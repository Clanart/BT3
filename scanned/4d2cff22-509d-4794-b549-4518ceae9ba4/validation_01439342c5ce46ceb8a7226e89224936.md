### Title
Missing Bounds Check in `decodeSwapDeltas` Silently Returns Zero Amounts, Enabling Bad-Price Execution - (File: metric-periphery/contracts/libraries/MetricOmmSwapQuoteDecode.sol)

### Summary

`MetricOmmSwapQuoteDecode.decodeSwapDeltas` reads two 32-byte words from a `bytes memory reason` buffer at fixed offsets without first verifying that the buffer is long enough to contain them. When the revert payload carries the correct 4-byte selector but fewer than 68 bytes of data, the assembly reads silently return zero, and the function returns `(0, 0, true)` — a "matched" result with both amounts zeroed — instead of returning `(0, 0, false)` or reverting.

### Finding Description

`decodeSwapDeltas` in `MetricOmmSwapQuoteDecode.sol`:

```solidity
function decodeSwapDeltas(bytes memory reason, bytes4 expectedSelector)
    internal pure
    returns (int128 amount0Delta, int128 amount1Delta, bool matched)
{
    if (bytes4(reason) != expectedSelector) return (0, 0, false);   // only 4-byte check

    int256 a0;
    int256 a1;
    assembly ("memory-safe") {
        a0 := mload(add(reason, 36))   // reads reason[4..35]  — first int256
        a1 := mload(add(reason, 68))   // reads reason[36..67] — second int256
    }
    amount0Delta = int128(a0);
    amount1Delta = int128(a1);
    return (amount0Delta, amount1Delta, true);          // matched = true
}
``` [1](#0-0) 

A valid ABI-encoded two-parameter error payload is `4 (selector) + 32 + 32 = 68 bytes`. The guard at line 13 only confirms the first 4 bytes match the expected selector. If `reason.length` is in the range `[4, 67]`, the selector check passes, but:

- `mload(add(reason, 36))` reads from `reason[36..67]` — partially or fully outside the allocated region.
- `mload(add(reason, 68))` reads from `reason[68..99]` — always outside the allocated region when `reason.length < 68`.

In the EVM, reads beyond the allocated memory region return zero. Therefore both `a0` and `a1` are zero, and the function returns `(0, 0, true)`.

By contrast, the analogous decode in `MetricOmmPoolLiquidityAdder._decodeLiquidityProbeOrBubble` correctly guards with an explicit length check before performing the same pattern of assembly reads:

```solidity
if (reason.length < 68) revert UnexpectedRevertLength(reason.length);
``` [2](#0-1) 

`decodeSwapDeltas` has no equivalent guard.

### Impact Explanation

`decodeSwapDeltas` is called in two quoter paths:

**Path 1 — `_quoteLiveSwap`:** [3](#0-2) 

**Path 2 — `_quoteHypotheticalSwap`:** [4](#0-3) 

In both paths, when `matched == true` the caller immediately returns the decoded amounts to the integrator. A zero `(amountIn, amountOut)` result propagates to `quoteLiveExactInSingle` / `quoteLiveExactIn` / `quoteLiveExactOut` and their hypothetical counterparts.

An integrator that trusts the quoter will:
- For exact-input: set `amountOutMinimum = 0` (or near-zero), allowing the pool to deliver zero output tokens while consuming the full input — complete loss of the input principal.
- For exact-output: set `amountInMaximum` based on a zero `amountIn` quote, causing the real swap to revert or, if the integrator adds a large buffer, to overpay.

The quoter does not validate pool addresses against the factory: [5](#0-4) 

Any caller can supply an arbitrary `pool` address. A pool that reverts with `abi.encodeWithSelector(QuoteSwapResult.selector)` (4 bytes, no parameters) passes the selector check and triggers the out-of-bounds reads, returning `(0, 0, true)`.

### Likelihood Explanation

- The quoter accepts any pool address without factory validation, so the trigger is fully unprivileged.
- Deploying a contract that reverts with a 4-byte payload matching `QuoteSwapResult.selector` requires no special access.
- Automated integrators (DEX aggregators, routing engines) that call the quoter before routing real swaps are the realistic victim class; they may not independently validate pool addresses.
- The `_decodeLiquidityProbeOrBubble` sibling already has the correct guard, confirming the omission in `decodeSwapDeltas` is an oversight rather than a design choice.

### Recommendation

Add a length guard immediately after the selector check, mirroring the pattern already used in `_decodeLiquidityProbeOrBubble`:

```solidity
function decodeSwapDeltas(bytes memory reason, bytes4 expectedSelector)
    internal pure
    returns (int128 amount0Delta, int128 amount1Delta, bool matched)
{
    if (bytes4(reason) != expectedSelector) return (0, 0, false);
+   if (reason.length < 68) return (0, 0, false);   // selector(4) + int256(32) + int256(32)

    int256 a0;
    int256 a1;
    assembly ("memory-safe") {
        a0 := mload(add(reason, 36));
        a1 := mload(add(reason, 68));
    }
    amount0Delta = int128(a0);
    amount1Delta = int128(a1);
    return (amount0Delta, amount1Delta, true);
}
```

### Proof of Concept

1. Deploy `MaliciousPool` whose `swap()` function executes:
   ```solidity
   bytes4 sel = MetricOmmSwapQuoter.QuoteSwapResult.selector;
   assembly { mstore(0, sel) revert(0, 4) }   // 4-byte payload only
   ```
2. Call `quoter.quoteLiveExactInSingle(address(maliciousPool), true, 1e18, 0)`.
3. Inside `_quoteLiveSwap`, the `try` block catches the 4-byte revert.
4. `decodeSwapDeltas(reason, QuoteSwapResult.selector)`:
   - `bytes4(reason) == expectedSelector` → passes the guard.
   - `reason.length == 4 < 68` → no length guard exists.
   - `a0 = mload(reason + 36)` → reads beyond allocated memory → `0`.
   - `a1 = mload(reason + 68)` → reads beyond allocated memory → `0`.
   - Returns `(0, 0, true)`.
5. `quoteLiveExactInSingle` returns `(amountIn = 0, amountOut = 0)`.
6. Integrator sets `amountOutMinimum = 0` and executes the real swap → receives zero output tokens, losing the full input amount.
