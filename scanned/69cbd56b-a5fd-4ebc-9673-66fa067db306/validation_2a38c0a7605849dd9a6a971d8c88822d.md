### Title
`sweepToken` / `unwrapWETH9` / `refundETH` carry no caller restriction, allowing any unprivileged address to drain router-held user funds — (File: `metric-periphery/contracts/base/PeripheryPayments.sol`)

---

### Summary

`PeripheryPayments.sweepToken`, `unwrapWETH9`, and `refundETH` are all externally callable with zero access control and accept an attacker-controlled `recipient`. Tokens accumulate in the router during normal exact-output WETH swaps (excess ETH is not auto-refunded). Any address can immediately drain that balance.

---

### Finding Description

All three settlement helpers are unrestricted public entry points:

```solidity
// metric-periphery/contracts/base/PeripheryPayments.sol
function sweepToken(address token, uint256 amountMinimum, address recipient)
    public payable override { ... }          // no caller check, arbitrary recipient

function unwrapWETH9(uint256 amountMinimum, address recipient)
    public payable override { ... }          // no caller check, arbitrary recipient

function refundETH() external payable override {
    uint256 balance = address(this).balance;
    if (balance > 0) { _transferETH(msg.sender, balance); }  // sends to any caller
}
``` [1](#0-0) 

The `pay` helper, called from the swap callback, wraps **exactly** the required `value` of native ETH and forwards it to the pool. Any surplus `msg.value` is left as raw ETH in the router:

```solidity

`exactOutputSingle` is `payable` (confirmed by tests calling it with `{value: nativePart}`). When a user calls it directly — not inside a `multicall` — there is no automatic `refundETH()` step. The surplus ETH sits in the router until any third party calls `refundETH()` and claims it. [3](#0-2) 

The intended recovery pattern (include `refundETH()` as a second leg of a `multicall`) is shown in tests but is never enforced by the contract itself: [4](#0-3) 

The same surface applies to `sweepToken`: any ERC-20 balance held by the router (e.g., WETH output routed through the contract in a multi-hop flow with `recipient: address(router)`) can be swept to an arbitrary address by any caller. [5](#0-4)
