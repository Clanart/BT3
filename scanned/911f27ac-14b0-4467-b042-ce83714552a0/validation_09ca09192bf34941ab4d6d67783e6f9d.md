### Title
Router's `pay()` Consumes Stale Native ETH From Prior Transactions to Subsidize Any Subsequent WETH Swap — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

### Summary

`PeripheryPayments.pay()` uses `address(this).balance` — the router's **total** native ETH balance — to decide how much ETH to wrap and forward when `token == WETH`. It does not distinguish between ETH sent in the **current** transaction and ETH left over from a **prior** transaction. Any ETH stranded in the router (e.g., from a user who called a `payable` swap function with excess ETH and omitted `refundETH()`) is silently consumed to subsidize the next caller's WETH swap input.

### Finding Description

`pay()` contains three branches for WETH payments: [1](#0-0) 

```
nativeBalance >= value   → wrap router's ETH, transfer to pool; pull nothing from payer
nativeBalance > 0        → wrap router's ETH, transfer partial; pull remainder from payer
nativeBalance == 0       → pull full amount from payer
```

`nativeBalance` is `address(this).balance` — the router's **entire** ETH balance at that moment, not scoped to the current `msg.value`. [2](#0-1) 

The `receive()` guard only blocks plain ETH transfers (no calldata): [3](#0-2) 

But every swap entry point is `payable`: [4](#0-3) [5](#0-4) 

When ETH is sent as part of a function call (not a plain transfer), `receive()` is **not** invoked — the ETH is credited to the contract balance silently. So a user can call `exactInputSingle{value: 2 ether}(amountIn: 500)` and leave `2 ether - 500 wei` stranded in the router if they omit `refundETH()`.

**Attack path:**

1. **User A** calls `exactInputSingle{value: 2 ether}(tokenIn=WETH, amountIn=500)` without `refundETH()`. The swap uses 500 wei of ETH; `~2 ether` remains in the router.
2. **User B** calls `exactInputSingle(tokenIn=WETH, amountIn=1000)` with no ETH sent.
3. Inside `_justPayCallback`, `pay(WETH, userB, pool, 1000)` is called: [6](#0-5) 

4. `nativeBalance = ~2 ether >= 1000`, so the router wraps 1000 wei of **User A's ETH** and transfers it to the pool. User B pays **zero** WETH from their own balance.

User A's ETH is consumed without their authorization to settle User B's swap.

### Impact Explanation

Direct loss of user principal. Any ETH left in the router (a realistic outcome for users who call payable swap functions with excess ETH and omit `refundETH()`) is permanently transferred to the pool on behalf of an unrelated caller. The victim loses the full stranded ETH amount; the attacker receives a free or discounted swap.

### Likelihood Explanation

- All four swap entry points (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`) are `payable`, so ETH can be sent with any of them.
- The `refundETH()` call is optional and must be composed manually by the user (typically via `multicall`). Omitting it is a realistic user error, especially for direct (non-multicall) invocations.
- An attacker can monitor the mempool or simply call a WETH swap after any block where the router's ETH balance is non-zero.

### Recommendation

Scope the native ETH available to `pay()` to only the ETH sent in the **current transaction**. The standard pattern is to pass `msg.value` (or a tracked per-call ETH budget) into `pay()` and decrement it as it is consumed, rather than reading `address(this).balance`. Alternatively, enforce that `address(this).balance` equals `msg.value` at the start of each top-level call and revert if it does not (i.e., reject calls when stale ETH is present).

```solidity
// Foundry fork test sketch
function test_staleEthSubsidizesSwap() public {
    // User A sends 2 ETH but only swaps 500 wei; forgets refundETH
    vm.prank(userA);
    router.exactInputSingle{value: 2 ether}(ExactInputSingleParams({
        pool: wethPool, tokenIn: weth, tokenOut: token1,
        zeroForOne: true, amountIn: 500, amountOutMinimum: 0,
        recipient: userA, deadline: block.timestamp + 1,
        priceLimitX64: 0, extensionData: ""
    }));
    // ~2 ether stranded in router
    assertGt(address(router).balance, 0);

    uint256 userBWethBefore = IERC20(weth).balanceOf(userB);

    // User B swaps 1000 WETH, paying nothing from their own balance
    vm.prank(userB);
    router.exactInputSingle(ExactInputSingleParams({
        pool: wethPool, tokenIn: weth, tokenOut: token1,
        zeroForOne: true, amountIn: 1000, amountOutMinimum: 0,
        recipient: userB, deadline: block.timestamp + 1,
        priceLimitX64: 0, extensionData: ""
    }));

    // Assert: User B's WETH balance unchanged (router's ETH paid for them)
    assertEq(IERC20(weth).balanceOf(userB), userBWethBefore);
    // Assert: User A's ETH is gone from the router (consumed for User B)
    assertEq(address(router).balance, 2 ether - 500 - 1000); // 1000 consumed for B
}
```
