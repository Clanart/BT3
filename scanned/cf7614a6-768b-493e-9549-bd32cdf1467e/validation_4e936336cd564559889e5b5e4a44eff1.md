### Title
`AnchoredPriceProvider` Uses L1-Style Future-Timestamp Rejection Without L2 Tolerance, Causing Swap DoS on L2 Deployments — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider` is described as "the one standard provider for public pools." Its `_isStale()` function unconditionally rejects any oracle `refTime` that is even one second ahead of `block.timestamp`. The codebase already recognises this is wrong on L2 — it ships `PriceProviderL2` and `ProtectedPriceProviderL2` with a configurable `FUTURE_TOLERANCE` — but `AnchoredPriceProvider` has no L2 counterpart. When `AnchoredProviderFactory` (whose deploy script sits in the root `script/` directory, not under `script/l1/`) is used on an L2 chain, every swap on every pool backed by this provider will revert with `FeedStalled` whenever the Pyth/Chainlink oracle's `refTime` is even one second ahead of the sequencer's `block.timestamp`.

---

### Finding Description

`PriceProvider.sol` and `AnchoredPriceProvider.sol` share the same L1-only staleness helper:

```solidity
/// @dev Pure staleness check (L1). Any future refTime is stale.
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta)
    internal pure returns (bool)
{
    if (refTime == 0) return true;
    if (refTime > nowTs) return true;          // ← hard reject
    return (nowTs - refTime) > maxDelta;
}
``` [1](#0-0) 

The L2-aware siblings fix this with a `FUTURE_TOLERANCE` window:

```solidity
if (refTime > nowTs) {
    return (refTime - nowTs) > futureTol;   // tolerate small skew
}
``` [2](#0-1) 

`AnchoredPriceProvider` has no such window. When `_isStale` returns `true`, `_readLeg` propagates `ok = false`:

```solidity
if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);
``` [3](#0-2) 

`_getBidAndAskPrice` then returns the sentinel `(0, type(uint128).max)`: [4](#0-3) 

And `getBidAndAskPrice` reverts:

```solidity
if (bid == 0 || ask == type(uint128).max) revert FeedStalled();
``` [5](#0-4) 

The pool calls `getBidAndAskPrice()` on every swap, so every swap reverts.

The deployment asymmetry is the root cause: `PriceProvider`/`ProtectedPriceProvider` each have an explicit L2 variant with `FUTURE_TOLERANCE`, but `AnchoredPriceProvider` does not, and its factory deploy script (`DeployAnchorFactory.s.sol`) lives in the root `script/` directory — not under `script/l1/` — making cross-chain deployment the expected path. [6](#0-5) [7](#0-6) 

---

### Impact Explanation

On Arbitrum and similar L2s, the sequencer's internal clock can run up to 1 hour ahead of the last L1-anchored timestamp. Pyth price updates published with a `refTime` even 1 second ahead of the current L2 `block.timestamp` will be unconditionally rejected. Because `AnchoredPriceProvider` is "the one standard provider for public pools," this makes every swap on every pool using it revert with `FeedStalled` for the duration of the skew — a complete, persistent DoS of core swap functionality. LPs cannot rebalance; traders cannot execute; protocol fees are not collected.

---

### Likelihood Explanation

Arbitrum's sequencer clock skew is a documented, routine occurrence (up to 1 hour ahead per Arbitrum docs). Pyth pushes price updates continuously; any update whose `refTime` exceeds the L2 `block.timestamp` by even 1 second triggers the bug. No attacker action is required — normal oracle operation on a live L2 deployment is sufficient.

---

### Recommendation

Add an `AnchoredPriceProviderL2` variant (or add a `FUTURE_TOLERANCE` immutable to `AnchoredPriceProvider`) and update `_isStale` to mirror the L2-aware logic already present in `PriceProviderL2`:

```solidity
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta, uint256 futureTol)
    internal pure returns (bool)
{
    if (refTime == 0) return true;
    if (refTime > nowTs) return (refTime - nowTs) > futureTol;
    return (nowTs - refTime) > maxDelta;
}
```

Cap `futureTol` at construction (e.g., `≤ 1 hours`) as `PriceProviderL2` does, and enforce it in the constructor:

```solidity
if (_futureTolerance > 1 hours) revert FutureToleranceOutOfBounds();
``` [8](#0-7)

### Proof of Concept

1. Deploy `AnchoredProviderFactory` on Arbitrum (as the root-level `DeployAnchorFactory.s.sol` script supports).
2. Create a pool backed by the resulting `AnchoredPriceProvider`.
3. Pyth publishes a price update with `refTime = block.timestamp + 2` (routine on Arbitrum when the sequencer clock is slightly ahead).
4. Any user calls `exactInputSingle` through `MetricOmmSimpleRouter`.
5. The pool calls `priceProvider.getBidAndAskPrice()`.
6. `_readLeg` calls `_isStale(refTime=T+2, nowTs=T, MAX_REF_STALENESS)`.
7. `refTime > nowTs` → `_isStale` returns `true` → `ok = false` → sentinel `(0, max)` returned.
8. `getBidAndAskPrice` reverts with `FeedStalled`.
9. The swap reverts. All swaps on this pool are blocked for the duration of the skew. [9](#0-8) [5](#0-4)
