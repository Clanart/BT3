### Title
Missing Class Existence Check in `execute_replace_class` Allows Replacing Contract Class with Undeclared Hash — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the new class hash provided by the calling contract has been previously declared. This is a state-transition bypass: the OS accepts a class replacement without the prerequisite state (a declared class) being satisfied. A malicious contract can replace its own class hash with an arbitrary, undeclared value, permanently rendering the contract unexecutable and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall: [1](#0-0) 

The function reads `request.class_hash` directly from the syscall request (attacker-controlled) and updates the contract's state entry with the new class hash: [2](#0-1) 

A TODO comment at line 898 explicitly acknowledges the missing check:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
``` [3](#0-2) 

The correct prerequisite state is that a class must first be declared via `execute_declare_transaction`, which writes the class hash into `contract_class_changes` with `prev_value=0`: [4](#0-3) 

`execute_replace_class` performs no analogous lookup into `contract_class_changes` to confirm the target class hash exists. The OS will accept any felt value as the new class hash and commit it to the state diff.

---

### Impact Explanation

**Permanent freezing of funds.**

Once a contract's class hash in the on-chain state is set to an undeclared value:

1. Any future call to that contract requires the OS to locate the compiled class for that hash in the `compiled_class_facts_bundle`.
2. Because the hash was never declared, no compiled class exists for it.
3. The OS cannot execute the contract; the sequencer cannot include calls to it in any future block.
4. The contract is permanently unexecutable.
5. Any ERC-20 tokens, ETH, or other assets held in the contract's storage are permanently inaccessible — there is no recovery path, since `replace_class` itself cannot be called again on a non-executable contract.

This satisfies the **Critical — Permanent freezing of funds** impact category.

---

### Likelihood Explanation

The attack path is fully reachable by any unprivileged user:

- No privileged role, leaked key, or sequencer cooperation is required.
- Any user can deploy a contract (via `deploy_account` + `deploy` syscall or `execute_deploy_account_transaction`).
- The deployed contract can immediately call `replace_class` with an arbitrary felt as the new class hash.
- The OS will process this syscall, update the state, and include the change in the proven block output.
- The attack is deterministic and requires only a single transaction after deployment.

The only gateway-level defense would be off-chain sequencer filtering, but the OS proof itself does not enforce the invariant, meaning a proof containing this state transition is valid and will be accepted on-chain.

---

### Recommendation

Before updating the contract state entry in `execute_replace_class`, verify that the requested `class_hash` exists in `contract_class_changes` (i.e., has been declared in the current or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` for `request.class_hash` and assert the returned value is non-zero (a valid compiled class hash), mirroring the `prev_value=0` enforcement used in `execute_declare_transaction`.

---

### Proof of Concept

1. **Attacker deploys a contract** (`ContractA`) that contains the following logic in one of its entry points:
   ```
   replace_class(class_hash=0xdeadbeef)  // arbitrary undeclared hash
   ```

2. **Attacker calls that entry point.** The OS processes the `replace_class` syscall via `execute_replace_class`:
   - `request.class_hash = 0xdeadbeef`
   - No check against `contract_class_changes` is performed.
   - `dict_update` writes `StateEntry(class_hash=0xdeadbeef, ...)` for `ContractA`.

3. **Block is proven.** The state diff is committed on-chain. `ContractA`'s class hash is now `0xdeadbeef`.

4. **Any future call to `ContractA`** requires the OS to find compiled class `0xdeadbeef`. It does not exist in any `compiled_class_facts_bundle`. The OS cannot execute `ContractA`. The sequencer cannot include any transaction targeting `ContractA`.

5. **Funds held by `ContractA` are permanently frozen.** The `replace_class` syscall cannot be called again to recover, since the contract is unexecutable.

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
