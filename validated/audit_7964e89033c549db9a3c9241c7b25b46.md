### Title
Non-EVM Bank Balances (IBC/CosmWasm Tokens) Orphaned After SELFDESTRUCT — (`x/evm/statedb/statedb.go`, `x/evm/keeper/statedb.go`)

### Summary
When a contract selfdestructs, `StateDB.Commit()` burns only the EVM-denom balance and removes the auth account. Any non-EVM-denom bank balances (IBC tokens, CosmWasm bridge tokens, etc.) held by the contract address are left untouched in the bank module. The auth account is gone, but the bank records remain at the address. If the address is later recreated via CREATE2, the new contract owner inherits those orphaned balances — constituting unauthorized access to Cosmos bank funds.

### Finding Description

**`StateDB.Commit()` — `x/evm/statedb/statedb.go` lines 800–825:**

The selfdestruct branch explicitly limits its burn to `s.evmDenom` only:

```go

// bridge) held by the destroyed address are not drained and may remain as
// orphaned bank balances.

    if _, err := s.keeper.SubBalance(cacheCtx, cosmosAddr, coin); err != nil { ... }
}
if err := s.keeper.DeleteAccount(cacheCtx, obj.Address()); err != nil { ... }
writeCache()
``` [1](#0-0) 

No loop over all bank denoms is performed. Only `s.evmDenom` is subtracted.

**`Keeper.DeleteAccount()` — `x/evm/keeper/statedb.go` lines 195–222:**

`DeleteAccount` clears EVM storage and removes the auth account record. It never calls `bankKeeper.SendCoins`, `bankKeeper.BurnCoins`, or any balance-clearing operation for non-EVM denoms. [2](#0-1) 

In Cosmos SDK, bank balances are stored by address independently of the auth account. Removing the auth account via `accountKeeper.RemoveAccount()` does not remove bank module balance records. [3](#0-2) 

### Impact Explanation

After `Commit()`:
- `eth_getBalance` returns 0 (EVM denom burned).
- `accountKeeper.GetAccount()` returns nil (auth account deleted).
- `bankKeeper.GetAllBalances()` returns non-zero IBC/CosmWasm token balances at the address.

The invariant "a selfdestructed account has zero total bank balance" is violated. The orphaned tokens are recoverable: if the contract was deployed via CREATE2, an attacker who knows the factory address, salt, and initcode can redeploy at the identical address. The new auth account at that address inherits the orphaned bank balances, giving the redeployer unauthorized access to Cosmos bank funds they did not own.

### Likelihood Explanation

The preconditions are reachable without privilege:
1. Pre-fund a deterministic CREATE2 address with IBC tokens (valid Cosmos bank send to any address).
2. Deploy and selfdestruct the contract in one transaction (satisfies EIP-6780).
3. IBC tokens remain orphaned.
4. Redeploy at the same address; new contract controls the orphaned tokens.

No validator collusion, governance action, or key compromise is required. The entire path is executable via standard transaction submission.

### Recommendation

In `StateDB.Commit()`, before calling `DeleteAccount`, iterate over **all** bank module balances at the address (not just `evmDenom`) and burn or escrow each one. The Cosmos SDK `bankKeeper.GetAllBalances(ctx, cosmosAddr)` call provides the full set of coins to drain.

### Proof of Concept

```
1. Compute CREATE2 address X for (factory, salt, initcode).
2. MsgSend: send 1000 uatom (IBC denom) to address X.
3. EVM tx: factory.deploy(salt, initcode) → contract at X calls SELFDESTRUCT.
   - StateDB.SelfDestruct(X): burns evmDenom balance, marks selfDestructed.
   - StateDB.Commit(): burns remaining evmDenom, calls DeleteAccount(X).
   - bankKeeper still holds 1000 uatom at X.
4. Assert: eth_getBalance(X) == 0 ✓, bankKeeper.GetAllBalances(X) == [1000uatom] ✓ (divergence).
5. EVM tx: factory.deploy(salt, initcode) → redeploy at X.
   - New auth account created at X.
   - bankKeeper.GetAllBalances(X) still shows 1000 uatom, now owned by new contract.
6. New contract calls IBC transfer or native action to drain 1000 uatom to attacker.

func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
	cosmosAddr := sdk.AccAddress(addr.Bytes())
	acct := k.accountKeeper.GetAccount(ctx, cosmosAddr)
	if acct == nil {
		return nil
	}

	// NOTE: only Ethereum accounts (contracts) can be selfdestructed
	_, ok := acct.(ethermint.EthAccountI)
	if !ok {
		return errorsmod.Wrapf(types.ErrInvalidAccount, "type %T, address %s", acct, addr)
	}

	// clear storage
	k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool {
		k.SetState(ctx, addr, key, nil)
		return true
	})

	// remove auth account
	k.accountKeeper.RemoveAccount(ctx, acct)

	k.debugLog(ctx, "account suicided",
		"ethereum-address", addr,
		"cosmos-address", cosmosAddr,
	)

	return nil
