### Title
Irreversible Eviction of Valid Transactions via Oversized Submission When Mempool Is Full — (`File: crates/apollo_mempool/src/mempool.rs`)

### Summary

`try_make_space` permanently evicts nonce-gapped transactions from `accounts_with_gap` even when the transaction that triggered the eviction is ultimately rejected because insufficient space was freed. An unprivileged attacker with a fresh account can repeatedly submit oversized-but-valid transactions (large signature, up to `max_signature_length = 4000` felts) to continuously drain legitimate pending transactions from the mempool at zero on-chain cost.

### Finding Description

The eviction path in `add_tx_validations` is:

```
add_tx_validations
  └─ exceeds_capacity(tx)?  → true
       └─ handle_capacity_overflow(tx, account_nonce)
            ├─ creating_gap = (account_has_gap || !account_has_txs) && !closing_gap
            ├─ closing_gap  = tx.nonce() == account_nonce
            └─ if !creating_gap → try_make_space(tx.total_bytes())
                                     evicts from accounts_with_gap (irreversibly)
                                     returns false if freed < required
                 → Err(MempoolError::MempoolFull)   ← attacker tx rejected
                   but evicted txs are already gone
``` [1](#0-0) 

For a fresh attacker account (`account_has_txs = false`, `account_has_gap = false`) submitting a transaction with `nonce == account_nonce` (the normal case, `closing_gap = true`):

```
creating_gap = (false || true) && !true = false
```

So `!creating_gap = true` and `try_make_space` is called unconditionally. [2](#0-1) 

Inside `try_make_space`, transactions are removed from the pool and the eviction metric is incremented. There is no rollback path: [3](#0-2) 

The codebase itself documents this as a known behavior in the test `rejects_or_accepts_tx_based_on_freed_space`:

> "Transactions tx1 and tx2 are evicted regardless of whether large_tx is accepted or rejected. We do not revert the eviction attempt even if adding large_tx ultimately fails." [4](#0-3) 

The gateway's stateless validator enforces `max_signature_length = 4000` felts (≈128 KB per transaction). [5](#0-4) 

An attacker can craft a transaction whose `total_bytes()` exceeds the combined byte size of all evictable accounts' transactions, ensuring that `try_make_space` returns `false` (not enough space freed) while still having permanently removed those transactions.

### Impact Explanation

Valid transactions from legitimate users that happen to be nonce-gapped (e.g., submitted nonce 1 before nonce 0 arrived) are permanently removed from the mempool without ever being sequenced. The attacker's transaction is rejected and never reaches a block, so no fee is charged on-chain. The sequencer bears the cost of stateful validation. This directly matches:

> **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

The `size_in_bytes()` used for capacity also includes `delayed_declares.size_in_bytes()`, but `try_make_space` only evicts from `accounts_with_gap` (which tracks only `tx_pool` entries, not `delayed_declares`). This means delayed-declare bytes count against capacity but are never evictable, further tightening the window in which the attack succeeds. [6](#0-5) [7](#0-6) 

### Likelihood Explanation

- **Trigger is unprivileged**: any account with `nonce == account_nonce` (the default for a fresh account) qualifies.
- **No on-chain cost**: the attacker's transaction is rejected at the mempool layer; no fee is deducted.
- **Repeatable**: the attacker can submit the oversized transaction in a tight loop.
- **Precondition**: the mempool must be at or near `capacity_in_bytes` (1 GiB in production config) and contain at least one `accounts_with_gap` entry. Both conditions are easily met in a live network. [8](#0-7) 

### Recommendation

1. **Atomic eviction**: record which transactions were evicted inside `try_make_space` and re-insert them if `handle_capacity_overflow` ultimately returns `Err`. Alternatively, perform a dry-run size check before committing any removal.
2. **Fee-weighted eviction**: instead of evicting only `accounts_with_gap`, also allow eviction of the lowest-tip gap-free transaction across all accounts when the incoming transaction has a higher tip, analogous to Geth's cross-address eviction.
3. **Reject oversized transactions early**: if `tx.total_bytes() > capacity_in_bytes`, reject immediately without attempting eviction.

### Proof of Concept

The existing test `rejects_or_accepts_tx_based_on_freed_space` (case `insufficient_eviction_space`) already demonstrates the exact bug:

```rust
// tx1 (nonce 1, account_nonce 0 → gap) and tx2 (nonce 2) fill the mempool.
// large_tx (nonce 0, account_nonce 0 → closing_gap=true, creating_gap=false)
// has a 20-felt signature, larger than tx1+tx2 combined.
// Result: tx1 and tx2 are evicted, large_tx is rejected with MempoolError::MempoolFull.
assert!(!mempool.tx_pool.contains_account(contract_address!("0x1")));
``` [4](#0-3) 

In production, an attacker would:
1. Deploy a fresh account (or reuse any account at nonce 0).
2. Wait for the mempool to fill (or fill it with their own gap-free transactions from many addresses).
3. Submit an invoke transaction with a 4000-felt signature (`total_bytes ≈ 128 KB`).
4. The mempool evicts all `accounts_with_gap` entries it can find, then rejects the attacker's transaction.
5. Repeat from step 3 using a new `tx_hash` (to avoid `DuplicateTransaction`).

    fn update_accounts_with_gap(&mut self, address_to_nonce: AddressToNonce) {
        for (address, account_nonce) in address_to_nonce {
            // Assumption: Future declares are not allowed — their nonce must match the account
            // nonce, so they fill a gap if one exists.
            if self.delayed_declares.contains(address, account_nonce) {
                self.accounts_with_gap.swap_remove(&address);
                continue;
            }

            // Gap exists when lowest transaction nonce is higher than account nonce.
            let gap_exists = match self.tx_pool.get_lowest_nonce(address) {
                Some(lowest_nonce) => account_nonce < lowest_nonce,
                None => false, // No transactions for the account, so no gap.
            };

            // Update the eviction tracking set accordingly.
            if gap_exists {
                self.accounts_with_gap.insert(address);
            } else {
                self.accounts_with_gap.swap_remove(&address);
            }
        }
    }

    pub fn try_make_space(&mut self, required_space: u64) -> bool {
        let mut total_space_freed = 0;

        while total_space_freed < required_space {
            let Some(address) = self.get_evictable_account() else {
                return false;
            };

            let txs: Vec<_> = self.tx_pool.account_txs_sorted_by_nonce(address).copied().collect();
            for tx_ref in txs.iter().rev() {
                let tx = self
                    .tx_pool
                    .remove(tx_ref.tx_hash)
                    .expect("Transaction must exist in the pool.");
                total_space_freed += tx.total_bytes();
                metric_count_evicted_txs(1);
                if total_space_freed >= required_space {
                    break;
                }
            }

            // Clean up if account is now empty.
            if !self.tx_pool.contains_account(address) {
                self.accounts_with_gap.swap_remove(&address);
            }
        }

        true
    }

    fn handle_capacity_overflow(
        &mut self,
        tx: &InternalRpcTransaction,
        account_nonce: Nonce,
    ) -> Result<(), MempoolError> {
        let address = tx.contract_address();

        let account_has_gap = self.accounts_with_gap.contains(&address);
        let account_has_txs = self.tx_pool.contains_account(address);
        let closing_gap = tx.nonce() == account_nonce;
        let creating_gap = (account_has_gap || !account_has_txs) && !closing_gap;

        if !creating_gap && self.try_make_space(tx.total_bytes()) {
            return Ok(());
        }

        Err(MempoolError::MempoolFull)
    }
