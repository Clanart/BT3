### Title
Unbounded `req.Predecessors` Loop in `TraceTx` Enables Node Resource Exhaustion via Public gRPC/JSON-RPC — (File: `x/evm/keeper/grpc_query.go`)

---

### Summary

The `TraceTx` gRPC handler iterates over a caller-supplied `req.Predecessors` slice with no size limit. Each element triggers a full `ApplyMessageWithConfig` EVM execution. An unprivileged caller can submit an arbitrarily large predecessors list, causing unbounded CPU and memory consumption that can crash the node process and halt the chain.

---

### Finding Description

In `x/evm/keeper/grpc_query.go`, the `TraceTx` handler replays every entry in `req.Predecessors` before tracing the target transaction:

```go
for i, tx := range req.Predecessors {
    ethTx := tx.AsTransaction()
    msg, err := core.TransactionToMessage(ethTx, signer, cfg.BaseFee)
    ...
    rsp, err := k.ApplyMessageWithConfig(ctx, msg, cfg, true)
    ...
    cfg.TxConfig.LogIndex += uint(len(rsp.Logs))
}
``` [1](#0-0) 

There is no validation or cap on `len(req.Predecessors)` anywhere in `TraceTx` or in the proto-layer before it reaches this loop. Each iteration calls `k.ApplyMessageWithConfig` with `commitMessage=true`, which performs a complete EVM state transition — including storage reads/writes, contract execution, and log accumulation — against the shared `statedb`. [2](#0-1) 

By contrast, the `SimulateV1` handler (same file) explicitly enforces `rpctypes.MaxSimulateBlocks` and a cross-block gas budget before entering its loop: [3](#0-2) 

No equivalent guard exists for `TraceTx`.

---

### Impact Explanation

An attacker submits a `debug_traceTransaction` JSON-RPC call (which routes to `TraceTx` via the RPC backend) with thousands of predecessor entries, each being a complex contract call (e.g., a tight loop that consumes the full gas limit). The node must execute every predecessor sequentially before it can respond. This causes:

1. **Unbounded CPU consumption** — each `ApplyMessageWithConfig` call runs the EVM interpreter to gas exhaustion.
2. **Unbounded memory growth** — the `statedb` accumulates dirty state, logs, and journal entries for every predecessor.
3. **Node process crash (OOM)** — if the node is killed by the OS, it halts block production for that validator.

If multiple validators are targeted simultaneously, the chain halts. This matches the allowed High impact: *"Public JSON-RPC, gRPC, simulation, tracing, receipt/log, or indexer path … exposes a reachable route to [halting the chain]."*

---

### Likelihood Explanation

- `debug_traceTransaction` is a standard, publicly documented JSON-RPC endpoint enabled by default in Ethermint's JSON-RPC server.
- No authentication is required; any client with network access to the RPC port can send the request.
- The attack requires only a single malformed RPC call; no on-chain transaction or gas payment is needed.
- The predecessor array is a standard field in the `QueryTraceTxRequest` proto message with no server-side size enforcement.

---

### Recommendation

Add an explicit cap on `len(req.Predecessors)` at the top of `TraceTx`, analogous to the `MaxSimulateBlocks` guard in `SimulateV1`:

```go
const maxTracePredecessors = 256 // or a configurable param

if len(req.Predecessors) > maxTracePredecessors {
    return nil, status.Errorf(codes.InvalidArgument,
        "too many predecessors: %d > %d", len(req.Predecessors), maxTracePredecessors)
}
```

Additionally, apply the existing `queryMaxGasLimit` budget across the entire predecessor replay loop (not just per-call), so that the total gas consumed by all predecessors is bounded even when the count is within the cap.

---

### Proof of Concept

1. Deploy or reference any contract that consumes close to the block gas limit per call (e.g., a storage-heavy loop).
2. Construct a `debug_traceTransaction` JSON-RPC request where `predecessors` contains 10,000 entries, each being a valid signed `MsgEthereumTx` calling that contract.
3. Send the request to a running Ethermint node's JSON-RPC endpoint.
4. Observe the node process consuming all available memory and CPU; the process is killed by the OS OOM killer, halting block production.

func (k Keeper) TraceTx(c context.Context, req *types.QueryTraceTxRequest) (*types.QueryTraceTxResponse, error) {
	var baseFee *big.Int
	if req == nil || req.Msg == nil {
		return nil, status.Error(codes.InvalidArgument, "request and message cannot be empty")
	}
	if req.BaseFee != nil && !req.BaseFee.IsNil() {
		baseFee = req.BaseFee.BigInt()
	}
	resultData, err := execTrace(
		c,
		req,
		k,
		baseFee,
		func(ctx sdk.Context, cfg *EVMConfig, traceConfig *types.TraceConfig) (*core.Message, error) {
			signer := ethtypes.MakeSigner(cfg.ChainConfig, big.NewInt(ctx.BlockHeight()), uint64(ctx.BlockTime().Unix())) //#nosec G115
			tracer, err := newTacer(&logger.Config{}, cfg.ChainConfig, cfg.TxConfig, traceConfig)
			if err != nil {
				return nil, status.Error(codes.Internal, err.Error())
			}
			cfg.Tracer = tracer.Hooks
			cfg.DebugTrace = true
			cfg.TraceReplay = traceConfig.GetTraceReplay()
			for i, tx := range req.Predecessors {
				ethTx := tx.AsTransaction()
				msg, err := core.TransactionToMessage(ethTx, signer, cfg.BaseFee)
				if err != nil {
					k.Logger(ctx).Debug("trace: skipping predecessor, failed to convert tx to message",
						"index", i, "hash", ethTx.Hash().Hex(), "err", err.Error())
					continue
				}
				cfg.TxConfig.TxHash = ethTx.Hash()
				cfg.TxConfig.TxIndex, err = ethermint.SafeUint(i)
				if err != nil {
					k.Logger(ctx).Debug("trace: skipping predecessor, invalid tx index",
						"index", i, "hash", ethTx.Hash().Hex(), "err", err.Error())
					continue
				}
				rsp, err := k.ApplyMessageWithConfig(ctx, msg, cfg, true)
				if err != nil {
					k.Logger(ctx).Error("trace: predecessor replay failed, trace state may be incomplete",
						"index", i, "hash", ethTx.Hash().Hex(), "err", err.Error())
					continue
				}
				cfg.TxConfig.LogIndex += uint(len(rsp.Logs))
			}

			tx := req.Msg.AsTransaction()
			cfg.TxConfig.TxHash = tx.Hash()
			if len(req.Predecessors) > 0 {
				cfg.TxConfig.TxIndex++
			}

			return core.TransactionToMessage(tx, signer, cfg.BaseFee)
		},
	)
	if err != nil {
		return nil, err
	}

	return &types.QueryTraceTxResponse{
		Data: resultData,
	}, nil
}
