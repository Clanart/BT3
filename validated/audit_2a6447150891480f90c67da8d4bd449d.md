### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the replacement class hash is actually declared before writing it into the contract's state entry. A malicious contract deployer can call `replace_class` with an arbitrary, undeclared class hash, permanently rendering the contract unexecutable and freezing any funds held in its storage.

---

### Finding Description

In `execute_replace_class`, the OS reads the `ReplaceClassRequest`, reduces gas, fetches the current `StateEntry` for the calling contract, and immediately writes a new `StateEntry` with the attacker-supplied `class_hash`:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The in-code `TODO` comment at line 898 explicitly acknowledges the missing check. There is no lookup into `contract_class_changes` to confirm that `class_hash` was ever declared. The OS Cairo program is the authoritative enforcement layer; if it accepts this state transition, the resulting proof is valid and the state is committed to L1 without any further semantic check.

Compare this to other state-mutating syscalls such as `execute_storage_write`, which correctly validates the storage key/value pair before writing, or `deploy_contract`, which asserts `state_entry.class_hash = UNINITIALIZED_CLASS_HASH` before proceeding. [2](#0-1) 

The `execute_replace_class` function has no equivalent guard. [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` field in `contract_state_changes` is set to an undeclared hash:

1. Every subsequent call to that contract causes the OS to look up the class in `contract_class_changes` and fail to find it, making the contract permanently unexecutable.
2. Because the contract cannot execute, it cannot call `replace_class` again to self-recover.
3. All ERC-20 balances, ETH, or other assets stored in the contract's storage segment are permanently inaccessible — no withdrawal, no transfer, no recovery path exists at the protocol level.
4. The resulting state is committed to L1 via a valid STARK proof, so the freeze is irreversible on-chain.

---

### Likelihood Explanation

The attack path requires only the ability to deploy a contract and call a function — both are available to any unprivileged user on StarkNet:

- The `replace_class` syscall is callable from any contract entry point with no additional privilege check beyond being the currently-executing contract.
- The OS imposes no restriction on the value of `class_hash` supplied to the syscall.
- A malicious contract author can embed a backdoor function that calls `replace_class(undeclared_hash)` after accumulating user deposits.
- The sequencer will include the transaction (it is a syntactically valid invoke), and the OS will produce a valid proof for the resulting state.

---

### Recommendation

Before writing the new `StateEntry`, add a lookup into `contract_class_changes` to assert that the supplied `class_hash` is declared:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the pattern used in `execute_declare_transaction`, where `prev_value=0` enforces that a class may only be declared once, and `assert_not_zero(compiled_class_hash)` ensures the compiled hash is valid. [4](#0-3) 

---

### Proof of Concept

1. **Attacker deploys** `MaliciousVault` with a legitimate declared class hash `A`. The contract exposes `deposit()` and a hidden `rug()` function.
2. **Users call** `deposit()`, transferring tokens into `MaliciousVault`'s storage.
3. **Attacker calls** `rug()`, which internally issues the syscall `replace_class(0xdeadbeef…)` where `0xdeadbeef…` is a felt that has never been declared in `contract_class_changes`.
4. **OS executes** `execute_replace_class`: gas is deducted, `GetContractAddressStateEntry` hint fetches the current state, and `dict_update` writes `new StateEntry(class_hash=0xdeadbeef…, …)` — **no declared-class check is performed**.
5. **Block is proven**: the STARK proof is valid; L1 accepts the state root update.
6. **All subsequent calls** to `MaliciousVault` fail at class lookup. Users' funds are permanently frozen with no recovery path. [3](#0-2)

        // Not enough gas to execute the syscall.
