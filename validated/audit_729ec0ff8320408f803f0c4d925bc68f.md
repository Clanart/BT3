### Title
Integer Truncation in `NewDynamicFeeChecker` Produces Zero Effective Fee for Cosmos SDK Transactions When `baseFee = 0` — (`ante/evm/fee_checker.go`)

### Summary

`NewDynamicFeeChecker` computes `feeCap` via integer division (`fee.Quo(gas)`). When the total fee amount is smaller than the gas limit — a common configuration — the quotient truncates to zero. If `baseFee` has reached zero (reachable by default because `MinGasPrice` defaults to 0), the effective fee returned to `DeductFeeDecorator` is also zero, allowing any Cosmos SDK transaction to execute without paying fees.

### Finding Description

In `ante/evm/fee_checker.go`, `NewDynamicFeeChecker` computes the per-gas fee cap using integer division:

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))   // line 83
```

`fee` is the total fee amount in the EVM denom (e.g., `1 aevmos`); `gas` is the gas limit (e.g., `21000`). Integer division truncates `1 / 21000 = 0`.

The guard that follows does not catch this:

```go
if feeCap.LT(baseFeeInt) {   // line 86
    return nil, 0, errorsmod.Wrapf(...)
}
```

When `baseFee = 0`, `feeCap (0) >= baseFeeInt (0)` passes silently.

The effective price is then:

```go
effectivePrice := sdkmath.NewIntFromBigInt(
    types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
// EffectiveGasPrice(0, 0, maxPriorityPrice) = min(0 + maxPriorityPrice, 0) = 0
```

And the effective fee returned to `DeductFeeDecorator`:

```go
effectiveFee := sdk.Coins{{Denom: denom, Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas))}}
// 0 * gas = 0
```

`DeductFeeDecorator` deducts exactly zero from the sender; the fee collector receives nothing.

The `baseFee = 0` state is reachable because `MinGasPrice` defaults to zero:

```go
// x/feemarket/types/params.go
DefaultMinGasPrice = sdkmath.LegacyZeroDec()   // line ~31
```

and `CalculateBaseFee` floors the base fee at `MinGasPrice`:

```go
minGasPrice := params.MinGasPrice.TruncateInt().BigInt()
return ethermint.BigMax(x.Sub(parentBaseFee, baseFeeDelta), minGasPrice)
// x/feemarket/keeper/eip1559.go:103-104
```

With `MinGasPrice = 0`, the base fee can decrease all the way to zero across consecutive low-activity blocks.

### Impact Explanation

Any Cosmos SDK transaction (e.g., `MsgSend`, governance votes, IBC transfers) submitted with a fee amount smaller than its gas limit executes with zero effective fee when `baseFee = 0`. The fee collector module receives no compensation. This is a direct fee mis-accounting bug in the ante handler path, matching the allowed High impact: *"ante handler bug that permits valid user funds/fees to be mis-accounted."*

### Likelihood Explanation

- `MinGasPrice = 0` is the **default** parameter value, so most deployments are affected.
- The base fee decreases by `1/BaseFeeChangeDenominator` (default `1/8`) per block when blocks are under the gas target. On a low-activity chain, `baseFee` reaches zero after a bounded number of empty blocks.
- The condition `fee < gas` is trivially satisfied for any transaction that sets a small absolute fee (e.g., `fee = 1 aevmos`, `gas = 21000`), which is a normal user configuration.
- No privileged role is required; any unprivileged user can submit such a transaction.

### Recommendation

Replace integer division with decimal arithmetic to preserve sub-unit precision:

```go
// Before (truncates to 0 when fee < gas):
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))

// After (preserves fractional gas price):
feeCapDec := sdkmath.LegacyNewDecFromInt(fee).QuoInt64(int64(gas))
feeCap := feeCapDec.TruncateInt()
```

Additionally, add an explicit guard when `baseFee = 0` to prevent zero-fee transactions from being accepted:

```go
if baseFeeInt.IsZero() && feeCap.IsZero() && !fee.IsZero() {
    return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee,
        "fee too small to cover gas: fee=%s gas=%d", fee, gas)
}
```

### Proof of Concept

1. Deploy Ethermint with default parameters (`MinGasPrice = 0`, `BaseFee = 1_000_000_000`).
2. Produce consecutive empty blocks until `baseFee` decreases to 0 (or set `MinGasPrice = 0` and wait).
3. Submit a Cosmos SDK `MsgSend` with `fee = 1 aevmos` and `gas = 21000`.
4. `NewDynamicFeeChecker` computes `feeCap = 1 / 21000 = 0` (integer truncation).
5. `baseFeeInt = 0`; the guard `feeCap.LT(baseFeeInt)` is `false` — no rejection.
6. `EffectiveGasPrice(0, 0, MaxInt64) = min(MaxInt64, 0) = 0`.
7. `effectiveFee = 0 * 21000 = 0 aevmos`.
8. `DeductFeeDecorator` deducts **0** from the sender; the fee collector receives **0**.
9. The transaction is included in a block and executes successfully at zero cost. [1](#0-0) [2](#0-1) [3](#0-2)
