### Title
Gas Refund for Rolled-Back SSTORE Operations Applied to Parent Context After `PostTxProcessing` Hook Failure — (`x/evm/keeper/state_transition.go`)

### Summary

`ApplyTransaction` calls `RefundGas` on the **parent** `ctx` unconditionally at line 252, using a `leftoverGas` value that already embeds the EVM's SSTORE gas refund (computed inside `ApplyMessageWithConfig`). When a `PostTxProcessing` hook fails, `tmpCtx` is abandoned (EVM state rolled back), but the SSTORE-inflated refund is still credited to the sender on `ctx`. The sender pays less than the actual gas consumed for a transaction whose state changes were entirely discarded.

### Finding Description

**Step 1 — Refund baked into `res.GasUsed` inside `ApplyMessageWithConfig`:** [1](#0-0) 

```
temporaryGasUsed := msg.GasLimit - leftoverGas          // raw EVM gas used
refund := GasToRefund(stateDB.GetRefund(), temporaryGasUsed, refundQuotient)
leftoverGas += refund                                    // refund folded in
```

After line 556 (`temporaryGasUsed = msg.GasLimit - leftoverGas`) and line 587 (`gasUsed = max(minimumGasUsed, temporaryGasUsed)`), `res.GasUsed` is already **reduced** by the SSTORE refund. `leftoverGas` returned to `ApplyTransaction` therefore **includes** the SSTORE refund. [2](#0-1) 

**Step 2 — Hook failure path in `ApplyTransaction`:** [3](#0-2) 

When `PostTxProcessing` returns an error, `commit()` is **not** called — `tmpCtx` is abandoned and all EVM state changes (including the SSTORE writes that generated the refund) are discarded.

**Step 3 — `RefundGas` called on `ctx` unconditionally:** [4](#0-3) 

```go
leftoverGas := msg.GasLimit - res.GasUsed   // includes SSTORE refund
if err = k.RefundGas(ctx, msg, leftoverGas, cfg.Params.EvmDenom); err != nil {
```

`RefundGas` is called on `ctx` (the **parent** context, not `tmpCtx`) regardless of whether the hook succeeded or failed. It sends `leftoverGas × gasPrice` from the fee collector to the sender: [5](#0-4) 

**Result:** The sender's SSTORE writes are rolled back (no committed state change), yet the sender is credited the SSTORE gas refund — up to `gasUsed / 5` post-EIP-3529 — on the live chain state.

### Impact Explanation

The sender pays `res.GasUsed × gasPrice` instead of `(res.GasUsed + refund) × gasPrice`. For a tx that uses the maximum EIP-3529 refund (20 % of gas used), the sender receives a ~20 % discount on gas fees for a transaction whose EVM effects were entirely discarded. This is a direct mis-accounting of user funds/fees: the fee collector module loses tokens it should have retained, and the sender gains tokens they are not entitled to.

### Likelihood Explanation

Exploitability requires a `PostTxProcessing` hook to be registered that can be made to return an error by a crafted transaction (e.g., by emitting a specific log pattern). The codebase itself ships a `FailureHook` test fixture confirming this is a designed scenario. [6](#0-5) 

Production chains built on Ethermint (e.g., Evmos) register ERC-20 hooks whose failure conditions can be triggered by emitting specific log signatures from a contract. An unprivileged user can deploy or call a contract that (a) performs SSTORE to accumulate a refund and (b) emits a log that causes the hook to fail.

### Recommendation

Move `RefundGas` inside the success branch (after `commit()`) **or** call it on `tmpCtx` so it is only applied when the hook succeeds and the cache context is committed. When the hook fails, `leftoverGas` passed to `RefundGas` should be recomputed without the SSTORE refund (i.e., using the raw `temporaryGasUsed` before `leftoverGas += refund`).

### Proof of Concept

1. Deploy a contract with bytecode that writes to storage (`SSTORE`) to accumulate a gas refund.
2. Register a `FailureHook` (or craft a log that triggers an existing hook to fail).
3. Submit the tx. Observe `res.VmError == ErrPostTxProcessing` and that the SSTORE write is absent from committed state.
4. Assert sender balance after = balance before − `res.GasUsed × gasPrice`, where `res.GasUsed` is already reduced by the SSTORE refund — i.e., the sender paid less than the raw EVM gas consumed, for a tx with no committed state changes. [7](#0-6) [1](#0-0)
