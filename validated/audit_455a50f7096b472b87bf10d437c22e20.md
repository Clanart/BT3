### Title
Unvalidated `l1_gas_price_wei` / `l1_data_gas_price_wei` in `ProposalInit` Allows Proposer to Inject Arbitrary ETH-Denominated Gas Prices and Corrupt Block Execution - (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_block_info_valid` validates only the FRI-denominated gas prices against the local oracle, but the WEI-denominated gas prices (`l1_gas_price_wei`, `l1_data_gas_price_wei`) carried in `ProposalInit` are accepted without any cross-check. A malicious proposer can set these wei fields to arbitrary values; validators accept the block, and `convert_to_sn_api_block_info` feeds the manipulated wei prices directly into the `BlockInfo` used for execution, corrupting ETH-denominated fee accounting and the derived `l2_gas_price_wei` for every transaction in the block.

### Finding Description

**Root cause — missing wei-price validation in `is_block_info_valid`:**

`is_block_info_valid` calls `get_l1_prices_in_fri_and_wei` and immediately discards the wei half of the result:

```rust
let (l1_gas_prices_fri, _l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
    l1_gas_price_provider,
    init_proposed.timestamp,
    block_info_validation.previous_block_info.as_ref(),
    gas_price_params,
)
.await;
```

Only the FRI prices are compared against the proposal:

```rust
if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
    && within_margin(l1_data_gas_price_fri_proposed, l1_data_gas_price_fri, l1_gas_price_margin_percent))
```

`init_proposed.l1_gas_price_wei` and `init_proposed.l1_data_gas_price_wei` are never compared to the locally computed oracle values. [1](#0-0) 

**Propagation — manipulated wei prices enter execution unchanged:**

After validation passes, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which copies the proposer-supplied wei prices verbatim into the `BlockInfo` handed to the blockifier:

```rust
let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
```

It also derives the implicit ETH/STRK rate from the proposal's own fri/wei pair:

```rust
let previous_block_info = PreviousBlockInfo::from(init);
let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;
let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)?;
``` [2](#0-1) 

`calculate_eth_to_fri_rate` computes `l1_gas_price_fri * WEI_PER_ETH / l1_gas_price_wei`, so a manipulated `l1_gas_price_wei` directly corrupts the derived `eth_to_fri_rate` and therefore `l2_gas_price_wei`. [3](#0-2) 

**Fallback path also uses the hardcoded `DEFAULT_ETH_TO_FRI_RATE = 10^21`** when both the oracle and previous block info are unavailable, which is the direct analog of the hardcoded USDC = $1 peg in the original report: [4](#0-3) [5](#0-4) 

### Impact Explanation

Every transaction in the accepted block is executed with the manipulated `BlockInfo`. Concretely:

- **ETH-denominated (v1/v2) transactions**: fees are charged using `l1_gas_price_wei` / `l1_data_gas_price_wei`. Setting these to near-zero lets a proposer drain the sequencer's fee revenue for ETH payers; setting them very high forces ETH payers to overpay or fail resource-bound checks.
- **`l2_gas_price_wei`**: derived from the corrupted implicit rate, affecting L2 gas pricing in ETH for all transaction types.
- **Block commitment**: the resulting receipts, balances, and state diff are committed to the state root with the wrong fee values, producing an incorrect but accepted block hash and state root.

This matches the allowed impact: *"Critical. Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact."*

### Likelihood Explanation

Any consensus participant that is selected as a proposer for a round can trigger this. No special privilege beyond normal proposer selection is required. The attack is silent — validators accept the block because the FRI-price margin check passes — and the manipulated wei prices are indistinguishable from legitimate ones in the committed block.

### Recommendation

In `is_block_info_valid`, validate the proposed wei prices against the locally computed oracle values with the same `within_margin` check already applied to FRI prices:

```rust
let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(...).await;

// existing FRI checks ...

if !(within_margin(init_proposed.l1_gas_price_wei, l1_gas_prices_wei.l1_gas_price, margin)
    && within_margin(init_proposed.l1_data_gas_price_wei, l1_gas_prices_wei.l1_data_gas_price, margin))
{
    return Err(ValidateProposalError::InvalidBlockInfo(...));
}
```

Alternatively, derive the wei prices inside `convert_to_sn_api_block_info` from the validated FRI prices and the oracle-supplied rate rather than trusting the proposer-supplied wei fields.

### Proof of Concept

1. Proposer is selected for height H.
2. Proposer calls `get_l1_prices_in_fri_and_wei` to obtain legitimate FRI prices (e.g., `l1_gas_price_fri = F`).
3. Proposer constructs `ProposalInit` with `l1_gas_price_fri = F` (within margin) but `l1_gas_price_wei = 1` (near-zero, far below the true oracle wei value).
4. Proposer broadcasts the `ProposalInit`.
5. Each validator runs `is_block_info_valid`: FRI margin check passes; wei prices are never checked (`_l1_gas_prices_wei` is discarded).
6. `initiate_validation` calls `convert_to_sn_api_block_info(init)`: `l1_gas_price_wei = 1` is accepted as `NonzeroGasPrice`; `eth_to_fri_rate = F * 10^18 / 1 = F * 10^18` (astronomically high); `l2_gas_price_wei ≈ 0` (rounds to zero, causing `NonzeroGasPrice::new` to fail or produce 1).
7. All ETH-denominated transactions in the block are executed with `l1_gas_price_wei = 1 wei`, effectively zeroing ETH fees.
8. The block is committed with a state root reflecting these incorrect fee deductions.

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let previous_block_info = PreviousBlockInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}
