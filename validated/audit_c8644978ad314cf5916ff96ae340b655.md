### Title
`CommitmentStateDiff` silently drops deprecated class declarations, producing wrong `state_diff_commitment` and block hash for blocks containing Cairo 0 `DECLARE` transactions — (File: `crates/blockifier/src/state/cached_state.rs`)

---

### Summary

The `From<CommitmentStateDiff> for ThinStateDiff` conversion in `crates/blockifier/src/state/cached_state.rs` unconditionally sets `deprecated_declared_classes: Vec::new()`. Because `CommitmentStateDiff` has no field for deprecated (Cairo 0) class declarations, any such declarations executed in a block are silently omitted from the `ThinStateDiff` that feeds `calculate_block_commitments`. This produces a wrong `state_diff_commitment` and a wrong `state_diff_length` (embedded in `concatenated_counts`) in the block header, corrupting the final block hash.

---

### Finding Description

**Root cause — `From<CommitmentStateDiff> for ThinStateDiff` drops deprecated class declarations**

`CommitmentStateDiff` tracks four categories of state change: `address_to_class_hash`, `address_to_nonce`, `storage_updates`, and `class_hash_to_compiled_class_hash`. It has no field for deprecated (Cairo 0) class declarations. [1](#0-0) 

The `From<StateMaps> for CommitmentStateDiff` conversion silently drops `StateMaps::declared_contracts` (the field that records Cairo 0 class declarations): [2](#0-1) 

Consequently, `From<CommitmentStateDiff> for ThinStateDiff` is forced to hard-code `deprecated_declared_classes: Vec::new()`: [3](#0-2) 

**Propagation into commitment calculation**

`BlockExecutionArtifacts::new` calls `ThinStateDiff::from(commitment_state_diff.clone())` and passes the result directly to `calculate_block_commitments`: [4](#0-3) 

`calculate_block_commitments` computes two values that are both wrong when `deprecated_declared_classes` is empty but should not be:

1. `state_diff_commitment` — computed by `calculate_state_diff_hash`, which explicitly chains `deprecated_declared_classes` into the Poseidon hash: [5](#0-4) 

2. `concatenated_counts` — computed by `concat_counts` using `state_diff.len()`, which counts `deprecated_declared_classes.len()`: [6](#0-5) 

Both corrupted values are chained into the final block hash: [7](#0-6) 

**Downstream effects**

- `BlockExecutionArtifacts::thin_state_diff()` returns the same incomplete `ThinStateDiff`, which is stored in `DecisionReachedResponse::state_diff` and written to the batcher's storage. [8](#0-7) 
- The P2P sync client uses `state_diff_length` from the block header as the termination condition for collecting state diff chunks. A too-small `state_diff_length` causes the client to accept an incomplete state diff from a peer. [9](#0-8) 
- The SNOS/transaction-prover path converts `CommitmentStateDiff` to the committer's `StateDiff` format via `commitment_state_diff_to_committer_state_diff`, which also has no path for deprecated class declarations, so the Patricia-trie commit and proof inputs are consistent with the wrong state diff. [10](#0-9) 

---

### Impact Explanation

For any block that contains a Cairo 0 `DECLARE` transaction:

- `state_diff_commitment` is computed over a `ThinStateDiff` that is missing the deprecated class entries → the commitment value is wrong.
- `concatenated_counts` encodes a `state_diff_length` that is too small by the number of deprecated class declarations → the counts field is wrong.
- Both wrong values are chained into `calculate_block_hash`, producing a wrong block hash and a wrong `ProposalCommitment` used in consensus.
- The stored `ThinStateDiff` is incomplete; any node that syncs the block via P2P will store the same incomplete state diff and will not learn about the deprecated class declarations.
- The SNOS proof input is built from the same incomplete state diff, so any ZK proof generated for the block will attest to a state transition that omits the deprecated class declarations.

This matches **Critical — Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input** and **Critical — Wrong compiled class, CASM/native artifact, class hash, or contract code selected for execution** (the class hash of the deprecated class is absent from the committed state diff).

---

### Likelihood Explanation

The trigger is any block that contains a Cairo 0 `DECLARE` (v0/v1) transaction accepted by the gateway. The `deprecated_declared_classes` field remains present in `ThinStateDiff`, `calculate_state_diff_hash`, and `ThinStateDiff::len()`, confirming the protocol still accounts for such declarations. The TODO comment (`// TODO(AlonH): Remove this when the structure of storage diffs changes`) confirms the omission is structural, not intentional removal of the feature.

---

### Recommendation

Add a `deprecated_declared_classes` field to `CommitmentStateDiff` and populate it from `StateMaps::declared_contracts` in `From<StateMaps> for CommitmentStateDiff`. Update `From<CommitmentStateDiff> for ThinStateDiff` to propagate the field instead of hard-coding `Vec::new()`. Remove the TODO once the field is correctly populated.

---

### Proof of Concept

1. Submit a Cairo 0 `DECLARE` (v0 or v1) transaction to the sequencer gateway.
2. The blockifier executes the transaction and records the class hash in `StateMaps::declared_contracts`.
3. `CommitmentStateDiff::from(StateMaps)` drops `declared_contracts`; `CommitmentStateDiff` has no `deprecated_declared_classes` field. [2](#0-1) 
4. `BlockExecutionArtifacts::new` converts `CommitmentStateDiff` to `ThinStateDiff` with `deprecated_declared_classes: Vec::new()`. [11](#0-10) 
5. `calculate_state_diff_hash` chains an empty `deprecated_declared_classes` list, producing a `state_diff_commitment` that differs from the correct value (which would include the class hash). [5](#0-4) 
6. `concat_counts` encodes `state_diff_length` as `N` instead of `N + 1` (missing the deprecated class entry). [12](#0-11) 
7. `calculate_block_hash` chains both wrong values, producing a wrong block hash. [13](#0-12) 
8. The P2P sync server serves the incomplete `ThinStateDiff`; the client accepts it because `current_state_diff_len == target_state_diff_len` (both wrong by the same amount). [9](#0-8)

/// Converts blockifier's CommitmentStateDiff to committer's StateDiff format.
pub fn commitment_state_diff_to_committer_state_diff(
    commitment_state_diff: CommitmentStateDiff,
) -> StateDiff {
    StateDiff {
        address_to_class_hash: commitment_state_diff.address_to_class_hash.into_iter().collect(),
        address_to_nonce: commitment_state_diff.address_to_nonce.into_iter().collect(),
        class_hash_to_compiled_class_hash: commitment_state_diff
            .class_hash_to_compiled_class_hash
            .into_iter()
            .map(|(class_hash, compiled_class_hash)| {
                (class_hash, CompiledClassHash(compiled_class_hash.0))
            })
            .collect(),
        storage_updates: commitment_state_diff
            .storage_updates
            .into_iter()
            .map(|(address, updates)| {
                (
                    address,
                    updates
                        .into_iter()
                        .map(|(key, value)| (StarknetStorageKey(key), StarknetStorageValue(value)))
                        .collect(),
                )
            })
            .collect(),
    }
}
