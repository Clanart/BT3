### Title
Unprivileged HyperCore User Can Permanently Lock Tokens by Setting ACTION_TRANSFER Recipient to `address(this)` — (`evm/src/omni-bridge/contracts/HlBridgeToken.sol`)

---

### Summary

`coreReceiveWithData` in `HyperliquedBridgeToken` dispatches `ACTION_TRANSFER` by decoding a user-controlled `recipient` address from the `data` field and calling `_update(_systemAddress, recipient, amount)`. There is no guard preventing `recipient == address(this)`. Because the `data` field originates from the HyperCore user's `sendToEvmWithData` call and is relayed verbatim by the system address, any unprivileged user can direct tokens into the token contract's own ERC20 balance, where no release path exists.

---

### Finding Description

In `coreReceiveWithData`, the only access control is:

```solidity
if (msg.sender != _systemAddress) revert NotSystemAddress();
``` [1](#0-0) 

The system address is a relay — it fires this callback when a HyperCore user calls `sendToEvmWithData`, passing through the user-supplied `data` unchanged. The `ACTION_TRANSFER` branch then does:

```solidity
if (action == ACTION_TRANSFER) {
    address recipient = abi.decode(tail, (address));
    _update(_systemAddress, recipient, amount);
}
``` [2](#0-1) 

There is no check that `recipient != address(this)`. If the attacker encodes `address(token)` as the recipient, `_update` moves `amount` tokens from `_systemAddress` into the token contract's own ERC20 balance.

Neither `HyperliquedBridgeToken` nor `BridgeToken` contains a sweep or rescue function for ERC20 tokens held by the contract itself: [3](#0-2) 

The only callable function that touches `address(this)`'s balance is `ACTION_INIT_TRANSFER`, but that path moves tokens **from `_systemAddress`** to `address(this)` and then burns them via `OmniBridge.initTransfer` — it does not touch tokens already sitting at `address(this)` from a prior `ACTION_TRANSFER` lock: [4](#0-3) 

`OmniBridge.initTransfer` burns from `msg.sender` (the token contract) when `isBridgeToken[tokenAddress]` is true: [5](#0-4) 

But this path is only reachable if the token contract itself initiates the call, which it cannot do for already-locked tokens. The owner (`OmniBridge`) can call `BridgeToken.burn(address(token), amount)` to destroy the stuck tokens, but this destroys value rather than returning it to the user.

---

### Impact Explanation

**Critical — Permanent freezing / irrecoverable lock of bridged assets.**

Tokens transferred to `address(this)` via `ACTION_TRANSFER` are permanently inaccessible to the user. The only admin recourse is to burn them (destroying value), not return them. The HyperCore-side balance is debited, but the EVM-side tokens are locked in the contract forever, breaking bridge collateralization.

---

### Likelihood Explanation

**High.** The attack requires no privilege — any HyperCore user can call `sendToEvmWithData` with arbitrary `data`. The encoding is trivial: `0x00 || abi.encode(address(token))`. The system address relays it without validation. The attacker loses their own tokens but can also grief other users by targeting any amount they control on HyperCore.

---

### Recommendation

Add a zero-address and self-address guard in the `ACTION_TRANSFER` branch:

```solidity
if (action == ACTION_TRANSFER) {
    address recipient = abi.decode(tail, (address));
    require(recipient != address(0) && recipient != address(this), "Invalid recipient");
    _update(_systemAddress, recipient, amount);
}
``` [2](#0-1)

    function coreReceiveWithData(
        address from,
        bytes32 /*destinationRecipient*/,
        uint32 /*destinationChainId*/,
        uint256 amount,
        uint64 /*coreNonce*/,
        bytes calldata data
    ) external override {
        if (msg.sender != _systemAddress) revert NotSystemAddress();
        if (data.length == 0) revert EmptyActionData();

        uint8 action = uint8(data[0]);
        bytes calldata tail = data[1:];

        if (action == ACTION_TRANSFER) {
            address recipient = abi.decode(tail, (address));
            _update(_systemAddress, recipient, amount);
        } else if (action == ACTION_INIT_TRANSFER) {
            (uint128 fee, string memory recipient, string memory message) = abi
                .decode(tail, (uint128, string, string));
            uint128 amount128 = amount.toUint128();
            _update(_systemAddress, address(this), amount);
            IOmniBridgeInitTransfer(owner()).initTransfer(
                address(this),
                amount128,
                fee,
                0,
                recipient,
                message
            );
        } else {
            revert UnknownAction(action);
        }

        emit CoreReceived(from, action, amount, data);
    }
