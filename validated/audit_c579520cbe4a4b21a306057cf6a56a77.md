### Title
Empty-Layers `ProofOfInclusion::valid()` Tautology Allows Forged DataLayer Inclusion Proofs - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` unconditionally returns `true` when the `layers` field is empty. Because `root_hash()` also falls back to `node_hash` when `layers` is empty, the final comparison inside `valid()` reduces to `node_hash == node_hash`, which is always true. An attacker can construct a zero-layer `ProofOfInclusion` for any arbitrary `node_hash` and have it pass validation, forging a DataLayer inclusion proof without any knowledge of the actual tree contents.

### Finding Description

`ProofOfInclusion` is a `Streamable` struct exposed through Python bindings:

```rust
pub struct ProofOfInclusion {
    pub node_hash: Hash,
    pub layers: Vec<ProofOfInclusionLayer>,
}
``` [1](#0-0) 

`root_hash()` returns `node_hash` when `layers` is empty:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash   // ← fallback when layers is empty
    }
}
``` [2](#0-1) 

`valid()` iterates over `layers` and then compares `existing_hash` (which starts as `node_hash` and is never updated when `layers` is empty) against `self.root_hash()` (which also returns `node_hash` when `layers` is empty):

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {          // ← loop body never executes
        ...
        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()    // ← node_hash == node_hash → always true
}
``` [3](#0-2)

### Impact Explanation

An attacker who can supply a serialized `ProofOfInclusion` to any verifier that calls only `proof.valid()` can forge inclusion of any arbitrary `node_hash`. Because `root_hash()` returns `node_hash` for an empty-layers proof, a verifier that additionally checks `proof.root_hash() == expected_root` can be bypassed by setting `node_hash = expected_root` — causing the proof to claim that the tree root itself is a leaf node. Any downstream logic that trusts `valid()` to confirm DataLayer state (e.g., key-value membership) will accept the forged proof, allowing an attacker to prove invalid state against a committed DataLayer tree root.

This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.**

### Likelihood Explanation

`ProofOfInclusion` is a `Streamable` type deserializable from raw bytes via `from_bytes()` / `from_bytes_unchecked()`, and is exposed to Python consumers. Any DataLayer client or full-node code that accepts a peer-supplied `ProofOfInclusion` and calls `valid()` is directly exploitable. The construction of a malicious proof requires only setting `layers` to an empty list and `node_hash` to the target value — no cryptographic work is needed.

### Recommendation

1. **Reject empty proofs in `valid()`**: Add an explicit check at the start of `valid()` that returns `false` when `layers` is empty (unless a separate single-leaf-root case is intentionally supported and documented).
2. **Bind `valid()` to an expected root**: Change the signature to `valid(&self, expected_root: &Hash) -> bool` so that the caller cannot accidentally omit the root comparison. The current two-step pattern (`valid()` + manual `root_hash()` comparison) is error-prone.
3. **Add a test for the empty-layers forgery**: Construct `ProofOfInclusion { node_hash: arbitrary_hash, layers: vec![] }` and assert that `valid()` returns `false`.

### Proof of Concept

```rust
use chia_datalayer::merkle::proof_of_inclusion::ProofOfInclusion;

// Forge a proof for any arbitrary node_hash with zero layers
let forged_proof = ProofOfInclusion {
    node_hash: [0xAB; 32],  // attacker-chosen hash, not in any real tree
    layers: vec![],
};

// valid() returns true — no cryptographic work required
assert!(forged_proof.valid());

// root_hash() returns node_hash, so if the verifier checks
// proof.root_hash() == expected_root, set node_hash = expected_root
let forged_root_proof = ProofOfInclusion {
    node_hash: actual_tree_root,  // set to the real root
    layers: vec![],
};
assert!(forged_root_proof.valid());
assert_eq!(forged_root_proof.root_hash(), actual_tree_root);
// Both checks pass — the proof is accepted as valid
