### Title
`CommitmentStateDiff` Silently Drops Deprecated Declared Classes, Producing Wrong `state_diff_commitment` and Block Hash — (`crates/blockifier/src/state/cached_state.rs`)

### Summary

The `CommitmentStateDiff` struct has no field for deprecated (Cairo 0) declared classes. The conversion chain `StateMaps` → `CommitmentStateDiff` → `ThinStateDiff` silently drops all deprecated declared class hashes by hardcoding `deprecated_declared_classes: Vec::new()`. Because `calculate_state_diff_hash` and `ThinStateDiff::len()` both consume `deprecated_declared_classes`, any block that contains a deprecated declare transaction will have a wrong `state_diff_commitment`, wrong `state_diff_length`, and therefore a wrong block hash committed to the chain.

### Finding Description

**Root cause — fixed struct with no slot for a protocol-level state-diff category:**

`StateMaps` tracks class declarations in two separate maps:

- `compiled_class_hashes: HashMap<ClassHash, CompiledClassHash>` — populated for Cairo 1 (Sierra) declarations
- `declared_contracts: HashMap<ClassHash, bool>` — populated for **all** declarations; the only record of a Cairo 0 (deprecated) declaration is `(class_hash, true)` here with **no** entry in `compiled_class_hashes` [1](#0-0) 

`CommitmentStateDiff::from(StateMaps)` maps `compiled_class_hashes` → `class_hash_to_compiled_class_hash` but **silently drops `declared_contracts`** entirely: [2](#0-1) 

`ThinStateDiff::from(CommitmentStateDiff)` then hardcodes the missing category to an empty vector: [3](#0-2) 

The TODO comment (`// TODO(AlonH): Remove this when the structure of storage diffs changes.`) acknowledges the structural gap. A parallel conversion in the cende client layer carries the same omission with an explicit uncertainty comment: [4](#0-3) 

**Propagation into the block commitment:**

`BlockExecutionArtifacts::new` calls `calculate_block_commitments` with the incomplete `ThinStateDiff`: [5](#0-4) 

`calculate_block_commitments` passes the diff to `calculate_state_diff_hash` and also calls `state_diff.len()` to build `concatenated_counts`: [6](#0-5) 

`calculate_state_diff_hash` explicitly chains `deprecated_declared_classes` into the Poseidon hash: [7](#0-6) 

`ThinStateDiff::len()` counts `deprecated_declared_classes.len()` toward `state_diff_length`: [8](#0-7) 

Because `deprecated_declared_classes` is always `[]`, both the Poseidon hash and the length counter are computed as if no deprecated class was ever declared, regardless of how many were actually included in the block.

**Analog to CVXStaker M-08:**

| CVXStaker | Sequencer |
|---|---|
| `rewardTokens[]` fixed at construction; new `extraRewards` added by Convex are never iterated | `CommitmentStateDiff` fixed struct; deprecated declared classes added by transactions are never captured |
| `getReward` loops only the stale list; new tokens stay stuck | `calculate_state_diff_hash` hashes only the stale (empty) list; deprecated classes are silently absent from the commitment |
| Fix: add `setRewardTokens` | Fix: add `deprecated_declared_classes` field to `CommitmentStateDiff` |

### Impact Explanation

Every block that contains at least one deprecated (Cairo 0) declare transaction will have:

1. A wrong `state_diff_commitment` — the Poseidon hash omits the deprecated class hashes that the protocol requires.
2. A wrong `state_diff_length` — the count is too small by the number of deprecated declared classes.
3. A wrong `concatenated_counts` field — derived from the wrong length.
4. A wrong block hash — all three corrupted values feed into `PartialBlockHashComponents` and ultimately `calculate_block_hash`.

All sequencer nodes run the same code and will agree on the wrong commitment, so the block is accepted. Any external verifier (prover, state-sync node, or L1 verifier) that independently recomputes the state diff commitment from the actual state diff will observe a mismatch and reject the block or proof.

This matches: **Critical — Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input.**

### Likelihood Explanation

Any unprivileged user who submits a valid deprecated declare transaction (transaction version 0 or 1) triggers the bug. The gateway and blockifier both retain code paths for deprecated declare transactions (the `ValidResourceBounds::L1Gas` variant covers pre-0.13.3 transactions; `StateMaps.declared_contracts` is populated during execution). No special privilege or coordination is required.

### Recommendation

1. Add `deprecated_declared_classes: Vec<ClassHash>` to `CommitmentStateDiff`.
2. In `CommitmentStateDiff::from(StateMaps)`, populate it from `diff.declared_contracts`: collect every `class_hash` where `is_declared == true` and no entry exists in `diff.compiled_class_hashes`.
3. In `ThinStateDiff::from(CommitmentStateDiff)`, propagate the field instead of hardcoding `Vec::new()`.
4. Remove the `// TODO(AlonH)` comment once the fix is in place.
5. Add a regression test that declares a Cairo 0 class, computes the state diff commitment, and verifies it matches `calculate_state_diff_hash` applied to the full diff including the deprecated class.

### Proof of Concept

```
1. Attacker submits a DeprecatedDeclare (v1) transaction declaring Cairo 0 class C.

2. Blockifier executes the transaction:
   StateMaps.declared_contracts  = { C → true }
   StateMaps.compiled_class_hashes = {}   // no entry for Cairo 0

3. CommitmentStateDiff::from(StateMaps):
   class_hash_to_compiled_class_hash = {}  // compiled_class_hashes is empty
   // declared_contracts is DROPPED

4. ThinStateDiff::from(CommitmentStateDiff):
   deprecated_declared_classes = Vec::new()  // hardcoded

5. calculate_state_diff_hash(&thin_state_diff):
   chain_deprecated_declared_classes(&[])   // chains count=0, no class hashes
   → state_diff_commitment = H(... 0 ...)   // WRONG: should be H(... 1, C ...)

6. ThinStateDiff::len() returns N (not N+1)
   → state_diff_length in block header = N  // WRONG: should be N+1

7. Block is committed with wrong state_diff_commitment and wrong block hash.
   Any independent verifier recomputes H(... 1, C ...) ≠ committed value → mismatch.
