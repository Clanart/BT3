### Title
Missing Declared Class Hash Validation in `execute_replace_class` Allows Permanent Contract Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS program unconditionally overwrites a contract's `class_hash` in `contract_state_changes` with any caller-supplied value, without verifying that the new class hash corresponds to a previously declared contract class. This is directly acknowledged by a developer TODO comment in the code. A contract can therefore replace its own class with an arbitrary, undeclared hash, rendering itself permanently uncallable and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function handles the `replace_class` syscall:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The function reads the current `state_entry` for the contract address and then writes a `new_state_entry` with the caller-supplied `class_hash` — with no assertion or lookup confirming that `class_hash` exists in `contract_class_changes`. The developer TODO at line 898 explicitly acknowledges this missing check.

The same omission exists in the deprecated syscall path:

```cairo
func execute_replace_class{contract_state_changes: DictAccess*, revert_log: RevertLogEntry*}(
    contract_address, syscall_ptr: ReplaceClass*
) {
    ...
    let class_hash = syscall_ptr.class_hash;
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}
    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );
    dict_update{dict_ptr=contract_state_changes}(...);
``` [2](#0-1) 

Contrast this with the `execute_declare_transaction` path, which correctly enforces `prev_value=0` to ensure a class is declared only once:

```cairo
// Note that prev_value=0 enforces that a class may be declared only once.
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [3](#0-2) 

No equivalent guard exists in `execute_replace_class` to verify the incoming `class_hash` is present in `contract_class_changes`.

Similarly, `deploy_contract` correctly asserts the contract address is uninitialized before writing:

```cairo
assert state_entry.class_hash = UNINITIALIZED_CLASS_HASH;
``` [4](#0-3) 

This pattern of validating state before writing is consistently applied everywhere except `execute_replace_class`.

---

### Impact Explanation

When a contract calls `replace_class` with a hash that has never been declared (i.e., has no entry in `contract_class_changes`), the OS commits the new `class_hash` to `contract_state_changes` without error. After the block is finalized:

- The contract's on-chain class hash points to a non-existent class.
- Every future call to that contract will fail at class resolution time, because no compiled class exists for the hash.
- Any ERC-20 tokens, ETH, or other assets held by the contract become permanently inaccessible.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

The `replace_class` syscall is a standard, documented StarkNet syscall callable from any Sierra contract's execution context. No privileged role is required. Any contract — including one deployed by an unprivileged user — can issue this syscall with an arbitrary `felt` value as the class hash. The OS performs no on-chain lookup to confirm the hash is declared. The risk is elevated because:

1. The missing check is explicitly flagged by a developer TODO, indicating it is a known gap.
2. Both the current (`syscall_impls.cairo`) and deprecated (`deprecated_execute_syscalls.cairo`) code paths are affected.
3. A single malformed transaction is sufficient to permanently brick a contract.

**Likelihood: Medium** — requires a contract to issue `replace_class` with an undeclared hash, which can occur through developer error, a buggy upgrade path, or deliberate exploitation of a contract that exposes `replace_class` to external callers.

---

### Recommendation

Before committing the new class hash, verify it exists in `contract_class_changes` (i.e., has been declared in the current or a prior block). A minimal fix mirrors the pattern already used in `execute_declare_transaction`:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

+   // Verify the class has been declared.
+   let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
+   assert_not_zero(compiled_class_hash);

    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}
    ...
}
```

Apply the same fix to `execute_replace_class` in `deprecated_execute_syscalls.cairo`.

---

### Proof of Concept

1. Attacker deploys Contract A holding user funds (e.g., an escrow or vault).
2. Contract A calls the `replace_class` syscall with `class_hash = 0xdeadbeef` (an arbitrary, never-declared hash).
3. The OS executes `execute_replace_class`:
   - Reads `state_entry` for Contract A (current, valid class hash).
   - Writes `new_state_entry` with `class_hash = 0xdeadbeef` — **no validation performed**.
   - `dict_update` succeeds because `prev_value` matches the current entry.
4. The block is finalized; Contract A's class hash is now `0xdeadbeef` on-chain.
5. Any subsequent call to Contract A fails at class resolution — no compiled class exists for `0xdeadbeef`.
6. All funds held by Contract A are permanently frozen with no recovery path.
