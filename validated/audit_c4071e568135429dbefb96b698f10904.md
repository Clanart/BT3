### Title
`CommitmentStateDiff` Silently Drops Deprecated (Cairo 0) Class Declarations, Producing Wrong `state_diff_commitment`, `concatenated_counts`, and Block Hash — (`crates/blockifier/src/state/cached_state.rs`)

---

### Summary

`CommitmentStateDiff` has no field for `deprecated_declared_classes` (Cairo 0 class declarations). The `From<CommitmentStateDiff> for ThinStateDiff` conversion hardcodes `deprecated_declared_classes: Vec::new()`. This `ThinStateDiff` is the sole input to `calculate_block_commitments` inside `BlockExecutionArtifacts::new`. Because `calculate_state_diff_hash` explicitly hashes `deprecated_declared_classes` and `ThinStateDiff::len()` counts them, any block that contains a Cairo 0 `Declare` transaction will produce a wrong `state_diff_commitment`, a wrong `concatenated_counts` (packed lengths field), a wrong `PartialBlockHash` (proposal commitment), and a wrong final block hash.

---

### Finding Description

**Root cause — the silent drop:**

`CommitmentStateDiff` has four fields: `address_to_class_hash`, `address_to_nonce`, `storage_updates`, and `class_hash_to_compiled_class_hash`. There is no `deprecated_declared_classes` field. [1](#0-0) 

When `StateMaps` is converted to `CommitmentStateDiff`, the `declared_contracts` map (which tracks whether a class hash is declared, including Cairo 0 classes) is silently dropped: [2](#0-1) 

When `CommitmentStateDiff` is then converted to `ThinStateDiff` (the type consumed by the commitment pipeline), `deprecated_declared_classes` is hardcoded to an empty `Vec`: [3](#0-2) 

**Propagation into the commitment pipeline:**

`BlockExecutionArtifacts::new` calls `ThinStateDiff::from(commitment_state_diff.clone())` and passes the result directly to `calculate_block_commitments`: [4](#0-3) 

Inside `calculate_block_commitments`, two values are derived from the `ThinStateDiff`:

1. `state_diff.len()` — used to build `concatenated_counts` (the packed lengths field in the block hash): [5](#0-4) 

2. `calculate_state_diff_hash(&state_diff)` — the Poseidon hash of the full state diff: [6](#0-5) 

`calculate_state_diff_hash` explicitly chains `deprecated_declared_classes` into the Poseidon hash: [7](#0-6) 

`ThinStateDiff::len()` also counts `deprecated_declared_classes`: [8](#0-7) 

Because the `ThinStateDiff` fed into both functions always has `deprecated_declared_classes = []`, both the `state_diff_commitment` and `concatenated_counts` are wrong whenever a block contains Cairo 0 class declarations.

**Propagation into the block hash:**

Both corrupted values flow into `BlockHeaderCommitments`, which is stored in `PartialBlockHashComponents`: [9](#0-8) 

`calculate_block_hash` chains both `concatenated_counts` and `state_diff_commitment` into the final block hash: [10](#0-9) 

The `PartialBlockHash` (proposal commitment used in consensus) is derived from the same corrupted `PartialBlockHashComponents`: [11](#0-10) 

**The P2P sync path is unaffected** — it assembles `ThinStateDiff` directly from `StateDiffChunk::DeprecatedDeclaredClass` chunks and correctly populates `deprecated_declared_classes`: [12](#0-11) 

This means a syncing node will compute a different (correct) block hash than the proposing sequencer, breaking consensus agreement.

---

### Impact Explanation

For any block containing at least one Cairo 0 (`Declare` v0/v1) transaction:

- `state_diff_commitment` is wrong (deprecated class hashes are absent from the Poseidon hash).
- `concatenated_counts` is wrong (state diff length is undercounted by the number of deprecated declared classes).
- `PartialBlockHash` (the proposal commitment broadcast during consensus) is wrong.
- The final block hash written to storage is wrong.

A syncing peer that correctly includes `deprecated_declared_classes` in its hash computation will compute a different block hash, causing consensus disagreement or proof verification failure. Any L1 state commitment or ZK proof that relies on the block hash will be anchored to a wrong value.

This matches: **Critical — Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input**, and **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value** (block hash queries via RPC will return the wrong hash).

---

### Likelihood Explanation

Cairo 0 `Declare` transactions (v0/v1) are still valid on Starknet. The P2P sync layer explicitly handles `DeprecatedDeclaredClass` chunks, confirming the protocol expects such blocks to exist. Any user submitting a Cairo 0 class declaration triggers the bug. No special privilege is required.

---

### Recommendation

Add `deprecated_declared_classes` to `CommitmentStateDiff` and populate it from `StateMaps.declared_contracts` (filtering for Cairo 0 classes, i.e., those with no entry in `compiled_class_hashes`). Propagate it through the `From<CommitmentStateDiff> for ThinStateDiff` conversion instead of hardcoding `Vec::new()`.

Remove the `// TODO(AlonH): Remove this when the structure of storage diffs changes.` comment and the hardcoded empty vec at: [13](#0-12) 

Add a test in `calculate_state_diff_hash` and `BlockExecutionArtifacts::new` that asserts the commitment changes when `deprecated_declared_classes` is non-empty.

---

### Proof of Concept

1. Submit a Cairo 0 `Declare` transaction (v0 or v1) to the sequencer gateway.
2. The blockifier executes it; `StateMaps.declared_contracts` records `{class_hash: true}` with no corresponding entry in `compiled_class_hashes`.
3. `CommitmentStateDiff::from(state_maps)` drops `declared_contracts` entirely.
4. `ThinStateDiff::from(commitment_state_diff)` sets `deprecated_declared_classes = []`.
5. `calculate_state_diff_hash` hashes `deprecated_declared_classes = []` → produces hash `H_wrong`.
6. `state_diff.len()` returns `N` (missing the 1 deprecated class) → `concatenated_counts` encodes `N` instead of `N+1`.
7. `calculate_block_hash` chains `H_wrong` and the wrong `concatenated_counts` → block hash `BH_wrong`.
8. A syncing peer receives the block, assembles `ThinStateDiff` with `deprecated_declared_classes = [class_hash]`, computes `H_correct ≠ H_wrong`, and derives `BH_correct ≠ BH_wrong`.
9. Consensus or proof verification fails, or the wrong block hash is committed to L1.

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
