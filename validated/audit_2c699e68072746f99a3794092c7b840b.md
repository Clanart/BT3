### Title
`ProofOfInclusion::valid()` Never Verifies Against an External Trusted Root — Forged Inclusion Proofs Always Pass - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` only checks internal chain self-consistency. Its final comparison `existing_hash == self.root_hash()` is a tautology when layers are present, because `root_hash()` returns `layers.last().combined_hash`, which is exactly the value `existing_hash` was set to in the last loop iteration. The function never accepts an external trusted root hash as a parameter and never compares against one. Any attacker who can supply a `ProofOfInclusion` object (via Python bindings, Streamable deserialization, or direct construction) can fabricate a self-consistent proof for any arbitrary key and have `valid()` return `true`.

### Finding Description

`ProofOfInclusion` is the DataLayer Merkle inclusion proof type, exposed to Python via `py_valid()` and serializable via `Streamable`. [1](#0-0) 

The `valid()` method iterates over layers, computing each `calculated_hash` and comparing it to `layer.combined_hash`. After the loop, `existing_hash` holds the value of `layers.last().combined_hash`. The final check is:

```rust
existing_hash == self.root_hash()
```

But `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← same value as existing_hash after the loop
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

When `layers` is non-empty, `existing_hash` and `self.root_hash()` are always identical — the check is a tautology. `valid()` therefore only verifies that the hash chain is internally self-consistent, but **never verifies the chain terminates at any externally committed, trusted root hash**. There is no parameter for a trusted root, and no caller-side enforcement is enforced by the API.

The struct is fully constructible from untrusted bytes via `Streamable` deserialization and via the Python `from_bytes` binding: [3](#0-2) 

The Python binding exposes `valid()` directly as the sole verification method: [4](#0-3) [5](#0-4) 

### Impact Explanation

Any DataLayer consumer (Python or Rust) that calls `proof.valid()` as the sole check for inclusion verification — without separately comparing `proof.root_hash()` against a known, trusted, on-chain committed root — can be fooled by a completely fabricated proof. An attacker constructs a `ProofOfInclusion` with arbitrary `node_hash`, `other_hash`, and a `combined_hash` computed to be internally consistent. `valid()` returns `true`. The attacker can thereby prove inclusion of any key-value pair in any fake tree root, enabling forged DataLayer state proofs. This maps to the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, or lets untrusted input prove invalid state.**

### Likelihood Explanation

The `valid()` method is the only verification API on `ProofOfInclusion`. Its name strongly implies it is a complete validity check. Any DataLayer integration that receives a proof from an untrusted peer and calls `proof.valid()` without an additional `proof.root_hash() == trusted_root` check is fully exploitable. The fuzz target itself only calls `proof.valid()` without a root check, demonstrating the pattern is easy to misuse: [6](#0-5) 

### Recommendation

`valid()` must accept an external trusted root hash parameter and compare the computed chain root against it:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    // ... existing chain check ...
    existing_hash == *trusted_root  // compare against external root, not self.root_hash()
}
```

Alternatively, rename the current method to `is_internally_consistent()` and add a separate `valid(trusted_root: &Hash) -> bool` that performs the full check. The Python binding and all callers must be updated accordingly.

### Proof of Concept

```python
from chia_rs import ProofOfInclusionLayer, ProofOfInclusion
from chia_rs.datalayer import calculate_internal_hash  # or equivalent

fake_node_hash = bytes([0xAA] * 32)
other_hash     = bytes([0xBB] * 32)
# compute a combined_hash that is internally consistent
combined_hash  = calculate_internal_hash(fake_node_hash, 0, other_hash)

layer = ProofOfInclusionLayer(
    other_hash_side=0,
    other_hash=other_hash,
    combined_hash=combined_hash,
)
proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

assert proof.valid()          # True — but this key was never in any real tree
assert proof.root_hash() == combined_hash  # fabricated root, not any committed root
```

The proof passes `valid()` despite corresponding to no real DataLayer tree state. Any verifier relying solely on `proof.valid()` accepts this forged inclusion claim. [1](#0-0)

    pub fn valid(&self) -> bool {
        let mut existing_hash = self.node_hash;

        for layer in &self.layers {
            let calculated_hash = crate::calculate_internal_hash(
                &existing_hash,
                layer.other_hash_side,
                &layer.other_hash,
            );

            if calculated_hash != layer.combined_hash {
                return false;
            }

            existing_hash = calculated_hash;
        }

        existing_hash == self.root_hash()
    }
