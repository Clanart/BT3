### Title
Unvalidated `l1_gas_price_wei` in `ProposalInit` Allows Proposer to Inject Arbitrary `l2_gas_price_wei` into Block Hash and Fee Charging - (File: `crates/apollo_consensus_orchestrator/src/utils.rs`)

### Summary

The `is_block_info_valid` function in the consensus orchestrator validates `l1_gas_price_fri` and `l1_data_gas_price_fri` within a margin against the local oracle, but **completely discards** the locally-computed `l1_gas_prices_wei`. The `l1_gas_price_wei` and `l1_data_gas_price_wei` fields from the proposer's `ProposalInit` are accepted without any bound check. These unvalidated wei prices are then used to derive `eth_to_fri_rate`, which in turn determines `l2_gas_price_wei`. Both `l1_gas_price_wei` and the derived `l2_gas_price_wei` are committed into the block hash and used for ETH-denominated fee charging.

### Finding Description

In `is_block_info_valid`:

```rust
let (l1_gas_prices_fri, _l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
    l1_gas_price_provider,
    init_proposed.timestamp,
    block_info_validation.previous_block_info.as_ref(),
    gas_price_params,
)
.await;
```

The `_l1_gas_prices_wei` is explicitly discarded. Only the FRI prices are validated within a margin. [1](#0-0) 

In `convert_to_sn_api_block_info`, the `eth_to_fri_rate` is derived from the proposer-supplied `l1_gas_price_fri` and `l1_gas_price_wei`:

```rust
let previous_block_info = PreviousBlockInfo::from(init);
let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;

let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
``` [2](#0-1) 

`calculate_eth_to_fri_rate` computes: `eth_to_fri_rate = l1_gas_price_fri * WEI_PER_ETH / l1_gas_price_wei`. [3](#0-2) 

Therefore: `l2_gas_price_wei = l2_gas_price_fri * WEI_PER_ETH / eth_to_fri_rate = l2_gas_price_fri * l1_gas_price_wei / l1_gas_price_fri`.

Since `l1_gas_price_wei` is never validated, a malicious proposer can set it to any value. Setting it to `u128::MAX / l1_gas_price_fri` makes `eth_to_fri_rate` approach 0 or 1, causing `l2_gas_price_wei` to become astronomically large.

Both `l1_gas_price_wei` and the derived `l2_gas_price_wei` are committed into the block hash via `gas_prices_to_hash` (for Starknet version ≥ 0.13.4): [4](#0-3) 

`PartialBlockHashComponents` is populated from `BlockInfo` which carries both values: [5](#0-4) 

The `ProposalInit` struct carries `l1_gas_price_wei` and `l1_data_gas_price_wei` as proposer-supplied fields with no upper bound enforced by validators: [6](#0-5) 

### Impact Explanation

**Wrong block hash commitment (Critical):** The block hash for Starknet ≥ 0.13.4 includes `l1_gas_price_wei` and `l2_gas_price_wei` directly. A malicious proposer can commit an arbitrarily wrong `l2_gas_price_wei` into the canonical block hash. All validators accept this because they all derive the same `l2_gas_price_wei` from the same unvalidated `l1_gas_price_wei`. The committed block hash diverges from what the protocol should produce, breaking the gas-price commitment invariant.

**Incorrect fee charging (Critical):** `l2_gas_price_wei` is placed into `eth_gas_prices.l2_gas_price` in `BlockInfo` and used for ETH-denominated (V1/V2) L2 gas fee charging. An inflated `l2_gas_price_wei` causes ETH-denominated transactions to be charged at a wrong price, causing spurious reverts or incorrect fee deductions. [7](#0-6) 

**Wrong RPC fee estimation (High):** `starknet_estimateFee` and `starknet_estimateMessageFee` use the block's `BlockInfo` gas prices. If the pending block carries a wrong `l2_gas_price_wei`, fee estimates returned to users are authoritative-looking but incorrect. [8](#0-7) 

### Likelihood Explanation

Any consensus validator that wins a proposal round can exploit this. Validators are not a small privileged set in a permissionless or semi-permissionless validator committee. The attack requires only crafting a `ProposalInit` with an out-of-range `l1_gas_price_wei` while keeping `l1_gas_price_fri` within the allowed margin. No special access beyond being a proposer is needed.

### Recommendation

In `is_block_info_valid`, validate `l1_gas_price_wei` and `l1_data_gas_price_wei` against the locally computed values (from `_l1_gas_prices_wei`) using the same `within_margin` check already applied to the FRI prices. Specifically, change:

```rust
let (l1_gas_prices_fri, _l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(...).await;
```

to use `l1_gas_prices_wei` and add margin checks for `init_proposed.l1_gas_price_wei` and `init_proposed.l1_data_gas_price_wei` against the locally computed wei values, analogous to the existing FRI checks at lines 302–318. [1](#0-0) 

### Proof of Concept

1. Attacker is a validator and wins a proposal round for block N.
2. Attacker constructs `ProposalInit` with:
   - `l1_gas_price_fri` = locally expected value (passes margin check)
   - `l1_gas_price_wei` = `u128::MAX / l1_gas_price_fri` (not validated)
   - `l2_gas_price_fri` = locally expected value (passes exact check)
3. `convert_to_sn_api_block_info` computes:
   - `eth_to_fri_rate = l1_gas_price_fri * 10^18 / (u128::MAX / l1_gas_price_fri)` ≈ 0 or 1
   - `l2_gas_price_wei = l2_gas_price_fri * 10^18 / eth_to_fri_rate` → near `u128::MAX`
4. All validators accept the `ProposalInit` (only FRI prices are checked).
5. Block hash is committed with `l2_gas_price_wei ≈ u128::MAX` — wrong commitment accepted by all nodes.
6. Any ETH-denominated V1/V2 transaction in this block that consumes L2 gas is charged at `l2_gas_price_wei ≈ u128::MAX`, causing all such transactions to revert with `MaxFeeExceeded` or `InsufficientFeeTokenBalance`, producing wrong revert results for accepted inputs.

fn calculate_eth_to_fri_rate(block_info: &PreviousBlockInfo) -> Result<u128, StarknetApiError> {
    let eth_to_fri_rate = block_info
        .l1_prices_fri
        .l1_gas_price
        .0
        .checked_mul(WEI_PER_ETH)
        .ok_or_else(|| {
            StarknetApiError::GasPriceConversionError(format!(
                "Gas price in Fri should be small enough to multiply by WEI_PER_ETH. Previous \
                 block info: {:?}",
                block_info
            ))
        })?
        .checked_div(block_info.l1_prices_wei.l1_gas_price.0)
        .ok_or_else(|| {
            StarknetApiError::GasPriceConversionError(format!(
                "Gas price in Wei should be non-zero. Previous block info: {:?}",
                block_info
            ))
        })?;
    if eth_to_fri_rate == 0 {
        return Err(StarknetApiError::GasPriceConversionError(format!(
            "Eth to fri rate is zero. Previous block info: {:?}",
            block_info
        )));
    }
    Ok(eth_to_fri_rate)

pub fn gas_prices_to_hash(
    l1_gas_price: &GasPricePerToken,
    l1_data_gas_price: &GasPricePerToken,
    l2_gas_price: &GasPricePerToken,
    block_hash_version: &BlockHashVersion,
) -> Vec<Felt> {
    if block_hash_version >= &BlockHashVersion::V0_13_4 {
        vec![
            HashChain::new()
                .chain(&STARKNET_GAS_PRICES0)
                .chain(&l1_gas_price.price_in_wei.0.into())

pub(crate) fn tx_execution_output_to_fee_estimation(
