### Title
`deprecated_declared_classes` Silently Dropped in `CommitmentStateDiff → ThinStateDiff` Conversion, Corrupting `state_diff_commitment` and `concatenated_counts` in Block Hash - (`File: crates/blockifier/src/state/cached_state.rs`)

### Summary

`CommitmentStateDiff` has no `deprecated_declared_classes` field. The `From<CommitmentStateDiff> for ThinStateDiff` conversion hardcodes `deprecated_declared_classes: Vec::new()`. Every call to `BlockExecutionArtifacts::new()` passes this truncated `ThinStateDiff` to `calculate_block_commitments()`, which uses it to compute both the `state_diff_commitment` hash and the `state_diff_length` packed into `concatenated_counts`. Both values are chained into the block hash and the `PartialBlockHash` used for consensus. Any block containing a deprecated (Cairo 0) class declaration produces a wrong `state_diff_commitment`, a wrong `concatenated_counts`, and therefore a wrong block hash — a direct analog to `_transferMint()`/`_transferBurn()` not updating `_totalSupply`.

### Finding Description

**Root cause — the missing field:**

`CommitmentStateDiff` in `crates/blockifier/src/state/cached_state.rs` tracks four categories of state change but has no slot for deprecated class declarations:

```rust
pub struct CommitmentStateDiff {
    pub address_to_class_hash: IndexMap<ContractAddress, ClassHash>,
    pub address_to_nonce: IndexMap<ContractAddress, Nonce>,
    pub storage_updates: IndexMap<ContractAddress, IndexMap<StorageKey, Felt>>,
    pub class_hash_to_compiled_class_hash: IndexMap<ClassHash, CompiledClassHash>,
}
``` [1](#0-0) 

The upstream `StateMaps` struct does carry `declared_contracts: HashMap<ClassHash, bool>` (populated by deprecated `DeclareV0`/`DeclareV1` transactions), but `CommitmentStateDiff::from(StateMaps)` silently discards it. [2](#0-1) 

**The conversion that propagates the omission:**

`ThinStateDiff::from(CommitmentStateDiff)` hardcodes `deprecated_declared_classes: Vec::new()`:

```rust
impl From<CommitmentStateDiff> for ThinStateDiff {
    fn from(commitment_state_diff: CommitmentStateDiff) -> Self {
        Self {
            deployed_contracts: commitment_state_diff.address_to_class_hash,
            storage_diffs: commitment_state_diff.storage_updates,
            class_hash_to_compiled_class_hash: commitment_state_diff
                .class_hash_to_compiled_class_hash,
            nonces: commitment_state_diff.address_to_nonce,
            // TODO(AlonH): Remove this when the structure of storage diffs changes.
            deprecated_declared_classes: Vec::new(),
        }
    }
}
``` [3](#0-2) 

**Where the corrupted value enters the commitment pipeline:**

`BlockExecutionArtifacts::new()` calls `calculate_block_commitments()` with this truncated `ThinStateDiff`:

```rust
let (header_commitments, measurements) = calculate_block_commitments(
    &transactions_data,
    ThinStateDiff::from(commitment_state_diff.clone()),  // deprecated_declared_classes = []
    l1_da_mode,
    &block_info.starknet_version,
)
.await;
``` [4](#0-3) 

**Two commitment values are corrupted:**

1. `state_diff_commitment` — `calculate_state_diff_hash()` explicitly chains `deprecated_declared_classes` into the Poseidon hash. With an empty list it chains `0` (count) instead of the real count and class hashes:

```rust
hash_chain = chain_deprecated_declared_classes(
    &state_diff.deprecated_declared_classes, hash_chain);
``` [5](#0-4) 

2. `concatenated_counts` — `calculate_block_commitments()` calls `state_diff.len()` to get `state_diff_length`, which counts `deprecated_declared_classes.len()`. With an empty list the count is too low:

```rust
let concatenated_counts = concat_counts(
    transactions_data.len(),
    event_leaf_elements.len(),
    state_diff.len(),   // undercounts: deprecated_declared_classes missing
    l1_da_mode,
);
``` [6](#0-5) 

`ThinStateDiff::len()` explicitly counts `deprecated_declared_classes`:

```rust
result += self.deprecated_declared_classes.len();
``` [7](#0-6) 

**Both corrupted values are chained into the block hash:**

```rust
.chain(&block_commitments.state_diff_commitment.0.0)
...
.chain(&block_commitments.concatenated_counts)
``` [8](#0-7) 

The `PartialBlockHash` used for consensus agreement is derived from the same `PartialBlockHashComponents`: [9](#0-8) 

### Impact Explanation

For any block containing a deprecated (Cairo 0) class declaration:

- The `state_diff_commitment` stored in the block header is wrong (missing deprecated class hashes from the Poseidon chain).
- The `state_diff_length` packed into `concatenated_counts` is wrong (too low by the number of deprecated declarations).
- The `PartialBlockHash` agreed upon by consensus is wrong.
- The final block hash is wrong.
- The SNOS/prover, which independently computes the correct state diff hash from the full state diff, will disagree with the sequencer's stored commitment, causing proof verification failure.

This matches the "Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input" Critical impact and the "Commitment construction" audit pivot.

### Likelihood Explanation

Deprecated (Cairo 0) class declarations via `DeclareTransactionV0V1` remain valid Starknet transactions. The blockifier still processes them (the `declared_contracts` field in `StateMaps` is populated). The bug is triggered by any such transaction reaching a sequenced block. The TODO comment (`// TODO(AlonH): Remove this when the structure of storage diffs changes`) confirms the omission is known but not yet fixed, and no compensating check exists anywhere in the commitment path.

### Recommendation

`CommitmentStateDiff` should be extended with a `deprecated_declared_classes: Vec<ClassHash>` field, populated from `StateMaps::declared_contracts` (filtering entries where the value is `true`). The `From<CommitmentStateDiff> for ThinStateDiff` conversion should then propagate this field instead of hardcoding `Vec::new()`. Alternatively, `finalize_block` should reconstruct the deprecated class list from `StateMaps::declared_contracts` and attach it to `BlockExecutionSummary` so `BlockExecutionArtifacts::new()` can pass the complete `ThinStateDiff` to `calculate_block_commitments()`.

### Proof of Concept

1. Submit a `DeclareTransactionV0V1` (deprecated Cairo 0 class declaration) to the sequencer.
2. The blockifier executes it; `StateMaps::declared_contracts` is populated with `{class_hash: true}`.
3. `finalize_block` calls `state_diff.into()` → `CommitmentStateDiff::from(StateMaps)` — `declared_contracts` is dropped.
4. `BlockExecutionArtifacts::new()` calls `ThinStateDiff::from(commitment_state_diff.clone())` → `deprecated_declared_classes = []`.
5. `calculate_block_commitments()` computes:
   - `state_diff_commitment` = `Poseidon("STARKNET_STATE_DIFF0", ..., 0 /*count=0, wrong*/, ...)` — missing the class hash.
   - `concatenated_counts` encodes `state_diff_length` that is 1 too low.
6. `PartialBlockHash` and final block hash are derived from these wrong values.
7. The SNOS prover, given the full state diff (with the deprecated class), computes a different `state_diff_commitment` and rejects the proof.

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
