### Title
P2P State Diff Sync Validates Only Length, Not Commitment Hash — Malicious Peer Can Substitute Arbitrary State Diff - (File: `crates/apollo_p2p_sync/src/client/state_diff.rs`)

### Summary

The P2P sync client assembles a block's `ThinStateDiff` from peer-supplied `StateDiffChunk` messages and validates only that the total element count matches `header.state_diff_length`. It never verifies the assembled diff against `header.state_diff_commitment` (the Poseidon hash committed in the block header). A malicious peer can therefore serve a state diff of the correct length but with entirely different storage values, nonces, or class hashes. The node accepts and persists this forged diff, causing every downstream consumer — the committer (global state root), the Patricia trie, and RPC state queries — to operate on wrong authoritative state.

### Finding Description

`parse_data_for_block` in `crates/apollo_p2p_sync/src/client/state_diff.rs` reads the block header to obtain `state_diff_length` and loops until the accumulated chunk length equals that target:

```rust
let target_state_diff_len = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    ...
    .state_diff_length          // ← only the count is read
    .ok_or(...)?;

while current_state_diff_len < target_state_diff_len {
    ...
    current_state_diff_len += state_diff_chunk.len();
    unite_state_diffs(&mut result, state_diff_chunk)?;
}
// ← no call to calculate_state_diff_hash(&result) and compare
//   against header.state_diff_commitment
validate_deprecated_declared_classes_non_conflicting(&result)?;
Ok(Some((result, block_number)))
``` [1](#0-0) 

The same `get_block_header` call that supplies `state_diff_length` also carries `state_diff_commitment` (a `StateDiffCommitment` / Poseidon hash), which was committed by the proposer and stored during header sync. That field is never read in this function. [2](#0-1) 

`calculate_state_diff_hash` exists and is used correctly in the committer's optional `verify_state_diff_hash` path, but that path is gated by a config flag and operates on the batcher's own output — not on P2P-received diffs. [3](#0-2) 

After `parse_data_for_block` returns, `write_to_storage` calls `append_state_diff` directly with no further hash check:

```rust
storage_writer.begin_rw_txn()?.append_state_diff(self.1, self.0)?.commit()?;
``` [4](#0-3) 

The stored `ThinStateDiff` is then fed to the committer as the authoritative state diff for that block number, which uses it to update the Patricia trie and derive the global state root. [5](#0-4) 

### Impact Explanation

A malicious P2P peer that knows the `state_diff_length` for a target block (publicly available from the synced header) can craft `StateDiffChunk` messages whose total element count matches but whose content differs — e.g., replacing a storage slot value, substituting a class hash, or injecting a nonce. The victim node:

1. Stores the forged `ThinStateDiff` in its MDBX storage.
2. The committer reads this diff and computes a wrong global state root (Patricia trie root).
3. RPC endpoints (`starknet_getStorageAt`, `starknet_getNonce`, `starknet_getClassHashAt`, `starknet_getStateUpdate`) return wrong authoritative values for the affected block.
4. The wrong global state root propagates into `calculate_block_hash` as the `state_root` argument, producing a wrong block hash for any block built on top.

This matches **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value**, and potentially **Critical — Wrong state, receipt, event, L1 message, class hash, storage value** if the corrupted state is used as input to subsequent execution or proving.

### Likelihood Explanation

Any node reachable on the P2P network can act as a state diff peer. An attacker needs only to establish a peer connection (standard libp2p) and respond to a state diff query with well-formed but forged chunks. No privileged access, no consensus participation, and no cryptographic secret is required. The only constraint is that the forged diff must have the same total `len()` as the legitimate diff, which is trivially satisfied by choosing entries of the same cardinality.

### Recommendation

After assembling the full `ThinStateDiff` from chunks, compute its Poseidon commitment and compare it against the value stored in the block header:

```rust
let expected_commitment = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    ...
    .state_diff_commitment          // already stored from header sync
    .ok_or(...)?;

let actual_commitment = calculate_state_diff_hash(&result);
if actual_commitment != expected_commitment {
    return Err(ParseDataError::BadPeer(BadPeerError::InvalidStateDiffCommitment));
}
```

This mirrors the check already present in `Committer::commit_block_inner` and closes the gap between the length-only guard and the cryptographic commitment.

### Proof of Concept

1. **Setup**: Victim node has synced headers up to block N. Block N's header contains `state_diff_length = 3` and `state_diff_commitment = H` (the correct Poseidon hash).

2. **Attacker action**: Attacker peer responds to the victim's state diff query for block N with three `StateDiffChunk::ContractDiff` messages. Each chunk has `len() == 1`, so `current_state_diff_len` reaches `target_state_diff_len == 3`. However, the storage slot values, nonces, or class hashes differ from the legitimate diff.

3. **Victim processing**: `parse_data_for_block` exits the loop, calls `validate_deprecated_declared_classes_non_conflicting` (passes, since no deprecated classes), and returns `Ok(Some((forged_diff, block_number)))`.

4. **Storage write**: `write_to_storage` calls `append_state_diff(block_number, forged_diff)` — the forged diff is now the canonical state for block N.

5. **Downstream corruption**:
   - `starknet_getStorageAt(contract, key, block_N)` returns the attacker-chosen value.
   - The committer reads the forged diff, updates the Patricia trie with wrong leaves, and stores a wrong `global_root` for block N.
   - `calculate_block_hash` for block N+1 chains this wrong `global_root`, producing a wrong block hash. [6](#0-5) [7](#0-6) [8](#0-7)

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,

    let block_hash_version: BlockHashVersion =

    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
