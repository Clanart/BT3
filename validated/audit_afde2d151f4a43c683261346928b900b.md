### Title
Missing Duplicate-Key Guard in `batch_insert` Fast Path Silently Corrupts Merkle Tree — (`crates/chia-datalayer/src/merkle/blob.rs`)

### Summary

`MerkleBlob::batch_insert` has two distinct insertion paths. The bootstrap path (for the last two items, popped from the end of the input vector) calls `self.insert()`, which checks `block_status_cache.contains_key()` before writing. The bulk fast path (all remaining items, i.e. the first N-2 items in the input vector) calls `insert_entry_to_blob` directly, with **no duplicate-key check**. `insert_entry_to_blob` calls `add_leaf`, which calls `HashMap::insert` and silently overwrites the existing cache entry. The result is two leaf nodes with the same key in the blob, a corrupted `key_to_index` cache, and a wrong Merkle root hash. `batch_insert` returns `Ok(())`.

**Concrete execution trace** with input `[(key1,v1,h1), (key2,v2,h2), (key1,v3,h3)]` on an empty blob:

1. `leaf_count = 0 ≤ 1` → bootstrap loop pops from the **end**:
   - Pop → `(key1,v3,h3)` → `insert(key1,…)` → OK, leaf_count=1
   - Pop → `(key2,v2,h2)` → `insert(key2,…)` → OK, leaf_count=2
2. Remaining vector: `[(key1,v1,h1)]`
3. Fast-path loop: `(key1,v1,h1)` → `insert_entry_to_blob` directly → **no duplicate check**
   - A second leaf node with `key1` is written to the blob at a new index
   - `key_to_index[key1]` is silently overwritten with the new index; the old key1 leaf is now orphaned in the blob
4. `batch_insert` returns `Ok(())`

**Post-condition:** the blob contains two leaf nodes with `key1`; the cache is inconsistent; the Merkle root hash is wrong.

**`check_integrity` would detect this** — `check_just_integrity` iterates all nodes and verifies that `get_index_by_key(node.key)` returns the node's actual index: [6](#0-5) 

The orphaned key1 leaf's actual index will not match the cache entry (which was overwritten), producing `IntegrityKeyToIndexCacheIndex`. The leaf count from iteration (3) will also not match the cache length (2), producing `IntegrityKeyToIndexCacheLength`. However, `batch_insert` itself returns `Ok(())` with no indication of corruption.

### Impact Explanation

The Merkle tree root hash is wrong after the call. Any proof of inclusion or exclusion generated from the corrupted blob is invalid. Downstream DataLayer logic that trusts the root hash (e.g., delta computation, proof verification) operates on a corrupted state. This matches the allowed High impact: "DataLayer Merkle proof/blob/delta logic… corrupts tree roots."

### Likelihood Explanation

The Python binding `py_batch_insert` exposes this directly: [7](#0-6) 

Any caller — including DataLayer sync code that assembles a batch from external peer data — that passes a vector containing a duplicate key will trigger silent corruption. The bug is triggered by a simple 3-element input with one repeated key; no special privileges are required beyond the ability to supply input to `batch_insert`.

### Recommendation

Add a duplicate-key check at the top of the fast-path loop, mirroring the guard already present in `insert()`:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // ... existing insert_entry_to_blob logic
}
```

Alternatively, deduplicate the input vector before processing, or make `add_leaf` return an error if the key is already present.

### Proof of Concept

```rust
use chia_datalayer::{Hash, KeyId, MerkleBlob, ValueId};
use chia_protocol::Bytes32;

fn main() {
    let mut blob = MerkleBlob::new(vec![]).unwrap();

    let key1 = KeyId(1);
    let key2 = KeyId(2);
    let h1 = Hash(Bytes32::new([1u8; 32]));
    let h2 = Hash(Bytes32::new([2u8; 32]));
    let h3 = Hash(Bytes32::new([3u8; 32]));

    // 3 items: key1 appears at index 0 AND index 2 (duplicate)
    // pop() takes from the end, so key1@index2 and key2@index1 go through insert()
    // key1@index0 goes through the unguarded fast path
    let batch = vec![
        ((key1, ValueId(10)), h1),  // ← fast path, no duplicate check
        ((key2, ValueId(20)), h2),  // ← bootstrap via insert()
        ((key1, ValueId(30)), h3),  // ← bootstrap via insert(), succeeds first
    ];

    let result = blob.batch_insert(batch);
    assert!(result.is_ok(), "batch_insert returned error: {:?}", result);

    // Tree is now corrupted; check_integrity detects it
    blob.check_integrity_on_drop = false;
    let integrity = blob.check_integrity();
    assert!(integrity.is_err(), "expected integrity failure, got Ok");
    println!("Corruption confirmed: {:?}", integrity.unwrap_err());
}
```
