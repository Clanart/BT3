### Title
Integer Division Truncation in `NewDynamicFeeChecker` Allows Zero-Fee Cosmos SDK Transactions When `baseFee = 0` - (File: `ante/evm/fee_checker.go`)

### Summary

`NewDynamicFeeChecker` derives the per-gas fee cap for Cosmos SDK transactions via integer (truncating) division: `feeCap = fee / gas`. When the declared `fee` is smaller than `gas`, `feeCap` truncates to `0`. If the chain's EIP-1559 `baseFee` is also `0` (reachable when `MinGasPrice = 0` and the base fee has been driven down), the guard `feeCap >= baseFeeInt` passes trivially, `effectivePrice` resolves to `0`, and `effectiveFee = 0` is returned to `NewDeductFeeDecorator` — meaning the sender pays nothing.

### Finding Description

In `NewDynamicFeeChecker` the per-gas fee cap is computed with integer division:

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))   // line 83
baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)

if feeCap.LT(baseFeeInt) {                          // line 86
    return nil, 0, errorsmod.Wrapf(...)
}
``` [1](#0-0) 

`sdkmath.Int.Quo` performs truncating integer division. When `fee < gas`, the result is `0`. The guard on line 86 only rejects the transaction when `feeCap < baseFeeInt`. If `baseFee = 0`, the condition `0 < 0` is false, so the check passes.

The effective fee is then computed as:

```go
effectivePrice := sdkmath.NewIntFromBigInt(
    types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
effectiveFee := sdk.Coins{{Denom: denom, Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas))}}
``` [2](#0-1) 

`EffectiveGasPrice = min(baseFee + tipCap, feeCap)`. With `feeCap = 0`, the result is `0` regardless of `maxPriorityPrice`. Therefore `effectiveFee = 0 * gas = 0`.

`NewDeductFeeDecorator` deducts exactly `effectiveFee` from the sender's balance. With `effectiveFee = 0`, no tokens are deducted and the transaction executes for free.

**When does `baseFee = 0`?**

`CalculateBaseFee` enforces a lower bound of `MinGasPrice`:

```go
minGasPrice := params.MinGasPrice.TruncateInt().BigInt()
return ethermint.BigMax(x.Sub(parentBaseFee, baseFeeDelta), minGasPrice)
``` [3](#0-2) 

When `MinGasPrice = 0` (the default on many deployments), the base fee can be driven to `0` over successive empty blocks. Additionally, a chain genesis or governance parameter update can set `BaseFee = 0` directly. `NewDynamicFeeChecker` reads `feemarketParams` at ante-handler construction time:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
``` [4](#0-3) 

**Secondary impact (always present):** Even when `baseFee > 0`, `effectiveFee = (fee / gas) * gas ≤ fee`. The remainder `fee mod gas` is never deducted. A sender declaring `fee = baseFee * gas + (gas − 1)` pays only `baseFee * gas`, saving up to `gas − 1` tokens per transaction.

The `MinGasPriceDecorator` that precedes `NewDeductFeeDecorator` in the Cosmos ante chain short-circuits when `MinGasPrice = 0`:

```go
if minGasPrice.IsZero() || simulate {
    return next(ctx, tx, simulate)
}
``` [5](#0-4) 

So when both `baseFee = 0` and `MinGasPrice = 0`, no ante decorator catches the zero-fee transaction.

### Impact Explanation

A Cosmos SDK transaction (non-EVM, e.g., `MsgSend`, `MsgDelegate`, IBC relayer messages) routed through `newCosmosAnteHandler` with `DynamicFeeChecker = true` can be submitted with `fee = 1` and any `gas > 1`. The truncation yields `feeCap = 0`, `effectiveFee = 0`, and the transaction commits with zero fee deducted. Validators and the fee-collector module receive nothing. This matches the "valid user funds/fees to be mis-accounted" High impact class and, at `baseFee = 0`, enables unbounded fee-free spam of Cosmos-layer messages.

### Likelihood Explanation

- `MinGasPrice = 0` is the default in many Ethermint-based deployments.
- With `MinGasPrice = 0`, `CalculateBaseFee` can drive `baseFee` to `0` after sustained low-traffic periods.
- The attacker needs only to craft a standard Cosmos SDK transaction with `fee < gas` (e.g., `fee = 1`, `gas = 21000`) — no special privileges required.
- The `newCosmosAnteHandler` path is used for all non-EVM Cosmos transactions when `DynamicFeeChecker = true`.

### Recommendation

1. **Reject zero `feeCap` explicitly:** After computing `feeCap`, add a guard:
   ```go
   if feeCap.IsZero() && !baseFeeInt.IsZero() {
       return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, "feeCap rounds to zero")
   }
   ```
2. **Use ceiling division for `feeCap`:** Replace `fee.Quo(gas)` with `(fee + gas - 1) / gas` so that any non-zero fee always produces a non-zero `feeCap` when `fee > 0`.
3. **Deduct the full declared `fee`, not `effectiveFee`:** Return `feeCoins` (the declared fee) rather than `effectiveFee` to prevent under-collection, or explicitly refund the difference `fee − effectiveFee` after execution.
4. **Enforce `MinGasPrice > 0`** in param validation to prevent `baseFee` from reaching `0`.

### Proof of Concept

**Setup:** Chain with `NoBaseFee = false`, `BaseFee = 0`, `MinGasPrice = 0`, `DynamicFeeChecker = true`.

**Attack:**
```
gas  = 21000
fee  = [1 aphoton]   // fee < gas → feeCap = 1/21000 = 0
```

**Trace through `NewDynamicFeeChecker`:**
1. `feeCap = 1.Quo(21000) = 0` [6](#0-5) 
2. `baseFeeInt = 0`
3. `feeCap.LT(baseFeeInt)` → `0 < 0` → **false** → check passes [7](#0-6) 
4. `effectivePrice = EffectiveGasPrice(0, 0, MaxInt64) = min(0+MaxInt64, 0) = 0`
5. `effectiveFee = 0 * 21000 = 0` [8](#0-7) 
6. `NewDeductFeeDecorator` deducts `0` tokens from sender.

**Result:** Cosmos SDK transaction executes and commits with zero fee paid. Repeated indefinitely, the attacker can spam the chain at no cost.
