### Title
`SpiceEndorsementStats` accumulator never seeded at state-sync boundary block, breaking all post-sync block processing in a spice epoch — (`core/store/src/columns.rs`, `chain/chain/src/spice/core.rs`)

---

### Summary

`DBCol::SpiceEndorsementStats` is a per-block running accumulator of chunk endorsement stats, where each block's value is derived from its parent's stored entry. After state sync, the sync-boundary block's entry is never written to `DBCol::SpiceEndorsementStats`. When the first post-sync block is processed, `compute_spice_endorsement_stats` attempts to read the parent's accumulator via `get_spice_endorsement_stats`. Because the entry is absent, the function returns a hard error, causing block processing to fail for every block in the same epoch after the sync point. The running endorsement accumulator is never initialized, so all validator endorsement stats for the epoch remain at zero — exactly the FairSide "conviction score never bootstrapped" pattern.

---

### Finding Description

`DBCol::SpiceEndorsementStats` stores, for every spice block, the running per-epoch accumulated `Vec<SpiceChunkEndorsementStats>` indexed by validator id. The column's own documentation acknowledges the gap:

```
// TODO(spice): each block's value is derived from its parent's, so this must
// be seeded at the state-sync boundary block (same as UncertifiedChunks) once
// spice state sync exists, otherwise the first post-sync block can't compute it.
``` [1](#0-0) 

Every normally-processed spice block writes its accumulator via `record_spice_endorsement_stats_for_block`: [2](#0-1) 

That function calls `compute_spice_endorsement_stats`, which reads the **parent block's** stored accumulator when the parent is in the same epoch:

```rust
let prev_stats = if !prev_header.is_genesis() && prev_header.epoch_id() == epoch_id {
    Some(get_spice_endorsement_stats(chain_store, prev_hash)?)
} else {
    None
};
``` [3](#0-2) 

`get_spice_endorsement_stats` treats a missing DB entry as a hard error:

```rust
let Some(stats) =
    chain_store.store_ref().get_ser(DBCol::spice_endorsement_stats(), block_hash.as_ref())
else {
    debug_assert!(false, "spice blocks in store should always have endorsement stats present");
    return Err(Error::Other(format!("missing spice endorsement stats for {block_hash}")));
};
``` [4](#0-3) 

`reset_heads_post_state_sync` sets the chain head to the sync-boundary block but never writes a `SpiceEndorsementStats` seed entry for it: [5](#0-4) 

**Consequence**: the first block after the sync point has `prev_header.epoch_id() == epoch_id` (both are in the same epoch), so `get_spice_endorsement_stats` is called for the sync-boundary block, finds no entry, and returns `Err(...)`. This error propagates through `record_spice_endorsement_stats_for_block` → `chain_update.rs` → block processing, causing every block in the epoch after the sync point to be rejected. The per-epoch endorsement accumulator is never initialized — the exact "conviction score never bootstrapped" failure mode from FairSide.

---

### Impact Explanation

Any node that performs state sync while the `protocol_feature_spice` feature is active enters a state where it cannot process a single block in the synced epoch. The running endorsement accumulator for the epoch is permanently zero. At epoch end, `finalize_epoch` reads `spice_chunk_endorsement_stats` from the last block's header (which is computed from the same broken accumulator), producing all-zero endorsement stats for every validator. This silently corrupts validator reward and kickout decisions for the entire epoch: validators who endorsed every chunk appear to have endorsed none, and may be incorrectly kicked out or denied rewards.

---

### Likelihood Explanation

The bug is reachable whenever `protocol_feature_spice` is compiled in and a node performs state sync (far-horizon sync) while the network is running a spice protocol version. The state sync path (`StateSync::run` → `reset_heads_post_state_sync`) is the standard catch-up path for any node that falls behind by more than the epoch-sync horizon. The missing seed is not conditional on any operator action; it is a structural omission in `reset_heads_post_state_sync`.

---

### Recommendation

In `reset_heads_post_state_sync` (or in the state-sync finalization path), after saving the sync-boundary block, write a zero-initialized `Vec<SpiceChunkEndorsementStats>` seed entry to `DBCol::SpiceEndorsementStats` for the sync-boundary block hash. This mirrors how `UncertifiedChunks` must also be seeded at the same boundary. The seed value should be `vec![SpiceChunkEndorsementStats::default(); num_validators]`, consistent with `fold_endorsement_stats` when `prev_stats` is `None` (epoch boundary reset).

---

### Proof of Concept

1. Enable `protocol_feature_spice` and run a 4-validator network past the epoch-sync horizon.
2. Add a fresh node; it enters the far-horizon sync pipeline and calls `reset_heads_post_state_sync` with `sync_hash` = first block of the current spice epoch.
3. The node attempts to process the second block of the epoch. `record_spice_endorsement_stats_for_block` is called; it calls `compute_spice_endorsement_stats(prev_hash = sync_hash)`.
4. `prev_header.epoch_id() == epoch_id` is `true` (both blocks are in the same epoch), so `get_spice_endorsement_stats(sync_hash)` is called.
5. `DBCol::SpiceEndorsementStats` has no entry for `sync_hash` → returns `Err("missing spice endorsement stats for {sync_hash}")`.
6. Block processing fails; the node cannot advance its head. All subsequent blocks in the epoch are also rejected. At epoch end, the last block's `spice_chunk_endorsement_stats` header field is never written, so `finalize_epoch` reads all-zero endorsement stats and incorrectly computes rewards and kickouts for every validator. fn get_spice_endorsement_stats(
    chain_store: &ChainStoreAdapter,
    block_hash: &CryptoHash,
) -> Result<Vec<SpiceChunkEndorsementStats>, Error> {
    let block = chain_store.get_block(block_hash)?;
    if block.header().is_genesis() || !block.is_spice_block() {
        return Ok(vec![]);
    }
    let Some(stats) =
        chain_store.store_ref().get_ser(DBCol::spice_endorsement_stats(), block_hash.as_ref())
    else {
        debug_assert!(false, "spice blocks in store should always have endorsement stats present");
        return Err(Error::Other(format!("missing spice endorsement stats for {block_hash}")));
    };
    Ok(stats)
}

pub fn compute_spice_endorsement_stats(
    chain_store: &ChainStoreAdapter,
    epoch_manager: &dyn EpochManagerAdapter,
    epoch_id: &EpochId,
    prev_hash: &CryptoHash,
) -> Result<Vec<SpiceChunkEndorsementStats>, Error> {
    let epoch_info = epoch_manager.get_epoch_info(epoch_id)?;
    let num_validators = epoch_info.validators_iter().len();
    let contribution =
        endorsement_contribution_of_block(chain_store, epoch_manager, &epoch_info, prev_hash)?;

    // Carry the previous block's accumulator only within the same epoch; at the
    // epoch boundary (and genesis) the per-epoch accumulator resets.
    let prev_header = chain_store.get_block_header(prev_hash)?;
    let prev_stats = if !prev_header.is_genesis() && prev_header.epoch_id() == epoch_id {
        Some(get_spice_endorsement_stats(chain_store, prev_hash)?)
    } else {
        None
    };
    fold_endorsement_stats(num_validators, prev_stats.as_deref(), &contribution)
}

```rust
    pub fn reset_heads_post_state_sync(
        &mut self,
        sync_hash: CryptoHash,
        block_processing_artifacts: &mut BlockProcessingArtifact,
        apply_chunks_done_sender: Option<ApplyChunksDoneSender>,
    ) -> Result<(), Error> {
        // Get header we were syncing into.
        let header = self.get_block_header(&sync_hash)?;
        let prev_hash = *header.prev_hash();
        let prev_block = self.get_block(&prev_hash)?;

        // Check which blocks were downloaded during state sync
        // and set the tail and chunk tail accordingly
        let tail_block_hash = self
            .get_extra_sync_block_hashes(&prev_hash)
            .into_iter()
            .min_by_key(|block_hash| self.get_block_header(block_hash).unwrap().height())
            .unwrap_or(prev_hash);
        let tail_block = self.get_block(&tail_block_hash)?;

        let new_tail = tail_block.header().height();
        let new_chunk_tail = tail_block.chunks().min_height_included().unwrap();
        tracing::debug!(target: "sync", ?new_tail, ?new_chunk_tail, "adjusting tail for sync blocks");

        let tip = Tip::from_header(prev_block.header());
        let final_head = Tip::from_header(self.genesis.header());
        // Update related heads now.
        let mut chain_store_update = self.mut_chain_store().store_update();
        chain_store_update.save_body_head(&tip)?;
        // Reset final head to genesis since at this point we don't have the last final block.
        chain_store_update.save_final_head(&final_head)?;
        // New Tail can not be earlier than `prev_block.header.inner_lite.height`
        chain_store_update.update_tail(new_tail);
        // New Chunk Tail can not be earlier than minimum of height_created in Block `prev_block`
        chain_store_update.update_chunk_tail(new_chunk_tail);
        chain_store_update.commit()?;

        // Check if there are any orphans unlocked by this state sync.
        // We can't fail beyond this point because the caller will not process accepted blocks
        //    and the blocks with missing chunks if this method fails
        self.check_orphans(prev_hash, block_processing_artifacts, apply_chunks_done_sender);
        Ok(())
    }
