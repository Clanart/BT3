### Title
Stale Parameter Snapshot in `newCosmosAnteHandler` Causes Permanent Fee-Market Bypass for Cosmos Transactions — (`evmd/ante/handler_options.go`)

---

### Summary

`newCosmosAnteHandler` captures `evmParams` and `feemarketParams` as local value-type snapshots at app-startup time and passes pointers to those snapshots into `NewDynamicFeeChecker` and `MinGasPriceDecorator`. Because the fee-market base fee is updated every block (via `feemarket.BeginBlock`) and `MinGasPrice` can be changed by governance, the Cosmos ante handler permanently uses genesis-time values for all fee validation of non-EVM (Cosmos SDK) transactions. Any user can submit a Cosmos transaction with fees calibrated to the genesis base fee, bypassing the live EIP-1559 base fee requirement for the entire lifetime of the node.

---

### Finding Description

Both `evmParams` and `feemarketParams` are local variables allocated on the stack of `newCosmosAnteHandler`. Pointers to them are passed into `NewDynamicFeeChecker` and `MinGasPriceDecorator`, which capture those pointers in their closures/structs. The function returns a `sdk.AnteHandler` closure; the local variables are promoted to the heap but are **never updated again**.

**The live base fee is updated every block but never reaches the Cosmos ante handler** [4](#0-3) 

`feemarket.BeginBlock` calls `k.SetBaseFee(ctx, baseFee)` which writes the new base fee into the KV store via `SetParams`. The Cosmos ante handler's snapshot is never refreshed.

**Contrast: EVM ante handler correctly fetches live params per transaction** [5](#0-4) 

`newEthAnteHandler` calls `options.EvmKeeper.EVMBlockConfig(ctx, ...)` on every invocation, which reads fresh params from the KV store (or the per-block object-store cache). Cosmos txs receive no equivalent refresh.

---

### Impact Explanation

This is a **High** impact finding matching: *"EVM state transition, gas refund, fee market, ante handler, mempool, or proposal handling bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

1. **Base fee bypass for Cosmos txs**: The base fee changes every block. After even one block of high usage, the live base fee exceeds the genesis snapshot. Any user can submit a Cosmos SDK transaction with fees calibrated to the genesis base fee and pass the `DynamicFeeChecker`, permanently under-paying fees relative to the current EIP-1559 requirement.

2. **`MinGasPrice` governance bypass**: If governance raises `MinGasPrice`, the `MinGasPriceDecorator` for Cosmos txs still enforces the old lower value. Transactions that should be rejected are accepted.

3. **`NoBaseFee` governance flip**: If governance sets `NoBaseFee = true`, the `DynamicFeeChecker` still enforces the old base fee, causing valid transactions to be rejected (liveness impact). The reverse (enabling base fee) also fails to take effect.

---

### Likelihood Explanation

- The base fee changes every block by design; the desynchronization begins at block 2 and grows continuously.
- Any unprivileged user submitting a Cosmos SDK transaction (governance votes, IBC relays, staking operations) is affected.
- No special privileges, key compromise, or validator collusion required.
- The `newCosmosAnteHandler` is called once at app startup from `evmd/app.go`; there is no refresh mechanism.

---

### Recommendation

Refactor `newCosmosAnteHandler` to fetch live params per transaction, mirroring `newEthAnteHandler`:

```go
func newCosmosAnteHandler(options HandlerOptions, extra ...sdk.AnteDecorator) sdk.AnteHandler {
    return func(ctx sdk.Context, tx sdk.Tx, simulate bool) (sdk.Context, error) {
        blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
        if err != nil {
            return ctx, err
        }
        evmParams := &blockCfg.Params
        feemarketParams := &blockCfg.FeeMarketParams
        evmDenom := evmParams.EvmDenom
        chainID := options.EvmKeeper.ChainID()
        ethCfg := evmParams.ChainConfig.EthereumConfig(chainID)

        var txFeeChecker ante.TxFeeChecker
        if options.DynamicFeeChecker {
            txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, evmParams, feemarketParams)
        }
        // build and run decorators with live params ...
    }
}
```

This ensures `feemarketParams.BaseFee`, `feemarketParams.MinGasPrice`, and `feemarketParams.NoBaseFee` are always the current block's values, consistent with how EVM transactions are validated.

---

### Proof of Concept

1. Chain starts with genesis base fee = 1,000,000,000 (1 Gwei).
2. Blocks fill up; after N blocks the live base fee rises to 10,000,000,000 (10 Gwei).
3. User submits a Cosmos governance `MsgVote` with fee = `1_000_000_000 * gasLimit` (1 Gwei effective price).
4. `newCosmosAnteHandler` runs `NewDynamicFeeChecker` which reads `feemarketParams.BaseFee` = 1 Gwei (genesis snapshot) → fee check passes.
5. The same transaction would be rejected by `newEthAnteHandler` (which reads the live 10 Gwei base fee from `EVMBlockConfig`).
6. The transaction is included in the block, paying 10× less than the current base fee requires. [6](#0-5) [2](#0-1) [3](#0-2) [4](#0-3) [7](#0-6)
