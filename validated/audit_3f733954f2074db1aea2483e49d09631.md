### Title
Missing Declared-Class Validation in `execute_replace_class` Enables Permanent Fund Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in `syscall_impls.cairo` accepts any arbitrary `class_hash` value without verifying that it corresponds to a previously declared contract class. An explicit `TODO` comment in the code acknowledges this missing check. Because the OS commits the unvalidated class hash to state, any contract that calls `replace_class` with an undeclared hash becomes permanently uncallable, freezing all funds it holds.

---

### Finding Description

In `execute_replace_class` (lines 877–916 of `syscall_impls.cairo`), the function reads the caller-supplied `class_hash` directly from the syscall request and writes it into `contract_state_changes` with no validation:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The `TODO` comment is the developers' own acknowledgement that the check is absent. [2](#0-1) 

When a subsequent transaction attempts to call the now-poisoned contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // ← the undeclared hash
);
// compiled_class_hash == 0 (uninitialized default)

let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,           // ← key == 0, not found → assertion failure
);
``` [3](#0-2) 

`find_element` is a Cairo built-in that asserts the element exists; if no compiled class with hash `0` is present, the OS execution aborts. The sequencer will therefore reject every future call to the contract, making its funds permanently inaccessible.

The analogy to M-02 is direct: just as `extcodesize` was a bypassable guard that let an attacker avoid the blocklist, the absent class-hash guard in `execute_replace_class` lets a caller bypass the implicit protocol invariant that only declared classes may be installed.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value and the block is finalized:

- Every future call to the contract causes the OS to fail to locate a compiled class, making the call impossible to include in any valid block.
- Any ERC-20 tokens, ETH-bridged assets, or protocol-controlled value stored in the contract's storage become permanently inaccessible.
- There is no recovery path: the contract cannot be called to self-rescue, and `replace_class` cannot be called again because the contract is already uncallable.

---

### Likelihood Explanation

The attack is reachable by any unprivileged transaction sender via two realistic paths:

1. **Direct self-inflicted freeze (attacker-controlled contract):** An attacker deploys a contract that holds third-party deposits (e.g., a fake yield vault), collects funds, then calls `replace_class(arbitrary_undeclared_hash)` to freeze depositors' funds while retaining any already-withdrawn value.

2. **Exploit of a vulnerable upgrade function:** Many upgradeable contracts expose a function such as `fn upgrade(new_class: ClassHash)` with insufficient access control. An attacker supplies an undeclared hash, triggering the missing OS-level guard and permanently bricking the contract.

The explicit `TODO` comment confirms the developers know the check is absent, meaning the window of exposure is open until the fix is deployed.

---

### Recommendation

Before committing the new class hash to state, verify it is declared by asserting a non-zero compiled class hash exists in `contract_class_changes`:

```cairo
// Inside execute_replace_class, before dict_update:
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the validation already performed implicitly in `execute_entry_point` and closes the gap acknowledged by the TODO.

### Proof of Concept

```
1. Attacker deploys VaultContract (class hash C_valid, declared).
   VaultContract exposes:
     fn freeze_vault(new_hash: felt) { replace_class_syscall(new_hash); }

2. Users deposit funds into VaultContract.

3. Attacker sends invoke tx:
     VaultContract.freeze_vault(0xdeadbeef)
   where 0xdeadbeef is never declared.

4. OS executes execute_replace_class:
   - Reads class_hash = 0xdeadbeef from request.
   - Skips the missing declared-class check (TODO).
   - Writes StateEntry{class_hash=0xdeadbeef, ...} to contract_state_changes.
   - Block is finalized; state is committed on-chain.

5. Any subsequent call to VaultContract:
   - execute_entry_point reads class_hash = 0xdeadbeef.
   - dict_read on contract_class_changes returns 0 (undeclared).
   - find_element(key=0) fails → OS execution aborts.
   - Sequencer rejects the tx; it can never be included.

6. All user funds in VaultContract are permanently frozen.
```
