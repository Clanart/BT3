### Title
`collect_source_receipt_proofs` Embeds Cryptographically Invalid Merkle Proofs in `ChunkStateWitness` at Resharding Boundaries — (`File: chain/chain/src/stateless_validation/state_witness.rs`)

---

### Summary

`collect_source_receipt_proofs` calls `get_incoming_receipts_for_shard` to build the `source_receipt_proofs` map embedded in a `ChunkStateWitness`. At a resharding epoch boundary, `get_incoming_receipts_for_shard` silently removes receipts destined for the sibling child shard from the `ReceiptProof` list, but the Merkle path inside each proof was computed against the **full, unfiltered** receipt set committed in `prev_outgoing_receipts_root`. The resulting proof is cryptographically invalid: it cannot verify against the committed root. Every chunk validator that receives the witness will reject it via `validate_receipt_proof → verify_against_receipt_root`, producing `InvalidChunkStateWitness`. No endorsements are emitted, and the first new chunk of a child shard after resharding cannot be included in a block.

The codebase itself acknowledges this in a `TODO(resharding)` comment at the exact call site.

---

### Finding Description

**Root cause — the caller strips receipts but keeps the proof computed over the full set.**

`collect_source_receipt_proofs` in `chain/chain/src/stateless_validation/state_witness.rs` calls:

```rust
let incoming_receipt_proofs = get_incoming_receipts_for_shard(
    &self,
    epoch_manager,
    prev_chunk_header.shard_id(),
    &shard_layout,
    *prev_chunk_original_block.hash(),
    prev_prev_chunk_header.height_included(),
    ReceiptFilter::All,   // <-- passes All, but crossing a resharding boundary
                          //     causes the function to silently drop receipts
)?;
``` [1](#0-0) 

`get_incoming_receipts_for_shard` detects a shard-layout change when it crosses the epoch boundary and switches `current_shard_id` to the parent shard. The `ReceiptProof` objects stored on disk for the parent shard contain receipts for **both** child shards. When `ReceiptFilter::All` is used, the function returns those proofs as-is, but the receipts inside them include entries destined for the sibling child. The Merkle path in each `ReceiptProof` was computed by the block producer over the full outgoing receipt set (both children), and that root is committed in `prev_outgoing_receipts_root` of the chunk header. [2](#0-1) 

**Downstream validation rejects the proof.**

`pre_validate_chunk_state_witness` calls `validate_source_receipt_proofs`, which calls `validate_receipt_proof` for every proof in the witness:

```rust
if !receipt_proof.verify_against_receipt_root(outgoing_receipts_root) {
    return Err(Error::InvalidChunkStateWitness(format!(
        "Receipt proof for chunk {:?} has invalid merkle path, doesn't match outgoing receipts root",
        ...
    )));
}
``` [3](#0-2) 

Because the receipts list in the proof was filtered (some receipts removed) but the Merkle path was not recomputed, `verify_against_receipt_root` returns `false` and the witness is rejected.

**The TODO comment in the codebase confirms the broken invariant:**

```
/// TODO(resharding): `get_incoming_receipts_for_shard` generates invalid proofs on resharding
/// boundaries, because it removes the receipts that target the other half of a split shard,
/// which makes the proof invalid. We need to collect the original proof and later, after verification,
/// filter it to remove the receipts that were meant for the other half of the split shard.
``` [4](#0-3) 

---

### Impact Explanation

The `ChunkStateWitness` produced by the chunk producer for the **first new chunk of a child shard** after a resharding event will contain `source_receipt_proofs` with Merkle paths that do not verify against the committed `prev_outgoing_receipts_root`. Every chunk validator will call `pre_validate_chunk_state_witness` → `validate_source_receipt_proofs` → `validate_receipt_proof` and return `InvalidChunkStateWitness`. No endorsements are produced. The chunk producer cannot collect enough endorsements to include the chunk in the block. This is a **liveness failure** at every resharding boundary for any child shard that has cross-shard incoming receipts from the pre-resharding epoch.

**Severity: High** — deterministic liveness failure at a protocol-defined event (resharding), affecting all child shards with incoming receipts.

---

### Likelihood Explanation

This is triggered **deterministically** at every resharding epoch boundary. No attacker action is required. Any shard split where the parent shard received cross-shard receipts in the epoch before resharding will trigger the bug. Given that resharding is a core protocol feature and cross-shard receipts are routine, this fires on every real resharding event.

---

### Recommendation

Follow the fix described in the TODO comment: in `collect_source_receipt_proofs`, collect the **original, unfiltered** `ReceiptProof` (with the full Merkle path intact) and embed it in the witness. After `validate_receipt_proof` verifies the proof against the committed root, apply the shard-layout filter to remove receipts destined for the sibling child shard before computing `applied_receipts_hash` and passing receipts to the runtime. This preserves the commitment invariant (proof verifies against the committed root) while still applying only the correct subset of receipts.

Concretely:
- Pass `ReceiptFilter::All` to `get_incoming_receipts_for_shard` but do **not** strip receipts from the `ReceiptProof` before embedding in the witness.
- In `validate_source_receipt_proofs`, after `validate_receipt_proof` succeeds, apply `filter_incoming_receipts_for_shard` to extract only the receipts belonging to the target child shard before adding them to `receipts_to_apply`. [5](#0-4) 

---

### Proof of Concept

1. Run a network with resharding enabled (e.g., `slow_test_resharding_v3` with stateless validation active).
2. At the resharding epoch boundary, observe the chunk producer for a child shard generating a `ChunkStateWitness`.
3. The witness's `source_receipt_proofs` will contain `ReceiptProof` entries where the receipts list has been filtered (sibling-destined receipts removed) but the Merkle path was computed over the full set.
4. Every chunk validator calls `validate_receipt_proof` → `verify_against_receipt_root` → returns `false`.
5. All validators emit `InvalidChunkStateWitness`; no endorsements are produced; the chunk is not included.

fn validate_source_receipt_proofs(
    epoch_manager: &dyn EpochManagerAdapter,
    source_receipt_proofs: &HashMap<ChunkHash, ReceiptProof>,
    receipt_source_blocks: &[Arc<Block>],
    target_shard_layout: ShardLayout,
    target_chunk_shard_id: ShardId,
) -> Result<Vec<Receipt>, Error> {
    if receipt_source_blocks.iter().any(|block| block.header().is_genesis()) {
        if receipt_source_blocks.len() != 1 {
            return Err(Error::Other(
                "Invalid chain state: receipt_source_blocks should not have any blocks alongside genesis".to_owned()
            ));
        }
        if !source_receipt_proofs.is_empty() {
            return Err(Error::InvalidChunkStateWitness(format!(
                "genesis source_receipt_proofs should be empty, actual len is {}",
                source_receipt_proofs.len()
            )));
        }
        return Ok(vec![]);
    }

    let mut receipts_to_apply = Vec::new();
    let mut expected_proofs_len = 0;
    let mut current_target_shard_id = target_chunk_shard_id;

    // Iterate over blocks between last_chunk_block (inclusive) and last_last_chunk_block (exclusive),
    // from the newest blocks to the oldest.
    for block in receipt_source_blocks {
        // Collect all receipts coming from this block.
        let mut block_receipt_proofs = Vec::new();

        for chunk in block.chunks().iter_new() {
            // Collect receipts coming from this chunk and validate that they are correct.
            let Some(receipt_proof) = source_receipt_proofs.get(&chunk.chunk_hash()) else {
                return Err(Error::InvalidChunkStateWitness(format!(
                    "Missing source receipt proof for chunk {:?}",
                    chunk.chunk_hash()
                )));
            };

            validate_receipt_proof(
                receipt_proof,
                chunk,
                current_target_shard_id,
                *chunk.prev_outgoing_receipts_root(),
            )?;

            expected_proofs_len += 1;
            block_receipt_proofs.push(receipt_proof.clone());
        }

        block_receipt_proofs = filter_incoming_receipts_for_shard(
            &target_shard_layout,
            target_chunk_shard_id,
            Arc::new(block_receipt_proofs),
        )?;

        // Arrange the receipts in the order in which they should be applied.
        shuffle_receipt_proofs(&mut block_receipt_proofs, get_receipts_shuffle_salt(block));
        for proof in block_receipt_proofs {
            receipts_to_apply.extend(proof.0.iter().cloned());
        }

        current_target_shard_id = epoch_manager
            .get_prev_shard_id_from_prev_hash(block.header().prev_hash(), current_target_shard_id)?
            .1;
    }

    // Check that there are no extraneous proofs in source_receipt_proofs.
    if source_receipt_proofs.len() != expected_proofs_len {
        return Err(Error::InvalidChunkStateWitness(format!(
            "source_receipt_proofs contains too many proofs. Expected {} proofs, found {}",
            expected_proofs_len,
            source_receipt_proofs.len()
        )));
    }
    Ok(receipts_to_apply)
}
