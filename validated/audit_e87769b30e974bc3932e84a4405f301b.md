### Title
Unchecked Upper Bound on `max_price_per_unit` Causes Felt Arithmetic Overflow in Fee Computation — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

`compute_max_possible_fee` performs unchecked felt-field multiplication of user-controlled `max_price_per_unit` values against `max_amount`. Because `max_price_per_unit` has no upper-bound constraint (only `assert_nn`, i.e., ≥ 0), an attacker can craft a V3 transaction whose resource-bounds fields cause the fee sum to wrap to 0 modulo the Cairo field prime. When `max_fee == 0`, `charge_fee` returns early and no fee is charged, allowing free execution of transactions.

---

### Finding Description

In `pack_resource_bounds`, only `max_amount` is bounded (`≤ 2^64 - 1`); `max_price_per_unit` is only checked to be non-negative:

```cairo
func pack_resource_bounds{range_check_ptr}(resource_bounds: ResourceBounds) -> felt {
    assert_nn_le(resource_bounds.max_amount, 2 ** 64 - 1);
    assert_nn(resource_bounds.max_price_per_unit);   // NO upper bound
    ...
}
``` [1](#0-0) 

`compute_max_possible_fee` then multiplies these unbounded values in felt arithmetic (mod the Cairo prime P ≈ 2^251):

```cairo
return l1_gas_bounds.max_amount * l1_gas_bounds.max_price_per_unit + l2_gas_bounds.max_amount *
    (l2_gas_bounds.max_price_per_unit + tx_info.tip) + l1_data_gas_bounds.max_amount *
    l1_data_gas_bounds.max_price_per_unit;
``` [2](#0-1) 

`charge_fee` then short-circuits on `max_fee == 0`:

```cairo
let max_fee = compute_max_possible_fee(tx_info=tx_info);
if (max_fee == 0) {
    return ();   // fee charging skipped entirely
}
...
assert_nn_le(calldata.amount.low, max_fee);
``` [3](#0-2) 

---

### Impact Explanation

**Direct loss of funds.** When `compute_max_possible_fee` returns 0 due to overflow, `charge_fee` returns immediately without executing the ERC-20 transfer to the sequencer. The fee recipient receives nothing for executing the transaction. Because the OS proof is still valid (no assertion fails), the L1 verifier accepts the block, permanently encoding the fee-free execution into the canonical chain state.

---

### Likelihood Explanation

The attacker is an unprivileged transaction sender who fully controls all resource-bounds fields of a V3 transaction. The overflow is deterministic and constructible with elementary modular arithmetic. The OS itself does not prevent a sequencer from including such a transaction; the only barrier is off-chain sequencer policy. A colluding or misconfigured sequencer would include it, and the OS proof would be accepted by L1 without error.

---

### Recommendation

Add an explicit upper-bound check on `max_price_per_unit` in `pack_resource_bounds`, consistent with the StarkNet transaction specification (e.g., `≤ 2^128 - 1`):

```cairo
assert_nn_le(resource_bounds.max_price_per_unit, 2 ** 128 - 1);
```

Additionally, `compute_max_possible_fee` should use safe bounded arithmetic or validate that intermediate products do not exceed the field prime before summing.

---

### Proof of Concept

The Cairo prime is P = 2^251 + 17·2^192 + 1. Set:

| Field | Value |
|---|---|
| L1 gas `max_amount` | 2 |
| L1 gas `max_price_per_unit` | (P − 1) / 2 |
| L2 gas `max_amount` | 1 |
| L2 gas `max_price_per_unit` | 1 |
| tip | 0 |
| L1 data gas `max_amount` | 0 |
| L1 data gas `max_price_per_unit` | 0 |

`(P − 1) / 2 < P / 2`, so `assert_nn` passes. `max_amount = 2 ≤ 2^64 − 1`, so `assert_nn_le` passes.

Fee computation:

```
max_fee = 2 * (P−1)/2  +  1 * (1 + 0)  +  0
        = (P − 1)      +  1
        = P
        ≡ 0  (mod P)
```

`compute_max_possible_fee` returns 0. `charge_fee` hits the `if (max_fee == 0) { return (); }` branch and exits without charging any fee. The transaction executes with L2 gas budget = 1 and pays zero fees. The OS proof is valid and accepted by L1.

func compute_max_possible_fee(tx_info: TxInfo*) -> felt {
    tempvar resource_bounds: ResourceBounds* = tx_info.resource_bounds_start;
    let n_resource_bounds = (tx_info.resource_bounds_end - resource_bounds) / ResourceBounds.SIZE;

    // Only V3 transactions with all resource bounds are supported.
    assert tx_info.version = 3;
    assert n_resource_bounds = 3;

    tempvar l1_gas_bounds: ResourceBounds = resource_bounds[L1_GAS_INDEX];
    tempvar l2_gas_bounds: ResourceBounds = resource_bounds[L2_GAS_INDEX];
    tempvar l1_data_gas_bounds = resource_bounds[L1_DATA_GAS_INDEX];

    return l1_gas_bounds.max_amount * l1_gas_bounds.max_price_per_unit + l2_gas_bounds.max_amount *
        (l2_gas_bounds.max_price_per_unit + tx_info.tip) + l1_data_gas_bounds.max_amount *
        l1_data_gas_bounds.max_price_per_unit;
}
```
