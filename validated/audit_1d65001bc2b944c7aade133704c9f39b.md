### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not validate that the new class hash supplied by a contract corresponds to any declared (compiled) contract class. An unprivileged contract can call this syscall with an arbitrary, undeclared class hash. The OS accepts and commits the change to state without any guard check. The affected contract becomes permanently non-functional, and any funds it holds are permanently frozen.

---

### Finding Description

In `syscall_impls.cairo`, the function `execute_replace_class` (lines 878–916) contains an explicit developer TODO acknowledging the missing guard:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The `class_hash` value comes directly from the syscall request struct (`ReplaceClassRequest`) and is never checked against the set of declared compiled classes (`compiled_class_facts`). Any felt value — including `0`, `1`, or any arbitrary undeclared hash — is accepted and written into `contract_state_changes` as the contract's new class hash.

This is structurally analogous to the reported `CollateralManager.removeCollateral()` issue: just as removing a collateral type causes a user's ICR to drop to zero with no guard, replacing a contract's class with an undeclared hash causes the contract's execution capability to drop to zero with no guard — permanently.

The `charge_fee` function in `transaction_impls.cairo` reads the fee token contract's class hash from `contract_state_changes` and uses it directly to dispatch execution:

```cairo
local fee_token_address = block_context.os_global_context.starknet_os_config.fee_token_address;
let (fee_state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
    key=fee_token_address
);
...
local execution_context: ExecutionContext = ExecutionContext(
    ...
    class_hash=fee_state_entry.class_hash,
    ...
);
``` [2](#0-1) 

If the fee token contract itself (or any contract holding user funds) has its class replaced with an invalid hash, all subsequent interactions with it fail permanently.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any ERC20 balances, escrowed assets, or other funds stored in the storage of a contract whose class hash has been replaced with an undeclared value are permanently inaccessible. The contract can no longer execute any entry point (transfer, withdraw, etc.) because the OS cannot resolve the class hash to a compiled class. The state change is committed on-chain and is irreversible — there is no recovery path once the invalid hash is in the committed state root. [3](#0-2) 

---

### Likelihood Explanation

**Medium.**

The attack requires an unprivileged transaction sender to:
1. Deploy a contract (e.g., a yield vault, escrow, or token wrapper) that accepts user deposits.
2. Attract user funds into the contract.
3. Call a function on the contract that issues `replace_class` with an arbitrary undeclared class hash.

Step 3 requires no privileged access — any contract can call `replace_class` on itself. The transaction succeeds (does not revert), the invalid class hash is committed to state, and the contract is permanently broken. The revert log records the old class hash, but since the transaction itself does not revert, the revert log entry is never used.

Additionally, a legitimate contract with a programming bug that passes an incorrect hash to `replace_class` would trigger the same outcome accidentally. [4](#0-3) 

---

### Recommendation

Implement the missing validation check inside `execute_replace_class` before updating `contract_state_changes`. Specifically, verify that `class_hash` exists in the set of declared compiled classes (i.e., it appears as a key in `contract_class_changes` with a non-zero compiled class hash, or is present in the pre-existing class tree). If the class hash is not declared, write a failure response and return without modifying state, consistent with how other invalid-argument cases are handled in the syscall dispatcher. [5](#0-4) 

---

### Proof of Concept

1. Attacker deploys `MaliciousVault`, a contract that:
   - Exposes a `deposit()` entry point that accepts user funds (ERC20 transfers into itself).
   - Exposes a `rug()` entry point that calls `replace_class(0x1)` — an arbitrary undeclared class hash.

2. Users call `deposit()` and transfer tokens into `MaliciousVault`. The contract's storage now holds user balances.

3. Attacker calls `rug()`. The OS executes `execute_replace_class`:
   - `class_hash = 0x1` is read from the syscall request.
   - The TODO check is absent; no validation is performed.
   - `dict_update` writes `StateEntry(class_hash=0x1, ...)` into `contract_state_changes`.
   - The revert log records the old class hash, but the transaction completes successfully — the revert log entry is never applied.

4. The block is proven and the state root is updated. `MaliciousVault`'s class hash is now `0x1` in the committed global state.

5. Any subsequent call to `MaliciousVault` (e.g., `withdraw()`) causes the OS to look up compiled class `0x1`, which does not exist in `compiled_class_facts`. Execution fails unconditionally.

6. All user funds stored in `MaliciousVault`'s storage are permanently frozen with no recovery path. [1](#0-0) [6](#0-5)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
