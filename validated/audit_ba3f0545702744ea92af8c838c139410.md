### Title
Integer Overflow in `check_time_locks` Enables Relative Timelock Bypass via `wrapping_add` — (File: `crates/chia-consensus/src/check_time_locks.rs`)

---

### Summary

`check_time_locks` has two arithmetic modes controlled by the `nowrap` boolean. When `nowrap=false`, all four relative-timelock checks use `wrapping_add` instead of `saturating_add`. For a coin whose puzzle emits `ASSERT_HEIGHT_RELATIVE 0xffff_ffff` (or `ASSERT_SECONDS_RELATIVE 0xffff_ffff_ffff_ffff`), the addition wraps around to a value *smaller* than the coin's confirmation height, making the check trivially pass at any block height. The intended timelock — effectively "never spendable in practice" — is silently discarded.

---

### Finding Description

`check_time_locks` in `crates/chia-consensus/src/check_time_locks.rs` is the consensus-layer function that enforces `ASSERT_HEIGHT_RELATIVE`, `ASSERT_SECONDS_RELATIVE`, `ASSERT_BEFORE_HEIGHT_RELATIVE`, and `ASSERT_BEFORE_SECONDS_RELATIVE` conditions against on-chain coin records. It is exposed as a Python binding and called by the full node during block/mempool validation.

The function accepts a `nowrap: bool` parameter. When `nowrap=false`, the relative-timelock arithmetic uses `wrapping_add`:

```rust
// ASSERT_HEIGHT_RELATIVE path, nowrap=false branch
} else if prev_transaction_block_height
    < unspent.confirmed_block_index.wrapping_add(height_relative)
{
    return Err(ValidationErr::Err(ErrorCode::AssertHeightRelativeFailed));
}
``` [1](#0-0) 

For `height_relative = 0xffff_ffff` and `confirmed_block_index = 100`:

```
wrapping_add(100u32, 0xffff_ffff) = 99   // wraps around u32::MAX
```

The check becomes `prev_height < 99`, which is `false` for any realistic block height, so **no error is returned** — the timelock is bypassed. The same overflow applies to `seconds_relative` (u64) with value `0xffff_ffff_ffff_ffff`. [2](#0-1) 

The inverse condition (`ASSERT_BEFORE_HEIGHT_RELATIVE`) wraps in the opposite direction: the check `prev_height >= 99` is always `true`, permanently locking the coin (DoS on the coin owner). [3](#0-2) 

The test suite explicitly documents this divergence:

```rust

### Impact Explanation

This matches the allowed High impact: **"Timelock or condition validation bypass enables unauthorized spend acceptance."**

A coin whose puzzle enforces `ASSERT_HEIGHT_RELATIVE 0xffff_ffff` is intended to be unspendable for ~4 billion blocks (centuries). When the full node calls `check_time_locks(..., nowrap=false)`, the overflow converts the required height to `confirmed_block_index - 1`, which is already satisfied. The coin is accepted as spendable immediately. Any value locked in such a coin — including coins created by smart-contract puzzles that use large relative timelocks as a security invariant — can be stolen or prematurely spent.

---

### Likelihood Explanation

The `nowrap` parameter is exposed as a first-class Python binding argument. The full node passes it from Python; the test suite exercises both branches, confirming both are live code paths. If the full node passes `nowrap=false` for any block height range (e.g., pre-fork blocks), the bypass is reachable by any unprivileged spender who can craft or find a coin with a max-value relative timelock condition. The attacker needs no special privilege — only the ability to submit a spend bundle.

---

### Recommendation

Remove the `nowrap` parameter entirely and unconditionally use `saturating_add` for all relative-timelock arithmetic. If backward compatibility with old blocks requires the wrapping behavior, gate it behind a hard-fork height check inside the function rather than exposing it as a caller-controlled boolean, and document that `nowrap=false` must never be used for any block that could contain attacker-crafted conditions.

---

### Proof of Concept

1. Craft a puzzle that emits `(ASSERT_HEIGHT_RELATIVE . 0xffff_ffff)`.
2. Fund a coin with that puzzle at block height `B`.
3. Submit a spend of that coin at block height `B + 1`.
4. The full node calls `check_time_locks(..., nowrap=false)`.
5. `wrapping_add(B, 0xffff_ffff) = B - 1`; check `B + 1 < B - 1` is `false` → no error.
6. The spend is accepted despite the intended ~4-billion-block timelock.

The same attack applies to `ASSERT_SECONDS_RELATIVE 0xffff_ffff_ffff_ffff` with `wrapping_add` on `u64`. [7](#0-6)

    if let Some(before_height_absolute) = bundle_conds.before_height_absolute {

            return Err(ValidationErr::Err(

    if let Some(before_seconds_absolute) = bundle_conds.before_seconds_absolute {
        if timestamp >= before_seconds_absolute {
            return Err(ValidationErr::Err(
                ErrorCode::AssertBeforeSecondsAbsoluteFailed,
            ));
        }
    }

    for spend in &bundle_conds.spends {
        let Some(unspent) = removal_coin_records.get(&Bytes32::from(spend.coin_id)) else {
            return Err(ValidationErr::Err(ErrorCode::InvalidCoinId));
        };

        if let Some(birth_height) = spend.birth_height {
            if birth_height != unspent.confirmed_block_index {
                return Err(ValidationErr::Err(ErrorCode::AssertMyBirthHeightFailed));
            }
        }
        if let Some(birth_seconds) = spend.birth_seconds {
            if birth_seconds != unspent.timestamp {
                return Err(ValidationErr::Err(ErrorCode::AssertMyBirthSecondsFailed));
            }
        }
        if let Some(height_relative) = spend.height_relative {
            if nowrap {

            }
        }
        if let Some(seconds_relative) = spend.seconds_relative {
            if nowrap {
                if timestamp < unspent.timestamp.saturating_add(seconds_relative) {
                    return Err(ValidationErr::Err(ErrorCode::AssertSecondsRelativeFailed));
                }
            } else if timestamp < unspent.timestamp.wrapping_add(seconds_relative) {
                return Err(ValidationErr::Err(ErrorCode::AssertSecondsRelativeFailed));
            }
        }
        if let Some(before_height_relative) = spend.before_height_relative {

                        .confirmed_block_index
                        .saturating_add(before_height_relative)
                {
                    return Err(ValidationErr::Err(
                        ErrorCode::AssertBeforeHeightRelativeFailed,
                    ));
                }
            } else if prev_transaction_block_height
                >= unspent
                    .confirmed_block_index
                    .wrapping_add(before_height_relative)
            {
                return Err(ValidationErr::Err(
                    ErrorCode::AssertBeforeHeightRelativeFailed,
                ));

                    return Err(ValidationErr::Err(

                return Err(ValidationErr::Err(
                    ErrorCode::AssertBeforeSecondsRelativeFailed,

            }
        }
    }

    Ok(())
