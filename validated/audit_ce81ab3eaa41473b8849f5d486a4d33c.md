### Title
Unclaimable Native Fee Lock — STRK Tokens Permanently Frozen in StarkNet Bridge - (File: `starknet/src/omni_bridge.cairo`)

### Summary
The StarkNet `OmniBridge.init_transfer` collects `native_fee` in STRK tokens from the caller but the contract exposes no function to pay those tokens out to a relayer or withdraw them. Every call with `native_fee > 0` permanently locks STRK in the bridge contract.

### Finding Description
`init_transfer` on StarkNet accepts a `native_fee: u128` parameter. When it is non-zero, the function pulls that many STRK tokens from the caller into the bridge contract:

```cairo
if native_fee > 0 {
    let native_token = self.strk_token_address.read();
    let success = IERC20Dispatcher { contract_address: native_token }
        .transfer_from(caller, get_contract_address(), native_fee.into());
    assert(success, 'ERR_FEE_TRANSFER_FAILED');
}
