### Title
Unauthorized Class Declaration via Unauthenticated `'BOOTSTRAP'` Sender Address Bypass — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

`execute_declare_transaction` contains a hardcoded special-case branch that, when `sender_address == 'BOOTSTRAP'`, `nonce == 0`, `version == 3`, and all resource bounds are zero, **completely skips signature verification, nonce enforcement, and fee collection**, and directly writes a new class hash into the global `contract_class_changes` dictionary. Because `'BOOTSTRAP'` is a predictable felt literal (the ASCII encoding of the string "BOOTSTRAP"), any transaction sender who can get such a transaction included in a block can declare arbitrary contract classes with no cryptographic authorization.

---

### Finding Description

In `execute_declare_transaction`, after computing and asserting the transaction hash, the following branch is evaluated:

```cairo
if (sender_address == 'BOOTSTRAP' and tx_info.nonce == 0 and tx_info.version == 3) {
    let max_possible_fee = compute_max_possible_fee(tx_info=tx_info);
    if (max_possible_fee == 0) {
        assert_not_zero(compiled_class_hash);
        dict_update{dict_ptr=contract_class_changes}(
            key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
        );
        %{ SkipTx %}
        return ();
    }
}
``` [1](#0-0) 

When this branch is taken, the function returns **before** calling:

- `check_and_increment_nonce` — nonce is never verified or incremented
- `run_validate` — the account contract's `__validate_declare__` entry point (signature check) is never executed
- `charge_fee` — no fee is deducted [2](#0-1) 

The three skipped security gates are the only mechanisms that enforce authorization for declare transactions:

- `run_validate` calls `__validate_declare__` which performs the ECDSA/account-level signature check: [3](#0-2) 

- `check_and_increment_nonce` enforces sequential nonce ordering: [4](#0-3) 

- `charge_fee` enforces economic cost: [5](#0-4) 

The `'BOOTSTRAP'` value is not a privileged cryptographic key or a restricted system address — it is simply the felt encoding of the ASCII string "BOOTSTRAP", a value any transaction sender can place in the `sender_address` field of a declare transaction. The `%{ AssertTransactionHash %}` hint only verifies internal hash consistency (that the OS computed the same hash as the sequencer provided); it does **not** verify a cryptographic signature over that hash.

The `prev_value=0` guard in `dict_update` prevents re-declaring an already-declared class, but it does not prevent declaring any new class.

---

### Impact Explanation

An attacker who can get a declare transaction with the BOOTSTRAP parameters included in a block can:

1. Declare an arbitrary malicious contract class (e.g., one whose `__execute__` function drains caller balances or has a hidden backdoor) into the global class registry — **without owning any account contract, without a valid signature, and without paying fees**.
2. Subsequently deploy contracts using that class via the `deploy` syscall or a `deploy_account` transaction.
3. Any user who interacts with a contract deployed from the malicious class (e.g., by calling it, sending funds to it, or using it as a proxy) suffers **direct loss of funds**.

Because the class registry is a global, persistent state commitment included in the OS output and verified on L1, a fraudulently declared class is indistinguishable from a legitimately declared one in the final state root. This constitutes a **Critical — Direct loss of funds** impact.

---

### Likelihood Explanation

The trigger condition is fully attacker-controlled: `sender_address = felt('BOOTSTRAP')`, `nonce = 0`, `version = 3`, all resource bounds zero. These are ordinary transaction fields with no cryptographic barrier. The sequencer's mempool may perform off-chain validation, but the OS program — which is the authoritative verifier — imposes no such restriction. A sequencer that follows the OS program's rules (or a malicious sequencer) can include such a transaction and produce a valid STARK proof. Because the `'BOOTSTRAP'` literal is publicly visible in the open-source OS program, the bypass is trivially discoverable.

---

### Recommendation

Remove the `'BOOTSTRAP'` special-case branch entirely. If a privileged bootstrap mechanism is genuinely required for system initialization, it must be protected by a cryptographic mechanism — for example, a signature from a well-known, hardware-secured key whose public key is hardcoded as a constant in the OS, verified inside the Cairo program before any state mutation is performed. A plain felt-literal address comparison provides no security.

---

### Proof of Concept

1. Attacker crafts a declare transaction with:
   - `sender_address = 0x424F4F545354524150` (felt encoding of `'BOOTSTRAP'`)
   - `nonce = 0`
   - `version = 3`
   - `L1_gas.max_amount = 0`, `L1_gas.max_price_per_unit = 0`
   - `L2_gas.max_amount = 0`, `L2_gas.max_price_per_unit = 0`
   - `L1_data_gas.max_amount = 0`, `L1_data_gas.max_price_per_unit = 0`
   - `class_hash` = hash of a malicious Sierra class
   - `compiled_class_hash` = compiled hash of the malicious class
   - Signature field: arbitrary (never checked)

2. The sequencer includes the transaction in a block.

3. The OS executes `execute_declare_transaction`:
   - Computes and asserts the transaction hash (consistency check only, no signature verification).
   - Evaluates the BOOTSTRAP condition — all criteria satisfied.
   - Calls `compute_max_possible_fee` → returns 0.
   - Calls `dict_update` to write `compiled_class_hash` into `contract_class_changes` for the malicious `class_hash`.
   - Returns without calling `check_and_increment_nonce`, `run_validate`, or `charge_fee`.

4. The malicious class is now part of the committed state. The attacker deploys a contract using this class. Users who interact with the deployed contract lose funds. [1](#0-0)

func check_and_increment_nonce{contract_state_changes: DictAccess*}(tx_info: TxInfo*) -> () {
    // Do not handle nonce for version 0.
    if (tx_info.version == 0) {
        return ();
    }

    tempvar state_entry: StateEntry*;
    %{ SetStateEntryToAccountContractAddress %}

    tempvar current_nonce = state_entry.nonce;
    with_attr error_message("Unexpected nonce.") {
        assert current_nonce = tx_info.nonce;
    }

    // Update contract_state_changes.
    tempvar new_state_entry = new StateEntry(
        class_hash=state_entry.class_hash,
        storage_ptr=state_entry.storage_ptr,
        nonce=current_nonce + 1,
    );
    dict_update{dict_ptr=contract_state_changes}(
        key=tx_info.account_contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    return ();

func run_validate{
    range_check_ptr,
    remaining_gas: felt,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, tx_execution_context: ExecutionContext*) {
    alloc_locals;
    local tx_execution_info: ExecutionInfo* = tx_execution_context.execution_info;

    // Do not run "__validate__" for version 0.
    if (tx_execution_info.tx_info.version == 0) {
        return ();
    }

    // "__validate__" is expected to get the same calldata as "__execute__".
    local validate_execution_context: ExecutionContext* = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=tx_execution_context.class_hash,
        calldata_size=tx_execution_context.calldata_size,
        calldata=tx_execution_context.calldata,
        execution_info=new ExecutionInfo(
            block_info=block_context.block_info_for_validate,
            tx_info=tx_execution_info.tx_info,
            caller_address=tx_execution_info.caller_address,
            contract_address=tx_execution_info.contract_address,
            selector=VALIDATE_ENTRY_POINT_SELECTOR,
        ),
        deprecated_tx_info=tx_execution_context.deprecated_tx_info,
    );

    // The __validate__ function should not revert.
    let (retdata_size, retdata, is_deprecated) = non_reverting_select_execute_entry_point_func(
        block_context=block_context, execution_context=validate_execution_context
    );
    if (is_deprecated == 0) {
        %{ CheckRetdataForDebug %}
        assert retdata_size = 1;
        assert retdata[0] = VALIDATED;
    }

    return ();
}
