### Title
Unvalidated `l1_gas_price_wei` in `ProposalInit` Allows Proposer to Corrupt ETH-Denominated Fee Calculations and Derived `l2_gas_price_wei` - (File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs)

---

### Summary

`is_block_info_valid` validates `l1_gas_price_fri` and `l1_data_gas_price_fri` within a configurable margin but explicitly discards the locally-computed `_l1_gas_prices_wei` without comparing it to the proposer-supplied `l1_gas_price_wei` / `l1_data_gas_price_wei`. A proposer selected by BFT consensus can therefore set those wei fields to any value. Because `convert_to_sn_api_block_info` derives `eth_to_fri_rate` directly from the proposer-supplied `l1_gas_price_fri / l1_gas_price_wei`, and then derives `l2_gas_price_wei` from that rate, the entire ETH-denominated gas-price vector fed to the blockifier is attacker-controlled. This is the sequencer analog of the Solana "Stake Mint Differentiation" bug: a numeric amount is accepted without validating the scale (unit) in which it is expressed.

---

### Finding Description

**Root cause — `is_block_info_valid`** [1](#0-0) 

```rust
let (l1_gas_prices_fri, _l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
    l1_gas_price_provider,
    init_proposed.timestamp,
    block_info_validation.previous_block_info.as_ref(),
    gas_price_params,
)
.await;
// ...
if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, ...)
    && within_margin(l1_data_gas_price_fri_proposed, l1_data_gas_price_fri, ...))
{
    return Err(...)
}
```

The locally-computed `_l1_gas_prices_wei` is prefixed with `_` and never compared to `init_proposed.l1_gas_price_wei` or `init_proposed.l1_data_gas_price_wei`. Only the FRI prices are range-checked.

**Downstream corruption — `convert_to_sn_api_block_info`** [2](#0-1) 

```rust
let previous_block_info = PreviousBlockInfo::from(init);   // uses init.l1_gas_price_wei
let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;
// eth_to_fri_rate = l1_gas_price_fri * WEI_PER_ETH / l1_gas_price_wei

let l2_gas_price_wei = NonzeroGasPrice::new(
    init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?
)?;
// l2_gas_price_wei = l2_gas_price_fri * l1_gas_price_wei / l1_gas_price_fri
``` [3](#0-2) 

Both `eth_gas_prices.l1_gas_price` and `eth_gas_prices.l2_gas_price` in the resulting `BlockInfo` are therefore fully controlled by the proposer's `l1_gas_price_wei`.

**Block-hash propagation**

`PartialBlockHashComponents::new` stores `l1_gas_price_per_token()` (which carries both `price_in_wei` and `price_in_fri`) and passes it to `gas_prices_to_hash` inside `calculate_block_hash`. [4](#0-3) 

If `gas_prices_to_hash` serialises the wei component (implementation not indexed), the block hash itself encodes the attacker-chosen wei price.

**Fallback rate poisoning**

`previous_block_info_from_block_header` reads `l1_gas_price.price_in_wei` from the stored header to reconstruct `PreviousBlockInfo`. When the L1 oracle fails on the next block, `calculate_eth_to_fri_rate` reuses that stored wei price, propagating the corrupted rate into the next block's FRI prices. [5](#0-4) 

---

### Impact Explanation

| Corrupted value | Mechanism | Effect |
|---|---|---|
| `eth_gas_prices.l1_gas_price` | `init.l1_gas_price_wei` used verbatim | V1/V2 (ETH-denominated) transactions pay fees at the attacker-chosen wei price |
| `eth_gas_prices.l2_gas_price` (`l2_gas_price_wei`) | Derived via `eth_to_fri_rate` from `l1_gas_price_wei` | L2-gas ETH fees scale proportionally with the manipulated wei price |
| Block hash (conditional) | `GasPricePerToken` with wei field fed to `gas_prices_to_hash` | Block hash encodes wrong gas-price scale |
| Next-block fallback rate | Stored wei price reused when oracle is unavailable | Corrupted `eth_to_fri_rate` propagates to subsequent blocks |

Setting `l1_gas_price_wei = 1` instead of the correct ~10⁹ makes `eth_gas_prices.l1_gas_price = 1 wei`, reducing ETH-denominated fees by ~10⁹×. This matches the "Incorrect fee … with economic impact" Critical scope.

---

### Likelihood Explanation

Any BFT-selected proposer (one of the sequencer nodes) can craft a `ProposalInit` with an arbitrary `l1_gas_price_wei`. The validator's `is_block_info_valid` will accept it because only the FRI fields are range-checked. No privileged access beyond being the current round's proposer is required.

---

### Recommendation

In `is_block_info_valid`, compare the proposer-supplied `l1_gas_price_wei` and `l1_data_gas_price_wei` against the locally-computed `_l1_gas_prices_wei` using the same `within_margin` check already applied to the FRI prices:

```rust
let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(...).await;

if !(within_margin(l1_gas_price_fri_proposed,  l1_gas_prices_fri.l1_gas_price,      margin)
  && within_margin(l1_data_gas_price_fri_proposed, l1_gas_prices_fri.l1_data_gas_price, margin)
  && within_margin(init_proposed.l1_gas_price_wei,  l1_gas_prices_wei.l1_gas_price,     margin)
  && within_margin(init_proposed.l1_data_gas_price_wei, l1_gas_prices_wei.l1_data_gas_price, margin))
{
    return Err(...)
}
```

    {

    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;

            },

        starknet_version: init.starknet_version,
    })

/// Poseidon (
///     block_hash_constant, block_number, global_state_root, sequencer_address,
///     block_timestamp, concat_counts, state_diff_hash, transaction_commitment,
///     event_commitment, receipt_commitment, gas_prices, starknet_version, 0, parent_block_hash

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
