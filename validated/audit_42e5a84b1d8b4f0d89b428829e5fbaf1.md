### Title
Incomplete Cache Update in `BlockStatusCache::move_index` Leaves Stale Key/Hash Mappings After Leaf Deletion — (File: `crates/chia-datalayer/src/merkle/blob.rs`)

### Summary
`BlockStatusCache::move_index` only marks the source index as free but never updates `key_to_index` or `leaf_hash_to_index`. When a leaf sibling is physically relocated to `TreeIndex(0)` during a 2-leaf tree deletion, the cache retains stale pointers to the old (now-free) index. All subsequent cache-driven operations (`get_proof_of_inclusion`, `upsert`, `delete`) on the surviving leaf operate on the wrong blob slot, causing panics or silent tree-root corruption.

### Finding Description

`BlockStatusCache::move_index` is called inside `MerkleBlob::delete` when the deleted leaf's parent is the root (no grandparent). The sibling node is physically written to `destination = TreeIndex(0)`, and then `move_index(sibling_index, destination)` is called to update the cache. [1](#0-0) 

`move_index` only inserts `source` into `free_indexes`. It never touches `key_to_index` or `leaf_hash_to_index`: [1](#0-0) 

The call site in `delete`: [2](#0-1) 

After this path executes with a leaf sibling:
- The leaf is physically at `TreeIndex(0)`.
- `key_to_index[sibling_key]` still maps to the old `sibling_index` (now in `free_indexes`).
- `leaf_hash_to_index[sibling_hash]` still maps to the old `sibling_index`.

Every subsequent cache-driven lookup (`get_leaf_by_key`, `get_index_by_key`) returns the stale index. [3](#0-2) 

### Impact Explanation

**`get_proof_of_inclusion` — panic / incorrect proof**

`get_proof_of_inclusion` calls `get_leaf_by_key`, which resolves to the stale `sibling_index`. The blob bytes there still carry the old parent pointer (`TreeIndex(0)`). Lineage traversal then reads `TreeIndex(0)`, which now holds the leaf itself (not an internal node), causing `expect_internal` to panic. [4](#0-3) 

**`upsert` — tree-root corruption**

`upsert` calls `get_leaf_by_key` (returns stale `sibling_index`), then `remove_leaf` (removes the key from `key_to_index`), then writes the updated leaf to `sibling_index` (the wrong slot). `TreeIndex(0)` retains the old, unmodified leaf and its old hash. The DataLayer root hash is now permanently stale — it no longer reflects the updated value — while the cache has lost the key entirely. [5](#0-4) 

The stale root hash can then be used to prove the old (pre-update) state as current, satisfying the "lets untrusted input prove invalid state" criterion.

### Likelihood Explanation

The trigger is a 2-leaf `MerkleBlob` followed by a single `delete`. This is a routine DataLayer operation sequence (insert two key-value pairs, then delete one). The Python binding `py_delete` is directly callable by any DataLayer client. No privileged access is required. [6](#0-5) 

### Recommendation

`move_index` must update `key_to_index` and `leaf_hash_to_index` when the moved node is a leaf. The function should accept the moved node's data (or the caller should pass the leaf) and re-map the key and hash to `destination`:

```rust
fn move_index(&mut self, source: TreeIndex, destination: TreeIndex, node: &Node) -> Result<(), Error> {
    if self.free_indexes.contains(&source) {
        return Err(Error::MoveSourceIndexNotInUse(source));
    }
    if self.free_indexes.contains(&destination) {
        return Err(Error::MoveDestinationIndexNotInUse(destination));
    }
    self.free_indexes.insert(source);
    // Update leaf-specific mappings
    if let Node::Leaf(leaf) = node {
        if let Some(idx) = self.key_to_index.get_mut(&leaf.key) {
            *idx = destination;
        }
        if let Some(idx) = self.leaf_hash_to_index.get_mut(&leaf.hash) {
            *idx = destination;
        }
    }
    Ok(())
}
```

The call in `delete` must pass `&sibling_block.node`.

### Proof of Concept

```rust
use chia_datalayer::{MerkleBlob, KeyId, ValueId};

let hash_a = [1u8; 32];
let hash_b = [2u8; 32];

let mut blob = MerkleBlob::new(vec![]).unwrap();
blob.insert(KeyId(1), ValueId(10), &hash_a, Default::default()).unwrap();
blob.insert(KeyId(2), ValueId(20), &hash_b, Default::default()).unwrap();

// Delete key 1 — triggers move_index with a leaf sibling (key 2)
blob.delete(KeyId(1)).unwrap();

// Cache now has key_to_index[2] pointing to the freed sibling_index, not TreeIndex(0).
// This panics inside expect_internal because TreeIndex(0) is now a leaf:
let _ = blob.get_proof_of_inclusion(KeyId(2)); // panic

// Or: upsert silently writes to the wrong slot and loses the key from the cache,
// leaving TreeIndex(0) with the old hash — stale root.
let new_hash = [3u8; 32];
blob.upsert(KeyId(2), ValueId(99), &new_hash).unwrap();
// blob.get_root_hash() returns old hash_b, not new_hash — root is corrupted.

            self.mark_lineage_as_dirty(parent)?;

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
