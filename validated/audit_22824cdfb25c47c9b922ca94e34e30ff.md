### Title
Panic in `ProofOfSpace::update_digest` via `.expect()` on Invalid v2 Proof Bytes — (`crates/chia-protocol/src/proof_of_space.rs`)

### Summary

A v2 `ProofOfSpace` (version=1) with structurally valid metadata but cryptographically invalid `proof` bytes passes `parse()` successfully. When any code subsequently calls `update_digest()` on that object (e.g., to compute a block hash), `quality_string()` returns `None` and the unconditional `.expect()` panics, crashing the node.

### Finding Description

The `parse()` implementation for version=1 accepts any byte sequence as the `proof` field — it only validates the structural constraint that exactly one of `pool_public_key` / `pool_contract_puzzle_hash` is set: [1](#0-0) 

No proof-content validation occurs at parse time. The `proof` field is stored verbatim.

Later, `update_digest()` for the v2 path calls `quality_string()` and immediately panics if it returns `None`: [2](#0-1) 

`quality_string()` delegates to `chia_pos2::quality_string_from_proof`, which is explicitly documented to return `None` for invalid proof format: [3](#0-2) 

The `k_size` is derived from `proof.len()` with no bounds check, and any random byte sequence will cause `quality_string_from_proof` to return `None`.

`update_digest` on `ProofOfSpace` is called transitively by `RewardChainBlock::update_digest`: [4](#0-3) 

This is invoked whenever any block structure containing the `ProofOfSpace` is hashed — including during block validation, header hash computation, and Python/wasm `get_hash()` calls exposed on `ProofOfSpace`, `RewardChainBlock`, `FullBlock`, and `HeaderBlock`. [5](#0-4) 

### Impact Explanation

Any peer on the network can broadcast a block containing a v2 `ProofOfSpace` with valid structural fields but random proof bytes. When a receiving node parses the block (succeeds) and then computes any hash of the block or its components, the Rust process panics. In Rust, a `panic!` in a non-`catch_unwind` context terminates the thread or process. This is a remotely-triggerable node crash matching the **Critical** impact: "serialized network object can trigger chain halt."

### Likelihood Explanation

The attack requires only crafting a well-formed serialized block with a v2 `ProofOfSpace` containing garbage proof bytes. No keys, privileged access, or chain state are needed. The structural parse constraints (exactly one of pool_pk/contract, valid G1Element bytes) are easy to satisfy. The panic is deterministic and reproducible.

### Recommendation

Move proof-content validation into `parse()` for version=1: call `quality_string_from_proof` (or an equivalent structural check) during parsing and return `Err(Error::InvalidPoS)` if it returns `None`. Alternatively, replace the `.expect()` in `update_digest` with a graceful error return — but this requires changing the `Streamable::update_digest` signature to be fallible, which is a larger refactor. The simplest fix is to validate proof content at parse time so that a successfully-parsed `ProofOfSpace` is always hashable.

### Proof of Concept

```rust
// Construct a v2 ProofOfSpace with valid structure but random proof bytes
let pos = ProofOfSpace::new(
    Bytes32::default(),          // challenge
    Some(valid_pool_pk),         // exactly one of pool_pk/contract
    None,
    valid_plot_pk,
    1,                           // version = 1 (v2)
    0,                           // plot_index
    0,                           // meta_group
    2,                           // strength
    0,                           // size (ignored for v2)
    Bytes::from(vec![0xde, 0xad, 0xbe, 0xef, /* ... random bytes ... */]),
);

// parse() succeeds — no proof-content check
let buf = pos.to_bytes().unwrap();
let parsed = ProofOfSpace::parse::<false>(&mut Cursor::new(&buf)).unwrap();

// update_digest panics: quality_string() returns None, .expect() fires
let mut sha = Sha256::new();
parsed.update_digest(&mut sha); // PANIC: "internal error. Can't compute hash of invalid ProofOfSpace"
```

The same panic is reachable via the Python binding `ProofOfSpace.get_hash()` or by parsing any `RewardChainBlock` / `FullBlock` containing such a proof and calling `.get_hash()` on it. [6](#0-5)

                    self.strength,
                ));

            })
