### Title
Missing Declared-Class Validation in `replace_class` Syscall Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS accepts any arbitrary felt value as a new class hash without verifying that the hash corresponds to a previously declared contract class. An unprivileged contract can call the `replace_class` syscall with an undeclared class hash, causing the OS to commit a permanently invalid class hash into the global state. Any contract whose class hash is set to an undeclared value becomes permanently unexecutable, freezing all funds held within it.

---

### Finding Description

In `execute_replace_class`, the OS reads the requested class hash directly from the syscall request and writes it into `contract_state_changes` with no validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The acknowledged TODO comment confirms the missing check. The function signature does not even carry `contract_class_changes` as an implicit argument, making it structurally impossible to perform the validation at this call site:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
```

When the block is committed, the state root is computed over `contract_state_changes` by `compute_contract_state_commitment` → `hash_contract_state_changes` → `get_contract_state_hash`, which blindly hashes whatever `class_hash` is stored in the `StateEntry`. No cross-check against `contract_class_changes` (the declared-class registry) is performed during commitment.

In any subsequent block, when `execute_entry_point` attempts to execute the frozen contract, it performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // undeclared hash → returns 0
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,           // key = 0, not present → proof fails
);
```

`dict_read` on an undeclared class hash returns 0 (the Cairo dict default). `find_element` with key `0` will fail to locate a compiled class fact, making any proof that includes a call to this contract invalid. An honest sequencer is therefore forced to permanently exclude all transactions targeting the frozen contract, making any funds held there irrecoverable.

---

### Impact Explanation

**Permanent freezing of funds (Critical).**

Once a `replace_class(undeclared_hash)` call is included in a committed block, the affected contract's class hash in the global state is permanently set to an undeclared value. No future block can include a transaction that calls this contract without producing an unprovable execution trace. Any ERC-20 balances, ETH, or other assets held by the contract are permanently inaccessible. The state cannot be repaired without a protocol-level upgrade or hard fork.

---

### Likelihood Explanation

The `replace_class` syscall is callable by any deployed contract without any privilege requirement. A contract that exposes a public function calling `replace_class` with an attacker-supplied argument (e.g., a proxy upgrade function with insufficient access control) is sufficient. Additionally, a contract developer who accidentally passes an undeclared hash (e.g., a hash computed off-chain that was never submitted as a `declare` transaction) would trigger the same outcome. Because the OS is the authoritative validator and performs no check, neither the sequencer's simulation nor the proof verifier will catch the error before the state is committed.

---

### Recommendation

1. Add `contract_class_changes: DictAccest*` as an implicit argument to `execute_replace_class`.
2. Before writing the new `StateEntry`, verify that `class_hash` is present in `contract_class_changes` (i.e., it was declared in the current block) **or** add a separate syscall/hint mechanism to verify existence in the historical class tree.
3. If the class hash is not declared, write a failure response via `write_failure_response` and return without updating the state, consistent with how other invalid syscall arguments are handled (e.g., `execute_call_contract` rejecting `EXECUTE_ENTRY_POINT_SELECTOR`).

---

### Proof of Concept

**Step 1 — Attacker deploys a contract** containing:
```
fn freeze_self(bad_hash: ClassHash) {
    replace_class_syscall(bad_hash);   // bad_hash was never declared
}
```

**Step 2 — Attacker calls `freeze_self(0xdeadbeef...)`.**

**Step 3 — OS execution path:**
- `execute_replace_class` is invoked with `class_hash = 0xdeadbeef...`
- No validation is performed (the TODO check is absent)
- `dict_update` writes `StateEntry { class_hash: 0xdeadbeef..., ... }` into `contract_state_changes`
- The revert log records the old class hash, but the transaction is not reverted

**Step 4 — Block commitment:**
- `state_update` → `squash_state_changes` → `compute_contract_state_commitment` → `get_contract_state_hash` hashes `0xdeadbeef...` into the state root
- The L1 verifier accepts the proof; the state root is finalized on-chain

**Step 5 — Any future call to the contract:**
- `execute_entry_point` calls `dict_read(contract_class_changes, 0xdeadbeef...)` → returns `0`
- `find_element(compiled_class_facts, key=0)` → element not found → proof is unprovable
- The sequencer cannot include any transaction targeting this contract
- All funds in the contract are permanently frozen [1](#0-0) [2](#0-1) [3](#0-2)

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
