Audit Report

## Title
Router `pay()` Consumes Residual ETH From Prior Transactions to Subsidize Subsequent WETH Swaps — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary

`PeripheryPayments.pay()` reads `address(this).balance` — the router's total native ETH balance — when deciding how much ETH to wrap for a WETH payment. It has no mechanism to distinguish ETH sent in the current transaction from ETH stranded by a prior caller. Any user who sends excess ETH to the router without calling `refundETH()` leaves funds that a subsequent WETH-swap caller can silently consume, resulting in direct, irreversible loss of the prior caller's ETH.

## Finding Description

The WETH branch of `pay()` reads the router's full native balance: [1](#0-0) 

`exactInputSingle`, `exactInput`, `exactOutputSingle`, and `exactOutput` are all `external payable`: [2](#0-1) 

The `receive()` guard only blocks direct ETH pushes (no calldata) from non-WETH addresses; it does **not** prevent ETH from being left over after a `payable` swap call that consumed less than `msg.value`: [3](#0-2) 

**Stranding path (User A):**
- User A calls `exactInputSingle{value: 1 ether}(tokenIn=WETH, amountIn=0.5 ether, ...)`.
- The swap callback calls `pay(WETH, userA, pool, 0.5 ether)`. `nativeBalance = 1 ETH ≥ 0.5 ETH`, so exactly 0.5 ETH is wrapped and sent to the pool.
- The remaining 0.5 ETH stays in the router. No `refundETH()` was called; it is stranded.

**Theft path (User B):**
- User B calls `exactInputSingle{value: 0.5 ether}(tokenIn=WETH, amountIn=1 ether, ...)`.
- The callback calls `pay(WETH, userB, pool, 1 ether)`.
- `nativeBalance = address(this).balance = 0.5 ETH (A's residual) + 0.5 ETH (B's msg.value) = 1 ETH`.
- `nativeBalance >= value` → the router wraps all 1 ETH and transfers it to the pool.
- User B's swap is fully settled using 0.5 ETH of User A's funds. User B never pays from their WETH allowance. User A's 0.5 ETH is gone.

Existing guards are insufficient: `receive()` only filters zero-calldata ETH pushes; there is no check that `msg.value` equals `amountIn` at any entry point; and `refundETH()` is optional and caller-initiated. [4](#0-3) 

## Impact Explanation

Direct, irreversible loss of native ETH belonging to a prior caller. The attacker receives a full swap output while paying only a fraction of the required input. The victim loses their stranded ETH with no recourse. This is a direct loss of user principal through an unprivileged public call, meeting the High threshold.

## Likelihood Explanation

The ETH-input multicall pattern is the documented and tested usage pattern for WETH swaps: [5](#0-4) 

The test suite itself demonstrates that `refundETH()` must be explicitly included in the multicall to recover excess ETH: [6](#0-5) 

Users who send slightly more ETH than needed (common for slippage headroom) and omit `refundETH()`, or who call `exactInputSingle` directly (not via multicall) with excess ETH, will strand funds. An attacker only needs to watch the router's ETH balance on-chain and follow any transaction that leaves residual ETH.

## Recommendation

Track only the ETH attributable to the current transaction. The standard fix (used by Uniswap v3 periphery) is to snapshot `address(this).balance - msg.value` before the swap entry and pass the available native amount into `pay()` as a parameter, rather than re-reading the live balance inside the callback. Alternatively, enforce that `address(this).balance == msg.value` at the start of every payable entry point (revert if the router already holds ETH), preventing cross-transaction contamination entirely.

## Proof of Concept

```solidity
function test_residualEth_stolenBySubsequentWethSwap() public {
    // User A strands 0.5 ETH by sending 1 ETH for a 0.5 ETH swap without refundETH
    vm.deal(userA, 1 ether);
    vm.prank(userA);
    router.exactInputSingle{value: 1 ether}(ExactInputSingleParams({
        pool: address(pool), tokenIn: address(weth), tokenOut: address(token1),
        zeroForOne: true, amountIn: 0.5 ether, amountOutMinimum: 0,
        recipient: userA, deadline: block.timestamp + 1, priceLimitX64: 0, extensionData: ""
    }));
    assertEq(address(router).balance, 0.5 ether, "A's ETH stranded");

    // User B sends only 0.5 ETH but requests a 1 ETH WETH swap
    vm.deal(userB, 0.5 ether);
    uint256 userBWethBefore = weth.balanceOf(userB);
    vm.prank(userB);
    router.exactInputSingle{value: 0.5 ether}(ExactInputSingleParams({
        pool: address(pool), tokenIn: address(weth), tokenOut: address(token1),
        zeroForOne: true, amountIn: 1 ether, amountOutMinimum: 0,
        recipient: userB, deadline: block.timestamp + 1, priceLimitX64: 0, extensionData: ""
    }));

    // Router used A's 0.5 ETH + B's 0.5 ETH = 1 ETH for B's swap
    assertEq(address(router).balance, 0, "router drained");
    assertEq(weth.balanceOf(userB), userBWethBefore, "B paid nothing from WETH");
}
```
