### Title
BLOCKHASH Opcode Returns Wrong Non-Zero Hash for Blocks Outside EIP-2935 History Window — (File: `x/evm/keeper/state_transition.go`, `x/evm/keeper/keeper.go`)

---

### Summary

`GetHashFn` in Ethermint applies a lower-bound window check only to the legacy staking-history fallback path, not to the EIP-2935 ring-buffer path. Once the chain has run for more than `HistoryServeWindow` (default 8191) blocks, a contract invoking the `BLOCKHASH` opcode for a block number older than the window receives a **wrong, non-zero hash** (the hash of a different, more recent block that shares the same ring-buffer slot) instead of the zero value mandated by the EVM specification. Any contract that relies on `BLOCKHASH` returning `0` to detect out-of-range queries is silently fed a plausible-looking but incorrect hash.

---

### Finding Description

**Root cause — `GetHashFn` (`x/evm/keeper/state_transition.go:97-144`)**

```go
func (k Keeper) GetHashFn(ctx sdk.Context, headerHashNum uint64) vm.GetHashFunc {
    return func(num64 uint64) common.Hash {
        ...
        var lower uint64
        if upper <= headerHashNum {
            lower = 0
        } else {
            lower = upper - headerHashNum   // window = 256 by default
        }

        if upper > num64 {
            // ① EIP-2935 path — NO lower-bound check
            headerHash := k.GetHeaderHash(ctx, num64)
            if headerHash.Cmp(common.Hash{}) != 0 {
                return headerHash           // ← returns wrong hash here
            } else if num64 >= lower {
                // ② legacy fallback — lower-bound check IS applied
                ...
            }
        }
        return common.Hash{}
    }
}
``` [1](#0-0) 

The `lower` guard (derived from `headerHashNum`, default 256) is only enforced on the legacy staking-history branch. The EIP-2935 branch at ① is entered for **any** historical block number without a lower-bound check.

**Root cause — `GetHeaderHash` ring-buffer collision (`x/evm/keeper/keeper.go:374-401`)**

```go
ringIndex := height % window          // window = HistoryServeWindow (default 8191)
var key common.Hash
binary.BigEndian.PutUint64(key[24:], ringIndex)
hash := k.GetState(ctx, ethparams.HistoryStorageAddress, key)
if hash.Cmp(common.Hash{}) != 0 {
    return hash                        // ← returns whatever is in the slot
}
``` [2](#0-1) 

`SetHeaderHash` writes `ctx.HeaderHash()` at slot `blockHeight % window` every block. [3](#0-2) 

After `window` blocks the ring wraps. Slot `s` now holds the hash of block `current - (current % window) + s` (the most recent block that maps to that slot), not the hash of the old block being queried. Because the slot is non-zero, `GetHeaderHash` returns it immediately, and `GetHashFn` propagates it to the EVM as the answer to `BLOCKHASH(oldBlock)`.

**Default parameter values**

| Parameter | Default | Purpose |
|---|---|---|
| `DefaultHeaderHashNum` | 256 | KV-store window / lower-bound for legacy path |
| `DefaultHistoryServeWindow` | 8191 | EIP-2935 ring-buffer size | [4](#0-3) 

The two constants are inconsistent: the lower-bound guard uses 256, but the ring buffer that is actually queried first spans 8191 slots — and has no guard at all for blocks older than 8191.

---

### Impact Explanation

The `BLOCKHASH` opcode is specified to return `0` for any block number outside the accessible history window. Returning a non-zero, wrong hash instead silently corrupts the EVM execution environment for any contract that:

- Uses `if blockhash(n) == 0` to detect an out-of-range query (e.g., dispute/proof contracts, bridges, commit-reveal schemes).
- Uses `blockhash` as a source of randomness or as a binding commitment to a past block.

A contract that guards a fund-release or dispute path with `require(blockhash(claimedBlock) == submittedHash)` can be bypassed: an attacker who knows the current ring-buffer contents can compute which recent block's hash occupies slot `claimedBlock % 8191` and submit that hash as "proof" for an arbitrarily old block. This maps directly to the allowed High impact: **EVM state transition bug that permits invalid transactions to commit or valid user funds to be mis-accounted**.

---

### Likelihood Explanation

- **Trigger condition 1**: Chain has run for more than 8191 blocks — trivially true on any live network after ~27 hours at 12 s/block.
- **Trigger condition 2**: A deployed contract uses `BLOCKHASH` for security-critical verification. This is a common pattern in bridges, optimistic rollup dispute games, and commit-reveal protocols — exactly the class of contracts Ethermint is designed to host.
- **Attacker capability**: Unprivileged; requires only a standard contract call or transaction. No validator access, governance, or key compromise needed.
- **Predictability**: The wrong hash returned is fully deterministic and publicly computable from on-chain state, so an attacker can pre-calculate the value before submitting the exploit transaction.

Likelihood: **Medium-High** once the chain exceeds 8191 blocks and any relevant contract is deployed.

---

### Recommendation

Add a lower-bound check to the EIP-2935 path in `GetHashFn`, using `HistoryServeWindow` (not `headerHashNum`) as the window:

```go
if upper > num64 {
    var eip2935Lower uint64
    if upper > historyServeWindow {
        eip2935Lower = upper - historyServeWindow
    }
    if num64 >= eip2935Lower {                  // ← add this guard
        headerHash := k.GetHeaderHash(ctx, num64)
        if headerHash.Cmp(common.Hash{}) != 0 {
            return headerHash
        }
    }
    if num64 >= lower {
        // legacy fallback ...
    }
}
```

`GetHashFn` should receive `historyServeWindow` as a second parameter (already available from `cfg.Params.HistoryServeWindow`) so the guard can be applied correctly. This ensures `BLOCKHASH` returns `0` for any block outside the ring-buffer window, matching the EVM specification.

### Proof of Concept

1. Deploy a contract on a chain that has run for > 8191 blocks:
   ```solidity
   function checkHash(uint256 blockNum, bytes32 expected) external view returns (bool) {
       return blockhash(blockNum) == expected;
   }
   ```
2. Pick any `blockNum` that is more than 8191 blocks in the past (e.g., `currentBlock - 10000`).
3. Compute `ringIndex = blockNum % 8191`. Read the hash stored at that ring-buffer slot from the EIP-2935 history contract storage — this is the hash of block `currentBlock - (currentBlock % 8191) + ringIndex` (a recent block).
4. Call `checkHash(blockNum, recentBlockHash)`. The call returns `true` even though `blockNum` is far outside the valid window and the EVM spec requires `blockhash(blockNum) == 0`.
5. Any contract that uses this pattern to verify a historical block hash (bridge proof, dispute bond, commit-reveal) accepts the forged value. [5](#0-4) [6](#0-5) [4](#0-3)

func (k Keeper) GetHashFn(ctx sdk.Context, headerHashNum uint64) vm.GetHashFunc {
	return func(num64 uint64) common.Hash {
		h, err := ethermint.SafeInt64(num64)
		if err != nil {
			return common.Hash{}
		}
		upper, err := ethermint.SafeUint64(ctx.BlockHeight())
		if err != nil {
			return common.Hash{}
		}
		if upper == num64 {
			headerHash := ctx.HeaderHash()
			if len(headerHash) > 0 {
				return common.BytesToHash(headerHash)
			}
		}
		// Align check with https://github.com/ethereum/go-ethereum/blob/release/1.11/core/vm/instructions.go#L433
		var lower uint64
		if upper <= headerHashNum {
			lower = 0
		} else {
			lower = upper - headerHashNum
		}

		if upper > num64 {
			// The requested height is historical, query EIP-2935 contract storage
			headerHash := k.GetHeaderHash(ctx, num64)
			if headerHash.Cmp(common.Hash{}) != 0 {
				return headerHash
			} else if num64 >= lower {
				// Pre upgrade case
				// In case EIP-2935 is not supported and data cannot be found, we fetch historical info
				histInfo, err := k.stakingKeeper.GetHistoricalInfo(ctx, h)
				if err != nil {
					k.Logger(ctx).Debug("historical info not found", "height", h, "err", err.Error())
					return common.Hash{}
				}
				header, err := cmttypes.HeaderFromProto(&histInfo.Header)
				if err != nil {
					k.Logger(ctx).Error("failed to cast tendermint header from proto", "error", err)
					return common.Hash{}
				}
				return common.BytesToHash(header.Hash())
			}
		}
		return common.Hash{}
	}
}

func (k Keeper) GetHeaderHash(ctx sdk.Context, height uint64) common.Hash {
	// check if history contract has been deployed
	acct := k.GetAccount(ctx, ethparams.HistoryStorageAddress)
	if acct != nil && acct.IsContract() {
		window := types.DefaultHistoryServeWindow
		params := k.GetParams(ctx)
		if params.HistoryServeWindow > 0 {
			window = params.HistoryServeWindow
		}

		ringIndex := height % window
		var key common.Hash
		binary.BigEndian.PutUint64(key[24:], ringIndex)
		hash := k.GetState(ctx, ethparams.HistoryStorageAddress, key)

		if hash.Cmp(common.Hash{}) != 0 {
			return hash
		}
	}
	// fall back to old behavior for retro compatibility
	// TODO can be removed along with DeleteHeaderHash once HistoryStorage has been filled up in next protocol upgrade
	store := ctx.KVStore(k.storeKey)
	hashByte := store.Get(types.GetHeaderHashKey(height))
	if len(hashByte) > 0 {
		return common.BytesToHash(hashByte)
	}
	return common.Hash{}
}
