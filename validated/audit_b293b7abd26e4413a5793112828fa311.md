### Title
`ProofOfInclusion::valid()` Contains a Tautological Root-Hash Check, Allowing Forged DataLayer Inclusion Proofs - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` is the sole public API for verifying DataLayer Merkle inclusion proofs. Its final correctness check — `existing_hash == self.root_hash()` — is a tautology: `root_hash()` returns `last.combined_hash`, which is the same value `existing_hash` was set to in the last loop iteration (after the loop already verified `calculated_hash == layer.combined_hash`). The function therefore only checks internal self-consistency of the proof struct, never comparing against any externally-supplied expected root. An attacker can construct a `ProofOfInclusion` with an arbitrary `node_hash` and internally consistent layers that passes `valid()` while proving inclusion of a key that does not exist in the actual DataLayer tree.

### Finding Description

**Root cause — `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`:**

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← derived entirely from the proof itself
    } else {
        self.node_hash
    }
}

pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {   // ← sets existing_hash = calculated_hash
            return false;
        }
        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()   // ← tautology: both sides equal last.combined_hash
}
```

After the loop, `existing_hash` holds the last `calculated_hash`, which was already asserted to equal `layer.combined_hash`. `self.root_hash()` returns that same `last.combined_hash`. The final comparison is therefore `last.combined_hash == last.combined_hash`, which is unconditionally `true` whenever the loop completes without returning `false`.

**Exploit path:**

An attacker constructs a `ProofOfInclusion` with:
- `node_hash` = any 32-byte value (the fake leaf hash they wish to "prove")
- `layers` = a chain of `ProofOfInclusionLayer` structs where each `combined_hash` is computed correctly from the previous hash and a chosen `other_hash`

Because `valid()` only checks that each layer's `combined_hash` equals the hash computed from the previous step, any internally consistent chain passes. The final root produced by this forged proof is whatever the attacker chose — it need not match the actual DataLayer tree root.

**Entry points:**

`ProofOfInclusion` is:
1. Fully serializable/deserializable via `Streamable` (`#[derive(Streamable)]`), so it can be received over the network as raw bytes.
2. Exposed to Python via `#[pymethods]` with `py_valid()` and `py_root_hash()`.
3. Constructible directly from Python via `from_py_object` and `__new__`. [1](#0-0) 

The Python binding exposes `valid()` and `root_hash()` as separate methods, making it natural for callers to write `proof.valid()` and trust the result, without realizing `valid()` never checks against an external root. [2](#0-1) 

The Python type stub confirms both methods are part of the public API: [3](#0-2) 

The `get_proof_of_inclusion` method on `MerkleBlob` is also exposed to Python, and the fuzz target and all tests call only `proof.valid()` — never comparing `proof.root_hash()` against an independently-known root: [4](#0-3) [5](#0-4) 

### Impact Explanation

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."**

Any DataLayer client or verifier that receives a `ProofOfInclusion` from an untrusted peer and calls `proof.valid()` to confirm a key-value pair is committed to a known root will be deceived. The attacker can prove inclusion of arbitrary key-value pairs in any DataLayer store, enabling:

- False attestation of data availability or state commitments
- Bypassing DataLayer-based access control or ownership checks that rely on inclusion proofs
- Corrupting application-layer logic that trusts `valid()` as a complete proof check

### Likelihood Explanation

The `ProofOfInclusion` struct is `Streamable` and constructible from Python, so any unprivileged network peer can craft and transmit a forged proof. The misleading API (`valid()` sounds complete but is not) makes incorrect usage highly likely. The fuzz target and all existing tests confirm that `valid()` is used in isolation without an external root comparison, suggesting the intended usage pattern is `proof.valid()` alone.

### Recommendation

The `valid()` method must accept an externally-supplied expected root hash and compare against it:

```rust
pub fn valid(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root   // compare against external root, not self.root_hash()
}
```

`root_hash()` can remain as a convenience accessor, but `valid()` must not use it as the ground truth. All call sites (Python bindings, fuzz targets, tests) must be updated to supply the expected root.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer, MerkleBlob
from chia_rs.sized_ints import uint8

# Build a real tree with one key
blob = MerkleBlob(bytearray())
real_key = 1
real_value = 2
real_hash = bytes([0xAA] * 32)
blob.insert(real_key, real_value, real_hash)
blob.calculate_lazy_hashes()
real_root = blob.get_root_hash()

# Forge a proof for a key that does NOT exist in the tree
fake_node_hash = bytes([0xBB] * 32)
fake_other_hash = bytes([0xCC] * 32)

import hashlib
# Compute a valid combined_hash for one layer (left side)
h = hashlib.sha256(b'\x00' * 30 + b'\x00\x02' + fake_node_hash + fake_other_hash).digest()
fake_layer = ProofOfInclusionLayer(
    other_hash_side=uint8(1),
    other_hash=fake_other_hash,
    combined_hash=h,
)
forged_proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[fake_layer])

# valid() returns True for the forged proof — root_hash() != real_root
assert forged_proof.valid()                    # passes — tautological check
assert forged_proof.root_hash() != real_root   # forged root ≠ actual tree root
# A verifier calling only proof.valid() is deceived
```
