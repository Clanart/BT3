### Title
Missing Nonce in `compute_meta_tx_v0_hash` Enables Unbounded Signature Replay — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/transaction_hash/transaction_hash.cairo`)

---

### Summary

`compute_meta_tx_v0_hash` omits the nonce from the transaction hash it produces. Because the OS also unconditionally skips nonce checking for version-0 transactions, a valid meta-tx-v0 signature can be replayed an unlimited number of times against the same target contract, enabling direct loss of funds.

---

### Finding Description

`compute_meta_tx_v0_hash` delegates to `deprecated_get_transaction_hash` with `additional_data_size=0` and `additional_data=cast(0, felt*)`:

```cairo
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(
    contract_address: felt,
    entry_point_selector: felt,
    calldata: felt*,
    calldata_size: felt,
    chain_id: felt,
) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        ...
        additional_data_size=0,          // ← nonce is NOT included
        additional_data=cast(0, felt*),
    );
    return tx_hash;
}
``` [1](#0-0) 

Compare this to `compute_l1_handler_transaction_hash`, which correctly passes `additional_data_size=1, additional_data=&nonce`, binding the hash to a specific nonce: [2](#0-1) 

The meta-tx-v0 hash therefore commits only to `(prefix, version=0, contract_address, selector, calldata_hash, max_fee=0, chain_id)` — no nonce.

At the OS level, `check_and_increment_nonce` explicitly skips all version-0 transactions:

```cairo
func check_and_increment_nonce{contract_state_changes: DictAccess*}(tx_info: TxInfo*) -> () {
    // Do not handle nonce for version 0.
    if (tx_info.version == 0) {
        return ();
    }
    ...
}
``` [3](#0-2) 

`execute_meta_tx_v0` constructs the new `TxInfo` with `nonce=0` and never calls `check_and_increment_nonce` for the target contract: [4](#0-3) 

The result is that the OS provides **zero replay protection** for meta-tx-v0 signatures. Any signature that was valid once remains valid forever for the same `(contract_address, selector, calldata, chain_id)` tuple.

---

### Impact Explanation

**Critical — Direct loss of funds.**

A victim contract C holds funds and has previously had a meta-tx-v0 executed on its behalf (e.g., a token transfer). An attacker who observes the signature can:

1. Deploy a contract that issues the `meta_tx_v0` syscall with the same `(contract_address=C, selector=__execute__, calldata, signature)`.
2. The OS recomputes the identical hash (no nonce), so the signature validates.
3. C's `__execute__` runs again, executing the same transfer.
4. Repeat until C is drained.

Because the OS enforces no nonce for version-0 and the hash commits to no nonce, there is no on-chain mechanism preventing this loop. The attacker pays only the gas cost of the outer transaction per replay.

---

### Likelihood Explanation

**Medium-High.** The `meta_tx_v0` syscall is callable by any deployed contract, making the entry path fully reachable by an unprivileged transaction sender. The only prerequisite is observing a prior valid meta-tx-v0 signature for the target contract, which is publicly visible on-chain once the original transaction is included in a block. Any v0 account contract that does not implement its own nonce storage (a common pattern for legacy Cairo 0 accounts) is immediately vulnerable.

---

### Recommendation

Include the nonce in the meta-tx-v0 hash, mirroring the pattern used by `compute_l1_handler_transaction_hash`:

```cairo
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(
    contract_address: felt,
    entry_point_selector: felt,
    calldata: felt*,
    calldata_size: felt,
    chain_id: felt,
    nonce: felt,                          // ← add nonce parameter
) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        ...
        chain_id=chain_id,
        additional_data_size=1,           // ← commit to nonce
        additional_data=&nonce,
    );
    return tx_hash;
}
```

Additionally, `execute_meta_tx_v0` must read and increment the target contract's nonce in `contract_state_changes` before executing, so that each meta-tx-v0 can only be used once.

---

### Proof of Concept

**Setup:**
- Victim contract `C` (a v0 Cairo account) holds 1000 STRK.
- A legitimate outer transaction previously triggered `meta_tx_v0` on `C` with `calldata = [transfer(attacker, 100)]` and a valid ECDSA signature `σ`.

**Attack:**
1. Attacker deploys `ReplayContract` whose `__execute__` issues:
   ```
   meta_tx_v0(contract_address=C, selector=__execute__, calldata=[transfer(attacker,100)], signature=σ)
   ```
2. Attacker submits 10 transactions calling `ReplayContract.__execute__`.
3. Each time, the OS calls `compute_meta_tx_v0_hash(C, __execute__, calldata, chain_id)` → same hash → `σ` validates → C transfers 100 STRK to attacker.
4. After 10 replays, attacker has received 1000 STRK; C is drained.
