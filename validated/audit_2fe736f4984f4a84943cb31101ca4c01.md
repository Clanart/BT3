### Title
Missing Declared Class Hash Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS Cairo program accepts an arbitrary attacker-controlled class hash and writes it directly into the contract state without verifying that the supplied hash corresponds to a previously declared contract class. This is the OS-level analog of SQL injection: unsanitized user-controlled input (`request.class_hash`) is injected into a privileged state-mutation operation (`dict_update` on `contract_state_changes`) without parameterized/bounded validation. The missing check is explicitly acknowledged by a `TODO` comment in the production code. If the supplied class hash is not present in the compiled class facts bundle, the affected contract becomes permanently uncallable, permanently freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` reads the new class hash directly from the syscall request buffer — which is populated by the executing contract (attacker-controlled) — and writes it into the OS state dictionary with no check that the hash corresponds to a declared class:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The `class_hash` value originates from `request.class_hash`, which is written by the calling contract into the syscall segment — fully attacker-controlled. No `assert`, `find_element`, or membership check is performed against the compiled class facts bundle before the state update is committed.

Contrast this with `execute_entry_point`, which correctly uses `find_element` to look up the compiled class hash in the bundle before dispatching execution:

```cairo
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [2](#0-1) 

The OS proves the state transition including the new class hash. Once the proof is accepted on L1, the contract's class hash in the canonical state is permanently set to the undeclared value. Any subsequent call to that contract will fail at the `find_element` lookup in `execute_entry_point` because the hash does not exist in the compiled class facts bundle, making the contract permanently uncallable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

A contract that holds ERC-20 balances, NFTs, or acts as a vault/escrow can have its class hash replaced with an arbitrary undeclared felt value. Once committed to the proven state:

- Every future call to the contract reaches `execute_entry_point`, which calls `find_element` on the compiled class facts bundle with the invalid hash.
- `find_element` fails (no matching entry), making the contract permanently non-executable.
- All assets stored in the contract's storage (token balances, locked funds, etc.) are permanently inaccessible — no withdrawal, transfer, or upgrade path remains.
- The state is L1-finalized via the STARK proof; there is no rollback mechanism.

---

### Likelihood Explanation

**Medium.**

- The attacker entry path requires only deploying a contract and submitting a transaction — both are unprivileged operations available to any network participant.
- The OS Cairo code is the authoritative proof layer. The missing check is explicitly marked as a `TODO` in production code, confirming it is not enforced at the proof level.
- The blockifier (off-chain execution engine) may independently reject such transactions before they reach the OS prover. However, the OS is the canonical enforcement boundary for what is proven on-chain; relying solely on the blockifier creates a defense-in-depth gap. Any blockifier bug, version mismatch, or sequencer deviation that allows the transaction through would result in a provably valid but semantically invalid state transition.
- The `TODO` comment targeting `1/1/2026` indicates this is a known, unresolved gap in the current production code.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that it exists in the compiled class facts bundle. The check should mirror the lookup already performed in `execute_entry_point`:

```cairo
// Verify the new class hash is a declared class.
let compiled_class_facts_bundle = block_context.os_global_context.compiled_class_facts_bundle;
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
// compiled_class_hash == 0 means undeclared; reject.
with_attr error_message("replace_class: class hash not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

Alternatively, use `find_element` against `compiled_class_facts_bundle.compiled_class_facts` with `key=compiled_class_hash` (after resolving via `contract_class_changes`) and assert success before proceeding with the state update. This closes the injection path by "parameterizing" the class hash input against the set of valid declared classes — the direct analog of using parameterized queries to prevent SQL injection.

### Proof of Concept

1. **Attacker deploys** a Cairo contract `MaliciousContract` whose `__execute__` function calls the `replace_class` syscall with `class_hash = 0xdeadbeef` (an arbitrary undeclared felt).

2. **Attacker submits** an invoke transaction targeting `MaliciousContract.__execute__`.

3. **OS processes** the transaction. `execute_syscalls` dispatches to `execute_replace_class`:
   - `class_hash = request.class_hash = 0xdeadbeef` (attacker-supplied).
   - The TODO-guarded check is absent; no membership validation occurs.
   - `dict_update` writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes` for the victim contract address.

4. **State commitment** (`compute_contract_state_commitment` in `state/commitment.cairo`) hashes the new `StateEntry` including `class_hash=0xdeadbeef` into the Patricia tree.

5. **Proof is generated and verified on L1.** The canonical state now records `class_hash=0xdeadbeef` for the victim contract.

6. **Any subsequent call** to the victim contract reaches `execute_entry_point`:
   - `dict_read` on `contract_class_changes` with key `0xdeadbeef` returns 0 (undeclared).
   - `find_element` on the compiled class facts bundle with key `0` fails — no matching compiled class exists.
   - The contract is permanently uncallable; all funds are permanently frozen. [3](#0-2) [4](#0-3) [5](#0-4)
