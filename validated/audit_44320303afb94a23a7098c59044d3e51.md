### Title
`ProofOfInclusion::valid()` Only Checks Internal Self-Consistency, Not Against a Trusted Root — Forged Inclusion Proofs Always Pass - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate contains a tautological final check: after verifying each layer's `combined_hash` is correctly computed, it checks `existing_hash == self.root_hash()`, where `root_hash()` returns the last layer's own `combined_hash`. Because `existing_hash` at loop exit already equals that same value (proven by the loop invariant), the final check is always `true`. The method therefore only validates internal self-consistency of the proof, never that the proof's root matches any external trusted root. An attacker can construct a fully internally-consistent `ProofOfInclusion` for any arbitrary `node_hash` and any arbitrary root, and `valid()` will return `true`.

### Finding Description

`ProofOfInclusion::valid()` is defined as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()   // ← always true if loop passes
}
``` [1](#0-0) 

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

**The tautology:** After the loop, `existing_hash` equals the last `calculated_hash`. The loop body already asserts `calculated_hash == layer.combined_hash` for every layer (returning `false` otherwise). Therefore at loop exit, `existing_hash == last_layer.combined_hash`. Since `root_hash()` returns exactly `last_layer.combined_hash`, the final comparison `existing_hash == self.root_hash()` reduces to `last_layer.combined_hash == last_layer.combined_hash`, which is unconditionally `true`.

An attacker can craft a `ProofOfInclusion` with:
- Any chosen `node_hash` (the leaf they wish to falsely claim is included)
- Any sequence of `ProofOfInclusionLayer` values where each `combined_hash` is correctly computed from the previous hash and a chosen `other_hash`

Such a proof passes `valid()` regardless of what the actual DataLayer tree root is.

`ProofOfInclusion` is a `Streamable` type exposed to Python via PyO3 bindings, meaning it can be deserialized from untrusted network bytes and passed to `valid()`. [3](#0-2) 

The Python binding exposes both `valid()` and `root_hash()` as separate methods, but the name `valid()` implies completeness. All existing usages in the codebase call `proof.valid()` without separately checking `proof.root_hash()` against a trusted external root: [4](#0-3) [5](#0-4) 

### Impact Explanation

A DataLayer peer or client that receives a `ProofOfInclusion` from an untrusted source and calls `proof.valid()` to verify it will accept any internally-consistent forged proof. The attacker can claim any key-value pair is present in a DataLayer store whose on-chain root hash they do not control. This allows untrusted input to prove invalid state — matching the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

### Likelihood Explanation

**High.** The method is named `valid()`, strongly implying it is the complete validation step. The Python binding exposes it as a standalone callable. Every test, fuzz target, and usage pattern in the repository calls `proof.valid()` without a separate root-hash check. Any consumer following the established pattern is vulnerable. The `ProofOfInclusion` struct is `Streamable` (deserializable from bytes), so an attacker can trivially craft and transmit a forged proof over the network.

### Recommendation

The `valid()` method must accept a trusted root hash as a parameter and compare against it, rather than comparing against the proof's own last `combined_hash`:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against EXTERNAL trusted root
}
```

The existing `valid()` method (which only checks internal consistency) should be renamed to `is_internally_consistent()` or removed to prevent misuse.

### Proof of Concept

```rust
use chia_datalayer::{Hash, MerkleBlob, KeyId, ValueId, InsertLocation};
use chia_datalayer::merkle::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};
use chia_datalayer::Side;

// Attacker constructs a forged proof for node_hash = [0xAA; 32]
// claiming it is in a tree with root [0xBB; 32] — neither is true.
let fake_leaf: Hash = [0xAA; 32];
let fake_other: Hash = [0x11; 32];

// Compute combined_hash honestly so the loop passes
let combined = chia_datalayer::calculate_internal_hash(&fake_leaf, Side::Right, &fake_other);

let forged_proof = ProofOfInclusion {
    node_hash: fake_leaf,
    layers: vec![ProofOfInclusionLayer {
        other_hash_side: Side::Right,
        other_hash: fake_other,
        combined_hash: combined,   // attacker sets this to match
    }],
};

// valid() returns true even though fake_leaf is in no real tree
assert!(forged_proof.valid());
// root_hash() returns `combined`, not any real tree root
assert_eq!(forged_proof.root_hash(), combined);
```

The loop verifies `calculated_hash == layer.combined_hash` (passes, since the attacker set them equal), then the final check compares `existing_hash` (= `combined`) against `root_hash()` (= `combined`): always `true`.
