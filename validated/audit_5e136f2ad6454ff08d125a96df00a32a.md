### Title
Missing Declared-Class Validation in `execute_replace_class` Enables Permanent Fund Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS Cairo program accepts any arbitrary class hash as the replacement target without verifying that the hash corresponds to a previously declared class. This is structurally analogous to the original report's pattern: a state flag is written without enforcing the invariant that should gate it, so the "current" state can be set to an invalid value that bypasses a downstream check. The consequence is that any contract can be permanently bricked — and any funds it holds permanently frozen — by a single unprivileged transaction.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` updates the contract's `StateEntry.class_hash` field in `contract_state_changes` with whatever hash the caller supplies:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The same omission exists in the deprecated path: [2](#0-1) 

The downstream consumer of this state is `execute_entry_point`, which reads the class hash from the contract's state entry and then performs a `dict_read` on `contract_class_changes` to obtain the compiled class hash, followed by a `find_element` lookup into the block's compiled-class bundle:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [3](#0-2) 

If the class hash stored in the contract's state entry was never declared, `dict_read` returns 0 (the default for an uninitialized key). `find_element` then searches for key `0` in the compiled-class facts array. Because `find_element` panics (fails the Cairo assertion) when the key is absent, any subsequent call to the bricked contract causes the OS proof to be unprovable for that block. The sequencer is forced to exclude every transaction that touches the contract, making the contract permanently uncallable and any funds it holds permanently frozen.

The root cause mirrors the original report exactly: the OS writes a new "current state" (the class hash) without creating a new checkpoint or enforcing the invariant that the value must be valid. The downstream check (the `find_element` lookup) reads only the current state and cannot distinguish between a legitimately replaced class and an attacker-supplied garbage hash.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value, the OS cannot prove any block that includes a call to that contract. The sequencer permanently excludes all such transactions. Any ERC-20 balances, NFTs, or protocol-level collateral held by the contract become irrecoverable. The state commitment is updated with the invalid class hash, so the damage persists across all future blocks.

---

### Likelihood Explanation

The attack requires only a single transaction from any account that controls (or can invoke `replace_class` on) a contract holding funds. The syscall is available to all Sierra contracts and all deprecated contracts. The OS explicitly documents the missing check as a TODO, confirming it is not enforced at the protocol level. No privileged role, leaked key, or external dependency is required.

---

### Recommendation

Before writing the new `StateEntry` in `execute_replace_class`, assert that the supplied `class_hash` is present in `contract_class_changes` (i.e., that `dict_read{dict_ptr=contract_class_changes}(key=class_hash)` returns a non-zero compiled class hash). This mirrors the existing pattern used in `execute_declare_transaction`, which enforces `prev_value=0` to guarantee the class was not previously declared. The check should be added in both `syscall_impls.cairo` and `deprecated_execute_syscalls.cairo`.

---

### Proof of Concept

1. Attacker deploys contract `A` (class hash `H_A`, declared). Contract `A` holds user funds.
2. Attacker sends a transaction that calls `replace_class(H_B)` from within contract `A`, where `H_B` is any felt value that has never been declared.
3. The OS executes `execute_replace_class`: no check on `H_B` is performed; `contract_state_changes` is updated so contract `A` now has `class_hash = H_B`.
4. The block is proven successfully — the `replace_class` call itself does not cause a proof failure.
5. In any subsequent block, a transaction calls contract `A`. `execute_entry_point` reads `class_hash = H_B`, calls `dict_read(key=H_B)` → returns `0`, then calls `find_element(key=0)` on the compiled-class bundle → panics because `0` is not a valid compiled class hash.
6. The OS proof for that block fails. The sequencer drops every transaction touching contract `A`.
7. All funds in contract `A` are permanently frozen with no recovery path.
