Looking at the exact code path in `revert.cairo` and `syscall_impls.cairo`:

### Title
Storage Write with Key `CHANGE_CLASS_ENTRY` Corrupts Class Hash During Revert — (`crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/revert.cairo`, `syscall_impls.cairo`)

---

### Summary

`execute_storage_write` appends a `RevertLogEntry` using the raw, unvalidated storage key as the `selector` field. Because `revert_contract_changes` uses the same `selector` field to distinguish storage-write entries from class-hash-change entries, a storage write with `key = CHANGE_CLASS_ENTRY (2**251+1)` is indistinguishable from a `replace_class` revert entry. On revert, the contract's `class_hash` is overwritten with the old storage value at that slot (typically `0`), permanently breaking dispatch and freezing all funds.

---

### Finding Description

**Root cause — no key range validation in `execute_storage_write`:** [1](#0-0) 

```cairo
tempvar storage_key = request.key;          // ← raw, unvalidated felt
assert [storage_ptr] = DictAccess(
    key=storage_key, prev_value=prev_value, new_value=request.value
);
...
assert [revert_log] = RevertLogEntry(selector=storage_key, value=prev_value);
```

There is no `assert_lt`, `assert_nn_le`, or any other constraint that prevents `storage_key` from equalling `CHANGE_CLASS_ENTRY` (`2**251+1`) or `CHANGE_CONTRACT_ENTRY` (`2**251`).

**Sentinel constants that the revert log relies on:** [2](#0-1) 

**Revert log entry format — the ambiguity:** [3](#0-2) 

The `selector` field serves double duty: it is either a storage key (entry type 3) or one of the two sentinel values (entry types 1 and 2). There is no out-of-band tag.

**`revert_contract_changes` — the misinterpretation:** [4](#0-3) 

When processing backwards, the function checks `selector == CHANGE_CLASS_ENTRY` **before** treating the entry as a storage write. A storage-write entry whose key happens to equal `CHANGE_CLASS_ENTRY` is therefore consumed as a class-hash restoration, setting `class_hash` to `prev_value` (the old storage value at that slot, typically `0`).

**Legitimate `replace_class` log entry for comparison:** [5](#0-4) 

Both a real `replace_class` and a crafted `storage_write(key=CHANGE_CLASS_ENTRY)` produce an identical `RevertLogEntry`; the revert logic cannot tell them apart.

---

### Impact Explanation

After the corrupted revert, `handle_revert` calls `dict_update` with the wrong `class_hash`: [6](#0-5) 

If `class_hash` is set to `0` (the default value of a never-written storage slot), the contract's state entry permanently records class hash `0`. Every subsequent call to the contract will attempt to dispatch through a non-existent class, causing all calls to fail. Any ERC-20 balances, ETH, or other assets held by the contract become permanently inaccessible — **permanent freezing of funds**.

---

### Likelihood Explanation

The attack requires a transaction that:
1. Issues `storage_write(key=2**251+1)` — no Sierra/CASM-level constraint prevents this; the key is a plain `felt` argument.
2. Reverts (e.g., the contract panics immediately after the write).

The OS itself imposes no key-range constraint. Whether an unprivileged attacker can reach this path end-to-end depends on whether the sequencer enforces `key < 2**251` before block inclusion. The OS code — which is the authoritative proven artifact — contains no such guard, making the invariant breakable at the proof level regardless of sequencer-side filtering.

---

### Recommendation

Add an explicit upper-bound assertion on the storage key inside `execute_storage_write`, immediately after reading `request.key`:

```cairo
// In execute_storage_write, after: tempvar storage_key = request.key;
assert_lt(storage_key, CONTRACT_ADDRESS_UPPER_BOUND);  // key must be < 2**251
```

This single constraint closes the ambiguity: storage keys are provably disjoint from both sentinel values (`CHANGE_CONTRACT_ENTRY = 2**251` and `CHANGE_CLASS_ENTRY = 2**251+1`).

---

### Proof of Concept

```
1. Declare and deploy contract C (class hash H ≠ 0).
2. Submit invoke tx calling C.__execute__:
     storage_write(key=2**251+1, value=1)   // key == CHANGE_CLASS_ENTRY
     panic()                                 // force revert
3. Sequencer includes tx; OS runs execute_storage_write:
     revert_log ← RevertLogEntry(selector=2**251+1, value=0)
                                             // 0 = old value of that slot
4. Transaction reverts; handle_revert called for C.
5. revert_contract_changes reads entry backwards:
     selector == CHANGE_CLASS_ENTRY → class_hash := 0
6. dict_update records StateEntry(class_hash=0, ...) for C.
7. Assert: contract_state_changes[C].class_hash == H  → FAILS (got 0).
8. All subsequent calls to C revert; funds frozen permanently.
```

```text
// Represents an entry of the revert log, which can be either:
// 1. contract address separator:
//   [CHANGE_CONTRACT_ENTRY, contact_address] - indicates that the preceding entries in the log
//   refer to the given `contract_address`.
// 2. change class entry - used to revert changes of class hash (due to deploy or replace_class):
//   [CHANGE_CLASS_ENTRY, old_class_hash]
// 3. storage write entry - used to revert changes to the contract's storage:
//   [storage_key, old_value]
//
// The first entry of the revert log is [CHANGE_CONTRACT_ENTRY, CONTRACT_ADDRESS_UPPER_BOUND].
struct RevertLogEntry {
    // Either the storage key, CHANGE_CONTRACT_ENTRY or CHANGE_CLASS_ENTRY.
    selector: felt,
    // The relevant (old) value.
    value: felt,
}
```

```text
func revert_contract_changes{
    class_hash: felt, storage_ptr: DictAccess*, revert_log_end: RevertLogEntry*
}() {
    alloc_locals;
    let revert_log_end = &revert_log_end[-1];

    tempvar selector = revert_log_end[0].selector;
    if (selector == CHANGE_CONTRACT_ENTRY) {
        // Change contract entries are handled by the caller.
        return ();
    }

    if (selector == CHANGE_CLASS_ENTRY) {
        // Change class entry.
        let class_hash = revert_log_end[0].value;
        return revert_contract_changes();
    }

    // Storage write entry.
    let storage_key = selector;
    let value = revert_log_end[0].value;
    local prev_value;
    %{ ReadStorageKeyForRevert %}
    assert storage_ptr[0] = DictAccess(key=storage_key, prev_value=prev_value, new_value=value);
    %{ WriteStorageKeyForRevert %}
    let storage_ptr = &storage_ptr[1];
    return revert_contract_changes();
```
