### Title
Missing shard-position check in `set_state_header` allows cross-shard state root substitution — (`chain/chain/src/state_sync/adapter.rs`)

---

### Summary

`set_state_header` verifies that the supplied chunk is *somewhere* in the block's Merkle tree via `verify_path`, but never asserts that the chunk occupies the tree position that corresponds to the requested `shard_id`. A malicious state-sync peer can supply a `ShardStateSyncResponseHeader` whose embedded chunk belongs to shard 0 while the call is made for `shard_id=1`. All existing guards pass, the header is stored under `StateHeaderKey(shard_id=1, sync_hash)`, and every subsequent `set_state_part` call for shard 1 validates parts against shard 0's `prev_state_root`, ultimately installing shard 0's state as shard 1's state.

---

### Finding Description

**The position-agnostic `verify_path` guard**

`set_state_header` performs the following Merkle check:

```rust
if !verify_path(
    *sync_prev_block_header.chunk_headers_root(),
    shard_state_header.chunk_proof(),
    &ChunkHashHeight(chunk.chunk_hash().clone(), chunk.height_included()),
) { … }
``` [1](#0-0) 

`verify_path` is defined as:

```rust
pub fn verify_path<T: BorshSerialize>(root: MerkleHash, path: &MerklePath, item: T) -> bool {
    verify_hash(root, path, CryptoHash::hash_borsh(item))
}
pub fn verify_hash(root: MerkleHash, path: &MerklePath, item_hash: MerkleHash) -> bool {
    compute_root_from_path(path, item_hash) == root
}
``` [2](#0-1) 

This only checks that the supplied chunk hash reconstructs the root via the supplied path. It does **not** check which leaf index the chunk occupies. The position-aware variant, `verify_path_with_index`, calls `verify_path_matches_index` to enforce the index: [3](#0-2) 

`set_state_header` uses the position-agnostic `verify_path`, not `verify_path_with_index`.

**No `chunk.shard_id() == shard_id` assertion**

`validate_chunk_proofs` checks hash integrity and body consistency (transactions, receipts) but contains no shard-id comparison: [4](#0-3) 

Nowhere in `set_state_header` is there an explicit `chunk.shard_id() == shard_id` guard.

**Storage under the wrong key**

After all checks pass, the header is stored under the caller-supplied `shard_id`:

```rust
let key = borsh::to_vec(&StateHeaderKey(shard_id, sync_hash)).unwrap();
store_update.set_ser(DBCol::StateHeaders, &key, &shard_state_header);
``` [5](#0-4) 

**Downstream corruption in `set_state_part`**

`set_state_part` reads the stored header and extracts the state root from the embedded chunk:

```rust
let shard_state_header = self.get_state_header(shard_id, sync_hash)?;
let chunk = shard_state_header.take_chunk();
let state_root = *chunk.take_header().take_inner().prev_state_root();
…
self.runtime_adapter.validate_state_part(shard_id, &state_root, part_id, part)
``` [6](#0-5) 

If the stored header contains shard 0's chunk, `state_root` is shard 0's `prev_state_root`. The attacker then supplies state parts that are valid for shard 0's state root; `validate_state_part` accepts them, and they are written to `DBCol::StateParts` under `StatePartKey(sync_hash, shard_id=1, part_id)`.

**Receipt proof validation does not block the attack**

The receipt proof loop hashes receipts as `ReceiptList(shard_id, receipts)` using the *argument* `shard_id`, not the chunk's shard id. The attacker can supply legitimately obtained incoming-receipt proofs for shard 1 (public blockchain data), satisfying steps 4a–4g independently of which chunk is embedded. [7](#0-6) 

---

### Impact Explanation

The exact corrupted value is `StateHeaderKey(shard_id=1, sync_hash)` → `ShardStateSyncResponseHeader` embedding shard 0's chunk and `prev_state_root`. Every `set_state_part` call for shard 1 validates against shard 0's state root. `set_state_finalize` then installs shard 0's trie as shard 1's canonical state, permanently corrupting the syncing node's view of shard 1.

---

### Likelihood Explanation

The attacker only needs to be a peer that the syncing node contacts for state headers — no validator or privileged role is required. All data needed to construct the malicious header (shard 0's chunk, its Merkle proof, shard 1's incoming receipt proofs, shard 0's `state_root_node`) is publicly available on-chain. The `StateHeaderValidationRequest → set_state_header` path is exercised on every state sync, making the window of opportunity broad. [8](#0-7) [9](#0-8) 

---

### Recommendation

Add an explicit shard-id guard immediately after extracting the chunk in `set_state_header`:

```rust
let chunk = shard_state_header.cloned_chunk();
// Guard: the embedded chunk must belong to the requested shard.
if chunk.shard_id() != shard_id {
    return Err(Error::Other(
        "set_shard_state failed: chunk shard_id does not match requested shard_id".into(),
    ));
}
```

Alternatively, replace the position-agnostic `verify_path` call with `verify_path_with_index`, passing the shard's index in the block as the expected position, mirroring the pattern already used in `validate_part` in `shards_manager_actor.rs`. [10](#0-9)

pub fn validate_chunk_proofs(
    chunk: &ShardChunk,
    epoch_manager: &dyn EpochManagerAdapter,
) -> Result<bool, Error> {
    let correct_chunk_hash = chunk.compute_header_hash();

    // 1. Checking chunk.header.hash
    let header_hash = chunk.header_hash();
    if header_hash != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }

    // 2. Checking that chunk body is valid
    // 2a. Checking chunk hash
    if chunk.chunk_hash() != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }
    let height_created = chunk.height_created();
    let outgoing_receipts_root = chunk.prev_outgoing_receipts_root();
    let (transactions, receipts) = (chunk.to_transactions(), chunk.prev_outgoing_receipts());

    // 2b. Checking that chunk transactions are valid
    let (tx_root, _) = merklize(transactions);
    if &tx_root != chunk.tx_root() {
        byzantine_assert!(false);
        return Ok(false);
    }
    // 2c. Checking that chunk receipts are valid
    if height_created == 0 {
        return Ok(receipts.is_empty() && outgoing_receipts_root == &CryptoHash::default());
    } else {
        let shard_layout = {
            let prev_block_hash = chunk.prev_block_hash();
            epoch_manager.get_shard_layout_from_prev_block(&prev_block_hash)?
        };
        let outgoing_receipts_hashes = Chain::build_receipts_hashes(receipts, &shard_layout)?;
        let (receipts_root, _) = merklize(&outgoing_receipts_hashes);
        if &receipts_root != outgoing_receipts_root {
            byzantine_assert!(false);
            return Ok(false);
        }
    }
    Ok(true)
}
