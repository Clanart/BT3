### Title
`processed_compiled_classes` deduplication by `CompiledClassHash` skips `add_compiled_class` for second `ClassHash` sharing same CASM — (`crates/apollo_central_sync/src/pending_sync.rs`)

### Summary

In `sync_pending_data`, the `processed_compiled_classes: HashSet<CompiledClassHash>` deduplicates compiled-class fetch tasks by `compiled_class_hash`. However, `PendingClasses.compiled_classes` is a `HashMap<ClassHash, …>` — keyed by `class_hash`. When two distinct `class_hash` values (H1, H2) share the same `compiled_class_hash` C, the second entry's `get_pending_compiled_class` call is silently skipped, leaving H2 with no entry in `pending_classes.compiled_classes`. Any RPC call that simulates or executes over the pending block using H2 then fails with `UndeclaredClassHash`.

### Finding Description

The deduplication guard in `sync_pending_data`:

```rust
if processed_compiled_classes.insert(compiled_class_hash) {   // dedup by CompiledClassHash
    tasks.push(
        get_pending_compiled_class(
            class_hash,          // but fetch+store is keyed by ClassHash
            ...
        ).boxed(),
    );
}
``` [1](#0-0) 

`get_pending_compiled_class` fetches and stores by `class_hash`:

```rust
let compiled_class = central_source.get_compiled_class(class_hash).await?;
pending_classes.write().await.add_compiled_class(class_hash, compiled_class);
``` [2](#0-1) 

`PendingClasses.compiled_classes` is a `HashMap<ClassHash, …>`: [3](#0-2) 

With `declared_classes = [{H1, C}, {H2, C}]`:
- H1: `insert(C)` → `true` → `get_pending_compiled_class(H1)` → CASM stored under H1 ✓
- H2: `insert(C)` → `false` → task skipped → **no entry for H2 in `compiled_classes`** ✗

### Impact Explanation

`ExecutionStateReader::get_compiled_class` for a pending block checks `pending_classes.get_class(H2)` (finds the Sierra class, since `processed_classes` deduplicates correctly by `class_hash`) and then `pending_classes.get_compiled_class(H2)` — which returns `None` because the entry was never stored: [4](#0-3) 

The code falls through to the class manager / committed storage path, which also has no entry for H2 (it is pending, not yet committed), yielding `StateError::UndeclaredClassHash(H2)`. Any `starknet_simulateTransactions`, `starknet_estimateFee`, or `starknet_traceTransaction` call over the pending block that touches H2 returns an authoritative-looking error or wrong result.

### Likelihood Explanation

The precondition requires two distinct Sierra classes whose CASM compilation produces the same `compiled_class_hash`. The Starknet protocol verifies that the `compiled_class_hash` in a declare transaction matches the actual CASM of the declared Sierra class, but does **not** prevent two different Sierra programs from compiling to identical CASM (Sierra compilation is not guaranteed to be injective). This is theoretically reachable by any unprivileged user who can submit two declare transactions, but practically very difficult to engineer deliberately. Likelihood is therefore **low**.

### Recommendation

Replace the `compiled_class_hash`-keyed deduplication with `class_hash`-keyed deduplication (consistent with `processed_classes`), or remove `processed_compiled_classes` entirely and rely solely on `processed_classes` to gate the compiled-class fetch — since each `class_hash` maps to exactly one compiled class:

```rust
// Replace:
if processed_compiled_classes.insert(compiled_class_hash) {
    tasks.push(get_pending_compiled_class(class_hash, ...).boxed());
}
// With (already gated by processed_classes above):
// Simply always push get_pending_compiled_class when processed_classes.insert(class_hash) is true.
``` [5](#0-4) 

### Proof of Concept

The existing test harness in `sync_test.rs` already exercises this path. A minimal reproduction:

1. Construct `declared_classes = [DeclaredClassHashEntry { class_hash: H1, compiled_class_hash: C }, DeclaredClassHashEntry { class_hash: H2, compiled_class_hash: C }]` in a single `PendingData` response.
2. Set `MockCentralSourceTrait::expect_get_compiled_class().times(1)` (only one call expected, for H1).
3. Run `sync_pending_data`.
4. Assert `pending_classes.compiled_classes.contains_key(&H2)` — this assertion **fails**, confirming H2's compiled class is never stored. [6](#0-5)

async fn test_pending_sync(
    reader: StorageReader,
    old_pending_data: PendingData,
    new_pending_datas: Vec<PendingData>,
    expected_pending_data: PendingData,
    old_pending_classes_data: Option<PendingClasses>,
    // Verifies that the classes will be requested in the given order.
    new_pending_classes: Vec<(ClassHash, ApiContractClass)>,
    new_pending_compiled_classes: Vec<(ClassHash, CasmContractClass)>,
    expected_pending_classes: Option<PendingClasses>,
) {
    let mut mock_pending_source = MockPendingSourceTrait::new();
    let mut mock_central_source = MockCentralSourceTrait::new();
    let pending_data_lock = Arc::new(RwLock::new(old_pending_data));
    let pending_classes_lock = Arc::new(RwLock::new(old_pending_classes_data.unwrap_or_default()));

    for new_pending_data in new_pending_datas {
        mock_pending_source
            .expect_get_pending_data()
            .times(1)
            .return_once(move || Ok(new_pending_data));
    }

    for (expected_class_hash, new_pending_class) in new_pending_classes {
        mock_central_source.expect_get_class().times(1).return_once(move |class_hash| {
            assert_eq!(class_hash, expected_class_hash);
            Ok(new_pending_class)
        });
    }
    for (expected_class_hash, new_pending_compiled_class) in new_pending_compiled_classes {
        mock_central_source.expect_get_compiled_class().times(1).return_once(move |class_hash| {
            assert_eq!(class_hash, expected_class_hash);
            Ok(new_pending_compiled_class)
        });
    }

    sync_pending_data(
        reader,
        Arc::new(mock_central_source),
        Arc::new(mock_pending_source),
        pending_data_lock.clone(),
        pending_classes_lock.clone(),
        Duration::ZERO,
    )
    .await
    .unwrap();

    assert_eq!(pending_data_lock.read().await.clone(), expected_pending_data);
    if let Some(expected_pending_classes) = expected_pending_classes {
        assert_eq!(pending_classes_lock.read().await.clone(), expected_pending_classes);
    }
