### Title
`replace_class` Syscall Accepts Undeclared Class Hash Without Validation — (`File: execution/syscall_impls.cairo`)

---

### Summary

The StarkNet OS `execute_replace_class` function in `syscall_impls.cairo` accepts any arbitrary `class_hash` value from the caller and writes it directly into the contract state without verifying that the hash corresponds to a previously declared contract class. The codebase itself acknowledges this gap with an explicit TODO. An unprivileged contract can call `replace_class` with a non-existent class hash, permanently bricking itself and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads `class_hash` directly from the syscall request and updates `contract_state_changes` with no membership check against `contract_class_changes`:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The same missing check exists in the deprecated path: [2](#0-1) 

By contrast, the `execute_declare_transaction` function enforces that a class hash is cryptographically valid (computed from actual Sierra source) and that `prev_value=0` (declared only once) before writing to `contract_class_changes`: [3](#0-2) [4](#0-3) 

`execute_replace_class` performs no equivalent lookup into `contract_class_changes` to confirm the target class hash was ever declared.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

After `replace_class` succeeds with an undeclared hash, the contract's `StateEntry.class_hash` is committed to the global state Merkle tree with a value that has no corresponding entry in the class tree. Every subsequent call to that contract will fail at entry-point resolution because the OS cannot find bytecode for the class. The contract becomes permanently non-functional. Any ERC-20 tokens, ETH, or other assets held by the contract are irrecoverably locked with no upgrade or rescue path, since the upgrade mechanism itself is what was corrupted. [5](#0-4) 

---

### Likelihood Explanation

The `replace_class` syscall is callable by any contract from any unprivileged transaction. No admin key or special role is required — the caller is the contract itself. Realistic trigger paths include:

1. **Malicious upgrade contract**: A contract advertised as a DeFi vault implements an upgrade function that calls `replace_class`. The deployer (or anyone who can invoke the upgrade entry point) passes an arbitrary felt as the new class hash. Funds deposited by users are permanently frozen.
2. **Buggy upgrade logic**: A legitimate contract passes a class hash that was never declared (e.g., a hash computed off-chain incorrectly, or a hash from a different chain). The OS silently accepts it.
3. **Griefing via library call**: A malicious library contract, when called via `library_call`, triggers `replace_class` on the calling contract's address with a garbage hash.

The TODO comment at line 898 confirms the development team is aware this check is absent and deferred it to a future date (`1/1/2026`), meaning the gap is present in the current production code path. [6](#0-5) 

---

### Recommendation

Before writing the new `StateEntry`, verify that `class_hash` exists in `contract_class_changes` (i.e., it was previously declared). The check should mirror the invariant enforced during `execute_declare_transaction`: the class hash must map to a non-zero compiled class hash in the class tree. Concretely, perform a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert the returned compiled class hash is non-zero before proceeding with the state update.

---

### Proof of Concept

1. Declare a valid class `C` (class hash `H_valid`, compiled hash `CH`). This writes `H_valid → CH` into `contract_class_changes`.
2. Deploy a contract `A` with class `C`. `A` holds user funds.
3. `A` exposes an `upgrade(new_class_hash)` entry point that calls `replace_class(new_class_hash)`.
4. Call `upgrade(0xdeadbeef)` where `0xdeadbeef` is never declared.
5. `execute_replace_class` in `syscall_impls.cairo` (line 896–910) reads `class_hash = 0xdeadbeef`, skips the missing membership check (line 898 TODO), and writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes`.
6. The block is proven. The global state root now commits `A → class_hash=0xdeadbeef`.
7. Any subsequent invoke of `A` fails: the OS looks up `0xdeadbeef` in the class tree, finds no entry, and cannot resolve an entry point.
8. All funds in `A` are permanently frozen with no recovery mechanism. [7](#0-6)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),

    );

func hash_contract_state_changes{hash_ptr: HashBuiltin*, range_check_ptr}(
    contract_address: felt,
    prev_state: StateEntry*,
    new_state: StateEntry*,
    patricia_update_constants: PatriciaUpdateConstants*,
    hashed_state_changes: DictAccess*,
) {
    alloc_locals;

    local initial_contract_state_root;
    local final_contract_state_root;

    %{ SetPreimageForCurrentCommitmentInfo %}

    local state_dict_start: DictAccess* = prev_state.storage_ptr;
    local state_dict_end: DictAccess* = new_state.storage_ptr;
    local n_updates = (state_dict_end - state_dict_start) / DictAccess.SIZE;
    // Call patricia_update_using_update_constants() (or the read-optimized variant) instead of
    // patricia_update() in order not to repeat globals_pow2 calculation.
    local should_use_read_optimized: felt;
    %{ ShouldUseReadOptimizedPatriciaUpdate %}
    if (should_use_read_optimized != 0) {
        patricia_update_read_optimized(
            patricia_update_constants=patricia_update_constants,
            update_ptr=state_dict_start,
            n_updates=n_updates,
            height=MERKLE_HEIGHT,
            prev_root=initial_contract_state_root,
            new_root=final_contract_state_root,
        );
    } else {
        patricia_update_using_update_constants(
            patricia_update_constants=patricia_update_constants,
            update_ptr=state_dict_start,
            n_updates=n_updates,
            height=MERKLE_HEIGHT,
            prev_root=initial_contract_state_root,
            new_root=final_contract_state_root,
        );
    }
    local range_check_ptr = range_check_ptr;

    let (prev_value) = get_contract_state_hash(
        class_hash=prev_state.class_hash,
        storage_root=initial_contract_state_root,
        nonce=prev_state.nonce,
    );
    assert hashed_state_changes.prev_value = prev_value;
    let (new_value) = get_contract_state_hash(
        class_hash=new_state.class_hash,
        storage_root=final_contract_state_root,
        nonce=new_state.nonce,
    );

    assert hashed_state_changes.new_value = new_value;
    assert hashed_state_changes.key = contract_address;

    return ();
