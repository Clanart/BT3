### Title
`NewDynamicFeeChecker` Uses Stale `feemarketParams` Snapshot for Cosmos Transactions, Allowing Base-Fee Bypass - (File: `ante/evm/fee_checker.go`)

### Summary
`NewDynamicFeeChecker` is constructed once at ante-handler initialization time with a pointer to a local copy of `feemarketParams`. Because the EIP-1559 base fee is updated every block via `BeginBlock → SetBaseFee`, the captured pointer always reflects the genesis/initialization base fee, not the current block's base fee. Any unprivileged user can submit Cosmos SDK transactions whose `feeCap` satisfies the stale (lower) base fee but falls below the current base fee, causing those transactions to commit with insufficient fees.

### Finding Description

**Root cause — stale params captured at construction time**

In `evmd/ante/handler_options.go`, `newCosmosAnteHandler` is called once at app initialization:

```go
feemarketParams := options.FeeMarketKeeper.GetParams(ctx)   // local copy
...
txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
```

`feemarketParams` is a local `feemarkettypes.Params` value (not a live store reference). `&feemarketParams` is a pointer to that local copy, which is captured inside the closure returned by `NewDynamicFeeChecker`. [1](#0-0) 

**Per-transaction fee check uses the frozen pointer**

Inside the returned `TxFeeChecker`, every Cosmos transaction is validated against this frozen pointer:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
...
if feeCap.LT(baseFeeInt) {
    return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, ...)
}
```

`feemarketParams` here is the pointer to the initialization-time copy; it is never refreshed. [2](#0-1) 

**Base fee changes every block, but the checker never sees the change**

`BeginBlock` calls `CalculateBaseFee` and writes the new value to the KV store via `SetBaseFee → SetParams`:

```go
func (k *Keeper) BeginBlock(ctx sdk.Context) error {
    baseFee := k.CalculateBaseFee(ctx)
    ...
    k.SetBaseFee(ctx, baseFee)
``` [3](#0-2) 

`SetBaseFee` writes to the persistent KV store, but the `feemarketParams` pointer inside `NewDynamicFeeChecker` points to a Go struct in memory that is never updated. [4](#0-3) 

**Contrast with the EVM ante handler — which correctly fetches fresh params**

The EVM ante handler (`newEthAnteHandler`) calls `EVMBlockConfig` per transaction, fetching live params from the store on every invocation:

```go
blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
...
feemarketParams := &blockCfg.FeeMarketParams
baseFee := blockCfg.BaseFee
``` [5](#0-4) 

The Cosmos ante handler has no equivalent live-fetch step; it relies entirely on the stale pointer.

**`MinGasPriceDecorator` for Cosmos transactions has the same staleness**

`MinGasPriceDecorator` is also constructed with `&feemarketParams` at initialization:

```go
cosmos.NewMinGasPriceDecorator(options.FeeMarketKeeper, evmDenom, &feemarketParams),
``` [6](#0-5) 

Inside the decorator, `mpd.feemarketParams.MinGasPrice` is read from the same frozen pointer. [7](#0-6) 

### Impact Explanation

When the EIP-1559 base fee rises above its genesis value (which happens whenever blocks are consistently above the gas target), any user can craft a Cosmos SDK transaction whose `feeCap` is:

```
genesis_base_fee  ≤  feeCap  <  current_base_fee
```

`NewDynamicFeeChecker` accepts the transaction (stale check passes), while the correct current-block check would reject it. The transaction commits with fees that are materially below the protocol-mandated base fee. This is a fee-market ante-handler bug that permits invalid transactions to commit and causes user/validator funds to be mis-accounted — matching the **High** allowed impact category.

### Likelihood Explanation

- The condition is always present when `DynamicFeeChecker: true` is set in `HandlerOptions`.
- No privileged access is required; any unprivileged user submitting a Cosmos SDK transaction can exploit this.
- The base fee drifts from genesis immediately after the first blocks with above-target gas usage, so the window opens quickly on any active chain.
- The attacker only needs to know the genesis base fee (public) and submit a transaction with `feeCap` just above it.

### Recommendation

Replace the captured-at-init pointer pattern with a live store read inside the closure. The simplest fix is to pass the `FeeMarketKeeper` into `NewDynamicFeeChecker` and call `GetParams(ctx)` (or `GetBaseFee(ctx)`) on every invocation, mirroring what `newEthAnteHandler` already does via `EVMBlockConfig`:

```go
// ante/evm/fee_checker.go
func NewDynamicFeeChecker(ethCfg *params.ChainConfig, evmParams *types.Params,
    fmKeeper FeeMarketKeeper) authante.TxFeeChecker {
    return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
        feemarketParams := fmKeeper.GetParams(ctx)          // live read
        baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, &feemarketParams)
        ...
    }
}
```

Apply the same fix to `MinGasPriceDecorator`.

### Proof of Concept

1. Deploy an Ethermint chain with `DynamicFeeChecker: true`, genesis `base_fee = 100_000_000_000`.
2. Submit enough EVM transactions to fill blocks above the gas target for several blocks. The base fee rises to, say, `200_000_000_000` (stored in the KV store, visible via `eth_gasPrice`).
3. Construct a Cosmos SDK transaction (e.g., `bank.MsgSend`) with `fee = 150_000_000_000 * gasLimit` (above genesis base fee, below current base fee).
4. Submit via `BroadcastTx`. The `NewDynamicFeeChecker` evaluates `feeCap = 150_000_000_000 >= stale_baseFee = 100_000_000_000` → passes. The transaction is included in a block despite paying only 75% of the required base fee.
5. Confirm: querying `feemarket` params shows `base_fee = 200_000_000_000`, yet the transaction was accepted at `150_000_000_000`. [8](#0-7) [9](#0-8)

func NewDynamicFeeChecker(ethCfg *params.ChainConfig, evmParams *types.Params, feemarketParams *feemarkettypes.Params) authante.TxFeeChecker {
	return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
		feeTx, ok := tx.(sdk.FeeTx)
		if !ok {
			return nil, 0, fmt.Errorf("tx must be a FeeTx")
		}

		if ctx.BlockHeight() == 0 {
			// genesis transactions: fallback to min-gas-price logic
			return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
		}

		denom := evmParams.EvmDenom

		baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
		if baseFee == nil {
			// london hardfork is not enabled: fallback to min-gas-prices logic
			return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
		}

		// default to `MaxInt64` when there's no extension option.
		maxPriorityPrice := sdkmath.NewInt(math.MaxInt64)

		// get the priority tip cap from the extension option.
		if hasExtOptsTx, ok := tx.(authante.HasExtensionOptionsTx); ok {
			for _, opt := range hasExtOptsTx.GetExtensionOptions() {
				if extOpt, ok := opt.GetCachedValue().(*ethermint.ExtensionOptionDynamicFeeTx); ok {
					maxPriorityPrice = extOpt.MaxPriorityPrice
					break
				}
			}
		}

		if maxPriorityPrice.Sign() == -1 {
			return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, "priority fee is negative")
		}

		gas := feeTx.GetGas()
		feeCoins := feeTx.GetFee()
		fee := feeCoins.AmountOf(denom)

		feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
		baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)

		if feeCap.LT(baseFeeInt) {
			return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, "insufficient gas prices; got: %s required: %s", feeCap, baseFeeInt)
		}

		// calculate the effective gas price using the EIP-1559 logic.
		effectivePrice := sdkmath.NewIntFromBigInt(types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))

		// NOTE: create a new coins slice without having to validate the denom
		effectiveFee := sdk.Coins{
			{
				Denom:  denom,
				Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas)),
			},
		}

		bigPriority := effectivePrice.Sub(baseFeeInt).Quo(types.DefaultPriorityReduction)
		priority := int64(math.MaxInt64)

		if bigPriority.IsInt64() {
			priority = bigPriority.Int64()
		}

		return effectiveFee, priority, nil
	}
}
