### Title
Missing Class Existence Validation in `execute_replace_class` Allows Permanent Freezing of Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary class hash from a calling contract without verifying that the hash corresponds to a previously declared contract class. This missing enforcement — explicitly acknowledged by a `TODO` comment in the production OS code — is the direct analog to the external report's pattern: a documented protocol constraint (replace_class must target a declared class) is not enforced in the OS code. Any unprivileged contract deployer can trigger this path, permanently freezing funds held by the affected contract.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` processes the syscall and writes the caller-supplied `class_hash` directly into the contract's `StateEntry` without any existence check:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The `TODO` comment is the OS team's own acknowledgment that this check is missing from production code. Compare this to `execute_declare_transaction`, which enforces `prev_value=0` to guarantee a class is declared only once, and `deploy_contract`, which asserts `state_entry.class_hash = UNINITIALIZED_CLASS_HASH` before deployment. The `replace_class` path has no analogous enforcement. [2](#0-1) [3](#0-2) 

Once a contract's `class_hash` is set to an undeclared value, any subsequent call to that contract reaches `execute_entry_point`, which performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [4](#0-3) 

`dict_read` returns 0 for an undeclared key. `find_element` with key `0` will fail if no compiled class with hash `0` exists, causing the OS proof to be unsatisfiable for any block that includes a call to the affected contract.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

A contract that successfully calls `replace_class` with an undeclared class hash has its on-chain `class_hash` permanently set to an undeclared value. The contract becomes permanently non-executable: no entry point can be dispatched, no withdrawal or transfer function can be called, and any tokens or ETH held by the contract are irrecoverably locked. The OS proof system provides no mechanism to undo a committed state-tree update once the block is finalized on L1.

---

### Likelihood Explanation

The `replace_class` syscall is reachable by any deployed contract — no privileged role
