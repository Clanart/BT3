### Title
`MerkleBlob::batch_insert` Skips Duplicate-Key and Duplicate-Hash Checks, Corrupting the DataLayer Merkle Tree Root - (File: `crates/chia-datalayer/src/merkle/blob.rs`)

### Summary

`MerkleBlob::batch_insert` bypasses the duplicate-key and duplicate-hash collision guards that are enforced by the single-item `insert` path. An untrusted caller supplying a batch that contains a repeated `KeyId` (or a `KeyId` already present in the tree) causes multiple leaf nodes with the same key to be written directly into the blob storage, corrupting the Merkle tree root and invalidating all subsequent proofs of inclusion and exclusion.

### Finding Description

The regular `insert` function guards against both forms of collision before touching the blob:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 369-374
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`batch_insert` uses `insert` only for the first two items (when the tree has ≤ 1 leaf). All remaining items in the batch are written directly to the blob via `insert_entry_to_blob` with **no collision check at all**:

```rust
// lines 587-603 – no contains_key / contains_leaf_hash guard
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ..., node: Node::Leaf(LeafNode { hash, key, value }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
``` [2](#0-1) 

This means:

1. **Intra-batch duplicates**: if the same `KeyId` appears more than once in a batch of ≥ 3 entries, both leaf blocks are written to the blob. The `block_status_cache` ends up with only one entry for that key (whichever `insert_entry_to_blob` wrote last), while the other leaf is an orphan in the blob that is still incorporated into the internal-node hash chain.

2. **Cross-batch duplicates**: a key already present in the tree from a prior `insert` call is not detected; a second leaf for the same key is appended and wired into the tree via `insert_subtree_at_key`.

In both cases the computed root hash covers the phantom duplicate leaf, so the root diverges from what a correct single-key tree would produce. `get_proof_of_inclusion` returns a proof anchored to the cached (single) leaf, but that proof's root hash is the corrupted one. Any verifier comparing the stored root against an independently computed root will see a mismatch. [3](#0-2)

### Impact Explanation

A corrupted `MerkleBlob` root means:

- **Forged inclusion proofs**: a proof generated by `get_proof_of_inclusion` passes `valid()` against the corrupted root, but the root itself does not match what an honest peer would compute for the same key-value set. A verifier that trusts the stored root accepts the proof; a verifier that recomputes the root from scratch rejects it — creating a cross-node disagreement on DataLayer state.
- **Corrupted tree root**: `check_integrity` will detect the inconsistency (`leaf_count != key_to_index_cache_length` or `leaf_count != leaf_hash_to_index_cache_length`), but `check_integrity` is not called automatically on every mutation; it is an explicit audit call.
- **Invalid exclusion proofs**: because the root is wrong, exclusion proofs for keys that are genuinely absent may also fail or be accepted incorrectly.

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."* [5](#0-4) 

### Likelihood Explanation

The DataLayer Python layer calls `batch_insert` when applying delta updates received from remote peers. A malicious or buggy peer that sends a delta containing a repeated key (or a key already present in the receiver's store) will trigger the unguarded fast path. No special privilege is required beyond the ability to participate in DataLayer data exchange. The fuzz targets for `insert` explicitly handle `KeyAlreadyPresent` and `HashAlreadyPresent`, confirming these are expected error conditions — but no equivalent fuzz target exercises `batch_insert` with colliding keys. [6](#0-5) 

### Recommendation

Add the same collision guards at the top of the fast-path loop in `batch_insert`:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // ... existing blob write ...
}
```

Alternatively, pre-validate the entire input vector for internal duplicates before touching the blob, and check each key against the cache, mirroring the contract of `insert`.

### Proof of Concept

```rust
let mut blob = MerkleBlob::new(vec![]).unwrap();

// Pre-populate so leaf_count > 1 (bypasses the guarded first-two-items path)
blob.insert(KeyId(100), ValueId(100), &sha256_num(&100i64), InsertLocation::Auto {}).unwrap();
blob.insert(KeyId(101), ValueId(101), &sha256_num(&101i64), InsertLocation::Auto {}).unwrap();

// batch with a duplicate key (KeyId(1) appears twice)
let batch = vec![
    ((KeyId(1), ValueId(10)), sha256_num(&1i64)),
    ((KeyId(2), ValueId(20)), sha256_num(&2i64)),
    ((KeyId(1), ValueId(99)), sha256_num(&99i64)), // duplicate of KeyId(1)
];
// Returns Ok(()) — no error raised
blob.batch_insert(batch).unwrap();

blob.calculate_lazy_hashes().unwrap();
// check_integrity() will now fail or the root will be inconsistent
// with a tree built by sequential insert() calls for the same keys.
```

The `batch_insert` call succeeds silently. The resulting root hash differs from the root that would be produced by inserting the same keys one at a time through `insert`, which would have rejected the second `KeyId(1)` with `KeyAlreadyPresent`. [2](#0-1) [1](#0-0)

    fn check_just_integrity(&self) -> Result<(), Error> {
        let mut leaf_count: usize = 0;
        let mut internal_count: usize = 0;
        let mut child_to_parent: HashMap<TreeIndex, TreeIndex> = HashMap::new();

        for item in ParentFirstIterator::new(&self.blob, None) {
            let (index, block) = item?;
            if let Some(parent) = block.node.parent().0 {
                if child_to_parent.remove(&index) != Some(parent) {
                    return Err(Error::IntegrityParentChildMismatch(index));
                }
            }
            match block.node {
                Node::Internal(node) => {
                    internal_count += 1;
                    child_to_parent.insert(node.left, index);
                    child_to_parent.insert(node.right, index);
                }
                Node::Leaf(node) => {
                    leaf_count += 1;
                    let cached_index = self
                        .block_status_cache
                        .get_index_by_key(node.key)
                        .ok_or(Error::IntegrityKeyNotInCache(node.key))?;
                    if *cached_index != index {
                        return Err(Error::IntegrityKeyToIndexCacheIndex(
                            node.key,
                            index,
                            *cached_index,
                        ));
                    }
                    assert!(
                        !self.block_status_cache.is_index_free(index),
                        "{}",
                        format!("active index found in free index list: {index:?}")
                    );
                }
            }
        }

        let key_to_index_cache_length = self.block_status_cache.key_to_index.len();
        if leaf_count != key_to_index_cache_length {
            return Err(Error::IntegrityKeyToIndexCacheLength(
                leaf_count,
                key_to_index_cache_length,
            ));
        }
        let leaf_hash_to_index_cache_length = self.block_status_cache.leaf_hash_to_index.len();
        if leaf_count != leaf_hash_to_index_cache_length {
            return Err(Error::IntegrityLeafHashToIndexCacheLength(
                leaf_count,
                leaf_hash_to_index_cache_length,
            ));
        }
        let total_count = leaf_count + internal_count + self.block_status_cache.free_index_count();
        let extend_index = self.extend_index();
        if total_count != extend_index.0 as usize {
            return Err(Error::IntegrityTotalNodeCount(extend_index, total_count));
        }
        if !child_to_parent.is_empty() {
            return Err(Error::IntegrityUnmatchedChildParentRelationships(
                child_to_parent.len(),
            ));
        }

        Ok(())
    }
