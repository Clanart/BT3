### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not validate that the caller-supplied class hash corresponds to a previously declared contract class. This is a direct analog to the reported vulnerability: just as the Wonderland modules accepted arbitrary `_request` data without validating it against the canonical `requestId`, the OS here accepts an arbitrary `class_hash` without checking it against the canonical `contract_class_changes` dictionary. A contract can replace its own class with any undeclared hash, permanently bricking itself and freezing all funds it holds.

---

### Finding Description

In `execute_replace_class` (lines 878–916 of `syscall_impls.cairo`), the new class hash is read directly from the syscall request and written into `contract_state_changes` with no validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment at line 898 is an explicit acknowledgment by the developers that this validation is absent. The `class_hash` value from `request.class_hash` is attacker-controlled and is never cross-checked against `contract_class_changes` (the dict that maps declared class hashes to their compiled class hashes).

When a subsequent call is made to the contract whose class was replaced with an undeclared hash, `execute_entry_point` performs the following lookup:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // arbitrary, undeclared hash → returns 0
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,           // key = 0, no such compiled class exists
);
``` [2](#0-1) 

Because `contract_class_changes` has no entry for the arbitrary hash, `dict_read` returns 0. `find_element` then searches for a compiled class with hash 0, which does not exist, causing the OS to be unable to produce a valid proof for any block containing a call to the bricked contract. The contract's state — including all held funds — is permanently inaccessible.

The `StateEntry` struct that is committed to the Merkle state tree contains the raw `class_hash` field with no on-chain enforcement that it must be declared:

```cairo
struct StateEntry {
    class_hash: felt,
    storage_ptr: DictAccess*,
    nonce: felt,
}
``` [3](#0-2) 

The state commitment (`get_contract_state_hash`) hashes this arbitrary class hash into the global state root without any validity check, making the corruption permanent and provable. [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any contract that (a) holds user funds and (b) exposes an upgrade path via `replace_class` where the new class hash is caller-supplied can have its class replaced with an arbitrary undeclared hash. Once committed to the state tree, the contract becomes permanently inaccessible: every future call to it will fail at the OS level because no compiled class can be resolved for the stored hash. All funds held by the contract are frozen with no recovery path.

---

### Likelihood Explanation

**Medium.** The `replace_class` syscall is callable only from within a contract's own execution context, so the attacker must be able to trigger it from the target contract. This is realistic in two scenarios:

1. **Malicious owner**: A contract owner deploys an upgradeable contract (e.g., a DeFi vault), attracts user deposits, then calls the upgrade function with an arbitrary undeclared class hash, freezing all deposited funds.
2. **Vulnerable upgrade function**: An upgradeable contract exposes a public or weakly-guarded upgrade function that passes the caller-supplied class hash directly to `replace_class`. An external attacker who can invoke this function can supply an invalid hash.

Both scenarios require no privileged protocol access — only the ability to submit a standard transaction.

---

### Recommendation

Implement the validation noted in the TODO comment at line 898. Before writing the new class hash into `contract_state_changes`, verify that it exists as a key in `contract_class_changes` (i.e., it has been declared):

```cairo
// Verify the class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the pattern already used in `execute_entry_point` and ensures that only valid, previously declared class hashes can be installed via `replace_class`.

---

### Proof of Concept

1. Attacker deploys **Contract A** — an upgradeable vault that accepts user deposits and exposes `upgrade(new_class_hash)` which calls `replace_class(new_class_hash)`.
2. Users deposit funds into Contract A.
3. Attacker calls `upgrade(0xdeadbeef)` where `0xdeadbeef` is an arbitrary value that was never declared on-chain.
4. The OS executes `execute_replace_class`: `request.class_hash = 0xdeadbeef` is written into `contract_state_changes[contract_a_address].class_hash` with no validation. The transaction succeeds and is committed to the state tree.
5. Any subsequent call to Contract A causes `execute_entry_point` to call `dict_read{dict_ptr=contract_class_changes}(key=0xdeadbeef)`, returning 0 (undeclared). `find_element(..., key=0)` finds no compiled class, making proof generation impossible for any block containing such a call.
6. Contract A is permanently inaccessible. All user funds are frozen with no recovery mechanism.

func get_contract_state_hash{hash_ptr: HashBuiltin*}(
    class_hash: felt, storage_root: felt, nonce: felt
) -> (hash: felt) {
    const CONTRACT_STATE_HASH_VERSION = 0;
    if (class_hash == UNINITIALIZED_CLASS_HASH) {
        if (storage_root == 0) {
            if (nonce == 0) {
                return (hash=0);
            }
        }
    }

    // Set res = H(H(class_hash, storage_root), nonce).
    let (hash_value) = hash2(class_hash, storage_root);
    let (hash_value) = hash2(hash_value, nonce);

    // Return H(hash_value, CONTRACT_STATE_HASH_VERSION). CONTRACT_STATE_HASH_VERSION must be in the
    // outermost hash to guarantee unique "decoding".
    let (hash) = hash2(hash_value, CONTRACT_STATE_HASH_VERSION);
    return (hash=hash);
