### Title
Fee-on-Transfer Token Accounting Mismatch Breaks Bridge Collateralization — (`evm/src/omni-bridge/contracts/OmniBridge.sol`)

### Summary

`OmniBridge.sol`'s `initTransfer` function records and emits the user-supplied `amount` parameter in the `InitTransfer` event, but for fee-on-transfer ERC20 tokens the contract actually receives `amount - transferFee`. The NEAR bridge finalizes the transfer using the emitted (inflated) amount, releasing more tokens than were ever locked on EVM. This breaks bridge collateralization.

### Finding Description

In `initTransfer`, when the token is a plain ERC20 (not a bridge token or custom minter), the contract executes: [1](#0-0) 

```solidity
} else {
    IERC20(tokenAddress).safeTransferFrom(
        msg.sender,
        address(this),
        amount          // user-supplied; actual receipt may be less
    );
}
```

Immediately after, the event is emitted with the same user-supplied `amount`: [2](#0-1) 

No balance-before/balance-after check is performed. For a fee-on-transfer token (e.g., USDT on some chains, STA, PAXG, etc.), `safeTransferFrom` deducts a fee from the transferred amount, so the bridge receives strictly less than `amount`. The emitted `InitTransfer` event still records the full `amount`.

On the NEAR side, `fin_transfer_callback` decodes the proof of this event and constructs a `TransferMessage` using the emitted amount directly: [3](#0-2) 

```rust
let transfer_message = TransferMessage {
    ...
    amount: Self::denormalize_amount(init_transfer.amount.0, decimals).into(),
    ...
};
```

NEAR then mints or releases `amount` tokens to the recipient — more than were ever locked on EVM. The EVM bridge's actual token balance is permanently short by `transferFee` per such transfer.

### Impact Explanation

Every fee-on-transfer token bridge-in inflates the NEAR-side supply relative to EVM-side reserves. Repeated transfers drain the EVM bridge's token balance. Eventually, legitimate users bridging back cannot withdraw their full entitlement — their funds are permanently frozen or lost. This matches the **Critical** impact class: irrecoverable lock of user funds and broken bridge collateralization.

### Likelihood Explanation

Any unprivileged user can call `initTransfer` with any ERC20 token address that has been registered (via `logMetadata` + NEAR-side token deployment). Fee-on-transfer tokens exist on mainnet and are not blocked by any allowlist or token-type check in `initTransfer`. No special role or leaked key is required — a normal bridge user triggers this by simply initiating a transfer with such a token.

### Recommendation

Measure the actual received amount using a balance-before/balance-after pattern:

```solidity
uint256 balanceBefore = IERC20(tokenAddress).balanceOf(address(this));
IERC20(tokenAddress).safeTransferFrom(msg.sender, address(this), amount);
uint256 actualReceived = IERC20(tokenAddress).balanceOf(address(this)) - balanceBefore;
require(actualReceived == amount, "Fee-on-transfer tokens not supported");
```

Alternatively, emit `actualReceived` instead of `amount` so the NEAR side finalizes based on what was truly locked. A token allowlist that explicitly excludes fee-on-transfer tokens is also a valid mitigation.

### Proof of Concept

1. A fee-on-transfer token (e.g., 1% fee) is registered with the bridge via `logMetadata` and deployed on NEAR.
2. Attacker calls `initTransfer(tokenAddress, 1000, 0, 0, nearRecipient, "")`.
3. `safeTransferFrom` transfers 1000 from attacker; token contract deducts 1% fee → bridge receives **990**.
4. `InitTransfer` event emits `amount = 1000`.
5. Relayer submits proof to NEAR `fin_transfer`; NEAR releases **1000** tokens to `nearRecipient`.
6. Bridge is now short 10 tokens per transfer.
7. After 100 such transfers, the bridge holds 99,000 tokens but NEAR has issued 100,000 — the 1,000-token shortfall is unrecoverable, permanently freezing the last legitimate withdrawer's funds. [4](#0-3) [5](#0-4)

        currentOriginNonce += 1;

            revert InvalidFee();

        uint256 extensionValue;
        if (tokenAddress == address(0)) {
            if (fee != 0) {
                revert InvalidFee();
            }
            extensionValue = msg.value - amount - nativeFee;

            extensionValue = msg.value - nativeFee;
            if (customMinters[tokenAddress] != address(0)) {

                    amount

    pub fn fin_transfer_callback(
        &mut self,
        #[serializer(borsh)] storage_deposit_actions: &Vec<StorageDepositAction>,
        #[serializer(borsh)] predecessor_account_id: AccountId,
    ) -> PromiseOrValue<Nonce> {
        let Ok(ProverResult::InitTransfer(init_transfer)) = Self::decode_prover_result(0) else {
            env::panic_str(BridgeError::InvalidProofMessage.to_string().as_str())
        };
        require!(
            self.factories
                .get(&init_transfer.emitter_address.get_chain())
                == Some(init_transfer.emitter_address),
            BridgeError::UnknownFactory.as_ref()
        );

        let decimals = self
            .token_decimals
            .get(&init_transfer.token)
            .near_expect(BridgeError::TokenDecimalsNotFound);

        let destination_nonce =
            self.get_next_destination_nonce(init_transfer.recipient.get_chain());
        let transfer_message = TransferMessage {
            origin_nonce: init_transfer.origin_nonce,
            token: init_transfer.token,
            amount: Self::denormalize_amount(init_transfer.amount.0, decimals).into(),
            recipient: init_transfer.recipient,
            fee: Self::denormalize_fee(&init_transfer.fee, decimals),
            sender: init_transfer.sender,
            msg: init_transfer.msg,
            destination_nonce,
            origin_transfer_id: None,
        };

        if let OmniAddress::Near(recipient) = transfer_message.recipient.clone() {
            self.process_fin_transfer_to_near(
                recipient,
                &predecessor_account_id,
                transfer_message,
                storage_deposit_actions,
            )
            .into()
        } else {
            self.process_fin_transfer_to_other_chain(predecessor_account_id, transfer_message);
            PromiseOrValue::Value(destination_nonce)
        }
    }
