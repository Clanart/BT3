### Title
Missing Declared Class Hash Validation in `execute_replace_class` Allows Permanent Contract Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the new class hash supplied by a contract is actually declared on-chain before committing the state update. The missing check is explicitly acknowledged by a TODO comment in the code. An unprivileged transaction sender can deploy a contract, invoke `replace_class` with an arbitrary undeclared class hash, and permanently render that contract un-executable — freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the requested class hash directly from the syscall request and writes it into `contract_state_changes` without any Cairo-level assertion that the hash exists in `contract_class_changes` (i.e., that it was previously declared via a `Declare` transaction):

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
``` [1](#0-0) 

The analogous check that *should* exist is performed for every class execution in `execute_entry_point`, which looks up the class hash in `contract_class_changes` and then calls `find_element` to locate the compiled class fact:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
``` [2](#0-1) 

If `class_hash` was never declared, `dict_read` returns 0 (the default), and `find_element` will fail to locate a compiled class with hash 0, making the contract permanently un-callable. The OS has no mechanism to recover from this state because `replace_class` writes no revert-log entry for the class hash change that would allow the sequencer to undo it in a future block — the state is committed.

The `replace_class` revert log entry *does* record the **old** class hash for within-transaction revert purposes:

```cairo
assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
``` [3](#0-2) 

But this only reverts within the same transaction if the transaction itself reverts. Once the transaction is finalized, the invalid class hash is committed to the global state tree permanently.

---

### Impact Explanation

**Impact: Critical — Permanent Freezing of Funds.**

Once a contract's class hash is set to an undeclared value, the sequencer cannot execute any future transaction targeting that contract (it cannot find the compiled class). The contract becomes permanently inert. Any ERC-20 balances, ETH, or other assets held in that contract's storage are irrecoverably frozen. No upgrade path exists because `replace_class` itself requires the contract to be executable.

---

### Likelihood Explanation

**Likelihood: High.**

The attack requires only a standard `Invoke` transaction from any unprivileged account. No special privilege, leaked key, or operator cooperation is needed. The attacker simply deploys a contract containing a call to `replace_class` with an arbitrary felt value as the class hash, then invokes it. The OS Cairo code imposes zero constraint on the value of `request.class_hash` beyond what the gas check enforces.

---

### Recommendation

Inside `execute_replace_class`, before committing the new class hash to `contract_state_changes`, add a Cairo-level assertion that the class hash exists in `contract_class_changes`:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the lookup already performed in `execute_entry_point` and closes the gap identified by the existing TODO comment.

---

### Proof of Concept

1. **Attacker deploys Contract A** — a contract whose constructor deposits user funds and whose body exposes a public function `freeze()` that executes the `replace_class` syscall with a felt value that has never been declared (e.g., `0xdeadbeef`).

2. **Users deposit funds** into Contract A (e.g., via an ERC-20 `transfer` or ETH deposit).

3. **Attacker calls `freeze()`** — this triggers `execute_replace_class` inside the OS. The OS reads `request.class_hash = 0xdeadbeef`, skips the missing declared-class check, and writes:
   ```
   contract_state_changes[contract_A_address].class_hash = 0xdeadbeef
   ```
   The transaction is valid and is included in the block. [4](#0-3) 

4. **State is committed** — the global state root now reflects Contract A with class hash `0xdeadbeef`.

5. **Any subsequent transaction targeting Contract A** causes the sequencer to attempt `execute_entry_point`, which calls `dict_read` on `contract_class_changes` for key `0xdeadbeef`, gets 0, then calls `find_element` for compiled class hash 0 — which is not present in `compiled_class_facts_bundle`. The sequencer cannot include any such transaction. [5](#0-4) 

6. **All funds in Contract A are permanently frozen.** No withdrawal, no upgrade, no recovery is possible.

        // Not enough gas to execute the syscall.
        return ();
