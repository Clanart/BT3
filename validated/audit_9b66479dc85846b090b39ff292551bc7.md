### Title
Non-EVM-Denom Cosmos Bank Tokens Permanently Locked After EVM Contract SELFDESTRUCT - (File: x/evm/statedb/statedb.go)

### Summary
When an EVM contract self-destructs, `StateDB.Commit()` burns only the EVM-native denomination balance and removes the auth account. Any non-EVM-denom Cosmos bank tokens (IBC assets, CosmWasm bridge tokens, etc.) held at the contract's Cosmos address are explicitly left as orphaned bank balances with no recovery path. The auth account is deleted, making those funds permanently inaccessible.

### Finding Description
In `x/evm/statedb/statedb.go`, the `Commit()` function handles self-destructed accounts by:
1. Burning the remaining EVM-denom balance via `keeper.SubBalance`
2. Deleting the auth account via `keeper.DeleteAccount`

The code contains an explicit acknowledgment of the gap:

```go
// Only the EVM denom is burned here. Non-EVM-native tokens (IBC, CosmWasm
// bridge) held by the destroyed address are not drained and may remain as
// orphaned bank balances.
if remaining := s.keeper.GetBalance(cacheCtx, cosmosAddr, s.evmDenom); remaining.Sign() > 0 {
    coin := sdk.NewCoin(s.evmDenom, sdkmath.NewIntFromBigInt(remaining.ToBig()))
    if _, err := s.keeper.SubBalance(cacheCtx, cosmosAddr, coin); err != nil {
        return errorsmod.Wrap(err, "failed to burn post-selfdestruct balance")
    }
}
if err := s.keeper.DeleteAccount(cacheCtx, obj.Address()); err != nil {
    return errorsmod.Wrap(err, "failed to delete account")
}
```

`DeleteAccount` in `x/evm/keeper/statedb.go` removes only auth metadata and contract storage — it never touches the bank module's balance store for non-EVM denoms:

```go
// NOTE: balance should be cleared separately
func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
    // clear storage
    k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool { ... })
    // remove auth account
    k.accountKeeper.RemoveAccount(ctx, acct)
    ...
}
```

After `RemoveAccount`, the Cosmos address has no associated private key, no code, and no auth record. Any non-EVM-denom bank balances at that address are stranded: the bank module still records them, but no entity can sign a transaction to move them, and no on-chain mechanism drains them.

### Impact Explanation
Cosmos bank funds denominated in non-EVM tokens (IBC assets such as ATOM, USDC, stATOM, etc.) sent to an EVM contract address are permanently locked when the contract self-destructs. The bank module retains the balance entry, but the auth account is gone, making the funds irrecoverable. This constitutes a mis-accounting of valid user funds through the EVM state transition path, matching the **High** impact category: *"EVM state transition... bug that permits... valid user funds/fees to be mis-accounted."*

### Likelihood Explanation
The attack path is fully unprivileged:
1. An attacker or user deploys an EVM contract (standard `CREATE`/`CREATE2`).
2. IBC tokens are transferred to the contract's Cosmos address (a normal `MsgTransfer` or bank send — no special permission required).
3. The contract calls `SELFDESTRUCT` (or the deployer calls a function that triggers it).
4. `StateDB.Commit()` burns only the EVM denom and deletes the auth account, leaving IBC tokens orphaned.

This is reachable on any Ethermint-based chain that supports IBC or any Cosmos-native token alongside EVM. The scenario is realistic wherever contracts act as escrows or receive multi-denom payments.

### Recommendation
In `StateDB.Commit()`, after burning the EVM-denom balance and before calling `DeleteAccount`, iterate over **all** bank balances of the self-destructed address and either burn them (if the module has burn permission for those denoms) or transfer them to a designated recovery address (e.g., the community pool or fee collector). The `BankKeeper.GetAllBalances` method can enumerate all denoms held at the address. This ensures no Cosmos bank funds are orphaned regardless of denomination.

```go
func (k *Keeper) SubBalance(ctx sdk.Context, addr sdk.AccAddress, coin sdk.Coin) (uint256.Int, error) {
	coins := sdk.NewCoins(coin)
	prevBalance := k.GetBalance(ctx, addr, coin.Denom)
	if err := k.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, coins); err != nil {
		return uint256.Int{}, err
	}
	if err := k.bankKeeper.BurnCoins(ctx, types.ModuleName, coins); err != nil {
		return uint256.Int{}, err
	}
	return prevBalance, nil
```

```go
// DeleteAccount handles contract's suicide call:
// - remove code
// - remove states
// - remove auth account
//
// NOTE: balance should be cleared separately
func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
	cosmosAddr := sdk.AccAddress(addr.Bytes())
	acct := k.accountKeeper.GetAccount(ctx, cosmosAddr)
	if acct == nil {
		return nil
	}

	// NOTE: only Ethereum accounts (contracts) can be selfdestructed
	_, ok := acct.(ethermint.EthAccountI)
	if !ok {
		return errorsmod.Wrapf(types.ErrInvalidAccount, "type %T, address %s", acct, addr)
	}

	// clear storage
	k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool {
		k.SetState(ctx, addr, key, nil)
		return true
	})

	// remove auth account
	k.accountKeeper.RemoveAccount(ctx, acct)

	k.debugLog(ctx, "account suicided",
		"ethereum-address", addr,
		"cosmos-address", cosmosAddr,
	)

	return nil
```
