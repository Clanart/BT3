### Title
Integer Truncation in `NewDynamicFeeChecker` Produces Zero `feeCap` That Passes Validation and Returns Zero `effectiveFee` for Cosmos SDK Transactions — (`File: ante/evm/fee_checker.go`)

---

### Summary

`NewDynamicFeeChecker` computes the per-gas fee cap for Cosmos SDK transactions via integer division `feeCap = fee / gas`. When `fee < gas`, this truncates to zero. When `baseFee` is also zero (a reachable state when `MinGasPrice = 0` and blocks are under-utilized), the truncated `feeCap = 0` silently passes the only guard check (`feeCap.LT(baseFeeInt)` → `0 < 0 = false`), and the downstream `effectiveFee` computed as `effectivePrice * gas = 0 * gas = 0` is returned to `DeductFeeDecorator`. This is the direct analog of the oracle report's division-truncation-to-zero pattern: a corrupted intermediate value passes a sanity check and propagates into a security-critical accounting path.

---

### Finding Description

In `ante/evm/fee_checker.go`, `NewDynamicFeeChecker` handles Cosmos SDK transactions under EIP-1559 fee market logic:

```go
gas := feeTx.GetGas()
feeCoins := feeTx.GetFee()
fee := feeCoins.AmountOf(denom)

feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))   // line 83 — integer division
baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)

if feeCap.LT(baseFeeInt) {                          // line 86 — only guard
    return nil, 0, errorsmod.Wrapf(...)
}

effectivePrice := sdkmath.NewIntFromBigInt(
    types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))

effectiveFee := sdk.Coins{{
    Denom:  denom,
    Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas)),  // line 97
}}
```

**Step 1 — Truncation.** `sdkmath.Int.Quo` performs integer (floor) division. Whenever `fee < gas` (e.g., `fee = 5`, `gas = 10`), `feeCap = 0`.

**Step 2 — Guard bypass.** The only validation is `feeCap.LT(baseFeeInt)`. When `baseFee = 0`, this evaluates to `0 < 0 = false`, so the check is silently passed with the corrupted zero value.

**Step 3 — Corrupted downstream computation.** `EffectiveGasPrice(0, 0, maxPriorityPrice)` = `min(0 + maxPriorityPrice, 0)` = `0`. Therefore `effectiveFee = 0 * gas = 0`. The checker returns `sdk.Coins{{Denom: denom, Amount: 0}}` to `DeductFeeDecorator`.

**When does `baseFee = 0` occur?** `CalculateBaseFee` in `x/feemarket/keeper/eip1559.go` lower-bounds the base fee at `minGasPrice`:

```go
minGasPrice := params.MinGasPrice.TruncateInt().BigInt()
return ethermint.BigMax(x.Sub(parentBaseFee, baseFeeDelta), minGasPrice)  // line 104
```

When `MinGasPrice = 0` (the default) and blocks are consistently under-utilized, the base fee decreases each block until it reaches zero. At that point `GetBaseFee()` returns `big.NewInt(0)` (not `nil`), so `NewDynamicFeeChecker` is entered (not the fallback path), and the truncation vulnerability is live.

**Attacker-controlled entry path.** Any unprivileged user can submit a Cosmos SDK transaction (e.g., `MsgSend`, `MsgDelegate`, any SDK message) with:
- `gasLimit` set to any value > 1 (e.g., 1,000,000)
- `fee` set to any value < `gasLimit` (e.g., 1 unit of the EVM denom)

No special privileges, governance access, or validator cooperation is required.

### Impact Explanation

`DeductFeeDecorator` receives `effectiveFee = sdk.Coins{{Denom: denom, Amount: 0}}`. Two outcomes are possible depending on the SDK version's coin validation in `DeductFees`:

1. **Fee bypass (most impactful):** If `DeductFeeDecorator` does not call `fees.IsValid()` before deducting (or treats zero-amount coins as a no-op), the user's account is debited 0 tokens. The transaction executes for free. This is a direct fee mis-accounting bug: the fee collector receives nothing, the user retains their funds, and the chain processes the transaction without compensation.

2. **Incorrect rejection of valid transactions:** If `DeductFees` validates coins and rejects zero-amount entries, a user who provided a fee satisfying `MinGasPriceDecorator` (e.g., `fee = ceil(minGasPrice * gas)` where `minGasPrice < 1`) is incorrectly rejected with "invalid fee amount" even though they paid the required minimum.
