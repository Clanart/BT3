### Title
Trusted Relayer Can Freely Set `fee_recipient` in `sign_transfer` to Redirect Fees — (`near/omni-bridge/src/lib.rs`)

---

### Summary

The `sign_transfer` function accepts a caller-supplied `fee_recipient: Option<AccountId>` parameter with no validation that it matches the calling relayer's identity. Any trusted relayer can call `sign_transfer` for any pending transfer and embed an arbitrary NEAR account as the fee recipient in the MPC-signed payload. Because the MPC signs whatever payload a trusted relayer submits, a malicious trusted relayer can front-run a legitimate relayer's `sign_transfer` call, get the MPC to sign a payload crediting themselves, submit it to the destination chain first, and claim the entire fee — while the legitimate relayer earns nothing.

---

### Finding Description

`sign_transfer` is the NEAR-side function that requests the MPC network to sign a `TransferMessagePayload` for delivery to a destination chain (EVM, Solana, Starknet). The signed payload is later submitted to the destination chain's `finTransfer`, which mints/unlocks tokens for the recipient and emits a `FinTransfer` event containing `feeRecipient`. That event is then proven back to NEAR via `claim_fee`, which pays the fee to whoever is named as `fee_recipient` in the proof.

The `fee_recipient` field flows as follows:

1. **Relayer calls `sign_transfer`** with an arbitrary `fee_recipient`: [1](#0-0) 

2. **`fee_recipient` is embedded verbatim** into the payload that the MPC signs, with no check that it equals `env::predecessor_account_id()` or any stored value: [2](#0-1) 

3. **`claim_fee_callback` enforces** that only the named `fee_recipient` can claim — but that name was set by the relayer in step 1: [3](#0-2) 

For non-zero-fee transfers, `sign_transfer_callback` does **not** remove the pending transfer after signing, so the same `transfer_id` can be signed multiple times by different relayers with different `fee_recipient` values: [4](#0-3) 

Both resulting MPC signatures are cryptographically valid. The first payload submitted to the destination chain wins (nonce replay protection prevents the second). The winner claims the full fee.

On the EVM side, `finTransfer` verifies the MPC signature over the Borsh-encoded payload (which includes `feeRecipient`) and emits the event — it has no way to distinguish which relayer was "supposed" to handle the transfer: [5](#0-4) 

The `feeRecipient` field in the EVM payload struct is a plain `string` with no on-chain binding to the submitter: [6](#0-5) 

---

### Impact Explanation

A malicious trusted relayer can steal the entire token fee from any pending transfer by:

1. Monitoring pending transfers on NEAR.
2. Calling `sign_transfer(transfer_id, Some("attacker.near"), None)` before the legitimate relayer does.
3. Receiving a valid MPC signature over a payload naming themselves as `fee_recipient`.
4. Submitting to the destination chain first, locking out the legitimate relayer via nonce replay protection.
5. Calling `claim_fee` on NEAR and receiving the full fee.

The legitimate relayer's `sign_transfer` call also succeeds (MPC signs it too), but their payload can never be submitted to the destination chain because the nonce is already consumed. They earn nothing despite having intended to service the transfer.

This maps to the allowed impact: **High — fee/accounting corruption that misdirects value**, as the fee is redirected away from the legitimate relayer to the attacker.

---

### Likelihood Explanation

The attack requires only a trusted relayer role, which is a permissioned but non-privileged role managed by `RelayerManager`. The codebase itself acknowledges that malicious relayer behavior is a concern (see the comment at line 1106 in `lib.rs`): [7](#0-6) 

Any trusted relayer who wishes to maximize their own fee income has a direct, low-effort incentive to front-run competing relayers' `sign_transfer` calls. The attack requires no special knowledge beyond monitoring the NEAR chain for pending transfers and calling a public function.

---

### Recommendation

Bind `fee_recipient` to the caller of `sign_transfer`. The simplest fix is to replace the caller-supplied `fee_recipient` parameter with `env::predecessor_account_id()` (or derive it from the caller), so the MPC always signs a payload that credits the relayer who actually requested the signature:

```rust
// Instead of accepting fee_recipient as a parameter:
let fee_recipient = Some(env::predecessor_account_id());
```

Alternatively, store the intended `fee_recipient` in the pending transfer record at the time the relayer first "claims" a transfer (a reservation mechanism), and validate it in `sign_transfer`.

---

### Proof of Concept

1. Alice (legitimate trusted relayer) monitors NEAR and sees `transfer_id = {origin_chain: Eth, origin_nonce: 42}` with a 100 USDC fee.
2. Alice prepares to call `sign_transfer({Eth, 42}, Some("alice.near"), None)`.
3. Bob (malicious trusted relayer) front-runs Alice and calls `sign_transfer({Eth, 42}, Some("bob.near"), None)` first.
4. MPC signs Bob's payload (with `fee_recipient = "bob.near"`) → signature `S_B`.
5. MPC also signs Alice's payload (with `fee_recipient = "alice.near"`) → signature `S_A`. Both are valid.
6. Bob submits `(S_B, payload_B)` to EVM `finTransfer`. `completedTransfers[destinationNonce]` is set to `true`.
7. Alice attempts to submit `(S_A, payload_A)` — reverts with `NonceAlreadyUsed`.
8. Bob calls `claim_fee` on NEAR with the EVM proof of the `FinTransfer` event (which contains `feeRecipient = "bob.near"`).
9. `claim_fee_callback` verifies `fee_recipient == predecessor_account_id` → passes → Bob receives 100 USDC.
10. Alice receives nothing. [8](#0-7) [9](#0-8)

    pub fn sign_transfer(
        &mut self,
        transfer_id: TransferId,
        fee_recipient: Option<AccountId>,
        fee: &Option<Fee>,
    ) -> Promise {
        let transfer_message = self.get_transfer_message(transfer_id);

        if let Some(fee) = &fee {
            require!(
                &transfer_message.fee == fee,
                BridgeError::InvalidFee.as_ref()
            );
        }

        let token_address = self
            .get_token_address(
                transfer_message.get_destination_chain(),
                self.get_token_id(&transfer_message.token),
            )
            .unwrap_or_else(|| {
                env::panic_str(BridgeError::FailedToGetTokenAddress.to_string().as_str())
            });

        let decimals = self
            .token_decimals
            .get(&token_address)
            .near_expect(BridgeError::TokenDecimalsNotFound);
        let amount_to_transfer = Self::normalize_amount(
            transfer_message
                .amount_without_fee()
                .near_expect(BridgeError::InvalidFee),
            decimals,
        );

        require!(
            amount_to_transfer > 0,
            BridgeError::InvalidAmountToTransfer.as_ref()
        );

        let message = DestinationChainMsg::from_json(&transfer_message.msg)
            .and_then(|s| s.destination_msg())
            .unwrap_or_default();

        let transfer_payload = TransferMessagePayload {
            prefix: PayloadType::TransferMessage,
            destination_nonce: transfer_message.destination_nonce,
            transfer_id,
            token_address,
            amount: U128(amount_to_transfer),
            recipient: transfer_message.recipient,
            fee_recipient,
            message,
        };

        let payload = near_sdk::env::keccak256_array(
            transfer_payload
                .encode_hashable()
                .near_expect(BridgeError::Borsh),
        );

        ext_signer::ext(self.mpc_signer.clone())
            .with_static_gas(MPC_SIGNING_GAS)
            .with_attached_deposit(env::attached_deposit())
            .sign(SignRequest {
                payload,
                path: SIGN_PATH.to_owned(),
                key_version: 0,
            })
            .then(
                Self::ext(env::current_account_id())
                    .with_static_gas(SIGN_TRANSFER_CALLBACK_GAS)
                    .sign_transfer_callback(transfer_payload, &transfer_message.fee),
            )
    }

    function finTransfer(
        bytes calldata signatureData,
        BridgeTypes.TransferMessagePayload calldata payload
    ) external payable whenNotPaused(PAUSED_FIN_TRANSFER) {
        if (completedTransfers[payload.destinationNonce]) {
            revert NonceAlreadyUsed(payload.destinationNonce);
        }

        completedTransfers[payload.destinationNonce] = true;

        bytes memory borshEncoded = bytes.concat(
            bytes1(uint8(BridgeTypes.PayloadType.TransferMessage)),
            Borsh.encodeUint64(payload.destinationNonce),
            bytes1(payload.originChain),
            Borsh.encodeUint64(payload.originNonce),
            bytes1(omniBridgeChainId),
            Borsh.encodeAddress(payload.tokenAddress),
            Borsh.encodeUint128(payload.amount),
            bytes1(omniBridgeChainId),
            Borsh.encodeAddress(payload.recipient),
            bytes(payload.feeRecipient).length == 0 // None or Some(String) in rust
                ? bytes("\x00")
                : bytes.concat(
                    bytes("\x01"),
                    Borsh.encodeString(payload.feeRecipient)

        );
