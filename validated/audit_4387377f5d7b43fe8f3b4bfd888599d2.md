### Title
`deprecated_declared_classes` Silently Dropped from State-Diff Commitment in `BlockExecutionArtifacts::new` — (`crates/blockifier/src/state/cached_state.rs`, `crates/apollo_batcher/src/block_builder.rs`)

---

### Summary

`BlockExecutionArtifacts::new` computes the block's `state_diff_commitment` and `state_diff_length` (both inputs to the block hash) using a `ThinStateDiff` that is always constructed with `deprecated_declared_classes: Vec::new()`. The `calculate_state_diff_hash` function explicitly hashes `deprecated_declared_classes`, and `ThinStateDiff::len()` counts them toward `state_diff_length`. Any block that contains a deprecated (Cairo 0) class declaration therefore produces a wrong `state_diff_commitment`, a wrong `state_diff_length` inside `concat_counts`, and consequently a wrong `ProposalCommitment` / block hash.

---

### Finding Description

**Root cause — the silent drop in `From<CommitmentStateDiff> for ThinStateDiff`:**

```rust
// crates/blockifier/src/state/cached_state.rs  lines 690-701
impl From<CommitmentStateDiff> for ThinStateDiff {
    fn from(commitment_state_diff: CommitmentStateDiff) -> Self {
        Self {
            deployed_contracts: commitment_state_diff.address_to_class_hash,
            storage_diffs: commitment_state_diff.storage_updates,
            class_hash_to_compiled_class_hash: commitment_state_diff
                .class_hash_to_compiled_class_hash,
            nonces: commitment_state_diff.address_to_nonce,
            // TODO(AlonH): Remove this when the structure of storage diffs changes.
            deprecated_declared_classes: Vec::new(),   // ← always empty
        }
    }
}
``` [1](#0-0) 

`CommitmentStateDiff` is built from `StateMaps` and drops `StateMaps::declared_contracts` (the Cairo 0 class-declaration tracker):

```rust
impl From<StateMaps> for CommitmentStateDiff {

        Self {
            address_to_class_hash: IndexMap::from_iter(diff.class_hashes),

            class_hash_to_compiled_class_hash: IndexMap::from_iter(diff.compiled_class_hashes),
            address_to_nonce: IndexMap::from_iter(diff.nonces),
            // diff.declared_contracts is silently discarded
        }
    }
}
``` [2](#0-1) 

**Propagation into the commitment pipeline — `BlockExecutionArtifacts::new`:**

```rust
// crates/apollo_batcher/src/block_builder.rs  lines 160-166
let (header_commitments, measurements) = calculate_block_commitments(
    &transactions_data,
    ThinStateDiff::from(commitment_state_diff.clone()),  // deprecated_declared_classes = []

``` [3](#0-2) 

The same empty `ThinStateDiff` is returned by `thin_state_diff()` and written to storage:

```rust

    ThinStateDiff::from(self.commitment_state_diff.clone())
}
``` [4](#0-3) 

**What the hash functions actually consume:**

`calculate_state_diff_hash` explicitly chains `deprecated_declared_classes`:

```rust
hash_chain =
    chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
``` [5](#0-4) 

`ThinStateDiff::len()` counts them toward `state_diff_length`:

```rust
result += self.deprecated_declared_classes.len();
``` [6](#0-5) 

`state_diff_length` feeds `concat_counts`, which is chained into the block hash:

```rust
let concatenated_counts = concat_counts(
    transactions_data.len(),
    event_leaf_elements.len(),
    state_diff.len(),   // wrong if deprecated_declared_classes were dropped
    l1_da_mode,
);
``` [7](#0-6) 

Both `state_diff_commitment` and `concatenated_counts` are chained into `calculate_block_hash`:

```rust
.chain(&block_commitments.concatenated_counts)
.chain(&block_commitments.state_diff_commitment.0.0)
``` [8](#0-7) 

---

### Impact Explanation

When a block contains one or more deprecated (Cairo 0) class declarations:

1. `state_diff_commitment` is computed over a `ThinStateDiff` with `deprecated_declared_classes = []` instead of the actual declared class hashes → **wrong `state_diff_commitment`**.
2. `state_diff_length` is under-counted by the number of deprecated declarations → **wrong `concatenated_counts`**.
3. Both wrong values are chained into `calculate_block_hash` → **wrong `ProposalCommitment` / block hash**.
4. The `ThinStateDiff` written to storage via `thin_state_diff()` is also missing the deprecated declarations → **wrong authoritative state diff returned by `starknet_getStateUpdate`**.

This matches the allowed impact: *Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input* and *RPC execution … returns an authoritative-looking wrong value*.

---

### Likelihood Explanation

The trigger requires a deprecated (Cairo 0) `Declare` transaction to be sequenced. The storage layer and RPC layer both have full support for `deprecated_declared_classes` (tests exist in `crates/apollo_storage/src/state/state_test.rs`), confirming the field is live in production paths. The TODO comment acknowledges the omission is not yet resolved. Any node or user that submits a valid `DeclareV0`/`DeclareV1` transaction can trigger the mismatch without any privileged access.

---

### Recommendation

`CommitmentStateDiff` must be extended with a `deprecated_declared_classes: Vec<ClassHash>` field, populated from `StateMaps::declared_contracts` (filtering for Cairo 0 entries). The `From<CommitmentStateDiff> for ThinStateDiff` conversion must then propagate this field instead of hardcoding `Vec::new()`. The TODO comment at line 698–699 of `cached_state.rs` marks exactly this gap.

---

### Proof of Concept

Construct a `DeclareV0` transaction for a Cairo 0 class and sequence it through the batcher. After `BlockExecutionArtifacts::new` returns:

```
assert!(artifacts.thin_state_diff().deprecated_declared_classes.is_empty());
// passes — but the class WAS declared; the commitment is wrong

// Recompute with the correct deprecated_declared_classes:
let correct_diff = ThinStateDiff {
    deprecated_declared_classes: vec![cairo0_class_hash],
    ..artifacts.thin_state_diff()
};
let correct_commitment = calculate_state_diff_hash(&correct_diff);
let wrong_commitment  = calculate_state_diff_hash(&artifacts.thin_state_diff());
assert_ne!(correct_commitment, wrong_commitment);
// The block hash stored on-chain uses wrong_commitment.
```

The divergence between `correct_commitment` and `wrong_commitment` is the exact corrupted value that propagates into the `ProposalCommitment`, the stored block hash, and the RPC `starknet_getStateUpdate` response.
