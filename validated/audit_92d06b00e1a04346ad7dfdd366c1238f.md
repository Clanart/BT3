### Title
Missing Zero/Undeclared Class Hash Validation in `execute_replace_class` Allows Permanent Contract Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not validate that the caller-supplied `class_hash` is non-zero or corresponds to a declared class. This is directly analogous to the reported "zero gauge" bug: just as Booster.sol accepted a zero address for `_gauge` without validation, the OS accepts a zero (or undeclared) `class_hash` in `replace_class` without validation. A contract can call `replace_class(0)`, setting its own class hash to `UNINITIALIZED_CLASS_HASH` (0). Any subsequent call to that contract causes the OS to attempt `find_element` with key=0 on the compiled class facts bundle, which panics/fails, permanently rendering the contract unexecutable and freezing all funds it holds.

---

### Finding Description

In `execute_replace_class` (lines 877–916 of `syscall_impls.cairo`), the OS reads `class_hash` directly from the syscall request and writes it into the contract's `StateEntry` without any validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-inserted TODO comment at line 898 explicitly acknowledges the missing check. There is no `assert_not_zero(class_hash)` and no check that `class_hash` exists in `contract_class_changes`.

When a subsequent transaction calls the affected contract, `execute_entry_point` runs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // key = 0
);
```

This returns 0 (uninitialized). Then:

```cairo
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,   // key = 0
);
```

`find_element` panics if the key is absent. Since no compiled class with hash=0 is ever declared, the OS cannot generate a valid proof for any block containing a call to the frozen contract. The contract is permanently unexecutable. [1](#0-0) [2](#0-1) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` is set to 0 via `replace_class(0)`:

1. The contract's `StateEntry.class_hash` is permanently 0 in the committed state.
2. Every future OS execution that touches this contract calls `find_element` with key=0, which panics because no compiled class with hash=0 exists.
3. The sequencer cannot include any transaction that calls the contract in a provable block.
4. All ERC-20 balances, vault deposits, or other assets held by the contract are permanently inaccessible — no transfer, withdrawal, or rescue is possible.

The nonce is not reset by `replace_class`, so the contract cannot be redeployed at the same address (the `deploy_contract` check `assert state_entry.nonce = 0` would fail). [3](#0-2) 

---

### Likelihood Explanation

**Medium.**

The attacker-controlled entry path is direct: any Cairo contract can issue the `replace_class` syscall with an arbitrary felt value, including 0. Realistic scenarios include:

- A DeFi protocol that exposes a governance-controlled upgrade path where the new class hash is passed as a parameter — an attacker who can influence governance (or exploit a reentrancy/input-validation bug in the upgrade function) passes 0.
- A contract with a public `upgrade(class_hash: felt)` function that forwards the argument directly to `replace_class` without validating it is non-zero and declared.
- A contract whose constructor calls `replace_class(0)` (intentionally or via a bug), freezing itself immediately upon deployment.

No privileged role, leaked key, or operator cooperation is required. The syscall is available to all Cairo contracts.

---

### Recommendation

Add two validations inside `execute_replace_class` before updating the state:

1. **Non-zero check**: `assert_not_zero(class_hash)` — rejects `UNINITIALIZED_CLASS_HASH` (0).
2. **Declared class check**: verify that `class_hash` has a non-zero entry in `contract_class_changes` (i.e., a compiled class hash has been registered for it via a prior `declare` transaction). This is exactly what the existing TODO comment calls for.

The analogous protection already exists in `execute_declare_transaction`:

```cairo
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
```

The same pattern should be applied in `execute_replace_class`. [4](#0-3) 

---

### Proof of Concept

1. **Attacker deploys a contract** `VictimVault` that:
   - Holds user funds (e.g., acts as an ERC-20 wrapper).
   - Exposes a public function `upgrade(new_class: felt)` that calls `replace_class(new_class)`.

2. **Attacker calls** `VictimVault.upgrade(0)`.

3. **OS execution** of `execute_replace_class`:
   - `class_hash = 0` (no validation).
   - `contract_state_changes[VictimVault_address].class_hash` is updated to 0.
   - Transaction succeeds; state is committed.

4. **Any subsequent call** to `VictimVault` (e.g., `withdraw`):
   - `execute_entry_point` reads `compiled_class_hash = dict_read(key=0)` → returns 0.
   - `find_element(..., key=0)` → panics; OS cannot produce a valid proof.
   - Sequencer must exclude all transactions touching `VictimVault`.

5. **Result**: All funds inside `VictimVault` are permanently frozen. No withdrawal, rescue, or redeployment is possible. [5](#0-4) [6](#0-5)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    let revert_log = &revert_log[1];

    return ();
}
