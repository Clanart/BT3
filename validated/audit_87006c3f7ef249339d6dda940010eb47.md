### Title
Legacy Proof Replay via ENearProxy Bypasses eNear's Own Pause When `PAUSED_LEGACY_FIN_TRANSFER` Is Unset — (`evm/src/eNear/contracts/ENearProxy.sol`)

### Summary
After migration, FakeProver is installed on eNear and ENearProxy is made eNear's admin. eNear's own `finaliseNearToEthTransfer` is paused for non-admin callers, but ENearProxy (as admin) bypasses that pause. The only guard on the public `ENearProxy.finaliseNearToEthTransfer` path is `whenNotPaused(PAUSED_LEGACY_FIN_TRANSFER)`. If that flag is not set before FakeProver is activated, any caller can submit a historical RainbowBridge proof with an unused receipt ID, pass both the ENearProxy prover check and eNear's FakeProver check, and mint eNear tokens.

### Finding Description

The migration plan (per README) is:
1. Replace eNear's prover with `FakeProver` (accepts any proof).
2. Pause `finaliseNearToEthTransfer` on eNear via `adminPause`.
3. Set ENearProxy as eNear's admin.

After this, `ENearProxy.mint` works because ENearProxy (as admin) can call eNear's paused function with a crafted fake proof.

The problem: `ENearProxy.finaliseNearToEthTransfer` is also public and also calls `eNear.finaliseNearToEthTransfer` as admin (bypassing eNear's pause). Its only guard is:

```solidity
// ENearProxy.sol line 83
external whenNotPaused(PAUSED_LEGACY_FIN_TRANSFER) {
    require(
        prover.proveOutcome(proofData, proofBlockHeight),  // line 85
        "Proof should be valid"
    );
    eNear.finaliseNearToEthTransfer(proofData, proofBlockHeight); // line 89
}
``` [1](#0-0) 

If `PAUSED_LEGACY_FIN_TRANSFER` is not set, an attacker can:
1. Supply any `proofData` — ENearProxy's own `prover` (which may also be FakeProver, or the real NearProver for a genuinely valid historical proof) passes.
2. `eNear.finaliseNearToEthTransfer` is called. ENearProxy is admin, so eNear's own pause is bypassed.
3. eNear's FakeProver returns `true` for any input.
4. eNear checks its `usedProofs` mapping. A historical proof whose receipt ID was never submitted passes this check.
5. eNear mints tokens to the address encoded in the proof. [2](#0-1) 

The `mint` function uses a monotonically incrementing `currentReceiptId` to avoid collisions with historical receipt IDs:

```solidity
currentReceiptId += 1;
eNear.finaliseNearToEthTransfer(fakeProofData, 0); // line 72
``` [3](#0-2) 

But `currentReceiptId` is initialized at deployment to the last known receipt ID, meaning all historical receipt IDs below that value that were never submitted to eNear remain valid replay targets.

### Impact Explanation
Unauthorized minting of eNear tokens. An attacker who finds any historical RainbowBridge proof (NEAR→ETH transfer) whose receipt ID was never recorded in eNear's `usedProofs` can mint an equivalent amount of eNear tokens with no backing. This directly inflates eNear supply without a corresponding NEAR lock, breaking bridge collateralization.

### Likelihood Explanation
The precondition — FakeProver installed on eNear and ENearProxy set as admin — is the **required migration state** for the `mint` function to work at all. The only missing condition is that `PAUSED_LEGACY_FIN_TRANSFER` has not yet been set on ENearProxy. There is no code-level enforcement of the ordering: nothing prevents FakeProver from being activated before the legacy path is paused. Historical NEAR proofs are publicly available on-chain. Any proof whose receipt ID was never submitted (e.g., a transfer that was initiated but the user never finalized on Ethereum) is a valid replay target. [4](#0-3) 

### Recommendation
Enforce the correct ordering at the code level. The simplest fix is to set `PAUSED_LEGACY_FIN_TRANSFER` in the `initialize` function so the legacy path is paused by default and must be explicitly unpaused only if needed. Alternatively, remove `ENearProxy.finaliseNearToEthTransfer` entirely — the README states this path should be permanently disabled after migration, so there is no reason to keep it callable.

### Proof of Concept
1. Deploy eNear (mainnet fork), ENearProxy, FakeProver.
2. Call `eNear.adminSstore(5, fakeProverAddress)` to install FakeProver on eNear.
3. Call `eNear.adminSstore(9, eNearProxyAddress)` to set ENearProxy as admin.
4. Do **not** call `ENearProxy.pauseAll()` or `ENearProxy.pause(PAUSED_LEGACY_FIN_TRANSFER)`.
5. Collect a historical RainbowBridge proof from NEAR whose receipt ID is not in eNear's `usedProofs`.
6. Call `ENearProxy.finaliseNearToEthTransfer(historicalProofData, blockHeight)` as any EOA.
7. Assert `eNear.balanceOf(attacker) > 0` — tokens are minted without a corresponding NEAR lock.
