### Title
RPC State Reader Returns Stale Compiled Class Hash After Migration, Corrupting Fee Estimation and Simulation - (File: `crates/apollo_rpc_execution/src/state_reader.rs`)

---

### Summary

When `enable_casm_hash_migration` is active (enabled in `blockifier_versioned_constants_0_14_1.json`), the blockifier migrates a class's compiled class hash from v1 (Poseidon) to v2 (Blake) and writes the new hash into the block's `ThinStateDiff`. The storage layer correctly records the v2 hash at the migration block. However, the RPC state reader's `get_compiled_class_hash` resolves the class hash by looking up the **first declaration block** in `declared_classes_block_table` (which is never updated on migration) and then reads the state diff at that original block — returning the stale v1 hash. Every RPC call that simulates, estimates, or traces a transaction touching a migrated class therefore operates on the wrong compiled class hash, producing authoritative-looking but incorrect results.

---

### Finding Description

**Step 1 — Migration writes a new compiled class hash into the state diff.**

In `finalize_block` (`crates/blockifier/src/blockifier/transaction_executor.rs`), after all transactions execute, `set_compiled_class_hash_migration` is called:

```rust
block_state.set_compiled_class_hash_migration(&class_hashes_to_migrate)?;
let state_diff = block_state.to_state_diff()?.state_maps;
```

`set_compiled_class_hash_migration` calls `set_compiled_class_hash(*class_hash, *compiled_class_hash_v2)` for every migrated class. [1](#0-0) 

The resulting `CommitmentStateDiff.class_hash_to_compiled_class_hash` therefore contains `{C → v2_hash}` for the migration block M. This flows through `BlockExecutionArtifacts::new` → `ThinStateDiff::from(commitment_state_diff)` and is persisted by `append_state_diff`. [2](#0-1) 

**Step 2 — `append_state_diff` updates the compiled-class-hash table but NOT the declaration-block table.**

```rust
for (class_hash, _) in &thin_state_diff.class_hash_to_compiled_class_hash {
    let not_declared = declared_classes_block_table.get(&self.txn, class_hash)?.is_none();
    if not_declared {                                          // ← skipped for migrated classes
        declared_classes_block_table.insert(&self.txn, class_hash, &block_number)?;
    }
}
write_compiled_class_hashes(                                  // ← always writes v2 at block M
    &thin_state_diff.class_hash_to_compiled_class_hash, ...
)?;
``` [3](#0-2) 

After migration, `declared_classes_block_table` still maps `C → N` (original declaration block), while `compiled_class_hash_table` has two entries: `(C, N) → v1` and `(C, M) → v2`.

**Step 3 — The storage reader resolves the correct v2 hash via cursor.**

`get_compiled_class_hash_at` does a cursor `lower_bound` / `prev` scan and returns the most recent entry before the requested block — correctly returning v2 after block M. [4](#0-3) 

**Step 4 — The RPC state reader resolves the wrong v1 hash via the stale declaration block.**

```rust
fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    // ...
    let maybe_block_number = self
        .storage_reader
        .get_state_reader()
        .get_class_definition_block_number(&class_hash)   // returns N, not M
        ...;

    let state_diff = self.storage_reader.get_state_diff(block_number)?;  // reads block N

    let compiled_class_hash = state_diff
        .class_hash_to_compiled_class_hash
        .get(&class_hash)   // returns v1_hash from block N
        ...;
    Ok(*compiled_class_hash)   // ← wrong: v1 instead of v2
}
``` [5](#0-4) 

`get_class_definition_block_number` reads `declared_classes_block_table`, which was never updated for the migration. It returns N. The state diff at N contains `{C → v1_hash}`. The RPC state reader therefore returns the stale v1 hash for every query after block M.

**Step 5 — Migration is enabled in the latest versioned constants.**

`enable_casm_hash_migration` is `true` in `blockifier_versioned_constants_0_14_1.json`, making this a live production path. [6](#0-5) 

---

### Impact Explanation

All RPC execution paths that use `ExecutionStateReader` — `starknet_call`, `starknet_estimateFee`, `starknet_simulateTransactions`, `starknet_traceTransaction` — call `get_compiled_class_hash` through `CachedState`. After a migration event, these calls receive the stale v1 hash for the affected class. The state diff produced by simulation will contain the wrong compiled class hash, and any fee estimate that depends on state-diff content or on the class's execution path will be incorrect. The result is returned to callers as an authoritative value with no indication of error.

This matches: **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

---

### Likelihood Explanation

`enable_casm_hash_migration = true` is the default for protocol version 0.14.1 and later. Any Cairo 1 class declared before the migration epoch and subsequently executed will be migrated. After that point, every RPC simulation or fee estimate touching that class returns the wrong compiled class hash. No special attacker capability is required; any user who calls `starknet_estimateFee` or `starknet_simulateTransactions` on a transaction that invokes a migrated class will receive the wrong answer.

---

### Recommendation

Replace the two-step `get_class_definition_block_number` + `get_state_diff` lookup in `get_compiled_class_hash` inside `crates/apollo_rpc_execution/src/state_reader.rs` with a direct call to `get_compiled_class_hash_at(state_number, &class_hash)`, which uses the cursor-based scan and always returns the most recent compiled class hash. This is the same approach used by the storage reader and is already correct.

---

### Proof of Concept

1. Deploy and declare class `C` in block `N` with `compiled_class_hash = v1`. `declared_classes_block_table[C] = N`.
2. In block `M > N`, execute a transaction that calls `C`. With `enable_casm_hash_migration = true`, `finalize_block` calls `set_compiled_class_hash_migration`, writing `{C → v2}` into the state diff for block M. `compiled_class_hash_table` now has `(C, N) → v1` and `(C, M) → v2`. `declared_classes_block_table[C]` remains `N`.
3. After block M is committed, call `starknet_estimateFee` with a transaction that invokes `C`.
4. `ExecutionStateReader::get_compiled_class_hash(C)` calls `get_class_definition_block_number(C)` → `N`, then `get_state_diff(N)` → `{C → v1}`. Returns `v1`.
5. The blockifier's `CachedState` caches `v1` for `C`. The simulation state diff records `{C → v1}` instead of `{C → v2}`.
6. The fee estimate and simulation result are computed against the wrong compiled class hash and returned to the caller as authoritative.

fn get_compiled_class_hash_at<'env, Mode: TransactionKind>(
    first_irrelevant_block: BlockNumber,
    class_hash: &ClassHash,
    txn: &'env DbTransaction<'env, Mode>,
    compiled_class_hash_table: &'env CompiledClassHashTable<'env>,
) -> StorageResult<Option<CompiledClassHash>> {
    let db_key = (*class_hash, first_irrelevant_block);
    // Find the previous db item.
    let mut cursor = compiled_class_hash_table.cursor(txn)?;
    cursor.lower_bound(&db_key)?;
    let res = cursor.prev()?;
    match res {
        None => Ok(None),
        Some(((got_class_hash, _got_block_number), value)) => {
            if got_class_hash != *class_hash {
                // The previous item belongs to different class hash, which means there is no
                // previous state diff for this item.
                return Ok(None);
            };
            // The previous db item indeed belongs to this address and key.
            Ok(Some(value))
        }
    }

    fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
        if let Some(pending_data) = &self.maybe_pending_data {
            for DeclaredClassHashEntry { class_hash: other_class_hash, compiled_class_hash } in
                &pending_data.declared_classes
            {
                if class_hash == *other_class_hash {
                    return Ok(*compiled_class_hash);
                }
            }
        }

        let maybe_block_number = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_reader()
            .map_err(storage_err_to_state_err)?
            .get_class_definition_block_number(&class_hash)
            .map_err(storage_err_to_state_err)?;

        // Cairo 0 classes (and undeclared classes) do not have a compiled class hash.
        // According to the trait, return the default value.
        let Some(block_number) = maybe_block_number else {
            return Ok(CompiledClassHash::default());
        };

        let state_diff = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_diff(block_number)
            .map_err(storage_err_to_state_err)?
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing state diff at block {block_number}."
            )))?;

        let compiled_class_hash = state_diff
            .class_hash_to_compiled_class_hash
            .get(&class_hash)
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing class declaration at block {block_number}, class \
                 {class_hash}."
            )))?;

        Ok(*compiled_class_hash)
    }
