### Title
`claim_fee` Unnecessarily Restricted to Trusted Relayers Locks Earned Fees Permanently - (File: `near/omni-bridge/src/lib.rs`)

### Summary

`claim_fee` carries a `#[trusted_relayer]` guard that prevents any non-trusted-relayer from calling it. However, the callback already enforces `fee_recipient == predecessor_account_id`, making the trusted-relayer check redundant and harmful: any fee_recipient who is not a current trusted relayer (e.g., a resigned relayer, or a non-relayer set as fee_recipient) is permanently blocked from collecting fees that are rightfully theirs.

### Finding Description

`claim_fee` in `near/omni-bridge/src/lib.rs` is decorated with both `#[trusted_relayer]` and `#[pause(except(roles(Role::DAO)))]`:

```rust
#[payable]
#[trusted_relayer]
#[pause(except(roles(Role::DAO)))]
pub fn claim_fee(&mut self, #[serializer(borsh)] args: ClaimFeeArgs) -> Promise {
    self.verify_proof(args.chain_kind, args.prover_args).then(
        Self::ext(env::current_account_id())
            .with_attached_deposit(env::attached_deposit())
            .with_static_gas(CLAIM_FEE_CALLBACK_GAS)
            .claim_fee_callback(&env::predecessor_account_id()),
    )
}
```

The `#[trusted_relayer]` macro (configured at the `impl` block level with `bypass_roles(Role::DAO, Role::UnrestrictedRelayer)`) rejects any caller who is not a currently-active trusted relayer, a DAO member, or an UnrestrictedRelayer.

Inside `claim_fee_callback`, the contract already enforces the only meaningful access control that is needed:

```rust
require!(
    fee_recipient == *predecessor_account_id,
    BridgeError::OnlyFeeRecipientCanClaim.as_ref()
);
```

This check ensures only the designated `fee_recipient` (embedded in the destination-chain proof) can collect the fee. The `#[trusted_relayer]` guard on the outer function adds nothing to security — it only restricts who can trigger the proof verification — but it creates a scenario where a legitimate fee_recipient who is not a trusted relayer is permanently blocked.

### Impact Explanation

**Permanent freezing / irrecoverable lock of user funds.**

When a transfer is finalized on the destination chain (EVM, Solana, StarkNet), the destination chain emits a `FinTransfer` event containing a `fee_recipient` NEAR account. That account is the only one authorized by the callback to collect the fee. If that account is not a trusted relayer at the time of the `claim_fee` call, the call reverts at the `#[trusted_relayer]` gate before the proof is even verified. The pending transfer entry remains in `pending_transfers` with the fee locked inside. There is no alternative code path that calls `send_fee_internal` for this case, so the fee is permanently unclaimable.

Concrete scenarios:
1. A relayer finalizes a transfer on Solana/EVM, setting itself as `fee_recipient`. It then calls `resign_trusted_relayer` on NEAR to recover its stake. It is no longer a trusted relayer. It can never call `claim_fee` to collect the fee it earned.
2. A protocol integration sets a non-relayer NEAR account as `fee_recipient` in the destination-chain finalization. That account can never call `claim_fee`.

In both cases the fee tokens remain locked in the bridge contract with no recovery mechanism.

### Likelihood Explanation

**Medium.** The resign-then-claim sequence is a natural lifecycle event for any relayer. The NEAR bridge explicitly supports `resign_trusted_relayer` to return stake. Any relayer who resigns before claiming outstanding fees loses them permanently. This requires no attacker — it is triggered by normal user behavior.

### Recommendation

Remove the `#[trusted_relayer]` attribute from `claim_fee`. The callback's `fee_recipient == predecessor_account_id` check is sufficient to ensure only the rightful recipient can claim. This mirrors the fix applied in the Audius reference report: allow any account to trigger the action; rely on the internal authorization check to enforce correctness.

```rust
#[payable]
// Remove: #[trusted_relayer]
#[pause(except(roles(Role::DAO)))]
pub fn claim_fee(&mut self, #[serializer(borsh)] args: ClaimFeeArgs) -> Promise { ... }
```

### Proof of Concept

1. Trusted relayer R finalizes a transfer on Solana, setting `fee_recipient = R.near` in the `FinalizeTransferResponse`.
2. R calls `resign_trusted_relayer` on NEAR, recovering its stake. `is_trusted_relayer(R)` now returns `false`.
3. R calls `claim_fee` with the Solana finalization proof.
4. The `#[trusted_relayer]` macro fires before the function body executes and panics with the relayer-not-active error.
5. The pending transfer entry for the fee remains in `pending_transfers` indefinitely. The fee tokens are locked.

    fn send_fee_internal(
        &mut self,
        transfer_message: &TransferMessage,
        fee_recipient: AccountId,
        token_fee: u128,
    ) -> PromiseOrValue<()> {
        if transfer_message.fee.native_fee.0 != 0 {
            let origin_chain = transfer_message.origin_transfer_id.as_ref().map_or_else(
                || transfer_message.get_origin_chain(),
                |origin_transfer_id| origin_transfer_id.origin_chain,
            );

            if origin_chain.is_utxo_chain() {
                env::panic_str(BridgeError::NativeFeeForUtxoChain.to_string().as_str())
            } else if origin_chain == ChainKind::Near {
                Promise::new(fee_recipient.clone())
                    .transfer(NearToken::from_yoctonear(transfer_message.fee.native_fee.0))
                    .detach();
            } else {
                ext_token::ext(self.get_native_token_id(origin_chain))
                    .with_static_gas(MINT_TOKEN_GAS)
                    .mint(fee_recipient.clone(), transfer_message.fee.native_fee, None)
                    .detach();
            }
        }

        let token = self.get_token_id(&transfer_message.token);
        env::log_str(
            &OmniBridgeEvent::ClaimFeeEvent {
                transfer_message: transfer_message.clone(),
            }
            .to_log_string(),
        );

        self.unlock_tokens_if_needed(transfer_message.get_destination_chain(), &token, token_fee);

        if token_fee > 0 {
            if self.is_deployed_token(&token) {
                ext_token::ext(token)
                    .with_static_gas(MINT_TOKEN_GAS)
                    .mint(fee_recipient, U128(token_fee), None)
                    .into()
            } else {
                ext_token::ext(token)
                    .with_static_gas(FT_TRANSFER_GAS)
                    .with_attached_deposit(ONE_YOCTO)
                    .ft_transfer(fee_recipient, U128(token_fee), None)
                    .into()
            }
        } else {
            PromiseOrValue::Value(())
        }
