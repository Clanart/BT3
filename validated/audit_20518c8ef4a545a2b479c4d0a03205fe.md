### Title
`ApplyMessage` Skips Gas Refund and Block-Gas Accounting Present in `ApplyTransaction` — (`x/evm/keeper/state_transition.go`)

### Summary

`ApplyTransaction` performs three critical post-execution accounting steps that `ApplyMessage` (the public API for native-module callers) entirely omits. When `ApplyMessage` is invoked with `commit=true` by any native module or precompile, EVM state is committed to the KVStore while gas refunds are never issued to the sender and block-gas usage is never recorded, causing the EIP-1559 fee-market base fee to be computed from stale data.

### Finding Description

`ApplyTransaction` (the normal EVM message-server path) performs three mandatory post-execution steps after calling `ApplyMessageWithConfig`:

```go
// x/evm/keeper/state_transition.go  ApplyTransaction
res, err := k.ApplyMessageWithConfig(tmpCtx, msg, cfg, true)   // commit state
...
if err = k.RefundGas(ctx, msg, leftoverGas, cfg.Params.EvmDenom); err != nil { ... }  // ① refund unused gas to sender
...
totalGasUsed, err := k.AddTransientGasUsed(ctx, res.GasUsed)   // ② accumulate per-tx gas in transient store
...
k.ResetGasMeterAndConsumeGas(ctx, totalGasUsed)                 // ③ update SDK gas meter → feeds ctx.BlockGasUsed()
```

`ApplyMessage`, the exported wrapper explicitly documented for use by "other native modules directly", does none of these:

```go
// x/evm/keeper/state_transition.go  ApplyMessage
func (k *Keeper) ApplyMessage(ctx sdk.Context, msg *core.Message, tracer *tracing.Hooks, commit bool) (*types.EVMResult, error) {
    cfg, err := k.EVMConfig(ctx, k.eip155ChainID, common.Hash{})
    ...
    result, err := k.ApplyMessageWithConfig(ctx, msg, cfg, commit)
    return result, nil   // ← no RefundGas, no AddTransientGasUsed, no ResetGasMeterAndConsumeGas
}
```

The three omitted steps are:

1. **`RefundGas`** — transfers `leftoverGas × gasPrice` from the fee-collector module account back to the sender. Without it, the sender permanently overpays; the fee collector retains funds it should not hold. [1](#0-0) 

2. **`AddTransientGasUsed`** — accumulates per-message gas into the object store keyed by `TxIndex`. Without it, `GetTransientGasUsed` returns 0 for the transaction, so `ResetGasMeterAndConsumeGas` would consume 0 gas even if it were called. [2](#0-1) 

3. **`ResetGasMeterAndConsumeGas`** — resets the SDK gas meter and records the actual EVM gas used. The SDK gas meter drives `ctx.BlockGasUsed()`, which `feemarket/keeper/abci.go` EndBlock reads to compute `blockGasWanted` and persist it for the next block's `CalculateBaseFee`. Without this call, `ctx.BlockGasUsed()` is stale (zero or the ante-handler value), so `SetBlockGasWanted` stores an incorrect value. [3](#0-2) 

The fee-market EndBlock then stores the wrong `blockGasWanted`:

```go
// x/feemarket/keeper/abci.go  EndBlock
gasWanted := ctx.BlockGasWanted()
gasUsed   := ctx.BlockGasUsed()          // stale if ApplyMessage was used
...
k.SetBlockGasWanted(ctx, gasWanted)      // persisted for next block's CalculateBaseFee
```

And `CalculateBaseFee` in the next block's BeginBlock reads this stale value:

```go
parentGasUsed := k.GetBlockGasWanted(ctx)   // wrong value → wrong base fee
``` [4](#0-3) [5](#0-4) 

### Impact Explanation

Two concrete impacts arise when `ApplyMessage` is called with `commit=true`:

- **User fund mis-accounting (High):** The sender's pre-paid gas fee (deducted in the ante handler via `DeductTxCostsFromUserBalance`) is never partially refunded. The fee collector permanently retains `leftoverGas × gasPrice` that belongs to the sender. [6](#0-5) 

- **Fee-market base fee corruption (High):** Block gas usage is underreported to the fee market. `CalculateBaseFee` computes a base fee that is lower than it should be (treating the block as under-target when it was actually at or over target). This allows subsequent transactions to pay less than the correct base fee, constituting a fee-market accounting bug. [7](#0-6) 

Both impacts fall within the allowed scope: "EVM state transition, gas refund, fee market … bug that permits … valid user funds/fees to be mis-accounted."

### Likelihood Explanation

`ApplyMessage` is a public, exported keeper method explicitly documented as the intended entry point for native modules and precompiles calling EVM logic. Any precompile registered via `customContractFns` that calls `ApplyMessage(ctx, msg, tracer, true)` triggers the bug from an ordinary, unprivileged EVM transaction. The asymmetry is structural and silent — no error is returned, so callers have no indication that accounting was skipped. [8](#0-7) 

### Recommendation

Move the three accounting steps into `ApplyMessageWithConfig` itself (gated on `commit == true`), or create a new `ApplyMessageCommit` helper that wraps `ApplyMessageWithConfig` and always performs `RefundGas`, `AddTransientGasUsed`, and `ResetGasMeterAndConsumeGas` — mirroring exactly what `ApplyTransaction` does. `ApplyMessage` should either call this helper or be deprecated in favour of it, so that any caller committing state automatically gets correct gas accounting.

### Proof of Concept

1. A native module or precompile calls `k.ApplyMessage(ctx, msg, nil, true)` — a documented, supported use case.
2. `ApplyMessageWithConfig` executes the EVM call and, because `commit=true`, calls `stateDB.Commit()`, persisting all state changes.
3. `ApplyMessage` returns — `RefundGas` is never called; the sender's overpaid fee stays in the fee collector.
4. `AddTransientGasUsed` is never called; `GetTransientGasUsed` returns 0 for this tx index.
5. `ResetGasMeterAndConsumeGas` is never called; `ctx.GasMeter().GasConsumed()` reflects only the ante-handler value, not the EVM gas used.
6. At EndBlock, `ctx.BlockGasUsed()` is lower than actual; `SetBlockGasWanted` stores an undercount.
7. At the next BeginBlock, `CalculateBaseFee` reads the undercount and computes a base fee that is lower than the EIP-1559 target demands, allowing subsequent transactions to underpay fees. [9](#0-8) [8](#0-7)

// RefundGasWithPrice transfers the leftover gas to sender using the provided gas price.
func (k *Keeper) RefundGasWithPrice(
	ctx sdk.Context,
	msg *core.Message,
	leftoverGas uint64,
	gasPrice *big.Int,
	denom string,
) error {
	if gasPrice == nil {
		gasPrice = new(big.Int)
	}

	// Return EVM tokens for remaining gas, exchanged at the original rate.
	remaining := new(big.Int).Mul(new(big.Int).SetUint64(leftoverGas), gasPrice)

	switch remaining.Sign() {
	case -1:
		// negative refund errors
		return errorsmod.Wrapf(types.ErrInvalidRefund, "refunded amount value cannot be negative %d", remaining.Int64())
	case 1:
		// positive amount refund
		refundedCoins := sdk.Coins{sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(remaining))}

		// refund to sender from the fee collector module account, which is the escrow account in charge of collecting tx fees
		err := k.bankKeeper.SendCoinsFromModuleToAccountVirtual(ctx, authtypes.FeeCollectorName, msg.From.Bytes(), refundedCoins)
		if err != nil {
			err = errorsmod.Wrapf(errortypes.ErrInsufficientFunds, "fee collector account failed to refund fees: %s", err.Error())
			return errorsmod.Wrapf(err, "failed to refund %d leftover gas (%s)", leftoverGas, refundedCoins.String())
		}
	default:
		// no refund, consume gas and update the tx gas meter
	}

	return nil
}

func (k *Keeper) EndBlock(ctx sdk.Context) error {
	gasWanted := ctx.BlockGasWanted()
	gw, err := ethermint.SafeInt64(gasWanted)
	if err != nil {
		return err
	}
	gasUsed, err := ethermint.SafeInt64(ctx.BlockGasUsed())
	if err != nil {
		return err
	}

	// to prevent BaseFee manipulation we limit the gasWanted so that
	// gasWanted = max(gasWanted * MinGasMultiplier, gasUsed)
	// this will be keep BaseFee protected from un-penalized manipulation
	// more info here https://github.com/evmos/ethermint/pull/1105#discussion_r888798925
	minGasMultiplier := k.GetParams(ctx).MinGasMultiplier
	limitedGasWanted := sdkmath.LegacyNewDec(gw).Mul(minGasMultiplier)
	gasWanted = sdkmath.LegacyMaxDec(limitedGasWanted, sdkmath.LegacyNewDec(gasUsed)).TruncateInt().Uint64()
	k.SetBlockGasWanted(ctx, gasWanted)

	// If the parent gasUsed is the same as the target, the baseFee remains
	// unchanged.
	if parentGasUsed == parentGasTarget {
		return new(big.Int).Set(parentBaseFee)
	}

	if parentGasUsed > parentGasTarget {
		// If the parent block used more gas than its target, the baseFee should
		// increase.
		gasUsedDelta := new(big.Int).SetUint64(parentGasUsed - parentGasTarget)
		x := new(big.Int).Mul(parentBaseFee, gasUsedDelta)
		y := x.Div(x, parentGasTargetBig)
		baseFeeDelta := ethermint.BigMax(
			x.Div(y, baseFeeChangeDenominator),
			common.Big1,
		)

		return x.Add(parentBaseFee, baseFeeDelta)

func (k *Keeper) ApplyTransaction(ctx sdk.Context, msgEth *types.MsgEthereumTx) (*types.EVMResult, error) {
	ethTx := msgEth.AsTransaction()
	cfg, err := k.EVMConfig(ctx, k.eip155ChainID, ethTx.Hash())
	if err != nil {
		return nil, errorsmod.Wrap(err, "failed to load evm config")
	}

	msg := msgEth.AsMessage(cfg.BaseFee)
	// snapshot to contain the tx processing and post processing in same scope
	var commit func()
	var commitDurableAuthorization func()
	tmpCtxCommitted := false
	tmpCtx := ctx
	if k.hooks != nil {
		// Create a cache context to revert state when tx hooks fails,
		// the cache context is only committed when both tx and hooks executed successfully.
		// Didn't use `Snapshot` because the context stack has exponential complexity on certain operations,
		// thus restricted to be used only inside `ApplyMessage`.
		tmpCtx, commit = ctx.CacheContext()

		// Keep the EIP-7702 authorization effects in a separate cache so they
		// survive a later EVM/post-hook failure that discards tmpCtx. Only needed
		// for txs that actually carry authorizations.
		if len(msg.SetCodeAuthorizations) > 0 {
			var durableAuthorizationCtx sdk.Context
			durableAuthorizationCtx, commitDurableAuthorization = ctx.CacheContext()
			cfg.DurableSetCodeAuthorizationCtx = &durableAuthorizationCtx
		}
	}

	// pass true to commit the StateDB
	res, err := k.ApplyMessageWithConfig(tmpCtx, msg, cfg, true)
	if err != nil {
		return nil, errorsmod.Wrap(err, "failed to apply ethereum core message")
	}

	logs := types.LogsToEthereum(res.Logs)

	// Compute block bloom filter
	if len(logs) > 0 {
		bloom := ethtypes.Bloom{}
		for _, log := range logs {
			bloom.Add(log.Address.Bytes())
			for _, topic := range log.Topics {
				bloom.Add(topic[:])
			}
		}
		k.SetTxBloom(tmpCtx, bloom.Big())
	}

	var contractAddr common.Address
	if msg.To == nil {
		contractAddr = crypto.CreateAddress(msg.From, msg.Nonce)
	}

	receipt := &ethtypes.Receipt{
		Type:            ethTx.Type(),
		PostState:       nil, // TODO: intermediate state root
		Logs:            logs,
		TxHash:          cfg.TxConfig.TxHash,
		ContractAddress: contractAddr,
		GasUsed:         res.GasUsed,
		BlockHash:       cfg.TxConfig.BlockHash,
		BlockNumber:     cfg.BlockNumber,
	}

	if !res.Failed() {
		receipt.Status = ethtypes.ReceiptStatusSuccessful
		// Only call hooks if tx executed successfully.
		if err = k.PostTxProcessing(tmpCtx, msg, receipt); err != nil {
			// If hooks return error, revert the whole tx.
			res.VmError = types.ErrPostTxProcessing.Error()
			k.Logger(ctx).Error("tx post processing failed", "error", err)

			// If the tx failed in post processing hooks, we should clear the logs
			res.Logs = nil
		} else if commit != nil {
			// PostTxProcessing is successful, commit the tmpCtx
			commit()
			tmpCtxCommitted = true
			// Since the post-processing can alter the log, we need to update the result
			res.Logs = types.NewLogsFromEth(receipt.Logs)
		}
	}

	// Get the tracer and add OnGasChange hook for gas refund
	leftoverGas := msg.GasLimit - res.GasUsed

	// refund gas in order to match the Ethereum gas consumption instead of the default SDK one.
	if err = k.RefundGas(ctx, msg, leftoverGas, cfg.Params.EvmDenom); err != nil {
		return nil, errorsmod.Wrapf(err, "failed to refund leftover gas to sender %s", msg.From)
	}

	tracer := cfg.GetTracer()
	if tracer != nil && tracer.OnGasChange != nil {
		tracer.OnGasChange(leftoverGas, 0, tracing.GasChangeTxLeftOverReturned)
	}

	totalGasUsed, err := k.AddTransientGasUsed(ctx, res.GasUsed)
	if err != nil {
		return nil, errorsmod.Wrap(err, "failed to add transient gas used")
	}

	if commitDurableAuthorization != nil && !tmpCtxCommitted {
		commitDurableAuthorization()
	}

	// reset the gas meter for current cosmos transaction
	k.ResetGasMeterAndConsumeGas(ctx, totalGasUsed)

	return res, nil
}

func (k *Keeper) ApplyMessage(ctx sdk.Context, msg *core.Message, tracer *tracing.Hooks, commit bool) (*types.EVMResult, error) {
	cfg, err := k.EVMConfig(ctx, k.eip155ChainID, common.Hash{})
	if err != nil {
		return nil, errorsmod.Wrap(err, "failed to load evm config")
	}

	cfg.Tracer = tracer
	result, err := k.ApplyMessageWithConfig(ctx, msg, cfg, commit)
	if err != nil {
		return nil, err
	}

	return result, nil
}
