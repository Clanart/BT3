### Title
Missing Class Hash Validation in `execute_replace_class` Enables Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not validate that the new class hash supplied by a contract corresponds to a previously declared contract class. A malicious contract deployer can exploit this to replace a contract's class with an arbitrary undeclared hash, permanently freezing any user funds held in that contract.

---

### Finding Description

In `execute_replace_class` (`syscall_impls.cairo`, lines 878–916), the OS accepts any `class_hash` value from the `ReplaceClassRequest` and immediately writes it into `contract_state_changes` without checking whether that hash exists in the set of declared classes:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges this missing invariant. The `compiled_class_facts_bundle` in `OsGlobalContext` holds the set of valid declared classes for the entire OS execution: [2](#0-1) 

When a future transaction attempts to call a contract whose class hash was replaced with an undeclared value, the OS hint machinery (`GetContractAddressStateEntry`, `select_execute_entry_point_func`) cannot resolve the class and the OS execution fails. Because Cairo programs have no exception handling, this failure is unrecoverable for that contract's state — the funds are permanently inaccessible.

The analog to the external report is direct: just as `setLiquidityPool` allowed changing a critical address reference without migrating existing state (breaking user withdrawals), `execute_replace_class` allows changing a contract's critical class reference to an invalid value without any guard, breaking all future interactions with that contract.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

After a successful `replace_class` call with an undeclared hash:

1. The new (invalid) class hash is committed into the global state Merkle tree via `compute_contract_state_commitment`.
2. All subsequent calls to the contract (including user withdrawal functions) fail during OS execution because the class cannot be resolved from `compiled_class_facts_bundle`.
3. The sequencer cannot include any transaction that calls the contract, making the funds permanently inaccessible.
4. There is no recovery path: the contract's own code cannot be invoked to call `replace_class` again with a valid hash, because any invocation of the contract fails first. [3](#0-2) 

---

### Likelihood Explanation

**Medium.**

The attack requires a contract deployer to:
1. Deploy a contract containing a callable function that invokes `replace_class` with an attacker-chosen hash.
2. Attract user deposits (social engineering / fake token / vault).
3. Trigger the `replace_class` call.

Step 3 is a normal, valid transaction that the sequencer will include (it succeeds at execution time). The damage only manifests in subsequent blocks. The attacker-controlled entry path is the **contract deployer** role, which is explicitly listed as an unprivileged entry point in the scope. No privileged sequencer cooperation is required.

---

### Recommendation

Implement the missing validation noted in the TODO comment. Before writing the new class hash into `contract_state_changes`, verify that `class_hash` exists in the `compiled_class_facts_bundle` (or the deprecated equivalent). Concretely, add a lookup against the `contract_class_changes` dict (which tracks declared classes) or against the global compiled class facts, and revert with an error if the hash is not found. This mirrors the existing guard in `execute_declare_transaction` that enforces `prev_value=0` to prevent re-declaration: [4](#0-3) 

---

### Proof of Concept

1. **Attacker declares** a legitimate class `C_legit` and deploys contract `V` (a fake vault) using `C_legit`. Users deposit tokens into `V`.

2. **Attacker calls** a function in `V` (part of `C_legit`) that issues the syscall:
   ```
   replace_class(class_hash=0xdeadbeef_undeclared)
   ```
   The OS executes `execute_replace_class`, skips the missing validation, and writes `class_hash=0xdeadbeef_undeclared` into `contract_state_changes` for address `V`.

3. **State is committed.** `compute_contract_state_commitment` hashes the new `StateEntry` (with the invalid class hash) into the global state root. The block is proven successfully.

4. **In the next block**, a user submits a `withdraw` transaction calling `V`. The sequencer pre-executes it: the OS reads `state_entry.class_hash = 0xdeadbeef_undeclared` for `V`, attempts to resolve it in `compiled_class_facts_bundle`, fails, and the OS execution aborts. The sequencer cannot include this transaction.

5. **All user funds in `V` are permanently frozen.** No transaction can ever successfully call `V` again. [5](#0-4)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
