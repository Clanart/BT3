### Title
`meta_tx_v0` Signature Replay Due to Missing Nonce in Hash — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_meta_tx_v0` syscall constructs a version-0 transaction hash that omits any nonce. The OS also explicitly skips nonce checking and `__validate__` for version-0 transactions. As a result, any valid `meta_tx_v0` signature observed on-chain can be replayed by an unprivileged attacker an unlimited number of times, leading to direct loss of funds.

---

### Finding Description

`execute_meta_tx_v0` in `syscall_impls.cairo` is a syscall callable from any executing contract. It computes a transaction hash via `compute_meta_tx_v0_hash` and then calls the target contract's `__execute__` entry point with that hash and a caller-supplied signature.

**Root cause 1 — No nonce in the hash:**

`compute_meta_tx_v0_hash` calls `deprecated_get_transaction_hash` with `additional_data_size=0` and `additional_data=cast(0, felt*)`:

```cairo
// transaction_hash.cairo lines 302-313
let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
    tx_hash_prefix=INVOKE_HASH_PREFIX,
    version=0,
    contract_address=contract_address,
    entry_point_selector=entry_point_selector,
    calldata_size=calldata_size,
    calldata=calldata,
    max_fee=0,
    chain_id=chain_id,
    additional_data_size=0,          // ← no nonce
    additional_data=cast(0, felt*),  // ← no nonce
);
```

The hash is fully determined by `(contract_address, selector, calldata, chain_id)`. There is no nonce, block number, or any other uniqueness field.

**Root cause 2 — OS skips nonce check for version 0:**

`check_and_increment_nonce` explicitly returns early for version-0 transactions:

```cairo
// execute_transaction_utils.cairo lines 64-67
if (tx_info.version == 0) {
    return ();
}
```

The `new_tx_info` built in `execute_meta_tx_v0` sets `nonce=0` and `version=0`, so no nonce is ever checked or incremented.

**Root cause 3 — OS skips `__validate__` for version 0:**

```cairo
// execute_transaction_utils.cairo lines 127-130
// Do not run "__validate__" for version 0.
if (tx_execution_info.tx_info.version == 0) {
    return ();
}
```

The target contract's `__validate__` is never called. Only `__execute__` runs, and it receives the same hash and signature every replay.

**Combined effect:** Given a previously-observed `(contract_address, calldata, signature)` tuple from any past `meta_tx_v0` syscall, an attacker can replay it by deploying a contract that calls the `meta_tx_v0` syscall with the same arguments. The target contract's `__execute__` will see an identical `transaction_hash` and identical `signature`, pass its own signature check, and execute the same action again.

---

### Impact Explanation

**Critical — Direct loss of funds.**

If the target contract's `__execute__` performs any fund-moving operation (token transfer, withdrawal, approval, etc.) based on the signed calldata, an attacker can replay the same signature to repeat that operation indefinitely. The target contract has no OS-level protection: the nonce is always 0, the hash is always the same, and the signature is always valid.

---

### Likelihood Explanation

Any unprivileged user can:
1. Observe a valid `meta_tx_v0` call in a past block (signature and calldata are public on-chain).
2. Deploy a contract that issues the `meta_tx_v0` syscall with the same `contract_address`, `selector`, `calldata`, and `signature`.
3. Repeat until the victim's funds are drained.

No privileged role, leaked key, or social engineering is required. The attacker only needs to have seen one successful `meta_tx_v0` transaction.

---

### Recommendation

Include a nonce in `compute_meta_tx_v0_hash` — specifically, pass the target contract's current on-chain nonce as `additional_data` and increment it after each successful `meta_tx_v0` execution, analogous to how `check_and_increment_nonce` works for version-3 transactions. This makes each meta-transaction hash unique and non-replayable.

---

### Proof of Concept

1. Alice signs a `meta_tx_v0` to transfer 1000 STRK from her account contract. The OS computes:
   `hash = H(INVOKE_PREFIX, 0, alice_addr, __execute__, calldata, 0, chain_id)`
   and calls `alice.__execute__(calldata)` with Alice's signature. Alice's account contract verifies the signature against `hash` and executes the transfer.

2. Attacker Bob observes the transaction on-chain, extracting `calldata` and `signature`.

3. Bob deploys `AttackerContract` with:
   ```
   func __execute__():
       meta_tx_v0(contract_address=alice_addr, selector=__execute__, calldata=alice_calldata, signature=alice_sig)
   ```

4. Bob calls `AttackerContract.__execute__()`. The OS computes the identical hash (same inputs, no nonce), sets `nonce=0`, skips `__validate__`, and calls `alice.__execute__` with the same hash and signature. Alice's account contract validates successfully and executes another 1000 STRK transfer.

5. Bob repeats step 4 until Alice's account is drained.
