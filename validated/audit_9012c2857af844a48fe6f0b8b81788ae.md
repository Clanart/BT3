### Title
Stale Pending Block Fallback Omits `l1_data_gas_price` and `l2_gas_price`, Causing Wrong Fee Estimates from `starknet_estimateFee` and `starknet_simulateTransactions` — (File: `crates/apollo_rpc/src/v0_8/api/api_impl.rs`)

---

### Summary

When the in-memory pending block is stale (its `parent_block_hash` does not match the latest accepted block hash), `read_pending_data` synthesises a fallback `DeprecatedPendingBlock`. That fallback correctly copies `l1_gas_price` from the latest header but cannot carry `l1_data_gas_price` or `l2_gas_price` because the deprecated struct has no such fields; both default to zero. `estimate_fee` and `simulate_transactions` called with `BlockId::Tag(Tag::Pending)` consume this fallback, so the block context used for execution has zero `l2_gas_price` and zero `l1_data_gas_price`. Fee estimates for V3 (`AllResources`) transactions and KZG-DA transactions are therefore systematically wrong (too low). Other pending-block queries (`get_nonce`, `get_storage_at`, `get_state_update`) correctly fall back to the latest accepted state without this defect.

---

### Finding Description

`read_pending_data` (lines 1553–1595 of `api_impl.rs`) checks whether the cached pending block's `parent_block_hash` matches the latest accepted block hash. When they differ (stale), it constructs a synthetic fallback:

```rust
PendingBlockOrDeprecated::Deprecated(DeprecatedPendingBlock {
    parent_block_hash: latest_header.block_hash,
    eth_l1_gas_price: latest_header.block_header_without_hash.l1_gas_price.price_in_wei,
    strk_l1_gas_price: latest_header.block_header_without_hash.l1_gas_price.price_in_fri,
    // DeprecatedPendingBlock has no l1_data_gas_price or l2_gas_price fields
    ..Default::default()
})
``` [1](#0-0) 

`DeprecatedPendingBlock` has no `l1_data_gas_price` or `l2_gas_price` fields. The `PendingBlockOrDeprecated::l2_gas_price()` method explicitly returns `GasPricePerToken::default()` (zero) for the `Deprecated` variant:

```rust
pub fn l2_gas_price(&self) -> GasPricePerToken {
    match self {
        // In older versions, L2 gas price was 0.
        PendingBlockOrDeprecated::Deprecated(_) => GasPricePerToken::default(),
        PendingBlockOrDeprecated::Current(block) => block.l2_gas_price,
    }
}
``` [2](#0-1) 

`estimate_fee` and `simulate_transactions` both call `read_pending_data` and pass the result through `client_pending_data_to_execution_pending_data`, which copies `l2_gas_price` and `l1_data_gas_price` directly from the pending block:

```rust
l1_gas_price: client_pending_data.block.l1_gas_price(),
l1_data_gas_price: client_pending_data.block.l1_data_gas_price(),
l2_gas_price: client_pending_data.block.l2_gas_price(),
``` [3](#0-2) 

The resulting `ExecutionPendingData` therefore has zero `l2_gas_price` and zero `l1_data_gas_price`. This is then used to construct the `BlockInfo` for execution inside `exec_estimate_fee` / `exec_simulate_transactions`. [4](#0-3) [5](#0-4) 

**The inconsistency (direct analog to the external report):**

| Gas price field | Fallback value | Correct? |
|---|---|---|
| `l1_gas_price` | Copied from latest header | ✓ |
| `l1_data_gas_price` | Zero (field absent in deprecated format) | ✗ |
| `l2_gas_price` | Zero (field absent in deprecated format) | ✗ |

Other pending-block RPC methods handle the stale case correctly — `get_nonce` returns the nonce from the latest accepted block, `get_state_update` returns an empty state diff — but `estimate_fee` and `simulate_transactions` silently use zero gas prices. [6](#0-5) 

---

### Impact Explanation

`starknet_estimateFee` and `starknet_simulateTransactions` with `BlockId::Tag(Tag::Pending)` return authoritative-looking but wrong fee estimates whenever the pending block is stale. The L2 gas fee component and L1 data gas fee component are both zero, making the total fee estimate too low. A client that uses this estimate to set `max_amount` in `AllResources` resource bounds will submit a transaction whose bounds are below the actual cost; the transaction will be reverted post-execution with `MaxGasAmountExceeded`, and the user is charged the capped fee for a failed transaction. This matches the allowed impact: **"High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."**

---

### Likelihood Explanation

The pending block becomes stale on every block acceptance, for the brief interval between the new block being committed and the pending-sync loop refreshing the in-memory `PendingData`. This is a normal, recurring condition on any live node. Any `starknet_estimateFee` or `starknet_simulateTransactions` call with the pending tag that lands in this window receives the wrong estimate. No special attacker capability is required; the window is unprivileged and observable by any RPC client.

---

### Recommendation

Replace the `DeprecatedPendingBlock` fallback in `read_pending_data` with a `PendingBlock` (current format) that copies all three gas price fields from the latest accepted block header:

```rust
PendingBlockOrDeprecated::Current(PendingBlock {
    parent_block_hash: latest_header.block_hash,
    l1_gas_price: latest_header.block_header_without_hash.l1_gas_price,
    l1_data_gas_price: latest_header.block_header_without_hash.l1_data_gas_price,
    l2_gas_price: latest_header.block_header_without_hash.l2_gas_price,
    // ... other fields from latest header
    ..Default::default()
})
```

This mirrors the fix applied to the external report: just as `maxDeposit`/`maxMint` were made to check the paused state and return 0, the stale-pending fallback must be made to carry the correct gas prices so that `estimate_fee` and `simulate_transactions` return values that are actually usable.

---

### Proof of Concept

1. Block N is accepted by the sequencer; the in-memory `pending_data` still has `parent_block_hash = hash(N-1)` (stale).
2. Before the pending-sync loop refreshes, a client calls `starknet_estimateFee` with `BlockId::Tag(Tag::Pending)` for a V3 `InvokeTransaction` using `AllResources` bounds.
3. `read_pending_data` detects the stale condition and returns a `DeprecatedPendingBlock` fallback with `l2_gas_price = GasPricePerToken { price_in_wei: 0, price_in_fri: 0 }` and `l1_data_gas_price = GasPricePerToken { price_in_wei: 0, price_in_fri: 0 }`.
4. `client_pending_data_to_execution_pending_data` copies these zero prices into `ExecutionPendingData`.
5. `exec_estimate_fee` builds a `BlockInfo` with zero `l2_gas_price` and zero `l1_data_gas_price`; the fee estimate returned to the client has `l2_gas_consumed * 0 = 0` and `l1_data_gas_consumed * 0 = 0`.
6. The client sets `l2_gas.max_amount` and `l1_data_gas.max_amount` based on this estimate and submits the transaction.
7. The actual block N+1 has non-zero `l2_gas_price`; post-execution fee check fires `FeeCheckError::MaxGasAmountExceeded`, the transaction is reverted, and the user is charged the capped fee for a failed transaction. [7](#0-6) [8](#0-7) [6](#0-5)

async fn read_pending_data<Mode: TransactionKind>(
    pending_data: &Arc<RwLock<PendingData>>,
    txn: &StorageTxn<'_, Mode>,
) -> RpcResult<PendingData> {
    let latest_header = match get_latest_block_number(txn)? {
        Some(latest_block_number) => get_block_header_by_number(txn, latest_block_number)?,
        None => starknet_api::block::BlockHeader {
            // TODO(Shahak): Consider adding genesis hash to the config to support chains that have
            // different genesis hash.
            block_header_without_hash: BlockHeaderWithoutHash {
                parent_hash: BlockHash::GENESIS_PARENT_HASH,
                ..Default::default()
            },
            ..Default::default()
        },
    };
    let pending_data = &pending_data.read().await;
    if pending_data.block.parent_block_hash() == latest_header.block_hash {
        Ok((*pending_data).clone())
    } else {
        Ok(PendingData {
            block: PendingBlockOrDeprecated::Deprecated(DeprecatedPendingBlock {
                parent_block_hash: latest_header.block_hash,
                eth_l1_gas_price: latest_header.block_header_without_hash.l1_gas_price.price_in_wei,
                strk_l1_gas_price: latest_header
                    .block_header_without_hash
                    .l1_gas_price
                    .price_in_fri,
                timestamp: latest_header.block_header_without_hash.timestamp,
                sequencer_address: latest_header.block_header_without_hash.sequencer,
                starknet_version: latest_header
                    .block_header_without_hash
                    .starknet_version
                    .to_string(),
                ..Default::default()
            }),
            state_update: ClientPendingStateUpdate {
                old_root: latest_header.block_header_without_hash.state_root,
                state_diff: Default::default(),
            },
        })
    }
}
