### Title
`ProofOfInclusion::valid()` Tautological Root-Hash Check Allows Forged Inclusion Proofs — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` contains a final comparison that is always `true` when the loop completes without returning `false`. The function never validates the proof against any external root hash, so any internally self-consistent `ProofOfInclusion` — including one crafted by an attacker for a key that does not exist in the tree — passes validation.

### Finding Description

`ProofOfInclusion::valid()` is the sole validation method exposed for DataLayer Merkle proofs, both in Rust and through the Python/wasm bindings. [1](#0-0) 

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;   // ← set to calculated_hash
    }

    existing_hash == self.root_hash()      // ← always true (see below)
}
```

`root_hash()` is defined as: [2](#0-1) 

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← returns the last layer's combined_hash
    } else {
        self.node_hash
    }
}
```

After the loop body executes for the last layer:

1. The loop body already asserted `calculated_hash == layer.combined_hash` (otherwise it returned `false`).
2. `existing_hash` is then set to `calculated_hash`.
3. Therefore `existing_hash == self.layers.last().combined_hash`.
4. `self.root_hash()` also returns `self.layers.last().combined_hash`.
5. The final comparison `existing_hash == self.root_hash()` is **unconditionally true** whenever the loop completes.

The function therefore only checks that the layers are internally self-consistent (each `combined_hash` matches the hash computed from the previous layer and `other_hash`). It never compares the computed root against any externally-known, trusted tree root.

An attacker can construct a `ProofOfInclusion` with an arbitrary `node_hash` (the hash of a key-value pair that is **not** in the tree) and a chain of internally consistent `ProofOfInclusionLayer` values. `valid()` will return `true` for this forged proof.

The struct is fully mutable from Python via `get_all` and `from_py_object`: [3](#0-2) 

and is exported to Python callers through the wheel: [4](#0-3) 

Every test and fuzz target in the repository calls only `proof.valid()` without separately comparing `proof.root_hash()` against the tree's actual root: [5](#0-4) [6](#0-5) [7](#0-6) 

### Impact Explanation

An attacker who can supply a `ProofOfInclusion` object (e.g., via the Python API, a network message, or deserialized bytes) can forge a proof that any arbitrary key-value pair is present in any DataLayer tree. Because `valid()` is the only validation gate and it never checks against the actual tree root, the forged proof passes. This lets untrusted input prove invalid state — matching the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

### Likelihood Explanation

The `ProofOfInclusion` struct is `Streamable` and constructible from raw bytes (`from_bytes`, `from_bytes_unchecked`) and from Python (`from_py_object`). Any code path that deserializes a proof from an untrusted peer and calls `valid()` is immediately exploitable. The misleading tautological check at line 57 gives implementers false confidence that the proof is fully validated, making it likely that callers omit the separate `root_hash()` comparison.

### Recommendation

`valid()` must accept the expected root hash as a parameter and compare `existing_hash` against it instead of against `self.root_hash()`:

```rust
pub fn valid_against_root(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root   // compare against the TRUSTED external root
}
```

Alternatively, if the zero-argument `valid()` is kept for internal-consistency checks only, it must be clearly documented as insufficient for security, and all call sites must be audited to ensure they also compare `proof.root_hash()` against the trusted tree root obtained from a trusted source (e.g., the on-chain commitment).

### Proof of Concept

```python
from chia_rs.datalayer import (
    MerkleBlob, ProofOfInclusion, ProofOfInclusionLayer, KeyId, ValueId
)

# Build a real tree with one entry
blob = MerkleBlob(blob=bytearray())
real_key   = KeyId(1)
real_value = ValueId(1)
real_hash  = bytes(range(32))
blob.insert(real_key, real_value, real_hash)
blob.calculate_lazy_hashes()

# Obtain a real proof to copy its layer structure
real_proof = blob.get_proof_of_inclusion(real_key)
assert real_proof.valid()

# Forge a proof for a key that is NOT in the tree.
# Use the same layers but substitute an arbitrary node_hash.
fake_node_hash = bytes([0xAB] * 32)
forged_proof = ProofOfInclusion(
    node_hash=fake_node_hash,
    layers=real_proof.layers,   # internally consistent layers
)

# valid() returns True for the forged proof even though fake_node_hash
# is not in the tree and the root_hash differs from the real tree root.
assert forged_proof.valid()          # ← passes: tautological check
assert forged_proof.root_hash() != blob.get_root_hash()  # ← different root
```
