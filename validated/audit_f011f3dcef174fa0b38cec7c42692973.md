### Title
Caller-Provided `shard_id` Not Cross-Validated Against Chunk's `shard_id` in State-Sync Header Acceptance — (File: `chain/chain/src/state_sync/adapter.rs`)

---

### Summary

In `ChainStateSyncAdapter::set_state_header`, the caller-supplied `shard_id` is used as the authoritative shard identity throughout receipt-hash computation and DB storage, but it is **never compared** against `chunk.shard_id()` extracted from the peer-supplied `ShardStateSyncResponseHeader`. A malicious peer can substitute a valid chunk for shard X into a response for shard Y, pass every existing validation gate, and cause the syncing node to store shard X's state root under shard Y's `StateHeaderKey`, then apply shard X's trie data into shard Y's storage.

---

### Finding Description

`set_state_header` accepts three inputs: the local `shard_id` (caller-controlled), `sync_hash` (local chain), and `shard_state_header` (peer-supplied). It extracts the chunk from the header and runs five validation steps:

1. **Step 1-2** – `validate_chunk_proofs` checks internal self-consistency of the chunk (hash, tx-root, receipts-root). It does **not** check `chunk.shard_id()` against the requested `shard_id`.

2. **Step 3a** – Verifies the chunk hash is present in `sync_prev_block_header.chunk_headers_root()` via a Merkle path. This proves the chunk exists in the block, but **not which shard-index position** it occupies. A chunk for shard X at position X in the tree can be proven present even when shard Y was requested.

3. **Step 3b** – Same structural gap for `prev_chunk_header`.

4. **Step 4** – Receipt-hash computation uses the caller's `shard_id`:
   ```rust
   let receipts_hash = CryptoHash::hash_borsh(ReceiptList(shard_id, receipts));
   ```
   The `to_shard_id` field of the peer-supplied `ShardProof` is explicitly discarded (`to_shard_id: _`). A peer can supply legitimate receipts from shard Z → shard Y (which hash correctly under `ReceiptList(Y, …)`) while the chunk in the header belongs to shard X. If `incoming_receipts_proofs` is empty (common at epoch boundaries), the loop body never executes and the check is trivially satisfied.

5. **Step 5** – `validate_state_root_node` validates the state-root node against `chunk_inner.prev_state_root()`. If the chunk is shard X's chunk, this validates shard X's state root — which is exactly what the malicious peer wants.

After all checks pass, the header is persisted:
```rust
let key = borsh::to_vec(&StateHeaderKey(shard_id, sync_hash)).unwrap();
store_update.set_ser(DBCol::StateHeaders, &key, &shard_state_header);
```

`shard_id` here is the caller's value (Y), but `shard_state_header` contains shard X's chunk and state root. Downstream, `set_state_part` reads the stored header, extracts shard X's `state_root`, validates incoming parts against it, and `apply_state_part` writes them into `shard_uid_Y`'s trie storage. ### Impact Explanation

A syncing node (validator or RPC) that accepts a poisoned header will:

- Store shard X's `prev_state_root` under `StateHeaderKey(shard_Y, sync_hash)`.
- Accept and apply shard X's trie nodes into shard Y's `ShardUId` column in RocksDB.
- After `set_state_finalize`, execute shard X's chunk transactions against shard Y's state, producing a completely wrong `ChunkExtra` for shard Y.
- Produce chunk headers with an incorrect `prev_state_root` for shard Y, which every honest validator will reject.

For a validator node this means inability to produce valid chunks for shard Y, potential ejection from the validator set, and possible slashing. For an RPC/archival node it means permanently corrupted state that cannot be detected without a full re-sync.

**Severity: High** — broken reconstruction invariant (`state_root` ↔ `shard_id` binding), unprivileged peer-controlled data, exact corrupted value is `StateHeaderKey(shard_Y, sync_hash)` → shard X's state root.

---

### Likelihood Explanation

Any peer node (no special role required) that the syncing node contacts for state sync can supply the malicious header. The syncing node downloads the header from a single peer before downloading parts. If the same peer (or a colluding set) also serves the matching state parts for shard X, the full attack completes silently. At epoch boundaries `incoming_receipts_proofs` is frequently empty, removing the only partial barrier. The attack is deterministic and requires no brute-force.

---

### Recommendation

Add an explicit shard-id binding check immediately after extracting the chunk from the peer-supplied header, before any other validation:

```rust
let chunk = shard_state_header.cloned_chunk();
// NEW: enforce that the peer-supplied chunk belongs to the requested shard.
if chunk.shard_id() != shard_id {
    byzantine_assert!(false);
    return Err(Error::Other(format!(
        "set_shard_state failed: chunk shard_id {:?} does not match requested shard_id {:?}",
        chunk.shard_id(), shard_id
    )));
}
```

Apply the same check to `prev_chunk_header.shard_id()` when present. Additionally, restore the `to_shard_id` field in the `ShardProof` destructuring and assert it equals the caller's `shard_id` to close the receipt-proof gap:

```rust
let ShardProof { from_shard_id, to_shard_id, proof } = shard_proof;
if to_shard_id != shard_id {
    return Err(Error::Other("set_shard_state failed: receipt to_shard_id mismatch".into()));
}

### Proof of Concept

**Setup**: Two-shard network (shard 0, shard 1). Syncing node requests state for shard 1.

**Attacker** (malicious peer) constructs a `ShardStateSyncResponseHeader` where:
- `chunk` = the honest shard-0 chunk from `sync_prev_block` (valid chunk hash, valid internal proofs)
- `chunk_proof` = the Merkle path proving shard-0's chunk hash is in `sync_prev_block.chunk_headers_root()` (valid, obtained from the honest chain)
- `prev_chunk_header` = shard-0's previous chunk header (valid, in the block)
- `prev_chunk_proof` = Merkle path for shard-0's prev chunk (valid)
- `incoming_receipts_proofs` = `[]` (empty; epoch-boundary blocks commonly have no cross-shard receipts for the target shard)
- `root_proofs` = `[]` (matches empty receipts)
- `state_root_node` = shard-0's honest state root node (passes `validate_state_root_node`)

**Execution on syncing node**:
1. `validate_chunk_proofs(shard_0_chunk)` → passes (internally consistent).
2. `verify_path(chunk_headers_root, shard_0_chunk_proof, ChunkHashHeight(shard_0_hash, …))` → passes (shard-0 chunk IS in the block).
3. Receipt loop: empty, skipped.
4. Step 4g: `sync_hash` height == `prev_chunk_header.height_included()` → passes.
5. `validate_state_root_node(shard_0_state_root_node, shard_0_prev_state_root)` → passes.
6. Stored: `StateHeaderKey(shard_1, sync_hash)` → shard-0's `ShardStateSyncResponseHeader`.

**Follow-up**: Attacker serves shard-0 state parts. `set_state_part` validates them against shard-0's state root (passes). `apply_state_part` writes shard-0's trie nodes into `ShardUId { shard_id: 1, … }`. Syncing node's shard-1 state is now shard-0's data. [8](#0-7) [5](#0-4) [9](#0-8)

    pub fn set_state_header(
        &self,
        shard_id: ShardId,
        sync_hash: CryptoHash,
        shard_state_header: ShardStateSyncResponseHeader,
    ) -> Result<(), Error> {
        let sync_block_header = self.chain_store.get_block_header(&sync_hash)?;

        let chunk = shard_state_header.cloned_chunk();
        let prev_chunk_header = shard_state_header.cloned_prev_chunk_header();

        // 1-2. Checking chunk validity
        if !validate_chunk_proofs(&chunk, self.epoch_manager.as_ref())? {
            byzantine_assert!(false);
            return Err(Error::Other(
                "set_shard_state failed: chunk header proofs are invalid".into(),
            ));
        }

        // Consider chunk itself is valid.

        // 3. Checking that chunks `chunk` and `prev_chunk` are included in appropriate blocks
        // 3a. Checking that chunk `chunk` is included into block at last height before sync_hash
        // 3aa. Also checking chunk.height_included
        let sync_prev_block_header =
            self.chain_store.get_block_header(sync_block_header.prev_hash())?;
        if !verify_path(
            *sync_prev_block_header.chunk_headers_root(),
            shard_state_header.chunk_proof(),
            &ChunkHashHeight(chunk.chunk_hash().clone(), chunk.height_included()),
        ) {
            byzantine_assert!(false);
            return Err(Error::Other(
                "set_shard_state failed: chunk isn't included into block".into(),
            ));
        }

        let block_header = get_block_header_on_chain_by_height(
            &self.chain_store,
            &sync_hash,
            chunk.height_included(),
        )?;
        // 3b. Checking that chunk `prev_chunk` is included into block at height before chunk.height_included
        // 3ba. Also checking prev_chunk.height_included - it's important for getting correct incoming receipts
        match (&prev_chunk_header, shard_state_header.prev_chunk_proof()) {
            (Some(prev_chunk_header), Some(prev_chunk_proof)) => {
                let prev_block_header =
                    self.chain_store.get_block_header(block_header.prev_hash())?;
                if !verify_path(
                    *prev_block_header.chunk_headers_root(),
                    prev_chunk_proof,
                    &ChunkHashHeight(prev_chunk_header.chunk_hash().clone(), prev_chunk_header.height_included()),
                ) {
                    byzantine_assert!(false);
                    return Err(Error::Other(
                        "set_shard_state failed: prev_chunk isn't included into block".into(),
                    ));
                }
            }
            (None, None) => {
                if chunk.height_included() != 0 {
                    return Err(Error::Other(
                    "set_shard_state failed: received empty state response for a chunk that is not at height 0".into()
                ));
                }
            }
            _ =>
                return Err(Error::Other("set_shard_state failed: `prev_chunk_header` and `prev_chunk_proof` must either both be present or both absent".into()))
        };

        // 4. Proving incoming receipts validity
        // 4a. Checking len of proofs
        if shard_state_header.root_proofs().len()
            != shard_state_header.incoming_receipts_proofs().len()
        {
            byzantine_assert!(false);
            return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
        }
        let mut hash_to_compare = sync_hash;
        for (i, receipt_response) in
            shard_state_header.incoming_receipts_proofs().iter().enumerate()
        {
            let ReceiptProofResponse(block_hash, receipt_proofs) = receipt_response;

            // 4b. Checking that there is a valid sequence of continuous blocks
            if *block_hash != hash_to_compare {
                byzantine_assert!(false);
                return Err(Error::Other(
                    "set_shard_state failed: invalid incoming receipts".into(),
                ));
            }
            let header = self.chain_store.get_block_header(&hash_to_compare)?;
            hash_to_compare = *header.prev_hash();

            let block_header = self.chain_store.get_block_header(block_hash)?;
            // 4c. Checking len of receipt_proofs for current block
            if receipt_proofs.len() != shard_state_header.root_proofs()[i].len()
                || receipt_proofs.len() != block_header.chunks_included() as usize
            {
                byzantine_assert!(false);
                return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
            }
            // We know there were exactly `block_header.chunks_included` chunks included
            // on the height of block `block_hash`.
            // There were no other proofs except for included chunks.
            // According to Pigeonhole principle, it's enough to ensure all receipt_proofs are distinct
            // to prove that all receipts were received and no receipts were hidden.
            let mut visited_shard_ids = HashSet::<ShardId>::new();
            for (j, receipt_proof) in receipt_proofs.iter().enumerate() {
                let ReceiptProof(receipts, shard_proof) = receipt_proof;
                let ShardProof { from_shard_id, to_shard_id: _, proof } = shard_proof;
                // 4d. Checking uniqueness for set of `from_shard_id`
                match visited_shard_ids.get(from_shard_id) {
                    Some(_) => {
                        byzantine_assert!(false);
                        return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
                    }
                    _ => visited_shard_ids.insert(*from_shard_id),
                };
                let RootProof(root, block_proof) = &shard_state_header.root_proofs()[i][j];
                let receipts_hash = CryptoHash::hash_borsh(ReceiptList(shard_id, receipts));
                // 4e. Proving the set of receipts is the subset of outgoing_receipts of shard `shard_id`
                if !verify_path(*root, proof, &receipts_hash) {
                    byzantine_assert!(false);
                    return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
                }
                // 4f. Proving the outgoing_receipts_root matches that in the block
                if !verify_path(
                    *block_header.prev_chunk_outgoing_receipts_root(),
                    block_proof,
                    root,
                ) {
                    byzantine_assert!(false);
                    return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
                }
            }
        }
        // 4g. Checking that there are no more heights to get incoming_receipts
        let header = self.chain_store.get_block_header(&hash_to_compare)?;
        if header.height() != prev_chunk_header.map_or(0, |h| h.height_included()) {
            byzantine_assert!(false);
            return Err(Error::Other("set_shard_state failed: invalid incoming receipts".into()));
        }

        // 5. Checking that state_root_node is valid
        let chunk_inner = chunk.take_header().take_inner();
        if matches!(
            self.runtime_adapter.validate_state_root_node(
                shard_state_header.state_root_node(),
                chunk_inner.prev_state_root(),
            ),
            StateRootNodeValidationResult::Invalid
        ) {
            byzantine_assert!(false);
            return Err(Error::Other("set_shard_state failed: state_root_node is invalid".into()));
        }

        // Saving the header data.
        let mut store_update = self.chain_store.store().store_update();
        let key = borsh::to_vec(&StateHeaderKey(shard_id, sync_hash)).unwrap();
        store_update.set_ser(DBCol::StateHeaders, &key, &shard_state_header);
        store_update.commit();

        Ok(())

    pub fn set_state_part(
        &self,
        shard_id: ShardId,
        sync_hash: CryptoHash,
        part_id: PartId,
        part: &StatePart,
    ) -> Result<(), Error> {
        let shard_state_header = self.get_state_header(shard_id, sync_hash)?;
        let chunk = shard_state_header.take_chunk();
        let state_root = *chunk.take_header().take_inner().prev_state_root();

                "set_state_part failed: validate_state_part failed. state_root={:?}",

        let key = borsh::to_vec(&StatePartKey(sync_hash, shard_id, part_id.idx)).unwrap();
        let bytes = part.to_bytes();
        store_update.set(DBCol::StateParts, &key, &bytes);
        store_update.commit();
        Ok(())
