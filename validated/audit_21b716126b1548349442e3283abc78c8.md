### Title
EIP-712 Amino Sign-Doc Omits `timeout_height` from Type Schema, Enabling Post-Expiry Replay — (`ethereum/eip712/types.go`, `ethereum/eip712/encoding.go`)

### Summary

The EIP-712 `Tx` struct type definition hard-codes five fields and explicitly omits `timeout_height`. Because EIP-712 `hashStruct` only encodes fields present in the type schema, any value of `timeout_height` in the Amino sign-doc JSON is silently dropped from the hash. A signature produced over a sign-doc with `timeout_height=N` is therefore cryptographically identical (under EIP-712 verification) to one produced over the same sign-doc with `timeout_height=0`. An unprivileged attacker who observes a broadcast-but-unconfirmed EIP-712/Amino tx can strip its `timeout_height`, resubmit after the deadline, and have the signature accepted.

### Finding Description

**Root cause — `ethereum/eip712/types.go` lines 74–81:**

The `Tx` type is hard-coded with exactly five fields. `timeout_height` is absent, with an inline comment acknowledging the omission:

```go
"Tx": {
    {Name: "account_number", Type: "string"},
    {Name: "chain_id",       Type: "string"},
    {Name: "fee",            Type: "Fee"},
    {Name: "memo",           Type: "string"},
    {Name: "sequence",       Type: "string"},
    // Note timeout_height was removed because it was not getting filled with the legacyTx
},
``` [1](#0-0) 

**Amino decode path — `ethereum/eip712/encoding.go` lines 86–132:**

`decodeAminoSignDoc` unmarshals the raw sign-doc bytes and passes them **unchanged** to `WrapTxToTypedData`. No check for a non-zero `timeout_height` is performed:

```go
typedData, err := WrapTxToTypedData(
    chainID.Uint64(),
    signDocBytes,   // full JSON including "timeout_height":"N"
)
``` [2](#0-1) 

Contrast with the **Protobuf decode path** (`decodeProtobufSignDoc`), which explicitly rejects any non-zero `timeout_height`:

```go
if body.TimeoutHeight != 0 || ... {
    return apitypes.TypedData{}, errors.New(
        "body contains unsupported fields: TimeoutHeight, ...")
}
``` [3](#0-2) 

The Amino path has no equivalent guard, so a sign-doc with `timeout_height=N` is accepted and forwarded to the EIP-712 hasher, which silently ignores the field.

**Verification entry point — `crypto/ethsecp256k1/ethsecp256k1.go` line 227:**

```go
func (pubKey PubKey) VerifySignature(msg, sig []byte) bool {
    return pubKey.verifySignatureECDSA(msg, sig) || pubKey.verifySignatureAsEIP712(msg, sig)
}
``` [4](#0-3) 

The `||` means: if the ECDSA path fails (bytes differ because `timeout_height` was stripped), the EIP-712 path is tried. Because the EIP-712 hash is identical for any `timeout_height` value, the EIP-712 path succeeds and `VerifySignature` returns `true`.

**Ante handler ordering — `evmd/ante/evm_handler.go` lines 46, 56:**

`TxTimeoutHeightDecorator` runs **before** signature verification in both the legacy EIP-712 handler and the standard Cosmos handler: [5](#0-4) 

A tx with `timeout_height=0` passes `TxTimeoutHeightDecorator` unconditionally (0 means "no deadline"), then reaches signature verification with the stripped sign-doc.

### Impact Explanation

An attacker who observes a user's EIP-712/Amino-signed tx with `timeout_height=N` that was broadcast but never included in a block (e.g., dropped from the mempool due to congestion) can:

1. Copy the tx body and signature verbatim.
2. Set `timeout_height=0` in the tx body.
3. Resubmit after block N.

The `TxTimeoutHeightDecorator` passes (no deadline). `VerifySignature` passes via the EIP-712 branch. The sequence number is still valid (the original tx was never executed). The tx executes — transferring funds, delegating, or performing any other action the user intended to be time-bounded.

This is a direct EIP-712 authorization bypass enabling replay of an expired transaction to authorize a fund transfer, matching the **High** impact category.

### Likelihood Explanation

- Requires no privileged access; any observer of the mempool or p2p gossip layer can perform this.
- The precondition (a tx with `timeout_height` set that was not confirmed before expiry) is realistic: users set `timeout_height` precisely to guard against late execution, and mempool drops are common under load.
- The manipulation (changing one field in the tx body) is trivial with any Cosmos SDK client library.

### Recommendation

Apply the same guard used in `decodeProtobufSignDoc` to `decodeAminoSignDoc`: reject any Amino sign-doc whose `TimeoutHeight` field is non-zero before constructing the EIP-712 typed data. Alternatively, add `timeout_height` to the `Tx` EIP-712 type definition so it is covered by the hash.

```go
// In decodeAminoSignDoc, after unmarshaling aminoDoc:
if aminoDoc.TimeoutHeight != "" && aminoDoc.TimeoutHeight != "0" {
    return apitypes.TypedData{}, errors.New(
        "body contains unsupported field: TimeoutHeight")
}
```

The same fix must be applied to `legacyDecodeAminoSignDoc` in `encoding_legacy.go`. [6](#0-5) 

### Proof of Concept

```
1. Alice signs MsgSend(from=Alice, to=Bob, amount=100) with timeout_height=500
   using SIGN_MODE_LEGACY_AMINO_JSON + EIP-712.
   Sign-doc JSON: {"account_number":"1","chain_id":"ethermint_9000-1",
                   "fee":{...},"memo":"","msgs":[...],"sequence":"0",
                   "timeout_height":"500"}
   EIP-712 hash H = hash(Tx without timeout_height field) — same as if timeout_height=0.

2. Tx is broadcast at block 490 but dropped from mempool; never confirmed.

3. At block 510 (past timeout), attacker copies Alice's signature S.
   Attacker builds a new tx: same body but timeout_height=0, same S.
   New sign-doc JSON: {"account_number":"1","chain_id":"ethermint_9000-1",
                       "fee":{...},"memo":"","msgs":[...],"sequence":"0"}
   EIP-712 hash H' = hash(Tx without timeout_height field) = H.

4. Ante handler:
   - TxTimeoutHeightDecorator: timeout_height=0 → pass.
   - SigVerificationDecorator → VerifySignature(new_sign_doc, S):
       ECDSA path: Keccak256(new_sign_doc) ≠ Keccak256(original_sign_doc) → fail.
       EIP-712 path: EIP712Hash(new_sign_doc) = H = H' → pass.
   - Returns true.

5. Tx executes: 100 tokens transferred from Alice to Bob after Alice's intended deadline.
```

func legacyDecodeAminoSignDoc(signDocBytes []byte) (apitypes.TypedData, error) {
	// Ensure codecs have been initialized
	if err := validateCodecInit(); err != nil {
		return apitypes.TypedData{}, err
	}

	var aminoDoc legacytx.StdSignDoc
	if err := aminoCodec.UnmarshalJSON(signDocBytes, &aminoDoc); err != nil {
		return apitypes.TypedData{}, err
	}

	var fees legacytx.StdFee
	if err := aminoCodec.UnmarshalJSON(aminoDoc.Fee, &fees); err != nil {
		return apitypes.TypedData{}, err
	}

	// Validate payload messages
	msgs := make([]sdk.Msg, len(aminoDoc.Msgs))
	for i, jsonMsg := range aminoDoc.Msgs {
		var m sdk.Msg
		if err := aminoCodec.UnmarshalJSON(jsonMsg, &m); err != nil {
			return apitypes.TypedData{}, fmt.Errorf("failed to unmarshal sign doc message: %w", err)
		}
		msgs[i] = m
	}

	if err := LegacyValidatePayloadMessages(msgs); err != nil {
		return apitypes.TypedData{}, err
	}

	// Use first message for fee payer and type inference
	msg := msgs[0]

	// By convention, the fee payer is the first address in the list of signers.
	signers, _, err := protoCodec.GetMsgV1Signers(msg)
	if err != nil {
		return apitypes.TypedData{}, err
	}
	feePayer := signers[0]
	feeDelegation := &FeeDelegationOptions{
		FeePayer: feePayer,
	}

	chainID, err := types.ParseChainID(aminoDoc.ChainID)
	if err != nil {
		return apitypes.TypedData{}, errors.New("invalid chain ID passed as argument")
	}

	typedData, err := LegacyWrapTxToTypedData(
		protoCodec,
		chainID.Uint64(),
		msg,
		signDocBytes,
		feeDelegation,
	)
	if err != nil {
		return apitypes.TypedData{}, fmt.Errorf("could not convert to EIP712 representation: %w", err)
	}

	return typedData, nil
}
