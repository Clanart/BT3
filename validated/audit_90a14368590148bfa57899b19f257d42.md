### Title
Missing Backward-Compatibility Branch in `calculate_global_state_root` Produces Incorrect State Commitment — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/state/commitment.cairo`)

---

### Summary

The `calculate_global_state_root` function in `commitment.cairo` is missing a required conditional branch. Its own specification comment documents **three** distinct cases, but the implementation only handles **two**. When the contract class tree is empty (`contract_class_root == 0`) but the contract state tree is non-empty (`contract_state_root != 0`), the OS computes `H(GLOBAL_STATE_VERSION, contract_state_root, 0)` instead of returning `contract_state_root` directly. This produces an incorrect global state root that will not match what the L1 verifier expects, causing proof rejection and blocking block finalization.

---

### Finding Description

The function comment at the top of `calculate_global_state_root` specifies three cases:

```
// If both the contract class and contract state trees are empty, the global root is set to 0.
// If the contract class tree is empty, the global state root is equal to the
// contract state root (for backward compatibility);
// Otherwise, the global root is obtained by:
//     global_root = H(GLOBAL_STATE_VERSION, contract_state_root, contract_class_root).
```

The actual implementation is:

```cairo
func calculate_global_state_root{poseidon_ptr: PoseidonBuiltin*, range_check_ptr}(
    contract_state_root: felt, contract_class_root: felt
) -> (global_root: felt) {
    if (contract_state_root == 0 and contract_class_root == 0) {
        // The state is empty.
        return (global_root=0);
    }

    tempvar elements: felt* = new (GLOBAL_STATE_VERSION, contract_state_root, contract_class_root);
    let (global_root) = poseidon_hash_many(n=3, elements=elements);
    return (global_root=global_root);
}
``` [1](#0-0) 

The condition `contract_state_root == 0 and contract_class_root == 0` uses a conjunction (`and`) that only catches the fully-empty state. The second specified case — **class tree empty, state tree non-empty** — falls through to the hash computation branch. This is the direct analog of the external report's `||` vs `&&` flaw: the guard condition is too narrow, causing a legitimate intermediate state to be handled by the wrong branch.

The correct implementation requires an additional guard:

```cairo
if (contract_class_root == 0) {
    // Backward compatibility: when class tree is empty, global root == state root.
    return (global_root=contract_state_root);
}
```

This branch is absent. The `and` condition on line 41 should instead be split into two sequential checks. [2](#0-1) 

---

### Impact Explanation

The global state root is the cryptographic commitment to the entire StarkNet state that is verified on L1. When `contract_class_root == 0` (no class declarations have occurred, e.g., a fresh chain or a block with no `declare` transactions) and `contract_state_root != 0` (any invoke or L1-handler transaction has modified storage), the OS outputs `H(GLOBAL_STATE_VERSION, contract_state_root, 0)` as the global root. The L1 verifier, which implements the correct three-case specification, expects `contract_state_root` in this scenario. The mismatch causes proof verification to fail on L1. No block can be finalized until the OS is corrected, constituting a **total network shutdown** — matching the allowed impact: *High. Network not being able to confirm new transactions.*

---

### Likelihood Explanation

The trigger condition (`contract_class_root == 0`, `contract_state_root != 0`) is reached in any block that contains state-modifying transactions (invoke, L1-handler) but no `declare` transactions, **and** where the cumulative class tree across all prior blocks is still empty. This is the normal state of a freshly deployed chain or a devnet/testnet reset. Any unprivileged user submitting a single invoke transaction into such a chain triggers the bug. No special privilege or coordination is required.

---

### Recommendation

Add the missing backward-compatibility branch between the two existing cases:

```cairo
func calculate_global_state_root{poseidon_ptr: PoseidonBuiltin*, range_check_ptr}(
    contract_state_root: felt, contract_class_root: felt
) -> (global_root: felt) {
    if (contract_state_root == 0 and contract_class_root == 0) {
        return (global_root=0);
    }
    // Backward compatibility: if class tree is empty, global root equals state root.
    if (contract_class_root == 0) {
        return (global_root=contract_state_root);
    }
    tempvar elements: felt* = new (GLOBAL_STATE_VERSION, contract_state_root, contract_class_root);
    let (global_root) = poseidon_hash_many(n=3, elements=elements);
    return (global_root=global_root);
}
```

---

### Proof of Concept

1. Deploy a fresh StarkNet OS instance (or reset state so no classes have been declared). At this point `contract_class_root == 0`.
2. Submit any invoke transaction that writes to storage (e.g., calls a pre-deployed system contract). After execution, `contract_state_root != 0`.
3. The OS calls `calculate_global_state_root(contract_state_root=X, contract_class_root=0)` where `X != 0`.
4. The `and` condition on line 41 evaluates to `FALSE` (because `contract_state_root != 0`), so execution falls through.
5. The OS computes and outputs `global_root = H(GLOBAL_STATE_VERSION, X, 0)`.
6. The L1 verifier, following the correct specification, expects `global_root = X`.
7. Verification fails; the block is rejected; no further blocks can be finalized. [3](#0-2)
