### Title
Missing Class Hash Declaration Validation in `execute_replace_class` Enables Permanent Contract Freezing - (File: `execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` function in `syscall_impls.cairo` does not verify that the new class hash supplied in a `ReplaceClassRequest` is actually declared in the protocol. Any contract can therefore replace its own class with an arbitrary, undeclared hash. Once committed, the contract becomes permanently non-callable, freezing any funds it holds.

### Finding Description
In `execute_replace_class`, the OS writes the caller-supplied `class_hash` directly into `contract_state_changes` with no validation. The code itself acknowledges the missing check with a TODO comment:

```cairo
let class_hash = request.class_hash;
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}
tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

When any subsequent call is made to this contract, `execute_entry_point` reads the compiled class hash from `contract_class_changes`:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
``` [2](#0-1) 

Because `contract_class_changes` is initialized via the `InitializeClassHashes` hint with the current on-chain declared classes, an undeclared hash returns the default value `0`. `find_element` is then called with `key=0`:

```cairo
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [3](#0-2) 

No compiled class with hash `0` exists, so `find_element` panics. The contract is permanently non-callable.

### Impact Explanation
**Critical — Permanent freezing of funds.** Any contract that calls `replace_class` with an undeclared hash becomes permanently non-callable. If the contract holds token balances or acts as a vault, those funds are irrecoverably frozen. There is no upgrade or recovery path once the invalid class hash is committed to the global state root.

### Likelihood Explanation
The attack requires no privileged access. A malicious deployer:
1. Deploys a contract that appears legitimate and accepts user deposits.
2. After accumulating funds, calls `replace_class(0)` (or any undeclared felt) from within the contract.
3. The OS accepts the state update without validation.
4. The contract is permanently frozen.

The `replace_class` syscall is reachable by any contract execution, making this an unprivileged, attacker-controlled entry path. The attack is irreversible once the block is proven and the state root is updated.

### Recommendation
Before writing the new class hash to `contract_state_changes`, verify it is present in `contract_class_changes` (i.e., it has been declared):

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=class_hash
);
assert_not_zero(compiled_class_hash);
```

This mirrors the validation already performed implicitly in `execute_entry_point` and makes the check explicit and enforced at the point of the syscall.

### Proof of Concept
1. Attacker deploys a malicious contract (e.g., a fake vault) that accepts deposits.
2. Users deposit funds; the contract accumulates a balance.
3. Attacker triggers `replace_class(0)` from within the contract.
4. `execute_replace_class` (line 895–915 of `syscall_impls.cairo`) writes `class_hash=0` to `contract_state_changes` with no validation.
5. The block is proven; the state root now reflects the contract with `class_hash=0`.
6. Any future call to the contract reaches `execute_entry_point`, reads `compiled_class_hash=0` from `contract_class_changes`, and `find_element` panics.
7. All funds in the contract are permanently frozen with no recovery mechanism.
