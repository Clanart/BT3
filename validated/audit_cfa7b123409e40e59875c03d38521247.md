### Title
Low-Level ETH Call to `address(0)` Silently Succeeds, Permanently Burning User Funds - (File: `evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

In `OmniBridge.finTransfer()`, when bridging native ETH (i.e., `payload.tokenAddress == address(0)`), the contract performs a low-level `.call{value: amount}("")` to `payload.recipient` with no zero-address guard. A call to `address(0)` returns `success = true` in Solidity, causing the ETH to be permanently burned. Because the destination nonce is marked consumed **before** the transfer executes, the transfer is irrecoverably finalized with no retry path.

---

### Finding Description

In `OmniBridge.sol`, `finTransfer()` handles native ETH delivery via a raw low-level call:

```solidity
// OmniBridge.sol line 287
completedTransfers[payload.destinationNonce] = true;

// ...signature verification...

// line 317-322
if (payload.tokenAddress == address(0)) {
    (bool success, ) = payload.recipient.call{value: payload.amount}("");
    if (!success) revert FailedToSendEther();
}
```

The `TransferMessagePayload.recipient` field is typed as `address` with no zero-address validation anywhere in `finTransfer()` or in the `TransferMessagePayload` struct definition. [1](#0-0) 

The nonce is consumed at line 287 before the transfer, so any outcome — including a silent success to `address(0)` — permanently finalizes the nonce. [2](#0-1) 

The low-level call at lines 319–322 has no guard against `payload.recipient == address(0)`: [3](#0-2) 

In Solidity, `address(0).call{value: N}("")` returns `(true, "")` — the ETH is burned and the call is considered successful. The `if (!success) revert FailedToSendEther()` guard does **not** catch this case.

---

### Impact Explanation

**Critical — Permanent, irrecoverable loss of user funds.**

When a user initiates a NEAR → EVM native ETH bridge transfer specifying `address(0)` as the EVM recipient:

1. The MPC signs a `TransferMessagePayload` with `recipient = address(0)`.
2. `finTransfer()` is called; `completedTransfers[nonce] = true` is set immediately.
3. `address(0).call{value: amount}("")` executes and returns `success = true`.
4. The ETH is burned at `address(0)`.
5. The nonce is permanently consumed — no retry, no refund, no recovery path exists.

This matches the allowed impact: **"Permanent freezing, irrecoverable lock, or unclaimable settlement of user or protocol funds in bridge, token, or vault flows."** [4](#0-3) 

---

### Likelihood Explanation

**Medium.** The recipient address is user-supplied on the NEAR side as a free-form string (e.g., `"evm:0x0000000000000000000000000000000000000000"`). Neither the NEAR bridge nor the EVM `finTransfer()` validates that the decoded EVM address is non-zero. A UI bug, copy-paste error, or a malicious relayer constructing a valid MPC-signed payload with `recipient = address(0)` can trigger this. The MPC signs whatever recipient is in the transfer intent — it does not enforce EVM address validity. [5](#0-4) 

---

### Recommendation

Add an explicit zero-address guard at the top of `finTransfer()` before the nonce is consumed:

```solidity
if (payload.recipient == address(0)) revert InvalidRecipient();
```

Additionally, consider moving `completedTransfers[payload.destinationNonce] = true` to **after** all transfer logic succeeds (or use a checks-effects-interactions pattern with a reentrancy guard), so that a failed or invalid transfer does not permanently consume the nonce. [6](#0-5) 

---

### Proof of Concept

1. Alice initiates a NEAR → EVM bridge transfer of 1 ETH, accidentally entering `0x0000000000000000000000000000000000000000` as the EVM recipient.
2. The NEAR MPC signs a `TransferMessagePayload` with `recipient = address(0)`, `tokenAddress = address(0)`, `amount = 1e18`.
3. A relayer calls `OmniBridge.finTransfer(signatureData, payload)`.
4. Line 287: `completedTransfers[nonce] = true` — nonce is consumed.
5. Line 311: `ECDSA.recover(hashed, signatureData) == nearBridgeDerivedAddress` — signature is valid.
6. Line 319: `address(0).call{value: 1e18}("")` — returns `(true, "")`.
7. Line 322: `if (!success)` is false — no revert.
8. 1 ETH is permanently burned at `address(0)`. The nonce is consumed. Alice has no recourse. [3](#0-2)
