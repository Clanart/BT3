### Title
Fee Charging Silently Skipped When All Resource Bound Prices Are Zero, Allowing Fee-Free Execution — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

The `charge_fee` function in the StarkNet OS unconditionally skips all fee collection when `compute_max_possible_fee` returns `0`. Because `compute_max_possible_fee` sums products of user-supplied `max_price_per_unit` values, a user can set all three prices to `0` while keeping `L2_GAS_INDEX.max_amount` non-zero. The OS then grants the transaction a full L2 gas execution budget, runs both `__validate__` and `__execute__`, and silently returns from `charge_fee` without transferring any tokens. The resulting proof is valid, so the sequencer cannot detect or reject the block after the fact.

---

### Finding Description

**Vulnerability class:** fee/accounting bug — fees are computed but the code path never charges them under an attacker-controlled condition.

**Root cause — `compute_max_possible_fee`:** [1](#0-0) 

The function returns the felt:

```
l1_gas.max_amount * l1_gas.max_price_per_unit
+ l2_gas.max_amount * (l2_gas.max_price_per_unit + tip)
+ l1_data_gas.max_amount * l1_data_gas.max_price_per_unit
```

Every term is a product. If the user sets all three `max_price_per_unit` fields to `0` and `tip = 0`, the entire expression evaluates to `0` regardless of `l2_gas.max_amount`.

**Root cause — `charge_fee` early-return:** [2](#0-1) 

When `max_fee == 0`, the function returns immediately. No ERC-20 transfer is initiated, no state change is written, and no assertion forces a non-zero payment.

**Root cause — gas budget is independent of price:** [3](#0-2) 

`get_initial_user_gas_bound` returns only `resource_bounds[L2_GAS_INDEX].max_amount`. It does not inspect any price field. A transaction with `l2_gas.max_amount = 1_100_000_000` (the `EXECUTE_MAX_SIERRA_GAS` cap) and all prices `= 0` receives the maximum execution budget.

**Execution flow confirming the bypass (invoke path):** [4](#0-3) 

The sequence is:
1. `initial_user_gas_bound = get_initial_user_gas_bound(...)` — non-zero because `l2_gas.max_amount > 0`.
2. `run_validate(...)` — executes `__validate__` consuming gas.
3. `non_reverting_select_execute_entry_point_func(...)` — executes `__execute__` consuming gas.
4. `charge_fee(...)` — computes `max_fee = 0`, returns immediately.

The same pattern applies to `execute_deploy_account_transaction` (line 687) and `execute_declare_transaction` (line 822). [5](#0-4) [6](#0-5) 

---

### Impact Explanation

**Impact: Critical — Direct loss of funds.**

An attacker can execute arbitrary contract logic (up to `EXECUTE_MAX_SIERRA_GAS = 1_100_000_000` gas units per transaction) without paying any fee. The OS proof produced for such a block is cryptographically valid; the L1 verifier accepts it. The sequencer's fee revenue is permanently zero for every such transaction. At scale, an attacker can drain sequencer resources (CPU, storage writes, L1 DA costs) while paying nothing, constituting a direct and permanent loss of funds for the protocol.

---

### Likelihood Explanation

The attacker controls all three `max_price_per_unit` fields and `tip` directly in the signed transaction body. Setting them to `0` is a trivial, one-line change to any SDK call. The OS imposes no lower-bound constraint on prices anywhere in the Cairo program. The only external gate is the sequencer's mempool policy; however:

- The OS proof is valid regardless of mempool policy, so a sequencer that is buggy, misconfigured, or running a modified client will include such transactions and produce an accepted proof.
- The OS is the final enforcement layer for the StarkNet protocol. Relying solely on off-chain mempool filtering for a critical invariant (fees must be non-zero) is an insufficient guarantee.

---

### Recommendation

Add an explicit lower-bound check inside `charge_fee` (or inside `compute_max_possible_fee`) that asserts `max_fee > 0` whenever `initial_user_gas_bound > 0`. Concretely, before the early-return branch, assert:

```cairo
// If the user requested any execution gas, they must have set a non-zero price.
if (initial_user_gas_bound != 0) {
    assert_not_zero(max_fee);
}
```

Alternatively, enforce in `get_account_tx_common_fields` that at least one `max_price_per_unit` is non-zero whenever `l2_gas.max_amount > 0`, mirroring the intent of the fee system.

---

### Proof of Concept

A malicious user constructs and signs a V3 `INVOKE_FUNCTION` transaction with the following resource bounds:

```json
{
  "resource_bounds": {
    "l1_gas":      { "max_amount": "0x0", "max_price_per_unit": "0x0" },
    "l2_gas":      { "max_amount": "0x41A4C380", "max_price_per_unit": "0x0" },
    "l1_data_gas": { "max_amount": "0x0", "max_price_per_unit": "0x0" }
  },
  "tip": "0x0"
}
```

(`0x41A4C380 = 1_100_000_000 = EXECUTE_MAX_SIERRA_GAS`)

**OS execution trace:**

1. `get_initial_user_gas_bound` → returns `1_100_000_000` (non-zero). [3](#0-2) 
2. `run_validate` and `non_reverting_select_execute_entry_point_func` execute normally with that gas budget. [7](#0-6) 
3. `charge_fee` is called. `compute_max_possible_fee` computes `0 * 0 + 1_100_000_000 * (0 + 0) + 0 * 0 = 0`. [8](#0-7) 
4. `if (max_fee == 0) { return (); }` — fee charging is skipped entirely. [9](#0-8) 
5. The OS proof is generated and accepted by the L1 verifier. The sequencer receives `0` tokens.

```text
func compute_max_possible_fee(tx_info: TxInfo*) -> felt {
    tempvar resource_bounds: ResourceBounds* = tx_info.resource_bounds_start;
    let n_resource_bounds = (tx_info.resource_bounds_end - resource_bounds) / ResourceBounds.SIZE;

    // Only V3 transactions with all resource bounds are supported.
    assert tx_info.version = 3;
    assert n_resource_bounds = 3;

    tempvar l1_gas_bounds: ResourceBounds = resource_bounds[L1_GAS_INDEX];
    tempvar l2_gas_bounds: ResourceBounds = resource_bounds[L2_GAS_INDEX];
    tempvar l1_data_gas_bounds = resource_bounds[L1_DATA_GAS_INDEX];

    return l1_gas_bounds.max_amount * l1_gas_bounds.max_price_per_unit + l2_gas_bounds.max_amount *
        (l2_gas_bounds.max_price_per_unit + tx_info.tip) + l1_data_gas_bounds.max_amount *
        l1_data_gas_bounds.max_price_per_unit;
}
```
