### Title
Non-EVM-native Token Balances Not Cleared During SelfDestruct Enables Orphaned-Balance Theft via CREATE2 Resurrection — (`x/evm/statedb/statedb.go`)

---

### Summary

When a contract self-destructs, `Commit()` in `x/evm/statedb/statedb.go` burns only the EVM-denom balance. Non-EVM-native tokens (IBC, CosmWasm bridge) held by the destroyed address are never drained from the Cosmos bank module. Because the auth account is deleted but the bank balances are not, an attacker can recreate the contract at the same address via CREATE2 and claim the orphaned Cosmos bank funds.

---

### Finding Description

In `x/evm/statedb/statedb.go`, the `Commit()` function handles the selfdestruct path as follows:

```go
// x/evm/statedb/statedb.go  lines 800-825
if obj.selfDestructed {
    cosmosAddr := sdk.AccAddress(obj.Address().Bytes())
    cacheCtx, writeCache := s.origCtx.CacheContext()
    // Only the EVM denom is burned here. Non-EVM-native tokens (IBC, CosmWasm
    // bridge) held by the destroyed address are not drained and may remain as
    // orphaned bank balances.
    if remaining := s.keeper.GetBalance(cacheCtx, cosmosAddr, s.evmDenom); remaining.Sign() > 0 {
        coin := sdk.NewCoin(s.evmDenom, sdkmath.NewIntFromBigInt(remaining.ToBig()))
        if _, err := s.keeper.SubBalance(cacheCtx, cosmosAddr, coin); err != nil {
            return errorsmod.Wrap(err, "failed to burn post-selfdestruct balance")
        }
    }
    if err := s.keeper.DeleteAccount(cacheCtx, obj.Address()); err != nil {
        return errorsmod.Wrap(err, "failed to delete account")
    }
    writeCache()
```

`DeleteAccount` in `x/evm/keeper/statedb.go` removes code, storage, and the auth account, but explicitly does **not** touch the bank module:

```go
// x/evm/keeper/statedb.go  line 194
// NOTE: balance should be cleared separately
func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
    ...
    k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool { ... })
    k.accountKeeper.RemoveAccount(ctx, acct)
    ...
}
```

The Cosmos SDK bank module stores balances keyed by address independently of the auth module. Removing the auth account does **not** remove bank-module balances for non-EVM-native denominations. After selfdestruct, those balances remain in the bank store under the now-deleted address.

Because CREATE2 produces a deterministic address from `(deployer, salt, initcode)`, an attacker who controls those parameters can recreate a contract at the exact same address. When the new account is registered in the auth module at that address, it inherits the orphaned bank balances that survived the selfdestruct.

This is the direct analog of the Notional H-4 pattern: a "full exit" (selfdestruct) validates and clears the primary obligation (EVM-denom balance) but silently skips the secondary obligation (non-EVM-native bank balances), trusting that they are already gone — when they are not.

---

### Impact Explanation

An attacker can steal Cosmos bank funds (IBC tokens, bridged assets) that were deposited into a contract before it self-destructed. The bank module retains those balances after the auth account is deleted; a CREATE2 resurrection at the same address gives the new contract full ownership of the orphaned funds. This is an unauthorized transfer of Cosmos bank funds through Ethermint stateDB/native action logic — matching the Critical impact tier.

---

### Likelihood Explanation

The entry path is fully unprivileged: any user can deploy a contract via CREATE2, arrange for it to receive IBC tokens (e.g., by being a bridge recipient or a DeFi vault), trigger selfdestruct, and then redeploy at the same address. No governance, validator cooperation, or key compromise is required. The only prerequisite is that the contract holds non-EVM-native tokens at the time of selfdestruct, which is a realistic condition for any IBC-integrated application on an Ethermint chain.

---

### Recommendation

Before calling `DeleteAccount`, enumerate and burn (or transfer to a designated recovery address) all non-EVM-native bank balances held by the destroyed address. Concretely, inside the `obj.selfDestructed` branch of `Commit()`, after burning the EVM-denom balance, iterate over all denominations held by `cosmosAddr` via `bankKeeper.GetAllBalances` and drain each one before calling `DeleteAccount`. This mirrors the complete-exit check recommended in the Notional report: verify (and enforce) that **all** obligations — not just the primary one — are cleared before the account is removed.

---

### Proof of Concept

1. Attacker deploys contract `C` at address `X` using CREATE2 (known `salt` + `initcode`).
2. A user or protocol sends IBC-USDC to address `X` (e.g., as part of a bridge deposit or DeFi interaction).
3. Attacker calls a function on `C` that triggers `SELFDESTRUCT`.
4. `Commit()` burns the EVM-denom balance at `X` and calls `DeleteAccount(X)`, removing the auth account and storage. IBC-USDC balance remains in the bank module under `X`.
5. Attacker redeploys `C` at address `X` using the same CREATE2 parameters. A fresh auth account is registered at `X`.
6. The new contract at `X` now owns the orphaned IBC-USDC balance. Attacker calls a bank precompile (or any native-action path) to transfer the funds out.

```go
// DeleteAccount handles contract's suicide call:
// - remove code
// - remove states
// - remove auth account
//
// NOTE: balance should be cleared separately
func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
	cosmosAddr := sdk.AccAddress(addr.Bytes())
	acct := k.accountKeeper.GetAccount(ctx, cosmosAddr)
	if acct == nil {
		return nil
	}

	// NOTE: only Ethereum accounts (contracts) can be selfdestructed
	_, ok := acct.(ethermint.EthAccountI)
	if !ok {
		return errorsmod.Wrapf(types.ErrInvalidAccount, "type %T, address %s", acct, addr)
	}

	// clear storage
	k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool {
		k.SetState(ctx, addr, key, nil)
		return true
	})

	// remove auth account
	k.accountKeeper.RemoveAccount(ctx, acct)

	k.debugLog(ctx, "account suicided",
		"ethereum-address", addr,
		"cosmos-address", cosmosAddr,
	)

	return nil
```
