### Title
Missing Declared Class Hash Validation in `execute_replace_class` Enables Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the new class hash supplied by a contract is actually a declared class. This is explicitly acknowledged by a TODO comment in the code. Because the OS accepts any arbitrary felt value as a replacement class hash without checking its existence in `contract_class_changes`, a contract can replace its own class hash with an undeclared value. Once committed, any subsequent transaction targeting that contract will cause the OS proof to fail (because `find_element` on the compiled class facts will not find the hash), permanently preventing execution and freezing all funds held in the contract.

---

### Finding Description

In `execute_replace_class` (lines 877–916 of `syscall_impls.cairo`), the OS reads `request.class_hash` directly from the syscall request and writes it into `contract_state_changes` without any check that the hash corresponds to a previously declared class:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The `REPLACE_CLASS_SELECTOR` branch in `execute_syscalls` dispatches to this function for any executing contract with no additional access control: [2](#0-1) 

When a subsequent transaction calls the now-replaced contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [3](#0-2) 

`find_element` is a Cairo assertion-based lookup; if the key is absent (because the class was never declared), the proof cannot be generated. The sequencer is therefore forced to exclude all future transactions targeting that contract, permanently freezing it.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value via `replace_class`, no transaction calling that contract can ever be proven by the OS. Any ERC-20 tokens, ETH, or other assets held in the contract's storage become permanently inaccessible. There is no recovery path: the state transition has been committed and proven valid by the OS, but the contract can never be executed again.

---

### Likelihood Explanation

The `replace_class` syscall is available to every Cairo 1 contract with no protocol-level access control beyond gas. An attacker who deploys a contract (a standard, permissionless action) can include a `replace_class` call targeting an arbitrary undeclared felt value. The OS will prove the block containing this call as valid. The attack requires no privileged role, no leaked key, and no operator cooperation — only the ability to deploy and call a contract, which any network participant can do.

---

### Recommendation

Inside `execute_replace_class`, before writing the new class hash to `contract_state_changes`, add a lookup into `contract_class_changes` (or the compiled class facts bundle) to assert that `class_hash` maps to a non-zero compiled class hash. Concretely, perform a `dict_read` on `contract_class_changes` for the requested `class_hash` and assert the result is non-zero, mirroring the check already performed at the start of `execute_entry_point`. This resolves the acknowledged TODO and closes the missing validation gap.

---

### Proof of Concept

1. Attacker deploys contract `C` (e.g., a token vault holding user deposits) whose `__execute__` entry point issues a `replace_class` syscall with `class_hash = 0xdeadbeef` (an undeclared felt).
2. Attacker (or any caller) sends an invoke transaction to `C`.
3. The OS processes the block: `execute_replace_class` writes `class_hash=0xdeadbeef` into `contract_state_changes` for address `C` with no validation.
4. The block is proven and accepted on-chain. `C`'s state entry now carries `class_hash=0xdeadbeef`.
5. Any subsequent invoke transaction targeting `C` reaches `execute_entry_point`, which calls `dict_read(key=0xdeadbeef)` → returns 0, then `find_element(..., key=0)` → no compiled class found → proof generation fails.
6. The sequencer cannot include any transaction calling `C`. All funds in `C` are permanently frozen.
