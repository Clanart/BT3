### Title
Silent Omission of Deprecated Class Declarations from `ThinStateDiff` Produces Wrong State-Diff Commitment and Block Hash - (File: `crates/blockifier/src/state/cached_state.rs`)

### Summary

The `From<CommitmentStateDiff> for ThinStateDiff` conversion unconditionally sets `deprecated_declared_classes: Vec::new()`, silently dropping every Cairo 0 (deprecated) class declaration from the state diff that feeds into `calculate_state_diff_hash`, `state_diff.len()`, and ultimately the block hash. Any block that includes a `DeprecatedDeclare` transaction will produce a wrong `state_diff_commitment`, a wrong `state_diff_length` inside `concatenated_counts`, and therefore a wrong block hash.

### Finding Description

**Root cause — the silent drop**

`CommitmentStateDiff` has no `deprecated_declared_classes` field: [1](#0-0) 

When the blockifier converts it to `ThinStateDiff`, the field is hard-coded to an empty `Vec`: [2](#0-1) 

The `TODO(AlonH)` comment acknowledges this is a known gap, but the gap has concrete commitment consequences.

**How the dropped data reaches the block hash**

`BlockExecutionArtifacts::new` calls `ThinStateDiff::from(commitment_state_diff.clone())` and immediately passes the result to `calculate_block_commitments`: [3](#0-2) 

Inside `calculate_block_commitments`, two values are derived from the (now-truncated) `ThinStateDiff`:

1. `state_diff_commitment` — computed by `calculate_state_diff_hash`, which explicitly chains `deprecated_declared_classes` into the Poseidon hash: [4](#0-3) 

2. `concatenated_counts` — computed by `concat_counts(…, state_diff.len(), …)`, where `ThinStateDiff::len()` counts `deprecated_declared_classes`: [5](#0-4) [6](#0-5) 

Both `state_diff_commitment` and `concatenated_counts` are chained directly into the block hash: [7](#0-6) 

**Secondary impact — P2P sync accepts truncated state diffs**

The `state_diff_length` stored in the block header (derived from the same truncated `ThinStateDiff.len()`) is used by P2P sync as the authoritative target length. A syncing node will stop consuming state-diff chunks as soon as `current_state_diff_len == target_state_diff_len`, accepting a state diff that is missing the deprecated class entries: [8](#0-7) 

**Secondary impact — `deprecated_declared_classes_block` table not updated**

`append_state_diff` iterates over `thin_state_diff.deprecated_declared_classes` to record which block first declared each Cairo 0 class. Because the list is always empty, the `deprecated_declared_classes_block` table is never updated for sequencer-produced blocks, breaking `get_deprecated_class_definition_block_number` and any logic that depends on it: [9](#0-8) 

### Impact Explanation

For every block that contains at least one `DeprecatedDeclare` transaction:

- `state_diff_commitment` is wrong → the block hash is wrong.
- `state_diff_length` inside `concatenated_counts` is wrong → the block hash is wrong a second time.
- P2P-synced nodes accept a state diff that is missing the deprecated class declarations, diverging from the canonical state.
- RPC `starknet_getStateUpdate` returns a `ThinStateDiff` without the declared class hashes, giving callers an authoritative-looking wrong value.

This matches **Critical: Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input** and **High: RPC execution … returns an authoritative-looking wrong value**.

### Likelihood Explanation

Cairo 0 (`DeprecatedDeclare`) transactions are still valid on Starknet and are accepted by the gateway. Any user can submit one. The trigger requires no privilege and no special network condition. The only mitigating factor is that such transactions are rare in practice on modern Starknet.

### Recommendation

Populate `deprecated_declared_classes` in `CommitmentStateDiff` (add the field, sourced from `StateMaps.declared_contracts` filtered to Cairo 0 classes), and propagate it through the `From<CommitmentStateDiff> for ThinStateDiff` conversion. Remove the `Vec::new()` placeholder and the associated TODO. Add a test that executes a `DeprecatedDeclare` transaction and asserts that the resulting `ThinStateDiff.deprecated_declared_classes` is non-empty and that the computed `state_diff_commitment` matches an independently calculated reference value.

### Proof of Concept

1. Submit a `DeprecatedDeclare` transaction (Cairo 0 class) to the sequencer.
2. The batcher includes it in a block; the blockifier executes it and records the class hash in `StateMaps.declared_contracts`.
3. `CommitmentStateDiff::from(StateMaps)` drops `declared_contracts`; `CommitmentStateDiff` has no `deprecated_declared_classes` field.
4. `ThinStateDiff::from(commitment_state_diff)` sets `deprecated_declared_classes: Vec::new()`.
5. `calculate_state_diff_hash(&thin_state_diff)` produces hash H₁ (missing the class hash).
6. Independently compute `calculate_state_diff_hash` on the same diff but with `deprecated_declared_classes = vec![class_hash]`; obtain H₂.
7. Assert H₁ ≠ H₂ — the sequencer's committed `state_diff_commitment` is wrong.
8. Observe that `thin_state_diff.len()` is N, but the correct length is N+1, so `concatenated_counts` encodes the wrong `state_diff_length`, producing a wrong block hash. [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
