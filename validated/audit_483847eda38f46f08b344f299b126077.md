### Title
Meta-Transaction Signature Replay Due to Missing Nonce in `compute_meta_tx_v0_hash` — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/transaction_hash/transaction_hash.cairo`)

---

### Summary

The `execute_meta_tx_v0` syscall in the StarkNet OS constructs a version-0 transaction context with a hardcoded `nonce=0` and computes its hash via `compute_meta_tx_v0_hash`, which passes `additional_data_size=0` — meaning **no nonce is included in the hash**. The OS also explicitly skips nonce checking for version-0 transactions. As a result, any valid meta-tx-v0 signature can be replayed an unlimited number of times by any caller, leading to direct loss of funds from the target account contract.

---

### Finding Description

`execute_meta_tx_v0` in `execution/syscall_impls.cairo` is a syscall callable by any executing contract. It constructs a synthetic version-0 `TxInfo` with `nonce=0` hardcoded:

```cairo
tempvar new_tx_info = new TxInfo(
    version=0,
    ...
    nonce=0,          // hardcoded — never incremented
    ...
);
```

The transaction hash is computed by `compute_meta_tx_v0_hash` in `transaction_hash/transaction_hash.cairo`:

```cairo
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(
    contract_address: felt,
    entry_point_selector: felt,
    calldata: felt*,
    calldata_size: felt,
    chain_id: felt,
) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        ...
        additional_data_size=0,           // NO nonce in hash
        additional_data=cast(0, felt*),
    );
    return tx_hash;
}
```

The hash is therefore a pure function of `(INVOKE_HASH_PREFIX, 0, contract_address, selector, calldata_hash, 0, chain_id)` — **identical for every replay of the same call**.

Additionally, `check_and_increment_nonce` in `execute_transaction_utils.cairo` explicitly skips nonce enforcement for version-0:

```cairo
// Do not handle nonce for version 0.
if (tx_info.version == 0) {
    return ();
}
```

And `run_validate` also skips `__validate__` for version-0:

```cairo
// Do not run "__validate__" for version 0.
if (tx_execution_info.tx_info.version == 0) {
    return ();
}
```

The OS then directly calls `__execute__` on the target contract via `contract_call_helper`. The target contract's `__execute__` receives `tx_info.transaction_hash` as the message to verify the signature against. Since this hash is nonce-free and static for a given `(contract_address, calldata)` pair, any observed valid signature can be replayed indefinitely.

---

### Impact Explanation

**Critical — Direct loss of funds.**

A Cairo 0 account contract's `__execute__` entry point verifies the user's signature against `tx_info.transaction_hash`. Because the OS-level hash for meta-tx-v0 contains no nonce, the same `(contract_address, calldata, signature)` triple always produces the same hash and always passes signature verification. An attacker who observes a single valid meta-tx-v0 invocation (e.g., a token transfer) can replay it an unlimited number of times, draining the target account's funds.

---

### Likelihood Explanation

**High.** The `meta_tx_v0` syscall is a public protocol feature callable by any executing contract. Any on-chain observer of a valid meta-tx-v0 invocation (which is visible in the transaction trace) can immediately construct a replay. No privileged access, leaked key, or off-chain coordination is required. The attacker only needs to submit a new transaction that calls a contract invoking `meta_tx_v0` with the same parameters.

---

### Recommendation

Include a nonce in the meta-tx-v0 hash computation. Specifically:

1. In `compute_meta_tx_v0_hash`, pass `additional_data_size=1` and `additional_data=&nonce` (analogous to how `compute_l1_handler_transaction_hash` includes the L1 handler nonce).
2. In `execute_meta_tx_v0`, accept a nonce from the caller request, include it in the hash, and enforce that the target contract's on-chain nonce matches and is incremented — or maintain a separate per-contract meta-tx nonce mapping.
3. Alternatively, enforce that the OS-level nonce check is not bypassed for meta-tx-v0 by removing the version-0 exemption in `check_and_increment_nonce` for this code path.

---

### Proof of Concept

1. Alice signs a meta-tx-v0 message authorizing a transfer of 100 tokens. The hash is `H(INVOKE_HASH_PREFIX, 0, alice_addr, __execute__, calldata_hash, 0, chain_id)`.
2. A relayer submits a transaction calling a contract that invokes `meta_tx_v0` with Alice's signature and calldata. The transfer executes. Alice's nonce is **not incremented** (version-0 exemption).
3. An attacker observes the signature from the transaction trace.
4. The attacker submits a new transaction calling any contract that invokes `meta_tx_v0` with the identical `(contract_address, selector, calldata, signature)`.
5. `compute_meta_tx_v0_hash` produces the identical hash. Alice's `__execute__` verifies the signature successfully. The transfer executes again.
6. Steps 4–5 repeat until Alice's account is drained.
