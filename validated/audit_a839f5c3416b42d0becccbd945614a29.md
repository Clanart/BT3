### Title
Unchecked Return Value of Fee Transfer Call Allows Fee-Free Transaction Execution — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

### Summary
The `charge_fee` function in the StarkNet OS calls `non_reverting_select_execute_entry_point_func` to execute an ERC20 `transfer` on the fee token contract, but completely discards the return value. If the fee transfer fails (reverts internally or returns `false`), the OS does not detect the failure and generates a valid proof for a block where fees were never actually paid. This is the direct Cairo/StarkNet analog of the unchecked `transferFrom` return value vulnerability.

### Finding Description
In `charge_fee` (`transaction_impls.cairo`, lines 160–163), the OS constructs an `ExecutionContext` targeting the fee token contract's `TRANSFER_ENTRY_POINT_SELECTOR` and calls `non_reverting_select_execute_entry_point_func`. The function returns `(retdata_size, retdata, is_deprecated)` — where `retdata` contains the ERC20 `transfer` success boolean — but the call site discards all return values:

```cairo
let remaining_gas = DEFAULT_INITIAL_GAS_COST;
non_reverting_select_execute_entry_point_func{remaining_gas=remaining_gas}(
    block_context=block_context, execution_context=&execution_context
);
return ();
``` [1](#0-0) 

Contrast this with every other call site of `non_reverting_select_execute_entry_point_func` in the same file, where the return value is captured and validated:

```cairo
let (retdata_size, retdata, is_deprecated) = non_reverting_select_execute_entry_point_func(
    block_context=block_context, execution_context=validate_deploy_execution_context
);
``` [2](#0-1) 

The `non_reverting_select_execute_entry_point_func` is specifically designed not to panic on inner execution failure — it returns the failure state in its return values. Because `charge_fee` ignores those values, a failed fee transfer is indistinguishable from a successful one at the OS level.

The fee transfer is constructed as:

```cairo
local calldata: TransferCallData = TransferCallData(
    recipient=block_context.block_info_for_execute.sequencer_address,
    amount=Uint256(low=low_actual_fee, high=0),
);
``` [3](#0-2)

### Impact Explanation
The OS is the source of truth for proof generation. If `charge_fee` does not assert that the ERC20 transfer succeeded, the STARK proof produced for the block is valid even when no fee was transferred to the sequencer. The L1 verifier accepts the proof, the block is finalized on-chain, and the sequencer receives no fee for the work performed. This constitutes **direct loss of funds** for the sequencer/protocol.

### Likelihood Explanation
An unprivileged transaction sender can trigger this by draining their fee token balance during the `__execute__` phase of their invoke transaction (e.g., by calling `transfer` to send all tokens to another address they control). Because `charge_fee` is called after execution completes, the account balance is zero when the fee transfer is attempted. The transfer fails silently, the OS generates a valid proof, and the attacker has executed a transaction at zero cost. This attack requires no privileged access — only the ability to submit a standard invoke transaction.

### Recommendation
Capture and assert the return value of `non_reverting_select_execute_entry_point_func` in `charge_fee`, verifying that:
1. The execution was not reverted (`is_reverted == 0`), and
2. The ERC20 `transfer` return data indicates success (`retdata[0] == TRUE`).

If the transfer fails, the OS should treat the transaction as invalid or halt proof generation for that block, consistent with how validation failures are handled elsewhere in the OS.

### Proof of Concept

1. Attacker deploys an account contract whose `__execute__` function, when called, transfers the entire fee token balance to an attacker-controlled address.
2. Attacker submits an invoke transaction with sufficient fee token balance to pass the sequencer's pre-inclusion balance check.
3. The OS executes the transaction: `__execute__` drains the fee token balance.
4. The OS calls `charge_fee`, which calls `non_reverting_select_execute_entry_point_func` to transfer the fee. The ERC20 `transfer` fails (zero balance) and returns `false` or reverts internally.
5. `charge_fee` discards the return value and returns normally.
6. The OS produces a valid STARK proof for the block.
7. The proof is verified and accepted on L1. The sequencer receives no fee. The attacker has executed a transaction for free. [5](#0-4)

func charge_fee{
    range_check_ptr,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, tx_execution_context: ExecutionContext*) {
    alloc_locals;

    local tx_info: TxInfo* = tx_execution_context.execution_info.tx_info;
    let max_fee = compute_max_possible_fee(tx_info=tx_info);

    if (max_fee == 0) {
        return ();
    }

    local low_actual_fee;
    %{ LoadActualFee %}
    local calldata: TransferCallData = TransferCallData(
        recipient=block_context.block_info_for_execute.sequencer_address,
        amount=Uint256(low=low_actual_fee, high=0),
    );

    // Verify that the charged amount is not larger than the transaction's max_fee field.
    assert_nn_le(calldata.amount.low, max_fee);

    // TODO(ilya, 01/01/2026): Consider caching the fee_token_class_hash.
    local fee_token_address = block_context.os_global_context.starknet_os_config.fee_token_address;
    let (fee_state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
        key=fee_token_address
    );
    let (__fp__, _) = get_fp_and_pc();
    // Use block_info directly from block_context, so that charge_fee will always run in
    // execute-mode rather than validate-mode.
    local execution_context: ExecutionContext = ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=fee_state_entry.class_hash,
        calldata_size=TransferCallData.SIZE,
        calldata=&calldata,
        execution_info=new ExecutionInfo(
            block_info=block_context.block_info_for_execute,
            tx_info=tx_info,
            caller_address=tx_info.account_contract_address,
            contract_address=fee_token_address,
            selector=TRANSFER_ENTRY_POINT_SELECTOR,
        ),
        deprecated_tx_info=tx_execution_context.deprecated_tx_info,
    );

    let remaining_gas = DEFAULT_INITIAL_GAS_COST;
    non_reverting_select_execute_entry_point_func{remaining_gas=remaining_gas}(
        block_context=block_context, execution_context=&execution_context
    );
    return ();
}
