### Title
Missing Class Hash Existence Validation in `execute_replace_class` Allows Permanent Contract Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` syscall in the StarkNet OS accepts an arbitrary `class_hash` from the caller and writes it directly to the contract state without verifying that the hash corresponds to a previously declared contract class. This is structurally analogous to the reported bug (calling into a non-contract address without validation): just as the EVM delegatecall silently succeeds on a non-contract address, `replace_class` silently succeeds with an undeclared class hash. The result is that the contract's class hash in the global state is permanently set to a hash that has no corresponding compiled class, making the contract permanently uncallable and freezing any funds it holds.

### Finding Description
In `execute_replace_class` (`syscall_impls.cairo`, line 877–916), the OS reads `request.class_hash` and immediately writes it into `contract_state_changes` with no check that the hash exists in `contract_class_changes` (the declared-class registry):

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-acknowledged TODO at line 898 confirms the missing check. The same omission exists in the deprecated path at `deprecated_execute_syscalls.cairo` lines 307–329.

When any future transaction attempts to call the affected contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
```

For an undeclared class hash, `dict_read` returns the default value `0` (`UNINITIALIZED_CLASS_HASH`). The subsequent `find_element` call then searches for a compiled class fact with hash `0`, which does not exist, causing the proof to be unprovable. No valid block can ever include a successful call to the frozen contract.

### Impact Explanation
**Critical — Permanent freezing of funds.**

Once `replace_class` is called with an undeclared class hash and the transaction is included in a block, the contract's `class_hash` field in the global state Merkle tree is permanently set to an undeclared value. Every subsequent attempt to call the contract will fail at the proof-generation stage (unprovable `find_element` assertion), making the contract permanently uncallable. Any ERC-20 balances, ETH, or other assets held by the contract are irreversibly locked with no recovery path.

### Likelihood Explanation
Any account that owns or controls a deployed contract can issue a `replace_class` syscall from within that contract. No privileged role, leaked key, or external dependency is required. The attacker only needs to craft and submit a transaction that invokes their contract's `replace_class` with an arbitrary undeclared felt value. The OS will accept and finalize the state change without error.

### Recommendation
Before writing the new class hash to `contract_state_changes`, assert that the hash exists as a key in `contract_class_changes` (i.e., has been declared). The check should mirror the pattern used in `execute_declare_transaction`, which enforces `prev_value=0` to guarantee a class is declared exactly once:

```cairo
let class_hash = request.class_hash;

// Verify the class hash has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);  // 0 == UNINITIALIZED_CLASS_HASH means undeclared.
```

The same fix must be applied to `execute_replace_class` in `deprecated_execute_syscalls.cairo`.

### Proof of Concept

1. Attacker deploys contract `C` holding user funds.
2. Attacker submits a transaction that calls `replace_class(class_hash=0xdeadbeef)` from within `C`. `0xdeadbeef` has never been declared.
3. The OS processes `execute_replace_class`: no validation is performed (line 898 TODO), `contract_state_changes[C].class_hash` is set to `0xdeadbeef`, and the transaction succeeds.
4. The block is finalized; the global state Merkle root now encodes `C.class_hash = 0xdeadbeef`.
5. Any future transaction calling `C` reaches `execute_entry_point` → `dict_read(contract_class_changes, 0xdeadbeef)` → returns `0` → `find_element(..., key=0)` → no compiled class fact found → proof fails.
6. No valid proof can ever be generated for a block that calls `C`. All funds in `C` are permanently frozen.
