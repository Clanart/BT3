### Title
`ProofOfInclusion::valid()` Trivially Returns `true` for Any Empty-Layers Proof, Enabling Forged Inclusion Attestations — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` contains a degenerate case where a proof with an empty `layers` vector unconditionally returns `true` for any `node_hash` value. Because `root_hash()` falls back to returning `self.node_hash` when `layers` is empty, the final equality check inside `valid()` reduces to `self.node_hash == self.node_hash`, which is always true. An attacker who can supply a deserialized `ProofOfInclusion` to a verifier that relies solely on `valid()` can forge an inclusion attestation for any claimed key without possessing a real Merkle path.

### Finding Description

`ProofOfInclusion` is a `Streamable`-derived struct exposed to Python via `PyStreamable` and `pyclass`. It can therefore be constructed from arbitrary untrusted bytes at the Python/wasm boundary.

`root_hash()` is defined as:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash   // ← fallback when layers is empty
    }
}
```

`valid()` is defined as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        // hash-chain check …
    }
    existing_hash == self.root_hash()   // ← when layers=[], this is node_hash == node_hash
}
```

When `layers` is empty the `for` loop is never entered, `existing_hash` remains `self.node_hash`, and `self.root_hash()` also returns `self.node_hash`. The final comparison is therefore `node_hash == node_hash`, which is unconditionally `true` for every possible `node_hash` value. [1](#0-0) 

The struct is fully deserializable from untrusted bytes because it derives `Streamable` and is registered as a Python class with `from_py_object`: [2](#0-1) 

The Python binding exposes `valid()` directly as the canonical verification method: [3](#0-2) 

The fuzz target and all tests call only `proof.valid()` / `proof_of_inclusion.valid()` without separately asserting `root_hash() == known_root`: [4](#0-3) 

### Impact Explanation

Any verifier that calls `proof.valid()` as its sole gate — without also independently asserting `proof.root_hash() == trusted_root` — will accept a forged `ProofOfInclusion(node_hash=<attacker_chosen>, layers=[])` as valid. Because `root_hash()` returns `node_hash` in the empty-layers case, an attacker who additionally knows the trusted root can set `node_hash = trusted_root`, causing both `valid() == true` and `root_hash() == trusted_root` to hold simultaneously, bypassing even a root-hash cross-check. The attacker can then assert that any arbitrary key-value pair is present in the DataLayer tree without possessing a real Merkle path, letting untrusted input prove invalid state — matching the **High** allowed impact: *DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion or lets untrusted input prove invalid state*.

### Likelihood Explanation

`ProofOfInclusion` is a first-class Streamable type exposed over the Python boundary. Any downstream consumer (chia-blockchain, wallet, DataLayer client) that deserializes a `ProofOfInclusion` from a peer and calls `.valid()` is reachable by an unprivileged network peer. The API design — a single `valid()` method with no root parameter — strongly encourages callers to treat it as a self-contained check, making the omission of a separate `root_hash()` comparison a natural mistake. The fuzz target and test suite reinforce this pattern by never performing the additional check.

### Recommendation

1. **Add a root parameter to `valid()`**: Change the signature to `pub fn valid(&self, expected_root: &Hash) -> bool` and replace the final comparison with `existing_hash == *expected_root`. This makes the function a complete, self-contained verifier and eliminates the degenerate empty-layers tautology.
2. **Reject empty-layers proofs for non-singleton trees**: If the API must remain parameter-free, add an explicit guard: if `layers` is empty, only return `true` when the caller has separately confirmed the tree is a singleton (i.e., `node_hash == root`).
3. **Update all call sites** — including the fuzz target and Python tests — to pass the known root, so the check is exercised end-to-end.

### Proof of Concept

```python
from chia_rs import ProofOfInclusion, ProofOfInclusionLayer
from chia_rs.sized_bytes import bytes32

# Attacker knows the trusted root of a multi-leaf tree
trusted_root = bytes32(b'\xab' * 32)

# Forge a proof: node_hash = trusted_root, layers = []
# valid() will return True because root_hash() == node_hash == trusted_root
forged = ProofOfInclusion(node_hash=trusted_root, layers=[])

assert forged.valid()                          # True  ← bypass
assert forged.root_hash() == trusted_root      # True  ← passes root check too

# A verifier doing only `proof.valid()` (or even `valid() and root_hash()==root`)
# accepts this as a legitimate inclusion proof for any claimed key.
```

    pub fn root_hash(&self) -> Hash {
        if let Some(last) = self.layers.last() {
            last.combined_hash
        } else {
            self.node_hash
        }

    pub fn valid(&self) -> bool {
        let mut existing_hash = self.node_hash;

        for layer in &self.layers {
            let calculated_hash = crate::calculate_internal_hash(
                &existing_hash,
                layer.other_hash_side,
                &layer.other_hash,
            );

            if calculated_hash != layer.combined_hash {
                return false;
            }

            existing_hash = calculated_hash;
        }

        existing_hash == self.root_hash()
    }
