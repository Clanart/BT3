### Title
`execute_replace_class` Accepts Undeclared Class Hash Without Validation — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the target class hash has been declared before updating a contract's class. Any deployed contract can replace its own class with an arbitrary, undeclared class hash. Any subsequent call to that contract causes the OS to fail during proof generation, permanently freezing all funds held by the contract.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` reads `class_hash` from the syscall request and directly writes it into `contract_state_changes` with no check that the class hash exists in `contract_class_changes`. The code itself acknowledges this with an explicit TODO: [1](#0-0) 

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The same missing check exists in the deprecated path: [2](#0-1) 

When a subsequent call is made to the contract whose class was replaced with an undeclared hash, `execute_entry_point` performs:

1. `dict_read{dict_ptr=contract_class_changes}(key=execution_context.class_hash)` — returns `0` (uninitialized default) because the class was never declared.
2. `find_element(..., key=compiled_class_hash)` with `compiled_class_hash=0` — this call **panics** because no `CompiledClassFact` with hash `0` exists in the bundle. [3](#0-2) 

The `find_element` call is not a graceful error path — it is a hard assertion. A failure here aborts OS execution entirely, making the block unprovable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value, every future call to that contract causes the OS to abort. The contract's storage (and any token balances or assets it holds) becomes permanently inaccessible. There is no recovery path: the state transition that set the invalid class hash was already committed, and the OS cannot prove any block that attempts to call the bricked contract.

---

### Likelihood Explanation

**High.** The `replace_class` syscall is a standard, publicly accessible syscall available to any deployed Cairo 1 contract. No privileged role, leaked key, or operator cooperation is required. A malicious contract owner can trigger this deliberately, or a buggy contract can trigger it accidentally. The attacker-controlled entry path is a standard user-submitted transaction invoking a contract that calls `replace_class` with an arbitrary felt value.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that the class hash exists as a key in `contract_class_changes` (i.e., its compiled class hash is non-zero). The existing TODO at line 898 of `syscall_impls.cairo` already identifies this exact gap. The fix should be applied to both `syscall_impls.cairo` (`execute_replace_class`) and `deprecated_execute_syscalls.cairo` (`execute_replace_class`).

---

### Proof of Concept

1. Attacker deploys a contract `C` with a valid declared class hash.
2. `C` executes a transaction that calls `replace_class(class_hash=0xdeadbeef)`, where `0xdeadbeef` is not present in `contract_class_changes`.
3. The OS processes this in `execute_replace_class` — gas is deducted, the response header is written, and `contract_state_changes[C].class_hash` is set to `0xdeadbeef` with no validation. [4](#0-3) 
4. The transaction succeeds (no revert). The state change is committed.
5. In the same or a subsequent block, any call to contract `C` reaches `execute_entry_point`.
6. `dict_read{dict_ptr=contract_class_changes}(key=0xdeadbeef)` returns `0` (undeclared). [5](#0-4) 
7. `find_element(..., key=0)` panics — no compiled class fact with hash `0` exists. [6](#0-5) 
8. OS execution aborts. The block containing any call to `C` cannot be proven. All funds in `C` are permanently frozen.
