### Title
Missing `chunk.shard_id() == shard_id` Guard in `set_state_header` Allows Cross-Shard State Root Substitution — (`chain/chain/src/state_sync/adapter.rs`)

### Summary

`ChainStateSyncAdapter::set_state_header` accepts a `ShardStateSyncResponseHeader` whose embedded chunk belongs to shard X while the caller requests shard Y. All five validation steps pass because none of them bind the chunk's `shard_id()` to the requested `shard_id` parameter. The corrupted header is then persisted under `StateHeaderKey(shard_Y, sync_hash)`, causing every downstream `set_state_part` and `apply_state_part` call to use shard X's `prev_state_root`, installing shard X's trie into shard Y's `ShardUId`.

---

### Finding Description

**Entrypoint and missing guard**

`set_state_header` receives `shard_id` (the shard the syncing node wants) and `shard_state_header` (the peer-supplied response). The function extracts the embedded chunk but never asserts `chunk.shard_id() == shard_id`: [1](#0-0) 

**Step 1-2 — `validate_chunk_proofs` does not check shard identity**

`validate_chunk_proofs` verifies the chunk's internal hash, tx root, and outgoing-receipts root. It has no parameter for the expected shard and performs no shard-identity check: [2](#0-1) 

**Step 3a — `verify_path` does not check Merkle-tree position**

The Merkle proof check uses `verify_path`, which only verifies that `hash(leaf) + path == root`. It does **not** verify that the leaf sits at the index corresponding to shard Y. The position-aware variant `verify_path_with_index` exists but is not used here: [3](#0-2) [4](#0-3) [5](#0-4) 

A valid Merkle proof for shard X's chunk at shard X's position in the block satisfies this check even when `shard_id` is shard Y.

**Step 4 — receipt validation uses the requested `shard_id`, not the chunk's**

The receipt hash is computed with the caller-supplied `shard_id` (shard Y): [6](#0-5) 

The `to_shard_id` field inside `ShardProof` is explicitly ignored: [7](#0-6) 

An attacker supplies shard Y's real incoming receipts (publicly available on-chain) together with shard X's chunk. The receipt proofs are valid because they are genuine blockchain data for shard Y.

**Step 5 — `validate_state_root_node` validates against the chunk's own `prev_state_root`**

The node is validated against `chunk_inner.prev_state_root()`, which is shard X's state root. The attacker obtains a valid `StateRootNode` for shard X from any state-sync provider: [8](#0-7) [9](#0-8) 

**Storage — header written under shard Y's key with shard X's state root**

After all checks pass, the header (containing shard X's chunk and shard X's `prev_state_root`) is stored under `StateHeaderKey(shard_Y, sync_hash)`: [10](#0-9) 

**Downstream corruption in `set_state_part` and `apply_state_part`**

`set_state_part` reads the stored header for shard Y and extracts the state root from the chunk — which is now shard X's root. It validates incoming parts against shard X's root: [11](#0-10) 

`apply_state_part` then writes those parts into the trie keyed by `shard_uid` derived from the requested `shard_id` (shard Y), installing shard X's trie data into shard Y's `ShardUId`: [12](#0-11) 

---

### Impact Explanation

After the attack completes, shard Y's trie and flat storage contain shard X's account/contract state. Any subsequent block application for shard Y reads the wrong state root, producing incorrect execution results, wrong balances, and corrupted contract storage — permanently, until the node re-syncs from scratch.

---

### Likelihood Explanation

Any peer on the network can respond to a state-sync header request. The attacker needs only publicly available data: a real chunk from shard X, its Merkle proof from the block header, shard Y's real incoming receipts, and a `StateRootNode` for shard X's state root (available from any state-sync provider). No validator or privileged role is required.

---

### Recommendation

Add an explicit shard-identity guard immediately after extracting the chunk, before any other validation:

```rust
if chunk.shard_id() != shard_id {
    return Err(Error::Other(format!(
        "set_shard_state failed: chunk shard_id {} != requested shard_id {}",
        chunk.shard_id(), shard_id
    )));
}
```

Additionally, replace the `verify_path` call for the chunk-inclusion check with `verify_path_with_index`, passing the shard's index in the epoch's shard layout as the expected position, so the Merkle proof is bound to the correct slot.

---

### Proof of Concept

```
Setup: two-shard epoch, node syncing shard Y (shard_id=1), sync_hash = H.

1. Obtain from any honest peer:
   - chunk_X: a valid ShardChunk for shard X (shard_id=0) included in block prev(H)
   - chunk_proof_X: its MerklePath in sync_prev_block_header.chunk_headers_root
   - prev_chunk_X / prev_chunk_proof_X: analogous for the previous block
   - incoming_receipts_Y: real ReceiptProofResponses for shard Y (to_shard_id=1)
   - root_proofs_Y: corresponding RootProofs valid against prev_chunk_outgoing_receipts_root
   - state_root_node_X: StateRootNode for chunk_X.prev_state_root()

2. Craft header:
   crafted = ShardStateSyncResponseHeaderV2 {
       chunk: chunk_X,
       chunk_proof: chunk_proof_X,
       prev_chunk_header: prev_chunk_X,
       prev_chunk_proof: prev_chunk_proof_X,
       incoming_receipts_proofs: incoming_receipts_Y,
       root_proofs: root_proofs_Y,
       state_root_node: state_root_node_X,
   }

3. Call:
   chain.state_sync_adapter.set_state_header(shard_Y, sync_hash, crafted)
   // Returns Ok(())

4. Assert:
   let stored = chain.state_sync_adapter.get_state_header(shard_Y, sync_hash).unwrap();
   let stored_root = stored.take_chunk().take_header().take_inner().prev_state_root();
   let expected_Y_root = prev_block.chunks()[shard_Y_index].prev_state_root();
   assert_ne!(stored_root, expected_Y_root);  // passes — shard X's root is stored for shard Y
```

pub fn validate_chunk_proofs(
    chunk: &ShardChunk,
    epoch_manager: &dyn EpochManagerAdapter,
) -> Result<bool, Error> {
    let correct_chunk_hash = chunk.compute_header_hash();

    // 1. Checking chunk.header.hash
    let header_hash = chunk.header_hash();
    if header_hash != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }

    // 2. Checking that chunk body is valid
    // 2a. Checking chunk hash
    if chunk.chunk_hash() != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }
    let height_created = chunk.height_created();
    let outgoing_receipts_root = chunk.prev_outgoing_receipts_root();
    let (transactions, receipts) = (chunk.to_transactions(), chunk.prev_outgoing_receipts());

    // 2b. Checking that chunk transactions are valid
    let (tx_root, _) = merklize(transactions);
    if &tx_root != chunk.tx_root() {
        byzantine_assert!(false);
        return Ok(false);
    }
    // 2c. Checking that chunk receipts are valid
    if height_created == 0 {
        return Ok(receipts.is_empty() && outgoing_receipts_root == &CryptoHash::default());
    } else {
        let shard_layout = {
            let prev_block_hash = chunk.prev_block_hash();
            epoch_manager.get_shard_layout_from_prev_block(&prev_block_hash)?
        };
        let outgoing_receipts_hashes = Chain::build_receipts_hashes(receipts, &shard_layout)?;
        let (receipts_root, _) = merklize(&outgoing_receipts_hashes);
        if &receipts_root != outgoing_receipts_root {
            byzantine_assert!(false);
            return Ok(false);
        }
    }
    Ok(true)

    fn apply_state_part(
        &self,
        shard_id: ShardId,
        state_root: &StateRoot,
        part_id: PartId,
        part: &StatePart,
        epoch_id: &EpochId,
    ) -> Result<(), Error> {
        let _timer = metrics::STATE_SYNC_APPLY_PART_DELAY
            .with_label_values(&[&shard_id.to_string()])
            .start_timer();

        let part = part
            .to_partial_state()
            .expect("Part was already validated earlier, so could never fail here");
        let ApplyStatePartResult { trie_changes, flat_state_delta, contract_codes } =
            Trie::apply_state_part(state_root, part_id, part);
        let tries = self.get_tries();
        let shard_uid = self.get_shard_uid_from_epoch_id(shard_id, epoch_id)?;
        let mut store_update = tries.store_update();
        tries.apply_all(&trie_changes, shard_uid, &mut store_update);
        tracing::debug!(target: "chain", %shard_id, values_count = %flat_state_delta.len(), "inserting values to flat storage");
        // TODO: `apply_to_flat_state` inserts values with random writes, which can be time consuming.
        //       Optimize taking into account that flat state values always correspond to a consecutive range of keys.
        flat_state_delta.apply_to_flat_state(&mut store_update.flat_store_update(), shard_uid);
        self.precompile_contracts(epoch_id, contract_codes)?;
        store_update.commit();
        Ok(())
    }
