### Title
Missing Declared Class Hash Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts an arbitrary `class_hash` value from a contract without verifying that the hash corresponds to a class that has been declared in the `contract_class_changes` dictionary. An explicit developer TODO acknowledges this missing check. This is the direct analog of the Serum v4 missing vault ownership check: just as that bug failed to verify that vault accounts were owned by the expected program, this bug fails to verify that the replacement class hash is owned/registered in the protocol's class registry.

---

### Finding Description

In `execute_replace_class` (lines 877–916 of `syscall_impls.cairo`), after deducting gas, the function reads `request.class_hash` and immediately writes it into `contract_state_changes` for the calling contract address, with no check that the hash exists in `contract_class_changes`:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The missing check should verify that `class_hash` is a key present in `contract_class_changes` (i.e., that `dict_read{dict_ptr=contract_class_changes}(key=class_hash)` returns a non-zero compiled class hash). This is exactly the check performed in `execute_entry_point` when a contract is actually called:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
``` [2](#0-1) 

If `class_hash` was never declared, `dict_read` returns 0 (`UNINITIALIZED_CLASS_HASH`), and `find_element` fails because no compiled class fact with hash 0 exists. The OS would be unable to generate a valid proof for any subsequent block that attempts to call the frozen contract.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract successfully calls `replace_class(undeclared_hash)`:

1. The OS commits the block with `contract_state_changes[contract_address].class_hash = undeclared_hash`.
2. The on-chain state root reflects a contract whose class hash has no corresponding compiled class.
3. Every future call to that contract causes `execute_entry_point` to look up `contract_class_changes[undeclared_hash]`, receive 0, then fail in `find_element`.
4. The sequencer must revert all such calls, making the contract permanently uncallable.
5. Any ERC-20 tokens, ETH, or other assets held in the contract's storage are permanently inaccessible — no `transfer`, `withdraw`, or any other entry point can ever execute again. [3](#0-2) 

---

### Likelihood Explanation

**Medium.**

The `replace_class` syscall is callable by any contract on itself — no privileged role is required. The realistic attack scenarios are:

- A malicious contract (e.g., a rug-pull DeFi pool) that accepts user deposits and then calls `replace_class(0xdeadbeef)` to permanently freeze all deposited funds.
- A contract with an upgradability bug that allows an attacker to trigger `replace_class` with an attacker-controlled hash that was never declared.

The OS provides zero enforcement against this. The only current mitigation is off-chain sequencer logic (blockifier), but the OS proof system itself does not enforce the invariant, meaning a proof can be generated and verified on-chain for a state containing a contract with an undeclared class hash.

---

### Recommendation

In `execute_replace_class`, before writing the new state entry, add a check that `class_hash` is present in `contract_class_changes` with a non-zero compiled class hash:

```cairo
let class_hash = request.class_hash;

// Verify the class hash is declared.
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=class_hash
);
with_attr error_message("replace_class: class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the validation already performed in `execute_entry_point` and closes the gap between what the OS accepts in state transitions and what it can actually execute. [4](#0-3) 

---

### Proof of Concept

1. Declare a class `ClassA` (hash `H_A`) and deploy a contract `C` using it. `C` holds user funds.
2. Deploy a second class `ClassB` (hash `H_B`) that is **never declared** (not submitted via a `declare` transaction, so `H_B` is absent from `contract_class_changes`).
3. From within `C`, call the `replace_class` syscall with `class_hash = H_B`.
4. The OS executes `execute_replace_class`: gas is deducted, `contract_state_changes[C].class_hash` is set to `H_B`, no validation of `H_B` occurs.
5. The block is proven and committed on-chain. The state root now encodes `C` with class hash `H_B`.
6. Any subsequent transaction calling `C` reaches `execute_entry_point`, which does `dict_read(contract_class_changes, H_B)` → returns 0, then `find_element(..., key=0)` → assertion failure.
7. The sequencer must revert all calls to `C`. `C` is permanently frozen. All funds in `C`'s storage are permanently inaccessible. [5](#0-4) [6](#0-5)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

        key=contract_address,

    );
