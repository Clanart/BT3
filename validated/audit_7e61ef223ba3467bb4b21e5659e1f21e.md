### Title
Floor Rounding in L2 Gas Price Threshold Admits Below-Minimum Transactions at Gateway ŌĆö (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`validate_tx_l2_gas_price_within_threshold` computes the minimum L2 gas price threshold using `.to_integer()` (floor/truncation) on a `Ratio`. When `min_gas_price_percentage ├Ś previous_l2_gas_price` is not evenly divisible by 100, the threshold is 1 unit lower than the intended minimum, allowing a transaction whose gas price is strictly below the intended floor to pass gateway admission.

---

### Finding Description

In `validate_tx_l2_gas_price_within_threshold`:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();          // ŌåÉ truncates (floor)
if tx_l2_gas_price.0 < threshold {
    return Err(ŌĆ”);
}
```

`Ratio::to_integer()` truncates toward zero, so `threshold = ŌīŖmin_gas_price_percentage ├Ś prev_price / 100Ōīŗ`.

The invariant the check is meant to enforce is: *"reject any transaction whose L2 gas price is below `min_gas_price_percentage %` of the previous block's L2 gas price."* For integer-valued gas prices the correct threshold is the ceiling, not the floor, because a gas price of `ŌīŖXŌīŗ` is strictly less than `X` whenever `X` is non-integer.

**Concrete example**

| Parameter | Value |
|---|---|
| `min_gas_price_percentage` | 50 |
| `previous_block_l2_gas_price` | 101 |
| True minimum (50 % of 101) | 50.5 ŌåÆ must be Ōēź 51 |
| Computed `threshold` (floor) | `ŌīŖ50 ├Ś 101 / 100Ōīŗ = ŌīŖ50.5Ōīŗ = 50` |

A transaction with `tx_l2_gas_price = 50` satisfies `50 < 50 ŌåÆ false`, so it passes. The intended minimum is 51. The rounding error directly mirrors the MagicLP pattern: a `mulFloor`-style operation in a comparison causes the wrong branch to be taken, admitting a value that should be rejected.

The root cause is the same as in the external report: using floor rounding in a comparison that determines which branch is taken, when ceiling rounding is required to preserve the stated invariant.

---

### Impact Explanation

A transaction whose `max_price_per_unit` for L2 gas is exactly `ŌīŖmin_gas_price_percentage ├Ś prev_price / 100Ōīŗ` ŌĆö one unit below the intended minimum ŌĆö bypasses the gateway's gas-price floor check and is admitted to the mempool. This is a **High** impact under the allowed scope:

> *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

Once admitted, the transaction proceeds through the normal sequencing pipeline with a gas price that violates the operator-configured fee-market floor, undermining the EIP-1559-style price signal and potentially allowing fee-market manipulation at scale (e.g., repeatedly submitting transactions at the floor-minus-one price to suppress the effective minimum).

---

### Likelihood Explanation

The condition is triggered whenever `min_gas_price_percentage ├Ś previous_l2_gas_price` is not divisible by 100. With a default `min_gas_price_percentage` of 50 and a continuously varying `previous_l2_gas_price` (updated every block via the EIP-1559 formula), roughly half of all blocks will produce a non-integer threshold. Any unprivileged user who observes the previous block's L2 gas price can craft a transaction at exactly the floor-rounded threshold.

---

### Recommendation

Replace `.to_integer()` with `.ceil().to_integer()` so the threshold is the smallest integer that is **at least** `min_gas_price_percentage %` of the previous price:

```rust
// Before (floor ŌĆö admits gas price one unit below the intended minimum)
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();

// After (ceiling ŌĆö correctly enforces the stated percentage floor)
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .ceil()
    .to_integer();
```

This is the direct sequencer analog of the MagicLP recommendation: replace `mulFloor` with `mulCeil` in the comparison that selects which branch is taken.

### Proof of Concept

1. Observe the previous block's L2 gas price, e.g. `previous_l2_gas_price = 101`.
2. Read the gateway's `min_gas_price_percentage`, e.g. `50`.
3. Compute the floor-rounded threshold: `ŌīŖ50 ├Ś 101 / 100Ōīŗ = 50`.
4. Submit a V3 `InvokeTransaction` with `AllResources` bounds where `l2_gas.max_price_per_unit = 50`.
5. `validate_tx_l2_gas_price_within_threshold` evaluates `50 < 50 ŌåÆ false` and returns `Ok(())`.
