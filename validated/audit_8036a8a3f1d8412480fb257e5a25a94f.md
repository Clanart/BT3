### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Bricking — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary `class_hash` value from the caller without verifying that the hash corresponds to a previously declared contract class. A developer-left TODO comment at the exact location of the missing check explicitly acknowledges this gap. Any contract can invoke `replace_class` with an undeclared hash, permanently setting its own class hash to a value that has no corresponding compiled class, making the contract permanently uncallable and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads `class_hash` directly from the syscall request and writes it into `contract_state_changes` with no validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment at line 898 is an explicit, in-code acknowledgment that the check is absent. No upper or lower bound, no membership check against `contract_class_changes`, and no cross-reference against `compiled_class_facts_bundle` is performed before the state update is committed.

The downstream consumer of this class hash is `execute_entry_point`, which performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
``` [2](#0-1) 

If `class_hash` is not declared, `dict_read` returns the default value `0`, and `find_element` will fail to locate a compiled class with hash `0`, producing an irrecoverable proof-level failure for any block that subsequently attempts to call the bricked contract.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is overwritten with an undeclared value:

1. Every future call to that contract causes `find_element` to fail inside the OS prover, making the block unprovable unless the sequencer excludes all such calls.
2. The contract becomes permanently uncallable at the protocol level.
3. Any ERC-20 balances, ETH, or other assets held by the contract are irrecoverably frozen with no upgrade or rescue path, because the very mechanism that would allow an upgrade (`replace_class`) has already been used to brick the contract.

---

### Likelihood Explanation

- The `replace_class` syscall is available to every deployed contract with no access restriction at the OS level.
- A single transaction from any unprivileged sender that controls (or can influence) a contract holding user funds is sufficient to trigger the issue.
- The TODO comment confirms the check was intentionally deferred, meaning the missing validation is present in the current production code path.
- No privileged role, leaked key, or external dependency is required.

---

### Recommendation

Before committing the `dict_update` in `execute_replace_class`, verify that `class_hash` exists as a key in `contract_class_changes` (i.e., it has been declared in the current or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` for the supplied `class_hash` and assert the returned compiled class hash is non-zero. This mirrors the check that already exists implicitly in `execute_entry_point` but must be enforced eagerly at the point of the setter, analogous to the `require(limit < IERC20(UMAMI).totalSupply(), ...)` fix applied in the reference report.

---

### Proof of Concept

1. Deploy contract `Vault` that holds user ETH and exposes a public `brick()` entry point that calls `replace_class(0xdeadbeef)` — a hash never declared on-chain.
2. Send a transaction invoking `Vault.brick()`.
3. The OS processes `execute_replace_class`: gas is deducted, no class-existence check is performed, and `contract_state_changes[Vault] = StateEntry(class_hash=0xdeadbeef, ...)` is written.
4. The block is proven successfully (the OS accepts the state update).
5. Any subsequent transaction calling `Vault` causes `execute_entry_point` to call `dict_read(contract_class_changes, 0xdeadbeef)` → returns `0` → `find_element(..., key=0)` → element not found → proof failure or sequencer forced exclusion.
6. All funds in `Vault` are permanently frozen with no recovery mechanism. [3](#0-2)
