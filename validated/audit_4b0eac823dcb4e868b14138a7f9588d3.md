### Title
Unchecked `class_hash` in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the caller-supplied `class_hash` corresponds to a previously declared contract class. This is an exact structural analog to the UniswapV3 report's root cause: an unchecked arbitrary input is accepted and committed to protocol state without validation. Any contract can replace its class with an arbitrary, undeclared felt value, permanently bricking itself and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class`, the OS reads `request.class_hash` directly from the syscall request and writes it into `contract_state_changes` without any check that the hash exists in `contract_class_changes` (i.e., that it was ever declared):

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The `TODO` comment explicitly acknowledges the missing check. The OS accepts and commits any arbitrary felt as the new class hash.

When the contract is subsequently called, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
```

If `class_hash` was never declared, `dict_read` returns `0` (uninitialized), and `find_element` with key `0` fails to find a witness — causing the OS to be unprovable for that block, or, if the contract is not called again in the same block, leaving the contract permanently bricked in the committed state.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Attack path:
1. Attacker deploys a contract that accepts user funds (e.g., a fake vault or DeFi protocol).
2. Users deposit funds into the contract.
3. Attacker calls `replace_class` with an arbitrary undeclared felt (e.g., `0xdeadbeef`).
4. The OS accepts this without validation and commits the corrupted `class_hash` to the state Merkle tree.
5. The contract's class hash now points to a non-existent compiled class.
6. All future calls to the contract fail at the OS level (no valid compiled class can be found).
7. Funds held by the contract are permanently frozen — no withdrawal, no recovery.

Additionally, if the contract is called again within the same block after `replace_class`, `find_element` fails to find a witness for `compiled_class_hash=0`, making the entire block unprovable — a network halt.

---

### Likelihood Explanation

**High.** The attack requires only:
- Deploying a contract (permissionless).
- Calling the `replace_class` syscall from within that contract (any contract can do this).
- No privileged role, no leaked key, no operator cooperation.

The missing check is explicitly flagged with a `TODO` comment, confirming it is a known gap in the current implementation.

---

### Recommendation

In `execute_replace_class`, before writing the new `class_hash` to `contract_state_changes`, verify that the hash exists as a key in `contract_class_changes` (i.e., it was declared in the current or a prior block). Concretely:

```cairo
// Verify the new class_hash is a declared class.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the validation that `execute_declare_transaction` enforces via `assert_not_zero(compiled_class_hash)` before writing to `contract_class_changes`.

---

### Proof of Concept

1. Deploy contract `VaultContract` that:
   - Accepts ETH/token deposits from users.
   - Exposes a `brick_self()` function that calls `replace_class(0xdeadbeef_arbitrary_undeclared_hash)`.
2. Users deposit funds into `VaultContract`.
3. Attacker calls `brick_self()`.
4. The OS executes `execute_replace_class`:
   - `class_hash = 0xdeadbeef_arbitrary_undeclared_hash` — no validation performed.
   - `contract_state_changes` is updated with the invalid class hash.
   - Proof is generated successfully (no OS-level rejection).
5. In any subsequent block, any call to `VaultContract` reaches `execute_entry_point`:
   - `dict_read(contract_class_changes, 0xdeadbeef...)` → returns `0`.
   - `find_element(..., key=0)` → no witness exists → OS fails or contract call fails.
6. `VaultContract` is permanently unusable. All deposited funds are frozen.

    alloc_locals;

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;
