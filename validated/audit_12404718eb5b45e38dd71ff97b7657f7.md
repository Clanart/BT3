### Title
Missing Declared-Class Validation in `execute_replace_class` Enables Permanent Fund Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the replacement class hash corresponds to a previously declared class. Any contract can invoke `replace_class` with an undeclared or zero class hash, rendering itself permanently uncallable and freezing all funds held within it. The developers acknowledge this gap with an explicit TODO comment at the exact missing-check location.

---

### Finding Description

In `syscall_impls.cairo`, the function `execute_replace_class` (lines 878–916) processes the `replace_class` syscall. It reads `request.class_hash` and directly commits it to the contract's state entry via `dict_update`, with **no check** that the hash corresponds to a declared class: [1](#0-0) 

The developers themselves acknowledge this gap with a TODO comment at line 898: [2](#0-1) 

The full missing-check context:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

This is structurally identical to the reported Vault.sol vulnerability: just as `utilize()` lacked `require(address(controller) != address(0))` before transferring tokens to `controller`, `execute_replace_class` lacks `assert_not_zero(class_hash)` (or an equivalent declared-class membership check) before committing the new class hash to state. In both cases, a zero/invalid default value is accepted without guard, and the destructive consequence is irreversible.

Contrast this with `execute_deploy` (lines 452–556), which also uses a class hash but immediately dispatches into the class — causing an early failure if the class is undeclared. `replace_class` defers the failure to the *next* call to the contract, by which point the state is already committed and the proof is finalized. [3](#0-2) 

---

### Impact Explanation

If a contract's class hash is set to `0` or any undeclared felt:

- All future entry-point dispatches to that contract fail — no compiled class exists for the hash.
- All ERC-20 balances, collateral, or other assets stored in the contract's storage become permanently inaccessible.
- The state transition is committed to the proven block output; it cannot be rolled back.

This constitutes **permanent freezing of funds** — a Critical-severity impact within the allowed scope.

---

### Likelihood Explanation

Any contract that exposes an upgrade path (e.g., a `set_implementation` or `upgrade` function) without independently validating the new class hash is exploitable. An attacker who can call such a function with an arbitrary class hash argument can trigger this path. Upgradeable patterns using `replace_class` are common in StarkNet DeFi protocols. The attacker-controlled entry path is a standard external transaction call — no privileged role is required. [4](#0-3) 

---

### Recommendation

Add a validation step inside `execute_replace_class` before committing the new class hash. The check should verify the hash exists in the declared-class registry (`contract_class_changes` dict or the global class store):

```cairo
// Ensure the replacement class has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

At minimum, add `assert_not_zero(class_hash)` to prevent the explicit zero-hash case, mirroring the pattern already used in `execute_declare_transaction`:

func execute_replace_class{
    range_check_ptr,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }
