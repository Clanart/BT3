### Title
Stale `feemarketParams` Snapshot in `newCosmosAnteHandler` Allows Cosmos Transactions to Bypass Current EIP-1559 Base Fee Enforcement - (File: `evmd/ante/handler_options.go`)

### Summary

`newCosmosAnteHandler` captures `feemarketParams` (including `BaseFee`) once at handler construction time and passes a pointer to that frozen snapshot into `NewDynamicFeeChecker`. Because the ante handler is constructed once at application startup, the `BaseFee` used for all subsequent Cosmos-native transaction fee validation never updates, allowing transactions to pass fee checks against a stale (genesis-era) base fee even when the live base fee has risen significantly.

### Finding Description

In `evmd/ante/handler_options.go`, `newCosmosAnteHandler` reads `feemarketParams` from the keeper at construction time:

```go
feemarketParams := options.FeeMarketKeeper.GetParams(ctx)
...
txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
``` [1](#0-0) 

The pointer `&feemarketParams` is passed into `NewDynamicFeeChecker`, which captures it in a closure:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
``` [2](#0-1) 

`feemarketParams` is a local variable in `newCosmosAnteHandler`; Go's escape analysis moves it to the heap, but its value is fixed at the moment `newCosmosAnteHandler` is called. Every subsequent invocation of the returned `TxFeeChecker` closure reads `feemarketParams.BaseFee` from this frozen copy, not from the live KV store.

By contrast, the EVM ante handler (`newEthAnteHandler`) fetches a fresh `EVMBlockConfig` on every transaction, which reads the current `BaseFee` from the store after `BeginBlock` has updated it:

```go
blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
...
baseFee := blockCfg.BaseFee
``` [3](#0-2) 

The feemarket `BeginBlock` correctly updates the base fee each block:

```go
func (k *Keeper) BeginBlock(ctx sdk.Context) error {
    baseFee := k.CalculateBaseFee(ctx)
    ...
    k.SetBaseFee(ctx, baseFee)
``` [4](#0-3) 

But this updated value is never reflected in the frozen `feemarketParams` pointer held by the Cosmos ante handler's `DynamicFeeChecker`.

The same frozen pointer is also passed to `MinGasPriceDecorator`:

```go
cosmos.NewMinGasPriceDecorator(options.FeeMarketKeeper, evmDenom, &feemarketParams),
``` [5](#0-4) 

causing `MinGasPrice` enforcement for Cosmos transactions to also use the startup-time value.

### Impact Explanation

When `DynamicFeeChecker: true` is set in `HandlerOptions`, every Cosmos-native transaction (bank sends, IBC transfers, governance votes, etc.) has its fee validated against the base fee that was current at node startup, not the live base fee. If the base fee rises (e.g., due to sustained high EVM gas usage), Cosmos transactions can be submitted and committed with fees that are below the current required base fee. This is a fee market accounting bug that permits invalid transactions to commit and causes valid user funds/fees to be mis-accounted — matching the "High" impact category for ante handler / fee market bugs.

### Likelihood Explanation

The vulnerability is active whenever `HandlerOptions.DynamicFeeChecker = true`. The base fee changes every block via `CalculateBaseFee`; on a busy chain the divergence between the frozen startup value and the live value grows continuously. Any unprivileged user submitting a Cosmos transaction can exploit this by setting fees just above the stale base fee but below the current one. No special privileges, keys, or validator cooperation are required.

### Recommendation

Move the `feemarketParams` read inside the `TxFeeChecker` closure so it is fetched fresh from the keeper on every invocation:

```go
func NewDynamicFeeChecker(ethCfg *params.ChainConfig, evmParams *types.Params, fmKeeper FeeMarketKeeper) authante.TxFeeChecker {
    return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
        feemarketParams := fmKeeper.GetParams(ctx)   // live read
        baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, &feemarketParams)
        ...
    }
}
```

Alternatively, mirror the pattern used in `newEthAnteHandler` and call `EVMBlockConfig` (which already caches per-block) to obtain the current `BaseFee` and `FeeMarketParams` inside the closure.

### Proof of Concept

1. Start a node with `DynamicFeeChecker: true`.
2. Record the genesis base fee `B0` (e.g., `1_000_000_000`).
3. Submit enough EVM transactions to drive the base fee up to `B1 > B0` over several blocks.
4. Submit a Cosmos `bank.MsgSend` with `fee = B0 * gasLimit` (below the current `B1 * gasLimit`).
5. Observe that the transaction passes `CheckTx` and `DeliverTx` because `NewDynamicFeeChecker` compares against the frozen `B0`, not the live `B1`.
6. The transaction is committed despite paying less than the current required base fee. [6](#0-5) [7](#0-6) [8](#0-7) [9](#0-8)

func NewDynamicFeeChecker(ethCfg *params.ChainConfig, evmParams *types.Params, feemarketParams *feemarkettypes.Params) authante.TxFeeChecker {
	return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
		feeTx, ok := tx.(sdk.FeeTx)
		if !ok {
			return nil, 0, fmt.Errorf("tx must be a FeeTx")
		}

		if ctx.BlockHeight() == 0 {
			// genesis transactions: fallback to min-gas-price logic
			return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
		}

		denom := evmParams.EvmDenom

		baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
		if baseFee == nil {
			// london hardfork is not enabled: fallback to min-gas-prices logic
			return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
		}

		// default to `MaxInt64` when there's no extension option.
		maxPriorityPrice := sdkmath.NewInt(math.MaxInt64)

		// get the priority tip cap from the extension option.
		if hasExtOptsTx, ok := tx.(authante.HasExtensionOptionsTx); ok {
			for _, opt := range hasExtOptsTx.GetExtensionOptions() {
				if extOpt, ok := opt.GetCachedValue().(*ethermint.ExtensionOptionDynamicFeeTx); ok {
					maxPriorityPrice = extOpt.MaxPriorityPrice
					break
				}
			}
		}

		if maxPriorityPrice.Sign() == -1 {
			return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, "priority fee is negative")
		}

		gas := feeTx.GetGas()
		feeCoins := feeTx.GetFee()
		fee := feeCoins.AmountOf(denom)

		feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
		baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)

		if feeCap.LT(baseFeeInt) {
			return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, "insufficient gas prices; got: %s required: %s", feeCap, baseFeeInt)
		}

		// calculate the effective gas price using the EIP-1559 logic.
		effectivePrice := sdkmath.NewIntFromBigInt(types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))

		// NOTE: create a new coins slice without having to validate the denom
		effectiveFee := sdk.Coins{
			{
				Denom:  denom,
				Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas)),
			},
		}

		bigPriority := effectivePrice.Sub(baseFeeInt).Quo(types.DefaultPriorityReduction)
		priority := int64(math.MaxInt64)

		if bigPriority.IsInt64() {
			priority = bigPriority.Int64()
		}

		return effectiveFee, priority, nil
	}

func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
}

func (k *Keeper) EVMBlockConfig(ctx sdk.Context, chainID *big.Int) (*EVMBlockConfig, error) {
	objStore := ctx.ObjectStore(k.objectKey)
	v := objStore.Get(types.KeyPrefixObjectParams)
	if v != nil {
		return v.(*EVMBlockConfig), nil
	}

	params := k.GetParams(ctx)
	ethCfg := params.ChainConfig.EthereumConfig(chainID)

	feemarketParams := k.feeMarketKeeper.GetParams(ctx)

	// get the coinbase address from the block proposer
	coinbase, err := k.GetCoinbaseAddress(ctx)
	if err != nil {
		return nil, errorsmod.Wrap(err, "failed to obtain coinbase address")
	}

	var baseFee *big.Int
	if types.IsLondon(ethCfg, ctx.BlockHeight()) {
		baseFee = feemarketParams.GetBaseFee()
		// should not be nil if london hardfork enabled
		if baseFee == nil {
			baseFee = new(big.Int)
		}
	}
	time := ctx.BlockHeader().Time
	var blockTime uint64
	if !time.IsZero() {
		blockTime, err = ethermint.SafeUint64(time.Unix())
		if err != nil {
			return nil, err
		}
	}
	blockNumber := big.NewInt(ctx.BlockHeight())
	rules := ethCfg.Rules(blockNumber, ethCfg.MergeNetsplitBlock != nil, blockTime)

	// Build the default precompile set once per block.
	contracts := make(map[common.Address]vm.PrecompiledContract)
	for addr, c := range vm.DefaultPrecompiles(rules) {
		contracts[addr] = c
	}

	var zero common.Hash
	cfg := &EVMBlockConfig{
		Params:             params,
		FeeMarketParams:    feemarketParams,
		ChainConfig:        ethCfg,
		CoinBase:           coinbase,
		BaseFee:            baseFee,
		Difficulty:         new(big.Int),
		Random:             &zero,
		BlobBaseFee:        new(big.Int),
		BlockNumber:        blockNumber,
		BlockTime:          blockTime,
		Rules:              rules,
		DefaultPrecompiles: contracts,
	}
	objStore.Set(types.KeyPrefixObjectParams, cfg)
	return cfg, nil
}
