### Title
`MerkleBlob::batch_insert` Bypasses Duplicate Key Check, Enabling DataLayer Tree Root Corruption — (File: `crates/chia-datalayer/src/merkle/blob.rs`)

### Summary

`MerkleBlob::batch_insert` skips the duplicate-key guard that `MerkleBlob::insert` enforces. When the tree already has two or more leaves (`leaf_count > 1`), every item in the batch is written directly to the blob without any existence check. When the tree has zero or one leaf, only the last two items in the batch go through the safe `self.insert()` path; all earlier items bypass the check. An attacker who can supply a batch containing a key that already exists in the tree (or duplicate keys within the batch itself) silently corrupts the `MerkleBlob`, producing an incorrect Merkle root and invalidating all subsequent inclusion/exclusion proofs.

### Finding Description

`MerkleBlob::insert` guards against duplicate keys at lines 369–371:

```rust
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
```

`MerkleBlob::batch_insert` does not replicate this guard for the bulk path:

```rust
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else { … };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;  // ← safe path
    }
}

for ((key, value), hash) in keys_values_hashes {          // ← NO duplicate check
    let new_leaf_index = self.get_new_index();
    let new_block = Block { … Node::Leaf(LeafNode { key, value, hash, … }) … };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
``` [1](#0-0) [2](#0-1) 

Two distinct bypass scenarios exist:

1. **`leaf_count > 1` (most common case):** The entire `if` block is skipped. Every item in the batch is written via `insert_entry_to_blob` with no existence check. A key already present in the tree, or a key repeated within the batch, is silently inserted again.

2. **`leaf_count <= 1`:** `keys_values_hashes.pop()` removes the *last* two items and routes them through the safe `self.insert()`. All *earlier* items (indices `0..n-3`) still go through the unchecked bulk loop.

`insert_entry_to_blob` writes the raw `Block` to the blob and updates `block_status_cache` via `add_leaf`, which calls `key_to_index.insert(leaf.key, index)`. For a duplicate key this silently overwrites the cached index, so the cache ends up pointing to the *last* inserted leaf while the blob contains *two* leaf nodes for the same key. This divergence causes `check_integrity` to fail (`leaf_count != key_to_index_cache_length`) and, critically, causes the computed Merkle root to be wrong because both leaf nodes contribute to the hash tree. [3](#0-2) [4](#0-3) 

The Python binding `py_batch_insert` exposes this path directly with no additional guard:

```rust
pub fn py_batch_insert(
    &mut self,
    keys_values: Vec<(KeyId, ValueId)>,
    hashes: Vec<Hash>,
) -> PyResult<()> {
    // only checks length parity, not key uniqueness
    self.batch_insert(zip(keys_values, hashes).collect())?;
    Ok(())
}
``` [5](#0-4) 

### Impact Explanation

A corrupted `MerkleBlob` produces an incorrect root hash. Any `get_proof_of_inclusion` call on the corrupted tree generates a proof that does not match the true committed root, so `confirm_included_already_hashed` / `confirm_not_included_already_hashed` return wrong answers. An adversary who controls the batch contents can therefore cause the DataLayer to attest to a false state: a key that was never legitimately inserted can be "proven" included, or a key that was inserted can be "proven" excluded. This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."* [6](#0-5) 

### Likelihood Explanation

The Python binding `py_batch_insert` is the primary DataLayer write path for bulk updates. Any caller that supplies a batch containing a repeated `KeyId` — whether by mistake or by deliberate crafting — triggers the corruption. Because the function returns `Ok(())` with no error, the caller has no indication that the tree is now invalid. The `check_integrity` call that would detect the problem is not invoked automatically; it must be called explicitly. [7](#0-6) 

### Recommendation

Add the same duplicate-key guard to the bulk loop in `batch_insert` that `insert` applies:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // … existing bulk-insert logic …
}
```

Alternatively, deduplicate the input vector before processing, or add a test that calls `batch_insert` with a repeated key and asserts `KeyAlreadyPresent` is returned.

### Proof of Concept

```python
from chia_rs.datalayer import MerkleBlob, KeyId, ValueId
from chia_rs.sized_bytes import bytes32

blob = MerkleBlob(blob=bytearray())

# Pre-populate so leaf_count > 1 (bypasses the safe self.insert() path entirely)
for i in range(3):
    blob.insert(KeyId(i), ValueId(i), bytes32(bytes([i]*32)))

key_dup = KeyId(99)
hash_a  = bytes32(bytes([0xAA]*32))
hash_b  = bytes32(bytes([0xBB]*32))

# Insert key_dup once legitimately
blob.insert(key_dup, ValueId(99), hash_a)
root_before = blob.get_root_hash()

# batch_insert with the same key — no KeyAlreadyPresent raised
blob.batch_insert(
    [(key_dup, ValueId(100))],
    [hash_b],
)
root_after = blob.get_root_hash()

# Root has changed silently; tree is now corrupt
assert root_before != root_after, "root changed without legitimate update"
# check_integrity will now raise because leaf_count != cache length
blob.check_integrity()   # expected to raise

    pub fn batch_insert(
        &mut self,
        mut keys_values_hashes: Vec<((KeyId, ValueId), Hash)>,
    ) -> Result<(), Error> {
        // OPT: perhaps go back to taking an iterator?
        // OPT: would it be worthwhile to hold the entire blocks?
        let mut indexes = vec![];

        if self.block_status_cache.leaf_count() <= 1 {
            for _ in 0..2 {
                let Some(((key, value), hash)) = keys_values_hashes.pop() else {
                    return Ok(());
                };
                self.insert(key, value, &hash, InsertLocation::Auto {})?;
            }
        }

        for ((key, value), hash) in keys_values_hashes {
            let new_leaf_index = self.get_new_index();
            let new_block = Block {
                metadata: NodeMetadata {
                    node_type: NodeType::Leaf,
                    dirty: false,
                },
                node: Node::Leaf(LeafNode {
                    parent: Parent(None),
                    hash,
                    key,
                    value,
                }),
            };
            self.insert_entry_to_blob(new_leaf_index, &new_block)?;
            indexes.push(new_leaf_index);
        }

        // OPT: can we insert the top node first?  maybe more efficient to update it's children
        //      than to update the parents of the children when traversing leaf to sub-root?
        while indexes.len() > 1 {
            let mut new_indexes = vec![];

            for chunk in indexes.chunks(2) {
                let [index_1, index_2] = match chunk {
                    [index] => {
                        new_indexes.push(*index);
                        continue;
                    }
                    [index_1, index_2] => [*index_1, *index_2],
                    _ => unreachable!(
                        "chunk should always be either one or two long and be handled above"
                    ),
                };

                let new_internal_node_index = self.get_new_index();

                let mut hashes = vec![];
                for index in [index_1, index_2] {
                    let block = self.update_parent(index, Some(new_internal_node_index))?;
                    hashes.push(block.node.hash());
                }

                let new_block = Block {
                    metadata: NodeMetadata {
                        node_type: NodeType::Internal,
                        dirty: false,
                    },
                    node: Node::Internal(InternalNode {
                        parent: Parent(None),
                        hash: internal_hash(&hashes[0], &hashes[1]),
                        left: index_1,
                        right: index_2,
                    }),
                };

                self.insert_entry_to_blob(new_internal_node_index, &new_block)?;
                new_indexes.push(new_internal_node_index);
            }

            indexes = new_indexes;
        }

        if indexes.len() == 1 {
            // OPT: can we avoid this extra min height leaf traversal?
            let min_height_leaf = self.get_min_height_leaf()?;
            self.insert_subtree_at_key(min_height_leaf.key, indexes[0], Side::Left)?;
        }

        Ok(())
    }
