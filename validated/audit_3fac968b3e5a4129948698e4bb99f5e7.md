### Title
Silent `ft_transfer` Failure in `fin_transfer_send_tokens_callback` Causes Permanent Fund Lock — (File: `near/omni-bridge/src/lib.rs`)

---

### Summary

When finalizing a cross-chain transfer to a NEAR recipient using a plain `ft_transfer` (no message payload), the `fin_transfer_send_tokens_callback` does not check whether the token transfer actually succeeded. If `ft_transfer` fails — for example because the recipient is blacklisted by a USDC-like NEP-141 token — the transfer is permanently marked as finalized while the tokens are never delivered, causing irrecoverable fund loss for the user.

---

### Finding Description

**Root cause — `is_refund_required` ignores `ft_transfer` failures:** [1](#0-0) 

```rust
fn is_refund_required(is_ft_transfer_call: bool) -> bool {
    if is_ft_transfer_call {
        match env::promise_result_checked(0, MAX_FT_TRANSFER_CALL_RESULT) {
            Ok(value) => { ... }
            Err(_) => false,
        }
    } else {
        // Not ft_transfer_call: don't refund
        false   // <-- unconditional, promise result is never inspected
    }
}
```

When `msg` is empty (no cross-contract message), `send_tokens` issues a plain `ft_transfer`. The callback receives `is_ft_transfer_call = false` and `is_refund_required` returns `false` **regardless of whether the promise succeeded or failed**. The promise result is never read.

**State is committed before the transfer:** [2](#0-1) 

`add_fin_transfer` records the transfer as finalized before `send_tokens` is even dispatched. If the subsequent `ft_transfer` panics, the finalization record is already written and cannot be undone by the callback.

**Callback takes the "success" path on failure:** [3](#0-2) 

```rust
pub fn fin_transfer_send_tokens_callback(...) {
    if Self::is_refund_required(is_ft_transfer_call) {
        // revert: burn, revert lock actions, remove_fin_transfer  <-- never reached for ft_transfer
    } else {
        // "success": send fee, emit FinTransferEvent              <-- always taken for ft_transfer
    }
}
```

When `ft_transfer` fails, `is_refund_required(false)` returns `false`, so `remove_fin_transfer` is never called, lock actions are never reverted, and `FinTransferEvent` is emitted as if delivery succeeded.

**Callback dispatch confirms `is_ft_transfer_call = !msg.is_empty()`:** [4](#0-3) 

```rust
.then(
    Self::ext(env::current_account_id())
        .with_static_gas(SEND_TOKENS_CALLBACK_GAS)
        .fin_transfer_send_tokens_callback(
            transfer_message,
            &fee_recipient,
            !msg.is_empty(),   // is_ft_transfer_call
            predecessor_account_id,
            lock_actions,
        ),
)
```

Any transfer without a message (`msg = ""`) is permanently vulnerable.

---

### Impact Explanation

- The transfer is recorded in `finalised_transfers` and cannot be re-submitted (replay protection blocks it).
- The tokens are never minted/transferred to the recipient.
- For native tokens locked in the bridge, they remain locked with no user-accessible recovery path.
- For deployed (bridge-minted) tokens, the mint never occurs.
- Recovery requires privileged DAO intervention via `transfer_token_as_dao`.

This matches: **Critical — Permanent freezing, irrecoverable lock, or unclaimable settlement of user or protocol funds in bridge, token, or vault flows.**

---

### Likelihood Explanation

NEAR hosts USDC.e and other tokens that implement operator-controlled transfer restrictions. The scenario requires:

1. A token with a rejection mechanism (blacklist, freeze, or custom `ft_transfer` guard).
2. The recipient's account being in a rejected state at the time the relayer submits `fin_transfer`.

This is directly analogous to the reported Clober OrderBook issue: a user-controlled address (the recipient) can be placed in a state where token delivery is impossible, and the protocol has no fallback. The attacker does not need any privileged role — they only need to be the recipient (or cause the recipient to be blacklisted) of a transfer involving a restrictive token.

---

### Recommendation

In `fin_transfer_send_tokens_callback`, check the promise result for plain `ft_transfer` failures, not only for `ft_transfer_call`. Specifically:

```rust
fn is_refund_required(is_ft_transfer_call: bool) -> bool {
    if is_ft_transfer_call {
        // existing logic
    } else {
        // NEW: treat a failed ft_transfer promise as requiring refund
        env::promise_result_checked(0, usize::MAX).is_err()
    }
}
```

Alternatively, restructure `fin_transfer_send_tokens_callback` to explicitly inspect `env::promise_result_checked(0, ...)` for both transfer types and revert finalization on any failure.

---

### Proof of Concept

1. User initiates a transfer of USDC.e from EVM → NEAR, recipient = `alice.near`, no message.
2. `alice.near` is blacklisted by the USDC.e contract (e.g., regulatory freeze).
3. Relayer calls `fin_transfer` on NEAR.
4. `fin_transfer_callback` → `process_fin_transfer_to_near`:
   - `add_fin_transfer(transfer_id)` — transfer permanently recorded as finalized. [5](#0-4) 
5. `send_tokens(token, alice.near, amount, "")` dispatches `ft_transfer(alice.near, amount)` on USDC.e.
6. USDC.e panics: `alice.near` is blacklisted — promise result = `Err`.
7. `fin_transfer_send_tokens_callback` fires with `is_ft_transfer_call = false`.
8. `is_refund_required(false)` → `false`. [6](#0-5) 
9. Callback enters the `else` branch, emits `FinTransferEvent`.
10. `alice.near` receives nothing. Transfer ID is finalized. Tokens are permanently locked in the bridge. No user-accessible recovery exists.

    pub fn fin_transfer_send_tokens_callback(
        &mut self,
        #[serializer(borsh)] transfer_message: TransferMessage,
        #[serializer(borsh)] fee_recipient: &AccountId,
        #[serializer(borsh)] is_ft_transfer_call: bool,
        #[serializer(borsh)] storage_owner: &AccountId,
        #[serializer(borsh)] lock_actions: Vec<LockAction>,
    ) {
        let token = self.get_token_id(&transfer_message.token);

        if Self::is_refund_required(is_ft_transfer_call) {
            self.burn_tokens_if_needed(
                token.clone(),
                U128(
                    transfer_message
                        .amount_without_fee()
                        .near_expect(BridgeError::InvalidFee),
                ),
            );

            self.revert_lock_actions(&lock_actions);

            self.remove_fin_transfer(&transfer_message.get_transfer_id(), storage_owner);

            env::log_str(
                &OmniBridgeEvent::FailedFinTransferEvent { transfer_message }.to_log_string(),
            );
        } else {
            // Send fee to the fee recipient
            if transfer_message.fee.fee.0 > 0 {
                if self.is_deployed_token(&token) {
                    ext_token::ext(token)
                        .with_static_gas(MINT_TOKEN_GAS)
                        .mint(fee_recipient.clone(), transfer_message.fee.fee, None)
                        .detach();
                } else {
                    ext_token::ext(token)
                        .with_attached_deposit(ONE_YOCTO)
                        .with_static_gas(FT_TRANSFER_GAS)
                        .ft_transfer(fee_recipient.clone(), transfer_message.fee.fee, None)
                        .detach();
                }
            }

            if transfer_message.fee.native_fee.0 > 0 {
                let native_token_id = self.get_native_token_id(transfer_message.get_origin_chain());

                ext_token::ext(native_token_id)
                    .with_static_gas(MINT_TOKEN_GAS)
                    .mint(fee_recipient.clone(), transfer_message.fee.native_fee, None)
                    .detach();
            }

            env::log_str(&OmniBridgeEvent::FinTransferEvent { transfer_message }.to_log_string());
        }
    }

    fn is_refund_required(is_ft_transfer_call: bool) -> bool {
        if is_ft_transfer_call {
            match env::promise_result_checked(0, MAX_FT_TRANSFER_CALL_RESULT) {
                Ok(value) => {
                    if let Ok(amount) = near_sdk::serde_json::from_slice::<U128>(&value) {
                        // Normal case: refund if the used token amount is zero
                        // The amount can be zero if the `ft_on_transfer` in the receiver contract returns an amount instead of `0`, or if it panics.
                        amount.0 == 0

                        // Unexpected case: don't refund
                        false
                    }
                }
                // Unexpected case: don't refund
                Err(_) => false,
            }
        } else {
            // Not ft_transfer_call: don't refund
