Audit Report

## Title
Stranded Router ETH Silently Subsidizes Subsequent User's WETH Input Payment — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary

`PeripheryPayments.pay()` uses `address(this).balance` — the entire router's ETH balance — to determine how much native ETH to wrap as WETH when processing a WETH payment. Any ETH left on the router from a prior payable call (e.g., excess `msg.value` not refunded) is silently consumed to fund a later user's swap input. The prior user loses that ETH with no recourse; the later user pays less than owed.

## Finding Description

In `pay()`, the WETH branch reads the global contract balance: [1](#0-0) 

`nativeBalance` at line 74 is `address(this).balance` — not scoped to the current caller's `msg.value`. If `nativeBalance >= value`, the entire `value` is sourced from the router's ETH balance with zero pull from the actual `payer`.

ETH strands on the router whenever a caller sends `msg.value > amountIn` to any payable entry point (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`, `multicall`) without appending a `refundETH()` call. None of these entry points issue an automatic refund: [2](#0-1) 

`refundETH()` is opt-in only: [3](#0-2) 

The `receive()` guard only blocks direct ETH pushes from non-WETH addresses; it does not prevent ETH accumulation via `msg.value` in any payable entry point: [4](#0-3) 

`multicall` is `payable` and loops via `delegatecall` with no automatic refund: [5](#0-4) 

**Exploit flow:**
1. Victim calls `exactInputSingle{value: 2 ether}(... amountIn: 1 ether, tokenIn: WETH ...)` without appending `refundETH()`. The callback fires `pay()` with `value = 1 ETH`; `nativeBalance = 2 ETH >= 1 ETH`, so 1 ETH is wrapped and sent to the pool. 1 ETH remains stranded on the router.
2. Attacker calls `exactInputSingle{value: 0}(... amountIn: 1 ether, tokenIn: WETH ...)`. The callback fires `pay()` with `value = 1 ETH`; `nativeBalance = 1 ETH >= 1 ETH`, so the stranded ETH is wrapped and sent to the pool. Zero is pulled from the attacker's wallet.

## Impact Explanation

Direct loss of user principal. The victim's stranded ETH is wrapped into WETH and transferred to a pool on behalf of an unrelated user. The victim receives nothing in return. This meets the "Critical/High direct loss of user principal" threshold under the allowed impact gate.

## Likelihood Explanation

No privileged access, malicious pool, or non-standard token is required. Stranding ETH is a common user error (sending excess `msg.value` without `refundETH()`). An attacker can monitor the router's ETH balance on-chain and immediately follow with a WETH `exactInputSingle{value: 0}` to drain it. The attack is repeatable and permissionless.

## Recommendation

Scope the native ETH available for WETH wrapping to the current transaction's `msg.value`, not the global contract balance. Options include:
- Track consumed `msg.value` in transient storage per top-level call and use only that amount in `pay()`.
- Automatically call `refundETH()` (or equivalent) at the end of every payable swap entry point.
- At the start of each top-level entry point, assert `address(this).balance == msg.value` to reject calls when pre-existing ETH is present.

## Proof of Concept

```solidity
function test_strandedEthSubsidizesWethSwap() public {
    // Step 1: Victim sends 2 ETH for a 1 ETH WETH swap, forgets refundETH
    vm.prank(victim);
    router.exactInputSingle{value: 2 ether}(
        ExactInputSingleParams({tokenIn: WETH, amountIn: 1 ether, ...})
    );
    assertEq(address(router).balance, 1 ether); // 1 ETH stranded

    // Step 2: Attacker swaps WETH input, sends 0 ETH
    uint256 attackerWethBefore = IERC20(WETH).balanceOf(attacker);
    vm.prank(attacker);
    router.exactInputSingle{value: 0}(
        ExactInputSingleParams({tokenIn: WETH, amountIn: 1 ether, ...})
    );

    // Attacker's WETH balance unchanged — stranded ETH paid for the swap
    assertEq(IERC20(WETH).balanceOf(attacker), attackerWethBefore);
    assertEq(address(router).balance, 0); // Router ETH drained
    // Victim lost 1 ETH with no recourse
}
```

In `pay()`, when `nativeBalance = 1 ETH` and `value = 1 ETH`, the `nativeBalance >= value` branch at line 75 fires, wraps the stranded ETH, and transfers it to the pool — pulling zero from the attacker's wallet.
