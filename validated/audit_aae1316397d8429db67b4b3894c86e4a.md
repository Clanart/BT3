### Title
Missing Authority Check in `execute_meta_tx_v0` Allows Arbitrary `__execute__` Invocation on Any Account — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_meta_tx_v0` syscall in the StarkNet OS calls `__execute__` on a caller-supplied `contract_address` without verifying that the invoking contract has any authority over that address, and without invoking `__validate__` on the target. Because standard StarkNet account contracts perform signature verification exclusively in `__validate__` (not in `__execute__`), any unprivileged contract can call `__execute__` on any victim account with attacker-controlled calldata, bypassing signature verification entirely and draining the victim's funds.

---

### Finding Description

In `execute_meta_tx_v0` (lines 286–400 of `syscall_impls.cairo`), the OS:

1. Reads `contract_address` directly from the caller-supplied `MetaTxV0Request` with no authority check.
2. Enforces only that `selector == EXECUTE_ENTRY_POINT_SELECTOR` (line 317–319) — i.e., it hard-requires calling `__execute__`.
3. Constructs a new `TxInfo` with `account_contract_address = contract_address`, `version = 0`, `nonce = 0`, and the attacker-supplied `signature_start/end`.
4. Calls `__execute__` on `contract_address` directly via `contract_call_helper`, **never calling `__validate__`**.
5. Performs no check that the calling contract has any relationship to `contract_address`. [1](#0-0) 

The meta-tx hash is computed over `(contract_address, selector, calldata, chain_id)` — the nonce is hardcoded to `0` and is excluded from the hash: [2](#0-1) 

The resulting `TxInfo` passed to the target's `__execute__` is: [3](#0-2) 

The execution context sets `caller_address = ORIGIN_ADDRESS` (0) and calls `__execute__` directly: [4](#0-3) 

The analog to the original report is exact: just as `liquidate_position` sent funds to an `owner_token_account` without checking its authority against the position owner, `execute_meta_tx_v0` invokes `__execute__` on a `contract_address` without checking the caller's authority over it. The OS is the necessary vulnerable step — it is the only entity that can invoke `__execute__` with a synthetic `TxInfo` that bypasses the normal `__validate__` flow.

---

### Impact Explanation

**Critical — Direct loss of funds.**

Standard StarkNet account contracts (following the account abstraction model) perform signature verification in `__validate__`, not in `__execute__`. The `__execute__` entry point simply iterates over the provided calls and dispatches them. By calling `__execute__` directly with attacker-controlled calldata (e.g., an ERC-20 `transfer` to the attacker's address), the attacker can drain any victim account's token balances without possessing the victim's private key.

---

### Likelihood Explanation

**High.** The attack path requires only:
- Deploying a contract (permissionless on StarkNet).
- Submitting a transaction that calls the attacker's contract (permissionless).
- The attacker's contract invokes `execute_meta_tx_v0` targeting any victim account.

No privileged role, leaked key, or operator cooperation is required. The syscall is available to any executing contract.

---

### Recommendation

Add an authority check before invoking `__execute__` on the target contract. Specifically, the OS should verify that the calling contract (`caller_execution_info.contract_address`) is authorized to act on behalf of `contract_address`. One approach is to require that `caller_execution_info.contract_address == contract_address` (i.e., a contract can only issue a meta-tx on behalf of itself), or alternatively, invoke `__validate__` on the target contract before `__execute__` to enforce the target's own signature verification logic.

---

### Proof of Concept

1. Attacker deploys `MaliciousRelayer` — a contract with a function that calls the `execute_meta_tx_v0` syscall.
2. Attacker submits an `invoke` transaction calling `MaliciousRelayer.attack(victim_account, transfer_calldata)`.
3. `MaliciousRelayer` issues `execute_meta_tx_v0` with:
   - `contract_address = victim_account`
   - `selector = EXECUTE_ENTRY_POINT_SELECTOR`
   - `calldata = [1, transfer_selector, 3, attacker_address, amount_low, amount_high]`
   - `signature = []` (empty)
4. The OS constructs `TxInfo { version=0, account_contract_address=victim_account, nonce=0, signature=[] }` and calls `victim_account.__execute__(calldata)`.
5. `victim_account.__execute__` dispatches the `transfer` call — no signature check occurs because `__validate__` was never called.
6. The victim's tokens are transferred to the attacker. [5](#0-4)

    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, caller_execution_context: ExecutionContext*) {
    alloc_locals;

    let request = cast(syscall_ptr + RequestHeader.SIZE, MetaTxV0Request*);
    local calldata_start: felt* = request.calldata_start;
    local calldata_size = request.calldata_end - calldata_start;

    let specific_base_gas_cost = (
        META_TX_V0_GAS_COST + META_TX_V0_CALLDATA_FACTOR_GAS_COST * calldata_size
    );
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=specific_base_gas_cost, request_struct_size=MetaTxV0Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    local contract_address = request.contract_address;
    local selector = request.selector;
    local caller_execution_info: ExecutionInfo* = caller_execution_context.execution_info;
    local old_tx_info: TxInfo* = caller_execution_info.tx_info;

    if (selector != EXECUTE_ENTRY_POINT_SELECTOR) {
        write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
        return ();
    }

    // Sanity check: Verify that `signature` is a valid Sierra array.
    assert_nn_le(request.signature_end - request.signature_start, SIERRA_ARRAY_LEN_BOUND - 1);

    let (state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
        key=contract_address

    let pedersen_ptr = builtin_ptrs.selectable.pedersen;
    with pedersen_ptr {
        let meta_tx_hash = compute_meta_tx_v0_hash(
            contract_address=contract_address,
            entry_point_selector=selector,
            calldata=calldata_start,
            calldata_size=calldata_size,
            chain_id=old_tx_info.chain_id,
        );
    }
    update_pedersen_in_builtin_ptrs(pedersen_ptr=pedersen_ptr);

    // Prepare execution context.
    tempvar new_tx_info = new TxInfo(
        version=0,
        account_contract_address=contract_address,
        max_fee=0,
        signature_start=request.signature_start,
        signature_end=request.signature_end,
        transaction_hash=meta_tx_hash,
        chain_id=old_tx_info.chain_id,
        nonce=0,
        resource_bounds_start=cast(0, ResourceBounds*),
        resource_bounds_end=cast(0, ResourceBounds*),
        tip=0,
        paymaster_data_start=cast(0, felt*),
        paymaster_data_end=cast(0, felt*),
        nonce_data_availability_mode=0,
        fee_data_availability_mode=0,
        account_deployment_data_start=cast(0, felt*),
        account_deployment_data_end=cast(0, felt*),
        proof_facts_start=cast(0, felt*),
        proof_facts_end=cast(0, felt*),
    );

    let (deprecated_tx_info_ptr: DeprecatedTxInfo*) = alloc();
    tempvar execution_context: ExecutionContext* = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=state_entry.class_hash,
        calldata_size=calldata_size,
        calldata=calldata_start,
        execution_info=new ExecutionInfo(
            block_info=caller_execution_info.block_info,
            tx_info=new_tx_info,
            caller_address=ORIGIN_ADDRESS,
            contract_address=contract_address,
            selector=selector,
        ),
        deprecated_tx_info=deprecated_tx_info_ptr,
    );
    fill_deprecated_tx_info(tx_info=new_tx_info, dst=execution_context.deprecated_tx_info);

    // Since we process the revert log backwards, entries before this point belong to the calling

    let revert_log = &revert_log[1];

        remaining_gas=remaining_gas,
        block_context=block_context,
        execution_context=execution_context,
    );

    // Entries before this point belong to the callee.
    assert [revert_log] = RevertLogEntry(selector=CHANGE_CONTRACT_ENTRY, value=contract_address);
    let revert_log = &revert_log[1];
