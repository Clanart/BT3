### Title
`ContractUpdates` from implicit state transitions silently discarded — validators never notified of contracts accessed during missing chunks - (File: chain/chain/src/stateless_validation/state_witness.rs)

### Summary

`collect_state_transition_data()` reads `ContractUpdates` (contract accesses and deploys) from every implicit transition stored in `DBCol::StateTransitionData`, but discards them with the `_` wildcard. Only the **main** transition's `ContractUpdates` is forwarded to validators via `ChunkContractAccesses`. When `ExcludeContractCodeFromStateWitness` is active, contract code is stripped from the implicit transitions' `base_state`, so validators that lack those contracts in their compiled-contract cache cannot reconstruct the correct post-state-root and will produce invalid endorsements or fail validation entirely.

### Finding Description

`get_state_transition()` returns a three-tuple `(ChunkStateTransition, CryptoHash, ContractUpdates)`. The `ContractUpdates` field carries the set of contract code hashes accessed (and any newly deployed contracts) during that block's chunk application. This data is persisted to `DBCol::StateTransitionData` for every block.

In `collect_state_transition_data()`, the loop that builds `implicit_transitions` calls `get_state_transition()` twice per block (once for a resharding cross-shard transition, once for the regular implicit transition), but in both call sites the `ContractUpdates` is thrown away:

```rust
// line 135-141 (resharding branch)
let (chunk_state_transition, _, _) = self.get_state_transition(
    epoch_manager, &current_block_hash, &next_epoch_id, next_shard_id,
)?;
implicit_transitions.push(chunk_state_transition);

// line 151-157 (regular implicit branch)
let (chunk_state_transition, _, _) = self.get_state_transition(
    epoch_manager, &current_block_hash, &current_epoch_id, current_shard_id,
)?;
implicit_transitions.push(chunk_state_transition);
```

Only the main transition's `ContractUpdates` survives into `StateTransitionData` and then into `CreateWitnessResult`:

```rust
Ok(StateTransitionData {
    ...
    contract_updates,   // main transition only
})
```

`handle_distribute_state_witness_request()` then sends `ChunkContractAccesses` and `PartialEncodedContractDeploys` exclusively from this single `contract_updates` field, leaving every implicit transition's contract accesses and deploys unannounced.

When `ExcludeContractCodeFromStateWitness` is enabled, `record_contract_call()` skips writing the contract bytecode into the trie recording for **all** chunk applications, including implicit (old-chunk) ones. The bytecode is therefore absent from `ChunkStateTransition.base_state` for every implicit transition. Validators must obtain it via the `ChunkContractAccesses` side-channel — but that side-channel is never populated for implicit transitions.

### Impact Explanation

A chunk validator that does not have the relevant contract in its compiled-contract cache will attempt to apply the implicit transition, fail to find the contract bytecode in the recorded partial state, and produce an incorrect post-state-root. This causes the validator to either:

- emit a chunk endorsement for a wrong state root (breaking the stateless-validation safety guarantee), or
- reject the witness entirely, withholding its endorsement and potentially stalling block production if enough validators are affected.

The corrupted value is the `post_state_root` field of the `ChunkStateTransition` for the implicit transition: the validator computes a root that diverges from the chunk producer's because the contract code is missing from the replay.

### Likelihood Explanation

The preconditions are:

1. `ProtocolFeature::ExcludeContractCodeFromStateWitness` is active (a protocol-version gate, not operator configuration).
2. At least one missing chunk exists between two consecutive new chunks of a shard (common under network stress or adversarial chunk withholding).
3. A delayed receipt or incoming receipt processed during that missing chunk invokes a `FunctionCallAction` against a contract not present in the validator's compiled-contract cache (e.g., a recently deployed contract, or a validator that restarted and lost its cache).

Conditions 2 and 3 are reachable by any unprivileged user: submitting a function-call transaction that produces a delayed receipt, then causing a missing chunk on the target shard (e.g., by targeting a chunk producer that is temporarily offline). No privileged role is required.

### Recommendation

Accumulate `ContractUpdates` across all implicit transitions and merge them with the main transition's `ContractUpdates` before constructing `CreateWitnessResult`. Concretely, change the implicit-transition loop to collect the returned `ContractUpdates`:

```rust
let (chunk_state_transition, _, implicit_contract_updates) = self.get_state_transition(...)?;
implicit_transitions.push(chunk_state_transition);
// merge implicit_contract_updates into an accumulated ContractUpdates
```

Then union the accumulated implicit `contract_accesses` and `contract_deploys` into the main transition's `ContractUpdates` before returning `StateTransitionData`. This ensures `handle_distribute_state_witness_request()` sends `ChunkContractAccesses` covering contracts from every transition in the witness.
