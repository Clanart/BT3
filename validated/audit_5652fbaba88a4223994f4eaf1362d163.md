### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Fund Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not validate that the requested new class hash corresponds to a previously declared contract class. A contract can replace its own class hash with an arbitrary, undeclared felt value. Once replaced, all future calls to that contract will fail because no executable class exists for the hash. Any funds held by the contract are permanently frozen. The developers themselves flagged this gap with a TODO comment that has not been resolved.

---

### Finding Description

In `execute_replace_class` (lines 878–916 of `syscall_impls.cairo`), the OS reads the requested `class_hash` directly from the syscall request and writes it into `contract_state_changes` without any check against the `contract_class_changes` dictionary (which tracks declared classes):

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

Critically, `execute_replace_class` does not receive `contract_class_changes` as an implicit argument — its signature only carries `contract_state_changes` and `revert_log`. There is therefore no mechanism within the function to verify that `class_hash` is a key present in the class tree. [2](#0-1) 

Compare this with `execute_declare_transaction` in `transaction_impls.cairo`, which enforces `prev_value=0` to guarantee a class is declared at most once and that `compiled_class_hash` is non-zero before writing to `contract_class_changes`:

```cairo
// Note that prev_value=0 enforces that a class may be declared only once.
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [3](#0-2) 

No equivalent guard exists in `execute_replace_class`. The OS will accept and commit any felt as the new class hash.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is overwritten with an undeclared value, the contract is permanently non-executable. The `StateEntry` struct stores `class_hash` as the sole pointer to the contract's logic:

```cairo
struct StateEntry {
    class_hash: felt,
    storage_ptr: DictAccess*,
    nonce: felt,
}
``` [4](#0-3) 

If `class_hash` points to nothing in the class tree, every subsequent call to the contract — including any withdrawal or transfer function — will fail at the entry-point lookup stage. There is no recovery path: `replace_class` itself requires the contract to be callable, and the contract is no longer callable. All ERC-20 balances, ETH, or other assets held in the contract's storage are permanently inaccessible.

---

### Likelihood Explanation

**Medium.**

The attack requires a contract to issue the `replace_class` syscall with an undeclared hash. Two realistic paths exist:

1. **Malicious deployer:** An attacker deploys a contract that appears legitimate (e.g., a token vault or DEX pool), attracts user deposits, and then calls `replace_class` with an arbitrary undeclared felt. The OS commits the state change without objection. All deposited funds are frozen.

2. **Vulnerable legitimate contract:** A legitimate contract exposes a governance or upgrade function that accepts a caller-supplied class hash and passes it to `replace_class`. An attacker supplies an undeclared hash. The OS does not catch the error; the contract is bricked.

Both paths are reachable by an unprivileged transaction sender with no special privileges beyond the ability to deploy or interact with contracts.

---

### Recommendation

Pass `contract_class_changes` as an implicit argument to `execute_replace_class` and assert that the requested `class_hash` is present as a key in that dictionary (i.e., `new_value != 0` after a `dict_read`). This mirrors the invariant already enforced by `execute_declare_transaction`. Remove the TODO comment only after the check is implemented and tested.

---

### Proof of Concept

1. Attacker deploys `VaultContract` with a `withdraw` function and a `upgrade(new_hash: felt)` function that calls `replace_class(new_hash)`.
2. Users deposit 1000 STRK into `VaultContract`.
3. Attacker calls `upgrade(0xdeadbeef)` — a felt that has never been declared as a class.
4. `execute_replace_class` runs, reads `class_hash = 0xdeadbeef` from the request, and writes:
   ```
   dict_update(contract_state_changes,
     key=vault_address,
     prev_value=<valid_state_entry>,
     new_value=StateEntry{class_hash=0xdeadbeef, ...})
   ```
   No validation occurs because `contract_class_changes` is not in scope.
5. The block is proven and accepted on L1. The state root now encodes `vault_address → class_hash=0xdeadbeef`.
6. Any subsequent call to `VaultContract` (including `withdraw`) fails at class resolution. The 1000 STRK are permanently frozen. [5](#0-4)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

        key=contract_address,
        prev_value=cast(state_entry, felt),

    );

    return ();
}
