### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not validate that the supplied `class_hash` corresponds to a previously declared contract class before permanently writing it to the contract's state. Any contract can call `replace_class` with an arbitrary, undeclared felt value as the class hash. Once committed, the contract's class pointer is irrecoverably corrupted, making the contract permanently uncallable and freezing all funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the function `execute_replace_class` (lines 878–916) reads the caller-supplied `class_hash` directly from the syscall request and writes it into `contract_state_changes` without any check that a class with that hash has ever been declared:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The developer-acknowledged `TODO` at line 898 explicitly confirms the missing check. [2](#0-1) 

The `dict_update` call at lines 906–911 is the irreversible write: once the block is proven and the state root is committed to L1, the corrupted class hash is final. [3](#0-2) 

The `execute_replace_class` handler is dispatched unconditionally from `execute_syscalls` whenever the selector matches `REPLACE_CLASS_SELECTOR`, with no pre-call guard on the class hash value: [4](#0-3) 

The analog to the external report is direct:

| External (`burnAndBridge`) | StarkNet OS (`execute_replace_class`) |
|---|---|
| `destChainId` not checked against a whitelist | `class_hash` not checked against declared classes |
| Tokens burned and bridged to an invalid chain | Contract class permanently set to an undeclared hash |
| Irrecoverable loss of funds | Contract permanently uncallable; funds frozen |

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

After `execute_replace_class` writes an undeclared `class_hash` into the state and the block is proven, every subsequent call to that contract will fail at class-lookup time because the OS cannot find an entry point table for the hash. The contract's storage (including any ERC-20 balances, ETH, or other assets it custodies) becomes permanently inaccessible. Because the state root is committed to L1 via the OS output, there is no on-chain mechanism to undo the change. [5](#0-4) 

---

### Likelihood Explanation

**Medium.**

The `replace_class` syscall is a standard, documented StarkNet syscall reachable by any deployed contract without any privileged role. An attacker can:

1. Deploy a contract that accepts user deposits (e.g., a fake vault or bridge).
2. Accumulate victim funds.
3. Call `replace_class` with an arbitrary non-zero felt (e.g., `1`) that has never been declared.
4. The OS accepts the call, the block is proven, and the contract is permanently frozen.

No leaked keys, operator collusion, or network-level attack is required. The entry path is a single unprivileged transaction from the attacker's own contract. [4](#0-3) 

---

### Recommendation

Before executing the `dict_update` in `execute_replace_class`, verify that `class_hash` is present in `contract_class_changes` (declared in the current block) or in the pre-existing state. Concretely:

```cairo
// Verify the class has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the check already enforced in `execute_declare_transaction`, which asserts `assert_not_zero(compiled_class_hash)` and uses `prev_value=0` to guarantee the class is newly declared. [6](#0-5)

### Proof of Concept

```cairo
// Attacker contract (pseudocode, Cairo 1 / Sierra)
#[starknet::contract]
mod AttackerVault {
    #[external]
    fn steal_funds(self: @ContractState) {
        // Step 1: Users have deposited tokens into this contract.
        // Step 2: Attacker calls replace_class with a non-existent class hash.
        starknet::replace_class_syscall(
            0xdeadbeef_00000000_00000000_00000000_00000000_00000000_00000000_00000001
        ).unwrap();
        // Step 3: After the block is proven, this contract is permanently uncallable.
        // All deposited funds are frozen forever.
    }
}
```

The OS's `execute_replace_class` will accept the call, write the invalid hash to `contract_state_changes`, and include it in the proven state update. No revert occurs because the only gas check is for the syscall cost itself, not for class validity. [7](#0-6)

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.

}
