### Title
Zero ETH-to-FRI Rate Bypasses Fallback Mechanism, Causing Block Production Halt with Incorrect Gas Price Committed - (File: `crates/apollo_l1_gas_price/src/eth_to_strk_oracle.rs`)

---

### Summary

`resolve_query` in `EthToStrkOracleClient` accepts a zero price from the external oracle without validation. Because `GasPrice::wei_to_fri(0)` silently returns `Ok(GasPrice(0))` instead of an error, the three-tier fallback in `get_l1_prices_in_fri_and_wei_and_conversion_rate` treats the zero-rate conversion as a success and skips both the previous-block-info fallback and the safe-default fallback. The resulting zero FRI gas prices are embedded in `ProposalInit` and propagated into block commitments, causing every block-build attempt to fail at `NonzeroGasPrice::new` for the duration of the cached zero-rate entry.

---

### Finding Description

**Root cause — `resolve_query` does not reject a zero rate:** [1](#0-0) 

```rust
let rate = u128::from_str_radix(price.trim_start_matches("0x"), 16)
    .expect("Failed to parse price as u128");
// ... only decimals are validated, rate is returned as-is
Ok(rate)   // rate == 0 is silently accepted
```

The function validates `decimals == 18` but performs no check on `rate`. A response of `{"price":"0x0","decimals":18}` passes through and is returned as `Ok(0)`.

**The zero rate is immediately cached:** [2](#0-1) 

Once cached, every subsequent call for the same quantized timestamp returns `Ok(0)` from the LRU cache without re-querying the oracle.

**`wei_to_fri(0)` silently produces `GasPrice(0)` — no error:** [3](#0-2) 

```rust
pub fn wei_to_fri(self, eth_to_fri_rate: u128) -> Result<GasPrice, StarknetApiError> {
    Ok(self
        .checked_mul_u128(eth_to_fri_rate)   // any_wei * 0 = 0 → Some(GasPrice(0))
        ...
        .checked_div(WEI_PER_ETH)            // 0 / WEI_PER_ETH = 0 → Some(GasPrice(0))
        .expect("WEI_PER_ETH must be non-zero"))
}
```

The asymmetry is confirmed by the test suite: `fri_to_wei_errors_on_conversion_rate_zero` exists, but there is no corresponding test for `wei_to_fri` with a zero rate. [4](#0-3) 

**The fallback is bypassed because the conversion returns `Ok`:** [5](#0-4) 

```rust
if let (Ok(eth_to_fri_rate), Ok(mut price_info)) = (eth_to_fri_rate, price_info) {
    ...
    let l1_gas_prices_fri_result =
        L1PricesInFri::convert_from_wei(&prices_in_wei, eth_to_fri_rate);  // eth_to_fri_rate=0
    if let Ok(prices_in_fri) = l1_gas_prices_fri_result {   // Ok(GasPrice(0)) — branch taken
        return (prices_in_fri, prices_in_wei, eth_to_fri_rate);  // returns zeros, skips fallback
    }
}
// ← previous-block-info fallback and safe-default fallback are NEVER reached
```

The comment on line 172 says "If conversion fails, leave return_value=None to try backup methods" — but the conversion does not fail; it silently succeeds with zeros.

**Zero FRI prices cause block building to fail:** [6](#0-5) 

```rust
let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;  // Err if 0
```

Every `build_proposal` call fails here for the entire cache window.

---

### Impact Explanation

The zero rate is cached for the quantized-timestamp window. During that window every proposer that queries the same oracle endpoint receives `Ok(0)`, the fallback to previous-block-info (which carries correct non-zero prices) is skipped, and `convert_to_sn_api_block_info` returns an error for every block-build attempt. Block production halts. Validators that also receive the zero rate compute zero expected FRI prices; `within_margin(0, 0, margin)` passes, so the validator accepts the malformed `ProposalInit` — but then also fails at `NonzeroGasPrice::new`, preventing execution. The net effect is a complete stall of sequencing with incorrect (zero) L1 gas prices embedded in every attempted `ProposalInit`, matching **"Critical — Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact."**

---

### Likelihood Explanation

The oracle is an external HTTP endpoint. A transient feed error, a misconfigured endpoint, or a targeted response returning `{"price":"0x0","decimals":18}` is sufficient. No privileged access to the sequencer process is required. The LRU cache amplifies the impact: a single zero response poisons the cache for the full `lag_interval_seconds` quantization window, blocking all block-build attempts during that period.

---

### Recommendation

1. **In `resolve_query`**, reject a zero rate before caching:
   ```rust
   if rate == 0 {
       return Err(EthToStrkOracleClientError::InvalidRateError("Rate must be non-zero".into()));
   }
   ```
2. **In `GasPrice::wei_to_fri`**, guard against a zero rate analogously to `fri_to_wei`:
   ```rust
   if eth_to_fri_rate == 0 {
       return Err(StarknetApiError::GasPriceConversionError(
           "ETH to FRI rate must be non-zero".to_string(),
       ));
   }
   ```
   This makes the conversion return `Err` on a zero rate, which causes `get_l1_prices_in_fri_and_wei_and_conversion_rate` to fall through to the previous-block-info fallback as intended.

### Proof of Concept

1. Stand up a mock oracle that returns `{"price":"0x0","decimals":18}` for any timestamp query.
2. Configure `EthToStrkOracleClient` to point at that mock.
3. Call `eth_to_fri_rate(timestamp)` — first call returns `QueryNotReadyError`; after the async query resolves, the second call returns `Ok(0)` and caches it.
4. Call `get_l1_prices_in_fri_and_wei_and_conversion_rate` with a valid `price_info` (non-zero WEI prices) and the zero rate. Observe that `L1PricesInFri::convert_from_wei` returns `Ok(L1PricesInFri { l1_gas_price: GasPrice(0), l1_data_gas_price: GasPrice(0) })` and the function returns immediately, skipping both fallback paths.
5. Pass the resulting zero FRI prices into `convert_to_sn_api_block_info`. Observe `NonzeroGasPrice::new(GasPrice(0))` returns `Err`, causing block building to fail.
6. Repeat step 3 for the same quantized timestamp — the cached zero rate is returned instantly, reproducing the failure on every block-build attempt for the cache window. [7](#0-6) [5](#0-4) [3](#0-2) [8](#0-7)

fn resolve_query(body: String) -> Result<u128, EthToStrkOracleClientError> {
    let Ok(json): Result<serde_json::Value, _> = serde_json::from_str(&body) else {
        return Err(EthToStrkOracleClientError::ParseError(format!(
            "Failed to parse JSON: {body}"
        )));
    };
    // Extract price from API response. Also returns MissingFieldError if value is not a string.
    let price = match json.get("price").and_then(|v| v.as_str()) {
        Some(price) => price,
        None => {
            return Err(EthToStrkOracleClientError::MissingFieldError("price".to_string(), body));
        }
    };
    let rate = u128::from_str_radix(price.trim_start_matches("0x"), 16)
        .expect("Failed to parse price as u128");
    // Extract decimals from API response. Also returns MissingFieldError if value is not a number.
    let decimals = match json.get("decimals").and_then(|v| v.as_u64()) {
        Some(decimals) => decimals,
        None => {
            return Err(EthToStrkOracleClientError::MissingFieldError(
                "decimals".to_string(),
                body,
            ));
        }
    };
    if decimals != ETH_TO_STRK_QUANTIZATION {
        return Err(EthToStrkOracleClientError::InvalidDecimalsError(
            ETH_TO_STRK_QUANTIZATION,
            decimals,
        ));
    }
    ETH_TO_STRK_SUCCESS_COUNT.increment(1);
    set_unix_now_seconds(&ETH_TO_STRK_LAST_SUCCESS_TIMESTAMP_SECONDS);
    ETH_TO_STRK_RATE.set_lossy(rate);
    Ok(rate)
}
