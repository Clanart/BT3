### Title
`ProofOfInclusion::valid()` Validates Only Internal Consistency, Never Checks Against a Trusted External Root — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` in the DataLayer Merkle proof subsystem only verifies that the hash chain within the proof is internally self-consistent. It never accepts or compares against a caller-supplied trusted root hash. Because `root_hash()` is derived entirely from the proof's own fields, the final equality check inside `valid()` is tautologically true whenever the loop completes. Any attacker who can supply a `ProofOfInclusion` object — trivially possible via the `Streamable` deserialization path or the Python/wasm binding — can forge a proof that passes `valid()` while proving inclusion in a completely fabricated tree.

### Finding Description

`ProofOfInclusion` is a `Streamable` struct exposed to Python via `pyclass` and `PyStreamable`:

```rust
#[derive(Clone, Debug, std::hash::Hash, Eq, PartialEq, Streamable)]
pub struct ProofOfInclusion {
    pub node_hash: Hash,
    pub layers: Vec<ProofOfInclusionLayer>,
}
``` [1](#0-0) 

The `valid()` method iterates over layers, checking that each `calculated_hash` equals `layer.combined_hash`, then sets `existing_hash = calculated_hash`. After the loop it checks `existing_hash == self.root_hash()`:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash, layer.other_hash_side, &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash { return false; }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()   // ← always true
}
``` [2](#0-1) 

`root_hash()` returns `last.combined_hash` when layers are non-empty:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash
    }
}
``` [3](#0-2) 

After the loop, `existing_hash` holds the last `calculated_hash`, which equals the last `layer.combined_hash` (the loop would have returned `false` otherwise). `root_hash()` also returns `last.combined_hash`. Therefore `existing_hash == self.root_hash()` is **always true** — the final check is dead code. `valid()` reduces to: "is the hash chain internally self-consistent?" with no reference to any external trusted root.

The Python binding exposes both `valid()` and `root_hash()` as separate, independent methods with no enforcement that callers compare `root_hash()` against a known trusted root:

```python
def root_hash(self) -> bytes32: ...
def valid(self) -> bool: ...
``` [4](#0-3) 

Every caller in the codebase — Rust tests, Python tests, and the fuzz target — calls only `proof.valid()` without comparing `proof.root_hash()` against the blob's actual root:

Contrast this with the consensus-layer `validate_merkle_proof`, which correctly requires a trusted root parameter and rejects any proof whose computed root does not match:

```rust
pub fn validate_merkle_proof(proof: &[u8], item: &[u8; 32], root: &[u8; 32]) -> Result<bool, SetError> {
    let tree = MerkleSet::from_proof(proof)?;
    if tree.get_root() != *root { return Err(SetError); }
    Ok(tree.generate_proof(item)?.0)
}
``` [8](#0-7) 

The DataLayer `ProofOfInclusion::valid()` has no equivalent guard.

### Impact Explanation

A DataLayer peer or client that receives a `ProofOfInclusion` over the network (deserialized via `Streamable::from_bytes` or the Python `from_bytes` binding) and calls `valid()` will accept any internally consistent proof as genuine, regardless of which tree root it actually corresponds to. An attacker can:

1. Construct a `ProofOfInclusion` with an arbitrary `node_hash` (the key-value pair they wish to falsely prove is present).
2. Build one or more `ProofOfInclusionLayer` entries with attacker-chosen `other_hash` and `combined_hash = calculate_internal_hash(node_hash, side, other_hash)`.
3. Serialize and transmit the proof.
4. Any receiver calling only `proof.valid()` accepts the forged inclusion proof.

This lets untrusted input prove invalid DataLayer state — matching the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.**

### Likelihood Explanation

`ProofOfInclusion` is `Streamable` and fully exposed via Python bindings, making it trivially constructable by any network peer. The API design of `valid()` — named to imply complete validation — actively encourages callers to omit the separate `root_hash()` comparison. Every existing caller in the codebase already omits it. Any DataLayer synchronization or verification code that follows the same pattern is immediately exploitable.

### Recommendation

`valid()` must accept a trusted external root hash and compare against it, rather than deriving the root from the proof itself:

```rust
pub fn valid(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against caller-supplied trusted root
}
```

The Python binding should mirror this signature. All callers must supply the root hash obtained from a trusted source (e.g., the on-chain coin commitment, not the proof itself). The standalone `root_hash()` accessor should be clearly documented as returning an untrusted value derived from proof-internal data only.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
import hashlib

# Attacker-chosen leaf hash (the key-value pair they want to falsely prove)
fake_node_hash = bytes(range(32))

# Attacker-chosen sibling hash
fake_other_hash = bytes(range(32, 64))

# Compute a valid combined_hash using the same internal hash function
# (SHA-256 of \x02 + left + right, per DataLayer convention)
combined = hashlib.sha256(b"\x02" + fake_node_hash + fake_other_hash).digest()

layer = ProofOfInclusionLayer(
    other_hash_side=1,          # Right
    other_hash=fake_other_hash,
    combined_hash=combined,
)

forged_proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

# valid() returns True for a completely fabricated proof
assert forged_proof.valid(), "Forged proof accepted!"

# root_hash() returns the attacker-controlled combined_hash, not any real tree root
print("Forged root:", forged_proof.root_hash().hex())
```

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
