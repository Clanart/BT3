### Title
Missing Class Hash Validity Check Allows Permanent Freezing of Contract Funds — (`File: execution/syscall_impls.cairo` + `execution/execute_entry_point.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not validate that the replacement class hash corresponds to a declared contract class. Combined with the absence of a guard in `execute_entry_point` before calling `find_element`, this allows any contract to replace its own class with an undeclared or zero hash. Once replaced, the contract becomes permanently uncallable at the OS/proving layer, freezing any funds it holds.

---

### Finding Description

**Root cause 1 — `execute_replace_class` (syscall_impls.cairo, line 896–916):**

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;
    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}
    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );
    dict_update{dict_ptr=contract_state_changes}(...);
    ...
}
```

The developer's own TODO comment acknowledges the missing check. `class_hash` from the request is written directly into the contract state with no validation that it corresponds to any declared class. An unprivileged user can supply `class_hash = 0` or any arbitrary undeclared felt value.

**Root cause 2 — `execute_entry_point` (execute_entry_point.cairo, lines 154–166):**

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
```

`dict_read` returns `0` (the default dict value) when `execution_context.class_hash` is `0` or maps to an undeclared hash. `find_element` is then called with `key = 0` (or any undeclared hash). Cairo's `find_element` raises a `ValueError` and aborts proof generation if the key is absent from the array. There is **no guard** between the dict read and the `find_element` call — no check for `compiled_class_hash == 0`, no graceful failure path analogous to the `ERROR_ENTRY_POINT_NOT_FOUND` path that exists for missing selectors.

This is the direct analog to the EVM report: just as a Solidity `try/catch` does not catch reverts when `codesize == 0`, the OS's graceful revert handling (the `is_reverted` / `failure_flag` path) is never reached when `find_element` aborts — the abort happens before the entry point is even called.

---

### Impact Explanation

Once a contract's class hash is replaced with an undeclared value:

- Every subsequent call to that contract (via `call_contract`, `library_call`, or direct transaction) reaches `execute_entry_point`, which calls `find_element` with the undeclared hash.
- `find_element` aborts proof generation. The sequencer cannot produce a valid proof for any block containing such a call.
- The sequencer is therefore forced to permanently exclude all calls to that contract from every block.
- Any ERC-20 tokens, ETH, or other assets held by the contract are **permanently frozen** — no withdrawal, transfer, or administrative function can ever be proven.

This satisfies the **Critical — Permanent freezing of funds** impact criterion.

---

### Likelihood Explanation

- The `replace_class` syscall is a standard, publicly documented StarkNet syscall callable by any contract from its own execution context.
- No privilege, admin key, or external dependency is required. Any deployed contract can invoke it.
- The missing check is explicitly flagged with a `TODO` comment, confirming it is an unimplemented guard, not an intentional design choice.
- A single transaction suffices to trigger the permanent freeze.

---

### Recommendation

**In `execute_replace_class`:** Before writing the new class hash to state, verify that `class_hash` is non-zero and that it exists in `contract_class_changes` (i.e., it has been declared). Remove the TODO and enforce the check:

```cairo
// Ensure the replacement class has been declared.
assert_not_zero(class_hash);
let (declared_compiled_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(declared_compiled_hash);
```

**In `execute_entry_point`:** Add a guard before `find_element` to handle the `compiled_class_hash == 0` case gracefully (return `is_reverted=1` with an appropriate error code) rather than allowing `find_element` to abort:

```cairo
if (compiled_class_hash == 0) {
    %{ ExitCall %}
    let (retdata: felt*) = alloc();
    assert retdata[0] = ERROR_ENTRY_POINT_NOT_FOUND;
    return (is_reverted=1, retdata_size=1, retdata=retdata);
}
```

---

### Proof of Concept

1. Attacker deploys `VaultContract` holding user funds. The contract exposes a `replace_class` call path (e.g., via an admin function or a reentrancy vector).
2. Attacker invokes `replace_class` with `class_hash = 0` (or any felt not present in the declared class set).
3. The OS executes `execute_replace_class` at [1](#0-0) , writes `class_hash = 0` into `contract_state_changes` with no validation.
4. Any subsequent call to `VaultContract` reaches `execute_entry_point` at [2](#0-1) .
5. `dict_read` returns `compiled_class_hash = 0`. `find_element` is called with `key = 0`. Since no compiled class with hash `0` exists in `compiled_class_facts_bundle`, `find_element` raises `ValueError: Element with key 0 was not found` and aborts proof generation.
6. The sequencer cannot include any call to `VaultContract` in any block. All funds are permanently frozen.
