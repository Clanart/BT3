### Title
STRK Native Fees Permanently Locked in StarkNet Bridge Contract with No Withdrawal Mechanism - (File: `starknet/src/omni_bridge.cairo`)

---

### Summary

Every call to `init_transfer` on the StarkNet bridge that includes a non-zero `native_fee` transfers STRK tokens into the bridge contract itself. The `IOmniBridge` interface exposes no function to withdraw or distribute these accumulated STRK tokens to relayers. The fees are permanently irrecoverable from the contract under the current code.

---

### Finding Description

In `starknet/src/omni_bridge.cairo`, the `init_transfer` function handles the optional `native_fee` parameter as follows:

```cairo
if native_fee > 0 {
    let native_token = self.strk_token_address.read();
    let success = IERC20Dispatcher { contract_address: native_token }
        .transfer_from(caller, get_contract_address(), native_fee.into());
    assert(success, 'ERR_FEE_TRANSFER_FAILED');
}
``` [1](#0-0) 

The STRK tokens are transferred to `get_contract_address()` — the bridge contract itself. The entire `IOmniBridge` trait, however, contains no function for withdrawing or distributing these accumulated fees: [2](#0-1) 

The only functions exposed are `log_metadata`, `deploy_token`, `fin_transfer`, `init_transfer`, `upgrade_token`, `set_pause_flags`, `pause_all`, `get_token_address`, `is_bridge_token`, and `is_transfer_finalised`. None of these allow retrieval of the accumulated STRK native fees.

On the NEAR side, when a StarkNet-origin transfer is finalized, `send_fee_internal` mints wrapped STRK tokens to the relayer:

```rust
ext_token::ext(self.get_native_token_id(origin_chain))
    .with_static_gas(MINT_TOKEN_GAS)
    .mint(fee_recipient.clone(), transfer_message.fee.native_fee, None)
    .detach();
``` [3](#0-2) 

This means the relayer receives freshly minted wrapped STRK on NEAR, while the actual STRK tokens paid by the user on StarkNet remain permanently locked in the bridge contract. There is no on-chain path to release them.

---

### Impact Explanation

**Critical — Permanent freezing / irrecoverable lock of user funds.**

Every user who calls `init_transfer` with `native_fee > 0` on StarkNet permanently contributes STRK tokens to a pool that cannot be retrieved by any party (relayer, admin, or user) under the current contract code. The STRK tokens are not burned, not forwarded to a relayer, and not tracked in any accounting structure — they simply accumulate in the contract balance with no exit path. This constitutes an irrecoverable lock of user-paid protocol funds in the bridge flow.

---

### Likelihood Explanation

**High.** The README explicitly documents that transfers support native chain token fees (e.g., STRK on StarkNet) as a first-class fee option. Any user or relayer-assisted transfer that sets `native_fee > 0` triggers this lock. Since relayers are incentivized to request native fees for gas compensation, this path is expected to be exercised routinely in production.

---

### Recommendation

Add a privileged withdrawal function to the `IOmniBridge` interface and its implementation that allows an authorized role (e.g., `DEFAULT_ADMIN_ROLE` or a dedicated `FEE_COLLECTOR_ROLE`) to transfer accumulated STRK native fees out of the contract to the intended fee recipient or a fee distribution contract. For example:

```cairo
fn withdraw_native_fees(ref self: TContractState, recipient: ContractAddress, amount: u128);
```

This function should be gated behind `assert_only_role(DEFAULT_ADMIN_ROLE)` or a dedicated role, and should transfer the specified amount of `strk_token_address` tokens to `recipient`.

---

### Proof of Concept

1. User calls `init_transfer` on the StarkNet bridge with `native_fee = 1_000_000` (1 STRK unit).
2. The bridge executes `transfer_from(caller, get_contract_address(), 1_000_000)` — STRK is now held by the bridge contract.
3. The `InitTransfer` event is emitted. A relayer picks it up and finalizes the transfer on NEAR via `fin_transfer`.
4. On NEAR, `send_fee_internal` mints wrapped STRK to the relayer's NEAR account.
5. The 1 STRK on StarkNet remains in the bridge contract balance indefinitely.
6. No function in `IOmniBridge` can be called to retrieve it — the STRK is permanently locked. [4](#0-3) [5](#0-4)

        fn init_transfer(
            ref self: ContractState,
            token_address: ContractAddress,
            amount: u128,
            fee: u128,
            native_fee: u128,
            recipient: ByteArray,
            message: ByteArray,
        ) {
            assert(!_is_paused(@self, PAUSE_INIT_TRANSFER), 'ERR_INIT_TRANSFER_PAUSED');

            assert(amount > 0, 'ERR_ZERO_AMOUNT');
            assert(fee < amount, 'ERR_INVALID_FEE');

            let origin_nonce = self.current_origin_nonce.read() + 1;
            self.current_origin_nonce.write(origin_nonce);

            let caller = get_caller_address();

            if self.is_bridge_token(token_address) {
                IBridgeTokenDispatcher { contract_address: token_address }
                    .burn(caller, amount.into());
            } else {
                let success = IERC20Dispatcher { contract_address: token_address }
                    .transfer_from(caller, get_contract_address(), amount.into());
                assert(success, 'ERR_TRANSFER_FROM_FAILED');
            }

            if native_fee > 0 {
                let native_token = self.strk_token_address.read();
                let success = IERC20Dispatcher { contract_address: native_token }
                    .transfer_from(caller, get_contract_address(), native_fee.into());
                assert(success, 'ERR_FEE_TRANSFER_FAILED');
            }

            self
                .emit(
                    Event::InitTransfer(
                        InitTransfer {
                            sender: caller,
                            token_address,
                            origin_nonce,
                            amount,
                            fee,
                            native_fee,
                            recipient,
                            message,
                        },
                    ),
                )

    fn send_fee_internal(
        &mut self,
        transfer_message: &TransferMessage,
        fee_recipient: AccountId,
        token_fee: u128,
    ) -> PromiseOrValue<()> {
        if transfer_message.fee.native_fee.0 != 0 {
            let origin_chain = transfer_message.origin_transfer_id.as_ref().map_or_else(
                || transfer_message.get_origin_chain(),
                |origin_transfer_id| origin_transfer_id.origin_chain,
            );

            if origin_chain.is_utxo_chain() {
                env::panic_str(BridgeError::NativeFeeForUtxoChain.to_string().as_str())
            } else if origin_chain == ChainKind::Near {
                Promise::new(fee_recipient.clone())
                    .transfer(NearToken::from_yoctonear(transfer_message.fee.native_fee.0))
                    .detach();
            } else {
                ext_token::ext(self.get_native_token_id(origin_chain))
                    .with_static_gas(MINT_TOKEN_GAS)
                    .mint(fee_recipient.clone(), transfer_message.fee.native_fee, None)
                    .detach();
            }
        }
