### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Bricking - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The OS-level implementation of the `replace_class` syscall does not verify that the caller-supplied `class_hash` corresponds to a previously declared contract class. This is an explicitly acknowledged gap (marked with a TODO). An unprivileged transaction sender can cause a contract to permanently replace its class with an undeclared, non-executable hash, making the contract permanently non-callable and freezing any funds it holds.

---

### Finding Description

The `execute_replace_class` function in `syscall_impls.cairo` accepts any arbitrary `class_hash` value from the syscall request and writes it directly to the contract's state entry without checking whether that hash exists in the `contract_class_changes` dictionary (i.e., whether it was ever declared via a `Declare` transaction).

The code at line 898 contains an explicit developer acknowledgment of this missing check:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The same missing validation exists in the deprecated syscall path:

```cairo
func execute_replace_class{contract_state_changes: DictAccess*, revert_log: RevertLogEntry*}(
    contract_address, syscall_ptr: ReplaceClass*
) {
    alloc_locals;
    let class_hash = syscall_ptr.class_hash;
    // No check that class_hash is declared.
    ...
    dict_update{dict_ptr=contract_state_changes}(...);
``` [2](#0-1) 

Both the modern (`execute_syscalls.cairo` line 195) and deprecated (`deprecated_execute_syscalls.cairo` line 676) syscall dispatch paths route to these unvalidated implementations. [3](#0-2) 

By contrast, the `Declare` transaction path correctly enforces that a class hash is valid before registering it — `prev_value=0` enforces uniqueness and `assert_not_zero(compiled_class_hash)` enforces non-zero compiled hash: [4](#0-3) 

The `replace_class` path has no equivalent guard.

---

### Impact Explanation

When a contract's class hash is set to an undeclared value, subsequent calls to that contract reach `execute_entry_point`, which performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
``` [5](#0-4) 

A `dict_read` on an undeclared class hash returns 0 (uninitialized), and `find_element` will fail to locate any compiled class for key `0`. The contract becomes permanently non-callable. Any ERC-20 tokens, ETH, or other assets held in that contract's storage are permanently frozen with no recovery path, since the contract's own logic (including any withdrawal or rescue function) is unreachable.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

The entry path is reachable by any unprivileged transaction sender:

1. A user deploys a contract (e.g., a wallet or vault) that holds funds.
2. The contract's logic (or a malicious callback/reentrancy from an external call) invokes `replace_class` with an arbitrary felt value that does not correspond to any declared class.
3. The OS accepts the syscall without validation and commits the invalid class hash to state.
4. All subsequent calls to the contract fail permanently.

This can occur accidentally (a bug in the contract's upgrade logic passes the wrong hash) or through deliberate exploitation (a malicious callee triggers a reentrancy that calls `replace_class` with a garbage hash). The OS's failure to enforce the declared-class invariant means the invalid state transition is proven valid and finalized on-chain.

**Likelihood: Medium** — Requires a contract that calls `replace_class` with an invalid hash, which can happen through buggy upgrade logic or reentrancy.

---

### Recommendation

In `execute_replace_class` (both `syscall_impls.cairo` and `deprecated_execute_syscalls.cairo`), before writing the new class hash to state, verify that the provided `class_hash` exists as a key in `contract_class_changes` with a non-zero compiled class hash value. This is exactly what the existing TODO comment at line 898 of `syscall_impls.cairo` calls for. Concretely:

```cairo
// Verify the class is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the invariant enforced during `Declare` transactions and closes the gap between what the sequencer is expected to enforce and what the OS actually proves.

---

### Proof of Concept

1. Deploy contract `Vault` holding 1000 STRK. `Vault` has an `upgrade(new_class_hash)` external function that calls `replace_class(new_class_hash)`.
2. Submit an `invoke` transaction calling `Vault.upgrade(0xdeadbeef)` where `0xdeadbeef` is not a declared class hash.
3. The OS executes `execute_replace_class` at `syscall_impls.cairo:878`. Line 898's TODO is skipped — no check is performed. `dict_update` writes `class_hash=0xdeadbeef` to `Vault`'s `StateEntry`.
4. The OS produces a valid proof for this block. The state commitment now records `Vault` with class hash `0xdeadbeef`.
5. Any subsequent call to `Vault` reaches `execute_entry_point`. `dict_read(contract_class_changes, 0xdeadbeef)` returns `0`. `find_element` with key `0` finds no compiled class. The call reverts with `ERROR_ENTRY_POINT_NOT_FOUND`.
6. The 1000 STRK in `Vault`'s storage are permanently frozen. No withdrawal, no rescue, no recovery. [6](#0-5) [7](#0-6)
