### Title
`proposed_split` Carried Over Uncleared by `next_for_old_chunk`, Causing `InvalidChunkHeaderShardSplit` Chain Halt — (File: `core/primitives/src/types.rs`)

---

### Summary

When a chunk is missing (an "old chunk") at a block height where the previous `ChunkExtra` carries a non-`None` `proposed_split` (i.e., near an epoch boundary during dynamic resharding), `ChunkExtra::next_for_old_chunk` blindly clones the previous `ChunkExtra` and only resets `state_root`. The `proposed_split` field is silently inherited. The next new chunk header produced by the chunk producer will have `proposed_split = None` (because `compute_proposed_split` returns `None` past the epoch boundary), causing `validate_chunk_with_chunk_extra_and_receipts_root` to reject every subsequent block with `InvalidChunkHeaderShardSplit`. The chain halts.

### Finding Description

**Step 1 — How `proposed_split` enters `ChunkExtra`.**

`NightshadeRuntime::compute_proposed_split` is called during chunk application. It returns `Some(TrieSplit{boundary_account, left_memory, right_memory})` only when all of the following hold:

- `ProtocolFeature::DynamicResharding` is enabled
- `is_next_block_possibly_last_in_epoch(height, prev_block_hash)` returns `true`
- `can_reshard(...)` returns `true`
- The shard is above the memory threshold
