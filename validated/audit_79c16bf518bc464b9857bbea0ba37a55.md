### Title
Missing Registered-Token Existence Check in `finalize_transfer` Allows Nonce Consumption with Unregistered Mint — (`solana/programs/bridge_token_factory/src/instructions/user/finalize_transfer.rs`)

---

### Summary

The Solana `FinalizeTransfer` instruction accepts an arbitrary `mint` account with no constraint that it is a registered bridge token. Because the `mint` address is also absent from the MPC-signed payload, an attacker can front-run a legitimate `finalize_transfer` call by substituting a fake mint (one whose mint-authority is the bridge authority PDA) while reusing the victim's valid MPC signature. The nonce is permanently consumed, the legitimate recipient receives nothing, and the real transfer is irrecoverably blocked.

---

### Finding Description

`FinalizeTransfer` account struct constrains `mint` only to be a valid SPL mint using the specified token program:

```rust
#[account(
    mut,
    mint::token_program = token_program,
)]
pub mint: Box<InterfaceAccount<'info, Mint>>,
``` [1](#0-0) 

There is no PDA constraint, no registry lookup, and no check that `mint` is an officially deployed bridge token. For the "bridged" (no-vault) path the only guard is:

```rust
require!(
    self.mint.mint_authority.contains(self.authority.key),
    ErrorCode::InvalidBridgedToken
);
``` [2](#0-1) 

This check is trivially satisfied by any SPL mint whose mint-authority has been set to the bridge's `authority` PDA — a permissionless operation on Solana.

The MPC signature is verified against `derived_near_bridge_address` and covers only `destination_nonce`, `transfer_id`, `amount`, `fee_recipient`, and `recipient`:

```rust
data.verify_signature(
    (Pubkey::default(), ctx.accounts.recipient.key()),
    &ctx.accounts.config.derived_near_bridge_address,
)?;
ctx.accounts.process(data.payload)?;
``` [3](#0-2) 

The `mint` account is **not** part of the signed payload, as confirmed by the `FinalizeTransferPayload` fields used in tests:

```rust
let payload_for_sign = FinalizeTransferPayload {
    destination_nonce: params.nonce,
    transfer_id: TransferId { origin_chain: ORIGIN_CHAIN, origin_nonce: ORIGIN_NONCE },
    amount: params.amount,
    fee_recipient: None,
};
``` [4](#0-3) 

After `use_nonce` marks the nonce as spent, the instruction mints tokens of whatever `mint` was supplied:

```rust
UsedNonces::use_nonce(
    data.destination_nonce,
    &self.used_nonces,
    ...
)?;
``` [5](#0-4) 

Once the nonce is consumed there is no recovery path; the transfer is permanently unclaimable.

---

### Impact Explanation

**Critical / High — Permanent freezing of user funds.**

An attacker who substitutes a fake mint consumes the victim's `destination_nonce`. Because `UsedNonces` is a bitmap that is never cleared, the legitimate `finalize_transfer` call for the same nonce will always revert with `NonceAlreadyUsed`. The bridged assets remain locked on the origin chain with no recourse. [5](#0-4) 

---

### Likelihood Explanation

**Medium.** MPC signatures are published as NEAR on-chain events and are therefore publicly observable. Any actor can read the signature, construct a fake mint (setting the bridge authority PDA as mint-authority), and submit `finalize_transfer` before the legitimate relayer. No privileged access, leaked key, or colluding MPC signer is required — only standard Solana transaction submission and the ability to create an SPL mint.

---

### Recommendation

Add a PDA-derived account constraint that proves the `mint` is a registered bridge token, analogous to how the `vault` PDA proves native-token registration. For example, require a `registered_token` PDA seeded from the mint key that is only created by the `deploy_token` instruction:

```rust
#[account(
    seeds = [REGISTERED_TOKEN_SEED, mint.key().as_ref()],
    bump,
)]
pub registered_token: Account<'info, RegisteredToken>,
```

Alternatively, include the `mint` public key inside `FinalizeTransferPayload` so it becomes part of the MPC-signed message, preventing substitution even without a registry account.

---

### Proof of Concept

1. Alice initiates a NEAR → Solana transfer of 1000 USDC. NEAR bridge emits a `SignTransferEvent` containing the MPC signature over `{destination_nonce: N, transfer_id: T, amount: 1000, fee_recipient: None}` with `recipient = Alice_Solana`.

2. Attacker observes the event and extracts the signature.

3. Attacker creates `FakeMint` on Solana and sets its mint-authority to the bridge `authority` PDA (permissionless).

4. Attacker calls `finalize_transfer` with:
   - `data.payload = {destination_nonce: N, transfer_id: T, amount: 1000, fee_recipient: None}`
   - `data.signature = <Alice's valid MPC signature>`
   - `recipient = Alice_Solana` (must match signed payload)
   - `mint = FakeMint`
   - `vault = None` (no vault for fake mint)

5. `verify_signature` passes (signature is valid for the payload + recipient).

6. `use_nonce(N)` marks nonce N as spent.

7. `mint.mint_authority.contains(authority.key)` passes (attacker set it up).

8. `mint_to(...)` mints 1000 `FakeMint` tokens to Alice's token account (worthless).

9. Alice's legitimate `finalize_transfer` call now reverts: nonce N is already used. Her 1000 USDC are permanently locked on NEAR with no recovery path. [6](#0-5)

impl FinalizeTransfer<'_> {
    pub fn process(&mut self, data: FinalizeTransferPayload) -> Result<()> {
        UsedNonces::use_nonce(
            data.destination_nonce,
            &self.used_nonces,
            &mut self.config,
            self.authority.to_account_info(),
            self.common.payer.to_account_info(),
            &Rent::get()?,
            self.system_program.to_account_info(),
        )?;

        if let Some(vault) = &self.vault {
            // Native version. We have a proof of token registration by vault existence
            transfer_checked(
                CpiContext::new_with_signer(
                    self.token_program.to_account_info(),
                    TransferChecked {
                        from: vault.to_account_info(),
                        to: self.token_account.to_account_info(),
                        authority: self.authority.to_account_info(),
                        mint: self.mint.to_account_info(),
                    },
                    &[&[AUTHORITY_SEED, &[self.config.bumps.authority]]],
                ),
                data.amount.try_into().map_err(|_| error!(ErrorCode::AmountOverflow))?,
                self.mint.decimals,
            )?;
        } else {
            // Bridged version. May be a fake token with our authority set but it will be ignored on the near side
            require!(
                self.mint.mint_authority.contains(self.authority.key),
                ErrorCode::InvalidBridgedToken
            );

            mint_to(
                CpiContext::new_with_signer(
                    self.token_program.to_account_info(),
                    MintTo {
                        mint: self.mint.to_account_info(),
                        to: self.token_account.to_account_info(),
                        authority: self.authority.to_account_info(),
                    },
                    &[&[AUTHORITY_SEED, &[self.config.bumps.authority]]],
                ),
                data.amount.try_into().map_err(|_| error!(ErrorCode::AmountOverflow))?,
            )?;
        }

        let payload = FinalizeTransferResponse {
            token: self.mint.key(),
            amount: data.amount,
            fee_recipient: data.fee_recipient.unwrap_or_default(),
            transfer_id: data.transfer_id,
        }
        .serialize_for_near(())?;

        self.common.post_message(payload)?;

        Ok(())
    }
