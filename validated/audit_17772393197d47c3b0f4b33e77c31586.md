### Title
Non-EVM-Denom Bank Balances Orphaned and Recoverable After SELFDESTRUCT via CREATE2 Redeploy - (File: x/evm/statedb/statedb.go)

### Summary

`StateDB.Commit()` only burns the EVM-native denom balance of a self-destructed contract. Non-EVM-native Cosmos bank balances (IBC tokens, CosmWasm bridge tokens, etc.) held by the destroyed address are explicitly left as orphaned bank balances. Because `DeleteAccount` removes only the auth account record — not the bank module balances for other denoms — an attacker who controls the CREATE2 factory and salt can redeploy a new contract at the same address and immediately gain access to those orphaned funds.

### Finding Description

In `StateDB.Commit()`, when processing a self-destructed account, the code drains only the EVM denom:

```go
// Only the EVM denom is burned here. Non-EVM-native tokens (IBC, CosmWasm
// bridge) held by the destroyed address are not drained and may remain as
// orphaned bank balances.
if remaining := s.keeper.GetBalance(cacheCtx, cosmosAddr, s.evmDenom); remaining.Sign() > 0 {
    coin := sdk.NewCoin(s.evmDenom, sdkmath.NewIntFromBigInt(remaining.ToBig()))
    if _, err := s.keeper.SubBalance(cacheCtx, cosmosAddr, coin); err != nil {
        return errorsmod.Wrap(err, "failed to burn post-selfdestruct balance")
    }
}
if err := s.keeper.DeleteAccount(cacheCtx, obj.Address()); err != nil {
    return errorsmod.Wrap(err, "failed to delete account")
}
``` [1](#0-0) 

`DeleteAccount` in the keeper removes code, storage, and the auth account record, but it never touches the bank module's balance store for any denom:

```go
// NOTE: balance should be cleared separately
func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
    // clear storage
    k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool { ... })
    // remove auth account
    k.accountKeeper.RemoveAccount(ctx, acct)
    ...

``` [2](#0-1) 

After `DeleteAccount`, the Cosmos address corresponding to the destroyed contract has no auth account, but its non-EVM-denom bank balances remain in the bank module's KV store. When a new contract is deployed at the same address (deterministically via CREATE2), a fresh auth account is created at that Cosmos address, and the bank module's existing non-EVM-denom balances become accessible to the new contract's controller.

### Impact Explanation

**Critical — Unauthorized transfer of Cosmos bank funds through Ethermint stateDB logic.**

Any non-EVM-native tokens (IBC-transferred assets, CosmWasm bridge tokens, or any `x/bank` denom other than the EVM denom) held by a self-destructed contract address survive the destruction in the bank module. An attacker who controls the CREATE2 factory and salt can redeploy a contract at the same address in a subsequent transaction, causing the bank module to associate those orphaned balances with the new account. The attacker then drains them. This is an unauthorized balance transfer of Cosmos bank funds enabled directly by Ethermint's `StateDB.Commit()` omission.

### Likelihood Explanation

**Medium.** The precondition is that a contract holds non-EVM-denom bank balances (e.g., IBC tokens sent to the contract's Cosmos address, which is a normal and documented operation). The attacker must control the CREATE2 factory and salt used to deploy the original contract, which is a realistic scenario for any contract the attacker themselves deployed or for any protocol where the factory is public. The SELFDESTRUCT + CREATE2 redeploy sequence is a standard EVM pattern and requires no privileged access.

### Recommendation

In `StateDB.Commit()`, when processing a self-destructed account, iterate over **all** bank module balances held by the destroyed Cosmos address — not just the EVM denom — and burn or transfer them before calling `DeleteAccount`. The Cosmos SDK `BankKeeper.GetAllBalances(ctx, cosmosAddr)` returns all denominations; each should be sent to the module account and burned (or to a community pool) before the account is removed.

```go
// Drain ALL bank balances, not just the EVM denom
allBalances := s.bankKeeper.GetAllBalances(cacheCtx, cosmosAddr)
if !allBalances.IsZero() {
    if err := s.bankKeeper.SendCoinsFromAccountToModule(cacheCtx, cosmosAddr, types.ModuleName, allBalances); err != nil {
        return errorsmod.Wrap(err, "failed to drain non-EVM balances pre-selfdestruct")
    }
    if err := s.bankKeeper.BurnCoins(cacheCtx, types.ModuleName, allBalances); err != nil {
        return errorsmod.Wrap(err, "failed to burn non-EVM balances pre-selfdestruct")
    }
}
```

### Proof of Concept

1. Attacker deploys a factory contract `F` via a standard EVM transaction.
2. Attacker calls `F.create2(salt)` to deploy child contract `C` at a deterministic address `A`.
3. A third party (or the attacker themselves) sends IBC tokens (e.g., `uatom`) to the Cosmos address corresponding to `A` via a standard IBC transfer.
4. Attacker calls `C.selfdestruct()`. `StateDB.SelfDestruct` burns the EVM denom balance. At `Commit()`, `StateDB.Commit()` burns only the EVM denom and calls `DeleteAccount`, which removes the auth account but leaves `uatom` in the bank module at address `A`.
5. In a subsequent transaction, attacker calls `F.create2(salt)` again with the same salt. A new contract `C'` is deployed at address `A`. The bank module now has an auth account at `A` again.
6. Attacker calls `C'.drain()` which transfers the `uatom` balance to the attacker. The attacker has recovered funds that should have been permanently inaccessible after the selfdestruct.

// DeleteAccount handles contract's suicide call:
// - remove code
// - remove states
// - remove auth account
//
// NOTE: balance should be cleared separately
func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
	cosmosAddr := sdk.AccAddress(addr.Bytes())
	acct := k.accountKeeper.GetAccount(ctx, cosmosAddr)
	if acct == nil {
		return nil
	}

	// NOTE: only Ethereum accounts (contracts) can be selfdestructed
	_, ok := acct.(ethermint.EthAccountI)
	if !ok {
		return errorsmod.Wrapf(types.ErrInvalidAccount, "type %T, address %s", acct, addr)
	}

	// clear storage
	k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool {
		k.SetState(ctx, addr, key, nil)
		return true
	})

	// remove auth account
	k.accountKeeper.RemoveAccount(ctx, acct)

	k.debugLog(ctx, "account suicided",
		"ethereum-address", addr,
		"cosmos-address", cosmosAddr,
	)

	return nil
