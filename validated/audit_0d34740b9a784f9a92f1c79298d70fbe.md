### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary `class_hash` value from a contract without verifying that the new class hash corresponds to a previously declared class. This mirrors the "Blocked LP tokens" vulnerability class: a state-mutating operation proceeds without first validating that the new state is coherent with existing protocol invariants. After the replacement, the contract's entry points become permanently unreachable, freezing all funds held by the contract.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function updates a contract's `class_hash` in `contract_state_changes` to whatever value the calling contract supplies in `request.class_hash`. There is no check that this new hash corresponds to a class that has been declared on-chain.

The developers themselves acknowledge this gap with an explicit TODO:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The OS writes the new `StateEntry` with the unvalidated `class_hash` directly into `contract_state_changes`. This dictionary is later squashed and committed to the global state root via `state_update`. [2](#0-1) 

Once committed, the contract's on-chain class hash is permanently set to the undeclared value. Every subsequent call to the contract will fail at class-lookup time because no compiled class exists for that hash. The contract's storage and any token balances it holds become permanently inaccessible.

The `contract_class_changes` dictionary, which tracks declared classes, is never consulted inside `execute_replace_class`. [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any ERC-20 balances, ETH, or other assets stored in the contract's storage become permanently inaccessible once the class hash is replaced with an undeclared value. Because Cairo's state model is append-only and the OS commits the corrupted `StateEntry` to the Merkle tree, there is no recovery path. The funds are frozen forever.

---

### Likelihood Explanation

The `replace_class` syscall is callable by any contract from within its own execution context — no privileged role is required. A malicious contract deployer can:

1. Deploy a contract that holds funds (e.g., receives ETH or ERC-20 tokens).
2. Call `replace_class` with a fabricated, never-declared class hash.
3. The OS accepts the call, commits the corrupted state, and the contract's funds are permanently frozen.

Additionally, a buggy but non-malicious contract upgrade could accidentally supply a wrong hash, triggering the same outcome. The missing validation is the sole guard against both scenarios.

---

### Recommendation

Inside `execute_replace_class`, before writing the new `StateEntry`, verify that `request.class_hash` exists in `contract_class_changes` (or in the global declared-class set). Concretely:

- Perform a `dict_read` on `contract_class_changes` for `request.class_hash` and assert the returned compiled class hash is non-zero.
- Alternatively, maintain a separate "declared classes" set and assert membership before accepting the replacement.

The developers have already identified this gap (the TODO comment at line 898); it must be resolved before the syscall is considered safe. [4](#0-3) 

---

### Proof of Concept

1. **Deploy** a contract `C` that holds 1000 STRK tokens in its storage.
2. **Craft** a transaction that calls `C.__execute__`, which internally invokes the `replace_class` syscall with `class_hash = 0xdeadbeef` (a hash that has never been declared).
3. **Observe** that `execute_replace_class` in the OS:
   - Reads the current `StateEntry` for `C`.
   - Writes a new `StateEntry` with `class_hash = 0xdeadbeef`, preserving `storage_ptr` and `nonce`.
   - Does **not** check `contract_class_changes` for `0xdeadbeef`.
4. **After block finalization**, `state_update` commits the corrupted entry to the global state root.
5. **Any subsequent call** to `C` (e.g., to transfer the 1000 STRK) fails at class resolution — no compiled class exists for `0xdeadbeef`.
6. The 1000 STRK are permanently frozen with no recovery mechanism. [3](#0-2) [5](#0-4)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    let revert_log = &revert_log[1];

    return ();
}
