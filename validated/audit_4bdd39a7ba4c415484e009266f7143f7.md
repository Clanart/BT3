### Title
Missing Class Hash Existence Check in `execute_replace_class` Allows Permanent Contract Bricking - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary `class_hash` value without verifying that the hash corresponds to a previously declared contract class. This is the direct Cairo analog of the Axelar finding: just as a Solidity low-level call to a non-existent address silently returns `true`, the OS here silently commits a `replace_class` to a non-existent class hash, permanently bricking the contract and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` reads the requested `class_hash` from the syscall request and immediately writes it into `contract_state_changes` with no validation that the hash was ever declared:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-inserted TODO at line 898 explicitly acknowledges this missing check. The same omission exists in the deprecated path in `deprecated_execute_syscalls.cairo` at `execute_replace_class` (lines 307–329), which also writes the new class hash directly without any existence validation.

After the state is committed, any future call to the affected contract reaches `execute_entry_point`, which performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // now the non-existent hash
);
// compiled_class_hash == 0 (dict default for undeclared hash)

let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,           // key == 0, not found → hard OS failure
);
```

`find_element` panics if the key is absent. The contract becomes permanently uncallable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to a non-existent value via `replace_class`, the state change is committed to the Merkle trie. No future transaction can successfully call the contract: the OS will fail to resolve the compiled class, making the contract permanently inaccessible. Any ERC-20 tokens, ETH, or other assets held in the contract's storage are irreversibly frozen. There is no recovery path because `replace_class` itself requires the contract to be callable.

---

### Likelihood Explanation

The `replace_class` syscall is reachable by any contract that includes it in its logic. A malicious contract owner can deploy a contract, attract user deposits, and then invoke `replace_class` with a fabricated, never-declared class hash (e.g., `0xdeadbeef...`). The OS accepts the syscall without complaint, the state is committed, and all deposited funds are frozen. No privileged role, leaked key, or external dependency is required — only the ability to deploy and call a contract, which is available to any unprivileged transaction sender.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that `class_hash` is present in `contract_class_changes` (i.e., it was previously declared). Concretely, perform a `dict_read` on `contract_class_changes` with `key=class_hash` and assert the result is non-zero:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);  // Revert if class was never declared
```

This mirrors the fix recommended in the Axelar report: check existence before the operation, rather than discovering the absence only at call time.

---

### Proof of Concept

1. Attacker deploys `VaultContract` that accepts user deposits and exposes a `drain()` function that calls `replace_class(0xdeadbeef_undeclared_hash)`.
2. Users deposit funds into `VaultContract`.
3. Attacker calls `drain()`. The OS executes `execute_replace_class`:
   - `class_hash = 0xdeadbeef_undeclared_hash` is written to `contract_state_changes` with no validation.
   - The TODO at line 898 of `syscall_impls.cairo` confirms no check is performed.
4. The block is proven and the state is committed.
5. Any future `call_contract` targeting `VaultContract` reaches `execute_entry_point`:
   - `dict_read(contract_class_changes, key=0xdeadbeef...)` → returns `0` (undeclared).
   - `find_element(..., key=0)` → element not found → OS hard failure / transaction reverts.
6. `VaultContract` is permanently uncallable. All deposited funds are frozen forever.

        // Not enough gas to execute the syscall.
        return ();
