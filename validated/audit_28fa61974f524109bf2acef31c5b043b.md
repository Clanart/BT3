### Title
`deprecated_declared_classes` Silently Dropped in `CommitmentStateDiff → ThinStateDiff` Conversion Produces Wrong `state_diff_commitment` and Block Hash — (`crates/blockifier/src/state/cached_state.rs`)

---

### Summary

The `From<CommitmentStateDiff> for ThinStateDiff` conversion unconditionally hardcodes `deprecated_declared_classes: Vec::new()`. Every downstream commitment computation — `calculate_state_diff_hash`, `concatenated_counts`, `PartialBlockHashComponents`, and the final block hash — is therefore wrong for any block that contains a Declare v0/v1 (Cairo 0 class declaration) transaction. The wrong block hash is persisted in the sequencer's storage and returned authoritatively by RPC, and it is the value against which proof-facts block-hash validation is performed.

---

### Finding Description

**Root cause — the silent drop** [1](#0-0) 

`CommitmentStateDiff` has no `deprecated_declared_classes` field; the conversion to `ThinStateDiff` therefore always produces an empty list, regardless of how many Cairo 0 classes were declared in the block.

**Propagation into commitment computation**

`BlockExecutionArtifacts::new` calls `ThinStateDiff::from(commitment_state_diff.clone())` and passes the result directly to `calculate_block_commitments`: [2](#0-1) 

`calculate_block_commitments` spawns a task that calls `calculate_state_diff_hash`, which chains `deprecated_declared_classes` into the Poseidon hash: [3](#0-2) 

`ThinStateDiff::len()` also counts `deprecated_declared_classes.len()`, so `concatenated_counts` (packed lengths) is wrong too: [4](#0-3) 

The wrong `state_diff_commitment` and `concatenated_counts` are baked into `PartialBlockHashComponents`: [5](#0-4) 

**Wrong block hash persisted and used for proof-facts validation**

`decision_reached` extracts the wrong `state_diff_commitment` from `partial_block_hash_components` and passes it to `write_commitment_results_and_add_new_task`: [6](#0-5) 

`finalize_commitment_output` then calls `calculate_block_hash` with the wrong `partial_block_hash_components` and the global root, producing a wrong block hash: [7](#0-6) 

This wrong block hash is written to the `block_hashes` storage table and is the value read by `validate_proof_block_hash` when a user submits an Invoke V3 transaction carrying SNOS proof facts: [8](#0-7) 

**Why the committer's internal check does not catch it**

`thin_state_diff()` also calls `ThinStateDiff::from(self.commitment_state_diff.clone())`, so the `state_diff` stored in Apollo storage has the same empty `deprecated_declared_classes`. The committer's `verify_state_diff_hash` check compares the provided commitment against `calculate_state_diff_hash(&state_diff)` — both sides are computed from the same truncated diff, so they match and no error is raised: [9](#0-8) 

---

### Impact Explanation

**High — RPC returns authoritative wrong value.** `starknet_getStateUpdate` returns the `ThinStateDiff` stored in Apollo storage, which is missing all `deprecated_declared_classes` for every block that contained a Declare v0/v1 transaction. `starknet_getBlockWithTxHashes` returns the block header whose `state_diff_commitment` and block hash are both wrong.

**Critical — wrong commitment/proof invariant.** The block hash written to the `block_hashes` contract (used by the `get_block_hash` syscall and by `validate_proof_block_hash`) does not correspond to the actual state transition. A SNOS prover that correctly includes `deprecated_declared_classes` in its output will compute a different block hash than the one stored by the sequencer. Proof-facts validation will therefore reject valid proofs and accept proof facts that reference the sequencer's wrong hash — breaking the binding between the on-chain block hash and the proven state diff.

---

### Likelihood Explanation

Any Declare v0 or Declare v1 transaction (Cairo 0 class declaration) triggers the bug. These transaction types remain valid in the current Starknet protocol. No special privilege is required; any user can submit such a transaction. The bug fires on every block that contains at least one such transaction.

---

### Recommendation

1. Add a `deprecated_declared_classes: Vec<ClassHash>` field to `CommitmentStateDiff` and populate it from `StateMaps.declared_contracts` (filtering for entries whose class has no compiled class hash, i.e., Cairo 0 classes).
2. Remove the hardcoded `deprecated_declared_classes: Vec::new()` in `From<CommitmentStateDiff> for ThinStateDiff` and propagate the field correctly.
3. Alternatively, thread the `deprecated_declared_classes` list separately from the blockifier's execution output into `BlockExecutionArtifacts::new` so it can be merged into the `ThinStateDiff` before `calculate_block_commitments` is called.
4. Add a test that executes a Declare v0 transaction, computes the state diff commitment from the resulting `BlockExecutionArtifacts`, and asserts it equals `calculate_state_diff_hash` applied to the full `ThinStateDiff` including the declared class.

---

### Proof of Concept

1. A user submits a Declare v0 transaction declaring Cairo 0 class `C`.
2. The sequencer executes the block; `CommitmentStateDiff` is produced with no record of `C` in any field (it has no `deprecated_declared_classes`).
3. `BlockExecutionArtifacts::new` calls `ThinStateDiff::from(commitment_state_diff)` → `deprecated_declared_classes = []`.
4. `calculate_state_diff_hash` hashes the diff without `C`; call this `H_wrong`.
5. `H_wrong` is stored as `state_diff_commitment` in the block header; `concatenated_counts` encodes `state_diff.len()` without counting `C`.
6. `calculate_block_hash` produces `BH_wrong` from `H_wrong` and the global root.
7. `BH_wrong` is written to the `block_hashes` contract at slot `block_number`.
8. A SNOS prover that correctly processes the block computes `H_correct ≠ H_wrong` (because it includes `C`) and therefore `BH_correct ≠ BH_wrong`.
9. When a user submits an Invoke V3 transaction with `proof_facts.block_hash = BH_correct`, `validate_proof_block_hash` reads `BH_wrong` from storage and returns `"Block hash mismatch"` — the valid proof is rejected.
10. A transaction carrying `proof_facts.block_hash = BH_wrong` passes the hash check even though `BH_wrong` does not commit to the actual state diff, allowing proof facts that do not correspond to a correct state transition to be admitted.

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}

    fn validate_proof_block_hash(
        proof_block_hash: Felt,
        proof_block_number: u64,
        os_constants: &OsConstants,
        state: &mut dyn State,
    ) -> TransactionPreValidationResult<()> {
        if proof_block_hash == Felt::ZERO {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Proof block hash is zero for block {proof_block_number}."
            )));
        }

        // Compare the proof's block hash with the stored block hash.
        let block_hash_contract_address =
            os_constants.os_contract_addresses.block_hash_contract_address();

        let stored_block_hash = state
            .get_storage_at(block_hash_contract_address, StorageKey::from(proof_block_number))?;

        if stored_block_hash != proof_block_hash {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Block hash mismatch for block {proof_block_number}. Proof block hash: \
                 {proof_block_hash}, stored block hash: {stored_block_hash}."
            )));
        }

        Ok(())
    }
