### Title
`ProofOfInclusion::valid()` Never Verifies Against an External Trusted Root — Forged DataLayer Inclusion Proofs Always Pass - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` in `chia-datalayer` only checks that each layer's `combined_hash` is internally self-consistent with the previous hash. Its final check — `existing_hash == self.root_hash()` — is a tautology: after the loop, `existing_hash` is always equal to `self.root_hash()` by construction. The function never compares the computed root against any externally committed, trusted DataLayer root. An attacker can craft a `ProofOfInclusion` for any arbitrary `node_hash` (any key-value claim) with a fabricated chain of internally consistent layers, and `valid()` will return `true`.

### Finding Description

`ProofOfInclusion::valid()` is defined as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;   // existing_hash := layer.combined_hash
    }

    existing_hash == self.root_hash()      // ← always true
}
``` [1](#0-0) 

And `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← the same value as existing_hash after the loop
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

After the loop completes without returning `false`, `existing_hash` holds the last `layer.combined_hash`. `self.root_hash()` also returns `self.layers.last().combined_hash`. These are the same value by definition, so the final comparison `existing_hash == self.root_hash()` is unconditionally `true`. It provides zero additional security.

The missing check is: `existing_hash == trusted_external_committed_root`. Without it, `valid()` only proves that the proof's internal hash chain is self-consistent — it does not prove that the chain terminates at the actual committed DataLayer Merkle root.

The struct is `Streamable` (deserializable from bytes) and fully exposed to Python via `py_valid()`: [3](#0-2) 

The Python type stub confirms `valid()` is the sole validation method, with no `valid_for_root(root)` alternative: [4](#0-3) 

**Exploit path:**

1. Attacker picks any `node_hash` they want to falsely claim is in the DataLayer tree (e.g., a key-value pair that does not exist).
2. Attacker builds a `ProofOfInclusion` with any number of `ProofOfInclusionLayer` entries where each `combined_hash` is correctly computed from the previous hash and the attacker-chosen `other_hash` — a fully attacker-controlled chain.
3. Attacker serializes this forged `ProofOfInclusion` and sends it to a DataLayer node.
4. The node deserializes it and calls `proof.valid()`.
5. `valid()` returns `true` because the internal chain is self-consistent, even though the root it terminates at is not the committed on-chain DataLayer root.

### Impact Explanation

This is a **High** severity finding matching the allowed impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

Any DataLayer consumer that calls `proof.valid()` as the sole check on a proof received from an untrusted source will accept forged inclusion proofs for arbitrary key-value pairs. This allows an attacker to falsely prove that any data exists in a DataLayer store, enabling incorrect state acceptance and cross-node disagreement about DataLayer contents.

### Likelihood Explanation

`ProofOfInclusion` is `Streamable` and exposed to Python. The method name `valid()` strongly implies complete validation. There is no `valid_for_root()` API, no documentation warning that callers must separately compare `proof.root_hash()` to a trusted root, and the tautological final check gives false confidence that the proof is fully validated. Any DataLayer node or client that receives proofs from peers and calls only `proof.valid()` is vulnerable.

### Recommendation

Replace the tautological self-referential check with a comparison against an externally supplied trusted root:

```rust
pub fn valid_for_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }

    &existing_hash == trusted_root   // ← compare against external committed root
}
```

The existing `valid()` method should either be removed or deprecated, since it provides only a self-consistency check that is insufficient for security. All callers that receive proofs from untrusted sources must use `valid_for_root(committed_root)` where `committed_root` is the on-chain committed DataLayer root hash.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
import hashlib

# Attacker wants to forge proof that node_hash X is in the tree
fake_node_hash = bytes([0xAA] * 32)
fake_sibling   = bytes([0xBB] * 32)

# Build one internally consistent layer: combined = H(fake_node_hash, fake_sibling)
# (using whatever calculate_internal_hash does — attacker controls both inputs)
def fake_combined(left, right):
    h = hashlib.sha256()
    h.update(b'\x02')  # MIDDLE node prefix used by chia merkle
    h.update(left)
    h.update(right)
    return h.digest()

combined = fake_combined(fake_node_hash, fake_sibling)

layer = ProofOfInclusionLayer(
    other_hash_side=1,          # right side
    other_hash=fake_sibling,
    combined_hash=combined,     # attacker sets this to match
)

proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

# valid() returns True even though this proves nothing about the real committed root
assert proof.valid() == True   # passes — forged proof accepted
# proof.root_hash() == combined, which is NOT the real DataLayer root
```

The `valid()` call returns `True` for a completely fabricated proof because the final check `existing_hash == self.root_hash()` is always satisfied by construction. [1](#0-0)
