### Title
Missing Class Declaration Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS Cairo program accepts an arbitrary `class_hash` from the contract's syscall request and writes it directly into the contract state without verifying that the hash corresponds to a previously declared class. A TODO comment in the code explicitly acknowledges this missing check. An attacker-controlled contract can exploit this to set its own class hash to an undeclared value, permanently rendering the contract uncallable and freezing any funds held within it.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The `class_hash` value is taken directly from the syscall request and committed to `contract_state_changes` with no validation that it exists in `contract_class_changes` (the declared class registry). The TODO comment at line 898 is an explicit in-code acknowledgment of this missing invariant.

The downstream enforcement point is in `execute_entry_point.cairo`, where any future call to the contract performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
``` [2](#0-1) 

If `class_hash` is undeclared, `dict_read` returns `0` (the default for an uninitialized Cairo dict entry), and `find_element` cannot resolve a compiled class — making the contract permanently uncallable at the OS level.

The analog to the `isContract` bypass is direct: just as a constructor call bypasses the `isContract` whitelist check because code is not yet present, the `replace_class` syscall bypasses the class-declaration check because the OS omits the validation entirely. In both cases, a state-transition guard that should be enforced is absent at the critical moment of state mutation.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is overwritten with an undeclared value and the state is committed to the chain:

- Every subsequent call to that contract resolves to a class hash of `0` (no declared compiled class), making the contract permanently uncallable.
- Any ERC-20 tokens, ETH, or other assets held in the contract's storage become permanently inaccessible.
- The state commitment (Patricia Merkle root) is updated with the corrupted `StateEntry`, making the corruption irreversible without a protocol-level intervention.

---

### Likelihood Explanation

**High.**

- The `replace_class` syscall is a standard, publicly available syscall callable by any Cairo 1 contract.
- No privileged role, leaked key, or external dependency is required.
- The attacker only needs to deploy (or control) a contract and issue a single `replace_class` syscall with an arbitrary undeclared hash.
- The missing check is confirmed by the in-code TODO comment, meaning no defensive logic exists at the OS layer.

---

### Recommendation

Before committing the new `class_hash` to `contract_state_changes`, the OS must verify that the hash exists as a key in `contract_class_changes` (i.e., it has been declared in the current or a prior block). Concretely, a `dict_read` on `contract_class_changes` with the requested `class_hash` should be performed, and the result must be asserted non-zero (a declared class always has a non-zero compiled class hash, enforced by `assert_not_zero(compiled_class_hash)` in `execute_declare_transaction`). [3](#0-2) 

---

### Proof of Concept

1. **Attacker deploys** a contract `C` holding user funds (e.g., an ERC-20 vault). The contract's class hash `H_valid` is properly declared.

2. **Attacker calls** `replace_class(H_fake)` from within contract `C`, where `H_fake` is any felt value that has never been declared (e.g., `0xdeadbeef`).

3. **OS execution** of `execute_replace_class`:
   - Reads `class_hash = H_fake` from the syscall request.
   - Skips the missing declaration check (the TODO).
   - Writes `StateEntry(class_hash=H_fake, ...)` into `contract_state_changes`.
   - The block is proven and the state root is updated with the corrupted entry.

4. **In any subsequent block**, a user attempts to withdraw funds by calling contract `C`:
   - `execute_entry_point` reads `class_hash = H_fake` from state.
   - `dict_read(contract_class_changes, H_fake)` returns `0` (never declared).
   - `find_element(..., key=0)` fails to locate a compiled class.
   - The call reverts; funds are permanently inaccessible. [4](#0-3)
