### Title
Missing Class Hash Validation in `execute_replace_class` Enables Permanent Contract State Corruption and Network Halt - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts an arbitrary attacker-supplied class hash without verifying it corresponds to a previously declared contract class. This is directly acknowledged by a TODO comment in the production code. An unprivileged contract deployer can replace their contract's class hash with an undeclared value, commit that invalid state to L1, and cause any subsequent OS proof attempt that touches the contract to fail via a hard Cairo assertion in `find_element`, halting block production.

---

### Finding Description

In `execute_replace_class`, the OS reads `request.class_hash` directly from the syscall request and writes it into `contract_state_changes` with no validation:

```cairo
let class_hash = request.class_hash;
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}
tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO at line 898 explicitly confirms the missing check is a known gap in the production OS code.

When any future call reaches `execute_entry_point` for a contract whose class hash was replaced with an undeclared value, the OS executes:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [2](#0-1) 

If `execution_context.class_hash` is an undeclared value, `dict_read` on `contract_class_changes` returns 0 (the Cairo dict default). `find_element` then searches for a compiled class with hash 0. Because no such class exists in `compiled_class_facts_bundle`, the Cairo hint inside `find_element` raises an unrecoverable exception, aborting the entire OS execution. This is not a graceful transaction revert — it is a hard proof-generation failure that prevents the block from being submitted to L1.

---

### Impact Explanation

**High — Network not being able to confirm new transactions (total network shutdown).**

The attack proceeds in two phases:

**Phase 1 — Poison the state (succeeds silently):** The attacker deploys a contract and calls `replace_class` with an arbitrary undeclared class hash. The OS processes this block successfully because the poisoned contract is not called again in the same block. The invalid class hash is committed to the global state root on L1.

**Phase 2 — Trigger the halt:** In any subsequent block, any transaction that calls the poisoned contract (including one submitted by the attacker themselves, or by an innocent user who does not know the class was poisoned) causes `find_element` to abort the OS execution. The sequencer cannot generate a valid STARK proof for that block and cannot advance the chain.

Because the invalid class hash is now part of the committed L1 state, the poisoned contract is permanently unexecutable. If the sequencer's transaction simulation layer also lacks this validation (consistent with the OS TODO), the sequencer will repeatedly attempt to include calls to the contract, repeatedly fail to prove the block, and stall.

---

### Likelihood Explanation

- **Attacker entry point is fully unprivileged.** Any user who can deploy a contract (a standard StarkNet operation) can call `replace_class` from within it.
- **The missing check is explicitly documented** in production code via the TODO comment, confirming it is not a latent edge case but a known absent guard.
- **No special conditions are required** beyond deploying a contract and issuing one syscall. The attacker does not need any privileged role, leaked key, or operator cooperation.
- **The blockifier simulation is likely also missing this check**, since the OS and blockifier are expected to be consistent; a discrepancy would itself be a separate bug. If the simulation did catch it, the transaction would be rejected before block inclusion, but the TODO suggests neither layer validates this.

---

### Recommendation

In `execute_replace_class`, before writing the new class hash to `contract_state_changes`, verify that the requested class hash exists in `contract_class_changes` (i.e., it has been declared in the current or a prior block). Concretely:

```cairo
// Verify the class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the validation already performed implicitly in `execute_entry_point` but must be enforced eagerly at the point of replacement, not lazily at the point of execution.

---

### Proof of Concept

1. Attacker deploys `ContractA` (any valid Sierra class).
2. `ContractA`'s `__execute__` calls the `replace_class` syscall with `class_hash = 0xdeadbeef` (an arbitrary felt not corresponding to any declared class).
3. The OS processes the block. `execute_replace_class` writes `class_hash=0xdeadbeef` into `contract_state_changes[ContractA]` with no validation. [1](#0-0) 
4. The block is proved and submitted to L1. The committed state now records `ContractA.class_hash = 0xdeadbeef`.
5. In the next block, the attacker (or any user) submits a transaction calling `ContractA`.
6. `execute_entry_point` reads `class_hash = 0xdeadbeef` from `contract_state_changes`, then calls `dict_read{dict_ptr=contract_class_changes}(key=0xdeadbeef)` → returns 0. [3](#0-2) 
7. `find_element` searches `compiled_class_facts_bundle` for a compiled class with hash 0. No such class exists. The Cairo hint raises an exception. [4](#0-3) 
8. The OS execution aborts. The STARK proof cannot be generated. The block cannot be submitted to L1. The network halts on this block.
