### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in `syscall_impls.cairo` updates `contract_state_changes` with an attacker-supplied class hash without verifying that the hash exists in `contract_class_changes`. This creates a persistent inconsistency between the two state dictionaries: the contract's on-chain state references a class that was never declared, making the contract permanently unexecutable and freezing any funds it holds.

---

### Finding Description

`execute_replace_class` reads the caller-supplied `class_hash` from the syscall request and immediately writes it into `contract_state_changes`:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The developer-acknowledged TODO at line 898 confirms the missing check. No lookup into `contract_class_changes` is performed before the update is committed.

The two state dictionaries that must remain consistent are:

| Dictionary | Role |
|---|---|
| `contract_state_changes` | Maps contract address → `StateEntry` (includes `class_hash`) |
| `contract_class_changes` | Maps `class_hash` → compiled class hash (the class tree) |

After `state_update` runs, both dictionaries are squashed and committed to their respective Patricia trees:

```cairo
let contract_state_tree_update_output = compute_contract_state_commitment(...);
let (contract_class_tree_update_output) = compute_class_commitment(...);
``` [2](#0-1) 

If `replace_class` is called with a hash that was never inserted into `contract_class_changes`, the contract state tree will reference a leaf that does not exist in the class tree. This is the direct analog of the external report: one data structure (contract state) is updated while the other (class registry) is neither checked nor updated.

---

### Impact Explanation

Once the block is committed:

1. The contract's `class_hash` field in the state tree points to an undeclared class.
2. Any future call to the contract (via `execute_call_contract`, `execute_library_call`, or direct invocation) reads `state_entry.class_hash` and attempts to load the class for execution.
3. Because the class does not exist in the class tree, execution cannot proceed.
4. `replace_class` itself can only be invoked from *within* a contract's execution context — so there is no recovery path once the class hash is corrupted.
5. All funds held by the contract (ERC-20 balances, NFTs, protocol TVL) are permanently inaccessible.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

The `replace_class` syscall is reachable by any unprivileged user who can deploy a contract. The attacker does not need any privileged role:

- Deploy a contract whose code calls `replace_class(0xdeadbeef)` (or any hash absent from `contract_class_changes`).
- The OS accepts the syscall without validation.
- The contract is permanently bricked.

For shared contracts (DEX, lending protocol, multisig wallet) that hold third-party funds, a single such call — whether intentional or triggered by a bug — freezes all deposited assets. The missing check is explicitly flagged by the development team, confirming it is a known gap in the current implementation.

---

### Recommendation

Inside `execute_replace_class`, before writing to `contract_state_changes`, verify that `class_hash` is present in `contract_class_changes`:

```cairo
// Verify the new class hash has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the check already enforced for `declare` transactions, where `prev_value=0` is asserted to guarantee a class is declared at most once:

```cairo
// Note that prev_value=0 enforces that a class may be declared only once.
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [3](#0-2) 

---

### Proof of Concept

1. **Deploy** a contract whose Sierra/CASM code contains a call to `replace_class(arbitrary_undeclared_hash)`.
2. **Invoke** the contract function that triggers the `replace_class` syscall.
3. The OS executes `execute_replace_class`:
   - Reads `class_hash = arbitrary_undeclared_hash` from the syscall request.
   - Skips the missing declared-class check (line 898 TODO).
   - Writes `new StateEntry(class_hash=arbitrary_undeclared_hash, ...)` into `contract_state_changes`.
4. `state_update` commits both trees. The contract state tree now contains `arbitrary_undeclared_hash`; the class tree does not.
5. **Any subsequent call** to the contract reads `state_entry.class_hash = arbitrary_undeclared_hash`, attempts class lookup, finds nothing, and fails irrecoverably.
6. All funds held by the contract are permanently frozen with no recovery mechanism, since `replace_class` requires the contract to be executable. [4](#0-3) [5](#0-4)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
