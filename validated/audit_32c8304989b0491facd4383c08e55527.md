### Title
Missing backward-compatibility branch in `calculate_global_state_root` due to overly restrictive `and` condition produces incorrect global state root — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/state/commitment.cairo`)

### Summary
`calculate_global_state_root` uses `contract_state_root == 0 and contract_class_root == 0` as its only early-return guard. The function's own comment documents a second required case — when only the class tree is empty (`contract_class_root == 0`), the global root must equal `contract_state_root` for backward compatibility. The `and` condition silently skips this branch, causing the function to hash `(GLOBAL_STATE_VERSION, contract_state_root, 0)` and commit a wrong global state root to the proof output.

### Finding Description
The function comment at lines 31–37 specifies three distinct cases:

| Condition | Expected result |
|---|---|
| `contract_state_root == 0 AND contract_class_root == 0` | `global_root = 0` |
| `contract_class_root == 0` (state non-empty) | `global_root = contract_state_root` (backward compat) |
| Otherwise | `global_root = H(VERSION, state_root, class_root)` |

The code at line 41 only handles case 1 and case 3:

```cairo
if (contract_state_root == 0 and contract_class_root == 0) {
    return (global_root=0);
}
// Jumps directly to poseidon_hash_many — case 2 is never reached
tempvar elements: felt* = new (GLOBAL_STATE_VERSION, contract_state_root, contract_class_root);
let (global_root) = poseidon_hash_many(n=3, elements=elements);
return (global_root=global_root);
``` [2](#0-1) 

When `contract_class_root == 0` and `contract_state_root != 0`, the function returns `poseidon_hash(GLOBAL_STATE_VERSION, contract_state_root, 0)` — a value that differs from `contract_state_root`. This incorrect root is then written into the `CommitmentUpdate` struct returned by `state_update` in `state.cairo`: [3](#0-2) 

The wrong `initial_global_root` or `final_global_root` propagates into the OS proof output, making the committed state root diverge from the true on-chain root.

### Impact Explanation
An incorrect global state root committed by the OS means the proof attests to a state that does not match the actual chain state. Any verifier or L1 contract that checks the state root against the expected value will reject the proof, halting block confirmation. Alternatively, if the incorrect root is accepted, nodes that independently compute the correct root will disagree, causing an unintended chain split. Both outcomes fall within the allowed impact scope:

- **High. Network not being able to confirm new transactions (total network shutdown)** — if the verifier rejects the malformed proof.
- **High. Unintended chain split (network partition)** — if different nodes accept different roots.

### Likelihood Explanation
StarkNet's class tree was introduced after the contract state tree. During the backward-compatibility window — any block where `contract_class_root == 0` but `contract_state_root != 0` — this function produces a wrong root. Any re-execution or proof generation over such historical blocks (a standard operation for verifiers and archive nodes) triggers the bug deterministically. No privileged access or key compromise is required; the trigger is the ordinary state of the chain at that point in history.

### Recommendation
Replace the two-condition guard with a single check on `contract_class_root`, which correctly handles all three documented cases:

```cairo
func calculate_global_state_root{poseidon_ptr: PoseidonBuiltin*, range_check_ptr}(
    contract_state_root: felt, contract_class_root: felt
) -> (global_root: felt) {
-   if (contract_state_root == 0 and contract_class_root == 0) {
-       return (global_root=0);
-   }
+   if (contract_class_root == 0) {
+       // Covers both: (a) both trees empty → returns 0, and
+       // (b) only class tree empty → returns contract_state_root (backward compat).
+       return (global_root=contract_state_root);
+   }

    tempvar elements: felt* = new (GLOBAL_STATE_VERSION, contract_state_root, contract_class_root);
    let (global_root) = poseidon_hash_many(n=3, elements=elements);
    return (global_root=global_root);
}
```

### Proof of Concept
1. Suppose the OS processes a block whose initial state has `contract_state_root = X` (non-zero) and `contract_class_root = 0` (class tree not yet populated).
2. `calculate_global_state_root(X, 0)` is called.
3. The condition `X == 0 and 0 == 0` evaluates to `FALSE` (because `X != 0`), so the early return is skipped.
4. The function computes `poseidon_hash_many([GLOBAL_STATE_VERSION, X, 0])` = some value `Y ≠ X`.
5. `Y` is written as `initial_global_root` into the proof output.
6. The true on-chain global root for that state is `X` (per the backward-compatibility rule).
7. The proof asserts `initial_root = Y` but the chain expects `X`; the proof is invalid or the committed root is wrong, causing a halt or split. [4](#0-3)
