### Title
`ProofOfInclusion::valid()` Is Trivially True for Any Proof with Empty `layers` — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` contains a tautological self-reference when `layers` is empty: `root_hash()` returns `self.node_hash`, so the final equality check becomes `self.node_hash == self.node_hash`, which is always `true`. An attacker can construct a `ProofOfInclusion` with an empty `layers` vector and any arbitrary `node_hash` and have `valid()` return `true`, forging a DataLayer inclusion proof for any hash value.

### Finding Description

In `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`, `ProofOfInclusion::valid()` is implemented as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()
}
``` [1](#0-0) 

And `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash   // <-- returned when layers is empty
    }
}
``` [2](#0-1) 

When `layers` is empty:
1. The `for` loop body never executes, so `existing_hash` remains `self.node_hash`.
2. `self.root_hash()` returns `self.node_hash` (the `else` branch).
3. The final check is `self.node_hash == self.node_hash` — unconditionally `true`.

This means `valid()` returns `true` for **any** `ProofOfInclusion` with an empty `layers` field, regardless of what `node_hash` contains. There is no external trusted root anchoring the check.

`ProofOfInclusion` derives `Streamable` and is exposed via Python bindings with `from_bytes()` / `from_json_dict()`, making it directly constructible from untrusted input: [3](#0-2) [4](#0-3) 

The struct is registered in the datalayer submodule and fully accessible to Python callers: [5](#0-4) 

### Impact Explanation

Any DataLayer client that calls `proof.valid()` as its sole validity check — which is the natural and documented usage pattern — can be deceived into accepting a forged inclusion proof for any arbitrary `node_hash`. The attacker does not need to know the actual tree contents or root; they simply supply `ProofOfInclusion { node_hash: <target_hash>, layers: [] }` and `valid()` returns `true`. This allows untrusted input to prove invalid state in the DataLayer Merkle tree, matching the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion**.

### Likelihood Explanation

`ProofOfInclusion` is a `Streamable` type exposed at the Python boundary. Any code path that deserializes a `ProofOfInclusion` from an untrusted peer and calls `.valid()` is exploitable. The fuzz target and integration tests confirm that `proof.valid()` is used as the sole correctness check without separately verifying `proof.root_hash()` against a trusted external root: [6](#0-5) [7](#0-6) 

The single-leaf tree case (where `layers` is legitimately empty) makes this indistinguishable from a forged proof at the API level.

### Recommendation

`valid()` must accept a trusted external root hash as a parameter and verify `self.root_hash() == trusted_root` as part of its check. Alternatively, add a guard that rejects proofs where `layers` is empty unless the caller explicitly handles the single-leaf case with a separate root comparison. The `root_hash()` method should not be the sole source of truth for the expected root — it must be anchored to an externally trusted value.

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    self.root_hash() == *trusted_root && self.valid_internal()
}
```

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer

# Forge a proof for any arbitrary hash with no knowledge of the tree
forged_node_hash = bytes([0xAB] * 32)  # attacker-chosen hash
forged_proof = ProofOfInclusion(node_hash=forged_node_hash, layers=[])

assert forged_proof.valid()          # True — no tree knowledge required
assert forged_proof.root_hash() == forged_node_hash  # self-referential
```

The `valid()` call returns `True` for any `node_hash` with empty `layers`, allowing an attacker to forge DataLayer inclusion proofs for arbitrary keys without access to the actual Merkle tree.
