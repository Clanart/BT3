### Title
`ProofOfInclusion::valid()` Final Root-Hash Check Is a Tautology — Forged Inclusion Proofs Always Pass — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

---

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate contains a final check that is a mathematical tautology: it compares `existing_hash` against `self.root_hash()`, but `root_hash()` is defined as `last.combined_hash` — the exact same value that `existing_hash` was just set to inside the loop. The check is therefore always `true` after the loop completes. The function validates only the internal self-consistency of the proof struct, never that the proof's computed root matches any externally trusted tree root. An attacker who controls a serialized `ProofOfInclusion` (received over the network or from any untrusted source) can forge a proof for any arbitrary `node_hash` and have `valid()` return `true`.

---

### Finding Description

`ProofOfInclusion::valid()` is implemented as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;   // ← set to layer.combined_hash
    }

    existing_hash == self.root_hash()      // ← always true
}
```

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← same field existing_hash was just set to
    } else {
        self.node_hash
    }
}
```

**Tautology proof:**

- After the last loop iteration, `existing_hash = calculated_hash` and the guard `calculated_hash != layer.combined_hash` did not fire, so `existing_hash == last.combined_hash`.
- `self.root_hash()` returns `last.combined_hash`.
- Therefore `existing_hash == self.root_hash()` ≡ `last.combined_hash == last.combined_hash` — always `true`.
- The empty-layers case: `existing_hash` is never modified, so it equals `self.node_hash`; `root_hash()` also returns `self.node_hash` — again always `true`.

The function never compares the computed root against any externally trusted value. A correct implementation would accept an `expected_root: &Hash` parameter and compare `existing_hash` against it, or the caller must separately assert `proof.root_hash() == trusted_root` — but the API as shipped gives no indication that this extra step is required, and the `valid()` name implies completeness.

**Forge construction:** An attacker crafts a `ProofOfInclusion` with:
- `node_hash` = any 32-byte value they wish to claim is in the tree
- `layers` = one layer where `other_hash` = `[0u8; 32]`, `other_hash_side` = `Left`, `combined_hash` = `calculate_internal_hash(node_hash, Left, [0u8; 32])`

`valid()` returns `true`. The `root_hash()` of this forged proof is the attacker-chosen `combined_hash`, not the real tree root. Any verifier that calls only `proof.valid()` accepts the forgery.

---

### Impact Explanation

The `ProofOfInclusion` struct is exposed through the Python bindings (`py_valid()`) and is `Streamable`-deserializable from untrusted bytes. DataLayer clients that receive proofs over the network and call `proof.valid()` to gate acceptance of key-value state will accept forged inclusion proofs for arbitrary keys. This lets untrusted input prove invalid DataLayer state — matching the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."*

---

### Likelihood Explanation

The `valid()` method is the sole public API for proof verification; its name implies it is sufficient. No documentation or inline comment warns callers to additionally check `root_hash()` against a trusted root. The fuzz target and all tests call only `proof.valid()` without a separate root comparison, reinforcing the false impression that the call is complete. Any DataLayer client that follows the same pattern is vulnerable.

---

### Recommendation

Change `valid()` to accept an externally trusted root hash and compare against it:

```rust
pub fn valid(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }

    &existing_hash == expected_root   // compare against trusted external root
}
```

Update all call sites (tests, fuzz targets, Python bindings) to supply the trusted root obtained from the `MerkleBlob` or from the on-chain commitment.

### Proof of Concept

```rust
use chia_datalayer::{Hash, Side};
use chia_datalayer::merkle::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};

fn forged_proof_passes_valid() {
    // Attacker wants to claim node_hash is in the tree.
    let node_hash: Hash = [0xAA; 32];

    // Construct one internally-consistent layer with arbitrary other_hash.
    let other_hash: Hash = [0x00; 32];
    let combined_hash = chia_datalayer::calculate_internal_hash(
        &node_hash, Side::Left, &other_hash,
    );

    let proof = ProofOfInclusion {
        node_hash,
        layers: vec![ProofOfInclusionLayer {
            other_hash_side: Side::Left,
            other_hash,
            combined_hash,
        }],
    };

    // valid() returns true even though combined_hash is not the real tree root.
    assert!(proof.valid());
    // proof.root_hash() == combined_hash (attacker-chosen), not the real root.
}

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
