### Title
Integer Division Truncation in `feeCap` Computation Enables Zero-Fee Cosmos Transaction Bypass — (`ante/evm/fee_checker.go`)

---

### Summary

`NewDynamicFeeChecker` computes the per-gas fee cap via integer division (`fee / gas`). When the total fee amount is smaller than the gas limit, the truncated `feeCap` becomes zero. When `baseFee` is also zero (a valid and reachable chain state), the zero `feeCap` passes the sufficiency check, and the resulting `effectiveFee` is zero — meaning the sender pays nothing despite having submitted a non-zero fee field.

---

### Finding Description

In `ante/evm/fee_checker.go` line 83:

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
```

`fee.Quo` is integer (truncating) division. If `fee = 1 aevmos` and `gas = 21000`, then `feeCap = 0`. [1](#0-0) 

The truncated `feeCap` is then compared against `baseFeeInt`:

```go
if feeCap.LT(baseFeeInt) {
    return nil, 0, errorsmod.Wrapf(...)
}
```

When `baseFee = 0`, the check `0 >= 0` passes. The effective price is then computed:

```go
effectivePrice := sdkmath.NewIntFromBigInt(types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
```

`EffectiveGasPrice` returns `min(tipCap + baseFee, feeCap)`: [2](#0-1) 

With `feeCap = 0` and `baseFee = 0`, `effectivePrice = min(tip + 0, 0) = 0`. The final deducted fee is:

```go
effectiveFee = effectivePrice * gas = 0 * gas = 0
``` [3](#0-2) 

The sender pays **zero** fees regardless of the `fee` field they submitted.

---

### Impact Explanation

When `baseFee = 0`, any Cosmos transaction whose `fee < gas` (in absolute token units) will have `feeCap` truncated to zero, pass the fee sufficiency check, and commit with `effectiveFee = 0`. The sender's account is not debited. This is a fee-market ante handler bug that permits transactions to commit with mis-accounted (zero) fees, matching the allowed High impact: *"ante handler bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

Additionally, the EIP-1559 path never falls back to `checkTxFeeWithValidatorMinGasPrices` when `baseFee = 0` (because `GetBaseFee` returns `big.NewInt(0)`, not `nil`): [4](#0-3) 

So validator-local `minGasPrices` enforcement is also bypassed in both `CheckTx` and `DeliverTx` modes.

---

### Likelihood Explanation

`baseFee = 0` is a reachable, non-adversarial chain state:

1. **`NoBaseFee = true`** is a governance-settable parameter. When set, `CalculateBaseFee` returns `nil` and the stored `BaseFee` param is not updated; if it was previously driven to zero it stays zero.
2. **`minGasPrice = 0`** (the default) means `CalculateBaseFee` has no lower bound and can drive `BaseFee` to zero over time when blocks are under the gas target.
3. `validateBaseFee` explicitly permits `BaseFee = 0` (only negative values are rejected): [5](#0-4) 

Any unprivileged user can craft a Cosmos tx with `fee = 1 aevmos` and `gas = 21000` to trigger this path.

---

### Recommendation

Scale the numerator before dividing, analogous to the TWAP oracle fix:

```go
// Before (truncates to 0 when fee < gas):
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))

// After: use LegacyDec for precision, then compare against baseFee in the same unit
feeCapDec := sdkmath.LegacyNewDecFromInt(fee).QuoInt64(int64(gas))
if feeCapDec.LT(sdkmath.LegacyNewDecFromInt(baseFeeInt)) {
    return nil, 0, errorsmod.Wrapf(...)
}
```

Alternatively, guard against the zero case explicitly:

```go
if gas == 0 {
    return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, "gas must be > 0")
}
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
if feeCap.IsZero() && !baseFeeInt.IsZero() { ... }
```

Also note: if `gas = 0` reaches line 83, `fee.Quo(sdkmath.NewIntFromUint64(0))` will **panic** (Go `math/big` division by zero), which is a separate chain-halt vector that should be guarded before this line.

---

### Proof of Concept

**Setup**: London HF enabled, `NoBaseFee = true` (or `minGasPrice = 0` and `BaseFee` has reached 0).

**Attack**:
1. Attacker submits a Cosmos tx (non-EVM) with `fee = [1 aevmos]` and `gas = 21000`.
2. `NewDynamicFeeChecker` is invoked. `baseFee = GetBaseFee(...) = big.NewInt(0)` (not nil, so EIP-1559 path is taken).
3. `feeCap = 1.Quo(21000) = 0` (integer truncation).
4. `feeCap.LT(baseFeeInt)` → `0 < 0` → **false** → check passes.
5. `effectivePrice = EffectiveGasPrice(0, 0, MaxInt64) = min(MaxInt64, 0) = 0`.
6. `effectiveFee = 0 * 21000 = 0 aevmos`.
7. Returned `effectiveFee = []sdk.Coins{}` (empty / zero). Cosmos SDK deducts nothing from the sender.
8. Transaction commits. Sender paid **0 fees**. [6](#0-5)
