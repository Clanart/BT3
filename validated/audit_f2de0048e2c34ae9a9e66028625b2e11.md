### Title
Missing Declared Class Validation in `execute_replace_class` Enables Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not validate that the new class hash supplied by a contract corresponds to a previously declared contract class. Any contract can call `replace_class` with an arbitrary, undeclared felt value as the new class hash. The OS silently commits this state change. All subsequent calls to that contract will fail permanently because the OS cannot resolve the class for execution. Any funds held in the contract's storage are permanently frozen.

---

### Finding Description

In `syscall_impls.cairo`, the function `execute_replace_class` processes the `replace_class` syscall: [1](#0-0) 

At line 896–898, the class hash is read from the syscall request and immediately used to update the contract state: [2](#0-1) 

The critical missing check is explicitly acknowledged by the developers themselves with a TODO comment at line 898:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
```

No assertion, dict lookup, or range check is performed to confirm that `request.class_hash` exists in `contract_class_changes` (the declared class registry). The `dict_update` at line 906 unconditionally writes the arbitrary hash into the contract's `StateEntry`.

This is structurally analogous to the ERC777 report's root cause: a privileged state-mutation path (there, `ERC1820Registry.setInterfaceImplementer`; here, `replace_class`) accepts attacker-controlled input without validating that the registered value is legitimate, enabling a future operation (token hook / contract call) to fail catastrophically.

---

### Impact Explanation

Once a contract's `class_hash` field in `contract_state_changes` is set to an undeclared value, every future call to that contract — whether via `call_contract`, `library_call`, or a top-level transaction — will attempt to look up and execute a class that does not exist. The execution will fail at the class-resolution step. Because the `replace_class` state change was already committed to the proven state, there is no recovery path. Any ERC-20 balances, vault deposits, or other assets stored in the contract's storage dictionary (`storage_ptr`) are permanently inaccessible.

**Impact class**: Critical — Permanent freezing of funds.

---

### Likelihood Explanation

The `replace_class` syscall is callable by any contract from within its own execution context. No privileged role is required beyond deploying a contract. Realistic trigger paths include:

1. A contract with a governance or upgrade mechanism that does not itself validate the new class hash before calling `replace_class` — an attacker who can influence the governance vote (e.g., via flash-loan-style token accumulation in a DAO) can pass an invalid hash.
2. A developer error: a contract passes a hash that was never declared (e.g., a Sierra class hash before the corresponding `declare` transaction is confirmed), permanently bricking the contract.
3. A malicious contract intentionally self-bricks after receiving user deposits, freezing those funds.

Because the OS provides no safety net, any of these paths results in an irreversible state corruption.

---

### Recommendation

Inside `execute_replace_class`, before writing the new class hash to `contract_state_changes`, add a lookup into `contract_class_changes` to assert that the requested `class_hash` maps to a non-zero compiled class hash:

```cairo
// Verify the class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the existing pattern used in `execute_deploy`, where the constructor is executed against the class — implicitly requiring the class to be resolvable. The `replace_class` path must enforce the same invariant explicitly, since it does not execute any entry point at the time of the call.

---

### Proof of Concept

1. Attacker deploys Contract A (a vault) that accepts user deposits and exposes an `upgrade(new_class_hash)` entry point that calls `replace_class`.
2. Users deposit funds into Contract A.
3. Attacker calls `upgrade(0xdeadbeef)` where `0xdeadbeef` is not a declared class hash.
4. The OS executes `execute_replace_class`:
   - Reads `class_hash = 0xdeadbeef` from the syscall request. [3](#0-2) 
   - Skips the missing declared-class check (the TODO line).
   - Writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes`. [4](#0-3) 
5. The transaction succeeds and the proof is generated. The on-chain state now records Contract A's class hash as `0xdeadbeef`.
6. Any subsequent `call_contract` or top-level invoke targeting Contract A fails at class resolution — no `withdraw` or `transfer` entry point can ever be executed again.
7. All user deposits in Contract A's storage are permanently frozen.

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
