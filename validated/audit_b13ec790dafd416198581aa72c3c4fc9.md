### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the replacement class hash supplied by a contract is actually declared in the class registry. A contract that calls `replace_class` with an undeclared hash will have its class permanently set to an inaccessible value, freezing all funds it holds with no recovery path.

### Finding Description
`execute_replace_class` in `syscall_impls.cairo` reads the current `StateEntry` for the calling contract and writes a new `StateEntry` with the caller-supplied `class_hash`, then commits the update to `contract_state_changes`:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment at line 898 explicitly acknowledges the missing validation. No check is performed against `contract_class_changes` (the declared-class registry) to confirm that `class_hash` has a corresponding compiled class hash entry. The OS accepts and commits any felt value as the new class hash.

Contrast this with `execute_declare_transaction`, which enforces `prev_value=0` to guarantee a class is declared at most once:

```cairo
// Note that prev_value=0 enforces that a class may be declared only once.
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
```

And `deploy_contract`, which asserts the target address is uninitialized before writing:

```cairo
assert state_entry.class_hash = UNINITIALIZED_CLASS_HASH;
assert state_entry.nonce = 0;
```

`execute_replace_class` has no equivalent guard.

### Impact Explanation
Once the OS commits a `StateEntry` whose `class_hash` points to an undeclared class:

1. The state root is updated on L1 with the invalid class hash.
2. Every subsequent attempt to execute any entry point on that contract will fail at class-lookup time — the class is not in the declared registry.
3. There is no OS-level mechanism to undo a committed state root; the contract is permanently bricked.
4. All ERC-20 balances, ETH, STRK, or any other token balances held by the contract's storage are permanently frozen with no withdrawal path.

This matches **Critical — Permanent freezing of funds**.

### Likelihood Explanation
The entry path is reachable by an unprivileged transaction sender whenever a deployed contract exposes a callable upgrade path (e.g., a governance-controlled `upgrade(new_class_hash)` function) that forwards the caller-supplied hash directly to `replace_class`. The OS is the intended last line of defense: even if the contract's own access control is bypassed or absent, the OS should reject an undeclared replacement hash. Because the OS performs no such check, any contract with an insufficiently guarded upgrade function becomes exploitable by any unprivileged caller. The pattern of upgradeable contracts on StarkNet is common, making this a realistic protocol-level risk.

### Recommendation
Before committing the new `StateEntry`, verify that `class_hash` exists as a key in `contract_class_changes` (or the global declared-class trie) with a non-zero compiled class hash:

```cairo
// Verify the replacement class is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the protection already present in `execute_declare_transaction` and `deploy_contract`.

### Proof of Concept

1. Deploy contract `C` holding funds, with a public `upgrade(new_class_hash: felt)` entry point that calls `replace_class(new_class_hash)` without access control (or with bypassed access control).
2. An unprivileged sender submits an invoke transaction calling `C.upgrade(0xdeadbeef)` where `0xdeadbeef` is not a declared class hash.
3. The OS executes `execute_replace_class` at [1](#0-0) , skips the missing declared-class check (acknowledged at line 898), and writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes`.
4. `state_update` in `state.cairo` squashes and commits the change, updating the global state root. [2](#0-1) 
5. The new state root is posted to L1. Contract `C`'s class hash is now permanently `0xdeadbeef`.
6. All future calls to `C` fail at class resolution; all funds in `C` are permanently frozen.

The root cause — the absent declared-class guard — is located at: [3](#0-2)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
