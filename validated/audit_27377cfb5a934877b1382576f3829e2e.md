### Title
Stale `feemarketParams` Snapshot in `newCosmosAnteHandler` Bypasses Current `MinGasPrice` and `BaseFee` Enforcement for Cosmos Transactions — (File: `evmd/ante/handler_options.go`)

### Summary

`newCosmosAnteHandler` reads `feemarketParams` from state exactly once at ante-handler construction time and passes a pointer to that snapshot into both `NewDynamicFeeChecker` and `MinGasPriceDecorator`. Because the EIP-1559 base fee is updated every block and `MinGasPrice` can be raised by governance, the Cosmos-tx ante handler enforces stale fee requirements for the entire lifetime of the node process. Any unprivileged user can submit Cosmos (non-EVM) transactions paying fees anchored to the startup-time base fee, bypassing the current, higher market rate.

### Finding Description

**Root cause — stale snapshot captured at construction time**

`newCosmosAnteHandler` in `evmd/ante/handler_options.go` reads params once and captures them:

```go
evmParams := options.EvmKeeper.GetParams(ctx)          // snapshot at construction
feemarketParams := options.FeeMarketKeeper.GetParams(ctx) // snapshot at construction
...
txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
...
cosmos.NewMinGasPriceDecorator(options.FeeMarketKeeper, evmDenom, &feemarketParams),
``` [1](#0-0) 

Both decorators receive a pointer to the same local `feemarketParams` variable. They use it directly for every subsequent transaction:

- `NewDynamicFeeChecker` calls `types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)`, which reads `feemarketParams.GetBaseFee()` — the stale snapshot value. [2](#0-1) [3](#0-2) 

- `MinGasPriceDecorator.AnteHandle` reads `mpd.feemarketParams.MinGasPrice` — also the stale snapshot. The `feesKeeper` field is stored but never queried inside `AnteHandle`. [4](#0-3) 

**Contrast with the EVM ante handler**

`newEthAnteHandler` is a closure that calls `options.EvmKeeper.EVMBlockConfig(ctx, ...)` on **every** invocation, reading fresh `feemarketParams` and `baseFee` from the KV store each time: [5](#0-4) 

`EVMBlockConfig` reads `feemarketParams.GetBaseFee()` from the live store on every call: [6](#0-5) 

**The base fee changes every block**

`feemarket.BeginBlock` recalculates and persists a new base fee every block via `k.SetBaseFee(ctx, baseFee)`, which overwrites `params.BaseFee` in the KV store: [7](#0-6) [8](#0-7) 

The Cosmos ante handler's `feemarketParams.BaseFee` is never updated after construction; it permanently reflects the value at node startup.

### Impact Explanation

Cosmos (non-EVM) transactions are fee-validated against the startup-time base fee and `MinGasPrice`. After a sustained period of high block utilization the live base fee can be orders of magnitude higher than the startup value. An attacker can submit Cosmos transactions paying fees anchored to the stale (lower) base fee, committing them at below-market cost. This is a direct fee mis-accounting: the chain accepts and commits transactions that should be rejected by the current fee market, and the fee collector receives less than the protocol requires. This matches the allowed High impact: *"ante handler bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

### Likelihood Explanation

The base fee is adjusted every block. On any chain that experiences even moderate sustained load the live base fee will diverge from the startup value within minutes. `DynamicFeeChecker` is enabled unconditionally in the production app (`DynamicFeeChecker: true`), so the stale path is always active. No special privileges are required; any user submitting a Cosmos transaction triggers the vulnerable code path. [9](#0-8) 

### Recommendation

Inside `newCosmosAnteHandler`, do not snapshot `feemarketParams` at construction time. Instead, follow the same pattern as `newEthAnteHandler`: read fresh params from the keeper on every transaction invocation. Concretely:

1. Remove the `feemarketParams` local variable from `newCosmosAnteHandler`.
2. Change `MinGasPriceDecorator.AnteHandle` to call `mpd.feesKeeper.GetParams(ctx)` (the keeper is already stored but unused) instead of reading `mpd.feemarketParams`.
3. Change `NewDynamicFeeChecker` to accept a `FeeMarketKeeper` interface and call `keeper.GetParams(ctx)` inside the returned closure, rather than capturing a `*feemarkettypes.Params` pointer at construction time.

### Proof of Concept

1. Node starts; startup base fee = **B₀** (e.g. 1 000 gwei). `newCosmosAnteHandler` captures `feemarketParams.BaseFee = B₀`.
2. Blocks fill above the gas target; after *N* blocks the live base fee rises to **Bₙ** (e.g. 10 000 gwei). `feemarket.BeginBlock` writes `Bₙ` to the KV store each block.
3. Attacker constructs a Cosmos `MsgSend` with `fee = B₀ × gasLimit` (one-tenth of the current required fee).
4. `NewDynamicFeeChecker` evaluates `feeCap = fee / gas = B₀`; compares against `baseFeeInt = B₀` (stale snapshot) → `feeCap >= baseFeeInt` → **passes**.
5. `MinGasPriceDecorator` reads `mpd.feemarketParams.MinGasPrice` (stale) → **passes**.
6. Transaction is committed. The fee collector receives `B₀ × gasLimit` instead of the correct `Bₙ × gasLimit`, mis-accounting fees by a factor of `Bₙ/B₀`.

func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
	if !IsLondon(ethCfg, height) {
		return nil
	}

	// should not be nil if london hardfork enabled
	if baseFee == nil {
		return new(big.Int)
	}
	return baseFee

func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
