### Title
`nativeFee` ETH Permanently Locked in `OmniBridge.sol` With No Withdrawal Mechanism - (File: `evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

Every call to `OmniBridge.initTransfer` that includes a non-zero `nativeFee` sends ETH into the contract. That ETH is never forwarded to any relayer and there is no function in the contract to withdraw it. The funds are permanently locked.

---

### Finding Description

`OmniBridge.initTransfer` is `payable` and accepts a `nativeFee` parameter. The function computes `extensionValue` by subtracting `nativeFee` from `msg.value`:

- For ERC20 transfers: `extensionValue = msg.value - nativeFee`
- For ETH transfers: `extensionValue = msg.value - amount - nativeFee` [1](#0-0) 

`extensionValue` (not `nativeFee`) is what gets passed to `initTransferExtension`. In the base `OmniBridge` contract, `initTransferExtension` simply reverts if `value != 0`, meaning `extensionValue` must be zero and the entire `msg.value` for ERC20 transfers equals `nativeFee` — all of which stays in the contract. [2](#0-1) 

In `OmniBridgeWormhole`, `initTransferExtension` uses `value` (i.e., `extensionValue`) to pay the Wormhole message fee — explicitly **not** `nativeFee`. The `nativeFee` portion of `msg.value` is still left in the contract. [3](#0-2) 

The relayer's native-fee compensation is handled entirely on the NEAR side: `send_fee_internal` mints wrapped native tokens to the relayer on NEAR for non-NEAR origin chains. The ETH `nativeFee` sent to the EVM contract plays no role in that payout and is never consumed. [4](#0-3) 

A search of the entire `OmniBridge.sol` reveals no `withdraw`, `withdrawFees`, or equivalent function. The contract does have a `receive()` fallback, confirming it can hold ETH, but provides no path to recover it. [5](#0-4) 

---

### Impact Explanation

Every user who calls `initTransfer` with `nativeFee > 0` permanently loses that ETH to the contract. Over time, the cumulative locked ETH grows with every bridging operation that includes a native fee. There is no upgrade path that can recover these funds without a full contract migration, and even an upgrade cannot retroactively return ETH already locked. This satisfies the **Critical** impact criterion: permanent freezing / irrecoverable lock of user funds in the bridge.

---

### Likelihood Explanation

`nativeFee` is a first-class parameter of the bridge's transfer flow, documented and emitted in `InitTransfer` events. Any user or dApp that sets `nativeFee > 0` to incentivize relayers triggers the lock. This is the normal operating path for time-sensitive transfers, making the likelihood **High**. [6](#0-5) 

---

### Recommendation

Add an admin-gated withdrawal function to `OmniBridge.sol` that allows the protocol to recover accumulated native fees:

```solidity
function withdrawNativeFees(address payable recipient, uint256 amount)
    external
    onlyRole(DEFAULT_ADMIN_ROLE)
{
    (bool success, ) = recipient.call{value: amount}("");
    require(success, "ETH transfer failed");
}
```

Alternatively, forward `nativeFee` directly to a designated fee-collector address inside `initTransfer` rather than leaving it in the contract.

---

### Proof of Concept

1. User calls `OmniBridge.initTransfer(tokenAddress=USDC, amount=1000e6, fee=10e6, nativeFee=0.01 ether, recipient="alice.near", message="")` sending `msg.value = 0.01 ether`.
2. Inside `initTransfer`: `extensionValue = msg.value - nativeFee = 0`. USDC is transferred from the user. `initTransferExtension` is called with `value=0` (passes the zero-check).
3. The `0.01 ether` `nativeFee` remains in the contract's ETH balance.
4. On NEAR, the relayer calls `claim_fee`; `send_fee_internal` mints wrapped ETH tokens to the relayer — the EVM contract's ETH balance is never touched.
5. Repeat for every user transfer. The ETH balance of `OmniBridge.sol` grows monotonically with no withdrawal path. [7](#0-6) [8](#0-7)

                || transfer_message.get_origin_chain(),
                |origin_transfer_id| origin_transfer_id.origin_chain,
            );

            if origin_chain.is_utxo_chain() {
                env::panic_str(BridgeError::NativeFeeForUtxoChain.to_string().as_str())
            } else if origin_chain == ChainKind::Near {
                Promise::new(fee_recipient.clone())
                    .transfer(NearToken::from_yoctonear(transfer_message.fee.native_fee.0))
                    .detach();
            } else {
                ext_token::ext(self.get_native_token_id(origin_chain))
                    .with_static_gas(MINT_TOKEN_GAS)
                    .mint(fee_recipient.clone(), transfer_message.fee.native_fee, None)
                    .detach();
            }

                transfer_message: transfer_message.clone(),
