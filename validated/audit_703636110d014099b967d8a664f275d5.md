### Title
Missing Nonce in `compute_meta_tx_v0_hash` Enables Unbounded Signature Replay — (`File: transaction_hash/transaction_hash.cairo`)

---

### Summary

The `compute_meta_tx_v0_hash` function in the StarkNet OS omits any nonce or sequencing field from the hash preimage. Because the resulting hash is purely a function of `(contract_address, selector, calldata, chain_id)`, a valid signature over a `meta_tx_v0` call is replayable indefinitely by any unprivileged caller, enabling direct loss of funds from any contract that relies on the `meta_tx_v0` syscall for gasless execution.

---

### Finding Description

`compute_meta_tx_v0_hash` is defined in `transaction_hash/transaction_hash.cairo` and delegates to `deprecated_get_transaction_hash` with `additional_data_size=0` and `additional_data=cast(0, felt*)`: [1](#0-0) 

The resulting hash covers only `(INVOKE_HASH_PREFIX, version=0, contract_address, selector, H(calldata), max_fee=0, chain_id)`. No nonce, no block number, no timestamp — nothing that changes between invocations with the same calldata. [2](#0-1) 

The syscall handler in `execution/syscall_impls.cairo` constructs a `TxInfo` with `nonce=0` and `version=0`, then calls `contract_call_helper` with this context: [3](#0-2) 

The OS-level nonce enforcement in `check_and_increment_nonce` explicitly skips version-0 transactions: [4](#0-3) 

This means:
1. The OS computes the same hash for the same `(contract_address, selector, calldata)` in every block.
2. No nonce is checked or incremented for the meta-transaction target.
3. A signature that was valid once remains valid forever.

This is structurally identical to the external report's finding: the signature has no expiration and the hash inputs are fully attacker-controlled.

---

### Impact Explanation

Any contract that uses `meta_tx_v0` for gasless execution (e.g., a token contract allowing off-chain-signed transfers) is vulnerable. An attacker who observes a valid `(calldata, signature)` pair on-chain can replay it in any subsequent block by calling a contract that issues the same `meta_tx_v0` syscall. Because the OS-computed hash is identical and the signature remains valid, the target contract's `__execute__` will accept it, executing the transfer again. This enables **direct, unbounded loss of funds** from any account or contract that relies on `meta_tx_v0` for authorization.

**Allowed impact matched:** Critical — Direct loss of funds.

---

### Likelihood Explanation

The attack requires only:
1. Observing a legitimate `meta_tx_v0` call on-chain (public information).
2. Deploying a contract that re-issues the same syscall with the captured signature.

No privileged access, leaked keys, or trusted-role compromise is needed. The attacker is a standard unprivileged transaction sender. The replay can be repeated in every block indefinitely.

---

### Recommendation

Include a nonce in the `compute_meta_tx_v0_hash` preimage, analogous to how `deprecated_get_transaction_hash` uses `additional_data` for the L1-handler nonce: [5](#0-4) 

Specifically, pass `additional_data_size=1` and `additional_data=&nonce` in `compute_meta_tx_v0_hash`, where `nonce` is fetched from the target contract's on-chain state and incremented by the OS after each successful `meta_tx_v0` execution — mirroring the L1-handler pattern.

---

### Proof of Concept

```
1. Alice signs a meta_tx_v0 authorizing:
     contract_address = TOKEN_CONTRACT
     selector        = __execute__
     calldata        = [transfer, Bob, 100_tokens]
   Signature S is broadcast on-chain with a legitimate transaction.

2. Attacker observes S from the mempool/chain.

3. Attacker deploys MaliciousContract with:
     function attack() {
         meta_tx_v0(TOKEN_CONTRACT, __execute__, calldata, S)
     }

4. OS computes:
     hash = H(INVOKE_PREFIX, 0, TOKEN_CONTRACT, __execute__, H(calldata), 0, chain_id)
   — identical to the original hash, because no nonce is included.

5. TOKEN_CONTRACT.__execute__ receives TxInfo{transaction_hash=hash, signature=S},
   verifies S against hash → valid → executes transfer(Bob, 100_tokens).

6. Attacker repeats step 3–5 in every block until Alice's balance is zero.
   Each replay costs only the attacker's gas; no new signature is needed.
``` [6](#0-5) [1](#0-0)
