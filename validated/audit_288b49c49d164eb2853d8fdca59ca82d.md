### Title
`ProofOfInclusion::valid()` Never Validates Against an External Trusted Root — Forged Inclusion Proofs Always Pass - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate is supposed to verify that a key is included in a Merkle tree with a specific root. However, the function derives its "root" from the proof's own data (`last.combined_hash`) rather than from an external trusted root. The final check `existing_hash == self.root_hash()` is a tautology — it is always `true` whenever the loop completes without returning `false`. An attacker can construct a fully fabricated, internally consistent `ProofOfInclusion` for any arbitrary `node_hash`, and `valid()` will return `true`, accepting the forged proof.

### Finding Description

The root cause is in `ProofOfInclusion::valid()` and `ProofOfInclusion::root_hash()`:

```rust
// crates/chia-datalayer/src/merkle/proof_of_inclusion.rs

pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // <-- derived from the proof itself
    } else {
        self.node_hash
    }
}

pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;  // <-- set to calculated_hash
    }

    existing_hash == self.root_hash()  // <-- TAUTOLOGY
}
```

Trace through the final iteration of the loop:
1. `calculated_hash` is computed from `existing_hash` and `layer.other_hash`.
2. The check `calculated_hash != layer.combined_hash` passes (they are equal), so no early return.
3. `existing_hash` is set to `calculated_hash`, which equals `layer.combined_hash` for the last layer.
4. After the loop: `existing_hash` == last `layer.combined_hash` == `self.root_hash()`.

The final assertion `existing_hash == self.root_hash()` is therefore **always `true`** when the loop completes. The function only validates internal self-consistency of the proof structure — it never checks the computed root against any external, independently trusted root value.

This is the direct analog of the external report's vulnerability class: the "authorization check" (proof validation) uses the wrong reference (the proof's own embedded root) instead of the correct reference (an external trusted root), making the check meaningless against adversarial input.

The `ProofOfInclusion` struct derives `Streamable` and is exposed via Python bindings with `pyclass`:

```rust
// crates/chia-datalayer/src/merkle/proof_of_inclusion.rs:8-28
#[cfg_attr(
    feature = "py-bindings",
    pyclass(get_all, from_py_object),
    derive(PyJsonDict, PyStreamable)
)]
#[derive(Clone, Debug, std::hash::Hash, Eq, PartialEq, Streamable)]
pub struct ProofOfInclusion {
    pub node_hash: Hash,
    pub layers: Vec<ProofOfInclusionLayer>,
}
```

An attacker can deserialize or directly construct a `ProofOfInclusion` with an arbitrary `node_hash` and any internally consistent `layers` (where each `combined_hash` equals the hash of the previous hash combined with `other_hash`), and `valid()` returns `true`.

The fuzz target and all tests only call `proof.valid()` on proofs generated from the same live blob, so the tautology is never exposed by existing tests:

```rust

assert!(proof.valid());
``` [1](#0-0) 

### Impact Explanation

Any DataLayer consumer — including Python DataLayer nodes — that calls `proof.valid()` to decide whether a key/value pair is committed to a specific Merkle root will accept a completely fabricated proof. An attacker can:

1. Claim any arbitrary `node_hash` (representing any key/value pair) is included in any tree.
2. Construct internally consistent `layers` that hash up to an attacker-chosen fake root.
3. Submit this proof; `valid()` returns `true`.

This allows forged inclusion proofs to be accepted as valid, enabling an untrusted party to prove invalid DataLayer state. This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."** [2](#0-1) 

### Likelihood Explanation

The `ProofOfInclusion` struct is fully serializable (`Streamable`) and exposed to Python via `pyclass`. Any Python DataLayer code that receives a `ProofOfInclusion` from an untrusted peer and calls `.valid()` to verify it is vulnerable. The attacker needs no privileged access — only the ability to send a crafted serialized `ProofOfInclusion` to a node that calls `.valid()` on it. [3](#0-2) [4](#0-3) 

### Recommendation

`valid()` must accept an external trusted root as a parameter and compare the computed root against it:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }

    &existing_hash == trusted_root  // compare against external trusted root
}
```

All call sites — including the Python binding — must be updated to pass the independently obtained root (e.g., from the local `MerkleBlob::get_root()` or a consensus-committed root hash) rather than relying on the proof's self-reported root.

### Proof of Concept

```rust
use chia_datalayer::{Hash, Side};
use chia_datalayer::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};

// Attacker wants to forge a proof that some arbitrary node_hash is "included"
let fake_node_hash: Hash = [0xAA; 32];
let fake_other_hash: Hash = [0xBB; 32];

// Compute what combined_hash must be for internal consistency
let fake_combined = calculate_internal_hash(&fake_node_hash, Side::Left, &fake_other_hash);

let forged_proof = ProofOfInclusion {
    node_hash: fake_node_hash,
    layers: vec![ProofOfInclusionLayer {
        other_hash_side: Side::Left,
        other_hash: fake_other_hash,
        combined_hash: fake_combined,  // internally consistent
    }],
};

// valid() returns true even though this key was never inserted into any real tree
assert!(forged_proof.valid());
// root_hash() returns fake_combined — an attacker-chosen value, not any real tree root
