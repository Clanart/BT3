### Title
Missing Class Hash Validation in `execute_replace_class` Enables Permanent Freezing of Contract Funds — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS program does not verify that the replacement class hash corresponds to a declared contract class. Any contract can replace its own class hash with an arbitrary, undeclared felt value. Once replaced, all subsequent calls to that contract fail because the OS cannot resolve the class, permanently freezing any funds held in the contract's storage. The OS program itself acknowledges this gap with an explicit TODO comment.

---

### Finding Description

In `execute_replace_class` (`execution/syscall_impls.cairo`, lines 878–916), the OS program accepts any class hash value from the syscall request and writes it directly into `contract_state_changes` without checking whether that hash exists in `contract_class_changes` (i.e., whether it was ever declared):

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

This is structurally inconsistent with the class declaration path. In `execute_declare_transaction`, the OS enforces that the class hash is a valid Sierra class hash (via `finalize_class_hash`) and that the compiled class hash is non-zero before writing to `contract_class_changes`:

```cairo
let expected_class_hash = finalize_class_hash(
    contract_class_component_hashes=contract_class_component_hashes
);
with_attr error_message("Invalid class hash pre-image.") {
    assert [class_hash_ptr] = expected_class_hash;
}
...
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [2](#0-1) 

The `replace_class` path performs none of these checks. The OS program generates a valid proof for a state transition where a contract's class hash is set to an arbitrary undeclared value, and the L1 verifier accepts that proof.

---

### Impact Explanation

After `replace_class` is called with an undeclared class hash, the contract's `StateEntry.class_hash` is set to a value that has no corresponding entry in `contract_class_changes`. Any subsequent call to that contract — including calls to withdraw funds — will fail at class resolution. There is no recovery path: the contract is permanently bricked and all funds in its storage are permanently frozen.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

The `execute_deploy` syscall (syscall_impls.cairo, lines 452–556) allows any contract to deploy child contracts without restriction. [3](#0-2) 

An unprivileged user can:
1. Deploy a malicious vault contract (via `deploy_account` or the `deploy` syscall — no permission check exists).
2. Advertise it as a legitimate yield vault to attract deposits.
3. After accumulating funds, trigger the contract to call `replace_class` with an arbitrary undeclared felt (e.g., `0x1`).
4. The OS program writes the invalid class hash into state without validation, producing a valid proof.
5. All deposited funds are permanently frozen.

No privileged role, leaked key, or operator action is required. The entry point is a standard unprivileged contract deployer and caller.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that the class hash exists in `contract_class_changes` (i.e., it has a non-zero compiled class hash entry). This is precisely what the acknowledged TODO at line 898 calls for:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
```

The fix should perform a `dict_read` on `contract_class_changes` keyed by `request.class_hash` and assert the result is non-zero before proceeding with the `dict_update` on `contract_state_changes`.

---

### Proof of Concept

1. Attacker deploys `MaliciousVault` (a contract whose constructor is valid and whose class is declared).
2. Users deposit tokens into `MaliciousVault` (e.g., via an `__execute__` call that transfers ERC-20 tokens to the contract's address).
3. Attacker calls a function on `MaliciousVault` that internally issues the syscall `replace_class(class_hash=0x1)`, where `0x1` is not present in `contract_class_changes`.
4. `execute_replace_class` in the OS program writes `class_hash=0x1` into `MaliciousVault`'s `StateEntry` with no validation.
5. The OS generates a valid STARK proof for this block; the L1 verifier accepts it.
6. All subsequent calls to `MaliciousVault` (including withdrawal calls) fail at class resolution — the class `0x1` does not exist.
7. All deposited funds are permanently inaccessible. [4](#0-3)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
