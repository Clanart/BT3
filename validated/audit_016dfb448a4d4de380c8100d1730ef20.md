### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Fund Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts an arbitrary, attacker-controlled `class_hash` value without verifying that the hash corresponds to a previously declared contract class. This is directly analogous to the open-redirect pattern in the reference report: a caller-supplied parameter is used without allow-list validation. A malicious or buggy contract can permanently replace its own class hash with an undeclared value, rendering the contract permanently non-executable and freezing all funds held within it. The OS will commit this invalid state transition to the state trie without objection.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the caller-supplied `class_hash` directly from the syscall request and writes it into `contract_state_changes` with no check that the hash exists in `contract_class_changes` (the set of declared classes):

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The developer-inserted TODO comment at line 898 explicitly acknowledges the missing validation: `// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.` [1](#0-0) 

The analogous fix in the reference report added an allow-list check (`allowDomains.includes(domain)`) before proceeding. Here, the equivalent check — verifying `class_hash` exists in `contract_class_changes` — is entirely absent.

By contrast, the `execute_declare_transaction` path correctly enforces `prev_value=0` to prevent re-declaration and calls `finalize_class_hash` to validate the class hash pre-image before writing to `contract_class_changes`: [2](#0-1) 

No equivalent guard exists in `execute_replace_class`.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value, every subsequent call to that contract will fail at class-lookup time because no bytecode exists for the hash. The contract becomes permanently non-executable. Any ERC-20 tokens, ETH, or other assets held in the contract's storage are irrecoverably frozen. The OS commits this broken state to the Merkle trie, making it part of the canonical chain state with no recovery path.

---

### Likelihood Explanation

The `replace_class` syscall is callable by any deployed contract on itself — no privileged role is required. An attacker needs only to:

1. Deploy a contract (permissionless on StarkNet).
2. Have the contract invoke the `replace_class` syscall with an arbitrary felt value as `class_hash`.

The OS will process the syscall, skip the missing validation, and commit the invalid class hash to state. The attack is deterministic, requires no race condition, no key compromise, and no operator cooperation.

---

### Recommendation

Before writing the new class hash to `contract_state_changes`, verify that `class_hash` is present in `contract_class_changes` (i.e., it has been declared). The check should assert that a `dict_read` on `contract_class_changes` for `class_hash` returns a non-zero compiled class hash, mirroring the invariant enforced during `execute_declare_transaction`.

```cairo
// Proposed fix (pseudocode):
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

---

### Proof of Concept

1. Attacker deploys contract `C` holding victim funds (e.g., users deposit tokens into `C`).
2. Attacker calls a function in `C` that invokes the `replace_class` syscall with `class_hash = 0xdeadbeef` (an arbitrary undeclared felt).
3. `execute_replace_class` in the OS reads `request.class_hash = 0xdeadbeef`, skips the missing declared-class check, and calls `dict_update` to set `C`'s class hash to `0xdeadbeef` in `contract_state_changes`.
4. The OS commits this state to the block output. The state trie now records `C` as having class hash `0xdeadbeef`.
5. Any subsequent `call_contract` or `invoke` targeting `C` fails at class resolution — no bytecode exists for `0xdeadbeef`.
6. All funds in `C`'s storage are permanently frozen with no recovery mechanism. [3](#0-2)
