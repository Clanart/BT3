### Title
Missing Nonce in `compute_meta_tx_v0_hash` Enables Unbounded Signature Replay - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/transaction_hash/transaction_hash.cairo`)

---

### Summary

`compute_meta_tx_v0_hash` computes the meta-transaction hash with `additional_data_size=0`, permanently omitting any nonce. Combined with `execute_meta_tx_v0` hardcoding `nonce=0` in the resulting `TxInfo` and `check_and_increment_nonce` explicitly skipping version-0 transactions, a valid meta-transaction signature can be replayed an unlimited number of times by any unprivileged caller, leading to direct loss of funds from the target account contract.

---

### Finding Description

`compute_meta_tx_v0_hash` delegates to `deprecated_get_transaction_hash` with `additional_data_size=0` and `additional_data=cast(0, felt*)`: [1](#0-0) 

This means the hash is a pure function of `(INVOKE_HASH_PREFIX, 0, contract_address, EXECUTE_SELECTOR, calldata, 0, chain_id)` — no nonce, no block number, no outer-transaction context. The hash is **identical** every time the same calldata is submitted to the same contract on the same chain.

In `execute_meta_tx_v0`, the synthesized `TxInfo` hardcodes `nonce=0`: [2](#0-1) 

`check_and_increment_nonce` explicitly skips all version-0 transactions, so no nonce is ever consumed: [3](#0-2) 

Compare with `compute_l1_handler_transaction_hash`, which correctly includes a nonce in `additional_data`: [4](#0-3) 

The `meta_tx_v0` syscall is reachable from any executing contract via the standard syscall dispatch: [5](#0-4) 

No restriction is placed on which contract may invoke `execute_meta_tx_v0` — any contract executing in the OS can issue this syscall with an arbitrary `contract_address`, `calldata`, and `signature`.

---

### Impact Explanation

**Critical — Direct loss of funds.**

If Alice signs a meta-transaction authorizing a token transfer (e.g., `transfer(Bob, 1000)`), the resulting signature commits to a hash that never changes. An attacker who observes the signature on-chain can:

1. Deploy a contract that calls `meta_tx_v0(contract=Alice, selector=__execute__, calldata=[transfer(Bob, 1000)], sig=alice_sig)`.
2. Submit this outer transaction in any subsequent block.
3. Alice's `__validate__` receives the identical hash and signature, accepts it, and `__execute__` transfers funds again.
4. Repeat until Alice's balance is zero.

The attacker does not need Alice's private key — only the previously broadcast signature.

---

### Likelihood Explanation

**High.** The `meta_tx_v0` syscall is a public protocol feature callable by any deployed contract. Any signature ever used in a meta-transaction is permanently replayable. An attacker only needs to observe one valid meta-transaction on-chain (e.g., from mempool or block history) and then submit their own outer transaction wrapping the same meta-tx parameters. No privileged access, leaked key, or social engineering is required.

---

### Recommendation

Include a nonce in the meta-transaction hash, analogous to how `compute_l1_handler_transaction_hash` includes its nonce in `additional_data`. Specifically:

1. **Add a nonce parameter to `compute_meta_tx_v0_hash`** and pass it as `additional_data_size=1, additional_data=&nonce`.
2. **Store and increment a per-contract meta-tx nonce** in `execute_meta_tx_v0` (separate from the outer transaction nonce, since the outer tx nonce is already consumed by the wrapping invoke).
3. **Remove the version-0 nonce skip** in `check_and_increment_nonce`, or handle meta-tx nonce accounting explicitly in `execute_meta_tx_v0`. [6](#0-5) 

---

### Proof of Concept

**Setup:**
- Alice's account contract is at address `0xALICE`. It uses a standard ECDSA `__validate__` that checks `get_tx_info().transaction_hash`.
- Alice signs a meta-tx authorizing `transfer(Bob, 1000 STRK)`. The OS computes:
  ```
  H = pedersen(INVOKE_PREFIX, 0, 0xALICE, EXECUTE_SELECTOR, hash(calldata), 0, chain_id)
  ```
  Alice's signature `σ` is valid for `H`.

**Attack (repeated in any block):**
1. Attacker deploys `ReplayContract` with an `__execute__` that calls:
   ```
   meta_tx_v0(contract_address=0xALICE, selector=EXECUTE_SELECTOR,
               calldata=[transfer(Bob, 1000)], signature=σ)
   ```
2. Attacker submits an invoke transaction calling `ReplayContract.__execute__`.
3. The OS executes `execute_meta_tx_v0`:
   - Computes the same hash `H` (no nonce → identical inputs → identical hash).
   - Presents `(H, σ)` to Alice's `__validate__`, which passes.
   - Executes Alice's `__execute__`, transferring 1000 STRK to Bob (attacker-controlled).
4. No nonce is incremented for the meta-tx. Step 2–3 can be repeated in every subsequent block until Alice's balance is zero. [7](#0-6) [8](#0-7)
