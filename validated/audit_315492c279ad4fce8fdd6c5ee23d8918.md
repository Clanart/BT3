### Title
`rejected_txs` Silently Dropped During CatchingUp Backlog — Rejected L1 Handler Txs Permanently Marked `Committed` — (`crates/apollo_l1_provider/src/l1_provider.rs`)

---

### Summary

When `L1Provider` is in `CatchingUp` state and the batcher calls `commit_block` with a non-empty `rejected_txs` set for a future height, the `rejected_txs` argument is silently discarded at the routing boundary. The `CommitBlockBacklog` struct stores only `committed_txs`. When the backlog is later drained, `apply_commit_block` is called with `Default::default()` (empty set) for `rejected_txs`, causing every tx that should have been marked `Rejected` to instead be marked `Committed`. The state corruption is permanent: the tx is removed from the `proposable_index` and can never be re-proposed.

---

### Finding Description

**Drop point 1 — routing in `commit_block`:**

`commit_block` accepts both `committed_txs` and `rejected_txs`, but when `is_catching_up()` is true it calls `accept_commit_while_catching_up(committed_txs, height)` — `rejected_txs` is not forwarded. [1](#0-0) 

**Drop point 2 — `CommitBlockBacklog` struct:**

The backlog entry type has no field for `rejected_txs`; only `committed_txs` and `height` are stored. [2](#0-1) 

**Drop point 3 — `add_commit_block_to_backlog` signature:**

The function only accepts `committed_txs`; there is no parameter for `rejected_txs`. [3](#0-2) 

**Drop point 4 — backlog drain uses `Default::default()`:**

When the backlog is applied after sync completes, every entry is committed with an empty `rejected_txs`. [4](#0-3) 

**Drop point 5 — `Equal` branch also uses `Default::default()`:**

Even the non-backlogged path (height matches current height during catchup) drops `rejected_txs`. A developer TODO comment immediately above this line acknowledges the gap. [5](#0-4) 

**How `apply_commit_block` partitions txs:**

The function partitions `consumed_txs` (the batcher's `committed_txs`) into `rejected_and_consumed` (intersection with `rejected_txs`) and `committed_txs` (remainder). With `rejected_txs = {}`, every tx lands in the committed bucket. [6](#0-5) 

**`mark_committed` sets the permanent flag:** [7](#0-6) 

Once `committed = true` and `state = Committed`, `is_validatable()` returns `false` and `is_proposable()` returns `false`, so the tx is evicted from `proposable_index` and can never be returned by `get_txs` again. [8](#0-7) 

---

### Impact Explanation

An L1 handler transaction that the batcher legitimately rejected (e.g., execution reverted) during a catchup window is permanently recorded as `Committed` in the provider. Consequences:

- `get_txs` never returns the tx again — the L1→L2 message is silently swallowed.
- `validate` returns `AlreadyIncludedOnL2` — an authoritative-looking wrong value served to RPC callers.
- The `proposable_index` invariant ("contains all hashes of transactions that are proposable, and only them") is violated. [9](#0-8) 

---

### Likelihood Explanation

The catchup path is exercised on every node restart or crash recovery. Any block committed by the batcher while the provider is still syncing — a routine occurrence — that contains a rejected L1 handler tx triggers the bug. The developer TODO comment at line 426 confirms the gap was known but unresolved. [5](#0-4) 

---

### Recommendation

1. Add `rejected_txs: IndexSet<TransactionHash>` to `CommitBlockBacklog`.
2. Thread `rejected_txs` through `accept_commit_while_catching_up` and `add_commit_block_to_backlog`.
3. Pass the stored `rejected_txs` (not `Default::default()`) in both the `Equal` branch and the backlog drain loop.

---

### Proof of Concept

```
1. Initialize provider (start_height=0, current_height=0, state=Pending).
2. Batcher calls commit_block(committed_txs={}, rejected_txs={}, height=5)
   → provider enters CatchingUp, target_height=5.
3. Scraper adds tx1 (L1 handler tx).
4. Batcher calls commit_block(committed_txs={tx1}, rejected_txs={tx1}, height=6)
   → height 6 > current_height → backlogged with committed_txs={tx1}; rejected_txs dropped.
5. Sync task sends commit_block for heights 0..=5 (no rejected txs).
   After height 5: current_height=6, is_caught_up(6) = 6 > 5 = true.
6. Backlog drain: apply_commit_block({tx1}, Default::default()).
   partition: rejected_and_consumed=[], committed=[tx1].
   commit_txs([tx1], []) → tx1.mark_committed().
7. Assert: tx1.state == Committed  ← BUG; expected Rejected.
8. get_txs() returns []; validate(tx1) returns AlreadyIncludedOnL2.
```

    fn apply_commit_block(
        &mut self,
        consumed_txs: IndexSet<TransactionHash>,
        rejected_txs: IndexSet<TransactionHash>,
    ) {
        debug!("Applying commit_block to height: {}", self.current_height);
        let (rejected_and_consumed, committed_txs): (Vec<_>, Vec<_>) =
            consumed_txs.iter().copied().partition(|tx| rejected_txs.contains(tx));
        self.tx_manager.commit_txs(&committed_txs, &rejected_and_consumed);

        self.current_height = self.current_height.unchecked_next();
    }
