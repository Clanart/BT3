### Title
`MerkleBlob::upsert` Missing Hash Collision Guard Corrupts `leaf_hash_to_index` Cache — (`crates/chia-datalayer/src/merkle/blob.rs`)

### Summary
`MerkleBlob::upsert` does not check whether `new_hash` already belongs to a different existing leaf before updating the cache. When called with a hash that belongs to another leaf, it silently overwrites the `leaf_hash_to_index` entry for that hash, causing `get_node_by_hash` to resolve to the wrong leaf and leaving two blob entries sharing the same hash.

### Finding Description
`insert` guards against hash collisions at lines 372–374 with `contains_leaf_hash`. `upsert` has no equivalent guard. The sequence in `upsert` is:
1. `remove_leaf(&leaf_A)` — removes `hash_A` from cache, making `hash_B` the only entry
2. `insert_entry_to_blob(index_A, block{hash=hash_B})` → `add_leaf` → `HashMap::insert(hash_B, index_A)` — silently overwrites `hash_B → index_B`

The blob now contains two leaves with identical hashes (`hash_B`), and the cache maps `hash_B` to the wrong index.

### Impact Explanation
- `get_node_by_hash(hash_B)` returns `(key_A, value_A)` instead of `(key_B, value_B)` — forged DataLayer state lookup
- The Merkle tree has two leaves with the same hash, making the root hash structurally invalid
- Proofs of inclusion for leaf B resolve to leaf A's data, enabling forged DataLayer state acceptance
- Matches: **High — DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state**

### Likelihood Explanation
The `upsert` API is public and exposed via Python/wasm bindings. The `new_hash` parameter is fully caller-controlled with no enforcement that it derives from the key/value. Any caller who controls one key (key_A) and knows another leaf's hash (hash_B) can trigger this. No privileged access is required.

### Recommendation
Add the same hash collision guard that `insert` uses, but only against *other* leaves (excluding the leaf being updated):

```rust
pub fn upsert(&mut self, key: KeyId, value: ValueId, new_hash: &Hash) -> Result<(), Error> {
    let Ok((leaf_index, mut leaf, mut block)) = self.get_leaf_by_key(key) else {
        self.insert(key, value, new_hash, InsertLocation::Auto {})?;
        return Ok(());
    };

    // Guard: new_hash must not already belong to a *different* leaf
    if leaf.hash != *new_hash && self.block_status_cache.contains_leaf_hash(new_hash) {
        return Err(Error::HashAlreadyPresent());
    }

    self.block_status_cache.remove_leaf(&leaf)?;
    // ... rest unchanged
```
