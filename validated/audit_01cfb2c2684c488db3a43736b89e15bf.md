### Title
Unvalidated `builder` Field in `ProposalInit` Allows Arbitrary `sequencer_address` in Block Hash and Fee Collection — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`ProposalInit` carries two distinct identity fields: `proposer` (the consensus-layer identity, checked by `single_height_consensus`) and `builder` (the execution-layer identity, mapped directly to `sequencer_address` in `BlockInfo`). The `is_block_info_valid` function in `validate_proposal.rs` never validates `builder` against any expected value. A malicious proposer can set `builder` to an arbitrary address; every validator will re-execute the block with that address as `sequencer_address`, producing a wrong block hash, redirecting all transaction fees, and exposing a wrong value through the `get_execution_info` syscall.

---

### Finding Description

`ProposalInit` has two address fields:

```
pub proposer: ContractAddress,  // consensus identity
pub builder: ContractAddress,   // execution/sequencer identity
``` [1](#0-0) 

`single_height_consensus::handle_proposal` checks `init.proposer` against the committee-derived proposer:

```rust
if init.proposer != proposer_id {
    warn!("Invalid proposer: expected {:?}, got {:?}", proposer_id, init.proposer);
    return VecDeque::new();
}
``` [2](#0-1) 

However, `is_block_info_valid` — the only block-info validation function called during proposal validation — checks `height`, `l1_da_mode`, `l2_gas_price_fri`, `timestamp`, `l1_gas_price_fri`, and `l1_data_gas_price_fri`, but **never touches `builder`**:

```rust
if !(init_proposed.height == block_info_validation.height
    && init_proposed.l1_da_mode == block_info_validation.l1_da_mode
    && init_proposed.l2_gas_price_fri == block_info_validation.l2_gas_price_fri)
``` [3](#0-2) 

`convert_to_sn_api_block_info` then maps `init.builder` directly to `sequencer_address`:

```rust
Ok(starknet_api::block::BlockInfo {
    block_number: init.height,
    block_timestamp: BlockTimestamp(init.timestamp),
    sequencer_address: init.builder,   // ← attacker-controlled
    ...
})
``` [4](#0-3) 

This `BlockInfo` is passed to the batcher via `ValidateBlockInput` in `initiate_validation`:

```rust

``` [5](#0-4) 

The `sequencer_address` then enters `PartialBlockHashComponents`:

```rust

``` [6](#0-5) 

And is chained into the final block hash:

```rust
.chain(&partial_block_hash_components.sequencer.0)
``` [7](#0-6) 

The same `sequencer_address` is used as the fee recipient:

```rust

    let sequencer_address = block_context.block_info.sequencer_address;
    get_address_balance_keys(sequencer_address)
}
``` [8](#0-7) 

And is exposed to contracts via the `get_execution_info` syscall:

```rust

``` [9](#0-8) 

The `update_state_sync_with_new_block` path also uses `init.builder` as the sequencer for the `SyncBlock` header sent to state sync:

```rust
let sequencer = SequencerContractAddress(init.builder);
``` [10](#0-9) 

---

### Impact Explanation

A malicious proposer sets `init.builder` to an attacker-controlled address `A`. Every validator calls `is_block_info_valid` (which ignores `builder`), then `initiate_validation` → `convert_to_sn_api_block_info` → batcher executes the block with `sequencer_address = A`. The resulting `PartialBlockHash` (the `ProposalCommitment` voted on by all validators) already encodes `A` as the sequencer. After quorum, the block is committed with:

1. **Wrong block hash** — `sequencer_address = A` is hashed into the canonical block hash stored in storage and propagated to state sync and RPC. Every downstream consumer (L1 verifier, proof system, RPC `starknet_getBlock`) sees the wrong sequencer address.
2. **Fee theft** — All transaction fees in the block are credited to `A` instead of the legitimate sequencer, constituting a direct economic loss.
3. **Wrong `get_execution_info` result** — Contracts that branch on `sequencer_address` during `__validate__` or `__execute__` see `A`, potentially bypassing sequencer-address guards.

---

### Likelihood Explanation

Any validator who is legitimately selected as proposer for a round can perform this attack with no additional privileges. The `builder` field is transmitted over the network in `ProposalInit` and is fully attacker-controlled. No cryptographic material (signature, RANDAO equivalent) binds `builder` to the proposer's identity. The attack requires only that the malicious node be selected as proposer for one round.

---

### Recommendation

In `is_block_info_valid`, add an explicit check that `init.builder` equals the node's own configured `builder_address` (already stored in `ContextConfig::static_config::builder_address`). Alternatively, add `expected_builder: ContractAddress` to `BlockInfoValidation` and assert equality:

```rust
if init_proposed.builder != block_info_validation.expected_builder {
    return Err(ValidateProposalError::InvalidBlockInfo(
        init_proposed.clone(),
        block_info_validation.clone(),
        format!(
            "Builder mismatch: expected={:?}, proposed={:?}",
            block_info_validation.expected_builder, init_proposed.builder
        ),
    ));
}
``` [11](#0-10)

### Proof of Concept

1. Malicious node M is selected as proposer for block N, round R.
2. M constructs `ProposalInit` with `builder = ATTACKER_ADDRESS` (any address M controls).
3. M broadcasts the proposal. Every validator calls `validate_proposal` → `is_block_info_valid` (passes — `builder` is never checked) → `initiate_validation` → `convert_to_sn_api_block_info` sets `sequencer_address = ATTACKER_ADDRESS`.
4. The batcher executes all transactions with `sequencer_address = ATTACKER_ADDRESS`; fees are credited to `ATTACKER_ADDRESS`.
5. `BlockExecutionArtifacts::new` calls `PartialBlockHashComponents::new(&block_info, ...)` which stores `sequencer = ATTACKER_ADDRESS`.
6. `commitment()` returns a `ProposalCommitment` (partial block hash) that encodes `ATTACKER_ADDRESS`.
7. All validators compute the same commitment (they all used `ATTACKER_ADDRESS`) and vote for it; quorum is reached.
8. `decision_reached` commits the block; `calculate_block_hash` produces a block hash containing `ATTACKER_ADDRESS` as sequencer.
9. `update_state_sync_with_new_block` propagates `sequencer = ATTACKER_ADDRESS` to state sync.
10. The canonical block hash stored in storage and returned by RPC is permanently wrong; all fees from block N are in `ATTACKER_ADDRESS`. [12](#0-11) [13](#0-12) [14](#0-13)

async fn is_block_info_valid(
    block_info_validation: &BlockInfoValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        block_info_validation.previous_block_info.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidBlockInfo(
            init_proposed.clone(),
            block_info_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + block_info_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidBlockInfo(
            init_proposed.clone(),
            block_info_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now, block_info_validation.block_timestamp_window_seconds, init_proposed.timestamp
            ),
        ));
    }
    if !(init_proposed.height == block_info_validation.height
        && init_proposed.l1_da_mode == block_info_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == block_info_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidBlockInfo(
            init_proposed.clone(),
            block_info_validation.clone(),
            "Block info validation failed".to_string(),
        ));
    }

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let previous_block_info = PreviousBlockInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
