### Title
`read_pending_data` Fallback Constructs `DeprecatedPendingBlock` With Zero `l1_data_gas_price` / `l2_gas_price`, Causing Wrong Fee Estimates on Pending Block - (File: `crates/apollo_rpc/src/v0_8/api/api_impl.rs`)

---

### Summary

When the in-memory pending data is stale (its `parent_block_hash` does not match the latest committed block hash), `read_pending_data` silently constructs a synthetic `DeprecatedPendingBlock` fallback. The `Deprecated` variant's `l1_data_gas_price()` and `l2_gas_price()` accessors unconditionally return `GasPricePerToken::default()` (zero). This zero propagates into the execution engine's `BlockContext`, where it is silently clamped to `NonzeroGasPrice::MIN`. Any RPC caller invoking `starknet_estimateFee`, `starknet_simulateTransactions`, or `starknet_estimateMessageFee` with `block_id: "pending"` during this window receives an authoritative-looking but wrong fee estimate — the data-gas and L2-gas components are replaced with the minimum possible non-zero price rather than the actual current prices.

---

### Finding Description

**Root cause — `read_pending_data` short-circuit to a zero-price fallback**

`read_pending_data` checks whether the cached pending block's `parent_block_hash` matches the latest committed block hash:

```
if pending_data.block.parent_block_hash() == latest_header.block_hash {
    Ok((*pending_data).clone())          // ← real pending data
} else {
    Ok(PendingData {
        block: PendingBlockOrDeprecated::Deprecated(DeprecatedPendingBlock {
            parent_block_hash: latest_header.block_hash,
            eth_l1_gas_price: ...,
            strk_l1_gas_price: ...,
            ..Default::default()         // ← no l1_data_gas_price, no l2_gas_price
        }),
        ...
    })
}
``` [1](#0-0) 

The `else` branch constructs a `DeprecatedPendingBlock`. That struct has no `l1_data_gas_price` or `l2_gas_price` fields. When the shared `PendingBlockOrDeprecated` accessors are called on the `Deprecated` variant, they return zero by design:

```rust
pub fn l1_data_gas_price(&self) -> GasPricePerToken {
    match self {
        // In older versions, data gas price was 0.
        PendingBlockOrDeprecated::Deprecated(_) => GasPricePerToken::default(),
        ...
    }
}
pub fn l2_gas_price(&self) -> GasPricePerToken {
    match self {
        // In older versions, L2 gas price was 0.
        PendingBlockOrDeprecated::Deprecated(_) => GasPricePerToken::default(),
        ...
    }
}
``` [2](#0-1) 

**Propagation into the execution engine**

`client_pending_data_to_execution_pending_data` copies these zero prices verbatim into `ExecutionPendingData`: [3](#0-2) 

The execution engine then builds a `BlockContext` from `maybe_pending_data`. When a price is zero, `NonzeroGasPrice::new(0)` fails and the code silently falls back to `NonzeroGasPrice::MIN`:

```rust
l1_data_gas_price: NonzeroGasPrice::new(l1_data_gas_price.price_in_wei)
    .unwrap_or(NonzeroGasPrice::MIN),
l2_gas_price: NonzeroGasPrice::new(l2_gas_price.price_in_wei)
    .unwrap_or(NonzeroGasPrice::MIN),
``` [4](#0-3) 

The `BlockContext` built with `NonzeroGasPrice::MIN` for data-gas and L2-gas is then used by `tx_execution_output_to_fee_estimation` to populate the `l1_data_gas_price` and `l2_gas_price` fields of the returned `FeeEstimation`: [5](#0-4) 

**Affected RPC methods**

All three pending-block execution paths call `read_pending_data` and then pass the result to the execution engine:

- `estimate_fee` (lines 1009–1016)
- `simulate_transactions` (lines 1079–1086)
- `estimate_message_fee` (lines 1437–1444) [6](#0-5) 

**Analog to the CErc721 bug**

| CErc721 | Sequencer |
|---|---|
| `accrueInterest()` short-circuits and returns `0` (NO_ERROR) when block is already current | `read_pending_data` short-circuits to `DeprecatedPendingBlock` when pending data is stale |
| Caller uses `0` as the exchange rate | Caller uses `GasPricePerToken::default()` (zero) as `l1_data_gas_price` / `l2_gas_price` |
| Freshness check `assetsExchangeRate == 0` causes revert | `NonzeroGasPrice::new(0).unwrap_or(MIN)` silently substitutes minimum price |
| Liquidation always fails | Fee estimation returns wrong (underestimated) values |

---

### Impact Explanation

Any RPC client calling `starknet_estimateFee`, `starknet_simulateTransactions`, or `starknet_estimateMessageFee` with `block_id: "pending"` during the stale-pending window receives a `FeeEstimation` where `l1_data_gas_price` and `l2_gas_price` are `NonzeroGasPrice::MIN` (effectively 1 fri/wei) instead of the actual current prices. The `overall_fee` field is correspondingly wrong. This is an authoritative-looking wrong value returned by the RPC layer.

Impact: **High — RPC fee estimation and simulation on the pending block returns an authoritative-looking wrong value.**

---

### Likelihood Explanation

The stale-pending window is a normal, frequent occurrence: it opens every time a new block is committed and the pending-data background task has not yet fetched a fresh pending block from the feeder gateway. Any unprivileged user can trigger the wrong result simply by calling `starknet_estimateFee` with `block_id: "pending"` during this window. No special privileges, no malicious peer, no privileged operation is required.

---

### Recommendation

In the `else` branch of `read_pending_data`, construct a `PendingBlockOrDeprecated::Current(PendingBlock { ... })` instead of a `Deprecated` block, copying `l1_data_gas_price`, `l2_gas_price`, and `l1_da_mode` from the latest committed block header. This mirrors the pattern already used for `l1_gas_price` and ensures that the fallback pending block carries the correct current gas prices rather than the legacy zero defaults.

Alternatively, add an explicit guard in the execution-context builder that rejects a zero `l1_data_gas_price` or `l2_gas_price` from pending data with an error rather than silently substituting `NonzeroGasPrice::MIN`.

---

### Proof of Concept

1. Node has committed block N. The in-memory `pending_data` still has `parent_block_hash = hash(N-1)` (stale).
2. Client calls `starknet_estimateFee` with `block_id: "pending"` for a V3 transaction that consumes data gas and L2 gas.
3. `read_pending_data` enters the `else` branch and returns a `DeprecatedPendingBlock`.
4. `l1_data_gas_price()` and `l2_gas_price()` return `GasPricePerToken { price_in_wei: 0, price_in_fri: 0 }`.
5. `client_pending_data_to_execution_pending_data` copies these zeros into `ExecutionPendingData`.
6. The execution engine builds `BlockContext` with `NonzeroGasPrice::MIN` for both prices.
7. `tx_execution_output_to_fee_estimation` returns `FeeEstimation { l1_data_gas_price: 1, l2_gas_price: 1, overall_fee: <underestimated> }`.
8. The client uses this underestimated fee, submits the transaction with insufficient resource bounds, and the transaction is rejected by the sequencer — or the client is misled about the true cost of execution. [7](#0-6) [8](#0-7) [4](#0-3)

async fn read_pending_data<Mode: TransactionKind>(
    pending_data: &Arc<RwLock<PendingData>>,
    txn: &StorageTxn<'_, Mode>,
) -> RpcResult<PendingData> {
    let latest_header = match get_latest_block_number(txn)? {
        Some(latest_block_number) => get_block_header_by_number(txn, latest_block_number)?,
        None => starknet_api::block::BlockHeader {
            // TODO(Shahak): Consider adding genesis hash to the config to support chains that have
            // different genesis hash.
            block_header_without_hash: BlockHeaderWithoutHash {
                parent_hash: BlockHash::GENESIS_PARENT_HASH,
                ..Default::default()
            },
            ..Default::default()
        },
    };
    let pending_data = &pending_data.read().await;
    if pending_data.block.parent_block_hash() == latest_header.block_hash {
        Ok((*pending_data).clone())
    } else {
        Ok(PendingData {
            block: PendingBlockOrDeprecated::Deprecated(DeprecatedPendingBlock {
                parent_block_hash: latest_header.block_hash,
                eth_l1_gas_price: latest_header.block_header_without_hash.l1_gas_price.price_in_wei,
                strk_l1_gas_price: latest_header
                    .block_header_without_hash
                    .l1_gas_price
                    .price_in_fri,
                timestamp: latest_header.block_header_without_hash.timestamp,
                sequencer_address: latest_header.block_header_without_hash.sequencer,
                starknet_version: latest_header
                    .block_header_without_hash
                    .starknet_version
                    .to_string(),
                ..Default::default()
            }),
            state_update: ClientPendingStateUpdate {
                old_root: latest_header.block_header_without_hash.state_root,
                state_diff: Default::default(),
            },
        })
    }
}

```rust
pub(crate) fn tx_execution_output_to_fee_estimation(
    tx_execution_output: &TransactionExecutionOutput,
    block_context: &BlockContext,
) -> ExecutionResult<FeeEstimation> {
    let gas_prices = &block_context.block_info().gas_prices;
    let (l1_gas_price, l1_data_gas_price, l2_gas_price) = (
        gas_prices.l1_gas_price(&tx_execution_output.price_unit.into()).get(),
        gas_prices.l1_data_gas_price(&tx_execution_output.price_unit.into()).get(),
        gas_prices.l2_gas_price(&tx_execution_output.price_unit.into()).get(),
    );

    let gas_vector = tx_execution_output.execution_info.receipt.gas;

    Ok(FeeEstimation {
        gas_consumed: gas_vector.l1_gas.0.into(),
        l1_gas_price,
        data_gas_consumed: gas_vector.l1_data_gas.0.into(),
        l1_data_gas_price,
        l2_gas_price,
        overall_fee: tx_execution_output.execution_info.receipt.fee,
        unit: tx_execution_output.price_unit,
    })
}
