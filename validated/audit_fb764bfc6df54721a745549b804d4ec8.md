### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Contract Lock and Fund Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary felt value as the new class hash without verifying that it corresponds to a declared contract class. This allows any contract to permanently replace its own class with an undeclared hash, rendering itself uncallable and freezing any funds held in its storage.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function unconditionally writes the caller-supplied `class_hash` into `contract_state_changes` with no validation that the hash is declared: [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges the missing check:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
```

The OS therefore commits a state entry whose `class_hash` field may point to a class that was never declared and has no corresponding entry in `contract_class_changes`.

When any future call reaches `execute_entry_point`, the OS performs a two-step lookup:

1. It reads the compiled class hash from `contract_class_changes` keyed by the contract's class hash: [2](#0-1) 

2. It then calls `find_element` against the globally fixed `compiled_class_facts_bundle` using that compiled class hash as the key.

If the class hash set by `replace_class` was never declared, step 1 returns 0 (the dict default), and `find_element` with key 0 will fail with a Cairo assertion error because no compiled class with hash 0 exists in the bundle. This is not a graceful revert — it is a proof-level failure that prevents the OS from producing a valid proof for any block containing a call to the affected contract.

The `replace_class` syscall is dispatched from `execute_syscalls` using the contract's own address, so the attacker controls only their own contract: [3](#0-2) 

The `execute_replace_class` function itself: [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

A contract that calls `replace_class` with an undeclared class hash transitions into a state where:

- Its `class_hash` field in the committed state points to a non-existent class.
- Every subsequent call to the contract causes `find_element` to fail at the OS proof level, making the contract permanently uncallable.
- Any ERC-20 balances, LP positions, or other assets stored in the contract's storage slots become permanently inaccessible.

The state corruption is irreversible: once the invalid class hash is committed to the Merkle tree, no future transaction can execute the contract's code to recover the funds.

---

### Likelihood Explanation

The `replace_class` syscall is available to any Cairo 1 contract with no access control at the OS level. An attacker needs only to:

1. Deploy a contract (or exploit a contract that exposes `replace_class` logic).
2. Call `replace_class` with any felt value that is not a declared class hash (e.g., `1`, or any random felt).
3. The OS accepts the syscall unconditionally and commits the invalid state.

No privileged role, leaked key, or external dependency is required. The entry path is a standard user-submitted invoke transaction.

---

### Recommendation

Inside `execute_replace_class`, before updating `contract_state_changes`, verify that the supplied `class_hash` has a non-zero entry in `contract_class_changes` (i.e., it has been declared and has a known compiled class hash). If the lookup returns 0, write a failure response and return without modifying state, consistent with how other invalid-argument cases are handled (e.g., `write_failure_response` with `ERROR_INVALID_ARGUMENT`). [5](#0-4) 

---

### Proof of Concept

**Step-by-step attack:**

1. Attacker deploys `ContractA` with a valid declared class hash `C1`. `ContractA` holds user funds in storage.
2. Attacker submits an invoke transaction that calls `ContractA.__execute__`, which internally issues the `replace_class` syscall with class hash `C2 = 0xdeadbeef` (never declared).
3. `execute_replace_class` in the OS writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes` with no validation. The transaction succeeds and the state is committed.
4. In any subsequent block, a user submits a transaction calling `ContractA`.
5. `execute_call_contract` reads `state_entry.class_hash = 0xdeadbeef` from `contract_state_changes`.
6. `execute_entry_point` calls `dict_read{dict_ptr=contract_class_changes}(key=0xdeadbeef)` → returns `0` (undeclared).
7. `find_element(..., key=0)` fails with a Cairo assertion error because no compiled class with hash `0` exists in `compiled_class_facts_bundle`.
8. The OS cannot produce a valid proof for the block. The sequencer must exclude all calls to `ContractA`.
9. All funds stored in `ContractA` are permanently frozen.

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
