### Title
Missing `chunk.shard_id()` vs `shard_id` parameter check in `set_state_header` allows unauthenticated peer to install wrong trie for a shard — (`chain/chain/src/state_sync/adapter.rs`)

### Summary
`ChainStateSyncAdapter::set_state_header` accepts a `ShardStateSyncResponseHeader` whose embedded chunk belongs to a different shard S' than the requested shard S. Because `verify_path` does not enforce the leaf's position in the Merkle tree, a chunk for S' with its own valid proof passes the inclusion check. With no explicit `chunk.shard_id() == shard_id` guard, the header is stored under `StateHeaderKey(S, sync_hash)` with `prev_state_root = R_S'`. All subsequent `set_state_part` calls validate against `R_S'`, and `apply_state_part` installs shard S's storage with the trie content of shard S'.

### Finding Description

In `set_state_header` the only inclusion check is:

```rust
// adapter.rs:394-403
if !verify_path(
    *sync_prev_block_header.chunk_headers_root(),
    shard_state_header.chunk_proof(),
    &ChunkHashHeight(chunk.chunk_hash().clone(), chunk.height_included()),
) { ... }
```

`verify_path` is defined as:

```rust
// merkle.rs:113-115
pub fn verify_path<T: BorshSerialize>(root: MerkleHash, path: &MerklePath, item: T) -> bool {
    verify_hash(root, path, CryptoHash::hash_borsh(item))
}
```

which reduces to `compute_root_from_path(path, item_hash) == root` — a pure path-traversal check with no position/index enforcement. The `chunk_headers_root` Merkle tree is built over all shards' chunks in order:

```rust
// block.rs:815-822
pub fn compute_chunk_headers_root(&self) -> (CryptoHash, Vec<MerklePath>) {
    merklize(&self.iter().map(|chunk|
        ChunkHashHeight(chunk.chunk_hash().clone(), chunk.height_included())
    ).collect_vec())
}
```

A chunk for shard S' at position `idx_S'` has a valid proof at that position. Supplying that chunk + proof for a request targeting shard S passes `verify_path` because the function does not check that the leaf is at the shard-S position.

`validate_chunk_proofs` only checks internal body/header consistency:

```rust
// validate.rs:22-66
pub fn validate_chunk_proofs(chunk: &ShardChunk, ...) -> Result<bool, Error> {
    // checks chunk hash, tx_root, receipts_root — no shard_id vs parameter check
}
```

After all checks pass, the header is stored:

```rust
// adapter.rs:526-529
let key = borsh::to_vec(&StateHeaderKey(shard_id, sync_hash)).unwrap();
store_update.set_ser(DBCol::StateHeaders, &key, &shard_state_header);
```

`set_state_part` then reads this header and uses its `prev_state_root` (which is `R_S'`) for validation:

```rust
// adapter.rs:541-543
let shard_state_header = self.get_state_header(shard_id, sync_hash)?;
let chunk = shard_state_header.take_chunk();
let state_root = *chunk.take_header().take_inner().prev_state_root();
```

`validate_state_part_impl` does not use `shard_id` in its trie validation:

```rust
// runtime/mod.rs:531-551
fn validate_state_part_impl(&self, state_root: &StateRoot, part_id: PartId, part: &StatePart)
    -> StatePartValidationResult {
    match Trie::validate_state_part(state_root, part_id, partial_state) { ... }
}
```

State parts of shard S' (valid against `R_S'`) therefore pass validation, and `apply_state_part` writes them into shard S's `ShardUId` storage.

### Impact Explanation
The node installs the trie of shard S' as the state of shard S. All account balances, contract storage, and access keys for shard S are replaced with those of shard S'. The node then operates on this corrupted state, producing invalid execution results and chunk headers, and may be slashed or permanently diverge from the canonical chain.

### Likelihood Explanation
Any peer participating in state sync can send a crafted `ShardStateSyncResponseHeader`. All required data (chunk for S', its Merkle proof, the `state_root_node` for `R_S'`, and state parts of S') are public information available from any full node. No validator or chunk-producer key is required.

### Recommendation
Add an explicit shard-id guard immediately after extracting the chunk in `set_state_header`:

```rust
// After line 376 in adapter.rs
if chunk.shard_id() != shard_id {
    return Err(Error::Other(format!(
        "set_shard_state failed: chunk shard_id {} != requested shard_id {}",
        chunk.shard_id(), shard_id
    )));
}
```

Additionally, replace the plain `verify_path` call with `verify_path_with_index` (already used in `validate_part` in `shards_manager_actor.rs:1269`) to enforce that the chunk is at the correct shard-index position in the Merkle tree, providing defense-in-depth.

pub fn validate_chunk_proofs(
    chunk: &ShardChunk,
    epoch_manager: &dyn EpochManagerAdapter,
) -> Result<bool, Error> {
    let correct_chunk_hash = chunk.compute_header_hash();

    // 1. Checking chunk.header.hash
    let header_hash = chunk.header_hash();
    if header_hash != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }

    // 2. Checking that chunk body is valid
    // 2a. Checking chunk hash
    if chunk.chunk_hash() != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }
    let height_created = chunk.height_created();
    let outgoing_receipts_root = chunk.prev_outgoing_receipts_root();
    let (transactions, receipts) = (chunk.to_transactions(), chunk.prev_outgoing_receipts());

    // 2b. Checking that chunk transactions are valid
    let (tx_root, _) = merklize(transactions);
    if &tx_root != chunk.tx_root() {
        byzantine_assert!(false);
        return Ok(false);
    }
    // 2c. Checking that chunk receipts are valid
    if height_created == 0 {
        return Ok(receipts.is_empty() && outgoing_receipts_root == &CryptoHash::default());
    } else {
        let shard_layout = {
            let prev_block_hash = chunk.prev_block_hash();
            epoch_manager.get_shard_layout_from_prev_block(&prev_block_hash)?
        };
        let outgoing_receipts_hashes = Chain::build_receipts_hashes(receipts, &shard_layout)?;
        let (receipts_root, _) = merklize(&outgoing_receipts_hashes);
        if &receipts_root != outgoing_receipts_root {
            byzantine_assert!(false);
            return Ok(false);
        }
    }
    Ok(true)
