### Title
EVM `initTransfer` Accepts Unregistered Tokens, Permanently Locking User Funds — (File: `evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.sol`'s `initTransfer` function accepts any ERC20 token address without verifying that the token is registered on the NEAR side. When a user transfers an unregistered token, it is locked in the EVM bridge contract. The NEAR `fin_transfer_callback` then panics with `TokenDecimalsNotFound`, leaving the tokens permanently stuck with no on-chain recovery path.

---

### Finding Description

`initTransfer` performs no check against the `ethToNearToken` / `nearToEthToken` mappings before accepting a token:

```solidity
} else {
    IERC20(tokenAddress).safeTransferFrom(
        msg.sender,
        address(this),
        amount
    );
}
```

Any ERC20 token that is neither a `isBridgeToken` nor a `customMinters` entry is silently accepted and locked in the bridge. The function then emits `InitTransfer` and increments `currentOriginNonce`. [1](#0-0) 

When a relayer submits the resulting proof to NEAR's `fin_transfer`, the callback immediately attempts to resolve the token's decimal configuration:

```rust
let decimals = self
    .token_decimals
    .get(&init_transfer.token)
    .near_expect(BridgeError::TokenDecimalsNotFound);
``` [2](#0-1) 

Because the token was never registered via `deploy_token` / `bind_token` / `add_deployed_tokens`, `token_decimals` has no entry for it. The NEAR transaction panics and is fully reverted — no nonce is consumed on NEAR, but the EVM-side transfer is already final and irreversible.

The `add_token` helper that populates `token_decimals` is only called through privileged admin paths: [3](#0-2) 

The EVM bridge contract exposes no `rescueTokens`, `recoverERC20`, or equivalent admin function. The only recovery path is for an admin to register the token on NEAR after the fact and for a relayer to re-submit the proof — an out-of-band, non-guaranteed operation.

---

### Impact Explanation

A user who calls `initTransfer` with any ERC20 token not yet registered on NEAR will have their tokens locked in the EVM bridge contract with no self-service recovery. If the token is never registered on NEAR (e.g., it is an arbitrary or obscure token), the lock is permanent. This matches the **Critical** impact criterion: *Permanent freezing, irrecoverable lock, or unclaimable settlement of user funds in bridge or vault flows.*

---

### Likelihood Explanation

`initTransfer` is a fully permissionless, public function. Any user can call it with any ERC20 token. Accidental misuse (wrong token address, token not yet deployed on NEAR) is a realistic scenario. No off-chain warning or on-chain guard prevents it. [4](#0-3) 

---

### Recommendation

Add a registration check at the top of `initTransfer` for non-native, non-bridge-token paths:

```solidity
require(
    tokenAddress == address(0) ||
    isBridgeToken[tokenAddress] ||
    customMinters[tokenAddress] != address(0) ||
    bytes(ethToNearToken[tokenAddress]).length > 0,
    "ERR_TOKEN_NOT_REGISTERED"
);
```

This mirrors the pattern already used in `deployToken` (`require(!isBridgeToken[nearToEthToken[metadata.token]], "ERR_TOKEN_EXIST")`) and ensures only tokens with a known NEAR-side mapping can be locked. [5](#0-4) 

---

### Proof of Concept

1. `arbitraryToken` is a standard ERC20 deployed on Ethereum. It has **no** entry in `ethToNearToken` and has never been registered on NEAR.
2. User approves the bridge and calls:
   ```solidity
   OmniBridge.initTransfer(
       address(arbitraryToken),
       1_000_000,   // amount
       0,           // fee
       0,           // nativeFee
       "victim.near",
       ""
   );
   ```
3. `safeTransferFrom(user, bridge, 1_000_000)` succeeds; `currentOriginNonce` increments; `InitTransfer` event is emitted.
4. Relayer picks up the event and calls NEAR `fin_transfer` with the EVM proof.
5. `fin_transfer_callback` executes:
   ```rust
   let decimals = self.token_decimals
       .get(&init_transfer.token)          // OmniAddress::Eth(arbitraryToken)
       .near_expect(BridgeError::TokenDecimalsNotFound);  // PANICS
   ```
6. NEAR transaction reverts. No NEAR-side nonce is consumed.
7. The 1,000,000 `arbitraryToken` units remain locked in the EVM bridge contract indefinitely. The EVM bridge has no function to release them. [6](#0-5) [7](#0-6)
