### Title
`replace_class` Syscall Accepts Undeclared Class Hash, Permanently Freezing Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` function in the StarkNet OS accepts any arbitrary `class_hash` value from a contract's syscall request without verifying that the class has been declared in `contract_class_changes`. A contract can replace its own class with a non-existent class hash, permanently freezing all funds held in that contract.

### Finding Description

In `execute_replace_class`, the OS updates a contract's `StateEntry` with a caller-supplied `class_hash` without any existence check:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-acknowledged TODO at line 898 confirms the check is missing. [1](#0-0) 

After this state update is committed, any subsequent call to the contract reaches `execute_entry_point`, which reads the class hash from state and then looks up the compiled class:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [2](#0-1) 

If `class_hash` was never declared, `dict_read` on `contract_class_changes` returns `0` (the default for an absent key). `find_element` then searches for a compiled class with hash `0`, which does not exist, causing an irrecoverable Cairo assertion failure in the OS prover. The blockifier also rejects any call to the contract (class not found), so no valid transaction can ever interact with the contract again.

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract calls `replace_class` with an undeclared class hash:
1. The contract's `StateEntry.class_hash` is committed to the Patricia Merkle tree with the invalid hash. [3](#0-2) 
2. The blockifier rejects all future calls to the contract (class not found), so the sequencer never includes them in a block.
3. Even if a call were included, the OS proof would fail at `find_element` because no compiled class with the bogus hash exists.
4. All ERC-20 tokens, ETH, or other assets held in the contract's storage are permanently inaccessible.

### Likelihood Explanation

**High.** The `replace_class` syscall is a standard, unprivileged operation available to any deployed contract. A malicious contract can call it with an arbitrary felt value as the class hash in a single transaction. No privileged role, leaked key, or external dependency is required. The missing check is explicitly acknowledged in the source with a TODO comment, confirming it is a known gap.

### Recommendation

Before updating `contract_state_changes`, verify that `class_hash` is present in `contract_class_changes` (i.e., it has been declared in the current or a prior block). The check should mirror the lookup performed in `execute_entry_point`:

```cairo
// Verify the class has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);  // 0 == UNINITIALIZED_CLASS_HASH means undeclared.
```

This is the direct analog of the `require(_exists(tokenId), "Nonexistent token")` pattern recommended in the original report. [4](#0-3) 

### Proof of Concept

1. Deploy contract `Victim` holding 1000 STRK in its storage.
2. `Victim` (or an attacker who controls it) calls the `replace_class` syscall with `class_hash = 0xdeadbeef` (any value never declared on-chain).
3. The OS executes `execute_replace_class`: no existence check is performed; `contract_state_changes` is updated with `class_hash = 0xdeadbeef`. The transaction succeeds and is included in a block. [5](#0-4) 
4. Any subsequent `invoke` targeting `Victim` is rejected by the blockifier (class `0xdeadbeef` not found) and never included in a block.
5. If somehow included, the OS reaches `execute_entry_point`, performs `dict_read(contract_class_changes, 0xdeadbeef)` → returns `0`, then calls `find_element(..., key=0)` → Cairo assertion failure; the block proof is invalid. [6](#0-5) 
6. The 1000 STRK in `Victim`'s storage are permanently frozen with no recovery path.

func get_contract_state_hash{hash_ptr: HashBuiltin*}(
    class_hash: felt, storage_root: felt, nonce: felt
) -> (hash: felt) {
    const CONTRACT_STATE_HASH_VERSION = 0;
    if (class_hash == UNINITIALIZED_CLASS_HASH) {
        if (storage_root == 0) {
            if (nonce == 0) {
                return (hash=0);
            }
        }
    }

    // Set res = H(H(class_hash, storage_root), nonce).
    let (hash_value) = hash2(class_hash, storage_root);
    let (hash_value) = hash2(hash_value, nonce);

    // Return H(hash_value, CONTRACT_STATE_HASH_VERSION). CONTRACT_STATE_HASH_VERSION must be in the
    // outermost hash to guarantee unique "decoding".
    let (hash) = hash2(hash_value, CONTRACT_STATE_HASH_VERSION);
    return (hash=hash);
}
