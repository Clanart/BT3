### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not validate that the caller-supplied `class_hash` corresponds to a previously declared contract class. An unprivileged contract can invoke `replace_class` with an arbitrary, undeclared class hash. The OS accepts and commits this update without any existence check, rendering the contract permanently uncallable and freezing any funds held within it.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads `class_hash` directly from the attacker-controlled syscall request and writes it into the contract's `StateEntry` without verifying that the hash exists in `contract_class_changes`:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The inline `TODO` comment explicitly acknowledges the missing check. No range check, no dictionary lookup against `contract_class_changes`, and no assertion that the hash is non-zero or corresponds to a live class is performed before the state is permanently updated.

This is the direct analog of the external report's root cause: user-supplied input (`class_hash`) is accepted and used in a critical protocol operation without any format or existence validation. [1](#0-0) 

---

### Impact Explanation

**Impact: Critical — Permanent freezing of funds.**

Once a contract's class hash is overwritten with an undeclared value, every subsequent call to that contract will fail at entry-point dispatch: the VM will attempt to look up the class in `contract_class_changes`, find no matching entry, and revert with `ENTRYPOINT_NOT_FOUND`. The state update is already committed and irreversible. Any ERC-20 tokens, ETH, or other assets held in the contract's storage become permanently inaccessible — a textbook permanent freeze of funds.

---

### Likelihood Explanation

The `replace_class` syscall is callable by any contract without privilege. The attack path requires only:

1. Deploying (or controlling) a contract that holds funds, or
2. Triggering `replace_class` through a vulnerability in a third-party contract (e.g., an unchecked external call that forwards calldata).

No trusted role, leaked key, or network-level capability is required. The missing check is confirmed by the codebase's own TODO comment, meaning the OS will accept any felt value as a valid class hash today.

---

### Recommendation

Before committing the class hash update, assert that the new class hash exists in `contract_class_changes`. Concretely, perform a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert the returned compiled class hash is non-zero (i.e., the class has been declared). This is precisely what the existing TODO comment calls for. [2](#0-1) 

---

### Proof of Concept

1. Attacker deploys **Contract A** (e.g., a simple vault) that accumulates user deposits.
2. Attacker (or any caller with `replace_class` access on Contract A) submits a transaction invoking the `replace_class` syscall with `class_hash = 0xdeadbeef` — a felt value that has never been declared on-chain.
3. `execute_replace_class` reads `request.class_hash = 0xdeadbeef`, skips the missing existence check (per the TODO), and calls `dict_update` to write `class_hash=0xdeadbeef` into Contract A's `StateEntry`.
4. The OS commits this state update as part of normal block execution.
5. All subsequent calls to Contract A — including withdrawal functions — attempt to dispatch to class `0xdeadbeef`, find no compiled class, and revert with `ENTRYPOINT_NOT_FOUND`.
6. All deposited funds in Contract A's storage are permanently frozen with no recovery path. [3](#0-2)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    );

        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    return ();
}
