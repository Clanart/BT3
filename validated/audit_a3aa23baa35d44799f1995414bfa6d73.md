### Title
Missing Validation of New Class Hash in `execute_replace_class` Enables Permanent Freezing of Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not validate that the caller-supplied `class_hash` is a declared class before committing the replacement to state. An acknowledged TODO comment in the code confirms this check is missing. This is a direct analog to the royalty bug's missing upper-bound/zero-address validation: a critical numeric/address input is accepted without any bounds or existence check, allowing an attacker to permanently brick a contract and freeze any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the new class hash directly from the syscall request and writes it into `contract_state_changes` without verifying that the hash corresponds to a declared class:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The `class_hash` field is taken verbatim from the syscall request with no check that:
1. `class_hash != 0` (the default/undeployed sentinel value), and
2. `class_hash` exists in `contract_class_changes` (i.e., has been declared).

The TODO comment at line 898 explicitly acknowledges this missing validation. After the state update is committed, any future call to `contract_address` will attempt to look up and execute the (non-existent) class, causing every call to fail permanently.

---

### Impact Explanation

**Impact: Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with an undeclared hash (including `0`), the contract becomes permanently uncallable. Any ERC-20 tokens, ETH, or other assets held in the contract's storage are irretrievably frozen. The replacement cannot be undone because the contract itself can no longer execute (including any recovery function that would call `replace_class` again). This satisfies the "Permanent freezing of funds" impact criterion.

---

### Likelihood Explanation

**Likelihood: Medium.**

The attack requires a malicious contract to be deployed that:
1. Accepts deposits from users (making it hold funds worth attacking), and
2. Contains logic — callable by the attacker — that invokes `replace_class` with `class_hash = 0` or any undeclared hash.

Deploying contracts is permissionless on StarkNet. Any unprivileged user can deploy such a contract. The OS is the last line of defense and should reject the syscall if the target class is undeclared, but it does not. The missing check is confirmed by the in-code TODO.

---

### Recommendation

Before committing the class replacement to `contract_state_changes`, the OS must verify that `request.class_hash` is non-zero and exists as a key in `contract_class_changes` (i.e., has been declared in the current block or a prior block). Concretely, add:

```cairo
assert_not_zero(class_hash);
// Verify class_hash is declared:
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the validation already applied in `execute_declare_transaction`, where `assert_not_zero(compiled_class_hash)` is enforced before updating `contract_class_changes`. [2](#0-1) 

---

### Proof of Concept

1. Attacker deploys **ContractA** (class hash `H_A`) with two entry points:
   - `deposit()`: accepts token transfers, records balances in storage.
   - `brick()`: calls the `replace_class` syscall with `class_hash = 0`.

2. Victims call `deposit()` and transfer funds into **ContractA**.

3. Attacker calls `brick()`. The OS executes `execute_replace_class`:
   - `request.class_hash = 0` is read from the syscall buffer.
   - No validation is performed (the TODO confirms this).
   - `contract_state_changes[ContractA_address].class_hash` is set to `0`.

4. The block is proven. The state transition is committed with `ContractA`'s class hash = `0`.

5. Any subsequent transaction calling **ContractA** (including `withdraw()`) looks up class hash `0`, finds no declared class, and fails.

6. All deposited funds are permanently frozen. [3](#0-2)
