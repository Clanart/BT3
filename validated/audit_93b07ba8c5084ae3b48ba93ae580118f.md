### Title
Missing Declared-Class Validation in `execute_replace_class` Enables Permanent Freezing of Contract Funds — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall in the StarkNet OS accepts any arbitrary felt as a replacement class hash without verifying that the hash corresponds to a previously declared contract class. Any contract — including one holding user funds — can call this syscall with an undeclared class hash, causing the OS to commit an invalid state transition that permanently renders the contract inoperable and freezes all funds it holds.

---

### Finding Description

In `execute_replace_class` (`crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`, lines 878–916), the replacement `class_hash` is read directly from the user-supplied syscall request and written into `contract_state_changes` with no validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment at line 898 explicitly acknowledges the missing invariant: *"Check that there is a declared contract class with the given hash."*

The OS never cross-references `class_hash` against `contract_class_changes` (classes declared in the current block) or the Patricia class tree (classes declared in prior blocks). The state transition is therefore accepted and committed to the Merkle tree unconditionally.

Compare this with the analogous path for `execute_declare_transaction` in `transaction_impls.cairo` (lines 816–819), which enforces `prev_value=0` to prevent double-declaration and asserts `compiled_class_hash != 0` — demonstrating that the codebase does apply class-level invariants elsewhere, but omits them in `replace_class`.

---

### Impact Explanation

Once the OS commits a `StateEntry` whose `class_hash` points to an undeclared class:

1. The Patricia state tree is updated with the invalid leaf.
2. The L1 verifier accepts the proof (the OS proof is valid; the missing check is the vulnerability).
3. Every subsequent call to the affected contract fails at class-resolution time because no bytecode exists for the hash.
4. Because the contract cannot execute *any* entry point — including `replace_class` itself — the state is irreversible.
5. All ERC-20 tokens, ETH, or other assets held by the contract are permanently frozen.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

The syscall is reachable by any unprivileged contract deployer or transaction sender via two concrete paths:

**Path A — Accidental (realistic for DeFi protocols):** A contract performing an upgrade calls `replace_class` with a class hash whose `declare` transaction was not included in the same block (e.g., due to block ordering or a sequencer reorg). The OS accepts the upgrade; the class is absent; the contract is bricked.

**Path B — Intentional rug-pull:** An attacker deploys a contract that accepts user deposits, then calls `replace_class` with an arbitrary undeclared hash (e.g., `1`). The contract becomes permanently inoperable, freezing deposited funds while the attacker has already extracted value through other means (e.g., a prior withdrawal or a separate storage manipulation).

Both paths require only a standard `INVOKE` transaction from an unprivileged sender — no privileged key, no operator access, no external dependency.

---

### Recommendation

Before updating `contract_state_changes`, verify that `class_hash` is a declared class. Concretely:

1. Attempt a read from `contract_class_changes` for `class_hash`; if the entry exists and `new_value != 0`, the class was declared in the current block — allow the replacement.
2. Otherwise, perform a Patricia read against the class commitment tree to confirm the class was declared in a prior block.
3. If neither check passes, write a failure response via `write_failure_response` and return without mutating state.

This mirrors the invariant already enforced in `execute_declare_transaction` (`prev_value=0` + `assert_not_zero(compiled_class_hash)`).

### Proof of Concept

```
1. Attacker deploys ContractA (holds 1000 STRK of user deposits).
2. Attacker submits INVOKE tx calling ContractA's entry point, which internally
   issues the replace_class syscall with class_hash = 0xdeadbeef
   (a felt that has never been declared on-chain).
3. execute_replace_class (syscall_impls.cairo:878-916) reads class_hash=0xdeadbeef
   from the request, fetches the current StateEntry, and calls dict_update with
   new_value = StateEntry{class_hash=0xdeadbeef, ...}.
4. No check against contract_class_changes or the Patricia class tree is performed.
5. The OS proof is generated and verified on L1; the state root now encodes
   ContractA.class_hash = 0xdeadbeef.
6. Any subsequent call to ContractA (transfer, withdraw, replace_class again)
   fails at class resolution — bytecode for 0xdeadbeef does not exist.
7. The 1000 STRK held by ContractA are permanently frozen with no recovery path.
```
