### Title
Stale `feemarketParams` Snapshot in `NewDynamicFeeChecker` Allows Cosmos SDK Transactions to Bypass Current EIP-1559 Base Fee Enforcement - (File: `ante/evm/fee_checker.go`)

### Summary

`NewDynamicFeeChecker` and `MinGasPriceDecorator` for Cosmos SDK (non-EVM) transactions capture a one-time snapshot of `feemarketParams` at ante-handler construction time. Because the base fee and `MinGasPrice` are never re-read from the live store during transaction execution, any increase in the base fee (driven by block gas usage via `BeginBlock`) is invisible to these checks. A user can submit a Cosmos SDK transaction with a fee below the current EIP-1559 base fee and have it accepted and committed.

### Finding Description

`NewDynamicFeeChecker` is constructed in `newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712`:

```go

    ...
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)

The pointer `&feemarketParams` points to a local variable whose value is fixed at construction time. Inside the returned closure, the fee check reads:

```go

``` [2](#0-1)

```go

    ...

    ...

``` [3](#0-2) 

The same stale pointer is passed to `MinGasPriceDecorator`, which also reads `MinGasPrice` from the snapshot:

```go

``` [4](#0-3)

```go

...

``` [5](#0-4) 

The base fee is updated every block in `BeginBlock` via `CalculateBaseFee` → `SetBaseFee`, which writes the new value into the persistent KV store:

```go
// x/feemarket/keeper/abci.go:31-38

    baseFee := k.CalculateBaseFee(ctx)
    ...
    k.SetBaseFee(ctx, baseFee)
``` [6](#0-5) 

The stale snapshot in `NewDynamicFeeChecker` never reflects these updates.

### Impact Explanation

A Cosmos SDK transaction (bank send, governance vote, IBC transfer, authz exec, etc.) that includes `ExtensionOptionDynamicFeeTx` is validated by `NewDynamicFeeChecker`. Because the checker uses the base fee from the construction-time snapshot rather than the current block's base fee, a user can submit a transaction with a `feeCap` that is below the current base fee but above the stale snapshot value. The check at line 86 (`feeCap.LT(baseFeeInt)`) passes, the effective fee is computed using the stale lower base fee, and the transaction is committed. The user pays less than the protocol-required base fee. This is a fee mis-accounting bug that permits invalid transactions to commit with under-priced fees, matching the **High** impact category: "fee market, ante handler bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."

### Likelihood Explanation

The base fee changes every block whenever gas usage deviates from the target. On any active chain, the base fee will drift from its genesis value within minutes. Any user who observes the stale base fee (e.g., by querying the handler's construction-time value) and submits a Cosmos SDK transaction with a fee between the stale and current base fee will bypass the check. No special privileges are required — any unprivileged user submitting a standard Cosmos SDK transaction triggers this path.

### Recommendation

Replace the construction-time snapshot with a live store read inside the closure. The `NewDynamicFeeChecker` should accept the `FeeMarketKeeper` interface and call `GetParams(ctx)` on each invocation, mirroring the pattern used in `newEthAnteHandler`:

```go
func NewDynamicFeeChecker(ethCfg *params.ChainConfig, evmKeeper EvmKeeper, fmKeeper FeeMarketKeeper) authante.TxFeeChecker {
    return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
        evmParams := evmKeeper.GetParams(ctx)
        feemarketParams := fmKeeper.GetParams(ctx)   // fresh read per tx
        baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, &feemarketParams)
        ...
    }
}
```

Similarly, `MinGasPriceDecorator.AnteHandle` should call `mpd.feesKeeper.GetParams(ctx).MinGasPrice` instead of reading from the cached `mpd.feemarketParams`.

### Proof of Concept

1. Chain launches; `newCosmosAnteHandler` is called once with genesis context where `BaseFee = 1_000_000_000` (1 gwei). The snapshot is frozen.
2. Blocks fill above the gas target. `BeginBlock` calls `CalculateBaseFee` → `SetBaseFee`, raising the live store value to `2_000_000_000` (2 gwei) after several blocks.
3. Attacker submits a Cosmos SDK `MsgSend` with `ExtensionOptionDynamicFeeTx{MaxPriorityPrice: 0}` and `fee = 1_500_000_000 * gasLimit` (1.5 gwei × gas, below the current 2 gwei base fee).
4. `NewDynamicFeeChecker` reads `baseFee` from the stale snapshot: `1_000_000_000`. The check `feeCap (1.5 gwei) < baseFeeInt (1 gwei)` is **false** — the transaction passes.
5. `effectivePrice = min(baseFee + tip, feeCap) = min(1 gwei + 0, 1.5 gwei) = 1 gwei`. The user is charged `1 gwei × gas` instead of the required `2 gwei × gas`.
6. The transaction is committed with a fee 50% below the current protocol-required base fee. [7](#0-6) [8](#0-7)

func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
