### Title
Unbounded Gas Substitution in Non-Sierra-Gas-Mode Entry Point Calls Bypasses Transaction Gas Accounting - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/entry_point_utils.cairo`)

---

### Summary

In `select_execute_entry_point_func`, when a Cairo 1 contract is not running in Sierra gas mode (`is_sierra_gas_mode == FALSE`), the OS substitutes the caller's actual remaining gas with `DEFAULT_INITIAL_GAS_COST` (10,000,000,000 units) for the callee, and then restores the caller's gas to its pre-call value after execution. This means the callee's gas consumption is never charged back to the caller, allowing a single transaction to consume resources far exceeding its declared gas bounds.

---

### Finding Description

In `select_execute_entry_point_func`:

```cairo
local caller_remaining_gas = remaining_gas;
local is_sierra_gas_mode;
%{ IsSierraGasMode %}
if (is_sierra_gas_mode != FALSE) {
    tempvar inner_remaining_gas = remaining_gas;
} else {
    // Run with high enough gas to avoid out-of-gas.
    tempvar inner_remaining_gas = DEFAULT_INITIAL_GAS_COST;  // 10,000,000,000
}

let (is_reverted, retdata_size, retdata) = execute_entry_point{
    remaining_gas=inner_remaining_gas
}(block_context=block_context, execution_context=execution_context);

if (is_sierra_gas_mode != FALSE) {
    tempvar remaining_gas = inner_remaining_gas;
} else {
    // Do not count Sierra gas for the caller in this case.
    tempvar remaining_gas = caller_remaining_gas;
}
```

Two compounding problems exist:

1. **Callee receives `DEFAULT_INITIAL_GAS_COST = 10,000,000,000` gas** regardless of the caller's actual remaining gas. This is ~9× the maximum allowed execute gas (`EXECUTE_MAX_SIERRA_GAS = 1,100,000,000`).

2. **The caller's `remaining_gas` is unconditionally restored to `caller_remaining_gas`** after the call returns. The gas consumed by the callee is silently discarded and never deducted from the caller's budget.

This path is reached from `contract_call_helper` in `syscall_impls.cairo`, which is invoked by both `execute_call_contract` and `execute_library_call`. Any `call_contract` or `library_call` syscall targeting a non-Sierra-gas-mode class triggers this path.

The `is_sierra_gas_mode` hint is set by the prover based on the actual contract class. An attacker who deploys a non-Sierra-gas-mode Cairo 1 class forces the prover to set `is_sierra_gas_mode = FALSE` for that class — the prover has no valid alternative, as setting it to `TRUE` would produce an incorrect proof. The attacker therefore indirectly but deterministically controls this branch. [1](#0-0) [2](#0-1) [3](#0-2) 

---

### Impact Explanation

**High — Network not being able to confirm new transactions (total network shutdown).**

A transaction's declared L2 gas bound (from `resource_bounds[L2_GAS_INDEX].max_amount`) is used to set `remaining_gas` at the top of `execute_invoke_function_transaction`. Because the callee's consumption is never subtracted from the caller's budget, the caller can issue an unbounded number of `call_contract` syscalls to expensive non-Sierra-gas-mode contracts within a single transaction. Each callee executes with up to 10 billion gas units. The sequencer must prove all of this computation but collects fees only for the declared gas bound. Sustained exploitation causes sequencers to operate at a severe loss, making block production economically unviable and halting the network. [4](#0-3) 

---

### Likelihood Explanation

**Medium-High.** The non-Sierra-gas-mode path is explicitly a transitional compatibility mode (marked with `TODO(Yoni): SIERRA_GAS_MODE - remove once all Cairo 1 contracts run with Sierra gas mode`). Any attacker can declare and deploy a Cairo 1 contract class compiled without Sierra gas mode. The class declaration is permissionless. Once deployed, the attacker calls it repeatedly via `call_contract` within a single transaction. No privileged access, leaked keys, or external dependencies are required. [5](#0-4) 

---

### Recommendation

1. **Remove the non-Sierra-gas-mode compatibility path entirely**, or enforce that it is only reachable for a hardcoded allowlist of legacy class hashes that cannot be deployed by unprivileged users.

2. If the compatibility path must remain, **always deduct the gas actually consumed by the callee from the caller's budget**, regardless of `is_sierra_gas_mode`. The `inner_remaining_gas` after the call reflects actual consumption and should be used to update `remaining_gas` in all branches.

3. **Cap `inner_remaining_gas` to `remaining_gas`** (never exceed the caller's actual budget) even in the non-Sierra-gas-mode branch, mirroring the intent of `cap_remaining_gas` used at the transaction level.

---

### Proof of Concept

1. Attacker compiles a Cairo 1 contract without Sierra gas mode that performs a maximally expensive loop (e.g., repeated `storage_write` syscalls up to the block step limit).
2. Attacker submits a `declare` transaction for this class (permissionless).
3. Attacker submits an `invoke` transaction with a minimal L2 gas bound (e.g., 1,000,000 units) targeting their own account contract.
4. The account contract's `__execute__` issues 100 `call_contract` syscalls to the declared non-Sierra-gas-mode contract.
5. For each call, the OS sets `inner_remaining_gas = DEFAULT_INITIAL_GAS_COST = 10,000,000,000` and executes the callee. After each call, `remaining_gas` is restored to the pre-call value (≈1,000,000), so the caller never runs out of gas.
6. Total gas consumed by callees: up to `100 × 10^10 = 10^12` units. Total gas declared and paid for: 1,000,000 units.
7. The sequencer proves the block at enormous computational cost while collecting negligible fees, making block production unprofitable and eventually halting the network. [6](#0-5) [7](#0-6)
