### Title
Missing `state_diff_commitment` Verification in P2P Sync State Diff Assembly Allows Malicious Peer to Inject Wrong State — (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

---

### Summary

The P2P sync client's `parse_data_for_block` function assembles a `ThinStateDiff` from peer-supplied chunks by verifying only that the accumulated chunk-length counter equals the `state_diff_length` stored in the block header. It never verifies that `calculate_state_diff_hash(&result)` equals the `state_diff_commitment` that is also present in the same header and committed to in the block hash. A single malicious peer can therefore supply chunks whose total length is correct but whose content is entirely fabricated, causing the node to persist a wrong state diff and subsequently serve wrong storage values, nonces, and class hashes from every RPC endpoint that reads state.

---

### Finding Description

`StateDiffStreamBuilder::parse_data_for_block` in `crates/apollo_p2p_sync/src/client/state_diff.rs` drives the sync loop with a single guard:

```rust
while current_state_diff_len < target_state_diff_len {
    ...
    current_state_diff_len += state_diff_chunk.len();
    unite_state_diffs(&mut result, state_diff_chunk)?;
}
if current_state_diff_len != target_state_diff_len {
    return Err(...WrongStateDiffLength...);
}
validate_deprecated_declared_classes_non_conflicting(&result)?;
Ok(Some((result, block_number)))
``` [1](#0-0) 

`target_state_diff_len` is read from `header.state_diff_length`: [2](#0-1) 

The same header also carries `state_diff_commitment`, which is the Poseidon hash of the structured state diff content, computed by `calculate_state_diff_hash`: [3](#0-2) 

This commitment is part of the block hash via `BlockHeaderCommitments.state_diff_commitment` and `concatenated_counts` (which encodes `state_diff_length`): [4](#0-3) [5](#0-4) 

After the loop exits, `result` is written directly to MDBX storage without any commitment check: [6](#0-5) 

The `state_diff_length` counter (`ThinStateDiff::len`) sums individual storage key-value pairs across all contracts: [7](#0-6) 

A malicious peer can craft chunks whose per-chunk `.len()` values sum to exactly `target_state_diff_len` while substituting arbitrary storage values, class hashes, or nonces. The length check passes; the commitment is never tested; the fabricated diff is committed to MDBX.

---

### Impact Explanation

Once the wrong `ThinStateDiff` is stored, every subsequent read of that block's state is corrupted:

- `get_storage_at`, `get_nonce_at`, `get_class_hash_at` via `StateStorageReader` return attacker-chosen values.
- The Patricia trie root computed by the committer from this diff will diverge from the authentic global root in the header, breaking proof inputs and SNOS consistency.
- Any RPC simulation or fee estimation that re-executes transactions against this state will produce wrong results.

This matches the allowed impact: **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value**, and potentially **Critical — wrong storage value or class hash from execution logic** if the node is also used for sequencing.

---

### Likelihood Explanation

Any single peer in the P2P network that the syncing node connects to can trigger this. No privileged position is required; the attacker only needs to respond to a state diff query with well-formed protobuf chunks whose lengths sum correctly. The header (with the authentic `state_diff_commitment`) is already stored before the state diff query is issued, so the correct commitment is available but simply unused.

---

### Recommendation

After the assembly loop exits and before returning `Ok(Some((result, block_number)))`, read `state_diff_commitment` from the stored header and verify it:

```rust
if let Some(expected_commitment) = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .and_then(|h| h.state_diff_commitment)
{
    let actual_commitment = calculate_state_diff_hash(&result);
    if actual_commitment != expected_commitment {
        return Err(ParseDataError::BadPeer(
            BadPeerError::WrongStateDiffCommitment { ... }
        ));
    }
}
```

This mirrors the check already present in the committer's `commit_block_inner` (guarded by `verify_state_diff_hash`): [8](#0-7) 

---

### Proof of Concept

1. A syncing node has stored a valid header for block N with `state_diff_length = 3` and `state_diff_commitment = H_real`.
2. The attacker peer responds to the state diff query with one `ContractDiff` chunk: address `0xDEAD`, storage key `0x1 → 0xATTACKER_VALUE` (chunk length = 1), and two more chunks totalling length 3.
3. `parse_data_for_block` accumulates `current_state_diff_len = 3 == target_state_diff_len = 3`, passes the length check, skips commitment verification, and calls `write_to_storage`.
4. `append_state_diff(block_number, fabricated_diff)` persists the attacker's storage values.
5. A subsequent `starknet_getStorageAt(0xDEAD, 0x1, block_N)` RPC call returns `0xATTACKER_VALUE` instead of the real value, while the block hash (which commits to `H_real`) remains unchanged — the node's state is silently diverged from the canonical chain. [9](#0-8) [3](#0-2) [10](#0-9)

    #[latency_histogram("p2p_sync_state_diff_parse_data_for_block_latency_seconds", true)]
    fn parse_data_for_block<'a>(
        state_diff_chunks_response_manager: &'a mut ClientResponsesManager<
            DataOrFin<StateDiffChunk>,
        >,
        block_number: BlockNumber,
        storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            let mut result = ThinStateDiff::default();
            let mut prev_result_len = 0;
            let mut current_state_diff_len = 0;
            let target_state_diff_len = storage_reader
                .begin_ro_txn()?
                .get_block_header(block_number)?
                .expect("A header with number lower than the header marker is missing")
                .state_diff_length
                .ok_or(P2pSyncClientError::OldHeaderInStorage {
                    block_number,
                    missing_field: "state_diff_length",
                })?;

            while current_state_diff_len < target_state_diff_len {
                let maybe_state_diff_chunk = state_diff_chunks_response_manager
                    .next()
                    .await
                    .ok_or(ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }))?;
                let Some(state_diff_chunk) = maybe_state_diff_chunk?.0 else {
                    if current_state_diff_len == 0 {
                        return Ok(None);
                    } else {
                        return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                            expected_length: target_state_diff_len,
                            possible_lengths: vec![current_state_diff_len],
                        }));
                    }
                };
                prev_result_len = current_state_diff_len;
                if state_diff_chunk.is_empty() {
                    return Err(ParseDataError::BadPeer(BadPeerError::EmptyStateDiffPart));
                }
                // It's cheaper to calculate the length of `state_diff_part` than the length of
                // `result`.
                current_state_diff_len += state_diff_chunk.len();
                unite_state_diffs(&mut result, state_diff_chunk)?;
            }

            if current_state_diff_len != target_state_diff_len {
                return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                    expected_length: target_state_diff_len,
                    possible_lengths: vec![prev_result_len, current_state_diff_len],
                }));
            }

            validate_deprecated_declared_classes_non_conflicting(&result)?;
            Ok(Some((result, block_number)))
        }
        .boxed()
    }

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
