### Title
Missing Declared Class Hash Validation in `execute_replace_class` Enables Permanent Contract Freezing â€” (File: `execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary class hash from a contract without verifying it corresponds to a previously declared class. A TODO comment in the production code explicitly acknowledges this missing check. An unprivileged contract caller can replace their contract's class with an undeclared hash, permanently bricking the contract and freezing any funds held within it, and causing OS proof generation to fail for any block that subsequently calls the bricked contract.

### Finding Description

In `execute_replace_class`, the OS writes the caller-supplied `class_hash` directly into the contract's state entry with no validation: [1](#0-0) 

The critical gap is at line 898:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment is a production acknowledgment that this check is absent. By contrast, `execute_entry_point` performs a **mandatory, non-optional** lookup of the compiled class for any class hash it encounters: [2](#0-1) 

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
```

If `class_hash` was never declared, `dict_read` on `contract_class_changes` returns `0` (the default for an uninitialized dict entry, as the dict is initialized via `dict_new()` in `initialize_state_changes`). [3](#0-2) 

`find_element` then searches for `compiled_class_hash = 0` in the compiled class facts bundle. Since `0` is not a valid compiled class hash, `find_element` panics, causing the entire OS proof to fail.

This is a direct analog to the external report: just as `SurplusGuildMinter::stake()` accepted a term from a different market without validating it belonged to the correct context, `execute_replace_class` accepts a class hash from an entirely different context (undeclared) without validation, corrupting the contract's state entry.

### Impact Explanation

**Critical â€” Permanent freezing of funds.** After a contract calls `replace_class` with an undeclared hash:

1. The contract's `class_hash` in `contract_state_changes` is set to the undeclared value.
2. Any future call to this contract in any subsequent block causes `execute_entry_point` to look up `compiled_class_hash` for the undeclared hash â†’ gets `0` â†’ `find_element` panics.
3. The OS proof for any block containing a call to this contract cannot be generated.
4. All funds held by the contract are permanently frozen with no recovery path.

**High â€” Network shutdown.** If the bricked contract is a widely-used contract (e.g., a token or DeFi contract), every block that includes any interaction with it will fail to be proven, halting finalization for all users interacting with it.

### Likelihood Explanation

The attack requires only:
1. Deploying a contract (permissionless on StarkNet).
2. Having the contract call the `replace_class` syscall with an arbitrary undeclared felt value.

The `replace_class` syscall is a standard, documented protocol feature accessible to any contract. The TODO comment in the production OS code confirms the check is currently absent. No privileged role, leaked key, or external dependency is required.

### Recommendation

Add a validation in `execute_replace_class` to verify the new `class_hash` is a declared class before writing it to state. The check should confirm the hash exists in `contract_class_changes` with a non-zero compiled class hash:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,  // add this implicit arg
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // Verify the class hash is declared.
    let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
    with_attr error_message("replace_class: undeclared class hash") {
        assert_not_zero(compiled_class_hash);
    }
    ...
}
```

### Proof of Concept

1. Deploy contract `C` holding user funds.
2. `C` calls `replace_class(class_hash=0xDEADBEEF)` where `0xDEADBEEF` is never declared.
3. OS executes `execute_replace_class`: no validation, `C`'s state entry is updated with `class_hash=0xDEADBEEF`.
4. In any subsequent block, a transaction calls `C`.
5. OS executes `execute_entry_point` for `C`:
   - Reads `C`'s class hash: `0xDEADBEEF`
   - `dict_read(contract_class_changes, 0xDEADBEEF)` â†’ returns `0` (never declared)
   - `find_element(..., key=0)` â†’ panics (0 not in compiled class facts)
6. OS proof generation fails for the entire block.
7. Funds in `C` are permanently frozen; no block including a call to `C` can ever be proven.
