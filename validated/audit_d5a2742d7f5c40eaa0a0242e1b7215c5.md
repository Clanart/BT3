### Title
EIP-712 Typed Data Hash Omits `timeout_height`, Allowing Attacker to Strip Transaction Expiry - (File: `ethereum/eip712/types.go`, `ethereum/eip712/eip712_legacy.go`, `ante/cosmos/eip712.go`)

---

### Summary

The EIP-712 `"Tx"` type schema in both the current and legacy Ethermint EIP-712 paths deliberately omits the `timeout_height` field. Because `timeout_height` is not committed to in the EIP-712 hash, an attacker who observes a signed `ExtensionOptionsWeb3Tx` transaction can strip or alter the `timeout_height` without invalidating the user's signature. This bypasses the user's intended transaction expiry, allowing the transaction to be executed at any block height for which the nonce is still valid.

---

### Finding Description

**Root cause — `timeout_height` excluded from the EIP-712 type schema:**

In `ethereum/eip712/types.go`, the `"Tx"` struct used to build the EIP-712 typed-data hash is:

```go
"Tx": {
    {Name: "account_number", Type: "string"},
    {Name: "chain_id",       Type: "string"},
    {Name: "fee",            Type: "Fee"},
    {Name: "memo",           Type: "string"},
    {Name: "sequence",       Type: "string"},
    // Note timeout_height was removed because it was not getting filled with the legacyTx
},
``` [1](#0-0) 

The identical omission exists in the legacy path:

```go
"Tx": {
    ...
    {Name: "sequence", Type: "string"},
    // Note timeout_height was removed because it was not getting filled with the legacyTx
    // {Name: "timeout_height", Type: "string"},
},
``` [2](#0-1) 

**How the ante handler uses this:**

In `ante/cosmos/eip712.go`, `VerifySignature` (called by `LegacyEip712SigVerificationDecorator`) constructs the Amino sign bytes from the *submitted* transaction, including its `timeout_height`:

```go
txBytes := legacytx.StdSignBytes(
    signerData.ChainID,
    signerData.AccountNumber,
    signerData.Sequence,
    tx.GetTimeoutHeight(),   // ← read from the submitted tx
    legacytx.StdFee{...},
    msgs, tx.GetMemo(),
)
``` [3](#0-2) 

These bytes are then passed to `LegacyWrapTxToTypedData`, which builds the EIP-712 typed data using the schema above. Because `timeout_height` is absent from the `"Tx"` type definition, the EIP-712 hash is **identical** regardless of whether `timeout_height` is `0`, `100`, or `10000` in the JSON payload.

**The same gap exists in the non-legacy `verifySignatureAsEIP712` path** (used by `ethsecp256k1.PubKey.VerifySignature`) when the sign doc is Amino-encoded, because `decodeAminoSignDoc` passes the raw sign bytes directly to `WrapTxToTypedData`, which uses the same schema: [4](#0-3) [5](#0-4) 

**The `TxTimeoutHeightDecorator` enforces the field from the submitted tx body, not from the signed hash:**

The legacy EIP-712 ante handler chain includes `authante.NewTxTimeoutHeightDecorator()`: [6](#0-5) 

This decorator reads `timeout_height` from the transaction body. If an attacker changes it to `0`, the decorator sees "no timeout" and allows the transaction through at any block height.

---

### Impact Explanation

A user who signs an `ExtensionOptionsWeb3Tx` (EIP-712 Cosmos tx) with `timeout_height = N` intends the transaction to be rejected if it is not included before block `N`. Because `timeout_height` is not part of the EIP-712 signed hash, an attacker who observes the signed transaction (e.g., from the mempool or a broadcast) can:

1. Change `timeout_height` from `N` to `0` (no expiry).
2. Rebroadcast the modified transaction.
3. The `LegacyEip712SigVerificationDecorator` recomputes the EIP-712 hash over the modified bytes — but since `timeout_height` is not in the type schema, the hash is identical to the one the user signed. Signature verification passes.
4. The `TxTimeoutHeightDecorator` sees `timeout_height = 0` and imposes no deadline.
5. The transaction executes at an arbitrary future block height the user explicitly tried to prevent.

Concrete harm: a user who signs a large token transfer or governance vote with a short `timeout_height` to limit execution risk (e.g., to prevent execution after a price move) can have that protection stripped, causing their funds to be transferred or their vote cast at an unintended time.

This matches the allowed High impact: **EIP-712 authorization bypass enabling forged execution**.

---

### Likelihood Explanation

- The attack requires only that the attacker observe the signed transaction before it is included in a block (e.g., from the public mempool or a broadcast endpoint). No privileged access is needed.
- The modification is trivial: change one integer field in the Cosmos tx body.
- The `ExtensionOptionsWeb3Tx` / legacy EIP-712 path is the standard path for Metamask/web3 wallet users interacting with Cosmos modules on Ethermint chains, so the affected transaction type is common.
- The vulnerability is structural and deterministic — it does not depend on timing, race conditions, or probabilistic factors.

---

### Recommendation

Include `timeout_height` in the EIP-712 `"Tx"` type schema in both `ethereum/eip712/types.go` and `ethereum/eip712/eip712_legacy.go`:

```go
"Tx": {
    {Name: "account_number",  Type: "string"},
    {Name: "chain_id",        Type: "string"},
    {Name: "fee",             Type: "Fee"},
    {Name: "memo",            Type: "string"},
    {Name: "sequence",        Type: "string"},
    {Name: "timeout_height",  Type: "string"},  // ← add this
},
```

The `StdSignBytes` function already serializes `timeout_height` as `"0"` when it is zero, so adding the field to the schema is backward-compatible for transactions that do not set a timeout (they will hash `"0"` consistently). For the Protobuf path, `decodeProtobufSignDoc` already rejects `TimeoutHeight != 0`, so no change is needed there; the fix is only required in the Amino-based paths.

---

### Proof of Concept

1. Alice signs a `MsgSend` wrapped in an `ExtensionOptionsWeb3Tx` with `timeout_height = 50` (current block is 45). She broadcasts it.

2. Bob observes the signed transaction in the mempool. He decodes the Cosmos tx, sets `timeout_height = 0` in the `TxBody`, and rebroadcasts.

3. The chain processes Bob's modified transaction:
   - `TxTimeoutHeightDecorator`: `timeout_height = 0` → no deadline, passes.
   - `LegacyEip712SigVerificationDecorator` → `VerifySignature`:
     - Calls `legacytx.StdSignBytes(..., 0 /*timeout_height*/, ...)` → JSON includes `"timeout_height":"0"`.
     - Calls `LegacyWrapTxToTypedData` → builds EIP-712 typed data with the schema that omits `timeout_height`.
     - EIP-712 hash is identical to the one Alice signed (because `timeout_height` is not in the type schema).
     - `ethcrypto.VerifySignature(pubKey, sigHash, sig)` → **passes**.
   - Transaction executes at block 200, long after Alice's intended expiry. [7](#0-6) [1](#0-0) [2](#0-1) [6](#0-5)

	return pubKey.verifySignatureECDSA(msg, sig) || pubKey.verifySignatureAsEIP712(msg, sig)

// Verifies the signature as an EIP-712 signature by first converting the message payload
// to EIP-712 object bytes, then performing ECDSA verification on the hash. This is to support
// signing a Cosmos payload using EIP-712.
func (pubKey PubKey) verifySignatureAsEIP712(msg, sig []byte) bool {
	eip712Bytes, err := eip712.GetEIP712BytesForMsg(msg)
	if err != nil {
		return false
	}

	if pubKey.verifySignatureECDSA(eip712Bytes, sig) {
		return true
	}

	// Try verifying the signature using the legacy EIP-712 encoding
	legacyEIP712Bytes, err := eip712.LegacyGetEIP712BytesForMsg(msg)
	if err != nil {
		return false
	}

	return pubKey.verifySignatureECDSA(legacyEIP712Bytes, sig)
}

```go
// decodeAminoSignDoc attempts to decode the provided sign doc (bytes) as an Amino payload
// and returns a signable EIP-712 TypedData object.
func decodeAminoSignDoc(signDocBytes []byte) (apitypes.TypedData, error) {
	// Ensure codecs have been initialized
	if err := validateCodecInit(); err != nil {
		return apitypes.TypedData{}, err
	}

	var aminoDoc legacytx.StdSignDoc
	if err := aminoCodec.UnmarshalJSON(signDocBytes, &aminoDoc); err != nil {
		return apitypes.TypedData{}, err
	}

	var fees legacytx.StdFee
	if err := aminoCodec.UnmarshalJSON(aminoDoc.Fee, &fees); err != nil {
		return apitypes.TypedData{}, err
	}

	// Validate payload messages
	msgs := make([]sdk.Msg, len(aminoDoc.Msgs))
	for i, jsonMsg := range aminoDoc.Msgs {
		var m sdk.Msg
		if err := aminoCodec.UnmarshalJSON(jsonMsg, &m); err != nil {
			return apitypes.TypedData{}, fmt.Errorf("failed to unmarshal sign doc message: %w", err)
		}
		msgs[i] = m
	}

	if err := validatePayloadMessages(msgs); err != nil {
		return apitypes.TypedData{}, err
	}

	chainID, err := types.ParseChainID(aminoDoc.ChainID)
	if err != nil {
		return apitypes.TypedData{}, errors.New("invalid chain ID passed as argument")
	}

	typedData, err := WrapTxToTypedData(
		chainID.Uint64(),
		signDocBytes,
	)
	if err != nil {
		return apitypes.TypedData{}, fmt.Errorf("could not convert to EIP712 representation: %w", err)
	}

	return typedData, nil
