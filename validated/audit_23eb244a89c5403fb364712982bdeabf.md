### Title
Missing `state_diff_commitment` binding in P2P state diff sync allows a malicious peer to corrupt local state storage — (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

### Summary
`StateDiffStreamBuilder::parse_data_for_block` validates only that the assembled `ThinStateDiff` has the correct *length* (`state_diff_length` from the stored header). It never verifies `calculate_state_diff_hash(assembled_diff) == header.state_diff_commitment`. A malicious P2P peer can send chunks that sum to the correct length but carry attacker-chosen storage keys, nonces, class hashes, or compiled-class-hash mappings. The assembled diff is committed to storage verbatim, permanently corrupting the per-contract tables that back every state-reader query.

### Finding Description
**Entrypoint**: `StateDiffStreamBuilder::parse_data_for_block` [6](#0-5) 

**Attacker-controlled data path**:
1. Syncing node has a header for block N stored with `state_diff_commitment = C` and `state_diff_length = L`.
2. Malicious peer sends `L` length-units of `StateDiffChunk` messages. Each chunk passes the non-empty check and the no-duplicate-key check (`ConflictingStateDiffParts`), but the storage values, nonces, or class hashes differ from the real block.
3. `unite_state_diffs` merges them into `result: ThinStateDiff`. `calculate_state_diff_hash(&result)` would yield `C' ≠ C`, but this call is never made.
4. `Ok(Some((result, block_number)))` is returned.
5. `write_to_storage` calls `append_state_diff(block_number, result)`, which writes the attacker's values into `storage_table`, `nonces_table`, `deployed_contracts_table`, `compiled_class_hash_table`, and the `state_diffs` mmap file. [7](#0-6) [8](#0-7) 

**No downstream guard**: the committer's `verify_state_diff_hash` flag is only exercised in the sequencer's own proposal-commit flow, not in the P2P sync client.

### Impact Explanation
After `append_state_diff` commits, every state-reader query for block N returns the attacker's values:
- `get_storage_at(contract, key)` → attacker's value
- `get_nonce_at(contract)` → attacker's nonce
- `get_class_hash_at(contract)` → attacker's class hash
- `get_compiled_class_hash(class_hash)` → attacker's compiled class hash

These values feed RPC responses (`starknet_getStorageAt`, `starknet_getNonce`, `starknet_getClassHashAt`), fee estimation (`starknet_estimateFee`), transaction simulation, and any execution that reads state at or after block N. The corruption is permanent until the block is reverted.

This matches: **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

### Likelihood Explanation
Any peer the syncing node connects to can exploit this. No special privilege is required — the peer only needs to respond to a `StateDiffQuery` with well-formed chunks that sum to the correct `state_diff_length`. The attack is silent: no error is returned, no report is filed, and the state marker advances normally.

### Recommendation
After the accumulation loop completes (line 106, before `Ok(Some(...))`), read `header.state_diff_commitment` and verify:


This mirrors the guard already present in `apollo_committer::commit_block_inner`. [5](#0-4)

```rust
    fn parse_data_for_block<'a>(
        state_diff_chunks_response_manager: &'a mut ClientResponsesManager<
            DataOrFin<StateDiffChunk>,
        >,
        block_number: BlockNumber,
        storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            let mut result = ThinStateDiff::default();
            let mut prev_result_len = 0;
            let mut current_state_diff_len = 0;
            let target_state_diff_len = storage_reader
                .begin_ro_txn()?
                .get_block_header(block_number)?
                .expect("A header with number lower than the header marker is missing")
                .state_diff_length
                .ok_or(P2pSyncClientError::OldHeaderInStorage {
                    block_number,
                    missing_field: "state_diff_length",
                })?;

            while current_state_diff_len < target_state_diff_len {
                let maybe_state_diff_chunk = state_diff_chunks_response_manager
                    .next()
                    .await
                    .ok_or(ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }))?;
                let Some(state_diff_chunk) = maybe_state_diff_chunk?.0 else {
                    if current_state_diff_len == 0 {
                        return Ok(None);
                    } else {
                        return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                            expected_length: target_state_diff_len,
                            possible_lengths: vec![current_state_diff_len],
                        }));
                    }
                };
                prev_result_len = current_state_diff_len;
                if state_diff_chunk.is_empty() {
                    return Err(ParseDataError::BadPeer(BadPeerError::EmptyStateDiffPart));
                }
                // It's cheaper to calculate the length of `state_diff_part` than the length of
                // `result`.
                current_state_diff_len += state_diff_chunk.len();
                unite_state_diffs(&mut result, state_diff_chunk)?;
            }

            if current_state_diff_len != target_state_diff_len {
                return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                    expected_length: target_state_diff_len,
                    possible_lengths: vec![prev_result_len, current_state_diff_len],
                }));
            }

            validate_deprecated_declared_classes_non_conflicting(&result)?;
            Ok(Some((result, block_number)))
        }
        .boxed()
    }
```

```rust
    #[sequencer_latency_histogram(STORAGE_APPEND_THIN_STATE_DIFF_LATENCY, false)]
    fn append_state_diff(
        self,
        block_number: BlockNumber,
        thin_state_diff: ThinStateDiff,
    ) -> StorageResult<Self> {

        let markers_table = self.open_table(&self.tables.markers)?;
        let state_diffs_table = self.open_table(&self.tables.state_diffs)?;
        let nonces_table = self.open_table(&self.tables.nonces)?;
        let deployed_contracts_table = self.open_table(&self.tables.deployed_contracts)?;
        let storage_table = self.open_table(&self.tables.contract_storage)?;
        let declared_classes_block_table = self.open_table(&self.tables.declared_classes_block)?;
        let deprecated_declared_classes_block_table =
            self.open_table(&self.tables.deprecated_declared_classes_block)?;
        let compiled_class_hash_table = self.open_table(&self.tables.compiled_class_hash)?;

        // Write state.

            block_number,
            &storage_table,

            &self.file_handlers,
```
