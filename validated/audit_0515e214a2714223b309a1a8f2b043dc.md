### Title
EIP-712 Typed Data Schema Omits `timeout_height`, Enabling Signature Replay After User-Intended Expiry — (`ethereum/eip712/types.go`, `ethereum/eip712/eip712_legacy.go`, `ante/cosmos/eip712.go`)

### Summary

The EIP-712 typed data schema for the `Tx` type deliberately omits the `timeout_height` field in both the current and legacy EIP-712 encoding paths. However, the signature verification logic in `ante/cosmos/eip712.go` builds the sign bytes with the actual `timeout_height` value from the transaction. Because `timeout_height` is absent from the EIP-712 type schema, `apitypes.TypedDataAndHash` silently ignores it when computing the hash. An attacker who obtains a broadcast-but-unconfirmed EIP-712-signed Cosmos transaction can strip or alter its `timeout_height` without invalidating the signature, replaying the transaction after the user-intended expiry block.

### Finding Description

**Root cause — schema omission:**

In `ethereum/eip712/types.go`, the `Tx` type definition used by the current EIP-712 path explicitly excludes `timeout_height`:

```go
"Tx": {
    {Name: "account_number", Type: "string"},
    {Name: "chain_id",       Type: "string"},
    {Name: "fee",            Type: "Fee"},
    {Name: "memo",           Type: "string"},
    {Name: "sequence",       Type: "string"},
    // Note timeout_height was removed because it was not getting filled with the legacyTx
},
``` [1](#0-0) 

The same omission exists in the legacy schema:

```go
"Tx": {
    ...
    {Name: "sequence", Type: "string"},
    // Note timeout_height was removed because it was not getting filled with the legacyTx
    // {Name: "timeout_height", Type: "string"},
},
``` [2](#0-1) 

**Root cause — sign bytes include `timeout_height`:**

In `ante/cosmos/eip712.go`, `VerifySignature` builds `txBytes` via `legacytx.StdSignBytes`, which **does** include `timeout_height` in the JSON payload:

```go
txBytes := legacytx.StdSignBytes(
    signerData.ChainID,
    signerData.AccountNumber,
    signerData.Sequence,
    tx.GetTimeoutHeight(),   // ← included in JSON
    legacytx.StdFee{Amount: tx.GetFee(), Gas: tx.GetGas()},
    msgs, tx.GetMemo(),
)
``` [3](#0-2) 

These `txBytes` are then passed to `LegacyWrapTxToTypedData`, which unmarshals them into `txData` (a `map[string]interface{}`). The `apitypes.TypedDataAndHash` call hashes only the fields declared in the type schema. Because `timeout_height` is not in the schema, it is silently dropped from the hash computation regardless of its value. [4](#0-3) 

The same structural issue exists in the non-legacy path via `decodeAminoSignDoc` → `WrapTxToTypedData`, where no guard rejects a non-zero `timeout_height` in the Amino sign doc before hashing. [5](#0-4) 

(The Protobuf decode path does guard against `TimeoutHeight != 0`, but the Amino path does not.) [6](#0-5) 

### Impact Explanation

A user who signs a legacy EIP-712 Cosmos transaction with `timeout_height = N` intends the transaction to be invalid after block `N`. Because the EIP-712 hash does not commit to `timeout_height`, an attacker who observes the signed transaction in the mempool can:

1. Copy the transaction and its `FeePayerSig`.
2. Set `timeout_height = 0` (no expiry) or any future block.
3. Rebroadcast. The `LegacyEip712SigVerificationDecorator` recomputes the EIP-712 hash over the modified `txBytes`; since `timeout_height` is absent from the schema, the hash is identical to the original, and the signature verifies successfully.
4. The Cosmos SDK `TxTimeoutHeightDecorator` passes because `timeout_height = 0` means "no expiry."

The result is unauthorized execution of a transaction after the signer's intended deadline — enabling replay of fund transfers, governance votes, staking operations, or any other Cosmos message type supported by the EIP-712 path.

### Likelihood Explanation

The legacy EIP-712 path (`ExtensionOptionsWeb3Tx`) is the standard mechanism for MetaMask and other Ethereum wallets to sign Cosmos transactions on Ethermint-based chains. Any user who sets `timeout_height` to limit transaction validity is exposed. The attacker only needs to observe the mempool (public) and submit a modified transaction before the original is confirmed.

### Recommendation

1. **Add `timeout_height` to the EIP-712 `Tx` type schema** in both `ethereum/eip712/types.go` and `ethereum/eip712/eip712_legacy.go`, so the field is included in the hash computation.
2. **Alternatively, reject transactions with non-zero `timeout_height`** in the Amino decode path (`decodeAminoSignDoc`) the same way the Protobuf path already does, preventing the field from being silently ignored.
3. Ensure the `LegacyEip712SigVerificationDecorator` explicitly validates that the `timeout_height` in the reconstructed `txBytes` matches the `timeout_height` in the submitted transaction.

### Proof of Concept

1. User signs a legacy EIP-712 `MsgSend` with `timeout_height = 100` and broadcasts it.
2. The transaction is not included before block 100.
3. Attacker intercepts the transaction from the mempool, copies `FeePayerSig`, and sets `timeout_height = 0`.
4. Attacker broadcasts the modified transaction after block 100.
5. `LegacyEip712SigVerificationDecorator.AnteHandle` calls `VerifySignature`, which builds `txBytes` with `timeout_height = 0`, calls `LegacyWrapTxToTypedData`, and computes the EIP-712 hash. Since `timeout_height` is absent from the `Tx` schema, the hash equals the original. `ethcrypto.VerifySignature` returns `true`. [7](#0-6) 
6. `TxTimeoutHeightDecorator` passes (no expiry). The transaction executes, transferring funds after the user-intended deadline.

```go
func decodeAminoSignDoc(signDocBytes []byte) (apitypes.TypedData, error) {
	// Ensure codecs have been initialized
	if err := validateCodecInit(); err != nil {
		return apitypes.TypedData{}, err
	}

	var aminoDoc legacytx.StdSignDoc
	if err := aminoCodec.UnmarshalJSON(signDocBytes, &aminoDoc); err != nil {
		return apitypes.TypedData{}, err
	}

	var fees legacytx.StdFee
	if err := aminoCodec.UnmarshalJSON(aminoDoc.Fee, &fees); err != nil {
		return apitypes.TypedData{}, err
	}

	// Validate payload messages
	msgs := make([]sdk.Msg, len(aminoDoc.Msgs))
	for i, jsonMsg := range aminoDoc.Msgs {
		var m sdk.Msg
		if err := aminoCodec.UnmarshalJSON(jsonMsg, &m); err != nil {
			return apitypes.TypedData{}, fmt.Errorf("failed to unmarshal sign doc message: %w", err)
		}
		msgs[i] = m
	}

	if err := validatePayloadMessages(msgs); err != nil {
		return apitypes.TypedData{}, err
	}

	chainID, err := types.ParseChainID(aminoDoc.ChainID)
	if err != nil {
		return apitypes.TypedData{}, errors.New("invalid chain ID passed as argument")
	}

	typedData, err := WrapTxToTypedData(
		chainID.Uint64(),
		signDocBytes,
	)
	if err != nil {
		return apitypes.TypedData{}, fmt.Errorf("could not convert to EIP712 representation: %w", err)
	}

	return typedData, nil
```
