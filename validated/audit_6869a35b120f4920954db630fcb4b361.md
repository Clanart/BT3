### Title
Stale `feemarketParams` Snapshot in `newCosmosAnteHandler` Allows Cosmos Txs to Bypass Current BaseFee Validation тАФ (`evmd/ante/handler_options.go`, `ante/evm/fee_checker.go`)

### Summary

`newCosmosAnteHandler` captures `feemarketParams` once at construction time and passes a pointer to that snapshot into `NewDynamicFeeChecker`. Because `BeginBlock` updates the baseFee in the KV store each block but never mutates the captured struct, every Cosmos (non-EVM) tx processed through this ante handler validates fees against the baseFee that was current when the ante handler was first constructed тАФ not the current block's baseFee.

### Finding Description

In `evmd/ante/handler_options.go`, `newCosmosAnteHandler` fetches params from the keeper once:

```go
feemarketParams := options.FeeMarketKeeper.GetParams(ctx)   // snapshot
txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
``` [1](#0-0) 

The local variable `feemarketParams` is captured by value; `&feemarketParams` is a pointer to that local. `NewDynamicFeeChecker` closes over this pointer:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
``` [2](#0-1) 

`GetBaseFee` reads directly from the captured struct, never from the live KV store:

```go
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    ...
    baseFee := feemarketParams.GetBaseFee()
``` [3](#0-2) 

Meanwhile, `BeginBlock` updates the baseFee in the KV store every block:

```go
func (k *Keeper) BeginBlock(ctx sdk.Context) error {
    baseFee := k.CalculateBaseFee(ctx)
    ...
    k.SetBaseFee(ctx, baseFee)
``` [4](#0-3) 

`SetBaseFee` writes to the KV store via `SetParams`, but the struct pointed to by the ante handler's captured pointer is never touched. The snapshot is permanently frozen at construction time. [5](#0-4) 

Contrast this with `newEthAnteHandler`, which correctly fetches a fresh `blockCfg` (including `FeeMarketParams`) on **every invocation**:

```go
blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
...
feemarketParams := &blockCfg.FeeMarketParams
``` [6](#0-5) 

The same stale-snapshot pattern also affects `newLegacyCosmosAnteHandlerEip712`: [7](#0-6) 

### Impact Explanation

An unprivileged Cosmos tx submitter can craft a tx whose `feeCap = fee/gas` satisfies the **stale** (lower) baseFee but not the **current** (higher) baseFee. The `NewDynamicFeeChecker` closure computes `effectiveFee` using the stale baseFee, the tx passes `DeductFeeDecorator`, and the user is charged less than `currentBaseFee ├Ч gas`. This is fee mis-accounting: the chain accepts and executes txs that should have been rejected, and the fee revenue is under-collected relative to the EIP-1559 invariant.

This matches the allowed High impact: *"fee market, ante handlerтАж bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

### Likelihood Explanation

- The baseFee rises whenever blocks are above the gas target. On a busy chain the stale snapshot diverges from the live baseFee within minutes of startup.
- The attacker needs only to know the genesis/startup baseFee (public on-chain data) and submit a Cosmos tx with `fee = staleFee ├Ч gas`.
- No special privileges, keys, or validator cooperation are required.

### Recommendation

Remove the snapshot pattern entirely. `NewDynamicFeeChecker` should accept the `FeeMarketKeeper` interface and call `keeper.GetBaseFee(ctx)` inside the closure on every invocation, mirroring the pattern already used in `newEthAnteHandler`. The same fix applies to `NewMinGasPriceDecorator`, which also captures `&feemarketParams` from the same stale snapshot.

### Proof of Concept

1. At app startup, baseFee = 100 (stored in snapshot).
2. Several blocks pass; `BeginBlock` raises baseFee to 200 in the KV store.
3. Attacker submits a Cosmos tx with `fee = 100 ├Ч gas`.
4. `NewDynamicFeeChecker` reads `feemarketParams.BaseFee = 100` (snapshot), computes `feeCap = 100 >= 100` тЖТ passes.
5. `effectiveFee = 100 ├Ч gas` is deducted; the tx executes.
6. The correct check would have required `fee = 200 ├Ч gas`; the attacker paid half.

Unit-test proof: construct `NewDynamicFeeChecker` with `feemarketParams.BaseFee = 100`, then invoke the returned checker with a `ctx` whose KV store has baseFee = 200 and a tx with `fee = 100 ├Ч gas`; assert the checker returns `effectiveFee = 100 ├Ч gas` and no error тАФ confirming the stale snapshot is used.

func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
	if !IsLondon(ethCfg, height) {
		return nil
	}
	baseFee := feemarketParams.GetBaseFee()
	// should not be nil if london hardfork enabled
	if baseFee == nil {
		return new(big.Int)
	}
	return baseFee
}
