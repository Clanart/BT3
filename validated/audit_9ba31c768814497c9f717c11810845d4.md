### Title
`compute_meta_tx_v0_hash` Omits Nonce, Enabling Unbounded Signature Replay — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/transaction_hash/transaction_hash.cairo`)

---

### Summary

`compute_meta_tx_v0_hash` constructs the hash that account contracts validate when a `meta_tx_v0` syscall is executed. The hash does not include any replay-prevention field (nonce, block number, or timestamp). Combined with the OS hardcoding `nonce=0` in the resulting `TxInfo`, any captured `meta_tx_v0` signature is permanently valid and can be replayed by an unprivileged attacker in any future transaction, leading to direct loss of funds.

---

### Finding Description

`compute_meta_tx_v0_hash` delegates to `deprecated_get_transaction_hash` with `additional_data_size=0` and `additional_data=cast(0, felt*)`: [1](#0-0) 

The resulting hash covers only `(INVOKE_HASH_PREFIX, version=0, contract_address, entry_point_selector, calldata, max_fee=0, chain_id)`. No nonce, block number, or any other replay-prevention field is included. [2](#0-1) 

When `execute_meta_tx_v0` builds the `TxInfo` presented to the account contract's `__execute__` entry point, it hardcodes `nonce=0`: [3](#0-2) 

Unlike regular account transactions, there is no `check_and_increment_nonce` call anywhere in `execute_meta_tx_v0`. The account contract therefore sees `nonce=0` and a hash that is identical for every replay of the same `(contract_address, selector, calldata, chain_id)` tuple. [4](#0-3) 

This is the direct analog to the MerkleDistributor bug: just as the token address was absent from the Merkle leaf hash (allowing a proof for one window to be used against another), the nonce is absent from the meta-transaction hash, allowing a signature for one invocation to be used in any future invocation.

---

### Impact Explanation

**Critical — Direct loss of funds.**

A `meta_tx_v0` is designed to let an account contract authorize an `__execute__` call on behalf of a user. If the user signs a `meta_tx_v0` to authorize a token transfer (or any state-changing call), an attacker who observes the on-chain signature can replay it indefinitely:

1. Attacker deploys a contract that issues a `meta_tx_v0` syscall.
2. Attacker submits a transaction calling that contract, supplying the victim's previously observed signature and the original calldata.
3. The OS computes the same hash (no nonce differentiates it), the account contract validates the signature as correct, and the transfer executes again.
4. Funds are drained until the victim's balance is zero.

---

### Likelihood Explanation

**High.** Every `meta_tx_v0` execution is recorded on-chain with its full calldata and signature. Any observer can extract these and replay them. No privileged access, leaked key, or social engineering is required — only the ability to submit a standard transaction.

---

### Recommendation

Include a nonce (or equivalent replay-prevention field) in `compute_meta_tx_v0_hash`. The nonce should be:

1. Passed as a parameter to `compute_meta_tx_v0_hash` and added to `additional_data`.
2. Stored and incremented per `(caller_contract, target_contract)` pair in contract storage, or sourced from the target account's on-chain nonce.
3. Reflected in the `TxInfo.nonce` field presented to the account contract so that the account's own validation logic can enforce uniqueness.

This mirrors the fix applied to the MerkleDistributor: include the distinguishing field (nonce here, token address there) inside the committed hash.

---

### Proof of Concept

**Setup:**
- Alice's account contract at address `A` supports `meta_tx_v0`.
- Alice signs a `meta_tx_v0` authorizing `transfer(Bob, 100 STRK)` via `__execute__`.
- The signed hash = `H(INVOKE_PREFIX, 0, A, __execute__, calldata, 0, chain_id)`.

**Step 1 — Legitimate execution:**
A parent transaction invokes `meta_tx_v0` with Alice's signature. The OS calls `compute_meta_tx_v0_hash`, gets hash `H`, presents `TxInfo(nonce=0, transaction_hash=H)` to Alice's `__execute__`. Alice's contract validates the signature → transfer executes.

**Step 2 — Replay by attacker:**
Attacker deploys `MaliciousContract` with a function that calls `meta_tx_v0(contract_address=A, selector=__execute__, calldata=<same>, signature=<Alice's captured signature>)`.

Attacker submits a transaction calling `MaliciousContract`. The OS again calls `compute_meta_tx_v0_hash` with the same inputs and produces the identical hash `H` (no nonce differentiates it). Alice's `__execute__` receives `TxInfo(nonce=0, transaction_hash=H)`, validates the signature as correct, and executes the transfer again.

**Result:** Alice loses another 100 STRK. This can be repeated until Alice's balance is zero, with no action required from Alice. [5](#0-4) [6](#0-5)

// Computes the hash of a v0 meta transaction. See the `meta_tx_v0` syscall.
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(
    contract_address: felt,
    entry_point_selector: felt,
    calldata: felt*,
    calldata_size: felt,
    chain_id: felt,
) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        tx_hash_prefix=INVOKE_HASH_PREFIX,
        version=0,
        contract_address=contract_address,
        entry_point_selector=entry_point_selector,
        calldata_size=calldata_size,
        calldata=calldata,
        max_fee=0,
        chain_id=chain_id,
        additional_data_size=0,
        additional_data=cast(0, felt*),
    );
    return tx_hash;
}

func execute_meta_tx_v0{
    range_check_ptr,
    syscall_ptr: felt*,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    revert_log: RevertLogEntry*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, caller_execution_context: ExecutionContext*) {
    alloc_locals;

    let request = cast(syscall_ptr + RequestHeader.SIZE, MetaTxV0Request*);
    local calldata_start: felt* = request.calldata_start;
    local calldata_size = request.calldata_end - calldata_start;

    let specific_base_gas_cost = (
        META_TX_V0_GAS_COST + META_TX_V0_CALLDATA_FACTOR_GAS_COST * calldata_size
    );
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=specific_base_gas_cost, request_struct_size=MetaTxV0Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    local contract_address = request.contract_address;
    local selector = request.selector;
    local caller_execution_info: ExecutionInfo* = caller_execution_context.execution_info;
    local old_tx_info: TxInfo* = caller_execution_info.tx_info;

    if (selector != EXECUTE_ENTRY_POINT_SELECTOR) {
        write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
        return ();
    }

    // Sanity check: Verify that `signature` is a valid Sierra array.
    assert_nn_le(request.signature_end - request.signature_start, SIERRA_ARRAY_LEN_BOUND - 1);

    let (state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
        key=contract_address
    );

    // Compute the meta-transaction hash.
    let pedersen_ptr = builtin_ptrs.selectable.pedersen;
    with pedersen_ptr {
        let meta_tx_hash = compute_meta_tx_v0_hash(
            contract_address=contract_address,
            entry_point_selector=selector,
            calldata=calldata_start,
            calldata_size=calldata_size,
            chain_id=old_tx_info.chain_id,
        );
    }
    update_pedersen_in_builtin_ptrs(pedersen_ptr=pedersen_ptr);

    // Prepare execution context.
    tempvar new_tx_info = new TxInfo(
        version=0,
        account_contract_address=contract_address,
        max_fee=0,
        signature_start=request.signature_start,
        signature_end=request.signature_end,
        transaction_hash=meta_tx_hash,
        chain_id=old_tx_info.chain_id,
        nonce=0,
        resource_bounds_start=cast(0, ResourceBounds*),
        resource_bounds_end=cast(0, ResourceBounds*),
        tip=0,
        paymaster_data_start=cast(0, felt*),
        paymaster_data_end=cast(0, felt*),
        nonce_data_availability_mode=0,
        fee_data_availability_mode=0,
        account_deployment_data_start=cast(0, felt*),
        account_deployment_data_end=cast(0, felt*),
        proof_facts_start=cast(0, felt*),
        proof_facts_end=cast(0, felt*),
    );

    let (deprecated_tx_info_ptr: DeprecatedTxInfo*) = alloc();
    tempvar execution_context: ExecutionContext* = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=state_entry.class_hash,
        calldata_size=calldata_size,
        calldata=calldata_start,
        execution_info=new ExecutionInfo(
            block_info=caller_execution_info.block_info,
            tx_info=new_tx_info,
            caller_address=ORIGIN_ADDRESS,
            contract_address=contract_address,
            selector=selector,
        ),
        deprecated_tx_info=deprecated_tx_info_ptr,
    );
    fill_deprecated_tx_info(tx_info=new_tx_info, dst=execution_context.deprecated_tx_info);

    // Since we process the revert log backwards, entries before this point belong to the calling
    // contract.
    assert [revert_log] = RevertLogEntry(
        selector=CHANGE_CONTRACT_ENTRY, value=caller_execution_info.contract_address
    );
    let revert_log = &revert_log[1];

    contract_call_helper(
        remaining_gas=remaining_gas,
        block_context=block_context,
        execution_context=execution_context,
    );

    // Entries before this point belong to the callee.
    assert [revert_log] = RevertLogEntry(selector=CHANGE_CONTRACT_ENTRY, value=contract_address);
    let revert_log = &revert_log[1];

    return ();
}
