### Title
Malicious P2P Peer Can Inject Arbitrary `state_root` into Block Header Storage Without Verification — (`crates/apollo_p2p_sync/src/client/header.rs`)

### Summary

`HeaderStreamBuilder::parse_data_for_block` accepts a `SignedBlockHeader` from any p2p peer and passes it directly to storage with no cryptographic verification of the `state_root` field (or the `block_hash` field). The only guards are a block-number ordering check and a signature-vector-length check. A malicious peer can supply an arbitrary `GlobalRoot` value in `block_header_without_hash.state_root`, which is committed to storage verbatim. Every downstream consumer that reads the stored header's `state_root` — most critically `starknet_getStateUpdate` — will serve the attacker-supplied value as an authoritative `new_root`/`old_root`.

---

### Finding Description

**Entrypoint**: `HeaderStreamBuilder::parse_data_for_block` [1](#0-0) 

The function performs exactly two checks on the incoming `SignedBlockHeader`:

1. `block_number` matches the expected sequence number.
2. `signatures.len() == ALLOWED_SIGNATURES_LENGTH` (vector length only, no cryptographic verification).

There is no check that:
- `state_root` corresponds to the actual Patricia trie root of the committed state.
- `block_hash` is the correct Poseidon hash over `(BLOCK_HASH_VERSION, block_number, state_root, ...)`.
- The `BlockSignature` is cryptographically valid against the sequencer's public key (the function `verify_block_signature` exists but is never called in this path). [3](#0-2) 

**Storage write**: `write_to_storage` calls `append_header` with the full, unvalidated `BlockHeader`, persisting the attacker-supplied `state_root` into the headers table. [4](#0-3) 

**`state_root` field location**: `BlockHeaderWithoutHash.state_root: GlobalRoot` — a plain `StarkHash` wrapper with no invariant enforcement. [5](#0-4) 

**RPC impact path**: `starknet_getStateUpdate` reads `state_root` directly from the stored header and returns it as `new_root` (current block) and `old_root` (previous block's header). The RPC test confirms this exact data flow: [6](#0-5) 

**Block hash impact**: `calculate_block_hash` chains `state_root` as the third element of the Poseidon hash. If the block hash is ever recomputed from the stored header (e.g., for proof or L1 verification), the wrong `state_root` propagates into the hash. [7](#0-6) 

The correct `GlobalRoot` invariant is `Poseidon(GLOBAL_STATE_VERSION, contract_state_root, contract_class_root)` over the actual Patricia tries: [8](#0-7) 

No guard in the p2p sync header path enforces this invariant.

---

### Impact Explanation

Any node running p2p header sync (i.e., a full node or a sequencer in sync mode) that connects to a malicious peer will store the attacker-supplied `state_root`. Consequences:

- **`starknet_getStateUpdate` returns wrong `old_root`/`new_root`** — authoritative-looking incorrect values served to all RPC clients.
- **Block hash mismatch** — if the block hash is recomputed from the stored header for L1 verification or proof production, the wrong `state_root` causes a hash that does not match the on-chain commitment.
- **Proof/SNOS input corruption** — the OS `calculate_block_hash` Cairo function takes `state_root` as a direct input; a wrong stored value fed into proof generation produces an invalid proof. [9](#0-8) 

Impact category: **High — RPC returns an authoritative-looking wrong value** (`starknet_getStateUpdate` `old_root`/`new_root`), with potential escalation to Critical if the corrupted header feeds into proof production or L1 verification.

---

### Likelihood Explanation

Any peer that can establish a p2p connection and respond to a `BlockHeadersRequest` query can trigger this. No operator privilege is required. The attacker only needs to:
1. Respond with a `SignedBlockHeader` whose `block_number` matches the expected sequence.
2. Include exactly one `BlockSignature` element (content irrelevant — it is never verified).
3. Set `block_header_without_hash.state_root` to any arbitrary `GlobalRoot`.

The node will accept and store the header.

---

### Recommendation

1. **Verify the block hash**: After receiving a `SignedBlockHeader`, recompute `calculate_block_hash` from the received fields and compare against `block_header.block_hash`. This binds `state_root` to the hash.
2. **Verify the block signature**: Call `verify_block_signature` against the known sequencer public key before accepting any header. The signature covers `Poseidon(block_hash, state_diff_commitment)`, so a valid signature over a verified block hash transitively authenticates `state_root`.
3. **Verify `parent_hash`**: The existing TODO at line 102 should be resolved — chain continuity prevents substitution attacks across blocks. [10](#0-9)

### Proof of Concept

```rust
// Attacker-controlled peer sends:
SignedBlockHeader {
    block_header: BlockHeader {
        block_hash: BlockHash(/* any value */),
        block_header_without_hash: BlockHeaderWithoutHash {
            block_number: BlockNumber(N),  // matches expected sequence
            state_root: GlobalRoot(Felt::from(0xdeadbeef_u64)),  // arbitrary
            ..Default::default()
        },
        state_diff_length: Some(0),
        ..Default::default()
    },
    signatures: vec![BlockSignature::default()],  // length == 1, content unchecked
}
```

`parse_data_for_block` passes both guards (block number matches, `signatures.len() == 1`) and returns `Ok(Some(signed_block_header))`. `write_to_storage` then calls `append_header`, persisting `state_root = 0xdeadbeef`. A subsequent `starknet_getStateUpdate` call for block N returns `new_root: 0xdeadbeef`.

```rust
pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
```

```text
func calculate_block_hash{poseidon_ptr: PoseidonBuiltin*}(
    block_info: BlockInfo*,
    header_commitments: BlockHeaderCommitments*,
    gas_prices_hash: felt,
    state_root: felt,
    previous_block_hash: felt,
    starknet_version: felt,
) -> felt {
    static_assert BlockInfo.SIZE == 3;
    static_assert BlockHeaderCommitments.SIZE == 5;

    let hash_state = hash_init();
    with hash_state {
        hash_update_single(BLOCK_HASH_VERSION);
        hash_update_single(block_info.block_number);
        hash_update_single(state_root);
        hash_update_single(block_info.sequencer_address);
        hash_update_single(block_info.block_timestamp);
        hash_update_single(header_commitments.packed_lengths);
        hash_update_single(header_commitments.state_diff_commitment);
        hash_update_single(header_commitments.transaction_commitment);
        hash_update_single(header_commitments.event_commitment);
        hash_update_single(header_commitments.receipt_commitment);
        hash_update_single(gas_prices_hash);
        hash_update_single(starknet_version);
        hash_update_single(0);
        hash_update_single(previous_block_hash);
    }

    let block_hash = hash_finalize(hash_state=hash_state);
    return block_hash;
}
```
