### Title
Missing Declaration Check in `execute_replace_class` Allows Permanent Contract Fund Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the new class hash has been declared before permanently updating a contract's class hash in state. Any contract can invoke this syscall with an arbitrary, undeclared class hash, rendering itself permanently non-executable and freezing all funds held within it. This is directly analogous to M-09: just as `openPosition()` failed to check `renewalCutoffTime`, `execute_replace_class` fails to check whether the replacement class hash is declared, allowing a state-transition bypass that permanently locks funds.

---

### Finding Description

In `execute_replace_class`, the OS accepts any `class_hash` value from the `ReplaceClassRequest` and writes it directly into the contract's `StateEntry` without verifying that the hash corresponds to a declared class: [1](#0-0) 

The critical missing check is explicitly acknowledged by the developers themselves: [2](#0-1) 

The state update proceeds unconditionally after gas deduction: [3](#0-2) 

The revert log entry is written, meaning the change is only undone if the **entire transaction** reverts. If the transaction succeeds, the class hash is permanently committed to state: [4](#0-3) 

For contrast, the `deploy_contract` path correctly enforces that the deployed class hash must be a known, pre-loaded compiled class (via `guess_compiled_class_facts` and `validate_compiled_class_facts_post_execution`): [5](#0-4) 

But `replace_class` bypasses this entire validation pipeline. The class hash written into state is never cross-checked against the compiled class facts bundle.

The `execute_declare_transaction` path enforces `prev_value=0` to prevent re-declaration, and the `deploy_contract` path enforces `state_entry.class_hash = UNINITIALIZED_CLASS_HASH`. Neither of these guards applies to `replace_class`, which accepts any arbitrary felt as the new class hash: [6](#0-5) [7](#0-6) 

---

### Impact Explanation

Once a contract's class hash is updated to an undeclared value via a successful (non-reverted) transaction:

1. The contract's `StateEntry.class_hash` is permanently set to the undeclared hash in the committed state trie.
2. Any future transaction invoking this contract causes the OS to look up the compiled class for the undeclared hash. Since no compiled class exists for it, execution cannot proceed correctly.
3. All funds (tokens, NFTs, or any assets) held in the contract's storage become permanently inaccessible — no withdrawal, transfer, or recovery function can execute.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

- The `replace_class` syscall is callable by **any contract** from within its own execution context — no privileged role, operator key, or special permission is required.
- The attacker only needs to deploy a contract (e.g., a fake vault or DeFi protocol), attract user deposits, and then invoke `replace_class` with a fabricated felt value as the class hash.
- The attack is a single transaction and requires no timing dependency, no Sybil attack, and no network-level access.
- The TODO comment confirms the developers are aware the check is absent, meaning the missing guard is not an oversight in documentation but a real gap in the enforced invariants of the OS.

---

### Recommendation

Before writing the new `StateEntry`, add a validation that `class_hash` exists in `contract_class_changes` (i.e., has been declared in the current or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` for the given `class_hash` and assert the returned compiled class hash is non-zero:

```cairo
// Verify the class hash has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the guard already present in `execute_declare_transaction`: [8](#0-7) 

---

### Proof of Concept

1. Attacker declares a legitimate class `C_legit` and deploys a vault contract `V` using it. `V` exposes a `deposit()` function and a hidden `freeze()` function.
2. Users call `deposit()` and transfer tokens into `V`'s storage.
3. Attacker calls `freeze()`, which internally invokes the `replace_class` syscall with `class_hash = 0xdeadbeef` (an arbitrary undeclared felt).
4. The OS executes `execute_replace_class`: gas is deducted, the state entry for `V` is updated to `class_hash=0xdeadbeef`, a revert log entry is written, and the function returns successfully.
5. The transaction is not reverted; the new class hash is committed to the state trie.
6. Any subsequent transaction calling `V` causes the OS to look up the compiled class for `0xdeadbeef`. No such compiled class exists in the facts bundle. Execution fails for every future call.
7. All user funds deposited in `V`'s storage are permanently frozen with no recovery path. [9](#0-8)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
