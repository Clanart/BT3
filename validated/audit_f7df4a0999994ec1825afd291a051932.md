### Title
Missing Zero Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts an attacker-controlled `class_hash` of zero without validation. Setting a contract's class hash to `0` (which equals `UNINITIALIZED_CLASS_HASH`) permanently renders the contract uncallable at the OS level, freezing any funds held within it. The developers themselves flagged this missing check with a TODO comment that has not been resolved.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the new class hash directly from the syscall request and writes it into the contract state without any zero-value guard:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The constant `UNINITIALIZED_CLASS_HASH = 0` is defined in `commitment.cairo`: [2](#0-1) 

When a contract's `class_hash` is set to `0`, any subsequent call to that contract enters `execute_entry_point`, which performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // = 0
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,           // = 0, no such compiled class exists
);
``` [3](#0-2) 

Since no compiled class with hash `0` is ever declared, `find_element` raises a Cairo assertion failure — a prover-level hard failure, not a graceful revert. The contract becomes permanently uncallable.

The same missing check exists in the deprecated syscall path: [4](#0-3) 

Additionally, because `deploy_contract` checks `assert state_entry.class_hash = UNINITIALIZED_CLASS_HASH` to guard against re-deployment: [5](#0-4) 

A contract whose class hash has been zeroed out appears "uninitialized" to this guard, opening a secondary path where — if the contract's nonce is also `0` — an attacker could re-deploy a malicious class to the same address and drain its storage.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any contract that calls `replace_class(0)` (or is tricked into doing so via a malicious inner call) will have its `class_hash` committed to the chain as `UNINITIALIZED_CLASS_HASH`. All subsequent calls to that contract fail at the OS prover level. Funds stored in the contract's storage become permanently inaccessible with no recovery path, because the state transition is committed on-chain and no future transaction can successfully execute against the contract.

---

### Likelihood Explanation

The `replace_class` syscall is a standard, permissionless operation available to any Cairo 1 contract. No privileged role or operator action is required. An attacker needs only to deploy a contract that holds user funds (e.g., a vault, escrow, or shared pool) and then invoke `replace_class(0)` from within it. The OS Cairo code performs no validation on the supplied class hash value, and the TODO comment at line 898 of `syscall_impls.cairo` explicitly acknowledges the missing check.

---

### Recommendation

Add an `assert_not_zero(class_hash)` guard immediately after reading `class_hash` from the request in both `execute_replace_class` implementations, before any state update is applied:

```cairo
let class_hash = request.class_hash;
// Reject zero (UNINITIALIZED_CLASS_HASH) to prevent permanent contract freezing.
assert_not_zero(class_hash);
```

This resolves the acknowledged TODO and closes the analogous gap in `deprecated_execute_syscalls.cairo`.

---

### Proof of Concept

1. Attacker deploys `VaultContract` holding user ETH (via ERC-20 storage).
2. `VaultContract` exposes a public function that internally calls the `replace_class` syscall with `class_hash = 0`.
3. Attacker invokes that function. The OS `execute_replace_class` handler accepts `class_hash = 0` without error and commits `StateEntry { class_hash: 0, ... }` for `VaultContract`'s address.
4. All future calls to `VaultContract` reach `execute_entry_point`, perform `dict_read` returning `compiled_class_hash = 0`, then call `find_element` with key `0` against the compiled class facts bundle — which contains no entry for hash `0` — triggering a Cairo assertion failure.
5. The contract is permanently uncallable. All user funds stored in its storage slots are frozen with no recovery mechanism. [6](#0-5)

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
