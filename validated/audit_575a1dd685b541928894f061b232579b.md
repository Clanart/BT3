### Title
Stale `feemarketParams` Pointer in `MinGasPriceDecorator` Allows Cosmos Transactions to Bypass Fee Enforcement After Governance `MinGasPrice` Change - (File: `ante/cosmos/min_gas_price.go`)

### Summary
`MinGasPriceDecorator` for Cosmos transactions reads `MinGasPrice` from a stale pointer captured at ante handler construction time rather than fetching the live value from the keeper on each call. After a governance-approved `MinGasPrice` increase, any unprivileged user can submit Cosmos transactions paying less than the new minimum and have them accepted and committed to the chain.

### Finding Description
In `newCosmosAnteHandler` (and `newLegacyCosmosAnteHandlerEip712`), `feemarketParams` is fetched once from the keeper at construction time and its address is passed to `NewMinGasPriceDecorator`:

    ...

The `feesKeeper` field is stored but **never used** in `AnteHandle` to fetch the current on-chain `MinGasPrice`. The pointed-to `feemarketParams` struct is never mutated after construction, so any governance change to `MinGasPrice` via `MsgUpdateParams` is invisible to this decorator for the lifetime of the node process.

This is in direct contrast to the Ethereum ante handler path (`newEthAnteHandler`), which fetches fresh params on every invocation:

```go
// evmd/ante/handler_options.go:88-95

``` [3](#0-2) 

The same stale-pointer pattern also affects `NewDynamicFeeChecker` when `options.DynamicFeeChecker` is enabled, where the stale `feemarketParams.BaseFee` is used to compute the EIP-1559 base fee for Cosmos transactions:

```go
// ante/evm/fee_checker.go:56

``` [4](#0-3) 

### Impact Explanation
After a governance-approved `MinGasPrice` increase (e.g., from 1 to 10 to combat spam), the `MinGasPriceDecorator` continues to enforce the old value (1). Any user can submit a Cosmos transaction with `gasPrice = 5` — below the new minimum — and it will pass the ante handler and be committed to the chain. This is an ante handler fee market bug that permits transactions violating the current fee floor to commit, and causes valid user fees to be mis-accounted relative to the governance-mandated minimum.

### Likelihood Explanation
`MinGasPrice` governance changes are a routine operational event (spam mitigation, token price adjustment). The attacker entry path requires no special privileges: submit any Cosmos transaction with a fee between the old and new `MinGasPrice` after the governance proposal executes. The bug is triggered on every such transaction for the entire lifetime of the node process until it is restarted.

### Recommendation
In `MinGasPriceDecorator.AnteHandle`, replace the stale pointer read with a live keeper fetch:

```go
// ante/cosmos/min_gas_price.go
func (mpd MinGasPriceDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
    minGasPrice := mpd.feesKeeper.GetParams(ctx).MinGasPrice  // live read
    ...
}
```

Apply the same fix to `NewDynamicFeeChecker`: pass the keeper and fetch `feemarketParams` inside the closure using `ctx`, rather than capturing a stale pointer at construction time.

### Proof of Concept
1. Chain starts with `MinGasPrice = 1`. `newCosmosAnteHandler` is called once; `feemarketParams.MinGasPrice = 1` is captured.
2. Governance proposal executes `MsgUpdateParams` setting `MinGasPrice = 10`.
3. Attacker broadcasts a Cosmos `MsgSend` with `gasPrice = 5`, `gasLimit = 100000`, total fee = `500000`.
4. `MinGasPriceDecorator.AnteHandle` reads `mpd.feemarketParams.MinGasPrice = 1` (stale). Required fee = `1 * 100000 = 100000`. Provided fee `500000 >= 100000` → check passes.
5. Transaction is included in a block despite paying `5` per gas, below the governance-mandated minimum of `10`.
6. The same bypass works for any fee between `1` and `10` for the entire node lifetime. [5](#0-4) [6](#0-5)

	feesKeeper      interfaces.FeeMarketKeeper
	evmDenom        string

// NewMinGasPriceDecorator creates a new MinGasPriceDecorator instance used only for
// Cosmos transactions.
func NewMinGasPriceDecorator(fk interfaces.FeeMarketKeeper, evmDenom string, feemarketParams *feemarkettypes.Params) MinGasPriceDecorator {
	return MinGasPriceDecorator{feesKeeper: fk, evmDenom: evmDenom, feemarketParams: feemarketParams}
}

func (mpd MinGasPriceDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
	feeTx, ok := tx.(sdk.FeeTx)
	if !ok {
		return ctx, errorsmod.Wrapf(errortypes.ErrInvalidType, "invalid transaction type %T, expected sdk.FeeTx", tx)
	}

	minGasPrice := mpd.feemarketParams.MinGasPrice

	// Short-circuit if min gas price is 0 or if simulating
	if minGasPrice.IsZero() || simulate {
		return next(ctx, tx, simulate)
	}
	minGasPrices := sdk.DecCoins{
		{
			Denom:  mpd.evmDenom,
			Amount: minGasPrice,
		},
	}

	feeCoins := feeTx.GetFee()
	gas := feeTx.GetGas()

	requiredFees := make(sdk.Coins, 0)

	// Determine the required fees by multiplying each required minimum gas
	// price by the gas limit, where fee = ceil(minGasPrice * gasLimit).
	gasLimit := sdkmath.LegacyNewDecFromBigInt(new(big.Int).SetUint64(gas))

	for _, gp := range minGasPrices {
		fee := gp.Amount.Mul(gasLimit).Ceil().RoundInt()
		if fee.IsPositive() {
			requiredFees = requiredFees.Add(sdk.Coin{Denom: gp.Denom, Amount: fee})
		}
	}

	if !feeCoins.IsAnyGTE(requiredFees) {
		return ctx, errorsmod.Wrapf(errortypes.ErrInsufficientFee,
			"provided fee < minimum global fee (%s < %s). Please increase the gas price.",
			feeCoins,
			requiredFees)
	}

	return next(ctx, tx, simulate)
}
