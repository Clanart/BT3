### Title
Missing Validation of `input_end >= input_start` in `execute_keccak` Allows Attacker to Cause Unprovable Block (Network Halt) - (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_keccak` function in `syscall_impls.cairo` computes `len = input_end - input_start` from an attacker-controlled syscall request and immediately passes it to `unsigned_div_rem` without first asserting that `len` is non-negative. If a contract submits a keccak syscall with `input_end < input_start`, the resulting `len` is a huge field element (≈ P − 1). `unsigned_div_rem` requires its quotient to fit in a range-check cell (`< 2^128`), but no valid quotient exists for such a large dividend, making the Cairo proof unprovable. The sequencer (Rust) handles the invalid syscall gracefully by writing a failure response, so the transaction is included in the block — but the OS Cairo program then fails irrecoverably when it tries to re-verify the same syscall, halting the network.

---

### Finding Description

In `execute_keccak`:

```cairo
tempvar input_start = request.input_start;
tempvar input_end = request.input_end;
let len = input_end - input_start;
let (local q, r) = unsigned_div_rem(len, KECCAK_FULL_RATE_IN_WORDS);
``` [1](#0-0) 

`input_start` and `input_end` come directly from the `KeccakRequest` struct written by the calling contract into the syscall segment. There is no `assert_nn` or `assert_nn_le` guard before the division. [2](#0-1) 

`unsigned_div_rem` in Cairo enforces `0 ≤ q < 2^128` via a range-check builtin and then asserts `value = q * div + r`. When `len ≈ P − 1` (because `input_end < input_start`), the required quotient is `(P−1) / 17 ≈ 2^247`, which far exceeds `2^128`. No valid witness satisfies both the range-check constraint and the equality constraint simultaneously, so the proof is unprovable.

The check `if (r != 0)` that would write `ERROR_INVALID_INPUT_LEN` is placed **after** the `unsigned_div_rem` call, so it is never reached. [3](#0-2) 

Compare with the signature-length validation present in `execute_meta_tx_v0`, which correctly uses `assert_nn_le` before using the length:

```cairo
assert_nn_le(request.signature_end - request.signature_start, SIERRA_ARRAY_LEN_BOUND - 1);
``` [4](#0-3) 

No equivalent guard exists for `input_end - input_start` in `execute_keccak`.

---

### Impact Explanation

The StarkNet OS is a Cairo program whose successful execution produces a STARK proof that finalizes a block on L1. If the OS Cairo program encounters an unsatisfiable constraint (range-check failure inside `unsigned_div_rem`), no valid proof can be generated for that block. The block is permanently stuck; subsequent blocks built on top of it also cannot be finalized. This constitutes a **total network shutdown** — matching the allowed impact "High. Network not being able to confirm new transactions."

---

### Likelihood Explanation

The attack requires only:
1. Deploying a contract (permissionless on StarkNet).
2. Invoking it so it issues a `keccak` syscall with `input_end < input_start`.

The Rust sequencer handles the invalid syscall gracefully (returns `ERROR_INVALID_INPUT_LEN` to the contract) and includes the transaction in the block. The OS Cairo program then re-processes the same syscall segment and fails irrecoverably. No privileged access, leaked key, or network-level attack is required. Any unprivileged account can trigger this with a single transaction.

---

### Recommendation

Add a non-negativity assertion on `len` before calling `unsigned_div_rem`, mirroring the pattern used elsewhere in the file:

```cairo
tempvar input_start = request.input_start;
tempvar input_end = request.input_end;
let len = input_end - input_start;
// Guard: reject if input_end < input_start.
assert_nn(len);  // or assert_nn_le(input_start, input_end)
let (local q, r) = unsigned_div_rem(len, KECCAK_FULL_RATE_IN_WORDS);
```

If `len` fails the range check, the OS should write a failure response (analogous to the `r != 0` branch) rather than aborting proof generation.

---

### Proof of Concept

1. **Deploy** a Cairo 1 contract containing:
   ```rust
   // Pseudocode: write keccak syscall request with input_end < input_start
   let input_start: felt252 = 0x1000;
   let input_end: felt252 = 0x0;   // input_end < input_start
   syscall_keccak(input_start, input_end);
   ```
2. **Invoke** the contract from any account. The sequencer executes the keccak syscall, detects the invalid range, writes `ERROR_INVALID_INPUT_LEN` as the response, and includes the transaction in the block.
3. **Observe** that when the prover runs the OS Cairo program against this block, it reaches `execute_keccak`, computes `len = 0x0 - 0x1000 ≡ P − 0x1000` (a value ≈ 2^251), and calls `unsigned_div_rem(P − 0x1000, 17)`. The required quotient ≈ 2^247 cannot satisfy the `< 2^128` range-check constraint. The proof generation fails.
4. **Result**: The block is permanently unprovable; the network cannot advance past it.

func execute_keccak{
    range_check_ptr, builtin_ptrs: BuiltinPointers*, syscall_ptr: felt*, outputs: OsCarriedOutputs*
}() {
    alloc_locals;
    let request_header = cast(syscall_ptr, RequestHeader*);
    let request = cast(syscall_ptr + RequestHeader.SIZE, KeccakRequest*);
    tempvar input_start = request.input_start;
    tempvar input_end = request.input_end;
    let len = input_end - input_start;
    let (local q, r) = unsigned_div_rem(len, KECCAK_FULL_RATE_IN_WORDS);

    // Note that if KECCAK_GAS_COST > SYSCALL_BASE_GAS_COST, we need to call
    // `reduce_syscall_base_gas` before the 'if' bellow to be consistent with the Sequencer.
    static_assert KECCAK_GAS_COST == SYSCALL_BASE_GAS_COST;
    if (r != 0) {
        let syscall_ptr = syscall_ptr + RequestHeader.SIZE + KeccakRequest.SIZE;
        write_failure_response(
            remaining_gas=request_header.gas, failure_felt=ERROR_INVALID_INPUT_LEN
        );
        return ();
    }

    let required_gas = KECCAK_GAS_COST + q * KECCAK_ROUND_COST_GAS_COST;
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=required_gas, request_struct_size=KeccakRequest.SIZE
    );
