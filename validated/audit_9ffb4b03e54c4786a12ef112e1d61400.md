### Title
Missing Class Declaration Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (`execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the new class hash supplied by a contract is actually declared before committing the replacement. This is directly analogous to the external report's root cause: a required prerequisite check (approval/existence) is absent before a state-mutating external call. Any contract can replace its own class hash with an undeclared value, permanently rendering itself uncallable and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function contains an explicit self-acknowledged TODO confirming the missing check:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The function reads `request.class_hash` and immediately writes it into `contract_state_changes` without performing any lookup into `contract_class_changes` (the declared class registry) to confirm the hash is known to the OS.

When any subsequent transaction calls the affected contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
``` [2](#0-1) 

Because the replaced class hash was never declared, `dict_read` returns `0` (the default for an unwritten dict entry). The OS then calls:

```cairo
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [3](#0-2) 

`find_element` with `key=0` (or any undeclared hash) will fail to locate a matching `CompiledClassFact`, causing a hint-level failure. There is no graceful revert path here — the block containing such a call becomes unprovable. The sequencer must permanently exclude all calls to the affected contract, making it and its funds permanently inaccessible.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any contract that holds ERC20 tokens, acts as a shared vault, or manages user balances can have its class hash replaced with an undeclared value. Once replaced:

1. Every subsequent call to the contract fails at the OS proof level (not a graceful revert).
2. The sequencer cannot include any transaction targeting the contract in a provable block.
3. All funds stored in the contract's storage are permanently frozen with no recovery path.

---

### Likelihood Explanation

**Medium.** The `replace_class` syscall is callable by any contract from within its own execution — no privileged role is required. A malicious contract owner who also manages a shared pool (e.g., a multi-user vault or escrow) can trigger this to freeze other users' funds. The attack requires only a standard invoke transaction from an unprivileged sender.

---

### Recommendation

Before committing the class hash update in `execute_replace_class`, verify that `request.class_hash` has a non-zero entry in `contract_class_changes` (i.e., it has been declared and has a known compiled class hash):

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(
    key=class_hash
);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the check already performed in `execute_entry_point` and closes the gap between the two code paths.

---

### Proof of Concept

1. **Deploy** a contract `VaultWithTrap` that holds user funds and exposes a function `trap()` which calls `replace_class(0xdeadbeef)` — an undeclared class hash.
2. **Submit** an invoke transaction calling `VaultWithTrap.trap()`. The OS executes `execute_replace_class`:
   - `request.class_hash = 0xdeadbeef`
   - No check against `contract_class_changes` is performed.
   - `contract_state_changes` is updated: `VaultWithTrap.class_hash = 0xdeadbeef`. [1](#0-0) 
3. **Submit** any subsequent transaction calling `VaultWithTrap` (e.g., `withdraw()`).
4. The OS enters `execute_entry_point` with `execution_context.class_hash = 0xdeadbeef`.
5. `dict_read` on `contract_class_changes` returns `0` (undeclared).
6. `find_element` searches for `compiled_class_hash=0` in the compiled class facts bundle — not found — hint failure. [3](#0-2) 
7. The block is unprovable. The sequencer must permanently exclude all calls to `VaultWithTrap`. All user funds in the contract are permanently frozen.
