### Title
Cosmos Native Transactions Bypass EIP-1559 Dynamic Base Fee by Default, Enabling Fixed-Cost Block Stuffing - (`evmd/ante/handler_options.go`)

### Summary

When `HandlerOptions.DynamicFeeChecker` is `false` (the Go zero-value default), all Cosmos-SDK native transactions routed through `newCosmosAnteHandler` are never checked against the EIP-1559 dynamic `baseFee`. They are only checked against the static `MinGasPrice` governance parameter (default `0`). This is the direct Ethermint analog of the Aleo `split`-transaction fixed-fee bypass: a whole class of transactions is permanently exempt from the congestion-responsive fee mechanism, allowing an attacker to stuff blocks at a fixed (or zero) cost while EVM transactions must pay an ever-rising base fee.

### Finding Description

`newCosmosAnteHandler` conditionally wires the dynamic fee checker:

```go
var txFeeChecker ante.TxFeeChecker
if options.DynamicFeeChecker {
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
}
// ...
ante.NewDeductFeeDecorator(options.AccountKeeper, options.BankKeeper, options.FeegrantKeeper, txFeeChecker),
``` [1](#0-0) 

`DynamicFeeChecker` is a plain `bool` field with no default enforcement:

```go
// use dynamic fee checker or the cosmos-sdk default one for native transactions
DynamicFeeChecker bool
``` [2](#0-1) 

When `DynamicFeeChecker = false` (the default), `txFeeChecker` is `nil`. The SDK's `NewDeductFeeDecorator` then falls back to `checkTxFeeWithValidatorMinGasPrices`:

```go
func checkTxFeeWithValidatorMinGasPrices(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
    // ...
    if ctx.IsCheckTx() && !minGasPrices.IsZero() {
        // only local validator min-gas-prices, CheckTx only
    }
    return feeCoins, priority, nil  // DeliverTx: no check at all
}
``` [3](#0-2) 

The only other guard for Cosmos txs is `MinGasPriceDecorator`, which short-circuits entirely when `MinGasPrice = 0` (the default):

```go
if minGasPrice.IsZero() || simulate {
    return next(ctx, tx, simulate)
}
``` [4](#0-3) 

Contrast this with EVM transactions, which always go through `newEthAnteHandler` → `CheckEthMinGasPrice` → `VerifyFee`, where `baseFee` is unconditionally enforced:

```go
if baseFee != nil && tx.GasFeeCap().Cmp(baseFee) < 0 {
    return nil, errorsmod.Wrapf(errortypes.ErrInsufficientFee, ...)
}
``` [5](#0-4) 

The dynamic base fee itself rises correctly with block utilization:

```go
if parentGasUsed > parentGasTarget {
    baseFeeDelta := ethermint.BigMax(x.Div(y, baseFeeChangeDenominator), common.Big1)
    return x.Add(parentBaseFee, baseFeeDelta)
}
``` [6](#0-5) 

But this rising base fee is never applied to Cosmos native transactions when `DynamicFeeChecker = false`.

### Impact Explanation

During network congestion, the EIP-1559 base fee rises to price out spam. EVM transactions must pay the rising `baseFee * gasLimit`. Cosmos native transactions (bank sends, staking, governance, IBC relays) are exempt: they pay at most `MinGasPrice * gasLimit` (default: zero). An attacker can flood the block with zero-fee Cosmos native transactions, consuming the entire block gas budget at no cost, while legitimate EVM transactions are priced out. DeFi protocols depending on timely EVM execution (liquidations, collateral adjustments, DEX operations) are disrupted. This matches the allowed High impact: "fee market, ante handler, or proposal handling bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."

### Likelihood Explanation

The `DynamicFeeChecker` field is a `bool` that defaults to `false` in Go. Any chain that does not explicitly set `HandlerOptions{DynamicFeeChecker: true}` is vulnerable. The attack requires only the ability to submit standard Cosmos SDK transactions (no special privileges). The attacker needs enough balance to cover `MinGasPrice * gasLimit` (default: zero). The attack is cheap, repeatable, and deterministic.

### Recommendation

1. Change the default to `DynamicFeeChecker: true` in the reference app setup, or add a validation check in `HandlerOptions.validate()` that rejects `DynamicFeeChecker = false` when the fee market is enabled.
2. Alternatively, remove the conditional and always wire `NewDynamicFeeChecker` for Cosmos txs when London HF is active.
3. Ensure `MinGasPriceDecorator` also enforces the current `baseFee` (not just the static `MinGasPrice`) for Cosmos native transactions.

### Proof of Concept

1. Deploy Ethermint with default `HandlerOptions` (i.e., `DynamicFeeChecker = false`, `MinGasPrice = 0`).
2. Fill several blocks with EVM transactions to drive the base fee up (e.g., to 100 gwei).
3. Submit a flood of `MsgSend` (bank transfer) Cosmos transactions with `fee = 0` and `gas = blockGasLimit / N`.
4. Observe: the Cosmos transactions pass both `MinGasPriceDecorator` (short-circuits at `MinGasPrice = 0`) and `DeductFeeDecorator` (no check during DeliverTx), consuming the full block gas budget at zero cost.
5. Observe: EVM transactions with `gasFeeCap < 100 gwei` are rejected; legitimate EVM users must pay 100 gwei while the attacker pays nothing.
6. The base fee continues to rise (blocks are full), but the attacker's cost remains zero, crowding out all EVM activity indefinitely.

func checkTxFeeWithValidatorMinGasPrices(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
	feeTx, ok := tx.(sdk.FeeTx)
	if !ok {
		return nil, 0, errorsmod.Wrap(errortypes.ErrTxDecode, "Tx must be a FeeTx")
	}

	feeCoins := feeTx.GetFee()
	gas, err := ethermint.SafeInt64(feeTx.GetGas())
	if err != nil {
		return nil, 0, err
	}
	minGasPrices := ctx.MinGasPrices()

	// Ensure that the provided fees meet a minimum threshold for the validator,
	// if this is a CheckTx. This is only for local mempool purposes, and thus
	// is only ran on check tx.
	if ctx.IsCheckTx() && !minGasPrices.IsZero() {
		requiredFees := make(sdk.Coins, len(minGasPrices))

		// Determine the required fees by multiplying each required minimum gas
		// price by the gas limit, where fee = ceil(minGasPrice * gasLimit).
		glDec := sdkmath.LegacyNewDec(gas)

		for i, gp := range minGasPrices {
			fee := gp.Amount.Mul(glDec)
			requiredFees[i] = sdk.NewCoin(gp.Denom, fee.Ceil().RoundInt())
		}

		if !feeCoins.IsAnyGTE(requiredFees) {
			return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, "insufficient fees; got: %s required: %s", feeCoins, requiredFees)
		}
	}

	priority := getTxPriority(feeCoins, gas)
	return feeCoins, priority, nil
}
