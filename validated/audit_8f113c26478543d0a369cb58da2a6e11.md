### Title
`MerkleBlob::batch_insert` Skips Duplicate Key/Hash Validation for Bulk-Inserted Leaves, Corrupting Tree State and Enabling Forged Inclusion Proofs — (File: `crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::batch_insert` uses a fast bulk-insertion path that bypasses the duplicate-key and duplicate-hash guards present in the regular `insert` function. When a caller supplies a key (or leaf hash) that already exists in the tree, `batch_insert` silently overwrites the `block_status_cache` entry without removing the old leaf from the blob, leaving the tree in a structurally corrupt state. Subsequent calls to `get_proof_of_inclusion` operate on the corrupted cache and produce proofs that do not correspond to the actual Merkle root, satisfying the "DataLayer Merkle proof/blob/delta logic … corrupts tree roots, or lets untrusted input prove invalid state" impact criterion.

---

### Finding Description

**Regular `insert` guards (lines 369–374):**

```rust
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

**`batch_insert` bulk path (lines 587–603) — no duplicate check:**

```rust
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    let new_block = Block { … Node::Leaf(LeafNode { parent: Parent(None), hash, key, value }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
``` [2](#0-1) 

`insert_entry_to_blob` unconditionally calls `block_status_cache.add_leaf`, which uses `HashMap::insert` and silently overwrites any existing entry:

```rust
fn add_leaf(&mut self, index: TreeIndex, leaf: LeafNode) {
    self.free_indexes.shift_remove(&index);
    self.key_to_index.insert(leaf.key, index);          // silent overwrite
    self.leaf_hash_to_index.insert(leaf.hash, index);   // silent overwrite
}
``` [3](#0-2) 

The guard at the top of `batch_insert` only routes the **first two** items through the safe `insert` path when `leaf_count() <= 1`; all remaining items — and all items when `leaf_count() > 1` — go through the unchecked bulk path: [4](#0-3) 

**Resulting corruption when a duplicate key K is submitted:**

| State | Before | After `batch_insert` with duplicate K |
|---|---|---|
| Old leaf (index I, key K) | In blob, reachable via cache | Still in blob, **orphaned** — index I not freed, not in `key_to_index` |
| New leaf (index J, key K) | — | In blob, `key_to_index[K] = J` |
| `free_indexes` | Does not contain I | Still does not contain I (leak) |
| Merkle root | Correct | Includes **both** leaves; root is wrong |

`get_proof_of_inclusion` resolves key K through the cache to index J, traverses J's lineage, and emits a proof anchored to the corrupted root. The proof is internally self-consistent for the new leaf but does not match any legitimate root: [5](#0-4) 

`check_integrity` would detect the mismatch (`leaf_count != key_to_index_cache_length`), but it is only invoked explicitly or in test builds (`check_integrity_on_drop: cfg!(test)`): [6](#0-5) 

---

### Impact Explanation

- **Corrupted Merkle root**: the blob contains two leaves for the same key; `calculate_lazy_hashes` propagates a root that no honest tree would produce.
- **Forged inclusion proof**: `get_proof_of_inclusion` returns a structurally valid proof for the new leaf against the corrupted root. A verifier that trusts the root (e.g., one that received it from the same node) will accept the proof even though the underlying tree state is invalid.
- **Orphaned index leak**: the old leaf's blob slot is never freed, so subsequent insertions may reuse the index, further corrupting the tree.

Matches: **High — DataLayer Merkle proof/blob/delta logic … corrupts tree roots, or lets untrusted input prove invalid state.**

---

### Likelihood Explanation

`batch_insert` is exposed directly to Python callers via the `py-bindings` feature: [7](#0-6) 

In the DataLayer sync protocol, a remote store peer supplies the key/value/hash tuples that are fed into `batch_insert`. A malicious or compromised remote peer can trivially include a key that already exists in the local store, triggering the corruption path without any special privilege. No key material, governance role, or chain reorg is required.

---

### Recommendation

Add the same duplicate-key and duplicate-hash guards to the bulk path inside `batch_insert` that already exist in `insert`:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // … existing bulk insertion logic …
}
```

Alternatively, validate the entire input batch for uniqueness before any blob mutation begins, so the function is atomic with respect to errors.

### Proof of Concept

```rust
let mut blob = MerkleBlob::new(vec![]).unwrap();

// Seed the tree with ≥2 leaves so batch_insert skips the safe path entirely.
blob.insert(KeyId(1), ValueId(1), &hash1, InsertLocation::Auto {}).unwrap();
blob.insert(KeyId(2), ValueId(2), &hash2, InsertLocation::Auto {}).unwrap();

// batch_insert with a key that already exists (KeyId(1)) — no error returned.
blob.batch_insert(vec![
    ((KeyId(1), ValueId(99)), hash_dup),   // duplicate key — bypasses guard
    ((KeyId(3), ValueId(3)), hash3),
]).unwrap();

blob.calculate_lazy_hashes().unwrap();

// check_integrity reveals the corruption:
// leaf_count (3 in blob) != key_to_index_cache_length (2 in cache)
blob.check_integrity().unwrap_err(); // IntegrityKeyToIndexCacheLength

// get_proof_of_inclusion returns a proof anchored to the corrupted root.
let proof = blob.get_proof_of_inclusion(KeyId(1)).unwrap();
assert!(proof.valid()); // passes — but root is wrong
```

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
