Audit Report

## Title
Pool admin bypasses `maxAdminSpreadFeeE6` cap via uncapped per-bin additional fees — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

## Summary

`setPoolBinAdditionalFees` forwards caller-supplied `addFeeBuyE6`/`addFeeSellE6` values directly to the pool with no upper-bound validation, while the parallel `setPoolAdminFees` path correctly enforces `maxAdminSpreadFeeE6`. A pool admin can set per-bin fees to `type(uint16).max` (65 535 ≈ 6.5 %) in a single transaction with no timelock, adding that amount on top of the global spread fee and exceeding the protocol's hard ceiling. Traders on the affected bin pay materially more than the advertised rate, with the excess accruing to the admin fee destination.

## Finding Description

`setPoolAdminFees` enforces the factory-level cap before writing any fee:

```solidity
// MetricOmmPoolFactory.sol:414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

`setPoolBinAdditionalFees` performs no analogous check — it passes the raw caller values straight through:

```solidity
// MetricOmmPoolFactory.sol:450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

`setBinAdditionalFees` in the pool stores the values with only a bin-index range check:

```solidity
// MetricOmmPool.sol:464-474
BinState storage s = _binStates[bin];
s.addFeeBuyE6 = addFeeBuyE6;
s.addFeeSellE6 = addFeeSellE6;
``` [3](#0-2) 

Every swap then adds the per-bin fee on top of the global base fee:

- Buy path (line 910): `params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)` [4](#0-3) 
- Sell path (line 1177): `params.baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6)` [5](#0-4) 

`HARD_MAX_SPREAD_FEE_E6 = 200_000` (20 %) is the protocol's absolute ceiling for global spread fees, and `maxAdminSpreadFeeE6` is initialised to that value. [6](#0-5) [7](#0-6) 

Because `addFeeBuyE6`/`addFeeSellE6` are `uint16`, the pool admin can set them to 65 535 (~6.5 %) with no cap check, no timelock, and no prior fee-collection step — pushing the effective per-bin fee above the hard ceiling and silently overcharging traders.

## Impact Explanation

This is a confirmed admin-boundary break: the pool admin exceeds `maxAdminSpreadFeeE6` (and potentially `HARD_MAX_SPREAD_FEE_E6`) via the per-bin path, which the factory owner intended as an absolute limit. The direct consequence is loss of user principal — traders on the affected bin receive less output than the advertised fee rate implies, with the excess flowing to the admin fee destination. The attack is repeatable on any bin, any pool, and any swap.

## Likelihood Explanation

The pool admin is a semi-trusted role (EOA or multisig) that requires only two transactions to execute the attack: raise the per-bin fee before a target swap, then reset it afterward. No special pool configuration, malicious initial setup, or privileged factory-owner cooperation is needed. Any admin watching the mempool can front-run victim swaps.

## Recommendation

1. **Add a cap check in `setPoolBinAdditionalFees`**: validate that `addFeeBuyE6` and `addFeeSellE6` do not exceed `maxAdminSpreadFeeE6` (or a dedicated per-bin cap stored on the factory), mirroring the guard already present in `setPoolAdminFees`.
2. **Collect accrued fees before updating per-bin fees**, mirroring the `collectFees` call at lines 418–425 of `setPoolAdminFees`, so that fee changes cannot retroactively re-attribute already-accrued amounts.
3. **Introduce a timelock** on per-bin fee increases so that users have advance notice before a fee hike takes effect.

## Proof of Concept

```solidity
function test_admin_frontRuns_swap_with_binFee() public {
    // Pool is live with addFeeBuyE6 = 0 (advertised to users)
    (,,, uint16 buyFeeBefore,) = PoolStateLibrary._binState(pool, 0);
    assertEq(buyFeeBefore, 0);

    // Admin front-runs Alice's swap: spike per-bin fee to type(uint16).max = 65535 (~6.5%)
    vm.prank(admin);
    factory.setPoolBinAdditionalFees(pool, 0, type(uint16).max, 0);
    // No revert — no cap check exists

    (,,, uint16 buyFeeAfter,) = PoolStateLibrary._binState(pool, 0);
    assertEq(buyFeeAfter, type(uint16).max);

    // Alice's swap executes at the inflated fee
    vm.prank(alice);
    router.exactInputSingle(
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            tokenIn: address(token1), tokenOut: address(token0),
            pool: pool, recipient: alice,
            amountIn: 1_000e18, amountOutMinimum: 0, sqrtPriceLimitX64: 0
        })
    );

    // Admin resets fee to 0
    vm.prank(admin);
    factory.setPoolBinAdditionalFees(pool, 0, 0, 0);
    // amountOut is materially lower than at fee=0; difference flows to admin fee destination
}
```
