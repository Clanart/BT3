### Title
Stale `feemarketParams` Captured at Ante Handler Construction Causes Base Fee Bypass for Cosmos Transactions — (`evmd/ante/handler_options.go`, `ante/evm/fee_checker.go`)

---

### Summary

`newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712` read `evmParams` and `feemarketParams` from the keeper **once at construction time** and capture them in the `NewDynamicFeeChecker` and `MinGasPriceDecorator` closures. Because `feemarketParams.BaseFee` is updated every block by `BeginBlock`, the captured value becomes stale immediately after the first block. Every subsequent Cosmos transaction processed by these ante handlers validates fees against the **genesis-time base fee**, not the current block's base fee. This allows Cosmos transactions to be included with fees below the live EIP-1559 base fee, bypassing the fee market entirely for the Cosmos transaction path.

---

### Finding Description

In `evmd/ante/handler_options.go`, `newCosmosAnteHandler` reads params at construction time:

```go
func newCosmosAnteHandler(ctx sdk.Context, options HandlerOptions, ...) sdk.AnteHandler {
    evmParams := options.EvmKeeper.GetParams(ctx)           // captured once
    feemarketParams := options.FeeMarketKeeper.GetParams(ctx) // captured once
    ...
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
    ...
    cosmos.NewMinGasPriceDecorator(options.FeeMarketKeeper, evmDenom, &feemarketParams)
``` [1](#0-0) 

The same pattern appears in `newLegacyCosmosAnteHandlerEip712`: [2](#0-1) 

The `NewDynamicFeeChecker` closure captures `feemarketParams` by pointer and uses it on every transaction call:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
``` [3](#0-2) 

`types.GetBaseFee` reads `feemarketParams.BaseFee` directly from the captured struct:

```go
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    ...
    baseFee := feemarketParams.GetBaseFee()
    ...
}
``` [4](#0-3) 

`feemarketParams.BaseFee` is updated every block by `BeginBlock` → `SetBaseFee` → `SetParams`, which writes the new base fee into the KV store and into the in-memory `Params` struct: [5](#0-4) [6](#0-5) 

However, the `feemarketParams` pointer captured in the `NewDynamicFeeChecker` closure points to the struct read at construction time. `SetParams` writes to the KV store and creates a new `Params` value — it does **not** update the captured pointer. The closure therefore always reads the genesis-time `BaseFee`.

By contrast, `newEthAnteHandler` (for EVM transactions) correctly reads fresh state on every call via `EVMBlockConfig`, which reads from the KV store each time: [7](#0-6) [8](#0-7) 

The same staleness affects `MinGasPriceDecorator`, which reads `mpd.feemarketParams.MinGasPrice` from the captured pointer: [9](#0-8) 

---

### Impact Explanation

**High.** This is an ante handler fee market bug that permits invalid Cosmos transactions to commit with fees below the current EIP-1559 base fee. After the first block, the captured `feemarketParams.BaseFee` diverges from the live base fee. Any Cosmos transaction whose `feeCap` satisfies `staleBaseFee ≤ feeCap < currentBaseFee` passes `NewDynamicFeeChecker` and is included in the block. The effective fee charged is computed using the stale (lower) base fee, so the sender underpays. This mis-accounts user fees and breaks the fee market invariant for the Cosmos transaction path. The impact matches: *"EVM state transition, gas refund, fee market, ante handler, mempool, or proposal handling bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

---

### Likelihood Explanation

**Medium.** The base fee changes every block whenever block gas usage deviates from the target. On any active chain, the base fee will diverge from the genesis value within a few blocks. Any unprivileged user can submit a Cosmos transaction with a fee cap between the stale and current base fee. No special access is required. The only limiting factor is that the attacker must know the stale base fee (genesis value, publicly known) and the current base fee (queryable via RPC).

---

### Recommendation

Read `feemarketParams` and `evmParams` fresh from the keeper **inside** the returned handler closure, not at construction time. Follow the same pattern as `newEthAnteHandler`, which calls `options.EvmKeeper.EVMBlockConfig(ctx, ...)` on every invocation:

```go
func newCosmosAnteHandler(options HandlerOptions, ...) sdk.AnteHandler {
    return func(ctx sdk.Context, tx sdk.Tx, simulate bool) (sdk.Context, error) {
        evmParams := options.EvmKeeper.GetParams(ctx)
        feemarketParams := options.FeeMarketKeeper.GetParams(ctx)
        // use fresh params below
        ...
    }
}
```

Remove the `ctx sdk.Context` parameter from `newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712` since it is only used to read params that should be read per-transaction instead.

---

### Proof of Concept

1. Chain starts with genesis base fee = 1,000,000,000 wei. `newCosmosAnteHandler` is called once at app startup; `feemarketParams.BaseFee = 1_000_000_000` is captured.
2. After several blocks of high gas usage, `CalculateBaseFee` raises the live base fee to 2,000,000,000 wei and `SetBaseFee` writes it to the KV store.
3. Attacker submits a Cosmos transaction (e.g., `MsgSend`) with `fee = 1,500,000,000 * gas` and `MaxPriorityPrice = MaxInt64`.
4. `NewDynamicFeeChecker` computes `baseFee = types.GetBaseFee(...)` using the captured stale `feemarketParams.BaseFee = 1_000_000_000`.
5. `feeCap = 1,500,000,000 ≥ baseFeeInt = 1,000,000,000` → check passes.
6. `effectivePrice = min(baseFee + tip, feeCap) = 1,500,000,000` — charged at stale rate.
7. Transaction is included in the block despite paying below the current base fee of 2,000,000,000 wei. The fee market is bypassed for this Cosmos transaction. [10](#0-9)

func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
