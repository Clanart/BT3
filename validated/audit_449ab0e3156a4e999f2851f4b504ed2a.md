### Title
`ProofOfInclusion::valid()` Final Root Check Is a Tautology — Forged Inclusion Proofs Always Pass - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` contains a final check `existing_hash == self.root_hash()` that is mathematically guaranteed to be `true` whenever the loop completes without returning `false`. Because `root_hash()` returns `last.combined_hash` from the proof itself — the same value that `existing_hash` was just set to — the check is a tautology. An attacker can construct a fully forged `ProofOfInclusion` for any arbitrary `node_hash` with internally consistent `layers`, and `valid()` will return `true`.

### Finding Description

`ProofOfInclusion::valid()` is defined as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()   // <-- tautology
}
``` [1](#0-0) 

And `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // <-- taken from the proof itself
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

**The tautology:** After the loop completes without returning `false`, `existing_hash` holds `calculated_hash` for the last layer, which was already verified to equal `layer.combined_hash`. `root_hash()` returns `last.combined_hash`. Therefore `existing_hash == self.root_hash()` is always `true` when `layers` is non-empty. The final check provides zero security guarantee.

This is the direct analog to the external report: just as `tokensNeeded = totalSupply * weight * newRatio` becomes 0 when `newRatio` is 0 (making `balance >= 0` trivially true), here `existing_hash == self.root_hash()` is trivially true because both sides resolve to the same attacker-controlled value.

**Forging a proof** requires only that each layer's `combined_hash` is correctly computed from the previous hash and `other_hash` — a computation the attacker performs themselves. No knowledge of the real tree is needed.

The struct is `Streamable` and fully deserializable from bytes, and `valid()` is exposed to Python via `py_valid()`: [3](#0-2)

### Impact Explanation

An attacker can construct a `ProofOfInclusion` claiming any arbitrary `node_hash` (representing any key-value pair) is included in a DataLayer Merkle tree, and `valid()` will return `true`. This lets untrusted input prove invalid state — forged inclusion of data that was never inserted into the tree. Any Python consumer that calls `proof.valid()` without separately comparing `proof.root_hash()` against a trusted external root is fully bypassed.

This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.**

### Likelihood Explanation

`ProofOfInclusion` is `Streamable` and constructible from Python via `ProofOfInclusion(node_hash=..., layers=[...])`. The attacker only needs to compute one SHA-256 hash per layer to produce a self-consistent proof. The method name `valid()` strongly implies complete proof validation, making it likely that callers rely on it without separately verifying `root_hash()` against a trusted source.

### Recommendation

The `valid()` method must accept an external trusted root hash and compare against it:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    // ... existing loop ...
    existing_hash == *trusted_root  // compare against external trusted root
}
```

Alternatively, remove the misleading `valid()` method and require callers to always compare `proof.root_hash()` against a trusted root explicitly. The current `valid()` should at minimum be renamed to `internally_consistent()` to prevent misuse.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
from chia_rs.sized_bytes import bytes32
import hashlib

# Attacker-chosen arbitrary node_hash (fake key-value pair hash)
fake_node_hash = bytes32(b'\xAA' * 32)
arbitrary_other = bytes32(b'\xBB' * 32)

# Compute combined_hash honestly (attacker does this themselves)
h = hashlib.sha256()
h.update(b'\x02')          # internal_hash prefix
h.update(arbitrary_other)  # other_hash on Left side
h.update(fake_node_hash)
combined = bytes32(h.digest())

layer = ProofOfInclusionLayer(
    other_hash_side=0,      # Side::Left
    other_hash=arbitrary_other,
    combined_hash=combined,
)

forged_proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

# valid() returns True for a completely forged proof
assert forged_proof.valid() == True
# root_hash() returns attacker-chosen combined hash, not any real tree root
assert forged_proof.root_hash() == combined
```

The loop verifies `calculated_hash == layer.combined_hash` (passes, attacker computed it correctly), then the final check `existing_hash == self.root_hash()` compares `combined` against `last.combined_hash` which is also `combined` — always `true`. [1](#0-0) [5](#0-4)
