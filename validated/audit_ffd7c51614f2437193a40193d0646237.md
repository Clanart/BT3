### Title
`execute_secp_get_xy` Uses Hard `assert_uint256_lt` on User-Controlled EC Point Coordinates, Causing Unprovable Block on Invalid Input - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

`execute_secp_get_xy` in the StarkNet OS syscall handler validates the coordinates of a user-supplied EC point using `assert_uint256_lt`, a hard Cairo assertion that panics the entire VM if the condition fails. Unlike every other secp syscall handler in the same file, this function does not use the graceful `write_failure_response` path for invalid inputs. A single transaction calling `secp256k1_get_xy` or `secp256r1_get_xy` with an EC point whose coordinates exceed the curve prime will cause the Cairo OS execution to panic, making the block unprovable and halting the network.

---

### Finding Description

`execute_secp_get_xy` is the shared handler for both `SECP256K1_GET_XY_SELECTOR` and `SECP256R1_GET_XY_SELECTOR` syscalls. Its implementation:

```cairo
func execute_secp_get_xy{range_check_ptr, syscall_ptr: felt*}(
    curve_prime: Uint256, gas_cost: felt
) {
    ...
    let success = reduce_syscall_gas_and_write_response_header(...);  // writes success header immediately
    if (success == 0) { return (); }

    tempvar ec_point = request.ec_point;
    let (x) = bigint_to_uint256(ec_point.x);
    let (y) = bigint_to_uint256(ec_point.y);

    assert_uint256_lt(x, curve_prime);   // <-- PANICS the entire VM if x >= prime
    assert_uint256_lt(y, curve_prime);   // <-- PANICS the entire VM if y >= prime
    ...
}
``` [1](#0-0) 

The function calls `reduce_syscall_gas_and_write_response_header`, which **immediately writes a success `ResponseHeader` with `failure_flag=0`** before any coordinate validation occurs. [2](#0-1) 

Then `assert_uint256_lt` — a hard Cairo assertion imported from `starkware.cairo.common.uint256` — is called on the user-supplied `ec_point` coordinates. If either coordinate is `>= curve_prime`, the assertion fails and the Cairo VM panics, aborting the entire OS execution. [3](#0-2) 

**Contrast with every other secp syscall handler** in the same file. For example, `execute_secp256k1_new` and `execute_secp256r1_new` use `reduce_syscall_base_gas` (which does not pre-write the success header), then check validity with the non-panicking `uint256_lt`, and call `write_failure_response` for invalid inputs: [4](#0-3) 

Similarly, `execute_secp256k1_get_point_from_x` and `execute_secp256r1_get_point_from_x` both validate `x` with `uint256_lt` and call `write_failure_response` before writing any success header: [5](#0-4) 

`execute_secp_get_xy` is the only secp syscall handler that skips this graceful validation pattern.

The `SecpGetXyRequest` struct takes an `ec_point` of type `SecpPoint*`, which contains `BigInt3` x and y fields. A `BigInt3` is three 86-bit limbs, capable of representing values up to ~2^258. The secp256k1 and secp256r1 primes are ~2^256, so a `BigInt3` can trivially encode a value exceeding the prime. A user's contract writes the syscall request directly to the syscall segment and can place any `BigInt3` values there. [6](#0-5) 

The `execute_secp_get_xy` handler is dispatched from `execute_syscalls` for both `SECP256K1_GET_XY_SELECTOR` and `SECP256R1_GET_XY_SELECTOR`: [7](#0-6) 

---

### Impact Explanation

When the Cairo OS program panics, the entire block proof fails. The block cannot be finalized on L1. If the sequencer's off-chain execution engine (Rust) handles the invalid coordinate gracefully and marks the transaction as failed/reverted, but the Cairo OS panics when the prover attempts to prove the same block, the sequencer and prover diverge. The block is unproducible, and the network cannot confirm new transactions — a total network shutdown.

This matches the allowed impact: **High — Network not being able to confirm new transactions (total network shutdown).**

---

### Likelihood Explanation

Any unprivileged user can deploy a contract that calls `secp256k1_get_xy` or `secp256r1_get_xy` with a crafted `SecpPoint` whose `BigInt3` coordinates encode a value `>= curve_prime`. No special role, leaked key, or privileged access is required. The attack requires only a single transaction and is trivially reproducible.

---

### Recommendation

Replace the hard `assert_uint256_lt` assertions with the graceful validation pattern used by all other secp syscall handlers:

1. Switch from `reduce_syscall_gas_and_write_response_header` to `reduce_syscall_base_gas` so the success header is not pre-written.
2. Use `uint256_lt` (non-panicking) to check `x < curve_prime` and `y < curve_prime`.
3. Call `write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT)` if either check fails.
4. Only write the success `ResponseHeader` and the `SecpGetXyResponse` if both coordinates are valid.

This mirrors the pattern in `execute_secp256k1_new` and `execute_secp256k1_get_point_from_x`. [8](#0-7) 

---

### Proof of Concept

1. Deploy a contract with the following logic (pseudocode):
   ```
   // Craft a SecpPoint with x = SECP256K1_PRIME + 1 (encoded as BigInt3)
   let invalid_point = SecpPoint {
       x: BigInt3 { d0: SECP256K1_PRIME_LOW + 1, d1: SECP256K1_PRIME_HIGH, d2: 0 },
       y: BigInt3 { d0: 0, d1: 0, d2: 0 }
   };
   // Call secp256k1_get_xy with the invalid point
   secp256k1_get_xy(invalid_point);
   ```
2. Submit a transaction invoking this contract.
3. The sequencer's Rust executor may handle the invalid coordinate gracefully and include the transaction as failed.
4. When the prover runs the Cairo OS to prove the block, `execute_secp_get_xy` is reached, `bigint_to_uint256` converts the `BigInt3` to a `Uint256 >= SECP256K1_PRIME`, and `assert_uint256_lt` panics.
5. The Cairo OS execution aborts. The block proof cannot be generated. The network halts.

func execute_secp_get_xy{range_check_ptr, syscall_ptr: felt*}(
    curve_prime: Uint256, gas_cost: felt
) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, SecpGetXyRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=gas_cost, request_struct_size=SecpGetXyRequest.SIZE
    );
    if (success == 0) {
        // Not enough gas to execute the syscall.
        return ();
    }

    tempvar ec_point = request.ec_point;
    let (x) = bigint_to_uint256(ec_point.x);
    let (y) = bigint_to_uint256(ec_point.y);

    assert_uint256_lt(x, curve_prime);
    assert_uint256_lt(y, curve_prime);

    assert [cast(syscall_ptr, SecpGetXyResponse*)] = SecpGetXyResponse(x=x, y=y);

    let syscall_ptr = syscall_ptr + SecpGetXyResponse.SIZE;
    return ();
}
