Audit Report

## Title
Stranded ETH in router consumed by subsequent WETH-input swap callers — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary
The `pay` function in `PeripheryPayments` settles WETH-input swaps by reading `address(this).balance` — the router's **total** native ETH balance — rather than the current transaction's `msg.value`. Because all four swap entry points (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`) are `payable` and none issue an automatic ETH refund, any surplus ETH deposited by one caller persists in the router and is silently consumed by the next WETH-input swap caller, who pays nothing.

## Finding Description

`PeripheryPayments.pay` branches on `token == WETH` and reads the router's global balance: [1](#0-0) 

When `nativeBalance >= value`, it wraps exactly `value` ETH and forwards WETH to the pool. The remainder (`address(this).balance − value`) is left in the router with no per-transaction accounting.

All four swap entry points are `payable` and none refund excess ETH after `_clearExpectedCallbackPool()`: [2](#0-1) [3](#0-2) 

The only refund path is `refundETH()`, which must be called explicitly (typically inside a `multicall` bundle): [4](#0-3) 

The callback that invokes `pay` is `_justPayCallback`, which passes `_getPayer()` (the original `msg.sender`) as payer: [5](#0-4) 

When `nativeBalance >= value`, the `safeTransferFrom` branch is never reached, so the attacker's payer address is irrelevant — no token pull from the attacker occurs. The `receive()` guard (`if (msg.sender != WETH) revert NotWETH()`) applies only to plain ETH transfers, not to `msg.value` attached to function calls, so all four `payable` entry points freely accept ETH from any caller. [6](#0-5) 

## Impact Explanation
Direct loss of user principal. A victim who sends excess ETH in a WETH-input `exactOutputSingle` or `exactOutput` call loses the surplus permanently to the next WETH-input swap caller. The attacker receives the full swap output at zero cost. The loss equals `msg.value − actualAmountIn`; in `exactOutput*` flows this can be as large as `amountInMaximum − actualAmountIn`. This meets the Sherlock High/Critical threshold for direct loss of user funds with no special permissions required.

## Likelihood Explanation
Medium-to-High. Users routinely overprovision `msg.value` in `exactOutput*` calls because the exact input amount is unknown at call time. The router's ETH balance is trivially readable on-chain. An attacker only needs to observe a non-zero router ETH balance and submit a WETH-input swap with `msg.value = 0` before the victim calls `refundETH()`. No privileged access, special setup, or non-standard token behavior is required. The attack is repeatable and front-runnable.

## Recommendation
Add an automatic ETH refund at the end of each swap entry point, immediately after `_clearExpectedCallbackPool()`:

```solidity
uint256 leftover = address(this).balance;
if (leftover > 0) _transferETH(msg.sender, leftover);
```

Alternatively, thread the current transaction's `msg.value` as a budget parameter through the call stack and restrict `pay` to consuming only that amount, falling back to `safeTransferFrom` for any shortfall.

## Proof of Concept

1. Alice calls `exactOutputSingle{value: 2000}(...)` with `tokenIn = WETH`, `amountOut = 500`, `amountInMaximum = 2000`.
2. The pool requests 1000 in the callback; `pay` reads `address(this).balance == 2000`, wraps 1000 ETH, sends WETH to the pool. **1000 ETH remains in the router.**
3. Bob calls `exactOutputSingle{value: 0}(...)` with `tokenIn = WETH`, `amountOut = 500`, `amountInMaximum = 1000`.
4. The pool requests 1000 in the callback; `pay` reads `address(this).balance == 1000`, wraps it, sends WETH to the pool.
5. Bob receives 500 output tokens at zero cost. Alice's 1000 ETH is permanently lost.

Foundry test plan: deploy router + mock WETH + mock pool that calls back with a fixed `amountToPay`; have Alice call `exactOutputSingle{value: 2000}` where pool requests 1000; assert router balance is 1000; have Bob call `exactOutputSingle{value: 0}` for the same pool/amount; assert Bob receives output and router balance is 0; assert Alice's 1000 ETH is unrecoverable.
