### Title
Unbounded ETH Transfer to Recipient Contract in `finTransfer` Enables Gas Bomb, Permanently Locking Sender Funds on NEAR — (File: `evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.finTransfer` sends native ETH to the recipient address with **no gas limit**, forwarding all remaining gas to the recipient's fallback function. A malicious recipient contract can implement a gas-exhausting fallback, making it impossible for any relayer to finalize the transfer. Because the NEAR bridge has no cancellation path for a pending outbound transfer, the sender's funds become permanently locked.

---

### Finding Description

In `OmniBridge.sol`, `finTransfer` marks the destination nonce as used and then dispatches the asset to the recipient. For the native-ETH branch:

```solidity
completedTransfers[payload.destinationNonce] = true;   // line 287 — nonce consumed first
...
if (payload.tokenAddress == address(0)) {
    (bool success, ) = payload.recipient.call{value: payload.amount}(""); // line 319 — no gas cap
    if (!success) revert FailedToSendEther();                              // line 322
}
``` [1](#0-0) [2](#0-1) 

The `.call{value: ...}("")` forwards **all remaining gas** to the callee. If `payload.recipient` is a contract whose `receive()` / `fallback()` runs an infinite loop or otherwise exhausts the block gas limit, the call returns `success = false`, the transaction reverts (including the nonce write), and the relayer's attempt fails. Because the gas cost of any attempt is unbounded, no relayer will ever successfully include the transaction, and the transfer is permanently unfinalizeable on EVM.

On the NEAR side, the sender's tokens are locked in the bridge at `init_transfer` time and are only released when `claim_fee` is called after a successful EVM finalization. There is no sender-callable cancellation function visible in the NEAR bridge contract. [3](#0-2) [4](#0-3) 

The same unbounded-gas pattern applies to the ERC-1155 branch, where `safeTransferFrom` triggers `onERC1155Received` on the recipient with no gas cap:

```solidity
IERC1155(multiToken.tokenAddress).safeTransferFrom(
    address(this), payload.recipient, multiToken.tokenId, payload.amount, ""
);
``` [5](#0-4) 

---

### Impact Explanation

A sender on NEAR who bridges native ETH to a malicious EVM contract recipient will have their funds permanently locked in the NEAR bridge. No relayer can finalize the transfer (gas bomb prevents inclusion), and no on-chain cancellation path exists on NEAR to recover the locked tokens. This satisfies **"Permanent freezing / irrecoverable lock of user funds in bridge flows"** (Critical).

---

### Likelihood Explanation

The recipient address is fully attacker-controlled: any party can deploy a gas-bomb contract on EVM and then initiate a NEAR→EVM transfer targeting it. The attack requires no privileged access, no key compromise, and no colluding validators. The only prerequisite is that the attacker controls both the NEAR sender account and the EVM recipient contract — a trivial setup. Likelihood is **Medium-High**.

---

### Recommendation

Cap the gas forwarded to the recipient in the native-ETH branch, analogous to the 2300-gas stipend used by `transfer()`, or use a configurable but bounded gas limit:

```solidity
uint256 constant ETH_TRANSFER_GAS = 2_300;

(bool success, ) = payload.recipient.call{value: payload.amount, gas: ETH_TRANSFER_GAS}("");
if (!success) revert FailedToSendEther();
```

For the ERC-1155 branch, consider wrapping `safeTransferFrom` in a try/catch or using a pull-payment pattern so a malicious `onERC1155Received` cannot block finalization. Additionally, implement a NEAR-side cancellation mechanism (with a timeout) so senders can recover funds if EVM finalization permanently fails.

---

### Proof of Concept

1. **Deploy gas-bomb contract on EVM:**
   ```solidity
   contract GasBomb {
       receive() external payable {
           while (true) {}   // exhausts all forwarded gas
       }
   }
   ```

2. **Initiate transfer on NEAR** targeting `GasBomb` as the EVM recipient, bridging any amount of native ETH equivalent.

3. **Relayer calls `finTransfer`** on `OmniBridge` with `payload.tokenAddress = address(0)` and `payload.recipient = address(GasBomb)`.

4. **Execution**: `GasBomb.receive()` consumes all remaining gas → `success = false` → `revert FailedToSendEther()` → entire transaction reverts, nonce not consumed.

5. **Result**: Every relayer attempt reverts with out-of-gas. No relayer will submit a transaction whose gas cost exceeds the block gas limit. The NEAR-side transfer remains in `pending_transfers` indefinitely with no cancellation path, permanently locking the sender's funds. [6](#0-5) [7](#0-6)
