### Title
EIP-712 Amino Path Silently Drops `timeout_height` from Typed-Data Hash, Enabling Post-Expiry Replay — (`ethereum/eip712/types.go`, `ethereum/eip712/encoding.go`)

---

### Summary

The EIP-712 Amino signing path in Ethermint omits `timeout_height` from the `Tx` type schema used to compute the EIP-712 struct hash. Because EIP-712 only commits to fields declared in the type schema, a transaction signed with a non-zero `timeout_height` produces an identical hash to the same transaction with `timeout_height = 0`. An attacker who observes such a transaction in the mempool can construct a clone with `timeout_height` stripped, submit it after the user's intended expiry block, and have it accepted — the user's signature remains valid.

---

### Finding Description

**Root cause — `timeout_height` removed from the `Tx` type schema**

In `ethereum/eip712/types.go`, `createEIP712Types` hard-codes the `Tx` EIP-712 type. The comment at line 80 explicitly acknowledges the omission:

```go
"Tx": {
    {Name: "account_number", Type: "string"},
    {Name: "chain_id",       Type: "string"},
    {Name: "fee",            Type: "Fee"},
    {Name: "memo",           Type: "string"},
    {Name: "sequence",       Type: "string"},
    // Note timeout_height was removed because it was not getting filled with the legacyTx
},
``` [1](#0-0) 

Because EIP-712 struct-hashing only encodes fields that appear in the declared type, any `timeout_height` value present in the JSON message payload is silently ignored when the hash is computed.

**Amino path does not reject non-zero `timeout_height`**

`decodeProtobufSignDoc` (the Protobuf EIP-712 path) correctly guards against this by rejecting any sign doc whose body carries `TimeoutHeight != 0`:

```go
if body.TimeoutHeight != 0 || body.GetTimeoutTimestamp() != nil || ... {
    return apitypes.TypedData{}, errors.New("body contains unsupported fields: ...")
}
``` [2](#0-1) 

However, `decodeAminoSignDoc` — the Amino EIP-712 path — contains **no equivalent guard**. It unmarshals the sign doc, validates messages, and immediately calls `WrapTxToTypedData` with the raw bytes, never inspecting `aminoDoc.TimeoutHeight`: [3](#0-2) 

The same omission exists in the legacy Amino path `legacyDecodeAminoSignDoc`: [4](#0-3) 

**How the hash is computed**

`WrapTxToTypedData` calls `createEIP712MessagePayload`, which parses the raw Amino JSON (including any `timeout_height` key) and passes it as the message body. `createEIP712Types` then builds the type schema without `timeout_height`. When `apitypes.TypedDataAndHash` encodes the struct, it only iterates over declared type fields — `timeout_height` is silently dropped. [5](#0-4) 

**Attack scenario**

1. A user constructs an Amino EIP-712 Cosmos tx (e.g., a `MsgSend` or governance vote) with `timeout_height = N`, intending it to expire if not included by block N.
2. The user signs the EIP-712 hash. Because `timeout_height` is absent from the type schema, the hash is identical to the hash of the same tx with `timeout_height = 0`.
3. The tx sits in the mempool (e.g., due to congestion or a low gas price).
4. At block N+1, the Cosmos SDK `TxTimeoutHeightDecorator` would reject the original tx.
5. An attacker observing the mempool clones the tx, sets `timeout_height = 0`, and reuses the user's signature verbatim.
6. Signature verification passes (same EIP-712 hash). `TxTimeoutHeightDecorator` passes (no timeout). The tx executes.

The entry point is the publicly reachable `ExtensionOptionsWeb3Tx` ante-handler path: [6](#0-5) 

---

### Impact Explanation

This is a **High** severity issue matching the allowed impact: *"EIP-712 authorization bypass enabling replay or forged execution."*

A user who sets `timeout_height` to bound the validity window of a Cosmos tx signed via EIP-712 Amino receives no protection. An attacker can execute the tx at any future block, potentially:
- Executing a token transfer the user no longer authorizes.
- Casting a governance vote after the user has changed their position.
- Triggering any other state-mutating Cosmos message the user intended to expire.

---

### Likelihood Explanation

The `ExtensionOptionsWeb3Tx` (Amino EIP-712) path is deprecated but still fully supported and reachable by any unprivileged user. Wallets that expose `timeout_height` as a user-facing option (for MEV protection or conditional execution) are the primary affected population. The attack requires only mempool observation and a trivial tx mutation — no privileged access, no key material, no validator collusion.

---

### Recommendation

1. **Reject non-zero `timeout_height` in the Amino path**, mirroring the existing Protobuf guard:

```go
// In decodeAminoSignDoc, after unmarshaling aminoDoc:
if aminoDoc.TimeoutHeight != 0 {
    return apitypes.TypedData{}, errors.New(
        "amino sign doc contains unsupported field: TimeoutHeight")
}
```

Apply the same fix to `legacyDecodeAminoSignDoc`.

2. **Long-term**: Add `timeout_height` to the `Tx` EIP-712 type schema so that the hash commits to it, and remove the rejection guard once the field is properly encoded.

---

### Proof of Concept

```
User signs Amino EIP-712 tx:
  { account_number: "1", chain_id: "ethermint_9000-1", sequence: "5",
    timeout_height: "500", fee: {...}, memo: "", msg0: {MsgSend...} }

EIP-712 Tx type schema (from createEIP712Types):
  Tx = { account_number, chain_id, fee, memo, sequence, msg0 }
  // timeout_height NOT present

Hash(Tx with timeout_height=500) == Hash(Tx with timeout_height=0)
  → user's secp256k1 signature is valid for BOTH

Attacker at block 501:
  1. Copies tx, sets timeout_height = 0
  2. Reuses user's signature
  3. Submits → TxTimeoutHeightDecorator: timeout_height=0, no rejection
  4. Sig verification: EIP-712 hash matches → passes
  5. Tx executes ✓
```

func decodeAminoSignDoc(signDocBytes []byte) (apitypes.TypedData, error) {
	// Ensure codecs have been initialized
	if err := validateCodecInit(); err != nil {
		return apitypes.TypedData{}, err
	}

	var aminoDoc legacytx.StdSignDoc
	if err := aminoCodec.UnmarshalJSON(signDocBytes, &aminoDoc); err != nil {
		return apitypes.TypedData{}, err
	}

	var fees legacytx.StdFee
	if err := aminoCodec.UnmarshalJSON(aminoDoc.Fee, &fees); err != nil {
		return apitypes.TypedData{}, err
	}

	// Validate payload messages
	msgs := make([]sdk.Msg, len(aminoDoc.Msgs))
	for i, jsonMsg := range aminoDoc.Msgs {
		var m sdk.Msg
		if err := aminoCodec.UnmarshalJSON(jsonMsg, &m); err != nil {
			return apitypes.TypedData{}, fmt.Errorf("failed to unmarshal sign doc message: %w", err)
		}
		msgs[i] = m
	}

	if err := validatePayloadMessages(msgs); err != nil {
		return apitypes.TypedData{}, err
	}

	chainID, err := types.ParseChainID(aminoDoc.ChainID)
	if err != nil {
		return apitypes.TypedData{}, errors.New("invalid chain ID passed as argument")
	}

	typedData, err := WrapTxToTypedData(
		chainID.Uint64(),
		signDocBytes,
	)
	if err != nil {
		return apitypes.TypedData{}, fmt.Errorf("could not convert to EIP712 representation: %w", err)
	}

	return typedData, nil

		return apitypes.TypedData{}, err

	// Validate payload messages
	msgs := make([]sdk.Msg, len(aminoDoc.Msgs))
	for i, jsonMsg := range aminoDoc.Msgs {
		var m sdk.Msg

			return apitypes.TypedData{}, fmt.Errorf("failed to unmarshal sign doc message: %w", err)
		}
		msgs[i] = m

		return apitypes.TypedData{}, err

		chainID.Uint64(),

		feeDelegation,

func WrapTxToTypedData(
	chainID uint64,
	data []byte,
) (apitypes.TypedData, error) {
	messagePayload, err := createEIP712MessagePayload(data)
	message := messagePayload.message
	if err != nil {
		return apitypes.TypedData{}, err
	}

	types, err := createEIP712Types(messagePayload)
	if err != nil {
		return apitypes.TypedData{}, err
	}

	value, err := ethermint.SafeInt64(chainID)
	if err != nil {
		return apitypes.TypedData{}, err
	}
	domain := createEIP712Domain(value)

	typedData := apitypes.TypedData{
		Types:       types,
		PrimaryType: txField,
		Domain:      domain,
		Message:     message,
	}

	return typedData, nil
