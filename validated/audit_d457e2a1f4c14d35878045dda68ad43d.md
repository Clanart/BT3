### Title
Missing Declared Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` syscall handler in the StarkNet OS does not validate that the new class hash supplied by the caller corresponds to a previously declared contract class. A contract can call `replace_class` with an arbitrary, undeclared felt value as the class hash. The OS accepts and commits this state transition. The contract's class hash in the global state tree then points to non-existent bytecode, making the contract permanently non-callable and freezing all funds held in its storage.

### Finding Description
In `syscall_impls.cairo`, the `execute_replace_class` function at line 898 contains an explicit developer acknowledgment of the missing check:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
```

After gas deduction, the function unconditionally writes the caller-supplied `class_hash` into the contract's `StateEntry` with no membership check against `contract_class_changes` (the dict of declared classes):

```cairo
let class_hash = request.class_hash;
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}
tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(...);
```

The identical omission exists in the deprecated path at `deprecated_execute_syscalls.cairo` lines 307–329, which also performs no declared-class membership check before writing the new class hash.

When the OS later attempts to execute any entry point on this contract, it calls `find_element` over `compiled_class_facts` keyed by `class_hash`. Because the hash was never declared, no matching fact exists, causing every future call to the contract to fail unconditionally. All token balances and other assets stored in the contract's storage dict are permanently inaccessible.

### Impact Explanation
**Critical — Permanent freezing of funds.**

Any ERC-20 balances, ETH, or other assets stored in the contract's storage become permanently unreachable. No recovery path exists: `replace_class` cannot be called again (the entry point itself is unreachable), and StarkNet has no admin override to forcibly re-assign a class hash. The state commitment produced by the OS will include the invalid class hash, and all subsequent proofs will attest to a state in which the funds are frozen forever.

### Likelihood Explanation
The attack surface is reachable by any unprivileged transaction sender who controls a contract (deployer, contract owner, or any contract whose logic calls `replace_class`). The syscall is available to both Cairo 1 contracts (via `syscall_impls.cairo`) and deprecated Cairo 0 contracts (via `deprecated_execute_syscalls.cairo`). No privileged role, leaked key, or external dependency is required. The developer TODO comment confirms the check was intentionally deferred, meaning the gap is present in the current production code path.

### Recommendation
Before committing the `dict_update` in `execute_replace_class`, assert that the supplied `class_hash` exists as a key in `contract_class_changes` (or in the pre-existing class commitment tree). This mirrors the validation already enforced during `execute_declare_transaction`, where `dict_update` with `prev_value=0` ensures a class is registered before it can be used. The same guard must be applied to both `syscall_impls.cairo` and `deprecated_execute_syscalls.cairo`.

### Proof of Concept

1. Deploy contract `A` that holds a balance of token `T` (e.g., an ERC-20 vault).
2. From within contract `A`, invoke the `replace_class` syscall with `class_hash = 0xdeadbeef` (any value not present in the declared-class tree).
3. The OS executes `execute_replace_class` in `syscall_impls.cairo`. Gas is deducted, the TODO check is skipped, and `StateEntry.class_hash` for contract `A` is set to `0xdeadbeef`.
4. The state diff is committed; the global state root now encodes `A.class_hash = 0xdeadbeef`.
5. Any subsequent transaction targeting contract `A` reaches `deprecated_execute_entry_point` or `execute_entry_point`, which calls `find_element` over `compiled_class_facts` with key `0xdeadbeef`. No fact is found; execution fails unconditionally.
6. All token balances stored under contract `A`'s storage key are permanently frozen.
