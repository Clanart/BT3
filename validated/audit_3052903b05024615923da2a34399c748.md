### Title
Missing Class Hash Validation in `execute_replace_class` Enables Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler does not verify that the replacement class hash corresponds to a declared contract class before committing the state change. This is a direct analog to the external report's missing authorization check: a required validation gate is absent, allowing an invalid state transition to proceed. Any contract can replace its own class hash with an arbitrary undeclared felt value, permanently freezing the contract and all funds it holds.

---

### Finding Description

In `execute_replace_class`, the function reads the new class hash from the syscall request and immediately writes it into `contract_state_changes` without checking that the hash exists in `contract_class_changes` (i.e., that it was previously declared). [1](#0-0) 

The acknowledged TODO at line 898 confirms the missing gate:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [2](#0-1) 

Once the state is committed with an undeclared class hash `H`, any future call to the contract enters `execute_entry_point`, which performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // = H, undeclared
);
// compiled_class_hash == 0 (default for missing key)

let (compiled_class_fact: CompiledClassFact*) = find_element(
    ...
    key=compiled_class_hash,           // key = 0
);
``` [3](#0-2) 

Since no compiled class with hash `0` exists (the declare path enforces `assert_not_zero(compiled_class_hash)`), `find_element` cannot satisfy its assertion, making any proof that includes a call to the frozen contract invalid. The sequencer is permanently unable to include calls to that contract in any valid block. [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

A malicious contract deployer can:
1. Deploy a contract that accepts user deposits (tokens, ETH-bridged assets, etc.).
2. Attract user deposits.
3. Issue a `replace_class` syscall with an arbitrary undeclared hash.
4. The state is committed with the invalid class hash in the same block (proof is valid because the contract is not called again in that block).
5. From the next block onward, every attempt to call the contract produces an unprovable execution trace.
6. All funds locked in the contract are permanently irrecoverable.

---

### Likelihood Explanation

**Medium.** Any unprivileged user can deploy a contract on StarkNet. The `replace_class` syscall is callable by the contract itself during any execution. No privileged role, leaked key, or external dependency is required. The attacker only needs to attract deposits before triggering the freeze — a realistic scenario in DeFi (token contracts, vaults, bridges). The attack is one transaction and irreversible.

---

### Recommendation

Inside `execute_replace_class`, before writing the new class hash to `contract_state_changes`, add a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert the result is non-zero:

```cairo
// Verify the replacement class is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the validation already present in `execute_entry_point` and closes the gap identified by the TODO comment. [5](#0-4) 

---

### Proof of Concept

1. Attacker deploys contract `C` (class `K`) with a `freeze()` function that calls `replace_class(0xdeadbeef)`.
2. Users deposit funds into `C`.
3. Attacker calls `freeze()`. `execute_replace_class` writes `C.class_hash = 0xdeadbeef` into `contract_state_changes` with no validation.
4. Block proof is generated and accepted — `C` is not called again in this block, so `execute_entry_point` is never reached for `C`.
5. State root is updated: `C.class_hash = 0xdeadbeef` is now canonical on-chain.
6. In block N+1, any user attempts to call `C`. `execute_entry_point` reads `contract_class_changes[0xdeadbeef]` → `0`. `find_element(..., key=0)` cannot find a compiled class; the Cairo assertion fails; the proof is unprovable.
7. The sequencer cannot include any call to `C` in any future valid block. All funds in `C` are permanently frozen. [6](#0-5) [7](#0-6)

    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

        key=contract_address,

    );

    return ();
}
