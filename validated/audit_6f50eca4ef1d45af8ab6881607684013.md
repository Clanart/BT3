### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Bricking — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

`execute_replace_class` updates a contract's `class_hash` in the OS state without verifying that the supplied hash is actually declared in `contract_class_changes`. A developer-acknowledged TODO comment confirms the check is intentionally absent. Any contract can therefore replace its own class with an arbitrary, undeclared hash, rendering itself permanently non-executable and freezing all funds it holds.

---

### Finding Description

The M-03 vulnerability class is **state-transition bypass**: a function mutates one piece of state (roles revoked) while leaving a correlated tracking variable (`collateralActive`) unchanged, producing an inconsistent state that permanently blocks a future transition.

The direct analog here is `execute_replace_class`:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The function writes the caller-supplied `class_hash` directly into the contract's `StateEntry` without consulting `contract_class_changes`. Compare this with `execute_declare_transaction`, which enforces that a class can only be declared once and that `compiled_class_hash != 0`:

```cairo
// Note that prev_value=0 enforces that a class may be declared only once.
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [2](#0-1) 

The two dictionaries — `contract_state_changes` (per-contract state, including `class_hash`) and `contract_class_changes` (the class registry) — are supposed to be kept consistent: a contract's `class_hash` must always refer to a declared entry in the class registry. `execute_replace_class` breaks this invariant by updating `contract_state_changes` without checking `contract_class_changes`.

The `StateEntry` struct that is written:

```cairo
struct StateEntry {
    class_hash: felt,
    storage_ptr: DictAccess*,
    nonce: felt,
}
``` [3](#0-2) 

After the replacement, the contract's `class_hash` field points to a hash that has no corresponding entry in the class tree. Every subsequent call to the contract will fail at execution time because the class bytecode does not exist, and the OS will have already committed the invalid state to the global root via `compute_contract_state_commitment`. [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` is set to an undeclared value, no entry point of that contract can ever be executed again. Any ERC-20 balances, ETH, or other assets stored in the contract's storage are permanently inaccessible. The OS produces a valid proof for the block containing the bad `replace_class` call, so the corrupted state is finalized on L1 with no recourse.

A concrete scenario: a deployer launches a shared vault or DEX contract, attracts user deposits, then issues a `replace_class` syscall with a random undeclared hash. All deposited funds are frozen permanently.

---

### Likelihood Explanation

The `replace_class` syscall is callable by any contract on itself — no privileged role is required. The missing check is explicitly flagged as a TODO, confirming it is not a deliberate design choice but an unimplemented guard. Any contract whose upgrade path (or any reachable code path) invokes `replace_class` with an invalid hash — whether maliciously or due to a bug — triggers the freeze. The attack surface is every upgradeable contract on the network.

---

### Recommendation

Inside `execute_replace_class`, after reducing gas and before writing the new `StateEntry`, assert that `class_hash` is present in `contract_class_changes` with a non-zero `compiled_class_hash`. Concretely:

1. Perform a `dict_read` on `contract_class_changes` keyed by `class_hash`.
2. Assert the returned `compiled_class_hash != UNINITIALIZED_CLASS_HASH`.

This mirrors the invariant already enforced by `execute_declare_transaction` and closes the gap between the two dictionaries.

---

### Proof of Concept

1. Declare class `C` (compiled hash `H`) via a normal `Declare` transaction — `contract_class_changes[C] = H`.
2. Deploy contract `Vault` with class `C`; users deposit funds into `Vault`'s storage.
3. `Vault` (or an attacker who controls it) calls the `replace_class` syscall with `class_hash = 0xdeadbeef` (any undeclared felt).
4. `execute_replace_class` writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes` for `Vault`'s address. No check against `contract_class_changes` is performed.
5. The block is proven and finalized. `Vault`'s committed state now has `class_hash = 0xdeadbeef`.
6. Any subsequent `call_contract` to `Vault` reads `state_entry.class_hash = 0xdeadbeef`, attempts to load the class, and fails — permanently. All deposited funds are frozen. [5](#0-4)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
