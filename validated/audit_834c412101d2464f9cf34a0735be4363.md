### Title
`types.GetBaseFee` Returns Non-Nil Zero When `NoBaseFee=true`, Bypassing Validator Min-Gas-Price Enforcement in `NewDynamicFeeChecker` — (`ante/evm/fee_checker.go`, `x/evm/types/utils.go`)

---

### Summary

There is a divergence in nil-return semantics between `types.GetBaseFee` (`x/evm/types/utils.go`) and `keeper.GetBaseFee` (`x/feemarket/keeper/params.go`) when `NoBaseFee=true`. The ante handler's `NewDynamicFeeChecker` relies on a nil return from `types.GetBaseFee` to fall back to validator min-gas-price logic, but `types.GetBaseFee` converts the nil it receives from `feemarketParams.GetBaseFee()` into `new(big.Int)` (a non-nil zero). This causes the EIP-1559 fee path to execute with `baseFee=0`, allowing Cosmos txs with zero fees to pass the ante handler and commit, bypassing the intended min-gas-price enforcement.

---

```go
func (p Params) GetBaseFee() *big.Int {
    if p.NoBaseFee {
        return nil   // correctly signals "disabled"
    }
    return p.BaseFee.BigInt()
}

**`types.GetBaseFee()`** (`x/evm/types/utils.go:244`):
```go
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    if !IsLondon(ethCfg, height) {
        return nil
    }
    baseFee := feemarketParams.GetBaseFee()
    // should not be nil if london hardfork enabled  ← WRONG when NoBaseFee=true
    if baseFee == nil {
        return new(big.Int)   // BUG: returns zero (non-nil) instead of nil
    }
    return baseFee
}
``` [3](#0-2) 

When `NoBaseFee=true` and London hardfork is active:
- `feemarketParams.GetBaseFee()` → `nil`
- `types.GetBaseFee` sees `nil` and returns `new(big.Int)` (zero, **non-nil**)

**`NewDynamicFeeChecker`** (`ante/evm/fee_checker.go:56`):
```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
if baseFee == nil {
    // london hardfork is not enabled: fallback to min-gas-prices logic
    return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
}
``` [4](#0-3) 

Because `baseFee` is `new(big.Int)` (zero, non-nil), the nil guard is **never triggered**. The function proceeds into EIP-1559 logic with `baseFeeInt = 0`.

The fee check then becomes:
```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))   // = 0 if fee=0
if feeCap.LT(baseFeeInt) { ... }                   // 0 < 0 → false, passes
``` [5](#0-4) 

A Cosmos tx with `fee=0` produces `feeCap=0`, which satisfies `0 >= 0`. The effective fee returned is `0 * gas = 0`, so zero fees are deducted from the sender's account.

---

### Impact Explanation

When a chain is configured with `NoBaseFee=true` (fee market disabled) but London hardfork is active, any unprivileged Cosmos tx submitter can:

1. Submit a Cosmos tx with `fee=0`.
2. The ante handler's `NewDynamicFeeChecker` applies EIP-1559 logic with `baseFee=0` instead of falling back to `checkTxFeeWithValidatorMinGasPrices`.
3. The fee check passes (`0 >= 0`).
4. `effectiveFee = 0` is returned and deducted — the tx commits with zero fees paid.
5. Validator min-gas-price enforcement (the intended protection when `NoBaseFee=true`) is completely bypassed.

This is a **fee mis-accounting** bug: the ante handler charges users less than the chain's configured minimum, and allows zero-fee transactions to commit when the operator intended min-gas-prices to be enforced.

---

### Likelihood Explanation

`NoBaseFee=true` is a documented, production-supported configuration (e.g., for chains that want fixed/zero fees or validator-controlled pricing). London hardfork is enabled by default in Ethermint chain configs. Any chain running both simultaneously is affected. The attacker needs no privileges — only the ability to submit a standard Cosmos tx with `fee=0`.

---

### Recommendation

In `types.GetBaseFee` (`x/evm/types/utils.go`), when `feemarketParams.GetBaseFee()` returns `nil`, the function must also return `nil` rather than `new(big.Int)`. The comment "should not be nil if london hardfork enabled" is incorrect — `NoBaseFee=true` is a valid reason for a nil base fee even when London is active.

```go
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    if !IsLondon(ethCfg, height) {
        return nil
    }
    // Return nil when NoBaseFee=true so callers fall back to min-gas-price logic
    return feemarketParams.GetBaseFee()
}
```

### Proof of Concept

```
Setup:
  feemarketParams.NoBaseFee = true
  ethCfg.LondonBlock = 0  (London active from genesis)
  ctx.BlockHeight() = 100

Call types.GetBaseFee(100, ethCfg, &feemarketParams):
  → IsLondon = true (does not return nil)
  → feemarketParams.GetBaseFee() = nil  (NoBaseFee=true)
  → baseFee == nil → return new(big.Int)  ← returns 0, non-nil

Call keeper.GetBaseFee(ctx):
  → params.NoBaseFee = true → return nil  ← returns nil

In NewDynamicFeeChecker:
  baseFee = new(big.Int)  (non-nil zero)
  baseFee == nil → FALSE  → EIP-1559 path taken
  feeCap = 0 / gas = 0
  0 < 0 → FALSE → fee check passes
  effectiveFee = 0 * gas = 0
  → Tx commits with zero fees deducted
  → Validator min-gas-price check (checkTxFeeWithValidatorMinGasPrices) never called
```
