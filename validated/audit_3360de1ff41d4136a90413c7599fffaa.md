### Title
EIP-2935 Ring Buffer Returns Stale Block Hash for Out-of-Range Heights Without Epoch Validation — (`File: x/evm/keeper/state_transition.go`, `x/evm/keeper/keeper.go`)

---

### Summary

`GetHashFn` in `x/evm/keeper/state_transition.go` calls `GetHeaderHash` for **any** historical height without first applying the valid-range lower-bound check. `GetHeaderHash` reads from the EIP-2935 ring buffer using `height % window` as the slot key, but stores no epoch tag — it cannot distinguish whether the stored hash belongs to the requested height or to a newer block that has already overwritten the same slot. After `window` (8191) blocks, the slot for height `h` is silently overwritten by height `h + 8191`. Because the range guard is only applied to the staking-fallback branch, the BLOCKHASH opcode returns a non-zero, wrong block hash for heights that are outside the valid 256-block window, violating the EVM specification.

---

### Finding Description

`GetHashFn` computes a lower bound `lower = upper - headerHashNum` (default: `upper - 256`) to enforce the EVM rule that BLOCKHASH returns `0` for blocks older than 256:

```go
// x/evm/keeper/state_transition.go  lines 114-140
var lower uint64
if upper <= headerHashNum {
    lower = 0
} else {
    lower = upper - headerHashNum
}

if upper > num64 {
    // EIP-2935 ring buffer — NO range check here
    headerHash := k.GetHeaderHash(ctx, num64)
    if headerHash.Cmp(common.Hash{}) != 0 {
        return headerHash          // ← returned without verifying num64 >= lower
    } else if num64 >= lower {    // ← range check only on the staking fallback
        ...
    }
}
```

`GetHeaderHash` reads the ring buffer slot `height % window` (window = 8191 by default):

```go
// x/evm/keeper/keeper.go  lines 384-391
ringIndex := height % window
var key common.Hash
binary.BigEndian.PutUint64(key[24:], ringIndex)
hash := k.GetState(ctx, ethparams.HistoryStorageAddress, key)
if hash.Cmp(common.Hash{}) != 0 {
    return hash   // ← no epoch/height tag stored; cannot verify ownership
}
```

No height tag is stored alongside the hash. After 8191 blocks, slot `h % 8191` is overwritten by block `h + 8191`. When a contract calls `BLOCKHASH(h)` where `h < upper - 8191`, `GetHeaderHash` reads the slot and finds the hash of block `h + 8191` (or `h + 2×8191`, etc.) — a non-zero value — and `GetHashFn` returns it without the range check.

The analog to the Aloe bug is exact: the circular buffer has no "initialized/epoch" field, so a stale slot written by a newer block is indistinguishable from a valid entry for the requested height.

---

### Impact Explanation

The BLOCKHASH opcode returns a **non-zero, incorrect hash** for block heights outside the valid 256-block window. The EVM specification mandates `0` for such heights. Smart contracts that rely on `blockhash(n) == 0` to detect out-of-range blocks (e.g., commit-reveal schemes, randomness guards, time-lock checks) will receive a plausible-looking but wrong hash, causing incorrect branching or security bypasses within EVM execution. This is an EVM state transition bug that feeds incorrect data into on-chain contract logic. The same `GetHashFn` is used by `eth_call`, `eth_estimateGas`, and `eth_simulateV1`, so the incorrect value also propagates through the JSON-RPC simulation path.

---

### Likelihood Explanation

The condition is automatically satisfied after the chain has been running for `window = 8191` blocks (roughly a few hours at 1-second block times). No special setup, governance action, or privileged role is required. Any unprivileged EVM transaction containing `BLOCKHASH(n)` where `n < current_height - 8191` will trigger the bug. The attacker only needs to deploy or call a contract that reads an old block hash.

---

### Recommendation

Apply the lower-bound range check **before** querying the ring buffer, not only in the staking fallback branch:

```go
if upper > num64 && num64 >= lower {
    headerHash := k.GetHeaderHash(ctx, num64)
    if headerHash.Cmp(common.Hash{}) != 0 {
        return headerHash
    }
    // staking fallback (pre-upgrade)
    histInfo, err := k.stakingKeeper.GetHistoricalInfo(ctx, h)
    ...
}
return common.Hash{}
```

Alternatively, store the block height alongside the hash in the ring buffer slot (as a two-word entry) so `GetHeaderHash` can verify the slot belongs to the requested height before returning it.

---

### Proof of Concept

1. Chain starts; EIP-2935 preinstall contract is deployed at genesis.
2. Chain runs for 8192+ blocks. At block `N = 8192`, `SetHeaderHash` writes block `N`'s hash to slot `8192 % 8191 = 1`, overwriting block `1`'s hash.
3. A contract executes `BLOCKHASH(1)` at block `N = 8192`.
4. `GetHashFn` computes `lower = 8192 - 256 = 7936`. Height `1 < 7936`, so it is out of range and should return `0`.
5. `GetHashFn` calls `GetHeaderHash(ctx, 1)`. Ring index = `1 % 8191 = 1`. The slot contains block `8192`'s hash (non-zero).
6. `GetHeaderHash` returns block `8192`'s hash. `GetHashFn` returns it **without** the `num64 >= lower` check.
7. The contract receives a non-zero hash for block `1` instead of `0`, violating the EVM spec. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

```go
// GetHeaderHash sets block hash into EIP-2935 compatible storage contract.
func (k Keeper) GetHeaderHash(ctx sdk.Context, height uint64) common.Hash {
	// check if history contract has been deployed
	acct := k.GetAccount(ctx, ethparams.HistoryStorageAddress)
	if acct != nil && acct.IsContract() {
		window := types.DefaultHistoryServeWindow
		params := k.GetParams(ctx)
		if params.HistoryServeWindow > 0 {
			window = params.HistoryServeWindow
		}

		ringIndex := height % window
		var key common.Hash
		binary.BigEndian.PutUint64(key[24:], ringIndex)
		hash := k.GetState(ctx, ethparams.HistoryStorageAddress, key)

		if hash.Cmp(common.Hash{}) != 0 {
			return hash
		}
	}
	// fall back to old behavior for retro compatibility
	// TODO can be removed along with DeleteHeaderHash once HistoryStorage has been filled up in next protocol upgrade
	store := ctx.KVStore(k.storeKey)
	hashByte := store.Get(types.GetHeaderHashKey(height))
	if len(hashByte) > 0 {
		return common.BytesToHash(hashByte)
	}
	return common.Hash{}
}
```

```go
func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	k.WithChainID(ctx)

	// cache parameters that's common for the whole block.
	cfg, err := k.EVMBlockConfig(ctx, k.ChainID())

		return err
	}

	headerHashNum, err := ethermint.SafeInt64(cfg.Params.GetHeaderHashNum())
	if err != nil {
		panic(err)
	}
	if i := ctx.BlockHeight() - headerHashNum; i > 0 {
		h, err := ethermint.SafeUint64(i)
		if err != nil {
			panic(err)
		}
		k.DeleteHeaderHash(ctx, h)
	}
	return nil
```
