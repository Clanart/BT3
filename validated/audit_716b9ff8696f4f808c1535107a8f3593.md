### Title
Missing Nonce in `meta_tx_v0` Hash Enables Unbounded Signature Replay — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_meta_tx_v0` syscall in the StarkNet OS creates a version-0 signed execution context with no nonce included in the transaction hash and no nonce tracking at the OS level. This means any signed `meta_tx_v0` message can be replayed an unlimited number of times by any party who has observed the signature, with no protocol-level mechanism to cancel or invalidate it.

---

### Finding Description

The `execute_meta_tx_v0` function constructs a new `TxInfo` for the target contract with `version=0` and `nonce=0` hardcoded: [1](#0-0) 

The hash for this meta-transaction is computed by `compute_meta_tx_v0_hash`, which calls `deprecated_get_transaction_hash` with `additional_data_size=0` — meaning **no nonce is included in the hash**: [2](#0-1) 

Compare this to `compute_l1_handler_transaction_hash`, which correctly passes `additional_data_size=1` with the nonce as additional data: [3](#0-2) 

Furthermore, `check_and_increment_nonce` explicitly skips nonce enforcement for version-0 transactions: [4](#0-3) 

And `run_validate` (which calls `__validate__`) is also skipped for version 0: [5](#0-4) 

The result is a three-layer absence of replay protection for `meta_tx_v0`:
1. No nonce in the signed hash.
2. No OS-level nonce increment.
3. No `__validate__` call to enforce account-level checks.

The OS provides no mechanism for a user to cancel or invalidate a previously signed `meta_tx_v0` message.

---

### Impact Explanation

**Critical — Direct loss of funds.**

A user who signs a `meta_tx_v0` message (e.g., authorizing a token transfer) produces a signature that is permanently valid. Any party who observes this signature on-chain can replay it in any subsequent outer transaction by calling the `meta_tx_v0` syscall with the same `contract_address`, `calldata`, and `signature`. Since the hash is deterministic and nonce-free, the signature verifies identically every time. If the target account contract does not implement its own nonce tracking (which the OS does not require or enforce), the same authorized action — such as a token transfer — executes repeatedly, draining the account.

---

### Likelihood Explanation

**High.** The `meta_tx_v0` syscall is callable by any contract during any transaction execution. All signatures for `meta_tx_v0` calls are visible on-chain once the outer transaction is included in a block. Any unprivileged actor can extract the signature from the transaction calldata and replay it. No special access or key compromise is required — only observation of a previously executed transaction.

---

### Recommendation

Include a nonce in `compute_meta_tx_v0_hash` analogously to how `compute_l1_handler_transaction_hash` includes the L1 nonce as `additional_data`. The OS should track and increment a per-account nonce for `meta_tx_v0` executions, or at minimum require the caller to supply a nonce that is committed into the hash and validated against on-chain state before execution proceeds.

---

### Proof of Concept

1. User signs a `meta_tx_v0` message authorizing `transfer(recipient, 1000)` on a token contract. The signed hash is `H = Pedersen(INVOKE_PREFIX, 0, contract_address, __execute__, calldata_hash, 0, chain_id)` — no nonce.
2. Relayer submits outer transaction (v3, nonce=N) calling a relay contract that invokes the `meta_tx_v0` syscall with the user's signature. Tokens are transferred.
3. Attacker observes the outer transaction on-chain, extracts `(contract_address, calldata, signature)` from the syscall data.
4. Attacker deploys a contract that calls `meta_tx_v0` with the same parameters and submits their own outer transaction (v3, nonce=M).
5. The OS computes the identical hash `H`, the account contract's `__execute__` receives the same `TxInfo` with the same signature, verifies it successfully, and executes the transfer again.
6. Step 4–5 repeats until the account is drained. No cancellation path exists at the OS level. [6](#0-5) [7](#0-6)

    let request = cast(syscall_ptr + RequestHeader.SIZE, MetaTxV0Request*);
    local calldata_start: felt* = request.calldata_start;
    local calldata_size = request.calldata_end - calldata_start;

    let specific_base_gas_cost = (
        META_TX_V0_GAS_COST + META_TX_V0_CALLDATA_FACTOR_GAS_COST * calldata_size
    );
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=specific_base_gas_cost, request_struct_size=MetaTxV0Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    local contract_address = request.contract_address;
    local selector = request.selector;
    local caller_execution_info: ExecutionInfo* = caller_execution_context.execution_info;
    local old_tx_info: TxInfo* = caller_execution_info.tx_info;

    if (selector != EXECUTE_ENTRY_POINT_SELECTOR) {
        write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
        return ();
    }

    // Sanity check: Verify that `signature` is a valid Sierra array.
    assert_nn_le(request.signature_end - request.signature_start, SIERRA_ARRAY_LEN_BOUND - 1);

    let (state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
        key=contract_address
    );

    // Compute the meta-transaction hash.
    let pedersen_ptr = builtin_ptrs.selectable.pedersen;
    with pedersen_ptr {
        let meta_tx_hash = compute_meta_tx_v0_hash(
            contract_address=contract_address,
            entry_point_selector=selector,
            calldata=calldata_start,
            calldata_size=calldata_size,
            chain_id=old_tx_info.chain_id,
        );
    }
    update_pedersen_in_builtin_ptrs(pedersen_ptr=pedersen_ptr);

    // Prepare execution context.
    tempvar new_tx_info = new TxInfo(
        version=0,
        account_contract_address=contract_address,
        max_fee=0,
        signature_start=request.signature_start,
        signature_end=request.signature_end,
        transaction_hash=meta_tx_hash,
        chain_id=old_tx_info.chain_id,
        nonce=0,
        resource_bounds_start=cast(0, ResourceBounds*),
        resource_bounds_end=cast(0, ResourceBounds*),
        tip=0,
        paymaster_data_start=cast(0, felt*),
        paymaster_data_end=cast(0, felt*),
        nonce_data_availability_mode=0,
        fee_data_availability_mode=0,
        account_deployment_data_start=cast(0, felt*),
        account_deployment_data_end=cast(0, felt*),
        proof_facts_start=cast(0, felt*),
        proof_facts_end=cast(0, felt*),
    );

    let (deprecated_tx_info_ptr: DeprecatedTxInfo*) = alloc();
    tempvar execution_context: ExecutionContext* = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=state_entry.class_hash,
        calldata_size=calldata_size,
        calldata=calldata_start,
        execution_info=new ExecutionInfo(
            block_info=caller_execution_info.block_info,
            tx_info=new_tx_info,
            caller_address=ORIGIN_ADDRESS,
            contract_address=contract_address,
            selector=selector,

        deprecated_tx_info=deprecated_tx_info_ptr,
    );
    fill_deprecated_tx_info(tx_info=new_tx_info, dst=execution_context.deprecated_tx_info);

    // Since we process the revert log backwards, entries before this point belong to the calling

    let revert_log = &revert_log[1];

        block_context=block_context,
        execution_context=execution_context,
    );

    // Entries before this point belong to the callee.
    assert [revert_log] = RevertLogEntry(selector=CHANGE_CONTRACT_ENTRY, value=contract_address);
    let revert_log = &revert_log[1];

    return ();
