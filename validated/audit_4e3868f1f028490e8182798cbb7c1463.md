### Title
P2P Sync Stores Empty `BlockBody` for Non-Empty Block via Malicious Header with `n_transactions=0` — (`crates/apollo_p2p_sync/src/client/transaction.rs`)

---

### Summary

`TransactionStreamFactory::parse_data_for_block` uses `header.n_transactions` as the sole loop bound for collecting transactions from a p2p peer. The header sync path (`HeaderStreamBuilder::parse_data_for_block`) performs no block-hash recomputation and no sequencer-signature verification before storing the header. A malicious peer that supplies `n_transactions=0` causes the transaction loop to be skipped entirely, writing an empty `BlockBody` for a block whose `transaction_commitment` encodes a non-empty Merkle tree.

---

### Finding Description

**Step 1 — Header accepted without commitment binding.**

`HeaderStreamBuilder::parse_data_for_block` applies exactly two checks to a peer-supplied `SignedBlockHeader`:

```
if block_number != signed_block_header.block_header.block_header_without_hash.block_number { … }
if signed_block_header.signatures.len() != ALLOWED_SIGNATURES_LENGTH { … }
``` [1](#0-0) 

Neither `calculate_block_hash` nor `verify_block_signature` is called anywhere in `crates/apollo_p2p_sync/`. A grep for both symbols returns zero matches. The block hash — which commits to `concatenated_counts` (encoding `n_transactions`) and `transaction_commitment` — is never recomputed or verified against the sequencer's public key.

The TODO at line 140 of `header.rs` explicitly acknowledges the missing cross-check:

```rust
// TODO(Shahak): Verify `n_transactions` and `state_diff_length` match values in
// concatenated_counts.
``` [2](#0-1) 

`append_header` stores the peer-supplied header verbatim, including the attacker-controlled `n_transactions` field: [3](#0-2) 

**Step 2 — Transaction loop bound derived from the poisoned field.**

`TransactionStreamFactory::parse_data_for_block` reads `target_transaction_len` directly from the stored header:

```rust
let target_transaction_len = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .expect("A header with number lower than the header marker is missing")
    .n_transactions;
while current_transaction_len < target_transaction_len {   // 0 < 0 → false
    …
}
Ok(Some((block_body, block_number)))   // block_body is BlockBody::default() = empty
``` [4](#0-3) 

When `n_transactions=0` the while condition is immediately false; the function returns `Ok(Some((BlockBody::default(), N)))` without consuming a single transaction from the peer stream.

**Step 3 — Empty body committed to storage.**

`write_to_storage` calls `append_body(N, empty_body)` unconditionally: [5](#0-4) 

The stored header for block N retains the peer-supplied `transaction_commitment` (which may be non-zero), while the stored body contains zero transactions. The invariant `|stored_transactions| == n_transactions == len(transaction_commitment_tree)` is broken.

**Block-hash binding does not rescue this.** The block hash chains `concatenated_counts` (which packs `n_transactions`) and `transaction_commitment` as independent inputs: [6](#0-5) 

Because neither the block hash nor the sequencer signature is verified in the p2p sync path, the attacker is free to set `n_transactions=0` with any `transaction_commitment` value and any fake signature.

---

### Impact Explanation

| Consumer | Effect |
|---|---|
| `starknet_getBlockWithTxs` / `starknet_getTransactionByBlockIdAndIndex` | Returns empty transaction list for block N |
| Execution replay / blockifier re-execution | Replays block N with no transactions; produces wrong post-state root |
| SNOS / proof generation | Receives empty body as input; `transaction_commitment` in the header disagrees with the computed commitment over zero leaves; proof is invalid or unprovable |
| `starknet_getBlockTransactionCount` | Returns 0 for a non-empty block |

The corrupted committed value is concrete: `BlockBody { transactions: [], transaction_outputs: [], transaction_hashes: [] }` stored for block N while `header.transaction_commitment` encodes a non-empty Poseidon tree.

---

### Likelihood Explanation

Any node that can establish a p2p connection and respond to a `BlockHeadersRequest` can trigger this. No sequencer key, operator privilege, or stake is required. The attack is a single malformed response message.

---

### Recommendation

1. **Verify the block hash** before storing a peer-supplied header: recompute `calculate_block_hash` from the received fields and compare against `block_header.block_hash`.
2. **Verify the sequencer signature** (`verify_block_signature`) against a trusted sequencer public key before accepting any `SignedBlockHeader`.
3. **Cross-validate `n_transactions`** against `concatenated_counts` extracted from the stored `BlockHeaderCommitments` (the TODO at `header.rs:140` tracks this).
4. As a defence-in-depth measure, after storing the body, assert `stored_body.transactions.len() == header.n_transactions`.

---

### Proof of Concept

```rust
// Pseudocode — store a header with n_transactions=0 but non-zero transaction_commitment,
// run parse_data_for_block, assert empty body is stored.
let header = BlockHeader {
    block_hash: BlockHash(Felt::ONE),          // arbitrary; not verified
    block_header_without_hash: BlockHeaderWithoutHash {
        block_number: BlockNumber(0),
        ..Default::default()
    },
    transaction_commitment: Some(TransactionCommitment(Felt::from(0xdeadbeef_u64))),
    n_transactions: 0,   // ← attacker-controlled
    state_diff_length: Some(0),
    ..Default::default()
};
storage_writer.begin_rw_txn().unwrap()
    .append_header(BlockNumber(0), &header).unwrap()
    .commit().unwrap();

// parse_data_for_block reads n_transactions=0, loop never executes,
// returns Ok(Some((BlockBody::default(), BlockNumber(0)))).
// write_to_storage stores the empty body.

let txn = storage_reader.begin_ro_txn().unwrap();
let body = txn.get_block_transactions(BlockNumber(0)).unwrap().unwrap();
assert!(body.is_empty());   // passes — commitment invariant violated
let stored_header = txn.get_block_header(BlockNumber(0)).unwrap().unwrap();
assert_eq!(stored_header.transaction_commitment,
           Some(TransactionCommitment(Felt::from(0xdeadbeef_u64))));  // non-zero root
```

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,

    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)

            .get_poseidon_hash(),
