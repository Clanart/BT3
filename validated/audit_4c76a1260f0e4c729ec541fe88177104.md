### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the supplied `class_hash` corresponds to a previously declared contract class. A contract can replace its own class hash with an arbitrary, undeclared value. Once committed to state, any future call to that contract will fail at the OS level because the execution engine cannot resolve the undeclared class hash to a compiled class, permanently freezing any funds held by the contract.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the requested `class_hash` from the syscall request and immediately writes it into `contract_state_changes` without checking whether that hash exists in `contract_class_changes` (the registry of declared classes):

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment at line 898 explicitly acknowledges this missing check. [1](#0-0) 

The same omission exists in the deprecated syscall path: [2](#0-1) 

When a contract is subsequently called, `execute_entry_point` performs a `dict_read` on `contract_class_changes` using the stored `class_hash` to obtain the compiled class hash:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
```

If the class hash was never declared, this returns `0` (the uninitialized default). The subsequent `find_element` call then searches for a compiled class with hash `0`, which does not exist, causing an assertion failure at the OS level. [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` in `contract_state_changes` is set to an undeclared value and the block is proven and finalized on L1, the state is irrevocably committed. Every subsequent transaction that attempts to call the affected contract will fail at the OS execution layer (not a graceful revert — an unresolvable class lookup). The sequencer must permanently exclude all calls to that contract. Any ERC-20 tokens, ETH, or other assets held in the contract's storage become permanently inaccessible. [4](#0-3) 

---

### Likelihood Explanation

The `replace_class` syscall is callable by any deployed contract — it requires no privileged role. A malicious contract deployer can deploy a contract that immediately calls `replace_class` with an arbitrary felt value (e.g., `1`) that has never been declared. This is a single transaction, requires no special permissions, and is reachable by any unprivileged transaction sender. The attack is deterministic and requires no race condition or external dependency.

---

### Recommendation

Before updating `contract_state_changes`, verify that the supplied `class_hash` exists in `contract_class_changes` (i.e., has a non-zero compiled class hash entry). This is exactly what the existing TODO comment prescribes:

```cairo
// Verify the class is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

Apply the same fix to `execute_replace_class` in `deprecated_execute_syscalls.cairo`. [5](#0-4) 

---

### Proof of Concept

1. Deploy a contract `MaliciousContract` whose constructor (or any callable function) executes the `replace_class` syscall with `class_hash = 0xdeadbeef` (an arbitrary value never declared on-chain).
2. The OS processes the transaction. `execute_replace_class` in `syscall_impls.cairo` line 906 writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes` with no validation.
3. The block is proven and the state root is updated on L1. `MaliciousContract`'s class hash is now `0xdeadbeef` in the committed state.
4. Any subsequent transaction calling `MaliciousContract` reaches `execute_entry_point`. At line 154, `dict_read(key=0xdeadbeef)` returns `0` (undeclared). At line 161, `find_element(..., key=0)` fails — no compiled class with hash `0` exists.
5. The OS cannot produce a valid proof for any block containing a call to `MaliciousContract`. All funds in `MaliciousContract` are permanently frozen. [6](#0-5) [7](#0-6)
