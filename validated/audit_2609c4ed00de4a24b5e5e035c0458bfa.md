### Title
Missing Duplicate-Key/Hash Validation in `batch_insert` Corrupts DataLayer Merkle Tree Root — (`File: crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::batch_insert` skips the duplicate-key and duplicate-hash guards that `MerkleBlob::insert` enforces for every item beyond the first two in a batch. Supplying a batch of three or more entries that contains a repeated `KeyId` or `Hash` silently writes multiple leaf nodes with the same key/hash into the blob, corrupting the committed Merkle root and enabling forged proofs of inclusion.

---

### Finding Description

`MerkleBlob::insert` enforces two invariants before writing any leaf:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 369-374
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`batch_insert` routes only the first two items through `insert` (the bootstrap path for an empty or single-leaf tree). Every subsequent item is written directly via `insert_entry_to_blob`, which performs no uniqueness check at all:

```rust
// lines 578-603
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else { return Ok(()); };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;  // ← validated
    }
}
for ((key, value), hash) in keys_values_hashes {          // ← NO duplicate check
    let new_leaf_index = self.get_new_index();
    ...
    self.insert_entry_to_blob(new_leaf_index, &new_block)?; // ← writes unconditionally
    indexes.push(new_leaf_index);
}
``` [2](#0-1) 

`insert_entry_to_blob` calls `block_status_cache.add_leaf`, which overwrites the key→index mapping for a repeated key, leaving the first leaf node orphaned in the blob but still wired into the tree structure via its parent pointer: [3](#0-2) 

The result is a tree whose blob contains two leaf nodes sharing the same `KeyId`, while the cache tracks only the last one. The Merkle root is computed over both leaves (including the hidden duplicate), so the committed root diverges from what the cache believes the tree contains.

`batch_insert` is directly exposed to Python callers via `py_batch_insert`: [4](#0-3) 

and is declared in the public Python stub: [5](#0-4) 

---

### Impact Explanation

The DataLayer Merkle tree root is committed on-chain. Downstream consumers call `get_proof_of_inclusion` and verify the returned `ProofOfInclusion` against that root to confirm that a specific key-value pair is present in the store. [6](#0-5) 

`ProofOfInclusion::valid` reconstructs the root by hashing up the layer chain and comparing to `combined_hash` of the last layer: [7](#0-6) 

When the tree is corrupted by a duplicate-key batch insert:

1. The committed on-chain root is computed over a tree that contains a hidden duplicate leaf.
2. `get_proof_of_inclusion(key)` returns a proof for the cache-tracked leaf (the last duplicate written). That proof hashes up to the corrupted root and `valid()` returns `true`.
3. A verifier comparing the proof root against the on-chain root sees a match — but the tree state the root actually encodes is not the state the cache represents.
4. Conversely, a proof for the *first* (orphaned) leaf cannot be generated at all, so its existence is unprovable and its exclusion is unprovable, breaking both inclusion and exclusion guarantees.

This satisfies the allowed High impact: **DataLayer Merkle proof/blob/delta logic corrupts tree roots and lets untrusted input prove invalid state.**

`check_integrity` would detect the corruption (leaf count vs. cache length mismatch), but it is not called inside `batch_insert` and is not called on every read path: [8](#0-7) 

---

### Likelihood Explanation

`batch_insert` is the primary bulk-load API used by the DataLayer Python layer (called in tests with hundreds of entries). Any code path that constructs the batch from external or user-supplied data without pre-deduplicating keys can trigger this. The batch size threshold is low: the bug activates for any batch of **three or more** entries where at least two share a `KeyId` or `Hash`. No privileged role is required; the Python binding is public.

---

### Recommendation

Add duplicate-key and duplicate-hash checks at the start of the fast path in `batch_insert`, mirroring the guards in `insert`:

```rust
pub fn batch_insert(
    &mut self,
    mut keys_values_hashes: Vec<((KeyId, ValueId), Hash)>,
) -> Result<(), Error> {
    // NEW: validate entire batch before any writes
    for ((key, _), hash) in &keys_values_hashes {
        if self.block_status_cache.contains_key(*key) {
            return Err(Error::KeyAlreadyPresent());
        }
        if self.block_status_cache.contains_leaf_hash(hash) {
            return Err(Error::HashAlreadyPresent());
        }
    }
    // ... existing logic
}
```

Alternatively, deduplicate the input vector before processing, or call `check_integrity` after the batch completes and treat failure as a hard error.

---

### Proof of Concept

```rust
#[test]
fn batch_insert_duplicate_key_corrupts_root() {
    use std::iter::zip;
    let mut blob = MerkleBlob::new(vec![]).unwrap();

    // Three entries: key 0 appears twice (indices 0 and 2 in the batch).
    // The first two go through insert() (validated); the third bypasses checks.
    let batch: Vec<((KeyId, ValueId), Hash)> = vec![
        ((KeyId(1), ValueId(1)), sha256_num(&1_i64)),
        ((KeyId(2), ValueId(2)), sha256_num(&2_i64)),
        ((KeyId(1), ValueId(99)), sha256_num(&99_i64)), // duplicate KeyId(1)
    ];

    // Should return Err, but currently succeeds:
    blob.batch_insert(batch).unwrap();
    blob.calculate_lazy_hashes().unwrap();

    // check_integrity detects the corruption:
    blob.check_integrity().expect_err("tree is corrupted but integrity check was not called");

    // get_proof_of_inclusion returns a proof that hashes to the corrupted root:
    let proof = blob.get_proof_of_inclusion(KeyId(1)).unwrap();
    assert!(proof.valid()); // passes against the wrong root
}
```

The `batch_insert` call succeeds without error. The resulting tree root encodes a hidden duplicate leaf. `get_proof_of_inclusion` produces a proof that validates against the corrupted root, while `check_integrity` (if called) reveals the inconsistency — demonstrating that the tree state and the committed root are out of sync.

    fn insert_entry_to_blob(&mut self, index: TreeIndex, block: &Block) -> Result<(), Error> {
        let new_block_bytes = block.to_bytes()?;
        let extend_index = self.extend_index();
        match index.cmp(&extend_index) {
            Ordering::Greater => return Err(Error::BlockIndexOutOfBounds(index)),
            Ordering::Equal => self.blob.extend_from_slice(&new_block_bytes),
            Ordering::Less => {
                self.blob[block_range(index)].copy_from_slice(&new_block_bytes);
            }
        }

        match block.node {
            Node::Leaf(leaf) => self.block_status_cache.add_leaf(index, leaf),
            Node::Internal(..) => self.block_status_cache.add_internal(index),
        }

        Ok(())
    }

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }

    pub fn valid(&self) -> bool {
        let mut existing_hash = self.node_hash;

        for layer in &self.layers {
            let calculated_hash = crate::calculate_internal_hash(
