### Title
Missing Vote Signature Verification in `handle_vote` Allows Forged Consensus Votes to Drive Wrong Block Commitment — (`File: crates/apollo_consensus/src/single_height_consensus.rs`)

### Summary

`SingleHeightConsensus::handle_vote` checks that the claimed `voter` address is a member of the validator set but never verifies the cryptographic signature on the vote. Any network peer can forge prevotes and precommits for any validator, manufacture a quorum, and drive the sequencer to commit a `DecisionReached` carrying an attacker-chosen `ProposalCommitment` (block hash / state-diff root).

### Finding Description

`handle_vote` in `crates/apollo_consensus/src/single_height_consensus.rs` performs two checks before accepting a vote:

1. Height match (`vote.height != height`)
2. Validator-set membership (`self.validators.contains(&vote.voter)`)

The function then immediately feeds the vote into the state machine. The code itself acknowledges the missing step with an explicit TODO:

### Impact Explanation

When `handle_vote` accepts a forged vote, the state machine accumulates it toward quorum. Once quorum is reached, `upon_decision` fires and emits `SMRequest::DecisionReached(decision)`: [6](#0-5) 

`DecisionReached` carries the `ProposalCommitment` (the partial block hash) that the forged votes named. The manager then calls `context.decision_reached(height, round, decision.block, ...)`, which commits that commitment to the batcher and ultimately writes the wrong global root and block hash to storage: [7](#0-6) 

The batcher's `set_global_root_and_block_hash` then persists the wrong root: [8](#0-7) 

This breaks the core commitment invariant: the stored global state root and block hash must correspond to a block that was agreed upon by a legitimate quorum of validators holding the correct private keys.

### Likelihood Explanation

The validator set is public (it is distributed as part of committee membership). Any peer connected to the consensus gossip network can observe the set of valid `voter` addresses and craft `Vote` messages with arbitrary `proposal_commitment` values and a zero/default `signature`. The only barrier — the validator-set membership check — is trivially satisfied by using a known validator address. No privileged access is required; a single malicious peer with network connectivity can send enough forged votes to reach quorum.

### Recommendation

Call `verify_precommit_vote_signature` (or an equivalent prevote verifier) inside `handle_vote` before the vote is forwarded to the state machine. The public key for each validator must be retrievable from the committee. Reject any vote whose signature does not verify against the claimed `voter`'s public key. Correspondingly, implement the signing side (`make_self_vote`) using the node's private key via `SignatureManager::sign_precommit_vote`.

### Proof of Concept

1. Attacker connects to the consensus gossip network as a peer.
2. Attacker observes the current committee to learn valid `voter` addresses (e.g., `V1`, `V2`, `V3`).
3. Attacker constructs `Vote { vote_type: Precommit, height: H, round: 0, proposal_commitment: Some(EVIL_HASH), voter: V1, signature: RawSignature::default() }` and broadcasts it.
4. Attacker repeats for `V2` and `V3` (or however many are needed for quorum).
5. `handle_vote` passes the height check and the validator-set check for each forged vote; the signature field is never inspected.
6. The state machine accumulates the forged precommits, reaches quorum, and emits `DecisionReached { block: EVIL_HASH }`.
7. The sequencer commits `EVIL_HASH` as the block commitment for height `H`, writing a wrong global state root to storage.

    pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
        // TODO(Asmaa): verify the signature
        trace!("Received {:?}", vote);
        let height = self.state_machine.height();
        if vote.height != height {
            warn!("Invalid vote height: expected {:?}, got {:?}", height, vote.height);
            return VecDeque::new();
        }
        if !self.validators.contains(&vote.voter) {
            debug!("Ignoring vote from non validator: vote={:?}", vote);
            return VecDeque::new();
        }

        // Check if vote has already been received.
        match self.state_machine.received_vote(&vote) {
            VoteStatus::Duplicate => {
                // Duplicate - ignore.
                trace_every_n_ms!(
                    DUPLICATE_VOTE_LOG_PERIOD_MS,
                    "Ignoring duplicate vote: {vote:?}"
                );
                return VecDeque::new();
            }
            VoteStatus::Conflict(old_vote, new_vote) => {
                // Conflict - ignore and record.
                warn!("Conflicting votes: old={old_vote:?}, new={new_vote:?}");
                CONSENSUS_CONFLICTING_VOTES.increment(1);
                return VecDeque::new();
            }
            VoteStatus::New => {
                // Vote is new, proceed to process it.
            }
        }

        info!("Accepting {:?}", vote);
        let sm_vote = match vote.vote_type {
            VoteType::Prevote => StateMachineEvent::Prevote(vote),
            VoteType::Precommit => StateMachineEvent::Precommit(vote),
        };
        self.state_machine.handle_event(sm_vote)
    }
