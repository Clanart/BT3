### Title
Stale `declared_classes_block_table` Index Causes RPC Execution State Reader to Return Pre-Migration Compiled Class Hash After `finalize_block` Migration — (File: `crates/apollo_rpc_execution/src/state_reader.rs`)

---

### Summary

When `finalize_block` applies a compiled-class-hash migration (v1 → v2) and the resulting state diff is persisted via `append_state_diff`, the `declared_classes_block_table` index is intentionally **not** updated for already-declared classes. The RPC execution state reader's `get_compiled_class_hash` resolves the compiled class hash by first looking up the original declaration block from that stale index, then reading the state diff at that block — returning the **pre-migration hash (v1)**. The correct storage path (`get_compiled_class_hash_at` via `compiled_class_hash_table`) returns the post-migration hash (v2). This split causes `starknet_call`, `starknet_estimateFee`, `starknet_simulateTransactions`, and `starknet_traceTransaction` to operate on the wrong compiled class hash, producing authoritative-looking wrong values.

---

### Finding Description

**Step 1 — Migration writes v2 into the state diff.**

`finalize_block` in `crates/blockifier/src/blockifier/transaction_executor.rs` calls `set_compiled_class_hash_migration`, which writes the new v2 hash into `writes.compiled_class_hashes` of the `CachedState`. After `to_state_diff()`, the resulting `CommitmentStateDiff.class_hash_to_compiled_class_hash` contains `class_hash → compiled_class_hash_v2`. This is confirmed by the test assertion at line 508–522 of `transaction_executor_test.rs`. [1](#0-0) 

**Step 2 — `append_state_diff` skips updating `declared_classes_block_table` for already-declared classes.**

In `crates/apollo_storage/src/state/mod.rs`, `append_state_diff` checks whether the class is already in `declared_classes_block_table`. Because the class was declared in an earlier block (block N), `not_declared` is `false` and the table is **not** updated. However, `write_compiled_class_hashes` unconditionally inserts the new v2 hash keyed by `(class_hash, block_M)` into `compiled_class_hash_table`. [2](#0-1) [3](#0-2) 

After this write, the storage state is:
- `declared_classes_block_table[class_hash]` = **block N** (original declaration, never updated)
- `compiled_class_hash_table[(class_hash, block_N)]` = v1 (original)
- `compiled_class_hash_table[(class_hash, block_M)]` = v2 (post-migration)

**Step 3 — RPC execution state reader resolves via the stale index.**

`get_compiled_class_hash` in `crates/apollo_rpc_execution/src/state_reader.rs` (lines 163–208):
1. Calls `get_class_definition_block_number` → reads `declared_classes_block_table` → returns **block N**
2. Calls `get_state_diff(block_N)` → reads the state diff at block N
3. Reads `state_diff.class_hash_to_compiled_class_hash[class_hash]` → returns **v1** [4](#0-3) 

**Step 4 — The correct path returns v2.**

`ApolloReader::get_compiled_class_hash` in `crates/apollo_state_reader/src/apollo_state.rs` calls `get_compiled_class_hash_at`, which uses a cursor-based lower-bound scan on `compiled_class_hash_table` keyed by `(class_hash, state_number)`. This correctly finds the latest entry ≤ the current state number, returning **v2**. [5](#0-4) [6](#0-5) 

The two paths diverge after migration: the RPC execution path returns v1; the block-building path returns v2.

---

### Impact Explanation

Every RPC call that uses `ExecutionStateReader` — `starknet_call`, `starknet_estimateFee`, `starknet_simulateTransactions`, `starknet_traceTransaction` — will resolve the wrong compiled class hash for any class that has been through a migration. This causes:

- **Wrong fee estimates**: gas accounting is performed against the wrong compiled class.
- **Wrong simulation results**: execution traces, revert decisions, and storage diffs are computed against the wrong class.
- **Wrong `Declare` validation in simulation**: a re-declare of the migrated class with v2 would be incorrectly rejected (the reader returns v1, so the hash check fails), while a re-declare with v1 would be incorrectly accepted.

These are authoritative-looking wrong values returned to callers, matching the **High** impact scope: *"RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."*

---

### Likelihood Explanation

The trigger requires `enable_casm_hash_migration = true` (a versioned-constants flag) and at least one class to have been executed after being declared with a v1 hash. Once those conditions hold — which is the intended production path for the migration feature — any unprivileged user calling the affected RPC methods will receive wrong results. No special privileges or adversarial network position are required.

---

### Recommendation

Replace the two-step `declared_classes_block_table → get_state_diff` lookup in `get_compiled_class_hash` (`crates/apollo_rpc_execution/src/state_reader.rs`, lines 174–207) with a direct call to `get_compiled_class_hash_at` on the `StateReader`, which reads from `compiled_class_hash_table` and correctly returns the latest hash for the given state number. This is the same path used by `ApolloReader` and eliminates the stale-index divergence.

---

### Proof of Concept

1. Declare class `C` with `compiled_class_hash = v1` in block N. `declared_classes_block_table[C] = N`.
2. Enable `enable_casm_hash_migration`. Execute a block M > N that invokes `C`. `finalize_block` detects v1 and calls `set_compiled_class_hash_migration`, writing `C → v2` into the state diff.
3. `append_state_diff(M, diff)` is called. Because `declared_classes_block_table[C]` already exists (= N), the table is not updated. `compiled_class_hash_table[(C, M)] = v2` is inserted.
4. Call `starknet_estimateFee` with a transaction that invokes `C`. `ExecutionStateReader::get_compiled_class_hash(C)`:
   - `get_class_definition_block_number(C)` → N
   - `get_state_diff(N).class_hash_to_compiled_class_hash[C]` → **v1** ← wrong
5. The fee estimate is computed against the v1 compiled class. The actual sequencer would execute against v2. The returned fee estimate is wrong. [7](#0-6) [8](#0-7) [9](#0-8)

fn write_compiled_class_hashes<'env>(
    compiled_class_hashes: &IndexMap<ClassHash, CompiledClassHash>,
    txn: &DbTransaction<'env, RW>,
    block_number: BlockNumber,
    compiled_class_hash_table: &'env CompiledClassHashTable<'env>,
) -> StorageResult<()> {
    for (class_hash, compiled_class_hash) in compiled_class_hashes {
        compiled_class_hash_table.insert(txn, &(*class_hash, block_number), compiled_class_hash)?;
    }
    Ok(())
}

    fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
        if let Some(pending_data) = &self.maybe_pending_data {
            for DeclaredClassHashEntry { class_hash: other_class_hash, compiled_class_hash } in
                &pending_data.declared_classes
            {
                if class_hash == *other_class_hash {
                    return Ok(*compiled_class_hash);
                }
            }
        }

        let maybe_block_number = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_reader()
            .map_err(storage_err_to_state_err)?
            .get_class_definition_block_number(&class_hash)
            .map_err(storage_err_to_state_err)?;

        // Cairo 0 classes (and undeclared classes) do not have a compiled class hash.
        // According to the trait, return the default value.
        let Some(block_number) = maybe_block_number else {
            return Ok(CompiledClassHash::default());
        };

        let state_diff = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_diff(block_number)
            .map_err(storage_err_to_state_err)?
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing state diff at block {block_number}."
            )))?;

        let compiled_class_hash = state_diff
            .class_hash_to_compiled_class_hash
            .get(&class_hash)
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing class declaration at block {block_number}, class \
                 {class_hash}."
            )))?;

        Ok(*compiled_class_hash)
    }

    fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
        let state_number = StateNumber(self.latest_block);
        match self
            .reader()?
            .get_state_reader()
            .and_then(|sr| sr.get_compiled_class_hash_at(state_number, &class_hash))
        {
            Ok(Some(compiled_class_hash)) => Ok(compiled_class_hash),
            Ok(None) => Ok(CompiledClassHash::default()),
            Err(err) => Err(StateError::StateReadError(err.to_string())),
        }
    }
