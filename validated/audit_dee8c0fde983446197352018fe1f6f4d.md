### Title
Missing Declared Class Hash Validation in `execute_replace_class` Allows Permanent Fund Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall in the StarkNet OS does not verify that the replacement class hash corresponds to a declared contract class. This creates a lax boundary between the "declared" and "undeclared" class hash categories — directly analogous to the lax boundary between `normalTokens` and `reservedTokens` in the external report. Any contract can replace its own class hash with an arbitrary undeclared value, permanently rendering the contract non-functional and freezing all funds it holds.

---

### Finding Description

In `execute_replace_class`, the OS reads the requested `class_hash` from the syscall request and writes it directly into `contract_state_changes` without checking whether that hash exists in `contract_class_changes` (the declared class registry). The developers themselves acknowledge this missing enforcement with an explicit TODO:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The `contract_class_changes` dict — which is the authoritative registry of declared classes — is never consulted. The OS enforces a one-time-only declaration invariant in `execute_declare_transaction` via `prev_value=0`:

```cairo
// Note that prev_value=0 enforces that a class may be declared only once.
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [2](#0-1) 

But `execute_replace_class` bypasses this registry entirely, writing any arbitrary felt as the new `class_hash` into the contract's `StateEntry`. Once committed, this state change is permanent — the `squash_dict` mechanism in `squash_class_changes` and `squash_state_changes` will finalize it into the global state root with no rollback path. [3](#0-2) 

The two categories with lax boundary enforcement are:

| Category | Meaning | Enforced? |
|---|---|---|
| Declared class hash | Exists in `contract_class_changes`; contract is callable | Yes, at declare time |
| Undeclared class hash | No corresponding class; contract is permanently broken | **No** — `replace_class` does not check |

---

### Impact Explanation

Once a contract's `StateEntry.class_hash` is set to an undeclared value, every subsequent call to that contract will fail at class lookup. There is no recovery mechanism — the state is committed to the Patricia Merkle tree and the class hash cannot be reset without another `replace_class` call, which itself requires the contract to be callable. Any funds (ERC-20 balances, locked collateral, etc.) stored in the contract's storage are permanently inaccessible.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

The entry path requires only an unprivileged contract deployer:

1. Attacker deploys a contract (e.g., a yield vault or escrow) that accepts user deposits.
2. Users deposit funds; the contract accumulates balances.
3. Attacker calls `replace_class` from within the contract, supplying an arbitrary undeclared hash (e.g., `0xdeadbeef`).
4. The OS writes the undeclared hash into `contract_state_changes` with no validation.
5. The state is squashed and committed to the global root.
6. All future calls to the contract revert at class resolution; funds are frozen permanently.

No privileged role, leaked key, or operator access is required. The attacker only needs to control a contract — a standard capability of any contract deployer. Contracts with on-chain upgrade governance (where token holders vote on a new class hash) are also vulnerable if the governance logic does not independently validate that the proposed hash is declared.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, `execute_replace_class` must verify that the hash exists in `contract_class_changes`. Concretely, perform a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert the returned value is non-zero (i.e., a valid compiled class hash was previously registered via `execute_declare_transaction`).

---

### Proof of Concept

```
1. Deploy contract C with class hash H_valid (declared).
   C exposes: deposit(), withdraw(), upgrade(new_hash).

2. Users call C.deposit() — funds accumulate in C's storage.

3. Attacker calls C.upgrade(0xdeadbeef):
   - C internally calls replace_class(0xdeadbeef).
   - OS executes execute_replace_class:
       class_hash = 0xdeadbeef  // undeclared
       // No check against contract_class_changes
       new_state_entry = StateEntry(class_hash=0xdeadbeef, ...)
       dict_update(contract_state_changes, key=C, new_value=new_state_entry)

4. Block is proven; state root now encodes C.class_hash = 0xdeadbeef.

5. Any call to C → OS looks up class 0xdeadbeef → not found → revert.
   C.withdraw() is permanently unreachable.
   All deposited funds are frozen.
```

The root cause is the absent cross-check between `contract_state_changes` and `contract_class_changes` in `execute_replace_class`. [4](#0-3)

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}

func squash_class_changes{range_check_ptr}(
    class_changes_start: DictAccess*, class_changes_end: DictAccess*
) -> (n_class_updates: felt, squashed_contract_state_dict: DictAccess*) {
    alloc_locals;

    local squashed_dict: DictAccess*;
    %{ GuessClassesPtr %}
    let (local squashed_dict_end) = squash_dict(
        dict_accesses=class_changes_start,
        dict_accesses_end=class_changes_end,
        squashed_dict=squashed_dict,
    );

    %{ UpdateClassesPtr %}

    return (
        n_class_updates=(squashed_dict_end - squashed_dict) / DictAccess.SIZE,
        squashed_contract_state_dict=squashed_dict,
    );
