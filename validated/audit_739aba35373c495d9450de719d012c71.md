### Title
`delete_declared_classes` Unconditionally Erases Sierra Class Body on Block Revert for Migrated-Compiled-Class Entries, Corrupting Class Storage and Proof Inputs — (File: `crates/apollo_storage/src/state/mod.rs`)

---

### Summary

`revert_state_diff` calls two sibling helpers to clean up class data. `delete_declared_classes_block` correctly guards deletion with a block-number equality check, but `delete_declared_classes` carries no such guard and unconditionally removes the Sierra class body from `declared_classes_table` for every entry in `thin_state_diff.class_hash_to_compiled_class_hash`. Because `ThinStateDiff::class_hash_to_compiled_class_hash` is populated from **both** freshly declared classes **and** `migrated_compiled_classes`, reverting any block that contains a compiled-class-hash migration for a class declared in an earlier block permanently destroys that class's body from storage. The result is an inconsistent DB state: `declared_classes_block_table` still points to the original declaration block while `declared_classes_table` has no entry, causing `get_class_definition_at` to return `None` or a `DBInconsistency` error for a legitimately declared class. Every downstream consumer — RPC class queries, SNOS proof inputs, and the class-manager's execution path — receives a wrong authoritative value.

---

### Finding Description

**Root cause — asymmetric block-number guard in `revert_state_diff`**

`revert_state_diff` calls two helpers in sequence:

```
delete_declared_classes_block   // line 642-647
delete_declared_classes         // line 648-653
```

`delete_declared_classes_block` (lines 828–851) reads the first-declaration block from `declared_classes_block_table` and only deletes the record when `class_block_entry == block_number`:

```rust
if class_block_entry == block_number {
    declared_classes_block_table.delete(txn, class_hash)?;
    deleted_data.push(*class_hash);
}
```

`delete_declared_classes` (lines 853–872) carries **no such guard**. It iterates every key in `thin_state_diff.class_hash_to_compiled_class_hash` and deletes the Sierra body unconditionally:

```rust
for class_hash in thin_state_diff.class_hash_to_compiled_class_hash.keys() {
    let Some(contract_class_location) = declared_classes_table.get(txn, class_hash)? else {
        continue;
    };
    declared_classes_table.delete(txn, class_hash)?;   // ← no block-number check
}
```

**Why `class_hash_to_compiled_class_hash` contains classes from earlier blocks**

`ThinStateDiff::from_state_diff` (lines 84–93 of `crates/starknet_api/src/state.rs`) merges both `declared_classes` and `migrated_compiled_classes` into the single `class_hash_to_compiled_class_hash` map:

```rust
class_hash_to_compiled_class_hash: diff
    .declared_classes
    .iter()
    .map(|(class_hash, (compiled_hash, _class))| (*class_hash, *compiled_hash))
    .chain(
        diff.migrated_compiled_classes
            .iter()
            .map(|(class_hash, compiled_hash)| (*class_hash, *compiled_hash)),
    )
    .collect(),
```

`append_state_diff` (lines 549–554) guards `declared_classes_block_table` with `if not_declared`, so a migrated class that was declared in block N will **not** update the block-number record when it appears again in block M. After appending block M, `declared_classes_block_table[C] == N` (unchanged).

**Inconsistent state after revert**

When block M is reverted:

| Table | Expected | Actual |
|---|---|---|
| `declared_classes_block_table[C]` | `N` (kept) | `N` (kept — correct) |
| `declared_classes_table[C]` | present | **deleted — wrong** |

**Corrupted read path**

`get_class_definition_at` (lines 428–454) first checks `declared_classes_block_table` (finds block N, proceeds), then reads `declared_classes_table` (finds nothing). Because the class marker has already advanced past block N, the function falls into the `is_after` branch and silently returns `Ok(None)` — the class appears to have never been declared.

---

### Impact Explanation

**Critical — Wrong compiled class / contract code selected for execution; wrong authoritative RPC value.**

After the revert the Sierra class body is gone from `declared_classes_table`. Any subsequent operation that needs the class body receives `None`:

- **Execution / blockifier**: a `deploy` or `call` targeting the class will fail to load the contract class, producing a wrong execution result or revert for an otherwise valid transaction.
- **SNOS / proof inputs**: `create_facts_db_from_storage_proof` and the transaction prover rely on class definitions being retrievable from storage; a missing body causes the prover to build facts from an incomplete set, yielding a wrong or unverifiable proof.
- **RPC `starknet_getClass`**: returns `None` for a class that is still legitimately declared, an authoritative-looking wrong value.
- **State root / commitment**: the Patricia classes trie still holds the class hash → compiled class hash leaf (the trie was not rolled back), but the body backing it is gone, breaking the invariant that every leaf in the classes trie has a retrievable body.

---

### Likelihood Explanation

**Moderate.** Two ordinary protocol conditions must coincide:

1. A block containing at least one `migrated_compiled_classes` entry for a class declared in an earlier block is committed. This is a standard protocol operation (compiled-class-hash v1→v2 migration) and is exercised in existing tests and production sync data.
2. That block is subsequently reverted. Block reverts occur during normal consensus operation (failed proposals, reorganisations) and are explicitly supported by `apollo_reverts`.

Neither condition requires privileged access or a malicious peer; both arise from ordinary sequencer operation.

---

### Recommendation

Mirror the block-number guard from `delete_declared_classes_block` inside `delete_declared_classes`. Pass `block_number` and `declared_classes_block_table` to the function and skip deletion when the recorded first-declaration block differs from the block being reverted:

```rust
fn delete_declared_classes<'env>(
    txn: &'env DbTransaction<'env, RW>,
    thin_state_diff: &ThinStateDiff,
    declared_classes_table: &'env DeclaredClassesTable<'env>,
    declared_classes_block_table: &'env DeclaredClassesBlockTable<'env>,
    block_number: BlockNumber,
    file_handlers: &FileHandlers<RW>,
) -> StorageResult<IndexMap<ClassHash, SierraContractClass>> {
    let mut deleted_data = IndexMap::new();
    for class_hash in thin_state_diff.class_hash_to_compiled_class_hash.keys() {
        // Only delete the body if this block is the one that first declared the class.
        let first_declared = declared_classes_block_table.get(txn, class_hash)?;
        if first_declared != Some(block_number) {
            continue;
        }
        let Some(location) = declared_classes_table.get(txn, class_hash)? else {
            continue;
        };
        deleted_data.insert(*class_hash, file_handlers.get_contract_class_unchecked(location)?);
        declared_classes_table.delete(txn, class_hash)?;
    }
    Ok(deleted_data)
}
```

A regression test analogous to `revert_doesnt_delete_previously_declared_classes` (which covers only deprecated classes) should be added for Cairo-1 classes that appear in a later block's `migrated_compiled_classes`.

### Proof of Concept

```
Block 0:
  append_state_diff(0, { class_hash_to_compiled_class_hash: { C → H1 } })
  append_classes(0, [(C, sierra_class_C)], [])
  → declared_classes_block_table[C] = 0
  → declared_classes_table[C]       = <location of sierra_class_C>

Block 1 (migration: C's compiled hash updated from H1 to H2):
  append_state_diff(1, { class_hash_to_compiled_class_hash: { C → H2 } })
  → declared_classes_block_table[C] = 0  (unchanged, "not_declared" guard fires)
  → declared_classes_table[C]       = <location of sierra_class_C>  (unchanged)

Revert block 1:
  delete_declared_classes_block: class_block_entry(C)=0 ≠ 1 → SKIP (correct)
  delete_declared_classes:       no guard → DELETE declared_classes_table[C]  ← BUG

After revert:
  declared_classes_block_table[C] = 0          (present — misleading)
  declared_classes_table[C]       = (missing)  (deleted — wrong)

get_class_definition_at(state_after_block_0, C):
  1. declared_classes_block_table[C] = 0 → found, proceed
  2. declared_classes_table[C]       → None
  3. state_number.is_after(class_marker) → return Ok(None)   ← class appears undeclared
```

fn delete_declared_classes_block<'env>(
    txn: &'env DbTransaction<'env, RW>,
    thin_state_diff: &ThinStateDiff,
    declared_classes_block_table: &'env DeclaredClassesBlockTable<'env>,
    block_number: BlockNumber,
) -> StorageResult<Vec<ClassHash>> {
    let mut deleted_data = Vec::new();
    for class_hash in thin_state_diff.class_hash_to_compiled_class_hash.keys() {
        let class_block_entry =
            declared_classes_block_table.get(txn, class_hash)?.ok_or_else(|| {
                StorageError::DBInconsistency {
                    msg: format!(
                        "Attempting to revert declaration of class {class_hash} but it doesn't \
                         exist in the DB"
                    ),
                }
            })?;
        if class_block_entry == block_number {
            declared_classes_block_table.delete(txn, class_hash)?;
            deleted_data.push(*class_hash);
        }
    }
    Ok(deleted_data)
}
