### Title
Missing Validation of `replace_class` Class Hash Against Declared Classes — (`File: execution/syscall_impls.cairo`)

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary class hash from a contract without verifying that the hash corresponds to a previously declared class. An unprivileged contract can call `replace_class` with an undeclared class hash, permanently making itself uncallable and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall. It reads the requested `class_hash` directly from the syscall request and updates the contract's state entry without any validation that the class hash is declared: [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges the missing check:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
```

After `replace_class` succeeds, the contract's `class_hash` field in `contract_state_changes` is set to the attacker-supplied value. When any subsequent transaction calls this contract, `execute_entry_point` performs a `find_element` lookup against `compiled_class_facts_bundle` using the stored `compiled_class_hash`: [2](#0-1) 

`find_element` is a Cairo standard library function that asserts the element exists. If the class hash is undeclared, the assertion fails, causing the entire Cairo VM execution to abort — the block cannot be proven. The sequencer will therefore permanently exclude any transaction that calls the frozen contract, making it permanently uncallable.

The analog to the oracle report is direct: just as `WstEthSpotOracle` and `EthXSpotOracle` use oracle-returned price data without validating it is non-zero or valid, `execute_replace_class` uses the user-supplied `class_hash` without validating it is a declared class.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract calls `replace_class` with an undeclared class hash:
1. The contract's class hash in state is set to the undeclared value.
2. Any future call to the contract causes `find_element` to abort the OS execution.
3. The sequencer permanently excludes all transactions targeting that contract.
4. All ERC-20 balances, NFTs, or other assets stored in or controlled by the contract become permanently inaccessible.
5. The contract cannot self-recover: calling `replace_class` again requires calling the contract, which is now impossible.

---

### Likelihood Explanation

**Medium.** Any deployed contract can invoke the `replace_class` syscall — no privileged role is required. The attack can be triggered by:
- A malicious contract author who intentionally calls `replace_class(undeclared_hash)` to rug-pull users after collecting funds.
- A buggy contract that accidentally passes an incorrect class hash.

The syscall is a standard, publicly accessible protocol feature. The missing check is explicitly noted as a TODO, confirming the code path is reachable and the validation is absent.

---

### Recommendation

Before updating the contract state in `execute_replace_class`, validate that `class_hash` exists in `contract_class_changes` (i.e., it was declared in the current or a prior block). Specifically, perform a `dict_read` on `contract_class_changes` for the given `class_hash` and assert the returned compiled class hash is non-zero:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the validation already performed in `execute_entry_point` but enforces it at the point of class replacement, preventing undeclared class hashes from ever entering the state.

---

### Proof of Concept

1. Deploy Contract A (e.g., a vault holding user funds).
2. From Contract A, invoke the `replace_class` syscall with `class_hash = 0xdeadbeef` (any value not present in `compiled_class_facts_bundle`).
3. `execute_replace_class` succeeds: no validation is performed, and Contract A's state entry is updated with `class_hash = 0xdeadbeef`.
4. Submit any transaction that calls Contract A.
5. The OS reaches `execute_entry_point`, reads `compiled_class_hash` for `0xdeadbeef` from `contract_class_changes` (returns 0, the default), then calls `find_element` with key `0` against `compiled_class_facts_bundle`.
6. `find_element` aborts the Cairo VM — the block cannot be proven.
7. The sequencer permanently drops all transactions targeting Contract A.
8. All funds in Contract A are permanently frozen. [3](#0-2) [4](#0-3)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
