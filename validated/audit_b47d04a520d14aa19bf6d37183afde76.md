### Title
`MerkleBlob::batch_insert` Skips Duplicate Key/Hash Validation, Corrupting Tree Roots and Proofs of Inclusion — (`File: crates/chia-datalayer/src/merkle/blob.rs`)

### Summary

`MerkleBlob::batch_insert` bypasses the `KeyAlreadyPresent` and `HashAlreadyPresent` guards that `insert` enforces for all but the last two items in a batch. An attacker who can supply duplicate keys or hashes to a `batch_insert` call silently corrupts the `BlockStatusCache`, orphans leaf nodes in the blob, and produces an incorrect Merkle root — invalidating all subsequent proofs of inclusion or exclusion derived from that tree.

### Finding Description

`MerkleBlob::insert` enforces two invariants before writing any leaf:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  L369-374
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`batch_insert` calls `insert` (with those guards) only for the **last two** items popped from the input vector when the tree has ≤ 1 existing leaves. Every other item is written directly via `insert_entry_to_blob` with no duplicate check:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  L578-603
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else { return Ok(()); };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;   // ← validated
    }
}
for ((key, value), hash) in keys_values_hashes {   // ← ALL remaining items, no validation
    let new_leaf_index = self.get_new_index();
    let new_block = Block { node: Node::Leaf(LeafNode { hash, key, value, .. }), .. };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;   // ← no duplicate check
    indexes.push(new_leaf_index);
}
``` [2](#0-1) 

When `leaf_count > 1` the `if` block is skipped entirely, so **all** items in the batch bypass validation.

`insert_entry_to_blob` calls `block_status_cache.add_leaf`, which performs a plain `HashMap::insert` — silently overwriting any existing entry for the same key or hash:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  L188-193
fn add_leaf(&mut self, index: TreeIndex, leaf: LeafNode) {
    self.free_indexes.shift_remove(&index);
    self.key_to_index.insert(leaf.key, index);          // ← silent overwrite
    self.leaf_hash_to_index.insert(leaf.hash, index);   // ← silent overwrite
}
``` [3](#0-2) 

The result is:

1. The **first** leaf for a duplicated key is written into the blob at its allocated index and wired into the tree's parent/child structure.
2. The cache entry for that key is then overwritten to point to the **second** leaf's index.
3. The first leaf becomes an **orphan**: it exists in the blob, participates in internal-node hash computation, but is invisible to the cache.
4. `calculate_lazy_hashes` propagates hashes upward through the actual blob structure (including the orphan), producing a root hash that does not match what the cache believes the tree contains.

The Python binding `py_batch_insert` exposes this path directly to callers:

```python

### Impact Explanation

The DataLayer Merkle tree is the authoritative source for key-value inclusion proofs. Corrupting it via `batch_insert` with duplicate keys:

- Produces an **incorrect root hash** that is committed on-chain, diverging from the actual tree state.
- Causes `get_proof_of_inclusion` to return a proof for the second (cache-visible) leaf while the orphaned first leaf still contributes to the root — making the proof structurally invalid against the committed root.
- Allows a malicious DataLayer peer to force a receiving node into a state where it **cannot generate valid proofs** for keys it believes it holds, or where it **accepts proofs** that do not correspond to the committed root.

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

### Likelihood Explanation

`batch_insert` is the primary bulk-insertion path used during DataLayer sync. A malicious DataLayer server that includes duplicate `(key, hash)` pairs in a sync response can trigger this on any receiving node. No privileged access is required — only the ability to serve DataLayer data to a syncing peer. The Python binding is directly reachable from the DataLayer sync code path.

### Recommendation

Add duplicate-key and duplicate-hash checks at the start of `batch_insert` (or inside the fast path loop) mirroring the guards in `insert`:

```rust
for ((key, value), hash) in &keys_values_hashes {
    if self.block_status_cache.contains_key(*key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(hash) {
        return Err(Error::HashAlreadyPresent());
    }
}
```

Alternatively, make `add_leaf` return an error on collision so that `insert_entry_to_blob` propagates it, ensuring the invariant is enforced at the lowest level regardless of which insertion path is used.

### Proof of Concept

```rust
use chia_datalayer::{Hash, KeyId, MerkleBlob, ValueId};
use chia_protocol::Bytes32;

fn main() {
    let mut blob = MerkleBlob::new(Vec::new()).unwrap();

    // Build a batch with a duplicate key (key=1 appears twice)
    // Tree starts empty (leaf_count == 0 <= 1), so the last 2 items
    // are validated; the first item (duplicate key=1) is not.
    let dup_key   = KeyId(1);
    let hash_a    = Hash(Bytes32::new([0xAA; 32]));
    let hash_b    = Hash(Bytes32::new([0xBB; 32]));
    let hash_c    = Hash(Bytes32::new([0xCC; 32]));

    // items[0] = duplicate key=1 (bypasses validation)
    // items[1] = key=1 (validated via insert — succeeds because cache is still empty at that point)
    // items[2] = key=2 (validated via insert)
    let batch = vec![
        ((dup_key,    ValueId(10)), hash_a),  // ← no duplicate check
        ((dup_key,    ValueId(20)), hash_b),  // ← validated, passes (cache empty)
        ((KeyId(2),   ValueId(30)), hash_c),  // ← validated, passes
    ];

    blob.batch_insert(batch).unwrap(); // succeeds — no error raised

    // Cache now points key=1 → hash_b leaf; hash_a leaf is orphaned in blob
    blob.calculate_lazy_hashes().unwrap();

    // check_integrity detects the orphan: IntegrityKeyToIndexCacheIndex
    blob.check_integrity().expect_err("tree is corrupted");
}

    fn insert_entry_to_blob(&mut self, index: TreeIndex, block: &Block) -> Result<(), Error> {
        let new_block_bytes = block.to_bytes()?;
        let extend_index = self.extend_index();
        match index.cmp(&extend_index) {
            Ordering::Greater => return Err(Error::BlockIndexOutOfBounds(index)),
            Ordering::Equal => self.blob.extend_from_slice(&new_block_bytes),
            Ordering::Less => {
                self.blob[block_range(index)].copy_from_slice(&new_block_bytes);
            }
        }

        match block.node {
            Node::Leaf(leaf) => self.block_status_cache.add_leaf(index, leaf),
            Node::Internal(..) => self.block_status_cache.add_internal(index),
        }

        Ok(())
    }
