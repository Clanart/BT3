### Title
Silent `big.Int` → `int64` Truncation in `SuggestGasTipCap` Produces Incorrect Gas Price Suggestions - (File: rpc/backend/chain_info.go)

### Summary
`SuggestGasTipCap` calls `baseFee.Int64()` on an unbounded `*big.Int` without any overflow guard. When the EIP-1559 base fee exceeds `math.MaxInt64` (9.2 × 10¹⁸), Go's `big.Int.Int64()` silently returns the low-order 64 bits reinterpreted as a signed integer — a negative value — causing the computed `maxDelta` to be clamped to zero. Every downstream JSON-RPC endpoint that relies on this function (`eth_gasPrice`, `eth_maxPriorityFeePerGas`, `eth_sendTransaction` via `SetTxDefaults`) then returns a zero tip, feeding incorrect fee data into transaction construction.

### Finding Description

In `SuggestGasTipCap`:

```go
// rpc/backend/chain_info.go:439
maxDelta := baseFee.Int64() * (int64(params.Params.ElasticityMultiplier) - 1) / int64(params.Params.BaseFeeChangeDenominator)
if maxDelta < 0 {
    maxDelta = 0
}
return big.NewInt(maxDelta), nil
```

`baseFee` is a `*big.Int` sourced from the feemarket module's `BaseFee` parameter, which is itself an `sdkmath.Int` (arbitrary-precision). Go's `(*big.Int).Int64()` documentation states the result is *undefined* when the value does not fit in 64 bits; in practice it returns the low-order 64 bits as a signed integer. When `baseFee > math.MaxInt64`, the result is negative, the `if maxDelta < 0` guard clamps it to zero, and the function returns `big.NewInt(0)`.

The correct arithmetic — `baseFee * (ElasticityMultiplier - 1) / Denominator` — must be performed entirely in `*big.Int` before any narrowing conversion. [1](#0-0) 

### Impact Explanation

Three public JSON-RPC endpoints are directly affected:

1. **`eth_gasPrice`** (`GasPrice()`) calls `SuggestGasTipCap(head.BaseFee)` and adds the result to `head.BaseFee`. With a zero tip the returned gas price equals exactly `baseFee`, the bare minimum. Wallets using this for legacy transactions will set `gasPrice = baseFee`; any subsequent base-fee increase causes those transactions to be stuck or dropped. [2](#0-1) 

2. **`eth_maxPriorityFeePerGas`** (`MaxPriorityFeePerGas()`) returns the raw result of `SuggestGasTipCap`, which becomes zero. EIP-1559 wallets set `maxPriorityFeePerGas = 0`, so the effective gas price equals exactly `baseFee` with no tip buffer. [3](#0-2) 

3. **`eth_sendTransaction` / `SetTxDefaults`** uses `SuggestGasTipCap` to populate `MaxPriorityFeePerGas` and derives `MaxFeePerGas = 0 + 2*baseFee`. Transactions constructed through this path carry zero tip and a `MaxFeePerGas` that provides no headroom above the current base fee, making them immediately invalid if the base fee rises even slightly. [4](#0-3) 

This matches the allowed High impact: *"Public JSON-RPC path feeds incorrect consensus-critical data into transaction execution."*

### Likelihood Explanation

The EIP-1559 base fee grows at most 12.5 % per block. Starting from the default `InitialBaseFee` of 1 Gwei (10⁹):

```
10⁹ × 1.125ⁿ > 9.2 × 10¹⁸  →  n ≈ 194 consecutive full blocks
```

At 6-second block times that is roughly 19 minutes of sustained maximum-throughput traffic. While expensive, a well-funded attacker can deliberately fill blocks to push the base fee past `MaxInt64`. Once triggered, the overflow persists for every subsequent block until the base fee falls back below the threshold, continuously serving incorrect gas price data to all connected clients. [5](#0-4) 

### Recommendation

Replace the narrowing `baseFee.Int64()` call with full `*big.Int` arithmetic throughout `SuggestGasTipCap`:

```go
func (b *Backend) SuggestGasTipCap(baseFee *big.Int) (*big.Int, error) {
    if baseFee == nil {
        return big.NewInt(0), nil
    }
    params, err := b.queryClient.FeeMarket.Params(b.ctx, &feemarkettypes.QueryParamsRequest{})
    if err != nil {
        return nil, err
    }
    // maxDelta = baseFee * (ElasticityMultiplier - 1) / BaseFeeChangeDenominator
    em := new(big.Int).SetUint64(uint64(params.Params.ElasticityMultiplier))
    denom := new(big.Int).SetUint64(uint64(params.Params.BaseFeeChangeDenominator))
    maxDelta := new(big.Int).Mul(baseFee, new(big.Int).Sub(em, big.NewInt(1)))
    maxDelta.Div(maxDelta, denom)
    if maxDelta.Sign() < 0 {
        maxDelta.SetInt64(0)
    }
    return maxDelta, nil
}
```

Also audit `x/feemarket/keeper/abci.go` line 41 where `baseFee.Int64()` is used for telemetry — while not security-critical, it will silently emit a wrong gauge value under the same condition. [6](#0-5) 

### Proof of Concept

1. Deploy an Ethermint chain with default parameters (`InitialBaseFee = 1e9`, `ElasticityMultiplier = 2`, `BaseFeeChangeDenominator = 8`).
2. For ~194 consecutive blocks, submit transactions that fill each block to its gas limit (e.g., via a script that continuously broadcasts high-gas-limit transactions).
3. Once `baseFee > 9.2 × 10¹⁸`:
   - Call `eth_gasPrice` → observe the returned value equals exactly `baseFee` (zero tip component).
   - Call `eth_maxPriorityFeePerGas` → observe `0x0`.
   - Call `eth_sendTransaction` with no explicit fee fields → observe `maxPriorityFeePerGas = 0x0` in the constructed transaction.
4. Any wallet relying on these RPC responses will construct transactions with zero tip and no fee headroom, causing them to be stuck or rejected as the base fee continues to fluctuate. [7](#0-6)

func (b *Backend) SuggestGasTipCap(baseFee *big.Int) (*big.Int, error) {
	if baseFee == nil {
		// london hardfork not enabled or feemarket not enabled
		return big.NewInt(0), nil
	}

	params, err := b.queryClient.FeeMarket.Params(b.ctx, &feemarkettypes.QueryParamsRequest{})
	if err != nil {
		return nil, err
	}
	// calculate the maximum base fee delta in current block, assuming all block gas limit is consumed
	// ```
	// GasTarget = GasLimit / ElasticityMultiplier
	// Delta = BaseFee * (GasUsed - GasTarget) / GasTarget / Denominator
	// ```
	// The delta is at maximum when `GasUsed` is equal to `GasLimit`, which is:
	// ```
	// MaxDelta = BaseFee * (GasLimit - GasLimit / ElasticityMultiplier) / (GasLimit / ElasticityMultiplier) / Denominator
	//          = BaseFee * (ElasticityMultiplier - 1) / Denominator
	// ```
	maxDelta := baseFee.Int64() * (int64(params.Params.ElasticityMultiplier) - 1) / int64(params.Params.BaseFeeChangeDenominator)
	if maxDelta < 0 {
		// impossible if the parameter validation passed.
		maxDelta = 0
	}
	return big.NewInt(maxDelta), nil
}
