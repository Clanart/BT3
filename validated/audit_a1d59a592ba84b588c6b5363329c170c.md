### Title
`locked_tokens` Accounting Desync: `claim_fee_callback` Only Unlocks Fee Portion, Leaving `amount_without_fee` Permanently Locked - (File: near/omni-bridge/src/lib.rs)

### Summary

When a cross-chain transfer to a non-origin chain is completed via `claim_fee_callback`, the `locked_tokens` map for the destination chain is only decremented by the relayer fee (`token_fee`), not by the full committed amount (`fee + amount_without_fee`). This permanently inflates `locked_tokens[(destination_chain, token)]` by `amount_without_fee` per completed transfer, breaking the bridge's collateralization accounting invariant.

### Finding Description

The `locked_tokens` map tracks how many tokens are committed to each chain. When a transfer is routed through NEAR to a non-origin destination (e.g., Eth→Sol), `process_fin_transfer_to_other_chain` increases `locked_tokens[(Sol, token)]` by the full transfer amount: [1](#0-0) 

Specifically, two `lock_tokens_if_needed` calls add `fee` and `amount_without_fee` to `locked_tokens[(Sol, token)]`, totaling `amount`.

Later, when the relayer delivers tokens to the Sol recipient and calls `claim_fee`, `claim_fee_callback` computes:

```
fee = transfer_message.amount.0 - denormalized_amount
```

and calls `send_fee_internal` with only this `fee` value: [2](#0-1) 

Inside `send_fee_internal`, only `token_fee` (the fee portion) is unlocked: [3](#0-2) 

The `amount_without_fee` portion — representing tokens already delivered to the Sol recipient — is **never unlocked** from `locked_tokens[(Sol, token)]`.

The same desync occurs in the BTC path via `submit_transfer_to_btc_connector_callback`: [4](#0-3) 

And for NEAR-originated transfers (NEAR→Sol via `init_transfer_internal`), `locked_tokens[(Sol, token)]` is increased by the full amount: [5](#0-4) 

but `claim_fee` only decrements by `fee`, leaving `amount_without_fee` permanently locked.

### Impact Explanation

**Impact: High** — Balance/accounting corruption that breaks bridge collateralization.

`locked_tokens` is the bridge's primary mechanism for tracking cross-chain obligations and enforcing collateralization. The invariant is: `locked_tokens[(chain, token)]` = tokens committed to `chain` not yet delivered. After a complete transfer cycle, this should return to its pre-transfer value. Instead, it permanently accumulates `amount_without_fee` per completed transfer.

Consequences:
1. **Collateralization tracking is broken**: The bridge's accounting permanently overstates its Sol-side obligations. The bridge believes it has more tokens committed to Sol than it actually does.
2. **Weakened secondary safety check**: `unlock_tokens` enforces `available >= amount`. With inflated `locked_tokens[(Sol, token)]`, this check passes more easily than it should, allowing more Sol→NEAR finalizations than the accounting supports.
3. **Unbounded inflation**: Each completed transfer to a non-origin chain inflates `locked_tokens[(destination_chain, token)]` by `amount_without_fee`. Over time this grows without bound, eventually risking overflow (though `checked_add` would panic rather than wrap).
4. **Potential over-release**: The inflated `locked_tokens` allows `unlock_tokens_if_needed(Sol, token, amount)` to succeed for amounts that exceed the actual backed commitments, weakening the bridge's collateralization guarantee.

### Likelihood Explanation

**Likelihood: High** — This affects every completed transfer to a non-origin chain. Any user who initiates a transfer from Eth/NEAR to Sol (or BTC) and has the fee claimed by a relayer triggers this inflation. It is systematic and cumulative.

### Recommendation

In `send_fee_internal`, unlock the full `transfer_message.amount.0` from `locked_tokens[(destination_chain, token)]` instead of only `token_fee`:

```rust
// Current (buggy):
self.unlock_tokens_if_needed(transfer_message.get_destination_chain(), &token, token_fee);

// Fixed:
self.unlock_tokens_if_needed(transfer_message.get_destination_chain(), &token, transfer_message.amount.0);
```

This ensures that after a complete transfer cycle, `locked_tokens[(destination_chain, token)]` is fully decremented by the committed amount, restoring the collateralization invariant.

### Proof of Concept

1. Token `T` is native to Eth, deployed on NEAR and Sol. `locked_tokens[(Sol, T)] = 0`.
2. User calls `fin_transfer` with an Eth→Sol `InitTransfer` proof for 1000 T (fee = 10 T).
3. `process_fin_transfer_to_other_chain` runs:
   - `lock_tokens_if_needed(Sol, T, 10)` → `locked_tokens[(Sol, T)] = 10`
   - `lock_tokens_if_needed(Sol, T, 990)` → `locked_tokens[(Sol, T)] = 1000`
4. Relayer delivers 990 T to Sol recipient, submits `FinTransfer` proof, calls `claim_fee`.
5. `claim_fee_callback` computes `fee = 1000 - 990 = 10`, calls `send_fee_internal(..., 10)`.
6. `send_fee_internal` calls `unlock_tokens_if_needed(Sol, T, 10)` → `locked_tokens[(Sol, T)] = 990`.
7. Transfer is complete. `locked_tokens[(Sol, T)]` should be 0, but is **990** — permanently inflated.
8. Repeat N times: `locked_tokens[(Sol, T)]` = N × 990, growing without bound, breaking the bridge's collateralization accounting. [6](#0-5) [7](#0-6)

    fn send_fee_internal(
        &mut self,
        transfer_message: &TransferMessage,
        fee_recipient: AccountId,
        token_fee: u128,
    ) -> PromiseOrValue<()> {
        if transfer_message.fee.native_fee.0 != 0 {
            let origin_chain = transfer_message.origin_transfer_id.as_ref().map_or_else(
                || transfer_message.get_origin_chain(),
                |origin_transfer_id| origin_transfer_id.origin_chain,
            );

            if origin_chain.is_utxo_chain() {
                env::panic_str(BridgeError::NativeFeeForUtxoChain.to_string().as_str())
            } else if origin_chain == ChainKind::Near {
                Promise::new(fee_recipient.clone())
                    .transfer(NearToken::from_yoctonear(transfer_message.fee.native_fee.0))
                    .detach();
            } else {
                ext_token::ext(self.get_native_token_id(origin_chain))
                    .with_static_gas(MINT_TOKEN_GAS)
                    .mint(fee_recipient.clone(), transfer_message.fee.native_fee, None)
                    .detach();
            }
        }

        let token = self.get_token_id(&transfer_message.token);
        env::log_str(
            &OmniBridgeEvent::ClaimFeeEvent {
                transfer_message: transfer_message.clone(),
            }
            .to_log_string(),
        );

        self.unlock_tokens_if_needed(transfer_message.get_destination_chain(), &token, token_fee);

        if token_fee > 0 {
            if self.is_deployed_token(&token) {
                ext_token::ext(token)
                    .with_static_gas(MINT_TOKEN_GAS)
                    .mint(fee_recipient, U128(token_fee), None)
                    .into()
            } else {
                ext_token::ext(token)
                    .with_static_gas(FT_TRANSFER_GAS)
                    .with_attached_deposit(ONE_YOCTO)
                    .ft_transfer(fee_recipient, U128(token_fee), None)
                    .into()
            }
        } else {
            PromiseOrValue::Value(())
        }

    fn unlock_tokens(
        &mut self,
        chain_kind: ChainKind,
        token_id: &AccountId,
        amount: u128,
    ) -> LockAction {
        let key = (chain_kind, token_id.clone());
        let Some(available) = self.locked_tokens.get(&key) else {
            return LockAction::Unchanged;
        };
        require!(
            available >= amount,
            TokenLockError::InsufficientLockedTokens.as_ref()
        );

        let remaining = available - amount;
        self.locked_tokens.insert(&key, &remaining);

        LockAction::Unlocked {
            chain_kind,
            token_id: token_id.clone(),
            amount,
        }
    }
