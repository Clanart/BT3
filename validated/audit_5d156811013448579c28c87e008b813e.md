### Title
`batch_insert` Bypasses Duplicate-Key and Duplicate-Hash Guards for All But the Last Two Batch Items, Enabling DataLayer Merkle Tree Corruption — (File: `crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::batch_insert` applies the duplicate-key (`KeyAlreadyPresent`) and duplicate-hash (`HashAlreadyPresent`) guards only to the **last two** items in the supplied batch (which are routed through the regular `insert` path). Every earlier item is written directly to the blob via `insert_entry_to_blob` with **no uniqueness check**. An attacker who controls the batch input can supply duplicate `KeyId` or `Hash` values, silently corrupting the Merkle tree and producing an incorrect root hash or forged inclusion/exclusion proofs.

---

### Finding Description

`MerkleBlob::insert` enforces two prerequisite guards before writing a leaf:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  L369-374
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`batch_insert` pops the **last two** items from the vector and routes them through `self.insert()`, which performs those checks. All remaining items (indices `0 … n-3`) are written directly:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  L578-603
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else { return Ok(()); };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;  // ← guarded
    }
}

for ((key, value), hash) in keys_values_hashes {          // ← remaining items
    let new_leaf_index = self.get_new_index();
    let new_block = Block { … LeafNode { hash, key, value, … } … };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?; // ← NO duplicate check
    indexes.push(new_leaf_index);
}
``` [2](#0-1) 

When a duplicate `KeyId` is written this way, `insert_entry_to_blob` updates `block_status_cache.key_to_index` to point to the new index, orphaning the original leaf node in the blob. The orphaned node is no longer reachable by key lookup but still occupies a slot in the tree, making the blob's structural invariants inconsistent. When `calculate_lazy_hashes` subsequently recomputes internal hashes bottom-up, it incorporates the orphaned leaf's hash, producing a root hash that does not correspond to the logical key-value set. A duplicate `Hash` similarly corrupts `leaf_hash_to_index`.

The Python binding `py_batch_insert` only validates that the lengths of the two input lists match; it performs no deduplication:

```python
# wheel/python/chia_rs/datalayer.pyi  L331
def batch_insert(self, keys_values: list[tuple[KeyId, ValueId]], hashes: list[bytes32]): ...
``` [3](#0-2) 

---

### Impact Explanation

A corrupted `MerkleBlob` produces an incorrect root hash. Any `ProofOfInclusion` generated from the corrupted tree will carry a `root_hash()` that does not match the true committed state:

```rust
// crates/chia-datalayer/src/merkle/proof_of_inclusion.rs  L32-58
pub fn root_hash(&self) -> Hash { … }
pub fn valid(&self) -> bool { … }   // self-consistent but against wrong root
``` [4](#0-3) 

This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic corrupts tree roots or lets untrusted input prove invalid state.**

---

### Likelihood Explanation

`batch_insert` is the primary bulk-insertion path exposed through the Python wheel (`py_batch_insert`) and used by the DataLayer sync protocol. Any DataLayer publisher or subscriber that processes externally-sourced key-value batches is a reachable entry point. No privileged role is required; the attacker only needs to supply a batch vector with a repeated `KeyId` or `Hash` in a position other than the last two.

---

### Recommendation

Add the same duplicate guards to the fast path inside `batch_insert` before calling `insert_entry_to_blob`:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // … existing blob-write logic …
}
```

Alternatively, refactor `batch_insert` to call `self.insert(…, InsertLocation::Auto {})` for every item, accepting the minor performance cost in exchange for uniform invariant enforcement.

### Proof of Concept

```rust
let mut blob = MerkleBlob::new(vec![]).unwrap();

// Build a batch of 4 items where item[0] duplicates item[2]'s key.
// Items [2] and [3] (last two) go through insert() and are checked.
// Items [0] and [1] bypass the check entirely.
let dup_key = KeyId(42);
let batch = vec![
    ((dup_key,   ValueId(1)), Hash([0xAA; 32])),  // ← duplicate, bypasses check
    ((KeyId(99), ValueId(2)), Hash([0xBB; 32])),  // ← bypasses check
    ((KeyId(77), ValueId(3)), Hash([0xCC; 32])),  // ← goes through insert()
    ((dup_key,   ValueId(4)), Hash([0xDD; 32])),  // ← goes through insert(), accepted
];

// This succeeds without error, silently corrupting the tree.
blob.batch_insert(batch).unwrap();
blob.calculate_lazy_hashes().unwrap();

// check_integrity() will now fail or produce a root hash
// inconsistent with the logical key-value set.

    pub fn batch_insert(
        &mut self,
        mut keys_values_hashes: Vec<((KeyId, ValueId), Hash)>,
    ) -> Result<(), Error> {
        // OPT: perhaps go back to taking an iterator?
        // OPT: would it be worthwhile to hold the entire blocks?
        let mut indexes = vec![];

        if self.block_status_cache.leaf_count() <= 1 {
            for _ in 0..2 {
                let Some(((key, value), hash)) = keys_values_hashes.pop() else {
                    return Ok(());
                };
                self.insert(key, value, &hash, InsertLocation::Auto {})?;
            }
        }

        for ((key, value), hash) in keys_values_hashes {
            let new_leaf_index = self.get_new_index();
            let new_block = Block {
                metadata: NodeMetadata {
                    node_type: NodeType::Leaf,
                    dirty: false,
                },
                node: Node::Leaf(LeafNode {
                    parent: Parent(None),
                    hash,
                    key,
                    value,
                }),
            };
            self.insert_entry_to_blob(new_leaf_index, &new_block)?;
            indexes.push(new_leaf_index);
        }

        // OPT: can we insert the top node first?  maybe more efficient to update it's children
        //      than to update the parents of the children when traversing leaf to sub-root?
        while indexes.len() > 1 {
            let mut new_indexes = vec![];

            for chunk in indexes.chunks(2) {
                let [index_1, index_2] = match chunk {
                    [index] => {
                        new_indexes.push(*index);
                        continue;
                    }
                    [index_1, index_2] => [*index_1, *index_2],
                    _ => unreachable!(
                        "chunk should always be either one or two long and be handled above"
                    ),
                };

                let new_internal_node_index = self.get_new_index();

                let mut hashes = vec![];
                for index in [index_1, index_2] {
                    let block = self.update_parent(index, Some(new_internal_node_index))?;
                    hashes.push(block.node.hash());
                }

                let new_block = Block {
                    metadata: NodeMetadata {
                        node_type: NodeType::Internal,
                        dirty: false,
                    },
                    node: Node::Internal(InternalNode {
                        parent: Parent(None),
                        hash: internal_hash(&hashes[0], &hashes[1]),
                        left: index_1,
                        right: index_2,
                    }),
                };

                self.insert_entry_to_blob(new_internal_node_index, &new_block)?;
                new_indexes.push(new_internal_node_index);
            }

            indexes = new_indexes;
        }

        if indexes.len() == 1 {
            // OPT: can we avoid this extra min height leaf traversal?
            let min_height_leaf = self.get_min_height_leaf()?;
            self.insert_subtree_at_key(min_height_leaf.key, indexes[0], Side::Left)?;
        }

        Ok(())
    }

    #[pyo3(name = "batch_insert")]
    pub fn py_batch_insert(
        &mut self,
        keys_values: Vec<(KeyId, ValueId)>,
        hashes: Vec<Hash>,
    ) -> PyResult<()> {
        if keys_values.len() != hashes.len() {
            Err(Error::UnmatchedKeysAndValues(
                keys_values.len(),
                hashes.len(),
            ))?;
        }

        self.batch_insert(zip(keys_values, hashes).collect())?;

        Ok(())
    }

    pub fn root_hash(&self) -> Hash {
        if let Some(last) = self.layers.last() {
            last.combined_hash
        } else {
            self.node_hash
        }
    }

    pub fn valid(&self) -> bool {
        let mut existing_hash = self.node_hash;

        for layer in &self.layers {
            let calculated_hash = crate::calculate_internal_hash(
                &existing_hash,
                layer.other_hash_side,
                &layer.other_hash,
            );

            if calculated_hash != layer.combined_hash {
                return false;
            }

            existing_hash = calculated_hash;
        }

        existing_hash == self.root_hash()
    }
