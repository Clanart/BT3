### Title
Irrecoverable Token Lock When `ft_transfer` Fails Silently in `fin_transfer_send_tokens_callback` — (File: `near/omni-bridge/src/lib.rs`)

---

### Summary

In `fin_transfer_send_tokens_callback`, the helper `is_refund_required` unconditionally returns `false` when `is_ft_transfer_call` is `false`. This means that if the underlying `ft_transfer` (or `mint`) cross-contract call to the recipient fails, the bridge never detects the failure: it marks the transfer as finalized, pays the fee to the fee recipient, and emits `FinTransferEvent` — while the principal tokens are permanently stranded inside the bridge contract with no recovery path.

---

### Finding Description

`fin_transfer_send_tokens_callback` is the NEAR callback that runs after the bridge attempts to deliver tokens to the recipient on the destination (NEAR) side of a cross-chain transfer. [1](#0-0) 

The entire refund/revert decision is delegated to `is_refund_required`: [2](#0-1) 

When `is_ft_transfer_call` is `false` (i.e., a plain `ft_transfer` or `mint` was used — the normal path for transfers without a DeFi `msg`), the function **always returns `false`** without ever inspecting `env::promise_result_checked`. The promise result of the `ft_transfer` / `mint` call is completely ignored.

Consequently, if the token delivery fails — for example because:
- the recipient account has no storage deposit on the token contract (the most common case; the relayer/caller of `fin_transfer` is responsible for supplying `storage_deposit_actions` but there is no enforcement),
- the token contract is paused or the recipient is blacklisted,
- the `mint` call panics for any reason —

the callback still takes the **"success" branch**: [3](#0-2) 

The `fin_transfer` record was already inserted into `finalised_transfers` by `add_fin_transfer` earlier in `process_fin_transfer_to_near`: [4](#0-3) 

Because the nonce is permanently consumed in `finalised_transfers`, the transfer **cannot be retried**. The principal tokens that were supposed to reach the recipient are now stranded in the bridge contract (for native/locked tokens) or simply never minted (for bridge tokens), with no admin rescue function and no cancel/refund path for the originating chain.

---

### Impact Explanation

**Critical — Permanent freezing of user funds.**

- On the source chain (EVM, Solana, StarkNet), the user's tokens were already burned or locked when `initTransfer` was called. Those tokens cannot be recovered.
- On the destination (NEAR) side, the principal amount is either stuck inside the bridge contract (native tokens) or simply not minted (bridge tokens).
- The `finalised_transfers` bitmap/set permanently records the nonce as used, so no retry is possible.
- There is no admin escape hatch to recover stranded tokens.

The result is a complete, irrecoverable loss of the bridged amount for the user.

---

### Likelihood Explanation

**Medium.**

The most direct trigger requires no privileged access: `fin_transfer` is a public function callable by anyone. A caller who submits a valid proof but omits the required `storage_deposit_actions` for the recipient will cause `ft_transfer` to fail (NEAR requires a storage deposit before a token account can receive funds). Because the failure is silently swallowed, this can happen:

1. **Accidentally** — a relayer that does not pre-check recipient storage registration.
2. **Deliberately** — a griefing caller who finalizes the transfer without storage deposit actions, permanently destroying the user's funds.
3. **Externally triggered** — a token contract that is paused or a recipient that is blacklisted after the source-chain transaction was submitted.

The condition is not exotic; missing storage deposits are a routine operational issue on NEAR.

---

### Recommendation

1. **Check the promise result for plain `ft_transfer` as well.** `is_refund_required` should call `env::promise_result_checked` regardless of `is_ft_transfer_call` and treat a failed promise as requiring a refund/revert.
2. **On failure, revert lock actions and remove the `fin_transfer` record** (as is already done in the `ft_transfer_call` refund branch) so the transfer can be retried with correct storage deposit actions.
3. **Do not mark the nonce as finalized** (`add_fin_transfer`) until after token delivery is confirmed successful, or implement a two-phase commit that can be rolled back.

---

### Proof of Concept

1. Alice initiates a transfer of 1000 USDC from EVM to her NEAR account `alice.near` via `initTransfer` on EVM. Tokens are locked in the EVM bridge.
2. Bob (any caller) calls `fin_transfer` on the NEAR bridge with a valid proof but supplies an empty `storage_deposit_actions` array.
3. `process_fin_transfer_to_near` runs: the transfer nonce is recorded in `finalised_transfers`, then `send_tokens` calls `ft_transfer(alice.near, 1000)` on the USDC token contract.
4. `ft_transfer` fails because `alice.near` has no storage deposit on the USDC contract. NEAR delivers a `PromiseResult::Failed` to the callback.
5. `fin_transfer_send_tokens_callback` is invoked with `is_ft_transfer_call = false`. `is_refund_required(false)` returns `false` unconditionally — the failed promise result is never read.
6. The callback takes the success branch: fee (if any) is sent to the fee recipient, `FinTransferEvent` is emitted.
7. Alice's 1000 USDC are permanently stranded in the NEAR bridge contract. The EVM tokens remain locked. The nonce is consumed; no retry is possible. Alice has lost her funds. [5](#0-4) [6](#0-5)

    #[allow(clippy::needless_pass_by_value)]
    #[private]
    pub fn fin_transfer_send_tokens_callback(
        &mut self,
        #[serializer(borsh)] transfer_message: TransferMessage,
        #[serializer(borsh)] fee_recipient: &AccountId,
        #[serializer(borsh)] is_ft_transfer_call: bool,
        #[serializer(borsh)] storage_owner: &AccountId,
        #[serializer(borsh)] lock_actions: Vec<LockAction>,
    ) {
        let token = self.get_token_id(&transfer_message.token);

        if Self::is_refund_required(is_ft_transfer_call) {
            self.burn_tokens_if_needed(
                token.clone(),
                U128(
                    transfer_message
                        .amount_without_fee()
                        .near_expect(BridgeError::InvalidFee),
                ),
            );

            self.revert_lock_actions(&lock_actions);

            self.remove_fin_transfer(&transfer_message.get_transfer_id(), storage_owner);

            env::log_str(
                &OmniBridgeEvent::FailedFinTransferEvent { transfer_message }.to_log_string(),
            );
        } else {
            // Send fee to the fee recipient
            if transfer_message.fee.fee.0 > 0 {
                if self.is_deployed_token(&token) {
                    ext_token::ext(token)
                        .with_static_gas(MINT_TOKEN_GAS)
                        .mint(fee_recipient.clone(), transfer_message.fee.fee, None)
                        .detach();
                } else {
                    ext_token::ext(token)
                        .with_attached_deposit(ONE_YOCTO)
                        .with_static_gas(FT_TRANSFER_GAS)
                        .ft_transfer(fee_recipient.clone(), transfer_message.fee.fee, None)
                        .detach();
                }
            }

            if transfer_message.fee.native_fee.0 > 0 {
                let native_token_id = self.get_native_token_id(transfer_message.get_origin_chain());

                ext_token::ext(native_token_id)
                    .with_static_gas(MINT_TOKEN_GAS)
                    .mint(fee_recipient.clone(), transfer_message.fee.native_fee, None)
                    .detach();
            }

            env::log_str(&OmniBridgeEvent::FinTransferEvent { transfer_message }.to_log_string());
        }
    }

impl Contract {
    fn is_refund_required(is_ft_transfer_call: bool) -> bool {
        if is_ft_transfer_call {
            match env::promise_result_checked(0, MAX_FT_TRANSFER_CALL_RESULT) {
                Ok(value) => {
                    if let Ok(amount) = near_sdk::serde_json::from_slice::<U128>(&value) {
                        // Normal case: refund if the used token amount is zero
                        // The amount can be zero if the `ft_on_transfer` in the receiver contract returns an amount instead of `0`, or if it panics.
                        amount.0 == 0
                    } else {
                        // Unexpected case: don't refund
                        false
                    }
                }
                // Unexpected case: don't refund
                Err(_) => false,
            }
        } else {
            // Not ft_transfer_call: don't refund
            false
        }
    }
