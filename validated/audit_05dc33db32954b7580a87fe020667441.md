### Title
Unverified `old_block_hash` Written to Block Hash Mapping Contract Enables Sequencer to Forge `get_block_hash` Results — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/os_utils.cairo`)

---

### Summary

In `write_block_number_to_block_hash_mapping`, the value `old_block_hash` is obtained from a sequencer-controlled hint (`%{ GetOldBlockNumberAndHash %}`) and written directly to the block hash mapping contract state with **no Cairo-level constraint**. A malicious sequencer can supply any arbitrary value and still produce a valid proof. Contracts that call the `get_block_hash` syscall will receive the forged value, enabling fund theft or permanent fund freezing in contracts that rely on historical block hashes for security-critical logic.

---

### Finding Description

In `os_utils.cairo`, the function `write_block_number_to_block_hash_mapping` is responsible for recording the hash of block `N − STORED_BLOCK_HASH_BUFFER` (i.e., 10 blocks ago) into the dedicated block hash mapping contract at `BLOCK_HASH_CONTRACT_ADDRESS = 0x1`.

The critical section:

```cairo
// Currently, the block hash mapping is not enforced by the OS.
// TODO(Yoni, 1/1/2026): output this hash.
local old_block_hash;
%{ GetOldBlockNumberAndHash %}

// Update mapping.
assert state_entry.class_hash = 0;
assert state_entry.nonce = 0;
tempvar storage_ptr = state_entry.storage_ptr;
assert [storage_ptr] = DictAccess(key=old_block_number, prev_value=0, new_value=old_block_hash);
``` [1](#0-0) 

The variable `old_block_hash` is declared as a `local` and populated exclusively by the hint `GetOldBlockNumberAndHash`. There is **no subsequent Cairo assertion** that constrains its value — no hash recomputation, no range check, no comparison against any committed value. The code comment itself acknowledges this gap: *"Currently, the block hash mapping is not enforced by the OS."*

The value is then committed into the contract state dict as the authoritative block hash for `old_block_number`, which is later readable by any contract via the `get_block_hash` syscall.

The function that calls this mapping write is `pre_process_block`, invoked once per block before any transactions execute: [4](#0-3)

### Impact Explanation

StarkNet's ZK-proving model is designed so that even a malicious sequencer cannot produce an incorrect state transition that passes proof verification. The proof is the enforcement mechanism — hints are the sequencer's witness, and Cairo assertions are the constraints. When a hint-supplied value has **no constraining assertion**, the sequencer can freely choose it and the proof remains valid.

`old_block_hash` is written into the state of the block hash mapping contract. Any contract that calls `get_block_hash(block_number)` for a block in the range `[current − 10, current − 10]` will read this forged value from contract storage. Contracts that use historical block hashes for:

- **Commit-reveal randomness** (e.g., `seed = get_block_hash(N)`)
- **Replay protection** (e.g., `assert get_block_hash(N) == expected`)
- **Fund unlocking conditions** (e.g., `if get_block_hash(N) == commitment { transfer() }`)

…will operate on attacker-controlled data. The sequencer can steer the outcome of any such check, enabling **direct loss of funds** or **permanent freezing of funds** in affected contracts.

**Impact: Critical — Direct loss of funds / Permanent freezing of funds.**

---

### Likelihood Explanation

The sequencer must be malicious. However, in StarkNet's threat model, the proof is the mechanism that is supposed to prevent a malicious sequencer from producing incorrect state. The absence of a constraint on `old_block_hash` is a **protocol-level failure**: the sequencer can exploit it while producing a proof that is cryptographically valid. The TODO comment confirms this is an acknowledged gap, not an intentional design choice. Any sequencer operator who controls block production can exploit this silently, with no on-chain evidence of manipulation.

**Likelihood: Medium** — requires a malicious sequencer, but no additional capability beyond normal block production.

---

### Recommendation

The OS must constrain `old_block_hash` with a Cairo assertion. The planned fix is already noted in the TODO:

> `// TODO(Yoni, 1/1/2026): output this hash.`

The correct remediation is to include `old_block_hash` in the OS output so that the L1 verifier can check its consistency against the on-chain block hash record, or to recompute it inside the Cairo program from committed block data and assert equality. Until this is done, the block hash mapping is entirely sequencer-controlled and unverifiable by the proof.

### Proof of Concept

1. Sequencer prepares a block. During `pre_process_block`, `write_block_number_to_block_hash_mapping` is called.
2. The hint `GetOldBlockNumberAndHash` is executed. The sequencer's hint implementation returns `old_block_hash = 0xdeadbeef` (arbitrary forged value) for `old_block_number = current_block - 10`.
3. The Cairo program executes:
   ```cairo
   assert [storage_ptr] = DictAccess(key=old_block_number, prev_value=0, new_value=0xdeadbeef);
   ```
   No assertion constrains `0xdeadbeef`. The proof is generated and is valid.
4. The forged value `0xdeadbeef` is now the authoritative block hash for `old_block_number` in the block hash mapping contract.
5. A victim contract calls `get_block_hash(old_block_number)` and receives `0xdeadbeef`.
6. If the contract contains logic such as:
   ```
   let seed = get_block_hash(committed_block);
   let winner = seed % n_participants;
   transfer_prize(participants[winner]);
   ```
   The sequencer controls `seed`, controls `winner`, and can direct the prize to any address — **direct loss of funds**. [5](#0-4)

func write_block_number_to_block_hash_mapping{range_check_ptr, contract_state_changes: DictAccess*}(
    block_context: BlockContext*
) {
    alloc_locals;
    tempvar old_block_number = block_context.block_info_for_execute.block_number -
        STORED_BLOCK_HASH_BUFFER;
    let is_old_block_number_non_negative = is_nn(old_block_number);
    if (is_old_block_number_non_negative == FALSE) {
        // Not enough blocks in the system - nothing to write.
        return ();
    }

    // Fetch the (block number -> block hash) mapping contract state.
    local state_entry: StateEntry*;
    %{ GetBlockHashMapping %}

    // Currently, the block hash mapping is not enforced by the OS.
    // TODO(Yoni, 1/1/2026): output this hash.
    local old_block_hash;
    %{ GetOldBlockNumberAndHash %}

    // Update mapping.
    assert state_entry.class_hash = 0;
    assert state_entry.nonce = 0;
    tempvar storage_ptr = state_entry.storage_ptr;
    assert [storage_ptr] = DictAccess(key=old_block_number, prev_value=0, new_value=old_block_hash);
    let storage_ptr = storage_ptr + DictAccess.SIZE;
    %{ WriteOldBlockToStorage %}

    // Update contract state.
    tempvar new_state_entry = new StateEntry(class_hash=0, storage_ptr=storage_ptr, nonce=0);
    dict_update{dict_ptr=contract_state_changes}(
        key=BLOCK_HASH_CONTRACT_ADDRESS,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    return ();
