### Title
Missing Validation of Replacement Class Hash Against Declared Classes in `execute_replace_class` — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in both the non-deprecated (`syscall_impls.cairo`) and deprecated (`deprecated_execute_syscalls.cairo`) paths updates a contract's `StateEntry` with an attacker-supplied class hash without verifying that the hash is present in the `contract_class_changes` dictionary (the block-level registry of declared classes). This is the direct StarkNet analog of the external report's "stale/unvalidated global state on new instance creation" class: just as `createPair` failed to read the current `periodSizeUpdated` variable, `execute_replace_class` fails to read the current `contract_class_changes` state to validate the new class hash.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` reads the caller-supplied `class_hash` from the syscall request and immediately writes it into a new `StateEntry` for the contract:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment explicitly acknowledges the missing check. Its deadline (1 January 2026) has already passed as of today (3 July 2026), yet the check is absent. The identical omission exists in the deprecated path:

```cairo
func execute_replace_class{contract_state_changes: DictAccess*, revert_log: RevertLogEntry*}(
    contract_address, syscall_ptr: ReplaceClass*
) {
    ...
    let class_hash = syscall_ptr.class_hash;
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}
    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );
    dict_update{dict_ptr=contract_state_changes}(...);
```

No lookup into `contract_class_changes` is performed in either path. The OS therefore accepts a state transition where a contract's `class_hash` field is set to a value that has never been declared and does not exist in the class commitment tree.

The analog to the external report is exact:
- **External report**: `createPair` does not read the updated `periodSizeUpdated` variable → new pair carries the stale default.
- **This finding**: `execute_replace_class` does not read the current `contract_class_changes` dict → the contract's new class hash is never validated against the global declared-class state.

---

### Impact Explanation

After `replace_class` is accepted with an undeclared hash `H`:

1. The contract's leaf in the contract state Patricia tree is updated to commit to `H`.
2. `H` has no corresponding entry in the class commitment tree.
3. Every subsequent transaction targeting this contract requires the sequencer/prover to supply the compiled class for `H`; none exists.
4. The contract becomes permanently unexecutable.
5. All funds (ERC-20 balances, NFTs, protocol liquidity) held by or attributed to the contract are permanently frozen with no recovery path.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

The `replace_class` syscall is a standard, unprivileged operation callable by any Sierra contract from within its own execution context. No admin key, operator privilege, or external dependency is required. Any contract developer who deploys a contract holding third-party funds (e.g., a vault, AMM pool, or bridge escrow) can trigger this path in a single transaction. The attacker-controlled entry point is the `replace_class` syscall request field `class_hash`, which is read directly from user-supplied calldata with no sanitization.

---

### Recommendation

Before writing the new `StateEntry`, add a lookup into `contract_class_changes` to assert the replacement class hash has been declared in the current block or in a prior block's class tree:

```cairo
// Assert the class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

If the class was declared in a prior block (not in `contract_class_changes` for the current block), the hint layer must supply a Merkle proof of membership in the class commitment tree, and the OS must verify it before accepting the replacement.

---

### Proof of Concept

1. Attacker deploys contract `C` (e.g., a vault holding user funds). `C`'s `StateEntry.class_hash` is set to the declared hash `D`.
2. Attacker calls `replace_class(class_hash=0xDEAD)` from within `C`, where `0xDEAD` is an arbitrary value never declared.
3. `execute_replace_class` in `syscall_impls.cairo` (lines 896–910) reads `class_hash = 0xDEAD` from the request, skips the missing validation, and writes `new StateEntry(class_hash=0xDEAD, ...)` into `contract_state_changes`.
4. The OS accepts the block. The contract state tree now commits to `class_hash=0xDEAD` for address `C`.
5. The class commitment tree has no entry for `0xDEAD`.
6. All future `invoke` or `call_contract` syscalls targeting `C` require a compiled class for `0xDEAD`; none can be provided.
7. All funds in `C` are permanently frozen. [1](#0-0) [2](#0-1) [3](#0-2)
