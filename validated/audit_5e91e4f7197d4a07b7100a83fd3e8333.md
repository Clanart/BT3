### Title
`ProofOfInclusion::valid()` Has No External Root Hash Sanity Check, Accepting Forged DataLayer Inclusion Proofs — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` only checks the internal self-consistency of a proof object. It derives the "trusted" root from the proof itself (`self.layers.last().combined_hash`), making the final equality check tautological. Any attacker who can supply a crafted `ProofOfInclusion` — possible because the struct is `Streamable` and exposed via Python bindings — can produce a proof that passes `valid()` for an arbitrary `node_hash` without possessing any real tree membership.

### Finding Description

`ProofOfInclusion` is defined in `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs` and derives `Streamable`, making it deserializable from untrusted bytes. Its `valid()` method is:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()   // ← tautological
}
```

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← taken from the proof itself
    } else {
        self.node_hash
    }
}
```

Two concrete missing-check cases:

**Case 1 — Zero-layer proof (always `true`).**
When `layers` is empty, the loop body never executes and the final check becomes `self.node_hash == self.node_hash`, which is unconditionally `true`. Any `ProofOfInclusion { node_hash: H_fake, layers: [] }` passes `valid()` regardless of `H_fake`.

**Case 2 — Forged single-layer proof.**
An attacker picks any `node_hash = H_fake`, any `other_hash = H_other`, computes `combined_hash = calculate_internal_hash(H_fake, side, H_other)`, and constructs:
```
ProofOfInclusion {
    node_hash: H_fake,
    layers: [ProofOfInclusionLayer {
        other_hash_side: side,
        other_hash: H_other,
        combined_hash,   // attacker-computed
    }]
}
```
`valid()` returns `true` because the loop check passes and the final check is `combined_hash == combined_hash`.

The missing sanity check is: **`valid()` never compares the derived root against a caller-supplied trusted root hash.** The function name implies full proof validity, but it only checks internal arithmetic consistency of attacker-controlled fields.

### Impact Explanation

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."*

A DataLayer client that receives a `ProofOfInclusion` over the network, deserializes it (via `Streamable`), and calls `proof.valid()` as its sole verification step will accept forged membership claims for arbitrary key-value pairs. This allows an untrusted peer to assert that any data is present in a DataLayer store without it actually being there, corrupting the integrity guarantees of the DataLayer protocol.

### Likelihood Explanation

The `ProofOfInclusion` struct is exposed directly to Python via `#[pyclass]` and `py_valid()`, and is `Streamable`, so it is routinely serialized and sent between nodes. The `valid()` method is the only verification interface provided; there is no documented requirement for callers to separately check `root_hash()` against a trusted value. Any DataLayer consumer that follows the natural API usage pattern (`proof.valid()`) is vulnerable. Constructing a passing forged proof requires only arithmetic (one hash computation per layer) and no secret knowledge.

### Recommendation

`valid()` must accept a trusted root hash parameter and compare against it instead of deriving the root from the proof itself:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against external trusted root
}
```

The existing `valid()` (no-argument form) should either be removed or clearly documented as an internal-consistency-only check that **must not** be used as a security boundary. All call sites — including the Python binding `py_valid()` — should be updated to supply the trusted root.

### Proof of Concept

```python
from chia_rs import ProofOfInclusion, ProofOfInclusionLayer
import hashlib

# Forge proof of inclusion for an arbitrary node hash
fake_node_hash = bytes([0xAB] * 32)

# Zero-layer proof: always passes valid()
forged = ProofOfInclusion(node_hash=fake_node_hash, layers=[])
assert forged.valid()   # True — no real tree needed

# One-layer proof: attacker computes combined_hash themselves
other_hash = bytes([0xCD] * 32)
# calculate_internal_hash(left, right) = sha256(b'\x01' + left + right) or similar
# attacker just needs to know the hash function, which is public
combined = hashlib.sha256(b'\x00' + fake_node_hash + other_hash).digest()  # illustrative
layer = ProofOfInclusionLayer(other_hash_side=0, other_hash=other_hash, combined_hash=combined)
forged2 = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])
# valid() checks combined == calculate_internal_hash(fake_node_hash, side, other_hash)
# which the attacker pre-computed — passes trivially
```
