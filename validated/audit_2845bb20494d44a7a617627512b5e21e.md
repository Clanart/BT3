### Title
`From<CommitmentStateDiff> for ThinStateDiff` Always Drops `deprecated_declared_classes`, Corrupting `state_diff_commitment` and Global Root for Blocks Containing Deprecated Declare Transactions — (`crates/blockifier/src/state/cached_state.rs`)

---

### Summary

The `From<CommitmentStateDiff> for ThinStateDiff` conversion unconditionally sets `deprecated_declared_classes: Vec::new()`. Because the Rust batcher path (`BlockExecutionArtifacts::new`) uses this conversion to produce both the `ThinStateDiff` fed to `calculate_block_commitments` and the `ThinStateDiff` stored and forwarded to the committer, any block that includes a deprecated Declare transaction will have a structurally wrong `state_diff_commitment` in its block hash and a wrong global root derived from a Patricia trie that is missing the deprecated class entry.

---

### Finding Description

**Step 1 — `CommitmentStateDiff` has no field for deprecated declared classes.**

`CommitmentStateDiff` tracks only `address_to_class_hash`, `address_to_nonce`, `storage_updates`, and `class_hash_to_compiled_class_hash`. [1](#0-0) 

**Step 2 — `From<StateMaps> for CommitmentStateDiff` silently drops `declared_contracts`.**

`StateMaps` carries `declared_contracts: HashMap<ClassHash, bool>`, which is set to `true` for every declared class (including deprecated ones). The conversion to `CommitmentStateDiff` maps only `class_hashes`, `storage`, `compiled_class_hashes`, and `nonces` — `declared_contracts` is completely discarded. [2](#0-1) 

**Step 3 — `From<CommitmentStateDiff> for ThinStateDiff` hardcodes `deprecated_declared_classes: Vec::new()`.**

The TODO comment acknowledges this is a known structural gap. [3](#0-2) 

**Step 4 — `BlockExecutionArtifacts::new` uses this conversion for commitment calculation.**

`ThinStateDiff::from(commitment_state_diff.clone())` is passed directly to `calculate_block_commitments`, which spawns `calculate_state_diff_hash` on it. The resulting `state_diff_commitment` is stored in `partial_block_hash_components`. [4](#0-3) 

**Step 5 — `thin_state_diff()` also uses the same conversion.**

The `ThinStateDiff` returned by `thin_state_diff()` — which is stored in the DB and forwarded to the committer — is produced by the same `From<CommitmentStateDiff>` conversion, so it also has `deprecated_declared_classes: Vec::new()`. [5](#0-4) 

**Step 6 — `calculate_state_diff_hash` chains `deprecated_declared_classes` into the Poseidon hash.**

With an empty vec, the hash contribution is `chain(0)` (count = 0, no class hashes), which is wrong for any block containing a deprecated Declare transaction. [6](#0-5) 

**Step 7 — The committer's `verify_state_diff_hash` guard does NOT catch this.**

The committer recomputes `calculate_state_diff_hash(&state_diff)` on the same `ThinStateDiff` it receives. Since that `ThinStateDiff` also has `deprecated_declared_classes: Vec::new()`, both the provided commitment and the recalculated commitment are identically wrong — the check passes, and the wrong commitment is stored. [7](#0-6) 

**Step 8 — The committer builds the Patricia trie from the same wrong `ThinStateDiff`.**

`commit_state_diff` converts `ThinStateDiff` into the forest's `StateDiff`. With `deprecated_declared_classes` empty, the classes trie is not updated for the deprecated class, producing a wrong global root. [8](#0-7) 

---

### Impact Explanation

Any unprivileged user can submit a deprecated Declare (Cairo 0) transaction. If the Rust batcher includes it in a block:

- `state_diff_commitment` in the block hash is wrong (deprecated class hash count = 0 instead of ≥ 1).
- The classes Patricia trie leaf for the deprecated class is never written, so the global root is wrong.
- Both values are committed to storage and propagated to consensus/proof systems.

This satisfies the "Critical — Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input" and "Critical — Wrong state_diff_commitment committed to block hash, global root derived from wrong Patricia trie leaf" impact categories.

---

### Likelihood Explanation

Deprecated Declare transactions remain valid on Starknet. Any user can craft and submit one. The Rust batcher path (`BlockExecutionArtifacts::new`) is the production path for the new sequencer. The bug is triggered deterministically whenever such a transaction is included in a block.

---

### Recommendation

`CommitmentStateDiff` must be extended with a `deprecated_declared_classes: Vec<ClassHash>` field. `From<StateMaps> for CommitmentStateDiff` must populate it from `StateMaps.declared_contracts` (filtering entries where the value is `true` and `compiled_class_hashes` does not contain the key, i.e., Cairo 0 classes). `From<CommitmentStateDiff> for ThinStateDiff` must then propagate this field instead of hardcoding `Vec::new()`. The TODO comment at line 698–699 of `cached_state.rs` must be resolved before the Rust batcher path is used in production for any network that accepts deprecated Declare transactions.

---

### Proof of Concept

The structural proof is direct from the code:

1. Execute a deprecated Declare transaction through the Rust executor. `StateMaps.declared_contracts` will contain `(deprecated_class_hash, true)` and `StateMaps.compiled_class_hashes` will be empty for that hash.
2. Call `CommitmentStateDiff::from(state_maps)` — `deprecated_class_hash` is not present in any field of the result.
3. Call `ThinStateDiff::from(commitment_state_diff)` — `deprecated_declared_classes` is `vec![]`.
4. Call `calculate_state_diff_hash(&thin_state_diff)` — the hash chains `0` for the deprecated declared classes count.
5. Construct a reference `ThinStateDiff` with `deprecated_declared_classes: vec![deprecated_class_hash]` and call `calculate_state_diff_hash` on it — the hash is different.
6. Assert the two hashes differ — the assertion holds, confirming the corruption.

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
