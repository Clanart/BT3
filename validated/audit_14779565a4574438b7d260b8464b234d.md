### Title
Integer Division Truncation in `NewDynamicFeeChecker` Allows Zero-Fee Cosmos Transactions When `baseFee = 0` - (File: ante/evm/fee_checker.go)

### Summary

`NewDynamicFeeChecker` computes `feeCap` via integer division `fee / gas`. When a user submits a Cosmos tx with `fee < gas` (trivially achievable), `feeCap` truncates to zero. If `baseFee` is also zero — a reachable state on any chain with `MinGasPrice = 0` (the default) and consistently under-utilized blocks — the baseFee guard passes, `effectivePrice` collapses to zero, and the `DeductFeeDecorator` deducts nothing. The user's submitted fee stays in their account and the transaction executes for free.

### Finding Description

In `ante/evm/fee_checker.go`, `NewDynamicFeeChecker` derives the per-gas price for a Cosmos tx as:

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))   // integer division, truncates toward zero
``` [1](#0-0) 

When `fee < gas` (e.g., `fee = 1 aevmos`, `gas = 200_000`), `feeCap` becomes `0`. The only guard against this is:

```go
if feeCap.LT(baseFeeInt) {
    return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, ...)
}
``` [2](#0-1) 

When `baseFee = 0`, the comparison is `0 < 0 = false`, so the check silently passes. The effective price is then:

```go
effectivePrice := sdkmath.NewIntFromBigInt(types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
// EffectiveGasPrice = min(tipCap + baseFee, feeCap) = min(maxPriorityPrice + 0, 0) = 0
``` [3](#0-2) 

`effectiveFee = 0 * gas = 0` is returned to `DeductFeeDecorator`, which deducts nothing.

**When can `baseFee = 0`?**

`CalculateBaseFee` decreases the base fee each block when gas used is below target, floored at `minGasPrice`:

```go
return ethermint.BigMax(x.Sub(parentBaseFee, baseFeeDelta), minGasPrice)
``` [4](#0-3) 

The default `MinGasPrice = 0`: [5](#0-4) 

With `MinGasPrice = 0`, the base fee can decrease all the way to `0` on any chain with consistently under-utilized blocks. Once at zero it stays there. This is confirmed by the test case `{"enable london HF, not enable feemarket", true, false, big.NewInt(0)}`. [6](#0-5) 

The `MinGasPriceDecorator` that runs before `DeductFeeDecorator` short-circuits immediately when `MinGasPrice = 0`:

```go
if minGasPrice.IsZero() || simulate {
    return next(ctx, tx, simulate)
}
``` [7](#0-6) 

So there is no secondary guard. The `NewDynamicFeeChecker` is the only fee enforcement for Cosmos txs, and it is bypassed entirely.

`NewDynamicFeeChecker` is wired into both `newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712` with `DynamicFeeChecker: true` in the production app: [8](#0-7) [9](#0-8) 

### Impact Explanation

Any unprivileged user can submit Cosmos transactions (governance votes, staking, IBC transfers, authz, etc.) with zero effective fee deducted from their account. This is a direct mis-accounting of user fees in the ante handler: the chain's fee collector receives nothing, validators are not compensated, and the economic spam-prevention mechanism is completely bypassed. This matches the allowed High impact: *"ante handler… bug that permits… valid user funds/fees to be mis-accounted."*

### Likelihood Explanation

The precondition `baseFee = 0` is reachable on any Ethermint chain that:
1. Uses the default `MinGasPrice = 0` (the documented default, disabled by design).
2. Has blocks that are consistently below the gas target — common on new, low-activity, or testnet deployments.

The attacker-controlled inputs are only the `fee` and `gas` fields of a standard Cosmos tx. No privileged access, validator collusion, or key compromise is required.

### Recommendation

Add an explicit zero-feeCap guard before proceeding with the EIP-1559 effective price calculation:

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
if feeCap.IsZero() && !fee.IsZero() {
    return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee,
        "fee too small for gas limit: fee %s, gas %d", fee, gas)
}
```

Additionally, enforce a non-zero `MinGasPrice` as the default, or add a chain-level minimum `feeCap` floor so that `baseFee = 0` does not open a free-transaction window.

### Proof of Concept

1. Deploy an Ethermint chain with default params (`MinGasPrice = 0`, `NoBaseFee = false`, London HF enabled).
2. Submit empty blocks until `baseFee` decreases to `0` (or set `BaseFee = 0` in genesis for immediate reproduction).
3. Craft a Cosmos tx (e.g., a `MsgDelegate`) with `fee = 1 aevmos` and `gas = 200_000`.
4. `feeCap = 1 / 200_000 = 0` (integer truncation).
5. `feeCap.LT(baseFeeInt)` → `0 < 0` → `false`, guard passes.
6. `effectivePrice = min(MaxInt64 + 0, 0) = 0`.
7. `effectiveFee = sdk.Coins{{Amount: 0}}` returned to `DeductFeeDecorator`.
8. `DeductFeeDecorator` deducts `0`; the user's `1 aevmos` is never taken.
9. The transaction is included in the block with zero fee paid.
