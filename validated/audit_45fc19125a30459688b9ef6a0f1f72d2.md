### Title
Felt Arithmetic Overflow in `execute_keccak` Gas Calculation via Missing Pointer Bounds Check Enables Unbounded VM Execution — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary

The `execute_keccak` syscall handler in the StarkNet OS does not validate that `input_end >= input_start` before computing the keccak input length. An attacker-controlled contract can supply `input_end < input_start` (as integers), causing `len = input_end - input_start` to wrap around to a value near the field prime P in felt arithmetic. The subsequent gas calculation `required_gas = KECCAK_GAS_COST + q * KECCAK_ROUND_COST_GAS_COST` is performed in felt arithmetic and can overflow back to a small value, bypassing the gas guard. `keccak_padded_input` is then invoked with `n_blocks ≈ P / KECCAK_FULL_RATE_IN_WORDS`, an astronomically large number, exhausting the Cairo VM's step budget and preventing the OS from producing a valid proof for the block.

### Finding Description

In `execute_keccak` (lines 919–982 of `syscall_impls.cairo`):

```cairo
tempvar input_start = request.input_start;
tempvar input_end   = request.input_end;
let len = input_end - input_start;                          // (1) no bounds check
let (local q, r) = unsigned_div_rem(len, KECCAK_FULL_RATE_IN_WORDS);
``` [1](#0-0) 

There is no assertion that `input_end >= input_start`. When `input_end < input_start` as integers, felt subtraction yields `len = P - (input_start - input_end)`, a value close to the field prime P ≈ 2^251.

The divisibility guard:

```cairo
if (r != 0) {
    write_failure_response(..., failure_felt=ERROR_INVALID_INPUT_LEN);
    return ();
}
``` [2](#0-1) 

only rejects inputs where `len mod KECCAK_FULL_RATE_IN_WORDS ≠ 0`. The attacker can trivially satisfy this by choosing `input_start - input_end = (P mod KECCAK_FULL_RATE_IN_WORDS)` so that `len` is divisible by `KECCAK_FULL_RATE_IN_WORDS` (17 words). For StarkNet's prime P, `P mod 17 = 9`, so setting `input_start = 9, input_end = 0` gives `len = P - 9`, which is divisible by 17.
