### Title
Missing `shard_id` Binding Check in `set_state_header` Allows Malicious Peer to Install Wrong Shard State During State Sync — (`File: chain/chain/src/state_sync/adapter.rs`)

---

### Summary

`compute_state_response_header` (the server-side state-sync header generator) validates that the requested `shard_id` belongs to the epoch's shard layout before producing a `ShardStateSyncResponseHeader`. The receiver-side counterpart `set_state_header` — which accepts and persists that header — performs no such check. It also never verifies that the chunk embedded in the received `ShardStateSyncResponseHeader` actually belongs to the caller-supplied `shard_id`. A malicious peer can supply a cryptographically valid chunk for shard X (with its correct merkle proof) while claiming it is the state header for shard Y. All existing checks pass, and the syncing node stores shard X's state root under shard Y's storage key, causing it to apply the wrong trie state for shard Y.

---

### Finding Description

**The paired check that exists in the generator but is absent in the receiver:**

`compute_state_response_header` explicitly validates the `shard_id` against the epoch's shard layout:

```rust
// chain/chain/src/state_sync/adapter.rs  lines 82-84
let shard_ids = self.epoch_manager.shard_ids(sync_block_epoch_id)?;
if !shard_ids.contains(&shard_id) {
    return Err(shard_id_out_of_bounds(shard_id));
}
``` [1](#0-0) 

`set_state_header` — the function that accepts a peer-supplied `ShardStateSyncResponseHeader` and writes it to `DBCol::StateHeaders` — contains no equivalent check:

```rust
// chain/chain/src/state_sync/adapter.rs  lines 368-529

    &self,
    shard_id: ShardId,          // caller-controlled
    sync_hash: CryptoHash,      // caller-controlled

    let chunk = shard_state_header.cloned_chunk();

    // ❌ No check: shard_ids.contains(&shard_id)
    // ❌ No check: chunk.shard_id() == shard_id

    store_update.set_ser(DBCol::StateHeaders, &key, &shard_state_header);

``` [2](#0-1) 

**Why the existing merkle-path check does not close the gap:**

Step 3a verifies that the chunk is *somewhere* in the block's `chunk_headers_root`:

```rust
if !verify_path(
    *sync_prev_block_header.chunk_headers_root(),
    shard_state_header.chunk_proof(),
    &ChunkHashHeight(chunk.chunk_hash().clone(), chunk.height_included()),
) { ... }
``` [3](#0-2) 

The proof is position-specific (it encodes the path from root to leaf), but the verifier never checks *which* position is expected. The `shard_id` parameter is not used in this check at all. A malicious peer can supply shard 0's chunk together with its correct position-0 proof; `verify_path` passes because the chunk genuinely is in the block. The `shard_id` parameter (e.g., 1) is then used only as the storage key and in receipt hashing — not to validate the chunk's identity.

**The `shard_id` parameter is used in two places after the unchecked acceptance:**

1. Receipt hash computation — `CryptoHash::hash_borsh(ReceiptList(shard_id, receipts))` — uses the caller-supplied `shard_id`, not the chunk's actual shard.
2. The storage key — `StateHeaderKey(shard_id, sync_hash)` — stores the wrong header under the wrong shard's slot. [4](#0-3) [5](#0-4) 

**Downstream propagation through `set_state_part` and `set_state_finalize`:**

`set_state_part` reads the stored header to obtain the `state_root` it validates parts against:

```rust

``` [6](#0-5) 

`set_state_finalize` then applies those parts to shard Y's trie slot, completing the state corruption: [7](#0-6) 

---

### Impact Explanation

A syncing node that accepts a crafted `ShardStateSyncResponseHeader` from a malicious peer will:

1. Store shard X's `prev_state_root` under shard Y's `StateHeaderKey`.
2. Download and validate state parts against shard X's state root (all parts pass because they are genuinely valid for shard X).
3. Apply shard X's full trie state into shard Y's trie prefix via `apply_state_part` / `set_state_finalize`.

After state sync completes, the node holds an incorrect state for shard Y. Every subsequent block it processes for shard Y will produce a wrong `ChunkExtra.state_root`, causing the node to diverge from the canonical chain. If the node is a chunk producer for shard Y, it will produce chunks with an invalid state root, which will be rejected by the network and may trigger slashing.

---

### Likelihood Explanation

State sync is exercised by every node that bootstraps or falls behind. During state sync the node opens Tier-3 connections to peers it has explicitly requested state from. Any peer in the network that is reachable and willing to serve state can be malicious. No privileged role is required — any ordinary network participant can serve a crafted `ShardStateSyncResponseHeader`. The attack requires only that the malicious peer be selected as the state-sync source for the target shard, which is a normal outcome of peer selection.

---

### Recommendation

Add two guards at the top of `set_state_header`, mirroring the checks already present in `compute_state_response_header`:

```rust
pub fn set_state_header(
    &self,
    shard_id: ShardId,
    sync_hash: CryptoHash,
    shard_state_header: ShardStateSyncResponseHeader,
) -> Result<(), Error> {
    let sync_block_header = self.chain_store.get_block_header(&sync_hash)?;

    // Guard 1: shard_id must be valid for the epoch of sync_hash
    let epoch_id = sync_block_header.epoch_id();
    let shard_ids = self.epoch_manager.shard_ids(epoch_id)?;
    if !shard_ids.contains(&shard_id) {
        return Err(shard_id_out_of_bounds(shard_id));
    }

    let chunk = shard_state_header.cloned_chunk();

    // Guard 2: the chunk in the header must belong to the requested shard
    if chunk.shard_id() != shard_id {
        byzantine_assert!(false);
        return Err(Error::Other(
            "set_shard_state failed: chunk shard_id does not match requested shard_id".into(),
        ));
    }
    // ... rest of existing validation ...
}
``` [8](#0-7)

### Proof of Concept

1. Network has 2 shards (shard 0, shard 1) at `sync_hash`.
2. Syncing node calls `set_state_header(shard_id=1, sync_hash, crafted_header)`.
3. Malicious peer constructs `crafted_header` containing:
   - `chunk` = the genuine `ShardChunk` for shard 0 (obtained from the block at `sync_hash - 1`)
   - `chunk_proof` = the correct merkle proof for shard 0's position in `chunk_headers_root`
   - `incoming_receipts_proofs` = empty (no incoming receipts for shard 0 in this epoch)
   - `state_root_node` = shard 0's genuine `StateRootNode`
4. `validate_chunk_proofs` passes (shard 0's chunk is internally consistent).
5. `verify_path(chunk_headers_root, proof_for_position_0, ChunkHashHeight(shard0_hash, ...))` passes — shard 0's chunk IS in the block.
6. Receipt proof loop passes (empty receipts, no iterations).
7. `validate_state_root_node` passes (shard 0's node is valid for shard 0's state root).
8. Header is committed to `DBCol::StateHeaders` under key `StateHeaderKey(shard_1, sync_hash)`.
9. Syncing node subsequently downloads and applies shard 0's state parts, validated against shard 0's state root, into shard 1's trie prefix.
10. After `set_state_finalize`, shard 1's trie contains shard 0's state. All future block processing for shard 1 produces wrong state roots. [9](#0-8) [10](#0-9)

    pub fn set_state_header(
        &self,
        shard_id: ShardId,
        sync_hash: CryptoHash,
        shard_state_header: ShardStateSyncResponseHeader,
    ) -> Result<(), Error> {
        let sync_block_header = self.chain_store.get_block_header(&sync_hash)?;

        let chunk = shard_state_header.cloned_chunk();
        let prev_chunk_header = shard_state_header.cloned_prev_chunk_header();

        // 1-2. Checking chunk validity
        if !validate_chunk_proofs(&chunk, self.epoch_manager.as_ref())? {
            byzantine_assert!(false);
            return Err(Error::Other(
                "set_shard_state failed: chunk header proofs are invalid".into(),
            ));
        }

        // Consider chunk itself is valid.

        // 3. Checking that chunks `chunk` and `prev_chunk` are included in appropriate blocks
        // 3a. Checking that chunk `chunk` is included into block at last height before sync_hash
        // 3aa. Also checking chunk.height_included
        let sync_prev_block_header =
            self.chain_store.get_block_header(sync_block_header.prev_hash())?;
        if !verify_path(
            *sync_prev_block_header.chunk_headers_root(),
            shard_state_header.chunk_proof(),
            &ChunkHashHeight(chunk.chunk_hash().clone(), chunk.height_included()),
        ) {
            byzantine_assert!(false);
            return Err(Error::Other(
                "set_shard_state failed: chunk isn't included into block".into(),
            ));
        }

        let block_header = get_block_header_on_chain_by_height(
            &self.chain_store,
            &sync_hash,
            chunk.height_included(),
        )?;
        // 3b. Checking that chunk `prev_chunk` is included into block at height before chunk.height_included
        // 3ba. Also checking prev_chunk.height_included - it's important for getting correct incoming receipts
        match (&prev_chunk_header, shard_state_header.prev_chunk_proof()) {
            (Some(prev_chunk_header), Some(prev_chunk_proof)) => {
                let prev_block_header =
                    self.chain_store.get_block_header(block_header.prev_hash())?;
                if !verify_path(
                    *prev_block_header.chunk_headers_root(),
                    prev_chunk_proof,
                    &ChunkHashHeight(prev_chunk_header.chunk_hash().clone(), prev_chunk_header.height_included()),
                ) {
                    byzantine_assert!(false);
                    return Err(Error::Other(
                        "set_shard_state failed: prev_chunk isn't included into block".into(),
                    ));
                }
            }
            (None, None) => {
                if chunk.height_included() != 0 {
                    return Err(Error::Other(
                    "set_shard_state failed: received empty state response for a chunk that is not at height 0".into()
                ));
                }
            }
            _ =>
                return Err(Error::Other("set_shard_state failed: `prev_chunk_header` and `prev_chunk_proof` must either both be present or both absent".into()))
        };

        // 4. Proving incoming receipts validity
        // 4a. Checking len of proofs
        if shard_state_header.root_proofs().len()
            != shard_state_header.incoming_receipts_proofs().len()
        {
            byzantine_assert!(false);
            return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
        }
        let mut hash_to_compare = sync_hash;
        for (i, receipt_response) in
            shard_state_header.incoming_receipts_proofs().iter().enumerate()
        {
            let ReceiptProofResponse(block_hash, receipt_proofs) = receipt_response;

            // 4b. Checking that there is a valid sequence of continuous blocks
            if *block_hash != hash_to_compare {
                byzantine_assert!(false);
                return Err(Error::Other(
                    "set_shard_state failed: invalid incoming receipts".into(),
                ));
            }
            let header = self.chain_store.get_block_header(&hash_to_compare)?;
            hash_to_compare = *header.prev_hash();

            let block_header = self.chain_store.get_block_header(block_hash)?;
            // 4c. Checking len of receipt_proofs for current block
            if receipt_proofs.len() != shard_state_header.root_proofs()[i].len()
                || receipt_proofs.len() != block_header.chunks_included() as usize
            {
                byzantine_assert!(false);
                return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
            }
            // We know there were exactly `block_header.chunks_included` chunks included
            // on the height of block `block_hash`.
            // There were no other proofs except for included chunks.
            // According to Pigeonhole principle, it's enough to ensure all receipt_proofs are distinct
            // to prove that all receipts were received and no receipts were hidden.
            let mut visited_shard_ids = HashSet::<ShardId>::new();
            for (j, receipt_proof) in receipt_proofs.iter().enumerate() {
                let ReceiptProof(receipts, shard_proof) = receipt_proof;
                let ShardProof { from_shard_id, to_shard_id: _, proof } = shard_proof;
                // 4d. Checking uniqueness for set of `from_shard_id`
                match visited_shard_ids.get(from_shard_id) {
                    Some(_) => {
                        byzantine_assert!(false);
                        return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
                    }
                    _ => visited_shard_ids.insert(*from_shard_id),
                };
                let RootProof(root, block_proof) = &shard_state_header.root_proofs()[i][j];
                let receipts_hash = CryptoHash::hash_borsh(ReceiptList(shard_id, receipts));
                // 4e. Proving the set of receipts is the subset of outgoing_receipts of shard `shard_id`
                if !verify_path(*root, proof, &receipts_hash) {
                    byzantine_assert!(false);
                    return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
                }
                // 4f. Proving the outgoing_receipts_root matches that in the block
                if !verify_path(
                    *block_header.prev_chunk_outgoing_receipts_root(),
                    block_proof,
                    root,
                ) {
                    byzantine_assert!(false);
                    return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
                }
            }
        }
        // 4g. Checking that there are no more heights to get incoming_receipts
        let header = self.chain_store.get_block_header(&hash_to_compare)?;
        if header.height() != prev_chunk_header.map_or(0, |h| h.height_included()) {
            byzantine_assert!(false);
            return Err(Error::Other("set_shard_state failed: invalid incoming receipts".into()));
        }

        // 5. Checking that state_root_node is valid
        let chunk_inner = chunk.take_header().take_inner();
        if matches!(
            self.runtime_adapter.validate_state_root_node(
                shard_state_header.state_root_node(),
                chunk_inner.prev_state_root(),
            ),
            StateRootNodeValidationResult::Invalid
        ) {
            byzantine_assert!(false);
            return Err(Error::Other("set_shard_state failed: state_root_node is invalid".into()));
        }

        // Saving the header data.
        let mut store_update = self.chain_store.store().store_update();
        let key = borsh::to_vec(&StateHeaderKey(shard_id, sync_hash)).unwrap();
        store_update.set_ser(DBCol::StateHeaders, &key, &shard_state_header);
        store_update.commit();

        Ok(())
    }
