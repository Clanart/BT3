### Title
`get_da_gas_cost` Applies Fee-Balance Discount Unconditionally to L1Handler Transactions — (`File: crates/blockifier/src/fee/gas_usage.rs`)

### Summary

`get_da_gas_cost` contains a hardcoded discount for the fee-token balance storage value being cheaper to encode (12 non-zero bytes). This discount is applied to **every** call regardless of transaction type. For `L1Handler` transactions — which have no fee transfer and no sender-balance storage update — the discount is applied to a state-changes count that never includes the fee-balance entry, producing an underestimated DA gas cost in calldata mode (`use_kzg_da = false`).

### Finding Description

`get_da_gas_cost` in `crates/blockifier/src/fee/gas_usage.rs` computes the L1 data-availability gas cost from a `StateChangesCount`. In the calldata branch it applies two discounts: one proportional to `n_modified_contracts`, and one unconditional flat discount for the fee-balance storage value:

```rust
// Up to balance of 8*(10**10) ETH.
let fee_balance_value_cost = eth_gas_constants::get_calldata_word_cost(12);
discount += eth_gas_constants::GAS_PER_MEMORY_WORD - fee_balance_value_cost;
``` [1](#0-0) 

The discount value is `GAS_PER_MEMORY_WORD − get_calldata_word_cost(12) = 512 − 272 = 240` gas units. [2](#0-1) 

For **account transactions**, `StateResources::new` is called with `sender_address: Some(...)`, which causes `count_for_fee_charge` to add `+1` to `n_storage_updates` for the sender's fee-token balance. The discount correctly offsets the cheaper encoding of that value. [3](#0-2) 

For **L1Handler transactions**, `TransactionReceipt::from_l1_handler` passes `sender_address: None`: [4](#0-3) 

No fee-balance storage update is added to `n_storage_updates`. The `naive_cost` is therefore computed without the fee-balance entry, yet the 240-unit discount is still subtracted unconditionally, underestimating the DA gas cost by exactly 240 gas units for every L1Handler transaction that has any storage updates.

### Impact Explanation

The underestimated `receipt.gas.l1_gas` flows into two places:

1. **Gas-bounds enforcement** — `FeeCheckReport::check_all_gas_amounts_within_bounds(&l1_handler_bounds, &receipt.gas)` in `l1_handler_transaction.rs` compares the computed gas against the protocol-level `l1_handler_max_amount_bounds`. An L1Handler whose true L1 gas consumption exceeds the bound by up to 240 units will pass the check and be committed to the block. [5](#0-4) 

2. **RPC fee estimation / execution resources** — `receipt.gas` is the authoritative gas vector returned to callers of `starknet_estimateFee` and stored in the transaction receipt. The reported `l1_gas` is 240 units lower than the true cost, constituting an authoritative-looking wrong value.

The `receipt.fee` for L1Handler transactions is forced to `Fee(0)` after the fact, so the fee itself is unaffected; the error is confined to the gas vector and the bounds check. [6](#0-5) 

### Likelihood Explanation

The bug is active whenever `use_kzg_da = false` (calldata DA mode). Any L1Handler transaction that writes at least one storage cell triggers the underestimation. The magnitude is fixed at 240 gas units, which is small relative to typical `l1_handler_max_amount_bounds`, so the practical window for bypassing the bounds check is narrow. In KZG/blob mode (`use_kzg_da = true`) the discount branch is not taken and the bug is dormant.

### Recommendation

Pass a boolean (or the transaction type) into `get_da_gas_cost` indicating whether a fee-balance storage update is present, and apply the flat discount only when it is:

```rust
if has_fee_balance_update {
    let fee_balance_value_cost = eth_gas_constants::get_calldata_word_cost(12);
    discount += eth_gas_constants::GAS_PER_MEMORY_WORD - fee_balance_value_cost;
}
```

Alternatively, derive the discount from the actual `n_storage_updates` count rather than a hardcoded assumption, consistent with how the `modified_contract_discount` is already scaled by `n_modified_contracts`.

### Proof of Concept

1. Deploy a contract on L1 that emits a `LogMessageToL2` event targeting a Starknet contract with a storage-writing handler.
2. In calldata mode (`use_kzg_da = false`), execute the resulting L1Handler transaction via the blockifier.
3. Observe that `receipt.gas.l1_gas` is 240 units lower than the value computed by manually applying `get_da_gas_cost` with the correct (no-fee-balance) `StateChangesCount`.
4. Craft the L1 message so the handler's true L1 gas consumption equals `l1_handler_max_amount_bounds.l1_gas + k` for `1 ≤ k ≤ 240`; the bounds check passes and the transaction is committed despite exceeding the protocol limit. [7](#0-6) [8](#0-7)

/// Returns the gas cost of data availability on L1.
pub fn get_da_gas_cost(state_changes_count: &StateChangesCount, use_kzg_da: bool) -> GasVector {
    let onchain_data_segment_length = get_onchain_data_segment_length(state_changes_count);

    let (l1_gas, blob_gas) = if use_kzg_da {
        (
            0_u8.into(),
            u64_from_usize(
                onchain_data_segment_length * eth_gas_constants::DATA_GAS_PER_FIELD_ELEMENT,
            )
            .into(),
        )
    } else {
        // TODO(Yoni, 1/5/2024): count the exact amount of nonzero bytes for each DA entry.
        let naive_cost = onchain_data_segment_length * eth_gas_constants::SHARP_GAS_PER_DA_WORD;

        // For each modified contract, the expected non-zeros bytes in the second word are:
        // 1 bytes for class hash flag; 2 for number of storage updates (up to 64K);
        // 3 for nonce update (up to 16M).
        let modified_contract_cost = eth_gas_constants::get_calldata_word_cost(1 + 2 + 3);
        let modified_contract_discount =
            eth_gas_constants::GAS_PER_MEMORY_WORD - modified_contract_cost;
        let mut discount = state_changes_count.n_modified_contracts * modified_contract_discount;

        // Up to balance of 8*(10**10) ETH.
        let fee_balance_value_cost = eth_gas_constants::get_calldata_word_cost(12);
        discount += eth_gas_constants::GAS_PER_MEMORY_WORD - fee_balance_value_cost;

        // Cost must be non-negative after discount.
        let gas = naive_cost.saturating_sub(discount);

        (u64_from_usize(gas).into(), 0_u8.into())
    };

    GasVector { l1_gas, l1_data_gas: blob_gas, ..Default::default() }
}

// Calldata.
pub const GAS_PER_MEMORY_ZERO_BYTE: usize = 4;
pub const GAS_PER_MEMORY_BYTE: usize = 16;
pub const WORD_WIDTH: usize = 32;
pub const GAS_PER_MEMORY_WORD: usize = GAS_PER_MEMORY_BYTE * WORD_WIDTH;

// Blob Data.
pub const FIELD_ELEMENTS_PER_BLOB: usize = 1 << 12;
pub const DATA_GAS_PER_BLOB: usize = 1 << 17;
pub const DATA_GAS_PER_FIELD_ELEMENT: usize = DATA_GAS_PER_BLOB / FIELD_ELEMENTS_PER_BLOB;

// Storage.
pub const GAS_PER_ZERO_TO_NONZERO_STORAGE_SET: usize = 20000;
pub const GAS_PER_COLD_STORAGE_ACCESS: usize = 2100;
pub const GAS_PER_NONZERO_TO_INT_STORAGE_SET: usize = 2900;
pub const GAS_PER_COUNTER_DECREASE: usize =
    GAS_PER_COLD_STORAGE_ACCESS + GAS_PER_NONZERO_TO_INT_STORAGE_SET;

// Events.
pub const GAS_PER_LOG: usize = 375;
pub const GAS_PER_LOG_TOPIC: usize = 375;
pub const GAS_PER_LOG_DATA_BYTE: usize = 8;
pub const GAS_PER_LOG_DATA_WORD: usize = GAS_PER_LOG_DATA_BYTE * WORD_WIDTH;

// SHARP empirical costs.
pub const SHARP_ADDITIONAL_GAS_PER_MEMORY_WORD: usize = 100; // This value is not accurate.
pub const SHARP_GAS_PER_MEMORY_WORD: usize =
    GAS_PER_MEMORY_WORD + SHARP_ADDITIONAL_GAS_PER_MEMORY_WORD;
// 10% discount for data availability.
pub const DISCOUNT_PER_DA_WORD: usize = (SHARP_GAS_PER_MEMORY_WORD * 10) / 100;
pub const SHARP_GAS_PER_DA_WORD: usize = SHARP_GAS_PER_MEMORY_WORD - DISCOUNT_PER_DA_WORD;

// TODO(Yoni, 1/1/2025): rename this file to `_utils`.
pub fn get_calldata_word_cost(n_nonzero_bytes: usize) -> usize {
    n_nonzero_bytes * GAS_PER_MEMORY_BYTE
        + (WORD_WIDTH - n_nonzero_bytes) * GAS_PER_MEMORY_ZERO_BYTE
}
