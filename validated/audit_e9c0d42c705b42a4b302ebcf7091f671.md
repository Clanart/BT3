### Title
`store_state_diff` Accepts State Diff Without Verifying Against Stored Block Header Commitment — (`crates/apollo_central_sync/src/lib.rs`)

---

### Summary

`GenericStateSync::store_state_diff` in `apollo_central_sync` receives a `(block_number, block_hash, state_diff)` tuple from the feeder gateway but never verifies that the state diff's Poseidon commitment matches the `state_diff_commitment` already stored in the block header. The `block_hash` parameter is accepted and logged but never used to cross-check the incoming state diff against the committed value. An explicit developer TODO at the exact entry point acknowledges this gap. The result is that an incorrect or tampered state diff is written unconditionally into the node's Patricia storage, producing a wrong global state root, wrong RPC responses, and wrong execution state.

---

### Finding Description

`store_state_diff` is called from `process_sync_event` whenever a `SyncEvent::StateDiffAvailable` arrives from the central (feeder-gateway) source stream.

```
crates/apollo_central_sync/src/lib.rs:435-442
async fn store_state_diff(
    &mut self,
    block_number: BlockNumber,
    block_hash: BlockHash,          // ← received from feeder, never verified
    mut state_diff: StateDiff,
    ...
) -> StateSyncResult {
    // TODO(dan): verifications - verify state diff against stored header.
``` [1](#0-0) 

The block header for `block_number` is already stored in the node's MDBX storage by the time the state diff stream reaches this block (the header stream runs ahead of the state diff stream). The stored header contains a `state_diff_commitment` (a Poseidon hash over the state diff content) that is part of the block hash chain. The function never reads the stored header, never computes `calculate_state_diff_hash(&thin_state_diff)`, and never compares the result to the stored commitment.

The `block_hash` argument is only used for a log line at line 598:

```
info!("SYNC_NEW_BLOCK: Added block {} with hash {:#064x}.", block_number, block_hash.0);
``` [2](#0-1) 

After the missing check, the function calls `append_state_diff(block_number, thin_state_diff)` unconditionally, writing every field of the incoming diff (deployed contracts, storage slots, nonces, class hashes, compiled class hashes) into the node's authoritative storage tables. [3](#0-2) 

The correct check — already implemented in the `apollo_committer` for the batcher path — is:

```rust
// apollo_committer/src/committer.rs:167-175
if self.config.verify_state_diff_hash {
    let calculated_commitment = calculate_state_diff_hash(&state_diff);
    if commitment != calculated_commitment {
        return Err(CommitterError::StateDiffHashMismatch { ... });
    }
}
``` [4](#0-3) 

That same `calculate_state_diff_hash` function is available in `starknet_api::block_hash::state_diff_hash` and is already imported in the codebase. The central sync path simply never calls it.

---

### Impact Explanation

Once a wrong `ThinStateDiff` is committed via `append_state_diff`, every downstream consumer reads corrupted state:

- **Storage values** (`contract_storage` table) are wrong → `starknet_getStorageAt` and `starknet_call` return wrong values.
- **Nonces** (`nonces` table) are wrong → account nonce checks during transaction validation use wrong values, causing valid transactions to be rejected or replayed.
- **Class hashes / compiled class hashes** (`deployed_contracts`, `compiled_class_hash` tables) are wrong → the wrong contract code is selected for execution.
- **The Patricia trie root** computed by `starknet_committer` over this storage is wrong → the global state root diverges from the canonical chain, breaking proof generation and L1 settlement.

This matches the allowed impact: *"Wrong state … class hash, storage value … from blockifier/syscall/execution logic"* and *"RPC execution … returns an authoritative-looking wrong value."* [5](#0-4) 

---

### Likelihood Explanation

The feeder gateway is a trusted source in the current deployment, but:

1. The TODO comment at line 442 is an explicit developer acknowledgment that the check is **intentionally deferred**, not that it is unnecessary.
2. The `block_hash` parameter is already threaded into the function signature, showing the intent to use it for verification.
3. The block header (which carries `state_diff_commitment`) is guaranteed to be present in storage before the state diff stream reaches the same block number, because the header stream runs ahead (`state_marker < header_marker` is enforced at line 818).
4. Any bug or compromise in the feeder gateway — or a network-level MITM between the node and the feeder — would silently corrupt state with no detection. [6](#0-5) 

---

### Recommendation

Inside `store_state_diff`, after computing `thin_state_diff`, read the stored block header and verify the state diff commitment:

```rust
let stored_header = self.reader.begin_ro_txn()?
    .get_block_header(block_number)?
    .ok_or(StateSyncError::MissingHeader { block_number })?;

if let Some(expected_commitment) = stored_header.state_diff_commitment {
    let actual_commitment = calculate_state_diff_hash(&thin_state_diff);
    if actual_commitment != expected_commitment {
        return Err(StateSyncError::StateDiffCommitmentMismatch {
            block_number,
            expected: expected_commitment,
            actual: actual_commitment,
        });
    }
}
```

This mirrors the check already present in `apollo_committer::commit_block_inner` and closes the gap noted by the TODO.

---

### Proof of Concept

1. Node starts central sync. The header stream stores block N's header, which contains `state_diff_commitment = H`.
2. The state diff stream yields `SyncEvent::StateDiffAvailable { block_number: N, block_hash: B, state_diff: FAKE_DIFF, ... }` where `calculate_state_diff_hash(FAKE_DIFF) ≠ H`.
3. `store_state_diff` is called. Line 442's TODO is the only guard — there is none. The function proceeds to `append_state_diff(N, fake_thin_diff)`.
4. The storage tables for block N now contain the attacker-chosen storage slots, nonces, and class hashes.
5. A subsequent `starknet_getStorageAt(contract, key, block=N)` returns the injected value.
6. The Patricia trie root computed from this storage diverges from the canonical root embedded in block N's hash, silently breaking proof generation for all subsequent blocks. [7](#0-6) [3](#0-2) [8](#0-7)

    fn append_state_diff(
        self,
        block_number: BlockNumber,
        thin_state_diff: ThinStateDiff,
    ) -> StorageResult<Self> {
        let file_offset_table = self.txn.open_table(&self.tables.file_offsets)?;
        let markers_table = self.open_table(&self.tables.markers)?;
        let state_diffs_table = self.open_table(&self.tables.state_diffs)?;
        let nonces_table = self.open_table(&self.tables.nonces)?;
        let deployed_contracts_table = self.open_table(&self.tables.deployed_contracts)?;
        let storage_table = self.open_table(&self.tables.contract_storage)?;
        let declared_classes_block_table = self.open_table(&self.tables.declared_classes_block)?;
        let deprecated_declared_classes_block_table =
            self.open_table(&self.tables.deprecated_declared_classes_block)?;
        let compiled_class_hash_table = self.open_table(&self.tables.compiled_class_hash)?;

        // Write state.
        write_deployed_contracts(
            &thin_state_diff.deployed_contracts,
            &self.txn,
            block_number,
            &deployed_contracts_table,
            &nonces_table,
        )?;
        write_storage_diffs(
            &thin_state_diff.storage_diffs,
            &self.txn,
            block_number,
            &storage_table,
        )?;
        // Must be called after write_deployed_contracts since the nonces are updated there.
        write_nonces(&thin_state_diff.nonces, &self.txn, block_number, &nonces_table)?;

        for (class_hash, _) in &thin_state_diff.class_hash_to_compiled_class_hash {
            let not_declared = declared_classes_block_table.get(&self.txn, class_hash)?.is_none();
            if not_declared {
                declared_classes_block_table.insert(&self.txn, class_hash, &block_number)?;
            }
        }

        write_compiled_class_hashes(
            &thin_state_diff.class_hash_to_compiled_class_hash,
            &self.txn,
            block_number,
            &compiled_class_hash_table,
        )?;

        for class_hash in thin_state_diff.deprecated_declared_classes.iter() {
            // Cairo0 classes can be declared in different blocks. The first block to declare the
            // class is recorded here.
            if deprecated_declared_classes_block_table.get(&self.txn, class_hash)?.is_none() {
                deprecated_declared_classes_block_table.insert(
                    &self.txn,
                    class_hash,
                    &block_number,
                )?;
            }
        }

        // Write state diff.
        let location = self.file_handlers.append_state_diff(&thin_state_diff);
        state_diffs_table.append(&self.txn, &block_number, &location)?;
        file_offset_table.upsert(&self.txn, &OffsetKind::ThinStateDiff, &location.next_offset())?;

        update_marker_to_next_block(&self.txn, &markers_table, MarkerKind::State, block_number)?;

        advance_compiled_class_marker_over_blocks_without_classes(
            &self.txn,
            &markers_table,
            &state_diffs_table,
            &self.file_handlers,
        )?;

        Ok(self)
