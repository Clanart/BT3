### Title
`compute_meta_tx_v0_hash` Omits Nonce, Enabling Signature Replay Across Blocks — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/transaction_hash/transaction_hash.cairo`)

---

### Summary

`compute_meta_tx_v0_hash` produces a hash over `(prefix, version=0, contract_address, selector, calldata, max_fee=0, chain_id)` with `additional_data_size=0` — no nonce is committed. The OS also explicitly skips nonce checking for version-0 transactions. Any signature produced for a `meta_tx_v0` call is therefore permanently valid and can be replayed by any observer, in any block, indefinitely.

---

### Finding Description

`compute_meta_tx_v0_hash` delegates to `deprecated_get_transaction_hash` with `additional_data_size=0`:

```cairo
// transaction_hash.cairo:302-313
let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
    tx_hash_prefix=INVOKE_HASH_PREFIX,
    version=0,
    contract_address=contract_address,
    entry_point_selector=entry_point_selector,
    calldata_size=calldata_size,
    calldata=calldata,
    max_fee=0,
    chain_id=chain_id,
    additional_data_size=0,           // ← no nonce
    additional_data=cast(0, felt*),
);
``` [1](#0-0) 

Contrast this with `compute_l1_handler_transaction_hash`, which commits a nonce in `additional_data`:

```cairo
// transaction_hash.cairo:233-234
additional_data_size=1,
additional_data=&nonce,
``` [2](#0-1) 

The OS-level nonce guard in `check_and_increment_nonce` explicitly exempts version-0 transactions:

```cairo
// execute_transaction_utils.cairo:64-66
// Do not handle nonce for version 0.
if (tx_info.version == 0) {
    return ();
}
``` [3](#0-2) 

And `execute_meta_tx_v0` sets `nonce=0` in the constructed `TxInfo`, confirming no per-invocation uniqueness is tracked:

```cairo
// syscall_impls.cairo:351
nonce=0,
``` [4](#0-3) 

The hash `H(INVOKE_PREFIX, 0, contract_address, selector, calldata, 0, chain_id)` is therefore **static** for a fixed `(contract_address, selector, calldata)` tuple. Once a valid signature over this hash is observed on-chain, it can be replayed by any party in any subsequent block.

---

### Impact Explanation

The `meta_tx_v0` syscall is designed to let a relayer submit a gasless transaction on behalf of a user. The target account contract's `__execute__` entry point receives `transaction_hash = meta_tx_hash` and verifies the user's signature against it. Because the hash never changes for the same calldata, a valid signature authorizes unlimited future executions of the same call. An attacker who observes a signed meta-tx v0 (e.g., a token transfer) can replay it in every subsequent block until the account is drained — **direct loss of funds (Critical)**.

---

### Likelihood Explanation

The `meta_tx_v0` syscall is a live protocol feature reachable by any unprivileged transaction sender. Any account contract that relies on the OS-provided `transaction_hash` for replay protection (the standard StarkNet account pattern) is immediately vulnerable. No privileged access, key compromise, or external dependency is required — only the ability to submit a transaction that invokes the syscall with a previously observed signature.

---

### Recommendation

Include a per-invocation nonce in `compute_meta_tx_v0_hash`, analogous to how `compute_l1_handler_transaction_hash` passes `additional_data=&nonce`. The OS must also track and increment this nonce in `execute_meta_tx_v0`, rather than unconditionally setting `nonce=0` and bypassing `check_and_increment_nonce`.

---

### Proof of Concept

1. User signs a meta-tx v0 payload: `(contract_address=ACCOUNT, selector=__execute__, calldata=[transfer 100 STRK to ATTACKER])`. The resulting hash is `H = compute_meta_tx_v0_hash(ACCOUNT, __execute__, calldata, chain_id)`.
2. Relayer submits the transaction. It is included in block N. The signature `sig` is now public.
3. Attacker constructs a new outer transaction calling any contract that issues the `meta_tx_v0` syscall with `(contract_address=ACCOUNT, selector=__execute__, calldata=[transfer 100 STRK to ATTACKER], signature=sig)`.
4. `execute_meta_tx_v0` recomputes the same hash `H` (no nonce, no block binding), sets `transaction_hash=H`, and calls `ACCOUNT.__execute__`.
5. `ACCOUNT.__execute__` verifies `sig` against `H` — it passes, and 100 STRK is transferred again.
6. Steps 3–5 repeat in blocks N+1, N+2, … until the account balance is zero. [5](#0-4) [6](#0-5)
