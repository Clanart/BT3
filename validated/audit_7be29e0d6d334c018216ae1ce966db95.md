### Title
Unbounded `Calls` per `SimBlock` in `eth_simulateV1` Enables Resource-Exhaustion DoS — (`x/evm/keeper/simulate.go`, `rpc/types/simulate.go`)

---

### Summary

`eth_simulateV1` enforces a cap of 256 simulated blocks (`MaxSimulateBlocks`) and a cross-block gas budget, but places **no limit on the number of calls per block** (`SimBlock.Calls []json.RawMessage`). Calls with `gas = 0` bypass both the per-block gas limit check and the cross-block `GasBudget`, so an attacker can submit a single request containing one block with an arbitrarily large number of zero-gas calls. The context timeout is only checked between blocks, not between calls within a block, so it provides no protection. The result is unbounded CPU and memory consumption inside the JSON-RPC/gRPC server process, which shares resources with the consensus node.

---

### Finding Description

**Root cause — no per-block call count limit:**

`SimBlock.Calls` is a plain slice with no length bound: [1](#0-0) 

`processBlock` pre-allocates three slices proportional to `len(block.Calls)` before any gas check: [2](#0-1) 

**Root cause — zero-gas calls bypass both guards:**

`sanitizeCall` only rejects a call when `*remainingBlockGas < uint64(*call.Gas)`. When `call.Gas` is explicitly set to `0`, the comparison is `remainingBlockGas < 0`, which is always `false` for a `uint64`. The call then passes `budget.Cap(0) = 0` and `budget.Consume(0)` succeeds unconditionally: [3](#0-2) 

The cross-block `GasBudget` is initialised from `gasCap` (default 25 M gas) and deducts `gasUsed` after each call. Because `gasUsed = 0` for a zero-gas call, the budget is never depleted: [4](#0-3) 

**Root cause — timeout not checked inside the call loop:**

`Execute` checks `ctx.Err()` only between blocks, not between individual calls within `processBlock`: [5](#0-4) 

The default `EVMTimeout` of 5 s is therefore ineffective against a single block with millions of calls.

**Block-count limit does not help:**

The 256-block cap is enforced at both the RPC layer and the keeper: [6](#0-5) [7](#0-6) 

But a single block with N calls is entirely unconstrained.

**Attack path:**

```
POST eth_simulateV1
{
  "blockStateCalls": [{
    "calls": [
      {"from":"0x…","to":"0x…","gas":"0x0"},  // repeated N million times
      ...
    ]
  }]
}
```

For each call the node:
1. JSON-unmarshals the call args
2. Calls `state.GetNonce` (store read)
3. Constructs an `ethtypes.Transaction` and computes its hash
4. Invokes `vm.NewEVM` + `applyCall` (EVM setup + intrinsic-gas failure)
5. Appends to `callResults`, `transactions`, `receipts`, `senders`

With N = 10 million calls, memory for the four pre-allocated slices alone exceeds several GB, and CPU time is proportional to N.

---

### Impact Explanation

The `eth_simulateV1` gRPC handler runs inside the same OS process as the CometBFT consensus engine. Exhausting the process's memory or saturating its CPU goroutines will stall block proposal and voting, causing the validator to miss blocks and potentially halt the chain. This is a reachable route to the Critical impact "valid unprivileged RPC submission can halt the chain." The finding is classified **High** under "Public JSON-RPC/gRPC simulation path exposes a reachable route to chain halt."

---

### Likelihood Explanation

The `eth_simulateV1` endpoint is publicly reachable on any node that exposes the JSON-RPC API (the default configuration). No authentication, privileged key, or on-chain transaction is required. A single HTTP POST is sufficient to trigger the condition. The only prerequisite is knowing the node's RPC address.

---

### Recommendation

1. **Add a per-block call count cap** (e.g., 1 000 calls per block) checked before the call loop in `processBlock`:
   ```go
   const MaxCallsPerBlock = 1000
   if len(block.Calls) > MaxCallsPerBlock {
       return nil, nil, nil, &rpctypes.ClientLimitExceededError{
           Message: fmt.Sprintf("too many calls per block: %d > %d", len(block.Calls), MaxCallsPerBlock),
       }
   }
   ```

2. **Reject zero-gas calls** in `sanitizeCall` (or treat `gas = 0` as `gas = nil`):
   ```go
   if call.Gas != nil && uint64(*call.Gas) == 0 {
       call.Gas = nil  // treat as unspecified
   }
   ```

3. **Check `ctx.Err()` inside the call loop** in `processBlock` so the EVMTimeout can interrupt a long-running block.

---

### Proof of Concept

```python
import json, requests

calls = [{"from": "0x" + "1"*40, "to": "0x" + "2"*40, "gas": "0x0"}] * 5_000_000

payload = {
    "jsonrpc": "2.0",
    "method": "eth_simulateV1",
    "params": [
        {"blockStateCalls": [{"calls": calls}]},
        "latest"
    ],
    "id": 1
}

# Single request; node OOMs or saturates CPU before responding.
requests.post("http://localhost:8545", json=payload, timeout=300)
```

The node pre-allocates `make([]SimCallResult, 5_000_000)`, `make([]*ethtypes.Transaction, 5_000_000)`, and `make(ethtypes.Receipts, 5_000_000)` before processing a single call, consuming several GB of RAM. Each subsequent zero-gas call then executes a full EVM setup cycle without decrementing the gas budget, so the loop runs to completion regardless of the configured `EVMTimeout`. // NewGasBudget creates a gas budget with the given cap. A cap of 0 is unlimited.
func NewGasBudget(gasCap uint64) *GasBudget {
	if gasCap == 0 {
		gasCap = math.MaxUint64
	}
	return &GasBudget{remaining: gasCap}
}

// Cap returns the given gas value clamped to the remaining budget.
func (b *GasBudget) Cap(gas uint64) uint64 {
	if gas > b.remaining {
		return b.remaining
	}
	return gas
}

// Consume deducts the given amount from the budget.
func (b *GasBudget) Consume(amount uint64) error {
	if amount > b.remaining {
		return fmt.Errorf("RPC gas cap exhausted: need %d, remaining %d", amount, b.remaining)
	}
	b.remaining -= amount
	return nil
}

func (sim *Simulator) sanitizeCall(call *evmtypes.TransactionArgs, state *statedb.StateDB, header *ethtypes.Header, remainingBlockGas *uint64) error {
	if call.Nonce == nil {
		nonce := state.GetNonce(call.GetFrom())
		call.Nonce = (*hexutil.Uint64)(&nonce)
	}
	// Let the call run wild unless explicitly specified.
	if call.Gas == nil {
		call.Gas = (*hexutil.Uint64)(remainingBlockGas)
	}
	if *remainingBlockGas < uint64(*call.Gas) {
		return &rpctypes.BlockGasLimitReachedError{
			Message: fmt.Sprintf(
				"block gas limit reached: remaining: %d, required: %d",
				*remainingBlockGas,
				*call.Gas,
			),
		}
	}
	// Clamp to the cross-block gas budget.
	gas := sim.budget.Cap(uint64(*call.Gas))
	call.Gas = (*hexutil.Uint64)(&gas)

	return call.CallDefaults(0, header.BaseFee, sim.chainConfig.ChainID)
