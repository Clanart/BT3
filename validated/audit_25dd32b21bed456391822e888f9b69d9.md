### Title
DataLayer `DeltaReader` Accepts Forged Hash-to-Node Mappings Without Hash Integrity Verification, Enabling Corrupted MerkleBlob Tree Roots - (File: `crates/chia-datalayer/src/merkle/deltas.rs`)

### Summary

`DeltaReader::new()` and its Python-binding mutation methods (`add_internal_nodes`, `add_leaf_nodes`) accept caller-supplied `(hash → node_content)` mappings with no verification that the hash key actually equals the cryptographic hash of the node content. When `create_merkle_blob_and_filter_unused_nodes()` is subsequently called, it builds a `MerkleBlob` that stores the caller-supplied (potentially forged) hash as the node's authoritative hash, producing a tree whose root hash is fabricated and whose internal node hashes do not match the hash of their children. Any `ProofOfInclusion` generated from this corrupted blob will pass `valid()` while proving inclusion in a tree with a forged root.

### Finding Description

`DeltaReader::new()` inserts caller-supplied entries directly into its internal `nodes` map without any hash integrity check:

```rust
pub fn new(internal_nodes: InternalNodesMap, leaf_nodes: LeafNodesMap) -> Result<Self, Error> {
    let mut nodes = NodeHashToDeltaReaderNode::new();
    for (hash, (left, right)) in internal_nodes {
        nodes.insert(hash, DeltaReaderNode::Internal { left, right });
    }
    for (hash, (key, value)) in leaf_nodes {
        nodes.insert(hash, DeltaReaderNode::Leaf { key, value });
    }
    Ok(Self { nodes })
}
``` [1](#0-0) 

The Python-binding mutation methods `py_add_internal_nodes` and `py_add_leaf_nodes` have the same flaw — they insert caller-controlled `(hash, node)` pairs with no verification: [2](#0-1) 

When `create_merkle_blob_and_filter_unused_nodes(root_hash, ...)` is called, it delegates to `MerkleBlob::build_blob_from_node_list`, which in `inner_build_blob_from_node_list` inserts each internal node into the blob using the caller-supplied `node_hash` as the stored hash — not a value computed from the children:

```rust
let block = Block {
    node: Node::Internal(InternalNode {
        hash: node_hash,   // ← caller-supplied, never verified
        parent: Parent(None),
        left: left_index,
        right: right_index,
    }),
};
``` [3](#0-2) 

There is no step anywhere in `inner_build_blob_from_node_list` that recomputes `internal_hash(left_child_hash, right_child_hash)` and compares it to `node_hash`: [4](#0-3) 

The resulting `MerkleBlob` therefore has internal nodes whose stored hash field is whatever the attacker supplied, not the true Merkle hash of the subtree. `get_proof_of_inclusion()` then reads these stored hashes verbatim when building `ProofOfInclusionLayer.combined_hash`: [5](#0-4) 

`ProofOfInclusion::valid()` only checks internal consistency of the proof chain (each layer's `combined_hash` equals the computed hash of the previous hash and `other_hash`). Its final check `existing_hash == self.root_hash()` is tautological — `root_hash()` returns `last.combined_hash`, which `existing_hash` was just set to: [6](#0-5) 

A proof built from a corrupted blob will pass `valid()` while asserting inclusion in a tree with a forged root hash.

### Impact Explanation

An untrusted caller (e.g., a malicious DataLayer peer, or any Python/wasm consumer of the `DeltaReader` API) can supply fabricated `internal_nodes` where `forged_root_hash → (left_hash, right_hash)` but `forged_root_hash ≠ internal_hash(left_hash, right_hash)`. After calling `create_merkle_blob_and_filter_unused_nodes(forged_root_hash, ...)`, the resulting `MerkleBlob` reports `forged_root_hash` as its root. Any `ProofOfInclusion` generated from this blob will pass `valid()` while proving inclusion in a tree that does not actually exist. This allows forged inclusion/exclusion proofs and corrupts committed DataLayer tree roots — matching the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

### Likelihood Explanation

The Python binding `DeltaReader.__new__(internal_nodes, leaf_nodes)` is directly reachable from any Python caller with no privilege requirement: [7](#0-6) 

In the DataLayer delta-sync workflow, a node receives node maps from a peer and constructs a `DeltaReader` from them. A malicious peer can supply forged mappings. No authentication or hash-integrity gate exists before the data enters `DeltaReader`. The attack requires only the ability to supply a crafted `internal_nodes` dict — a capability any network peer or local Python caller possesses.

### Recommendation

In `DeltaReader::new()`, `py_add_internal_nodes`, and `py_add_leaf_nodes`, verify that each supplied `(hash, node)` pair is cryptographically consistent before inserting it:

- For internal nodes: compute `internal_hash(left, right)` and assert it equals the supplied `hash`.
- For leaf nodes: compute the leaf hash from `(key, value)` (or require the caller to supply the pre-image and compute the hash internally) and assert it equals the supplied `hash`.

Alternatively, enforce this check inside `inner_build_blob_from_node_list` before writing the node to the blob, rejecting any node whose stored hash does not match the recomputed hash of its children. [8](#0-7) 

### Proof of Concept

```python
from chia_rs.datalayer import DeltaReader, KeyId, ValueId
import hashlib

# Real leaf: hash_A is the true hash of leaf data
hash_A = bytes([0xAA] * 32)
hash_B = bytes([0xBB] * 32)

# Forged root: attacker claims forged_root = internal_hash(hash_A, hash_B)
# but supplies a completely different value
forged_root = bytes([0xFF] * 32)

# Build DeltaReader with forged internal node mapping
# forged_root -> (hash_A, hash_B) but forged_root != internal_hash(hash_A, hash_B)
dr = DeltaReader(
    internal_nodes={forged_root: (hash_A, hash_B)},
    leaf_nodes={
        hash_A: (KeyId(1), ValueId(1)),
        hash_B: (KeyId(2), ValueId(2)),
    }
)

# No missing hashes reported — the forged root is "present"
assert dr.get_missing_hashes(forged_root) == set()

# Build a MerkleBlob with the forged root
blob = dr.create_merkle_blob_and_filter_unused_nodes(forged_root, {forged_root, hash_A, hash_B})

# The blob reports forged_root as its root — a hash that was never computed
# from any real tree structure
assert blob.get_root_hash() == forged_root

# ProofOfInclusion for key 1 passes valid() despite the forged root
proof = blob.get_proof_of_inclusion(KeyId(1))
assert proof.valid()          # True — proof is internally consistent
assert proof.root_hash() == forged_root  # But root is forged
```
