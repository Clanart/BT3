### Title
`SyncStateReader::get_compiled_class_hash` Implemented as `todo!()` Causes Panic on Every Declare Transaction Gateway Validation — (File: `crates/apollo_gateway/src/sync_state_reader.rs`)

### Summary

The `get_compiled_class_hash` method on `SyncStateReader` — the live state reader used by the gateway for stateful transaction validation — is implemented as `todo!()`. Every Declare transaction submitted to the gateway triggers a full blockifier execution during stateful validation, which calls `get_compiled_class_hash` to check for prior class declaration. This unconditionally panics, causing the gateway to reject all valid Declare transactions before they can reach the mempool.

### Finding Description

In `crates/apollo_gateway/src/sync_state_reader.rs`, `SyncStateReader` implements the `BlockifierStateReader` trait. All methods are properly implemented except `get_compiled_class_hash`, which is left as a stub: [1](#0-0) 

The gateway's stateful validator (`StatefulValidator::perform_validations`) handles Declare transactions by calling `self.execute(tx)` — a full blockifier execution — rather than the lighter validate-only path used for Invoke transactions: [2](#0-1) 

During that full execution, `CachedState::get_compiled_class_hash` is called to check whether the class was previously declared (to enforce the "declare once" invariant). On a cache miss, it delegates directly to the underlying state reader: [3](#0-2) 

`StateReaderAndContractManager` — the wrapper used in the gateway — passes the call straight through to `SyncStateReader`: [4](#0-3) 

This reaches the `todo!()` stub and panics. The `SyncStateReaderFactory` is the production factory that creates `SyncStateReader` instances for every gateway stateful validation: [5](#0-4) 

The RPC execution path (`apollo_rpc_execution`) has a correct implementation that reads from storage and pending data: [6](#0-5) 

`SyncStateReader` has no equivalent implementation.

### Impact Explanation

Every Declare transaction (v1, v2, v3) submitted to the gateway panics unconditionally during stateful validation. The gateway cannot admit any new class declaration to the mempool. No new Sierra/CASM contract classes can be deployed on the sequencer node. This is a complete denial of the Declare transaction type at the admission layer.

### Likelihood Explanation

Any user submitting a Declare transaction to the gateway triggers the panic. No special privileges or unusual conditions are required — it is the normal production code path for every Declare transaction.

### Recommendation

Implement `get_compiled_class_hash` in `SyncStateReader` by querying the state sync client, mirroring the pattern used for `get_storage_at`, `get_nonce_at`, and `get_class_hash_at`. The `apollo_state_sync_types::communication::StateSyncClient` trait already exposes `is_class_declared_at`; a compiled class hash lookup should be added there, or the method should read from the class manager, following the pattern in `apollo_rpc_execution/src/state_reader.rs` lines 163–208. [1](#0-0) 

### Proof of Concept

1. Start the sequencer node with the default `SyncStateReaderFactory` configuration.
2. Submit any valid Declare V2 or V3 transaction to the gateway's `add_transaction` endpoint.
3. The gateway calls `StatefulValidator::perform_validations` → `self.execute(tx)` (Declare branch).
4. The blockifier calls `CachedState::get_compiled_class_hash` → `StateReaderAndContractManager::get_compiled_class_hash` → `SyncStateReader::get_compiled_class_hash`.
5. `todo!()` fires; the task panics with `not yet implemented`.
6. The gateway returns an error to the caller; the transaction is never forwarded to the mempool.
7. Repeat for any Declare transaction — the result is always a panic.

        let mut cache = self.cache.borrow_mut();

        if cache.get_compiled_class_hash(class_hash).is_none() {
            let compiled_class_hash = self.state.get_compiled_class_hash(class_hash)?;
            cache.set_compiled_class_hash_initial_value(class_hash, compiled_class_hash);
        }

        let compiled_class_hash = cache
            .get_compiled_class_hash(class_hash)
            .unwrap_or_else(|| panic!("Cannot retrieve '{class_hash:?}' from the cache."));
        Ok(*compiled_class_hash)

    fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
        if let Some(pending_data) = &self.maybe_pending_data {
            for DeclaredClassHashEntry { class_hash: other_class_hash, compiled_class_hash } in
                &pending_data.declared_classes
            {
                if class_hash == *other_class_hash {
                    return Ok(*compiled_class_hash);
                }
            }
        }

        let maybe_block_number = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_reader()
            .map_err(storage_err_to_state_err)?
            .get_class_definition_block_number(&class_hash)
            .map_err(storage_err_to_state_err)?;

        // Cairo 0 classes (and undeclared classes) do not have a compiled class hash.
        // According to the trait, return the default value.
        let Some(block_number) = maybe_block_number else {
            return Ok(CompiledClassHash::default());
        };

        let state_diff = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_diff(block_number)
            .map_err(storage_err_to_state_err)?
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing state diff at block {block_number}."
            )))?;

        let compiled_class_hash = state_diff
            .class_hash_to_compiled_class_hash
            .get(&class_hash)
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing class declaration at block {block_number}, class \
                 {class_hash}."
            )))?;

        Ok(*compiled_class_hash)
    }
