### Title
Underestimated Gas Cost for Storage Syscalls Missing Patricia Update and Dict Squash Overhead — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/constants.cairo`)

---

### Summary

`STORAGE_READ_GAS_COST` and `STORAGE_WRITE_GAS_COST` in `constants.cairo` are explicitly acknowledged by a developer TODO to **not include** the overhead of Patricia tree updates and dictionary squashing. These operations are performed by the OS during state commitment but are not charged to the caller. Any unprivileged contract can exploit this underpricing to submit storage-heavy transactions at below-actual-cost gas prices, economically attacking the sequencer and potentially halting the network.

---

### Finding Description

In `constants.cairo`, the gas costs for storage syscalls are defined as:

```cairo
// TODO(Yoni, 1/1/2026): take into account Patricia updates and dict squash.
const STORAGE_READ_GAS_COST = 18070;
const STORAGE_WRITE_GAS_COST = 44970;
``` [1](#0-0) 

The inline TODO is a developer acknowledgment that these constants are **incomplete**: they do not account for the cost of Patricia trie updates and dictionary squashing that the OS must perform for every storage access during state commitment.

These constants are consumed directly in `syscall_impls.cairo` to charge gas for every `storage_read` and `storage_write` syscall:

```cairo
let success = reduce_syscall_gas_and_write_response_header(
    total_gas_cost=STORAGE_READ_GAS_COST, request_struct_size=StorageReadRequest.SIZE
);
``` [2](#0-1) 

```cairo
let success = reduce_syscall_gas_and_write_response_header(
    total_gas_cost=STORAGE_WRITE_GAS_COST, request_struct_size=StorageWriteRequest.SIZE
);
``` [3](#0-2) 

The actual OS work for each storage access includes:

1. A `DictAccess` entry appended to the contract's `storage_ptr` (per read/write).
2. At block finalization, `squash_state_changes` squashes all inner storage dicts and the outer contract dict.
3. `compute_contract_state_commitment` then runs Patricia trie updates over the squashed changes. [4](#0-3) [5](#0-4) 

None of the gas cost of these operations is reflected in `STORAGE_READ_GAS_COST` or `STORAGE_WRITE_GAS_COST`. The analogy to the external report is exact: just as `MIN_DEPOSIT_LAMPORTS` omitted `ACCOUNT_STORAGE_OVERHEAD`, these constants omit the Patricia/squash overhead.

---

### Impact Explanation

**High — Network not being able to confirm new transactions.**

An attacker can deploy a contract that performs a large number of storage reads and/or writes in a single transaction. The gas charged to the attacker is `N × STORAGE_READ_GAS_COST` (or write equivalent), but the actual proving cost incurred by the sequencer/prover is `N × (STORAGE_READ_GAS_COST + Patricia_overhead + squash_overhead)`. Because the Patricia and squash overhead is unmetered, the attacker pays less than the true computational cost.

At sufficient scale (many such transactions filling blocks), the sequencer is forced to perform provably more work than it is compensated for. This constitutes an economic griefing attack: the sequencer operates at a loss, and if the loss is large enough, rational sequencer operators shut down, halting the network's ability to confirm new transactions.

---

### Likelihood Explanation

**High.** The attack requires no privileges. Any user can deploy a Cairo contract that loops over `storage_write` or `storage_read` syscalls. The underpricing is structural and present in every block. The TODO comment confirms the developers are aware the values are incomplete, meaning the gap between charged and actual cost is real and non-zero.

---

### Recommendation

Compute the true gas cost of a storage access to include:
- The amortized cost of one `DictAccess` entry in the inner storage dict.
- The amortized cost of one entry in the outer contract state dict.
- The amortized cost of Patricia trie update steps for both the storage trie and the contract state trie.
- The amortized cost of `squash_dict` over the inner and outer dicts.

Update `STORAGE_READ_GAS_COST` and `STORAGE_WRITE_GAS_COST` in `constants.cairo` (and the corresponding template in `constants_template.txt`) to reflect the full cost, resolving the acknowledged TODO. [1](#0-0) 

---

### Proof of Concept

1. Attacker deploys a Cairo 1 contract with an entry point that calls `storage_write(key=i, value=i)` in a loop for `i = 0..N`.
2. Attacker submits an invoke transaction with `l2_gas_bounds.max_amount` set to cover `N × STORAGE_WRITE_GAS_COST`.
3. The OS charges exactly `N × STORAGE_WRITE_GAS_COST` gas from the attacker's budget via `reduce_syscall_gas_and_write_response_header`.
4. At block finalization, `squash_state_changes` squashes `N` inner dict entries and `compute_contract_state_commitment` runs Patricia updates over all `N` changed keys — work that was never charged.
5. By submitting many such transactions, the attacker forces the sequencer/prover to perform unbounded uncompensated Patricia and squash work, degrading throughput and potentially making the network economically unviable to operate.

func squash_state_changes{range_check_ptr}(
    contract_state_changes_start: DictAccess*, contract_state_changes_end: DictAccess*
) -> (n_contract_state_changes: felt, squashed_contract_state_dict: DictAccess*) {
    alloc_locals;

    // State changes after squashing the outer dictionary.
    let (local squashed_dict: DictAccess*) = alloc();

    // Squash the global dictionary to get a list of triples (addr, dict_begin, dict_end).
    let (squashed_dict_end) = squash_dict(
        dict_accesses=contract_state_changes_start,
        dict_accesses_end=contract_state_changes_end,
        squashed_dict=squashed_dict,
    );

    local n_contract_state_changes = (squashed_dict_end - squashed_dict) / DictAccess.SIZE;

    // State changes after squashing the outer dictionary and the inner dictionaries.
    let (local fully_squashed_dict: DictAccess*) = alloc();
    squash_state_changes_inner(
        n_contract_state_changes=n_contract_state_changes,
        state_changes=squashed_dict,
        squashed_state_changes=fully_squashed_dict,
    );

    return (
        n_contract_state_changes=n_contract_state_changes,
        squashed_contract_state_dict=fully_squashed_dict,
    );
}
