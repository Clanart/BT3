### Title
Cross-Deployment Signature Replay via Missing Contract-Address Binding in `to_borsh` — (`starknet/src/bridge_types.cairo`)

### Summary

`TransferMessagePayload::to_borsh` does not include the OmniBridge contract address in the signed message. The only deployment-specific binding is `omni_bridge_chain_id` (a `u8`). If two OmniBridge deployments share the same `omni_bridge_chain_id` and `omni_bridge_derived_address`, an unprivileged relayer can replay a valid MPC-signed `fin_transfer` payload from one deployment on the other, bypassing the per-contract nonce bitmap and minting unbacked tokens.

---

### Finding Description

**Root cause — `to_borsh` omits the OmniBridge contract address:** [1](#0-0) 

The serialized message contains:
- `PayloadType` (1 byte)
- `destination_nonce` (8 bytes)
- `origin_chain` (1 byte)
- `origin_nonce` (8 bytes)
- `chain_id` (1 byte) — from `omni_bridge_chain_id`
- `token_address` (32 bytes)
- `amount` (16 bytes)
- `chain_id` again (1 byte)
- `recipient` (32 bytes)
- optional fields

The OmniBridge contract address (`get_contract_address()`) is **never included**. The only deployment discriminator is the `u8` field `omni_bridge_chain_id`.

**Signature verification uses only `omni_bridge_chain_id` and `omni_bridge_derived_address`:** [2](#0-1) 

**Nonce replay protection is per-contract storage only:** [3](#0-2) [4](#0-3) 

`completed_transfers` is a `Map<u64, felt252>` in each contract's own storage. Marking nonce `N` as used in deployment A has zero effect on deployment B's bitmap.

**`fin_transfer` marks nonce then verifies — both checks are local:** [5](#0-4) 

---

### Impact Explanation

If two OmniBridge instances share the same `omni_bridge_chain_id` (e.g., both set to `0x10` for "StarkNet") and the same `omni_bridge_derived_address` (MPC key):

1. MPC legitimately signs a `TransferMessagePayload` for deployment A (mainnet).
2. An unprivileged relayer submits the identical `(signature, payload)` to deployment B (testnet or a second mainnet instance).
3. Deployment B's `completed_transfers` has no record of `destination_nonce`, so the check passes.
4. `_verify_borsh_signature` produces the same hash (same `chain_id`, same `token_address`, same `recipient`) and the same MPC signature verifies.
5. If deployment B is on a different network but deployed at the same contract address (common when using identical deployer + salt), its bridge tokens are at the same addresses and it is their owner — `mint` succeeds.

Result: bridge token `totalSupply` on deployment B increases with no corresponding locked collateral on the origin chain — unbacked supply / double-mint.

---

### Likelihood Explanation

- **Realistic deployment pattern:** testnet and mainnet are routinely deployed from the same scripts with the same deployer key and salt, producing identical contract addresses on both networks.
- **Single chain-type ID:** `omni_bridge_chain_id` is a `u8` likely representing the chain type ("StarkNet"), not a unique per-deployment identifier. Both testnet and mainnet would naturally receive the same value.
- **Same MPC key across environments** is common during early protocol phases.
- The relayer is fully unprivileged — no key compromise or operator collusion required.

---

### Recommendation

Include the OmniBridge contract address in the Borsh-serialized message so the MPC signature is cryptographically bound to a single deployment:

```cairo
// In TransferMessagePayloadTrait::to_borsh
borsh_bytes.append(@borsh::encode_address(get_contract_address())); // add this
```

Alternatively, enforce that `omni_bridge_chain_id` is globally unique per deployment (not per chain type) and document this as a hard invariant checked at deployment time.

---

### Proof of Concept

1. Deploy `OmniBridge_A` (mainnet) and `OmniBridge_B` (testnet) with identical `omni_bridge_chain_id = 0x10`, identical `omni_bridge_derived_address`, and identical deployer+salt (so contract addresses match).
2. On both, call `deploy_token` with the same NEAR token metadata → bridge token deployed at the same address on both networks.
3. User locks 100 tokens on NEAR; MPC signs `TransferMessagePayload { destination_nonce: 1, token_address: 0xABC, amount: 100, ... }` for `OmniBridge_A`.
4. Relayer calls `OmniBridge_A.fin_transfer(sig, payload)` → 100 tokens minted on mainnet. ✓
5. Relayer calls `OmniBridge_B.fin_transfer(sig, payload)` → `is_transfer_finalised(1)` returns `false` (separate storage), signature verifies (same hash), 100 tokens minted on testnet. ✓
6. Assert: `BridgeToken.totalSupply()` on testnet = 100, with zero collateral locked for testnet — unbacked supply confirmed.

    fn to_borsh(self: @TransferMessagePayload, chain_id: u8) -> ByteArray {
        let mut borsh_bytes: ByteArray = Default::default();
        borsh_bytes.append_byte(PayloadType::TransferMessage.into());
        borsh_bytes.append(@borsh::encode_u64(*self.destination_nonce));
        borsh_bytes.append_byte(*self.origin_chain);

        borsh_bytes.append_byte(chain_id);
        borsh_bytes.append(@borsh::encode_address(*self.token_address));
        borsh_bytes.append(@borsh::encode_u128(*self.amount));
        borsh_bytes.append_byte(chain_id);
        borsh_bytes.append(@borsh::encode_address(*self.recipient));
        match self.fee_recipient {
            Option::None => { borsh_bytes.append_byte(0); },
            Option::Some(fee_recipient) => {
                borsh_bytes.append_byte(1);
                borsh_bytes.append(@borsh::encode_byte_array(fee_recipient));
            },
        }
        match self.message {
            Option::None => {},
            Option::Some(message) => { borsh_bytes.append(@borsh::encode_byte_array(message)); },
        }
        borsh_bytes
    }

    fn _verify_borsh_signature(
        ref self: ContractState, borsh_bytes: @ByteArray, signature: Signature,
    ) {
        let message_hash_le = compute_keccak_byte_array(borsh_bytes);
        let message_hash = reverse_u256_bytes(message_hash_le);

        let sig = signature_from_vrs(signature.v, signature.r, signature.s);
        verify_eth_signature(message_hash, sig, self.omni_bridge_derived_address.read());
    }

        let (slot, bit) = _nonce_slot_and_bit(nonce);
