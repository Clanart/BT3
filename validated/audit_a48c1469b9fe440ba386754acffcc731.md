### Title
Missing Zero-Value Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts a `class_hash` of `0` without validation. A contract can invoke `replace_class(0)`, causing the OS to overwrite its own class hash with the uninitialized sentinel value. All subsequent calls to that contract will fail to resolve an entry point, permanently freezing any funds held in its storage.

---

### Finding Description

In `execute_replace_class`, the OS reads `class_hash` directly from the syscall request and writes it into the contract's state entry with no check that it is non-zero or that a declared class with that hash actually exists:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The in-code `TODO` comment at line 898 explicitly acknowledges the missing validation. The value `0` is the uninitialized class hash sentinel (`UNINITIALIZED_CLASS_HASH`), confirmed by its use in `deploy_contract.cairo`:

```cairo
assert state_entry.class_hash = UNINITIALIZED_CLASS_HASH;
``` [2](#0-1) 

Once a contract's class hash is set to `0`, the OS cannot resolve any entry point for it. The contract's storage (and any token balances it holds) becomes permanently inaccessible.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any ERC-20 balance, ETH equivalent, or other asset stored in the contract's storage slot becomes unreachable. There is no recovery path: the OS state update commits the zero class hash to the Patricia Merkle Tree, and no future transaction can execute an entry point on the contract to withdraw assets.

---

### Likelihood Explanation

The `replace_class` syscall is a standard, publicly documented StarkNet syscall callable by any contract. The attack path requires only:

1. Deploying (or interacting with) a contract that holds user funds.
2. Issuing a transaction that causes the contract to call `replace_class` with `class_hash = 0`.

No privileged role, leaked key, or external dependency is required. A malicious contract factory or a buggy upgrade path can trigger this for any depositor. The missing check is confirmed by the developer TODO comment, indicating the team is aware the guard is absent.

---

### Recommendation

Add an explicit non-zero validation before writing the new class hash, analogous to the `assert_not_zero(compiled_class_hash)` guard already present in `execute_declare_transaction`:

```cairo
// In execute_replace_class, before dict_update:
with_attr error_message("Invalid class hash: zero is not a valid class.") {
    assert_not_zero(class_hash);
}
``` [3](#0-2) 

Ideally, also verify that a compiled class with `class_hash` has been declared (resolving the existing TODO), mirroring the `prev_value=0` enforcement used in `dict_update` for class declarations.

---

### Proof of Concept

1. Attacker deploys `VaultContract` which accepts token deposits from users and internally calls `replace_class(class_hash=0)` in a function gated behind a time-lock or governance trigger.
2. Users deposit tokens; balances are recorded in `VaultContract`'s storage.
3. Attacker submits an invoke transaction calling the trigger function.
4. The OS executes `execute_replace_class`:
   - `request.class_hash = 0` is read.
   - No zero-check is performed (line 896).
   - `dict_update` writes `StateEntry(class_hash=0, ...)` for `VaultContract`'s address.
5. State is committed to the Patricia Merkle Tree with class hash `0`.
6. Any subsequent call to `VaultContract` causes the OS to look up compiled class `0`, which does not exist → execution fails unconditionally.
7. All deposited tokens are permanently frozen with no withdrawal path. [4](#0-3)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
