### Title
Missing Declared Class Existence Check in `execute_replace_class` Allows Permanent Fund Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS accepts any arbitrary felt value as a new class hash without verifying that the hash corresponds to a declared class in the contract class tree. This is a direct analog to the reported GithubHint.sol bug: just as a missing contract existence check allowed calls to an invalid address to silently succeed and return empty data, the missing declared-class existence check here allows `replace_class` to silently succeed with an invalid class hash, permanently corrupting the contract's state and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` reads the requested `class_hash` from the syscall request and immediately writes it into the contract's `StateEntry` without any validation:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The developer-acknowledged TODO comment at line 898 explicitly states the check is missing. [1](#0-0) 

The same missing check exists in the deprecated Cairo 0 path: [2](#0-1) 

When a subsequent call is made to the contract with the invalid class hash, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [3](#0-2) 

If `class_hash` is undeclared, `dict_read` returns 0 (uninitialized), and `find_element` with key `0` will fail to locate a compiled class fact, making it impossible to generate a valid proof for any block that calls the corrupted contract.

The `UNINITIALIZED_CLASS_HASH` constant is defined as `0` in `commitment.cairo`: [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once `replace_class` is called with an undeclared class hash:
1. The OS accepts the syscall and commits the corrupted `StateEntry` (with invalid `class_hash`) to the global state tree.
2. Any future call to the contract in a subsequent block causes `execute_entry_point` to fail at `find_element`, making it impossible to produce a valid STARK proof for that block.
3. The sequencer must permanently exclude all calls to the corrupted contract.
4. Any ERC-20 tokens, ETH, or other assets held by the contract are permanently inaccessible — no withdrawal, transfer, or recovery is possible.

---

### Likelihood Explanation

**High.** The attack requires only:
- Deploying a contract (or using an existing one the attacker controls) that calls `replace_class` with an arbitrary undeclared felt value.
- Submitting a single transaction.

No privileged access, leaked keys, or operator cooperation is required. The attacker can target their own contract holding funds, or any contract they control. The TODO comment confirms the developers are aware the check is absent.

---

### Recommendation

Before updating the contract's `StateEntry`, verify that `class_hash` is present in `contract_class_changes` (i.e., it was declared in the current or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` with the requested `class_hash` and assert the result is non-zero (i.e., a valid compiled class hash exists):

```cairo
// Verify the class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the check already implicitly performed in `execute_entry_point` when dispatching a call, making it explicit and enforced at the point of `replace_class`.

---

### Proof of Concept

1. Attacker deploys `MaliciousContract` holding 1000 STRK, with a function:
   ```
   fn poison_self() {
       replace_class(class_hash=0xdeadbeef);  // undeclared class hash
   }
   ```
2. Attacker calls `poison_self()`. The OS executes `execute_replace_class` in `syscall_impls.cairo`. No existence check is performed. The contract's `StateEntry.class_hash` is set to `0xdeadbeef` and committed to the global state.
3. In the next block, any party attempts to call `MaliciousContract` (e.g., to withdraw funds). `execute_entry_point` calls `dict_read` for class hash `0xdeadbeef`, gets `0` (undeclared), then calls `find_element` with key `0`. No compiled class fact with hash `0` exists; `find_element` fails. The block cannot be proven.
4. The sequencer must permanently skip all calls to `MaliciousContract`. The 1000 STRK are permanently frozen.
