### Title
`ProofOfInclusion::valid()` Performs Tautological Root Check, Enabling Forged DataLayer Inclusion Proofs — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate contains a self-referential final check that is always `true` when the proof has at least one layer. The method never validates the proof against an externally trusted root hash. An unprivileged caller can construct a `ProofOfInclusion` for any arbitrary `node_hash` that passes `valid()`, making the function useless as a security gate and enabling forged DataLayer state proofs.

### Finding Description

The `valid()` method walks the proof layers, verifying that each `combined_hash` equals the hash of the running hash combined with `other_hash`. After the loop, it performs a final check:

```rust
existing_hash == self.root_hash()
```

But `root_hash()` is defined as:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // <-- attacker-controlled
    } else {
        self.node_hash
    }
}
```

After the last loop iteration, `existing_hash` is set to `calculated_hash`, which was just asserted equal to `layer.combined_hash`. Therefore, after the loop, `existing_hash` **is** `last.combined_hash`, which is exactly what `root_hash()` returns. The final check `existing_hash == self.root_hash()` is a tautology — it compares a value to itself and is always `true` when layers are present.

The method never accepts an external trusted root as a parameter. The only real work done is verifying internal chain consistency (each layer hashes correctly from the previous), but the chain is anchored to nothing external. An attacker can freely choose any `node_hash` and build a chain of internally consistent layers, and `valid()` will return `true`. [1](#0-0) 

The `root_hash()` method returns the last `combined_hash` from the proof itself, not from any trusted external source: [2](#0-1) 

Both `valid()` and `root_hash()` are exposed to Python via `py_valid()` and `py_root_hash()`, making this reachable from any Python DataLayer consumer: [3](#0-2) 

The Python type stub confirms both methods are part of the public API: [4](#0-3) 

The fuzz target and tests call `proof.valid()` as the sole validity check, establishing the pattern that callers are expected to rely on `valid()` alone: [5](#0-4) 

### Impact Explanation

Any code that calls `proof.valid()` to gate DataLayer state decisions — without separately comparing `proof.root_hash()` against a known-good, externally trusted root — will accept a fully forged proof. An attacker can:

1. Choose any `node_hash` (e.g., a key-value pair they want to falsely claim is in the DataLayer tree).
2. Build a chain of `ProofOfInclusionLayer` values where each `combined_hash = calculate_internal_hash(prev, side, other_hash)` — trivially computable.
3. Submit the `ProofOfInclusion` to any verifier that calls `proof.valid()`.
4. `valid()` returns `true`; `root_hash()` returns the attacker-chosen last `combined_hash`.

This matches the allowed High impact: **DataLayer Merkle proof logic accepts forged inclusion, letting untrusted input prove invalid state.**

### Likelihood Explanation

The `valid()` method is the only validation method on `ProofOfInclusion`. Its name and signature strongly imply it is a complete validity check. There is no documentation warning that callers must additionally compare `root_hash()` against a trusted external root. The fuzz target and tests use `proof.valid()` as the sole check. Any Python DataLayer consumer following this pattern is vulnerable.

### Recommendation

`valid()` must accept a trusted external root hash as a parameter and compare the computed root against it:

```rust
pub fn valid(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root  // compare against externally trusted root
}
```

Alternatively, keep the current signature but rename it to `is_internally_consistent()` and add a separate `valid(trusted_root: &Hash) -> bool` that calls the internal check and then compares `root_hash()` against `trusted_root`. Update all call sites, the Python binding, and the fuzz target accordingly.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
import hashlib

# Attacker wants to forge proof that fake_node_hash is in some tree
fake_node_hash = bytes([0xAB] * 32)
other_hash     = bytes([0xCD] * 32)

# Compute a valid combined_hash for one layer (left side = 0)
h = hashlib.sha256(b"\x01" + fake_node_hash + other_hash).digest()  # simplified
layer = ProofOfInclusionLayer(
    other_hash_side=0,   # Side.LEFT
    other_hash=other_hash,
    combined_hash=h,     # attacker sets this to whatever calculate_internal_hash produces
)

proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

# valid() returns True — fake_node_hash was never in any real tree
assert proof.valid()
# root_hash() returns attacker-chosen h, not any real tree root
assert proof.root_hash() == h
```

The tautological final check `existing_hash == self.root_hash()` passes because both sides equal `layer.combined_hash` — a value the attacker freely chose. [1](#0-0)
