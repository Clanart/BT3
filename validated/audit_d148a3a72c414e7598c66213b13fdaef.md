### Title
Zero-Fee DynamicFeeTx Commits When `NoBaseFee=true` and `MinGasPrice=0` Due to `nil`→`0` BaseFee Conversion - (`x/evm/keeper/config.go`)

---

### Summary

When `NoBaseFee=true` and London HF is enabled, `EVMBlockConfig` converts a `nil` baseFee from `feemarketParams.GetBaseFee()` into `big.NewInt(0)`. This non-nil zero value propagates through the entire ante handler chain, causing `CheckEthMempoolFee` to skip its local min-gas-price check, allowing a `DynamicFeeTx` with `gasFeeCap=0` and `gasTipCap=0` to pass all fee guards and commit with zero fees deducted from the sender.

### Impact Explanation

An unprivileged attacker can submit a `DynamicFeeTx` with `gasFeeCap=0` and `gasTipCap=0` that executes and commits on-chain without paying any fees. This violates the invariant that committed EVM transactions must deduct fees from the sender. The attacker can:

- Execute arbitrary EVM calls (including value transfers, contract interactions) at zero cost.
- Spam the network with zero-cost transactions, exhausting block gas without economic penalty.
- Drain contract state or trigger side effects without paying for gas.

This matches the **High** impact scope: *"EVM state transition, gas refund, fee market, ante handler, mempool, or proposal handling bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

---

### Likelihood Explanation

The preconditions are realistic and documented:

- `NoBaseFee=true` is an explicitly supported deployment mode documented in the feemarket spec for chains that want to disable dynamic fee adjustment.
- `MinGasPrice=0` is the **default value** (`DefaultMinGasPrice = sdkmath.LegacyZeroDec()`).
- London HF is enabled on virtually all modern Ethermint deployments. [10](#0-9) 

Any chain operator who sets `NoBaseFee=true` without also setting a non-zero `MinGasPrice` (which is the default) is fully exposed. No privileged access is required — any account with a valid signature can exploit this.

---

### Recommendation

In `EVMBlockConfig`, do not silently convert `nil` to `big.NewInt(0)` when `NoBaseFee=true`. Instead, propagate `nil` to signal "no base fee enforcement" and let the ante handlers handle it correctly:

```go
var baseFee *big.Int
if types.IsLondon(ethCfg, ctx.BlockHeight()) {
    baseFee = feemarketParams.GetBaseFee()
    // Do NOT convert nil to 0 here when NoBaseFee=true.
    // nil signals "London HF active but fee market disabled".
}
```

Alternatively, enforce a minimum effective fee floor in `VerifyFee` when `baseFee == big.NewInt(0)` and the effective fee is zero, or require `MinGasPrice > 0` when `NoBaseFee=true` via parameter validation. [1](#0-0)

// DeductFees deducts fees from the given account.
func DeductFees(bankKeeper types.BankKeeper, ctx sdk.Context, acc sdk.AccountI, fees sdk.Coins) error {
	if !fees.IsValid() {
		return errorsmod.Wrapf(errortypes.ErrInsufficientFee, "invalid fee amount: %s", fees)
	}
	if ctx.BlockHeight() > 0 {
		if err := bankKeeper.SendCoinsFromAccountToModuleVirtual(ctx, acc.GetAddress(), authtypes.FeeCollectorName, fees); err != nil {
			return errorsmod.Wrap(errortypes.ErrInsufficientFunds, err.Error())
		}
	}
	return nil
}

func (msg MsgEthereumTx) GetEffectiveGasPrice(baseFee *big.Int) *big.Int {
	tx := msg.AsTransaction()
	if baseFee == nil {
		return tx.GasPrice()
	}
	// for legacy tx, both gasTipCap and gasFeeCap are gasPrice, the result is equavalent.
	return ethermint.BigMin(new(big.Int).Add(tx.GasTipCap(), baseFee), tx.GasFeeCap())
}
