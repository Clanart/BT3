Audit Report

## Title
Stale ETH in `PeripheryPayments.pay` Is Silently Consumed by Subsequent WETH Swaps, Stealing Prior Users' Funds — (File: `metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary
`PeripheryPayments.pay` reads `address(this).balance` — the router's entire native ETH balance — when settling a WETH payment leg. ETH accumulates in the router whenever a caller uses `multicall{value: X}(...)` without appending `refundETH()`. Any subsequent WETH swap by any address then silently consumes that stranded ETH, causing the original depositor to permanently lose their ETH while the later caller pays nothing or receives a discount.

## Finding Description
The root cause is in `pay` at [1](#0-0)  — when `token == WETH` and `payer != address(this)`, the function reads `address(this).balance` (the router's total ETH, not the current call's `msg.value`) and uses it first before pulling from the payer.

ETH enters the router through the `payable` `multicall` function: [2](#0-1) 

The `receive()` guard only blocks plain ETH transfers from non-WETH addresses: [3](#0-2)  — it does not prevent ETH from entering via a `payable` function call, so ETH sent with `multicall` is not rejected.

When a user calls `multicall{value: X}(...)` and omits `refundETH()`, the surplus ETH is permanently stranded. The next WETH swap triggers `_justPayCallback` → `pay(WETH, payer, pool, value)`, which reads the stranded balance and uses it to wrap and transfer WETH to the pool without ever calling `safeTransferFrom` on the actual payer. The same drain path exists in `MetricOmmPoolLiquidityAdder` via the shared `pay` call: [4](#0-3) 

## Impact Explanation
Direct, irreversible loss of user principal. When `address(this).balance >= value`, the stranded ETH fully covers the swap cost and `safeTransferFrom` is never called on the attacker — they receive full swap output for free. When `0 < address(this).balance < value`, the victim loses their stranded ETH and the attacker receives a proportional discount. This is a Critical/High direct loss of user funds with no recovery path, meeting Sherlock thresholds.

## Likelihood Explanation
Medium-to-high. The trigger — sending ETH via `multicall` without `refundETH()` — is a realistic and documented usage pattern for native-ETH swaps. No special permissions are required to exploit it: any address can call `exactInputSingle` or `exactOutputSingle` with WETH as `tokenIn` immediately after ETH becomes stranded. The exploit is repeatable and permissionless.

## Recommendation
Track only the ETH belonging to the current call. The standard fix (used by Uniswap v3's `SwapRouter`) is to thread `msg.value` through the call chain and cap native consumption to that amount inside `pay`. A minimal targeted fix is to compare `address(this).balance` against `msg.value` and only consume native ETH when it was explicitly provided in the current top-level call. Alternatively, enforce at the `multicall` level that any batch sending ETH must include `refundETH()` as its final call — but this is a social/documentation control, not a code-level invariant, and is insufficient on its own.

## Proof of Concept
```
1. User A calls:
       router.multicall{value: 1 ether}([
           exactInputSingle(tokenIn=WETH, amountIn=0.5 ether, ...)
           // refundETH() omitted
       ])
   → 0.5 ETH remains stranded in the router.

2. User B (attacker) calls with zero msg.value:
       router.exactInputSingle(
           tokenIn=WETH, amountIn=0.5 ether, recipient=attacker, ...
       )

3. Pool calls router.metricOmmSwapCallback(...)
   → _justPayCallback → pay(WETH, userB, pool, 0.5 ether)

4. Inside pay() [PeripheryPayments.sol L74]:
       nativeBalance = address(this).balance = 0.5 ether  // User A's stranded ETH
       nativeBalance >= value → branch 1 taken
       IWETH9(WETH).deposit{value: 0.5 ether}()           // wraps User A's ETH
       IERC20(WETH).safeTransfer(pool, 0.5 ether)         // pays pool
       // safeTransferFrom(userB, ...) is NEVER called

5. Result:
   - User B receives full swap output, pays 0 WETH.
   - User A loses 0.5 ETH with no recourse.
```
