### Title
Missing Destination Chain ID in `deployToken` Signature Hash Enables Cross-Chain Replay of Token Deployment Signatures — (`evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

The `deployToken` function in `OmniBridge.sol` constructs its signature hash without including the destination chain ID (`omniBridgeChainId`). In contrast, `finTransfer` correctly binds its hash to the destination chain. Because all EVM OmniBridge deployments share the same `nearBridgeDerivedAddress` (the NEAR MPC-derived key), a valid `deployToken` signature obtained for one EVM chain (e.g., Ethereum) can be replayed verbatim on every other EVM chain (Arbitrum, Base, BNB, Polygon, HyperEvm, Abstract) to deploy the same bridge token without explicit NEAR MPC authorization for those chains.

---

### Finding Description

`OmniBridge.deployToken` builds its borsh-encoded hash as:

```solidity
bytes memory borshEncoded = bytes.concat(
    bytes1(uint8(BridgeTypes.PayloadType.Metadata)),
    Borsh.encodeString(metadata.token),
    Borsh.encodeString(metadata.name),
    Borsh.encodeString(metadata.symbol),
    bytes1(metadata.decimals)
);
bytes32 hashed = keccak256(borshEncoded);
``` [1](#0-0) 

No chain-identifying field is included. The signature is then verified against `nearBridgeDerivedAddress`: [2](#0-1) 

Compare this with `finTransfer`, which correctly embeds `omniBridgeChainId` twice (once for the destination token address, once for the recipient) in its borsh payload:

```solidity
bytes1(omniBridgeChainId),
Borsh.encodeAddress(payload.tokenAddress),
...
bytes1(omniBridgeChainId),
Borsh.encodeAddress(payload.recipient),
``` [3](#0-2) 

Because `deployToken`'s hash is chain-agnostic, the identical `(signatureData, metadata)` tuple accepted on Ethereum is also accepted on Arbitrum, Base, BNB, Polygon, HyperEvm, and Abstract — all of which share the same `nearBridgeDerivedAddress` derived from the NEAR MPC key.

The `PayloadType` enum confirms `Metadata = 1` is the only prefix used for token deployment across all chains: [4](#0-3) 

The Solana `DeployTokenPayload` has the same structural omission — no chain ID in its serialization: [5](#0-4) 

---

### Impact Explanation

An attacker who observes a valid `deployToken` transaction on any EVM chain can replay it on every other EVM OmniBridge deployment:

1. **Unauthorized bridge token deployment**: The replayed call passes signature verification and deploys a bridge token on the target chain. The NEAR MPC never authorized deployment on that specific chain.
2. **Permanent blocking of legitimate deployment**: After the replay, `nearToEthToken[metadata.token]` is set on the target chain. Any future legitimate `deployToken` call for the same NEAR token on that chain will revert with `ERR_TOKEN_EXIST`, permanently preventing the NEAR bridge from deploying the canonical token there. [6](#0-5) 

This matches the allowed impact: **"Proof, signature, MPC … verification bypass enabling unauthorized … token deployment."**

---

### Likelihood Explanation

- The attack requires only a publicly visible `deployToken` transaction on any EVM chain — no privileged access, no key material, no MPC collusion.
- Any unprivileged observer of on-chain data can extract `(signatureData, metadata)` and submit it to any other EVM OmniBridge.
- The NEAR bridge is deployed on at least six EVM chains (`Eth`, `Arb`, `Base`, `Bnb`, `Pol`, `HyperEvm`, `Abs`), all sharing the same MPC-derived signer address.
- Likelihood is **High**.

---

### Recommendation

Include `omniBridgeChainId` in the borsh-encoded payload for `deployToken`, mirroring the pattern already used in `finTransfer`:

```solidity
bytes memory borshEncoded = bytes.concat(
    bytes1(uint8(BridgeTypes.PayloadType.Metadata)),
+   bytes1(omniBridgeChainId),          // bind to destination chain
    Borsh.encodeString(metadata.token),
    Borsh.encodeString(metadata.name),
    Borsh.encodeString(metadata.symbol),
    bytes1(metadata.decimals)
);
```

Apply the same fix to the Solana `DeployTokenPayload::serialize_for_near` by prepending `SOLANA_OMNI_BRIDGE_CHAIN_ID` before the metadata fields. [5](#0-4) 

The NEAR MPC signing path must also include the destination chain kind in the `MetadataPayload` hash so that signatures are chain-specific.

---

### Proof of Concept

1. NEAR MPC signs a `MetadataPayload` for token `"usdc.near"` with `name="USD Coin"`, `symbol="USDC"`, `decimals=6`. The resulting `signatureData` is broadcast in a `deployToken` transaction on Ethereum (`omniBridgeChainId = 0`).

2. Attacker copies `(signatureData, metadata)` from the Ethereum transaction.

3. Attacker calls `deployToken(signatureData, metadata)` on the Arbitrum OmniBridge (`omniBridgeChainId = 3`).

4. The hash computed on Arbitrum is identical to the one computed on Ethereum (no chain ID in the hash), so `ECDSA.recover(hashed, signatureData) == nearBridgeDerivedAddress` passes.

5. A new `BridgeToken` for `"usdc.near"` is deployed on Arbitrum. `nearToEthToken["usdc.near"]` is now set on Arbitrum.

6. When the NEAR bridge later legitimately attempts to deploy USDC on Arbitrum, the call reverts with `ERR_TOKEN_EXIST`, permanently blocking the canonical deployment. [7](#0-6)

    function deployToken(
        bytes calldata signatureData,
        BridgeTypes.MetadataPayload calldata metadata
    ) external payable whenNotPaused(PAUSED_DEPLOY_TOKEN) returns (address) {
        if (tokenImplementationAddress == address(0)) {
            revert TokenImplementationNotSet();
        }
        bytes memory borshEncoded = bytes.concat(
            bytes1(uint8(BridgeTypes.PayloadType.Metadata)),
            Borsh.encodeString(metadata.token),
            Borsh.encodeString(metadata.name),
            Borsh.encodeString(metadata.symbol),
            bytes1(metadata.decimals)
        );
        bytes32 hashed = keccak256(borshEncoded);

        if (ECDSA.recover(hashed, signatureData) != nearBridgeDerivedAddress) {
            revert InvalidSignature();
        }

        require(
            !isBridgeToken[nearToEthToken[metadata.token]],
            "ERR_TOKEN_EXIST"
        );
        uint8 decimals = _normalizeDecimals(metadata.decimals);

        // slither-disable-next-line reentrancy-no-eth
        address bridgeTokenProxy = address(
            new ERC1967Proxy(
                tokenImplementationAddress,
                abi.encodeWithSelector(
                    BridgeToken.initialize.selector,
                    metadata.name,
                    metadata.symbol,
                    decimals
                )
            )
        );

        deployTokenExtension(
            metadata.token,
            bridgeTokenProxy,
            decimals,
            metadata.decimals
        );

        emit BridgeTypes.DeployToken(
            bridgeTokenProxy,
            metadata.token,
            metadata.name,
            metadata.symbol,
            decimals,
            metadata.decimals
        );

        isBridgeToken[address(bridgeTokenProxy)] = true;
        ethToNearToken[address(bridgeTokenProxy)] = metadata.token;
        nearToEthToken[metadata.token] = address(bridgeTokenProxy);

        return bridgeTokenProxy;
    }
