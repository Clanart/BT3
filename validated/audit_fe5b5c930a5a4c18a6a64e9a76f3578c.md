### Title
Missing Zero-Check on `class_hash` in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `execution/deprecated_execute_syscalls.cairo` and `execution/syscall_impls.cairo`)

---

### Summary

Both implementations of `execute_replace_class` accept an attacker-supplied `class_hash` of zero (`UNINITIALIZED_CLASS_HASH`) without validation. A contract that calls `replace_class(0)` will have its class hash overwritten with the sentinel "uninitialized" value, making the contract permanently uncallable and freezing any funds it holds.

---

### Finding Description

`UNINITIALIZED_CLASS_HASH` is defined as `0` in `state/commitment.cairo`: [1](#0-0) 

The deprecated syscall implementation of `execute_replace_class` reads the caller-supplied `class_hash` and writes it directly to `contract_state_changes` with no non-zero check: [2](#0-1) 

The new syscall implementation in `syscall_impls.cairo` has the identical omission. The developer even left a TODO acknowledging the missing validation: [3](#0-2) 

By contrast, `deploy_contract` correctly asserts that the *existing* class hash is `UNINITIALIZED_CLASS_HASH` before deploying: [4](#0-3) 

And `execute_declare_transaction` correctly asserts `compiled_class_hash` is non-zero before writing it: [5](#0-4) 

Neither `execute_replace_class` implementation applies an equivalent guard on the *incoming* `class_hash`.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

When a contract's `class_hash` is set to `0` (`UNINITIALIZED_CLASS_HASH`), the OS treats it as an undeployed address. All subsequent calls to that contract will fail to resolve any entry point, making the contract permanently uncallable. Any ERC-20 tokens, ETH, or other assets held in the contract's storage become irretrievably frozen. The state change is committed to the Merkle tree and cannot be reversed by any protocol mechanism.

---

### Likelihood Explanation

The attack is reachable by any contract deployer — an explicitly listed unprivileged actor. A malicious deployer can:

1. Deploy a contract that appears legitimate (e.g., a token vault or multisig).
2. Attract user deposits.
3. Trigger an internal call to `replace_class(0)` (e.g., via a hidden backdoor or a rug-pull condition).

The OS will accept the syscall, commit `class_hash=0` to state, and the contract is permanently bricked. No privileged role or external dependency is required. The attack is a single transaction.

---

### Recommendation

Add an `assert_not_zero(class_hash)` guard in both `execute_replace_class` implementations immediately after reading the request field, mirroring the pattern used in `execute_declare_transaction`:

```cairo
let class_hash = request.class_hash;  // or syscall_ptr.class_hash
with_attr error_message("replace_class: class_hash must not be zero.") {
    assert_not_zero(class_hash);
}
```

This is the direct analog of the audited recommendation: "Add the check that the signatoryAddress is not zero address (either into the initializer or into the function)."

---

### Proof of Concept

1. Deploy a Cairo contract containing:
   ```cairo
   // Inside any external entry point:
   replace_class(class_hash=0);
   ```
2. Call that entry point via an `invoke` transaction.
3. The OS executes `execute_replace_class` (either path), reads `class_hash=0`, and writes `StateEntry(class_hash=0, ...)` to `contract_state_changes` via `dict_update`.
4. The block is proven and the state root is updated.
5. Any subsequent call to the contract resolves `class_hash=0`, finds no compiled class, and reverts — permanently. All funds in the contract are frozen.

The root cause is the absence of `assert_not_zero(class_hash)` in:

```text
func execute_replace_class{contract_state_changes: DictAccess*, revert_log: RevertLogEntry*}(
    contract_address, syscall_ptr: ReplaceClass*
) {
    alloc_locals;
    let class_hash = syscall_ptr.class_hash;

    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
```
