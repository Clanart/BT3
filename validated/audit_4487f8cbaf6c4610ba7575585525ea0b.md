### Title
Integer Truncation in `NewDynamicFeeChecker` Produces Zero `effectiveFee`, Allowing Fee-Free Cosmos Tx Execution - (File: `ante/evm/fee_checker.go`)

---

### Summary

`NewDynamicFeeChecker` computes `feeCap` via integer division `fee / gas`. When the total fee amount (in the smallest denomination) is less than the gas limit, integer truncation yields `feeCap = 0`. With `baseFee = 0` (London enabled + `NoBaseFee = true`), `EffectiveGasPrice` returns 0, so `effectiveFee = 0 * gas = 0`. The `DeductFeeDecorator` then deducts zero fees, allowing any Cosmos transaction to execute without paying fees.

---

### Finding Description

In `ante/evm/fee_checker.go`, the `NewDynamicFeeChecker` function computes the per-gas fee cap using integer division:

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))   // line 83
```

`fee` is `sdk.Int` (integer, in smallest denomination, e.g. aevmos). `gas` is `uint64`. Go's `sdk.Int.Quo` performs truncating integer division. If `fee < gas` (e.g. `fee = 1`, `gas = 21000`), `feeCap = 0`.

The subsequent guard:

```go
if feeCap.LT(baseFeeInt) { ... }   // line 86
```

only rejects the tx when `baseFee > 0`. When `baseFee = 0` (London enabled, `NoBaseFee = true`), `0 < 0` is false — the tx is not rejected.

`EffectiveGasPrice` is then called:

```go
effectivePrice := sdkmath.NewIntFromBigInt(types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
// x/evm/types/utils.go line 232-234:
// return BigMin(tipCap + baseFee, feeCap) = BigMin(maxPriorityPrice + 0, 0) = 0
```

With `feeCap = 0` and `baseFee = 0`, `EffectiveGasPrice` returns `min(maxPriorityPrice, 0) = 0`.

The returned `effectiveFee` is therefore:

```go
effectiveFee := sdk.Coins{{Denom: denom, Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas))}}
// = 0 * gas = 0
```

`DeductFeeDecorator` (used via `ante.NewDeductFeeDecorator` in `newCosmosAnteHandler`) deducts the `effectiveFee` returned by `txFeeChecker`. The custom `evm.DeductFeeDecorator` also explicitly skips deduction when `fee.IsZero()`:

```go
if !fee.IsZero() {
    err := evmkeeper.DeductFees(dfd.bankKeeper, ctx, deductFeesFromAcc, fee)
}
```

Result: the transaction executes with zero fees deducted.

The `MinGasPriceDecorator` (which runs before `DeductFeeDecorator` in `newCosmosAnteHandler`) short-circuits when `MinGasPrice.IsZero()`, providing no backstop when the global minimum gas price is also zero.

---

### Impact Explanation

An unprivileged attacker can submit Cosmos SDK transactions (non-Ethereum, e.g. `MsgSend`, `MsgDelegate`, IBC relayer messages) and have them executed with zero fee deduction. This is a direct fee mis-accounting bug: the user's declared fee coins are never transferred to the fee collector. At scale, this drains validator/staker fee revenue and allows spam transactions at no cost, matching the allowed High impact: *"ante handler bug that permits valid user funds/fees to be mis-accounted."*

---

### Likelihood Explanation

Requires three conditions to hold simultaneously:
1. `DynamicFeeChecker = true` in `HandlerOptions` (opt-in per chain deployment).
2. London hardfork enabled (so `GetBaseFee` returns `new(big.Int)` = 0 rather than `nil`, keeping the EIP-1559 path active).
3. `NoBaseFee = true` in feemarket params (so `baseFee = 0`), AND `MinGasPrice = 0`.

Conditions 2+3 together describe a chain that has enabled London but disabled the dynamic fee market — a common configuration for chains that want EIP-1559 tx types without fee burning. Condition 1 is a deployment choice. All three are realistic and documented configurations. The attacker input is trivial: any Cosmos tx with `fee_amount < gas_limit` in the smallest denomination (e.g. 1 aevmos with gas 21000).

---

### Recommendation

Replace the truncating integer division with decimal arithmetic before comparing against `baseFee`, mirroring the approach used in `MinGasPriceDecorator`:

```go
// Current (truncates to 0 when fee < gas):
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))

// Fixed (preserves sub-unit precision):
feeCapDec := sdkmath.LegacyNewDecFromInt(fee).QuoInt64(int64(gas))
feeCap := feeCapDec.TruncateInt()
```

Alternatively, restructure the check to avoid dividing first: verify `fee >= baseFee * gas` directly (multiply before dividing), consistent with how `CheckEthMinGasPrice` computes `requiredFee = minGasPrice.Mul(gasLimit)` without dividing the fee first.

### Proof of Concept

**Setup:** Chain with London hardfork enabled, `NoBaseFee = true`, `MinGasPrice = 0`, `DynamicFeeChecker = true`.

**Attacker action:** Submit a Cosmos `MsgSend` tx with:
- `fee_amount = [{denom: "aevmos", amount: "1"}]`
- `gas_limit = 21000`

**Trace through `NewDynamicFeeChecker`:**

