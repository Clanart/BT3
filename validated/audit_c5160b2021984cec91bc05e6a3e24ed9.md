### Title
Caller-Controlled `vault = None` in `finalize_transfer` Allows Mint-Path Bypass for Vault-Registered Tokens, Enabling Unbacked Token Minting — (`solana/programs/bridge_token_factory/src/instructions/user/finalize_transfer.rs`)

### Summary
`finalize_transfer` selects between vault-release and mint-to based solely on whether the caller passes the `vault` account. An attacker who controls a token's `mint_authority` can register the token via `log_metadata` (creating the vault), transfer `mint_authority` to the bridge authority, then call `finalize_transfer` with `vault = None` using a legitimately obtained NEAR-signed payload, causing the program to mint new tokens instead of releasing from the vault — doubling supply while vault collateral remains locked.

### Finding Description
`log_metadata` enforces `!mint.mint_authority.contains(authority.key)` and creates a vault PDA via `init_if_needed`. This vault signals to `finalize_transfer` that the token is "native" (vault-based). However, `finalize_transfer` declares `vault` as `Option<Box<InterfaceAccount<'info, TokenAccount>>>`. In Anchor, `Option<Account>` is purely caller-controlled: the caller can pass `None` regardless of whether the PDA exists on-chain. The program never checks for vault PDA existence when `vault = None`. The branch at lines 101–136 of `finalize_transfer.rs` unconditionally takes the mint path when `vault` is `None`, after only verifying `mint_authority == authority`. Since the attacker transferred `mint_authority` to the bridge authority in step 2, this check passes, and `mint_to` is called.

### Impact Explanation
The attacker mints unbacked tokens on Solana while the vault retains the original collateral. This breaks the collateralization invariant: vault-registered tokens must only be released from the vault, never minted. The vault tokens become permanently locked (no mechanism releases them), and the circulating supply is inflated by the minted amount. Impact: **unauthorized mint of unbacked tokens + permanent lock of vault collateral**.

### Likelihood Explanation
The attacker only needs to: (1) create an SPL token (trivial, permissionless), (2) call `log_metadata` (permissionless), (3) transfer their own `mint_authority` to the bridge PDA (trivial SPL instruction), (4) perform a round-trip bridge of their own tokens to obtain a valid NEAR-signed payload (normal bridge operation), (5) call `finalize_transfer` with `vault = None` (permissionless). No privileged access, no key compromise, no colluding validators required.

### Recommendation
Remove the `Option` from the `vault` field and instead derive whether to use vault or mint path from on-chain state. Specifically:
- Make `vault` a required account but allow it to be uninitialized, checking `vault.data_is_empty()` to distinguish paths; OR
- Store a per-mint flag in a PDA during `log_metadata` that records "this token uses vault path" and enforce it in `finalize_transfer`; OR
- At minimum, when `vault = None`, verify that no vault PDA exists for the mint by checking the derived PDA address is uninitialized.

### Proof of Concept
```
1. Create SPL mint M with mint_authority = attacker keypair A
2. Call log_metadata(M) → vault PDA V = PDA[VAULT_SEED, M] is created
3. Call init_transfer(M, 1000) → 1000 tokens deposited into V; NEAR mints 1000 NEAR-M to A
4. Call SPL set_authority(M, mint_authority, bridge_authority_PDA) as A
5. On NEAR: call init_transfer(NEAR-M, 1000) → NEAR MPC signs FinalizeTransferPayload(nonce, M, 1000, A)
6. On Solana: call finalize_transfer(signed_payload, vault=None)
   → mint_authority check passes (bridge authority == authority)
   → mint_to(M, A, 1000) executes
7. Assert: A holds 1000 freshly minted M tokens on Solana
8. Assert: V still holds 1000 M tokens (never released)
9. Total supply of M increased by 1000 with no backing
```

    pub fn verify_signature(
        &self,
        params: P::AdditionalParams,
        derived_near_bridge_address: &[u8; 64],
    ) -> Result<()> {
        let serialized = self.payload.serialize_for_near(params)?;
        let hash = keccak::hash(&serialized);

        let signature_bytes = &self.signature[0..64];

        let signature = libsecp256k1::Signature::parse_standard_slice(signature_bytes)
            .map_err(|_| ProgramError::InvalidArgument)?;
        require!(!signature.s.is_high(), ErrorCode::MalleableSignature);

        let signer = secp256k1_recover(&hash.to_bytes(), self.signature[64], signature_bytes)
            .map_err(|_| error!(ErrorCode::SignatureVerificationFailed))?;

        require!(
            signer.0 == *derived_near_bridge_address,
            ErrorCode::SignatureVerificationFailed
        );

        Ok(())
    }
