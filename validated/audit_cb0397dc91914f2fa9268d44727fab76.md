### Title
Missing Signature Verification in Bootstrap Declare Path Allows Unauthorized Class Declaration - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

The `execute_declare_transaction` function contains a special bootstrap path that skips both signature verification (`run_validate`) and nonce checking (`check_and_increment_nonce`) for any transaction whose fields satisfy trivially-controllable conditions. An unprivileged attacker can craft a declare transaction meeting these conditions to register an arbitrary `compiled_class_hash` for any Sierra class hash without possessing any private key.

---

### Finding Description

In `execute_declare_transaction`, after computing the transaction hash and filling `tx_info` (including loading a signature that is never verified), the following early-return path exists: [1](#0-0) 

```cairo
if (sender_address == 'BOOTSTRAP' and tx_info.nonce == 0 and tx_info.version == 3) {
    let max_possible_fee = compute_max_possible_fee(tx_info=tx_info);
    if (max_possible_fee == 0) {
        assert_not_zero(compiled_class_hash);
        dict_update{dict_ptr=contract_class_changes}(
            key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
        );
        %{ SkipTx %}
        return ();
    }
}
```

This path returns **before** reaching:

- `check_and_increment_nonce` at line 779 — nonce is never incremented for the `'BOOTSTRAP'` address
- `run_validate` at lines 799–812 — the account contract's `__validate_declare__` entry point (which performs signature verification) is never called [2](#0-1) 

The signature is loaded into `tx_info` via `%{ GenSignatureArg %}` inside `fill_account_tx_info`: [3](#0-2) 

but is never cryptographically verified because `run_validate` is bypassed.

The four conditions gating this path are all trivially satisfiable by any unprivileged user:

| Condition | How attacker satisfies it |
|---|---|
| `sender_address == 'BOOTSTRAP'` | Use felt literal `0x424f4f545354524150` as sender |
| `tx_info.nonce == 0` | Set nonce field to 0 |
| `tx_info.version == 3` | Use standard v3 transaction |
| `max_possible_fee == 0` | Set all resource bounds to 0 |

Because the nonce is **never incremented** for the `'BOOTSTRAP'` address in this path, the attacker can submit multiple transactions each with `nonce=0`, declaring multiple different class hashes in the same or successive blocks.

The `class_hash` itself is constrained to be a valid Sierra class hash (verified via `finalize_class_hash` at lines 738–743): [4](#0-3) 

However, the `compiled_class_hash` stored in `contract_class_changes` is **entirely attacker-controlled** and is not verified against any actual compiled class artifact. The `dict_update` call at line 770 enforces `prev_value=0`, meaning each class hash can only be declared once — making a malicious declaration permanent and irreversible.

---

### Impact Explanation

**Permanent freezing of funds (Critical).**

An attacker front-runs a legitimate class declaration by submitting a bootstrap-path declare transaction for the same `class_hash` with an arbitrary (invalid or malicious) `compiled_class_hash`. Because `prev_value=0` is enforced, the legitimate declaration subsequently fails. All contracts deployed under that `class_hash` will reference a `compiled_class_hash` that either does not exist or maps to wrong bytecode. Execution of those contracts will permanently fail, freezing any funds held within them. Since class declarations are irreversible on StarkNet, there is no recovery path.

---

### Likelihood Explanation

High. The four conditions are trivially satisfiable with no privileged access, no leaked key, and no special tooling. Any user who can submit a declare transaction to the sequencer can trigger this path. The attacker only needs to observe the mempool or anticipate which class hashes will be declared (e.g., by watching a deployment pipeline) and submit a racing transaction first.

---

### Recommendation

Remove the bootstrap path entirely, or gate it with a proper authorization mechanism. If bootstrapping is a genuine protocol requirement, it should be enforced at the block-level (e.g., only permitted in block 0, or only when a specific sequencer-signed field is present and verified). At minimum, the path must not skip `run_validate`.

```cairo
// Remove or restrict:
// if (sender_address == 'BOOTSTRAP' and tx_info.nonce == 0 and tx_info.version == 3) { ... }
```

If the bootstrap path must be retained, add a verifiable authorization check — analogous to the fix in the reference report — before the early return:

```cairo
// Require a valid signature from a known bootstrap authority key,
// verified against the transaction hash, before allowing the bypass.
```

---

### Proof of Concept

1. Attacker identifies a Sierra class `C` with `class_hash = H` that a legitimate protocol deployer is about to declare.
2. Attacker crafts a declare transaction:
   - `sender_address = 'BOOTSTRAP'` (felt `0x424f4f545354524150`)
   - `nonce = 0`
   - `version = 3`
   - All resource bounds = `0` (so `max_possible_fee = 0`)
   - `class_hash = H` (valid Sierra class hash, with matching `contract_class_component_hashes`)
   - `compiled_class_hash = ATTACKER_VALUE` (arbitrary felt, e.g., `1`)
   - Signature: any value (never verified)
3. Attacker submits this transaction. The OS processes it via the bootstrap path:
   - `run_validate` is skipped — no signature check occurs
   - `check_and_increment_nonce` is skipped — nonce stays at 0
   - `dict_update(key=H, prev_value=0, new_value=ATTACKER_VALUE)` executes
4. The legitimate declare transaction for `H` now fails: `prev_value` is `ATTACKER_VALUE`, not `0`.
5. All contracts deployed with `class_hash = H` reference `compiled_class_hash = ATTACKER_VALUE`, which does not correspond to any valid compiled class. Their execution permanently fails, freezing all funds held in those contracts.
6. Attacker repeats with `nonce=0` for additional class hashes, since the `'BOOTSTRAP'` address nonce is never incremented. [5](#0-4)

func execute_declare_transaction{
    range_check_ptr,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*) {
    alloc_locals;

    local tx_version;
    %{ TxVersion %}
    if (tx_version == 0) {
        %{ SkipTx %}
        return ();
    }

    // Guess transaction fields.
    local sender_address;
    local class_hash_ptr: felt*;
    local compiled_class_hash;
    local account_deployment_data_size;
    local account_deployment_data: felt*;
    %{ DeclareTxFields %}
    let common_tx_fields = get_account_tx_common_fields(
        block_context=block_context,
        tx_hash_prefix=DECLARE_HASH_PREFIX,
        sender_address=sender_address,
    );

    let poseidon_ptr = builtin_ptrs.selectable.poseidon;
    with poseidon_ptr {
        // Compute transaction hash.
        let transaction_hash = compute_declare_transaction_hash(
            common_fields=common_tx_fields,
            class_hash=[class_hash_ptr],
            compiled_class_hash=compiled_class_hash,
            account_deployment_data_size=account_deployment_data_size,
            account_deployment_data=account_deployment_data,
        );
        %{ AssertTransactionHash %}

        // Ensure the given class hash is a result of a Sierra class hash calculation.
        local contract_class_component_hashes: ContractClassComponentHashes*;
        %{ SetComponentHashes %}

        let expected_class_hash = finalize_class_hash(
            contract_class_component_hashes=contract_class_component_hashes
        );
        with_attr error_message("Invalid class hash pre-image.") {
            assert [class_hash_ptr] = expected_class_hash;
        }
    }
    update_poseidon_in_builtin_ptrs(poseidon_ptr=poseidon_ptr);

    // Get the account transaction info.
    let (tx_info: TxInfo*) = alloc();
    let (deprecated_tx_info: DeprecatedTxInfo*) = alloc();
    fill_account_tx_info(
        transaction_hash=transaction_hash,
        common_tx_fields=common_tx_fields,
        account_deployment_data_size=account_deployment_data_size,
        account_deployment_data=account_deployment_data,
        proof_facts_size=0,
        proof_facts=cast(0, felt*),
        tx_info_dst=tx_info,
        deprecated_tx_info_dst=deprecated_tx_info,
    );

    // Do not run validate or perform any account-related actions for declare transactions that
    // meet the following conditions.
    // This flow is used for the sequencer to bootstrap a new system.
    if (sender_address == 'BOOTSTRAP' and tx_info.nonce == 0 and tx_info.version == 3) {
        let max_possible_fee = compute_max_possible_fee(tx_info=tx_info);
        if (max_possible_fee == 0) {
            // Declare the class hash and skip the rest of the transaction.
            // Note that prev_value=0 enforces that a class may be declared only once.
            assert_not_zero(compiled_class_hash);
            dict_update{dict_ptr=contract_class_changes}(
                key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
            );
            %{ SkipTx %}
            return ();
        }
    }
