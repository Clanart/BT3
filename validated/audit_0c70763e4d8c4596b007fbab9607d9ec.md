### Title
`_build_thin_state_diff` and `_compute_state_diff_length` silently drop L2-DA storage updates and nonces, producing a wrong `state_diff_commitment` and block hash — (`echonet/echo_center.py`)

### Summary

`BlobTransformer._build_thin_state_diff` and `_compute_state_diff_length` in `echonet/echo_center.py` unconditionally read only `storage_updates["L1"]` and `nonces["L1"]` from the blob's state diff, silently discarding every entry stored under the `"L2"` key. When a block contains storage updates or nonces with L2 DA mode, the `state_diff_commitment` fed to `calculate_block_commitments` is derived from an incomplete state diff, `state_diff_length` is under-counted, `concatenated_counts` is wrong, and the final block hash stored in the block document is wrong. The state update document produced by `transform_state_update` is also missing those L2 entries, so `starknet_getStateUpdate` returns an authoritative-looking but incorrect value.

### Finding Description

`_build_thin_state_diff` (line 494–502) constructs the thin state diff that is passed directly to the block-hash CLI for commitment calculation:

```python
return {
    "deployed_contracts": state_diff["address_to_class_hash"],
    "storage_diffs": state_diff["storage_updates"]["L1"],   # L2 entries silently dropped
    "class_hash_to_compiled_class_hash": state_diff["class_hash_to_compiled_class_hash"],
    "deprecated_declared_classes": [],
    "nonces": state_diff["nonces"]["L1"],                   # L2 entries silently dropped
}
``` [1](#0-0) 

`_compute_state_diff_length` (line 552–560) repeats the same omission when computing the length that is stored as `state_diff_length` in the block document:

```python
storage_updates = state_diff["storage_updates"]["L1"]   # L2 entries silently dropped
return (
    len(state_diff["address_to_class_hash"])
    + len(state_diff["class_hash_to_compiled_class_hash"])
    + len(state_diff["nonces"]["L1"])                   # L2 entries silently dropped
    + sum(len(slots) for slots in storage_updates.values())
)
``` [2](#0-1) 

`transform_block` calls both functions and stores their results:

```python
block_commitments = self._compute_block_commitments(blob, ...)   # uses _build_thin_state_diff
block_document["state_diff_length"] = self._compute_state_diff_length(blob)
block_document["block_hash"] = self._compute_block_hash(...)
``` [3](#0-2) 

`transform_state_update` (line 780–829) also reads only `storage_updates["L1"]` and `nonces["L1"]`, so the stored state-update document is missing L2 entries: [4](#0-3) 

The downstream commitment path is:

1. `_build_thin_state_diff` → `_compute_block_commitments` → block-hash CLI → `state_diff_commitment` (wrong, missing L2 slots).
2. `_compute_state_diff_length` → `state_diff_length` (under-counted).
3. `state_diff_length` feeds `concat_counts` → `concatenated_counts` (wrong packed field).
4. Both wrong values enter `calculate_block_hash` → wrong `block_hash` stored in the block document.

The canonical Rust path shows that `calculate_block_commitments` takes a full `ThinStateDiff` and encodes `state_diff.len()` into `concatenated_counts`, which is then chained into the block hash: [5](#0-4) [6](#0-5) 

`ThinStateDiff.len()` counts every storage slot across all contracts: [7](#0-6) 

Dropping L2 slots from the thin state diff therefore produces a strictly smaller `len()`, a wrong `concatenated_counts`, a wrong `state_diff_commitment`, and a wrong block hash.

### Impact Explanation

Any block whose blob carries storage updates or nonces under `storage_updates["L2"]` / `nonces["L2"]` will have:

- A `state_diff_commitment` that does not match the commitment computed by the sequencer's own Rust path (`BlockExecutionArtifacts::new` → `calculate_block_commitments`).
- A `state_diff_length` that is smaller than the true length, corrupting `concatenated_counts` inside the block hash.
- A `block_hash` in the stored block document that diverges from the canonical hash.
- A `starknet_getStateUpdate` response that silently omits L2-DA storage diffs and nonces — an authoritative-looking wrong value.

This matches **High: RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value**, and potentially **Critical: Wrong state, receipt, event, L1 message, class hash, storage value, or revert result** if downstream consumers rely on the stored block hash or state update for proof verification or state reconstruction.

### Likelihood Explanation

The blob's `block_info.use_kzg_da` flag is already read by `_compute_block_commitments` to set the `l1_da_mode` parameter, confirming that KZG/blob DA mode is a live code path. Any block produced with `use_kzg_da = true` that has storage updates or nonce changes will have those entries under `"L2"`, triggering the omission on every such block. [8](#0-7) 

### Recommendation

Merge both DA-mode buckets before building the thin state diff and computing its length:

```python
@staticmethod
def _build_thin_state_diff(blob: JsonObject) -> JsonObject:
    state_diff = blob["state_diff"]
    storage_updates = {
        **state_diff["storage_updates"].get("L1", {}),
        **state_diff["storage_updates"].get("L2", {}),
    }
    nonces = {
        **state_diff["nonces"].get("L1", {}),
        **state_diff["nonces"].get("L2", {}),
    }
    return {
        "deployed_contracts": state_diff["address_to_class_hash"],
        "storage_diffs": storage_updates,
        "class_hash_to_compiled_class_hash": state_diff["class_hash_to_compiled_class_hash"],
        "deprecated_declared_classes": [],
        "nonces": nonces,
    }

@staticmethod
def _compute_state_diff_length(blob: JsonObject) -> int:
    state_diff = blob["state_diff"]
    storage_updates = {
        **state_diff["storage_updates"].get("L1", {}),
        **state_diff["storage_updates"].get("L2", {}),
    }
    nonces = {
        **state_diff["nonces"].get("L1", {}),
        **state_diff["nonces"].get("L2", {}),
    }
    return (
        len(state_diff["address_to_class_hash"])
        + len(state_diff["class_hash_to_compiled_class_hash"])
        + len(nonces)
        + sum(len(slots) for slots in storage_updates.values())
    )
```

Apply the same merge in `transform_state_update`. Add a unit test that supplies a blob with non-empty `storage_updates["L2"]` and asserts that the computed `state_diff_length` and `state_diff_commitment` match the values produced by the Rust `calculate_block_commitments` path.

### Proof of Concept

1. Construct a blob with `use_kzg_da = true` and place one storage update under `state_diff["storage_updates"]["L2"]` (e.g., `{"0xABC": {"0x1": "0x2"}}`), leaving `state_diff["storage_updates"]["L1"]` empty.
2. Call `BlobTransformer._build_thin_state_diff(blob)` — the returned dict has `"storage_diffs": {}` (empty), silently dropping the L2 slot.
3. Call `BlobTransformer._compute_state_diff_length(blob)` — returns 0 for storage, under-counting by 1.
4. The `state_diff_commitment` computed by the block-hash CLI from the empty thin state diff differs from the commitment the sequencer's Rust code would compute from the full diff.
5. The `concatenated_counts` field encodes the wrong `state_diff_length`, so the final `block_hash` diverges from the canonical hash — confirmed by the `_log_block_hash_comparison` warning path that already exists in `transform_block`. [9](#0-8)

    def _log_block_hash_comparison(
        self,
        block_number: int,
        block_document: JsonObject,
        source_block: JsonObject,
    ) -> None:
        block_hash = block_document["block_hash"]
        mainnet_block_hash = source_block["block_hash"]
        if block_hash == mainnet_block_hash:
            self._logger.info(f"Block {block_number}: block hash MATCH with mainnet: {block_hash}")
            return

        self._logger.warning(
            f"Block {block_number}: block hash MISMATCH. "
            f"echonet={block_hash}, mainnet={mainnet_block_hash}"
        )
        self._shared.record_block_hash_mismatch(
            block_number, echonet=block_hash, mainnet=mainnet_block_hash
        )
        mismatches = []
        for key in [
            "transaction_commitment",
            "event_commitment",
            "receipt_commitment",
            "state_diff_commitment",
            "parent_block_hash",
        ]:
            echonet = block_document[key]
            mainnet_val = source_block[key]
            if echonet != mainnet_val:
                mismatches.append(f"{key}: echonet={echonet}, mainnet={mainnet_val}")
                if key == "transaction_commitment":
                    self._shared.record_transaction_commitment_mismatch(
                        block_number, echonet=echonet, mainnet=mainnet_val
                    )
        if mismatches:
            self._logger.info(f"Block {block_number}: mismatched fields: {'; '.join(mismatches)}")
        else:
            self._logger.info(
                f"Block {block_number}: block hash mismatch with no difference in compared "
                f"commitments or parent_block_hash (other header / hash inputs differ)."
            )
