### Title
`MerkleBlob::batch_insert` Bypasses Duplicate Key/Hash Validation for All Items Beyond the First Two, Corrupting the Merkle Tree Root - (File: `crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::batch_insert` silently inserts duplicate keys and duplicate leaf hashes into the DataLayer Merkle tree for all items beyond the first two in a batch, because it bypasses the duplicate-detection guards that `MerkleBlob::insert` enforces. This corrupts the tree's blob and root hash, causing `get_proof_of_inclusion` to produce proofs that are structurally inconsistent with the committed root, and causing `check_integrity` to fail. An unprivileged caller who can supply a crafted batch (e.g., via the Python binding `py_batch_insert` or a DataLayer sync delta) can trigger this path.

---

### Finding Description

`MerkleBlob::insert` (the single-item path) enforces two guards before writing anything to the blob:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 369-374
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
```

`MerkleBlob::batch_insert` uses `insert` only for the **last two** items popped from the input vector (the bootstrap phase when `leaf_count <= 1`). Every remaining item — which is the majority of any large batch — is written directly via `insert_entry_to_blob`, with no duplicate check at all:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 587-602
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    let new_block = Block {
        ...
        node: Node::Leaf(LeafNode { parent: Parent(None), hash, key, value }),
    };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;  // no key/hash guard
    indexes.push(new_leaf_index);
}
```

`insert_entry_to_blob` unconditionally calls `block_status_cache.add_leaf(index, leaf)`. When a duplicate `KeyId` is added, the cache's `key_to_index` HashMap overwrites the old entry, leaving the first leaf node as an unreachable "ghost" in the raw blob. The ghost node is still traversed during `calculate_lazy_hashes` (which walks the blob structure), so it contributes to the computed root hash. However, `get_proof_of_inclusion` follows only the cache-tracked path, producing a proof whose reconstructed root does not match the actual stored root. `check_integrity` subsequently detects the mismatch between `leaf_count` and `key_to_index_cache_length` and fails.

The Python binding `py_batch_insert` is the direct public entry point:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 1503-1518
#[pyo3(name = "batch_insert")]
pub fn py_batch_insert(
    &mut self,
    keys_values: Vec<(KeyId, ValueId)>,
    hashes: Vec<Hash>,
) -> PyResult<()> {
    if keys_values.len() != hashes.len() { ... }
    self.batch_insert(zip(keys_values, hashes).collect())?;
    Ok(())
}
```

The only pre-call check is a length mismatch guard; there is no deduplication of the input before it reaches `batch_insert`.

---

### Impact Explanation

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."**

Concretely:

1. **Root hash corruption.** After a batch containing a duplicate key, `calculate_lazy_hashes` produces a root that incorporates the ghost leaf. Any on-chain commitment derived from this root is wrong.
2. **Proof invalidity.** `get_proof_of_inclusion` for the duplicated key returns a proof whose `valid()` check fails, because the proof path does not account for the ghost leaf's contribution to the root.
3. **Integrity failure.** `check_integrity` raises `IntegrityKeyToIndexCacheLength` (leaf count in blob ≠ cache size), permanently marking the tree as corrupt.
4. **Forged exclusion.** The ghost leaf exists in the tree but is invisible to the cache, so `get_proof_of_inclusion` returns `UnknownKey` for it — a key that is actually present in the committed tree appears excluded.

---

### Likelihood Explanation

The Python DataLayer sync layer calls `batch_insert` when applying received delta updates. A malicious DataLayer peer, or any caller that can supply a `keys_values` list with a repeated `KeyId` (or repeated hash), can trigger the corruption with a single crafted batch of three or more items. No privileged role is required; the Python binding is unprivileged and accepts arbitrary caller-supplied input. The bug is deterministic and reproducible.

---

### Recommendation

Add the same duplicate guards to the bulk loop inside `batch_insert` that `insert` already enforces:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    let new_leaf_index = self.get_new_index();
    ...
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
```

Alternatively, deduplicate the input vector before processing, or refactor `batch_insert` to delegate all leaf writes through `insert` so the guards are never bypassed.

### Proof of Concept

```rust
use chia_datalayer::{Hash, InsertLocation, KeyId, MerkleBlob, ValueId};
use chia_protocol::Bytes32;

fn main() {
    let mut blob = MerkleBlob::new(vec![]).unwrap();

    // Pre-populate with 2 items so leaf_count > 1 (bypasses the bootstrap path)
    for i in 0i64..2 {
        blob.insert(
            KeyId(i), ValueId(i),
            &Hash(Bytes32::new([i as u8; 32])),
            InsertLocation::Auto {},
        ).unwrap();
    }

    // batch_insert with a duplicate key (KeyId(0) already present)
    // Items 0..N-3 skip the contains_key check entirely
    let batch = vec![
        ((KeyId(0), ValueId(99)), Hash(Bytes32::new([0xAA; 32]))), // duplicate key!
        ((KeyId(10), ValueId(10)), Hash(Bytes32::new([10u8; 32]))),
        ((KeyId(11), ValueId(11)), Hash(Bytes32::new([11u8; 32]))),
    ];
    blob.batch_insert(batch).unwrap(); // succeeds — no error raised

    blob.calculate_lazy_hashes().unwrap();

    // Proof for KeyId(0) is now structurally inconsistent with the root
    let proof = blob.get_proof_of_inclusion(KeyId(0)).unwrap();
    assert!(!proof.valid(), "proof is invalid due to ghost leaf in root hash");

    // Integrity check detects the corruption
    blob.check_integrity().expect_err("tree is corrupt");
}
```

    fn insert_entry_to_blob(&mut self, index: TreeIndex, block: &Block) -> Result<(), Error> {
        let new_block_bytes = block.to_bytes()?;
        let extend_index = self.extend_index();
        match index.cmp(&extend_index) {
            Ordering::Greater => return Err(Error::BlockIndexOutOfBounds(index)),
            Ordering::Equal => self.blob.extend_from_slice(&new_block_bytes),
            Ordering::Less => {
                self.blob[block_range(index)].copy_from_slice(&new_block_bytes);
            }
        }

        match block.node {
            Node::Leaf(leaf) => self.block_status_cache.add_leaf(index, leaf),
            Node::Internal(..) => self.block_status_cache.add_internal(index),
        }

        Ok(())
    }

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
