### Title
DataLayer `ProofOfInclusion::valid()` Does Not Verify Against a Trusted Root — Forged Inclusion Proofs Pass Validation - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

---

### Summary

`ProofOfInclusion::valid()` only checks the internal hash-chain consistency of the proof against hashes embedded within the proof itself. It never accepts a trusted external root hash as a parameter. Because `root_hash()` is derived entirely from the proof's own last layer, an attacker can craft an arbitrary `ProofOfInclusion` that passes `valid()` while proving inclusion of any chosen `node_hash` under an attacker-chosen root — not the real tree root. Any caller that uses `proof.valid()` as the sole acceptance gate, without separately comparing `proof.root_hash()` to a trusted root, accepts forged inclusion proofs.

---

### Finding Description

`ProofOfInclusion` is a Streamable struct exposed to Python via `pyclass` with `from_bytes`, `from_bytes_unchecked`, and `__new__` constructors. Its `valid()` method is:

```rust
// crates/chia-datalayer/src/merkle/proof_of_inclusion.rs
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()   // ← tautology: root_hash() IS last.combined_hash
}
```

`root_hash()` returns:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← attacker-controlled field
    } else {
        self.node_hash
    }
}
```

The final comparison `existing_hash == self.root_hash()` is a tautology: if the chain is internally consistent, `existing_hash` will equal `last.combined_hash`, which is exactly what `root_hash()` returns. The method never compares against any externally-supplied trusted root. An attacker who controls the bytes of a `ProofOfInclusion` can trivially satisfy `valid()` for any `node_hash` of their choice.

The struct is fully deserializable from untrusted bytes via the `Streamable` derive and is exposed to Python callers via `from_bytes` / `from_bytes_unchecked`. The Python binding exposes `valid()` as `py_valid()` with no trusted-root parameter.

The existing tests and fuzz targets confirm the pattern: they call only `proof.valid()` without any root-hash comparison to a trusted external value.

---

### Impact Explanation

An attacker who can deliver a crafted `ProofOfInclusion` to a DataLayer client or verifier can prove inclusion of any arbitrary `node_hash` under any attacker-chosen root. If the verifier calls only `proof.valid()` — the natural and only API provided for this purpose — the forged proof is accepted. This lets untrusted input prove invalid state, matching the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

---

### Likelihood Explanation

The method is named `valid()` and is the sole proof-verification API exposed to Python. Its signature gives no indication that a separate root-hash comparison is required. The fuzz target, the Rust unit tests, and the Python integration tests all call `proof.valid()` without a root-hash check, establishing the pattern that callers are expected to follow. Any DataLayer peer-to-peer proof exchange that relies on this API is vulnerable.

---

### Recommendation

Add a `trusted_root: Hash` parameter to `valid()` (or add a separate `valid_for_root(trusted_root: &Hash) -> bool` method) and compare the computed root against it as the final step:

```rust
pub fn valid_for_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // ← compare against externally-trusted root
}
```

Expose this as the primary Python API and deprecate or remove the root-agnostic `valid()`.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer, Side
from chia_rs import MerkleBlob

# Real tree with one leaf
blob = MerkleBlob(blob=bytearray())
real_hash = bytes([0xAA] * 32)
blob.insert(1, 1, real_hash)
blob.calculate_lazy_hashes()
real_root = blob.get_root()

# Attacker wants to forge inclusion of a fake_hash under real_root
# (or any root of their choice)
fake_hash = bytes([0xBB] * 32)
sibling   = bytes([0xCC] * 32)

# Compute a combined_hash that is internally consistent
import hashlib
# DataLayer internal hash: sha256(b"\x01" + left + right) or similar
# Attacker just needs one consistent layer
from chia_rs.datalayer import calculate_internal_hash  # if exposed, else compute manually
combined = calculate_internal_hash(fake_hash, Side.Left, sibling)

forged = ProofOfInclusion(
    node_hash=fake_hash,
    layers=[ProofOfInclusionLayer(
        other_hash_side=Side.Left,
        other_hash=sibling,
        combined_hash=combined,
    )]
)

assert forged.valid()          # ← passes: internal consistency satisfied
assert forged.root_hash() != real_root  # ← root is attacker-chosen, not the real tree root
# A caller that only checks forged.valid() accepts this as a valid proof of fake_hash
