### Title
Unprivileged Caller Can Pre-Consume `currentReceiptId` Nonces via Unguarded Legacy Path, Permanently DoS-ing `mint()` — (`evm/src/eNear/contracts/ENearProxy.sol`)

---

### Summary

`ENearProxy.finaliseNearToEthTransfer` is a public entry point guarded only by `whenNotPaused(PAUSED_LEGACY_FIN_TRANSFER)`. In the intended production deployment, `prover` is set to `FakeProver`, which unconditionally returns `true` for any proof. This lets any unprivileged caller submit an arbitrary `proofData` blob directly to `eNear.finaliseNearToEthTransfer`, causing eNear to mark any chosen receipt ID as used. Because `mint()` constructs its fake proof using the current value of `currentReceiptId` and that value is publicly readable, an attacker can pre-consume it before `mint()` runs, causing every subsequent mint to revert. `currentReceiptId` can only be incremented inside `mint()` itself, so there is no recovery path short of a contract upgrade.

---

### Finding Description

**`ENearProxy.finaliseNearToEthTransfer` — the unguarded legacy path** [1](#0-0) 

The function is `external` with a single guard: `whenNotPaused(PAUSED_LEGACY_FIN_TRANSFER)`. `PAUSED_LEGACY_FIN_TRANSFER` is **not set during `initialize()`** — `__Pausable_init()` leaves all flags cleared — and the deployment script passes no initial pause flags. [2](#0-1) 

**`FakeProver` accepts every proof unconditionally** [3](#0-2) 

The README explicitly states FakeProver is the intended production prover for ENearProxy: [4](#0-3) 

**`mint()` embeds `currentReceiptId` at a fixed, known offset** [5](#0-4) 

The receipt ID is placed at byte offset 76 (`72 + 4` header bytes) as a 32-byte big-endian `uint256`. This layout is fully public and deterministic. An attacker replicates the same structure with the target receipt ID, submits it through the legacy path, and eNear marks that ID as used before `mint()` can.

**No mechanism to skip a poisoned `currentReceiptId`**

`currentReceiptId` is incremented only inside `mint()` (line 71). If `mint()` reverts because the ID is already used, the counter never advances. There is no admin function to skip or reset it. [6](#0-5) 

---

### Impact Explanation

Every call to `mint()` by the MINTER_ROLE (i.e., OmniBridge finalizing a NEAR→EVM transfer of eNear) can be permanently blocked. The attacker reads `currentReceiptId`, submits a crafted legacy proof consuming that ID, and repeats for every subsequent value. Users who have locked NEAR on the NEAR side cannot receive eNear on Ethereum; their funds are irrecoverably stuck until an admin pauses the legacy path **and** upgrades the contract to repair `currentReceiptId`. This matches the "Permanent freezing / irrecoverable lock of user funds" critical impact criterion.

---

### Likelihood Explanation

- `PAUSED_LEGACY_FIN_TRANSFER` is unset by default and requires a separate admin transaction after deployment.
- FakeProver is the documented production prover — no valid proof is required.
- `currentReceiptId` is a public storage variable; no off-chain information is needed.
- The proof layout is fully deterministic and visible in the contract source.
- The attack requires a single cheap transaction per nonce and can be automated to front-run every mint.

---

### Recommendation

1. **Set `PAUSED_LEGACY_FIN_TRANSFER` inside `initialize()`** so the legacy path is disabled from the moment of deployment, eliminating the attack surface entirely.
2. Alternatively, remove `ENearProxy.finaliseNearToEthTransfer` entirely — the README states the legacy path should be paused permanently once ENearProxy takes over.
3. If the legacy path must remain, replace `FakeProver` with a real proof verifier so arbitrary proofs cannot be submitted.

---

### Proof of Concept

```
1. Deploy ENearProxy with FakeProver; currentReceiptId = 0.
2. Attacker reads currentReceiptId → 0.
3. Attacker constructs:
     craftedProof = bytes(72 zeros)
                  + 0x01000000
                  + abi.encodePacked(uint256(0))   // receipt ID = 0
                  + bytes(24 zeros)
                  + <nearConnector length + data>
                  + 0x022500000000
                  + <any 16-byte amount>
                  + <any 20-byte address>
                  + bytes(280 zeros)
4. Attacker calls ENearProxy.finaliseNearToEthTransfer(craftedProof, 0).
   → FakeProver returns true.
   → eNear.finaliseNearToEthTransfer marks receipt 0 as used.
5. MINTER_ROLE calls ENearProxy.mint(eNear, recipient, amount).
   → mint() builds fakeProofData with currentReceiptId = 0.
   → eNear.finaliseNearToEthTransfer reverts: "proof already used".
   → currentReceiptId is never incremented.
6. Attacker repeats for receipt ID 1, 2, … blocking all future mints.
```
