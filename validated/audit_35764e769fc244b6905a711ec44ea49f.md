### Title
Unbounded `req.Predecessors` Loop in `TraceTx` gRPC Handler Enables Resource Exhaustion Leading to Node Crash - (File: `x/evm/keeper/grpc_query.go`)

### Summary
The `TraceTx` gRPC handler iterates over a caller-supplied `req.Predecessors` slice with no length bound. Each predecessor is replayed via a full `ApplyMessageWithConfig` call with `commit=true`, meaning real EVM execution (including storage writes to a cache context) is performed for every entry. An unprivileged caller can submit a single `TraceTx` (or the corresponding `debug_traceTransaction` JSON-RPC call) carrying thousands of complex predecessor transactions, exhausting CPU and memory on the serving node.

### Finding Description
In `x/evm/keeper/grpc_query.go`, the `TraceTx` handler contains:

```go
for i, tx := range req.Predecessors {
    ethTx := tx.AsTransaction()
    msg, err := core.TransactionToMessage(ethTx, signer, cfg.BaseFee)
    ...
    rsp, err := k.ApplyMessageWithConfig(ctx, msg, cfg, true)   // commit=true
    ...
    cfg.TxConfig.LogIndex += uint(len(rsp.Logs))
}
``` [1](#0-0) 

There is no cap on `len(req.Predecessors)`. Each iteration invokes `ApplyMessageWithConfig` with `commit=true`, which runs the full EVM state machine, writes dirty state to the branched cache context, and accumulates logs. The cost per iteration is proportional to the gas limit of the predecessor transaction; a predecessor with `gasLimit = blockGasLimit` can consume the maximum EVM work per call.

By contrast, the `TraceBlock` handler iterates over `req.Txs` with the same absence of a length bound:

```go
for i, tx := range req.Txs {
    ...
    traceResult, logIndex, err := k.prepareTrace(ctx, cfg, msg, req.TraceConfig, true)
    ...
}
``` [2](#0-1) 

Both endpoints are reachable over the public gRPC port (and via the JSON-RPC `debug_*` namespace) without any authentication.

### Impact Explanation
The gRPC server and the CometBFT consensus engine share the same OS process on a Cosmos SDK node. A single malicious `TraceTx` request carrying N predecessors, each with maximum gas, forces N sequential full EVM executions. Memory grows proportionally (cache context, logs, state journal). If the node is also a validator, CPU starvation or OOM causes it to miss `propose`/`prevote`/`precommit` rounds, triggering a consensus timeout and, if enough validators are targeted simultaneously, a chain halt. Even on a dedicated RPC node, the node becomes unresponsive, severing the JSON-RPC surface used by wallets and relayers.

This matches the allowed Critical impact: *"Valid unprivileged … RPC submission … can halt the chain … or cause deterministic validator consensus failure."*

### Likelihood Explanation
- The `debug_traceTransaction` JSON-RPC method (which routes to `TraceTx`) is commonly enabled on public RPC endpoints and on validator nodes that double as RPC providers.
- No authentication or rate-limiting is enforced at the Ethermint layer.
- Constructing a request with hundreds of high-gas predecessors requires only a standard Ethereum JSON-RPC client.
- The attack is repeatable and can be sustained with minimal cost to the attacker (no on-chain fees for a query).

### Recommendation
Add an explicit cap on `req.Predecessors` length before the loop, analogous to the `MaxSimulateBlocks = 256` guard used in `eth_simulateV1`:

```go
const maxTracePredecessors = 128

if len(req.Predecessors) > maxTracePredecessors {
    return nil, status.Errorf(codes.InvalidArgument,
        "too many predecessors: %d > %d", len(req.Predecessors), maxTracePredecessors)
}
```

Apply the same pattern to `TraceBlock`'s `req.Txs` slice. Additionally, consider enforcing a cumulative gas budget across all predecessor executions so that even within the cap, a single request cannot consume unbounded EVM work. [3](#0-2) 

### Proof of Concept
1. Obtain the gRPC/JSON-RPC address of a target node.
2. Construct a `QueryTraceTxRequest` where `Predecessors` contains 10,000 entries, each being a valid `MsgEthereumTx` with `gasLimit = blockGasLimit` and calldata that exercises storage-heavy EVM opcodes (e.g., repeated `SSTORE`).
3. Submit via `grpcurl` or `eth_sendRawTransaction`-style debug call.
4. Observe: the node's CPU pegs at 100%, memory climbs until OOM or the process is killed by the OS, and the node stops producing/signing blocks.

The root cause is the absence of any bound check before the loop at `x/evm/keeper/grpc_query.go:542`, directly analogous to the unbounded `for` loop over `contracts` in the referenced `BatchRequests.sol` report. [4](#0-3)

func (k Keeper) TraceTx(c context.Context, req *types.QueryTraceTxRequest) (*types.QueryTraceTxResponse, error) {
	var baseFee *big.Int
	if req == nil || req.Msg == nil {
		return nil, status.Error(codes.InvalidArgument, "request and message cannot be empty")
	}
	if req.BaseFee != nil && !req.BaseFee.IsNil() {
		baseFee = req.BaseFee.BigInt()
	}
	resultData, err := execTrace(
		c,
		req,
		k,
		baseFee,
		func(ctx sdk.Context, cfg *EVMConfig, traceConfig *types.TraceConfig) (*core.Message, error) {
			signer := ethtypes.MakeSigner(cfg.ChainConfig, big.NewInt(ctx.BlockHeight()), uint64(ctx.BlockTime().Unix())) //#nosec G115
			tracer, err := newTacer(&logger.Config{}, cfg.ChainConfig, cfg.TxConfig, traceConfig)
			if err != nil {
				return nil, status.Error(codes.Internal, err.Error())
			}
			cfg.Tracer = tracer.Hooks
			cfg.DebugTrace = true
			cfg.TraceReplay = traceConfig.GetTraceReplay()
			for i, tx := range req.Predecessors {
				ethTx := tx.AsTransaction()
				msg, err := core.TransactionToMessage(ethTx, signer, cfg.BaseFee)
				if err != nil {
					k.Logger(ctx).Debug("trace: skipping predecessor, failed to convert tx to message",
						"index", i, "hash", ethTx.Hash().Hex(), "err", err.Error())
					continue
				}
				cfg.TxConfig.TxHash = ethTx.Hash()
				cfg.TxConfig.TxIndex, err = ethermint.SafeUint(i)
				if err != nil {
					k.Logger(ctx).Debug("trace: skipping predecessor, invalid tx index",
						"index", i, "hash", ethTx.Hash().Hex(), "err", err.Error())
					continue
				}
				rsp, err := k.ApplyMessageWithConfig(ctx, msg, cfg, true)
				if err != nil {
					k.Logger(ctx).Error("trace: predecessor replay failed, trace state may be incomplete",
						"index", i, "hash", ethTx.Hash().Hex(), "err", err.Error())
					continue
				}
				cfg.TxConfig.LogIndex += uint(len(rsp.Logs))
			}

			tx := req.Msg.AsTransaction()
			cfg.TxConfig.TxHash = tx.Hash()
			if len(req.Predecessors) > 0 {
				cfg.TxConfig.TxIndex++
			}

			return core.TransactionToMessage(tx, signer, cfg.BaseFee)
		},
	)
	if err != nil {
		return nil, err
	}

	return &types.QueryTraceTxResponse{
		Data: resultData,
	}, nil
}
