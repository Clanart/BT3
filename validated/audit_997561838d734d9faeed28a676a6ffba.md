### Title
Stale Parameter Snapshot in Cosmos Ante Handler Bypasses Governance-Updated Fee Market Rules — (`File: evmd/ante/handler_options.go`)

---

### Summary

`newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712` snapshot `evmParams` and `feemarketParams` once at construction time and pass pointers to those local copies into `NewDynamicFeeChecker` and `NewMinGasPriceDecorator`. Because the ante handler is built once at app startup, any subsequent governance update to `feemarket` or `evm` parameters is never reflected in the running fee-check closures. After governance enables the EIP-1559 base fee or raises `MinGasPrice`, any user can submit Cosmos-SDK transactions that bypass the new fee requirements, because the ante handler still enforces the old (stale) values.

---

### Finding Description

**Root cause — `evmd/ante/handler_options.go`, `newCosmosAnteHandler` (lines 178–211):**

```go
func newCosmosAnteHandler(ctx sdk.Context, options HandlerOptions, ...) sdk.AnteHandler {
    evmParams        := options.EvmKeeper.GetParams(ctx)          // one-time snapshot
    feemarketParams  := options.FeeMarketKeeper.GetParams(ctx)    // one-time snapshot
    ...
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
    ...
    cosmos.NewMinGasPriceDecorator(options.FeeMarketKeeper, evmDenom, &feemarketParams),
```

`evmParams` and `feemarketParams` are **local variables** whose addresses are captured by the returned `sdk.AnteHandler` closure. They are never refreshed from the store.

The same pattern appears in `newLegacyCosmosAnteHandlerEip712` (lines 29–37 of `evmd/ante/evm_handler.go`).

**How the stale snapshot is consumed:**

`NewDynamicFeeChecker` calls `types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)` on every transaction:

```go
// x/evm/types/utils.go:244
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    if !IsLondon(ethCfg, height) { return nil }
    baseFee := feemarketParams.GetBaseFee()   // reads stale snapshot
    if baseFee == nil { return new(big.Int) }
    return baseFee
}
```

`MinGasPriceDecorator.AnteHandle` reads `mpd.feemarketParams.MinGasPrice` directly (line 54 of `ante/cosmos/min_gas_price.go`), also from the stale snapshot.

**Contrast with the EVM ante handler:** `newEthAnteHandler` (lines 86–176 of `handler_options.go`) calls `options.EvmKeeper.EVMBlockConfig(ctx, ...)` on **every** invocation, fetching live params. The Cosmos ante handler has no equivalent live-fetch path.

---

### Impact Explanation

**Scenario A — Governance enables the EIP-1559 base fee after node startup:**

1. At startup, `NoBaseFee = true` → snapshot captures `NoBaseFee = true`.
2. Governance passes `MsgUpdateParams` setting `NoBaseFee = false` and `BaseFee = 1_000_000_000`.
3. The live feemarket store now enforces a non-zero base fee for EVM txs (via `newEthAnteHandler`).
4. The Cosmos ante handler still reads the stale snapshot where `feemarketParams.GetBaseFee()` returns `nil` (or 0), so `GetBaseFee` returns `new(big.Int)` (zero).
5. `feeCap.LT(baseFeeInt)` is `feeCap < 0`, which is always false — **any Cosmos tx passes fee validation regardless of the actual base fee**.

Result: Cosmos-SDK transactions (including `authz`, `gov`, IBC, etc.) bypass the EIP-1559 base fee entirely, paying zero fees while EVM txs are correctly charged. This is a mis-accounting of user fees and an invalid-transaction admission bug.

**Scenario B — Governance raises `MinGasPrice`:**

`MinGasPriceDecorator` reads `mpd.feemarketParams.MinGasPrice` from the stale snapshot. After governance raises `MinGasPrice`, Cosmos txs with fees between the old and new minimum are accepted when they should be rejected.

**Scenario C — Governance changes `EvmDenom`:**

`denom := evmParams.EvmDenom` (line 54 of `fee_checker.go`) is read from the stale `evmParams`. Fee coins are checked against the wrong denomination, causing fee accounting to break entirely.

---

### Likelihood Explanation

- Governance parameter updates to `feemarket` and `evm` modules are a normal, documented operational procedure (e.g., enabling EIP-1559 at a future height, adjusting `MinGasPrice`).
- After any such update, **every** Cosmos-SDK transaction submitted by any user exploits the stale snapshot automatically — no special knowledge or tooling is required.
- The EVM ante handler already demonstrates the correct pattern (`EVMBlockConfig` per call), confirming this is an oversight rather than an intentional design.

---

### Recommendation

Replace the one-time snapshot with a live fetch inside the returned closure, mirroring the EVM ante handler pattern:

```go
func newCosmosAnteHandler(options HandlerOptions, ...) sdk.AnteHandler {
    return func(ctx sdk.Context, tx sdk.Tx, simulate bool) (sdk.Context, error) {
        evmParams       := options.EvmKeeper.GetParams(ctx)
        feemarketParams := options.FeeMarketKeeper.GetParams(ctx)
        ...
    }
}
```

Apply the same fix to `newLegacyCosmosAnteHandlerEip712`.

---

### Proof of Concept

1. Deploy Ethermint with `NoBaseFee = true` (default). Node starts; `newCosmosAnteHandler` snapshots `feemarketParams.NoBaseFee = true`.
2. Submit and pass a governance proposal: `MsgUpdateParams` with `NoBaseFee = false`, `BaseFee = 1_000_000_000`.
3. Confirm via `eth_gasPrice` that EVM txs now require base fee ≥ 1 Gwei.
4. Submit a Cosmos-SDK tx (e.g., `MsgSend`) with `fee = 0uatom`. Observe it passes `CheckTx` and `DeliverTx` successfully — the ante handler uses the stale `feemarketParams` where `GetBaseFee()` returns 0, so `feeCap.LT(0)` is never true.
5. Confirm that an EVM tx with `gasPrice = 0` is correctly rejected by `newEthAnteHandler` (which fetches live params), demonstrating the asymmetry.
