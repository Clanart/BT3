### Title
Missing `parent_hash` Continuity Check in P2P Header Sync Allows Malicious Peer to Store Headers with Arbitrary `parent_hash` — (`crates/apollo_p2p_sync/src/client/header.rs`)

---

### Summary

`HeaderStreamBuilder::parse_data_for_block` accepts a `SignedBlockHeader` from a p2p peer and stores it verbatim without verifying that the header's `parent_hash` equals the previously stored block's `block_hash`. The check is explicitly acknowledged as missing via a TODO comment. No cryptographic signature verification over the block hash is performed anywhere in the p2p sync client path. A single malicious peer can therefore store a header for block N with an arbitrary `parent_hash`, corrupting the stored block hash chain permanently.

---

### Finding Description

`parse_data_for_block` performs exactly two checks on the peer-supplied `SignedBlockHeader`:

1. `block_number` matches the expected value [1](#0-0) 
2. `signatures.len() == ALLOWED_SIGNATURES_LENGTH` (length only, no cryptographic verification) [2](#0-1) 

The `_storage_reader` parameter is intentionally unused (underscore prefix), confirming no lookup of the previous block's hash is performed. The TODO comment at line 102 explicitly acknowledges the missing guard: [3](#0-2) 

After passing these two checks, the `SignedBlockHeader` is returned as `Ok(Some(...))` and immediately written to storage via `write_to_storage`, which calls `append_header` with no further validation: [4](#0-3) 

`append_header` in storage blindly copies `block_header.block_header_without_hash.parent_hash` into `StorageBlockHeader.parent_hash` and commits it: [5](#0-4) 

There is no call to `verify_block_signature` anywhere in `crates/apollo_p2p_sync/` (confirmed by search). The `verify_block_signature` function exists in `starknet_api` and signs `poseidon_hash(block_hash, state_diff_commitment)`, but it is never invoked in this path: [6](#0-5) 

Additionally, the `block_hash` field itself is also peer-supplied and never recomputed or verified against the header fields, so the attacker controls both `block_hash` and `parent_hash` of the stored header.

For contrast, the central sync path does perform this check via `verify_parent_block_hash`, which reads the previous block's hash from storage and rejects mismatches: [7](#0-6) 

The p2p sync path has no equivalent guard.

---

### Impact Explanation

A malicious p2p peer (unprivileged, no operator access required) can store a `BlockHeader` for block N with:
- Correct `block_number = N` (to pass the only structural check)
- Arbitrary `parent_hash` (e.g., `0x0` or any chosen value)
- Arbitrary `block_hash` (unverified)
- Exactly one signature entry (to pass the length check)

The result is a permanently stored header with a broken `parent_hash` chain. This corrupted value is then served authoritatively by the RPC layer (e.g., `starknet_getBlockWithTxHashes` returns `parent_hash` directly from storage). Any downstream consumer — proof manager, SNOS input construction, or another syncing node — that reads the stored `parent_hash` or `block_hash` receives the attacker-controlled value.

Impact: **High** — RPC returns an authoritative-looking wrong `parent_hash`/`block_hash` value for the affected block. The stored block hash chain is broken from block N onward.

---

### Likelihood Explanation

P2P sync is an opt-in mode but is a production feature. Any peer that can establish a p2p sync session (no authentication required beyond network connectivity) can trigger this. The attack requires sending a single well-formed `SignedBlockHeader` message with the correct `block_number` and a single signature entry. No cryptographic material is needed.

---

### Recommendation

In `parse_data_for_block`, use the already-available `storage_reader` (currently ignored with `_` prefix) to read the previous block's header and assert that `signed_block_header.block_header.block_header_without_hash.parent_hash` equals the stored `block_hash` of block `block_number - 1`. A mismatch should return `ParseDataError::BadPeer`. This mirrors the existing guard in `verify_parent_block_hash` in central sync. [8](#0-7) 

Additionally, implement cryptographic signature verification (calling `verify_block_signature` with the known sequencer public key) before accepting any peer-supplied header.

---

### Proof of Concept

```
1. Enable p2p sync on a node that has synced blocks 0..N-1.
2. Act as a p2p peer responding to the node's HeaderQuery for block N.
3. Send a SignedBlockHeader with:
     block_number = N
     parent_hash  = 0xdeadbeef  (not equal to stored block N-1's block_hash)
     block_hash   = 0xcafebabe  (arbitrary)
     signatures   = [any single BlockSignature]
4. parse_data_for_block passes: block_number matches, signatures.len() == 1.
5. write_to_storage calls append_header, which stores parent_hash=0xdeadbeef.
6. Query the node's RPC: starknet_getBlockWithTxHashes(block_number=N)
   → returns "parent_hash": "0xdeadbeef"  (wrong, attacker-controlled)
```

The stored `parent_hash` for block N is now `0xdeadbeef` instead of the actual hash of block N-1, breaking the block hash chain. [9](#0-8)

        signed_headers_response_manager: &'a mut ClientResponsesManager<
            DataOrFin<SignedBlockHeader>,
        >,
        block_number: BlockNumber,
        _storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            // TODO(noamsp): investigate and remove this timeout.
            let maybe_signed_header =
                timeout(Duration::from_secs(15), signed_headers_response_manager.next())
                    .await
                    .ok()
                    .flatten()
                    .ok_or(ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }))?;
            let Some(signed_block_header) = maybe_signed_header?.0 else {
                return Ok(None);
            };
            // TODO(shahak): Check that parent_hash is the same as the previous block's hash
            // and handle reverts.
            if block_number
                != signed_block_header.block_header.block_header_without_hash.block_number
            {
                return Err(ParseDataError::BadPeer(BadPeerError::HeadersUnordered {
                    expected_block_number: block_number,
                    actual_block_number: signed_block_header
                        .block_header
                        .block_header_without_hash
                        .block_number,
                }));
            }
            if signed_block_header.signatures.len() != ALLOWED_SIGNATURES_LENGTH {
                return Err(ParseDataError::BadPeer(BadPeerError::WrongSignaturesLength {
                    signatures: signed_block_header.signatures,
                }));
            }
            Ok(Some(signed_block_header))
        }
        .boxed()
    }

    fn verify_parent_block_hash(
        &self,
        block_number: BlockNumber,
        block: &Block,
    ) -> StateSyncResult {
        let prev_block_number = match block_number.prev() {
            None => return Ok(()),
            Some(bn) => bn,
        };
        let prev_hash = self
            .reader
            .begin_ro_txn()?
            .get_block_header(prev_block_number)?
            .ok_or(StorageError::DBInconsistency {
                msg: format!(
                    "Missing block {prev_block_number} in the storage (for verifying block \
                     {block_number}).",
                ),
            })?
            .block_hash;

        if prev_hash != block.header.block_header_without_hash.parent_hash {
            // Block parent hash mismatch, log and restart sync loop.
            warn!(
                "Detected reorg in central. block number {block_number} had hash {prev_hash} but \
                 the next block's parent hash is {}.",
                block.header.block_header_without_hash.parent_hash
            );
            CENTRAL_SYNC_FORKS_FROM_FEEDER.increment(1);
            return Err(StateSyncError::ParentBlockHashMismatch {
                block_number,
                expected_parent_block_hash: block.header.block_header_without_hash.parent_hash,
                stored_parent_block_hash: prev_hash,
            });
        }

        Ok(())
