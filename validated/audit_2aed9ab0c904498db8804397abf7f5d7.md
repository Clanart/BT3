### Title
Missing Class Hash Existence Validation in `execute_replace_class` Enables Permanent Fund Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS unconditionally accepts any arbitrary felt value as a new class hash without verifying that the hash corresponds to a previously declared class. This is directly analogous to the external report's root cause of "trusting a return value without independent verification." A contract can replace its own class hash with an undeclared value, rendering itself permanently non-callable and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall: [1](#0-0) 

The critical section is:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment at line 898 explicitly acknowledges the missing check. The OS writes `request.class_hash` — an attacker-controlled value — directly into `contract_state_changes` with no validation that this hash exists in `contract_class_changes`.

When a subsequent call is made to the contract after the replacement, `execute_entry_point` performs: [2](#0-1) 

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=cast(compiled_class_facts_bundle.compiled_class_facts, felt*),
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
```

If `execution_context.class_hash` is an undeclared value, `dict_read` on `contract_class_changes` returns 0 (the default for an uninitialized dict entry). `find_element` then searches for a compiled class with hash 0. If no such class exists, the proof fails. The sequencer, detecting this during simulation, will refuse to include any subsequent call to the contract. The contract becomes permanently non-callable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any funds stored in the contract's storage (e.g., ERC-20 token balances, ETH, protocol state) become permanently inaccessible. No withdrawal, transfer, or recovery function can be called because every entry point dispatch goes through `execute_entry_point`, which will fail to resolve the invalid class hash. The state change from `replace_class` is committed on-chain and irreversible.

---

### Likelihood Explanation

The attack is reachable by any unprivileged transaction sender:

1. A user deploys a contract (or interacts with an existing contract that exposes `replace_class` logic).
2. Funds accumulate in the contract (either deposited by the attacker to lure victims, or already present in a legitimate protocol contract that has a logic flaw allowing an external caller to trigger `replace_class`).
3. The attacker sends a transaction that causes the contract to invoke the `replace_class` syscall with an arbitrary undeclared felt value as the class hash.
4. The OS accepts this transaction — `execute_replace_class` does not revert; it successfully updates `contract_state_changes`.
5. The contract is now permanently non-callable. All funds are frozen.

The TODO comment at line 898 confirms the development team is aware this check is missing and has not yet implemented it.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, validate that the hash exists in `contract_class_changes` (i.e., that it was previously declared in the current or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` with `key=class_hash` and assert the result is non-zero. If the class hash is undeclared, write a failure response via `write_failure_response` instead of updating state. This mirrors the external report's recommendation to "not trust the return value" and instead independently verify the value against a known-good source.

---

### Proof of Concept

1. Deploy `VictimContract` with storage holding 1000 ETH worth of token balances.
2. `VictimContract` exposes a public function `self_destruct()` that calls `replace_class(0xdeadbeef)` — an undeclared class hash.
3. Attacker calls `self_destruct()`. The OS executes `execute_replace_class`, writes `class_hash=0xdeadbeef` into `contract_state_changes`, and the transaction is included in a block.
4. Any subsequent transaction attempting to call any function on `VictimContract` causes `execute_entry_point` to call `dict_read{dict_ptr=contract_class_changes}(key=0xdeadbeef)` → returns 0 → `find_element(..., key=0)` fails → block is unprovable → sequencer rejects all calls to `VictimContract`.
5. The 1000 ETH equivalent is permanently frozen. [3](#0-2) [2](#0-1)

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
