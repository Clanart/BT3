### Title
Missing Declared Class Existence Check in `execute_replace_class` Allows Permanent Contract Bricking — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS does not verify that the new class hash supplied by a contract corresponds to a class that has actually been declared on-chain. The OS silently accepts the syscall and commits the invalid class hash into the contract's state entry. Any subsequent call to that contract causes an unrecoverable OS-level failure (`find_element` assertion trap), permanently freezing any funds the contract holds.

---

### Finding Description

`execute_replace_class` processes the `replace_class` syscall. After deducting gas it reads `request.class_hash` and immediately writes it into the contract's `StateEntry` via `dict_update` on `contract_state_changes`:

```cairo
// syscall_impls.cairo  lines 896-910
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The in-code TODO at line 898 explicitly acknowledges the missing check. No assertion against `contract_class_changes` (the class-hash → compiled-class-hash mapping) is performed. [2](#0-1) 

When a future transaction calls the bricked contract, `execute_entry_point` runs:

```cairo
// execute_entry_point.cairo  lines 154-166
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash          // ← the undeclared hash
);
// dict_read returns 0 for an absent key
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,                  // ← 0, not in the facts array
);
``` [3](#0-2) 

`dict_read` returns `0` for an absent key. `find_element` with key `0` will fail with a Cairo assertion trap (not a graceful revert) because `0` is not present in the compiled class facts bundle. This is an OS-level proof failure, not a contract-level revert.

The analogy to the Timelock bug is exact: just as `target.call(callData)` silently succeeds when `target` has no code, `execute_replace_class` silently succeeds when the new class hash has no corresponding compiled class, leaving the caller to assume the operation was valid when in fact the contract is now permanently inaccessible.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `StateEntry.class_hash` is set to an undeclared hash and the block is committed, the state root encodes that invalid hash via `get_contract_state_hash`: [4](#0-3) 

There is no on-chain mechanism to undo a committed state root. Every future call to the contract causes `find_element` to trap, making it impossible to generate a valid proof for any block that touches the contract. The contract — and all funds it holds — are permanently frozen.

---

### Likelihood Explanation

- The `replace_class` syscall is callable by any Cairo 1 contract without any privileged role.
- The attacker-controlled input is `request.class_hash`, a raw `felt` value read directly from the syscall segment with no range or existence constraint.
- A buggy upgrade path (e.g., a contract that computes its own next class hash incorrectly) or a deliberate attacker can supply an undeclared hash.
- The TODO comment confirms the development team is aware the check is absent, meaning the gap is not protected by any compensating control in the OS today.

---

### Recommendation

In `execute_replace_class`, before writing the new `StateEntry`, assert that `request.class_hash` is present in `contract_class_changes`:

```cairo
// After reading class_hash from request:
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
// compiled_class_hash == 0 means the class is not declared
if (compiled_class_hash == 0) {
    write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
    return ();
}
```

This mirrors the pattern already used in `execute_entry_point` and ensures that `replace_class` can only target classes that are verifiably declared in the current block's class tree. [5](#0-4) 

---

### Proof of Concept

1. **Attacker deploys** `VictimContract` holding user funds. `VictimContract` exposes an `upgrade(class_hash: felt)` entry point that calls `replace_class(class_hash)`.

2. **Attacker calls** `upgrade(0xdeadbeef)` where `0xdeadbeef` is not a declared class hash.

3. **OS processes `execute_replace_class`:**
   - Gas is deducted successfully.
   - `class_hash = 0xdeadbeef` is read from the request.
   - **No existence check is performed** (line 898 TODO).
   - `dict_update` writes `StateEntry{class_hash=0xdeadbeef, ...}` into `contract_state_changes`.
   - The syscall returns success; the transaction is included in the block.

4. **State is committed.** `compute_contract_state_commitment` hashes the new `StateEntry` (including `class_hash=0xdeadbeef`) into the Patricia tree root. The invalid hash is now part of the canonical chain state. [6](#0-5) 

5. **Any future call to `VictimContract`** reaches `execute_entry_point`:
   - `dict_read(contract_class_changes, 0xdeadbeef)` → returns `0` (not declared).
   - `find_element(..., key=0)` → Cairo assertion trap; OS cannot produce a valid proof.

6. **Result:** `VictimContract` is permanently inaccessible. All funds it holds are frozen with no recovery path.

func get_contract_state_hash{hash_ptr: HashBuiltin*}(
    class_hash: felt, storage_root: felt, nonce: felt
) -> (hash: felt) {
    const CONTRACT_STATE_HASH_VERSION = 0;
    if (class_hash == UNINITIALIZED_CLASS_HASH) {
        if (storage_root == 0) {
            if (nonce == 0) {
                return (hash=0);
            }
        }
    }

    // Set res = H(H(class_hash, storage_root), nonce).
    let (hash_value) = hash2(class_hash, storage_root);
    let (hash_value) = hash2(hash_value, nonce);

    // Return H(hash_value, CONTRACT_STATE_HASH_VERSION). CONTRACT_STATE_HASH_VERSION must be in the
    // outermost hash to guarantee unique "decoding".
    let (hash) = hash2(hash_value, CONTRACT_STATE_HASH_VERSION);
    return (hash=hash);
