### Title
Unvalidated v2 `ProofOfSpace` Proof Bytes Cause Panic in `update_digest` on Any Block Hash Call — (`File: crates/chia-protocol/src/proof_of_space.rs`)

---

### Summary

`ProofOfSpace::parse()` for v2 proofs accepts arbitrary proof bytes without validating them against the PoS2 proof structure. However, `ProofOfSpace::update_digest()` for v2 proofs calls `self.quality_string().expect(...)`, which panics if the proof bytes are structurally invalid. Because `update_digest` is invoked whenever any containing block type (`RewardChainBlock`, `UnfinishedBlock`, `FullBlock`) computes its hash, any unprivileged peer can send a network block carrying a v2 `ProofOfSpace` with malformed proof bytes, causing a deterministic panic in the Rust layer on every node that processes it.

---

### Finding Description

`ProofOfSpace` was extended in Chia 3.0 to support v2 proofs. The version is encoded in the high bits of the `pool_contract_puzzle_hash` Option prefix byte. For v2 proofs (`version == 1`), `parse()` reads `plot_index`, `meta_group`, `strength`, and `proof` bytes, and only checks that exactly one of `pool_public_key` / `pool_contract_puzzle_hash` is set:

```rust
} else if version == 1 {
    let plot_index = <u16 as Streamable>::parse::<TRUSTED>(input)?;
    let meta_group = <u8 as Streamable>::parse::<TRUSTED>(input)?;
    let strength = <u8 as Streamable>::parse::<TRUSTED>(input)?;
    let proof = <Bytes as Streamable>::parse::<TRUSTED>(input)?;

    if pool_public_key.is_some() == pool_contract_puzzle_hash.is_some() {
        return Err(Error::InvalidPoS);
    }
    Ok(ProofOfSpace { ..., proof, ... })
}
```

The `proof` bytes themselves are accepted without any structural validation. [1](#0-0) 

However, `update_digest()` for v2 proofs does not hash the raw proof bytes. Instead it calls `quality_string()`, which invokes `chia_pos2::quality_string_from_proof()` and returns `None` if the proof is structurally invalid. The result is immediately unwrapped with `.expect()`:

```rust
self.quality_string()
    .expect("internal error. Can't compute hash of invalid ProofOfSpace")
    .update_digest(digest);
``` [2](#0-1) 

`quality_string()` itself:

```rust
pub fn quality_string(&self) -> Option<Bytes32> {
    if self.version != 1 { return None; }
    let k_size = (self.proof.len() * 8 / 128) as u8;
    let plot_id = self.compute_plot_id().to_bytes();
    chia_pos2::quality_string_from_proof(&plot_id, k_size, self.strength, self.proof.as_slice())
        .map(|quality| { ... })
}
``` [3](#0-2) 

`chia_pos2::quality_string_from_proof` returns `None` for malformed proof bytes (wrong length, invalid structure). The `.expect()` then panics.

`update_digest` on `ProofOfSpace` is called from `RewardChainBlock::update_digest`:

```rust
self.proof_of_space.update_digest(digest);
``` [4](#0-3) 

`RewardChainBlock::update_digest` is called from `FullBlock::update_digest` and `UnfinishedBlock::update_digest`, and also from `UnfinishedBlock::partial_hash()`:

```rust
pub fn partial_hash(&self) -> Bytes32 {
    self.reward_chain_block.hash().into()
}
``` [5](#0-4) 

All of `FullBlock`, `UnfinishedBlock`, `RewardChainBlock`, and `ProofOfSpace` expose `get_hash()` to Python via pyo3. [6](#0-5) 

---

### Impact Explanation

Any node that receives a network block (via `RespondBlock`, `RespondUnfinishedBlock`, or weight-proof messages) containing a v2 `ProofOfSpace` with invalid proof bytes and then calls `get_hash()`, `partial_hash()`, or any hashing operation on the block will trigger a Rust panic. In pyo3, this surfaces as a `PanicException` in Python. If the Python full node does not catch this exception at the right boundary, it will crash the block-processing coroutine or the entire node process, causing a chain halt. Because the panic is deterministic (same bytes → same panic on every node), all nodes processing the same malformed block will crash identically, enabling a network-wide halt.

This matches the Critical allowed impact: **a valid unprivileged serialized network object can trigger deterministic consensus divergence or chain halt**.

---

### Likelihood Explanation

The attack requires only sending a single malformed network message. No keys, governance access, or privileged roles are needed. Any peer connection is sufficient. The malformed block only needs to pass `parse()` (which it will, since proof bytes are not validated there), and then any subsequent hash computation triggers the panic. The attack is trivially constructable: take a valid v2 `ProofOfSpace` wire encoding and corrupt the proof bytes while keeping the structural prefix bytes intact.

---

### Recommendation

1. **Validate proof bytes in `parse()`**: For v2 proofs, call `quality_string_from_proof` (or a lightweight length/format check) during `parse()` and return `Err(Error::InvalidPoS)` if the proof is structurally invalid. This ensures that any `ProofOfSpace` that successfully parses is also safe to hash.

2. **Remove the `.expect()` in `update_digest()`**: Replace the panic with a graceful fallback. Since `update_digest` cannot return an error today, the cleanest fix is option 1 (reject at parse time). Alternatively, hash a sentinel value on `None` and document that the hash is only meaningful for valid proofs, but this changes the hash function semantics.

3. **Add a fuzz target** that feeds arbitrary bytes into `ProofOfSpace::parse()` followed by `update_digest()` to catch this class of bug.

### Proof of Concept

```rust
// Craft a v2 ProofOfSpace with garbage proof bytes that passes parse()
// prefix byte = 0b10 (version=1, no pool_contract), pool_public_key = Some(...)
// plot_index, meta_group, strength = arbitrary
// proof = garbage bytes of length 128 (so k_size = 8, but content is invalid)

let mut buf = Vec::new();
// challenge (32 bytes)
buf.extend_from_slice(&[0u8; 32]);
// pool_public_key Option prefix = 1 (Some), followed by 48-byte G1 point
buf.push(1u8);
buf.extend_from_slice(&valid_g1_bytes); // 48 bytes
// pool_contract_puzzle_hash prefix = 0b10 (version=1, not set)
buf.push(0b10u8);
// plot_public_key (48 bytes)
buf.extend_from_slice(&valid_g1_bytes);
// plot_index (u16), meta_group (u8), strength (u8)
buf.extend_from_slice(&[0u8, 0u8, 0u8, 1u8]);
// proof: length-prefixed garbage (128 bytes, wrong internal structure)
buf.extend_from_slice(&[0u8, 0u8, 0u8, 128u8]);
buf.extend_from_slice(&[0xffu8; 128]);

// This succeeds:
let pos = ProofOfSpace::parse::<false>(&mut Cursor::new(&buf)).unwrap();
assert_eq!(pos.version, 1);

// This panics:
let mut digest = Sha256::new();
pos.update_digest(&mut digest); // PANIC: "internal error. Can't compute hash of invalid ProofOfSpace"
