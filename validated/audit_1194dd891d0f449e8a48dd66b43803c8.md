### Title
Missing Declared-Class Validation in `execute_replace_class` Enables Permanent Freezing of Contract Funds — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary felt value as the new class hash without verifying that it corresponds to a declared contract class. This is an exact structural analog to the external report's missing precondition guard: just as `setGoalReached()` sets a critical flag without checking `totalRaised >= fundraisingGoal`, `execute_replace_class` transitions a contract's class hash to an unvalidated value. The consequence is that any contract can permanently brick itself (and freeze all funds it holds) by replacing its class with an undeclared hash, after which no transaction calling that contract can ever be included in a valid block.

---

### Finding Description

In `execute_replace_class`, the OS reads the requested `class_hash` directly from the syscall request and writes it into `contract_state_changes` with no check that the hash is a declared class: [1](#0-0) 

The code itself contains an explicit acknowledgment of the missing guard:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

After this update, any future call to the contract reaches `execute_entry_point`, which performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // ← the invalid/undeclared hash
);
``` [2](#0-1) 

For an undeclared class hash, `dict_read` returns `0` (the default for an unwritten key). The OS then calls `find_element` searching for a `CompiledClassFact` with `hash == 0`: [3](#0-2) 

`find_element` is a hard Cairo assertion — if no compiled class with that hash exists in `compiled_class_facts_bundle`, the OS execution aborts with an assertion failure. This is not a graceful per-transaction revert; it is a fatal failure of the entire OS run. The sequencer therefore cannot include any transaction that calls the bricked contract, making all funds held by it permanently inaccessible.

The `replace_class` syscall is dispatched from `execute_syscalls` using the executing contract's own address: [4](#0-3) 

Any contract — deployed by any unprivileged user — can invoke this syscall.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value, every subsequent call to that contract causes a hard OS abort. The sequencer is forced to exclude all such transactions. Any ERC-20 tokens, ETH, or other assets held in the contract's storage become permanently unrecoverable. There is no upgrade path, no rescue mechanism, and no way to revert the class hash change after the block containing the `replace_class` call is finalized.

---

### Likelihood Explanation

The attack requires only that a contract deployer (an unprivileged role) deploy a contract that calls `replace_class` with an arbitrary felt. This is a single transaction. A realistic scenario is a malicious DeFi contract that attracts user deposits before the deployer triggers the class replacement, or a buggy contract that accidentally passes an undeclared hash. The TODO comment confirms the developers are aware the check is absent and have deferred it.

---

### Recommendation

In `execute_replace_class`, before writing the new class hash to state, verify that it is present in `contract_class_changes` (i.e., has a non-zero compiled class hash):

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // Validate that the class hash is declared.
    let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
    if (compiled_class_hash == 0) {
        write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
        return ();
    }
    ...
}
```

### Proof of Concept

1. Attacker deploys `MaliciousVault` — a contract that accepts token deposits from users.
2. Users deposit funds; the contract accumulates a balance.
3. Attacker sends a transaction that calls `replace_class(class_hash=0xdeadbeef)` from within `MaliciousVault`. The OS processes this without error because `execute_replace_class` performs no declared-class check.
4. The block is finalized. `MaliciousVault`'s `StateEntry.class_hash` is now `0xdeadbeef` in the global state.
5. Any user who submits a withdrawal transaction targeting `MaliciousVault` causes the OS to call `dict_read(key=0xdeadbeef)` → returns `0` → `find_element(..., key=0)` → hard assertion failure.
6. The sequencer cannot include any such transaction. All user funds are permanently frozen.
