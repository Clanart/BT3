### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Arbitrary Class Hash Substitution — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS does not verify that the caller-supplied `class_hash` corresponds to a previously declared contract class. Any contract can invoke the `replace_class` syscall with an arbitrary, undeclared class hash. The OS proof system accepts this state transition as valid, permanently corrupting the contract's class pointer and freezing any funds held by the contract.

---

### Finding Description

`execute_replace_class` reads the new class hash directly from the syscall request and writes it unconditionally into `contract_state_changes`:

```cairo
// Replaces the class.
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0)

By contrast, `execute_declare_transaction` in `transaction_impls.cairo` is the only place that legitimately populates `contract_class_changes` (the class hash → compiled class hash mapping), and it enforces `prev_value=0` to prevent re-declaration:

```cairo
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [3](#0-2) 

`execute_replace_class` only updates `contract_state_changes` (the per-contract class pointer), never `contract_class_changes`. There is therefore no OS-level enforcement that the hash written by `replace_class` has a corresponding compiled class entry.

The syscall is dispatched unconditionally from `execute_syscalls` whenever the selector matches `REPLACE_CLASS_SELECTOR`:

```cairo
if (selector == REPLACE_CLASS_SELECTOR) {
    execute_replace_class(contract_address=execution_context.execution_info.contract_address);
    ...
}
``` [4](#0-3)

### Impact Explanation

**Critical — Permanent freezing of funds.**

After `replace_class` writes an undeclared hash into `contract_state_changes`, the state commitment records the contract as having a class hash for which no compiled class exists. Any subsequent call to that contract will fail to resolve its class, making the contract permanently unexecutable. All tokens or assets held in the contract's storage (e.g., a DeFi pool, escrow, or multisig) are irrecoverably frozen. Because the OS proof system accepts the transition as valid (no assertion fails), the corrupted state is committed on-chain and cannot be rolled back.

---

### Likelihood Explanation

Any contract deployer — an unprivileged role — can deploy a contract whose logic calls `replace_class` with an arbitrary felt value (e.g., `1`, `0xdeadbeef`, or any hash that was never declared). The deployer then submits a normal invoke transaction to trigger that path. The OS has no gate on this path; the only acknowledged guard is the unimplemented TODO. If the sequencer's blockifier does not independently enforce the declared-class invariant (or if a future sequencer version relaxes it), the OS proof system will produce a valid proof for the corrupted state, and the L1 verifier will accept it.

---

### Recommendation

Inside `execute_replace_class`, before writing the new class hash, assert that `class_hash` has a non-zero entry in `contract_class_changes` (i.e., it was previously declared). This mirrors the check already present in `execute_declare_transaction` and closes the gap acknowledged by the TODO comment. Concretely:

```cairo
// Assert the class has been declared (has a compiled class hash entry).
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This ensures the OS proof system — not just the sequencer — enforces the invariant.

---

### Proof of Concept

1. Deploy contract `MaliciousContract` whose `__execute__` function issues a `replace_class` syscall with `class_hash = 0xdeadbeef` (never declared).
2. Submit an invoke transaction targeting `MaliciousContract.__execute__`.
3. The OS processes the transaction: `execute_replace_class` reads `request.class_hash = 0xdeadbeef`, skips the missing declared-class check, and writes `contract_state_changes[MaliciousContract].class_hash = 0xdeadbeef`.
4. `contract_class_changes` has no entry for `0xdeadbeef` — no assertion in the OS catches this.
5. `state_update` commits the corrupted state; the proof is valid; the L1 verifier accepts it.
6. Any subsequent call to `MaliciousContract` fails to resolve its class. All funds in its storage are permanently frozen.

        // Not enough gas to execute the syscall.
        return ();
