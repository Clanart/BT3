### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (`execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not validate that the requested new class hash corresponds to a declared contract class. Any contract can call `replace_class` with an arbitrary, undeclared hash. The OS accepts this state transition without complaint, permanently bricking the contract and freezing all user funds held within it.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall and directly writes the caller-supplied `class_hash` into the contract state with no check that the hash has ever been declared:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The TODO comment at line 898 explicitly acknowledges the missing invariant enforcement. The OS is the authoritative verifier of state transitions; if it does not enforce that `replace_class` targets a declared class, a proof of an invalid state transition (contract class → undeclared hash) is accepted by the protocol.

Once a contract's `class_hash` is set to an undeclared value, every subsequent call to that contract fails at class resolution. There is no recovery path: the contract is permanently bricked and all funds it holds are permanently frozen. [1](#0-0) 

For comparison, the `execute_declare_transaction` path in `transaction_impls.cairo` does enforce `assert_not_zero(compiled_class_hash)` and writes to `contract_class_changes` only after verifying the class hash pre-image — but `execute_replace_class` performs no equivalent lookup into `contract_class_changes`. [2](#0-1) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any user funds deposited into a contract whose class is replaced with an undeclared hash become permanently inaccessible. The contract cannot be called, upgraded, or recovered. Because the OS accepts the proof of this state transition, the freeze is enshrined in the canonical chain state.

---

### Likelihood Explanation

The attack path requires no privileged role:

1. An attacker deploys a contract (permissionless on StarkNet) that exposes a `deposit` entry point and a `freeze` entry point. The `freeze` entry point calls `replace_class(arbitrary_undeclared_hash)`.
2. Users deposit funds into the contract.
3. The attacker submits an invoke transaction calling `freeze`.
4. The sequencer executes and includes the transaction; the OS verifies the resulting proof without checking the class hash.
5. The contract's class is now set to an undeclared hash. All subsequent calls revert. Funds are permanently frozen.

The attacker is an ordinary contract deployer — an unprivileged transaction sender. No operator key, sequencer compromise, or privileged access is required.

---

### Recommendation

Inside `execute_replace_class`, before writing the new class hash to `contract_state_changes`, verify that the hash exists in `contract_class_changes` (i.e., it was declared in the current or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` for `class_hash` and assert the returned compiled class hash is non-zero:

```cairo
// Enforce that the replacement class has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the invariant already enforced in `execute_declare_transaction` and closes the gap acknowledged by the existing TODO comment. [3](#0-2) 

---

### Proof of Concept

```
// Malicious contract (pseudocode Sierra/Cairo 1)
#[external]
fn deposit(amount: u256) { ... }   // accepts user funds

#[external]
fn freeze() {
    // 0xdeadbeef is not a declared class hash
    replace_class(class_hash: 0xdeadbeef);
}
```

1. Attacker deploys the contract above.
2. Users call `deposit`, locking funds in the contract.
3. Attacker calls `freeze` via an invoke transaction.
4. OS processes `execute_replace_class` with `class_hash = 0xdeadbeef`; no validation occurs; state is updated.
5. All future calls to the contract fail at class resolution — funds are permanently frozen.
