### Title
Stale Fee Market Params Snapshot in Cosmos Ante Handler Allows Bypass of Updated MinGasPrice and BaseFee Enforcement — (File: `ante/cosmos/min_gas_price.go`, `evmd/ante/handler_options.go`)

---

### Summary

`MinGasPriceDecorator` and `NewDynamicFeeChecker` for Cosmos (non-EVM) transactions use fee market parameters captured once at ante handler construction time rather than reading live state from the keeper. After governance raises `MinGasPrice`, or as the EIP-1559 base fee rises block-by-block, Cosmos transactions continue to be validated against the stale (lower) threshold, allowing them to commit with fees below the enforced minimum.

---

### Finding Description

**Root cause — stale snapshot captured at construction time**

In `newCosmosAnteHandler` (`evmd/ante/handler_options.go:178`), `feemarketParams` is fetched from the keeper exactly once, at the moment the ante handler chain is built (app startup):

```go
feemarketParams := options.FeeMarketKeeper.GetParams(ctx)   // line 180
```

A pointer to this local copy is then passed into both `MinGasPriceDecorator` and `NewDynamicFeeChecker`:

```go
cosmos.NewMinGasPriceDecorator(options.FeeMarketKeeper, evmDenom, &feemarketParams),  // line 198
...
txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)         // line 187
```

**`MinGasPriceDecorator` — keeper stored but never consulted**

`MinGasPriceDecorator` stores `feesKeeper` but reads `MinGasPrice` exclusively from the stale pointer:

```go
// ante/cosmos/min_gas_price.go:54
minGasPrice := mpd.feemarketParams.MinGasPrice
```

`mpd.feesKeeper` is never called inside `AnteHandle`. When governance executes a parameter change via `SetParams`, the KV-store is updated but the snapshot pointer is not. Every subsequent Cosmos transaction is checked against the old, lower floor.

**`NewDynamicFeeChecker` — stale base fee used for every block**

`NewDynamicFeeChecker` (`ante/evm/fee_checker.go:56`) derives the base fee from the same stale pointer:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
```

The EIP-1559 base fee is updated every block by `BeginBlock → CalculateBaseFee → SetBaseFee → SetParams` (`x/feemarket/keeper/abci.go:31-51`, `x/feemarket/keeper/params.go:72-78`). Because `feemarketParams.BaseFee` is never refreshed, the dynamic fee checker for Cosmos transactions always uses the base fee from app startup.

**Contrast with EVM ante handler — correct per-execution fetch**

`newEthAnteHandler` (`evmd/ante/handler_options.go:86`) correctly fetches fresh state on every invocation:

```go
blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
baseFee := blockCfg.BaseFee
```

EVM transactions are always checked against the current base fee; Cosmos transactions are not.

---

### Impact Explanation

This is a fee market ante handler bug that permits invalid Cosmos transactions to commit:

1. **MinGasPrice bypass**: After a governance proposal raises `MinGasPrice`, any user can submit Cosmos transactions (bank sends, IBC transfers, governance votes, staking operations) with a gas price between the old and new minimum. `MinGasPriceDecorator` accepts them because it reads the stale value. Fees that should be rejected are collected at the lower rate, mis-accounting user fees relative to the enforced policy.

2. **BaseFee bypass (dynamic fee checker path)**: In active networks the base fee rises continuously when blocks are full. Cosmos transactions using `NewDynamicFeeChecker` are checked against the base fee at app startup. As the base fee climbs, the gap between the enforced threshold and the stale threshold widens, allowing progressively under-priced Cosmos transactions to be included in blocks.

Both paths allow transactions that violate the current fee floor to commit to state, directly matching the allowed High impact: *"fee market, ante handler … bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

---

### Likelihood Explanation

- **BaseFee path**: Triggered automatically. Any network with non-trivial activity will see the base fee rise above its startup value within minutes. No attacker action beyond submitting a normal Cosmos transaction is required.
- **MinGasPrice path**: Requires a governance parameter update, which is a routine operational event on live chains. Once the update passes, every Cosmos transaction submitted at the old price exploits the bypass until the node is restarted.

The entry path is entirely unprivileged: a standard Cosmos SDK transaction submitted via the normal broadcast RPC.

---

### Recommendation

Replace the stale snapshot read in `MinGasPriceDecorator.AnteHandle` with a live keeper call:

```go
// ante/cosmos/min_gas_price.go — AnteHandle
params := mpd.feesKeeper.GetParams(ctx)   // read live state
minGasPrice := params.MinGasPrice
```

For `NewDynamicFeeChecker`, remove the `feemarketParams` parameter and fetch current params from the keeper inside the returned closure, mirroring the pattern used in `newEthAnteHandler`.

---

### Proof of Concept

**MinGasPrice bypass:**
1. Chain starts with `MinGasPrice = 1`.
2. Governance proposal passes setting `MinGasPrice = 100`; `SetParams` updates the KV-store.
3. Attacker submits `MsgSend` with `gasPrice = 1`.
4. `MinGasPriceDecorator.AnteHandle` reads `mpd.feemarketParams.MinGasPrice = 1` (stale), computes `requiredFees` using `1`, passes the check.
5. Transaction is included in a block despite violating the current `MinGasPrice = 100`.

**BaseFee bypass (dynamic fee checker):**
1. Chain starts with `BaseFee = 100` (stored in `feemarketParams` snapshot).
2. After sustained high utilization, `CalculateBaseFee` raises `BaseFee` to `1000` in the KV-store.
3. Attacker submits a Cosmos transaction with `fee / gas = 100`.
4. `NewDynamicFeeChecker` calls `types.GetBaseFee(…, feemarketParams)` → returns `100` (stale), `feeCap (100) >= baseFeeInt (100)` passes.

}

func (mpd MinGasPriceDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
	feeTx, ok := tx.(sdk.FeeTx)
	if !ok {
		return ctx, errorsmod.Wrapf(errortypes.ErrInvalidType, "invalid transaction type %T, expected sdk.FeeTx", tx)
	}

	minGasPrice := mpd.feemarketParams.MinGasPrice

	// Short-circuit if min gas price is 0 or if simulating
	if minGasPrice.IsZero() || simulate {
		return next(ctx, tx, simulate)
	}
	minGasPrices := sdk.DecCoins{
		{
			Denom:  mpd.evmDenom,
			Amount: minGasPrice,
		},
	}

	feeCoins := feeTx.GetFee()
	gas := feeTx.GetGas()

	requiredFees := make(sdk.Coins, 0)

	// Determine the required fees by multiplying each required minimum gas
	// price by the gas limit, where fee = ceil(minGasPrice * gasLimit).
	gasLimit := sdkmath.LegacyNewDecFromBigInt(new(big.Int).SetUint64(gas))

	for _, gp := range minGasPrices {
		fee := gp.Amount.Mul(gasLimit).Ceil().RoundInt()
		if fee.IsPositive() {
			requiredFees = requiredFees.Add(sdk.Coin{Denom: gp.Denom, Amount: fee})
		}
	}

	if !feeCoins.IsAnyGTE(requiredFees) {
		return ctx, errorsmod.Wrapf(errortypes.ErrInsufficientFee,
			"provided fee < minimum global fee (%s < %s). Please increase the gas price.",
			feeCoins,
			requiredFees)
	}

	return next(ctx, tx, simulate)
}

func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
