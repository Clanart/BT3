### Title
`execute_deploy` Unconditionally Hardcodes `failure_flag=0`, Masking Constructor Failures from Caller Contracts — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_deploy` function in the StarkNet OS always writes `failure_flag=0` (success) into the deploy syscall response, regardless of whether the constructor execution actually succeeded or reverted. Any caller contract using the `deploy` syscall will always observe a "successful" deployment, even when the constructor failed. This is the direct analog of the PreimageOracle bug: a failed sub-execution's outcome is silently suppressed and the caller is given a false success signal, enabling incorrect downstream state transitions.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_deploy` function calls `deploy_contract` (which internally calls `execute_entry_point` for the constructor), then unconditionally writes a hardcoded `failure_flag=0` into the response header:

```cairo
// Write the response header.
// TODO(Yoni, 1/1/2026): support failures.
assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=0);
``` [1](#0-0) 

The TODO comment is an explicit developer acknowledgment that failure propagation is not implemented. Critically, `deploy_contract` only returns `(retdata_size, retdata)` — it does not return `is_reverted` — so `execute_deploy` has no mechanism to detect constructor failure even if it tried:

```cairo
with remaining_gas {
    let (retdata_size, retdata) = deploy_contract(
        block_context=block_context, constructor_execution_context=constructor_execution_context
    );
}
``` [2](#0-1) 

Compare this with the correct pattern used in `contract_call_helper` for the `call_contract` syscall, which properly propagates `is_reverted`:

```cairo
assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=is_reverted);
``` [3](#0-2) 

The `execute_entry_point` function does correctly call `handle_revert` when a constructor reverts, undoing the constructor's storage writes. However, this revert is invisible to `execute_deploy` because `deploy_contract` swallows the `is_reverted` signal. The OS then certifies the deploy as successful in the proven trace. [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent Freezing of Funds / Direct Loss of Funds.**

A caller contract (e.g., a factory or account abstraction wallet) that deploys a sub-contract via the `deploy` syscall will always receive `failure_flag=0`. If the constructor fails (reverts), the caller proceeds as if deployment succeeded:

1. The caller stores the returned `contract_address` as a trusted, initialized contract.
2. In a subsequent transaction, the caller transfers funds (tokens, ETH-bridged assets) to that address.
3. The deployed contract is in an uninitialized state — its constructor storage writes were reverted by `handle_revert`, so any access-control variables, owner fields, or withdrawal keys are absent.
4. Funds sent to the uninitialized contract are permanently frozen: no withdrawal path exists because the constructor never completed.

Additionally, the OS proves a state transition where the deploy succeeded, while the sequencer's blockifier may record the deploy as failed. This divergence between the proven state and the actual execution state constitutes an incorrect state commitment submitted to L1, which is the root cause of a potential chain split between nodes that trust the proof and nodes that re-execute.

---

### Likelihood Explanation

Any unprivileged contract deployer can trigger this. Constructors fail under attacker-reachable conditions:

- **Out-of-gas**: A caller contract can be invoked with a gas budget that is sufficient for the outer call but insufficient for the constructor (the 63/64 gas forwarding rule applies in nested calls).
- **Explicit revert**: A constructor can be written to revert under specific calldata or storage conditions.
- **Failing inner syscall**: A constructor that calls another contract or reads storage can be made to fail.

No privileged role, leaked key, or operator cooperation is required. The attack is fully reachable from an unprivileged transaction sender or contract deployer.

---

### Recommendation

1. Modify `deploy_contract` to return `is_reverted` alongside `retdata_size` and `retdata`.
2. In `execute_deploy`, use the returned `is_reverted` to set `failure_flag` correctly, mirroring the pattern in `contract_call_helper`.
3. Ensure that when `is_reverted=1`, the contract address registration (class hash entry in `contract_state_changes`) is also reverted, so the address does not persist in state after a failed constructor.
4. Remove the hardcoded `failure_flag=0` and the associated TODO comment once the fix is in place.

---

### Proof of Concept

**Setup:**
- Contract A: a factory contract that calls `deploy` to deploy Contract B, stores the returned address, and later sends funds to it.
- Contract B: a contract whose constructor explicitly reverts (or is invoked with insufficient gas to complete).

**Attack Steps:**

1. Attacker submits Transaction 1: calls Contract A's `deploy_vault()` function.
2. Contract A issues a `deploy` syscall targeting Contract B's class hash.
3. Contract B's constructor reverts (out-of-gas or explicit revert).
4. `execute_entry_point` calls `handle_revert`, undoing constructor storage writes.
5. `execute_deploy` writes `ResponseHeader(gas=remaining_gas, failure_flag=0)` — hardcoded success.
6. Contract A reads `failure_flag=0`, stores Contract B's address as `trusted_vault`.
7. Transaction 1 completes successfully; the OS proves this state.
8. Attacker submits Transaction 2: triggers Contract A to send funds to `trusted_vault`.
9. Contract B exists at the address (class hash registered) but has no initialized storage — no owner, no withdrawal key.
10. Funds are permanently frozen in Contract B with no recovery path. [5](#0-4)

    let request = cast(syscall_ptr + RequestHeader.SIZE, DeployRequest*);

    local constructor_calldata_size = request.constructor_calldata_end - constructor_calldata_start;

    let specific_base_gas_cost = DEPLOY_GAS_COST + DEPLOY_CALLDATA_FACTOR_GAS_COST *
        constructor_calldata_size;
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=specific_base_gas_cost, request_struct_size=DeployRequest.SIZE

    if (success == FALSE) {

        return ();

    local caller_execution_info: ExecutionInfo* = caller_execution_context.execution_info;

    // Verify deploy_from_zero is either 0 (FALSE) or 1 (TRUE).
    tempvar deploy_from_zero = request.deploy_from_zero;
    assert deploy_from_zero * (deploy_from_zero - 1) = 0;

    let deployer_address = (1 - deploy_from_zero) * caller_address;

    let hash_ptr = selectable_builtins.pedersen;
    with hash_ptr {
        let (contract_address) = get_contract_address(

            class_hash=request.class_hash,
            constructor_calldata_size=constructor_calldata_size,
            constructor_calldata=constructor_calldata_start,
            deployer_address=deployer_address,
        );
    }
    tempvar builtin_ptrs = new BuiltinPointers(
        selectable=SelectableBuiltins(
            pedersen=hash_ptr,
            range_check=selectable_builtins.range_check,
            ecdsa=selectable_builtins.ecdsa,
            bitwise=selectable_builtins.bitwise,
            ec_op=selectable_builtins.ec_op,
            poseidon=selectable_builtins.poseidon,
            segment_arena=selectable_builtins.segment_arena,
            range_check96=selectable_builtins.range_check96,
            add_mod=selectable_builtins.add_mod,
            mul_mod=selectable_builtins.mul_mod,
        ),
        non_selectable=builtin_ptrs.non_selectable,
    );

    tempvar constructor_execution_context = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_CONSTRUCTOR,
        class_hash=request.class_hash,
        calldata_size=constructor_calldata_size,
        calldata=constructor_calldata_start,
        execution_info=new ExecutionInfo(
            block_info=caller_execution_info.block_info,
            tx_info=caller_execution_info.tx_info,
            caller_address=caller_address,
            contract_address=contract_address,
            selector=CONSTRUCTOR_ENTRY_POINT_SELECTOR,
        ),
        deprecated_tx_info=caller_execution_context.deprecated_tx_info,
    );

    with remaining_gas {
        let (retdata_size, retdata) = deploy_contract(
            block_context=block_context, constructor_execution_context=constructor_execution_context
        );
    }

    // TODO(Yoni, 1/1/2026): consider sharing code with call_contract_helper.
    let response_header = cast(syscall_ptr, ResponseHeader*);
    let syscall_ptr = syscall_ptr + ResponseHeader.SIZE;

    // Write the response header.
    // TODO(Yoni, 1/1/2026): support failures.
    assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=0);

    let response = cast(syscall_ptr, DeployResponse*);
    // Advance syscall pointer to the next syscall.
    let syscall_ptr = syscall_ptr + DeployResponse.SIZE;

    %{ CheckNewDeployResponse %}

    // Write the response.
    relocate_segment(src_ptr=response.constructor_retdata_start, dest_ptr=retdata);
    assert [response] = DeployResponse(
        contract_address=contract_address,
        constructor_retdata_start=retdata,
        constructor_retdata_end=retdata + retdata_size,
    );

    return ();
