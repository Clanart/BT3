### Title
Unbounded `AnteCache` Memory Exhaustion via Stale Nonce Entries from Mempool-Evicted Transactions - (File: `ante/cache/antecache.go`)

### Summary
The `AnteCache` in Ethermint's ante handler accumulates `(address, nonce)` entries for every Ethereum transaction that passes `CheckTx`. These entries are only deleted when the **same transaction is later delivered** (DeliverTx) or when a **nonce mismatch occurs during ReCheckTx**. Transactions that pass `CheckTx` but are subsequently evicted from the mempool (e.g., because the EIP-1559 base fee rises above their gas price) leave permanent stale entries in the cache. When the cache is configured with `maxTx == 0` (unbounded), this constitutes an unbounded memory growth path reachable by any unprivileged user via standard `eth_sendRawTransaction` submissions.

### Finding Description

**Root cause 1 — Unbounded cache when `maxTx == 0`:**

The `AnteCache.Set` eviction guard is gated on `maxTx > 0`:

```go
if c.maxTx > 0 && len(c.cache) >= c.maxTx {
    // evict oldest
}
```

When `maxTx == 0`, the condition is never true and the map grows without bound. [1](#0-0) 

**Root cause 2 — `setAnteHandler` passes `mempoolMaxTxs` without the `DefaultMaxTxs` fallback:**

`setupMempoolAndProposalHandlers` (the mempool wiring) guards against a zero value:
```go
if maxTxs <= 0 {
    maxTxs = srvconfig.DefaultMaxTxs
}
```
But `setAnteHandler` passes the raw value directly to `NewAnteCache`:
```go
AnteCache: cache.NewAnteCache(mempoolMaxTxs),
``` [2](#0-1) [3](#0-2) 

If `--mempool.max-txs` is unset or explicitly `0`, the mempool is bounded by `DefaultMaxTxs` but the `AnteCache` is **unbounded**.

**Root cause 3 — Stale entry leak: cleanup only on DeliverTx or nonce-mismatch ReCheckTx:**

Entries are added to the cache after a successful `CheckTx` ante chain: [4](#0-3) 

Entries are deleted only in two cases inside `CheckAndSetEthSenderNonce`:
1. DeliverTx when the entry exists (`ex == true`)
2. ReCheckTx when there is a **nonce mismatch** [5](#0-4) 

**The gap:** When a transaction fails ReCheckTx due to an **insufficient fee** (base fee rose), `CheckEthMempoolFee` returns an error before `CheckAndSetEthSenderNonce` is ever reached. The cache entry is never deleted. The transaction is then evicted from the mempool, but its `(address, nonce)` entry remains in the `AnteCache` permanently. [6](#0-5) 

There is no background goroutine, timer, or periodic sweep that cleans up stale entries.

### Impact Explanation

With `maxTx == 0`, an attacker can accumulate an unbounded number of stale `(address, nonce)` entries in the `AnteCache`. Each entry is a `map[TxNonce]*list.Element` entry plus a `list.Element` node. At scale (millions of entries), this exhausts the heap of the validator process, causing an OOM kill. A coordinated attack targeting multiple validators simultaneously can halt consensus. This is a valid unprivileged transaction path that can halt the chain.

### Likelihood Explanation

The attack requires only:
1. Generating Ethereum keypairs (free, off-chain)
2. Funding each address with the minimum amount to pass `CheckEthCanTransfer` (a small, one-time cost per address)
3. Submitting transactions with a gas price just above the current base fee via `eth_sendRawTransaction`
4. Waiting for the base fee to rise (normal network activity), causing the transactions to fail `CheckEthMempoolFee` during ReCheckTx and be evicted from the mempool without triggering cache cleanup

No validator keys, governance access, or privileged roles are required. The cost per stale cache entry is minimal (one cheap EVM transaction fee). The `maxTx == 0` condition is reachable in any deployment where `--mempool.max-txs` is not explicitly set to a positive value or where the app options do not propagate the flag default.

### Recommendation

1. **Apply the same `DefaultMaxTxs` fallback to `AnteCache`** that `setupMempoolAndProposalHandlers` already applies to the mempool — never pass `0` to `NewAnteCache` in production.
2. **Add timeout-based cleanup**: run a background goroutine that periodically sweeps the `AnteCache` and removes entries older than a configurable TTL (e.g., 5 minutes).
3. **Hook mempool eviction**: when the mempool removes a transaction, call `AnteCache.Delete` for its `(address, nonce)` pair.
4. **Expand the ReCheckTx deletion path** in `CheckAndSetEthSenderNonce` to delete the cache entry on any ante failure, not only on nonce mismatch.

### Proof of Concept

```go
// Attacker controls N Ethereum keypairs, each funded with minimal balance.
// Base fee is currently F. Attacker submits txs with gasPrice = F + 1 (just above minimum).

for i := 0; i < 1_000_000; i++ {
    key := attackerKeys[i]
    // Craft a valid EIP-1559 tx: nonce=0, gasPrice=baseFee+1, valid sig
    tx := buildEthTx(key, nonce=0, gasPrice=currentBaseFee+1)
    // Passes CheckTx: valid sig, sufficient balance, correct nonce
    // → AnteCache.Set(address_i, 0) called
    broadcastCheckTx(tx)
}

// Now cause base fee to rise (e.g., by filling blocks with high-gas txs,
// or simply wait for organic network activity).
// During the next ReCheckTx round, each attacker tx fails at CheckEthMempoolFee
// (fee too low) BEFORE CheckAndSetEthSenderNonce is reached.
// → AnteCache entries are NEVER deleted.
// → 1,000,000 stale (address, nonce) entries remain in memory permanently.
// → With maxTx==0, the AnteCache map grows without bound → OOM.
```

type AnteCache struct {
	mu    sync.RWMutex
	cache map[TxNonce]*list.Element
	order *list.List
	// - if maxTx == 0, there is no cap on the number of transactions in the cache
	// - if maxTx > 0, the cache will cap the number of transactions it stores,
	// - if maxTx < 0, the cache is a no-op cache.
	maxTx int
}

func NewAnteCache(mempoolMaxTxs int) *AnteCache {
	return &AnteCache{
		cache: make(map[TxNonce]*list.Element),
		order: list.New(),
		maxTx: mempoolMaxTxs,
	}
}

// Set the TxNonce
func (c *AnteCache) Set(address string, nonce uint64) {
	if c.maxTx < 0 {
		return
	}
	c.mu.Lock()
	defer c.mu.Unlock()
	key := TxNonce{address, nonce}
	if elem, ok := c.cache[key]; ok {
		c.order.MoveToBack(elem)
		return
	}
	if c.maxTx > 0 && len(c.cache) >= c.maxTx {
		if front := c.order.Front(); front != nil {
			oldKey := front.Value.(TxNonce)
			delete(c.cache, oldKey)
			c.order.Remove(front)
		}
	}
	elem := c.order.PushBack(key)
	c.cache[key] = elem
```

```go
func CheckAndSetEthSenderNonce(
	ctx sdk.Context, tx sdk.Tx, ak evmtypes.AccountKeeper, unsafeUnOrderedTx bool, accountGetter AccountGetter, nonceCache *cache.AnteCache,
) ([]cache.TxNonce, error) {
	var pending []cache.TxNonce

	for _, msg := range tx.GetMsgs() {
		msgEthTx, ok := msg.(*evmtypes.MsgEthereumTx)
		if !ok {
			return nil, errorsmod.Wrapf(errortypes.ErrUnknownRequest, "invalid message type %T, expected %T", msg, (*evmtypes.MsgEthereumTx)(nil))
		}

		tx := msgEthTx.AsTransaction()

		from := msgEthTx.GetFrom()
		acc := accountGetter(from)
		if acc == nil {
			return nil, errorsmod.Wrapf(
				errortypes.ErrUnknownAddress,
				"account %s is nil", common.BytesToAddress(from.Bytes()),
			)
		}
		expectedNonce := acc.GetSequence()
		txNonce := tx.Nonce()
		fromStr := from.String()

		// if flag is set, we bypass nonce all check verification
		if !unsafeUnOrderedTx {
			ex := nonceCache.Exists(fromStr, txNonce)
			// to support tx replacement, we check if the transaction nonce exists in the cache and if yes we skip
			// nonce verification, and we don't set the sequence
			// We allow skip verification only during CheckTx to keep sequence safe during the execution.
			if ctx.IsCheckTx() && !ctx.IsReCheckTx() && ex {
				continue
			}

			// nonce verification, the sequence needs to be in order
			if txNonce != expectedNonce {
				// delete in case of recheck tx
				if ex {
					nonceCache.Delete(fromStr, txNonce)
				}
				return nil, errorsmod.Wrapf(
					errortypes.ErrInvalidSequence,
					"invalid nonce; got %d, expected %d", txNonce, expectedNonce,
				)
			}

			if ctx.IsCheckTx() {
				if !ctx.IsReCheckTx() {
					pending = append(pending, cache.TxNonce{Address: fromStr, Nonce: txNonce})
				}
			} else if ex {
				// delete in case of deliver tx
				nonceCache.Delete(fromStr, txNonce)
			}
