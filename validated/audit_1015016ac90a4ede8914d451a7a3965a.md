### Title
Missing Duplicate-Key Validation in `batch_insert` Bulk Path Corrupts DataLayer Merkle Tree - (File: `crates/chia-datalayer/src/merkle/blob.rs`)

### Summary
`MerkleBlob::batch_insert` skips the duplicate-key existence check for all items processed through its bulk insertion path (items beyond the first two when the tree already has ≥2 leaves). An attacker-controlled call with repeated `KeyId` values silently overwrites the `key_to_index` cache while writing multiple orphaned leaf nodes into the blob, corrupting the Merkle tree and enabling forged or invalid proofs of inclusion.

### Finding Description
`MerkleBlob::insert` (the single-item path) correctly guards against duplicate keys: [1](#0-0) 

`batch_insert` uses `insert` only for the last 1–2 items when the tree has ≤1 leaf (popped from the end of the input vector): [2](#0-1) 

All remaining items — the bulk of the batch — are written directly via `insert_entry_to_blob` with **no duplicate-key check**: [3](#0-2) 

`insert_entry_to_blob` calls `block_status_cache.add_leaf`, which silently overwrites the `key_to_index` HashMap entry for any repeated key: [4](#0-3) 

`add_leaf` itself performs no existence check before overwriting: [5](#0-4) 

The result is that the blob contains multiple leaf nodes sharing the same `KeyId`, while `key_to_index` only tracks the last-written index. The earlier leaf nodes become structurally orphaned — they are reachable by the tree iterator but not by the cache — causing `check_just_integrity` to fail with `IntegrityKeyToIndexCacheIndex` or `IntegrityKeyToIndexCacheLength`. The root hash is computed over a structurally invalid tree.

The same bulk path also does not check whether a key already exists in the tree before the batch, so `batch_insert` can silently shadow an existing leaf, making the old leaf unreachable through the cache while it remains in the blob.

### Impact Explanation
A corrupted `key_to_index` cache means `get_proof_of_inclusion` generates a proof for the wrong (last-written) leaf node for a duplicated key, while the tree root is computed over a blob containing orphaned leaves. This constitutes DataLayer Merkle tree root corruption and invalid proof-of-inclusion generation — an attacker can cause the DataLayer to attest to state that does not correspond to the actual committed key-value mapping. [6](#0-5) 

### Likelihood Explanation
`batch_insert` is directly exposed as a public Python binding (`py_batch_insert`) with no additional validation layer: [7](#0-6) 

It is declared in the public Python stub and called in production DataLayer sync flows: [8](#0-7) 

Any caller — including the Chia full node's DataLayer sync code — that passes a batch containing a repeated `KeyId` (or a `KeyId` already present in the tree) will silently corrupt the tree. The existing test `test_batch_insert` only uses unique keys and does not exercise the duplicate case. [9](#0-8) 

### Recommendation
Add a duplicate-key guard at the start of the bulk loop in `batch_insert`, mirroring the check in `insert`:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    // ... existing bulk insertion logic
}
```

Alternatively, collect all keys from the batch into a temporary `HashSet` before writing any nodes, and reject the entire batch if any duplicate is detected. The guard in `insert` at line 369 is the correct reference pattern. [10](#0-9) 

### Proof of Concept

```python
from chia_rs.datalayer import MerkleBlob, KeyId, ValueId
from chia_protocol import bytes32

blob = MerkleBlob(blob=bytearray())

# Pre-populate with 2 leaves so batch_insert uses the bulk path
blob.insert(KeyId(100), ValueId(100), bytes32(b'\x01' * 32))
blob.insert(KeyId(200), ValueId(200), bytes32(b'\x02' * 32))

# Now batch_insert with a duplicate key in the bulk path (items 1..N-2)
dup_key = KeyId(999)
keys_values = [(dup_key, ValueId(1)), (dup_key, ValueId(2)), (KeyId(300), ValueId(3))]
hashes = [bytes32(b'\xaa' * 32), bytes32(b'\xbb' * 32), bytes32(b'\xcc' * 32)]

blob.batch_insert(keys_values, hashes)
blob.calculate_lazy_hashes()

# check_integrity will raise, demonstrating tree corruption:
blob.check_integrity()  # raises IntegrityKeyToIndexCacheIndex or similar
```

The blob now contains two leaf nodes with `KeyId(999)` but `key_to_index` only maps to the last-written one. The root hash is computed over a structurally invalid tree, and any proof of inclusion for `KeyId(999)` reflects only the last-written value, not the first. [11](#0-10)

    fn check_just_integrity(&self) -> Result<(), Error> {
        let mut leaf_count: usize = 0;
        let mut internal_count: usize = 0;
        let mut child_to_parent: HashMap<TreeIndex, TreeIndex> = HashMap::new();

        for item in ParentFirstIterator::new(&self.blob, None) {
            let (index, block) = item?;
            if let Some(parent) = block.node.parent().0 {
                if child_to_parent.remove(&index) != Some(parent) {
                    return Err(Error::IntegrityParentChildMismatch(index));
                }
            }
            match block.node {
                Node::Internal(node) => {
                    internal_count += 1;
                    child_to_parent.insert(node.left, index);
                    child_to_parent.insert(node.right, index);
                }
                Node::Leaf(node) => {
                    leaf_count += 1;
                    let cached_index = self
                        .block_status_cache
                        .get_index_by_key(node.key)
                        .ok_or(Error::IntegrityKeyNotInCache(node.key))?;
                    if *cached_index != index {
                        return Err(Error::IntegrityKeyToIndexCacheIndex(
                            node.key,
                            index,
                            *cached_index,
                        ));
                    }
                    assert!(
                        !self.block_status_cache.is_index_free(index),
                        "{}",
                        format!("active index found in free index list: {index:?}")
                    );
                }
            }
        }

        let key_to_index_cache_length = self.block_status_cache.key_to_index.len();
        if leaf_count != key_to_index_cache_length {
            return Err(Error::IntegrityKeyToIndexCacheLength(
                leaf_count,
                key_to_index_cache_length,
            ));
        }
        let leaf_hash_to_index_cache_length = self.block_status_cache.leaf_hash_to_index.len();
        if leaf_count != leaf_hash_to_index_cache_length {
            return Err(Error::IntegrityLeafHashToIndexCacheLength(
                leaf_count,
                leaf_hash_to_index_cache_length,
            ));
        }
        let total_count = leaf_count + internal_count + self.block_status_cache.free_index_count();
        let extend_index = self.extend_index();
        if total_count != extend_index.0 as usize {
            return Err(Error::IntegrityTotalNodeCount(extend_index, total_count));
        }
        if !child_to_parent.is_empty() {
            return Err(Error::IntegrityUnmatchedChildParentRelationships(
                child_to_parent.len(),
            ));
        }

        Ok(())
    }

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }

    fn test_batch_insert(
        #[values(0, 1, 2, 10)] pre_inserts: usize,
        #[values(0, 1, 2, 8, 9)] count: usize,

        let mut blob = MerkleBlob::new(vec![]).unwrap();
        for i in 0..pre_inserts {
            let i = i as i64;
            blob.insert(
                KeyId(i),
                ValueId(i),
                &sha256_num(&i),

            )
            .unwrap();
        }
        open_dot(blob.to_dot().unwrap().set_note("initial"));

        let mut batch: Vec<((KeyId, ValueId), Hash)> = vec![];

        let mut batch_map: HashMap<KeyId, ValueId> = HashMap::new();
        for i in pre_inserts..(pre_inserts + count) {
            let i = i as i64;
            batch.push(((KeyId(i), ValueId(i)), sha256_num(&i)));
            batch_map.insert(KeyId(i), ValueId(i));
        }

        let before = blob.get_keys_values().unwrap();
        blob.batch_insert(batch).unwrap();
        let after = blob.get_keys_values().unwrap();

        open_dot(
            blob.to_dot()
                .unwrap()
                .set_note(&format!("after batch insert of {count} values")),
        );

        let mut expected = before.clone();
        expected.extend(batch_map);

        assert_eq!(after, expected);
    }
