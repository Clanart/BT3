### Title
`ProofOfSpace::update_digest` Panics on v2 Proof with Invalid Proof Bytes, Enabling Deterministic Node Crash via Crafted Network Message - (File: crates/chia-protocol/src/proof_of_space.rs)

### Summary
The `ProofOfSpace::update_digest` implementation for v2 proofs (version == 1) calls `self.quality_string().expect("internal error. Can't compute hash of invalid ProofOfSpace")`. Because `quality_string()` returns `None` when the raw proof bytes fail cryptographic validation, and because `parse` never validates those bytes, an attacker can deliver a well-formed v2 `ProofOfSpace` (or any block embedding one) that parses successfully but causes an unconditional `panic!` the moment any code path computes its hash. This is a deterministic, unprivileged, network-reachable crash.

### Finding Description

**Root cause — divergence between `parse` and `update_digest` for v2 proofs.**

`ProofOfSpace::parse` for version 1 reads `proof` as raw `Bytes` with no cryptographic validation. The only structural check is that exactly one of `pool_public_key` / `pool_contract_puzzle_hash` is set:

```rust
// parse: version 1 branch
let proof = <Bytes as Streamable>::parse::<TRUSTED>(input)?;
if pool_public_key.is_some() == pool_contract_puzzle_hash.is_some() {
    return Err(Error::InvalidPoS);
}
``` [1](#0-0) 

`update_digest` for the same version 1 branch does **not** hash the raw proof bytes. Instead it hashes the derived `quality_string()`, and hard-panics if that derivation fails:

```rust
self.quality_string()
    .expect("internal error. Can't compute hash of invalid ProofOfSpace")
    .update_digest(digest);
``` [2](#0-1) 

`quality_string()` returns `None` whenever `chia_pos2::quality_string_from_proof` fails — which it does for any proof bytes that do not satisfy the v2 proof-of-space relation:

```rust
pub fn quality_string(&self) -> Option<Bytes32> {
    if self.version != 1 { return None; }
    let k_size = (self.proof.len() * 8 / 128) as u8;
    let plot_id = self.compute_plot_id().to_bytes();
    chia_pos2::quality_string_from_proof(
        &plot_id, k_size, self.strength, self.proof.as_slice()
    )
    .map(|quality| { ... })
}
``` [3](#0-2) 

**Propagation path to consensus-critical hashing.**

`RewardChainBlock::update_digest` unconditionally delegates to `proof_of_space.update_digest`:

```rust
self.proof_of_space.update_digest(digest);
``` [4](#0-3) 

`UnfinishedBlock::partial_hash()` calls `self.reward_chain_block.hash()`, which calls `update_digest`:

```rust
pub fn partial_hash(&self) -> Bytes32 {
    self.reward_chain_block.hash().into()
}
``` [5](#0-4) 

Any node that receives an `UnfinishedBlock` (or `FullBlock` via `RewardChainBlock`) containing a v2 `ProofOfSpace` with invalid proof bytes and calls `partial_hash()` / `hash()` / `update_digest()` before completing proof-of-space validation will hit the unconditional `panic!`.

**Contrast with `stream`.**

`stream` for version 1 serializes the raw `self.proof` bytes directly and never calls `quality_string()`:

```rust
self.proof.stream(out)
``` [6](#0-5) 

This means `parse → stream` round-trips succeed for any proof bytes, while `parse → update_digest` panics for invalid ones. The two halves of the `Streamable` contract are inconsistent for v2 proofs.

**Additional panic in `FullBlock::update_digest`.**

`FullBlock::update_digest` also contains an unconditional `panic!` for any version field other than 0 or 1:

```rust
} else {
    panic!("version field must be 0 or 1, but it's {}", self.version);
}
``` [7](#0-6) 

`FullBlock::stream` returns `Err(Error::InvalidFullBlock)` for the same case — a second `stream`/`update_digest` divergence that can crash a node if a parsed `FullBlock` with an out-of-range version byte reaches any hashing code path.

### Impact Explanation

Any unprivileged peer can craft and broadcast an `UnfinishedBlock` or `FullBlock` containing a v2 `ProofOfSpace` with syntactically valid structure (one of pool_pk / contract_ph set, valid G1 point, arbitrary proof bytes). The message passes `parse` cleanly. The first call to `partial_hash()`, `header_hash()` via `RewardChainBlock`, or any other code that invokes `update_digest` on the embedded `ProofOfSpace` triggers an unconditional `panic!`, crashing the Rust process. Because the panic is deterministic and reproducible from the same bytes, every node that processes the message crashes identically — a consensus-halting, chain-stopping event.

This matches: **Critical — valid serialized network object triggers deterministic consensus divergence / chain halt.**

### Likelihood Explanation

- No privileged access, leaked keys, or governance role is required.
- The attacker only needs to construct a valid-looking `UnfinishedBlock` with a v2 `ProofOfSpace` carrying garbage proof bytes — a trivial byte-level operation.
- The crash fires the moment any node computes a hash of the block (e.g., for deduplication, logging, or partial validation) before completing proof-of-space verification.
- The v2 proof format is new (Chia 3.0), so the code path is not yet hardened by years of adversarial traffic.

### Recommendation

1. **Validate proof bytes in `parse`**: For version 1, call `quality_string_from_proof` (or an equivalent lightweight check) during `parse` and return `Err(Error::InvalidPoS)` if it fails. This makes the invariant "a successfully parsed v2 `ProofOfSpace` always has a valid quality string" hold at the boundary.

2. **Remove the `panic!` from `update_digest`**: Replace `.expect(...)` with a graceful fallback (e.g., hash a sentinel value or propagate an error). `update_digest` is called in contexts where panicking is never acceptable.

3. **Align `stream` and `update_digest` error handling**: Both methods should either succeed or fail together for the same input. The current asymmetry (stream succeeds, update_digest panics) violates the `Streamable` contract and makes the struct unsafe to use after parsing.

### Proof of Concept

```rust
// Craft a v2 ProofOfSpace with invalid proof bytes
let pos = ProofOfSpace::new(
    Bytes32::default(),
    Some(G1Element::default()),  // pool_public_key set → passes parse check
    None,                         // pool_contract_puzzle_hash not set
    G1Element::default(),
    1,                            // version = 1 (v2 proof)
    0, 0, 2,                      // plot_index, meta_group, strength
    0,                            // size (unused for v2)
    Bytes::from(vec![0xde, 0xad, 0xbe, 0xef]),  // invalid proof bytes
);

// parse succeeds — no proof validation in parse()
let buf = pos.to_bytes().unwrap();
let parsed = ProofOfSpace::parse::<false>(&mut Cursor::new(&buf)).unwrap();

// update_digest panics: quality_string() returns None for invalid proof bytes,
// .expect() fires unconditionally
let mut digest = Sha256::new();
parsed.update_digest(&mut digest);  // ← PANIC
```

impl Streamable for ProofOfSpace {
    fn update_digest(&self, digest: &mut Sha256) {
        self.challenge.update_digest(digest);
        self.pool_public_key.update_digest(digest);

        if self.version == 0 {
            self.pool_contract_puzzle_hash.update_digest(digest);
            self.plot_public_key.update_digest(digest);
            self.size.update_digest(digest);
            self.proof.update_digest(digest);
        } else if self.version == 1 {
            if let Some(pool_contract) = self.pool_contract_puzzle_hash {
                0b11_u8.update_digest(digest);
                pool_contract.update_digest(digest);
            } else {
                0b10_u8.update_digest(digest);
            }

            self.plot_public_key.update_digest(digest);
            self.plot_index.update_digest(digest);
            self.meta_group.update_digest(digest);
            self.strength.update_digest(digest);

            // for v2 proofs, we don't hash the full proof directly. The full
            // proof is the witness to this quality string commitment.
            self.quality_string()

                .update_digest(digest);
        } else {
            panic!("version field must be 0 or 1, but it's {}", self.version);
        }
    }
