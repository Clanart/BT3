### Title
`NextBaseFee` and `processBlock` Use Raw `computeGasUsed` Instead of On-Chain `BlockGasWanted`, Causing `eth_baseFee` / `eth_feeHistory` to Return Systematically Incorrect Base Fee - (File: `rpc/backend/chain_info.go`, `rpc/backend/utils.go`)

### Summary

The JSON-RPC `eth_baseFee` endpoint and `eth_feeHistory` compute the next base fee using raw `computeGasUsed` (sum of actual gas consumed by transactions) instead of the adjusted `BlockGasWanted` value that the on-chain `CalculateBaseFee` actually uses. Because the feemarket `EndBlock` applies a `MinGasMultiplier` floor before storing `BlockGasWanted`, the two values diverge whenever transactions use less gas than `MinGasMultiplier × gasLimit`. The RPC therefore returns a systematically lower base fee than the chain will actually set, causing wallets and users who rely on `eth_baseFee` to submit transactions with a `gasFeeCap` that is too low, resulting in rejected transactions.

### Finding Description

**On-chain base fee calculation** (`x/feemarket/keeper/eip1559.go`, `CalculateBaseFee`):

```go
parentGasUsed := k.GetBlockGasWanted(ctx)   // adjusted value from EndBlock
```

`GetBlockGasWanted` returns the value stored by `EndBlock` in `x/feemarket/keeper/abci.go`:

```go
limitedGasWanted := sdkmath.LegacyNewDec(gw).Mul(minGasMultiplier)
gasWanted = sdkmath.LegacyMaxDec(limitedGasWanted, sdkmath.LegacyNewDec(gasUsed)).TruncateInt().Uint64()
k.SetBlockGasWanted(ctx, gasWanted)
```

So on-chain: `parentGasUsed = max(gasWanted × MinGasMultiplier, gasUsed)`. [1](#0-0) 

**RPC `NextBaseFee`** (`rpc/backend/chain_info.go`):

```go
gasUsed, err := computeGasUsed(blockRes)   // raw sum of GasUsed from tx results
...
header := ethtypes.Header{GasUsed: gasUsed, ...}
nextBaseFee, err := CalcBaseFee(cfg, &header, feeParams.Params)
``` [2](#0-1) 

`computeGasUsed` simply sums `txsResult.GetGasUsed()` from block results — the raw actual gas consumed, with no `MinGasMultiplier` adjustment: [3](#0-2) 

The same pattern appears in `processBlock` used by `eth_feeHistory`: [4](#0-3) 

**Divergence scenario** (default `MinGasMultiplier = 0.5`):

| Quantity | On-chain | RPC |
|---|---|---|
| `gasWanted` (sum of gas limits) | 1,000,000 | — |
| `gasUsed` (actual) | 100,000 | 100,000 |
| `parentGasUsed` used for base fee | `max(500,000, 100,000) = 500,000` | `100,000` |

With `gasTarget = gasLimit / 2 = 500,000`:
- On-chain: `parentGasUsed == gasTarget` → base fee **unchanged**
- RPC: `parentGasUsed (100,000) < gasTarget (500,000)` → base fee **decreases**

The RPC reports a lower base fee than the chain will actually set. [5](#0-4) 

The `eth_baseFee` endpoint directly calls `NextBaseFee()`: [6](#0-5) 

### Impact Explanation

Any wallet or user calling `eth_baseFee` to set their EIP-1559 `gasFeeCap` receives a value lower than the actual on-chain base fee. Transactions submitted with `gasFeeCap < actualBaseFee` are rejected by the ante handler:

```go
if baseFee != nil && tx.GasFeeCap().Cmp(baseFee) < 0 {
    return nil, errorsmod.Wrapf(errortypes.ErrInsufficientFee, ...)
}
``` [7](#0-6) 

This is a **High** impact: the public JSON-RPC path feeds incorrect consensus-critical data (the base fee) into transaction execution, causing valid user transactions to be rejected. The discrepancy is structural and permanent whenever `MinGasMultiplier > 0` and blocks are not fully utilized.

### Likelihood Explanation

**High likelihood.** The default `MinGasMultiplier` is `0.5` (50%): [8](#0-7) 

Any block where transactions use less than `MinGasMultiplier × gasLimit` gas triggers the divergence. This is the common case for most real-world blocks (e.g., simple ETH transfers with `gasLimit=21000` and `gasUsed=21000` but submitted with a higher limit, or contract calls that use a fraction of their limit). An unprivileged attacker can deliberately submit transactions with inflated gas limits and minimal execution to maximize the discrepancy, causing other users' transactions to fail.

### Recommendation

In `NextBaseFee()` and `processBlock()`, replace `computeGasUsed(blockRes)` with a query to the feemarket module's `BlockGas` gRPC endpoint (which returns the stored `BlockGasWanted`) to match the value used by `CalculateBaseFee` on-chain:

```go
// Instead of:
gasUsed, err := computeGasUsed(blockRes)

// Use:
blockGasRes, err := b.queryClient.FeeMarket.BlockGas(
    rpctypes.ContextWithHeight(blockHeight),
    &feemarkettypes.QueryBlockGasRequest{},
)
gasUsed = uint64(blockGasRes.Gas)
```

This ensures the RPC-reported base fee matches the value the chain will actually compute in `BeginBlock`. [9](#0-8) 

### Proof of Concept

1. Deploy an Ethermint chain with default params (`MinGasMultiplier = 0.5`, `ElasticityMultiplier = 2`).
2. Submit a block containing one transaction with `gasLimit = 1,000,000` that actually uses only `gasUsed = 21,000` (e.g., a simple ETH transfer with an inflated gas limit).
3. Call `eth_baseFee` (→ `NextBaseFee()`):
   - `computeGasUsed = 21,000`
   - `gasTarget = blockGasLimit / 2`
   - RPC computes base fee as if `parentGasUsed = 21,000` → base fee decreases significantly
4. On-chain `BeginBlock` → `CalculateBaseFee`:
   - `BlockGasWanted = max(1,000,000 × 0.5, 21,000) = 500,000`
   - If `gasTarget = 500,000`, base fee is **unchanged**
5. A user who sets `gasFeeCap = eth_baseFee` (the decreased RPC value) submits a transaction. The ante handler rejects it with `ErrInsufficientFee` because the actual base fee is higher than what the RPC reported.

func (k *Keeper) EndBlock(ctx sdk.Context) error {
	gasWanted := ctx.BlockGasWanted()
	gw, err := ethermint.SafeInt64(gasWanted)
	if err != nil {
		return err
	}
	gasUsed, err := ethermint.SafeInt64(ctx.BlockGasUsed())
	if err != nil {
		return err

	// to prevent BaseFee manipulation we limit the gasWanted so that
	// gasWanted = max(gasWanted * MinGasMultiplier, gasUsed)

	// more info here https://github.com/evmos/ethermint/pull/1105#discussion_r888798925
	minGasMultiplier := k.GetParams(ctx).MinGasMultiplier
	limitedGasWanted := sdkmath.LegacyNewDec(gw).Mul(minGasMultiplier)

func computeGasUsed(blockRes *tmrpctypes.ResultBlockResults) (uint64, error) {
	var gasUsed uint64
	for _, txsResult := range blockRes.TxsResults {
		if ShouldIgnoreGasUsed(txsResult) {
			// block gas limit has exceeded, other txs must have failed with same reason.
			break
		}
		gas, err := ethermint.SafeUint64(txsResult.GetGasUsed())
		if err != nil {
			return 0, err
		}
		gasUsed += gas
	}
	return gasUsed, nil
}

```go
	parentGasUsed := k.GetBlockGasWanted(ctx)

	// NOTE: a MaxGas equal to -1 means that block gas is unlimited
	if consParams.Block == nil || consParams.Block.MaxGas <= -1 {
		panic(fmt.Sprintf("get invalid consensus params: %s", consParams))
	}
	gasLimit := big.NewInt(consParams.Block.MaxGas)
	// CONTRACT: ElasticityMultiplier cannot be 0 as it's checked in the params
	// validation
	parentGasTargetBig := new(big.Int).Div(gasLimit, new(big.Int).SetUint64(uint64(params.ElasticityMultiplier)))
	if !parentGasTargetBig.IsUint64() {
		return nil
	}

	parentGasTarget := parentGasTargetBig.Uint64()
	baseFeeChangeDenominator := new(big.Int).SetUint64(uint64(params.BaseFeeChangeDenominator))

	// If the parent gasUsed is the same as the target, the baseFee remains
	// unchanged.
	if parentGasUsed == parentGasTarget {
		return new(big.Int).Set(parentBaseFee)
	}

	if parentGasUsed > parentGasTarget {
		// If the parent block used more gas than its target, the baseFee should
		// increase.
		gasUsedDelta := new(big.Int).SetUint64(parentGasUsed - parentGasTarget)
		x := new(big.Int).Mul(parentBaseFee, gasUsedDelta)
		y := x.Div(x, parentGasTargetBig)
		baseFeeDelta := ethermint.BigMax(
			x.Div(y, baseFeeChangeDenominator),
			common.Big1,
		)

		return x.Add(parentBaseFee, baseFeeDelta)
	}

	// Otherwise if the parent block used less gas than its target, the baseFee
	// should decrease.
	gasUsedDelta := new(big.Int).SetUint64(parentGasTarget - parentGasUsed)
	x := new(big.Int).Mul(parentBaseFee, gasUsedDelta)
	y := x.Div(x, parentGasTargetBig)
	baseFeeDelta := x.Div(y, baseFeeChangeDenominator)

	// Set global min gas price as lower bound of the base fee, transactions below
	// the min gas price don't even reach the mempool.
	minGasPrice := params.MinGasPrice.TruncateInt().BigInt()
	return ethermint.BigMax(x.Sub(parentBaseFee, baseFeeDelta), minGasPrice)

func (e *PublicAPI) BaseFee() (*hexutil.Big, error) {

	baseFee, err := e.backend.NextBaseFee()
	if err != nil {
		return nil, err
	}

		return (*hexutil.Big)(new(big.Int)), nil

// BlockGas implements the Query/BlockGas gRPC method
func (k Keeper) BlockGas(c context.Context, _ *types.QueryBlockGasRequest) (*types.QueryBlockGasResponse, error) {
	ctx := sdk.UnwrapSDKContext(c)
	gas, err := ethermint.SafeInt64(k.GetBlockGasWanted(ctx))

		return nil, err
	}
	return &types.QueryBlockGasResponse{
