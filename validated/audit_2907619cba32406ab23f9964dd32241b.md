### Title
`starknet_estimateFee` / `starknet_simulateTransactions` Return `overall_fee` That Cannot Be Reproduced From Reported Components Due to Missing `l2_gas_consumed` - (File: crates/apollo_rpc_execution/src/objects.rs)

### Summary

The `FeeEstimation` struct returned by `starknet_estimateFee` and `starknet_simulateTransactions` omits `l2_gas_consumed` while including `l2_gas_price` and an `overall_fee` that already incorporates L2 gas costs. The OpenRPC spec explicitly states `overall_fee = gas_consumed * gas_price + data_gas_consumed * data_gas_price`, but this formula is false for any V3 transaction that consumes L2 gas. The RPC endpoint returns an authoritative-looking wrong value: the `overall_fee` is higher than what the reported components can explain, and users cannot derive or verify the correct fee from the response.

### Finding Description

In `tx_execution_output_to_fee_estimation`, the `FeeEstimation` response is assembled from the execution receipt:

```rust
Ok(FeeEstimation {
    gas_consumed: gas_vector.l1_gas.0.into(),
    l1_gas_price,
    data_gas_consumed: gas_vector.l1_data_gas.0.into(),
    l1_data_gas_price,
    l2_gas_price,
    overall_fee: tx_execution_output.execution_info.receipt.fee,
    unit: tx_execution_output.price_unit,
})
```

`gas_vector.l2_gas` is silently dropped. The `overall_fee` is taken directly from `receipt.fee`, which the blockifier computes as `l1_gas * l1_gas_price + l1_data_gas * l1_data_gas_price + l2_gas * l2_gas_price`. The L2 gas component is included in `overall_fee` but is not exposed in any response field.

The code itself acknowledges this with a TODO:

```rust
// TODO(Tzahi): Add l2_gas_consumed. Verify overall_fee estimation of l1_gas_price only is
// close enough (as there are roundings) to the fee of both l1_gas_price and l2_gas_price.
```

The OpenRPC schema at `crates/apollo_rpc/resources/V0_8/starknet_api_openrpc.json` describes `overall_fee` as:

> "The estimated fee for the transaction (in wei or fri, depending on the tx version), **equals to gas_consumed\*gas_price + data_gas_consumed\*data_gas_price**"

This description is false for any V3 transaction with non-zero L2 gas. The transaction prover's own RPC records confirm that real transactions carry significant `l2_gas_consumed` (e.g., `0xb56b6`) and `l2_gas_price` (e.g., `0x1dcd65000`), making the L2 gas component a material fraction of `overall_fee`.

### Impact Explanation

Any caller of `starknet_estimateFee` or `starknet_simulateTransactions` who uses the returned components to set `AllResources` resource bounds for a V3 transaction will observe:

1. `gas_consumed` (L1 gas) and `data_gas_consumed` (L1 data gas) are reported.
2. `l2_gas_consumed` is absent â€” the caller has no on-chain-verifiable way to determine the required L2 gas bound.
3. `overall_fee` is higher than `gas_consumed * l1_gas_price + data_gas_consumed * l1_data_gas_price` by exactly `l2_gas * l2_gas_price`, which is unaccounted for.

A user who sets their L2 gas resource bound to zero or to an under-estimated value based on the incomplete response will have their transaction reverted with `Insufficient max L2Gas`. Conversely, a user who inflates their L2 gas bound to compensate for the missing information may overpay. The RPC endpoint presents an internally inconsistent, authoritative-looking result that cannot be used to correctly construct a V3 transaction.

This matches the allowed impact: **High â€” RPC fee estimation returns an authoritative-looking wrong value.**

### Likelihood Explanation

Every V3 transaction (`TransactionVersion::THREE`) that executes Sierra code consumes L2 gas. The `AllResources` resource bounds variant, which requires an explicit L2 gas bound, is the standard path for V3 transactions. Any wallet, SDK, or dApp that calls `starknet_estimateFee` to populate resource bounds for a V3 transaction is affected. The trigger requires no privilege â€” any user submitting a V3 transaction is exposed.

### Recommendation

Add `l2_gas_consumed` to the `FeeEstimation` struct and populate it from `gas_vector.l2_gas`:

```rust
pub struct FeeEstimation {
    pub gas_consumed: Felt,
    pub l1_gas_price: GasPrice,
    pub data_gas_consumed: Felt,
    pub l1_data_gas_price: GasPrice,
    pub l2_gas_consumed: Felt,   // ADD THIS
    pub l2_gas_price: GasPrice,
    pub overall_fee: Fee,
    pub unit: PriceUnit,
}
```

In `tx_execution_output_to_fee_estimation`:

```rust
Ok(FeeEstimation {
    gas_consumed: gas_vector.l1_gas.0.into(),
    l1_gas_price,
    data_gas_consumed: gas_vector.l1_data_gas.0.into(),
    l1_data_gas_price,
    l2_gas_consumed: gas_vector.l2_gas.0.into(),   // ADD THIS
    l2_gas_price,
    overall_fee: tx_execution_output.execution_info.receipt.fee,
    unit: tx_execution_output.price_unit,
})
```

Update the OpenRPC schema description for `overall_fee` to reflect the correct formula including L2 gas.

### Proof of Concept

1. Submit a V3 invoke transaction to `starknet_estimateFee`.
2. Observe the response: `gas_consumed`, `l1_gas_price`, `data_gas_consumed`, `l1_data_gas_price`, `l2_gas_price`, `overall_fee`.
3. Compute `derived_fee = gas_consumed * l1_gas_price + data_gas_consumed * l1_data_gas_price`.
4. Compare `derived_fee` with `overall_fee`. For any transaction with non-zero L2 gas, `overall_fee > derived_fee` by `l2_gas * l2_gas_price`.
5. The difference is the L2 gas cost, which is included in `overall_fee` but not in any reported component.

```rust
pub(crate) fn tx_execution_output_to_fee_estimation(
    tx_execution_output: &TransactionExecutionOutput,
    block_context: &BlockContext,
) -> ExecutionResult<FeeEstimation> {
    let gas_prices = &block_context.block_info().gas_prices;
    let (l1_gas_price, l1_data_gas_price, l2_gas_price) = (
        gas_prices.l1_gas_price(&tx_execution_output.price_unit.into()).get(),
        gas_prices.l1_data_gas_price(&tx_execution_output.price_unit.into()).get(),
        gas_prices.l2_gas_price(&tx_execution_output.price_unit.into()).get(),
    );

    let gas_vector = tx_execution_output.execution_info.receipt.gas;

    Ok(FeeEstimation {
        gas_consumed: gas_vector.l1_gas.0.into(),
        l1_gas_price,
        data_gas_consumed: gas_vector.l1_data_gas.0.into(),
        l1_data_gas_price,
        l2_gas_price,
        overall_fee: tx_execution_output.execution_info.receipt.fee,
        unit: tx_execution_output.price_unit,
    })
}
```
