### Title
P2P Sync State Diff Accepted on Length Match Alone Without Cryptographic Commitment Verification — (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

---

### Summary

The P2P sync client assembles a `ThinStateDiff` from peer-supplied chunks and accepts it as valid once the accumulated entry count equals `header.state_diff_length`. It never verifies the assembled diff's Poseidon hash against `header.state_diff_commitment`. A malicious peer can therefore supply a state diff with the correct number of entries but arbitrary storage values, nonces, or class hashes, causing the node to persist a wrong state diff, compute a wrong global state root, and return wrong authoritative values from RPC.

---

### Finding Description

`StateDiffStreamBuilder::parse_data_for_block` in `crates/apollo_p2p_sync/src/client/state_diff.rs` reads `target_state_diff_len` from the stored block header and loops until `current_state_diff_len == target_state_diff_len`: [1](#0-0) 

After the loop it performs exactly two checks: the accumulated length equals the target, and deprecated declared classes are non-conflicting: [2](#0-1) 

The block header also carries `state_diff_commitment: Option<StateDiffCommitment>`, the Poseidon hash of the canonical state diff: [3](#0-2) 

`parse_data_for_block` never reads `state_diff_commitment` and never calls `calculate_state_diff_hash` on the assembled result. The assembled diff is then written directly to storage: [4](#0-3) 

`append_state_diff` writes every field of the diff (deployed contracts, storage diffs, nonces, compiled class hashes) into the MDBX tables without any hash check: [5](#0-4) 

The analog to the external report is exact: just as `L1NativeTokenVault._bridgeBurnNativeToken` returned `depositChecked = true` without verifying the post-transfer balance, `parse_data_for_block` returns `Ok(Some((result, block_number)))` without verifying `calculate_state_diff_hash(&result) == header.state_diff_commitment`. In both cases an assumed-correct quantity (token amount / state diff content) is accepted on a weaker proxy check (transfer did not revert / entry count matches) rather than a post-operation equality check.

The central sync path carries the same unimplemented check, acknowledged by a developer TODO: [6](#0-5) 

The committer does have an optional `verify_state_diff_hash` flag, but that flag guards the committer's own internal consistency check between the `state_diff` and `state_diff_commitment` fields passed to it by the batcher — it does not protect the P2P sync ingestion path: [7](#0-6) 

---

### Impact Explanation

Once the wrong `ThinStateDiff` is committed to storage:

1. **Wrong storage values from RPC** — `StateStorageReader::get_storage_at` reads directly from the MDBX tables populated by `append_state_diff`. `starknet_getStorageAt` returns the attacker-chosen value as authoritative.
2. **Wrong global state root** — The batcher's `CommitmentManager` sends the stored state diff to the Patricia-trie committer, which computes a `GlobalRoot` from the wrong leaves. This wrong root is chained into `calculate_block_hash` and stored as the canonical state root.
3. **Wrong state diff propagated to other peers** — The P2P sync server reads the stored state diff and serves it to downstream peers, spreading the corruption.
4. **Wrong proof inputs** — The transaction prover's `create_commitment_infos_with_state_changes` builds Patricia-trie commitment infos from the stored state diff; a wrong diff produces wrong `previous_root`/`updated_root` pairs fed to the SNOS. [8](#0-7) [9](#0-8) 

---

### Likelihood Explanation

Any node participating in the P2P network can act as a sync server. The attacker only needs to:

1. Connect to the victim node as a peer.
2. Respond to a `StateDiff` query for a target block with chunks whose total `len()` equals `header.state_diff_length` but whose storage values, nonces, or class hashes differ from the canonical ones.

No privileged access, no cryptographic key, and no consensus participation is required. The `state_diff_length` value is public (transmitted in the signed block header). Crafting a replacement diff of the same length is trivial.

---

### Recommendation

After the `while` loop in `parse_data_for_block`, before returning `Ok(Some(...))`, compute and compare the Poseidon commitment:

```rust
if let Some(expected_commitment) = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .and_then(|h| h.state_diff_commitment)
{
    let actual_commitment = calculate_state_diff_hash(&result);
    if actual_commitment != expected_commitment {
        return Err(ParseDataError::BadPeer(
            BadPeerError::StateDiffHashMismatch {
                block_number,
                expected: expected_commitment,
                actual: actual_commitment,
            },
        ));
    }
}
```

Apply the same check in `GenericStateSync::store_state_diff` in `crates/apollo_central_sync/src/lib.rs` to resolve the existing TODO. [2](#0-1) [10](#0-9)

### Proof of Concept

1. Run a victim node syncing via P2P.
2. Operate a malicious peer that, when queried for the state diff of block `N`, responds with `StateDiffChunk` entries whose total `len()` equals `header.state_diff_length` for block `N` but whose storage values are all `Felt::ONE` instead of the canonical values.
3. Observe that `parse_data_for_block` returns `Ok(Some((tampered_diff, N)))`.
4. Observe that `write_to_storage` calls `append_state_diff(N, tampered_diff)` without error.
5. Query `starknet_getStorageAt` for any storage slot updated in block `N`; the response is `0x1` instead of the canonical value.
6. Observe that the committer computes a `GlobalRoot` that differs from the canonical root for block `N`, causing the block hash stored for block `N+1` to diverge from the network's canonical hash. [11](#0-10) [12](#0-11)

    #[latency_histogram("p2p_sync_state_diff_parse_data_for_block_latency_seconds", true)]
    fn parse_data_for_block<'a>(
        state_diff_chunks_response_manager: &'a mut ClientResponsesManager<
            DataOrFin<StateDiffChunk>,
        >,
        block_number: BlockNumber,
        storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            let mut result = ThinStateDiff::default();
            let mut prev_result_len = 0;
            let mut current_state_diff_len = 0;
            let target_state_diff_len = storage_reader
                .begin_ro_txn()?
                .get_block_header(block_number)?
                .expect("A header with number lower than the header marker is missing")
                .state_diff_length
                .ok_or(P2pSyncClientError::OldHeaderInStorage {
                    block_number,
                    missing_field: "state_diff_length",
                })?;

            while current_state_diff_len < target_state_diff_len {
                let maybe_state_diff_chunk = state_diff_chunks_response_manager
                    .next()
                    .await
                    .ok_or(ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }))?;
                let Some(state_diff_chunk) = maybe_state_diff_chunk?.0 else {
                    if current_state_diff_len == 0 {
                        return Ok(None);
                    } else {
                        return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                            expected_length: target_state_diff_len,
                            possible_lengths: vec![current_state_diff_len],
                        }));
                    }
                };
                prev_result_len = current_state_diff_len;
                if state_diff_chunk.is_empty() {
                    return Err(ParseDataError::BadPeer(BadPeerError::EmptyStateDiffPart));
                }
                // It's cheaper to calculate the length of `state_diff_part` than the length of
                // `result`.
                current_state_diff_len += state_diff_chunk.len();
                unite_state_diffs(&mut result, state_diff_chunk)?;
            }

            if current_state_diff_len != target_state_diff_len {
                return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                    expected_length: target_state_diff_len,
                    possible_lengths: vec![prev_result_len, current_state_diff_len],
                }));
            }

            validate_deprecated_declared_classes_non_conflicting(&result)?;
            Ok(Some((result, block_number)))
        }
        .boxed()
    }

        self,

        thin_state_diff: ThinStateDiff,

        let markers_table = self.open_table(&self.tables.markers)?;
        let state_diffs_table = self.open_table(&self.tables.state_diffs)?;
        let nonces_table = self.open_table(&self.tables.nonces)?;
        let deployed_contracts_table = self.open_table(&self.tables.deployed_contracts)?;
        let storage_table = self.open_table(&self.tables.contract_storage)?;
        let declared_classes_block_table = self.open_table(&self.tables.declared_classes_block)?;
        let deprecated_declared_classes_block_table =

        let compiled_class_hash_table = self.open_table(&self.tables.compiled_class_hash)?;

        // Write state.
        write_deployed_contracts(

            block_number,
            &deployed_contracts_table,
            &nonces_table,

            &thin_state_diff.storage_diffs,
            &self.txn,
            block_number,
            &storage_table,
        )?;

        write_nonces(&thin_state_diff.nonces, &self.txn, block_number, &nonces_table)?;

        for (class_hash, _) in &thin_state_diff.class_hash_to_compiled_class_hash {

            &self.txn,
            block_number,

        update_marker_to_next_block(&self.txn, &markers_table, MarkerKind::State, block_number)?;

        advance_compiled_class_marker_over_blocks_without_classes(
            &self.txn,
            &markers_table,
            &state_diffs_table,
            &self.file_handlers,

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
