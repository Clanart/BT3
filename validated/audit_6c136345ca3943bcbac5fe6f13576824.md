### Title
Missing Class Hash Existence Validation in `execute_replace_class` Allows Setting Undeclared Class Hash, Causing Proof Failure and Network Halt - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS Cairo program accepts an arbitrary user-supplied `class_hash` without validating that the hash corresponds to a declared (existing) contract class. This is directly analogous to the external report's pattern of accepting a user-supplied ID for a non-existent entity. An unprivileged contract can call `replace_class` with a class hash that has never been declared, writing an invalid class hash into the contract state. When the contract is subsequently called (in the same or a future block), the OS prover cannot locate a compiled class for the non-existent hash, causing proof generation to fail and preventing the block from being confirmed — a total network halt.

---

### Finding Description

In `syscall_impls.cairo`, the function `execute_replace_class` processes the `replace_class` syscall:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The `class_hash` value comes directly from `request.class_hash`, which is attacker-controlled calldata. There are **two missing validations**:

1. No check that `class_hash` is present in `contract_class_changes` (declared in the current block).
2. No check that `class_hash` is present in the `compiled_class_facts` bundle (declared in any prior block).

The developer-acknowledged TODO comment at line 898 explicitly confirms this is a known missing check. [1](#0-0) 

By contrast, `execute_declare_transaction` correctly enforces that a class can only be declared once by asserting `prev_value=0` and `assert_not_zero(compiled_class_hash)`, but `execute_replace_class` performs no equivalent existence check on the incoming hash. [2](#0-1) 

The OS global context assembles the compiled class facts bundle at startup via `guess_compiled_class_facts()`, and `validate_compiled_class_facts_post_execution` validates that every class hash used during execution is present in the bundle. If a contract's class hash is set to an undeclared value and that contract is subsequently called, the prover cannot supply a valid compiled class entry, making the proof unsound. [3](#0-2) 

---

### Impact Explanation

**High — Network not being able to confirm new transactions (total network shutdown).**

When a contract's class hash is replaced with a non-existent hash and that contract is subsequently called (either in the same block or a future block), the StarkNet OS prover cannot construct a valid STARK proof for the block. The block cannot be confirmed on L1. Because the poisoned state persists across blocks (the invalid class hash is committed to the state trie), every subsequent block that touches the affected contract will also fail to prove. This constitutes a persistent, total network halt for all transactions that interact with the affected contract, and can be extended to a global halt if the attacker targets a widely-used contract or the fee token.

---

### Likelihood Explanation

**High.** The `replace_class` syscall is callable by any deployed contract without any privileged role. An attacker needs only to:
1. Deploy a contract (standard, permissionless operation).
2. Call `replace_class` with an arbitrary felt value as the class hash.

No leaked keys, operator collusion, or Sybil attack is required. The OS Cairo code — the authoritative source for what the prover accepts — performs no validation, and the developer TODO confirms the check is absent. The sequencer's blockifier may or may not independently validate class existence; if it does not, the malicious transaction is included in a block and the prover fails. Even if the blockifier rejects it, the OS-level missing check remains a protocol-layer vulnerability.

---

### Recommendation

Add an existence check inside `execute_replace_class` before updating the contract state. The check should verify that the supplied `class_hash` is present in either `contract_class_changes` (declared in the current block) or the `compiled_class_facts` bundle (declared in a prior block). Concretely, resolve the existing TODO:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
```

by adding a lookup into `contract_class_changes` (analogous to how `execute_declare_transaction` uses `dict_update` with `prev_value=0` to enforce uniqueness) or by verifying membership in the compiled class facts bundle before writing the new state entry. Additionally, add `assert_not_zero(class_hash)` to prevent replacement with the zero hash.

---

### Proof of Concept

1. **Setup**: Attacker deploys contract `A` (class hash `H_A`, which is valid and declared).
2. **Attack transaction (Block N)**: Attacker's contract `A` calls the `replace_class` syscall with `class_hash = 0xDEAD` (an arbitrary felt that has never been declared).
3. **OS processing**: `execute_replace_class` in `syscall_impls.cairo` reads `request.class_hash = 0xDEAD`, skips any existence check (the TODO line), and writes `StateEntry(class_hash=0xDEAD, ...)` into `contract_state_changes` for contract `A`.
4. **State commitment**: `state_update` in `state.cairo` squashes and commits the state diff, including the poisoned class hash `0xDEAD` for contract `A`, into the global state root.
5. **Trigger (Block N or N+1)**: Any user calls contract `A`. The OS attempts to execute contract `A` using class hash `0xDEAD`.
6. **Proof failure**: `validate_compiled_class_facts_post_execution` cannot find a compiled class entry for `0xDEAD` in the `compiled_class_facts` bundle. The STARK proof is invalid.
7. **Network halt**: The block containing the call to contract `A` cannot be proven or confirmed on L1. All subsequent blocks touching contract `A` also fail. The network cannot confirm new transactions. [4](#0-3) [5](#0-4) [3](#0-2)

        // Not enough gas to execute the syscall.
        return ();
