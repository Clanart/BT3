### Title
`ProofOfInclusion::valid()` Never Anchors to an External Tree Root — Forged Inclusion Proofs Always Pass - (File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs)

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate contains a tautological final check that is always `true`, meaning the method only validates internal hash-chain consistency but never verifies the proof's claimed root against the actual committed Merkle tree root. An attacker can construct a `ProofOfInclusion` for any arbitrary `node_hash` (any key-value pair) against any tree, and `valid()` will return `true`.

### Finding Description

`ProofOfInclusion::valid()` is implemented as follows:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()   // ← always true
}
``` [1](#0-0) 

The loop invariant guarantees that after each iteration, `existing_hash` is set to `calculated_hash`, which was just verified to equal `layer.combined_hash`. After the final iteration, `existing_hash` equals the last `layer.combined_hash`.

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← same value existing_hash was just set to
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

So `existing_hash == self.root_hash()` is a **tautology** — it is always `true` regardless of what the proof contains. The method never compares the proof's claimed root against any externally-committed tree root.

An attacker can forge a `ProofOfInclusion` for any `node_hash` by:
1. Choosing an arbitrary `node_hash` (the leaf they want to falsely prove is included).
2. Choosing arbitrary `other_hash` values for each layer.
3. Computing `combined_hash = calculate_internal_hash(existing_hash, other_hash_side, other_hash)` for each layer — making the per-layer check pass trivially.
4. The resulting proof passes `valid()` with any `node_hash` and any sibling hashes, regardless of the actual tree state.

`ProofOfInclusion` is a `Streamable` type exposed via Python bindings, meaning it can be deserialized from untrusted bytes via `from_bytes()` and then validated with `valid()`. [3](#0-2) 

The Python binding exposes `valid()` and `root_hash()` as separate methods, with no combined "validate against this root" API: [4](#0-3) 

The fuzz target for DataLayer proofs only calls `proof.valid()` without any external root check, confirming that `valid()` is treated as a complete validation: [5](#0-4) 

### Impact Explanation

Any verifier that calls `proof.valid()` without separately checking `proof.root_hash() == actual_committed_root` will accept forged inclusion proofs. Since the DataLayer uses on-chain Merkle roots (stored in DataLayer singletons) as commitments to off-chain key-value state, a forged proof of inclusion lets an attacker prove that an arbitrary key-value pair exists in a DataLayer store when it does not. This matches the allowed impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion and lets untrusted input prove invalid state.**

### Likelihood Explanation

`ProofOfInclusion` is a `Streamable` type deserializable from untrusted bytes. The `valid()` method is the sole public validation API and is misleadingly named as a complete check. Any Python or Rust consumer that deserializes a `ProofOfInclusion` from network input and calls only `valid()` — the natural and documented usage — is vulnerable. The fuzz harness itself demonstrates this pattern.

### Recommendation

`valid()` must accept an external root hash and compare the proof's computed root against it:

```rust
pub fn valid_for_root(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root   // anchor to external committed root
}
```

The existing `valid()` method (or the Python `valid()` binding) should be removed or deprecated, and all call sites updated to pass the actual committed tree root.

### Proof of Concept

```rust
use chia_datalayer::{Hash, MerkleBlob, KeyId, ValueId, InsertLocation};
use chia_datalayer::merkle::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};
use chia_datalayer::Side;

// Build a real tree with one entry
let mut blob = MerkleBlob::new(Vec::new()).unwrap();
let real_key = KeyId(1);
let real_hash = Hash([0xAA; 32]);
blob.insert(real_key, ValueId(1), &real_hash, InsertLocation::Auto {}).unwrap();
blob.calculate_lazy_hashes().unwrap();
let actual_root = blob.get_root().unwrap();

// Forge a proof claiming a completely different node_hash is included
let fake_node_hash = Hash([0xFF; 32]);  // not in the tree
let fake_other_hash = Hash([0x11; 32]);
// compute combined_hash so the per-layer check passes
let combined = chia_datalayer::calculate_internal_hash(&fake_node_hash, Side::Right, &fake_other_hash);

let forged_proof = ProofOfInclusion {
    node_hash: fake_node_hash,
    layers: vec![ProofOfInclusionLayer {
        other_hash_side: Side::Right,
        other_hash: fake_other_hash,
        combined_hash: combined,
    }],
};

// valid() returns true even though fake_node_hash is not in the tree
assert!(forged_proof.valid());  // passes — tautological final check
// The forged proof's root_hash() == combined, which != actual_root
// but valid() never checks this
```
