### Title
Missing Shard-ID Binding in `set_state_header` Allows Cross-Shard State Substitution — (`File: chain/chain/src/state_sync/adapter.rs`)

### Summary

`set_state_header` validates that the supplied `ShardStateSyncResponseHeader` contains a chunk that is cryptographically included in the sync block's merkle tree, but **never verifies that the chunk belongs to the requested `shard_id`**. Because the block's `chunk_headers_root` covers all shards, a valid merkle proof can be produced for any shard's chunk. A malicious peer can therefore supply a chunk from shard N as the state header for shard M, causing the syncing node to download and apply shard N's trie data into shard M's storage, permanently corrupting that shard's state.

---

### Finding Description

`set_state_header` in `chain/chain/src/state_sync/adapter.rs` accepts a `shard_id` parameter and a `ShardStateSyncResponseHeader` from a remote peer. It performs five validation steps:

1. **`validate_chunk_proofs`** — verifies internal chunk hash/tx-root/receipts-root consistency. Does **not** inspect `chunk.shard_id()`.
2. **`verify_path` against `chunk_headers_root`** — proves the chunk is a leaf in the block's merkle tree. The leaf is `ChunkHashHeight(chunk.chunk_hash(), chunk.height_included())`. The merkle tree covers **all shards**; the proof is valid for any shard's chunk.
3. **`verify_path` for `prev_chunk_header`** — same structure, same gap.
4. **Receipt proof chain** — hashes receipts as `ReceiptList(shard_id, receipts)` using the *requested* `shard_id`, then verifies merkle paths. An attacker can supply the legitimate shard M receipts (observable on-chain) to satisfy this check independently of which chunk is embedded.
5. **`validate_state_root_node`** — checks that `state_root_node.hash == chunk.prev_state_root()`. Passes as long as the node matches the embedded chunk's state root, regardless of which shard that root belongs to.

**The missing check** — nowhere in the function is there a comparison:

```rust
// ABSENT:
if chunk.shard_id() != shard_id { return Err(...); }
if prev_chunk_header.shard_id() != shard_id { return Err(...); }
``` [1](#0-0) 

`validate_chunk_proofs` confirms this: it checks hash, tx-root, and receipts-root but never reads `shard_id` from the chunk. [2](#0-1) 

After passing all checks, the header is persisted under `StateHeaderKey(shard_id, sync_hash)` — keyed to the *requested* shard — but its embedded chunk and `prev_state_root` belong to a different shard. [3](#0-2) 

---

### Impact Explanation

Once the poisoned header is stored, the downstream pipeline uses it without re-checking shard identity:

- **`set_state_part`** reads the header, extracts `chunk.prev_state_root()` (shard N's root), and validates each downloaded part against that root. Parts for shard N's trie pass; they are written to `DBCol::StateParts` under `StatePartKey(sync_hash, shard_M_id, part_id)`. [4](#0-3) 

- **`apply_state_part`** writes those trie nodes into shard M's storage (`shard_uid` derived from the requested `shard_id`), overwriting shard M's trie with shard N's data. [5](#0-4) 

- **`set_state_finalize`** applies shard N's chunk (transactions, receipts) against shard M's `shard_uid`, producing a `ChunkExtra` whose `state_root` is shard N's post-state root. The node's recorded state root for shard M is now permanently wrong. [6](#0-5) 

The syncing node subsequently produces chunks for shard M with an invalid `prev_state_root`, which all honest validators reject. The node is effectively DoS'd out of chunk production for that shard and will be kicked from the validator set.

---

### Likelihood Explanation

State sync headers are requested from any peer that advertises the data. The `StateSyncDownloader` downloads from a `StateSyncDownloadSource` and forwards the header to `set_state_header` for validation. [7](#0-6) 

Any unprivileged network peer can respond to a `StateRequestHeader` message. The attacker needs only:
1. A real chunk from shard N in the target block (publicly observable).
2. Its merkle proof from the block (publicly computable).
3. The legitimate incoming receipts for shard M (publicly observable).

No validator key, no stake, no privileged role is required.

---

### Recommendation

Add an explicit shard-ID binding check immediately after extracting the chunk, before any other validation:

```rust
pub fn set_state_header(
    &self,
    shard_id: ShardId,
    sync_hash: CryptoHash,
    shard_state_header: ShardStateSyncResponseHeader,
) -> Result<(), Error> {
    let sync_block_header = self.chain_store.get_block_header(&sync_hash)?;
    let chunk = shard_state_header.cloned_chunk();
    let prev_chunk_header = shard_state_header.cloned_prev_chunk_header();

    // NEW: bind chunk to the requested shard
    if chunk.shard_id() != shard_id {
        byzantine_assert!(false);
        return Err(Error::Other(
            "set_shard_state failed: chunk shard_id does not match requested shard_id".into(),
        ));
    }
    if let Some(ref pch) = prev_chunk_header {
        if pch.shard_id() != shard_id {
            byzantine_assert!(false);
            return Err(Error::Other(
                "set_shard_state failed: prev_chunk shard_id does not match requested shard_id".into(),
            ));
        }
    }
    // ... existing checks follow
```

This mirrors the canonical-binding fix recommended in the external report: instead of relying on partial field checks (chunk inclusion in block), enforce that the chunk is the canonical one for the requested shard.

### Proof of Concept

**Setup:** A 2-shard network. Honest node V is state-syncing shard 0 at `sync_hash`.

1. **Attacker A** (any peer) observes the block at `sync_hash - 1`. It contains chunks for shard 0 and shard 1.

2. A constructs a `ShardStateSyncResponseHeaderV2` where:
   - `chunk` = shard 1's `ShardChunk` from that block
   - `chunk_proof` = shard 1's merkle path in `chunk_headers_root` (valid, computable from the block)
   - `prev_chunk_header` = shard 0's actual prev chunk header (correct `height_included` for shard 0)
   - `prev_chunk_proof` = shard 0's prev chunk merkle path (valid)
   - `incoming_receipts_proofs` = legitimate receipts sent to shard 0 (observable on-chain)
   - `root_proofs` = valid merkle paths for those receipts (observable on-chain)
   - `state_root_node` = shard 1's state root node (hash matches shard 1's `prev_state_root`)

3. A sends this as the response to V's `StateRequestHeader { shard_id: 0, sync_hash }`.

4. V calls `set_state_header(shard_id=0, sync_hash, crafted_header)`:
   - Step 1 (`validate_chunk_proofs`): passes — shard 1's chunk is internally consistent.
   - Step 3a (`verify_path` on `chunk_headers_root`): passes — shard 1's chunk IS in the block's merkle tree.
   - Step 3b (`verify_path` for prev_chunk): passes — shard 0's prev chunk is in its block.
   - Step 4 (receipts): passes — legitimate shard 0 receipts with valid proofs.
   - Step 5 (`validate_state_root_node`): passes — shard 1's node matches shard 1's state root.
   - Header stored under `StateHeaderKey(shard_id=0, sync_hash)`.

5. V downloads state parts for shard 1's `prev_state_root`, validates them (they are valid for shard 1's trie), and writes them into shard 0's trie via `apply_state_part`.

6. `set_state_finalize` applies shard 1's chunk to shard 0's `shard_uid`. V's shard 0 state root is now shard 1's post-state root.

7. V produces chunks for shard 0 with the wrong `prev_state_root`; all honest validators reject them. V is DoS'd out of shard 0 chunk production. [8](#0-7)

    pub fn set_state_header(
        &self,
        shard_id: ShardId,
        sync_hash: CryptoHash,
        shard_state_header: ShardStateSyncResponseHeader,
    ) -> Result<(), Error> {
        let sync_block_header = self.chain_store.get_block_header(&sync_hash)?;

        let chunk = shard_state_header.cloned_chunk();
        let prev_chunk_header = shard_state_header.cloned_prev_chunk_header();

        // 1-2. Checking chunk validity
        if !validate_chunk_proofs(&chunk, self.epoch_manager.as_ref())? {
            byzantine_assert!(false);
            return Err(Error::Other(
                "set_shard_state failed: chunk header proofs are invalid".into(),
            ));
        }

        // Consider chunk itself is valid.

        // 3. Checking that chunks `chunk` and `prev_chunk` are included in appropriate blocks
        // 3a. Checking that chunk `chunk` is included into block at last height before sync_hash
        // 3aa. Also checking chunk.height_included
        let sync_prev_block_header =
            self.chain_store.get_block_header(sync_block_header.prev_hash())?;
        if !verify_path(
            *sync_prev_block_header.chunk_headers_root(),
            shard_state_header.chunk_proof(),
            &ChunkHashHeight(chunk.chunk_hash().clone(), chunk.height_included()),
        ) {
            byzantine_assert!(false);
            return Err(Error::Other(
                "set_shard_state failed: chunk isn't included into block".into(),
            ));
        }

        let block_header = get_block_header_on_chain_by_height(
            &self.chain_store,
            &sync_hash,
            chunk.height_included(),
        )?;
        // 3b. Checking that chunk `prev_chunk` is included into block at height before chunk.height_included
        // 3ba. Also checking prev_chunk.height_included - it's important for getting correct incoming receipts
        match (&prev_chunk_header, shard_state_header.prev_chunk_proof()) {
            (Some(prev_chunk_header), Some(prev_chunk_proof)) => {
                let prev_block_header =
                    self.chain_store.get_block_header(block_header.prev_hash())?;
                if !verify_path(
                    *prev_block_header.chunk_headers_root(),
                    prev_chunk_proof,
                    &ChunkHashHeight(prev_chunk_header.chunk_hash().clone(), prev_chunk_header.height_included()),
                ) {
                    byzantine_assert!(false);
                    return Err(Error::Other(
                        "set_shard_state failed: prev_chunk isn't included into block".into(),
                    ));
                }
            }
            (None, None) => {
                if chunk.height_included() != 0 {
                    return Err(Error::Other(
                    "set_shard_state failed: received empty state response for a chunk that is not at height 0".into()
                ));
                }
            }
            _ =>
                return Err(Error::Other("set_shard_state failed: `prev_chunk_header` and `prev_chunk_proof` must either both be present or both absent".into()))
        };

        // 4. Proving incoming receipts validity
        // 4a. Checking len of proofs
        if shard_state_header.root_proofs().len()
            != shard_state_header.incoming_receipts_proofs().len()
        {
            byzantine_assert!(false);
            return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
        }
        let mut hash_to_compare = sync_hash;
        for (i, receipt_response) in
            shard_state_header.incoming_receipts_proofs().iter().enumerate()
        {
            let ReceiptProofResponse(block_hash, receipt_proofs) = receipt_response;

            // 4b. Checking that there is a valid sequence of continuous blocks
            if *block_hash != hash_to_compare {
                byzantine_assert!(false);
                return Err(Error::Other(
                    "set_shard_state failed: invalid incoming receipts".into(),
                ));
            }
            let header = self.chain_store.get_block_header(&hash_to_compare)?;
            hash_to_compare = *header.prev_hash();

            let block_header = self.chain_store.get_block_header(block_hash)?;
            // 4c. Checking len of receipt_proofs for current block
            if receipt_proofs.len() != shard_state_header.root_proofs()[i].len()
                || receipt_proofs.len() != block_header.chunks_included() as usize
            {
                byzantine_assert!(false);
                return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
            }
            // We know there were exactly `block_header.chunks_included` chunks included
            // on the height of block `block_hash`.
            // There were no other proofs except for included chunks.
            // According to Pigeonhole principle, it's enough to ensure all receipt_proofs are distinct
            // to prove that all receipts were received and no receipts were hidden.
            let mut visited_shard_ids = HashSet::<ShardId>::new();
            for (j, receipt_proof) in receipt_proofs.iter().enumerate() {
                let ReceiptProof(receipts, shard_proof) = receipt_proof;
                let ShardProof { from_shard_id, to_shard_id: _, proof } = shard_proof;
                // 4d. Checking uniqueness for set of `from_shard_id`
                match visited_shard_ids.get(from_shard_id) {
                    Some(_) => {
                        byzantine_assert!(false);
                        return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
                    }
                    _ => visited_shard_ids.insert(*from_shard_id),
                };
                let RootProof(root, block_proof) = &shard_state_header.root_proofs()[i][j];
                let receipts_hash = CryptoHash::hash_borsh(ReceiptList(shard_id, receipts));
                // 4e. Proving the set of receipts is the subset of outgoing_receipts of shard `shard_id`
                if !verify_path(*root, proof, &receipts_hash) {
                    byzantine_assert!(false);
                    return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
                }
                // 4f. Proving the outgoing_receipts_root matches that in the block
                if !verify_path(
                    *block_header.prev_chunk_outgoing_receipts_root(),
                    block_proof,
                    root,
                ) {
                    byzantine_assert!(false);
                    return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
                }
            }
        }
        // 4g. Checking that there are no more heights to get incoming_receipts
        let header = self.chain_store.get_block_header(&hash_to_compare)?;
        if header.height() != prev_chunk_header.map_or(0, |h| h.height_included()) {
            byzantine_assert!(false);
            return Err(Error::Other("set_shard_state failed: invalid incoming receipts".into()));
        }

        // 5. Checking that state_root_node is valid
        let chunk_inner = chunk.take_header().take_inner();
        if matches!(
            self.runtime_adapter.validate_state_root_node(
                shard_state_header.state_root_node(),
                chunk_inner.prev_state_root(),
            ),
            StateRootNodeValidationResult::Invalid
        ) {
            byzantine_assert!(false);
            return Err(Error::Other("set_shard_state failed: state_root_node is invalid".into()));
        }

        // Saving the header data.
        let mut store_update = self.chain_store.store().store_update();
        let key = borsh::to_vec(&StateHeaderKey(shard_id, sync_hash)).unwrap();
        store_update.set_ser(DBCol::StateHeaders, &key, &shard_state_header);
        store_update.commit();

        Ok(())

pub fn validate_chunk_proofs(
    chunk: &ShardChunk,
    epoch_manager: &dyn EpochManagerAdapter,
) -> Result<bool, Error> {
    let correct_chunk_hash = chunk.compute_header_hash();

    // 1. Checking chunk.header.hash
    let header_hash = chunk.header_hash();
    if header_hash != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }

    // 2. Checking that chunk body is valid
    // 2a. Checking chunk hash
    if chunk.chunk_hash() != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }
    let height_created = chunk.height_created();
    let outgoing_receipts_root = chunk.prev_outgoing_receipts_root();
    let (transactions, receipts) = (chunk.to_transactions(), chunk.prev_outgoing_receipts());

    // 2b. Checking that chunk transactions are valid
    let (tx_root, _) = merklize(transactions);
    if &tx_root != chunk.tx_root() {
        byzantine_assert!(false);
        return Ok(false);
    }
    // 2c. Checking that chunk receipts are valid
    if height_created == 0 {
        return Ok(receipts.is_empty() && outgoing_receipts_root == &CryptoHash::default());
    } else {
        let shard_layout = {
            let prev_block_hash = chunk.prev_block_hash();
            epoch_manager.get_shard_layout_from_prev_block(&prev_block_hash)?
        };
        let outgoing_receipts_hashes = Chain::build_receipts_hashes(receipts, &shard_layout)?;
        let (receipts_root, _) = merklize(&outgoing_receipts_hashes);
        if &receipts_root != outgoing_receipts_root {
            byzantine_assert!(false);
            return Ok(false);
        }
    }
    Ok(true)
