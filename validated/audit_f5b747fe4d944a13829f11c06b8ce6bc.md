### Title
Missing Zero-Address Validation on `nearBridgeDerivedAddress_` in `OmniBridge.initialize()` Enables Signature Verification Bypass — (`evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.initialize()` stores `nearBridgeDerivedAddress_` without any zero-address guard. This address is the sole authority used to verify MPC-signed payloads in both `finTransfer` and `deployToken`. If it is initialized to `address(0)` — whether by human error or a malicious wrapper — and the deployed OpenZeppelin ECDSA library is a version prior to 4.7.3 (where `ECDSA.recover` returns `address(0)` for invalid signatures rather than reverting), the signature check degenerates to `address(0) != address(0)`, which is always `false`, meaning the check silently passes for any garbage signature. An unprivileged attacker can then call `finTransfer` with an arbitrary payload and a random byte string as the signature to mint unbacked bridge tokens or drain native ETH and ERC-20 tokens held by the bridge.

---

### Finding Description

`OmniBridge.initialize()` accepts three parameters and writes them directly to storage with no validation:

```solidity
function initialize(
    address tokenImplementationAddress_,
    address nearBridgeDerivedAddress_,   // ← no address(0) check
    uint8 omniBridgeChainId_
) public initializer {
    tokenImplementationAddress = tokenImplementationAddress_;
    nearBridgeDerivedAddress = nearBridgeDerivedAddress_;   // stored as-is
    omniBridgeChainId = omniBridgeChainId_;
    ...
}
``` [1](#0-0) 

`nearBridgeDerivedAddress` is then used as the expected signer in both security-critical paths:

```solidity
// deployToken
if (ECDSA.recover(hashed, signatureData) != nearBridgeDerivedAddress) {
    revert InvalidSignature();
}
``` [2](#0-1) 

```solidity
// finTransfer
if (ECDSA.recover(hashed, signatureData) != nearBridgeDerivedAddress) {
    revert InvalidSignature();
}
``` [3](#0-2) 

In OpenZeppelin ECDSA versions prior to 4.7.3, `ECDSA.recover` returns `address(0)` for a malformed or invalid signature instead of reverting. If `nearBridgeDerivedAddress` is `address(0)`, the comparison becomes:

```
ECDSA.recover(hashed, <garbage>) → address(0)
address(0) != address(0)         → false   ← check passes
```

Both `finTransfer` and `deployToken` proceed as if the signature were valid.

The `setNearBridgeDerivedAddress` setter, which could be used to correct the value post-deployment, is also unchecked:

```solidity
function setNearBridgeDerivedAddress(
    address nearBridgeDerivedAddress_
) external onlyRole(DEFAULT_ADMIN_ROLE) {
    nearBridgeDerivedAddress = nearBridgeDerivedAddress_;
}
``` [4](#0-3) 

---

### Impact Explanation

With `nearBridgeDerivedAddress == address(0)` and OZ ECDSA < 4.7.3:

- **`finTransfer`**: An attacker submits any `TransferMessagePayload` (e.g., `tokenAddress = address(0)` for native ETH, `amount = bridge_balance`, `recipient = attacker`) with a random 65-byte signature. The signature check passes. The bridge transfers native ETH, mints bridge tokens, or transfers any ERC-20 it holds to the attacker. This is **direct theft and unauthorized mint of bridged assets** (Critical).
- **`deployToken`**: An attacker deploys a bridge token with arbitrary metadata, poisoning the `nearToEthToken` / `ethToNearToken` mappings and potentially blocking legitimate token deployments for those token IDs.

The `completedTransfers[payload.destinationNonce]` replay guard is per-nonce, so the attacker simply increments the nonce across calls to drain the entire bridge.

---

### Likelihood Explanation

The `initialize` function is called once at deployment, typically by a deployment script or multisig. A typo, a zero-filled calldata field, or a factory wrapper that passes `address(0)` for this parameter (analogous to the AeraVault report's factory-wrapper scenario) would silently produce a broken and exploitable bridge. The OmniBridgeWormhole variant's `initializeWormhole` delegates directly to `initialize` with the same unchecked parameters:

```solidity
function initializeWormhole(
    address tokenImplementationAddress,
    address nearBridgeDerivedAddress,   // ← passed through unchecked
    ...
) external initializer {
    initialize(tokenImplementationAddress, nearBridgeDerivedAddress, omniBridgeChainId);
``` [5](#0-4) [6](#0-5) 

**Note on OZ version dependency**: The Critical exploit path (signature bypass) requires OZ ECDSA < 4.7.3. With OZ >= 4.7.3 or v5, `ECDSA.recover` reverts on invalid signatures, so the impact degrades to a non-functional bridge (DoS) until the admin calls `setNearBridgeDerivedAddress`. The exact OZ version in use could not be confirmed from the available index; if OZ >= 4.7.3 is confirmed, this finding's severity reduces to a deployment-safety / DoS concern rather than a theft vector.

---

### Recommendation

Add an explicit zero-address guard in `initialize()` and `setNearBridgeDerivedAddress()`:

```solidity
require(nearBridgeDerivedAddress_ != address(0), "ERR_ZERO_BRIDGE_ADDRESS");
```

Apply the same guard to `tokenImplementationAddress_` (even though a runtime check exists in `deployToken`, failing fast at initialization is safer). Consider also validating that `omniBridgeChainId_` is non-zero or matches a known set of chain IDs to prevent cross-chain replay if two deployments accidentally share the same chain ID.

---

### Proof of Concept

1. Deploy `OmniBridge` (or `OmniBridgeWormhole`) with `nearBridgeDerivedAddress_ = address(0)` (e.g., via a deployment script bug or factory wrapper).
2. Fund the bridge with native ETH via `receive()`.
3. Craft a `TransferMessagePayload` with `tokenAddress = address(0)`, `amount = bridge_balance`, `recipient = attacker`, `destinationNonce = 1`.
4. Call `finTransfer(randomBytes65, payload)`.
5. `ECDSA.recover(keccak256(borshEncoded), randomBytes65)` returns `address(0)` (OZ < 4.7.3).
6. `address(0) != address(0)` → `false` → check passes.
7. `completedTransfers[1] = true`; bridge sends `bridge_balance` ETH to attacker.
8. Repeat with `destinationNonce = 2, 3, …` for ERC-20 tokens and bridge-token minting. [7](#0-6)

            revert NonceAlreadyUsed(payload.destinationNonce);
        }

        completedTransfers[payload.destinationNonce] = true;

        bytes memory borshEncoded = bytes.concat(
            bytes1(uint8(BridgeTypes.PayloadType.TransferMessage)),
            Borsh.encodeUint64(payload.destinationNonce),
            bytes1(payload.originChain),
            Borsh.encodeUint64(payload.originNonce),
            bytes1(omniBridgeChainId),
            Borsh.encodeAddress(payload.tokenAddress),
            Borsh.encodeUint128(payload.amount),
            bytes1(omniBridgeChainId),
            Borsh.encodeAddress(payload.recipient),
            bytes(payload.feeRecipient).length == 0 // None or Some(String) in rust
                ? bytes("\x00")
                : bytes.concat(
                    bytes("\x01"),
                    Borsh.encodeString(payload.feeRecipient)
                ),
            bytes(payload.message).length == 0
                ? bytes("")
                : Borsh.encodeBytes(payload.message)
        );
        bytes32 hashed = keccak256(borshEncoded);

        if (ECDSA.recover(hashed, signatureData) != nearBridgeDerivedAddress) {
            revert InvalidSignature();

                IBridgeToken(payload.tokenAddress).mint(

                    payload.message

                payload.amount
            );
