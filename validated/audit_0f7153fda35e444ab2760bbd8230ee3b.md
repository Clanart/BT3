### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts an arbitrary `class_hash` from the caller and writes it directly to contract state without verifying that the hash corresponds to a previously declared class. An explicit `TODO` comment in the code acknowledges this check is missing. As a result, any contract can replace its own class hash with an undeclared value, permanently freezing all funds held in that contract.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The `class_hash` value from the syscall request is written directly into `contract_state_changes` with no check that it exists in `contract_class_changes` (the declared class registry). The TODO comment at line 898 explicitly acknowledges this missing validation.

When any subsequent call is made to that contract (in the same or a future block), `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [2](#0-1) 

If the class hash was never declared, `dict_read` returns the default value `0`, and `find_element` will fail to locate a matching `CompiledClassFact`, causing a Cairo assertion failure that makes the block unprovable. The contract becomes permanently uncallable.

This is structurally identical to the M-03 pattern: a security enforcement function (`_setupDelayForRenouncingOwnership` / the class-existence check) is defined and documented as necessary but is never actually invoked, leaving the mechanism completely bypassed.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value via `replace_class`, every future call to that contract causes `find_element` to fail inside `execute_entry_point`. The block containing such a call becomes unprovable. The sequencer must permanently exclude calls to that contract, making any funds held in it irretrievable. There is no recovery path because the corrupted class hash is committed to the Merkle state.

---

### Likelihood Explanation

**Medium.** The `replace_class` syscall is available to any deployed contract without privilege restrictions. A malicious contract author can deliberately call `replace_class` with an arbitrary undeclared hash to permanently freeze their own contract's funds (e.g., as a griefing or denial-of-service attack against users who have deposited funds). It can also be triggered accidentally by a buggy contract. No special role or key is required — only the ability to deploy and call a contract.

---

### Recommendation

Before writing the new class hash to state, validate that it exists in `contract_class_changes` (i.e., it has a non-zero compiled class hash entry). The TODO comment at line 898 already identifies this requirement:

```cairo
// Add before the dict_update in execute_replace_class:
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors how `execute_entry_point` itself validates class existence before execution, and closes the gap between the documented intent and the actual enforcement.

---

### Proof of Concept

1. Attacker deploys contract `C` holding user funds (e.g., an escrow).
2. Attacker calls a function in `C` that internally invokes the `replace_class` syscall with `class_hash = 0xdeadbeef` (any value not present in `contract_class_changes`).
3. `execute_replace_class` accepts the value without validation and commits `StateEntry(class_hash=0xdeadbeef, ...)` for contract `C`.
4. In any subsequent block, when a user calls contract `C`, `execute_entry_point` reads `class_hash = 0xdeadbeef`, performs `dict_read` on `contract_class_changes` returning `0`, then calls `find_element(..., key=0)`.
5. `find_element` fails (no compiled class with hash `0` exists), making the block unprovable.
6. The sequencer must permanently exclude all calls to `C`. Funds are frozen forever. [3](#0-2) [4](#0-3)
