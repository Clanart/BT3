### Title
`ProofOfInclusion::valid()` Does Not Verify Against an External Root Hash — Forged Inclusion Proofs Always Pass - (File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs)

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate only checks internal hash-chain consistency. It never accepts or compares against an external, trusted Merkle root. The final guard `existing_hash == self.root_hash()` is a tautology — it is always `true` when the loop completes — because `existing_hash` is set to `calculated_hash`, which was already asserted equal to `layer.combined_hash`, and `root_hash()` returns that same `last.combined_hash`. An attacker can therefore craft a `ProofOfInclusion` with arbitrary `node_hash` and layers that form a self-consistent chain, pass `valid()`, and present any attacker-chosen value as the tree root.

### Finding Description

`ProofOfInclusion::valid()` iterates over layers, computing `calculated_hash = calculate_internal_hash(existing_hash, side, other_hash)`, asserting `calculated_hash == layer.combined_hash`, then setting `existing_hash = calculated_hash`. [1](#0-0) 

After the loop, `existing_hash` equals the last `layer.combined_hash`. `root_hash()` also returns `last.combined_hash`: [2](#0-1) 

So the final assertion `existing_hash == self.root_hash()` reduces to `last.combined_hash == last.combined_hash` — always `true`. The function never compares against any externally supplied, trusted root.

The struct is fully constructible from untrusted bytes via the Python-exposed `from_bytes` / `from_bytes_unchecked` / `parse_rust` methods generated by `PyStreamable`: [3](#0-2) 

And is directly instantiable from Python: [4](#0-3) 

**Forge recipe (no privileges required):**

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
from chia_rs.sized_bytes import bytes32
import hashlib

# Claim any node_hash is "included"
node_hash = bytes32(b'\xde' * 32)
other_hash = bytes32(b'\xab' * 32)
# compute combined_hash using the same calculate_internal_hash logic
combined_hash = bytes32(hashlib.sha256(b'\x01' + node_hash + other_hash).digest())  # simplified

layer = ProofOfInclusionLayer(other_hash_side=0, other_hash=other_hash, combined_hash=combined_hash)
proof = ProofOfInclusion(node_hash=node_hash, layers=[layer])

assert proof.valid()          # True — passes with no external root check
assert proof.root_hash() == combined_hash  # attacker-controlled root
```

Even simpler — an empty-layers proof is always valid:

```python
proof = ProofOfInclusion(node_hash=bytes32(b'\xde'*32), layers=[])
assert proof.valid()   # True — root_hash() == node_hash, tautology
```

### Impact Explanation

Any consumer that calls `proof.valid()` as the sole verification step — the natural reading of the API — will accept a forged proof for any `node_hash` the attacker chooses, with any attacker-chosen `root_hash()`. This allows an untrusted peer to convince a DataLayer client that an arbitrary key/value hash is included in a tree it controls, enabling forged state proofs. This matches the allowed High impact: **DataLayer Merkle proof logic accepts forged inclusion, letting untrusted input prove invalid state.**

### Likelihood Explanation

The `valid()` method is the only verification entry point exposed to Python callers. There is no `valid_for_root(expected_root)` variant. The method name implies completeness. Any DataLayer client that receives a `ProofOfInclusion` over the network and calls only `proof.valid()` — without separately comparing `proof.root_hash()` against a locally-trusted root — is fully exploitable by a malicious peer. The Python wheel exposes `ProofOfInclusion` with `from_bytes` deserialization, making the attack path direct.

### Recommendation

Add an `expected_root` parameter to `valid()` (or provide a separate `valid_for_root(root: Hash) -> bool` method) that compares the computed root against the caller-supplied trusted value:

```rust
pub fn valid_for_root(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash, layer.other_hash_side, &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root   // compare against EXTERNAL trusted root
}
```

Expose this as the primary Python API and deprecate the no-argument `valid()`.

### Proof of Concept

**Root cause lines:** [1](#0-0) 

**Tautology**: after the loop, `existing_hash` = last `calculated_hash` = last `layer.combined_hash` (enforced by the `!= combined_hash → return false` guard). `root_hash()` returns `last.combined_hash`. So `existing_hash == self.root_hash()` is `last.combined_hash == last.combined_hash` — unconditionally `true`.

**Attacker path**: deserialize a crafted `ProofOfInclusion` via `from_bytes` (Python-exposed), call `valid()` → `true`, present `root_hash()` as the claimed tree root. No cryptographic material, no privileged access, no existing tree required.
