### Title
Resigned or Revoked Trusted Relayer Permanently Loses Accrued Fee Claims Due to `#[trusted_relayer]` Guard on `claim_fee` - (File: `near/omni-bridge/src/lib.rs`)

---

### Summary

The `claim_fee` function in the NEAR bridge locker is gated by the `#[trusted_relayer]` macro, which checks the **current** trusted-relayer status of the caller at the time of the call. A relayer who signed a transfer (becoming the `fee_recipient` embedded in the proof) and then resigned or was forcibly revoked by the DAO loses permanent access to all fees they legitimately earned. The fee amount remains locked inside the bridge contract with no recovery path.

---

### Finding Description

`claim_fee` is decorated with `#[trusted_relayer]`: [1](#0-0) 

The macro is imported from `omni_utils::macros::trusted_relayer` and enforces that `env::predecessor_account_id()` is currently a trusted relayer at call time. [2](#0-1) 

Inside `claim_fee_callback`, a second check enforces that only the specific `fee_recipient` embedded in the proof can receive the fee: [3](#0-2) 

Together these two checks mean:
- Only the exact `fee_recipient` can claim (no substitute caller).
- That `fee_recipient` must **still** be a trusted relayer at claim time.

A relayer can lose trusted status in two ways confirmed by the test suite:

1. **Voluntary resignation** via `resign_trusted_relayer` — stake is returned, trusted status is removed.
2. **Forced revocation** by DAO via `reject_relayer_application` on an active relayer — stake is seized, trusted status is removed. [4](#0-3) [5](#0-4) 

After either event, `is_trusted_relayer` returns `false` for that account. Any subsequent call to `claim_fee` by the former relayer is rejected by `#[trusted_relayer]` before the proof is even verified. The `pending_transfers` entry is never removed, and the fee amount remains permanently locked in the bridge. [6](#0-5) 

---

### Impact Explanation

The fee amount specified by the user at `init_transfer` time is deducted from the bridged amount and held inside the bridge contract. It is only released when `claim_fee` succeeds and calls `send_fee_internal`. If `claim_fee` is permanently blocked for the legitimate `fee_recipient`, the fee is irrecoverably locked — no other account can claim it (blocked by the `fee_recipient == predecessor_account_id` check), and no administrative function exists to rescue it.

This matches the allowed impact: **Permanent freezing / irrecoverable lock of user or protocol funds in bridge flows.** [7](#0-6) 

---

### Likelihood Explanation

The scenario is realistic and requires no privileged attacker:

- A relayer signs many transfers over time, accumulating unclaimed fees in `pending_transfers`.
- The relayer voluntarily calls `resign_trusted_relayer` (e.g., to exit the protocol and recover their stake), or the DAO forcibly revokes them.
- All pending fee claims become permanently unclaimable.

The relayer's own voluntary resignation is an entirely unprivileged, self-initiated action that triggers the loss — no admin collusion is required.

---

### Recommendation

Remove the `#[trusted_relayer]` guard from `claim_fee`. The function already enforces that only the exact `fee_recipient` embedded in the cryptographic proof can receive the fee, so there is no security benefit to additionally requiring current trusted-relayer status. Any account that holds a valid proof naming itself as `fee_recipient` has legitimately earned that fee.

```rust
// Before
#[payable]
#[trusted_relayer]          // <-- remove this
#[pause(except(roles(Role::DAO)))]
pub fn claim_fee(&mut self, #[serializer(borsh)] args: ClaimFeeArgs) -> Promise { ... }

// After
#[payable]
#[pause(except(roles(Role::DAO)))]
pub fn claim_fee(&mut self, #[serializer(borsh)] args: ClaimFeeArgs) -> Promise { ... }
```

---

### Proof of Concept

1. Relayer `R` calls `apply_for_trusted_relayer` with the required stake and waits for the activation period.
2. `R` calls `sign_transfer` for several pending transfers, setting `fee_recipient = R`. Each signed transfer is finalized on the destination chain, generating valid `FinTransfer` proofs.
3. Before calling `claim_fee`, `R` calls `resign_trusted_relayer`. Stake is returned; `is_trusted_relayer(R)` → `false`.
4. `R` attempts `claim_fee` with a valid proof. The `#[trusted_relayer]` macro panics immediately — the proof is never verified, `remove_transfer_message` is never called, and the fee is never sent.
5. No other account can call `claim_fee` for these transfers because `claim_fee_callback` enforces `fee_recipient == predecessor_account_id`, and `fee_recipient` is `R`.
6. All accrued fees are permanently locked in the bridge contract. [1](#0-0) [8](#0-7)

    fn remove_transfer_message(&mut self, transfer_id: TransferId) -> TransferMessage {
        let storage_usage = env::storage_usage();
        let transfer = self
            .pending_transfers
            .remove(&transfer_id)
            .map(storage::TransferMessageStorage::into_main)
            .near_expect(BridgeError::TransferNotExist);

        let refund =
            env::storage_byte_cost().saturating_mul((storage_usage - env::storage_usage()).into());

        if let Some(mut storage) = self.accounts_balances.get(&transfer.owner) {
            storage.available = storage.available.saturating_add(refund);
            self.accounts_balances.insert(&transfer.owner, &storage);
        }

        transfer.message
    }

    fn send_fee_internal(
        &mut self,
        transfer_message: &TransferMessage,
        fee_recipient: AccountId,
        token_fee: u128,
    ) -> PromiseOrValue<()> {
        if transfer_message.fee.native_fee.0 != 0 {
            let origin_chain = transfer_message.origin_transfer_id.as_ref().map_or_else(
                || transfer_message.get_origin_chain(),
                |origin_transfer_id| origin_transfer_id.origin_chain,
            );

            if origin_chain.is_utxo_chain() {
                env::panic_str(BridgeError::NativeFeeForUtxoChain.to_string().as_str())
            } else if origin_chain == ChainKind::Near {
                Promise::new(fee_recipient.clone())
                    .transfer(NearToken::from_yoctonear(transfer_message.fee.native_fee.0))
                    .detach();
            } else {
                ext_token::ext(self.get_native_token_id(origin_chain))
                    .with_static_gas(MINT_TOKEN_GAS)
                    .mint(fee_recipient.clone(), transfer_message.fee.native_fee, None)
                    .detach();
            }
        }

        let token = self.get_token_id(&transfer_message.token);
        env::log_str(
            &OmniBridgeEvent::ClaimFeeEvent {
                transfer_message: transfer_message.clone(),
            }
            .to_log_string(),
        );

        self.unlock_tokens_if_needed(transfer_message.get_destination_chain(), &token, token_fee);

        if token_fee > 0 {
            if self.is_deployed_token(&token) {
                ext_token::ext(token)
                    .with_static_gas(MINT_TOKEN_GAS)
                    .mint(fee_recipient, U128(token_fee), None)
                    .into()
            } else {
                ext_token::ext(token)
                    .with_static_gas(FT_TRANSFER_GAS)
                    .with_attached_deposit(ONE_YOCTO)
                    .ft_transfer(fee_recipient, U128(token_fee), None)
                    .into()
            }
        } else {
            PromiseOrValue::Value(())
        }
