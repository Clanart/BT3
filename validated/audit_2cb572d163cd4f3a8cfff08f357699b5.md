### Title
Deprecated (Cairo 0) Class Returned Without State-Number Declaration Check in `ExecutionStateReader::get_compiled_class` - (File: `crates/apollo_rpc_execution/src/state_reader.rs`)

---

### Summary

`ExecutionStateReader::get_compiled_class` skips the block-number-bounded declaration check for Cairo 0 (deprecated) classes when the class manager path is active. The class manager is a global, block-agnostic store; a deprecated class declared at block N is returned unconditionally for any historical query at block M < N. This is the direct sequencer analog of the Aave Lens M-07 bug: the existence check passes against a global secondary store (class manager) rather than against the block-scoped primary record (storage state diff), so a class that does not yet exist at the queried state is treated as if it does.

---

### Finding Description

In `ExecutionStateReader::get_compiled_class` (lines 115–141 of `crates/apollo_rpc_execution/src/state_reader.rs`), when `class_manager_handle` is `Some`, the function dispatches on the class variant returned by `class_manager_client.get_executable(class_hash)`:

```rust
// For Cairo 1 (V1): declaration check IS performed
ContractClass::V1(casm_contract_class) => {
    let is_declared = is_contract_class_declared(
        &self.storage_reader.begin_ro_txn()...,
        &class_hash,
        self.state_number,   // ← bounded to the queried block
    )?;
    if is_declared {
        Ok(RunnableCompiledClass::V1(casm_contract_class.try_into()?))
    } else {
        Err(StateError::UndeclaredClassHash(class_hash))
    }
}

// For Cairo 0 (V0): declaration check IS NOT performed
// TODO(shahak): Verify cairo0 as well after get_class_definition_block_number is fixed.
ContractClass::V0(deprecated_contract_class) => {
    Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
}
``` [1](#0-0) 

The class manager stores deprecated classes globally via `set_deprecated_class` and retrieves them via `get_executable` without any block-number scope. Once a deprecated class is stored in the class manager (at declaration time), it is returned for any future query regardless of the `state_number` of the `ExecutionStateReader`.

The correct per-block check for deprecated classes is `get_deprecated_class_definition_block_number`, which lives in a separate storage table (`deprecated_declared_classes_block_table`) from the Cairo 1 table (`declared_classes_block_table`) that `is_contract_class_declared` queries. The TODO comment acknowledges this gap. [2](#0-1) 

By contrast, `SyncStateReader::get_contract_class_from_client` (used by the gateway) calls `is_class_declared_at` **before** fetching the class, and `is_class_declared_at` correctly checks both Cairo 1 and Cairo 0 declaration block numbers: [3](#0-2) 

`ExecutionStateReader` is the reader used for RPC execution paths (fee estimation, simulation, `starknet_call`, trace).

---

### Impact Explanation

Any RPC call that triggers `ExecutionStateReader::get_compiled_class` for a deprecated class at a historical block number before that class was declared will receive the class body and execute against it, producing:

- A wrong execution result (wrong return data, wrong events, wrong storage changes) from `starknet_call`, `starknet_simulateTransactions`, or `starknet_traceTransaction`.
- A wrong fee estimate from `starknet_estimateFee` or `starknet_estimateMessageFee`.

These are authoritative-looking responses — the RPC server returns `200 OK` with a plausible result, not an error. Callers (wallets, dApps, bridges) relying on fee estimation or simulation at historical blocks will receive incorrect data.

This matches: **High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

---

### Likelihood Explanation

The trigger requires:
1. A deprecated (Cairo 0) class declared at some block N.
2. An RPC call at a block number M < N that involves that class hash (e.g., a `starknet_call` to a contract whose class hash is the deprecated class, or a `starknet_estimateFee` for a transaction that deploys or calls such a contract).
3. The RPC node must have `class_manager_handle` set (i.e., the class manager is configured, which is the production sequencer deployment mode).

Condition 1 is satisfied by any existing Cairo 0 class on the network. Condition 2 is achievable by any unprivileged user who knows a deprecated class hash and can craft an RPC call referencing it at a historical block. Condition 3 is the default production configuration.

---

### Recommendation

In the `ContractClass::V0` arm of `ExecutionStateReader::get_compiled_class`, add a declaration check analogous to the Cairo 1 check, using `get_deprecated_class_definition_block_number` from the storage state reader:

```rust
ContractClass::V0(deprecated_contract_class) => {
    let txn = self.storage_reader.begin_ro_txn()
        .map_err(storage_err_to_state_err)?;
    let state_reader = txn.get_state_reader()
        .map_err(storage_err_to_state_err)?;
    let maybe_block = state_reader
        .get_deprecated_class_definition_block_number(&class_hash)
        .map_err(|e| StateError::StateReadError(e.to_string()))?;
    let is_declared = maybe_block
        .map_or(false, |bn| !self.state_number.is_before(bn));
    if is_declared {
        Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
    } else {
        Err(StateError::UndeclaredClassHash(class_hash))
    }
}
```

This mirrors the Cairo 1 path and resolves the acknowledged TODO. [4](#0-3)

### Proof of Concept

**Setup:**
- Block 0: deprecated class `H` is declared → class manager stores `H`.
- Block 1: some other state change (no re-declaration of `H`).

**Attack:**
1. Attacker calls `starknet_call` (or `starknet_estimateFee`) at `block_id = 0` (i.e., `state_number = right_before_block(0)`, before `H` was declared) with a contract whose class hash is `H`.
2. `ExecutionStateReader::get_compiled_class(H)` is invoked with `state_number` corresponding to block -1 (genesis state, before block 0).
3. `class_manager_client.get_executable(H)` returns `ContractClass::V0(...)` — the class is in the global store.
4. The `V0` arm returns `Ok(RunnableCompiledClass::V0(...))` without checking `state_number`.
5. The execution engine runs the class body as if it were declared at genesis, producing a wrong result.

**Expected:** `StateError::UndeclaredClassHash(H)` → RPC returns `CLASS_HASH_NOT_FOUND`.
**Actual:** Execution proceeds with the class body, returning a wrong authoritative result. [5](#0-4) [6](#0-5)
