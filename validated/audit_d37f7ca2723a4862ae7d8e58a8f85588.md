### Title
`validate_virtual_os_state_diff` Rejects Legitimate Zero-Value Storage Writes, Blocking Transaction Proof Generation — (`File: crates/starknet_transaction_prover/src/running/committer_utils.rs`)

---

### Summary

The `validate_virtual_os_state_diff` function in the transaction prover path unconditionally rejects any storage update whose value is `Felt::ZERO`. Writing zero to a storage slot is a valid, unprivileged Starknet operation (it is the canonical way to clear/delete a storage entry). Because `state_maps_to_committer_state_diff` passes zero-valued storage writes through without filtering, any virtual-OS proof run over a transaction that clears storage will hit this check and return `ProofProviderError::InvalidStateDiff`, permanently blocking proof generation for that transaction.

---

### Finding Description

**Root cause — the zero-value guard in `validate_virtual_os_state_diff`:**

```rust

---

### Impact Explanation

Any unprivileged user can submit a valid Invoke transaction that writes `Felt::ZERO` to a storage slot (e.g., clearing an ERC-20 allowance, resetting a counter, or any `storage_write(key, 0)` syscall). When the `starknet_transaction_prover` attempts to generate a proof for a virtual block containing such a transaction, `validate_virtual_os_state_diff` returns `ProofProviderError::InvalidStateDiff`. The proof pipeline is aborted; the transaction cannot be proven. This maps to **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value**, and potentially **Critical — Wrong state or revert result from blockifier/syscall/execution logic for accepted input** if the proving failure propagates to an incorrect or missing state commitment.

---

### Likelihood Explanation

Writing zero to storage is a routine, unprivileged operation in any Starknet contract (clearing balances, resetting flags, revoking approvals). Any block that contains such a transaction will trigger the failure. No special privileges or adversarial setup are required.

---

### Recommendation

Mirror the GiantLP fix: skip (or strip) zero-valued storage entries before the validation gate. The minimal fix is to filter them out in `state_maps_to_committer_state_diff` before the diff reaches `validate_virtual_os_state_diff`:

```rust
storage_updates: StorageDiff::from(StorageView(state_maps.storage))
    .into_iter()
    .filter_map(|(address, updates)| {
        let non_zero: HashMap<_, _> = updates
            .into_iter()
            .filter(|(_, v)| v != &Felt::ZERO)   // skip deletions
            .collect();
        if non_zero.is_empty() { None } else { Some((address, non_zero)) }
    })
    .collect(),
```

Alternatively, remove the zero-value guard from `validate_virtual_os_state_diff` entirely if the committer already handles zero-value leaves correctly (which the Patricia trie implementation does — a zero leaf hash is the canonical "empty" leaf).

---

### Proof of Concept

1. Deploy any Cairo contract with a `storage_write` entry point.
2. Submit an Invoke transaction that calls `storage_write(key=0x1, value=0x0)` (clearing a previously set slot).
3. The blockifier executes the transaction successfully; `StateMaps.storage` contains `((contract, key), Felt::ZERO)`.
4. `state_maps_to_committer_state_diff` converts this to `StateDiff { storage_updates: { contract: { key: StarknetStorageValue(Felt::ZERO) } } }`.
5. `validate_virtual_os_state_diff` iterates the updates, finds `value.0 == Felt::ZERO`, and returns:
   ```
   ProofProviderError::InvalidStateDiff(
       "Storage deletion not allowed: try to delete storage at address …, key …"
   )
   ```
6. Proof generation for the virtual block is aborted; the transaction cannot be proven despite being valid and accepted on-chain. [7](#0-6)

/// Validates that the committer state diff contains only allowed state transitions.
///
/// This function enforces the following constraints:
/// * **No Storage Deletions:** Storage entries cannot be updated to `Felt::ZERO`.
/// * **No Class Declarations:** The `class_hash_to_compiled_class_hash` map must be empty.
/// * **No Contract Deployments:** The `address_to_class_hash` map must be empty.
pub(crate) fn validate_virtual_os_state_diff(
    state_diff: &StateDiff,
) -> Result<(), ProofProviderError> {
    // validate no storage deletions.
    for (address, storage_diffs) in &state_diff.storage_updates {
        for (key, value) in storage_diffs {
            if value.0 == Felt::ZERO {
                return Err(ProofProviderError::InvalidStateDiff(format!(
                    "Storage deletion not allowed: try to delete storage at address {address:?}, \
                     key {key:?}"
                )));
            }
        }
    }
    // validate no contract deployments (or replaced classes).
    if !state_diff.address_to_class_hash.is_empty() {
        return Err(ProofProviderError::InvalidStateDiff(format!(
            "Contract deployments not allowed: try to deploy contracts(address to class hash): \
             {0:?}",
            state_diff.address_to_class_hash
        )));
    }
    // validate no contract declarations (or compiled class hash updates).
    if !state_diff.class_hash_to_compiled_class_hash.is_empty() {
        return Err(ProofProviderError::InvalidStateDiff(format!(
            "Contract declarations not allowed: try to declare classes(class hash to compiled \
             class hash): {0:?}",
            state_diff.class_hash_to_compiled_class_hash
        )));
    }
    Ok(())
}
