### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the new class hash supplied by a contract is a previously declared class. A contract (or an attacker who can influence a contract's upgrade path) can call `replace_class` with an arbitrary, undeclared class hash. Once the contract's on-chain class hash is overwritten with an undeclared value, every subsequent attempt to execute that contract causes a hard, non-recoverable OS-level failure inside `execute_entry_point`. The contract becomes permanently non-executable, and any funds it holds are permanently frozen.

---

### Finding Description

**Root cause — `execute_replace_class` (syscall_impls.cairo, line 898)**

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The function accepts any `class_hash` value from the syscall request and writes it directly into `contract_state_changes` without consulting `contract_class_changes` to confirm the hash was ever declared. The developer-authored TODO at line 898 explicitly acknowledges this missing guard.

**Downstream failure — `execute_entry_point` (execute_entry_point.cairo, lines 154–166)**

When any future transaction targets the affected contract, the OS executes:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // ← the undeclared hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,           // ← 0 or garbage; not in the bundle
);
```

`dict_read` on an undeclared key returns 0 (the dict default). `find_element` then searches the compiled-class bundle for hash `0`, which does not exist, producing a hard Cairo assertion failure — not a graceful revert. The sequencer cannot include any call to this contract in a provable block. The contract is permanently bricked.

The identical missing check exists in the deprecated path: `execute_replace_class` in `deprecated_execute_syscalls.cairo` (lines 307–329) also performs no declared-class validation before writing the new class hash.

---

### Impact Explanation

**Permanent freezing of funds (Critical).**

Once a contract's class hash is overwritten with an undeclared value:

- Every call to the contract causes a non-recoverable OS-level failure in `execute_entry_point`.
- There is no recovery path: the contract cannot execute `replace_class` again to self-heal because it is already non-executable.
- All ERC-20 balances, ETH, or other assets stored in the contract's storage are permanently inaccessible.

This maps directly to the allowed impact: *Critical — Permanent freezing of funds.*

---

### Likelihood Explanation

The `replace_class` syscall is callable by any contract on itself during normal execution. Realistic trigger paths include:

1. **Upgradeable contracts with external governance**: A contract that lets token holders or a DAO vote on a new implementation class hash. An attacker submits a proposal with an undeclared hash; if it passes, the contract is bricked.
2. **Proxy/upgrade patterns with insufficient input validation**: A contract that forwards a caller-supplied class hash directly to `replace_class` without checking whether it is declared.
3. **Accidental misuse**: A developer passes a Sierra class hash instead of the corresponding compiled (CASM) class hash, or passes a hash that was never declared on-chain.

No privileged role, leaked key, or external dependency is required. Any unprivileged transaction sender who can influence the argument to `replace_class` in a target contract can trigger this.

---

### Recommendation

Inside `execute_replace_class` (and its deprecated counterpart), before writing the new class hash to `contract_state_changes`, assert that the hash exists in `contract_class_changes`:

```cairo
// Verify the new class hash has been declared.
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=class_hash
);
with_attr error_message("Class hash not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the guard already present in `execute_declare_transaction` (`transaction_impls.cairo`, line 816: `assert_not_zero(compiled_class_hash)`) and closes the gap acknowledged by the TODO comment.

---

### Proof of Concept

1. **Declare** a contract class `C` and **deploy** a contract instance `A` using class `C`. Contract `A` holds user funds and exposes an `upgrade(new_class_hash)` entry point that calls `replace_class(new_class_hash)`.

2. **Attacker** calls `A.upgrade(0xdeadbeef)` where `0xdeadbeef` is never declared on-chain.

3. The OS executes `execute_replace_class` in `syscall_impls.cairo`:
   - Line 896: `let class_hash = request.class_hash;` → `0xdeadbeef`
   - Line 898: TODO check is absent — no validation performed.
   - Lines 906–910: `dict_update` writes `class_hash=0xdeadbeef` into `contract_state_changes` for contract `A`.
   - Transaction succeeds and is included in a block.

4. **Any subsequent call** to contract `A` reaches `execute_entry_point` (`execute_entry_point.cairo`):
   - Line 154–156: `dict_read(key=0xdeadbeef)` on `contract_class_changes` → returns `0` (undeclared).
   - Lines 161–166: `find_element(..., key=0)` → key not found in `compiled_class_facts_bundle` → hard Cairo assertion failure.
   - The sequencer cannot produce a valid proof for any block containing a call to `A`.

5. Contract `A` is permanently non-executable. All funds stored in its storage are permanently frozen. [1](#0-0) [2](#0-1) [3](#0-2)
