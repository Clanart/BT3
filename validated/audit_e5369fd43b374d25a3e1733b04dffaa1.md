### Title
`as u8` truncation of `k_size` from oversized proof causes `update_digest` panic in consensus hash path — (`crates/chia-protocol/src/proof_of_space.rs`)

### Summary
A v2 `ProofOfSpace` with `proof.len()` equal to any multiple of 4096 bytes (e.g. 4096, 8192, …) causes `k_size` to silently wrap to 0 via Rust's `as u8` truncation. `quality_string_from_proof` returns `None` for `k_size = 0`, so `quality_string()` returns `None`. `update_digest` then hits the unconditional `.expect(…)` and panics. Because `update_digest` is called whenever the block hash is computed, any node that receives and attempts to hash a block containing this crafted proof crashes.

### Finding Description

**Root cause — integer truncation:** [1](#0-0) 

`(self.proof.len() * 8 / 128) as u8` performs the division in `usize` and then truncates to `u8`. For `proof.len() = 4096`, the intermediate value is `256`, which wraps to `0`.

**No proof-length guard in `parse`:** [2](#0-1) 

The v2 parse path reads `proof` as an unconstrained `Bytes` field. The only structural check is that exactly one of `pool_public_key` / `pool_contract_puzzle_hash` is set. Proof length is never validated.

**Unconditional panic in `update_digest`:** [3](#0-2) 

`quality_string()` returning `None` (because `quality_string_from_proof` rejects `k_size = 0`) causes `.expect(…)` to panic. This is inside the `Streamable` implementation, which is called whenever the block hash is computed.

### Impact Explanation

`update_digest` is the `Streamable` hashing primitive. Any code path that computes the hash of a block record containing a crafted `ProofOfSpace` — including block validation, block propagation, and block storage — will panic and crash the node process. A single crafted block broadcast to the network can crash all nodes that attempt to hash it, causing a chain halt. This matches the Critical impact: *"serialized network object can trigger deterministic consensus divergence, chain halt, or committed state corruption."*

### Likelihood Explanation

The exploit requires only:
1. Constructing a syntactically valid v2 `ProofOfSpace` with `proof.len() = 4096` (or any multiple of 4096 ≥ 4096).
2. Embedding it in a block and broadcasting it.

No privileged access, leaked keys, or social engineering is required. The crafted bytes parse successfully through `ProofOfSpace::parse`.

### Recommendation

1. **Validate proof length in `parse`** for v2 proofs: reject any proof whose length does not satisfy `proof.len() * 8 % 128 == 0` and `(proof.len() * 8 / 128) > 0 && (proof.len() * 8 / 128) <= 255`.
2. **Replace the `as u8` cast** in `quality_string` with a checked conversion:
   ```rust
   let k_size = u8::try_from(self.proof.len() * 8 / 128).ok()?;
   if k_size == 0 { return None; }
   ```
   This makes `quality_string` return `None` gracefully instead of passing an invalid `k_size` downstream.
3. **Remove the `.expect(…)` in `update_digest`** or add a pre-validation step before hashing so that structurally invalid proofs are rejected at parse time, never at hash time.
