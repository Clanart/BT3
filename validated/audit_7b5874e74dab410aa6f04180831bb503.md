### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS Cairo program does not verify that the new class hash supplied via the `replace_class` syscall corresponds to any previously declared class. The OS unconditionally commits the arbitrary class hash into the global state tree. Any contract whose class hash is replaced with an undeclared value becomes permanently uncallable, freezing all funds it holds.

---

### Finding Description

`execute_replace_class` processes the `replace_class` syscall. It reads `request.class_hash` directly from the syscall segment and writes it into the contract's `StateEntry` without any cross-check against `contract_class_changes` (the dictionary of declared classes):

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment at line 898 is the codebase's own acknowledgment that this check is absent. The function is dispatched directly from the main syscall loop in `execute_syscalls`: [2](#0-1) 

`UNINITIALIZED_CLASS_HASH` is defined as `0` in `commitment.cairo`: [3](#0-2) 

`get_contract_state_hash` treats `class_hash == 0` as the "uninitialized" sentinel and returns a leaf hash of `0`, meaning a contract whose class is replaced with `0` is indistinguishable from a never-deployed address in the commitment tree: [4](#0-3) 

By contrast, `deploy_contract` correctly guards against deploying to an already-occupied address by asserting `state_entry.class_hash = UNINITIALIZED_CLASS_HASH` and `state_entry.nonce = 0`: [5](#0-4) 

And `execute_declare_transaction` enforces that a class can only be declared once by using `prev_value=0`: [6](#0-5) 

`execute_replace_class` has no analogous guard. The OS will produce a valid proof for a state transition that sets a contract's class hash to any arbitrary felt — including `0` or any value that was never declared — because the OS never consults `contract_class_changes` to verify the new hash exists.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once the OS commits a state root in which a contract's `class_hash` is an undeclared value, every future call to that contract will fail at class-lookup time. There is no recovery path: the contract cannot be called to withdraw funds, and no other mechanism in the OS allows overriding the committed class hash without the contract itself executing a valid `replace_class`. Because the contract is uncallable, all ERC-20 balances, NFTs, or other assets held by the contract are permanently frozen. The corrupted state root is published to L1 as part of the verified proof output, making the freeze irreversible at the protocol level.

---

### Likelihood Explanation

Any unprivileged contract deployer can trigger this path:

1. Deploy a contract (e.g., a token vault or DeFi pool) that accepts user deposits.
2. Embed a call to `replace_class(arbitrary_undeclared_hash)` — either intentionally (malicious rug) or accidentally (developer error, analogous to a user calling `lock()` a second time in the reference report).
3. The sequencer includes the transaction; the OS processes it without objection.
4. The proof is verified on L1 with the corrupted class hash committed.

No privileged role is required beyond the ability to deploy and call a contract, which is available to every network participant. The `replace_class` syscall is also exposed to deprecated (Cairo 0) contracts via `deprecated_execute_syscalls.cairo`: [7](#0-6) 

widening the attack surface further.

---

### Recommendation

Inside `execute_replace_class`, after reading `class_hash` from the request, perform a `dict_read` on `contract_class_changes` to confirm the key exists with a non-zero compiled class hash, mirroring the invariant enforced by `execute_declare_transaction`. Concretely, resolve the TODO at line 898 of `syscall_impls.cairo`:

```cairo
// Verify the new class hash has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

Additionally, add an explicit `assert_not_zero(class_hash)` to reject attempts to replace with `UNINITIALIZED_CLASS_HASH = 0`.

---

### Proof of Concept

**Vulnerability class mapping:** The external report shows `createLock()` overwrites an existing lock record without checking for a prior lock, losing the original tokens. Here, `execute_replace_class` overwrites a contract's class hash without checking that the new hash is declared, losing the contract's callability and all funds inside it.

**Step-by-step:**

1. Attacker deploys `VaultContract` (a token vault) — users deposit 1 000 000 STRK into it.
2. Attacker deploys a second transaction that calls `replace_class(0xdeadbeef)` from within `VaultContract`. `0xdeadbeef` is never declared.
3. The OS executes `execute_replace_class`:
   - `class_hash = 0xdeadbeef` is read from the syscall segment.
   - No check against `contract_class_changes` is performed (line 898 TODO).
   - `dict_update` writes `StateEntry(class_hash=0xdeadbeef, ...)` for `VaultContract`.
4. `state_update` in `state.cairo` squashes and commits this entry into the Patricia tree, producing a new global root that includes `VaultContract.class_hash = 0xdeadbeef`.
5. The proof is verified on L1. The new state root is accepted.
6. All subsequent calls to `VaultContract` fail — the class `0xdeadbeef` does not exist in the class tree.
7. The 1 000 000 STRK are permanently frozen. [8](#0-7) [9](#0-8)

func get_contract_state_hash{hash_ptr: HashBuiltin*}(
    class_hash: felt, storage_root: felt, nonce: felt
) -> (hash: felt) {
    const CONTRACT_STATE_HASH_VERSION = 0;
    if (class_hash == UNINITIALIZED_CLASS_HASH) {
        if (storage_root == 0) {
            if (nonce == 0) {
                return (hash=0);
            }
        }
    }

    // Set res = H(H(class_hash, storage_root), nonce).
    let (hash_value) = hash2(class_hash, storage_root);
    let (hash_value) = hash2(hash_value, nonce);

    // Return H(hash_value, CONTRACT_STATE_HASH_VERSION). CONTRACT_STATE_HASH_VERSION must be in the
    // outermost hash to guarantee unique "decoding".
    let (hash) = hash2(hash_value, CONTRACT_STATE_HASH_VERSION);
    return (hash=hash);
}
