### Title
Stale `feemarketParams` Snapshot in `NewDynamicFeeChecker` Allows Cosmos Transactions to Bypass Current Fee Market Base Fee - (File: `ante/evm/fee_checker.go`, `evmd/ante/handler_options.go`)

### Summary
`newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712` capture `feemarketParams` once at app-initialization time and pass a pointer to that stale local copy into `NewDynamicFeeChecker`. Every subsequent Cosmos SDK transaction is fee-checked against the frozen initial base fee rather than the current block's base fee. EVM transactions, by contrast, always read fresh params via `EVMBlockConfig`. The result is a two-path fee-check inconsistency: Cosmos-wrapped transactions can be admitted with fees that would be rejected if they were EVM transactions, mis-accounting fees whenever the dynamic base fee has risen above its initial value.

### Finding Description

`newCosmosAnteHandler` (and its EIP-712 sibling) snapshot module params at construction time:

```go
// evmd/ante/handler_options.go:178-187
func newCosmosAnteHandler(ctx sdk.Context, options HandlerOptions, extra ...sdk.AnteDecorator) sdk.AnteHandler {
    evmParams      := options.EvmKeeper.GetParams(ctx)       // snapshot at init
    feemarketParams := options.FeeMarketKeeper.GetParams(ctx) // snapshot at init
    ...
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
``` [1](#0-0) 

The returned `sdk.AnteHandler` closure is stored once and reused for every transaction. `feemarketParams` is a local variable; `&feemarketParams` is a pointer to that stack-allocated copy, which is never refreshed.

Inside `NewDynamicFeeChecker`, the base fee is derived exclusively from this frozen pointer:

```go
// ante/evm/fee_checker.go:56
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
``` [2](#0-1) 

`types.GetBaseFee` simply calls `feemarketParams.GetBaseFee()` on the captured snapshot:

```go
// x/evm/types/utils.go:244-253
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    ...
    baseFee := feemarketParams.GetBaseFee()
    ...
}
``` [3](#0-2) 

The EVM ante handler takes the opposite, correct approach — it calls `EVMBlockConfig` on every transaction, which reads fresh params from the keeper each block:

```go
// evmd/ante/handler_options.go:88-95
blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
...
feemarketParams := &blockCfg.FeeMarketParams
baseFee         := blockCfg.BaseFee
``` [4](#0-3) 

`EVMBlockConfig` in turn reads the live base fee from the fee-market keeper:

```go
// x/evm/keeper/config.go:85-99
feemarketParams := k.feeMarketKeeper.GetParams(ctx)
...
baseFee = feemarketParams.GetBaseFee()
``` [5](#0-4) 

The fee-market keeper's `GetBaseFee` reads from the KV store, which is updated every block by `CalculateBaseFee` / `SetBaseFee`:

```go
// x/feemarket/keeper/params.go:57-68
func (k Keeper) GetBaseFee(ctx sdk.Context) *big.Int {
    params := k.GetParams(ctx)
    ...
    baseFee := params.BaseFee.BigInt()
    ...
}
``` [6](#0-5) 

### Impact Explanation

The EIP-1559 base fee rises dynamically when blocks are above the gas target. After any period of sustained high usage, the live base fee will be materially higher than the value frozen at node startup. Any Cosmos SDK transaction (including EIP-712 signed transactions) routed through `newCosmosAnteHandler` is fee-checked against the stale, lower base fee. The `feeCap < baseFee` guard in `NewDynamicFeeChecker` (line 86) therefore passes with a fee that would be rejected for an equivalent EVM transaction. The effective fee deducted from the sender is computed from the stale base fee, so the sender pays less than the protocol requires — a direct fee mis-accounting. This matches the allowed High impact: *"fee market, ante handler … bug that permits … valid user funds/fees to be mis-accounted."*

### Likelihood Explanation

Medium. The base fee starts at its genesis value and increases whenever blocks are above the gas target. On any active chain the base fee will diverge from the initialization snapshot within minutes to hours of sustained load. An attacker who observes the current on-chain base fee and the node's startup base fee can craft Cosmos transactions that pay only the lower startup fee and have them accepted through `newCosmosAnteHandler`, while identical EVM transactions would be rejected.

### Recommendation

Remove the snapshot pattern. `NewDynamicFeeChecker` should read the base fee from the keeper at call time, not from a frozen pointer. The simplest fix is to pass the `FeeMarketKeeper` interface into `NewDynamicFeeChecker` and call `keeper.GetBaseFee(ctx)` inside the returned closure, mirroring what `newEthAnteHandler` already does via `EVMBlockConfig`. Alternatively, construct the Cosmos ante handler inside a per-block wrapper so that `feemarketParams` is refreshed each block, consistent with the EVM path.

### Proof of Concept

1. Node starts; genesis base fee = 1 000 000 000 (1 Gwei). `newCosmosAnteHandler` snapshots `feemarketParams.BaseFee = 1e9`.
2. Network sustains high load for several blocks. `CalculateBaseFee` raises the live base fee to 5 000 000 000 (5 Gwei) and stores it via `SetBaseFee` → `SetParams`.
3. Attacker submits a Cosmos SDK transaction (e.g., a `MsgEthereumTx` wrapped in a Cosmos tx, or any Cosmos msg with `ExtensionOptionDynamicFeeTx`) with `feeCap = 1e9`.
4. `NewDynamicFeeChecker` evaluates `feeCap (1e9) >= baseFee (1e9, stale)` → passes. The transaction is admitted and the sender pays fees at 1 Gwei.
5. An equivalent EVM transaction with `GasFeeCap = 1e9` is evaluated by `newEthAnteHandler` → `EVMBlockConfig` returns live `baseFee = 5e9` → `VerifyFee` rejects it with "the tx gasfeecap is lower than the tx baseFee".
6. The Cosmos path accepts what the EVM path rejects, and the sender is charged 5× less than the protocol requires.

func (k Keeper) GetBaseFee(ctx sdk.Context) *big.Int {
	params := k.GetParams(ctx)
	if params.NoBaseFee {
		return nil
	}

	baseFee := params.BaseFee.BigInt()
	if baseFee == nil || baseFee.Sign() == 0 {
		// try v1 format
		return k.GetBaseFeeV1(ctx)
	}
	return baseFee
}
