### Title
Unchecked Fee Token Transfer Return Value in `charge_fee` Allows Silent Fee Bypass - (File: `execution/transaction_impls.cairo`)

### Summary
The `charge_fee` function in the StarkNet OS executes the fee token's `transfer` entry point but completely discards the return value. For deprecated (CairoZero) fee token implementations that return `false` on failure instead of reverting, a transaction sender with insufficient balance can have their transaction processed without paying fees, resulting in direct loss of funds for the sequencer.

### Finding Description
In `charge_fee`, the call to `non_reverting_select_execute_entry_point_func` for the fee token transfer discards all return values:

```cairo
let remaining_gas = DEFAULT_INITIAL_GAS_COST;
non_reverting_select_execute_entry_point_func{remaining_gas=remaining_gas}(
    block_context=block_context, execution_context=&execution_context
);
return ();
``` [1](#0-0) 

The function signature returns `(retdata_size: felt, retdata: felt*, is_deprecated: felt)`, but none of these are captured or checked at the call site in `charge_fee`. [2](#0-1) 

`non_reverting_select_execute_entry_point_func` only asserts `is_reverted = 0` — it catches panics/reverts, but **not** a silent `false` return from a deprecated CairoZero ERC20 that signals failure without reverting. [3](#0-2) 

Contrast this with how `run_validate` and `__validate_declare__` handle their return values — they explicitly assert `retdata[0] = VALIDATED`: [4](#0-3) [5](#0-4) 

The fee transfer has no equivalent success check. The `TRANSFER_ENTRY_POINT_SELECTOR` is used to call the fee token: [6](#0-5) 

The `charge_fee` function is invoked for every account transaction type — invoke, deploy account, and declare: [7](#0-6) [8](#0-7) [9](#0-8) 

### Impact Explanation
**Critical — Direct loss of funds.** If the fee token is a deprecated (CairoZero) ERC20 contract that returns `(success: felt)` where `success = 0` on insufficient balance without reverting (a valid CairoZero pattern), the OS has no mechanism to detect the failed transfer. The transaction is processed and committed to the block with state changes applied, but the sequencer receives no fee. Since `charge_fee` is called after execution and validation for every non-zero-fee transaction, an attacker can drain sequencer revenue across all transaction types.

### Likelihood Explanation
**Medium.** The StarkNet ETH fee token is a deployed deprecated (CairoZero) contract. CairoZero ERC20 implementations commonly use the `(success: felt)` return pattern. If the deployed fee token contract has any code path that returns `false` without panicking (e.g., on zero-amount transfers, overflow edge cases, or specific balance conditions), this is directly exploitable by any transaction sender. The attacker-controlled input is the transaction itself — any unprivileged user submitting a transaction with `max_fee > 0` but insufficient balance triggers this path.

### Recommendation
After calling `non_reverting_select_execute_entry_point_func` for the fee transfer, capture and validate the return value. For non-deprecated contracts (`is_deprecated == 0`), assert that `retdata_size >= 1` and `retdata[0] != 0` (transfer returned `true`). This mirrors the pattern already used for `__validate__`, `__validate_declare__`, and `__validate_deploy__`:

```cairo
let (retdata_size, retdata, is_deprecated) = non_reverting_select_execute_entry_point_func{
    remaining_gas=remaining_gas
}(block_context=block_context, execution_context=&execution_context);
if (is_deprecated == 0) {
    assert retdata_size = 1;
    assert retdata[0] = 1;  // transfer must return true
}
```

### Proof of Concept
1. The attacker submits a V3 transaction (invoke/declare/deploy_account) with `max_fee > 0` but holds zero fee token balance.
2. The OS executes the transaction, runs `__validate__` and `__execute__`, then calls `charge_fee`.
3. `charge_fee` constructs a `TransferCallData` with `amount = low_actual_fee` and calls the fee token's `transfer` entry point via `non_reverting_select_execute_entry_point_func`.
4. The deprecated CairoZero fee token's `transfer` function checks the balance, finds it insufficient, and returns `(success=0)` **without reverting** (a valid CairoZero pattern).
5. `non_reverting_select_execute_entry_point_func` sees `is_reverted = 0` and returns `(retdata_size=1, retdata=[0], is_deprecated=1)`.
6. `charge_fee` discards all return values and returns normally.
7. The OS commits the block. The transaction's state changes are finalized. The sequencer receives no fee. The attacker's transaction executed for free. [10](#0-9)

// Charges a fee from the user.
// If max_fee is not 0, validates that the selector matches the entry point of an account contract
// and executes an ERC20 transfer on the behalf of that account contract.
//
// Arguments:
// block_context - a global context that is fixed throughout the block.
// tx_execution_context - The execution context of the transaction that pays the fee.
func charge_fee{
    range_check_ptr,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, tx_execution_context: ExecutionContext*) {
    alloc_locals;

    local tx_info: TxInfo* = tx_execution_context.execution_info.tx_info;
    let max_fee = compute_max_possible_fee(tx_info=tx_info);

    if (max_fee == 0) {
        return ();
    }

    local low_actual_fee;
    %{ LoadActualFee %}
    local calldata: TransferCallData = TransferCallData(
        recipient=block_context.block_info_for_execute.sequencer_address,
        amount=Uint256(low=low_actual_fee, high=0),
    );

    // Verify that the charged amount is not larger than the transaction's max_fee field.
    assert_nn_le(calldata.amount.low, max_fee);

    // TODO(ilya, 01/01/2026): Consider caching the fee_token_class_hash.
    local fee_token_address = block_context.os_global_context.starknet_os_config.fee_token_address;
    let (fee_state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
        key=fee_token_address
    );
    let (__fp__, _) = get_fp_and_pc();
    // Use block_info directly from block_context, so that charge_fee will always run in
    // execute-mode rather than validate-mode.
    local execution_context: ExecutionContext = ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=fee_state_entry.class_hash,
        calldata_size=TransferCallData.SIZE,
        calldata=&calldata,
        execution_info=new ExecutionInfo(
            block_info=block_context.block_info_for_execute,
            tx_info=tx_info,
            caller_address=tx_info.account_contract_address,
            contract_address=fee_token_address,
            selector=TRANSFER_ENTRY_POINT_SELECTOR,
        ),
        deprecated_tx_info=tx_execution_context.deprecated_tx_info,
    );

    let remaining_gas = DEFAULT_INITIAL_GAS_COST;
    non_reverting_select_execute_entry_point_func{remaining_gas=remaining_gas}(
        block_context=block_context, execution_context=&execution_context
    );
    return ();
}
