### Title
`ProofOfInclusion::valid()` Tautological Root-Hash Check Allows Forged Proof Acceptance — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

---

### Summary

`ProofOfInclusion::valid()` contains a final check (`existing_hash == self.root_hash()`) that is a logical tautology: it is always `true` whenever the loop completes without returning `false`. As a result, the function only verifies the internal hash-chain consistency of the proof, but never validates the proof's root hash against any externally known tree root. An attacker who can supply a `ProofOfInclusion` (a `Streamable` type deserializable from arbitrary bytes) can construct a forged proof for any key, with any fabricated root hash, and `valid()` will return `true`.

---

### Finding Description

**Root cause — tautological final check:**

```rust
// crates/chia-datalayer/src/merkle/proof_of_inclusion.rs  lines 40-58
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {   // (A)
            return false;
        }

        existing_hash = calculated_hash;              // (B)
    }

    existing_hash == self.root_hash()                 // (C)  ← always true
}
```

After the loop, `existing_hash` equals the last `calculated_hash`, which equals the last `layer.combined_hash` (guaranteed by check A + assignment B). `root_hash()` is defined as:

```rust
// lines 32-38
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← same value as existing_hash
    } else {
        self.node_hash
    }
}
```

So line (C) reduces to `last.combined_hash == last.combined_hash`, which is always `true`. The function never compares the proof's root against any externally trusted tree root.

**Forged proof construction:**

Because `ProofOfInclusion` is `Streamable` (deserializable from raw bytes) and exposed via Python bindings (`from_bytes`, `valid`), an attacker can craft:

```
node_hash   = sha256(fake_leaf_data)          // any 32-byte value
other_hash  = <arbitrary 32 bytes>
combined_hash = calculate_internal_hash(node_hash, Side::Left, other_hash)
```

This single-layer `ProofOfInclusion` passes `valid()` with a completely fabricated root hash, proving inclusion of a key that is not in any real tree.
