### Title
Pause Check Missing in `fin_transfer_callback` Allows Transfer Finalization to Bypass Bridge Pause - (File: `near/omni-bridge/src/lib.rs`)

### Summary

The NEAR bridge's `fin_transfer` function enforces a pause guard via `#[pause(except(roles(Role::DAO)))]`, but its asynchronous callback `fin_transfer_callback` carries no equivalent pause check. Because NEAR executes cross-contract callbacks as independent transactions, a pause applied between the outer call and the callback execution is silently ignored, and the transfer is finalized unconditionally.

### Finding Description

`fin_transfer` (line 673) is decorated with both `#[trusted_relayer]` and `#[pause(except(roles(Role::DAO)))]`:

```rust
#[payable]
#[trusted_relayer]
#[pause(except(roles(Role::DAO)))]
pub fn fin_transfer(&mut self, #[serializer(borsh)] args: FinTransferArgs) -> Promise {
    ...
    main_promise.then(
        Self::ext(env::current_account_id())
            ...
            .fin_transfer_callback(
                &args.storage_deposit_actions,
                env::predecessor_account_id(),
            ),
    )
}
``` [1](#0-0) 

The callback that performs the actual state-changing work — verifying the proof result, decrementing locked-token balances, minting bridge tokens, and recording the finalized transfer — carries only `#[private]` and no pause guard:

```rust
#[private]
#[payable]
pub fn fin_transfer_callback(
    &mut self,
    ...
) -> PromiseOrValue<Nonce> {
    // No pause check here
    let Ok(ProverResult::InitTransfer(init_transfer)) = Self::decode_prover_result(0) else { ... };
    ...
    // Proceeds to mint tokens / release locked tokens
}
``` [2](#0-1) 

In NEAR's async execution model, the callback is a separate receipt that is enqueued and executed in a later block. Any pause applied after `fin_transfer` is submitted but before `fin_transfer_callback` executes is invisible to the callback. The same structural gap exists in `sign_transfer` / `sign_transfer_callback` and `claim_fee` / `claim_fee_callback`: [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) 

### Impact Explanation

`fin_transfer_callback` calls `process_fin_transfer_to_near` or `process_fin_transfer_to_other_chain`, which mint bridge tokens to the recipient or release locked collateral. Executing these paths while the bridge is paused directly contradicts the purpose of the pause mechanism and can result in:

- Unauthorized minting of bridge tokens during a security incident (e.g., after a double-spend is detected on the source chain and the bridge is paused to contain damage).
- Unauthorized release of locked native tokens to a recipient when the bridge operator intended to halt all outflows.

This matches the allowed impact: **permanent or unauthorized release of bridged assets** and **proof/verification bypass enabling unauthorized transfer finalization**. [7](#0-6) 

### Likelihood Explanation

The window between `fin_transfer` submission and `fin_transfer_callback` execution is at least one NEAR block (~1 second). Emergency pauses are typically triggered in response to an on-chain event (e.g., anomalous proof submission, exploit detected), meaning the pause and the in-flight callback can easily overlap. A trusted relayer need only have submitted `fin_transfer` one block before the pause is applied for the bypass to occur. The `#[trusted_relayer]` requirement limits the initiating caller to registered relayers, but does not prevent the race condition once the outer call is already in flight.

### Recommendation

Add a pause check at the top of `fin_transfer_callback`, `sign_transfer_callback`, and `claim_fee_callback`. For `fin_transfer_callback`, the check should mirror the one on the outer function:

```rust
#[private]
#[payable]
pub fn fin_transfer_callback(...) -> PromiseOrValue<Nonce> {
    // Add pause guard analogous to the outer function
    require!(
        !self.is_paused(/* fin_transfer pause flag */),
        BridgeError::Paused.as_ref()
    );
    ...
}
```

Because `#[private]` callbacks cannot carry the `#[pause]` proc-macro directly (it would be a no-op since only the contract itself can call them), the check must be an explicit inline `require!` or a helper call at the start of the callback body.

### Proof of Concept

1. Trusted relayer calls `fin_transfer` with a valid proof at NEAR block N. The pause check passes; `verify_proof` is dispatched and `fin_transfer_callback` is enqueued as a receipt for block N+1.
2. At block N (same block or N+1 before the callback receipt is processed), the bridge operator calls `pause` / `pauseAll` in response to a detected exploit.
3. At block N+1, `fin_transfer_callback` executes. It reads no pause flag, decodes the prover result, decrements locked tokens, and mints bridge tokens to the recipient — completing the transfer as if the bridge were fully operational.
4. The recipient receives minted tokens backed by source-chain collateral that the operator intended to freeze, defeating the entire purpose of the emergency pause. [2](#0-1)

    #[payable]
    pub fn fin_transfer_callback(
        &mut self,
        #[serializer(borsh)] storage_deposit_actions: &Vec<StorageDepositAction>,
        #[serializer(borsh)] predecessor_account_id: AccountId,
    ) -> PromiseOrValue<Nonce> {
        let Ok(ProverResult::InitTransfer(init_transfer)) = Self::decode_prover_result(0) else {
            env::panic_str(BridgeError::InvalidProofMessage.to_string().as_str())
        };
        require!(
            self.factories
                .get(&init_transfer.emitter_address.get_chain())
                == Some(init_transfer.emitter_address),
            BridgeError::UnknownFactory.as_ref()
        );

        let decimals = self
            .token_decimals
            .get(&init_transfer.token)
            .near_expect(BridgeError::TokenDecimalsNotFound);

        let destination_nonce =
            self.get_next_destination_nonce(init_transfer.recipient.get_chain());
        let transfer_message = TransferMessage {
            origin_nonce: init_transfer.origin_nonce,
            token: init_transfer.token,
            amount: Self::denormalize_amount(init_transfer.amount.0, decimals).into(),
            recipient: init_transfer.recipient,
            fee: Self::denormalize_fee(&init_transfer.fee, decimals),
            sender: init_transfer.sender,
            msg: init_transfer.msg,
            destination_nonce,
            origin_transfer_id: None,
        };

        if let OmniAddress::Near(recipient) = transfer_message.recipient.clone() {
            self.process_fin_transfer_to_near(
                recipient,
                &predecessor_account_id,
                transfer_message,
                storage_deposit_actions,
            )
            .into()
        } else {
            self.process_fin_transfer_to_other_chain(predecessor_account_id, transfer_message);
            PromiseOrValue::Value(destination_nonce)
        }
    }
