### Title
`batch_insert` Bypasses Duplicate-Key and Duplicate-Hash Validation, Corrupting DataLayer Merkle Tree Root - (File: `crates/chia-datalayer/src/merkle/blob.rs`)

### Summary
`MerkleBlob::batch_insert()` contains a fast-path bulk-insertion loop that skips the duplicate `KeyId` and duplicate `Hash` checks enforced by the single-item `MerkleBlob::insert()`. Supplying duplicate keys or hashes via the Python binding `py_batch_insert` silently writes multiple leaf nodes with the same key/hash into the blob while the `block_status_cache` retains only the last-written entry. The resulting tree has a corrupted root hash and inconsistent proof-of-inclusion state.

### Finding Description

`MerkleBlob::insert()` guards against duplicates at lines 369–374:

```rust
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`batch_insert()` uses `insert()` only for the first two items when `leaf_count <= 1` (the bootstrap path). Once the tree has two or more leaves, all remaining items in the batch are written directly via `insert_entry_to_blob()` with **no duplicate check**:

```rust
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ... Node::Leaf(LeafNode { hash, key, value, ... }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
``` [2](#0-1) 

`insert_entry_to_blob()` calls `block_status_cache.add_leaf()`, which performs a silent `HashMap::insert` — overwriting any existing entry for the same key or hash without error:

```rust
fn add_leaf(&mut self, index: TreeIndex, leaf: LeafNode) {
    self.free_indexes.shift_remove(&index);
    self.key_to_index.insert(leaf.key, index);
    self.leaf_hash_to_index.insert(leaf.hash, index);
}
``` [3](#0-2) 

The Python binding `py_batch_insert` is directly reachable by untrusted callers:

```rust
#[pyo3(name = "batch_insert")]
pub fn py_batch_insert(&mut self, keys_values: Vec<(KeyId, ValueId)>, hashes: Vec<Hash>) -> PyResult<()> {
    ...
    self.batch_insert(zip(keys_values, hashes).collect())?;
    Ok(())
}
``` [4](#0-3) 

### Impact Explanation

When duplicate `KeyId` or `Hash` values are passed in the batch:

1. The blob contains **multiple leaf nodes** with the same key or hash, but the `block_status_cache` maps that key/hash to only the last-written index.
2. The Merkle tree's internal hash structure is computed over all nodes including the duplicates, producing a **root hash that does not correspond to any valid unique-key tree**.
3. `get_proof_of_inclusion()` traverses from the cache-tracked index and produces a proof whose `root_hash()` is derived from the corrupted tree — the proof appears internally self-consistent but proves membership in a tree with a forged/inconsistent root.
4. A DataLayer client comparing the returned root hash against a stored commitment will either accept a proof for a key mapped to the wrong value, or accept a root hash that encodes hidden duplicate state.

This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.** [5](#0-4) 

### Likelihood Explanation

The Python binding is the primary DataLayer interface. Any caller constructing a batch with repeated keys (e.g., an update batch that accidentally includes the same key twice, or a malicious DataLayer peer supplying a crafted batch) triggers the bug. No privileged access is required — only the ability to call `batch_insert` with attacker-controlled input. [6](#0-5) 

### Recommendation

Replace the raw `insert_entry_to_blob` loop in the fast path with calls to `self.insert()`, or add explicit duplicate checks at the start of `batch_insert` before entering the fast path:

```rust
// Before the fast-path loop, validate all inputs:
for ((key, _value), hash) in &keys_values_hashes {
    if self.block_status_cache.contains_key(*key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(hash) {
        return Err(Error::HashAlreadyPresent());
    }
}
// Also check for intra-batch duplicates using a local HashSet.
```

Additionally, `add_leaf` in `BlockStatusCache` should assert or error on silent overwrites to prevent future regressions. [7](#0-6) 

### Proof of Concept

```python
from chia_datalayer import MerkleBlob, KeyId, ValueId
import hashlib

blob = MerkleBlob(blob=bytearray())
# Pre-insert 2 items so leaf_count > 1, activating the fast path
blob.insert(KeyId(1), ValueId(1), hashlib.sha256(b"seed1").digest())
blob.insert(KeyId(2), ValueId(2), hashlib.sha256(b"seed2").digest())

# batch_insert with a duplicate KeyId — no error is raised
dup_key = KeyId(99)
hash_a = hashlib.sha256(b"a").digest()
hash_b = hashlib.sha256(b"b").digest()  # different hash, same key
blob.batch_insert(
    [(dup_key, ValueId(10)), (dup_key, ValueId(20)), (KeyId(3), ValueId(3))],
    [hash_a, hash_b, hashlib.sha256(b"seed3").digest()]
)
blob.calculate_lazy_hashes()

# The blob now contains two leaf nodes with key=99.
# The cache tracks only the last-written one (ValueId(20), hash_b).
# The root hash is computed over a tree with both leaves present.
proof = blob.get_proof_of_inclusion(dup_key)
# proof.valid() returns True, but the root encodes a corrupted tree
# containing a hidden duplicate leaf (ValueId(10), hash_a) that is
# unreachable through the cache but contributes to the root hash.
print(proof.valid())  # True — inconsistent state accepted
``` [2](#0-1)

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
