### Title
Missing Backward-Compatibility Branch in `calculate_global_state_root` Produces Incorrect Global State Root — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/state/commitment.cairo`)

---

### Summary

`calculate_global_state_root` documents three distinct cases in its comment but only implements two of them. When `contract_class_root == 0` and `contract_state_root != 0` (the backward-compatibility case), the function falls through to the general hash branch and computes `H(GLOBAL_STATE_VERSION, contract_state_root, 0)` instead of returning `contract_state_root` directly. This produces a global state root that diverges from what the L1 verifier expects, causing proof rejection and a network halt.

---

### Finding Description

The function comment at lines 31–37 explicitly specifies three cases:

```
// If both the contract class and contract state trees are empty, the global root is set to 0.
// If the contract class tree is empty, the global state root is equal to the
// contract state root (for backward compatibility);
// Otherwise, the global root is obtained by:
//     global_root = H(GLOBAL_STATE_VERSION, contract_state_root, contract_class_root).
```

The implementation at lines 41–48 only handles cases 1 and 3:

```cairo
if (contract_state_root == 0 and contract_class_root == 0) {
    return (global_root=0);                          // Case 1 ✓
}
// Case 2 (contract_class_root == 0, contract_state_root != 0) is MISSING
tempvar elements: felt* = new (GLOBAL_STATE_VERSION, contract_state_root, contract_class_root);
let (global_root) = poseidon_hash_many(n=3, elements=elements);  // Case 3 ✓
return (global_root=global_root);
```

When `contract_class_root == 0` but `contract_state_root != 0`, the code falls into the `poseidon_hash_many` branch and returns `H(GLOBAL_STATE_VERSION, contract_state_root, 0)` instead of `contract_state_root`.

This function is called twice in `state_update` (state.cairo lines 90–97) — once for the initial root and once for the final root — and the resulting `CommitmentUpdate` is serialized directly into the OS output header (output.cairo lines 161–162) as the state roots committed to L1. ### Impact Explanation

The global state root is the single value committed to L1 and verified by the StarkNet core contract. If the OS computes `H(GLOBAL_STATE_VERSION, contract_state_root, 0)` while the L1 verifier applies the backward-compatible formula and expects `contract_state_root` directly, every proof generated during the affected state will be rejected by L1. No new blocks can be finalized, resulting in a **total network halt**. Alternatively, if different OS versions or aggregator paths apply different formulas, nodes will disagree on the canonical state root, producing an **unintended chain split**. [4](#0-3) 

---

### Likelihood Explanation

The trigger condition — `contract_class_root == 0` with `contract_state_root != 0` — occurs when the contract class Patricia tree is empty (no classes ever declared) while the contract state tree is non-empty. This is the exact state during network bootstrapping: the bootstrapping flow in `execute_declare_transaction` (transaction_impls.cairo lines 764–775) allows declaring classes with `sender_address == 'BOOTSTRAP'`, `nonce == 0`, and `max_fee == 0` before any class tree root is established. Any block processed in this window — including the very first block containing state changes but no class declarations — triggers the incorrect branch. Because bootstrapping is a necessary protocol step for every new StarkNet deployment, this path is reachable with certainty on any fresh chain. [5](#0-4) 

---

### Recommendation

Add the missing backward-compatibility branch before the general hash computation:

```cairo
func calculate_global_state_root{poseidon_ptr: PoseidonBuiltin*, range_check_ptr}(
    contract_state_root: felt, contract_class_root: felt
) -> (global_root: felt) {
    if (contract_state_root == 0 and contract_class_root == 0) {
        return (global_root=0);
    }

    // Backward compatibility: if the class tree is empty, return the state root directly.
    if (contract_class_root == 0) {
        return (global_root=contract_state_root);
    }

    tempvar elements: felt* = new (GLOBAL_STATE_VERSION, contract_state_root, contract_class_root);
    let (global_root) = poseidon_hash_many(n=3, elements=elements);
    return (global_root=global_root);
}
```

This aligns the implementation with the documented specification and with what the L1 verifier expects for the backward-compatible state root format. [6](#0-5) 

---

### Proof of Concept

1. Deploy a fresh StarkNet chain. At genesis, `contract_class_root == 0` (empty class tree).
2. Submit any transaction that modifies contract state (e.g., an L1→L2 message handler or a deploy), making `contract_state_root != 0`.
3. The OS calls `state_update`, which calls `calculate_global_state_root(contract_state_root=X, contract_class_root=0)` where `X != 0`.
4. The missing branch causes the function to compute `H(GLOBAL_STATE_VERSION, X, 0)` instead of returning `X`.
5. This incorrect value is written into `CommitmentUpdate.final_root` and serialized into the OS output header.
6. The L1 StarkNet core contract, applying the backward-compatible formula, expects `X` but receives `H(GLOBAL_STATE_VERSION, X, 0)`.
7. Proof verification fails on L1; no further blocks can be finalized — network halt. [7](#0-6)

func calculate_global_state_root{poseidon_ptr: PoseidonBuiltin*, range_check_ptr}(

        // The state is empty.
        return (global_root=0);

    let (global_root) = poseidon_hash_many(n=3, elements=elements);
    return (global_root=global_root);
}
