### Title
Stale-Pending Fallback Silently Substitutes `NonzeroGasPrice::MIN` for Data-Gas and L2-Gas Prices in RPC Fee Estimation — (`crates/apollo_rpc/src/v0_8/api/api_impl.rs`, `crates/apollo_rpc_execution/src/lib.rs`)

---

### Summary

When the in-memory pending block is stale (its `parent_block_hash` does not match the latest committed block hash), the RPC layer silently falls back to a `DeprecatedPendingBlock` that carries **zero** for both `l1_data_gas_price` and `l2_gas_price`. The execution layer then substitutes `NonzeroGasPrice::MIN` (= 1) for every zero price. As a result, `starknet_estimateFee`, `starknet_simulateTransactions`, and `starknet_estimateMessageFee` called with `block_id = "pending"` return authoritative-looking fee estimates that are orders-of-magnitude too low for the data-gas and L2-gas components, while the actual next block will use the real, much higher prices computed by the sequencer.

---

### Finding Description

**Step 1 – Stale-pending detection and fallback construction**

`read_pending_data` compares the cached pending block's `parent_block_hash` against the latest committed block hash. When they differ it constructs a synthetic `DeprecatedPendingBlock`:

```rust

**Step 3 – `create_block_context` silently replaces zero with `NonzeroGasPrice::MIN`**

```rust
// crates/apollo_rpc_execution/src/lib.rs  lines 382-396
l1_data_gas_price: NonzeroGasPrice::new(l1_data_gas_price.price_in_wei)
    .unwrap_or(NonzeroGasPrice::MIN),   // ← 1 instead of real price
l2_gas_price: NonzeroGasPrice::new(l2_gas_price.price_in_wei)
    .unwrap_or(NonzeroGasPrice::MIN),   // ← 1 instead of real price
``` [3](#0-2) 

No error is returned; the `BlockContext` is built with price = 1 and execution proceeds normally.

**Step 4 – The actual next block uses real, transformed prices**

The proposer computes the next block's gas prices through `apply_fee_transformations`, which adds a tip, applies a multiplier to blob fees, and clamps to configured min/max bounds:

```rust
// crates/apollo_consensus_orchestrator/src/utils.rs  lines 272-285
pub(crate) fn apply_fee_transformations(price_info: &mut PriceInfo, gas_price_params: &GasPriceParams) {
    price_info.base_fee_per_gas = price_info.base_fee_per_gas
        .saturating_add(gas_price_params.l1_gas_tip_wei)
        .clamp(gas_price_params.min_l1_gas_price_wei, gas_price_params.max_l1_gas_price_wei);
    price_info.blob_fee = GasPrice(
        (gas_price_params.l1_data_gas_price_multiplier * price_info.blob_fee.0).to_integer(),
    ).clamp(gas_price_params.min_l1_data_gas_price_wei, gas_price_params.max_l1_data_gas_price_wei);
}
``` [4](#0-3) 

The real `l1_data_gas_price` and `l2_gas_price` for the next block are therefore bounded by configured minimums (e.g., `min_l1_data_gas_price_wei`) that are far above 1. The "view" path (stale-pending RPC) and the "write" path (actual block execution) diverge on exactly these two prices.

---

### Impact Explanation

`starknet_estimateFee`, `starknet_simulateTransactions`, and `starknet_estimateMessageFee` with `block_id = "pending"` return fee estimates whose `l1_data_gas_price` and `l2_gas_price` components are `NonzeroGasPrice::MIN` (= 1) instead of the real values. For v3 transactions using `AllResources` bounds, a user who sets `l1_data_gas.max_price_per_unit` or `l2_gas.max_price_per_unit` from this estimate will have their transaction rejected by the sequencer with `MaxGasPriceTooLow` when the actual block is built. The RPC returns no error — the wrong value looks authoritative.

This matches the allowed impact: **High — RPC fee estimation returns an authoritative-looking wrong value.**

---

### Likelihood Explanation

The stale-pending condition occurs naturally and repeatedly: every time a new block is committed, the cached pending block becomes stale until the sequencer publishes a fresh pending block for the next height. On a busy network this window is short; on a slow or recovering node it can persist for many seconds. Any unprivileged caller of `starknet_estimateFee` with `block_id = "pending"` during this window receives the wrong estimate.

---

### Recommendation

In `read_pending_data`, when constructing the stale-pending fallback, populate `l1_data_gas_price` and `l2_gas_price` from the latest committed block header rather than leaving them at zero:

```rust
block: PendingBlockOrDeprecated::Current(PendingBlock {
    parent_block_hash: latest_header.block_hash,
    l1_gas_price: latest_header.block_header_without_hash.l1_gas_price,
    l1_data_gas_price: latest_header.block_header_without_hash.l1_data_gas_price,
    l2_gas_price: latest_header.block_header_without_hash.l2_gas_price,
    l1_da_mode: latest_header.block_header_without_hash.l1_da_mode,
    timestamp: latest_header.block_header_without_hash.timestamp,
    sequencer_address: latest_header.block_header_without_hash.sequencer,
    starknet_version: latest_header.block_header_without_hash.starknet_version.to_string(),
    ..Default::default()
}),
```

This ensures the fee estimate uses the most recently committed prices rather than `NonzeroGasPrice::MIN`, making the "view" path consistent with the "write" path.

---

### Proof of Concept

```
1. Commit block N (latest committed block has l1_data_gas_price = 1000 wei, l2_gas_price = 5000 fri).
2. The pending block cache still holds a block whose parent_block_hash = block N-1's hash
   (stale condition — has not yet been refreshed by the sequencer).
3. Call starknet_estimateFee([v3_invoke_tx], [], "pending").
4. read_pending_data detects parent_block_hash mismatch → constructs DeprecatedPendingBlock.
5. DeprecatedPendingBlock.l1_data_gas_price() → GasPricePerToken { price_in_wei: 0, price_in_fri: 0 }.
6. create_block_context: NonzeroGasPrice::new(0).unwrap_or(NonzeroGasPrice::MIN) → 1.
7. Fee estimate returned: l1_data_gas_price = 1, l2_gas_price = 1.
8. User sets resource_bounds.l1_data_gas.max_price_per_unit = 1 based on estimate.
9. Actual block N+1 is built with l1_data_gas_price = 1000 wei.
10. Transaction rejected: MaxGasPriceTooLow { resource: L1DataGas, max_gas_price: 1, actual_gas_price: 1000 }.
