### Title
`validate_compiled_class_facts` Uses `blake_compiled_class_hash` Instead of `poseidon_compiled_class_hash`, Silently Replacing the Protocol-Standard Hash Safeguard — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/contract_class/compiled_class.cairo`)

---

### Summary

`compiled_class.cairo` imports and calls `blake_compiled_class_hash` (Blake2s) to validate compiled class hashes, while the protocol-standard implementation `poseidon_compiled_class_hash` (Poseidon) exists in the same directory. The OS appears to enforce that `compiled_class_fact.hash` equals the canonical hash of the compiled class bytes, but the hash function used is Blake2s — not the Poseidon function that the StarkNet protocol specifies for compiled class hashes. This is a direct analog to the external report: a perceived safeguard (correct hash binding) exists in name but uses the wrong underlying implementation.

---

### Finding Description

Two parallel implementations of `compiled_class_hash` exist in the codebase:

- `poseidon_compiled_class_hash.cairo` — uses Poseidon, the protocol-standard hash for compiled class hashes per the StarkNet specification.
- `blake_compiled_class_hash.cairo` — uses Blake2s, a newer addition.

`compiled_class.cairo` imports only the Blake2s variant:

```cairo
from starkware.starknet.core.os.contract_class.blake_compiled_class_hash import (
    compiled_class_hash as blake_compiled_class_hash,
)
```

And uses it exclusively in `validate_compiled_class_facts`:

```cairo
let (hash) = blake_compiled_class_hash(compiled_class, full_contract=FALSE);
...
assert compiled_class_fact.hash = hash;
``` [1](#0-0) [2](#0-1) 

This means the OS enforces `compiled_class_fact.hash = blake2s(compiled_class_bytes)`, not `poseidon(compiled_class_bytes)`. The `compiled_class_fact.hash` is the value stored in `contract_class_changes` during `execute_declare_transaction` and ultimately committed into the class commitment tree via `get_contract_class_leaf_hash`. [3](#0-2) [4](#0-3) 

The class commitment tree leaf hash is `poseidon(CONTRACT_CLASS_LEAF_VERSION, compiled_class_hash)`, where `compiled_class_hash` is whatever value the OS accepted — which is now a Blake2s hash, not a Poseidon hash as the protocol specifies.

---

### Impact Explanation

**High — Unintended chain split / network partition.**

The StarkNet protocol specifies that the `compiled_class_hash` field in a declare transaction must be the Poseidon hash of the compiled class (per SNIP-9 and the class commitment scheme). The OS now enforces Blake2s instead. This creates a divergence between:

1. What the OS proof asserts as valid (Blake2s-bound compiled class hashes in the state tree).
2. What the protocol specification and off-chain verifiers/clients expect (Poseidon-bound compiled class hashes).

Any node or verifier that independently checks compiled class hashes against the Poseidon specification will disagree with the OS-produced state root, causing a chain split. Additionally, legitimate declare transactions that supply a Poseidon-based `compiled_class_hash` (as the protocol requires) will fail OS validation, halting new class declarations and potentially the network.

---

### Likelihood Explanation

**High.** Every declare transaction triggers `validate_compiled_class_facts`. Any class declarer — an unprivileged protocol participant — submitting a declare transaction causes this code path to execute. No special privileges, leaked keys, or operator collusion are required. The mismatch is structural and fires on every class declaration.

---

### Recommendation

Change `compiled_class.cairo` to import and use `poseidon_compiled_class_hash` instead of `blake_compiled_class_hash`:

```cairo
- from starkware.starknet.core.os.contract_class.blake_compiled_class_hash import (
-     compiled_class_hash as blake_compiled_class_hash,
- )
+ from starkware.starknet.core.os.contract_class.poseidon_compiled_class_hash import (
+     compiled_class_hash as poseidon_compiled_class_hash,
+ )
```

And update the call site in `validate_compiled_class_facts`:

```cairo
- let (hash) = blake_compiled_class_hash(compiled_class, full_contract=FALSE);
+ let (hash) = poseidon_compiled_class_hash(compiled_class, full_contract=FALSE);
``` [5](#0-4) 

---

### Proof of Concept

1. A class declarer submits a declare transaction. The OS calls `validate_compiled_class_facts` on the compiled class bytes.
2. `validate_compiled_class_facts` computes `hash = blake2s(compiled_class_bytes)` and asserts `compiled_class_fact.hash == hash`.
3. The `compiled_class_hash` stored in `contract_class_changes` is therefore a Blake2s digest.
4. This Blake2s digest is committed into the class commitment tree via `get_contract_class_leaf_hash(compiled_class_hash)`.
5. Any off-chain verifier or client that recomputes `poseidon(compiled_class_bytes)` and compares it to the on-chain `compiled_class_hash` finds a mismatch — the state root is inconsistent with the protocol specification, causing a chain split between nodes that trust the OS proof and nodes that independently verify compiled class hashes. func compiled_class_hash{range_check_ptr, poseidon_ptr: PoseidonBuiltin*}(
    compiled_class: CompiledClass*, full_contract: felt
) -> (hash: felt) {
    alloc_locals;
    assert compiled_class.compiled_class_version = COMPILED_CLASS_VERSION;

    let hash_state: HashState = hash_init();
    with hash_state {
        hash_update_single(item=compiled_class.compiled_class_version);

        // Hash external entry points.
        hash_entry_points(
            entry_points=compiled_class.external_functions,
            n_entry_points=compiled_class.n_external_functions,
        );

        // Hash L1 handler entry points.
        hash_entry_points(
            entry_points=compiled_class.l1_handlers, n_entry_points=compiled_class.n_l1_handlers
        );

        // Hash constructor entry points.
        hash_entry_points(
            entry_points=compiled_class.constructors, n_entry_points=compiled_class.n_constructors
        );

        // Hash bytecode.
        let bytecode_hash = bytecode_hash_node(
            data_ptr=compiled_class.bytecode_ptr,
            data_length=compiled_class.bytecode_length,
            full_contract=full_contract,
        );
        hash_update_single(item=bytecode_hash);
    }

    let hash: felt = hash_finalize(hash_state=hash_state);
    return (hash=hash);
