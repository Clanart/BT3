### Title
`ProofOfSpace::update_digest` Panics on Invalid v2 Proof and Produces Hash Divergent from Serialized Bytes — (`File: crates/chia-protocol/src/proof_of_space.rs`)

### Summary

For v2 `ProofOfSpace` objects (version == 1), `update_digest` hashes `quality_string()` — a 32-byte value derived from the raw proof — while `stream` serializes the raw proof bytes. This intentional design divergence means `ProofOfSpace.hash() != sha256(ProofOfSpace.to_bytes())` for all v2 proofs, propagating the mismatch into `RewardChainBlock` and `FullBlock` hashes. More critically, `update_digest` calls `self.quality_string().expect("internal error. Can't compute hash of invalid ProofOfSpace")`: if a v2 `ProofOfSpace` with a syntactically valid but semantically invalid proof is parsed from the network and any code path subsequently calls `update_digest` (e.g., to compute a block header hash), the process panics unconditionally. The `parse` function performs no proof validity check, so such an object is freely constructible from attacker-controlled network bytes.

### Finding Description

**Root cause — `update_digest` vs `stream` divergence:**

`ProofOfSpace::update_digest` for version == 1:

```rust
self.quality_string()
    .expect("internal error. Can't compute hash of invalid ProofOfSpace")
    .update_digest(digest);
```

`ProofOfSpace::stream` for version == 1:

```rust
self.proof.stream(out)   // raw proof bytes, not quality_string
```

`quality_string()` returns `None` when `chia_pos2::quality_string_from_proof` fails (invalid proof). The `.expect()` then panics.

**Parse does not validate the proof:**

```rust
} else if version == 1 {
    let plot_index = <u16 as Streamable>::parse::<TRUSTED>(input)?;
    let meta_group  = <u8  as Streamable>::parse::<TRUSTED>(input)?;
    let strength    = <u8  as Streamable>::parse::<TRUSTED>(input)?;
    let proof       = <Bytes as Streamable>::parse::<TRUSTED>(input)?;
    // only structural check: exactly one of pool_pk / contract_ph
    if pool_public_key.is_some() == pool_contract_puzzle_hash.is_some() {
        return Err(Error::InvalidPoS);
    }
    Ok(ProofOfSpace { ..., proof, ... })
}
```

No call to `quality_string_from_proof` or any proof-content check. An attacker can craft a network message containing a v2 `ProofOfSpace` with arbitrary (invalid) proof bytes that passes `parse` cleanly.

**Panic propagation chain:**

`get_hash()` (Python binding, generated by `chia_py_streamable_macro`) calls `Streamable::update_digest`:

```rust
pub fn get_hash<'p>(&self, py: pyo3::Python<'p>) -> pyo3::PyResult<...> {
    let mut ctx = chia_sha2::Sha256::new();
    #crate_name::Streamable::update_digest(self, &mut ctx);
    ...
}
```

`RewardChainBlock::update_digest` calls `self.proof_of_space.update_digest(digest)`. `FullBlock::update_digest` calls `self.reward_chain_block.update_digest(digest)`. Any code computing a block header hash on a block containing an invalid v2 proof will panic.

**Hash mismatch (secondary):**

Even for a valid v2 proof, `ProofOfSpace.hash() != sha256(ProofOfSpace.to_bytes())` because `update_digest` feeds `quality_string()` (32 bytes) while `stream` feeds the raw proof (variable length). This propagates: `RewardChainBlock.hash() != sha256(RewardChainBlock.to_bytes())` and `FullBlock.hash() != sha256(FullBlock.to_bytes())` for any block containing a v2 proof. Any external code or cross-language component that computes the block hash as `sha256(block.to_bytes())` will disagree with `block.get_hash()`, producing consensus-critical hash divergence.

### Impact Explanation

**Panic path (Critical/High):** An attacker broadcasts a syntactically valid block whose `ProofOfSpace` carries an invalid v2 proof. Any node that attempts to compute the block header hash before completing proof validation will call `update_digest`, hit the `.expect()`, and panic. In a Rust full-node context this terminates the processing thread or process. In the Python binding (`get_hash()` via pyo3), it surfaces as a `PanicException`. If triggered on the critical block-processing path before proof validation, this is a deterministic chain halt.

**Hash mismatch path (High):** The `Streamable` trait's implicit contract — that `hash()` equals `sha256(to_bytes())` — is violated for all v2 `ProofOfSpace` objects. Any component (Python full-node, wasm client, external verifier) that computes block hashes via `sha256(block.to_bytes())` will produce a different value than `block.get_hash()`, causing cross-language consensus disagreement on block identity for every v2-proof block.

### Likelihood Explanation

The panic path requires only that a node attempts to hash a block before proof validation completes. The `parse` function imposes no barrier. The hash mismatch affects every v2-proof block unconditionally. Both paths are reachable by any unprivileged peer that can submit a crafted block to the network.

### Recommendation

1. **Eliminate the panic:** Replace `.expect(...)` in `update_digest` with a graceful error path. Either return a `Result` from `update_digest`, or validate the proof during `parse` so that an invalid v2 proof is rejected at deserialization time before any hash computation is attempted.

2. **Document or enforce the hash/stream invariant:** Either explicitly document that `hash() != sha256(to_bytes())` for v2 proofs and audit all callers to ensure none rely on their equality, or restructure `update_digest` to feed the same bytes as `stream` (appending the raw proof) and compute the quality-string commitment separately in the consensus validation layer.

### Proof of Concept

// Craft a v2 ProofOfSpace with an invalid proof (all-zero bytes)
let pos = ProofOfSpace::new(
    Bytes32::default(),
    Some(G1Element::default()),  // pool_public_key set, contract not set
    None,
    G1Element::default(),
    1,          // version == 1 → v2 proof
    0, 0, 2,    // plot_index, meta_group, strength

// parse succeeds — no proof validation

// propagates through RewardChainBlock::update_digest → FullBlock::update_digest

    pub fn quality_string(&self) -> Option<Bytes32> {
        if self.version != 1 {
            return None;
        }

        let k_size = (self.proof.len() * 8 / 128) as u8;
        let plot_id = self.compute_plot_id().to_bytes();
        chia_pos2::quality_string_from_proof(&plot_id, k_size, self.strength, self.proof.as_slice())
            .map(|quality| {
                let mut sha256 = Sha256::new();
                sha256.update(chia_pos2::serialize_quality(
                    &quality.chain_links,
                    self.strength,
                ));
                sha256.finalize().into()
            })
    }
