### Title
Stale MPC-Signed `MetadataPayload` Replay Enables Token Deployment with Wrong Decimals - (`near/omni-bridge/src/lib.rs`, `evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

The `log_metadata` function on NEAR is permissionless and produces an MPC-signed `MetadataPayload` that contains no nonce, block number, or sequence counter. Because the signed payload is permanently valid, an attacker can retain an old signature (emitted publicly as a NEAR event) and use it later to call `deployToken` on EVM with stale metadata — most critically, stale `decimals` — causing decimal/accounting corruption that breaks bridge collateralization.

---

### Finding Description

**Step 1 — Permissionless metadata signing with no freshness field.**

`log_metadata` on NEAR carries no access guard:

```rust
// near/omni-bridge/src/lib.rs:316-327
#[pause(except(roles(Role::DAO)))]
pub fn log_metadata(&self, token_id: &AccountId) -> Promise {
    ext_token::ext(token_id.clone())
        .with_static_gas(LOG_METADATA_GAS)
        .ft_metadata()
        .then(
            Self::ext(env::current_account_id())
                .with_static_gas(LOG_METADATA_CALLBACK_GAS)
                .with_attached_deposit(env::attached_deposit())
                .log_metadata_callback(token_id),
        )
}
``` [1](#0-0) 

The callback assembles and signs a `MetadataPayload`:

```rust
// near/omni-bridge/src/lib.rs:341-347
let metadata_payload = MetadataPayload {
    prefix: PayloadType::Metadata,
    token: token_id.to_string(),
    name: metadata.name,
    symbol: metadata.symbol,
    decimals: metadata.decimals,
};
``` [2](#0-1) 

The payload contains **no nonce, no block height, no sequence number**. The MPC signature over this payload is therefore valid forever.

**Step 2 — EVM `deployToken` verifies the signature but has no freshness check.**

```solidity

``` [3](#0-2) 

The only guard against re-use is that the token must not already be deployed:

```solidity

require(
    !isBridgeToken[nearToEthToken[metadata.token]],
    "ERR_TOKEN_EXIST"
);
``` [4](#0-3) 

There is no check that the signature was produced from a recent or canonical metadata snapshot.

---

### Impact Explanation

If a NEAR token's `decimals` field changes (e.g., via an upgrade) between two `log_metadata` calls, an attacker who holds the older MPC-signed payload can call `deployToken` on EVM with the stale decimals value. The EVM bridge then normalises all transfer amounts using the wrong decimal base:

```solidity
// evm/src/omni-bridge/contracts/OmniBridge.sol:159
uint8 decimals = _normalizeDecimals(metadata.decimals);
``` [5](#0-4) 

A token deployed with `decimals=18` when the canonical value is `decimals=6` causes every subsequent bridge transfer to mint or release 10¹² times the intended amount, directly breaking bridge collateralisation. This matches the allowed impact: **"Balance, decimal, fee, token-mapping, or accounting corruption that breaks bridge collateralization or misdirects value."**

---

### Likelihood Explanation

- `log_metadata` is callable by any unprivileged user at any time; no stake or special role is required.
- All `LogMetadataEvent` emissions (including the MPC signature) are publicly visible in NEAR transaction receipts, so an attacker can harvest old signatures without any privileged access.
- Upgradeable ERC-20/NEP-141 tokens that change decimals exist in practice (e.g., rebasing tokens, token migrations).
- The attacker only needs to act before the legitimate `deployToken` call is made on EVM — a race that is easy to win by front-running or by simply calling `deployToken` first with the stale signature.

---

### Recommendation

Add a monotonically increasing nonce (or NEAR block height) to `MetadataPayload` and record the last-used nonce on the EVM side, rejecting any payload whose nonce is not strictly greater than the stored value. This mirrors the fix applied to the ECO Protocol: include a block number in the cross-chain message so that stale signed payloads cannot be replayed.

```rust
// Proposed addition to MetadataPayload
pub struct MetadataPayload {
    pub prefix: PayloadType,
    pub nonce: u64,        // <-- add this
    pub token: String,
    pub name: String,
    pub symbol: String,
    pub decimals: u8,
}
```

On the EVM side, store and enforce `lastMetadataNonce[token] < payload.nonce` before accepting a `deployToken` call.

---

### Proof of Concept

1. Token `token.near` exists on NEAR with `decimals = 18`.
2. Attacker calls `log_metadata("token.near")` → NEAR MPC signs `MetadataPayload { token: "token.near", decimals: 18, ... }` → signature `S_old` is emitted in a public NEAR event.
3. `token.near` is upgraded; its `ft_metadata()` now returns `decimals = 6`.
4. A legitimate user calls `log_metadata("token.near")` → MPC signs `MetadataPayload { token: "token.near", decimals: 6, ... }` → signature `S_new`.
5. Attacker front-runs the legitimate `deployToken(S_new, ...)` call on EVM and instead submits `deployToken(S_old, { token: "token.near", decimals: 18, ... })`.
6. `ECDSA.recover` succeeds (signature is valid), `!isBridgeToken[nearToEthToken["token.near"]]` is true (token not yet deployed), so the token is deployed with `decimals = 18`.
7. All subsequent bridge transfers normalise amounts using 18 decimals instead of 6, minting 10¹² × the correct amount of EVM-side tokens per transfer. [6](#0-5) [7](#0-6)

    #[private]
    #[result_serializer(borsh)]
    pub fn log_metadata_callback(
        &self,
        #[callback] metadata: FungibleTokenMetadata,
        token_id: &AccountId,
    ) -> Promise {
        require!(
            !metadata.name.is_empty() && !metadata.symbol.is_empty(),
            BridgeError::InvalidMetadata.as_ref()
        );

        let metadata_payload = MetadataPayload {
            prefix: PayloadType::Metadata,
            token: token_id.to_string(),
            name: metadata.name,
            symbol: metadata.symbol,
            decimals: metadata.decimals,
        };

        let payload = near_sdk::env::keccak256_array(
            borsh::to_vec(&metadata_payload).near_expect(BridgeError::Borsh),
        );

        ext_signer::ext(self.mpc_signer.clone())
            .with_static_gas(MPC_SIGNING_GAS)
            .with_attached_deposit(env::attached_deposit())
            .sign(SignRequest {
                payload,
                path: SIGN_PATH.to_owned(),
                key_version: 0,
            })
            .then(
                Self::ext(env::current_account_id())
                    .with_static_gas(SIGN_LOG_METADATA_CALLBACK_GAS)
                    .sign_log_metadata_callback(metadata_payload),
            )
    }

    #[private]
    #[result_serializer(borsh)]
    pub fn sign_log_metadata_callback(
        &self,
        #[callback_result] call_result: Result<SignatureResponse, PromiseError>,
        #[serializer(borsh)] metadata_payload: MetadataPayload,
    ) {
        if let Ok(signature) = call_result {
            env::log_str(
                &OmniBridgeEvent::LogMetadataEvent {
                    signature,
                    metadata_payload,
                }
                .to_log_string(),
            );
        }
    }

    function deployToken(
        bytes calldata signatureData,
        BridgeTypes.MetadataPayload calldata metadata
    ) external payable whenNotPaused(PAUSED_DEPLOY_TOKEN) returns (address) {
        if (tokenImplementationAddress == address(0)) {
            revert TokenImplementationNotSet();
        }
        bytes memory borshEncoded = bytes.concat(
            bytes1(uint8(BridgeTypes.PayloadType.Metadata)),
            Borsh.encodeString(metadata.token),
            Borsh.encodeString(metadata.name),
            Borsh.encodeString(metadata.symbol),
            bytes1(metadata.decimals)
        );
        bytes32 hashed = keccak256(borshEncoded);

        if (ECDSA.recover(hashed, signatureData) != nearBridgeDerivedAddress) {
            revert InvalidSignature();
        }

        require(
            !isBridgeToken[nearToEthToken[metadata.token]],
            "ERR_TOKEN_EXIST"
        );
        uint8 decimals = _normalizeDecimals(metadata.decimals);

        // slither-disable-next-line reentrancy-no-eth
        address bridgeTokenProxy = address(
            new ERC1967Proxy(
                tokenImplementationAddress,
                abi.encodeWithSelector(
                    BridgeToken.initialize.selector,
                    metadata.name,
                    metadata.symbol,
                    decimals
                )
            )
        );

        deployTokenExtension(
            metadata.token,
            bridgeTokenProxy,
            decimals,
            metadata.decimals
        );

        emit BridgeTypes.DeployToken(
            bridgeTokenProxy,
            metadata.token,
            metadata.name,
            metadata.symbol,
            decimals,
            metadata.decimals
        );

        isBridgeToken[address(bridgeTokenProxy)] = true;
        ethToNearToken[address(bridgeTokenProxy)] = metadata.token;
        nearToEthToken[metadata.token] = address(bridgeTokenProxy);

        return bridgeTokenProxy;
    }
