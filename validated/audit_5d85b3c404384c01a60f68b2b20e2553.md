### Title
`ChainUpdate::set_state_finalize` Does Not Save `StateTransitionData`, Breaking State Witness Production After State Sync — (`File: chain/chain/src/chain_update.rs`)

---

### Summary

`ChainUpdate::set_state_finalize` and `ChainUpdate::set_state_finalize_on_height` apply chunks during state sync finalization but never call `save_state_transition_data()`. As a result, `DBCol::StateTransitionData` entries are permanently absent for every block processed during state sync. When the node subsequently becomes a chunk producer for the synced shard, `collect_state_transition_data` → `get_state_transition` fails with "Missing transition state proof", making it impossible to produce a valid `ChunkStateWitness` for the first chunk after state sync.

---

---

### Impact Explanation

Any node that completes state sync for a shard it will track in the next epoch is affected. After state sync, the node processes new blocks normally and `should_save_state_transition_data` becomes true, so new blocks are stored correctly. However, the blocks from `height_included` through `sync_hash − 1` — processed exclusively by `set_state_finalize` / `set_state_finalize_on_height` — have no `StateTransitionData` entries. When the node is assigned as chunk producer for the synced shard, the very first `ChunkStateWitness` it attempts to produce will fail because `collect_state_transition_data` cannot find the proof for those state-synced blocks. The node misses its assigned chunks, accumulates missed-chunk penalties, and risks validator kickout. [8](#0-7) 

---

### Likelihood Explanation

State sync is the standard recovery path for any node that falls behind by more than one epoch, and is also the mandatory path for nodes newly tracking a shard after resharding. Both scenarios are routine in production. The bug is triggered deterministically on every state sync that results in the node becoming a chunk producer for the synced shard in the following epoch. [9](#0-8) 

---

### Recommendation

In `ChainUpdate::set_state_finalize`, after `apply_chunk_postprocessing`, add a `save_state_transition_data` call conditioned on `should_produce_state_witness_for_this_or_next_epoch`, mirroring the pattern in `process_apply_chunk_result`:

```rust
// After apply_chunk_postprocessing in set_state_finalize:
if self.epoch_manager
    .should_produce_state_witness_for_this_or_next_epoch(block_header.epoch_id(), shard_id)?
{
    self.chain_store_update.save_state_transition_data(
        *block_header.hash(),
        shard_id,
        apply_result.proof.take(),
        apply_result.applied_receipts_hash,
        mem::take(&mut apply_result.contract_updates),
    );
}
```

Apply the same fix to `set_state_finalize_on_height` for the old-chunk (missing-chunk) blocks, using `apply_result.proof` and `apply_result.contract_updates` from the `OldChunk` apply result. [10](#0-9) 

---

### Proof of Concept

1. Run a two-node test network with stateless validation enabled.
2. Stop node B for more than one epoch so it falls behind and must state sync.
3. Restart node B; it completes state sync via `set_state_finalize` for shard 0.
4. Advance the chain until node B is assigned as chunk producer for shard 0.
5. Observe that node B's attempt to call `create_chunk_state_witness` → `collect_state_transition_data` → `get_state_transition` returns `Error::Other("Missing transition state proof for block <sync_hash-1> and shard 0")`.
6. Node B fails to produce the chunk; the missed-chunk counter increments each round until kickout threshold is reached.

The exact corrupted value is the absent `DBCol::StateTransitionData` entry keyed by `get_block_shard_id(block_hash, shard_id)` for every block hash in the range `[height_included, sync_hash)` processed during state sync finalization. [11](#0-10)

    /// Read state transition data from chain.
    fn get_state_transition(
        &self,
        epoch_manager: &dyn EpochManagerAdapter,
        block_hash: &CryptoHash,
        epoch_id: &EpochId,
        shard_id: ShardId,
    ) -> Result<(ChunkStateTransition, CryptoHash, ContractUpdates), Error> {
        let shard_uid = shard_id_to_uid(epoch_manager, shard_id, epoch_id)?;
        let stored_chunk_state_transition_data = self
            .store()
            .get_ser(
                near_store::DBCol::StateTransitionData,
                &near_primitives::utils::get_block_shard_id(block_hash, shard_id),
            )
            .ok_or_else(|| {
                let message = format!(
                    "Missing transition state proof for block {block_hash} and shard {shard_id}"
                );
                if !cfg!(feature = "shadow_chunk_validation") {
                    log_assert_fail!("{message}");
                }
                Error::Other(message)
            })?;
        let StoredChunkStateTransitionData::V1(StoredChunkStateTransitionDataV1 {
            base_state,
            receipts_hash,
            contract_accesses,
            contract_deploys,
        }) = stored_chunk_state_transition_data;
        let contract_updates = ContractUpdates {
            contract_accesses: contract_accesses.into_iter().collect(),
            contract_deploys: contract_deploys.into_iter().map(|c| c.into()).collect(),
        };
        Ok((
            ChunkStateTransition {
                block_hash: *block_hash,
                base_state,
                post_state_root: *self.get_chunk_extra(block_hash, &shard_uid)?.state_root(),
            },
            receipts_hash,
            contract_updates,
        ))
    }

    pub fn set_state_finalize(
        &mut self,
        shard_id: ShardId,
        sync_hash: CryptoHash,
    ) -> Result<(), Error> {
        let shard_state_header = self.state_sync_adapter.get_state_header(shard_id, sync_hash)?;
        let mut height = shard_state_header.chunk_height_included();
        let mut chain_update = self.chain_update();
        let shard_uid = chain_update.set_state_finalize(shard_id, sync_hash, shard_state_header)?;
        chain_update.commit()?;

        // We restored the state on height `shard_state_header.chunk.header.height_included`.
        // Now we should build a chain up to height of `sync_hash` block.
        loop {
            height += 1;
            let mut chain_update = self.chain_update();
            // Result of successful execution of set_state_finalize_on_height is bool,
            // should we commit and continue or stop.
            if chain_update.set_state_finalize_on_height(height, shard_id, sync_hash)? {
                chain_update.commit()?;
            } else {
                break;
            }
        }

        let flat_storage_manager = self.runtime_adapter.get_flat_storage_manager();
        if let Some(flat_storage) = flat_storage_manager.get_flat_storage_for_shard(shard_uid) {
            let header = self.get_block_header(&sync_hash)?;
            flat_storage.update_flat_head(header.prev_hash()).unwrap();
        }

        Ok(())
    }
