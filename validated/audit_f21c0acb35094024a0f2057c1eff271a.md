Audit Report

## Title
Router `pay()` Consumes Unattributed Native ETH Balance to Settle Any Caller's WETH Obligation, Enabling Theft of Stranded ETH — (File: `metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary
The `pay()` function reads `address(this).balance` — the router's entire native ETH balance — when settling WETH obligations, with no per-user attribution. ETH legitimately sent via `msg.value` to `payable` entry points (`multicall`, `exactInputSingle`, `exactOutputSingle`, `exactInput`, `exactOutput`) can remain on the router if the caller omits `refundETH()`. Any subsequent caller whose `tokenIn` is WETH will have that stranded ETH consumed on their behalf, permanently losing the original depositor's funds.

## Finding Description
`pay()` at [1](#0-0)  reads `address(this).balance` unconditionally when `token == WETH` and `payer != address(this)`. There is no snapshot, per-call accounting, or `msg.value` forwarding — the entire router balance is treated as available ETH for the current caller's obligation.

The `receive()` guard at [2](#0-1)  blocks unsolicited ETH pushes, but does not prevent ETH from arriving via `msg.value` on `payable` functions. `multicall` at [3](#0-2)  is `payable` and performs no post-call balance check. `exactInputSingle` at [4](#0-3)  is also `payable` and never enforces that `msg.value == amountIn`.

The exploit path:
1. User A calls `multicall{value: 1 ether}([exactInputSingle(WETH→token1, amountIn=0.5 ether)])` without appending `refundETH()`. During the callback, `_justPayCallback` calls `pay(WETH, userA, pool, 0.5 ether)` at [5](#0-4) ; `address(this).balance == 1 ether`, so 0.5 ETH is deposited and sent to the pool. After the swap, 0.5 ETH remains on the router with no owner record.
2. User B calls `exactInputSingle(WETH→token1, amountIn=0.5 ether)` with zero ETH and zero WETH approval. The callback fires `pay(WETH, userB, pool, 0.5 ether)`; `address(this).balance == 0.5 ether` (User A's ETH). The branch at [6](#0-5)  deposits and transfers that ETH as WETH to the pool. User B receives full swap output; User A's 0.5 ETH is permanently gone.

`refundETH()` at [7](#0-6)  exists but is optional and not enforced at the contract level.

## Impact Explanation
Direct loss of user principal. The stranded ETH is deposited as WETH and transferred to the pool on behalf of an unrelated caller. The original depositor receives nothing. Loss equals 100% of the stranded amount, which exceeds the Critical threshold for any non-dust value (>$100 USD, >20% of position). This is a swap conservation failure: the pool receives WETH input that was not paid by the declared `payer`, and the declared `payer` (User B) contributes nothing.

## Likelihood Explanation
Medium. ETH stranding requires `msg.value > amountIn` without `refundETH()`. This is a realistic omission: integrators commonly over-estimate `msg.value` as a safety buffer, and the protocol's own documented pattern (`multicall{value}(swap, refundETH)`) is advisory, not enforced. Once ETH is stranded, exploitation requires zero privileges — any address can trigger it with a single public call, no approvals, no ETH.

## Recommendation
Replace the unconditional `address(this).balance` read with only the ETH attributable to the current call. Two viable approaches:

1. **Delta snapshot**: record `address(this).balance - msg.value` before the swap entry point and pass the delta into `pay()`, so only the current call's ETH is consumed.
2. **Enforce no residual**: revert at the end of `multicall` if `address(this).balance > 0` after all delegatecalls complete, ensuring no ETH can ever be stranded.
3. **Pass `msg.value` explicitly**: thread `msg.value` through the call stack into `pay()` and use it instead of `address(this).balance`.

## Proof of Concept
```solidity
// Setup: Router deployed, Pool(WETH, token1) with liquidity.
// UserA has 1 ETH. UserB has 0 ETH, 0 WETH, no approvals.

// Step 1 — UserA strands 0.5 ETH
vm.prank(userA);
bytes[] memory calls = new bytes[](1);
calls[0] = abi.encodeCall(router.exactInputSingle, ExactInputSingleParams({
    pool: pool, tokenIn: WETH, tokenOut: token1,
    zeroForOne: true, amountIn: 0.5 ether,
    amountOutMinimum: 0, recipient: userA,
    deadline: block.timestamp + 1, priceLimitX64: 0, extensionData: ""
}));
router.multicall{value: 1 ether}(calls);
// 0.5 ETH used for swap; 0.5 ETH stranded (no refundETH).
assertEq(address(router).balance, 0.5 ether);

// Step 2 — UserB steals stranded ETH (no ETH, no WETH, no approval)
vm.prank(userB);
router.exactInputSingle(ExactInputSingleParams({
    pool: pool, tokenIn: WETH, tokenOut: token1,
    zeroForOne: true, amountIn: 0.5 ether,
    amountOutMinimum: 0, recipient: userB,
    deadline: block.timestamp + 1, priceLimitX64: 0, extensionData: ""
}));
// pay(WETH, userB, pool, 0.5 ether) fires; address(this).balance == 0.5 ether.
// Router deposits UserA's ETH as WETH, sends to pool. UserB gets token1 output.
assertEq(address(router).balance, 0);
assertGt(token1.balanceOf(userB), 0);
```
