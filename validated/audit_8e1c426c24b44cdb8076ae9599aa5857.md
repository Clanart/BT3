### Title
Native Fee Tokens (STRK) Permanently Locked in Starknet Bridge Contract With No Withdrawal Mechanism — (File: `starknet/src/omni_bridge.cairo`)

---

### Summary

The Starknet `OmniBridge` contract collects `native_fee` (STRK tokens) from users during every `init_transfer` call, but the contract exposes no function — admin or otherwise — to withdraw those accumulated STRK tokens. Every STRK fee paid by any user is irrecoverably locked in the contract forever.

---

### Finding Description

In `starknet/src/omni_bridge.cairo`, the `init_transfer` function unconditionally pulls `native_fee` STRK tokens from the caller into the contract itself:

```cairo
if native_fee > 0 {
    let native_token = self.strk_token_address.read();
    let success = IERC20Dispatcher { contract_address: native_token }
        .transfer_from(caller, get_contract_address(), native_fee.into());
    assert(success, 'ERR_FEE_TRANSFER_FAILED');
}
``` [1](#0-0) 

The entire public interface of the contract is:

```cairo
pub trait IOmniBridge<TContractState> {
    fn log_metadata(...)
    fn deploy_token(...)
    fn fin_transfer(...)
    fn init_transfer(...)
    fn upgrade_token(...)
    fn set_pause_flags(...)
    fn pause_all(...)
    fn get_token_address(...)
    fn is_bridge_token(...)
    fn is_transfer_finalised(...)
}
``` [2](#0-1) 

None of these functions — nor the `upgrade` function from `UpgradeableImpl` — allow withdrawing ERC-20 tokens (STRK) held by the contract. There is no `withdraw_fees`, `rescue_tokens`, or equivalent function anywhere in the contract. [3](#0-2) 

By contrast, the NEAR bridge has an explicit `claim_fee` mechanism that lets relayers prove and claim their earned fees: [4](#0-3) 

The Starknet bridge has no equivalent. The STRK tokens transferred to `get_contract_address()` during `init_transfer` are owned by the contract with no code path to move them out.

---

### Impact Explanation

**Critical — Permanent freezing / irrecoverable lock of user funds in bridge flows.**

Every user who calls `init_transfer` with `native_fee > 0` permanently loses those STRK tokens. The funds are not burned (they remain as an ERC-20 balance of the bridge contract), but they are functionally unrecoverable because no function exists to transfer them out. Over the lifetime of the bridge, the total locked STRK accumulates proportionally to protocol usage. Relayers, who are the intended recipients of native fees as gas compensation, can never claim them on Starknet.

---

### Likelihood Explanation

**High.** The `native_fee` parameter is a first-class field of `init_transfer` and is expected to be non-zero whenever a relayer requires gas compensation. Any user who follows the documented fee-paying flow triggers this lock. No special conditions, attacker privileges, or external failures are required — the lock occurs on every ordinary fee-paying transfer.

---

### Recommendation

Add an admin-gated withdrawal function to the Starknet contract that allows the `DEFAULT_ADMIN_ROLE` holder to transfer accumulated STRK (and any other ERC-20 tokens) out of the contract to a designated fee-recipient address. For example:

```cairo
fn withdraw_fees(
    ref self: ContractState,
    token: ContractAddress,
    recipient: ContractAddress,
    amount: u256,
) {
    self.accesscontrol.assert_only_role(DEFAULT_ADMIN_ROLE);
    let success = IERC20Dispatcher { contract_address: token }
        .transfer(recipient, amount);
    assert(success, 'ERR_WITHDRAW_FAILED');
}
```

Alternatively, mirror the NEAR bridge's `claim_fee` design: route the `native_fee` directly to the relayer's address at the time of `init_transfer` rather than accumulating it in the contract.

---

### Proof of Concept

1. Alice calls `init_transfer(token_address, 1_000_000, 0, 500_000, "near:bob.near", "")` on the Starknet bridge, paying 500,000 STRK as `native_fee`.
2. The contract executes `transfer_from(Alice, bridge_contract, 500_000)` — STRK moves to the bridge.
3. An event is emitted; the transfer proceeds normally on NEAR.
4. Alice's 500,000 STRK sits in the bridge contract's ERC-20 balance indefinitely.
5. No relayer, admin, or user can call any function to retrieve those STRK tokens — the interface has no such entry point.
6. Repeat for every fee-paying user; the locked STRK balance grows monotonically and is never recoverable under the current code. [5](#0-4)

            token_address: ContractAddress,
            amount: u128,
            fee: u128,
            native_fee: u128,

            message: ByteArray,
        ) {
            assert(!_is_paused(@self, PAUSE_INIT_TRANSFER), 'ERR_INIT_TRANSFER_PAUSED');

            assert(amount > 0, 'ERR_ZERO_AMOUNT');
            assert(fee < amount, 'ERR_INVALID_FEE');

            let origin_nonce = self.current_origin_nonce.read() + 1;
            self.current_origin_nonce.write(origin_nonce);

            let caller = get_caller_address();

            if self.is_bridge_token(token_address) {
                IBridgeTokenDispatcher { contract_address: token_address }
                    .burn(caller, amount.into());
            } else {
                let success = IERC20Dispatcher { contract_address: token_address }
                    .transfer_from(caller, get_contract_address(), amount.into());
                assert(success, 'ERR_TRANSFER_FROM_FAILED');
            }

            if native_fee > 0 {
                let native_token = self.strk_token_address.read();
                let success = IERC20Dispatcher { contract_address: native_token }
                    .transfer_from(caller, get_contract_address(), native_fee.into());
                assert(success, 'ERR_FEE_TRANSFER_FAILED');
            }

            self
                .emit(

                            token_address,
                            origin_nonce,
                            amount,
                            fee,

                            message,
