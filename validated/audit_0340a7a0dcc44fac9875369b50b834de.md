### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Contract Freezing - (File: `execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` function in `syscall_impls.cairo` accepts an attacker-controlled class hash without verifying it corresponds to a declared contract class. This allows any unprivileged contract caller to permanently corrupt a contract's class pointer, making the contract uncallable and freezing all funds it holds.

### Finding Description

In `execute_replace_class` (lines 878–916 of `syscall_impls.cairo`), the new class hash is taken directly from the syscall request and written into `contract_state_changes` without any validation against the set of declared classes:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );
    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The explicit `TODO` comment acknowledges the missing check. [1](#0-0) 

Once the invalid class hash is committed to state, any subsequent call to that contract reaches `execute_entry_point`, which performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // undeclared hash → returns 0
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,           // key=0, not found → panic
);
``` [2](#0-1) 

`find_element` panics when the key is absent. Because the state was already committed with the invalid class hash, the contract is permanently uncallable.

The vulnerability class maps directly to the external report: just as `delegatecall` with dynamic, unvalidated inputs executes arbitrary code in the caller's context (enabling self-destruction), `replace_class` with a dynamic, unvalidated class hash permanently corrupts the contract's execution context, destroying its functionality.

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any funds (ERC-20 balances, NFTs, or other assets) held by a contract whose class hash has been replaced with an undeclared value are permanently frozen. The contract becomes uncallable at the OS level; no withdrawal, transfer, or recovery path exists once the state is committed.

### Likelihood Explanation

**High.** The attacker only needs to control a contract (deploy one or already own one). No privileged role, leaked key, or external dependency is required. The `replace_class` syscall is a standard, publicly accessible syscall reachable from any Cairo 1 contract's execution context. The missing validation is explicitly acknowledged in the code via the TODO comment, confirming it is not a design choice but an unimplemented guard.

### Recommendation

Before writing the new class hash to `contract_state_changes`, validate that `request.class_hash` is present in `contract_class_changes` (i.e., it has been declared). Concretely, perform a `dict_read` on `contract_class_changes` with `request.class_hash` as the key and assert the result is non-zero before proceeding with the state update. This mirrors the lookup already performed in `execute_entry_point` and closes the gap between what the OS accepts and what it can later prove.

### Proof of Concept

1. Attacker deploys contract `C` holding user funds.
2. Contract `C` issues a `replace_class` syscall with `class_hash = 0xdeadbeef` (an arbitrary, undeclared felt).
3. `execute_replace_class` accepts the request without validation and writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes[C]`. [3](#0-2) 
4. The block is proven and the state is committed on-chain with `C.class_hash = 0xdeadbeef`.
5. Any subsequent transaction calling `C` reaches `execute_entry_point`:
   - `dict_read(contract_class_changes, 0xdeadbeef)` → returns `0` (not declared).
   - `find_element(..., key=0)` → panics; the OS cannot generate a valid proof for any block containing a call to `C`. [4](#0-3) 
6. Contract `C` is permanently uncallable. All funds are frozen with no recovery path.
