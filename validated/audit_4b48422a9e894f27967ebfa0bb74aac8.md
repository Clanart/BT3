### Title
`execute_send_message_to_l1` Charges Flat Gas Fee But Performs Unbounded `memcpy` Proportional to Payload Size — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary

The `execute_send_message_to_l1` syscall implementation in the StarkNet OS charges a flat, fixed gas cost (`SEND_MESSAGE_TO_L1_GAS_COST = 14470`) regardless of the message payload size, but then performs a `memcpy` of `payload_size` felt elements into the OS output segment. This `memcpy` consumes Cairo VM steps proportional to `payload_size` — steps that are part of the OS's own proving execution and are not charged back to the user through the syscall gas. An unprivileged transaction sender can exploit this to force the OS prover to perform disproportionate work relative to the gas paid, potentially exhausting the block's step budget and preventing new transactions from being confirmed.

### Finding Description

In `execute_send_message_to_l1`, the gas deduction is a flat constant:

```cairo
let success = reduce_syscall_gas_and_write_response_header(
    total_gas_cost=SEND_MESSAGE_TO_L1_GAS_COST, request_struct_size=SendMessageToL1Request.SIZE
);
``` [1](#0-0) 

After the gas check succeeds, the function computes `payload_size` from attacker-controlled pointer arithmetic and immediately performs an unbounded `memcpy`:

```cairo
tempvar payload_size = request.payload_end - payload_start;
...
memcpy(
    dst=outputs.messages_to_l1 + MessageToL1Header.SIZE, src=payload_start, len=payload_size
);
``` [2](#0-1) 

There is **no per-element gas charge** and **no upper-bound assertion** on `payload_size`. Compare this with `execute_deploy`, which explicitly adds a per-element factor:

```cairo
let specific_base_gas_cost = DEPLOY_GAS_COST + DEPLOY_CALLDATA_FACTOR_GAS_COST * constructor_calldata_size;
``` [3](#0-2) 

And `execute_meta_tx_v0`, which similarly charges per calldata element:

```cairo
let specific_base_gas_cost = (
    META_TX_V0_GAS_COST + META_TX_V0_CALLDATA_FACTOR_GAS_COST * calldata_size
);
``` [4](#0-3) 

The constants confirm the design intent for variable-size inputs: [5](#0-4) 

`DEPLOY_CALLDATA_FACTOR_GAS_COST` and `META_TX_V0_CALLDATA_FACTOR_GAS_COST` exist precisely to meter per-element OS work. No equivalent `SEND_MESSAGE_TO_L1_PAYLOAD_FACTOR_GAS_COST` exists.

The `memcpy` in Cairo consumes one VM step per element copied. These steps are part of the OS program's execution budget (the block step limit), not the user's Sierra gas budget. The user's Sierra gas pays for building the payload in their own contract execution, but the OS-side copy cost is entirely unmetered from the user's perspective.

### Impact Explanation

The OS Cairo program runs within a block-level step limit. An attacker who sends a transaction with a maximally-sized `send_message_to_l1` payload forces the OS to consume a large number of steps during block proving, while paying only the flat `SEND_MESSAGE_TO_L1_GAS_COST = 14470` gas for the syscall. By packing multiple such transactions into a block (each within their own Sierra gas budget), the attacker can exhaust the OS's step budget, causing the block to fail proving or preventing the sequencer from including legitimate transactions. This maps to **High: Network not being able to confirm new transactions**.

### Likelihood Explanation

Any unprivileged account can call `send_message_to_l1` with an arbitrarily large payload. The Sierra gas budget (`EXECUTE_MAX_SIERRA_GAS = 1,100,000,000`) limits how large a payload a single transaction can build, but the ratio of OS memcpy steps to user gas paid is unbounded by design. The attack requires only deploying a contract and submitting invoke transactions — no privileged access, no leaked keys, no external dependencies. [6](#0-5) 

### Recommendation

Add a per-element payload gas factor to `execute_send_message_to_l1`, mirroring the pattern used in `execute_deploy` and `execute_meta_tx_v0`:

```cairo
const SEND_MESSAGE_TO_L1_PAYLOAD_FACTOR_GAS_COST = <calibrated_value>;

// In execute_send_message_to_l1:
let payload_size = request.payload_end - payload_start;
let specific_base_gas_cost = SEND_MESSAGE_TO_L1_GAS_COST + SEND_MESSAGE_TO_L1_PAYLOAD_FACTOR_GAS_COST * payload_size;
let (success, remaining_gas) = reduce_syscall_base_gas(
    specific_base_gas_cost=specific_base_gas_cost, request_struct_size=SendMessageToL1Request.SIZE
);
```

Additionally, add an upper-bound assertion on `payload_size` (analogous to the `assert_nn_le(constructor_calldata_size, SIERRA_ARRAY_LEN_BOUND - 1)` pattern used elsewhere) before the `memcpy`.

### Proof of Concept

1. Attacker deploys a contract whose `__execute__` function allocates a felt array of maximum feasible size (bounded by `EXECUTE_MAX_SIERRA_GAS`) and calls `send_message_to_l1` with that array as payload.
2. Attacker submits an invoke transaction targeting this contract.
3. The sequencer includes the transaction. The user's Sierra gas is consumed for array allocation. The syscall gas deduction is only `SEND_MESSAGE_TO_L1_GAS_COST = 14470`.
4. During OS execution, `execute_send_message_to_l1` performs `memcpy` of the full payload — consuming OS steps proportional to payload size with no additional gas charged.
5. By submitting multiple such transactions in a single block, the attacker exhausts the OS step budget, preventing the block from being proven and halting transaction confirmation. [7](#0-6)

// Sends a message to L1.
func execute_send_message_to_l1{range_check_ptr, syscall_ptr: felt*, outputs: OsCarriedOutputs*}(
    contract_address: felt
) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, SendMessageToL1Request*);
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=SEND_MESSAGE_TO_L1_GAS_COST, request_struct_size=SendMessageToL1Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    tempvar payload_start = request.payload_start;
    tempvar payload_size = request.payload_end - payload_start;

    assert [outputs.messages_to_l1] = MessageToL1Header(
        from_address=contract_address, to_address=request.to_address, payload_size=payload_size
    );
    memcpy(
        dst=outputs.messages_to_l1 + MessageToL1Header.SIZE, src=payload_start, len=payload_size
    );
    let (outputs) = os_carried_outputs_new(
        messages_to_l1=outputs.messages_to_l1 + MessageToL1Header.SIZE + payload_size,
        messages_to_l2=outputs.messages_to_l2,
    );

    return ();
}
```
