### Title
Unvalidated Class Hash in `execute_replace_class` Enables Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not validate that the new class hash supplied by the caller corresponds to a previously declared contract class. An unprivileged contract can invoke this syscall with an arbitrary, undeclared class hash. Once committed to state, the contract becomes permanently non-executable, and any funds it holds are irreversibly frozen.

---

### Finding Description

The `execute_replace_class` function processes the `replace_class` syscall. After deducting gas, it reads the requested new class hash directly from the syscall request and writes it into `contract_state_changes` without any check that the hash is present in the declared class registry (`contract_class_changes`):

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment at line 898 is an explicit, unresolved acknowledgment of this missing validation. The function only checks that sufficient gas is available; it imposes no constraint on the semantic validity of `class_hash`.

When a subsequent call is made to the affected contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
```

If `class_hash` is undeclared, `dict_read` returns `0` (the default for an uninitialized dict entry). `find_element` then searches for a compiled class with hash `0`, which does not exist, causing the OS execution to fail. The contract is permanently non-executable.

---

### Impact Explanation

Any funds (ETH, STRK, or arbitrary ERC-20 tokens) held in a contract whose class hash has been replaced with an undeclared value are permanently frozen. The contract cannot be called, upgraded, or self-destructed. There is no recovery path because the OS state commitment records the invalid class hash as the authoritative state. This satisfies the **Critical — Permanent freezing of funds** impact criterion.

---

### Likelihood Explanation

The attack is reachable by any unprivileged contract deployer or by any caller who can trigger a `replace_class` syscall in a target contract (e.g., through an exposed upgrade function that does not independently validate the new class hash). The OS is the last line of defense; because it performs no validation, any contract-level omission of input checking is directly exploitable. The missing check is confirmed by the in-code TODO, indicating the team is aware the guard is absent.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, `execute_replace_class` must verify that the hash is present in `contract_class_changes` (i.e., that a non-zero compiled class hash is associated with it):

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the validation already performed implicitly during `execute_entry_point` and closes the gap identified by the TODO comment.

---

### Proof of Concept

1. Attacker deploys `VaultContract` holding user funds. The contract exposes an `upgrade(new_class_hash)` function that calls the `replace_class` syscall.
2. Attacker calls `upgrade(0xdeadbeef)` where `0xdeadbeef` is never declared on-chain.
3. `execute_replace_class` deducts gas, reads `class_hash = 0xdeadbeef` from the request, and writes it to `contract_state_changes` with no further checks. [1](#0-0) 
4. State is committed. `VaultContract`'s `StateEntry.class_hash` is now `0xdeadbeef`.
5. Any subsequent call to `VaultContract` reaches `execute_entry_point`, which calls `dict_read` on `contract_class_changes` for key `0xdeadbeef`, receiving `0`. [2](#0-1) 
6. `find_element` searches for compiled class hash `0`, finds nothing, and the OS execution fails. All funds in `VaultContract` are permanently frozen with no recovery mechanism.
