### Title
`ProofOfInclusion::valid()` Trivially Returns `true` for Any Proof with Empty `layers` — Forged DataLayer Inclusion Proof Accepted - (File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs)

### Summary

`ProofOfInclusion::valid()` contains a degenerate zero-case: when `layers` is empty, the function reduces to `self.node_hash == self.node_hash`, which is unconditionally `true`. An unprivileged attacker can construct a `ProofOfInclusion` with any arbitrary `node_hash` and an empty `layers` vector, serialize it via the `Streamable` interface, and have `valid()` return `true`. This allows forging a DataLayer proof of inclusion for any hash without knowledge of the actual tree contents.

### Finding Description

In `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`, the `valid()` method iterates over `self.layers` to verify the hash chain from `node_hash` up to the root:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()   // ← degenerate when layers is empty
}
```

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash          // ← returns node_hash when layers is empty
    }
}
```

When `layers` is empty:
- The loop body never executes.
- `existing_hash` remains `self.node_hash`.
- `self.root_hash()` returns `self.node_hash`.
- The final check becomes `self.node_hash == self.node_hash` → always `true`.

This is structurally identical to the reported `valueToShares` pattern: a zero-denominator special case (`layers.is_empty()` ↔ `currentEquity == 0`) causes the validation function to bypass all real checks and return a trivially correct result.

The `ProofOfInclusion` struct derives `Streamable`, is constructible directly from Python via `__new__`, and is exposed through the Python wheel as `chia_rs.datalayer.ProofOfInclusion`. Any caller — including untrusted network peers — can supply a serialized `ProofOfInclusion` with empty `layers` and an arbitrary `node_hash`.

### Impact Explanation

An attacker can forge a `ProofOfInclusion` that:
1. Claims any arbitrary hash `H` is a leaf in the DataLayer Merkle tree.
2. Passes `proof.valid() == True` unconditionally.
3. Returns `proof.root_hash() == H` — making the forged proof appear to commit to a root equal to the claimed leaf.

Any verifier that calls only `proof.valid()` (as the fuzz target does) will accept the forged proof. Even verifiers that additionally check `proof.root_hash() == known_root` are vulnerable if the attacker can influence what `known_root` is (e.g., by supplying the root alongside the proof in a protocol message).

This allows untrusted input to prove invalid DataLayer state — matching the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion or lets untrusted input prove invalid state**.

### Likelihood Explanation

- `ProofOfInclusion` is `Streamable` and directly constructible from Python (`ProofOfInclusion(node_hash=H, layers=[])`).
- No privilege is required; any network participant can craft and submit such a proof.
- The fuzz target (`proofs_of_inclusion.rs`) only asserts `proof.valid()` without cross-checking the root against the actual tree, confirming that `valid()` is treated as a self-contained correctness oracle.
- The single-element tree case legitimately produces empty `layers`, so the degenerate path is exercised in normal operation and is unlikely to be caught by integration tests.

### Recommendation

`valid()` must reject a proof with empty `layers` unless the caller can independently confirm the tree has exactly one element. The simplest fix is to require that `valid()` be called with an expected root hash parameter, or to add an explicit check:

```rust
pub fn valid(&self) -> bool {
    // A proof with no layers is only valid if the caller separately
    // confirms the tree root equals node_hash (single-element tree).
    // Without that context, reject empty-layer proofs to prevent forgery.
    if self.layers.is_empty() {
        return false; // or require caller to pass expected_root
    }
    // ... existing hash-chain verification ...
}
```

Alternatively, expose a `valid_for_root(expected_root: &Hash) -> bool` method that always checks `self.root_hash() == expected_root` as part of validation, and deprecate the no-argument `valid()`.
