### Title
Fixed Gas Budget for L1 Handler Execution Does Not Scale with Calldata Size, Enabling Permanent Fund Loss — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

`execute_l1_handler_transaction` allocates a hard-coded gas budget of `L1_HANDLER_L2_GAS_MAX_AMOUNT = 100,000,000` for every L1→L2 handler execution, regardless of the calldata size carried by the message. Because the message is irrevocably consumed **before** execution begins, any handler that exhausts this fixed budget due to large attacker-controlled calldata will silently fail while the message is permanently marked as consumed. Funds locked on L1 expecting a corresponding L2 credit are permanently frozen.

---

### Finding Description

In `execute_l1_handler_transaction`, the execution sequence is:

```cairo
// Step 1 – message is consumed and written to the proof output (irreversible)
consume_l1_to_l2_message(execution_context=tx_execution_context, nonce=nonce);

// Step 2 – handler is executed with a FIXED gas budget, independent of calldata size
let remaining_gas = L1_HANDLER_L2_GAS_MAX_AMOUNT;          // always 100_000_000
non_reverting_select_execute_entry_point_func{remaining_gas=remaining_gas}(
    block_context=block_context, execution_context=tx_execution_context
);
``` [1](#0-0) 

`L1_HANDLER_L2_GAS_MAX_AMOUNT` is a compile-time constant:

```cairo
const L1_HANDLER_L2_GAS_MAX_AMOUNT = 100000000;
``` [2](#0-1) 

The only calldata bound enforced by the OS is the Sierra array length ceiling:

```cairo
assert_nn_le(tx_execution_context.calldata_size, SIERRA_ARRAY_LEN_BOUND - 1);
// SIERRA_ARRAY_LEN_BOUND = 4_294_967_296  (2^32)
``` [3](#0-2) 

There is **no lower bound** on the gas budget relative to calldata size, and **no check** that `L1_HANDLER_L2_GAS_MAX_AMOUNT` is sufficient for the supplied calldata before the message is consumed.

`consume_l1_to_l2_message` writes the message header and payload directly into `outputs.messages_to_l2`, which is part of the OS proof output. Once written, the message is treated as consumed by the L1 verifier and cannot be replayed. [4](#0-3) 

`non_reverting_select_execute_entry_point_func` is explicitly non-reverting: if the handler runs out of gas, the OS continues and the block is finalized with the message marked consumed but the handler's effects absent.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Token bridge contracts on L1 (e.g., StarkGate) lock ETH or ERC-20 tokens on L1 and emit an L1→L2 message instructing the L2 handler to mint the equivalent tokens. If the handler fails due to out-of-gas:

- The L1 tokens remain locked in the bridge contract (no refund path once the message is consumed).
- The L2 mint never executes.
- The message nonce is permanently spent; the message cannot be re-submitted.

The result is a permanent, irrecoverable loss of the bridged funds for the victim.

---

### Likelihood Explanation

An unprivileged L1 sender controls the calldata of any L1→L2 message. With `STEP_GAS_COST = 100`, the 100 M gas budget corresponds to roughly **1 million Sierra steps**. A handler that iterates over its calldata (e.g., a batch-mint bridge processing an array of recipients) consumes O(calldata\_size) steps. Sending a message with ~100 k–1 M felt elements — well within the 2^32 bound and affordable on L1 — is sufficient to exhaust the budget. No privileged access is required; any L1 account can call the bridge's deposit function with an oversized payload array.

---

### Recommendation

Scale the gas budget for L1 handler execution with the calldata size, analogous to how Optimism's fix updated `minimumGasLimit()` to use 40 gas per byte. Concretely:

```cairo
let calldata_gas = tx_execution_context.calldata_size * CALLDATA_WORD_GAS_COST;
let remaining_gas = L1_HANDLER_L2_GAS_MAX_AMOUNT + calldata_gas;
```

Alternatively, enforce a pre-execution check that the calldata size does not exceed a threshold derivable from `L1_HANDLER_L2_GAS_MAX_AMOUNT`, and reject (without consuming) messages that exceed it.

---

### Proof of Concept

1. Attacker calls a StarkGate-style bridge on L1, supplying a `recipients` array of 500,000 felt elements as the message payload.
2. The sequencer picks up the L1→L2 message and calls `execute_l1_handler_transaction`.
3. `consume_l1_to_l2_message` writes the message to `outputs.messages_to_l2` — the message is now permanently consumed. [5](#0-4) 
4. `non_reverting_select_execute_entry_point_func` is invoked with `remaining_gas = 100_000_000`. [6](#0-5) 
5. The handler iterates over the 500,000-element array; each iteration costs multiple Sierra steps. Gas is exhausted before the loop completes.
6. The handler exits with `ERROR_OUT_OF_GAS`; `non_reverting_select_execute_entry_point_func` absorbs the failure silently.
7. The block is proven and accepted. The L1 message is marked consumed. The victim's bridged funds are permanently frozen with no recourse.

func consume_l1_to_l2_message{outputs: OsCarriedOutputs*}(
    execution_context: ExecutionContext*, nonce: felt
) {
    assert_not_zero(execution_context.calldata_size);
    // The payload is the calldata without the from_address argument (which is the first).
    let payload: felt* = execution_context.calldata + 1;
    tempvar payload_size = execution_context.calldata_size - 1;

    tempvar execution_info = execution_context.execution_info;

    // Write the given transaction to the output.
    assert [outputs.messages_to_l2] = MessageToL2Header(
        from_address=[execution_context.calldata],
        to_address=execution_info.contract_address,
        nonce=nonce,
        selector=execution_info.selector,
        payload_size=payload_size,
    );

    let message_payload = cast(outputs.messages_to_l2 + MessageToL2Header.SIZE, felt*);
    memcpy(dst=message_payload, src=payload, len=payload_size);

    let (outputs) = os_carried_outputs_new(
        messages_to_l1=outputs.messages_to_l1,
        messages_to_l2=outputs.messages_to_l2 + MessageToL2Header.SIZE +
        outputs.messages_to_l2.payload_size,
    );
    return ();
