### Title
Zero-Fee Cosmos Tx Execution via Integer Truncation in `NewDynamicFeeChecker` — (`ante/evm/fee_checker.go`)

---

### Summary

When `NoBaseFee=true` and the London hardfork is enabled, `NewDynamicFeeChecker` computes `feeCap = fee / gas` using integer division. Any attacker who sets `gas > fee` (e.g., `fee=1`, `gas=2`) causes `feeCap` to truncate to `0`. Because `baseFee` is also `0` in this configuration, the `feeCap < baseFee` guard passes, `effectivePrice` resolves to `0`, and `effectiveFee = 0` is returned. The `checkDeductFee` function then skips deduction entirely, granting free execution.

---

### Finding Description

**Step 1 — `NoBaseFee=true` + London HF enabled does NOT fall back to min-gas-price logic.**

`types.GetBaseFee` in `x/evm/types/utils.go`:

```go
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    if !IsLondon(ethCfg, height) {
        return nil
    }
    baseFee := feemarketParams.GetBaseFee()
    if baseFee == nil {
        return new(big.Int)   // ← returns 0, not nil
    }
    return baseFee
}
``` [1](#0-0) 

`feemarketParams.GetBaseFee()` returns `nil` when `NoBaseFee=true`: [2](#0-1) 

So `GetBaseFee` returns `new(big.Int)` (value `0`), **not** `nil`. The nil-guard in `fee_checker.go` that would redirect to `checkTxFeeWithValidatorMinGasPrices` is never triggered:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
if baseFee == nil {   // ← NOT taken; baseFee == 0
    return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
}
``` [3](#0-2) 

**Step 2 — Integer truncation produces `feeCap = 0`.**

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))   // 1 / 2 = 0
baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)    // 0

if feeCap.LT(baseFeeInt) {   // 0 < 0 → false → passes
    return nil, 0, errorsmod.Wrapf(...)
}
``` [4](#0-3) 

**Step 3 — `effectivePrice = 0`, `effectiveFee = 0`.**

`EffectiveGasPrice(baseFee=0, feeCap=0, tip)` = `min(0 + tip, 0)` = `0`.

```go
effectivePrice := sdkmath.NewIntFromBigInt(
    types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
// effectivePrice = 0

effectiveFee := sdk.Coins{{Denom: denom, Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas))}}
// Amount = 0 * gas = 0
``` [5](#0-4) 

**Step 4 — `checkDeductFee` skips deduction for zero fee.**

```go
if !fee.IsZero() {
    err := evmkeeper.DeductFees(dfd.bankKeeper, ctx, deductFeesFromAcc, fee)
    ...
}
``` [6](#0-5) 

Zero fees are never deducted. The attacker's account balance is unchanged.

---

### Impact Explanation

An unprivileged attacker can submit Cosmos txs with `fee=1` (or any value) and `gas > fee`, paying **zero** effective fees. This is a direct mis-accounting of user fees in the ante handler, matching the High impact category: *"fee market, ante handler bug that permits valid user funds/fees to be mis-accounted."*

The attacker can spam the network with zero-cost transactions, exhausting block gas and degrading chain liveness without any economic cost.

---

### Likelihood Explanation

- **Precondition:** `NoBaseFee=true` + London HF enabled. This is a documented, supported deployment mode (chains that want EIP-1559 tx format without dynamic base fee adjustment).
- **Attacker action:** Submit any Cosmos tx with `fee=1`, `gas=2`. No special privileges, keys, or network position required.
- **Trigger:** Deterministic — every such tx passes the fee check with zero deduction.

Likelihood is **High** given the precondition is a common chain configuration.

---

### Recommendation

Replace the truncating integer division with a check that rejects any tx where the declared fee is insufficient to cover at least `baseFee * gas` (or `1 * gas` when `baseFee=0`). Concretely:

1. When `baseFee == 0` (NoBaseFee mode), enforce a minimum `feeCap >= minGasPrice` using the validator's `MinGasPrices`, or require `fee >= gas * minGasPrice`.
2. Alternatively, reject the tx if `feeCap == 0` and `fee > 0` (truncation occurred): `if fee.IsPositive() && feeCap.IsZero() { return error }`.
3. Use `fee.Quo(gas)` with a remainder check: if `fee mod gas != 0` and `feeCap == 0`, reject.

---

### Proof of Concept

```
NoBaseFee = true, LondonBlock = 0 (enabled)
Attacker tx: fee = [1 aphoton], gas = 2

feeCap     = 1 / 2 = 0          (sdkmath integer truncation)
baseFeeInt = 0                   (NoBaseFee=true → GetBaseFee returns new(big.Int))
0 < 0      = false               (guard passes)
effectivePrice = EffectiveGasPrice(0, 0, MaxInt64) = 0
effectiveFee   = 0 * 2 = 0
checkDeductFee(0) → IsZero() → skip → 0 aphoton deducted

Assert: attacker balance unchanged despite executing tx.
Fuzz: gas ∈ [2, MaxUint64], fee = 1 → effectiveFee always 0.
```
