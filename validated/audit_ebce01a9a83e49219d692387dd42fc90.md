### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS program accepts any arbitrary felt value as a new class hash without verifying that it corresponds to a declared and compiled contract class. An unprivileged contract can invoke this syscall with a non-existent class hash, permanently replacing its own class hash with an invalid value. Once committed to state, the contract becomes permanently unexecutable, and any funds held in its storage are irreversibly frozen.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function handles the `replace_class` syscall. After deducting gas, it reads `request.class_hash` directly from the syscall request and writes it into the contract's `StateEntry` without any validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment at line 898 explicitly acknowledges the missing check. The OS program — which is the authoritative source of truth for the STARK proof — accepts and commits this state transition without verifying that `class_hash` exists in the compiled class facts bundle (`block_context.os_global_context.compiled_class_facts_bundle`).

This is the direct analog to the external report's vulnerability: just as a messenger address can be set to a value that cannot be removed or zeroed (preventing emergency response), here a contract's class hash can be permanently set to an invalid value that cannot be corrected — because `replace_class` can only be called by the contract itself, and once the class is invalid, the contract can never execute again.

The `replace_class` syscall is designed to take effect for **future** calls (the current transaction continues with the old class). This means the transaction that calls `replace_class(invalid_hash)` completes successfully, the state change is committed, and the invalid class hash is written into the global state root. No revert occurs. [1](#0-0) 

The `OsGlobalContext` carries the `compiled_class_facts_bundle` which contains all valid class hashes, but this bundle is never consulted during `execute_replace_class`: [2](#0-1) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with a non-existent value and the block is proven and accepted on L1:

1. The new state root encodes the contract with the invalid class hash.
2. Any subsequent transaction targeting that contract will fail at class lookup — the OS cannot find a compiled class for the hash.
3. The contract's storage (including token balances, locked collateral, etc.) is permanently inaccessible.
4. There is no recovery path: `replace_class` can only be called by the contract itself, which can no longer execute.

This matches the allowed impact: **Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

**Medium.** The attack is reachable by any unprivileged contract deployer or transaction sender:

- Any contract can call the `replace_class` syscall on itself.
- No privileged role is required.
- A malicious contract author can intentionally trigger this (e.g., a rug-pull variant where a DeFi contract holding user funds is bricked).
- A buggy contract with an unguarded `replace_class` call path can be exploited by an attacker who crafts calldata to reach it.
- The sequencer executes the transaction successfully (the syscall itself does not fail), so the block is produced and proven normally.

---

### Recommendation

Before committing the class hash change, validate that `request.class_hash` exists in the compiled class facts bundle available in `block_context.os_global_context.compiled_class_facts_bundle`. This is already noted as a TODO in the code and should be treated as a security-critical fix, not a deferred optimization.

Analogously to the external report's fix (allowing the admin to set the messenger to zero address), the OS should enforce that `replace_class` can only target a hash that has been declared — or explicitly allow zeroing the class hash as a "disable" mechanism with appropriate access controls at the contract level.

---

### Proof of Concept

1. **Attacker deploys** a contract `VaultContract` that holds user token balances in storage and exposes a public `drain_and_brick()` entry point.
2. Inside `drain_and_brick()`, the attacker calls the `replace_class` syscall with `class_hash = 0xdeadbeef` (an arbitrary felt not corresponding to any declared class).
3. The OS processes `execute_replace_class`:
   - Gas is deducted. [3](#0-2) 
   - `class_hash = 0xdeadbeef` is read from the request with **no validation**. [4](#0-3) 
   - The contract's `StateEntry` is updated with `class_hash=0xdeadbeef`.
   - A revert log entry is written for the old class hash. [5](#0-4) 
4. The transaction completes successfully. The block is proven. L1 accepts the new state root.
5. `VaultContract`'s class hash in the global state is now `0xdeadbeef`.
6. Any future `call_contract` or `invoke` targeting `VaultContract` fails — the OS cannot locate a compiled class for `0xdeadbeef`.
7. All user funds stored in `VaultContract` are permanently frozen.

The state commitment path confirms the invalid class hash is durably written into the Patricia tree: [6](#0-5)

func get_contract_state_hash{hash_ptr: HashBuiltin*}(
    class_hash: felt, storage_root: felt, nonce: felt
) -> (hash: felt) {
    const CONTRACT_STATE_HASH_VERSION = 0;
    if (class_hash == UNINITIALIZED_CLASS_HASH) {
        if (storage_root == 0) {
            if (nonce == 0) {
                return (hash=0);
            }
        }
    }

    // Set res = H(H(class_hash, storage_root), nonce).
    let (hash_value) = hash2(class_hash, storage_root);
    let (hash_value) = hash2(hash_value, nonce);

    // Return H(hash_value, CONTRACT_STATE_HASH_VERSION). CONTRACT_STATE_HASH_VERSION must be in the
    // outermost hash to guarantee unique "decoding".
    let (hash) = hash2(hash_value, CONTRACT_STATE_HASH_VERSION);
    return (hash=hash);
}
