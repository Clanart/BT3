### Title
EIP-712 Amino Sign-Doc `timeout_height` Exclusion Enables Replay of Expired Cosmos Transactions via `ethsecp256k1.PubKey.VerifySignature` Fallback - (File: `ethereum/eip712/types.go`, `crypto/ethsecp256k1/ethsecp256k1.go`)

---

### Summary

`ethsecp256k1.PubKey.VerifySignature` accepts a signature if it verifies against either raw ECDSA or the EIP-712 encoding of the same sign bytes. The EIP-712 `Tx` type schema intentionally omits `timeout_height`. Because of this, a signature produced over a Cosmos Amino-encoded transaction with `timeout_height = X` will also verify against the same transaction with `timeout_height = 0`. An unprivileged attacker who observes the mempool can strip the timeout from an expired, EIP-712-signed transaction and re-submit it, causing it to execute after the user intended to cancel it.

---

### Finding Description

**Root cause 1 — dual-path `VerifySignature`:**

`crypto/ethsecp256k1/ethsecp256k1.go` line 226–227 accepts a signature if *either* the raw-ECDSA path or the EIP-712 fallback path succeeds:

```go
func (pubKey PubKey) VerifySignature(msg, sig []byte) bool {
    return pubKey.verifySignatureECDSA(msg, sig) || pubKey.verifySignatureAsEIP712(msg, sig)
}
``` [1](#0-0) 

This function is invoked by the Cosmos SDK's `ante.NewSigVerificationDecorator` inside `newCosmosAnteHandler` for every standard Cosmos tx whose account holds an `ethsecp256k1` key. [2](#0-1) 

**Root cause 2 — `timeout_height` absent from EIP-712 `Tx` schema:**

`ethereum/eip712/types.go` defines the EIP-712 `Tx` type. The comment explicitly states `timeout_height` was removed:

```go
"Tx": {
    {Name: "account_number", Type: "string"},
    {Name: "chain_id",       Type: "string"},
    {Name: "fee",            Type: "Fee"},
    {Name: "memo",           Type: "string"},
    {Name: "sequence",       Type: "string"},
    // Note timeout_height was removed because it was not getting filled with the legacyTx
},
``` [3](#0-2) 

**Root cause 3 — Amino decode path does not reject non-zero `timeout_height`:**

`decodeProtobufSignDoc` (Protobuf path) explicitly rejects `TimeoutHeight != 0`:

```go
if body.TimeoutHeight != 0 || body.GetTimeoutTimestamp() != nil || ... {
    return apitypes.TypedData{}, errors.New("body contains unsupported fields: ...")
}
``` [4](#0-3) 

But `decodeAminoSignDoc` (Amino path) performs **no such check**. It calls `WrapTxToTypedData` directly with the full Amino JSON, which includes `timeout_height` in the raw bytes but not in the EIP-712 hash:
