### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Bricking — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS Cairo program does not verify that the new class hash supplied by the caller corresponds to a previously declared contract class. The OS accepts and commits any arbitrary felt value as the new class hash. A malicious sequencer can include a `replace_class` call with an undeclared class hash, produce a valid STARK proof, and permanently render the target contract non-executable — freezing all funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the requested class hash directly from the syscall request and writes it into `contract_state_changes` without any membership check against `contract_class_changes` (the dictionary of declared classes):

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The inline `TODO` comment explicitly acknowledges the missing check. The same omission exists in the deprecated syscall path: [2](#0-1) 

The OS enforces that a class hash can only be *declared* once (`prev_value=0` guard in `execute_declare_transaction`): [3](#0-2) 

But `execute_replace_class` never cross-references `contract_class_changes` to confirm the target hash exists there. The OS proof is therefore valid even when a contract's class hash is set to a value that has no corresponding compiled class.

**Analog to the external report:** The report describes stale/invalid authorization state being accepted without clearing or validating it — votes from removed holders persist and authorize an upgrade. Here, the OS accepts a class-hash state transition without validating the new value is a legitimate declared class, analogous to accepting an authorization from a removed/invalid voter.

---

### Impact Explanation

Once a contract's class hash is set to an undeclared value and committed to the state Merkle tree, every subsequent call to that contract will fail at class-lookup time. Any ERC-20 token contract, vault, or bridge contract that self-calls `replace_class` (or is tricked into doing so via a malicious inner call) with an undeclared hash becomes permanently non-executable. All funds stored in its storage are irrecoverably frozen.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

- The attack requires a **malicious or compromised sequencer** to include a transaction where a contract calls `replace_class` with an undeclared hash. The sequencer controls which transactions enter a block.
- Alternatively, a contract that exposes an unguarded `replace_class` call (e.g., a proxy with an open upgrade path) can be exploited by any unprivileged user who can invoke that entry point.
- The OS Cairo program is the final arbiter of what constitutes a valid state transition for proof purposes. Because it does not enforce the declared-class constraint, a valid STARK proof can be generated for this invalid state transition.
- The `TODO` comment confirms the StarkNet team is aware the check is absent, meaning the window of exposure is active and acknowledged.

---

### Recommendation

1. In `execute_replace_class` (both `syscall_impls.cairo` and `deprecated_execute_syscalls.cairo`), assert that `class_hash` exists in `contract_class_changes` (i.e., its compiled class hash is non-zero) before writing the new `StateEntry`. This mirrors the guard already applied in `deploy_contract` which asserts `state_entry.class_hash = UNINITIALIZED_CLASS_HASH` before deployment.
2. Remove the `TODO` and implement the check as a Cairo `assert` or `dict_read` against `contract_class_changes`, reverting the syscall if the class is undeclared.

---

### Proof of Concept

1. Declare class `A` (legitimate). Deploy contract `C` using class `A`. `C` holds user funds.
2. Craft a transaction that calls `C.__execute__`, which internally calls the `replace_class` syscall with `class_hash = 0xdeadbeef` (never declared).
3. The OS processes `execute_replace_class`: no check is performed against `contract_class_changes`; `contract_state_changes[C].class_hash` is updated to `0xdeadbeef`.
4. `state_update` squashes and commits this to the Patricia tree — a valid STARK proof is produced.
5. All subsequent calls to `C` fail at class resolution. Funds in `C`'s storage are permanently frozen. [4](#0-3) [5](#0-4)
