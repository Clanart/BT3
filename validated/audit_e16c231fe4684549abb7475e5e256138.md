### Title
Missing Declared Class Validation in `execute_replace_class` Allows Undeclared Class Hash to Corrupt Contract State and Break OS Proof - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the new class hash supplied by a contract corresponds to a previously declared contract class. This is a direct analog to the external report's missing whitelist check: just as `setPosMode` allowed blacklisted collateral to be moved to a new mode, `execute_replace_class` allows any arbitrary, undeclared class hash to be written into a contract's state. When the contract is subsequently called, the OS's `execute_entry_point` function fails hard (non-reverted, proof-breaking failure) because `find_element` cannot locate the undeclared compiled class, making the block unprovable and causing a network halt.

---

### Finding Description

In `execute_replace_class` (lines 877–916 of `syscall_impls.cairo`), the function reads the requested class hash directly from the syscall request and writes it into `contract_state_changes` without any check that the hash corresponds to a declared class:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment at line 898 explicitly acknowledges the missing check and was dated January 1, 2026 — a date that has already passed — confirming the check was planned but never implemented.

When any subsequent call is made to the contract (in the same block or a later block), `execute_entry_point` performs the following lookup chain:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // <-- the undeclared hash
);
```

Since the undeclared hash was never registered in `contract_class_changes`, `dict_read` returns the default value `0`. The OS then calls:

```cairo
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,   // <-- 0, not present in facts
);
```

`find_element` raises a `ValueError` (hard Cairo VM failure) when the key is absent. This is not a transaction revert — it is an OS-level proof failure. The block containing the subsequent call cannot be proven or finalized.

---

### Impact Explanation

**High. Network not being able to confirm new transactions (total network shutdown).**

Once a contract's class hash is replaced with an undeclared value, every subsequent transaction that invokes that contract causes the OS to fail at `find_element`. The block containing such a transaction cannot be proven. If the attacker targets a widely-called contract (e.g., a popular DeFi contract or a system contract), every block that includes calls to it becomes unprovable, halting the network's ability to finalize new blocks.

---

### Likelihood Explanation

**High.** The attack requires only:
1. Deploying a contract (permissionless).
2. Sending a transaction that calls `replace_class` with an arbitrary felt value as the class hash.
3. Having the contract called again (by anyone, including the attacker) in a subsequent transaction.

No privileged access, leaked keys, or operator cooperation is required. The attacker is a standard unprivileged transaction sender. The TODO comment confirms the team is aware the check is missing.

---

### Recommendation

In `execute_replace_class`, before updating the contract state, verify that `class_hash` is present in `contract_class_changes` (i.e., it has been declared in the current or a prior block). The check should mirror the lookup performed in `execute_entry_point`:

```cairo
// Verify the new class hash is a declared class.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This ensures that only classes with a valid compiled class mapping can be used as replacement targets, consistent with the invariant enforced everywhere else in the OS.

---

### Proof of Concept

1. Attacker deploys **Contract A** (any valid class).
2. Attacker sends **Transaction T1** invoking Contract A, which internally issues `replace_class(0xdeadbeef)` where `0xdeadbeef` is an arbitrary felt never declared as a class.
3. `execute_replace_class` processes the syscall without validation. Contract A's `class_hash` in `contract_state_changes` is now `0xdeadbeef`.
4. Attacker (or any user) sends **Transaction T2** invoking Contract A.
5. The OS calls `execute_entry_point` for Contract A with `class_hash = 0xdeadbeef`.
6. `dict_read{dict_ptr=contract_class_changes}(key=0xdeadbeef)` returns `0` (default, never declared).
7. `find_element(..., key=0)` raises `ValueError: Key 0 was not found in the array` — a hard OS failure.
8. The OS cannot generate a valid proof for the block containing T2. The block is permanently unfinalizeable.
9. Any future block including a call to Contract A suffers the same fate, causing a sustained network halt. [1](#0-0) [2](#0-1) [3](#0-2)
