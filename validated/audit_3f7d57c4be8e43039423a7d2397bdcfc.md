### Title
Deploy Syscall Unconditionally Reports Success Regardless of Constructor Outcome — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_deploy` syscall handler in `syscall_impls.cairo` hardcodes `failure_flag=0` in its response header unconditionally, even when the deployed contract's constructor fails and its state changes are reverted via the revert log. Any caller contract that relies on the success flag to decide whether to transfer funds to the newly deployed address will send funds to an uninitialized (class_hash=0) contract, permanently freezing them.

---

### Finding Description

In `execute_deploy`, after calling `deploy_contract`, the response header is written with a hardcoded `failure_flag=0`: [1](#0-0) 

```cairo
with remaining_gas {
    let (retdata_size, retdata) = deploy_contract(
        block_context=block_context, constructor_execution_context=constructor_execution_context
    );
}
// Write the response header.
// TODO(Yoni, 1/1/2026): support failures.
assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=0);
```

The developer-acknowledged TODO confirms this is a known gap. Compare this with the correct pattern used in `contract_call_helper`, which propagates the actual revert status: [2](#0-1) 

```cairo
let (is_reverted, retdata_size, retdata, _is_deprecated) = select_execute_entry_point_func(...);
...
assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=is_reverted);
```

`execute_deploy` has `revert_log` as an implicit argument: [3](#0-2) 

This means `deploy_contract` is called with a live revert log and **can** handle constructor reverts — it will undo the state changes (restoring `class_hash=0` for the new address) and return failure retdata. But the response to the caller always says success.

The nonce is not incremented for the deployed contract during a syscall-level deploy (only the deployer's nonce is tracked at the transaction level via `check_and_increment_nonce`): [4](#0-3) 

So after a failed constructor, the target address has `class_hash=0`, `nonce=0`, and empty storage — it is indistinguishable from a never-deployed address. Any ETH/STRK sent there is unrecoverable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

A factory or vault contract that:
1. Calls the `deploy` syscall to create a child contract, and
2. Immediately transfers tokens to the returned `contract_address` (a standard pattern)

…will send funds to an address with `class_hash=0` whenever the constructor reverts. Because there is no code at that address, no withdrawal function exists. The funds are permanently frozen with no recovery path.

---

### Likelihood Explanation

**Medium-High.** The `deploy` syscall is a standard, widely-used primitive. Constructor failures are a normal occurrence (out-of-gas, assertion failures, invalid initial state). Any contract following the deploy-then-fund pattern — factory contracts, proxy deployers, token vaults — is affected. An attacker can deliberately craft a class whose constructor fails under controlled conditions (e.g., specific calldata or gas amount) to trigger the bug against a victim factory contract.

---

### Recommendation

Mirror the pattern used in `contract_call_helper`. `deploy_contract` must return an `is_reverted` flag, and `execute_deploy` must propagate it:

```cairo
with remaining_gas {
    let (is_reverted, retdata_size, retdata) = deploy_contract(
        block_context=block_context,
        constructor_execution_context=constructor_execution_context,
    );
}
...
assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=is_reverted);
```

When `is_reverted=1`, the response `contract_address` field should be set to `0` (or a sentinel) so callers cannot accidentally use it.

func check_and_increment_nonce{contract_state_changes: DictAccess*}(tx_info: TxInfo*) -> () {
    // Do not handle nonce for version 0.
    if (tx_info.version == 0) {
        return ();
    }

    tempvar state_entry: StateEntry*;
    %{ SetStateEntryToAccountContractAddress %}

    tempvar current_nonce = state_entry.nonce;
    with_attr error_message("Unexpected nonce.") {
        assert current_nonce = tx_info.nonce;
    }

    // Update contract_state_changes.
    tempvar new_state_entry = new StateEntry(
        class_hash=state_entry.class_hash,
        storage_ptr=state_entry.storage_ptr,
        nonce=current_nonce + 1,
    );
    dict_update{dict_ptr=contract_state_changes}(
        key=tx_info.account_contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    return ();
