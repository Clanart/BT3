### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

`execute_replace_class` in the StarkNet OS is missing a check that the supplied `class_hash` is an already-declared class. An unprivileged contract can call this syscall with an arbitrary, undeclared class hash. The OS accepts the state update unconditionally, permanently replacing the contract's class with a non-existent one. All subsequent calls to that contract will fail at class-lookup time, permanently freezing any funds held by the contract.

---

### Finding Description

In `execute_replace_class`, the OS updates `contract_state_changes` with the caller-supplied `class_hash` without verifying that the hash exists in `contract_class_changes`:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-acknowledged TODO comment confirms the check is intentionally deferred but not yet enforced. [1](#0-0) 

By contrast, `execute_declare_transaction` enforces that a class is registered in `contract_class_changes` via `dict_update` with `prev_value=0` before it can be used. [2](#0-1) 

When the OS later executes a call to the affected contract, it reads the class hash from the contract's `StateEntry` and attempts to look it up in `contract_class_changes`. Because the hash was never declared, the lookup yields nothing and execution fails — permanently, since the state update was already committed.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once `replace_class` is called with an undeclared hash and the enclosing transaction succeeds, the contract's `StateEntry.class_hash` is permanently set to a non-existent class. Every subsequent call to that contract (including withdrawal functions) will fail at class resolution. Any ERC-20 tokens, ETH, or other assets held in the contract's storage become irrecoverable. The `revert_log` entry written by `execute_replace_class` only enables rollback within the same transaction; once the transaction is finalized, the state change is irreversible. [3](#0-2) 

---

### Likelihood Explanation

**Low to Medium.**

The attack requires a contract that either:
1. Is controlled by the attacker (attacker deploys a contract, attracts user deposits, then calls `replace_class` with an invalid hash to freeze deposited funds), or
2. Exposes an upgrade function callable by the attacker (e.g., a DeFi protocol with an insufficiently guarded `replace_class` path).

No privileged role is required. Any unprivileged user who can deploy a contract or trigger a `replace_class` call in an existing contract can exploit this. The syscall is available to all Cairo 1 contracts via the standard syscall interface. [4](#0-3) 

---

### Recommendation

Before committing the `dict_update` in `execute_replace_class`, assert that the supplied `class_hash` exists in `contract_class_changes`:

```cairo
// Verify the class has been declared.
let (declared_compiled_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash not declared.") {
    assert_not_zero(declared_compiled_hash);
}
```

This mirrors the protection already present in `execute_declare_transaction`, which uses `prev_value=0` to enforce that a class can only be declared once and must exist before use. [5](#0-4) 

---

### Proof of Concept

1. Attacker deploys `MaliciousVault` — a contract that accepts token deposits and exposes a public `freeze()` function that calls `replace_class(0xdeadbeef...)` where `0xdeadbeef...` is an arbitrary felt never declared as a class.
2. Users deposit funds into `MaliciousVault`.
3. Attacker calls `freeze()`. The OS executes `execute_replace_class`:
   - Gas is deducted. [6](#0-5) 
   - `contract_state_changes` is updated with `class_hash = 0xdeadbeef...` — **no declared-class check is performed**. [3](#0-2) 
   - The transaction succeeds and is finalized.
4. Any user attempting to call `withdraw()` on `MaliciousVault` triggers class resolution for `0xdeadbeef...`, which does not exist in `contract_class_changes`. Execution fails unconditionally.
5. All deposited funds are permanently frozen with no recovery path.

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    let revert_log = &revert_log[1];

    return ();
}
