### Title
Missing Duplicate-Hash Guard in `MerkleBlob::upsert` Update Path Corrupts `leaf_hash_to_index` Cache and Merkle Tree Root — (`crates/chia-datalayer/src/merkle/blob.rs`)

### Summary

`MerkleBlob::upsert` has two code paths: when the key is absent it delegates to `insert` (which checks `HashAlreadyPresent`); when the key is present it takes an inline update path that **never checks whether `new_hash` is already held by a different leaf**. This allows two leaf nodes to share the same hash value, breaking the `leaf_hash_to_index` bijection and corrupting the Merkle tree root.

### Finding Description

In the update branch of `upsert` (line 792):

1. `block_status_cache.remove_leaf(&leaf)` removes the old hash `H1` from `leaf_hash_to_index` and marks the slot as free.
2. `leaf.hash.clone_from(new_hash)` sets the leaf's hash to `H2` — **without checking `contains_leaf_hash(H2)`**.
3. `insert_entry_to_blob(leaf_index, &block)` writes the block and calls `add_leaf`, which does `leaf_hash_to_index.insert(leaf.hash, index)` — a silent overwrite if `H2` already exists.

After the call sequence `insert(k1,v1,H1)` → `insert(k2,v2,H2)` → `upsert(k1,v_new,H2)`:

- The blob contains two leaf nodes, both with hash `H2`.
- `leaf_hash_to_index` has only one entry for `H2` (pointing to `k1`'s index); `k2`'s mapping is silently overwritten.
- `leaf_count` = 2, `leaf_hash_to_index.len()` = 1 → `check_integrity` returns `IntegrityLeafHashToIndexCacheLength`.
- The Merkle root is now computed from two identical leaf hashes, producing a root that does not correspond to the actual key-value state.

`insert` guards against this at lines 372–374, but the `upsert` update path has no equivalent guard.

### Impact Explanation

**High — DataLayer Merkle tree root corruption.** The root hash stored on-chain no longer reflects the actual key-value contents. Downstream consumers (proof-of-inclusion verifiers, delta readers, sync logic) operate against a corrupted root, enabling:
- Forged inclusion proofs (a key not in the store can be "proven" present under the wrong root).
- Forged exclusion proofs (a key that is present can be "proven" absent).
- Persistent, silent state divergence between the blob and its cache.

### Likelihood Explanation

The call sequence requires no special privileges beyond write access to a DataLayer store (which any store owner has). No hash collision is needed — the attacker simply passes an already-known leaf hash as `new_hash`. The bug is reachable through the public Rust API and the Python binding (`py_upsert`, line 1436). [6](#0-5) 

### Recommendation

Add the same guard that `insert` uses, at the start of the update branch in `upsert`:

```rust
if self.block_status_cache.contains_leaf_hash(new_hash) {
    // only allow if the hash belongs to the key being updated
    if self.block_status_cache.get_index_by_leaf_hash(new_hash)
        != self.block_status_cache.get_index_by_key(key) {
        return Err(Error::HashAlreadyPresent());
    }
}
```

Or equivalently, after `remove_leaf` (which removes the old hash), check `contains_leaf_hash(new_hash)` before writing.

### Proof of Concept

```rust
use chia_datalayer::{Hash, InsertLocation, KeyId, MerkleBlob, ValueId};
use chia_protocol::Bytes32;

let h1 = Hash(Bytes32::new([1u8; 32]));
let h2 = Hash(Bytes32::new([2u8; 32]));

let mut blob = MerkleBlob::new(vec![]).unwrap();
blob.insert(KeyId(1), ValueId(1), &h1, InsertLocation::Auto {}).unwrap();
blob.insert(KeyId(2), ValueId(2), &h2, InsertLocation::Auto {}).unwrap();

// upsert k1 with h2 — no error returned
blob.upsert(KeyId(1), ValueId(99), &h2).unwrap();

// integrity check fails: leaf_count=2, leaf_hash_to_index.len()=1
blob.check_integrity().expect_err("invariant violated");
```

    pub fn upsert(&mut self, key: KeyId, value: ValueId, new_hash: &Hash) -> Result<(), Error> {
        let Ok((leaf_index, mut leaf, mut block)) = self.get_leaf_by_key(key) else {
            self.insert(key, value, new_hash, InsertLocation::Auto {})?;
            return Ok(());
        };

        self.block_status_cache.remove_leaf(&leaf)?;
        leaf.hash.clone_from(new_hash);
        leaf.value = value;
        // OPT: maybe just edit in place?
        block.node = Node::Leaf(leaf);
        self.insert_entry_to_blob(leaf_index, &block)?;

        if let Some(parent) = block.node.parent().0 {
            self.mark_lineage_as_dirty(parent)?;
        }

        Ok(())
    }
