Looking at the full code path from `parse_data_for_block` through `write_to_storage` → `append_header`, I need to determine whether any guard rejects an attacker-supplied `block_hash`.

## Verdict

### Title
Missing `block_hash` Cryptographic Verification in P2P Header Sync Allows Malicious Peer to Corrupt Block Hash Storage — (`crates/apollo_p2p_sync/src/client/header.rs`)

---

### Summary

`HeaderStreamBuilder::parse_data_for_block` accepts a `SignedBlockHeader` from an untrusted P2P peer and passes it directly to `write_to_storage` → `append_header` without ever recomputing or verifying that `block_header.block_hash` equals `calculate_block_hash(partial_components, state_root, parent_hash)`. The attacker-supplied `block_hash` is written verbatim into both the `headers` table (keyed by block number) and the `block_hash_to_number` reverse index (keyed by the fake hash). No cryptographic guard exists anywhere in this path.

---

### Finding Description

**Entrypoint:** `HeaderStreamBuilder::parse_data_for_block` [1](#0-0) 

The only checks performed on the incoming `SignedBlockHeader` are:

1. `block_number` matches the expected sequence number.
2. `signatures.len() == ALLOWED_SIGNATURES_LENGTH` (length only, no cryptographic verification). [2](#0-1) 

There is an explicit TODO acknowledging that even `parent_hash` chain continuity is unverified:

```
// TODO(shahak): Check that parent_hash is the same as the previous block's hash
// and handle reverts.
``` [3](#0-2) 

The returned `signed_block_header` (with attacker-chosen `block_hash`) is then passed to `write_to_storage`, which calls `append_header` unconditionally: [4](#0-3) 

Inside `append_header`, the fake `block_hash` is stored verbatim in two places:

- `headers_table.append(block_number, storage_block_header)` — stores `block_header.block_hash` as-is.
- `update_hash_mapping(block_hash_to_number_table, storage_block_header, block_number)` — inserts the fake hash into the reverse index. [5](#0-4) 

No call to `calculate_block_hash` or any equivalent recomputation exists anywhere in this pipeline. A grep for `verify.*signature` or `check.*block_hash` in `crates/apollo_p2p_sync/**` returns zero matches.

---

### Impact Explanation

**Concrete corrupted values:**

| Table | Key | Stored value |
|---|---|---|
| `headers` | `BlockNumber(N)` | `block_hash = Felt::from(0xdeadbeef)` (attacker-chosen) |
| `block_hash_to_number` | `Felt::from(0xdeadbeef)` | `BlockNumber(N)` |

The real hash (the Poseidon hash of the canonical inputs) is never inserted into `block_hash_to_number`. Consequently:

- `starknet_getBlockByHash(real_hash)` → `BlockNotFound` (authoritative-looking wrong RPC response, matching the "High. RPC execution… returns an authoritative-looking wrong value" impact category).
- The stored `block_hash` of block N is the value downstream consumers read as `parent_hash` for block N+1. Since `parent_hash` chain continuity is also unverified (the TODO above), the peer can send a consistent fake chain: block N+1 carries `parent_hash = 0xdeadbeef`, which is stored and propagated. The entire hash chain from block N onward is corrupted.

---

### Likelihood Explanation

Any node operating in P2P sync mode (i.e., `BLOCK_NUMBER_LIMIT::Unlimited`, the header sync path) that connects to a single malicious peer is vulnerable. No operator privilege is required; the attacker only needs to be a reachable P2P peer. The attack is deterministic and requires no brute force.

---

### Recommendation

After receiving `signed_block_header` in `parse_data_for_block`, recompute the expected hash using `calculate_block_hash` from the header's own fields and reject the peer if `signed_block_header.block_header.block_hash != computed_hash`. Additionally, verify the `BlockSignature` cryptographically against the block hash using the known sequencer public key (analogous to `sequencer_pub_key` used in `apollo_central_sync`). The existing TODO for `parent_hash` chain continuity should also be resolved.

---

### Proof of Concept

```rust
// Inject a SignedBlockHeader with attacker-chosen block_hash
let fake_hash = BlockHash(Felt::from(0xdeadbeef_u64));
let real_hash = calculate_block_hash(&partial, state_root, parent_hash).unwrap();

let signed = SignedBlockHeader {
    block_header: BlockHeader {
        block_hash: fake_hash,          // attacker-controlled
        block_header_without_hash: BlockHeaderWithoutHash {
            block_number: BlockNumber(0),
            state_root,
            ..Default::default()
        },
        state_diff_length: Some(0),
        ..Default::default()
    },
    signatures: vec![BlockSignature::default()], // length check passes
};

// write_to_storage calls append_header with no hash recomputation
storage_writer.begin_rw_txn().unwrap()
    .append_header(BlockNumber(0), &signed.block_header).unwrap()
    .commit().unwrap();

let txn = storage_reader.begin_ro_txn().unwrap();
// Real hash is absent from the reverse index:
assert!(txn.get_block_number_by_hash(&real_hash).unwrap().is_none());
// Fake hash is present:
assert_eq!(txn.get_block_number_by_hash(&fake_hash).unwrap(), Some(BlockNumber(0)));
// Stored header carries the fake hash:
assert_eq!(txn.get_block_header(BlockNumber(0)).unwrap().unwrap().block_hash, fake_hash);
```
