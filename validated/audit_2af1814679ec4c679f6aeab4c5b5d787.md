### Title
`replace_class` Syscall Accepts Undeclared Class Hashes Without Internal Accounting Verification, Enabling Permanent Freezing of Funds — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS does not verify that the new class hash supplied by the caller has been declared (i.e., exists in `contract_class_changes`). The OS accepts any arbitrary felt value as a replacement class hash. A subsequent call to the contract with the invalid class hash causes `execute_entry_point` to look up a compiled class for hash `0` (the dict default for an undeclared key), which panics via `find_element`, making the contract permanently uncallable and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class` (Cairo 1 path), the new class hash is taken directly from the syscall request and written into `contract_state_changes` with no check against `contract_class_changes`:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The same missing check exists in the deprecated (Cairo 0) path: [2](#0-1) 

When the contract is subsequently called, `execute_entry_point` reads the compiled class hash from `contract_class_changes`:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
``` [3](#0-2) 

Because the class hash was never declared, `dict_read` returns `0` (the Cairo dict default for an uninitialized key, equal to `UNINITIALIZED_CLASS_HASH`). The OS then calls `find_element` with key `0`:

```cairo
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [4](#0-3) 

`find_element` panics if the key is absent. Since no compiled class has hash `0` (`UNINITIALIZED_CLASS_HASH = 0`), the OS aborts for any block containing a call to the affected contract. [5](#0-4) 

The analog to the external report is exact: just as `PosManager` reads live `balanceOfLp()` instead of internally tracked amounts (bypassing the pause), the OS reads `contract_class_changes` for the new class hash but never writes to it during `replace_class`, so the internal accounting (declared classes) is never consulted to validate the replacement.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

1. An attacker deploys a contract (or controls an existing one) that holds user funds (e.g., a vault or DEX pool).
2. The attacker calls `replace_class(0xdeadbeef)` where `0xdeadbeef` is not a declared class hash.
3. The OS accepts the state transition without verification.
4. The contract's class hash in the committed state is now `0xdeadbeef`.
5. Any subsequent call to the contract causes the OS to look up `contract_class_changes[0xdeadbeef]` → `0`, then `find_element(..., key=0)` → panic.
6. The sequencer cannot include any call to this contract in a provable block.
7. All funds held by the contract are permanently inaccessible — no withdrawal, no recovery, no upgrade path without a protocol-level intervention.

**High — Network halt (secondary path).**

If the attacker additionally sends an L1→L2 message to the affected contract, the sequencer is eventually obligated to process the L1 handler. Attempting to prove a block containing that L1 handler causes the OS to abort, blocking block finalization.

---

### Likelihood Explanation

The `replace_class` syscall is available to any contract without restriction. Any unprivileged user can:
- Deploy a contract (permissionless on StarkNet).
- Call `replace_class` with an arbitrary felt value from within that contract.
- The OS imposes no constraint on the value; the TODO comment in the source code explicitly acknowledges the missing check.

The attack requires no special privileges, no leaked keys, and no third-party compromise.

---

### Recommendation

In `execute_replace_class` (both `syscall_impls.cairo` and `deprecated_execute_syscalls.cairo`), add a `dict_read` against `contract_class_changes` to verify the requested class hash has a non-zero compiled class hash entry before updating `contract_state_changes`:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors how `execute_entry_point` consumes the class hash — the same internal accounting (`contract_class_changes`) must be used at write time (replace) as at read time (execute), closing the bypass.

---

### Proof of Concept

**Step 1 — Attacker deploys a vault contract** that accepts user deposits and exposes a `replace_class` call path.

**Step 2 — Users deposit funds** into the vault.

**Step 3 — Attacker calls `replace_class(0xdeadbeef)`** from within the vault contract. The OS executes:

```cairo
// syscall_impls.cairo execute_replace_class
let class_hash = request.class_hash;  // = 0xdeadbeef (undeclared)
// No check against contract_class_changes here.
dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new StateEntry(class_hash=0xdeadbeef, ...), felt),
);
```

The state is committed with `class_hash = 0xdeadbeef`.

**Step 4 — Any user attempts to withdraw.** The OS executes `execute_entry_point` for the vault:

```cairo
// execute_entry_point.cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=0xdeadbeef);
// compiled_class_hash = 0  (never declared)
let (compiled_class_fact) = find_element(..., key=0);
// PANIC: 0 not found in compiled_class_facts_bundle
```

**Step 5 — OS aborts.** No proof can be generated for any block containing a call to the vault. All deposited funds are permanently frozen.
