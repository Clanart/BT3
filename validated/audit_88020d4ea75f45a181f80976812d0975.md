### Title
Missing Class Hash Existence Validation in `execute_replace_class` Enables Permanent Contract Bricking via Library Call — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS does not verify that the new class hash corresponds to a declared contract class. A malicious declared class can exploit this via a `library_call` — the StarkNet analog of `delegatecall` — to permanently replace the calling contract's class hash with an invalid value, bricking the contract and permanently freezing any funds it holds.

---

### Finding Description

**Root Cause — Missing Validation in `execute_replace_class`:**

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall. At line 898, there is an explicit TODO comment acknowledging a missing OS-level validation:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
``` [1](#0-0) 

The function reads `class_hash` directly from the request and writes it into the contract's `StateEntry` without any existence check. Any value — including `0` or an arbitrary undeclared hash — is accepted by the OS proof.

**Attack Vector — `library_call` as the Entry Point:**

`execute_library_call` (line 237) is the StarkNet analog of Ethereum's `delegatecall`. It executes code from an arbitrary `request.class_hash` in the full context of the calling contract: same `contract_address`, same storage, same nonce. [2](#0-1) 

Critically, during a library call, `execution_context.execution_info.contract_address` is the **caller's** address. When `execute_syscalls` dispatches `REPLACE_CLASS_SELECTOR`, it passes `execution_context.execution_info.contract_address` as the target: [3](#0-2) 

This means any `replace_class` syscall issued inside a library call replaces the **calling contract's** class hash — not the library's.

**Full Attack Path:**

1. Attacker declares a valid malicious class `MaliciousLib` whose logic calls `replace_class(class_hash=0)` (or any undeclared hash).
2. A victim contract `VictimProxy` (holding ERC-20 tokens) makes a `library_call` to `MaliciousLib` — e.g., as part of a plugin, upgrade, or arbitrary computation pattern.
3. `MaliciousLib` executes `replace_class(class_hash=0)` within the victim's execution context.
4. `execute_replace_class` in the OS processes this without validating the new class hash (line 898 TODO is unimplemented).
5. `VictimProxy`'s `StateEntry.class_hash` is permanently set to `0`.
6. All future calls to `VictimProxy` fail: the OS cannot resolve a class with hash `0`.
7. Any tokens held by `VictimProxy` are permanently inaccessible. [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an invalid/undeclared value, the contract cannot execute any code. It cannot self-recover via another `replace_class` call because there is no valid class to execute. Any ERC-20 tokens, NFTs, or other assets held by the contract are permanently inaccessible with no recovery path.

---

### Likelihood Explanation

**Medium.** The attack requires a victim contract to make a `library_call` to an attacker-controlled class. This is a realistic scenario for:
- Contracts using plugin architectures (the exact pattern described in the original report's proxy/plugin model).
- Upgradeable contracts that call arbitrary library classes.
- Any contract that performs arbitrary computations via library calls.

The missing validation is explicitly acknowledged in the codebase via the TODO comment at line 898, confirming it is a known, unimplemented gap in the OS proof system. The attacker's class is fully valid and declared — only the `replace_class` argument is invalid, and the OS does not check it.

---

### Recommendation

Implement the check noted in the TODO at line 898 of `syscall_impls.cairo`: before updating the contract's `StateEntry`, verify that `request.class_hash` exists in `contract_class_changes` (i.e., it is a declared class). This enforcement must occur at the OS level so the proof system rejects invalid class hash replacements regardless of sequencer behavior.

---

### Proof of Concept

```
// Step 1: Attacker declares MaliciousLib (a valid, declared class)
// MaliciousLib.__execute__() calls: replace_class(class_hash=0)

// Step 2: VictimProxy holds 1000 STRK and calls:
library_call(
    class_hash = MaliciousLib.class_hash,
    selector   = __execute__,
    calldata   = []
)

// Step 3: OS dispatches execute_library_call (syscall_impls.cairo:237)
//   → execution_context.contract_address = VictimProxy.address
//   → MaliciousLib code runs, issues replace_class(class_hash=0)

// Step 4: OS dispatches execute_replace_class (syscall_impls.cairo:878)
//   → line 898 TODO: no declared-class check performed
//   → VictimProxy.StateEntry.class_hash := 0

// Step 5: VictimProxy.class_hash == 0 in finalized state
//   → All future calls to VictimProxy: OS cannot find class 0 → revert
//   → 1000 STRK permanently frozen
```

This is the direct StarkNet analog of the reported SELFDESTRUCT-via-delegatecall vulnerability:

| EVM (Original Report) | StarkNet (This Finding) |
|---|---|
| `delegatecall` to plugin | `library_call` to malicious class |
| Plugin calls `SELFDESTRUCT` | Class calls `replace_class(0)` |
| Proxy contract destroyed | Contract class hash set to invalid |
| ETH drained / proxy bricked | Tokens frozen / contract bricked |
| Registry mapping stale | State entry permanently corrupted |

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
