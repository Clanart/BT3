### Title
Missing Class Declaration Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not validate that the requested new class hash corresponds to a previously declared contract class. This is an acknowledged gap, marked with a `TODO` in production code. A contract can replace its own class with an arbitrary, undeclared class hash. The OS commits this invalid state to the global state tree. Any funds held by the affected contract become permanently inaccessible, as all future calls to it will fail to resolve the class.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall. After deducting gas, it reads the requested `class_hash` directly from the syscall request and writes it into `contract_state_changes` without any check that the hash corresponds to a declared class:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The `contract_class_changes` dict — which tracks declared classes — is never consulted. The OS has no mechanism to reject a `replace_class` call that targets an undeclared class hash.

This is directly analogous to the SDLPool report: in SDLPool, a two-level authorization system (operator approval + token-level approval) fails to enforce that revoking the first level clears the second. Here, a two-level class system (class declaration + class replacement) fails to enforce that the replacement target satisfies the first level (declaration). In both cases, a prerequisite authorization/state check is absent, allowing an invalid state transition to be committed. [1](#0-0) 

---

### Impact Explanation

**Critical. Permanent freezing of funds.**

Once `execute_replace_class` commits an undeclared class hash to `contract_state_changes`, the state is finalized and committed to the global Patricia Merkle Tree via `state_update`. There is no rollback path at the OS level for a committed state root. Any subsequent transaction targeting the affected contract will attempt to load a class that does not exist in `contract_class_changes`, causing execution to fail unconditionally. All funds (tokens, ETH, STRK) held in the contract's storage become permanently inaccessible. [2](#0-1) 

---

### Likelihood Explanation

The `replace_class` syscall is callable by any contract from within its own execution — no privileged role is required. Realistic trigger paths include:

1. **Malicious contract deployer**: A deployer creates a contract that accepts user deposits, then calls `replace_class` with an arbitrary felt (e.g., `1`) as the class hash. The OS accepts it. Deposited funds are permanently frozen. This is a griefing/extortion vector distinct from simple theft.
2. **Buggy upgradeable contract**: A legitimate upgradeable contract has a flaw (e.g., missing access control on its upgrade function) that allows an attacker to supply an arbitrary class hash to `replace_class`. The OS does not act as a safety net.
3. **Reentrancy-triggered replacement**: A contract vulnerable to reentrancy could be manipulated into calling `replace_class` with attacker-controlled data mid-execution.

The missing check is confirmed by the production `TODO` comment, meaning this is not a theoretical gap — it is an explicitly known, unimplemented validation. [3](#0-2) 

---

### Recommendation

Before updating `contract_state_changes`, `execute_replace_class` must verify that `class_hash` exists as a key in `contract_class_changes` with a non-zero value (i.e., it has been declared). Concretely:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors how `execute_declare_transaction` enforces `prev_value=0` to prevent double-declaration — the same dict should be used as the authoritative source of declared classes. [4](#0-3) 

---

### Proof of Concept

1. Attacker deploys `VaultContract` with a legitimate class hash `C_valid`. Users deposit funds.
2. Attacker (or any caller with access to the contract's execute path) triggers a call that internally invokes the `replace_class` syscall with `class_hash = 0xdeadbeef` (an arbitrary felt never declared).
3. The OS executes `execute_replace_class`:
   - Gas is deducted.
   - `class_hash = 0xdeadbeef` is read from the request.
   - **No check against `contract_class_changes`.**
   - `contract_state_changes[VaultContract.address]` is updated: `class_hash` field set to `0xdeadbeef`.
   - Revert log entry is written.
4. Block is finalized. `state_update` squashes and commits the state. The global state root now encodes `VaultContract` with class `0xdeadbeef`.
5. Any subsequent `invoke` to `VaultContract` fails: the OS reads `class_hash = 0xdeadbeef` from state, looks it up in the class tree, finds nothing, and cannot execute. All funds are permanently frozen. [5](#0-4) [6](#0-5)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
