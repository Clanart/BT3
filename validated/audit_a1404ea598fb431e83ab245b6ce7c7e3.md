### Title
Stale `feemarketParams` Snapshot in `NewDynamicFeeChecker` Allows Cosmos Transactions to Bypass Current EIP-1559 Base Fee Enforcement - (File: ante/evm/fee_checker.go)

### Summary
`NewDynamicFeeChecker`, used for Cosmos-native transactions with `ExtensionOptionDynamicFeeTx`, captures `feemarketParams` (including `BaseFee`) at handler construction time via a pointer to a local variable. Because the base fee is never refreshed from the live store, every subsequent Cosmos transaction is fee-checked against a permanently stale base fee, not the current block's base fee. An unprivileged user can submit Cosmos transactions with fees calibrated to the stale (potentially much lower) base fee and have them accepted even when the live base fee is significantly higher.

### Finding Description

`newCosmosAnteHandler` in `evmd/ante/handler_options.go` is called once at application initialization. It reads `feemarketParams` from the store at that moment and passes a pointer to the local copy into `NewDynamicFeeChecker`:

```go
func newCosmosAnteHandler(ctx sdk.Context, options HandlerOptions, ...) sdk.AnteHandler {
    evmParams := options.EvmKeeper.GetParams(ctx)
    feemarketParams := options.FeeMarketKeeper.GetParams(ctx)   // ← snapshot at init
    ...
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)

`types.GetBaseFee` reads directly from the struct field `feemarketParams.BaseFee`, which is the value frozen at construction time:

```go

The base fee is updated every block in `BeginBlock` via `SetBaseFee`, which writes the new value into the persistent KV store and into `params.BaseFee`: [4](#0-3) 

But the local `feemarketParams` struct captured in `newCosmosAnteHandler` is never updated; it remains frozen at the genesis/initialization value for the entire lifetime of the process.

**Contrast with the EVM ante handler**, which correctly reads the current base fee on every transaction by calling `EVMBlockConfig(ctx, ...)`, which fetches the live value from the object store: [5](#0-4) [6](#0-5) 

### Impact Explanation

Cosmos transactions that carry `ExtensionOptionDynamicFeeTx` (the EIP-1559 fee extension for native Cosmos txs) are fee-validated against the stale initial base fee rather than the live one. If the base fee has risen since node startup (e.g., after sustained high-gas blocks), an attacker can submit Cosmos transactions with fees just above the initial base fee. `NewDynamicFeeChecker` accepts them because `feeCap >= staleBaseFee`, while the actual required fee is `feeCap >= currentBaseFee`. The fee collector receives less than the protocol requires, and the block proposer's fee income is mis-accounted. This matches: **"High. EVM state transition, gas refund, fee market, ante handler, mempool, or proposal handling bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."**

### Likelihood Explanation

The base fee adjusts every block. On any chain with sustained above-target gas usage, the live base fee will diverge from the initialization value within minutes. Any user who knows the initial base fee (it is public) can exploit this immediately by submitting Cosmos txs with `ExtensionOptionDynamicFeeTx` at the stale price. No special privileges are required.

### Recommendation

`NewDynamicFeeChecker` must read the live base fee from the store on every invocation rather than from a captured snapshot. The simplest fix is to pass the `FeeMarketKeeper` (or `EVMKeeper`) into the checker and call `GetBaseFee(ctx)` inside the returned closure, mirroring what `newEthAnteHandler` does via `EVMBlockConfig`:

```go
// Instead of capturing &feemarketParams at construction time:
func NewDynamicFeeChecker(ethCfg *params.ChainConfig, evmParams *types.Params,
    fmKeeper FeeMarketKeeper) authante.TxFeeChecker {
    return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
        liveParams := fmKeeper.GetParams(ctx)          // fresh every call
        baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, &liveParams)
        ...
    }
}
```

Alternatively, align `newCosmosAnteHandler` with `newEthAnteHandler` by calling `EVMBlockConfig(ctx, ...)` inside the returned closure so that all fee checks use the same per-block cached config.

### Proof of Concept

1. Node starts; initial base fee = B₀ (e.g., 1 gwei). `newCosmosAnteHandler` captures `feemarketParams.BaseFee = B₀`.
2. Over N blocks of high gas usage, `BeginBlock` raises the live base fee to B₁ >> B₀ (e.g., 100 gwei). The KV store and `params.BaseFee` are updated each block.
3. Attacker submits a Cosmos tx with `ExtensionOptionDynamicFeeTx`, setting `MaxPriorityPrice` and fees such that `feeCap = B₀ + 1`.
4. `NewDynamicFeeChecker` evaluates `feeCap.LT(baseFeeInt)` where `baseFeeInt = B₀` (stale). The check passes.
5. The tx is included in the block. The fee collector receives `(B₀ + 1) * gas` instead of the required `B₁ * gas`, underpaying by `(B₁ - B₀ - 1) * gas` per transaction. [7](#0-6) [8](#0-7)

func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
}
