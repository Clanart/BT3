### Title
`ProofOfSpace::update_digest` Panics on Garbage v2 Proof Bytes via `.expect()` — (`crates/chia-protocol/src/proof_of_space.rs`)

---

### Summary

`ProofOfSpace::update_digest` unconditionally calls `self.quality_string().expect(...)` for v2 proofs. `quality_string()` returns `None` for any proof bytes that fail cryptographic validation. The `parse` function accepts arbitrary proof bytes without validating them. Any call to `get_hash()` on a parsed block containing a v2 `ProofOfSpace` with invalid proof bytes will panic, crashing the node.

---

### Finding Description

**`parse` accepts garbage proof bytes without validation.**

For v2 proofs (`version == 1`), `parse` only checks that exactly one of `pool_public_key` / `pool_contract_puzzle_hash` is set. The `proof` field is stored as raw bytes with no cryptographic check: [1](#0-0) 

**`quality_string()` returns `None` for invalid proof bytes.**

The doc comment explicitly states it: *"returns None if this is a v1 proof or if the proof is invalid."* It delegates to `chia_pos2::quality_string_from_proof`, which returns `None` on any cryptographic failure: [2](#0-1) 

**`update_digest` panics via `.expect()` on that `None`.**

For v2 proofs, `update_digest` calls `quality_string()` and immediately unwraps with `.expect()`. There is no error return path — it panics: [3](#0-2) 

**The panic propagates up through `RewardChainBlock` and `FullBlock`.**

`RewardChainBlock::update_digest` calls `self.proof_of_space.update_digest(digest)` unconditionally: [4](#0-3) 

`FullBlock::update_digest` calls `self.reward_chain_block.update_digest(digest)`: [5](#0-4) 

`get_hash()` is the public entry point that calls `update_digest`. It is exposed on `ProofOfSpace`, `RewardChainBlock`, and `FullBlock`: [6](#0-5) [7](#0-6) 

---

### Impact Explanation

A single malformed `FullBlock` or `RewardChainBlock` with a v2 `ProofOfSpace` containing garbage `proof` bytes will cause any node that calls `get_hash()` on it to panic. In Rust, an unhandled `panic!` in a non-`catch_unwind` context terminates the thread or process. Since `get_hash()` is called during normal block processing (header hash computation for deduplication, caching, and consensus), this is a **consensus-critical node crash / chain halt** triggerable by a single network packet from an unprivileged attacker.

---

### Likelihood Explanation

The attack requires only:
1. Constructing a syntactically valid `FullBlock` with `version=1` in the `ProofOfSpace` and random bytes in the `proof` field.
2. Broadcasting it to any node.

No keys, privileges, or chain state are needed. The `parse` function accepts the block without error. The panic fires the moment the node computes the block's hash.

---

### Recommendation

Replace the `.expect()` with proper error propagation. The `Streamable::update_digest` trait method signature (`fn update_digest(&self, digest: &mut Sha256)`) does not return a `Result`, which is the root design issue. Options:

1. **Validate proof bytes during `parse`**: Call `quality_string_from_proof` inside `parse` for v2 proofs and return `Err(Error::InvalidPoS)` if it fails. This rejects malformed blocks at the network boundary before any hash computation.
2. **Change `update_digest` to return `Result`**: Propagate the error up through `RewardChainBlock` and `FullBlock` so callers can handle it gracefully instead of panicking.

Option 1 is simpler and aligns with the existing structural validation already present in `parse`.

---

### Proof of Concept

```rust
use chia_protocol::{Bytes, Bytes32, ProofOfSpace};
use chia_bls::G1Element;
use chia_traits::Streamable;

// Construct a v2 ProofOfSpace with garbage proof bytes
let pos = ProofOfSpace::new(
    Bytes32::default(),
    Some(G1Element::default()), // pool_pk set, pool_contract not set (valid structure)
    None,
    G1Element::default(),
    1,       // version = 1 (v2)
    0u16,    // plot_index
    0u8,     // meta_group
    2u8,     // strength
    0u8,     // size (unused for v2)
    Bytes::from(vec![0xde, 0xad, 0xbe, 0xef]), // garbage proof bytes
);

// This panics: quality_string() returns None, .expect() fires
let _hash = pos.get_hash(); // PANIC: "internal error. Can't compute hash of invalid ProofOfSpace"
```

The same panic is triggered transitively by `RewardChainBlock::get_hash()` and `FullBlock::get_hash()` when they contain such a `ProofOfSpace`.

    pub fn quality_string(&self) -> Option<Bytes32> {
        if self.version != 1 {
            return None;
        }

        let k_size = (self.proof.len() * 8 / 128) as u8;
        let plot_id = self.compute_plot_id().to_bytes();
        chia_pos2::quality_string_from_proof(&plot_id, k_size, self.strength, self.proof.as_slice())
            .map(|quality| {
                let mut sha256 = Sha256::new();
                sha256.update(chia_pos2::serialize_quality(
                    &quality.chain_links,
                    self.strength,
                ));
                sha256.finalize().into()
            })
    }
