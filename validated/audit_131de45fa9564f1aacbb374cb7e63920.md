### Title
`replace_class` Allows Setting `class_hash` to `UNINITIALIZED_CLASS_HASH` (0), Enabling Contract Re-Deployment at Occupied Addresses — (`File: execution/syscall_impls.cairo`, `execution/deprecated_execute_syscalls.cairo`, `execution/deploy_contract.cairo`)

---

### Summary

`UNINITIALIZED_CLASS_HASH = 0` is the sentinel value used by `deploy_contract` to determine whether a contract address is unoccupied. However, neither `execute_replace_class` in `syscall_impls.cairo` nor its deprecated counterpart in `deprecated_execute_syscalls.cairo` validates that the supplied `class_hash` is non-zero. A deployed contract can therefore call `replace_class(class_hash=0)`, resetting its own `class_hash` to the uninitialized sentinel. The OS then cannot distinguish this "reset" state from a genuinely undeployed address, allowing a second `deploy` syscall to succeed at the same address, running a new constructor over the original contract's preserved storage.

---

### Finding Description

**Root cause — `UNINITIALIZED_CLASS_HASH = 0` is both the "undeployed" sentinel and a value `replace_class` can freely set.**

`UNINITIALIZED_CLASS_HASH` is defined as the literal `0`:

```cairo
const UNINITIALIZED_CLASS_HASH = 0;
```

`deploy_contract` uses this value as the sole guard against re-deployment:

```cairo
assert state_entry.class_hash = UNINITIALIZED_CLASS_HASH;
assert state_entry.nonce = 0;
```

`execute_replace_class` (non-deprecated path) performs no validation on the incoming `class_hash`:

```cairo
let class_hash = request.class_hash;
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}
tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(...);
```

The deprecated path in `deprecated_execute_syscalls.cairo` has the identical omission:

```cairo
let class_hash = syscall_ptr.class_hash;
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}
tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(...);
```

Because `class_hash=0` is accepted, a contract can call `replace_class(0)` and set its own `class_hash` to `UNINITIALIZED_CLASS_HASH`. After this call, the contract's `StateEntry` has `class_hash=0` and (for non-account contracts) `nonce=0`, which is byte-for-byte identical to the state of a never-deployed address. The `deploy_contract` guard then passes unconditionally, and a second deployment to the same address succeeds, running a new constructor over the original contract's preserved storage.

---

### Impact Explanation

**Critical — Direct loss of funds.**

A contract that holds user funds (vault, DEX pool, multisig) can be redeployed with an attacker-controlled class if the contract can be induced to call `replace_class(0)`. The new constructor executes against the original contract's storage (the `storage_ptr` is carried over verbatim in `deploy_contract`). A malicious replacement class can read and drain any token balances or ETH stored in that storage, causing direct, irreversible loss of funds for all depositors.

---

### Likelihood Explanation

**Medium.**

- Any deployed contract can call `replace_class` with an arbitrary `class_hash`, including 0. No privileged role is required.
- The attack is constrained to contracts whose nonce is 0 (i.e., non-account contracts that have never sent a transaction), which is the common case for DeFi contracts.
- The attacker must be able to trigger a `replace_class(0)` call from within the target contract. This is achievable if: (a) the attacker controls the contract, or (b) the contract exposes a `replace_class` path reachable by an unprivileged caller (e.g., via a reentrancy or an unguarded upgrade function).
- The TODO comment in `syscall_impls.cairo` (`// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.`) confirms the validation gap is known but unresolved.

---

### Recommendation

Add an explicit non-zero check for `class_hash` in both `execute_replace_class` implementations:

```cairo
// In syscall_impls.cairo execute_replace_class:
let class_hash = request.class_hash;
with_attr error_message("Invalid class hash: class hash cannot be zero.") {
    assert_not_zero(class_hash);
}
```

```cairo
// In deprecated_execute_syscalls.cairo execute_replace_class:
let class_hash = syscall_ptr.class_hash;
with_attr error_message("Invalid class hash: class hash cannot be zero.") {
    assert_not_zero(class_hash);
}
```

This mirrors the guard already applied in `execute_declare_transaction`:

```cairo
assert_not_zero(compiled_class_hash);
```

### Proof of Concept

1. Attacker deploys contract **A** at address `addr` using class **X**. Class X contains a function `reset()` that calls `replace_class(class_hash=0)`. Contract A accumulates user deposits in its storage (e.g., a balance mapping). At this point: `state_entry[addr] = {class_hash: X, storage_ptr: <deposits>, nonce: 0}`.

2. Attacker calls `reset()` on contract A. `execute_replace_class` in `syscall_impls.cairo` (line 896–910) accepts `class_hash=0` without validation and writes: `state_entry[addr] = {class_hash: 0, storage_ptr: <deposits>, nonce: 0}`.

3. Attacker submits a `deploy` syscall targeting `addr` with malicious class **Y**. `deploy_contract` (line 53–54) checks:
   - `assert state_entry.class_hash = UNINITIALIZED_CLASS_HASH` → `0 == 0` ✓
   - `assert state_entry.nonce = 0` → `0 == 0` ✓
   Both assertions pass. The new state entry is written with `class_hash=Y` and the original `storage_ptr` (line 56–60).

4. Class Y's constructor executes with full access to the original storage containing user deposits and drains them.

func execute_replace_class{contract_state_changes: DictAccess*, revert_log: RevertLogEntry*}(
    contract_address, syscall_ptr: ReplaceClass*

    alloc_locals;
    let class_hash = syscall_ptr.class_hash;

    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),

    );
