### Title
`ProofOfInclusion::valid()` Is Tautologically True for Empty `layers` — Forged Inclusion Proof Accepted Without Root Verification - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

---

### Summary

`ProofOfInclusion::valid()` contains a self-referential final check that is unconditionally `true` when the `layers` vector is empty. Because `root_hash()` falls back to `self.node_hash` in the empty-layers case, the terminal comparison `existing_hash == self.root_hash()` reduces to `self.node_hash == self.node_hash`, which is always satisfied. An attacker can construct a `ProofOfInclusion` with any arbitrary `node_hash` and an empty `layers` list, submit it through the Python/wasm binding or Streamable deserialization path, and have `valid()` return `true` — without the proof being checked against any actual Merkle tree root.

---

### Finding Description

`ProofOfInclusion` is a `Streamable` struct with two fields: `node_hash: Hash` and `layers: Vec<ProofOfInclusionLayer>`. The `valid()` method is the sole self-contained correctness check for the proof:

```rust
// crates/chia-datalayer/src/merkle/proof_of_inclusion.rs

pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash          // ← fallback when layers is empty
    }
}

pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {   // ← loop body never executes when layers == []
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()   // ← node_hash == node_hash → always true
}
```

When `layers` is empty:

| Step | Value |
|---|---|
| `existing_hash` initialized | `self.node_hash` |
| Loop body | skipped |
| `self.root_hash()` | `self.node_hash` (empty-layers branch) |
| Final check | `self.node_hash == self.node_hash` → **`true`** |

The check is tautological. No layer hash is verified, and no external root hash is consulted. The `valid()` function never rejects a proof with empty `layers`, regardless of what `node_hash` contains.

This is the direct analog of the Linea bug: just as `startingParentFinalStateRootHash` is empty when `startingDataParentHash` is empty — making the comparison `startingParentFinalStateRootHash != _finalizationData.parentStateRootHash` fail to guard the initial state — here `root_hash()` collapses to `node_hash` when `layers` is empty, making the comparison `existing_hash == self.root_hash()` fail to guard against any forged proof.

The struct is fully constructible from untrusted input via:
- `Streamable` deserialization (`from_bytes`, `parse_rust`)
- Python binding constructor (`ProofOfInclusion(node_hash=..., layers=[])`)
- JSON deserialization (`from_json_dict`)

All callers in the codebase — the fuzz target, the Rust unit tests, and the Python test suite — call only `proof.valid()` without separately asserting `proof.root_hash() == known_root`:

```rust
// crates/chia-datalayer/fuzz/fuzz_targets/proofs_of_inclusion.rs
let proof = blob.get_proof_of_inclusion(key).unwrap();
assert!(proof.valid());   // ← no root check
```

```

---

### Impact Explanation

An attacker who can supply a `ProofOfInclusion` to any consumer that calls only `proof.valid()` can forge a proof of inclusion for any `node_hash` value in any DataLayer tree. The forged proof passes `valid()` unconditionally. Because DataLayer Merkle trees are used to commit to key-value state on-chain, accepting a forged inclusion proof allows an untrusted party to assert that an arbitrary key-value pair exists in a tree whose on-chain root hash does not actually contain it. This matches the allowed High impact: **DataLayer Merkle proof logic accepts forged inclusion, letting untrusted input prove invalid state.**

---

### Likelihood Explanation

`ProofOfInclusion` is a `Streamable` type exposed through the Python wheel and wasm bindings. Any code path that receives a `ProofOfInclusion` from an external peer and validates it solely with `proof.valid()` is exploitable with a trivially crafted two-field object. The attacker needs only to set `node_hash` to the hash they wish to claim is included and leave `layers` empty. No cryptographic knowledge or key material is required.

---

### Recommendation

`valid()` must be changed to require an external expected root hash parameter, or the final check must explicitly reject the empty-layers case. Two options:

**Option A** — Reject empty `layers` (a single-node tree is a degenerate case that should not be proven this way):
```rust
pub fn valid(&self) -> bool {
    if self.layers.is_empty() {
        return false;
    }
    // ... existing logic
}
```

**Option B** — Require the caller to supply the expected root and compare against it:
```rust
pub fn valid_against_root(&self, expected_root: &Hash) -> bool {
    // ... existing layer-chain logic ...
    existing_hash == *expected_root
}
```

Option B is the more correct fix: it removes the self-referential `root_hash()` fallback entirely and forces every call site to supply the authoritative root hash from an independent source (e.g., the on-chain commitment).

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, MerkleBlob
from chia_rs import bytes32

# Build a real tree with one entry
blob = MerkleBlob(bytearray())
# ... insert real entries, compute root ...
real_root = blob.get_root()

# Forge a proof claiming an arbitrary hash is in the tree
arbitrary_node_hash = bytes32(b"\xde\xad" * 16)
forged_proof = ProofOfInclusion(node_hash=arbitrary_node_hash, layers=[])

# valid() returns True — no layers means no checks
assert forged_proof.valid()                        # passes unconditionally
assert forged_proof.root_hash() == arbitrary_node_hash  # "root" is just node_hash

# The forged proof is accepted even though arbitrary_node_hash != real_root
# and the key was never inserted into the tree.
