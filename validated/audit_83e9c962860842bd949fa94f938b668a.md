### Title
Missing Zero-Check on `class_hash` in `execute_replace_class` Allows Permanent Contract Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts a caller-supplied `class_hash` of zero without any validation. Because `0` is the sentinel `UNINITIALIZED_CLASS_HASH` value, a contract that calls `replace_class(0)` will have its class permanently set to the uninitialized state. Any subsequent call to that contract will fail to resolve a compiled class, permanently freezing all funds held by the contract. The same missing check exists in the deprecated syscall path.

---

### Finding Description

In `execute_replace_class` (`syscall_impls.cairo`, lines 878–916), the OS reads `class_hash` directly from the syscall request and writes it into `contract_state_changes` with no validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-left TODO explicitly acknowledges the missing check: `// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.`

`class_hash = 0` is the `UNINITIALIZED_CLASS_HASH` sentinel (imported and asserted as such in `deploy_contract.cairo`, line 53: `assert state_entry.class_hash = UNINITIALIZED_CLASS_HASH`). The OS will commit this zero class hash to the state trie without objection.

The identical missing check exists in the deprecated path at `deprecated_execute_syscalls.cairo`, lines 307–329, in `execute_replace_class`. [1](#0-0) [2](#0-1) [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to `0` (UNINITIALIZED_CLASS_HASH) via `replace_class(0)`, every subsequent call to that contract will attempt to look up compiled class facts for hash `0`. No such compiled class exists or can be declared (the OS enforces `prev_value=0` on class declaration, meaning class hash `0` can never be legitimately declared). The contract becomes permanently unexecutable. Any ERC-20 tokens, ETH, STRK, or other assets held in that contract's storage are irrecoverably frozen on-chain.

---

### Likelihood Explanation

**Medium.** The syscall is reachable by any unprivileged contract deployer. A contract that contains a bug (e.g., an unchecked external input passed to `replace_class`) or a deliberately malicious contract can trigger this path. Because the OS provides no guard, the invalid state is committed to the canonical state trie and is irreversible. The attack requires no privileged role, no leaked key, and no operator cooperation — only the ability to deploy and execute a contract.

---

### Recommendation

Add an `assert_not_zero(class_hash)` guard immediately after reading `class_hash` from the request in both `execute_replace_class` implementations:

```cairo
// In syscall_impls.cairo
let class_hash = request.class_hash;
with_attr error_message("Class hash must not be zero.") {
    assert_not_zero(class_hash);
}
```

```cairo
// In deprecated_execute_syscalls.cairo
let class_hash = syscall_ptr.class_hash;
with_attr error_message("Class hash must not be zero.") {
    assert_not_zero(class_hash);
}
```

Additionally, resolve the existing TODO by verifying that `class_hash` corresponds to a previously declared class in `contract_class_changes` before committing the state update.

---

### Proof of Concept

1. Deploy a contract `VictimContract` that holds STRK tokens and exposes a public function `self_destruct()`.
2. Inside `self_destruct()`, emit the `replace_class` syscall with `class_hash = 0`.
3. Call `VictimContract.self_destruct()` from any unprivileged account.
4. The OS executes `execute_replace_class`:
   - Reads `class_hash = 0` from the syscall request.
   - No `assert_not_zero` check is present.
   - Writes `StateEntry(class_hash=0, ...)` into `contract_state_changes`.
5. The state update is committed to the Patricia Merkle trie with `VictimContract`'s class hash set to `UNINITIALIZED_CLASS_HASH = 0`.
6. Any subsequent invoke transaction targeting `VictimContract` will fail at class resolution — the OS finds no compiled class for hash `0`.
7. All STRK held by `VictimContract` is permanently frozen with no recovery path. [4](#0-3)

        // Not enough gas to execute the syscall.
        return ();
