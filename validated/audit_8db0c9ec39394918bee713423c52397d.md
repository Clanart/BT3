### Title
Missing Declared-Class Existence Check in `execute_replace_class` Allows Permanent Contract Freezing — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS Cairo program accepts any arbitrary `class_hash` value from a `ReplaceClassRequest` and writes it directly into `contract_state_changes` without verifying that the class hash is actually declared in `contract_class_changes`. This is the direct analog of the BeaconDS bug: just as `setImplementation` accepted an undeployed address, `execute_replace_class` accepts an undeclared class hash. A contract that replaces its class with an undeclared hash becomes permanently uncallable, freezing any funds it holds.

---

### Finding Description

`execute_replace_class` in `syscall_impls.cairo` reads the requested class hash from the syscall request and immediately writes it to `contract_state_changes` with no cross-check against `contract_class_changes`:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-inserted TODO at line 898 explicitly acknowledges the missing check. The same omission exists in the deprecated path in `deprecated_execute_syscalls.cairo`.

When any future call targets the affected contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
```

For an undeclared class hash, `dict_read` returns `0` (the uninitialized default). `find_element` with `key=0` then asserts the element exists in `compiled_class_facts_bundle` — which it does not — causing the OS proof to fail for any block that includes a call to the affected contract.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value and committed to the state tree (via `compute_contract_state_commitment` / `hash_contract_state_changes`), the state root permanently encodes the broken class hash. Every subsequent block that includes any call to that contract — including calls needed to withdraw funds — will produce an unprovable OS execution. The sequencer cannot include such calls without invalidating the block proof. Funds held by the contract (e.g., an account contract or vault) are irrecoverably frozen.

---

### Likelihood Explanation

**High.** The `replace_class` syscall is callable by any deployed contract with no privileged role required. The missing check is confirmed by the explicit `TODO` comment in the production code. Any contract whose logic (or a bug in its logic) issues `replace_class` with an arbitrary felt value — including a value that was never declared — will trigger this path. An attacker can deliberately deploy a contract that calls `replace_class(arbitrary_undeclared_hash)` to freeze itself (and any funds it holds) or to demonstrate the bug.

---

### Recommendation

In `execute_replace_class` (both `syscall_impls.cairo` and `deprecated_execute_syscalls.cairo`), after reading `class_hash` from the request, perform a `dict_read` on `contract_class_changes` and assert the returned `compiled_class_hash` is non-zero (i.e., the class is declared):

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the fix recommended for BeaconDS (`extcodesize` check before accepting a new implementation) and ensures the OS enforces class existence at the point of replacement, not only at the point of execution.

---

### Proof of Concept

1. Attacker deploys contract `A` with class `C_valid` (declared).
2. Contract `A` calls `replace_class(0xdeadbeef)` where `0xdeadbeef` is never declared.
3. The OS processes `execute_replace_class`:
   - Reads `class_hash = 0xdeadbeef` from the request.
   - No check against `contract_class_changes`.
   - Writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes`.
4. The block is proven successfully (the OS has no enforcement here).
5. The state root now encodes `contract_A.class_hash = 0xdeadbeef`.
6. In any subsequent block, a call to contract `A` reaches `execute_entry_point`:
   - `dict_read(contract_class_changes, key=0xdeadbeef)` → returns `0`.
   - `find_element(..., key=0)` → asserts element found → **assertion fails**.
   - The block proof is invalid; the sequencer cannot include any call to `A`.
7. Any ETH/tokens held by contract `A` are permanently frozen.
