### Title
Missing Backward-Compatibility Branch Due to Overly Strict `and` Condition in `calculate_global_state_root` — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/state/commitment.cairo`)

---

### Summary

`calculate_global_state_root` uses a single `and` condition to gate the only early-return path. The function's own documentation describes **three** distinct cases, but the code only implements **two**. When `contract_class_root == 0` and `contract_state_root != 0`, the backward-compatibility branch is silently skipped and the function returns a Poseidon hash over a zero class root instead of returning `contract_state_root` directly. This produces a globally incorrect state root, which causes every verifier that follows the documented protocol to reject the OS output — a network-wide commitment failure.

---

### Finding Description

The function comment explicitly documents three cases:

```
// If both the contract class and contract state trees are empty, the global root is set to 0.
// If the contract class tree is empty, the global state root is equal to the
// contract state root (for backward compatibility);
// Otherwise, the global root is obtained by:
//     global_root = H(GLOBAL_STATE_VERSION, contract_state_root, contract_class_root).
```

The implementation is:

```cairo
if (contract_state_root == 0 and contract_class_root == 0) {
    // The state is empty.
    return (global_root=0);
}

tempvar elements: felt* = new (GLOBAL_STATE_VERSION, contract_state_root, contract_class_root);
let (global_root) = poseidon_hash_many(n=3, elements=elements);
return (global_root=global_root);
``` [1](#0-0) 

The `and` guard on line 41 requires **both** roots to be zero before returning early. When `contract_class_root == 0` but `contract_state_root != 0` (the backward-compatibility case), the condition is `false`, execution falls through, and the function returns:

```
H(GLOBAL_STATE_VERSION, contract_state_root, 0)
```

instead of the documented:

```
contract_state_root
```

These are cryptographically distinct values. The OS output therefore carries a wrong `final_root`, which propagates into `state_update_output` and ultimately into the block's OS output header. [2](#0-1) 

---

### Impact Explanation

`calculate_global_state_root` is called inside `state_update`, which feeds `CommitmentUpdate.final_root` into the OS output. Any verifier or L1 contract that independently computes the expected global root using the documented formula will derive `contract_state_root`, while the OS proof asserts `H(GLOBAL_STATE_VERSION, contract_state_root, 0)`. The mismatch causes proof rejection. If the OS is the canonical source of truth, nodes that accept the proof will diverge from nodes that recompute the root — an **unintended chain split**. Alternatively, if all verifiers reject the proof, the network cannot confirm new blocks — a **network halt**.

**Allowed impact matched:** High — Unintended chain split (network partition) / High — Network not being able to confirm new transactions.

---

### Likelihood Explanation

The trigger condition is `contract_class_root == 0` with `contract_state_root != 0`. This occurs in any deployment where contract state has been modified but no Sierra class has yet been declared (i.e., the contract class Patricia tree is still at its genesis root of 0). This is the normal state of a fresh StarkNet deployment or any isolated test chain. An unprivileged user can reach this path simply by submitting any state-modifying transaction (invoke, deploy-account, L1 handler) before the first `declare` transaction is processed. No privileged access is required.

---

### Recommendation

Add the missing backward-compatibility branch between the two existing cases:

```cairo
if (contract_state_root == 0 and contract_class_root == 0) {
    return (global_root=0);
}
// Backward compatibility: when no classes have been declared yet,
// the global root equals the contract state root.
if (contract_class_root == 0) {
    return (global_root=contract_state_root);
}
tempvar elements: felt* = new (GLOBAL_STATE_VERSION, contract_state_root, contract_class_root);
let (global_root) = poseidon_hash_many(n=3, elements=elements);
return (global_root=global_root);
```

---

### Proof of Concept

1. Deploy a fresh StarkNet OS instance. The contract class Patricia tree is empty; `contract_class_root = 0`.
2. Submit any invoke transaction that writes to contract storage (e.g., increment a counter). This makes `contract_state_root != 0`.
3. `state_update` calls `calculate_global_state_root(contract_state_root=X, contract_class_root=0)`.
4. The `and` condition `(X == 0 and 0 == 0)` evaluates to `false` (because `X != 0`).
5. The function computes `poseidon_hash_many([GLOBAL_STATE_VERSION, X, 0])` and returns it as `global_root`.
6. Any independent verifier following the documented protocol computes `global_root = X` (backward-compatibility branch).
7. The two values differ → proof rejected or chain split. [3](#0-2)
