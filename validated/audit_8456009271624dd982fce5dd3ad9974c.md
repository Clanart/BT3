### Title
`compute_meta_tx_v0_hash` Missing Nonce Enables Signature Replay — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/transaction_hash/transaction_hash.cairo`)

---

### Summary

`compute_meta_tx_v0_hash` computes a transaction hash that omits any nonce or expiration timestamp. Because the signed preimage is static for a given `(contract_address, selector, calldata, chain_id)` tuple, any observer who captures a valid user signature can replay it an unlimited number of times against the same or future contract deployments, without the signer's consent.

---

### Finding Description

`compute_meta_tx_v0_hash` delegates to `deprecated_get_transaction_hash` with `additional_data_size=0` and `additional_data=cast(0, felt*)`: [1](#0-0) 

The resulting hash covers only: `INVOKE_HASH_PREFIX | version=0 | contract_address | entry_point_selector | H(calldata) | max_fee=0 | chain_id`. No nonce, no expiration, no block number.

Contrast this with `compute_l1_handler_transaction_hash`, which explicitly passes `additional_data_size=1, additional_data=&nonce` to bind the hash to a single use: [2](#0-1) 

The function signature of `compute_meta_tx_v0_hash` does not even accept a nonce parameter, making it structurally impossible to include one: [3](#0-2) 

The `deprecated_get_transaction_hash` helper that both functions share shows the `additional_data` slot is the only place a nonce could be injected: [4](#0-3) 

---

### Impact Explanation

**Critical — Direct loss of funds.**

A user who signs a `meta_tx_v0` authorizing, for example, an ERC-20 transfer produces a signature that is permanently valid for that exact `(contract_address, selector, calldata)` tuple. Any party who observes the signature on-chain (e.g., from a prior block) can re-submit it as a new `meta_tx_v0` syscall invocation, draining the user's account repeatedly. Because the OS enforces no nonce on this path, every replay is accepted as a fresh, valid transaction.

---

### Likelihood Explanation

**High.** The `meta_tx_v0` syscall is a public protocol input path reachable by any unprivileged transaction sender. Signatures are visible on-chain the moment the first use is included in a block. No special privilege, leaked key, or Sybil attack is required — the attacker only needs to copy the calldata and signature from a confirmed transaction and re-submit.

---

### Recommendation

1. Add a `nonce` parameter to `compute_meta_tx_v0_hash` and pass it as `additional_data` (mirroring `compute_l1_handler_transaction_hash`).
2. Enforce nonce uniqueness in the syscall handler (increment or mark-used after first acceptance).
3. Optionally include an expiration block number in the signed preimage to bound the replay window even if nonce tracking is delayed.

---

### Proof of Concept

1. User signs a `meta_tx_v0` authorizing `transfer(recipient, 100)` on token contract `T`. The hash is `H(PREFIX | 0 | T | transfer_selector | H(calldata) | 0 | chain_id)`.
2. Transaction is included in block N; signature `(r, s)` is now public.
3. Attacker constructs a new transaction invoking the same contract with the same `meta_tx_v0` syscall parameters and the captured `(r, s)`.
4. `compute_meta_tx_v0_hash` produces the identical hash; ECDSA verification passes.
5. The OS executes the transfer again. Steps 3–5 repeat indefinitely, draining the user's balance.

No privileged access, no leaked key, no network-level attack required — only observation of a single confirmed transaction.

func deprecated_get_transaction_hash{hash_ptr: HashBuiltin*}(
    tx_hash_prefix: felt,
    version: felt,
    contract_address: felt,
    entry_point_selector: felt,
    calldata_size: felt,
    calldata: felt*,
    max_fee: felt,
    chain_id: felt,
    additional_data_size: felt,
    additional_data: felt*,
) -> (tx_hash: felt) {
    let (hash_state_ptr) = hash_init();
    let (hash_state_ptr) = hash_update_single(hash_state_ptr=hash_state_ptr, item=tx_hash_prefix);
    let (hash_state_ptr) = hash_update_single(hash_state_ptr=hash_state_ptr, item=version);
    let (hash_state_ptr) = hash_update_single(hash_state_ptr=hash_state_ptr, item=contract_address);
    let (hash_state_ptr) = hash_update_single(
        hash_state_ptr=hash_state_ptr, item=entry_point_selector
    );
    let (hash_state_ptr) = hash_update_with_hashchain(
        hash_state_ptr=hash_state_ptr, data_ptr=calldata, data_length=calldata_size
    );
    let (hash_state_ptr) = hash_update_single(hash_state_ptr=hash_state_ptr, item=max_fee);
    let (hash_state_ptr) = hash_update_single(hash_state_ptr=hash_state_ptr, item=chain_id);

    let (hash_state_ptr) = hash_update(
        hash_state_ptr=hash_state_ptr, data_ptr=additional_data, data_length=additional_data_size
    );

    let (tx_hash) = hash_finalize(hash_state_ptr=hash_state_ptr);

    return (tx_hash=tx_hash);
