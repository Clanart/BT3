### Title
Missing Declared Class Hash Validation in `replace_class` Syscall Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary class hash as a replacement without verifying that the hash corresponds to a previously declared contract class. An unprivileged contract deployer can call `replace_class` with a non-existent class hash, permanently rendering the contract non-callable and freezing all funds held within it.

### Finding Description
In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall by reading the requested class hash directly from the syscall request and writing it into `contract_state_changes` without any validation that the hash corresponds to a declared class:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-acknowledged TODO comment at line 898 explicitly confirms the missing check: `"Check that there is a declared contract class with the given hash."` The OS unconditionally commits the arbitrary class hash to state.

When any subsequent transaction attempts to call this contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
```

A `dict_read` on an undeclared class hash returns `0` (the Cairo dict default). `find_element` with key `0` will then fail with an assertion error if no compiled class with hash `0` exists, making any proof for a block containing such a call invalid. The sequencer is therefore forced to permanently exclude all transactions targeting that contract address, freezing any funds held there.

### Impact Explanation
**Critical — Permanent freezing of funds.**

Any contract that holds tokens or acts as a vault/wallet can have its class replaced with an undeclared hash. Once committed to state, the contract becomes permanently non-callable: every future transaction targeting it causes an OS-level proof failure, so the sequencer must exclude them indefinitely. All assets stored in the contract's storage or sent to its address are irrecoverably frozen with no upgrade or recovery path.

### Likelihood Explanation
**High.** The `replace_class` syscall is available to any deployed contract with no access restriction. A malicious actor needs only to:
1. Deploy a contract (permissionless on StarkNet).
2. Induce users to deposit funds (e.g., by presenting it as a legitimate vault or wallet).
3. Call `replace_class` with an arbitrary undeclared felt value as the class hash.

No privileged role, leaked key, or operator cooperation is required. The OS Cairo program itself enforces no constraint on the new class hash, and the TODO comment confirms this is a known gap in the current implementation.

### Recommendation
Before committing the new class hash to `contract_state_changes`, verify that it exists in `contract_class_changes` (i.e., it has been declared in the current or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` with `class_hash` as the key and assert the result is non-zero:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("replace_class: class hash not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the validation already implicitly performed in `execute_entry_point` and closes the gap identified by the TODO.

### Proof of Concept
1. Deploy contract `MaliciousVault` whose `__execute__` calls `replace_class(class_hash=0xDEAD)` where `0xDEAD` is never declared.
2. Convince users to deposit STRK/ETH into `MaliciousVault`.
3. Submit an invoke transaction calling `MaliciousVault.__execute__`.
4. The OS executes `execute_replace_class`: no validation occurs; `contract_state_changes[MaliciousVault_addr].class_hash = 0xDEAD` is committed.
5. In any subsequent block, a transaction calling `MaliciousVault` reaches `execute_entry_point`: `dict_read(contract_class_changes, 0xDEAD)` returns `0`; `find_element(..., key=0)` fails; the block proof is invalid.
6. The sequencer must permanently exclude all calls to `MaliciousVault`. All deposited funds are frozen.
