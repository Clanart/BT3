### Title
Fee Accounting Bypass via Felt Arithmetic Overflow in `compute_max_possible_fee` - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

`compute_max_possible_fee` computes the maximum chargeable fee using unchecked felt arithmetic (modulo the StarkNet field prime P ≈ 2^251). Because `max_price_per_unit` is only validated to be non-negative (≤ (P−1)/2) while `max_amount` is bounded to 2^64−1, an attacker can craft resource bounds whose product sum wraps to exactly 0 mod P. The `charge_fee` function then hits its early-exit guard `if (max_fee == 0) { return (); }` and skips the ERC-20 fee transfer entirely, executing the transaction at zero cost.

---

### Finding Description

`compute_max_possible_fee` in `transaction_impls.cairo` computes:

```cairo
return l1_gas_bounds.max_amount * l1_gas_bounds.max_price_per_unit
     + l2_gas_bounds.max_amount * (l2_gas_bounds.max_price_per_unit + tx_info.tip)
     + l1_data_gas_bounds.max_amount * l1_data_gas_bounds.max_price_per_unit;
``` [1](#0-0) 

All arithmetic is done in Cairo's felt field (mod P). The only bounds enforced on the inputs come from `pack_resource_bounds` during hash computation:

```cairo
assert_nn_le(resource_bounds.max_amount, 2 ** 64 - 1);
assert_nn(resource_bounds.max_price_per_unit);
``` [2](#0-1) 

`assert_nn` only checks `max_price_per_unit ≥ 0`, bounding it to `[0, (P−1)/2]`. There is **no upper bound** on `max_price_per_unit` beyond (P−1)/2. With `max_amount` up to 2^64−1 and `max_price_per_unit` up to (P−1)/2, a single product can reach ~2^314, far exceeding P, and the sum of three such products can wrap to any value including 0.

`charge_fee` then uses the result as a gate:

```cairo
let max_fee = compute_max_possible_fee(tx_info=tx_info);
if (max_fee == 0) {
    return ();
}
``` [3](#0-2) 

When `max_fee` wraps to 0, the function returns immediately, skipping the ERC-20 `transfer` call to the sequencer entirely. [4](#0-3) 

---

### Impact Explanation

**Direct loss of funds (Critical).** The sequencer receives zero fee for a fully executed transaction. The OS proof is valid — the Cairo constraints are satisfied — so the block is accepted on-chain with the fee transfer omitted. At scale, sequencers that do not independently detect the overflow in their mempool logic will process transactions for free, suffering direct, provable revenue loss. Additionally, if sequencers detect the pattern and halt inclusion of all v3 transactions to protect themselves, the result is a **total network shutdown** (no new transactions confirmed).

---

### Likelihood Explanation

The attack requires only:
1. Knowledge of the StarkNet field prime P = 2^251 + 17·2^192 + 1 (public).
2. Crafting three resource-bound fields whose product sum ≡ 0 (mod P) — a trivial linear algebra problem over a known prime.
3. Submitting a valid v3 transaction (no privileged access required).

Any unprivileged transaction sender can execute this.

---

### Recommendation

Add an explicit upper-bound range check on `max_price_per_unit` in `pack_resource_bounds` (e.g., `assert_nn_le(resource_bounds.max_price_per_unit, 2 ** 128 - 1)`) so that the maximum product per resource type is bounded to 2^192, making the three-term sum provably less than P and eliminating the overflow path. Alternatively, recompute `max_fee` using 256-bit arithmetic (analogous to how `TransferCallData.amount` is already typed as `Uint256`) and compare against a 256-bit cap.

---

### Proof of Concept

Let P = 2^251 + 17·2^192 + 1 (StarkNet field prime).

Choose:
- `l1_gas_bounds.max_amount = 2`, `l1_gas_bounds.max_price_per_unit = (P−1)/2`
- `l2_gas_bounds.max_amount = 1`, `l2_gas_bounds.max_price_per_unit = 1`, `tip = 0`
- `l1_data_gas_bounds.max_amount = 0`, `l1_data_gas_bounds.max_price_per_unit = 0`

All inputs satisfy the enforced constraints:
- `max_amount ∈ {0, 1, 2}` ≤ 2^64−1 ✓
- `max_price_per_unit = (P−1)/2` satisfies `assert_nn` (it equals the maximum non-negative felt) ✓
- `tip = 0` ≤ 2^64−1 ✓

`compute_max_possible_fee` evaluates to:

```
2 · (P−1)/2  +  1 · (1 + 0)  +  0 · 0
= (P − 1) + 1
= P
≡ 0  (mod P)
```

`charge_fee` receives `max_fee = 0`, hits `if (max_fee == 0) { return (); }`, and exits without transferring any fee to the sequencer. The transaction is fully executed and included in the proven block at zero cost to the attacker. [5](#0-4) [6](#0-5) [7](#0-6)

// Returns the maximum possible fee that can be charged for the transaction.
func compute_max_possible_fee(tx_info: TxInfo*) -> felt {
    tempvar resource_bounds: ResourceBounds* = tx_info.resource_bounds_start;
    let n_resource_bounds = (tx_info.resource_bounds_end - resource_bounds) / ResourceBounds.SIZE;

    // Only V3 transactions with all resource bounds are supported.
    assert tx_info.version = 3;
    assert n_resource_bounds = 3;

    tempvar l1_gas_bounds: ResourceBounds = resource_bounds[L1_GAS_INDEX];
    tempvar l2_gas_bounds: ResourceBounds = resource_bounds[L2_GAS_INDEX];
    tempvar l1_data_gas_bounds = resource_bounds[L1_DATA_GAS_INDEX];

    return l1_gas_bounds.max_amount * l1_gas_bounds.max_price_per_unit + l2_gas_bounds.max_amount *
        (l2_gas_bounds.max_price_per_unit + tx_info.tip) + l1_data_gas_bounds.max_amount *
        l1_data_gas_bounds.max_price_per_unit;
}

func pack_resource_bounds{range_check_ptr}(resource_bounds: ResourceBounds) -> felt {
    assert_nn_le(resource_bounds.max_amount, 2 ** 64 - 1);
    assert_nn(resource_bounds.max_price_per_unit);
    return (resource_bounds.resource * 2 ** 64 + resource_bounds.max_amount) * 2 ** 128 +
        resource_bounds.max_price_per_unit;
}

func hash_fee_fields{range_check_ptr, poseidon_ptr: PoseidonBuiltin*}(
    tip: felt, resource_bounds: ResourceBounds*, n_resource_bounds: felt

    alloc_locals;

    let (local data_to_hash: felt*) = alloc();
    assert data_to_hash[0] = tip;
    assert_nn_le(tip, 2 ** 64 - 1);

    static_assert L1_GAS_INDEX == 0;
    static_assert L2_GAS_INDEX == 1;
    static_assert L1_DATA_GAS_INDEX == 2;

    with_attr error_message("Invalid number of resource bounds: {n_resource_bounds}.") {
        assert n_resource_bounds = 3;
    }

    // L1 gas.
    let l1_gas_bounds = resource_bounds[L1_GAS_INDEX];
    assert l1_gas_bounds.resource = L1_GAS;
    assert data_to_hash[1] = pack_resource_bounds(l1_gas_bounds);

    // L2 gas.
    let l2_gas_bounds = resource_bounds[L2_GAS_INDEX];
    assert l2_gas_bounds.resource = L2_GAS;
    assert data_to_hash[2] = pack_resource_bounds(l2_gas_bounds);

    // L1 data gas.

    assert l1_data_gas_bounds.resource = L1_DATA_GAS;
    assert data_to_hash[3] = pack_resource_bounds(l1_data_gas_bounds);

    let (hash) = poseidon_hash_many(n=n_resource_bounds + 1, elements=data_to_hash);
    return hash;
