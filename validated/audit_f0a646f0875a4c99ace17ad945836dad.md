### Title
P2P Sync Accepts Cairo0 Class Body With Mismatched Hash, Storing Wrong Code Under Committed Key - (`crates/apollo_p2p_sync/src/client/class.rs`)

### Summary
A malicious p2p sync peer can send a `protobuf::Cairo0Class` whose `program` field decompresses to a different program than the one whose hash matches `class_hash`. No step in the receive-parse-store pipeline verifies `compute_deprecated_class_hash(body) == class_hash`. The class is stored under the attacker-supplied key, so any subsequent RPC call, fee estimation, or execution that loads that class uses the wrong bytecode.

### Finding Description

**Step 1 — Deserialization without hash check.**

`TryFrom<protobuf::Cairo0Class>` decodes and decompresses the program, constructs `ContractClass{program, entry_points_by_type, abi}`, and returns it — with no call to `compute_deprecated_class_hash`. [1](#0-0) 

**Step 2 — Class hash taken verbatim from the wire.**

`TryFrom<protobuf::Class>` pairs the deserialized body with `value.class_hash`, which is a raw attacker-controlled field. [2](#0-1) 

**Step 3 — Only membership in the state diff is checked, not body integrity.**

`ClassStreamBuilder::parse_data_for_block` verifies `is_declared` (the class_hash must appear in the already-committed state diff) and `duplicate_class`. It does **not** verify that `compute_deprecated_class_hash(deprecated_contract_class) == class_hash`. [3](#0-2) 

**Step 4 — Stored under the attacker-supplied key.**

`write_to_storage` calls `add_deprecated_class(class_hash, deprecated_class)` where `class_hash` is the attacker-supplied value and `deprecated_class` contains the wrong program. A developer TODO at line 39 explicitly acknowledges the missing verification:

> `// TODO(shahak): Verify class hash matches class manager response. report if not.` [4](#0-3) 

### Impact Explanation

After storage, `starknet_getClass(H)` returns a class body whose `compute_deprecated_class_hash` does not equal `H`. Any `starknet_estimateFee`, `starknet_call`, or `starknet_simulateTransactions` that invokes a contract declared as class `H` executes the wrong bytecode, producing wrong fee estimates, wrong return values, and wrong state transitions — all presented as authoritative results to callers.

This matches the allowed impact: **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

### Likelihood Explanation

Any peer that the syncing node connects to can trigger this. The `is_declared` guard only requires that the attacker supply a `class_hash` that genuinely appears in the committed state diff for the block being synced — the attacker does not need to forge the state diff itself. The attacker simply substitutes a different (valid, decompressible) program blob for the correct one.

### Recommendation

After `TryFrom<protobuf::Cairo0Class>` returns the `ContractClass`, compute its hash and compare it to the `class_hash` extracted from the outer `protobuf::Class` before inserting into either the result map or the class manager:

```rust
let computed = compute_deprecated_class_hash(&deprecated_contract_class)
    .map_err(|_| BadPeerError::InvalidClassHash { class_hash })?;
if computed != class_hash {
    return Err(ParseDataError::BadPeer(BadPeerError::InvalidClassHash { class_hash }));
}
```

This resolves the open TODO at line 39 of `class.rs`.

### Proof of Concept

```rust
// 1. Construct two distinct Cairo0 programs: body_A and body_B.
// 2. Compute H = compute_deprecated_class_hash(body_B).
// 3. Build protobuf::Cairo0Class { program: compress_and_encode(body_A), ... }.
// 4. Build protobuf::Class { class_hash: H, class: Cairo0(...) }.
// 5. Call TryFrom<protobuf::Class> -> (ApiContractClass::DeprecatedContractClass(body_A), H).
// 6. Insert into deprecated_declared_classes_result under key H.
// 7. Call add_deprecated_class(H, body_A).
// 8. Assert: compute_deprecated_class_hash(storage.get_deprecated_class(H)) != H.  // PASSES
```

The `is_declared` guard is satisfied as long as `H` is a class hash that legitimately appears in the block's committed state diff, which the attacker knows from the public chain.
