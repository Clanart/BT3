### Title
Integer Division Truncation in `NewDynamicFeeChecker` Causes Zero Effective Fee Deduction for Cosmos Txs — (`File: ante/evm/fee_checker.go`)

### Summary

`NewDynamicFeeChecker` derives the per-gas fee cap via integer division (`fee.Quo(gas)`). When the submitted fee amount is smaller than the gas limit, this truncates to zero. With `baseFee = 0` (the `NoBaseFee = true` configuration), the zero `feeCap` passes the base-fee guard, `EffectiveGasPrice` returns zero, and the `DeductFeeDecorator` deducts nothing — the Cosmos transaction executes with zero fees charged.

### Finding Description

In `ante/evm/fee_checker.go`, `NewDynamicFeeChecker` computes the per-gas fee cap using integer (truncating) division:

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))   // line 83
```

`sdkmath.Int.Quo` is floor division. When `fee < gas` (e.g., `fee = 100_000`, `gas = 200_000`), `feeCap = 0`.

The subsequent base-fee guard:

```go
if feeCap.LT(baseFeeInt) {   // line 86
    return nil, 0, errorsmod.Wrapf(...)
}
```

evaluates `0 < 0 → false` when `baseFee = 0`, so no error is returned.

`EffectiveGasPrice` is then called with `feeCap = 0` and `baseFee = 0`:

```go
effectivePrice := sdkmath.NewIntFromBigInt(
    types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
// = min(maxPriorityPrice + 0, 0) = 0
```

The returned `effectiveFee` is:

```go
effectiveFee := sdk.Coins{{Denom: denom, Amount: effectivePrice.Mul(gas)}}
// = {denom: 0}
```

The SDK's `DeductFeeDecorator` (wired in `evmd/ante/handler_options.go` line 201) receives this zero-amount coin slice. Because `effectiveFee.IsZero()` is `true`, the deduction branch is skipped entirely — the user's account is not charged.

The `MinGasPriceDecorator` that runs earlier uses `Ceil()` for its threshold:

```go
fee := gp.Amount.Mul(gasLimit).Ceil().RoundInt()   // ante/cosmos/min_gas_price.go line 77
```

A user can satisfy this check (e.g., pay `ceil(0.5 × 200_000) = 100_000` coins) while still having `feeCap = 100_000 / 200_000 = 0` in the dynamic checker. The two decorators are inconsistent: one uses ceiling arithmetic, the other uses floor division.

### Impact Explanation

Any Cosmos SDK transaction routed through `newCosmosAnteHandler` with `DynamicFeeChecker = true` can be executed with zero fees deducted whenever:

1. The London hardfork is active (so `types.GetBaseFee` returns `0` rather than `nil`), and
2. `NoBaseFee = true` (base fee forced to 0), and
3. The submitted fee amount is less than the gas limit.

The fee collector receives nothing. The user retains the fee coins they declared in the transaction. This is a direct mis-accounting of user funds/fees in the ante handler path.

### Likelihood Explanation

`NoBaseFee = true` is a documented, governance-settable parameter used on chains that need zero-price calls. Gas limits for non-trivial Cosmos messages (e.g., IBC, staking, governance) routinely exceed the fee amount in the smallest denomination, making `fee < gas` the common case rather than the exception. An unprivileged user needs only to submit a standard Cosmos tx with any fee amount below the gas limit.

### Recommendation

Replace the truncating integer division with a ceiling division for `feeCap`, mirroring the `MinGasPriceDecorator`:

```go
// Before (truncates, can produce 0):
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))

// After (ceiling, consistent with MinGasPriceDecorator):
gasInt := sdkmath.NewIntFromUint64(gas)
feeCap := fee.Add(gasInt).Sub(sdkmath.OneInt()).Quo(gasInt)
```

Alternatively, add an explicit guard after the division:

```go
if feeCap.IsZero() && !fee.IsZero() {
    return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee,
        "fee too small relative to gas limit: feeCap rounds to zero")
}
```

### Proof of Concept

**Setup:** London hardfork active, `NoBaseFee = true`, `MinGasPrice = 0.5` (Dec), `DynamicFeeChecker = true`.

**Attacker submits a Cosmos tx with:**
- `gas = 200_000`
- `fee = 100_000 aevmos` (satisfies `MinGasPriceDecorator`: `ceil(0.5 × 200_000) = 100_000`)

**Execution trace in `NewDynamicFeeChecker`:**

```
baseFee  = GetBaseFee(...)  = 0          (NoBaseFee=true, London active)
fee      = 100_000
gas      = 200_000

feeCap   = 100_000 / 200_000 = 0         ← integer truncation (line 83)

feeCap.LT(baseFeeInt) → 0 < 0 → false   ← guard passes (line 86)

effectivePrice = EffectiveGasPrice(0, 0, MaxInt64)
               = min(MaxInt64 + 0, 0) = 0

effectiveFee   = {aevmos: 0 × 200_000} = {aevmos: 0}
```

**`DeductFeeDecorator`:** `effectiveFee.IsZero() == true` → no deduction.

**Result:** The attacker's Cosmos transaction is included in a block. Zero `aevmos` is deducted. The attacker retains their 100_000 `aevmos` fee coins while the transaction executes fully. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

// EffectiveGasPrice compute the effective gas price based on eip-1159 rules

func EffectiveGasPrice(baseFee *big.Int, feeCap *big.Int, tipCap *big.Int) *big.Int {
	return ethermint.BigMin(new(big.Int).Add(tipCap, baseFee), feeCap)
}
