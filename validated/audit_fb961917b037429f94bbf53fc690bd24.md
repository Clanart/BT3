### Title
`ProofOfInclusion::valid()` Performs Only Internal Consistency Check, Not Root-Anchored Verification — (`File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate only verifies that each layer's `combined_hash` is internally self-consistent with the previous layer. It never checks the final computed root against any external, trusted tree root. The terminal guard `existing_hash == self.root_hash()` is a tautology — it is always `true` when the loop completes — so the function provides a false sense of complete proof validation. An attacker who can supply a `ProofOfInclusion` object (via the Streamable/Python binding surface) can forge an internally consistent proof for any arbitrary `node_hash` and have `valid()` return `true`.

### Finding Description

`ProofOfInclusion` is a `Streamable` struct exposed to Python via `py_valid()`:

```rust
// crates/chia-datalayer/src/merkle/proof_of_inclusion.rs
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← just the last field the attacker wrote
    } else {
        self.node_hash
    }
}

pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;   // ← existing_hash = layer.combined_hash
    }

    existing_hash == self.root_hash()      // ← always true: both sides equal last.combined_hash
}
```

After the loop, `existing_hash` equals the last `calculated_hash`, which the loop already asserted equals `layer.combined_hash`. `self.root_hash()` returns that same `last.combined_hash`. The final comparison is therefore a tautology — it can never be `false` when the loop exits normally.

**Analog to the external report:** The external report's `enterTickets()` checks only `winningTickets[currentRound_][0].length == 0` (index 0) to decide whether prizes have been distributed, missing all higher indices. Here, `valid()` checks only the *internal chain* of the proof, missing the one check that actually matters: whether the chain's terminal hash equals the *externally trusted tree root*.

**Attacker construction:** An attacker can craft a `ProofOfInclusion` with:
- `node_hash` = any 32-byte value (claiming any key is "included")
- `layers` = a chain of `ProofOfInclusionLayer` values where each `combined_hash` is computed correctly from the previous hash and a chosen `other_hash`

`valid()` returns `true` for this forged proof. The struct is `Streamable` and fully deserializable from bytes via `from_bytes()` / `parse_rust()`, and `py_valid()` is exposed to Python callers.

### Impact Explanation

Any Python DataLayer consumer that calls `proof.valid()` to decide whether a key is present in a specific tree — without separately asserting `proof.root_hash() == expected_root` — will accept a forged proof of inclusion. This lets an untrusted peer prove that an arbitrary key/value pair exists in a DataLayer store when it does not, satisfying the allowed impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."*

### Likelihood Explanation

`valid()` is the sole validation method on `ProofOfInclusion` and takes no expected-root parameter. Its name implies completeness. Python DataLayer code that receives proofs from peers and calls `proof.valid()` without the additional `root_hash()` cross-check is a natural and likely usage pattern. The struct is fully deserializable from attacker-supplied bytes through the Streamable/Python binding surface.

### Recommendation

`valid()` must accept an expected root hash and compare against it:

```rust
pub fn valid(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root   // ← anchor to the trusted root
}
```

Alternatively, rename the current function to `is_internally_consistent()` and add a separate `valid(expected_root: &Hash) -> bool` that calls it and then checks the root. Update `py_valid()` accordingly to require the expected root as a parameter.

### Proof of Concept

```rust
use chia_datalayer::{Hash, Side, ProofOfInclusionLayer, ProofOfInclusion};
use chia_datalayer::calculate_internal_hash;

// Attacker wants to forge proof that node_hash X is in the tree
let fake_node_hash: Hash = [0xAA; 32];
let other_hash:     Hash = [0xBB; 32];

// Compute a combined_hash that is internally consistent
let combined = calculate_internal_hash(&fake_node_hash, Side::Left, &other_hash);

let forged = ProofOfInclusion {
    node_hash: fake_node_hash,
    layers: vec![ProofOfInclusionLayer {
        other_hash_side: Side::Left,
        other_hash,
        combined_hash: combined,   // attacker controls this
    }],
};

// valid() returns true — no external root is checked
assert!(forged.valid());
// root_hash() returns `combined`, which the attacker chose freely
assert_eq!(forged.root_hash(), combined);
