### Title
Missing Class Hash Existence Validation in `execute_replace_class` Enables Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS Cairo program accepts any arbitrary felt value as a new class hash without verifying it is actually declared in the contract class tree. This is a state-transition bypass: the OS writes an invalid class hash into contract state without validation, permanently freezing any contract that calls `replace_class` with an undeclared hash — and all funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` processes the `replace_class` syscall. It reads the requested `class_hash` directly from the syscall request and writes it into `contract_state_changes` with no check that the hash corresponds to a declared class. The developers themselves flagged this as a missing invariant: [1](#0-0) 

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The `class_hash` field comes directly from the `ReplaceClassRequest` struct in the syscall segment — it is attacker-controlled calldata. The OS writes it unconditionally into the contract's `StateEntry` without consulting `contract_class_changes` or the existing class tree.

For contrast, the `execute_declare_transaction` path enforces that a declared class hash must be a valid Sierra class hash computed from component hashes: [2](#0-1) 

```cairo
let expected_class_hash = finalize_class_hash(
    contract_class_component_hashes=contract_class_component_hashes
);
with_attr error_message("Invalid class hash pre-image.") {
    assert [class_hash_ptr] = expected_class_hash;
}
```

No equivalent check exists in `execute_replace_class`. The class commitment tree update in `compute_class_commitment` only processes entries that were explicitly declared: [3](#0-2) 

A class hash written via `replace_class` that was never declared will never appear in the class tree, meaning the contract's state permanently points to a non-existent class.

---

### Impact Explanation

When a contract calls `replace_class` with an arbitrary undeclared felt (e.g., `0xdeadbeef`, or any value that does not correspond to a valid Sierra class), the OS:

1. Updates `contract_state_changes` with the invalid class hash (no revert, no error).
2. Commits this to the Patricia Merkle Tree via `compute_contract_state_commitment`. [4](#0-3) 

After the block is finalized, the contract's on-chain class hash points to a class that does not exist in the class tree. Every subsequent call to the contract fails at class lookup. Any ERC-20 tokens, ETH, or STRK held by the contract are permanently inaccessible — there is no recovery path because the contract's `withdraw` / `transfer` entry points cannot be executed.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

The `replace_class` syscall is reachable by any unprivileged contract deployer or caller:

- A user deploys a contract (e.g., a multisig or vault) that exposes an upgrade function.
- The upgrade function calls `replace_class` with a caller-supplied class hash.
- If access control on the upgrade function is insufficient, an attacker supplies an undeclared hash.
- The OS accepts it without validation.

Even without an access-control bug in the target contract, a user can deploy their own contract, call `replace_class` with an invalid hash, and permanently freeze any funds they or others have deposited into it. The entry path is a standard `invoke` transaction — no privileged role required.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify it exists in the declared class set. Concretely, perform a `dict_read` on `contract_class_changes` for the requested `class_hash` and assert the returned compiled class hash is non-zero. If the class is not declared, write a failure response and return without modifying state — consistent with how other syscalls handle invalid inputs.

---

### Proof of Concept

**Vulnerability class (analog to external report):** The external report's root cause is that the archiver stores crafted receipt data without validating the `globalModification` flag, allowing state to be overwritten. The analog here is that the OS stores a crafted class hash from `replace_class` without validating it is declared, allowing contract state to be permanently corrupted.

**Steps:**

1. Deploy a contract `Vault` holding STRK tokens, with a public `upgrade(new_class_hash: felt)` function that calls `replace_class(new_class_hash)`.
2. Submit an `invoke` transaction calling `Vault.upgrade(0xdeadbeef)` where `0xdeadbeef` is never declared.
3. The OS executes `execute_replace_class`:
   - Reads `class_hash = 0xdeadbeef` from the syscall request.
   - Skips the missing existence check (line 898 TODO).
   - Writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes`.
4. `compute_contract_state_commitment` commits this to the Patricia tree.
5. All subsequent calls to `Vault` fail — class `0xdeadbeef` does not exist.
6. STRK tokens in `Vault` are permanently frozen. [5](#0-4)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
