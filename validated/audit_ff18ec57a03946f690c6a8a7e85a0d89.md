### Title
Unregistered Token Accepted in `initTransfer()` Causes Permanent Irrecoverable Fund Lock - (File: `evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.initTransfer()` on EVM (and the analogous function on StarkNet) accepts any arbitrary ERC20 token address without verifying the token is registered in the bridge. When a user calls `initTransfer()` with an unregistered token, the tokens are immediately locked in the bridge contract, but the NEAR-side finalization will always revert because the token has no registered decimals mapping. There is no cancel or rescue path on the EVM side, making the lock permanent and irrecoverable.

---

### Finding Description

In `OmniBridge.initTransfer()`, the function handles three cases for a non-zero `tokenAddress`:

1. `customMinters[tokenAddress] != address(0)` → burn via custom minter
2. `isBridgeToken[tokenAddress]` → burn bridge token
3. **else** → `safeTransferFrom(msg.sender, address(this), amount)` — lock in bridge [1](#0-0) 

There is **no check** that `ethToNearToken[tokenAddress]` is non-empty, that `isBridgeToken[tokenAddress]` is true, or that the token is otherwise registered before accepting it. Any arbitrary ERC20 token falls through to the `else` branch and is locked in the contract. [2](#0-1) 

The `isBridgeToken` and `ethToNearToken` mappings are only populated by `deployToken()` (MPC-signed) or `addCustomToken()` (admin-only): [3](#0-2) 

On the NEAR side, `fin_transfer_callback` attempts to look up the token's decimal configuration. For any token not registered via `deploy_token` or `bind_token`, `token_decimals.get()` returns `None` and the call panics with `TokenDecimalsNotFound`: [4](#0-3) 

Because the NEAR side will always reject finalization of an unregistered token, and because the EVM `OmniBridge` contract exposes no cancel, refund, or emergency-rescue function for locked ERC20 tokens, the user's funds are permanently locked.

The same pattern exists on StarkNet: `init_transfer()` has no registration check and falls through to `transfer_from` for any non-bridge token: [5](#0-4) 

On NEAR, `ft_on_transfer` → `init_transfer` → `init_transfer_internal` also accepts any token (the predecessor account ID is used as the token ID with no registration check), returns `U128(0)` (keeping the tokens), and the subsequent `sign_transfer` call panics with `FailedToGetTokenAddress` for unregistered tokens: [6](#0-5) [7](#0-6) 

---

### Impact Explanation

**Critical — Permanent irrecoverable lock of user funds.**

A user who calls `initTransfer()` (EVM or StarkNet) with any ERC20 token that is not registered in the bridge will have their tokens permanently locked in the bridge contract. The NEAR-side finalization always fails for unregistered tokens, and there is no on-chain cancel or refund path on the EVM side. The `currentOriginNonce` is incremented and the `InitTransfer` event is emitted, but the transfer can never be completed or reversed.

---

### Likelihood Explanation

**Medium.** Any unprivileged user can call `initTransfer()` directly with an arbitrary token address. This can occur through:
- Accidental use of a token that has not yet been registered (e.g., a newly listed token whose `deployToken` has not been called yet)
- A user bridging a token that was removed from the bridge (via `removeCustomToken`) after they initiated the transfer
- A malicious actor tricking a user into calling `initTransfer()` with an unregistered token address

No special privileges are required; the function is fully public and unpaused by default.

---

### Recommendation

Add a registration check at the start of `initTransfer()` on EVM and StarkNet to reject unregistered tokens before any token transfer occurs:

```solidity
// EVM
require(
    tokenAddress == address(0) ||
    isBridgeToken[tokenAddress] ||
    customMinters[tokenAddress] != address(0) ||
    bytes(ethToNearToken[tokenAddress]).length > 0,
    "OmniBridge: token not registered"
);
```

```cairo
// StarkNet
assert(
    self.is_bridge_token(token_address) ||
    self.starknet_to_near_token.read(token_address).len() > 0,
    'ERR_TOKEN_NOT_REGISTERED'
);
```

Additionally, consider adding an emergency token rescue function (admin-only) to recover tokens that were locked before this fix was applied.

On NEAR, add a check in `init_transfer_internal` (or `init_transfer`) that the token is registered (present in `token_id_to_address` for the destination chain) before returning `U128(0)`.

---

    function initTransfer(
        address tokenAddress,
        uint128 amount,
        uint128 fee,
        uint128 nativeFee,
        string calldata recipient,
        string calldata message
    ) external payable whenNotPaused(PAUSED_INIT_TRANSFER) {
        currentOriginNonce += 1;
        if (fee >= amount) {
            revert InvalidFee();
        }

        uint256 extensionValue;
        if (tokenAddress == address(0)) {
            if (fee != 0) {
                revert InvalidFee();
            }
            extensionValue = msg.value - amount - nativeFee;
        } else {
            extensionValue = msg.value - nativeFee;
            if (customMinters[tokenAddress] != address(0)) {
                IERC20(tokenAddress).safeTransferFrom(
                    msg.sender,
                    customMinters[tokenAddress],
                    amount
                );
                ICustomMinter(customMinters[tokenAddress]).burn(
                    tokenAddress,
                    amount
                );
            } else if (isBridgeToken[tokenAddress]) {
                BridgeToken(tokenAddress).burn(msg.sender, amount);
            } else {
                IERC20(tokenAddress).safeTransferFrom(
                    msg.sender,
                    address(this),
                    amount
                );
            }
        }

        initTransferExtension(
            msg.sender,
            tokenAddress,
            currentOriginNonce,
            amount,
            fee,
            nativeFee,
            recipient,
            message,
            extensionValue
        );

        emit BridgeTypes.InitTransfer(
            msg.sender,
            tokenAddress,
            currentOriginNonce,
            amount,
            fee,
            nativeFee,
            recipient,
            message
        );
    }

        fn init_transfer(
            ref self: ContractState,
            token_address: ContractAddress,
            amount: u128,
            fee: u128,
            native_fee: u128,
            recipient: ByteArray,
            message: ByteArray,
        ) {
            assert(!_is_paused(@self, PAUSE_INIT_TRANSFER), 'ERR_INIT_TRANSFER_PAUSED');

            assert(amount > 0, 'ERR_ZERO_AMOUNT');
            assert(fee < amount, 'ERR_INVALID_FEE');

            let origin_nonce = self.current_origin_nonce.read() + 1;
            self.current_origin_nonce.write(origin_nonce);

            let caller = get_caller_address();

            if self.is_bridge_token(token_address) {
                IBridgeTokenDispatcher { contract_address: token_address }
                    .burn(caller, amount.into());
            } else {
                let success = IERC20Dispatcher { contract_address: token_address }
                    .transfer_from(caller, get_contract_address(), amount.into());
                assert(success, 'ERR_TRANSFER_FROM_FAILED');
            }

            if native_fee > 0 {
                let native_token = self.strk_token_address.read();
                let success = IERC20Dispatcher { contract_address: native_token }
                    .transfer_from(caller, get_contract_address(), native_fee.into());
                assert(success, 'ERR_FEE_TRANSFER_FAILED');
            }

            self
                .emit(
                    Event::InitTransfer(
                        InitTransfer {
                            sender: caller,
                            token_address,
                            origin_nonce,
                            amount,
                            fee,
                            native_fee,
                            recipient,
                            message,
                        },
                    ),
                )
        }
