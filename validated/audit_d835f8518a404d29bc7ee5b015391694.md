### Title
Unsigned `ContractCodeResponse::V1` Accepted Without Sender Identity Validation, Allowing Any Connected Peer to Corrupt a Chunk Validator's Witness Reconstruction Cache — (`chain/client/src/stateless_validation/validate.rs`)

### Summary

`validate_contract_code_response` skips all sender-identity checks when the `SignedContractCodeResponse` protocol feature is not yet active. Any connected peer can send a `ContractCodeResponse::V1` carrying arbitrary contract bytes for a valid `ChunkProductionKey`. The validator stores those bytes unconditionally, corrupting the contract-code cache used during witness reconstruction and preventing the validator from endorsing the chunk.

### Finding Description

`ContractCodeResponse` has two variants:

- `V1` — carries only `next_chunk: ChunkProductionKey` and compressed contract bytes; no `responder` field, no signature.
- `V2` — adds a `responder: AccountId` and a `Signature` over the inner struct. [1](#0-0) 

The receive-side validator is `validate_contract_code_response`:

```rust
pub fn validate_contract_code_response(...) -> Result<ChunkRelevance, Error> {
    let key = response.chunk_production_key();
    require_relevant!(validate_chunk_relevant(epoch_manager, key, store)?);
    let protocol_version = epoch_manager.get_epoch_protocol_version(&key.epoch_id)?;
    if ProtocolFeature::SignedContractCodeResponse.enabled(protocol_version) {
        validate_witness_contract_code_response_signature(epoch_manager, response)?;
    }
    Ok(ChunkRelevance::Relevant)
}
``` [2](#0-1) 

When `SignedContractCodeResponse` is not active, the only check performed is `validate_chunk_relevant` — a bounds check on `(epoch_id, shard_id, height_created)`. No check is made that the sender is a legitimate chunk producer for that shard, and no signature is verified. `ContractCodeResponse::V1::verify_signature` always returns `false` and `responder()` returns `None`, so the feature-gated path is the only place where identity is ever asserted. [3](#0-2) 

After passing this check, the handler stores the attacker-supplied bytes directly:

```rust
fn handle_contract_code_response(&self, response: ContractCodeResponse) -> Result<(), Error> {
    if !validate_contract_code_response(...)?. is_relevant() { return Ok(()); }
    let key = response.chunk_production_key().clone();
    let contracts = response.decompress_contracts()?;
    self.partial_witness_tracker.store_accessed_contract_codes(key, contracts)
}
``` [4](#0-3) 

Those bytes are later merged into the decoded `ChunkStateWitness` base state before `validate_chunk_state_witness` runs: [5](#0-4) 

### Impact Explanation

An attacker who is a connected peer and knows the current `ChunkProductionKey` (publicly derivable from block headers) can:

1. Race the legitimate chunk producer's `ContractCodeResponse` to the target validator.
2. Send a `ContractCodeResponse::V1` with arbitrary (wrong) contract bytes for the same key.
3. The validator stores the wrong bytes, merges them into the witness, and runs `validate_chunk_state_witness`. The computed post-state root will not match the committed root in the chunk header.
4. The validator does not emit a `ChunkEndorsement` for that chunk.

If enough validators are targeted simultaneously, the chunk fails to accumulate the required endorsement threshold, stalling block production for that shard. [6](#0-5) 

### Likelihood Explanation

- The `ChunkProductionKey` (`epoch_id`, `shard_id`, `height_created`) is publicly visible from block headers; no privileged access is needed to construct a valid key.
- `ContractCodeResponse` is a T1 routed message; any peer with a T1 connection can send it.
- The timing window is the interval between the validator sending a `ContractCodeRequest` and receiving the legitimate response — typically a few hundred milliseconds, wide enough for a well-positioned peer.
- The attack is repeatable every chunk (~1 second) and requires no cryptographic capability.

### Recommendation

1. **Immediate**: Reject `ContractCodeResponse::V1` messages when the receiving node's epoch already supports `SignedContractCodeResponse`, or unconditionally require V2 once the feature is universally active.
2. **Short-term**: In `validate_contract_code_response`, add a check that the `responder` field (when present) is a known chunk producer for the shard, and that the signature is valid, regardless of the feature flag — treating unsigned V1 as always-invalid once V2 is the expected format.
3. **Structural**: Track which peer/account the `ContractCodeRequest` was sent to and reject responses that do not originate from that peer, mirroring the origin-binding that `ContractCodeRequest` already enforces on the request side via `requester` + signature. [7](#0-6) 

### Proof of Concept

```
1. Observe a block header to extract (epoch_id, shard_id, height_created) for the next chunk.
2. As a connected T1 peer, construct:
     ContractCodeResponse::V1 {
         next_chunk: ChunkProductionKey { epoch_id, shard_id, height_created },
         compressed_contracts: <compress(vec![wrong_bytes])>,
     }
3. Send this message to a target chunk validator immediately after the chunk is produced
   (before the legitimate chunk producer's response arrives).
4. The validator calls validate_contract_code_response → passes (only bounds-checks the key).
5. store_accessed_contract_codes stores wrong_bytes under the key.
6. When witness parts complete, wrong_bytes are merged into the witness base_state.
7. validate_chunk_state_witness computes a mismatched post_state_root → returns Err.
8. No ChunkEndorsement is emitted by this validator for the chunk.
```

pub enum ContractCodeResponse {
    V1(ContractCodeResponseV1) = 0,
    V2(ContractCodeResponseV2) = 1,
}

impl ContractCodeResponse {
    pub fn encode(
        next_chunk: ChunkProductionKey,
        contracts: &Vec<CodeBytes>,
        signer: &ValidatorSigner,
        protocol_version: ProtocolVersion,
    ) -> std::io::Result<Self> {
        if ProtocolFeature::SignedContractCodeResponse.enabled(protocol_version) {
            ContractCodeResponseV2::encode(next_chunk, contracts, signer).map(Self::V2)
        } else {
            ContractCodeResponseV1::encode(next_chunk, contracts).map(Self::V1)
        }
    }

    pub fn chunk_production_key(&self) -> &ChunkProductionKey {
        match self {
            Self::V1(v1) => &v1.next_chunk,
            Self::V2(v2) => &v2.inner.next_chunk,
        }
    }

    /// Account that produced this response. Available only for signed variants.
    pub fn responder(&self) -> Option<&AccountId> {
        match self {
            Self::V1(_) => None,
            Self::V2(v2) => Some(&v2.inner.responder),
        }
    }

    pub fn decompress_contracts(&self) -> std::io::Result<Vec<CodeBytes>> {
        let compressed_contracts = match self {
            Self::V1(v1) => &v1.compressed_contracts,
            Self::V2(v2) => &v2.inner.compressed_contracts,
        };
        compressed_contracts.decode().map(|(data, _size)| data)
    }

    /// Verifies the signature for signed variants. Returns `false` for
    /// unsigned variants since there is nothing to verify.
    pub fn verify_signature(&self, public_key: &PublicKey) -> bool {
        match self {
            Self::V1(_) => false,
            Self::V2(v2) => v2.verify_signature(public_key),
        }
    }
}

fn validate_witness_contract_code_response_signature(
    epoch_manager: &dyn EpochManagerAdapter,
    response: &ContractCodeResponse,
) -> Result<(), Error> {
    let Some(responder) = response.responder() else {
        return Err(Error::Other(
            "Unsigned contract code response in epoch where signature is required".to_owned(),
        ));
    };
    let key = response.chunk_production_key();
    let chunk_producers =
        epoch_manager.get_epoch_chunk_producers_for_shard(&key.epoch_id, key.shard_id)?;
    if !chunk_producers.contains(responder) {
        return Err(Error::Other(format!(
            "Contract code response responder {responder} is not a chunk producer for shard {} in epoch {:?}",
            key.shard_id, key.epoch_id,
        )));
    }
    let validator = epoch_manager.get_validator_by_account_id(&key.epoch_id, responder)?;
    if !response.verify_signature(validator.public_key()) {
        return Err(Error::Other("Invalid witness contract code response signature".to_owned()));
    }
    Ok(())
