### Title
`ExecutionStateReader::get_compiled_class_hash` Returns Stale Compiled Class Hash After Migration - (`File: crates/apollo_rpc_execution/src/state_reader.rs`)

### Summary

`ExecutionStateReader::get_compiled_class_hash` resolves a class's compiled class hash by looking up the **original declaration block** and reading from that block's state diff. When a class's compiled class hash is later updated via `migrated_compiled_classes` (a V1→V2 hash migration), the declaration-block pointer is never updated, so the function permanently returns the pre-migration hash. The correct, time-aware lookup (`get_compiled_class_hash_at`) already exists in the storage layer and is used by `ApolloReader`, but `ExecutionStateReader` bypasses it.

### Finding Description

**Storage invariant for compiled class hash migration**

`append_state_diff` maintains two independent structures for compiled class hashes:

1. `declared_classes_block_table`: maps `class_hash → first_declaration_block_number`. Written once, on first declaration, and never updated:

```rust
// crates/apollo_storage/src/state/mod.rs:549-553
for (class_hash, _) in &thin_state_diff.class_hash_to_compiled_class_hash {
    let not_declared = declared_classes_block_table.get(&self.txn, class_hash)?.is_none();
    if not_declared {
        declared_classes_block_table.insert(&self.txn, class_hash, &block_number)?;
    }
}
```

2. `compiled_class_hash_table`: maps `(class_hash, block_number) → compiled_class_hash`. Written for every block that touches the class, including migrations:

```rust
// crates/apollo_storage/src/state/mod.rs:801-810
fn write_compiled_class_hashes(...) {
    for (class_hash, compiled_class_hash) in compiled_class_hashes {
        compiled_class_hash_table.insert(txn, &(*class_hash, block_number), compiled_class_hash)?;
    }
}
```

The correct time-aware reader `get_compiled_class_hash_at` uses a cursor on `compiled_class_hash_table` to find the latest entry ≤ `state_number`:

```rust
// crates/apollo_storage/src/state/mod.rs:1066-1088
fn get_compiled_class_hash_at(...) -> StorageResult<Option<CompiledClassHash>> {
    let db_key = (*class_hash, first_irrelevant_block);
    let mut cursor = compiled_class_hash_table.cursor(txn)?;
    cursor.lower_bound(&db_key)?;
    let res = cursor.prev()?;
    // returns the most recent compiled_class_hash ≤ state_number
}
```

`ApolloReader::get_compiled_class_hash` (used during block building) calls this correctly:

```rust
// crates/apollo_state_reader/src/apollo_state.rs:243-253
fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    let state_number = StateNumber(self.latest_block);
    match self.reader()?.get_state_reader()
        .and_then(|sr| sr.get_compiled_class_hash_at(state_number, &class_hash)) {
        Ok(Some(compiled_class_hash)) => Ok(compiled_class_hash),
        Ok(None) => Ok(CompiledClassHash::default()),
        ...
    }
}
```

**The broken path in `ExecutionStateReader`**

`ExecutionStateReader::get_compiled_class_hash` (used for RPC simulation, fee estimation, tracing) takes a different path:

```rust
// crates/apollo_rpc_execution/src/state_reader.rs:163-207
fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    // ... pending check only covers declared_classes, not migrated_compiled_classes ...

    let maybe_block_number = self.storage_reader
        .begin_ro_txn()?
        .get_state_reader()?
        .get_class_definition_block_number(&class_hash)  // ← always returns ORIGINAL declaration block N
        ?;

    let Some(block_number) = maybe_block_number else {
        return Ok(CompiledClassHash::default());
    };

    let state_diff = self.storage_reader
        .begin_ro_txn()?
        .get_state_diff(block_number)  // ← reads state diff at block N, not migration block M
        ?
        .ok_or(...)?;

    let compiled_class_hash = state_diff
        .class_hash_to_compiled_class_hash
        .get(&class_hash)  // ← returns H_old from block N, ignoring migration at block M
        .ok_or(...)?;

    Ok(*compiled_class_hash)
}
```

**Concrete scenario**

- Block N: class `C` declared with `compiled_class_hash = H_old` (Poseidon/V1). `declared_classes_block_table[C] = N`.
- Block M (M > N): `migrated_compiled_classes` updates `C` to `H_new` (Blake/V2). `compiled_class_hash_table[(C, M)] = H_new`. `declared_classes_block_table[C]` remains N.
- RPC call at block M+k: `ExecutionStateReader::get_compiled_class_hash(C)` → `get_class_definition_block_number(C)` = N → `get_state_diff(N).class_hash_to_compiled_class_hash[C]` = H_old → returns **H_old** instead of H_new.

The pending-data path has the same gap: it only iterates `pending_data.declared_classes` and never checks `migrated_compiled_classes` entries in the pending state update.

### Impact Explanation

`get_compiled_class_hash` feeds directly into `should_migrate`:

```rust
// crates/blockifier/src/utils.rs:122-142
pub fn should_migrate(state_reader: &impl StateReader, class_hash: ClassHash)
    -> StateResult<Option<(ClassHash, CompiledClassHashV2ToV1)>>
{
    let state_compiled_class_hash = state_reader.get_compiled_class_hash(class_hash)?;
    // ...
    let compiled_class_hash_v2 = state_reader.get_compiled_class_hash_v2(...)?;
    if state_compiled_class_hash == compiled_class_hash_v2 {
        return Ok(None);  // no migration needed
    }
    Ok(Some((class_hash, (compiled_class_hash_v2, state_compiled_class_hash))))
}
```

When `ExecutionStateReader` returns H_old (V1) instead of H_new (V2), `should_migrate` sees `H_old ≠ H_new` and incorrectly concludes migration is still needed. During RPC simulation/fee estimation/tracing this causes:

- The simulated `CommitmentStateDiff.class_hash_to_compiled_class_hash` to include a spurious migration entry for `C`, producing a wrong state diff commitment.
- Fee estimation to include migration gas costs that will not actually be charged on-chain (or vice versa after migration), returning an authoritative-looking wrong fee.
- `starknet_simulateTransactions` / `starknet_traceTransaction` to return execution results that diverge from what the sequencer actually executes, because the sequencer's `ApolloReader` correctly sees H_new and skips migration.

This matches the allowed impact: **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

### Likelihood Explanation

The `migrated_compiled_classes` feature is active (the `enable_casm_hash_migration` flag in `VersionedConstants` is toggled in production). Any class declared before the migration epoch and executed after it will trigger this divergence. The condition is reachable by any caller of `starknet_estimateFee`, `starknet_simulateTransactions`, or `starknet_traceTransaction` on a node that has processed at least one migration block.

### Recommendation

Replace the declaration-block + state-diff lookup in `ExecutionStateReader::get_compiled_class_hash` with the same `get_compiled_class_hash_at` call used by `ApolloReader`:

```rust
fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    // pending check (also add migrated_compiled_classes scan here)
    if let Some(pending_data) = &self.maybe_pending_data {
        // check declared_classes AND migrated_compiled_classes
        ...
    }

    Ok(self.storage_reader
        .begin_ro_txn()
        .map_err(storage_err_to_state_err)?
        .get_state_reader()
        .map_err(storage_err_to_state_err)?
        .get_compiled_class_hash_at(self.state_number, &class_hash)
        .map_err(storage_err_to_state_err)?
        .unwrap_or_default())
}
```

Also extend the pending-data scan to cover `migrated_compiled_classes` entries so that pending-block simulation is consistent.

### Proof of Concept

1. Declare class `C` at block N with compiled class hash H_old (V1/Poseidon). Confirm `declared_classes_block_table[C] = N`.
2. Append a migration state diff at block M > N: `ThinStateDiff { class_hash_to_compiled_class_hash: {C: H_new}, .. }`. Confirm `compiled_class_hash_table[(C, M)] = H_new` and `declared_classes_block_table[C]` is still N.
3. Construct an `ExecutionStateReader` with `state_number = right_after_block(M)`.
4. Call `exec_state_reader.get_compiled_class_hash(C)`.
5. Observe it returns H_old (from `get_state_diff(N)`), not H_new.
6. Call `ApolloReader::get_compiled_class_hash(C)` at the same state number.
7. Observe it returns H_new (from `get_compiled_class_hash_at`).
8. Pass both readers into `should_migrate(reader, C)`: the `ExecutionStateReader` path incorrectly returns `Some(...)` (migration needed), while the `ApolloReader` path correctly returns `None`.

fn get_compiled_class_hash_at<'env, Mode: TransactionKind>(
    first_irrelevant_block: BlockNumber,
    class_hash: &ClassHash,
    txn: &'env DbTransaction<'env, Mode>,
    compiled_class_hash_table: &'env CompiledClassHashTable<'env>,
) -> StorageResult<Option<CompiledClassHash>> {
    let db_key = (*class_hash, first_irrelevant_block);
    // Find the previous db item.
    let mut cursor = compiled_class_hash_table.cursor(txn)?;
    cursor.lower_bound(&db_key)?;
    let res = cursor.prev()?;
    match res {
        None => Ok(None),
        Some(((got_class_hash, _got_block_number), value)) => {
            if got_class_hash != *class_hash {
                // The previous item belongs to different class hash, which means there is no
                // previous state diff for this item.
                return Ok(None);
            };
            // The previous db item indeed belongs to this address and key.
            Ok(Some(value))
        }
    }
