### Title
Missing Duplicate-Key/Hash Validation in `MerkleBlob::batch_insert()` Bulk Path Corrupts DataLayer Merkle Tree Root - (File: `crates/chia-datalayer/src/merkle/blob.rs`)

### Summary

`MerkleBlob::batch_insert()` contains a fast-path for bulk leaf insertion that bypasses the duplicate-key and duplicate-hash guards enforced by the single-item `insert()` function. When the tree already holds two or more leaves, every item in a batch is written directly to the blob without any uniqueness check. An unprivileged caller who controls the input to `batch_insert()` can silently insert duplicate keys or hashes, corrupting the tree structure, invalidating the Merkle root, and enabling forged or inconsistent inclusion proofs.

### Finding Description

`MerkleBlob::insert()` enforces two invariants before writing a leaf:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 369-374
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
```

`batch_insert()` calls `insert()` only for the first two items when the tree has ≤1 existing leaves. All remaining items are written directly via `insert_entry_to_blob()`, which does not consult the cache for uniqueness:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 578-603
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else {
            return Ok(());
        };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;  // checked
    }
}

for ((key, value), hash) in keys_values_hashes {          // ALL remaining: unchecked
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ... Node::Leaf(LeafNode { hash, key, value, ... }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;  // no duplicate guard
    indexes.push(new_leaf_index);
}
```

When the tree already has ≥2 leaves, the `if self.block_status_cache.leaf_count() <= 1` branch is skipped entirely, so **every** item in the batch bypasses both guards.

`BlockStatusCache::add_leaf()` uses `HashMap::insert()`, which silently overwrites an existing key entry. A duplicate key therefore causes the cache to point to the new leaf index while the old leaf node remains in the blob, orphaned. The blob now contains two leaves with the same key, but the cache tracks only one. The internal-node hashes built over both leaves produce a tree root that is inconsistent with the cache-visible state.

The Python binding `py_batch_insert` exposes this path directly to any caller:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 1503-1518
#[pyo3(name = "batch_insert")]
pub fn py_batch_insert(
    &mut self,
    keys_values: Vec<(KeyId, ValueId)>,
    hashes: Vec<Hash>,
) -> PyResult<()> {
    ...
    self.batch_insert(zip(keys_values, hashes).collect())?;
    Ok(())
}
```

### Impact Explanation

**High — DataLayer Merkle proof/blob/delta logic corrupts tree roots and lets untrusted input prove invalid state.**

1. **Corrupted tree root.** The root hash is computed over all leaf nodes including the orphaned duplicate. Any downstream consumer that trusts the root (e.g., a DataLayer sync peer) will accept a root that does not correspond to the canonical key→value mapping.

2. **Forged or inconsistent inclusion proofs.** `get_proof_of_inclusion()` walks the cache to find the leaf index for a key. After a duplicate insertion the cache points to the new leaf; the old leaf is invisible to the cache but still participates in the hash chain. A proof generated for the new leaf will verify against the corrupted root, while the old leaf's hash silently inflates the root. A verifier comparing the root against an honest tree will detect a mismatch, but a verifier that only checks `proof.valid()` against the stored root will accept the forged state.

3. **`check_integrity()` failure.** The function detects `leaf_count != key_to_index_cache_length` and raises an error, but this check is opt-in and is not called automatically in production paths.

### Likelihood Explanation

The `MerkleBlob.batch_insert()` Python method is the primary bulk-insertion API for the DataLayer. Any code that constructs a batch from external or partially-trusted data (e.g., a sync delta received from a peer) and passes it to `batch_insert()` without pre-deduplication can trigger the corruption. No privileged access is required; the caller only needs to supply a list of `(key, value, hash)` triples where at least one key or hash duplicates an existing entry, and the tree must already contain ≥2 leaves (a trivially satisfied precondition in any real DataLayer store).

### Recommendation

Add the same duplicate-key and duplicate-hash guards to the bulk path inside `batch_insert()`. Before writing each leaf in the fast path, check:

```rust
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(&hash) {
    return Err(Error::HashAlreadyPresent());
}
```

Alternatively, refactor `batch_insert()` to route every item through `insert()` (accepting the performance trade-off), or maintain a local `HashSet` of keys and hashes seen within the current batch to catch intra-batch duplicates as well.

### Proof of Concept

```python
from chia_rs.datalayer import MerkleBlob, KeyId, ValueId
from chia_rs.sized_bytes import bytes32
import hashlib

def make_hash(n: int) -> bytes32:
    return bytes32(hashlib.sha256(n.to_bytes(8, "big")).digest())

blob = MerkleBlob(blob=bytearray())

# Seed the tree with 3 leaves so leaf_count > 1
seed_kvs = [(KeyId(i), ValueId(i)) for i in range(3)]
seed_hashes = [make_hash(i) for i in range(3)]
blob.batch_insert(seed_kvs, seed_hashes)
blob.calculate_lazy_hashes()
root_before = blob.get_root_hash()

# Now batch_insert a duplicate of key=0 (already present)
# The bulk path skips the KeyAlreadyPresent guard
dup_kvs  = [(KeyId(0), ValueId(999))]   # key 0 already exists
dup_hash = [make_hash(999)]
blob.batch_insert(dup_kvs, dup_hash)    # should raise KeyAlreadyPresent, but does not
blob.calculate_lazy_hashes()
root_after = blob.get_root_hash()

# root_after differs from root_before and from any honest tree
# containing {0:999, 1:1, 2:2}; the tree is now in an inconsistent state.
assert root_before != root_after, "root changed due to duplicate insertion"

# get_proof_of_inclusion returns a proof that verifies against the corrupted root
proof = blob.get_proof_of_inclusion(KeyId(0))
assert proof.valid()   # passes, but root is wrong
