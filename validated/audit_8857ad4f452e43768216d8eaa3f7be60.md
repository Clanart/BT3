### Title
Missing Shard-ID Binding in `set_state_header` Allows Cross-Shard State Root Substitution — (`chain/chain/src/state_sync/adapter.rs`)

---

### Summary

`set_state_header` accepts a `ShardStateSyncResponseHeader` for a requested `shard_id=A` that internally contains a chunk belonging to `shard_id=B`. No check enforces `chunk.shard_id() == shard_id`. All five validation steps pass with shard B's data, and the corrupted header is committed to `StateHeaderKey(shard_id=A, sync_hash)`, binding shard B's `prev_state_root` to shard A's sync slot.

---

### Finding Description

`set_state_header` in `chain/chain/src/state_sync/adapter.rs` performs five validation steps before writing to `DBCol::StateHeaders`:

**Step 1–2** — `validate_chunk_proofs(&chunk, ...)` verifies the chunk's internal hash, tx root, and receipts root. It does not inspect `shard_id` against any external parameter. [1](#0-0) 

**Step 3a** — `verify_path` checks that `ChunkHashHeight(chunk.chunk_hash(), chunk.height_included())` is a member of `sync_prev_block_header.chunk_headers_root()`. The `chunk_headers_root` is a Merkle tree over **all** shards' chunks. `verify_path` is purely path-based — it has no position/index check: [2](#0-1) 

Shard B's chunk with shard B's own Merkle proof satisfies `compute_root_from_path(proof_B, hash(ChunkHashHeight_B)) == chunk_headers_root`. The function returns `true` regardless of which shard the chunk belongs to. [3](#0-2) 

**Step 4** — Receipt proof validation uses the caller-supplied `shard_id` (A) in `ReceiptList(shard_id, receipts)`. An attacker who has access to blockchain data (as any state-sync provider does) can supply valid receipt proofs for shard A's incoming receipts, satisfying this check independently of which chunk is embedded. [4](#0-3) 

**Step 5** — `validate_state_root_node` checks the supplied `state_root_node` against `chunk_inner.prev_state_root()`. The attacker supplies shard B's `state_root_node`, which is consistent with shard B's `prev_state_root`. This passes. [5](#0-4) 

**Storage** — The header is then written under `StateHeaderKey(shard_id=A, sync_hash)`, embedding shard B's `prev_state_root` as the committed state root for shard A: [6](#0-5) 

The missing guard is a single check: `chunk.shard_id() != shard_id → return Err(...)`.

---

### Impact Explanation

After the corrupted header is stored, two downstream paths are affected:

1. **`set_state_part`** reads the stored header and extracts `state_root = *chunk.take_header().take_inner().prev_state_root()` — shard B's root. All subsequent state parts are validated against shard B's root. If the attacker also serves shard B's parts, they pass validation and `apply_state_part` writes shard B's trie nodes into shard A's `shard_uid` slot. [7](#0-6) 

2. **`set_state_finalize` / `chain_update.set_state_finalize`** reads the stored header, extracts `chunk_header.prev_state_root()` (shard B's root), and calls `apply_chunk` with `RuntimeStorageConfig::new(chunk_header.prev_state_root(), true)` for `shard_uid` of shard A. This either panics/errors (if shard B's trie nodes are absent for shard A's uid) or installs shard B's state into shard A's trie. [8](#0-7) 

The invariant violated: `StateHeaderKey(shard_id=A, sync_hash).prev_state_root` must equal shard A's actual state root at `sync_hash`. After the attack it equals shard B's state root.

---

### Likelihood Explanation

Any network peer can act as a state-sync data source. The syncing node requests headers from peers and calls `set_state_header` on the response without authenticating the sender's identity. The attacker needs:
- A valid chunk from shard B (obtainable from the public chain)
- Shard B's Merkle proof in the block (computable from the public block)
- Valid receipt proofs for shard A's incoming receipts (computable from the public chain)
- Shard B's `state_root_node` (obtainable from any full node)

No validator key, no privileged access, no genesis edit is required.

---

### Recommendation

Add an explicit shard-id binding check immediately after extracting the chunk, before any other validation:

```rust
// In set_state_header, after: let chunk = shard_state_header.cloned_chunk();
if chunk.shard_id() != shard_id {
    byzantine_assert!(false);
    return Err(Error::Other(
        "set_shard_state failed: chunk shard_id does not match requested shard_id".into(),
    ));
}
``` [9](#0-8) 

Additionally, consider using `verify_path_with_index` (which enforces position) instead of `verify_path` for the `chunk_headers_root` check, passing the shard's index in the layout as the expected position: [10](#0-9)

pub fn validate_chunk_proofs(
    chunk: &ShardChunk,
    epoch_manager: &dyn EpochManagerAdapter,
) -> Result<bool, Error> {
    let correct_chunk_hash = chunk.compute_header_hash();

    // 1. Checking chunk.header.hash
    let header_hash = chunk.header_hash();
    if header_hash != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }

    // 2. Checking that chunk body is valid
    // 2a. Checking chunk hash
    if chunk.chunk_hash() != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }
    let height_created = chunk.height_created();
    let outgoing_receipts_root = chunk.prev_outgoing_receipts_root();
    let (transactions, receipts) = (chunk.to_transactions(), chunk.prev_outgoing_receipts());

    // 2b. Checking that chunk transactions are valid
    let (tx_root, _) = merklize(transactions);
    if &tx_root != chunk.tx_root() {
        byzantine_assert!(false);
        return Ok(false);
    }
    // 2c. Checking that chunk receipts are valid
    if height_created == 0 {
        return Ok(receipts.is_empty() && outgoing_receipts_root == &CryptoHash::default());
    } else {
        let shard_layout = {
            let prev_block_hash = chunk.prev_block_hash();
            epoch_manager.get_shard_layout_from_prev_block(&prev_block_hash)?
        };
        let outgoing_receipts_hashes = Chain::build_receipts_hashes(receipts, &shard_layout)?;
        let (receipts_root, _) = merklize(&outgoing_receipts_hashes);
        if &receipts_root != outgoing_receipts_root {
            byzantine_assert!(false);
            return Ok(false);
        }
    }
    Ok(true)
}
```
