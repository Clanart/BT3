### Title
Missing Class Hash Validation in `execute_replace_class` Enables Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the replacement class hash corresponds to a declared contract class. This allows any contract to replace its own class with an arbitrary undeclared hash, permanently breaking the contract and freezing any funds held within it. The missing check is explicitly acknowledged by a TODO comment in the production code.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function accepts any `class_hash` from the syscall request and directly updates the contract's state entry without checking whether the new class hash is declared in `contract_class_changes`.

The critical section:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges the missing validation. No assertion, `dict_read`, or any other check is performed to confirm that `class_hash` maps to a known compiled class before the state is committed.

When a contract's class hash is set to an undeclared value, subsequent calls to that contract fail inside `execute_entry_point`. Specifically:

1. `dict_read{dict_ptr=contract_class_changes}(key=execution_context.class_hash)` returns `0` (the default for an undeclared key).
2. `find_element` then searches the compiled class facts bundle for a compiled class with hash `0`.
3. If no such compiled class exists, `find_element` raises an assertion failure, making the proof invalid. [2](#0-1) 

The sequencer would then be forced to revert or exclude any transaction that attempts to call the broken contract. The state with the invalid class hash, however, is already committed from the successful `replace_class` call — leaving the contract permanently inaccessible.

---

### Impact Explanation

**Critical: Permanent freezing of funds.**

A malicious contract deployer can:
1. Deploy a contract (e.g., a token vault, escrow, or DeFi protocol).
2. Attract user deposits into the contract.
3. Call `replace_class` with an arbitrary undeclared hash (e.g., `0xdeadbeef`).
4. The OS updates the contract's class hash without any validation.
5. Any subsequent call to the contract fails — the compiled class cannot be found.
6. All user funds in the contract are permanently frozen with no recovery path.

This is a direct analog to the external report: just as the MANAGER role could add a malicious token and oracle to drain funds, a contract deployer can use the unvalidated `replace_class` syscall to permanently freeze all funds in their contract.

---

### Likelihood Explanation

**High.** The attack requires:
- No special privileges, roles, or leaked keys.
- No external dependencies or third-party compromise.
- Only the ability to deploy a contract and call the `replace_class` syscall — available to any unprivileged user.

The attack is straightforward and can be executed by any contract deployer. The missing check is even acknowledged in the production code via a TODO comment, confirming the developers are aware the invariant is not enforced.

---

### Recommendation

Add a validation check in `execute_replace_class` to verify that the new class hash is declared before updating the contract's state entry. Concretely, perform a `dict_read` on `contract_class_changes` with the new class hash and assert the result is non-zero (indicating a declared class). If the class hash is not declared, write a failure response and return without modifying state.

---

### Proof of Concept

1. Attacker deploys a malicious vault contract containing a `rug_pull` function that calls `replace_class(0xdeadbeef)` — an undeclared hash.
2. Users deposit funds into the vault (e.g., 1000 STRK).
3. Attacker calls `rug_pull` on the vault.
4. The OS executes `execute_replace_class` with `class_hash = 0xdeadbeef`, updating the vault's state entry without any validation. [1](#0-0) 
5. The transaction succeeds and the state is committed with the invalid class hash.
6. Any subsequent call to the vault (e.g., `withdraw`) fails: `dict_read` on `contract_class_changes` returns `0` for the undeclared hash, and `find_element` cannot locate a compiled class with hash `0`, causing proof failure. [2](#0-1) 
7. All 1000 STRK in the vault are permanently frozen — no withdrawal, no recovery.
