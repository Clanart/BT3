### Title
Unauthenticated `AddGasPrice`/`Initialize` on `RemoteL1GasPriceServer` Allows Arbitrary L1 Gas Price Injection into Block Headers - (File: `crates/apollo_l1_gas_price/src/l1_gas_price_provider.rs`, `crates/apollo_l1_gas_price/src/communication.rs`)

---

### Summary

The `L1GasPriceProvider` exposes its write endpoints — `Initialize` (ring-buffer wipe) and `AddGasPrice` (price sample injection) — through a `RemoteComponentServer` HTTP/2 listener that performs **zero caller authentication**. Any process that can reach the TCP port can reset the price history and inject arbitrary `base_fee_per_gas` / `blob_fee` values. Those values flow directly into every block proposal's `BlockInfo.gas_prices`, corrupting the L1 gas price embedded in block headers and all downstream fee calculations.

---

### Finding Description

`L1GasPriceProvider` is the authoritative source of L1 gas prices for the sequencer. It is served over the network via `RemoteL1GasPriceServer`:

---

### Impact Explanation

An attacker who can reach the `RemoteL1GasPriceServer` port can:

1. **Call `Initialize`** — wipes the ring buffer, bypassing the sequential block-number guard.
2. **Call `AddGasPrice`** with `price_info.base_fee_per_gas = u128::MAX` and `blob_fee = u128::MAX` — fills the ring buffer with a single attacker-controlled sample.
3. `get_price_info` returns the mean of the ring buffer (now a single attacker sample), clamped to `max_l1_gas_price_wei` / `max_l1_data_gas_price_wei`.
4. Every subsequent block proposal embeds the manipulated gas price in its `BlockInfo`, causing **all transaction fees in those blocks to be computed against the wrong L1 gas price**.

This matches the impact scope: **"Critical. Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact"** and **"High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value"** (fee estimation queries the same provider).

The `apply_fee_transformations` clamp limits the magnitude but does not prevent the attack — the attacker can still force the price to the configured maximum or minimum, both of which are wrong values relative to the true L1 market price.

---

### Likelihood Explanation

- The `RemoteComponentServer` is a plain TCP/HTTP2 listener with no TLS, no token, and no IP allowlist at the application layer.
- In the hybrid deployment, `bind_ip` is `0.0.0.0`, making the port reachable by any pod in the cluster network.
- The `L1GasPriceRequest` enum and `GasPriceData` struct are fully serializable; a valid request requires only knowledge of the wire format (bincode/serde), which is public.
- After calling `Initialize`, the sequential block-number constraint is gone, so the attacker does not even need to observe the current L1 block number.

---

### Recommendation

Restrict write access to `L1GasPriceProvider` to the `L1GasPriceScraper` only. Concretely:

1. **Split the request enum** into a read-only interface (`GetGasPrice`, `GetEthToFriRate`) exposed on the `RemoteComponentServer`, and a write interface (`Initialize`, `AddGasPrice`) accessible only via the `LocalComponentServer` (in-process channel).
2. Alternatively, add **mutual TLS or a shared secret** to the `RemoteComponentServer` so that only the authorized scraper process can call write endpoints.
3. As a defense-in-depth measure, validate that the `timestamp` in injected `GasPriceData` is within a reasonable window of wall-clock time, preventing replay of stale or future-dated samples.

    pub fn initialize(&mut self) -> L1GasPriceProviderResult<()> {
        info!("Initializing L1GasPriceProvider with config: {:?}", self.config);
        self.price_samples_by_block = Some(RingBuffer::new(self.config.storage_limit));
        Ok(())
    }

    pub fn add_price_info(&mut self, new_data: GasPriceData) -> L1GasPriceProviderResult<()> {
        // In case the provider has been restarted while the scraper is still running,
        // a NotInitializedError will be returned to the scraper. We expect the scraper to exit with
        // an error, and that infrastructure will restart it, leading to initialization.
        let Some(samples) = &mut self.price_samples_by_block else {
            return Err(L1GasPriceProviderError::NotInitializedError);
        };
        if let Some(data) = samples.back() {
            if new_data.block_number != data.block_number + 1 {
                return Err(L1GasPriceProviderError::UnexpectedBlockNumberError {
                    expected: data.block_number + 1,
                    found: new_data.block_number,
                });
            }
        }
        trace!("Received price sample for L1 block: {:?}", new_data);
        info_every_n_ms!(1_000, "Received price sample for L1 block: {:?}", new_data);
        samples.push(new_data);
        Ok(())
    }
