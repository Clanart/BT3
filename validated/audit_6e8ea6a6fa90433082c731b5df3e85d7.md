### Title
OS Cairo `execute_replace_class` Skips Declared-Class Validation, Enabling Wrong State Root Commitment — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The OS Cairo implementation of `execute_replace_class` does not verify that the supplied class hash is declared in `contract_class_changes` before writing it into `contract_state_changes`. An explicit `TODO` comment in the source acknowledges this missing check. The Rust blockifier enforces the equivalent check and rejects undeclared class hashes. Because the OS is the authoritative proof system and relies on prover-supplied hints, a malicious prover can supply a hint that causes the OS to accept a `replace_class` with an undeclared class hash, producing a provably-valid but wrong state root.

---

### Finding Description

**OS Cairo — missing check (new syscall path)**

In `execute_replace_class` the class hash is read from the request and written directly into `contract_state_changes` with no lookup into `contract_class_changes` to confirm the class was ever declared:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The same omission exists in the deprecated syscall path: [2](#0-1) 

**Rust blockifier — check is present**

The blockifier's `replace_class` calls `get_compiled_class` which returns `StateError::UndeclaredClassHash` if the class is absent, and additionally rejects Cairo 0 class hashes:

```rust
pub fn replace_class(&mut self, class_hash: ClassHash) -> SyscallResult<()> {
    // Ensure the class is declared (by reading it), and of type V1.
    let compiled_class = self.state.get_compiled_class(class_hash)?;
    if !is_cairo1(&compiled_class) {
        return Err(SyscallExecutionError::ForbiddenClassReplacement { class_hash });
    }
    self.state.set_class_hash_at(self.call.storage_address, class_hash)?;
    Ok(())
}
``` [3](#0-2) 

The blockifier test confirms that calling `replace_class` with an undeclared hash produces an `"is not declared"` error: [4](#0-3) 

**State commitment path**

After all transactions execute, the OS squashes `contract_state_changes` and feeds it into `compute_contract_state_commitment`, which hashes `(class_hash, storage_root, nonce)` per contract and updates the Patricia trie to produce the new global state root: [5](#0-4) 

An undeclared class hash written into `contract_state_changes` flows directly into this commitment, producing a wrong state root that is nonetheless provably valid under the STARK proof.

---

### Impact Explanation

The OS Cairo program is the authoritative proof system. It relies on prover-supplied hints (`%{ GetContractAddressStateEntry %}`) and does not independently verify that the class hash in a `replace_class` request exists in `contract_class_changes`. Because the check is absent:

1. A malicious prover crafts a block in which a contract calls `replace_class(X)` where `X` is not declared.
2. The blockifier reverts the transaction; the honest state diff does not include the class-hash update.
3. The malicious prover supplies a hint to the OS claiming the syscall succeeded.
4. The OS writes `class_hash = X` into `contract_state_changes` without objection.
5. `compute_contract_state_commitment` hashes the corrupted entry and produces a wrong Patricia trie root.
6. The STARK proof over this OS execution is valid — the OS ran correctly according to its own (incomplete) logic.
7. The wrong state root is committed on-chain; the affected contract now points to an undeclared class, making all future calls to it fail or behave incorrectly.

This matches the "Wrong state … class hash … from blockifier/syscall/execution logic" critical impact category.

---

### Likelihood Explanation

Exploiting this requires a malicious or compromised prover that can supply crafted hints to the OS. In the current centralized sequencer architecture the prover is trusted, so the practical likelihood is low. However, the OS is designed to be the trustless verifier; the missing check is an acknowledged gap (the TODO targets 2026) and represents a latent critical invariant violation that becomes exploitable if the prover role is ever decentralized or compromised.

---

### Recommendation

**Short term:** In `execute_replace_class` (both the new and deprecated paths), add a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert the result is non-zero before writing the new `StateEntry`. This mirrors the blockifier's `get_compiled_class` guard and closes the divergence.

**Long term:** Systematically audit all OS Cairo syscall implementations for hint-reliant state mutations that lack independent verification, and track them against their Rust blockifier counterparts to ensure invariant parity.

---

### Proof of Concept

```
1. Deploy contract A (class hash C_A, declared).
2. Submit invoke tx: A calls replace_class(X) where X is not in contract_class_changes.
3. Blockifier executes the block:
   - replace_class(X) → get_compiled_class(X) → UndeclaredClassHash → tx reverted.
   - Honest state diff: A's class hash unchanged.
4. Malicious prover runs SNOS with a manipulated hint for the replace_class syscall:
   - %{ GetContractAddressStateEntry %} returns a StateEntry with class_hash = C_A.
   - OS writes new StateEntry(class_hash=X, ...) into contract_state_changes.
   - No assertion fires (the TODO check is absent).
5. compute_contract_state_commitment hashes (X, storage_root_A, nonce_A) for address A.
6. Patricia trie root R' ≠ honest root R is output.
7. STARK proof over the OS execution with root R' is generated and verified successfully.
8. R' is committed on-chain; contract A now maps to undeclared class X.
   All subsequent calls to A fail with ENTRY_POINT_NOT_FOUND or equivalent.
```
