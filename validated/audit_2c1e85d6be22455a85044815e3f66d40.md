### Title
`ExecutionStateReader::get_compiled_class_hash` Returns Stale Compiled Class Hash for Migrated Classes — (File: `crates/apollo_rpc_execution/src/state_reader.rs`)

---

### Summary

`ExecutionStateReader::get_compiled_class_hash` resolves a class's compiled class hash by first looking up the class's **original declaration block** in `declared_classes_block_table`, then reading the state diff at that block. Because `declared_classes_block_table` is never updated when a class's compiled class hash is **migrated** in a later block, the function always returns the stale pre-migration hash. The correct cursor-based lookup (`get_compiled_class_hash_at`) exists and is used by `ApolloReader`, but is not used in the RPC execution path. This causes RPC fee estimation, simulation, tracing, and pending-block views to compute wrong state diffs and return authoritative-looking wrong values.

---

### Finding Description

**Storage layout for compiled class hashes**

`append_state_diff` writes two separate tables:

1. `declared_classes_block_table`: keyed by `class_hash` → first-declaration `block_number`. Written **only if the class is not already present** (i.e., never updated on migration).
2. `compiled_class_hash_table`: keyed by `(class_hash, block_number)` → `compiled_class_hash`. Written for **every** entry in `class_hash_to_compiled_class_hash`, including migrations. [1](#0-0) 

`ThinStateDiff::class_hash_to_compiled_class_hash` is populated from **both** `declared_classes` and `migrated_compiled_classes`: [2](#0-1) 

So after block N (original declaration, hash H1) and block M > N (migration, hash H2):
- `declared_classes_block_table[class_hash] = N` (unchanged)
- `compiled_class_hash_table[(class_hash, N)] = H1`
- `compiled_class_hash_table[(class_hash, M)] = H2`

**The buggy lookup path in `ExecutionStateReader`**

`get_compiled_class_hash` in `ExecutionStateReader`:
1. Calls `get_class_definition_block_number(&class_hash)` → returns `N` (original declaration block, never updated on migration).
2. Calls `get_state_diff(N)` → returns the state diff at block N.
3. Reads `state_diff.class_hash_to_compiled_class_hash[class_hash]` → returns **H1** (the stale pre-migration hash). [3](#0-2) 

**The correct lookup path (not used here)**

`get_compiled_class_hash_at` uses a cursor on `compiled_class_hash_table` keyed by `(class_hash, block_number)` to find the **latest** compiled class hash strictly before the given state number, correctly returning H2 after migration: [4](#0-3) 

`ApolloReader::get_compiled_class_hash` (used in the batcher/sequencer path) correctly calls `get_compiled_class_hash_at`: [5](#0-4) 

**The divergence**

`ExecutionStateReader` (RPC path) uses the wrong two-step lookup via `declared_classes_block_table` + `get_state_diff`, while `ApolloReader` (batcher path) uses the correct cursor-based `get_compiled_class_hash_at`. The two implementations diverge silently for any class that has been migrated.

---

### Impact Explanation

`ExecutionStateReader` is the state reader for all RPC-driven execution: `starknet_estimateFee`, `starknet_simulateTransactions`, `starknet_traceTransaction`, `starknet_call`, and pending-block views. For any transaction that touches a migrated class:

- `CachedState::get_compiled_class_hash` caches the stale H1 as the initial value.
- If the transaction re-declares the class with H2, the state diff records a spurious `H1 → H2` migration that does not reflect the actual on-chain state, producing a wrong `state_diff_commitment` in the RPC response.
- Fee estimation computes gas costs against the wrong state diff, returning wrong fee values to callers.
- Simulation and trace responses carry the wrong `class_hash_to_compiled_class_hash` entries, which are authoritative-looking wrong values.

The batcher's actual block execution is unaffected (it uses `ApolloReader`), so the on-chain state is correct, but the RPC layer presents a divergent view.

**Impact category:** High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.

---

### Likelihood Explanation

The compiled class hash migration feature (`enable_casm_hash_migration`) is a production feature. Any Cairo 1 class declared before the migration epoch and subsequently executed will have its compiled class hash migrated. Once any such class exists in the chain, every RPC fee estimation or simulation involving that class triggers the stale-hash return. The trigger requires no special privileges: any user submitting a transaction that calls a migrated contract class will observe wrong fee estimates.

---

### Recommendation

Replace the two-step `declared_classes_block_table` + `get_state_diff` lookup in `ExecutionStateReader::get_compiled_class_hash` with the cursor-based `get_compiled_class_hash_at`, consistent with `ApolloReader`:

```rust
fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    if let Some(pending_data) = &self.maybe_pending_data {
        for DeclaredClassHashEntry { class_hash: other_class_hash, compiled_class_hash }
            in &pending_data.declared_classes
        {
            if class_hash == *other_class_hash {
                return Ok(*compiled_class_hash);
            }
        }
    }

    // Use cursor-based lookup to get the latest compiled class hash
    // (handles both original declarations and migrations).
    let compiled_class_hash = self
        .storage_reader
        .begin_ro_txn()
        .map_err(storage_err_to_state_err)?
        .get_state_reader()
        .map_err(storage_err_to_state_err)?
        .get_compiled_class_hash_at(self.state_number, &class_hash)
        .map_err(storage_err_to_state_err)?
        .unwrap_or_default();

    Ok(compiled_class_hash)
}
```

This aligns `ExecutionStateReader` with `ApolloReader` and ensures that migrated compiled class hashes are correctly resolved in all RPC execution contexts.

---

### Proof of Concept

1. Declare class `C` in block N with compiled class hash H1. `declared_classes_block_table[C] = N`, `compiled_class_hash_table[(C, N)] = H1`.
2. In block M > N, a migration block writes `compiled_class_hash_table[(C, M)] = H2` (H2 ≠ H1). `declared_classes_block_table[C]` remains N.
3. Call `starknet_estimateFee` for a transaction that invokes contract `C` at state number S > M.
4. `ExecutionStateReader::get_compiled_class_hash(C)`:
   - `get_class_definition_block_number(C)` → N
   - `get_state_diff(N).class_hash_to_compiled_class_hash[C]` → H1 ✗ (should be H2)
5. `CachedState` caches H1 as the initial compiled class hash for C.
6. The resulting state diff records `C → H1` instead of `C → H2`, producing a wrong `state_diff_commitment` in the fee estimation response.
7. Contrast: `ApolloReader::get_compiled_class_hash(C)` calls `get_compiled_class_hash_at(S, C)` → cursor finds `(C, M) → H2` ✓. [6](#0-5) [7](#0-6) [4](#0-3)

fn get_compiled_class_hash_at<'env, Mode: TransactionKind>(
    first_irrelevant_block: BlockNumber,
    class_hash: &ClassHash,
    txn: &'env DbTransaction<'env, Mode>,
    compiled_class_hash_table: &'env CompiledClassHashTable<'env>,
) -> StorageResult<Option<CompiledClassHash>> {
    let db_key = (*class_hash, first_irrelevant_block);
    // Find the previous db item.
    let mut cursor = compiled_class_hash_table.cursor(txn)?;
    cursor.lower_bound(&db_key)?;
    let res = cursor.prev()?;
    match res {
        None => Ok(None),
        Some(((got_class_hash, _got_block_number), value)) => {
            if got_class_hash != *class_hash {
                // The previous item belongs to different class hash, which means there is no
                // previous state diff for this item.
                return Ok(None);
            };
            // The previous db item indeed belongs to this address and key.
            Ok(Some(value))
        }
    }
