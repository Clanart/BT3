### Title
DataLayer `ProofOfInclusion::valid()` Returns `true` for Any Empty-Layers Proof, Enabling Forged Inclusion Proofs - (File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs)

### Summary

`ProofOfInclusion::valid()` in `chia-datalayer` unconditionally returns `true` when the `layers` field is empty, regardless of what `node_hash` is set to. This is structurally identical to the EigenLayer bug: when no proof path is provided, the validation degenerates to a trivially-true self-referential check. An attacker who can supply a serialized `ProofOfInclusion` to a DataLayer client can forge an inclusion proof for any hash value by constructing a zero-layer proof.

### Finding Description

`ProofOfInclusion` is a `Streamable` struct with two fields: `node_hash` (the claimed leaf hash) and `layers` (the Merkle path from leaf to root). [1](#0-0) 

The `root_hash()` method falls back to `node_hash` when `layers` is empty: [2](#0-1) 

The `valid()` method initializes `existing_hash = self.node_hash`, skips the loop entirely when `layers` is empty, then checks `existing_hash == self.root_hash()`. Since `root_hash()` also returns `self.node_hash` in the empty case, this reduces to `self.node_hash == self.node_hash`, which is **always `true`**: [3](#0-2) 

An attacker can construct:
```rust
ProofOfInclusion {
    node_hash: <any_32_byte_value>,
    layers: vec![],
}
```
and `valid()` returns `true`. `root_hash()` returns `node_hash`, so if the attacker sets `node_hash` to the known root of a target tree, both `proof.valid()` and `proof.root_hash() == known_root` pass — without any actual Merkle path being provided.

The struct is fully serializable/deserializable via `Streamable` and exposed through Python bindings: [4](#0-3) [5](#0-4) 

The `get_proof_of_inclusion` method on `MerkleBlob` legitimately produces empty-layers proofs only for single-element trees (where the leaf is the root). For all multi-element trees, a non-empty `layers` is required. The `valid()` method does not enforce this minimum depth requirement. [6](#0-5) 

### Impact Explanation

Any DataLayer client that receives a `ProofOfInclusion` from an untrusted source and calls `proof.valid()` (and optionally `proof.root_hash() == known_root`) without separately verifying that `proof.node_hash` corresponds to the actual key-value leaf hash can be deceived into accepting a forged inclusion proof. This lets untrusted input prove invalid DataLayer state — matching the allowed impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."**

### Likelihood Explanation

`ProofOfInclusion` is a `Streamable` type with Python bindings, meaning it is designed to be transmitted over the network and deserialized by clients. The `valid()` method is the designated API for proof verification. DataLayer clients that follow the natural API pattern of calling `proof.valid()` and checking `proof.root_hash()` against a known root are vulnerable. The fuzz target and tests only ever call `valid()` on proofs generated by `get_proof_of_inclusion`, so the zero-layer forgery case is never exercised. [7](#0-6) 

### Recommendation

`valid()` should require at least one layer when the tree has more than one element, or — more robustly — the method should accept the expected root hash as a parameter and verify `proof.root_hash() == expected_root` internally, making the check atomic. At minimum, add a guard:

```rust
pub fn valid(&self) -> bool {
    // A zero-layer proof is only valid for a single-element tree (leaf == root).
    // Callers must separately verify node_hash matches the expected leaf.
    // To prevent trivial forgery, require callers to pass the expected root:
    // consider changing the signature to valid(&self, expected_root: &Hash) -> bool
    // and checking self.root_hash() == *expected_root here.
    ...
}
```

Alternatively, document clearly that callers **must** verify `proof.node_hash == expected_leaf_hash` in addition to `proof.valid()` and `proof.root_hash() == known_root`, and add a test that a zero-layer proof with `node_hash = root` is rejected when the tree has more than one element.

### Proof of Concept

```rust
use chia_datalayer::{Hash, KeyId, MerkleBlob, ValueId, InsertLocation};
use chia_datalayer::ProofOfInclusion;

// Build a two-element tree
let mut blob = MerkleBlob::new(vec![]).unwrap();
let hash_a = [0xAAu8; 32];
let hash_b = [0xBBu8; 32];
blob.insert(KeyId(1), ValueId(1), &hash_a, InsertLocation::Auto {}).unwrap();
blob.insert(KeyId(2), ValueId(2), &hash_b, InsertLocation::Auto {}).unwrap();
blob.calculate_lazy_hashes().unwrap();

let root: Hash = blob.get_root_hash().unwrap().unwrap();

// Forge a proof: claim root is a leaf with no path
let forged = ProofOfInclusion {
    node_hash: root,
    layers: vec![],
};

assert!(forged.valid());           // true — no path checked
assert_eq!(forged.root_hash(), root); // matches known root
// A client checking only these two conditions accepts the forged proof.
```

    pub fn valid(&self) -> bool {
        let mut existing_hash = self.node_hash;

        for layer in &self.layers {
            let calculated_hash = crate::calculate_internal_hash(
                &existing_hash,
                layer.other_hash_side,
                &layer.other_hash,
            );

            if calculated_hash != layer.combined_hash {
                return false;
            }

            existing_hash = calculated_hash;
        }

        existing_hash == self.root_hash()
    }

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
