### Title
Unchecked Class Hash in `execute_replace_class` Enables Permanent Contract Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall in the StarkNet OS does not verify that the replacement class hash corresponds to a declared contract class. Any contract can replace its own class with an arbitrary, undeclared class hash. Once replaced, all subsequent calls to that contract will fail because the class does not exist in the class tree, permanently freezing any funds held within it. The code itself acknowledges this gap with a TODO comment.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the requested class hash from the syscall request and immediately writes it into `contract_state_changes` without any validation against the declared class tree (`contract_class_changes`): [1](#0-0) 

The critical lines are:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment at line 898 explicitly acknowledges the missing check. The OS unconditionally accepts any felt value as the new class hash and commits it to the contract state. There is no cross-reference against `contract_class_changes` (the class tree) to confirm the hash was ever declared.

By contrast, the `execute_declare_transaction` path in `transaction_impls.cairo` enforces `prev_value=0` to prevent double-declaration and verifies the class hash pre-image via `finalize_class_hash`: [2](#0-1) 

No equivalent guard exists in `execute_replace_class`.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with an undeclared value, every subsequent call to that contract will fail at class resolution time (the class does not exist in the class tree). The contract becomes permanently inoperable. Any ERC-20 tokens, ETH, or other assets held in the contract's storage are irrecoverably locked. Because the state transition is committed to the Merkle tree and included in the ZK proof, there is no protocol-level mechanism to undo it.

---

### Likelihood Explanation

**Medium.**

The `replace_class` syscall is callable by any contract from within its own execution context — no privileged role is required. A realistic attack path is:

1. An attacker deploys a contract that appears legitimate (e.g., a vault, token, or bridge contract).
2. Users deposit funds into the contract.
3. The attacker invokes a function in the contract that calls `replace_class` with an arbitrary felt value that has never been declared.
4. The OS accepts the syscall, commits the invalid class hash to state, and includes it in the proof.
5. All future calls to the contract revert at class lookup; deposited funds are permanently frozen.

The attacker does not need any privileged sequencer access. The syscall is a standard user-reachable operation. The only prerequisite is that the attacker controls a contract (i.e., is a contract deployer), which is an unprivileged role.

---

### Recommendation

Before committing the new class hash in `execute_replace_class`, verify that the hash exists in `contract_class_changes` (or the underlying class commitment tree). Concretely, perform a `dict_read` on `contract_class_changes` for `class_hash` and assert the returned compiled class hash is non-zero:

```cairo
// Verify the class is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the enforcement already present in the declare transaction path and closes the gap identified by the existing TODO.

---

### Proof of Concept

**Step 1 — Attacker deploys a vault contract** that accepts user deposits and exposes a public `drain` function:

```cairo
@external
func drain() {
    // Calls replace_class with a felt that was never declared.
    replace_class(class_hash=0xdeadbeef);
    return ();
}
```

**Step 2 — Users deposit funds** into the vault (e.g., via `transfer` of an ERC-20 token to the vault's address).

**Step 3 — Attacker calls `drain()`**, which triggers the `replace_class` syscall with `class_hash = 0xdeadbeef`.

**Step 4 — OS execution** reaches `execute_replace_class` in `syscall_impls.cairo`. The function reads `class_hash = 0xdeadbeef` from the request, skips any existence check (per the TODO at line 898), and writes the new `StateEntry` with `class_hash=0xdeadbeef` into `contract_state_changes`.

**Step 5 — Block is proven.** The ZK proof is valid because the OS accepted the syscall. The state root now reflects the vault contract having class hash `0xdeadbeef`.

**Step 6 — All subsequent calls to the vault fail** at class resolution. The deposited funds are permanently frozen with no recovery path. [3](#0-2)
