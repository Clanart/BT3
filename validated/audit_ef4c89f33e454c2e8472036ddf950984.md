### Title
P2P State Diff Sync Validates Only Declared Length, Never Verifies Poseidon Commitment — (`File: crates/apollo_p2p_sync/src/client/state_diff.rs`)

---

### Summary

The P2P sync client assembles a block's `ThinStateDiff` from peer-supplied chunks and accepts it as valid once the total entry count matches the `state_diff_length` declared in the stored block header. The `state_diff_commitment` (a Poseidon hash of the diff's full content) is present in the same header and is never read or compared. Any peer can therefore supply a state diff whose entry count is correct but whose content is entirely fabricated, causing the syncing node to persist wrong storage values, nonces, and class hashes, compute a wrong global state root, and serve authoritative-looking wrong values from every RPC endpoint that reads state.

---

### Finding Description

**Root cause — `parse_data_for_block` in `crates/apollo_p2p_sync/src/client/state_diff.rs`**

The function reads the block header from storage to obtain `state_diff_length` and uses it as the sole termination and acceptance criterion:

```rust
let target_state_diff_len = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .expect("...")
    .state_diff_length          // ← only this field is used
    .ok_or(P2pSyncClientError::OldHeaderInStorage { … })?;

while current_state_diff_len < target_state_diff_len {
    …
    current_state_diff_len += state_diff_chunk.len();
    unite_state_diffs(&mut result, state_diff_chunk)?;
}

if current_state_diff_len != target_state_diff_len {
    return Err(…WrongStateDiffLength…);
}

validate_deprecated_declared_classes_non_conflicting(&result)?;
Ok(Some((result, block_number)))   // ← accepted with no hash check
``` [1](#0-0) 

The same header also carries `state_diff_commitment: Option<StateDiffCommitment>`, a Poseidon hash computed by `calculate_state_diff_hash` over the full diff content (deployed contracts, declared classes, storage diffs, nonces). [2](#0-1) 

`calculate_state_diff_hash` chains every field of the diff into a single Poseidon digest, so two diffs with the same entry count but different values produce different commitments. [3](#0-2) 

The commitment is part of the signed block header (`SignedBlockHeader.state_diff_commitment` in the protobuf wire format) and is included in the block hash computation, making it a cryptographically trusted value. [4](#0-3) 

After `parse_data_for_block` returns, the assembled diff is written directly to storage without any further hash check:

```rust
storage_writer.begin_rw_txn()?.append_state_diff(self.1, self.0)?.commit()?;
``` [5](#0-4) 

**Analog to the MonoX bug**

| MonoX | Sequencer P2P sync |

### Impact Explanation

A malicious peer constructs `N` state diff chunks whose total `len()` equals `target_state_diff_len` but whose content differs from the real diff (e.g., wrong storage values, wrong compiled class hashes, wrong nonces). The syncing node:

1. Accepts the fabricated diff (length check passes, no hash check performed).
2. Writes it to `apollo_storage` via `append_state_diff`, overwriting the correct state entries.
3. Passes it to the Patricia trie committer, which computes a wrong global state root.
4. Serves wrong storage values, nonces, and class hashes from every RPC endpoint (`starknet_getStorageAt`, `starknet_getNonce`, `starknet_getClassHashAt`, fee estimation, simulation, tracing).

This matches **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value**, and potentially **Critical — Wrong state, storage value, or class hash from blockifier/syscall/execution logic for accepted input** if the corrupted state is subsequently used for execution.

---

### Likelihood Explanation

Any peer the syncing node connects to can trigger this. No privileged position is required. The only prerequisite is that the attacker is a reachable P2P peer, which is the normal operating assumption for a syncing node. The missing check is a single absent call to `calculate_state_diff_hash` and compare.

---

### Recommendation

After assembling the full `ThinStateDiff` and before writing it to storage, compute its Poseidon commitment and compare it against the `state_diff_commitment` stored in the block header:

```rust
// After the while loop, before Ok(Some(...))
if let Some(expected_commitment) = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .and_then(|h| h.state_diff_commitment)
{
    let actual_commitment = calculate_state_diff_hash(&result);
    if actual_commitment != expected_commitment {
        return Err(ParseDataError::BadPeer(
            BadPeerError::StateDiffCommitmentMismatch { block_number }
        ));
    }
}
```

This mirrors the existing check in `apollo_committer` (`verify_state_diff_hash`) and closes the gap between the declared length and the actual content. [6](#0-5)

### Proof of Concept

1. A syncing node connects to a malicious peer.
2. The malicious peer serves a valid `SignedBlockHeader` for block N with `state_diff_length = 2` and `state_diff_commitment = H(real_diff)`.
3. The syncing node stores the header and requests the state diff.
4. The malicious peer sends two `StateDiffChunk` entries whose combined `len()` equals 2 but whose content sets an arbitrary storage slot to an attacker-chosen value (e.g., sets the fee-token balance of an address to `u128::MAX`).
5. `parse_data_for_block` exits the while loop with `current_state_diff_len == 2 == target_state_diff_len`. [7](#0-6) 
6. `validate_deprecated_declared_classes_non_conflicting` passes (no deprecated classes involved). [8](#0-7) 
7. The fabricated diff is written to storage. [9](#0-8) 
8. `starknet_getStorageAt` for that slot now returns the attacker-chosen value. The Patricia trie root diverges from the honest chain's root.

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
