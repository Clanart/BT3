### Title
`CommitmentStateDiff`→`ThinStateDiff` conversion silently drops `deprecated_declared_classes`, producing a wrong `state_diff_commitment` and block hash for any block containing Cairo 0 class declarations - (File: `crates/blockifier/src/state/cached_state.rs`)

### Summary

The sequencer's block-finalization path converts `CommitmentStateDiff` to `ThinStateDiff` with `deprecated_declared_classes` hard-coded to `Vec::new()`. Because `calculate_state_diff_hash` chains that field into the Poseidon commitment, any block that includes a Cairo 0 (`DeclareV1`) transaction will produce a `state_diff_commitment` — and therefore a block hash — that is structurally wrong: it commits to a state diff that omits the deprecated class declarations that were actually applied.

### Finding Description

**Root cause — the conversion:**

```rust
// crates/blockifier/src/state/cached_state.rs  line 690-701
impl From<CommitmentStateDiff> for ThinStateDiff {
    fn from(commitment_state_diff: CommitmentStateDiff) -> Self {
        Self {
            deployed_contracts: commitment_state_diff.address_to_class_hash,
            storage_diffs: commitment_state_diff.storage_updates,
            class_hash_to_compiled_class_hash: commitment_state_diff
                .class_hash_to_compiled_class_hash,
            nonces: commitment_state_diff.address_to_nonce,
            // TODO(AlonH): Remove this when the structure of storage diffs changes.
            deprecated_declared_classes: Vec::new(),   // ← always empty
        }
    }
}
``` [1](#0-0) 

`CommitmentStateDiff` itself has no `deprecated_declared_classes` field; it is derived from `StateMaps`, which also does not track Cairo 0 class declarations. [2](#0-1) 

**How the wrong value propagates to the block commitment:**

`finalize_block` calls `block_state.to_state_diff()?.state_maps` and converts the result to `CommitmentStateDiff`. That diff is then passed to `BlockExecutionArtifacts::new`, which calls `ThinStateDiff::from(commitment_state_diff.clone())` and feeds the result directly to `calculate_block_commitments`. [3](#0-2) [4](#0-3) 

`calculate_block_commitments` calls `calculate_state_diff_hash`, which chains `deprecated_declared_classes` into the Poseidon hash, and also calls `state_diff.len()` to build `concatenated_counts`. [5](#0-4) 

`calculate_state_diff_hash` explicitly chains the deprecated-class list: [6](#0-5) 

`ThinStateDiff::len()` counts `deprecated_declared_classes.len()` as part of `state_diff_length`: [7](#0-6) 

**Contrast with the sync path**, which correctly merges `migrated_compiled_classes` and `deprecated_declared_classes` via `ThinStateDiff::from_state_diff`: [8](#0-7) 

The `native_blockifier` storage path also manually inserts migrated hashes and passes the full `ThinStateDiff` (with deprecated classes) to `append_state_diff`: [9](#0-8) 

### Impact Explanation

For every block that contains at least one `DeclareV1` (Cairo 0) transaction:

1. `state_diff_commitment` is computed over a `ThinStateDiff` that omits the deprecated class hashes → the Poseidon hash is wrong.
2. `concatenated_counts` encodes a `state_diff_length` that is too small by the number of deprecated declarations → the packed-length field in the block hash is wrong.
3. Both values feed `PartialBlockHashComponents` → `calculate_block_hash`, so the final block hash is wrong.

Because all sequencer nodes (proposer and validators) share the same code path, consensus converges on the wrong hash. Any downstream consumer that independently recomputes the state-diff commitment — a proof verifier, an L1 contract, a full node running `compare_block_hash` — will observe a mismatch. [10](#0-9) 

This matches the allowed impact: **"Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input"** and **"Wrong compiled class, CASM/native artifact, class hash, or contract code selected for execution"** (the class hash of a deprecated class is omitted from the authoritative commitment).

### Likelihood Explanation

Any unprivileged user who submits a valid `DeclareV1` transaction that is accepted by the mempool and sequenced into a block triggers the condition. The sequencer's blockifier supports `DeclareTransaction` V1; no special privilege is required. The only mitigating factor is whether the gateway currently admits Cairo 0 declarations — if it does, the bug is trivially reachable.

### Recommendation

`CommitmentStateDiff` must be extended with a `deprecated_declared_classes: Vec<ClassHash>` field. The blockifier should populate it whenever a Cairo 0 class declaration is executed (analogous to how `class_hash_to_compiled_class_hash` is populated for Cairo 1 declarations). The `From<CommitmentStateDiff> for ThinStateDiff` conversion should then propagate that field instead of substituting `Vec::new()`. The TODO comment at line 699 already acknowledges this debt.

### Proof of Concept

1. Submit a `DeclareV1` transaction declaring Cairo 0 class `C` with class hash `H`.
2. The sequencer executes it; `StateMaps` records no entry for `H` in `compiled_class_hashes` (Cairo 0 has no compiled class hash) and no entry in any deprecated-class list.
3. `finalize_block` produces `CommitmentStateDiff` with `class_hash_to_compiled_class_hash` not containing `H`.
4. `ThinStateDiff::from(commitment_state_diff)` sets `deprecated_declared_classes = []`.
5. `calculate_state_diff_hash` chains `0` deprecated classes → hash value `X`.
6. An independent verifier builds the correct `ThinStateDiff` (with `deprecated_declared_classes = [H]`) and computes hash `Y ≠ X`.
7. The block header stores `state_diff_commitment = X`; the verifier rejects the block or the proof fails.

    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}

/// Note: Blocks before v0.14.0 may include deprecated (Cairo 0) declared classes which are not
/// represented in [`CommitmentStateDiff`]; those blocks skip hash comparison below.
pub async fn compare_block_hash(
    txs_hashing_data: Vec<TransactionHashingData>,
    actual_state_diff: CommitmentStateDiff,
    block_header: &BlockHeader,
    block_number: BlockNumber,
) -> ReexecutionResult<bool> {
    let starknet_version: StarknetVersion = block_header.starknet_version.clone().try_into()?;

    let min_version: StarknetVersion =
        MIN_VERSION_FOR_BLOCK_HASH_COMPARISON.try_into().expect("Invalid min version constant.");
    if starknet_version < min_version {
        tracing::debug!(
            "Block {block_number}: skipping block hash comparison (version {} < {}).",
            block_header.starknet_version,
            MIN_VERSION_FOR_BLOCK_HASH_COMPARISON
        );
        return Ok(true);
    }

    let (commitments, _measurements) = calculate_block_commitments(
        &txs_hashing_data,
        actual_state_diff.into(),
        block_header.l1_da_mode,
        &starknet_version,
    )
    .await;

    let block_info: BlockInfo = block_header.clone().try_into()?;
    let partial_block_hash_components = PartialBlockHashComponents::new(&block_info, commitments);

    let computed_hash = calculate_block_hash(
        &partial_block_hash_components,
        block_header.new_root,
        block_header.parent_hash,
    )?;

    if computed_hash == block_header.block_hash {
        Ok(true)
    } else {
        tracing::warn!(
            "Block hash mismatch for block {block_number}.\n  expected: {}\n  actual:   {}",
            block_header.block_hash,
            computed_hash,
        );
        Ok(false)
    }
