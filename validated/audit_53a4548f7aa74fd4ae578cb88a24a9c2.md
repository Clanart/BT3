### Title
Fixed Gas Cost for Variable-Payload `send_message_to_l1` Syscall Enables Prover Resource Exhaustion — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

`execute_send_message_to_l1` charges a fixed constant gas cost (`SEND_MESSAGE_TO_L1_GAS_COST = 14470`) regardless of the message payload size, while the OS performs an O(payload\_size) `memcpy` to write the payload to the output segment. This is a direct analog of the Linea `REFUND_OVERHEAD_IN_GAS` bug: a fixed accounting constant fails to cover a variable-cost operation. A malicious contract can send arbitrarily large L1 message payloads while paying a fixed gas cost, causing the prover to perform unbounded work relative to fees collected.

---

### Finding Description

In `syscall_impls.cairo`, `execute_send_message_to_l1` charges `SEND_MESSAGE_TO_L1_GAS_COST` as a flat fee:

```cairo
let success = reduce_syscall_gas_and_write_response_header(
    total_gas_cost=SEND_MESSAGE_TO_L1_GAS_COST,
    request_struct_size=SendMessageToL1Request.SIZE
);
``` [1](#0-0) 

The constant is defined as a single fixed value with no per-element factor:

```cairo
const SEND_MESSAGE_TO_L1_GAS_COST = 14470;
``` [2](#0-1) 

After charging this fixed cost, the OS then copies the **entire variable-length payload** to the output segment:

```cairo
tempvar payload_size = request.payload_end - payload_start;
...
memcpy(
    dst=outputs.messages_to_l1 + MessageToL1Header.SIZE, src=payload_start, len=payload_size
);
``` [3](#0-2) 

The `memcpy` is O(payload\_size) work for the prover, but this cost is entirely absent from the gas charged to the caller.

**The codebase itself demonstrates the correct pattern for variable-cost syscalls.** Both `execute_meta_tx_v0` and `execute_deploy` use a base cost plus a per-element factor:

```cairo
// META TX:
let specific_base_gas_cost = (
    META_TX_V0_GAS_COST + META_TX_V0_CALLDATA_FACTOR_GAS_COST * calldata_size
);
``` [4](#0-3) 

The corresponding constants confirm the pattern:

```cairo
const META_TX_V0_GAS_COST = 167950;
const META_TX_V0_CALLDATA_FACTOR_GAS_COST = 4850;
const DEPLOY_GAS_COST = 147120;
const DEPLOY_CALLDATA_FACTOR_GAS_COST = 4850;
``` [5](#0-4) 

`SEND_MESSAGE_TO_L1_GAS_COST` has **no corresponding payload factor constant**, making it the only variable-payload syscall without proportional gas accounting.

---

### Impact Explanation

**High — Network not being able to confirm new transactions (total network shutdown).**

The prover's work is proportional to the total data written to the output segment. Because `send_message_to_l1` charges a fixed `14470` gas regardless of payload size, an attacker can fill a block with transactions that each send a large payload (up to `SIERRA_ARRAY_LEN_BOUND = 2^32` felts per Sierra array), causing the prover to process orders of magnitude more data than the gas fees cover. If proving cost systematically exceeds fee revenue, the economic incentive to prove blocks collapses, and the network cannot finalize new transactions on L1.

---

### Likelihood Explanation

**High.** The attack requires no privileged access. Any unprivileged contract deployer can deploy a contract that allocates a large felt array and calls `send_message_to_l1` with it as the payload. The `execute_send_message_to_l1` function imposes no explicit bound on `payload_size` beyond what the caller's gas budget allows for array allocation — and array allocation gas is far cheaper than the prover cost of copying the data.

---

### Recommendation

Add a per-element payload factor constant and apply it in `execute_send_message_to_l1`, mirroring the existing pattern:

```cairo
const SEND_MESSAGE_TO_L1_GAS_COST = <base>;
const SEND_MESSAGE_TO_L1_PAYLOAD_FACTOR_GAS_COST = <per_felt_cost>;
```

Then compute the total cost as:

```cairo
let specific_gas_cost = SEND_MESSAGE_TO_L1_GAS_COST
    + SEND_MESSAGE_TO_L1_PAYLOAD_FACTOR_GAS_COST * payload_size;
let success = reduce_syscall_gas_and_write_response_header(
    total_gas_cost=specific_gas_cost,
    request_struct_size=SendMessageToL1Request.SIZE
);
```

The per-element cost should be calibrated to cover the prover's actual cost of writing one felt to the output segment (at minimum `STEP_GAS_COST = 100` per element for the `memcpy` step).

---

### Proof of Concept

1. Deploy a contract with an entry point that:
   - Allocates a felt array of size N (e.g., N = 100,000)
   - Fills it with arbitrary values
   - Calls `send_message_to_l1(to_address=<any>, payload=array)`
2. Submit an invoke transaction calling this entry point.
3. The OS charges `14470` gas for the syscall regardless of N.
4. `execute_send_message_to_l1` performs `memcpy(..., len=100000)`, writing 100,000 felts to the output segment — O(N) prover work paid for at O(1) gas cost.
5. Repeat across many transactions in a block to make the block's proving cost exceed the total fees collected, rendering the block economically unviable to prove. [6](#0-5)

// Sends a message to L1.
func execute_send_message_to_l1{range_check_ptr, syscall_ptr: felt*, outputs: OsCarriedOutputs*}(
    contract_address: felt
) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, SendMessageToL1Request*);
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=SEND_MESSAGE_TO_L1_GAS_COST, request_struct_size=SendMessageToL1Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    tempvar payload_start = request.payload_start;
    tempvar payload_size = request.payload_end - payload_start;

    assert [outputs.messages_to_l1] = MessageToL1Header(
        from_address=contract_address, to_address=request.to_address, payload_size=payload_size
    );
    memcpy(
        dst=outputs.messages_to_l1 + MessageToL1Header.SIZE, src=payload_start, len=payload_size
    );
    let (outputs) = os_carried_outputs_new(
        messages_to_l1=outputs.messages_to_l1 + MessageToL1Header.SIZE + payload_size,
        messages_to_l2=outputs.messages_to_l2,
    );

    return ();
}
```
