### Title
`within_margin` computes tolerance from the **proposed** gas price instead of the **expected** gas price, allowing a proposer to commit L1 gas prices ~11.1% above the validator's computed value instead of the intended 10% — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

The `within_margin` helper used to validate a proposer's L1 gas prices computes its percentage tolerance band relative to `number1` (the **proposed** value) rather than `number2` (the **expected** value). Because the margin grows with the proposed price, a proposer can set a price that is ~11.1 % above the validator's oracle-derived price and still pass validation. The inflated price is then committed into the block's `BlockInfo`, used for every fee calculation in that block, and hashed into the block hash — permanently recording a gas price that exceeds the protocol's intended 10 % tolerance.

---

### Finding Description

`is_block_info_valid` in `validate_proposal.rs` calls:

```rust
within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
``` [1](#0-0) 

The helper is:

```rust
fn within_margin(number1: GasPrice, number2: GasPrice, margin_percent: u128) -> bool {
    if number1.0.abs_diff(number2.0) <= GAS_PRICE_ABS_DIFF_MARGIN {
        return true;
    }
    let margin = (number1.0 * margin_percent) / 100;   // ← margin from proposed, not expected
    number1.0.abs_diff(number2.0) <= margin
}
``` [2](#0-1) 

`margin_percent` is the versioned-constant `l1_gas_price_margin_percent = 10`. [3](#0-2) 

**Algebra of the asymmetry.** Let `E` = expected price, `P` = proposed price, `m = 10`.

The check passes when `|P − E| ≤ P·m/100`.

For `P > E`:
```
P − E ≤ P/10  →  9P/10 ≤ E  →  P ≤ 10E/9 ≈ 1.111·E
```

So the proposer can set a price **11.1 % above** the expected value and still pass, not the intended 10 %. The symmetric case (deflation) is capped at ~9.1 % below, so the band is not centred on `E`.

Once `is_block_info_valid` returns `Ok`, `convert_to_sn_api_block_info` converts the accepted `ProposalInit` directly into the `BlockInfo` fed to the batcher: [4](#0-3) 

The inflated `l1_gas_price_fri` / `l1_data_gas_price_fri` values are stored in `GasPrices::strk_gas_prices` and used for every fee calculation in the block. They are also hashed into the block hash via `gas_prices_hash`: [5](#0-4) 

---

### Impact Explanation

A malicious proposer (any consensus participant whose turn it is to propose) can set `l1_gas_price_fri` and `l1_data_gas_price_fri` to `floor(10·E/9)` — the maximum value that passes `within_margin`. All validators accept the proposal because the check passes. The block is executed with the inflated price, so:

1. Every transaction in the block pays fees computed against a gas price up to ~11.1 % above the oracle value instead of the intended ≤ 10 %.
2. The block hash permanently commits to the inflated price.
3. The extra fee revenue accrues to the sequencer/proposer.

The deviation is hardcoded (it follows from the formula every time the proposer maximises their price) and affects every block — exactly the pattern identified in the seed report where a hardcoded 1 % offset was applied to every user on every call.

---

### Likelihood Explanation

Any validator that becomes the proposer for a round can exploit this with zero additional cost: they simply set their proposed gas price to `floor(10·E/9)` where `E` is the price they observe from the oracle. No special access, no external dependency, and no on-chain cost beyond normal block production.

---

### Recommendation

Compute the margin from the **expected** value (`number2`) so the tolerance band is symmetric around `E`:

```rust
fn within_margin(number1: GasPrice, number2: GasPrice, margin_percent: u128) -> bool {
    if number1.0.abs_diff(number2.0) <= GAS_PRICE_ABS_DIFF_MARGIN {
        return true;
    }
    let margin = (number2.0 * margin_percent) / 100;  // use expected, not proposed
    number1.0.abs_diff(number2.0) <= margin
}
```

With this fix the allowed range is exactly `[E·(1 − m/100), E·(1 + m/100)]` for both inflation and deflation.

---

### Proof of Concept

```
Expected oracle price  E = 1_000_000 FRI
Intended max proposed  = E + 10% = 1_100_000 FRI

Attacker sets proposed P = 1_111_111 FRI  (≈ 11.11% above E)

within_margin check:
  margin = P * 10 / 100 = 111_111
  |P − E| = 111_111  ≤  111_111  → PASS ✓

Block executes with l1_gas_price_fri = 1_111_111 instead of ≤ 1_100_000.
Every fee in the block is inflated by an extra ~1.01% beyond the intended cap.
The block hash commits to 1_111_111.
```

fn within_margin(number1: GasPrice, number2: GasPrice, margin_percent: u128) -> bool {
    // For small numbers (e.g., less than 10 wei, if margin is 10%), even an off-by-one
    // error might be bigger than the margin, even if it is just a rounding error.
    // We make an exception for such mismatch, and don't bother checking percentages
    // if the difference in price is only one wei.
    if number1.0.abs_diff(number2.0) <= GAS_PRICE_ABS_DIFF_MARGIN {
        return true;
    }
    let margin = (number1.0 * margin_percent) / 100;
    number1.0.abs_diff(number2.0) <= margin
}

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let previous_block_info = PreviousBlockInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)

        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {

                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}

func calculate_block_hash{poseidon_ptr: PoseidonBuiltin*}(
    block_info: BlockInfo*,
    header_commitments: BlockHeaderCommitments*,
    gas_prices_hash: felt,
    state_root: felt,
    previous_block_hash: felt,
    starknet_version: felt,
) -> felt {
    static_assert BlockInfo.SIZE == 3;
    static_assert BlockHeaderCommitments.SIZE == 5;

    let hash_state = hash_init();
    with hash_state {
        hash_update_single(BLOCK_HASH_VERSION);
        hash_update_single(block_info.block_number);
        hash_update_single(state_root);
        hash_update_single(block_info.sequencer_address);
        hash_update_single(block_info.block_timestamp);
        hash_update_single(header_commitments.packed_lengths);
        hash_update_single(header_commitments.state_diff_commitment);
        hash_update_single(header_commitments.transaction_commitment);
        hash_update_single(header_commitments.event_commitment);
        hash_update_single(header_commitments.receipt_commitment);
        hash_update_single(gas_prices_hash);
        hash_update_single(starknet_version);
        hash_update_single(0);
        hash_update_single(previous_block_hash);
    }

    let block_hash = hash_finalize(hash_state=hash_state);
    return block_hash;
}
