### Title
Missing Declared Class Hash Validation in `execute_replace_class` Allows Permanent Contract Bricking — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in `syscall_impls.cairo` is missing a guard that validates whether the new class hash supplied by the caller is actually a declared class. The OS unconditionally writes the caller-supplied class hash into the contract state without checking its existence in `contract_class_changes`. This is the direct analog of the Sablier bug: a state-mutating operation that should reject invalid/non-existent entities but instead silently accepts them, producing permanently corrupt state.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` processes the `replace_class` syscall for Cairo 1 contracts:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The TODO comment at line 898 explicitly acknowledges the missing check. The function accepts any arbitrary `class_hash` value from the caller and commits it to `contract_state_changes` without verifying that a compiled class with that hash exists. [1](#0-0) 

The same missing guard exists in the deprecated path: [2](#0-1) 

When a subsequent call is made to the bricked contract, `execute_entry_point` reads the compiled class hash:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // undeclared hash → returns 0
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,           // key=0, not found → OS execution fails
);
``` [3](#0-2) 

`dict_read` on `contract_class_changes` for an undeclared class hash returns the default value `0`. `find_element` then searches for a compiled class with hash `0`, which does not exist, causing the OS to fail to produce a valid proof for any block containing a call to the bricked contract.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

An attacker who controls a contract holding funds (e.g., a multisig, vault, or any ERC-20 holder) can call `replace_class` with an arbitrary undeclared class hash. The OS commits this invalid class hash to the contract state without any validation. All future calls to that contract will fail at the OS level (proof generation fails), and the sequencer's blockifier will also reject them. The contract becomes permanently unexecutable, and any funds it holds are irretrievably frozen. No recovery path exists once the state is committed.

---

### Likelihood Explanation

Any unprivileged contract can invoke the `replace_class` syscall with an arbitrary felt value as the class hash. No special role, key, or operator access is required. The attacker only needs to deploy a contract (or control an existing one) and submit a single transaction. The missing check is explicitly acknowledged in the codebase via a TODO comment, confirming it is a known gap in the OS validation logic.

---

### Recommendation

Add a validation step inside `execute_replace_class` (both in `syscall_impls.cairo` and `deprecated_execute_syscalls.cairo`) that checks whether the supplied `class_hash` has a non-zero entry in `contract_class_changes` before committing the state update. If the class hash is not declared, the syscall should write a failure response instead of updating the state:

```cairo
// After reducing gas, before updating state:
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
if (compiled_class_hash == 0) {
    write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
    return ();
}
```

This mirrors the pattern used in `execute_entry_point` where the compiled class is looked up before execution proceeds, and ensures the OS rejects `replace_class` calls referencing undeclared classes.

---

### Proof of Concept

1. Attacker deploys contract `C` holding funds, with a valid declared class hash `H_valid`.
2. Attacker submits a transaction calling `replace_class(H_undeclared)` on `C`, where `H_undeclared` is any felt value not present in `contract_class_changes`.
3. `execute_replace_class` in the OS accepts the call, writes `StateEntry(class_hash=H_undeclared, ...)` to `contract_state_changes`, and the block is proved successfully (the `replace_class` transaction itself does not trigger the broken path).
4. In any subsequent block, any transaction calling `C` causes `execute_entry_point` to call `dict_read(contract_class_changes, H_undeclared)` → returns `0`, then `find_element(..., key=0)` → element not found → OS proof generation fails.
5. The sequencer cannot include any call to `C` in a provable block. All funds in `C` are permanently frozen.
