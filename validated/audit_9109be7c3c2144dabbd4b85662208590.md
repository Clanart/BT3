### Title
Missing Zero-Value Validation of `class_hash` Parameter in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS program accepts a `class_hash` parameter from the syscall request and directly writes it into the contract state without validating that it is non-zero. An attacker controlling a contract (e.g., a shared token contract or vault) can invoke the `replace_class` syscall with `class_hash = 0`, setting the contract's class hash to the uninitialized sentinel value. Any subsequent attempt to execute an entry point on that contract will cause the OS to panic inside `find_element`, permanently freezing all funds held by the contract.

---

### Finding Description

In `execute_replace_class`, the `class_hash` value is read directly from the syscall request and used without any non-zero check:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges the missing check. The value `0` is the `UNINITIALIZED_CLASS_HASH` sentinel used by `deploy_contract` to assert a contract slot is empty before deployment: [2](#0-1) 

When `execute_entry_point` is subsequently called for a contract whose `class_hash` is 0, it performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // key = 0
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,           // key = hash of class 0 → not found → panic
);
``` [3](#0-2) 

`find_element` asserts the element exists; if no class with hash 0 is declared, the OS Cairo program panics and the block proof cannot be generated.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to 0 via `replace_class(0)`, no entry point of that contract can ever be executed again. Any ERC-20 tokens, ETH, or other assets held by the contract become permanently inaccessible. The state update is committed to the Merkle trie and cannot be reversed without a protocol-level intervention.

---

### Likelihood Explanation

Any contract that exposes a `replace_class` call path reachable by an unprivileged user (e.g., a proxy pattern, an upgradeable vault, or a contract whose upgrade logic has a bug) is vulnerable. The attacker only needs to submit a valid transaction that causes the target contract to invoke the `replace_class` syscall with `class_hash = 0`. No privileged key or operator access is required beyond the ability to trigger the syscall from within the contract.

---

### Recommendation

Add an explicit non-zero assertion on `class_hash` before updating the state in `execute_replace_class`:

```cairo
let class_hash = request.class_hash;
assert_not_zero(class_hash);  // <-- add this check
```

Additionally, the existing TODO should be resolved: verify that `class_hash` corresponds to a previously declared class in `contract_class_changes` before accepting the replacement. [4](#0-3) 

---

### Proof of Concept

1. Deploy a contract `Vault` that holds user funds and exposes an `upgrade(class_hash)` function that calls the `replace_class` syscall.
2. Submit a transaction calling `Vault.upgrade(0)`.
3. The OS executes `execute_replace_class` with `class_hash = 0`; no validation fires; `dict_update` writes `StateEntry(class_hash=0, ...)` for `Vault`'s address.
4. Submit any subsequent transaction calling any entry point on `Vault` (e.g., `withdraw`).
5. The OS calls `execute_entry_point` for `Vault`; `dict_read(key=0)` returns the compiled class hash for class 0; `find_element` cannot find a compiled class fact with that hash and panics.
6. All funds in `Vault` are permanently frozen; the block containing step 4 cannot be proven. [5](#0-4) [6](#0-5)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
