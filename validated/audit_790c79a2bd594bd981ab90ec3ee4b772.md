### Title
Missing Class Hash Existence Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not validate that the new class hash supplied by a caller corresponds to a declared contract class. This is the direct analog of the SocketGateway bug: just as `delegatecall` to a hardcoded-but-undeployed route address silently succeeds, `replace_class` with an undeclared class hash is silently accepted by the OS, corrupting the contract's class pointer and permanently freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` (the new-syscall path) accepts any arbitrary `class_hash` value and writes it directly into the contract's `StateEntry` without checking whether that hash corresponds to a declared class:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
``` [1](#0-0) 

The same omission exists in the deprecated-syscall path: [2](#0-1) 

After the state is updated, any subsequent call to the contract reaches `execute_entry_point`, which performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash          // ← the invalid hash
);
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,                  // ← resolves to 0 or unknown hash
);
``` [3](#0-2) 

Because the class was never declared, `dict_read` returns the default value (0), and `find_element` cannot locate a matching `CompiledClassFact`. The contract becomes permanently uncallable.

The TODO comment at line 898 of `syscall_impls.cairo` is an explicit acknowledgment by the development team that this check is absent: [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with a hash that has no corresponding declared class, every future call to that contract fails at the OS level when `find_element` cannot resolve the compiled class. Any ERC-20 tokens, ETH, or other assets held by the contract become permanently inaccessible. The state corruption is committed to the proven state root and cannot be undone without a protocol-level upgrade.

---

### Likelihood Explanation

The `replace_class` syscall is callable by any contract without privilege restrictions. The OS performs no existence check on the supplied class hash — this is confirmed by the explicit TODO comment. If the sequencer-side blockifier also lacks this guard (which the TODO implies, since the OS is the authoritative validator), any unprivileged user can trigger this by deploying a contract that calls `replace_class` with an arbitrary felt value as the class hash. Even if the blockifier rejects such transactions, a malicious or compromised prover can include the syscall in a crafted execution trace that the OS will accept, since the OS itself performs no validation.

---

### Recommendation

In both `execute_replace_class` implementations, add a check that the supplied `class_hash` is present in `contract_class_changes` (i.e., it was declared in the current or a prior block) before updating the `StateEntry`. This mirrors the validation already performed in `execute_entry_point` when resolving a class hash to its compiled form, and closes the gap acknowledged by the TODO comment.

---

### Proof of Concept

1. Attacker deploys contract `A` that holds user funds and exposes a `drain` entry point.
2. Attacker calls a function on `A` that internally invokes `replace_class(0xdeadbeef...)` where `0xdeadbeef...` is a felt that has never been declared as a class.
3. The OS processes `execute_replace_class`: no existence check is performed; `A`'s `StateEntry.class_hash` is set to `0xdeadbeef...`.
4. The block is proven successfully — the OS accepted the syscall.
5. In any subsequent block, any call to contract `A` reaches `execute_entry_point`. `dict_read` on `contract_class_changes` with key `0xdeadbeef...` returns 0. `find_element` with key 0 finds no `CompiledClassFact`. The call fails unconditionally.
6. All funds held by `A` are permanently frozen.
