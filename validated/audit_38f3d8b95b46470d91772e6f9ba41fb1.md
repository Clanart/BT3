### Title
Stale `feemarketParams` Snapshot in `NewDynamicFeeChecker` Allows Cosmos Transactions to Bypass Current BaseFee Enforcement - (File: `ante/evm/fee_checker.go` / `evmd/ante/handler_options.go`)

### Summary
`newCosmosAnteHandler` captures a one-time value-copy snapshot of `feemarketParams` (including `BaseFee`) at construction time and passes a pointer to that local variable into `NewDynamicFeeChecker`. Because `BaseFee` is recalculated and written to the KV store every block by `CalculateBaseFee`, the closure used to validate every subsequent Cosmos transaction always reads a permanently stale `BaseFee`. Transactions whose `feeCap` falls between the stale and the live `BaseFee` pass the ante-handler check and commit with fees below the protocol-required minimum.

### Finding Description

In `evmd/ante/handler_options.go`, `newCosmosAnteHandler` reads params from the store once at construction time:

```go
evmParams := options.EvmKeeper.GetParams(ctx)
feemarketParams := options.FeeMarketKeeper.GetParams(ctx)
...
txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
```

`feemarketParams` is a **local value variable**. `&feemarketParams` is a pointer to that local variable, which is captured by the closure returned from `NewDynamicFeeChecker`. [1](#0-0) 

Inside the closure, every fee check calls:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
```

which reads `feemarketParams.BaseFee` — the stale snapshot — not the live store value. [2](#0-1) 

`GetBaseFee` simply dereferences the pointer and returns `feemarketParams.GetBaseFee()`, which reads `p.BaseFee.BigInt()` from the captured struct. [3](#0-2) 

Meanwhile, `CalculateBaseFee` writes a new `BaseFee` to the KV store on every `BeginBlock`, but this write never touches the local variable captured in the closure. [4](#0-3) 

The fee-cap comparison at line 86 therefore compares the user-supplied `feeCap` against the stale `baseFeeInt`:

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)   // stale
if feeCap.LT(baseFeeInt) { ... }                  // wrong lower bound
``` [5](#0-4) 

The `effectiveFee` deducted from the sender is also computed with the stale `baseFeeInt`, so the actual coins removed from the account are less than the protocol requires. [6](#0-5) 

The same stale pointer is passed to `MinGasPriceDecorator`, compounding the mis-accounting for `MinGasPrice` enforcement as well. [7](#0-6) 

### Impact Explanation

Every Cosmos (non-EVM) transaction processed through `newCosmosAnteHandler` is validated against a permanently stale `BaseFee`. After the first block in which `BaseFee` rises, any user can submit a Cosmos transaction with a `feeCap` strictly between `stale_baseFee` and `current_baseFee`. The transaction passes the ante-handler, is included in a block, and commits while paying less than the protocol-mandated minimum fee. This is a fee-market ante-handler accounting bug that permits invalid transactions to commit and causes valid user fees to be mis-accounted — matching the High impact category.

### Likelihood Explanation

`BaseFee` changes every block whenever block gas usage deviates from the target. After the very first such deviation, the stale snapshot diverges from the live value. No special privileges are required: any unprivileged user can craft a Cosmos transaction with a `feeCap` in the gap between the stale and live `BaseFee` and submit it via the standard transaction broadcast path.

### Recommendation

Read `feemarketParams` from the live store inside the closure rather than capturing a one-time snapshot at construction time. Replace the captured pointer with a live keeper call:

```go
// Inside the NewDynamicFeeChecker closure:
liveFeemarketParams := feemarketKeeper.GetParams(ctx)
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, &liveFeemarketParams)
```

Alternatively, pass the `FeeMarketKeeper` interface into `NewDynamicFeeChecker` and call `GetParams(ctx)` on each invocation, consistent with how `newEthAnteHandler` already reads live params via `EVMBlockConfig`. [8](#0-7) 

### Proof of Concept

1. At block N, `CalculateBaseFee` sets `BaseFee = 1000` in the KV store. `newCosmosAnteHandler` is called (e.g., at app startup or upgrade), capturing `feemarketParams.BaseFee = 1000`.
2. Blocks N+1 … N+k have high gas usage; `CalculateBaseFee` raises the live `BaseFee` to `1500` in the KV store. The closure's captured `feemarketParams.BaseFee` remains `1000`.
3. An attacker submits a Cosmos transaction with `fee = 1200 * gas` (so `feeCap = 1200`).
4. `NewDynamicFeeChecker` evaluates `feeCap (1200) >= stale_baseFeeInt (1000)` → **passes**.
5. `effectivePrice = min(1000 + MaxInt64, 1200) = 1200`; `effectiveFee = 1200 * gas` is deducted — still below the live required `1500 * gas`.
6. The transaction commits. The protocol collected `300 * gas` less than the current fee market requires, and the block proposer's fee revenue is mis-accounted.

func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
	if !IsLondon(ethCfg, height) {
		return nil
	}
	baseFee := feemarketParams.GetBaseFee()

	if baseFee == nil {
		return new(big.Int)
	}
	return baseFee
