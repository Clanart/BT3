### Title
`CommitmentStateDiff→ThinStateDiff` Conversion Silently Drops `deprecated_declared_classes`, Producing Wrong State-Diff Commitment and Block Hash - (`crates/blockifier/src/state/cached_state.rs`)

### Summary

`CommitmentStateDiff` has no `deprecated_declared_classes` field. The `From<CommitmentStateDiff> for ThinStateDiff` implementation hard-codes that field to `Vec::new()`. Because `BlockExecutionArtifacts::new` feeds this `ThinStateDiff` directly into `calculate_block_commitments` → `calculate_state_diff_hash`, any block that contains a deprecated (Cairo 0 / Declare v0/v1) class declaration will produce a `state_diff_commitment`, `state_diff_length`, `concatenated_counts`, and ultimately a `PartialBlockHash` that are all wrong.

### Finding Description

**Root cause — two divergent code paths for `ThinStateDiff`:**

`StateMaps` tracks deprecated class declarations in its `declared_contracts: HashMap<ClassHash, bool>` field. When the sequencer converts execution output to a commitment, it goes through:

```
StateMaps  →  CommitmentStateDiff  →  ThinStateDiff
```

Step 1 (`From<StateMaps> for CommitmentStateDiff`) silently drops `declared_contracts`:

```rust
// crates/blockifier/src/state/cached_state.rs  lines 723-731
impl From<StateMaps> for CommitmentStateDiff {
    fn from(diff: StateMaps) -> Self {
        Self {
            address_to_class_hash: IndexMap::from_iter(diff.class_hashes),
            storage_updates: StorageDiff::from(StorageView(diff.storage)),
            class_hash_to_compiled_class_hash: IndexMap::from_iter(diff.compiled_class_hashes),
            address_to_nonce: IndexMap::from_iter(diff.nonces),
            // diff.declared_contracts is DROPPED — no field for it in CommitmentStateDiff
        }
    }
}
```

Step 2 (`From<CommitmentStateDiff> for ThinStateDiff`) hard-codes the field to empty:

```rust
// crates/blockifier/src/state/cached_state.rs  lines 756-767
impl From<CommitmentStateDiff> for ThinStateDiff {
    fn from(commitment_state_diff: CommitmentStateDiff) -> Self {
        Self {
            deployed_contracts: commitment_state_diff.address_to_class_hash,
            storage_diffs: commitment_state_diff.storage_updates,
            class_hash_to_compiled_class_hash: commitment_state_diff.class_hash_to_compiled_class_hash,
            nonces: commitment_state_diff.address_to_nonce,
            // TODO(AlonH): Remove this when the structure of storage diffs changes.
            deprecated_declared_classes: Vec::new(),   // ← always empty
        }
    }
}
```

**Propagation into commitment construction:**

`BlockExecutionArtifacts::new` calls this conversion and immediately feeds the result into `calculate_block_commitments`:

```rust
// crates/apollo_batcher/src/block_builder.rs  lines 170-176
let (header_commitments, measurements) = calculate_block_commitments(
    &transactions_data,
    ThinStateDiff::from(commitment_state_diff.clone()),  // deprecated_declared_classes = []
    l1_da_mode,
    &block_info.starknet_version,
).await;
```

`calculate_block_commitments` calls `calculate_state_diff_hash`, which chains `deprecated_declared_classes` into the Poseidon hash:

```rust
// crates/starknet_api/src/block_hash/state_diff_hash.rs  lines 30-41
pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    ...
    hash_chain = chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    ...
}
```

`chain_deprecated_declared_classes` chains `[count, class_hash_0, class_hash_1, …]`. When the list is always empty the hash encodes `[0]` regardless of how many deprecated classes were actually declared.

`calculate_block_commitments` also computes `state_diff_length = state_diff.len()`, and `ThinStateDiff::len()` counts `deprecated_declared_classes.len()`:

```rust
// crates/starknet_api/src/state.rs  lines 111-122
pub fn len(&self) -> usize {
    ...
    result += self.deprecated_declared_classes.len();   // always 0
    ...
}
```

Both the wrong `state_diff_commitment` and the wrong `concatenated_counts` (which embeds `state_diff_length`) are stored in `BlockHeaderCommitments`, which feeds `PartialBlockHashComponents` and ultimately the block hash.

The same wrong `ThinStateDiff` is also stored in the batcher's storage via `thin_state_diff()`:

```rust
// crates/apollo_batcher/src/block_builder.rs  lines 210-213
pub fn thin_state_diff(&self) -> ThinStateDiff {
    ThinStateDiff::from(self.commitment_state_diff.clone())  // deprecated_declared_classes = []
}
```

**Contrast with the correct path (sync / storage):**

`ThinStateDiff::from_state_diff` (used by the sync and storage layers) correctly populates `deprecated_declared_classes`:

```rust
// crates/starknet_api/src/state.rs  lines 80-108
deprecated_declared_classes: diff.deprecated_declared_classes.keys().copied().collect(),
```

This is the exact analog of the ProxyFactory/ProxyRegistry bypass: the sequencer's "factory" path (`CommitmentStateDiff` → `ThinStateDiff`) bypasses the "registry" step that records deprecated class declarations, while the sync/storage "registry" path (`StateDiff` → `ThinStateDiff::from_state_diff`) correctly includes them.

### Impact Explanation

Any block containing at least one accepted Declare v0 or v1 transaction will have:

- A wrong `state_diff_commitment` (Poseidon hash encodes `0` deprecated classes instead of the actual count and hashes).
- A wrong `state_diff_length` embedded in `concatenated_counts`.
- A wrong `PartialBlockHash` / block hash derived from those commitments.
- A wrong `ThinStateDiff` persisted to batcher storage, causing downstream consumers (provers, sync peers, RPC `starknet_getStateUpdate`) to receive an authoritative-looking but incorrect state diff.

This matches **Critical — Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input** and **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value**.

### Likelihood Explanation

Deprecated Declare transactions (v0/v1) are still accepted by the sequencer (evidenced by OS flow tests and storage tests that exercise `deprecated_declared_classes`). Any unprivileged user can submit such a transaction. The bug fires on the very first block that includes one.

### Recommendation

Add `deprecated_declared_classes` to `CommitmentStateDiff` and populate it from `StateMaps.declared_contracts` in `From<StateMaps> for CommitmentStateDiff`. Then propagate it through `From<CommitmentStateDiff> for ThinStateDiff` instead of hard-coding `Vec::new()`. Remove the `TODO(AlonH)` comment once the field is properly threaded through.

### Proof of Concept

1. Submit a Declare v1 transaction declaring a Cairo 0 class. The gateway accepts it; the blockifier records `declared_contracts[class_hash] = true` in `StateMaps`.

2. At block finalization, `BlockExecutionArtifacts::new` calls:
   ```rust
   ThinStateDiff::from(commitment_state_diff.clone())
   // → deprecated_declared_classes: Vec::new()   ← class_hash missing
   ```

3. `calculate_state_diff_hash` computes:
   ```
   Poseidon("STARKNET_STATE_DIFF0", …, 0 /* deprecated count */, …)
   ```
   instead of:
   ```
   Poseidon("STARKNET_STATE_DIFF0", …, 1, class_hash, …)
   ```

4. The wrong `StateDiffCommitment` is stored in `BlockHeaderCommitments.state_diff_commitment` and propagated into `PartialBlockHashComponents`, producing a block hash that does not match what any honest verifier, prover, or sync peer would compute from the actual state diff.
