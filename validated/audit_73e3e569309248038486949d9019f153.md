### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS accepts any arbitrary felt value as a new class hash without verifying that the class has been declared. This is an explicitly acknowledged missing check (marked with a TODO). Because the OS enforces no such invariant, any contract can irreversibly set its own class hash to an undeclared value, permanently bricking itself and freezing all funds it holds.

---

### Finding Description

In `execute_replace_class` (`syscall_impls.cairo`, lines 877–916), after deducting gas, the OS reads `request.class_hash` and immediately writes it into `contract_state_changes` with no cross-reference against `contract_class_changes`:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The `contract_class_changes` dict (keyed by class hash, valued by compiled class hash) is the authoritative record of declared classes. The OS never consults it here. The TODO comment at line 898 is the developers' own acknowledgment that this check is absent. [2](#0-1) 

For contrast, the normal declare path in `execute_declare_transaction` enforces `prev_value=0` (class declared at most once) and requires `__validate_declare__` to pass before writing to `contract_class_changes`: [3](#0-2) 

`execute_replace_class` bypasses all of that and writes any felt directly into the contract state tree.

The `StateEntry` struct stores `class_hash` as a plain felt: [4](#0-3) 

Once committed, `get_contract_state_hash` hashes the new (invalid) class hash into the Patricia tree leaf, making the state change part of the proven global root with no recovery path. [5](#0-4) 

---

### Impact Explanation

Once a contract's class hash is set to an undeclared value:

- Every subsequent call to that contract requires the OS to look up the class in `contract_class_changes`. The class does not exist, so execution fails unconditionally.
- The contract cannot call `replace_class` again to self-repair, because calling it requires the contract to be executable.
- All tokens, NFTs, or other assets held in the contract's storage are permanently inaccessible.

This satisfies **Critical — Permanent freezing of funds**.

---

### Likelihood Explanation

The `replace_class` syscall is a standard, publicly documented StarkNet syscall reachable by any Sierra contract. The attack surface is broad:

1. **Malicious deployer scenario**: An attacker deploys a contract that accepts user deposits and contains a backdoor function calling `replace_class(0xdeadbeef)`. After accumulating funds, the attacker triggers the backdoor. The OS commits the invalid class hash. All deposited funds are frozen.
2. **Buggy upgrade mechanism**: A legitimate protocol's upgrade function passes an unvalidated class hash argument. An attacker supplies an undeclared hash. Same outcome.

No privileged role, leaked key, or network-level attack is required. Any unprivileged contract deployer can reach this code path via the normal `REPLACE_CLASS_SELECTOR` dispatch in `execute_syscalls`: [6](#0-5) 

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that it exists in `contract_class_changes`. Concretely, perform a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert the returned compiled class hash is non-zero (i.e., the class has been declared). This mirrors the invariant already enforced during `execute_declare_transaction` and closes the gap the TODO comment identifies.

---

### Proof of Concept

1. Attacker deploys contract `C` (class `K`, declared normally). `C` holds a public `deposit()` and an owner-only `brick()` that calls `replace_class(0x1)` — an undeclared hash.
2. Users call `deposit()`, locking funds in `C`'s storage.
3. Attacker calls `brick()`. The OS dispatches to `execute_replace_class` with `class_hash = 0x1`.
4. `execute_replace_class` skips the missing declared-class check and writes `StateEntry { class_hash: 0x1, ... }` into `contract_state_changes`.
5. The block is proven. The global state root now commits `C` with class hash `0x1`.
6. Any subsequent call to `C` (e.g., `withdraw()`) causes the OS to look up class `0x1` in `contract_class_changes`, find nothing, and fail.
7. All user funds in `C` are permanently frozen with no recovery mechanism.

func get_contract_state_hash{hash_ptr: HashBuiltin*}(
    class_hash: felt, storage_root: felt, nonce: felt
) -> (hash: felt) {
    const CONTRACT_STATE_HASH_VERSION = 0;
    if (class_hash == UNINITIALIZED_CLASS_HASH) {
        if (storage_root == 0) {
            if (nonce == 0) {
                return (hash=0);
            }
        }
    }

    // Set res = H(H(class_hash, storage_root), nonce).
    let (hash_value) = hash2(class_hash, storage_root);
    let (hash_value) = hash2(hash_value, nonce);

    // Return H(hash_value, CONTRACT_STATE_HASH_VERSION). CONTRACT_STATE_HASH_VERSION must be in the
    // outermost hash to guarantee unique "decoding".
    let (hash) = hash2(hash_value, CONTRACT_STATE_HASH_VERSION);
    return (hash=hash);
```
