### Title
`batch_insert` Bypasses Duplicate-Key and Duplicate-Hash Guards Present in `insert` — (File: `crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::batch_insert` skips the duplicate-key and duplicate-hash validation that `MerkleBlob::insert` enforces. When a batch of three or more entries is supplied, the first N-2 entries are written directly into the blob via `insert_entry_to_blob` with no uniqueness checks, allowing an attacker to silently corrupt the Merkle tree with duplicate keys or hashes. This is the exact structural analog of the Saddle Finance M02 finding: values that are validated on every incremental update are accepted without bounds/uniqueness checks during the "bulk initialization" path.

---

### Finding Description

`MerkleBlob::insert` (the single-entry update path) enforces two invariants before writing anything:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 369-374
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`MerkleBlob::batch_insert` (the bulk path) only routes the **last two** entries through `insert` when the tree has ≤ 1 existing leaf. Every earlier entry in the input vector is written directly via `insert_entry_to_blob`, which performs no uniqueness check:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 578-602
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else { ... };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;  // validated
    }
}

for ((key, value), hash) in keys_values_hashes {          // ← remaining N-2 entries
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ... Node::Leaf(LeafNode { hash, key, value, ... }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?; // ← NO duplicate check
    indexes.push(new_leaf_index);
}
``` [2](#0-1) 

`insert_entry_to_blob` unconditionally calls `block_status_cache.add_leaf`, so the cache ends up with duplicate key or hash entries, and the tree structure is silently corrupted. [3](#0-2) 

`batch_insert` is a public method exposed to Python callers through the DataLayer wheel bindings. [4](#0-3) 

---

### Impact Explanation

A caller who supplies a batch containing duplicate `KeyId` values or duplicate `Hash` values (three or more entries total) will silently produce a `MerkleBlob` whose internal `block_status_cache` maps the same key or hash to multiple tree indexes. Downstream operations that rely on the uniqueness invariant — root hash computation, proof-of-inclusion generation, and proof verification — will operate on a structurally invalid tree. This enables:

- **Forged inclusion proofs**: two leaves with the same hash produce ambiguous proof paths; a verifier may accept a proof for a key/value pair that was never legitimately inserted.
- **Corrupted tree root**: the root hash computed over a tree with duplicate leaves does not correspond to any valid committed state, causing cross-node disagreement on the DataLayer root.

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

---

### Likelihood Explanation

`batch_insert` is a public API reachable from Python. Any DataLayer client that calls `batch_insert` with a list containing a repeated key or hash (e.g., due to a bug in the caller, or a deliberately crafted input) will silently corrupt the tree. No privileged role or key material is required.

---

### Recommendation

Apply the same duplicate-key and duplicate-hash guards at the top of `batch_insert` (or inside the fast-path loop) that `insert` already enforces:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // ... existing insert_entry_to_blob call
}
```

Alternatively, refactor `batch_insert` to route all entries through `insert` (accepting the performance cost), or extract the uniqueness check into a shared helper called by both paths.

### Proof of Concept

```rust
use chia_datalayer::merkle::blob::{MerkleBlob, KeyId, ValueId, Hash, InsertLocation};
use chia_protocol::Bytes32;

let mut blob = MerkleBlob::new(vec![]).unwrap();

let dup_key   = KeyId(1);
let dup_hash  = Hash(Bytes32::new([0xAB; 32]));

// Three entries: first two bypass validation, last two go through insert().
// Entries are pop()-ed from the end, so index 0 and 1 are the "fast path".
let batch = vec![
    ((dup_key,        ValueId(1)), dup_hash),   // index 0 – fast path, no check
    ((KeyId(2),       ValueId(2)), Hash(Bytes32::new([0x01; 32]))),
    ((KeyId(3),       ValueId(3)), Hash(Bytes32::new([0x02; 32]))),
    ((dup_key,        ValueId(4)), dup_hash),   // index 3 – also fast path, DUPLICATE
];

// This succeeds even though dup_key and dup_hash appear twice.
blob.batch_insert(batch).unwrap();

// The tree now contains two leaves with the same KeyId and Hash,
// violating the uniqueness invariant and corrupting the Merkle root.

    fn insert_entry_to_blob(&mut self, index: TreeIndex, block: &Block) -> Result<(), Error> {
        let new_block_bytes = block.to_bytes()?;
        let extend_index = self.extend_index();
        match index.cmp(&extend_index) {
            Ordering::Greater => return Err(Error::BlockIndexOutOfBounds(index)),
            Ordering::Equal => self.blob.extend_from_slice(&new_block_bytes),
            Ordering::Less => {
                self.blob[block_range(index)].copy_from_slice(&new_block_bytes);
            }
        }

        match block.node {
            Node::Leaf(leaf) => self.block_status_cache.add_leaf(index, leaf),
            Node::Internal(..) => self.block_status_cache.add_internal(index),
        }

        Ok(())
    }
