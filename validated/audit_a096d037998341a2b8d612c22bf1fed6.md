### Title
Unregistered Token Accepted in `init_transfer` Causes Permanent Fund Lock â€” (File: `starknet/src/omni_bridge.cairo`)

---

### Summary

The `init_transfer` function in the Starknet OmniBridge contract accepts any caller-supplied `token_address` without verifying it is registered in the bridge's token mapping. This is the direct analog of M-09: just as `SurplusGuildMinter::stake()` lacked a check that the term's credit token matched the SGM's credit token, `init_transfer` lacks a check that `token_address` is a known bridge token. The result is that a user's tokens are irrecoverably locked in the Starknet contract while the NEAR hub rejects the corresponding proof.

---

### Finding Description

In `starknet/src/omni_bridge.cairo`, `init_transfer` performs only three input validations before accepting tokens: [1](#0-0) 

```cairo
assert(!_is_paused(@self, PAUSE_INIT_TRANSFER), 'ERR_INIT_TRANSFER_PAUSED');
assert(amount > 0, 'ERR_ZERO_AMOUNT');
assert(fee < amount, 'ERR_INVALID_FEE');
```

It then branches on `is_bridge_token(token_address)` solely to decide **how** to take custody (burn vs. `transfer_from`), not **whether** the token is valid for bridging: [2](#0-1) 

```cairo
if self.is_bridge_token(token_address) {
    IBridgeTokenDispatcher { contract_address: token_address }
        .burn(caller, amount.into());
} else {
    let success = IERC20Dispatcher { contract_address: token_address }
        .transfer_from(caller, get_contract_address(), amount.into());
    assert(success, 'ERR_TRANSFER_FROM_FAILED');
}
```

`is_bridge_token` only checks whether the token was deployed by the bridge itself: [3](#0-2) 

```cairo
fn is_bridge_token(self: @ContractState, token_address: ContractAddress) -> bool {
    self.starknet_to_near_token.read(token_address).len() > 0
}
```

A token that is **not** a bridge-deployed token but is also **not** registered in the NEAR hub's `token_decimals` / `token_address_to_id` maps passes both branches silently. The `InitTransfer` event is emitted and the nonce is incremented, but the NEAR hub will reject any proof for this transfer because `fin_transfer_callback` panics on an unregistered token: [4](#0-3) 

```rust
let decimals = self
    .token_decimals
    .get(&init_transfer.token)
    .near_expect(BridgeError::TokenDecimalsNotFound);
```

There is no `withdraw` or `recover` function in the Starknet contract. The only egress path for locked tokens is `fin_transfer`, which requires a valid MPC signature. The MPC will never sign for an unregistered token. The funds are therefore permanently locked.

---

### Impact Explanation

**Permanent freezing / irrecoverable lock of user funds** â€” matching the allowed Critical/High impact class.

Any ERC-20 token on Starknet that has not been registered in the NEAR hub can be deposited into the bridge contract. Once deposited, the tokens cannot be recovered: the NEAR hub rejects the proof, the MPC will not sign a release, and the Starknet contract has no administrative recovery path.

---

### Likelihood Explanation

**Medium.** The attack surface is reachable by any unprivileged user who:
1. Holds any ERC-20 token on Starknet (registered or not).
2. Approves the bridge and calls `init_transfer` with that token address.

A user could trigger this accidentally (wrong token address, token not yet registered) or deliberately (griefing another user's funds by front-running a registration). No privileged access is required.

---

### Recommendation

Add an explicit registration check at the top of `init_transfer`, mirroring the fix recommended in M-09:

```cairo
fn init_transfer(
    ref self: ContractState,
    token_address: ContractAddress,
    ...
) {
+   // Ensure the token is registered in the bridge before accepting it
+   let near_token_id = self.starknet_to_near_token.read(token_address);
+   assert(near_token_id.len() > 0 || _is_registered_native(@self, token_address),
+          'ERR_TOKEN_NOT_REGISTERED');
    assert(!_is_paused(@self, PAUSE_INIT_TRANSFER), 'ERR_INIT_TRANSFER_PAUSED');
    ...
}
```

For native (non-bridge-deployed) tokens, maintain a separate allowlist of approved native token addresses that have corresponding NEAR-side registrations, and check membership before accepting the deposit.

---

### Proof of Concept

1. Deploy any arbitrary ERC-20 token `FakeToken` on Starknet (not registered in the NEAR hub).
2. Approve the Starknet OmniBridge to spend `FakeToken`.
3. Call `init_transfer(fake_token_address, 1000e18, 0, 0, "near_recipient.near", "")`.
4. The call succeeds: `transfer_from` moves `FakeToken` into the bridge, `origin_nonce` is incremented, and `InitTransfer` is emitted.
5. A relayer submits the proof to the NEAR hub via `fin_transfer`.
6. `fin_transfer_callback` calls `self.token_decimals.get(&init_transfer.token).near_expect(BridgeError::TokenDecimalsNotFound)` and panics â€” the proof is rejected.
7. `FakeToken` remains locked in the Starknet bridge forever with no recovery path. [5](#0-4) [6](#0-5)
