### Title
Missing Duplicate Key/Hash Validation in `MerkleBlob::batch_insert` Fast Path Corrupts DataLayer Tree Root - (File: crates/chia-datalayer/src/merkle/blob.rs)

### Summary
`MerkleBlob::batch_insert` skips the duplicate-key and duplicate-hash guards that `MerkleBlob::insert` enforces when the tree already contains two or more leaves. An untrusted caller can supply a batch containing repeated `KeyId` or `Hash` values; the fast path writes those duplicate leaf nodes directly into the blob, orphaning old entries and producing an incorrect Merkle root. Any inclusion or exclusion proof derived from that root is thereafter forged.

### Finding Description
`MerkleBlob::insert` explicitly rejects duplicate keys and hashes before writing anything to the blob:

```rust
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`batch_insert` delegates to `insert` only for the first two items when the tree has ≤ 1 existing leaf:

```rust
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else { return Ok(()); };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;
    }
}
``` [2](#0-1) 

All remaining items — and **all** items when the tree already has ≥ 2 leaves — are written directly via `insert_entry_to_blob`, which performs no duplicate check:

```rust
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ..., node: Node::Leaf(LeafNode { hash, key, value, ... }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
``` [3](#0-2) 

The Python binding `py_batch_insert` only validates that the lengths of the key/value and hash slices match; it adds no duplicate check:

```rust
if keys_values.len() != hashes.len() {
    Err(Error::UnmatchedKeysAndValues(...))?;
}
self.batch_insert(zip(keys_values, hashes).collect())?;
``` [4](#0-3) 

When a duplicate `KeyId` is inserted, `block_status_cache.add_leaf` overwrites the old `key → index` mapping, leaving the original leaf node orphaned in the blob. The tree's internal hash chain then incorporates both the orphaned node and the new duplicate, producing a root hash that does not correspond to any valid tree state. Proofs of inclusion or exclusion generated against this root are structurally invalid.

### Impact Explanation
The corrupted root hash is committed on-chain as the DataLayer store root. Any proof of inclusion or exclusion derived from it is forged: a key that appears twice in the blob can be "proven" present at an index that does not match the canonical key-to-index mapping, and a key whose old entry was orphaned can be "proven" absent even though it was previously committed. This matches the allowed High impact: "DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."

### Likelihood Explanation
The `py_batch_insert` binding is the primary bulk-insertion API used during DataLayer sync. A peer DataLayer node — or any Python caller that processes externally-sourced key-value deltas — can supply a batch containing repeated keys without triggering any error. The missing guard is entirely absent from the fast path, not gated behind a flag, so the condition is trivially reachable.

### Recommendation
Add the same duplicate-key and duplicate-hash checks to the fast path of `batch_insert` that `insert` already enforces. Before calling `insert_entry_to_blob` for each item in the loop, verify:

```rust
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(&hash) {
    return Err(Error::HashAlreadyPresent());
}
```

Alternatively, pre-validate the entire input vector for uniqueness before entering the fast path, consistent with how `insert` guards individual insertions. [5](#0-4) 

### Proof of Concept
1. Construct a `MerkleBlob` with two existing leaves (so `leaf_count > 1`).
2. Call `batch_insert` (or `py_batch_insert`) with a vector containing two entries sharing the same `KeyId` (e.g., `KeyId(42)`) but different `Hash` values.
3. Observe that no error is returned.
4. Call `get_root_hash()` — the returned hash incorporates both the orphaned original leaf and the duplicate, producing a root that no valid proof can satisfy.
5. Call `check_integrity()` — it will report a mismatch between `leaf_count` and `key_to_index_cache_length` or a `IntegrityLeafHashToIndexCacheLength` error, confirming tree corruption. [6](#0-5)

    fn check_just_integrity(&self) -> Result<(), Error> {
        let mut leaf_count: usize = 0;
        let mut internal_count: usize = 0;
        let mut child_to_parent: HashMap<TreeIndex, TreeIndex> = HashMap::new();

        for item in ParentFirstIterator::new(&self.blob, None) {
            let (index, block) = item?;
            if let Some(parent) = block.node.parent().0 {
                if child_to_parent.remove(&index) != Some(parent) {
                    return Err(Error::IntegrityParentChildMismatch(index));
                }
            }
            match block.node {
                Node::Internal(node) => {
                    internal_count += 1;
                    child_to_parent.insert(node.left, index);
                    child_to_parent.insert(node.right, index);
                }
                Node::Leaf(node) => {
                    leaf_count += 1;
                    let cached_index = self
                        .block_status_cache
                        .get_index_by_key(node.key)
                        .ok_or(Error::IntegrityKeyNotInCache(node.key))?;
                    if *cached_index != index {
                        return Err(Error::IntegrityKeyToIndexCacheIndex(
                            node.key,
                            index,
                            *cached_index,
                        ));
                    }
                    assert!(
                        !self.block_status_cache.is_index_free(index),
                        "{}",
                        format!("active index found in free index list: {index:?}")
                    );
                }
            }
        }

        let key_to_index_cache_length = self.block_status_cache.key_to_index.len();
        if leaf_count != key_to_index_cache_length {
            return Err(Error::IntegrityKeyToIndexCacheLength(
                leaf_count,
                key_to_index_cache_length,
            ));
        }
        let leaf_hash_to_index_cache_length = self.block_status_cache.leaf_hash_to_index.len();
        if leaf_count != leaf_hash_to_index_cache_length {
            return Err(Error::IntegrityLeafHashToIndexCacheLength(
                leaf_count,
                leaf_hash_to_index_cache_length,
            ));
        }
        let total_count = leaf_count + internal_count + self.block_status_cache.free_index_count();
        let extend_index = self.extend_index();
        if total_count != extend_index.0 as usize {
            return Err(Error::IntegrityTotalNodeCount(extend_index, total_count));
        }
        if !child_to_parent.is_empty() {
            return Err(Error::IntegrityUnmatchedChildParentRelationships(
                child_to_parent.len(),
            ));
        }

        Ok(())
    }
