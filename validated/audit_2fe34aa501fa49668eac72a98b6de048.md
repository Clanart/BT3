### Title
Precision Loss in `NewDynamicFeeChecker` Due to Intermediate Integer Division Causes Systematic Fee Under-Collection - (File: `ante/evm/fee_checker.go`)

---

### Summary

`NewDynamicFeeChecker` in `ante/evm/fee_checker.go` derives `feeCap` by performing integer division of the total submitted fee by gas (`fee / gas`) before computing the effective fee. Because Go integer division truncates the remainder, the effective fee deducted from the sender is always `Ōēż` the fee the sender submitted. The protocol systematically under-collects up to `gas ŌłÆ 1` units of the EVM denom per Cosmos transaction, and the sender retains the truncated remainder without paying it.

---

### Finding Description

In `NewDynamicFeeChecker`:

```go
// ante/evm/fee_checker.go lines 79-98
gas := feeTx.GetGas()
feeCoins := feeTx.GetFee()
fee := feeCoins.AmountOf(denom)

feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))   // ŌåÉ integer division, truncates
baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)

// ...

effectivePrice := sdkmath.NewIntFromBigInt(
    types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()),
)

effectiveFee := sdk.Coins{{
    Denom:  denom,
    Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas)),  // ŌåÉ multiply back
}}
``` [1](#0-0) 

The sequence is:

1. `feeCap = fee / gas` ŌĆö integer (truncating) division
2. `effectivePrice = min(baseFee + tip, feeCap)`
3. `effectiveFee = effectivePrice ├Ś gas`

When `feeCap` is the binding constraint (i.e., `feeCap Ōēż baseFee + tip`, which is the common case when the user does not set a large priority tip), step 3 yields:

```
effectiveFee = feeCap ├Ś gas = (fee / gas) ├Ś gas  Ōēż  fee
```

The difference `fee mod gas` is never deducted. The Cosmos SDK deducts exactly the `effectiveFee` returned by the fee checker, so the sender retains `fee mod gas` units without paying them.

This is the direct structural analog of the external report: division is performed in the middle of the calculation rather than at the end, causing the remainder to be silently discarded.

---

### Impact Explanation

Every Cosmos transaction routed through `NewDynamicFeeChecker` (i.e., all non-EVM Cosmos txs when the London hardfork / EIP-1559 fee market is active) under-pays fees by up to `gas ŌłÆ 1` units of the EVM denom per transaction.

- **Maximum loss per tx**: `gas ŌłÆ 1` units. For a tx with `gas = 1,000,000`, up to 999,999 units of the smallest denomination are not collected.
- **Worst-case percentage loss**: when `baseFee` is near its minimum (e.g., `MinGasPrice` floor), the ratio `(gas ŌłÆ 1) / (baseFee ├Ś gas)` approaches `1 / baseFee`. If `baseFee = 1`, the loss approaches 100% of the fee.
- **Exploitability**: a sender can craft `fee = baseFee ├Ś gas + (gas ŌłÆ 1)` to maximize the truncation. The `feeCap` check at line 86 passes (`feeCap = baseFee`), but only `baseFee ├Ś gas` is deducted instead of `baseFee ├Ś gas + (gas ŌłÆ 1)`.

This fits the allowed High impact: **ante handler bug that permits valid user funds/fees to be mis-accounted**. [2](#0-1) 

---

### Likelihood Explanation

- The `NewDynamicFeeChecker` is wired into the production ante handler chain via `evmd/ante/handler_options.go` and `evmd/ante/evm_handler.go`.
- It is triggered on every Cosmos (non-EVM) transaction when the London hardfork is active and EIP-1559 fee market is enabled ŌĆö a standard production configuration.
- No special privileges are required. Any unprivileged user submitting a Cosmos tx with a fee amount that is not an exact multiple of the gas limit triggers the truncation.
- The truncation is deterministic and reproducible on every such transaction. [3](#0-2) 

---

### Recommendation

Avoid the intermediate per-gas division. Compute the effective fee directly from the total submitted fee without first truncating to a per-gas price. One approach:

```go
// Instead of:
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
// ...
effectiveFee = effectivePrice.Mul(gas)

// Compute effectiveFee without intermediate truncation:
// Use fee directly as the cap in total-fee space, not per-gas space.
// Or, if a per-gas feeCap is needed for the baseFee comparison,
// use ceiling division so no remainder is silently dropped:
feeCap := fee.Add(sdkmath.NewIntFromUint64(gas - 1)).Quo(sdkmath.NewIntFromUint64(gas)) // ceil
```

Alternatively, restructure so all multiplications precede all divisions, mirroring the fix recommended in the external report.

---

### Proof of Concept

**Setup**: EIP-1559 enabled, `baseFee = 100`, `gas = 21000`, `tip = MaxInt64` (default, no extension option).

**Submitted fee**: `fee = 100 ├Ś 21000 + 20999 = 2,120,999`

**Execution**:
```
feeCap = 2,120,999 / 21000 = 100  (truncated; remainder 20999 discarded)
effectivePrice = min(baseFee + MaxInt64, feeCap) = min(huge, 100) = 100
effectiveFee = 100 ├Ś 21000 = 2,100,000
```

**Result**: sender submitted `2,120,999` but only `2,100,000` is deducted. The sender retains `20,999` units without paying them. The protocol under-collects `20,999` units on this single transaction. [4](#0-3)

```go
func NewDynamicFeeChecker(ethCfg *params.ChainConfig, evmParams *types.Params, feemarketParams *feemarkettypes.Params) authante.TxFeeChecker {
	return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
		feeTx, ok := tx.(sdk.FeeTx)
		if !ok {
			return nil, 0, fmt.Errorf("tx must be a FeeTx")
		}

		if ctx.BlockHeight() == 0 {
			// genesis transactions: fallback to min-gas-price logic
			return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
		}

		denom := evmParams.EvmDenom

		baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
		if baseFee == nil {
			// london hardfork is not enabled: fallback to min-gas-prices logic
			return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
		}

		// default to `MaxInt64` when there's no extension option.
		maxPriorityPrice := sdkmath.NewInt(math.MaxInt64)

		// get the priority tip cap from the extension option.
		if hasExtOptsTx, ok := tx.(authante.HasExtensionOptionsTx); ok {
			for _, opt := range hasExtOptsTx.GetExtensionOptions() {
				if extOpt, ok := opt.GetCachedValue().(*ethermint.ExtensionOptionDynamicFeeTx); ok {
					maxPriorityPrice = extOpt.MaxPriorityPrice
					break
				}
			}
		}

		if maxPriorityPrice.Sign() == -1 {
			return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, "priority fee is negative")
		}

		gas := feeTx.GetGas()
		feeCoins := feeTx.GetFee()
		fee := feeCoins.AmountOf(denom)

		feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
		baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)

		if feeCap.LT(baseFeeInt) {
			return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, "insufficient gas prices; got: %s required: %s", feeCap, baseFeeInt)
		}

		// calculate the effective gas price using the EIP-1559 logic.
		effectivePrice := sdkmath.NewIntFromBigInt(types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))

		// NOTE: create a new coins slice without having to validate the denom
		effectiveFee := sdk.Coins{
			{
				Denom:  denom,
				Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas)),
			},
		}
