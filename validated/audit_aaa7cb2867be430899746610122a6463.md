### Title
Solana `deploy_token` Missing Pause Check Allows Token Deployment During Emergency Pause — (`solana/programs/bridge_token_factory/src/lib.rs`)

### Summary

The Solana `bridge_token_factory` program's `deploy_token` instruction has no pause guard. When `ALL_PAUSED` is set, `finalize_transfer` and `init_transfer` are blocked, but `deploy_token` remains callable with any valid pre-obtained MPC-signed payload. This is directly inconsistent with the EVM implementation, which explicitly includes `PAUSED_DEPLOY_TOKEN` in its `pauseAll()` flag set.

### Finding Description

In `solana/programs/bridge_token_factory/src/lib.rs`, the three user-facing state-changing instructions are:

- `finalize_transfer` — checks `FINALIZE_TRANSFER_PAUSED` before proceeding [1](#0-0) 
- `init_transfer` — checks `INIT_TRANSFER_PAUSED` before proceeding [2](#0-1) 
- `deploy_token` — **no pause check at all** [3](#0-2) 

The `pause` instruction sets `config.paused = ALL_PAUSED`, which is the only pause state available on Solana. [4](#0-3) 

There is no `DEPLOY_TOKEN_PAUSED` constant defined anywhere in the Solana program. By contrast, the EVM `pauseAll()` explicitly includes `PAUSED_DEPLOY_TOKEN` in the flags it sets: [5](#0-4) 

And the EVM `deployToken` is guarded: [6](#0-5) 

The NEAR bridge similarly guards `deploy_token` with `#[pause(except(roles(Role::DAO)))]`: [7](#0-6) 

Solana's `deploy_token` verifies the MPC signature and then calls `initialize_token_metadata`, which writes new token state (metadata accounts, token-address mappings) to the program's on-chain accounts. This state persists after the pause is lifted and directly governs how `finalize_transfer` routes minting for that token.

### Impact Explanation

During an emergency pause on Solana, an actor holding a valid pre-signed `DeployTokenPayload` (obtained before the pause was triggered) can still call `deploy_token` to register a new token and its on-chain metadata. This creates new token-mapping state while the bridge is supposed to be fully halted.

The concrete harm: if the pause was triggered because of a discovered issue in the token deployment pipeline (e.g., a decimal normalization bug, a malicious token contract, or a compromised MPC key), the attacker can still complete the deployment using a signature obtained before the pause. The newly deployed token's metadata and address mapping then govern how `finalize_transfer` mints tokens to recipients once the pause is lifted — potentially with corrupted decimals, a wrong token address, or a mapping that misdirects minted supply. This fits **High: token-mapping or accounting corruption that breaks bridge collateralization or misdirects value**.

### Likelihood Explanation

Medium. The attacker only needs a valid signed `DeployTokenPayload` obtained before the pause. In practice, token deployments are initiated on NEAR and the MPC signs the payload asynchronously; a signed payload can exist in-flight at the moment of an emergency pause. Any relayer or user who observed the signed payload (e.g., from NEAR event logs) before the pause can submit it to Solana during the pause window.

### Recommendation

Add a `DEPLOY_TOKEN_PAUSED` bit constant to `solana/programs/bridge_token_factory/src/constants.rs` and include it in `ALL_PAUSED`. Then add the same `require!` guard at the top of `deploy_token` that `finalize_transfer` and `init_transfer` already use:

```rust
pub fn deploy_token(
    ctx: Context<DeployToken>,
    data: SignedPayload<DeployTokenPayload>,
) -> Result<()> {
    require!(
        ctx.accounts.common.config.paused & DEPLOY_TOKEN_PAUSED == 0,
        error::ErrorCode::Paused
    );
    // ...
}
```

This aligns Solana's pause coverage with the EVM and NEAR implementations.

### Proof of Concept

1. NEAR bridge operator initiates a token deployment for token `T`; the MPC signs a `DeployTokenPayload` for `T`. The signed payload is visible in NEAR event logs.
2. A security incident is discovered; the Solana admin calls `pause`, setting `config.paused = ALL_PAUSED`. `finalize_transfer` and `init_transfer` are now blocked.
3. An attacker (or a racing relayer)
