### Title
Stale `feemarketParams` Snapshot in `newCosmosAnteHandler` Bypasses Current Fee Market for Cosmos Transactions - (File: evmd/ante/handler_options.go)

### Summary
`newCosmosAnteHandler` captures `feemarketParams` as a one-time snapshot at construction time and passes a pointer to that snapshot into both `NewDynamicFeeChecker` and `MinGasPriceDecorator`. Because the EIP-1559 base fee is recalculated and persisted every block and `MinGasPrice` can be raised by governance, the snapshot diverges from the live store state. Cosmos transactions are therefore validated against a stale (potentially much lower) base fee and minimum gas price, allowing them to commit while paying less than the current fee market requires. The EVM ante handler (`newEthAnteHandler`) does not share this flaw — it fetches fresh params per transaction via `EVMBlockConfig`.

### Finding Description

In `evmd/ante/handler_options.go`, `newCosmosAnteHandler` reads params once from the store at the moment it is called:

```go
evmParams := options.EvmKeeper.GetParams(ctx)          // snapshot
feemarketParams := options.FeeMarketKeeper.GetParams(ctx)  // snapshot
```

Pointers to these local variables are then handed to two fee-enforcement components:

```go
txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
// ...
cosmos.NewMinGasPriceDecorator(options.FeeMarketKeeper, evmDenom, &feemarketParams),
```

`NewDynamicFeeChecker` resolves the base fee through `types.GetBaseFee`, which calls `feemarketParams.GetBaseFee()` — reading `feemarketParams.BaseFee` from the snapshot, not from the live KV store. [1](#0-0) 

`MinGasPriceDecorator.AnteHandle` reads `mpd.feemarketParams.MinGasPrice` directly from the same snapshot pointer. [2](#0-1) 

The base fee is updated on every block in `BeginBlock` → `CalculateBaseFee` → `SetBaseFee` → `SetParams`, which writes a new `BaseFee` value into the KV store. [3](#0-2)  The snapshot held by the Cosmos ante handler is never refreshed, so it permanently reflects the base fee at the moment `newCosmosAnteHandler` was constructed.

By contrast, `newEthAnteHandler` fetches a fresh `blockCfg` (including `FeeMarketParams` and `BaseFee`) on every invocation: [4](#0-3) 

The construction site for the Cosmos handler: [5](#0-4) 

### Impact Explanation

**Fee market bypass for Cosmos transactions.** Once the live base fee rises above the snapshot value (which happens whenever blocks are consistently above the gas target), any Cosmos transaction whose `feeCap` satisfies `snapshot_baseFee ≤ feeCap < live_baseFee` passes `NewDynamicFeeChecker`'s check at line 86 of `ante/evm/fee_checker.go` and is accepted. [6](#0-5)  The effective fee deducted is computed against the stale base fee, so the user pays less than the current fee market demands — valid user funds/fees are mis-accounted.

**Governance `MinGasPrice` increase rendered ineffective.** If governance raises `MinGasPrice` to combat spam or protect the network, `MinGasPriceDecorator` continues to enforce the old, lower value because it reads from the frozen snapshot. Cosmos transactions with fees between the old and new `MinGasPrice` are admitted, defeating the governance action. This matches the allowed High impact: "fee market, ante handler … bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."

### Likelihood Explanation

The base fee changes every block whenever gas usage deviates from the target. On a live chain with any meaningful load the snapshot diverges immediately after the first block following app startup. No special attacker capability is required — any user who observes the current base fee and submits a Cosmos transaction priced just above the stale snapshot value exploits the gap. The governance `MinGasPrice` vector requires a governance proposal to have passed, but that is a normal chain operation, not a privileged attack.

### Recommendation

Read `feemarketParams` from the keeper inside the returned closure (per-transaction), not at construction time, mirroring the pattern used in `newEthAnteHandler`:

```go
func newCosmosAnteHandler(options HandlerOptions, extra ...sdk.AnteDecorator) sdk.AnteHandler {
    return func(ctx sdk.Context, tx sdk.Tx, simulate bool) (sdk.Context, error) {
        evmParams := options.EvmKeeper.GetParams(ctx)
        feemarketParams := options.FeeMarketKeeper.GetParams(ctx)
        // build decorators using fresh params ...
    }
}
```

Alternatively, have `MinGasPriceDecorator` use its already-stored `feesKeeper` to read `MinGasPrice` live from the store on each call, rather than from the cached pointer.

### Proof of Concept

1. Chain starts; genesis `BaseFee = 1 Gwei`. `newCosmosAnteHandler` is called once, capturing `feemarketParams.BaseFee = 1 Gwei`.
2. Blocks run at full capacity. After N blocks, `CalculateBaseFee` has raised the live base fee to 10 Gwei and written it to the KV store via `SetBaseFee`. [7](#0-6) 
3. Attacker submits a Cosmos `MsgSend` (or any Cosmos SDK message) with `gasPrice = 2 Gwei` — above the stale snapshot (1 Gwei) but below the live base fee (10 Gwei).
4. `NewDynamicFeeChecker` evaluates `feeCap (2) >= baseFee (1, stale)` → passes. Effective fee is deducted at 2 Gwei, not 10 Gwei. [8](#0-7) 
5. `MinGasPriceDecorator` checks against the stale `MinGasPrice` from the snapshot — also passes.
6. Transaction commits. The protocol collects 2 Gwei per gas instead of the required 10 Gwei; the fee market invariant is violated for all Cosmos-path transactions for the lifetime of the node process.

	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
