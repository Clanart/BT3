### Title
Missing Class Existence Validation in `replace_class` Syscall Allows Permanent Freezing of Contract Funds - (File: `execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` function in `syscall_impls.cairo` (and its deprecated counterpart in `deprecated_execute_syscalls.cairo`) does not validate that the new class hash provided to `replace_class` corresponds to a declared contract class. An unprivileged contract can call `replace_class` with an arbitrary, non-existent class hash, permanently bricking the contract and freezing any funds it holds, because no valid proof can ever be generated for a transaction that subsequently calls the bricked contract.

### Finding Description
In `syscall_impls.cairo`, the `execute_replace_class` function contains an explicit TODO acknowledging the missing check:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}
``` [1](#0-0) 

The function directly updates the contract's class hash in `contract_state_changes` without verifying that the new class hash has a corresponding entry in the declared classes (`contract_class_changes`). The deprecated path in `deprecated_execute_syscalls.cairo` has the same omission: [2](#0-1) 

The downstream consequence is in `execute_entry_point`, which resolves a contract's class hash through `contract_class_changes` and then calls `find_element` to locate the compiled class:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
``` [3](#0-2) 

If the class hash set by `replace_class` is not declared, `dict_read` returns 0 (the default for an uninitialized dict entry), and `find_element` with key `0` will fail as a Cairo-level assertion, making the proof for any block containing a call to that contract unprovable.

### Impact Explanation
Any funds held in a contract whose class hash has been replaced with a non-existent value are **permanently frozen**. No valid STARK proof can be generated for any future transaction that calls the bricked contract, because `find_element` will fail at the OS level — not as a transaction revert, but as a proof-generation failure. The sequencer is forced to exclude all such transactions indefinitely. This matches the **Critical: Permanent freezing of funds** impact category.

### Likelihood Explanation
The `replace_class` syscall is callable by any contract on itself. A malicious or buggy contract can invoke it with an arbitrary felt as the new class hash. No privileged role is required. Any user who deploys or interacts with such a contract can trigger this. The explicit TODO comment in the production code confirms the check is intentionally deferred, meaning the gap is present in the current deployed OS.

### Recommendation
Before updating the contract's class hash in `contract_state_changes`, verify that the new class hash has a non-zero entry in `contract_class_changes` (i.e., it has been declared). This mirrors the existing pattern used in `execute_entry_point` where `dict_read` on `contract_class_changes` is the authoritative source of declared classes. The check should be added to both `execute_replace_class` in `syscall_impls.cairo` and its deprecated counterpart in `deprecated_execute_syscalls.cairo`.

### Proof of Concept
1. An unprivileged user deploys a contract `C` that holds user funds and contains logic to call `replace_class(0xdeadbeef)`, where `0xdeadbeef` is not a declared class hash.
2. The user sends a transaction invoking that logic. The OS executes `execute_replace_class`:
   - `request.class_hash = 0xdeadbeef`
   - No existence check is performed.
   - `contract_state_changes` is updated: contract `C` now has `class_hash = 0xdeadbeef`. [4](#0-3) 
3. The block is proven and committed to L1 with `C`'s class hash set to `0xdeadbeef`.
4. In any subsequent block, a transaction calling contract `C` causes `execute_entry_point` to call `dict_read{dict_ptr=contract_class_changes}(key=0xdeadbeef)`, returning `compiled_class_hash = 0`.
5. `find_element(..., key=0)` fails as a Cairo assertion — the proof cannot be generated. [5](#0-4) 
6. All funds in contract `C` are permanently inaccessible.
