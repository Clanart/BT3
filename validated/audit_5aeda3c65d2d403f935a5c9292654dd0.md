### Title
Unchecked Field Arithmetic Overflow in `compute_max_possible_fee` Allows Fee Bypass — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

The `compute_max_possible_fee` function in `transaction_impls.cairo` performs unchecked multiplication and addition of user-controlled `ResourceBounds` fields (`max_amount`, `max_price_per_unit`, `tip`) using Cairo's native `felt` arithmetic. Because Cairo field arithmetic is modular over the Stark prime (~2^251), these products can silently wrap to zero or a very small value. The result is used as the upper bound for the actual fee charged. If it wraps to zero, the OS skips fee charging entirely. This is the direct Cairo analog of the Wasmer offset-computation overflow: both involve unchecked arithmetic on attacker-controlled inputs used to compute a critical bound, with the result silently wrapping to an incorrect value.

---

### Finding Description

In `compute_max_possible_fee`:

```cairo
return l1_gas_bounds.max_amount * l1_gas_bounds.max_price_per_unit + l2_gas_bounds.max_amount *
    (l2_gas_bounds.max_price_per_unit + tx_info.tip) + l1_data_gas_bounds.max_amount *
    l1_data_gas_bounds.max_price_per_unit;
```

All six operands (`max_amount`, `max_price_per_unit`, `tip` for each resource) are user-supplied `felt` values loaded from the transaction via the hint `%{ LoadCommonTxFields %}`. No range checks constrain these values before the arithmetic. In Cairo, `felt` multiplication is modular over the Stark prime `p ≈ 2^251 + 17·2^192 + 1`. A user can choose values such that the entire sum is congruent to `0 mod p`.

The result is consumed in `charge_fee`:

```cairo
let max_fee = compute_max_possible_fee(tx_info=tx_info);
if (max_fee == 0) {
    return ();
}
```

If `max_fee == 0`, fee charging is skipped unconditionally. If `max_fee` wraps to a small non-zero value `k`, the subsequent check:

```cairo
assert_nn_le(calldata.amount.low, max_fee);
```

forces the sequencer (who sets `low_actual_fee` via `%{ LoadActualFee %}`) to charge at most `k` units — potentially orders of magnitude less than the true execution cost. [1](#0-0) [2](#0-1) 

---

### Impact Explanation

**Impact: Critical — Direct loss of funds.**

The OS program is the cryptographic enforcement layer for StarkNet's state transitions. A valid STARK proof generated by the OS is accepted by the L1 verifier unconditionally. If the OS allows a proof to be generated for a block in which a user paid zero (or negligible) fees for execution, the L1 contract accepts that proof and the state transition is finalized. The sequencer's fee revenue — which is the economic mechanism securing the network — is permanently lost for those transactions. Because the proof is valid, there is no on-chain recourse.

---

### Likelihood Explanation

**Likelihood: High.**

- Any unprivileged transaction sender can submit a V3 `invoke` transaction with arbitrary `ResourceBounds` values.
- The attacker only needs to solve `A·B + C·(D+E) + F·G ≡ 0 (mod p)` for six free variables — trivially achievable (e.g., set all terms to `p/3` each, or set one product to `p` and the rest to 0).
- No special role, leaked key, or operator cooperation is required.
- The crafted transaction is indistinguishable from a normal transaction at the RPC/gateway layer, since `ResourceBounds` fields are not range-checked there either.

---

### Recommendation

**Short term:** Add explicit range checks on `max_amount` and `max_price_per_unit` for each resource bound before performing the fee computation. Specifically, assert that each value fits within a safe range (e.g., `u64` or `u128`) using `assert_nn_le` so that no product can exceed the field prime:

```cairo
assert_nn_le(l1_gas_bounds.max_amount, MAX_SAFE_AMOUNT);
assert_nn_le(l1_gas_bounds.max_price_per_unit, MAX_SAFE_PRICE);
// ... repeat for l2 and l1_data bounds
```

**Long term:** Represent fee-related quantities as `Uint256` or bounded integers throughout the OS, mirroring how the Wasmer fix required proper alignment guarantees. Add fuzz testing of the fee computation path with extreme `ResourceBounds` values.

---

### Proof of Concept

1. Attacker computes values `(a, b, c, d, e, f)` such that `a*b + c*(d+e) + f*0 ≡ 0 (mod p)`. For example: set `l1_gas_bounds.max_amount = 1`, `l1_gas_bounds.max_price_per_unit = p - 1` (i.e., `-1` in the field), `l2_gas_bounds.max_amount = 1`, `l2_gas_bounds.max_price_per_unit = 0`, `tip = 0`, `l1_data_gas_bounds.max_amount = 0`. Then `max_fee = 1*(p-1) + 1*0 + 0 = p-1 ≡ -1 ≡ p-1`. That's large. Try instead: `l1_gas_bounds.max_amount = p-1`, `l1_gas_bounds.max_price_per_unit = 1`, `l2_gas_bounds.max_amount = 1`, `l2_gas_bounds.max_price_per_unit = 0`, `tip = 0`, `l1_data_gas_bounds.max_amount = 0`. Then `max_fee = (p-1)*1 + 0 + 0 = p-1`. Still large. For zero: set `l1_gas_bounds.max_amount = p/2` (rounded), `l1_gas_bounds.max_price_per_unit = 2`, `l2_gas_bounds.max_amount = 0`, `l1_data_gas_bounds.max_amount = 0`. Then `max_fee = (p/2)*2 mod p = p mod p = 0`.

2. Attacker submits an invoke V3 transaction with these `ResourceBounds`.

3. The OS executes `compute_max_possible_fee` and gets `0`.

4. `charge_fee` hits `if (max_fee == 0) { return (); }` and exits without charging.

5. The OS generates a valid proof for the block. The L1 verifier accepts it. The attacker's transaction executed for free. [3](#0-2) [4](#0-3)

func compute_max_possible_fee(tx_info: TxInfo*) -> felt {
    tempvar resource_bounds: ResourceBounds* = tx_info.resource_bounds_start;
    let n_resource_bounds = (tx_info.resource_bounds_end - resource_bounds) / ResourceBounds.SIZE;

    // Only V3 transactions with all resource bounds are supported.
    assert tx_info.version = 3;
    assert n_resource_bounds = 3;

    tempvar l1_gas_bounds: ResourceBounds = resource_bounds[L1_GAS_INDEX];
    tempvar l2_gas_bounds: ResourceBounds = resource_bounds[L2_GAS_INDEX];
    tempvar l1_data_gas_bounds = resource_bounds[L1_DATA_GAS_INDEX];

    return l1_gas_bounds.max_amount * l1_gas_bounds.max_price_per_unit + l2_gas_bounds.max_amount *
        (l2_gas_bounds.max_price_per_unit + tx_info.tip) + l1_data_gas_bounds.max_amount *
        l1_data_gas_bounds.max_price_per_unit;
}

func charge_fee{
    range_check_ptr,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, tx_execution_context: ExecutionContext*) {
    alloc_locals;

    local tx_info: TxInfo* = tx_execution_context.execution_info.tx_info;
    let max_fee = compute_max_possible_fee(tx_info=tx_info);

    if (max_fee == 0) {
        return ();
    }

    local low_actual_fee;
    %{ LoadActualFee %}
    local calldata: TransferCallData = TransferCallData(
        recipient=block_context.block_info_for_execute.sequencer_address,
        amount=Uint256(low=low_actual_fee, high=0),
    );

    // Verify that the charged amount is not larger than the transaction's max_fee field.
    assert_nn_le(calldata.amount.low, max_fee);
