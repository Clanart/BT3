### Title
`get_compiled_class` ignores `pending_classes` for `BlockId::Tag(Tag::Pending)`, returning `CLASS_HASH_NOT_FOUND` for valid pending-declared classes — (`crates/apollo_rpc/src/v0_8/api/api_impl.rs`)

### Summary

`starknet_getCompiledContractClass` called with `block_id = Pending` and a class hash that exists only in the pending block (not yet committed) always returns `CLASS_HASH_NOT_FOUND`. The sibling method `starknet_getClass` handles this correctly by consulting `self.pending_classes` first; `get_compiled_class` has no such check.

### Finding Description

`get_compiled_class` resolves the block number via `get_accepted_block_number`, which maps both `Tag::Latest` and `Tag::Pending` to the latest *committed* block number. [1](#0-0) 

It then queries only committed storage — first `get_class_definition_block_number` (Cairo 1), then `get_deprecated_class_definition_at` (Cairo 0) — and returns `CLASS_HASH_NOT_FOUND` if both return `None`. [2](#0-1) 

`self.pending_classes` is never consulted. By contrast, `get_class` explicitly reads `self.pending_classes` before touching committed storage when `block_id` is `Pending`: [3](#0-2) 

### Impact Explanation

Any caller of `starknet_getCompiledContractClass` with `block_id = Pending` for a class declared in the current pending block receives `CLASS_HASH_NOT_FOUND` instead of the compiled class. This is a wrong authoritative-looking RPC response for a valid pending state query. The impact matches the allowed scope: **High — pending view returns an authoritative-looking wrong value**.

The concrete corrupted value is: the RPC response is the error `CLASS_HASH_NOT_FOUND` where the correct response is the CASM/compiled class body present in `pending_classes`.

### Likelihood Explanation

The scenario is straightforward: a declare transaction lands in the pending block, `pending_classes` is populated, and any client (including the node's own fee-estimation or simulation path) calls `starknet_getCompiledContractClass` with `Pending`. No special privileges are required — any RPC client can trigger this.

### Recommendation

Mirror the pattern used in `get_class`: at the top of `get_compiled_class`, when `block_id` is `BlockId::Tag(Tag::Pending)`, read `self.pending_classes` and, if the class hash is found there, return the compiled class directly (or fall through to committed storage if not found in pending).
