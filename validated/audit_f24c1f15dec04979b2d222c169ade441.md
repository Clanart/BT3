### Title
Missing Declared Class Validation in `execute_replace_class` Allows Permanent Contract Freezing — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary `class_hash` from the caller without verifying that the hash corresponds to a previously declared contract class. This is directly acknowledged by a TODO comment in the code. An unprivileged contract can invoke `replace_class` with an undeclared or fabricated class hash, permanently corrupting its own on-chain class pointer. Any funds held by that contract become permanently frozen because all future calls to it will fail when the OS cannot resolve the invalid class hash to compiled code.

---

### Finding Description

Inside `execute_replace_class`, the OS reads `request.class_hash` directly from the syscall request and writes it into `contract_state_changes` without any check that the hash exists in `contract_class_changes` (the declared-class registry):

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment explicitly acknowledges the missing guard. By contrast, `execute_declare_transaction` enforces `prev_value=0` to prevent double-declaration and calls `finalize_class_hash` to verify the class hash pre-image:

```cairo
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [2](#0-1) 

When `execute_call_contract` later resolves a callee, it reads `state_entry.class_hash` and passes it directly as the `class_hash` field of the `ExecutionContext`:

```cairo
let (state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(key=contract_address);
tempvar execution_context: ExecutionContext* = new ExecutionContext(
    ...
    class_hash=state_entry.class_hash,
    ...
);
``` [3](#0-2) 

If `state_entry.class_hash` is an undeclared value, the OS cannot resolve it to compiled bytecode, causing every subsequent call to that contract to fail irreversibly.

**Analog to the external report:** In the Optimism `PreimageOracle`, anyone could call `loadPrecompilePreimagePart()` with a future precompile address and arbitrary data, setting `preimagePartOk[key][offset] = true` before the precompile existed. Here, any contract can call `replace_class` with an arbitrary or future class hash, writing it into the committed state before (or instead of) a legitimate declaration — the missing validation is structurally identical.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` field in `contract_state_changes` is set to an undeclared value and the block is committed, the state root encodes the corrupted pointer. Every future `call_contract` or `library_call` targeting that address will fail at class resolution. There is no recovery path: `replace_class` itself requires the contract to be callable, and the contract is no longer callable. All ERC-20 balances, NFTs, or protocol liquidity held by the contract are permanently inaccessible.

---

### Likelihood Explanation

**Medium-High.** The `replace_class` syscall is the standard upgrade mechanism for Cairo 1 contracts. Any contract that exposes an upgrade entry point callable by an admin, a governance vote, or a DAO proposal is reachable. A malicious or compromised admin, a governance attack, or a reentrancy path that reaches the upgrade function can supply an arbitrary class hash. The TODO comment confirms the developers are aware the check is absent and have deferred it, meaning the window is open in the current production code.

---

### Recommendation

Before writing the new `StateEntry`, verify that `class_hash` is present in `contract_class_changes` (current block declarations) or in the pre-existing global class registry supplied to the OS. Concretely, add a `dict_read` on `contract_class_changes` for `class_hash` and assert the returned compiled class hash is non-zero, mirroring the guard already present in `execute_declare_transaction`.

---

### Proof of Concept

1. Deploy contract `Vault` holding user funds. `Vault` exposes an `upgrade(new_class_hash)` entry point that calls `replace_class(new_class_hash)`.
2. Attacker (admin or governance attacker) calls `upgrade(0xdeadbeef)` where `0xdeadbeef` is never declared.
3. The OS executes `execute_replace_class`:
   - `class_hash = 0xdeadbeef` is read from the request.
   - The TODO-guarded check is absent; no validation occurs.
   - `contract_state_changes[Vault_address].class_hash` is updated to `0xdeadbeef`.
4. The block is proven and committed; the state root now encodes `Vault.class_hash = 0xdeadbeef`.
5. Any subsequent `call_contract(Vault_address, ...)` reads `class_hash = 0xdeadbeef`, fails to find compiled bytecode, and reverts.
6. All funds inside `Vault` are permanently frozen with no recovery path. [4](#0-3)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    let revert_log = &revert_log[1];

    return ();
}
