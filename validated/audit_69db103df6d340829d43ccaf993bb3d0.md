Audit Report

## Title
Unprivileged `refundETH()` drains any residual ETH left on the router by a prior payable call — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary
`refundETH()` transfers the router's entire native ETH balance to `msg.sender` with no access control. Every swap entry-point (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`) is `payable`, and the `pay()` helper wraps only exactly the required ETH amount, leaving any excess on the contract. Any user who calls a payable swap with `msg.value > amountIn` leaves residual ETH on the router that any attacker can immediately steal by calling `refundETH()`.

## Finding Description
`refundETH()` contains no caller check and sends the full contract balance to whoever calls it: [1](#0-0) 

The `pay()` helper, when `token == WETH` and `nativeBalance >= value`, wraps exactly `value` wei and leaves the remainder untouched on the contract: [2](#0-1) 

All four swap entry-points are `payable` and do not refund excess ETH after the swap completes: [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) 

The `receive()` guard (only WETH can push ETH via a plain transfer) does not protect against ETH arriving via `msg.value` in a payable call — that path bypasses `receive()` entirely: [7](#0-6) 

**Exploit flow:**
1. Victim calls `exactInputSingle{value: 2 ether}(...)` with `amountIn = 1 ether`.
2. The swap callback triggers `pay()`, which deposits exactly 1 ether as WETH and sends it to the pool; the remaining 1 ether stays on the router.
3. Attacker calls `refundETH()` in the same or next block and receives the full router balance (1 ether).

The `multicall` wrapper is the intended composition mechanism for bundling `refundETH()` with a swap, but this is not enforced at the contract level — a direct call to any payable swap function is fully valid and leaves excess ETH exposed.

## Impact Explanation
Direct, complete theft of user principal. A victim who sends excess ETH to cover price uncertainty (a common pattern) loses that excess to the first caller of `refundETH()`. There is no recovery path. This meets the High severity threshold for direct loss of user principal.

## Likelihood Explanation
Requires a user to call a payable swap function directly (not via `multicall`) with `msg.value > amountIn`. This is a realistic and common pattern for handling price uncertainty. A MEV bot monitoring the mempool can back-run the victim's transaction atomically in the same block. The project's own test suite confirms the behavior: [8](#0-7) 

## Recommendation
Change `refundETH()` to only refund ETH that arrived in the current `multicall` context. The simplest fix is to track the original `msg.value` in transient storage at `multicall` entry and only refund up to that amount to the original `multicall` caller. Alternatively, add a `recipient` parameter (as `unwrapWETH9` does) so the caller explicitly designates who receives the refund, and enforce that `refundETH` is only callable within a `multicall` context by checking a transient "in-multicall" flag.

## Proof of Concept
```solidity
function test_refundETH_attackerStealsVictimExcessETH() public {
    address victim   = makeAddr("victim");
    address attacker = makeAddr("attacker");
    vm.deal(victim, 2 ether);

    // Victim calls exactInputSingle directly with excess ETH.
    // amountIn = 1 ether, msg.value = 2 ether → 1 ether excess stays on router.
    vm.prank(victim);
    router.exactInputSingle{value: 2 ether}(
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(pool),
            tokenIn: address(weth),
            tokenOut: address(token1),
            zeroForOne: true,
            amountIn: 1 ether,
            amountOutMinimum: 0,
            recipient: victim,
            deadline: block.timestamp + 60,
            priceLimitX64: 0,
            extensionData: ""
        })
    );

    assertEq(address(router).balance, 1 ether);

    uint256 attackerBefore = attacker.balance;
    vm.prank(attacker);
    router.refundETH();

    assertEq(attacker.balance - attackerBefore, 1 ether, "attacker stole victim ETH");
    assertEq(victim.balance, 0, "victim lost excess ETH");
}
```
