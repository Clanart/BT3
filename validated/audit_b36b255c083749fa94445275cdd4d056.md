### Title
`meta_tx_v0` Hash Excludes Nonce, Enabling Signature Replay to Drain Account Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_meta_tx_v0` syscall constructs a version-0 transaction hash that commits to no nonce and is not rooted to any specific outer transaction. Because the hash is purely a function of `(contract_address, selector, calldata, chain_id)`, any valid `meta_tx_v0` signature can be replayed by any unprivileged actor across any number of outer transactions, causing the target account contract's `__execute__` entry point to be invoked repeatedly with the same signed calldata — enabling direct, unbounded loss of funds.

---

### Finding Description

`execute_meta_tx_v0` in `syscall_impls.cairo` is a syscall available to any executing contract. It constructs a synthetic version-0 `TxInfo` and calls the target contract's `__execute__` entry point using a caller-supplied signature.

The transaction hash is computed by `compute_meta_tx_v0_hash`:

```cairo
// transaction_hash.cairo L302-L313
let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
    tx_hash_prefix=INVOKE_HASH_PREFIX,
    version=0,
    contract_address=contract_address,
    entry_point_selector=entry_point_selector,
    calldata_size=calldata_size,
    calldata=calldata,
    max_fee=0,
    chain_id=chain_id,
    additional_data_size=0,           // ← no nonce
    additional_data=cast(0, felt*),   // ← no nonce
);
``` [1](#0-0) 

The `additional_data_size=0` means no nonce is committed into the hash. The synthetic `TxInfo` also hardcodes `nonce=0`:

```cairo
// syscall_impls.cairo L343-L363
tempvar new_tx_info = new TxInfo(
    version=0,
    ...
    nonce=0,   // ← always zero
    ...
);
``` [2](#0-1) 

Two OS-level replay guards are both explicitly bypassed for version-0 transactions:

**1. Nonce check is skipped:**
```cairo
// execute_transaction_utils.cairo L63-L67
func check_and_increment_nonce(...) {
    // Do not handle nonce for version 0.
    if (tx_info.version == 0) {
        return ();
    }
``` [3](#0-2) 

**2. `__validate__` is skipped:**
```cairo
// execute_transaction_utils.cairo L127-L130
    // Do not run "__validate__" for version 0.
    if (tx_execution_info.tx_info.version == 0) {
        return ();
    }
``` [4](#0-3) 

The result: the hash `H = Pedersen(INVOKE_PREFIX, 0, contract_address, selector, calldata_hash, 0, chain_id)` is fully determined by public, static inputs. Any party who observes a valid `(calldata, signature)` pair for a given target account can replay it by invoking `meta_tx_v0` from any contract they control, any number of times, across any block.

---

### Impact Explanation

**Critical — Direct loss of funds.**

A `meta_tx_v0` call whose calldata encodes a token transfer (e.g., ERC-20 `transfer` or `__execute__` wrapping a transfer) produces a signature that is permanently valid for that `(contract_address, calldata)` tuple. An attacker who observes one legitimate use can replay it in subsequent blocks until the target account is drained. No privileged access is required; any contract can issue the `meta_tx_v0` syscall.

---

### Likelihood Explanation

**High.** The `meta_tx_v0` syscall is a public protocol feature callable from any deployed contract. A single on-chain observation of a legitimate `meta_tx_v0` invocation (e.g., from a block explorer or mempool) yields a replayable signature. The attacker only needs to deploy a contract that re-issues the same syscall with the captured `(contract_address, selector, calldata, signature)`. No key material, privileged role, or off-chain coordination is required.

---

### Recommendation

Include a nonce in the `meta_tx_v0` hash to bind each signature to a single execution:

1. **Add a nonce field to `MetaTxV0Request`** and pass it as `additional_data` in `compute_meta_tx_v0_hash` (analogous to how `compute_l1_handler_transaction_hash` passes `nonce` as `additional_data`).
2. **Enforce nonce uniqueness** at the OS level: read the target account's current nonce from `contract_state_changes`, assert it matches the request nonce, and increment it — even for version-0 meta transactions.
3. Alternatively, root the hash to the outer transaction hash (e.g., include `old_tx_info.transaction_hash` in the hash inputs), so each `meta_tx_v0` signature is bound to exactly one outer transaction.

---

### Proof of Concept

1. Alice's account contract `A` legitimately calls `meta_tx_v0` with `calldata = [transfer(Bob, 100)]` and a valid ECDSA signature `σ` over `H(A, __execute__, calldata, chain_id)`. This executes and transfers 100 tokens to Bob.

2. Attacker Eve observes `σ` and `calldata` on-chain.

3. Eve deploys contract `Evil` containing:
   ```
   func attack():
       meta_tx_v0(contract_address=A, selector=__execute__, calldata=calldata, signature=σ)
   ```

4. Eve submits an invoke transaction calling `Evil.attack()`. The OS computes the same hash `H` (no nonce, no outer-tx binding), the account contract `A` verifies `σ` against `H` and accepts it, and the transfer executes again.

5. Eve repeats step 4 until `A`'s balance is zero. Each repetition is a valid, provable state transition accepted by the StarkNet OS.
