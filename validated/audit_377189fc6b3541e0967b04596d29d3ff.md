### Title
Missing Recipient Address Validation in `initTransfer` Enables Permanent Irrecoverable Token Lock - (File: evm/src/omni-bridge/contracts/OmniBridge.sol, starknet/src/omni_bridge.cairo)

### Summary
The `initTransfer` function in `OmniBridge.sol` and `init_transfer` in `omni_bridge.cairo` accept an arbitrary `recipient` string/`ByteArray` with no format, length, or emptiness validation before locking or burning user tokens. A user who supplies an empty string or a structurally invalid cross-chain address will have their tokens permanently locked in the source-chain bridge with no on-chain recovery path, because the destination-chain finalization will always fail and no refund mechanism exists.

### Finding Description

**Root cause — EVM bridge (`OmniBridge.sol::initTransfer`):**

```solidity
function initTransfer(
    address tokenAddress,
    uint128 amount,
    uint128 fee,
    uint128 nativeFee,
    string calldata recipient,   // ← no validation whatsoever
    string calldata message
) external payable whenNotPaused(PAUSED_INIT_TRANSFER) {
    currentOriginNonce += 1;
    if (fee >= amount) { revert InvalidFee(); }
    // tokens locked / burned immediately below
    ...
    IERC20(tokenAddress).safeTransferFrom(msg.sender, address(this), amount);
    ...
    emit BridgeTypes.InitTransfer(..., recipient, message);
}
``` [1](#0-0) 

The function performs only a fee check. The `recipient` string is never tested for emptiness, minimum length, or structural validity (e.g., a valid NEAR account ID, a 20-byte hex EVM address, a base58 Solana address, etc.). Tokens are locked/burned and the `InitTransfer` event is emitted before any cross-chain validation can occur.

**Root cause — StarkNet bridge (`omni_bridge.cairo::init_transfer`):**

```cairo
fn init_transfer(
    ref self: ContractState,
    token_address: ContractAddress,
    amount: u128,
    fee: u128,
    native_fee: u128,
    recipient: ByteArray,   // ← no validation
    message: ByteArray,
) {
    assert(!_is_paused(@self, PAUSE_INIT_TRANSFER), 'ERR_INIT_TRANSFER_PAUSED');
    assert(amount > 0, 'ERR_ZERO_AMOUNT');
    assert(fee < amount, 'ERR_INVALID_FEE');
    // tokens burned/locked immediately below
    ...
    self.emit(Event::InitTransfer(InitTransfer { ..., recipient, ... }))
}
``` [2](#0-1) 

Same pattern: only amount and fee are checked; `recipient` is accepted as any `ByteArray` including zero-length.

**Exploit path:**

1. Attacker or user calls `initTransfer(tokenAddress, amount, 0, 0, "", "")` (empty recipient) on the EVM or StarkNet bridge.
2. Tokens are immediately locked in the vault (native tokens) or burned (bridge tokens). The `InitTransfer` event is emitted with `recipient = ""`.
3. A relayer picks up the event and submits the Merkle/MPC proof to the NEAR bridge's `fin_transfer`.
4. The NEAR bridge attempts to parse `""` as an `OmniAddress`. Parsing fails; the NEAR transaction panics.
5. Because the NEAR transaction reverts, the proof is never consumed and the transfer is never finalized.
6. The tokens remain locked on the source chain. There is no `rescue`, `refund`, or `unlock` function in `OmniBridge.sol` or `omni_bridge.cairo` that can release them without a successful NEAR-side finalization.
7. The lock is permanent.

The same outcome occurs with a structurally malformed recipient (e.g., a 20-byte hex string supplied when the destination chain expects a NEAR account ID), because the NEAR bridge will reject it during deserialization.

### Impact Explanation

**Critical — Permanent irrecoverable lock of user funds in bridge vault flows.**

Any user who supplies an empty or malformed `recipient` string loses their tokens permanently. The source-chain bridge has no refund path: `initTransfer` does not record the sender for later recovery, and there is no admin rescue function. The locked/burned tokens cannot be recovered without an out-of-band contract upgrade or admin key action, neither of which is part of the standard protocol.

This matches the allowed impact: *"Permanent freezing, irrecoverable lock, or unclaimable settlement of user or protocol funds in bridge, token, or vault flows."*

### Likelihood Explanation

**Medium.** The entry point is fully public and requires no privilege. Realistic trigger scenarios include:

- A front-end bug that submits an empty recipient field.
- A user copy-pasting an address in the wrong format for the destination chain (e.g., a `0x…` EVM address when bridging to NEAR, which is not a valid NEAR account ID).
- A programmatic integrator that omits the recipient field.
- A deliberate griefing attack where an adversary burns their own tokens to demonstrate the flaw or to lock protocol-owned liquidity.

No privileged access, leaked key, or external dependency compromise is required.

### Recommendation

Add explicit recipient validation before any token movement in both `initTransfer` (EVM) and `init_transfer` (StarkNet):

1. **Non-empty check**: `require(bytes(recipient).length > 0, "ERR_EMPTY_RECIPIENT")` (Solidity) / `assert(recipient.len() > 0, 'ERR_EMPTY_RECIPIENT')` (Cairo).
2. **Maximum length check**: Enforce an upper bound consistent with the longest valid cross-chain address the NEAR bridge accepts (e.g., ≤ 64 bytes for a NEAR account ID).
3. **Structural check** (recommended): Validate that the recipient string matches the expected format for the destination chain (e.g., contains only valid NEAR account ID characters, or is a valid hex-encoded address for EVM destinations).

These checks mirror the BLS parameter length checks recommended in the reference report and prevent the analogous permanent-lock outcome.

    function initTransfer(
        address tokenAddress,
        uint128 amount,
        uint128 fee,

        string calldata recipient,

        currentOriginNonce += 1;

            revert InvalidFee();

        uint256 extensionValue;
        if (tokenAddress == address(0)) {

                revert InvalidFee();

            extensionValue = msg.value - amount - nativeFee;

            extensionValue = msg.value - nativeFee;
            if (customMinters[tokenAddress] != address(0)) {
                IERC20(tokenAddress).safeTransferFrom(
                    msg.sender,
                    customMinters[tokenAddress],
                    amount

                    amount
                );

                BridgeToken(tokenAddress).burn(msg.sender, amount);

                IERC20(tokenAddress).safeTransferFrom(
                    msg.sender,
                    address(this),
                    amount
                );

            fee,

        fn init_transfer(
            ref self: ContractState,
            token_address: ContractAddress,
            amount: u128,
            fee: u128,
            native_fee: u128,
            recipient: ByteArray,
            message: ByteArray,
        ) {
            assert(!_is_paused(@self, PAUSE_INIT_TRANSFER), 'ERR_INIT_TRANSFER_PAUSED');

            assert(amount > 0, 'ERR_ZERO_AMOUNT');
            assert(fee < amount, 'ERR_INVALID_FEE');

            let origin_nonce = self.current_origin_nonce.read() + 1;
            self.current_origin_nonce.write(origin_nonce);

            let caller = get_caller_address();

            if self.is_bridge_token(token_address) {
                IBridgeTokenDispatcher { contract_address: token_address }
                    .burn(caller, amount.into());
            } else {
                let success = IERC20Dispatcher { contract_address: token_address }
                    .transfer_from(caller, get_contract_address(), amount.into());
                assert(success, 'ERR_TRANSFER_FROM_FAILED');
            }

            if native_fee > 0 {
                let native_token = self.strk_token_address.read();
                let success = IERC20Dispatcher { contract_address: native_token }
                    .transfer_from(caller, get_contract_address(), native_fee.into());
                assert(success, 'ERR_FEE_TRANSFER_FAILED');
            }

            self
                .emit(
                    Event::InitTransfer(
                        InitTransfer {
                            sender: caller,
                            token_address,
                            origin_nonce,
                            amount,
                            fee,
                            native_fee,
                            recipient,
                            message,
                        },
                    ),
                )
        }
