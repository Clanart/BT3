### Title
Unbounded `memcpy` in `execute_send_message_to_l1` with Fixed Gas Cost Enables OS Proof Exhaustion — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

`execute_send_message_to_l1` charges a fixed, payload-size-independent gas cost (`SEND_MESSAGE_TO_L1_GAS_COST`) but then performs a `memcpy` whose iteration count is directly proportional to the attacker-controlled `payload_size`. No upper-bound assertion on `payload_size` exists, unlike every other variable-length array processed by the OS. An unprivileged contract deployer can craft a transaction that passes the gas check at a fixed cost while forcing the OS Cairo program to execute an arbitrarily large number of steps during proof generation, potentially exhausting the prover's step budget and preventing block finalization.

---

### Finding Description

In `execute_send_message_to_l1` the gas reduction is performed first with a **constant** cost:

```cairo
let success = reduce_syscall_gas_and_write_response_header(
    total_gas_cost=SEND_MESSAGE_TO_L1_GAS_COST,   // fixed constant
    request_struct_size=SendMessageToL1Request.SIZE
);
```

After the gas check passes, the function computes an unbounded length and copies it:

```cairo
tempvar payload_size = request.payload_end - payload_start;
...
memcpy(
    dst=outputs.messages_to_l1 + MessageToL1Header.SIZE,
    src=payload_start,
    len=payload_size          // no bound check
);
```

`memcpy` in Cairo is a recursive loop that consumes one step per felt copied. `payload_size` is derived from two attacker-supplied felt pointers with **no assertion** constraining its magnitude.

**Contrast with every other variable-length field in the same OS:**

| Location | Array | Bound check |
|---|---|---|
| `fill_account_tx_info` L218 | `signature_len` | `assert_nn_le(signature_len, SIERRA_ARRAY_LEN_BOUND - 1)` |
| `get_invoke_tx_execution_context` L485 | `calldata_size` | `assert_nn_le(tx_execution_context.calldata_size, SIERRA_ARRAY_LEN_BOUND - 1)` |
| `execute_meta_tx_v0` L323 | `signature` span | `assert_nn_le(request.signature_end - request.signature_start, SIERRA_ARRAY_LEN_BOUND - 1)` |
| `prepare_constructor_execution_context` L534 | `constructor_calldata_size` | `assert_nn_le(constructor_calldata_size, SIERRA_ARRAY_LEN_BOUND - 1)` |
| **`execute_send_message_to_l1`** | **`payload_size`** | **none** | [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

### Impact Explanation

The StarkNet OS Cairo program is executed by the prover to generate a STARK proof for each block. The prover operates under a hard step-count ceiling. If `execute_send_message_to_l1` is invoked with a payload of N felts, the OS program consumes O(N) additional steps beyond what the fixed `SEND_MESSAGE_TO_L1_GAS_COST` accounts for. A sufficiently large N causes the OS program to exceed the prover's step budget, making proof generation impossible. A block whose proof cannot be generated cannot be finalized, halting the network's ability to confirm new transactions — matching the **High: Network not being able to confirm new transactions** impact class.

---

### Likelihood Explanation

The attack path is fully reachable by an unprivileged contract deployer:

1. Deploy a contract whose `execute` entry point calls `send_message_to_l1` with a large `Span<felt252>` payload (e.g., 10^6 elements stored in contract storage or calldata).
2. Submit an invoke transaction targeting that contract.
3. The sequencer's gas check passes because `SEND_MESSAGE_TO_L1_GAS_COST` is fixed and the user's L2 gas budget covers it.
4. The sequencer includes the transaction in a block.
5. The OS program runs `memcpy` for 10^6 iterations, consuming far more prover steps than the gas accounting predicted.
6. If the prover's step budget is exhausted, proof generation fails and the block cannot be finalized.

The only mitigating factor is whether the sequencer's Bouncer independently tracks OS-level step overhead for message payloads. The OS code itself provides no such protection.

---

### Recommendation

Add the same `SIERRA_ARRAY_LEN_BOUND` guard used for every other variable-length field immediately after computing `payload_size`:

```cairo
tempvar payload_size = request.payload_end - payload_start;
assert_nn_le(payload_size, SIERRA_ARRAY_LEN_BOUND - 1);  // add this line
```

Alternatively, make the gas cost proportional to payload size (analogous to `DEPLOY_CALLDATA_FACTOR_GAS_COST` and `META_TX_V0_CALLDATA_FACTOR_GAS_COST` patterns already used in the same file):

```cairo
let specific_base_gas_cost = SEND_MESSAGE_TO_L1_GAS_COST +
    SEND_MESSAGE_TO_L1_PAYLOAD_FACTOR_GAS_COST * payload_size;
``` [5](#0-4)

### Proof of Concept

```
1. Attacker deploys contract C with storage array of 500_000 felts.

2. C.__execute__() calls:
       send_message_to_l1(
           to_address=0xdeadbeef,
           payload=array_of_500_000_felts.span()
       )

3. Attacker submits InvokeV3 tx targeting C with sufficient L2 gas to cover
   SEND_MESSAGE_TO_L1_GAS_COST (fixed, small).

4. Sequencer gas check: passes (fixed cost).

5. OS execute_send_message_to_l1:
       payload_size = 500_000          // no bound check
       memcpy(..., len=500_000)        // 500_000 Cairo VM steps

6. Repeated across multiple txs in the same block, total OS step count
   exceeds prover ceiling → proof generation fails → block not finalized
   → network halted.
```

func execute_send_message_to_l1{range_check_ptr, syscall_ptr: felt*, outputs: OsCarriedOutputs*}(
    contract_address: felt
) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, SendMessageToL1Request*);
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=SEND_MESSAGE_TO_L1_GAS_COST, request_struct_size=SendMessageToL1Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    tempvar payload_start = request.payload_start;
    tempvar payload_size = request.payload_end - payload_start;

    assert [outputs.messages_to_l1] = MessageToL1Header(
        from_address=contract_address, to_address=request.to_address, payload_size=payload_size
    );
    memcpy(
        dst=outputs.messages_to_l1 + MessageToL1Header.SIZE, src=payload_start, len=payload_size
    );
    let (outputs) = os_carried_outputs_new(
        messages_to_l1=outputs.messages_to_l1 + MessageToL1Header.SIZE + payload_size,
        messages_to_l2=outputs.messages_to_l2,
    );

    return ();
}
