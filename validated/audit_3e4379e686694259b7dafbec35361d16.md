### Title
Missing `state_diff_commitment` verification in P2P state diff sync allows malicious peer to persist arbitrary storage key/value pairs — (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

### Summary

`StateDiffStreamBuilder::parse_data_for_block` reads only `state_diff_length` from the stored block header to gate acceptance of peer-supplied `StateDiffChunk` data. It never reads or checks `state_diff_commitment`. A malicious P2P peer that knows the target length K can send exactly K length-units of chunks carrying arbitrary `contract_address`/`storage_key`/`value` triples. These pass the sole guard at line 99, are assembled into a `ThinStateDiff`, and are written unconditionally to the node's `contract_storage` table via `append_state_diff`. The stored state diff is permanently inconsistent with the `state_diff_commitment` recorded in the block header.

### Finding Description

**Entry point — `parse_data_for_block`:**

The function reads `target_state_diff_len` from the stored header:

```rust
let target_state_diff_len = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    ...
    .state_diff_length   // ← only field read; state_diff_commitment is ignored
    ...
``` [1](#0-0) 

It then accumulates chunks and applies the sole structural guard:

```rust
if current_state_diff_len != target_state_diff_len {
    return Err(ParseDataError::BadPeer(...));
}
``` [2](#0-1) 

After passing this check, the assembled `ThinStateDiff` is returned with no hash computation and no comparison against `state_diff_commitment`. [3](#0-2) 

**Write path — `write_to_storage`:**

```rust
storage_writer.begin_rw_txn()?.append_state_diff(self.1, self.0)?.commit()?;
``` [4](#0-3) 

`append_state_diff` writes directly into the `contract_storage` table (and `nonces`, `deployed_contracts`, etc.) with no commitment check: [5](#0-4) 

**The `state_diff_commitment` field exists in the header and is populated** — the block header proto carries it, and the header sync stores it — but `parse_data_for_block` never touches it: [6](#0-5) 

**No downstream guard closes the gap.** The `apollo_committer`'s `verify_state_diff_hash` check exists but is in a completely separate component that receives state diffs via `CommitBlockRequest` messages from the sequencer — it does not read from `apollo_storage` and is not invoked in the P2P sync path: [7](#0-6) 

### Impact Explanation

The `contract_storage` table written by `append_state_diff` is the same table read by `StateReader::get_storage_at`, which backs both RPC storage queries and blockifier execution state reads. Corrupted storage key/value pairs for block N propagate forward into every subsequent state read at `StateNumber ≥ N`. This satisfies **Critical** impact: *wrong storage value from blockifier/syscall/execution logic for accepted input*, and also **High** impact: *RPC execution / `starknet_getStorageAt` / `starknet_getStateUpdate` returns an authoritative-looking wrong value*.

### Likelihood Explanation

The attacker needs only to:
1. Be a reachable P2P peer (no special privilege).
2. Know `state_diff_length` for block N — this is public information in the block header.
3. Craft `StateDiffChunk` messages whose `len()` sum equals `target_state_diff_len` but whose storage key/value content is arbitrary.

No cryptographic material needs to be forged. The `unite_state_diffs` function only rejects duplicate keys within the same chunk stream, not wrong values. [8](#0-7) 

### Recommendation

After assembling `result` and before returning `Ok(Some(...))`, read `state_diff_commitment` from the stored header and verify:

```rust
let expected_commitment = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .expect(...)
    .state_diff_commitment
    .ok_or(...)?;

let actual_commitment = calculate_state_diff_hash(&result);
if actual_commitment != expected_commitment {
    return Err(ParseDataError::BadPeer(BadPeerError::StateDiffCommitmentMismatch { ... }));
}
```

This mirrors the guard already present in `apollo_committer::commit_block_inner`. [7](#0-6) 

### Proof of Concept

```rust
// Precondition: block header for block N is stored with
//   state_diff_length = K, state_diff_commitment = H_correct
//
// Attacker sends K length-units of StateDiffChunk with wrong storage values:
let malicious_chunk = StateDiffChunk::ContractDiff(ContractDiff {
    contract_address: victim_contract,
    storage_diffs: indexmap! { victim_key => attacker_value },
    // len() of this chunk == K (attacker controls the number of entries)
    ..Default::default()
});
// parse_data_for_block accumulates: current_state_diff_len = K == target → passes line 99
// write_to_storage calls append_state_diff → contract_storage[victim_contract][victim_key] = attacker_value
//
// Verification:
// calculate_state_diff_hash(&stored_diff) != H_correct  ← invariant violated
// starknet_getStorageAt(victim_contract, victim_key, block=N) → attacker_value  ← wrong committed value served
```
