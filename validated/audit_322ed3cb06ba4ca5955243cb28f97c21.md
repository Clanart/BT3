### Title
`ProofOfInclusion::valid()` Does Not Validate Against a Trusted Root — Forged DataLayer Inclusion Proofs Pass Validation - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

---

### Summary

`ProofOfInclusion::valid()` only checks internal hash-chain consistency within the proof itself. It never compares the computed root against any externally-supplied trusted root. Because `root_hash()` is derived entirely from the proof's own data, the final equality check inside `valid()` is a tautology. An attacker who supplies a crafted `ProofOfInclusion` (via Python `from_bytes` / `from_py_object`) can make `valid()` return `true` for any claimed key-value pair in any arbitrary tree, bypassing DataLayer state verification.

---

### Finding Description

`ProofOfInclusion` is a Streamable struct exposed to Python via PyO3 bindings:

```rust
// crates/chia-datalayer/src/merkle/proof_of_inclusion.rs
pub struct ProofOfInclusion {
    pub node_hash: Hash,
    pub layers: Vec<ProofOfInclusionLayer>,
}
``` [1](#0-0) 

The `valid()` method iterates over layers, verifying each layer's `combined_hash` equals the hash of the previous hash combined with `other_hash`. After the loop, `existing_hash` holds the last layer's `combined_hash`. The final check is:

```rust
existing_hash == self.root_hash()
``` [2](#0-1) 

But `root_hash()` is defined as:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← taken directly from the proof itself
    } else {
        self.node_hash
    }
}
``` [3](#0-2) 

Since `existing_hash` at loop-end is exactly `last.combined_hash`, the final comparison `existing_hash == self.root_hash()` is **always true** after the loop completes without an early return. `valid()` therefore only checks that the proof's hash chain is internally self-consistent — it never verifies the claimed root against any trusted external value.

The struct is fully constructible from untrusted bytes via the Python binding:

```python
# wheel/python/chia_rs/datalayer.pyi
class ProofOfInclusion:
    node_hash: bytes32
    layers: list[ProofOfInclusionLayer]
    def root_hash(self) -> bytes32: ...
    def valid(self) -> bool: ...
    @classmethod
    def from_bytes(cls, blob: bytes) -> Self: ...
``` [4](#0-3) 

The `pyclass(get_all, from_py_object)` attribute also allows direct Python-side construction of arbitrary `ProofOfInclusion` objects with attacker-chosen field values. [5](#0-4) 

---

### Impact Explanation

Any caller — Python or Rust — that uses `proof.valid()` as the sole check for DataLayer state inclusion will accept a forged proof. An attacker can craft a `ProofOfInclusion` with an arbitrary `node_hash` (representing any key-value pair they wish to falsely prove) and a set of layers whose hashes chain consistently to an attacker-chosen root. `valid()` returns `true`. The caller has no indication that the root does not correspond to any real DataLayer tree.

This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.**

---

### Likelihood Explanation

The Python API exposes `valid()` as the primary and only named validation method on `ProofOfInclusion`. There is no `valid_for_root(trusted_root)` variant. The method name implies complete validation. Callers who receive a `ProofOfInclusion` from an untrusted peer and call `proof.valid()` will believe the proof is sound. The `from_bytes` and `from_py_object` entry points are reachable by any unprivileged input.

---

### Recommendation

1. **Add a `trusted_root` parameter to `valid()`** (or add a separate `valid_for_root(root: &Hash) -> bool` method) that compares the computed root against a caller-supplied trusted root, rather than against `self.root_hash()` which is derived from the proof itself.

2. **Deprecate or remove the standalone `valid()` method** from the public Python API, or document clearly that it does not verify against any trusted root and must always be paired with an explicit `root_hash()` comparison against a known-good root.

3. **Update `get_proof_of_inclusion`** to return the trusted root alongside the proof, making it structurally impossible to forget the root check.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
import hashlib

# Attacker wants to forge proof that node_hash X is in some tree
fake_node_hash = bytes([0xAA] * 32)
fake_other_hash = bytes([0xBB] * 32)

# Compute a combined_hash that is internally consistent
# (left side = 0, meaning fake_node_hash is on the left)
h = hashlib.sha256()
h.update(b'\x00')  # internal node prefix (simplified)
h.update(fake_node_hash)
h.update(fake_other_hash)
fake_combined = h.digest()  # exact hash depends on calculate_internal_hash impl

layer = ProofOfInclusionLayer(
    other_hash_side=1,          # right side
    other_hash=fake_other_hash,
    combined_hash=fake_combined, # attacker sets this to match
)

proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

# valid() returns True because the chain is internally consistent.
# root_hash() returns fake_combined — an attacker-chosen value.
assert proof.valid()  # passes — no trusted root was ever checked
print("Forged proof accepted. Claimed root:", proof.root_hash().hex())
```

The `valid()` call succeeds for a completely fabricated proof. The caller has no way to know the returned `root_hash()` does not correspond to any real DataLayer commitment unless they independently compare it to a trusted root — which the API does not require or enforce.

    pub fn valid(&self) -> bool {
        let mut existing_hash = self.node_hash;

        for layer in &self.layers {
            let calculated_hash = crate::calculate_internal_hash(
                &existing_hash,
                layer.other_hash_side,
                &layer.other_hash,
            );

                return false;
            }

            existing_hash = calculated_hash;
        }

        existing_hash == self.root_hash()
    }
