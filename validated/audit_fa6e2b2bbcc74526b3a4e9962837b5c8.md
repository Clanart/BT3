### Title
Felt Arithmetic Overflow in `compute_max_possible_fee` Allows Fee-Free Transaction Execution — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

`compute_max_possible_fee` computes the maximum chargeable fee using unchecked felt-field multiplication. Because `max_price_per_unit` is only constrained to be non-negative (≤ (P−1)/2 ≈ 2^250) while `max_amount` is bounded to 2^64−1, their product can silently wrap around modulo the field prime P ≈ 2^251. An attacker can choose resource-bound values whose combined felt sum equals exactly P, making `max_fee ≡ 0 (mod P)`. The early-exit guard `if (max_fee == 0) { return (); }` then skips fee charging entirely, so the transaction is executed at zero cost.

---

### Finding Description

`compute_max_possible_fee` at lines 87–102 of `transaction_impls.cairo` computes:

```
max_fee = l1_gas.max_amount * l1_gas.max_price_per_unit
        + l2_gas.max_amount * (l2_gas.max_price_per_unit + tip)
        + l1_data.max_amount * l1_data.max_price_per_unit
```

all in felt arithmetic (mod P). [1](#0-0) 

The only bounds enforced on `max_price_per_unit` come from `pack_resource_bounds`, called during transaction-hash computation:

```cairo
assert_nn_le(resource_bounds.max_amount, 2 ** 64 - 1);
assert_nn(resource_bounds.max_price_per_unit);          // only: ≥ 0, i.e. ≤ (P-1)/2
``` [2](#0-1) 

`assert_nn` does **not** impose an upper bound; it only verifies the value is in the "non-negative" felt range `[0, (P−1)/2]`. With `max_amount ≤ 2^64−1` and `max_price_per_unit ≤ (P−1)/2 ≈ 2^250`, the product can reach ≈ 2^314, overflowing P ≈ 2^251 many times.

The result of `compute_max_possible_fee` is therefore an arbitrary felt value, including 0.

When `max_fee == 0`, `charge_fee` returns immediately without executing any ERC-20 transfer:

```cairo
if (max_fee == 0) {
    return ();
}
``` [3](#0-2) 

This is the direct analog of the external report: just as `steth_amount − steth_amount_adj` can be less than the transfer fee cost, here the felt-overflowed `max_fee` can be less than (or equal to) zero, making the fee-charging step a no-op.

---

### Impact Explanation

**Critical — Direct loss of funds.**

A transaction sender can execute arbitrary contract calls (invoke, deploy-account, declare) on the StarkNet OS without paying any fee. The sequencer's fee revenue is zeroed for every such crafted transaction. At scale, an attacker can drain sequencer compensation entirely, making block production economically unviable and causing a network-level loss of funds.

---

### Likelihood Explanation

**Medium.**

- The attacker controls `max_amount`, `max_price_per_unit`, and `tip` directly in the signed transaction body — a fully unprivileged entry path.
- The overflow arithmetic is deterministic and requires only arithmetic knowledge of P; no key material or privileged access is needed.
- The sequencer's off-chain mempool logic typically uses native 64/128-bit or arbitrary-precision arithmetic (not felt arithmetic), so it would compute a large `max_fee` and include the transaction, while the OS computes `max_fee = 0` and skips charging.
- The only mitigation is if the sequencer independently validates resource bounds using felt arithmetic before inclusion — which is not enforced by the OS protocol itself.

---

### Recommendation

1. **Add an explicit upper bound on `max_price_per_unit`** in `pack_resource_bounds` (e.g., `assert_nn_le(resource_bounds.max_price_per_unit, 2 ** 128 - 1)`). This keeps the product `max_amount * max_price_per_unit ≤ (2^64−1)(2^128−1) < 2^192 ≪ P`, preventing overflow.

2. **Alternatively**, perform the fee summation using a multi-limb (Uint256) accumulator rather than raw felt arithmetic, mirroring how `calldata.amount` is already represented as `Uint256`.

3. **Add a minimum-fee guard** analogous to the external report's recommendation: after computing `max_fee`, assert it is at least some protocol-defined minimum before proceeding, so that a near-zero overflowed result is caught rather than silently accepted.

---

### Proof of Concept

Let P = 3618502788666131213697322783095070105623107215331596699973092056135872020481 (StarkNet field prime).

Choose:
```
A = floor((P-1)/3)   = 1206167596222043737899107594365023368541035738443865566657697352045290673493
B = floor((P-1)/3)   = 1206167596222043737899107594365023368541035738443865566657697352045290673493
C = P - A - B        = 1206167596222043737899107594365023368541035738443865566657697352045290673495
```

All three satisfy `assert_nn` (each ≤ (P−1)/2 ≈ 1.8×10^75 ✓) and `assert_nn_le(max_amount, 2^64−1)` with `max_amount = 1`.

Craft a V3 transaction with:
```
l1_gas_bounds   = { max_amount: 1, max_price_per_unit: A }
l2_gas_bounds   = { max_amount: 1, max_price_per_unit: B }
l1_data_bounds  = { max_amount: 1, max_price_per_unit: C }
tip             = 0
```

OS computes:
```
max_fee = 1*A + 1*(B+0) + 1*C = A + B + C = P ≡ 0 (mod P)
```

`charge_fee` hits `if (max_fee == 0) { return (); }` and exits — **zero fee charged**, transaction fully executed. [4](#0-3) [5](#0-4)

func compute_max_possible_fee(tx_info: TxInfo*) -> felt {
    tempvar resource_bounds: ResourceBounds* = tx_info.resource_bounds_start;
    let n_resource_bounds = (tx_info.resource_bounds_end - resource_bounds) / ResourceBounds.SIZE;

    // Only V3 transactions with all resource bounds are supported.
    assert tx_info.version = 3;
    assert n_resource_bounds = 3;

    tempvar l1_gas_bounds: ResourceBounds = resource_bounds[L1_GAS_INDEX];
    tempvar l2_gas_bounds: ResourceBounds = resource_bounds[L2_GAS_INDEX];
    tempvar l1_data_gas_bounds = resource_bounds[L1_DATA_GAS_INDEX];

    return l1_gas_bounds.max_amount * l1_gas_bounds.max_price_per_unit + l2_gas_bounds.max_amount *
        (l2_gas_bounds.max_price_per_unit + tx_info.tip) + l1_data_gas_bounds.max_amount *
        l1_data_gas_bounds.max_price_per_unit;
}

// Charges a fee from the user.
// If max_fee is not 0, validates that the selector matches the entry point of an account contract
// and executes an ERC20 transfer on the behalf of that account contract.

// Arguments:

func charge_fee{

func pack_resource_bounds{range_check_ptr}(resource_bounds: ResourceBounds) -> felt {
    assert_nn_le(resource_bounds.max_amount, 2 ** 64 - 1);
    assert_nn(resource_bounds.max_price_per_unit);
