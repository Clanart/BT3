### Title
Non-EVM Bank Token Orphaning on SELFDESTRUCT — (`x/evm/statedb/statedb.go`)

### Summary

When a self-destructed contract address holds non-EVM-denom bank balances (IBC vouchers, CosmWasm bridge tokens, etc.), `stateDB.Commit()` only burns the EVM denom and then calls `DeleteAccount`, which removes the auth account. The non-EVM tokens remain in the bank module under an address that no longer has an account, making them permanently unrecoverable.

### Finding Description

The self-destruct path in `Commit()` explicitly handles only the EVM denom: [1](#0-0) 

The comment at lines 812–815 is a self-admission of the gap:

```
// Only the EVM denom is burned here. Non-EVM-native tokens (IBC, CosmWasm
// bridge) held by the destroyed address are not drained and may remain as
// orphaned bank balances.
```

`DeleteAccount` then removes the auth account record and all EVM storage, but never touches the bank module for non-EVM denoms: [2](#0-1) 

`DeleteAccount` calls `accountKeeper.RemoveAccount` (line 215), which removes the account from the auth store. After this, the bank module still holds the IBC/bridge token balance under that address, but there is no account to own or spend it. The `Keeper.SubBalance` used for the EVM denom burn is single-denom only: [3](#0-2) 

There is no call to `bankKeeper.GetAllBalances` or any multi-denom drain before `DeleteAccount`.

### Impact Explanation

Non-EVM bank tokens (IBC vouchers, CW20 bridge tokens, etc.) held by a self-destructed contract address become permanently orphaned in the bank module. No account exists to send or burn them; they are unrecoverable. This constitutes an unauthorized permanent destruction of Cosmos bank funds through the EVM state transition path — matching the Critical impact: *"Unauthorized theft, mint, burn bypass, or balance transfer of EVM-denom or Cosmos bank funds through Ethermint transaction execution or stateDB/native action logic."*

### Likelihood Explanation

The precondition is realistic:
1. A contract with a `SELFDESTRUCT` opcode is deployed (attacker-controlled or any public selfdestruct function).
2. IBC tokens are sent to that contract address (common in DeFi bridges, escrow contracts, or simply by user mistake).
3. The attacker (contract owner or anyone with access to the selfdestruct function) triggers `SELFDESTRUCT`.

No privileged role, governance, or validator collusion is required. The attacker only needs to control a contract and have non-EVM tokens present at that address.

### Recommendation

Before calling `DeleteAccount`, drain **all** bank balances held by the contract address, not just the EVM denom. The fix should:

1. Call `bankKeeper.GetAllBalances(ctx, cosmosAddr)` to enumerate every denom.
2. For each non-EVM denom, transfer the balance to a designated burn/community address or to the IBC module for proper escrow release, rather than leaving it orphaned.
3. Only then call `DeleteAccount`.

Alternatively, block `SELFDESTRUCT` (return an error from `Commit`) if the address holds any non-EVM bank balance, forcing the contract to drain those balances first via a native precompile call.

### Proof of Concept

```
1. Deploy contract C with a public selfdestruct() function.
2. IBC-transfer 100 uatom to address(C) via the IBC transfer module.
   → bankKeeper.GetAllBalances(address(C)) = [100 ibc/XXXX]
3. Call C.selfdestruct() via EthereumTx.
   → stateDB.SelfDestruct burns EVM denom (0 in this case).
   → stateDB.Commit():
       - SubBalance(evmDenom) = no-op (zero balance)
       - DeleteAccount removes auth account for address(C)
4. Assert: bankKeeper.GetAllBalances(address(C)) still returns [100 ibc/XXXX]
5. Assert: accountKeeper.GetAccount(address(C)) == nil
6. The 100 ibc/XXXX are permanently locked — no account can spend them.
```

```go
func (k *Keeper) SubBalance(ctx sdk.Context, addr sdk.AccAddress, coin sdk.Coin) (uint256.Int, error) {
	coins := sdk.NewCoins(coin)
	prevBalance := k.GetBalance(ctx, addr, coin.Denom)
	if err := k.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, coins); err != nil {
		return uint256.Int{}, err
	}
	if err := k.bankKeeper.BurnCoins(ctx, types.ModuleName, coins); err != nil {
		return uint256.Int{}, err
	}
	return prevBalance, nil
```

```go
func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
	cosmosAddr := sdk.AccAddress(addr.Bytes())
	acct := k.accountKeeper.GetAccount(ctx, cosmosAddr)
	if acct == nil {
		return nil
	}

	// NOTE: only Ethereum accounts (contracts) can be selfdestructed
	_, ok := acct.(ethermint.EthAccountI)
	if !ok {
		return errorsmod.Wrapf(types.ErrInvalidAccount, "type %T, address %s", acct, addr)
	}

	// clear storage
	k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool {
		k.SetState(ctx, addr, key, nil)
		return true
	})

	// remove auth account
	k.accountKeeper.RemoveAccount(ctx, acct)

	k.debugLog(ctx, "account suicided",
		"ethereum-address", addr,
		"cosmos-address", cosmosAddr,
	)

	return nil
```
