### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary felt value as a new class hash without verifying that the hash corresponds to a previously declared contract class. This allows any contract to atomically replace its own class hash with an undeclared (non-existent) value, permanently rendering the contract un-callable and freezing all funds held within it.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall. After deducting gas, it reads the caller's current `StateEntry` and writes a new `StateEntry` with the attacker-supplied `class_hash` directly into `contract_state_changes`, with no check that the new class hash has ever been declared:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment explicitly acknowledges the missing check. The OS commits this state change unconditionally. Once the block is proven and the state root is updated, the contract's class hash is permanently set to the invalid value. Any subsequent call to the contract will fail at class-lookup time because no compiled class exists for that hash, making the contract permanently inaccessible.

The analog to the external report's vulnerability class is direct: just as `unwrapToken()` lacked time-based restrictions that allowed an atomic flashloan loop to bypass the intended graduation constraint, `execute_replace_class` lacks a declared-class check that allows an atomic single-transaction call to bypass the intended constraint that only declared classes may be used as contract implementations. [1](#0-0) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

A contract that holds user assets (ETH, ERC-20 tokens, or any other on-chain value) can have its class hash replaced with an arbitrary undeclared felt. After the block is proven:

- The state root permanently encodes the invalid class hash.
- Every subsequent `call_contract` or `invoke` targeting that address will fail at class resolution, reverting with no way to recover.
- No upgrade, migration, or rescue path exists because the contract's own `__execute__` entry point is unreachable.
- All funds stored in the contract's storage are permanently frozen with no protocol-level recovery mechanism. [2](#0-1) 

---

### Likelihood Explanation

**High.** The attack requires only:

1. Deploying a contract (permissionless on StarkNet).
2. Attracting user deposits into the contract.
3. Calling `replace_class` with any non-zero felt that has no corresponding declared class (e.g., `0xdeadbeef`).

No privileged role, leaked key, or external dependency is required. The `replace_class` syscall is reachable from any Cairo 1 contract via the standard syscall interface. The missing check is explicitly flagged in the source with a TODO dated 2026, confirming it is absent in the current production code path. The `execute_syscalls` dispatcher routes `REPLACE_CLASS_SELECTOR` directly to `execute_replace_class` with no pre-filter. [3](#0-2) 

---

### Recommendation

Inside `execute_replace_class`, before writing the new `StateEntry`, verify that `class_hash` exists in `contract_class_changes` (i.e., it was declared in the current block) **or** that it exists in the committed class tree (i.e., it was declared in a prior block). Concretely:

- Perform a `dict_read` on `contract_class_changes` for `class_hash` and assert the result is non-zero, **or**
- Extend the OS to carry a "declared classes" set and assert membership before accepting the replacement.

Until this check is implemented, the OS proof system accepts state transitions that permanently destroy contract executability. [4](#0-3) 

---

### Proof of Concept

**Attack flow (single transaction, no flashloan needed):**

1. Attacker deploys `VaultContract` with a legitimate class hash. Users deposit funds.
2. Attacker submits an `invoke` transaction calling `VaultContract.__execute__`, which internally issues the `replace_class` syscall with `class_hash = 0x1` (or any felt with no declared class).
3. The OS executes `execute_replace_class`:
   - Gas is deducted.
   - `contract_state_changes` is updated: `VaultContract → StateEntry(class_hash=0x1, ...)`.
   - A revert-log entry is written (but the transaction does not revert).
4. The block is proven. The new state root encodes `VaultContract.class_hash = 0x1`.
5. Any subsequent call to `VaultContract` triggers class lookup for `0x1`. No compiled class exists → execution fails unconditionally.
6. All user funds in `VaultContract`'s storage are permanently frozen.

The state-transition path through the OS:

```
invoke tx → execute_invoke_function_transaction
  → non_reverting_select_execute_entry_point_func (execute)
    → execute_syscalls (REPLACE_CLASS_SELECTOR)
      → execute_replace_class  ← no class-existence check
        → dict_update(contract_state_changes, key=vault_addr, new_value=StateEntry(class_hash=0x1))
  → charge_fee (succeeds, tx not reverted)
→ state_update: new state root encodes class_hash=0x1 for vault_addr
``` [5](#0-4) [3](#0-2) [6](#0-5)

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
