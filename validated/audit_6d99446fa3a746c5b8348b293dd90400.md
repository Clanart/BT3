### Title
Pending-data/pending-classes TOCTOU race causes `UndeclaredClassHash` or stale class body during RPC execution — (`crates/apollo_rpc/src/pending.rs`, `crates/apollo_rpc/src/v0_8/api/api_impl.rs`)

---

### Summary

`JsonRpcServerImpl::call` (and `estimate_fee`, `simulate_transactions`) reads `Arc<RwLock<PendingData>>` and `Arc<RwLock<PendingClasses>>` under two **separate, non-atomic** lock acquisitions. The pending sync writes `pending_data` (with `declared_classes=[{H,C}]`) and then, in a subsequent async task, writes H into `pending_classes`. An RPC call that lands between those two writes assembles an `ExecutionPendingData` where `declared_classes` contains H but `classes` does not. `ExecutionStateReader::get_compiled_class` then fails to find H in the pending classes, falls through to committed storage, and either returns `UndeclaredClassHash(H)` or a stale committed class body.

---

### Finding Description

**Write side — `pending_sync.rs`:**

`get_pending_data` writes the two shared objects in two separate critical sections:

```
// Step 1: write pending_data (declared_classes=[{H,C}] is now visible)
*pending_data.write().await = new_pending_data;          // line 182
// lock released — window opens here
```

Then, back in `sync_pending_data`, the loop reads `declared_classes` from `pending_data` and spawns **separate async tasks** that eventually call:

```
pending_classes.write().await.add_class(class_hash, class);          // line 196
pending_classes.write().await.add_compiled_class(class_hash, casm);  // line 206
```

These tasks involve a network round-trip to the central source (`central_source.get_class(class_hash).await`), so the window can last hundreds of milliseconds. [1](#0-0) [2](#0-1) 

**Read side — `api_impl.rs`:**

```rust
Some(client_pending_data_to_execution_pending_data(
    read_pending_data(&self.pending_data, &txn).await?,   // lock 1: acquired + released
    self.pending_classes.read().await.clone(),             // lock 2: acquired + released
))
```

The two locks are acquired sequentially. Between the release of lock 1 and the acquisition of lock 2, the pending sync has not yet written H into `pending_classes`. [3](#0-2) 

**Assembly — `pending.rs`:**

`client_pending_data_to_execution_pending_data` blindly combines the two snapshots:

```rust
ExecutionPendingData {
    declared_classes: client_pending_data.state_update.state_diff.declared_classes,  // has H
    classes: pending_classes,   // does NOT have H yet
    ...
}
``` [4](#0-3) 

**Execution — `state_reader.rs`:**

`get_compiled_class` checks `pending_data.classes` first. H is absent, so it falls through:

1. If `class_manager_handle` is set: fetches from class manager, then calls `is_contract_class_declared` against committed storage — H is not committed → `UndeclaredClassHash(H)`.
2. If no class manager: calls `get_contract_class` against committed storage — H is not committed → `UndeclaredClassHash(H)`.
3. If H happens to be committed with an older body: returns the stale committed class. [5](#0-4) 

---

### Impact Explanation

Any unprivileged user who calls `starknet_call`, `starknet_estimateFee`, or `starknet_simulateTransactions` with `block_id = "pending"` targeting a contract that uses a class declared in the current pending block will receive an authoritative-looking wrong result — either a spurious `UndeclaredClassHash` error or execution against a stale committed class body — during the window between the two writes. The window is bounded by a network fetch from the central source, typically hundreds of milliseconds, and recurs on every pending block update.

Impact: **High** — "RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."

---

### Likelihood Explanation

The race is structural and unconditional: every time the pending sync receives new pending data containing a newly declared class, the window opens. The window duration equals the latency of `central_source.get_class` + `central_source.get_compiled_class` (two sequential network fetches). Under normal network conditions this is 50–500 ms. Any RPC call arriving during that window hits the bug. No special attacker capability is required.

---

### Recommendation

Acquire both locks atomically before assembling `ExecutionPendingData`. Replace the two separate reads with a single critical section that holds both read guards simultaneously, or restructure the pending sync to write both `pending_data` and the corresponding classes under a single composite lock before making the new state visible to RPC readers.

---

### Proof of Concept

```rust
// Pseudocode property test
let pending_data = Arc::new(RwLock::new(PendingData::default()));
let pending_classes = Arc::new(RwLock::new(PendingClasses::default()));

// Simulate pending sync: write pending_data with declared_classes=[{H,C}]
// but do NOT yet write H into pending_classes (intermediate state)
*pending_data.write().await = PendingData {
    state_update: PendingStateUpdate {
        state_diff: ClientStateDiff {
            declared_classes: vec![DeclaredClassHashEntry {
                class_hash: H,
                compiled_class_hash: C,
            }],
            ..Default::default()
        },
        ..Default::default()
    },
    ..Default::default()
};
// pending_classes intentionally left empty — H not yet inserted

// Simulate RPC call arriving in the window
let client_pd = pending_data.read().await.clone();
let pc = pending_classes.read().await.clone(); // empty
let exec_pd = client_pending_data_to_execution_pending_data(client_pd, pc);

// exec_pd.declared_classes contains H, but exec_pd.classes does not
// ExecutionStateReader::get_compiled_class(H) will return UndeclaredClassHash(H)
assert!(exec_pd.classes.get_class(H).is_none());
assert!(exec_pd.declared_classes.iter().any(|e| e.class_hash == H));
// Invariant violated: declared_classes ⊄ classes
```

    fn get_compiled_class(&self, class_hash: ClassHash) -> StateResult<RunnableCompiledClass> {
        if let Some(pending_classes) =
            self.maybe_pending_data.as_ref().map(|pending_data| &pending_data.classes)
        {
            if let Some(api_contract_class) = pending_classes.get_class(class_hash) {
                match api_contract_class {
                    ApiContractClass::ContractClass(sierra) => {
                        if let Some(pending_casm) = pending_classes.get_compiled_class(class_hash) {
                            let sierra_version = sierra.get_sierra_version()?;
                            let runnable_compiled_class = RunnableCompiledClass::V1(
                                CompiledClassV1::try_from((pending_casm, sierra_version))
                                    .map_err(StateError::ProgramError)?,
                            );
                            return Ok(runnable_compiled_class);
                        }
                    }
                    ApiContractClass::DeprecatedContractClass(pending_deprecated_class) => {
                        return Ok(RunnableCompiledClass::V0(
                            CompiledClassV0::try_from(pending_deprecated_class)
                                .map_err(StateError::ProgramError)?,
                        ));
                    }
                }
            }
        }

        if let Some((class_manager_client, run_time_handle)) = &self.class_manager_handle {
            let contract_class = run_time_handle
                .block_on(class_manager_client.get_executable(class_hash))
                .map_err(|e| StateError::StateReadError(e.to_string()))?
                .ok_or(StateError::UndeclaredClassHash(class_hash))?;

            return match contract_class {
                ContractClass::V1(casm_contract_class) => {
                    let is_declared = is_contract_class_declared(
                        &self.storage_reader.begin_ro_txn().map_err(storage_err_to_state_err)?,
                        &class_hash,
                        self.state_number,
                    )
                    .map_err(|e| StateError::StateReadError(e.to_string()))?;

                    if is_declared {
                        Ok(RunnableCompiledClass::V1(casm_contract_class.try_into()?))
                    } else {
                        Err(StateError::UndeclaredClassHash(class_hash))
                    }
                }
                // TODO(shahak): Verify cairo0 as well after get_class_definition_block_number is
                // fixed.
                ContractClass::V0(deprecated_contract_class) => {
                    Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
                }
            };
        }

        match get_contract_class(
            &self.storage_reader.begin_ro_txn().map_err(storage_err_to_state_err)?,
            &class_hash,
            self.state_number,
        ) {
            Ok(Some(contract_class)) => Ok(contract_class),
            Ok(None) => Err(StateError::UndeclaredClassHash(class_hash)),
            Err(ExecutionUtilsError::CasmTableNotSynced) => {
                self.missing_compiled_class.set(Some(class_hash));
                Err(StateError::StateReadError("Casm table not fully synced".to_string()))
            }
            Err(ExecutionUtilsError::ProgramError(err)) => Err(StateError::ProgramError(err)),
            Err(ExecutionUtilsError::StorageError(err)) => Err(storage_err_to_state_err(err)),
            Err(ExecutionUtilsError::SierraValidationError(err)) => {
                Err(StateError::StarknetApiError(err))
            }
        }
    }
