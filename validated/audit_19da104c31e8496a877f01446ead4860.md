### Title
`compute_meta_tx_v0_hash` Lacks Nonce — Unbounded Signature Replay of `meta_tx_v0` Syscall - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/transaction_hash/transaction_hash.cairo`)

---

### Summary

The `compute_meta_tx_v0_hash` function omits any nonce from the hash preimage, and the OS never checks or increments the target contract's nonce for version-0 meta-transactions. A valid meta-tx-v0 signature over `(contract_address, selector, calldata, chain_id)` is therefore reusable across an unlimited number of distinct outer transactions, enabling an attacker to replay the same authorized action — including token transfers — indefinitely.

---

### Finding Description

`compute_meta_tx_v0_hash` delegates to `deprecated_get_transaction_hash` with `additional_data_size=0` and `additional_data=cast(0, felt*)`:

```cairo
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(
    contract_address: felt,
    entry_point_selector: felt,
    calldata: felt*,
    calldata_size: felt,
    chain_id: felt,
) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        tx_hash_prefix=INVOKE_HASH_PREFIX,
        version=0,
        contract_address=contract_address,
        entry_point_selector=entry_point_selector,
        calldata_size=calldata_size,
        calldata=calldata,
        max_fee=0,
        chain_id=chain_id,
        additional_data_size=0,          // ← no nonce
        additional_data=cast(0, felt*),
    );
    return tx_hash;
}
``` [1](#0-0) 

The resulting hash is a pure function of `(INVOKE_HASH_PREFIX, 0, contract_address, selector, calldata, 0, chain_id)` — identical for every invocation with the same arguments, regardless of block or time.

`execute_meta_tx_v0` then constructs the inner `TxInfo` with `nonce=0` hardcoded:

```cairo
tempvar new_tx_info = new TxInfo(
    version=0,
    ...
    nonce=0,
    ...
);
``` [2](#0-1) 

The OS then explicitly skips nonce enforcement for version-0 transactions in `check_and_increment_nonce`:

```cairo
// Do not handle nonce for version 0.
if (tx_info.version == 0) {
    return ();
}
``` [3](#0-2) 

And `run_validate` also skips `__validate__` for version 0:

```cairo
// Do not run "__validate__" for version 0.
if (tx_execution_info.tx_info.version == 0) {
    return ();
}
``` [4](#0-3) 

The three defects combine: the hash has no nonce, the TxInfo carries `nonce=0`, and the OS never increments the target contract's nonce. A signature that was valid once remains valid forever for the same `(contract_address, selector, calldata, chain_id)` tuple.

---

### Impact Explanation

**Direct loss of funds (Critical).** Any account contract whose `__execute__` entry point authorizes actions based on the transaction hash (the standard pattern) will accept the same meta-tx-v0 signature repeatedly. If the calldata encodes a token transfer or any other value-moving operation, an attacker who obtains one valid signature can drain the target account by replaying it across arbitrarily many outer transactions.

---

### Likelihood Explanation

The `meta_tx_v0` syscall is reachable by any unprivileged contract deployed on StarkNet. An attacker needs only to:

1. Observe or obtain a single valid meta-tx-v0 signature (e.g., from a prior on-chain execution, which is public).
2. Deploy a contract that issues `meta_tx_v0` with that signature and the same `(contract_address, selector, calldata)`.
3. Submit multiple outer invoke transactions calling that contract.

No privileged access, leaked key, or social engineering beyond obtaining the original signature is required. The original signature is visible on-chain after the first use.

---

### Recommendation

Include a per-use nonce in the meta-tx-v0 hash preimage. Pass it as `additional_data` in `deprecated_get_transaction_hash` (mirroring how `compute_l1_handler_transaction_hash` passes the L1-handler nonce), and enforce that the target contract's nonce matches and is incremented in `execute_meta_tx_v0`, removing the version-0 exemption in `check_and_increment_nonce` for this path. [5](#0-4) 

---

### Proof of Concept

1. Alice signs a meta-tx-v0 hash for `(victim_account, __execute__, [transfer(attacker, 1000_STRK)], chain_id)`. The hash is `H = pedersen(INVOKE_HASH_PREFIX, 0, victim_account, __execute__, calldata_hash, 0, chain_id)`.
2. Attacker deploys `ReplayContract` whose body issues `meta_tx_v0(contract_address=victim_account, selector=__execute__, calldata=[transfer(attacker,1000_STRK)], signature=Alice_sig)`.
3. Attacker submits outer invoke tx #1 → `ReplayContract.__execute__` → `meta_tx_v0` syscall → OS calls `compute_meta_tx_v0_hash` → produces the same `H` → victim's `__execute__` sees `tx_info.transaction_hash = H`, validates Alice's signature, executes the transfer. Victim's nonce is **not** incremented (version 0 exemption).
4. Attacker submits outer invoke tx #2 (different nonce on attacker's account, same calldata to `ReplayContract`) → identical `meta_tx_v0` syscall → same `H` → same Alice signature accepted again → second transfer executes.
5. Steps 3–4 repeat until the victim account is drained. [6](#0-5) [7](#0-6)
