### Title
Missing Declared Class Validation in `execute_replace_class` Allows Permanent Contract Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary class hash as the replacement without verifying that the hash corresponds to a previously declared class. An unprivileged contract deployer can exploit this to permanently freeze funds held in any contract by replacing its class with an undeclared hash, after which no valid OS proof can be generated for any transaction targeting that contract.

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall. It reads the requested new `class_hash` from the syscall request and directly updates `contract_state_changes` with it, with an explicit TODO acknowledging the missing validation:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

No check is performed against `contract_class_changes` (the declared class registry) to confirm that `class_hash` has a corresponding compiled class hash entry.

When a subsequent transaction calls the affected contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,  // resolves to 0 for undeclared hash
);
``` [2](#0-1) 

`dict_read` returns `0` for an undeclared class hash. `find_element` with key `0` then fails with a Cairo assertion error (since `0` is not a valid compiled class fact hash), making it impossible to generate a valid proof for any block containing a call to the affected contract. The sequencer is forced to permanently exclude all transactions targeting that contract.

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any funds (tokens, NFTs, or other assets) held in a contract whose class has been replaced with an undeclared hash become permanently inaccessible. No transaction can ever successfully execute against that contract again, because the OS proof generation will always fail at the `find_element` call. The state is irrecoverable without a protocol-level upgrade.

### Likelihood Explanation

The attack is reachable by any unprivileged actor in two realistic scenarios:

1. **Malicious deployer**: A deployer creates a contract that accepts user deposits, accumulates funds, then calls `replace_class` with an arbitrary undeclared hash to freeze all deposited assets.
2. **Exploit of a vulnerable contract**: Any contract that exposes a function allowing user-controlled input to flow into a `replace_class` syscall (e.g., an upgradeable contract with insufficient access control) can be exploited by any transaction sender.

The `replace_class` syscall is a standard, gas-metered operation available to all contracts. No privileged role is required to invoke it.

### Recommendation

Before updating `contract_state_changes` in `execute_replace_class`, verify that the requested `class_hash` has a non-zero entry in `contract_class_changes` (i.e., it has been declared and has a valid compiled class hash). This is already acknowledged by the existing TODO comment. The check should assert:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the enforcement already present for `execute_declare_transaction`, which asserts `assert_not_zero(compiled_class_hash)` before writing to `contract_class_changes`. [3](#0-2) 

### Proof of Concept

1. Attacker deploys contract `C` (class hash `H_valid`, compiled class hash `CH_valid` — both declared).
2. Users deposit funds into `C`.
3. Attacker sends an invoke transaction to `C` that internally calls the `replace_class` syscall with `class_hash = 0xDEAD` (an undeclared felt).
4. The OS processes this: `execute_replace_class` writes `StateEntry(class_hash=0xDEAD, ...)` for contract `C` into `contract_state_changes` with no validation. The block is proven successfully.
5. Any subsequent transaction targeting `C` causes `execute_entry_point` to call `dict_read(key=0xDEAD)` on `contract_class_changes`, returning `0`. Then `find_element(..., key=0)` fails — no valid proof can be generated.
6. All funds in `C` are permanently frozen. [4](#0-3) [5](#0-4)

    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.

    );

        key=contract_address,
