### Title
Unsigned `ContractCodeResponse` (V1) Allows Any Peer to Inject Arbitrary Contract Bytes into Witness Reconstruction Cache — (`chain/client/src/stateless_validation/validate.rs`)

### Summary

`validate_contract_code_response` skips signature verification when `ProtocolFeature::SignedContractCodeResponse` is not yet enabled. Any network peer can send a `ContractCodeResponseV1` carrying arbitrary contract bytes under any valid `ChunkProductionKey`. Those bytes are stored unconditionally in the `PartialEncodedStateWitnessTracker` cache and later merged into the witness's `base_state` before re-execution. The re-executed state transition produces a wrong state root, witness validation fails, and the targeted validator never emits a chunk endorsement.

### Finding Description

**Two code paths — one with authentication, one without:**

`validate_contract_code_response` conditionally calls `validate_witness_contract_code_response_signature` only when `SignedContractCodeResponse` is enabled: [1](#0-0) 

When the feature is off, the only gate is `validate_chunk_relevant` — a window check on `height_created` and `epoch_id` that any peer can satisfy by reading the chain head from the network. No sender identity is verified.

**Attacker-controlled data flows into the reconstruction cache:**

`handle_contract_code_response` trusts the key and bytes from the response directly: [2](#0-1) 

`store_accessed_contract_codes` stores the bytes under the attacker-supplied `ChunkProductionKey` with no hash verification against the hashes that were originally requested: [3](#0-2) 

**Injected bytes corrupt witness base_state:**

When enough witness parts arrive and the witness is decoded, the stored contract bytes are merged into `witness.mut_main_state_transition().base_state`: [4](#0-3) 

The witness validator then re-executes the state transition using these wrong bytes. The resulting state root does not match `state_witness.main_state_transition().post_state_root`, so `validate_chunk_state_witness_impl` returns an error: [5](#0-4) 

The validator never produces a `ChunkEndorsement` for that chunk.

**Contrast with the authenticated path:**

`ContractCodeResponseV2` carries a signature from the responder, and `validate_witness_contract_code_response_signature` verifies that the responder is a chunk producer for the correct shard/epoch and that the signature is valid: [6](#0-5) 

The V1 path has no equivalent check — exactly the same asymmetry as PythOracle's `commit()` vs `commitRequested()`.

**The unsigned V1 struct carries no sender binding:** [7](#0-6) 

`ContractCodeResponseV1` contains only `next_chunk` (fully attacker-controlled) and `compressed_contracts` (arbitrary bytes). There is no `signature` field and no `responder` field.

### Impact Explanation

A single malicious peer can race the legitimate chunk producer's response and inject wrong contract bytes for any `ChunkProductionKey` that is within the relevance window. The targeted validator's witness reconstruction produces a wrong state root, validation fails, and the validator misses the endorsement deadline for that chunk. If the attacker targets a sufficient fraction of the chunk validator set simultaneously, the chunk accumulates fewer endorsements than the required threshold, the block producer cannot include it, and block production stalls for that shard.

### Likelihood Explanation

The attack requires only a network peer relationship with the target validator — no stake, no validator role, no privileged access. The attacker must win a race against the legitimate chunk producer's response, which is feasible given that the validator sends the `ContractCodeRequest` to a randomly chosen chunk producer and the attacker can observe the request on the network. The attack is repeatable every chunk interval.

### Recommendation

1. **Always verify the response signature**, regardless of protocol version. The `SignedContractCodeResponse` gate should be removed; V1 should be retired.
2. **Verify received contract bytes against the requested hashes** inside `handle_contract_code_response` before storing: compute `CodeHash(hash(&bytes))` for each received `CodeBytes` and reject any response whose hashes do not match the set from the originating `ContractCodeRequest`.

### Proof of Concept

1. Validator V sends `ContractCodeRequest` for `ChunkProductionKey { shard_id=0, epoch_id=E, height_created=H }` with contract hash `h`.
2. Attacker A (any peer) sends `ContractCodeResponseV1 { next_chunk: { shard_id=0, epoch_id=E, height_created=H }, compressed_contracts: <garbage bytes> }` to V before the legitimate producer responds.
3. `validate_contract_code_response` passes (height H is within the relevance window, epoch E is plausible).
4. `store_accessed_contract_codes` stores the garbage bytes under key `(shard_id=0, epoch_id=E, height_created=H)`.
5. When witness parts for that key are decoded, garbage bytes are merged into `base_state`.
6. `validate_chunk_state_witness_impl` re-executes the state transition, produces a wrong state root, returns `Err(InvalidChunkStateWitness(...))`.
7. V emits no `ChunkEndorsement` for chunk `(shard_id=0, height_created=H)`. [1](#0-0) [8](#0-7) [2](#0-1) [9](#0-8) [5](#0-4)

fn validate_witness_contract_code_response_signature(
    epoch_manager: &dyn EpochManagerAdapter,
    response: &ContractCodeResponse,
) -> Result<(), Error> {
    let Some(responder) = response.responder() else {
        return Err(Error::Other(
            "Unsigned contract code response in epoch where signature is required".to_owned(),
        ));
    };
    let key = response.chunk_production_key();
    let chunk_producers =
        epoch_manager.get_epoch_chunk_producers_for_shard(&key.epoch_id, key.shard_id)?;
    if !chunk_producers.contains(responder) {
        return Err(Error::Other(format!(
            "Contract code response responder {responder} is not a chunk producer for shard {} in epoch {:?}",
            key.shard_id, key.epoch_id,
        )));
    }
    let validator = epoch_manager.get_validator_by_account_id(&key.epoch_id, responder)?;
    if !response.verify_signature(validator.public_key()) {
        return Err(Error::Other("Invalid witness contract code response signature".to_owned()));
    }
    Ok(())
}

pub enum ContractCodeResponse {
    V1(ContractCodeResponseV1) = 0,
    V2(ContractCodeResponseV2) = 1,
}

impl ContractCodeResponse {
    pub fn encode(
        next_chunk: ChunkProductionKey,
        contracts: &Vec<CodeBytes>,
        signer: &ValidatorSigner,
        protocol_version: ProtocolVersion,
    ) -> std::io::Result<Self> {
        if ProtocolFeature::SignedContractCodeResponse.enabled(protocol_version) {
            ContractCodeResponseV2::encode(next_chunk, contracts, signer).map(Self::V2)
        } else {
            ContractCodeResponseV1::encode(next_chunk, contracts).map(Self::V1)
        }
    }

    pub fn chunk_production_key(&self) -> &ChunkProductionKey {
        match self {
            Self::V1(v1) => &v1.next_chunk,
            Self::V2(v2) => &v2.inner.next_chunk,
        }
    }

    /// Account that produced this response. Available only for signed variants.
    pub fn responder(&self) -> Option<&AccountId> {
        match self {
            Self::V1(_) => None,
            Self::V2(v2) => Some(&v2.inner.responder),
        }
    }

    pub fn decompress_contracts(&self) -> std::io::Result<Vec<CodeBytes>> {
        let compressed_contracts = match self {
            Self::V1(v1) => &v1.compressed_contracts,
            Self::V2(v2) => &v2.inner.compressed_contracts,
        };
        compressed_contracts.decode().map(|(data, _size)| data)
    }

    /// Verifies the signature for signed variants. Returns `false` for
    /// unsigned variants since there is nothing to verify.
    pub fn verify_signature(&self, public_key: &PublicKey) -> bool {
        match self {
            Self::V1(_) => false,
            Self::V2(v2) => v2.verify_signature(public_key),
        }
    }
}

pub struct ContractCodeResponseV1 {
    // The same as `next_chunk` in `ContractCodeRequest`
    next_chunk: ChunkProductionKey,
    /// Code for the contracts.

        next_chunk: ChunkProductionKey,
        contracts: &Vec<CodeBytes>,
    ) -> std::io::Result<Self> {

        Ok(Self { next_chunk, compressed_contracts })
    }
