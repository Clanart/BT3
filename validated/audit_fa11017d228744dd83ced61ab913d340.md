### Title
Unimplemented `get_compiled_class_hash` in `SyncStateReader` Causes Gateway Panic on Declare Transactions - (File: `crates/apollo_gateway/src/sync_state_reader.rs`)

### Summary

The `get_compiled_class_hash` method in `SyncStateReader` — the gateway's live state reader backed by the state-sync client — is implemented as `todo!()`. Any gateway stateful-validation path that causes the blockifier to call this method will panic, causing the gateway to reject the triggering transaction with an internal error rather than a proper validation result.

### Finding Description

`SyncStateReader` implements the `BlockifierStateReader` trait for the gateway's stateful transaction validator. All other methods (`get_storage_at`, `get_nonce_at`, `get_class_hash_at`, `get_compiled_class`) delegate to the state-sync client. The `get_compiled_class_hash` method, however, is left as `todo!()`:

```rust
fn get_compiled_class_hash(&self, _class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    todo!()
}
``` [1](#0-0) 

The blockifier calls `get_compiled_class_hash` during stateful validation of `DeclareV2`/`DeclareV3` transactions to verify that the `compiled_class_hash` field in the transaction matches the value already stored in state (re-declaration guard). Because `SyncStateReader` is the concrete reader injected into the gateway's stateful validator via `SyncStateReaderFactory`, any `Declare` transaction that reaches this check will trigger the `todo!()` panic. [2](#0-1) 

The `GenesisStateReader` (used only when no blocks exist yet) does implement `get_compiled_class_hash` by returning `CompiledClassHash::default()`, but `SyncStateReader` — used for all normal operation — does not. [3](#0-2) 

This is the direct sequencer analog of the `getAmountsOut` stub: a function that is part of a required interface, is called on a live code path, and returns no real value — here it panics instead of returning `0`, but the effect is the same: the caller cannot obtain the authoritative value it needs.

### Impact Explanation

A `todo!()` panic in a synchronous blockifier call unwinds through the spawned blocking task. The tokio runtime catches the panic and surfaces it as an internal server error to the gateway, which rejects the transaction. A valid `DeclareV2`/`DeclareV3` transaction is therefore permanently rejected at the gateway admission stage — it never reaches the mempool or the sequencer. This matches the impact:

> **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

### Likelihood Explanation

Any user who submits a `DeclareV2` or `DeclareV3` transaction through the gateway (the standard path for deploying new Cairo 1 contracts) will trigger this code path. The trigger requires no special privilege — it is a normal, unprivileged RPC call (`starknet_addDeclareTransaction`). The only precondition is that at least one block exists (so `SyncStateReader` is used rather than `GenesisStateReader`).

### Recommendation

Implement `get_compiled_class_hash` in `SyncStateReader` by delegating to the state-sync client, analogously to the other methods. The `StateSyncClient` trait already exposes `is_class_declared_at`; a `get_compiled_class_hash` query should be added to the `StateSyncClient` interface and forwarded to storage, or the compiled class hash should be retrieved from the class manager. [4](#0-3) 

### Proof of Concept

1. Start a sequencer node with at least one committed block (so `SyncStateReader` is active).
2. Submit a `starknet_addDeclareTransaction` RPC call with a valid `DeclareV2` or `DeclareV3` transaction body.
3. The gateway's stateful validator constructs a `SyncStateReader` and passes it to the blockifier.
4. The blockifier calls `get_compiled_class_hash` on `SyncStateReader` to check the re-declaration guard.
5. `todo!()` panics; the tokio blocking task unwinds; the gateway returns an internal error.
6. The valid declare transaction is rejected and never enters the mempool.

        &self,

/// Use any of these factory methods only once per transaction to make sure metrics are accurate.
#[async_trait]
impl StateReaderFactory for SyncStateReaderFactory {
    type TGatewayStateReaderWithCompiledClasses = SyncOrGenesisStateReader;
    type TGatewayFixedBlockStateReader = SyncOrGenesisFixedBlockStateReader;

    // TODO(guy.f): The call to `get_latest_block_number()` is not counted in the storage metrics as
    // it is done prior to the creation of SharedStateSyncClientMetricWrapper, directly via the
    // SharedStateSyncClient.
    async fn get_blockifier_state_reader_and_gateway_fixed_block_from_latest_block(
        &self,
    ) -> StateSyncClientResult<(
        Self::TGatewayStateReaderWithCompiledClasses,
        Self::TGatewayFixedBlockStateReader,
    )> {
        let latest_block_number = self.shared_state_sync_client.get_latest_block_number().await?;

        // If no blocks exist yet, return genesis state readers for bootstrap transactions.
        let Some(latest_block_number) = latest_block_number else {
            info!("No blocks found yet; using genesis state readers for bootstrap transactions.");
            return Ok((GenesisStateReader.into(), GenesisFixedBlockStateReader.into()));
        };

        let blockifier_state_reader = SyncStateReader::from_number(
            self.shared_state_sync_client.clone(),
            self.class_manager_client.clone(),
            latest_block_number,
            self.runtime.clone(),
        );
        let gateway_fixed_block_sync_state_client = GatewayFixedBlockSyncStateClient::new(
            self.shared_state_sync_client.clone(),
            latest_block_number,
        );
        Ok((blockifier_state_reader.into(), gateway_fixed_block_sync_state_client.into()))
    }
