### Title
`ProofOfInclusion::valid()` Does Not Verify Against a Trusted Root — Forged Proofs Always Pass — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

---

### Summary

The `ProofOfInclusion::valid()` method in the DataLayer Merkle implementation contains a tautological final check that makes the function verify only internal self-consistency of the proof structure, never binding it to any external trusted root hash. An attacker can construct a fully forged `ProofOfInclusion` for any arbitrary `node_hash` (key-value pair) and `valid()` will return `true`, allowing untrusted input to prove invalid DataLayer state.

---

### Finding Description

`ProofOfInclusion::valid()` is implemented as follows:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // <-- returns the last layer's combined_hash
    } else {
        self.node_hash
    }
}

pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;   // <-- existing_hash == layer.combined_hash
    }

    existing_hash == self.root_hash()      // <-- TAUTOLOGY
}
```

After the loop body, `existing_hash` is set to `calculated_hash`, which was just verified to equal `layer.combined_hash`. The final check `existing_hash == self.root_hash()` compares `existing_hash` (which equals the last `layer.combined_hash`) against `self.root_hash()` (which also returns `last.combined_hash`). **These two values are always identical by construction**, making the final check unconditionally true whenever the loop completes.

The result: `valid()` only checks that each layer's `combined_hash` is correctly computed from the previous hash and `other_hash` — it never verifies the proof against any external, trusted root. The function accepts any internally self-consistent proof regardless of whether it corresponds to the actual tree.

The `ProofOfInclusion` struct is `Streamable` and fully deserializable from bytes via `from_bytes` / `parse_rust`, and is exposed to Python via `py_valid()` and `py_root_hash()`. A caller receiving a proof from an untrusted source and calling only `proof.valid()` — as the API name implies is sufficient — will be deceived. [1](#0-0)

### Impact Explanation

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion... lets untrusted input prove invalid state."**

An attacker who can deliver a crafted `ProofOfInclusion` to a verifier (e.g., over the network, via a DataLayer delta file, or through the Python/Rust API) can:

1. Claim any arbitrary key-value pair is present in a DataLayer store.
2. Have `valid()` return `true` for that forged claim.
3. Cause the verifier to accept false DataLayer state as proven.

The `ProofOfInclusion` is `Streamable` (deserializable from raw bytes), so the attacker-controlled entry path is any code path that deserializes a `ProofOfInclusion` from an untrusted source and calls `valid()` without separately comparing `proof.root_hash()` against a known trusted root. [2](#0-1) 

The Python binding exposes `valid()` as the sole verification method with no required root parameter, making misuse the natural usage pattern: [3](#0-2)

### Recommendation

`valid()` must accept a `trusted_root: Hash` parameter and compare the computed root against it:

```rust
pub fn valid(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }

    &existing_hash == trusted_root   // compare against external trusted root
}
```

The Python binding `py_valid()` should be updated to require the trusted root as an argument. All callers must supply the root hash obtained from a trusted source (e.g., the on-chain committed root, not from the proof itself). [7](#0-6)

### Proof of Concept

```rust
use chia_datalayer::{Hash, Side};
use chia_datalayer::merkle::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};

fn forge_proof(fake_node_hash: Hash, fake_sibling: Hash) -> ProofOfInclusion {
    // Compute a fake combined_hash from attacker-chosen inputs
    let fake_combined = chia_datalayer::calculate_internal_hash(
        &fake_node_hash,
        Side::Left,
        &fake_sibling,
    );

    ProofOfInclusion {
        node_hash: fake_node_hash,
        layers: vec![ProofOfInclusionLayer {
            other_hash_side: Side::Left,
            other_hash: fake_sibling,
            combined_hash: fake_combined,  // internally consistent
        }],
    }
}

fn main() {
    let fake_node_hash = [0xAA; 32];
    let fake_sibling   = [0xBB; 32];
    let proof = forge_proof(fake_node_hash, fake_sibling);

    // valid() returns true for a completely forged proof
    assert!(proof.valid());
    // root_hash() is attacker-controlled, not the real tree root
    println!("Forged root: {:?}", proof.root_hash());
}
```

The forged proof passes `valid()` with no connection to any real `MerkleBlob` or committed root hash. [7](#0-6) [8](#0-7)

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
