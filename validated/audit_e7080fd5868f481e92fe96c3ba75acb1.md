### Title
Detached `swap_migrated_token` Promise Permanently Locks User Tokens on Failure — (`near/omni-bridge/src/lib.rs`)

---

### Summary

When a user sends old (migrated) tokens to the NEAR bridge contract via `ft_transfer_call` with a `SwapMigratedToken` message, the bridge immediately consumes **all** tokens by returning `U128(0)` from `ft_on_transfer`, while the actual swap promise is fired with `.detach()`. If the swap fails for any reason, the user's tokens are irrecoverably locked inside the bridge contract with no automatic refund path and no user-callable recovery function.

---

### Finding Description

In `near/omni-bridge/src/lib.rs`, the `ft_on_transfer` handler for `BridgeOnTransferMsg::SwapMigratedToken` is:

```rust
BridgeOnTransferMsg::SwapMigratedToken => {
    self.swap_migrated_token(sender_id, token_id, amount)
        .detach();
    PromiseOrPromiseIndexOrValue::Value(U128(0))
}
``` [1](#0-0) 

Under NEP-141, returning `U128(0)` from `ft_on_transfer` signals to the token contract that **all transferred tokens were consumed** — the bridge takes full custody of the old tokens immediately. The `.detach()` call means the result of `swap_migrated_token` is never observed by the bridge contract. If the swap promise fails (e.g., the new token contract has no storage registered for the recipient, the new token contract panics, or gas is exhausted), the old tokens remain in the bridge's balance with no automatic refund and no user-callable recovery function.

The only administrative escape hatch is `transfer_token_as_dao`, which is restricted to the `DAO` role:

```rust
#[access_control_any(roles(Role::DAO))]
pub fn transfer_token_as_dao(...)
``` [2](#0-1) 

This means an ordinary user has **zero recourse** without DAO intervention. If `swap_migrated_token` internally burns the old tokens before attempting to mint new ones (a common pattern in migration swaps), even the DAO cannot restore the user's balance — the loss becomes permanent and irrecoverable.

The migration setup is performed by `migrate_deployed_token`, which records the `old_token → new_token` mapping: [3](#0-2) 

---

### Impact Explanation

Any user holding old (migrated) tokens can trigger this path. If the swap fails:

- The user's old tokens are consumed by the bridge (returned `U128(0)` to the token contract's resolver).
- No new tokens are delivered.
- The user has no function to call to recover their funds.
- If the implementation burns old tokens before minting new ones, the loss is **permanent and irrecoverable** even by the DAO.

This matches the allowed Critical impact: **Permanent freezing, irrecoverable lock, or unclaimable settlement of user funds in bridge or token flows.**

---

### Likelihood Explanation

The failure mode is realistic and reachable by any unprivileged token holder:

- **Storage not registered**: If the new token contract does not have a storage account registered for the recipient, `ft_transfer` or `mint` will panic, causing the swap promise to fail.
- **New token contract bug or pause**: Any panic in the new token contract causes the detached promise to silently fail.
- **Gas exhaustion**: The swap involves at least one cross-contract call; if gas is insufficient, the promise fails silently.

The entry path requires no special privileges — any holder of old migrated tokens can trigger it by calling `ft_transfer_call` on the old token contract with `msg = "SwapMigratedToken"`.

---

### Recommendation

1. **Do not return `U128(0)` eagerly.** Instead, chain the swap as a proper callback and return the appropriate refund amount if the swap fails:

   ```rust
   BridgeOnTransferMsg::SwapMigratedToken => {
       self.swap_migrated_token(sender_id, token_id, amount)
           .then(Self::ext(...).swap_migrated_token_callback(amount))
           .into()
   }
   ```

   In `swap_migrated_token_callback`, if the swap failed, return `amount` so the NEP-141 resolver refunds the tokens to the sender.

2. **Alternatively**, ensure `swap_migrated_token` is atomic: only burn old tokens after new tokens are successfully minted, so a failure at any step leaves the old tokens intact in the bridge for DAO recovery.

---

### Proof of Concept

1. DAO calls `migrate_deployed_token(origin_chain, old_token, new_token)` — maps `old_token → new_token`.
2. Alice holds 1000 units of `old_token` and calls:
   ```
   old_token.ft_transfer_call(bridge, 1000, None, "SwapMigratedToken")
   ```
3. `old_token` transfers 1000 units to `bridge` and calls `bridge.ft_on_transfer(alice, 1000, "SwapMigratedToken")`.
4. Bridge returns `U128(0)` — all 1000 tokens are consumed. `swap_migrated_token` is fired with `.detach()`.
5. `swap_migrated_token` fails because `new_token` has no storage registered for Alice (or any other reason).
6. Alice's 1000 old tokens are now in the bridge's balance. Alice has no function to call to recover them. The DAO must manually intervene via `transfer_token_as_dao`, or if old tokens were burned inside the swap, they are gone permanently.

            ext_token::ext(token)
                .with_attached_deposit(ONE_YOCTO)
                .with_static_gas(FT_TRANSFER_CALL_GAS)
                .ft_transfer_call(recipient, amount, None, msg)

                .with_attached_deposit(ONE_YOCTO)
                .with_static_gas(FT_TRANSFER_GAS)

        }
    }

    #[access_control_any(roles(Role::DAO))]
    #[payable]
    pub fn migrate_deployed_token(
        &mut self,
        origin_chain: ChainKind,
        old_token: AccountId,
        new_token: AccountId,
    ) {
        require!(
            env::attached_deposit() >= NEP141_DEPOSIT,
            BridgeError::NotEnoughAttachedDeposit.as_ref()
        );

        require!(
            self.deployed_tokens.remove(&old_token),
            BridgeError::OldTokenNotDeployed.as_ref(),
        );
        require!(
            self.deployed_tokens.insert(&new_token),
            BridgeError::TokenExists.as_ref()
        );
        self.deployed_tokens_v2.remove(&old_token);
        self.deployed_tokens_v2.insert(&new_token, &origin_chain);

        let origin_address = self
            .token_id_to_address
            .remove(&(origin_chain, old_token.clone()))
            .near_expect(BridgeError::FailedToGetTokenAddress);

        require!(
            self.token_id_to_address
                .insert(&(origin_chain, new_token.clone()), &origin_address)
                .is_none(),
            BridgeError::TokenExists.as_ref()
        );

        self.token_address_to_id
            .insert(&origin_address, &new_token)
            .near_expect(BridgeError::ExpectedToOverwriteTokenAddress);

        require!(
            self.migrated_tokens
                .insert(&old_token, &new_token)
                .is_none(),
            BridgeError::TokenAlreadyMigrated.as_ref()
        );

        ext_token::ext(new_token.clone())
            .with_static_gas(STORAGE_DEPOSIT_GAS)
            .with_attached_deposit(NEP141_DEPOSIT)
            .storage_deposit(&env::current_account_id(), Some(true))
            .detach();

        env::log_str(
            &OmniBridgeEvent::MigrateTokenEvent {
                old_token_id: old_token,
                new_token_id: new_token,
            }
            .to_log_string(),
        );
    }
