### Title
Missing shard-identity check in `set_state_header` allows cross-shard state substitution — (`chain/chain/src/state_sync/adapter.rs`)

---

### Summary

`set_state_header` accepts a `ShardStateSyncResponseHeader` whose embedded chunk belongs to shard A and stores it under `StateHeaderKey(shard_id=B, sync_hash)` because it never asserts `chunk.shard_id() == shard_id`. The `verify_path` call confirms the chunk exists somewhere in the block's Merkle tree but does not verify the chunk occupies the tree position that corresponds to shard B. All subsequent part validation and `apply_state_part` then use shard A's `prev_state_root` as the binding root, writing shard A's trie nodes into shard B's `ShardUId` storage.

---

### Finding Description

`set_state_header` in `chain/chain/src/state_sync/adapter.rs` performs five validation steps before persisting the header:

**Step 1-2 — `validate_chunk_proofs`** [1](#0-0) 

This function checks the chunk's internal hash, tx root, and receipts root. It does not compare `chunk.shard_id()` against the caller-supplied `shard_id` parameter. A grep of `adapter.rs` for `chunk.shard_id()` returns zero matches, confirming no such check exists anywhere in `set_state_header`.

**Step 3a — `verify_path` (position-blind)** [2](#0-1) 

`verify_path` is imported from `near_primitives::merkle`: [3](#0-2) 

It only checks that `compute_root_from_path(path, hash(item)) == root`. It does **not** verify that the leaf sits at index `shard_layout.get_shard_index(shard_id_B)`. The position-aware variant `verify_path_with_index` exists but is not called here: [4](#0-3) 

**Step 4 — receipt proof validation uses the parameter `shard_id`, not the chunk's shard_id** [5](#0-4) 

`receipts_hash = CryptoHash::hash_borsh(ReceiptList(shard_id, receipts))` uses the caller-supplied `shard_id` (B). An attacker who populates `incoming_receipts_proofs` with shard B's actual on-chain receipt proofs satisfies this check while the embedded chunk still belongs to shard A.

**Step 5 — state root node validated against shard A's `prev_state_root`** [6](#0-5) 

The node is validated against `chunk_inner.prev_state_root()`, which is shard A's root. An attacker who supplies a valid state root node for shard A passes this check.

**Persistence under the wrong key** [7](#0-6) 

The header is written to `StateHeaderKey(shard_id=B, sync_hash)`.

**Downstream binding in `set_state_part`** [8](#0-7) 

`set_state_part` reads the stored header for `(shard_id=B, sync_hash)`, extracts `state_root = chunk.prev_state_root()` (shard A's root), and validates every incoming part against it. Parts that are valid for shard A pass; they are then written to `StatePartKey(sync_hash, shard_id=B, part_id)` and later applied into shard B's `ShardUId` storage via `apply_state_part`.

---

### Impact Explanation

After `set_state_finalize`, the node's committed `ChunkExtra.state_root` for shard B equals shard A's `prev_state_root`. The node's trie and flat-state storage for shard B's `ShardUId` contain shard A's data. Any subsequent chunk application for shard B reads the wrong state, producing incorrect execution results (wrong account balances, wrong contract storage, wrong receipts). The corrupted node diverges silently from the canonical chain.

---

### Likelihood Explanation

Any peer the syncing node contacts for state sync can trigger this. No validator key, chunk producer key, or operator privilege is required. The attacker only needs to:
1. Be a reachable peer during state sync.
2. Respond to the header request for shard B with a crafted `ShardStateSyncResponseHeader` containing shard A's chunk (with its legitimate Merkle proof) and shard B's actual on-chain receipt proofs (publicly available from any full node).

All required inputs are public blockchain data. No cryptographic forgery is needed.

---

### Recommendation

In `set_state_header`, immediately after extracting the chunk, assert that the chunk's declared shard matches the requested shard:

```rust
// After: let chunk = shard_state_header.cloned_chunk();
if chunk.shard_id() != shard_id {
    byzantine_assert!(false);
    return Err(Error::Other(
        "set_shard_state failed: chunk shard_id does not match requested shard_id".into(),
    ));
}
```

Additionally, replace the position-blind `verify_path` call with `verify_path_with_index`, passing `shard_layout.get_shard_index(shard_id)?` as the expected leaf index, so the Merkle proof is bound to the correct position in the chunk headers tree. [2](#0-1)

8. Call set_state_finalize(shard_B, sync_hash).

   chunk_extra = client[1].chain.get_chunk_extra(sync_hash, shard_B_uid)

pub fn validate_chunk_proofs(
    chunk: &ShardChunk,
    epoch_manager: &dyn EpochManagerAdapter,
) -> Result<bool, Error> {
    let correct_chunk_hash = chunk.compute_header_hash();

    // 1. Checking chunk.header.hash
    let header_hash = chunk.header_hash();
    if header_hash != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }

    // 2. Checking that chunk body is valid
    // 2a. Checking chunk hash
    if chunk.chunk_hash() != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }
    let height_created = chunk.height_created();
    let outgoing_receipts_root = chunk.prev_outgoing_receipts_root();
    let (transactions, receipts) = (chunk.to_transactions(), chunk.prev_outgoing_receipts());

    // 2b. Checking that chunk transactions are valid
    let (tx_root, _) = merklize(transactions);
    if &tx_root != chunk.tx_root() {
        byzantine_assert!(false);
        return Ok(false);
    }
    // 2c. Checking that chunk receipts are valid
    if height_created == 0 {
        return Ok(receipts.is_empty() && outgoing_receipts_root == &CryptoHash::default());
    } else {
        let shard_layout = {
            let prev_block_hash = chunk.prev_block_hash();
            epoch_manager.get_shard_layout_from_prev_block(&prev_block_hash)?
        };
        let outgoing_receipts_hashes = Chain::build_receipts_hashes(receipts, &shard_layout)?;
        let (receipts_root, _) = merklize(&outgoing_receipts_hashes);
        if &receipts_root != outgoing_receipts_root {
            byzantine_assert!(false);
            return Ok(false);
        }
    }
    Ok(true)
}
