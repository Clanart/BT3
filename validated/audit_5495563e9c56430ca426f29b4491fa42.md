Audit Report

## Title
`AnchoredPriceProvider._isStale()` Hard-Rejects Future `refTime` Without L2 Tolerance, Causing Swap DoS on L2 Deployments — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

## Summary

`AnchoredPriceProvider`, described as "the one standard provider for public pools," contains a staleness helper that unconditionally rejects any oracle `refTime` exceeding `block.timestamp` by even one second. The codebase ships `PriceProviderL2` and `ProtectedPriceProviderL2` with a configurable `FUTURE_TOLERANCE` to handle L2 sequencer clock skew, but no equivalent L2 variant exists for `AnchoredPriceProvider`. The root-level `DeployAnchorFactory.s.sol` script explicitly targets L2 chains (its usage example shows `NETWORK=base`), making L2 deployment the expected path. On any L2 where oracle `refTime` exceeds `block.timestamp`, every swap on every pool backed by this provider reverts with `FeedStalled`.

## Finding Description

`AnchoredPriceProvider._isStale()` at lines 222–230 hard-rejects any future `refTime`:

```solidity
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta)
    internal pure returns (bool)
{
    if (refTime == 0) return true;
    if (refTime > nowTs) return true;   // ← unconditional hard reject
    return (nowTs - refTime) > maxDelta;
}
``` [1](#0-0) 

`_readLeg()` propagates `ok = false` when `_isStale` returns `true`: [2](#0-1) 

`_getBidAndAskPrice()` returns the sentinel `(0, type(uint128).max)` on `!ok`: [3](#0-2) 

`getBidAndAskPrice()` then reverts with `FeedStalled`: [4](#0-3) 

The L2-aware siblings fix this with a `FUTURE_TOLERANCE` window. `PriceProviderL2._isStale()` tolerates small forward skew:

```solidity
if (refTime > nowTs) {
    return (refTime - nowTs) > futureTol;   // tolerate small skew
}
``` [5](#0-4) 

`ProtectedPriceProviderL2` applies the identical fix: [6](#0-5) 

Both L2 variants enforce a bounded `FUTURE_TOLERANCE` at construction: [7](#0-6) 

No `AnchoredPriceProviderL2` exists in the codebase. The deployment script `DeployAnchorFactory.s.sol` lives in the root `script/` directory (not under `script/l1/`) and its inline usage example explicitly targets `NETWORK=base` — an L2 chain — confirming cross-chain deployment is the intended path: [8](#0-7) 

## Impact Explanation

On L2 chains (Base, Arbitrum, etc.), the sequencer's internal clock routinely runs ahead of the last L1-anchored timestamp. Any Pyth price update published with `refTime` even one second ahead of the L2 `block.timestamp` causes `_isStale` to return `true`, propagating `ok = false` through `_readLeg` → `_getBidAndAskPrice` → `getBidAndAskPrice` → `FeedStalled` revert. Since the pool calls `getBidAndAskPrice()` on every swap, this is a complete, persistent DoS of core swap functionality for the duration of the clock skew — matching the allowed impact of "broken core pool functionality causing unusable swap/liquidity flows." LPs cannot rebalance, traders cannot execute, and protocol fees are not collected.

## Likelihood Explanation

L2 sequencer clock skew is a documented, routine occurrence requiring no attacker action. Pyth pushes price updates continuously; any update whose `refTime` exceeds the L2 `block.timestamp` by even one second triggers the revert. The deployment script's own example (`NETWORK=base`) confirms L2 deployment is the expected use case. The condition is triggered by normal oracle operation, not adversarial input, and repeats for the full duration of any skew episode.

## Recommendation

Add a `FUTURE_TOLERANCE` immutable to `AnchoredPriceProvider` (or create an `AnchoredPriceProviderL2` variant) and update `_isStale` to mirror the L2-aware logic already present in `PriceProviderL2`:

```solidity
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta, uint256 futureTol)
    internal pure returns (bool)
{
    if (refTime == 0) return true;
    if (refTime > nowTs) return (refTime - nowTs) > futureTol;
    return (nowTs - refTime) > maxDelta;
}
```

Cap `futureTol` at construction (e.g., `≤ 1 hours`) as `PriceProviderL2` does:

```solidity
if (_futureTolerance > 1 hours) revert FutureToleranceOutOfBounds();
FUTURE_TOLERANCE = _futureTolerance;
```

## Proof of Concept

1. Deploy `AnchoredProviderFactory` on Base using `DeployAnchorFactory.s.sol` with `NETWORK=base REFERENCE=pyth`.
2. Create a pool backed by the resulting `AnchoredPriceProvider` (e.g., WETH/USDC).
3. Pyth publishes a price update with `refTime = block.timestamp + 2` (routine on Base when the sequencer clock is slightly ahead).
4. Any user calls `exactInputSingle` through `MetricOmmSimpleRouter`.
5. The pool calls `priceProvider.getBidAndAskPrice()`.
6. `_readLeg` calls `_isStale(refTime=T+2, nowTs=T, MAX_REF_STALENESS)`.
7. `refTime > nowTs` → `_isStale` returns `true` → `ok = false` → sentinel `(0, type(uint128).max)` returned by `_getBidAndAskPrice`.
8. `getBidAndAskPrice` reverts with `FeedStalled`.
9. The swap reverts. All swaps on this pool are blocked for the duration of the skew.

Foundry fork test: fork Base mainnet, deploy the factory and provider, mock a Pyth update with `refTime = block.timestamp + 1`, call `getBidAndAskPrice()`, and assert revert with `FeedStalled`.
