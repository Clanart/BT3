### Title
`MerkleBlob::batch_insert` Skips Duplicate-Key Validation, Corrupting Tree Root and Enabling Forged Inclusion Proofs — (`File: crates/chia-datalayer/src/merkle/blob.rs`)

### Summary

`MerkleBlob::batch_insert` bypasses the duplicate-key and duplicate-hash guards that `MerkleBlob::insert` enforces for all items beyond the first two. Supplying a batch that contains a repeated `KeyId` silently writes two leaf nodes with the same key into the blob at distinct tree indices, producing a structurally invalid Merkle tree whose root hash is computed over phantom duplicate state. Any proof of inclusion generated from this corrupted tree is forged relative to the true committed key-value set.

### Finding Description

`MerkleBlob::insert` (the single-item path) opens with two explicit guards:

```rust
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`batch_insert` calls `self.insert` (with those guards) only for the first 0–2 items when the tree has ≤ 1 existing leaf:

```rust
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else { ... };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;
    }
}
``` [2](#0-1) 

All remaining items in the vector are written directly via `insert_entry_to_blob` with **no duplicate check**:

```rust
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ... Node::Leaf(LeafNode { hash, key, value, ... }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
``` [3](#0-2) 

`insert_entry_to_blob` calls `self.block_status_cache.add_leaf(index, leaf)` for every leaf it writes:

```rust
match block.node {
    Node::Leaf(leaf) => self.block_status_cache.add_leaf(index, leaf),
    Node::Internal(..) => self.block_status_cache.add_internal(index),
}
``` [4](#0-3) 

`add_leaf` inserts into `key_to_index` (a `HashMap`). When a duplicate key appears, the HashMap silently overwrites the first entry, so the cache ends up pointing only to the *last* leaf index for that key — but the blob contains **two** leaf nodes with the same key at different indices. The tree-building loop then constructs internal nodes referencing both indices and computes their hashes, producing a root hash that encodes phantom duplicate state.

`check_integrity` would detect the mismatch (`leaf_count` from blob iteration > `key_to_index_cache_length`), but it is not called automatically after `batch_insert`. [5](#0-4) 

The Python binding `py_batch_insert` exposes this path directly to callers:

```rust
#[pyo3(name = "batch_insert")]
pub fn py_batch_insert(&mut self, keys_values: Vec<(KeyId, ValueId)>, hashes: Vec<Hash>) -> PyResult<()> {
    ...
    self.batch_insert(zip(keys_values, hashes).collect())?;
    Ok(())
}
``` [6](#0-5) 

### Impact Explanation

The corrupted root hash is committed on-chain as the authoritative DataLayer state root. Because the blob contains two leaf nodes for the same key, `get_proof_of_inclusion` generates a proof path to whichever leaf the cache happens to point to, while the other phantom leaf silently inflates the root hash. This lets the store owner:

1. **Forge inclusion proofs**: produce a proof for a key-value pair that was never legitimately inserted (the phantom duplicate leaf contributes to the root hash, so a proof path through it verifies against the committed root).
2. **Corrupt tree roots permanently**: the committed root no longer corresponds to any valid key-value set, breaking all downstream verifiers that rely on the DataLayer Merkle root for state integrity.

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

### Likelihood Explanation

`py_batch_insert` is a public Python API callable by any DataLayer store operator or any code path that drives bulk DataLayer updates. No privileged keys or special roles are required — only the ability to supply a list of `(KeyId, ValueId)` pairs with a repeated `KeyId`. The bug is triggered deterministically with a batch of ≥ 3 items (so the fast-path `insert` calls are exhausted) where at least two share the same key.

### Recommendation

Add duplicate-key and duplicate-hash validation at the start of `batch_insert`, mirroring the guards in `insert`:

```rust
pub fn batch_insert(&mut self, keys_values_hashes: Vec<((KeyId, ValueId), Hash)>) -> Result<(), Error> {
    // Validate no duplicates within the batch or against existing keys
    for ((key, _), hash) in &keys_values_hashes {
        if self.block_status_cache.contains_key(*key) {
            return Err(Error::KeyAlreadyPresent());
        }
        if self.block_status_cache.contains_leaf_hash(hash) {
            return Err(Error::HashAlreadyPresent());
        }
    }
    // Also check for intra-batch duplicates using a local HashSet
    ...
}
```

Alternatively, route all items through `self.insert` (accepting the performance cost) or call `check_integrity` after every `batch_insert` to at least detect corruption before it propagates.

### Proof of Concept

```python
from chia_rs import MerkleBlob, KeyId, ValueId
import hashlib

blob = MerkleBlob(bytearray())

# Build a batch with 3 items where key_0 appears twice (indices 0 and 2)
# The first two items go through self.insert() (guarded), the third bypasses it
key_dup = KeyId(0)
key_b   = KeyId(1)

hash_a = bytes(hashlib.sha256(b"a").digest())
hash_b = bytes(hashlib.sha256(b"b").digest())
hash_c = bytes(hashlib.sha256(b"c").digest())  # duplicate key, different hash

# batch_insert pops from the END, so the last two items are processed first via insert()
# item order: [(key_dup,0,hash_a), (key_b,1,hash_b), (key_dup,2,hash_c)]
# pop() takes hash_c then hash_b through guarded insert(); hash_a goes through unguarded path
blob.batch_insert(
    [(key_dup, ValueId(0)), (key_b, ValueId(1)), (key_dup, ValueId(2))],
    [hash_a, hash_b, hash_c],
)

# Tree now has two leaf nodes for key_dup at different indices.
# Root hash is computed over phantom duplicate state.
# check_integrity() will raise IntegrityKeyToIndexCacheLength.
blob.calculate_lazy_hashes()
root = blob.get_root_hash()
print("Corrupted root:", root)  # non-None, but encodes duplicate state

# Proof generated here is forged relative to the true key-value set
proof = blob.get_proof_of_inclusion(key_dup)
print("Proof valid against corrupted root:", proof.valid())
```
