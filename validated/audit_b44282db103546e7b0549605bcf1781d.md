### Title
Double-Mint via Legacy Path After `mint()` Uses Fake Receipt ID — (`evm/src/eNear/contracts/ENearProxy.sol`)

---

### Summary

`ENearProxy.mint()` constructs a fake Borsh proof embedding an internal sequential counter (`currentReceiptId`) as the receipt ID — not the real NEAR receipt ID. Because `ENearProxy.finaliseNearToEthTransfer()` is unpaused by default and carries no role restriction, an unprivileged relayer can submit the real NEAR proof for the same transfer. The real proof carries a different receipt ID, which eNear's `usedEvents` has never seen, so eNear processes it a second time and mints a duplicate amount.

---

### Finding Description

**Deployment context (from README):**

The intended production setup is:
- ENearProxy becomes eNear's admin via `adminSstore`
- eNear's internal prover is replaced with `FakeProver` (always returns `true`)
- eNear's own `finaliseNearToEthTransfer` is paused for non-admin callers; only ENearProxy (as admin) can call it [1](#0-0) 

**`mint()` path — fake receipt ID:**

When the MINTER_ROLE holder (the omni-bridge contract) processes a NEAR→ETH transfer, it calls `mint()`, which builds a fake Borsh-encoded proof embedding `currentReceiptId` (an internal `uint256` counter, starting at 0 and incrementing) at the receipt-ID field: [2](#0-1) 

eNear parses this proof, extracts the receipt ID `N = currentReceiptId`, marks `usedEvents[N] = true`, and mints `amount` tokens. `N` is a sequential integer — it has no relationship to the real NEAR receipt hash for transfer T.

**`finaliseNearToEthTransfer()` path — real receipt ID:**

`ENearProxy.finaliseNearToEthTransfer()` is `external` with no role restriction, guarded only by `whenNotPaused(PAUSED_LEGACY_FIN_TRANSFER)`: [3](#0-2) 

`PAUSED_LEGACY_FIN_TRANSFER` is **not set by default** — `initialize()` calls `__Pausable_init()` which initialises the pause bitmask to `0`: [4](#0-3) 

An unprivileged relayer submits the real NEAR proof for transfer T (obtainable from the NEAR blockchain). ENearProxy validates it against the real `NearProver`, then calls `eNear.finaliseNearToEthTransfer(realProofData, blockHeight)`. Because ENearProxy is eNear's admin, eNear's own pause is bypassed. eNear extracts the real receipt ID `M` from the proof, checks `usedEvents[M]` — which is `false` (only `usedEvents[N]` was set by `mint()`) — and mints `amount` tokens again.

**Root cause:** `mint()` uses a synthetic, sequentially-assigned receipt ID (`currentReceiptId`) instead of the real NEAR receipt ID. The two IDs occupy the same field in the Borsh proof but are always different values, so eNear's replay guard never fires on the second call.

---

### Impact Explanation

For every NEAR→ETH transfer processed via `mint()`, an unprivileged relayer can replay the real NEAR proof through the legacy path and mint a second batch of eNear tokens. This creates unbacked eNear supply — a direct double-spend / collateralization break. Impact: **High** (cross-chain duplicate settlement enabling unbacked supply).

---

### Likelihood Explanation

- `PAUSED_LEGACY_FIN_TRANSFER` is `0` after `initialize()` — no explicit admin action is required to leave the legacy path open.
- Real NEAR proofs for any transfer are publicly available from the NEAR blockchain.
- `ENearProxy.finaliseNearToEthTransfer()` requires no role — any EOA can call it.
- The MINTER_ROLE holder (omni-bridge) legitimately calls `mint()` for every NEAR→ETH transfer, creating an exploitable gap for each one.

---

### Recommendation

1. **Pause by default:** In `initialize()`, call `_pause(PAUSED_LEGACY_FIN_TRANSFER)` so the legacy path is closed unless explicitly re-opened by the admin.
2. **Or remove the legacy path entirely** if Rainbow Bridge proofs are no longer accepted in production.
3. **Or track the real receipt ID in `mint()`:** Accept the real NEAR receipt ID as a parameter and pass it in the fake proof so eNear marks the correct ID as used, preventing the real proof from being replayed.

---

```

   Set ENearProxy as eNear's admin; set eNear's prover to FakeProver.
   Do NOT call pauseAll() — PAUSED_LEGACY_FIN_TRANSFER remains 0.

2. MINTER_ROLE calls:
     ENearProxy.mint(eNear, recipient, 1_000e24)
   → fakeProofData embeds currentReceiptId = 0 (N)
   → eNear.usedEvents[0] = true
   → eNear.totalSupply() == 1_000e24

3. Unprivileged relayer obtains the real NEAR proof for transfer T

   → prover.proveOutcome() passes (real NearProver validates real proof)
   → eNear.usedEvents[M] == false  ← replay guard does NOT fire
   → eNear mints 1_000e24 again
   → eNear.totalSupply() == 2_000e24

4. Assert eNear.totalSupply() == 2 * amount  ✓
   Unbacked eNear has been minted.
