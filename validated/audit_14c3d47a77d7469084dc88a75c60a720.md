### Title
Integer Truncation in `NewDynamicFeeChecker` Allows Zero-Fee Cosmos SDK Transactions When `NoBaseFee = true` - (File: ante/evm/fee_checker.go)

### Summary
`NewDynamicFeeChecker` derives a per-gas `feeCap` by integer-dividing the total submitted fee by the gas limit. When `NoBaseFee = true` (London HF still active), the base fee is `0` rather than `nil`, so the code does not fall back to the validator min-gas-price path. Any Cosmos SDK transaction whose total fee is less than its gas limit produces `feeCap = 0`, `effectivePrice = 0`, and `effectiveFee = 0 coins`. The `DeductFeeDecorator` then deducts nothing, even though the `MinGasPriceDecorator` already accepted the transaction.

### Finding Description

In `ante/evm/fee_checker.go`, `NewDynamicFeeChecker` computes the per-gas fee cap with integer (truncating) division:

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))   // line 83 – truncates
```

`GetBaseFee` in `x/evm/types/utils.go` returns `big.Int(0)` — **not** `nil` — when London HF is enabled but `NoBaseFee = true`:

```go
baseFee := feemarketParams.GetBaseFee()   // nil when NoBaseFee=true
if baseFee == nil {
    return new(big.Int)                   // returns 0, not nil
}
```

Because `baseFee` is `0` (not `nil`), the nil-guard at line 57–60 of `fee_checker.go` does **not** redirect to `checkTxFeeWithValidatorMinGasPrices`. Execution continues with `baseFeeInt = 0`.

The subsequent check:

```go
if feeCap.LT(baseFeeInt) { ... }   // 0 < 0 → false, passes
```

passes trivially. Then:

```go
effectivePrice := sdkmath.NewIntFromBigInt(
    types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
// = min(0 + MaxInt64, 0) = 0

effectiveFee := sdk.Coins{{Denom: denom, Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas))}}
// = 0 * gas = 0
```

`DeductFeeDecorator.AnteHandle` overwrites `fee` with the checker's return value and then calls `checkDeductFee`:

```go
fee, priority, err = dfd.txFeeChecker(ctx, tx)   // fee = 0 coins
...
if !fee.IsZero() {                                 // false → skipped
    err := evmkeeper.DeductFees(...)
}
```

Zero fees are deducted.

### Impact Explanation

Any unprivileged user can submit a Cosmos SDK transaction (including EIP-712 signed transactions routed through `newLegacyCosmosAnteHandlerEip712`) with `fee < gas` and pay **zero** fees, regardless of the configured `MinGasPrice`. The `MinGasPriceDecorator` validates the raw submitted fee coins and passes, but `NewDynamicFeeChecker` then computes an effective fee of zero. This is a direct mis-accounting of user fees in the ante handler, matching the allowed High impact: *"ante handler bug that permits valid user funds/fees to be mis-accounted."*

### Likelihood Explanation

**High.** The condition `NoBaseFee = true` with London HF enabled is explicitly documented as a supported configuration ("needed for 0 price calls"). Any chain with `MinGasPrice < 1` (fractional gas price, common for chains using small-denomination tokens like `aevmos` or `aphoton`) is vulnerable: the minimum required fee `ceil(minGasPrice × gas)` is less than `gas`, so `feeCap = fee / gas = 0`. No special privileges are required; any user can craft such a transaction.

### Recommendation

Replace the truncating integer division with a check that rejects any fee submission where `fee < gas` (i.e., where `feeCap` would round to zero below the base fee), or compute `feeCap` using ceiling division:

```go
// Option A: reject if effective feeCap would be zero while baseFee is also zero
if fee.IsZero() && baseFeeInt.IsZero() {
    return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, "fee must be positive")
}

// Option B: ceiling division to avoid truncation to zero
// feeCap = ceil(fee / gas) = (fee + gas - 1) / gas
one := sdkmath.NewIntFromUint64(1)
gasInt := sdkmath.NewIntFromUint64(gas)
feeCap := fee.Add(gasInt).Sub(one).Quo(gasInt)
```

Additionally, add a post-check that `effectiveFee > 0` whenever `MinGasPrice > 0`.

### Proof of Concept

**Chain configuration:**
- London HF enabled
- `NoBaseFee = true` → `GetBaseFee()` returns `big.Int(0)`
- `MinGasPrice = 0.001` (fractional, common for `aphoton`-denominated chains)

**Attack:**
1. Attacker constructs a Cosmos SDK tx with `gas = 21000`, `fee = 21aphoton` (= `ceil(0.001 × 21000)`).
2. `MinGasPriceDecorator`: `requiredFee = ceil(0.001 × 21000) = 21` → `21 >= 21` → **passes**.
3. `NewDynamicFeeChecker`:
   - `feeCap = 21 / 21000 = 0` (integer truncation)
   - `baseFeeInt = 0`
   - `0 < 0` → false → passes fee cap check
   - `effectivePrice = min(0 + MaxInt64, 0) = 0`
   - `effectiveFee = 0 × 21000 = 0 aphoton`
4. `DeductFeeDecorator`: `fee.IsZero()` → **deducts nothing**.
5. Transaction executes with zero fees paid.

The attacker can repeat this for every transaction, consuming chain resources without paying fees. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)
