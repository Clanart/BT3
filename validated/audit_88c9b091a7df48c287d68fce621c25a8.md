### Title
`ProofOfInclusion::valid()` Validates Only Internal Consistency, Not Against a Trusted Root — Forged DataLayer Inclusion Proofs Always Pass - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate contains a tautological final check: it compares `existing_hash` against `self.root_hash()`, but `root_hash()` is derived entirely from the attacker-controlled proof payload (`last.combined_hash`). The function never validates against any externally-supplied trusted root. An attacker can construct a self-consistent but completely fabricated `ProofOfInclusion` for any key-value pair, and `valid()` will return `true`. This is the direct analog of the origin-spoofing pattern in the reference report: the "trusted source" (the on-chain Merkle root) is replaced by a value extracted from the attacker-controlled payload itself.

### Finding Description

In `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`, the `valid()` method is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← derived from attacker-controlled proof data
    } else {
        self.node_hash
    }
}

pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;   // ← always equals layer.combined_hash here
    }
    existing_hash == self.root_hash()      // ← TAUTOLOGY: existing_hash IS last.combined_hash
}
```

After the loop completes without returning `false`, `existing_hash` equals `last.combined_hash` by construction (the loop only continues when `calculated_hash == layer.combined_hash`, then sets `existing_hash = calculated_hash`). `root_hash()` also returns `last.combined_hash`. Therefore the final check `existing_hash == self.root_hash()` is always `true` when the loop passes — it is a tautology.

**Root cause:** `valid()` never accepts a trusted external root as a parameter. It derives the root entirely from the proof payload itself. This is the exact analog of the reference vulnerability: the origin/identity value is read from the attacker-controlled message payload instead of from a trusted out-of-band source.

**Exploit path:**

1. Attacker chooses any target `node_hash` (e.g., the hash of a fabricated key-value pair they want to claim is in the DataLayer store).
2. Attacker constructs a chain of `ProofOfInclusionLayer` structs where each `combined_hash` is computed correctly from the previous hash and a freely chosen `other_hash`. This is trivially possible since `calculate_internal_hash` is a public deterministic function.
3. Attacker serializes the forged `ProofOfInclusion` (the struct is `Streamable` and constructable from Python via `__new__`).
4. Any consumer that calls `proof.valid()` — including Python DataLayer sync code that receives proofs from peers — receives `true`.
5. The forged proof's `root_hash()` will return whatever value the attacker chose as the top `combined_hash`, allowing the attacker to claim the proof corresponds to any tree root they wish.

The `ProofOfInclusion` struct is fully exposed via Python bindings (exported in `wheel/python/chia_rs/datalayer.pyi` with `valid()`, `root_hash()`, `from_bytes()`, and `__new__`), making it directly reachable from untrusted serialized input.

### Impact Explanation

**High.** DataLayer Merkle proof logic accepts forged inclusion proofs, letting untrusted input prove invalid state. Any DataLayer peer or client that calls `proof.valid()` on a received `ProofOfInclusion` without separately verifying `proof.root_hash() == known_trusted_root` will accept fabricated proofs. This allows an attacker to falsely prove that arbitrary key-value pairs are included in a DataLayer store, corrupting the integrity guarantees of the authenticated data layer. The `valid()` function name and its Python binding (`py_valid`) strongly imply it is a complete verification — the missing root-binding is not documented or enforced anywhere in the API.

### Likelihood Explanation

Medium-High. The `ProofOfInclusion` struct is a first-class Python-exposed type designed for use in DataLayer sync. Any DataLayer node that receives proofs from peers and calls `proof.valid()` as the sole check is vulnerable. The construction of a valid-looking forged proof requires only knowledge of `calculate_internal_hash` (public) and the ability to serialize a `ProofOfInclusion` (trivially possible via `from_bytes` or `__new__`).

### Recommendation

`valid()` must accept a trusted external root hash as a parameter and compare the computed root against it:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against EXTERNAL trusted root
}
```

The existing `valid()` method (which only checks internal consistency) should be renamed to `is_internally_consistent()` or removed to prevent misuse. The Python binding should expose only the root-anchored variant.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
from chia_rs import calculate_internal_hash  # or compute manually

# Attacker wants to forge a proof that node_hash X is in some tree
node_hash = bytes([0xAA] * 32)   # arbitrary fake leaf hash
other_hash = bytes([0xBB] * 32)  # arbitrary sibling hash

# Compute combined_hash using the public hash function
# (left/right ordering is attacker-chosen)
combined_hash = calculate_internal_hash(node_hash, 0, other_hash)

layer = ProofOfInclusionLayer(
    other_hash_side=0,
    other_hash=other_hash,
    combined_hash=combined_hash,
)
forged_proof = ProofOfInclusion(node_hash=node_hash, layers=[layer])

assert forged_proof.valid()          # True — forged proof passes
assert forged_proof.root_hash() == combined_hash  # attacker controls this value
```
