### Title
DataLayer `ProofOfInclusion::valid()` Always Returns `true` for Empty `layers` — Forged Single-Leaf Inclusion Proof Accepted - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` in `chia-datalayer` contains an empty-initial-state bypass: when the `layers` field is empty (the single-leaf tree case), the verification loop is skipped entirely and the method unconditionally returns `true` for any `node_hash` value. An unprivileged caller can construct or deserialize a `ProofOfInclusion { node_hash: <arbitrary_hash>, layers: [] }` and `valid()` will return `true`, forging a DataLayer inclusion proof without any knowledge of the actual tree.

### Finding Description

`ProofOfInclusion::valid()` is implemented as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()
}
``` [1](#0-0) 

And `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash   // ← returns node_hash when layers is empty
    }
}
``` [2](#0-1) 

When `layers` is empty:
1. The `for layer in &self.layers` loop body never executes.
2. `existing_hash` remains equal to `self.node_hash`.
3. `self.root_hash()` returns `self.node_hash` (the `else` branch).
4. The final check `existing_hash == self.root_hash()` becomes `self.node_hash == self.node_hash`, which is **always `true`**.

This is the direct analog of the DEX report's root cause: just as an empty pool triggers a special-case path that skips fee/swap simulation, an empty `layers` vector triggers a degenerate path that skips all hash verification.

The `ProofOfInclusion` struct is `Streamable` (deserializable from bytes) and is exposed to Python via `pyclass(get_all, from_py_object)` with a public constructor: [3](#0-2) 

The Python binding exposes `valid()` directly: [4](#0-3) 

The Python type stub confirms `ProofOfInclusion` is constructible with arbitrary `node_hash` and `layers`: [5](#0-4) 

The existing test suite only calls `proof.valid()` without comparing `root_hash()` to a known trusted root, confirming the intended usage pattern is that `valid()` alone is the complete verification: [6](#0-5) 

### Impact Explanation

An attacker who can supply a `ProofOfInclusion` object to any verifier that calls `proof.valid()` can forge a proof claiming that any arbitrary hash is included in a DataLayer store. The forged proof `ProofOfInclusion { node_hash: <target_hash>, layers: [] }` passes `valid()` unconditionally and reports `root_hash()` as `<target_hash>`. This allows untrusted input to prove invalid DataLayer state — matching the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."**

### Likelihood Explanation

The `ProofOfInclusion` struct is directly constructible from Python (via `from_py_object` and `__new__`), deserializable from bytes (`from_bytes`), and `valid()` is the sole public verification method. Any DataLayer client that receives a proof over the network and calls `proof.valid()` is vulnerable. No privileged access, key material, or tree knowledge is required.

### Recommendation

`valid()` must be given an external trusted root hash to compare against, or the single-leaf case must be explicitly validated. For example:

```rust
pub fn valid_against_root(&self, expected_root: &Hash) -> bool {
    // existing loop ...
    existing_hash == *expected_root
}
```

Alternatively, when `layers` is empty, `valid()` should require the caller to separately confirm that `node_hash` equals the known tree root — and this should be documented and enforced at the API level. The current `valid()` method must not be used as a standalone proof check.

### Proof of Concept

```python
from chia_rs import ProofOfInclusion, ProofOfInclusionLayer
from chia_rs.sized_bytes import bytes32

# Forge a proof for any arbitrary hash with no knowledge of the tree
forged_hash = bytes32(b'\xde\xad' * 16)
forged_proof = ProofOfInclusion(node_hash=forged_hash, layers=[])

assert forged_proof.valid()          # True — bypass confirmed
assert forged_proof.root_hash() == forged_hash  # attacker controls claimed root
```
