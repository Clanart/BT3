### Title
Unauthenticated `AddGasPrice`/`Initialize` on `L1GasPriceProvider` Remote Server Corrupts L1 Gas Price Feed — (File: `crates/apollo_l1_gas_price/src/communication.rs`)

---

### Summary

In the distributed deployment topology, `L1GasPriceProvider` runs with `LocalExecutionWithRemoteEnabled` and its `RemoteComponentServer` binds to `0.0.0.0` with no authentication. The server exposes every variant of `L1GasPriceRequest`, including the write operations `Initialize` (resets all accumulated samples) and `AddGasPrice` (appends a new `GasPriceData` sample). Any network-reachable peer can call these operations without restriction, allowing an attacker to corrupt the L1 gas price feed that the consensus orchestrator uses to set `l1_gas_price` and `l1_data_gas_price` in every block proposal.

---

### Finding Description

`L1GasPriceProvider::handle_request` dispatches all five `L1GasPriceRequest` variants — including the two mutating ones — without any caller identity check:

```rust
// crates/apollo_l1_gas_price/src/communication.rs
L1GasPriceRequest::Initialize =>
    L1GasPriceResponse::Initialize(self.initialize()),
L1GasPriceRequest::AddGasPrice(data) =>
    L1GasPriceResponse::AddGasPrice(self.add_price_info(data)),
``` [1](#0-0) 

`initialize()` resets `price_samples_by_block` to a fresh empty `RingBuffer`, discarding all accumulated samples:

```rust
pub fn initialize(&mut self) -> L1GasPriceProviderResult<()> {
    self.price_samples_by_block = Some(RingBuffer::new(self.config.storage_limit));
    Ok(())
}
``` [2](#0-1) 

`add_price_info()` enforces only a sequential block-number check (`expected = last + 1`). After a reset the buffer is empty, so the first insertion bypasses even that guard:

```rust
if let Some(data) = samples.back() {
    if new_data.block_number != data.block_number + 1 {
        return Err(...UnexpectedBlockNumberError...);
    }
}
samples.push(new_data);
``` [3](#0-2) 

The `RemoteComponentServer` that wraps this handler performs no authentication; it accepts any TCP connection, deserializes the request, and forwards it: [4](#0-3) 

In the distributed `l1` service configuration the provider is explicitly bound to all interfaces:

```json
"components.l1_gas_price_provider.execution_mode": "LocalExecutionWithRemoteEnabled",
"components.l1_gas_price_provider.remote_server_config.bind_ip": "0.0.0.0",
``` [5](#0-4) 

The same configuration appears in the hybrid topology: [6](#0-5) 

The corrupted prices flow directly into block proposals. The consensus orchestrator calls `get_price_info(timestamp)` and `get_eth_to_fri_rate(timestamp)` on the provider, then embeds the results as `l1_gas_price` and `l1_data_gas_price` in `ProposalInit` and the block header: [7](#0-6) 

---

### Impact Explanation

An attacker who can reach the L1 service port executes the following two-step sequence:

1. **Send `L1GasPriceRequest::Initialize`** — resets the ring buffer to empty, erasing all legitimate samples accumulated by the scraper.
2. **Send `L1GasPriceRequest::AddGasPrice(GasPriceData { block_number: N, timestamp: T, price_info: <attacker-chosen> })`** — because the buffer is now empty the sequential guard is skipped; the attacker's entry is accepted as the first sample. Subsequent injections with `block_number = N+1, N+2, …` pass the guard trivially.

Once the ring buffer contains only attacker-controlled samples, every call to `get_price_info(timestamp)` returns the attacker's chosen `base_fee_per_gas` and `blob_fee`. These values are embedded verbatim in the `ProposalInit` sent to the consensus manager and ultimately in the committed block header, producing wrong `l1_gas_price` / `l1_data_gas_price` for every transaction in every subsequent block.

The corrupted value is the authoritative L1 gas price used by the blockifier for fee validation and resource accounting. Setting it to zero or near-zero allows transactions that should be rejected for insufficient fees to be accepted; setting it to an extreme value causes all transactions to fail fee checks.

Matching impact scope: **Critical — Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact.**

---

### Likelihood Explanation

- The `RemoteComponentServer` is bound to `0.0.0.0` in both the distributed and hybrid production topologies.
- No authentication, TLS client certificate, or IP allowlist is implemented at the server layer.
- The attacker needs only TCP reachability to the configured port — a network-adjacent attacker (same VPC/cluster without strict `NetworkPolicy`) satisfies this.
- The `Initialize` + `AddGasPrice` sequence requires no secret knowledge; L1 block numbers are public.
- The legitimate scraper will eventually call `Initialize` again on restart (triggered by the `UnexpectedBlockNumberError` the attacker's injection causes), but the attacker can repeat the sequence faster than the scraper recovers, sustaining the corruption.

---

### Recommendation

1. **Restrict write operations to local callers only.** Split `L1GasPriceRequest` into a read-only variant (served by the `RemoteComponentServer`) and a write variant (served only by the `LocalComponentServer`). The `Initialize` and `AddGasPrice` variants must never be reachable over the network.

2. **Add caller authentication to `RemoteComponentServer`.** Implement mutual TLS or a shared-secret header check so that only the designated scraper process can reach the write endpoint, even if the split above is not immediately feasible.

3. **Validate the source of `AddGasPrice` calls.** At minimum, assert inside `add_price_info` that the call originates from the local scraper task (e.g., via a capability token passed at construction time), rejecting any call that arrives through the remote server path.

---

### Proof of Concept

```
# 1. Discover the L1 gas price provider port from the deployment config
PORT=$(kubectl get configmap l1-node-config -o json | jq '."components.l1_gas_price_provider.port"')

# 2. Serialize and send Initialize to wipe the ring buffer
curl -s -X POST http://<l1-service-ip>:$PORT \
  -H "Content-Type: application/octet-stream" \
  --data-binary @<(serialize_request L1GasPriceRequest::Initialize)

# 3. Inject a fake GasPriceData entry with attacker-chosen prices
#    block_number=0 is accepted because the buffer is now empty (no predecessor check)
curl -s -X POST http://<l1-service-ip>:$PORT \
  -H "Content-Type: application/octet-stream" \
  --data-binary @<(serialize_request \
      L1GasPriceRequest::AddGasPrice(GasPriceData {
          block_number: 0,
          timestamp: BlockTimestamp(<current_unix_ts>),
          price_info: PriceInfo {
              base_fee_per_gas: GasPrice(0),   // zero out fees
              blob_fee: GasPrice(0),
          }
      }))

# 4. Repeat step 3 with block_number=1,2,... to fill the ring buffer
#    with attacker-controlled samples before the scraper can recover.
#
# Result: the next call to get_price_info() by the consensus orchestrator
# returns base_fee_per_gas=0 / blob_fee=0, embedding zero L1 gas prices
# in every subsequent block proposal and committed block header.
```

    pub fn get_price_info(&self, timestamp: BlockTimestamp) -> L1GasPriceProviderResult<PriceInfo> {
        let Some(samples) = &self.price_samples_by_block else {
            return Err(L1GasPriceProviderError::NotInitializedError);
        };
        // timestamp of the newest price sample
        let last_timestamp = samples
            .back()
            .ok_or(L1GasPriceProviderError::MissingDataError {
                timestamp: timestamp.0,
                lag: self.config.lag_margin_seconds.as_secs(),
            })?
            .timestamp;

        // Check if the prices are stale.
        if timestamp.0 > (*last_timestamp + self.config.max_time_gap_seconds) {
            return Err(L1GasPriceProviderError::StaleL1GasPricesError {
                current_timestamp: timestamp.0,
                last_valid_price_timestamp: *last_timestamp,
            });
        }

        // This index is for the last block in the mean (inclusive).
        let index_last_timestamp_rev = samples.iter().rev().position(|data| {
            data.timestamp <= timestamp.saturating_sub(&self.config.lag_margin_seconds.as_secs())
        });

        // Could not find a block with the requested timestamp and lag.
        let Some(last_index_rev) = index_last_timestamp_rev else {
            return Err(L1GasPriceProviderError::MissingDataError {
                timestamp: timestamp.0,
                lag: self.config.lag_margin_seconds.as_secs(),
            });
        };
        // Convert the index to the forward direction.
        // `last_index` should be one past the final entry we will include in our calculation.
        // The index returned from `position` is guaranteed to be less than `len()`,
        // so `last_index` is guaranteed to be >= 1.
        let last_index = samples.len() - last_index_rev;

        let num_blocks = usize::try_from(self.config.number_of_blocks_for_mean)
            .expect("number_of_blocks_for_mean is too large to fit into a usize");

        let first_index = if last_index >= num_blocks {
            last_index - num_blocks
        } else {
            warn!(
                "Not enough history to calculate the mean gas price. Using blocks {}-{}, \
                 inclusive.",
                samples[0].block_number,
                samples[last_index - 1].block_number,
            );
            L1_GAS_PRICE_PROVIDER_INSUFFICIENT_HISTORY.increment(1);
            0
        };
        debug_assert!(first_index < last_index, "error calculating indices");
        let actual_number_of_blocks = last_index - first_index;

        // Go over all elements between `first_index` and `last_index` (non-inclusive).
        let price_info_summed: PriceInfo = samples
            .iter()
            .skip(first_index)
            .take(actual_number_of_blocks)
            .map(|data| &data.price_info)
            .sum();
        let actual_number_of_blocks =
            u128::try_from(actual_number_of_blocks).expect("Cannot convert to u128");
        let price_info_out = price_info_summed
            .checked_div(actual_number_of_blocks)
            .expect("Actual number of blocks should be non-zero");
        info_every_n_ms!(
            1_000,
            "Calculated L1 gas price for timestamp {}: {:?} (based on blocks {}-{}, inclusive)",
            timestamp.0,
            price_info_out,
            samples[first_index].block_number,
            samples[last_index - 1].block_number,
        );
        L1_GAS_PRICE_LATEST_MEAN_VALUE.set_lossy(price_info_out.base_fee_per_gas.0);
        L1_DATA_GAS_PRICE_LATEST_MEAN_VALUE.set_lossy(price_info_out.blob_fee.0);
        Ok(price_info_out)
    }
