### Title
Unbounded Stale ETH→STRK Rate Propagation Through Block Commitments When Oracle Is Unavailable - (File: crates/apollo_consensus_orchestrator/src/utils.rs)

### Summary
The `L1GasPriceProvider` has a `max_time_gap_seconds` staleness guard for L1 gas prices, but the ETH→STRK oracle path has no equivalent check. When the oracle is unavailable, `get_l1_prices_in_fri_and_wei_and_conversion_rate` silently falls back to the ETH→STRK rate embedded in `previous_block_info`. Because `previous_block_info` is refreshed on every `decision_reached` from the committed `ProposalInit`, the stale rate propagates forward through every subsequent block header indefinitely, embedding wrong FRI gas prices into block hashes and fee calculations.

### Finding Description

**Asymmetric staleness protection.** `L1GasPriceProvider::get_price_info()` has an explicit staleness guard:

```rust
// crates/apollo_l1_gas_price/src/l1_gas_price_provider.rs:118-124
if timestamp.0 > (*last_timestamp + self.config.max_time_gap_seconds) {
    return Err(L1GasPriceProviderError::StaleL1GasPricesError { ... });
}
```

`EthToStrkOracleClient::eth_to_fri_rate()` has no equivalent check. The LRU cache (`cached_prices`) stores rates keyed by `quantized_timestamp` with no TTL or age validation. When the oracle API is unreachable, the function returns `QueryNotReadyError` or `AllUrlsFailedError` after exhausting the one-interval fallback:

```rust
// crates/apollo_l1_gas_price/src/eth_to_strk_oracle.rs:214-227
if !handle.is_finished() {
    if let Some(rate) = cache.get(&(quantized_timestamp - NUMBER_OF_TIMESTAMPS_BACK)) {
        return Ok(*rate);  // bounded one-interval fallback
    }
    return Err(EthToStrkOracleClientError::QueryNotReadyError(timestamp));
}
```

**Silent fallback to unbounded-age previous block info.** Because `eth_to_fri_rate` returns an error, the `if let (Ok(...), Ok(...))` guard at line 158 of `utils.rs` is false, and execution falls through to the `previous_block_info` path with no staleness bound:

```rust
// crates/apollo_consensus_orchestrator/src/utils.rs:185-198
if let Some(block_info) = previous_block_info {
    let eth_to_fri_rate = calculate_eth_to_fri_rate(block_info);
    // No check on how old block_info is
    return (prev_l1_gas_price, prev_l1_gas_price_wei, eth_to_fri_rate);
}
```

**Stale rate propagates through every committed block.** `previous_block_info` is refreshed on every `decision_reached` from the committed `ProposalInit`:

```rust
// crates/apollo_consensus_orchestrator/src/sequencer_consensus_context.rs:817
self.previous_block_info = Some(PreviousBlockInfo::from(&init));
```

The `ProposalInit` carries the FRI gas prices that were themselves derived from the stale rate. So block N+1 inherits the stale rate from block N, block N+2 from N+1, and so on — the rate from the last oracle-working block propagates indefinitely with no decay or bound.

**Stale rate reaches block hash.** The stale FRI prices are embedded in `PartialBlockHashComponents` via `PartialBlockHashComponents::new` (which reads `block_info.gas_prices.l1_gas_price_per_token()` and `l1_data_gas_price_per_token()`), and are then committed into the block hash by `calculate_block_hash` through `gas_prices_to_hash`.

**Validator accepts the stale rate.** Both proposer and validator call `get_l1_prices_in_fri_and_wei` with the same `self.previous_block_info` from `SequencerConsensusContext`. Both fall back to the same stale rate, so the `within_margin` check in `is_block_info_valid` passes:

```rust
// crates/apollo_consensus_orchestrator/src/validate_proposal.rs:302-307
if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
    && within_margin(...))
```

The stale rate is therefore accepted by consensus and committed to the block hash.

### Impact Explanation

Wrong ETH→STRK rate → wrong `l1_gas_price.price_in_fri` / `l1_data_gas_price.price_in_fri` in every block header → wrong block hash → wrong fee calculations for all transactions in every block produced during oracle downtime. If the ETH/STRK price moves significantly while the oracle is down, fees in FRI are systematically mispriced. Users can underpay (if ETH appreciated) or overpay (if ETH depreciated), with unbounded economic impact proportional to oracle downtime and price movement. This matches: **Critical. Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact.**

### Likelihood Explanation

The ETH→STRK oracle is an external HTTP API (`EthToStrkOracleClient`). It can become unavailable due to network partitions, API outages, or maintenance. The `L1GasPriceProvider` already acknowledges this risk and guards against it with `max_time_gap_seconds` (default 900 s). The ETH→STRK path has no equivalent guard, so any oracle downtime longer than one `lag_interval_seconds` (default 1 s) silently activates the unbounded fallback.

### Recommendation

Add a configurable `max_oracle_gap_seconds` to `EthToStrkOracleConfig` (mirroring `max_time_gap_seconds` in `L1GasPriceProviderConfig`). Track the wall-clock time of the last successful oracle response. In `get_l1_prices_in_fri_and_wei_and_conversion_rate`, before accepting `previous_block_info` as the ETH→STRK source, check whether the oracle has been silent longer than the threshold and, if so, either return an error that halts block production or fall through to the hard-coded `DEFAULT_ETH_TO_FRI_RATE` fallback rather than silently propagating the stale rate.

### Proof of Concept

1. Oracle API becomes unavailable at block N (e.g., network outage).
2. For block N+1, `eth_to_fri_rate(timestamp)` exhausts the one-interval LRU fallback and returns `QueryNotReadyError`.
3. `get_l1_prices_in_fri_and_wei_and_conversion_rate` falls back to `previous_block_info` (from block N's `ProposalInit`), which carries the ETH→STRK rate from block N.
4. FRI gas prices for block N+1 are computed from this stale rate and embedded in `PartialBlockHashComponents`.
5. Validator also calls `get_l1_prices_in_fri_and_wei` with the same `previous_block_info`; `within_margin` passes; block N+1 is committed.
6. `decision_reached` sets `self.previous_block_info = Some(PreviousBlockInfo::from(&init_N+1))`, where `init_N+1` carries the stale FRI prices.
7. For block N+2, step 3 repeats using block N+1's `previous_block_info`, which still encodes the rate from block N.
8. The stale rate from block N propagates through blocks N+1, N+2, … indefinitely, with no staleness check ever firing. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) [7](#0-6) [8](#0-7)

    async fn eth_to_fri_rate(&self, timestamp: u64) -> Result<u128, EthToStrkOracleClientError> {
        const NUMBER_OF_TIMESTAMPS_BACK: u64 = 1;
        let quantized_timestamp = (timestamp - self.config.lag_interval_seconds)
            .checked_div(self.config.lag_interval_seconds)
            .expect("lag_interval_seconds should be non-zero");

        let mut cache = self.cached_prices.lock().unwrap();

        if let Some(rate) = cache.get(&quantized_timestamp) {
            debug!("Cached conversion rate for timestamp {timestamp} is {rate}");
            return Ok(*rate);
        }

        // Check if there is a query already sent out for this timestamp, if not, start one.
        let mut queries = self.queries.lock().unwrap();
        let handle = queries
            .get_or_insert_mut(quantized_timestamp, || self.spawn_query(quantized_timestamp));
        // If the query is not finished, return an error.
        if !handle.is_finished() {
            debug!("Query not yet resolved: timestamp={timestamp}");
            // If the previous quantized timestamp is in the cache, use it.
            if let Some(rate) = cache.get(&(quantized_timestamp - NUMBER_OF_TIMESTAMPS_BACK)) {
                debug!(
                    "Query not yet resolved: timestamp={timestamp}, using previous rate {rate} \
                     from quantized timestamp={}",
                    (quantized_timestamp - NUMBER_OF_TIMESTAMPS_BACK)
                        * self.config.lag_interval_seconds
                );
                return Ok(*rate);
            }
            // If not, return a query not ready error.
            return Err(EthToStrkOracleClientError::QueryNotReadyError(timestamp));
        }
