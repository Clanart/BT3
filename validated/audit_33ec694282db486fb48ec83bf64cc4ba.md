### Title
`MerkleBlob::batch_insert` Skips Duplicate-Key/Hash Validation for All But the Last Two Items, Enabling Silent Merkle Tree Root Corruption — (File: `crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::batch_insert` applies the full uniqueness guard (via the checked `insert` path) only to the **last two** items popped from the input vector. Every earlier item is written directly to the blob through `insert_entry_to_blob` with **no check** for a duplicate `KeyId` or duplicate leaf `Hash`. Supplying a batch that contains a repeated key — or a key already present in the tree — silently produces a structurally invalid Merkle tree. The resulting root hash is computed over duplicate leaf nodes, so any proof of inclusion or exclusion derived from it is either forged or broken.

---

### Finding Description

`MerkleBlob::insert` guards against duplicates at lines 369–374:

```rust
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
```

`batch_insert` calls `self.insert(…)` only for the last two items it pops from the vector (lines 578–585). All remaining items — the head of the vector — bypass those checks entirely and are written directly via `insert_entry_to_blob` (lines 587–603):

```rust
for ((key, value), hash) in keys_values_hashes {   // no duplicate check
    let new_leaf_index = self.get_new_index();
    let new_block = Block { … Node::Leaf(LeafNode { hash, key, value, … }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;   // updates cache unconditionally
    indexes.push(new_leaf_index);
}
```

`insert_entry_to_blob` calls `self.block_status_cache.add_leaf(index, leaf)`, which inserts into the `key_to_index: HashMap<KeyId, TreeIndex>` and `leaf_hash_to_index: HashMap<Hash, TreeIndex>` maps. Because `HashMap::insert` silently overwrites existing entries, a duplicate key causes the cache to point to the **newer** leaf while the **older** leaf remains in the blob, still referenced by its parent internal node. The tree is now structurally inconsistent:

* Two leaf nodes with the same `KeyId` exist in the blob.
* The cache tracks only the newer one.
* The subtree hashes propagated upward (lines 607–648) are computed from this corrupted structure, producing an incorrect root hash.
* `check_integrity` (lines 821–887) would detect the mismatch (`IntegrityKeyToIndexCacheIndex` / `IntegrityKeyToIndexCacheLength`), but it is only enabled in test builds (`check_integrity_on_drop: cfg!(test)`); production builds never call it.

The Python binding `py_batch_insert` (lines 1503–1519) performs no additional uniqueness check — it only verifies that the lengths of the two input slices match — so the bug is directly reachable from Python/wasm callers.

---

### Impact Explanation

A caller that supplies a `batch_insert` input containing a repeated `KeyId` (or a `KeyId` already present in the tree) silently corrupts the Merkle tree root. Any `ProofOfInclusion` subsequently generated by `get_proof_of_inclusion` will be computed against the wrong root hash. A verifier that trusts the root will either:

* Accept a proof for a key that was overwritten (forged inclusion), or
* Reject a proof for a legitimately present key (broken inclusion).

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."**

---

### Likelihood Explanation

The vulnerability class is identical to the external report: a uniqueness invariant that is enforced in the single-item path (`insert`) is silently absent in the bulk path (`batch_insert`). Any caller — including the Python/wasm DataLayer bindings — that passes a batch with ≥ 3 items where any of the first `n-2` items share a key with another item in the batch (or with an existing tree key) triggers the corruption. Because `batch_insert` is the performance-optimised path used for bulk DataLayer updates, it is the natural target for high-volume writes, increasing the probability of accidental or deliberate duplicate submission.

---

### Recommendation

Add the same duplicate guards at the top of the fast-path loop in `batch_insert`, or refactor the loop to call the checked `insert` helper for every item. At minimum:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // … existing fast-path logic …
}
```

Alternatively, pre-validate the entire input vector for internal duplicates before entering either branch.

### Proof of Concept

```rust
use chia_datalayer::{Hash, InsertLocation, KeyId, MerkleBlob, ValueId};
use chia_protocol::Bytes32;

fn sha256_of(n: u8) -> Hash {
    use chia_sha2::Sha256;
    let mut h = Sha256::new();
    h.update([n]);
    Hash(Bytes32::new(h.finalize()))
}

fn main() {
    let mut blob = MerkleBlob::new(Vec::new()).unwrap();

    // Pre-populate so leaf_count > 1 (forces the fast path for items 0..n-2)
    blob.insert(KeyId(100), ValueId(100), &sha256_of(100), InsertLocation::Auto {}).unwrap();
    blob.insert(KeyId(101), ValueId(101), &sha256_of(101), InsertLocation::Auto {}).unwrap();

    // Batch: items 0 and 1 go through the fast path (no duplicate check).
    // Item 0 duplicates KeyId(100) which is already in the tree.
    let batch = vec![
        ((KeyId(100), ValueId(999)), sha256_of(200)), // duplicate key — NOT checked
        ((KeyId(102), ValueId(102)), sha256_of(102)), // last-2: checked via insert()
        ((KeyId(103), ValueId(103)), sha256_of(103)), // last-1: checked via insert()
    ];

    // Succeeds silently — no KeyAlreadyPresent error
    blob.batch_insert(batch).unwrap();

    // Tree is now corrupt: two leaf nodes with KeyId(100) exist in the blob.
    // check_integrity() would return an error, but it is disabled in production.
    // The root hash is computed from the corrupted structure.
    blob.check_integrity_on_drop = false; // suppress the test-only panic
    // blob.check_integrity().unwrap_err(); // would fire IntegrityKeyToIndexCacheIndex
}

    fn check_just_integrity(&self) -> Result<(), Error> {
        let mut leaf_count: usize = 0;
        let mut internal_count: usize = 0;
        let mut child_to_parent: HashMap<TreeIndex, TreeIndex> = HashMap::new();

        for item in ParentFirstIterator::new(&self.blob, None) {
            let (index, block) = item?;
            if let Some(parent) = block.node.parent().0 {
                if child_to_parent.remove(&index) != Some(parent) {
                    return Err(Error::IntegrityParentChildMismatch(index));
                }
            }
            match block.node {
                Node::Internal(node) => {
                    internal_count += 1;
                    child_to_parent.insert(node.left, index);
                    child_to_parent.insert(node.right, index);
                }
                Node::Leaf(node) => {
                    leaf_count += 1;
                    let cached_index = self
                        .block_status_cache
                        .get_index_by_key(node.key)
                        .ok_or(Error::IntegrityKeyNotInCache(node.key))?;
                    if *cached_index != index {
                        return Err(Error::IntegrityKeyToIndexCacheIndex(
                            node.key,
                            index,
                            *cached_index,
                        ));
                    }
                    assert!(
                        !self.block_status_cache.is_index_free(index),
                        "{}",
                        format!("active index found in free index list: {index:?}")
                    );
                }
            }
        }

        let key_to_index_cache_length = self.block_status_cache.key_to_index.len();
        if leaf_count != key_to_index_cache_length {
            return Err(Error::IntegrityKeyToIndexCacheLength(
                leaf_count,
                key_to_index_cache_length,
            ));
        }
        let leaf_hash_to_index_cache_length = self.block_status_cache.leaf_hash_to_index.len();
        if leaf_count != leaf_hash_to_index_cache_length {
            return Err(Error::IntegrityLeafHashToIndexCacheLength(
                leaf_count,
                leaf_hash_to_index_cache_length,
            ));
        }
        let total_count = leaf_count + internal_count + self.block_status_cache.free_index_count();
        let extend_index = self.extend_index();
        if total_count != extend_index.0 as usize {
            return Err(Error::IntegrityTotalNodeCount(extend_index, total_count));
        }
        if !child_to_parent.is_empty() {
            return Err(Error::IntegrityUnmatchedChildParentRelationships(
                child_to_parent.len(),
            ));
        }

        Ok(())

    fn insert_entry_to_blob(&mut self, index: TreeIndex, block: &Block) -> Result<(), Error> {
        let new_block_bytes = block.to_bytes()?;
        let extend_index = self.extend_index();
        match index.cmp(&extend_index) {
            Ordering::Greater => return Err(Error::BlockIndexOutOfBounds(index)),
            Ordering::Equal => self.blob.extend_from_slice(&new_block_bytes),
            Ordering::Less => {
                self.blob[block_range(index)].copy_from_slice(&new_block_bytes);
            }
        }

        match block.node {
            Node::Leaf(leaf) => self.block_status_cache.add_leaf(index, leaf),
            Node::Internal(..) => self.block_status_cache.add_internal(index),
        }

        Ok(())
    }
