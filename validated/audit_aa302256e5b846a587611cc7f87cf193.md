### Title
Hardcoded `'BOOTSTRAP'` Magic Address Bypasses All Declare Transaction Security Checks — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

`execute_declare_transaction` in `transaction_impls.cairo` contains a hardcoded magic felt literal `'BOOTSTRAP'` that, when used as the `sender_address` of a declare transaction with `nonce=0`, `version=3`, and zero resource bounds, completely skips signature verification, nonce enforcement, and fee charging. Any party who can include such a transaction in a block can register an arbitrary `class_hash → compiled_class_hash` mapping without owning the sender account, without a valid signature, and without paying fees. This is the direct analog of the "hardcoded credentials" class from the external report: a hardcoded privileged value baked into the OS that bypasses the authentication layer.

---

### Finding Description

In `execute_declare_transaction`, after computing the transaction hash and verifying the Sierra class hash pre-image, the OS evaluates the following branch:

```cairo
if (sender_address == 'BOOTSTRAP' and tx_info.nonce == 0 and tx_info.version == 3) {
    let max_possible_fee = compute_max_possible_fee(tx_info=tx_info);
    if (max_possible_fee == 0) {
        assert_not_zero(compiled_class_hash);
        dict_update{dict_ptr=contract_class_changes}(
            key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
        );
        %{ SkipTx %}
        return ();
    }
}
``` [1](#0-0) 

When this branch is taken, the function returns immediately, skipping all three security gates that follow in the normal path:

1. **`check_and_increment_nonce`** — nonce state is never read or incremented; the `'BOOTSTRAP'` account need not exist at all.
2. **`non_reverting_select_execute_entry_point_func` with `VALIDATE_DECLARE_ENTRY_POINT_SELECTOR`** — the `__validate_declare__` entry point is never called; no signature is verified.
3. **`charge_fee`** — no fee is deducted. [2](#0-1) 

The `'BOOTSTRAP'` literal is a Cairo short-string felt (`0x424f4f545354524150`), publicly visible in the OS source. The only constraint on the declared class is:

- `class_hash` must pass `finalize_class_hash` (valid Sierra class hash pre-image). [3](#0-2) 
- `compiled_class_hash` must be non-zero. [4](#0-3) 
- The class must not already be declared (`prev_value=0`).

Critically, **`compiled_class_hash` is never verified to correspond to `class_hash`** in the bootstrap path. The normal path relies on the sequencer's off-chain validation, but the bootstrap path skips even that gate.

The hardcoded nature of this value is the direct analog of the "hardcoded credentials" finding: a fixed, publicly known value embedded in the OS that grants unconditional privilege. [5](#0-4) 

---

### Impact Explanation

An attacker who can include a block (i.e., any sequencer in a decentralized StarkNet) can:

1. **Front-run a legitimate class declaration**: take a valid Sierra class whose `class_hash` has not yet been registered, and map it to a malicious `compiled_class_hash`. Contracts subsequently deployed under that `class_hash` will execute the attacker's CASM, not the Sierra source the `class_hash` commits to.
2. **Declare a new class with a mismatched compiled hash**: create a fresh Sierra class, declare it with a malicious `compiled_class_hash`, and deploy contracts that appear legitimate (matching a known Sierra source) but execute arbitrary code.

In both cases, users who deploy or interact with contracts of the poisoned class can have their funds stolen. This satisfies **Critical — Direct loss of funds**.

---

### Likelihood Explanation

The `'BOOTSTRAP'` felt value is hardcoded in publicly auditable OS source code. The conditions required (`nonce=0`, `version=3`, all resource bounds zero) are trivially constructable. In a decentralized StarkNet where sequencer rotation occurs, any sequencer node — not just the current trusted operator — can include such a transaction. The OS proof will be valid, and the class registration will be accepted by verifiers. No key material, phishing, or external dependency is required.

---

### Recommendation

Remove the `'BOOTSTRAP'` special-case branch from `execute_declare_transaction`. If system bootstrapping is a genuine operational requirement, replace it with a governance-controlled mechanism (e.g., a time-bounded bootstrapping window enforced by block number, or a multi-sig controlled address whose class hash is verified against a known commitment rather than bypassed entirely). The `compiled_class_hash` must always be cryptographically bound to `class_hash` before registration.

### Proof of Concept

```
1. Attacker constructs a valid Sierra contract C with class_hash H.
   - finalize_class_hash(component_hashes_of_C) == H  ✓

2. Attacker constructs a malicious CASM program M with compiled_class_hash M_hash ≠ 0.
   - M_hash is NOT the legitimate compiled hash of C.

3. Attacker (as sequencer) includes a declare transaction in a block:
     sender_address      = 0x424f4f545354524150  ('BOOTSTRAP')
     nonce               = 0
     version             = 3
     tip                 = 0
     resource_bounds     = [{L1_GAS, max_amount=0, max_price=0},
                            {L2_GAS, max_amount=0, max_price=0},
                            {L1_DATA, max_amount=0, max_price=0}]
     class_hash          = H
     compiled_class_hash = M_hash

4. OS execution path:
   a. compute_declare_transaction_hash() → tx_hash (includes sender='BOOTSTRAP')
   b. finalize_class_hash() asserts class_hash == H  ✓
   c. fill_account_tx_info() fills TxInfo with version=3, nonce=0, max_fee=0
   d. compute_max_possible_fee() returns 0  ✓
   e. Bootstrap branch taken → dict_update(key=H, prev_value=0, new_value=M_hash)
   f. return() — no __validate_declare__, no nonce check, no fee

5. Proof is generated and verified. H → M_hash is now canonical on-chain.

6. Any user who deploys a contract with class_hash=H executes M (malicious CASM),
   not the Sierra source C. Attacker drains funds from users of that contract.

func check_and_increment_nonce{contract_state_changes: DictAccess*}(tx_info: TxInfo*) -> () {
    // Do not handle nonce for version 0.
    if (tx_info.version == 0) {
        return ();
    }

    tempvar state_entry: StateEntry*;
    %{ SetStateEntryToAccountContractAddress %}

    tempvar current_nonce = state_entry.nonce;
    with_attr error_message("Unexpected nonce.") {
        assert current_nonce = tx_info.nonce;
    }

    // Update contract_state_changes.
    tempvar new_state_entry = new StateEntry(
        class_hash=state_entry.class_hash,
        storage_ptr=state_entry.storage_ptr,
        nonce=current_nonce + 1,
    );
    dict_update{dict_ptr=contract_state_changes}(
        key=tx_info.account_contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    return ();
