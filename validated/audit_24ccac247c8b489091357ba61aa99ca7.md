### Title
Unbounded Predecessor Replay Loop in `TraceTx` gRPC Handler Enables Resource-Exhaustion DoS - (File: x/evm/keeper/grpc_query.go)

### Summary

The `TraceTx` gRPC handler iterates over a caller-supplied `Predecessors` slice of arbitrary length, executing a full `ApplyMessageWithConfig` call for every entry with no count limit and no timeout covering the replay loop. An unprivileged caller can submit a single `TraceTx` (or `debug_traceTransaction` JSON-RPC) request carrying thousands of complex predecessor transactions, forcing the node to execute unbounded EVM work before the traced transaction even begins.

### Finding Description

`TraceTx` in `x/evm/keeper/grpc_query.go` accepts a `QueryTraceTxRequest` whose `Predecessors` field is a `repeated MsgEthereumTx` with no upper-bound enforced anywhere in the proto definition or in the handler. [1](#0-0) 

Inside the handler, the predecessor slice is iterated unconditionally: [2](#0-1) 

Each iteration calls `ApplyMessageWithConfig(ctx, msg, cfg, true)` — a full EVM state transition committed to the in-memory `stateDB`. The only timeout in the tracing path is set inside `prepareTrace`, which is called **after** the predecessor loop completes: [3](#0-2) 

The predecessor loop therefore runs entirely outside any deadline. An attacker can supply N predecessors, each being a gas-heavy contract deployment or call, and the server will execute all N EVM transactions before the timeout goroutine is even started.

The same structural issue exists in `TraceBlock`, where `req.Txs` is iterated without a count limit and each iteration calls `prepareTrace` (each with its own independent 5-second timeout, so N transactions yield up to N × 5 s of wall-clock work): [4](#0-3) 

The `SimulateV1` path does enforce `MaxSimulateBlocks` on the block count: [5](#0-4) 

but no analogous guard exists for `Predecessors` in `TraceTx` or for `Txs` in `TraceBlock`.

### Impact Explanation

The `TraceTx` gRPC method is exposed publicly via the `debug_traceTransaction` JSON-RPC endpoint. Any unauthenticated caller can submit a request with an arbitrarily large `Predecessors` array. Because each predecessor undergoes a full EVM execution (`ApplyMessageWithConfig` with `commit=true`), the CPU and memory cost scales linearly with the number of predecessors and the gas complexity of each one. A single malicious request can monopolise the node's goroutine for minutes, starving the consensus engine of CPU time and causing the validator to miss blocks — a reachable route to chain-level liveness failure when multiple validators are targeted.

### Likelihood Explanation

The `debug_traceTransaction` JSON-RPC endpoint is enabled by default on Ethermint nodes and requires no authentication. The `Predecessors` field is documented as a normal part of the public API (it is used by the backend's `TraceTransaction` to replay prior block transactions). Any user who can reach the RPC port can craft a malicious request. No special privileges, keys, or on-chain state are required.

### Recommendation

1. **Enforce a hard cap on `len(req.Predecessors)`** before entering the replay loop in `TraceTx`, analogous to the `MaxSimulateBlocks` guard in `SimulateV1`. A reasonable limit is the maximum number of transactions per block.
2. **Wrap the entire predecessor replay loop in a context with deadline**, not just the final `prepareTrace` call, so that a slow or large predecessor set is bounded by the same configurable trace timeout.
3. **Apply a similar cap on `len(req.Txs)`** in `TraceBlock`.

### Proof of Concept

1. Attacker prepares M `MsgEthereumTx` objects, each deploying a large contract (near the `MaxInitCodeSize` limit) — maximising per-predecessor EVM work.
2. Attacker sends a single `debug_traceTransaction` JSON-RPC call (which maps to `TraceTx`) with those M transactions in the `predecessors` field and any valid target transaction as `msg`.
3. The server enters the loop at `grpc_query.go:542` and calls `ApplyMessageWithConfig` M times with no timeout.
4. The node's RPC goroutine is blocked for the duration of all M EVM executions; the consensus engine is starved of CPU, causing missed blocks and potential slashing. [6](#0-5) [7](#0-6)

func (k Keeper) TraceTx(c context.Context, req *types.QueryTraceTxRequest) (*types.QueryTraceTxResponse, error) {
	var baseFee *big.Int
	if req == nil || req.Msg == nil {
		return nil, status.Error(codes.InvalidArgument, "request and message cannot be empty")
	}
	if req.BaseFee != nil && !req.BaseFee.IsNil() {
		baseFee = req.BaseFee.BigInt()
	}
	resultData, err := execTrace(
		c,
		req,
		k,
		baseFee,
		func(ctx sdk.Context, cfg *EVMConfig, traceConfig *types.TraceConfig) (*core.Message, error) {
			signer := ethtypes.MakeSigner(cfg.ChainConfig, big.NewInt(ctx.BlockHeight()), uint64(ctx.BlockTime().Unix())) //#nosec G115
			tracer, err := newTacer(&logger.Config{}, cfg.ChainConfig, cfg.TxConfig, traceConfig)
			if err != nil {
				return nil, status.Error(codes.Internal, err.Error())
			}
			cfg.Tracer = tracer.Hooks
			cfg.DebugTrace = true
			cfg.TraceReplay = traceConfig.GetTraceReplay()
			for i, tx := range req.Predecessors {
				ethTx := tx.AsTransaction()
				msg, err := core.TransactionToMessage(ethTx, signer, cfg.BaseFee)
				if err != nil {
					k.Logger(ctx).Debug("trace: skipping predecessor, failed to convert tx to message",
						"index", i, "hash", ethTx.Hash().Hex(), "err", err.Error())
					continue
				}
				cfg.TxConfig.TxHash = ethTx.Hash()
				cfg.TxConfig.TxIndex, err = ethermint.SafeUint(i)
				if err != nil {
					k.Logger(ctx).Debug("trace: skipping predecessor, invalid tx index",
						"index", i, "hash", ethTx.Hash().Hex(), "err", err.Error())
					continue
				}
				rsp, err := k.ApplyMessageWithConfig(ctx, msg, cfg, true)
				if err != nil {
					k.Logger(ctx).Error("trace: predecessor replay failed, trace state may be incomplete",
						"index", i, "hash", ethTx.Hash().Hex(), "err", err.Error())
					continue
				}
				cfg.TxConfig.LogIndex += uint(len(rsp.Logs))
			}

			tx := req.Msg.AsTransaction()
			cfg.TxConfig.TxHash = tx.Hash()
			if len(req.Predecessors) > 0 {
				cfg.TxConfig.TxIndex++
			}

			return core.TransactionToMessage(tx, signer, cfg.BaseFee)
		},
	)
