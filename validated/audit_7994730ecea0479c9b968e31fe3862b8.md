### Title
Gateway Silently Swallows Proof-Manager Store Failure, Permanently Breaking Reproposal for Affected Transactions — (`crates/apollo_gateway/src/gateway.rs`)

### Summary

`store_proof_and_spawn_archiving` in the Gateway swallows a proof-manager `set_proof` failure and still forwards the transaction to the mempool. The internal representation of the transaction strips the raw proof bytes; the only copy is the proof manager. When the proposer later calls `send_reproposal`, `convert_internal_rpc_tx_to_rpc_tx` unconditionally calls `get_proof` and returns `ProofNotFound`, making every reproposal attempt for that block fail permanently.

### Finding Description

**Step 1 — Error is swallowed in the gateway.**

`store_proof_and_spawn_archiving` calls `store_proof_in_proof_manager`. On failure it logs an error and falls through; the transaction is still sent to the mempool. [1](#0-0) 

**Step 2 — The internal transaction no longer carries the proof.**

`convert_rpc_tx_to_internal` separates the proof from the transaction body. The `InternalRpcTransaction` stored in the mempool contains only `proof_facts`; the raw `Proof` bytes are expected to live exclusively in the proof manager. [2](#0-1) 

**Step 3 — Reproposal unconditionally fetches the proof from the proof manager.**

`convert_internal_rpc_tx_to_rpc_tx`, called by `send_reproposal` → `convert_internal_consensus_tx_to_consensus_tx`, calls `self.get_proof(&tx.proof_facts).await?`. The comment explicitly states the invariant that the gateway is supposed to guarantee: [3](#0-2) 

If the proof was never stored, `get_proof` returns `Ok(None)` which is mapped to `Err(ProofNotFound)`: [4](#0-3) 

**Step 4 — Reproposal propagates the error and fails.**

`send_reproposal` collects all conversion results with `collect::<Result<Vec<_>, _>>()?`, so a single `ProofNotFound` aborts the entire reproposal. [5](#0-4) 

**Analogy to the external report.** In `shutdownPool`, `withdrawAll` is wrapped in a try/catch whose failure is silently ignored, yet `pool.shutdown = true` is set unconditionally. Here, `store_proof_in_proof_manager` failure is silently ignored, yet the transaction is accepted unconditionally. Both create an irreversible state (pool marked shutdown / transaction in mempool without its proof) that breaks every subsequent operation that assumes the prior step succeeded.

### Impact Explanation

Every time the proposer must re-broadcast a previously built block that contains a client-side-proving invoke transaction whose proof was not stored (due to the swallowed error), `send_reproposal` returns `ReproposeError` and the round fails. Because the `valid_proposals` cache retains the same block for the same height, the proposer cannot recover without building a completely new block. If the proof manager experiences even a transient write failure at gateway admission time, the affected transaction permanently poisons any block it is included in for reproposal rounds, stalling consensus progress for that height.

### Likelihood Explanation

The proof manager uses filesystem-backed atomic rename (`write_proof_atomically`). Any transient I/O error (disk pressure, concurrent rename collision, permission issue) causes `set_proof` to return an error. The gateway's `store_proof_and_spawn_archiving` does not retry and does not reject the transaction. A single such event during a busy period is sufficient to trigger the condition.

### Recommendation

`store_proof_and_spawn_archiving` must propagate the proof-manager store failure back to `add_tx` and reject the transaction, mirroring how proof *verification* failure is handled (which correctly returns an error and does not forward the transaction). The fix is analogous to the recommended mitigation in the external report: only mark the operation as successful (accept the transaction) if the critical sub-operation (proof storage) also succeeded.

### Proof of Concept

1. Submit an `InvokeV3` transaction carrying non-empty `proof_facts` to the gateway.
2. Arrange for the proof manager's `set_proof` to fail transiently (e.g., inject an I/O error on the filesystem rename).
3. Observe: gateway logs `"Failed to set proof in proof manager"` but returns success; transaction enters the mempool.
4. The proposer builds a block containing this transaction and broadcasts it (first round may succeed because validators receive the proof inline in the `ConsensusTransaction`).
5. Trigger a reproposal (e.g., simulate a round timeout so the proposer re-enters `send_reproposal`).
6. Observe: `convert_internal_rpc_tx_to_rpc_tx` calls `get_proof` → `ProofNotFound` → `send_reproposal` returns `Err(ReproposeError)` → the proposer cannot re-broadcast the block → consensus round fails. [6](#0-5) [7](#0-6) [8](#0-7)

```rust
    async fn store_proof_and_spawn_archiving(
        &self,
        proof_data: Option<(ProofFacts, Proof)>,
        tx_hash: TransactionHash,
    ) -> ProofArchiveHandle {
        let (proof_facts, proof) = proof_data?;

        // Proof is verified during conversion to internal tx. It is stored here, after
        // validation, to avoid storing proofs for rejected transactions.
        let store_result = self
            .transaction_converter
            .store_proof_in_proof_manager(proof_facts.clone(), proof.clone())
            .await;
        match store_result {
            Ok(proof_manager_store_duration) => {
                GATEWAY_PROOF_MANAGER_STORE_LATENCY
                    .record(proof_manager_store_duration.as_secs_f64());
                info!(
                    "Proof manager store in the gateway took: {proof_manager_store_duration:?} \
                     for tx hash: {tx_hash:?}"
                );
            }
            Err(e) => {
                error!("Failed to set proof in proof manager: {}", e);
            }
        }

        let proof_archive_writer = self.proof_archive_writer.clone();
        let handle = tokio::spawn(async move {
            let proof_facts_hash = proof_facts.hash();
            let proof_archive_writer_start = Instant::now();
            let result = proof_archive_writer.set_proof(proof_facts, proof).await;
            let proof_archive_writer_duration = proof_archive_writer_start.elapsed();
            info!(
                "Proof archive writer took: {proof_archive_writer_duration:?} for tx hash: \
                 {tx_hash:?}"
            );
            (proof_facts_hash, result)
        });

        Some((handle, tx_hash))
    }
```

```rust
    async fn convert_internal_rpc_tx_to_rpc_tx(
        &self,
        tx: InternalRpcTransaction,
    ) -> TransactionConverterResult<RpcTransaction> {
        match tx.tx {
            InternalRpcTransactionWithoutTxHash::Invoke(tx) => {
                // We expect the proof to be available here because it has already been verified
                // and stored by the proof manager in the gateway.
                let proof = if tx.proof_facts.is_empty() {
                    Proof::default()
                } else {
                    self.get_proof(&tx.proof_facts).await?
                };

                Ok(RpcTransaction::Invoke(RpcInvokeTransaction::V3(RpcInvokeTransactionV3 {
                    resource_bounds: tx.resource_bounds,
                    signature: tx.signature,
                    nonce: tx.nonce,
                    tip: tx.tip,
                    paymaster_data: tx.paymaster_data,
                    nonce_data_availability_mode: tx.nonce_data_availability_mode,
                    fee_data_availability_mode: tx.fee_data_availability_mode,
                    sender_address: tx.sender_address,
                    calldata: tx.calldata,
                    account_deployment_data: tx.account_deployment_data,
                    proof_facts: tx.proof_facts,
                    proof,
                })))
            }
```

```rust
async fn send_reproposal(
    proposal_commitment: ProposalCommitment,
    init: ProposalInit,
    txs: Vec<Vec<InternalConsensusTransaction>>,
    finished_info: FinishedProposalInfo,
    next_l2_gas_price: GasPrice,
    stream_sender: &mut StreamSender,
    transaction_converter: Arc<dyn TransactionConverterTrait>,
) -> Result<(), ReproposeError> {
    stream_sender.send(ProposalPart::Init(init)).await?;
    for batch in txs.iter() {
        let transactions = futures::future::join_all(batch.iter().map(|tx| {
            // transaction_converter is an external dependency (class manager) and so
            // we can't assume success on reproposal.
            transaction_converter.convert_internal_consensus_tx_to_consensus_tx(tx.clone())
        }))
        .await
        .into_iter()
        .collect::<Result<Vec<_>, _>>()?;
        stream_sender.send(ProposalPart::Transactions(TransactionBatch { transactions })).await?;
    }
    let executed_transaction_count: u64 = finished_info
        .final_n_executed_txs
        .try_into()
        .expect("Number of executed transactions should fit in u64");
    let fin_payload = ProposalFinPayload {
        commitment_parts: CommitmentParts::from(&finished_info),
        l2_gas_info: L2GasInfo {
            next_l2_gas_price_fri: next_l2_gas_price,
            l2_gas_used: finished_info.l2_gas_used,
        },
    };
    let fin = ProposalFin {
        proposal_commitment,
        executed_transaction_count,
        fin_payload: Some(fin_payload),
    };
    stream_sender.send(ProposalPart::Fin(fin)).await?;

    Ok(())
}
```
