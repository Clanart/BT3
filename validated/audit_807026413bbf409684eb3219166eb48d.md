### Title
`execute_replace_class` Permits Setting Class Hash to `UNINITIALIZED_CLASS_HASH` (0), Enabling Re-deployment of Existing Contracts - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not validate that the requested new class hash is non-zero. Because `UNINITIALIZED_CLASS_HASH = 0` is the exact sentinel value the OS uses in `deploy_contract` to determine whether an address is available for deployment, a contract can call `replace_class(0)` to "uninitialize" itself. If the contract's nonce is also zero (the common case for non-account contracts), the address becomes re-deployable by any party who knows the original deployment parameters.

---

### Finding Description

**Root cause — missing zero-check in `execute_replace_class`:** [1](#0-0) 

The function reads `class_hash = request.class_hash` and immediately writes it into the contract's `StateEntry` with no validation. The only acknowledged gap is a TODO comment:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
```

There is no `assert class_hash != 0` or equivalent guard. Passing `class_hash = 0` is silently accepted.

**The sentinel value that is violated:** [2](#0-1) 

`UNINITIALIZED_CLASS_HASH = 0` is the protocol-level marker for "no contract deployed here."

**The initialization guard that is bypassed:** [3](#0-2) 

`deploy_contract` enforces two conditions before allowing deployment to an address:
1. `state_entry.class_hash = UNINITIALIZED_CLASS_HASH` (must be 0)
2. `state_entry.nonce = 0`

After a successful `replace_class(0)`, condition 1 is satisfied. For any contract that has never acted as a transaction sender (i.e., its nonce was never incremented by `check_and_increment_nonce`), condition 2 is also satisfied. Both conditions are met simultaneously, and the address is treated as undeployed.

**Nonce is only incremented for the account sender, not the called contract:** [4](#0-3) 

`check_and_increment_nonce` operates on `tx_info` (the sending account), not on the contract being called. Any contract invoked via `call_contract` or an L1 handler retains `nonce = 0` in its `StateEntry`.

---

### Impact Explanation

**Critical — Direct loss of funds.**

An attacker can re-deploy an arbitrary class to an address that previously held a legitimate contract (e.g., a token vault, multisig, or bridge escrow). The new deployment:

- Runs a fresh constructor against the **existing storage** of the original contract (because `deploy_contract` preserves `state_entry.storage_ptr`).
- Can overwrite ownership, authorization state, or any other storage slot the constructor touches.
- Gives the attacker full control over the contract's logic while the original funds remain in storage, enabling direct theft. [5](#0-4) 

---

### Likelihood Explanation

**Medium.** The attack requires:

1. A deployed contract (with `nonce = 0`) that executes `replace_class(0)` — either because it contains a malicious or buggy function that passes user-controlled input to `replace_class`, or because the contract intentionally self-destructs.
2. The attacker knowing the original deployment parameters (salt, class hash, constructor calldata, deployer address) to recompute the same address and submit a new deploy transaction.

Condition 2 is trivially satisfied for publicly visible on-chain deployments. Condition 1 is the primary constraint, but it is realistic: any contract with an upgradeable pattern that does not validate the new class hash before calling `replace_class` is vulnerable. The acknowledged TODO in the OS code confirms this check was intentionally deferred.

---

### Recommendation

Add an explicit non-zero check inside `execute_replace_class` before updating the state:

```cairo
// Reject class_hash = 0 (UNINITIALIZED_CLASS_HASH) to prevent re-initialization.
assert_not_zero(class_hash);
```

This mirrors the guard already present in `execute_declare_transaction`: [6](#0-5) 

where `assert_not_zero(compiled_class_hash)` is enforced before writing to the class tree.

---

### Proof of Concept

1. **Deploy** contract `C` at address `X` using parameters `(salt=S, class_hash=H, calldata=[], deployer=0)`. The OS sets `StateEntry(class_hash=H, nonce=0)` for `X`.

2. **Invoke** contract `X` via a `call_contract` syscall (so `X`'s nonce stays 0). Inside the execution, `X` calls the `replace_class` syscall with `class_hash = 0`.

3. **OS executes** `execute_replace_class` with `class_hash = 0`. No assertion fires. The OS writes `StateEntry(class_hash=0, nonce=0)` for `X`.

4. **Attacker submits** a deploy transaction with `(salt=S, class_hash=H, calldata=[], deploy_from_zero=TRUE)`. The address computation yields `X`. The OS calls `deploy_contract`, which checks `state_entry.class_hash == 0` ✓ and `state_entry.nonce == 0` ✓ — both pass.

5. **Attacker's constructor** runs against `X`'s existing storage. The attacker overwrites ownership or authorization slots and subsequently drains any funds held in `X`'s storage. [7](#0-6) [8](#0-7) [2](#0-1)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
