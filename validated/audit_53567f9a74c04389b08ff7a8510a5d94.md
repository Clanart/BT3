### Title
Missing shard_id binding in `set_state_header` allows cross-shard state root substitution — (`chain/chain/src/state_sync/adapter.rs`)

### Summary

`ChainStateSyncAdapter::set_state_header` validates a peer-supplied `ShardStateSyncResponseHeader` but never checks that the embedded chunk's `shard_id` matches the requested `shard_id`. An unprivileged snapshot-host peer can supply shard_X's chunk (with a valid Merkle proof at position X in the block) in response to a state-sync request for shard_Y. All five validation steps pass, and shard_Y's `StateHeaders` DB entry is written with shard_X's `prev_state_root` and `memory_usage`, corrupting the syncing node's state root and `num_state_parts` for shard_Y.

---

### Finding Description

`set_state_header` performs five validation steps:

**Step 1–2** — `validate_chunk_proofs` checks the chunk's internal hash, `tx_root`, and `receipts_root`. It does **not** check `chunk.shard_id()`. [1](#0-0) 

**Step 3a** — `verify_path` confirms the chunk hash is a leaf in `sync_prev_block_header.chunk_headers_root()`. The Merkle proof is position-specific (it encodes left/right siblings), so a proof for shard_X at position X is valid. The code **never checks that position X corresponds to shard_Y**. [2](#0-1) 

**Step 3b** — Same position-agnostic `verify_path` for `prev_chunk_header`; no shard_id binding. [3](#0-2) 

**Step 4** — Receipt validation computes `CryptoHash::hash_borsh(ReceiptList(shard_id, receipts))` using the **requested** `shard_id` (shard_Y). An attacker supplies the actual public incoming receipts for shard_Y with their real Merkle proofs; these are on-chain public data and satisfy steps 4e–4f against the trusted block headers. [4](#0-3) 

**Step 5** — `validate_state_root_node` checks `hash(state_root_node.data) == chunk_inner.prev_state_root()`. `chunk_inner` is extracted from the attacker-supplied chunk (shard_X's chunk), so this check passes when the attacker provides shard_X's `state_root_node`. **No check that `chunk_inner` belongs to shard_Y.** [5](#0-4) 

After all checks pass, the header is stored under key `StateHeaderKey(shard_Y, sync_hash)`: [6](#0-5) 

The exact corrupted values written to `DBCol::StateHeaders`:
- **state root**: `chunk_inner.prev_state_root()` from shard_X (not shard_Y)
- **memory_usage**: `state_root_node.memory_usage` from shard_X, used downstream to compute `num_state_parts` for shard_Y via `get_num_state_parts`

---

### Impact Explanation

Once the corrupted header is stored, `set_state_part` reads it back and validates incoming parts against shard_X's state root with shard_Y's `shard_id`: [7](#0-6) 

The syncing node requests the wrong number of parts (derived from shard_X's `memory_usage`) and validates them against shard_X's state root. If the attacker also serves shard_X's state parts, the node installs shard_X's committed state as shard_Y's state — a concrete cross-shard state substitution.

---

### Likelihood Explanation

Any peer can advertise as a snapshot host by sending a signed `SnapshotHostInfo` message — no validator or operator privilege is required. The syncing node selects from advertised snapshot hosts: [8](#0-7) 

The attacker needs only: (1) run a node and advertise as a snapshot host for shard_Y; (2) use shard_X's actual on-chain chunk (public); (3) supply the actual incoming receipts for shard_Y (public on-chain data); (4) supply shard_X's `state_root_node` (obtainable from any archive node). No cryptographic forgery is required.

---

### Recommendation

In `set_state_header`, after extracting `chunk_inner`, add an explicit shard_id binding check:

```rust
let chunk_inner = chunk.take_header().take_inner();
// ADD: verify the chunk belongs to the requested shard
if chunk_inner.shard_id() != shard_id {
    return Err(Error::Other(
        "set_shard_state failed: chunk shard_id does not match requested shard_id".into(),
    ));
}
```

This check should be placed before step 5 (or immediately after extracting `chunk_inner`). [9](#0-8) 

---

### Proof of Concept

1. Set up a two-shard network (shard_0, shard_1) and let it run for a few epochs.
2. At the sync epoch boundary, obtain the `sync_hash` block.
3. From the `sync_prev_block`, extract shard_0's chunk and its Merkle proof at position 0.
4. Obtain shard_0's `state_root_node` from an archive node.
5. Collect the actual incoming receipts for shard_1 from the chain (public data).
6. Construct a `ShardStateSyncResponseHeaderV2` with shard_0's chunk/proof/state_root_node but shard_1's receipt proofs.
7. Call `set_state_header(shard_1, sync_hash, crafted_header)`.
8. Assert: the stored header's `state_root_node.memory_usage` equals shard_0's memory_usage (not shard_1's), and `get_num_state_parts(stored.memory_usage)` differs from shard_1's actual `num_state_parts`.

pub fn validate_chunk_proofs(
    chunk: &ShardChunk,
    epoch_manager: &dyn EpochManagerAdapter,
) -> Result<bool, Error> {
    let correct_chunk_hash = chunk.compute_header_hash();

    // 1. Checking chunk.header.hash
    let header_hash = chunk.header_hash();
    if header_hash != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }

    // 2. Checking that chunk body is valid
    // 2a. Checking chunk hash
    if chunk.chunk_hash() != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }
    let height_created = chunk.height_created();
    let outgoing_receipts_root = chunk.prev_outgoing_receipts_root();
    let (transactions, receipts) = (chunk.to_transactions(), chunk.prev_outgoing_receipts());

    // 2b. Checking that chunk transactions are valid
    let (tx_root, _) = merklize(transactions);
    if &tx_root != chunk.tx_root() {
        byzantine_assert!(false);
        return Ok(false);
    }
    // 2c. Checking that chunk receipts are valid
    if height_created == 0 {
        return Ok(receipts.is_empty() && outgoing_receipts_root == &CryptoHash::default());
    } else {
        let shard_layout = {
            let prev_block_hash = chunk.prev_block_hash();
            epoch_manager.get_shard_layout_from_prev_block(&prev_block_hash)?
        };
        let outgoing_receipts_hashes = Chain::build_receipts_hashes(receipts, &shard_layout)?;
        let (receipts_root, _) = merklize(&outgoing_receipts_hashes);
        if &receipts_root != outgoing_receipts_root {
            byzantine_assert!(false);
            return Ok(false);
        }
    }
    Ok(true)
