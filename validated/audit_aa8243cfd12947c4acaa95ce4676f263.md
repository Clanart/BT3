### Title
Missing Declared-Class Validation in `replace_class` Syscall Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS does not verify that the new class hash supplied by a contract corresponds to a previously declared class. An unprivileged contract can call `replace_class` with an arbitrary, undeclared class hash. The OS will commit this state transition to the global state without any validation, permanently breaking the contract and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall. After deducting gas, it reads the current `StateEntry` for the calling contract and writes a new `StateEntry` with the caller-supplied `class_hash` directly into `contract_state_changes`, with no check that the new class hash has ever been declared on-chain:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The acknowledged TODO at line 898 confirms the missing check is known but unimplemented. The `contract_class_changes` dictionary, which tracks declared class hashes (keyed by class hash, value = compiled class hash), is never consulted during `replace_class` execution.

By contrast, the `execute_declare_transaction` function correctly enforces that a class can only be declared once by using `dict_update` with `prev_value=0`:

```cairo
// Note that prev_value=0 enforces that a class may be declared only once.
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [2](#0-1) 

No equivalent guard exists in `execute_replace_class`. The OS will produce a valid proof for a block that contains a `replace_class` call with a non-existent class hash, and the resulting state will be committed to the chain.

---

### Impact Explanation

Once a contract's class hash is replaced with an undeclared hash, every future call to that contract will fail at class resolution time because no bytecode exists for that hash. Any assets held in the contract's storage (ETH, ERC20 balances, NFTs, etc.) become permanently inaccessible. This constitutes **permanent freezing of funds**, which is in the allowed impact scope.

---

### Likelihood Explanation

The attack path is fully reachable by an unprivileged actor:

1. An attacker deploys a contract (or exploits an existing contract with a user-accessible `replace_class` code path).
2. The contract accumulates funds (e.g., acts as a vault or wallet).
3. The attacker (or a malicious contract logic path) calls `replace_class` with an arbitrary felt value that has never been declared as a class hash.
4. The OS executes `execute_replace_class`, writes the undeclared hash into `contract_state_changes`, and produces a valid proof.
5. The state is committed; the contract is permanently broken and all funds are frozen.

No privileged role, leaked key, or external dependency is required. The `replace_class` syscall is a standard, publicly accessible syscall available to any Sierra contract.

---

### Recommendation

Inside `execute_replace_class`, after reading the `class_hash` from the request, perform a lookup in `contract_class_changes` (or an equivalent declared-class registry) to confirm the hash maps to a non-zero compiled class hash before writing the new `StateEntry`. This mirrors the invariant already enforced in `execute_declare_transaction` via `prev_value=0` in `dict_update`. The TODO comment at line 898 already identifies this gap; it must be resolved before the syscall is considered safe.

---

### Proof of Concept

1. Declare class `A` (valid, compiled class hash `H_A`).
2. Deploy contract `C` using class `A`; fund it with tokens.
3. From within `C`, invoke `replace_class(class_hash=0xdeadbeef)` where `0xdeadbeef` has never been declared.
4. `execute_replace_class` reads `state_entry` for `C`, constructs `new StateEntry(class_hash=0xdeadbeef, ...)`, and calls `dict_update` — no validation of `0xdeadbeef` against `contract_class_changes` occurs.
5. The OS produces a valid proof; the block is accepted; `C`'s class hash is now `0xdeadbeef` on-chain.
6. Any subsequent call to `C` fails at class resolution; all funds in `C` are permanently frozen. [3](#0-2)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
