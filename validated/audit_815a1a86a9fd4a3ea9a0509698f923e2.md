### Title
`add_events()` Panics on Duplicate `TransactionConsumed` L1 Event, Crashing the L1 Provider — (File: `crates/apollo_l1_provider/src/l1_provider.rs`)

---

### Summary

`add_events()` iterates over a batch of L1 events in a `for` loop. When it encounters a `TransactionConsumed` event for a transaction that was already marked consumed (e.g., due to an L1 reorg causing the scraper to re-deliver the same `ConsumedMessageToL2` event), it calls `panic!()` unconditionally. This terminates the L1 provider process, halting all L1 handler transaction admission until the service is manually restarted. The analog to the external report is exact: a loop over external items, a single already-processed item causes the whole loop to abort, and recovery requires operator intervention.

---

### Finding Description

In `add_events()`, every event type except `TransactionConsumed` is handled gracefully:

- `L1HandlerTransaction` — `add_tx()` silently ignores a duplicate full payload with a `warn!`.
- `TransactionCancellationStarted` — duplicate cancellation requests are logged and dropped with `continue`.
- `TransactionCanceled` — idempotent removal.

Only `TransactionConsumed` is fatal:

```rust

        // log and ignore.

            "Double consumption of {tx_hash} at {consumed_at}, previously \
             consumed at {previously_consumed_at}."

}

`consume_tx()` in the transaction manager returns `Err(previously_consumed_at)` whenever the record already carries a `consumed_at` timestamp:

```rust

The consumed record is only removed from `records` after the `l1_handler_consumption_timelock_seconds` expires via `clear_old_tx_from_consumed_queue`. Until that window closes, any re-delivery of the same `ConsumedMessageToL2` event hits the `Err` branch and panics. [3](#0-2)

A single bad event in the batch aborts the entire loop and crashes the process — identical in structure to the Frax `depositEther()` loop reverting on one bad validator.

---

### Impact Explanation

The L1 provider is the sole source of `L1HandlerTransaction` objects for the batcher's `ProposeTransactionProvider`. A crash prevents the batcher from calling `get_txs()` for any subsequent block proposal, effectively rejecting all pending L1 handler transactions from sequencing until the service is restarted. [6](#0-5) 

If the scraper's last-scraped cursor is behind the re-delivered event (e.g., after a reorg rollback), the same panic will recur on every restart until the cursor advances past the offending block — matching the "inoperable until governance pops the bad validator" pattern from the external report.

Matches: **High — Mempool/gateway/RPC admission rejects valid transactions before sequencing.**

---

### Likelihood Explanation

Ethereum L1 reorgs of 1–2 blocks occur regularly. The scraper polls `get_latest_block()` and re-scrapes `events(last_scraped..latest, ...)` on each iteration. After a reorg, `last_scraped` is rolled back and the same `ConsumedMessageToL2` event is re-delivered to `add_events()`. If the `l1_handler_consumption_timelock_seconds` has not yet expired for that transaction, the record is still present, `consume_tx` returns `Err`, and the panic fires. No privileged access is required; a natural short reorg suffices. [7](#0-6) 

---

### Recommendation

Mirror the graceful handling already applied to `add_tx` for double-scrapes: replace the `panic!` with a `warn!` and `continue`, consistent with the existing TODO comment acknowledging the uncertainty:

```rust
Event::TransactionConsumed { tx_hash, timestamp: consumed_at } => {
    if let Err(previously_consumed_at) =
        self.tx_manager.consume_tx(tx_hash, consumed_at, self.clock.unix_now())
    {
        warn!(
            "Duplicate ConsumedMessageToL2 for {tx_hash} at {consumed_at}; \
             previously consumed at {previously_consumed_at}. Ignoring."
        );
    }
}
```

This matches the pattern already used for `TransactionCancellationStarted` duplicates. [8](#0-7) 

---

### Proof of Concept

1. L1 handler tx `H` is scraped; `add_tx()` adds it to the proposable index.
2. `H` is consumed on L1; scraper delivers `Event::TransactionConsumed { tx_hash: H, timestamp: T1 }` to `add_events()`.
3. `consume_tx()` succeeds; record for `H` now has `consumed_at = T1`. The consumption timelock (e.g., 1000 s) has not expired.
4. A 1-block L1 reorg occurs; the scraper rolls `last_scraped` back by one block and re-scrapes.
5. The same `ConsumedMessageToL2` event is re-delivered: `Event::TransactionConsumed { tx_hash: H, timestamp: T1 }`.
6. `consume_tx()` finds `record.consumed_at = Some(T1)` and returns `Err(T1)`.
7. `add_events()` hits `panic!("Double consumption of H …")` — the L1 provider process terminates.
8. The batcher's next `get_txs()` call fails; no L1 handler transactions are included in subsequent blocks until the service is restarted and the scraper cursor advances past the reorged block. [9](#0-8) [10](#0-9)

    pub fn get_txs(
        &mut self,

    ) -> L1ProviderResult<Vec<L1HandlerTransaction>> {
        if self.state.is_uninitialized() {
            return Err(L1ProviderError::Uninitialized);

        self.check_height_with_error(height)?;

        match self.state {

                let txs = self.tx_manager.get_txs(n_txs, self.clock.unix_now());
                info!(

                    txs.len(),
                    n_txs

                            "L2 tx hash: {}, L1-L2 msg hash: {}",
                            tx.tx_hash,
                            tx.tx.calc_msg_hash()
                        ))
                        .collect::<Vec<_>>()
                );
                Ok(txs)
            }
            _ => Err(L1ProviderError::UnexpectedProviderState {
                expected: ProviderState::Propose,

    pub fn get_txs(&mut self, n_txs: usize, now: u64) -> Vec<L1HandlerTransaction> {
        // Oldest        Now.sub(timelock)     Newest       Now
        //  |<---  passed  --->|                 |           |
        //  |<--- cooldown --->|                 |           |
        // t-------------------------------------------------->
        let cutoff = now.saturating_sub(self.config.l1_handler_proposal_cooldown_seconds.as_secs());
        let past_cooldown_txs = self.proposable_index.range(..cutoff);

        // Linear scan, but we expect this to be a small number of transactions (< 10 roughly).
        let unstaged_tx_hashes: Vec<_> = past_cooldown_txs
            .flat_map(|(_timestamp, tx_hashes)| tx_hashes.iter())
            .skip_while(|&&tx_hash| self.is_staged(tx_hash))
            .take(n_txs)
            .copied()
            .collect();

        for &tx_hash in unstaged_tx_hashes.iter() {
            let record = self.records.get(&tx_hash).expect("transaction should exist");
            assert_eq!(
                record.state,
                TransactionState::Pending,
                "Transaction {tx_hash} has state {:?}. Only Pending transactions should be in the \
                 proposable index.",
                record.state
            );
        }

        let mut txs = Vec::with_capacity(n_txs);
        let current_staging_epoch = self.current_staging_epoch; // borrow-checker constraint.
        for tx_hash in unstaged_tx_hashes {
            let newly_staged =
                self.with_record(tx_hash, |record| record.try_mark_staged(current_staging_epoch));
            assert_eq!(
                newly_staged,
                Some(true),
                "Inconsistent storage state: indexed l1 handler {tx_hash} is not in storage or \
                 wasn't marked as staged."
            );

            txs.push(self.records[&tx_hash].get_unchecked().clone());

    pub fn consume_tx(
        &mut self,
        tx_hash: TransactionHash,
        consumed_at: BlockTimestamp,
        unix_now: u64,
    ) -> Result<(), BlockTimestamp> {
        self.clear_old_tx_from_consumed_queue(unix_now);

        let Some(record) = self.records.get(&tx_hash) else {
            debug!(
                "Attempted to consume an unknown transaction: {tx_hash}. This can happen if the \
                 transaction was too old to be scraped (e.g. it was created before we started \
                 scraping)."
            );
            return Ok(());
        };

        // Double consumption is a bug.
        if let Some(previously_consumed_at) = record.get_consumed_at_timestamp() {
            return Err(previously_consumed_at);
        }

        // Mark the transaction as consumed.
        self.with_record(tx_hash, |record| record.mark_consumed(consumed_at));
        Ok(())
    }

    pub fn clear_old_tx_from_consumed_queue(&mut self, unix_now: u64) {
        let cutoff =
            unix_now.saturating_sub(self.config.l1_handler_consumption_timelock_seconds.as_secs());
        let still_timelocked = self.consumed_queue.split_off(&BlockTimestamp(cutoff));
        let passed_timelock = std::mem::replace(&mut self.consumed_queue, still_timelocked);
        for tx_hashes in passed_timelock.values() {
            for tx_hash in tx_hashes {
                self.records.remove(tx_hash);
            }
        }
