### Title
`execute_meta_tx_v0` Omits Nonce from Transaction Hash, Enabling Unbounded Replay of Signed Meta-Transactions — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `meta_tx_v0` syscall constructs a version-0 transaction context with `nonce=0` and computes a transaction hash that contains no nonce and no block-specific data. Because the OS explicitly skips nonce enforcement for version-0 transactions, and because `__validate__` is never invoked for meta-tx calls, the same signed meta-transaction can be replayed an unlimited number of times by any party who observed the original signature. This is a direct analog of the reported nonce-reset replay vulnerability: instead of a nonce being reset to zero on ownership transfer, the nonce is structurally absent from the meta-tx hash, producing the same replay-enabling condition.

---

### Finding Description

**Root cause 1 — nonce omitted from hash.**

`compute_meta_tx_v0_hash` in `transaction_hash.cairo` calls `deprecated_get_transaction_hash` with `additional_data_size=0`, meaning the nonce is never hashed into the transaction commitment:

```cairo
// transaction_hash/transaction_hash.cairo lines 295-314
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(...) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        tx_hash_prefix=INVOKE_HASH_PREFIX,
        version=0,
        contract_address=contract_address,
        entry_point_selector=entry_point_selector,
        calldata_size=calldata_size,
        calldata=calldata,
        max_fee=0,
        chain_id=chain_id,
        additional_data_size=0,          // ← nonce absent
        additional_data=cast(0, felt*),
    );
    return tx_hash;
}
```

The resulting hash is a pure function of `(chain_id, contract_address, selector, calldata)`. It is identical across every block and every invocation.

**Root cause 2 — OS nonce enforcement skipped for version 0.**

`check_and_increment_nonce` in `execute_transaction_utils.cairo` explicitly returns early for version-0 transactions, so the account's on-chain nonce is never read or incremented:

```cairo
// execute_transaction_utils.cairo lines 63-67
func check_and_increment_nonce{contract_state_changes: DictAccess*}(tx_info: TxInfo*) -> () {
    // Do not handle nonce for version 0.
    if (tx_info.version == 0) {
        return ();
    }
    ...
}
```

**Root cause 3 — `__validate__` is never called.**

`execute_meta_tx_v0` in `syscall_impls.cairo` constructs the execution context with `version=0, nonce=0` and immediately calls `contract_call_helper`, bypassing `run_validate` entirely:

```cairo
// syscall_impls.cairo lines 343-393
tempvar new_tx_info = new TxInfo(
    version=0,
    ...
    nonce=0,          // ← always zero
    ...
);
...
contract_call_helper(
    remaining_gas=remaining_gas,
    block_context=block_context,
    execution_context=execution_context,
);
```

`run_validate` itself also skips validation for version 0 (`execute_transaction_utils.cairo` lines 127-130), but it is not even reached here.

**Combined effect.** The hash presented to the target account's `__execute__` is permanently fixed for a given `(chain_id, contract_address, selector, calldata)` tuple. Any signature that was valid once will remain valid forever, and the OS provides no mechanism to detect or prevent re-execution.

---

### Impact Explanation

**Critical — Direct loss of funds.**

The `meta_tx_v0` syscall is the StarkNet mechanism for gasless/relayed transactions. A relayer contract accepts a user-supplied signature and forwards it via `meta_tx_v0` to the user's account contract, which executes `__execute__`. Standard account contracts verify the signature against the transaction hash. Because the hash never changes for the same calldata, a captured signature can be submitted to the relayer (or any contract that accepts the same parameters) in any future block, causing `__execute__` to run again with the same calldata. If that calldata encodes a token transfer or other value-moving operation, the attacker can drain the account by replaying the call repeatedly until the account balance is exhausted.

---

### Likelihood Explanation

**Low–Medium.**

The attack requires: (a) a deployed relayer contract that accepts user-provided signatures and passes them to `meta_tx_v0`; (b) the attacker to observe at least one successful meta-tx call on-chain. Both conditions are realistic in production: gasless transaction relayers are a primary use case for `meta_tx_v0`, and all on-chain calldata and signatures are publicly visible. No privileged access, leaked key, or operator cooperation is required.

---

### Recommendation

Include the account's current on-chain nonce in the `compute_meta_tx_v0_hash` computation (analogous to how `additional_data` carries the nonce in regular v1/v3 transactions), and enforce that the OS increments the account nonce after each successful `meta_tx_v0` execution. This mirrors the fix recommended in the original report: do not allow a nonce-free execution path that can be replayed.

---

### Proof of Concept

1. **Setup**: Deploy a relayer contract `R` that accepts `(calldata, signature)` from any caller and invokes `meta_tx_v0(contract_address=A, selector=EXECUTE, calldata=calldata, signature=signature)`.

2. **Victim action**: User `U` signs a meta-transaction to transfer 100 tokens from account `A` to address `B`. The signed hash is:
   ```
   H(INVOKE_PREFIX, 0, A, EXECUTE_SELECTOR, [transfer_calldata], 0, chain_id)
   ```
   `U` submits `(calldata, sig)` to `R`. The OS processes the meta-tx: `A.__execute__` is called, 100 tokens are transferred. The OS does **not** increment `A`'s nonce (version=0 path, `execute_transaction_utils.cairo` line 65).

3. **Replay**: Attacker `X` calls `R` again with the identical `(calldata, sig)`. The OS recomputes the same hash (no nonce, no block data), `A.__execute__` verifies the same signature against the same hash, and executes again. Another 100 tokens are transferred.

4. **Iteration**: `X` repeats until `A` is drained. Each replay costs only the gas of calling `R`; no new signature from `U` is needed.
