### Title
Malformed `recipient` in StarkNet `init_transfer` permanently locks user tokens — (`near/omni-types/src/starknet/events.rs`)

---

### Summary

The StarkNet `init_transfer` function accepts any `ByteArray` as `recipient` with no format validation. The NEAR-side `parse_init_transfer` strictly validates the same field via `OmniAddress::from_str`. If the string is valid UTF-8 but fails `OmniAddress` parsing (unknown chain prefix, malformed EVM hex, invalid Solana base58, etc.), the prover returns an error, `fin_transfer_callback` panics, and the consumed StarkNet nonce can never be settled on NEAR. The user's tokens are permanently locked/burned on StarkNet with no recovery path.

---

### Finding Description

**Step 1 — No recipient validation on StarkNet.**

`init_transfer` in `starknet/src/omni_bridge.cairo` accepts `recipient: ByteArray` and only checks `amount > 0` and `fee < amount`. The recipient string is emitted verbatim in the `InitTransfer` event. [1](#0-0) 

**Step 2 — Strict parsing on the NEAR side.**

`parse_init_transfer` in `near/omni-types/src/starknet/events.rs` calls:

```rust
let recipient: OmniAddress = recipient_str.parse().map_err(stringify)?;
```

`OmniAddress::from_str` returns `Err` for any unrecognised chain prefix or malformed address part. [2](#0-1) 

The full set of accepted prefixes is `eth`, `near`, `sol`, `arb`, `base`, `bnb`, `pol`, `hlevm`, `abs`, `btc`, `zcash`, `strk`, `fogo`. Any other prefix (e.g. `ethereum:`, `polygon:`, `avax:`) or a malformed address part returns `Err("Chain X is not supported")` or a parse error. [3](#0-2) 

**Step 3 — Prover propagates the error.**

`verify_callback` in `mpc-omni-prover` is annotated `#[handle_result]`. When `parse_starknet_result` → `parse_init_transfer` returns `Err`, the function returns `Err(String)`, which the NEAR SDK converts to a panicking promise result (`PromiseResult::Failed`). [4](#0-3) 

**Step 4 — `fin_transfer_callback` panics unconditionally.**

`decode_prover_result` returns `Err(PromiseError::Failed)` on a failed promise. The callback's `let Ok(...) = ... else { env::panic_str(...) }` arm fires, aborting the entire NEAR transaction. [5](#0-4) [6](#0-5) 

**Step 5 — No recovery mechanism exists.**

There is no admin function, user-callable refund, or alternative settlement path for a StarkNet `InitTransfer` whose NEAR-side proof verification fails. The `origin_nonce` on StarkNet is already consumed (incremented before the token transfer), and the tokens are already burned or locked in the StarkNet contract. [7](#0-6) 

---

### Impact Explanation

**Critical — Permanent, irrecoverable lock of user funds on StarkNet.**

Any user who calls `init_transfer` with a recipient string that is valid UTF-8 but fails `OmniAddress::from_str` (e.g. `"ethereum:0xABCD..."`, `"polygon:0xABCD..."`, `"avax:0xABCD..."`, `"eth:0x123"` (too short), or any unsupported chain prefix) will have their tokens permanently locked/burned on StarkNet with no settlement possible on NEAR.

---

### Likelihood Explanation

**Medium.** The StarkNet contract is a public interface with no documentation or on-chain enforcement of the `chain:address` format. A user interacting directly with the contract (e.g. via a script, CLI, or a third-party UI) can trivially supply a plausible-looking but unsupported recipient string. Common mistakes include using `"ethereum:"` instead of `"eth:"`, `"polygon:"` instead of `"pol:"`, or omitting the chain prefix entirely for a non-EVM address. Each such mistake results in permanent fund loss.

---

### Recommendation

Add recipient format validation in the StarkNet `init_transfer` function before consuming the nonce and transferring tokens. The simplest approach is to require the recipient to match a known `chain:address` pattern on-chain (e.g. check that it contains `:` and that the prefix is one of the supported chain identifiers). Alternatively, validate the recipient off-chain in the relayer/UI layer and reject malformed inputs before submission, but this is not a sufficient on-chain fix.

pub fn parse_init_transfer(
    from_address: &[u8; 32],
    keys: &[[u8; 32]],
    data: &[[u8; 32]],
) -> Result<InitTransferMessage, String> {
    if keys.len() < 4 {
        return Err(format!(
            "InitTransfer: expected at least 4 keys, got {}",
            keys.len()
        ));
    }
    if keys[0] != INIT_TRANSFER_SELECTOR {
        return Err("InitTransfer: selector mismatch".to_string());
    }

    let sender = OmniAddress::Strk(H256(keys[1]));
    let token = OmniAddress::Strk(H256(keys[2]));
    let origin_nonce = felt_to_u64(&keys[3])?;

    let mut cursor = FeltCursor::new(data);
    let amount = cursor.read_u128()?;
    let fee = cursor.read_u128()?;
    let native_fee = cursor.read_u128()?;
    let recipient_str = cursor.read_byte_array()?;
    let msg = cursor.read_byte_array()?;

    let emitter_address = OmniAddress::Strk(H256(*from_address));
    let recipient: OmniAddress = recipient_str.parse().map_err(stringify)?;

    Ok(InitTransferMessage {
        origin_nonce,
        token,
        amount: near_sdk::json_types::U128(amount),
        recipient,
        fee: Fee {
            fee: near_sdk::json_types::U128(fee),
            native_fee: near_sdk::json_types::U128(native_fee),
        },
        sender,
        msg,
        emitter_address,
    })
}
