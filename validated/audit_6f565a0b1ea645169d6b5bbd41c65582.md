### Title
`compute_meta_tx_v0_hash` Omits Nonce from Hash, Enabling Unbounded Signature Replay — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/transaction_hash/transaction_hash.cairo`)

---

### Summary

`compute_meta_tx_v0_hash` computes a transaction hash that does not include any nonce or sequencing field. Because the hash is fully determined by `(contract_address, entry_point_selector, calldata, chain_id)`, a signature that was valid for one meta_tx_v0 invocation remains valid for every future invocation with the same parameters. An attacker who observes a signed meta_tx_v0 call can replay it an unlimited number of times across arbitrary outer transactions, causing direct, repeated loss of funds from the victim account.

---

### Finding Description

**Root cause — missing nonce in hash:**

`compute_meta_tx_v0_hash` delegates to `deprecated_get_transaction_hash` with `additional_data_size=0`:

```cairo
// transaction_hash/transaction_hash.cairo  lines 295-315
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(
    contract_address: felt,
    entry_point_selector: felt,
    calldata: felt*,
    calldata_size: felt,
    chain_id: felt,
) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        ...
        additional_data_size=0,          // ← no nonce
        additional_data=cast(0, felt*),  // ← no nonce
    );
    return tx_hash;
}
``` [1](#0-0) 

Compare with `compute_l1_handler_transaction_hash`, which correctly passes the nonce as `additional_data`:

```cairo
// lines 220-238
func compute_l1_handler_transaction_hash{pedersen_ptr: HashBuiltin*}(
    execution_context: ExecutionContext*, chain_id: felt, nonce: felt
) -> felt {
    let (transaction_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        ...
        additional_data_size=1,   // ← nonce included
        additional_data=&nonce,
    );
    return transaction_hash;
}
``` [2](#0-1) 

**No nonce check or increment in `execute_meta_tx_v0`:**

`execute_meta_tx_v0` hard-codes `nonce=0` in the synthesised `TxInfo` and never calls `check_and_increment_nonce`:

```cairo
// syscall_impls.cairo  lines 343-363
tempvar new_tx_info = new TxInfo(
    version=0,
    ...
    transaction_hash=meta_tx_hash,
    nonce=0,          // ← always zero, never incremented
    ...
);
``` [3](#0-2) 

`check_and_increment_nonce` explicitly skips version-0 transactions:

```cairo
// execute_transaction_utils.cairo  lines 63-67
func check_and_increment_nonce{...}(tx_info: TxInfo*) -> () {
    if (tx_info.version == 0) {
        return ();   // ← nonce never checked or incremented for meta_tx_v0
    }
    ...
}
``` [4](#0-3) 

**`__validate__` is also skipped for version-0:**

`run_validate` returns immediately for version-0 transactions, so the inner contract's validation entry point is never invoked:

```cairo
// execute_transaction_utils.cairo  lines 127-130
if (tx_execution_info.tx_info.version == 0) {
    return ();
}
``` [5](#0-4) 

The inner contract's `__execute__` entry point is called directly via `contract_call_helper`. The inner contract reads `tx_info.transaction_hash` and `tx_info.signature_*` through `get_execution_info` to perform its own signature check. Because the hash is nonce-free, any previously captured `(calldata, signature)` pair is permanently valid. [6](#0-5) 

---

### Impact Explanation

**Critical — Direct loss of funds.**

A v0 account contract that authorises a token transfer (or any state-mutating call) via meta_tx_v0 produces a signature over a hash that is permanently valid for the same `(contract_address, selector, calldata, chain_id)` tuple. An attacker who observes the signature can replay the identical meta_tx_v0 call from any outer transaction they control, causing the inner contract to re-execute the authorised action (e.g., transfer tokens) an unlimited number of times until the account is drained.

---

### Likelihood Explanation

**Medium-High.** The meta_tx_v0 syscall is callable from any deployed contract. An attacker only needs to:
1. Observe a single on-chain meta_tx_v0 invocation (calldata and signature are public in the transaction trace).
2. Deploy a thin wrapper contract that re-issues the same `meta_tx_v0` syscall with the captured signature.
3. Submit an outer invoke transaction calling that wrapper.

No privileged access, leaked key, or operator cooperation is required. The only prerequisite is that a victim account has previously used meta_tx_v0 with a signature over a call that has side-effects (e.g., a transfer).

---

### Recommendation

Include a nonce in `compute_meta_tx_v0_hash`, exactly as `compute_l1_handler_transaction_hash` does:

```cairo
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(
    contract_address: felt,
    entry_point_selector: felt,
    calldata: felt*,
    calldata_size: felt,
    chain_id: felt,
    nonce: felt,          // ← add nonce parameter
) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        ...
        additional_data_size=1,   // ← include nonce
        additional_data=&nonce,
    );
    return tx_hash;
}
```

`execute_meta_tx_v0` must then read and increment the target contract's nonce (from `contract_state_changes`) before computing the hash, and pass it both to the hash function and into the synthesised `TxInfo.nonce` field.

### Proof of Concept

1. Alice's v0 account at address `A` holds 1000 tokens.
2. Alice issues an outer invoke transaction that calls a relay contract `R`.
3. `R` calls `meta_tx_v0(contract_address=A, selector=__execute__, calldata=[transfer, Bob, 100], signature=sig_alice)`.
4. The OS computes `H = Pedersen(INVOKE_PREFIX, 0, A, __execute__, hash(calldata), 0, chain_id)` — **no nonce**.
5. `A.__execute__` verifies `sig_alice` against `H` and transfers 100 tokens to Bob. Transaction succeeds.
6. Attacker Eve observes `calldata` and `sig_alice` from the public transaction trace.
7. Eve deploys contract `E` that calls `meta_tx_v0(contract_address=A, selector=__execute__, calldata=[transfer, Eve, 100], signature=sig_alice)`.

   Wait — Eve must use the *exact same calldata* to get the same hash. So Eve replays the *exact same calldata* `[transfer, Bob, 100]` with `sig_alice`.

8. The OS recomputes `H` identically (same inputs, no nonce). `A.__execute__` verifies `sig_alice` against `H` — **still valid**. Another 100 tokens are transferred to Bob (or Eve if she controls Bob, or she can front-run Alice's future identical calls).
9. Eve repeats step 7–8 until Alice's account is drained.

The attack is bounded only by Alice's token balance and the gas Eve is willing to spend. No cryptographic break is required.
