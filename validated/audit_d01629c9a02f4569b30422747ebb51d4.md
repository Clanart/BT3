### Title
Missing Class Hash Validation in `execute_replace_class` Enables Permanent Fund Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the new class hash provided by a contract corresponds to a declared (compiled) class before committing the state update. This missing validation — explicitly acknowledged with a TODO comment in the code — is the direct analog to M-18's pattern: a required precondition check is skipped before a state-modifying operation executes. Any contract can call `replace_class` with an arbitrary, undeclared felt value, permanently setting its class hash to an invalid entry. Any funds held in that contract become permanently inaccessible.

---

### Finding Description

In `execute_replace_class` (lines 878–916 of `syscall_impls.cairo`), the OS reads the new class hash from the syscall request and immediately updates `contract_state_changes` without verifying that the new class hash exists in the set of declared compiled classes:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The structural parallel to M-18 is exact:

| M-18 (Astaria) | StarkNet OS Analog |
|---|---|
| `buyoutLien` modifies vault accounting without first calling `transferWithdrawReserve()` | `execute_replace_class` modifies contract class-hash state without first verifying the new hash is declared |
| Required precondition: transfer reserve funds to withdrawProxy | Required precondition: confirm new class hash exists in compiled class facts |
| Result: withdrawProxy receives insufficient funds; LPs permanently locked out | Result: contract class hash set to invalid value; all callers permanently locked out |

The OS validates compiled class facts post-execution via `validate_compiled_class_facts_post_execution`: [2](#0-1) 

However, this validation only checks that the compiled class facts *used during execution* are consistent. It does not retroactively validate that a class hash written into `contract_state_changes` via `replace_class` corresponds to any real compiled class. Once the block is proven and the state committed, the invalid class hash is canonical. In every subsequent block, the sequencer cannot supply a compiled class for a non-existent hash, so every call to the affected contract fails at class lookup — permanently.

---

### Impact Explanation

**Critical. Permanent freezing of funds.**

After `replace_class` is called with an undeclared hash and the block is proven:

1. The contract's class hash in the committed state is set to an arbitrary invalid value.
2. In all future blocks, the OS reads this class hash from state and attempts to look it up in the compiled class facts bundle.
3. No compiled class exists for this hash; every call to the contract fails.
4. All ERC-20 balances, ETH, or other assets stored in the contract's storage are permanently inaccessible with no recovery path.

The `execute_blocks` function in `os.cairo` shows that `state_update` is called after all transactions execute, committing the corrupted class hash to the final state root: [3](#0-2) 

---

### Likelihood Explanation

The `replace_class` syscall is available to any contract executing on StarkNet. No privileged role is required:

1. An unprivileged user deploys a contract (standard `deploy_account` or `deploy` syscall — both are processed by `execute_deploy_account_transaction` or `execute_deploy` without any special permission).
2. The contract calls `replace_class` with an arbitrary felt (e.g., `0x1`) that has no corresponding compiled class.
3. `execute_replace_class` processes this without validation.
4. The block is proven; the invalid class hash is committed to state.

The `execute_deploy` syscall handler in `syscall_impls.cairo` confirms that any caller can deploy a contract: [4](#0-3) 

For multi-user impact (matching the "funds of other users frozen" pattern from M-18): a contract acting as a shared vault or token contract — where multiple users have deposited assets — can have its class replaced with an invalid hash by anyone who can invoke the relevant function, permanently freezing all depositors' funds.

---

### Recommendation

Before updating `contract_state_changes` in `execute_replace_class`, add an assertion that `class_hash` exists in `contract_class_changes` (declared in the current block) or was previously declared (exists in the pre-block state's class mapping). This is precisely the check the TODO comment calls for. The fix mirrors how `execute_declare_transaction` enforces `prev_value=0` to prevent double-declaration: [5](#0-4) 

A symmetric enforcement — asserting `new_value != 0` (i.e., the class is declared) — should be applied in `execute_replace_class` before the `dict_update` call.

### Proof of Concept

```
1. Attacker deploys contract C that holds user deposits and exposes:
      fn freeze() { replace_class(0x1); }  // 0x1 is not a declared class

2. Users deposit funds into C.

3. Attacker submits invoke tx calling C.freeze().

4. OS executes execute_replace_class:
      - Reads class_hash = 0x1 from request.
      - No validation performed (TODO comment, line 898).
      - dict_update sets C's class_hash = 0x1 in contract_state_changes.

5. Block proven; state committed with C.class_hash = 0x1.

6. In block N+1, any tx calling C:
      - OS reads C.class_hash = 0x1 from state.
      - Looks up compiled class for 0x1 in compiled_class_facts_bundle.
      - Not found → execution fails.

7. All user deposits in C are permanently frozen.
   No upgrade, no rescue, no recovery path exists.
``` [6](#0-5)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
