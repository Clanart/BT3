### Title
Missing Zero-Value Validation in `execute_replace_class` Enables Contract Re-initialization and Direct Loss of Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

`execute_replace_class` does not validate that the caller-supplied `class_hash` is non-zero. This allows any contract to reset its own `class_hash` to `UNINITIALIZED_CLASS_HASH = 0`. Once reset, the `deploy_contract` guard — which checks `state_entry.class_hash = UNINITIALIZED_CLASS_HASH` — passes, enabling re-deployment to the same address with an attacker-controlled class. The re-deployment preserves the original contract's storage while replacing its code, allowing the new class to drain any funds held in that storage.

---

### Finding Description

`UNINITIALIZED_CLASS_HASH` is defined as `0` and is the sentinel value used by `deploy_contract` to determine whether an address is already occupied: [1](#0-0) 

`deploy_contract` enforces two preconditions before allowing deployment to an address: [2](#0-1) 

`execute_replace_class` reads the caller-supplied `class_hash` directly from the syscall request and writes it to the contract's state entry **without any validation**: [3](#0-2) 

The existing TODO comment at line 898 acknowledges the missing class existence check, but no zero-value guard exists either. Passing `class_hash = 0` is accepted silently, resetting the contract's state entry to the uninitialized sentinel.

After the reset, the `deploy_contract` preconditions at lines 53–54 are satisfied for the same address (assuming the contract is non-account, so `nonce = 0` holds). A subsequent `deploy` syscall targeting that address succeeds, invoking the constructor of an attacker-chosen class while **preserving the original contract's storage pointer**: [4](#0-3) 

The new class inherits the original storage, and any function in the new class can read and transfer those storage values.

---

### Impact Explanation

**Critical — Direct loss of funds.**

A contract holding funds (e.g., a vault, token contract, or escrow) can have its class hash reset to 0 via `replace_class(0)`. An attacker then re-deploys a malicious class to the same address. The malicious class's constructor or any external function can read the preserved storage (which contains balances, allowances, or locked assets) and transfer them to the attacker. The original contract's address, storage, and all associated funds are permanently compromised.

---

### Likelihood Explanation

`replace_class` is a standard, publicly accessible syscall callable by any contract on itself. Any contract that exposes an upgrade function accepting a user-supplied class hash (a common pattern) can be triggered by an unprivileged transaction sender to call `replace_class(0)`. The attacker entry path is:

1. Send a transaction to a target contract's upgrade function with `class_hash = 0`.
2. The OS processes `execute_replace_class` without rejecting the zero value.
3. Send a `deploy` syscall targeting the now-uninitialized address.

No privileged access, leaked keys, or operator cooperation is required. The attack is executable by any unprivileged L2 transaction sender against any contract that does not independently validate the class hash before calling `replace_class`.

---

### Recommendation

Add an explicit non-zero assertion in `execute_replace_class` before updating the state entry:

```cairo
// In execute_replace_class, after reading class_hash from request:
let class_hash = request.class_hash;
assert_not_zero(class_hash);  // Prevent reset to UNINITIALIZED_CLASS_HASH.
```

Additionally, resolve the existing TODO at line 898 by verifying that `class_hash` corresponds to a declared contract class in `contract_class_changes`, which would also subsume the zero check since no class is declared with hash 0. [5](#0-4) 

---

### Proof of Concept

1. Attacker deploys contract `C` at address `A` (non-account contract; `nonce = 0` throughout its lifetime).
2. `C` exposes an `upgrade(class_hash: felt)` function that calls `replace_class(class_hash)`.
3. Attacker sends a transaction calling `C.upgrade(0)`.
4. OS executes `execute_replace_class` with `class_hash = 0`; no assertion fires.
5. `contract_state_changes[A].class_hash` is now `0 = UNINITIALIZED_CLASS_HASH`.
6. Attacker sends a transaction calling `deploy(class_hash=MALICIOUS, salt=..., deployer=...)` that resolves to address `A`.
7. `deploy_contract` checks `state_entry.class_hash = UNINITIALIZED_CLASS_HASH` → **passes** (it is 0).
8. `deploy_contract` checks `state_entry.nonce = 0` → **passes**.
9. `MALICIOUS` class is written as the new class hash for `A`; its constructor executes.
10. Constructor reads `A`'s preserved storage (original balances/assets) and transfers them to the attacker. [6](#0-5) [7](#0-6)

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
