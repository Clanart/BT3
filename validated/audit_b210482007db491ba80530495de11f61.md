### Title
Unvalidated Class Hash in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts an arbitrary caller-supplied `class_hash` and writes it directly to the contract's state entry without verifying that the hash corresponds to a previously declared contract class. This is the direct analog of the Wido "unconfirmed address" vulnerability: just as `performCollateralSwap` accepted an unverified Comet address, `execute_replace_class` accepts an unverified class hash. A contract can replace its own class with an undeclared hash, permanently rendering itself uncallable and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` reads `class_hash` directly from the syscall request and writes it into `contract_state_changes` with no validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The developer-inserted TODO at line 898 explicitly acknowledges the missing check. The same omission exists in the deprecated syscall path: [2](#0-1) 

When a subsequent call to the contract is processed, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
``` [3](#0-2) 

If `class_hash` was never declared, `dict_read` returns 0 (the default uninitialized value), and `find_element` with key `0` will fail to locate a matching compiled class fact, making the contract permanently unexecutable. Because the OS Cairo program is the authoritative enforcement layer whose output is proven on-chain, this state corruption is irreversible once committed.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value via `replace_class`, every future call to that contract fails at OS proof-generation time. The sequencer cannot include any call to the contract in a provable block. Any ERC-20 tokens, ETH, or STRK held in the contract's storage become permanently inaccessible. The state corruption is committed to the Merkle trie and cannot be undone without a protocol-level upgrade.

---

### Likelihood Explanation

`replace_class` is a standard, permissionless syscall available to any Cairo 1 contract. No privileged role is required. An attacker needs only to:
1. Deploy or control a contract that holds (or will receive) funds.
2. Issue a transaction that calls `replace_class` with an arbitrary felt value that is not a declared class hash.

The OS will accept and prove this state transition because it performs no validation of the supplied hash.

---

### Recommendation

Before writing the new class hash to `contract_state_changes`, verify that the hash exists as a key in `contract_class_changes` (i.e., it was declared in the current or a prior block). Concretely, add a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert the returned compiled class hash is non-zero:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors how `execute_entry_point` resolves a class before execution and closes the gap identified in the TODO comment. [4](#0-3) 

---

### Proof of Concept

1. Attacker deploys contract `A` that holds 1000 STRK.
2. Attacker sends a transaction from `A` invoking the `replace_class` syscall with `class_hash = 0xdeadbeef` (an arbitrary felt never declared on StarkNet).
3. The OS processes `execute_replace_class`: gas is deducted, `class_hash = 0xdeadbeef` is written to `contract_state_changes[A]`. No validation occurs. The block is proven and the state is committed.
4. In the next block, any user attempts to call contract `A`. The OS executes `execute_entry_point` for `A`: `dict_read(contract_class_changes, 0xdeadbeef)` returns `0`; `find_element(..., key=0)` finds no matching compiled class fact and the proof fails.
5. The sequencer cannot include any call to `A` in any provable block. The 1000 STRK are permanently frozen. [5](#0-4) [6](#0-5)
