### Title
`collect_source_receipt_proofs` Embeds Parent-Shard `to_shard_id` in Witness at Resharding Boundaries, Permanently Breaking Child-Shard Chunk Validation — (File: `chain/chain/src/stateless_validation/state_witness.rs`)

---

### Summary

When a shard is split during resharding, `collect_source_receipt_proofs` fetches receipt proofs that were committed against the **parent** shard's `to_shard_id`. These proofs are embedded verbatim into the `ChunkStateWitness.source_receipt_proofs` map. Validators then call `validate_receipt_proof`, which enforces `proof.to_shard_id == target_chunk_shard_id` (the child shard). Because the stored proof carries the parent shard ID, the check always fails. No chunk for a child shard that spans a resharding boundary can ever be endorsed. The codebase itself acknowledges this with an explicit TODO that has no accompanying fix.

---

### Finding Description

**Proof generation — `collect_source_receipt_proofs`** [1](#0-0) 

The comment admits the defect: `get_incoming_receipts_for_shard` is called with `ReceiptFilter::All`, but when the traversal crosses an epoch boundary where a shard split occurred, `get_incoming_receipts_for_shard` maps the child shard back to its parent: [2](#0-1) 

It then reads the on-chain `IncomingReceipts` keyed by the **parent** shard ID. Those stored `ReceiptProof` objects carry `ShardProof { to_shard_id: parent, proof: <merkle-path-over-parent-receipts> }`. They are inserted into the witness map unchanged: [3](#0-2) 

**Validation — `validate_source_receipt_proofs` / `validate_receipt_proof`**

On the validator side, `validate_receipt_proof` enforces two hard invariants: [4](#0-3) 

The check at line 547 (`receipt_proof.1.to_shard_id != target_chunk_shard_id`) compares the proof's embedded `to_shard_id` (parent) against the expected child shard ID. They differ → `InvalidChunkStateWitness` is returned. The Merkle-path check at line 556 would also fail independently, because the path was computed over the full parent-shard receipt set, not the filtered child-shard subset.

**The fixed-reference invariant**

`source_receipt_proofs` is a `HashMap<ChunkHash, ReceiptProof>`. The `ChunkHash` key and the `ShardProof.to_shard_id` inside the value are both fixed at witness-generation time against the pre-resharding shard layout. After resharding, the child shard ID is a new identifier that was not in scope when the proof was committed on-chain. There is no migration path: the on-chain Merkle commitment (`prev_outgoing_receipts_root`) was computed with `to_shard_id = parent`, so no re-keying can produce a valid proof for `to_shard_id = child`. [5](#0-4) 

---

### Impact Explanation

Every chunk produced for a child shard whose receipt window spans the resharding epoch boundary will carry an unverifiable `source_receipt_proofs`. Validators uniformly reject the witness with `InvalidChunkStateWitness`. No endorsements are issued, so the child shard's chunks cannot be included in blocks. All cross-shard receipts destined for that child shard are permanently undeliverable — the exact "balance rendered unclaimable" pattern from the external report, translated to NEAR's receipt-delivery invariant.

**Severity: Critical** — consensus for newly-created child shards is broken at every resharding event while stateless validation is active.

---

### Likelihood Explanation

The defect fires deterministically at every resharding boundary when stateless validation is enabled. No adversarial action is required; honest chunk producers following the protocol trigger it automatically. The codebase has already undergone three mainnet resharding events (protocol versions 75, 76, 78): [6](#0-5) 

If stateless validation is activated concurrently with or after a future resharding, the bug manifests immediately.

---

### Recommendation

Implement the fix described in the TODO: collect the **unfiltered** original proof (preserving the parent-shard Merkle commitment), include it in the witness, and perform receipt filtering **after** the validator has verified the proof against the on-chain root. Concretely:

1. In `collect_source_receipt_proofs`, pass `ReceiptFilter::All` (already done) but also record the pre-resharding `to_shard_id` alongside each proof so validators know which Merkle root to check against.
2. In `validate_source_receipt_proofs`, when crossing a resharding boundary, verify the proof against the parent-shard outgoing-receipts root, then apply `reassign_outgoing_receipts_for_resharding` to filter the receipt list before computing `applied_receipts_hash`. [7](#0-6) 

---

### Proof of Concept

Assume parent shard `P` is split into children `L` (lower ID) and `R` at epoch boundary `E`.

1. In epoch `E-1`, chunk `C` in shard `A` sends receipts to `P`. On-chain commitment: `prev_outgoing_receipts_root = merkle({ hash(receipts_to_P, to_shard_id=P) })`.

2. In epoch `E`, chunk producer for child `L` calls `collect_source_receipt_proofs`.

3. `get_incoming_receipts_for_shard(shard_id=L, ...)` crosses the epoch boundary, maps `L → P`, and reads `IncomingReceipts[block_in_E-1][P]` = `ReceiptProof(receipts_to_P, ShardProof { from: A, to: P, proof: <path> })`.

4. Witness is emitted with `source_receipt_proofs[hash(C)] = ReceiptProof(..., ShardProof { to_shard_id: P })`.

5. Validator calls `validate_receipt_proof(proof, chunk_C, target=L, root)`:
   - Line 547: `proof.to_shard_id (P) != target_chunk_shard_id (L)` → **`InvalidChunkStateWitness`**.

6. No endorsement is produced. Child shard `L`'s chunk is never finalized. All receipts from epoch `E-1` destined for `L` are permanently stranded. [8](#0-7)

pub fn validate_receipt_proof(
    receipt_proof: &ReceiptProof,
    from_chunk: &ShardChunkHeader,
    target_chunk_shard_id: ShardId,
    outgoing_receipts_root: CryptoHash,
) -> Result<(), Error> {
    // Validate that from_shard_id is correct. The receipts must match the outgoing receipt root
    // for this shard, so it's impossible to fake it.
    if receipt_proof.1.from_shard_id != from_chunk.shard_id() {
        return Err(Error::InvalidChunkStateWitness(format!(
            "Receipt proof for chunk {:?} is from shard {}, expected shard {}",
            from_chunk.chunk_hash(),
            receipt_proof.1.from_shard_id,
            from_chunk.shard_id(),
        )));
    }
    // Validate that to_shard_id is correct. to_shard_id is also encoded in the merkle tree,
    // so it's impossible to fake it.
    if receipt_proof.1.to_shard_id != target_chunk_shard_id {
        return Err(Error::InvalidChunkStateWitness(format!(
            "Receipt proof for chunk {:?} is for shard {}, expected shard {}",
            from_chunk.chunk_hash(),
            receipt_proof.1.to_shard_id,
            target_chunk_shard_id
        )));
    }
    // Verify that (receipts, to_shard_id) belongs to the merkle tree of outgoing receipts.
    if !receipt_proof.verify_against_receipt_root(outgoing_receipts_root) {
        return Err(Error::InvalidChunkStateWitness(format!(
            "Receipt proof for chunk {:?} has invalid merkle path, doesn't match outgoing receipts root",
            from_chunk.chunk_hash()
        )));
    }
    Ok(())

    pub fn reassign_outgoing_receipts_for_resharding(
        receipts: &mut Vec<Receipt>,
        shard_layout: &ShardLayout,
        shard_id: ShardId,
        receipts_shard_id: ShardId,
    ) -> Result<(), Error> {
        tracing::debug!(target: "resharding", %shard_id, ?receipts_shard_id, "reassign_outgoing_receipts_for_resharding");

        let split_shard_ids = shard_layout.get_children_shards_ids(receipts_shard_id);
        let split_shard_ids =
            split_shard_ids.ok_or(Error::InvalidSplitShardsIds(shard_id, receipts_shard_id))?;

        // The target shard id is the split shard with the lowest shard id.
        let target_shard_id = split_shard_ids.iter().min();
        let target_shard_id =
            *target_shard_id.ok_or(Error::InvalidSplitShardsIds(shard_id, receipts_shard_id))?;

        if shard_id == target_shard_id {
            // This shard_id is the lowest index child, it gets all the receipts.
            Ok(())
        } else {
            // This shard_id is not the lowest index child, it gets no receipts.
            receipts.clear();
            Ok(())
