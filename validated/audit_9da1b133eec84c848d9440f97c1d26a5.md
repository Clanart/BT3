### Title
Unvalidated Class Hash in `execute_replace_class` Enables Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary felt value as the replacement class hash without verifying it corresponds to a declared contract class. An attacker-controlled contract can call `replace_class` with an undeclared class hash, permanently writing an unresolvable class reference into the contract's on-chain state. All subsequent calls to that contract will revert, permanently freezing any funds held within it.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` reads the requested class hash directly from the syscall request and writes it into `contract_state_changes` with no declaration check:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The `TODO` comment at line 898 explicitly acknowledges the missing validation. The `class_hash` field of the new `StateEntry` is committed to the Patricia Merkle tree during `state_update`, making the change durable across blocks.

When any future transaction calls this contract, `execute_entry_point` resolves the class hash through two sequential lookups:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [2](#0-1) 

For an undeclared class hash, `dict_read` on `contract_class_changes` returns `0` (the default uninitialized value). `find_element` then searches for a compiled class with hash `0`, which does not exist. The blockifier, executing the same logic, will revert every call to this contract. Because the class hash is durably committed to state, this revert is permanent — no future transaction can recover the contract or its funds.

---

### Impact Explanation

**Critical: Permanent freezing of funds.**

A contract deployer creates a contract that accepts user deposits (e.g., a vault or liquidity pool). After accumulating funds, the deployer triggers `replace_class` with an arbitrary undeclared felt value (e.g., `0xdeadbeef`). The OS writes this value into the contract's `StateEntry` without any validation. From that point forward, every call to the contract — including withdrawal functions — is permanently reverted because neither the blockifier nor the OS can resolve the class hash to a compiled class. The deposited funds are irrecoverably frozen with no upgrade or recovery path.

---

### Likelihood Explanation

The attack requires only:
1. Deploying a contract (permissionless on StarkNet — any "contract deployer" qualifies).
2. Calling `replace_class` with an arbitrary felt value from within that contract.

No privileged access, leaked keys, or operator cooperation is required. The missing validation is explicitly documented as a `TODO` in production code at line 898, confirming it is not enforced at the OS level. The attack is deterministic and requires no brute force or probabilistic conditions.

---

### Recommendation

Before committing the new class hash to `contract_state_changes`, validate that it corresponds to a previously declared class by asserting a non-zero entry exists in `contract_class_changes`:

```cairo
// Validate that the class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the pattern already used in `execute_entry_point` and ensures `replace_class` can only target classes that have been properly declared and whose compiled class facts are available to the OS.

### Proof of Concept

**Step 1 — Attacker deploys `MaliciousVault`:**
A contract with a `deposit()` entry point that accepts funds and a `freeze()` entry point that calls `replace_class(0xdeadbeef)`.

**Step 2 — Users deposit funds:**
Users call `deposit()`, transferring tokens to `MaliciousVault`. The contract state is valid at this point.

**Step 3 — Attacker calls `freeze()`:**
The contract executes the `replace_class` syscall. The OS processes it via `execute_replace_class`:
- `class_hash = 0xdeadbeef` (undeclared)
- No validation is performed (line 898 TODO)
- `dict_update` writes `StateEntry(class_hash=0xdeadbeef, ...)` to `contract_state_changes`
- This is committed to the Patricia tree during `state_update` [3](#0-2) 

**Step 4 — Users attempt to withdraw:**
Any call to `MaliciousVault` reaches `execute_entry_point`. `dict_read` on `contract_class_changes` for key `0xdeadbeef` returns `0`. `find_element` cannot locate a compiled class with hash `0`. The blockifier reverts the call. This revert is permanent because the state commitment already contains `class_hash=0xdeadbeef`. [4](#0-3) 

**Result:** All user funds in `MaliciousVault` are permanently frozen. No recovery mechanism exists within the protocol.

---

**Vulnerability class analog:** This is the StarkNet OS analog of CVE-2021-23358 (arbitrary code injection via underscore templates). Just as that vulnerability allowed injection of arbitrary executable code through an unvalidated template parameter, this vulnerability allows injection of an arbitrary (undeclared) class hash into durable contract state through an unvalidated syscall parameter — with irreversible protocol-level consequences.
