### Title
`execute_replace_class` Unconditionally Updates Class Hash Without Validating the New Class Is Declared, Permanently Freezing Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS unconditionally updates a contract's class hash in `contract_state_changes` without verifying that the new class hash corresponds to a declared class. A contract deployer can exploit this to permanently brick any contract (including ones holding user funds) by calling `replace_class` with an undeclared class hash, making the contract permanently uncallable and freezing all funds it holds.

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall. After deducting gas, it reads the current `StateEntry` for the contract and unconditionally writes a new `StateEntry` with the attacker-supplied `class_hash` into `contract_state_changes`:

    key=contract_address,

The TODO comment at line 898 explicitly acknowledges the missing validation. The OS never checks `contract_class_changes` (the dictionary of declared class hashes) to confirm the new `class_hash` is a known, compiled class. The state update is committed unconditionally.

This is structurally identical to the M-03 pattern: a state-transition (class hash replacement) proceeds unconditionally without verifying a critical precondition (the new class is declared), leaving the system in a broken state with no recovery path.

### Impact Explanation

Once a contract's class hash is set to an undeclared value, every subsequent call to that contract will fail at the OS level when it attempts to resolve the class hash to compiled bytecode — the class simply does not exist in the compiled class facts bundle. The contract's storage (and any token balances or ETH held there) becomes permanently inaccessible. There is no syscall or protocol mechanism to recover from this state, because the contract itself cannot execute any entry point to call `replace_class` again.

**Impact: Critical — Permanent freezing of funds.**

### Likelihood Explanation

The attacker-controlled entry path is direct and requires no privileged access:

1. An attacker deploys a contract (valid attacker role: contract deployer).
2. The contract accumulates user funds (e.g., acts as a pool, vault, or escrow that users deposit into).
3. The attacker calls a function on their contract that internally invokes the `replace_class` syscall with an arbitrary felt value that is not a declared class hash.
4. The OS processes the syscall via `execute_replace_class`, passes the gas check, and unconditionally writes the invalid class hash into `contract_state_changes`.
5. The transaction succeeds and the state is committed. The contract is permanently bricked.

No privileged role, leaked key, or external dependency is required. The `replace_class` syscall is available to any Sierra contract.

### Recommendation

Before committing the `dict_update`, verify that `class_hash` exists in `contract_class_changes` (i.e., it was declared in the current block) or in the pre-existing global state. Concretely, perform a `dict_read` on `contract_class_changes` for the supplied `class_hash` and assert the result is non-zero (a valid compiled class hash). If the class is not declared, write a failure response and return without updating `contract_state_changes`. This is exactly what the existing TODO comment calls for.

### Proof of Concept

1. Declare class `A` (legitimate class with a `replace_and_lock` entry point).
2. Deploy contract `C` using class `A`. Users deposit funds into `C`.
3. Call `replace_and_lock` on `C`, which internally executes:
   ```
   replace_class(class_hash=0xdeadbeef)  // undeclared felt
   ```
4. The OS calls `execute_replace_class`. Gas is deducted. The TODO check is absent. `dict_update` sets `C`'s class hash to `0xdeadbeef`.
5. Transaction succeeds. `C`'s class hash in the committed state is `0xdeadbeef`.
6. Any subsequent call to `C` fails: the OS cannot find `0xdeadbeef` in the compiled class facts bundle. All funds in `C`'s storage are permanently frozen. [1](#0-0) [2](#0-1)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.

        key=contract_address,

    );

}
