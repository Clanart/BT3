### Title
Proof-facts hash reuse bypasses cryptographic proof verification for subsequent transactions - (File: `crates/apollo_transaction_converter/src/transaction_converter.rs`)

### Summary

`run_proof_verification` gates proof verification on whether the **proof-facts hash** was previously stored, not on whether the submitted proof bytes are valid. Once any transaction with a given `ProofFacts` set is verified and stored, every subsequent transaction carrying the same `ProofFacts` — regardless of its actual proof bytes — bypasses cryptographic verification entirely and is admitted by the gateway/consensus converter.

### Finding Description

`run_proof_verification` performs a `contains_proof(proof_facts)` lookup keyed on `proof_facts.hash()` (a Poseidon hash of the proof-facts felts). If the lookup returns `true`, the function returns `Ok(false)` immediately — the submitted `Proof` bytes are never passed to `starknet_proof_verifier::verify_proof`. [1](#0-0) 

The proof manager stores proofs keyed by `facts_hash` only: [2](#0-1) 

`contains_proof` checks only whether *any* proof was previously stored for those facts — it does not compare the stored proof bytes against the newly submitted ones: [3](#0-2) 

The proof-facts hash is derived solely from the `ProofFacts` felt vector, which encodes `proof_version`, `VIRTUAL_SNOS`, `program_hash`, `output_version`, `block_number`, `block_hash`, and `config_hash`: [4](#0-3) 

`validate_proof_facts` in the blockifier validates structural fields (block number recency, block hash match, config hash, program hash) but contains no uniqueness check — the same proof facts can appear in arbitrarily many transactions: [5](#0-4) 

The OS-level `check_proof_facts` in Cairo mirrors this: it validates the block hash and config hash but performs no cross-transaction deduplication: [6](#0-5) 

### Impact Explanation

An attacker controlling two accounts can:

1. **Account A** submits `Tx1` with valid `proof_facts = P` and a valid `proof` → `contains_proof(P)` = false → proof verified → stored under `P.hash()`.
2. **Account B** submits `Tx2` with the same `proof_facts = P` and **arbitrary/invalid proof bytes** → `contains_proof(P)` = true → `run_proof_verification` returns `Ok(false)` → proof bytes are never verified → `Tx2` is admitted.

The proof bytes are not part of the transaction hash (only `proof_facts` felts are hashed into the invoke transaction hash via `compute_invoke_transaction_hash`), so `Tx2` has a distinct transaction hash and passes nonce/duplicate checks normally. [7](#0-6) 

The result is that the gateway/consensus converter admits a transaction whose attached SNOS proof was never cryptographically verified. This violates the SNIP-36 invariant that every proof-carrying transaction must carry a valid proof, and falls under **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing**.

### Likelihood Explanation

The attack requires only two funded accounts and one previously accepted proof-carrying transaction. The proof facts for a given historical block are public (block number and block hash are on-chain). Any attacker who observes a valid proof-carrying transaction in the mempool or in a finalized block can immediately construct a second transaction with the same proof facts and garbage proof bytes. No privileged access is required.

### Recommendation

Change the `contains_proof` short-circuit to also verify that the **stored proof matches the submitted proof**, or remove the short-circuit entirely and always verify the submitted proof bytes:

```rust
async fn run_proof_verification(
    proof_facts: ProofFacts,
    proof: Proof,
    proof_manager_client: SharedProofManagerClient,
) -> Result<bool, TransactionConverterError> {
    // Always verify; use the cache only to skip re-storing, not re-verifying.
    let facts_hash = proof_facts.hash();
    tokio::task::spawn_blocking(move || {
        starknet_proof_verifier::verify_proof(proof_facts, proof)
    })
    .await
    .expect("proof verification task panicked")?;

    // Store only if not already present (idempotent).
    if !proof_manager_client.contains_proof_by_hash(facts_hash).await? {
        proof_manager_client.set_proof(...).await?;
    }
    Ok(true)
}
```

Alternatively, key the `contains_proof` check on `hash(proof_facts || proof_bytes)` so that different proof bytes for the same facts are treated as distinct entries and each is independently verified.

### Proof of Concept

```
1. Observe or submit Tx1 from Account A:
     proof_facts = [PROOF_VERSION_V1, VIRTUAL_SNOS, program_hash,
                    VIRTUAL_OS_OUTPUT_VERSION, block_N, block_hash_N, config_hash]
     proof       = <valid zstd-compressed proof>
   → run_proof_verification: contains_proof = false → verify_proof succeeds → stored

2. Submit Tx2 from Account B (different sender_address, fresh nonce):
     proof_facts = identical to Tx1
     proof       = b"garbage_bytes_that_fail_verification"
   → run_proof_verification: contains_proof(proof_facts) = true → returns Ok(false)
   → Tx2 admitted to mempool/block with unverified proof bytes
```

`Tx2` passes `validate_proof_facts` (structural checks pass), passes nonce validation (different account), and is included in a block. The SNIP-36 proof-verification requirement is silently bypassed. [8](#0-7)

    fn validate_proof_facts(
        &self,
        block_context: &BlockContext,
        state: &mut dyn State,
    ) -> TransactionPreValidationResult<()> {
        // Only Invoke V3 transactions can carry proof facts.
        let Transaction::Invoke(invoke_tx) = &self.tx else {
            return Ok(());
        };
        if invoke_tx.version() < TransactionVersion::THREE {
            return Ok(());
        }

        // Parse proof facts.
        let proof_facts = invoke_tx.proof_facts();
        let snos_proof_facts = match ProofFactsVariant::try_from(&proof_facts)
            .map_err(|e| TransactionPreValidationError::InvalidProofFacts(e.to_string()))?
        {
            ProofFactsVariant::Empty => return Ok(()),
            ProofFactsVariant::Snos(snos_proof_facts) => snos_proof_facts,
        };
        let os_constants = &block_context.versioned_constants.os_constants;

        if !os_constants.allowed_proof_versions.contains(&snos_proof_facts.proof_version.as_felt())
        {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Proof version {} is not allowed under this protocol version.",
                snos_proof_facts.proof_version
            )));
        }

        // Validate the program hash.
        let allowed = &os_constants.allowed_virtual_os_program_hashes;
        if !allowed.contains(&snos_proof_facts.program_hash) {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS program hash {} is not allowed",
                snos_proof_facts.program_hash
            )));
        }

        // Validate the block hash and block number.
        let proof_block_hash = snos_proof_facts.block_hash.0;
        let proof_block_number = snos_proof_facts.block_number.0;
        Self::validate_proof_block_number(
            proof_block_number,
            block_context.block_info.block_number,
        )?;
        Self::validate_proof_block_hash(proof_block_hash, proof_block_number, os_constants, state)?;

        // Validate the config hash.
        let virtual_os_config_hash = block_context.virtual_os_config_hash();
        let proof_config_hash = snos_proof_facts.config_hash;
        if virtual_os_config_hash != proof_config_hash {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS config hash mismatch. Computed virtual OS config hash: \
                 {virtual_os_config_hash}, expected virtual OS config hash: {proof_config_hash}."
            )));
        }

        Ok(())
    }

func check_proof_facts{range_check_ptr, contract_state_changes: DictAccess*}(
    proof_facts_size: felt,
    proof_facts: felt*,
    current_block_number: felt,
    virtual_os_config_hash: felt,
) {
    if (proof_facts_size == 0) {
        return ();
    }

    assert_le(ProofHeader.SIZE + VirtualOsOutputHeader.SIZE, proof_facts_size);

    // Validate the proof header.
    static_assert ProofHeader.SIZE == 3;
    let proof_header = cast(proof_facts, ProofHeader*);
    assert proof_header.proof_variant = VIRTUAL_SNOS;
    assert is_program_hash_allowed(proof_header.program_hash) = TRUE;
    // Proof version may be V0 (legacy) or V1 (current).
    with_attr error_message("Unsupported proof version") {
        assert proof_header.proof_version = PROOF_VERSION_V1;
    }

    // Validate the virtual OS output header.
    let os_output_header = cast(&proof_facts[ProofHeader.SIZE], VirtualOsOutputHeader*);

    with_attr error_message("Virtual OS output version is not supported") {
        assert os_output_header.output_version = VIRTUAL_OS_OUTPUT_VERSION;
    }

    // Validate that the proof facts block number is not too recent.
    // (This is a sanity check - the following non-zero check ensures that the block hash is
    // not trivial).
    assert_nn_le(
        os_output_header.base_block_number, current_block_number - STORED_BLOCK_HASH_BUFFER
    );
    // Not all block hashes are stored in the contract; Make sure the requested one is not trivial.
    assert_not_zero(os_output_header.base_block_hash);

    // validate that the proof facts block hash is the true hash of the proof facts block number.
    read_block_hash_from_storage(
        block_number=os_output_header.base_block_number,
        expected_block_hash=os_output_header.base_block_hash,
    );

    // validate that the proof facts config hash is the true hash of the OS config.
    assert os_output_header.starknet_os_config_hash = virtual_os_config_hash;

    return ();
}
