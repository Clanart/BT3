### Title
Missing Class Hash Existence Validation in `replace_class` Syscall Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS Cairo program accepts any arbitrary felt value as a new class hash without verifying that the hash corresponds to a declared contract class. An attacker-controlled contract can call `replace_class` with an undeclared or invalid class hash, permanently corrupting its own on-chain class pointer. Because the OS commits this invalid state to the proven state root, any funds held by the affected contract become permanently inaccessible — a critical, irreversible freezing of funds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
``` [1](#0-0) 

The `class_hash` value from the request is written directly into `contract_state_changes` with no check that it exists in `contract_class_changes` (the declared class registry). The inline `TODO` comment at line 898 explicitly acknowledges this missing validation.

The same flaw exists in the deprecated syscall path:

```cairo
func execute_replace_class{contract_state_changes: DictAccess*, revert_log: RevertLogEntry*}(
    contract_address, syscall_ptr: ReplaceClass*
) {
    alloc_locals;
    let class_hash = syscall_ptr.class_hash;

    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );
    dict_update{dict_ptr=contract_state_changes}(...);
``` [2](#0-1) 

When a subsequent block attempts to execute the affected contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
``` [3](#0-2) 

If `class_hash` is undeclared, `dict_read` returns 0 (the default), and `find_element` with key `0` will fail with a hard Cairo assertion — the contract is permanently unexecutable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value via `replace_class`, the contract can never be executed again: the OS cannot locate the compiled class, and there is no recovery path (calling `replace_class` again requires executing the contract, which is now impossible). Any ERC-20 tokens, ETH, or other assets held by the contract are permanently frozen. Because the OS proves and commits this state transition to L1, the corruption is irreversible at the protocol level.

---

### Likelihood Explanation

**High.** The `replace_class` syscall is available to any deployed contract — no special privilege is required. An attacker can:

1. Deploy a contract (standard user operation).
2. Have the contract call `replace_class` with an arbitrary felt (e.g., `0xdeadbeef`).
3. The OS accepts and commits the invalid class hash to the proven state root.

No sequencer collusion, leaked keys, or privileged access is needed. The attacker only needs to control a deployed contract, which is a standard capability of any StarkNet user.

---

### Recommendation

Before writing the new class hash to `contract_state_changes`, verify that it exists in `contract_class_changes` (i.e., it has been declared). Concretely, perform a `dict_read` on `contract_class_changes` with the new `class_hash` and assert the result is non-zero:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This check should be added in both `execute_replace_class` in `syscall_impls.cairo` and `execute_replace_class` in `deprecated_execute_syscalls.cairo`, resolving the acknowledged TODO.

---

### Proof of Concept

1. Attacker deploys contract `VaultAttacker` that:
   - Accepts ETH/token deposits from users.
   - Exposes a function `attack()` that calls `replace_class(0xdeadbeef)`.

2. Users deposit funds into `VaultAttacker`.

3. Attacker calls `attack()`. The OS executes `execute_replace_class` with `class_hash = 0xdeadbeef`. No validation occurs. `contract_state_changes` is updated with the invalid class hash. The block is proven and committed to L1.

4. In any subsequent block, any transaction targeting `VaultAttacker` causes `execute_entry_point` to call `dict_read(contract_class_changes, 0xdeadbeef)` → returns `0` → `find_element(..., key=0)` → hard failure. The sequencer cannot include any transaction calling `VaultAttacker`.

5. All deposited funds are permanently frozen. The state root on L1 reflects this invalid class hash, and no valid proof can ever recover the funds.
