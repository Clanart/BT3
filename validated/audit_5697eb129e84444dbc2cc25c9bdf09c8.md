### Title
Missing Declared Class Validation in `replace_class` Syscall Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary

The `execute_replace_class` function in the StarkNet OS does not validate that the new class hash supplied by a caller has been previously declared. An unprivileged contract can call `replace_class` with an arbitrary, undeclared class hash, permanently rendering the contract uncallable at the OS level and freezing all funds held within it.

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function accepts any `class_hash` value from the syscall request and writes it directly into `contract_state_changes` without verifying that a corresponding compiled class exists:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;
    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}
    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );
    dict_update{dict_ptr=contract_state_changes}(...);
``` [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges the missing check. Contrast this with `execute_declare_transaction`, which enforces `prev_value=0` and `assert_not_zero(compiled_class_hash)` to register a class: [2](#0-1) 

After the invalid `replace_class` call, any future invocation of the affected contract reaches `execute_entry_point`, which does:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
``` [3](#0-2) 

Because the class was never declared, `dict_read` returns `0` (the default). `find_element` with key `0` will fail with a constraint violation if no compiled class with hash `0` exists, making it impossible to generate a valid proof for any block containing a call to this contract. The sequencer is forced to permanently exclude all transactions targeting the contract.

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any ETH, ERC-20 tokens, or other assets held in the affected contract become permanently inaccessible. No transaction calling the contract can ever be proven again. The state entry for the contract remains in the Merkle tree with a broken class hash, and there is no recovery path within the OS.

### Likelihood Explanation

The attack is reachable by any unprivileged user who controls a contract (e.g., a deployed vault, multisig, or token contract). The attacker:
1. Deploys a contract or controls an existing one holding user funds.
2. Issues a `replace_class` syscall with an arbitrary undeclared felt value as the class hash.
3. The OS accepts the call without validation and commits the corrupted state.

No privileged role, leaked key, or external dependency is required. The missing check is self-contained in the OS Cairo code.

### Recommendation

Add a validation step inside `execute_replace_class` to confirm that the supplied `class_hash` has a non-zero entry in `contract_class_changes` (i.e., it has been declared) before updating the contract state. This mirrors the invariant enforced during `execute_declare_transaction` where `assert_not_zero(compiled_class_hash)` is required. The existing TODO comment at line 898 already identifies this gap.

### Proof of Concept

1. Attacker deploys `VaultContract` which accepts ETH deposits from users.
2. Users deposit funds; `VaultContract` accumulates a balance.
3. Attacker calls `replace_class(class_hash=0xdeadbeef)` from within `VaultContract` (where `0xdeadbeef` is never declared).
4. OS executes `execute_replace_class`: no validation, `contract_state_changes` updated with `class_hash=0xdeadbeef`.
5. Any subsequent transaction calling `VaultContract` reaches `execute_entry_point` → `dict_read` returns `compiled_class_hash=0` → `find_element(..., key=0)` fails → block proof cannot be generated.
6. Sequencer permanently excludes all calls to `VaultContract`. All deposited funds are frozen with no recovery mechanism. [4](#0-3)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
