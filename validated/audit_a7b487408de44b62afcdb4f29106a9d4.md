### Title
Unbounded Predecessor Replay Loop in `TraceTx` gRPC Handler Enables Node Resource Exhaustion — (File: `x/evm/keeper/grpc_query.go`)

---

### Summary

The `TraceTx` gRPC handler iterates over a caller-supplied `req.Predecessors` slice with no size bound. Each iteration invokes `ApplyMessageWithConfig` with `commit=true`, which is a full EVM execution plus StateDB commit. The 5-second timeout applied inside `prepareTrace` covers only the final traced transaction, not the predecessor replay loop. An unprivileged caller can submit a single gRPC/JSON-RPC request carrying thousands of complex predecessor transactions, forcing the node to execute all of them serially and without any deadline, exhausting CPU and memory.

---

### Finding Description

In `x/evm/keeper/grpc_query.go`, `TraceTx` replays predecessor transactions before tracing the target:

```go
for i, tx := range req.Predecessors {          // ← no length cap
    ethTx := tx.AsTransaction()
    msg, err := core.TransactionToMessage(ethTx, signer, cfg.BaseFee)
    ...
    rsp, err := k.ApplyMessageWithConfig(ctx, msg, cfg, true)   // commit=true
    ...
    cfg.TxConfig.LogIndex += uint(len(rsp.Logs))
}
``` [1](#0-0) 

`ApplyMessageWithConfig` with `commit=true` creates a fresh `StateDB`, runs the full EVM (including all opcode execution, storage writes, and balance mutations), then calls `stateDB.Commit()`, which flushes every dirty storage slot and account to the underlying Cosmos KV store: [2](#0-1) [3](#0-2) 

The 5-second deadline is set only inside `prepareTrace`, which is called **after** the predecessor loop completes:

```go
result, _, err := k.prepareTrace(ctx, cfg, msg, req.GetTraceConfig(), false)
``` [4](#0-3) 

The timeout goroutine inside `prepareTrace` therefore never fires during predecessor replay: [5](#0-4) 

There is no analogous deadline, no `MaxPredecessors` constant, and no per-request gas budget applied to the predecessor loop.

---

### Impact Explanation

Each predecessor execution is a full EVM state transition. A single malicious request with N predecessors, each consuming the block gas limit in storage writes, forces the node to perform O(N × blockGasLimit / SSTOREcost) store writes before the gRPC call returns. At N = 1 000 predecessors each consuming 30 M gas, this is ~1.5 billion gas worth of work in one synchronous call. This exhausts CPU and heap on the responding node. If the targeted node is a validator, it will stall its consensus reactor, miss `PrecommitTimeout` windows, and be slashed or cause the chain to lose liveness — matching the "chain halt" Critical impact. Even on a non-validator full node, the crash degrades the network's RPC availability and can be used to silence specific nodes before a targeted attack.

---

### Likelihood Explanation

The `TraceTx` gRPC method is exposed publicly via the `debug_traceTransaction` JSON-RPC endpoint on any node that enables the debug namespace (the default for archive/RPC nodes). No authentication, stake, or governance action is required. The attacker only needs to craft a `QueryTraceTxRequest` with a large `Predecessors` slice containing high-gas transactions. This is a single HTTP/gRPC call with no rate limiting at the application layer.

---

### Recommendation

1. **Cap predecessor count**: Enforce a hard limit (e.g., `MaxTracePredecessors = 256`) at the top of `TraceTx` before the loop.
2. **Apply a single deadline to the entire handler**: Wrap the predecessor loop and `prepareTrace` call in a single `context.WithTimeout` so the 5-second (or configurable) budget covers all work.
3. **Bound per-predecessor gas**: Reject any predecessor whose `GasLimit` exceeds the block gas limit, preventing a single predecessor from consuming unbounded resources.

---

func (s *StateDB) Commit() error {
	if s.committed {
		return errors.New("statedb already committed")
	}
	s.committed = true
	// if there's any errors during the execution, abort
	if s.err != nil {
		return s.err
	}

	// Enforce the non-overlap invariant BEFORE flushing the native cache store.
	// A nested native action (via ExecuteNativeAction) commits its writes into s.cacheMS
	// (readable via s.ctx). If any EVM-dirty key was also written by such an action, the
	// store value visible through s.ctx will differ from originStorage. Detecting this
	// before flushing means we can abort cleanly — the parent context is never touched.
	//
	// Note: only EVM-dirty keys are checked; native-only writes have no EVM dirty bit
	// and are not in scope.
	for _, addr := range s.journal.sortedDirties() {
		obj, exist := s.stateObjects[addr]
		if !exist || obj.selfDestructed {
			continue
		}
		for _, key := range obj.dirtyStorage.SortedKeys() {
			origin := obj.originStorage[key]
			dirty := obj.dirtyStorage[key]
			if dirty == origin {
				continue
			}
			// A native action wrote this slot iff the store value differs from origin.
			// If it also differs from the EVM-dirty value, the two sides disagree — conflict.
			store := s.keeper.GetState(s.ctx, obj.Address(), key)
			if store != origin && store != dirty {
				return fmt.Errorf(
					"%w: address %s key %s modified by both EVM execution and native action (origin=%s, store=%s, dirty=%s)",
					ErrStateConflict,
					obj.Address().Hex(), key.Hex(), origin.Hex(), store.Hex(), dirty.Hex(),
				)
			}
		}
	}

	s.flushNativeCacheLayers()
	if len(s.nativeEvents) > 0 {
		s.origCtx.EventManager().EmitEvents(s.nativeEvents)
	}

	for _, addr := range s.journal.sortedDirties() {
		obj, exist := s.stateObjects[addr]
		if !exist {
			continue
		}
		if obj.selfDestructed {
			// Burn any balance that arrived after SelfDestruct was called (e.g., via a
			// value-bearing CALL to the destroyed address within the same transaction).
			// SelfDestruct already burned the balance present at destruction time, but
			// subsequent AddBalance calls write to the bank without a matching burn.
			// DeleteAccount only removes auth metadata and storage; it never touches the
			// bank balance, so we must drain it here before removing the account.
			//
			// Both operations run inside a single CacheContext so that if DeleteAccount
			// fails after SubBalance, the partial burn is rolled back and the bank is
			// left consistent.
			cosmosAddr := sdk.AccAddress(obj.Address().Bytes())
			cacheCtx, writeCache := s.origCtx.CacheContext()
			// Only the EVM denom is burned here. Non-EVM-native tokens (IBC, CosmWasm
			// bridge) held by the destroyed address are not drained and may remain as
			// orphaned bank balances.
			if remaining := s.keeper.GetBalance(cacheCtx, cosmosAddr, s.evmDenom); remaining.Sign() > 0 {
				coin := sdk.NewCoin(s.evmDenom, sdkmath.NewIntFromBigInt(remaining.ToBig()))
				if _, err := s.keeper.SubBalance(cacheCtx, cosmosAddr, coin); err != nil {
					return errorsmod.Wrap(err, "failed to burn post-selfdestruct balance")
				}
			}
			if err := s.keeper.DeleteAccount(cacheCtx, obj.Address()); err != nil {
				return errorsmod.Wrap(err, "failed to delete account")
			}
			writeCache()
		} else {
			codeDirty := obj.codeDirty()
			if codeDirty && obj.code != nil {
				s.keeper.SetCode(s.origCtx, obj.CodeHash(), obj.code)
			}
			if codeDirty || obj.nonceDirty() {
				if err := s.keeper.SetAccount(s.origCtx, obj.Address(), obj.account); err != nil {
					return errorsmod.Wrap(err, "failed to set account")
				}
			}
			for _, key := range obj.dirtyStorage.SortedKeys() {
				value := obj.dirtyStorage[key]
				if value == obj.originStorage[key] {
					continue
				}
				s.keeper.SetState(s.origCtx, obj.Address(), key, value.Bytes())
			}
		}
	}
	return nil
}
