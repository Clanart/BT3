### Title
MinGasPriceDecorator Validates Declared Fee While DeductFeeDecorator Deducts Lower Effective Fee, Bypassing MinGasPrice Enforcement for Cosmos Txs with ExtensionOptionDynamicFeeTx — (`ante/cosmos/min_gas_price.go` / `ante/evm/fee_checker.go`)

### Summary
When `MinGasPrice > 0` and `baseFee < MinGasPrice`, a Cosmos tx submitter can attach `ExtensionOptionDynamicFeeTx.MaxPriorityPrice = 0` and set `feeCap = MinGasPrice`. `MinGasPriceDecorator` passes because it checks the declared fee (`feeCap * gas = MinGasPrice * gas`), but `NewDynamicFeeChecker` returns `effectiveFee = baseFee * gas`, which is what `DeductFeeDecorator` actually deducts. The sender pays `baseFee * gas < MinGasPrice * gas`, violating the MinGasPrice invariant.

### Finding Description

`MinGasPriceDecorator.AnteHandle` checks `feeCoins.IsAnyGTE(requiredFees)` where `feeCoins = feeTx.GetFee()` — the raw declared fee coins from the tx body: [1](#0-0) 

This check passes when `feeCap * gas >= MinGasPrice * gas`, i.e., `feeCap >= MinGasPrice`.

Later, `NewDeductFeeDecorator` calls `txFeeChecker = NewDynamicFeeChecker(...)`, which computes:

```
effectivePrice = min(baseFee + maxPriorityPrice, feeCap)
               = min(baseFee + 0, MinGasPrice)
               = baseFee          // since baseFee < MinGasPrice
effectiveFee   = baseFee * gas
``` [2](#0-1) 

The `EffectiveGasPrice` function confirms this: [3](#0-2) 

`DeductFeeDecorator` deducts `effectiveFee = baseFee * gas`, not `MinGasPrice * gas`. The ante chain order in `newCosmosAnteHandler` places `MinGasPriceDecorator` before `DeductFeeDecorator`: [4](#0-3) 

This is enabled in production with `DynamicFeeChecker: true`: [5](#0-4) 

### Impact Explanation

The `MinGasPrice` feemarket parameter is a **consensus-level** global minimum enforced on both `CheckTx` and `DeliverTx`. An attacker can submit Cosmos txs that pass the `MinGasPrice` gate check but are deducted at `baseFee` (which can be arbitrarily lower than `MinGasPrice` during low-traffic periods). This is a fee mis-accounting bug: the sender pays less than the protocol-mandated minimum, and the fee pool receives less than intended. This fits the High impact category: "ante handler bug that permits valid user funds/fees to be mis-accounted."

### Likelihood Explanation

The precondition `baseFee < MinGasPrice` is reachable in normal operation: `baseFee` decreases when blocks are below the gas target, while `MinGasPrice` is a governance-set floor. The attack requires no privileges — any tx submitter can attach `ExtensionOptionDynamicFeeTx` with `MaxPriorityPrice = 0` and set `feeCap = MinGasPrice`. The path is deterministic and locally testable.

### Recommendation

`MinGasPriceDecorator` should validate the **effective fee** (as computed by EIP-1559 logic), not the declared fee. Either:
1. Move the MinGasPrice check into `NewDynamicFeeChecker` after computing `effectivePrice`, rejecting if `effectivePrice < MinGasPrice`, or
2. In `MinGasPriceDecorator`, when `ExtensionOptionDynamicFeeTx` is present, compute `effectivePrice = min(baseFee + maxPriorityPrice, feeCap)` and validate that against `MinGasPrice`.

Note that `CheckEthMinGasPrice` (used for Ethereum txs) already does this correctly by calling `ethMsg.GetEffectiveFee(baseFee)`: [6](#0-5) 

The Cosmos tx path lacks the equivalent effective-fee check.

### Proof of Concept

```
MinGasPrice = 2, baseFee = 1, gas = 1000
MaxPriorityPrice = 0, feeCap = 2 (= MinGasPrice)
declared fee = feeCap * gas = 2000

MinGasPriceDecorator: feeCoins(2000) >= requiredFees(2000) → PASS
NewDynamicFeeChecker:
  effectivePrice = min(1 + 0, 2) = 1
  effectiveFee   = 1 * 1000 = 1000
DeductFeeDecorator deducts: 1000  (not 2000)

Invariant violated: deducted(1000) < MinGasPrice*gas(2000)
```
