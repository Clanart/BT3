### Title
Missing Declared Class Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS does not verify that the new class hash provided to the `replace_class` syscall corresponds to a declared contract class. This is an acknowledged missing check (marked with a TODO). An unprivileged transaction sender can exploit a contract with a user-controlled upgrade path to permanently replace its class with an undeclared hash, making the contract permanently non-executable and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class` (lines 877–916 of `syscall_impls.cairo`), the OS reads the new class hash from the syscall request and directly updates the contract's `StateEntry` in `contract_state_changes` without verifying that the new class hash is present in the declared class registry (`contract_class_changes`):

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges the missing check. The `contract_class_changes` dictionary (populated by `execute_declare_transaction`) is the authoritative registry of declared classes, but it is never consulted here. [2](#0-1) 

By contrast, `execute_declare_transaction` enforces `prev_value=0` to prevent double-declaration and uses `assert_not_zero(compiled_class_hash)` to ensure validity: [3](#0-2) 

No equivalent guard exists in `execute_replace_class`. The class hash written into the state is never cross-checked against `contract_class_changes` or the compiled class facts bundle.

The analog to the external report is direct: in the Ajna bug, a tracked LP balance is zeroed out without first processing the rewards that depend on it. Here, a tracked class hash is overwritten without first validating that the new value is a declared, executable class — the dependent state (class existence) is never checked before the state transition is committed.

---

### Impact Explanation

When a contract's class hash is set to an undeclared value, every subsequent call to that contract will fail at execution time because the OS cannot locate the class in the compiled class facts bundle (validated post-execution in `validate_compiled_class_facts_post_execution`). The state commitment, however, is finalized with the invalid class hash. There is no recovery path: the contract is permanently non-executable.

Any funds (ETH, ERC20 tokens, or other assets) held in the contract's storage are permanently frozen. This matches the **Critical — Permanent freezing of funds** impact category. [4](#0-3) 

---

### Likelihood Explanation

The `replace_class` syscall is reachable by any contract via `execute_syscalls` when `selector == REPLACE_CLASS_SELECTOR`. An unprivileged transaction sender can trigger this path if a target contract exposes any publicly callable function that passes user-controlled input to `replace_class` — a common pattern in upgradeable proxy contracts. The attacker does not need any privileged role; they only need to submit a valid transaction to a vulnerable contract. [5](#0-4) 

---

### Recommendation

Before committing the new class hash in `execute_replace_class`, add a Cairo-level assertion that the class hash exists in `contract_class_changes` (i.e., it was declared in the current or a prior block). This mirrors the validation already enforced in `execute_declare_transaction` and closes the gap acknowledged by the TODO comment.

---

### Proof of Concept

1. Attacker deploys contract `C` with a public `upgrade(new_class_hash)` function that calls `replace_class(new_class_hash)`.
2. Attacker (or any user) deposits funds into `C`.
3. Attacker calls `C.upgrade(undeclared_hash)` where `undeclared_hash` is any felt not present in `contract_class_changes`.
4. The OS executes `execute_replace_class`: gas is deducted, the state entry for `C` is updated to `class_hash = undeclared_hash`, and the old class hash is logged in the revert log. No declared-class check is performed.
5. The block is finalized. The state commitment now records `C.class_hash = undeclared_hash`.
6. In any subsequent block, any transaction targeting `C` fails: the OS reads `class_hash = undeclared_hash` from state, cannot find it in the compiled class facts, and the execution aborts.
7. All funds in `C`'s storage are permanently inaccessible. [6](#0-5)

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
