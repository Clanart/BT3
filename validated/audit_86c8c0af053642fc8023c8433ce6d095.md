### Title
P2P Sync Accepts State Diff Chunks Without Verifying Hash Against Signed Header Commitment - (File: `crates/apollo_p2p_sync/src/client/state_diff.rs`)

### Summary
The P2P sync client assembles a `ThinStateDiff` from peer-supplied chunks and validates only the total element count against the header's `state_diff_length` field. It never verifies the assembled diff's Poseidon hash against the `state_diff_commitment` stored in the signed block header. A malicious peer can serve chunks that sum to the correct length but carry wrong storage/nonce/class data, causing the node to persist a corrupted state diff. The RPC then returns authoritative-looking wrong values for `starknet_getStateUpdate`, and — when `verify_state_diff_hash` is disabled — the committer derives a wrong global state root.

### Finding Description

`parse_data_for_block` in `crates/apollo_p2p_sync/src/client/state_diff.rs` reads `target_state_diff_len` from the stored block header and accumulates `StateDiffChunk` messages until `current_state_diff_len == target_state_diff_len`: [1](#0-0) 

The only post-assembly check is `validate_deprecated_declared_classes_non_conflicting`, which detects duplicate deprecated class hashes but does not compute or compare the state diff Poseidon hash. [2](#0-1) 

The signed block header carries both `state_diff_length` (used above) and `state_diff_commitment` (never consulted here): [3](#0-2) 

The canonical commitment algorithm is `calculate_state_diff_hash`, which hashes deployed contracts, declared classes, deprecated classes, storage diffs, and nonces into a single Poseidon digest: [4](#0-3) 

Because `chain_storage_diffs` counts only non-empty inner maps, a diff with N non-empty storage entries and M empty-inner-map entries has `len() == N` but a different hash than a diff with only N entries and no empty maps. A malicious peer can therefore craft chunks that pass the length gate while producing a different `StateDiffCommitment`.

The assembled (wrong) diff is written directly to `apollo_storage` via `append_state_diff`: [5](#0-4) 

The central sync path has an analogous gap, explicitly noted as a TODO: [6](#0-5) 

### Impact Explanation

Once the corrupted `ThinStateDiff` is in storage, `get_state_update` in the RPC layer reads it directly and returns it to callers: [7](#0-6) 

This satisfies the **High** impact: *"RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."*

When `verify_state_diff_hash` is `false` (a supported configuration), the committer skips the cross-check and applies the wrong diff to the Patricia trie, producing a wrong `GlobalRoot` and therefore a wrong block hash — satisfying the **Critical** impact: *"Wrong state … storage value … from blockifier/syscall/execution logic."* [8](#0-7) [9](#0-8) 

### Likelihood Explanation

Any node on the P2P network can act as a peer. No special privilege is required. The attacker only needs to serve chunks whose element count matches `state_diff_length` while substituting different storage keys, values, nonces, or class hashes. The length check is the sole gate, and it is trivially satisfied.

### Recommendation

After `unite_state_diffs` accumulates all chunks and `current_state_diff_len == target_state_diff_len`, compute `calculate_state_diff_hash(&result)` and compare it against `header.state_diff_commitment`. If the header's commitment is `None` (pre-commitment blocks), skip the check. Reject the peer and abort the session on mismatch, consistent with the existing `BadPeerError` pattern.

### Proof of Concept

1. A syncing node connects to a malicious peer.
2. The node requests state diff chunks for block N whose header (from a trusted source) has `state_diff_length = 3` and `state_diff_commitment = H_correct`.
3. The malicious peer sends three `StateDiffChunk::ContractDiff` messages, each contributing 1 to the length count, but with storage values replaced by attacker-chosen felts.
4. `current_state_diff_len` reaches `target_state_diff_len = 3`; `validate_deprecated_declared_classes_non_conflicting` passes (no deprecated classes involved).
5. The corrupted `ThinStateDiff` is written to storage. `calculate_state_diff_hash` on this diff yields `H_wrong ≠ H_correct`.
6. `starknet_getStateUpdate(block_id=N)` now returns the attacker-chosen storage values as authoritative chain state.
7. If `verify_state_diff_hash = false`, the committer applies the wrong diff to the trie, producing a wrong `GlobalRoot` that propagates into the block hash and all downstream proof inputs. [10](#0-9)

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}

        thin_state_diff: ThinStateDiff,

        let markers_table = self.open_table(&self.tables.markers)?;
        let state_diffs_table = self.open_table(&self.tables.state_diffs)?;
        let nonces_table = self.open_table(&self.tables.nonces)?;
        let deployed_contracts_table = self.open_table(&self.tables.deployed_contracts)?;
        let storage_table = self.open_table(&self.tables.contract_storage)?;
        let declared_classes_block_table = self.open_table(&self.tables.declared_classes_block)?;
        let deprecated_declared_classes_block_table =
            self.open_table(&self.tables.deprecated_declared_classes_block)?;
        let compiled_class_hash_table = self.open_table(&self.tables.compiled_class_hash)?;

        // Write state.
        write_deployed_contracts(
            &thin_state_diff.deployed_contracts,
            &self.txn,
            block_number,
            &deployed_contracts_table,
            &nonces_table,
        )?;
        write_storage_diffs(
            &thin_state_diff.storage_diffs,
            &self.txn,
            block_number,
            &storage_table,
        )?;
        // Must be called after write_deployed_contracts since the nonces are updated there.
        write_nonces(&thin_state_diff.nonces, &self.txn, block_number, &nonces_table)?;

        for (class_hash, _) in &thin_state_diff.class_hash_to_compiled_class_hash {
            let not_declared = declared_classes_block_table.get(&self.txn, class_hash)?.is_none();
            if not_declared {
                declared_classes_block_table.insert(&self.txn, class_hash, &block_number)?;
            }

        write_compiled_class_hashes(
            &thin_state_diff.class_hash_to_compiled_class_hash,
            &self.txn,
            block_number,
            &compiled_class_hash_table,

            // Cairo0 classes can be declared in different blocks. The first block to declare the

            }
