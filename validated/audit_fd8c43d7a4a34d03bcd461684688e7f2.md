### Title
RPC `CalcBaseFee` Omits `MinGasMultiplier` Adjustment, Causing Systematic Base Fee Underestimation - (File: `rpc/backend/utils.go`)

---

### Summary

The RPC-layer `CalcBaseFee` function uses raw `gasUsed` as the gas input, while the consensus `CalculateBaseFee` uses `gasWanted` adjusted by `MinGasMultiplier`. This mirrors the Teller Finance bug class exactly: a cap/threshold variable (`MinGasMultiplier`) is missing from a ratio calculation, making the computed value appear lower than it actually is. The `eth_feeHistory` and `NextBaseFee` JSON-RPC endpoints therefore return systematically underestimated base fees, feeding incorrect consensus-critical data to every client.

---

### Finding Description

**Consensus path** (`x/feemarket/keeper/abci.go`, `EndBlock`):

```go
minGasMultiplier := k.GetParams(ctx).MinGasMultiplier
limitedGasWanted := sdkmath.LegacyNewDec(gw).Mul(minGasMultiplier)
gasWanted = sdkmath.LegacyMaxDec(limitedGasWanted, sdkmath.LegacyNewDec(gasUsed)).TruncateInt().Uint64()
k.SetBlockGasWanted(ctx, gasWanted)
```

`gasWanted = max(originalGasWanted × MinGasMultiplier, gasUsed)` is persisted as `BlockGasWanted`.

**Consensus path** (`x/feemarket/keeper/eip1559.go`, `CalculateBaseFee`):

```go
parentGasUsed := k.GetBlockGasWanted(ctx)   // ← the MinGasMultiplier-adjusted value
```

The base fee is computed from the adjusted `gasWanted`, which is ≥ `gasUsed` whenever `originalGasWanted × MinGasMultiplier > gasUsed`.

**RPC path** (`rpc/backend/utils.go`, `CalcBaseFee`):

```go
func CalcBaseFee(config *params.ChainConfig, parent *ethtypes.Header, p feemarkettypes.Params) (*big.Int, error) {
    ...
    parentGasTarget := parent.GasLimit / uint64(p.ElasticityMultiplier)
    if parent.GasUsed == parentGasTarget { ... }   // ← uses raw gasUsed, no MinGasMultiplier
    ...
}
```

`CalcBaseFee` receives `parent.GasUsed` set to the raw block gas consumed:

- In `NextBaseFee` (`rpc/backend/chain_info.go` line 151/168): `gasUsed, err := computeGasUsed(blockRes)` → `header.GasUsed = gasUsed`
- In `processBlock` (`rpc/backend/utils.go` line 186/201): `gasUsed, ok := (*ethBlock)["gasUsed"].(hexutil.Uint64)` → `header.GasUsed = uint64(gasUsed)`

Neither call applies the `MinGasMultiplier` adjustment. The `feemarkettypes.Params` struct (which includes `MinGasMultiplier`) is passed to `CalcBaseFee` but the field is never read inside the function.

**Concrete divergence**: With default `MinGasMultiplier = 0.5`, if a block has `originalGasWanted = 1 000 000` and `gasUsed = 400 000`:

| Path | Gas input to base fee formula |
|---|---|
| Consensus | `max(1 000 000 × 0.5, 400 000) = 500 000` |
| RPC | `400 000` |

The RPC predicts a lower (or even decreasing) base fee while the consensus sets a higher one.

---

### Impact Explanation

Every client that calls `eth_feeHistory` or `eth_nextBaseFee` receives a `nextBaseFee` that is lower than the base fee the chain will actually commit. Wallets and dApps use these values to set `maxFeePerGas`. When the actual committed base fee exceeds the client-predicted value, EIP-1559 transactions are rejected at the ante handler with "max fee per gas less than block base fee". This is a systematic, always-reachable mis-accounting of consensus-critical fee data through the public JSON-RPC surface, matching the allowed High impact: *"Public JSON-RPC path feeds incorrect consensus-critical data into transaction execution."*

---

### Likelihood Explanation

The divergence is triggered whenever `gasUsed < originalGasWanted × MinGasMultiplier`. With the default `MinGasMultiplier = 0.5`, this occurs for any block where transactions collectively use less than half their declared gas limit — a common pattern for reverting transactions, complex contract calls with conservative gas estimates, or any block that is not fully packed. The entry path is entirely unprivileged: any user submitting a normal EVM transaction can trigger the condition.

---

### Recommendation

Apply the same `MinGasMultiplier` adjustment inside `CalcBaseFee` before using `parent.GasUsed` as the gas input:

```go
func CalcBaseFee(config *params.ChainConfig, parent *ethtypes.Header, p feemarkettypes.Params) (*big.Int, error) {
    ...
    // Mirror the EndBlock adjustment: gasWanted = max(gasUsed * MinGasMultiplier, gasUsed)
    // Here parent.GasUsed represents gasWanted from the block.
    adjustedGasUsed := sdkmath.LegacyNewDec(int64(parent.GasUsed)).Mul(p.MinGasMultiplier)
    effectiveGasUsed := sdkmath.LegacyMaxDec(adjustedGasUsed, sdkmath.LegacyNewDec(int64(parent.GasUsed))).TruncateInt().Uint64()
    parentGasTarget := parent.GasLimit / uint64(p.ElasticityMultiplier)
    if effectiveGasUsed == parentGasTarget { ... }
    ...
}
```

Alternatively, expose `BlockGasWanted` via a query endpoint and use it directly in `NextBaseFee` and `processBlock` instead of recomputing from `gasUsed`.

---

### Proof of Concept

1. Deploy any contract with a function that reverts after consuming ~30% of its declared gas limit.
2. Fill a block with such transactions (`gasWanted` = block gas limit, `gasUsed` ≈ 30% of limit).
3. With `MinGasMultiplier = 0.5`: consensus stores `gasWanted = max(0.5 × limit, 0.3 × limit) = 0.5 × limit`.
4. `CalculateBaseFee` sees `parentGasUsed = 0.5 × limit`; if `gasTarget = limit / 2`, base fee is unchanged or increases.
5. `CalcBaseFee` (RPC) sees `parent.GasUsed = 0.3 × limit`; since `0.3 × limit < gasTarget`, it predicts a **decreasing** base fee.
6. Call `eth_feeHistory` — the returned `nextBaseFee` is lower than the actual committed base fee.
7. Submit a transaction with `maxFeePerGas = nextBaseFee` from step 6 — it is rejected by the ante handler. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

// CalcBaseFee calculates the basefee of the header.
func CalcBaseFee(config *params.ChainConfig, parent *ethtypes.Header, p feemarkettypes.Params) (*big.Int, error) {
	// If the current block is the first EIP-1559 block, return the InitialBaseFee.
	if !config.IsLondon(parent.Number) {
		return new(big.Int).SetUint64(params.InitialBaseFee), nil
	}
	if p.ElasticityMultiplier == 0 {
		return nil, errors.New("ElasticityMultiplier cannot be 0 as it's checked in the params validation")
	}
	parentGasTarget := parent.GasLimit / uint64(p.ElasticityMultiplier)
	// If the parent gasUsed is the same as the target, the baseFee remains unchanged.
	if parent.GasUsed == parentGasTarget {
		return new(big.Int).Set(parent.BaseFee), nil
	}

	var (
		num   = new(big.Int)
		denom = new(big.Int)
	)

	if parent.GasUsed > parentGasTarget {
		// If the parent block used more gas than its target, the baseFee should increase.
		// max(1, parentBaseFee * gasUsedDelta / parentGasTarget / baseFeeChangeDenominator)
		num.SetUint64(parent.GasUsed - parentGasTarget)
		num.Mul(num, parent.BaseFee)
		num.Div(num, denom.SetUint64(parentGasTarget))
		num.Div(num, denom.SetUint64(uint64(p.BaseFeeChangeDenominator)))
		baseFeeDelta := ethermint.BigMax(num, common.Big1)
		return num.Add(parent.BaseFee, baseFeeDelta), nil
	}

	// Otherwise if the parent block used less gas than its target, the baseFee should decrease.
	// max(0, parentBaseFee * gasUsedDelta / parentGasTarget / baseFeeChangeDenominator)
	num.SetUint64(parentGasTarget - parent.GasUsed)
	num.Mul(num, parent.BaseFee)
	num.Div(num, denom.SetUint64(parentGasTarget))
	num.Div(num, denom.SetUint64(uint64(p.BaseFeeChangeDenominator)))
	baseFee := num.Sub(parent.BaseFee, num)
	minGasPrice := p.MinGasPrice.TruncateInt().BigInt()
	return ethermint.BigMax(baseFee, minGasPrice), nil
}

// computeGasUsed sums the gas used by the transactions in the block result,
// stopping at the first tx that should be ignored per ShouldIgnoreGasUsed
// (cosmos-sdk bug: https://github.com/cosmos/cosmos-sdk/issues/10832).
func computeGasUsed(blockRes *tmrpctypes.ResultBlockResults) (uint64, error) {
	var gasUsed uint64
	for _, txsResult := range blockRes.TxsResults {
		if ShouldIgnoreGasUsed(txsResult) {
			// block gas limit has exceeded, other txs must have failed with same reason.
			break
		}
		gas, err := ethermint.SafeUint64(txsResult.GetGasUsed())
		if err != nil {
			return 0, err
		}
		gasUsed += gas
	}
	return gasUsed, nil
}
