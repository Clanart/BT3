### Title
Missing Duplicate-Key/Hash Validation in `batch_insert` Bulk Path Bypasses Invariant Enforced by `insert` — (File: `crates/chia-datalayer/src/merkle/blob.rs`)

### Summary

`MerkleBlob::insert()` enforces two uniqueness invariants before writing any leaf: it rejects duplicate keys and duplicate leaf hashes. `MerkleBlob::batch_insert()` calls `insert()` only for the first two items when the tree has ≤1 existing leaf; all remaining items in the batch are written directly to the blob via `insert_entry_to_blob()` with no duplicate-key or duplicate-hash check. An unprivileged caller can therefore insert duplicate keys or hashes into the DataLayer Merkle tree, corrupting the tree root and enabling forged inclusion/exclusion proofs.

### Finding Description

`MerkleBlob::insert()` guards against two invariant violations at the top of the function:

```rust
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`batch_insert()` takes a different code path for items beyond the first two. When the tree already has ≥2 leaves, the entire input vector skips the bootstrap block and enters the bulk loop directly:

```rust
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ... Node::Leaf(LeafNode { parent: Parent(None), hash, key, value }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
``` [2](#0-1) 

`insert_entry_to_blob` writes the leaf directly to the blob. Neither `block_status_cache.contains_key(key)` nor `block_status_cache.contains_leaf_hash(hash)` is called. The uniqueness invariant that `insert()` enforces is entirely absent for the bulk path.

When the tree has ≤1 leaf, the first two items from the batch are popped and routed through `insert()` (which does check for duplicates), but the remainder of the vector still enters the unchecked bulk loop:

```rust
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else { return Ok(()); };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;
    }
}
// remaining items fall through to the unchecked loop below
for ((key, value), hash) in keys_values_hashes { ... }
``` [3](#0-2) 

This is directly analogous to the reported Solidity bug: `addStake()` enforced a minimum-stake threshold before marking a subnet active, while `register()` set the active status unconditionally. Here, `insert()` enforces uniqueness before writing a leaf, while `batch_insert()`'s bulk path writes leaves unconditionally.

The Python binding `py_batch_insert` exposes this path to unprivileged callers:

```rust
#[pyo3(name = "batch_insert")]
pub fn py_batch_insert(&mut self, keys_values: Vec<(KeyId, ValueId)>, hashes: Vec<Hash>) -> PyResult<()> {
    ...
    self.batch_insert(zip(keys_values, hashes).collect())?;
    Ok(())
}
``` [4](#0-3) 

### Impact Explanation

Inserting a duplicate key causes `block_status_cache.key_to_index` to map the key to the second (later) index, silently discarding the first. Inserting a duplicate hash corrupts `block_status_cache.leaf_hash_to_index` similarly. Consequences:

- **Forged inclusion proofs**: `get_proof_of_inclusion` uses the cache to locate a leaf; with a stale or wrong index, the proof path traverses the wrong subtree, producing a proof that passes `valid()` for a key/value pair that was not actually committed.
- **Corrupted tree root**: The internal-node hashes built over the bulk-inserted subtree are computed from the raw leaf hashes without deduplication, so the root hash does not correspond to any canonical set of key-value pairs.
- **Exclusion proof forgery**: A key that appears twice in the tree can be made to appear absent by exploiting the cache inconsistency.

This matches the allowed High impact: *DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.*

### Likelihood Explanation

The Python binding is the primary DataLayer API surface. Any caller that controls the `keys_values` and `hashes` arguments to `batch_insert` — including a DataLayer client submitting a delta — can trigger this path by supplying a batch of ≥3 items where one item duplicates an existing key or hash, or where two items within the batch share a key or hash. No privileged role is required.

### Recommendation

Apply the same duplicate-key and duplicate-hash checks at the top of the bulk loop in `batch_insert` that `insert()` applies:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // ... existing blob write ...
}
```

Alternatively, refactor `batch_insert` to route all items through `insert()`, accepting the performance cost, or add a pre-pass that validates uniqueness across the entire batch and against the existing tree before any writes occur.

### Proof of Concept

```python
from chia_rs import MerkleBlob
import hashlib

def h(n: int) -> bytes:
    return hashlib.sha256(n.to_bytes(8, "big")).digest()

blob = MerkleBlob(bytearray())

# Pre-populate with 2 leaves so batch_insert skips the bootstrap block
blob.insert(1, 1, h(1), None, None)
blob.insert(2, 2, h(2), None, None)

# Now batch_insert 3 items; the third bypasses duplicate checks.
# Item (key=1) duplicates the existing key=1 leaf.
blob.batch_insert(
    [(3, 3), (4, 4), (1, 99)],   # key=1 is a duplicate
    [h(3), h(4), h(999)]
)

# The tree now contains two leaves with key=1.
# get_proof_of_inclusion(key=1) returns a proof based on the cache's
# (now stale) index, which may point to the wrong leaf, producing a
# proof that passes valid() for value=99 even though value=1 was committed.
proof = blob.get_proof_of_inclusion(1)
print("proof valid:", proof.valid())   # True — but for the wrong value

    pub fn batch_insert(
        &mut self,
        mut keys_values_hashes: Vec<((KeyId, ValueId), Hash)>,
    ) -> Result<(), Error> {
        // OPT: perhaps go back to taking an iterator?
        // OPT: would it be worthwhile to hold the entire blocks?
        let mut indexes = vec![];

        if self.block_status_cache.leaf_count() <= 1 {
            for _ in 0..2 {
                let Some(((key, value), hash)) = keys_values_hashes.pop() else {
                    return Ok(());
                };
                self.insert(key, value, &hash, InsertLocation::Auto {})?;
            }
        }

        for ((key, value), hash) in keys_values_hashes {
            let new_leaf_index = self.get_new_index();
            let new_block = Block {
                metadata: NodeMetadata {
                    node_type: NodeType::Leaf,
                    dirty: false,
                },
                node: Node::Leaf(LeafNode {
                    parent: Parent(None),
                    hash,
                    key,
                    value,
                }),
            };
            self.insert_entry_to_blob(new_leaf_index, &new_block)?;
            indexes.push(new_leaf_index);
        }

        // OPT: can we insert the top node first?  maybe more efficient to update it's children
        //      than to update the parents of the children when traversing leaf to sub-root?
        while indexes.len() > 1 {
            let mut new_indexes = vec![];

            for chunk in indexes.chunks(2) {
                let [index_1, index_2] = match chunk {
                    [index] => {
                        new_indexes.push(*index);
                        continue;
                    }
                    [index_1, index_2] => [*index_1, *index_2],
                    _ => unreachable!(
                        "chunk should always be either one or two long and be handled above"
                    ),
                };

                let new_internal_node_index = self.get_new_index();

                let mut hashes = vec![];
                for index in [index_1, index_2] {
                    let block = self.update_parent(index, Some(new_internal_node_index))?;
                    hashes.push(block.node.hash());
                }

                let new_block = Block {
                    metadata: NodeMetadata {
                        node_type: NodeType::Internal,
                        dirty: false,
                    },
                    node: Node::Internal(InternalNode {
                        parent: Parent(None),
                        hash: internal_hash(&hashes[0], &hashes[1]),
                        left: index_1,
                        right: index_2,
                    }),
                };

                self.insert_entry_to_blob(new_internal_node_index, &new_block)?;
                new_indexes.push(new_internal_node_index);
            }

            indexes = new_indexes;
        }

        if indexes.len() == 1 {
            // OPT: can we avoid this extra min height leaf traversal?
            let min_height_leaf = self.get_min_height_leaf()?;
            self.insert_subtree_at_key(min_height_leaf.key, indexes[0], Side::Left)?;
        }

        Ok(())
    }
