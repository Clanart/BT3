### Title
Unbounded `epoch_height` in `SnapshotHostInfo` Enables Permanent Eviction of Legitimate Snapshot Hosts from Bounded LRU Cache — (`chain/network/src/snapshot_hosts/mod.rs`)

---

### Summary

Any unprivileged network peer can craft `SnapshotHostInfo` messages with an arbitrarily large `epoch_height` (e.g., `u64::MAX`) signed under freshly-generated key pairs. Because the bounded `SnapshotHostsCache` is an LRU cache keyed by `PeerId`, and because `epoch_height` is never validated against actual chain state, an attacker with ≥ `snapshot_hosts_cache_size` (default 1000) distinct key pairs can permanently fill the cache with fake entries, evicting every legitimate snapshot host. Entries carrying `epoch_height = u64::MAX` can never be replaced or discarded by the epoch-advancement mechanism, so state sync on the victim node stalls indefinitely until a restart.

---

### Finding Description

`SnapshotHostsCache` stores one `SnapshotHostInfo` per `PeerId` in a fixed-capacity LRU cache:

```rust
// chain/network/src/snapshot_hosts/mod.rs  L138
hosts: LruCache<PeerId, Arc<SnapshotHostInfo>>,
``` [1](#0-0) 

The only admission check is `is_new`, which compares the incoming `epoch_height` against the stored one for the same `peer_id`:

```rust
fn is_new(&self, h: &SnapshotHostInfo) -> bool {
    if self.discard_snapshot_infos_below_epoch_height
        .is_some_and(|min_epoch| min_epoch > h.epoch_height) {
        return false;
    }
    match self.hosts.peek(&h.peer_id) {
        Some(old) if old.epoch_height >= h.epoch_height => false,
        _ => true,
    }
}
``` [2](#0-1) 

`epoch_height` is **never validated against the actual chain**. The `SnapshotHostInfo` signature only covers `(sync_hash, epoch_height, shards)`:

```rust
pub(crate) fn verify(&self) -> Result<(), SnapshotHostInfoVerificationError> {
    if self.shards.len() > MAX_SHARDS_PER_SNAPSHOT_HOST_INFO {
        return Err(...);
    }
    if !self.signature.verify(self.hash().as_ref(), self.peer_id.public_key()) {
        return Err(...);
    }
    Ok(())
}
``` [3](#0-2) 

So a peer can sign a message with `epoch_height = u64::MAX` and any `sync_hash`, and it passes all checks.

When the cache is full, `insert` calls `hosts.push`, which evicts the LRU entry and removes it from `hosts_for_shard`:

```rust
fn insert(&mut self, d: &Arc<SnapshotHostInfo>) {
    self.add_to_shard_hosts(d.as_ref());
    if let Some((displaced, _)) = self.hosts.push(d.peer_id.clone(), d.clone()) {
        if displaced != d.peer_id {
            self.remove_from_shard_hosts(&displaced);
        }
    }
}
``` [4](#0-3) 

The epoch-advancement discard path (`update_discard_epoch_threshold`) computes `min_epoch_to_keep = epoch_height - 1` and removes entries below it. An entry with `epoch_height = u64::MAX` satisfies `info.epoch_height >= min_epoch_to_keep` for every possible `min_epoch_to_keep`, so it is **never discarded**:

```rust
fn update_discard_epoch_threshold(&mut self, epoch_height: EpochHeight) {
    let min_epoch_to_keep = epoch_height.saturating_sub(self.epoch_retention_window);
    ...
    if info.epoch_height >= min_epoch_to_keep {
        new_hosts.push(peer_id, info);   // fake entry always survives
    }
}
``` [5](#0-4) 

The default cache size is 1000:

```rust
fn default_snapshot_hosts_cache_size() -> u32 { 1000 }
``` [6](#0-5) 

Incoming `SyncSnapshotHosts` messages are accepted from any connected peer and processed without banning unless the signature is invalid, shards exceed the limit, or duplicate `peer_id`s appear in a single batch:

```rust

A valid entry with `epoch_height = u64::MAX` triggers **no ban**.

---

### Impact Explanation

Once the attacker fills the 1000-slot cache with fake entries:

1. All legitimate `SnapshotHostInfo` entries are evicted and removed from `hosts_for_shard`.
2. `update_current_state_sync_hash` rebuilds `hosts_for_shard` from the current cache, but finds only fake entries whose `sync_hash` does not match the real sync hash, so `hosts_for_shard` remains empty.
3. `select_host_for_header` and `select_host_for_part` return `None`, and state sync cannot proceed.
4. The fake entries can never be replaced (no real `epoch_height` can exceed `u64::MAX`) and are never discarded by epoch advancement.
5. The victim node's state sync stalls permanently until a restart clears the in-memory cache.

**Impact: High** — state sync is a critical path for nodes joining or catching up to the network. A stalled state sync means the node cannot participate in consensus or serve data.

---

### Likelihood Explanation

- Generating 1000 ED25519 key pairs is trivial (milliseconds on commodity hardware).
- The attacker needs only a single TCP connection to the victim node to deliver the messages.
- No stake, validator role, or privileged access is required.
- The attack is permanent per-run: the victim must restart to recover.
- The `snapshot_hosts_broadcast_rate_limit` (0.1 qps) limits outbound re-broadcasting but does **not** limit how fast the victim ingests incoming messages.

---

### Recommendation

1. **Validate `epoch_height` against chain state**: Reject `SnapshotHostInfo` entries whose `epoch_height` exceeds the current chain epoch by more than a small tolerance (e.g., 2 epochs). This requires passing the current epoch height into the verification path.

2. **Cap accepted `epoch_height`**: In `verify()`, add a check that `epoch_height <= current_epoch + TOLERANCE`, analogous to the existing `TooManyShards` check, and ban peers that violate it.

3. **Rate-limit per-peer insertions**: Track how many new entries each connected peer has contributed and ban peers that insert more than a small number of distinct `peer_id` entries per epoch.

4. **Prefer a score-based eviction policy over pure LRU**: Evict entries with the lowest `epoch_height` (least relevant) rather than the least-recently-used, so that fake entries with implausibly high `epoch_height` stand out and can be filtered.

    fn update_discard_epoch_threshold(&mut self, epoch_height: EpochHeight) {
        let min_epoch_to_keep = epoch_height.saturating_sub(self.epoch_retention_window);
        if self.discard_snapshot_infos_below_epoch_height == Some(min_epoch_to_keep) {
            return;
        }

        self.discard_snapshot_infos_below_epoch_height = Some(min_epoch_to_keep);

        // Remove snapshot infos that are now below the retention window
        let mut new_hosts = LruCache::new(NonZeroUsize::new(self.hosts.cap().get()).unwrap());

        loop {
            let Some((peer_id, info)) = self.hosts.pop_lru() else { break };
            if info.epoch_height >= min_epoch_to_keep {
                new_hosts.push(peer_id, info);
            } else {
                self.remove_from_shard_hosts(&peer_id);
            }
        }
        self.hosts = new_hosts;
    }
