### Title
`baseFee.Int64()` Silent Truncation in `SuggestGasTipCap` Produces Incorrect Gas Price Data Fed Into Transaction Construction - (File: rpc/backend/chain_info.go)

### Summary

`SuggestGasTipCap` in `rpc/backend/chain_info.go` calls `baseFee.Int64()` on an unbounded `*big.Int` without any overflow guard. When the EIP-1559 base fee grows beyond `math.MaxInt64` (9.2 ├Ś 10┬╣ŌüĖ), the low-64-bit truncation returns a negative value, which is silently clamped to zero. The function then returns `big.NewInt(0)` as the suggested tip cap. This incorrect value is consumed directly by the `eth_gasPrice`, `eth_maxPriorityFeePerGas` JSON-RPC endpoints and by `SetTxDefaults`, which uses it to construct `MaxPriorityFeePerGas` and `MaxFeePerGas` for transactions submitted via `eth_sendTransaction`.

### Finding Description

In `SuggestGasTipCap`:

```go
maxDelta := baseFee.Int64() * (int64(params.Params.ElasticityMultiplier) - 1) / int64(params.Params.BaseFeeChangeDenominator)
if maxDelta < 0 {
    // impossible if the parameter validation passed.
    maxDelta = 0
}
return big.NewInt(maxDelta), nil
```

`baseFee` is a `*big.Int` sourced from the feemarket module's `BaseFee` parameter, which is stored as `sdkmath.Int` (a 256-bit integer) and computed via `big.Int` arithmetic in `CalculateBaseFee`. There is no upper bound enforced on `BaseFee` by `validateBaseFee` ŌĆö it only checks `value.IsNegative()`. [1](#0-0) [2](#0-1) 

Go's `big.Int.Int64()` returns the low 64 bits of the absolute value with the sign of x. When `baseFee > math.MaxInt64`, the result is a negative `int64` (e.g., `baseFee = 2^63` ŌåÆ `Int64() = -2^63`). The `maxDelta < 0` branch clamps this to 0, and the function returns `big.NewInt(0)`.

The comment "impossible if the parameter validation passed" is incorrect ŌĆö parameter validation only prevents `BaseFee` from being negative, not from exceeding `math.MaxInt64`. [3](#0-2) 

`SuggestGasTipCap` is called from three production paths:

1. `GasPrice()` ŌåÆ `eth_gasPrice` RPC: returns `baseFee + 0 = baseFee` instead of `baseFee + correctTip`.
2. `MaxPriorityFeePerGas()` ŌåÆ `eth_maxPriorityFeePerGas` RPC: returns `0` instead of the correct tip.
3. `SetTxDefaults()` ŌåÆ used when constructing transactions via `eth_sendTransaction`: sets `MaxPriorityFeePerGas = 0` and `MaxFeePerGas = 2 * baseFee`. [4](#0-3) [5](#0-4) [6](#0-5) 

### Impact Explanation

When `baseFee > math.MaxInt64`, `eth_gasPrice` and `eth_maxPriorityFeePerGas` return incorrect (too-low) values. `SetTxDefaults` constructs transactions with `MaxFeePerGas = 2 * baseFee` and `MaxPriorityFeePerGas = 0`. Because the base fee is actively growing (blocks are full ŌĆö that is what drove it above `MaxInt64`), the next block's base fee will be up to `1.125 * baseFee`. A transaction with `MaxFeePerGas = 2 * baseFee` remains valid for several more blocks, but with zero priority tip it will be consistently deprioritized or dropped from the mempool as the base fee continues to rise. Clients relying on `eth_gasPrice` to set a legacy `gasPrice` will also receive a value that is too low, causing their transactions to be rejected by the ante handler's `EthMempoolFeeDecorator` / `CheckEthMinGasPrice` checks.

This is a public JSON-RPC path feeding incorrect fee-market data into transaction construction and execution.

### Likelihood Explanation

The feemarket `CalculateBaseFee` uses `big.Int` arithmetic with no upper cap on `BaseFee`. [7](#0-6) 

With default parameters (`BaseFeeChangeDenominator = 8`, `ElasticityMultiplier = 2`, `InitialBaseFee = 10^9`), the base fee can increase by at most 12.5% per block. Starting from 1 Gwei, reaching `math.MaxInt64` (Ōēł 9.2 ├Ś 10┬╣ŌüĖ) requires approximately 195 consecutive fully-saturated blocks. On a chain with high sustained throughput or a low initial base fee, this threshold is reachable. Additionally, governance can set `BaseFee` directly to any non-negative value via parameter update, instantly triggering the overflow.

### Recommendation

Replace the `int64` arithmetic with `big.Int` arithmetic throughout `SuggestGasTipCap`:

```go
elasticity := new(big.Int).SetUint64(uint64(params.Params.ElasticityMultiplier))
denominator := new(big.Int).SetUint64(uint64(params.Params.BaseFeeChangeDenominator))
one := big.NewInt(1)

// maxDelta = baseFee * (ElasticityMultiplier - 1) / BaseFeeChangeDenominator
maxDelta := new(big.Int).Mul(baseFee, new(big.Int).Sub(elasticity, one))
maxDelta.Div(maxDelta, denominator)
if maxDelta.Sign() < 0 {
    maxDelta = new(big.Int)
}
return maxDelta, nil
```

### Proof of Concept

1. Deploy an Ethermint chain with default feemarket parameters (`BaseFeeChangeDenominator=8`, `ElasticityMultiplier=2`, `InitialBaseFee=1_000_000_000`).
2. Fill every block to 100% gas capacity for ~195 consecutive blocks. The base fee grows by 12.5% per block: after 195 blocks, `baseFee Ōēł 1e9 * 1.125^195 Ōēł 9.4e18 > math.MaxInt64`.
3. Alternatively, submit a governance proposal to set `BaseFee` to `math.MaxInt64 + 1` directly.
4. Call `eth_maxPriorityFeePerGas` ŌĆö it returns `0x0` instead of the correct positive tip.
5. Call `eth_gasPrice` ŌĆö it returns exactly `baseFee` with no tip component.
6. Call `eth_sendTransaction` without specifying `maxFeePerGas` ŌĆö `SetTxDefaults` sets `MaxPriorityFeePerGas=0` and `MaxFeePerGas=2*baseFee`. As the base fee continues to rise past `2*baseFee`, the submitted transaction is rejected by the ante handler with `insufficient gas prices`. [8](#0-7) [9](#0-8)

func (b *Backend) SuggestGasTipCap(baseFee *big.Int) (*big.Int, error) {
	if baseFee == nil {
		// london hardfork not enabled or feemarket not enabled
		return big.NewInt(0), nil
	}

	params, err := b.queryClient.FeeMarket.Params(b.ctx, &feemarkettypes.QueryParamsRequest{})
	if err != nil {
		return nil, err
	}
	// calculate the maximum base fee delta in current block, assuming all block gas limit is consumed
	// ```
	// GasTarget = GasLimit / ElasticityMultiplier
	// Delta = BaseFee * (GasUsed - GasTarget) / GasTarget / Denominator
	// ```
	// The delta is at maximum when `GasUsed` is equal to `GasLimit`, which is:
	// ```
	// MaxDelta = BaseFee * (GasLimit - GasLimit / ElasticityMultiplier) / (GasLimit / ElasticityMultiplier) / Denominator
	//          = BaseFee * (ElasticityMultiplier - 1) / Denominator
	// ```
	maxDelta := baseFee.Int64() * (int64(params.Params.ElasticityMultiplier) - 1) / int64(params.Params.BaseFeeChangeDenominator)
	if maxDelta < 0 {
		// impossible if the parameter validation passed.
		maxDelta = 0
	}
	return big.NewInt(maxDelta), nil
}

// GasPrice returns the current gas price based on Ethermint's gas price oracle.
func (b *Backend) GasPrice() (*hexutil.Big, error) {
	var (
		result *big.Int
		err    error
	)
	head, err := b.CurrentHeader()
	if err != nil {
		return nil, err
	}
	if head.BaseFee != nil {
		result, err = b.SuggestGasTipCap(head.BaseFee)
		if err != nil {
			return nil, err
		}
		result = result.Add(result, head.BaseFee)
	} else {
		result = b.RPCMinGasPrice()
	}

	// return at least GlobalMinGasPrice from FeeMarket module
	minGasPrice, err := b.GlobalMinGasPrice()
	if err != nil {
		return nil, err
	}
	minGasPriceInt := minGasPrice.TruncateInt().BigInt()
	if result.Cmp(minGasPriceInt) < 0 {
		result = minGasPriceInt
	}

	return (*hexutil.Big)(result), nil
}
