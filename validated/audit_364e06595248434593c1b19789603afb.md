### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not validate that the new class hash supplied by a contract corresponds to a previously declared class. This is an explicitly acknowledged missing check (marked with a `TODO`). A malicious contract deployer can call `replace_class` with an arbitrary, undeclared class hash, permanently bricking the contract and freezing all funds held within it.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The function accepts any `class_hash` value from the `ReplaceClassRequest` and writes it directly into `contract_state_changes` without verifying that a compiled class with that hash has been declared. The `TODO` comment at line 898 explicitly acknowledges this missing validation.

When any subsequent call is made to the contract with the replaced (undeclared) class hash, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [2](#0-1) 

`dict_read` on `contract_class_changes` for an undeclared class hash returns `0` (the default for uninitialized dict entries). `find_element` then searches for a compiled class with hash `0`. If no such class exists, `find_element` raises a hard Cairo assertion failure — the OS proof cannot be generated for any block containing a call to the bricked contract.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

A malicious contract deployer can:
1. Deploy a contract that accepts user deposits (e.g., a vault or escrow).
2. After accumulating user funds, call `replace_class` with an arbitrary undeclared felt value (e.g., `1`).
3. The OS commits the new class hash to state without any validation.
4. All future calls to the contract fail at the OS level (hard assertion in `find_element`), making the contract permanently uncallable.
5. All funds held in the contract are permanently frozen with no recovery path.

The contract's storage (and thus its funds) remains in the Merkle state tree but is inaccessible forever, as no valid proof can be generated for any block that executes a call to it.

---

### Likelihood Explanation

**High.** The attack requires only:
- Deploying a contract (permissionless on StarkNet).
- Calling the `replace_class` syscall with an arbitrary felt value.

No privileged access, leaked keys, or external dependencies are required. The `replace_class` syscall is available to any executing contract. The missing check is explicitly documented as a `TODO`, confirming it is a known gap in the current implementation.

---

### Recommendation

Before committing the new class hash to `contract_state_changes`, verify that the supplied `class_hash` corresponds to a declared class. Specifically, perform a lookup in `contract_class_changes` (or the global compiled class facts bundle) to confirm the class hash maps to a valid compiled class hash. If the class is not declared, the syscall should return a failure response rather than updating state.

```cairo
// Proposed fix (pseudocode):
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
if (compiled_class_hash == 0) {
    write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
    return ();
}
```

### Proof of Concept

1. Deploy contract `VaultA` that accepts ETH-equivalent deposits from users.
2. Users deposit funds; `VaultA` holds balances in storage.
3. `VaultA`'s owner calls `replace_class(class_hash=0xdeadbeef)` where `0xdeadbeef` is not a declared class.
4. The OS `execute_replace_class` writes `class_hash=0xdeadbeef` to `contract_state_changes` with no validation. [3](#0-2) 
5. The block is proven successfully (no call to `VaultA` occurs in this block after the replacement).
6. State is committed: `VaultA.class_hash = 0xdeadbeef`.
7. Any future transaction calling `VaultA` causes `execute_entry_point` to call `dict_read` returning `0`, then `find_element` with key `0` fails with a hard assertion. [2](#0-1) 
8. The sequencer cannot include any call to `VaultA` in any future block.
9. All user funds in `VaultA` are permanently frozen.
