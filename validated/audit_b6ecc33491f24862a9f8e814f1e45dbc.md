### Title
Unprivileged Caller Can Invoke `__execute__` on Any Account Contract via `execute_meta_tx_v0` Syscall, Bypassing Signature Validation — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_meta_tx_v0` syscall in the StarkNet OS allows any contract to directly invoke the `__execute__` entry point of any other account contract, setting `caller_address = ORIGIN_ADDRESS (0)` and supplying arbitrary calldata and signature, without the OS ever calling `__validate__` on the target contract. Because standard account contracts perform signature verification only in `__validate__` (not in `__execute__`), an unprivileged attacker can drain any account on the network.

---

### Finding Description

`execute_meta_tx_v0` is exposed as a syscall reachable by any contract execution. Its handler in `syscall_impls.cairo` constructs a synthetic `TxInfo` and `ExecutionContext` for the target contract and calls `contract_call_helper` directly: [1](#0-0) 

The synthetic `TxInfo` is built with:
- `version = 0` — which causes `check_and_increment_nonce` to skip nonce validation entirely
- `nonce = 0`
- `account_contract_address = contract_address` (the victim)
- `signature_start/end` = **attacker-supplied**
- `transaction_hash` = `compute_meta_tx_v0_hash(contract_address, selector, calldata, chain_id)` — no nonce in the hash, enabling replay [2](#0-1) 

The `ExecutionContext` is built with `caller_address = ORIGIN_ADDRESS` (the value `0`): [3](#0-2) 

`contract_call_helper` then executes the target's `__execute__` entry point directly — **no call to `__validate__` is made**: [4](#0-3) 

The syscall is reachable by any contract — it is the final branch in `execute_syscalls`, with no caller restriction: [5](#0-4) 

The normal transaction flow enforces `__validate__` → `__execute__` sequencing in `execute_invoke_function_transaction`: [6](#0-5) 

`execute_meta_tx_v0` entirely skips this sequencing for the target contract.

Standard account contracts guard `__execute__` only with `assert get_caller_address() == 0`. Because `execute_meta_tx_v0` sets `caller_address = ORIGIN_ADDRESS (0)`, this guard passes. Signature verification lives exclusively in `__validate__`, which is never invoked here.

The nonce check is also bypassed: `check_and_increment_nonce` returns immediately for `version == 0`: [7](#0-6) 

---

### Impact Explanation

**Critical — Direct loss of funds.**

An attacker can call `execute_meta_tx_v0` targeting any account contract, supplying arbitrary calldata (e.g., ERC-20 `transfer` to the attacker) and an empty or random signature. The victim's `__execute__` runs without any signature check, executing the attacker's chosen calls. Every account contract on the network that follows the standard `__validate__`/`__execute__` split is vulnerable. Funds can be completely drained in a single transaction.

---

### Likelihood Explanation

**High.** The attack requires only:
1. Deploying a malicious contract (permissionless on StarkNet).
2. Invoking it to emit a `META_TX_V0_SELECTOR` syscall targeting any victim address.

No privileged access, leaked keys, or operator cooperation is needed. The syscall is unconditionally available to all contracts.

---

### Recommendation

The OS must enforce that `__validate__` is called on the target contract before `__execute__` is invoked via `execute_meta_tx_v0`, mirroring the sequencing in `execute_invoke_function_transaction`. Alternatively, restrict `execute_meta_tx_v0` so it can only be called by contracts that have explicitly registered as meta-transaction relayers, or require the OS to verify the provided signature against the meta-tx hash using the target contract's declared verification key before dispatching `__execute__`.

---

### Proof of Concept

```
// Attacker deploys MaliciousRelayer with this logic:
func attack{syscall_ptr: felt*, ...}(victim: felt, transfer_calldata: felt*) {
    // Emit META_TX_V0_SELECTOR syscall:
    //   contract_address = victim
    //   selector         = EXECUTE_ENTRY_POINT_SELECTOR
    //   calldata         = [transfer(attacker, victim_balance)]
    //   signature        = [] (empty — never checked)
    //
    // OS executes victim.__execute__(calldata) with caller_address=0.
    // Victim's __execute__ sees caller==0, passes its guard, runs the transfer.
    // Attacker receives victim's entire token balance.
}
```

1. Attacker deploys `MaliciousRelayer`.
2. Attacker calls `MaliciousRelayer.attack(victim=alice, ...)`.
3. Inside execution, `execute_meta_tx_v0` is triggered with `contract_address=alice`, `calldata=[transfer(attacker, alice_balance)]`, `signature=[]`.
4. OS sets `caller_address=0`, skips `__validate__`, calls `alice.__execute__`.
5. Alice's `__execute__` passes the `caller==0` guard and executes the transfer.
6. Alice's funds are sent to the attacker. The attack is also **replayable** because the meta-tx hash contains no nonce.
