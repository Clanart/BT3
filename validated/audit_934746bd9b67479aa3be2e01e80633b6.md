### Title
Deploy Syscall Unconditionally Reports Success Regardless of Constructor Failure, Enabling Permanent Fund Freeze - (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_deploy` syscall handler in the StarkNet OS always writes `failure_flag=0` in its response header, regardless of whether the deployed contract's constructor actually succeeded or reverted. A calling contract that relies on this response to determine whether a deploy succeeded will always proceed as if the deploy was successful — even when the constructor failed and the contract was never actually registered. If the calling contract subsequently transfers funds to the "deployed" address, those funds are permanently frozen at an address with no contract code.

---

### Finding Description

In `execute_deploy` (`execution/syscall_impls.cairo`, lines 533–539), after calling `deploy_contract`, the OS writes the syscall response with a hardcoded `failure_flag=0`:

```cairo
// Write the response header.
// TODO(Yoni, 1/1/2026): support failures.
assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=0);
``` [1](#0-0) 

The `deploy_contract` function returns only `(retdata_size, retdata)` — it does not surface an `is_reverted` flag to the caller:

```cairo
with remaining_gas {
    let (retdata_size, retdata) = deploy_contract(
        block_context=block_context, constructor_execution_context=constructor_execution_context
    );
}
``` [2](#0-1) 

This is in direct contrast to `contract_call_helper`, which correctly propagates the `is_reverted` flag:

```cairo
assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=is_reverted);
``` [3](#0-2) 

When a constructor reverts, the revert mechanism (via `revert_log` and `handle_revert`) undoes all state changes — including the `CHANGE_CLASS_ENTRY` that registered the class hash at the new contract address. The contract address is left with no class hash (uninitialized, `class_hash = 0`). However, the calling contract receives `failure_flag=0` and has no way to detect the failure.

The revert log mechanism that undoes class hash assignments is defined in `revert.cairo`:

```cairo
if (selector == CHANGE_CLASS_ENTRY) {
    // Change class entry.
    let class_hash = revert_log_end[0].value;
    return revert_contract_changes();
}
``` [4](#0-3) 

So after a constructor revert: the address has no contract, but the deploying contract is told the deploy succeeded.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

A contract that deploys a sub-contract (e.g., a vault, escrow, or pool) via the `deploy` syscall and then transfers tokens to the deployed address will proceed even when the constructor failed. The tokens are transferred to an address with `class_hash = 0` — no contract exists there. There is no entry point to withdraw those tokens. The funds are permanently frozen.

This matches the external report's root cause: a partial failure (constructor revert) is not surfaced to the caller, causing the caller to proceed with incorrect assumptions and lock funds irreversibly.

---

### Likelihood Explanation

**Medium-High.** The `deploy` syscall is a standard, unprivileged operation available to any Cairo contract. DeFi protocols routinely deploy sub-contracts (vaults, pools, escrows) and immediately interact with them. A constructor can fail due to:

- Out-of-gas during constructor execution (attacker can influence gas by crafting calldata)
- Any assertion or `require`-equivalent in the constructor
- Recursive deploy depth limits

An unprivileged transaction sender can trigger this by submitting a transaction that calls a contract which deploys a sub-contract with a constructor that reverts under attacker-controlled conditions, then sends funds to the resulting address.

---

### Recommendation

Propagate the constructor's revert status from `deploy_contract` back to `execute_deploy`, and set `failure_flag` accordingly — mirroring the pattern already used in `contract_call_helper`. The `deploy_contract` function should be updated to return an `is_reverted` flag, and the response header should be:

```cairo
assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=is_reverted);
```

This is already acknowledged as a known gap in the codebase via the TODO comment at line 538.

---

### Proof of Concept

1. Attacker (or legitimate user) deploys Contract A, which contains logic to:
   - Call the `deploy` syscall targeting Contract B (whose constructor reverts under a specific condition, e.g., when `calldata[0] == 1`)
   - After the deploy call, transfer 1000 tokens to the address returned by the deploy syscall
2. Attacker submits an invoke transaction calling Contract A with the triggering calldata
3. The OS executes `execute_deploy`:
   - `deploy_contract` runs Contract B's constructor → constructor reverts
   - Revert log undoes the `CHANGE_CLASS_ENTRY` → Contract B's address has `class_hash = 0`
   - OS writes `ResponseHeader(gas=..., failure_flag=0)` ← **bug**
4. Contract A reads `failure_flag=0`, assumes success, transfers 1000 tokens to Contract B's address
5. The ERC20 transfer succeeds (it only updates storage in the token contract)
6. Contract B's address has no code — the 1000 tokens are permanently frozen [5](#0-4)

func execute_deploy{
    range_check_ptr,
    syscall_ptr: felt*,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    revert_log: RevertLogEntry*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, caller_execution_context: ExecutionContext*) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, DeployRequest*);
    local constructor_calldata_start: felt* = request.constructor_calldata_start;
    local constructor_calldata_size = request.constructor_calldata_end - constructor_calldata_start;

    let specific_base_gas_cost = DEPLOY_GAS_COST + DEPLOY_CALLDATA_FACTOR_GAS_COST *
        constructor_calldata_size;
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=specific_base_gas_cost, request_struct_size=DeployRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    local caller_execution_info: ExecutionInfo* = caller_execution_context.execution_info;
    local caller_address = caller_execution_info.contract_address;

    // Verify deploy_from_zero is either 0 (FALSE) or 1 (TRUE).
    tempvar deploy_from_zero = request.deploy_from_zero;
    assert deploy_from_zero * (deploy_from_zero - 1) = 0;
    // Set deployer_address to 0 if request.deploy_from_zero is TRUE.
    let deployer_address = (1 - deploy_from_zero) * caller_address;

    let selectable_builtins = &builtin_ptrs.selectable;
    let hash_ptr = selectable_builtins.pedersen;
    with hash_ptr {
        let (contract_address) = get_contract_address(
            salt=request.contract_address_salt,
            class_hash=request.class_hash,
            constructor_calldata_size=constructor_calldata_size,
            constructor_calldata=constructor_calldata_start,
            deployer_address=deployer_address,
        );
    }
    tempvar builtin_ptrs = new BuiltinPointers(
        selectable=SelectableBuiltins(
            pedersen=hash_ptr,
            range_check=selectable_builtins.range_check,
            ecdsa=selectable_builtins.ecdsa,
            bitwise=selectable_builtins.bitwise,
            ec_op=selectable_builtins.ec_op,
            poseidon=selectable_builtins.poseidon,
            segment_arena=selectable_builtins.segment_arena,
            range_check96=selectable_builtins.range_check96,
            add_mod=selectable_builtins.add_mod,
            mul_mod=selectable_builtins.mul_mod,
        ),
        non_selectable=builtin_ptrs.non_selectable,
    );

    tempvar constructor_execution_context = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_CONSTRUCTOR,
        class_hash=request.class_hash,
        calldata_size=constructor_calldata_size,
        calldata=constructor_calldata_start,
        execution_info=new ExecutionInfo(
            block_info=caller_execution_info.block_info,
            tx_info=caller_execution_info.tx_info,
            caller_address=caller_address,
            contract_address=contract_address,
            selector=CONSTRUCTOR_ENTRY_POINT_SELECTOR,
        ),
        deprecated_tx_info=caller_execution_context.deprecated_tx_info,
    );

    with remaining_gas {
        let (retdata_size, retdata) = deploy_contract(
            block_context=block_context, constructor_execution_context=constructor_execution_context
        );
    }

    // TODO(Yoni, 1/1/2026): consider sharing code with call_contract_helper.
    let response_header = cast(syscall_ptr, ResponseHeader*);
    let syscall_ptr = syscall_ptr + ResponseHeader.SIZE;

    // Write the response header.
    // TODO(Yoni, 1/1/2026): support failures.
    assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=0);

    let response = cast(syscall_ptr, DeployResponse*);
    // Advance syscall pointer to the next syscall.
    let syscall_ptr = syscall_ptr + DeployResponse.SIZE;

    %{ CheckNewDeployResponse %}

    // Write the response.
    relocate_segment(src_ptr=response.constructor_retdata_start, dest_ptr=retdata);
    assert [response] = DeployResponse(
        contract_address=contract_address,
        constructor_retdata_start=retdata,
        constructor_retdata_end=retdata + retdata_size,
    );

    return ();
}
