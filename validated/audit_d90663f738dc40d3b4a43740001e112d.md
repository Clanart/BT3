### Title
L2ŌåÆL1 Messages Written to OS Output Unconditionally Even When Sub-Call Reverts ŌĆö (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

`execute_send_message_to_l1` writes L2ŌåÆL1 messages to `outputs.messages_to_l1` unconditionally (after a gas check). The revert mechanism (`handle_revert` in `revert.cairo`) only rolls back storage writes and class-hash changes; it never resets the `outputs` pointer. Consequently, when a sub-call executed via `call_contract` reverts, any L2ŌåÆL1 messages it emitted remain in the OS output and are later serialised into the proof and processed by the L1 StarkNet core contract.

---

### Finding Description

**Root cause ŌĆö `execute_send_message_to_l1`** unconditionally advances `outputs.messages_to_l1` after a successful gas deduction, with no awareness of whether the calling context will ultimately revert: [1](#0-0) 

```cairo
func execute_send_message_to_l1{..., outputs: OsCarriedOutputs*}(contract_address: felt) {
    ...
    if (success == FALSE) { return (); }          // only gas guard

    assert [outputs.messages_to_l1] = MessageToL1Header(...);
    memcpy(...);
    let (outputs) = os_carried_outputs_new(
        messages_to_l1=outputs.messages_to_l1 + MessageToL1Header.SIZE + payload_size, ...
    );
    return ();
}
```

**Root cause ŌĆö `contract_call_helper`** receives `is_reverted` from `select_execute_entry_point_func` but never restores the `outputs` pointer to its pre-call value when the callee reverts: [2](#0-1) 

```cairo

    with remaining_gas {
        let (is_reverted, retdata_size, retdata, _) = select_execute_entry_point_func(...);
    }
    if (is_reverted != FALSE) {
        assert retdata[retdata_size] = ERROR_ENTRY_POINT_FAILED;
        // ŌåÉ outputs is NOT reset here; messages written by the callee remain
    }
    ...
    assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=is_reverted);
    ...
}
```

**Root cause ŌĆö `handle_revert`** only processes storage writes and class-hash changes; `outputs` is not an implicit argument and is never touched during revert: [3](#0-2)

---

### Impact Explanation

**Critical ŌĆö Direct loss of funds.**

L2ŌåÆL1 messages are the mechanism by which L2 contracts instruct the L1 StarkNet core contract to release bridged assets (e.g., ETH, ERC-20 tokens). The L1 contract trusts the OS proof output unconditionally. If a reverted sub-call's withdrawal message appears in the proof output, the L1 contract will honour it and transfer funds to the attacker's L1 address, even though the corresponding L2 state change was rolled back. The attacker can repeat this to drain the bridge.

---

### Likelihood Explanation

**High.** The attack requires only an unprivileged invoke transaction. Any user can:

1. Deploy a contract that calls `send_message_to_l1` (e.g., "withdraw X tokens to my L1 address") and then deliberately reverts (e.g., `assert 0 = 1`).
2. Call that contract from their account's `__execute__` entry point via the `call_contract` syscall.
3. `__execute__` receives the `failure_flag=1` response and returns successfully (it is executed via `non_reverting_select_execute_entry_point_func`, which only panics if `__execute__` itself reverts, not its callees).
4. The L2ŌåÆL1 message is in the OS output; the L1 bridge releases funds.

No privileged access, leaked keys, or external dependency is required.

---

### Recommendation

Before executing a sub-call in `contract_call_helper`, save the current `outputs` pointer. If `is_reverted != FALSE` after the call returns, restore `outputs` to the saved value, discarding any L2ŌåÆL1 messages the reverted callee wrote. Concretely:

```cairo
// Save outputs before sub-call
let outputs_before = outputs;

with remaining_gas {
    let (is_reverted, ...) = select_execute_entry_point_func(...);
}

if (is_reverted != FALSE) {
    // Discard messages written by the reverted callee
    let outputs = outputs_before;
    ...
}
```

The same fix must be applied symmetrically in `execute_library_call` and any other helper that wraps `select_execute_entry_point_func` and propagates `outputs`.

---

### Proof of Concept

1. Attacker deploys **ContractB** (Cairo 1):
   ```rust
   #[external]
   fn exploit(to: felt252) {
       starknet::send_message_to_l1_syscall(to, array![1000_u128.into()].span()).unwrap();
       panic_with_felt252('revert');   // deliberately revert
   }
   ```

2. Attacker's account `__execute__` calls `ContractB::exploit(attacker_l1_address)` via `call_contract`. The call returns `failure_flag=1`; `__execute__` ignores it and returns `VALIDATED`.

3. The OS executes the block. `execute_send_message_to_l1` writes the withdrawal message to `outputs.messages_to_l1` before the revert. `contract_call_helper` detects `is_reverted=1` and rolls back storage via the revert log, but `outputs` is not reset.

4. `serialize_messages` serialises the message into the proof output: [6](#0-5) 

5. The L1 StarkNet core contract verifies the proof and processes the withdrawal message, transferring funds to the attacker's L1 address ŌĆö despite the L2 call having reverted.

func execute_send_message_to_l1{range_check_ptr, syscall_ptr: felt*, outputs: OsCarriedOutputs*}(
    contract_address: felt
) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, SendMessageToL1Request*);
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=SEND_MESSAGE_TO_L1_GAS_COST, request_struct_size=SendMessageToL1Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    tempvar payload_start = request.payload_start;
    tempvar payload_size = request.payload_end - payload_start;

    assert [outputs.messages_to_l1] = MessageToL1Header(
        from_address=contract_address, to_address=request.to_address, payload_size=payload_size
    );
    memcpy(
        dst=outputs.messages_to_l1 + MessageToL1Header.SIZE, src=payload_start, len=payload_size
    );
    let (outputs) = os_carried_outputs_new(
        messages_to_l1=outputs.messages_to_l1 + MessageToL1Header.SIZE + payload_size,
        messages_to_l2=outputs.messages_to_l2,
    );

    return ();
}

func handle_revert{contract_state_changes: DictAccess*}(
    contract_address, revert_log_end: RevertLogEntry*
) {
    alloc_locals;

    local state_entry: StateEntry*;

    %{ PrepareStateEntryForRevert %}

    let class_hash = state_entry.class_hash;
    let storage_ptr = state_entry.storage_ptr;
    with class_hash, storage_ptr, revert_log_end {
        revert_contract_changes();
    }

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(
            new StateEntry(class_hash=class_hash, storage_ptr=storage_ptr, nonce=state_entry.nonce),
            felt,
        ),
    );

    // `revert_contract_changes()` stops where
    // `revert_log_end[0].selector == CHANGE_CONTRACT_ENTRY`.
    tempvar next_contract_address = revert_log_end[0].value;

    if (next_contract_address == CONTRACT_ADDRESS_UPPER_BOUND) {
        // Finish backward processing: this entry marks the beginning of the revert log.
        return ();
    }

    return handle_revert(contract_address=next_contract_address, revert_log_end=revert_log_end);
}
