### Title
Missing `recipient` Empty-String Validation in `initTransfer()` Enables Permanent Fund Lock — (File: `evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.sol::initTransfer()` and `initTransfer1155()` accept an arbitrary `string calldata recipient` without validating it is non-empty. A user who submits an empty (or unparseable) recipient string will have their tokens irreversibly burned or locked on the EVM side while the corresponding NEAR-side finalization permanently fails, with no on-chain refund path.

---

### Finding Description

`initTransfer()` performs only one input check before consuming user funds:

```solidity
if (fee >= amount) {
    revert InvalidFee();
}
```

There is no guard on `recipient`:

```solidity
function initTransfer(
    address tokenAddress,
    uint128 amount,
    uint128 fee,
    uint128 nativeFee,
    string calldata recipient,   // ← never validated
    string calldata message
) external payable whenNotPaused(PAUSED_INIT_TRANSFER) {
    currentOriginNonce += 1;
    if (fee >= amount) { revert InvalidFee(); }
    // tokens burned/locked here …
    emit BridgeTypes.InitTransfer(…, recipient, …);
}
``` [1](#0-0) 

The same omission exists in `initTransfer1155()`: [2](#0-1) 

After the call succeeds, the NEAR bridge's `fin_transfer_callback` receives the proof and attempts to parse `init_transfer.recipient` as an `OmniAddress`. An empty string cannot be parsed as any `OmniAddress` variant, so the NEAR-side callback panics and the finalization is permanently blocked. Because the EVM side already burned (bridge tokens) or locked (native tokens) the funds, there is no on-chain mechanism to refund them. [3](#0-2) 

The Solana bridge's own `SECURITY.md` explicitly acknowledges the identical class of issue for the Solana path ("No validation of `recipient` string in `InitTransferPayload` — An invalid recipient causes the transfer to fail on the NEAR side after tokens are locked/burned on Solana. Manual intervention would be needed."), confirming the cross-chain impact model. The EVM path carries the same root cause but has no such acknowledgment or mitigation. [4](#0-3) 

By contrast, the StarkNet `init_transfer` already validates `amount > 0` but similarly omits a `recipient` length check: [5](#0-4) 

---

### Impact Explanation

**Critical — Permanent, irrecoverable lock of user funds in the bridge flow.**

- For bridge tokens (`isBridgeToken[tokenAddress] == true`): tokens are **burned** on EVM. There is no mint-back path if NEAR finalization fails.
- For native/ERC-20 tokens: tokens are **locked** in the bridge contract with no on-chain refund instruction.

In both cases the `InitTransfer` event is emitted and the nonce is consumed, but the NEAR-side `fin_transfer_callback` will panic on the unparseable recipient, leaving the transfer in a permanently unresolvable state on-chain.

---

### Likelihood Explanation

**Medium.** `initTransfer` is a public, permissionless function callable by any token holder. A user who accidentally passes `""` as the recipient (e.g., from a front-end bug, a script error, or a direct contract call) will permanently lose their funds. A malicious actor could also deliberately trigger this on behalf of a victim if they can influence the call parameters (e.g., via a wrapper contract).

---

### Recommendation

Add an explicit non-empty check for `recipient` in both `initTransfer` and `initTransfer1155` before any state changes or token movements:

```solidity
function initTransfer(
    address tokenAddress,
    uint128 amount,
    uint128 fee,
    uint128 nativeFee,
    string calldata recipient,
    string calldata message
) external payable whenNotPaused(PAUSED_INIT_TRANSFER) {
+   require(bytes(recipient).length > 0, "ERR_EMPTY_RECIPIENT");
    currentOriginNonce += 1;
    if (fee >= amount) { revert InvalidFee(); }
    …
}
```

Apply the same guard to `initTransfer1155`. Optionally add a minimum-length sanity check (e.g., `> 5` bytes) to reject obviously malformed OmniAddress strings before funds are consumed.

---

### Proof of Concept

1. Deploy or interact with the live `OmniBridge` contract on any supported EVM chain.
2. Obtain bridge tokens (e.g., via a prior `finTransfer`).
3. Call:
   ```solidity
   OmniBridge.initTransfer(
       bridgeTokenAddress,
       1_000_000,   // amount
       0,           // fee
       0,           // nativeFee
       "",          // recipient — empty string, no revert
       ""
   );
   ```
4. The transaction succeeds: tokens are burned, `currentOriginNonce` is incremented, `InitTransfer` is emitted.
5. A relayer submits the proof to the NEAR bridge's `fin_transfer`. The callback attempts `init_transfer.recipient.get_chain()` on an empty-string `OmniAddress`, which panics with `BridgeError::InvalidProofMessage` or an OmniAddress parse error.
6. No further finalization is possible; the burned tokens are unrecoverable on-chain. [6](#0-5) [7](#0-6)

        fn init_transfer(
            ref self: ContractState,
            token_address: ContractAddress,
            amount: u128,
            fee: u128,
            native_fee: u128,
            recipient: ByteArray,
            message: ByteArray,
        ) {
            assert(!_is_paused(@self, PAUSE_INIT_TRANSFER), 'ERR_INIT_TRANSFER_PAUSED');

            assert(amount > 0, 'ERR_ZERO_AMOUNT');
            assert(fee < amount, 'ERR_INVALID_FEE');

            let origin_nonce = self.current_origin_nonce.read() + 1;
            self.current_origin_nonce.write(origin_nonce);

            let caller = get_caller_address();

            if self.is_bridge_token(token_address) {
                IBridgeTokenDispatcher { contract_address: token_address }
                    .burn(caller, amount.into());
            } else {
                let success = IERC20Dispatcher { contract_address: token_address }
                    .transfer_from(caller, get_contract_address(), amount.into());
                assert(success, 'ERR_TRANSFER_FROM_FAILED');
            }

            if native_fee > 0 {
                let native_token = self.strk_token_address.read();
                let success = IERC20Dispatcher { contract_address: native_token }
                    .transfer_from(caller, get_contract_address(), native_fee.into());
                assert(success, 'ERR_FEE_TRANSFER_FAILED');
            }

            self
                .emit(
                    Event::InitTransfer(
                        InitTransfer {
                            sender: caller,
                            token_address,
                            origin_nonce,
                            amount,
                            fee,
                            native_fee,
                            recipient,
                            message,
                        },
                    ),
                )
        }
