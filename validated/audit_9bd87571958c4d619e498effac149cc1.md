### Title
`ProofOfInclusion::valid()` Self-Referential Root Check Allows Forged DataLayer Inclusion Proofs — (`File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` contains a tautological final check: it compares `existing_hash` against `self.root_hash()`, but `root_hash()` is derived entirely from the proof's own `combined_hash` field. The check is always true after the loop, meaning any internally self-consistent proof — regardless of whether it corresponds to the actual DataLayer Merkle tree root — passes validation. An unprivileged attacker can forge a proof claiming any key is included in any tree.

### Finding Description

`ProofOfInclusion::valid()` is the sole public API for verifying DataLayer inclusion proofs:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()   // ← always true
}
``` [1](#0-0) 

The final line compares `existing_hash` against `self.root_hash()`. But `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← taken from the proof itself
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

After the loop body executes for the last layer:
- `existing_hash` is set to `calculated_hash`
- The loop guard already verified `calculated_hash == layer.combined_hash`
- Therefore `existing_hash == last.combined_hash`
- `self.root_hash()` also returns `last.combined_hash`
- The final comparison is `last.combined_hash == last.combined_hash` — **always `true`**

For the degenerate case of an empty `layers` vec, `existing_hash` stays as `self.node_hash` and `root_hash()` returns `self.node_hash`, so the check is again trivially true for any `node_hash`.

The function never compares against an **external, trusted root hash**. It only verifies that the proof's own chain of hashes is internally self-consistent.

The Python binding exposes this directly:

```python
def valid(self) -> bool: ...
def root_hash(self) -> bytes32: ...
``` [3](#0-2) [4](#0-3) 

The fuzz target and all tests only call `valid()` on proofs generated from the real tree, so the forgery path is never exercised: [5](#0-4) 

### Impact Explanation

Any party that receives a `ProofOfInclusion` from an untrusted source and calls `proof.valid()` — the only provided verification method — will accept a completely fabricated proof. The attacker can:

1. Claim any arbitrary key/value pair is present in a DataLayer store whose root they do not control.
2. Claim any key is absent (by constructing a forged exclusion proof using the same pattern).
3. Prove invalid state against any tree root, because `valid()` never checks against the real root.

This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.**

### Likelihood Explanation

The `valid()` method is the only public verification API. Its name strongly implies sufficiency. No documentation warns callers to additionally compare `proof.root_hash()` against a known root. Any DataLayer client that follows the natural API usage pattern is vulnerable. The attacker needs only to craft a `ProofOfInclusion` struct with a self-consistent chain of hashes — trivially achievable with no cryptographic secrets.

### Recommendation

`valid()` must accept an external trusted root hash and verify against it:

```rust
pub fn valid_against_root(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }

    &existing_hash == expected_root   // compare against caller-supplied root
}
```

The no-argument `valid()` should either be removed or clearly documented as an internal-consistency-only check that does **not** prove inclusion against any specific tree.

### Proof of Concept

Forge a single-leaf proof claiming key hash `0xAA…AA` is included in a tree whose real root is `0xBB…BB`:

```rust
use chia_datalayer::{Hash, Side, ProofOfInclusion, ProofOfInclusionLayer};

let fake_node_hash: Hash = [0xAA; 32];
// Empty layers: root_hash() returns node_hash, final check is node_hash == node_hash
let forged = ProofOfInclusion {
    node_hash: fake_node_hash,
    layers: vec![],
};
assert!(forged.valid());   // passes — no external root checked
// root_hash() returns 0xAA…AA, not the real tree root 0xBB…BB
// but valid() never compared against 0xBB…BB

// Multi-layer forgery: attacker picks any other_hash and computes combined_hash themselves
let other_hash: Hash = [0xCC; 32];
let combined = chia_datalayer::calculate_internal_hash(&fake_node_hash, Side::Left, &other_hash);
let forged_multi = ProofOfInclusion {
    node_hash: fake_node_hash,
    layers: vec![ProofOfInclusionLayer {
        other_hash_side: Side::Left,
        other_hash,
        combined_hash: combined,
    }],
};
assert!(forged_multi.valid());   // also passes
