### Title
Deprecated Account Contract `__validate__` Return Value Never Checked, Bypassing Signature Verification - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/execute_transaction_utils.cairo`)

---

### Summary

The StarkNet OS `run_validate()` function, and every other validate call site, gates the `VALIDATED` return-value assertion on `is_deprecated == 0`. When the account contract is a deprecated (CairoZero) contract, `is_deprecated` is `1` and the assertion is **never executed**. Any deprecated account contract whose `__validate__` entry point does not revert — including one that ignores the signature entirely, or one that is matched via the `DEFAULT_ENTRY_POINT_SELECTOR` fallback — passes OS-level signature verification unconditionally.

---

### Finding Description

In `run_validate()`:

```cairo
let (retdata_size, retdata, is_deprecated) = non_reverting_select_execute_entry_point_func(
    block_context=block_context, execution_context=validate_execution_context
);
if (is_deprecated == 0) {
    %{ CheckRetdataForDebug %}
    assert retdata_size = 1;
    assert retdata[0] = VALIDATED;
}
``` [1](#0-0) 

When `is_deprecated == 1` (a CairoZero account), the branch is skipped. The OS never asserts `retdata[0] = VALIDATED`. Whatever the deprecated `__validate__` function returns — including nothing meaningful — is silently accepted.

The same unchecked pattern appears in `execute_deploy_account_transaction()`:

```cairo
if (is_deprecated == 0) {
    assert retdata_size = 1;
    assert retdata[0] = VALIDATED;
}
``` [2](#0-1) 

And in `execute_declare_transaction()`:

```cairo
if (is_deprecated == 0) {
    assert retdata_size = 1;
    assert retdata[0] = VALIDATED;
}
``` [3](#0-2) 

The `VALIDATED` constant is `'VALID'` and is the sole mechanism by which a Cairo 1 account signals successful signature verification: [4](#0-3) 

For deprecated contracts, `select_execute_entry_point_func` routes to `deprecated_execute_entry_point`, which returns `is_deprecated=1`: [5](#0-4) 

Additionally, `get_entry_point_offset` in the deprecated path allows a `DEFAULT_ENTRY_POINT_SELECTOR` (value `0x0`) to match **any** selector, including `VALIDATE_ENTRY_POINT_SELECTOR`:

```cairo
// If the selector was not found, check if we have a default entry point.
if (n_entry_points != 0 and entry_points[0].selector == DEFAULT_ENTRY_POINT_SELECTOR) {
    return (success=1, entry_point_offset=entry_points[0].offset);
}
``` [6](#0-5) [7](#0-6) 

So a deprecated account contract with a default entry point that simply returns without reverting will have its `__validate__` call dispatched to that default handler, which returns `is_reverted=0` with arbitrary retdata, and the OS accepts it.

---

### Impact Explanation

An attacker deploys a deprecated (CairoZero) account contract whose `__validate__` function (or default entry point) always returns without reverting and without checking the transaction signature. The OS accepts every invoke, declare, or deploy-account transaction from this account as validly signed. The attacker can drain any funds held by the account or execute arbitrary state changes on behalf of it without possessing the private key.

**Impact: Critical — Direct loss of funds.**

---

### Likelihood Explanation

- Any unprivileged user can deploy a deprecated account contract via a `deploy_account` transaction.
- Writing a CairoZero contract whose `__validate__` is a no-op (or registering a default entry point) is trivial.
- No privileged access, leaked key, or external dependency is required.
- The attacker controls the class hash they declare and deploy, making this fully self-contained.

---

### Recommendation

Remove the `is_deprecated` gate on the `VALIDATED` assertion. For deprecated contracts, the OS should still verify that the `__validate__` entry point returns the expected sentinel value. If the CairoZero ABI does not use `VALIDATED`, a separate but equally strict check (e.g., `retdata_size = 0` for a clean return, or a dedicated deprecated-validate sentinel) must be enforced. The current unconditional skip is equivalent to having no signature check at all for deprecated accounts.

---

### Proof of Concept

1. Declare a CairoZero class whose external entry point table contains a single entry with `selector = DEFAULT_ENTRY_POINT_SELECTOR (0x0)` pointing to a function that immediately returns `(retdata_size=0, retdata=[])`.
2. Submit a `deploy_account` transaction deploying this class. The OS calls `__validate_deploy__` → dispatched to the default entry point → returns without reverting → `is_deprecated=1` → `VALIDATED` check skipped → transaction accepted.
3. Now submit an `invoke` transaction from this account with a completely invalid (empty) signature. The OS calls `__validate__` → dispatched to the default entry point → returns without reverting → `is_deprecated=1` → `VALIDATED` check skipped → transaction accepted and executed.
4. The attacker has executed an arbitrary transaction from an account they do not own the key to, achieving direct loss of funds. [8](#0-7)

func run_validate{
    range_check_ptr,
    remaining_gas: felt,
    builtin_ptrs: BuiltinPointers*,

    outputs: OsCarriedOutputs*,

    alloc_locals;
    local tx_execution_info: ExecutionInfo* = tx_execution_context.execution_info;

    // Do not run "__validate__" for version 0.
    if (tx_execution_info.tx_info.version == 0) {
        return ();
    }

    // "__validate__" is expected to get the same calldata as "__execute__".
    local validate_execution_context: ExecutionContext* = new ExecutionContext(

        class_hash=tx_execution_context.class_hash,
        calldata_size=tx_execution_context.calldata_size,
        calldata=tx_execution_context.calldata,
        execution_info=new ExecutionInfo(
            block_info=block_context.block_info_for_validate,
            tx_info=tx_execution_info.tx_info,
            caller_address=tx_execution_info.caller_address,
            contract_address=tx_execution_info.contract_address,
            selector=VALIDATE_ENTRY_POINT_SELECTOR,
        ),
        deprecated_tx_info=tx_execution_context.deprecated_tx_info,
    );

    // The __validate__ function should not revert.
    let (retdata_size, retdata, is_deprecated) = non_reverting_select_execute_entry_point_func(

    );

        %{ CheckRetdataForDebug %}
        assert retdata_size = 1;
        assert retdata[0] = VALIDATED;
    }
