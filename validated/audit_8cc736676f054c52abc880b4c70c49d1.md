### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not validate that the new class hash provided by a contract is actually declared in the contract class tree. This allows any contract to irreversibly set its own class hash to an arbitrary undeclared value, permanently breaking the contract and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class`, the OS reads the requested new class hash from the syscall request and immediately writes it into `contract_state_changes` without checking whether that class hash exists in `contract_class_changes`. The missing check is explicitly acknowledged by a TODO comment in the code: [1](#0-0) 

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

After this update, any future call to the contract reaches `execute_entry_point`, which performs: [2](#0-1) 

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
```

If `class_hash` was never declared, `dict_read` returns `0` (the uninitialized default). `find_element` with key `0` then fails to locate any compiled class (since `0` is `UNINITIALIZED_CLASS_HASH`), causing the OS to panic and the block to be invalid. The contract is permanently uncallable. [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

A malicious actor deploys a contract that accepts deposits from users. After accumulating funds, the attacker calls `replace_class` with an arbitrary undeclared class hash. The OS updates the contract's class hash without any validation. All subsequent calls to the contract cause the OS to panic at `find_element`, making any block containing such a call invalid. The contract becomes permanently uncallable and all deposited funds are irreversibly frozen.

This is directly analogous to the external report: just as the bonding curve sale becomes permanently stuck when `_createPair` fails after the attacker manipulates state during the ETH refund callback, here the contract becomes permanently stuck after `replace_class` writes an invalid class hash that the OS cannot resolve.

---

### Likelihood Explanation

**High.** The attack requires no privileged access. Any unprivileged user can:
1. Deploy a contract (class hash H, legitimately declared).
2. Collect deposits from other users into the contract.
3. Call a function that invokes `replace_class(H')` where `H'` is any value not present in `contract_class_changes`.

The OS performs no validation on `H'`. The state update is committed. The contract is permanently broken. The attack is deterministic and requires no race condition or timing dependency.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that it is present in `contract_class_changes` (i.e., that it has been declared). The check should assert that `dict_read{dict_ptr=contract_class_changes}(key=class_hash)` returns a non-zero compiled class hash. This is exactly what the existing TODO comment calls for: [4](#0-3) 

```diff
  let class_hash = request.class_hash;

- // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
+ let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
+ if (compiled_class_hash == 0) {
+     write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
+     return ();
+ }
```

### Proof of Concept

**Attacker contract (pseudocode Sierra/Cairo):**
```
@external
fn collect_and_freeze() {
    // Step 1: Accept deposits from users (omitted for brevity).
    // Step 2: Call replace_class with an arbitrary undeclared hash.
    replace_class(class_hash: 0xdeadbeef);  // Not declared in contract_class_changes.
}
```

**Attack sequence:**
1. Attacker deploys the contract with a legitimate declared class hash `H`.
2. Users deposit funds into the contract (e.g., via a `deposit()` entry point).
3. Attacker calls `collect_and_freeze()`.
4. The OS executes `execute_replace_class` — no validation is performed on `0xdeadbeef`.
5. `contract_state_changes[attacker_contract].class_hash` is set to `0xdeadbeef`.
6. Any subsequent transaction calling the attacker's contract reaches `execute_entry_point`, which calls `dict_read{dict_ptr=contract_class_changes}(key=0xdeadbeef)` → returns `0`.
7. `find_element(..., key=0)` fails — OS panics, block is invalid.
8. The contract is permanently uncallable. All deposited funds are frozen. [5](#0-4) [6](#0-5)

        // Not enough gas to execute the syscall.
