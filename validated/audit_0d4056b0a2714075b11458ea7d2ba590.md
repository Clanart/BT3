### Title
`ProofOfInclusion::valid()` Tautological Root Check Enables Forged DataLayer Inclusion Proofs - (File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs)

### Summary
`ProofOfInclusion::valid()` contains a final comparison that is always `true` regardless of input. The method never validates the proof's computed root against any externally-supplied trusted root. Any caller that relies on `valid()` as its sole gate for accepting a DataLayer inclusion proof can be deceived by a forged `ProofOfInclusion` constructed from untrusted bytes.

### Finding Description
`valid()` iterates over each `ProofOfInclusionLayer`, computing `calculated_hash` from the running hash and the sibling hash, and returns `false` if `calculated_hash != layer.combined_hash`. After the loop, `existing_hash` holds the last `calculated_hash`. The final statement is:

```rust
existing_hash == self.root_hash()
```

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← always equals existing_hash after the loop
    } else {
        self.node_hash       // ← always equals existing_hash when loop never ran
    }
}
``` [1](#0-0) 

**When layers are present:** the loop only continues when `calculated_hash == layer.combined_hash`, so after the loop `existing_hash` is exactly `last.combined_hash`. `root_hash()` also returns `last.combined_hash`. The comparison is `last.combined_hash == last.combined_hash` — always `true`.

**When no layers are present:** `existing_hash` is `self.node_hash` (loop never executes) and `root_hash()` also returns `self.node_hash`. Again always `true`.

In both cases the final check is a tautology. The function never compares the computed root against any externally-supplied trusted root. An attacker can construct a `ProofOfInclusion` with an arbitrary `node_hash` (e.g., the hash of a key-value pair they wish to falsely prove is included) and any set of internally-consistent layers (where each `combined_hash` is correctly derived from the previous hash and a chosen sibling hash). `valid()` returns `true` for this forged proof even though the proof's root hash does not match the actual DataLayer tree root.

The struct is fully exposed via Python bindings with `from_bytes`, `from_py_object`, and a public constructor, giving untrusted callers a direct path to supply forged proofs: [2](#0-1) 

The `valid()` method is the only validation method on the struct, and its name strongly implies it performs a complete proof check.

### Impact Explanation
Any verifier that calls `proof.valid()` as its sole gate for accepting a DataLayer inclusion proof can be convinced that an arbitrary key-value pair is present in a store whose actual root does not contain that pair. This lets untrusted input prove invalid state in the DataLayer Merkle tree, matching the allowed High impact:

> High. DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.

### Likelihood Explanation
The `ProofOfInclusion` struct is serializable/deserializable from bytes and exposed to Python via bindings. The `valid()` method is the only validation method on the struct, and its name strongly implies a complete proof check. A Python consumer receiving a proof from an untrusted peer (e.g., a DataLayer sync partner) and calling `proof.valid()` without separately verifying `proof.root_hash()` against a locally-trusted root would be fully deceived. The likelihood is medium — it depends on whether callers omit the root-hash cross-check, which the misleading API name encourages. [3](#0-2) 

### Recommendation
The `valid()` method should accept a trusted root hash parameter and compare the computed root against it:

```rust
pub fn valid(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root
}
```

Alternatively, rename the current method to `is_internally_consistent()` and add a separate `valid(trusted_root: &Hash) -> bool` that performs the full check including root comparison.

### Proof of Concept
```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer

# Forge a proof for an arbitrary node_hash not present in any real tree
fake_node_hash   = bytes([0xAA] * 32)
fake_sibling     = bytes([0xBB] * 32)

# Compute a valid combined_hash using the same calculate_internal_hash logic
# (left-side: sha256(prefix || fake_node_hash || fake_sibling))
import hashlib
combined = hashlib.sha256(fake_node_hash + fake_sibling).digest()  # simplified

layer = ProofOfInclusionLayer(
    other_hash_side=0,          # Side.Left
    other_hash=fake_sibling,
    combined_hash=combined,
)

forged = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

# valid() returns True — tautological check passes
assert forged.valid()

# forged.root_hash() == combined, which is NOT the real tree root.
# Any verifier that only calls forged.valid() accepts this fabricated proof.
