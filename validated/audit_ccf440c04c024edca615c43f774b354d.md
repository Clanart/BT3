### Title
Unvalidated `l1_gas_price_wei` / `l1_data_gas_price_wei` in `ProposalInit` Committed into Block Hash and Used for Fee Calculation - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_block_info_valid` validates the FRI-denominated L1 gas prices from a proposer's `ProposalInit` against the validator's own oracle reading, but silently discards the validator's independently-computed WEI prices and accepts the proposer's `l1_gas_price_wei` / `l1_data_gas_price_wei` without any cross-check. Both WEI values are then committed verbatim into the block hash via `gas_prices_to_hash` and used to compute ETH-denominated fees for L1 handler transactions and to derive `l2_gas_price_wei`. A malicious proposer (any consensus participant) can therefore inject arbitrary WEI prices that pass consensus validation, producing a wrong block hash commitment and incorrect fee accounting with economic impact.

### Finding Description

**Validation gap in `is_block_info_valid`**

`is_block_info_valid` calls `get_l1_prices_in_fri_and_wei` to obtain the validator's own oracle-derived prices, but only uses the FRI half:

```rust
let (l1_gas_prices_fri, _l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
    l1_gas_price_provider,
    init_proposed.timestamp,
    block_info_validation.previous_block_info.as_ref(),
    gas_price_params,
).await;
```

The `_l1_gas_prices_wei` result is discarded. The function then checks only `l1_gas_price_fri` and `l1_data_gas_price_fri` against the margin:

```rust
if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
    && within_margin(l1_data_gas_price_fri_proposed, l1_data_gas_price_fri, l1_gas_price_margin_percent))
```

`init_proposed.l1_gas_price_wei` and `init_proposed.l1_data_gas_price_wei` are never compared against the validator's oracle-derived WEI values. [1](#0-0) 

**WEI prices enter the block hash commitment**

After validation passes, `convert_to_sn_api_block_info(init)` builds a `BlockInfo` directly from the proposer's `init.l1_gas_price_wei` and `init.l1_data_gas_price_wei`. `PartialBlockHashComponents::new` then captures these values:

```rust
l1_gas_price: block_info.gas_prices.l1_gas_price_per_token(),   // includes price_in_wei
l1_data_gas_price: block_info.gas_prices.l1_data_gas_price_per_token(),
```

`calculate_block_hash` chains them through `gas_prices_to_hash`, which for Starknet ≥ 0.13.4 hashes all six price fields (wei + fri for each of L1 gas, L1 data gas, L2 gas) into the block hash:

```rust
.chain(&l1_gas_price.price_in_wei.0.into())
.chain(&l1_gas_price.price_in_fri.0.into())
.chain(&l1_data_gas_price.price_in_wei.0.into())
.chain(&l1_data_gas_price.price_in_fri.0.into())
``` [2](#0-1) [3](#0-2) 

**Downstream corruption of `l2_gas_price_wei` and fallback rate**

`convert_to_sn_api_block_info` derives `l2_gas_price_wei` by calling `init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)`, where `eth_to_fri_rate` is computed via `calculate_eth_to_fri_rate(&previous_block_info)`. `PreviousBlockInfo::from(init)` populates `l1_prices_wei` directly from `init.l1_gas_price_wei`. A manipulated WEI price therefore corrupts the reconstructed ETH-to-FRI rate, which in turn corrupts `l2_gas_price_wei` used for all L2 gas fee calculations in the block. [4](#0-3) [5](#0-4) 

**Fallback propagation to subsequent blocks**

When the oracle fails for a later block, `get_l1_prices_in_fri_and_wei_and_conversion_rate` falls back to `previous_block_info`, which was populated from the accepted (manipulated) `ProposalInit`. The corrupted WEI prices then propagate as the fallback rate for the next block. [6](#0-5) 

### Impact Explanation

**Critical – Wrong block hash commitment**: The `PartialBlockHash` / `ProposalCommitment` computed by both proposer and validator is derived from the proposer-supplied WEI prices. Because validators re-execute with the same `init`, they reproduce the same wrong hash and reach consensus on it. The committed block hash encodes incorrect gas price data, violating the block hash invariant.

**Critical – Incorrect fee with economic impact**: ETH-denominated fees for L1 handler transactions are computed from `l1_gas_price_wei` / `l1_data_gas_price_wei`. A proposer setting these to near-zero causes L1 handler transactions to be undercharged; setting them to `u128::MAX` causes overcharge or arithmetic overflow in fee paths. The derived `l2_gas_price_wei` is also wrong, affecting all L2 gas fee calculations in the block.

### Likelihood Explanation

Any consensus participant can act as proposer in a round. No special key or admin privilege is required. The attacker only needs to craft a `ProposalInit` with valid FRI prices (within the oracle margin) and arbitrary WEI prices. The existing `within_margin` check on FRI prices will pass; no check exists for WEI prices. This is directly analogous to the external report's oracle front-running: the oracle (here, the proposer) can set the price variable to an adversarial value before the transaction (here, block execution) is processed.

### Recommendation

In `is_block_info_valid`, compare the proposer's WEI prices against the validator's independently-computed WEI prices using the same `within_margin` check already applied to FRI prices:

```rust
let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(...).await;

// existing FRI check ...

if !(within_margin(init_proposed.l1_gas_price_wei, l1_gas_prices_wei.l1_gas_price, margin)
    && within_margin(init_proposed.l1_data_gas_price_wei, l1_gas_prices_wei.l1_data_gas_price, margin))
{
    return Err(ValidateProposalError::InvalidBlockInfo(...));
}
```

This mirrors the existing FRI validation and closes the gap between the two price representations that are both committed into the block hash.

### Proof of Concept

1. Validator node V is validating a proposal from proposer P at height H.
2. Oracle reports `base_fee_per_gas = 1_000_000_000 wei`, `eth_to_fri_rate = 2_000_000_000_000_000_000_000` (2000 STRK/ETH), giving `l1_gas_price_fri ≈ 2_000_000_000_000 fri`.
3. P crafts `ProposalInit` with:
   - `l1_gas_price_fri = 2_000_000_000_000` (valid, within margin)
   - `l1_gas_price_wei = 1` (manipulated, should be `1_000_000_000`)
4. `is_block_info_valid` checks only FRI prices → passes.
5. `convert_to_sn_api_block_info(init)` builds `BlockInfo` with `l1_gas_price.price_in_wei = 1`.
6. `PartialBlockHashComponents::new` captures `price_in_wei = 1`.
7. `gas_prices_to_hash` chains `1` into the Poseidon hash → wrong `PartialBlockHash` committed.
8. All L1 handler transactions in the block are charged `1 wei` per gas unit instead of `1_000_000_000 wei` → economic loss.
9. `calculate_eth_to_fri_rate` reconstructs rate as `fri / (wei * ETH_TO_WEI_FACTOR)` = `2_000_000_000_000 / (1 * 10^18)` ≈ 0 → `l2_gas_price_wei` collapses to 0 or 1, corrupting all L2 fee calculations in the block. [7](#0-6) [8](#0-7) [9](#0-8)

#[instrument(level = "warn", skip_all, fields(?block_info_validation, ?init_proposed))]
async fn is_block_info_valid(
    block_info_validation: &BlockInfoValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        block_info_validation.previous_block_info.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidBlockInfo(
            init_proposed.clone(),
            block_info_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + block_info_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidBlockInfo(
            init_proposed.clone(),
            block_info_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now, block_info_validation.block_timestamp_window_seconds, init_proposed.timestamp
            ),
        ));
    }
    if !(init_proposed.height == block_info_validation.height
        && init_proposed.l1_da_mode == block_info_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == block_info_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidBlockInfo(
            init_proposed.clone(),
            block_info_validation.clone(),
            "Block info validation failed".to_string(),
        ));
    }
    let (l1_gas_prices_fri, _l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
        l1_gas_price_provider,
        init_proposed.timestamp,
        block_info_validation.previous_block_info.as_ref(),
        gas_price_params,
    )
    .await;
    let l1_gas_price_margin_percent =
        VersionedConstants::latest_constants().l1_gas_price_margin_percent.into();
    debug!("L1 price info: {l1_gas_prices_fri:?}");

    let l1_gas_price_fri = l1_gas_prices_fri.l1_gas_price;
    let l1_data_gas_price_fri = l1_gas_prices_fri.l1_data_gas_price;
    let l1_gas_price_fri_proposed = init_proposed.l1_gas_price_fri;
    let l1_data_gas_price_fri_proposed = init_proposed.l1_data_gas_price_fri;

    if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_fri_proposed,
            l1_data_gas_price_fri,
            l1_gas_price_margin_percent,
        ))
    {
        return Err(ValidateProposalError::InvalidBlockInfo(
            init_proposed.clone(),
            block_info_validation.clone(),
            format!(
                "L1 gas price mismatch: expected L1 gas price FRI={l1_gas_price_fri}, \
                 proposed={l1_gas_price_fri_proposed}, expected L1 data gas price \
                 FRI={l1_data_gas_price_fri}, proposed={l1_data_gas_price_fri_proposed}, \
                 l1_gas_price_margin_percent={l1_gas_price_margin_percent}"
            ),
        ));
    }
    Ok(())
}

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let previous_block_info = PreviousBlockInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;

    Ok(starknet_api::block::BlockInfo {

        block_timestamp: BlockTimestamp(init.timestamp),
