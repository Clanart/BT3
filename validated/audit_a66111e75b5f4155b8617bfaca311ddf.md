### Title
Missing Class Hash Existence Validation in `execute_replace_class` Allows Permanent Fund Freeze — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS updates a contract's class hash in the contract state tree without verifying that the new class hash exists in the contract class tree. This creates a two-layer accounting mismatch analogous to the WrappedVault share-transfer bug: the outer layer (contract state tree) is updated, but the inner layer (contract class tree) is never validated. A contract deployer can exploit this to permanently freeze all funds held by any contract.

---

### Finding Description

In `execute_replace_class`, the OS reads the current `StateEntry` for the calling contract and writes a new `StateEntry` with the caller-supplied `class_hash`:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The `class_hash` value written into `contract_state_changes` is never cross-checked against `contract_class_changes`. The contract class tree maps `class_hash → compiled_class_hash`; if the supplied `class_hash` has no entry there, the contract state tree and the class tree are permanently inconsistent.

The two-layer structure is explicit in the OS:

- **Layer 1 (contract state tree)**: maps `contract_address → StateEntry { class_hash, storage_ptr, nonce }` — updated by `execute_replace_class`.
- **Layer 2 (contract class tree)**: maps `class_hash → compiled_class_hash` — **never consulted** by `execute_replace_class`. [2](#0-1) 

Once the state is committed (end of block), the contract's `class_hash` field permanently points to a non-existent class. Every subsequent call to the contract will fail at class resolution, and because the broken `replace_class` transaction already succeeded and was committed, no future transaction can undo it. The contract's storage (and any token balances it holds) is permanently inaccessible.

---

### Impact Explanation

**Permanent freezing of funds.**

Any ERC-20 balance, ETH, or other asset stored in the contract's storage becomes unreachable. The contract cannot self-repair: `replace_class` can only be called from within the contract's own execution context, but every attempt to execute the contract fails because the class does not exist. There is no protocol-level recovery path.

---

### Likelihood Explanation

The entry path requires only a contract deployer:

1. Deploy any contract (or use an existing one the attacker controls).
2. Call an entry point that invokes `replace_class` with an arbitrary felt that is not a declared class hash (e.g., `1`).
3. The OS accepts the syscall, commits the block, and the contract is permanently broken.

No privileged role, leaked key, or external dependency is required. The attacker needs only the ability to submit a transaction — the minimum privilege level on StarkNet.

---

### Recommendation

Before writing the new `StateEntry`, verify that `class_hash` exists in `contract_class_changes` (i.e., its compiled class hash is non-zero). The developers have already identified this gap:

```
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
``` [3](#0-2) 

The fix should perform a `dict_read` on `contract_class_changes` for the supplied `class_hash` and assert the result is non-zero before allowing the state update to proceed.

### Proof of Concept

**Attack sequence (Cairo pseudocode):**

```cairo
// Attacker's contract entry point
@external
func freeze_self{syscall_ptr: felt*, ...}() {
    // Pass a class hash that was never declared — e.g., felt 0xDEAD
    replace_class(class_hash=0xDEAD);
    return ();
}
```

**OS execution path:**

1. `execute_replace_class` is called with `request.class_hash = 0xDEAD`.
2. `GetContractAddressStateEntry` hint loads the current `StateEntry`.
3. A new `StateEntry` with `class_hash=0xDEAD` is written to `contract_state_changes` — **no lookup in `contract_class_changes`**.
4. Block is finalized; `state_update` commits both trees. The class tree has no entry for `0xDEAD`.
5. In any subsequent block, any call to the contract reaches `execute_call_contract`, which reads `state_entry.class_hash = 0xDEAD` and attempts to resolve the compiled class — resolution fails, transaction reverts.
6. The contract's storage (funds) is permanently inaccessible. [4](#0-3) [5](#0-4)
