### Title
`ACTION_INIT_TRANSFER` in `coreReceiveWithData` Never Passes Wormhole Message Fee, Making HyperCore→Bridge Path Permanently Non-Functional — (File: `evm/src/omni-bridge/contracts/HlBridgeToken.sol`)

---

### Summary

`HyperliquedBridgeToken.coreReceiveWithData` calls `OmniBridgeWormhole.initTransfer` without attaching any ETH value. Because `OmniBridgeWormhole.initTransferExtension` forwards exactly the value it receives to `_wormhole.publishMessage{value: value}(...)`, and Wormhole's `messageFee()` is non-zero on every live deployment, every `ACTION_INIT_TRANSFER` invocation reverts unconditionally. The entire HyperCore→Omni-Bridge transfer path is permanently non-functional.

---

### Finding Description

`HyperliquedBridgeToken.coreReceiveWithData` handles two actions. For `ACTION_INIT_TRANSFER` it:

1. Moves tokens from the system pool to itself via `_update(_systemAddress, address(this), amount)`.
2. Calls `IOmniBridgeInitTransfer(owner()).initTransfer(address(this), amount128, fee, 0, recipient, message)` — **with no `{value: ...}` attached and `nativeFee = 0`**. [1](#0-0) 

Because no ETH is forwarded, `msg.value` inside `OmniBridge.initTransfer` is `0`. The base contract computes:

```
extensionValue = msg.value - nativeFee = 0 - 0 = 0
```

and passes `extensionValue` to `initTransferExtension`. [2](#0-1) 

`OmniBridgeWormhole.initTransferExtension` then calls:

```solidity
_wormhole.publishMessage{value: value}(wormholeNonce, payload, _consistencyLevel);
```

with `value = 0`. [3](#0-2) 

Wormhole's `publishMessage` requires `msg.value >= messageFee()`. On every live Wormhole deployment `messageFee() > 0`, so the call reverts unconditionally. The entire transaction reverts, including the `_update` in step 1.

Contrast this with the direct user-facing `initTransfer` path, where the caller is expected to supply `nativeFee + wormholeFee` as `msg.value`, and `extensionValue = msg.value - nativeFee` correctly funds Wormhole. The `coreReceiveWithData` path has no equivalent mechanism. [4](#0-3) 

---

### Impact Explanation

The `ACTION_INIT_TRANSFER` branch of `coreReceiveWithData` is **completely non-functional** on any live Wormhole deployment. No HyperCore user can bridge tokens to NEAR or other chains via this path.

Additionally, HyperLiquid's system calls `coreReceiveWithData` after the user has committed a `sendToEvmWithData` action on HyperCore. If HyperLiquid's system does not atomically restore the user's HyperCore-side balance when the EVM transaction reverts (a realistic behavior for a system that treats the EVM call as a one-way dispatch), the user's funds are permanently lost: deducted on HyperCore and never bridged on EVM. This constitutes an irrecoverable lock of user funds in the bridge flow.

---

### Likelihood Explanation

Any HyperCore user who attempts to bridge tokens outbound via `sendToEvmWithData` with `ACTION_INIT_TRANSFER` triggers this path. On a live Wormhole deployment the revert is 100% deterministic — no special conditions are required. The vulnerability is reachable by any unprivileged HyperCore token holder.

---

### Recommendation

In `coreReceiveWithData`, forward the Wormhole message fee when calling `initTransfer`. The contract should either:

1. Accept ETH from the HyperLiquid system call (if the system supports attaching value) and forward it: `IOmniBridgeInitTransfer(owner()).initTransfer{value: msg.value}(...)`.
2. Or maintain an internal ETH reserve (topped up via `receive()`) and forward the exact `IWormhole(_wormhole).messageFee()` amount when calling `initTransfer`.

Additionally, add a `receive() external payable {}` to `HyperliquedBridgeToken` so the contract can be pre-funded with ETH to cover Wormhole fees.

---

### Proof of Concept

1. Deploy `OmniBridgeWormhole` with a live Wormhole endpoint where `messageFee() = X > 0`.
2. Deploy `HyperliquedBridgeToken` with `owner = OmniBridgeWormhole`.
3. Mint tokens to `_systemAddress` via the 3-arg `mint` (simulating HyperCore deposits).
4. As `_systemAddress`, call:
   ```solidity
   token.coreReceiveWithData(
       user,
       bytes32(0), uint32(0),
       100e18,
       0,
       abi.encodePacked(uint8(1), abi.encode(uint128(0), "near:alice.near", ""))
   );
   ```
5. The call reverts with Wormhole's fee-insufficient error. No tokens are bridged. If HyperLiquid does not restore the user's HyperCore balance, the 100e18 tokens are permanently lost. [5](#0-4) [6](#0-5)

    function coreReceiveWithData(
        address from,
        bytes32 /*destinationRecipient*/,
        uint32 /*destinationChainId*/,
        uint256 amount,
        uint64 /*coreNonce*/,
        bytes calldata data
    ) external override {
        if (msg.sender != _systemAddress) revert NotSystemAddress();
        if (data.length == 0) revert EmptyActionData();

        uint8 action = uint8(data[0]);
        bytes calldata tail = data[1:];

        if (action == ACTION_TRANSFER) {
            address recipient = abi.decode(tail, (address));
            _update(_systemAddress, recipient, amount);
        } else if (action == ACTION_INIT_TRANSFER) {
            (uint128 fee, string memory recipient, string memory message) = abi
                .decode(tail, (uint128, string, string));
            uint128 amount128 = amount.toUint128();
            _update(_systemAddress, address(this), amount);
            IOmniBridgeInitTransfer(owner()).initTransfer(
                address(this),
                amount128,
                fee,
                0,
                recipient,
                message
            );
        } else {
            revert UnknownAction(action);
        }

        emit CoreReceived(from, action, amount, data);
    }

    function initTransferExtension(
        address sender,
        address tokenAddress,
        uint64 originNonce,
        uint128 amount,
        uint128 fee,
        uint128 nativeFee,
        string calldata recipient,
        string calldata message,
        uint256 value
    ) internal override {
        bytes memory payload = bytes.concat(
            bytes1(uint8(MessageType.InitTransfer)),
            bytes1(omniBridgeChainId),
            Borsh.encodeAddress(sender),
            bytes1(omniBridgeChainId),
            Borsh.encodeAddress(tokenAddress),
            Borsh.encodeUint64(originNonce),
            Borsh.encodeUint128(amount),
            Borsh.encodeUint128(fee),
            Borsh.encodeUint128(nativeFee),
            Borsh.encodeString(recipient),
            Borsh.encodeString(message)
        );
        // slither-disable-next-line reentrancy-eth
        _wormhole.publishMessage{value: value}(
            wormholeNonce,
            payload,
            _consistencyLevel
        );

        wormholeNonce++;
    }
