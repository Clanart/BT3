### Title
Unauthenticated `logMetadata` Allows Anyone to Emit `LogMetadata` Event with Attacker-Controlled Token Metadata, Enabling Unauthorized Token Deployment on NEAR - (File: evm/src/omni-bridge/contracts/OmniBridge.sol)

### Summary
`OmniBridge.logMetadata` and `OmniBridge.logMetadata1155` are external functions with no authentication constraints. Any unprivileged caller can supply an arbitrary `tokenAddress` — including a malicious contract returning attacker-chosen `name`, `symbol`, and `decimals` — causing the legitimate OmniBridge contract to emit a `LogMetadata` event. Because the NEAR bridge prover validates only that the event was emitted by a registered factory (the OmniBridge address), this event is accepted as valid proof and triggers deployment of a NEAR bridge token with attacker-controlled metadata, including a corrupted decimal value.

### Finding Description
`logMetadata` is declared `external payable` with no role check, no pause guard, and no validation that `tokenAddress` is a legitimate or pre-approved token:

```solidity
function logMetadata(address tokenAddress) external payable {
    string memory name = IERC20Metadata(tokenAddress).name();
    string memory symbol = IERC20Metadata(tokenAddress).symbol();
    uint8 decimals = IERC20Metadata(tokenAddress).decimals();

    logMetadataExtension(tokenAddress, name, symbol, decimals);

    emit BridgeTypes.LogMetadata(tokenAddress, name, symbol, decimals);
}
```

The emitted `LogMetadata` event is the first step in the cross-chain token deployment pipeline. On NEAR, `deploy_token` consumes a proof of this event and deploys a bridged token. The NEAR-side validation only checks that the `emitter_address` is a registered factory:

```rust
require!(
    self.factories.get(&chain) == Some(metadata.emitter_address),
    BridgeError::UnknownFactory.as_ref()
);
```

Since the event is genuinely emitted by the OmniBridge contract (a registered factory), this check passes. The NEAR bridge then deploys a token using the attacker-supplied `name`, `symbol`, and `decimals` from the malicious ERC20.

`logMetadata1155` has the same missing access control and additionally writes on-chain state (`multiTokens` mapping) without authentication:

```solidity
function logMetadata1155(address tokenAddress, uint256 tokenId) external payable {
    ...
    if (multiToken.tokenAddress == address(0)) {
        multiToken.tokenAddress = tokenAddress;
        multiToken.tokenId = tokenId;
    }
    ...
    emit BridgeTypes.LogMetadata(...);
}
```

The StarkNet `log_metadata` function has the identical missing-access-control pattern.

### Impact Explanation
An attacker deploys a malicious ERC20 at address X returning `decimals() = 0` (or any wrong value), `name() = "USD Coin"`, `symbol() = "USDC"`. They call `logMetadata(X)`. The NEAR bridge deploys a NEAR bridge token for EVM address X with 0 decimals. Any subsequent bridging through address X uses this corrupted decimal configuration, causing severe amount miscalculation: amounts are normalized/denormalized against 0 decimals, breaking bridge collateralization and misdirecting value for any user who bridges through that EVM address. This fits the allowed impact: **decimal/accounting corruption that breaks bridge collateralization or misdirects value**, and **unauthorized token deployment**.

### Likelihood Explanation
The entry path requires only a standard EVM transaction from any unprivileged account. No special role, leaked key, or privileged access is needed. The attacker only needs to deploy a minimal malicious ERC20 (trivial) and call `logMetadata`. The NEAR bridge relayer will automatically process the resulting event.

### Recommendation
Add an access control guard to `logMetadata`, `logMetadata1155`, and the StarkNet `log_metadata` so that only authorized accounts (e.g., `DEFAULT_ADMIN_ROLE` or a dedicated `METADATA_MANAGER_ROLE`) can trigger metadata logging, analogous to how `setMetadata` is already protected:

```solidity
function logMetadata(address tokenAddress) external payable onlyRole(DEFAULT_ADMIN_ROLE) {
    ...
}
```

Alternatively, restrict `tokenAddress` to addresses that are already registered ERC20 tokens or that pass a whitelist check, preventing arbitrary malicious contracts from being used as the metadata source.

### Proof of Concept
1. Attacker deploys `MaliciousToken` at address `X`:
   - `name()` returns `"USD Coin"`
   - `symbol()` returns `"USDC"`
   - `decimals()` returns `0`
2. Attacker calls `OmniBridge.logMetadata(X)` — no authentication required.
3. OmniBridge emits `LogMetadata(X, "USD Coin", "USDC", 0)` from the legitimate factory address.
4. NEAR bridge relayer picks up the event; prover validates `emitter_address == registered_factory` — passes.
5. NEAR `deploy_token_callback` calls `deploy_token_internal` with `decimals = 0`, deploying a NEAR bridge token for EVM address `X` with 0 decimals.
6. Any user who bridges tokens through EVM address `X` has their amounts normalized against 0 decimals, causing catastrophic accounting corruption.

        fn log_metadata(ref self: ContractState, token: ContractAddress) {
            // There are two possible metadata standards in use.
            // 1. Old style: name and symbol are felt252 values.
            // 2. New style: name and symbol are ByteArray values (ERC20 ABI).
            // We are using low-level contract calls to determine the type.

            let call_data: Array<felt252> = array![];
            let mut res = syscalls::call_contract_syscall(
                token, selector!("name"), call_data.span(),
            )
                .unwrap_syscall();

            let name = if res.len() == 1 {
                // Old standard (felt252)
                let name = OptionTrait::expect(
                    Serde::<felt252>::deserialize(ref res), 'Could not deserialize name',
                );
                utils::felt252_to_string(name)
            } else {
                // New standard (ByteArray)
                OptionTrait::expect(
                    Serde::<ByteArray>::deserialize(ref res), 'Could not deserialize name',
                )
            };

            let mut res = syscalls::call_contract_syscall(
                token, selector!("symbol"), call_data.span(),
            )
                .unwrap_syscall();

            let symbol = if res.len() == 1 {
                // Old standard (felt252)
                let symbol = OptionTrait::expect(
                    Serde::<felt252>::deserialize(ref res), 'Could not deserialize symbol',
                );
                utils::felt252_to_string(symbol)
            } else {
                // New standard (ByteArray)
                OptionTrait::expect(
                    Serde::<ByteArray>::deserialize(ref res), 'Could not deserialize symbol',
                )
            };

            let decimals = {
                let mut res = syscalls::call_contract_syscall(
                    token, selector!("decimals"), call_data.span(),
                )
                    .unwrap_syscall();

                let decimals = OptionTrait::expect(
                    Serde::<u8>::deserialize(ref res), 'Could not deserialize decimals',
                );
                decimals
            };

            self.emit(Event::LogMetadata(LogMetadata { address: token, name, symbol, decimals }))
        }
