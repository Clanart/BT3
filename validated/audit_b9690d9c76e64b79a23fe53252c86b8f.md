### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary felt value as the new class hash without verifying that the hash corresponds to a previously declared class. A malicious contract can exploit this to replace its own class hash with `UNINITIALIZED_CLASS_HASH` (0) or any undeclared hash, permanently rendering the contract uncallable and freezing all funds held within it. The OS itself contains a TODO comment acknowledging this missing check.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the requested new class hash directly from the syscall request and writes it into the contract's `StateEntry` without any validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO at line 898 explicitly acknowledges the missing check. No constraint prevents `class_hash` from being `0` (`UNINITIALIZED_CLASS_HASH`) or any arbitrary undeclared felt.

The constant `UNINITIALIZED_CLASS_HASH = 0` is defined in `commitment.cairo` and is the sentinel value used by `deploy_contract` to detect undeployed addresses: [2](#0-1) 

When a contract's `class_hash` is set to 0 or an undeclared value, any subsequent `call_contract` syscall targeting that address will construct an `ExecutionContext` with an invalid `class_hash`: [3](#0-2) 

The OS will attempt to execute an entry point under a class that does not exist in the class tree, causing every future call to that contract to fail irreversibly. The contract's storage and any token balances it holds become permanently inaccessible.

The `deploy_contract` function enforces `assert state_entry.class_hash = UNINITIALIZED_CLASS_HASH` and `assert state_entry.nonce = 0` before allowing redeployment to the same address: [4](#0-3) 

Because a used contract will have a nonce > 0 (incremented by `check_and_increment_nonce`), redeployment to the same address is also blocked, making the freeze permanent. [5](#0-4) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any contract that calls `replace_class` with `class_hash = 0` or an undeclared hash becomes permanently uncallable. All ERC-20 token balances, ETH/STRK, or other assets held in that contract's storage are irretrievably frozen. Because the OS state commitment faithfully records the invalid class hash in the Patricia tree, the freeze is reflected in the canonical chain state and cannot be undone without a protocol-level upgrade.

---

### Likelihood Explanation

**Medium.** The attack requires a malicious contract operator to first attract user deposits (e.g., by deploying a contract that mimics a legitimate DeFi protocol), then execute `replace_class(0)` in a single transaction. The entry path is fully unprivileged: any contract deployer can issue the `REPLACE_CLASS` syscall. No special role, leaked key, or external dependency is required. The OS unconditionally accepts the syscall as shown by the code and the acknowledged TODO.

---

### Recommendation

Add an explicit validation inside `execute_replace_class` that the supplied `class_hash` exists in `contract_class_changes` (i.e., has a non-zero `compiled_class_hash` entry). Concretely, after reading `class_hash` from the request, perform a `dict_read` on `contract_class_changes` and assert the result is non-zero before writing the new `StateEntry`. This mirrors the pattern already used in `execute_declare_transaction` where `assert_not_zero(compiled_class_hash)` is enforced before updating the class tree. [6](#0-5) 

---

### Proof of Concept

1. **Attacker deploys a contract** (e.g., a token vault) via `execute_deploy_account_transaction` or the `deploy` syscall. The contract's `StateEntry` is initialized with a valid declared `class_hash` and `nonce = 0`.

2. **Users deposit funds** into the vault contract over multiple transactions. The contract's nonce increments and its storage holds user balances.

3. **Attacker calls `replace_class(0)`** from within the vault contract in a single invoke transaction. The OS executes `execute_replace_class`:
   - Reads `class_hash = 0` from the syscall request.
   - Skips the missing declared-class check (TODO at line 898).
   - Writes `new StateEntry(class_hash=0, storage_ptr=..., nonce=N)` into `contract_state_changes`.

4. **State is committed.** The Patricia tree now records the vault contract with `class_hash = 0`.

5. **Any subsequent call to the vault** constructs `ExecutionContext` with `class_hash = 0`. The OS cannot find an entry point under the uninitialized class hash. Every call fails.

6. **Redeployment is blocked** because `nonce = N > 0`, so `deploy_contract`'s `assert state_entry.nonce = 0` fails.

7. **All user funds are permanently frozen** with no recovery path at the OS level.

func check_and_increment_nonce{contract_state_changes: DictAccess*}(tx_info: TxInfo*) -> () {
    // Do not handle nonce for version 0.
    if (tx_info.version == 0) {
        return ();
    }

    tempvar state_entry: StateEntry*;
    %{ SetStateEntryToAccountContractAddress %}

    tempvar current_nonce = state_entry.nonce;
    with_attr error_message("Unexpected nonce.") {
        assert current_nonce = tx_info.nonce;
    }

    // Update contract_state_changes.
    tempvar new_state_entry = new StateEntry(
        class_hash=state_entry.class_hash,
        storage_ptr=state_entry.storage_ptr,
        nonce=current_nonce + 1,
    );
    dict_update{dict_ptr=contract_state_changes}(
        key=tx_info.account_contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    return ();
