### Title
`batch_insert` Bypasses Duplicate-Key Validation, Silently Corrupting the DataLayer Merkle Tree Root - (File: `crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::batch_insert` applies duplicate-key and duplicate-hash guards only to the last two items it pops from the input vector. All remaining items are written directly into the blob via `insert_entry_to_blob` with no uniqueness check. An unprivileged caller supplying a batch that contains a repeated `KeyId` or `Hash` among the non-popped items silently produces a structurally corrupt Merkle tree whose root hash is wrong, invalidating every subsequent proof of inclusion or exclusion derived from it.

---

### Finding Description

`MerkleBlob::insert` (the single-item path) enforces two invariants before writing anything:

```rust
// blob.rs:369-374
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
```

`batch_insert` calls `self.insert(…)` for at most the **last two** items it pops from the vector:

```rust
// blob.rs:578-585
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else {
            return Ok(());
        };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;
    }
}
```

Every other item in the batch is written through the fast path:

```rust
// blob.rs:587-603
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    let new_block = Block { … Node::Leaf(LeafNode { hash, key, value, … }) … };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
```

`insert_entry_to_blob` writes the raw block bytes and updates `block_status_cache` via `add_leaf`, which calls `HashMap::insert` — silently overwriting the existing cache entry for a repeated key. The blob therefore ends up with **two physical leaf nodes sharing the same `KeyId`**, while the cache tracks only the later one. The subtree built from `indexes` is then grafted onto the existing tree, producing an internal-node hash computed over the duplicate leaf's hash — a root that does not correspond to any valid set of key-value pairs.

`check_just_integrity` will detect the mismatch (`leaf_count != key_to_index_cache_length`) only if called explicitly; `batch_insert` itself returns `Ok(())`.

The Python binding `py_batch_insert` (line 1503) exposes this path directly to any caller:

```rust
// blob.rs:1503-1519
#[pyo3(name = "batch_insert")]
pub fn py_batch_insert(
    &mut self,
    keys_values: Vec<(KeyId, ValueId)>,
    hashes: Vec<Hash>,
) -> PyResult<()> {
    …
    self.batch_insert(zip(keys_values, hashes).collect())?;
    Ok(())
}
```

---

### Impact Explanation

A corrupt root hash means every `get_proof_of_inclusion` call returns a proof that fails external verification against the committed root, and `get_proof_of_inclusion` for the shadowed duplicate leaf is unreachable through the cache. An attacker who controls the batch input can therefore:

- Commit a root that does not faithfully represent the key-value set, enabling forged inclusion proofs (the tree "contains" a key whose proof verifies against the wrong root).
- Make a legitimately inserted key unprovable (the cache points to the wrong physical leaf).
- Cause `check_integrity` to raise an error on a tree that was accepted without error, creating a cross-node disagreement about tree validity.

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."**

---

### Likelihood Explanation

The Python binding is the primary DataLayer API surface. Any caller — including a DataLayer client constructing a sync delta — can supply a `batch_insert` list with a repeated key. No privilege, key material, or network position is required. The bug is triggered by ordinary API input.

---

### Recommendation

Add a pre-pass deduplication check inside `batch_insert` before the fast-path loop, mirroring the guards in `insert`:

```rust
pub fn batch_insert(
    &mut self,
    mut keys_values_hashes: Vec<((KeyId, ValueId), Hash)>,
) -> Result<(), Error> {
    // Reject duplicates within the batch itself
    let mut seen_keys: HashSet<KeyId> = HashSet::new();
    let mut seen_hashes: HashSet<Hash> = HashSet::new();
    for ((key, _), hash) in &keys_values_hashes {
        if self.block_status_cache.contains_key(*key) || !seen_keys.insert(*key) {
            return Err(Error::KeyAlreadyPresent());
        }
        if self.block_status_cache.contains_leaf_hash(hash) || !seen_hashes.insert(*hash) {
            return Err(Error::HashAlreadyPresent());
        }
    }
    // … existing logic …
}
```

Alternatively, call `self.insert(…)` for every item (accepting the performance cost) or add a post-insert `check_integrity()` call that returns an error instead of panicking.

### Proof of Concept

```rust
use chia_datalayer::{Hash, InsertLocation, KeyId, MerkleBlob, ValueId};
use chia_protocol::Bytes32;

fn main() {
    let mut blob = MerkleBlob::new(Vec::new()).unwrap();

    // Seed with 2 leaves so leaf_count > 1 — the bootstrap branch is skipped.
    blob.insert(KeyId(0), ValueId(0), &Hash(Bytes32::new([0u8; 32])), InsertLocation::Auto {}).unwrap();
    blob.insert(KeyId(1), ValueId(1), &Hash(Bytes32::new([1u8; 32])), InsertLocation::Auto {}).unwrap();

    // Batch with a duplicate KeyId(2) — both entries bypass the duplicate check.
    let batch = vec![
        ((KeyId(2), ValueId(2)), Hash(Bytes32::new([2u8; 32]))),
        ((KeyId(2), ValueId(99)), Hash(Bytes32::new([3u8; 32]))), // duplicate key
        ((KeyId(3), ValueId(3)), Hash(Bytes32::new([4u8; 32]))),
    ];

    // Returns Ok(()) — no error raised despite duplicate KeyId(2)
    blob.batch_insert(batch).unwrap();

    // Tree is now corrupt: two physical leaf nodes share KeyId(2).
    // check_integrity() will return an error.
    blob.check_integrity().unwrap_err(); // IntegrityKeyToIndexCacheLength or similar
}
```
