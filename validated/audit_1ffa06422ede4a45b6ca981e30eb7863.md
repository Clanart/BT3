### Title
Missing Class Hash Validation in `execute_replace_class` Enables Permanent Freezing of Contract Funds - (File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo)

### Summary
The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary felt value as the new class hash without verifying it corresponds to a declared contract class. This is the direct analog of the reported path traversal: just as `ReadProposalFlag` reads any file path without validation, `execute_replace_class` accepts any class hash "path" into the class registry without bounds-checking. A contract can permanently disable itself — and freeze all funds it holds — by calling `replace_class` with an undeclared hash. The OS itself acknowledges this gap with an explicit TODO comment at the vulnerable line.

### Finding Description

In `execute_replace_class`, the new class hash supplied by the calling contract is written directly into `contract_state_changes` with no check that the hash exists in the declared class registry (`contract_class_changes`): [1](#0-0) 

The comment at line 898 is an explicit, developer-acknowledged admission of the missing guard:

```cairo
let class_hash = request.class_hash;
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
```

After `replace_class` succeeds with an undeclared hash, any subsequent call into that contract reaches `execute_entry_point`, which performs a two-step lookup:

1. `dict_read` on `contract_class_changes` for the (undeclared) class hash → returns `0` (the Cairo dict default for an unseen key).
2. `find_element` on the compiled class facts bundle searching for compiled-class-hash `0`. [2](#0-1) 

`find_element` is a hint-plus-assertion primitive: if the key is absent from the array, the assertion fails and the Cairo program aborts — it does **not** produce a graceful revert. Because the validate step of every subsequent invoke transaction from that account runs through `execute_entry_point` unconditionally, the OS cannot produce a valid proof for any block that includes such a transaction, even a reverted one.

### Impact Explanation

**Permanent freezing of funds (Critical).** Once a contract's class hash is replaced with an undeclared value:

- The contract is permanently non-executable. No entry point (external, L1 handler, or constructor) can be reached.
- All ERC-20 tokens, ETH, STRK, or any other asset held in the contract's storage are irrecoverably locked. There is no upgrade path, no escape hatch, and no admin override at the protocol level.
- Because the OS aborts (rather than reverts) when `find_element` cannot locate the compiled class, any block containing a subsequent call to the frozen contract becomes unprovable, creating a secondary risk of network halt for the affected block.

### Likelihood Explanation

The attack surface is broad:

- **Any deployed contract** can issue the `replace_class` syscall — no privileged role is required.
- A malicious contract author can deploy a contract, attract user deposits (e.g., a vault, multisig, or liquidity pool), and then call `replace_class(0xdeadbeef)` in a single transaction to freeze all deposited assets permanently.
- The gas cost of `REPLACE_CLASS_GAS_COST` is the only barrier, which is trivially affordable.
- The TODO comment confirms the team is aware the check is absent, meaning the window is open until the fix is shipped.

### Recommendation

Inside `execute_replace_class`, before writing the new class hash to `contract_state_changes`, verify that the hash exists in `contract_class_changes` (i.e., it was previously declared via a `declare` transaction). A `dict_read` on `contract_class_changes` with the requested `class_hash` as the key, followed by an assertion that the returned compiled class hash is non-zero, would close the gap — mirroring the validation already implicitly required by `execute_entry_point`.

### Proof of Concept

1. **Deploy** a Cairo 1 contract (`VaultContract`) that accepts ERC-20 deposits and exposes a `replace_class` wrapper entry point.
2. **Fund** the contract: users deposit 1 000 000 STRK into `VaultContract`.
3. **Attack**: the contract owner calls `VaultContract::replace_class(class_hash=0x41414141)`. The OS handler at `execute_replace_class` writes `class_hash=0x41414141` into `contract_state_changes` with no validation.
4. **Observe**: `0x41414141` was never declared; `contract_class_changes` has no entry for it.
5. **Consequence**: any subsequent call to `VaultContract` (including the owner's own withdrawal attempt) reaches `execute_entry_point` → `dict_read` returns `0` → `find_element(key=0)` aborts the OS → the block is unprovable → the 1 000 000 STRK are permanently frozen.

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
