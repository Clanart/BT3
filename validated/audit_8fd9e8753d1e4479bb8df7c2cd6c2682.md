### Title
Missing Declared Class Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS accepts any arbitrary felt value as a new class hash without verifying that the hash corresponds to a previously declared class. This is the direct analog of the external report's pattern: a shared resource (the compiled class) is referenced by a contract, but the OS performs no referential-integrity check when the contract replaces its class pointer. A contract can set its own class hash to an undeclared value, permanently rendering itself non-executable and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` reads `request.class_hash` and immediately writes it into the contract's `StateEntry` via `dict_update` on `contract_state_changes`:

```cairo
let class_hash = request.class_hash;
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}
tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The developer-acknowledged TODO at line 898 explicitly confirms the missing check. There is no cross-reference against `contract_class_changes` (the dictionary that tracks declared classes) to verify that `class_hash` was ever declared.

By contrast, `execute_declare_transaction` enforces referential integrity in the opposite direction: it uses `prev_value=0` to guarantee a class is declared exactly once before it can be used:

```cairo
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [2](#0-1) 

The `contract_class_changes` dictionary is the authoritative registry of declared classes. `execute_replace_class` writes to `contract_state_changes` (the per-contract state) but never reads from `contract_class_changes` to validate the new hash. [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `StateEntry.class_hash` is set to an undeclared value, the OS has no compiled class to dispatch execution to. Every subsequent call to that contract — including calls needed to withdraw or transfer funds — will fail at the class-lookup stage. The state transition is committed to the Patricia Merkle tree with the invalid class hash, and there is no recovery mechanism. Any ERC-20 tokens, ETH, or other assets held in the contract's storage become permanently inaccessible. [4](#0-3) 

---

### Likelihood Explanation

The `replace_class` syscall is callable by any contract from within its own execution context — no privileged role is required. The attacker-controlled entry paths include:

1. **Malicious contract**: A contract deployer publishes a contract that accepts user deposits and then calls `replace_class(0xDEAD)` (any undeclared felt), freezing all deposited funds.
2. **Reentrancy / calldata injection**: A contract that passes an externally-supplied class hash to `replace_class` can be exploited by any caller who provides an undeclared hash value.
3. **Cross-contract call**: `execute_call_contract` and `execute_deploy` both create execution contexts that can invoke `replace_class` on the callee's behalf. [5](#0-4) 

The syscall is reachable from any unprivileged transaction sender or contract deployer.

---

### Recommendation

Inside `execute_replace_class`, before writing the new `StateEntry`, add a lookup into `contract_class_changes` to assert that `class_hash` maps to a non-zero compiled class hash (i.e., it was previously declared). Concretely:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the pattern used in `execute_declare_transaction` and closes the referential-integrity gap between the class registry (`contract_class_changes`) and the per-contract state (`contract_state_changes`). [6](#0-5) 

---

### Proof of Concept

1. **Deploy** contract `C` that holds user funds (e.g., an ERC-20 vault). `C` is deployed with a valid, declared class hash `H_valid`.
2. **Trigger** `replace_class` inside `C` with `class_hash = 0x1337` (any felt that was never passed to a `declare` transaction).
3. **OS processing** in `execute_replace_class` reads `request.class_hash = 0x1337`, skips the missing declared-class check, and calls `dict_update` on `contract_state_changes`, setting `C`'s `StateEntry.class_hash = 0x1337`.
4. **State commitment** in `state_update` → `compute_contract_state_commitment` hashes the new `StateEntry` (with `class_hash=0x1337`) into the Patricia tree and produces a new valid global root. The invalid state is now committed on-chain. [7](#0-6) 

5. **Any subsequent call** to `C` (e.g., `withdraw`) causes the OS to look up compiled class `0x1337`, find nothing, and fail. The funds in `C`'s storage are permanently frozen with no recovery path. [3](#0-2)

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;
