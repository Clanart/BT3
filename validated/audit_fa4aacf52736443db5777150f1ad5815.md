### Title
`MerkleBlob::batch_insert` Skips Duplicate-Key/Hash Validation, Enabling Silent Tree Corruption and Forged DataLayer Merkle Root — (`File: crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::batch_insert` bypasses the duplicate-key and duplicate-hash guards that `MerkleBlob::insert` always enforces. When the tree already holds ≥ 2 leaves, **every** item in the batch is written directly to the blob via `insert_entry_to_blob` without any uniqueness check. When the tree holds 0–1 leaves, only the last two items are checked; all earlier items in the batch are unchecked. Because `BlockStatusCache::add_leaf` uses `HashMap::insert` (which silently overwrites), a duplicate key causes the cache to lose track of the first leaf while both leaves remain physically present in the blob and structurally connected to the tree. The resulting Merkle root is computed over both leaves and is therefore incorrect. Any DataLayer inclusion/exclusion proof anchored to that root is forged relative to the true intended state.

**Concrete corruption sequence** when a batch contains duplicate key `K` (hashes `H1`, `H2`):

1. Leaf `(K, H1)` is written to blob at `index1`; cache: `key_to_index[K] = index1`.
2. Leaf `(K, H2)` is written to blob at `index2`; cache: `key_to_index[K] = index2` (overwrites).
3. Both leaves are physically present in the blob and are connected to internal nodes during the subtree-building phase.
4. The Merkle root is computed over **both** leaves — it is wrong.
5. `get_leaf_by_key(K)` returns only the leaf at `index2`; the leaf at `index1` is unreachable via the cache but still contributes to the root hash.
6. `get_proof_of_inclusion(K)` produces a proof valid for the corrupted root, not the correct single-key root.
7. `check_integrity` detects the divergence via `IntegrityKeyToIndexCacheIndex` (the blob leaf at `index1` maps to a cache entry pointing to `index2`) or `IntegrityKeyToIndexCacheLength` (blob leaf count > cache size), but `check_integrity` is not called automatically after `batch_insert`. ---

### Impact Explanation

The DataLayer Merkle tree root hash is committed to the Chia blockchain. A corrupted root — one that encodes two leaves for the same key — is a **forged tree state**: it does not correspond to any valid key-value mapping. Proofs of inclusion generated against this root are valid for the corrupted root but invalid for the correct root. Proofs of exclusion for the orphaned leaf's key will incorrectly succeed (the cache says the key maps to `index2`, so a lookup for the orphaned leaf's distinct hash will not find it). This satisfies the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.**

---

### Likelihood Explanation

`batch_insert` is a public method exposed via the Python wheel. Any caller — including DataLayer RPC handlers or peer-sync code — that passes a batch containing a key already present in the tree, or a batch with an internal duplicate key, silently corrupts the tree with no error returned. The corruption is invisible to the caller unless `check_integrity` is explicitly invoked afterward. The condition is easy to trigger accidentally (e.g., retrying a partially-applied batch) and trivially triggerable by any party that can influence the key-value pairs submitted to a DataLayer store.

---

### Recommendation

Add the same uniqueness guards to `batch_insert` that `insert` enforces, before writing any item to the blob:

```rust
for ((key, value), hash) in keys_values_hashes {
    // Mirror the guards from `insert`:
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    let new_leaf_index = self.get_new_index();
    ...
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
```

Alternatively, pre-validate the entire batch for internal duplicates before touching the blob, so that the operation is atomic (all-or-nothing).

    fn check_just_integrity(&self) -> Result<(), Error> {
        let mut leaf_count: usize = 0;
        let mut internal_count: usize = 0;
        let mut child_to_parent: HashMap<TreeIndex, TreeIndex> = HashMap::new();

        for item in ParentFirstIterator::new(&self.blob, None) {
            let (index, block) = item?;
            if let Some(parent) = block.node.parent().0 {
                if child_to_parent.remove(&index) != Some(parent) {
                    return Err(Error::IntegrityParentChildMismatch(index));
                }
            }
            match block.node {
                Node::Internal(node) => {
                    internal_count += 1;
                    child_to_parent.insert(node.left, index);
                    child_to_parent.insert(node.right, index);
                }
                Node::Leaf(node) => {
                    leaf_count += 1;
                    let cached_index = self
                        .block_status_cache
                        .get_index_by_key(node.key)
                        .ok_or(Error::IntegrityKeyNotInCache(node.key))?;
                    if *cached_index != index {
                        return Err(Error::IntegrityKeyToIndexCacheIndex(
                            node.key,
                            index,
                            *cached_index,
                        ));
                    }
                    assert!(
                        !self.block_status_cache.is_index_free(index),
                        "{}",
                        format!("active index found in free index list: {index:?}")
                    );
                }
            }
        }

        let key_to_index_cache_length = self.block_status_cache.key_to_index.len();
        if leaf_count != key_to_index_cache_length {
            return Err(Error::IntegrityKeyToIndexCacheLength(
                leaf_count,
                key_to_index_cache_length,
            ));
        }
        let leaf_hash_to_index_cache_length = self.block_status_cache.leaf_hash_to_index.len();
        if leaf_count != leaf_hash_to_index_cache_length {
            return Err(Error::IntegrityLeafHashToIndexCacheLength(
                leaf_count,
                leaf_hash_to_index_cache_length,
            ));
        }
        let total_count = leaf_count + internal_count + self.block_status_cache.free_index_count();
        let extend_index = self.extend_index();
        if total_count != extend_index.0 as usize {
            return Err(Error::IntegrityTotalNodeCount(extend_index, total_count));
        }
        if !child_to_parent.is_empty() {
            return Err(Error::IntegrityUnmatchedChildParentRelationships(
                child_to_parent.len(),
            ));
        }

        Ok(())
    }
