### Title
`CommitmentStateDiff → ThinStateDiff` Conversion Always Drops `deprecated_declared_classes`, Producing Wrong `state_diff_commitment`, `state_diff_length`, `concatenated_counts`, and Block Hash for Any Block Containing a Deprecated (Cairo 0) Class Declaration — (`File: crates/blockifier/src/state/cached_state.rs`)

---

### Summary

The `From<CommitmentStateDiff> for ThinStateDiff` implementation unconditionally sets `deprecated_declared_classes: Vec::new()`. This conversion is the sole source of the `ThinStateDiff` fed into `calculate_block_commitments` inside `BlockExecutionArtifacts::new()`. For any block that includes a deprecated (Cairo 0) class declaration, the resulting `state_diff_commitment`, `state_diff_length`, `concatenated_counts`, and final block hash are all computed over a structurally incomplete state diff, producing permanently wrong authoritative values that are stored, broadcast over P2P, and used by the proof path.

---

### Finding Description

**Root cause — the silent drop:**

`CommitmentStateDiff` has no field for deprecated declared classes: [1](#0-0) 

When the blockifier converts `CommitmentStateDiff` to `ThinStateDiff`, the `deprecated_declared_classes` field is hard-coded to an empty `Vec`: [2](#0-1) 

The `TODO(AlonH)` comment acknowledges this is a known structural gap, but no guard prevents the incomplete diff from being used for commitment calculation.

**Where the wrong diff is consumed:**

`BlockExecutionArtifacts::new()` calls `ThinStateDiff::from(commitment_state_diff.clone())` and passes the result directly to `calculate_block_commitments`: [3](#0-2) 

`calculate_block_commitments` uses `state_diff.len()` for both `concatenated_counts` and the `state_diff_length` measurement, and passes the diff to `calculate_state_diff_hash`: [4](#0-3) 

**`ThinStateDiff::len()` counts `deprecated_declared_classes`:** [5](#0-4) 

Because `deprecated_declared_classes` is always `Vec::new()` in the proposer path, `len()` returns a value that is too small by exactly the number of deprecated declared classes in the block.

**`calculate_state_diff_hash` hashes `deprecated_declared_classes`:** [6](#0-5) 

With an empty `deprecated_declared_classes`, `chain_deprecated_declared_classes` chains `0` (the count) instead of the actual class hashes, producing a wrong `StateDiffCommitment`.

**`calculate_block_hash` chains both corrupted values:** [7](#0-6) 

Both `block_commitments.concatenated_counts` (which encodes the wrong `state_diff_length`) and `block_commitments.state_diff_commitment.0.0` (the wrong hash) are chained into the Poseidon block hash, making the final block hash wrong.

**The wrong `state_diff_length` is stored in the block header and used by P2P sync:**

`state_diff_length` from `BlockHeaderCommitments` is stored in `StorageBlockHeader`: [8](#0-7) 

The P2P sync client reads this stored `state_diff_length` as the authoritative target length and stops consuming state diff chunks once it reaches that count: [9](#0-8) 

A syncing node that receives the correct (full) state diff from a peer will see `current_state_diff_len > target_state_diff_len` and reject the valid diff as a `WrongStateDiffLength` bad-peer error, or accept a truncated diff that is missing the deprecated declared classes.

**The same wrong `ThinStateDiff` is committed to storage:**

`thin_state_diff()` uses the same conversion: [10](#0-9) 

This is passed to `commit_proposal_and_block` in `decision_reached`: [11](#0-10) 

The stored `ThinStateDiff` is missing the deprecated declared classes, so any subsequent read of the state diff (RPC `starknet_getStateUpdate`, proof inputs, etc.) returns an incomplete diff.

---

### Impact Explanation

For any block containing a `DeclareV0` or `DeclareV1` transaction:

1. **Wrong `state_diff_commitment`** is stored in the block header and chained into the block hash. The block hash is permanently wrong.
2. **Wrong `state_diff_length`** is stored in the block header. P2P sync nodes use this as the authoritative termination condition for state diff streaming, causing them to either reject valid full diffs or accept truncated diffs.
3. **Wrong `concatenated_counts`** (which packs `state_diff_length`) is chained into the block hash, compounding the block hash error.
4. **Wrong stored `ThinStateDiff`** causes `starknet_getStateUpdate` and proof inputs to return an incomplete state diff missing the deprecated declared classes.
5. The SNOS/proof path receives an incomplete `CommitmentStateDiff` and produces proof inputs that do not match the actual on-chain state, breaking proof verification.

This matches: **Critical — Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input** and **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value**.

---

### Likelihood Explanation

Deprecated (Cairo 0) class declarations (`DeclareV0`, `DeclareV1`) remain valid Starknet transaction types. Any user submitting such a transaction triggers the bug. The trigger requires no privilege — any unprivileged account can submit a deprecated declare. The bug is deterministic: every block containing at least one deprecated declare will have a wrong block hash and wrong stored state diff.

---

### Recommendation

`CommitmentStateDiff` must be extended to carry deprecated declared class hashes, or the conversion path must be changed to source `deprecated_declared_classes` from a separate tracking mechanism (e.g., `StateMaps.declared_contracts` where the value is `false`).

Concretely:

1. Add `deprecated_declared_classes: Vec<ClassHash>` to `CommitmentStateDiff`.
2. Populate it in `From<StateMaps> for CommitmentStateDiff` from `diff.declared_contracts.iter().filter(|(_, &is_new)| !is_new).map(|(h, _)| *h)`.
3. Remove the `deprecated_declared_classes: Vec::new()` hard-coding in `From<CommitmentStateDiff> for ThinStateDiff`.
4. Remove the `TODO(AlonH)` comment once the structural gap is closed.

---

### Proof of Concept

```
1. Deploy a Cairo 0 (deprecated) contract class via a DeclareV0 or DeclareV1 transaction.
2. The batcher executes the block containing this transaction.
3. In BlockExecutionArtifacts::new():
     - commitment_state_diff has no deprecated_declared_classes field
     - ThinStateDiff::from(commitment_state_diff) sets deprecated_declared_classes = []
     - calculate_block_commitments receives ThinStateDiff with deprecated_declared_classes = []
     - state_diff.len() returns N (missing the K deprecated declared classes)
     - calculate_state_diff_hash chains count=0 for deprecated classes instead of count=K + hashes
     - concatenated_counts encodes state_diff_length=N instead of N+K
     - calculate_block_hash produces hash H_wrong ≠ H_correct

4. The wrong block hash H_wrong is stored in StorageBlockHeader.
5. The wrong state_diff_length N is stored in StorageBlockHeader.state_diff_length.
6. A syncing peer fetches the block header (state_diff_length=N) and requests state diff chunks.
7. The peer sends K+N items (the correct full diff).
8. The sync client reaches current_state_diff_len == N < target=N, but the peer sends K more items.
9. The sync client either:
   a. Accepts the diff at length N (missing K deprecated declared classes), or
   b. Rejects the peer as BadPeer::WrongStateDiffLength when the peer sends Fin after K+N items.
10. starknet_getStateUpdate for this block returns a ThinStateDiff missing the K deprecated classes.
11. SNOS proof inputs built from this block's CommitmentStateDiff are missing the deprecated classes,
    causing proof verification to fail or produce a wrong state root.
```

    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =

    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
