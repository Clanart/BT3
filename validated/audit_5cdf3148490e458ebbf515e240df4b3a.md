### Title
`calculate_next_base_gas_price` and `calculate_next_l2_gas_price_for_fin` use hardcoded `VersionedConstants::latest_constants()` instead of block-version-specific constants, causing incorrect `next_l2_gas_price` - (File: `crates/apollo_consensus_orchestrator/src/fee_market/mod.rs`)

---

### Summary

The EIP-1559 fee market functions always call `VersionedConstants::latest_constants()` to obtain `gas_target`, `max_block_size`, `gas_price_max_change_denominator`, and `min_gas_price`, regardless of the block's actual Starknet version. These parameters differ materially between versions (e.g., `gas_target` is `3_200_000_000` for V0_14_0 vs `1_040_000_000` for V0_14_3/V0_14_4 — a 3× difference). When the deployed sequencer binary is at a newer version than the network version being produced, the wrong constants are used, producing an incorrect `next_l2_gas_price` that is persisted in the block header and used as the starting L2 gas price for the next block's execution.

### Finding Description

In `crates/apollo_consensus_orchestrator/src/fee_market/mod.rs`, two functions unconditionally call `VersionedConstants::latest_constants()`:

**`calculate_next_l2_gas_price_for_fin` (line 70):**
```rust
let gas_target = VersionedConstants::latest_constants().gas_target;
``` [1](#0-0) 

**`calculate_next_base_gas_price` (line 92):**
```rust
let versioned_constants = VersionedConstants::latest_constants();
``` [2](#0-1) 

Neither function accepts a `starknet_version` parameter. The `gas_target` and `max_block_size` values used in the EIP-1559 price-change formula are therefore always taken from the latest compiled-in version, not from the version of the block being finalized.
