### Title
Missing Declared Class Hash Validation in `execute_replace_class` Enables Permanent Contract Fund Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the caller-supplied class hash corresponds to a previously declared contract class before committing the state update. This is an exact structural analog to the PaladinRewardReserve issue — an unrestricted state-mutating operation with no validation on the supplied value — and allows any contract to permanently brick itself (or be bricked by an attacker who can trigger its upgrade path), freezing all funds held within it.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` (lines 878–916) processes the `replace_class` syscall. After deducting gas and reading the current state entry, it writes the caller-supplied `class_hash` directly into the contract's `StateEntry` with **no check that a contract class with that hash has ever been declared**. The code itself acknowledges this gap with an explicit TODO:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The syscall is dispatched unconditionally from `execute_syscalls` for any executing contract:

```cairo
if (selector == REPLACE_CLASS_SELECTOR) {
    execute_replace_class(contract_address=execution_context.execution_info.contract_address);
``` [2](#0-1) 

The OS commits the block with the invalid class hash in the state diff. There is no post-hoc correction: once the state root is updated via `state_update` → `compute_contract_state_commitment`, the invalid entry is permanent. [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

If a contract's class hash is replaced with an undeclared hash, every subsequent call to that contract fails at class resolution. The OS cannot find the corresponding contract class, so no entry point can be dispatched. Any ERC-20 tokens, ETH, or other assets stored in the contract's storage slots become permanently inaccessible. The state transition is accepted by the OS and committed to the global state root, making it irreversible at the protocol level.

---

### Likelihood Explanation

The `replace_class` syscall is reachable by **any executing contract** — no privileged role is required at the OS level. The realistic attack surface is:

- **Upgradeable contracts** (a standard DeFi pattern) that expose a public or role-gated `upgrade(new_class_hash)` function. If the access control is bypassed (compromised key, reentrancy, insufficient guard), an attacker supplies an undeclared hash.
- **Contracts with no access control on upgrade** — the attacker calls directly.

The OS provides zero safety net. The missing check is explicitly acknowledged in the source as a known gap (`TODO`), confirming the code path is live and unguarded.

---

### Recommendation

Before writing the new `StateEntry`, perform a lookup in `contract_class_changes` to assert the class hash has been declared:

```cairo
// Verify the class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the existing pattern used in `execute_declare_transaction`, which enforces `assert_not_zero(compiled_class_hash)` before writing to `contract_class_changes`. [4](#0-3) 

---

### Proof of Concept

1. Deploy a contract `Vault` holding user funds, with an `upgrade(new_class_hash: felt)` function that calls the `replace_class` syscall.
2. An attacker (or compromised owner) calls `Vault.upgrade(0xdeadbeef)` where `0xdeadbeef` is not a declared class hash.
3. The OS dispatches `execute_replace_class` via `execute_syscalls.cairo` line 195–203.
4. `execute_replace_class` deducts gas, reads the current `StateEntry`, and writes `new StateEntry(class_hash=0xdeadbeef, ...)` to `contract_state_changes` — **no validation performed** (line 898 TODO is unimplemented).
5. `state_update` squashes and commits the state diff; the global state root now encodes `Vault.class_hash = 0xdeadbeef`.
6. All subsequent calls to `Vault` fail: the OS cannot resolve class `0xdeadbeef`, no entry point is dispatched.
7. All user funds in `Vault` are permanently frozen. [5](#0-4)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

        key=contract_address,

    );

    return ();
}
