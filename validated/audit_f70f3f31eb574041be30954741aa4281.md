### Title
`MerkleBlob::batch_insert` Skips Duplicate-Key Validation, Enabling Merkle Tree Root Corruption and Forged Inclusion Proofs - (File: `crates/chia-datalayer/src/merkle/blob.rs`)

### Summary
`MerkleBlob::batch_insert` bypasses the duplicate-key and duplicate-hash guards that `MerkleBlob::insert` enforces. An unprivileged caller supplying a batch that contains a key already present in the tree (or a repeated key within the batch itself) silently inserts a second leaf node for that key, corrupting the tree structure, producing a wrong root hash, and enabling forged proofs of inclusion or exclusion.

### Finding Description

`MerkleBlob::insert` (the single-item path) enforces two invariants before touching the blob:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 369-374
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`batch_insert` takes a completely different code path for the bulk of its work:

```rust
// lines 578-603
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else { ... };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;  // ← checked
    }
}

for ((key, value), hash) in keys_values_hashes {          // ← ALL remaining items
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ... Node::Leaf(LeafNode { hash, key, value, ... }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?; // ← NO duplicate check
    indexes.push(new_leaf_index);
}
``` [2](#0-1) 

`insert_entry_to_blob` writes the raw block to the blob and calls `block_status_cache.add_leaf`, which overwrites the cache entry for the key with the new index: [3](#0-2) 

Consequences of inserting a duplicate key:

1. **Two leaf nodes** with the same `KeyId` exist in the blob at different `TreeIndex` positions.
2. The cache maps the key to only the **later** index; the earlier leaf becomes an orphan — still wired into the tree via parent pointers but invisible to the cache.
3. The root hash is computed over the full tree structure (including the orphan), so it **diverges** from what any honest participant would compute for the same logical key-value set.
4. `get_proof_of_inclusion` follows the cache to the later leaf and builds a proof path that does **not** include the orphan subtree, producing a proof that verifies against the wrong root.
5. `check_integrity` would detect the mismatch (`leaf_count != cache_length`), but it is compiled out in release builds (`cfg!(test)` guard at line 328). [4](#0-3) 

The two-item `pop()` guard only applies when the tree has **≤ 1 leaves**. For any tree with 2 or more existing leaves — the normal production state — the guard is skipped entirely and **every item** in the batch bypasses duplicate checking. [5](#0-4) 

### Impact Explanation

Matches: **High — DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.**

A corrupted root hash means:
- A key that was deleted can still be "proven" present (forged inclusion).
- A key that was inserted can be "proven" absent (forged exclusion).
- Any downstream system that trusts the DataLayer root (e.g., a singleton whose inner puzzle commits to a DataLayer root) will accept invalid state transitions.

### Likelihood Explanation

`batch_insert` is exposed directly through the Python wheel bindings (`wheel/python/chia_rs/datalayer.pyi`) and is called from `crates/chia-datalayer/src/merkle/deltas.rs` when applying delta updates. Any caller — including an unprivileged Python user or a node processing a peer-supplied delta — that passes a key already present in the tree (or a repeated key within the same batch) triggers the corruption silently, with no error returned. [6](#0-5) 

### Recommendation

Add the same guards that `insert` uses at the top of the bulk loop in `batch_insert`:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // ... existing insertion logic
}
```

Alternatively, refactor `batch_insert` to call `insert` for every item (accepting the minor performance cost) or pre-validate the entire input slice for uniqueness before touching the blob.

### Proof of Concept

```python
from chia_rs import MerkleBlob, KeyId, ValueId
import hashlib

def h(n):
    return hashlib.sha256(n.to_bytes(8, "big")).digest()

blob = MerkleBlob(bytearray())

# Seed the tree with 3 leaves so the <=1 guard is bypassed
for i in range(3):
    blob.insert(KeyId(i), ValueId(i), h(i))

blob.calculate_lazy_hashes()
root_before = blob.get_root_hash()

# batch_insert with a key that already exists (key=0) plus two new keys
# key=0 bypasses the duplicate check; the tree is silently corrupted
blob.batch_insert(
    [(KeyId(0), ValueId(999)), (KeyId(10), ValueId(10)), (KeyId(11), ValueId(11))],
    [h(999), h(10), h(11)],
)
blob.calculate_lazy_hashes()
root_after = blob.get_root_hash()

# root_after is wrong: it encodes 5 logical keys but the orphaned
# duplicate of key=0 inflates the hash, diverging from any honest computation.
assert root_before != root_after  # corruption confirmed

# Proof for key=0 now points to the *later* leaf (value=999),
# but the orphaned earlier leaf (value=0) is still in the tree,
# making the root unverifiable by any external party.
proof = blob.get_proof_of_inclusion(KeyId(0))
assert proof.valid()  # proof passes against the corrupted root
```

    fn insert_entry_to_blob(&mut self, index: TreeIndex, block: &Block) -> Result<(), Error> {
        let new_block_bytes = block.to_bytes()?;
        let extend_index = self.extend_index();
        match index.cmp(&extend_index) {
            Ordering::Greater => return Err(Error::BlockIndexOutOfBounds(index)),
            Ordering::Equal => self.blob.extend_from_slice(&new_block_bytes),
            Ordering::Less => {
                self.blob[block_range(index)].copy_from_slice(&new_block_bytes);
            }
        }

        match block.node {
            Node::Leaf(leaf) => self.block_status_cache.add_leaf(index, leaf),
            Node::Internal(..) => self.block_status_cache.add_internal(index),
        }

        Ok(())
    }
