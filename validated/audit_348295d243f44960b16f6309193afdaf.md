### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Fund Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the caller-supplied class hash corresponds to a previously declared contract class. Any unprivileged contract can call `replace_class` with an arbitrary undeclared hash, causing the OS to commit an invalid state transition that permanently bricks the contract and freezes all funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The code itself acknowledges the missing check with a `TODO` comment. The `class_hash` from the syscall request is written directly into `contract_state_changes` without any lookup into `contract_class_changes` to confirm the hash is declared. [1](#0-0) 

Critically, the function's implicit argument list does **not** include `contract_class_changes`:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,   // ← only state changes, not class changes
    revert_log: RevertLogEntry*,
}(contract_address: felt)
``` [2](#0-1) 

This means the function is architecturally incapable of performing the validation even if the TODO were addressed without a signature change. The identical omission exists in the deprecated syscall path: [3](#0-2) 

The OS's `execute_entry_point` function resolves a contract's class hash by looking it up in `contract_class_changes` and then finding the corresponding `CompiledClassFact` in the global `compiled_class_facts` bundle:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
``` [4](#0-3) 

If `replace_class` sets a contract's class hash to a value that has no entry in `contract_class_changes`, the `find_element` call will fail (or return garbage), making the contract permanently uncallable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once the OS commits a block where a contract's class hash is set to an undeclared value:

1. The state transition is proven on L1 and is irreversible.
2. Every subsequent call to that contract fails at the `find_element` lookup in `execute_entry_point`.
3. Any ERC-20 tokens, ETH, or STRK held by the contract are permanently inaccessible.

This is a direct, on-chain, provably-committed loss of user funds with no recovery path.

---

### Likelihood Explanation

The `replace_class` syscall is a standard, permissionless syscall available to every Cairo 1 contract. The attacker-controlled entry path is:

1. An unprivileged user deploys a contract `C` that holds funds (e.g., a multisig, vault, or DeFi contract).
2. `C` contains an entry point that calls `replace_class(arbitrary_undeclared_hash)`.
3. The user (or any caller of that entry point) triggers the call.
4. The OS processes the syscall, writes the undeclared hash into `contract_state_changes`, and generates a valid proof.
5. The L1 verifier accepts the proof; `C` is permanently bricked.

No privileged role, leaked key, or malicious sequencer is required. The sequencer is obligated to include valid user transactions, and the OS has no constraint that rejects this one.

---

### Recommendation

1. Add `contract_class_changes: DictAccest*` as an implicit argument to `execute_replace_class` in both `syscall_impls.cairo` and `deprecated_execute_syscalls.cairo`.
2. Before writing the new class hash, perform a `dict_read` on `contract_class_changes` to confirm the hash is present (i.e., has been declared in this or a prior block).
3. If the hash is absent, write a failure response via `write_failure_response` instead of updating state.

---

### Proof of Concept

```
1. Deploy contract C with storage holding 1000 STRK.
2. C has entry point `brick_self()`:
       replace_class(0xdeadbeef)   // 0xdeadbeef is not declared
3. Call C.brick_self() via an invoke transaction.
4. OS executes execute_replace_class:
       - reads class_hash = 0xdeadbeef from syscall request
       - skips declared-class check (TODO, not implemented)
       - writes StateEntry(class_hash=0xdeadbeef, ...) into contract_state_changes
5. Block is proven and settled on L1.
6. Any subsequent call to C triggers execute_entry_point:
       - dict_read(contract_class_changes, key=0xdeadbeef) → not found / zero
       - find_element fails → contract is uncallable
7. 1000 STRK in C are permanently frozen.
```

        // Not enough gas to execute the syscall.
        return ();
