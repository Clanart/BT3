### Title
`execute_meta_tx_v0` Allows Arbitrary `__execute__` Invocation on Any Account Without `__validate__` — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_meta_tx_v0` syscall handler in `syscall_impls.cairo` accepts a fully attacker-controlled `contract_address` field from the `MetaTxV0Request` struct and directly invokes `__execute__` on that target account without ever calling `__validate__`. Because standard Cairo 1 account contracts perform signature verification only in `__validate__`, any unprivileged contract can impersonate any victim account and execute arbitrary calls on its behalf, leading to direct loss of funds.

---

### Finding Description

The `execute_meta_tx_v0` function reads the target address from the user-supplied syscall request:

```cairo
local contract_address = request.contract_address;   // fully attacker-controlled
local selector = request.selector;
``` [1](#0-0) 

It then constructs a brand-new `TxInfo` that sets `account_contract_address` to the attacker-chosen address, hardcodes `nonce = 0`, and accepts an attacker-supplied `signature`:

```cairo
tempvar new_tx_info = new TxInfo(
    version=0,
    account_contract_address=contract_address,   // victim's address
    ...
    signature_start=request.signature_start,     // attacker-controlled
    signature_end=request.signature_end,
    ...
    nonce=0,                                     // hardcoded, no replay protection
    ...
);
``` [2](#0-1) 

The execution context is then built with `caller_address = ORIGIN_ADDRESS` (0) and `selector = EXECUTE_ENTRY_POINT_SELECTOR`, making it indistinguishable from a legitimate OS-initiated `__execute__` call:

```cairo
tempvar execution_context: ExecutionContext* = new ExecutionContext(
    ...
    execution_info=new ExecutionInfo(
        ...
        tx_info=new_tx_info,
        caller_address=ORIGIN_ADDRESS,
        contract_address=contract_address,
        selector=selector,           // = EXECUTE_ENTRY_POINT_SELECTOR
    ),
    ...
);
``` [3](#0-2) 

Finally, `contract_call_helper` is invoked, which calls `select_execute_entry_point_func` — directly executing `victim.__execute__()` — **without ever calling `victim.__validate__()`**:

```cairo
contract_call_helper(
    remaining_gas=remaining_gas,
    block_context=block_context,
    execution_context=execution_context,
);
``` [4](#0-3) 

The only guard present is a selector check that restricts the call to `__execute__`, which is precisely the most dangerous entry point to reach without validation:

```cairo
if (selector != EXECUTE_ENTRY_POINT_SELECTOR) {
    write_failure_response(...);
    return ();
}
``` [5](#0-4) 

**Analog to the external report:** In `OmoRouter.sol`, the `from` parameter of `transferFrom()` was user-controlled, allowing theft of approved allowances. Here, `contract_address` in `MetaTxV0Request` is user-controlled, allowing execution of arbitrary calls on behalf of any account — the same class of authorization bypass, applied to the OS-level account execution model.

---

### Impact Explanation

**Critical — Direct loss of funds.**

Standard Cairo 1 account contracts (e.g., OpenZeppelin) perform signature verification exclusively in `__validate__`. Their `__execute__` function trusts that `__validate__` was already called by the OS and does not re-verify the signature. By bypassing `__validate__`, an attacker can instruct any victim account to execute arbitrary calls — including ERC-20 `transfer` calls — draining the victim's token balances. Because `nonce = 0` is hardcoded and no nonce is ever incremented for meta-tx-v0, the same attack can be replayed indefinitely against the same victim.

---

### Likelihood Explanation

**High.** The attack requires no privileged role, no leaked key, and no trusted operator. Any unprivileged user can:
1. Deploy a malicious contract (permissionless on StarkNet).
2. Submit a standard invoke transaction calling the malicious contract.
3. Have the malicious contract issue a `meta_tx_v0` syscall targeting any victim.

There is no on-chain access control restricting which contracts may call `execute_meta_tx_v0`.

---

### Recommendation

Apply one or more of the following mitigations:

1. **Restrict `contract_address` to the caller itself.** A contract should only be permitted to issue a meta-tx on its own behalf:
   ```cairo
   // Enforce: contract_address == caller_execution_info.contract_address
   assert contract_address = caller_execution_info.contract_address;
   ```

2. **Call `__validate__` before `__execute__`.** Invoke the target contract's `__validate__` entry point with the meta-tx hash and the provided signature before proceeding to `__execute__`, mirroring the normal transaction flow.

3. **Add nonce tracking.** Include a nonce in the meta-tx hash and enforce monotonic nonce increments per `(caller, contract_address)` pair to prevent replay attacks.

---

### Proof of Concept

```
1. Attacker deploys MaliciousContract containing:
       fn attack(victim: felt252, calldata: Array<felt252>) {
           // Issues meta_tx_v0 syscall:
           //   contract_address = victim
           //   selector         = __execute__
           //   calldata         = [transfer(attacker_address, victim_balance)]
           //   signature        = []   (empty — __execute__ won't check it)
       }

2. Attacker submits: invoke MaliciousContract.attack(victim=0xVICTIM, ...)

3. OS processes execute_meta_tx_v0:
   - Builds new_tx_info { account_contract_address=0xVICTIM, version=0, nonce=0, signature=[] }
   - Builds ExecutionContext { caller_address=0, contract_address=0xVICTIM,
                               selector=__execute__ }
   - Calls contract_call_helper → select_execute_entry_point_func
     → 0xVICTIM.__execute__([transfer(attacker, balance)])
     (NO __validate__ is called)

4. 0xVICTIM.__execute__ sees caller_address=0 (legitimate-looking OS call),
   executes the transfer without signature verification.

5. Victim's tokens are transferred to the attacker.
   Step 2 can be repeated indefinitely (nonce=0 is never incremented).
```
