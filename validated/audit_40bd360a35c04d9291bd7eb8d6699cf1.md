### Title
Missing Caller Authentication on `L1GasPriceProvider::add_price_info()` Enables Unauthenticated Gas Price Injection with Economic Impact — (File: `crates/apollo_l1_gas_price/src/communication.rs`)

### Summary

The `L1GasPriceProvider` exposes a write endpoint (`L1GasPriceRequest::AddGasPrice`) through the `RemoteComponentServer` with no caller authentication. Any process that can reach the remote server socket can inject arbitrary `GasPriceData` (block number, timestamp, `base_fee_per_gas`, `blob_fee`) into the provider's ring-buffer. The corrupted samples are then averaged and used as the canonical L1 gas prices in every block proposal, causing all transactions in those blocks to be priced against attacker-controlled values.

### Finding Description

`L1GasPriceProvider::handle_request` in `communication.rs` dispatches `L1GasPriceRequest::AddGasPrice(data)` unconditionally to `self.add_price_info(data)`:

```rust
// crates/apollo_l1_gas_price/src/communication.rs  lines 29-31
L1GasPriceRequest::AddGasPrice(data) => {
    L1GasPriceResponse::AddGasPrice(self.add_price_info(data))
}
```

`add_price_info` performs only one structural check — that the incoming block number equals `last_stored + 1` — and then unconditionally pushes the caller-supplied `PriceInfo { base_fee_per_gas, blob_fee }` into the ring-buffer:

```rust
// crates/apollo_l1_gas_price/src/l1_gas_price_provider.rs  lines 84-102
pub fn add_price_info(&mut self, new_data: GasPriceData) -> L1GasPriceProviderResult<()> {
    let Some(samples) = &mut self.price_samples_by_block else {
        return Err(L1GasPriceProviderError::NotInitializedError);
    };
    if let Some(data) = samples.back() {
        if new_data.block_number != data.block_number + 1 {
            return Err(L1GasPriceProviderError::UnexpectedBlockNumberError { ... });
        }
    }
    samples.push(new_data);   // ← no validation of price values
    Ok(())
}
```

There is no token, TLS client certificate, IP allowlist, or any other identity check at any layer between the network socket and this push. The `RemoteComponentServer` is a generic framed-message server; the `ComponentRequestHandler` trait carries no caller context.

The attacker's only prerequisite is knowing the current highest stored L1 block number (trivially obtained by observing the scraper's public metrics or querying the base layer directly) so that the consecutive-block-number guard passes.

### Impact Explanation

`get_price_info` computes a rolling mean over the ring-buffer and returns it as the authoritative `PriceInfo` consumed by the consensus orchestrator:

```rust
// crates/apollo_l1_gas_price/src/l1_gas_price_provider.rs  lines 163-173
let price_info_summed: PriceInfo = samples
    .iter().skip(first_index).take(actual_number_of_blocks)
    .map(|data| &data.price_info)
    .sum();
let price_info_out = price_info_summed.checked_div(actual_number_of_blocks)...;
```

The orchestrator converts this to FRI and embeds it in every block proposal. Because the gas prices are clamped only at the proposal level (min/max bounds in `ContextDynamicConfig`), an attacker who injects prices within those bounds — but systematically biased — will cause every committed block to carry wrong `l1_gas_price_wei` / `l1_data_gas_price_wei` values. All transaction fee calculations, resource-bound checks, and refund computations for that block are then performed against the attacker-controlled price, producing incorrect economic outcomes for every user whose transaction is included.

If the attacker can reach all validator nodes' `RemoteComponentServer` endpoints (all nodes share the same unauthenticated interface), they can synchronise the injected prices across the network, ensuring validators accept the corrupted proposal without disagreement.

**Matched impact**: *Critical — Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact.*

### Likelihood Explanation

- The `RemoteComponentServer` is a standard TCP/HTTP framed server with no built-in authentication layer.
- In any distributed deployment where the L1 gas price provider runs as a remote component, the `AddGasPrice` endpoint is reachable by any process on the same network segment.
- The only bypass required is knowing the current L1 block number, which is public information.
- The attack is repeatable: the scraper pushes one block per polling interval; the attacker simply races each legitimate push with a crafted one carrying the same block number but manipulated prices.

### Recommendation

1. **Authenticate callers at the `RemoteComponentServer` level**: add mutual TLS or a shared-secret token verified before dispatching any `ComponentRequestHandler::handle_request` call.
2. **Restrict write endpoints**: split the `L1GasPriceProviderClient` trait into a read-only interface (for the consensus orchestrator) and a write interface (for the scraper only), and bind the write interface exclusively to a localhost or Unix-domain socket.
3. **Validate price plausibility**: even after authentication, reject `GasPriceData` whose `base_fee_per_gas` or `blob_fee` deviates by more than a configurable factor from the previous sample, providing a defence-in-depth layer.
