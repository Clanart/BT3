### Title
Unvalidated `global_roots` from RPC Proof Response Used as SNOS Initial State Roots Without Cross-Check Against Stored Global Root — (File: `crates/starknet_transaction_prover/src/running/storage_proofs.rs`)

---

### Summary

`RpcStorageProofsProvider::get_storage_proofs()` fetches a Patricia Merkle storage proof from an external RPC node and blindly promotes the `global_roots.contracts_tree_root` / `classes_tree_root` fields from the response into the SNOS `CommitmentInfo.previous_root` fields. No check is ever made against the sequencer's own authoritative global root for that block number, which is stored locally in `apollo_storage`. A compromised or stale RPC node can therefore supply a self-consistent proof bundle anchored to a *different* block's state root, causing the virtual OS to produce a proof whose `initial_root` and `final_root` attest to a state transition that never occurred.

---

### Finding Description

**Data flow**

1. `Runner::create_virtual_os_hints()` extracts `block_number` from `execution_data.base_block_info` and calls `storage_proofs_provider.get_storage_proofs(block_number, …)`. [1](#0-0) 

2. `RpcStorageProofsProvider::get_storage_proofs()` calls `fetch_proofs(block_number, &query)`, which issues one or more `starknet_getStorageProof` RPC calls to an external node. [2](#0-1) 

3. The returned `RpcStorageProof` carries a `global_roots` struct. In `create_commitment_infos_with_state_changes()`, these values are taken verbatim as the initial Patricia roots:

```rust
let contracts_trie_root_hash = HashOutput(rpc_proof.global_roots.contracts_tree_root);
let classes_trie_root_hash   = HashOutput(rpc_proof.global_roots.classes_tree_root);
``` [3](#0-2) 

4. These become `previous_root` in `CommitmentInfo`, which is placed directly into `VirtualOsBlockInput.contract_state_commitment_info` / `contract_class_commitment_info`. [4](#0-3) 

5. The SNOS reads these as `initial_root` and derives `final_root` from them; both appear verbatim in the proof output / proof facts. [5](#0-4) 

**What is never checked**

The sequencer stores the authoritative global root for every committed block in `apollo_storage`: [6](#0-5) 

`get_storage_proofs()` receives `block_number` but never reads `get_global_root(block_number)` from storage and never compares it against `rpc_proof.global_roots`. The `global_roots` field is consumed without any cross-validation.

**Secondary gap in `merge_storage_proofs`**

When the query is chunked (total keys > 100), each chunk is fetched separately and the merged result silently takes `global_roots` from only the first chunk's response:

```rust
let global_roots = proofs[0].global_roots.clone();
``` [7](#0-6) 

Divergent `global_roots` across chunks (possible under a race condition or a partially-compromised node) are never detected.

---

### Impact Explanation

The `global_roots` from the RPC response become the `previous_root` of the contracts trie and classes trie `CommitmentInfo`. The SNOS uses these as `initial_root` and computes `final_root` by applying the state diff on top. If the roots belong to a different block, the proof output's `initial_root` and `final_root` are both wrong. Any on-chain verifier or proof-fact consumer that trusts these values will accept a proof attesting to a state transition that never happened — wrong state roots in the proof output.

This matches: **Critical — Wrong state … from blockifier/syscall/execution logic for accepted input** and **High — RPC execution … returns an authoritative-looking wrong value**.

---

### Likelihood Explanation

Requires the RPC node configured for the transaction prover to return a `global_roots` that does not match the sequencer's stored global root for the requested block. This can occur via:

- A compromised or misconfigured RPC node serving proofs anchored to a stale or different block.
- A race condition during chunked fetching where the node's view of the chain tip advances between chunk requests (secondary gap above).

The sequencer already possesses the ground-truth value (`get_global_root(block_number)`) and could reject the mismatch, but does not.

---

### Recommendation

1. **Pass the expected global root into `get_storage_proofs`** — the caller (`create_virtual_os_hints`) already has `block_number`; the batcher/committer can supply the corresponding stored `GlobalRoot`.

2. **Validate immediately after fetching**:

```rust
let expected = stored_global_root; // fetched from apollo_storage
let actual = StateRoots {
    contracts_trie_root_hash: HashOutput(rpc_proof.global_roots.contracts_tree_root),
    classes_trie_root_hash:   HashOutput(rpc_proof.global_roots.classes_tree_root),
}.global_root();
if actual != expected {
    return Err(ProofProviderError::GlobalRootMismatch { block_number, expected, actual });
}
```

3. **In `merge_storage_proofs`**, assert that all chunks return identical `global_roots` before selecting the first one.

---

### Proof of Concept

1. Configure the transaction prover's RPC URL to point to a node that, for block `N`, returns `global_roots` matching block `N-K` (an older block with different state).
2. Submit any valid invoke transaction for block `N` to `prove_transaction(BlockId::Number(N), tx)`.
3. `get_storage_proofs(N, …)` fetches the proof; `rpc_proof.global_roots` carries block `N-K`'s roots.
4. `create_commitment_infos_with_state_changes` sets `contracts_trie_root_hash = root_of(N-K)` and `classes_trie_root_hash = root_of(N-K)`.
5. The SNOS runs with these as `initial_root`; `final_root` is computed by applying the (correct) state diff on top of the wrong base.
6. The resulting proof's `proof_facts` contains `initial_root = global_root(N-K)` and `final_root ≠ global_root(N)`, attesting to a state transition that never occurred on-chain.
7. No error is raised anywhere in `get_storage_proofs`, `create_virtual_os_hints`, or the SNOS runner because the Patricia Merkle proofs are internally consistent with the wrong roots. [8](#0-7)

    async fn get_storage_proofs(
        &self,
        block_number: BlockNumber,
        execution_data: &VirtualBlockExecutionData,
        config: &StorageProofConfig,
    ) -> Result<StorageProofs, ProofProviderError> {
        let query = Self::prepare_query(execution_data);

        let rpc_proof = self.fetch_proofs(block_number, &query).await?;

        // Validate that contract_leaves_data matches contract_addresses length.
        let leaves_len = rpc_proof.contracts_proof.contract_leaves_data.len();
        let addresses_len = query.contract_addresses.len();
        if leaves_len != addresses_len {
            return Err(ProofProviderError::InvalidProofResponse(format!(
                "Contract leaves length mismatch: expected {addresses_len} leaves for requested \
                 contracts, got {leaves_len}"
            )));
        }

        // Update initial reads with class hashes and nonces from the proof.
        // We've validated the lengths match, so this zip is safe.
        let mut extended_initial_reads = StateMaps::default();
        for (leaf, addr) in
            rpc_proof.contracts_proof.contract_leaves_data.iter().zip(&query.contract_addresses)
        {
            extended_initial_reads.class_hashes.insert(*addr, ClassHash(leaf.class_hash));
            extended_initial_reads.nonces.insert(*addr, Nonce(leaf.nonce));
        }

        // Include storage values from execution.
        extended_initial_reads.storage.extend(&execution_data.initial_reads.storage);

        let commitment_infos = match config.include_state_changes {
            true => {
                Self::create_commitment_infos_with_state_changes(
                    &rpc_proof,
                    &query,
                    &extended_initial_reads,
                    &execution_data.state_diff,
                )
                .await?
            }
            false => Self::create_commitment_infos_without_state_changes(&rpc_proof, &query)?,
        };

        Ok(StorageProofs { extended_initial_reads, commitment_infos })
    }
