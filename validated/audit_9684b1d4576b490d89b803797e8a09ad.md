### Title
Mempool Capacity Checked Against Total Size Including Non-Evictable `delayed_declares`, Enabling Griefing DoS - (`crates/apollo_mempool/src/mempool.rs`)

### Summary

The mempool's byte-capacity enforcement measures total occupancy as `tx_pool + delayed_declares`, but the eviction path (`try_make_space`) can only remove transactions from `tx_pool` via `accounts_with_gap`. An attacker who fills `delayed_declares` up to `capacity_in_bytes` with valid-nonce declare transactions leaves the eviction path with nothing to free, causing every subsequent legitimate transaction to be rejected with `MempoolError::MempoolFull`.

### Finding Description

`size_in_bytes()` aggregates both sub-stores:

```rust
fn size_in_bytes(&self) -> u64 {
    self.tx_pool.size_in_bytes() + self.delayed_declares.size_in_bytes()
}
``` [1](#0-0) 

`exceeds_capacity()` gates admission against this combined total:

```rust
fn exceeds_capacity(&self, tx: &InternalRpcTransaction, freed_bytes: u64) -> bool {
    (self.size_in_bytes() + tx.total_bytes()).saturating_sub(freed_bytes)
        > self.config.static_config.capacity_in_bytes
}
``` [2](#0-1) 

When capacity is exceeded, `handle_capacity_overflow` calls `try_make_space`, which exclusively evicts transactions from `tx_pool` by iterating `accounts_with_gap`:

```rust
pub fn try_make_space(&mut self, required_space: u64) -> bool {
    ...
    while total_space_freed < required_space {
        let Some(address) = self.get_evictable_account() else { break; };
        let txs: Vec<_> = self.tx_pool.account_txs_sorted_by_nonce(address)...
        ...
    }
    total_space_freed >= required_space
}
``` [3](#0-2) 

`delayed_declares` are never touched by `try_make_space`. Meanwhile, `add_tx` routes every non-FIFO declare transaction directly into `delayed_declares` (not `tx_pool`):

```rust
let should_delay_declare =
    matches!(&args.tx.tx, InternalRpcTransactionWithoutTxHash::Declare(_))
        && !self.is_fifo();
if should_delay_declare {
    self.delayed_declares.push_back(self.clock.now(), args);
} else {
    self.add_tx_inner(args);
}
``` [4](#0-3) 

Because declares with `nonce == account_nonce` never enter `accounts_with_gap`, the attacker's transactions are invisible to the eviction loop. The result is a structural asymmetry identical to M-16: the cap is checked against the **total** (pool + delayed), but only the **pool** portion is evictable.

### Impact Explanation

Once `delayed_declares.size_in_bytes()` alone reaches `capacity_in_bytes` (default 1 GB), every incoming invoke, deploy, or declare transaction triggers `exceeds_capacity → handle_capacity_overflow → try_make_space → false → MempoolError::MempoolFull`. The gateway propagates this rejection to the RPC caller. No legitimate transaction can enter the mempool until the delay window expires and `add_ready_declares` drains the backlog — but the attacker can continuously replenish it.

This matches: **High — Mempool/gateway/RPC admission rejects valid transactions before sequencing.** [5](#0-4) 

### Likelihood Explanation

- The `declare_delay` is configurable (default 20 blocks/seconds). [6](#0-5) 
- The attacker must pay fees for each declare, but the 1 GB cap and typical declare sizes mean a sustained flood is feasible for a well-resourced adversary.
- No privilege is required; any account can submit declare transactions via the public RPC.
- The attack is self-sustaining: as old declares graduate to `tx_pool` and become evictable, the attacker submits fresh ones to refill `delayed_declares`.

### Recommendation

Enforce the capacity limit only against the evictable portion, mirroring the fix recommended in M-16:

```rust
fn size_in_bytes(&self) -> u64 {
    // Only count tx_pool; delayed_declares are non-evictable and must not
    // consume capacity that legitimate transactions cannot reclaim.
    self.tx_pool.size_in_bytes()
}
```

Alternatively, make `delayed_declares` evictable in `try_make_space` (evict the lowest-priority delayed declare when no gap-account exists), or impose a separate, smaller cap on `delayed_declares` independent of the main `capacity_in_bytes`.

### Proof of Concept

1. Attacker holds accounts `A_0 … A_k`, each with `account_nonce = 0`.
2. Attacker submits declare transactions with `nonce = 0` from each account until `delayed_declares.size_in_bytes() ≥ capacity_in_bytes`.
   - Each declare passes `add_tx_validations` (valid nonce, no duplicate) and is routed to `delayed_declares`.
   - No gap is created, so no entry appears in `accounts_with_gap`.
3. Victim submits a legitimate invoke transaction with correct nonce.
4. `exceeds_capacity` returns `true` (total size ≥ cap).
5. `handle_capacity_overflow` computes `creating_gap = false` (correct nonce), calls `try_make_space`.
6. `try_make_space` iterates `accounts_with_gap` — empty — returns `false`.
7. `handle_capacity_overflow` returns `Err(MempoolError::MempoolFull)`.
8. Victim's transaction is rejected at the gateway before sequencing.
9. Attacker repeats step 2 every `declare_delay` interval to maintain the block. [7](#0-6)

    pub fn try_make_space(&mut self, required_space: u64) -> bool {
        let mut total_space_freed = 0;
        let mut evicted_txs = Vec::new();

        while total_space_freed < required_space {
            let Some(address) = self.get_evictable_account() else {
                break;
            };

            let txs: Vec<_> = self.tx_pool.account_txs_sorted_by_nonce(address).copied().collect();
            for tx_ref in txs.iter().rev() {
                let tx = self
                    .tx_pool
                    .remove(tx_ref.tx_hash)
                    .expect("Transaction must exist in the pool.");
                total_space_freed += tx.total_bytes();
                evicted_txs.push(*tx_ref);
                metric_count_evicted_txs(1);
                self.decrement_stuck_txs_if_gap_account(address, 1);
                if total_space_freed >= required_space {
                    break;
                }
            }

            // Clean up if account is now empty.
            if !self.tx_pool.contains_account(address) {
                self.accounts_with_gap.swap_remove(&address);
            }
        }

        // Keep the queue consistent with the pool: the evicted txs were removed from the pool, so
        // drop their (now-orphaned) queue references too. In fee mode gap accounts are never
        // queued, so this is a no-op there; in Echonet/FIFO mode they are, and skipping this leaves
        // a dangling reference that panics the next `get_txs`.
        self.tx_queue.remove_txs(&evicted_txs);

        total_space_freed >= required_space
    }

```rust
    fn handle_capacity_overflow(
        &mut self,
        tx: &InternalRpcTransaction,
        account_nonce: Nonce,
        freed_bytes: u64,
    ) -> Result<(), MempoolError> {
        let address = tx.contract_address();

        let account_has_gap = self.accounts_with_gap.contains(&address);
        let account_has_txs = self.tx_pool.contains_account(address);
        let closing_gap = tx.nonce() == account_nonce;
        let creating_gap = (account_has_gap || !account_has_txs) && !closing_gap;

        // Only the net growth must be evicted: an accompanying replacement removal frees
        // `freed_bytes` (0 when there is no replacement).
        let required_space = tx.total_bytes().saturating_sub(freed_bytes);
        if !creating_gap && self.try_make_space(required_space) {
            return Ok(());
        }

        Err(MempoolError::MempoolFull)
    }
