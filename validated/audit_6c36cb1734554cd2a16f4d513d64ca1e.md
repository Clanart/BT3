### Title
Missing Zero-Value Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` syscall handler in the StarkNet OS accepts `class_hash = 0` (`UNINITIALIZED_CLASS_HASH`) as a valid replacement class, with no guard against it. A contract can call `replace_class(0)` on itself, setting its class hash to the sentinel uninitialized value. Because the contract's nonce is non-zero after use, it cannot be redeployed to the same address. Any funds (token balances) stored in the contract's storage are permanently frozen.

### Finding Description
`execute_replace_class` in `syscall_impls.cairo` reads the requested class hash directly from the syscall request and writes it into the contract state without any validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The constant `UNINITIALIZED_CLASS_HASH = 0` is the sentinel value used throughout the OS to represent a contract that has never been deployed: [2](#0-1) 

`deploy_contract` enforces that a contract address must have `class_hash == UNINITIALIZED_CLASS_HASH` before it can be deployed, and also that `nonce == 0`: [3](#0-2) 

After a contract has been used (nonce > 0), it cannot be redeployed to the same address even if its class hash is reset to 0, because the nonce check will fail. The contract's storage (holding user funds) persists in the state tree, but with no executable class attached, making those funds permanently inaccessible.

The TODO comment in the source code explicitly acknowledges the missing validation: [4](#0-3) 

### Impact Explanation
**Critical — Permanent freezing of funds.**

When a contract's `class_hash` is set to `0` via `replace_class(0)`:
1. The contract's storage (e.g., ERC-20 balances, vault positions) remains in the global state tree, as `get_contract_state_hash` only returns `hash=0` when *all three* of `class_hash`, `storage_root`, and `nonce` are zero simultaneously.
2. Any subsequent call to the contract will fail at entry-point dispatch because there is no class to execute.
3. Redeployment to the same address is blocked by the nonce check in `deploy_contract`.
4. There is no recovery path: the funds are permanently frozen. [5](#0-4) 

### Likelihood Explanation
Any contract can issue the `replace_class` syscall on itself. A malicious deployer can:
1. Deploy a contract that accepts user deposits (e.g., a fake vault or DEX).
2. Wait for users to deposit funds.
3. Invoke an internal function that calls `replace_class(0)`.

The OS will accept the syscall, commit the state change, and produce a valid proof. No privileged operator access is required. The attack is fully executable by an unprivileged transaction sender who controls their own contract's code.

### Recommendation
Add an `assert_not_zero(class_hash)` check (or equivalently `assert class_hash != UNINITIALIZED_CLASS_HASH`) immediately after reading `class_hash` from the request in `execute_replace_class`, before updating the state. Additionally, implement the noted TODO to verify that the given class hash corresponds to a previously declared class in `contract_class_changes`.

```cairo
let class_hash = request.class_hash;
// Guard: reject zero (UNINITIALIZED_CLASS_HASH) as a replacement class.
assert_not_zero(class_hash);
```

### Proof of Concept
1. Deploy contract `MaliciousVault` with a `drain()` entry point that calls `replace_class(0)` via the `replace_class` syscall.
2. Users call `deposit()` on `MaliciousVault`, storing their token balances in the contract's storage.
3. Attacker calls `drain()`. The OS executes `execute_replace_class` with `request.class_hash = 0`. No validation fires. The state entry for `MaliciousVault` is updated to `StateEntry(class_hash=0, storage_ptr=..., nonce=N)`.
4. The block is proven and committed. The global state root reflects `class_hash=0` for `MaliciousVault`.
5. Any subsequent call to `MaliciousVault` fails — no class exists to dispatch to.
6. `deploy_contract` cannot redeploy to the same address because `state_entry.nonce = N ≠ 0`.
7. All user funds stored in `MaliciousVault`'s storage are permanently frozen. [6](#0-5) [7](#0-6)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
