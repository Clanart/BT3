### Title
Stale Base Fee Snapshot in `NewDynamicFeeChecker` Allows Cosmos Txs to Bypass Current EIP-1559 Fee Enforcement - (File: `ante/evm/fee_checker.go`, `evmd/ante/handler_options.go`)

---

### Summary

`newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712` capture `evmParams` and `feemarketParams` as **value snapshots at construction time** and pass them into `NewDynamicFeeChecker`. Because the base fee changes every block, the fee checker enforces a stale base fee for all subsequent Cosmos-wrapped transactions, allowing under-priced transactions to pass or over-priced ones to be rejected.

---

### Finding Description

In `evmd/ante/handler_options.go`, both `newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712` read params once at construction time and pass them by pointer into the fee checker:

```go
// evmd/ante/handler_options.go:179-187
evmParams := options.EvmKeeper.GetParams(ctx)
feemarketParams := options.FeeMarketKeeper.GetParams(ctx)
...
txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
``` [1](#0-0) 

`NewDynamicFeeChecker` closes over the `feemarketParams` pointer and calls `types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)` on every transaction:

```go
// ante/evm/fee_checker.go:56
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
``` [2](#0-1) 

`types.GetBaseFee` reads the base fee directly from the captured params struct, not from the live KV store:

```go
// x/evm/types/utils.go:244-254
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    ...
    baseFee := feemarketParams.GetBaseFee()
    ...
}
``` [3](#0-2) 

The base fee is updated every block by `CalculateBaseFee` → `SetBaseFee` → `SetParams`, which writes to the persistent KV store:

```go
// x/feemarket/keeper/params.go:72-78
func (k Keeper) SetBaseFee(ctx sdk.Context, baseFee *big.Int) {
    params := k.GetParams(ctx)
    params.BaseFee = ethermint.SaturatedNewInt(baseFee)
    err := k.SetParams(ctx, params)
    ...
}
``` [4](#0-3) 

The snapshot struct captured at construction time is never updated. The same staleness affects `MinGasPriceDecorator`, which also stores `feemarketParams` at construction:

```go
// ante/cosmos/min_gas_price.go:44-46
func NewMinGasPriceDecorator(fk interfaces.FeeMarketKeeper, evmDenom string, feemarketParams *feemarkettypes.Params) MinGasPriceDecorator {
    return MinGasPriceDecorator{feesKeeper: fk, evmDenom: evmDenom, feemarketParams: feemarketParams}
}
``` [5](#0-4) 

**Contrast with `newEthAnteHandler`**, which correctly reads params fresh on every transaction inside the closure:

```go
// evmd/ante/handler_options.go:87-96
return func(ctx sdk.Context, tx sdk.Tx, simulate bool) (sdk.Context, error) {
    blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
    ...
    feemarketParams := &blockCfg.FeeMarketParams
    baseFee := blockCfg.BaseFee
``` [6](#0-5) 

The EVM ante handler always uses the live base fee; the Cosmos ante handler does not.

---

### Impact Explanation

Cosmos SDK transactions (non-EVM, e.g. IBC, governance, staking) that go through `newCosmosAnteHandler` with `DynamicFeeChecker = true` are validated against a stale base fee snapshot:

- **If the base fee has risen** since construction (normal under load): transactions priced between the stale (lower) base fee and the current (higher) base fee pass the fee check and are committed to the chain with fees below the protocol-required minimum. This is a fee under-payment that mis-accounts user funds and undermines the EIP-1559 fee market.
- **If the base fee has fallen**: legitimate transactions are incorrectly rejected, causing valid user funds to be denied.

The same stale `MinGasPrice` is used by `MinGasPriceDecorator`, compounding the mis-accounting.

This matches the allowed High impact: *"ante handler bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

---

### Likelihood Explanation

- Any deployment enabling `DynamicFeeChecker: true` for Cosmos txs is affected.
- The base fee changes every block under normal network activity, so the staleness grows continuously from the moment the ante handler is constructed.
- An unprivileged user only needs to submit a Cosmos tx with a fee between the stale and current base fee — no special access required.
- The longer the node runs without restarting (which would reconstruct the ante handler), the larger the divergence between stale and live base fee.

---

### Recommendation

Move the params read **inside** the returned closure, mirroring the pattern used in `newEthAnteHandler`. The `NewDynamicFeeChecker` signature should accept the keeper interfaces and read live params on each invocation:

```go
// Instead of capturing params at construction:
func NewDynamicFeeChecker(evmKeeper EVMKeeper, feeMarketKeeper FeeMarketKeeper) authante.TxFeeChecker {
    return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
        evmParams := evmKeeper.GetParams(ctx)
        feemarketParams := feeMarketKeeper.GetParams(ctx)
        baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, &feemarketParams)
        ...
    }
}
```

Similarly, `MinGasPriceDecorator.AnteHandle` should call `mpd.feesKeeper.GetParams(ctx).MinGasPrice` instead of reading from the stale `mpd.feemarketParams`.

---

### Proof of Concept

1. Node starts; `newCosmosAnteHandler` is called with block height N. Base fee at height N = **100 wei**. The snapshot `feemarketParams.BaseFee = 100` is captured.
2. Network becomes congested. By block N+50, `CalculateBaseFee` has raised the live base fee to **500 wei** in the KV store.
3. Attacker submits a Cosmos SDK tx (e.g., `MsgDelegate`) with `fee = 150 wei * gasLimit`.
4. `NewDynamicFeeChecker` reads `baseFee` from the stale snapshot → **100 wei**. Check: `feeCap (150) >= baseFee (100)` → **passes**.
5. The live store has base fee = 500 wei, but the tx was accepted at 150 wei — a 70% fee underpayment committed to chain.
6. The fee market's burn/distribution accounting is mis-stated for every such transaction. [7](#0-6) [8](#0-7)
