### Title
Meta-Transaction V0 Hash Excludes Nonce, Enabling Unbounded Signature Replay - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/transaction_hash/transaction_hash.cairo`)

---

### Summary

The `compute_meta_tx_v0_hash` function omits a nonce from the signed data, and `execute_meta_tx_v0` hardcodes `nonce=0` while the OS-level nonce check explicitly skips version-0 transactions. This is the direct structural analog of the reported vulnerability: a meta-tx-v0 signature binds only to `(prefix, version, contract_address, selector, calldata, max_fee, chain_id)` — never to a replay-preventing counter. Any attacker who observes a valid meta-tx-v0 signature can replay it indefinitely against the target contract's `__execute__` entry point.

---

### Finding Description

**Root cause 1 — nonce absent from hash.**

`compute_meta_tx_v0_hash` delegates to `deprecated_get_transaction_hash` with `additional_data_size=0` and `additional_data=cast(0, felt*)`:

```cairo
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(
    contract_address: felt,
    entry_point_selector: felt,
    calldata: felt*,
    calldata_size: felt,
    chain_id: felt,
) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        ...
        additional_data_size=0,
        additional_data=cast(0, felt*),   // ← no nonce committed
    );
``` [1](#0-0) 

The resulting hash covers only `(INVOKE_HASH_PREFIX, 0, contract_address, selector, calldata_hash, 0, chain_id)`. No nonce, no block number, no caller — nothing that changes between two identical calls.

**Root cause 2 — nonce hardcoded to 0 in execution context.**

`execute_meta_tx_v0` constructs the `TxInfo` with `nonce=0`:

```cairo
tempvar new_tx_info = new TxInfo(
    version=0,
    ...
    nonce=0,          // ← always zero
    ...
);
``` [2](#0-1) 

**Root cause 3 — OS nonce guard explicitly skips version 0.**

`check_and_increment_nonce` contains an early return for version-0 transactions:

```cairo
// Do not handle nonce for version 0.
if (tx_info.version == 0) {
    return ();
}
``` [3](#0-2) 

The three roots combine: the hash commits no nonce, the TxInfo carries nonce=0, and the OS never increments or checks any counter for version-0 meta-transactions. A signature produced for one meta-tx-v0 invocation is cryptographically indistinguishable from a signature for any future invocation with the same `(contract_address, selector, calldata, chain_id)`.

**Dispatch path.**

`execute_syscalls` routes the `META_TX_V0_SELECTOR` directly to `execute_meta_tx_v0` with no additional guard:

```cairo
assert selector = META_TX_V0_SELECTOR;
execute_meta_tx_v0(block_context=block_context, caller_execution_context=execution_context);
``` [4](#0-3) 

Any Sierra contract can emit this syscall; no privileged role is required.

---

### Impact Explanation

**Critical — Direct loss of funds.**

The target contract's `__execute__` entry point is invoked with the attacker-supplied signature and the OS-computed hash. Any account contract that performs signature verification inside `__execute__` (the standard pattern for meta-transaction relayers and session-key schemes) will accept the replayed signature because the hash is identical to the original. The attacker can drain the target account by replaying a single observed signature as many times as gas allows, executing the same token-transfer or approval calldata repeatedly.

---

### Likelihood Explanation

**High.** The attacker's only prerequisite is observing one successful meta-tx-v0 call on-chain (public data). They then deploy a contract that emits the same `MetaTxV0Request` with the captured signature. No key material, no privileged access, and no race condition is required. The replay succeeds in every subsequent block until the target account is drained or the contract is upgraded.

---

### Recommendation

Include a monotonically-increasing nonce in `compute_meta_tx_v0_hash` by passing it as `additional_data`:

```cairo
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(
    contract_address: felt,
    entry_point_selector: felt,
    calldata: felt*,
    calldata_size: felt,
    chain_id: felt,
    nonce: felt,          // ← add nonce parameter
) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        ...
        additional_data_size=1,
        additional_data=&nonce,   // ← commit nonce
    );
```

Correspondingly, `execute_meta_tx_v0` must read the target contract's current nonce from `contract_state_changes`, pass it into the hash, and increment it — mirroring the logic in `check_and_increment_nonce` for version ≥ 1 transactions.

---

### Proof of Concept

1. **Legitimate use**: Relayer contract `R` calls `meta_tx_v0` targeting victim account `V` with calldata `transfer(attacker, 100_STRK)` and a valid signature `σ` from `V`'s key. The OS computes `H = hash(prefix, 0, V, __execute__, calldata, 0, chain_id)` — no nonce. `V.__execute__` verifies `σ` against `H` and executes the transfer.

2. **Replay**: Attacker deploys contract `A`. `A` emits `MetaTxV0Request { contract_address=V, selector=__execute__, calldata=transfer(attacker,100_STRK), signature=σ }`. The OS recomputes the identical `H` (same inputs, no nonce), sets `nonce=0` in `TxInfo`, and calls `V.__execute__`. `V.__execute__` verifies `σ` against `H` — still valid — and executes the transfer again.

3. **Iteration**: Step 2 repeats in every block. `V`'s balance is drained to zero. The attacker never needed `V`'s private key after the initial observation. [5](#0-4) [6](#0-5) [7](#0-6)

func check_and_increment_nonce{contract_state_changes: DictAccess*}(tx_info: TxInfo*) -> () {
    // Do not handle nonce for version 0.
    if (tx_info.version == 0) {
        return ();
    }

    tempvar state_entry: StateEntry*;
    %{ SetStateEntryToAccountContractAddress %}

    tempvar current_nonce = state_entry.nonce;
    with_attr error_message("Unexpected nonce.") {
        assert current_nonce = tx_info.nonce;
    }

    // Update contract_state_changes.
    tempvar new_state_entry = new StateEntry(
        class_hash=state_entry.class_hash,
        storage_ptr=state_entry.storage_ptr,
        nonce=current_nonce + 1,
    );
    dict_update{dict_ptr=contract_state_changes}(
        key=tx_info.account_contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    return ();
}
