### Title
Missing shard-identity check on `prev_chunk_header` in `set_state_header` allows receipt-chain termination height corruption — (`chain/chain/src/state_sync/adapter.rs`)

### Summary

`ChainStateSyncAdapter::set_state_header` verifies that `prev_chunk_header` is included in a block's Merkle tree of chunk headers, but uses a **position-agnostic** `verify_path` call and never asserts `prev_chunk_header.shard_id() == shard_id`. An unprivileged state-sync peer can substitute a genuine chunk header from a different shard (with its valid Merkle proof) to corrupt the receipt-chain termination height used in step 4g, causing `set_state_finalize` to apply the wrong set of incoming receipts and produce a state root that diverges from the canonical chain.

---

### Finding Description

**Step 3b — position-agnostic Merkle proof for `prev_chunk_header`**

The check at lines 416–425 of `chain/chain/src/state_sync/adapter.rs` is:

```rust
if !verify_path(
    *prev_block_header.chunk_headers_root(),
    prev_chunk_proof,
    &ChunkHashHeight(prev_chunk_header.chunk_hash().clone(),
                     prev_chunk_header.height_included()),
) { … }
```

`chunk_headers_root` is a Merkle tree over **all** shards' chunk headers in the block: [1](#0-0) 

`verify_path` only checks that the supplied `ChunkHashHeight` is *somewhere* in the tree — it does not verify which leaf index (shard position) the proof corresponds to: [2](#0-1) 

The test `test_incorrect_path` confirms that a proof for leaf *i* does not verify for leaf *j* when the *items* differ, but it equally confirms that a proof for shard-1's `ChunkHashHeight` verifies correctly when shard-1's `ChunkHashHeight` is the supplied item — regardless of which shard was requested. [3](#0-2) 

**No shard-identity assertion exists anywhere in `set_state_header`:** [4](#0-3) 

**Step 4g — receipt-chain termination uses the attacker-controlled height**

After iterating the receipt chain, the code checks:

```rust
if header.height() != prev_chunk_header.map_or(0, |h| h.height_included()) {
    return Err(…);
}
``` [5](#0-4) 

If `prev_chunk_header` belongs to shard 1 with `height_included() = H1`, the chain is accepted as long as it terminates at height H1, regardless of shard 0's actual `prev_chunk_height_included = H0`.

**Receipt proofs are bound to `shard_id` (the requested shard), not to `prev_chunk_header.shard_id()`:** [6](#0-5) 

So the attacker cannot forge receipt content, but can freely choose the *length* of the receipt chain by picking a `prev_chunk_header` whose `height_included()` differs from H0.

**`set_state_finalize` applies whatever receipts are in the stored header**

Receipts are filtered only by `chunk.height_included()`, not by `prev_chunk_header.height_included()`: [7](#0-6) 

If the chain was truncated (H1 > H0), receipts from blocks at heights H0+1…H1 are absent; the syncing node applies too few receipts. If the chain was extended (H1 < H0), receipts from blocks H1+1…H0 are included a second time; the syncing node double-applies receipts. Either way the resulting state root diverges from the canonical value.

---

### Impact Explanation

A syncing node that accepts the crafted header will:
1. Store it under `DBCol::StateHeaders` keyed by `(shard_id, sync_hash)`.
2. Call `set_state_finalize`, which reads the stored header and applies the wrong receipt set.
3. Produce an incorrect `ChunkExtra` / state root for the synced shard.

If the node is a chunk producer or chunk validator for that shard in the next epoch, it will produce or endorse chunks built on the wrong state, causing it to be slashed or to diverge from the rest of the network. [8](#0-7) 

---

### Likelihood Explanation

Any peer that the syncing node contacts for state can mount this attack. State-sync peers are not required to be validators; an unprivileged node that has the relevant block data can respond to state-sync requests. The attacker needs only publicly observable on-chain data (block headers, chunk headers, receipt Merkle proofs) — no key material or privileged access is required.

---

### Recommendation

Add an explicit shard-identity assertion immediately after extracting `prev_chunk_header` in `set_state_header`:

```rust
if let Some(ref pch) = prev_chunk_header {
    if pch.shard_id() != shard_id {
        return Err(Error::Other(
            "set_shard_state failed: prev_chunk_header belongs to wrong shard".into(),
        ));
    }
}
```

Similarly add the same guard for the main `chunk` field (`chunk.shard_id() != shard_id`), since step 3a's `verify_path` is equally position-agnostic. [9](#0-8) 

---

### Proof of Concept

**Test-loop sketch (production code path only):**

1. Set up a two-shard network. Let shard 0's last-included chunk before `sync_hash` be at height H0, and shard 1's at height H1 > H0.
2. Craft a `ShardStateSyncResponseHeaderV2` for shard 0 where:
   - `chunk` = the real shard-0 chunk (passes `validate_chunk_proofs` and step 3a).
   - `prev_chunk_header` = shard 1's chunk header from the block at height `chunk.height_included() - 1` (genuinely present in `prev_block_header.chunk_headers_root()`).
   - `prev_chunk_proof` = the valid Merkle path for shard 1's leaf in that block.
   - `incoming_receipts_proofs` = the real receipt chain for shard 0, but truncated to cover only blocks from `sync_hash` back to H1+1 (omitting blocks H0+1…H1).
3. Call `set_state_header(shard_id=0, sync_hash, crafted_header)`.
   - Step 3b passes: shard 1's `ChunkHashHeight` is in the tree with a valid proof.
   - Step 4g passes: the chain terminates at height H1 == `prev_chunk_header.height_included()`.
4. Assert `set_state_header` returns `Ok(())`.
5. Call `set_state_finalize(shard_id=0, sync_hash)`.
6. Assert the resulting `ChunkExtra` state root differs from the canonical state root (receipts from blocks H0+1…H1 were not applied).

    pub fn set_state_finalize(
        &mut self,
        shard_id: ShardId,
        sync_hash: CryptoHash,
        shard_state_header: ShardStateSyncResponseHeader,
    ) -> Result<ShardUId, Error> {
        let _span =
            tracing::debug_span!(target: "sync", "chain_update_set_state_finalize", %shard_id, ?sync_hash).entered();
        let (chunk, incoming_receipts_proofs) = match shard_state_header {
            ShardStateSyncResponseHeader::V1(shard_state_header) => (
                ShardChunk::V1(shard_state_header.chunk),
                shard_state_header.incoming_receipts_proofs,
            ),
            ShardStateSyncResponseHeader::V2(shard_state_header) => {
                (shard_state_header.chunk, shard_state_header.incoming_receipts_proofs)
            }
        };

        // Note that block headers are already synced and can be taken
        // from store on disk.
        let block_header = get_block_header_on_chain_by_height(
            &self.chain_store_update.chain_store(),
            &sync_hash,
            chunk.height_included(),
        )?;

        // Getting actual incoming receipts.
        let mut receipt_proof_responses: Vec<ReceiptProofResponse> = vec![];
        for incoming_receipt_proof in &incoming_receipts_proofs {
            let ReceiptProofResponse(hash, _) = incoming_receipt_proof;
            let block_header = self.chain_store_update.get_block_header(hash)?;
            if block_header.height() <= chunk.height_included() {
                receipt_proof_responses.push(incoming_receipt_proof.clone());
            }
        }
        let receipts = collect_receipts_from_response(&receipt_proof_responses);
        let is_genesis = block_header.height() == self.chain_store_update.get_genesis_height();
        let prev_block_header = (!is_genesis)
            .then(|| self.chain_store_update.get_block_header(block_header.prev_hash()))
            .transpose()?;

        // Prev block header should be present during state sync, since headers have been synced at
        // this point, except for genesis.
        let gas_price = if let Some(prev_block_header) = &prev_block_header {
            prev_block_header.next_gas_price()
        } else {
            block_header.next_gas_price()
        };

        let chunk_header = chunk.cloned_header();
        let gas_limit = chunk_header.gas_limit();
        let block = self.chain_store_update.get_block(block_header.hash())?;
        let transactions = chunk.to_transactions().to_vec();
        let transaction_validity = if let Some(prev_block_header) = prev_block_header {
            self.chain_store_update
                .chain_store()
                .compute_transaction_validity(&prev_block_header, &chunk)
        } else {
            vec![true; transactions.len()]
        };
        let transactions = SignedValidPeriodTransactions::new(transactions, transaction_validity);
        let shard_uid =
            shard_id_to_uid(self.epoch_manager.as_ref(), shard_id, block_header.epoch_id())?;
        let memtrie_pin = self
            .runtime_adapter
            .get_tries()
            .maybe_pin_memtrie_root(shard_uid, chunk_header.prev_state_root())?;
        let apply_result = self.runtime_adapter.apply_chunk(
            RuntimeStorageConfig::new(chunk_header.prev_state_root(), true),
            ApplyChunkReason::UpdateTrackedShard,
            ApplyChunkShardContext {
                shard_uid,
                gas_limit,
                last_validator_proposals: chunk_header.prev_validator_proposals(),
                is_new_chunk: true,
                on_post_state_ready: None,
                memtrie_pin,
            },
            ApplyChunkBlockContext {
                block_type: BlockType::Normal,
                height: chunk_header.height_included(),
                prev_block_hash: *chunk_header.prev_block_hash(),
                block_timestamp: block_header.raw_timestamp(),
                gas_price,
                random_seed: *block_header.random_value(),
                congestion_info: block.block_congestion_info(),
                bandwidth_requests: block.block_bandwidth_requests(),

                shard_id,
                receipt_proof_response.1,
            );
        }
        Ok(shard_uid)
    }
