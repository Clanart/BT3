### Title
Precommit Vote Signatures Omit `chain_id`, Enabling Cross-Chain Replay — (`crates/apollo_signature_manager/src/signature_manager.rs`)

---

### Summary

`build_precommit_vote_message_digest` constructs the BFT consensus precommit vote digest as `blake2s("PRECOMMIT_VOTE" || block_hash)`. Neither the domain separator nor the block hash itself contains the Starknet `chain_id`. A precommit vote signature produced on one Starknet network (e.g., mainnet) is cryptographically valid on any other Starknet network (e.g., testnet, integration, or a forked chain) for the same `block_hash` value.

---

---

### Impact Explanation

A validator operating on two Starknet networks (mainnet + testnet, or mainnet + a hard-forked chain) with the same private key produces identical precommit vote signatures for any `block_hash` value that appears on both networks. Because `starknet_version` is the same string across all networks and `chain_id` is absent from the block hash, a collision is structurally possible:

- At a fork point, both chains share the same parent block hash and may share the same block number, sequencer address, and timestamp for the diverging block.
- An adversary who collects quorum-many precommit signatures from validators on chain A (e.g., testnet) for a block_hash that also appears on chain B (mainnet) can inject those signatures into chain B's consensus, causing chain B to finalize a block it should not.
- This maps to **wrong state committed** on the target chain, satisfying the "Critical. Wrong state … from blockifier/syscall/execution logic" impact, and "High. Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."

---

### Likelihood Explanation

- Requires the same `block_hash` to appear on two chains simultaneously. Because `chain_id` is absent from the block hash formula, this is structurally possible (not prevented by design), especially at a fork point where state roots have not yet diverged.
- Validators running on multiple Starknet networks with the same key (a common operational pattern) are directly exposed.
- No privileged access is required; any party that can observe consensus messages on one network can attempt replay on another.

---

### Recommendation

Include `chain_id` in the precommit vote message digest, looked up dynamically (not cached at construction), so that signatures are chain-scoped:

```rust
fn build_precommit_vote_message_digest(
    block_hash: BlockHash,
    chain_id: &ChainId,
) -> MessageDigest {
    let block_hash = block_hash.to_bytes_be();
    let chain_id_bytes = chain_id.to_string().into_bytes();
    let mut message = Vec::new();
    message.extend_from_slice(PRECOMMIT_VOTE);
    message.extend_from_slice(&chain_id_bytes);
    message.extend_from_slice(&block_hash);
    MessageDigest(blake2s_to_felt(&message))
}
```

`SignatureManager` should receive `chain_id` at construction or per-call, and `verify_precommit_vote_signature` must accept and bind the same `chain_id`.

---

### Proof of Concept

1. Validator V holds private key `k` and runs on both Starknet mainnet and a forked/testnet chain.
2. On testnet, block N is produced with `block_hash = H` (where `H` is determined by block number, state root, sequencer, timestamp, commitments — none of which include `chain_id`).
3. V signs a precommit vote on testnet: `sig = ecdsa_sign(k, blake2s("PRECOMMIT_VOTE" || H))`.
4. An adversary observes `sig` on testnet.
5. On mainnet, if block N also produces `block_hash = H` (possible at fork point or by construction), the adversary submits `sig` as V's precommit vote on mainnet.
6. `verify_precommit_vote_signature(H, sig, V.public_key)` returns `true` on mainnet — the replayed vote is accepted.
7. If the adversary collects quorum-many such replayed signatures, mainnet consensus finalizes block N with the replayed votes, committing potentially wrong state. [6](#0-5) [1](#0-0) [3](#0-2)

func calculate_block_hash{poseidon_ptr: PoseidonBuiltin*}(
    block_info: BlockInfo*,
    header_commitments: BlockHeaderCommitments*,
    gas_prices_hash: felt,
    state_root: felt,
    previous_block_hash: felt,
    starknet_version: felt,
) -> felt {
    static_assert BlockInfo.SIZE == 3;
    static_assert BlockHeaderCommitments.SIZE == 5;

    let hash_state = hash_init();
    with hash_state {
        hash_update_single(BLOCK_HASH_VERSION);
        hash_update_single(block_info.block_number);
        hash_update_single(state_root);
        hash_update_single(block_info.sequencer_address);
        hash_update_single(block_info.block_timestamp);
        hash_update_single(header_commitments.packed_lengths);
        hash_update_single(header_commitments.state_diff_commitment);
        hash_update_single(header_commitments.transaction_commitment);

        hash_update_single(header_commitments.receipt_commitment);
        hash_update_single(gas_prices_hash);
        hash_update_single(starknet_version);
        hash_update_single(0);
        hash_update_single(previous_block_hash);
    }

    let block_hash = hash_finalize(hash_state=hash_state);
    return block_hash;
