### Title
Missing Declared Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the new class hash supplied by the caller corresponds to a previously declared contract class. An attacker-controlled contract can call `replace_class` with an arbitrary, undeclared class hash, permanently replacing its own class with one that has no associated compiled class. All subsequent calls to that contract will fail at execution time, permanently freezing any funds held by the contract.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function accepts a caller-supplied `class_hash` and writes it directly into `contract_state_changes` without checking whether that hash exists in `contract_class_changes` (the registry of declared classes):

```cairo
// Replaces the class.
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges this missing check. Contrast this with the `execute_declare_transaction` flow, which properly enforces that a class hash is cryptographically derived from the Sierra class components and registers it in `contract_class_changes` with `prev_value=0` (enforcing uniqueness): [2](#0-1) 

When any subsequent call is made to the affected contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
``` [3](#0-2) 

If `class_hash` was never declared, `dict_read` returns 0 (the default uninitialized value). The subsequent `find_element` call searching for a `CompiledClassFact` with `hash=0` will fail to locate a valid compiled class, causing every future call to the contract to revert. The deprecated `execute_replace_class` in `deprecated_execute_syscalls.cairo` has the identical omission: [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with an undeclared hash, the contract becomes permanently inaccessible. Any ETH/STRK or ERC-20 tokens held in that contract's storage are irrecoverably frozen. There is no recovery path: the class hash is committed to the global state tree, and no mechanism exists to revert it without a valid `replace_class` call — which itself requires the contract to be callable.

---

### Likelihood Explanation

The `replace_class` syscall is callable by any contract on itself. No privileged role is required. An attacker who controls a contract (or who can trick a contract into calling `replace_class` with an attacker-supplied argument) can trigger this. The entry path is fully unprivileged and reachable from any deployed contract.

---

### Recommendation

Before writing the new class hash to `contract_state_changes`, verify that it exists in `contract_class_changes` (i.e., has been declared). Concretely, perform a `dict_read` on `contract_class_changes` with the supplied `class_hash` and assert the returned `compiled_class_hash` is non-zero:

```cairo
let class_hash = request.class_hash;

// Enforce that the class has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the invariant already enforced during `execute_declare_transaction` and closes the gap identified by the TODO comment.

---

### Proof of Concept

1. Attacker deploys contract `A` holding 1000 STRK.
2. Contract `A` executes the `replace_class` syscall with `class_hash = 0xdeadbeef` (an arbitrary value never declared via a `Declare` transaction).
3. The OS calls `execute_replace_class`, which skips the missing declared-class check and writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes`.
4. The block is proven and the state root is updated; contract `A` now has `class_hash=0xdeadbeef` on-chain.
5. Any subsequent invoke transaction targeting contract `A` reaches `execute_entry_point`, which calls `dict_read{dict_ptr=contract_class_changes}(key=0xdeadbeef)` and receives `0` (undeclared).
6. `find_element` with `key=0` finds no valid `CompiledClassFact`; execution fails unconditionally.
7. The 1000 STRK in contract `A`'s storage is permanently frozen with no recovery path.
