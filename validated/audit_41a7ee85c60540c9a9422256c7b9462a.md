### Title
`execute_replace_class` Accepts Undeclared Class Hash Without Validation, Enabling Permanent Fund Freeze ŌĆö (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS Cairo program updates a contract's class hash in `contract_state_changes` without verifying that the supplied class hash has ever been declared (i.e., exists in `contract_class_changes`). An acknowledged `TODO` comment at line 898 confirms the check is intentionally absent. An unprivileged contract can call `replace_class` with an arbitrary, undeclared class hash, permanently setting its own class hash to a value for which no compiled class exists. The contract becomes permanently un-callable, freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` reads the caller-supplied `class_hash` from the syscall request and writes it directly into `contract_state_changes` with no cross-check against `contract_class_changes`:

```cairo
// syscall_impls.cairo, lines 877ŌĆō916

    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );
    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The `contract_class_changes` dict maps `class_hash ŌåÆ compiled_class_hash`. When a class is declared via a `Declare` transaction, an entry is written into this dict. The `execute_replace_class` function never reads from or asserts membership in `contract_class_changes` for the new `class_hash`.

When the contract is subsequently called (in the same or a later block), `execute_entry_point` does:

```cairo
// execute_entry_point.cairo, lines 154ŌĆō166
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // ŌåÉ the undeclared hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,           // ŌåÉ returns 0 for undeclared hash
);
``` [2](#0-1) 

`dict_read` returns `0` for a key never written into `contract_class_changes`. `find_element` with `key=0` then fails with a Cairo assertion error (no compiled class has hash `0`), making the contract permanently un-callable. The same missing check exists in the deprecated syscall path: [3](#0-2) 

---

### Impact Explanation

**Critical ŌĆö Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value and the block is committed, the contract state tree permanently records the invalid class hash. No subsequent transaction can successfully execute against that contract: the OS will always fail at `find_element` when trying to resolve the compiled class. Any ERC-20 tokens, ETH, STRK, or other assets held by the contract are irrecoverably frozen. The contract cannot self-rescue via another `replace_class` call because it cannot be entered at all.

---

### Likelihood Explanation

Any deployed contract can call `replace_class` from within its own logic. A malicious contract author needs only to deploy a contract that calls `replace_class(arbitrary_undeclared_hash)` ŌĆö no privileged role, no leaked key, no operator cooperation required. The attacker can target their own contract (to grief users who sent funds to it) or, if the contract is a shared protocol contract, affect all users of that contract. The entry path is fully unprivileged and requires only a standard `InvokeTransaction`.

---

### Recommendation

Inside `execute_replace_class`, after reading `class_hash` from the request, assert that the class hash has a non-zero entry in `contract_class_changes` before writing to `contract_state_changes`:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=class_hash
);
with_attr error_message("replace_class: class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the validation already performed in `execute_entry_point` and closes the gap acknowledged by the existing TODO comment. Apply the same fix to the deprecated path in `deprecated_execute_syscalls.cairo`.

---

### Proof of Concept

1. **Deploy** `MaliciousContract` holding user funds. Its `__execute__` function calls `replace_class(0xdeadbeef)` where `0xdeadbeef` is never declared.
2. **Invoke** `MaliciousContract.__execute__`. The OS processes `execute_replace_class`:
   - `class_hash = 0xdeadbeef` is written into `contract_state_changes` with no validation.
   - The block is proven and committed. The contract state tree now records `class_hash = 0xdeadbeef`.
3. **Attempt** any subsequent call to `MaliciousContract`:
   - `execute_entry_point` calls `dict_read(contract_class_changes, key=0xdeadbeef)` ŌåÆ returns `0`.
   - `find_element(..., key=0)` fails: no compiled class with hash `0` exists.
   - The OS cannot produce a valid proof for any block containing a call to `MaliciousContract`.
4. **Result**: All funds inside `MaliciousContract` are permanently frozen. The contract is bricked with no recovery path.

func execute_replace_class{
    range_check_ptr,

    alloc_locals;

    // Reduce gas.

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.

    %{ GetContractAddressStateEntry %}

        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,

    );
