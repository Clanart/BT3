### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Fund Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the new class hash supplied by a contract is actually declared in the protocol's class registry (`contract_class_changes`). This is a state-transition bypass: the OS performs a class-replacement state transition without checking the validity of the target state. A contract can replace its class with an arbitrary, undeclared felt value, permanently rendering itself non-executable and freezing any funds tracked in its own storage.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the caller's current `StateEntry`, constructs a new `StateEntry` with the requested `class_hash`, and writes it back to `contract_state_changes` — all without verifying that `class_hash` is present in `contract_class_changes`:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment at line 898 is an explicit acknowledgment by the authors that this check is absent. Compare this with the analogous protection present in `execute_declare_transaction`, which uses `prev_value=0` in `dict_update` to enforce that a class can only be declared once, and with `deploy_contract`, which asserts `state_entry.class_hash = UNINITIALIZED_CLASS_HASH` before deploying. No equivalent guard exists for `replace_class`.

The vulnerability class is identical to the external report: a state transition is executed without validating the current or target state. In the external report, an order in a terminal "filled" state could be cancelled again because `_remaining[orderHash] != 1` was never checked. Here, a class replacement is accepted even when the target class hash has never been declared, because no membership check against `contract_class_changes` is performed.

---

### Impact Explanation

Once a contract's `class_hash` in `contract_state_changes` is set to an undeclared value, every subsequent call to that contract — whether via `call_contract`, `library_call`, or a direct transaction — will fail at class-lookup time. The contract's storage (including any internally tracked token balances, escrow records, or vault positions) becomes permanently inaccessible. There is no recovery path: the OS has no mechanism to "un-replace" a class, and the state commitment will reflect the invalid class hash as the canonical state.

**Impact category**: Critical — Permanent freezing of funds.

---

### Likelihood Explanation

The `replace_class` syscall is callable by any Sierra contract from within its own execution context. A realistic path:

1. A contract (e.g., a vault, escrow, or multisig) contains an upgrade function that accepts a new class hash from an authorized caller.
2. An attacker exploits a missing access-control check or an authorization bypass in that upgrade function to supply an arbitrary, undeclared class hash.
3. The OS accepts the syscall without validation.
4. The contract is permanently non-functional; all internally tracked balances are frozen.

Alternatively, a malicious contract author deploys a "honeypot" contract that accepts deposits and then self-destructs via `replace_class` with an invalid hash, permanently locking deposited funds. The OS provides no protocol-level backstop.

Likelihood is **medium**: the syscall is reachable by any unprivileged contract caller, and the missing check is confirmed by the authors' own TODO. The attack requires either a vulnerable upgrade path in a target contract or a deliberate honeypot deployment.

---

### Recommendation

Before writing the new `StateEntry`, verify that the requested `class_hash` is present in `contract_class_changes` (i.e., has been declared in the current or a prior block). The analogous check already exists for class declarations:

```cairo
// In execute_replace_class, after gas reduction:
let (declared_compiled_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("replace_class: class hash not declared.") {
    assert_not_zero(declared_compiled_hash);
}
```

This mirrors the invariant enforced by `execute_declare_transaction` (`prev_value=0` on `dict_update`) and `deploy_contract` (`assert state_entry.class_hash = UNINITIALIZED_CLASS_HASH`).

---

### Proof of Concept

1. Declare and deploy a vault contract `V` that holds user deposits tracked in its own storage and exposes an `upgrade(new_class_hash)` function with a missing or bypassable access-control check.
2. Users deposit funds into `V`; balances are recorded in `V`'s storage.
3. Attacker calls `V.upgrade(0xdeadbeef)` where `0xdeadbeef` is never declared.
4. `execute_replace_class` in `syscall_impls.cairo` (lines 878–916) accepts the call, writes `class_hash=0xdeadbeef` into `contract_state_changes` for `V`'s address, and returns success.
5. The state commitment is updated with `V` pointing to an undeclared class.
6. Any subsequent call to `V` (withdraw, transfer, etc.) fails at class resolution — no entry point can be dispatched.

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

        key=contract_address,
