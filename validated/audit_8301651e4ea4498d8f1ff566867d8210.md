### Title
Missing Transaction Commitment Verification in P2P Sync Allows Malicious Peer to Corrupt Stored Block Body — (`crates/apollo_p2p_sync/src/client/transaction.rs`)

### Summary
`TransactionStreamFactory::parse_data_for_block` accepts exactly `n_transactions` `FullTransaction` objects from a p2p peer and stores them via `append_body` without ever verifying that the assembled `BlockBody` satisfies `calculate_transaction_commitment(stored_transactions) == stored_header.transaction_commitment`. A malicious peer can therefore corrupt the stored body for any block it serves.

### Finding Description

The call path is:

1. **`parse_data_for_block`** reads `n_transactions` from the already-stored header and collects that many `FullTransaction` objects from the peer network stream. No commitment check is performed. The code even contains an explicit acknowledgement of the missing validation:

   > `// TODO(eitan): Validate transaction hash from untrusted sources` [1](#0-0) 

2. **`write_to_storage`** calls `append_body` directly on the assembled `BlockBody` with no pre-check against the header's `transaction_commitment` field: [2](#0-1) 

3. **`append_body`** in the storage layer updates the body marker and writes all transactions to disk. It performs only a sequential-marker check (`MarkerMismatch`) and a duplicate-hash check; it does **not** recompute or compare the transaction commitment: [3](#0-2) 

The stored header carries a `transaction_commitment` that was set during header sync and is part of the block hash preimage: [4](#0-3) 

The commitment is a Poseidon Patricia root over `H(tx_hash, tx_signature)` leaves: [5](#0-4) 

Nothing in the sync pipeline recomputes this root and compares it to `stored_header.transaction_commitment` before or after `append_body`. The invariant is therefore entirely unenforced on the body-sync path.

### Impact Explanation

After a successful attack:
- `get_block_transactions` / `get_transaction` return the attacker-supplied transactions.
- RPC endpoints (`starknet_getBlockWithTxs`, `starknet_getTransactionByHash`, etc.) serve the corrupted data to clients.
- Any execution-replay path that reads transactions from body storage (e.g., `starknet_simulateTransactions`, `starknet_traceBlockTransactions`) executes the wrong transactions, producing wrong state diffs, receipts, and events.

The stored `transaction_commitment` in the header remains correct (it was set during header sync), but the stored body is silently inconsistent with it.

### Likelihood Explanation

Any p2p peer the syncing node connects to can trigger this. No special privilege is required — the attacker only needs to be a peer that responds to the transaction query for a given block range. The node will accept and store whatever T transactions the peer sends, as long as the count equals `n_transactions` from the header.

### Recommendation

After collecting all transactions in `parse_data_for_block`, recompute the transaction commitment from the assembled `BlockBody` and compare it to `stored_header.transaction_commitment`. Reject with `BadPeerError` if they differ. The function `calculate_transaction_commitment` already exists and is used in the batcher path; it should be called here before returning `Ok(Some((block_body, block_number)))`. [6](#0-5)

    fn parse_data_for_block<'a>(
        transactions_response_manager: &'a mut ClientResponsesManager<DataOrFin<FullTransaction>>,
        block_number: BlockNumber,
        storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            let mut block_body = BlockBody::default();
            let mut current_transaction_len = 0;
            let target_transaction_len = storage_reader
                .begin_ro_txn()?
                .get_block_header(block_number)?
                .expect("A header with number lower than the header marker is missing")
                .n_transactions;
            while current_transaction_len < target_transaction_len {
                let maybe_transaction = transactions_response_manager.next().await.ok_or(
                    ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }),
                )?;
                let Some(FullTransaction { transaction, transaction_output, transaction_hash }) =
                    maybe_transaction?.0
                else {
                    if current_transaction_len == 0 {
                        return Ok(None);
                    } else {
                        return Err(ParseDataError::BadPeer(BadPeerError::NotEnoughTransactions {
                            expected: target_transaction_len,
                            actual: current_transaction_len,
                            block_number: block_number.0,
                        }));
                    }
                };
                block_body.transactions.push(transaction);
                block_body.transaction_outputs.push(transaction_output);
                // TODO(eitan): Validate transaction hash from untrusted sources
                block_body.transaction_hashes.push(transaction_hash);
                current_transaction_len += 1;
            }
            Ok(Some((block_body, block_number)))
        }
        .boxed()
    }

    fn append_body(self, block_number: BlockNumber, block_body: BlockBody) -> StorageResult<Self> {
        let markers_table = self.open_table(&self.tables().markers)?;
        update_marker(self.txn(), &markers_table, block_number)?;

        if self.scope() != StorageScope::StateOnly {
            let events_table = self.open_table(&self.tables().events)?;

            let transaction_metadata_table =
                self.open_table(&self.tables().transaction_metadata)?;
            let file_offset_table = self.txn().open_table(&self.tables().file_offsets)?;

            write_transactions(
                &block_body,
                self.txn(),
                self.file_handlers(),
                &file_offset_table,
                &transaction_hash_to_idx_table,
                &transaction_metadata_table,
                &events_table,
                block_number,
            )?;
        }

        Ok(self)
    }
