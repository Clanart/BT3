Audit Report

## Title
Missing Bounds Check in `decodeSwapDeltas` Silently Returns Zero Amounts, Enabling Bad-Price Execution - (File: metric-periphery/contracts/libraries/MetricOmmSwapQuoteDecode.sol)

## Summary
`MetricOmmSwapQuoteDecode.decodeSwapDeltas` checks only the first 4 bytes of the revert payload against the expected selector but never verifies `reason.length >= 68`. When a malicious pool reverts with exactly 4 bytes matching `QuoteSwapResult.selector`, the assembly reads at offsets 36 and 68 fall outside the allocated buffer and silently return zero, causing the function to return `(0, 0, true)` ã a "matched" result with both amounts zeroed. Integrators that trust the quoter will set `amountOutMinimum = 0` and execute real swaps with no output protection, enabling complete loss of input principal.

## Finding Description
`decodeSwapDeltas` in `MetricOmmSwapQuoteDecode.sol` (L7ã26) performs only a 4-byte selector check before executing two unchecked assembly reads:

```solidity
if (bytes4(reason) != expectedSelector) return (0, 0, false);  // only 4-byte check
// no reason.length >= 68 guard
assembly ("memory-safe") {
    a0 := mload(add(reason, 36))   // reads reason[4..35]
    a1 := mload(add(reason, 68))   // reads reason[36..67]
}
```

A valid ABI-encoded two-`int256` error payload is `4 + 32 + 32 = 68 bytes`. When `reason.length == 4`, both `mload` calls read beyond the allocated memory region and return zero per EVM semantics. The function then returns `(0, 0, true)`.

This is called in two paths without any post-decode validation:
- `_quoteLiveSwap` (L295ã307): `if (matched) return (amount0Delta, amount1Delta);`
- `_quoteHypotheticalSwap` (L318ã331): `if (matched) return (amount0Delta, amount1Delta);`

The quoter accepts arbitrary pool addresses ã `_validateQuotePath` (L333ã343) only checks path length and connectivity, not factory registration. A malicious pool can revert `swap()` with `abi.encodePacked(QuoteSwapResult.selector)` (4 bytes), bypassing the only guard.

The sibling function `_decodeLiquidityProbeOrBubble` in `MetricOmmPoolLiquidityAdder.sol` (L209ã224) already has the correct guard: `if (reason.length < 68) revert UnexpectedRevertLength(reason.length);`, confirming the omission in `decodeSwapDeltas` is an oversight.

## Impact Explanation
This is a bad-price execution vulnerability. An integrator (DEX aggregator, routing engine) calling `quoteLiveExactInSingle` or `quoteHypotheticalExactInputSingle` against a malicious pool address receives `(amountIn=0, amountOut=0)`. If the integrator derives `amountOutMinimum` from the quote, it will be set to 0, and the subsequent real swap through a legitimate pool will execute with no slippage protection ã the trader receives zero output tokens while the full input is consumed. This is a direct loss of user principal meeting Critical/High severity thresholds.

## Likelihood Explanation
The attack is fully unprivileged: any caller can supply an arbitrary `pool` address to the quoter since no factory validation exists. Deploying a contract that reverts `swap()` with a 4-byte payload matching `QuoteSwapResult.selector` requires no special access. The selector is deterministic from the error signature. Automated integrators (aggregators, routing engines) that call the quoter before routing real swaps are the realistic victim class and may not independently validate pool addresses.

## Recommendation
Add a length guard immediately after the selector check, mirroring `_decodeLiquidityProbeOrBubble`:

```solidity
function decodeSwapDeltas(bytes memory reason, bytes4 expectedSelector)
    internal pure
    returns (int128 amount0Delta, int128 amount1Delta, bool matched)
{
    if (bytes4(reason) != expectedSelector) return (0, 0, false);
+   if (reason.length < 68) return (0, 0, false);  // selector(4) + int256(32) + int256(32)

    int256 a0;
    int256 a1;
    assembly ("memory-safe") {
        a0 := mload(add(reason, 36));
        a1 := mload(add(reason, 68));
    }
    amount0Delta = int128(a0);
    amount1Delta = int128(a1);
    return (amount0Delta, amount1Delta, true);
}
```

## Proof of Concept
1. Deploy `MaliciousPool` whose `swap()` executes:
   ```solidity
   bytes4 sel = MetricOmmSwapQuoter.QuoteSwapResult.selector;
   assembly { mstore(0, sel) revert(0, 4) }
   ```
2. Call `quoter.quoteLiveExactInSingle(address(maliciousPool), true, 1e18, 0)`.
3. `_quoteLiveSwap` catches the 4-byte revert in the `catch (bytes memory reason)` block.
