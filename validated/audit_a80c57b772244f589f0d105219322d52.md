### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezal of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in `syscall_impls.cairo` accepts any attacker-supplied class hash from the syscall request and writes it directly into `contract_state_changes` without verifying that the new class hash is actually declared in `contract_class_changes`. The OS itself acknowledges this missing check via a `TODO` comment. Any contract can invoke the `replace_class` syscall with an undeclared class hash, permanently corrupting its own on-chain class pointer. All funds held by that contract become permanently frozen because every subsequent call to it will fail at the proof level.

---

### Finding Description

In `execute_replace_class` (lines 877–916 of `syscall_impls.cairo`):

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;   // ← fully attacker-controlled

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash,             // ← written unconditionally
        storage_ptr=state_entry.storage_ptr,
        nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

`request.class_hash` is the value supplied by the executing contract through the syscall interface — it is fully user-controlled. The function performs no lookup into `contract_class_changes` to confirm that a compiled class with that hash has been declared. The `TODO` comment at line 898 is the OS developers' own acknowledgement that this invariant is missing. [1](#0-0) 

The downstream consequence is in `execute_entry_point` (lines 154–166 of `execute_entry_point.cairo`). When any future transaction calls the corrupted contract, the OS does:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // ← now the invalid hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,           // ← resolves to 0 (dict default) or unknown
);
```

`dict_read` on an undeclared key returns 0. `find_element` then searches for a compiled class fact with hash 0, which does not exist. `find_element` is a hint-assisted Cairo primitive that **asserts** the element is present; if it is absent the proof cannot be generated, making the contract permanently uncallable. [2](#0-1) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value, no valid proof can ever be produced for a transaction that calls it. Any ERC-20 tokens, ETH, or other assets stored in that contract's storage are permanently inaccessible. The state corruption is committed to the Merkle tree and cannot be reversed without a protocol-level intervention.

---

### Likelihood Explanation

The `replace_class` syscall is a standard, documented StarkNet syscall callable by any Sierra contract. The attack surface is:

1. **Self-inflicted by a buggy contract**: A contract with a logic flaw (e.g., an unguarded `replace_class` call whose argument is derived from user-supplied calldata) can be triggered by any unprivileged transaction sender to set an invalid class hash.
2. **Intentional griefing**: A contract author can deliberately deploy a contract that calls `replace_class(0xdeadbeef)` and then invoke it, permanently bricking the contract and any funds inside.

No privileged role, leaked key, or operator cooperation is required. The entry path is a standard user-submitted invoke transaction.

---

### Recommendation

Inside `execute_replace_class`, before writing the new class hash to `contract_state_changes`, assert that the class hash is present in `contract_class_changes`:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the check already performed implicitly in `execute_entry_point` and closes the gap the `TODO` comment identifies.

---

### Proof of Concept

1. **Deploy** a Sierra contract `VictimVault` that holds token balances and exposes a function `nuke()` whose body is:
   ```
   replace_class(0xdeadbeef_undeclared_hash)
   ```
2. **Submit** an invoke transaction calling `VictimVault::nuke()`.
3. The OS executes `execute_replace_class` with `class_hash = 0xdeadbeef`. No declared-class check is performed. `contract_state_changes` is updated: `VictimVault.class_hash ← 0xdeadbeef`.
4. The block is proven and committed. The state root now encodes `VictimVault` with class hash `0xdeadbeef`.
5. **Submit** any subsequent transaction calling `VictimVault` (e.g., a token transfer). The OS reaches `execute_entry_point`, calls `dict_read(contract_class_changes, 0xdeadbeef)` → returns `0`, then calls `find_element(..., key=0)`. No compiled class fact with hash `0` exists; the prover cannot satisfy the assertion. The block cannot be proven.
6. All funds inside `VictimVault` are permanently frozen. [3](#0-2) [4](#0-3)
