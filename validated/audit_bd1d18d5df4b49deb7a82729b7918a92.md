### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Bricking — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary felt as the new `class_hash` without verifying that the value corresponds to a class that has actually been declared in `contract_class_changes`. This is a state-transition bypass: the OS permits a contract to transition to a class-hash state that is invalid (undeclared), analogous to the reference bug where `joinGame` permitted joining in the `Ongoing` state when only `Created` should have been allowed. Any contract that calls `replace_class` with an undeclared hash becomes permanently uncallable, freezing all funds it holds.

---

### Finding Description

In `execute_replace_class` (syscall_impls.cairo, lines 878–916), the OS reads the requested `class_hash` from the syscall request and immediately writes it into `contract_state_changes` with no check that the hash exists in `contract_class_changes`:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges the missing check. The OS codebase itself flags this as an unimplemented invariant.

When a subsequent transaction calls into the bricked contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
``` [2](#0-1) 

If `class_hash` is undeclared, `dict_read` returns `0` (`UNINITIALIZED_CLASS_HASH`). `find_element` then searches for a compiled class with hash `0`. Since no such class exists, the prover cannot satisfy the Cairo constraint, making the block unprovable. The sequencer is therefore forced to exclude all future calls to the bricked contract, permanently locking any funds it holds.

The `UNINITIALIZED_CLASS_HASH` constant is defined as `0`: [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

A contract that calls `replace_class(undeclared_hash)` has its `class_hash` field in `contract_state_changes` set to an undeclared value. The state update is committed to the Patricia Merkle Tree via `compute_contract_state_commitment`: [4](#0-3) 

Once committed, the contract's class_hash is permanently invalid on-chain. Any ERC-20 token balance, ETH, or other asset stored in the contract's storage becomes inaccessible forever, because no valid proof can be generated for a block that executes a call into that contract.

---

### Likelihood Explanation

**Medium-High.** The `replace_class` syscall is a standard, documented StarkNet syscall available to any Sierra contract. An attacker needs only to:
1. Deploy a contract (permissionless via `deploy` syscall or `deploy_account` transaction).
2. Have that contract call `replace_class` with any felt value that is not a declared class hash.

No privileged role, leaked key, or external dependency is required. The attack is reachable by any unprivileged contract deployer. The only friction is that the attacker must pay gas for the deploy and the `replace_class` call. A malicious contract targeting a victim protocol (e.g., a DeFi vault) could trigger this via a reentrancy or callback if the victim contract delegates class replacement to an untrusted caller.

---

### Recommendation

Add a validation step inside `execute_replace_class` that reads `contract_class_changes` and asserts the new `class_hash` maps to a non-zero compiled class hash before updating the contract state:

```cairo
func execute_replace_class{..., contract_class_changes: DictAccess*, ...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // Enforce that the new class hash is declared.
    let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
    assert_not_zero(compiled_class_hash);  // Revert if class is not declared.

    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}
    ...
}
```

This mirrors the invariant already enforced at declaration time in `execute_declare_transaction`, where `assert_not_zero(compiled_class_hash)` is called before writing to `contract_class_changes`: [5](#0-4) 

---

### Proof of Concept

1. **Attacker deploys a malicious contract** `MaliciousContract` whose constructor or any external function calls `replace_class(0xDEADBEEF)` where `0xDEADBEEF` is not a declared class hash.

2. **Attacker targets a victim contract** (e.g., a token vault holding user funds) that has a callback or upgrade mechanism that calls into `MaliciousContract`.

3. **`MaliciousContract` calls `replace_class(0xDEADBEEF)`** on itself (or, if the victim delegates class replacement, on the victim).

4. **The OS processes `execute_replace_class`**: no validation of `0xDEADBEEF` against `contract_class_changes` is performed. The state entry for the contract is updated with `class_hash = 0xDEADBEEF`.

5. **State is committed** via `state_update` → `compute_contract_state_commitment`. The invalid class_hash is now part of the on-chain state root.

6. **Any subsequent call to the bricked contract** causes `execute_entry_point` to call `dict_read(contract_class_changes, 0xDEADBEEF)` → returns `0` → `find_element(..., key=0)` → no compiled class found → proof generation fails.

7. **The sequencer cannot include any transaction that calls the bricked contract.** All funds in the contract are permanently frozen. [6](#0-5)
