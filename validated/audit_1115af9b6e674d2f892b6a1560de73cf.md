### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not validate that the caller-supplied `class_hash` corresponds to a previously declared contract class. The OS unconditionally overwrites the contract's class hash with any arbitrary felt value. If the new hash is undeclared, the contract becomes permanently non-executable, freezing all funds it holds. The codebase itself acknowledges this gap with a `TODO` comment.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The `class_hash` field is taken directly from `request.class_hash` — a caller-controlled value — and written into `contract_state_changes` without any check that the hash exists in `contract_class_changes` (declared in the current block) or in the pre-existing committed state. The `TODO` comment at line 898 explicitly acknowledges this missing check.

Compare this to the `execute_declare_transaction` path, which enforces `prev_value=0` and validates the class hash pre-image via `finalize_class_hash`: [2](#0-1) 

No equivalent guard exists in `execute_replace_class`.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

If a contract's class hash is replaced with an undeclared value:

1. Every subsequent call to that contract will fail at the class-lookup stage, because no compiled class exists for the hash.
2. The contract's storage and balance are permanently inaccessible — there is no recovery path in the OS.
3. Any ERC-20 tokens, ETH, or other assets held in the contract's storage are frozen forever.

This maps directly to the external report's impact: just as locking tokens before validating the destination handler causes permanent fund loss, replacing a class hash without validating it is declared causes permanent contract paralysis and fund loss.

---

### Likelihood Explanation

The `replace_class` syscall is reachable by any unprivileged contract deployer or caller:

1. A contract deployer writes a contract whose logic calls `replace_class(arbitrary_felt)`.
2. Any user invokes a function on that contract (or the deployer self-invokes it).
3. The sequencer includes the transaction; the OS executes it without the missing guard.
4. The proof is valid — the OS accepted the state transition — so the broken class hash is committed on-chain.

No privileged role, leaked key, or operator collusion is required. The attack is also reachable accidentally: a buggy contract that computes a class hash incorrectly (e.g., off-by-one in a hash chain) will silently brick itself and freeze user funds.

---

### Recommendation

Before writing the new `StateEntry`, verify that `class_hash` is present in `contract_class_changes` (declared in the current block) or in the committed contract-class trie. Concretely, add a dict-read against `contract_class_changes` and assert the result is non-zero:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the guard already present in `execute_declare_transaction` (`prev_value=0` enforces uniqueness; the analogous check here enforces existence).

---

### Proof of Concept

1. Deploy contract `A` holding 1000 tokens, with an entry point `brick()` that executes:
   ```
   replace_class(0xdeadbeef)   // 0xdeadbeef is never declared
   ```
2. Call `brick()` on contract `A`.
3. The OS executes `execute_replace_class`:
   - `class_hash = 0xdeadbeef`
   - No declared-class check is performed (the `TODO` path is skipped).
   - `contract_state_changes[A].class_hash` is updated to `0xdeadbeef`.
4. The block is proven and committed on-chain.
5. All subsequent calls to contract `A` fail — no compiled class exists for `0xdeadbeef`.
6. The 1000 tokens in `A`'s storage are permanently frozen with no recovery mechanism. [3](#0-2)

    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
