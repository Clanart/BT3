### Title
Mandatory fee-token transfer in `charge_fee` uses `non_reverting_select_execute_entry_point_func` with no revert handling, allowing a transaction sender to brick block proving — (File: `execution/transaction_impls.cairo`)

---

### Summary

The `charge_fee` function in `transaction_impls.cairo` executes the ERC-20 fee-token `transfer` via `non_reverting_select_execute_entry_point_func`, which unconditionally asserts `is_reverted = 0`. If the fee-token transfer reverts — for example because the user's `__execute__` function drained the fee-token balance before `charge_fee` runs — the Cairo assertion fails, the block proof is invalid, and the network cannot finalize any further blocks.

---

### Finding Description

`charge_fee` is invoked **after** the main transaction body (`__execute__`, constructor, or `__validate_declare__`) for every invoke, deploy-account, and declare transaction. [1](#0-0) 

Inside `charge_fee`, the fee-token `transfer` entry point is dispatched through `non_reverting_select_execute_entry_point_func`: [2](#0-1) 

`non_reverting_select_execute_entry_point_func` is defined as:

```cairo
func non_reverting_select_execute_entry_point_func{...}(...) -> (...) {
    let revert_log = init_revert_log();
    let (is_reverted, retdata_size, retdata, is_deprecated) = select_execute_entry_point_func{
        revert_log=revert_log
    }(block_context=block_context, execution_context=execution_context);
    assert is_reverted = 0;          // <-- hard assertion, no error path
    return (retdata_size, retdata, is_deprecated);
}
``` [3](#0-2) 

`select_execute_entry_point_func` can legitimately return `is_reverted = 1` when the callee reverts: [4](#0-3) 

There is **no try/catch, no graceful fallback, and no alternative code path** if the fee-token transfer reverts. The `assert is_reverted = 0` statement is a Cairo constraint: a failing assertion produces an unsatisfiable constraint system, making the entire block proof invalid.

The fee transfer is called with `caller_address = tx_info.account_contract_address` (the user's own account): [5](#0-4) 

A standard ERC-20 `transfer` reverts when the caller's balance is insufficient. Because `charge_fee` runs **after** `__execute__`, a user whose `__execute__` body transfers all fee tokens to a third address will have a zero balance when `charge_fee` runs, causing the transfer to revert and the assertion to fail.

The same `non_reverting_select_execute_entry_point_func` is also used for `__validate__`, `__validate_deploy__`, and `__validate_declare__`, but those run before `__execute__` and cannot be drained by the user's own execution body. The `charge_fee` call site is uniquely vulnerable because it runs last, after the user-controlled execution has already modified state.

---

### Impact Explanation

A single transaction in a block that causes `charge_fee` to revert makes the **entire block unprovable**. StarkNet's L1 settlement depends on valid STARK proofs; an unprovable block means no further state updates can be committed to Ethereum. This constitutes a **total network shutdown** — no new transactions can be confirmed until the block is discarded and re-sequenced, which itself requires out-of-band operator intervention.

---

### Likelihood Explanation

The sequencer is expected to simulate the full transaction (including fee charging) before inclusion. However:

1. **Simulation/execution discrepancy**: The sequencer simulation runs off-chain with its own state snapshot. If the simulation does not replay `charge_fee` against the post-`__execute__` state, the drain will go undetected.
2. **No OS-level guard**: The OS itself has zero protection. If the sequencer makes any mistake — a bug, a race condition, or an incomplete simulation — the block proof fails.
3. **Unprivileged entry**: Any transaction sender can craft this scenario with no special access. The user only needs to hold enough fee tokens to pass the sequencer's pre-execution balance check, then drain them inside `__execute__`.

Likelihood is **low-to-medium** (sequencer simulation is the primary mitigation, but it is not enforced by the OS).

---

### Recommendation

Replace `non_reverting_select_execute_entry_point_func` in `charge_fee` with `select_execute_entry_point_func` and handle the `is_reverted = 1` case explicitly — for example, by marking the transaction as failed, reverting its state changes, and skipping fee collection (or collecting a minimum penalty fee from a pre-locked deposit). This mirrors how reverted invoke transactions are already handled elsewhere in the OS and prevents a single bad transaction from making an entire block unprovable.

---

### Proof of Concept

1. User account holds exactly `F` fee tokens — enough to satisfy the sequencer's declared `max_fee` check.
2. User submits an invoke transaction. The `__execute__` body calls `transfer(recipient=attacker_address, amount=F)` on the fee token contract, draining the account to zero.
3. Sequencer simulates the transaction, observes balance `F` at simulation time, and includes the transaction in a block.
4. OS executes `__execute__`: fee token balance is now `0`.
5. OS calls `charge_fee` → calls `non_reverting_select_execute_entry_point_func` → dispatches `transfer(sequencer, actual_fee)` on the fee token.
6. Fee token's `transfer` reverts (balance = 0 < actual_fee).
7. `select_execute_entry_point_func` returns `is_reverted = 1`.
8. `assert is_reverted = 0` fails — unsatisfiable Cairo constraint.
9. The block proof is invalid. The network cannot confirm any new transactions.

func select_execute_entry_point_func{
    range_check_ptr,
    remaining_gas: felt,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    revert_log: RevertLogEntry*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, execution_context: ExecutionContext*) -> (
    is_reverted: felt, retdata_size: felt, retdata: felt*, is_deprecated: felt
) {
    alloc_locals;
    // TODO(Yoni): SIERRA_GAS_MODE - move back inside `execute_entry_point` functions.
    %{ EnterCall %}

    local is_deprecated;
    %{ CheckIsDeprecated %}
    // Note that the class_hash is validated in both the `if` and `else` cases, so a malicious
    // prover won't be able to produce a proof if guesses the wrong case.
    if (is_deprecated != FALSE) {
        let (is_reverted, retdata_size, retdata: felt*) = deprecated_execute_entry_point(
            block_context=block_context, execution_context=execution_context
        );
        return (
            is_reverted=is_reverted, retdata_size=retdata_size, retdata=retdata, is_deprecated=1
        );
    }

    // TODO(Yoni): SIERRA_GAS_MODE - remove once all Cairo 1 contracts run with Sierra gas mode.
    local caller_remaining_gas = remaining_gas;
    local is_sierra_gas_mode;
    %{ IsSierraGasMode %}
    if (is_sierra_gas_mode != FALSE) {
        tempvar inner_remaining_gas = remaining_gas;
    } else {
        // Run with high enough gas to avoid out-of-gas.
        tempvar inner_remaining_gas = DEFAULT_INITIAL_GAS_COST;
    }
    %{ DebugExpectedInitialGas %}

    let (is_reverted, retdata_size, retdata) = execute_entry_point{
        remaining_gas=inner_remaining_gas
    }(block_context=block_context, execution_context=execution_context);

    if (is_sierra_gas_mode != FALSE) {
        tempvar remaining_gas = inner_remaining_gas;
    } else {
        // Do not count Sierra gas for the caller in this case.
        tempvar remaining_gas = caller_remaining_gas;
    }
    return (is_reverted=is_reverted, retdata_size=retdata_size, retdata=retdata, is_deprecated=0);
}
