### Title
Missing Class Hash Validation in `execute_replace_class` Enables Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts and commits any caller-supplied class hash — including zero or any undeclared hash — without verifying it corresponds to a declared contract class. Once committed, the contract's class hash is permanently invalid. Every subsequent call to that contract causes the OS to attempt a `find_element` lookup for a non-existent compiled class, producing an irrecoverable hint failure that makes the OS proof invalid for any block containing such a call. Funds held by the frozen contract become permanently inaccessible.

---

### Finding Description

In `execute_replace_class`, the OS reads `class_hash` directly from the user-supplied syscall request and writes it into the contract state with no validation: [1](#0-0) 

The critical lines are:

```cairo
let class_hash = request.class_hash;
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}
tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment at line 898 is the OS developers' own acknowledgment that this check is missing. No Cairo-level assertion prevents `class_hash = 0` or any undeclared value from being stored.

Once the invalid class hash is committed to state, any subsequent call to the contract reaches `execute_entry_point`: [2](#0-1) 

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // 0 or undeclared hash
);
// compiled_class_hash == 0 (default dict value)

let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,           // 0 — not present in the bundle
);
```

`find_element` uses a hint to locate the element. When the key does not exist in `compiled_class_facts_bundle`, the hint fails. A hint failure in the Cairo VM makes the entire OS proof invalid — the block cannot be proven. The sequencer must exclude every transaction that touches the frozen contract, making the contract and its funds permanently inaccessible.

---

### Impact Explanation

Any contract that calls `replace_class` with an invalid class hash (zero or undeclared) is permanently frozen. No entry point can ever be executed on it again, because every attempt causes an irrecoverable OS-level hint failure. All funds (tokens, ETH, etc.) held by the contract are permanently locked with no recovery path. This satisfies **Critical — Permanent freezing of funds**.

---

### Likelihood Explanation

The `replace_class` syscall is available to every contract without privilege restrictions. Realistic triggers include:

- A contract upgrade mechanism that passes an incorrect class hash due to a logic bug.
- An attacker who exploits a contract's upgrade function (e.g., via reentrancy or access-control flaw) to supply `class_hash = 0`.
- A developer mistake during testing on a live network.

The OS developers themselves flagged this gap with a TODO comment, confirming it is a known unimplemented safety check.

---

### Recommendation

Inside `execute_replace_class`, before writing the new state entry, assert that `class_hash` is non-zero and that it exists as a key in `contract_class_changes` (i.e., it has been declared in the current or a prior block). If the check fails, write a failure response via `write_failure_response` and return, rather than committing the invalid state.

---

### Proof of Concept

1. Deploy contract `C` that holds user funds and exposes an upgrade function that calls `replace_class(0)` (or any undeclared hash).
2. Invoke the upgrade function. `execute_replace_class` accepts `class_hash = 0` without validation and commits it to `contract_state_changes`. [3](#0-2) 
3. The block is proven successfully — the invalid state is now canonical.
4. In the next block, any transaction calling `C` reaches `execute_entry_point` with `execution_context.class_hash = 0`. [4](#0-3) 
5. `dict_read` returns `compiled_class_hash = 0`. `find_element` cannot locate a compiled class with key `0` in `compiled_class_facts_bundle` and the hint fails.
6. The OS proof for that block is invalid. The sequencer must drop every transaction interacting with `C`.
7. `C`'s funds are permanently inaccessible — no `replace_class` recovery is possible because the contract itself cannot be executed.
