### Title
Missing shard-id binding in `set_state_header` allows cross-shard state substitution — (`chain/chain/src/state_sync/adapter.rs`)

### Summary

`ChainStateSyncAdapter::set_state_header` accepts a `shard_id` parameter and a `ShardStateSyncResponseHeader` but never checks that `header.chunk.shard_id() == shard_id`. A peer serving state-sync data can supply shard B's real chunk (with its real Merkle proof) while the caller passes `shard_id = A`. All five validation steps pass, the header is stored under `StateHeaderKey(A, sync_hash)`, and subsequent `apply_state_part` calls install shard B's trie data into shard A's `ShardUId`, corrupting shard A's committed state root.

### Finding Description

**Step 1-2 — `validate_chunk_proofs`** checks the chunk's own hash, `tx_root`, and `outgoing_receipts_root`. It does not inspect `chunk.shard_id()`. [1](#0-0) 

**Step 3a — `verify_path`** verifies that `ChunkHashHeight(chunk.chunk_hash(), chunk.height_included())` is a leaf in the `chunk_headers_root` Merkle tree. Because `verify_path` only reconstructs the root from the path and leaf — it does **not** check the leaf's position/index — a valid proof for shard B's chunk at index B passes even when `shard_id = A` is expected. [2](#0-1) [3](#0-2) 

Compare with `verify_path_with_index`, which does enforce position — but is not used here: [4](#0-3) 

**Step 3b** — same `verify_path` (no index check) for `prev_chunk_header`. [5](#0-4) 

**Step 4 — incoming receipt proofs** use the caller's `shard_id` parameter (A), not the chunk's embedded shard id. An attacker supplies the real public receipt proofs for shard A (observable on-chain), which verify correctly. [6](#0-5) 

**Step 5 — `validate_state_root_node`** only checks `hash(node.data) == state_root`. The attacker supplies shard B's real `state_root_node`, which is consistent with shard B's `prev_state_root`. No shard binding. [7](#0-6) 

**Storage** — the header is written under `StateHeaderKey(shard_id_A, sync_hash)` containing shard B's chunk: [8](#0-7) 

**`apply_state_part`** derives `shard_uid` from the caller's `shard_id` (A), so shard B's trie nodes are inserted into shard A's column: [9](#0-8) 

**`set_state_finalize`** then calls `apply_chunk` with `shard_uid = A` but `prev_state_root = shard_B_root`, producing a `ChunkExtra` for shard A whose `state_root` is shard B's post-execution root: [10](#0-9) 

### Impact Explanation

A victim node completing state sync for shard A will have its committed `ChunkExtra.state_root` for shard A set to shard B's state root. All subsequent chunk applications for shard A will read from the wrong trie, producing incorrect execution results, wrong receipts, and divergence from the canonical chain. The node cannot self-heal without a full re-sync.

### Likelihood Explanation

The attacker must be a peer from which the victim downloads state-sync headers (peer-to-peer state sync path). The attacker needs only read access to the canonical chain (public data) to construct all required proofs. No validator, chunk-producer, or operator privilege is required. The attack is silent — the victim node accepts the header without error.

### Recommendation

Add an explicit shard-id binding check immediately after extracting the chunk, before any other validation:

```rust
let chunk = shard_state_header.cloned_chunk();
if chunk.shard_id() != shard_id {
    return Err(Error::Other(
        "set_shard_state failed: chunk shard_id does not match requested shard_id".into(),
    ));
}
```

Additionally, replace `verify_path` with `verify_path_with_index` in step 3a, passing the shard index derived from `shard_id` and the epoch's shard layout, so the Merkle proof is bound to the correct position in the tree. [11](#0-10) 

### Proof of Concept

1. Set up a two-shard test-loop environment; let both shards accumulate state.
2. Compute `sync_hash` and call `compute_state_response_header(shard_id=1, sync_hash)` on the honest node to obtain `header_B` (shard 1's real header with its real Merkle proof).
3. On the victim node call `set_state_header(shard_id=0, sync_hash, header_B)` — all five checks pass.
4. Serve shard 1's state parts via `set_state_part(shard_id=0, ...)` — they validate against shard 1's `prev_state_root` stored in the header.
5. Call `apply_state_part(shard_id=0, state_root=shard1_root, ...)` — shard 1's trie nodes are written under shard 0's `ShardUId`.
6. Call `set_state_finalize(shard_id=0, sync_hash)`.
7. Assert: `ChunkExtra.state_root` for shard 0 equals shard 1's pre-sync state root — confirming cross-shard state substitution.

pub fn validate_chunk_proofs(
    chunk: &ShardChunk,
    epoch_manager: &dyn EpochManagerAdapter,
) -> Result<bool, Error> {
    let correct_chunk_hash = chunk.compute_header_hash();

    // 1. Checking chunk.header.hash
    let header_hash = chunk.header_hash();
    if header_hash != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }

    // 2. Checking that chunk body is valid
    // 2a. Checking chunk hash
    if chunk.chunk_hash() != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }
    let height_created = chunk.height_created();
    let outgoing_receipts_root = chunk.prev_outgoing_receipts_root();
    let (transactions, receipts) = (chunk.to_transactions(), chunk.prev_outgoing_receipts());

    // 2b. Checking that chunk transactions are valid
    let (tx_root, _) = merklize(transactions);
    if &tx_root != chunk.tx_root() {
        byzantine_assert!(false);
        return Ok(false);
    }
    // 2c. Checking that chunk receipts are valid
    if height_created == 0 {
        return Ok(receipts.is_empty() && outgoing_receipts_root == &CryptoHash::default());
    } else {
        let shard_layout = {
            let prev_block_hash = chunk.prev_block_hash();
            epoch_manager.get_shard_layout_from_prev_block(&prev_block_hash)?

        let outgoing_receipts_hashes = Chain::build_receipts_hashes(receipts, &shard_layout)?;
        let (receipts_root, _) = merklize(&outgoing_receipts_hashes);
        if &receipts_root != outgoing_receipts_root {
            byzantine_assert!(false);
            return Ok(false);
        }
    }
    Ok(true)
