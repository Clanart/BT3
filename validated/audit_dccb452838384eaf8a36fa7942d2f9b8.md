### Title
Deprecated Declared Classes Structurally Omitted from State Diff Commitment and Block Hash — (`crates/blockifier/src/state/cached_state.rs`)

### Summary

The `From<CommitmentStateDiff> for ThinStateDiff` conversion hard-codes `deprecated_declared_classes: Vec::new()`. Because `CommitmentStateDiff` (produced by the blockifier's `StateMaps`) has no field for deprecated (Cairo 0) class declarations, any block that includes a deprecated-declare transaction will produce a `ThinStateDiff` with an empty `deprecated_declared_classes` list. That list is a direct input to both `calculate_state_diff_hash` and `ThinStateDiff::len()`, so the resulting `state_diff_commitment` and the `concatenated_counts` packed field (which encodes `state_diff_length`) inside the block hash are both wrong.

### Finding Description

**Root cause — structural omission in the conversion:**

`CommitmentStateDiff` is built from `StateMaps`, which tracks `class_hashes`, `compiled_class_hashes`, `storage`, and `nonces` but has no `deprecated_declared_classes` field. The conversion to `ThinStateDiff` therefore always produces an empty list:

```rust
// crates/blockifier/src/state/cached_state.rs
impl From<CommitmentStateDiff> for ThinStateDiff {
    fn from(commitment_state_diff: CommitmentStateDiff) -> Self {
        Self {
            deployed_contracts: commitment_state_diff.address_to_class_hash,
            storage_diffs: commitment_state_diff.storage_updates,
            class_hash_to_compiled_class_hash: commitment_state_diff
                .class_hash_to_compiled_class_hash,
            nonces: commitment_state_diff.address_to_nonce,
            // TODO(AlonH): Remove this when the structure of storage diffs changes.
            deprecated_declared_classes: Vec::new(),   // ← always empty
        }
    }
}
``` [1](#0-0) 

**Propagation into commitment calculation:**

`BlockExecutionArtifacts::new` converts `CommitmentStateDiff` to `ThinStateDiff` and passes it directly to `calculate_block_commitments`:

```rust
let (header_commitments, measurements) = calculate_block_commitments(
    &transactions_data,
    ThinStateDiff::from(commitment_state_diff.clone()),  // deprecated_declared_classes = []
    l1_da_mode,
    &block_info.starknet_version,
)
.await;
``` [2](#0-1) 

**Two commitment fields are corrupted:**

Inside `calculate_block_commitments`, the empty list affects:

1. `state_diff.len()` — used to build `concatenated_counts` (the packed `tx_count | event_count | state_diff_length | DA_mode` field chained into the block hash).
2. `calculate_state_diff_hash(&state_diff)` — which explicitly chains `deprecated_declared_classes` via `chain_deprecated_declared_classes`. [3](#0-2) [4](#0-3) 

`ThinStateDiff::len()` explicitly counts `deprecated_declared_classes.len()`: [5](#0-4) 

Both corrupted values are chained into `calculate_block_hash`: [6](#0-5) 

### Impact Explanation

For any block that contains one or more deprecated (Cairo 0) declare transactions:

- `state_diff_commitment` is computed over a `ThinStateDiff` that is missing the declared class hashes → wrong `StateDiffCommitment` stored and broadcast.
- `concatenated_counts` encodes a `state_diff_length` that is too small by the number of deprecated declared classes → wrong packed field in the block hash.
- The final `block_hash` (and `PartialBlockHash` used for consensus commitment) is therefore wrong.

This matches **Critical — Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input**, and **Critical — Incorrect fee, gas, bouncer, resource accounting** (the `state_diff_length` in `concatenated_counts` is an authoritative on-chain value).

### Likelihood Explanation

Deprecated declare transactions are valid Starknet transactions. The `native_blockifier` path and `apollo_rpc_execution` path both explicitly handle `deprecated_declared_classes`, confirming the system is designed to accept them. Any user who submits a Cairo 0 declare transaction that is sequenced into a block triggers the corruption without any privileged access.

### Recommendation

1. Add a `deprecated_declared_classes: Vec<ClassHash>` field to `CommitmentStateDiff` (and correspondingly to `StateMaps`).
2. Populate it during deprecated-declare transaction execution in the blockifier.
3. Remove the hard-coded `Vec::new()` in `From<CommitmentStateDiff> for ThinStateDiff` and propagate the real list.
4. Add a test that sequences a deprecated declare transaction and asserts that `calculate_state_diff_hash` and `ThinStateDiff::len()` produce values consistent with the on-chain formula.

### Proof of Concept

1. Submit a `DeclareV0` or `DeclareV1` (Cairo 0) transaction to the sequencer gateway.
2. The batcher executes it; `finalize_block` produces a `CommitmentStateDiff` with no `deprecated_declared_classes` field.
3. `BlockExecutionArtifacts::new` calls `ThinStateDiff::from(commitment_state_diff)` → `deprecated_declared_classes = []`.
4. `calculate_block_commitments` computes `state_diff.len()` = N (missing the K deprecated declared classes) and `calculate_state_diff_hash` over the truncated diff.
5. The resulting `PartialBlockHash` / `BlockHash` differs from the value that a correct implementation (or the SNOS verifier) would compute for the same block, breaking proof verification and consensus agreement on the block hash.

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
