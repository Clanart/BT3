### Title
Missing `chunk.shard_id() == shard_id` Guard in `set_state_header` Allows Unprivileged Peer to Corrupt Cached State Header and Permanently Block State Sync - (`chain/chain/src/state_sync/adapter.rs`)

---

### Summary

`ChainStateSyncAdapter::set_state_header` accepts a `ShardStateSyncResponseHeader` for shard A but never verifies that the embedded `chunk.shard_id()` equals the requested `shard_id`. An unprivileged peer can respond to a state header request for shard A with shard B's legitimate on-chain chunk (paired with shard B's valid Merkle proof), pass every validation gate, and cause the syncing node to permanently store a corrupted `StateHeaderKey(A, H)` whose `chunk.prev_state_root()` is shard B's state root. All subsequent `set_state_part` calls for shard A then validate parts against shard B's state root, causing every legitimate part to fail. Because `ensure_shard_header` returns the cached (corrupted) header on the next call without re-downloading, the node is permanently stuck in state sync for shard A until the DB is manually cleared.

---

### Finding Description

**Entrypoint**: `set_state_header` in `chain/chain/src/state_sync/adapter.rs`, called from the chain actor after `StateSyncDownloader::ensure_shard_header` receives a peer response.

**Validation gates and why each passes for a cross-shard substitution:**

**Gate 1 — `validate_chunk_proofs`** (`chain/chain/src/validate.rs` lines 22–66):
Checks hash self-consistency, `tx_root`, and `receipts_root` of the chunk body. It takes no `shard_id` parameter and never compares `chunk.shard_id()` to anything external. Shard B's legitimate on-chain chunk passes trivially. [1](#0-0) 

**Gate 2 — `verify_path` for chunk inclusion** (`adapter.rs` lines 394–403):
```rust
if !verify_path(
    *sync_prev_block_header.chunk_headers_root(),
    shard_state_header.chunk_proof(),
    &ChunkHashHeight(chunk.chunk_hash().clone(), chunk.height_included()),
) {
```
Verifies that the chunk's hash appears *somewhere* in the Merkle tree rooted at `chunk_headers_root`. It does not verify *which leaf index* (i.e., which shard position) the proof corresponds to. Shard B's chunk with shard B's Merkle proof passes this check. [2](#0-1) 

**Gate 3 — `verify_path` for prev_chunk inclusion** (lines 416–425): Same structural gap — position is not checked. [3](#0-2) 

**Gate 4 — Receipt proof validation** (lines 446–510):
Uses the `shard_id` *parameter* (A), not `chunk.shard_id()`, to compute `receipts_hash = CryptoHash::hash_borsh(ReceiptList(shard_id, receipts))`. The attacker supplies the real on-chain receipt proofs for shard A, which are publicly available. This gate passes independently of which chunk is embedded. [4](#0-3) 

**Gate 5 — `validate_state_root_node`** (lines 512–523):
```rust
let chunk_inner = chunk.take_header().take_inner();
self.runtime_adapter.validate_state_root_node(
    shard_state_header.state_root_node(),
    chunk_inner.prev_state_root(),   // ← shard B's prev_state_root
)
```
Validates the supplied `state_root_node` against `chunk_inner.prev_state_root()`. Since the attacker supplies shard B's actual state root node (consistent with shard B's `prev_state_root`), this passes. [5](#0-4) 

**Storage** (lines 525–529):
```rust
let key = borsh::to_vec(&StateHeaderKey(shard_id, sync_hash)).unwrap();
store_update.set_ser(DBCol::StateHeaders, &key, &shard_state_header);
```
The header is stored under key `(A, H)` but contains shard B's chunk. The invariant `StateHeaderKey(A, H).chunk.shard_id() == A` is violated. [6](#0-5) 

---

### Impact Explanation

**Exact corrupted value**: `StateHeaderKey(A, H).chunk.prev_state_root()` = shard B's `prev_state_root` (instead of shard A's).

**`set_state_part` consequence** (lines 541–553):
```rust
let shard_state_header = self.get_state_header(shard_id, sync_hash)?;
let chunk = shard_state_header.take_chunk();
let state_root = *chunk.take_header().take_inner().prev_state_root();
// state_root is now shard B's prev_state_root
self.runtime_adapter.validate_state_part(shard_id, &state_root, part_id, part)
```
Every legitimate state part for shard A is validated against shard B's state root and rejected. [7](#0-6) 

**`set_state_finalize` consequence** (`chain_update.rs` lines 519–542):
`apply_chunk` is called with `chunk_header.prev_state_root()` (shard B's root) as the storage root, shard B's transactions, and shard A's receipts — a semantically invalid state transition for shard A. [8](#0-7) 

**Persistence**: `ensure_shard_header` checks storage before downloading. Once the corrupted header is written, it is returned immediately on every subsequent call, preventing any retry from fetching a correct header. [9](#0-8) 

---

### Likelihood Explanation

The attack requires only a connected peer (no validator key, no special role). All inputs are publicly available on-chain: shard B's chunk body, its Merkle proof from `sync_prev_block.chunk_headers_root`, shard A's receipt proofs, and shard B's state root node. The attack is deterministic and requires no brute force. Any node that enters state sync for shard A while connected to a malicious peer is vulnerable.

---

### Recommendation

Add an explicit shard identity check immediately after `validate_chunk_proofs` in `set_state_header`:

```rust
if chunk.shard_id() != shard_id {
    return Err(Error::Other(
        "set_shard_state failed: chunk.shard_id() does not match requested shard_id".into(),
    ));
}
```

Similarly add the same check for `prev_chunk_header.shard_id()` when present. Optionally, also verify that the Merkle proof leaf index corresponds to the correct shard position (i.e., `prev_shard_layout.get_shard_index(shard_id)` matches the proof path direction), which would close the positional ambiguity in `verify_path` independently.

---

### Proof of Concept

A property test in `chain/chain/src/state_sync/adapter.rs` (or a dedicated integration test) would:

1. Set up a two-shard epoch with sync hash H.
2. Compute a legitimate `ShardStateSyncResponseHeader` for shard B using `compute_state_response_header(B, H)`.
3. Obtain the legitimate receipt proofs for shard A.
4. Construct a crafted header: `chunk` = shard B's chunk, `chunk_proof` = shard B's Merkle proof, `incoming_receipts_proofs`/`root_proofs` = shard A's real proofs, `state_root_node` = shard B's state root node.
5. Call `set_state_header(A, H, crafted_header)`.
6. Assert the call returns `Err(...)` — currently it returns `Ok(())`, confirming the bug.
7. Assert that `get_state_header(A, H).chunk.shard_id() == A` — currently it equals B, confirming the invariant violation.

pub fn validate_chunk_proofs(
    chunk: &ShardChunk,
    epoch_manager: &dyn EpochManagerAdapter,
) -> Result<bool, Error> {
    let correct_chunk_hash = chunk.compute_header_hash();

    // 1. Checking chunk.header.hash
    let header_hash = chunk.header_hash();
    if header_hash != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }

    // 2. Checking that chunk body is valid
    // 2a. Checking chunk hash
    if chunk.chunk_hash() != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }
    let height_created = chunk.height_created();
    let outgoing_receipts_root = chunk.prev_outgoing_receipts_root();
    let (transactions, receipts) = (chunk.to_transactions(), chunk.prev_outgoing_receipts());

    // 2b. Checking that chunk transactions are valid
    let (tx_root, _) = merklize(transactions);
    if &tx_root != chunk.tx_root() {
        byzantine_assert!(false);
        return Ok(false);
    }
    // 2c. Checking that chunk receipts are valid
    if height_created == 0 {
        return Ok(receipts.is_empty() && outgoing_receipts_root == &CryptoHash::default());
    } else {
        let shard_layout = {
            let prev_block_hash = chunk.prev_block_hash();
            epoch_manager.get_shard_layout_from_prev_block(&prev_block_hash)?
        };
        let outgoing_receipts_hashes = Chain::build_receipts_hashes(receipts, &shard_layout)?;
        let (receipts_root, _) = merklize(&outgoing_receipts_hashes);
        if &receipts_root != outgoing_receipts_root {
            byzantine_assert!(false);
            return Ok(false);
        }
    }
    Ok(true)
