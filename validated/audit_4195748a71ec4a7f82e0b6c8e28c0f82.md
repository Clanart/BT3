### Title
Missing Contract Existence Check Before Void-Returning `mint()` Calls in `finTransfer` Silently Consumes Nonce Without Minting Tokens — (`evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.finTransfer()` marks the destination nonce as used **before** calling `mint()` on either a `customMinter` or a `bridgeToken` contract. Both `ICustomMinter.mint()` and `IBridgeToken.mint()` return `void`. Per the EVM specification (and the Solidity documentation warning), a call to an address with no deployed code returns `(true, "")`. Because the return type is `void`, Solidity performs no ABI-decoding and does not revert. The nonce is permanently consumed, but no tokens are minted to the recipient — their cross-chain transfer is irrecoverably lost.

---

### Finding Description

In `OmniBridge.finTransfer()`, the destination nonce is marked used unconditionally at the top of the function: [1](#0-0) 

Then, for the `customMinters` branch, a high-level call is made to `ICustomMinter.mint()`: [2](#0-1) 

And for the `isBridgeToken` branch, a high-level call is made to `IBridgeToken.mint()`: [3](#0-2) 

Both interfaces declare `mint` as returning nothing (`void`): [4](#0-3) [5](#0-4) 

The Solidity documentation states:

> The low-level functions `call`, `delegatecall`, and `staticcall` return `true` as their first return value if the account called is non-existent, as part of the design of the EVM.

This applies equally to high-level Solidity external calls: when the callee has no code, the EVM returns `(true, "")`. Because `mint` is declared `void`, Solidity performs no return-data decoding and does not revert. The call silently "succeeds."

No contract existence check (e.g., `tokenAddress.code.length > 0`) is performed before either `mint` call.

---

### Impact Explanation

If the `customMinter` contract or the `BridgeToken` proxy/implementation is destroyed (via `selfdestruct` or any upgrade path that leaves the address codeless):

1. A relayer submits a valid MPC-signed `finTransfer` payload.
2. `completedTransfers[payload.destinationNonce] = true` is written — the nonce is permanently consumed.
3. The `mint()` call to the codeless address returns `(true, "")` and does not revert.
4. No tokens are minted to `payload.recipient`.
5. The nonce cannot be replayed (`NonceAlreadyUsed` revert on retry).
6. The user's cross-chain transfer is permanently frozen with no recovery path.

This matches the allowed impact: **Permanent freezing / irrecoverable lock of user funds in bridge token flows.**

---

### Likelihood Explanation

- `customMinters` are admin-registered arbitrary contracts. Any such contract that contains a `selfdestruct` path (or is itself upgradeable and has its implementation wiped) becomes a trigger.
- `BridgeToken` proxies share a single `tokenImplementationAddress`. If that implementation is selfdestructed (possible pre-EIP-6780, or on chains that have not adopted EIP-6780), every bridge token's `mint` call silently succeeds, affecting all pending `finTransfer` calls across all bridge tokens simultaneously.
- The external report itself acknowledges the analogous risk is "not null" and cites a prior audit finding (TOB-NESTED-1) confirming real-world occurrence.
- Likelihood is **Low-to-Medium**: requires contract destruction, but the consequence when triggered is total and irreversible.

---

### Recommendation

Add a contract existence check before each `mint` call in `finTransfer`:

```solidity
// Before calling customMinter or bridgeToken mint:
require(target.code.length > 0, "OmniBridge: target has no code");
```

Or use OpenZeppelin's `Address.functionCall` wrapper (which performs this check internally) instead of bare interface dispatch for void-returning external calls.

Additionally, consider moving `completedTransfers[payload.destinationNonce] = true` to **after** all token-transfer logic succeeds, so that a failed or silent mint does not permanently consume the nonce.

---

### Proof of Concept

1. Admin registers a `customMinter` contract `M` for token `T` via `addCustomToken`.
2. `M` is later selfdestructed (e.g., it contains an owner-callable `selfdestruct`, or its implementation is wiped).
3. NEAR MPC signs a `TransferMessagePayload` for token `T`, recipient `Bob`, amount `1000`.
4. Relayer calls `OmniBridge.finTransfer(sig, payload)`.
5. `completedTransfers[nonce]` is set to `true` (line 287).
6. `customMinters[T]` is non-zero (still points to destroyed `M`).
7. `ICustomMinter(M).mint(T, Bob, 1000)` — EVM returns `(true, "")` because `M` has no code; Solidity does not revert (void return).
8. `finTransfer` emits `FinTransfer` and returns normally.
9. Bob receives zero tokens. Retry reverts with `NonceAlreadyUsed`. Funds are permanently lost. [6](#0-5)
