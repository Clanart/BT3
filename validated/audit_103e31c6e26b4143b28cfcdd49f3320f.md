### Title
Unvalidated `l1_gas_price_wei` / `l1_data_gas_price_wei` in `ProposalInit` allows a malicious proposer to inject arbitrary ETH-denominated gas prices into execution and the block hash — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_block_info_valid` validates only the FRI-denominated L1 gas prices (`l1_gas_price_fri`, `l1_data_gas_price_fri`) against the local oracle. The WEI-denominated counterparts (`l1_gas_price_wei`, `l1_data_gas_price_wei`) carried in `ProposalInit` are explicitly computed but then silently discarded (`_l1_gas_prices_wei`). Because `l2_gas_price_wei` is derived from `l2_gas_price_fri` via an `eth_to_fri_rate` that is itself computed from the unvalidated WEI prices, a malicious proposer can inject any ETH-denominated gas price into the `BlockInfo` used for execution and into the block hash commitment, with no rejection by any validator.

---

### Finding Description

**Two independent data paths, no cross-check.**

**Path 1 — Proposer (`build_proposal.rs`):**
`initiate_build` calls `get_l1_prices_in_fri_and_wei` to obtain both FRI and WEI prices from the oracle, then packs all four values into `ProposalInit`:

```
l1_gas_price_wei:      l1_prices_wei.l1_gas_price      (oracle-derived)
l1_data_gas_price_wei: l1_prices_wei.l1_data_gas_price  (oracle-derived)
l1_gas_price_fri:      l1_prices_fri.l1_gas_price       (oracle-derived)
l1_data_gas_price_fri: l1_prices_fri.l1_data_gas_price  (oracle-derived)
l2_gas_price_fri:      args.l2_gas_price                (local state)
```

`l2_gas_price_wei` is **not** in `ProposalInit`; it is computed later in `convert_to_sn_api_block_info` as:

```rust
let eth_to_fri_rate = calculate_eth_to_fri_rate(&PreviousBlockInfo::from(init))?;
let l2_gas_price_wei = init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?;
```

`eth_to_fri_rate` is derived from `init.l1_gas_price_wei` and `init.l1_gas_price_fri`.

**Path 2 — Validator (`validate_proposal.rs`):**
`is_block_info_valid` calls the same `get_l1_prices_in_fri_and_wei` but only uses the FRI half:

```rust
let (l1_gas_prices_fri, _l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(...).await;
```

The WEI result is bound to `_l1_gas_prices_wei` (underscore = intentionally unused). The only checks performed are:

```rust
within_margin(init_proposed.l1_gas_price_fri,      l1_gas_prices_fri.l1_gas_price,      margin)
within_margin(init_proposed.l1_data_gas_price_fri, l1_gas_prices_fri.l1_data_gas_price, margin)
```

`init_proposed.l1_gas_price_wei` and `init_proposed.l1_data_gas_price_wei` are **never compared** to any expected value.

**Downstream effect:**
After validation passes, `convert_to_sn_api_block_info(init)` is called with the accepted (but unvalidated) `init`. It builds the `BlockInfo` that drives execution:

```rust
eth_gas_prices: GasPriceVector {
    l1_gas_price:      NonzeroGasPrice::new(init.l1_gas_price_wei)?,      // ← attacker-controlled
    l1_data_gas_price: NonzeroGasPrice::new(init.l1_data_gas_price_wei)?, // ← attacker-controlled
    l2_gas_price:      l2_gas_price_wei,                                  // ← derived from attacker-controlled eth_to_fri_rate
},
```

`PartialBlockHashComponents::new(&block_info, header_commitments)` then captures all three WEI prices into the partial block hash, and `gas_prices_to_hash` (for version ≥ 0.13.4) hashes all six prices — including the three manipulated WEI values — into the final block hash commitment.

---

### Impact Explanation

**Incorrect fee / gas accounting with economic impact (Critical).**

A malicious proposer sets `l1_gas_price_wei = 1` (or any value) while keeping `l1_gas_price_fri` within the allowed margin. Every validator accepts the proposal because only the FRI prices are checked. The resulting `BlockInfo` used for execution carries:

- `eth_gas_prices.l1_gas_price = 1 wei` — all ETH-paying transactions are charged near-zero L1 gas fees.
- `eth_gas_prices.l2_gas_price = l2_gas_price_fri / (1/l1_gas_price_fri * 10^18)` — `l2_gas_price_wei` is inflated or deflated by the ratio of the forged WEI price to the true WEI price.

Transactions whose `max_fee` (ETH) would normally be insufficient are accepted and executed at the forged price. The block hash commits to the forged prices, so the committed state is permanently wrong. Because all replicas receive the same `ProposalInit` and apply the same `convert_to_sn_api_block_info`, they all compute the same wrong fees and the same wrong block hash — there is no split, but the ledger permanently records incorrect fee accounting.

---

### Likelihood Explanation

Any consensus participant who is scheduled as proposer can trigger this. No special privilege beyond being a validator/proposer is required. The attacker only needs to craft a `ProposalInit` with a manipulated `l1_gas_price_wei` while keeping `l1_gas_price_fri` within the oracle-derived margin. The margin check (`l1_gas_price_margin_percent`) is the only gate, and it operates entirely on FRI values.

---

### Recommendation

In `is_block_info_valid`, validate the WEI prices against the locally computed expected values, analogously to the FRI check. The computed `_l1_gas_prices_wei` should be renamed and used:

```rust
let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(...).await;

// existing FRI checks ...

// Add WEI checks:
if !(within_margin(init_proposed.l1_gas_price_wei,      l1_gas_prices_wei.l1_gas_price,      margin)
  && within_margin(init_proposed.l1_data_gas_price_wei, l1_gas_prices_wei.l1_data_gas_price, margin))
{
    return Err(ValidateProposalError::InvalidBlockInfo(...));
}
```

Alternatively, derive `l1_gas_price_wei` deterministically from the validated FRI price and the locally computed `eth_to_fri_rate` rather than accepting it from the proposer at all.

---

### Proof of Concept

1. Attacker is scheduled as proposer for height H.
2. Attacker calls `get_l1_prices_in_fri_and_wei` locally to obtain the oracle's current FRI prices (so the FRI check passes).
3. Attacker constructs `ProposalInit` with:
   - `l1_gas_price_fri` = oracle value (passes `within_margin`)
   - `l1_data_gas_price_fri` = oracle value (passes `within_margin`)
   - `l1_gas_price_wei` = **1** (not checked)
   - `l1_data_gas_price_wei` = **1** (not checked)
   - `l2_gas_price_fri` = expected value (passes exact equality check)
4. Every validator calls `is_block_info_valid`:
   - FRI checks pass → proposal accepted.
   - `_l1_gas_prices_wei` is discarded without comparison.
5. Every validator calls `convert_to_sn_api_block_info(init)`:
   - `eth_to_fri_rate = calculate_eth_to_fri_rate(PreviousBlockInfo::from(init))` → computed from `l1_gas_price_wei=1` and `l1_gas_price_fri=oracle_value` → rate is `oracle_value / 1 * 10^18` (massively inflated).
   - `l2_gas_price_wei = l2_gas_price_fri.fri_to_wei(inflated_rate)` → rounds to 0 or 1 (minimum).
   - `eth_gas_prices.l1_gas_price = 1 wei`.
6. Execution runs with `l1_gas_price_wei = 1` and `l2_gas_price_wei ≈ 0`. All ETH-paying transactions are charged near-zero fees regardless of their actual resource consumption.
7. `gas_prices_to_hash` hashes `l1_gas_price_wei=1`, `l2_gas_price_wei≈0` into the block hash. The committed block hash is permanently wrong.

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let previous_block_info = PreviousBlockInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}

async fn initiate_build(args: &mut ProposalBuildArguments) -> BuildProposalResult<ProposalInit> {
    let timestamp = get_proposal_timestamp(
        args.override_timestamp,
        args.deps.batcher.as_ref(),
        args.deps.clock.as_ref(),
    )
    .await;
    let (l1_prices_fri, l1_prices_wei) = get_l1_prices_in_fri_and_wei(
        args.deps.l1_gas_price_provider.clone(),
        timestamp,
        args.previous_block_info.as_ref(),
        &args.gas_price_params,
    )
    .await;
    let init = ProposalInit {
        height: args.build_param.height,
        round: args.build_param.round,
        valid_round: args.build_param.valid_round,
        proposer: args.build_param.proposer,
        builder: args.builder_address,
        timestamp,
        l1_da_mode: args.l1_da_mode,
        l2_gas_price_fri: args.l2_gas_price,
        l1_gas_price_wei: l1_prices_wei.l1_gas_price,
        l1_data_gas_price_wei: l1_prices_wei.l1_data_gas_price,
        l1_gas_price_fri: l1_prices_fri.l1_gas_price,
        l1_data_gas_price_fri: l1_prices_fri.l1_data_gas_price,
        starknet_version: starknet_api::block::StarknetVersion::LATEST,
        // TODO(Asmaa): Put the real value once we have it.
        version_constant_commitment: Default::default(),
    };

    let retrospective_block_hash = wait_for_retrospective_block_hash(
        args.deps.batcher.clone(),
        args.deps.state_sync_client.clone(),
        &init,
        args.deps.clock.as_ref(),
        args.retrospective_block_hash_deadline,
        args.retrospective_block_hash_retry_interval_millis,
        args.compare_retrospective_block_hash,
    )
    .await?;

    let build_proposal_input = ProposeBlockInput {
        proposal_id: args.proposal_id,
        deadline: args.batcher_deadline,
        retrospective_block_hash,
        block_info: convert_to_sn_api_block_info(&init)?,
        proposal_round: args.proposal_round,
    };
    debug!("Initiating build proposal: {build_proposal_input:?}");
    args.deps.batcher.propose_block(build_proposal_input.clone()).await.map_err(|err| {
        BuildProposalError::Batcher(
            format!("Failed to initiate build proposal {build_proposal_input:?}."),
            err,
        )
    })?;
    Ok(init)
}

pub fn gas_prices_to_hash(
    l1_gas_price: &GasPricePerToken,
    l1_data_gas_price: &GasPricePerToken,
    l2_gas_price: &GasPricePerToken,
    block_hash_version: &BlockHashVersion,
) -> Vec<Felt> {
    if block_hash_version >= &BlockHashVersion::V0_13_4 {
        vec![
            HashChain::new()
                .chain(&STARKNET_GAS_PRICES0)
                .chain(&l1_gas_price.price_in_wei.0.into())
                .chain(&l1_gas_price.price_in_fri.0.into())
                .chain(&l1_data_gas_price.price_in_wei.0.into())
                .chain(&l1_data_gas_price.price_in_fri.0.into())
                .chain(&l2_gas_price.price_in_wei.0.into())
                .chain(&l2_gas_price.price_in_fri.0.into())
                .get_poseidon_hash(),
        ]
    } else {
        vec![
            l1_gas_price.price_in_wei.0.into(),
            l1_gas_price.price_in_fri.0.into(),
            l1_data_gas_price.price_in_wei.0.into(),
            l1_data_gas_price.price_in_fri.0.into(),
        ]
    }
}

/// This message must be sent first when proposing a new block.
/// This struct differs from `BlockInfo` in `starknet_api` because we send L1 gas prices in ETH and

/// comparing after multiplication.

    pub height: BlockNumber,

    /// Block timestamp.
    pub timestamp: u64,

    pub builder: ContractAddress,

    pub starknet_version: starknet_api::block::StarknetVersion,

}
