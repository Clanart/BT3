### Title
Wrong `state_diff_commitment` and `state_diff_length` for Blocks Containing Deprecated Class Declarations Due to Structurally Missing Field in `CommitmentStateDiff` - (`crates/blockifier/src/state/cached_state.rs`)

---

### Summary

The `CommitmentStateDiff` struct has no `deprecated_declared_classes` field. The `From<CommitmentStateDiff> for ThinStateDiff` conversion therefore always hard-codes `deprecated_declared_classes: Vec::new()`. This `ThinStateDiff` is the sole input to `calculate_block_commitments` inside `BlockExecutionArtifacts::new`, which computes both the `state_diff_commitment` (Poseidon hash) and the `state_diff_length` packed into `concatenated_counts`. For any block that contains a deprecated (Cairo 0) class declaration, both values are structurally under-counted, producing a wrong `state_diff_commitment` and a wrong `concatenated_counts` field that propagate into the final block hash and the block header stored in the sequencer's storage.

---

### Finding Description

**Root cause — missing field, analogous to the missing `approve()` call:**

`CommitmentStateDiff` is defined without a `deprecated_declared_classes` member:

```rust
// crates/blockifier/src/state/cached_state.rs  lines 669-677
pub struct CommitmentStateDiff {
    pub address_to_class_hash: IndexMap<ContractAddress, ClassHash>,
    pub address_to_nonce: IndexMap<ContractAddress, Nonce>,
    pub storage_updates: IndexMap<ContractAddress, IndexMap<StorageKey, Felt>>,
    pub class_hash_to_compiled_class_hash: IndexMap<ClassHash, CompiledClassHash>,
}
``` [1](#0-0) 

The `From` implementation that converts it to `ThinStateDiff` hard-codes the missing field to an empty vector:

```rust
// crates/blockifier/src/state/cached_state.rs  lines 690-702
impl From<CommitmentStateDiff> for ThinStateDiff {
    fn from(commitment_state_diff: CommitmentStateDiff) -> Self {
        Self {
            deployed_contracts: commitment_state_diff.address_to_class_hash,
            storage_diffs: commitment_state_diff.storage_updates,
            class_hash_to_compiled_class_hash: commitment_state_diff
                .class_hash_to_compiled_class_hash,
            nonces: commitment_state_diff.address_to_nonce,
            // TODO(AlonH): Remove this when the structure of storage diffs changes.
            deprecated_declared_classes: Vec::new(),
        }
    }
}
``` [2](#0-1) 

**Propagation into commitment calculation:**

`BlockExecutionArtifacts::new` calls `ThinStateDiff::from(commitment_state_diff.clone())` and passes the result directly to `calculate_block_commitments`:

```rust
// crates/apollo_batcher/src/block_builder.rs  lines 160-166

    &transactions_data,
    ThinStateDiff::from(commitment_state_diff.clone()),
    l1_da_mode,
    &block_info.starknet_version,

.await;
``` [3](#0-2) 

Inside `calculate_block_commitments`, two values are derived from the (now empty) `deprecated_declared_classes`:

1. `state_diff.len()` — used as `state_diff_length` and packed into `concatenated_counts`:

```rust
// crates/starknet_api/src/state.rs  lines 110-121
pub fn len(&self) -> usize {
    let mut result = 0usize;
    result += self.deployed_contracts.len();
    result += self.class_hash_to_compiled_class_hash.len();
    result += self.deprecated_declared_classes.len();   // always 0
    result += self.nonces.len();
    for (_contract_address, storage_diffs) in &self.storage_diffs {
        result += storage_diffs.len();
    }
    result
}
``` [4](#0-3) 

2. `calculate_state_diff_hash` — chains `deprecated_declared_classes` into the Poseidon hash:

```rust
// crates/starknet_api/src/block_hash/state_diff_hash.rs  lines 30-42
pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    // ...
}
``` [5](#0-4) 

Both the wrong `state_diff_commitment` and the wrong `concatenated_counts` (which encodes `state_diff_length`) are stored in `BlockHeaderCommitments` and flow into `PartialBlockHashComponents`, which is the direct input to the final block hash computation.

**Why tests do not catch this — the mock analog:**

`CommitmentStateDiff` has no field for deprecated classes, so every test that constructs a `CommitmentStateDiff` directly (the normal path in unit tests) produces a `ThinStateDiff` with an empty `deprecated_declared_classes`. The commitment calculation succeeds and matches, masking the structural omission — exactly as the empty `MockVesting.vest()` masked the missing `approve()` in the original report.

The `verify_state_diff_hash` guard in the committer also does not catch this: it re-derives the commitment from the same `ThinStateDiff` (with the same empty field), so both sides of the comparison are identically wrong:

```rust
// crates/apollo_committer/src/committer.rs  lines 165-179
let state_diff_commitment = match state_diff_commitment {
    Some(commitment) => {
        if self.config.verify_state_diff_hash {
            let calculated_commitment = calculate_state_diff_hash(&state_diff);
            if commitment != calculated_commitment {
                return Err(CommitterError::StateDiffHashMismatch { ... });
            }
        }
        commitment
    }
    None => calculate_state_diff_hash(&state_diff),
};
``` [6](#0-5) 

---

### Impact Explanation

For any block that includes a deprecated (Cairo 0) class declaration (submitted via a DECLARE v0/v1 transaction):

- The `state_diff_commitment` stored in the block header is wrong (the Poseidon hash omits the deprecated class hashes).
- The `concatenated_counts` field (which packs `state_diff_length`) is wrong (undercounted by the number of deprecated declarations).
- Both fields are inputs to the block hash, so the block hash itself is wrong.
- The wrong `state_diff_commitment` is returned by `starknet_getStateUpdate` RPC, presenting an authoritative-looking incorrect value to clients and provers.
- Consensus nodes that independently recompute the block hash will disagree with the proposer, causing consensus failures.
- The SNOS/prover, which independently computes the state diff commitment, will produce a different value, causing proof failures.

This matches the **High** impact category: "RPC execution … returns an authoritative-looking wrong value" and "Wrong state … from blockifier/syscall/execution logic for accepted input."

---

### Likelihood Explanation

DECLARE v0 and v1 transactions (which declare Cairo 0 / deprecated classes) remain valid Starknet transaction types. Any unprivileged user who submits such a transaction that is included in a sequenced block triggers the wrong commitment. No special privilege is required. The trigger is a normal, accepted transaction type.

---

### Recommendation

Add `deprecated_declared_classes: Vec<ClassHash>` to `CommitmentStateDiff` and populate it from the blockifier's execution output (analogous to how `StateDiff::deprecated_declared_classes` is populated in `ThinStateDiff::from_state_diff`). Remove the `Vec::new()` hard-code and the associated TODO comment once the field is present. Ensure the `verify_state_diff_hash` check in the committer is exercised with a state diff that contains deprecated class declarations.

---

### Proof of Concept

1. Submit a DECLARE v0/v1 transaction declaring a Cairo 0 class to the sequencer gateway.
2. Wait for the transaction to be included in a block.
3. Call `starknet_getStateUpdate` for that block. Observe that `state_diff.old_declared_contracts` is non-empty (the deprecated class is present in the state diff returned by the full node), but the `state_diff_commitment` in the block header was computed with `deprecated_declared_classes = []`.
4. Independently compute `calculate_state_diff_hash` over the full `ThinStateDiff` (including the deprecated class). The result will differ from the `state_diff_commitment` stored in the block header, confirming the mismatch.
5. Observe that `state_diff_length` in the block header is undercounted by the number of deprecated class declarations, causing `concatenated_counts` and the block hash to be wrong. [7](#0-6) [3](#0-2) [5](#0-4) [8](#0-7)
