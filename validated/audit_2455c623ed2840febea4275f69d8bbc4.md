### Title
`ProofOfInclusion::valid()` Does Not Verify Against an External Expected Root Hash, Allowing Forged Inclusion Proofs - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` only checks internal hash-chain self-consistency. It derives the root hash it validates against from the proof's own data (`self.root_hash()` returns `last.combined_hash`), making the final equality check tautological. No external expected root hash is ever compared. An attacker who can supply a `ProofOfInclusion` object — trivially possible through the Python/Streamable bindings — can forge a proof that passes `valid()` for any arbitrary `node_hash`, proving inclusion of a key-value pair that does not exist in the real tree.

### Finding Description

`ProofOfInclusion` is defined as a `Streamable` + `pyclass(get_all, from_py_object)` type, meaning it can be freely constructed from Python or deserialized from untrusted bytes. [1](#0-0) 

The `valid()` method walks the `layers` array, recomputing each `combined_hash` from the previous hash and the sibling hash: [2](#0-1) 

The final line is `existing_hash == self.root_hash()`. But `root_hash()` returns `last.combined_hash`: [3](#0-2) 

After the loop, `existing_hash` is exactly `last.combined_hash` (the loop would have returned `false` earlier if they differed). Therefore the final equality check is **always true** once the loop completes — it is a tautology. `valid()` never compares the proof's root hash against any externally-supplied expected root.

An attacker constructs a forged `ProofOfInclusion`:
- `node_hash` = any hash they wish to prove included (e.g., `H(fake_key, fake_value)`)
- One layer: `other_hash = attacker_chosen`, `other_hash_side = Left`, `combined_hash = calculate_internal_hash(node_hash, Left, attacker_chosen)`

This passes `valid()` because the hash chain is internally consistent. The `root_hash()` returns the attacker-controlled `combined_hash`. Any caller that only checks `proof.valid()` — without also asserting `proof.root_hash() == known_tree_root` — accepts the forged proof.

The Python binding exposes this directly: [4](#0-3) 

### Impact Explanation

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."**

A caller (Python or Rust) that receives a `ProofOfInclusion` from an untrusted source (e.g., a DataLayer peer, a network message, or a deserialized blob) and calls only `proof.valid()` will accept a forged proof. The attacker can assert that any key-value pair is present in any DataLayer tree, corrupting the application's view of committed state without touching the actual `MerkleBlob`.

### Likelihood Explanation

- `ProofOfInclusion` is `Streamable` and `from_py_object`, so any Python caller can receive or construct one from untrusted bytes.
- The method is named `valid()`, strongly implying it is a complete validity check. Callers are likely to rely on it alone.
- The `get_proof_of_inclusion` API returns proofs that happen to be correct, masking the flaw during normal use and making it easy to miss in code review.
- No privileged role or key is required; any unprivileged network peer can supply a forged `ProofOfInclusion`.

### Recommendation

`valid()` must accept an externally-provided expected root hash and compare against it:

```rust
pub fn valid_against_root(&self, expected_root: &Hash) -> bool {
    // existing internal chain check ...
    // final check:
    self.root_hash() == *expected_root
}
```

Alternatively, rename the current method to `internally_consistent()` to make its limited scope explicit, and require all callers to separately assert `proof.root_hash() == known_root`. The Python binding should expose only the root-aware variant.

### Proof of Concept

```python
from chia_rs import MerkleBlob, ProofOfInclusion, ProofOfInclusionLayer, Side
import hashlib

# Attacker wants to forge proof that fake_node_hash is in the tree
fake_node_hash = bytes([0xAB] * 32)
other_hash     = bytes([0xCD] * 32)

# Compute combined_hash the same way calculate_internal_hash does
# (left || right with a domain prefix — exact formula from chia-datalayer)
combined = hashlib.sha256(b"\x01" + fake_node_hash + other_hash).digest()

forged = ProofOfInclusion(
    node_hash=fake_node_hash,
    layers=[ProofOfInclusionLayer(
        other_hash_side=Side.Left,
        other_hash=other_hash,
        combined_hash=combined,
    )],
)

assert forged.valid()          # passes — no external root checked
assert forged.root_hash() == combined  # attacker-controlled root
# Any caller that only checks forged.valid() accepts this as a valid inclusion proof.

    pub fn valid(&self) -> bool {
        let mut existing_hash = self.node_hash;

        for layer in &self.layers {
            let calculated_hash = crate::calculate_internal_hash(
                &existing_hash,
                layer.other_hash_side,
                &layer.other_hash,
            );

            if calculated_hash != layer.combined_hash {
                return false;
            }

            existing_hash = calculated_hash;
        }

        existing_hash == self.root_hash()
    }

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
