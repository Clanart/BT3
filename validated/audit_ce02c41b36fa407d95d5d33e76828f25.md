### Title
Cached Cairo0 (`V0`) Class Skips Revert-Validity Check in `get_compiled_from_class_manager`, Allowing Execution with an Undeclared Class After Block Revert — (`crates/blockifier/src/state/state_reader_and_contract_manager.rs`)

---

### Summary

`StateReaderAndContractManager::get_compiled_from_class_manager` applies an `is_declared` guard on cache hits **only for Cairo1 classes**. For `RunnableCompiledClass::V0` (Cairo0 / deprecated) classes the guard is silently skipped. Because the shared `ContractClassManager` cache is never flushed on a block revert, a Cairo0 class that was declared in a reverted block remains in the cache and is returned to the executor without any declaration check. The executor then runs the transaction against code that does not exist in the canonical state, producing a wrong execution result, wrong state diff, and a wrong state root / block commitment.

---

### Finding Description

`get_compiled_from_class_manager` in `StateReaderAndContractManager` handles cache hits as follows:

```rust
match &runnable_class {
    RunnableCompiledClass::V0(_) => {}          // ← no check at all
    _ => {
        // The Cairo1 class is cached; verify it is declared,
        // since existence in the cache does not guarantee that
        // (it might contain a declared class from a reverted block, for example).
        if !self.state_reader.is_declared(class_hash)? {
            return Err(StateError::UndeclaredClassHash(class_hash));
        }
    }
}
``` [1](#0-0) 

The comment explicitly acknowledges the revert-block scenario, but the protection is only wired for Cairo1. The `FetchCompiledClasses::is_declared` trait is documented to always return `false` for Cairo0 classes, so the developers intentionally omitted the check for `V0`:

```rust
/// Returns whether the given class hash corresponds to a declared Cairo 1 class.
/// Cairo 0 classes always return `false`.
fn is_declared(&self, class_hash: ClassHash) -> StateResult<bool>;
``` [2](#0-1) 

The `ContractClassManager` (both `TrivialClassManager` and `NativeClassManager`) is a process-wide, Arc-shared LRU cache. `BlockBuilderFactory` clones (shares) it into every new `StateReaderAndContractManager`:

```rust
let state_reader = StateReaderAndContractManager::new_with_native_classes_whitelist(
    apollo_reader,
    self.contract_class_manager.clone(),   // shared across all blocks
    ...
);
``` [3](#0-2) 

`revert_block` reverts storage and advances the class-manager marker, but it does **not** flush the in-memory class cache:

```rust
pub fn revert_block(storage_writer: &mut StorageWriter, target_block_marker: BlockNumber) {
    storage_writer
        ...
        .try_revert_class_manager_marker(target_block_marker)   // only the marker
        ...
``` [4](#0-3) 

`try_revert_class_manager_marker` only decrements a stored block number; it does not touch the `RawClassCache`: [5](#0-4) 

The test suite confirms the intended (but incomplete) design: the `cairo_0_cached_scenario` expects `is_declared_result: None` — i.e., no declaration check is ever made for a cached V0 class — and the test passes successfully:

---

### Impact Explanation

**Impact: Critical — Wrong state/execution result from blockifier/syscall/execution logic for accepted input.**

After a block revert, a subsequent block can execute a `call_contract`, `library_call`, or `deploy` targeting a Cairo0 class hash that is no longer declared in the canonical state. The blockifier returns the cached bytecode without error, the transaction succeeds instead of reverting with `UndeclaredClassHash`, and the resulting state diff (storage writes, events, L2→L1 messages) is committed to storage. The Patricia-trie state root computed from this diff is wrong, the block commitment (`calculate_block_hash`) chains from a wrong global root, and any SNOS proof built over this block proves an incorrect state transition.

---

### Likelihood Explanation

**Likelihood: High.**

Block reverts are a normal operational event in the sequencer (reorgs, proposal failures). The `revert_block` path is production code with its own metrics. Any Cairo0 class declared and accessed (cached) in a reverted block — including classes declared by the sequencer's own L1-handler transactions — satisfies the precondition. No special privilege or network position is required beyond submitting a `Declare v0` transaction that gets included and then reverted.

---

### Recommendation

Add a declaration check for `V0` cache hits analogous to the existing Cairo1 check. Because `is_declared` is intentionally Cairo1-only, introduce a parallel `is_deprecated_declared` query (backed by `get_deprecated_class_definition_block_number`) and call it in the `RunnableCompiledClass::V0(_)` arm:

```rust
RunnableCompiledClass::V0(_) => {
    if !self.state_reader.is_deprecated_declared(class_hash)? {
        return Err(StateError::UndeclaredClassHash(class_hash));
    }
}
```

Alternatively, extend `is_declared` to cover Cairo0 classes and remove the "Cairo 0 classes always return `false`" contract.

---

### Proof of Concept

1. **Block N**: Submit a `Declare v0` transaction for class hash `C`. The transaction is included and executed; `get_compiled_class(C)` is called during execution, populating the shared `ContractClassManager` cache with `CompiledClasses::V0(...)` keyed by `C`.

2. **Revert block N**: The sequencer calls `revert_block(N)`. Storage is rolled back — `deprecated_declared_classes_table` no longer contains `C` — but the in-memory `RawClassCache` still holds the `V0` entry for `C`.

3. **Block N+1**: Submit an `Invoke` transaction that calls `library_call(C, selector, ...)` or deploys a contract with class hash `C`.

4. **Execution**: `get_compiled_class(C)` → `get_compiled_from_class_manager(C)` → cache hit → `RunnableCompiledClass::V0(_)` arm → **no `is_declared` check** → returns the cached bytecode.

5. **Result**: The transaction executes successfully with code from a reverted block. The state diff records storage writes / events that should not exist. The Patricia trie produces a wrong `contracts_trie_root_hash` / `classes_trie_root_hash`, the block hash chains from a wrong global root, and any proof generated over this block is a proof of an invalid state transition.
