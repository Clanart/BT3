### Title
`ExecutionStateReader::get_compiled_class_hash` Ignores `migrated_compiled_classes` in Pending Data, Returning Stale v1 Hash for Pending-Block RPC Execution - (File: crates/apollo_rpc_execution/src/state_reader.rs)

### Summary

`ExecutionStateReader::get_compiled_class_hash` only consults `pending_data.declared_classes` when resolving a compiled class hash for a pending block. It never checks `migrated_compiled_classes`. Because `PendingData` has no `migrated_compiled_classes` field and `client_pending_data_to_execution_pending_data` silently drops that field during conversion, any class whose CASM hash was migrated (v1 Poseidon → v2 Blake) inside the pending block is invisible to the pending-state reader. The function falls through to the on-chain storage and returns the old v1 hash. Every RPC call that executes against the pending block — `starknet_estimateFee`, `starknet_simulateTransactions`, `starknet_call` — therefore operates with a wrong authoritative compiled-class-hash value.

### Finding Description

**Root cause 1 — `PendingData` has no `migrated_compiled_classes` field.**

`ExecutionPendingData` in `crates/apollo_rpc_execution/src/objects.rs` carries `declared_classes: Vec<DeclaredClassHashEntry>` but has no field for migrated compiled classes. [1](#0-0) 

**Root cause 2 — Conversion silently drops `migrated_compiled_classes`.**

`client_pending_data_to_execution_pending_data` in `crates/apollo_rpc/src/pending.rs` copies `declared_classes` from the client state diff but never copies `migrated_compiled_classes`, even though `ClientStateDiff` carries that field. [2](#0-1) 

**Root cause 3 — `get_compiled_class_hash` only checks `declared_classes`.**

When resolving a compiled class hash for a pending block, `ExecutionStateReader::get_compiled_class_hash` iterates only over `pending_data.declared_classes`. If the class was migrated (not freshly declared) in the pending block, the loop finds nothing and falls through to the on-chain storage, which returns the old v1 `CompiledClassHash`. [3](#0-2) 

**The stale value propagates into `CachedState`.**

`CachedState::get_compiled_class_hash` caches the first value it reads as the `initial_reads` baseline. Because the pending-state reader returns v1, the cache records v1 as the pre-existing value. Any subsequent write of v2 (e.g., by the migration logic or a declare transaction) produces a spurious v1→v2 diff entry, inflating the state diff and the fee charged for that diff. [4](#0-3) 

**`should_migrate` is also misled.**

`CasmHashMigrationData::from_state` calls `should_migrate`, which calls `get_compiled_class_hash` to decide whether a class still holds a v1 hash. If the pending block already migrated the class to v2 but the reader returns v1, `should_migrate` incorrectly marks the class for migration again, charging extra migration gas and producing a wrong state diff. [5](#0-4) 

**The `migrated_compiled_classes` field exists in the client state diff and is used in the committed path.**

The central sync path correctly handles `migrated_compiled_classes` by merging it into `class_hash_to_compiled_class_hash` in `ThinStateDiff::from_state_diff`, and `calculate_state_diff_hash` hashes the merged map. The pending execution path is the only place where the field is dropped. [6](#0-5) [7](#0-6) 

### Impact Explanation

Every RPC method that executes against the pending block (`starknet_estimateFee`, `starknet_simulateTransactions`, `starknet_call`, `starknet_getStateUpdate` for pending) uses `ExecutionStateReader` with `maybe_pending_data` populated. For any class whose CASM hash was migrated in the pending block:

- `get_compiled_class_hash` returns the old v1 hash instead of the new v2 hash.
- Fee estimation is wrong: migration gas is double-counted, or declare-transaction validation fails/succeeds incorrectly.
- Simulation produces a wrong state diff (spurious v1→v2 entry or missing entry).
- `starknet_call` results that depend on the compiled class hash are wrong.

This matches: **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

### Likelihood Explanation

The CASM hash migration feature (`enable_casm_hash_migration`, `migrated_compiled_classes`) is active in production. Any block that triggers a migration will populate `migrated_compiled_classes` in the pending state diff. Any RPC caller querying the pending block during that window will receive wrong values. No special privilege is required; any user calling `starknet_estimateFee` or `starknet_simulateTransactions` against the pending block triggers the path.

### Recommendation

1. Add a `migrated_compiled_classes: Vec<MigratedCompiledClassHashEntry>` field to `PendingData` in `crates/apollo_rpc_execution/src/objects.rs`.
2. Populate it in `client_pending_data_to_execution_pending_data` in `crates/apollo_rpc/src/pending.rs`.
3. In `ExecutionStateReader::get_compiled_class_hash`, after the `declared_classes` loop, add a second loop over `pending_data.migrated_compiled_classes` and return the migrated hash if found.

### Proof of Concept

1. Deploy a Cairo 1 class declared with a v1 (Poseidon) compiled class hash.
2. Construct a pending block that includes a `migrated_compiled_classes` entry upgrading that class to a v2 (Blake) hash.
3. Call `starknet_estimateFee` for a transaction that uses that class, with `block_id = "pending"`.
4. Observe that `ExecutionStateReader::get_compiled_class_hash` returns the v1 hash (from on-chain storage) instead of the v2 hash (from the pending migration).
5. The fee estimate includes spurious migration gas, and the simulated state diff contains a wrong `class_hash_to_compiled_class_hash` entry — neither of which would appear in the actually committed block.

    fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
        if let Some(pending_data) = &self.maybe_pending_data {
            for DeclaredClassHashEntry { class_hash: other_class_hash, compiled_class_hash } in
                &pending_data.declared_classes
            {
                if class_hash == *other_class_hash {
                    return Ok(*compiled_class_hash);
                }
            }
        }

        let maybe_block_number = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_reader()
            .map_err(storage_err_to_state_err)?
            .get_class_definition_block_number(&class_hash)
            .map_err(storage_err_to_state_err)?;

        // Cairo 0 classes (and undeclared classes) do not have a compiled class hash.
        // According to the trait, return the default value.
        let Some(block_number) = maybe_block_number else {
            return Ok(CompiledClassHash::default());
        };

        let state_diff = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_diff(block_number)
            .map_err(storage_err_to_state_err)?
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing state diff at block {block_number}."
            )))?;

        let compiled_class_hash = state_diff
            .class_hash_to_compiled_class_hash
            .get(&class_hash)
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing class declaration at block {block_number}, class \
                 {class_hash}."
            )))?;

        Ok(*compiled_class_hash)
    }

    fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
        let mut cache = self.cache.borrow_mut();

        if cache.get_compiled_class_hash(class_hash).is_none() {
            let compiled_class_hash = self.state.get_compiled_class_hash(class_hash)?;
            cache.set_compiled_class_hash_initial_value(class_hash, compiled_class_hash);
        }

        let compiled_class_hash = cache
            .get_compiled_class_hash(class_hash)
            .unwrap_or_else(|| panic!("Cannot retrieve '{class_hash:?}' from the cache."));
        Ok(*compiled_class_hash)
    }

    fn from_state<S: StateReader>(
        state_reader: &S,
        executed_class_hashes: &HashSet<ClassHash>,
        versioned_constants: &VersionedConstants,
    ) -> TransactionExecutionResult<Self> {
        if !versioned_constants.enable_casm_hash_migration {
            return Ok(Self::empty());
        }

        executed_class_hashes.iter().try_fold(Self::empty(), |mut migration_data, &class_hash| {
            if let Some((class_hash, casm_hash_v2_to_v1)) =
                should_migrate(state_reader, class_hash)?
            {
                // Add class hash mapping to the migration data.
                migration_data.class_hashes_to_migrate.insert(class_hash, casm_hash_v2_to_v1);

                // Accumulate the class's migration resources.
                let class = state_reader.get_compiled_class(class_hash)?;
                migration_data.resources +=
                    &class.estimate_compiled_class_hash_migration_resources();
            }
            Ok(migration_data)
        })
    }
