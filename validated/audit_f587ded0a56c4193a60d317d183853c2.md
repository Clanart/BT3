### Title
Duplicate Key Bypass in `MerkleBlob::batch_insert` Bootstrap Path Corrupts DataLayer Merkle Tree — (`crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::batch_insert` uses LIFO `pop()` to bootstrap the first two items through the checked `insert()` path, but then iterates the **remaining prefix** of the Vec through a fast path (`insert_entry_to_blob`) that has **no duplicate-key guard**. An attacker who supplies a Vec where a key duplicated in the last two elements also appears earlier in the Vec will silently insert that key twice, corrupting the committed Merkle tree.

---

### Finding Description

`batch_insert` has two distinct insertion paths:

**Bootstrap path** (lines 578–585) — used when `leaf_count <= 1`: [1](#0-0) 

It calls `pop()` twice (LIFO), extracting the **last two** elements of the Vec and routing them through `self.insert()`, which enforces uniqueness: [2](#0-1) 

**Fast path** (lines 587–603) — processes the **remaining prefix** of the Vec after `pop()` has consumed the tail: [3](#0-2) 

This path calls `insert_entry_to_blob` directly, which has **no duplicate-key check**. It calls `add_leaf`, which uses `HashMap::insert` and silently overwrites any existing cache entry: [4](#0-3) 

The result: the blob gets two leaf nodes with the same `KeyId`, but the cache's `key_to_index` map is silently overwritten to point only to the second one. The first duplicate leaf remains in the blob, referenced by the tree structure, but invisible to the cache.

---

### Impact Explanation

The committed Merkle root is computed over a blob containing a duplicate key. This means:

- Inclusion proofs for the first `key_A` leaf (at the bootstrap-inserted index) will fail because the cache points to the wrong index.
- `check_integrity()` would detect the mismatch (`IntegrityKeyToIndexCacheIndex`), but `check_integrity_on_drop` is `false` in production: [5](#0-4) 
- The corrupted root is silently committed to the DataLayer store, permanently invalidating the tree's Merkle proofs.

This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic corrupts tree roots or lets untrusted input prove invalid state**.

---

### Likelihood Explanation

The trigger condition is:
1. Tree starts with `leaf_count <= 1` (empty tree or single-leaf tree — the common initial state).
2. The batch Vec has ≥ 3 items.
3. A key appearing in the last two elements (which go through the checked path) also appears in an earlier element (which goes through the unchecked path).

The Python binding `py_batch_insert` accepts caller-supplied `keys_values` and `hashes` lists with no pre-validation of uniqueness: [6](#0-5) 

Any DataLayer client that can submit a batch update controls this input directly.

### Recommendation

Before the fast-path loop, validate that no key in the remaining prefix already exists in the cache:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // ... existing fast-path logic ...
}
```

Alternatively, pre-deduplicate the entire input Vec before splitting it into bootstrap and fast-path portions, or unify both paths through `insert()`.

```rust
let key_b = KeyId(2);
let h_a = Hash(Bytes32::new([0xAA; 32]));
let h_b = Hash(Bytes32::new([0xBB; 32]));
let h_c = Hash(Bytes32::new([0xCC; 32]));

// Vec order: [key_A (index 0), key_B (index 1), key_A (index 2)]
// pop() takes index 2 (key_A/h_C) → checked insert → ok
// pop() takes index 1 (key_B/h_B) → checked insert → ok
// fast path iterates index 0 (key_A/h_A) → insert_entry_to_blob → NO CHECK
let batch = vec![
    ((key_a, ValueId(1)), h_a),
    ((key_b, ValueId(2)), h_b),
    ((key_a, ValueId(3)), h_c),  // duplicate key_A at tail
];

blob.batch_insert(batch).unwrap(); // succeeds — no error

// Tree now has two leaf nodes with key_A; cache is inconsistent
// check_integrity() will return Err(IntegrityKeyToIndexCacheIndex(...))
assert!(blob.check_integrity().is_err());
```
