### Title
Division-Before-Multiplication in `NewDynamicFeeChecker` Truncates `feeCap` to Zero, Enabling Fee-Free Cosmos Transactions - (File: `ante/evm/fee_checker.go`)

### Summary

`NewDynamicFeeChecker` in `ante/evm/fee_checker.go` computes `feeCap` via integer division (`fee / gas`) before using it in a subsequent multiplication to derive `effectiveFee`. When `fee < gas`, integer truncation sets `feeCap = 0`. If the chain runs with London HF enabled but `NoBaseFee = true` (making `baseFee = 0`), the guard `feeCap.LT(baseFeeInt)` evaluates to `0 < 0 = false` and passes. The downstream `EffectiveGasPrice(0, 0, maxPriorityPrice)` then returns `0`, and `effectiveFee = 0 * gas = 0`. The Cosmos SDK `DeductFeeDecorator` deducts the returned fee (zero), so the sender pays nothing for their transaction.

### Finding Description

In `ante/evm/fee_checker.go`, `NewDynamicFeeChecker` performs the following sequence:

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))          // line 83: integer division
baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)

if feeCap.LT(baseFeeInt) { ... }                           // line 86: guard

effectivePrice := sdkmath.NewIntFromBigInt(
    types.EffectiveGasPrice(baseFeeInt.BigInt(),
                            feeCap.BigInt(),
                            maxPriorityPrice.BigInt()))    // line 91

effectiveFee := sdk.Coins{{
    Denom:  denom,
    Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas)), // line 97: multiply back
}}
```

`fee.Quo(gas)` is `sdkmath.Int` integer division. Any `fee < gas` truncates to `feeCap = 0`.

`baseFee` is obtained from `types.GetBaseFee` (line 56):

```go
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    if !IsLondon(ethCfg, height) {
        return nil   // falls back to min-gas-prices
    }
    baseFee := feemarketParams.GetBaseFee()
    if baseFee == nil {
        return new(big.Int)   // returns 0 (non-nil) when NoBaseFee=true
    }
    return baseFee
}
```

When `NoBaseFee = true`, `feemarketParams.GetBaseFee()` returns `nil`, so `GetBaseFee` returns `new(big.Int)` (= 0, **non-nil**). Because it is non-nil, the checker does **not** fall back to `checkTxFeeWithValidatorMinGasPrices`; it proceeds with `baseFeeInt = 0`.

With `feeCap = 0` and `baseFeeInt = 0`:
- Guard `feeCap.LT(baseFeeInt)` → `0 < 0` → **false** (passes)
- `EffectiveGasPrice(0, 0, maxPriorityPrice)` = `min(0 + maxPriorityPrice, 0)` = **0**
- `effectiveFee = 0 * gas` = **0**

The Cosmos SDK `DeductFeeDecorator` deducts the coins returned by the checker. Zero coins are deducted; the sender pays nothing.

### Impact Explanation

Any unprivileged user can submit a Cosmos SDK transaction (e.g., `MsgSend`, `MsgDelegate`, or any wrapped Cosmos message) with `fee = 1 aevmos` and `gas = 1_000_000`. The ante handler accepts it and charges 0 fees. This is a direct mis-accounting of user fees: the fee collector receives nothing, validators receive no fee revenue, and the economic spam-prevention mechanism is completely bypassed. This matches the allowed High impact: *"ante handler … bug that permits … valid user funds/fees to be mis-accounted."*

### Likelihood Explanation

The condition requires:
1. London HF enabled (default on all modern Ethermint chains).
2. `NoBaseFee = true` — a documented, supported configuration used by chains that want static or zero base fees.
3. `MinGasPrice = 0` — common default; `MinGasPriceDecorator` short-circuits when `minGasPrice.IsZero()`.

All three conditions are simultaneously reachable in production deployments. No privileged access is required; any user can craft the transaction.

### Recommendation

Replace the integer-division-first pattern with a precision-preserving comparison. The correct approach is to avoid deriving `feeCap` via integer division at all, and instead compare the total fee directly:

```go
// Instead of: feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
// Use Dec arithmetic to avoid truncation:
gasInt := sdkmath.NewIntFromUint64(gas)
feeCapDec := sdkmath.LegacyNewDecFromInt(fee).Quo(sdkmath.LegacyNewDecFromInt(gasInt))
baseFeeCapDec := sdkmath.LegacyNewDecFromInt(baseFeeInt)
if feeCapDec.LT(baseFeeCapDec) {
    return nil, 0, errorsmod.Wrapf(...)
}
// Then compute effectiveFee without re-multiplying a truncated feeCap:
effectivePrice := sdkmath.LegacyNewDecFromInt(baseFeeInt).Add(
    sdkmath.LegacyNewDecFromInt(maxPriorityPrice)).Min(feeCapDec)
effectiveFeeAmt := effectivePrice.MulInt(gasInt).TruncateInt()
```

Alternatively, guard against the zero-baseFee + zero-feeCap case explicitly, or ensure that when `NoBaseFee = true` the checker falls back to `checkTxFeeWithValidatorMinGasPrices` (i.e., treat a `baseFee` of 0 returned due to `NoBaseFee` the same as `nil`).

### Proof of Concept

**Setup:** Ethermint chain with London HF enabled, `NoBaseFee = true`, `MinGasPrice = 0`.

**Attack:**
```
fee    = 1 aevmos
gas    = 1_000_000
```

**Trace through `NewDynamicFeeChecker`:**
```
baseFee    = GetBaseFee(...)  → 0 (non-nil, because NoBaseFee=true)
feeCap     = 1 / 1_000_000   → 0  (integer truncation)
guard:       0 < 0            → false (passes)
effectivePrice = EffectiveGasPrice(0, 0, MaxInt64)
             = min(0 + MaxInt64, 0) = 0
effectiveFee = 0 * 1_000_000 = 0
```

**Result:** `DeductFeeDecorator` deducts 0 coins. The transaction is included in a block with zero fee paid.
