### Title
`initTransfer` Accepts Unregistered Tokens, Permanently Locking Funds — (`evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.sol::initTransfer` does not validate that the supplied `tokenAddress` is registered in the bridge (i.e., that `ethToNearToken[tokenAddress]` is non-empty). For any arbitrary ERC-20 that is not a bridge token and has no custom minter, the function silently locks the tokens in the bridge contract and emits an `InitTransfer` event. When a relayer submits the corresponding proof to NEAR, `fin_transfer_callback` panics with `TokenDecimalsNotFound` because the token has no registered decimals entry. There is no refund path back to EVM, so the tokens are permanently frozen.

---

### Finding Description

`OmniBridge::initTransfer` branches on whether the token is a custom-minter token, a bridge token, or neither:

```solidity
// OmniBridge.sol lines 394–412
if (customMinters[tokenAddress] != address(0)) {
    IERC20(tokenAddress).safeTransferFrom(msg.sender, customMinters[tokenAddress], amount);
    ICustomMinter(customMinters[tokenAddress]).burn(tokenAddress, amount);
} else if (isBridgeToken[tokenAddress]) {
    BridgeToken(tokenAddress).burn(msg.sender, amount);
} else {
    IERC20(tokenAddress).safeTransferFrom(msg.sender, address(this), amount);  // ← locks tokens
}
``` [1](#0-0) 

For the `else` branch (any ERC-20 not registered in the bridge), tokens are transferred into the bridge contract with no check that `ethToNearToken[tokenAddress]` is non-empty. The event is then emitted unconditionally:

```solidity
emit BridgeTypes.InitTransfer(msg.sender, tokenAddress, currentOriginNonce, amount, fee, nativeFee, recipient, message);
``` [2](#0-1) 

On the NEAR side, when a relayer submits the proof, `fin_transfer_callback` immediately panics:

```rust
let decimals = self
    .token_decimals
    .get(&init_transfer.token)
    .near_expect(BridgeError::TokenDecimalsNotFound);
``` [3](#0-2) 

Because the NEAR callback panics, its state changes are reverted, but the EVM-side lock is already committed. There is no admin rescue function in `OmniBridge.sol` and no cross-chain refund path, so the tokens are irrecoverably frozen.

The same root cause exists in the Starknet bridge: `init_transfer` performs no check that `starknet_to_near_token.read(token_address)` is non-empty before locking tokens via `transfer_from`. [4](#0-3) 

---

### Impact Explanation

Any ERC-20 tokens sent through `initTransfer` with an unregistered `tokenAddress` are permanently frozen in the EVM bridge contract. There is no admin sweep, no cross-chain refund, and no cancel mechanism. This matches the allowed impact: **Permanent freezing, irrecoverable lock, or unclaimable settlement of user or protocol funds in bridge, token, or vault flows.**

---

### Likelihood Explanation

This can happen accidentally (a user bridges a token before it has been registered via `deployToken`/`addCustomToken`) or deliberately (an attacker front-runs a token registration, submitting a transfer for the soon-to-be-registered token address before the `DeployToken` event is processed on EVM, mirroring the M-10 front-run scenario). The `tokenAddress` parameter is fully attacker-controlled and requires no special privilege. The only constraint is that the caller must hold the ERC-20 tokens.

---

### Recommendation

Add a registration guard at the top of `initTransfer` (and `initTransfer1155`):

```solidity
require(
    tokenAddress == address(0) ||
    bytes(ethToNearToken[tokenAddress]).length > 0 ||
    multiTokens[tokenAddress].tokenAddress != address(0),
    "ERR_TOKEN_NOT_REGISTERED"
);
```

Apply the equivalent guard in the Starknet `init_transfer`:

```cairo
assert(
    self.is_bridge_token(token_address) ||
    self.starknet_to_near_token.read(token_address).len() > 0,
    'ERR_TOKEN_NOT_REGISTERED'
);
```

---

### Proof of Concept

1. Deploy any ERC-20 token `T` that is **not** registered in `OmniBridge` (i.e., `ethToNearToken[T] == ""`).
2. Approve `OmniBridge` to spend `amount` of `T`.
3. Call `OmniBridge.initTransfer(address(T), amount, 0, 0, "victim.near", "")`.
   - Branch taken: `else` → `safeTransferFrom(msg.sender, address(this), amount)`.
   - `InitTransfer` event emitted with `tokenAddress = T`.
4. Relayer picks up the event and calls `fin_transfer` on NEAR with the proof.
5. `fin_transfer_callback` executes: `self.token_decimals.get(&init_transfer.token).near_expect(BridgeError::TokenDecimalsNotFound)` → **panics**.
6. NEAR state reverts; EVM lock is permanent.
7. `amount` of token `T` is frozen in `OmniBridge` forever with no recovery path.

        fn init_transfer(
            ref self: ContractState,
            token_address: ContractAddress,
            amount: u128,
            fee: u128,
            native_fee: u128,
            recipient: ByteArray,
            message: ByteArray,
        ) {
            assert(!_is_paused(@self, PAUSE_INIT_TRANSFER), 'ERR_INIT_TRANSFER_PAUSED');

            assert(amount > 0, 'ERR_ZERO_AMOUNT');
            assert(fee < amount, 'ERR_INVALID_FEE');

            let origin_nonce = self.current_origin_nonce.read() + 1;
            self.current_origin_nonce.write(origin_nonce);

            let caller = get_caller_address();

            if self.is_bridge_token(token_address) {
                IBridgeTokenDispatcher { contract_address: token_address }
                    .burn(caller, amount.into());
            } else {
                let success = IERC20Dispatcher { contract_address: token_address }
                    .transfer_from(caller, get_contract_address(), amount.into());
                assert(success, 'ERR_TRANSFER_FROM_FAILED');
            }

            if native_fee > 0 {
                let native_token = self.strk_token_address.read();
                let success = IERC20Dispatcher { contract_address: native_token }
                    .transfer_from(caller, get_contract_address(), native_fee.into());
                assert(success, 'ERR_FEE_TRANSFER_FAILED');
            }

            self
                .emit(
                    Event::InitTransfer(
                        InitTransfer {
                            sender: caller,
                            token_address,
                            origin_nonce,
                            amount,
                            fee,
                            native_fee,
                            recipient,
                            message,
                        },
                    ),
                )
        }
