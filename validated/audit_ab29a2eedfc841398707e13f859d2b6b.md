### Title
Gateway `max_l2_gas_amount` Exceeds OS Per-Transaction Execution Gas Cap, Admitting Transactions That Will Revert Unexpectedly - (File: `crates/apollo_gateway_config/src/config.rs`)

### Summary

The `StatelessTransactionValidatorConfig::max_l2_gas_amount` (1,200,000,000) enforced at gateway admission is higher than the effective OS-level per-transaction gas ceiling (`execute_max_sierra_gas` + `validate_max_sierra_gas` = 1,000,000,000 + 100,000,000 = 1,100,000,000). These two limits live in completely separate configuration objects with no programmatic cross-check. Any transaction whose declared `l2_gas.max_amount` falls in the gap (1.1 B – 1.2 B) is admitted by the gateway but will be executed with at most 1.1 B gas by the blockifier. If the transaction requires more than 1.1 B gas to succeed it will revert with "Out of gas" even though the user declared a sufficient bound.

### Finding Description

**Source 1 – Gateway admission limit**

`StatelessTransactionValidatorConfig::max_l2_gas_amount` is set to `1_200_000_000` by default and enforced in `validate_resource_bounds`:

```rust
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { … });
```
