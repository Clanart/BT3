### Title
Missing Class Hash Validation in `execute_replace_class` Enables Permanent Contract Freezing — (File: `execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` syscall in the StarkNet OS does not validate that the replacement class hash has been declared. Any contract can replace its own class with an arbitrary, undeclared hash, permanently rendering itself unexecutable and freezing all funds it holds. The OS itself acknowledges this missing check via a TODO comment.

### Finding Description
In `execute_replace_class` (lines 878–916 of `syscall_impls.cairo`), the new class hash supplied by the calling contract is written directly into the contract's `StateEntry` without any check that the hash exists in `contract_class_changes` (declared in the current block) or in the existing class tree:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

After this update, any subsequent call to the contract causes the OS to look up the compiled class for the stored hash. Because the hash is undeclared, no compiled class exists, and every call fails. Since `replace_class` can only be invoked by the contract itself, and the contract can no longer execute, the freeze is irreversible.

This is the direct analog of the external report: just as `FantiumMinterV1` changes `fantiumNFTContractAddress` without clearing `collectionIdToAllowList`, the OS here changes the contract's class hash without verifying the new hash is backed by any declared class — leaving the contract's associated execution context permanently broken.

The `declare_transaction` path enforces `prev_value=0` to prevent re-declaration of the same class:

```cairo
// Note that prev_value=0 enforces that a class may be declared only once.
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [2](#0-1) 

But `execute_replace_class` performs no symmetric check that the target hash was ever declared, creating an asymmetry: declaration is strictly validated; replacement is not.

### Impact Explanation
A contract that calls `replace_class` with an undeclared hash becomes permanently unexecutable. Every call to it will fail at class lookup. Because `replace_class` itself requires calling the contract, there is no recovery path. All funds held by the contract are permanently frozen.

**Impact: Critical — Permanent freezing of funds.**

### Likelihood Explanation
The attack is reachable by any unprivileged contract deployer:

1. Deploy contract A (class hash `H_A`) with a `deposit()` function and a `freeze(felt class_hash)` function that calls the `replace_class` syscall with the supplied hash.
2. Attract user deposits into contract A.
3. Call `freeze(0xdeadbeef)` where `0xdeadbeef` is any value not present in `contract_class_changes` or the class tree.
4. The OS writes `0xdeadbeef` as the contract's class hash with no validation.
5. All subsequent calls to contract A fail; all deposited funds are permanently frozen.

No privileged role, leaked key, or malicious sequencer is required. The attacker is solely a contract deployer exercising a standard syscall that the OS fails to constrain.

### Recommendation
Inside `execute_replace_class`, before writing the new class hash to `contract_state_changes`, verify that the hash exists either in `contract_class_changes` (declared in the current block) or in the existing contract class Patricia tree. Concretely, perform a `dict_read` on `contract_class_changes` for `class_hash` and assert the result is non-zero, or add an equivalent membership proof against the class tree root. The TODO comment at line 898 already identifies this as a required check.

### Proof of Concept

```
// Attacker-controlled contract (Sierra/Cairo)
#[starknet::contract]
mod MaliciousVault {
    #[external]
    fn deposit() { /* accept ETH/tokens */ }

    #[external]
    fn freeze(new_class_hash: felt252) {
        // Calls replace_class syscall with arbitrary undeclared hash.
        starknet::replace_class_syscall(new_class_hash).unwrap();
    }
}
```

Execution trace through the OS:

1. `execute_replace_class` is invoked with `class_hash = 0xdeadbeef`.
2. `dict_update` writes `StateEntry { class_hash: 0xdeadbeef, ... }` for the contract address — **no validation performed**.
3. `state_update` in `state.cairo` squashes and commits this entry to the Patricia tree.
4. Any future call reads `class_hash = 0xdeadbeef` from state, fails to find a compiled class, and reverts.
5. No `replace_class` can be issued to recover, because issuing it requires executing the contract.
6. All funds in the contract are permanently frozen. [3](#0-2) [4](#0-3)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
