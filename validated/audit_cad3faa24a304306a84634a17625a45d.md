### Title
`set_state_finalize_on_height` omits `StateTransitionData` write, breaking state witness generation after state sync — (File: `chain/chain/src/chain_update.rs`)

---

### Summary

`set_state_finalize_on_height` applies missing-chunk transitions for gap blocks during state sync finalization with `record_storage = true`, producing a `PartialStorage` proof in the apply result, but never persists it to `DBCol::StateTransitionData`. The normal block-processing path (`process_apply_chunk_result`) does persist it. When a chunk producer that just completed state sync calls `create_state_witness`, `collect_state_transition_data` walks back through those gap blocks and calls `get_state_transition`, which hard-fails on the missing DB entry. The chunk producer cannot produce a valid state witness and therefore cannot produce chunks.

---

---

### Impact Explanation

A chunk producer that completes state sync via `reset_heads_post_state_sync` has its gap blocks (between the state sync point and the sync hash) processed exclusively by `set_state_finalize_on_height`. Those blocks have `ChunkExtra` and trie changes committed, but no `StateTransitionData`. When the node subsequently produces its first chunk and calls `create_state_witness`, `get_state_transition` returns `Error::Other("Missing transition state proof …")` for every gap block that contained an implicit (missing-chunk) transition. The chunk producer cannot distribute a state witness, validators cannot endorse the chunk, and the chunk producer fails to earn rewards and risks kickout.

The corrupted invariant is: every block processed with `record_storage = true` that a chunk producer may later need for witness generation must have a corresponding `DBCol::StateTransitionData` entry. `set_state_finalize_on_height` breaks this invariant for all gap blocks.

---

### Likelihood Explanation

State sync is triggered whenever a node falls more than ~2 epochs behind, including new validators joining the network. With dynamic resharding and shard shuffling enabled, validators are regularly assigned to new shards and must complete catchup. The catchup path (`block_catch_up_postprocess`) correctly saves `StateTransitionData`. However, the far-horizon state sync path (used when a node is very far behind) goes through `set_state_finalize_on_height` and does not. Any validator that uses the far-horizon path and then becomes a chunk producer in the next epoch will hit this bug on its first chunk production attempt.

---

### Recommendation

In `set_state_finalize_on_height`, after `apply_chunk` returns, compute `should_save_state_transition_data` (as done in `postprocess_block_only`) and call `save_state_transition_data` with `apply_result.proof`, `apply_result.applied_receipts_hash`, and `apply_result.contract_updates`, mirroring the `OldChunk` branch of `process_apply_chunk_result`. The `PartialStorage` is already produced (because `record_storage = true` is passed), so no additional runtime cost is incurred — only the DB write is missing.

---

### Proof of Concept

1. Start a validator node that is more than 2 epochs behind the network tip, triggering the far-horizon sync path.
2. State sync completes; `reset_heads_post_state_sync` is called, which internally calls `set_state_finalize` → `set_state_finalize_on_height` for each gap block.
3. The node is elected as a chunk producer for the next epoch.
4. The node calls `produce_chunk` → `create_state_witness` → `collect_state_transition_data`.
5. `collect_state_transition_data` iterates back through gap blocks and calls `get_state_transition` for each one.
6. `get_state_transition` calls `store.get_ser(DBCol::StateTransitionData, …)` and finds no entry (because `set_state_finalize_on_height` never wrote one).
7. `get_state_transition` returns `Error::Other("Missing transition state proof for block … and shard …")`.
8. `create_state_witness` propagates the error; no state witness is distributed; the chunk is not endorsed; the chunk producer earns no reward and risks kickout. [7](#0-6) [8](#0-7) [9](#0-8)

    pub fn set_state_finalize_on_height(
        &mut self,
        height: BlockHeight,
        shard_id: ShardId,
        sync_hash: CryptoHash,
    ) -> Result<bool, Error> {
        let _span =
            tracing::debug_span!(target: "sync", "set_state_finalize_on_height", height, %shard_id)
                .entered();
        // Note that block headers are already synced and can be taken
        // from store on disk.
        let block_header_result = get_block_header_on_chain_by_height(
            &self.chain_store_update.chain_store(),
            &sync_hash,
            height,
        );
        if let Err(_) = block_header_result {
            // No such height, go ahead.
            return Ok(true);
        }
        let block_header = block_header_result?;
        if block_header.hash() == &sync_hash {
            // Don't continue
            return Ok(false);
        }
        let block = self.chain_store_update.get_block(block_header.hash())?;

        let prev_hash = block_header.prev_hash();
        let prev_block_header = self.chain_store_update.get_block_header(prev_hash)?;

        let shard_uid =
            shard_id_to_uid(self.epoch_manager.as_ref(), shard_id, block_header.epoch_id())?;
        let chunk_extra = self.chain_store_update.get_chunk_extra(prev_hash, &shard_uid)?;
        let memtrie_pin = self
            .runtime_adapter
            .get_tries()
            .maybe_pin_memtrie_root(shard_uid, *chunk_extra.state_root())?;

        let apply_result = self.runtime_adapter.apply_chunk(
            RuntimeStorageConfig::new(*chunk_extra.state_root(), true),
            ApplyChunkReason::UpdateTrackedShard,
            ApplyChunkShardContext {
                shard_uid,
                last_validator_proposals: chunk_extra.validator_proposals(),
                gas_limit: chunk_extra.gas_limit(),
                is_new_chunk: false,
                on_post_state_ready: None,
                memtrie_pin,
            },
            ApplyChunkBlockContext::from_header(
                &block_header,
                prev_block_header.next_gas_price(),
                block.block_congestion_info(),
                block.block_bandwidth_requests(),
            ),
            &[],
            SignedValidPeriodTransactions::empty(),
        )?;
        let flat_storage_manager = self.runtime_adapter.get_flat_storage_manager();
        let store_update = flat_storage_manager.save_flat_state_changes(
            *block_header.hash(),
            *prev_block_header.hash(),
            height,
            shard_uid,
            apply_result.trie_changes.state_changes(),
        )?;
        self.chain_store_update.merge(store_update.into());
        self.chain_store_update.save_trie_changes(*block_header.hash(), apply_result.trie_changes);

        // The chunk is missing but some fields may need to be updated
        // anyway. Prepare a chunk extra as a copy of the old chunk
        // extra and apply changes to it.
        let mut new_chunk_extra = ChunkExtra::clone(&chunk_extra);
        *new_chunk_extra.state_root_mut() = apply_result.new_root;
        self.chain_store_update.save_chunk_extra(
            block_header.hash(),
            &shard_uid,
            new_chunk_extra.into(),
        );
        Ok(true)
    }

    fn collect_state_transition_data(
        &self,
        epoch_manager: &dyn EpochManagerAdapter,
        chunk_header: &ShardChunkHeader,
        prev_chunk_header: &ShardChunkHeader,
    ) -> Result<StateTransitionData, Error> {
        let prev_chunk_height_included = prev_chunk_header.height_included();

        // Iterate over blocks in chain from `chunk_header.prev_block_hash()`
        // (inclusive) until the block with height `prev_chunk_height_included`
        // (exclusive).
        // Every block corresponds to one implicit state transition between
        // `prev_chunk_header` and `chunk_header`.
        // There may be one additional implicit transition for a block, if
        // resharding happens after its processing.
        // TODO(logunov): consider uniting with `get_incoming_receipts_for_shard`
        // because it has the same purpose.
        let mut current_block_hash = *chunk_header.prev_block_hash();
        let mut next_epoch_id = epoch_manager.get_epoch_id_from_prev_block(&current_block_hash)?;
        let mut next_shard_id = chunk_header.shard_id();
        let mut implicit_transitions = vec![];

        loop {
            let header = self.get_block_header(&current_block_hash)?;
            if header.height() < prev_chunk_height_included {
                return Err(Error::InvalidBlockHeight(prev_chunk_height_included));
            }

            let current_epoch_id = *header.epoch_id();
            let current_shard_id = epoch_manager
                .get_prev_shard_id_from_prev_hash(&current_block_hash, next_shard_id)?
                .1;
            if current_shard_id != next_shard_id {
                // If shard id changes, we need to get implicit state
                // transition from current shard id to the next shard id.
                let (chunk_state_transition, _, _) = self.get_state_transition(
                    epoch_manager,
                    &current_block_hash,
                    &next_epoch_id,
                    next_shard_id,
                )?;
                implicit_transitions.push(chunk_state_transition);
            }
            next_epoch_id = current_epoch_id;
            next_shard_id = current_shard_id;

            if header.height() == prev_chunk_height_included {
                break;
            }

            // Add implicit state transition.
            let (chunk_state_transition, _, _) = self.get_state_transition(
                epoch_manager,
                &current_block_hash,
                &current_epoch_id,
                current_shard_id,
            )?;
            implicit_transitions.push(chunk_state_transition);

            current_block_hash = *header.prev_hash();
        }

        let main_block = current_block_hash;
        let epoch_id = next_epoch_id;
        let main_transition_shard_id = next_shard_id;
        implicit_transitions.reverse();

        // Get the main state transition.
        let (main_transition, receipts_hash, contract_updates) = if prev_chunk_header.is_genesis() {
            self.get_genesis_state_transition(
                epoch_manager,
                &main_block,
                &epoch_id,
                main_transition_shard_id,
            )?
        } else {
            self.get_state_transition(
                epoch_manager,
                &main_block,
                &epoch_id,
                main_transition_shard_id,
            )?
        };

        Ok(StateTransitionData {
            main_transition,
            main_transition_shard_id,
            implicit_transitions,
            applied_receipts_hash: receipts_hash,
            contract_updates,
        })
    }
