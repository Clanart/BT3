### Title
`deprecated_declared_classes` silently dropped in `CommitmentStateDiff → ThinStateDiff` conversion, producing wrong `state_diff_commitment` and `concatenated_counts` in block hash — (File: `crates/blockifier/src/state/cached_state.rs`)

---

### Summary

`StateMaps.declared_contracts` tracks every deprecated (Cairo 0) class declaration made during block execution. The `CommitmentStateDiff` struct has no field for these entries, so `CommitmentStateDiff::from(StateMaps)` silently drops them. The downstream `From<CommitmentStateDiff> for ThinStateDiff` then hardcodes `deprecated_declared_classes: Vec::new()`. Because `BlockExecutionArtifacts::new` uses this conversion to feed both `calculate_block_commitments` (which computes `state_diff_commitment` and `state_diff_length`) and `thin_state_diff()` (which is stored to the DB), any block containing a deprecated Declare transaction will have a wrong `state_diff_commitment`, a wrong `state_diff_length` in `concatenated_counts`, and therefore a wrong block hash and wrong stored state diff.

---

### Finding Description

**Step 1 — Deprecated declared classes are tracked in `StateMaps` but dropped in `CommitmentStateDiff`.**

`StateMaps` carries `declared_contracts: HashMap<ClassHash, bool>` which is set to `true` for every deprecated (Cairo 0) class declared in a transaction. A test in the blockifier confirms this:

```rust

    StateMaps {
        compiled_class_hashes: HashMap::from([(declare_tx.class_hash(), compiled_class_hash)]),
        declared_contracts: HashMap::from([(declare_tx.class_hash(), true)]),
        ..Default::default()
    }
);
``` [1](#0-0) 

The conversion `CommitmentStateDiff::from(StateMaps)` maps only `class_hashes`, `storage`, `compiled_class_hashes`, and `nonces` — `declared_contracts` is never read: [2](#0-1) 

`CommitmentStateDiff` itself has no `deprecated_declared_classes` field: [3](#0-2) 

**Step 2 — `From<CommitmentStateDiff> for ThinStateDiff` hardcodes an empty list.** [4](#0-3) 

The `// TODO(AlonH): Remove this when the structure of storage diffs changes.` comment at line 698 acknowledges the structural gap but leaves the field permanently empty in the proposer path.

**Step 3 — `BlockExecutionArtifacts::new` uses this conversion for commitment calculation.** [5](#0-4) 

`ThinStateDiff::from(commitment_state_diff.clone())` always produces `deprecated_declared_classes: []`.

**Step 4 — `calculate_state_diff_hash` includes `deprecated_declared_classes` in the Poseidon hash.** [6](#0-5) 

With an empty list the hash chains `0` (count) and nothing else, instead of the actual count and class hashes.

**Step 5 — `ThinStateDiff::len()` counts `deprecated_declared_classes` in `state_diff_length`.** [7](#0-6) 

With an empty list, `state_diff_length` is under-counted by the number of deprecated declared classes.

**Step 6 — Both wrong values feed into the block hash.**

`calculate_block_commitments` passes `state_diff.len()` to `concat_counts` and runs `calculate_state_diff_hash`: [8](#0-7) 

`calculate_block_hash` chains both `concatenated_counts` (which encodes the wrong `state_diff_length`) and `state_diff_commitment` into the final block hash: [9](#0-8) 

**Step 7 — The stored `ThinStateDiff` is also wrong.**

`BlockExecutionArtifacts::thin_state_diff()` uses the same conversion: [10](#0-9) 

This is what `commit_proposal` writes to storage: [11](#0-10) 

So the stored state diff, the `state_diff_commitment`, and the `concatenated_counts` are all consistently wrong — they all omit deprecated declared classes.

---

### Impact Explanation

For any block that includes a deprecated Declare (v1) transaction:

1. **Wrong `state_diff_commitment`** — `calculate_state_diff_hash` chains `0` deprecated classes instead of the actual set, producing a different Poseidon hash.
2. **Wrong `state_diff_length` in `concatenated_counts`** — the packed field encoding transaction count, event count, and state diff length is wrong, changing the block hash.
3. **Wrong block hash** — both corrupted values are chained into `calculate_block_hash`, so the `PartialBlockHash` / `ProposalCommitment` is wrong.
4. **Wrong stored state diff** — the DB entry for the block's `ThinStateDiff` has `deprecated_declared_classes: []`, so any consumer reading it (RPC `starknet_getStateUpdate`, sync, prover) sees an incomplete state diff.

Because all sequencer nodes run the same code, they agree on the wrong hash. However, any external verifier (L1 proof verifier, independent block hash checker) that correctly includes deprecated declared classes will compute a different hash, causing proof/L1 verification failure for such blocks.

This matches: **Critical — Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input.**

---

### Likelihood Explanation

Deprecated Declare (v1) transactions are still valid in the Starknet protocol and are accepted by the gateway (confirmed by `simulate_declare_deprecated` test in `crates/apollo_rpc_execution/src/execution_test.rs`). Any user who submits such a transaction triggers the bug without any privileged access. The trigger is a normal, accepted transaction type.

---

### Recommendation

1. Add a `deprecated_declared_classes: Vec<ClassHash>` field to `CommitmentStateDiff`.
2. In `CommitmentStateDiff::from(StateMaps)`, populate it from `diff.declared_contracts` by collecting all keys where the value is `true` and no corresponding `compiled_class_hash` exists (i.e., Cairo 0 classes).
3. Remove the `deprecated_declared_classes: Vec::new()` hardcode in `From<CommitmentStateDiff> for ThinStateDiff` and use the new field instead.
4. Remove the TODO comment at line 698 of `cached_state.rs`.

---

### Proof of Concept

1. Submit a deprecated Declare (v1) transaction declaring Cairo 0 class `C` to the sequencer.
2. The blockifier executes it; `StateMaps.declared_contracts` is set to `{C: true}`.
3. `CommitmentStateDiff::from(StateMaps)` drops `declared_contracts`; `CommitmentStateDiff.class_hash_to_compiled_class_hash` does not contain `C` (no compiled class hash for Cairo 0).
4. `ThinStateDiff::from(CommitmentStateDiff)` produces `deprecated_declared_classes: []`.
5. `calculate_state_diff_hash` chains `0` (count) for deprecated classes instead of `1, C`.
6. `ThinStateDiff::len()` returns a value that is 1 less than correct; `concat_counts` encodes the wrong `state_diff_length`.
7. `calculate_block_hash` produces a hash that differs from the correct hash by both the `state_diff_commitment` and `concatenated_counts` fields.
8. `starknet_getStateUpdate` for this block returns a `ThinStateDiff` with `deprecated_declared_classes: []`, omitting `C`.

    fn commit_proposal(
        &mut self,
        height: BlockNumber,
        state_diff: ThinStateDiff,

        // TODO(AlonH): write casms.

        match storage_commitment_block_hash {
            StorageCommitmentBlockHash::ParentHash(parent_block_hash) => {
