### Title
`personal_sendTransaction` and `personal_sign` Ignore Password Parameter, Enabling Unauthorized Transaction Signing from Node Keyring ‚Äî (`rpc/namespaces/ethereum/personal/api.go`)

---

### Summary

The `personal_sendTransaction` and `personal_sign` JSON-RPC endpoints accept a password parameter per the Ethereum `personal_` namespace spec but discard it entirely. Because `UnlockAccount` is also unimplemented, every address stored in the node's keyring is permanently accessible to any caller who can reach the endpoint. When the `personal` namespace is enabled, an unprivileged network caller can drain funds from any keyring-managed address (e.g., the public validator address) by submitting a crafted `personal_sendTransaction` call with an arbitrary or empty password.

---

### Finding Description

**`personal_sendTransaction` ‚Äî password silently discarded**

```go
// rpc/namespaces/ethereum/personal/api.go
func (api *PrivateAccountAPI) SendTransaction(
    _ context.Context,
    args evmtypes.TransactionArgs,
    _ string,          // ‚Üê password parameter is unnamed and never read
) (common.Hash, error) {
    api.logger.Debug("personal_sendTransaction", "address", args.To.String())
    return api.backend.SendTransaction(args)
}
```

The backend implementation (`rpc/backend/sign_tx.go`) only verifies that the `from` address exists in the keyring, then signs and broadcasts the transaction unconditionally ‚Äî no password check occurs at any layer:

```go
// rpc/backend/sign_tx.go

    if err != nil { ... }
    ...
    if err := msg.Sign(signer, b.clientCtx.Keyring); err != nil { ... }
    ...
    return txHash, nil
}
```

**`personal_sign` ‚Äî same pattern**

```go

    _ context.Context,
    data hexutil.Bytes,
    addr common.Address,
    _ string,          // ‚Üê password discarded

    return api.backend.Sign(addr, data)

```

**`UnlockAccount` is permanently stubbed out**

```go
func (api *PrivateAccountAPI) UnlockAccount(
    _ context.Context, addr common.Address, _ string, _ *uint64,
) (bool, error) {
    // TODO: Not supported. See underlying issue https://github.com/99designs/keyring/issues/85
    return false, nil
}
```

Because unlocking is never implemented, every key in the keyring is always "unlocked." The password field in `personal_sendTransaction` and `personal_sign` therefore provides zero access control.

**Namespace enablement**

The `personal` namespace is not in the default list but is routinely enabled in documented deployment examples:

```bash
# init.sh
ethermintd start ... --json-rpc.api eth,txpool,personal,net,debug,web3 ...
```

Once enabled, the endpoint is bound to the configured JSON-RPC address (default `0.0.0.0:8545`) with no per-method authentication.

---

### Impact Explanation

**Critical ‚Äî Unauthorized balance transfer of EVM-denom funds.**

An attacker who can reach the JSON-RPC port and knows any address held in the node's keyring (the validator address is on-chain public information) can:

1. Call `personal_sendTransaction` with `from` = validator/operator address, `to` = attacker address, `value` = full balance, and any string as the password.
2. The node signs the transaction with the real private key and broadcasts it.
3. All EVM-denom funds are transferred to the attacker with no further preconditions.

The same applies to `personal_sign`: an attacker can obtain a valid ECDSA signature over arbitrary data from any keyring address, enabling downstream authorization bypasses in contracts or off-chain systems that rely on node-signed messages.

---

### Likelihood Explanation

**Medium.** The `personal` namespace must be explicitly enabled, but:
- The canonical `init.sh` and documented quick-start guides enable it.
- Many production deployments expose the full JSON-RPC surface (including `personal`) on a public or semi-public port.
- The validator address required as `from` is always publicly visible on-chain.
- No exploit tooling is needed ‚Äî a single `curl` call suffices.

---

### Recommendation

1. **Enforce password verification**: Before signing in `SendTransaction` and `Sign`, decrypt the keyring entry with the supplied password and return an error if decryption fails. Use `keyring.ExportPrivKeyArmor` + `crypto.UnarmorDecryptPrivKey` (the same pattern used in `UnsafeExportEthKeyCommand`) to validate the password before any signing operation.
2. **Implement `UnlockAccount` / `LockAccount`**: Maintain an in-memory map of unlocked addresses with expiry; reject signing requests for locked addresses.
3. **Restrict the `personal` namespace by default**: Gate it behind an explicit opt-in flag distinct from the general `--json-rpc.api` list, and document that it must never be exposed on a public interface.
4. **Add IP allowlist or authentication middleware** for the `personal` namespace when it must be exposed remotely.

func (b *Backend) SendTransaction(args evmtypes.TransactionArgs) (common.Hash, error) {
	// Look up the wallet containing the requested signer
	_, err := b.clientCtx.Keyring.KeyByAddress(sdk.AccAddress(args.GetFrom().Bytes()))
	if err != nil {
		b.logger.Error("failed to find key in keyring", "address", args.GetFrom(), "error", err.Error())
		return common.Hash{}, fmt.Errorf("failed to find key in the node's keyring; %s; %s", keystore.ErrNoMatch, err.Error())
	}

	if args.ChainID != nil && (b.chainID).Cmp((*big.Int)(args.ChainID)) != 0 {
		return common.Hash{}, fmt.Errorf("chainId does not match node's (have=%v, want=%v)", args.ChainID, (*hexutil.Big)(b.chainID))
	}

	args, err = b.SetTxDefaults(args)
	if err != nil {
		return common.Hash{}, err
	}

	msg := args.ToTransaction()
	if err := msg.ValidateBasic(); err != nil {
		b.logger.Debug("tx failed basic validation", "error", err.Error())
		return common.Hash{}, err
	}

	bn, err := b.BlockNumber()
	if err != nil {
		b.logger.Debug("failed to fetch latest block number", "error", err.Error())
		return common.Hash{}, err
	}

	header, err := b.CurrentHeader()
	if err != nil {
		b.logger.Debug("failed to fetch latest block header", "error", err.Error())
		return common.Hash{}, err
	}

	signer := ethtypes.MakeSigner(b.ChainConfig(), new(big.Int).SetUint64(uint64(bn)), header.Time)

	// Sign transaction
	if err := msg.Sign(signer, b.clientCtx.Keyring); err != nil {
		b.logger.Debug("failed to sign tx", "error", err.Error())
		return common.Hash{}, err
	}

	// Query params to use the EVM denomination
	res, err := b.queryClient.Params(b.ctx, &evmtypes.QueryParamsRequest{})
	if err != nil {
		b.logger.Error("failed to query evm params", "error", err.Error())
		return common.Hash{}, err
	}

	// Assemble transaction from fields
	tx, err := msg.BuildTx(b.clientCtx.TxConfig.NewTxBuilder(), res.Params.EvmDenom)
	if err != nil {
		b.logger.Error("build cosmos tx failed", "error", err.Error())
		return common.Hash{}, err
	}

	// Encode transaction by default Tx encoder
	txEncoder := b.clientCtx.TxConfig.TxEncoder()
	txBytes, err := txEncoder(tx)
	if err != nil {
		b.logger.Error("failed to encode eth tx using default encoder", "error", err.Error())
		return common.Hash{}, err
	}

	ethTx := msg.AsTransaction()

	// check the local node config in case unprotected txs are disabled
	if !b.UnprotectedAllowed() && !ethTx.Protected() {
		// Ensure only eip155 signed transactions are submitted if EIP155Required is set.
		return common.Hash{}, errors.New("only replay-protected (EIP-155) transactions allowed over RPC")
	}

	txHash := ethTx.Hash()

	// Broadcast transaction in sync mode (default)
	// NOTE: If error is encountered on the node, the broadcast will not return an error
	rsp, err := b.broadcastTx(txBytes)
	if rsp != nil && rsp.Code != 0 {
		err = errorsmod.ABCIError(rsp.Codespace, rsp.Code, rsp.RawLog)
	}
	if err != nil {
		b.logger.Error("failed to broadcast tx", "error", err.Error())
		return txHash, err
	}

	// Return transaction hash
	return txHash, nil
}

// Sign signs the provided data using the private key of address via Geth's signature standard.
func (b *Backend) Sign(address common.Address, data hexutil.Bytes) (hexutil.Bytes, error) {
	from := sdk.AccAddress(address.Bytes())

	_, err := b.clientCtx.Keyring.KeyByAddress(from)
	if err != nil {
		b.logger.Error("failed to find key in keyring", "address", address.String())
		return nil, fmt.Errorf("%s; %s", keystore.ErrNoMatch, err.Error())
	}

	// Apply EIP-191 signed-message prefix to domain-separate personal
	// signatures from transaction signatures (matching Geth's eth_sign).
	signature, _, err := b.clientCtx.Keyring.SignByAddress(from, accounts.TextHash(data), signingtypes.SignMode_SIGN_MODE_TEXTUAL)
	if err != nil {
		b.logger.Error("keyring.SignByAddress failed", "address", address.Hex())
		return nil, err
	}

	signature[crypto.RecoveryIDOffset] += 27 // Transform V from 0/1 to 27/28 according to the yellow paper
	return signature, nil
}
