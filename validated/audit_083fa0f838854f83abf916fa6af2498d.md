### Title
Unrestricted `execute_meta_tx_v0` Allows Any Contract to Execute `__execute__` on Behalf of Any Victim Account — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

`execute_meta_tx_v0` is a syscall available to any executing contract. It accepts a caller-supplied `contract_address` and directly invokes that contract's `__execute__` entry point under a version-0 transaction context — with no check that the calling contract is authorized to act on behalf of the specified `contract_address`. This is a direct OS-level analog of the "unrestricted on-behalf withdrawal" pattern: an attacker contract can name any victim as the `contract_address` and the OS will execute the victim's `__execute__` without ever running `__validate__`.

---

### Finding Description

`execute_meta_tx_v0` in `syscall_impls.cairo` (lines 286–400) constructs a new `ExecutionContext` where:

- `contract_address` is taken verbatim from `request.contract_address` — fully attacker-controlled.
- `account_contract_address` in the new `TxInfo` is set to that same attacker-supplied address (line 345).
- `caller_address` is set to `ORIGIN_ADDRESS` (0) (line 374).
- The entry point executed is always `EXECUTE_ENTRY_POINT_SELECTOR` (`__execute__`) — enforced at line 317.
- `__validate__` is **never called** — `contract_call_helper` is invoked directly (line 389). [1](#0-0) 

The only checks performed are:
1. Gas sufficiency (line 304–310).
2. `selector == EXECUTE_ENTRY_POINT_SELECTOR` (line 317).
3. Signature array bounds (line 323).

**There is no assertion that `request.contract_address == caller_execution_context.execution_info.contract_address`**, i.e., no check that the calling contract is the same as the target account. [2](#0-1) 

This syscall is reachable by any contract execution via the standard `execute_syscalls` dispatcher (line 343–350 of `execute_syscalls.cairo`), with no caller restriction. [3](#0-2) 

The `run_validate` function explicitly skips `__validate__` for version-0 transactions (line 128–130 of `execute_transaction_utils.cairo`), and `execute_meta_tx_v0` does not call `run_validate` at all — it goes directly to `contract_call_helper`. [4](#0-3) 

Standard account contracts' `__execute__` entry points check that `get_caller_address() == 0` (i.e., the caller is the OS/origin). Since `caller_address` is set to `ORIGIN_ADDRESS` (0) at line 374, this check passes for any victim account. The victim's `__execute__` then runs the attacker-supplied calldata without any signature verification.

---

### Impact Explanation

**Critical — Direct loss of funds.**

An attacker can execute arbitrary calldata through any victim account's `__execute__` entry point. This includes transferring all ERC-20 tokens held by the victim, approving the attacker as a spender, or any other state-mutating action the victim account is capable of. Because `__validate__` is bypassed entirely and `caller_address = 0` satisfies the standard caller guard in `__execute__`, no victim-side defense prevents this.

---

### Likelihood Explanation

Any unprivileged transaction sender can deploy a contract that emits the `META_TX_V0` syscall with an arbitrary `contract_address`. The attacker needs only to:
1. Know the victim's contract address (public on-chain).
2. Craft calldata for the victim's `__execute__` (e.g., a token transfer to the attacker).
3. Submit a transaction that calls their malicious contract.

No privileged access, leaked keys, or social engineering is required. The attack is fully on-chain and deterministic.

---

### Recommendation

Add an authorization check inside `execute_meta_tx_v0` asserting that the target `contract_address` equals the calling contract's own address:

```cairo
// Enforce that a contract can only issue a meta-tx on its own behalf.
assert contract_address = caller_execution_context.execution_info.contract_address;
```

This mirrors the fix recommended in the reference report: enforce `msg.sender == _supplier` before processing an on-behalf action. [5](#0-4) 

---

### Proof of Concept

1. **Victim** (`0xVICTIM`) is a deployed account contract holding 1000 STRK. Its `__execute__` checks `assert get_caller_address() == 0` and then executes a multicall.

2. **Attacker** deploys `MaliciousContract` with the following logic in its `__execute__`:

```cairo
// Attacker's contract calls execute_meta_tx_v0 syscall:
// contract_address = 0xVICTIM
// selector        = EXECUTE_ENTRY_POINT_SELECTOR (__execute__)
// calldata        = [transfer(attacker_address, 1000_STRK)]
// signature       = [] (empty — __validate__ is never called)
starknet.meta_tx_v0(
    contract_address=VICTIM,
    selector=EXECUTE_ENTRY_POINT_SELECTOR,
    calldata=[transfer_calldata],
    signature=[],
)
```

3. **Attacker** submits an invoke transaction calling `MaliciousContract.__execute__`.

4. The OS dispatches `execute_meta_tx_v0`:
   - `contract_address = 0xVICTIM` (attacker-supplied, no auth check).
   - New `TxInfo`: `version=0`, `account_contract_address=0xVICTIM`, `caller_address=0`.
   - `__validate__` is **not called**.
   - `victim.__execute__([transfer_calldata])` runs with `caller_address=0` → passes the caller guard.
   - 1000 STRK is transferred to the attacker.

5. Victim's funds are drained with no action on the victim's part. [6](#0-5)

func execute_meta_tx_v0{
    range_check_ptr,
    syscall_ptr: felt*,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    revert_log: RevertLogEntry*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, caller_execution_context: ExecutionContext*) {
    alloc_locals;

    let request = cast(syscall_ptr + RequestHeader.SIZE, MetaTxV0Request*);
    local calldata_start: felt* = request.calldata_start;
    local calldata_size = request.calldata_end - calldata_start;

    let specific_base_gas_cost = (
        META_TX_V0_GAS_COST + META_TX_V0_CALLDATA_FACTOR_GAS_COST * calldata_size
    );
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=specific_base_gas_cost, request_struct_size=MetaTxV0Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    local contract_address = request.contract_address;
    local selector = request.selector;
    local caller_execution_info: ExecutionInfo* = caller_execution_context.execution_info;
    local old_tx_info: TxInfo* = caller_execution_info.tx_info;

    if (selector != EXECUTE_ENTRY_POINT_SELECTOR) {
        write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
        return ();
    }

    // Sanity check: Verify that `signature` is a valid Sierra array.
    assert_nn_le(request.signature_end - request.signature_start, SIERRA_ARRAY_LEN_BOUND - 1);

    let (state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
        key=contract_address
    );

    // Compute the meta-transaction hash.
    let pedersen_ptr = builtin_ptrs.selectable.pedersen;
    with pedersen_ptr {
        let meta_tx_hash = compute_meta_tx_v0_hash(
            contract_address=contract_address,
            entry_point_selector=selector,
            calldata=calldata_start,
            calldata_size=calldata_size,
            chain_id=old_tx_info.chain_id,
        );
    }
    update_pedersen_in_builtin_ptrs(pedersen_ptr=pedersen_ptr);

    // Prepare execution context.
    tempvar new_tx_info = new TxInfo(
        version=0,
        account_contract_address=contract_address,
        max_fee=0,
        signature_start=request.signature_start,
        signature_end=request.signature_end,
        transaction_hash=meta_tx_hash,
        chain_id=old_tx_info.chain_id,
        nonce=0,
        resource_bounds_start=cast(0, ResourceBounds*),
        resource_bounds_end=cast(0, ResourceBounds*),
        tip=0,
        paymaster_data_start=cast(0, felt*),
        paymaster_data_end=cast(0, felt*),
        nonce_data_availability_mode=0,
        fee_data_availability_mode=0,
        account_deployment_data_start=cast(0, felt*),
        account_deployment_data_end=cast(0, felt*),
        proof_facts_start=cast(0, felt*),
        proof_facts_end=cast(0, felt*),
    );

    let (deprecated_tx_info_ptr: DeprecatedTxInfo*) = alloc();
    tempvar execution_context: ExecutionContext* = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=state_entry.class_hash,
        calldata_size=calldata_size,
        calldata=calldata_start,
        execution_info=new ExecutionInfo(
            block_info=caller_execution_info.block_info,
            tx_info=new_tx_info,
            caller_address=ORIGIN_ADDRESS,
            contract_address=contract_address,
            selector=selector,
        ),
        deprecated_tx_info=deprecated_tx_info_ptr,
    );
    fill_deprecated_tx_info(tx_info=new_tx_info, dst=execution_context.deprecated_tx_info);

    // Since we process the revert log backwards, entries before this point belong to the calling
    // contract.
    assert [revert_log] = RevertLogEntry(
        selector=CHANGE_CONTRACT_ENTRY, value=caller_execution_info.contract_address
    );
    let revert_log = &revert_log[1];

    contract_call_helper(
        remaining_gas=remaining_gas,
        block_context=block_context,
        execution_context=execution_context,
    );

    // Entries before this point belong to the callee.
    assert [revert_log] = RevertLogEntry(selector=CHANGE_CONTRACT_ENTRY, value=contract_address);
    let revert_log = &revert_log[1];

    return ();
}
