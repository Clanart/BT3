Looking at the actual code for `compute_meta_tx_v0_hash` and `execute_meta_tx_v0` to verify the claim precisely.

### Title
Meta-tx v0 hash excludes outer-transaction context, enabling cross-outer-tx signature replay — (`crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`, `transaction_hash/transaction_hash.cairo`)

---

### Summary

`compute_meta_tx_v0_hash` hashes only `(INVOKE_HASH_PREFIX, version=0, contract_address, selector, calldata, max_fee=0, chain_id)` with no outer-transaction hash, no outer-transaction nonce, and no per-invocation counter. `execute_meta_tx_v0` also sets `nonce=0` in the inner `TxInfo` and never increments the target contract's nonce. Two distinct outer invoke transactions carrying identical `MetaTxV0Request` fields therefore produce an identical `meta_tx_hash`, and the target account's `__execute__` sees the same `(transaction_hash, signature)` pair both times, passing signature validation twice.

---

### Finding Description

**`compute_meta_tx_v0_hash` inputs (transaction_hash.cairo lines 295–315):**

```
INVOKE_HASH_PREFIX | version=0 | contract_address | selector
| calldata | max_fee=0 | chain_id | (no additional_data)
``` [1](#0-0) 

`additional_data_size=0` is passed explicitly, so no nonce, no outer-tx hash, and no replay counter enters the hash.

**`execute_meta_tx_v0` calls the hash function with only outer `chain_id`:**

```cairo
let meta_tx_hash = compute_meta_tx_v0_hash(
    contract_address=contract_address,
    entry_point_selector=selector,
    calldata=calldata_start,
    calldata_size=calldata_size,
    chain_id=old_tx_info.chain_id,   // ← only outer-tx field used
);
``` [2](#0-1) 

**Inner `TxInfo` is constructed with `nonce=0` and no uniqueness anchor:** [3](#0-2) 

The target contract's `StateEntry` nonce is never incremented inside `execute_meta_tx_v0`; only the outer transaction's nonce (managed elsewhere) prevents replay of the *outer* transaction, not of the embedded meta-tx.

---

### Impact Explanation

If a victim account's `__execute__` transfers tokens (the canonical use-case for meta-transactions / gasless relaying), an attacker who possesses a valid `(calldata, signature)` tuple can embed it in two separate outer invoke transactions. Both outer transactions carry different nonces and are independently valid. The OS executes the meta-tx twice, the target account validates the same `meta_tx_hash` and signature twice, and the token transfer executes twice — **direct loss of funds via double-spend**.

---

### Likelihood Explanation

The attacker must possess a valid victim signature. This is realistic in the primary intended use-case: gasless relaying, where the victim signs a meta-tx and hands it to a relayer. Any party who observes the signed meta-tx (the relayer itself, a mempool observer, or anyone who sees the first outer transaction on-chain) can replay it in a second outer transaction. No privileged access, leaked key, or operator collusion is required beyond obtaining the already-broadcast signed payload.

---

### Recommendation

Bind the meta-tx hash to a unique per-invocation value that cannot be reused across outer transactions. Two standard approaches:

1. **Include the outer transaction hash**: pass `old_tx_info.transaction_hash` as an `additional_data` element to `deprecated_get_transaction_hash`. This ties each meta-tx execution to exactly one outer transaction.
2. **Introduce a meta-tx nonce on the target account**: increment the target contract's nonce in `execute_meta_tx_v0` (analogous to how top-level invoke transactions increment the sender's nonce) and include it in the hash.

Either change makes `meta_tx_hash` unique per outer transaction, breaking the replay path.

---

### Proof of Concept

1. Deploy a relay contract `R` that accepts `(calldata, sig)` and issues a `meta_tx_v0` syscall targeting victim account `V`.
2. Victim `V` signs `meta_tx_hash = compute_meta_tx_v0_hash(V, __execute__, calldata, chain_id)` and provides `sig`.
3. Submit outer invoke tx₁ (nonce=N) calling `R` with `(calldata, sig)`.
4. Submit outer invoke tx₂ (nonce=N+1) calling `R` with the same `(calldata, sig)`.
5. Both outer transactions are accepted (different nonces).
6. Assert: `compute_meta_tx_v0_hash` returns the same value for both — confirmed by inspection of the hash inputs above.
7. `V.__execute__` validates `sig` against the same hash in both calls and executes the token transfer twice. [4](#0-3) [1](#0-0)

func execute_meta_tx_v0{
    range_check_ptr,
    syscall_ptr: felt*,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    revert_log: RevertLogEntry*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, caller_execution_context: ExecutionContext*) {
    alloc_locals;

    let request = cast(syscall_ptr + RequestHeader.SIZE, MetaTxV0Request*);
    local calldata_start: felt* = request.calldata_start;
    local calldata_size = request.calldata_end - calldata_start;

    let specific_base_gas_cost = (
        META_TX_V0_GAS_COST + META_TX_V0_CALLDATA_FACTOR_GAS_COST * calldata_size
    );
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=specific_base_gas_cost, request_struct_size=MetaTxV0Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    local contract_address = request.contract_address;
    local selector = request.selector;
    local caller_execution_info: ExecutionInfo* = caller_execution_context.execution_info;
    local old_tx_info: TxInfo* = caller_execution_info.tx_info;

    if (selector != EXECUTE_ENTRY_POINT_SELECTOR) {
        write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
        return ();
    }

    // Sanity check: Verify that `signature` is a valid Sierra array.
    assert_nn_le(request.signature_end - request.signature_start, SIERRA_ARRAY_LEN_BOUND - 1);

    let (state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
        key=contract_address
    );

    // Compute the meta-transaction hash.
    let pedersen_ptr = builtin_ptrs.selectable.pedersen;
    with pedersen_ptr {
        let meta_tx_hash = compute_meta_tx_v0_hash(
            contract_address=contract_address,
            entry_point_selector=selector,
            calldata=calldata_start,
            calldata_size=calldata_size,
            chain_id=old_tx_info.chain_id,
        );
    }
    update_pedersen_in_builtin_ptrs(pedersen_ptr=pedersen_ptr);

    // Prepare execution context.
    tempvar new_tx_info = new TxInfo(
        version=0,
        account_contract_address=contract_address,
        max_fee=0,
        signature_start=request.signature_start,
        signature_end=request.signature_end,
        transaction_hash=meta_tx_hash,
        chain_id=old_tx_info.chain_id,
        nonce=0,
        resource_bounds_start=cast(0, ResourceBounds*),
        resource_bounds_end=cast(0, ResourceBounds*),
        tip=0,
        paymaster_data_start=cast(0, felt*),
        paymaster_data_end=cast(0, felt*),
        nonce_data_availability_mode=0,
        fee_data_availability_mode=0,
        account_deployment_data_start=cast(0, felt*),
        account_deployment_data_end=cast(0, felt*),
        proof_facts_start=cast(0, felt*),
        proof_facts_end=cast(0, felt*),
    );

    let (deprecated_tx_info_ptr: DeprecatedTxInfo*) = alloc();
    tempvar execution_context: ExecutionContext* = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=state_entry.class_hash,
        calldata_size=calldata_size,
        calldata=calldata_start,
        execution_info=new ExecutionInfo(
            block_info=caller_execution_info.block_info,
            tx_info=new_tx_info,
            caller_address=ORIGIN_ADDRESS,
            contract_address=contract_address,
            selector=selector,
        ),
        deprecated_tx_info=deprecated_tx_info_ptr,
    );
    fill_deprecated_tx_info(tx_info=new_tx_info, dst=execution_context.deprecated_tx_info);

    // Since we process the revert log backwards, entries before this point belong to the calling
    // contract.
    assert [revert_log] = RevertLogEntry(
        selector=CHANGE_CONTRACT_ENTRY, value=caller_execution_info.contract_address
    );
    let revert_log = &revert_log[1];

    contract_call_helper(
        remaining_gas=remaining_gas,
        block_context=block_context,
        execution_context=execution_context,
    );

    // Entries before this point belong to the callee.
    assert [revert_log] = RevertLogEntry(selector=CHANGE_CONTRACT_ENTRY, value=contract_address);
    let revert_log = &revert_log[1];

    return ();
}
