### Title
Stale `feemarketParams` Snapshot in Cosmos Ante Handler Enables Persistent Fee Market Bypass — (`File: evmd/ante/handler_options.go`)

---

### Summary

`newCosmosAnteHandler` captures `feemarketParams` once at app-startup time and passes a pointer to that frozen local variable into both `NewDynamicFeeChecker` and `NewMinGasPriceDecorator`. Because `BeginBlock` updates the base fee in the KV store every block but the Cosmos ante handler never re-reads it, every Cosmos transaction is fee-validated against a stale base fee and a stale `MinGasPrice`. The EVM ante handler (`newEthAnteHandler`) does not share this flaw — it calls `EVMBlockConfig` fresh on every invocation. The asymmetry is the root cause.

---

### Finding Description

In `evmd/ante/handler_options.go`, `newCosmosAnteHandler` is constructed once at app startup:

```go
func newCosmosAnteHandler(ctx sdk.Context, options HandlerOptions, ...) sdk.AnteHandler {
    evmParams        := options.EvmKeeper.GetParams(ctx)       // snapshot
    feemarketParams  := options.FeeMarketKeeper.GetParams(ctx) // snapshot – never refreshed
    ...
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
    ...
    cosmos.NewMinGasPriceDecorator(options.FeeMarketKeeper, evmDenom, &feemarketParams),
    ...
    ante.NewDeductFeeDecorator(..., txFeeChecker),
``` [1](#0-0) 

Both decorators receive `&feemarketParams` — a pointer to a local variable whose value is frozen at construction time. Go's escape analysis moves it to the heap, so the pointer stays valid, but the value it points to is never mutated again.

`NewDynamicFeeChecker` uses that pointer directly:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
// feemarketParams is the stale snapshot
if feeCap.LT(baseFeeInt) { return error }
``` [2](#0-1) 

`MinGasPriceDecorator` does the same:

```go
minGasPrice := mpd.feemarketParams.MinGasPrice  // stale pointer
``` [3](#0-2) 

Meanwhile, `BeginBlock` updates the base fee in the KV store every block:

```go
func (k *Keeper) BeginBlock(ctx sdk.Context) error {
    baseFee := k.CalculateBaseFee(ctx)
    k.SetBaseFee(ctx, baseFee)   // writes to KV store
``` [4](#0-3) 

`SetBaseFee` writes into `params.BaseFee` in the KV store:

```go
func (k Keeper) SetBaseFee(ctx sdk.Context, baseFee *big.Int) {
    params := k.GetParams(ctx)
    params.BaseFee = ethermint.SaturatedNewInt(baseFee)
    k.SetParams(ctx, params)
}
``` [5](#0-4) 

The Cosmos ante handler's frozen `feemarketParams.BaseFee` is never updated to reflect these writes.

By contrast, `newEthAnteHandler` calls `EVMBlockConfig` on every invocation, which reads fresh params from the object store (populated from the KV store at the start of each block):

```go
blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
baseFee := blockCfg.BaseFee   // always current
``` [6](#0-5) 

`EVMBlockConfig` reads `feemarketParams.GetBaseFee()` fresh from the KV store on the first call per block:

```go
feemarketParams := k.feeMarketKeeper.GetParams(ctx)
baseFee = feemarketParams.GetBaseFee()
``` [7](#0-6) 

---

### Impact Explanation

**Fee market bypass for Cosmos transactions.** Any Cosmos SDK transaction (non-EVM) that pays fees in the EVM denom — including those using `ExtensionOptionDynamicFeeTx` — is validated against the base fee and `MinGasPrice` that were current at the last app startup, not the current block's values.

1. **Underpayment of fees**: If the base fee has risen since startup (e.g., due to sustained high block utilization), a user can submit a Cosmos transaction with `feeCap = stale_base_fee < current_base_fee`. `NewDynamicFeeChecker` accepts it; the fee collector receives `stale_base_fee * gas` instead of `current_base_fee * gas`. EVM-denom funds are mis-accounted.

2. **`MinGasPrice` governance bypass**: If `MinGasPrice` is raised via governance after startup, `MinGasPriceDecorator` for Cosmos transactions still enforces the old lower bound. Cosmos transactions that should be rejected are admitted.

3. **Potential non-determinism across validators**: Validators that restart at different block heights capture different stale snapshots. A Cosmos transaction whose fee falls between two validators' stale base fees would be accepted by one and rejected by the other during `FinalizeBlock`, risking consensus divergence.

This matches the allowed High impact: *"ante handler … bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

---

### Likelihood Explanation

- The base fee changes every block (EIP-1559 formula in `CalculateBaseFee`). After a single block, the Cosmos ante handler is already using a stale value.
- No special privileges are required. Any user submitting a Cosmos transaction with `ExtensionOptionDynamicFeeTx` is affected.
- The discrepancy compounds over time: a chain running for thousands of blocks under load can have a base fee orders of magnitude higher than the startup value.
- Governance `MinGasPrice` changes take effect immediately for EVM transactions but never for Cosmos transactions, making the bypass permanent until the next node restart.

---

### Recommendation

Replace the frozen local-variable snapshot with a per-invocation read from the keeper inside the returned closure, mirroring the pattern used by `newEthAnteHandler`:

```go
// In newCosmosAnteHandler, replace:
feemarketParams := options.FeeMarketKeeper.GetParams(ctx)
cosmos.NewMinGasPriceDecorator(options.FeeMarketKeeper, evmDenom, &feemarketParams)
evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)

// With a decorator that reads fresh params on each AnteHandle call:
// e.g., MinGasPriceDecorator.AnteHandle should call mpd.feesKeeper.GetParams(ctx)
// and NewDynamicFeeChecker should accept a keeper interface, not a *Params pointer.
```

`MinGasPriceDecorator` already stores `feesKeeper interfaces.FeeMarketKeeper` but never uses it in `AnteHandle` — it should call `mpd.feesKeeper.GetParams(ctx).MinGasPrice` instead of reading from the stale pointer. [8](#0-7) 

---

### Proof of Concept

1. Start the chain with `EnableHeight=1`, `BaseFee=1_000_000_000` (1 Gwei), `MinGasPrice=0`.
2. Flood the chain with EVM transactions for N blocks until `CalculateBaseFee` drives the base fee to `10_000_000_000` (10 Gwei). The KV store now holds `BaseFee=10 Gwei`.
3. Construct a Cosmos SDK transaction with `ExtensionOptionDynamicFeeTx{MaxPriorityPrice: 0}` and `fee = 1 Gwei * gasLimit` (i.e., `feeCap = 1 Gwei`).
4. Submit the transaction. `newEthAnteHandler` would reject an EVM tx with `GasFeeCap < baseFee` at step `CheckEthCanTransfer`. But `newCosmosAnteHandler`'s `NewDynamicFeeChecker` reads `baseFee` from the frozen `feemarketParams` (still `1 Gwei`), so `feeCap >= baseFeeInt` passes, and the transaction is included with a fee 10× below the current market rate.
5. The fee collector receives `1 Gwei * gas` instead of `10 Gwei * gas`, mis-accounting EVM-denom funds. [1](#0-0) [9](#0-8)

func NewDynamicFeeChecker(ethCfg *params.ChainConfig, evmParams *types.Params, feemarketParams *feemarkettypes.Params) authante.TxFeeChecker {
	return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
		feeTx, ok := tx.(sdk.FeeTx)
		if !ok {
			return nil, 0, fmt.Errorf("tx must be a FeeTx")
		}

		if ctx.BlockHeight() == 0 {
			// genesis transactions: fallback to min-gas-price logic
			return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
		}

		denom := evmParams.EvmDenom

		baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
		if baseFee == nil {
			// london hardfork is not enabled: fallback to min-gas-prices logic
			return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
		}

		// default to `MaxInt64` when there's no extension option.
		maxPriorityPrice := sdkmath.NewInt(math.MaxInt64)

		// get the priority tip cap from the extension option.
		if hasExtOptsTx, ok := tx.(authante.HasExtensionOptionsTx); ok {
			for _, opt := range hasExtOptsTx.GetExtensionOptions() {
				if extOpt, ok := opt.GetCachedValue().(*ethermint.ExtensionOptionDynamicFeeTx); ok {
					maxPriorityPrice = extOpt.MaxPriorityPrice
					break
				}
			}
		}

		if maxPriorityPrice.Sign() == -1 {
			return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, "priority fee is negative")
		}

		gas := feeTx.GetGas()
		feeCoins := feeTx.GetFee()
		fee := feeCoins.AmountOf(denom)

		feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
		baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)

		if feeCap.LT(baseFeeInt) {
			return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, "insufficient gas prices; got: %s required: %s", feeCap, baseFeeInt)
		}

		// calculate the effective gas price using the EIP-1559 logic.
		effectivePrice := sdkmath.NewIntFromBigInt(types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))

		// NOTE: create a new coins slice without having to validate the denom
		effectiveFee := sdk.Coins{
			{
				Denom:  denom,
				Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas)),
			},
		}

		bigPriority := effectivePrice.Sub(baseFeeInt).Quo(types.DefaultPriorityReduction)
		priority := int64(math.MaxInt64)

		if bigPriority.IsInt64() {
			priority = bigPriority.Int64()
		}

		return effectiveFee, priority, nil
	}

func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
