### Title
Unauthenticated Bootstrap Path in `execute_declare_transaction` Allows Fee-Free Class Declarations Without Signature Verification — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

The `execute_declare_transaction` function in the StarkNet OS contains a special "bootstrap" code path that completely bypasses signature verification, nonce enforcement, and fee charging. The only guard is a check that `sender_address == 'BOOTSTRAP'` (a plain felt value), `nonce == 0`, `version == 3`, and `max_possible_fee == 0` — all conditions trivially satisfiable by any transaction sender. Because the OS is the reference enforcement layer for the protocol, this missing access control allows fee-free, signature-free class declarations to be proven as valid.

---

### Finding Description

In `execute_declare_transaction`, after computing the transaction hash and class hash, the following branch exists:

```cairo
// Do not run validate or perform any account-related actions for declare transactions that
// meet the following conditions.
// This flow is used for the sequencer to bootstrap a new system.
if (sender_address == 'BOOTSTRAP' and tx_info.nonce == 0 and tx_info.version == 3) {
    let max_possible_fee = compute_max_possible_fee(tx_info=tx_info);
    if (max_possible_fee == 0) {
        assert_not_zero(compiled_class_hash);
        dict_update{dict_ptr=contract_class_changes}(
            key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
        );
        %{ SkipTx %}
        return ();
    }
}
``` [1](#0-0) 

When this branch is taken, the OS skips:
1. **`check_and_increment_nonce`** — no nonce enforcement
2. **`run_validate`** (i.e., `__validate_declare__`) — no signature verification
3. **`charge_fee`** — no fee payment [2](#0-1) 

The four conditions required to enter this path are:

| Condition | Attacker control |
|---|---|
| `sender_address == 'BOOTSTRAP'` | Set in transaction fields (felt literal, no crypto binding) |
| `nonce == 0` | Set in transaction fields |
| `version == 3` | Set in transaction fields |
| All resource bounds = 0 → `max_possible_fee == 0` | Set in transaction fields |

`compute_max_possible_fee` returns `l1_gas.max_amount * l1_gas.max_price + l2_gas.max_amount * (l2_gas.max_price + tip) + l1_data_gas.max_amount * l1_data_gas.max_price`, which is zero when all bounds are zero. [3](#0-2) 

There is **no cryptographic check** that the sender is the legitimate owner of the `'BOOTSTRAP'` address. The OS does not verify a signature from that address, does not check that a contract is deployed there, and does not restrict which sequencer or user may trigger this path. The `'BOOTSTRAP'` felt value (`0x424f4f545354524150`) is not a privileged key — it is an arbitrary constant that any transaction can claim as its `sender_address`.

---

### Impact Explanation

**Direct loss of funds (Critical).** The `charge_fee` call is entirely skipped for any declare transaction satisfying the four trivial conditions. The fee token transfer to the sequencer address never occurs:

```cairo
// charge_fee is called for normal declare transactions:
charge_fee(
    block_context=block_context, tx_execution_context=validate_declare_execution_context
);
``` [4](#0-3) 

Because the OS is the proof-enforcing layer of the protocol, a block containing such a transaction will produce a valid STARK proof with zero fees charged. The fee collector (sequencer) receives no STRK/ETH for the class declaration. Additionally, `__validate_declare__` is never called, meaning the class is registered in `contract_class_changes` without any account authorization — any class hash can be declared by anyone who can get such a transaction included in a block.

---

### Likelihood Explanation

The OS is the canonical protocol enforcement layer. Because the OS skips signature verification for the bootstrap path, the sequencer's own pre-inclusion validation logic is expected to mirror this rule (the sequencer implements the same protocol). If the sequencer's mempool/gateway also exempts `'BOOTSTRAP'`-addressed declare transactions from signature checks (consistent with the OS), an unprivileged user can submit such a transaction directly. Even if the gateway independently enforces signatures, a sequencer operator (or a sequencer with a validation gap) can include the transaction, and the OS will produce a valid proof — making the fee bypass provably correct on-chain with no recourse.

---

### Recommendation

The bootstrap path must be cryptographically gated. Options:

1. **Remove the bootstrap path entirely** once the system is live; bootstrap should be a one-time, off-chain operation.
2. **Require a valid signature** from a hardcoded, well-known bootstrap public key (analogous to role-based access control), verified inside the OS before entering the fee-free path.
3. **Restrict by block number**: only allow the bootstrap path for blocks below a fixed threshold, enforced with a Cairo `assert`.

The current check `sender_address == 'BOOTSTRAP'` provides no cryptographic access control and is the direct analog of the `setPreferredFeeType()` missing-access-control bug in the external report.

---

### Proof of Concept

1. Attacker crafts a `DECLARE` transaction with:
   - `sender_address = 'BOOTSTRAP'` (felt `0x424f4f545354524150`)
   - `nonce = 0`
   - `version = 3`
   - `resource_bounds`: all `max_amount = 0`, all `max_price_per_unit = 0`, `tip = 0`
   - `compiled_class_hash`: any valid non-zero compiled class hash
   - `class_hash`: any Sierra class hash pre-image

2. The transaction is submitted to the sequencer (or included directly by a sequencer operator).

3. The OS executes `execute_declare_transaction`:
   - `compute_declare_transaction_hash` runs normally.
   - The bootstrap `if` branch is entered (all four conditions satisfied).
   - `compute_max_possible_fee` returns `0`.
   - `dict_update` registers the class hash → compiled class hash mapping.
   - `%{ SkipTx %}` is called; the function returns.
   - `check_and_increment_nonce`, `run_validate`, and `charge_fee` are **never reached**.

4. The STARK proof is valid. The class is declared on-chain. Zero fees are paid. No signature from `'BOOTSTRAP'` was ever required. [5](#0-4)

func compute_max_possible_fee(tx_info: TxInfo*) -> felt {
    tempvar resource_bounds: ResourceBounds* = tx_info.resource_bounds_start;
    let n_resource_bounds = (tx_info.resource_bounds_end - resource_bounds) / ResourceBounds.SIZE;

    // Only V3 transactions with all resource bounds are supported.
    assert tx_info.version = 3;
    assert n_resource_bounds = 3;

    tempvar l1_gas_bounds: ResourceBounds = resource_bounds[L1_GAS_INDEX];
    tempvar l2_gas_bounds: ResourceBounds = resource_bounds[L2_GAS_INDEX];
    tempvar l1_data_gas_bounds = resource_bounds[L1_DATA_GAS_INDEX];

    return l1_gas_bounds.max_amount * l1_gas_bounds.max_price_per_unit + l2_gas_bounds.max_amount *
        (l2_gas_bounds.max_price_per_unit + tx_info.tip) + l1_data_gas_bounds.max_amount *
        l1_data_gas_bounds.max_price_per_unit;
}

    range_check_ptr,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*) {
    alloc_locals;

    local tx_version;
    %{ TxVersion %}
    if (tx_version == 0) {
        %{ SkipTx %}
        return ();
    }

    // Guess transaction fields.
    local sender_address;
    local class_hash_ptr: felt*;
    local compiled_class_hash;
    local account_deployment_data_size;
    local account_deployment_data: felt*;
    %{ DeclareTxFields %}
    let common_tx_fields = get_account_tx_common_fields(
        block_context=block_context,
        tx_hash_prefix=DECLARE_HASH_PREFIX,
        sender_address=sender_address,
    );

    let poseidon_ptr = builtin_ptrs.selectable.poseidon;
    with poseidon_ptr {
        // Compute transaction hash.
        let transaction_hash = compute_declare_transaction_hash(
            common_fields=common_tx_fields,
            class_hash=[class_hash_ptr],
            compiled_class_hash=compiled_class_hash,
            account_deployment_data_size=account_deployment_data_size,
            account_deployment_data=account_deployment_data,
        );
        %{ AssertTransactionHash %}

        // Ensure the given class hash is a result of a Sierra class hash calculation.
        local contract_class_component_hashes: ContractClassComponentHashes*;
        %{ SetComponentHashes %}

        let expected_class_hash = finalize_class_hash(
            contract_class_component_hashes=contract_class_component_hashes
        );
        with_attr error_message("Invalid class hash pre-image.") {
            assert [class_hash_ptr] = expected_class_hash;
        }
    }
    update_poseidon_in_builtin_ptrs(poseidon_ptr=poseidon_ptr);

    // Get the account transaction info.
    let (tx_info: TxInfo*) = alloc();
    let (deprecated_tx_info: DeprecatedTxInfo*) = alloc();
    fill_account_tx_info(
        transaction_hash=transaction_hash,
        common_tx_fields=common_tx_fields,
        account_deployment_data_size=account_deployment_data_size,
        account_deployment_data=account_deployment_data,
        proof_facts_size=0,
        proof_facts=cast(0, felt*),
        tx_info_dst=tx_info,
        deprecated_tx_info_dst=deprecated_tx_info,
    );

    // Do not run validate or perform any account-related actions for declare transactions that
    // meet the following conditions.
    // This flow is used for the sequencer to bootstrap a new system.
    if (sender_address == 'BOOTSTRAP' and tx_info.nonce == 0 and tx_info.version == 3) {
        let max_possible_fee = compute_max_possible_fee(tx_info=tx_info);
        if (max_possible_fee == 0) {
            // Declare the class hash and skip the rest of the transaction.
            // Note that prev_value=0 enforces that a class may be declared only once.
            assert_not_zero(compiled_class_hash);
            dict_update{dict_ptr=contract_class_changes}(
                key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
            );
            %{ SkipTx %}
            return ();
        }
    }
