### Title
Missing Declared Class Validation in `execute_replace_class` Allows Permanent Fund Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function accepts an arbitrary attacker-controlled `class_hash` from the syscall request and writes it directly into the contract's state entry with no check that the hash corresponds to a declared class. After the replacement, any future call to the contract will fail irrecoverably inside `execute_entry_point`, permanently freezing all funds held by the contract. The missing check is explicitly acknowledged by a TODO comment in the code.

---

### Finding Description

In `execute_replace_class`, the `class_hash` field is read directly from the syscall request and applied to the contract's `StateEntry` without any validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges the missing check. There is no assertion that `class_hash` exists in `contract_class_changes`.

When any subsequent transaction attempts to call the affected contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [2](#0-1) 

For an undeclared `class_hash`, `dict_read` returns `0` (the default uninitialized value). `find_element` then fails to locate a compiled class with hash `0`, making the contract permanently unexecutable. The state commitment is already written; there is no recovery path.

The analog to M-02 is direct: in M-02, an unvalidated external contract address bypassed a `balanceOf`/`transferFrom` check to drain funds. Here, an unvalidated `class_hash` bypasses the declared-class check to permanently corrupt a contract's state, freezing its funds.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any contract that calls `replace_class` with an undeclared class hash will have its `class_hash` state permanently set to an invalid value. All assets (tokens, native currency equivalents) held by the contract become permanently inaccessible because no future execution of the contract can succeed. The state is committed to the Merkle tree and cannot be reversed without a protocol-level intervention.

---

### Likelihood Explanation

**High.** The `replace_class` syscall is available to any deployed contract. An attacker who controls a contract — including one that holds other users' funds (e.g., a vault, AMM, or bridge) — can call `replace_class` with an arbitrary undeclared hash at any time. No privileged role is required. The missing check is explicitly flagged in the source code as a known gap (`TODO`), confirming it is not an oversight of the auditor but a confirmed unimplemented guard.

---

### Recommendation

Before writing the new class hash to the contract state, verify that it corresponds to a declared class by asserting a non-zero compiled class hash exists in `contract_class_changes`:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the validation already performed implicitly in `execute_entry_point` but makes it an explicit, enforced precondition at the point of replacement.

---

### Proof of Concept

1. Attacker deploys a contract `VaultContract` that holds user-deposited funds and exposes a `replace_class` syscall call path.
2. Attacker submits a transaction invoking `VaultContract` with a call that triggers the `replace_class` syscall with `class_hash = 0xBADC0DE` (any value not present in `contract_class_changes`).
3. `execute_replace_class` reads `request.class_hash = 0xBADC0DE` and writes it directly into `contract_state_changes` for `VaultContract` with no validation. [1](#0-0) 
4. The block is proven and committed. `VaultContract`'s `StateEntry.class_hash` is now `0xBADC0DE`.
5. Any future transaction calling `VaultContract` reaches `execute_entry_point`:
   - `dict_read{dict_ptr=contract_class_changes}(key=0xBADC0DE)` returns `0` (undeclared).
   - `find_element(..., key=0)` fails — no compiled class with hash `0` exists. [2](#0-1) 
6. `VaultContract` is permanently unexecutable. All deposited user funds are frozen with no recovery mechanism.
