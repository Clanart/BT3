### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary
The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary felt value as a new class hash without verifying that the hash corresponds to a previously declared class. A contract that calls `replace_class` with an undeclared class hash will have its class pointer permanently set to an invalid value, making every subsequent entry-point call fail. Any funds held in the contract are permanently frozen.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The developer-acknowledged TODO at line 898 confirms the missing check. The function writes the caller-supplied `class_hash` directly into `contract_state_changes` with no assertion that the hash exists in `contract_class_changes` (the declared-class dictionary).

When any subsequent transaction attempts to execute an entry point on this contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
``` [2](#0-1) 

If `class_hash` was never declared, `dict_read` returns 0 (the Cairo dict default for an uninitialized key). `find_element` then searches for a compiled class with hash 0, which does not exist, causing an irrecoverable assertion failure. The contract is permanently bricked.

The analog to the external report is direct: just as `startUnlock()` retroactively applies a new condition to users who locked tokens before the condition existed — here, `replace_class` applies a state transition (class pointer update) without enforcing the invariant that the new class must be declared, leaving the contract in an irreversible broken state.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any ERC-20 tokens, ETH, or other assets held in the storage of a contract whose class is replaced with an undeclared hash become permanently inaccessible. No entry point (including withdrawal or transfer functions) can ever be executed again on that contract. The state root commitment will include the frozen contract's storage, but no proof can ever be generated that moves those funds out.

---

### Likelihood Explanation

**Medium.**

The attack surface is any contract that:
1. Exposes a code path reachable by an unprivileged caller that internally invokes `replace_class`, or
2. Is a shared-custody contract (multisig, vault, escrow) where one party can call `replace_class` with a crafted hash to grief other depositors.

Because `replace_class` is a standard Sierra syscall available to all Cairo 1 contracts, any contract that calls it — even accidentally with a mistyped hash — triggers the freeze. The TODO comment in the production code confirms this is a known, unmitigated gap.

---

### Recommendation

Inside `execute_replace_class`, before writing the new class hash to `contract_state_changes`, assert that the hash exists in `contract_class_changes`:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=class_hash
);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the validation already performed implicitly in `execute_entry_point` and makes the check explicit and enforced at the point of the syscall, preventing the irreversible bad state from ever being committed.

---

### Proof of Concept

1. Attacker deploys a vault contract (Sierra/Cairo 1) that accepts deposits from multiple users. Users deposit funds.
2. Attacker calls a function on the vault that internally executes `replace_class(class_hash=0xdeadbeef)` where `0xdeadbeef` is never declared.
3. The OS processes the block. `execute_replace_class` writes `class_hash=0xdeadbeef` into `contract_state_changes` for the vault address with no validation.
4. The new state root is committed on-chain with the vault's class pointer set to `0xdeadbeef`.
5. Any subsequent transaction targeting the vault reaches `execute_entry_point`, which calls `dict_read{dict_ptr=contract_class_changes}(key=0xdeadbeef)` → returns 0. `find_element(..., key=0)` fails. The transaction reverts.
6. All user funds in the vault are permanently frozen with no recovery path. [3](#0-2) [4](#0-3)
