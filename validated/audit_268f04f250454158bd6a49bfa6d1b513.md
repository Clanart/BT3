### Title
Unauthenticated Class Declaration via Hardcoded Bootstrap Sender Address Bypass - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

### Summary
`execute_declare_transaction` contains a special "bootstrap" code path that completely skips signature verification, nonce enforcement, and fee charging when the user-supplied `sender_address` field equals the felt literal `'BOOTSTRAP'`. Because `sender_address` is an attacker-controlled transaction field, any unprivileged user can craft a declare transaction that satisfies all four conditions and register any class hash with an arbitrary `compiled_class_hash` — permanently poisoning the class registry without authorization.

### Finding Description

In `execute_declare_transaction`, after computing the transaction hash and verifying the Sierra class hash, the OS checks a set of conditions that are entirely derived from user-supplied transaction fields:

```cairo
if (sender_address == 'BOOTSTRAP' and tx_info.nonce == 0 and tx_info.version == 3) {
    let max_possible_fee = compute_max_possible_fee(tx_info=tx_info);
    if (max_possible_fee == 0) {
        assert_not_zero(compiled_class_hash);
        dict_update{dict_ptr=contract_class_changes}(
            key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
        );
        %{ SkipTx %}
        return ();
    }
}
``` [1](#0-0) 

When this branch is taken, the OS returns immediately, skipping:

- `check_and_increment_nonce` (line 779) — nonce validation against the on-chain state
- `non_reverting_select_execute_entry_point_func` with `VALIDATE_DECLARE_ENTRY_POINT_SELECTOR` (lines 804–806) — the account contract's `__validate_declare__` entry point, which is the sole signature-verification step
- `charge_fee` (line 822) — fee deduction [2](#0-1) 

All four triggering conditions are fully attacker-controlled:

| Condition | Source |
|---|---|
| `sender_address == 'BOOTSTRAP'` | Declare tx field, loaded via `%{ DeclareTxFields %}` |
| `tx_info.nonce == 0` | Declare tx field, loaded via `%{ LoadCommonTxFields %}` |
| `tx_info.version == 3` | Enforced by `compute_declare_transaction_hash` anyway |
| `max_possible_fee == 0` | All three resource bounds set to 0 by attacker | [3](#0-2) 

The `compiled_class_hash` is also attacker-controlled and is only checked to be non-zero (`assert_not_zero(compiled_class_hash)`), with no verification that it corresponds to the actual compiled form of the declared Sierra class. [4](#0-3) 

The transaction hash computation in `compute_declare_transaction_hash` commits to `sender_address`, `nonce`, `compiled_class_hash`, and all resource bounds, so the hash is self-consistent — the `%{ AssertTransactionHash %}` hint passes — but no cryptographic signature over that hash is ever verified in the bootstrap path. [5](#0-4) 

### Impact Explanation

An attacker who triggers the bootstrap path can:

1. **Front-run any legitimate class declaration** — because `dict_update` with `prev_value=0` enforces that a class hash can be written exactly once, the attacker can race to declare a target `class_hash` first with an arbitrary `compiled_class_hash` (e.g., a hash of a different, malicious compiled program, or simply a random non-zero felt).
2. **Permanently poison the class registry** — once written, the mapping `class_hash → compiled_class_hash` cannot be overwritten. The legitimate declarer can never correct it.
3. **Freeze funds in all contracts of that class** — any contract subsequently deployed under the poisoned `class_hash` will reference the wrong compiled code. Every call to such a contract will fail, permanently locking any ETH/tokens held by those contracts.

This satisfies the **Critical — Permanent freezing of funds** impact category.

### Likelihood Explanation

The attack requires no privileged access, no leaked keys, and no operator collusion. Any user who can submit a StarkNet declare transaction can set `sender_address` to the felt encoding of `'BOOTSTRAP'` (`0x424f4f545354524150`), set all resource bounds to zero, and set `nonce = 0`. The OS enforces no whitelist on `sender_address` and performs no out-of-band check that the bootstrap path is only reachable during system initialization. The bootstrap path is permanently open to any transaction sender.

### Recommendation

1. **Remove the bootstrap path from the OS entirely.** System initialization should be handled off-chain or through a privileged, sequencer-controlled mechanism that is not reachable via a user-submitted transaction.
2. If the bootstrap path must remain, **restrict it to a hardcoded, protocol-controlled address** (e.g., the sequencer's own address read from `block_context`) rather than a magic string that any user can supply as `sender_address`.
3. **Validate `compiled_class_hash`** against the actual compiled class in both the bootstrap and normal paths, not merely assert it is non-zero.

### Proof of Concept

An attacker submits a StarkNet declare transaction with the following fields:

```
sender_address    = 0x424f4f545354524150   // felt('BOOTSTRAP')
nonce             = 0
version           = 3
tip               = 0
l1_gas_bounds     = { resource: L1_GAS, max_amount: 0, max_price_per_unit: 0 }
l2_gas_bounds     = { resource: L2_GAS, max_amount: 0, max_price_per_unit: 0 }
l1_data_gas_bounds= { resource: L1_DATA_GAS, max_amount: 0, max_price_per_unit: 0 }
class_hash        = <any valid Sierra class hash H>
compiled_class_hash = 1   // arbitrary non-zero value, not the real compiled hash
```

The OS execution path:

1. `%{ DeclareTxFields %}` loads the attacker's fields — `sender_address = 'BOOTSTRAP'`, `compiled_class_hash = 1`.
2. `compute_declare_transaction_hash` produces a self-consistent hash; `%{ AssertTransactionHash %}` passes.
3. `finalize_class_hash` verifies `class_hash = H` is a valid Sierra class hash (the attacker uses a real Sierra class).
4. `fill_account_tx_info` populates `tx_info` with `nonce = 0`, `version = 3`, all resource bounds zero.
5. The condition at line 764 evaluates to true; `compute_max_possible_fee` returns 0.
6. `dict_update` writes `H → 1` into `contract_class_changes` with no signature check.
7. The OS returns, having declared class `H` with a poisoned `compiled_class_hash = 1`.

Any contract subsequently deployed under class `H` will be permanently non-executable, freezing all funds it holds. [6](#0-5)

func compute_declare_transaction_hash{range_check_ptr, poseidon_ptr: PoseidonBuiltin*}(
    common_fields: CommonTxFields*,
    class_hash: felt,
    compiled_class_hash: felt,
    account_deployment_data_size: felt,
    account_deployment_data: felt*,
) -> felt {
    alloc_locals;

    // TODO(Noa, 01/01/2026): remove the following `assert` once the field is supported.
    assert account_deployment_data_size = 0;
    with_attr error_message("Invalid transaction version: {version}.") {
        assert common_fields.version = 3;
    }

    let hash_state: PoseidonHashState = poseidon_hash_init();
    with hash_state {
        hash_tx_common_fields(common_fields=common_fields);
        poseidon_hash_update_with_nested_hash(
            data_ptr=account_deployment_data, data_length=account_deployment_data_size
        );
        // Add the class hash to the hash state.
        poseidon_hash_update_single(item=class_hash);
        poseidon_hash_update_single(item=compiled_class_hash);
    }
    let transaction_hash = poseidon_hash_finalize(hash_state=hash_state);

    return transaction_hash;
}
