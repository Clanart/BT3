Audit Report

## Title
Stranded Router ETH Consumed by Unprivileged WETH Swap — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary
`PeripheryPayments.pay` uses the router's ambient native ETH balance to fund WETH-input swaps for any external caller, without verifying that the ETH was contributed by the current caller. Any ETH left on the router by a prior user who omitted `refundETH()` can be silently consumed by an attacker who calls `exactInputSingle` with `tokenIn=WETH`, `msg.value=0`, and zero WETH approval, receiving real swap output at no cost.

## Finding Description
**Root cause:** In `pay` (`PeripheryPayments.sol` L73–84), when `token == WETH` and `payer != address(this)`, the function checks `address(this).balance` and, if sufficient, wraps the router's own ETH and transfers the resulting WETH to the pool. The payer's WETH balance and allowance are never consulted in this branch. There is no check that the ambient ETH balance was contributed by the current top-level call.

**ETH stranding precondition:** `exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`, and `multicall` are all `payable`. A user who sends `msg.value > amountIn` for a WETH-input swap without appending `refundETH()` leaves the surplus on the router. The `receive()` guard (L32–34) only rejects direct ETH pushes from non-WETH addresses; it does not prevent ETH from accumulating via `msg.value` in legitimate function calls.

**Exploit path:**
1. User A calls `exactInputSingle{value: 2*V}(tokenIn=WETH, amountIn=V)` without `refundETH()` → `V` ETH stranded on router.
2. Attacker calls `exactInputSingle(tokenIn=WETH, amountIn=V, msg.value=0)` with zero WETH approval.
3. Router stores `payer=attacker`, `tokenToPay=WETH` in transient storage (L71), then calls `pool.swap(...)`.
4. Pool calls back `metricOmmSwapCallback` → `_justPayCallback` → `pay(WETH, attacker, pool, V)`.
5. Inside `pay`: `token == WETH`, `payer != address(this)`, `address(this).balance >= V` → router wraps its own ETH and transfers WETH to pool (L75–77). Attacker's WETH balance/approval never touched.
6. Attacker receives swap output; user A's `V` ETH is permanently consumed.

**Existing guards insufficient:** The `_requireExpectedCallbackCaller` check (L49) only validates that the callback comes from the expected pool, not that the payer has authorized the ETH spend. The `receive()` guard is irrelevant to this path.

## Impact Explanation
Direct, irreversible loss of user principal (native ETH). Any ETH stranded on the router — a routine outcome of the documented multicall pattern when `refundETH()` is omitted — is freely claimable by any address. The attacker pays nothing and receives real ERC-20 swap output. This is a direct loss of user funds meeting the High severity threshold.

## Likelihood Explanation
The Uniswap v3 multicall pattern (send ETH → swap → `refundETH`) is the documented usage model. Omitting `refundETH()` is a common user mistake. Additionally, a front-running attacker can observe a pending multicall that sends excess ETH and exploit it before `refundETH()` executes. No special permissions, no malicious pool, and no non-standard tokens are required. Likelihood: Medium–High.

## Recommendation
In the `nativeBalance >= value` and `nativeBalance > 0` branches of `pay`, only consume ETH that was explicitly authorized by the current top-level call. One safe approach: record `msg.value` in a transient slot at the entry of each external function and deduct from it inside `pay`, falling through to `safeTransferFrom` for any shortfall. Alternatively, when `payer != address(this)` and the top-level `msg.value` is zero, always pull WETH from the payer via `safeTransferFrom` regardless of the router's ambient ETH balance.

## Proof of Concept
```solidity
function test_strandedEthStolenByAttacker() public {
    uint128 V = 2_500;

    // User A sends 2*V ETH, swaps V, forgets refundETH — V ETH stranded
    vm.prank(userA);
    router.exactInputSingle{value: 2 * V}(
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(pool), tokenIn: address(weth), tokenOut: address(token1),
            zeroForOne: true, amountIn: V, amountOutMinimum: 0,
            recipient: userA, deadline: block.timestamp + 1,
            priceLimitX64: 0, extensionData: ""
        })
    );
    assertEq(address(router).balance, V, "V ETH stranded");

    // Attacker: msg.value=0, zero WETH approval
    address attacker = makeAddr("attacker");
    vm.prank(attacker);
    uint256 out = router.exactInputSingle(
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(pool), tokenIn: address(weth), tokenOut: address(token1),
            zeroForOne: true, amountIn: V, amountOutMinimum: 0,
            recipient: attacker, deadline: block.timestamp + 1,
            priceLimitX64: 0, extensionData: ""
        })
    );

    assertGt(out, 0, "attacker received output");
    assertEq(address(router).balance, 0, "router ETH drained");
}
```
