### Title
Missing Zero Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not validate that the supplied `class_hash` is non-zero before writing it to state. Because `0` is the protocol-wide sentinel value `UNINITIALIZED_CLASS_HASH`, any contract can call `replace_class(0)` to permanently set its own class hash to the uninitialized sentinel, making itself permanently non-executable and freezing all funds held in its storage.

---

### Finding Description

In `execute_replace_class` inside `syscall_impls.cairo`, the OS reads the caller-supplied `class_hash` directly from the syscall request and writes it to `contract_state_changes` without any non-zero guard:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The value `0` is the protocol-wide `UNINITIALIZED_CLASS_HASH` sentinel. The `deploy_contract` function enforces that a contract cannot be deployed to address `0` (ORIGIN_ADDRESS), and it asserts `state_entry.class_hash = UNINITIALIZED_CLASS_HASH` to confirm the slot is empty before deployment: [2](#0-1) 

This means `class_hash = 0` is the canonical marker for "no contract here." Allowing `replace_class(0)` to succeed writes that sentinel back into a live contract's state entry, making it indistinguishable from an undeployed slot. The identical missing check exists in the deprecated path: [3](#0-2) 

The TODO comment in the modern path (`// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.`) explicitly acknowledges that class hash validation is absent.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` is set to `0`, every subsequent call to that contract will fail to find any entry point (the OS will look up class `0`, which has no code). Storage — including ERC-20 balances, vault positions, or any other user funds — remains in the contract's storage trie but is permanently inaccessible. There is no recovery path: `replace_class` cannot be called again because the contract has no executable class to dispatch the syscall from.

---

### Likelihood Explanation

**Medium.**

The attack surface is any deployed contract that:
1. Exposes a `replace_class` call path with a user-controlled or insufficiently validated `class_hash` argument, **or**
2. Contains a logic bug that passes `0` as the class hash.

The OS is the last line of defense. Analogously to how `deploy_contract` explicitly blocks deployment to address `0` (ORIGIN_ADDRESS), `execute_replace_class` must block replacement with class hash `0`. Without this OS-level guard, a single malformed or malicious transaction can permanently destroy a contract's executability.

---

### Recommendation

Add an `assert_not_zero` guard at the top of `execute_replace_class` in both `syscall_impls.cairo` and `deprecated_execute_syscalls.cairo`, immediately after reading the class hash:

```cairo
let class_hash = request.class_hash;
assert_not_zero(class_hash);  // Prevent replacing with UNINITIALIZED_CLASS_HASH (0)
```

This mirrors the existing pattern used in `deploy_contract` and in `execute_declare_transaction` (which already asserts `assert_not_zero(compiled_class_hash)` before writing to `contract_class_changes`): [4](#0-3) 

---

### Proof of Concept

1. Deploy a contract `VaultContract` that holds user ERC-20 balances and exposes a function `set_implementation(class_hash: felt)` that calls the `replace_class` syscall with the provided argument.
2. As an unprivileged user, call `VaultContract.set_implementation(0)`.
3. The OS `execute_replace_class` handler reads `class_hash = 0` and writes `StateEntry(class_hash=0, ...)` to `contract_state_changes` without reverting.
4. The block is proven and the state transition is committed on-chain.
5. All subsequent calls to `VaultContract` (e.g., `withdraw`, `transfer`) fail at entry-point dispatch because class hash `0` has no code.
6. All user balances stored in `VaultContract`'s storage are permanently frozen with no recovery path. [5](#0-4)

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
