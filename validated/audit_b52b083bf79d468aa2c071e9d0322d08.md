### Title
`build_precommit_vote_message_digest` Omits Chain ID, Enabling Cross-Network Precommit Signature Replay — (File: `crates/apollo_signature_manager/src/signature_manager.rs`)

---

### Summary

`build_precommit_vote_message_digest` constructs the ECDSA message digest for consensus precommit votes as `blake2s("PRECOMMIT_VOTE" || block_hash)`. Neither this digest nor the underlying `block_hash` (produced by `calculate_block_hash`) includes the chain ID. A precommit signature produced on one Starknet network is therefore structurally valid on any other Starknet network for any block that shares the same hash value — the direct sequencer analog of the cross-chain signature replay described in the external report.

---

### Finding Description

In `build_precommit_vote_message_digest`:

```rust
fn build_precommit_vote_message_digest(block_hash: BlockHash) -> MessageDigest {
    let block_hash = block_hash.to_bytes_be();
    let mut message = Vec::with_capacity(PRECOMMIT_VOTE.len() + block_hash.len());
    message.extend_from_slice(PRECOMMIT_VOTE);
    message.extend_from_slice(&block_hash);
    MessageDigest(blake2s_to_felt(&message))
}

The `block_hash` input is produced by `calculate_block_hash`, which hashes:

**No chain ID appears anywhere in this chain.** Compare this with every Starknet transaction hash function (e.g., `get_invoke_transaction_v3_hash`, `get_declare_transaction_v3_hash`), which all explicitly chain `Felt::try_from(chain_id)?` into their hash: [3](#0-2) 

The `SignatureManager` component exposes `sign_precommit_vote` and `verify_precommit_vote_signature` as production RPC endpoints: [4](#0-3) 

The state machine currently emits votes with `signature: RawSignature::default()` and a `// TODO(Asmaa): sign the vote` comment, indicating signing integration is in progress and this path will become active: [5](#0-4)

### Impact Explanation

A validator's ECDSA precommit signature for block hash `H` on Starknet Sepolia is byte-for-byte identical to a valid precommit signature for block hash `H` on Starknet Mainnet. `verify_precommit_vote_signature` accepts only `(block_hash, signature, public_key)` — no chain context: [6](#0-5) 

If an attacker collects enough replayed precommit signatures to form a 2/3+ BFT quorum on the target network, the wrong block is finalized and committed to storage, corrupting the canonical state root and block hash chain. This falls under **High — signature/hash logic binds the wrong hash/payload**.

---

### Likelihood Explanation

Low-to-medium. Block hashes encode block-specific data (state root, sequencer address, timestamp, etc.), making collisions unlikely across live production networks. Risk is elevated in:

- Test/staging environments sharing genesis state (Sepolia ↔ IntegrationSepolia)
- Network bootstrapping phases where early blocks share parameters
- Deliberate fork scenarios

The `SignatureManager` is a live production component; once the TODO signing integration is completed, this becomes directly reachable without any privileged access.

---

### Recommendation

Include the chain ID in the precommit vote message digest, mirroring the pattern used by all transaction hash functions:

```rust
fn build_precommit_vote_message_digest(
    block_hash: BlockHash,
    chain_id: &ChainId,
) -> MessageDigest {
    let block_hash = block_hash.to_bytes_be();
    let chain_id_bytes = chain_id.to_string().into_bytes();
    let mut message = Vec::new();
    message.extend_from_slice(PRECOMMIT_VOTE);
    message.extend_from_slice(&chain_id_bytes);
    message.extend_from_slice(&block_hash);
    MessageDigest(blake2s_to_felt(&message))
}
```

Propagate `chain_id` through `sign_precommit_vote`, `verify_precommit_vote_signature`, and the `SignatureManagerRequest::SignPrecommitVote` variant. Optionally, include `chain_id` in `calculate_block_hash` itself for defense-in-depth, consistent with how Starknet transaction hashes are domain-separated. [7](#0-6) 

---

### Proof of Concept

1. Validator `V` signs a precommit on Starknet Sepolia for block hash `H`:
   `sig = ECDSA_sign(sk_V, blake2s("PRECOMMIT_VOTE" || H))`
2. Attacker observes `sig` on the Sepolia P2P network.
3. Attacker injects `sig` as a precommit vote for block hash `H` on Starknet Mainnet (or IntegrationSepolia).
4. `verify_precommit_vote_signature(H, sig, pk_V)` returns `true` on Mainnet — the digest is identical because chain ID is absent from both the message and the block hash.
5. With enough such replayed signatures forming a 2/3+ quorum, the wrong block is finalized on Mainnet, producing a wrong committed state root and corrupting the block hash chain. [8](#0-7) [1](#0-0)
