### Title
Propeller Publisher Signature Does Not Bind Channel — Cross-Channel Shard Replay Delivers Wrong Message to Wrong Consumer - (File: `crates/apollo_propeller/src/signature.rs`)

---

### Summary

The `sign_message_id` function in the Propeller protocol signs only the raw `MessageRoot` (the Merkle root of the erasure-coded shards), without including the `Channel` identifier in the signed payload. This is the direct analog of the MerkleDistributor bug: just as the token address was absent from the Merkle leaf (allowing a claimant to substitute a different token), the channel is absent from the signed commitment (allowing an attacker to substitute a different channel). Any network peer can take a valid set of shards published for Channel A and rebroadcast them with the `channel` field set to Channel B. The signature check passes, the Merkle proof passes, and the wrong message is delivered to Channel B's consumer.

---

### Finding Description

**Root cause — `sign_message_id` omits the channel:** [1](#0-0) 

The signed payload is `<propeller>{message_root}</propeller>`. The `Channel` value is never included.

**Publishing — one signature covers all channels:** [2](#0-1) 

`create_units_to_publish` computes `signature = sign_message_id(&message_root, &keypair)` and attaches the same signature to every `PropellerUnit`, regardless of which channel the units are destined for. The `channel` field is stored in the unit struct but is never fed into the signing operation.

**Verification — only the root is checked cryptographically:** [3](#0-2) 

`verify_signature` calls `signature::verify_message_id_signature(&unit.root(), unit.signature(), &self.publisher_public_key)`. The channel is not part of the verified message.

**The channel "check" is a no-op assertion, not a security gate:** [4](#0-3) 

`assert_eq!(self.channel, unit.channel())` compares the validator's own channel (which was initialised from the unit's claimed channel in the engine) against the unit's channel — they are always equal by construction. This provides zero security against cross-channel replay.

**Engine routing — channel is taken from the attacker-controlled unit field:** [5](#0-4) 

`handle_unit` reads `claimed_channel = unit.channel()` and uses it to look up the registered `ChannelData` and to key the `MessageKey`. A unit with a forged channel field is routed to the target channel's `MessageProcessor` without any cryptographic check that the publisher actually signed for that channel.

**Emitted event omits the channel:** [6](#0-5) 

`Event::MessageReceived` carries `publisher`, `message_root`, and `message` — but not the channel. The consumer of Channel B therefore cannot distinguish a legitimately published Channel-B message from a replayed Channel-A message.

---

### Impact Explanation

The Propeller protocol is the data-dissemination layer for the consensus subsystem. Channels multiplex different logical streams (e.g., different consensus rounds, different proposal types). Because the publisher's signature does not bind the channel, an attacker who can observe network traffic can replay any set of shards from Channel A to Channel B. The receiving node's `UnitValidator` will accept the shards (Merkle proof valid, signature valid), reconstruct the original message, and emit `Event::MessageReceived` to Channel B's consumer. The consumer processes a message that was never intended for that channel — wrong block proposal delivered to the wrong consensus round, or wrong payload type delivered to the wrong handler — matching the **High** impact: *"Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."*

---

### Likelihood Explanation

The attack requires only:
1. The ability to observe network traffic (any peer on the libp2p overlay).
2. The target publisher to be registered on both the source and target channels — a normal condition when the same validator set participates in consecutive consensus rounds.
3. Modifying the `channel` field of intercepted `PropellerUnit` protobuf messages before forwarding them.

No privileged access, no key material, and no cryptographic break are required.

---

### Recommendation

Include the channel in the signed payload so that a signature for Channel A cannot be accepted for Channel B:

```rust
// In sign_message_id / verify_message_id_signature, add channel bytes:
let msg = [SIGNING_PREFIX, &channel.0.to_le_bytes(), &message_id.0, SIGNING_POSTFIX].concat();
```

Both `sign_message_id` and `verify_message_id_signature` must be updated to accept and incorporate the `Channel` parameter, and all call sites in `create_units_to_publish` and `UnitValidator::verify_signature` must pass the channel through.

### Proof of Concept

```
Setup:
  Publisher P is registered on Channel(1) (consensus round N)
                          and Channel(2) (consensus round N+1).

Step 1 — Legitimate publish on Channel 1:
  P calls create_units_to_publish(message_B1, Channel(1), keypair, ...)
  → message_root = R1
  → signature    = sign(<propeller>R1</propeller>)   // channel NOT included
  → units[i].channel = Channel(1), units[i].root = R1, units[i].signature = sig

Step 2 — Attacker intercepts shards for Channel 1.

Step 3 — Attacker rewrites channel field:
  for each unit: unit.channel = Channel(2)   // shard, root, signature unchanged

Step 4 — Attacker forwards modified units to peers.

Step 5 — Peer engine receives unit with channel=2:
  handle_unit: claimed_channel = Channel(2)  ← attacker-controlled
  channels.get(&Channel(2)) → Some(channel_data)  ← registered, passes
  MessageKey { channel: Channel(2), publisher: P, root: R1 }
  MessageProcessor spawned for Channel(2).

Step 6 — UnitValidator for Channel(2):
  assert_eq!(Channel(2), unit.channel())  → Channel(2)==Channel(2) ✓ (no-op)

  Event::MessageReceived { publisher: P, message_root: R1, message: B1 }
  to Channel(2)'s consumer.

        is delivered to consensus round N+1 / Channel 2's handler,
        which processes it as a valid Channel-2 message.

    fn verify_signature(
        &mut self,
        unit: &PropellerUnit,
    ) -> Result<(), ShardSignatureVerificationError> {
        if let Some(signature) = &self.verified_signature {
            return if signature == unit.signature() {
                Ok(())
            } else {
                Err(ShardSignatureVerificationError::VerificationFailed)
            };
        }

        let result = signature::verify_message_id_signature(
            &unit.root(),
            unit.signature(),
            &self.publisher_public_key,
        );

        if let Ok(()) = &result {
            self.verified_signature = Some(unit.signature().to_vec());
        }

        result
    }

    fn handle_unit(&mut self, sender_peer_id: PeerId, unit: PropellerUnit) {
        let claimed_channel = unit.channel();
        let claimed_publisher = unit.publisher();
        let claimed_root = unit.root();

        // Track received shard.
        if let Some(metrics) = &self.metrics {
            metrics.shards_received.increment(1);
        }

        // Check if channel is registered.
        let Some(channel_data) = self.channels.get(&claimed_channel) else {
            warn!(?claimed_channel, "Received shard for unregistered channel, dropping");
            return;
        };

        // Skip if message already finalized.
        let message_key = MessageKey {
            channel: claimed_channel,
            publisher: claimed_publisher,
            root: claimed_root,
        };

        // TODO(guyn): Add timestamps to message key to avoid replay attacks or issues with very
        // late shards.
        if self.finalized_messages.contains(&message_key) {
            trace!("Message already finalized, dropping unit");
            return;
        }

        // Spawn tasks if this is a new message.
        if !self.message_to_unit_tx.contains_key(&message_key) {
            debug!(
                ?claimed_channel,
                ?claimed_publisher,
                ?claimed_root,
                "[ENGINE] Spawning new message processor"
            );

            let schedule_manager = channel_data.schedule_manager.clone();
            let Some(publisher_public_key) =
                channel_data.peer_public_keys.get(&claimed_publisher).cloned()
            else {
                warn!(?claimed_publisher, "Received shard for unregistered publisher, dropping");
                return;
            };
            let my_shard_index_result =
                schedule_manager.get_my_shard_index_given_publisher(&claimed_publisher);
            let Ok(my_shard_index) = my_shard_index_result else {
                warn!(
                    ?claimed_publisher,
                    ?claimed_channel,
                    ?my_shard_index_result,
                    "Received shard for publisher not in channel, dropping"
                );
                return;
            };

            // Create channel for Engine -> MessageProcessor communication
            let (unit_tx, unit_rx) = mpsc::unbounded_channel();

            // Create and spawn message processor
            let processor = MessageProcessor {
                channel: claimed_channel,
                publisher: claimed_publisher,
                message_root: claimed_root,
                my_shard_index,
                publisher_public_key,
                tree_manager: Arc::clone(&schedule_manager),
                local_peer_id: self.local_peer_id,
                unit_rx,
                engine_tx: self.state_manager_tx.clone(),
                timeout: self.config.stale_message_timeout,
            };

            // TODO(AndrewL): track task handle to see if it panics or is killed.
            // TODO(AndrewL): abort the task if channel is removed.
            tokio::spawn(processor.run());

            self.message_to_unit_tx.insert(message_key, unit_tx);
        }

        // Send unit to message processor
        let unit_tx =
            self.message_to_unit_tx.get(&message_key).expect("Message processor must exist");

        // This may fail if the message is already finalized
        let _ = unit_tx.send((sender_peer_id, unit));
