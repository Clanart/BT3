### Title
Missing `chunk.shard_id() == shard_id` Guard in `set_state_header` Allows Cross-Shard State Header Injection - (`chain/chain/src/state_sync/adapter.rs`)

### Summary

`set_state_header` verifies that the supplied chunk is *somewhere* in the block's chunk-header merkle tree, but never asserts that it is at the position that corresponds to the requested `shard_id`. Because `verify_path` is a pure membership proof (no index check), an unprivileged peer can respond to a shard-B state-header request with shard-A's chunk and a valid shard-A merkle proof, pass every guard in the function, and cause the node to persist `StateHeaderKey(shard_id=B, sync_hash)` → header containing shard-A's `prev_state_root`. Every subsequent `set_state_part` call for shard B then validates parts against shard A's state root, and `set_state_finalize` applies shard A's chunk body under shard B's `ShardUId`.

### Finding Description

**Entrypoint.** During state sync a node calls `set_state_header(shard_id, sync_hash, peer_response)` with data received from an arbitrary peer.

**Guard 1 – `validate_chunk_proofs`** (`chain/chain/src/validate.rs` lines 22-66): checks internal self-consistency of the chunk (hash, tx-root, receipts-root). It never compares `chunk.shard_id()` to the `shard_id` parameter. [1](#0-0) 

**Guard 2 – `verify_path`** (`core/primitives/src/merkle.rs` lines 113-118): computes `compute_root_from_path(path, hash(leaf)) == root`. This is a pure membership check; it does not verify the leaf's *index* in the tree. [2](#0-1) 

A position-aware variant `verify_path_with_index` exists (lines 121-128) but is not used here. [3](#0-2) 

**The missing check.** In a two-shard block the merkle tree is:

```
root = combine_hash(hash(ChunkHashHeight(chunk0)), hash(ChunkHashHeight(chunk1)))
proof_for_chunk0 = [{hash: hash(ChunkHashHeight(chunk1)), direction: Right}]
```

`verify_path(root, proof_for_chunk0, ChunkHashHeight(chunk0))` reconstructs `combine_hash(hash(chunk0), hash(chunk1)) == root` — **true** — even though chunk0 is at index 0 (shard 0), not index 1 (shard 1). The call site never checks `chunk.shard_id() == shard_id`. [4](#0-3) 

**Storage.** The header is then persisted under `StateHeaderKey(shard_id=B, sync_hash)` carrying shard A's `prev_state_root`, `prev_outgoing_receipts_root`, and chunk body. [5](#0-4) 

**Propagation to `set_state_part`.** `set_state_part` reads the stored header and extracts `prev_state_root` from it to validate every incoming state part: [6](#0-5) 

With shard A's root installed under shard B's key, the attacker supplies state parts for shard A's trie; they pass `validate_state_part(shard_id=B, state_root=A_root, ...)` and are committed to `DBCol::StateParts` under shard B's `StatePartKey`.

**Propagation to `set_state_finalize`.** `set_state_finalize` reads the stored header's chunk (shard A's chunk), extracts `prev_state_root` and `gas_limit` from it, and calls `apply_chunk` with `shard_uid` derived from the *parameter* `shard_id=B` but with shard A's state root and transactions: [7](#0-6) 

### Impact Explanation

The node installs shard A's trie as shard B's post-sync state. All subsequent chunk applications for shard B start from the wrong state root, producing wrong outcomes, wrong outgoing receipts, and wrong `ChunkExtra`. The node diverges from the canonical chain for shard B and cannot re-join without a full re-sync.

### Likelihood Explanation

Any peer reachable during state sync can mount this attack. All required data (chunk body, merkle proof, receipt proofs, state-root node) are publicly observable on-chain. No validator key or privileged role is required. The attack is deterministic and requires no brute-force.

### Recommendation

Add an explicit shard-id guard immediately after `validate_chunk_proofs`:

```rust
if chunk.shard_id() != shard_id {
    return Err(Error::Other(
        "set_shard_state failed: chunk shard_id does not match requested shard_id".into(),
    ));
}
```

Alternatively, replace `verify_path` with `verify_path_with_index`, passing the shard index derived from `shard_id` via `epoch_manager.get_shard_layout(...).get_shard_index(shard_id)`, so the proof is bound to the correct leaf position.

### Proof of Concept

1. Observe a finalized block B_prev with two shards. Extract:
   - `chunk0`: the full `ShardChunk` for shard 0 (available from any archive node).
   - `proof0`: the merkle path for `ChunkHashHeight(chunk0.chunk_hash(), chunk0.height_included())` at index 0 in B_prev's chunk-header tree (recompute via `merklize` over all `ChunkHashHeight` values).
   - `receipts_proofs_for_shard1`: valid `incoming_receipts_proofs` and `root_proofs` for shard 1 (available from block data).
   - `state_root_node0`: shard 0's `StateRootNode` (available from any state-sync provider).

2. Construct a `ShardStateSyncResponseHeaderV2` with `chunk = chunk0`, `chunk_proof = proof0`, and the shard-1 receipt proofs.

3. Call `set_state_header(shard_id = 1, sync_hash, malicious_header)`.

4. Assert: `chain_store.get_state_header(1, sync_hash).chunk().shard_id() == 0` (shard 0's chunk stored under shard 1's key).

5. Assert: `chain_store.get_state_header(1, sync_hash).chunk().take_header().take_inner().prev_state_root() == shard_0_state_root` (shard 0's state root used for all shard-1 part validation).

pub fn validate_chunk_proofs(
    chunk: &ShardChunk,
    epoch_manager: &dyn EpochManagerAdapter,
) -> Result<bool, Error> {
    let correct_chunk_hash = chunk.compute_header_hash();

    // 1. Checking chunk.header.hash
    let header_hash = chunk.header_hash();
    if header_hash != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }

    // 2. Checking that chunk body is valid
    // 2a. Checking chunk hash
    if chunk.chunk_hash() != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }
    let height_created = chunk.height_created();
    let outgoing_receipts_root = chunk.prev_outgoing_receipts_root();
    let (transactions, receipts) = (chunk.to_transactions(), chunk.prev_outgoing_receipts());

    // 2b. Checking that chunk transactions are valid
    let (tx_root, _) = merklize(transactions);
    if &tx_root != chunk.tx_root() {
        byzantine_assert!(false);
        return Ok(false);
    }
    // 2c. Checking that chunk receipts are valid
    if height_created == 0 {
        return Ok(receipts.is_empty() && outgoing_receipts_root == &CryptoHash::default());
    } else {
        let shard_layout = {
            let prev_block_hash = chunk.prev_block_hash();
            epoch_manager.get_shard_layout_from_prev_block(&prev_block_hash)?
        };
        let outgoing_receipts_hashes = Chain::build_receipts_hashes(receipts, &shard_layout)?;
        let (receipts_root, _) = merklize(&outgoing_receipts_hashes);
        if &receipts_root != outgoing_receipts_root {
            byzantine_assert!(false);
            return Ok(false);
        }
    }
    Ok(true)
