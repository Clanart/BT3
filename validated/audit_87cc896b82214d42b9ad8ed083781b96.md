### Title
Missing Class Hash Validation in `execute_replace_class` Enables Permanent Fund Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall in `syscall_impls.cairo` accepts any arbitrary class hash from the caller without verifying that the hash corresponds to a declared contract class. This is directly analogous to the external report's pattern of partial validation: one dimension (gas) is checked, but the critical semantic constraint (class existence) is not. Any deployed contract can call `replace_class` with an undeclared hash, permanently breaking itself and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class`, the OS reads `class_hash` directly from the syscall request and writes it into `contract_state_changes` without any existence check:

```cairo
let class_hash = request.class_hash;
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
``` [1](#0-0) 

The OS has full access to `compiled_class_facts_bundle` (via `block_context.os_global_context`) which contains every declared class, but this bundle is never consulted. The state is unconditionally updated:

```cairo
dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [2](#0-1) 

When any subsequent call targets the now-broken contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
``` [3](#0-2) 

For an undeclared hash, `dict_read` returns `0` (the Cairo dict default). Then `find_element` is called with `key=0`:

```cairo
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [4](#0-3) 

`find_element` is a non-optional search that panics if the key is absent. If no compiled class with hash `0` exists, the OS cannot prove any block containing a call to this contract. The contract becomes permanently uncallable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any ERC-20/ETH/STRK balance held by the contract becomes irrecoverable:
- The contract's on-chain class hash is set to an undeclared value.
- Every future call to the contract fails at the OS level before any entry point executes.
- There is no upgrade or recovery path because `replace_class` itself requires a successful call.

---

### Likelihood Explanation

The attack is reachable by any unprivileged contract deployer with no special privileges:

1. Deploy a contract that accepts user deposits (e.g., a fake vault or DEX).
2. Attract users to deposit funds.
3. Call `replace_class(undeclared_hash)` — e.g., `class_hash = 1` — from within the contract.
4. The OS writes the invalid class hash to state without complaint.
5. All deposited funds are permanently frozen.

The TODO comment in the production code explicitly acknowledges the missing check, confirming it is a real, unmitigated gap. [5](#0-4) 

---

### Recommendation

Before writing the new state entry in `execute_replace_class`, verify that `class_hash` is present in `compiled_class_facts_bundle`. Concretely:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

Alternatively, perform a `find_element` lookup against `compiled_class_facts_bundle.compiled_class_facts` and revert the syscall (write `failure_flag=1`) if the class is not found, consistent with how other invalid-argument cases are handled (e.g., `ERROR_INVALID_ARGUMENT` in `execute_call_contract`). [6](#0-5) 

---

### Proof of Concept

```
1. Attacker deploys `MaliciousVault` — a contract with a `deposit()` entry point that
   accepts STRK and a `rug()` entry point that calls replace_class(0x1).

2. Users call deposit() and transfer funds to MaliciousVault.
   State: MaliciousVault.balance = N STRK, class_hash = <valid_declared_hash>

3. Attacker calls rug().
   OS executes execute_replace_class:
     - class_hash = 0x1 (not in compiled_class_facts_bundle)
     - No existence check performed (TODO comment)
     - dict_update writes class_hash=0x1 to contract_state_changes

4. State committed: MaliciousVault.class_hash = 0x1

5. Any user calls withdraw() on MaliciousVault:
   - execute_entry_point: dict_read(key=0x1) → compiled_class_hash = 0
   - find_element(key=0) → not found → OS panics / block proof fails
   - Sequencer drops the transaction; it can never be included.

6. N STRK permanently frozen. No recovery possible.
```
