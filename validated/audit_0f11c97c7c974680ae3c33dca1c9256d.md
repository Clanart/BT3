### Title
Non-EVM-Denom Bank Balances Not Burned on SELFDESTRUCT Enables Orphaned Fund Theft via CREATE2 Address Reuse - (File: x/evm/statedb/statedb.go)

### Summary
When a contract self-destructs, `StateDB.Commit()` only burns the EVM-native denomination balance. Non-EVM-denom tokens (IBC, CosmWasm bridge tokens, etc.) held by the destroyed contract address are silently left as orphaned bank balances. After `DeleteAccount` removes the auth account, those tokens are stranded. An attacker who controls the CREATE2 salt can redeploy a new contract at the same address and drain the orphaned Cosmos bank funds via a bank precompile or native action.

### Finding Description

In `StateDB.Commit()`, the self-destruct path explicitly acknowledges the gap:

```go

// bridge) held by the destroyed address are not drained and may remain as
// orphaned bank balances.

        return errorsmod.Wrap(err, "failed to burn post-selfdestruct balance")

The `Keeper.SubBalance` used for the burn is denom-specific — it only operates on `s.evmDenom`: [3](#0-2) 

After `DeleteAccount`, the Cosmos bank module still holds any non-EVM-denom coins at the destroyed address. The auth account is gone, but the bank balance record persists. When a new contract is deployed at the same address (deterministically via CREATE2), a fresh auth account is created at that address and the bank module's pre-existing balance for that address becomes accessible to the new contract owner.

### Impact Explanation

**Critical — Unauthorized theft of Cosmos bank funds through Ethermint stateDB/native action logic.**

Any non-EVM-denom tokens (IBC vouchers, CosmWasm bridge tokens, or any `sdk.Coin` with a denom other than the EVM denom) held by a self-destructed contract address are not burned. They remain in the bank module. An attacker who controls the CREATE2 factory and salt can redeploy a contract at the same address and, via a bank precompile or `ExecuteNativeAction`, transfer those orphaned coins to themselves. The original token holders lose their funds permanently.

### Likelihood Explanation

The attack is reachable by any unprivileged user who can:
1. Predict or control a CREATE2 address (standard Ethereum tooling).
2. Pre-fund that address with non-EVM-denom tokens via a Cosmos bank send (no EVM required).
3. Deploy and immediately SELFDESTRUCT the contract in one transaction (EIP-6780 permits this when creation and destruction occur in the same tx).
4. Redeploy at the same address and drain the orphaned balance via a bank precompile.

The `SelfDestructExploitFactory` test contract already demonstrates steps 1–3 for the EVM-denom case; extending it to non-EVM denoms requires only that the chain exposes a bank precompile or native-action hook. [4](#0-3) 

### Recommendation

In `StateDB.Commit()`, before calling `DeleteAccount`, enumerate **all** bank denominations held by the destroyed address and burn or redirect each one. The `BankKeeper` interface already exposes `GetAllBalances` (or equivalent) on the underlying Cosmos SDK bank keeper. The burn loop should be:

```go
allCoins := bankKeeper.SpendableCoins(cacheCtx, cosmosAddr)
if !allCoins.IsZero() {
    if err := bankKeeper.SendCoinsFromAccountToModule(cacheCtx, cosmosAddr, types.ModuleName, allCoins); err != nil {
        return errorsmod.Wrap(err, "failed to sweep non-evm balances pre-selfdestruct")
    }
    if err := bankKeeper.BurnCoins(cacheCtx, types.ModuleName, allCoins); err != nil {
        return errorsmod.Wrap(err, "failed to burn non-evm balances pre-selfdestruct")
    }
}
```

Alternatively, if burning IBC vouchers is undesirable (it would reduce IBC channel supply), redirect them to a governance-controlled address or the fee collector, and document the policy explicitly — mirroring the recommendation in the original audit report.

// DeleteAccount handles contract's suicide call:
// - remove code
// - remove states
// - remove auth account

// NOTE: balance should be cleared separately
func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
	cosmosAddr := sdk.AccAddress(addr.Bytes())
	acct := k.accountKeeper.GetAccount(ctx, cosmosAddr)
	if acct == nil {
		return nil
	}

	// NOTE: only Ethereum accounts (contracts) can be selfdestructed
	_, ok := acct.(ethermint.EthAccountI)
	if !ok {
		return errorsmod.Wrapf(types.ErrInvalidAccount, "type %T, address %s", acct, addr)
	}

	// clear storage
	k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool {
		k.SetState(ctx, addr, key, nil)
		return true
	})

	// remove auth account
	k.accountKeeper.RemoveAccount(ctx, acct)

	k.debugLog(ctx, "account suicided",
		"ethereum-address", addr,
		"cosmos-address", cosmosAddr,
	)

	return nil
