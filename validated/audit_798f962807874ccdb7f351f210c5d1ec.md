### Title
EIP-7702 Authorization with Zero Chain ID Accepted Cross-Chain, Enabling Unauthorized EOA Code Mutation on Any Ethermint Fork - (File: x/evm/keeper/set_code_authorizations.go)

---

### Summary

`validateAuthorization` in `x/evm/keeper/set_code_authorizations.go` explicitly permits EIP-7702 `SetCodeAuthorization` entries whose `ChainID` field is zero (`0`). A zero chain ID is treated as a wildcard: the authorization is valid on **every** Ethermint chain simultaneously. An attacker who observes a valid EIP-7702 SetCode transaction on one Ethermint chain can replay the exact same signed authorization tuple on any other Ethermint chain (fork, testnet, or independent deployment sharing the same address space), causing the victim's EOA to have its code pointer mutated to an attacker-chosen delegate contract without the victim's consent on that chain.

---

### Finding Description

The root cause is a single conditional in `validateAuthorization`:

```go
// x/evm/keeper/set_code_authorizations.go, line 17
if !auth.ChainID.IsZero() && auth.ChainID.CmpBig(k.eip155ChainID) != 0 {
    return authority, core.ErrAuthorizationWrongChainID
}
```

The guard only rejects a mismatched chain ID when `auth.ChainID` is **non-zero**. When `auth.ChainID == 0`, the check is skipped entirely and the authorization is accepted on any chain. This is the same design as go-ethereum's EIP-7702 reference implementation (which also allows chain-ID 0 as a cross-chain wildcard per the EIP-7702 spec), but it creates a concrete replay surface in Ethermint's multi-chain deployment model.

The analog to the external report is direct: just as the Merkle-drop contracts lacked chain-ID binding in their leaf encoding (allowing anyone to replay a `withdraw()` call on a fork), EIP-7702 authorizations signed with `chainId = 0` carry no chain binding and can be replayed verbatim on any Ethermint chain.

The full execution path is:

1. Victim signs a `SetCodeAuthorization{ChainID: 0, Address: <delegate>, Nonce: N}` on chain A (e.g., to use a smart-wallet delegate).
2. Attacker observes the transaction on chain A.
3. Attacker submits a new `SetCodeTx` on chain B (a fork or sibling Ethermint chain) embedding the same authorization tuple.
4. `validateAuthorization` on chain B passes the chain-ID check because `auth.ChainID.IsZero()` is `true`.
5. The nonce check (`stateDB.GetNonce(authority) == auth.Nonce`) passes if the victim's nonce on chain B happens to match (likely on a fresh fork or if the victim has the same nonce history).
6. `setAuthorizationDelegation` writes `types.AddressToDelegation(auth.Address)` as the victim's code on chain B, pointing their EOA at the attacker-chosen delegate.
7. The durable commit path (`applyDurableAuthorization` → `durableStateDB.Commit()`) persists this mutation even if the outer EVM call fails. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

---

### Impact Explanation

Once the victim's EOA code is mutated to point at an attacker-controlled delegate contract on chain B:

- Every call to the victim's address on chain B executes the delegate's code in the victim's context (storage, balance, `msg.sender` = victim).
- The delegate can drain the victim's entire EVM-denom balance via a single `CALL` or `SELFDESTRUCT`.
- The delegate can perform arbitrary storage writes under the victim's address, corrupting any state the victim's address is relied upon to hold.
- The victim's nonce is also bumped by the authorization processing (`SetNonce(authority, auth.Nonce+1)`), which can break pending legitimate transactions.

This matches the **High** allowed impact: "EIP-155/EIP-712/EIP-7702 authorization, nonce, chain-id, or signer verification bypass enabling replay, forged execution, or unauthorized account/code mutation." [5](#0-4) 

---

### Likelihood Explanation

- **Realistic trigger**: Any user who signs a chain-ID-0 authorization (which wallets may do for portability, or which an attacker can socially engineer) is immediately vulnerable on all Ethermint chains.
- **No privileged access required**: The attacker only needs to submit a standard `SetCodeTx` (type 4) via the public JSON-RPC (`eth_sendRawTransaction`). The outer transaction signer can be any funded account; only the inner authorization needs to match the victim.
- **Nonce condition**: The nonce check is the only remaining barrier. On a fresh fork or testnet where the victim has the same nonce history as on the original chain, this is trivially satisfied. On a live chain, the attacker can wait for the victim's nonce to reach the value embedded in the captured authorization.
- **Durable commit amplifies impact**: Even if the outer EVM call reverts, the authorization effects are committed via the durable stateDB path, so the attacker does not need the call to succeed. [6](#0-5) 

---

### Recommendation

1. **Reject chain-ID-0 authorizations at the Ethermint layer**: In `validateAuthorization`, add an explicit check that rejects zero chain IDs:

```go
if auth.ChainID.IsZero() {
    return authority, core.ErrAuthorizationWrongChainID
}
if auth.ChainID.CmpBig(k.eip155ChainID) != 0 {
    return authority, core.ErrAuthorizationWrongChainID
}
```

2. **Reject at `ValidateBasic`**: Add a corresponding check in `SetCodeTx.Validate()` and in the `SetCodeAuthorization` proto validation to reject zero-chain-ID authorization entries before they reach the keeper.

3. **Document the policy**: If chain-ID-0 authorizations are intentionally supported for cross-chain portability, document the replay risk explicitly and require users to opt in. [7](#0-6) [8](#0-7)

### Proof of Concept

```
Chain A (Ethermint, chainID=9000):
  Victim signs: SetCodeAuthorization{ChainID: 0, Address: <safe_delegate>, Nonce: 5}
  Victim submits SetCodeTx → delegation applied on chain A ✓

Chain B (Ethermint fork, chainID=9001, victim nonce=5):
  Attacker constructs SetCodeTx{
    ChainID: 9001,
    Nonce: <attacker_nonce>,
    To: <any address>,
    AuthList: [SetCodeAuthorization{ChainID: 0, Address: <drain_contract>, Nonce: 5}]
  }
  Attacker signs outer tx with their own key and submits via eth_sendRawTransaction.

  validateAuthorization:
    auth.ChainID.IsZero() == true  →  chain-ID check SKIPPED
    auth.Nonce (5) == stateDB.GetNonce(victim) (5)  →  PASS
    authority recovered from (V,R,S) == victim address  →  PASS

    stateDB.SetNonce(victim, 6)
    stateDB.SetCode(victim, AddressToDelegation(<drain_contract>))

  durableStateDB.Commit() → mutation persisted even if EVM call reverts

  Result: victim's EOA on chain B now executes <drain_contract> code.
  Next call to victim's address drains their balance.

func (k *Keeper) validateAuthorization(auth *types.SetCodeAuthorization, stateDB vm.StateDB) (authority common.Address, err error) {
	// Verify chain ID is null or equal to current chain ID.
	if !auth.ChainID.IsZero() && auth.ChainID.CmpBig(k.eip155ChainID) != 0 {
		return authority, core.ErrAuthorizationWrongChainID
	}
	// Limit nonce to 2^64-1 per EIP-2681.
	if auth.Nonce+1 < auth.Nonce {
		return authority, core.ErrAuthorizationNonceOverflow
	}
	// Validate signature values and recover authority.
	authority, err = auth.Authority()
	if err != nil {
		return authority, fmt.Errorf("%w: %v", core.ErrAuthorizationInvalidSignature, err)
	}
	// Check the authority account
	//  1) doesn't have code or has exisiting delegation
	//  2) matches the auth's nonce
	//
	// Note it is added to the access list even if the authorization is invalid.
	stateDB.AddAddressToAccessList(authority)
	code := stateDB.GetCode(authority)
	if _, ok := types.ParseDelegation(code); len(code) != 0 && !ok {
		return authority, core.ErrAuthorizationDestinationHasCode
	}
	if have := stateDB.GetNonce(authority); have != auth.Nonce {
		return authority, core.ErrAuthorizationNonceMismatch
	}
	return authority, nil
}

func (k *Keeper) setAuthorizationDelegation(auth *types.SetCodeAuthorization, authority common.Address, stateDB vm.StateDB) {
	// Update nonce and account code.
	stateDB.SetNonce(authority, auth.Nonce+1, tracing.NonceChangeAuthorization)
	if auth.Address == (common.Address{}) {
		// Delegation to zero address means clear.
		stateDB.SetCode(authority, nil, tracing.CodeChangeAuthorizationClear)
		return
	}

	// Otherwise install delegation to auth.Address.
	stateDB.SetCode(authority, types.AddressToDelegation(auth.Address), tracing.CodeChangeAuthorization)
}

func (tx SetCodeTx) Validate() error {
	if len(tx.To) == 0 {
		return errorsmod.Wrap(core.ErrSetCodeTxCreate, "to address cannot be empty")
	}

	if len(tx.AuthList) == 0 {
		return errorsmod.Wrap(core.ErrEmptyAuthList, "auth list cannot be empty")
	}

	// V is the signature y-parity byte; an empty slice would panic at auth.V[0]
	// in ToEthAuthList.
	for i := range tx.AuthList {
		if len(tx.AuthList[i].V) != 1 {
			return errorsmod.Wrapf(core.ErrAuthorizationInvalidSignature, "auth %d: V must be a single byte", i)
		}
	}

	if tx.GasTipCap == nil {
		return errorsmod.Wrap(ErrInvalidGasCap, "gas tip cap cannot nil")
	}

	if tx.GasFeeCap == nil {
		return errorsmod.Wrap(ErrInvalidGasCap, "gas fee cap cannot nil")
	}

	if tx.GasTipCap.IsNegative() {
		return errorsmod.Wrapf(ErrInvalidGasCap, "gas tip cap cannot be negative %s", tx.GasTipCap)
	}

	if tx.GasFeeCap.IsNegative() {
		return errorsmod.Wrapf(ErrInvalidGasCap, "gas fee cap cannot be negative %s", tx.GasFeeCap)
	}

	if !types.IsValidInt256(tx.GetGasTipCap()) {
		return errorsmod.Wrap(ErrInvalidGasCap, "out of bound")
	}

	if !types.IsValidInt256(tx.GetGasFeeCap()) {
		return errorsmod.Wrap(ErrInvalidGasCap, "out of bound")
	}

	if tx.GasFeeCap.LT(*tx.GasTipCap) {
		return errorsmod.Wrapf(
			ErrInvalidGasCap, "max priority fee per gas higher than max fee per gas (%s > %s)",
			tx.GasTipCap, tx.GasFeeCap,
		)
	}

	if !types.IsValidInt256(tx.Fee()) {
		return errorsmod.Wrap(ErrInvalidGasFee, "out of bound")
	}

	amount := tx.GetValue()
	// Amount can be 0
	if amount != nil && amount.Sign() == -1 {
		return errorsmod.Wrapf(ErrInvalidAmount, "amount cannot be negative %s", amount)
	}
	if !types.IsValidInt256(amount) {
		return errorsmod.Wrap(ErrInvalidAmount, "out of bound")
	}

	if tx.To != "" {
		if err := types.ValidateAddress(tx.To); err != nil {
			return errorsmod.Wrap(err, "invalid to address")
		}
	}

	if tx.GetChainID() == nil {
		return errorsmod.Wrap(
			errortypes.ErrInvalidChainID,
			"chain ID must be present on SetCode txs",
		)
	}

	return nil
}
