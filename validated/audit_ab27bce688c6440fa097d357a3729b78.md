### Title
Unvalidated Class Hash in `execute_replace_class` Allows Permanent Fund Freezing and OS Proof Failure - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary class hash from a contract without verifying that the hash corresponds to a previously declared class. This is structurally analogous to the ReferralFeeReceiver bug where user-provided contract addresses were never validated against the factory. A contract can replace its own class with an undeclared hash, corrupting the on-chain state entry. Any subsequent call to that contract causes the OS Cairo program to fail during proof generation, permanently freezing all funds held by the contract and potentially halting the network.

### Finding Description

In `execute_replace_class`, the OS reads `request.class_hash` directly from the syscall request and writes it into `contract_state_changes` without any check that the hash exists in `contract_class_changes` (the declared-class registry):

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-inserted TODO comment at line 898 explicitly acknowledges this missing validation. [1](#0-0) 

When the affected contract is subsequently called, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
```

Because the arbitrary hash was never declared, `dict_read` returns the default value `0`. The OS then calls:

```cairo
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
```

`find_element` with `key=0` panics (Cairo assertion failure) because no compiled class has hash `0` — `execute_declare_transaction` enforces `assert_not_zero(compiled_class_hash)` for all declarations. This panic aborts OS proof generation for the entire block. [2](#0-1) [3](#0-2) 

### Impact Explanation

**Critical — Permanent freezing of funds.** Any contract holding user assets (a vault, a DEX pool, a multi-sig) that calls `replace_class` with an undeclared hash — whether intentionally by a malicious deployer or via a reentrancy/callback path — has its class hash permanently corrupted in state. All future calls to that contract fail at the OS level, making the funds irrecoverable.

**High — Network halt.** If the sequencer's blockifier does not independently enforce this validation (consistent with the OS having the same TODO), the transaction is included in a block. The OS then fails to generate a valid proof for that block. Because the block is already committed, the network cannot advance, constituting a total network shutdown.

### Likelihood Explanation

The attack requires only an unprivileged transaction sender who can deploy a contract and invoke `replace_class`. No privileged access, leaked keys, or external dependencies are needed. The explicit TODO comment confirms the validation is absent in the current production code. Any contract that exposes an upgrade path callable by users (a common DeFi pattern) is a direct attack surface.

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that it exists in `contract_class_changes` with a non-zero compiled class hash:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the fix recommended for ReferralFeeReceiver: enforce that user-provided identifiers were registered through the authoritative registry before trusting them.

### Proof of Concept

1. Attacker deploys `VaultContract` which accepts user deposits and exposes an `upgrade(new_hash)` function that calls `replace_class(new_hash)`.
2. Users deposit funds into `VaultContract`.
3. Attacker calls `upgrade(0xdeadbeef)` where `0xdeadbeef` is never declared.
4. OS writes `contract_state_changes[vault_address].class_hash = 0xdeadbeef`.
5. Any subsequent call to `VaultContract` (withdraw, transfer, etc.) reaches `execute_entry_point`:
   - `dict_read(contract_class_changes, 0xdeadbeef)` → returns `0`
   - `find_element(compiled_class_facts, key=0)` → Cairo panic
6. OS proof generation for the block aborts. All user funds in `VaultContract` are permanently frozen. [4](#0-3) [5](#0-4)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
