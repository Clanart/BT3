### Title
Missing Class Hash Existence Validation in `execute_replace_class` Enables Permanent Contract Fund Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not validate that the supplied class hash corresponds to a previously declared contract class. This is an explicitly acknowledged gap (marked with a `TODO`). An unprivileged transaction sender who can reach a contract's upgrade path — or any contract that exposes `replace_class` without strict access control — can supply an undeclared class hash. Once committed to state, the contract becomes permanently uncallable, freezing all funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function updates a contract's `class_hash` in `contract_state_changes` without verifying that the new hash exists in the declared class registry:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The `class_hash` field is taken directly from the request with no existence check. The `TODO` comment at line 898 explicitly acknowledges this missing validation.

When a subsequent transaction calls the now-bricked contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // undeclared hash → returns 0
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,           // key = 0, not present → panic
);
``` [2](#0-1) 

`dict_read` on `contract_class_changes` for an undeclared class hash returns the default value `0`. `find_element` then searches for compiled class hash `0`, which is absent from `compiled_class_facts_bundle`, causing a Cairo panic. The contract becomes permanently uncallable at the OS level.

This is structurally identical to the external report's pattern: **shared state (storage) is preserved across the class update, but the new class (interface) is incompatible or nonexistent**, making the shared state permanently inaccessible.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any ERC-20, vault, or DeFi contract deployed on StarkNet that:
1. Exposes a public or insufficiently guarded upgrade path (calling `replace_class`), **or**
2. Is controlled by a malicious/compromised admin

can have its class hash replaced with an arbitrary undeclared felt value. Once this state is committed to the Patricia tree, the contract's storage — including all token balances — is permanently inaccessible. No recovery path exists at the protocol level because the OS has no mechanism to revert a committed state root.

The `StateEntry` struct ties `class_hash`, `storage_ptr`, and `nonce` together: [3](#0-2) 

The storage is shared (analogous to shared balances in the report), but the class (analogous to the interface/allowance logic) is replaced with a nonexistent one, making the shared storage permanently unreachable.

---

### Likelihood Explanation

**Medium.**

The `replace_class` syscall is callable by any contract from within its own execution. Contracts implementing upgradeable proxy patterns — a common design on StarkNet — frequently expose an `upgrade(new_class_hash)` function. If that function lacks strict access control (e.g., `assert caller == owner`), any unprivileged transaction sender can supply an undeclared class hash. The OS provides zero safety net: it accepts any felt value as the new class hash. The missing check is self-documented in the codebase as a known gap.

---

### Recommendation

In `execute_replace_class`, before updating `contract_state_changes`, verify that `class_hash` exists in `contract_class_changes` (i.e., it has been declared in the current or a prior block). Concretely:

```cairo
// Verify the class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the check already performed implicitly in `execute_entry_point` but must be enforced eagerly at the point of `replace_class` to prevent invalid state from being committed. The `TODO` at line 898 should be resolved before production deployment. [4](#0-3) 

---

### Proof of Concept

1. **Attacker deploys** (or identifies) a contract `C` that exposes a public `upgrade(new_class_hash: felt)` function with no access control, which internally calls `replace_class(new_class_hash)`.

2. **Victim deposits** tokens into contract `C` (e.g., an LP pool or vault). Contract `C`'s storage now holds the victim's balance.

3. **Attacker sends** an invoke transaction calling `C.upgrade(0xdeadbeef)` where `0xdeadbeef` is an arbitrary felt that has never been declared as a class.

4. **OS executes** `execute_replace_class`:
   - No existence check is performed (line 898 TODO).
   - `contract_state_changes` is updated: `C.class_hash = 0xdeadbeef`.
   - The transaction succeeds and is committed to the state root.

5. **Victim attempts** to withdraw by calling `C.__execute__` in a subsequent transaction.

6. **OS executes** `execute_entry_point` for contract `C`:
   - `dict_read(contract_class_changes, key=0xdeadbeef)` → returns `0` (undeclared).
   - `find_element(..., key=0)` → panics (0 not in `compiled_class_facts`).
   - The call fails permanently.

7. **Result**: All funds in contract `C`'s storage are permanently frozen. The victim cannot recover their tokens. [5](#0-4) [6](#0-5)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    let revert_log = &revert_log[1];

    return ();
}
