### Title
Bouncer `state_diff_size` Omits Deprecated Declared Classes, Causing `concatenated_counts` in Block Hash to Commit a `state_diff_length` Exceeding the Enforced Limit ŌĆö (File: crates/blockifier/src/state/cached_state.rs)

---

### Summary

The `Bouncer`'s `state_diff_size` dimension is derived from `StateChangesKeys`, which has **no field for deprecated (Cairo 0) class declarations**. However, `ThinStateDiff::len()` ŌĆö the value used as `state_diff_length` in `concat_counts` and therefore committed inside `concatenated_counts` in the block hash ŌĆö **does** count `deprecated_declared_classes`. An unprivileged user can submit deprecated-declare transactions to make the block's actual `state_diff_length` exceed the bouncer's configured `state_diff_size` limit, producing a `concatenated_counts` commitment that encodes a larger `state_diff_length` than the bouncer's accounting permits.

---

### Finding Description

**Step 1 ŌĆö `StateChangesKeys` has no slot for deprecated declared classes.**

`StateChangesKeys` tracks five sets:

```
nonce_keys, class_hash_keys, storage_keys, compiled_class_hash_keys, modified_contracts
``` [1](#0-0) 

`StateChangesKeys::count()` converts these into `StateChangesCount`:

```
n_storage_updates, n_class_hash_updates, n_compiled_class_hash_updates, n_modified_contracts
``` [2](#0-1) 

There is no `n_deprecated_declared_classes` field. Deprecated (Cairo 0) class declarations write only to `StateMaps::declared_contracts`, which is never mirrored into any `StateChangesKeys` field.

**Step 2 ŌĆö The bouncer's `state_diff_size` is derived from `StateChangesKeys`.**

`Bouncer::try_update` computes `marginal_state_changes_keys` as the set-difference of the transaction's keys against the already-accumulated keys, then passes them to `get_tx_weights` to produce `BouncerWeights::state_diff_size`. [3](#0-2) 

Because `StateChangesKeys` carries no deprecated-class slot, every deprecated `DECLARE` transaction contributes **zero** to `state_diff_size`.

**Step 3 ŌĆö `ThinStateDiff::len()` counts `deprecated_declared_classes`.**

```rust
result += self.deprecated_declared_classes.len();
``` [4](#0-3) 

The docstring explicitly states: *"This has the same value as `state_diff_length` in the corresponding `BlockHeader`."*

**Step 4 ŌĆö `state_diff_length` is committed in the block hash via `concat_counts`.**

`calculate_block_commitments` calls `state_diff.len()` and passes it to `concat_counts`:

```rust
let state_diff_length = state_diff.len();
...
let concatenated_counts = concat_counts(
    transactions_data.len(),
    event_leaf_elements.len(),
    state_diff.len(),   // ŌåÉ includes deprecated_declared_classes
    l1_da_mode,
);
``` [5](#0-4) 

`concatenated_counts` is then chained directly into the Poseidon block hash: [6](#0-5) 

**Step 5 ŌĆö `BlockExecutionArtifacts::new` uses `commitment_state_diff` (which includes deprecated classes) for commitments.**

```rust
let (header_commitments, measurements) = calculate_block_commitments(
    &transactions_data,
    ThinStateDiff::from(commitment_state_diff.clone()),  // includes deprecated_declared_classes
    l1_da_mode,
    &block_info.starknet_version,
)
.await;
``` [7](#0-6) 

**The invariant broken:** `bouncer_weights.state_diff_size` should upper-bound the actual `state_diff_length` committed in the block hash. With N deprecated declared classes in a block, the committed `state_diff_length` equals `bouncer_weights.state_diff_size + N`, violating the bouncer's configured `block_max_capacity.state_diff_size` limit.

---

### Impact Explanation

**Impact: Critical ŌĆö Incorrect bouncer resource accounting with economic impact.**

1. **Block size limit bypass:** A block can commit to a `state_diff_length` that exceeds `block_max_capacity.state_diff_size`. The bouncer never rejects the block because it never counts deprecated declared classes.

2. **Wrong `concatenated_counts` relative to bouncer accounting:** The `concatenated_counts` field committed in the block hash encodes a `state_diff_length` that is inconsistent with the bouncer's `state_diff_size` dimension. Downstream consumers (P2P sync client, proof manager) that rely on `state_diff_length` from the block header will observe a value larger than the bouncer's enforced limit.

3. **Economic impact on proving and DA:** A larger-than-intended `state_diff_length` increases Patricia-trie proving work and L1 DA costs. The bouncer's `state_diff_size` limit exists precisely to bound these costs; bypassing it shifts unbounded cost onto the sequencer and provers.

4. **P2P sync integrity:** The P2P sync client uses `state_diff_length` from the stored block header as the termination condition for accumulating state diff chunks: [8](#0-7) 

If the committed `state_diff_length` exceeds what the bouncer's limit would allow, syncing nodes will accept a state diff larger than the sequencer's own capacity configuration.

---

### Likelihood Explanation

Deprecated (Cairo 0) `DECLARE` transactions (version 0/1) remain valid in the current protocol. `deprecated_declared_classes` is still a live field in `ThinStateDiff` and is written by `append_state_diff` in production storage. [9](#0-8) 

Any unprivileged user can submit deprecated declare transactions. There is no gateway-level cap on the number of deprecated declared classes per block independent of the bouncer's `state_diff_size` check. The trigger requires no special privilege and is reachable through the normal transaction submission path.

---

### Recommendation

Add a `deprecated_declared_class_keys: HashSet<ClassHash>` field to `StateChangesKeys` and populate it whenever a deprecated class declaration is processed. Include `deprecated_declared_class_keys.len()` in the `state_diff_size` contribution returned by `get_tx_weights`, mirroring the `deprecated_declared_classes.len()` term in `ThinStateDiff::len()`. This restores the invariant that `bouncer_weights.state_diff_size` upper-bounds the actual `state_diff_length` committed in `concatenated_counts`.

---

### Proof of Concept

1. Configure a sequencer with `block_max_capacity.state_diff_size = S`.
2. Submit `S + K` deprecated (Cairo 0) `DECLARE` transactions (each declaring a distinct class hash) in a single block proposal, with no other state-changing transactions.
3. The bouncer accumulates `state_diff_size = 0` (no `StateChangesKeys` field is incremented), so the block is never closed on the `state_diff_size` dimension regardless of K.
4. `BlockExecutionArtifacts::new` calls `calculate_block_commitments` with `ThinStateDiff::from(commitment_state_diff)`, where `ThinStateDiff::len() = K` (all deprecated declared classes).
5. `concat_counts` encodes `state_diff_length = K` into `concatenated_counts`.
6. The final block hash commits to `concatenated_counts` encoding `state_diff_length = K > S`, while `bouncer_weights.state_diff_size = 0 Ōēż S`.

The committed `state_diff_length` in the block hash exceeds the bouncer's configured limit by `K`, with no enforcement path to prevent it.
