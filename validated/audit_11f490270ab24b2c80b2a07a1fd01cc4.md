### Title
P2P State Diff Assembly Verifies Only Length, Not Commitment Hash, Allowing a Malicious Peer to Inject a Wrong `ThinStateDiff` into Storage - (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

---

### Summary

`StateDiffStreamBuilder::parse_data_for_block` assembles a `ThinStateDiff` from peer-supplied `StateDiffChunk` messages by checking only that the accumulated length equals the `state_diff_length` field of the stored block header. It never computes `calculate_state_diff_hash` on the assembled result and never compares it against the `state_diff_commitment` that is also stored in the same header. A malicious peer can therefore send chunks whose total length is correct but whose content is arbitrary, causing the node to persist a wrong state diff, advance the state marker, and subsequently commit a wrong global state root to the Patricia trie.

---

### Finding Description

The analog to the LPIssuer deposit bug is exact:

| LPIssuer | Sequencer P2P sync |
|---|---|
| User provides `tokenAmounts`; balanced amounts are computed from current TVL | Sync client assembles `ThinStateDiff` from peer chunks; termination is driven by `state_diff_length` from the header |
| Strategist changes TVL between submission and execution | Malicious peer sends chunks with correct total length but wrong storage/class/nonce values |
| No minimum-return check on the minted LP amount | No hash check of the assembled diff against `state_diff_commitment` |
| User receives fewer LP tokens than expected | Node stores a wrong `ThinStateDiff` and commits a wrong global root |

**Root cause — `parse_data_for_block`** (`crates/apollo_p2p_sync/src/client/state_diff.rs`, lines 51–110):

```rust
let target_state_diff_len = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    ...
    .state_diff_length          // ← only the length is read
    ...;

while current_state_diff_len < target_state_diff_len {
    ...
    current_state_diff_len += state_diff_chunk.len();
    unite_state_diffs(&mut result, state_diff_chunk)?;
}

if current_state_diff_len != target_state_diff_len { ... }   // length check only

validate_deprecated_declared_classes_non_conflicting(&result)?;
Ok(Some((result, block_number)))   // ← returned without hash verification
```

The same block header that supplies `state_diff_length` also carries `state_diff_commitment` — the Poseidon hash of the correct state diff, as confirmed by the test that asserts `block_header.state_diff_commitment.unwrap() == block_header_commitments.state_diff_commitment`. That commitment is never read inside `parse_data_for_block`, and `calculate_state_diff_hash` is never called on `result`.

The assembled diff is immediately written to persistent storage and the state marker is advanced:

```rust
storage_writer.begin_rw_txn()?.append_state_diff(self.1, self.0)?.commit()?;
STATE_SYNC_STATE_MARKER.set_lossy(self.1.unchecked_next().0);
```

From this point the wrong diff is the authoritative state for that block number.

The committer does have an optional `verify_state_diff_hash` flag, but:
1. It is a configurable option (default is `false` in tests; production deployments may vary).
2. Even when enabled, it fires only after the diff is already in storage and the state marker has advanced — RPC queries issued in that window return wrong values.
3. When `verify_state_diff_hash` is `false`, the committer calls `commit_state_diff` with the wrong diff, producing a wrong Patricia trie root and a wrong `global_root`.

---

### Impact Explanation

**Wrong storage value / state root (Critical / High):**

- The wrong `ThinStateDiff` is committed to the Patricia trie via `commit_state_diff`, producing a wrong `contracts_trie_root_hash` / `classes_trie_root_hash` and therefore a wrong `GlobalRoot`.
- `calculate_block_hash` chains `state_root.0` directly into the block hash; a wrong global root produces a wrong block hash for every subsequent block.
- RPC endpoints that read storage values, nonces, or class hashes from the state storage return wrong authoritative-looking values for the affected block and all blocks that inherit the corrupted trie.
- SNOS proof inputs derived from the Patricia witness paths (`StateCommitmentInfos`) are built from the wrong trie, making proofs invalid or proving wrong facts.

---

### Likelihood Explanation

Any node reachable on the P2P network can serve `StateDiffChunk` responses. The attacker needs only to be the peer that answers the state diff query for a given block — either by being the first to respond or by eclipsing the victim. No privileged key material is required. The attack is silent: the victim node logs no error and advances its state marker normally.

---

### Recommendation

After the length-equality check at line 99, compute the Poseidon commitment of the assembled diff and compare it against the header's `state_diff_commitment`:

```rust
// After: if current_state_diff_len != target_state_diff_len { ... }
let expected_commitment = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .expect("header must exist")
    .state_diff_commitment
    .ok_or(P2pSyncClientError::OldHeaderInStorage {
        block_number,
        missing_field: "state_diff_commitment",
    })?;

let actual_commitment = calculate_state_diff_hash(&result);
if actual_commitment != expected_commitment {
    return Err(ParseDataError::BadPeer(
        BadPeerError::WrongStateDiffCommitment { block_number }
    ));
}
```

This mirrors the check already present in `Committer::commit_or_load` when `verify_state_diff_hash` is enabled, but moves it to the point of ingestion — before the diff is written to storage.

---

### Proof of Concept

1. Victim node starts P2P sync. It has already stored the block header for block N, which contains `state_diff_length = 2` and `state_diff_commitment = H(correct_diff)`.
2. Attacker peer answers the state diff query for block N with two `StateDiffChunk::ContractDiff` messages whose combined `len()` equals 2 but whose storage values differ from the real block (e.g., replacing `storage[key] = 100` with `storage[key] = 0`).
3. `parse_data_for_block` accumulates both chunks, passes the length check (`current_state_diff_len == 2 == target_state_diff_len`), passes `validate_deprecated_declared_classes_non_conflicting`, and returns `Ok(Some((result, block_number)))`.
4. `write_to_storage` calls `append_state_diff(block_number, result)` and advances `STATE_SYNC_STATE_MARKER`.
5. Any RPC call to `starknet_getStorageAt` for the affected contract now returns `0` instead of `100`.
6. The committer (if `verify_state_diff_hash = false`) calls `commit_state_diff` with the wrong diff, producing a wrong `contracts_trie_root_hash`. This wrong root propagates into `calculate_block_hash` for block N and all subsequent blocks, and into SNOS proof inputs. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) [7](#0-6)

    fn parse_data_for_block<'a>(
        state_diff_chunks_response_manager: &'a mut ClientResponsesManager<
            DataOrFin<StateDiffChunk>,
        >,
        block_number: BlockNumber,
        storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            let mut result = ThinStateDiff::default();
            let mut prev_result_len = 0;
            let mut current_state_diff_len = 0;
            let target_state_diff_len = storage_reader
                .begin_ro_txn()?
                .get_block_header(block_number)?
                .expect("A header with number lower than the header marker is missing")
                .state_diff_length
                .ok_or(P2pSyncClientError::OldHeaderInStorage {
                    block_number,
                    missing_field: "state_diff_length",
                })?;

            while current_state_diff_len < target_state_diff_len {
                let maybe_state_diff_chunk = state_diff_chunks_response_manager
                    .next()
                    .await
                    .ok_or(ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }))?;
                let Some(state_diff_chunk) = maybe_state_diff_chunk?.0 else {
                    if current_state_diff_len == 0 {
                        return Ok(None);
                    } else {
                        return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                            expected_length: target_state_diff_len,
                            possible_lengths: vec![current_state_diff_len],
                        }));
                    }
                };
                prev_result_len = current_state_diff_len;
                if state_diff_chunk.is_empty() {
                    return Err(ParseDataError::BadPeer(BadPeerError::EmptyStateDiffPart));
                }
                // It's cheaper to calculate the length of `state_diff_part` than the length of
                // `result`.
                current_state_diff_len += state_diff_chunk.len();
                unite_state_diffs(&mut result, state_diff_chunk)?;
            }

            if current_state_diff_len != target_state_diff_len {
                return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                    expected_length: target_state_diff_len,
                    possible_lengths: vec![prev_result_len, current_state_diff_len],
                }));
            }

            validate_deprecated_declared_classes_non_conflicting(&result)?;
            Ok(Some((result, block_number)))
        }
        .boxed()
    }

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}

impl<T: StorageTransaction<Mode = RW>> StateStorageWriter for T {
    #[sequencer_latency_histogram(STORAGE_APPEND_THIN_STATE_DIFF_LATENCY, false)]
    fn append_state_diff(
        self,
        block_number: BlockNumber,
        thin_state_diff: ThinStateDiff,
    ) -> StorageResult<Self> {
        let tables = self.tables();
        let inner_txn = self.txn();
        let file_offset_table = inner_txn.open_table(&tables.file_offsets)?;
        let markers_table = self.open_table(&tables.markers)?;
        let state_diffs_table = self.open_table(&tables.state_diffs)?;
        let nonces_table = self.open_table(&tables.nonces)?;
        let deployed_contracts_table = self.open_table(&tables.deployed_contracts)?;
        let storage_table = self.open_table(&tables.contract_storage)?;
        let declared_classes_block_table = self.open_table(&tables.declared_classes_block)?;
        let deprecated_declared_classes_block_table =
            self.open_table(&tables.deprecated_declared_classes_block)?;
        let compiled_class_hash_table = self.open_table(&tables.compiled_class_hash)?;

        // Write state.
        write_deployed_contracts(
            &thin_state_diff.deployed_contracts,
            inner_txn,
            block_number,
            &deployed_contracts_table,
            &nonces_table,
        )?;
        write_storage_diffs(
            &thin_state_diff.storage_diffs,
            inner_txn,
            block_number,
            &storage_table,
        )?;
        // Must be called after write_deployed_contracts since the nonces are updated there.
        write_nonces(&thin_state_diff.nonces, inner_txn, block_number, &nonces_table)?;

        for (class_hash, _) in &thin_state_diff.class_hash_to_compiled_class_hash {
            let not_declared = declared_classes_block_table.get(inner_txn, class_hash)?.is_none();
            if not_declared {
                declared_classes_block_table.insert(inner_txn, class_hash, &block_number)?;
            }
        }

        write_compiled_class_hashes(
            &thin_state_diff.class_hash_to_compiled_class_hash,
            inner_txn,
            block_number,
            &compiled_class_hash_table,
        )?;

        for class_hash in thin_state_diff.deprecated_declared_classes.iter() {
            // Cairo0 classes can be declared in different blocks. The first block to declare the
            // class is recorded here.
            if deprecated_declared_classes_block_table.get(inner_txn, class_hash)?.is_none() {
                deprecated_declared_classes_block_table.insert(
                    inner_txn,
                    class_hash,
                    &block_number,
                )?;
            }
        }

        // Write state diff.
        let location = self.file_handlers().append_state_diff(&thin_state_diff);
        state_diffs_table.append(inner_txn, &block_number, &location)?;
        file_offset_table.upsert(inner_txn, &OffsetKind::ThinStateDiff, &location.next_offset())?;

        update_marker_to_next_block(inner_txn, &markers_table, MarkerKind::State, block_number)?;

        advance_compiled_class_marker_over_blocks_without_classes(
            inner_txn,
            &markers_table,
            &state_diffs_table,
            self.file_handlers(),
        )?;

        Ok(self)
    }
