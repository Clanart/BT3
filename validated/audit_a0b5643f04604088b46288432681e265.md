### Title
Unvalidated Class Hash in `execute_replace_class` Allows OS Proof Failure Leading to Network Halt — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary class hash without verifying it corresponds to a declared contract class. A malicious contract can replace its own class hash with an undeclared value and then invoke itself via `call_contract` in the same transaction. When the OS processes the subsequent `call_contract` syscall, it attempts to look up the compiled class for the invalid hash, causing `find_element` to fail with an assertion error — an unrecoverable OS-level failure that prevents proof generation and halts the network.

---

### Finding Description

In `execute_replace_class`, the new class hash is taken directly from the syscall request with no validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges the missing validation. The `class_hash` field from the request is written directly into `contract_state_changes` with no check against `contract_class_changes` or `compiled_class_facts_bundle`.

When a subsequent `call_contract` targets this contract, `execute_call_contract` reads the state entry and constructs an `ExecutionContext` with the invalid `class_hash`:

```cairo
let (state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
    key=contract_address
);
tempvar execution_context: ExecutionContext* = new ExecutionContext(
    ...
    class_hash=state_entry.class_hash,   // ← invalid hash
    ...
);
``` [2](#0-1) 

This execution context is passed to `execute_entry_point`, which performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // ← undeclared hash → returns 0
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,           // ← 0, not present → assertion failure
);
``` [3](#0-2) 

`find_element` uses a hint to locate the element and then asserts the result. If no compiled class with hash `0` exists in `compiled_class_facts_bundle`, the assertion fails at the Cairo VM level — this is not a graceful transaction revert but an unrecoverable OS execution failure. The proof cannot be generated for the block.

The syscalls are processed sequentially in `execute_syscalls`. The `replace_class` syscall commits the invalid class hash to `contract_state_changes` before the `call_contract` syscall is processed, so the invalid hash is visible when the inner call is dispatched. [4](#0-3) 

---

### Impact Explanation

If the OS Cairo program fails with an assertion error, the STARK proof for the block cannot be generated. The block cannot be finalized on L1. No subsequent blocks can be built on top of it. This constitutes a **total network shutdown** matching the allowed High impact: *"Network not being able to confirm new transactions."*

Additionally, any funds held by the contract whose class was replaced with an invalid hash are permanently inaccessible, matching the Critical impact: *"Permanent freezing of funds."*

---

### Likelihood Explanation

The attack is reachable by any unprivileged user who can:
1. Deploy a contract (permissionless on StarkNet).
2. Send a transaction that calls a function in that contract which issues `replace_class` with an arbitrary felt value followed by `call_contract` to itself.

No privileged role, leaked key, or malicious operator is required. The only mitigation that could block this at the sequencer level would be if the blockifier independently validates the class hash in `replace_class` — but the TODO comment in the OS code strongly suggests this validation is absent at the OS layer, and a discrepancy between blockifier acceptance and OS provability is itself a critical correctness gap.

---

### Recommendation

In `execute_replace_class`, before writing the new class hash to `contract_state_changes`, verify that the class hash exists in `contract_class_changes` (i.e., it has been declared):

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This is exactly what the existing TODO comment at line 898 calls for. The check should mirror the lookup already performed in `execute_entry_point`. [5](#0-4) 

---

### Proof of Concept

1. **Deploy** a malicious Cairo contract `MaliciousContract` with an entry point `attack()`:
   ```
   fn attack() {
       replace_class(0xDEADBEEF);          // arbitrary undeclared class hash
       call_contract(self_address, 'attack', []);  // call itself
   }
   ```

2. **Send** an invoke transaction calling `MaliciousContract.attack()`.

3. **OS syscall processing** (in `execute_syscalls`):
   - Processes `REPLACE_CLASS_SELECTOR` → `execute_replace_class` writes `class_hash=0xDEADBEEF` into `contract_state_changes` for `MaliciousContract`. No validation occurs.
   - Processes `CALL_CONTRACT_SELECTOR` → `execute_call_contract` reads `state_entry.class_hash = 0xDEADBEEF` and calls `execute_entry_point`.

4. **In `execute_entry_point`**:
   - `dict_read(contract_class_changes, key=0xDEADBEEF)` → returns `0` (undeclared).
   - `find_element(..., key=0)` → no compiled class with hash `0` exists → **assertion failure**.

5. **Result**: The OS Cairo program aborts. The STARK proof cannot be generated. The block is unprovable. The network halts. [6](#0-5) [7](#0-6)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
