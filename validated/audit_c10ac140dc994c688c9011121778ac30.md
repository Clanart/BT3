### Title
Missing Validation of `class_hash` in `execute_replace_class` Allows Permanent Fund Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts a caller-supplied `class_hash` and writes it directly into the contract state without verifying that the hash corresponds to a previously declared class. This is the exact same vulnerability class as the reported DCA bug: a user-provided parameter is accepted and applied to state with no cross-validation against authoritative stored data. A contract can replace its own class with an arbitrary, undeclared felt value, permanently rendering itself uncallable and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class`, the OS reads `request.class_hash` directly from the syscall buffer and writes it into `contract_state_changes` for the executing contract address. There is no check that the supplied hash exists in the declared class registry (`contract_class_changes`).

The code itself contains an explicit acknowledgment of this gap:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The missing check is: `assert class_hash in contract_class_changes` (i.e., the class must have been declared before it can be used as a replacement target).

By contrast, the `execute_declare_transaction` path in `transaction_impls.cairo` enforces `prev_value=0` to guarantee a class is declared at most once and always writes a valid `compiled_class_hash`:

```cairo
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [2](#0-1) 

`execute_replace_class` performs no equivalent guard.

---

### Impact Explanation

When a contract's `class_hash` in `contract_state_changes` is set to an undeclared value, any subsequent transaction that calls that contract reaches `execute_entry_point`, which does:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
``` [3](#0-2) 

`dict_read` on an undeclared hash returns `0`. `find_element` with key `0` will fail with an assertion error if no compiled class with hash `0` exists in the bundle, causing the OS execution to abort. Even if the sequencer defensively excludes such transactions from blocks, the contract is permanently uncallable: its class hash in the Merkle state tree points to a non-existent class, and no future transaction can invoke it to call `replace_class` again to recover. Any ERC-20 balances, NFTs, or protocol funds held in the contract's storage are permanently frozen.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

The `replace_class` syscall is a standard, publicly documented StarkNet syscall callable by any contract. An attacker needs only to:

1. Deploy a contract (permissionless on StarkNet).
2. Attract user deposits (e.g., by presenting as a legitimate vault or DeFi protocol).
3. Call `replace_class` with an arbitrary undeclared felt (e.g., `1`).

No privileged role, leaked key, or external dependency is required. The attacker-controlled entry path is entirely on-chain and reachable by any unprivileged contract deployer.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, assert that it exists in `contract_class_changes` (i.e., it was previously declared):

```cairo
// Verify the replacement class has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the validation already present in `execute_declare_transaction` and closes the gap noted in the existing TODO comment.

---

### Proof of Concept

1. Attacker deploys `MaliciousVault` — a contract that accepts user token deposits and exposes a `drain()` function callable only by the owner.
2. Users deposit funds; the contract accumulates balances in its storage.
3. Attacker calls `drain()` which internally invokes the `replace_class` syscall with `class_hash = 0xdeadbeef` (an undeclared felt).
4. The OS executes `execute_replace_class` at lines 878–916 of `syscall_impls.cairo`, writes `class_hash=0xdeadbeef` into `contract_state_changes` for `MaliciousVault`'s address — **no validation occurs**.
5. The block is proven and the state root is updated; `MaliciousVault`'s leaf in the contract state Merkle tree now encodes `class_hash=0xdeadbeef`.
6. Any subsequent call to `MaliciousVault` causes `execute_entry_point` to look up compiled class hash for `0xdeadbeef`, get `0` from `contract_class_changes`, and fail at `find_element` — the contract is permanently uncallable.
7. All user funds stored in `MaliciousVault`'s storage are permanently frozen with no recovery path. [4](#0-3)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    let revert_log = &revert_log[1];

    return ();
}
