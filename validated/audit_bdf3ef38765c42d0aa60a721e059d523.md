### Title
Missing Class Hash Existence Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS does not validate that the new class hash provided to the `replace_class` syscall corresponds to a previously declared class. This is an exact structural analog to H-4: the OS checks one condition (sufficient gas) but skips a second critical condition (class existence), trusting the caller to supply a valid hash. Any contract that calls `replace_class` with an undeclared class hash will have its class permanently set to a value with no associated code, making the contract permanently non-callable and freezing all funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function (lines 878–916) handles the `replace_class` syscall. It deducts gas and unconditionally writes the new class hash into `contract_state_changes`, but explicitly omits the check that the new class hash is a declared class. This is acknowledged by a developer TODO at line 898:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The `contract_class_changes` dictionary (populated by `execute_declare_transaction` in `transaction_impls.cairo`) tracks all declared class hashes. The OS enforces that a class may be declared only once via `prev_value=0`:

```cairo
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [2](#0-1) 

However, `execute_replace_class` never consults `contract_class_changes` to verify that `class_hash` has a corresponding compiled class hash entry. The OS simply writes the arbitrary felt value into the contract's `StateEntry.class_hash` field and commits it to the state tree via `state_update` in `state.cairo`. [3](#0-2) 

The `StateEntry` struct stores `class_hash` as a plain felt with no runtime invariant enforcing it points to declared code: [4](#0-3) 

---

### Impact Explanation

Once `replace_class` is executed with an undeclared class hash, the contract's `StateEntry.class_hash` is permanently committed to the global state root. Every subsequent call to that contract will attempt to look up entry points in a class that does not exist in the compiled class facts bundle. The call will fail unconditionally. There is no recovery path in the OS — no syscall exists to reset a class hash without calling `replace_class` again, which itself requires the contract to be callable. Any ERC-20 balances, ETH-equivalent tokens, or other assets stored in the contract's storage are permanently inaccessible.

This matches the **Critical: Permanent freezing of funds** impact category.

---

### Likelihood Explanation

The `replace_class` syscall is reachable by any unprivileged transaction sender through the following realistic paths:

1. **Upgradeable proxy contracts**: A proxy that exposes an `upgrade(new_class_hash)` function callable by any user (or with insufficient access control) can be called with an arbitrary, undeclared class hash. The OS will accept it.
2. **Contracts with buggy upgrade logic**: A contract that computes a class hash from user-supplied input and passes it to `replace_class` without pre-validating it against declared classes.
3. **Malicious contract self-destruction**: A contract intentionally calls `replace_class` with `0xdeadbeef` after receiving user funds, permanently freezing them.

The OS is the last line of defense. Because the OS does not enforce the invariant, no amount of care in contract code can guarantee safety if the OS accepts the invalid state transition.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that it exists in `contract_class_changes` (i.e., has a non-zero compiled class hash entry). A sketch of the fix:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,  // add this implicit arg
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // Verify the new class hash is a declared class.
    let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
    with_attr error_message("Class hash is not declared.") {
        assert_not_zero(compiled_class_hash);
    }

    // Proceed with the state update.
    ...
}
```

### Proof of Concept

1. Declare and deploy a contract `Proxy` that exposes `upgrade(new_class_hash: felt)` which internally calls the `replace_class` syscall with `new_class_hash`.
2. Send 1000 STRK to `Proxy` (stored in its ERC-20 balance slot in storage).
3. As an unprivileged user, call `Proxy.upgrade(0xdeadbeef)` where `0xdeadbeef` has never been declared.
4. The OS executes `execute_replace_class`: gas is deducted, `contract_state_changes` is updated with `class_hash=0xdeadbeef`, the revert log records the old class hash, and the state root is updated.
5. Any subsequent call to `Proxy` (e.g., to transfer the 1000 STRK out) fails: the OS looks up class hash `0xdeadbeef` in the compiled class facts bundle, finds nothing, and the call reverts.
6. The 1000 STRK is permanently frozen. No recovery is possible because `replace_class` itself requires the contract to be callable.

The root cause — the missing `contract_class_changes` lookup — is confirmed at: [5](#0-4)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
