### Title
Stale `feemarketParams` Snapshot in `NewDynamicFeeChecker` Allows Cosmos SDK Transactions to Bypass Current Base Fee — (`ante/evm/fee_checker.go`, `evmd/ante/handler_options.go`)

### Summary

`NewDynamicFeeChecker` for Cosmos SDK (non-EVM) transactions captures a one-time snapshot of `feemarketParams` at ante-handler construction time. Because the EIP-1559 base fee changes every block, the fee checker permanently uses a stale base fee, allowing Cosmos SDK transactions to be accepted with fees below the live base fee requirement.

### Finding Description

`newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712` both call `NewDynamicFeeChecker` once at app initialization, passing a pointer to a local `feemarketParams` snapshot:

```go
// evmd/ante/handler_options.go
func newCosmosAnteHandler(ctx sdk.Context, options HandlerOptions, ...) sdk.AnteHandler {
    evmParams    := options.EvmKeeper.GetParams(ctx)       // snapshot at init
    feemarketParams := options.FeeMarketKeeper.GetParams(ctx) // snapshot at init
    ...
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
    ...
}
``` [1](#0-0) 

The returned closure permanently captures the pointer to this local variable:

```go

    feemarketParams *feemarkettypes.Params) authante.TxFeeChecker {
    return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
        ...
        baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams) // reads stale snapshot
        ...
    }
}
``` [2](#0-1) 

`types.GetBaseFee` reads `feemarketParams.GetBaseFee()` — the `BaseFee` field of the captured snapshot, not the live store:

```go

    ...
    baseFee := feemarketParams.GetBaseFee()

The live base fee is updated every block in `BeginBlock`:

```go

    ...

``` [4](#0-3) 

`SetBaseFee` writes the new value into the KV store via `SetParams`:

```go
func (k Keeper) SetBaseFee(ctx sdk.Context, baseFee *big.Int) {
    params := k.GetParams(ctx)
    params.BaseFee = ethermint.SaturatedNewInt(baseFee)
    err := k.SetParams(ctx, params)
    ...
}
``` [5](#0-4) 

The snapshot captured in `newCosmosAnteHandler` is **never refreshed**. The `feemarketParams` local variable is heap-allocated (its address escapes into the closure), but its `BaseFee` field is never updated after construction.

By contrast, `newEthAnteHandler` (for EVM transactions) correctly fetches fresh params on every invocation via `EVMBlockConfig`:

```go
func newEthAnteHandler(options HandlerOptions) sdk.AnteHandler {
    return func(ctx sdk.Context, tx sdk.Tx, simulate bool) (sdk.Context, error) {
        blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
        ...
        baseFee := blockCfg.BaseFee  // always current
        ...
    }
}
``` [6](#0-5) 

This creates an asymmetry: EVM transactions are validated against the live base fee; Cosmos SDK transactions are validated against the genesis base fee.

### Impact Explanation

The `NewDynamicFeeChecker` is wired into `authante.NewDeductFeeDecorator` for Cosmos SDK transactions. When the base fee rises above the genesis value (which happens whenever blocks are above the gas target), Cosmos SDK transactions with fees calibrated to the old lower base fee pass the fee check and are deducted at the stale (lower) effective price. This is a fee market bypass: the protocol accepts and commits Cosmos SDK transactions that do not meet the current EIP-1559 base fee requirement, causing valid user funds/fees to be mis-accounted (under-charged). This matches the allowed High impact: "ante handler bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."

### Likelihood Explanation

The base fee changes every block whenever gas usage deviates from the target. On any active chain the base fee will diverge from the genesis value within minutes of launch. Any user who submits a Cosmos SDK transaction (e.g., governance, IBC, staking) with fees calibrated to the current (stale) base fee will trigger this path. No special privileges are required; any unprivileged user submitting a standard Cosmos SDK transaction exercises this code path.

### Recommendation

`newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712` should not capture a one-time snapshot of `feemarketParams`. Instead, `NewDynamicFeeChecker` should read the live base fee from the store on every invocation, mirroring the pattern used in `newEthAnteHandler`:

```go
// Option A: read live params inside the closure
func NewDynamicFeeChecker(fmKeeper FeeMarketKeeper, evmKeeper EVMKeeper) authante.TxFeeChecker {
    return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
        feemarketParams := fmKeeper.GetParams(ctx)   // live read
        evmParams       := evmKeeper.GetParams(ctx)  // live read
        baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, &feemarketParams)
        ...
    }
}
```

Alternatively, pass the keepers rather than snapshots so the closure always reads current state.

### Proof of Concept

1. Chain starts with genesis `BaseFee = 1_000_000_000` (1 Gwei). `newCosmosAnteHandler` is called; `feemarketParams.BaseFee = 1_000_000_000` is captured.
2. Blocks run at above-target gas usage. After N blocks, `BeginBlock` has updated the live `BaseFee` to `2_000_000_000` (2 Gwei) in the KV store.
3. A user submits a Cosmos SDK governance `MsgVote` with `fee = 1_000_000_000 * gasLimit` (1 Gwei price).
4. `newEthAnteHandler` would reject an EVM tx at this price (it reads the live 2 Gwei base fee). But `newCosmosAnteHandler` routes the governance tx through `NewDynamicFeeChecker`, which reads `feemarketParams.BaseFee = 1_000_000_000` (the stale snapshot) and accepts the transaction.
5. The transaction is committed with fees 50% below the current base fee requirement, bypassing the EIP-1559 fee market for Cosmos SDK transactions. [7](#0-6) [2](#0-1) [3](#0-2) [4](#0-3)

func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
	if !IsLondon(ethCfg, height) {
		return nil
	}
	baseFee := feemarketParams.GetBaseFee()
	// should not be nil if london hardfork enabled
	if baseFee == nil {
		return new(big.Int)
	}
	return baseFee
}

func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
}
