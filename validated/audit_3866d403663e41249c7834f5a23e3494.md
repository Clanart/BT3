### Title
Expensive Cryptographic Proof Verification Runs Before Fee/Balance Checks in Gateway Admission — (`crates/apollo_transaction_converter/src/transaction_converter.rs`)

### Summary

The gateway performs the expensive `verify_recursive_circuit` call (inside `spawn_proof_verification`) before the stateful fee-bounds and balance checks. An attacker can submit Invoke V3 transactions with structurally valid `proof_facts` metadata but a corrupted proof blob, forcing the sequencer to run the full cryptographic verifier at no cost to the sender.

### Finding Description

The gateway admission path in `add_tx_inner` follows this order:

1. **Stateless validation** — checks format, signature, and `max_proof_size` limit.
2. **`convert_rpc_tx_to_internal_and_executable_txs`** — internally calls `convert_rpc_tx_to_internal_rpc_tx`, which calls `spawn_proof_verification`. This spawns a blocking tokio task that runs `starknet_proof_verifier::verify_proof` → `verify_recursive_circuit`. The gateway **awaits** this handle before proceeding.
3. **Stateful validation** — only after step 2 does `extract_state_nonce_and_run_validations` run, which calls `check_fee_bounds` and `verify_can_pay_committed_bounds`. [1](#0-0) 

The proof verification task is spawned and awaited in the converter: [2](#0-1) [3](#0-2) 

The gateway test confirms that a failed verification causes an early return **before** `convert_internal_rpc_tx_to_executable_tx` is called, meaning stateful validation (fee/balance) is never reached: [4](#0-3) 

The fee/balance check only runs inside `perform_pre_validation_stage`, which is called during stateful validation — after the expensive verification has already completed or failed: [5](#0-4) 

The `validate_proof_facts` function (called at line 365) only validates cheap metadata (program hash, block hash, config hash). The expensive cryptographic work is entirely in the converter's `verify_recursive_circuit` call, which runs before any fee check. [6](#0-5) 

### Impact Explanation

An attacker submits Invoke V3 transactions with:
- Valid `proof_facts` metadata: correct `PROOF_VERSION`, an allowed `program_hash` from `allowed_virtual_os_program_hashes`, a valid `block_hash`/`block_number` pair, and a matching `config_hash`. These pass `validate_proof_facts` cheaply.
- A corrupted proof blob (e.g., a single flipped bit), up to the `max_proof_size` limit of 480,000 bytes.

The sequencer runs the full `verify_recursive_circuit` on the corrupted proof, which fails. The transaction is rejected. The sender is never charged because the fee/balance check is never reached. The attacker can repeat this at scale, consuming sequencer CPU resources for free.

This matches: **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing** — the admission path performs unbounded expensive work before the economic gate.

It also touches: **Critical. Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact** — the sequencer absorbs the cost of proof verification that should be priced into the transaction fee.

The `gas_per_proof` constant (75,000,000 L2 gas in v0.14.2) is meant to price this work, but it is only charged if the transaction reaches execution — which it never does when verification fails before the fee check. [7](#0-6) 

### Likelihood Explanation

The attack requires only a valid Starknet account address (to pass stateless signature checks) and knowledge of a recent allowed `program_hash` and a stored block hash — all publicly available on-chain. The proof blob can be any byte sequence up to 480 KB. No privileged access is needed. The attacker pays zero fees per attempt.

### Recommendation

**Short term:** Move the fee-bounds and balance checks (`check_fee_bounds`, `verify_can_pay_committed_bounds`) to run **before** `spawn_proof_verification` is called. Specifically, perform a lightweight pre-check on the sender's balance against the minimum fee implied by `gas_per_proof` before spawning the blocking verification task.

**Long term:** Ensure that any computationally expensive gateway-side operation (proof verification, Sierra compilation) is gated behind a fee/balance pre-check so that the sequencer is never forced to perform unbounded work for free.

### Proof of Concept

1. Obtain a valid `program_hash` from `allowed_virtual_os_program_hashes` in the latest versioned constants.
2. Read a stored block hash from the block-hash contract for a block number satisfying `current - STORED_BLOCK_HASH_BUFFER`.
3. Compute the correct `config_hash` via `OsChainInfo::compute_virtual_os_config_hash`.
4. Construct an Invoke V3 transaction with these valid `proof_facts` fields but a 480,000-byte proof blob with a corrupted last byte.
5. Submit to the gateway. Observe that `verify_recursive_circuit` runs (latency spike, `PROOF_VERIFICATION_LATENCY` metric increments) and the transaction is rejected with `InvalidProof` — but no fee is deducted from the sender.
6. Repeat in a tight loop. Each iteration forces the sequencer to run the full recursive circuit verifier at zero cost to the attacker. [8](#0-7)

    async fn run_proof_verification(
        proof_facts: ProofFacts,
        proof: Proof,
        proof_manager_client: SharedProofManagerClient,
    ) -> Result<bool, TransactionConverterError> {
        let contains_proof = proof_manager_client.contains_proof(proof_facts.clone()).await?;

        if contains_proof {
            return Ok(false);
        }

        let proof_facts_hash = proof_facts.hash();
        let verify_start = Instant::now();
        tokio::task::spawn_blocking(move || {
            starknet_proof_verifier::verify_proof(proof_facts, proof)
        })
        .await
        .expect("proof verification task panicked")?;
        let verify_duration = verify_start.elapsed();
        PROOF_VERIFICATION_LATENCY.record(verify_duration.as_secs_f64());
        info!(
            "Proof verification took: {verify_duration:?} for proof facts hash: \
             {proof_facts_hash:?}"
        );

        Ok(true)
    }

    fn validate_proof_facts(
        &self,
        block_context: &BlockContext,
        state: &mut dyn State,
    ) -> TransactionPreValidationResult<()> {
        // Only Invoke V3 transactions can carry proof facts.
        let Transaction::Invoke(invoke_tx) = &self.tx else {
            return Ok(());
        };
        if invoke_tx.version() < TransactionVersion::THREE {
            return Ok(());
        }

        // Parse proof facts.
        let proof_facts = invoke_tx.proof_facts();
        let snos_proof_facts = match ProofFactsVariant::try_from(&proof_facts)
            .map_err(|e| TransactionPreValidationError::InvalidProofFacts(e.to_string()))?
        {
            ProofFactsVariant::Empty => return Ok(()),
            ProofFactsVariant::Snos(snos_proof_facts) => snos_proof_facts,
        };
        let os_constants = &block_context.versioned_constants.os_constants;

        // Validate the program hash.
        let allowed = &os_constants.allowed_virtual_os_program_hashes;
        if !allowed.contains(&snos_proof_facts.program_hash) {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS program hash {} is not allowed",
                snos_proof_facts.program_hash
            )));
        }

        // Validate the block hash and block number.
        let proof_block_hash = snos_proof_facts.block_hash.0;
        let proof_block_number = snos_proof_facts.block_number.0;
        Self::validate_proof_block_number(
            proof_block_number,
            block_context.block_info.block_number,
        )?;
        Self::validate_proof_block_hash(proof_block_hash, proof_block_number, os_constants, state)?;

        // Validate the config hash.
        let chain_info = &block_context.chain_info;
        // TODO(Meshi): Cache this computation as part of the chain context.
        let virtual_os_config_hash = OsChainInfo::from(chain_info)
            .compute_virtual_os_config_hash()
            .expect("Failed to compute OS config hash");
        let proof_config_hash = snos_proof_facts.config_hash;
        if virtual_os_config_hash != proof_config_hash {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS config hash mismatch. Computed virtual OS config hash: \
                 {virtual_os_config_hash}, expected virtual OS config hash: {proof_config_hash}."
            )));
        }

        Ok(())
    }

    // Performs static checks before executing validation entry point.
    // Note that nonce is incremented during these checks.
    pub fn perform_pre_validation_stage<S: State + StateReader>(
        &self,
        state: &mut S,
        tx_context: &TransactionContext,
    ) -> TransactionPreValidationResult<()> {
        let tx_info = &tx_context.tx_info;
        Self::handle_nonce(state, tx_info, self.execution_flags.strict_nonce_check)?;

        if self.execution_flags.charge_fee {
            self.check_fee_bounds(tx_context)?;

            verify_can_pay_committed_bounds(state, tx_context).map_err(Box::new)?;
        }

        self.validate_proof_facts(&tx_context.block_context, state)?;

        Ok(())
    }

pub fn verify_proof(proof_facts: ProofFacts, proof: Proof) -> Result<(), VerifyProofError> {
    // Reject empty proof payloads before running the verifier.
    if proof.is_empty() {
        return Err(VerifyProofError::EmptyProof);
    }

    // Validate that the first element of proof facts is PROOF_VERSION.
    let expected_proof_version = PROOF_VERSION;
    let actual_first = proof_facts.0.first().copied().unwrap_or_default();
    if actual_first != expected_proof_version {
        return Err(VerifyProofError::InvalidProofVersion {
            expected: expected_proof_version,
            actual: actual_first,
        });
    }

    // Reconstruct the output preimage from proof facts and verify the proof.
    let output_preimage = reconstruct_output_preimage(&proof_facts)?;
    // TODO(Avi): Avoid cloning the proof.
    let proof_output = PrivacyProofOutput { proof: proof.0.to_vec(), output_preimage };
    verify_recursive_circuit(&proof_output)
        .map_err(|e| VerifyProofError::Verification(e.to_string()))?;

    Ok(())
