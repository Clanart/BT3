### Title
Unauthenticated `RemoteProofManagerServer` Allows Pre-Seeding Proof Cache to Bypass Cryptographic Proof Verification - (File: `crates/apollo_proof_manager/src/proof_manager.rs`, `crates/apollo_infra/src/component_server/remote_component_server.rs`)

### Summary

The `ProofManager::set_proof` endpoint is exposed over an unauthenticated HTTP/2 remote server that binds to `0.0.0.0`. Any network-reachable caller can inject an arbitrary `(ProofFacts, Proof)` pair into the proof manager's persistent store. The `run_proof_verification` helper in `TransactionConverter` skips `starknet_proof_verifier::verify_proof` whenever `contains_proof` returns `true`. An attacker who pre-seeds the store with a fake proof for a chosen `ProofFacts` value causes the gateway and consensus flows to accept a transaction carrying those proof facts without ever running the cryptographic circuit verifier.

### Finding Description

**Unprotected setter — `ProofManager::set_proof`**

`ProofManager::set_proof` stores any `(ProofFacts, Proof)` pair unconditionally (after a duplicate check) with no caller identity check:

```rust
pub async fn set_proof(&self, proof_facts: ProofFacts, proof: Proof) -> Result<(), FsProofStorageError> {
    if self.contains_proof(proof_facts.clone()).await? { return Ok(()); }
    let facts_hash = proof_facts.hash();
    self.proof_storage.set_proof(facts_hash, proof.clone()).await?;
    self.cache.insert(facts_hash, proof);
    Ok(())
}
``` [1](#0-0) 

The `ComponentRequestHandler` for `ProofManager` dispatches `SetProof` requests directly to this function with no authentication gate: [2](#0-1) 

**Unauthenticated remote server bound to `0.0.0.0`**

`RemoteComponentServer::remote_component_server_handler` deserializes any incoming HTTP body and forwards it to the local client. There is no token, TLS client certificate, IP allowlist, or any other authentication mechanism in the handler: [3](#0-2) 

The production distributed deployment config explicitly binds the proof manager's remote server to all interfaces:

```json
"components.proof_manager.remote_server_config.bind_ip": "0.0.0.0",
"components.proof_manager.remote_server_config.#is_none": false
``` [4](#0-3) 

**Verification skip in `run_proof_verification`**

`run_proof_verification` treats a `contains_proof == true` result as proof that the cryptographic verifier already ran and returns `Ok(false)` ("not verified this time, already done"):

```rust
let contains_proof = proof_manager_client.contains_proof(proof_facts.clone()).await?;
if contains_proof {
    return Ok(false);   // skips starknet_proof_verifier::verify_proof entirely
}
// ... verify_proof called only here ...
``` [5](#0-4) 

In the **gateway flow** (`spawn_proof_verification`), the spawned task returns `Ok(())` when `run_proof_verification` returns `Ok(false)`, so the gateway proceeds to accept the transaction: [6](#0-5) 

In the **consensus flow** (`spawn_verify_and_store_proof`), `verified == false` causes an early return without calling `set_proof` again, and the transaction is accepted: [7](#0-6) 

**Blockifier structural checks do not substitute for cryptographic verification**

`validate_proof_facts` in the blockifier checks that `program_hash` is in the allowed set, `block_number` is old enough, `block_hash` matches the stored on-chain value, and `config_hash` matches the chain config. These are all structural checks on the *content* of `proof_facts`; they do not verify the cryptographic proof: [8](#0-7) 

An attacker who constructs `proof_facts` referencing a real historical block (correct `block_hash`, allowed `program_hash`, correct `config_hash`) passes all blockifier checks while the circuit verifier is never invoked.

### Impact Explanation

A transaction carrying `proof_facts` that reference a real block but are backed by a fabricated proof is admitted through the gateway and consensus flows and included in a sequenced block. The invariant that every `proof_facts`-bearing transaction has been cryptographically verified against the SNOS circuit is broken. This falls under:

> **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

The accepted transaction carries unverified SNOS proof facts (`block_number`, `block_hash`, `program_hash`, `config_hash`) that feed into the proof-manager store and are propagated to any downstream consumer that relies on `contains_proof` as a verified-proof signal.

### Likelihood Explanation

The proof manager remote server is deployed with `bind_ip: 0.0.0.0` and no authentication in the production distributed configuration. Any attacker with TCP reachability to the proof manager pod/host can issue a raw HTTP/2 POST with a serialized `ProofManagerRequest::SetProof` payload. No credentials, keys, or privileged position are required beyond network access.

### Recommendation

1. **Add caller authentication to `RemoteComponentServer`**: Implement mutual TLS or a shared-secret bearer token checked in `remote_component_server_handler` before dispatching any request.
2. **Restrict `SetProof` to verified callers only**: Separate the `SetProof` endpoint from `GetProof`/`ContainsProof` and only expose `SetProof` over the local (in-process) channel, never over the remote HTTP server.
3. **Do not treat `contains_proof == true` as a verification bypass**: `run_proof_verification` should only skip the circuit verifier if the proof was stored by a trusted internal path, not if it was found in the store (which may have been written by an external caller).

### Proof of Concept

```
# 1. Construct valid-looking SnosProofFacts for a real historical block N:
#    proof_version = PROOF_VERSION
#    program_hash  = any value in allowed_virtual_os_program_hashes
#    block_number  = N  (old enough: current_block - STORED_BLOCK_HASH_BUFFER)
#    block_hash    = actual stored hash for block N (publicly readable via RPC)
#    config_hash   = compute_virtual_os_config_hash() for the chain

# 2. Send SetProof to the unauthenticated remote proof manager HTTP/2 endpoint:
POST http://<proof_manager_host>:<port>/
Content-Type: application/octet-stream
Body: bincode-serialized ProofManagerRequest::SetProof(proof_facts, fake_proof_bytes)

# 3. Submit an InvokeV3 transaction to the gateway with:
#    proof_facts = the constructed SnosProofFacts above
#    proof       = the same fake_proof_bytes

# 4. Gateway calls contains_proof(proof_facts) → true
#    → run_proof_verification returns Ok(false), skipping verify_proof
#    → blockifier validate_proof_facts passes (block_hash matches, program_hash allowed)
#    → transaction is accepted into the mempool and sequenced
```

        trace!("Received HTTP request: {http_request:?}");

                    request_id = %request_id,
                    request_type = request.request_label(),

                // Even if the current task is cancelled, the inner task will continue to run.
                // Note: this creates a new request ID for the local client.

                metrics.increment_processed();

                    Ok(response) => {
                        trace!("Local client processed request successfully: {response:?}");
                        HyperResponse::builder()
                            .status(StatusCode::OK)
                            .header(CONTENT_TYPE, APPLICATION_OCTET_STREAM)
                            .body(Full::new(Bytes::from(

                                    .expect("Response serialization should succeed"),
                            )))
                    }
                    Err(error) => {
                        panic!(

                        );
                    }
                }

            Err(error) => {
                error!("Failed to deserialize request: {error:?}");
                let server_error = ServerError::RequestDeserializationFailure(error.to_string());
                HyperResponse::builder().status(StatusCode::BAD_REQUEST).body(Full::new(
                    Bytes::from(
                        SerdeWrapper::new(server_error)
                            .wrapper_serialize()
                            .expect("Server error serialization should succeed"),
                    ),
                ))
            }
        }
        .expect("Response building should succeed");
        trace!("Built HTTP response: {http_response:?}");

        Ok(http_response)
    }

        info!(
            "Proof verification took: {verify_duration:?} for proof facts hash: \
             {proof_facts_hash:?}"
        );

        Ok(true)
    }

    fn validate_proof_facts(
        &self,
        block_context: &BlockContext,
        state: &mut dyn State,
    ) -> TransactionPreValidationResult<()> {
        // Only Invoke V3 transactions can carry proof facts.
        let Transaction::Invoke(invoke_tx) = &self.tx else {
            return Ok(());
        };
        if invoke_tx.version() < TransactionVersion::THREE {
            return Ok(());
        }

        // Parse proof facts.
        let proof_facts = invoke_tx.proof_facts();
        let snos_proof_facts = match ProofFactsVariant::try_from(&proof_facts)
            .map_err(|e| TransactionPreValidationError::InvalidProofFacts(e.to_string()))?
        {
            ProofFactsVariant::Empty => return Ok(()),
            ProofFactsVariant::Snos(snos_proof_facts) => snos_proof_facts,
        };
        let os_constants = &block_context.versioned_constants.os_constants;

        // Validate the program hash.
        let allowed = &os_constants.allowed_virtual_os_program_hashes;
        if !allowed.contains(&snos_proof_facts.program_hash) {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS program hash {} is not allowed",
                snos_proof_facts.program_hash
            )));
        }

        // Validate the block hash and block number.
        let proof_block_hash = snos_proof_facts.block_hash.0;
        let proof_block_number = snos_proof_facts.block_number.0;
        Self::validate_proof_block_number(
            proof_block_number,
            block_context.block_info.block_number,
        )?;
        Self::validate_proof_block_hash(proof_block_hash, proof_block_number, os_constants, state)?;

        // Validate the config hash.
        let chain_info = &block_context.chain_info;
        // TODO(Meshi): Cache this computation as part of the chain context.
        let virtual_os_config_hash = OsChainInfo::from(chain_info)
            .compute_virtual_os_config_hash()
            .expect("Failed to compute OS config hash");
        let proof_config_hash = snos_proof_facts.config_hash;
        if virtual_os_config_hash != proof_config_hash {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS config hash mismatch. Computed virtual OS config hash: \
                 {virtual_os_config_hash}, expected virtual OS config hash: {proof_config_hash}."
            )));
        }

        Ok(())
    }
