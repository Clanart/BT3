### Title
Integer Division Truncation in `NewDynamicFeeChecker` Enables Zero-Fee Cosmos Transaction Submission When `baseFee = 0` — (File: `ante/evm/fee_checker.go`)

---

### Summary

In `ante/evm/fee_checker.go`, the `NewDynamicFeeChecker` function derives a per-gas fee cap via integer division: `feeCap = fee / gas`. When `fee < gas`, this truncates to zero. If `baseFee = 0` (reachable when `MinGasPrice = 0` and the chain has low gas usage), the subsequent fee-sufficiency check passes trivially, `EffectiveGasPrice` resolves to zero, and `DeductFeeDecorator` deducts nothing — allowing any Cosmos transaction to be included with near-zero fees paid.

---

### Finding Description

In `NewDynamicFeeChecker` (`ante/evm/fee_checker.go`):

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))   // integer division — truncates
baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)

if feeCap.LT(baseFeeInt) {                          // 0 < 0 → false → passes
    return nil, 0, errorsmod.Wrapf(...)
}

effectivePrice := sdkmath.NewIntFromBigInt(
    types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()),
)
// EffectiveGasPrice(0, 0, MaxInt64) = min(0+MaxInt64, 0) = 0

effectiveFee := sdk.Coins{{Denom: denom, Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas))}}
// effectiveFee = 0 * gas = 0
``` [1](#0-0) 

The `DeductFeeDecorator` in `ante/evm/nativefee.go` then calls `checkDeductFee` with this zero-coin slice:

```go
if !fee.IsZero() {
    err := evmkeeper.DeductFees(dfd.bankKeeper, ctx, deductFeesFromAcc, fee)
    ...
}
``` [2](#0-1) 

Because `effectiveFee = 0`, the `!fee.IsZero()` guard is false and no fees are deducted.

The `MinGasPriceDecorator` that runs earlier short-circuits immediately when `MinGasPrice = 0`:

```go
if minGasPrice.IsZero() || simulate {
    return next(ctx, tx, simulate)
}
``` [3](#0-2) 

So when both `MinGasPrice = 0` and `baseFee = 0`, no ante decorator catches the zero-fee submission.

The `baseFee = 0` state is reachable: `CalculateBaseFee` in `x/feemarket/keeper/eip1559.go` applies `max(baseFee - delta, minGasPrice)` as the lower bound. When `MinGasPrice = 0`, the base fee can decrease to zero through sustained low gas usage. [4](#0-3) 

The `EndBlock` gas-wanted computation also uses `TruncateInt()`:

```go
gasWanted = sdkmath.LegacyMaxDec(limitedGasWanted, sdkmath.LegacyNewDec(gasUsed)).TruncateInt().Uint64()
``` [5](#0-4) 

When `MinGasMultiplier` is small and `gasWanted` is small, `limitedGasWanted` truncates to zero, and `gasWanted = gasUsed`. This accelerates the base fee's descent to zero, compounding the precondition.

---

### Impact Explanation

Once `baseFee = 0` and `MinGasPrice = 0`, any unprivileged user can submit Cosmos SDK transactions (e.g., `MsgSend`, governance votes, IBC relays) with `fee = gas - 1` and pay zero fees. The `effectiveFee` returned by `NewDynamicFeeChecker` is zero, so `DeductFeeDecorator` deducts nothing from the sender's balance. This is a direct mis-accounting of user fees: the fee collector module receives no revenue, and the sender retains funds that should have been deducted. This matches the allowed High impact: **"ante handler... bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."**

---

### Likelihood Explanation

The precondition (`baseFee = 0`, `MinGasPrice = 0`) is reachable on any Ethermint-based chain that:
1. Sets `MinGasPrice = 0` (common on testnets and some production chains), and
2. Experiences a period of low block gas usage (empty or near-empty blocks), which drives the EIP-1559 base fee to zero over time.

Once the precondition is met, exploitation requires only submitting a standard Cosmos tx with `fee = gas - 1`. No special privileges, keys, or network access are needed. The attacker can sustain this indefinitely as long as `baseFee` remains at zero.

---

### Recommendation

Replace the integer division for `feeCap` with ceiling division, or validate that the raw fee amount satisfies `fee >= baseFee * gas` before computing `feeCap`:

```go
// Option 1: ceiling division
feeCap := fee.Add(sdkmath.NewIntFromUint64(gas - 1)).Quo(sdkmath.NewIntFromUint64(gas))

// Option 2: explicit raw-fee check
if fee.LT(baseFeeInt.Mul(sdkmath.NewIntFromUint64(gas))) {
    return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, ...)
}
```

Additionally, enforce a non-zero `MinGasPrice` floor in parameter validation to prevent `baseFee` from reaching zero.

---

### Proof of Concept

1. Deploy an Ethermint chain with `MinGasPrice = 0` and `NoBaseFee = false`, London enabled.
2. Submit empty blocks until `CalculateBaseFee` drives `baseFee` to `0` (confirmed via `eth_gasPrice` or `feemarket` query).
3. Construct a Cosmos `MsgSend` tx with `gas = 21000`, `fee = 20999` (one less than gas).
4. Submit the tx. In `NewDynamicFeeChecker`:
   - `feeCap = 20999 / 21000 = 0` (integer truncation)
   - `baseFeeInt = 0`
   - `feeCap.LT(baseFeeInt)` → `0 < 0` → `false` → passes
   - `effectivePrice = EffectiveGasPrice(0, 0, MaxInt64) = min(MaxInt64, 0) = 0`
   - `effectiveFee = 0 * 21000 = 0`
5. `DeductFeeDecorator` deducts `0` coins. Tx is included. Sender balance is unchanged. [6](#0-5) [7](#0-6)
