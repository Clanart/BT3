### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts an arbitrary `class_hash` from the caller and writes it directly into `contract_state_changes` **without verifying that the class hash was ever declared** in the `contract_class_changes` registry. This is the direct StarkNet analog of M-01: just as Perennial's `Collateral` contract did not verify a product was created by the factory, the OS does not verify a class was created (declared) through the proper channel. A contract can permanently replace its own class with a non-existent hash, making itself forever non-executable and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class` (`syscall_impls.cairo`, lines 877–916), the OS reads `request.class_hash` from the syscall buffer and immediately writes it into the contract's `StateEntry` via `dict_update`:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The `class_hash` value is taken directly from `request.class_hash` (line 896) with no lookup into `contract_class_changes` to confirm the hash corresponds to a previously declared class. The code itself acknowledges this gap with an explicit TODO comment at line 898.

By contrast, `execute_declare_transaction` in `transaction_impls.cairo` (lines 816–819) enforces `prev_value=0` (class declared at most once) and calls `finalize_class_hash` to verify the Sierra pre-image before writing to `contract_class_changes`. No equivalent guard exists in `execute_replace_class`.

The same root cause exists in `execute_deploy` (`syscall_impls.cairo`, lines 512–525): `request.class_hash` is used to construct the `constructor_execution_context` and is written into `contract_state_changes` via `deploy_contract` without checking `contract_class_changes`.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` in `contract_state_changes` is set to an undeclared hash:

1. Every subsequent call to that contract will fail at class-lookup time because the hash has no entry in the class tree.
2. The contract cannot execute any entry point — including a corrective `replace_class` call — so the state is irrecoverable.
3. All tokens or assets held in the contract's storage are permanently frozen.

This matches the allowed critical impact: *Permanent freezing of funds*.

---

### Likelihood Explanation

The attack requires only that a contract (controlled by the attacker) calls the `replace_class` syscall with an arbitrary felt value as the class hash. This is a standard, unprivileged syscall available to any deployed contract. No special role, leaked key, or external dependency is needed. The attacker can:

- Deploy a contract that appears legitimate and accepts user deposits.
- After accumulating funds, call `replace_class(0xdeadbeef)` (any undeclared hash).
- The OS writes the invalid hash into state; the contract is permanently frozen.

The explicit TODO comment in production code confirms the check is known to be absent and has not yet been implemented.

---

### Recommendation

Inside `execute_replace_class`, after reading `request.class_hash`, perform a lookup in `contract_class_changes` (or a read from the class commitment tree) to assert the hash maps to a non-zero compiled class hash before writing the new `StateEntry`. Concretely:

```cairo
// Assert the class is declared (compiled_class_hash != 0).
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

Apply the same guard in `execute_deploy` before constructing `constructor_execution_context`.

---

### Proof of Concept

1. Attacker declares and deploys contract `VaultTrap` that:
   - Accepts ERC-20 deposits from users (stores balances in its storage).
   - Exposes an owner-only `freeze()` function that calls `replace_class(0x1)` — an undeclared hash.

2. Users deposit funds; `VaultTrap` accumulates token balances.

3. Attacker calls `freeze()`. The OS executes `execute_replace_class`:
   - Reads `class_hash = 0x1` from the syscall buffer.
   - Skips the (missing) declared-class check.
   - Writes `StateEntry(class_hash=0x1, ...)` into `contract_state_changes`.

4. Block is proven and state is committed. `VaultTrap`'s on-chain class hash is now `0x1`, which has no entry in the class tree.

5. Any subsequent call to `VaultTrap` (including withdrawal) fails at class resolution. User funds are permanently frozen.

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    let revert_log = &revert_log[1];

    return ();
}
