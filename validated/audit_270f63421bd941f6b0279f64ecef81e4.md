### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezal of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the supplied `class_hash` has been declared in the protocol's class registry before updating the contract's state entry. An unprivileged contract can call `replace_class` with an arbitrary, undeclared class hash, permanently bricking itself and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the requested `class_hash` directly from the syscall request and writes it into `contract_state_changes` without any check that the hash corresponds to a declared class:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-acknowledged TODO comment at line 898 explicitly flags this missing check. The analogous missing check exists in the deprecated path as well.

The `contract_class_changes` dict — which tracks declared classes — is never consulted. The OS will accept any felt value as a valid replacement class hash and commit it to the global state. [1](#0-0)

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` field in `contract_state_changes` is set to a hash that does not exist in `contract_class_changes`, every subsequent call to that contract will fail at class-lookup time. The contract becomes permanently non-executable. Any ERC-20 tokens, ETH, STRK, or other assets held in the contract's storage are irrecoverably frozen: no `transfer`, `withdraw`, or any other entry point can ever be invoked again. [3](#0-2) 

---

### Likelihood Explanation

The `replace_class` syscall is callable by any contract during normal execution — no privileged role is required. A malicious contract author can deliberately supply a bogus class hash. Additionally, a legitimate contract with a programming error (e.g., computing a class hash incorrectly before calling `replace_class`) can trigger this accidentally, mirroring the "user error" scenario described in the original report. Because the OS provides no guard, the state corruption is committed to the proven block output and is irreversible. [4](#0-3) 

---

### Recommendation

Before writing the new `class_hash` into `contract_state_changes`, verify that the hash exists in `contract_class_changes` (i.e., that it was declared in the current block or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` with `key=class_hash` and assert the returned `compiled_class_hash` is non-zero. This mirrors the check already enforced in `execute_declare_transaction`:

```cairo
// Declare the class hash.
// Note that prev_value=0 enforces that a class may be declared only once.
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [5](#0-4) 

Apply the same fix to the deprecated `execute_replace_class` in `deprecated_execute_syscalls.cairo`.

---

### Proof of Concept

1. **Deploy** a contract `Victim` that holds user funds (e.g., acts as a vault). The contract's class hash `H_real` is properly declared.
2. **Craft** a malicious class hash `H_fake = 0xdeadbeef...` that has never been declared on-chain.
3. **Invoke** `Victim.__execute__` (or any entry point) which internally calls the `replace_class(H_fake)` syscall.
4. The OS calls `execute_replace_class`:
   - `class_hash = H_fake` is read from the request.
   - No lookup in `contract_class_changes` is performed.
   - `contract_state_changes[Victim_address].class_hash` is updated to `H_fake`.
5. The block is proven and finalized. The state root now encodes `Victim` with class `H_fake`.
6. Any subsequent transaction targeting `Victim` fails at class resolution — `H_fake` has no bytecode.
7. All funds in `Victim`'s storage are permanently frozen. [6](#0-5)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
