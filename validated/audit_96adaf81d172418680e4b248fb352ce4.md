### Title
Missing Token Registration Validation Allows Permanent Irrecoverable Lock of User Funds - (`evm/src/omni-bridge/contracts/OmniBridge.sol`, `starknet/src/omni_bridge.cairo`)

### Summary
`OmniBridge.sol::initTransfer` and `omni_bridge.cairo::init_transfer` accept any arbitrary user-supplied `tokenAddress` without validating that it is a registered bridge token (i.e., has a corresponding NEAR token ID and decimals entry). Tokens deposited for an unregistered address are locked in the bridge contract with no recovery path, because the NEAR-side `fin_transfer_callback` panics on `TokenDecimalsNotFound` for any token not in `token_decimals`, making finalization permanently impossible.

### Finding Description

**EVM (`OmniBridge.sol`):**

`initTransfer` accepts any `tokenAddress` from the caller. For tokens that are neither `address(0)`, a custom-minter token, nor a bridge token, it executes:

```solidity
IERC20(tokenAddress).safeTransferFrom(msg.sender, address(this), amount);
```

There is no check that `tokenAddress` is present in `ethToNearToken` (i.e., has a registered NEAR counterpart). The tokens are transferred into the bridge contract and an `InitTransfer` event is emitted. [1](#0-0) 

**StarkNet (`omni_bridge.cairo`):**

`init_transfer` similarly accepts any `token_address`. For non-bridge tokens it calls `transfer_from` into the contract with no registration check:

```cairo
let success = IERC20Dispatcher { contract_address: token_address }
    .transfer_from(caller, get_contract_address(), amount.into());
assert(success, 'ERR_TRANSFER_FROM_FAILED');
``` [2](#0-1) 

**NEAR-side rejection:**

When the relayer attempts to finalize the transfer on NEAR via `fin_transfer_callback`, the contract immediately panics if the token has no registered decimals:

```rust
let decimals = self
    .token_decimals
    .get(&init_transfer.token)
    .near_expect(BridgeError::TokenDecimalsNotFound);
``` [3](#0-2) 

Because `fin_transfer_callback` is `#[private]` and only callable as a callback from the prover, and because `OmniBridge.sol` exposes no admin-rescue or sweep function for locked ERC-20 balances, the deposited tokens are permanently irrecoverable.

### Impact Explanation

Any user who calls `initTransfer` (EVM) or `init_transfer` (StarkNet) with an ERC-20 token address that has not been registered in the bridge (no `ethToNearToken` / `nearToEthToken` mapping, no `token_decimals` entry on NEAR) will have their tokens permanently locked in the bridge contract. There is no cancel, refund, or admin-rescue path visible in `OmniBridge.sol`. This matches the allowed critical impact: **permanent freezing / irrecoverable lock of user funds in bridge flows**. [4](#0-3) 

### Likelihood Explanation

The entry point is a public, permissionless function callable by any token holder. A user who bridges a token that was recently de-listed, or who makes a typo in the token address, or who is deliberately griefed by a front-end pointing to an unregistered address, will trigger this path. No privileged access is required. Likelihood is **medium**: the bridge UI would normally guard against this, but the on-chain contract itself provides no safety net.

### Recommendation

Add an explicit check in both `initTransfer` (EVM) and `init_transfer` (StarkNet) that the supplied token address is a registered bridge token before accepting the deposit:

**EVM:**
```solidity
require(
    tokenAddress == address(0) ||
    bytes(ethToNearToken[tokenAddress]).length > 0 ||
    customMinters[tokenAddress] != address(0),
    "ERR_TOKEN_NOT_REGISTERED"
);
```

**StarkNet:**
```cairo
assert(
    self.is_bridge_token(token_address) ||
    self.starknet_to_near_token.read(token_address).len() > 0,
    'ERR_TOKEN_NOT_REGISTERED'
);
```

Additionally, consider adding an admin-only token-rescue function for any ERC-20 tokens that are not tracked by the bridge, to recover from accidental deposits.

### Proof of Concept

1. Attacker (or user) deploys or obtains any ERC-20 token `T` that is **not** registered in the bridge (`ethToNearToken[T] == ""`).
2. Calls `OmniBridge.initTransfer(T, 1000e18, 0, 0, "victim.near", "")` with sufficient `msg.value` for `nativeFee = 0`.
3. `OmniBridge.sol` executes `IERC20(T).safeTransferFrom(msg.sender, address(this), 1000e18)` — tokens are now held by the bridge.
4. An `InitTransfer` event is emitted with `tokenAddress = T`.
5. A relayer picks up the event and calls `fin_transfer` on NEAR with the prover result.
6. `fin_transfer_callback` on NEAR executes `self.token_decimals.get(&init_transfer.token).near_expect(BridgeError::TokenDecimalsNotFound)` — **panics**, finalization permanently fails.
7. The 1000e18 tokens of `T` remain locked in `OmniBridge` with no recovery mechanism. [5](#0-4) [3](#0-2)
