### Title
`ProofOfInclusion::valid()` Never Verifies Against an External Committed Root — Forged Inclusion Proofs Always Pass - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate only checks the internal hash-chain consistency of the proof. Its final assertion `existing_hash == self.root_hash()` is a tautology — it always evaluates to `true` after the loop — because `self.root_hash()` returns the very same `combined_hash` that `existing_hash` was just set to. No external committed Merkle root is ever compared. An untrusted party can therefore construct a fully fake `ProofOfInclusion` for any arbitrary key-value pair, serialize it via the `Streamable` / Python binding interface, and have it pass `valid()` unconditionally.

### Finding Description

`ProofOfInclusion` is a `Streamable` struct exposed to Python and wasm. Its `valid()` method is the sole validation entry point:

```rust
// crates/chia-datalayer/src/merkle/proof_of_inclusion.rs  lines 40-58
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;   // ← existing_hash := layer.combined_hash
    }

    existing_hash == self.root_hash()      // ← always true (see below)
}
```

`root_hash()` is:

```rust
// lines 32-38
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← the same value existing_hash was just set to
    } else {
        self.node_hash
    }
}
```

After the loop completes without an early `return false`, `existing_hash` holds the last `calculated_hash`, which the loop already asserted equals `last.combined_hash`. `self.root_hash()` returns that same `last.combined_hash`. The final comparison is therefore `last.combined_hash == last.combined_hash` — a tautology in every case (single-layer, multi-layer, and zero-layer proofs).

The analog to the external report is direct: the Uniswap bug used the **stored initial liquidity** instead of the **current actual liquidity**; here `valid()` uses the **proof's own self-referential root** instead of the **actual committed on-chain Merkle root**. In both cases the critical check is performed against a value that is trivially equal to itself, making the guard meaningless.

Because `ProofOfInclusion` derives `Streamable` and `PyStreamable`, an attacker can:

1. Craft any `node_hash` (e.g., `SHA256("fake_key" || "fake_value")`).
2. Build one or more `ProofOfInclusionLayer` entries whose `combined_hash` values form an internally consistent chain (each `combined_hash` is the correct `internal_hash` of the previous hash and the chosen `other_hash`).
3. Serialize the struct and pass it to any DataLayer consumer that calls `proof.valid()`.
4. `valid()` returns `true` — the forged proof is accepted.

No knowledge of the actual tree contents or the committed on-chain root is required.

### Impact Explanation

The DataLayer commits a Merkle root on-chain. Downstream consumers (Python DataLayer service, wasm clients) rely on `proof.valid()` to verify that a key-value pair is genuinely included in the committed tree before acting on it. Because `valid()` never compares against the committed root, any attacker who can supply a `ProofOfInclusion` object — through the Python binding (`from_bytes`, `from_json_dict`, direct construction via `from_py_object`) or the wasm boundary — can prove inclusion of arbitrary data in any tree. This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, letting untrusted input prove invalid state**.

### Likelihood Explanation

The `ProofOfInclusion` type is fully serializable and constructible from Python/wasm without any privileged access. The `valid()` method is the only validation API; there is no `valid_for_root(committed_root)` alternative. Any DataLayer client that calls `proof.valid()` without separately comparing `proof.root_hash()` against the on-chain commitment is exploitable. Given that `valid()` is named and documented as a complete validity check, and the Python type stub exposes only `valid()` and `root_hash()` with no guidance to combine them, this pattern is highly likely to be used incorrectly.

### Recommendation

Replace the self-referential final check with a parameter-driven root comparison:

```rust
pub fn valid_for_root(&self, committed_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == committed_root   // compare against the external committed root
}
```

Deprecate or remove the current `valid()` method, or redefine it to require the committed root as an argument. Update the Python/wasm bindings accordingly so callers cannot accidentally omit the root check.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer, Side
from hashlib import sha256

# Fake leaf hash for a key-value pair that does NOT exist in any real tree
fake_node_hash = sha256(b"fake_key" + b"fake_value").digest()

# Build one internally-consistent layer with an arbitrary sibling hash
other_hash = bytes(32)  # all-zero sibling
combined = sha256(b"\x02" + fake_node_hash + other_hash).digest()  # internal_hash formula

layer = ProofOfInclusionLayer(
    other_hash_side=Side.Right,
    other_hash=other_hash,
    combined_hash=combined,
)

proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

# valid() returns True for a completely fabricated proof
assert proof.valid(), "Forged proof accepted!"
# root_hash() returns the attacker-chosen combined hash, not the real committed root
print("Forged root:", proof.root_hash().hex())
```

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;

                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
