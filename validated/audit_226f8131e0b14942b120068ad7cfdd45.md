### Title
Stale `feemarketParams` Snapshot in `NewDynamicFeeChecker` Allows Cosmos Transactions to Bypass Current EIP-1559 Base Fee Enforcement - (File: `ante/evm/fee_checker.go`)

---

### Summary

`newCosmosAnteHandler` captures `feemarketParams` once at ante handler construction time (app startup) and passes a pointer to `NewDynamicFeeChecker`. The closure uses the stale `feemarketParams.BaseFee` for all subsequent Cosmos SDK transaction fee validation, never re-reading the live base fee from the KV store. As the EIP-1559 base fee evolves block-by-block, the stale snapshot diverges from the live on-chain base fee, allowing Cosmos transactions to be accepted with fees below the current base fee (underpayment) or causing legitimate transactions to be incorrectly rejected (false rejection).

---

### Finding Description

In `evmd/ante/handler_options.go`, `newCosmosAnteHandler` reads `feemarketParams` once from the store at construction time and passes a pointer to `NewDynamicFeeChecker`:

```go
feemarketParams := options.FeeMarketKeeper.GetParams(ctx)
...
txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
``` [1](#0-0) 

The pointer `&feemarketParams` points to a local variable that is never updated after construction. The `NewDynamicFeeChecker` closure captures this pointer and uses `feemarketParams.BaseFee` via `types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)` for all subsequent fee validation:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
...
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
baseFeeInt := sdkmath.NewIntFromBigInt(baseFee) // stale baseFee

if feeCap.LT(baseFeeInt) {
    return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, ...)
}
``` [2](#0-1) 

The effective fee deducted is also computed using the stale base fee:

```go
effectivePrice := sdkmath.NewIntFromBigInt(types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
effectiveFee := sdk.Coins{{Denom: denom, Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas))}}
``` [3](#0-2) 

In contrast, the EVM ante handler (`newEthAnteHandler`) correctly reads live params on every transaction via `EVMBlockConfig`:

```go
blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
...
feemarketParams := &blockCfg.FeeMarketParams
baseFee := blockCfg.BaseFee
``` [4](#0-3) 

The base fee is updated every `BeginBlock` via `k.SetBaseFee(ctx, baseFee)`: [5](#0-4) 

After app startup, the live base fee diverges from the stale snapshot. The same stale-capture pattern also exists in `newLegacyCosmosAnteHandlerEip712`: [6](#0-5) 

Additionally, `MinGasPriceDecorator` is constructed with the same stale pointer and uses `mpd.feemarketParams.MinGasPrice` rather than querying the live keeper: [7](#0-6) 

---

### Impact Explanation

**High.** This is a fee market ante handler bug that permits invalid transactions to commit with mis-accounted fees, matching the allowed impact: *"fee market, ante handler, or proposal handling bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

**Underpayment (base fee increased):** If the base fee has risen since app startup due to sustained high block utilization, Cosmos SDK transactions with fee-per-gas between the stale base fee and the current base fee are incorrectly accepted. The effective fee deducted is computed using the stale (lower) base fee, resulting in systematic underpayment. The EIP-1559 fee market mechanism is bypassed for all Cosmos SDK transactions (governance, staking, IBC, etc.) on chains with `DynamicFeeChecker = true`.

**False rejection (base fee decreased):** If the base fee has fallen since app startup, legitimate Cosmos SDK transactions with fee-per-gas above the current base fee but below the stale base fee are incorrectly rejected, constituting a DoS for legitimate users.

---

### Likelihood Explanation

High. The base fee changes every block based on block utilization per `CalculateBaseFee`: [8](#0-7) 

With the default `BaseFeeChangeDenominator = 8`, the base fee can shift by up to 12.5% per block. After app startup, the stale snapshot diverges monotonically. Any user submitting a Cosmos SDK transaction on a chain with `DynamicFeeChecker = true` and an active EIP-1559 fee market is affected. No special privileges are required — any unprivileged user can submit a Cosmos SDK transaction.

---

### Recommendation

Pass the `FeeMarketKeeper` into `NewDynamicFeeChecker` and call `GetParams(ctx)` inside the returned closure to obtain live params on every invocation, mirroring the pattern used in `newEthAnteHandler` via `EVMBlockConfig`. The stale pointer capture of `&feemarketParams` and `&evmParams` must be replaced with live store reads inside the closure body.

---

### Proof of Concept

1. App starts; `newCosmosAnteHandler` captures `feemarketParams.BaseFee = 1,000 gwei` into the `NewDynamicFeeChecker` closure.
2. Network sustains high block utilization for N blocks; `BeginBlock` updates the live base fee to `2,000 gwei` in the KV store.
3. Attacker submits a Cosmos SDK transaction (e.g., `MsgDelegate`) with `fee-per-gas = 1,500 gwei` (between stale 1,000 and current 2,000).
4. `NewDynamicFeeChecker` evaluates: `feeCap (1,500) >= stale baseFee (1,000)` → **accepted**.
5. Correct evaluation should be: `feeCap (1,500) < current baseFee (2,000)` → **should be rejected**.
6. The effective fee deducted is `min(1,000 + tip, 1,500) * gas` instead of the required `2,000 * gas`.
7. The attacker pays ~25% less than the current base fee requires; the fee market is bypassed for all Cosmos SDK transactions.

This is directly analogous to the external report: a gas price condition (base fee spike) causes the fee enforcement mechanism to fail, allowing transactions to be processed at incorrect prices — except here the root cause is a stale parameter snapshot rather than a hard-coded gas price check.

func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
}
