The vulnerability is real. The `pay` function in `PeripheryPayments.sol` uses `address(this).balance` unconditionally when `token == WETH` and `nativeBalance > 0`, without any check that the native ETH belongs to the current caller. ETH can be stranded in the router between transactions via any `payable` entry point (`multicall`, `exactInputSingle`, `exactOutputSingle`, etc.) when a user sends excess ETH and omits `refundETH()`. A subsequent WETH swap by any caller silently consumes that stranded ETH.

### Title
Stranded ETH in Router Silently Consumed by Subsequent WETH Swaps — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

### Summary
`PeripheryPayments.pay` uses the router's entire native ETH balance as a subsidy for any WETH payment, regardless of which user deposited that ETH. ETH left in the router by a prior user (e.g., via `multicall{value: X}` without a trailing `refundETH()`) is silently wrapped and forwarded to the pool on behalf of the next WETH-paying caller, who has their `safeTransferFrom` pull reduced by exactly that amount.

### Finding Description

The hybrid WETH branch of `pay` reads `address(this).balance` and, when `0 < nativeBalance < value`, wraps and transfers the full native balance first, then pulls only the shortfall from `payer`: [1](#0-0) 

There is no check that `nativeBalance` was contributed by the current `payer`. Any ETH sitting in the router — from any prior transaction — is treated as belonging to the current swap.

ETH enters the router legitimately via any `payable` entry point: [2](#0-1) 

The `receive()` guard only blocks *direct* ETH pushes; it does not prevent ETH from being forwarded via `multicall{value: X}` or `exactInputSingle{value: X}` where `X > amountIn`: [3](#0-2) 

If user A calls `multicall{value: 1 ETH}([exactInputSingle(WETH, 0.5 ETH)])` without appending `refundETH()`, 0.5 ETH remains in the router after the transaction. User B then calls `exactInputSingle(WETH, 1 ETH)` with `msg.value = 0`. Inside `pay(WETH, userB, pool, 1e18)`:

- `nativeBalance = 0.5 ETH` (user A's residual)
- Router wraps and sends 0.5 ETH to the pool
- Router pulls only `0.5 ETH` worth of WETH from user B

User A's 0.5 ETH is permanently consumed; user B's WETH cost is halved.

### Impact Explanation

Direct loss of user principal. User A loses their stranded ETH with no recourse; it is irreversibly wrapped and delivered to the pool as part of user B's payment. User B (the attacker) profits by paying less WETH for the same swap output. Impact is **High**: the loss equals the full stranded ETH amount, which can be arbitrarily large.

### Likelihood Explanation

**Medium.** The precondition — ETH stranded in the router — arises naturally whenever a user sends excess ETH without `refundETH()`, a common mistake in the Uniswap v3 multicall pattern. An attacker can monitor the mempool or chain state for a non-zero `address(router).balance` and immediately issue a WETH swap to drain it. No privileged access is required.

### Recommendation

Track only the ETH contributed by the current `msg.value` (i.e., `msg.value` at entry, decremented as it is consumed), rather than reading `address(this).balance` at payment time. Alternatively, record the router's ETH balance at the start of each top-level call and use only that delta as the available native subsidy, refusing to consume any pre-existing balance.

### Proof of Concept

```
// 1. User A sends excess ETH via multicall, forgets refundETH
router.multicall{value: 1 ether}([
    abi.encodeCall(router.exactInputSingle, (ExactInputSingleParams({
        tokenIn: WETH, amountIn: 0.5 ether, ...
    })))
    // no refundETH call
]);
// router.balance == 0.5 ether (user A's ETH stranded)

// 2. User B (attacker) calls a WETH swap with msg.value = 0
router.exactInputSingle(ExactInputSingleParams({
    tokenIn: WETH, amountIn: 1 ether, ...
}));
// pay(WETH, userB, pool, 1e18):
//   nativeBalance = 0.5e18 > 0
//   wraps 0.5 ETH (user A's) → sends to pool
//   safeTransferFrom(userB, pool, 0.5e18)  ← only half pulled from B
// assert: user A lost 0.5 ETH; user B paid 0.5 ETH less WETH
```
