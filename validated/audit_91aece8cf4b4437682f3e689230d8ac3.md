### Title
Missing Declared-Class Validation in `replace_class` Syscall Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS directly overwrites a contract's `class_hash` field in `contract_state_changes` with an attacker-supplied value, without verifying that the new class hash corresponds to a previously declared class. An explicit `TODO` in the code acknowledges this missing check. This is structurally analogous to the ERC-20 `approve` direct-overwrite pattern: a critical mapping value is replaced without validating the new value's legitimacy, enabling a state transition that permanently bricks a contract and freezes any funds it holds.

---

### Finding Description

In `execute_replace_class`:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The `class_hash` value comes directly from `request.class_hash`, which is caller-controlled. The OS performs **no cross-check** against `contract_class_changes` (the dict that tracks declared class hashes). The `dict_update` call unconditionally commits the new `StateEntry` with the arbitrary class hash into the global state. [1](#0-0) 

Compare this to the declare path, which correctly enforces `prev_value=0` to prevent re-declaration and validates the class hash pre-image: [2](#0-1) 

No equivalent forward-validation exists in `execute_replace_class`. The `contract_class_changes` dict is never consulted.

The `StateEntry` struct stores `class_hash` as a raw `felt`: [3](#0-2) 

When `compute_contract_state_commitment` later hashes this entry into the Patricia tree, it uses whatever `class_hash` is stored — including an undeclared one — producing a valid-looking state root that references a non-existent class: [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` is set to an undeclared value:

1. The state commitment is accepted by the OS (the class hash is just a `felt` in the state tree; no cross-tree validation occurs at commitment time).
2. Any subsequent call to the contract will fail at class lookup, because the class does not exist in the class tree.
3. The contract's storage — including any token balances or ETH/STRK held — becomes permanently inaccessible. There is no recovery path: the contract cannot be called to withdraw funds, and the OS has no mechanism to undo a committed state root.

---

### Likelihood Explanation

**Medium-High.**

- The `replace_class` syscall is available to any contract without privilege restrictions.
- The missing check is explicitly acknowledged in the production code with a `TODO` comment, confirming the developers are aware it is absent.
- Realistic attack vectors include: (a) a proxy/upgradeable contract whose upgrade function accepts a caller-supplied class hash — an attacker who can invoke the upgrade function passes an undeclared hash; (b) a reentrancy scenario where an attacker triggers `replace_class` mid-execution on a contract holding funds; (c) a buggy contract that accidentally passes an invalid class hash.
- No privileged role or leaked key is required — only the ability to trigger a `replace_class` syscall from within a contract execution.

---

### Recommendation

Inside `execute_replace_class`, after reading `request.class_hash`, perform a `dict_read` on `contract_class_changes` keyed by the new class hash and assert that the returned compiled class hash is non-zero (i.e., the class has been declared). This mirrors the existing pattern used in `execute_declare_transaction` where `prev_value=0` enforces declaration uniqueness. The check should be:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This ensures the OS rejects any `replace_class` call that references an undeclared class, preventing the permanent-freeze state transition from being committed.

---

### Proof of Concept

1. Declare class `A` (valid, compiled class hash `C_A`). Deploy contract `Victim` with class `A`, holding 1000 STRK in storage.
2. Craft class `B_fake = 0xdeadbeef...` — never declared; `contract_class_changes` has no entry for it.
3. From within `Victim`'s execution (e.g., via a public `upgrade(class_hash)` entry point), issue the `replace_class` syscall with `class_hash = B_fake`.
4. `execute_replace_class` reads `request.class_hash = B_fake`, skips any class-tree lookup (the TODO is unimplemented), and calls `dict_update` writing `StateEntry(class_hash=B_fake, ...)` into `contract_state_changes`.
5. The OS proceeds to `compute_contract_state_commitment`, which hashes `B_fake` into the Patricia tree as the new leaf for `Victim`'s address. The proof is generated and accepted.
6. In the next block, any call to `Victim` fails: the OS looks up `B_fake` in the class tree, finds no entry, and the execution aborts.
7. The 1000 STRK in `Victim`'s storage is permanently frozen — no entry point can be reached to withdraw it. [5](#0-4)

    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;
