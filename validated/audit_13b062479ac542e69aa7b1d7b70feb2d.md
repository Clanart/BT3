### Title
`execute_replace_class` Does Not Verify the New Class Hash Is Declared Before Updating Contract State — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS processes the `replace_class` syscall by writing an arbitrary caller-supplied class hash directly into `contract_state_changes`, without verifying that the new class hash corresponds to a previously declared class in `contract_class_changes`. This is structurally impossible given the function's implicit arguments — `contract_class_changes` is not in scope. The omission is explicitly acknowledged by a TODO comment in the code. A contract that calls `replace_class` with an undeclared class hash will have its class hash permanently set to a non-existent class, making all future calls to that contract fail and permanently freezing any funds it holds.

---

### Finding Description

In `execute_replace_class`, the new class hash is read directly from the syscall request and written unconditionally into `contract_state_changes`:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The function's implicit argument list is:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt)
``` [2](#0-1) 

`contract_class_changes` is absent from the implicit arguments. This means the OS cannot — and does not — cross-check whether `request.class_hash` has a corresponding entry in the class tree. Any felt value, including one that was never declared, is accepted as the new class hash.

By contrast, the `execute_declare_transaction` path does write to `contract_class_changes` with `prev_value=0` to enforce that a class is declared exactly once:

```cairo
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [3](#0-2) 

There is no symmetric read-check in `execute_replace_class` to confirm the target class hash exists in that tree.

The state commitment logic in `commitment.cairo` hashes whatever `class_hash` is stored in `StateEntry` into the global state root without any validity check:

```cairo
let (new_value) = get_contract_state_hash(
    class_hash=new_state.class_hash,
    storage_root=final_contract_state_root,
    nonce=new_state.nonce,
);
``` [4](#0-3) 

This means an undeclared class hash is committed to L1 as a valid state root, with no on-chain mechanism to detect or reject it.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value and committed to the global state root on L1:

1. Every subsequent call to the contract will fail at class resolution time, because the class does not exist in the class tree.
2. The contract's storage and balance remain in the state trie but are permanently inaccessible.
3. There is no recovery path: `replace_class` itself requires a successful call to the contract, which is now impossible.

Any ERC-20 balances, ETH, or other assets held by the bricked contract are permanently frozen.

---

### Likelihood Explanation

**Medium.**

The attack requires a contract to call `replace_class` with an undeclared hash. This is reachable via two realistic paths:

1. **Malicious rug-pull contract**: An attacker deploys a contract that appears legitimate (e.g., a vault or token), attracts user deposits, then calls `replace_class` with an arbitrary undeclared hash to permanently brick the contract and freeze all deposited funds. The OS provides no protection against this.

2. **Vulnerable contract with attacker-controlled argument**: A contract with a logic bug that allows an external caller to supply the class hash argument to an internal `replace_class` call. The OS's missing check means the bug's impact is maximized — a valid declared hash would at least leave the contract functional.

The OS is the last line of defense. If it enforced that the new class hash is declared, both scenarios would result in a revert rather than a permanent freeze.

---

### Recommendation

Pass `contract_class_changes` as an implicit argument to `execute_replace_class` and add a read-check before updating the contract state:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
+   contract_class_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

+   // Verify the class hash is declared.
+   let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
+   assert_not_zero(compiled_class_hash);

    local state_entry: StateEntry*;
    ...
}
```

This mirrors the invariant already enforced in `execute_declare_transaction` (where `prev_value=0` ensures a class is declared exactly once) and closes the gap at the `replace_class` syscall level.

### Proof of Concept

1. Attacker deploys `VaultContract` with a `drain()` function that calls `replace_class(undeclared_hash)` where `undeclared_hash` is any felt not present in `contract_class_changes`.
2. Users deposit funds into `VaultContract`.
3. Attacker calls `drain()`. The OS executes `execute_replace_class`, reads `request.class_hash = undeclared_hash`, and writes it to `contract_state_changes` without any check against `contract_class_changes`.
4. The block is proven. The state diff — including `VaultContract.class_hash = undeclared_hash` — is committed to L1 as part of the valid global state root.
5. All subsequent calls to `VaultContract` fail at class resolution. User funds are permanently frozen with no recovery path.

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
