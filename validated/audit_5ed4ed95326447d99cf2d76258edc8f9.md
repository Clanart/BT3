### Title
Native ETH Push-Transfer in `finTransfer` Permanently Freezes User Funds When Recipient Cannot Receive ETH — (`evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.finTransfer` uses a push pattern to deliver native ETH to the recipient. If the recipient is a contract without a `receive()` or `fallback()` function, the low-level call fails, the entire transaction reverts (including the nonce-used flag), and the transfer can never be finalized. Because the originating NEAR tokens are already burned/locked with no cross-chain refund path, the user's funds are permanently frozen.

---

### Finding Description

In `OmniBridge.finTransfer`, the destination nonce is marked used at line 287 **before** the asset delivery:

```solidity
completedTransfers[payload.destinationNonce] = true;   // line 287
```

For native ETH transfers (`payload.tokenAddress == address(0)`), the contract then pushes ETH directly to the recipient:

```solidity
(bool success, ) = payload.recipient.call{value: payload.amount}("");
if (!success) revert FailedToSendEther();              // line 322
```

If `payload.recipient` is a contract that cannot receive ETH (no `receive()` / `fallback()`), the call returns `success = false`, and `FailedToSendEther()` is thrown. Solidity reverts **all** state changes, so `completedTransfers[payload.destinationNonce]` is rolled back to `false`. The nonce is never permanently consumed, and every subsequent attempt to finalize the same transfer will hit the same revert. There is no alternative delivery path and no cross-chain refund mechanism from EVM back to NEAR.

The `recipient` field is embedded in the MPC-signed payload and cannot be changed without a new signature over a different message, which the MPC will not produce for an already-initiated transfer.

---

### Impact Explanation

**Critical — Permanent irrecoverable lock of user funds.**

1. User burns/locks NEAR tokens on the NEAR side by calling `init_transfer`.
2. MPC signs the transfer message (including the non-ETH-receivable `recipient`).
3. Every call to `finTransfer` on EVM reverts; the nonce is never consumed.
4. No admin rescue function exists in `OmniBridge.sol` to redirect the delivery.
5. No cross-chain refund path exists from EVM back to NEAR.

The user permanently loses the bridged value.

---

### Likelihood Explanation

**Medium.** Smart-contract wallets (e.g., Gnosis Safe without a fallback handler, custom multisigs, protocol treasuries) are common EVM recipients. A user who controls such a contract on EVM and initiates a native-ETH bridge from NEAR will trigger this path. No privileged access is required; the attacker-controlled input is the `recipient` field supplied during `init_transfer` on NEAR.

---

### Recommendation

Replace the push pattern with a pull (withdrawal) pattern for native ETH:

1. On a failed ETH push, record the claimable amount in a `pendingWithdrawals[recipient]` mapping instead of reverting.
2. Expose a `withdraw()` function that lets the recipient pull their ETH.
3. Alternatively, wrap native ETH as WETH before delivery so that `safeTransfer` is used and the nonce is consumed regardless of the recipient's ETH-receive capability.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

// A contract with no receive() or fallback() — cannot accept ETH
contract NoReceive {}

contract PoC {
    OmniBridge bridge = OmniBridge(<deployed_address>);

    function attack() external payable {
        NoReceive noReceive = new NoReceive();

        // Attacker already burned NEAR tokens on NEAR side,
        // specifying address(noReceive) as the EVM recipient.
        // MPC signed the payload. Relayer now calls finTransfer:

        BridgeTypes.TransferMessagePayload memory payload = BridgeTypes.TransferMessagePayload({
            destinationNonce: 1,
            originChain: 1,          // NEAR
            originNonce: 42,
            tokenAddress: address(0), // native ETH
            amount: 1 ether,
            recipient: address(noReceive), // cannot receive ETH
            feeRecipient: "",
            message: ""
        });

        // This call always reverts with FailedToSendEther().
        // completedTransfers[1] is rolled back each time.
        // The transfer can never be finalized.
        // NEAR tokens are permanently lost.
        bridge.finTransfer{value: 1 ether}(validSignature, payload);
    }
}
```
