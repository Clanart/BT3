### Title
Integer Division Truncation in `NewDynamicFeeChecker` Causes Systematic Fee Under-Collection — (File: `ante/evm/fee_checker.go`)

---

### Summary

`NewDynamicFeeChecker` derives the per-gas fee cap by dividing the total fee by gas using integer (truncating) division, then reconstructs the effective fee by multiplying back. This division-before-multiplication pattern is the direct analog of the external report's vulnerability: the truncation permanently discards up to `gas − 1` tokens of fee per transaction, causing the ante handler to deduct less than the user actually offered and systematically under-collecting fees into the fee collector.

---

### Finding Description

In `ante/evm/fee_checker.go`, `NewDynamicFeeChecker` computes:

```go
// line 83 — integer division, truncates floor(fee / gas)
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
```

`sdkmath.Int.Quo` is integer (truncating) division. `feeCap` is therefore `⌊fee / gas⌋`, which is strictly less than `fee / gas` whenever `fee mod gas ≠ 0`.

`feeCap` is then fed into `EffectiveGasPrice`:

```go
// line 91
effectivePrice := sdkmath.NewIntFromBigInt(
    types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
```

and the effective fee is reconstructed by multiplying back:

```go
// lines 94-99
effectiveFee := sdk.Coins{{
    Denom:  denom,
    Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas)),
}}
```

Because `effectivePrice ≤ feeCap = ⌊fee / gas⌋`, we have:

```
effectiveFee = effectivePrice × gas ≤ ⌊fee / gas⌋ × gas ≤ fee
```

The shortfall is `fee mod gas`, which is in the range `[0, gas − 1]`.

This `effectiveFee` is what `DeductFeeDecorator` actually deducts from the user's account:

```go
// ante/evm/nativefee.go line 59
fee, priority, err = dfd.txFeeChecker(ctx, tx)
// ...
// line 111
err := evmkeeper.DeductFees(dfd.bankKeeper, ctx, deductFeesFromAcc, fee)
```

The remainder `fee mod gas` is never deducted — it stays in the sender's account. The fee collector receives `effectiveFee < fee`.

The `DeductFeeDecorator` does guard against `gas = 0` before calling the checker (line 48–50 of `nativefee.go`), so division-by-zero is not reachable. The truncation bug is present for every non-zero gas value where `fee mod gas ≠ 0`.

---

### Impact Explanation

This matches the allowed High impact: **"ante handler bug that permits valid user funds/fees to be mis-accounted."**

The fee collector (and therefore validators/stakers) systematically receives fewer fees than the transaction declared. The shortfall per transaction is up to `gas − 1` tokens in the EVM denomination. For a Cosmos tx with a gas limit of 1,000,000 (well within typical block gas limits), up to 999,999 base-denomination units are silently retained by the sender on every transaction. Across many transactions this constitutes a persistent, exploitable drain on protocol fee revenue. A sender can deliberately set `fee = baseFee × gas + (gas − 1)` to pass the `feeCap ≥ baseFee` check while maximising the retained amount.

---

### Likelihood Explanation

The path is reachable on every Cosmos (non-EVM) transaction when `DynamicFeeChecker` is enabled and the London hardfork is active — both are the default production configuration. No special privilege is required; any unprivileged user submitting a Cosmos tx with a fee amount that is not an exact multiple of the gas limit triggers the truncation. The condition `fee mod gas ≠ 0` is trivially satisfied by any odd fee amount.

---

### Recommendation

Reorder operations so that multiplication precedes division, mirroring the fix in the external report:

```diff
-feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
-baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)
-
-if feeCap.LT(baseFeeInt) {
-    return nil, 0, errorsmod.Wrapf(...)
-}
-
-effectivePrice := sdkmath.NewIntFromBigInt(
-    types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
-
-effectiveFee := sdk.Coins{{
-    Denom:  denom,
-    Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas)),
-}}
+// Use ceiling division so feeCap = ⌈fee / gas⌉, preserving the full fee
+gasInt := sdkmath.NewIntFromUint64(gas)
+feeCap := fee.Add(gasInt.Sub(sdkmath.OneInt())).Quo(gasInt)   // ceiling
+baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)
+
+if feeCap.LT(baseFeeInt) {
+    return nil, 0, errorsmod.Wrapf(...)
+}
+
+effectivePrice := sdkmath.NewIntFromBigInt(
+    types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
+
+// Deduct the full declared fee (capped at effectivePrice × gas)
+effectiveFee := sdk.Coins{{
+    Denom:  denom,
+    Amount: effectivePrice.Mul(gasInt),
+}}
```

Alternatively, cap `effectiveFee` at the original `fee` to ensure the full declared amount is always collected.

### Proof of Concept

**Parameters:** `baseFee = 100`, `gas = 3`, `maxPriorityPrice = MaxInt64`

**Case — fee not a multiple of gas:**
```
fee    = 302   (user declares 302 tokens)
feeCap = 302 / 3 = 100   (truncated; actual ratio is 100.666…)
effectivePrice = min(baseFee + MaxInt64, feeCap) = 100
effectiveFee   = 100 × 3 = 300
```
`DeductFeeDecorator` deducts **300**, not 302. The sender retains **2** tokens that should have gone to the fee collector.

**Maximised shortfall with gas = 1,000,000:**
```
fee    = baseFee × 1_000_000 + 999_999
feeCap = baseFee   (truncated)
effectiveFee = baseFee × 1_000_000
shortfall    = 999_999 tokens per transaction

func (dfd DeductFeeDecorator) checkDeductFee(ctx sdk.Context, sdkTx sdk.Tx, fee sdk.Coins) error {
	feeTx, ok := sdkTx.(sdk.FeeTx)
	if !ok {
		return errorsmod.Wrap(sdkerrors.ErrTxDecode, "Tx must be a FeeTx")
	}

	if addr := dfd.accountKeeper.GetModuleAddress(types.FeeCollectorName); addr == nil {
		return fmt.Errorf("fee collector module account (%s) has not been set", types.FeeCollectorName)
	}

	feePayer := feeTx.FeePayer()
	feeGranter := feeTx.FeeGranter()
	deductFeesFrom := feePayer

	// if feegranter set deduct fee from feegranter account.
	// this works with only when feegrant enabled.
	if feeGranter != nil {
		feeGranterAddr := sdk.AccAddress(feeGranter)

		if dfd.feegrantKeeper == nil {
			return sdkerrors.ErrInvalidRequest.Wrap("fee grants are not enabled")
		} else if !bytes.Equal(feeGranterAddr, feePayer) {
			err := dfd.feegrantKeeper.UseGrantedFees(ctx, feeGranterAddr, feePayer, fee, sdkTx.GetMsgs())
			if err != nil {
				return errorsmod.Wrapf(err, "%s does not allow to pay fees for %s", feeGranter, feePayer)
			}
		}

		deductFeesFrom = feeGranterAddr
	}

	deductFeesFromAcc := dfd.accountKeeper.GetAccount(ctx, deductFeesFrom)
	if deductFeesFromAcc == nil {
		return sdkerrors.ErrUnknownAddress.Wrapf("fee payer address: %s does not exist", deductFeesFrom)
	}

	// deduct the fees
	if !fee.IsZero() {
		err := evmkeeper.DeductFees(dfd.bankKeeper, ctx, deductFeesFromAcc, fee)
		if err != nil {
			return err
		}
	}
