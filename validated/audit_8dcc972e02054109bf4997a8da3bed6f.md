### Title
Stale `feemarketParams` Snapshot in Cosmos Ante Handler Bypasses Dynamic Fee Market Enforcement - (File: `evmd/ante/handler_options.go`)

### Summary
`newCosmosAnteHandler` captures a one-time snapshot of `feemarketParams` at construction time and passes a pointer to that snapshot into `NewDynamicFeeChecker` and `NewMinGasPriceDecorator`. Because the EIP-1559 base fee is updated every block via `BeginBlock → SetBaseFee`, the snapshot diverges from the live KV-store value immediately after the first block. Every Cosmos-native transaction is then validated against the stale (genesis-time) base fee rather than the current one, allowing any unprivileged user to submit Cosmos transactions whose fee cap is below the real current base fee.

### Finding Description

**Construction-time snapshot — `newCosmosAnteHandler`**

```go

    ...

    ...

```

Both decorators receive `&feemarketParams` — a pointer to a local variable that is fixed at the moment `newCosmosAnteHandler` is called (once, at app startup via `setAnteHandler`).

**How the stale value is consumed — `NewDynamicFeeChecker`**

```go
// ante/evm/fee_checker.go  line 56
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
```

`GetBaseFee` reads `feemarketParams.BaseFee` directly from the captured struct:

```go
// x/evm/types/utils.go  lines 244-254

    ...
    baseFee := feemarketParams.GetBaseFee()   // reads the stale snapshot field
    ...

```

The live base fee is updated every block:

```go
// x/feemarket/keeper/abci.go  lines 30-51
func (k *Keeper) BeginBlock(ctx sdk.Context) error {
    baseFee := k.CalculateBaseFee(ctx)
    ...
    k.SetBaseFee(ctx, baseFee)   // writes new value to KV store / params
```

`SetBaseFee` writes back into the KV-store params, but the pointer held by the Cosmos ante handler still points to the old struct value — it is never updated.

**Contrast with the EVM ante handler — fresh per-tx**

```go

        ...
        baseFee := blockCfg.BaseFee   // live value read from KV store on every tx
```

EVM transactions always see the current base fee; Cosmos-native transactions always see the genesis-time base fee.

**`MinGasPriceDecorator` — same stale pointer**

```go
// ante/cosmos/min_gas_price.go  line 54
minGasPrice := mpd.feemarketParams.MinGasPrice   // stale snapshot
```

If `MinGasPrice` is raised via governance, Cosmos transactions continue to be accepted at the old (lower) floor.

### Impact Explanation

Any unprivileged user submitting a Cosmos-native transaction (governance votes, IBC relays, staking messages, etc.) can set a `feeCap` that satisfies the stale genesis-time base fee but is below the real current base fee. The `NewDynamicFeeChecker` will accept the transaction, deduct the lower fee, and the transaction will be committed. This is a fee market ante handler bug that permits invalid transactions to commit and causes valid fee revenue to be mis-accounted — matching the **High** allowed impact.

### Likelihood Explanation

The base fee changes every block automatically. After even a modest period of above-target gas usage the live base fee will exceed the genesis value. Any user who inspects the genesis params can craft Cosmos transactions that underpay fees indefinitely without any special privilege, governance action, or key compromise.

### Recommendation

Replace the construction-time snapshot with a live read inside the returned handler closure, mirroring the pattern used by `newEthAnteHandler`:

```go
func newCosmosAnteHandler(options HandlerOptions, ...) sdk.AnteHandler {
    return func(ctx sdk.Context, tx sdk.Tx, simulate bool) (sdk.Context, error) {
        evmParams       := options.EvmKeeper.GetParams(ctx)
        feemarketParams := options.FeeMarketKeeper.GetParams(ctx)
        // build decorators with fresh params and execute
    }
}
```

Alternatively, pass the `FeeMarketKeeper` directly into `NewDynamicFeeChecker` and `MinGasPriceDecorator` so they can call `GetParams(ctx)` / `GetBaseFee(ctx)` on each invocation, as the EVM path already does via `EVMBlockConfig`.

### Proof of Concept

1. Chain starts; genesis base fee = 1 Gwei (`params.InitialBaseFee`). `newCosmosAnteHandler` is called once; `feemarketParams.BaseFee = 1 Gwei` is captured.
2. Network experiences sustained high gas usage. After N blocks `BeginBlock → CalculateBaseFee → SetBaseFee` raises the live base fee to, say, 10 Gwei.
3. Attacker submits a Cosmos-native `MsgDelegate` with `feeCap = 2 Gwei/gas`.
4. `NewDynamicFeeChecker` evaluates `feeCap (2) >= baseFee (1, stale)` → **passes**.
5. `authante.NewDeductFeeDecorator` deducts only `2 Gwei × gasLimit` from the sender.
6. Transaction is committed despite paying 5× less than the current fee market requires.
7. An EVM transaction with the same `feeCap = 2 Gwei` would be **rejected** by `newEthAnteHandler` because it reads the live base fee of 10 Gwei. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
	if !IsLondon(ethCfg, height) {
		return nil
	}
	baseFee := feemarketParams.GetBaseFee()
	// should not be nil if london hardfork enabled
	if baseFee == nil {
		return new(big.Int)
	}
	return baseFee
}

func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
