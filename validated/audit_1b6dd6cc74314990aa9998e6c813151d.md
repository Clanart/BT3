### Title
Unvalidated `l1_gas_price_wei` / `l1_data_gas_price_wei` in `ProposalInit` Allows Proposer to Commit Arbitrary ETH Gas Prices to Block Header — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_block_info_valid` fetches both FRI and WEI L1 gas prices from the local oracle but only validates the FRI prices against the proposer's `ProposalInit`. The WEI prices returned by the oracle are silently discarded (`_l1_gas_prices_wei`), while the proposer-supplied `l1_gas_price_wei` and `l1_data_gas_price_wei` are accepted without any cross-check. These unvalidated WEI values are then used verbatim to construct the `BlockInfo` passed to the batcher, setting `eth_gas_prices` for the entire block. A malicious proposer can therefore commit an arbitrarily wrong ETH gas price to the block header and cause incorrect ETH-denominated fee calculations for every L1-handler transaction in that block.

---

### Finding Description

**Root cause — oracle WEI prices are fetched but thrown away**

In `is_block_info_valid` the oracle is queried for both FRI and WEI prices:

```rust
let (l1_gas_prices_fri, _l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
    l1_gas_price_provider,
    init_proposed.timestamp,
    block_info_validation.previous_block_info.as_ref(),
    gas_price_params,
)
.await;
``` [1](#0-0) 

The variable `_l1_gas_prices_wei` is prefixed with `_` and never used again. Only the FRI prices are checked against the proposer's values:

```rust
if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
    && within_margin(
        l1_data_gas_price_fri_proposed,
        l1_data_gas_price_fri,
        l1_gas_price_margin_percent,
    ))
``` [2](#0-1) 

`l1_gas_price_wei` and `l1_data_gas_price_wei` from the proposer's `ProposalInit` are never compared to the oracle's WEI prices.

**Propagation — unvalidated WEI prices enter block execution**

After `is_block_info_valid` returns `Ok`, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which reads the proposer-supplied WEI values directly:

```rust
let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
``` [3](#0-2) 

These become `eth_gas_prices.l1_gas_price` and `eth_gas_prices.l1_data_gas_price` in the `BlockInfo` handed to the batcher:

```rust
eth_gas_prices: GasPriceVector {
    l1_gas_price: l1_gas_price_wei,
    l1_data_gas_price: l1_data_gas_price_wei,
    l2_gas_price: l2_gas_price_wei,
},
``` [4](#0-3) 

The same `init` is reused in `finalize_decision` to produce the `cende_block_info` that is committed to state sync and the blob:

```rust
let Ok(cende_block_info) = convert_to_sn_api_block_info(init) else { ... };
``` [5](#0-4) 

**Implicit constraint does not prevent manipulation**

`convert_to_sn_api_block_info` derives `eth_to_fri_rate` from the proposer-supplied WEI and FRI prices, then uses it to compute `l2_gas_price_wei`. This creates a consistency constraint: if the WEI price is set too far from the FRI price, `l2_gas_price_wei` becomes zero and the conversion fails. However, this constraint is loose. A proposer can choose any WEI price `P'_wei` such that:

```
l2_gas_price_fri / (P_fri / P'_wei * ETH_TO_WEI) > 0
```

For typical mainnet values this allows the WEI price to be manipulated by an order of magnitude or more while still passing `convert_to_sn_api_block_info`.

**`ProposalInit` carries both WEI and FRI prices precisely to enable this validation**

The struct comment explicitly states the WEI prices are included to allow distinguishing L1-price errors from conversion-rate errors:

```
/// This struct differs from `BlockInfo` in `starknet_api` because we send L1 gas prices in ETH and
/// include the ETH to STRK conversion rate. This allows for more informative validations…
``` [6](#0-5) 

The WEI prices are present for validation purposes but the validation is never performed.

---

### Impact Explanation

1. **Wrong ETH gas prices committed to the block header** — `l1_gas_price.price_in_wei` and `l1_data_gas_price.price_in_wei` stored in the block header are taken from the unvalidated proposer values. Every downstream consumer of the block header (state sync, P2P peers, RPC `starknet_getBlockWithTxHashes`, proof inputs) receives a wrong authoritative value. This matches the **Critical** impact: *Wrong state … storage value … from blockifier/syscall/execution logic for accepted input*.

2. **Incorrect ETH-denominated fee for every L1-handler transaction in the block** — The blockifier uses `eth_gas_prices` to compute the ETH fee charged for L1-handler transactions. A proposer who sets `l1_gas_price_wei = 1` causes every L1-handler in the block to be charged near-zero ETH fees, regardless of actual L1 gas cost. This matches the **Critical** impact: *Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact*.

---

### Likelihood Explanation

Any consensus participant whose turn it is to propose a block can trigger this. No special privilege beyond being a validator is required. The manipulation is invisible to other validators because `is_block_info_valid` only checks FRI prices. The attack can be repeated every block the malicious proposer is selected.

---

### Recommendation

Inside `is_block_info_valid`, after fetching `_l1_gas_prices_wei` from the oracle, validate the proposer's WEI prices against the oracle's WEI prices using the same `within_margin` logic already applied to FRI prices:

```rust
let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(...).await;

// existing FRI checks …

if !(within_margin(init_proposed.l1_gas_price_wei,    l1_gas_prices_wei.l1_gas_price,    margin)
  && within_margin(init_proposed.l1_data_gas_price_wei, l1_gas_prices_wei.l1_data_gas_price, margin))
{
    return Err(ValidateProposalError::InvalidBlockInfo(...));
}
```

Alternatively, derive the WEI prices deterministically from the validated FRI prices and the oracle's ETH/STRK rate, removing the proposer's ability to supply them independently.

---

### Proof of Concept

Assume the oracle reports:
- `l1_gas_price_wei` = 100 gwei (10¹¹ wei)
- `l1_gas_price_fri` = 10¹⁴ fri
- `l2_gas_price_fri` = 10⁹ fri

A malicious proposer constructs a `ProposalInit` with:
- `l1_gas_price_fri` = 10¹⁴ fri ← passes `within_margin` check
- `l1_gas_price_wei` = 10 gwei (10¹⁰ wei) ← 10× lower than true, **never checked**

`is_block_info_valid` returns `Ok` because only FRI prices are validated.

`convert_to_sn_api_block_info` computes:
- `eth_to_fri_rate` = 10¹⁴ / 10¹⁰ × 10¹⁸ = 10²² (10× higher than true)
- `l2_gas_price_wei` = 10⁹ / 10²² × 10¹⁸ = 10⁵ (non-zero, conversion succeeds)

The batcher receives `BlockInfo` with `eth_gas_prices.l1_gas_price = 10 gwei`. Every L1-handler transaction in the block is charged ETH fees at 10× below the true market rate. The block header stored in state records `l1_gas_price.price_in_wei = 10 gwei`, a wrong authoritative value propagated to all peers and proof inputs. [7](#0-6) [8](#0-7)

#[instrument(level = "warn", skip_all, fields(?block_info_validation, ?init_proposed))]
async fn is_block_info_valid(
    block_info_validation: &BlockInfoValidation,
    init_proposed: &ProposalInit,
    clock: &dyn Clock,
    l1_gas_price_provider: Arc<dyn L1GasPriceProviderClient>,
    gas_price_params: &GasPriceParams,
) -> ValidateProposalResult<()> {
    let now: u64 = clock.unix_now();
    let last_block_timestamp =
        block_info_validation.previous_block_info.as_ref().map_or(0, |info| info.timestamp);
    if init_proposed.timestamp < last_block_timestamp {
        return Err(ValidateProposalError::InvalidBlockInfo(
            init_proposed.clone(),
            block_info_validation.clone(),
            format!(
                "Timestamp is too old: last_block_timestamp={}, proposed={}",
                last_block_timestamp, init_proposed.timestamp
            ),
        ));
    }
    if init_proposed.timestamp > now + block_info_validation.block_timestamp_window_seconds {
        return Err(ValidateProposalError::InvalidBlockInfo(
            init_proposed.clone(),
            block_info_validation.clone(),
            format!(
                "Timestamp is in the future: now={}, block_timestamp_window_seconds={}, \
                 proposed={}",
                now, block_info_validation.block_timestamp_window_seconds, init_proposed.timestamp
            ),
        ));
    }
    if !(init_proposed.height == block_info_validation.height
        && init_proposed.l1_da_mode == block_info_validation.l1_da_mode
        && init_proposed.l2_gas_price_fri == block_info_validation.l2_gas_price_fri)
    {
        return Err(ValidateProposalError::InvalidBlockInfo(
            init_proposed.clone(),
            block_info_validation.clone(),
            "Block info validation failed".to_string(),
        ));
    }
    let (l1_gas_prices_fri, _l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
        l1_gas_price_provider,
        init_proposed.timestamp,
        block_info_validation.previous_block_info.as_ref(),
        gas_price_params,
    )
    .await;
    let l1_gas_price_margin_percent =
        VersionedConstants::latest_constants().l1_gas_price_margin_percent.into();
    debug!("L1 price info: {l1_gas_prices_fri:?}");

    let l1_gas_price_fri = l1_gas_prices_fri.l1_gas_price;
    let l1_data_gas_price_fri = l1_gas_prices_fri.l1_data_gas_price;
    let l1_gas_price_fri_proposed = init_proposed.l1_gas_price_fri;
    let l1_data_gas_price_fri_proposed = init_proposed.l1_data_gas_price_fri;

    if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
        && within_margin(
            l1_data_gas_price_fri_proposed,
            l1_data_gas_price_fri,
            l1_gas_price_margin_percent,
        ))
    {
        return Err(ValidateProposalError::InvalidBlockInfo(
            init_proposed.clone(),
            block_info_validation.clone(),
            format!(
                "L1 gas price mismatch: expected L1 gas price FRI={l1_gas_price_fri}, \
                 proposed={l1_gas_price_fri_proposed}, expected L1 data gas price \
                 FRI={l1_data_gas_price_fri}, proposed={l1_data_gas_price_fri_proposed}, \
                 l1_gas_price_margin_percent={l1_gas_price_margin_percent}"
            ),
        ));
    }
    Ok(())
}

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let previous_block_info = PreviousBlockInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}
