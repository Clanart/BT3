### Title
`MerkleBlob::batch_insert` Bypasses Duplicate-Key Validation, Corrupting Tree Root — (`File: crates/chia-datalayer/src/merkle/blob.rs`)

### Summary

`MerkleBlob::batch_insert` skips the duplicate-key and duplicate-hash guards that `MerkleBlob::insert` enforces. An unprivileged caller supplying a batch that contains a key already present in the tree (or a repeated key within the batch itself) silently writes extra leaf nodes into the flat blob. The `block_status_cache` HashMap then silently overwrites the old index for that key, leaving the blob with more physical leaves than cache entries. Every subsequent Merkle-root computation incorporates the phantom leaf, producing a root that does not correspond to the declared key-value set. Inclusion and exclusion proofs derived from that root are therefore forged.

### Finding Description

`MerkleBlob::insert` enforces two guards before writing anything:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 369-374
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
```

`batch_insert` calls `insert` (with those guards) only for the **last two** items popped from the input vector when the tree starts with ≤ 1 leaf. Every other item — the entire batch when the tree already has ≥ 2 leaves, or the first `N-2` items when it starts smaller — is written directly through `insert_entry_to_blob` with **no duplicate check**:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 578-603
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else {
            return Ok(());
        };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;  // guarded
    }
}

for ((key, value), hash) in keys_values_hashes {          // ← NO duplicate check
    let new_leaf_index = self.get_new_index();
    let new_block = Block { … Node::Leaf(LeafNode { hash, key, value, … }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
```

`insert_entry_to_blob` calls `block_status_cache.add_leaf(index, leaf)`, which updates the `key_to_index` and `leaf_hash_to_index` HashMaps via `HashMap::insert` — silently overwriting the old entry for a duplicate key. The old leaf node remains physically in the blob at its original index, but the cache no longer tracks it. The tree now has:

- **blob leaf count** = N + 1 (original N plus the phantom duplicate)
- **cache key count** = N (the HashMap overwrote the old entry)

`check_integrity` would detect this mismatch (`IntegrityKeyToIndexCacheLength`), but that check is only compiled in under `#[cfg(any(test, debug_assertions))]` and is not called on every mutation in production.

The Python binding `py_batch_insert` exposes this path directly to callers:

```rust

    &mut self,
    keys_values: Vec<(KeyId, ValueId)>,
    hashes: Vec<Hash>,

    …
    self.batch_insert(zip(keys_values, hashes).collect())?;
    Ok(())

```

### Impact Explanation

After a duplicate leaf is silently inserted:

1. **Corrupted root hash.** `calculate_lazy_hashes` traverses all physical leaf nodes, including the phantom duplicate, and folds them into the root. The resulting root does not match the canonical root for the declared key-value set.

2. **Forged inclusion proofs.** A lineage proof constructed against the corrupted root can "prove" membership for a key at a position that does not correspond to its actual value, or prove membership for a key that was supposed to have been deleted.

3. **Forged exclusion proofs.** Because the cache points to the *new* index for the duplicate key, a lookup for the *old* index returns nothing, making the tree appear to exclude a key that is physically present.

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

### Likelihood Explanation

`py_batch_insert` is part of the public Python API (`datalayer.pyi`) and is called by the DataLayer service whenever it bulk-inserts new key-value pairs. Any caller — including an untrusted client that can submit DataLayer store operations — who supplies a batch containing a key already present in the store (or a repeated key within the batch) triggers the corruption silently, with no error returned. No special privilege is required.

### Recommendation

Add the same duplicate-key and duplicate-hash guards inside the fast-path loop of `batch_insert`, mirroring what `insert` already does:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    let new_leaf_index = self.get_new_index();
    …
}
```

Alternatively, deduplicate the input vector before processing and return an error if any duplicate is detected.

### Proof of Concept

```rust
// Demonstrates silent duplicate insertion via batch_insert
let mut blob = MerkleBlob::new(vec![]).unwrap();

// Pre-populate with 2 leaves so leaf_count > 1 → ALL batch items skip the guard
let k0 = KeyId(0); let v0 = ValueId(0); let h0 = sha256_num(&0i64);
let k1 = KeyId(1); let v1 = ValueId(1); let h1 = sha256_num(&1i64);
blob.insert(k0, v0, &h0, InsertLocation::Auto {}).unwrap();
blob.insert(k1, v1, &h1, InsertLocation::Auto {}).unwrap();

// batch_insert with k0 duplicated — no error is returned
let k2 = KeyId(2); let v2 = ValueId(2); let h2 = sha256_num(&2i64);
blob.batch_insert(vec![
    ((k2, v2), h2),
    ((k0, ValueId(99)), sha256_num(&99i64)),  // duplicate key k0
]).unwrap();  // succeeds silently

// blob now has 4 physical leaf nodes but cache has only 3 entries
// root hash is computed over 4 leaves → corrupted root
blob.calculate_lazy_hashes().unwrap();

    fn insert_entry_to_blob(&mut self, index: TreeIndex, block: &Block) -> Result<(), Error> {
        let new_block_bytes = block.to_bytes()?;
        let extend_index = self.extend_index();
        match index.cmp(&extend_index) {
            Ordering::Greater => return Err(Error::BlockIndexOutOfBounds(index)),
            Ordering::Equal => self.blob.extend_from_slice(&new_block_bytes),
            Ordering::Less => {
                self.blob[block_range(index)].copy_from_slice(&new_block_bytes);
            }
        }

        match block.node {
            Node::Leaf(leaf) => self.block_status_cache.add_leaf(index, leaf),
            Node::Internal(..) => self.block_status_cache.add_internal(index),
        }

        Ok(())
    }
