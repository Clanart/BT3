### Title
`deploy_token` Callable When Bridge Is Paused, Bypassing Emergency Stop — (`solana/programs/bridge_token_factory/src/lib.rs`)

### Summary

The Solana `bridge_token_factory` program defines a selective pause system covering `init_transfer` and `finalize_transfer`, but the `deploy_token` instruction has no pause guard. When the bridge is paused via `pause()`, which sets `ALL_PAUSED`, token deployment remains fully operational. This is a direct analog of the reported vulnerability class: a state-mutating, cross-chain-consequential function missing a pause check.

### Finding Description

`constants.rs` defines only two pause flags: [1](#0-0) 

```rust
pub const INIT_TRANSFER_PAUSED: u8 = 1 << 0;
pub const FINALIZE_TRANSFER_PAUSED: u8 = 1 << 1;
pub const ALL_PAUSED: u8 = INIT_TRANSFER_PAUSED | FINALIZE_TRANSFER_PAUSED;
```

There is no `DEPLOY_TOKEN_PAUSED` flag. The `pause()` instruction sets `config.paused = ALL_PAUSED`: [2](#0-1) 

`finalize_transfer`, `finalize_transfer_sol`, `init_transfer`, and `init_transfer_sol` all guard with a `require!` check against the pause flags: [3](#0-2) 

But `deploy_token` has no such check: [4](#0-3) 

```rust
pub fn deploy_token(
    ctx: Context<DeployToken>,
    data: SignedPayload<DeployTokenPayload>,
) -> Result<()> {
    msg!("Deploying token");
    data.verify_signature((), &ctx.accounts.common.config.derived_near_bridge_address)?;
    ctx.accounts.initialize_token_metadata(data.payload)?;
    Ok(())
}
```

`initialize_token_metadata` creates a new SPL mint on-chain (via the `init` account constraint on `mint`) and posts a Wormhole message back to NEAR: [5](#0-4) 

### Impact Explanation

When the bridge is paused (e.g., due to a security incident), any caller holding a valid MPC-signed `DeployTokenPayload` — which is produced by a prior `log_metadata` call and signed by the NEAR MPC signer — can still:

1. Create a new irreversible SPL mint on Solana.
2. Emit a Wormhole cross-chain message to NEAR registering the token mapping.

On NEAR, `deploy_token` is guarded by `#[pause(except(roles(Role::DAO)))]`: [6](#0-5) 

So the Wormhole message is queued and processed when NEAR unpauses. The Solana mint creation is irreversible. This means the pause does not fully halt token deployment: new token-to-mint mappings are created on Solana and propagated to NEAR outside the emergency window, corrupting the token-mapping state the protocol relies on for correct collateralization accounting. This matches the allowed impact: **token-mapping corruption that breaks bridge collateralization or misdirects value**.

### Likelihood Explanation

Likelihood is medium. Valid signed `DeployTokenPayload` messages exist whenever a `log_metadata` call has been processed by the NEAR MPC signer but the corresponding `deploy_token` on Solana has not yet been submitted. These are normal in-flight protocol states. Any unprivileged user who observes such a signed payload (e.g., from Wormhole VAA logs) can submit it to `deploy_token` regardless of pause state.

### Recommendation

1. Add a `DEPLOY_TOKEN_PAUSED: u8 = 1 << 2` constant to `constants.rs`.
2. Update `ALL_PAUSED` to include it: `ALL_PAUSED = INIT_TRANSFER_PAUSED | FINALIZE_TRANSFER_PAUSED | DEPLOY_TOKEN_PAUSED`.
3. Add a pause guard to `deploy_token` in `lib.rs`:

```rust
pub fn deploy_token(
    ctx: Context<DeployToken>,
    data: SignedPayload<DeployTokenPayload>,
) -> Result<()> {
    require!(
        ctx.accounts.common.config.paused & DEPLOY_TOKEN_PAUSED == 0,
        error::ErrorCode::Paused
    );
    msg!("Deploying token");
    data.verify_signature((), &ctx.accounts.common.config.derived_near_bridge_address)?;
    ctx.accounts.initialize_token_metadata(data.payload)?;
    Ok(())
}
```

### Proof of Concept

1. Admin calls `pause()` on the Solana bridge due to a security incident. `config.paused` is set to `ALL_PAUSED = 0x03` (covers only `INIT_TRANSFER_PAUSED | FINALIZE_TRANSFER_PAUSED`).
2. Attacker holds a valid Wormhole VAA containing a MPC-signed `DeployTokenPayload` for token `X` (obtained from a prior `log_metadata` Wormhole message).
3. Attacker calls `deploy_token` with this payload. The function has no pause check, so it proceeds.
4. A new SPL mint for token `X` is created on Solana (irreversible).
5. A Wormhole message is posted registering the Solana mint address for token `X`.
6. When NEAR unpauses, the Wormhole relayer delivers the message; NEAR's `deploy_token_callback` creates the token mapping.
7. The token mapping was created outside the intended operational window, bypassing the emergency stop. [1](#0-0) [4](#0-3) [2](#0-1)

    pub fn initialize_token_metadata(&self, mut metadata: DeployTokenPayload) -> Result<()> {
        let bump = &[self.common.config.bumps.authority];
        let signer_seeds = &[&[AUTHORITY_SEED, bump][..]];
        let origin_decimals = metadata.decimals;
        metadata.decimals = std::cmp::min(MAX_ALLOWED_DECIMALS, metadata.decimals);

        let cpi_accounts = CreateMetadataAccountsV3 {
            payer: self.common.payer.to_account_info(),
            update_authority: self.authority.to_account_info(),
            mint: self.mint.to_account_info(),
            metadata: self.metadata.to_account_info(),
            mint_authority: self.authority.to_account_info(),
            system_program: self.system_program.to_account_info(),
            rent: self.common.rent.to_account_info(),
        };
        let cpi_ctx = CpiContext::new_with_signer(
            self.token_metadata_program.to_account_info(),
            cpi_accounts,
            signer_seeds,
        );
        create_metadata_accounts_v3(
            cpi_ctx,
            DataV2 {
                name: metadata.name,
                symbol: metadata.symbol,
                uri: String::new(),
                seller_fee_basis_points: 0,
                creators: None,
                collection: None,
                uses: None,
            },
            true, // TODO: Maybe better to make it immutable
            true,
            None,
        )?;

        let payload = DeployTokenResponse {
            token: metadata.token,
            solana_mint: self.mint.key(),
            decimals: metadata.decimals,
            origin_decimals,
        }
        .serialize_for_near(())?;

        self.common.post_message(payload)?;

        Ok(())
    }
