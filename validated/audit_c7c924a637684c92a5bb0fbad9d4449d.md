### Title
`MerkleBlob::batch_insert` Bypasses Duplicate Key/Hash Checks, Corrupting Tree Root and Enabling Forged Inclusion Proofs - (File: `crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::batch_insert` performs duplicate-key and duplicate-hash validation only for the first 1–2 items (via the guarded `self.insert()` path), but silently bypasses both checks for all remaining items in the batch. An attacker who can supply a batch containing a repeated `KeyId` or repeated `Hash` will corrupt the Merkle tree's blob and root hash, and can subsequently obtain a proof of inclusion that validates against a false committed state.

---

### Finding Description

`MerkleBlob::insert` enforces two invariants before writing any leaf:

```rust
// blob.rs:369-374
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`batch_insert` calls `self.insert()` (with those guards) only for the first two items when the tree has ≤ 1 existing leaves:

```rust
// blob.rs:578-585
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else { ... };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;
    }
}
``` [2](#0-1) 

Every subsequent item in the batch is written directly via `insert_entry_to_blob`, with **no** duplicate-key or duplicate-hash check:

```rust
// blob.rs:587-602
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ... Node::Leaf(LeafNode { hash, key, value, ... }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
``` [3](#0-2) 

`insert_entry_to_blob` calls `block_status_cache.add_leaf(index, leaf)`, which overwrites the cache entry for a duplicate key with the new index, leaving the first leaf node physically present in the blob but no longer tracked:

```rust
// blob.rs:1024-1027
match block.node {
    Node::Leaf(leaf) => self.block_status_cache.add_leaf(index, leaf),
    Node::Internal(..) => self.block_status_cache.add_internal(index),
}
``` [4](#0-3) 

The result is a blob that contains two leaf nodes sharing the same `KeyId` at different tree indexes. The internal-node subtree built in the second phase of `batch_insert` incorporates both leaf hashes into the computed root, producing a root hash that does not correspond to any valid single-key-per-entry Merkle tree.

This is the structural analog to the DIMO-Identity collision: `insert()` checks new entries against the existing registry, but `batch_insert`'s fast path never checks new entries against the existing tree **or** against other entries in the same batch.

The Python binding `py_batch_insert` exposes this path directly to callers:

```rust
// blob.rs:1503-1519
#[pyo3(name = "batch_insert")]
pub fn py_batch_insert(&mut self, keys_values: Vec<(KeyId, ValueId)>, hashes: Vec<Hash>) -> PyResult<()> {
    ...
    self.batch_insert(zip(keys_values, hashes).collect())?;
    Ok(())
}
``` [5](#0-4) 

The public Python stub confirms the surface is reachable:

```python
def batch_insert(self, keys_values: list[tuple[KeyId, ValueId]], hashes: list[bytes32]): ...
``` [6](#0-5) 

---

### Impact Explanation

A corrupted tree root means:

1. **Forged inclusion proof**: `get_proof_of_inclusion` walks the cache to find a leaf's index, then traverses parent hashes. With two leaves sharing a key, the cache points to the second (overwriting) leaf, but the first leaf's hash is still embedded in the internal-node subtree. The proof path is built from a structurally inconsistent tree, so `proof.valid()` may return `true` for a leaf hash that was never legitimately committed.

2. **Root hash mismatch**: Any downstream system that trusts the root hash (e.g., a DataLayer store root committed on-chain) will accept a root that encodes duplicate state, allowing an operator or attacker to prove inclusion of a key-value pair that was silently overwritten.

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

---

### Likelihood Explanation

The `batch_insert` Python binding is the primary high-throughput insertion path for DataLayer. Any caller that passes a batch with a repeated `KeyId` (e.g., two updates to the same key in one batch, or a programming error in the caller) will silently corrupt the tree rather than receiving an error. The fuzz target for single `insert` explicitly handles `KeyAlreadyPresent` and `HashAlreadyPresent`, confirming these are expected error conditions — but no equivalent fuzz target exists for `batch_insert` with duplicate inputs. [7](#0-6) 

---

### Recommendation

Add duplicate-key and duplicate-hash checks at the start of the bulk loop in `batch_insert`, mirroring the guards in `insert`:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // ... existing insertion logic
}
```

Alternatively, pre-validate the entire input batch for uniqueness before any writes begin, and add a fuzz target that exercises `batch_insert` with duplicate keys/hashes.

### Proof of Concept

```python
from chia_rs.datalayer import MerkleBlob, KeyId, ValueId
from chia.types.blockchain_format.sized_bytes import bytes32

blob = MerkleBlob(blob=bytearray())

# Batch with 3+ items so the fast path is taken; items 0 and 1 are popped
# first (via insert()), item 2 onward bypass duplicate checks.
# Duplicate key in positions 2 and 3 (both processed by the unchecked loop).
dup_key   = KeyId(999)
dup_value = ValueId(1)
hash_a    = bytes32(b'\xaa' * 32)
hash_b    = bytes32(b'\xbb' * 32)

keys_values = [
    (KeyId(1), ValueId(1)),   # popped last  → insert() with guard
    (KeyId(2), ValueId(2)),   # popped first → insert() with guard
    (dup_key,  dup_value),    # unchecked fast path
    (dup_key,  ValueId(2)),   # DUPLICATE KEY — no error raised
]
hashes = [
    bytes32(b'\x01' * 32),
    bytes32(b'\x02' * 32),
    hash_a,
    hash_b,   # different hash, same key → bypasses HashAlreadyPresent too
]

blob.batch_insert(keys_values, hashes)   # succeeds silently
blob.calculate_lazy_hashes()

# Tree now contains two leaf nodes for dup_key; root hash is corrupted.
proof = blob.get_proof_of_inclusion(dup_key)
print("proof valid:", proof.valid())   # may print True for corrupted state
