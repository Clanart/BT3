### Title
DataLayer `ProofOfInclusion::valid()` Does Not Verify Against an External Root Hash, Enabling Forged Inclusion Proofs — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
The `ProofOfInclusion::valid()` method only checks internal hash-chain self-consistency. The final guard `existing_hash == self.root_hash()` is tautologically true whenever the loop passes, so the method never compares the computed root against any externally-supplied trusted root. An attacker who can deliver a deserialized `ProofOfInclusion` to a verifier that calls only `valid()` can forge a proof of inclusion for any key in any claimed tree root.

### Finding Description
`ProofOfInclusion::valid()` walks the `layers` vector, recomputing each `combined_hash` from the running hash and `other_hash`, and returns `false` on any mismatch. After the loop it checks:

```rust
existing_hash == self.root_hash()
```

`root_hash()` is defined as:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← always equals existing_hash after the loop
    } else {
        self.node_hash
    }
}
```

After the loop, `existing_hash` holds `calculated_hash` from the last iteration, which the loop already asserted equals `layer.combined_hash`. `root_hash()` returns that same `last.combined_hash`. The final equality is therefore always `true` when the loop passes — it is a dead check that provides no security guarantee.

The method never accepts an expected root as a parameter and never compares against one. A `ProofOfInclusion` is a `Streamable` type fully constructible from untrusted bytes via `from_bytes` / `parse_rust`. [1](#0-0) [2](#0-1) 

### Impact Explanation
Any party that receives a `ProofOfInclusion` from an untrusted source and calls only `proof.valid()` will accept a completely fabricated proof. The attacker constructs:

- `node_hash` = any 32-byte value they wish to claim is a leaf
- One or more `ProofOfInclusionLayer` entries whose `combined_hash` values are computed correctly from the attacker-chosen `other_hash` values

The resulting object passes `valid()` and reports an attacker-chosen `root_hash()`. This lets untrusted input prove invalid DataLayer state — forged key-value inclusion against an arbitrary claimed tree root — matching the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."* [3](#0-2) [4](#0-3) 

### Likelihood Explanation
`ProofOfInclusion` is a `Streamable` type exposed through the Python binding (`get_proof_of_inclusion`, `from_bytes`, `parse_rust`). DataLayer clients that receive proofs over the network and rely on `proof.valid()` as the sole check — a natural reading of the API — are directly vulnerable. The existing test suite calls `valid()` only on proofs produced locally by a trusted `MerkleBlob`, masking the gap. [5](#0-4) [6](#0-5) 

### Recommendation
1. Add an `expected_root` parameter to `valid()` (or provide a companion `valid_against_root(expected_root: &Hash) -> bool`) that compares the computed root against the caller-supplied trusted root before returning `true`.
2. Remove or document the tautological final check so the intent is unambiguous.
3. Update the Python binding and all call sites to pass the expected root, mirroring how `validate_merkle_proof` in `crates/chia-consensus/src/merkle_tree.rs` correctly checks `tree.get_root() != *root` before accepting a proof. [7](#0-6) 

### Proof of Concept
```rust
use chia_datalayer::{Hash, Side, calculate_internal_hash};
use chia_datalayer::merkle::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};

// Attacker-chosen values — nothing to do with any real MerkleBlob
let fake_leaf  = Hash::from([0xAA; 32]);
let fake_other = Hash::from([0xBB; 32]);
let fake_root  = calculate_internal_hash(&fake_leaf, Side::Left, &fake_other);

let forged = ProofOfInclusion {
    node_hash: fake_leaf,
    layers: vec![ProofOfInclusionLayer {
        other_hash_side: Side::Left,
        other_hash:      fake_other,
        combined_hash:   fake_root,   // attacker controls the claimed root
    }],
};

// Passes — no external root is checked
assert!(forged.valid());

// The "root" is whatever the attacker chose, not the real tree root
assert_eq!(forged.root_hash(), fake_root);
```

A verifier that trusts `forged.valid()` alone will accept this as proof that `fake_leaf` is included in a tree whose root is `fake_root`, even though no such tree exists. [1](#0-0)

    pub fn valid(&self) -> bool {
        let mut existing_hash = self.node_hash;

        for layer in &self.layers {
            let calculated_hash = crate::calculate_internal_hash(
                &existing_hash,
                layer.other_hash_side,
                &layer.other_hash,
            );

            if calculated_hash != layer.combined_hash {
                return false;
            }

            existing_hash = calculated_hash;
        }

        existing_hash == self.root_hash()
    }

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
