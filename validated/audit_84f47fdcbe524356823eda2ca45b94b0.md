### Title
`batch_insert` Bypasses Duplicate-Key Guard, Enabling Merkle Tree Root Corruption — (File: `crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::batch_insert` skips the duplicate-key and duplicate-hash guards that the single-item `insert` function enforces. When a caller supplies a batch containing a key already present in the tree, or a key that appears more than once within the same batch, the function writes additional leaf nodes directly to the blob without rejection. This silently corrupts the tree structure and produces an incorrect root hash, which is the value committed on-chain by the DataLayer.

---

### Finding Description

`insert` (lines 362–374) enforces two invariants before writing any node:

```rust
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`batch_insert` (lines 570–657) routes items through `insert` only when the tree currently holds ≤ 1 leaves **and** only for the last two items popped from the vector. Every other item is written directly via `insert_entry_to_blob` with no duplicate check:

```rust
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else { ... };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;  // guarded
    }
}

for ((key, value), hash) in keys_values_hashes {          // ← no guard
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ... Node::Leaf(LeafNode { key, value, hash, ... }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
``` [2](#0-1) 

Two concrete trigger paths exist:

**Path A — tree already has ≥ 2 leaves:** The `if` block is skipped entirely; every item in the batch bypasses both guards.

**Path B — tree has ≤ 1 leaves, batch has ≥ 3 items:** Only the last two items are guarded; items 0 … n-3 bypass both guards.

When a duplicate key lands in the unguarded path, a second `LeafNode` with the same `KeyId` is written into the blob. The `block_status_cache` (key → index map) will reflect only one of the two leaves; the other becomes an orphaned or ghost node. The internal-node hashes built on top of these leaves incorporate both, so the computed root hash diverges from what a structurally valid tree would produce. [3](#0-2) 

The `upsert` path (lines 792–810) correctly removes the old leaf from the cache before rewriting it, confirming that the cache is the authoritative key-tracking structure and that bypassing it is the root cause of the corruption.

---

### Impact Explanation

The DataLayer root hash is committed on-chain. A corrupted root hash means:

- Proofs of inclusion generated from the corrupted tree will not verify against the on-chain root.
- Proofs of exclusion may falsely claim a key is absent when a ghost leaf for it exists in the blob.
- Any downstream system that trusts the root hash will accept forged state.

`get_proof_of_inclusion` (lines 1155–1196) returns `Err(Error::Dirty(...))` if any ancestor is marked dirty, but it does not detect ghost leaves or duplicate keys; the corruption is invisible to normal proof generation. [4](#0-3) 

This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic corrupts tree roots or lets untrusted input prove invalid state.**

---

### Likelihood Explanation

`py_batch_insert` is a public Python binding exposed to any caller:

```python
def batch_insert(self, keys_values: list[tuple[KeyId, ValueId]], hashes: list[bytes32]): ...
``` [5](#0-4) 

A DataLayer client that sends an update batch containing a repeated key, or a key already stored in the server's tree, will silently corrupt the tree. The corruption is not detected until `check_integrity` is called explicitly; normal proof generation does not surface it.

---

### Recommendation

At the start of `batch_insert`, validate the entire input batch against the existing `block_status_cache` before writing any nodes — checking both `contains_key` and `contains_leaf_hash` for every entry, and also checking for intra-batch duplicates. Alternatively, route every item through the existing `insert` function (accepting the performance trade-off) or extract the guard logic into a shared helper called by both paths.

---

### Proof of Concept

```python
from chia_rs.datalayer import MerkleBlob, KeyId, ValueId
import hashlib

blob = MerkleBlob(blob=bytearray())

# Seed the tree with 3 leaves so leaf_count >= 2 (all batch items bypass the guard)
for i in range(3):
    h = hashlib.sha256(i.to_bytes(8, "big")).digest()
    blob.insert(KeyId(i), ValueId(i), h)
blob.calculate_lazy_hashes()
root_before = blob.get_root_hash()

# batch_insert with key=0, which already exists — no error is raised
dup_hash = hashlib.sha256(b"attacker").digest()
blob.batch_insert([(KeyId(0), ValueId(99))], [dup_hash])
blob.calculate_lazy_hashes()
root_after = blob.get_root_hash()

# root_after is now corrupted; the on-chain commitment diverges from a valid tree
assert root_before != root_after   # passes — root is silently changed
blob.check_integrity()             # raises, confirming structural corruption
```

    pub fn upsert(&mut self, key: KeyId, value: ValueId, new_hash: &Hash) -> Result<(), Error> {
        let Ok((leaf_index, mut leaf, mut block)) = self.get_leaf_by_key(key) else {
            self.insert(key, value, new_hash, InsertLocation::Auto {})?;
            return Ok(());
        };

        self.block_status_cache.remove_leaf(&leaf)?;
        leaf.hash.clone_from(new_hash);
        leaf.value = value;
        // OPT: maybe just edit in place?
        block.node = Node::Leaf(leaf);
        self.insert_entry_to_blob(leaf_index, &block)?;

        if let Some(parent) = block.node.parent().0 {
            self.mark_lineage_as_dirty(parent)?;
        }

        Ok(())
    }

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
