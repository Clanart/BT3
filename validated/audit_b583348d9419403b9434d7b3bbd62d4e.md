### Title
`ProofOfInclusion::valid()` Tautological Root Check Allows Forged DataLayer Inclusion Proofs — (`File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` contains a tautological final check that makes it impossible to verify a proof against any external/known tree root. The method only verifies internal hash-chain consistency, not that the proof corresponds to a specific committed root. Because `ProofOfInclusion` is `Streamable` (deserializable from untrusted bytes) and `valid()` is the sole exposed verification method, an attacker can construct a forged proof claiming any key is included in any DataLayer tree, and `valid()` will return `true`.

### Finding Description

In `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`, the `valid()` method is:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;  // existing_hash = layer.combined_hash
    }

    existing_hash == self.root_hash()     // TAUTOLOGY
}
``` [1](#0-0) 

And `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // returns attacker-controlled value
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

**Trace of the tautology (non-empty `layers` case):**

1. After the loop body executes for the last layer, `existing_hash` is set to `calculated_hash`, which was just verified to equal `layer.combined_hash`.
2. So after the loop: `existing_hash == last_layer.combined_hash`.
3. `self.root_hash()` returns `last_layer.combined_hash` (the `Some` branch).
4. Therefore `existing_hash == self.root_hash()` reduces to `last_layer.combined_hash == last_layer.combined_hash` — always `true`.

The final check is dead code. `valid()` never compares the computed root against any external, trusted root. It only verifies that the proof's own internal hash chain is self-consistent — a property the attacker fully controls.

**Forge construction:** An attacker can build a valid-passing `ProofOfInclusion` for any claimed `node_hash`:

1. Choose arbitrary `node_hash` (e.g., hash of a key not in the tree).
2. For each layer, choose arbitrary `other_hash` and `other_hash_side`, then compute `combined_hash = calculate_internal_hash(existing_hash, side, other_hash)`.
3. The resulting struct passes `valid()` and `root_hash()` returns the attacker-chosen final hash.

`ProofOfInclusion` is `Streamable` and fully deserializable from bytes via `from_bytes()` / `from_bytes_unchecked()`, exposed in both Rust and Python bindings. [3](#0-2) 

The Python binding exposes `valid()` directly: [4](#0-3) 

And the Python type stub documents it as the canonical verification method: [5](#0-4) 

All existing tests call `proof.valid()` without comparing `root_hash()` to any external root, confirming the pattern is established: [6](#0-5) [7](#0-6) 

### Impact Explanation

Any DataLayer client that receives a `ProofOfInclusion` from an untrusted source (e.g., a DataLayer server or peer) and calls `valid()` to verify it will accept a forged proof. The attacker can claim any key-value pair is included in any DataLayer tree root, enabling:

- **Forged state proofs**: Proving inclusion of data that does not exist in the committed tree.
- **Invalid state acceptance**: Downstream logic that gates on `proof.valid()` (the only exposed verification method) will be deceived.

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

### Likelihood Explanation

`ProofOfInclusion` is `Streamable`, Python-exposed, and `valid()` is the sole verification method. The method name implies completeness. Any DataLayer client verifying received proofs using only `valid()` — the natural and documented usage — is vulnerable. The tautology is non-obvious and missed by all existing tests.

### Recommendation

`valid()` must accept the expected root as a parameter and compare the computed root against it:

```rust
pub fn valid_for_root(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }

    &existing_hash == expected_root  // compare against EXTERNAL known root
}
```

The existing `valid()` (no-argument form) should either be removed or clearly documented as only checking internal self-consistency, not proof authenticity. All call sites must be updated to supply the known committed root.

### Proof of Concept

```rust
use chia_datalayer::{Hash, MerkleBlob, KeyId, ValueId, InsertLocation};
use chia_datalayer::merkle::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};
use chia_datalayer::Side;

// Build a real tree with one entry
let mut blob = MerkleBlob::new(Vec::new()).unwrap();
let real_key = KeyId(1);
let real_hash = Hash(Bytes32::new([0xAA; 32]));
blob.insert(real_key, ValueId(1), &real_hash, InsertLocation::Auto {}).unwrap();
blob.calculate_lazy_hashes().unwrap();
let real_root = blob.get_root_hash().unwrap();

// Forge a proof for a key NOT in the tree
let fake_node_hash = Hash(Bytes32::new([0xFF; 32]));
let other_hash    = Hash(Bytes32::new([0x11; 32]));
let combined      = calculate_internal_hash(&fake_node_hash, Side::Left, &other_hash);

let forged = ProofOfInclusion {
    node_hash: fake_node_hash,
    layers: vec![ProofOfInclusionLayer {
        other_hash_side: Side::Left,
        other_hash,
        combined_hash: combined,
    }],
};

// valid() returns true — forged proof accepted
assert!(forged.valid());

// root_hash() returns attacker-controlled value, NOT real_root
assert_ne!(forged.root_hash(), real_root);
// But callers relying solely on valid() never check this
```
