### Title
Stale Compiled Class Hash in `ExecutionStateReader::get_compiled_class_hash` Causes Wrong Fee Estimation After CASM Hash Migration — (File: `crates/apollo_rpc_execution/src/state_reader.rs`)

### Summary

`ExecutionStateReader::get_compiled_class_hash` reads the compiled class hash from the class's **declaration block's state diff**, ignoring any subsequent `migrated_compiled_classes` updates written to later blocks. After a CASM hash migration (H_v1 → H_v2) is committed in block N+K, the function still returns H_v1. This causes `should_migrate` to fire again during RPC fee estimation and simulation for every transaction that invokes a migrated class, inflating fee estimates with migration costs that will not be incurred in actual block execution.

### Finding Description

`ExecutionStateReader::get_compiled_class_hash` resolves the compiled class hash via a two-step path:

1. `get_class_definition_block_number(&class_hash)` — returns block N, the block where the class was **first declared**.
2. `get_state_diff(N)` — reads the raw state diff at block N.
3. `state_diff.class_hash_to_compiled_class_hash.get(&class_hash)` — returns H_v1, the hash recorded at declaration time. [1](#0-0) 

The function never consults `self.state_number` and never calls `get_compiled_class_hash_at`, so it is blind to any entry written to the `compiled_class_hash_table` after block N.

`ApolloReader::get_compiled_class_hash` — the path used during actual block execution — does the opposite: it calls `get_compiled_class_hash_at(state_number, &class_hash)`, which performs a cursor lower-bound scan over the `compiled_class_hash_table` keyed by `(class_hash, block_number)` and returns the most recent entry at or before `state_number`. [2](#0-1) 

The `compiled_class_hash_table` is written unconditionally for every entry in `class_hash_to_compiled_class_hash` of each block's `ThinStateDiff`, including migration blocks: [3](#0-2) 

When `enable_casm_hash_migration` is active, `finalize_block` calls `set_compiled_class_hash_migration`, which writes H_v2 into the `CachedState` writes map and ultimately into the `compiled_class_hash_table` at block N+K: [4](#0-3) 

After block N+K is committed, `get_compiled_class_hash_at` returns H_v2 for any state number ≥ N+K. But `ExecutionStateReader::get_compiled_class_hash` still returns H_v1 because it reads from block N's state diff.

During RPC fee estimation, the `TransactionExecutor` is constructed with `ExecutionStateReader` as the underlying state reader. The bouncer's `CasmHashMigrationData::from_state` calls `should_migrate` with the `CachedState` wrapping `ExecutionStateReader`: [5](#0-4) 

`should_migrate` calls `state_reader.get_compiled_class_hash(class_hash)`. Through `CachedState::get_compiled_class_hash`, this reaches `ExecutionStateReader::get_compiled_class_hash` and returns H_v1. Since H_v1 ≠ H_v2, `should_migrate` returns `Some(...)`, migration resources are accumulated, and the fee estimate is inflated: [6](#0-5) 

In actual block execution the same class is read via `ApolloReader`, which returns H_v2, `should_migrate` returns `None`, no migration resources are charged, and the actual fee is lower than the estimate.

### Impact Explanation

`starknet_estimateFee` and `starknet_simulateTransactions` return authoritative-looking wrong values (inflated fees and spurious migration entries in simulated state diffs) for any transaction that invokes a class that has undergone CASM hash migration. The discrepancy is the full migration cost for the affected class, which can be substantial for large contracts. This matches the **High** impact scope: *RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.*

### Likelihood Explanation

The condition requires `enable_casm_hash_migration = true` in `VersionedConstants` and at least one class to have been migrated. Once those conditions hold, the bug is persistent and affects every subsequent fee estimate for transactions invoking any migrated class — no special attacker action is required. Any ordinary user calling `starknet_estimateFee` observes the wrong value.

### Recommendation

Replace the declaration-block lookup in `ExecutionStateReader::get_compiled_class_hash` with a call to `get_compiled_class_hash_at(self.state_number, &class_hash)`, consistent with `ApolloReader::get_compiled_class_hash`. This ensures the function returns the most recent compiled class hash at the requested state, including any post-declaration migrations.

### Proof of Concept

1. Class `C` is declared in block N with `compiled_class_hash = H_v1` (Poseidon-based). `declared_classes_block_table[C] = N`; `compiled_class_hash_table[(C, N)] = H_v1`.
2. `enable_casm_hash_migration = true`. In block N+K a transaction invokes `C`; the bouncer detects H_v1 ≠ H_v2 and triggers migration. `finalize_block` calls `set_compiled_class_hash_migration`; block N+K's `ThinStateDiff.class_hash_to_compiled_class_hash` contains `C → H_v2`; `write_compiled_class_hashes` inserts `compiled_class_hash_table[(C, N+K)] = H_v2`.
3. After block N+K is committed, a user calls `starknet_estimateFee` for a transaction invoking `C` at the current head.
4. `ExecutionStateReader::get_compiled_class_hash(C)`:
   - `get_class_definition_block_number(C)` → N
   - `get_state_diff(N).class_hash_to_compiled_class_hash[C]` → **H_v1** (stale)
5. `CachedState::get_compiled_class_hash(C)` caches H_v1 as the initial value.
6. `should_migrate(cached_state, C)`: `state_compiled_class_hash = H_v1`; `compiled_class_hash_v2 = H_v2`; H_v1 ≠ H_v2 → returns `Some((C, (H_v2, H_v1)))`.
7. Migration resources for `C` are added to the bouncer weights; fee estimate is inflated by the migration cost.
8. In actual block execution, `ApolloReader::get_compiled_class_hash(C)` calls `get_compiled_class_hash_at(state_number, C)` → cursor scan finds `(C, N+K) = H_v2` → returns **H_v2**.
9. `should_migrate` returns `None`; no migration resources charged; actual fee < estimated fee.
10. The user either overpays (if they use the estimate as the fee cap) or the transaction is rejected for insufficient fee (if the estimate is used as an exact bound), depending on the client's fee-setting strategy.

        if let Some(pending_data) = &self.maybe_pending_data {
            for DeclaredClassHashEntry { class_hash: other_class_hash, compiled_class_hash } in
                &pending_data.declared_classes
            {
                if class_hash == *other_class_hash {
                    return Ok(*compiled_class_hash);
                }
            }
        }

        let maybe_block_number = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_reader()
            .map_err(storage_err_to_state_err)?
            .get_class_definition_block_number(&class_hash)
            .map_err(storage_err_to_state_err)?;

        // Cairo 0 classes (and undeclared classes) do not have a compiled class hash.
        // According to the trait, return the default value.
        let Some(block_number) = maybe_block_number else {
            return Ok(CompiledClassHash::default());
        };

            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_diff(block_number)
            .map_err(storage_err_to_state_err)?
            .ok_or(StateError::StateReadError(format!(

            )))?;

            .class_hash_to_compiled_class_hash
            .get(&class_hash)
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing class declaration at block {block_number}, class \
                 {class_hash}."
            )))?;

        Ok(*compiled_class_hash)

    fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {

pub fn should_migrate(
    state_reader: &impl StateReader,
    class_hash: ClassHash,
) -> StateResult<Option<(ClassHash, CompiledClassHashV2ToV1)>> {
    let state_compiled_class_hash = state_reader.get_compiled_class_hash(class_hash)?;
    match state_compiled_class_hash {
        // Class hash does not exist in the state, or is a Cairo 0 class.
        CompiledClassHash(hash) if hash == StarkHash::ZERO => Ok(None),
        state_compiled_class_hash => {
            let compiled_class_hash_v2 = state_reader.get_compiled_class_hash_v2(
                class_hash,
                &state_reader.get_compiled_class(class_hash)?,
            )?;
            // If the state compiled class hash is compiled class hash v2, the class should not
            // migrate.
            if state_compiled_class_hash == compiled_class_hash_v2 {
                return Ok(None);
            }
            Ok(Some((class_hash, (compiled_class_hash_v2, state_compiled_class_hash))))
        }
    }
