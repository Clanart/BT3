### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the replacement class hash has been declared on-chain before updating a contract's class. Any contract can call `replace_class` with an arbitrary, undeclared class hash. The OS accepts this state transition without validation, permanently rendering the contract non-executable and freezing any funds it holds.

### Finding Description
In `execute_replace_class`, the OS reads the requested `class_hash` from the syscall request and directly writes it into `contract_state_changes` without checking whether that hash exists in `contract_class_changes` (the declared-class registry). The function signature does not even receive `contract_class_changes` as an implicit argument, making the check structurally impossible at this call site. A developer TODO comment in the code explicitly acknowledges this gap:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
```

The relevant code path is:

```cairo
let class_hash = request.class_hash;
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

By contrast, the `execute_declare_transaction` path enforces `prev_value=0` and validates the class hash via `finalize_class_hash` before writing to `contract_class_changes`, demonstrating that the OS does have the machinery to enforce class-existence invariants — it simply does not apply them in `execute_replace_class`. [2](#0-1) 

### Impact Explanation
When a contract's class is set to an undeclared hash, every subsequent call to that contract fails because the execution engine cannot locate the class bytecode. Any ERC-20 tokens, ETH, or other assets held in the contract's storage become permanently inaccessible — a **Critical: Permanent freezing of funds** outcome. Because the OS proof accepts this state transition as valid, the frozen state is committed to the global state root and cannot be reversed without a protocol-level upgrade. [3](#0-2) 

### Likelihood Explanation
The attack surface is broad. Any contract that exposes a `replace_class` call path — whether intentionally (upgradeable proxy) or through a reentrancy/logic bug — can be exploited. A malicious contract owner can deploy a contract, attract user deposits, and then call `replace_class` with a random felt value to freeze all deposited funds. No privileged role, leaked key, or external dependency is required. The attacker only needs to be a standard contract deployer and transaction sender.

### Recommendation
Add `contract_class_changes: DictAccess*` as an implicit argument to `execute_replace_class` and perform a `dict_read` (or equivalent lookup) to assert that `request.class_hash` is present in the declared-class registry before updating `contract_state_changes`. This mirrors the invariant already enforced in `execute_declare_transaction` where `prev_value=0` guarantees a class is declared at most once. [4](#0-3) 

### Proof of Concept

1. **Attacker deploys** a contract `C` holding user funds (e.g., an ERC-20 vault). The contract's class hash `H_valid` is properly declared.

2. **Attacker crafts** a `replace_class` syscall request with `class_hash = 0xdeadbeef` (an arbitrary felt that has never been declared).

3. **OS execution**: `execute_replace_class` is called with `contract_address = C`. It reads `request.class_hash = 0xdeadbeef`, skips any declared-class check (none exists), and calls:
   ```cairo
   dict_update{dict_ptr=contract_state_changes}(
       key=C,
       prev_value=cast(state_entry, felt),
       new_value=cast(new StateEntry(class_hash=0xdeadbeef, ...), felt),
   );
   ```

4. **State commitment**: `state_update` in `state.cairo` squashes and commits the change. The global state root now reflects `C.class_hash = 0xdeadbeef`.

5. **Funds frozen**: All subsequent transactions targeting `C` fail at class-lookup time. The user funds stored in `C`'s storage are permanently inaccessible. [5](#0-4) [6](#0-5)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
