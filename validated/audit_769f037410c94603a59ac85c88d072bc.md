### Title
`starknet_getStorageAt` for Pending Block Returns `CONTRACT_NOT_FOUND` for Contracts Deployed Only in Pending State — (`File: crates/apollo_rpc/src/v0_8/api/api_impl.rs`)

### Summary

`starknet_getStorageAt` with `block_id = "pending"` performs a contract-existence guard only against the **committed** state reader. A contract deployed in the pending block but not yet committed is invisible to that guard. When the queried storage key resolves to `Felt::ZERO` (key absent from pending storage diffs, or explicitly written as zero), the RPC returns `CONTRACT_NOT_FOUND` instead of the correct value `0`.

### Finding Description

In `get_storage_at` (lines 347–384 of `api_impl.rs`):

1. **Pending storage diffs are fetched** from `pending_data.state_update.state_diff.storage_diffs` when `block_id` is `Pending`.
2. `get_accepted_block_number` for `Tag::Pending` resolves to the **latest committed block number** (block.rs line 129–131: `BlockId::Tag(Tag::Latest | Tag::Pending) => get_latest_block_number(txn)?`).
3. `execution_utils::get_storage_at` checks pending storage diffs first; if the key is absent (or present with value `0`), it falls back to the committed state reader, returning `Felt::ZERO`.
4. The guard at line 375 fires: `if res == Felt::default() && contract_address != BLOCK_HASH_TABLE_ADDRESS`.
5. The guard calls `txn.get_state_reader().get_class_hash_at(state_number, &contract_address)` — **committed state only**. Pending `deployed_contracts` are never consulted.
6. The contract is absent from committed state → `ok_or_else(|| CONTRACT_NOT_FOUND)` fires.

The pending `deployed_contracts` list is read for other endpoints (e.g., `get_class_hash_at` at line 1656–1663 correctly reads `pending_state_diff.deployed_contracts`), but `get_storage_at` omits this check entirely.

### Impact Explanation

`starknet_getStorageAt` is an authoritative RPC endpoint. Returning `CONTRACT_NOT_FOUND` for a contract that **does** exist in the pending block is a wrong value. Clients (wallets, dApps, indexers) that rely on this endpoint to verify contract state during the pending window will incorrectly conclude the contract does not exist, potentially blocking interactions or triggering redundant deploy attempts.

Matches: **High — RPC pending view returns an authoritative-looking wrong value.**

### Likelihood Explanation

The trigger is unprivileged and routine: any `DeployAccount` or `deploy` syscall that lands in the pending block, followed by a `starknet_getStorageAt` query for any storage key that has not been explicitly written to a non-zero value. This is the normal state for a freshly deployed contract (all storage slots default to zero). No special attacker capability is required.

### Recommendation

Before returning `CONTRACT_NOT_FOUND`, also check the pending `deployed_contracts` and `replaced_classes` lists when `block_id` is `Pending`. Concretely, after the committed-state check fails, additionally check whether `contract_address` appears in `pending_data.state_update.state_diff.deployed_contracts` or `replaced_classes`. If found there, return `Ok(Felt::ZERO)` instead of the error. This mirrors the pattern already used in `maybe_get_class_hash_at` (lines 1656–1663).

### Proof of Concept

1. Submit a `DeployAccount` transaction; it lands in the pending block. The contract address `A` now appears in `pending_data.state_update.state_diff.deployed_contracts` but is absent from committed storage.
2. Call `starknet_getStorageAt(A, key=0x1, block_id="pending")` where `key=0x1` has never been written.
3. `execution_utils::get_storage_at` finds no entry for `A` in pending storage diffs → falls back to committed state → returns `Felt::ZERO`.
4. Guard fires: `res == Felt::default()` → checks `txn.get_state_reader().get_class_hash_at(state_number, A)` → `None` (committed state has no record of `A`).
5. RPC returns `CONTRACT_NOT_FOUND` (error code 20) instead of `0`.

pub fn get_storage_at<Mode: TransactionKind>(
    txn: &StorageTxn<'_, Mode>,
    state_number: StateNumber,
    pending_storage_diffs: Option<&IndexMap<ContractAddress, Vec<StorageEntry>>>,
    contract_address: ContractAddress,
    key: StorageKey,
) -> StorageResult<Felt> {
    if let Some(pending_storage_diffs) = pending_storage_diffs {
        if let Some(storage_entries) = pending_storage_diffs.get(&contract_address) {
            if let Some(StorageEntry { key: _, value }) = storage_entries
                .iter()
                .find(|StorageEntry { key: other_key, value: _ }| key == *other_key)
            {
                return Ok(*value);
            }
        }
    }
