### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The StarkNet OS `execute_replace_class` syscall handler (both the new and deprecated paths) accepts any arbitrary felt value as the new `class_hash` without verifying that a class with that hash has actually been declared on-chain. This breaks the protocol invariant that every contract's `class_hash` must correspond to a declared class. A contract that replaces its class with an undeclared hash becomes permanently non-executable, permanently freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` reads the requested class hash directly from the syscall request and writes it into `contract_state_changes` with no validation:

```cairo
// syscall_impls.cairo, lines 877–916
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The `TODO` comment explicitly acknowledges the missing check. The same omission exists in the deprecated path:

```cairo
// deprecated_execute_syscalls.cairo, lines 307–329
func execute_replace_class{contract_state_changes: DictAccess*, revert_log: RevertLogEntry*}(
    contract_address, syscall_ptr: ReplaceClass*
) {
    alloc_locals;
    let class_hash = syscall_ptr.class_hash;
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}
    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );
    dict_update{dict_ptr=contract_state_changes}(...);
    ...
}
```

The protocol invariant is: every contract's `class_hash` field in `contract_state_changes` must correspond to a class that exists in `contract_class_changes`. The OS enforces this invariant at deploy time (via `deploy_contract`, which checks `state_entry.class_hash = UNINITIALIZED_CLASS_HASH` before writing the new class), but not at upgrade time via `replace_class`.

This is the direct analog of M-08: just as `ODSafeManager.transferCollateral()` failed to check that the destination address was a registered `SAFEHandler`, the StarkNet OS `execute_replace_class` fails to check that the new class hash is a registered (declared) class.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

If a contract's `class_hash` is set to an undeclared value, the OS will be unable to locate any entry point for that contract in `contract_class_changes`. Every subsequent call to the contract will fail at the class-lookup stage. Any ERC-20 tokens, ETH, or other assets held in the contract's storage become permanently inaccessible — no `transfer`, `withdraw`, or any other function can execute.

This satisfies the "Critical. Permanent freezing of funds." impact category.

---

### Likelihood Explanation

The attack path is reachable by any unprivileged contract deployer:

1. An attacker deploys a contract that accepts user deposits (e.g., a fake vault or staking contract).
2. Users deposit funds into the contract.
3. The attacker's contract calls the `replace_class` syscall with an arbitrary felt value that does not correspond to any declared class.
4. The OS processes the syscall without validation and writes the invalid class hash into `contract_state_changes`.
5. All future calls to the contract revert at class resolution; deposited funds are permanently frozen.

Alternatively, a legitimate contract with an upgrade mechanism that does not validate the new class hash before calling `replace_class` can be exploited by an attacker who can influence the upgrade parameter (e.g., via a governance attack or a bug in the upgrade access control). In that scenario, the OS's missing validation is the last line of defense that fails.

The `replace_class` syscall is a standard, publicly accessible syscall available to any deployed contract. No privileged role or leaked key is required.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, the OS must verify that the class hash exists in `contract_class_changes` (i.e., has been declared). This check should be added in both `execute_replace_class` implementations:

- `syscall_impls.cairo` (new syscall path, line 898)
- `deprecated_execute_syscalls.cairo` (deprecated syscall path, line 311)

The check should assert that `dict_read{dict_ptr=contract_class_changes}(key=class_hash)` returns a non-null/non-zero entry, consistent with how `deploy_contract` enforces that the deployed class hash is known.

---

### Proof of Concept

1. Attacker deploys `VaultContract` with a legitimate class hash. Users deposit 1000 ETH.
2. `VaultContract` calls `replace_class(class_hash=0xdeadbeef)` where `0xdeadbeef` is not declared.
3. The OS executes `execute_replace_class`:
   - Reads `request.class_hash = 0xdeadbeef`.
   - Skips the missing declared-class check (the TODO).
   - Writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes`.
4. Block is proven and committed. `VaultContract`'s on-chain class hash is now `0xdeadbeef`.
5. Any user calling `VaultContract.withdraw()` causes the OS to look up class `0xdeadbeef` in `contract_class_changes` — it does not exist. Execution fails unconditionally.
6. The 1000 ETH is permanently frozen. [1](#0-0) [2](#0-1) [3](#0-2)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    );

        key=contract_address,

    );
