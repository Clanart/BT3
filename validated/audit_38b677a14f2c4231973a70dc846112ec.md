### Title
Missing Declared Class Hash Validation in `execute_replace_class` Allows Permanent Contract Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the new class hash supplied by a contract is actually a declared class before updating the contract's state. This is directly analogous to the Particle M-12 finding: just as Particle allowed a position to be opened against an NFT that the PositionManager did not actually hold (only approval was checked), the StarkNet OS allows a contract to replace its class hash with an arbitrary, undeclared value (only gas is checked). The missing validation is even acknowledged by a developer TODO comment in the code.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
```

The function:
1. Reads `request.class_hash` — a fully attacker-controlled value.
2. Explicitly skips the check (noted by the TODO) that the new class hash corresponds to a declared class in `contract_class_changes`.
3. Unconditionally writes the new (potentially undeclared) class hash into `contract_state_changes` for `contract_address`.

There is no cross-reference against `contract_class_changes` (the dictionary of declared class hashes → compiled class hashes) before committing the state update. The only gate is gas sufficiency. [1](#0-0) 

Compare this to the declare flow in `transaction_impls.cairo`, where `dict_update` on `contract_class_changes` enforces `prev_value=0` (a class may only be declared once, and must exist before use): [2](#0-1) 

No equivalent existence check is performed in `execute_replace_class`.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` in `contract_state_changes` is set to an undeclared hash value, every subsequent call to that contract will fail at the class-lookup stage (the OS cannot find a compiled class for the hash). The contract becomes permanently non-callable. Any ERC-20 tokens, ETH, or other assets held in that contract's storage are irrecoverably frozen, because no entry point (including `transfer`, `withdraw`, etc.) can ever be executed again.

If the OS produces and proves a block containing such a state transition, the resulting on-chain state commitment is final and irreversible.

---

### Likelihood Explanation

**High.** The attack requires only:
1. Deploying a contract (permissionless on StarkNet).
2. That contract issuing a `replace_class` syscall with an arbitrary felt value as the new class hash.

No privileged role, leaked key, or operator cooperation is needed. Any unprivileged contract deployer can trigger this. The developer TODO comment (`// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.`) — with a date now in the past — confirms the check was known to be missing and has not yet been implemented. [3](#0-2) 

---

### Recommendation

Before committing the `dict_update` in `execute_replace_class`, add a lookup into `contract_class_changes` (or the pre-existing declared class set) to assert that `class_hash` maps to a non-zero compiled class hash. Concretely:

```cairo
// Assert the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the existing pattern used during `execute_declare_transaction`, where `prev_value=0` enforces that a class must be newly declared, and ensures that `replace_class` can only point to a class that already exists in the system.

---

### Proof of Concept

1. **Deploy** a Cairo contract `VictimVault` that holds user funds and exposes a `drain` entry point.
2. **Deploy** an attacker contract `Attacker` that calls `replace_class(0xdeadbeef)` — where `0xdeadbeef` is not a declared class hash.
3. **Invoke** `Attacker`. The OS processes `execute_replace_class`:
   - Gas check passes.
   - TODO check is absent.
   - `contract_state_changes[attacker_address].class_hash` is set to `0xdeadbeef`.
4. The OS produces a valid proof for this block. The state commitment on L1 is updated.
5. Any subsequent transaction targeting `attacker_address` fails at class resolution — the contract is permanently bricked.
6. All funds in `attacker_address` storage are permanently frozen.

The same attack applies to any contract the attacker controls that holds funds, including contracts that users have deposited into trustingly.
