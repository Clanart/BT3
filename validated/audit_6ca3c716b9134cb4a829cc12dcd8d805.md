### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS Cairo program accepts any arbitrary `class_hash` value from the caller without verifying that the hash corresponds to a previously declared class in `contract_class_changes`. This is directly analogous to the "Killed Gauge Can Be Voted For" pattern: a check for one condition (the syscall is structurally valid) is performed, but a critical state condition (the target class hash is actually declared) is not. Any contract can call `replace_class` with an undeclared class hash, permanently bricking itself and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall: [1](#0-0) 

The function reads `request.class_hash` and immediately writes it into `contract_state_changes` via `dict_update`, replacing the contract's class hash with the caller-supplied value. The only gas check is performed; there is **no verification** that `class_hash` exists as a key in `contract_class_changes` (i.e., that it was previously declared via a `declare` transaction). The developer-acknowledged TODO at line 898 confirms this check is absent:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
``` [2](#0-1) 

When any subsequent call is made to the bricked contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
``` [3](#0-2) 

Because the class hash was never declared, `dict_read` returns the default value `0` (`UNINITIALIZED_CLASS_HASH`). The OS then calls `find_element` searching for a compiled class with hash `0`: [4](#0-3) 

No compiled class with hash `0` exists (it is the sentinel `UNINITIALIZED_CLASS_HASH` value defined in `commitment.cairo`): [5](#0-4) 

`find_element` is a non-optional search that panics on failure, making the contract permanently uncallable. All funds held by the contract are irrecoverably frozen.

The `declare` transaction enforces `prev_value=0` and `new_value=compiled_class_hash` with `assert_not_zero(compiled_class_hash)`, so only non-zero compiled class hashes are ever legitimately registered: [6](#0-5) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with an undeclared value, every future call to that contract causes the OS to look up a compiled class hash of `0`, which does not exist in the compiled class facts bundle. The contract becomes permanently inaccessible. Any ERC-20 balances, ETH, or other assets held in the contract's storage are irreversibly frozen. The state transition is committed to the global state root and cannot be undone.

---

### Likelihood Explanation

The `replace_class` syscall is available to any deployed contract without privilege restrictions. An attacker can:

1. Deploy a contract that calls `replace_class(arbitrary_undeclared_hash)`.
2. Induce users to deposit funds into the contract (e.g., by presenting it as a legitimate DeFi vault).
3. Trigger the `replace_class` call with an undeclared hash.
4. All deposited funds are permanently frozen.

Alternatively, a buggy or malicious contract upgrade path could accidentally supply an undeclared hash. The OS provides no safety net. The attacker-controlled entry path is a standard user-submitted transaction — no privileged access is required.

---

### Recommendation

In `execute_replace_class`, before writing the new class hash into `contract_state_changes`, verify that the requested `class_hash` has a non-zero entry in `contract_class_changes` (i.e., it was previously declared). This mirrors the check that `execute_entry_point` implicitly relies on:

```cairo
// After gas reduction, before dict_update:
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This resolves the acknowledged TODO at line 898 and closes the state-transition bypass.

---

### Proof of Concept

1. Declare class `A` (legitimate). Deploy contract `C` using class `A`. Users deposit funds into `C`.
2. Attacker submits an invoke transaction that calls `C.__execute__`, which internally calls the `replace_class` syscall with `new_class_hash = 0xdeadbeef` (never declared).
3. `execute_replace_class` in the OS accepts this: gas is deducted, `contract_state_changes[C].class_hash` is updated to `0xdeadbeef`, no declared-class check is performed.
4. The block is committed. The global state root now encodes `C` with class hash `0xdeadbeef`.
5. Any subsequent transaction calling `C` causes `execute_entry_point` to call `dict_read(contract_class_changes, 0xdeadbeef)` → returns `0`. Then `find_element(..., key=0)` panics. The OS cannot prove any block containing a call to `C`.
6. The sequencer excludes all calls to `C` to keep the network alive, but `C`'s storage (and all funds) are permanently inaccessible — **permanent freezing of funds**.
