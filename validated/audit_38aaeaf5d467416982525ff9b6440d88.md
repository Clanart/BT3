### Title
Stale Compiled Class Hash Returned by `ExecutionStateReader::get_compiled_class_hash` After Migration Causes Wrong RPC Simulation and Fee Estimation — (`crates/apollo_rpc_execution/src/state_reader.rs`)

---

### Summary

After a compiled class hash migration (v1 → v2) is committed to storage, `ExecutionStateReader::get_compiled_class_hash` returns the stale pre-migration v1 hash. It reads the state diff at the original **declaration block** rather than using the cursor-based lookup that returns the most recent hash. This causes `should_migrate` to incorrectly flag the class as still needing migration during RPC simulation, producing wrong simulated state diffs and wrong fee estimates.

---

### Finding Description

`ExecutionStateReader::get_compiled_class_hash` resolves a class's compiled class hash in three steps:

1. Call `get_class_definition_block_number(&class_hash)` to find the block where the class was **declared** (block M).
2. Call `get_state_diff(block_number)` to read the state diff at block M.
3. Look up `state_diff.class_hash_to_compiled_class_hash.get(&class_hash)`. [1](#0-0) 

When a compiled class hash migration occurs, `ThinStateDiff::from_state_diff` merges `migrated_compiled_classes` into `class_hash_to_compiled_class_hash` for the **migration block** (block N > M): [2](#0-1) 

The storage writer records the migration at block N in the `compiled_class_hash_table` as `(class_hash, N) → compiled_class_hash_v2`: [3](#0-2) 

The cursor-based `get_compiled_class_hash_at` correctly returns the most recent entry (v2 after migration): [4](#0-3) 

`ApolloReader::get_compiled_class_hash` (used by the main sequencer path) correctly calls `get_compiled_class_hash_at`: [5](#0-4) 

But `ExecutionStateReader::get_compiled_class_hash` reads the state diff at the **declaration block M**, which still contains the old v1 hash. The migration block N's state diff is never consulted.

After migration:
- `ApolloReader::get_compiled_class_hash(class_hash)` → v2 (correct)
- `ExecutionStateReader::get_compiled_class_hash(class_hash)` → v1 (stale)

This discrepancy directly feeds `should_migrate`: [6](#0-5) 

Because `state_compiled_class_hash` (v1, stale) ≠ `compiled_class_hash_v2` (v2, current), `should_migrate` returns `Some(...)` — migration needed — even though migration already occurred. The simulation then includes spurious migration entries in the state diff.

The migration entries inflate `state_diff_length`, which feeds `concat_counts` in the block hash commitment: [7](#0-6) 

This causes `starknet_estimateFee` and `starknet_simulateTransactions` to return wrong gas costs and wrong state diffs.

Additionally, the transaction prover's `fetch_class` uses `get_compiled_class_hash_v2` via `StateReaderAndContractManager`, but the RPC simulation path uses `ExecutionStateReader` which returns the wrong v1 hash, causing the proof input to bind the wrong compiled class hash: [8](#0-7) 

---

### Impact Explanation

**High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

After a compiled class hash migration is committed, any `starknet_estimateFee` or `starknet_simulateTransactions` call for a transaction that touches the migrated class will:
- Include spurious migration entries in the simulated state diff (wrong `class_hash_to_compiled_class_hash` entries)
- Return inflated fee estimates because `state_diff_length` is overcounted
- Return a wrong `state_diff_commitment` in the simulated block header commitments

---

### Likelihood Explanation

Requires `enable_casm_hash_migration` (a `VersionedConstants` flag) to be true and at least one compiled class hash migration to have been committed to storage. The migration feature is explicitly designed for production use and is tested end-to-end: [9](#0-8) 

Once migration is enabled and any class is migrated, every subsequent RPC simulation touching that class is affected.

---

### Recommendation

Replace the declaration-block state diff lookup in `ExecutionStateReader::get_compiled_class_hash` with the cursor-based `get_compiled_class_hash_at(self.state_number, &class_hash)`, consistent with `ApolloReader`:

```rust
fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    if let Some(pending_data) = &self.maybe_pending_data {
        for DeclaredClassHashEntry { class_hash: other_class_hash, compiled_class_hash }
            in &pending_data.declared_classes
        {
            if class_hash == *other_class_hash {
                return Ok(*compiled_class_hash);
            }
        }
    }
    // Use cursor-based lookup to get the most recent compiled class hash,
    // including any post-declaration migration updates.
    match self.storage_reader
        .begin_ro_txn().map_err(storage_err_to_state_err)?
        .get_state_reader().map_err(storage_err_to_state_err)?
        .get_compiled_class_hash_at(self.state_number, &class_hash)
        .map_err(storage_err_to_state_err)?
    {
        Some(h) => Ok(h),
        None => Ok(CompiledClassHash::default()),
    }
}
```

---

### Proof of Concept

1. Declare a Cairo1 contract at block M with compiled class hash v1. Storage records `(class_hash, M) → v1`.
2. Execute a transaction that triggers migration at block N. Storage records `(class_hash, N) → v2`. The migration block's `ThinStateDiff.class_hash_to_compiled_class_hash` contains `{class_hash → v2}`.
3. Call `starknet_estimateFee` for a transaction that invokes the migrated contract.
4. `ExecutionStateReader::get_compiled_class_hash(class_hash)`:
   - `get_class_definition_block_number` → M
   - `get_state_diff(M).class_hash_to_compiled_class_hash.get(class_hash)` → v1 (stale)
5. `should_migrate` compares v1 (stale) with v2 (current) → returns `Some(class_hash, (v2, v1))`.
6. The simulated state diff gains a spurious `{class_hash → v2}` migration entry.
7. `state_diff_length` is overcounted by 1, inflating the fee estimate.
8. The returned `state_diff_commitment` does not match what the sequencer would actually produce.

fn get_compiled_class_hash_at<'env, Mode: TransactionKind>(
    first_irrelevant_block: BlockNumber,
    class_hash: &ClassHash,
    txn: &'env DbTransaction<'env, Mode>,
    compiled_class_hash_table: &'env CompiledClassHashTable<'env>,
) -> StorageResult<Option<CompiledClassHash>> {
    let db_key = (*class_hash, first_irrelevant_block);
    // Find the previous db item.
    let mut cursor = compiled_class_hash_table.cursor(txn)?;
    cursor.lower_bound(&db_key)?;
    let res = cursor.prev()?;
    match res {
        None => Ok(None),
        Some(((got_class_hash, _got_block_number), value)) => {
            if got_class_hash != *class_hash {
                // The previous item belongs to different class hash, which means there is no
                // previous state diff for this item.
                return Ok(None);
            };
            // The previous db item indeed belongs to this address and key.
            Ok(Some(value))
        }
    }
