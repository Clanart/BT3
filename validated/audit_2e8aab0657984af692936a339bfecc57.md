### Title
`From<CommitmentStateDiff> for ThinStateDiff` Silently Drops `deprecated_declared_classes`, Producing Wrong `state_diff_commitment` and `concatenated_counts` in Block Hash - (File: crates/blockifier/src/state/cached_state.rs)

---

### Summary

The `From<CommitmentStateDiff> for ThinStateDiff` conversion in `crates/blockifier/src/state/cached_state.rs` unconditionally sets `deprecated_declared_classes: Vec::new()`. This `ThinStateDiff` is the sole input to `calculate_state_diff_hash` and `ThinStateDiff::len()` (used for `concatenated_counts`) inside `BlockExecutionArtifacts::new()`. For any block that contains a deprecated (Cairo 0) class declaration, both the `state_diff_commitment` field and the `state_diff_length` component of `concatenated_counts` in the block hash will be computed over an incomplete state diff, producing a wrong `PartialBlockHashComponents` and ultimately a wrong block hash.

---

### Finding Description

`CommitmentStateDiff` (produced by the blockifier from `StateMaps`) has no field for deprecated class declarations:

```rust
pub struct CommitmentStateDiff {
    pub address_to_class_hash: IndexMap<ContractAddress, ClassHash>,
    pub address_to_nonce: IndexMap<ContractAddress, Nonce>,
    pub storage_updates: IndexMap<ContractAddress, IndexMap<StorageKey, Felt>>,
    pub class_hash_to_compiled_class_hash: IndexMap<ClassHash, CompiledClassHash>,
}
``` [1](#0-0) 

The `From` implementation that converts it to `ThinStateDiff` therefore hardcodes the field to an empty vector, with a TODO acknowledging the structural gap:

---

### Impact Explanation

For any block that includes a deprecated (Cairo 0) `DECLARE` transaction:

1. **`state_diff_commitment`** is computed with `deprecated_declared_classes = []` instead of the actual declared class hashes. The Poseidon hash over the state diff is wrong.
2. **`concatenated_counts`** encodes `state_diff_length = ThinStateDiff::len()` which is also short by the number of deprecated declarations.
3. Both wrong values are chained into `calculate_block_hash`, producing a wrong `PartialBlockHash` / final block hash.
4. The SNOS program independently computes the state diff commitment with the correct deprecated classes (supplied via hints). The proof will fail to verify because the sequencer's committed `state_diff_commitment` disagrees with the OS-computed value.
5. The `ThinStateDiff` written to storage via `append_state_diff` also has `deprecated_declared_classes: Vec::new()`, so any RPC or sync consumer reading `starknet_getStateUpdate` for that block will receive an incomplete state diff.

This matches **Critical – Wrong state/receipt/event/class hash from blockifier/execution logic for accepted input** and **High – RPC returns an authoritative-looking wrong value**.

---

### Likelihood Explanation

Deprecated (Cairo 0) `DECLARE` transactions (v0/v1) are valid Starknet transactions. The storage layer, central sync, and P2P sync all handle `deprecated_declared_classes` correctly, confirming the sequencer is expected to process them. The `native_blockifier` path explicitly populates `deprecated_declared_classes` via `ThinStateDiff::from_state_diff(state_diff)`. Only the Rust batcher path uses the lossy `From<CommitmentStateDiff>` conversion. Any user who submits a deprecated declare transaction that is accepted and sequenced triggers the wrong commitment.

---

### Recommendation

`CommitmentStateDiff` (and the underlying `StateMaps`) must be extended to track deprecated class declarations, mirroring how `ThinStateDiff::from_state_diff` populates `deprecated_declared_classes` from `StateDiff::deprecated_declared_classes`. The `From<CommitmentStateDiff> for ThinStateDiff` conversion and the `BlockExecutionArtifacts::thin_state_diff()` method must propagate the real set of deprecated class hashes instead of `Vec::new()`. The TODO comment at line 698 of `cached_state.rs` marks this as a known structural gap that must be resolved before the commitment path is correct.

---

### Proof of Concept

1. Submit a valid deprecated `DECLARE` transaction (v0/v1, Cairo 0 class) to the sequencer gateway.
2. The batcher includes it in a block. The blockifier executes it; `StateMaps` records no entry for the deprecated class declaration (no `compiled_class_hashes` entry, no `class_hashes` entry for the class itself).
3. `BlockExecutionArtifacts::new()` calls `ThinStateDiff::from(commitment_state_diff.clone())`. The resulting `ThinStateDiff` has `deprecated_declared_classes: []`.
4. `calculate_state_diff_hash` chains `0` (length) for `deprecated_declared_classes` instead of `1` (the actual class hash). The `StateDiffCommitment` value is wrong.
5. `ThinStateDiff::len()` returns `N` instead of `N+1`; `concatenated_counts` encodes the wrong state diff length.
6. `calculate_block_hash` produces a block hash that differs from what the SNOS Cairo program computes for the same block, causing proof verification failure.
7. `starknet_getStateUpdate` for the block returns a `ThinStateDiff` with an empty `deprecated_declared_classes` list, an authoritative-looking wrong value. [2](#0-1) [3](#0-2) [9](#0-8)

    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =

    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
