### Title
Stale `feemarketParams` Captured at Construction Time Used in Fee Validation, Bypassing Current EIP-1559 Base Fee — (File: `ante/evm/fee_checker.go`)

---

### Summary

`NewDynamicFeeChecker` captures `feemarketParams *feemarkettypes.Params` at construction time and uses it to derive the base fee for every subsequent Cosmos SDK transaction fee validation. Because `SetBaseFee` only updates the KV store — never the in-memory pointer — the fee checker permanently validates against the initial (stale) base fee rather than the current block's base fee. This is the direct structural analog of the reported "health check on outdated state" vulnerability: a critical validation is performed against state that is never refreshed.

---

### Finding Description

`feemarketParams` is a pointer to a `Params` struct fetched once at app startup. The actual base fee is updated every block in `x/feemarket/keeper/abci.go`:

```go
func (k *Keeper) BeginBlock(ctx sdk.Context) error {
    baseFee := k.CalculateBaseFee(ctx)
    ...
    k.SetBaseFee(ctx, baseFee)
``` [2](#0-1) 

`SetBaseFee` writes the new base fee into the KV store by marshalling a new `Params` struct:

```go
func (k Keeper) SetBaseFee(ctx sdk.Context, baseFee *big.Int) {
    params := k.GetParams(ctx)
    params.BaseFee = ethermint.SaturatedNewInt(baseFee)
    err := k.SetParams(ctx, params)
``` [3](#0-2) 

This creates a new local `params` copy and stores it in the KV store. It never mutates the in-memory struct that `feemarketParams` points to. After the first block where the base fee changes, the pointer held by `NewDynamicFeeChecker` is permanently stale.

**Contrast with the EVM ante handler**, which correctly fetches the base fee fresh on every transaction:

```go
blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
...
baseFee := blockCfg.BaseFee
``` [4](#0-3) 

EVM transactions always validate against the live base fee. Cosmos SDK transactions validated by `NewDynamicFeeChecker` (governance, staking, IBC, etc.) validate against the initial base fee captured at startup — a permanent divergence.

---

### Impact Explanation

Any Cosmos SDK transaction routed through `NewDynamicFeeChecker` can be submitted with a fee above the stale initial base fee but below the current live base fee and will pass fee validation. If the base fee has risen significantly since app startup (e.g., from 1 Gwei to 100 Gwei under sustained high load), an attacker can include Cosmos transactions at a fraction of the required cost. This is a fee market ante handler bug that permits transactions with insufficient fees to commit, directly mis-accounting user fees and bypassing the EIP-1559 congestion-control mechanism for the Cosmos transaction path.

This matches the allowed impact: **High — fee market / ante handler bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted.**

---

### Likelihood Explanation

The base fee changes every block via `BeginBlock`. After the very first block where `CalculateBaseFee` returns a value different from the initial `params.BaseFee`, the pointer held by `NewDynamicFeeChecker` is stale. No special conditions are required: any user submitting a Cosmos transaction with a fee between the stale initial base fee and the current live base fee will exploit this. The entry path is fully unprivileged (standard Cosmos SDK transaction submission).

---

### Recommendation

Replace the captured `feemarketParams *feemarkettypes.Params` parameter with a keeper interface that allows `NewDynamicFeeChecker` to fetch the current params from the KV store on every invocation, mirroring the pattern used in `newEthAnteHandler` where `EVMBlockConfig` is called fresh per transaction. Alternatively, read `feemarketParams` from the store inside the returned closure using the live `ctx`.

---

### Proof of Concept

1. App starts; `feemarketParams.BaseFee = 1_000_000_000` (1 Gwei). `NewDynamicFeeChecker` captures this pointer.
2. Over many blocks of high gas usage, `BeginBlock` → `CalculateBaseFee` → `SetBaseFee` raises the KV-store base fee to `100_000_000_000` (100 Gwei).
3. The `feemarketParams` pointer still holds `BaseFee = 1_000_000_000` (stale).
4. Attacker submits a Cosmos governance `MsgVote` with `fee = 2_000_000_000` (2 Gwei).
5. `NewDynamicFeeChecker` calls `types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)` → returns `1_000_000_000` (stale).
6. Check: `feeCap (2 Gwei) >= baseFeeInt (1 Gwei)` → **passes**.
7. Transaction is included in the block with fees 50× below the current required base fee, bypassing the EIP-1559 fee market entirely for the Cosmos transaction path. [5](#0-4) [6](#0-5) [7](#0-6) [8](#0-7)

		}

		if ctx.BlockHeight() == 0 {
			// genesis transactions: fallback to min-gas-price logic
			return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
		}

		denom := evmParams.EvmDenom

		baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
		if baseFee == nil {
			// london hardfork is not enabled: fallback to min-gas-prices logic
			return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
		}

		// default to `MaxInt64` when there's no extension option.
		maxPriorityPrice := sdkmath.NewInt(math.MaxInt64)

		// get the priority tip cap from the extension option.
		if hasExtOptsTx, ok := tx.(authante.HasExtensionOptionsTx); ok {

				if extOpt, ok := opt.GetCachedValue().(*ethermint.ExtensionOptionDynamicFeeTx); ok {

func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
}

func (k Keeper) GetBaseFee(ctx sdk.Context) *big.Int {
	params := k.GetParams(ctx)
	if params.NoBaseFee {
		return nil
	}

	baseFee := params.BaseFee.BigInt()
	if baseFee == nil || baseFee.Sign() == 0 {
		// try v1 format
		return k.GetBaseFeeV1(ctx)
	}
	return baseFee
}

// SetBaseFee set's the base fee in the store
func (k Keeper) SetBaseFee(ctx sdk.Context, baseFee *big.Int) {

	err := k.SetParams(ctx, params)
	if err != nil {
		return
	}
