### Title
Missing Declared-Class Validation in `execute_replace_class` Enables Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS does not verify that the new class hash supplied in a `replace_class` syscall corresponds to a declared contract class. This is a direct state-transition bypass: the OS unconditionally commits a class-hash update to the global state without checking whether the target class exists. Any contract can replace its class with an arbitrary undeclared hash — including `UNINITIALIZED_CLASS_HASH = 0` — permanently rendering itself uncallable and freezing all funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` processes the `replace_class` syscall. After deducting gas, it reads `class_hash` from the request and directly updates the contract's `StateEntry` with no validation that the new class hash is present in `contract_class_changes` (i.e., that it was ever declared). The code itself contains an explicit TODO acknowledging this missing check:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

This is the direct analog of the Liquity trove state-transition bypass. In the Liquity bug, `adjustTrove()` was called without checking whether the trove was in the `active` state — the only valid state for that operation. Here, `execute_replace_class` updates the contract's class hash without checking whether the target class is in the `declared` state — the only valid state for a class hash stored in contract state. The OS accepts the state transition unconditionally and commits the invalid class hash to the global state root.

For comparison, the `execute_declare_transaction` path correctly enforces `prev_value=0` to prevent double-declaration:

```cairo
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [2](#0-1) 

But `execute_replace_class` performs no symmetric check that the *destination* class hash is declared. The `contract_class_changes` dict, which maps class hash → compiled class hash, is never consulted.

The `deploy_contract` path does enforce a state check — it asserts `state_entry.class_hash = UNINITIALIZED_CLASS_HASH` before deploying — showing the pattern is known and applied elsewhere: [3](#0-2) 

The omission in `execute_replace_class` is therefore an inconsistency in the OS's own state-transition enforcement.

---

### Impact Explanation

Once a contract's class hash is set to an undeclared value (or `UNINITIALIZED_CLASS_HASH = 0`):

1. The OS accepts the state transition and commits it to the state root — the proof is valid from the OS's perspective.
2. The contract's class hash in the global state is permanently invalid.
3. Any subsequent transaction targeting this contract fails because the class does not exist in the class tree.
4. Funds held by the contract are permanently frozen with no recovery path, since the contract cannot execute any entry point (including withdrawal logic).

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

The `replace_class` syscall is reachable by any unprivileged contract deployer. The attack path requires no privileged role:

1. An unprivileged user deploys a contract that accepts deposits from other users (a vault, pool, or escrow).
2. After accumulating third-party funds, the deployer calls a function on the contract that invokes `replace_class(undeclared_hash)`, where `undeclared_hash` is any felt value not present in `contract_class_changes` (e.g., the felt `1`, or `UNINITIALIZED_CLASS_HASH = 0`).
3. The OS executes `execute_replace_class` without consulting `contract_class_changes`.
4. The contract's `StateEntry.class_hash` is permanently set to the invalid value.
5. All deposited user funds are frozen.

Additionally, any existing contract with a logic path that allows an unprivileged caller to trigger `replace_class` with an attacker-controlled argument is exploitable in the same way.

---

### Recommendation

Add a check in `execute_replace_class` to verify that the new class hash corresponds to a declared contract class by reading from `contract_class_changes` and asserting the compiled class hash is non-zero:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the invariant already enforced in `execute_declare_transaction` and is consistent with the existing `deploy_contract` state check.

---

### Proof of Concept

1. Attacker deploys Contract A (a "vault") that accepts token deposits from users.
2. Users deposit funds into Contract A; the contract accumulates a non-zero balance.
3. Attacker calls a function on Contract A that executes `replace_class(1)`, where `1` is a felt value with no corresponding declared class.
4. The OS dispatches to `execute_replace_class`:
   - Reads `class_hash = 1` from the `ReplaceClassRequest`.
   - Skips the declared-class check (the TODO-marked missing validation).
   - Updates Contract A's `StateEntry` to `StateEntry(class_hash=1, storage_ptr=..., nonce=...)`.
   - Writes the update to `contract_state_changes`.
5. The block is proven; the state root is updated with Contract A having class hash `1`.
6. Any future transaction calling Contract A fails: the class `1` is not in the class tree, so no entry point can be dispatched.
7. All user funds in Contract A are permanently frozen with no recovery mechanism. [4](#0-3)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;

}
