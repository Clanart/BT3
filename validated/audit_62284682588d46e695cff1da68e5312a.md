Audit Report

## Title
Unattributed native ETH balance in `pay()` hybrid branch allows any WETH swap to consume prior user's stranded `msg.value` — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary
`PeripheryPayments.pay` reads `address(this).balance` with no per-caller accounting. When a user over-sends `msg.value` and omits `refundETH()`, the residual ETH persists on the router. Any subsequent WETH swap by any user triggers the hybrid branch, silently consuming that residual ETH to fund the new swap. The original sender loses their ETH with no recourse.

## Finding Description
`pay` contains three branches for WETH payment (lines 73–84 of `PeripheryPayments.sol`). The hybrid branch fires when `0 < address(this).balance < value`: it deposits the entire contract ETH balance as WETH and transfers it to the pool, then pulls only the remainder from `payer` via `safeTransferFrom`. `address(this).balance` is a global contract-level value with no per-transaction or per-caller attribution.

ETH is stranded when a user calls any `payable` router entry point (e.g., `multicall{value: 2 ETH}` containing `exactInputSingle(amountIn=1 ETH, tokenIn=WETH)`) without a trailing `refundETH()` call. The first branch of `pay` deposits exactly `value` (1 ETH), leaving 1 ETH on the router. `receive()` blocks direct ETH pushes from non-WETH addresses but is irrelevant here because the ETH arrived via `msg.value`. The transient callback context (`_setNextCallbackContext`, `_clearExpectedCallbackPool`) correctly binds pool and payer for the current swap but places no restriction on which ETH `pay` may consume.

Exploit path:
1. User-A calls `multicall{value: 2 ETH}([exactInputSingle(amountIn=1 ETH, tokenIn=WETH)])` with no `refundETH()`. 1 ETH is stranded on the router.
2. User-B calls `exactInputSingle(tokenIn=WETH, amountIn=1.5 ETH)` with `msg.value=0`.
3. `pay(WETH, userB, pool, 1.5 ETH)` reads `nativeBalance=1 ETH`; hybrid branch fires.
4. Router deposits User-A's 1 ETH as WETH → pool, then pulls only 0.5 ETH of WETH from User-B.
5. User-A loses 1 ETH; User-B's swap input cost is reduced by 1 ETH.

## Impact Explanation
Direct loss of user principal. User-A's stranded ETH is transferred to a pool on behalf of User-B. User-A receives nothing in return. The loss equals the full stranded ETH balance, which can be up to the entire `msg.value` of User-A's transaction. This meets the Sherlock threshold for High severity (direct, quantifiable loss of user funds by an unprivileged third party).

## Likelihood Explanation
Medium. The precondition — ETH stranded on the router — occurs whenever a caller sends excess `msg.value` and omits `refundETH()`. This is a realistic integration mistake: `refundETH()` is opt-in and not enforced by the router. Frontends, aggregators, or users constructing multicalls manually can easily omit it. Once ETH is stranded, any subsequent WETH swap by any user automatically triggers the theft with no special attacker setup required.

## Recommendation
Track per-transaction ETH attribution using transient storage. Record `msg.value` at the start of each top-level call and deduct from it inside `pay`, reverting if the deduction would exceed the recorded value. Alternatively, remove the hybrid branch entirely and require callers to pre-wrap ETH to WETH before calling the router, eliminating the ambiguity at the cost of one extra user step.

function test_residualEthConsumedByLaterSwapper() public {
    // User-A sends 2 ETH; swap needs only 1 ETH; no refundETH()
    vm.prank(userA);
    router.multicall{value: 2 ether}(
        _encodeExactInputSingle(address(weth), token1, 1 ether, userA)
    );
    assertEq(address(router).balance, 1 ether); // 1 ETH stranded

    // User-B swaps 1.5 ETH worth of WETH, sends no msg.value
    uint256 userBWethBefore = weth.balanceOf(userB);
    vm.prank(userB);
    router.exactInputSingle(
        ExactInputSingleParams({
            tokenIn: address(weth),
            amountIn: 1.5 ether,
            // ... other params, msg.value = 0
        })
    );

    // Router is empty; User-A's 1 ETH was consumed for User-B's swap
    assertEq(address(router).balance, 0);
    // User-B only spent 0.5 ETH of their own WETH
    assertEq(weth.balanceOf(userB), userBWethBefore - 0.5 ether);
}
