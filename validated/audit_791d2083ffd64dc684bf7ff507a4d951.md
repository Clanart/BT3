### Title
Non-EVM-Native Cosmos Bank Tokens Permanently Locked as Orphaned Balances After Contract Self-Destruct — (File: `x/evm/statedb/statedb.go`)

### Summary
When a self-destructed EVM contract holds non-EVM-native Cosmos bank tokens (e.g., IBC-transferred tokens, CosmWasm bridge tokens), those tokens are permanently orphaned. `StateDB.Commit()` only burns the EVM denom balance and deletes the auth account, but explicitly skips draining any other bank denominations. The tokens remain in the bank module at the now-deleted address with no recovery path.

### Finding Description
In `StateDB.Commit()`, when processing a self-destructed account, the code reads and burns only the EVM denom balance, then calls `DeleteAccount` to remove auth metadata and storage. The code contains an explicit acknowledgment of the gap:

```go
// Only the EVM denom is burned here. Non-EVM-native tokens (IBC, CosmWasm
// bridge) held by the destroyed address are not drained and may remain as
// orphaned bank balances.
if remaining := s.keeper.GetBalance(cacheCtx, cosmosAddr, s.evmDenom); remaining.Sign() > 0 {
    coin := sdk.NewCoin(s.evmDenom, sdkmath.NewIntFromBigInt(remaining.ToBig()))
    if _, err := s.keeper.SubBalance(cacheCtx, cosmosAddr, coin); err != nil {
        return errorsmod.Wrap(err, "failed to burn post-selfdestruct balance")
    }
}
if err := s.keeper.DeleteAccount(cacheCtx, obj.Address()); err != nil {
    return errorsmod.Wrap(err, "failed to delete account")
}
writeCache()
``` [1](#0-0) 

After `DeleteAccount` removes the auth account and all EVM storage, the Cosmos bank module still holds any non-EVM-denom balances at that address. Because the auth account no longer exists, no EVM call can be made to the address, and no Cosmos-side withdrawal is possible. The tokens are permanently inaccessible.

The `Keeper.DeleteAccount` implementation confirms it only clears EVM storage and removes the auth account — it never touches bank balances: [2](#0-1) 

The `AddBalance` / `SubBalance` keeper methods operate exclusively on the EVM denom: [3](#0-2) 

### Impact Explanation
Any non-EVM-native Cosmos bank token (IBC vouchers, CosmWasm bridge tokens, or any other denom registered in the bank module) held at a contract address that is self-destructed becomes permanently unrecoverable. The bank module retains the balance entry, but the owning auth account is gone and no EVM code exists to call. This is a permanent, irreversible loss of valid user funds — a mis-accounting of Cosmos bank funds caused by the EVM state transition path.

This matches the allowed High impact: **"EVM state transition… bug that permits… valid user funds/fees to be mis-accounted."**

### Likelihood Explanation
The scenario is reachable by any unprivileged user:

1. A contract is deployed that accepts IBC tokens (e.g., via an IBC transfer to the contract address, or via a precompile that routes IBC tokens to the contract).
2. Users or the deployer send non-EVM-denom tokens to the contract.
3. The contract self-destructs — either via a `SELFDESTRUCT` opcode in a pre-EIP-6780 chain config, or via EIP-6780 (contract created and destroyed in the same transaction).
4. At `StateDB.Commit()`, only the EVM denom is burned; all other bank balances are orphaned.

A malicious deployer can exploit this deliberately: deploy a contract that accepts IBC tokens from users, then self-destruct it, permanently locking all deposited non-EVM tokens. The EIP-6780 path (`SelfDestruct6780` → `SelfDestruct` when `newContract == true`) is fully supported in this codebase: [4](#0-3) 

### Recommendation
In the `selfDestructed` branch of `StateDB.Commit()`, iterate over **all** bank denominations held by the destroyed address (not just the EVM denom) and either burn them or transfer them to a designated recovery address (e.g., the community pool or a governance-controlled module account). The `BankKeeper` interface already exposes the necessary methods (`GetAllBalances` or equivalent iteration). This ensures no Cosmos bank funds are orphaned when a contract self-destructs. [5](#0-4) 

### Proof of Concept

**Attack flow (EIP-6780 path, single transaction):**

```solidity
// Attacker deploys this factory; users send IBC tokens to childAddr before redeployChild is called.
contract AttackFactory {
    function attackInOneTx(bytes32 salt) external payable returns (address childAddr) {
        // 1. Deploy child via CREATE2
        childAddr = create2deploy(salt);
        // 2. Self-destruct child in same tx (EIP-6780: newContract == true → selfDestructed = true)
        IChild(childAddr).destroy();
        // 3. Forward msg.value (EVM denom) to destroyed address — this is burned at commit.
        //    IBC tokens previously sent to childAddr are NOT burned — they are orphaned.
    }
}
```

At `StateDB.Commit()`:
- `obj.selfDestructed == true` → enters the burn branch.
- `s.keeper.GetBalance(cacheCtx, cosmosAddr, s.evmDenom)` → burns only the EVM denom.
- `s.keeper.DeleteAccount(cacheCtx, obj.Address())` → removes auth account and EVM storage.
- IBC token balance at `cosmosAddr` in the bank module: **untouched, permanently orphaned**. [6](#0-5)

func (k *Keeper) AddBalance(ctx sdk.Context, addr sdk.AccAddress, coin sdk.Coin) (uint256.Int, error) {
	coins := sdk.NewCoins(coin)
	prevBalance := k.GetBalance(ctx, addr, coin.Denom)
	if err := k.bankKeeper.MintCoins(ctx, types.ModuleName, coins); err != nil {
		return uint256.Int{}, err
	}
	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, addr, coins); err != nil {
		return uint256.Int{}, err
	}
	return prevBalance, nil
}

func (k *Keeper) SubBalance(ctx sdk.Context, addr sdk.AccAddress, coin sdk.Coin) (uint256.Int, error) {
	coins := sdk.NewCoins(coin)
	prevBalance := k.GetBalance(ctx, addr, coin.Denom)
	if err := k.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, coins); err != nil {
		return uint256.Int{}, err
	}
	if err := k.bankKeeper.BurnCoins(ctx, types.ModuleName, coins); err != nil {
		return uint256.Int{}, err
	}
	return prevBalance, nil

func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
	cosmosAddr := sdk.AccAddress(addr.Bytes())
	acct := k.accountKeeper.GetAccount(ctx, cosmosAddr)
	if acct == nil {
		return nil
	}

	// NOTE: only Ethereum accounts (contracts) can be selfdestructed
	_, ok := acct.(ethermint.EthAccountI)
	if !ok {
		return errorsmod.Wrapf(types.ErrInvalidAccount, "type %T, address %s", acct, addr)
	}

	// clear storage
	k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool {
		k.SetState(ctx, addr, key, nil)
		return true
	})

	// remove auth account
	k.accountKeeper.RemoveAccount(ctx, acct)

	k.debugLog(ctx, "account suicided",
		"ethereum-address", addr,
		"cosmos-address", cosmosAddr,
	)

	return nil
