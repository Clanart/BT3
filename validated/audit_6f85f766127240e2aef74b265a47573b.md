### Title
`ProofOfInclusion::valid()` Final Root-Hash Check Is a Tautology — Forged Inclusion Proof Always Passes - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` contains a loop that verifies each layer's hash is internally consistent with the previous layer, then ends with a final check `existing_hash == self.root_hash()`. Because `root_hash()` is derived from the proof's own last `combined_hash` — the same value that `existing_hash` was just set to inside the loop — the final check is a tautology that is always `true` after the loop passes. The function therefore never validates the proof against any external trusted root. An attacker can construct a `ProofOfInclusion` with an arbitrary `node_hash` and a self-consistent chain of fake layers, and `valid()` will return `true` even though the proof's root does not match the actual DataLayer tree root.

### Finding Description

In `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`, the `valid()` method iterates over `self.layers`, computing `calculated_hash` from the running `existing_hash` and each layer's `other_hash`, then checks `calculated_hash != layer.combined_hash`. At the end of each iteration it sets `existing_hash = calculated_hash`. [1](#0-0) 

After the loop exits normally, `existing_hash` equals `calculated_hash` from the last iteration, which the loop already verified equals `last.combined_hash`. The final line:

```rust
existing_hash == self.root_hash()
```

calls `root_hash()`, which returns:

```rust
if let Some(last) = self.layers.last() {
    last.combined_hash   // ← same value existing_hash was just set to
} else {
    self.node_hash
}
``` [2](#0-1) 

This is structurally identical to the external report's "missing bracket" class: the algorithm's final verification step is a no-op. The loop body correctly checks internal consistency, but the concluding guard that should compare against a **trusted external root** is replaced by a self-referential comparison that is always `true`.

The struct is fully constructable from Python via `__new__`, deserializable via `from_bytes` (Streamable), and `valid()` is exposed as a Python binding `py_valid`. [3](#0-2) [4](#0-3) 

### Impact Explanation

An attacker can craft a `ProofOfInclusion` with:
- `node_hash` = hash of a fake leaf for key K (not present in the real tree)
- `layers[i].other_hash` = arbitrary bytes
- `layers[i].combined_hash` = `calculate_internal_hash(prev_hash, side, other_hash)` (self-consistent)

This proof passes `valid()` unconditionally, yet `root_hash()` returns a value that does not match the actual DataLayer tree root. Any consumer that calls only `proof.valid()` — without separately asserting `proof.root_hash() == trusted_root` — will accept a forged inclusion proof for a key-value pair that does not exist in the tree.

This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion, letting untrusted input prove invalid state.**

### Likelihood Explanation

The `valid()` method is the primary, self-contained API for proof validation. Its name implies completeness. The fuzz target and all internal tests call only `proof.valid()` without an external root comparison: [5](#0-4) [6](#0-5) 

Any downstream Python consumer following the same pattern is vulnerable. The attack requires no privileges — only the ability to construct a `ProofOfInclusion` object and call `valid()`.

### Recommendation

`valid()` must accept a trusted root as a parameter and compare against it, or the final line must be changed to compare `existing_hash` against a caller-supplied trusted root rather than the proof's own `root_hash()`. The corrected signature should be:

```rust
pub fn valid(&self, trusted_root: &Hash) -> bool {
    // ... loop unchanged ...
    existing_hash == *trusted_root
}
```

All call sites (Python bindings, fuzz targets, tests) must be updated to supply the trusted root obtained from the authoritative `MerkleBlob`.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
from chia_rs.datalayer import calculate_internal_hash  # or equivalent

# Fake leaf hash for a key NOT in the tree
fake_node_hash = bytes([0xAA] * 32)
fake_other_hash = bytes([0xBB] * 32)

# Build a self-consistent single-layer proof
combined = calculate_internal_hash(fake_node_hash, 0, fake_other_hash)
layer = ProofOfInclusionLayer(
    other_hash_side=0,
    other_hash=fake_other_hash,
    combined_hash=combined,
)
proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

# valid() returns True even though this proof has nothing to do with the real tree
assert proof.valid()                        # passes — tautology
assert proof.root_hash() != real_tree_root  # root mismatch never checked
```
