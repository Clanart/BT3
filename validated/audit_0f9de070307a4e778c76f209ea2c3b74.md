### Title
Missing State Diff Commitment Verification in P2P Sync Allows Malicious Peer to Corrupt Stored State - (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

### Summary

`StateDiffStreamBuilder::parse_data_for_block` assembles a `ThinStateDiff` from peer-supplied `StateDiffChunk` messages and writes it to storage via `append_state_diff` without ever verifying that `calculate_state_diff_hash(assembled_diff) == header.state_diff_commitment`. The only guard is a length check against `header.state_diff_length`. A malicious peer can craft chunks whose union has the correct `len()` but different field values, causing wrong `storage_diffs`, `nonces`, and `class_hash_to_compiled_class_hash` entries to be committed to the state tables.

### Finding Description

`parse_data_for_block` reads exactly one field from the stored header:

```rust
let target_state_diff_len = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    ...
    .state_diff_length   // ← only field read
``` [1](#0-0) 

It then accumulates chunks until `current_state_diff_len == target_state_diff_len`, checking only for empty chunks and duplicate keys via `unite_state_diffs`: [2](#0-1) 

`state_diff_commitment` is **never read** from the header and **never compared** against `calculate_state_diff_hash(&result)`. After the loop, the assembled diff is returned as `Ok(Some((result, block_number)))` and written unconditionally:

```rust
storage_writer.begin_rw_txn()?.append_state_diff(self.1, self.0)?.commit()?;
``` [3](#0-2) 

`append_state_diff` in storage performs no commitment check either — it writes directly to `state_diffs`, `nonces`, `deployed_contracts`, `contract_storage`, and `compiled_class_hash` tables: [4](#0-3) 

The commitment check exists only in the **batcher/committer path** (`apollo_committer`), and even there it is gated behind a config flag `verify_state_diff_hash`: [5](#0-4) 

That path is entirely separate from the P2P sync write path.

### Impact Explanation

Once `append_state_diff` commits, every downstream consumer reads the corrupted tables:
- `starknet_getStateUpdate` calls `get_state_diff` which reads from the `state_diffs` mmap file written by `append_state_diff`.
- The execution state reader reads `contract_storage`, `nonces`, and `deployed_contracts` tables — all written by the same call.

The corrupted values are indistinguishable from legitimate sequencer-produced state: the header's `state_diff_commitment` says one thing, but the stored diff says another, and no runtime check ever reconciles them.

**Impact: High** — RPC `starknet_getStateUpdate` and execution state reader return authoritative-looking wrong storage/nonce/class values for any block whose state diff was supplied by the malicious peer.

### Likelihood Explanation

Any peer that can respond to a `StateDiffQuery` can exploit this. The attacker only needs to:
1. Know `state_diff_length` for the target block (public from the header).
2. Craft `StateDiffChunk` messages whose `len()` values sum to exactly `state_diff_length`, with no duplicate keys, but with arbitrary wrong field values.

No privileged access is required.

### Recommendation

After the accumulation loop in `parse_data_for_block`, read `state_diff_commitment` from the stored header and assert:

```rust
let expected_commitment = header.state_diff_commitment; // already stored
let actual_commitment = calculate_state_diff_hash(&result);
if actual_commitment != expected_commitment {
    return Err(ParseDataError::BadPeer(BadPeerError::StateDiffCommitmentMismatch {
        expected: expected_commitment,
        actual: actual_commitment,
        block_number,
    }));
}
```

This mirrors the guard already present in `apollo_committer` and closes the gap in the P2P sync path.

### Proof of Concept

Precondition: store a header for block 0 with `state_diff_length = 1` and `state_diff_commitment = calculate_state_diff_hash(correct_diff)` where `correct_diff` has one storage entry `(addr_A, key_K) = V_correct`.

Attack: peer responds with a single `StateDiffChunk::ContractDiff` containing `(addr_A, key_K) = V_wrong`. This chunk has `len() == 1`, passes the empty-chunk and duplicate-key checks, and is accepted.

After `write_to_storage`:
- `get_state_diff(BlockNumber(0))` returns a diff with `V_wrong`.
- `calculate_state_diff_hash(stored_diff) != header.state_diff_commitment`.
- `starknet_getStateUpdate` and the execution state reader serve `V_wrong` as the authoritative storage value for `(addr_A, key_K)`. [6](#0-5)

    fn parse_data_for_block<'a>(
        state_diff_chunks_response_manager: &'a mut ClientResponsesManager<
            DataOrFin<StateDiffChunk>,
        >,
        block_number: BlockNumber,
        storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            let mut result = ThinStateDiff::default();
            let mut prev_result_len = 0;
            let mut current_state_diff_len = 0;
            let target_state_diff_len = storage_reader
                .begin_ro_txn()?
                .get_block_header(block_number)?
                .expect("A header with number lower than the header marker is missing")
                .state_diff_length
                .ok_or(P2pSyncClientError::OldHeaderInStorage {
                    block_number,
                    missing_field: "state_diff_length",
                })?;

            while current_state_diff_len < target_state_diff_len {
                let maybe_state_diff_chunk = state_diff_chunks_response_manager
                    .next()
                    .await
                    .ok_or(ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }))?;
                let Some(state_diff_chunk) = maybe_state_diff_chunk?.0 else {
                    if current_state_diff_len == 0 {
                        return Ok(None);
                    } else {
                        return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                            expected_length: target_state_diff_len,
                            possible_lengths: vec![current_state_diff_len],
                        }));
                    }
                };
                prev_result_len = current_state_diff_len;
                if state_diff_chunk.is_empty() {
                    return Err(ParseDataError::BadPeer(BadPeerError::EmptyStateDiffPart));
                }
                // It's cheaper to calculate the length of `state_diff_part` than the length of
                // `result`.
                current_state_diff_len += state_diff_chunk.len();
                unite_state_diffs(&mut result, state_diff_chunk)?;
            }

            if current_state_diff_len != target_state_diff_len {
                return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                    expected_length: target_state_diff_len,
                    possible_lengths: vec![prev_result_len, current_state_diff_len],
                }));
            }

            validate_deprecated_declared_classes_non_conflicting(&result)?;
            Ok(Some((result, block_number)))
        }
        .boxed()
    }

        let declared_classes_block_table = self.open_table(&self.tables.declared_classes_block)?;
        let deprecated_declared_classes_block_table =
            self.open_table(&self.tables.deprecated_declared_classes_block)?;
        let compiled_class_hash_table = self.open_table(&self.tables.compiled_class_hash)?;

        // Write state.
        write_deployed_contracts(

            &self.txn,
            block_number,
            &deployed_contracts_table,
            &nonces_table,
        )?;
        write_storage_diffs(
            &thin_state_diff.storage_diffs,
            &self.txn,
            block_number,
            &storage_table,
        )?;
        // Must be called after write_deployed_contracts since the nonces are updated there.
        write_nonces(&thin_state_diff.nonces, &self.txn, block_number, &nonces_table)?;

        for (class_hash, _) in &thin_state_diff.class_hash_to_compiled_class_hash {
            let not_declared = declared_classes_block_table.get(&self.txn, class_hash)?.is_none();
            if not_declared {

            }
