### Title
`ProofOfInclusion::valid()` Does Not Verify Against a Trusted Root Hash, Enabling Forged Inclusion Proofs - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` only checks the internal self-consistency of the proof chain. Its final assertion `existing_hash == self.root_hash()` is a tautology — it always evaluates to `true` when the loop completes — because `root_hash()` is derived from the proof's own last `combined_hash`, not from any externally-supplied trusted root. No caller-supplied trusted root hash is ever compared. An attacker can construct a fully self-consistent `ProofOfInclusion` for any arbitrary `node_hash` anchored to any arbitrary root, and `valid()` will return `true`.

### Finding Description

`ProofOfInclusion::valid()` is implemented as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()   // ← always true here
}
``` [1](#0-0) 

And `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

After the loop, `existing_hash` equals the last `layer.combined_hash` (because the loop verified `calculated_hash == layer.combined_hash` and then set `existing_hash = calculated_hash`). `root_hash()` also returns `last.combined_hash`. Therefore the final check `existing_hash == self.root_hash()` is always `true` when the loop completes without returning `false`. The method never compares against any externally-supplied, trusted root hash.

The `ProofOfInclusion` struct is fully attacker-controlled: it is a `Streamable` type exposed to Python via `pyo3` with a public constructor accepting arbitrary `node_hash` and `layers`. [3](#0-2) 

The Python type stub exposes `valid()` and `root_hash()` as separate, independent methods with no documentation requiring callers to also compare `root_hash()` against a trusted value: [4](#0-3) 

`get_proof_of_inclusion` on `MerkleBlob` generates proofs from the live tree, but the returned `ProofOfInclusion` object is a plain serializable struct that any attacker can fabricate independently: [5](#0-4) 

### Impact Explanation

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."**

Any DataLayer verifier that calls `proof.valid()` as its sole check — a natural and reasonable reading of the API — will accept a forged proof claiming that an arbitrary `node_hash` is included in an arbitrary tree root. The attacker controls both the claimed leaf hash and the claimed root, so they can assert any key-value pair is present in any DataLayer store state without it actually being so.

### Likelihood Explanation

The method is named `valid()`, which strongly implies it is a complete validity check. The Python type stub exposes it with no caveat. There is no parameter for a trusted root hash, so callers have no in-API prompt to perform the additional `proof.root_hash() == trusted_root` comparison. The `ProofOfInclusion` struct is `Streamable` and constructible from raw bytes or from Python, making it trivially forgeable by any unprivileged party. The attack requires no special privileges, no key material, and no access to any node.

### Recommendation

Add a `trusted_root_hash` parameter to `valid()` (or add a separate `valid_for_root(root: &Hash) -> bool` method) and compare the computed root against it:

```rust
pub fn valid_for_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against externally-supplied trusted root
}
```

The existing `valid()` method (or its Python binding) should either be removed or clearly documented as only checking internal self-consistency, not proof authenticity.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer, Side
import hashlib

def sha256(data: bytes) -> bytes:
    return hashlib.sha256(data).digest()

def internal_hash(left: bytes, right: bytes) -> bytes:
    return sha256(b"\x02" + left + right)

# Attacker-chosen leaf hash (claims this node is in the tree)
fake_node_hash = bytes([0xAA] * 32)

# Attacker-chosen sibling hash
fake_sibling = bytes([0xBB] * 32)

# Attacker computes a valid combined_hash for the single layer
combined = internal_hash(fake_node_hash, fake_sibling)

layer = ProofOfInclusionLayer(
    other_hash_side=1,          # Side.Right
    other_hash=fake_sibling,
    combined_hash=combined,
)

proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

assert proof.valid()            # True — forged proof passes
assert proof.root_hash() == combined  # Attacker-controlled root
# No check against any real DataLayer store root was performed.
```

`valid()` returns `True` for a completely fabricated proof. A verifier that trusts `valid()` alone will accept that `fake_node_hash` is included in a DataLayer store with root `combined`, even though no such store exists. [1](#0-0) [6](#0-5)

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
