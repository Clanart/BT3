### Title
`ProofOfInclusion::valid()` Performs Tautological Root-Hash Check, Accepting Forged Inclusion Proofs - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` in the DataLayer Merkle implementation contains a tautological final comparison: it checks `existing_hash == self.root_hash()`, but `root_hash()` returns `last.combined_hash` from within the proof itself — the exact value `existing_hash` was just assigned in the final loop iteration. The check is always `true` when `layers` is non-empty. As a result, any attacker-supplied `ProofOfInclusion` with an internally self-consistent chain passes `valid()` regardless of what the actual Merkle tree root is, enabling forged inclusion proofs for arbitrary keys.

### Finding Description

`ProofOfInclusion::valid()` is defined as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;  // existing_hash = layer.combined_hash
    }

    existing_hash == self.root_hash()     // always true when layers is non-empty
}
``` [1](#0-0) 

And `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // same value as existing_hash after the loop
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

**The tautology:** After the loop, `existing_hash` equals the last `calculated_hash`. The loop only continues when `calculated_hash == layer.combined_hash`, so `existing_hash` is guaranteed to equal `last.combined_hash`. `self.root_hash()` also returns `last.combined_hash`. Therefore `existing_hash == self.root_hash()` is unconditionally `true` whenever `layers` is non-empty.

The function only verifies that the proof's internal chain is self-consistent — it never checks that the chain's terminal hash matches any externally trusted root. An attacker can construct a `ProofOfInclusion` for any arbitrary `node_hash` by:

1. Choosing any target `node_hash` (the key they want to falsely prove is included).
2. Building a chain of `ProofOfInclusionLayer` entries where each `combined_hash` is computed correctly from the previous hash and an arbitrary `other_hash`.
3. Serializing the struct via the `Streamable` implementation and submitting it.

`valid()` returns `true` for this forged proof.

### Impact Explanation

This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.**

Any consumer that relies on `proof.valid()` alone to accept a DataLayer inclusion proof — without separately comparing `proof.root_hash()` against a known trusted root — will accept forged proofs for arbitrary keys. The `ProofOfInclusion` type is `Streamable` and exposed through Python bindings, making it directly reachable from untrusted network input. [3](#0-2) 

### Likelihood Explanation

- `ProofOfInclusion` is a `Streamable` type exposed via Python bindings (`py_valid`, `py_root_hash`), meaning it is deserialized from untrusted bytes in production DataLayer workflows.
- The test suite calls `proof_of_inclusion.valid()` without checking `root_hash()` against a known value, confirming the intended API is `valid()` alone.
- An attacker needs only to construct a self-consistent hash chain — a straightforward computation requiring no cryptographic secrets. [4](#0-3) 

### Recommendation

`valid()` must accept an externally trusted root hash as a parameter and compare against it, not against `self.root_hash()`:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root
}
```

The existing `root_hash()` accessor is still useful for extracting the claimed root from a locally-generated proof, but `valid()` must not use it for security-critical verification. All call sites that accept externally-supplied `ProofOfInclusion` objects must be updated to pass the known tree root.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer, MerkleBlob
import hashlib

# Real tree with one key
blob = MerkleBlob(bytearray())
real_key = 0x0102030405060708
real_value = 0x0102030405060708
real_hash = bytes(range(32))
blob.insert(real_key, real_value, real_hash)
blob.calculate_lazy_hashes()
real_root = blob.get_root_hash()

# Forge a proof for a key that does NOT exist in the tree
fake_node_hash = bytes([0xAA] * 32)
other_hash    = bytes([0xBB] * 32)

def internal_hash(left, right):
    h = hashlib.sha256()
    h.update(b'\x01')   # internal node prefix (adjust to match calculate_internal_hash)
    h.update(left)
    h.update(right)
    return h.digest()

combined = internal_hash(fake_node_hash, other_hash)

layer = ProofOfInclusionLayer(
    other_hash_side=...,   # Side.Right
    other_hash=other_hash,
    combined_hash=combined,
)
forged_proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

# valid() returns True for a completely fabricated proof
assert forged_proof.valid(), "Forged proof accepted!"
# forged_proof.root_hash() == combined  (attacker-controlled, not real_root)
assert forged_proof.root_hash() != real_root
```

The `valid()` call succeeds because the final check compares `existing_hash` (= `combined`) against `self.root_hash()` (= `combined`), which is always equal. [1](#0-0) [5](#0-4)

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
