### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary `class_hash` from the caller and commits it to the contract's state without verifying that the hash corresponds to a previously declared class. An unprivileged contract can call `replace_class` with an undeclared class hash, permanently setting its own class to an unreachable value and freezing all funds held within it.

---

### Finding Description

In `execute_replace_class`, the OS reads `class_hash` directly from the syscall request and writes it into `contract_state_changes` with no validation that the hash exists in the declared class set:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The inline `TODO` comment explicitly acknowledges the missing check. The `class_hash` field originates entirely from the attacker-controlled syscall request — there is no floor or existence constraint applied before the state update is committed. [1](#0-0) 

This is the direct analog of M-07's pattern: a critical operation (swap/deposit in M-07; class replacement here) accepts an unconstrained value (`minOut = 0` there; any arbitrary `class_hash` here) and commits the result without a post-operation validity check.

Once the transaction succeeds and the state is committed via the Patricia tree update in `compute_contract_state_commitment`, the contract's `class_hash` field in the global state root is permanently set to the undeclared value. [2](#0-1) 

Any subsequent call to the frozen contract will fail at class lookup time because the compiled class facts bundle does not contain the undeclared hash. The revert log entry written by `execute_replace_class` only helps if the *current* transaction reverts; once the transaction is finalized, the change is irreversible. [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

After a successful `replace_class` call with an undeclared hash:

1. The contract's `class_hash` in the committed state root is set to an undeclared value.
2. Every future call to the contract fails at entry-point dispatch (no compiled class found).
3. There is no recovery path: the contract cannot call `replace_class` again because executing any entry point requires the class to be resolvable.
4. All ERC-20 balances, NFTs, or other assets held in the contract's storage are permanently inaccessible.

---

### Likelihood Explanation

**Low.**

The attacker must own or control a deployed contract (reachable by any unprivileged deployer). The attack requires a deliberate, single transaction. No privileged role, leaked key, or external dependency is needed. The missing check is self-documented in the codebase (`TODO` comment), indicating it is a known gap rather than an oversight that might be caught by other defenses.

---

### Recommendation

Before committing the new class hash to `contract_state_changes`, verify that `class_hash` exists in the set of declared classes. Concretely, perform a `dict_read` on `contract_class_changes` (or the equivalent declared-class registry) and assert the result is non-zero:

```cairo
// Verify the class is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the guard already present in `execute_declare_transaction`, which asserts `assert_not_zero(compiled_class_hash)` before writing to `contract_class_changes`. [4](#0-3) 

---

### Proof of Concept

1. **Deploy** a contract `VictimVault` that holds user funds and exposes a public `freeze_self()` entry point.
2. Inside `freeze_self()`, emit a `replace_class` syscall with `class_hash = 0xdeadbeef` (any value not present in the declared class set).
3. **Call** `freeze_self()` from any unprivileged account. The OS executes `execute_replace_class`:
   - `request.class_hash = 0xdeadbeef` is read.
   - No declared-class check is performed (the `TODO` branch is absent).
   - `contract_state_changes` is updated: `VictimVault.class_hash ← 0xdeadbeef`.
4. The transaction succeeds. `state_update` in `state.cairo` squashes and commits the change into the Patricia tree, producing a new global state root that encodes `VictimVault.class_hash = 0xdeadbeef`. [5](#0-4) 

5. Any subsequent call to `VictimVault` (withdraw, transfer, etc.) fails: the OS cannot resolve `0xdeadbeef` in the compiled class facts bundle, so execution aborts.
6. All funds in `VictimVault`'s storage are permanently frozen with no recovery mechanism.
