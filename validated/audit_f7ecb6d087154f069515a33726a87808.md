### Title
Missing Zero-Value Check in `execute_replace_class` Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

Both the Cairo 1 and deprecated (CairoZero) implementations of the `replace_class` syscall accept an attacker-controlled `class_hash` value of `0` (`UNINITIALIZED_CLASS_HASH`) without any validation. Setting a contract's `class_hash` to `0` permanently renders it unexecutable, freezing any funds held within it.

---

### Finding Description

`UNINITIALIZED_CLASS_HASH` is defined as `0` in the state commitment module: [1](#0-0) 

The Cairo 1 `execute_replace_class` function in `syscall_impls.cairo` reads `class_hash` directly from the syscall request and writes it into the contract's `StateEntry` with no check that it is non-zero: [2](#0-1) 

A developer-left TODO at line 898 explicitly acknowledges that a class existence check is missing, but no zero-value guard exists either: [3](#0-2) 

The deprecated (CairoZero) path in `deprecated_execute_syscalls.cairo` has the identical omission: [4](#0-3) 

By contrast, `deploy_contract` correctly enforces that a contract's `class_hash` must equal `UNINITIALIZED_CLASS_HASH` before deployment, and `execute_declare_transaction` enforces `assert_not_zero(compiled_class_hash)` before writing to the class tree — but no equivalent guard exists for `replace_class`: [5](#0-4) 

---

### Impact Explanation

When a contract's `class_hash` is set to `0` via `replace_class(0)`:

1. `update_class_hash_in_execution_context` reads `state_entry.class_hash = 0` for any subsequent call to that contract address.
2. The OS attempts to dispatch execution using class hash `0`, which has no declared class — every call to the contract fails permanently.
3. Any ERC-20 tokens, ETH, or other assets held in the contract's storage become permanently inaccessible.

`get_contract_state_hash` treats `class_hash=0` with non-zero storage or nonce as a valid (non-empty) leaf, so the contract's storage is preserved in the state tree but is forever unreachable: [6](#0-5) 

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

Any contract that executes the `replace_class` syscall with `class_hash=0` triggers this. A malicious contract developer can:

- Deploy a contract that appears legitimate (e.g., a vault or token contract).
- Attract user deposits.
- Trigger `replace_class(0)` from within the contract's logic (e.g., via a hidden backdoor or a time-locked call), permanently freezing all deposited funds.

The entry path is fully unprivileged: any contract deployer can reach this code path. No operator key, leaked secret, or network-level attack is required.

---

### Recommendation

Add an `assert_not_zero(class_hash)` guard in both `execute_replace_class` implementations before writing to `contract_state_changes`:

In `syscall_impls.cairo`, after line 896:
```cairo
let class_hash = request.class_hash;
assert_not_zero(class_hash);  // Prevent setting UNINITIALIZED_CLASS_HASH
```

In `deprecated_execute_syscalls.cairo`, after line 311:
```cairo
let class_hash = syscall_ptr.class_hash;
assert_not_zero(class_hash);  // Prevent setting UNINITIALIZED_CLASS_HASH
```

This mirrors the existing pattern used in `execute_declare_transaction` and `deploy_contract`.

---

### Proof of Concept

1. Deploy a contract `VaultWithBackdoor` that:
   - Accepts user deposits (stores balances in its storage).
   - Contains a function callable by the deployer that invokes the `replace_class` syscall with `class_hash = 0`.

2. Users deposit funds; the contract accumulates assets.

3. The deployer calls the backdoor function. The OS executes `execute_replace_class` in `syscall_impls.cairo`:
   - `class_hash = 0` is read from the request (line 896).
   - No zero-check exists; `dict_update` writes `StateEntry(class_hash=0, ...)` to `contract_state_changes` (lines 906–910).

4. All subsequent calls to `VaultWithBackdoor` resolve `class_hash=0` → no declared class → permanent execution failure.

5. All user funds are permanently frozen. The state tree still records the contract's storage (non-zero leaf), but no execution path can reach it. [7](#0-6) [4](#0-3)

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
