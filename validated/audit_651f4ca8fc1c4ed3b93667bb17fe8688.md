### Title
Stale `feemarketParams` in `NewDynamicFeeChecker` Allows Cosmos Transactions to Bypass the Current EIP-1559 Base Fee - (File: `ante/evm/fee_checker.go`, `evmd/ante/handler_options.go`)

---

### Summary

`newCosmosAnteHandler` captures `feemarketParams` once at construction time and passes a pointer to it into `NewDynamicFeeChecker`. Because the EIP-1559 base fee is updated every block, the fee checker permanently uses a stale `BaseFee` value for all Cosmos-SDK transactions, allowing them to be accepted and committed with fees below the current block's base fee. The analogous EVM path (`newEthAnteHandler`) correctly reads fresh params per transaction.

---

### Finding Description

In `evmd/ante/handler_options.go`, `newCosmosAnteHandler` reads `feemarketParams` exactly once at handler-construction time:

```go
feemarketParams := options.FeeMarketKeeper.GetParams(ctx)   // captured once
...
txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
``` [1](#0-0) 

The pointer `&feemarketParams` is stored inside the closure returned by `NewDynamicFeeChecker`. Every time a Cosmos transaction is processed, the checker calls:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
``` [2](#0-1) 

`GetBaseFee` reads `feemarketParams.BaseFee` directly from the captured struct — not from the live KV store:

```go
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    ...
    baseFee := feemarketParams.GetBaseFee()   // reads the stale in-memory value
    ...
}
``` [3](#0-2) 

`SetBaseFee` writes the updated base fee to the KV store and updates the params stored there, but it never touches the in-memory `feemarketParams` struct captured at construction time:

```go
func (k Keeper) SetBaseFee(ctx sdk.Context, baseFee *big.Int) {
    params := k.GetParams(ctx)
    params.BaseFee = ethermint.SaturatedNewInt(baseFee)
    err := k.SetParams(ctx, params)
``` [4](#0-3) 

By contrast, `newEthAnteHandler` (for EVM transactions) reads fresh params on every transaction via `EVMBlockConfig`, which queries the live KV store:

```go
blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
feemarketParams := &blockCfg.FeeMarketParams
baseFee := blockCfg.BaseFee
``` [5](#0-4) 

`EVMBlockConfig` calls `k.feeMarketKeeper.GetParams(ctx)` live:

```go
feemarketParams := k.feeMarketKeeper.GetParams(ctx)
...
baseFee = feemarketParams.GetBaseFee()
``` [6](#0-5) 

The `NewDynamicFeeChecker` then uses the stale `baseFee` to compute the `effectiveFee` that is actually deducted from the sender:

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
...
if feeCap.LT(baseFeeInt) { return error }   // stale baseFeeInt
effectivePrice := types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), ...)
effectiveFee := effectivePrice.Mul(gas)     // deducted amount uses stale base
``` [7](#0-6) 

The same stale pointer is also passed to `NewMinGasPriceDecorator`, which reads `MinGasPrice` from it:

```go
cosmos.NewMinGasPriceDecorator(options.FeeMarketKeeper, evmDenom, &feemarketParams),
``` [8](#0-7) 

```go
minGasPrice := mpd.feemarketParams.MinGasPrice   // stale
``` [9](#0-8) 

---

### Impact Explanation

When the EIP-1559 base fee rises above its genesis value (which happens whenever blocks are above the gas target), any Cosmos-SDK transaction processed through `newCosmosAnteHandler` with `DynamicFeeChecker = true` is validated against the stale, lower base fee. The `effectiveFee` deducted from the sender is computed using the stale base fee, so:

- Cosmos transactions are accepted and committed with fees below the current block's base fee.
- The fee collector receives less than the protocol-mandated minimum, constituting fee mis-accounting.
- The discrepancy grows monotonically as the base fee drifts further from the genesis value.

This matches the allowed High impact: *"fee market, ante handler … bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

---

### Likelihood Explanation

- The base fee changes every block under EIP-1559 (bounded by `BaseFeeChangeDenominator = 8`, i.e., ±12.5% per block). After sustained load the base fee diverges significantly from genesis.
- `DynamicFeeChecker` is the standard configuration for any EIP-1559-enabled Ethermint chain.
- Any unprivileged user can craft a Cosmos transaction with `fee = stale_base_fee × gas` and submit it via the standard Cosmos RPC or JSON-RPC path; no special access is required.
- The bug is permanent for the lifetime of the node process — it is not self-healing.

---

### Recommendation

`NewDynamicFeeChecker` must not rely on params captured at construction time. The fee checker closure should query the live base fee from the keeper on every invocation, mirroring the pattern used in `newEthAnteHandler`:

```go
// Instead of capturing feemarketParams at construction:
return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
    // Read live params from the keeper each time
    feemarketParams := feeMarketKeeper.GetParams(ctx)
    baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, &feemarketParams)
    ...
}
```

Similarly, `NewMinGasPriceDecorator` should read `MinGasPrice` from the keeper at ante-handle time rather than from a stale pointer.

---

### Proof of Concept

1. Chain launches; genesis `BaseFee = 1_000_000_000` (1 gwei). `newCosmosAnteHandler` is called once, capturing `feemarketParams.BaseFee = 1 gwei`.
2. Sustained high block utilization causes `CalculateBaseFee` → `SetBaseFee` to raise the live base fee to `2_000_000_000` (2 gwei) after ~8 blocks.
3. Attacker submits a Cosmos `MsgSend` with `fee = 1_000_000_000 × gasLimit` (priced at the stale 1 gwei base fee).
4. `NewDynamicFeeChecker` computes `baseFee = 1 gwei` (stale), `feeCap = 1 gwei`, check `feeCap < baseFee` → false → transaction passes.
5. `effectiveFee = 1 gwei × gasLimit` is deducted; the transaction is committed.
6. The fee collector receives `1 gwei × gasLimit` instead of the required `2 gwei × gasLimit` — a 50% fee shortfall per transaction, exploitable indefinitely.

func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
	if !IsLondon(ethCfg, height) {
		return nil
	}
	baseFee := feemarketParams.GetBaseFee()
	// should not be nil if london hardfork enabled
	if baseFee == nil {
		return new(big.Int)
	}
	return baseFee
}
