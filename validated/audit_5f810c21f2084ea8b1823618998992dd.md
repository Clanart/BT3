### Title
`deprecated_declared_classes` Silently Zeroed in `CommitmentStateDiff → ThinStateDiff` Conversion Produces Wrong `state_diff_commitment` and Block Hash — (File: `crates/blockifier/src/state/cached_state.rs`)

---

### Summary

`From<CommitmentStateDiff> for ThinStateDiff` unconditionally sets `deprecated_declared_classes: Vec::new()`. Because `BlockExecutionArtifacts::new` derives the `ThinStateDiff` for commitment calculation exclusively through this conversion, any block that includes a deprecated (Cairo 0) class declaration will have a `state_diff_commitment` and a `state_diff_length` that are both computed as if no deprecated classes were declared. Both values are embedded in `BlockHeaderCommitments`, which feeds `PartialBlockHash` (the consensus proposal commitment) and the final `BlockHash`.

---

### Finding Description

**Root cause — the silent drop:**

`CommitmentStateDiff` carries no `deprecated_declared_classes` field. Its `From<StateMaps>` constructor never populates one because `StateMaps` itself does not track deprecated declarations. The conversion to `ThinStateDiff` therefore hard-codes an empty vector:

```rust
// crates/blockifier/src/state/cached_state.rs  lines 690-701
impl From<CommitmentStateDiff> for ThinStateDiff {
    fn from(commitment_state_diff: CommitmentStateDiff) -> Self {
        Self {
            deployed_contracts: commitment_state_diff.address_to_class_hash,
            storage_diffs: commitment_state_diff.storage_updates,
            class_hash_to_compiled_class_hash: commitment_state_diff
                .class_hash_to_compiled_class_hash,
            nonces: commitment_state_diff.address_to_nonce,
            // TODO(AlonH): Remove this when the structure of storage diffs changes.
            deprecated_declared_classes: Vec::new(),   // ← always empty
        }
    }
}
```

**Propagation into the block hash:**

`BlockExecutionArtifacts::new` calls `calculate_block_commitments` with exactly this deficient `ThinStateDiff`:

```rust
// crates/apollo_batcher/src/block_builder.rs  lines 160-166
let (header_commitments, measurements) = calculate_block_commitments(
    &transactions_data,
    ThinStateDiff::from(commitment_state_diff.clone()),  // deprecated_declared_classes = []
    l1_da_mode,
    &block_info.starknet_version,
)
.await;
```

Inside `calculate_block_commitments`, two values are derived from the deficient diff:

1. **`state_diff_commitment`** — computed by `calculate_state_diff_hash`, which explicitly chains `deprecated_declared_classes` into the Poseidon hash:

```rust
// crates/starknet_api/src/block_hash/state_diff_hash.rs  lines 30-41
pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    ...
    hash_chain = chain_deprecated_declared_classes(
        &state_diff.deprecated_declared_classes,   // always []
        hash_chain,
    );
    ...
}
```

With an empty slice, `chain_deprecated_declared_classes` chains `[count=0]` instead of the actual class hashes, producing a different Poseidon output than the spec requires.

2. **`state_diff_length`** (packed into `concatenated_counts`) — computed by `ThinStateDiff::len()`, which adds `deprecated_declared_classes.len()`:

```rust
// crates/starknet_api/src/state.rs  lines 110-121
pub fn len(&self) -> usize {
    let mut result = 0usize;
    result += self.deployed_contracts.len();
    result += self.class_hash_to_compiled_class_hash.len();
    result += self.deprecated_declared_classes.len();   // always 0
    result += self.nonces.len();
    for (_, storage_diffs) in &self.storage_diffs { result += storage_diffs.len(); }
    result
}
```

Both wrong values are stored in `BlockHeaderCommitments`, which is then embedded in `PartialBlockHashComponents` and used to compute both the `PartialBlockHash` (the consensus proposal commitment) and the final `BlockHash`.

---

### Impact Explanation

For every block that contains at least one `DeclareV1` (deprecated Cairo 0 class declaration) transaction:

| Corrupted value | Where it appears |
|---|---|
| `state_diff_commitment` | `BlockHeaderCommitments`, block header, block hash |
| `state_diff_length` in `concatenated_counts` | `BlockHeaderCommitments`, block hash |
| `PartialBlockHash` | Consensus proposal commitment |
| Final `BlockHash` | Stored in storage, returned by RPC, used by L1 |

The block hash diverges from the value that any correct independent implementation of the Starknet spec would compute. Downstream consumers — the SNOS prover, L1 verifier contracts, and any external block-hash verifier — will reject or disagree with the sequencer's output for such blocks.

This matches the allowed impact: **"Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input"** — specifically a wrong `state_diff_commitment` and wrong block hash produced for accepted transactions.

---

### Likelihood Explanation

`DeclareV1` (deprecated declare) transactions remain valid on Starknet v0.13.x. Any unprivileged user can submit one. The gateway performs stateful validation but does not reject deprecated declare transactions on the basis of their type alone. A single such transaction included in any block is sufficient to trigger the wrong commitment. No special privileges, no malicious peer, and no oracle manipulation are required.

---

### Recommendation

1. **Track deprecated declared classes in the blockifier.** Add a `deprecated_declared_classes: Vec<ClassHash>` field to `StateMaps` and populate it when a `DeclareV1` transaction is executed. Propagate it through `CommitmentStateDiff`.

2. **Fix the conversion.** Remove the hard-coded `deprecated_declared_classes: Vec::new()` from `From<CommitmentStateDiff> for ThinStateDiff` once the field is available upstream.

3. **Until (1) is done, pass the deprecated class hash explicitly in the block-building path**, mirroring the pattern already used in `induced_state_diff` in `crates/apollo_rpc_execution/src/execution_utils.rs` (lines 130–144), where `deprecated_declared_class_hash: Option<ClassHash>` is accepted as a parameter and injected into the resulting `ThinStateDiff`.

4. **Add a regression test** that builds a block containing a `DeclareV1` transaction and asserts that the resulting `state_diff_commitment` and `state_diff_length` match an independently computed reference value.

---

### Proof of Concept

1. Submit a valid `DeclareV1` transaction (deprecated Cairo 0 class declaration) to the sequencer gateway.
2. Wait for it to be included in a block and for `BlockExecutionArtifacts::new` to be called.
3. Observe that `ThinStateDiff::from(commitment_state_diff.clone())` produces a `ThinStateDiff` with `deprecated_declared_classes = []`.
4. Compute the expected `state_diff_commitment` independently using `calculate_state_diff_hash` with the correct `deprecated_declared_classes` populated.
5. Compare with the `state_diff_commitment` stored in the block header: they differ.
6. Compute the expected `BlockHash` using the correct `state_diff_commitment` and `state_diff_length`: it differs from the hash stored and returned by the sequencer.

The corrupted values flow through:

```
DeclareV1 tx executed
  → StateMaps (no deprecated_declared_classes field)

  → ThinStateDiff::from(…) → deprecated_declared_classes = []   ← BUG

  → ThinStateDiff::len()     → wrong state_diff_length
  → BlockHeaderCommitments   → wrong concatenated_counts
  → PartialBlockHashComponents
  → PartialBlockHash (consensus proposal commitment) — WRONG
  → calculate_block_hash     → BlockHash — WRONG

pub fn induced_state_diff(
    transactional_state: &mut CachedState<MutRefState<'_, CachedState<ExecutionStateReader>>>,
    deprecated_declared_class_hash: Option<ClassHash>,
) -> ExecutionResult<ThinStateDiff> {
    let blockifier_state_diff =
        CommitmentStateDiff::from(transactional_state.to_state_diff()?.state_maps);

    Ok(ThinStateDiff {
        deployed_contracts: blockifier_state_diff.address_to_class_hash,
        storage_diffs: blockifier_state_diff.storage_updates,
        class_hash_to_compiled_class_hash: blockifier_state_diff.class_hash_to_compiled_class_hash,
        deprecated_declared_classes: deprecated_declared_class_hash
            .map_or_else(Vec::new, |class_hash| vec![class_hash]),
        nonces: blockifier_state_diff.address_to_nonce,
    })
