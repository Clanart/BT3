### Title
Cross-Chain Signature Replay in `deployToken` / `deploy_token` Enables Unauthorized Token Deployment on Unintended Chains — (Files: `evm/src/omni-bridge/contracts/OmniBridge.sol`, `starknet/src/omni_bridge.cairo`, `solana/programs/bridge_token_factory/src/instructions/user/deploy_token.rs`)

---

### Summary

The `deployToken` / `deploy_token` signed-message format is identical across all destination chains (EVM, StarkNet, Solana). It does not include a chain identifier or contract address. A valid NEAR-MPC-signed `deployToken` signature for chain A can be extracted from a submitted transaction and replayed verbatim on chain B, deploying the same token on an unintended chain and causing NEAR to register an unauthorized cross-chain token mapping.

---

### Finding Description

When NEAR's MPC signs a `deployToken` (metadata) message, the borsh-encoded payload that is hashed and signed is:

**EVM** (`OmniBridge.sol` lines 142–149):
```
PayloadType.Metadata (0x01) | borsh(token) | borsh(name) | borsh(symbol) | decimals
```

**StarkNet** (`bridge_types.cairo` `MetadataPayloadTrait::to_borsh`, lines 36–44):
```
PayloadType::Metadata (0x01) | borsh(token) | borsh(name) | borsh(symbol) | decimals
```

**Solana** (`deploy_token.rs` `DeployTokenPayload::serialize_for_near`, lines 19–26):
```
IncomingMessageType::Metadata (0x01) | borsh(token) | borsh(name) | borsh(symbol) | decimals
```

All three formats are byte-for-byte identical. No `chain_id`, no `address(this)` / program ID, no destination-chain discriminator is included.

Contrast this with `finTransfer` / `fin_transfer`, which explicitly encodes `omniBridgeChainId` / `SOLANA_OMNI_BRIDGE_CHAIN_ID` twice in the payload (for token address and recipient address chain prefixes), preventing cross-chain replay for transfers. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

The only replay guard in `deployToken` is an idempotency check — "token already deployed on this chain":

- EVM: `require(!isBridgeToken[nearToEthToken[metadata.token]], "ERR_TOKEN_EXIST")` (line 155–158)
- StarkNet: `assert(existing_token.is_zero(), 'ERR_TOKEN_ALREADY_DEPLOYED')` (line 209)
- Solana: `init` constraint on the PDA `[WRAPPED_MINT_SEED, token_hash]` (line 48) [5](#0-4) [6](#0-5) [7](#0-6) 

These guards only prevent re-deployment on the **same** chain. They do not prevent the same signature from being used on a **different** chain.

---

### Impact Explanation

An attacker who observes a valid `deployToken` transaction on chain A can:

1. Extract the `signatureData` / `signature` from the calldata.
2. Submit the identical `(signature, payload)` to chain B's bridge contract.
3. The token is deployed on chain B with the bridge as mint authority.
4. Chain B's bridge emits a Wormhole message to NEAR.
5. NEAR's `deploy_token_callback` verifies the emitter is a registered factory for chain B and registers the token for chain B. [8](#0-7) 

The result is an **unauthorized token registration on NEAR for chain B** — a chain the protocol had not yet authorized for that token. Once registered, users can bridge the token to chain B. If the protocol's off-chain infrastructure (relayers, liquidity, risk controls) is not ready for chain B, this can lead to:

- Unbacked or unmonitored bridged token supply on chain B.
- Disruption of the protocol's controlled token rollout.
- Potential accounting desynchronization between NEAR's `locked_tokens` tracking and actual on-chain supply.

This matches the allowed High impact: **"Proof, signature, MPC, Wormhole, or light-client verification bypass enabling unauthorized transfer finalization, token deployment, or message execution."**

---

### Likelihood Explanation

- The `deployToken` transaction is public on-chain; the signature is trivially extractable from calldata.
- No privileged access is required — any unprivileged observer can perform the replay.
- The attacker only needs to know the address of the bridge contract on the target chain (publicly known).
- The attack is executable as soon as a single `deployToken` transaction is confirmed on any chain.

---

### Recommendation

Include the destination chain identifier and contract address in the signed metadata payload, mirroring the pattern already used in `finTransfer`. For example:

**EVM** — add `omniBridgeChainId` and `address(this)` to the borsh-encoded hash input:
```solidity
bytes memory borshEncoded = bytes.concat(
    bytes1(uint8(BridgeTypes.PayloadType.Metadata)),
    bytes1(omniBridgeChainId),          // ADD: destination chain
    Borsh.encodeAddress(address(this)), // ADD: contract address
    Borsh.encodeString(metadata.token),
    Borsh.encodeString(metadata.name),
    Borsh.encodeString(metadata.symbol),
    bytes1(metadata.decimals)
);
```

Apply the equivalent fix to `MetadataPayloadTrait::to_borsh` in StarkNet and `DeployTokenPayload::serialize_for_near` in Solana. The NEAR MPC signing logic must be updated to include these fields when producing `deployToken` signatures.

---

### Proof of Concept

1. NEAR MPC signs a `deployToken` message for `usdc.near` (name="USD Coin", symbol="USDC", decimals=6) targeting Ethereum. The borsh payload is `0x01 | borsh("usdc.near") | borsh("USD Coin") | borsh("USDC") | 0x06`.

2. A relayer submits `OmniBridge.deployToken(sig, payload)` on Ethereum. The token is deployed. The transaction is public.

3. An attacker extracts `sig` from the Ethereum transaction calldata.

4. The attacker calls `OmniBridge.deployToken(sig, payload)` on BSC (or any other EVM chain where the bridge is deployed), or `deploy_token(sig, payload)` on StarkNet.

5. `ECDSA.recover(keccak256(borshEncoded), sig) == nearBridgeDerivedAddress` passes — the payload is identical, the signature is valid.

6. The token is deployed on BSC/StarkNet. A Wormhole message is emitted.

7. NEAR's `deploy_token_callback` receives the VAA, verifies the emitter is the registered factory for BSC/StarkNet, and registers `usdc.near` for that chain — without any new NEAR MPC authorization. [9](#0-8) [10](#0-9) [11](#0-10)

    function deployToken(
        bytes calldata signatureData,
        BridgeTypes.MetadataPayload calldata metadata
    ) external payable whenNotPaused(PAUSED_DEPLOY_TOKEN) returns (address) {
        if (tokenImplementationAddress == address(0)) {
            revert TokenImplementationNotSet();
        }
        bytes memory borshEncoded = bytes.concat(
            bytes1(uint8(BridgeTypes.PayloadType.Metadata)),
            Borsh.encodeString(metadata.token),
            Borsh.encodeString(metadata.name),
            Borsh.encodeString(metadata.symbol),
            bytes1(metadata.decimals)
        );
        bytes32 hashed = keccak256(borshEncoded);

        if (ECDSA.recover(hashed, signatureData) != nearBridgeDerivedAddress) {
            revert InvalidSignature();
        }

        require(
            !isBridgeToken[nearToEthToken[metadata.token]],
            "ERR_TOKEN_EXIST"
        );
        uint8 decimals = _normalizeDecimals(metadata.decimals);

        // slither-disable-next-line reentrancy-no-eth
        address bridgeTokenProxy = address(
            new ERC1967Proxy(
                tokenImplementationAddress,
                abi.encodeWithSelector(
                    BridgeToken.initialize.selector,
                    metadata.name,
                    metadata.symbol,
                    decimals
                )
            )
        );

        deployTokenExtension(

            bridgeTokenProxy,
            decimals,
            metadata.decimals

            metadata.symbol,

            metadata.decimals

        ethToNearToken[address(bridgeTokenProxy)] = metadata.token;
        nearToEthToken[metadata.token] = address(bridgeTokenProxy);

    fn to_borsh(self: @MetadataPayload) -> ByteArray {
        let mut borsh_bytes: ByteArray = Default::default();
        borsh_bytes.append_byte(PayloadType::Metadata.into());
        borsh_bytes.append(@borsh::encode_byte_array(self.token));

        borsh_bytes.append(@borsh::encode_byte_array(self.symbol));
        borsh_bytes.append_byte(*self.decimals);
        borsh_bytes
    }

        fn deploy_token(ref self: ContractState, signature: Signature, payload: MetadataPayload) {
            assert(!_is_paused(@self, PAUSE_DEPLOY_TOKEN), 'ERR_DEPLOY_TOKEN_PAUSED');

            _verify_borsh_signature(ref self, @payload.to_borsh(), signature);

            let token_id_hash = compute_keccak_byte_array(@payload.token);
            let existing_token = self.near_to_starknet_token.read(token_id_hash);
            assert(existing_token.is_zero(), 'ERR_TOKEN_ALREADY_DEPLOYED');

            let decimals = _normalizeDecimals(payload.decimals);

            let mut constructor_calldata: Array<felt252> = array![];
            (payload.name.clone(), payload.symbol.clone(), decimals)
                .serialize(ref constructor_calldata);

            // Use the low part of the u256 hash to ensure it fits in felt252
            let salt: felt252 = token_id_hash.low.into();
            let (contract_address, _) = deploy_syscall(
                self.bridge_token_class_hash.read(), salt, constructor_calldata.span(), false,
            )
                .unwrap_syscall();

            self.starknet_to_near_token.write(contract_address, payload.token.clone());
            self.near_to_starknet_token.write(token_id_hash, contract_address);

                .emit(

                        DeployToken {
                            token_address: contract_address,
                            near_token_id: payload.token,
                            name: payload.name,
                            symbol: payload.symbol,
                            decimals,
                            origin_decimals: payload.decimals,

    #[private]
    pub fn deploy_token_callback(
        &mut self,
        attached_deposit: NearToken,
        #[callback_result]
        #[serializer(borsh)]
        call_result: Result<ProverResult, PromiseError>,
    ) -> Promise {
        let Ok(ProverResult::LogMetadata(metadata)) = call_result else {
            env::panic_str(BridgeError::InvalidProofMessage.to_string().as_str());
        };

        let chain = metadata.emitter_address.get_chain();
        require!(
            self.factories.get(&chain) == Some(metadata.emitter_address),
            BridgeError::UnknownFactory.as_ref()
        );

        self.deploy_token_internal(
            chain,
            &metadata.token_address,
            BasicMetadata {
                name: metadata.name,
                symbol: metadata.symbol,
                decimals: metadata.decimals,
            },
            attached_deposit,
        )
    }
