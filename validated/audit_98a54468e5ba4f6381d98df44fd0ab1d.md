### Title
Missing Declared Class Hash Validation in `replace_class` Syscall Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS does not validate that the new class hash supplied by a contract is a previously declared class. A contract can call `replace_class` with an arbitrary, undeclared class hash. Once the state is committed with this invalid class hash, all future calls to that contract will fail at the compiled-class lookup step, permanently bricking the contract and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class`, the OS accepts any `class_hash` value from the syscall request and writes it directly into `contract_state_changes` without verifying that a compiled class with that hash has been declared:

```cairo
// execute_replace_class (syscall_impls.cairo, ~line 896-914)
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment at line 898 explicitly acknowledges this missing check. [1](#0-0) 

After the state is committed with an invalid `class_hash`, any subsequent call to that contract reaches `execute_entry_point`, which performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // = invalid/undeclared hash
);
// compiled_class_hash == 0 (default for undeclared key)

let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,   // = 0
);
```

`find_element` with key `0` will fail (no compiled class fact with hash `0` exists), making any proof for a call to this contract invalid. The sequencer must revert every future transaction targeting this contract, but the contract's class hash in the state tree remains the invalid value — the contract is permanently unexecutable. [2](#0-1) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any ERC-20 token balance, ETH, or other asset stored in the contract's storage slots becomes permanently inaccessible. Because every call to the contract fails at the class-lookup step, there is no way to invoke `replace_class` again to recover the contract. The freeze is irreversible without a protocol-level state override.

---

### Likelihood Explanation

**High.** The attack path requires only:
1. Deploying a contract (permissionless on StarkNet).
2. Receiving funds into that contract (e.g., acting as a vault, bridge endpoint, or token holder).
3. Calling the `replace_class` syscall with any value not present in the declared class tree (e.g., `felt(1)`, a random felt, or `0`).

No privileged role, leaked key, or external dependency is required. The missing check is confirmed by the in-code TODO comment, meaning the OS currently accepts this call unconditionally.

---

### Recommendation

Inside `execute_replace_class`, before writing the new class hash to `contract_state_changes`, verify that the requested `class_hash` has a non-zero entry in `contract_class_changes` (i.e., it has been declared in the current or a prior block):

```cairo
// After reading class_hash from the request:
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("replace_class: class hash not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the validation already performed implicitly in `execute_entry_point` and makes the constraint explicit and enforced at the point of the syscall. [3](#0-2) 

---

### Proof of Concept

1. **Deploy** a contract `VaultContract` that holds user token balances in its storage and exposes a `replace_class` syscall call in one of its entry points.
2. **Users deposit** tokens into `VaultContract` (balances recorded in storage).
3. **Attacker** (who controls `VaultContract`, or a malicious contract that users interact with) invokes the entry point that calls `replace_class(class_hash=0x41414141)` — an arbitrary undeclared felt.
4. The OS executes `execute_replace_class`: no validation occurs; `contract_state_changes` is updated with `class_hash = 0x41414141`.
5. The block is finalized; the state root now encodes `VaultContract.class_hash = 0x41414141`.
6. **Any subsequent transaction** targeting `VaultContract` reaches `execute_entry_point`:
   - `dict_read(contract_class_changes, 0x41414141)` → returns `0`.
   - `find_element(..., key=0)` → fails; proof cannot be generated.
   - Sequencer must revert the transaction.
7. All user balances in `VaultContract`'s storage are permanently frozen — no `withdraw`, `transfer`, or `replace_class` call can ever succeed again.
