### Title
Missing `shard_id` Binding in `set_state_header` Allows Cross-Shard State Corruption During State Sync — (`File: chain/chain/src/state_sync/adapter.rs`)

---

### Summary

`ChainStateSyncAdapter::set_state_header` accepts a caller-supplied `shard_id` and a `ShardStateSyncResponseHeader` that embeds a `ShardChunk`, but never verifies that `chunk.shard_id() == shard_id`. A malicious peer can respond to a state-sync header request for shard X by supplying a structurally valid chunk from shard Y (with a genuine Merkle proof for shard Y's position in the block). All existing checks pass, and the header is committed to `DBCol::StateHeaders` under the key `StateHeaderKey(X, sync_hash)`. Every downstream step — `set_state_part`, `apply_state_part`, and `set_state_finalize` — then operates on shard Y's state root while believing it is shard X's, permanently corrupting the node's trie and flat storage for shard X.

---

### Finding Description

`set_state_header` performs five categories of checks:

1. `validate_chunk_proofs` — validates the chunk's internal encoded-merkle-root and signature.
2. `verify_path(sync_prev_block_header.chunk_headers_root(), chunk_proof, ChunkHashHeight(chunk.chunk_hash(), chunk.height_included()))` — proves the chunk hash appears somewhere in the block's chunk-headers Merkle tree.
3. Prev-chunk Merkle proof.
4. Incoming-receipts chain continuity.
5. `validate_state_root_node` — proves the state-root-node hashes to `chunk.prev_state_root()`.

None of these checks bind the chunk's own `shard_id` field to the `shard_id` parameter supplied by the caller. Check 2 only proves that the chunk hash is a leaf in the block's Merkle tree; it does not constrain which leaf position (i.e., which shard slot) was used. A chunk from shard Y at leaf index Y has a perfectly valid Merkle proof that `verify_path` accepts regardless of what `shard_id` the caller passed.

After all checks pass, the header is stored:

```rust
let key = borsh::to_vec(&StateHeaderKey(shard_id, sync_hash)).unwrap();
store_update.set_ser(DBCol::StateHeaders, &key, &shard_state_header);
```

`shard_id` here is the attacker-controlled value X, while `shard_state_header` contains shard Y's chunk and state root. [1](#0-0) 

`set_state_part` then retrieves this header by `(X, sync_hash)`, extracts `state_root = chunk_Y.prev_state_root()`, validates the part against that root, and stores it under `StatePartKey(sync_hash, X, part_id)`: [2](#0-1) 

`apply_state_part` in `NightshadeRuntime` resolves `shard_uid` from the caller-supplied `shard_id=X` and `epoch_id`, then writes shard Y's trie nodes and flat-state delta into shard X's column: [3](#0-2) 

`set_state_finalize` then calls `apply_chunk` with `shard_uid=X` but `prev_state_root = chunk_Y.prev_state_root()`, completing the cross-shard state transplant: [4](#0-3) 

The missing guard is a single equality check that is never performed anywhere in the pipeline:

```rust
// MISSING:
if chunk.shard_id() != shard_id {
    return Err(Error::Other("set_shard_state failed: chunk shard_id mismatch".into()));
}
```

---

### Impact Explanation

A syncing node that accepts a cross-shard header will:

- Write shard Y's trie nodes into shard X's RocksDB column family under `ShardUId` for X.
- Write shard Y's flat-state delta into shard X's flat-storage.
- Commit a `ChunkExtra` for shard X whose `state_root` is shard Y's root.

From that point the node's shard X state is irrecoverably wrong. If the node is a validator or chunk producer for shard X it will produce chunks with an incorrect `prev_state_root`, which other validators will reject, causing the node to be slashed or kicked out. If it is an RPC node, all state queries for shard X accounts return wrong data. Recovery requires a full re-sync.

**Severity: High** — permanent local state corruption, validator slashing risk, incorrect RPC responses.

---

### Likelihood Explanation

State sync is triggered whenever a node falls behind by more than one epoch. Any peer the syncing node connects to can serve state-sync responses. The NEAR P2P network is open; an attacker can run a node, become a peer of the victim, and respond to `StateRequestHeader` messages with a crafted `ShardStateSyncResponse`. No stake, no validator role, and no privileged access is required. The crafted response passes all five existing validation checks because it uses a genuine chunk from the same block with a genuine Merkle proof.

---

### Recommendation

Add an explicit shard-id binding check at the top of `set_state_header`, immediately after extracting the chunk:

```rust
let chunk = shard_state_header.cloned_chunk();
// Bind the chunk's own shard_id to the caller-supplied shard_id.
if chunk.shard_id() != shard_id {
    byzantine_assert!(false);
    return Err(Error::Other(format!(
        "set_shard_state failed: chunk shard_id {} does not match requested shard_id {}",
        chunk.shard_id(), shard_id
    )));
}
```

This mirrors the pattern already used in `compute_state_response_header`, which validates `shard_ids.contains(&shard_id)` before selecting the chunk by shard index, ensuring the chunk's position in the block is always consistent with the requested shard. [5](#0-4) 

---

### Proof of Concept

**Setup:** Two-shard network (shard 0, shard 1). Victim node V is state-syncing shard 0. Attacker A is a peer of V.

1. V sends `StateRequestHeader { shard_id: 0, sync_hash: H }` to A.
2. A calls its own `compute_state_response_header(1, H)` to obtain a valid `ShardStateSyncResponseHeaderV2` for shard 1 (chunk₁, chunk_proof₁, state_root_node₁, …).
3. A responds with `StateResponseInfo { shard_id: 0, sync_hash: H, state_response: header_for_shard_1 }`.
4. V calls `set_state_header(0, H, header_for_shard_1)`.
   - `validate_chunk_proofs(chunk₁)` → passes (chunk₁ is internally valid).
   - `verify_path(block.chunk_headers_root, chunk_proof₁, ChunkHashHeight(chunk₁.hash, chunk₁.height_included))` → passes (chunk_proof₁ is the genuine Merkle proof for shard 1's slot).
   - All receipt and state-root-node checks → pass (they are internally consistent for shard 1).
   - Header stored under `StateHeaderKey(0, H)` with shard 1's state root.
5. V requests and applies all state parts for "shard 0" — actually shard 1's trie nodes — via `set_state_part` and `apply_state_part`.
6. V calls `set_state_finalize(0, H)`: applies shard 1's chunk transactions against shard 1's state root, but writes the result into shard 0's `ChunkExtra`.
7. V's shard 0 state is now shard 1's state. All shard 0 account queries return wrong data; any block V produces for shard 0 is invalid.

    pub fn set_state_header(
        &self,
        shard_id: ShardId,
        sync_hash: CryptoHash,
        shard_state_header: ShardStateSyncResponseHeader,
    ) -> Result<(), Error> {
        let sync_block_header = self.chain_store.get_block_header(&sync_hash)?;

        let chunk = shard_state_header.cloned_chunk();
        let prev_chunk_header = shard_state_header.cloned_prev_chunk_header();

        // 1-2. Checking chunk validity
        if !validate_chunk_proofs(&chunk, self.epoch_manager.as_ref())? {
            byzantine_assert!(false);
            return Err(Error::Other(
                "set_shard_state failed: chunk header proofs are invalid".into(),
            ));
        }

        // Consider chunk itself is valid.

        // 3. Checking that chunks `chunk` and `prev_chunk` are included in appropriate blocks
        // 3a. Checking that chunk `chunk` is included into block at last height before sync_hash
        // 3aa. Also checking chunk.height_included
        let sync_prev_block_header =
            self.chain_store.get_block_header(sync_block_header.prev_hash())?;
        if !verify_path(
            *sync_prev_block_header.chunk_headers_root(),
            shard_state_header.chunk_proof(),
            &ChunkHashHeight(chunk.chunk_hash().clone(), chunk.height_included()),
        ) {
            byzantine_assert!(false);
            return Err(Error::Other(
                "set_shard_state failed: chunk isn't included into block".into(),
            ));
        }

        let block_header = get_block_header_on_chain_by_height(
            &self.chain_store,
            &sync_hash,
            chunk.height_included(),
        )?;
        // 3b. Checking that chunk `prev_chunk` is included into block at height before chunk.height_included
        // 3ba. Also checking prev_chunk.height_included - it's important for getting correct incoming receipts
        match (&prev_chunk_header, shard_state_header.prev_chunk_proof()) {
            (Some(prev_chunk_header), Some(prev_chunk_proof)) => {
                let prev_block_header =
                    self.chain_store.get_block_header(block_header.prev_hash())?;
                if !verify_path(
                    *prev_block_header.chunk_headers_root(),
                    prev_chunk_proof,
                    &ChunkHashHeight(prev_chunk_header.chunk_hash().clone(), prev_chunk_header.height_included()),
                ) {
                    byzantine_assert!(false);
                    return Err(Error::Other(
                        "set_shard_state failed: prev_chunk isn't included into block".into(),
                    ));
                }
            }
            (None, None) => {
                if chunk.height_included() != 0 {
                    return Err(Error::Other(
                    "set_shard_state failed: received empty state response for a chunk that is not at height 0".into()
                ));
                }
            }
            _ =>
                return Err(Error::Other("set_shard_state failed: `prev_chunk_header` and `prev_chunk_proof` must either both be present or both absent".into()))
        };

        // 4. Proving incoming receipts validity
        // 4a. Checking len of proofs
        if shard_state_header.root_proofs().len()
            != shard_state_header.incoming_receipts_proofs().len()
        {
            byzantine_assert!(false);
            return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
        }
        let mut hash_to_compare = sync_hash;
        for (i, receipt_response) in
            shard_state_header.incoming_receipts_proofs().iter().enumerate()
        {
            let ReceiptProofResponse(block_hash, receipt_proofs) = receipt_response;

            // 4b. Checking that there is a valid sequence of continuous blocks
            if *block_hash != hash_to_compare {
                byzantine_assert!(false);
                return Err(Error::Other(
                    "set_shard_state failed: invalid incoming receipts".into(),
                ));
            }
            let header = self.chain_store.get_block_header(&hash_to_compare)?;
            hash_to_compare = *header.prev_hash();

            let block_header = self.chain_store.get_block_header(block_hash)?;
            // 4c. Checking len of receipt_proofs for current block
            if receipt_proofs.len() != shard_state_header.root_proofs()[i].len()
                || receipt_proofs.len() != block_header.chunks_included() as usize
            {
                byzantine_assert!(false);
                return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
            }
            // We know there were exactly `block_header.chunks_included` chunks included
            // on the height of block `block_hash`.
            // There were no other proofs except for included chunks.
            // According to Pigeonhole principle, it's enough to ensure all receipt_proofs are distinct
            // to prove that all receipts were received and no receipts were hidden.
            let mut visited_shard_ids = HashSet::<ShardId>::new();
            for (j, receipt_proof) in receipt_proofs.iter().enumerate() {
                let ReceiptProof(receipts, shard_proof) = receipt_proof;
                let ShardProof { from_shard_id, to_shard_id: _, proof } = shard_proof;
                // 4d. Checking uniqueness for set of `from_shard_id`
                match visited_shard_ids.get(from_shard_id) {
                    Some(_) => {
                        byzantine_assert!(false);
                        return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
                    }
                    _ => visited_shard_ids.insert(*from_shard_id),
                };
                let RootProof(root, block_proof) = &shard_state_header.root_proofs()[i][j];
                let receipts_hash = CryptoHash::hash_borsh(ReceiptList(shard_id, receipts));
                // 4e. Proving the set of receipts is the subset of outgoing_receipts of shard `shard_id`
                if !verify_path(*root, proof, &receipts_hash) {
                    byzantine_assert!(false);
                    return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
                }
                // 4f. Proving the outgoing_receipts_root matches that in the block
                if !verify_path(
                    *block_header.prev_chunk_outgoing_receipts_root(),
                    block_proof,
                    root,
                ) {
                    byzantine_assert!(false);
                    return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
                }
            }
        }
        // 4g. Checking that there are no more heights to get incoming_receipts
        let header = self.chain_store.get_block_header(&hash_to_compare)?;
        if header.height() != prev_chunk_header.map_or(0, |h| h.height_included()) {
            byzantine_assert!(false);
            return Err(Error::Other("set_shard_state failed: invalid incoming receipts".into()));
        }

        // 5. Checking that state_root_node is valid
        let chunk_inner = chunk.take_header().take_inner();
        if matches!(
            self.runtime_adapter.validate_state_root_node(
                shard_state_header.state_root_node(),
                chunk_inner.prev_state_root(),
            ),
            StateRootNodeValidationResult::Invalid
        ) {
            byzantine_assert!(false);
            return Err(Error::Other("set_shard_state failed: state_root_node is invalid".into()));
        }

        // Saving the header data.
        let mut store_update = self.chain_store.store().store_update();
        let key = borsh::to_vec(&StateHeaderKey(shard_id, sync_hash)).unwrap();
        store_update.set_ser(DBCol::StateHeaders, &key, &shard_state_header);
        store_update.commit();

        Ok(())
    }

    pub fn set_state_part(
        &self,
        shard_id: ShardId,
        sync_hash: CryptoHash,
        part_id: PartId,
        part: &StatePart,
    ) -> Result<(), Error> {
        let shard_state_header = self.get_state_header(shard_id, sync_hash)?;
        let chunk = shard_state_header.take_chunk();
        let state_root = *chunk.take_header().take_inner().prev_state_root();
        if matches!(
            self.runtime_adapter.validate_state_part(shard_id, &state_root, part_id, part),
            StatePartValidationResult::Invalid

                "set_state_part failed: validate_state_part failed. state_root={:?}",
                state_root
            )));
        }
        // Saving the part data.
        let mut store_update = self.chain_store.store().store_update();
        let key = borsh::to_vec(&StatePartKey(sync_hash, shard_id, part_id.idx)).unwrap();
        let bytes = part.to_bytes();
        store_update.set(DBCol::StateParts, &key, &bytes);
        store_update.commit();
        Ok(())

    fn apply_state_part(
        &self,
        shard_id: ShardId,
        state_root: &StateRoot,
        part_id: PartId,
        part: &StatePart,
        epoch_id: &EpochId,
    ) -> Result<(), Error> {
        let _timer = metrics::STATE_SYNC_APPLY_PART_DELAY
            .with_label_values(&[&shard_id.to_string()])
            .start_timer();

        let part = part
            .to_partial_state()
            .expect("Part was already validated earlier, so could never fail here");
        let ApplyStatePartResult { trie_changes, flat_state_delta, contract_codes } =
            Trie::apply_state_part(state_root, part_id, part);
        let tries = self.get_tries();
        let shard_uid = self.get_shard_uid_from_epoch_id(shard_id, epoch_id)?;
        let mut store_update = tries.store_update();
        tries.apply_all(&trie_changes, shard_uid, &mut store_update);
        tracing::debug!(target: "chain", %shard_id, values_count = %flat_state_delta.len(), "inserting values to flat storage");
        // TODO: `apply_to_flat_state` inserts values with random writes, which can be time consuming.
        //       Optimize taking into account that flat state values always correspond to a consecutive range of keys.
        flat_state_delta.apply_to_flat_state(&mut store_update.flat_store_update(), shard_uid);
        self.precompile_contracts(epoch_id, contract_codes)?;
        store_update.commit();
        Ok(())
