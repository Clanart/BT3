### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS does not validate that the new class hash supplied via the `replace_class` syscall is actually declared in the contract class tree. A contract can therefore replace its own class hash with an arbitrary, undeclared value. Because the only way to subsequently fix the class hash is to call `replace_class` again — which requires executing the (now-broken) contract — any funds held by the contract are permanently frozen. The missing check is even acknowledged by a developer TODO comment in the source.

---

### Finding Description

`execute_replace_class` (lines 877–916 of `syscall_impls.cairo`) processes the `replace_class` syscall. After deducting gas it reads the current `StateEntry` for the calling contract from `contract_state_changes` and writes a new `StateEntry` whose `class_hash` field is set to whatever value the contract supplied in `request.class_hash`. No cross-reference against `contract_class_changes` (the class tree) is performed.

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The contract state tree (`contract_state_changes`) and the class tree (`contract_class_changes`) are entirely separate dictionaries. `execute_replace_class` only touches the former. The class tree is only updated by `execute_declare_transaction`, which enforces `prev_value=0` (a class may be declared exactly once):

```cairo
// Note that prev_value=0 enforces that a class may be declared only once.
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [2](#0-1) 

Because `execute_replace_class` never reads `contract_class_changes`, it cannot detect that the requested class hash has no corresponding compiled class hash. The OS will commit the updated contract state (with the invalid class hash) to the Patricia tree via `compute_contract_state_commitment`, which hashes the `StateEntry` fields blindly: [3](#0-2) 

The class tree commitment (`compute_class_commitment`) is computed independently and will not contain a leaf for the undeclared hash, creating a permanent inconsistency between the two trees that is baked into the global state root.

**Analog to the Venus bug:** Venus's `_mint` checked "recipient holds *any* Prime token" instead of "recipient holds an *irrevocable* Prime token," blocking a valid upgrade path. Here the OS performs the inverse error: it applies *no* check on the new class hash, allowing an invalid downgrade (valid class → non-existent class). Both are state-transition validation failures in a class/token management context.

---

### Impact Explanation

Once a contract's `class_hash` field in the state tree points to an undeclared hash:

1. Every subsequent call to the contract requires the OS to look up the class in the class tree. The class tree has no leaf for the hash; the sequencer cannot supply valid class data; execution of the contract is impossible.
2. The only on-chain mechanism to repair the class hash is `replace_class` — but that requires executing the contract, which is now impossible. The circular dependency is unbreakable.
3. All ERC-20 balances, NFTs, or other assets held by the contract are permanently inaccessible.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

- Any deployed contract can issue the `replace_class` syscall; no privileged role is required.
- A contract with a public upgrade function that accepts a caller-supplied class hash (a common pattern) can be exploited by any external caller who passes an undeclared hash.
- A contract developer who makes a typo or off-by-one error in a class hash constant will silently freeze their own contract's funds with no OS-level safety net.
- The TODO comment (`// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.`) confirms the gap is known and unresolved.

---

### Recommendation

Inside `execute_replace_class`, after reading `request.class_hash`, perform a `dict_read` on `contract_class_changes` for that key and assert the returned compiled class hash is non-zero (i.e., the class is declared):

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);  // Revert if class is not declared.
```

This mirrors the invariant already enforced in `execute_declare_transaction` (`assert_not_zero(compiled_class_hash)`) and closes the gap between the two trees.

---

### Proof of Concept

1. **Deploy** a contract `Victim` that holds user funds and exposes a public `upgrade(new_class_hash)` function that calls `replace_class(new_class_hash)`.
2. **Craft** a transaction calling `Victim.upgrade(0xdeadbeef)` where `0xdeadbeef` is a felt that has never been passed to `declare`.
3. **OS execution**: `execute_replace_class` deducts gas, reads the current `StateEntry`, writes a new `StateEntry` with `class_hash=0xdeadbeef` into `contract_state_changes`, and returns successfully — no assertion fires.
4. **State commitment**: `compute_contract_state_commitment` hashes the new `StateEntry` (including `class_hash=0xdeadbeef`) and updates the contract state Patricia tree. `compute_class_commitment` processes only `contract_class_changes`, which has no entry for `0xdeadbeef`. Both roots are committed to the global state.
5. **Post-state**: Any call to `Victim` requires the OS to resolve `class_hash=0xdeadbeef` in the class tree. The tree has no leaf; the sequencer cannot provide a valid compiled class; execution is permanently impossible.
6. **Result**: All funds in `Victim` are permanently frozen with no recovery path. [4](#0-3) [5](#0-4) [6](#0-5)

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
