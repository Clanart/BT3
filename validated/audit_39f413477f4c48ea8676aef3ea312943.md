### Title
Missing `state_diff_commitment` Hash Verification in P2P State Diff Sync Allows Malicious Peer to Corrupt Stored State — (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

### Summary

`StateDiffStreamBuilder::parse_data_for_block` validates only the **count** of state diff entries (`state_diff_length`) against the stored block header, but never verifies the **cryptographic commitment** (`state_diff_commitment` = Poseidon hash). A malicious p2p peer can send `StateDiffChunk` values whose entries sum to the correct `target_state_diff_len` but carry arbitrary wrong storage/nonce/class content. The assembled `ThinStateDiff` passes all checks and is written unconditionally to storage via `write_to_storage → append_state_diff`, permanently corrupting `state_diffs_table[block_number]` with a diff whose hash does not match `header.state_diff_commitment`.

### Finding Description

In `parse_data_for_block`:

```
target_state_diff_len = header.state_diff_length   // ← count only, not the hash
while current_state_diff_len < target_state_diff_len { ... }
if current_state_diff_len != target_state_diff_len { return Err(...) }
validate_deprecated_declared_classes_non_conflicting(&result)?;
Ok(Some((result, block_number)))   // ← no hash check before returning
``` [1](#0-0) 

The stored `StorageBlockHeader` carries **both** `state_diff_length: Option<usize>` (the count) and `state_diff_commitment: Option<StateDiffCommitment>` (the Poseidon hash): [2](#0-1) 

`parse_data_for_block` reads only `state_diff_length` and never reads or checks `state_diff_commitment`. After the length check passes, `write_to_storage` calls `append_state_diff` with no further validation: [3](#0-2) 

`append_state_diff` writes the diff directly to the `state_diffs_table` and all derived per-contract tables (nonces, storage, deployed contracts, compiled class hashes) without any hash guard: [4](#0-3) 

The `state_diff_commitment` field **is** populated in the stored header during p2p sync (confirmed by the integration test that asserts `block_header.state_diff_commitment.unwrap() == block_header_commitments.state_diff_commitment`): [5](#0-4) 

So the guard value exists in storage but is never consulted during state diff assembly.

### Impact Explanation

The corrupted `ThinStateDiff` at `state_diffs_table[block_number]` is the authoritative source for:

1. **RPC state queries** — wrong contract storage values, nonces, and class hashes are served to clients.
2. **Class sync** — `ClassStreamBuilder::parse_data_for_block` reads the stored state diff to determine which classes to fetch; a wrong diff causes it to fetch wrong or missing classes.
3. **Global root computation** — any downstream component that applies the stored state diff to compute the Patricia trie root will produce a wrong global root. [6](#0-5) 

The committer's `verify_state_diff_hash` guard exists only in the **sequencer/batcher path** (`CommitBlockRequest`), not in the p2p sync path: [7](#0-6) 

### Likelihood Explanation

Any peer that the syncing node connects to can exploit this. The attacker only needs to craft `StateDiffChunk` messages whose `len()` values sum to `target_state_diff_len` while substituting arbitrary storage keys, nonce values, or class hashes. No operator privilege is required.

### Recommendation

After assembling `result` and before returning `Ok(Some(...))`, add:

```rust
let expected_commitment = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .expect("...")
    .state_diff_commitment
    .ok_or(P2pSyncClientError::OldHeaderInStorage {
        block_number,
        missing_field: "state_diff_commitment",
    })?;
let calculated_commitment = calculate_state_diff_hash(&result);
if calculated_commitment != expected_commitment {
    return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffCommitment {
        expected: expected_commitment,
        calculated: calculated_commitment,
        block_number: block_number.0,
    }));
}
``` [8](#0-7) 

### Proof of Concept

A Rust p2p-sync unit test following the existing `validate_state_diff_fails` pattern would:

1. Store a header for block 0 with `state_diff_length = 2` and `state_diff_commitment = calculate_state_diff_hash(&legitimate_diff)`.
2. Send two `StateDiffChunk::ContractDiff` messages each with `len() == 1` (correct total count) but with a different contract address or storage value than the legitimate diff.
3. Assert that `get_state_diff(BlockNumber(0))` returns the injected wrong diff, and that `calculate_state_diff_hash(&stored_diff) != header.state_diff_commitment`.

The test would currently **pass** (wrong diff is stored), demonstrating the missing guard. [9](#0-8)

    fn append_state_diff(
        self,
        block_number: BlockNumber,
        thin_state_diff: ThinStateDiff,
    ) -> StorageResult<Self> {
        let file_offset_table = self.txn.open_table(&self.tables.file_offsets)?;
        let markers_table = self.open_table(&self.tables.markers)?;
        let state_diffs_table = self.open_table(&self.tables.state_diffs)?;
        let nonces_table = self.open_table(&self.tables.nonces)?;
        let deployed_contracts_table = self.open_table(&self.tables.deployed_contracts)?;
        let storage_table = self.open_table(&self.tables.contract_storage)?;
        let declared_classes_block_table = self.open_table(&self.tables.declared_classes_block)?;
        let deprecated_declared_classes_block_table =
            self.open_table(&self.tables.deprecated_declared_classes_block)?;
        let compiled_class_hash_table = self.open_table(&self.tables.compiled_class_hash)?;

        // Write state.
        write_deployed_contracts(
            &thin_state_diff.deployed_contracts,
            &self.txn,
            block_number,
            &deployed_contracts_table,
            &nonces_table,
        )?;
        write_storage_diffs(
            &thin_state_diff.storage_diffs,
            &self.txn,
            block_number,
            &storage_table,
        )?;
        // Must be called after write_deployed_contracts since the nonces are updated there.
        write_nonces(&thin_state_diff.nonces, &self.txn, block_number, &nonces_table)?;

        for (class_hash, _) in &thin_state_diff.class_hash_to_compiled_class_hash {
            let not_declared = declared_classes_block_table.get(&self.txn, class_hash)?.is_none();
            if not_declared {
                declared_classes_block_table.insert(&self.txn, class_hash, &block_number)?;
            }
        }

        write_compiled_class_hashes(
            &thin_state_diff.class_hash_to_compiled_class_hash,
            &self.txn,
            block_number,
            &compiled_class_hash_table,
        )?;

        for class_hash in thin_state_diff.deprecated_declared_classes.iter() {
            // Cairo0 classes can be declared in different blocks. The first block to declare the
            // class is recorded here.
            if deprecated_declared_classes_block_table.get(&self.txn, class_hash)?.is_none() {
                deprecated_declared_classes_block_table.insert(
                    &self.txn,
                    class_hash,
                    &block_number,
                )?;
            }
        }

        // Write state diff.
        let location = self.file_handlers.append_state_diff(&thin_state_diff);
        state_diffs_table.append(&self.txn, &block_number, &location)?;
        file_offset_table.upsert(&self.txn, &OffsetKind::ThinStateDiff, &location.next_offset())?;

        update_marker_to_next_block(&self.txn, &markers_table, MarkerKind::State, block_number)?;

        advance_compiled_class_marker_over_blocks_without_classes(
            &self.txn,
            &markers_table,
            &state_diffs_table,
            &self.file_handlers,
        )?;

        Ok(self)

async fn validate_state_diff_fails(
    header_state_diff_lengths: Vec<usize>,
    state_diff_chunks: Vec<Option<StateDiffChunk>>,
) {
    let mut rng = get_rng();

    let mut actions = vec![
        Action::RunP2pSync,
        // We already validate the header query content in other tests.
        Action::ReceiveQuery(Box::new(|_query| ()), DataType::Header),
    ];

    // Send headers with corresponding state diff length
    for (i, state_diff_length) in header_state_diff_lengths.iter().copied().enumerate() {
        actions.push(Action::SendHeader(DataOrFin(Some(random_header(
            &mut rng,
            BlockNumber(i.try_into().unwrap()),
            Some(state_diff_length),
            None,
        )))));
    }
    actions.push(Action::SendHeader(DataOrFin(None)));

    actions.push(
        // We already validate the state diff query content in other tests.
        Action::ReceiveQuery(Box::new(|_query| ()), DataType::StateDiff),
    );

    // Send state diff chunks.
    for state_diff_chunk in state_diff_chunks {
        actions.push(Action::SendStateDiff(DataOrFin(state_diff_chunk)));
    }

    actions.push(Action::ValidateReportSent(DataType::StateDiff));

    run_test(
        HashMap::from([
            (DataType::Header, header_state_diff_lengths.len().try_into().unwrap()),
            (DataType::StateDiff, header_state_diff_lengths.len().try_into().unwrap()),
        ]),
        None,
        actions,
    )
    .await;
}
