### Title
Cairo0 Class Declaration Not Verified Against State Number in `ExecutionStateReader::get_compiled_class` When Class Manager Is Active - (`File: crates/apollo_rpc_execution/src/state_reader.rs`)

### Summary

`ExecutionStateReader::get_compiled_class` skips the state-number declaration check for Cairo0 (deprecated) classes when the class manager handle is present. For Cairo1 classes the code explicitly calls `is_contract_class_declared` against `self.state_number` before returning the compiled class; for Cairo0 classes it returns the class unconditionally with a `// TODO(shahak): Verify cairo0 as well` comment. This means any Cairo0 class that exists in the class manager is treated as declared at every historical state number, causing `estimate_fee`, `simulate_transactions`, `starknet_call`, and `trace_transaction` to execute against a class that was not yet declared at the queried block, producing authoritative-looking wrong values.

### Finding Description

In `ExecutionStateReader::get_compiled_class`, when `self.class_manager_handle` is `Some`, the code branches on the class type returned by the class manager:

```rust
// crates/apollo_rpc_execution/src/state_reader.rs  lines 121-141
return match contract_class {
    ContractClass::V1(casm_contract_class) => {
        let is_declared = is_contract_class_declared(
            &self.storage_reader.begin_ro_txn()...,
            &class_hash,
            self.state_number,          // ← checked against state_number
        )?;
        if is_declared {
            Ok(RunnableCompiledClass::V1(casm_contract_class.try_into()?))
        } else {
            Err(StateError::UndeclaredClassHash(class_hash))
        }
    }
    // TODO(shahak): Verify cairo0 as well after get_class_definition_block_number is fixed.
    ContractClass::V0(deprecated_contract_class) => {
        Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
        // ↑ NO state_number check — class returned unconditionally
    }
};
``` [1](#0-0) 

The class manager stores classes globally, without any block-number scope. `get_executable` returns a class as long as it was ever added to the manager, regardless of when it was declared on-chain. [2](#0-1) 

The storage layer does have the correct per-block check via `get_deprecated_class_definition_block_number` / `get_deprecated_class_definition_at`, which returns `None` when the class was not yet declared at the queried state number: [3](#0-2) 

The fallback path (when `class_manager_handle` is `None`) correctly uses `get_contract_class`, which calls `get_deprecated_class_definition_at` and returns `None` → `UndeclaredClassHash` for pre-declaration queries: [4](#0-3) 

The `ExecutionStateReader` with a live class manager handle is constructed for every RPC execution call (`estimate_fee`, `simulate_transactions`, `call`, `trace_transaction`): [5](#0-4) 

The gateway's `SyncStateReader` is **not** affected: it calls `state_sync_client.is_class_declared_at` before fetching from the class manager, and that function correctly checks `get_deprecated_class_definition_block_number <= block_number` for Cairo0 classes: [6](#0-5) [7](#0-6) 

### Impact Explanation

Any RPC caller can query `starknet_estimateFee`, `starknet_simulateTransactions`, `starknet_call`, or `starknet_traceTransaction` at a historical block number that precedes the declaration of a Cairo0 class. The node will execute the transaction using the undeclared class and return a fee estimate, simulation trace, or call result as if the class were valid at that block. The correct response is `UndeclaredClassHash`. The wrong authoritative value can mislead wallets, relayers, and off-chain tooling into believing a transaction is executable when it is not (or into computing wrong fees/traces for historical analysis).

**Scope match:** High — RPC execution, fee estimation, tracing, simulation returns an authoritative-looking wrong value.

### Likelihood Explanation

- The class manager handle is present in all production RPC execution paths.
- Cairo0 (deprecated) classes are widely deployed on Starknet mainnet.
- Any user can craft a query to `estimate_fee` or `simulate_transactions` at a block number before a known Cairo0 class was declared.
- No privileged access is required; the trigger is a standard JSON-RPC call.

### Recommendation

Apply the same `is_contract_class_declared` guard to the `ContractClass::V0` branch. The storage already exposes `get_deprecated_class_definition_block_number` which returns the first block at which the deprecated class was declared. A minimal fix:

```rust
ContractClass::V0(deprecated_contract_class) => {
    // Mirror the Cairo1 check using the deprecated-class block table.
    let declared_block = self.storage_reader
        .begin_ro_txn()
        .map_err(storage_err_to_state_err)?
        .get_state_reader()
        .map_err(storage_err_to_state_err)?
        .get_deprecated_class_definition_block_number(&class_hash)
        .map_err(storage_err_to_state_err)?;

    let is_declared = declared_block
        .map(|bn| !self.state_number.is_before(bn))
        .unwrap_or(false);

    if is_declared {
        Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
    } else {
        Err(StateError::UndeclaredClassHash(class_hash))
    }
}
```

This resolves the TODO comment and makes Cairo0 handling consistent with Cairo1.

### Proof of Concept

1. On a node with a class manager configured, identify a Cairo0 class `C` declared at block `N` (e.g., any legacy account class on mainnet).
2. Send `starknet_estimateFee` with a transaction that invokes a contract whose class hash is `C`, specifying `block_id = N-1` (one block before declaration).
3. **Expected:** The node returns `CLASS_HASH_NOT_FOUND` or `UndeclaredClassHash`.
4. **Actual:** The node fetches `C` from the class manager (which has no block-number scope), skips the declaration check, and returns a non-zero fee estimate — an authoritative-looking wrong value.

The same applies to `starknet_simulateTransactions`, `starknet_call`, and `starknet_traceTransaction` at any `block_id < N`. [8](#0-7) [9](#0-8)

pub(crate) fn get_contract_class(
    txn: &StorageTxn<'_, RO>,
    class_hash: &ClassHash,
    state_number: StateNumber,
) -> Result<Option<RunnableCompiledClass>, ExecutionUtilsError> {
    match txn.get_state_reader()?.get_class_definition_block_number(class_hash)? {
        Some(block_number) if state_number.is_before(block_number) => return Ok(None),
        Some(_block_number) => {
            let (Some(casm), Some(sierra)) = txn.get_casm_and_sierra(class_hash)? else {
                return Err(ExecutionUtilsError::CasmTableNotSynced);
            };
            let sierra_version =
                sierra.get_sierra_version().map_err(ExecutionUtilsError::SierraValidationError)?;
            return Ok(Some(RunnableCompiledClass::V1(CompiledClassV1::try_from((
                casm,
                sierra_version,
            ))?)));
        }
        None => {}
    };

    let Some(deprecated_class) =
        txn.get_state_reader()?.get_deprecated_class_definition_at(state_number, class_hash)?
    else {
        return Ok(None);
    };
    Ok(Some(RunnableCompiledClass::V0(
        CompiledClassV0::try_from(deprecated_class).map_err(ExecutionUtilsError::ProgramError)?,
    )))
}

    fn get_contract_class_from_client(&self, class_hash: ClassHash) -> StateResult<ContractClass> {
        let is_class_declared = self
            .runtime
            .block_on(self.state_sync_client.is_class_declared_at(self.block_number, class_hash))
            .map_err(|e| StateError::StateReadError(e.to_string()))?;

        if !is_class_declared {

            .runtime
            .block_on(self.class_manager_client.get_executable(class_hash))
            .map_err(|e| StateError::StateReadError(e.to_string()))?

                    "Class with hash {class_hash:?} doesn't appear in class manager even though \
                     it was declared"
                )
            });

        Ok(contract_class)
    }
