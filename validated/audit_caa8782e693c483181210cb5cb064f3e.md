### Title
Unchecked Bootstrap Declare Bypass Allows Fee-Free, Validation-Free Class Declaration - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

In `execute_declare_transaction`, a special "bootstrap" code path allows any unprivileged user to declare contract classes without paying fees, without running `__validate_declare__`, and without incrementing the sender's nonce. All four conditions that gate this path are fully attacker-controllable, enabling unlimited fee-free class declarations per block.

---

### Finding Description

`execute_declare_transaction` contains a bootstrap shortcut at lines 764–776:

```cairo
if (sender_address == 'BOOTSTRAP' and tx_info.nonce == 0 and tx_info.version == 3) {
    let max_possible_fee = compute_max_possible_fee(tx_info=tx_info);
    if (max_possible_fee == 0) {
        assert_not_zero(compiled_class_hash);
        dict_update{dict_ptr=contract_class_changes}(
            key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
        );
        %{ SkipTx %}
        return ();
    }
}
```

When this branch is taken, the function returns immediately, skipping:

1. **`check_and_increment_nonce`** (line 779) — the nonce at the bootstrap address is never incremented.
2. **`non_reverting_select_execute_entry_point_func`** for `__validate_declare__` (line 804) — no signature or account validation is performed.
3. **`charge_fee`** (line 822) — no fee is transferred to the sequencer.

All four gate conditions are fully user-controlled in a declare transaction:

| Condition | Attacker control |
|---|---|
| `sender_address == 'BOOTSTRAP'` | Felt literal; any user sets `sender_address` in the tx |
| `tx_info.nonce == 0` | User-supplied nonce field |
| `tx_info.version == 3` | User-supplied version field |
| `max_possible_fee == 0` | User sets all resource bounds to zero |

Because the nonce is **never incremented** in this path, the bootstrap address's nonce remains `0` in `contract_state_changes` for the entire block. An attacker can therefore submit an unbounded number of declare transactions in the same block, each satisfying `nonce == 0`, each declaring a distinct class hash for free.

The only per-class guard is `prev_value=0` in `dict_update`, which prevents re-declaring the same class hash. This does not limit the number of *distinct* classes an attacker can declare. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

---

### Impact Explanation

**Direct loss of funds (Critical).**

The sequencer earns revenue by collecting fees from every transaction. The bootstrap path completely bypasses `charge_fee`. An attacker who submits declare transactions with `sender_address = felt('BOOTSTRAP')`, `nonce = 0`, `version = 3`, and all resource bounds set to zero will have those transactions accepted and proven by the OS without any fee being transferred to the sequencer. Because the nonce is never incremented, the attacker can repeat this for every distinct class hash they wish to declare, draining the sequencer of all fee revenue for declare transactions at zero cost.

Additionally, classes are declared without running `__validate_declare__`, meaning the account abstraction security model is bypassed: no signature is checked, and no account contract at the bootstrap address is required to exist. [5](#0-4) [6](#0-5) 

---

### Likelihood Explanation

**High.** The attack requires no privileged access, no leaked keys, and no external dependency. Any user who can submit a declare transaction to the network can exploit this. The felt value of `'BOOTSTRAP'` is a deterministic, publicly computable constant. The remaining conditions (nonce=0, version=3, zero resource bounds) are trivially satisfiable. The attack is repeatable within every block. [7](#0-6) 

---

### Recommendation

The bootstrap path must be restricted to a privileged, operator-controlled context. Concretely:

1. **Remove the bootstrap shortcut from the on-chain OS Cairo code entirely.** Bootstrap class declarations should be handled off-chain or via a separate, access-controlled mechanism before the network is live.
2. If the bootstrap path must remain, gate it with a **cryptographic proof of operator authorization** (e.g., a signature from a known sequencer key verified inside the Cairo code), not a plain felt comparison on `sender_address`.
3. At minimum, **increment the nonce** even in the bootstrap path to prevent the same address from declaring multiple classes with `nonce == 0` in a single block. [8](#0-7) 

---

### Proof of Concept

1. Compute `felt('BOOTSTRAP')` — the ASCII encoding of the string "BOOTSTRAP" as a Cairo felt. This is a fixed, public value.
2. Construct a v3 declare transaction with:
   - `sender_address = felt('BOOTSTRAP')`
   - `nonce = 0`
   - `version = 3`
   - `l1_gas_max_amount = 0`, `l2_gas_max_amount = 0`, `l1_data_gas_max_amount = 0` (all resource bounds zero → `max_possible_fee = 0`)
   - `class_hash` = hash of any valid Sierra class not yet declared
   - `compiled_class_hash` = corresponding CASM hash (non-zero)
3. Submit the transaction. The OS executes `execute_declare_transaction`, hits the bootstrap branch at line 764, passes both `if` checks, calls `dict_update` to register the class, and returns — skipping nonce increment, `__validate_declare__`, and `charge_fee`.
4. Repeat step 2–3 with a different `class_hash` in the same block. Because the nonce at `felt('BOOTSTRAP')` was never incremented, `nonce == 0` is still satisfied.
5. The OS proof is generated and accepted with all classes declared and zero fees paid. [1](#0-0) [9](#0-8)

func get_initial_user_gas_bound(common_tx_fields: CommonTxFields*) -> felt {
    assert common_tx_fields.n_resource_bounds = 3;
    return common_tx_fields.resource_bounds[L2_GAS_INDEX].max_amount;

// Represents the calldata of an ERC20 transfer.
struct TransferCallData {
    recipient: felt,
    amount: Uint256,
}

// Returns the maximum possible fee that can be charged for the transaction.
func compute_max_possible_fee(tx_info: TxInfo*) -> felt {
    tempvar resource_bounds: ResourceBounds* = tx_info.resource_bounds_start;
    let n_resource_bounds = (tx_info.resource_bounds_end - resource_bounds) / ResourceBounds.SIZE;

    // Only V3 transactions with all resource bounds are supported.
    assert tx_info.version = 3;
    assert n_resource_bounds = 3;

    tempvar l1_gas_bounds: ResourceBounds = resource_bounds[L1_GAS_INDEX];
    tempvar l2_gas_bounds: ResourceBounds = resource_bounds[L2_GAS_INDEX];
    tempvar l1_data_gas_bounds = resource_bounds[L1_DATA_GAS_INDEX];

    return l1_gas_bounds.max_amount * l1_gas_bounds.max_price_per_unit + l2_gas_bounds.max_amount *
        (l2_gas_bounds.max_price_per_unit + tx_info.tip) + l1_data_gas_bounds.max_amount *
        l1_data_gas_bounds.max_price_per_unit;

        block_context=block_context, execution_context=&execution_context
