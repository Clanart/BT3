### Title
Partial `ft_on_transfer` Return Not Handled in `fin_transfer_send_tokens_callback` ŌĆö Tokens Permanently Locked and Accounting Corrupted - (File: near/omni-bridge/src/lib.rs)

### Summary

`is_refund_required` uses a binary check (`ft_resolve_transfer result == 0`) to decide whether to revert a finalized transfer. When a recipient's `ft_on_transfer` returns a **partial** unused amount (between 1 and `amount_without_fee - 1`), the bridge treats the transfer as fully successful, emits `FinTransferEvent`, but never burns (for deployed tokens) or re-locks (for native tokens) the returned portion. Those tokens are permanently stranded in the bridge contract with no recovery path.

### Finding Description

**Root cause ŌĆö `is_refund_required`** [1](#0-0) 

```rust
fn is_refund_required(is_ft_transfer_call: bool) -> bool {
    if is_ft_transfer_call {
        match env::promise_result_checked(0, MAX_FT_TRANSFER_CALL_RESULT) {
            Ok(value) => {
                if let Ok(amount) = near_sdk::serde_json::from_slice::<U128>(&value) {
                    amount.0 == 0   // ŌåÉ only triggers refund when ALL tokens are returned
                } else { false }
            }
            Err(_) => false,
        }
    } else { false }
}
```

Under NEP-141, `ft_resolve_transfer` returns the **used** amount. If `ft_on_transfer` returns N unused tokens, `ft_resolve_transfer` returns `amount - N`. Because `amount - N > 0`, `is_refund_required` returns `false` and the bridge takes the success branch.

**Caller ŌĆö `fin_transfer_send_tokens_callback`** [2](#0-1) 

When `is_refund_required` is `false`:
- `burn_tokens_if_needed` is **not** called ŌåÆ N returned deployed tokens sit in the bridge, unburned.
- `revert_lock_actions` is **not** called ŌåÆ `locked_tokens` stays decremented by the full amount even though N tokens came back.
- `FinTransferEvent` is emitted as if the full transfer succeeded.

**`send_tokens` path for deployed tokens** [3](#0-2) 

For deployed tokens with a non-empty `msg`, `send_tokens` calls `mint(recipient, amount, Some(msg))`.

**`OmniToken::mint` with `msg`** [4](#0-3) 

```rust
fn mint(&mut self, account_id: AccountId, amount: U128, msg: Option<String>) -> PromiseOrValue<U128> {
    if let Some(msg) = msg {
        self.token.internal_deposit(&env::predecessor_account_id(), amount.into()); // mints to bridge
        self.ft_transfer_call(account_id, amount, None, msg)                        // then transfers to recipient
    } ...
}
```

Tokens are first minted to the bridge, then forwarded via `ft_transfer_call`. If `ft_on_transfer` returns N, those N tokens are transferred back to the bridge by `ft_resolve_transfer`. The bridge now holds N deployed tokens that are never burned.

**`send_tokens` path for native tokens** [5](#0-4) 

For native tokens with a non-empty `msg`, `ft_transfer_call` is called directly. N returned tokens land in the bridge's token balance. `locked_tokens` was already decremented by the full amount in `unlock_tokens_if_needed`: [6](#0-5) 

The bridge now holds N native tokens that are not tracked in `locked_tokens`.

**`process_fin_transfer_to_near` ŌĆö where `lock_actions` and `is_ft_transfer_call` are set** [7](#0-6) 

### Impact Explanation

**Deployed tokens (bridged assets, e.g., ETH.omni):**  
N tokens are minted and returned to the bridge but never burned. Total supply of the bridged token exceeds the amount locked on the origin chain. This is unbacked supply ŌĆö a direct break of bridge collateralization. If the bridge is ever upgraded to allow token recovery, those tokens can be extracted without any origin-chain backing.

**Native tokens (e.g., USDC locked on NEAR):**  
`locked_tokens` is decremented by the full amount, but N tokens are returned to the bridge's balance. These N tokens are permanently stranded ŌĆö no bridge function can recover them. This is an irrecoverable lock of user/protocol funds.

Both cases match the allowed impact: **permanent freezing/irrecoverable lock of funds** and **accounting corruption that breaks bridge collateralization**.

### Likelihood Explanation

Any bridge transfer to a NEAR recipient with a non-empty `msg` field triggers the `ft_transfer_call` path. A malicious or even a legitimately-designed recipient contract that returns a partial amount from `ft_on_transfer` (valid NEP-141 behavior) will trigger this bug. The attacker controls both the recipient address and the `msg` field when initiating a transfer on the origin chain. No privileged access is required.

### Recommendation

Replace the binary `amount.0 == 0` check with a comparison against the full expected amount. If `ft_resolve_transfer` returns less than the full amount, the difference (returned tokens) must be handled:

- For deployed tokens: burn the returned amount (`burn_tokens_if_needed(token, returned_amount)`).
- For native tokens: re-lock the returned amount (`revert_lock_actions` proportionally, or call `lock_tokens_if_needed` for the returned portion).

Alternatively, block transfers with a non-empty `msg` to NEAR recipients entirely (analogous to the Sentiment fix of blocking intermediate amounts > 0), accepting only the all-or-nothing semantics that the current binary check already handles correctly.

### Proof of Concept

1. Attacker deploys `MaliciousReceiver` on NEAR implementing `ft_on_transfer` to always return `amount / 2` (half unused).
2. On Ethereum, attacker calls `initTransfer` sending 1000 USDC to `MaliciousReceiver` on NEAR with `msg = "trigger"`.
3. Relayer calls `fin_transfer` on NEAR. Bridge verifies proof, calls `process_fin_transfer_to_near`.
4. `send_tokens` calls `ft_transfer_call(MaliciousReceiver, 999, None, "trigger")` (999 = amount_without_fee).
5. `MaliciousReceiver.ft_on_transfer` returns `U128(499)` ŌåÆ 499 tokens unused.
6. `ft_resolve_transfer` returns `500` (used amount). Bridge's `is_refund_required` sees `500 != 0` ŌåÆ returns `false`.
7. `fin_transfer_send_tokens_callback` takes the success branch: emits `FinTransferEvent`, sends fee to relayer, does NOT burn or re-lock.
8. Result: 499 USDC tokens are permanently stuck in the bridge contract. `locked_tokens` for USDC is decremented by 999 but the bridge holds 499 untracked tokens. For deployed tokens, 499 unbacked tokens inflate total supply.

    pub fn fin_transfer_send_tokens_callback(
        &mut self,
        #[serializer(borsh)] transfer_message: TransferMessage,
        #[serializer(borsh)] fee_recipient: &AccountId,
        #[serializer(borsh)] is_ft_transfer_call: bool,
        #[serializer(borsh)] storage_owner: &AccountId,
        #[serializer(borsh)] lock_actions: Vec<LockAction>,
    ) {
        let token = self.get_token_id(&transfer_message.token);

        if Self::is_refund_required(is_ft_transfer_call) {
            self.burn_tokens_if_needed(
                token.clone(),
                U128(
                    transfer_message
                        .amount_without_fee()
                        .near_expect(BridgeError::InvalidFee),
                ),
            );

            self.revert_lock_actions(&lock_actions);

            self.remove_fin_transfer(&transfer_message.get_transfer_id(), storage_owner);

            env::log_str(
                &OmniBridgeEvent::FailedFinTransferEvent { transfer_message }.to_log_string(),
            );
        } else {
            // Send fee to the fee recipient
            if transfer_message.fee.fee.0 > 0 {
                if self.is_deployed_token(&token) {
                    ext_token::ext(token)
                        .with_static_gas(MINT_TOKEN_GAS)
                        .mint(fee_recipient.clone(), transfer_message.fee.fee, None)
                        .detach();
                } else {
                    ext_token::ext(token)
                        .with_attached_deposit(ONE_YOCTO)
                        .with_static_gas(FT_TRANSFER_GAS)
                        .ft_transfer(fee_recipient.clone(), transfer_message.fee.fee, None)
                        .detach();
                }
            }

            if transfer_message.fee.native_fee.0 > 0 {
                let native_token_id = self.get_native_token_id(transfer_message.get_origin_chain());

                ext_token::ext(native_token_id)
                    .with_static_gas(MINT_TOKEN_GAS)
                    .mint(fee_recipient.clone(), transfer_message.fee.native_fee, None)
                    .detach();
            }

            env::log_str(&OmniBridgeEvent::FinTransferEvent { transfer_message }.to_log_string());
        }
    }

    fn unlock_tokens(
        &mut self,
        chain_kind: ChainKind,
        token_id: &AccountId,
        amount: u128,
    ) -> LockAction {
        let key = (chain_kind, token_id.clone());
        let Some(available) = self.locked_tokens.get(&key) else {
            return LockAction::Unchanged;
        };
        require!(
            available >= amount,
            TokenLockError::InsufficientLockedTokens.as_ref()
        );

        let remaining = available - amount;
        self.locked_tokens.insert(&key, &remaining);

        LockAction::Unlocked {
            chain_kind,
            token_id: token_id.clone(),
            amount,
        }
    }
