### Title
Integer Division Truncation in `NewDynamicFeeChecker` Allows Cosmos SDK Transactions to Bypass Fee Payment When BaseFee Is Zero - (File: ante/evm/fee_checker.go)

### Summary
`NewDynamicFeeChecker` computes `feeCap` via integer division (`fee / gas`). When the total fee amount is less than the gas limit (in absolute token units), `feeCap` truncates to zero. When `baseFee` is also zero (either `NoBaseFee=true` or the base fee has been driven to zero), the fee sufficiency check passes and the computed `effectiveFee` is zero — meaning the sender pays nothing despite specifying a non-zero fee.

### Finding Description
In `ante/evm/fee_checker.go`, `NewDynamicFeeChecker` computes the per-gas fee cap using integer division:

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
```

`sdkmath.Int.Quo` performs truncating integer division. For any Cosmos SDK transaction where `fee < gas` in absolute token units (e.g., fee = 1 attotoken, gas = 1,000,000), `feeCap` truncates to `0`.

The subsequent sufficiency check is:

```go
if feeCap.LT(baseFeeInt) {
    return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, ...)
}
```

When `baseFee = 0` (reachable when `NoBaseFee = true` with London enabled, or when the base fee has been driven to zero via sustained low-gas blocks), `feeCap(0) < baseFeeInt(0)` evaluates to `false`, so the check passes silently.

The effective fee is then computed as:

```go
effectivePrice := sdkmath.NewIntFromBigInt(types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
// EffectiveGasPrice(0, 0, MaxInt64) = min(MaxInt64 + 0, 0) = 0

effectiveFee := sdk.Coins{{Denom: denom, Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas))}}
// effectiveFee = 0 * gas = 0
```

`DeductFeeDecorator.checkDeductFee` then deducts `effectiveFee` (zero), so the sender pays nothing.

The zero-baseFee condition is reachable in two ways:
1. `NoBaseFee = true` — `GetBaseFee` in `x/evm/types/utils.go` returns `new(big.Int)` (zero) rather than `nil` when London is enabled and `NoBaseFee = true`, so the code does not fall back to `checkTxFeeWithValidatorMinGasPrices`.
2. Base fee driven to zero — `CalculateBaseFee` in `x/feemarket/keeper/eip1559.go` floors the base fee at `MinGasPrice`, which defaults to zero, so sustained low-gas blocks can drive `baseFee` to zero.

### Impact Explanation
Any unprivileged user can submit Cosmos SDK transactions (non-EVM) with a nominal fee of 1 attotoken and an arbitrarily large gas limit, paying zero effective fees. Fee revenue is entirely lost for those transactions. With `MinGasPrice = 0` (default), no other ante decorator catches this. This constitutes a fee mis-accounting bug in the ante handler path, matching the allowed High impact: *"ante handler... bug that permits... valid user funds/fees to be mis-accounted."*

### Likelihood Explanation
- `DynamicFeeChecker = true` is the intended production configuration for EIP-1559 chains.
- `MinGasPrice = 0` is the default value.
- `NoBaseFee = true` is a documented, supported configuration mode.
- Even without `NoBaseFee`, the base fee can be driven to zero naturally when `MinGasPrice = 0` and blocks are consistently below the gas target.
- No privileged access is required; any user submitting a Cosmos SDK transaction can trigger this.

### Recommendation
Replace the integer division with decimal arithmetic to preserve sub-unit precision:

```go
// Before (truncates to 0 when fee < gas):
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))

// After (preserves fractional gas price):
feeCapDec := sdkmath.LegacyNewDecFromInt(fee).QuoInt64(int64(gas))
```

Compare `feeCapDec` against `baseFeeInt` using decimal comparison, and compute `effectivePrice` accordingly. Alternatively, enforce a minimum `feeCap > 0` check before proceeding when `baseFee > 0`, and reject the transaction when `baseFee = 0` and `feeCap = 0` unless `MinGasPrice` also permits zero-fee transactions explicitly.

	}
