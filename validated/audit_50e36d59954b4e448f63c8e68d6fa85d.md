### Title
Missing Declared Class Validation in `execute_replace_class` Enables Permanent Fund Freezing — (`execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the new class hash supplied by a contract corresponds to a previously declared class. This allows a contract to replace its own class hash with an arbitrary, undeclared value, creating an irrecoverable state where the contract can never be called again and any funds it holds are permanently frozen.

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the `class_hash` from the syscall request and directly writes it into the contract's `StateEntry` without checking whether that hash exists in `contract_class_changes` (the set of declared classes for the current block) or in the global class commitment tree.

The code itself contains an explicit acknowledgment of this missing check:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The same missing validation exists in the deprecated syscall path: [2](#0-1) 

By contrast, the `execute_deploy` path implicitly validates the class hash because the OS must locate and execute the constructor entry point — if the class is undeclared, execution fails before state is committed. `replace_class` has no such implicit check: it only writes to state. [3](#0-2) 

The class commitment tree maps class hashes to compiled class hashes via `contract_class_changes`: [4](#0-3) 

A class hash written into `StateEntry` that has no entry in `contract_class_changes` and no entry in the global class tree will cause every future OS execution attempt against that contract to fail at class lookup, with no recovery path.

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` field in `StateEntry` is set to an undeclared hash, the OS cannot locate a `CompiledClassFact` for it. Every subsequent transaction targeting that contract — including any attempt to withdraw or transfer funds held by it — will fail at the class-lookup stage. Because the OS is the proof system enforcing state validity, this state is accepted on-chain and is irrecoverable without a protocol-level upgrade. Any ERC-20 balances, ETH, or other assets held by the affected contract are permanently frozen. [5](#0-4) 

### Likelihood Explanation

The `replace_class` syscall is callable by any contract on itself. The attacker-controlled entry path requires no privileged role:

1. An attacker deploys a contract (or exploits an existing contract that exposes `replace_class` to user-supplied input).
2. The contract issues a `replace_class` syscall with an arbitrary, undeclared `class_hash`.
3. The OS accepts the state update without validation (the missing check is explicitly noted in the TODO).
4. The contract's `class_hash` in `StateEntry` is now an undeclared value.
5. All future calls to the contract fail; funds are permanently frozen.

The OS is the last line of defense for protocol invariants. The invariant that every `class_hash` stored in `StateEntry` must correspond to a declared class is broken here.

### Recommendation

Before writing the new `class_hash` into `StateEntry`, verify that it exists in `contract_class_changes` (declared in the current block) or in the global class commitment tree. Concretely, perform a `dict_read` on `contract_class_changes` for the given `class_hash` and assert the result is non-zero, analogous to how `execute_declare_transaction` enforces `assert_not_zero(compiled_class_hash)` before writing to `contract_class_changes`. [6](#0-5) 

### Proof of Concept

1. Attacker deploys contract `V` (a vault holding user funds) whose implementation calls `replace_class(request.new_hash)` when triggered by an external call.
2. Attacker calls `V` with `new_hash = 0xdeadbeef` (an undeclared class hash).
3. `execute_replace_class` in `syscall_impls.cairo` writes `class_hash=0xdeadbeef` into `V`'s `StateEntry` with no validation.
4. The OS generates a valid proof for this block (the missing check means no constraint is violated).
5. In any subsequent block, any transaction targeting `V` causes the OS to look up compiled class `0xdeadbeef` in `validate_compiled_class_facts` — it is not found, execution fails.
6. All funds in `V` are permanently frozen with no recovery path.

// Validates the compiled class facts structure and hash, using the hint variable
// `bytecode_segment_structures` - a mapping from compilied class hash to the structure.
func validate_compiled_class_facts{poseidon_ptr: PoseidonBuiltin*, range_check_ptr}(
    n_compiled_class_facts, compiled_class_facts: CompiledClassFact*, builtin_costs: felt*
) {
    if (n_compiled_class_facts == 0) {
        return ();
    }
    alloc_locals;

    let compiled_class_fact = &compiled_class_facts[0];
    let compiled_class = compiled_class_fact.compiled_class;

    validate_entry_points(
        n_entry_points=compiled_class.n_external_functions,
        entry_points=compiled_class.external_functions,
    );

    validate_entry_points(
        n_entry_points=compiled_class.n_l1_handlers, entry_points=compiled_class.l1_handlers
    );
    // Compiled classes are expected to end with a `ret` opcode followed by a pointer to the
    // builtin costs.
    assert compiled_class.bytecode_ptr[compiled_class.bytecode_length] = 0x208b7fff7fff7ffe;
    assert compiled_class.bytecode_ptr[compiled_class.bytecode_length + 1] = cast(
        builtin_costs, felt
    );

    // Calculate the compiled class hash.
    // This hint enters a new scope that contains the bytecode segment structure of the class.
    %{ EnterScopeWithBytecodeSegmentStructure %}
    let (hash) = blake_compiled_class_hash(compiled_class, full_contract=FALSE);
    %{ LoadClass %}

    assert compiled_class_fact.hash = hash;

    return validate_compiled_class_facts(
        n_compiled_class_facts=n_compiled_class_facts - 1,
        compiled_class_facts=&compiled_class_facts[1],
        builtin_costs=builtin_costs,
    );
}
