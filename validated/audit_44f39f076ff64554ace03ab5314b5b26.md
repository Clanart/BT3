### Title
Missing Class Hash Existence Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS accepts and commits an arbitrary `class_hash` value into the contract state without verifying that the hash corresponds to a previously declared contract class. This is the direct analog of using `_mint` instead of `_safeMint`: a critical state transition is performed without the safety check that ensures the target is valid. Any contract that calls `replace_class` with an undeclared class hash will have its state permanently corrupted, freezing all funds held within it.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the requested `class_hash` from the syscall request and immediately writes it into `contract_state_changes` with no validation that the hash is a declared class:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The developer-acknowledged TODO at line 898 explicitly states the missing check. The `class_hash` field is taken directly from the `ReplaceClassRequest` struct without any cross-reference against `contract_class_changes` (the dictionary of declared classes). [2](#0-1) 

By contrast, the `execute_declare_transaction` function in `transaction_impls.cairo` does enforce that a class hash is valid before writing it — it calls `finalize_class_hash` and asserts the computed hash matches the declared one, and uses `prev_value=0` to enforce uniqueness: [3](#0-2) 

No equivalent guard exists in `execute_replace_class`.

---

### Impact Explanation

Once `execute_replace_class` commits an undeclared `class_hash` into `contract_state_changes`, the contract's `StateEntry` permanently records that invalid hash: [4](#0-3) 

Every subsequent call to that contract — whether via `call_contract`, `invoke`, or `l1_handler` — will attempt to dispatch execution using the stored `class_hash`. Since no class bytecode exists for that hash, execution will fail unconditionally. There is no recovery path: `replace_class` itself requires a successful execution to run, which is impossible once the class hash is invalid. All ERC-20 tokens, ETH, or other assets held in the contract's storage are permanently inaccessible.

**Allowed impact matched**: Critical — Permanent freezing of funds.

---

### Likelihood Explanation

The `replace_class` syscall is callable by any contract during its own execution. Reachable scenarios include:

1. **Malicious contract deployer**: An attacker deploys a contract (e.g., a fake vault or pool) that accepts user deposits and then calls `replace_class` with a fabricated, undeclared hash, permanently freezing deposited funds.
2. **Buggy upgrade logic**: A legitimate contract with a user-controlled upgrade path (e.g., a DAO-governed proxy) can be tricked into passing an attacker-supplied hash to `replace_class`.
3. **Direct self-destruction pattern**: Any contract can call `replace_class(0)` or any non-existent hash to irreversibly brick itself, taking user funds with it.

No privileged role, leaked key, or external dependency is required. The attacker entry path is a standard `invoke` transaction targeting a contract that calls the `replace_class` syscall.

---

### Recommendation

Before committing the new `class_hash` to `contract_state_changes`, verify that it exists in `contract_class_changes` (the declared class dictionary). Specifically, perform a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert the result is non-zero (i.e., a compiled class hash has been registered for it). This mirrors the safety check that `_safeMint` adds over `_mint` — ensuring the target state is valid before the transition is committed.

```cairo
// Proposed fix (pseudocode):
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

---

### Proof of Concept

1. Attacker deploys contract `VaultAttack` with an `__execute__` entry point that:
   - Accepts ETH/ERC-20 deposits from users.
   - On a trigger call, invokes the `replace_class` syscall with `class_hash = 0xdeadbeef` (any undeclared hash).
2. Attacker submits an `invoke` transaction calling the trigger.
3. The OS processes `execute_replace_class`:
   - Reads `request.class_hash = 0xdeadbeef`.
   - Skips the missing existence check (line 898 TODO).
   - Writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes`.
4. Block is proven and state is committed. `VaultAttack`'s `class_hash` is now `0xdeadbeef`.
5. Any subsequent call to `VaultAttack` dispatches against class `0xdeadbeef`, which has no bytecode. Execution fails permanently.
6. All user deposits are frozen with no recovery mechanism. [5](#0-4)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

        key=contract_address,

    );

}
