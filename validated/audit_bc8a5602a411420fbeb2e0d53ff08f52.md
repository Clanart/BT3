### Title
Missing Validation of `class_hash` in `execute_replace_class` Allows Permanent Contract Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts an arbitrary `class_hash` from the caller without verifying that the hash corresponds to a previously declared class. This is directly analogous to the external report's pattern: a user-supplied parameter is accepted and applied to state without validation. A contract can replace its own class with an undeclared class hash, permanently rendering itself unexecutable and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads `class_hash` directly from the syscall request and writes it into `contract_state_changes` without any check that the hash exists in `contract_class_changes`:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment at line 898 explicitly acknowledges the missing check. The same flaw exists in the deprecated path in `deprecated_execute_syscalls.cairo` at `execute_replace_class` (lines 307–329), which also applies `syscall_ptr.class_hash` directly to state with no existence check.

The OS is the authoritative enforcer of the protocol invariant that every contract's `class_hash` must point to a declared class. Because this check is absent, the invariant can be broken by any contract that issues a `replace_class` syscall.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` is set to an undeclared value, every subsequent call to that contract will fail at class resolution time (the OS cannot find the class bytecode). The contract becomes permanently unexecutable. Any ERC-20 tokens, ETH, or other assets held in the contract's storage are irrecoverably locked, because no entry point — including withdrawal or rescue functions — can ever be invoked again.

---

### Likelihood Explanation

The `replace_class` syscall is callable by any deployed contract on behalf of itself. No privileged role is required. A single transaction from an unprivileged account that deploys a contract and immediately calls `replace_class` with an arbitrary felt value is sufficient to trigger the bug. The attack requires no special knowledge beyond the existence of the syscall.

---

### Recommendation

Before applying the new class hash to `contract_state_changes`, assert that the hash is present in `contract_class_changes` (i.e., that it was declared in the current or a prior block). The TODO comment at line 898 already identifies this as the required fix:

```cairo
// Verify the class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

Apply the same fix to the deprecated path in `deprecated_execute_syscalls.cairo`.

---

### Proof of Concept

1. Attacker deploys contract `C` holding user funds (e.g., a simple vault).
2. Attacker calls any entry point on `C` that issues the `replace_class` syscall with `class_hash = 0xdeadbeef` (an arbitrary undeclared felt).
3. The OS executes `execute_replace_class`:
   - Reads `class_hash = 0xdeadbeef` from the request. [1](#0-0) 
   - Skips the missing existence check (acknowledged by the TODO). [2](#0-1) 
   - Writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes` for contract `C`. [3](#0-2) 
4. The block is proven and the new state root commits `C.class_hash = 0xdeadbeef`.
5. Any future call to `C` fails: the OS looks up `0xdeadbeef` in the class tree, finds nothing, and cannot execute any entry point.
6. All funds in `C`'s storage are permanently frozen.

The same path is reachable through the deprecated syscall handler: [4](#0-3)

    contract_address, syscall_ptr: ReplaceClass*

    let class_hash = syscall_ptr.class_hash;

    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(

        key=contract_address,
