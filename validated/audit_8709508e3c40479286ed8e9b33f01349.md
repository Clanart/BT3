### Title
Missing Hash-Collision Guard in `upsert` Corrupts `leaf_hash_to_index` Cache and Merkle Tree State — (`crates/chia-datalayer/src/merkle/blob.rs`)

### Summary

`MerkleBlob::upsert` does not check whether the supplied `new_hash` already exists in `leaf_hash_to_index` for a **different** key. `MerkleBlob::insert` has this guard; `upsert` does not. An unprivileged caller can overwrite another key's cache entry, causing `get_node_by_hash` to return the wrong key/value and leaving the blob with two leaves sharing the same hash, corrupting the Merkle tree root.

---

### Finding Description

`insert` guards against duplicate leaf hashes: [1](#0-0) 

```rust
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
```

`upsert` has no equivalent check: [2](#0-1) 

The call sequence inside `upsert` is:

1. `block_status_cache.remove_leaf(&leaf)` — removes `hash_A` from `leaf_hash_to_index` and frees `idx_A`. [3](#0-2) 

2. The leaf's hash field is overwritten with `new_hash` (= `hash_B`). [4](#0-3) 

3. `insert_entry_to_blob` calls `add_leaf`, which does an unconditional `HashMap::insert` — silently overwriting `leaf_hash_to_index[hash_B]` from `idx_B` to `idx_A`. [5](#0-4) 

After the call:

| Structure | State |
|---|---|
| `leaf_hash_to_index` | `{hash_B → idx_A}` — key_B's entry is gone |
| `key_to_index` | `{key_A → idx_A, key_B → idx_B}` — 2 entries |
| blob at `idx_A` | key_A leaf with `hash_B` |
| blob at `idx_B` | key_B leaf with `hash_B` (unchanged) |

Two leaves now share `hash_B` in the blob, and the cache maps `hash_B` to the wrong index.

---

### Impact Explanation

- **`get_node_by_hash(hash_B)`** resolves through `get_index_by_leaf_hash` → returns `idx_A` → returns key_A's `(key, value)` instead of key_B's. State lookups are silently wrong. [6](#0-5) 

- **Tree root corruption**: `calculate_lazy_hashes` recomputes internal hashes from the blob. With two leaves carrying `hash_B`, the recomputed root hash is wrong, invalidating all subsequent Merkle proofs. [7](#0-6) 

- **`check_integrity` detects it** — `leaf_count (2) != leaf_hash_to_index_cache_length (1)` raises `IntegrityLeafHashToIndexCacheLengthError` — but only if called after the fact; the corruption is already committed. [8](#0-7) 

This matches the allowed High impact: *DataLayer Merkle proof/blob/delta logic corrupts tree roots or lets untrusted input prove invalid state.*

---

### Likelihood Explanation

The Python binding exposes `upsert` with a fully caller-controlled `new_hash: bytes32`: [9](#0-8) 

No hash collision is required. The attacker only needs to know `hash_B` (publicly readable from the tree via `get_hashes_indexes` or `get_node_by_hash`) and call `upsert(key_A, any_value, hash_B)`. The operation succeeds silently with no error.

---

### Recommendation

Add the same guard present in `insert` at the top of `upsert`, **after** confirming the key exists (to avoid false-positives where the key's own current hash equals `new_hash`):

```rust
pub fn upsert(&mut self, key: KeyId, value: ValueId, new_hash: &Hash) -> Result<(), Error> {
    let Ok((leaf_index, mut leaf, mut block)) = self.get_leaf_by_key(key) else {
        self.insert(key, value, new_hash, InsertLocation::Auto {})?;
        return Ok(());
    };

    // Guard: new_hash must not already belong to a *different* key
    if leaf.hash != *new_hash && self.block_status_cache.contains_leaf_hash(new_hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // ... rest unchanged
```

### Proof of Concept

```rust
let mut blob = MerkleBlob::new(vec![]).unwrap();

let key_a = KeyId(1);
let key_b = KeyId(2);
let hash_a = Hash(/* some hash */);
let hash_b = Hash(/* different hash */);

blob.insert(key_a, ValueId(10), &hash_a, InsertLocation::Auto {}).unwrap();
blob.insert(key_b, ValueId(20), &hash_b, InsertLocation::Auto {}).unwrap();

// Attacker upserts key_A with key_B's existing hash — no error returned
blob.upsert(key_a, ValueId(99), &hash_b).unwrap();

// get_node_by_hash(hash_b) now returns key_A's data instead of key_B's
let (returned_key, returned_value) = blob.get_node_by_hash(hash_b).unwrap();
assert_eq!(returned_key, key_a);   // PASSES — wrong key returned for hash_b
assert_eq!(returned_value, ValueId(99));

// Integrity check confirms corruption
assert!(blob.check_integrity().is_err()); // IntegrityLeafHashToIndexCacheLengthError
```
