### Title
Missing Declared Class Hash Validation in `execute_replace_class` Allows Permanent Contract Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not validate that the caller-supplied `class_hash` corresponds to a previously declared class. Any contract can invoke `replace_class` with an arbitrary, undeclared felt value as the new class hash. The OS will accept this, commit the broken state, and produce a valid proof — permanently freezing any funds held by the affected contract.

---

### Finding Description

`execute_replace_class` in `syscall_impls.cairo` reads the caller-supplied `class_hash` directly from the syscall request and writes it into `contract_state_changes` without checking that the hash exists in `contract_class_changes` (the declared-class registry):

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-authored TODO at line 898 explicitly acknowledges this missing check. The same omission exists in the deprecated path `execute_replace_class` in `deprecated_execute_syscalls.cairo` (lines 307–329), which has no TODO at all. [1](#0-0) [2](#0-1) 

By contrast, the `execute_declare_transaction` flow correctly enforces that a class hash must be the result of a valid Sierra class hash calculation before writing it to `contract_class_changes`:

```cairo
let expected_class_hash = finalize_class_hash(...);
assert [class_hash_ptr] = expected_class_hash;
...
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [3](#0-2) 

The `replace_class` syscall bypasses this entire gate entirely — there is no lookup into `contract_class_changes` to confirm the target hash was ever declared.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

When a contract's class hash is set to an undeclared value:

1. The OS writes the arbitrary hash into `contract_state_changes` and produces a valid proof.
2. All subsequent calls to that contract attempt to execute the undeclared class. Since no compiled class exists for that hash, every call fails unconditionally.
3. Any tokens, ETH, or other assets held in the contract's storage become permanently inaccessible — there is no recovery path once the state is committed on-chain.

The state commitment machinery (`hash_class_changes`, `compute_class_commitment`) operates on whatever values are in `contract_class_changes` and `contract_state_changes`; it does not cross-validate that a contract's class hash appears in the declared-class tree. [4](#0-3) 

---

### Likelihood Explanation

**Medium.**

- Any deployed contract can issue the `replace_class` syscall; no privileged role is required.
- The `REPLACE_CLASS_SELECTOR` branch is reachable from both the new (`execute_syscalls`) and deprecated (`execute_deprecated_syscalls`) syscall dispatch loops.
- A malicious contract author can intentionally freeze a contract that holds user deposits (e.g., a vault or escrow) by calling `replace_class(0xdeadbeef)` after users have deposited funds.
- A buggy contract with an unguarded `replace_class` call is also exploitable by any transaction sender who can trigger that code path. [5](#0-4) [6](#0-5) 

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that it exists in `contract_class_changes` (i.e., it was previously declared). Concretely, perform a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert the returned compiled class hash is non-zero. This mirrors the guard already present in `execute_declare_transaction` and closes the gap acknowledged by the existing TODO comment.

---

### Proof of Concept

1. Attacker deploys contract `A` that holds user funds (e.g., users call `deposit()` which writes to storage).
2. After users deposit, attacker calls a function in `A` that executes `replace_class(0x1)` — an undeclared hash.
3. The OS dispatches to `execute_replace_class`, reads `class_hash = 0x1`, skips any declared-class check, and writes `StateEntry(class_hash=0x1, ...)` into `contract_state_changes`.
4. The block is proven and committed on-chain with contract `A`'s class hash set to `0x1`.
5. All subsequent calls to `A` (including `withdraw()`) fail because no compiled class for `0x1` exists.
6. User funds are permanently frozen with no recovery mechanism. [7](#0-6)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
