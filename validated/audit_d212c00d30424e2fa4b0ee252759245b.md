### Title
`ProofOfInclusion::valid()` Checks Only Internal Consistency, Not Against a Trusted Root — Forged DataLayer Inclusion Proofs Always Pass - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate verifies only that the proof's internal hash chain is self-consistent. The final comparison `existing_hash == self.root_hash()` is a tautology: after the loop, `existing_hash` is always equal to `self.root_hash()` by construction. No external trusted root is ever checked. An attacker who can supply a `ProofOfInclusion` (a streamable, deserializable type) can forge any internally consistent proof and have it accepted as valid.

### Finding Description

`ProofOfInclusion::valid()` is implemented as follows:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()
}
``` [1](#0-0) 

And `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

**The tautology:** After the loop completes without returning `false`, `existing_hash` holds the last `calculated_hash`, which equals the last `layer.combined_hash` (the loop invariant enforces this). `self.root_hash()` also returns `last.combined_hash`. Therefore `existing_hash == self.root_hash()` is **always true** once the loop passes. The final check adds zero security.

For a zero-layer proof, `existing_hash = self.node_hash` and `self.root_hash() = self.node_hash`, so `valid()` trivially returns `true` for any `node_hash`.

**The analog to the external report:** `onlyExecutor` checks whether the caller *has* the `EXECUTOR_ROLE` (a weaker property) instead of checking whether the caller *is* the timelock contract (the actual required property). Here, `valid()` checks whether the proof is *internally self-consistent* (a weaker property) instead of checking whether the proof's root *matches a caller-supplied trusted root* (the actual required property).

The Python binding exposes `valid()` directly: [5](#0-4) 

### Impact Explanation

Any code that deserializes a `ProofOfInclusion` from an untrusted source and calls `proof.valid()` as the sole verification step will accept a forged proof. An attacker can construct a `ProofOfInclusion` with an arbitrary `node_hash` (claiming any key-value pair is included) and a single consistent layer whose `combined_hash = calculate_internal_hash(node_hash, side, sibling)`. This proof passes `valid()` regardless of the actual DataLayer tree root. This allows untrusted input to prove invalid state — forged inclusion of arbitrary key-value pairs in any DataLayer tree.

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."**

### Likelihood Explanation

`ProofOfInclusion` is a streamable type exposed to Python with `from_bytes()`. The method name `valid()` implies a complete validity check. Any DataLayer peer protocol that accepts a serialized `ProofOfInclusion` from a remote node and calls `proof.valid()` without also separately comparing `proof.root_hash()` to a locally-known trusted root is vulnerable. The fuzz target itself demonstrates the intended usage pattern of calling only `proof.valid()`: [6](#0-5) 

### Recommendation

`valid()` must accept an expected root hash parameter and compare the computed root against it:

```rust
pub fn valid_against_root(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root
}
```

Alternatively, deprecate the no-argument `valid()` and require callers to always check `proof.root_hash()` against a trusted root obtained from a local, authoritative source (not from the proof itself).

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer, Side
from chia_rs.sized_bytes import bytes32
import hashlib

# Attacker wants to forge proof that arbitrary_node_hash is in some tree
arbitrary_node_hash = bytes32(b'\xde\xad' * 16)
sibling_hash       = bytes32(b'\xbe\xef' * 16)

# Compute a consistent combined_hash (attacker controls all inputs)
h = hashlib.sha256(b'\x00' * 30 + b'\x00\x02' + arbitrary_node_hash + sibling_hash).digest()
combined = bytes32(h)

layer = ProofOfInclusionLayer(
    other_hash_side=Side.RIGHT,   # or LEFT
    other_hash=sibling_hash,
    combined_hash=combined,
)

forged_proof = ProofOfInclusion(node_hash=arbitrary_node_hash, layers=[layer])

# valid() returns True for this completely forged proof
assert forged_proof.valid(), "Expected True — tautology confirmed"
# root_hash() returns the attacker-chosen combined_hash, not any real tree root
print("Forged proof root:", forged_proof.root_hash().hex())
```

The `valid()` call returns `True` because the loop's invariant guarantees `existing_hash == last.combined_hash == self.root_hash()` after completion — no external trusted root is ever consulted. [1](#0-0)
