### Title
Missing Declared Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts an arbitrary class hash without verifying that the hash corresponds to a previously declared contract class. An unprivileged contract can call `replace_class` with any felt value as the new class hash. Once committed to state, any future attempt to execute that contract will cause the OS to fail when looking up the undeclared compiled class, permanently making the contract unexecutable and freezing all funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function updates a contract's class hash in `contract_state_changes` without validating that the new class hash has been declared:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment explicitly acknowledges the missing check. The `class_hash` value comes directly from the syscall request (`request.class_hash`) with no validation against `contract_class_changes` (the declared class registry).

When the contract is subsequently called, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
```

If `execution_context.class_hash` is undeclared, `dict_read` on `contract_class_changes` returns 0 (the default for uninitialized dict entries). `find_element` then searches for a compiled class with hash 0. If no such class exists, `find_element` panics, causing the OS to fail to generate a valid proof for any block containing a call to that contract. The sequencer is forced to permanently exclude all calls to the contract, freezing its funds.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any contract that holds funds (e.g., a DeFi vault, escrow, or multi-sig) can have those funds permanently frozen. The attacker calls `replace_class` with an arbitrary undeclared felt value. The state is committed on-chain with the invalid class hash. No future transaction can execute the contract (the OS proof would be invalid), and there is no recovery path: the contract cannot call `replace_class` back to a valid hash because it cannot be executed at all. Funds held by the contract — including funds belonging to third-party users — are permanently inaccessible.

---

### Likelihood Explanation

**High.** The `replace_class` syscall is a standard, publicly accessible syscall available to any Cairo 1 contract. No special privilege is required. Any contract author or user who can trigger a contract's execution path that calls `replace_class` can supply an arbitrary class hash. The missing validation is explicitly noted in a TODO comment, confirming it is a known gap in the current implementation.

---

### Recommendation

Before committing the new class hash in `execute_replace_class`, validate that the provided `class_hash` exists in `contract_class_changes` (i.e., has a non-zero compiled class hash entry). Specifically, perform a `dict_read` on `contract_class_changes` with the new `class_hash` and assert the result is non-zero before updating `contract_state_changes`. This mirrors the check already implicitly required by `execute_entry_point` when it calls `find_element`.

---

### Proof of Concept

1. Attacker deploys Contract A (a vault holding user funds) with a valid class hash `H_valid`.
2. Attacker triggers Contract A's execution (e.g., via an invoke transaction) such that Contract A calls the `replace_class` syscall with `new_class_hash = 0xdeadbeef` (an arbitrary undeclared felt).
3. `execute_replace_class` in `syscall_impls.cairo` (lines 878–915) accepts the call, performs no validation, and updates `contract_state_changes[contract_A_address].class_hash = 0xdeadbeef`.
4. The block is proven and committed. Contract A's on-chain class hash is now `0xdeadbeef`.
5. Any future transaction calling Contract A causes `execute_entry_point` to call `dict_read{dict_ptr=contract_class_changes}(key=0xdeadbeef)`, returning 0. `find_element` then searches for compiled class hash 0, fails to find it, and panics.
6. The OS cannot produce a valid proof for any block containing a call to Contract A. The sequencer permanently excludes Contract A from execution. All funds held by Contract A are frozen with no recovery path. [1](#0-0) [2](#0-1) [3](#0-2)
