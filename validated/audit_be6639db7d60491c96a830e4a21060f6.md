### Title
`execute_deploy` Syscall Unconditionally Reports Success Regardless of Constructor Failure — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_deploy` syscall implementation in the StarkNet OS always writes `failure_flag=0` in the response header, even when the deployed contract's constructor execution fails. Calling contracts therefore cannot detect deployment failures and may proceed to transfer funds to a contract address that was never successfully initialized, leading to permanent freezing of those funds.

---

### Finding Description

In `execute_deploy` (`syscall_impls.cairo`, lines 452–556), after invoking `deploy_contract`, the response header is written with a hardcoded `failure_flag=0`:

```cairo
// Write the response header.
// TODO(Yoni, 1/1/2026): support failures.
assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=0);
``` [1](#0-0) 

The TODO comment explicitly acknowledges this limitation. Additionally, the `contract_address` is always written into the `DeployResponse` regardless of whether the constructor succeeded or reverted:

```cairo
assert [response] = DeployResponse(
    contract_address=contract_address,
    constructor_retdata_start=retdata,
    constructor_retdata_end=retdata + retdata_size,
);
``` [2](#0-1) 

Compare this with `contract_call_helper`, which correctly propagates the `is_reverted` flag:

```cairo
assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=is_reverted);
``` [3](#0-2) 

When a constructor fails, the revert log (passed implicitly to `deploy_contract`) unwinds the constructor's state changes, meaning the contract is not actually registered in `contract_state_changes`. The contract at the computed address does not exist. Yet the calling contract receives `failure_flag=0` and a valid-looking `contract_address`, causing it to believe the deployment succeeded.

This is the direct StarkNet analog of the ERC1155Bridgable bug: just as ERC1155Bridgable lacked a `payable receive` function and therefore could not accept ETH sent to it (silently failing), the OS `execute_deploy` syscall lacks a failure-propagation mechanism and therefore cannot communicate to the caller that the deployed contract does not actually exist.

---

### Impact Explanation

Any contract implementing a factory pattern — deploying a child contract and then transferring funds to it — is vulnerable. The sequence is:

1. Factory contract calls the `deploy` syscall; constructor of the child contract reverts.
2. OS writes `failure_flag=0` and the computed `contract_address` into the response.
3. Factory contract, trusting the OS response, transfers ERC-20 tokens to `contract_address`.
4. The ERC-20 `transfer` call succeeds (token contracts do not require the recipient to be a deployed contract; they simply update a storage mapping).
5. The child contract does not exist at `contract_address` (constructor reverted, deployment undone by the revert log).
6. The tokens are now recorded in the token contract's storage under `contract_address`, but no contract exists there to claim or forward them.
7. If the constructor is permanently broken (e.g., depends on an invariant that can never be satisfied), the funds are **permanently frozen**.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

- Factory/deployer patterns are standard in DeFi and account-abstraction infrastructure on StarkNet.
- Any unprivileged user can submit a transaction that triggers this path: write a contract that calls `deploy` and then immediately transfers tokens to the returned address.
- No privileged role, leaked key, or external dependency is required.
- The bug is unconditional — every single `deploy` syscall, regardless of constructor outcome, returns `failure_flag=0`.

---

### Recommendation

Propagate the constructor's revert status into the response header, mirroring the pattern already used in `contract_call_helper`:

```cairo
// Capture whether the constructor reverted.
let (is_reverted, retdata_size, retdata) = deploy_contract(...);

assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=is_reverted);
```

When `is_reverted=1`, the `contract_address` field in `DeployResponse` should be set to `0` (or left undefined) so that callers cannot accidentally use it.

---

### Proof of Concept

1. User deploys a **Factory** contract containing:
   ```
   fn deploy_and_fund(token: ContractAddress, amount: u256) {
       let (addr, _) = deploy_syscall(BROKEN_CLASS, salt, [], false);
       // OS returns failure_flag=0 even though constructor reverted
       IERC20(token).transfer(addr, amount);  // succeeds; funds now at addr
   }
   ```
2. `BROKEN_CLASS` has a constructor that always panics (e.g., `assert(false, 'init failed')`).
3. The OS executes the constructor, which reverts; the revert log unwinds the deployment; the contract at `addr` does not exist.
4. The OS writes `failure_flag=0` and `addr` into the deploy response.
5. Factory calls `transfer(addr, amount)` — the ERC-20 contract updates its storage; the transfer succeeds.
6. `amount` tokens are now permanently locked at `addr` with no contract to recover them. [4](#0-3)

func execute_deploy{
    range_check_ptr,
    syscall_ptr: felt*,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    revert_log: RevertLogEntry*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, caller_execution_context: ExecutionContext*) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, DeployRequest*);
    local constructor_calldata_start: felt* = request.constructor_calldata_start;
    local constructor_calldata_size = request.constructor_calldata_end - constructor_calldata_start;

    let specific_base_gas_cost = DEPLOY_GAS_COST + DEPLOY_CALLDATA_FACTOR_GAS_COST *
        constructor_calldata_size;
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=specific_base_gas_cost, request_struct_size=DeployRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    local caller_execution_info: ExecutionInfo* = caller_execution_context.execution_info;
    local caller_address = caller_execution_info.contract_address;

    // Verify deploy_from_zero is either 0 (FALSE) or 1 (TRUE).
    tempvar deploy_from_zero = request.deploy_from_zero;
    assert deploy_from_zero * (deploy_from_zero - 1) = 0;
    // Set deployer_address to 0 if request.deploy_from_zero is TRUE.
    let deployer_address = (1 - deploy_from_zero) * caller_address;

    let selectable_builtins = &builtin_ptrs.selectable;
    let hash_ptr = selectable_builtins.pedersen;
    with hash_ptr {
        let (contract_address) = get_contract_address(
            salt=request.contract_address_salt,
            class_hash=request.class_hash,
            constructor_calldata_size=constructor_calldata_size,
            constructor_calldata=constructor_calldata_start,
            deployer_address=deployer_address,
        );
    }
    tempvar builtin_ptrs = new BuiltinPointers(
        selectable=SelectableBuiltins(
            pedersen=hash_ptr,
            range_check=selectable_builtins.range_check,
            ecdsa=selectable_builtins.ecdsa,
            bitwise=selectable_builtins.bitwise,
            ec_op=selectable_builtins.ec_op,
            poseidon=selectable_builtins.poseidon,
            segment_arena=selectable_builtins.segment_arena,
            range_check96=selectable_builtins.range_check96,
            add_mod=selectable_builtins.add_mod,
            mul_mod=selectable_builtins.mul_mod,
        ),
        non_selectable=builtin_ptrs.non_selectable,
    );

    tempvar constructor_execution_context = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_CONSTRUCTOR,
        class_hash=request.class_hash,
        calldata_size=constructor_calldata_size,
        calldata=constructor_calldata_start,
        execution_info=new ExecutionInfo(
            block_info=caller_execution_info.block_info,
            tx_info=caller_execution_info.tx_info,
            caller_address=caller_address,
            contract_address=contract_address,
            selector=CONSTRUCTOR_ENTRY_POINT_SELECTOR,
        ),
        deprecated_tx_info=caller_execution_context.deprecated_tx_info,
    );

    with remaining_gas {
        let (retdata_size, retdata) = deploy_contract(
            block_context=block_context, constructor_execution_context=constructor_execution_context
        );
    }

    // TODO(Yoni, 1/1/2026): consider sharing code with call_contract_helper.
    let response_header = cast(syscall_ptr, ResponseHeader*);
    let syscall_ptr = syscall_ptr + ResponseHeader.SIZE;

    // Write the response header.
    // TODO(Yoni, 1/1/2026): support failures.
    assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=0);

    let response = cast(syscall_ptr, DeployResponse*);
    // Advance syscall pointer to the next syscall.
    let syscall_ptr = syscall_ptr + DeployResponse.SIZE;

    %{ CheckNewDeployResponse %}

    // Write the response.
    relocate_segment(src_ptr=response.constructor_retdata_start, dest_ptr=retdata);
    assert [response] = DeployResponse(
        contract_address=contract_address,
        constructor_retdata_start=retdata,
        constructor_retdata_end=retdata + retdata_size,
    );

    return ();
}
