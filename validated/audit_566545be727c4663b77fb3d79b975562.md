### Title
Integer Truncation in `NewDynamicFeeChecker` Causes Zero Effective Fee for Cosmos Transactions When `baseFee = 0` - (File: ante/evm/fee_checker.go)

### Summary

`NewDynamicFeeChecker` derives the per-gas fee cap via integer division `feeCap = fee / gas`. When the total fee amount is less than the gas limit (a common condition when `MinGasPrice < 1 aphoton/gas`), the division truncates to zero. Because `GetBaseFee` returns `big.Int(0)` — not `nil` — when `NoBaseFee = true` with London HF enabled, the EIP-1559 path is taken instead of the `checkTxFeeWithValidatorMinGasPrices` fallback. With `feeCap = 0` and `baseFeeInt = 0`, the sufficiency check `feeCap.LT(baseFeeInt)` evaluates `0 < 0 = false` and passes. `EffectiveGasPrice(0, 0, MaxInt64) = min(MaxInt64, 0) = 0`, so `effectiveFee = 0 * gas = 0` is returned. The `DeductFeeDecorator` then deducts zero from the sender even though `MinGasPriceDecorator` already accepted a non-zero fee.

### Finding Description

**Root cause — integer truncation:**

```go
// ante/evm/fee_checker.go:83
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
```

`sdkmath.Int.Quo` is integer (floor) division. When `fee < gas`, `feeCap = 0`.

**Why `baseFee = 0` is reached instead of the nil-fallback:**

```go
// x/evm/types/utils.go:244-254
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    if !IsLondon(ethCfg, height) {
        return nil
    }
    baseFee := feemarketParams.GetBaseFee()
    if baseFee == nil {          // NoBaseFee=true → GetBaseFee() returns nil
        return new(big.Int)      // returns 0, NOT nil
    }
    return baseFee
}
```

When `NoBaseFee = true`, `feemarketParams.GetBaseFee()` returns `nil`, but `GetBaseFee` converts that to `big.Int(0)`. Back in `NewDynamicFeeChecker`:

```go
// ante/evm/fee_checker.go:56-60
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
if baseFee == nil {
    return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
}
```

`baseFee = 0` is not `nil`, so the fallback is never taken. The EIP-1559 path continues with `baseFeeInt = 0`.

**Sufficiency check passes silently:**

```go
// ante/evm/fee_checker.go:86-88
if feeCap.LT(baseFeeInt) {   // 0 < 0 → false → no error
    return nil, 0, errorsmod.Wrapf(...)
}
```

**Effective fee collapses to zero:**

```go
// x/evm/types/utils.go:232-234
func EffectiveGasPrice(baseFee, feeCap, tipCap *big.Int) *big.Int {
    return ethermint.BigMin(new(big.Int).Add(tipCap, baseFee), feeCap)
    // = min(MaxInt64 + 0, 0) = 0
}
```

```go
// ante/evm/fee_checker.go:94-99
effectiveFee := sdk.Coins{{
    Denom:  denom,
    Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas)), // 0 * gas = 0
}}
```

**`DeductFeeDecorator` deducts zero:**

```go
// ante/evm/nativefee.go:57-63
fee := feeTx.GetFee()
if !simulate {
    fee, priority, err = dfd.txFeeChecker(ctx, tx) // returns 0
    ...
}
// checkDeductFee called with fee=0 → no bank deduction
```

**Concrete attack path:**

1. Chain configured with `NoBaseFee = true`, London HF enabled, `MinGasPrice = 0.5 aphoton/gas` (fractional, non-zero).
2. Attacker submits a Cosmos tx with `gas = 1_000_000`, `fee = 500_000 aphoton` (exactly `ceil(0.5 × 1_000_000)`).
3. `MinGasPriceDecorator` passes: `500_000 ≥ 0.5 × 1_000_000 = 500_000`. ✓
4. `NewDynamicFeeChecker`: `feeCap = 500_000 / 1_000_000 = 0`. `effectiveFee = 0`.
5. `DeductFeeDecorator` deducts 0. Attacker pays nothing.

The same applies when `baseFee = 0` because `EnableHeight` has not been reached yet (a transient window at chain start).

### Impact Explanation

Cosmos transactions (staking, governance, IBC, authz, etc.) can be submitted with zero effective fees even when `MinGasPrice > 0` is configured to prevent spam. This mis-accounts user funds (the fee the chain expects to collect is never deducted), enables fee-free spam of the Cosmos tx mempool, and undermines the economic security model of the fee market for any chain running with `NoBaseFee = true` and a fractional `MinGasPrice`.

This matches the allowed High impact: *"ante handler… bug that permits… valid user funds/fees to be mis-accounted."*

### Likelihood Explanation

- `NoBaseFee = true` is a documented, supported configuration used by chains that want a static or zero base fee.
- `MinGasPrice < 1 aphoton/gas` is reachable whenever the chain sets a fractional global minimum (e.g., `0.001`, `0.5`). The default is `0`, but any non-zero fractional value triggers the bypass.
- The attacker needs only to craft a standard Cosmos SDK transaction with `fee = ceil(MinGasPrice × gas)` and `gas > fee` (guaranteed when `MinGasPrice < 1`). No special privileges or keys are required.

### Recommendation

Replace the integer division with a comparison that avoids precision loss. The correct check is whether the total fee satisfies `fee >= baseFee * gas`:

```go
// Instead of:
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
if feeCap.LT(baseFeeInt) { ... }

// Use:
requiredFee := baseFeeInt.Mul(sdkmath.NewIntFromUint64(gas))
if fee.LT(requiredFee) { ... }
```

For the effective fee calculation, use the actual fee amount (capped at `feeCap * gas`) rather than the truncated `feeCap`:

```go
effectiveGasPrice := sdkmath.NewIntFromBigInt(
    types.EffectiveGasPrice(baseFeeInt.BigInt(), fee.BigInt(), maxPriorityPrice.Mul(sdkmath.NewIntFromUint64(gas)).BigInt()),
)
```

Alternatively, normalize `fee` to a per-gas price using `LegacyDec` before comparison to preserve sub-integer precision.

Additionally, `GetBaseFee` in `x/evm/types/utils.go` should return `nil` (not `big.Int(0)`) when `NoBaseFee = true`, so that `NewDynamicFeeChecker` correctly falls back to `checkTxFeeWithValidatorMinGasPrices` for chains that have disabled the fee market.
