### Title
Wrong `state_diff_commitment` and Block Hash Due to Structural Omission of `deprecated_declared_classes` in Sequencer Commitment Path — (File: `crates/blockifier/src/state/cached_state.rs`)

---

### Summary

The `From<CommitmentStateDiff> for ThinStateDiff` conversion unconditionally sets `deprecated_declared_classes: Vec::new()`. Because `BlockExecutionArtifacts::new()` uses this conversion to feed `calculate_block_commitments`, the `state_diff_commitment` (and therefore the block hash and the `ProposalCommitment` used by consensus) is computed as if no deprecated (Cairo 0) classes were declared, even when a block contains such declarations. The stored `ThinStateDiff` written to MDBX storage via `thin_state_diff()` is produced by the same conversion and is therefore also missing the deprecated class entries. This is the sequencer-native analog of the weak Fiat-Shamir issue: a commitment is computed over an incomplete view of the state, so the block hash does not uniquely bind to the full state diff.

---

### Finding Description

**Root cause — always-empty `deprecated_declared_classes`:**

`CommitmentStateDiff` has no `deprecated_declared_classes` field. The `From` impl that converts it to `ThinStateDiff` hard-codes the field to an empty vector:

```rust
// crates/blockifier/src/state/cached_state.rs:690-701
impl From<CommitmentStateDiff> for ThinStateDiff {
    fn from(commitment_state_diff: CommitmentStateDiff) -> Self {
        Self {
            deployed_contracts: commitment_state_diff.address_to_class_hash,
            storage_diffs: commitment_state_diff.storage_updates,
            class_hash_to_compiled_class_hash: commitment_state_diff
                .class_hash_to_compiled_class_hash,
            nonces: commitment_state_diff.address_to_nonce,
            // TODO(AlonH): Remove this when the structure of storage diffs changes.
            deprecated_declared_classes: Vec::new(),
        }
    }
}
``` [1](#0-0) 

**Commitment computation path:**

`BlockExecutionArtifacts::new()` calls `ThinStateDiff::from(commitment_state_diff.clone())` and passes the result directly to `calculate_block_commitments`:

```rust
// crates/apollo_batcher/src/block_builder.rs:160-166
let (header_commitments, measurements) = calculate_block_commitments(
    &transactions_data,
    ThinStateDiff::from(commitment_state_diff.clone()),
    l1_da_mode,
    &block_info.starknet_version,
)
.await;
``` [2](#0-1) 

`calculate_block_commitments` calls `calculate_state_diff_hash`, which explicitly chains `deprecated_declared_classes` into the Poseidon hash:

```rust
// crates/starknet_api/src/block_hash/state_diff_hash.rs:30-42
pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    ...
    hash_chain = chain_deprecated_declared_classes(
        &state_diff.deprecated_declared_classes, hash_chain);
    ...
}
``` [3](#0-2) 

Because `deprecated_declared_classes` is always `[]`, the `state_diff_commitment` is identical whether or not the block contains deprecated Declare transactions.

**Block hash propagation:**

The wrong `state_diff_commitment` is embedded in `BlockHeaderCommitments`, which is stored in `PartialBlockHashComponents`: [4](#0-3) 

`PartialBlockHashComponents` is then used to compute both the `ProposalCommitment` (the value consensus nodes vote on) and the final block hash via `calculate_block_hash`: [5](#0-4) 

**Stored state diff is also wrong:**

`thin_state_diff()` uses the same conversion, so the `ThinStateDiff` written to MDBX storage is also missing the deprecated class entries: [6](#0-5) 

**No existing check catches the discrepancy:**

The committer verifies the state diff commitment against the stored `ThinStateDiff`. Because both are derived from the same incomplete conversion, the check always passes even when deprecated classes are missing: [7](#0-6) 

---

### Impact Explanation

For any block containing a deprecated (Cairo 0) Declare transaction:

1. **Wrong `state_diff_commitment`** — the `BlockHeaderCommitments.state_diff_commitment` field does not reflect the actual state diff.
2. **Wrong block hash** — `calculate_block_hash` chains `state_diff_commitment` directly; the resulting block hash is therefore wrong.
3. **Wrong `ProposalCommitment`** — consensus nodes vote on `PartialBlockHash::from_partial_block_hash_components`, which is derived from the same wrong commitments; the agreed-upon commitment does not bind to the full state diff.
4. **Wrong stored `ThinStateDiff`** — the MDBX record for the block's state diff is missing the deprecated class entries, so RPC calls returning state updates (`starknet_getStateUpdate`) will return an authoritative-looking but incomplete value.
5. **Downstream proof / sync breakage** — the SNOS and state-sync paths consume the stored `ThinStateDiff` and the `state_diff_commitment`; both are wrong, so any proof or sync that cross-checks these values against the actual Patricia trie state will observe a mismatch.

This matches the allowed impact: *"Wrong state … or revert result from blockifier/syscall/execution logic for accepted input"* and *"RPC … returns an authoritative-looking wrong value."*

---

### Likelihood Explanation

Deprecated Declare (v0/v1) transactions are still valid Starknet protocol transactions and the blockifier executes them. Whether the gateway admits them depends on gateway-level validation not fully visible in this search, but no hard rejection was found. If even one such transaction is sequenced, the block hash and state diff commitment are provably wrong. The issue is structural and deterministic — it cannot be mitigated at runtime without a code change.

---

### Recommendation

1. Add a `deprecated_declared_classes: Vec<ClassHash>` field to `CommitmentStateDiff` and populate it in the blockifier when a deprecated Declare transaction is executed.
2. Remove the hard-coded `deprecated_declared_classes: Vec::new()` from `From<CommitmentStateDiff> for ThinStateDiff` and propagate the field correctly.
3. Ensure `thin_state_diff()` in `BlockExecutionArtifacts` returns the complete diff including deprecated class entries.
4. Add a cross-check in the committer (or in `BlockExecutionArtifacts::new`) that asserts the `state_diff_commitment` computed from the full `ThinStateDiff` matches the one stored in `PartialBlockHashComponents`.

---

### Proof of Concept

1. Submit a `DeclareTransactionV1` (Cairo 0 class) to the sequencer gateway.
2. The blockifier executes it; `CommitmentStateDiff` is produced with no `deprecated_declared_classes` field.
3. `BlockExecutionArtifacts::new()` calls `ThinStateDiff::from(commitment_state_diff.clone())` → `deprecated_declared_classes = []`.
4. `calculate_block_commitments` → `calculate_state_diff_hash` → `chain_deprecated_declared_classes(&[], ...)` → the deprecated class hash is absent from the Poseidon chain.
5. The resulting `state_diff_commitment` equals the commitment of a block with no deprecated class declarations.
6. `calculate_block_hash` embeds this wrong commitment; the block hash is wrong.
7. The `ThinStateDiff` stored in MDBX via `thin_state_diff()` is also missing the deprecated class entry.
8. Any verifier (L1 contract, state-sync peer, SNOS) that recomputes the state diff hash from the on-chain data will observe a mismatch with the block header's `state_diff_commitment`.

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
