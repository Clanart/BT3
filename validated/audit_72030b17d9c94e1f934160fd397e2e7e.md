### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Fund Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary felt as the new class hash without verifying that the hash corresponds to a previously declared contract class. This allows a contract to irreversibly replace its own class hash with an undeclared value, rendering the contract permanently uncallable and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the requested class hash directly from the syscall request and writes it into `contract_state_changes` with no validation:

```cairo
let class_hash = request.class_hash;
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}
tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment explicitly acknowledges the missing check. The OS developers intended to add a validation that the new class hash exists in `contract_class_changes` (i.e., was previously declared), but it is absent.

When the contract is subsequently called, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
``` [2](#0-1) 

If `execution_context.class_hash` is an undeclared value, `dict_read` returns the default value `0`. `find_element` then searches for a compiled class with hash `0`, which does not exist, causing the OS execution to fail. The sequencer simulation catches this and excludes any future calls to the contract. The contract is permanently uncallable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with an undeclared value, no transaction can successfully call that contract again. Any ERC-20 tokens, ETH, STRK, or other assets held in the contract's storage are permanently inaccessible. There is no recovery path: `replace_class` cannot be called again (the entry point lookup fails before execution reaches the syscall handler), and the state change is committed on-chain.

---

### Likelihood Explanation

The attack is straightforward and requires no privileged access:

1. An attacker deploys a contract (e.g., a yield-farming vault or token staking contract) that contains a `freeze()` function calling `replace_class(0xdeadbeef)` (any felt not in `contract_class_changes`).
2. Victims deposit funds into the contract.
3. The attacker calls `freeze()`. The OS accepts the `replace_class` syscall without validation.
4. The contract's class hash is now `0xdeadbeef`. All subsequent calls to the contract fail at the OS level.
5. Victim funds are permanently frozen.

This is a one-transaction, zero-privilege attack once the contract is deployed. It is directly analogous to the race condition in the report: just as a compromised recovery address could keep a wallet permanently in the "in recovery" state (blocking all owner operations), a malicious contract can keep itself permanently in an "invalid class" state (blocking all fund withdrawals).

---

### Recommendation

Add a validation in `execute_replace_class` that the requested `class_hash` exists in `contract_class_changes` (i.e., has a non-zero compiled class hash entry), consistent with the existing TODO comment. The check should mirror the lookup already performed in `execute_entry_point`:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);  // Reject undeclared class hashes.
``` [3](#0-2) 

---

### Proof of Concept

1. Declare a valid class `C` and deploy contract `A` using it.
2. Deploy a second contract `B` with a `freeze()` entry point that issues `replace_class(0x1337)` where `0x1337` is never declared.
3. Send funds to contract `B`.
4. Call `freeze()` on `B`. The OS executes `execute_replace_class`, writes `class_hash=0x1337` into `contract_state_changes` for `B`'s address, and returns success.
5. Attempt any call to `B`. The OS reads `class_hash=0x1337` from state, calls `dict_read{dict_ptr=contract_class_changes}(key=0x1337)` → returns `0`, then `find_element(..., key=0)` fails. The sequencer simulation rejects the call.
6. Funds in `B` are permanently frozen with no recovery mechanism. [4](#0-3) [5](#0-4)

func execute_replace_class{

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
