### Title
P2P State Diff Sync Validates Length But Not Commitment Hash, Allowing Malicious Peer to Inject Wrong State Into Storage - (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

---

### Summary

The `StateDiffStreamBuilder::parse_data_for_block` function assembles a `ThinStateDiff` from P2P-supplied chunks and validates only that the total entry count matches the header's `state_diff_length`. It never verifies that `calculate_state_diff_hash(&assembled_diff) == header.state_diff_commitment`. A malicious peer can send chunks whose lengths sum correctly but whose content is fabricated, causing the node to persist a wrong state diff, compute a wrong global root, and store a wrong block hash — all without any cryptographic check catching the deviation.

---

### Finding Description

**Root cause — missing commitment check in `parse_data_for_block`**

`StateDiffStreamBuilder::parse_data_for_block` reads the expected length from the stored header:

```rust
let target_state_diff_len = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    ...
    .state_diff_length   // ← only the count is read
    .ok_or(...)?;
```

It accumulates chunks until `current_state_diff_len == target_state_diff_len`, then runs a single post-assembly check:

```rust
validate_deprecated_declared_classes_non_conflicting(&result)?;
Ok(Some((result, block_number)))
``` [1](#0-0) 

`validate_deprecated_declared_classes_non_conflicting` only detects duplicate deprecated class hashes; it does not compute or compare the Poseidon commitment. [2](#0-1) 

The header already carries the authoritative `state_diff_commitment: Option<StateDiffCommitment>` in `StorageBlockHeader`: [3](#0-2) 

That value is never fetched inside `parse_data_for_block`. The assembled diff is written straight to MDBX: [4](#0-3) 

**What `calculate_state_diff_hash` actually covers**

The hash function commits to every field — deployed contracts, declared classes, deprecated classes, storage diffs, and nonces — all sorted by key: [5](#0-4) 

A peer can craft chunks that each have a non-zero `len()` (passing the `EmptyStateDiffPart` guard), whose combined count equals `target_state_diff_len`, but whose storage values, nonces, or class hashes differ from the real block. `unite_state_diffs` only rejects duplicate keys, not wrong values: [6](#0-5) 

**Downstream propagation**

After the wrong diff is stored, `BlockExecutionArtifacts::new` (proposer path) and `read_commitment_input_and_add_task` (sync path) both read the diff back from storage and forward it to the committer, which applies it to the Patricia trie to produce `global_root`: [7](#0-6) 

The committer's `verify_state_diff_hash` guard is a **config flag** (`self.config.verify_state_diff_hash`), not always enabled: [8](#0-7) 

Even when enabled, it fires only after the wrong diff is already durably stored. The wrong `global_root` is then chained into `calculate_block_hash`: [9](#0-8) 

producing a block hash that diverges from the consensus-agreed value, with no subsequent check comparing the two.

---

### Impact Explanation

A node syncing via P2P will store a fabricated state diff, compute a wrong Patricia-trie root, and record a wrong block hash. Every downstream consumer of that storage — RPC `starknet_getStorageAt`, `starknet_getStateUpdate`, fee estimation, execution tracing, and proof inputs — will operate on the corrupted state. This matches:

- **High**: RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.
- **Critical** (if the corrupted node is the sequencer itself or feeds proof inputs): Wrong state, storage value, or class hash from blockifier/syscall/execution logic; wrong compiled class or class hash selected for execution.

---

### Likelihood Explanation

Any peer in the P2P network can serve state diff responses. The only admission gate is that chunk lengths sum to `target_state_diff_len` and no deprecated class appears twice. Both constraints are trivially satisfied by a crafted response. No signature, stake, or reputation check prevents a peer from sending wrong content. The attack is therefore reachable by any unprivileged network participant.

---

### Recommendation

Inside `parse_data_for_block`, after the length check passes, read the header's `state_diff_commitment` and verify the assembled diff:

```rust
let expected_commitment = header.state_diff_commitment
    .ok_or(P2pSyncClientError::OldHeaderInStorage {
        block_number,
        missing_field: "state_diff_commitment",
    })?;
let actual_commitment = calculate_state_diff_hash(&result);
if actual_commitment != expected_commitment {
    return Err(ParseDataError::BadPeer(
        BadPeerError::WrongStateDiffCommitment { block_number }
    ));
}
```

This mirrors the check already present in the committer (`verify_state_diff_hash`) but moves it to the admission boundary, before any storage write occurs. [10](#0-9) 

---

### Proof of Concept

1. Node A is syncing block N via P2P. Its storage already holds the header for block N with `state_diff_length = 2` and `state_diff_commitment = H_real`.
2. Malicious peer sends two `StateDiffChunk::ContractDiff` messages, each with `len() == 1`, but with fabricated storage values (e.g., balance slot set to `u128::MAX` instead of the real value). Total length = 2 = `target_state_diff_len`. No duplicate keys. `validate_deprecated_declared_classes_non_conflicting` passes.
3. `parse_data_for_block` returns `Ok(Some((fabricated_diff, block_N)))`.
4. `write_to_storage` calls `append_state_diff(block_N, fabricated_diff)` — durably stored.
5. Commitment manager reads `fabricated_diff` from storage, sends to committer. Committer computes `global_root_wrong` from the fabricated Patricia trie update.
6. `calculate_block_hash(..., global_root_wrong, ...)` produces `block_hash_wrong ≠ block_hash_consensus`.
7. `starknet_getStorageAt(contract, slot, block_N)` returns the fabricated value. Proof inputs derived from `global_root_wrong` are invalid. [11](#0-10) [12](#0-11) [13](#0-12)

fn unite_state_diffs(
    state_diff: &mut ThinStateDiff,
    state_diff_chunk: StateDiffChunk,
) -> Result<(), BadPeerError> {
    match state_diff_chunk {
        StateDiffChunk::ContractDiff(contract_diff) => {
            if let Some(class_hash) = contract_diff.class_hash {
                if state_diff
                    .deployed_contracts
                    .insert(contract_diff.contract_address, class_hash)
                    .is_some()
                {
                    return Err(BadPeerError::ConflictingStateDiffParts);
                }
            }
            if let Some(nonce) = contract_diff.nonce {
                if state_diff.nonces.insert(contract_diff.contract_address, nonce).is_some() {
                    return Err(BadPeerError::ConflictingStateDiffParts);
                }
            }
            if !contract_diff.storage_diffs.is_empty() {
                match state_diff.storage_diffs.get_mut(&contract_diff.contract_address) {
                    Some(storage_diffs) => {
                        for (k, v) in contract_diff.storage_diffs {
                            if storage_diffs.insert(k, v).is_some() {
                                return Err(BadPeerError::ConflictingStateDiffParts);
                            }
                        }
                    }
                    None => {
                        state_diff
                            .storage_diffs
                            .insert(contract_diff.contract_address, contract_diff.storage_diffs);
                    }
                }
            }
        }
        StateDiffChunk::DeclaredClass(declared_class) => {
            if state_diff
                .class_hash_to_compiled_class_hash
                .insert(declared_class.class_hash, declared_class.compiled_class_hash)
                .is_some()
            {
                return Err(BadPeerError::ConflictingStateDiffParts);
            }
        }
        StateDiffChunk::DeprecatedDeclaredClass(deprecated_declared_class) => {
            state_diff.deprecated_declared_classes.push(deprecated_declared_class.class_hash);
        }
    }
    Ok(())

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
