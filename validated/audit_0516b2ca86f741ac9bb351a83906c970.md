### Title
Missing Recipient Validation in `initTransfer` Allows Permanent Irrecoverable Lock of Bridged Assets — (File: `evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.sol::initTransfer` (and `initTransfer1155`) accept a `string calldata recipient` parameter with no validation that it is non-empty or parseable as a valid `OmniAddress`. A user who calls `initTransfer` with `recipient = ""` (empty string) will have their tokens burned or locked on the EVM side, while the NEAR bridge will permanently fail to finalize the transfer, resulting in an irrecoverable loss of assets.

---

### Finding Description

`OmniBridge.sol::initTransfer` burns or locks the caller's tokens before emitting the `InitTransfer` event. The `recipient` string is passed through entirely without validation:

```solidity
function initTransfer(
    address tokenAddress,
    uint128 amount,
    uint128 fee,
    uint128 nativeFee,
    string calldata recipient,   // ← no non-empty / format check
    string calldata message
) external payable whenNotPaused(PAUSED_INIT_TRANSFER) {
    currentOriginNonce += 1;
    if (fee >= amount) { revert InvalidFee(); }
    // tokens burned or locked here ...
    emit BridgeTypes.InitTransfer(..., recipient, message);
}
``` [1](#0-0) 

On the NEAR side, `fin_transfer_callback` decodes the prover result and parses the recipient field:

```rust
recipient: event.data.recipient.parse().map_err(stringify)?,
``` [2](#0-1) 

An empty string `""` cannot be parsed as an `OmniAddress`. The parse call returns an error, the prover result is invalid, and `fin_transfer_callback` panics:

```rust
let Ok(ProverResult::InitTransfer(init_transfer)) = Self::decode_prover_result(0) else {
    env::panic_str(BridgeError::InvalidProofMessage.to_string().as_str())
};
``` [3](#0-2) 

Because the NEAR finalization always fails for an empty recipient, and there is no admin rescue or cancel function in `OmniBridge.sol`, the locked/burned tokens are permanently unrecoverable.

The same missing validation exists in `initTransfer1155`: [4](#0-3) 

The StarkNet bridge has the identical gap — `init_transfer` accepts `recipient: ByteArray` with no non-empty check before burning/locking tokens: [5](#0-4) 

---

### Impact Explanation

**Critical — Permanent freezing / irrecoverable lock of user funds.**

- For bridge tokens (`isBridgeToken[tokenAddress] == true`): tokens are **burned** via `BridgeToken.burn`. There is no mint-back path; the supply is permanently destroyed.
- For native ERC-20 tokens: tokens are **locked** in the bridge contract. No rescue function exists in `OmniBridge.sol`, so they are permanently frozen.
- For native ETH (`tokenAddress == address(0)`): ETH is sent with the transaction and locked in the contract; same outcome.

The nonce `currentOriginNonce` is incremented and the event is emitted, but the NEAR finalization will always revert. There is no on-chain mechanism to cancel or refund the transfer. [6](#0-5) 

---

### Likelihood Explanation

Any unprivileged bridge user can trigger this by calling `initTransfer` with `recipient = ""`. Realistic causes include:

- A UI bug or wallet integration that submits the transaction before the recipient field is populated.
- A developer scripting a transfer and accidentally passing an empty string.
- A copy-paste error where the recipient field is cleared.

No special role or privileged access is required. The function is public and callable by anyone.

The Solana bridge's `SECURITY.md` explicitly acknowledges the same class of issue for Solana (`InitTransferPayload` with invalid recipient), noting "Manual intervention would be needed." However, the EVM contract has no equivalent manual-intervention path, making the EVM instance a permanent freeze rather than a recoverable one. [7](#0-6) 

---

### Recommendation

Add a non-empty recipient check at the top of `initTransfer` and `initTransfer1155` in `OmniBridge.sol`, before any token movement occurs:

```solidity
if (bytes(recipient).length == 0) {
    revert InvalidRecipient();
}
```

Optionally, enforce a minimum format check (e.g., that the string contains a chain prefix separator) to catch obviously malformed addresses early on the EVM side, consistent with the `OmniAddress` parsing rules on NEAR.

Apply the same guard to the StarkNet `init_transfer`:

```cairo
assert(recipient.len() > 0, 'ERR_EMPTY_RECIPIENT');
```

---

### Proof of Concept

1. Deploy `OmniBridge` on a local fork. Register a bridge token.
2. Mint bridge tokens to `user`.
3. Call:
   ```solidity
   OmniBridge.initTransfer(
       bridgeTokenAddress,
       1_000_000,   // amount
       0,           // fee
       0,           // nativeFee
       "",          // recipient — empty string
       ""           // message
   );
   ```
4. Observe: `BridgeToken.burn(user, 1_000_000)` executes successfully; `InitTransfer` event is emitted with `recipient = ""`.
5. Attempt to call `fin_transfer` on the NEAR bridge with a proof of this event. The prover decodes `recipient = ""`, `"".parse::<OmniAddress>()` fails, `fin_transfer_callback` panics with `InvalidProofMessage`.
6. Repeat step 5 indefinitely — it always fails. The burned tokens are unrecoverable. [1](#0-0) [8](#0-7) [9](#0-8)

        fn init_transfer(
            ref self: ContractState,
            token_address: ContractAddress,
            amount: u128,

            message: ByteArray,
        ) {
            assert(!_is_paused(@self, PAUSE_INIT_TRANSFER), 'ERR_INIT_TRANSFER_PAUSED');

            assert(amount > 0, 'ERR_ZERO_AMOUNT');
            assert(fee < amount, 'ERR_INVALID_FEE');

            let origin_nonce = self.current_origin_nonce.read() + 1;
            self.current_origin_nonce.write(origin_nonce);

            let caller = get_caller_address();

            if self.is_bridge_token(token_address) {
                IBridgeTokenDispatcher { contract_address: token_address }
                    .burn(caller, amount.into());
            } else {
                let success = IERC20Dispatcher { contract_address: token_address }
                    .transfer_from(caller, get_contract_address(), amount.into());

            if native_fee > 0 {

                let success = IERC20Dispatcher { contract_address: native_token }
                    .transfer_from(caller, get_contract_address(), native_fee.into());
                assert(success, 'ERR_FEE_TRANSFER_FAILED');

            self

                    Event::InitTransfer(

                            token_address,
                            origin_nonce,
                            amount,
                            fee,
                            native_fee,
                            recipient,
                            message,
                        },
                    ),
