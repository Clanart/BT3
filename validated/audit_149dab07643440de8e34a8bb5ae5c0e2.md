### Title
Meta-Transaction Replay via Nonce-Absent `execute_meta_tx_v0` Allows Arbitrary Draining of Target Contracts — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_meta_tx_v0` syscall allows any executing contract to invoke the `__execute__` entry point of an arbitrary target contract with a caller-supplied signature. The OS-computed meta-transaction hash (`compute_meta_tx_v0_hash`) does not include a nonce, and the OS unconditionally writes `nonce=0` into the target's `TxInfo`. Because `check_and_increment_nonce` explicitly skips all version-0 transactions, no replay protection exists at the OS level. An attacker who observes a valid meta-transaction on-chain can replay it indefinitely, draining the target contract's funds.

---

### Finding Description

**Root cause — three cooperating defects:**

**1. `execute_meta_tx_v0` sets `nonce=0` unconditionally and passes no nonce to the hash function.** [1](#0-0) 

The call to `compute_meta_tx_v0_hash` receives only `contract_address`, `entry_point_selector`, `calldata`, `calldata_size`, and `chain_id`. No nonce is passed, so the resulting hash is a pure function of those five fields. [2](#0-1) 

The `TxInfo` written for the target contract hardcodes `nonce=0` and `version=0`.

**2. `check_and_increment_nonce` is a no-op for version 0.** [3](#0-2) 

Because `execute_meta_tx_v0` sets `version=0`, the OS never reads or increments the target contract's on-chain nonce. The state is left unchanged between replays.

**3. `execute_meta_tx_v0` is callable by any contract with no restriction on the target.** [4](#0-3) 

The only guard is `selector == EXECUTE_ENTRY_POINT_SELECTOR`. Any deployed contract can issue this syscall against any `contract_address` with any `calldata` and any `signature`.

**Contrast with L1 handlers**, which do include a nonce in both the hash and the output, providing replay protection: [5](#0-4) 

The deliberate inclusion of `nonce` for L1 handlers and its deliberate omission for meta-transactions confirms this is a structural gap, not an oversight in a single line.

---

### Impact Explanation

**Critical — Direct loss of funds.**

A target account contract's `__execute__` entry point validates the caller-supplied signature against `tx_info.transaction_hash`. Because the hash is deterministic (no nonce), a signature that was valid once is valid forever for the same `(contract_address, calldata, chain_id)` tuple. An attacker can:

- Replay a fund-transfer meta-transaction as many times as the target's balance allows.
- The target contract receives `nonce=0` every time and has no OS-provided mechanism to distinguish the first execution from the hundredth.
- Standard account-contract implementations do not implement custom storage-based nonce tracking for version-0 meta-transactions, because the OS nonce field appears to carry that meaning.

---

### Likelihood Explanation

**High.**

- All meta-transaction executions are public on-chain; the attacker needs only to observe `calldata` and `signature`.
- Deploying a contract that issues `execute_meta_tx_v0` requires no privilege — any unprivileged user can do it.
- No cryptographic break is required; the attacker reuses a legitimately produced signature.
- The attack is repeatable in a tight loop within a single block or across blocks until the target is drained.

---

### Recommendation

1. **Include a nonce in `compute_meta_tx_v0_hash`**: add a per-target-contract nonce field to the hash inputs, mirroring how L1 handler hashes are constructed.
2. **Enforce nonce in `check_and_increment_nonce`**: remove the version-0 early-return, or add a parallel path that reads and increments the target contract's nonce for meta-transactions.
3. **Alternatively**, restrict `execute_meta_tx_v0` so that only the target contract itself can issue the syscall (i.e., `caller_execution_context.execution_info.contract_address == request.contract_address`), eliminating the unauthorized-caller surface entirely.

---

### Proof of Concept

1. Alice's account contract signs a meta-transaction: transfer 100 STRK to Bob. The signature covers `meta_tx_hash(Alice, __execute__, [transfer 100 to Bob], chain_id)`.
2. A legitimate caller submits this via `execute_meta_tx_v0`. Alice's `__execute__` validates the signature → passes. 100 STRK transferred.
3. Eve observes the calldata and signature on-chain.
4. Eve deploys `AttackContract` with a single function that calls `execute_meta_tx_v0(contract_address=Alice, selector=__execute__, calldata=[transfer 100 to Bob], signature=Alice_sig)`.
5. Eve submits a transaction invoking `AttackContract`.
6. The OS computes `meta_tx_hash(Alice, __execute__, [transfer 100 to Bob], chain_id)` — identical to step 2.
7. Alice's `__execute__` receives `nonce=0`, `transaction_hash=same_hash`, `signature=Alice_sig` → signature check passes → 100 STRK transferred again.
8. Eve repeats steps 4–7 until Alice's balance is zero.

No privileged access, no key compromise, no cryptographic break required. The OS's own `execute_meta_tx_v0` implementation is the necessary and sufficient vulnerable step. [6](#0-5) [3](#0-2)

    syscall_ptr: felt*,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    revert_log: RevertLogEntry*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, caller_execution_context: ExecutionContext*) {
    alloc_locals;

    let request = cast(syscall_ptr + RequestHeader.SIZE, MetaTxV0Request*);
    local calldata_start: felt* = request.calldata_start;
    local calldata_size = request.calldata_end - calldata_start;

    let specific_base_gas_cost = (
        META_TX_V0_GAS_COST + META_TX_V0_CALLDATA_FACTOR_GAS_COST * calldata_size
    );
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=specific_base_gas_cost, request_struct_size=MetaTxV0Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    local contract_address = request.contract_address;
    local selector = request.selector;
    local caller_execution_info: ExecutionInfo* = caller_execution_context.execution_info;
    local old_tx_info: TxInfo* = caller_execution_info.tx_info;

    if (selector != EXECUTE_ENTRY_POINT_SELECTOR) {
        write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
        return ();
    }

    // Sanity check: Verify that `signature` is a valid Sierra array.
    assert_nn_le(request.signature_end - request.signature_start, SIERRA_ARRAY_LEN_BOUND - 1);

    let (state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
        key=contract_address
    );

    // Compute the meta-transaction hash.
    let pedersen_ptr = builtin_ptrs.selectable.pedersen;
    with pedersen_ptr {
        let meta_tx_hash = compute_meta_tx_v0_hash(
            contract_address=contract_address,
            entry_point_selector=selector,
            calldata=calldata_start,
            calldata_size=calldata_size,
            chain_id=old_tx_info.chain_id,
        );
    }
    update_pedersen_in_builtin_ptrs(pedersen_ptr=pedersen_ptr);

    // Prepare execution context.
    tempvar new_tx_info = new TxInfo(
        version=0,
        account_contract_address=contract_address,
        max_fee=0,
        signature_start=request.signature_start,
        signature_end=request.signature_end,
        transaction_hash=meta_tx_hash,
        chain_id=old_tx_info.chain_id,
        nonce=0,
        resource_bounds_start=cast(0, ResourceBounds*),
        resource_bounds_end=cast(0, ResourceBounds*),
        tip=0,
        paymaster_data_start=cast(0, felt*),
        paymaster_data_end=cast(0, felt*),
        nonce_data_availability_mode=0,
        fee_data_availability_mode=0,
        account_deployment_data_start=cast(0, felt*),
        account_deployment_data_end=cast(0, felt*),
        proof_facts_start=cast(0, felt*),
        proof_facts_end=cast(0, felt*),
    );

    let (deprecated_tx_info_ptr: DeprecatedTxInfo*) = alloc();
    tempvar execution_context: ExecutionContext* = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=state_entry.class_hash,
        calldata_size=calldata_size,
        calldata=calldata_start,
        execution_info=new ExecutionInfo(
            block_info=caller_execution_info.block_info,
            tx_info=new_tx_info,
            caller_address=ORIGIN_ADDRESS,
            contract_address=contract_address,
            selector=selector,
        ),
        deprecated_tx_info=deprecated_tx_info_ptr,
    );
    fill_deprecated_tx_info(tx_info=new_tx_info, dst=execution_context.deprecated_tx_info);

    // Since we process the revert log backwards, entries before this point belong to the calling
    // contract.
    assert [revert_log] = RevertLogEntry(
        selector=CHANGE_CONTRACT_ENTRY, value=caller_execution_info.contract_address
    );
    let revert_log = &revert_log[1];

    contract_call_helper(
        remaining_gas=remaining_gas,
        block_context=block_context,
        execution_context=execution_context,
    );

    // Entries before this point belong to the callee.
    assert [revert_log] = RevertLogEntry(selector=CHANGE_CONTRACT_ENTRY, value=contract_address);
    let revert_log = &revert_log[1];

    return ();
}
