### Title
`deprecated_declared_classes` Silently Dropped from State Diff Commitment — (`crates/blockifier/src/state/cached_state.rs`)

### Summary

The `From<CommitmentStateDiff> for ThinStateDiff` conversion unconditionally sets `deprecated_declared_classes: Vec::new()`. Every code path that produces a block commitment — `BlockExecutionArtifacts::new()`, `thin_state_diff()` — goes through this conversion. When a block contains a deprecated (Cairo 0) class declaration, the resulting `state_diff_commitment`, `state_diff_length` inside `concatenated_counts`, and the stored `ThinStateDiff` all silently omit those class hashes. The committed `partial_block_hash` is therefore wrong for any block that includes a `DECLARE v1` transaction.

### Impact Explanation

For any block that includes a `DECLARE v1` (deprecated Cairo 0 class) transaction:

1. **Wrong `state_diff_commitment`** — `chain_deprecated_declared_classes` receives an empty slice instead of the actual class hashes; the Poseidon hash is wrong.
2. **Wrong `state_diff_length` in `concatenated_counts`** — the packed field encodes a count that is too small by the number of deprecated declarations; `concatenated_counts` is wrong.
3. **Wrong `partial_block_hash`** — both corrupted fields feed directly into `calculate_block_hash`, so the proposal commitment and the final block hash are wrong.
4. **Wrong stored `ThinStateDiff`** — the DB record is missing the deprecated class hashes; any downstream consumer (SNOS prover, RPC `starknet_getStateUpdate`, P2P sync) receives an authoritative-looking but incorrect state diff.

This matches the impact category: *Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input.*

### Likelihood Explanation

`DECLARE v1` transactions (deprecated Cairo 0 class declarations) are valid Starknet transactions that the gateway must accept. Any user who submits such a transaction triggers the discrepancy. No special privilege is required; the attacker only needs to submit a well-formed `DECLARE v1` transaction.

### Recommendation

`CommitmentStateDiff` must be extended with a `deprecated_declared_classes: Vec<ClassHash>` field, populated by the blockifier when a deprecated class declaration is executed. The `From<CommitmentStateDiff> for ThinStateDiff` conversion must propagate this field instead of hard-coding `Vec::new()`. The TODO comment at line 699 of `cached_state.rs` already acknowledges this debt; it must be resolved before the sequencer handles `DECLARE v1` transactions in production.

### Proof of Concept

1. Submit a `DECLARE v1` transaction declaring a Cairo 0 class to the sequencer.
2. The transaction is accepted and included in a block.
3. Capture the `state_diff_commitment` from the block header.
4. Independently compute `calculate_state_diff_hash` on the `ThinStateDiff` returned by `starknet_getStateUpdate` — it will match the sequencer's (wrong) value because the stored diff is also missing the deprecated class hash.
5. Manually compute the correct hash by including the deprecated class hash in `chain_deprecated_declared_classes` — the two values differ, proving the commitment is wrong.
6. The SNOS prover, which processes the full execution trace including the deprecated class declaration, will produce a state diff that includes the class hash; its computed `state_diff_commitment` will not match the one in the block header, causing proof verification to fail.

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
