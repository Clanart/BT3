### Title
Integer Division Truncation in `NewDynamicFeeChecker` Enables Zero-Fee Cosmos SDK Transactions When Base Fee Is Zero - (File: `ante/evm/fee_checker.go`)

### Summary
`NewDynamicFeeChecker` computes `feeCap` via integer division (`fee / gas`). When a user submits a Cosmos SDK transaction with a total fee amount smaller than the gas limit, the division truncates `feeCap` to zero. If the chain's base fee has also reached zero (possible when `MinGasPrice = 0`, the default), the base-fee check passes with `feeCap = 0 >= baseFeeInt = 0`, `effectivePrice` resolves to zero, and the returned `effectiveFee` carries a zero-amount coin. The Cosmos SDK's `DeductFeeDecorator` skips deduction when `fee.IsZero()` is true, so the transaction is accepted and committed with no fees charged.

### Finding Description
In `ante/evm/fee_checker.go`, `NewDynamicFeeChecker` derives the per-gas price cap for a Cosmos SDK transaction using integer (truncating) division:

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))   // line 83
```

`sdkmath.Int.Quo` performs truncating integer division. Whenever `fee < gas` (e.g., `fee = 1 aphoton`, `gas = 1 000 000`), `feeCap` truncates to `0`.

The guard that follows compares `feeCap` against `baseFeeInt`:

```go
if feeCap.LT(baseFeeInt) {                          // line 86
    return nil, 0, errorsmod.Wrapf(...)
}
```

When `baseFee = 0` (a reachable state, see below), `feeCap (0) < baseFeeInt (0)` is false, so the check passes silently.

`effectivePrice` is then computed as:

```go
effectivePrice := sdkmath.NewIntFromBigInt(
    types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
// = min(maxPriorityPrice + 0, 0) = 0
```

and the returned fee coins are:

```go
effectiveFee := sdk.Coins{{Denom: denom, Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas))}}
// Amount = 0 * gas = 0
```

The Cosmos SDK `DeductFeeDecorator.checkDeductFee` guards deduction with `if !fee.IsZero()`. Because `sdk.Coins{{Denom: "aphoton", Amount: 0}}.IsZero()` evaluates to `true`, the deduction is skipped entirely and the transaction is committed with zero fees paid.

**How `baseFee = 0` is reached**: `CalculateBaseFee` in `x/feemarket/keeper/eip1559.go` lower-bounds the base fee at `minGasPrice`:

```go
minGasPrice := params.MinGasPrice.TruncateInt().BigInt()
return ethermint.BigMax(x.Sub(parentBaseFee, baseFeeDelta), minGasPrice)  // line 104
```

The default `MinGasPrice = 0` (`DefaultMinGasPrice = sdkmath.LegacyZeroDec()`). On a chain with low block utilisation, the base fee decreases by up to `1/BaseFeeChangeDenominator` per block. Once it reaches `0`, `baseFeeDelta = 0` and it remains at `0` indefinitely. `GetBaseFee` then returns `big.NewInt(0)` (not `nil`), so `NewDynamicFeeChecker` does **not** fall back to the validator min-gas-prices path.

### Impact Explanation
Any unprivileged user can submit Cosmos SDK transactions (bank sends, staking, governance votes, IBC, etc.) with `fee = 1 aphoton` and an arbitrarily large `gas` limit. When `baseFee = 0`, every such transaction is accepted and committed with zero fees deducted from the sender's account. This constitutes a complete fee-market bypass for Cosmos SDK transactions, enabling unlimited free transaction submission and potential mempool/block-space exhaustion.

This matches the allowed High impact: *"ante handler … bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

### Likelihood Explanation
- `MinGasPrice` defaults to `0` on all Ethermint-based chains.
- Any chain that experiences sustained low block utilisation (common during off-peak periods or early deployment) will see the base fee decay to `0` and remain there.
- Once `baseFee = 0`, the exploit requires only a standard Cosmos SDK transaction with `fee < gas`, which is trivially constructable by any wallet or CLI user.
- No special privileges, keys, or validator cooperation are required.

### Recommendation
Replace the truncating integer division with a ceiling division or a pre-multiplication check:

```go
// Option A: reject if fee < baseFee * gas before dividing
requiredFee := baseFeeInt.Mul(sdkmath.NewIntFromUint64(gas))
if fee.LT(requiredFee) {
    return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, ...)
}
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
```

Additionally, enforce a non-zero lower bound for `MinGasPrice` in `validateMinGasPrice`, or ensure `CalculateBaseFee` never returns `0` when the fee market is active.

### Proof of Concept

**Precondition**: Chain running with default `MinGasPrice = 0`; base fee has decayed to `0` after sustained low utilisation.

**Attacker action** (Cosmos SDK CLI):
```

  --fees 1aphoton \
  --gas 10000000 \
  --chain-id ethermint-1
```

**Trace through `NewDynamicFeeChecker`**:
