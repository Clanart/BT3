### Title
Zero-Fee Bypass via Non-EVM Denom + `ExtensionOptionDynamicFeeTx` When `NoBaseFee=true` — (`ante/evm/fee_checker.go`, `x/evm/types/utils.go`)

### Summary
When `NoBaseFee=true` and the London hardfork is active, `GetBaseFee` returns `0` instead of `nil`. This prevents the fallback to `checkTxFeeWithValidatorMinGasPrices`. An attacker submitting a Cosmos tx with fees in a non-EVM denom and `ExtensionOptionDynamicFeeTx` causes `effectiveFee = [{evmDenom: 0}]`, which is treated as zero and never deducted — resulting in a completely fee-free transaction.

### Finding Description
The root cause is in `x/evm/types/utils.go` `GetBaseFee`:

```go
func GetBaseFee(...) *big.Int {
    if !IsLondon(ethCfg, height) {
        return nil
    }
    baseFee := feemarketParams.GetBaseFee()
    // should not be nil if london hardfork enabled
    if baseFee == nil {
        return new(big.Int)  // BUG: returns 0, not nil
    }
    return baseFee
}
```

`feemarkettypes.Params.GetBaseFee()` legitimately returns `nil` when `NoBaseFee=true`. The comment "should not be nil if london hardfork enabled" is incorrect — `NoBaseFee=true` is a valid governance-settable configuration. By returning `new(big.Int)` (zero) instead of `nil`, the code bypasses the intended fallback path in `NewDynamicFeeChecker`.

With `baseFee=0` and `fee=feeCoins.AmountOf(evmDenom)=0`:
- `feeCap = 0`
- `feeCap.LT(baseFeeInt)` → `0 < 0` = false → passes
- `effectivePrice = min(maxPriorityPrice + 0, 0) = 0`
- `effectiveFee = [{evmDenom: 0}]`
- `fee.IsZero() = true` → `DeductFees` not called [1](#0-0) [8](#0-7) [7](#0-6) 

### Impact Explanation
Any unprivileged user can submit Cosmos transactions with zero effective fees (neither EVM denom nor the declared non-EVM denom is deducted) on any chain with `NoBaseFee=true`. This is a fee mis-accounting bug: the ante handler certifies the tx as fee-paid while deducting nothing. This maps to the allowed High impact: "ante handler bug that permits valid user funds/fees to be mis-accounted."

### Likelihood Explanation
`NoBaseFee=true` is a governance-settable parameter with a defined default of `false`, but it is a documented, supported configuration for chains that opt out of EIP-1559 fee market dynamics. Chains using fixed-fee or zero-fee models (testnets, permissioned chains, or chains with `MinGasPrice=0`) are directly affected. The attack requires no special privileges — any tx submitter can trigger it. [9](#0-8) [10](#0-9) 

### Recommendation
In `GetBaseFee` (`x/evm/types/utils.go`), when `NoBaseFee=true` causes `feemarketParams.GetBaseFee()` to return `nil`, return `nil` (not `new(big.Int)`) so that `NewDynamicFeeChecker` correctly falls back to `checkTxFeeWithValidatorMinGasPrices`:

```go
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    if !IsLondon(ethCfg, height) {
        return nil
    }
    if feemarketParams.NoBaseFee {
        return nil  // signal: use validator min-gas-prices fallback
    }
    return feemarketParams.GetBaseFee()
}
```

Alternatively, add an explicit denom check in `NewDynamicFeeChecker` to reject fees not denominated in the EVM denom when the dynamic fee path is active.

### Proof of Concept
Unit test preconditions:
- `feemarketParams.NoBaseFee = true`
- London hardfork enabled at height 1
- `evmDenom = "aphoton"`
- Tx fee: `[{1000uatom}]`, gas: `100000`
- `ExtensionOptionDynamicFeeTx` attached

Expected (buggy) result:
1. `GetBaseFee` returns `0` (not nil)
2. `fee = feeCoins.AmountOf("aphoton") = 0`
3. `feeCap = 0`, `baseFeeInt = 0`, check `0 < 0` passes
4. `effectiveFee = [{aphoton: 0}]`
5. `DeductFees` not called
6. Attacker's `aphoton` balance unchanged, `uatom` balance unchanged
7. Tx executes with zero fees paid [11](#0-10)

```go
func NewDynamicFeeChecker(ethCfg *params.ChainConfig, evmParams *types.Params, feemarketParams *feemarkettypes.Params) authante.TxFeeChecker {
	return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
		feeTx, ok := tx.(sdk.FeeTx)
		if !ok {
			return nil, 0, fmt.Errorf("tx must be a FeeTx")
		}

		if ctx.BlockHeight() == 0 {
			// genesis transactions: fallback to min-gas-price logic
			return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
		}

		denom := evmParams.EvmDenom

		baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
		if baseFee == nil {
			// london hardfork is not enabled: fallback to min-gas-prices logic
			return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
		}

		// default to `MaxInt64` when there's no extension option.
		maxPriorityPrice := sdkmath.NewInt(math.MaxInt64)

		// get the priority tip cap from the extension option.
		if hasExtOptsTx, ok := tx.(authante.HasExtensionOptionsTx); ok {
			for _, opt := range hasExtOptsTx.GetExtensionOptions() {
				if extOpt, ok := opt.GetCachedValue().(*ethermint.ExtensionOptionDynamicFeeTx); ok {
					maxPriorityPrice = extOpt.MaxPriorityPrice
					break
				}
			}
		}

		if maxPriorityPrice.Sign() == -1 {
			return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, "priority fee is negative")
		}

		gas := feeTx.GetGas()
		feeCoins := feeTx.GetFee()
		fee := feeCoins.AmountOf(denom)

		feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
		baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)

		if feeCap.LT(baseFeeInt) {
			return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, "insufficient gas prices; got: %s required: %s", feeCap, baseFeeInt)
		}

		// calculate the effective gas price using the EIP-1559 logic.
		effectivePrice := sdkmath.NewIntFromBigInt(types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))

		// NOTE: create a new coins slice without having to validate the denom
		effectiveFee := sdk.Coins{
			{
				Denom:  denom,
				Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas)),
			},
		}

		bigPriority := effectivePrice.Sub(baseFeeInt).Quo(types.DefaultPriorityReduction)
		priority := int64(math.MaxInt64)

		if bigPriority.IsInt64() {
			priority = bigPriority.Int64()
		}

		return effectiveFee, priority, nil
	}
```
