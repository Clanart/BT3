Looking at the actual code in `batch_insert` and `add_leaf`, I need to trace the exact execution path to determine if the guards are truly bypassed.

**Key code paths:**

`batch_insert` (lines 578–603): only the last ≤2 items (via `pop()`) go through `insert()`, which has the guards. All remaining items go through the bulk path directly to `insert_entry_to_blob`. [1](#0-0) 

`insert_entry_to_blob` calls `add_leaf` unconditionally for leaf nodes: [2](#0-1) 

`add_leaf` uses `HashMap::insert`, which **silently overwrites** existing entries — no duplicate check: [3](#0-2) 

By contrast, `insert()` has explicit guards before reaching `add_leaf`: [4](#0-3) 

**What actually happens with a duplicate key in the bulk path:**

1. Original leaf at blob index A with `key = K` → `key_to_index[K] = A`
2. Duplicate leaf at blob index B with `key = K` → `add_leaf` runs `key_to_index.insert(K, B)`, silently overwriting to `key_to_index[K] = B`
3. Both leaves now exist in the blob, but the cache points only to B
4. `get_proof_of_inclusion(K)` looks up `key_to_index[K]` → gets B → returns a proof for the attacker-controlled leaf [5](#0-4) 

**Does `check_integrity` catch this?** Yes — it would fire `IntegrityKeyToIndexCacheIndex` (original leaf at A finds cache pointing to B) and `IntegrityKeyToIndexCacheLength` (blob has N+1 leaves but cache has N entries). However, `check_integrity_on_drop` is only enabled in test mode (`cfg!(test)`), and `check_integrity` is **not called automatically** in production after `batch_insert`. [6](#0-5) [7](#0-6) 

**Severity assessment:**

The bug is real and locally testable: a `Vec` with `leaf_count > 1` already in the tree, plus any new entry sharing an existing `KeyId`, bypasses all guards. The tree ends up with two leaves sharing a key, the cache is corrupted, and `get_proof_of_inclusion` returns a proof for the wrong leaf.

However, the **exploitability as a bounty-grade vulnerability** depends on whether an unprivileged external attacker can control the input to `batch_insert`. The function is a Rust library API — it is not directly exposed to network input. Whether a DataLayer node exposes a path where untrusted data flows into `batch_insert` without prior deduplication is not determinable from this codebase alone. The DataLayer layer above may deduplicate keys before calling `batch_insert`.

### Title
`batch_insert` Bulk Path Bypasses Duplicate-Key Guard, Corrupting `BlockStatusCache` and Enabling Forged Proof of Inclusion — (`crates/chia-datalayer/src/merkle/blob.rs`)

### Summary
`batch_insert` only routes the last ≤2 items through `insert()`, which checks `KeyAlreadyPresent`/`HashAlreadyPresent`. All remaining items go directly to `insert_entry_to_blob` → `add_leaf`, which calls `HashMap::insert` without any duplicate check, silently overwriting `key_to_index` and `leaf_hash_to_index`. The result is a blob containing two leaves with the same `KeyId`, with the cache pointing only to the later one, causing `get_proof_of_inclusion` to return a proof for the wrong leaf.

### Finding Description
In `batch_insert`, when `self.block_status_cache.leaf_count() > 1`, the entire input Vec bypasses `insert()` and goes through the bulk path. Even when `leaf_count <= 1`, only the last two items (via `pop()`) are checked; all earlier items in the Vec are unchecked. `add_leaf` calls `self.key_to_index.insert(leaf.key, index)` and `self.leaf_hash_to_index.insert(leaf.hash, index)` — both are plain `HashMap::insert` with no collision detection. [3](#0-2) 

### Impact Explanation
`get_proof_of_inclusion` resolves a key exclusively through `key_to_index`. After the overwrite, it returns a proof anchored to the attacker-controlled leaf hash, not the original. This matches the allowed High impact: *DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion or lets untrusted input prove invalid state.* [8](#0-7) 

### Likelihood Explanation
**Conditional.** The bug is unambiguously present in the production code path. Exploitability requires that an unprivileged caller can supply the `Vec` passed to `batch_insert` without prior deduplication at the DataLayer application layer. If the caller deduplicates, the bug is latent but not directly exploitable. If not, it is directly triggerable.

### Recommendation
Add duplicate-key and duplicate-hash checks at the top of `batch_insert` before any items are processed, mirroring the guards in `insert()`. Alternatively, replace `HashMap::insert` in `add_leaf` with an entry-API check that returns an error on collision, and propagate that error through `insert_entry_to_blob` and `batch_insert`.

    fn check_just_integrity(&self) -> Result<(), Error> {
        let mut leaf_count: usize = 0;
        let mut internal_count: usize = 0;
        let mut child_to_parent: HashMap<TreeIndex, TreeIndex> = HashMap::new();

        for item in ParentFirstIterator::new(&self.blob, None) {
            let (index, block) = item?;
            if let Some(parent) = block.node.parent().0 {
                if child_to_parent.remove(&index) != Some(parent) {
                    return Err(Error::IntegrityParentChildMismatch(index));
                }
            }
            match block.node {
                Node::Internal(node) => {
                    internal_count += 1;
                    child_to_parent.insert(node.left, index);
                    child_to_parent.insert(node.right, index);
                }
                Node::Leaf(node) => {
                    leaf_count += 1;
                    let cached_index = self
                        .block_status_cache
                        .get_index_by_key(node.key)
                        .ok_or(Error::IntegrityKeyNotInCache(node.key))?;
                    if *cached_index != index {
                        return Err(Error::IntegrityKeyToIndexCacheIndex(
                            node.key,
                            index,
                            *cached_index,
                        ));
                    }
                    assert!(
                        !self.block_status_cache.is_index_free(index),
                        "{}",
                        format!("active index found in free index list: {index:?}")
                    );
                }
            }
        }

        let key_to_index_cache_length = self.block_status_cache.key_to_index.len();
        if leaf_count != key_to_index_cache_length {
            return Err(Error::IntegrityKeyToIndexCacheLength(
                leaf_count,
                key_to_index_cache_length,
            ));
        }
        let leaf_hash_to_index_cache_length = self.block_status_cache.leaf_hash_to_index.len();
        if leaf_count != leaf_hash_to_index_cache_length {
            return Err(Error::IntegrityLeafHashToIndexCacheLength(
                leaf_count,
                leaf_hash_to_index_cache_length,
            ));
        }
        let total_count = leaf_count + internal_count + self.block_status_cache.free_index_count();
        let extend_index = self.extend_index();
        if total_count != extend_index.0 as usize {
            return Err(Error::IntegrityTotalNodeCount(extend_index, total_count));
        }
        if !child_to_parent.is_empty() {
            return Err(Error::IntegrityUnmatchedChildParentRelationships(
                child_to_parent.len(),
            ));
        }

        Ok(())
    }

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
