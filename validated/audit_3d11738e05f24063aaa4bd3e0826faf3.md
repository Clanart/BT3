### Title
`_pause()` Replaces All Pause Flags Instead of Accumulating Them, Silently Unpausing Previously Paused Operations - (File: evm/src/omni-bridge/contracts/SelectivePausableUpgradable.sol)

---

### Summary

`SelectivePausableUpgradable._pause()` uses a plain assignment (`$._pausedFlags = flags`) rather than a bitwise OR accumulation. Any call to `pause(X)` silently clears every previously set pause bit that is not included in `X`. In an emergency where an admin first pauses `fin_transfer` and then separately pauses `init_transfer`, the second call atomically unpauses `fin_transfer`, removing the security protection that was just applied.

---

### Finding Description

`_pause()` in `SelectivePausableUpgradable.sol` writes the caller-supplied bitmap directly into storage:

```solidity
function _pause(uint256 flags) internal virtual {
    SelectivePausableStorage storage $ = _getSelectivePausableStorage();
    $._pausedFlags = flags;          // ← assignment, not |=
    emit Paused(_msgSender(), $._pausedFlags);
}
```

`OmniBridge.sol` exposes this through two entry points:

```solidity
function pause(uint256 flags) external onlyRole(DEFAULT_ADMIN_ROLE) {
    _pause(flags);
}
function pauseAll() external onlyRole(PAUSABLE_ADMIN_ROLE) {
    uint256 flags = PAUSED_FIN_TRANSFER | PAUSED_INIT_TRANSFER | PAUSED_DEPLOY_TOKEN;
    _pause(flags);
}
```

The three selective pause constants are:

```
PAUSED_INIT_TRANSFER = 1 << 0  (= 1)
PAUSED_FIN_TRANSFER  = 1 << 1  (= 2)
PAUSED_DEPLOY_TOKEN  = 1 << 2  (= 4)
```

A natural emergency sequence:

| Step | Call | Resulting `_pausedFlags` | `fin_transfer` paused? |
|------|------|--------------------------|------------------------|
| 1 | `pause(PAUSED_FIN_TRANSFER)` | `2` | ✅ yes |
| 2 | `pause(PAUSED_INIT_TRANSFER)` | `1` | ❌ **no** |

After step 2, `fin_transfer` is silently unpaused. The `Paused` event emits only the new value (`1`), giving no indication that `PAUSED_FIN_TRANSFER` was cleared. The admin has no on-chain signal that the protection was removed.

The same pattern exists in StarkNet's `set_pause_flags`:

```cairo
fn set_pause_flags(ref self: ContractState, flags: u8) {
    self.accesscontrol.assert_only_role(DEFAULT_ADMIN_ROLE);
    let old_flags = self.pause_flags.read();
    self.pause_flags.write(flags);   // ← assignment, not |=
    ...
}
```

---

### Impact Explanation

`fin_transfer` is the function that mints bridged tokens or releases locked native tokens on EVM. If it was paused because a security incident was detected (e.g., a compromised MPC key is being used to forge transfer payloads), and the admin then issues a second `pause()` call to also halt `init_transfer`, `fin_transfer` becomes callable again. An attacker holding a valid (or forged) MPC-signed payload can immediately call `finTransfer()` to mint bridged tokens or release locked ERC-20/ETH, constituting an unauthorized mint or release of bridged assets.

---

### Likelihood Explanation

In a real incident, operators act under time pressure and may issue multiple targeted `pause()` calls in sequence rather than computing the combined bitmask in advance. The `Paused` event does not highlight which bits were *cleared*, only which bits are now set, so the operator has no immediate feedback that a prior protection was removed. The `pauseAll()` helper exists precisely because the selective path is error-prone, but it requires `PAUSABLE_ADMIN_ROLE` and pauses everything — it cannot be used to add a single flag to an existing partial pause.

---

### Recommendation

Change `_pause` to accumulate flags rather than replace them, and introduce a symmetric `_unpause` that clears specific bits:

```solidity
function _pause(uint256 flags) internal virtual {
    SelectivePausableStorage storage $ = _getSelectivePausableStorage();
    $._pausedFlags |= flags;          // accumulate
    emit Paused(_msgSender(), $._pausedFlags);
}

function _unpause(uint256 flags) internal virtual {
    SelectivePausableStorage storage $ = _getSelectivePausableStorage();
    $._pausedFlags &= ~flags;         // clear specific bits
    emit Unpaused(_msgSender(), $._pausedFlags);
}
```

Apply the same fix to StarkNet's `set_pause_flags`. If the replacement semantics are intentional, at minimum emit both the old and new flag values in the event so operators can detect accidental clearance.

---

### Proof of Concept

1. A compromised MPC key is detected; admin calls `pause(PAUSED_FIN_TRANSFER)` → `_pausedFlags = 2`. `finTransfer` reverts for all callers.
2. Admin decides to also halt new outbound transfers and calls `pause(PAUSED_INIT_TRANSFER)` → `_pausedFlags = 1`. `PAUSED_FIN_TRANSFER` bit is silently cleared.
3. Attacker calls `finTransfer(signature, payload)` with a payload signed by the compromised MPC key. The `whenNotPaused(PAUSED_FIN_TRANSFER)` modifier passes because `(1 & 2) == 0`.
4. `finTransfer` mints bridged tokens to the attacker's address or releases locked ERC-20/ETH, completing an unauthorized asset release.
