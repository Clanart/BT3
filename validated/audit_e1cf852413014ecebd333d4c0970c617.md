### Title
`SyncStateReader::get_compiled_class_hash` Is `todo!()` — Gateway Panics and Rejects Valid Declare Transactions - (File: `crates/apollo_gateway/src/sync_state_reader.rs`)

---

### Summary

The production `SyncStateReader` used by the gateway for stateful transaction validation implements the `BlockifierStateReader` trait but leaves `get_compiled_class_hash` as a bare `todo!()`. Any Declare transaction submitted to the gateway triggers this code path during stateful validation, causing a runtime panic that rejects the transaction with an internal error (or crashes the gateway task).

---

### Finding Description

`SyncStateReader` is the live state reader used by the gateway to validate transactions against the current chain state. It implements `blockifier::state::state_api::StateReader` (aliased as `BlockifierStateReader`). Every method in the trait is properly implemented — `get_storage_at`, `get_nonce_at`, `get_class_hash_at`, `get_compiled_class` — except one:

```rust
fn get_compiled_class_hash(&self, _class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    todo!()
}
``` [1](#0-0) 

The `StateReader` trait requires this method to return the `CompiledClassHash` associated with a declared Cairo 1 class: [2](#0-1) 

When the gateway validates a Declare transaction, the blockifier wraps `SyncStateReader` in a `CachedState<SyncStateReader>`. `CachedState::get_compiled_class_hash` checks its local cache first; on a cache miss it calls through to the underlying reader:

```rust
fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    let mut cache = self.cache.borrow_mut();
    if cache.get_compiled_class_hash(class_hash).is_none() {
        let compiled_class_hash = self.state.get_compiled_class_hash(class_hash)?;  // ← hits todo!()
        ...
    }
``` [3](#0-2) 

During Declare transaction execution the blockifier reads the current compiled class hash to enforce the "class may be declared only once" invariant. This read is a cache miss on the first call, so it falls through to `SyncStateReader::get_compiled_class_hash`, which panics.

The `SyncStateReaderFactory` is the concrete factory wired into the gateway for all stateful validation: [4](#0-3) 

The `StateReaderAndContractManager` wrapper also delegates `get_compiled_class_hash` directly to the inner state reader, so there is no interception layer that could mask the panic: [5](#0-4) 

The analog to the external `mintBatch()` finding is exact: a method is declared in the interface (`StateReader` trait), called unconditionally by the consumer (`CachedState`), and the production implementation (`SyncStateReader`) contains no real body — only `todo!()`.

---

### Impact Explanation

**High — Mempool/gateway admission rejects valid transactions before sequencing.**

Every Declare V2 or V3 transaction submitted to the gateway will panic inside the stateful validation task. Depending on the Tokio task boundary, this either:
- propagates as an unhandled panic that crashes the gateway worker, or
- is caught and converted to an internal server error, causing the transaction to be silently dropped.

In either case, no Declare transaction can pass gateway admission. New Cairo 1 class declarations are permanently blocked at the gateway layer, preventing any new contract class from entering the mempool or being sequenced.

---

### Likelihood Explanation

**High.** The trigger is any Declare transaction (V2 or V3) submitted to the gateway. This is a standard, unprivileged user action. No special privileges, malformed inputs, or race conditions are required. The code path is deterministic: every Declare transaction causes a cache miss on `get_compiled_class_hash` for the newly declared class hash, which unconditionally calls the `todo!()` body.

---

### Recommendation

Implement `get_compiled_class_hash` in `SyncStateReader` by querying the state sync client, mirroring the pattern used by the other implemented methods. The `StateSyncClient` trait already exposes the necessary data (compiled class hash is part of the state diff stored per block). Alternatively, delegate to the `class_manager_client` if it exposes a compiled-class-hash lookup, consistent with how `get_compiled_class` is implemented.

The `apollo_rpc_execution` state reader provides a reference implementation that reads the compiled class hash from storage via `get_class_definition_block_number` and the block's state diff: [6](#0-5) 

---

### Proof of Concept

1. Start the Apollo sequencer node with the gateway enabled.
2. Submit any valid Declare V3 transaction (e.g., a minimal Cairo 1 contract) via the RPC endpoint.
3. The gateway calls `SyncStateReaderFactory::get_blockifier_state_reader_and_gateway_fixed_block_from_latest_block`, which returns a `SyncStateReader`.
4. The blockifier wraps it in `CachedState<SyncStateReader>` and begins stateful validation of the Declare transaction.
5. During execution, `CachedState::get_compiled_class_hash(class_hash)` is called for the newly declared class hash — a guaranteed cache miss.
6. `SyncStateReader::get_compiled_class_hash` is invoked and panics: `not yet implemented`.
7. The gateway returns an internal error to the caller; the Declare transaction is rejected and never reaches the mempool or batcher.

    fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
        if let Some(pending_data) = &self.maybe_pending_data {
            for DeclaredClassHashEntry { class_hash: other_class_hash, compiled_class_hash } in
                &pending_data.declared_classes
            {
                if class_hash == *other_class_hash {
                    return Ok(*compiled_class_hash);
                }
            }
        }

        let maybe_block_number = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_reader()
            .map_err(storage_err_to_state_err)?
            .get_class_definition_block_number(&class_hash)
            .map_err(storage_err_to_state_err)?;

        // Cairo 0 classes (and undeclared classes) do not have a compiled class hash.
        // According to the trait, return the default value.
        let Some(block_number) = maybe_block_number else {
            return Ok(CompiledClassHash::default());
        };

        let state_diff = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_diff(block_number)
            .map_err(storage_err_to_state_err)?
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing state diff at block {block_number}."
            )))?;

        let compiled_class_hash = state_diff
            .class_hash_to_compiled_class_hash
            .get(&class_hash)
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing class declaration at block {block_number}, class \
                 {class_hash}."
            )))?;

        Ok(*compiled_class_hash)
    }
