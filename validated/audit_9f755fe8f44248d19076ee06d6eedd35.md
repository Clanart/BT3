### Title
Missing Declared Class Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the new class hash corresponds to a declared contract class. Any contract can replace its own class hash with an arbitrary, undeclared value. Once this happens, the contract becomes permanently uncallable at the OS proof level, freezing all funds held within it.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function writes the caller-supplied `class_hash` directly into `contract_state_changes` with no validation that the hash has ever been declared:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges the missing check. The consequence is fatal at the proof layer. When any subsequent transaction calls the affected contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [2](#0-1) 

For an undeclared class hash, `dict_read` returns `0` (the default for an uninitialized dict entry). `find_element` then searches the compiled class facts for a compiled class whose hash is `0`. No such entry can legitimately exist (Poseidon/Blake hash outputs are never `0` for valid inputs), so `find_element` fails with an assertion error. The proof is invalid. The sequencer cannot include any transaction calling this contract in any future block.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with an undeclared value, the OS cannot construct a valid proof for any call to that contract. The sequencer is permanently blocked from including such transactions. All funds held by the contract are irrecoverably frozen. No upgrade, migration, or recovery path exists at the protocol level.

---

### Likelihood Explanation

**High.**

- The `replace_class` syscall is a standard, publicly accessible syscall callable by any executing contract.
- The missing check is explicitly acknowledged in the codebase as a TODO with a future date, confirming it is absent in the current production code.
- No privileged role, leaked key, or operator cooperation is required. Any user who can deploy a contract and invoke it can trigger this path.
- A realistic attack vector exists: deploy a contract that appears legitimate (e.g., a vault or token), attract user deposits, then call `replace_class` with a random undeclared felt value.

---

### Recommendation

In `execute_replace_class`, before writing the new class hash to `contract_state_changes`, verify that the new class hash has a non-zero compiled class hash entry in `contract_class_changes`:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the invariant enforced during class declaration (`dict_update` with `prev_value=0, new_value=compiled_class_hash`) and ensures `replace_class` can only target classes that the OS can actually execute.

---

### Proof of Concept

1. Attacker deploys a malicious vault contract whose code contains a call to `replace_class(0xdeadbeef)` (a random, never-declared felt).
2. Users deposit funds into the vault (the contract holds ETH/tokens in its storage).
3. Attacker calls the vault, triggering `replace_class(0xdeadbeef)`.
4. `execute_replace_class` in `syscall_impls.cairo` (line 896–914) writes `class_hash=0xdeadbeef` into `contract_state_changes` with no validation.
5. In the next block, any transaction calling the vault reaches `execute_entry_point` (line 154–166 of `execute_entry_point.cairo`).
6. `dict_read(key=0xdeadbeef)` returns `0` (undeclared → no entry in `contract_class_changes`).
7. `find_element(..., key=0)` fails — no compiled class with hash `0` exists.
8. The proof is invalid; the sequencer cannot include the transaction.
9. The vault and all its funds are permanently frozen. [3](#0-2) [4](#0-3)
