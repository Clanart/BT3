### Title
ETH→STRK Oracle Rate Accepted Without Bounds Validation, Corrupting Block Gas Prices and Fee Accounting — (`crates/apollo_l1_gas_price/src/eth_to_strk_oracle.rs`)

---

### Summary

`resolve_query` in `EthToStrkOracleClient` fetches the ETH→STRK conversion rate from an external HTTP oracle and validates only that `decimals == 18`. It performs no min/max sanity check on the returned `rate` value. Because all sequencer nodes share the same oracle, a wrong-but-structurally-valid rate propagates identically to every node, passes the validator's relative margin check, and is committed into the block header as authoritative `l1_gas_price_fri` / `l1_data_gas_price_fri`. Every transaction in that block is then charged fees computed from the corrupted rate.

---

### Finding Description

**Root cause — `resolve_query` (lines 153–188):**

```rust
let rate = u128::from_str_radix(price.trim_start_matches("0x"), 16)
    .expect("Failed to parse price as u128");
// ...
if decimals != ETH_TO_STRK_QUANTIZATION {          // only structural check
    return Err(EthToStrkOracleClientError::InvalidDecimalsError(...));
}
Ok(rate)   // ← rate accepted with no min/max guard
```

`EthToStrkOracleConfig` carries no `min_rate` / `max_rate` fields, so there is no configuration-level bound either. [1](#0-0) [2](#0-1) 

**Propagation path:**

`get_l1_prices_in_fri_and_wei_and_conversion_rate` calls `apply_fee_transformations` (which clamps the **wei** prices to `min/max_l1_gas_price_wei`) and then calls `L1PricesInFri::convert_from_wei(&prices_in_wei, eth_to_fri_rate)`. The clamping of wei prices does not protect the fri prices: if `eth_to_fri_rate` is N× wrong, the resulting `l1_gas_price_fri` and `l1_data_gas_price_fri` are also N× wrong. [3](#0-2) [4](#0-3) 

**Why the validator does not catch it:**

The validator's margin check (`gas_price_fri_out_of_range` test) compares the proposer's fri prices against the validator's **own locally computed** fri prices. Because all nodes query the same oracle endpoint, they all receive the same wrong rate, compute the same wrong fri prices, and the relative comparison passes. The proposal is accepted. [5](#0-4) 

**Wrong values committed to the block:**

`convert_to_sn_api_block_info` constructs `GasPrices` directly from the proposal's fri and wei fields. These become the authoritative gas prices for every transaction executed in the block. [6](#0-5) 

---

### Impact Explanation

A wrong `eth_to_fri_rate` (e.g., 10× too high) causes `l1_gas_price_fri` and `l1_data_gas_price_fri` to be 10× too high. Every transaction in the block is charged 10× more STRK than it should be (or 10× less if the rate is too low). The corrupted prices are committed into the block header and used by the blockifier for all fee deductions, producing wrong account balances and wrong state. This matches **Critical — Incorrect fee, gas, resource accounting, balance, or L1 gas price effect with economic impact**.

---

### Likelihood Explanation

The oracle is an external HTTP service. A bug in the oracle backend, a stale or replayed response, a misconfigured decimal scaling, or a market-event-driven anomaly (analogous to the Chainlink circuit-breaker scenario in the seed report) can produce a rate that is structurally valid (`decimals == 18`, parseable hex) but economically wrong by orders of magnitude. No attacker privilege is required beyond the ability to influence the oracle's HTTP response (e.g., via a compromised CDN, DNS poisoning, or oracle-side bug).

---

### Recommendation

Add a configurable `min_eth_to_fri_rate` / `max_eth_to_fri_rate` to `EthToStrkOracleConfig` and enforce them in `resolve_query`:

```rust
if rate < config.min_eth_to_fri_rate || rate > config.max_eth_to_fri_rate {
    return Err(EthToStrkOracleClientError::RateOutOfBoundsError(rate));
}
```

If the rate is out of bounds, `resolve_query` returns an error; `get_l1_prices_in_fri_and_wei_and_conversion_rate` already has a fallback chain (previous block info → default min prices) that will be used instead. [7](#0-6) 

---

### Proof of Concept

1. Oracle returns `{"price": "0x84595161401484A000000", "decimals": 18}` — a rate 10× the expected `DEFAULT_ETH_TO_FRI_RATE` of `10^21`. `decimals == 18` passes; the rate is a valid `u128`.
2. `resolve_query` returns `Ok(rate)` with no rejection.
3. `get_l1_prices_in_fri_and_wei_and_conversion_rate` calls `apply_fee_transformations` (wei prices clamped correctly) then `L1PricesInFri::convert_from_wei` with the 10× rate → `l1_gas_price_fri` is 10× too high.
4. All validators query the same oracle, compute the same 10× fri prices, and the relative margin check passes.
5. The block is committed with `l1_gas_price_fri` = 10× correct value.
6. Every transaction in the block is charged 10× the correct STRK fee; account balances are decremented by 10× the correct amount, producing wrong state. [8](#0-7) [9](#0-8)

fn resolve_query(body: String) -> Result<u128, EthToStrkOracleClientError> {
    let Ok(json): Result<serde_json::Value, _> = serde_json::from_str(&body) else {
        return Err(EthToStrkOracleClientError::ParseError(format!(
            "Failed to parse JSON: {body}"
        )));
    };
    // Extract price from API response. Also returns MissingFieldError if value is not a string.
    let price = match json.get("price").and_then(|v| v.as_str()) {
        Some(price) => price,
        None => {
            return Err(EthToStrkOracleClientError::MissingFieldError("price".to_string(), body));
        }
    };
    let rate = u128::from_str_radix(price.trim_start_matches("0x"), 16)
        .expect("Failed to parse price as u128");
    // Extract decimals from API response. Also returns MissingFieldError if value is not a number.
    let decimals = match json.get("decimals").and_then(|v| v.as_u64()) {
        Some(decimals) => decimals,
        None => {
            return Err(EthToStrkOracleClientError::MissingFieldError(
                "decimals".to_string(),
                body,
            ));
        }
    };
    if decimals != ETH_TO_STRK_QUANTIZATION {
        return Err(EthToStrkOracleClientError::InvalidDecimalsError(
            ETH_TO_STRK_QUANTIZATION,
            decimals,
        ));
    }
    ETH_TO_STRK_SUCCESS_COUNT.increment(1);
    set_unix_now_seconds(&ETH_TO_STRK_LAST_SUCCESS_TIMESTAMP_SECONDS);
    ETH_TO_STRK_RATE.set_lossy(rate);
    Ok(rate)
}

pub(crate) fn apply_fee_transformations(
    price_info: &mut PriceInfo,
    gas_price_params: &GasPriceParams,
) {
    price_info.base_fee_per_gas = price_info
        .base_fee_per_gas
        .saturating_add(gas_price_params.l1_gas_tip_wei)
        .clamp(gas_price_params.min_l1_gas_price_wei, gas_price_params.max_l1_gas_price_wei);

    price_info.blob_fee = GasPrice(
        (gas_price_params.l1_data_gas_price_multiplier * price_info.blob_fee.0).to_integer(),
    )
    .clamp(gas_price_params.min_l1_data_gas_price_wei, gas_price_params.max_l1_data_gas_price_wei);
}

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let previous_block_info = PreviousBlockInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}
