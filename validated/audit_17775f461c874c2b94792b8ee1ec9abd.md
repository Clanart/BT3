### Title
`ProofOfInclusion.valid()` Validates Against Self-Derived Root Instead of Externally Committed Root, Enabling Forged Inclusion Proofs - (File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs)

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate only verifies the internal hash-chain consistency of the proof, but the final root comparison is made against `self.root_hash()` — a value derived from the proof's own data — rather than against any externally committed, trusted tree root. This makes the final check tautologically true and means `valid()` never actually anchors the proof to a canonical committed state. An attacker can construct a fully self-consistent `ProofOfInclusion` for any arbitrary key-value pair, and `valid()` will return `true`.

### Finding Description

In `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`, the `root_hash()` helper returns the `combined_hash` field of the **last layer of the proof itself**:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← comes from the proof, not from a trusted external source
    } else {
        self.node_hash
    }
}
```

The `valid()` method then iterates over layers, checking that each `calculated_hash == layer.combined_hash`, and accumulates `existing_hash = calculated_hash`. After the loop, `existing_hash` is exactly `last.combined_hash` (because the loop enforced equality at every step). The final assertion:

```rust
existing_hash == self.root_hash()
```

is therefore **always true** when the loop completes without returning `false`. It is a tautology — it compares `last.combined_hash` against itself.

The complete `valid()` function therefore only checks that the hash chain is internally self-consistent. It never checks whether the chain's root matches any externally committed, trusted DataLayer tree root. Because `ProofOfInclusion` is `Streamable` (deserializable from raw bytes) and its `valid()` method is exposed directly through the Python bindings (`py_valid`), an attacker who can deliver a crafted `ProofOfInclusion` to any consumer that calls `proof.valid()` as its sole check can prove the inclusion of any arbitrary key-value pair.

**Analogy to the reference report**: In `UniswapImplementation.beforeSwap()`, the spot price (`sqrtPriceX96` from `getSlot0`) is used instead of a TWAP — a value the attacker can manipulate within the same execution context. Here, `self.root_hash()` plays the role of the spot price: it is derived from the attacker-supplied proof object rather than from a committed canonical source (the actual DataLayer tree root stored on-chain). In both cases, the validation anchors to a value the attacker controls rather than to a committed canonical value.

### Impact Explanation

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."**

Any Python or Rust consumer that calls `proof.valid()` as the sole verification step — without separately comparing `proof.root_hash()` against a trusted, externally committed root — will accept a forged proof. Because `ProofOfInclusion` is `Streamable` and fully exposed via the Python wheel, an attacker can craft a proof object that claims any key-value pair is present in any DataLayer store, bypassing the Merkle integrity guarantee entirely.

### Likelihood Explanation

- `ProofOfInclusion` is `Streamable` and can be deserialized from arbitrary bytes over the network.
- The `valid()` method is the only validation method on the struct and its name implies completeness.
- The Python bindings expose `valid()` directly with no additional root-checking wrapper.
- The fuzz target and all internal tests call `proof.valid()` without a separate root check, establishing a pattern that callers are expected to follow.
- No privileged role or leaked key is required; the attacker only needs to deliver a crafted serialized `ProofOfInclusion`.

### Recommendation

`valid()` must accept an externally trusted root hash as a parameter and compare the computed chain root against it:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // ← compare against externally committed root
}
```

The no-argument `valid()` should either be removed or clearly documented as an internal-consistency-only check that provides no security guarantee against a forged proof. All call sites — including the Python bindings and the fuzz target — must be updated to supply the trusted root.

### Proof of Concept

```rust
use chia_datalayer::{Hash, Side};
use chia_datalayer::merkle::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};

fn forge_proof_for_arbitrary_key(fake_node_hash: Hash, fake_sibling_hash: Hash) -> ProofOfInclusion {
    // Compute a self-consistent combined_hash from attacker-chosen inputs.
    let combined = chia_datalayer::calculate_internal_hash(
        &fake_node_hash,
        Side::Right,
        &fake_sibling_hash,
    );
    ProofOfInclusion {
        node_hash: fake_node_hash,
        layers: vec![ProofOfInclusionLayer {
            other_hash_side: Side::Right,
            other_hash: fake_sibling_hash,
            combined_hash: combined,   // self-consistent; root_hash() returns this
        }],
    }
}

fn main() {
    let fake_node_hash = [0xAA; 32];
    let fake_sibling_hash = [0xBB; 32];
    let forged = forge_proof_for_arbitrary_key(fake_node_hash, fake_sibling_hash);
    // valid() returns true for a completely fabricated proof
    assert!(forged.valid(), "forged proof accepted");
    // root_hash() is whatever the attacker chose, not the real DataLayer root
    println!("forged root: {:?}", forged.root_hash());
}
```
