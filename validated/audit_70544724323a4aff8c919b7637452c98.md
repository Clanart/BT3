### Title
Proposer-Supplied L1 WEI Gas Prices Bypass Oracle Validation, Corrupting ETH-Denominated Fees and Block Hash - (`File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_block_info_valid` computes oracle-derived WEI gas prices but immediately discards them (binding the result to `_l1_gas_prices_wei`). The proposer's claimed `l1_gas_price_wei` and `l1_data_gas_price_wei` from `ProposalInit` are accepted without any validation and are then used verbatim in `convert_to_sn_api_block_info` to populate the `eth_gas_prices` vector fed to the blockifier. This is the direct sequencer analog of the BribeVault M-15 finding: a protocol-computed fee variable exists but is unused, and an external (proposer-supplied) value is used instead.

### Finding Description

In `is_block_info_valid` (`validate_proposal.rs`), the validator calls `get_l1_prices_in_fri_and_wei` to obtain both oracle-computed FRI and WEI prices:

```rust
let (l1_gas_prices_fri, _l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
    l1_gas_price_provider,
    init_proposed.timestamp,
    block_info_validation.previous_block_info.as_ref(),
    gas_price_params,
)
.await;
``` [1](#0-0) 

The `_l1_gas_prices_wei` binding (underscore prefix = intentionally unused) is the oracle-authoritative WEI price. Only the FRI prices are then validated:

```rust
if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
    && within_margin(l1_data_gas_price_fri_proposed, l1_data_gas_price_fri, ...))
``` [2](#0-1) 

After this validation passes, `initiate_validation` calls `convert_to_sn_api_block_info(init)`, which uses the proposer's raw WEI values directly:

```rust
let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
// ...
let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;
let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
``` [3](#0-2) 

`calculate_eth_to_fri_rate` derives the ETH/FRI rate from `init.l1_gas_price_fri / init.l1_gas_price_wei`. Because `l1_gas_price_wei` is unvalidated, the proposer controls the implied conversion rate, which in turn controls `l2_gas_price_wei`.

These WEI prices flow into `BlockInfo.gas_prices.eth_gas_prices`, which the blockifier uses for ETH-denominated fee calculations (deprecated v0/v1/v2 transactions). They also flow into `PartialBlockHashComponents`:

```rust
l1_gas_price: block_info.gas_prices.l1_gas_price_per_token(),
l1_data_gas_price: block_info.gas_prices.l1_data_gas_price_per_token(),
l2_gas_price: block_info.gas_prices.l2_gas_price_per_token(),
``` [4](#0-3) 

These are chained into `calculate_block_hash` via `gas_prices_to_hash`, so the committed block hash encodes the manipulated WEI prices. [5](#0-4) 

The same manipulated WEI prices are stored in the `BlockHeaderWithoutHash` sent to state sync:

```rust
let l1_gas_price = cende_block_info.gas_prices.l1_gas_price_per_token();
let block_header_without_hash = BlockHeaderWithoutHash { l1_gas_price, ... };
``` [6](#0-5) 

### Impact Explanation

A Byzantine proposer sets `l1_gas_price_wei = 1` (or any arbitrary value) while keeping `l1_gas_price_fri` within the allowed oracle margin. All validators accept the `ProposalInit` because only FRI prices are checked. The blockifier then executes the block with a manipulated `eth_to_fri_rate`, producing:

1. **Wrong ETH-denominated fees** for deprecated (v0/v1/v2) transactions — users can be overcharged or undercharged.
2. **Wrong `l2_gas_price_wei`** derived from the manipulated rate — affects ETH-paying L2 gas costs.
3. **Wrong block hash commitment** — the partial block hash encodes the manipulated WEI prices; all validators agree on this wrong hash because they all use the same `ProposalInit`.
4. **Wrong block header in storage and sync** — downstream consumers (RPC, state sync, proof inputs) receive authoritative-looking wrong WEI prices.

This matches: *Critical. Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact* and *High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.*

### Likelihood Explanation

Any validator selected as proposer in a BFT round can trigger this. No special capability beyond being a proposer is required. The manipulation is invisible to other validators because the validation logic explicitly discards the oracle WEI result. The `ProposalInit` WEI fields are transmitted over the P2P consensus wire format and accepted verbatim. [7](#0-6) 

### Recommendation

In `is_block_info_valid`, validate the proposed WEI prices against the oracle-computed WEI prices using the same `within_margin` check already applied to FRI prices:

```rust
let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(...).await;
// existing FRI checks ...
if !(within_margin(init_proposed.l1_gas_price_wei, l1_gas_prices_wei.l1_gas_price, margin)
    && within_margin(init_proposed.l1_data_gas_price_wei, l1_gas_prices_wei.l1_data_gas_price, margin))
{
    return Err(ValidateProposalError::InvalidBlockInfo(...));
}
```

Alternatively, derive the WEI prices in `convert_to_sn_api_block_info` from the validated FRI prices and the oracle ETH/FRI rate rather than trusting the proposer's WEI fields.

### Proof of Concept

1. Proposer is selected for height N.
2. Oracle reports `l1_gas_price_wei = 1000 gwei`, `l1_gas_price_fri = 1000 fri` (ETH/FRI rate = 1).
3. Proposer constructs `ProposalInit` with `l1_gas_price_fri = 1000 fri` (within margin, passes validation) but `l1_gas_price_wei = 1 wei` (1000× lower, not validated).
4. `calculate_eth_to_fri_rate` computes rate = `1000 fri / 1 wei * WEI_PER_ETH` = 1000× the true rate.
5. `l2_gas_price_wei = l2_gas_price_fri / (1000 * true_rate)` = 1000× lower than correct.
6. All validators accept the `ProposalInit` (FRI check passes).
7. Blockifier executes deprecated transactions with `l2_gas_price_wei` 1000× too low.
8. Users paying ETH fees are charged 1000× less than the protocol intends.
9. The block hash commits to these wrong WEI prices; the committed state is permanently wrong. [8](#0-7) [9](#0-8)

```rust
pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let previous_block_info = PreviousBlockInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}
