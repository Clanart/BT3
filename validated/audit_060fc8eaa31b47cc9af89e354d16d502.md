### Title
`ProofOfInclusion::valid()` Contains a Tautological Root-Hash Check, Failing to Validate Against an External Tree Root - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` performs only internal self-consistency checks on the proof's layer chain. Its final assertion `existing_hash == self.root_hash()` is a mathematical tautology — it always evaluates to `true` when the loop completes — because `root_hash()` returns `last.combined_hash`, which is the same value `existing_hash` holds at loop exit. No external, trusted tree root is ever compared. Any caller that relies solely on `valid()` to accept or reject a deserialized `ProofOfInclusion` can be deceived by a crafted proof whose internal hashes are consistent but whose embedded root does not match the actual DataLayer tree root.

### Finding Description
`ProofOfInclusion::valid()` iterates over `self.layers`, recomputing each `combined_hash` from the running hash and `other_hash`, and returns `false` on any mismatch. After the loop it executes:

```rust
existing_hash == self.root_hash()
```

`root_hash()` is defined as:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← always equals existing_hash after the loop
    } else {
        self.node_hash
    }
}
```

Because the loop already enforced `calculated_hash == layer.combined_hash` at every step, `existing_hash` at loop exit is exactly `last.combined_hash`. The final comparison is therefore `last.combined_hash == last.combined_hash` — unconditionally `true`. [1](#0-0) 

`valid()` accepts no external root parameter and never compares against a trusted source of truth. [2](#0-1) 

`ProofOfInclusion` is a `Streamable` type, fully deserializable from raw bytes, and its `valid()` method is directly exposed as a Python binding (`py_valid`). [3](#0-2) 

The Python stub confirms the public API surface: `valid()` and `root_hash()` are separate, independent methods with no coupling enforced by the type system. [4](#0-3) 

### Impact Explanation
An attacker can craft a `ProofOfInclusion` byte string where:
- `node_hash` is set to the hash of any key-value pair they wish to falsely prove is included.
- `layers` contains an internally consistent chain (each `combined_hash` correctly computed from the previous hash and a chosen `other_hash`), anchoring to an attacker-chosen root.

`valid()` returns `true` for this proof. Any consumer that calls `proof.valid()` without separately asserting `proof.root_hash() == actual_tree_root` will accept the forged inclusion proof. This lets untrusted input prove invalid DataLayer state — a forged proof of inclusion for a key-value pair that does not exist in the tree.

This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, or lets untrusted input prove invalid state.**

### Likelihood Explanation
`ProofOfInclusion` is a `Streamable` type deserializable from network bytes. The Python binding exposes `valid()` as a standalone method with no documentation warning that `root_hash()` must be separately checked against a trusted root. The fuzz target and integration tests only call `valid()` on proofs generated from the same in-memory blob, never on externally supplied proofs, so the tautology has not been caught by existing tests. [5](#0-4) 

Any downstream caller (e.g., chia-blockchain Python code) that treats `proof.valid() == True` as sufficient authorization to accept a DataLayer state claim is exploitable.

### Recommendation
`valid()` must accept an external trusted root hash and compare against it instead of the self-referential `self.root_hash()`:

```rust
pub fn valid(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root   // compare against external trusted root
}
```

All call sites — including the Python binding `py_valid` — must be updated to supply the actual tree root obtained from a trusted source (e.g., the on-chain committed root hash).

### Proof of Concept

```rust
use chia_datalayer::{Hash, Side, proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer}};

fn forge_proof() {
    // Attacker-chosen leaf hash (key-value pair NOT in the real tree)
    let fake_node_hash: Hash = [0xAA; 32];
    let fake_sibling:   Hash = [0xBB; 32];

    // Compute a valid combined_hash so the loop passes
    let combined = chia_datalayer::calculate_internal_hash(
        &fake_node_hash, Side::Left, &fake_sibling,
    );

    let forged = ProofOfInclusion {
        node_hash: fake_node_hash,
        layers: vec![ProofOfInclusionLayer {
            other_hash_side: Side::Left,
            other_hash: fake_sibling,
            combined_hash: combined,
        }],
    };

    // valid() returns true; forged.root_hash() == combined (attacker-controlled)
    assert!(forged.valid());   // ← passes, proof is forged
}
```

The tautological check `existing_hash == self.root_hash()` at line 57 always passes, so `valid()` returns `true` for any internally consistent but externally unanchored proof. [1](#0-0)
