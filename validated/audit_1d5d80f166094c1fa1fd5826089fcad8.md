### Title
Stale `feemarketParams` Snapshot in Cosmos Ante Handler Allows Fee-Market Bypass for Non-EVM Transactions - (File: `evmd/ante/handler_options.go`)

### Summary

`newCosmosAnteHandler` captures `feemarketParams` (including `BaseFee`) as a one-time snapshot at handler-creation time and passes a pointer to that snapshot into `NewDynamicFeeChecker`. Every subsequent Cosmos (non-EVM) transaction is fee-validated against this frozen base fee, not the live on-chain value that is recalculated every block. The EVM ante handler reads fresh params on every transaction; the Cosmos ante handler does not. This is the direct analog of the reported "user-controlled initial value when reference state is zero/stale" class: when the reference price (base fee) is stale or was zero at genesis, user-submitted fee values that would be rejected under the current base fee are silently accepted.

### Finding Description

In `evmd/ante/handler_options.go`, `newCosmosAnteHandler` is called once (at app startup or upgrade):

```go
func newCosmosAnteHandler(ctx sdk.Context, options HandlerOptions, extra ...sdk.AnteDecorator) sdk.AnteHandler {
    evmParams := options.EvmKeeper.GetParams(ctx)
    feemarketParams := options.FeeMarketKeeper.GetParams(ctx)   // ← snapshot
    ...
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)  // ← pointer to snapshot
``` [1](#0-0) 

The closure returned by `NewDynamicFeeChecker` then uses this frozen `feemarketParams` for every Cosmos tx:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
...
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
if feeCap.LT(baseFeeInt) { ... }   // baseFeeInt derived from stale snapshot
``` [2](#0-1) 

By contrast, `newEthAnteHandler` reads live params on every transaction:

```go
blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
feemarketParams := &blockCfg.FeeMarketParams
baseFee := blockCfg.BaseFee
``` [3](#0-2) 

`CalculateBaseFee` updates `BaseFee` in the persistent store every block via `BeginBlock`. The Cosmos ante handler never re-reads this updated value; it always compares against the base fee that existed when the handler was first constructed. [4](#0-3) 

### Impact Explanation

A user submitting a Cosmos SDK transaction (e.g., `MsgSend`, `MsgDelegate`, or any non-EVM message) can set `fee = stale_baseFee * gas` and have it accepted even when the live base fee is significantly higher. The effective fee deducted from the sender is therefore lower than what the current EIP-1559 fee market requires. This is a fee-market ante handler bug that permits transactions with insufficient fees to commit, mis-accounting user fee obligations and undermining the fee-market mechanism for Cosmos-native messages.

**Allowed impact matched**: *High — fee market / ante handler bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted.*

### Likelihood Explanation

The base fee rises whenever blocks are above the gas target (the common case under load). Any period of sustained chain activity causes the live base fee to diverge upward from the genesis/upgrade-time snapshot. An attacker simply needs to submit a Cosmos tx with `fee = snapshot_baseFee * gas` after the live base fee has risen. No special privileges, keys, or network position are required — only a standard unprivileged transaction submission.

### Recommendation

Read `feemarketParams` (and derive `baseFee`) from the live store inside the per-transaction closure, mirroring the pattern already used in `newEthAnteHandler`:

```go
// Inside NewDynamicFeeChecker closure, replace the snapshot lookup with:
feemarketParams := feemarketKeeper.GetParams(ctx)
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, &feemarketParams)
```

Alternatively, pass a `FeeMarketKeeper` reference into `NewDynamicFeeChecker` so it can call `GetParams(ctx)` on every invocation, exactly as `newEthAnteHandler` does via `EVMBlockConfig`.

### Proof of Concept

1. Chain starts with `BaseFee = 1_000_000_000` (1 Gwei). `newCosmosAnteHandler` is called; `feemarketParams.BaseFee = 1_000_000_000` is frozen in the closure.
2. Over subsequent blocks, high utilization causes `CalculateBaseFee` to raise the live `BaseFee` to `2_000_000_000` (2 Gwei) and store it via `SetBaseFee`.
3. Attacker submits a `MsgSend` Cosmos tx with `fee = 1_000_000_000 * gasLimit` (1 Gwei × gas).
4. `NewDynamicFeeChecker` computes `baseFeeInt = 1_000_000_000` (from the stale snapshot), `feeCap = fee / gas = 1_000_000_000`, and the check `feeCap.LT(baseFeeInt)` → `1e9 < 1e9` → **false** → tx passes.
5. The tx is included in the block paying only half the required base fee. The EVM ante handler for a parallel EVM tx would have correctly rejected the same gas price against the live `2_000_000_000` base fee. [5](#0-4) [6](#0-5)

func (k Keeper) CalculateBaseFee(ctx sdk.Context) *big.Int {
	params := k.GetParams(ctx)

	// Ignore the calculation if not enabled
	if !params.IsBaseFeeEnabled(ctx.BlockHeight()) {
		return nil
	}
	consParams := ctx.ConsensusParams()

	// If the current block is the first EIP-1559 block, return the base fee
	// defined in the parameters (DefaultBaseFee if it hasn't been changed by
	// governance).
	if ctx.BlockHeight() == params.EnableHeight {
		return params.BaseFee.BigInt()
	}
