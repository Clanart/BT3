### Title
Missing Declared-Class Validation in `execute_replace_class` Enables Permanent Contract Freezing — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the replacement class hash corresponds to a previously declared contract class. The OS itself acknowledges this missing check via a `TODO` comment. Any unprivileged contract can invoke `replace_class` with an arbitrary, undeclared class hash, permanently replacing its own class pointer with one that has no corresponding compiled class. All subsequent calls to that contract will fail to resolve the class and revert, permanently freezing any funds held by the contract.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall. After deducting gas, it reads the current `StateEntry` for the calling contract and writes a new `StateEntry` with the caller-supplied `class_hash` directly into `contract_state_changes`:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The `class_hash` value comes directly from `request.class_hash`, which is attacker-controlled calldata. There is **no assertion** that `class_hash` exists in the `compiled_class_facts` bundle loaded at OS startup. The `TODO` comment at line 898 explicitly acknowledges this missing check.

This is structurally identical to the original report's pattern: the OS *inherits* the responsibility to validate class declarations (the compiled class facts bundle is loaded and validated post-execution in `validate_compiled_class_facts_post_execution`), but the `replace_class` path never consults that bundle, leaving the critical guard permanently absent.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` in the committed state trie is set to an undeclared hash, every future call to that contract will attempt to look up the class in the OS's compiled class facts and fail. The contract becomes permanently uncallable. Any ETH, STRK, or ERC-20 tokens held in that contract's storage are irrecoverably frozen. There is no recovery path: the contract cannot call `replace_class` again because the entry point dispatch itself fails before reaching any code.

---

### Likelihood Explanation

**High.** The `replace_class` syscall is a standard, publicly accessible syscall available to any Sierra contract. No privileged role is required. A malicious contract author can:

1. Deploy a contract that accepts user deposits.
2. After accumulating funds, call `replace_class` with a fabricated, undeclared hash (e.g., `felt(1)`).
3. The OS commits the invalid class hash to state.
4. All deposited funds are permanently frozen.

This is a single-transaction, zero-privilege attack requiring only the ability to deploy and call a contract — the most basic unprivileged capability on StarkNet.

---

### Recommendation

Before writing the new `StateEntry`, verify that `class_hash` is present in the `compiled_class_facts` bundle available in the OS global context. The `validate_compiled_class_facts_post_execution` function already iterates over declared facts; a lookup helper should be extracted and called inline within `execute_replace_class`:

```cairo
// Verify the new class hash is a declared compiled class.
assert_class_hash_is_declared(
    class_hash=class_hash,
    compiled_class_facts=os_global_context.compiled_class_facts_bundle,
);
```

This mirrors how `execute_deploy` already requires a `class_hash` that was declared before the block (the class must be in the compiled class facts to be deployable).

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

}
