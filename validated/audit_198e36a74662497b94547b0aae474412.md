### Title
Missing Declared-Class Existence Check in SNOS `execute_replace_class` Allows Wrong Class Hash to Be Committed to State Root — (`crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The Starknet OS (SNOS) Cairo program's `execute_replace_class` function does not verify that the new class hash supplied to the `replace_class` syscall is actually declared before writing it into the contract state dict. The blockifier enforces this check and reverts the transaction if the class is undeclared. The SNOS does not, creating a proof-system invariant gap: a valid STARK proof can be generated for a state transition where a contract's class hash is set to an undeclared (non-existent) value, producing a wrong global state root.

---

### Impact Explanation

The SNOS is the Cairo program executed inside the STARK prover to produce the validity proof for each Starknet block. It is supposed to independently enforce every state-transition invariant so that the proof itself is the guarantee of correctness. When the SNOS skips the declared-class check:

1. A `replace_class` syscall carrying an undeclared (or arbitrary) class hash passes through `execute_replace_class` without error.
2. The contract's `class_hash` field in `contract_state_changes` is updated to the undeclared value.
3. This corrupted entry propagates into the Patricia Merkle Tree commitment, producing a wrong global state root.
4. The STARK proof over this execution is valid (no constraint was violated inside the OS), so the wrong state root is accepted as proven.

Downstream effects: any subsequent call to that contract will attempt to load a class that does not exist, causing execution failures or — if a class with that hash is later declared — silently executing unintended code. The wrong class hash also appears in the state diff committed to L1, corrupting the L1-verifiable state.

**Impact category:** Critical — wrong class hash / wrong state root from OS/syscall execution logic.

---

### Likelihood Explanation

The blockifier always rejects `replace_class` with an undeclared hash, so in the honest-sequencer path the SNOS never receives such a syscall. However:

- The SNOS hint system (`GetContractAddressStateEntry`, etc.) is populated by the sequencer. A compromised or malicious sequencer can supply hints that make the SNOS process a `replace_class` with an undeclared hash as if it succeeded.
- Because the SNOS has no independent check, it cannot detect the lie and will produce a valid proof for the wrong state.
- The TODO comment (`// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.`) confirms the developers are aware the check is absent and intend to add it, meaning the gap is known and unmitigated.

---

### Recommendation

**Short term:** Add a declared-class existence assertion inside `execute_replace_class` in both OS Cairo files, mirroring the blockifier check. In the new-syscall path (`syscall_impls.cairo`), look up the class hash in `contract_class_changes` (or the equivalent OS-level declared-class dict) and assert it is present before calling `dict_update`. In the deprecated path (`deprecated_execute_syscalls.cairo`), apply the same guard.

**Long term:** Establish a systematic cross-check between blockifier syscall handlers and their SNOS counterparts to ensure every validation performed by the blockifier is also enforced as a Cairo constraint in the OS, so the proof system is the authoritative enforcer of all invariants rather than relying on the sequencer's honesty.

---

### Proof of Concept

1. Deploy a Cairo 1 contract `C` that calls `replace_class(0xdeadbeef)` where `0xdeadbeef` is not declared.
2. Submit an invoke transaction targeting `C.__execute__`.
3. **Blockifier path:** `get_compiled_class(0xdeadbeef)` returns `StateError::UndeclaredClassHash`; the transaction is reverted; the state diff records no class-hash change for `C`. [1](#0-0) 
4. **Crafted SNOS hint path:** Supply hints that present the `replace_class` syscall as having succeeded (failure_flag = 0). The SNOS `execute_replace_class` reads `class_hash = 0xdeadbeef` from the request, skips the existence check (line 898 TODO), and writes `new StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes`. [5](#0-4) 
5. The Patricia Merkle Tree commitment is computed over the corrupted `contract_state_changes` dict, yielding a state root where `C` maps to class hash `0xdeadbeef`.
6. The STARK proof is valid; the wrong state root is submitted to L1 as proven.
7. Any subsequent call to `C` will attempt to load class `0xdeadbeef`, which does not exist, producing wrong execution results or silent misbehavior if the hash is later claimed by a different class.
