### Title
Meta-Transaction v0 Hash Excludes Nonce, Enabling Signature Replay Across Executions - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/transaction_hash/transaction_hash.cairo`)

---

### Summary

The `execute_meta_tx_v0` syscall constructs a version-0 transaction context whose hash (`compute_meta_tx_v0_hash`) does not include any nonce or uniqueness token. Combined with the OS explicitly skipping nonce enforcement for version-0 transactions, the same caller-supplied signature over the same `(contract_address, selector, calldata, chain_id)` tuple is cryptographically indistinguishable across repeated invocations. Any contract can replay a previously-observed meta-tx signature to re-execute the target account's `__execute__` entry point an arbitrary number of times.

---

### Finding Description

`compute_meta_tx_v0_hash` delegates to `deprecated_get_transaction_hash` with `additional_data_size=0`, meaning no nonce is mixed into the hash:

```cairo
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(
    contract_address, entry_point_selector, calldata, calldata_size, chain_id
) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        tx_hash_prefix=INVOKE_HASH_PREFIX,
        version=0,
        contract_address=contract_address,
        entry_point_selector=entry_point_selector,
        calldata_size=calldata_size,
        calldata=calldata,
        max_fee=0,
        chain_id=chain_id,
        additional_data_size=0,          // ← no nonce
        additional_data=cast(0, felt*),
    );
    return tx_hash;
}
``` [1](#0-0) 

`execute_meta_tx_v0` then constructs the inner `TxInfo` with `version=0` and `nonce=0` (hardcoded), and directly calls `contract_call_helper` — bypassing both `run_validate` and `check_and_increment_nonce`:

```cairo
tempvar new_tx_info = new TxInfo(
    version=0,
    ...
    nonce=0,          // ← always zero
    transaction_hash=meta_tx_hash,
    signature_start=request.signature_start,
    ...
);
...
contract_call_helper(remaining_gas, block_context, execution_context);
``` [2](#0-1) 

The OS-level nonce guard explicitly opts out for version 0:

```cairo
func check_and_increment_nonce{...}(tx_info: TxInfo*) {
    // Do not handle nonce for version 0.
    if (tx_info.version == 0) {
        return ();
    }
    ...
}
``` [3](#0-2) 

And `run_validate` is similarly skipped:

```cairo
// Do not run "__validate__" for version 0.
if (tx_execution_info.tx_info.version == 0) {
    return ();
}
``` [4](#0-3) 

The syscall is reachable from any contract execution via the `META_TX_V0_SELECTOR` dispatch in `execute_syscalls`: [5](#0-4) 

---

### Impact Explanation

**Critical — Direct loss of funds.**

Because the meta-tx hash is a pure function of `(contract_address, selector, calldata, chain_id)` with no nonce, a signature produced once is permanently valid for that tuple. An attacker who observes a legitimate meta-tx on-chain (signature is public) can:

1. Deploy a malicious contract.
2. Call `execute_meta_tx_v0` with the captured `(contract_address, selector, calldata, signature)`.
3. The OS computes the identical `meta_tx_hash`, places the replayed signature in `TxInfo`, and invokes the target account's `__execute__` entry point.
4. If `__execute__` verifies the signature against `tx_info.transaction_hash` (the standard pattern for account contracts that support meta-tx), the check passes — the hash is identical.
5. Any token transfer or state mutation inside `__execute__` executes again.
6. Repeat until the target account is drained.

---

### Likelihood Explanation

**Medium.** The attacker must:
- Be an unprivileged transaction sender who can deploy a contract (no special privilege required).
- Observe a prior meta-tx on-chain (all data is public).
- Target an account whose `__execute__` implementation verifies the signature against the OS-provided `transaction_hash` without maintaining its own used-hash registry.

Account contracts designed to accept meta-tx calls are precisely the ones that perform this signature check, making them the natural target population.

---

### Recommendation

Include a caller-supplied or OS-tracked nonce in `compute_meta_tx_v0_hash` so that each meta-tx produces a unique hash:

```cairo
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(
    contract_address, entry_point_selector, calldata, calldata_size, chain_id,
    nonce: felt,   // ← add nonce
) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        ...
        additional_data_size=1,
        additional_data=&nonce,
    );
    return tx_hash;
}
```

Alternatively, maintain a boolean mapping of consumed meta-tx hashes in the OS state (analogous to the Connext fix in PR 1626) and assert `prev_value=0` before marking a hash as used.

---

### Proof of Concept

1. Alice's account contract supports meta-tx: its `__execute__` verifies `get_tx_info().transaction_hash` against a provided signature and then calls `transfer(bob, 100_tokens)`.
2. A relayer submits a legitimate meta-tx. The OS computes `H = hash('invoke', 0, alice, '__execute__', calldata, 0, chain_id)`. Alice's `__execute__` verifies `sig` over `H` and transfers 100 tokens. The signature and calldata are visible on-chain.
3. Attacker deploys `ReplayContract` with:
   ```
   fn attack():
       meta_tx_v0(alice, '__execute__', calldata, captured_sig)
   ```
4. Attacker calls `ReplayContract.attack()`. The OS recomputes the identical `H` (no nonce), places `captured_sig` in `TxInfo`, and calls Alice's `__execute__`. The signature check passes. 100 tokens transfer again.
5. Attacker repeats until Alice's balance is zero.

No privileged access, leaked key, or operator cooperation is required. The only inputs are a deployed contract and a previously-observed on-chain signature. [6](#0-5) [7](#0-6)

func execute_meta_tx_v0{
    range_check_ptr,
    syscall_ptr: felt*,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    revert_log: RevertLogEntry*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, caller_execution_context: ExecutionContext*) {
    alloc_locals;

    let request = cast(syscall_ptr + RequestHeader.SIZE, MetaTxV0Request*);
    local calldata_start: felt* = request.calldata_start;
    local calldata_size = request.calldata_end - calldata_start;

    let specific_base_gas_cost = (
        META_TX_V0_GAS_COST + META_TX_V0_CALLDATA_FACTOR_GAS_COST * calldata_size
    );
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=specific_base_gas_cost, request_struct_size=MetaTxV0Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    local contract_address = request.contract_address;
    local selector = request.selector;
    local caller_execution_info: ExecutionInfo* = caller_execution_context.execution_info;
    local old_tx_info: TxInfo* = caller_execution_info.tx_info;

    if (selector != EXECUTE_ENTRY_POINT_SELECTOR) {
        write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
        return ();
    }

    // Sanity check: Verify that `signature` is a valid Sierra array.
    assert_nn_le(request.signature_end - request.signature_start, SIERRA_ARRAY_LEN_BOUND - 1);

    let (state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
        key=contract_address
    );

    // Compute the meta-transaction hash.
    let pedersen_ptr = builtin_ptrs.selectable.pedersen;
    with pedersen_ptr {
        let meta_tx_hash = compute_meta_tx_v0_hash(
            contract_address=contract_address,
            entry_point_selector=selector,
            calldata=calldata_start,
            calldata_size=calldata_size,
            chain_id=old_tx_info.chain_id,
        );
    }
    update_pedersen_in_builtin_ptrs(pedersen_ptr=pedersen_ptr);

    // Prepare execution context.
    tempvar new_tx_info = new TxInfo(
        version=0,
        account_contract_address=contract_address,
        max_fee=0,
        signature_start=request.signature_start,
        signature_end=request.signature_end,
        transaction_hash=meta_tx_hash,
        chain_id=old_tx_info.chain_id,
        nonce=0,
        resource_bounds_start=cast(0, ResourceBounds*),
        resource_bounds_end=cast(0, ResourceBounds*),
        tip=0,
        paymaster_data_start=cast(0, felt*),
        paymaster_data_end=cast(0, felt*),
        nonce_data_availability_mode=0,
        fee_data_availability_mode=0,
        account_deployment_data_start=cast(0, felt*),
        account_deployment_data_end=cast(0, felt*),
        proof_facts_start=cast(0, felt*),
        proof_facts_end=cast(0, felt*),
    );

    let (deprecated_tx_info_ptr: DeprecatedTxInfo*) = alloc();
    tempvar execution_context: ExecutionContext* = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=state_entry.class_hash,
        calldata_size=calldata_size,
        calldata=calldata_start,
        execution_info=new ExecutionInfo(
            block_info=caller_execution_info.block_info,
            tx_info=new_tx_info,
            caller_address=ORIGIN_ADDRESS,
            contract_address=contract_address,
            selector=selector,
        ),
        deprecated_tx_info=deprecated_tx_info_ptr,
    );
    fill_deprecated_tx_info(tx_info=new_tx_info, dst=execution_context.deprecated_tx_info);

    // Since we process the revert log backwards, entries before this point belong to the calling
    // contract.
    assert [revert_log] = RevertLogEntry(
        selector=CHANGE_CONTRACT_ENTRY, value=caller_execution_info.contract_address
    );
    let revert_log = &revert_log[1];

    contract_call_helper(
        remaining_gas=remaining_gas,
        block_context=block_context,
        execution_context=execution_context,
    );

    // Entries before this point belong to the callee.
    assert [revert_log] = RevertLogEntry(selector=CHANGE_CONTRACT_ENTRY, value=contract_address);
    let revert_log = &revert_log[1];

    return ();
