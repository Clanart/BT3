### Title
Unvalidated Class Hash in `execute_replace_class` Allows Permanent Freezing of Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not validate that the supplied class hash corresponds to a declared contract class. Any contract can replace its own class with an arbitrary felt value (including `0` or a random non-existent hash), rendering itself permanently uncallable. Funds held by or owed to such a contract are permanently frozen, directly analogous to the external report's pattern where a recipient closes their account to prevent repayment.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the new class hash directly from the syscall request and writes it into `contract_state_changes` without any check that the hash corresponds to a declared class:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-acknowledged TODO at line 898 confirms the missing validation. After this update, any subsequent call to the contract will attempt to dispatch against a class hash that has no corresponding compiled class, causing every call to fail unconditionally. [1](#0-0) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to a non-existent value, no entry point can be dispatched on it. Any ERC-20 tokens, collateral, or other assets held in that contract's storage become permanently inaccessible. Because the OS state transition is proven and finalized on-chain, there is no recovery path: the state root commits to the invalid class hash, and no future transaction can call into the contract to withdraw funds.

---

### Likelihood Explanation

The `replace_class` syscall is a standard, permissionless operation available to any deployed contract. A malicious contract author needs only to:

1. Deploy a contract that accepts deposits from other users (e.g., a lending pool, vault, or escrow).
2. After accumulating user funds, invoke `replace_class` with `class_hash = 0` (or any undeclared felt).
3. The contract is immediately and permanently uncallable; all deposited funds are frozen.

No privileged role, leaked key, or external dependency is required. The attacker is a standard contract deployer — an explicitly listed unprivileged entry point in the bounty scope.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that it exists in `contract_class_changes` (i.e., it has been declared in the current or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` for the supplied `class_hash` and assert the returned compiled class hash is non-zero:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the existing guard in `execute_declare_transaction` that enforces `assert_not_zero(compiled_class_hash)` before writing a class declaration. [2](#0-1) 

---

### Proof of Concept

1. **Attacker deploys a lending/vault contract** (`MaliciousVault`) that accepts ERC-20 deposits from users and exposes a `withdraw` entry point.
2. **Users deposit funds** into `MaliciousVault`. The contract's storage now holds balances.
3. **Attacker calls `replace_class`** from within `MaliciousVault`, supplying `class_hash = 1` (an undeclared felt). The OS executes `execute_replace_class`, which writes `class_hash=1` into `contract_state_changes` for `MaliciousVault`'s address with no validation. [3](#0-2) 
4. **State is committed.** The proven state root now records `MaliciousVault.class_hash = 1`.
5. **Users attempt to call `withdraw`.** The OS reads `class_hash=1` from state, finds no compiled class for it, and every call fails unconditionally.
6. **Funds are permanently frozen.** No entry point can ever be dispatched on `MaliciousVault` again; the balances in its storage are irrecoverable.

The `charge_fee` path in `transaction_impls.cairo` further illustrates the systemic risk: if the `fee_token_address` contract were subject to this attack, every subsequent fee transfer would fail, halting the network. [4](#0-3)

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
