### Title
`batch_insert` Bypasses Key/Hash Uniqueness Checks, Enabling DataLayer Merkle Tree Root Corruption — (File: `crates/chia-datalayer/src/merkle/blob.rs`)

### Summary

`MerkleBlob::batch_insert` contains a fast-path code branch that writes leaf nodes directly to the blob via `insert_entry_to_blob` without performing the key-uniqueness or hash-uniqueness checks that `insert` enforces. An unprivileged caller supplying a batch with duplicate `KeyId` or `Hash` values (≥3 items when the tree has ≤1 existing leaves) can silently insert duplicate leaves, corrupting the tree root and making DataLayer Merkle proofs unreliable.

### Finding Description

`MerkleBlob::insert` guards against duplicate insertions with two explicit checks:

```rust
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`batch_insert` uses `pop()` to pull the **last two** items from the input vector and routes them through the guarded `insert` path:

```rust
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else { ... };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;
    }
}
``` [2](#0-1) 

All **remaining items** (the first N−2 elements of the input) are then written directly via `insert_entry_to_blob` with **no uniqueness check**:

```rust
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ... Node::Leaf(LeafNode { hash, key, value, ... }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
``` [3](#0-2) 

If the batch contains duplicate `KeyId` or `Hash` values among those first N−2 items, both copies are written to the blob. The `block_status_cache` (a `HashMap`) will only retain the last-written index for a given key, leaving the earlier duplicate leaf as an orphaned node that is part of the tree structure but invisible to the cache. This causes:

1. `check_integrity` to fail (`leaf_count != key_to_index_cache_length`). [4](#0-3) 
2. The computed tree root to reflect a tree with duplicate leaves — a root that does not correspond to any valid key-value mapping.
3. `get_proof_of_inclusion` to generate a proof anchored to the corrupted root. [5](#0-4) 

The Python binding `py_batch_insert` exposes this path directly to callers:

```rust
#[pyo3(name = "batch_insert")]
pub fn py_batch_insert(&mut self, keys_values: Vec<(KeyId, ValueId)>, hashes: Vec<Hash>) -> PyResult<()> {
    ...
    self.batch_insert(zip(keys_values, hashes).collect())?;
    Ok(())
}
``` [6](#0-5) 

### Impact Explanation

A corrupted tree root means the DataLayer's committed state is inconsistent: the root hash no longer faithfully represents the set of key-value pairs. `ProofOfInclusion::valid()` only checks internal hash chaining and does not verify against an external expected root:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(...);
        if calculated_hash != layer.combined_hash { return false; }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()
}
``` [7](#0-6) 

A proof generated from the corrupted tree will pass `valid()` while being anchored to a wrong root, enabling forged inclusion proofs against the corrupted committed state. This matches the **High** allowed impact: *DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.*

### Likelihood Explanation

The Python binding is directly callable. Any DataLayer workflow that calls `batch_insert` with ≥3 items — where the caller or an upstream data source supplies duplicate keys — triggers the bug silently (no error is returned). The condition `leaf_count <= 1` is satisfied at tree initialization, which is the common case for a fresh DataLayer store.

### Recommendation

Add the same key and hash uniqueness checks to the bulk path in `batch_insert` before calling `insert_entry_to_blob`. Either:

- Pre-validate the entire input vector for duplicates (against each other and against the existing cache) before any writes, or
- Route all items through the guarded `insert` path (accepting the performance cost), or
- Maintain a local `HashSet` of keys and hashes seen within the current batch and reject duplicates before writing.

### Proof of Concept

```python
from chia_rs.datalayer import MerkleBlob, KeyId, ValueId
import hashlib

def h(n):
    return hashlib.sha256(n.to_bytes(8, 'big')).digest()

blob = MerkleBlob(blob=bytearray())

# Batch of 4 items: first 2 (indices 0,1) bypass uniqueness checks;
# last 2 (indices 2,3) go through insert() with checks.
# Items 0 and 1 share the same KeyId — duplicate silently accepted.
dup_key = KeyId(42)
keys_values = [
    (dup_key,   ValueId(1)),   # item 0 — bypasses check
    (dup_key,   ValueId(2)),   # item 1 — bypasses check (duplicate key!)
    (KeyId(10), ValueId(3)),   # item 2 — goes through insert()
    (KeyId(11), ValueId(4)),   # item 3 — goes through insert()
]
hashes = [h(0), h(1), h(2), h(3)]

blob.batch_insert(keys_values, hashes)
blob.calculate_lazy_hashes()

# Tree root is now computed from a structure with two leaves sharing KeyId(42).
# check_integrity() will fail; proofs are anchored to a corrupted root.
print("root:", blob.get_root().hex())
# check_integrity raises — leaf_count != key_to_index_cache_length
blob.check_integrity()  # expected to raise

    pub fn batch_insert(
        &mut self,
        mut keys_values_hashes: Vec<((KeyId, ValueId), Hash)>,
    ) -> Result<(), Error> {
        // OPT: perhaps go back to taking an iterator?
        // OPT: would it be worthwhile to hold the entire blocks?
        let mut indexes = vec![];

        if self.block_status_cache.leaf_count() <= 1 {
            for _ in 0..2 {
                let Some(((key, value), hash)) = keys_values_hashes.pop() else {
                    return Ok(());
                };
                self.insert(key, value, &hash, InsertLocation::Auto {})?;
            }
        }

        for ((key, value), hash) in keys_values_hashes {
            let new_leaf_index = self.get_new_index();
            let new_block = Block {
                metadata: NodeMetadata {
                    node_type: NodeType::Leaf,
                    dirty: false,
                },
                node: Node::Leaf(LeafNode {
                    parent: Parent(None),
                    hash,
                    key,
                    value,
                }),
            };
            self.insert_entry_to_blob(new_leaf_index, &new_block)?;
            indexes.push(new_leaf_index);
        }

        // OPT: can we insert the top node first?  maybe more efficient to update it's children
        //      than to update the parents of the children when traversing leaf to sub-root?
        while indexes.len() > 1 {
            let mut new_indexes = vec![];

            for chunk in indexes.chunks(2) {
                let [index_1, index_2] = match chunk {
                    [index] => {
                        new_indexes.push(*index);
                        continue;
                    }
                    [index_1, index_2] => [*index_1, *index_2],
                    _ => unreachable!(
                        "chunk should always be either one or two long and be handled above"
                    ),
                };

                let new_internal_node_index = self.get_new_index();

                let mut hashes = vec![];
                for index in [index_1, index_2] {
                    let block = self.update_parent(index, Some(new_internal_node_index))?;
                    hashes.push(block.node.hash());
                }

                let new_block = Block {
                    metadata: NodeMetadata {
                        node_type: NodeType::Internal,
                        dirty: false,
                    },
                    node: Node::Internal(InternalNode {
                        parent: Parent(None),
                        hash: internal_hash(&hashes[0], &hashes[1]),
                        left: index_1,
                        right: index_2,
                    }),
                };

                self.insert_entry_to_blob(new_internal_node_index, &new_block)?;
                new_indexes.push(new_internal_node_index);
            }

            indexes = new_indexes;
        }

        if indexes.len() == 1 {
            // OPT: can we avoid this extra min height leaf traversal?
            let min_height_leaf = self.get_min_height_leaf()?;
            self.insert_subtree_at_key(min_height_leaf.key, indexes[0], Side::Left)?;
        }

        Ok(())
    }

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
