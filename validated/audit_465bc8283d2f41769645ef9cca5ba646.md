### Title
Missing Validation of Class Hash in `execute_replace_class` Allows Permanent Contract Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS Cairo program accepts an arbitrary `class_hash` from the caller without verifying that the hash corresponds to a previously declared contract class. This is structurally identical to M-11: a privileged setter (here, any contract invoking `replace_class`) can supply an unchecked parameter that permanently corrupts protocol state. A contract that calls `replace_class` with an undeclared class hash will have its on-chain class hash permanently set to an invalid value, making the contract permanently non-executable and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` reads the requested class hash directly from the syscall request and writes it into `contract_state_changes` without any check that the hash exists in `contract_class_changes` (the set of declared classes):

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The developer-acknowledged TODO at line 898 explicitly confirms the missing check. The `class_hash` value is fully attacker-controlled (it comes from the contract's own syscall request) and is committed to state with no validation.

Contrast this with `execute_declare_transaction`, which enforces that a class hash is the result of a valid Sierra class hash computation before writing it to `contract_class_changes`:

```cairo
let expected_class_hash = finalize_class_hash(...);
with_attr error_message("Invalid class hash pre-image.") {
    assert [class_hash_ptr] = expected_class_hash;
}
...
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [2](#0-1) [3](#0-2) 

`replace_class` bypasses this gate entirely.

When a future transaction calls the affected contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
``` [4](#0-3) 

If `class_hash` was never declared, `dict_read` returns 0 and `find_element` fails to locate a compiled class fact, making the contract permanently non-executable. The state mutation from `replace_class` is already committed and irreversible.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any contract that calls `replace_class` with an undeclared class hash (whether by attacker design, contract bug, or user error) will have its class hash permanently corrupted in the global state. Because the class hash is stored in `contract_state_changes` and committed to the state root, and because there is no recovery mechanism, all assets held by that contract (tokens, NFTs, etc.) become permanently inaccessible. The contract address is effectively bricked.

---

### Likelihood Explanation

The `replace_class` syscall is a standard, publicly documented StarkNet feature available to any deployed contract. No privileged role is required — any unprivileged user can deploy a contract and invoke `replace_class` from within it. Realistic trigger paths include:

1. A contract with an upgrade mechanism that does not independently validate the new class hash before calling `replace_class`.
2. A user or developer accidentally supplying a wrong hash (e.g., a Sierra class hash instead of the compiled class hash, or a hash from a different chain).
3. A malicious actor deliberately supplying an invalid hash to grief a contract they control (e.g., a shared multisig or escrow).

The missing check is confirmed by the in-code TODO comment, indicating the developers are aware of the gap.

---

### Recommendation

Before writing the new class hash to `contract_state_changes`, verify that it exists in `contract_class_changes` (i.e., it has been declared). Concretely, after reading `class_hash` from the request, perform a `dict_read` on `contract_class_changes` and assert the result is non-zero:

```cairo
let class_hash = request.class_hash;

// Validate that the class hash has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the protection already present in `execute_declare_transaction` and closes the gap identified by the TODO comment.

---

### Proof of Concept

1. Deploy contract `A` that exposes a public function `freeze()` which calls the `replace_class` syscall with `class_hash = 0xdeadbeef` (an undeclared hash).
2. Deposit funds (e.g., ERC-20 tokens) into contract `A`.
3. Send an invoke transaction calling `A.freeze()`.
4. The OS processes `execute_replace_class`: `class_hash = 0xdeadbeef` is written to `contract_state_changes[A]` with no validation. The transaction succeeds.
5. Send any subsequent transaction calling any function on contract `A`.
6. `execute_entry_point` reads `class_hash = 0xdeadbeef` from state, calls `dict_read` on `contract_class_changes` (returns 0 — never declared), then `find_element` fails to locate a compiled class fact.
7. Contract `A` is permanently non-executable. All funds are frozen with no recovery path. [5](#0-4) [6](#0-5)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
