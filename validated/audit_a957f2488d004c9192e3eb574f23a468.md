### Title
`deprecated_declared_classes` Silently Dropped in `CommitmentStateDiff` → `ThinStateDiff` Conversion Produces Wrong `state_diff_commitment` and Block Hash - (`crates/blockifier/src/state/cached_state.rs`)

### Summary

The `From<CommitmentStateDiff> for ThinStateDiff` conversion hardcodes `deprecated_declared_classes: Vec::new()`, silently discarding any Cairo 0 (deprecated) class declarations from the block's state diff. Because `calculate_state_diff_hash` and `ThinStateDiff::len()` both include `deprecated_declared_classes` in their computations, any block that contains a Cairo 0 declare transaction will have a wrong `state_diff_commitment` and wrong `state_diff_length` committed into its block header, and therefore a wrong block hash.

### Finding Description

`CommitmentStateDiff` in `crates/blockifier/src/state/cached_state.rs` has no `deprecated_declared_classes` field:

```rust
pub struct CommitmentStateDiff {
    pub address_to_class_hash: IndexMap<ContractAddress, ClassHash>,
    pub address_to_nonce: IndexMap<ContractAddress, Nonce>,
    pub storage_updates: IndexMap<ContractAddress, IndexMap<StorageKey, Felt>>,
    pub class_hash_to_compiled_class_hash: IndexMap<ClassHash, CompiledClassHash>,
}
``` [1](#0-0) 

When the batcher finalizes a block, `finalize_block` calls `block_state.to_state_diff()?.state_maps` and converts the result to `CommitmentStateDiff`. Cairo 1 declared classes are captured in `class_hash_to_compiled_class_hash`, but Cairo 0 (deprecated) declared classes only appear in `StateMaps.declared_contracts` — a field that `CommitmentStateDiff` has no slot for. They are silently dropped.

The `From<CommitmentStateDiff> for ThinStateDiff` implementation then hardcodes the missing field:

```rust
impl From<CommitmentStateDiff> for ThinStateDiff {
    fn from(commitment_state_diff: CommitmentStateDiff) -> Self {
        Self {
            ...
            // TODO(AlonH): Remove this when the structure of storage diffs changes.
            deprecated_declared_classes: Vec::new(),
        }
    }
}
``` [2](#0-1) 

`BlockExecutionArtifacts::new()` calls `calculate_block_commitments` with this always-empty `ThinStateDiff`:

```rust
let (header_commitments, measurements) = calculate_block_commitments(
    &transactions_data,
    ThinStateDiff::from(commitment_state_diff.clone()),  // deprecated_declared_classes = []
    l1_da_mode,
    &block_info.starknet_version,
)
.await;
``` [3](#0-2) 

`calculate_state_diff_hash` chains `deprecated_declared_classes` into the Poseidon hash:

```rust
hash_chain = chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
``` [4](#0-3) 

`ThinStateDiff::len()` also counts `deprecated_declared_classes.len()` toward `state_diff_length`:

```rust
result += self.deprecated_declared_classes.len();
``` [5](#0-4) 

The resulting wrong `state_diff_commitment` is embedded in `PartialBlockHashComponents` and propagates into the final block hash. The same wrong `ThinStateDiff` is also stored in MDBX via `append_state_diff`, so the persisted state diff is missing the deprecated declared classes entirely. [6](#0-5) 

### Impact Explanation

Any block that includes a valid Cairo 0 declare transaction (DeclareV0/DeclareV1) will have:

1. A wrong `state_diff_commitment` in the block header (the Poseidon hash omits the declared class hashes).
2. A wrong `state_diff_length` (too small by the number of deprecated declared classes).
3. A wrong block hash derived from those wrong commitments.
4. A persisted `ThinStateDiff` in storage with `deprecated_declared_classes = []`, causing `starknet_getStateUpdate` RPC responses to omit those class declarations.

This matches **Critical: Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input** and **High: RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value**.

### Likelihood Explanation

Cairo 0 declare transactions (DeclareV0/DeclareV1) are still valid Starknet transaction types. Any unprivileged user can submit one. If the gateway admits it and the batcher includes it in a block, the corruption is triggered automatically — no special privilege or coordination is required.

### Recommendation

Add a `deprecated_declared_classes: Vec<ClassHash>` field to `CommitmentStateDiff` and populate it from `StateMaps.declared_contracts` (filtering for entries where the class hash has no corresponding compiled class hash, i.e., Cairo 0 classes). Propagate this field through `From<CommitmentStateDiff> for ThinStateDiff` instead of hardcoding `Vec::new()`. Remove the `TODO(AlonH)` comment once the fix is in place.

### Proof of Concept

1. Submit a `DeclareV1` transaction declaring a Cairo 0 contract class with hash `C`.
2. The batcher includes it in block `N`. `finalize_block` produces a `CommitmentStateDiff` with no record of `C` (it only appears in `StateMaps.declared_contracts`, which is discarded).
3. `BlockExecutionArtifacts::new()` calls `ThinStateDiff::from(commitment_state_diff)`, yielding `deprecated_declared_classes = []`.
4. `calculate_block_commitments` computes `state_diff_commitment = Poseidon("STARKNET_STATE_DIFF0", ..., 0 /*n_deprecated*/, ...)` — missing `C`.
5. The correct commitment would be `Poseidon("STARKNET_STATE_DIFF0", ..., 1, C, ...)`.
6. The block header records the wrong `state_diff_commitment` and `state_diff_length`, and the block hash is wrong.
7. A call to `starknet_getStateUpdate(block_N)` returns a state diff with `deprecated_declared_classes = []`, hiding the class declaration from all consumers. [7](#0-6) [8](#0-7) [4](#0-3)

    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =

    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
