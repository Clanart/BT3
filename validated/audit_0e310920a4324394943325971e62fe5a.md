### Title
Missing Class Hash Validation in `execute_replace_class` Enables Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary class hash without verifying it is actually declared on-chain. A contract can call `replace_class` with a non-existent class hash, permanently rendering itself uncallable and freezing all funds it holds. The OS code itself acknowledges this missing check with a TODO comment. This is the direct analog of the "Destructible" pattern: just as `destroy()` permanently removes a contract's code and causes any subsequent ETH transfers to be lost, `replace_class` with an invalid hash permanently removes a contract's ability to execute, freezing all funds within it.

---

### Finding Description

In `execute_replace_class` (`syscall_impls.cairo`, lines 877–916), after deducting gas, the function reads `class_hash` from the syscall request and unconditionally updates the contract's `StateEntry` with the new class hash. There is **no validation** that the supplied class hash corresponds to a declared contract class:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges the missing check. The OS writes the new (potentially invalid) class hash directly into the state without consulting `contract_class_changes` or any declared-class registry.

Compare this to the "Destructible" pattern: `Destructible.destroy()` could be called at any time to permanently erase a contract's code, making any ETH sent afterward lost forever. Here, `replace_class(invalid_hash)` permanently erases a contract's ability to dispatch any function call — the contract's storage and balance remain on-chain but become permanently inaccessible.

---

### Impact Explanation

After a contract replaces its class with a non-existent class hash:

1. The contract's storage (including token balances, ETH deposits, vault state) remains on-chain.
2. Every subsequent call to the contract fails at class resolution — the class does not exist.
3. No function (withdraw, transfer, rescue) can ever be invoked again.
4. All funds held in the contract are **permanently frozen** with no recovery path short of a protocol upgrade.

This satisfies **Critical — Permanent freezing of funds**.

---

### Likelihood Explanation

The attack is reachable by any unprivileged contract deployer:

- No special role, key, or operator access is required.
- `replace_class` is a standard syscall available to every contract.
- The OS provides zero protection against supplying an undeclared class hash.
- The missing check is explicitly acknowledged in the production code.

A realistic scenario: a malicious actor deploys a contract that appears to be a legitimate vault or token contract, attracts user deposits, then calls an internal function that invokes `replace_class(0)` (or any random undeclared felt). The OS executes the replacement without complaint, and all deposited funds are permanently frozen.

---

### Recommendation

Before updating the contract's `StateEntry`, validate that `class_hash` is present in `contract_class_changes` (i.e., it was declared in the current or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` for `class_hash` and assert the returned compiled class hash is non-zero (i.e., the class is declared). This mirrors the check already enforced in `execute_declare_transaction` via `assert_not_zero(compiled_class_hash)`. [2](#0-1) 

---

### Proof of Concept

1. Deploy `MaliciousVault` — a contract with:
   - `deposit()`: accepts user STRK/ETH transfers and records balances in storage.
   - `freeze()`: calls the `replace_class` syscall with class hash `0` (guaranteed undeclared).

2. Users call `deposit()` and lock funds in `MaliciousVault`.

3. Attacker calls `freeze()`. The OS dispatches `execute_replace_class`:
   - Gas is deducted (line 888–893).
   - `class_hash = 0` is read from the request (line 896).
   - The TODO check is absent (line 898).
   - `MaliciousVault`'s `StateEntry.class_hash` is set to `0` (lines 902–910).
   - The revert log records the old class hash (line 912), but the transaction is not reverted.

4. All subsequent calls to `MaliciousVault` fail: the OS looks up class hash `0`, finds no compiled class, and the call panics.

5. All user funds stored in `MaliciousVault`'s storage are permanently frozen — no withdraw, no rescue, no recovery. [3](#0-2)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

        key=contract_address,

    );

    return ();
}
