### Title
Pre-EIP-155 Legacy Transaction Cross-Chain Replay When `AllowUnprotectedTxs=true` - (`x/evm/types/tx_data.go`, `ante/interfaces/setup.go`, `x/evm/types/legacy_tx.go`)

---

### Summary

When the EVM parameter `AllowUnprotectedTxs` is `true`, an unprivileged attacker can replay any pre-EIP-155 signed Ethereum transaction (v=27 or v=28) from any other chain on the Ethermint chain. The combination of three independent gaps — `DeriveChainID` returning `big.Int(0)` for v=27/28, `LegacyTx.Validate()` performing no chain-ID check, and `EIP155Signer.Sender()` silently falling back to `HomesteadSigner` for unprotected transactions — creates a complete replay path.

---

### Finding Description

**Gap 1 — `DeriveChainID` returns zero for v=27/28, not an error** [1](#0-0) 

For v=27 or v=28, `DeriveChainID` returns `new(big.Int)` (i.e., chain ID = 0) rather than `nil` or an error. This is not treated as an invalid state anywhere downstream.

**Gap 2 — `LegacyTx.Validate()` performs no chain-ID check** [2](#0-1) 

`LegacyTx.Validate()` checks gas price, fee, amount, and `to` address, but never inspects the chain ID derived from the signature. A tx signed for Ethereum mainnet (chain ID 1) or any other chain passes `Validate()` identically to one signed for the local chain.

**Gap 3 — `ValidateEthBasic` bypasses the protection check when `AllowUnprotectedTxs=true`** [3](#0-2) 

The only ante-handler guard against unprotected transactions is this single conditional. When the governance parameter is `true`, the check is entirely skipped.

**Gap 4 — `EIP155Signer.Sender()` silently falls back to `HomesteadSigner` for unprotected txs** [4](#0-3) [5](#0-4) 

`VerifyEthSig` is called with the signer produced by `ethtypes.MakeSigner(blockCfg.ChainConfig, ...)`. Because Ethermint's default chain config has `EIP155Block=0`, this produces an `EIP155Signer`. [6](#0-5) 

In go-ethereum, `EIP155Signer.Sender()` contains the following logic (upstream go-ethereum, not Ethermint-specific):

```go
func (s EIP155Signer) Sender(tx *Transaction) (common.Address, error) {
    if !tx.Protected() {
        return HomesteadSigner{}.Sender(tx)  // silent fallback
    }
    if tx.ChainId().Cmp(s.chainId) != 0 {
        return common.Address{}, ErrInvalidChainId
    }
    ...
}
```

When `tx.Protected()` is false (v=27/28), the EIP-155 signer **does not reject the transaction** — it falls back to `HomesteadSigner`, which successfully recovers the sender from the pre-EIP-155 signature. No chain-ID mismatch error is ever raised.

`VerifySender` then compares the recovered address against `msg.From`: [7](#0-6) 

If the attacker sets `msg.From` to the victim's address (which is the same on all EVM chains for the same key), this check passes.

---

### Impact Explanation

An attacker who obtains any pre-EIP-155 signed transaction from the victim on any Ethereum-compatible network (e.g., Ethereum mainnet, a testnet, or another Ethermint chain) can submit it verbatim to an Ethermint chain with `AllowUnprotectedTxs=true`. If the victim's nonce on the target chain matches the nonce in the replayed transaction, the transaction executes: ETH/token transfers drain the victim's balance, and contract calls execute under the victim's identity without their consent.

This is a **High** impact: chain-ID replay protection bypass enabling unauthorized account mutation and fund transfer.

---

### Likelihood Explanation

**Preconditions:**
1. `AllowUnprotectedTxs` governance parameter must be `true`. This is not the default, but it is a documented, reachable configuration that operators enable to support legacy tooling.
2. The attacker must possess a valid pre-EIP-155 signed transaction from the victim. Pre-EIP-155 transactions are publicly visible on Ethereum mainnet (pre-2016 history) and on any chain that accepted them.
3. The victim's nonce on the target Ethermint chain must match the nonce in the replayed transaction. This is most trivially satisfied for nonce=0 (fresh accounts) or low nonces, which is common for newly funded accounts.

All three preconditions are realistic and independently achievable by an unprivileged attacker.

---

### Recommendation

1. **In `LegacyTx.Validate()`**: Reject transactions where `DeriveChainID(v)` returns zero (i.e., v=27 or v=28) unless `AllowUnprotectedTxs` is explicitly enabled. Even when it is enabled, consider emitting a warning or requiring an explicit opt-in per-transaction.

2. **In `ValidateEthBasic`**: When `allowUnprotectedTxs=true` and the transaction is unprotected, additionally verify that the derived chain ID is either zero (pre-EIP-155) or matches the current chain ID. Reject any unprotected tx whose derived chain ID is non-zero and does not match.

3. **Stronger guard**: Add an explicit check in the ante handler that, for unprotected legacy transactions, the `From` address is not being used to replay a transaction whose nonce has already been consumed on another chain — or simply document that `AllowUnprotectedTxs=true` carries this replay risk and should only be used in isolated/private networks.

---

### Proof of Concept

```go
// 1. On Ethereum mainnet (or any chain), victim signs a pre-EIP-155 tx:
signer := ethtypes.HomesteadSigner{}
tx := ethtypes.NewTx(&ethtypes.LegacyTx{
    Nonce:    0,
    To:       &attackerAddr,
    Value:    big.NewInt(1e18), // 1 ETH
    Gas:      21000,
    GasPrice: big.NewInt(1e9),
})
signedTx, _ := ethtypes.SignTx(tx, signer, victimKey)
// v = 27 or 28

// 2. Attacker submits the same RLP-encoded tx to Ethermint
//    (AllowUnprotectedTxs=true, victim has nonce=0 on this chain):
msgEthTx := evmtypes.NewTxFromEthereumTx(signedTx)
msgEthTx.From = victimAddr.Bytes()

// 3. Ante handler flow:
//    ValidateEthBasic: allowUnprotectedTxs=true → skip Protected() check ✓
//    VerifyEthSig: EIP155Signer.Sender() → !tx.Protected() → HomesteadSigner fallback → recovers victimAddr ✓
//    VerifySender: recovered == msg.From ✓
//    Nonce check: victim nonce=0 matches ✓
//    → Transaction commits, 1 ETH transferred to attacker

func DeriveChainID(v *big.Int) *big.Int {
	if v == nil || v.Sign() < 1 {
		return nil
	}

	if v.BitLen() <= 64 {
		v := v.Uint64()
		if v == 27 || v == 28 {
			return new(big.Int)
		}

		if v < 35 {
			return nil
		}

		// V MUST be of the form {0,1} + CHAIN_ID * 2 + 35
		return new(big.Int).SetUint64((v - 35) / 2)
	}
	v = new(big.Int).Sub(v, big.NewInt(35))
	return v.Div(v, big.NewInt(2))
}

// Validate performs a stateless validation of the tx fields.
func (tx LegacyTx) Validate() error {
	gasPrice := tx.GetGasPrice()
	if gasPrice == nil {
		return errorsmod.Wrap(ErrInvalidGasPrice, "gas price cannot be nil")
	}

	if gasPrice.Sign() == -1 {
		return errorsmod.Wrapf(ErrInvalidGasPrice, "gas price cannot be negative %s", gasPrice)
	}
	if !types.IsValidInt256(gasPrice) {
		return errorsmod.Wrap(ErrInvalidGasPrice, "out of bound")
	}
	if !types.IsValidInt256(tx.Fee()) {
		return errorsmod.Wrap(ErrInvalidGasFee, "out of bound")
	}

	amount := tx.GetValue()
	// Amount can be 0
	if amount != nil && amount.Sign() == -1 {
		return errorsmod.Wrapf(ErrInvalidAmount, "amount cannot be negative %s", amount)
	}
	if !types.IsValidInt256(amount) {
		return errorsmod.Wrap(ErrInvalidAmount, "out of bound")
	}

	if tx.To != "" {
		if err := types.ValidateAddress(tx.To); err != nil {
			return errorsmod.Wrap(err, "invalid to address")
		}
	}

	return nil
}

func VerifyEthSig(tx sdk.Tx, signer ethtypes.Signer) error {
	for _, msg := range tx.GetMsgs() {
		msgEthTx, ok := msg.(*evmtypes.MsgEthereumTx)
		if !ok {
			return errorsmod.Wrapf(errortypes.ErrUnknownRequest, "invalid message type %T, expected %T", msg, (*evmtypes.MsgEthereumTx)(nil))
		}

		if err := msgEthTx.VerifySender(signer); err != nil {
			return errorsmod.Wrapf(errortypes.ErrorInvalidSigner, "signature verification failed: %s", err.Error())
		}
	}

	return nil
}

func (msg *MsgEthereumTx) VerifySender(signer ethtypes.Signer) error {
	from, err := msg.recoverSender(signer)
	if err != nil {
		return err
	}

	if !bytes.Equal(msg.From, from.Bytes()) {
		return fmt.Errorf("sender verification failed. got %s, expected %s", HexAddress(from.Bytes()), HexAddress(msg.From))
	}
	return nil
}
