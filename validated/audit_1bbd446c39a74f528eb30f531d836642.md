### Title
Cross-Chain Homestead Transaction Replay via `allow_unprotected_txs=true` — (`evmd/ante/handler_options.go`, `ante/interfaces/setup.go`)

---

### Summary

When `AllowUnprotectedTxs=true`, `ValidateEthBasic` skips the EIP-155 protection check. The subsequent `VerifyEthSig` call uses `ethtypes.MakeSigner(blockCfg.ChainConfig, ...)`, which on a fully-forked chain returns a `PragueSigner`/`LondonSigner`. However, go-ethereum's `EIP155Signer.Sender()` (the base of all modern signers) explicitly falls back to `HomesteadSigner{}.Sender(tx)` for any unprotected transaction. This means a Homestead-signed transaction from chain A is accepted and its sender correctly recovered on chain B, breaking the invariant that only chain-B-specific signatures may commit state.

---

### Finding Description

**Step 1 — `ValidateEthBasic` gate is bypassed** [1](#0-0) 

When `allowUnprotectedTxs=true`, the entire `!tx.Protected()` rejection is skipped. A Homestead-signed legacy tx (V=27 or V=28, no chain ID encoded) passes through.

**Step 2 — Signer construction** [2](#0-1) 

`ethtypes.MakeSigner(blockCfg.ChainConfig, blockCfg.BlockNumber, blockCfg.BlockTime)` is called. With the default `DefaultChainConfig()` (all forks at block 0, including Prague/Cancun/London/Berlin/EIP155), this returns a `PragueSigner`. [3](#0-2) 

**Step 3 — The critical go-ethereum fallback**

`PragueSigner` inherits from `londonSigner → EIP2930Signer → EIP155Signer`. In go-ethereum's `EIP155Signer.Sender()`:

```go
func (s EIP155Signer) Sender(tx *Transaction) (common.Address, error) {
    if !tx.Protected() {
        return HomesteadSigner{}.Sender(tx)  // ← fallback for unprotected txs
    }
    ...
}
```

For an unprotected tx (V=27/28), the signer falls back to `HomesteadSigner{}.Sender()`, which hashes the tx **without** chain ID and recovers the sender. This succeeds for any Homestead-signed tx regardless of which chain it was originally signed for.

**Step 4 — `VerifySender` passes** [4](#0-3) 

`VerifySender` only checks that the recovered address equals `msg.From`. Since `HomesteadSigner` correctly recovers the original signer's address (the same key is used on both chains), this check passes. The tx commits.

---

### Impact Explanation

An attacker who possesses any pre-EIP155 (Homestead-signed) transaction from chain A — e.g., a historical ETH mainnet tx, a tx from a testnet, or one they crafted themselves — can submit it verbatim to chain B with `allowUnprotectedTxs=true`. If the victim's nonce on chain B matches the replayed tx's nonce, the transaction executes: ETH/token transfers, contract calls, contract deployments all commit under the victim's address without their consent. The chain-ID invariant is completely absent for this tx class.

---

### Likelihood Explanation

- `AllowUnprotectedTxs` defaults to `false` [5](#0-4)  but is a governance-settable parameter. Chains enabling it for legacy tooling compatibility are directly exposed.
- The nonce must match. For accounts with nonce=0 on chain B (new accounts), any pre-EIP155 tx with nonce=0 from chain A is immediately replayable.
- No privileged access is required once the parameter is set. Any public RPC endpoint accepting `eth_sendRawTransaction` is the entry point.

---

### Recommendation

1. **Reject unprotected txs unconditionally** regardless of `AllowUnprotectedTxs` when `EIP155Block=0` (already active). The parameter should only gate txs from before the EIP-155 fork height on the *same* chain.
2. Alternatively, when `AllowUnprotectedTxs=true`, add an explicit check in `VerifyEthSig` that the signer used to recover the address is `HomesteadSigner` only if the chain has not yet reached `EIP155Block`, rejecting unprotected txs at any block height past `EIP155Block`.
3. Document clearly that `AllowUnprotectedTxs=true` disables cross-chain replay protection.

---

### Proof of Concept

```
// Chain A (e.g., Ethereum pre-EIP155, chainID irrelevant)
privKey := <victim's secp256k1 key>
tx_A := sign(HomesteadSigner{}, LegacyTx{Nonce: 0, To: attacker, Value: 1 ETH, ...}, privKey)

// Chain B: AllowUnprotectedTxs=true, EIP155Block=0
// Victim has nonce=0 on chain B (same address, same key)
eth_sendRawTransaction(rlp_encode(tx_A))
// → ValidateEthBasic: allowUnprotectedTxs=true, tx.Protected()=false → PASS
// → MakeSigner → PragueSigner
// → PragueSigner.Sender(tx_A): tx_A.Protected()==false → HomesteadSigner{}.Sender(tx_A) → recovers victim address
// → VerifySender: recovered == msg.From → PASS
// → tx commits: 1 ETH transferred from victim to attacker on chain B
```

```go
func (msg *MsgEthereumTx) VerifySender(signer ethtypes.Signer) error {
	from, err := msg.recoverSender(signer)
	if err != nil {
		return err
	}

	if !bytes.Equal(msg.From, from.Bytes()) {
		return fmt.Errorf("sender verification failed. got %s, expected %s", HexAddress(from.Bytes()), HexAddress(msg.From))
	}
	return nil
```
