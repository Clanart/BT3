### Title
OS Proof Does Not Constrain `BlockHeaderCommitments` Against Actual Execution — Block Hash Commits to Hint-Provided Transaction/Event/Receipt/StateDiff Values - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/block_hash.cairo`)

---

### Summary

The Starknet OS Cairo program's `get_block_hashes` function accepts `header_commitments` (containing `transaction_commitment`, `event_commitment`, `receipt_commitment`, `state_diff_commitment`, and `packed_lengths`) and `gas_prices_hash` entirely from hints, with no Cairo-level constraint tying them to the transactions actually executed in the block. The sequencer correctly computes these commitments off-chain in `BlockExecutionArtifacts::new`, but the SNOS proof never enforces that the hint-provided values match the actual execution output. The only check is a hint-level sanity assertion (`CheckBlockHashConsistency`) that compares the computed hash against `os_input.new_block_hash` — but both inputs and expected output come from the same `OsBlockInput` struct, so a consistent but fabricated set of values passes trivially. The OS flow test explicitly confirms this by using `BlockHeaderCommitments::default()` (all zeros) for every block.

---

### Finding Description

**Root cause — `get_block_hashes` in `block_hash.cairo`:**

```cairo
func get_block_hashes{poseidon_ptr: PoseidonBuiltin*}(block_info: BlockInfo*, state_root: felt) -> (
    previous_block_hash: felt, new_block_hash: felt
) {
    alloc_locals;
    local previous_block_hash;
    // Currently, the header commitments and gas prices are not computed by the OS.
    // TODO(Yoni, 1/1/2027): compute the header commitments and gas prices.
    local header_commitments: BlockHeaderCommitments*;
    local gas_prices_hash;
    ...
    %{ GetBlockHashes %}          // fills all locals from hints — no Cairo constraint
    let block_hash = calculate_block_hash(...);
    %{ CheckBlockHashConsistency %} // hint-only sanity check, not a proof constraint
    return (previous_block_hash=previous_block_hash, new_block_hash=block_hash);
}
``` [1](#0-0) 

The `%{ GetBlockHashes %}` hint populates `header_commitments` from `os_input.block_hash_commitments` and `gas_prices_hash` from `block_info.gas_prices`. [2](#0-1) 

The `%{ CheckBlockHashConsistency %}` hint then checks `calculated_block_hash == os_input.new_block_hash`. [3](#0-2) 

Since `os_input.block_hash_commitments` and `os_input.new_block_hash` are both fields of the same `OsBlockInput` struct, a prover can set `block_hash_commitments` to any value, recompute `new_block_hash` to match, and the hint check passes. [4](#0-3) 

**Contrast with the off-chain path:** The sequencer correctly computes all five commitment fields from actual execution data in `calculate_block_commitments` (transaction Merkle root, event Merkle root, receipt Merkle root, state-diff Poseidon hash, packed counts), and stores them in `BlockExecutionArtifacts`. [5](#0-4) 

These commitments are then placed into `PartialBlockHashComponents` and stored in the batcher's storage for later block-hash finalization. [6](#0-5) 

None of this off-chain computation is enforced inside the Cairo proof. The OS flow test makes this explicit:

```rust
// TODO(Nimrod): Remove dummy block hashes once the OS verifies them.
let block_hash_commitments = BlockHeaderCommitments::default(); // all zeros
let new_block_hash = calculate_block_hash(
    &PartialBlockHashComponents::new(&block_info, block_hash_commitments.clone()),
    ...
).unwrap();
``` [7](#0-6) 

The block hash produced by the OS is later written to the block-hash contract storage via `WriteOldBlockToStorage`, making it the authoritative on-chain commitment to block N's content. [8](#0-7) 

---

### Impact Explanation

The block hash is the cryptographic commitment that binds a block number to its transactions, events, receipts, and state diff. Because the OS proof does not constrain `header_commitments` to match the actual executed transactions:

1. **Wrong block hash stored on-chain.** The value written to the block-hash contract (used by `get_block_hash` syscall) does not actually commit to the transactions in the block. Contracts that rely on `get_block_hash` for cross-block verification receive a value that is cryptographically meaningless with respect to block content.

2. **Authoritative-looking wrong RPC values.** `starknet_getBlockWithTxHashes`, `starknet_getTransactionReceipt`, and `starknet_getEvents` return data that clients expect to be committed to by the block hash. Since the block hash does not commit to the actual transaction/event/receipt Merkle roots, these RPC responses carry a false appearance of proof-backed integrity — matching the "High. RPC execution … returns an authoritative-looking wrong value" impact category.

3. **Fabricated transaction inclusion proofs.** Any system that constructs a Merkle proof of transaction inclusion against the block's `transaction_commitment` field can be deceived, because that field is not constrained by the proof.

---

### Likelihood Explanation

The sequencer controls the `OsBlockInput` it feeds to the SNOS. The only barrier is the hint-level `CheckBlockHashConsistency` check, which is trivially satisfied by computing `new_block_hash` from the fabricated `block_hash_commitments`. The OS flow test already demonstrates that `BlockHeaderCommitments::default()` (all zeros) is accepted without error. No privileged external party is required; the sequencer itself is the prover.

---

### Recommendation

Implement the computation of `header_commitments` and `gas_prices_hash` inside the Cairo OS program, as indicated by the existing TODO comment (`TODO(Yoni, 1/1/2027): compute the header commitments and gas prices`). Until then, the block hash does not cryptographically bind the proof to the transactions, events, receipts, or state diff it purports to cover.

Concretely:
- Compute `transaction_commitment`, `event_commitment`, and `receipt_commitment` from the transaction execution outputs already available inside the OS execution loop.
- Compute `state_diff_commitment` from the state changes already tracked by the OS Patricia trie update.
- Compute `packed_lengths` from the counts of transactions, events, and state-diff entries.
- Replace the `%{ GetBlockHashes %}` hint for these fields with Cairo-native computation, retaining the hint only for `previous_block_hash` (which requires a storage lookup).
- Demote `CheckBlockHashConsistency` to a debug-only assertion once the Cairo computation is in place.

---

### Proof of Concept

The OS flow test manager already demonstrates the issue without any modification. In `crates/starknet_os_flow_tests/src/test_manager.rs`, every block is prepared with:

```rust
let block_hash_commitments = BlockHeaderCommitments::default(); // zeros for all 5 fields
let new_block_hash = calculate_block_hash(
    &PartialBlockHashComponents::new(&block_info, block_hash_commitments.clone()),
    block_hash_state_root,
    prev_block_hash,
).unwrap();
let os_block_input = OsBlockInput {
    ...
    block_hash_commitments,   // all zeros — no real tx/event/receipt/state-diff commitment
    new_block_hash,           // consistent with the zeros above
    ...
};
```

The SNOS runs successfully and produces a valid proof. The `CheckBlockHashConsistency` hint passes because `new_block_hash` was computed from the same zero commitments. The resulting block hash stored on-chain commits to zero transaction/event/receipt/state-diff roots regardless of what transactions were actually executed.

To demonstrate fabrication: replace `BlockHeaderCommitments::default()` with any arbitrary `BlockHeaderCommitments { transaction_commitment: TransactionCommitment(Felt::from(0xdeadbeef)), ... }`, recompute `new_block_hash` accordingly, and the OS proof still passes — while the on-chain block hash now falsely claims the block contains a transaction set with Merkle root `0xdeadbeef`.

    let os_input = &hint_processor.get_current_execution_helper()?.os_block_input;

}

pub(crate) fn get_block_hashes<S: StateReader>(
    hint_processor: &mut SnosHintProcessor<'_, S>,
    mut ctx: HintContext<'_>,
) -> OsHintResult {
    let os_input = &hint_processor.get_current_execution_helper()?.os_block_input;
    let block_info = &os_input.block_info;
    let gas_prices = &block_info.gas_prices;
    let commitments = &os_input.block_hash_commitments;

    ctx.insert_value(Ids::PreviousBlockHash, os_input.prev_block_hash.0)?;

    let header_commitments_ptr = ctx.vm.add_memory_segment();
    ctx.insert_values_to_fields(
        header_commitments_ptr,
        CairoStruct::BlockHeaderCommitments,
        &[
            ("transaction_commitment", commitments.transaction_commitment.0.into()),
            ("event_commitment", commitments.event_commitment.0.into()),
            ("receipt_commitment", commitments.receipt_commitment.0.into()),
            ("state_diff_commitment", commitments.state_diff_commitment.0.0.into()),
            ("packed_lengths", commitments.concatenated_counts.into()),
        ],
    )?;
    ctx.insert_value(Ids::HeaderCommitments, header_commitments_ptr)?;

    // The on-chain block hash commits to the exact ASCII encoding of the version string, which
    // may be newer than the latest known `StarknetVersion` variant.
    let starknet_version_string = os_input
        .starknet_version_override
        .clone()
        .unwrap_or_else(|| block_info.starknet_version.to_string());
    ctx.insert_value(Ids::StarknetVersion, ascii_as_felt(&starknet_version_string)?)?;

    let [gas_prices_hash]: [Felt; 1] = gas_prices_to_hash(
        &gas_prices.l1_gas_price_per_token(),
        &gas_prices.l1_data_gas_price_per_token(),
        &gas_prices.l2_gas_price_per_token(),
        &block_info.starknet_version.try_into()?,
    )
    .try_into()
    .map_err(|_| {
        OsHintError::UnsupportedStarknetVersionForBlockHash(block_info.starknet_version)
    })?;

    ctx.insert_value(Ids::GasPricesHash, gas_prices_hash)?;

    Ok(())
}

/// Calculates the commitments of the transactions data for the block hash.
pub async fn calculate_block_commitments(
    transactions_data: &[TransactionHashingData],
    state_diff: ThinStateDiff,
    l1_da_mode: L1DataAvailabilityMode,
    starknet_version: &StarknetVersion,
) -> (BlockHeaderCommitments, BlockCommitmentsMeasurements) {
    let transaction_leaf_elements: Vec<TransactionLeafElement> = transactions_data
        .iter()
        .map(|tx_leaf| {
            let mut tx_leaf_element = TransactionLeafElement::from(tx_leaf);
            if starknet_version < &BlockHashVersion::V0_13_4.into()
                && tx_leaf.transaction_signature.0.is_empty()
            {
                tx_leaf_element.transaction_signature =
                    TransactionSignature(vec![Felt::ZERO].into());
            }
            tx_leaf_element
        })
        .collect();

    let event_leaf_elements: Vec<EventLeafElement> = transactions_data
        .iter()
        .flat_map(|transaction_data| {
            transaction_data.transaction_output.events.iter().map(|event| EventLeafElement {
                event: event.clone(),
                transaction_hash: transaction_data.transaction_hash,
            })
        })
        .collect();

    let receipt_elements: Vec<ReceiptElement> =
        transactions_data.iter().map(ReceiptElement::from).collect();

    let concatenated_counts = concat_counts(
        transactions_data.len(),
        event_leaf_elements.len(),
        state_diff.len(),
        l1_da_mode,
    );

    let n_txs = transactions_data.len();
    let n_events = event_leaf_elements.len();
    let state_diff_length = state_diff.len();

    // Spawn tasks for parallel execution; each measures its own duration.
    let transaction_task = spawn_measured_task(move || {
        calculate_transaction_commitment::<Poseidon>(&transaction_leaf_elements)
    });

    let event_task =
        spawn_measured_task(move || calculate_event_commitment::<Poseidon>(&event_leaf_elements));

    let receipt_task =
        spawn_measured_task(move || calculate_receipt_commitment::<Poseidon>(&receipt_elements));

    let state_diff_task = spawn_measured_task(move || calculate_state_diff_hash(&state_diff));

    // Wait for all tasks to complete.
    let (
        (transaction_commitment, transaction_duration),
        (event_commitment, event_duration),
        (receipt_commitment, receipt_duration),
        (state_diff_commitment, state_diff_duration),
    ) = tokio::try_join!(transaction_task, event_task, receipt_task, state_diff_task)
        .expect("Failed to join block commitments tasks.");

    let commitments = BlockHeaderCommitments {
        transaction_commitment,
        event_commitment,
        receipt_commitment,
        state_diff_commitment,
        concatenated_counts,
    };
    let measurements = BlockCommitmentsMeasurements {
        transaction_commitment_duration: transaction_duration,
        event_commitment_duration: event_duration,
        receipt_commitment_duration: receipt_duration,
        state_diff_commitment_duration: state_diff_duration,
        n_txs,
        n_events,
        state_diff_length,
    };
    (commitments, measurements)
}

pub(crate) fn write_old_block_to_storage<S: StateReader>(
    hint_processor: &mut SnosHintProcessor<'_, S>,
    ctx: HintContext<'_>,
) -> OsHintResult {
    let execution_helper = &mut hint_processor.get_mut_current_execution_helper()?;

    let block_hash_contract_address = ctx.fetch_const(Const::BlockHashContractAddress)?;
    let old_block_number = ctx.get_integer(Ids::OldBlockNumber)?;
    let old_block_hash = ctx.get_integer(Ids::OldBlockHash)?;

    log::debug!("writing block number: {old_block_number} -> block hash: {old_block_hash}");

    execution_helper.cached_state.set_storage_at(
        ContractAddress(PatriciaKey::try_from(*block_hash_contract_address)?),
        StorageKey(PatriciaKey::try_from(old_block_number)?),
        old_block_hash,
    )?;
    Ok(())
}
