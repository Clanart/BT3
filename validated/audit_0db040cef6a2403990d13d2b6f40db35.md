### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The OS-level implementation of the `replace_class` syscall does not verify that the supplied new class hash corresponds to a previously declared contract class. A malicious or buggy contract can call `replace_class` with an arbitrary, undeclared class hash. The OS accepts the update unconditionally, permanently setting the contract's class hash to a value for which no executable code exists. All subsequent calls to that contract will fail at class-lookup time, and any funds held by the contract are permanently frozen with no recovery path.

---

### Finding Description

In `execute_replace_class` (lines 878–916 of `syscall_impls.cairo`), the OS reads the requested new class hash directly from the syscall request buffer and writes it into `contract_state_changes` without any check that the hash is present in `contract_class_changes` (the declared-class registry):

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-inserted TODO comment at line 898 explicitly acknowledges the missing check. No other code path in the OS enforces this invariant for `replace_class`. By contrast, `execute_declare_transaction` (in `transaction_impls.cairo`) enforces `prev_value=0` to prevent double-declaration, and `execute_deploy` computes the contract address from the class hash — but neither of these guards applies to `replace_class`.

Once the state entry's `class_hash` field is set to an undeclared value, every future invocation of the contract will fail at class-lookup time (the OS cannot find compiled code for the hash). Because `replace_class` itself can only be called from within a successful contract execution, and the contract can no longer execute, there is no on-chain recovery path. [1](#0-0) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any funds (tokens, ETH-bridged assets, protocol reserves) held in a contract whose class hash has been replaced with an undeclared value are permanently inaccessible. The contract cannot execute any entry point (including withdrawal or recovery functions) because the OS cannot resolve the class hash to executable code. There is no OS-level mechanism to revert a committed state update after the block is proven and accepted on L1.

---

### Likelihood Explanation

**Medium.**

The attack requires a contract that exposes a `replace_class` call path reachable by an attacker. Two realistic scenarios:

1. **Malicious deployer rug-pull**: A deployer creates a contract that accepts user deposits and includes a privileged (or even public) function that calls `replace_class` with an attacker-chosen hash. After accumulating user funds, the deployer triggers the call, permanently bricking the contract and freezing deposited assets.

2. **Proxy/upgrade pattern with weak access control**: Many StarkNet contracts use an upgradeable proxy pattern where an `upgrade` function calls `replace_class`. If the access control on that function has a bug, an attacker can supply an invalid hash and permanently disable the proxy, freezing all protocol funds routed through it.

Both scenarios are reachable by an unprivileged transaction sender or contract deployer with no special privileges beyond the ability to submit transactions.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that the hash exists in `contract_class_changes` (the squashed class registry). Concretely, perform a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert that the returned compiled class hash is non-zero (i.e., the class has been declared):

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("replace_class: class hash not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the invariant already enforced in `execute_declare_transaction` where `assert_not_zero(compiled_class_hash)` is called before updating `contract_class_changes`. [2](#0-1) 

---

### Proof of Concept

**Step 1 — Deploy a trap contract.**

Deploy a Cairo contract whose constructor or any callable entry point issues the `replace_class` syscall with a caller-supplied (arbitrary) class hash, e.g.:

```cairo
#[external(v0)]
fn brick_self(ref self: ContractState, fake_hash: ClassHash) {
    starknet::replace_class_syscall(fake_hash).unwrap();
}
```

**Step 2 — Accumulate victim funds.**

Have users deposit tokens into the contract (e.g., via a standard ERC-20 `transfer` or a DeFi deposit function).

**Step 3 — Trigger the invalid replacement.**

Submit an `INVOKE_FUNCTION` transaction calling `brick_self` with `fake_hash = 0xdeadbeef` (any value not present in the declared-class tree).

**Step 4 — OS accepts without validation.**

`execute_replace_class` in `syscall_impls.cairo` reads `class_hash = 0xdeadbeef` from the syscall request and calls `dict_update` on `contract_state_changes` with no membership check against `contract_class_changes`. [3](#0-2) 

**Step 5 — State is committed.**

`state_update` in `state/state.cairo` squashes and commits the updated `contract_state_changes` (including the poisoned class hash) into the Patricia Merkle Tree and produces a valid proof accepted by L1. [4](#0-3) 

**Step 6 — Contract is permanently bricked.**

All subsequent calls to the contract fail at class resolution. Deposited user funds are permanently frozen with no recovery path.

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
