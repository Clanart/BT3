### Title
`StateDB.Transfer()` to a Cosmos-Blocked Address Silently Stores Error, Causing Cosmos-Level Tx Failure and Missing Gas Refund — (File: `x/evm/statedb/statedb.go`)

### Summary

When an EVM transaction sends value to an address that maps to a Cosmos SDK blocked address (e.g., a module account), `bankKeeper.SendCoins()` fails inside `StateDB.Transfer()`. Rather than propagating the error immediately as an EVM revert, the error is silently stored in `s.err`. The EVM continues executing with incorrect state assumptions. At commit time, `stateDB.Commit()` returns the stored error as a cosmos-level error (not a `VmError`), causing `ApplyMessageWithConfig()` to return an error. `ApplyTransaction()` then returns early — **before** calling `RefundGas()` — so the gas refund owed to the sender is never applied. The user's full upfront fee (deducted in the ante handler) is permanently lost.

---

### Finding Description

**Root cause — `x/evm/statedb/statedb.go`, `Transfer()` (lines 434–450):**

```go
func (s *StateDB) Transfer(sender, recipient common.Address, amount *uint256.Int) {
    // ...
    if err := s.ExecuteNativeAction(common.Address{}, nil, func(ctx sdk.Context) error {
        return s.keeper.Transfer(ctx, senderAddr, recipientAddr, coins)
    }); err != nil {
        s.err = err   // ← error stored, EVM continues unaware
    }
}
```

`s.keeper.Transfer()` calls `bankKeeper.SendCoins()`. The Cosmos SDK bank module rejects sends to blocked addresses (module accounts). The error is stored in `s.err` but **not** returned to the EVM, so the EVM proceeds as if the transfer succeeded.

**Propagation — `x/evm/statedb/statedb.go`, `Commit()` (lines 748–756):**

```go

    // ...
    if s.err != nil {
        return s.err   // ← cosmos-level error, not ErrStateConflict
    }
    // ...
}
```

**Cosmos-level error path — `x/evm/keeper/state_transition.go`, `ApplyMessageWithConfig()` (lines 602–623):**


Only `ErrStateConflict` is demoted to a `VmError`. A bank-blocked-address error is **not** `ErrStateConflict`, so it propagates as a cosmos-level error.

**Gas refund skipped — `x/evm/keeper/state_transition.go`, `ApplyTransaction()` (lines 194–254):**

```go

if err != nil {
    return nil, errorsmod.Wrap(err, "failed to apply ethereum core message")
    // ← returns here; RefundGas() on line 252 is never reached
}
// ...

```

The ante handler already deducted `gasLimit × effectiveGasPrice` from the sender via `DeductTxCostsFromUserBalance()`. Because `ApplyTransaction()` returns an error before `RefundGas()` is called, the `leftoverGas × gasPrice` refund is never returned to the sender.

---

### Impact Explanation

**High — EVM state transition / gas refund / fee mis-accounting.**

1. **Gas refund permanently lost**: The sender paid `gasLimit × gasPrice` upfront. In a normal EVM failure (VmError), `ApplyMessageWithConfig()` still returns a result (not an error), so `RefundGas()` is called and unused gas is returned. In this path, `ApplyMessageWithConfig()` returns an error, `RefundGas()` is skipped, and the sender forfeits the entire upfront fee.

2. **EVM executes with incorrect state**: The EVM believes the value transfer succeeded (no revert signal was sent). Any subsequent opcodes in the same call frame that depend on the transfer having occurred (e.g., balance checks, reentrancy guards) operate on a false premise. The entire state is rolled back at commit time, but the EVM's execution trace was already corrupted.

3. **Inconsistent error surface**: A normal failed EVM call is included in the block with `status=0` (VmError). This path causes a cosmos-level error, which is a different failure mode that breaks Ethereum-compatible tooling (receipts, explorers, RPC clients) that expect all EVM failures to surface as `VmError`.

---

### Likelihood Explanation

**Medium.** Module account addresses in Cosmos SDK are deterministic and publicly derivable from their module name. Any user can compute the EVM-format address of the fee collector, distribution module, staking module, etc., and send a value-bearing EVM transaction to it. No special privileges are required. A contract that programmatically sends ETH to such an address (e.g., a fee-splitting contract that routes to a hardcoded address that was later registered as a module account) would trigger this silently on every call.

---

### Recommendation

1. **Treat blocked-address transfer failures as EVM reverts, not cosmos errors.** In `StateDB.Transfer()`, if `ExecuteNativeAction` returns an error, set `s.err` **and** signal the EVM to revert the current call frame (e.g., by returning a sentinel that the EVM interprets as an out-of-gas or revert condition), consistent with how go-ethereum handles failed transfers.

2. **Alternatively**, in `ApplyMessageWithConfig()`, treat any `stateDB.Commit()` error that originates from a bank transfer failure the same way `ErrStateConflict` is treated — demote it to a `VmError` so the transaction is included in the block with `status=0` and `RefundGas()` is still called.

3. **Ensure `RefundGas()` is always called** when the ante handler has already deducted fees, regardless of whether `ApplyMessageWithConfig()` returns an error or a result.

---

### Proof of Concept

1. Identify the EVM-format address of the Cosmos fee collector module account:
   ```
   feeCollectorAddr = sdk.AccAddress(authtypes.NewModuleAddress(authtypes.FeeCollectorName))
   evmAddr = common.BytesToAddress(feeCollectorAddr)
   ```

2. Submit an EVM transaction from any EOA:
   ```
   eth_sendRawTransaction({
     to:    evmAddr,   // fee collector module account
     value: 1,
     gas:   100000,
     ...
   })
   ```

3. Execution path:
   - Ante handler deducts `100000 × gasPrice` from sender.
   - `ApplyMessageWithConfig()` → `evm.Call()` → `StateDB.Transfer()` → `bankKeeper.SendCoins()` fails (blocked address) → `s.err` set.
   - `stateDB.Commit()` returns `s.err` (not `ErrStateConflict`).
   - `ApplyMessageWithConfig()` returns cosmos-level error.
   - `ApplyTransaction()` returns early; `RefundGas()` is never called.
   - Sender loses full `100000 × gasPrice` fee; no gas refund is applied.

4. Contrast with a normal EVM revert (e.g., calling a contract that `REVERT`s): `ApplyMessageWithConfig()` returns `(result, nil)` with `result.VmError != ""`, `RefundGas()` is called, and unused gas is returned. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) [7](#0-6)

```go
func (k *Keeper) RefundGas(ctx sdk.Context, msg *core.Message, leftoverGas uint64, denom string) error {
	return k.RefundGasWithPrice(ctx, msg, leftoverGas, msg.GasPrice, denom)
}

// RefundGasWithPrice transfers the leftover gas to sender using the provided gas price.
func (k *Keeper) RefundGasWithPrice(
	ctx sdk.Context,
	msg *core.Message,
	leftoverGas uint64,
	gasPrice *big.Int,
	denom string,
) error {
	if gasPrice == nil {
		gasPrice = new(big.Int)
	}

	// Return EVM tokens for remaining gas, exchanged at the original rate.
	remaining := new(big.Int).Mul(new(big.Int).SetUint64(leftoverGas), gasPrice)

	switch remaining.Sign() {
	case -1:
		// negative refund errors
		return errorsmod.Wrapf(types.ErrInvalidRefund, "refunded amount value cannot be negative %d", remaining.Int64())
	case 1:
		// positive amount refund
		refundedCoins := sdk.Coins{sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(remaining))}

		// refund to sender from the fee collector module account, which is the escrow account in charge of collecting tx fees
		err := k.bankKeeper.SendCoinsFromModuleToAccountVirtual(ctx, authtypes.FeeCollectorName, msg.From.Bytes(), refundedCoins)
		if err != nil {
			err = errorsmod.Wrapf(errortypes.ErrInsufficientFunds, "fee collector account failed to refund fees: %s", err.Error())
			return errorsmod.Wrapf(err, "failed to refund %d leftover gas (%s)", leftoverGas, refundedCoins.String())
		}
	default:
		// no refund, consume gas and update the tx gas meter
	}

	return nil
}
```

```go
// DeductFees deducts fees from the given account.
func DeductFees(bankKeeper types.BankKeeper, ctx sdk.Context, acc sdk.AccountI, fees sdk.Coins) error {
	if !fees.IsValid() {
		return errorsmod.Wrapf(errortypes.ErrInsufficientFee, "invalid fee amount: %s", fees)
	}
	if ctx.BlockHeight() > 0 {
		if err := bankKeeper.SendCoinsFromAccountToModuleVirtual(ctx, acc.GetAddress(), authtypes.FeeCollectorName, fees); err != nil {
			return errorsmod.Wrap(errortypes.ErrInsufficientFunds, err.Error())
		}
	}
	return nil
}
```
