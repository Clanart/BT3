### Title
`CalculateBaseFee` Panics on Zero or Unlimited `MaxGas` Consensus Parameter, Halting the Chain - (File: x/feemarket/keeper/eip1559.go)

### Summary

`CalculateBaseFee` in `x/feemarket/keeper/eip1559.go` contains an unconditional `panic` when `consParams.Block == nil` or `consParams.Block.MaxGas <= -1`. Because this function is called every block in `BeginBlock`, any chain configuration where `MaxGas` is set to `-1` (unlimited) or where consensus params are temporarily unavailable will cause every validator to panic deterministically, halting the chain. This is a reachable, consensus-critical code path triggered by a valid on-chain governance parameter change — no privileged key compromise is required.

### Finding Description

In `CalculateBaseFee`, after confirming that the base fee is enabled and the block height is past `EnableHeight`, the function reads the consensus parameters to derive the gas target:

```go
if consParams.Block == nil || consParams.Block.MaxGas <= -1 {
    panic(fmt.Sprintf("get invalid consensus params: %s", consParams))
}
```

The CometBFT/Tendermint specification explicitly defines `MaxGas = -1` as meaning "no block gas limit." This is a valid, documented chain configuration. If a chain operator or governance proposal sets `MaxGas` to `-1` while the feemarket `NoBaseFee` is `false` and `EnableHeight` has been reached, every call to `BeginBlock` will panic on every validator node simultaneously, causing a deterministic consensus failure and chain halt.

The panic is not recoverable — it propagates up through the ABCI `BeginBlock` handler and crashes the node process (or causes the SDK's panic recovery to mark the block as failed, which is equally fatal for consensus).

The entry path is:
1. `feemarket.BeginBlock` → `CalculateBaseFee` (called every block)
2. `consParams.Block.MaxGas <= -1` evaluates to `true`
3. `panic(...)` is executed on all validators simultaneously [1](#0-0) 

The `BeginBlock` caller does not recover from this panic: [2](#0-1) 

The `MaxGas = -1` semantics are explicitly documented in the Cosmos/CometBFT ecosystem as "unlimited gas," making this a valid governance-reachable state: [3](#0-2) 

Note that `types.BlockGasLimit` in the same codebase correctly handles `MaxGas == -1` by returning `math.MaxUint64`, but `CalculateBaseFee` does not apply the same defensive handling — it panics instead.

### Impact Explanation

**Critical.** A valid unprivileged governance transaction (or a chain launched with `MaxGas = -1`) triggers a deterministic panic in `BeginBlock` on every validator simultaneously. This halts the chain. No funds need to be at risk for this to qualify — the impact is a complete consensus failure and chain halt, matching the allowed Critical impact: *"Valid unprivileged transaction, RPC submission, or block-processing path can halt the chain, corrupt committed state, or cause deterministic validator consensus failure."*

### Likelihood Explanation

**Medium-High.** The condition is reachable via:
1. A governance proposal that sets `MaxGas = -1` (unlimited) — a common configuration on testnets and some mainnets.
2. A chain genesis that sets `MaxGas = -1` while also enabling the feemarket with `NoBaseFee = false`.
3. Any future upgrade that resets consensus params to defaults where `MaxGas` is unset or `-1`.

The feemarket is enabled by default in Ethermint-based chains, and `MaxGas = -1` is a standard CometBFT configuration. The combination is realistic and does not require any attacker — it can be triggered by a well-intentioned governance vote.

### Recommendation

Replace the `panic` with a graceful error return or a safe fallback. The function already returns `nil` in other error cases (e.g., `parentGasTargetBig.IsUint64()` check). Apply the same pattern:

```go
if consParams.Block == nil || consParams.Block.MaxGas <= 0 {
    // MaxGas == -1 means unlimited; base fee adjustment cannot be computed.
    // Return the current base fee unchanged rather than panicking.
    return new(big.Int).Set(parentBaseFee)
}
```

Alternatively, mirror the logic in `types.BlockGasLimit` which maps `MaxGas == -1` to `math.MaxUint64`, and use that as the gas limit for the EIP-1559 calculation.

### Proof of Concept

1. Deploy an Ethermint chain with `feemarket` enabled (`NoBaseFee = false`, `EnableHeight = 1`).
2. Submit a governance proposal to set `ConsensusParams.Block.MaxGas = -1` (unlimited gas).
3. Once the proposal passes and takes effect, the next block's `BeginBlock` calls `CalculateBaseFee`.
4. `consParams.Block.MaxGas` is `-1`, so `consParams.Block.MaxGas <= -1` is `true`.
5. `panic(fmt.Sprintf("get invalid consensus params: %s", consParams))` executes on every validator.
6. All validators crash/halt simultaneously → chain halt.

The condition `consParams.Block.MaxGas <= -1` also fires when `MaxGas == 0`, which is another valid "unset" state in some SDK versions, broadening the attack surface further. [4](#0-3) [5](#0-4)

```go
func (k Keeper) CalculateBaseFee(ctx sdk.Context) *big.Int {
	params := k.GetParams(ctx)

	// Ignore the calculation if not enabled
	if !params.IsBaseFeeEnabled(ctx.BlockHeight()) {
		return nil
	}
	consParams := ctx.ConsensusParams()

	// If the current block is the first EIP-1559 block, return the base fee
	// defined in the parameters (DefaultBaseFee if it hasn't been changed by
	// governance).
	if ctx.BlockHeight() == params.EnableHeight {
		return params.BaseFee.BigInt()
	}

	// get the block gas used and the base fee values for the parent block.
	// NOTE: this is not the parent's base fee but the current block's base fee,
	// as it is retrieved from the transient store, which is committed to the
	// persistent KVStore after EndBlock (ABCI Commit).
	parentBaseFee := params.BaseFee.BigInt()
	if parentBaseFee == nil {
		return nil
	}

	parentGasUsed := k.GetBlockGasWanted(ctx)

	// NOTE: a MaxGas equal to -1 means that block gas is unlimited
	if consParams.Block == nil || consParams.Block.MaxGas <= -1 {
		panic(fmt.Sprintf("get invalid consensus params: %s", consParams))
	}
```

```go
// BeginBlock updates base fee
func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
}
```
