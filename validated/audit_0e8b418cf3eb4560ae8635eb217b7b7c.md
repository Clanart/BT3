### Title
Unguarded `uint256.MustFromBig` in `StateOverride.Apply` Panics on Overflow, Crashing the Node via `EthCall`/`EstimateGas`/`TraceCall` — (`rpc/types/types.go`)

### Summary

`StateOverride.Apply` calls `uint256.MustFromBig` on an attacker-controlled balance value without first checking for overflow. An unprivileged caller can submit an `EthCallRequest` (or `EstimateGas`/`TraceCall`) with an `Overrides` balance field set to any value ≥ 2^256, triggering an unrecovered panic in the gRPC handler goroutine and crashing the node process.

### Finding Description

`StateOverride.Apply` in `rpc/types/types.go` handles the balance override path as follows:

```go
if account.Balance != nil {
    balance := (*big.Int)(*account.Balance)
    db.SetBalance(addr, *uint256.MustFromBig(balance))  // panics if balance >= 2^256
}
``` [1](#0-0) 

`uint256.MustFromBig` is documented to panic when the input `*big.Int` does not fit in 256 bits. There is no overflow check before this call.

The `OverrideAccount.Balance` field is typed `**hexutil.Big`, which is a thin wrapper over `*big.Int`. The `json.Unmarshal` step for `hexutil.Big` accepts any hex-encoded integer of arbitrary size — there is no size cap at the JSON layer.

The full attacker-controlled call chain is:

1. Attacker sends `EthCallRequest` with `Overrides: {"0x<addr>": {"balance": "0x1<64 zeros>"}}` (i.e., 2^256).
2. `EthCall` in `grpc_query.go` unmarshals `req.Overrides` into `rpctypes.StateOverride` — succeeds, no size check.
3. Sets `cfg.Overrides = &overrides` and calls `k.ApplyMessageWithConfig(...)`.
4. `ApplyMessageWithConfig` in `state_transition.go` calls `cfg.Overrides.Apply(stateDB)`.
5. `StateOverride.Apply` calls `uint256.MustFromBig(balance)` → **panic**. [2](#0-1) [3](#0-2) 

There is no `recover()` anywhere in this call stack. A panic in a gRPC handler goroutine that is not recovered crashes the entire node process.

The same vulnerable `StateOverride.Apply` is also reachable via `EstimateGas` (same `EthCallRequest` proto) and via `prepareTrace` (used by `TraceCall`, `TraceTx`, `TraceBlock`): [4](#0-3) 

### Impact Explanation

A single malformed RPC request causes an unrecovered panic in the node process, halting the chain. This matches the Critical impact: **valid unprivileged RPC submission can halt the chain via node process crash**.

### Likelihood Explanation

The endpoint is publicly accessible (no authentication required for `eth_call`, `eth_estimateGas`, `debug_traceCall`). The payload is trivial to construct — any hex value with more than 64 hex digits after `0x` suffices. No special privileges, keys, or chain state are required.

### Recommendation

Replace `uint256.MustFromBig` with `uint256.FromBig` and return an error on overflow, exactly as the already-fixed `SimStateOverride.Apply` does:

```go
// SimStateOverride.Apply — correct pattern (rpc/types/simulate.go:155-160)
u256Balance, overflow := uint256.FromBig((*big.Int)(account.Balance))
if overflow {
    return &InvalidParamsError{Message: fmt.Sprintf("account %s balance overflows uint256", addr.Hex())}
}
stateDB.SetBalance(addr, *u256Balance)
``` [5](#0-4) 

Apply the same pattern to `StateOverride.Apply` in `rpc/types/types.go` at line 100–103.

### Proof of Concept

```go
// Unit test: send EthCallRequest with balance = 2^256 (overflows uint256)
// Expected: handler returns error; Actual: panic crashes the process
overflowBal := new(big.Int).Lsh(big.NewInt(1), 256) // 2^256
overrides := rpctypes.StateOverride{
    common.HexToAddress("0xdeadbeef"): rpctypes.OverrideAccount{
        Balance: func() **hexutil.Big {
            b := (*hexutil.Big)(overflowBal)
            return &b
        }(),
    },
}
bz, _ := json.Marshal(overrides)
req := &types.EthCallRequest{
    Args:      []byte(`{"from":"0x0000000000000000000000000000000000000001","to":"0x0000000000000000000000000000000000000002","gas":"0x5208"}`),
    GasCap:    25_000_000,
    Overrides: bz,
}
// This call panics instead of returning an error:
_, err := keeper.EthCall(ctx, req)
```
