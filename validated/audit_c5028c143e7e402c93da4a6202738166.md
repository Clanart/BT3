### Title
Native ETH Sent With Non-WETH Swap Is Silently Trapped and Stealable by Any Caller via `refundETH()` — (File: `metric-periphery/contracts/MetricOmmSimpleRouter.sol`, `metric-periphery/contracts/base/PeripheryPayments.sol`)

---

### Summary

All four public swap entry points (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`) are `payable` but contain no guard that reverts when `msg.value > 0` and `tokenIn != WETH`. When a user sends native ETH alongside a pure ERC-20 swap, the `pay()` helper silently ignores the native balance and pulls the ERC-20 via `safeTransferFrom`. The ETH is stranded on the router. Because `refundETH()` unconditionally transfers the entire native balance to `msg.sender`, any subsequent caller can drain the stranded ETH in the same or a later block.

---

### Finding Description

**`PeripheryPayments.pay()` ignores native ETH for non-WETH tokens** [1](#0-0) 

The `pay()` function has three branches:
1. `payer == address(this)` → ERC-20 transfer from router (mid-path hop)
2. `token == WETH` → wraps native ETH balance first, then pulls remainder from payer
3. `else` → pure `safeTransferFrom(payer, recipient, value)` — **native ETH balance is never touched**

When `token != WETH`, any ETH sitting on the router is completely ignored. No revert, no refund.

**All swap functions are `payable` with no ETH-guard** [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) 

None of these functions check `if (msg.value > 0 && tokenIn != WETH) revert`. The EVM accepts the ETH because the functions are `payable`, the swap executes normally via ERC-20 pull, and the ETH is left on the contract.

**`refundETH()` sends to `msg.sender`, not the original depositor** [6](#0-5) 

`refundETH()` is a public, permissionless function that transfers the router's entire native balance to `msg.sender`. There is no record of who deposited the ETH, so any caller — including a front-running bot — can claim it.

**Same issue in `MetricOmmPoolLiquidityAdder`** [7](#0-6) [8](#0-7) 

`addLiquidityExactShares` and `addLiquidityWeighted` are also `payable`. If the pool's WETH leg is `token1` (not `token0`) and the user sends ETH while the callback pays `token0` first, or if neither token is WETH, the ETH is stranded and stealable via the same `refundETH()` path inherited from `PeripheryPayments`.

---

### Impact Explanation

**Direct loss of user principal.** A user who calls `exactInputSingle{value: 1 ether}(tokenIn=USDC, ...)` loses 1 ETH. The swap succeeds (USDC is pulled via `safeTransferFrom`), the user receives the expected output token, and has no on-chain indication that their ETH was not consumed. A MEV bot monitoring the mempool can call `refundETH()` immediately after and receive the stranded ETH. The loss is unbounded — it equals whatever `msg.value` the victim sent.

---

### Likelihood Explanation

**Medium.** The Uniswap v3 multicall pattern (`multicall{value}([swap(..., tokenIn=WETH), refundETH()])`) trains users to send ETH with swap calls. A user or integrator who mistakenly sets `tokenIn` to a non-WETH ERC-20 while still sending ETH (e.g., copy-paste error, UI bug, or off-chain path-building mistake) triggers the loss silently. The interface comment notes "ERC-20 routes only" but the functions remain `payable` with no enforcement. [9](#0-8) 

---

### Recommendation

Add a guard at the top of each swap entry point (and each `addLiquidity*` entry point in `MetricOmmPoolLiquidityAdder`) that reverts when native ETH is sent but the input token is not WETH:

```solidity
// In exactInputSingle, exactInput:
if (msg.value > 0 && params.tokenIn != WETH) revert NativeNotAccepted();

// In exactOutputSingle, exactOutput:
if (msg.value > 0 && params.tokenIn != WETH) revert NativeNotAccepted();
```

Alternatively, remove `payable` from all non-WETH-aware entry points and keep it only on `multicall`, `refundETH`, `unwrapWETH9`, and `sweepToken`.

---

### Proof of Concept

```solidity
// Alice calls exactInputSingle with USDC as tokenIn but accidentally sends 1 ETH
router.exactInputSingle{value: 1 ether}(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: usdcPool,
        tokenIn: address(usdc),   // NOT WETH
        tokenOut: address(token1),
        zeroForOne: true,
        amountIn: 1000e6,
        amountOutMinimum: 0,
        recipient: alice,
        deadline: block.timestamp + 1,
        priceLimitX64: 0,
        extensionData: ""
    })
);
// pay() branch: token != WETH → safeTransferFrom(alice, pool, 1000e6)
// 1 ETH is now stranded on the router; swap succeeded normally

// Bob (attacker) calls refundETH() in the next transaction
router.refundETH();
// Bob receives Alice's 1 ETH
assert(bob.balance == 1 ether);
```

The `pay()` function's `else` branch executes `safeTransferFrom` and never touches `address(this).balance`, confirming the ETH is silently ignored. [10](#0-9)

  function pay(address token, address payer, address recipient, uint256 value) internal {
    // If the payer is contract it means we are in the middle of a path. In the middle of a path we operate on ERC20 only.
    if (payer == address(this)) {
      IERC20(token).safeTransfer(recipient, value);
    } else if (token == WETH) {
      uint256 nativeBalance = address(this).balance;
      if (nativeBalance >= value) {
        IWETH9(WETH).deposit{value: value}();
        IERC20(WETH).safeTransfer(recipient, value);
      } else if (nativeBalance > 0) {
        IWETH9(WETH).deposit{value: nativeBalance}();
        IERC20(WETH).safeTransfer(recipient, nativeBalance);
        IERC20(WETH).safeTransferFrom(payer, recipient, value - nativeBalance);
      } else {
        IERC20(WETH).safeTransferFrom(payer, recipient, value);
      }
    } else {
      IERC20(token).safeTransferFrom(payer, recipient, value);
    }
  }

  function exactInput(ExactInputParams calldata params) external payable returns (uint256 amountOut) {
    _checkDeadline(params.deadline);
    _validatePath(params.tokens, params.pools, params.extensionDatas);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

      amount = MetricOmmSwapResults.extractAmountOut(zeroForOne, amount0Delta, amount1Delta);
    }

    if (amount <= 0) revert InvalidSwapDeltas();
    amountOut = MetricOmmSwapInputs.int128ToUint128(amount);
    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }

    _clearExpectedCallbackPool();
  }

  function exactOutput(ExactOutputParams calldata params) external payable returns (uint256 amountIn) {
    _checkDeadline(params.deadline);

    uint8 tradesLeftAfterThis = uint8(params.pools.length - 1);
    address pool = params.pools[tradesLeftAfterThis];

    int128 expectedAmountOut = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountOut);

      pool, CALLBACK_MODE_EXACT_OUTPUT_ITERATE, tradesLeftAfterThis, msg.sender, params.tokens[0]
    );

      .swap(
        params.recipient,
        zeroForOne,

          tokens: params.tokens,
          pools: params.pools,
          extensionDatas: params.extensionDatas,
          zeroForOneBitMap: params.zeroForOneBitMap,
          amountInMax: params.amountInMaximum
        })
