### Title
Unverified `state_root` in P2P-synced `SignedBlockHeader` allows malicious peer to corrupt stored global state root — (`crates/apollo_p2p_sync/src/client/header.rs`)

### Summary
`HeaderStreamBuilder::parse_data_for_block` accepts a `SignedBlockHeader` from an untrusted p2p peer and passes it directly to `write_to_storage` / `append_header` without verifying the `state_root` field against any trusted source. The stored `state_root` is then served by the RPC layer as `new_root` and re-propagated to downstream peers by the p2p server.

### Finding Description
`parse_data_for_block` performs only two checks on the received `SignedBlockHeader`:

1. The block number matches the expected value.
2. The signatures vector has exactly one element. [1](#0-0) 

No check is performed on `state_root`, `block_hash`, `parent_hash` (a TODO comment at line 102 acknowledges this gap), or the cryptographic validity of the signature. `verify_block_signature` exists in `starknet_api` but is called only in tests — never in the p2p sync client path. [2](#0-1) 

`write_to_storage` then calls `append_header`, which stores `block_header.block_header_without_hash.state_root` verbatim into the `headers` table: [3](#0-2) 

`state_root` lives inside `BlockHeaderWithoutHash`, which is a direct field of the network-supplied `SignedBlockHeader`: [4](#0-3) 

The stored `state_root` is then:

1. **Served by RPC as `new_root`** — `BlockHeader::from` maps `state_root` → `new_root`: [5](#0-4) 
and `get_state_update` returns it as `new_root` in `AcceptedStateUpdate`: [6](#0-5) 

2. **Re-served by the p2p server** to other syncing nodes via `FetchBlockData for SignedBlockHeader`, which reads the stored header directly: [7](#0-6) 

3. **Consumed by `blockifier_reexecution`**, which reads `new_root` from the RPC `starknet_getBlockWithTxHashes` response: [8](#0-7) 

### Impact Explanation
A malicious p2p peer can inject an arbitrary `state_root` (e.g., `GlobalRoot(Felt::ONE)`) into the stored block header for block N. The RPC layer then serves this wrong value as `new_root` to any client querying the block — an authoritative-looking wrong committed value. The wrong `state_root` also propagates to other p2p-syncing nodes via the p2p server, and is consumed by `blockifier_reexecution` which uses `new_root` from RPC. This matches the allowed impact: **High — RPC returns an authoritative-looking wrong value** (`new_root` / global state root).

### Likelihood Explanation
Any node syncing headers via p2p from an untrusted peer is vulnerable. The attacker only needs to be a reachable p2p peer that responds to `HeaderQuery` messages. No operator or validator privileges are required.

### Recommendation
In `parse_data_for_block`, recompute the block hash from the received header fields using `calculate_block_hash` and assert it matches the `block_hash` field in the `SignedBlockHeader`. Additionally, cryptographically verify the block signature using `verify_block_signature` against a known sequencer public key before accepting and storing the header. [9](#0-8) 

### Proof of Concept
1. Run a malicious p2p peer that responds to `HeaderQuery` for block N with a `SignedBlockHeader` where `block_header_without_hash.state_root = GlobalRoot(Felt::ONE)` and a one-element (but otherwise arbitrary) signature vector.
2. The syncing node's `parse_data_for_block` accepts the header — block number matches, `signatures.len() == 1`.
3. `write_to_storage` calls `append_header`, storing `state_root = GlobalRoot(Felt::ONE)` for block N.
4. A subsequent RPC call `starknet_getBlockWithTxHashes` for block N returns `new_root = 0x1`.
5. The correct `state_root` for block N (computed from the actual Patricia trie) is a different value, demonstrating the corruption. Any downstream consumer of `new_root` (including other p2p peers that re-sync from this node) will receive the wrong global state root.

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
