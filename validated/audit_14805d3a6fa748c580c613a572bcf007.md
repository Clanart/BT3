### Title
`eth_signTypedData` Signs Attacker-Controlled EIP-712 Typed Data Without Domain or Content Validation, Enabling Forged Legacy EIP-712 Cosmos Transactions - (File: `rpc/backend/sign_tx.go`)

### Summary

The `Backend.SignTypedData` function, exposed as the `eth_signTypedData` JSON-RPC endpoint, accepts a fully caller-controlled `apitypes.TypedData` struct and signs its EIP-712 hash using the node's keyring without validating the domain separator, chain ID, or message content. An attacker with access to the public JSON-RPC endpoint can craft a `typedData` payload whose domain and message exactly match the format produced by `LegacyWrapTxToTypedData`, obtain a valid `FeePayerSig` for any address managed by the node, and submit a Legacy EIP-712 Cosmos transaction that passes `LegacyEip712SigVerificationDecorator` — transferring funds from the victim account without possessing its private key.

### Finding Description

**Root cause — `rpc/backend/sign_tx.go` lines 150–174:**

```go
func (b *Backend) SignTypedData(address common.Address, typedData apitypes.TypedData) (hexutil.Bytes, error) {
    from := sdk.AccAddress(address.Bytes())
    _, err := b.clientCtx.Keyring.KeyByAddress(from)
    if err != nil { ... }

    sigHash, _, err := apitypes.TypedDataAndHash(typedData)   // ← typedData is 100% caller-controlled
    if err != nil { return nil, err }

    signature, _, err := b.clientCtx.Keyring.SignByAddress(from, sigHash, ...)
    ...
    signature[crypto.RecoveryIDOffset] += 27
    return signature, nil
}
```

The only guard is that `address` must exist in the node keyring. There is no check that:
- `typedData.Domain.ChainId` matches the node's chain ID
- `typedData.Domain.Name` / `VerifyingContract` / `Salt` match the Cosmos EIP-712 domain
- The message content is safe or restricted to non-financial operations

**Contrast with `Sign` (eth_sign):** `Sign` applies `accounts.TextHash(data)` (EIP-191 prefix `"\x19Ethereum Signed Message:\n"`), which domain-separates the output from transaction signatures. The codebase even has a dedicated test `TestSign_CannotForgeTransaction` proving this protection. `SignTypedData` has no equivalent protection.

**The Legacy EIP-712 verification path (`ante/cosmos/eip712.go` lines 248–295) accepts a `FeePayerSig` that is exactly the output of `SignTypedData` when given the correct domain:**

```go
typedData, err := eip712.LegacyWrapTxToTypedData(ethermintCodec, extOpt.TypedDataChainID, msgs[0], txBytes, feeDelegation)
sigHash, _, err := apitypes.TypedDataAndHash(typedData)
...
if !ethcrypto.VerifySignature(pubKey.Bytes(), sigHash, feePayerSig[:len(feePayerSig)-1]) {
    return errorsmod.Wrap(errortypes.ErrorInvalidSigner, "unable to verify signer signature of EIP712 typed data")
}
```

The domain used by `LegacyWrapTxToTypedData` is fixed and fully public (`ethereum/eip712/eip712_legacy.go` lines 66–72):

```go
domain := apitypes.TypedDataDomain{
    Name:              "Cosmos Web3",
    Version:           "1.0.0",
    ChainId:           math.NewHexOrDecimal256(value),
    VerifyingContract: "cosmos",
    Salt:              "0",
}
```

An attacker who knows the chain ID (public) can reproduce this domain exactly and craft a `typedData` whose `sigHash` will pass the `VerifySignature` check in the ante handler.

**Exploit flow:**

1. Attacker queries the node's chain ID (e.g., `9000`) and the victim's account number and sequence (both public via gRPC/REST).
2. Attacker constructs `typedData` with:
   - `Domain = {Name:"Cosmos Web3", Version:"1.0.0", ChainId:9000, VerifyingContract:"cosmos", Salt:"0"}`
   - `PrimaryType = "Tx"`, `Types` matching the Amino JSON Cosmos tx schema
   - `Message` = a `MsgSend` from victim to attacker with the victim's current nonce/account number
3. Attacker calls `eth_signTypedData(victim_address, crafted_typed_data)` on the public JSON-RPC endpoint → receives `sig`.
4. Attacker builds a Cosmos tx with `ExtensionOptionsWeb3Tx{TypedDataChainID:9000, FeePayer:victim, FeePayerSig:sig}` and the same `MsgSend`.
5. `LegacyEip712SigVerificationDecorator` reconstructs the identical `typedData` via `LegacyWrapTxToTypedData`, computes the same `sigHash`, and `VerifySignature` passes — the transaction is accepted and funds are transferred.

### Impact Explanation

An attacker with access to the node's JSON-RPC endpoint can obtain a valid EIP-712 `FeePayerSig` for any address managed by the node's keyring, then submit a Legacy EIP-712 Cosmos transaction that passes all ante handler signature checks. This enables unauthorized transfer of EVM-denom or Cosmos bank funds from the victim account. The impact is **unauthorized theft of funds** via a forged EIP-712 authorization — matching the "High: EIP-712 authorization bypass enabling forged execution" category.

### Likelihood Explanation

- The `eth_signTypedData` endpoint is part of the standard Ethereum JSON-RPC API and is publicly accessible on most RPC nodes.
- The Cosmos EIP-712 domain parameters are fixed, public, and documented.
- The attacker needs no privileged access — only the ability to call the JSON-RPC endpoint and knowledge of the victim's address (public on-chain).
- The attack is applicable to any custodial node or validator node that manages user keys and exposes the JSON-RPC interface.

### Recommendation

**Short term:** In `Backend.SignTypedData`, validate that `typedData.Domain.ChainId` matches the node's configured chain ID, and that the domain name/verifying contract do not match the Cosmos EIP-712 domain (`"Cosmos Web3"` / `"cosmos"`). Reject requests whose typed data could be replayed as a Cosmos transaction.

**Long term:** Restrict `eth_signTypedData` to only sign typed data whose domain is explicitly allowlisted or whose primary type is not `"Tx"` with the Cosmos EIP-712 domain. Consider deprecating or gating the endpoint behind authentication, consistent with how Geth treats `personal_*` methods.

func (b *Backend) Sign(address common.Address, data hexutil.Bytes) (hexutil.Bytes, error) {
	from := sdk.AccAddress(address.Bytes())

	_, err := b.clientCtx.Keyring.KeyByAddress(from)

	}

	return signature, nil

// SignTypedData signs EIP-712 conformant typed data
func (b *Backend) SignTypedData(address common.Address, typedData apitypes.TypedData) (hexutil.Bytes, error) {
	from := sdk.AccAddress(address.Bytes())

	_, err := b.clientCtx.Keyring.KeyByAddress(from)
	if err != nil {
		b.logger.Error("failed to find key in keyring", "address", address.String())
		return nil, fmt.Errorf("%s; %s", keystore.ErrNoMatch, err.Error())
	}

	sigHash, _, err := apitypes.TypedDataAndHash(typedData)
	if err != nil {
		return nil, err
	}

	// Sign the requested hash with the wallet
	signature, _, err := b.clientCtx.Keyring.SignByAddress(from, sigHash, signingtypes.SignMode_SIGN_MODE_TEXTUAL)
	if err != nil {
		b.logger.Error("keyring.SignByAddress failed", "address", address.Hex())
		return nil, err
	}

	signature[crypto.RecoveryIDOffset] += 27 // Transform V from 0/1 to 27/28 according to the yellow paper
	return signature, nil
}
