### Title
Non-EVM-Native Token Balances (IBC/CosmWasm) Permanently Orphaned on Contract Self-Destruct - (File: `x/evm/statedb/statedb.go`)

---

### Summary

When a contract self-destructs, `StateDB.Commit()` only burns the EVM-native denom balance. Non-EVM-native tokens (IBC tokens, CosmWasm bridge tokens, or any other Cosmos bank-module denominations) held by the contract address are explicitly skipped and left as permanently orphaned bank balances with no recovery mechanism.

---

### Finding Description

In `x/evm/statedb/statedb.go`, the `Commit()` function handles self-destructed accounts by burning only the EVM denom and then deleting the account. The code explicitly acknowledges that non-EVM-native tokens are not handled:

```go
// Only the EVM denom is burned here. Non-EVM-native tokens (IBC, CosmWasm
// bridge) held by the destroyed address are not drained and may remain as
// orphaned bank balances.
if remaining := s.keeper.GetBalance(cacheCtx, cosmosAddr, s.evmDenom); remaining.Sign() > 0 {
    coin := sdk.NewCoin(s.evmDenom, sdkmath.NewIntFromBigInt(remaining.ToBig()))
    if _, err := s.keeper.SubBalance(cacheCtx, cosmosAddr, coin); err != nil {
        return errorsmod.Wrap(err, "failed to burn post-selfdestruct balance")
    }
}
if err := s.keeper.DeleteAccount(cacheCtx, obj.Address()); err != nil {
    return errorsmod.Wrap(err, "failed to delete account")
}
writeCache()

After `DeleteAccount` removes the auth account entry, the address no longer has an associated account in the auth module. Any non-EVM-native tokens (IBC tokens, staking rewards, CosmWasm bridge tokens, etc.) that were held by the contract address remain in the bank module but are permanently inaccessible because:

1. The account no longer exists in the auth module (removed by `DeleteAccount`).
2. There is no private key associated with the address (it was a contract).
3. There is no rescue or recovery mechanism anywhere in the EVM module.

`DeleteAccount` only removes auth metadata and contract storage; it never touches the bank module for non-EVM denoms:

```go
// NOTE: balance should be cleared separately
func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
    ...
    // clear storage
    k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool { ... })
    // remove auth account
    k.accountKeeper.RemoveAccount(ctx, acct)
    ...
}
``` [2](#0-1) 

In Cosmos SDK, any address — including EVM contract addresses — can receive arbitrary bank-module tokens via `MsgSend` or IBC transfers. The bank module does not distinguish between EOA and contract addresses. This means a contract can legitimately accumulate IBC tokens, and those tokens are permanently lost upon self-destruct.

---

### Impact Explanation

Any user who sends IBC tokens or other non-EVM-native Cosmos tokens to a contract address that subsequently self-destructs will permanently lose those tokens. The tokens remain in the bank module as orphaned balances with no way to recover them. This constitutes a permanent, irreversible mis-accounting of valid user funds through the EVM state transition path.

This maps to the allowed High impact: **"EVM state transition … bug that permits … valid user funds/fees to be mis-accounted."**

---

### Likelihood Explanation

The entry path is fully unprivileged:

1. A user sends IBC tokens (e.g., `ibc/27394FB092D2ECCD56123C74F36E4C1F926001CEADA9CA97EA622B25F41E5EB2`) to a deployed contract address via a standard `MsgSend` Cosmos transaction.
2. The contract self-destructs (via the `SELFDESTRUCT` / `SELFDESTRUCT6780` EVM opcode, callable by any EVM transaction targeting the contract).
3. `StateDB.Commit()` burns only the EVM denom and removes the account; the IBC tokens remain as an orphaned bank balance.

This can happen accidentally (a user sends IBC tokens to a contract that later self-destructs) or be triggered intentionally (an attacker deploys a contract, induces users to send IBC tokens to it, then calls `selfdestruct`). No privileged role or validator cooperation is required.

---

### Recommendation

In the self-destruct branch of `StateDB.Commit()`, iterate over **all** bank-module balances of the destroyed address — not just the EVM denom — and either burn them or transfer them to a designated recovery address (e.g., the fee collector or a community pool):

```go
// Drain ALL bank balances, not just the EVM denom.
allBalances := s.bankKeeper.GetAllBalances(cacheCtx, cosmosAddr)
if !allBalances.IsZero() {
    if err := s.bankKeeper.SendCoinsFromAccountToModule(cacheCtx, cosmosAddr, authtypes.FeeCollectorName, allBalances); err != nil {
        return errorsmod.Wrap(err, "failed to drain non-EVM balances on selfdestruct")
    }
}
```

Alternatively, expose a governance-controlled rescue function that can recover orphaned balances from addresses with no associated auth account.

---

### Proof of Concept

1. Deploy a contract at address `0xABCD...` on an Ethermint chain that supports IBC.
2. Send IBC tokens (e.g., `100ibc/...`) to `cosmos1...` (the bech32 form of `0xABCD...`) via `MsgSend`.
3. Verify: `bank.GetAllBalances(0xABCD...)` returns `[100ibc/...]`.
4. Submit an EVM transaction calling `SELFDESTRUCT` on the contract.
5. After the block commits, `keeper.GetAccount(ctx, 0xABCD...)` returns `nil` (account deleted).
6. Query: `bank.GetAllBalances(cosmos1...)` still returns `[100ibc/...]` — the tokens are permanently stuck with no account to authorize their movement and no recovery path in the EVM module. [3](#0-2) [4](#0-3)

// DeleteAccount handles contract's suicide call:
// - remove code
// - remove states
// - remove auth account
//
// NOTE: balance should be cleared separately
func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
	cosmosAddr := sdk.AccAddress(addr.Bytes())
	acct := k.accountKeeper.GetAccount(ctx, cosmosAddr)
	if acct == nil {
		return nil
	}

	// NOTE: only Ethereum accounts (contracts) can be selfdestructed
	_, ok := acct.(ethermint.EthAccountI)
	if !ok {
		return errorsmod.Wrapf(types.ErrInvalidAccount, "type %T, address %s", acct, addr)
	}

	// clear storage
	k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool {
		k.SetState(ctx, addr, key, nil)
		return true
	})

	// remove auth account
	k.accountKeeper.RemoveAccount(ctx, acct)

	k.debugLog(ctx, "account suicided",
		"ethereum-address", addr,
		"cosmos-address", cosmosAddr,
	)

	return nil
