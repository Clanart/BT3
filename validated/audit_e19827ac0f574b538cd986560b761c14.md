### Title
Unvalidated `hexutil.Big` Balance in `StateOverride.Apply()` Causes Reachable `uint256.MustFromBig` Panic via Public JSON-RPC — (`rpc/types/types.go`)

### Summary

`StateOverride.Apply()` in `rpc/types/types.go` converts a caller-supplied `hexutil.Big` balance directly with `uint256.MustFromBig`, which panics unconditionally on overflow. Any value larger than 2²⁵⁶−1 is accepted by the JSON decoder but triggers the panic. The identical code path in `SimStateOverride.Apply()` (`rpc/types/simulate.go`) was already fixed to use `uint256.FromBig` with an explicit overflow check and a graceful error return. The unfixed path is reachable by any unprivileged caller through `eth_call`, `eth_estimateGas`, `eth_createAccessList`, and `debug_traceCall`.

### Finding Description

**Root cause — `rpc/types/types.go`, `StateOverride.Apply()`, lines 100–103:**

```go
if account.Balance != nil {
    balance := (*big.Int)(*account.Balance)
    db.SetBalance(addr, *uint256.MustFromBig(balance))   // panics on overflow
}
```

`OverrideAccount.Balance` is typed `**hexutil.Big` (line 173). `hexutil.Big` wraps `big.Int`, which has no upper bound. The JSON decoder accepts any hex-encoded integer, including values ≥ 2²⁵⁶. `uint256.MustFromBig` is defined as:

```go
func MustFromBig(b *big.Int) *Int {
    i, overflow := FromBig(b)
    if overflow { panic("overflow") }
    return i
}
```

**Fixed path — `rpc/types/simulate.go`, `SimStateOverride.Apply()`, lines 155–159:**

```go
u256Balance, overflow := uint256.FromBig((*big.Int)(account.Balance))
if overflow {
    return &InvalidParamsError{Message: fmt.Sprintf("account %s balance overflows uint256", addr.Hex())}
}
stateDB.SetBalance(addr, *u256Balance)
```

The simulation path was corrected; the production `StateOverride` path was not.

**Call chain to the panic:**

1. Attacker sends `eth_call` (or `eth_estimateGas`, `eth_createAccessList`, `debug_traceCall`) with a crafted `stateOverrides` object.
2. `EthCall` in `x/evm/keeper/grpc_query.go` (line 257): `json.Unmarshal(req.Overrides, &overrides)` — succeeds because `hexutil.Big` accepts any hex integer.
3. `cfg.Overrides = &overrides` is set (line 261).
4. `k.ApplyMessageWithConfig(ctx, msg, cfg, false)` is called (line 284); internally it calls `cfg.Overrides.Apply(stateDB)`.
5. `StateOverride.Apply()` reaches `uint256.MustFromBig` with a value ≥ 2²⁵⁶ → **panic**.

The same trigger exists in `EstimateGas` (line 368), `CreateAccessList` (line 968), and `prepareTrace` (line 800) in `x/evm/keeper/grpc_query.go`.

### Impact Explanation

A Go panic in a goroutine that is not covered by a `recover()` terminates the entire process. If the gRPC server does not wrap these handlers with a panic-recovery interceptor, a single crafted RPC call crashes the node, halting the chain. Even with a recovery interceptor the panic propagates through the gRPC stack in an uncontrolled manner and may corrupt in-flight state or leave the stateDB in an inconsistent snapshot. This matches the **Critical** allowed impact: *"Valid unprivileged RPC submission can halt the chain or corrupt committed state."*

### Likelihood Explanation

The entry point is the standard public `eth_call` JSON-RPC method, requiring no authentication, no funds, and no prior on-chain state. The payload is a single JSON field set to a hex integer one bit wider than 256 bits. Any external user or automated script can trigger it repeatedly.

### Recommendation

Replace `uint256.MustFromBig` with `uint256.FromBig` and return a graceful error on overflow, mirroring the already-correct `SimStateOverride.Apply()` implementation:

```go
if account.Balance != nil {
    balance := (*big.Int)(*account.Balance)
    u256Balance, overflow := uint256.FromBig(balance)
    if overflow {
        return fmt.Errorf("account %s balance overflows uint256", addr.Hex())
    }
    db.SetBalance(addr, *u256Balance)
}
```
