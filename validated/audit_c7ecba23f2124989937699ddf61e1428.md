### Title
`deprecated_declared_classes` Hardcoded to Empty in Proposer State Diff Commitment — Wrong Block Hash for Cairo 0 Declare Transactions - (File: `crates/blockifier/src/state/cached_state.rs`)

### Summary

The `From<CommitmentStateDiff> for ThinStateDiff` conversion permanently hardcodes `deprecated_declared_classes: Vec::new()`. This is the direct sequencer analog of the DSClock `wireID == ++next` bug: just as `wireID` is never assigned so every scheduled action overwrites slot 0, the deprecated-class field is never populated from execution so every proposed block silently drops Cairo 0 class declarations from the state-diff commitment. Any block the sequencer proposes that contains a Cairo 0 Declare transaction will carry a wrong `state_diff_commitment` and a wrong `concatenated_counts` (state-diff length), producing a wrong `PartialBlockHash` and ultimately a wrong block hash.

### Finding Description

**Root cause — `crates/blockifier/src/state/cached_state.rs` lines 690–701:** [1](#0-0) 

```rust
impl From<CommitmentStateDiff> for ThinStateDiff {
    fn from(commitment_state_diff: CommitmentStateDiff) -> Self {
        Self {
            deployed_contracts: commitment_state_diff.address_to_class_hash,
            storage_diffs: commitment_state_diff.storage_updates,
            class_hash_to_compiled_class_hash: commitment_state_diff
                .class_hash_to_compiled_class_hash,
            nonces: commitment_state_diff.address_to_nonce,
            // TODO(AlonH): Remove this when the structure of storage diffs changes.
            deprecated_declared_classes: Vec::new(),
        }
    }
}
```

`CommitmentStateDiff` (and the `StateMaps` it is built from) has no field for deprecated class declarations. The blockifier never records Cairo 0 Declare transactions during execution, so the conversion cannot populate `deprecated_declared_classes`.

**Propagation — `crates/apollo_batcher/src/block_builder.rs` lines 160–166:** [2](#0-1) 

`BlockExecutionArtifacts::new()` calls `calculate_block_commitments` with `ThinStateDiff::from(commitment_state_diff.clone())`. That function does two things with the `ThinStateDiff`:

1. **`calculate_state_diff_hash`** — chains `deprecated_declared_classes` into the Poseidon hash: [3](#0-2) 

2. **`state_diff.len()`** — counts `deprecated_declared_classes.len()` for `concatenated_counts`: [4](#0-3) 

Because `deprecated_declared_classes` is always `Vec::new()`, both outputs are wrong whenever a Cairo 0 Declare transaction is included in the block.

**Commitment construction — `crates/starknet_api/src/block_hash/block_hash_calculator.rs` lines 318–357:** [5](#0-4) 

The wrong `state_diff_commitment` and wrong `concatenated_counts` are packed into `BlockHeaderCommitments`, which feeds `PartialBlockHashComponents::new()` and ultimately `calculate_block_hash`. [6](#0-5) 

### Impact Explanation

For every block the sequencer proposes that contains one or more Cairo 0 Declare transactions:

- `state_diff_commitment` is computed without the deprecated class hashes → wrong `StateDiffCommitment` stored in `BlockHeaderCommitments`.
- `state_diff_length` packed into `concatenated_counts` is too small by the number of deprecated declared classes → wrong `concatenated_counts`.
- Both fields are stored in `PartialBlockHashComponents` and committed to storage via `set_partial_block_hash_components`. [7](#0-6) 

The final block hash computed by `finalize_commitment_output` is therefore wrong, breaking the chain's commitment invariant. This matches **Critical — Wrong state/class hash/storage value from blockifier/execution logic for accepted input**.

### Likelihood Explanation

Cairo 0 Declare transactions (version 1) remain valid Starknet protocol transactions. Any unprivileged user can submit one. The gateway accepts it, the blockifier executes it, but the resulting `CommitmentStateDiff` silently omits the declaration. No existing check in the proposer path detects the omission before the wrong commitment is sealed into the block.

### Recommendation

- Add a `deprecated_declared_classes: Vec<ClassHash>` field to `CommitmentStateDiff` (and to the underlying `StateMaps`).
- Populate it during execution of Cairo 0 Declare transactions in the blockifier.
- Remove the hardcoded `Vec::new()` from `From<CommitmentStateDiff> for ThinStateDiff` and derive the field from the execution output.
- Add a unit test that proposes a block containing a Cairo 0 Declare transaction and asserts that `state_diff_commitment` and `state_diff_length` match an independently computed reference value.

### Proof of Concept

1. User submits a Cairo 0 Declare transaction (Declare v1) to the sequencer gateway.
2. The batcher includes it in a block; the blockifier executes it and produces a `CommitmentStateDiff` with no record of the declaration.
3. `BlockExecutionArtifacts::new()` calls `ThinStateDiff::from(commitment_state_diff)` → `deprecated_declared_classes = []`.
4. `calculate_state_diff_hash` hashes `chain_deprecated_declared_classes(&[], ...)` → count field is `0`, class hash is absent.
5. `state_diff.len()` returns `N` instead of `N+1` → `concatenated_counts` encodes the wrong state-diff length.
6. Both wrong values are stored in `PartialBlockHashComponents` and committed.
7. The final block hash diverges from what any correct verifier (or syncing node using `ThinStateDiff::from_state_diff`) would compute, breaking the commitment invariant.

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
