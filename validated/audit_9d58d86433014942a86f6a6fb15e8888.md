### Title
Missing Cross-Type Class Declaration Conflict Check in `unite_state_diffs` Allows Malicious Peer to Inject Corrupted State Diff with Dual-Declared Class Hash â€” (`File: crates/apollo_p2p_sync/src/client/state_diff.rs`)

---

### Summary

`unite_state_diffs` in the P2P sync client checks for intra-type duplicate class declarations but has no cross-type check: a class hash can appear simultaneously in a `DeclaredClass` chunk (Cairo 1, written to `class_hash_to_compiled_class_hash`) and a `DeprecatedDeclaredClass` chunk (Cairo 0, written to `deprecated_declared_classes`). The deferred validator `validate_deprecated_declared_classes_non_conflicting` only checks for duplicates within `deprecated_declared_classes` and never compares against `class_hash_to_compiled_class_hash`. A malicious P2P peer can craft chunks that pass the length gate and both intra-type checks, producing a `ThinStateDiff` with the same class hash in two mutually exclusive fields, which is then written unconditionally to storage.

---

### Finding Description

`parse_data_for_block` in `StateDiffStreamBuilder` assembles a `ThinStateDiff` by calling `unite_state_diffs` for each received `StateDiffChunk`, then calls `validate_deprecated_declared_classes_non_conflicting` on the final result. [1](#0-0) 

Inside `unite_state_diffs`, the three chunk variants are handled asymmetrically:

- `DeclaredClass` â†’ inserts into `class_hash_to_compiled_class_hash` (IndexMap); the `.is_some()` guard immediately returns `ConflictingStateDiffParts` on a duplicate key. [2](#0-1) 

- `DeprecatedDeclaredClass` â†’ **unconditionally pushes** to `deprecated_declared_classes` (Vec); no inline check at all. [3](#0-2) 

The deferred validator only checks that `deprecated_declared_classes` has no internal duplicates by comparing its length to a deduplicated `HashSet`: [4](#0-3) 

There is **no check** that a class hash present in `class_hash_to_compiled_class_hash` is absent from `deprecated_declared_classes`. A class hash cannot legitimately be both a Cairo 1 class and a deprecated Cairo 0 class; these are mutually exclusive declaration types.

`write_to_storage` for `(ThinStateDiff, BlockNumber)` calls `append_state_diff` directly with no further validation: [5](#0-4) 

`append_state_diff` in storage also performs no cross-type check; it writes `class_hash_to_compiled_class_hash` entries to `declared_classes_block_table` and `deprecated_declared_classes` entries to `deprecated_declared_classes_block_table` independently: [6](#0-5) 

---

### Impact Explanation

A malicious P2P peer crafts two chunks for the same class hash `X`:

1. `StateDiffChunk::DeclaredClass { class_hash: X, compiled_class_hash: Y }` â€” length contribution 1
2. `StateDiffChunk::DeprecatedDeclaredClass { class_hash: X }` â€” length contribution 1

By replacing one legitimate entry (e.g., a second `DeclaredClass { Z }`) with `DeprecatedDeclaredClass { X }`, the total chunk length still equals `target_state_diff_len` read from the stored header. The loop exits normally, `validate_deprecated_declared_classes_non_conflicting` passes (no intra-list duplicate), and the corrupted `ThinStateDiff` â€” with `X` in both `class_hash_to_compiled_class_hash` and `deprecated_declared_classes` â€” is committed to storage.

Downstream consequences:

- **Wrong state diff commitment**: `chain_declared_classes` and `chain_deprecated_declared_classes` both include `X` when computing the state diff hash, producing a hash that diverges from the one signed into the block header. [7](#0-6) 

- **Wrong storage values returned by RPC**: `starknet_getStateUpdate` reads the stored `ThinStateDiff` directly; it will report `X` as both a declared Cairo 1 class and a deprecated Cairo 0 class, which is an authoritative-looking wrong value.

- **Wrong class hash / storage value in Patricia trie**: if the syncing node also runs the committer, the classes trie is updated with `X â†’ compiled_class_hash Y` while the deprecated classes table also records `X`, producing an incorrect global state root.

This matches the allowed impact: **High â€” RPC execution returns an authoritative-looking wrong value** and **Wrong state / class hash / storage value from accepted input**.

---

### Likelihood Explanation

The trigger requires only a malicious P2P peer. No privileged role, no administrator action, and no cryptographic break is needed. The peer must craft chunks whose total `len()` matches the header's `state_diff_length`, which is straightforward: substitute one legitimate `DeclaredClass` entry with a `DeprecatedDeclaredClass` carrying the same class hash as an already-sent `DeclaredClass` entry. The existing test suite (`state_diff_conflicting`) does not cover this cross-type case. [8](#0-7) 

---

### Recommendation

Add a cross-type conflict check either inline in `unite_state_diffs` when processing a `DeprecatedDeclaredClass` chunk, or inside `validate_deprecated_declared_classes_non_conflicting` after all chunks are assembled:

```diff
// Option A â€“ inline in unite_state_diffs
StateDiffChunk::DeprecatedDeclaredClass(deprecated_declared_class) => {
+   if state_diff
+       .class_hash_to_compiled_class_hash
+       .contains_key(&deprecated_declared_class.class_hash)
+   {
+       return Err(BadPeerError::ConflictingStateDiffParts);
+   }
    state_diff.deprecated_declared_classes.push(deprecated_declared_class.class_hash);
}

// Option B â€“ in validate_deprecated_declared_classes_non_conflicting
fn validate_deprecated_declared_classes_non_conflicting(
    state_diff: &ThinStateDiff,
) -> Result<(), BadPeerError> {
    let unique_len = state_diff.deprecated_declared_classes.iter()
        .cloned().collect::<HashSet<_>>().len();
    if unique_len != state_diff.deprecated_declared_classes.len() {
        return Err(BadPeerError::ConflictingStateDiffParts);
    }
+   for class_hash in &state_diff.deprecated_declared_classes {
+       if state_diff.class_hash_to_compiled_class_hash.contains_key(class_hash) {
+           return Err(BadPeerError::ConflictingStateDiffParts);
+       }
+   }
    Ok(())
}
```

A corresponding test case should be added to `state_diff_conflicting` covering the cross-type scenario.

// For performance reasons, this function does not check if a deprecated class was declared twice.
// That check is done after we get the final state diff.
#[latency_histogram("p2p_sync_state_diff_unite_state_diffs_latency_seconds", true)]
fn unite_state_diffs(
    state_diff: &mut ThinStateDiff,
    state_diff_chunk: StateDiffChunk,
) -> Result<(), BadPeerError> {
    match state_diff_chunk {
        StateDiffChunk::ContractDiff(contract_diff) => {
            if let Some(class_hash) = contract_diff.class_hash {
                if state_diff
                    .deployed_contracts
                    .insert(contract_diff.contract_address, class_hash)
                    .is_some()
                {
                    return Err(BadPeerError::ConflictingStateDiffParts);
                }
            }
            if let Some(nonce) = contract_diff.nonce {
                if state_diff.nonces.insert(contract_diff.contract_address, nonce).is_some() {
                    return Err(BadPeerError::ConflictingStateDiffParts);
                }
            }
            if !contract_diff.storage_diffs.is_empty() {
                match state_diff.storage_diffs.get_mut(&contract_diff.contract_address) {
                    Some(storage_diffs) => {
                        for (k, v) in contract_diff.storage_diffs {
                            if storage_diffs.insert(k, v).is_some() {
                                return Err(BadPeerError::ConflictingStateDiffParts);
                            }
                        }
                    }
                    None => {
                        state_diff
                            .storage_diffs
                            .insert(contract_diff.contract_address, contract_diff.storage_diffs);
                    }
                }
            }
        }
        StateDiffChunk::DeclaredClass(declared_class) => {
            if state_diff
                .class_hash_to_compiled_class_hash
                .insert(declared_class.class_hash, declared_class.compiled_class_hash)
                .is_some()
            {
                return Err(BadPeerError::ConflictingStateDiffParts);
            }
        }
        StateDiffChunk::DeprecatedDeclaredClass(deprecated_declared_class) => {
            state_diff.deprecated_declared_classes.push(deprecated_declared_class.class_hash);
        }
    }
    Ok(())
}

fn chain_declared_classes(
    declared_classes: &IndexMap<ClassHash, CompiledClassHash>,
    mut hash_chain: HashChain,
) -> HashChain {
    hash_chain = hash_chain.chain(&declared_classes.len().into());
    for (class_hash, compiled_class_hash) in sorted_index_map(declared_classes) {
        hash_chain = hash_chain.chain(&class_hash).chain(&compiled_class_hash.0)
    }
    hash_chain
}

// Chains: [number_of_old_declared_classes, class_hash_0, class_hash_1, ...].
fn chain_deprecated_declared_classes(
    deprecated_declared_classes: &[ClassHash],
    hash_chain: HashChain,
) -> HashChain {
    let mut sorted_deprecated_declared_classes = deprecated_declared_classes.to_vec();
    sorted_deprecated_declared_classes.sort_unstable();
    hash_chain
        .chain(&sorted_deprecated_declared_classes.len().into())
        .chain_iter(sorted_deprecated_declared_classes.iter().map(|class_hash| &class_hash.0))
}
