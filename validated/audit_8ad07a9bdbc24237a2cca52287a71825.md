### Title
Stale `feemarketParams` Snapshot in `NewDynamicFeeChecker` and `MinGasPriceDecorator` Allows Cosmos SDK Transactions to Bypass Current Fee Market Base Fee - (`ante/evm/fee_checker.go`, `evmd/ante/handler_options.go`)

### Summary

`newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712` capture `feemarketParams` (including `BaseFee` and `MinGasPrice`) as a value-copy at construction time and pass a pointer to that copy into `NewDynamicFeeChecker` and `NewMinGasPriceDecorator`. Because the copy is never refreshed from the KV store, every Cosmos SDK transaction processed by those handlers is validated against the base fee that existed when the ante handler was constructed, not the current block's base fee. This is the direct analog of the external report: a stored/snapshot value is used as the enforcement threshold instead of the live value.

### Finding Description

In `evmd/ante/handler_options.go`, `newCosmosAnteHandler` reads params once at construction time:

```go
evmParams := options.EvmKeeper.GetParams(ctx)          // snapshot
feemarketParams := options.FeeMarketKeeper.GetParams(ctx)  // snapshot
...
txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
``` [1](#0-0) 

The pointer `&feemarketParams` points to a local copy that is never updated. `NewDynamicFeeChecker` closes over this pointer and reads `feemarketParams.BaseFee` on every invocation:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
``` [2](#0-1) 

`types.GetBaseFee` calls `feemarketParams.GetBaseFee()`, which returns `params.BaseFee.BigInt()` — the value frozen at construction time, not the live store value. [3](#0-2) 

The same stale pointer is passed to `NewMinGasPriceDecorator`, which reads `mpd.feemarketParams.MinGasPrice` directly: [4](#0-3) 

Meanwhile, `newEthAnteHandler` (for EVM transactions) correctly reads fresh params inside the closure on every call via `EVMBlockConfig`:

```go
blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
feemarketParams := &blockCfg.FeeMarketParams
baseFee := blockCfg.BaseFee
``` [5](#0-4) 

The base fee is updated every block in `BeginBlock` → `SetBaseFee` → `SetParams`, writing the new value to the KV store: [6](#0-5) 

But the snapshot held by the Cosmos ante handler is never refreshed from that store.

### Impact Explanation

Any unprivileged user submitting a Cosmos SDK transaction (non-EVM, e.g. governance, staking, IBC, or any `sdk.Msg`) on a chain where `DynamicFeeChecker` is enabled can pay fees computed against the stale (potentially much lower) base fee. When the base fee rises due to network congestion, these transactions pass fee validation and are committed despite carrying insufficient fees. This mis-accounts fees: the fee collector receives less than the current fee market requires, and the fee market's congestion-control invariant is broken for the Cosmos SDK transaction path. The same stale `MinGasPrice` means governance-raised minimum gas prices are also not enforced until the ante handler is reconstructed.

### Likelihood Explanation

The base fee changes every block. After the first block following ante handler construction, the snapshot is already stale. Any Cosmos SDK transaction submitted to the chain is processed through this path. No special privileges or conditions are required — any user can craft a transaction with fees calibrated to the stale base fee and have it accepted.

### Recommendation

Read `feemarketParams` from the live store inside the returned closure, mirroring the pattern used in `newEthAnteHandler`:

```go
// Inside NewDynamicFeeChecker closure:
feemarketParams := feemarketKeeper.GetParams(ctx)
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, &feemarketParams)
```

Alternatively, pass the `FeeMarketKeeper` interface into `NewDynamicFeeChecker` and `NewMinGasPriceDecorator` so they can call `GetParams(ctx)` on each invocation, exactly as `EVMBlockConfig` does for EVM transactions.

### Proof of Concept

1. Chain starts with `BaseFee = 1_000_000_000` (1 Gwei). `newCosmosAnteHandler` is called once; `feemarketParams.BaseFee = 1_000_000_000` is frozen in the closure.
2. Over subsequent blocks, high gas usage causes `BeginBlock` → `CalculateBaseFee` → `SetBaseFee` to raise the live base fee to `10_000_000_000` (10 Gwei).
3. Attacker submits a Cosmos SDK `MsgDelegate` with `fee = 1_000_000_000 * gasLimit` (1 Gwei price).
4. `NewDynamicFeeChecker` reads `baseFee` from the stale snapshot: `1_000_000_000`. The check `feeCap.LT(baseFeeInt)` passes. The transaction is accepted and committed.
5. The fee collector receives 1 Gwei × gas instead of the required 10 Gwei × gas — a 10× underpayment. The fee market's congestion signal is bypassed for all Cosmos SDK transactions. [7](#0-6) [8](#0-7)

func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {

	}

func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
}
