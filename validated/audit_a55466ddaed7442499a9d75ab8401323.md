### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS accepts a user-supplied `class_hash` from the syscall request and writes it directly into the contract's state entry **without verifying that the class hash corresponds to a previously declared contract class**. This is structurally identical to the external report's root cause: a privileged operation accepts an unvalidated external reference (program ID / class hash) from user-controlled input. A contract can replace its own class with an arbitrary, undeclared hash, permanently rendering itself unexecutable and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class` (lines 878–916 of `syscall_impls.cairo`), the `class_hash` field is read directly from the syscall request and used to overwrite the contract's on-chain class hash with no existence check:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment is an explicit acknowledgment by the development team that this validation is absent.

When any subsequent call targets this contract, `execute_entry_point` resolves the class hash through two steps:

**Step 1** — look up the class hash in `contract_class_changes` to obtain the compiled class hash:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // ← now the attacker-supplied, undeclared hash
);
``` [2](#0-1) 

Because the hash was never declared, `dict_read` returns the default value `0`.

**Step 2** — search the compiled class facts bundle for `compiled_class_hash = 0`:

```cairo
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,   // ← 0, not present
);
``` [3](#0-2) 

`find_element` is a non-optional search; if the key is absent the hint fails and the execution cannot proceed. The contract is permanently unexecutable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any contract that calls `replace_class` with an undeclared class hash becomes permanently inaccessible. All assets held by that contract (native ETH, ERC-20 balances, NFTs, or any other on-chain value) are irreversibly frozen because no future transaction can successfully execute against it. The state change is committed on-chain and cannot be undone without a protocol-level intervention.

---

### Likelihood Explanation

**Medium.**

- `replace_class` is a standard, permissionless syscall available to any Sierra contract.
- A malicious actor can deploy a contract, fund it (or attract user funds into it), and then call `replace_class` with an arbitrary felt value as the class hash.
- Legitimate upgrade-pattern contracts that expose a public or semi-public `upgrade` function are also at risk: if the caller-supplied class hash is not independently validated by the contract before invoking `replace_class`, an attacker can pass an undeclared hash and freeze the contract.
- No privileged role, leaked key, or external dependency is required.

---

### Recommendation

Before writing the new class hash into the contract state, verify that it exists in `contract_class_changes` (i.e., it was previously declared in this or a prior block):

```cairo
let class_hash = request.class_hash;

// Validate that the class hash has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
// A compiled_class_hash of 0 means the class was never declared.
assert_not_zero(compiled_class_hash);

local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}
...
```

This mirrors the pattern already used in `execute_entry_point` and closes the gap the TODO comment identifies.

---

### Proof of Concept

1. **Attacker deploys Contract A** — a Sierra contract that (a) accepts ETH deposits from users and (b) exposes a public `freeze_self` entry point that calls `replace_class(class_hash=0xdeadbeef)` where `0xdeadbeef` is an arbitrary felt that has never been declared.

2. **Users deposit funds** into Contract A (e.g., via a `deposit` entry point), trusting it as a legitimate vault.

3. **Attacker calls `freeze_self`**. The OS executes `execute_replace_class`:
   - `request.class_hash = 0xdeadbeef`
   - No existence check is performed (the TODO confirms this).
   - `contract_state_changes` is updated: Contract A's `class_hash` is now `0xdeadbeef`.
   - The transaction succeeds and is committed in block N.

4. **In block N+1**, any user attempts to call Contract A's `withdraw` entry point:
   - `execute_entry_point` reads `class_hash = 0xdeadbeef` from state.
   - `dict_read{dict_ptr=contract_class_changes}(key=0xdeadbeef)` returns `0` (never declared).
   - `find_element(..., key=0)` fails — no compiled class with hash 0 exists.
   - The call cannot execute. The blockifier rejects it.

5. **All user funds in Contract A are permanently frozen.** No withdrawal, migration, or recovery is possible at the protocol level.
