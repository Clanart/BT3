### Title
Zero-Fee Cosmos Tx Execution via Non-evmDenom Fee Coin When `NoBaseFee=true` ŌĆö (`ante/evm/fee_checker.go`)

---

### Summary

When the chain is configured with `NoBaseFee=true` and London hardfork enabled, `NewDynamicFeeChecker` silently accepts a Cosmos tx whose fee is denominated entirely in a non-evmDenom coin (e.g., `stake`). The checker computes `feeCap=0` and returns `effectiveFee={evmDenom:0}`. The downstream `DeductFeeDecorator` skips deduction because `fee.IsZero()==true`, so the tx executes paying zero fees in any denom.

---

### Finding Description

**Step 1 ŌĆö `NoBaseFee=true` does NOT cause a nil return from `types.GetBaseFee`**

`x/feemarket/types/params.go` `GetBaseFee()` returns `nil` when `NoBaseFee=true`: [1](#0-0) 

But `x/evm/types/utils.go` `GetBaseFee()` wraps that and converts `nil` ŌåÆ `new(big.Int)` (i.e., 0) when London is enabled: [2](#0-1) 

So in `NewDynamicFeeChecker`, the `baseFee == nil` guard at line 57 is **not triggered** ŌĆö the code proceeds into the EIP-1559 path with `baseFee = 0`: [3](#0-2) 

**Step 2 ŌĆö Non-evmDenom fee yields `feeCap=0`**

```
feeCoins = [{stake: 1000}]
fee      = feeCoins.AmountOf("aphoton") = 0
feeCap   = 0 / gas = 0
``` [4](#0-3) 

**Step 3 ŌĆö `feeCap(0) >= baseFee(0)` passes**

The only rejection guard is `feeCap.LT(baseFeeInt)`. With both equal to 0, this is `false`, so no error is returned: [5](#0-4) 

**Step 4 ŌĆö `effectiveFee = {evmDenom: 0}` is returned**

`EffectiveGasPrice(0, 0, maxPriorityPrice)` = `min(maxPriorityPrice+0, 0)` = 0. The returned coins slice is `{evmDenom: 0}`: [6](#0-5) 

**Step 5 ŌĆö `DeductFeeDecorator` skips deduction**

`checkDeductFee` is called with `fee = {evmDenom: 0}`. `fee.IsZero()` returns `true`, so `DeductFees` is never called: [7](#0-6) 

The original `{stake: 1000}` fee declared in the tx is never deducted by any decorator in the Cosmos ante chain: [8](#0-7) 

**Step 6 ŌĆö `MinGasPriceDecorator` also short-circuits**

The only other fee guard short-circuits when `minGasPrice.IsZero()`, which is the default (`MinGasPrice=0`): [9](#0-8) 

---

### Impact Explanation

An unprivileged attacker can submit any Cosmos message (bank send, staking, governance, IBC, etc.) with zero effective fees. The fee collector receives nothing. This enables:
- **Fee bypass**: executing arbitrary Cosmos messages without paying the required evmDenom fee
- **Spam/DoS**: flooding the mempool and block space at zero cost, degrading chain liveness

This matches the allowed High impact: *"ante handler bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

---

### Likelihood Explanation

`NoBaseFee=true` is an explicitly documented, supported configuration ("needed for 0 price calls"). Many Ethermint-based chains use it during early deployment or for specific use cases. `MinGasPrice=0` is the default. London hardfork is enabled by default. All three preconditions are simultaneously satisfied in a standard Ethermint deployment with `NoBaseFee=true`.

---

### Recommendation

In `NewDynamicFeeChecker`, after computing `fee = feeCoins.AmountOf(denom)`, explicitly reject the tx if the fee is not denominated in evmDenom:

```go
// Reject if no evmDenom fee is provided
if feeCoins.Len() > 0 && fee.IsZero() {
    return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee,
        "fee must be paid in %s; got %s", denom, feeCoins)
}
```

Alternatively, validate that `feeCoins` contains only the evmDenom before proceeding with the EIP-1559 path.

---

### Proof of Concept

```go
// Preconditions: NoBaseFee=true, MinGasPrice=0, London HF enabled
feemarketParams := feemarkettypes.Params{
    NoBaseFee:   true,
    MinGasPrice: sdkmath.LegacyZeroDec(),
    BaseFee:     sdkmath.ZeroInt(),
}
evmParams := types.DefaultParams() // EvmDenom = "aphoton"

// Attacker tx: fee in "stake", not "aphoton"
txBuilder := encodingConfig.TxConfig.NewTxBuilder()
txBuilder.SetGasLimit(100_000)
txBuilder.SetFeeAmount(sdk.NewCoins(sdk.NewCoin("stake", sdkmath.NewInt(1_000_000))))
tx := txBuilder.GetTx()

fees, _, err := NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)(deliverCtx, tx)
// err == nil  (passes!)
// fees == [{Denom:"aphoton", Amount:0}]
// fee.IsZero() == true ŌåÆ DeductFees never called
// fee_collector aphoton balance: unchanged
```
