### Title
`ProofOfInclusion::valid()` Tautological Root Check Allows Forged Inclusion Proofs — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate performs only an internal chain-consistency check. Its final comparison is a tautology: after the loop, `existing_hash` is always equal to `self.root_hash()` when the loop passes, because `root_hash()` returns `last.combined_hash` — the exact value `existing_hash` was set to in the last iteration. No external trusted root is ever consulted. An attacker can construct any internally-consistent `ProofOfInclusion` from untrusted bytes, call `valid()`, and receive `true`, regardless of whether the claimed leaf is actually present in the real DataLayer tree.

### Finding Description

`ProofOfInclusion::valid()` is defined as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()   // ← tautology
}
```

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← sourced from the proof itself
    } else {
        self.node_hash
    }
}
```

After the loop completes without returning `false`, `existing_hash` holds the last `calculated_hash`, which equals `layer.combined_hash` for the last layer (the loop would have returned `false` otherwise). `self.root_hash()` returns that same `last.combined_hash`. Therefore `existing_hash == self.root_hash()` is unconditionally `true` whenever the loop passes — the final line adds zero security.

The struct is fully Streamable and exposed to Python:

```rust
#[derive(Clone, Debug, std::hash::Hash, Eq, PartialEq, Streamable)]
pub struct ProofOfInclusion {
    pub node_hash: Hash,
    pub layers: Vec<ProofOfInclusionLayer>,
}
```

`from_bytes`, `from_bytes_unchecked`, `from_json_dict`, and `parse_rust` are all available to Python callers, meaning an attacker can supply arbitrary bytes that deserialize into a `ProofOfInclusion` and then call `valid()` on it.

### Impact Explanation

Any Python or Rust consumer that calls `proof.valid()` as the sole check for DataLayer inclusion accepts forged proofs. The function name and signature (`fn valid(&self) -> bool`) imply a complete validity check. There is no parameter for an external trusted root, and the Python type stub exposes `valid()` and `root_hash()` as separate, independent methods with no documentation requiring callers to also compare `root_hash()` against a known store root. A verifier that omits the `root_hash()` comparison will accept a proof claiming any leaf is in any tree, enabling forged inclusion proofs against DataLayer state.

This matches the allowed impact: **High — DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, or lets untrusted input prove invalid state.**

### Likelihood Explanation

- `ProofOfInclusion` is a Streamable type fully deserializable from attacker-controlled bytes via `from_bytes()` / `from_json_dict()`.
- The Python API (`wheel/python/chia_rs/datalayer.pyi`) exposes `valid()` and `root_hash()` as separate methods with no guidance that both must be used together.
- The function name `valid()` strongly implies a complete check, making it likely that callers omit the separate `root_hash()` comparison.
- No privilege is required; any network peer can supply a crafted `ProofOfInclusion`.

### Recommendation

1. **Add an external root parameter to `valid()`**: Change the signature to `fn valid(&self, expected_root: &Hash) -> bool` and compare `existing_hash` against `expected_root` instead of `self.root_hash()`. This makes it impossible to call `valid()` without supplying a trusted root.
2. **Alternatively**, rename the current function to `is_internally_consistent()` and add a separate `verify_against_root(root: &Hash) -> bool` that performs the external comparison.
3. **Update the Python bindings** to reflect the new signature and add documentation making the required usage pattern explicit.
4. **Update the fuzz target** (`crates/chia-datalayer/fuzz/fuzz_targets/proofs_of_inclusion.rs`) to also verify `proof.root_hash() == blob.get_root_hash()` after `proof.valid()`.

### Proof of Concept

```rust
use chia_datalayer::{Hash, KeyId, MerkleBlob, Side, ValueId};
use chia_datalayer::merkle::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};
use chia_protocol::Bytes32;

fn forged_proof_passes_valid() {
    // Attacker-chosen arbitrary hashes — not from any real MerkleBlob
    let fake_leaf_hash = Hash(Bytes32::new([0xAA; 32]));
    let fake_sibling   = Hash(Bytes32::new([0xBB; 32]));

    // Compute a combined_hash that is internally consistent
    let combined = chia_datalayer::calculate_internal_hash(
        &fake_leaf_hash,
        Side::Left,
        &fake_sibling,
    );

    let forged = ProofOfInclusion {
        node_hash: fake_leaf_hash,
        layers: vec![ProofOfInclusionLayer {
            other_hash_side: Side::Left,
            other_hash: fake_sibling,
            combined_hash: combined,   // attacker sets this to match
        }],
    };

    // valid() returns true even though this proof was never generated
    // from any real MerkleBlob and proves nothing about any real tree.
    assert!(forged.valid());   // ← passes; tautology confirmed

    // The "root" this proof claims is entirely attacker-controlled:
    assert_eq!(forged.root_hash(), combined);
}
```

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
