### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Bricking - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS accepts any arbitrary felt value as a new class hash without verifying that the hash corresponds to a previously declared class. This is the direct analog of the reported "failure to validate initialization" pattern: an uninitialized/undeclared slot is accepted as valid, leading to an irrecoverably invalid contract state. A contract that replaces its class with an undeclared hash (including `0 = UNINITIALIZED_CLASS_HASH`) becomes permanently non-executable, freezing any funds it holds.

---

### Finding Description

In `execute_replace_class`, the OS reads `request.class_hash` from the syscall buffer and immediately writes it into `contract_state_changes` with no check that the value exists in `contract_class_changes`:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-inserted `TODO` comment explicitly acknowledges the missing check. [1](#0-0) 

By contrast, `deploy_contract` correctly enforces that the target address is uninitialized before writing a class hash:

```cairo
assert state_entry.class_hash = UNINITIALIZED_CLASS_HASH;
assert state_entry.nonce = 0;
``` [2](#0-1) 

And `execute_declare_transaction` enforces that a class can only be declared once by using `prev_value=0`:

```cairo
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [3](#0-2) 

`execute_replace_class` performs neither check. `UNINITIALIZED_CLASS_HASH = 0` is defined as the sentinel for a non-existent class: [4](#0-3)

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` is set to an undeclared value (e.g., `0` or any felt not present in `contract_class_changes`), the OS has no entry-point table to dispatch calls against. Every future invocation of the contract fails at the class-lookup stage. There is no upgrade path because `replace_class` itself requires the contract to be callable. Any ERC-20 tokens, ETH, STRK, or other assets held in the contract's storage are permanently inaccessible — the contract is bricked with no recovery mechanism.

---

### Likelihood Explanation

The `replace_class` syscall is a standard, permissionless operation available to any Sierra contract. The attacker-controlled path requires only:

1. Deploy a contract (permissionless, any user can do this).
2. Have the contract call `replace_class` with `class_hash = 0` or any undeclared felt.
3. The OS OS Cairo program accepts the call, writes the invalid class hash into `contract_state_changes`, and the state transition is proven and finalized on L1.

No privileged role, leaked key, or operator cooperation is required. The attack is reachable from a standard `INVOKE_FUNCTION` transaction sent by any unprivileged L2 user. A malicious contract author could deploy a vault-like contract, attract user deposits, then call `replace_class(0)` to permanently freeze all deposited funds.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, assert that the hash is present in `contract_class_changes` (i.e., it was previously declared). Concretely, perform a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert the returned compiled class hash is non-zero:

```cairo
// Assert the new class hash is a declared class.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the invariant already enforced by `execute_declare_transaction` (`prev_value=0` / `assert_not_zero(compiled_class_hash)`) and closes the gap identified by the existing TODO comment.

---

### Proof of Concept

1. Attacker deploys contract `VaultTrap` with a legitimate class hash `C`.
2. Users deposit funds into `VaultTrap` (storage balances updated).
3. Attacker sends an `INVOKE_FUNCTION` transaction calling `VaultTrap.__execute__`, which internally calls the `replace_class` syscall with `class_hash = 0`.
4. `execute_replace_class` runs: no declared-class check is performed; `contract_state_changes` is updated with `StateEntry(class_hash=0, ...)` for `VaultTrap`'s address.
5. The OS proves the block; the state root on L1 now reflects `VaultTrap.class_hash = 0`.
6. Any subsequent call to `VaultTrap` attempts to dispatch against class `0`, which has no entry points. All calls revert permanently.
7. All user funds stored in `VaultTrap`'s storage are permanently frozen with no recovery path. [6](#0-5)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),

    );

    let revert_log = &revert_log[1];

    return ();
}
