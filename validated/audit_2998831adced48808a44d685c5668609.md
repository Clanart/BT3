### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Contract Bricking — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS Cairo program accepts any attacker-supplied class hash in the `replace_class` syscall without verifying that the hash corresponds to a declared contract class. A malicious contract deployer can replace a contract's class with an undeclared hash, permanently bricking the contract and freezing all funds it holds. The same gap exists in the deprecated syscall path.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` processes the `replace_class` syscall by directly writing the caller-supplied `class_hash` into `contract_state_changes` with no check that the hash is declared on-chain. The code itself contains an explicit acknowledgment of the missing guard: [1](#0-0) 

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The identical omission exists in the deprecated path: [2](#0-1) 

Once the state is committed with an undeclared class hash, any future call to the affected contract reaches `execute_entry_point`, which performs: [3](#0-2) 

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
```

Because the class hash is undeclared, `dict_read` returns the default value `0`, and `find_element` with key `0` panics when no compiled class with that hash exists. The OS cannot prove any block that includes a call to the bricked contract. The contract becomes permanently unexecutable.

The vulnerability class maps directly to the external report: a state transition (analogous to transferring veNFT ownership to the AutoCompounder) is accepted by the OS without validating that the resulting state preserves access to the contract's functionality. Just as the AutoCompounder permanently removes delegation and governance capabilities with no opt-out, `replace_class` with an undeclared hash permanently removes all entry-point dispatch with no recovery path.

---

### Impact Explanation

Any ERC-20 tokens, ETH, or other assets stored in the bricked contract's storage become permanently inaccessible. No valid entry point can be dispatched because `find_element` panics on the undeclared hash. This satisfies **Critical. Permanent freezing of funds.**

---

### Likelihood Explanation

The attack is reachable by any unprivileged contract deployer:

1. Deploy a contract whose logic calls `replace_class(undeclared_hash)`.
2. Submit an invoke transaction triggering that function.
3. The OS Cairo code has no guard; the state is committed with the undeclared hash.
4. All subsequent calls to the contract cause `find_element` to panic.

The missing validation is explicitly acknowledged in the production source with a TODO dated 2026, confirming the gap is known and currently unaddressed in the OS layer. The `replace_class` syscall is available to any Cairo 1 contract, making the entry path fully unprivileged.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, validate that it corresponds to a declared class. Concretely, perform a lookup in `contract_class_changes` (or the compiled class facts bundle) and assert a non-zero compiled class hash is returned:

```cairo
// Proposed guard in execute_replace_class:
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

Apply the same guard to the deprecated `execute_replace_class` in `deprecated_execute_syscalls.cairo`.

---

### Proof of Concept

1. Attacker deploys `MaliciousContract` containing an external function that executes `replace_class(0xDEADBEEF)`, where `0xDEADBEEF` is never declared.
2. Attacker submits an invoke transaction calling that function. The OS processes `execute_replace_class`:
   - `class_hash = 0xDEADBEEF` is written into `contract_state_changes` with no validation.
   - The block is proven successfully; the state is committed.
3. In any subsequent block, a transaction calls `MaliciousContract`.
4. `execute_entry_point` reads `class_hash = 0xDEADBEEF` from state, calls `dict_read` on `contract_class_changes` → returns `0`, then calls `find_element` with key `0`.
5. No compiled class with hash `0` exists → `find_element` panics → the OS cannot prove the block containing that call.
6. The contract is permanently unexecutable; all funds in its storage are frozen with no recovery mechanism.
