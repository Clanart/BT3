### Title
Missing Declared-Class Prerequisite Check in `execute_replace_class` Allows Permanent Contract Freezing - (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the supplied `class_hash` corresponds to a previously declared class before committing the state update. This is a direct analog of the external report's vulnerability class: a state-transition is permitted without satisfying a required prerequisite. Any contract can call `replace_class` with an arbitrary, undeclared class hash, permanently corrupting its own state entry and freezing all funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the function `execute_replace_class` (the non-deprecated path, invoked for Cairo 1 contracts via `execute_syscalls.cairo`) reads the caller-supplied `class_hash` from the syscall request and immediately writes it into `contract_state_changes` with no validation:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-acknowledged TODO at line 898 confirms the missing check. The identical omission exists in the deprecated path at `deprecated_execute_syscalls.cairo` lines 307–329, which also performs no existence check.

The prerequisite that must hold — but is never enforced by the OS — is: **the new class hash must already exist in `contract_class_changes`** (i.e., it must have been declared via a `Declare` transaction). This is exactly the same structural gap as the original report: an action (replace class) is permitted without satisfying a required prior state (class declaration).

When any subsequent call is made to the affected contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
```

If `class_hash` was never declared, `dict_read` returns 0 (the default for an uninitialized dict entry). The OS then calls `find_element` searching for a compiled class with hash `0`. If no such class exists in `compiled_class_facts_bundle`, the Cairo assertion inside `find_element` fails, making the block unprovable for any transaction that touches the affected contract.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` field in `contract_state_changes` is set to an undeclared hash, no future transaction can successfully execute any entry point of that contract. The OS will fail to resolve the compiled class, making any block containing such a call unprovable. All assets (tokens, ETH, etc.) held in the contract's storage become permanently inaccessible. The state commitment will include the corrupted class hash, and there is no recovery path within the protocol.

---

### Likelihood Explanation

The `replace_class` syscall is a standard, publicly available syscall callable by any deployed contract. Any contract that implements an upgrade mechanism using `replace_class` is exposed. The attack requires only that the new class hash supplied to `replace_class` has not yet been declared. This can occur:

1. **Accidentally**: A contract calls `replace_class` with a class hash that was declared in a later transaction in the same block (ordering dependency), or with a hash that was never declared due to a bug.
2. **Deliberately**: An attacker who controls a contract (or can influence its upgrade logic) supplies an arbitrary undeclared hash.

The OS is the enforcement layer for the proof system. Because it does not enforce the prerequisite, neither honest nor malicious sequencers are constrained by the proof from committing this invalid state.

---

### Recommendation

In `execute_replace_class` (both `syscall_impls.cairo` and `deprecated_execute_syscalls.cairo`), add a check that the supplied `class_hash` exists in `contract_class_changes` before updating the state entry. Concretely, perform a `dict_read` on `contract_class_changes` with `key=class_hash` and assert the returned compiled class hash is non-zero:

```cairo
// Verify the class has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the invariant enforced during `execute_declare_transaction`, where `prev_value=0` is asserted to ensure a class is declared exactly once, and `assert_not_zero(compiled_class_hash)` is called before writing.

---

### Proof of Concept

1. Deploy contract `A` (a vault holding user funds) with a valid declared class hash `H_old`.
2. From within contract `A`, call the `replace_class` syscall with `class_hash = 0xDEAD` (an arbitrary value never declared via any `Declare` transaction).
3. The OS executes `execute_replace_class`:
   - Line 896: `let class_hash = request.class_hash;` → `0xDEAD`
   - Line 898: The TODO comment confirms no declared-class check is performed.
   - Lines 902–910: `contract_state_changes` is updated; contract `A`'s state entry now has `class_hash = 0xDEAD`.
4. In any subsequent block, a user submits a transaction calling any entry point of contract `A`.
5. The OS executes `execute_entry_point` (line 154): `dict_read{dict_ptr=contract_class_changes}(key=0xDEAD)` → returns `0` (undeclared).
6. `find_element` is called with `key=0` on `compiled_class_facts_bundle.compiled_class_facts`. No such entry exists → Cairo assertion failure → block is unprovable.
7. All funds in contract `A` are permanently frozen.
