### Title
`ProofOfInclusion::valid()` Checks Proof Against Its Own Claimed Root, Not a Trusted External Root - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` is intended to verify that a DataLayer Merkle inclusion proof is valid against the actual tree root. Instead, its final check compares the computed hash against `self.root_hash()`, which is derived from the proof's own data (`last.combined_hash`). This makes the final check a tautology — it is always `true` after the loop — and the function never validates the proof against any external trusted root. An attacker who can supply a `ProofOfInclusion` struct can forge a self-consistent proof for any key-value pair and have `valid()` return `true`.

### Finding Description

`ProofOfInclusion::valid()` operates as follows: [1](#0-0) 

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()
}
```

And `root_hash()` is: [2](#0-1) 

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash
    }
}
```

**The tautology:** After the loop, `existing_hash` holds `calculated_hash` from the last iteration. The loop already enforced `calculated_hash == layer.combined_hash` (or returned `false`). Therefore `existing_hash == last.combined_hash`. But `self.root_hash()` also returns `last.combined_hash`. The final check `existing_hash == self.root_hash()` is always `true` when the loop completes — it checks the proof's own claimed root against itself.

The function only verifies **internal self-consistency** of the proof chain. It never compares the proof's root against any externally trusted root hash. A correct implementation would require a trusted root parameter, analogous to the Python reference: [3](#0-2) 

```python
def _confirm(root: bytes32, val: bytes, proof: bytes) -> bool:
    p = deserialize_proof(proof)
    if p.get_root() != root:   # <-- external trusted root compared here
        return False
    ...
```

The Rust `valid()` has no equivalent external root parameter.

### Impact Explanation

Any caller that relies solely on `proof.valid()` to authenticate a `ProofOfInclusion` received from an untrusted source (e.g., a DataLayer peer) will accept forged proofs. An attacker constructs a `ProofOfInclusion` with an arbitrary `node_hash` and a chain of `ProofOfInclusionLayer` values where each `combined_hash` is correctly computed from the previous hash and a chosen `other_hash`. The chain is internally consistent, so `valid()` returns `true`, yet the proof's root does not correspond to the actual committed tree root. This allows an untrusted party to prove inclusion of arbitrary key-value pairs that are not in the tree, matching the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, or lets untrusted input prove invalid state**. ### Likelihood Explanation

`ProofOfInclusion` and its `valid()` method are exposed via Python bindings: [4](#0-3) 

Any Python-layer DataLayer code that calls `proof.valid()` on a peer-supplied proof without separately comparing `proof.root_hash()` against a locally trusted root is vulnerable. The misleading name `valid()` and the presence of the tautological final check (which looks like a root comparison) make it likely that callers treat `valid()` as a complete verification.

### Recommendation

`valid()` must accept a trusted external root hash and compare against it:

```rust
pub fn valid(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root  // compare against externally trusted root
}
```

All call sites (Python bindings, fuzz targets, tests) must be updated to supply the trusted root from the locally committed `MerkleBlob`. [5](#0-4) 

### Proof of Concept

1. Obtain any valid `ProofOfInclusion` for key K in tree T (root R₁).
2. Construct a forged `ProofOfInclusion` for a key K′ not in T:
   - Set `node_hash` = SHA256(K′ || fake_value).
   - Build one layer: `other_hash` = any 32-byte value, `other_hash_side` = Left, `combined_hash` = `calculate_internal_hash(node_hash, Left, other_hash)`.
3. Call `forged_proof.valid()` → returns `true`.
4. Call `forged_proof.root_hash()` → returns a root R₂ ≠ R₁, but the caller using only `valid()` never detects the mismatch. [1](#0-0) [6](#0-5)

        parents_iter.next();

                return Err(Error::Dirty(*next_index));
            }
            let parent = block

                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;

                other_hash: sibling.hash(),
