### Title
Flat Gas Cost for `send_message_to_l1` Does Not Account for Variable-Length Payload `memcpy`, Enabling Unbounded OS Prover Resource Consumption — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_send_message_to_l1` OS handler charges a flat, fixed gas cost (`SEND_MESSAGE_TO_L1_GAS_COST = 14470`) regardless of the message payload size, but then performs an unbounded `memcpy` proportional to the attacker-controlled `payload_size`. This creates a gap between the gas charged to the user and the actual OS-level proving work performed. A malicious contract can craft a transaction that sends a message to L1 with an arbitrarily large payload, consuming excessive Cairo steps in the OS prover without paying proportional gas, potentially making the block unprovable and halting the network.

---

### Finding Description

In `execute_send_message_to_l1`:

```cairo
func execute_send_message_to_l1{...}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, SendMessageToL1Request*);
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=SEND_MESSAGE_TO_L1_GAS_COST,   // flat: 14470
        request_struct_size=SendMessageToL1Request.SIZE
    );
    if (success == FALSE) { return (); }

    tempvar payload_start = request.payload_start;
    tempvar payload_size = request.payload_end - payload_start;  // attacker-controlled

    assert [outputs.messages_to_l1] = MessageToL1Header(..., payload_size=payload_size);
    memcpy(
        dst=outputs.messages_to_l1 + MessageToL1Header.SIZE,
        src=payload_start,
        len=payload_size   // unbounded OS-level work
    );
    ...
}
``` [1](#0-0) 

The gas charged is the constant `SEND_MESSAGE_TO_L1_GAS_COST = 14470`: [2](#0-1) 

There is **no upper-bound check** on `payload_size` and **no per-element gas charge** for the payload copy. The `memcpy` executes entirely within the OS program, consuming Cairo steps that are not accounted for in the user's L2 gas budget.

**Contrast with `execute_keccak`**, which correctly charges proportional to input size:

```cairo
let required_gas = KECCAK_GAS_COST + q * KECCAK_ROUND_COST_GAS_COST;
``` [3](#0-2) 

Also contrast with `DEPLOY_CALLDATA_FACTOR_GAS_COST` and `META_TX_V0_CALLDATA_FACTOR_GAS_COST`, which show the codebase already recognizes the need for per-element gas in variable-length syscalls: [4](#0-3) 

The `send_message_to_l1` syscall is the only variable-length syscall that omits this per-element factor.

---

### Impact Explanation

**Impact: High — Network not being able to confirm new transactions (total network shutdown).**

The OS prover runs the Cairo program in `os.cairo` to produce a STARK proof for each block. The `memcpy` inside `execute_send_message_to_l1` executes as Cairo steps within the OS program itself — not within the user contract's gas-metered execution. If `payload_size` is large enough, the OS program's step count can exceed the prover's block step limit, making the block unprovable. The sequencer cannot finalize the block on L1, and if it lacks proper detection/exclusion logic for such transactions, block production stalls. An attacker who repeatedly submits such transactions can sustain the disruption, effectively halting the network's ability to confirm new transactions.

---

### Likelihood Explanation

**Likelihood: Low.**

An attacker must deploy a contract and submit a transaction that calls `send_message_to_l1` with a very large payload. The Sierra/CASM execution of the contract does charge L2 gas for constructing the payload array, so the payload size is indirectly bounded by `EXECUTE_MAX_SIERRA_GAS = 1_100_000_000`. However, the OS-level `memcpy` cost is not reflected in that budget. The gap between Sierra gas consumed (for array construction) and OS prover steps consumed (for `memcpy`) is the exploitable surface. The attack requires a funded account and knowledge of the gas accounting gap, but no privileged access. [5](#0-4) 

---

### Recommendation

Introduce a per-element gas factor for the `send_message_to_l1` payload, analogous to `DEPLOY_CALLDATA_FACTOR_GAS_COST`. The total gas cost should be:

```
SEND_MESSAGE_TO_L1_GAS_COST + payload_size * SEND_MESSAGE_TO_L1_PAYLOAD_FACTOR_GAS_COST
```

This mirrors the pattern already used for `deploy` and `meta_tx_v0` syscalls and ensures the OS-level `memcpy` work is fully covered by the user's gas budget.

---

### Proof of Concept

1. Deploy a contract with an entry point that calls `send_message_to_l1` with a payload of `N` felts (e.g., `N = 500_000`).
2. Submit a transaction invoking that entry point. The Sierra execution charges gas for building the `N`-element array — within `EXECUTE_MAX_SIERRA_GAS`.
3. The OS handler `execute_send_message_to_l1` charges only the flat `14470` gas, then calls `memcpy(..., len=N)`, executing `N` additional Cairo steps inside the OS program.
4. If `N` is large enough to push the OS program's total step count past the prover limit, the block proof fails.
5. The sequencer cannot submit the block to L1; no transactions in that block are confirmed.
6. Repeating this attack across successive blocks sustains the network halt. [6](#0-5)

// Sends a message to L1.
func execute_send_message_to_l1{range_check_ptr, syscall_ptr: felt*, outputs: OsCarriedOutputs*}(
    contract_address: felt
) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, SendMessageToL1Request*);
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=SEND_MESSAGE_TO_L1_GAS_COST, request_struct_size=SendMessageToL1Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    tempvar payload_start = request.payload_start;
    tempvar payload_size = request.payload_end - payload_start;

    assert [outputs.messages_to_l1] = MessageToL1Header(
        from_address=contract_address, to_address=request.to_address, payload_size=payload_size
    );
    memcpy(
        dst=outputs.messages_to_l1 + MessageToL1Header.SIZE, src=payload_start, len=payload_size
    );
    let (outputs) = os_carried_outputs_new(
        messages_to_l1=outputs.messages_to_l1 + MessageToL1Header.SIZE + payload_size,
        messages_to_l2=outputs.messages_to_l2,
    );

    return ();
}
