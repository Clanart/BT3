### Title
`ProofOfInclusion::valid()` Does Not Verify Root Hash Against a Trusted/Expected Value — Forged DataLayer Inclusion Proof Accepted — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

---

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate only checks the internal hash-chain consistency of the proof. It derives the root hash from the proof itself and never compares it against any externally-known, trusted tree root. A caller who receives a `ProofOfInclusion` from an untrusted source and calls only `valid()` will accept any internally-consistent proof, regardless of which tree it actually proves membership in. This is the direct analog of the Linea ENS bug: just as `getStorageValues` accepted whatever block number the gateway supplied (only checking it was non-zero), `valid()` accepts whatever root hash the proof encodes (only checking internal consistency).

---

### Finding Description

`ProofOfInclusion` is a `Streamable` struct exposed through Python bindings. It can be deserialized from bytes received over the network. [1](#0-0) 

Its `root_hash()` method derives the root entirely from the proof's own last layer:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← comes from the proof itself, not from a trusted source
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

`valid()` then checks that each layer's `combined_hash` is correctly computed from the previous hash and `other_hash`, and finally asserts `existing_hash == self.root_hash()`:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(...);
        if calculated_hash != layer.combined_hash { return false; }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()   // ← tautological: root_hash() IS existing_hash
}
``` [3](#0-2) 

The final comparison `existing_hash == self.root_hash()` is tautological: `root_hash()` returns `last.combined_hash`, which is exactly the value `existing_hash` was just set to. The check never involves any externally-supplied, trusted root hash. A proof is "valid" if and only if its own internal hash chain is self-consistent — which an attacker can trivially guarantee for any tree they construct.

**Contrast with the consensus Merkle set verifier**, which correctly requires the caller to supply the expected root:

```rust
pub fn validate_merkle_proof(proof: &[u8], item: &[u8; 32], root: &[u8; 32]) -> Result<bool, SetError> {
    let tree = MerkleSet::from_proof(proof)?;
    if tree.get_root() != *root { return Err(SetError); }   // ← trusted root compared here
    Ok(tree.generate_proof(item)?.0)
}
``` [4](#0-3) 

The DataLayer `ProofOfInclusion::valid()` has no equivalent check.

Every call site in the codebase — including the fuzz target and the Rust/Python tests — calls only `proof.valid()` without comparing `proof.root_hash()` against the known tree root: [5](#0-4) [6](#0-5) 

The Python bindings expose `valid()` and `root_hash()` as separate, independent methods with no guidance that both must be used together: [7](#0-6) 

---

### Impact Explanation

An attacker operating a DataLayer server can:

1. Build an attacker-controlled Merkle tree containing a fabricated key-value pair (e.g., claiming a coin record, ownership record, or any DataLayer state).
2. Call `MerkleBlob::get_proof_of_inclusion()` on their own blob to produce a structurally valid `ProofOfInclusion` for that fabricated entry.
3. Serialize it via `Streamable` and send it to a verifying client.
4. The client calls `proof.valid()` → `true`, and accepts the forged state as proven.

The proof's `root_hash()` will be the attacker's tree root, not the on-chain committed root — but since `valid()` never checks against the on-chain root, the forgery passes. This allows untrusted input to prove invalid DataLayer state, matching the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion, letting untrusted input prove invalid state.**

---

### Likelihood Explanation

The `ProofOfInclusion` struct is `Streamable` and fully exposed through Python bindings with `from_bytes()` deserialization. The API design actively encourages the incomplete pattern: `valid()` is named and documented as the verification method, with no parameter for an expected root and no warning that root-hash comparison is also required. Any DataLayer client that follows the natural API usage pattern is vulnerable. The fuzz target and all existing tests demonstrate this exact incomplete pattern.

---

### Recommendation

`valid()` should accept the expected trusted root hash as a mandatory parameter and compare against it:

```rust
pub fn valid(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root   // ← compare against the caller-supplied trusted root
}
```

This mirrors the correct pattern already used in `validate_merkle_proof` in `crates/chia-consensus/src/merkle_tree.rs`. [4](#0-3) 

Alternatively, rename the current `valid()` to `internally_consistent()` and add a separate `verify(expected_root: &Hash) -> bool` that performs both checks, making the API impossible to misuse.

use chia_datalayer::{Hash, InsertLocation, KeyId, MerkleBlob, ValueId};

// Attacker builds a fake tree with a fabricated entry
let mut attacker_blob = MerkleBlob::new(Vec::new()).unwrap();
let fake_key = KeyId(0xdeadbeef);
let fake_value = ValueId(0xcafebabe);
let fake_hash: Hash = [0xAA; 32];
attacker_blob.insert(fake_key, fake_value, &fake_hash, InsertLocation::Auto {}).unwrap();
attacker_blob.calculate_lazy_hashes().unwrap();

// Attacker generates a proof for the fabricated entry
let forged_proof = attacker_blob.get_proof_of_inclusion(fake_key).unwrap();

// Victim's on-chain committed root is completely different
let real_on_chain_root: Hash = [0x00; 32];

// Victim calls valid() — returns true, forgery accepted
assert!(forged_proof.valid());

// The proof's root does NOT match the on-chain root — but valid() never checks this
assert_ne!(forged_proof.root_hash(), real_on_chain_root);

// A correct check would be:
// assert_eq!(forged_proof.root_hash(), real_on_chain_root); // ← this would catch the forgery

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
