### Title
Missing Declared Class Hash Validation in `execute_replace_class` Allows Permanent Fund Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary
The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the new class hash provided by a contract corresponds to a previously declared class. This is explicitly acknowledged by a TODO comment in the code. An unprivileged contract deployer can exploit this to permanently brick any contract they control (or any contract with a bug that allows arbitrary `replace_class` calls), freezing all funds held by that contract.

---

### Finding Description

In `execute_replace_class`, the OS accepts the `class_hash` from the syscall request and updates the contract's state entry without any validation that the hash is declared: [1](#0-0) 

The critical missing check is acknowledged by the developer:

```cairo
let class_hash = request.class_hash;
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The `class_hash` field is taken directly from the contract-controlled `ReplaceClassRequest` with no constraint that it exists in `contract_class_changes`. Any felt value is accepted.

When a subsequent call is made to the bricked contract, `execute_entry_point` performs: [2](#0-1) 

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,  // returns 0 for undeclared hash
);
```

`dict_read` on an undeclared key returns 0. `find_element` with key=0 will fail to locate a compiled class, causing an OS-level assertion failure. The contract becomes permanently uncallable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any funds (ERC-20 tokens, ETH, or other assets) held in a contract whose class is replaced with an undeclared hash are permanently frozen. The contract cannot be called, upgraded, or recovered. There is no mechanism in the OS to undo a committed `replace_class` after the block is finalized.

---

### Likelihood Explanation

**Medium.** The attack requires:
1. Deploying a contract (permissionless on StarkNet), OR exploiting a missing access control in an existing contract's `replace_class` logic.
2. Calling `replace_class` with an arbitrary undeclared felt as the class hash.

No privileged role is required. Any transaction sender can deploy a contract and call `replace_class` with an undeclared hash. The OS will accept it without complaint. The scenario where users deposit funds into a contract before the attacker triggers `replace_class` is realistic (e.g., a malicious token contract or vault).

---

### Recommendation

Before updating `contract_state_changes` in `execute_replace_class`, validate that the provided `class_hash` has a non-zero entry in `contract_class_changes`:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This is exactly what the existing TODO comment calls for.

---

### Proof of Concept

1. Deploy contract `A` with a valid class hash `H_valid`. Contract `A` has a function `brick()` that calls `replace_class(0xdeadbeef)` where `0xdeadbeef` is never declared.
2. Users deposit funds into contract `A`.
3. Attacker calls `brick()`. The OS executes `execute_replace_class` with `class_hash=0xdeadbeef`.
4. The OS performs no validation — `dict_update` sets `A`'s class to `0xdeadbeef`.
5. Block is proven and finalized on L1.
6. Any subsequent call to contract `A` reaches `execute_entry_point`, which does `dict_read(key=0xdeadbeef)` → returns 0, then `find_element(key=0)` → fails.
7. Contract `A` is permanently uncallable. All deposited funds are frozen with no recovery path. [3](#0-2)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

        key=contract_address,

    );

}
