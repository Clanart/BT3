### Title
Missing Class Hash Existence Validation in `execute_replace_class` Enables Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary felt value as a new class hash without verifying that the hash corresponds to a declared class. A contract can replace its own class hash with an undeclared value, permanently making itself uncallable and freezing any funds it holds. The OS itself acknowledges this gap with an explicit TODO comment.

---

### Finding Description

In `execute_replace_class`, the OS reads `class_hash` directly from the syscall request and writes it into the contract's `StateEntry` with no existence check: [1](#0-0) 

The comment at line 898 explicitly acknowledges the missing guard:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
```

After the state is committed with an undeclared class hash, any future call to that contract reaches `execute_entry_point`, which performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
``` [2](#0-1) 

Because the undeclared hash has no entry in `contract_class_changes`, `dict_read` returns the default value `0`. The OS then calls `find_element` searching for a compiled class with hash `0`: [3](#0-2) 

`find_element` is a non-optional search; if no compiled class with hash `0` exists in the bundle, it panics, causing the entire OS proof to fail. This means **no valid proof can ever be generated for any block that includes a call to the bricked contract**. The contract is permanently uncallable at the protocol level.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once the state is committed on-chain with an invalid class hash, the contract is irreversibly bricked. Any ERC-20 balances, ETH, or other assets held by the contract are permanently inaccessible. There is no recovery path short of a protocol-level upgrade. The OS proof system itself enforces the freeze: the prover cannot generate a valid proof for any block that attempts to call the contract.

---

### Likelihood Explanation

**Medium.**

The attack surface is any contract that exposes a code path leading to `replace_class` with attacker-controlled input. Concrete scenarios:

1. A contract with an upgradeable pattern that accepts a new class hash from an external caller without validating it against declared classes.
2. A cross-contract call where a malicious callee calls `replace_class` on itself with a crafted hash, bricking a shared vault or escrow.
3. A contract bug that allows arbitrary storage writes that corrupt the class hash indirectly.

No privileged role is required. Any unprivileged transaction sender can deploy a contract and trigger this path. The OS-level missing check is the necessary vulnerable step — if the OS rejected undeclared hashes, none of these scenarios would succeed.

---

### Recommendation

Inside `execute_replace_class`, before updating the `StateEntry`, add a validation that the new `class_hash` is present in `contract_class_changes` (i.e., has been declared in the current or a prior block):

```cairo
// Ensure the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the check already implicitly performed in `execute_entry_point` and makes the invariant explicit and enforced at the point of the syscall.

---

### Proof of Concept

1. Deploy **Contract A** (holds user funds) with a public function `brick(new_hash: felt)` that calls the `replace_class` syscall with `new_hash`.
2. An attacker calls `brick(0xdeadbeef)` where `0xdeadbeef` is an undeclared class hash.
3. `execute_replace_class` accepts the call — no existence check is performed. [4](#0-3) 
4. The block is proven and committed. Contract A's `StateEntry.class_hash` is now `0xdeadbeef` on-chain.
5. In any subsequent block, a call to Contract A reaches `execute_entry_point`. `dict_read(contract_class_changes, 0xdeadbeef)` returns `0`. `find_element` searches for compiled class hash `0` and panics. [5](#0-4) 
6. No valid OS proof can be produced for any block containing a call to Contract A.
7. All funds held by Contract A are permanently frozen.
