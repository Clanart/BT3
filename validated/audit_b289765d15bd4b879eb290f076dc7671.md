### Title
`ProofOfInclusion::valid()` Tautological Root-Hash Check Allows Forged Inclusion Proofs - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` in the DataLayer crate contains a tautological final check that makes the function verify only the internal self-consistency of the proof chain, never the proof against any external trusted root hash. An attacker who can supply a `ProofOfInclusion` object (which is `Streamable` and fully exposed via Python/WASM bindings) can forge a proof for an arbitrary `node_hash` that always passes `valid()`.

### Finding Description
The `valid()` method iterates through each `ProofOfInclusionLayer`, computing `calculated_hash = calculate_internal_hash(existing_hash, layer.other_hash_side, layer.other_hash)`, asserting `calculated_hash == layer.combined_hash`, and then advancing `existing_hash = calculated_hash`. After the loop, `existing_hash` equals `last_layer.combined_hash` by construction (the loop body just verified this). The final line is:

```rust
existing_hash == self.root_hash()
```

where `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← same field existing_hash was just set to
    } else {
        self.node_hash
    }
}
```

This reduces to `last_layer.combined_hash == last_layer.combined_hash`, which is unconditionally `true`. The function therefore never compares the computed root against any externally-trusted tree root. Any `ProofOfInclusion` whose layer hashes form a self-consistent chain passes `valid()` regardless of whether `node_hash` is actually present in the tree.

The analog to the external report is direct: just as `repegCheck()` adjusts `k` once and then re-computes cost without verifying the new cost against the budget (assuming one adjustment is sufficient), `valid()` performs a chain of hash checks and then executes a final "budget" check (`existing_hash == self.root_hash()`) that is structurally guaranteed to pass — the adjustment (setting `existing_hash = calculated_hash`) already made the check trivially true.

### Impact Explanation
`ProofOfInclusion` is a `Streamable` type exposed through both the Python wheel and WASM bindings. Any DataLayer client that receives a `ProofOfInclusion` from an untrusted peer and calls `.valid()` as its sole gate — without separately comparing `.root_hash()` against a trusted on-chain or locally-computed root — will accept a completely fabricated proof. This lets an attacker prove inclusion of an arbitrary key-value pair that does not exist in the DataLayer tree, corrupting the verified state visible to the client. This matches the allowed High impact: "DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."

### Likelihood Explanation
`ProofOfInclusion` is `Streamable` (serializable/deserializable), is constructed via `MerkleBlob::get_proof_of_inclusion`, and is returned directly to Python callers. The Python test suite calls `proof_of_inclusion.valid()` as the sole validity assertion with no external root comparison. Any DataLayer peer protocol that transmits proofs and relies on `valid()` alone is exploitable by a malicious peer with no special privileges.

### Recommendation
The `valid()` method should accept a trusted root hash parameter and compare the computed root against it:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against externally-supplied root
}
```

Alternatively, rename the current `valid()` to `internally_consistent()` to make its limited scope explicit, and require all call sites that receive proofs from untrusted sources to additionally verify `proof.root_hash() == trusted_root`.

### Proof of Concept
Construct a forged `ProofOfInclusion` for an arbitrary `node_hash` `H`:

1. Pick any `other_hash` `S` and `other_hash_side`.
2. Compute `combined = calculate_internal_hash(H, side, S)`.
3. Build `ProofOfInclusion { node_hash: H, layers: [ProofOfInclusionLayer { other_hash_side: side, other_hash: S, combined_hash: combined }] }`.
4. Call `.valid()` → returns `true` because `existing_hash` after the loop equals `combined`, and `root_hash()` also returns `combined`.

The proof passes validation for any `H`, including one that was never inserted into the tree.
