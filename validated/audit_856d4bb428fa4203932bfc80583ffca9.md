### Title
P2P State Diff Sync Accepts Attacker-Supplied State Diff Without Commitment Verification - (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

### Summary

`parse_data_for_block` in the P2P sync state diff client assembles a `ThinStateDiff` from peer-supplied `StateDiffChunk` messages by verifying only that the accumulated chunk lengths equal `header.state_diff_length`. It never verifies the assembled diff's Poseidon hash against `header.state_diff_commitment`. A single malicious peer can therefore inject an arbitrary state diff that passes all existing checks, causing the node to persist a wrong state diff and subsequently derive a wrong global state root.

### Finding Description

The loop in `parse_data_for_block` terminates when `current_state_diff_len == target_state_diff_len`, where `target_state_diff_len` is read from the already-stored block header's `state_diff_length` field: [1](#0-0) 

`current_state_diff_len` is accumulated by summing `state_diff_chunk.len()` for each received chunk: [2](#0-1) 

`StateDiffChunk::len()` and `ThinStateDiff::len()` are consistent with each other, so the length check is arithmetically sound. However, the block header also carries a `state_diff_commitment` field — the Poseidon hash of the canonical state diff: [3](#0-2) 

After the loop completes, `parse_data_for_block` calls only `validate_deprecated_declared_classes_non_conflicting` and then returns `Ok(Some((result, block_number)))`: [4](#0-3) 

There is no call to `calculate_state_diff_hash` and no comparison against `header.state_diff_commitment`. The assembled `ThinStateDiff` is then written directly to MDBX storage: [5](#0-4) 

`append_state_diff` writes the diff into all state tables (deployed contracts, nonces, storage, compiled class hashes) without any commitment check: [6](#0-5) 

The commitment hash function that *should* be used for verification is: [7](#0-6) 

The committer, which does perform a commitment check, is only invoked in the batcher/proposer flow, not in the P2P sync path: [8](#0-7) 

### Impact Explanation

A malicious P2P peer sends a valid signed block header (correct block hash, correct `state_diff_length = L`, correct `state_diff_commitment = H`) followed by `StateDiffChunk` messages whose lengths sum to `L` but whose content differs from what `H` commits to. All existing checks pass:

- `current_state_diff_len == target_state_diff_len` ✓  
- No `ConflictingStateDiffParts` (attacker uses distinct keys) ✓  
- `validate_deprecated_declared_classes_non_conflicting` ✓  

The wrong `ThinStateDiff` is persisted. The committer subsequently applies it to the Patricia trie and derives a wrong `GlobalRoot`. Every downstream consumer — `starknet_getStorageAt`, `starknet_getClassHashAt`, `starknet_getNonce`, proof-manager storage proofs, SNOS inputs — reads from these corrupted tables and returns authoritative-looking wrong values.

**Matching impact**: *High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value* (wrong storage values, class hashes, nonces); and *Critical — Wrong state … storage value* if the corrupted state is subsequently fed into blockifier re-execution or proof generation.

### Likelihood Explanation

Any node that participates in P2P sync and connects to at least one adversarial peer is vulnerable. No special privilege is required; the attacker only needs to be a reachable P2P peer. The attack is silent — no error is logged, no peer report is filed, and the node continues operating normally with corrupted state.

### Recommendation

After the loop in `parse_data_for_block`, compute the Poseidon commitment of the assembled diff and compare it against the header's `state_diff_commitment`:

```rust
// After the loop and validate_deprecated_declared_classes_non_conflicting:
if let Some(expected_commitment) = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .expect("header must exist")
    .state_diff_commitment
{
    let actual_commitment = calculate_state_diff_hash(&result);
    if actual_commitment != expected_commitment {
        return Err(ParseDataError::BadPeer(
            BadPeerError::WrongStateDiffCommitment {
                expected: expected_commitment,
                actual: actual_commitment,
            },
        ));
    }
}
```

This mirrors the check already present in the committer: [8](#0-7) 

### Proof of Concept

1. Honest block N has `state_diff_length = 2` and `state_diff_commitment = H` (Poseidon hash of `{deployed: {addr_A: class_X}, nonces: {addr_A: 1}}`).

2. Attacker peer sends the valid signed header for block N.

3. Instead of the honest chunks, attacker sends:
   - `StateDiffChunk::ContractDiff { contract_address: addr_B, class_hash: Some(class_Y), nonce: None, storage_diffs: {} }` → `chunk.len() = 1`
   - `StateDiffChunk::ContractDiff { contract_address: addr_B, class_hash: None, nonce: Some(2), storage_diffs: {} }` → `chunk.len() = 1`

4. `current_state_diff_len` reaches 2 == `target_state_diff_len`. Loop exits. No duplicate keys. `validate_deprecated_declared_classes_non_conflicting` passes.

5. The assembled diff `{deployed: {addr_B: class_Y}, nonces: {addr_B: 2}}` is written to storage.

6. `starknet_getClassHashAt(addr_A)` returns `None` (wrong); `starknet_getClassHashAt(addr_B)` returns `class_Y` (wrong). The Patricia trie root diverges from the canonical root committed in the block hash. [9](#0-8)

impl StateDiffChunk {
    pub fn len(&self) -> usize {
        match self {
            StateDiffChunk::ContractDiff(contract_diff) => {
                let mut result = contract_diff.storage_diffs.len();
                if contract_diff.class_hash.is_some() {
                    result += 1;
                }
                if contract_diff.nonce.is_some() {
                    result += 1;
                }
                result
            }
            StateDiffChunk::DeclaredClass(_) => 1,
            StateDiffChunk::DeprecatedDeclaredClass(_) => 1,
        }
    }

    pub fn is_empty(&self) -> bool {
        self.len() == 0
    }
}

impl StateStorageWriter for StorageTxn<'_, RW> {
    #[sequencer_latency_histogram(STORAGE_APPEND_THIN_STATE_DIFF_LATENCY, false)]
    fn append_state_diff(
        self,
        block_number: BlockNumber,
        thin_state_diff: ThinStateDiff,
    ) -> StorageResult<Self> {
        let file_offset_table = self.txn.open_table(&self.tables.file_offsets)?;
        let markers_table = self.open_table(&self.tables.markers)?;
        let state_diffs_table = self.open_table(&self.tables.state_diffs)?;
        let nonces_table = self.open_table(&self.tables.nonces)?;
        let deployed_contracts_table = self.open_table(&self.tables.deployed_contracts)?;
        let storage_table = self.open_table(&self.tables.contract_storage)?;
        let declared_classes_block_table = self.open_table(&self.tables.declared_classes_block)?;
        let deprecated_declared_classes_block_table =
            self.open_table(&self.tables.deprecated_declared_classes_block)?;
        let compiled_class_hash_table = self.open_table(&self.tables.compiled_class_hash)?;

        // Write state.
        write_deployed_contracts(
            &thin_state_diff.deployed_contracts,
            &self.txn,
            block_number,
            &deployed_contracts_table,
            &nonces_table,
        )?;
        write_storage_diffs(
            &thin_state_diff.storage_diffs,
            &self.txn,
            block_number,
            &storage_table,
        )?;
        // Must be called after write_deployed_contracts since the nonces are updated there.
        write_nonces(&thin_state_diff.nonces, &self.txn, block_number, &nonces_table)?;

        for (class_hash, _) in &thin_state_diff.class_hash_to_compiled_class_hash {
            let not_declared = declared_classes_block_table.get(&self.txn, class_hash)?.is_none();
            if not_declared {
                declared_classes_block_table.insert(&self.txn, class_hash, &block_number)?;
            }
        }

        write_compiled_class_hashes(
            &thin_state_diff.class_hash_to_compiled_class_hash,
            &self.txn,
            block_number,
            &compiled_class_hash_table,
        )?;

        for class_hash in thin_state_diff.deprecated_declared_classes.iter() {
            // Cairo0 classes can be declared in different blocks. The first block to declare the
            // class is recorded here.
            if deprecated_declared_classes_block_table.get(&self.txn, class_hash)?.is_none() {
                deprecated_declared_classes_block_table.insert(
                    &self.txn,
                    class_hash,
                    &block_number,
                )?;
            }
        }

        // Write state diff.
        let location = self.file_handlers.append_state_diff(&thin_state_diff);
        state_diffs_table.append(&self.txn, &block_number, &location)?;
        file_offset_table.upsert(&self.txn, &OffsetKind::ThinStateDiff, &location.next_offset())?;

        update_marker_to_next_block(&self.txn, &markers_table, MarkerKind::State, block_number)?;

        advance_compiled_class_marker_over_blocks_without_classes(
            &self.txn,
            &markers_table,
            &state_diffs_table,
            &self.file_handlers,
        )?;

        Ok(self)

    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);

    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
