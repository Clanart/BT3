Audit Report

## Title
Residual ETH Left by Prior Caller Is Silently Consumed by Subsequent WETH-Input Swap, Causing Direct ETH Loss — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary
`PeripheryPayments.pay()` reads `address(this).balance` to subsidize WETH-input swaps, but this balance includes ETH deposited by prior callers via `msg.value` on any `payable` swap function. Because no automatic refund occurs after any swap, a subsequent WETH-input swap permanently consumes a prior caller's ETH, transferring it to the pool as WETH on behalf of the new caller.

## Finding Description
`receive()` reverts for any `msg.sender != WETH`, but this guard only applies to bare ETH transfers — it does not prevent ETH from entering via `msg.value` on `payable` functions. [1](#0-0) 

All swap entry points are `payable`: [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) 

Inside `pay()`, when `token == WETH`, the function reads the entire `address(this).balance` without any attribution to the current caller: [7](#0-6) 

If `nativeBalance >= value`, the router wraps the full `value` from its own ETH balance and transfers WETH to the pool — pulling **zero** WETH from `payer`. If `0 < nativeBalance < value`, it partially subsidizes from its balance and pulls only the remainder from `payer`. No automatic refund is issued at the end of any swap function; `refundETH()` must be called explicitly: [8](#0-7) 

**Concrete exploit path:**
1. User A calls `exactInputSingle{value: 1 ether}` with `tokenIn = someERC20` (not WETH). `pay()` takes the `else` branch (line 86), pulling ERC20 from User A via `transferFrom`. The 1 ETH `msg.value` is never consumed and remains in the router. User A does not call `refundETH()`. [9](#0-8) 
2. User B calls `exactInputSingle` (no ETH sent) with `tokenIn = WETH`, `amountIn = 1 ether`. Inside the swap callback, `pay(WETH, userB, pool, 1e18)` is called. `nativeBalance = 1 ether >= value = 1 ether`, so the router wraps User A's 1 ETH and transfers WETH to the pool. User B's WETH allowance is never touched.
3. User A's 1 ETH is permanently lost; User B swapped for free.

## Impact Explanation
Direct, permanent loss of User A's ETH principal. The ETH is not locked — it is actively transferred to a pool as WETH on behalf of User B. User A receives nothing in return. This is a Critical/High direct loss of user principal.

## Likelihood Explanation
Medium. The `receive()` guard creates a false sense of security. The mixed ETH/WETH payment feature is an intended use case that actively encourages users to send ETH with WETH swaps. Any user who sends slightly more ETH than needed, or who batches a non-WETH swap with ETH in a `multicall` without a trailing `refundETH()`, creates a residual that any subsequent WETH swapper can drain. No special attacker capability is required — a normal swap call suffices.

## Recommendation
Track the ETH balance attributable to the current transaction rather than the entire contract balance. The standard fix is to snapshot `address(this).balance - msg.value` at entry and only treat `msg.value` as available ETH for the current caller. Pass this value down through the call stack to `pay()` instead of reading `address(this).balance`. Alternatively, auto-refund any unused ETH at the end of each swap function rather than relying on the caller to invoke `refundETH()`.

## Proof of Concept

```solidity
function test_residualEthStolenByWethSwap() public {
    // User A calls exactInputSingle with tokenIn=token1 but accidentally sends 1 ETH
    vm.deal(userA, 1 ether);
    vm.prank(userA);
    // tokenIn is token1 (not WETH), so msg.value is never consumed
    router.exactInputSingle{value: 1 ether}(
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(token1WethPool),
            tokenIn: address(token1),
            tokenOut: address(weth),
            zeroForOne: false,
            amountIn: 1000,
            amountOutMinimum: 0,
            recipient: userA,
            deadline: block.timestamp + 1,
            priceLimitX64: 0,
            extensionData: ""
        })
    );
    // userA forgets refundETH(); 1 ETH is now stuck in router
    assertEq(address(router).balance, 1 ether);

    // User B swaps WETH->token1; router uses userA's ETH
    uint256 userBWethBefore = weth.balanceOf(userB);
    vm.prank(userB);
    router.exactInputSingle(  // no msg.value
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(wethToken1Pool),
            tokenIn: address(weth),
            tokenOut: address(token1),
            zeroForOne: true,
            amountIn: 1 ether,
            amountOutMinimum: 0,
            recipient: userB,
            deadline: block.timestamp + 1,
            priceLimitX64: 0,
            extensionData: ""
        })
    );

    assertEq(address(router).balance, 0, "router ETH drained");
    assertEq(weth.balanceOf(userB), userBWethBefore, "userB paid no WETH");
    assertEq(userA.balance, 0, "userA lost 1 ETH");
}
```
