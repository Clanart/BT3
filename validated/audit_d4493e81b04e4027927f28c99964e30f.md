### Title
Fire-and-Forget `burn_tokens_if_needed` Allows Unbacked Token Minting on Destination Chain — (`File: near/omni-bridge/src/lib.rs`)

### Summary
`burn_tokens_if_needed` dispatches a cross-contract `burn` call using `.detach()`, meaning the bridge records the transfer message and proceeds to MPC signing regardless of whether the burn actually succeeds. If the burn fails silently, the bridge-deployed tokens remain in the bridge contract while the MPC signature authorizes minting an equivalent amount on the destination chain, creating unbacked supply.

### Finding Description

`burn_tokens_if_needed` is a fire-and-forget cross-contract call:

```rust
fn burn_tokens_if_needed(&self, token: AccountId, amount: U128) {
    if self.is_deployed_token(&token) {
        ext_token::ext(token)
            .with_static_gas(BURN_TOKEN_GAS)
            .burn(amount)
            .detach();   // ← no callback, no success check
    }
}
``` [1](#0-0) 

This function is called in `init_transfer_internal` **after** the transfer message is already committed to state:

```rust
// transfer message already stored via add_transfer_message above
self.burn_tokens_if_needed(token_id.clone(), transfer_message.amount);  // detached
self.lock_tokens_if_needed(...);
// ...
env::log_str(&OmniBridgeEvent::InitTransferEvent { transfer_message }.to_log_string());
U128(0)   // ← returns 0 to ft_transfer_call, tokens stay with bridge
``` [2](#0-1) 

The same pattern appears in `fast_fin_transfer_to_other_chain` (line 932) and `resolve_fast_transfer` (line 904), where the comment explicitly states the burn is the mechanism preventing double-minting: [3](#0-2) 

After `init_transfer_internal` returns `U128(0)`, the NEP-141 `ft_transfer_call` protocol keeps the full token amount with the bridge contract. The transfer message is now stored and `sign_transfer` can be called by any relayer: [4](#0-3) 

`sign_transfer` does **not** re-verify that the burn succeeded before requesting the MPC signature. It only checks that the transfer message exists and the fee matches.

### Impact Explanation

If the `.detach()`-ed burn fails (e.g., due to insufficient `BURN_TOKEN_GAS`, a token contract panic, or a storage edge case), the following state holds simultaneously:

1. The bridge contract holds the full token amount (tokens were transferred in via `ft_transfer_call`, never burned).
2. The transfer message is stored and signable.
3. The MPC produces a valid signature authorizing minting on the destination chain.
4. Tokens are minted on the destination chain without a corresponding burn on NEAR.

This creates unbacked supply: the same economic value exists on both NEAR (in the bridge contract) and the destination chain. An attacker who can reliably trigger burn failure can repeat this to mint unbacked tokens on any destination chain. This matches **Critical — unauthorized mint of bridged assets**.

### Likelihood Explanation

The burn targets bridge-deployed tokens (`is_deployed_token` check) where the bridge is the controller. Under normal conditions the burn succeeds. Likelihood is low-to-medium because:

- `BURN_TOKEN_GAS` is a fixed constant; if it is ever insufficient for the token's `burn` execution (e.g., after a token contract upgrade that adds storage operations), every `init_transfer` for that token silently skips the burn.
- The failure is silent and produces no on-chain error — it would not be detected until supply discrepancy is noticed off-chain.
- No privileged access is required to trigger `ft_transfer_call`; any token holder can initiate the flow. [5](#0-4) 

### Recommendation

Replace the `.detach()` pattern with a chained callback that verifies burn success before the transfer message is considered finalized. The transfer message should only be committed (or the `InitTransferEvent` emitted) after the burn callback confirms success. If the burn fails, the callback should return the full token amount to the sender (non-zero return from `ft_on_transfer`).

Alternatively, record the transfer message in a "pending burn" state and only promote it to a signable state inside a burn-success callback.

### Proof of Concept

1. User holds bridge-deployed token `T` on NEAR.
2. User calls `T.ft_transfer_call(receiver_id=bridge, amount=1000, msg=InitTransfer{recipient="eth:0x...", fee=0, ...})`.
3. NEP-141 transfers 1000 T to bridge; bridge's `ft_on_transfer` → `init_transfer` → `init_transfer_internal` is called.
4. `add_transfer_message` stores the transfer message. `burn_tokens_if_needed` fires `.detach()` with `BURN_TOKEN_GAS`.
5. If the burn cross-contract call fails (gas exhaustion, token panic), the bridge contract retains 1000 T. `init_transfer_internal` returns `U128(0)`, so `ft_transfer_call` keeps the tokens with the bridge.
6. `InitTransferEvent` is emitted. A relayer calls `sign_transfer`; MPC signs the payload.
7. Relayer submits the signature to the EVM `OmniBridge.finTransfer`, which mints 1000 T on Ethereum.
8. 1000 T now exist on both NEAR (in bridge contract) and Ethereum — unbacked supply. [1](#0-0) [6](#0-5)

    pub fn ft_on_transfer(&mut self, sender_id: AccountId, amount: U128, msg: String) {
        let token_id = env::predecessor_account_id();
        let parsed_msg: BridgeOnTransferMsg = serde_json::from_str(&msg)
            .or_else(|_| serde_json::from_str(&msg).map(BridgeOnTransferMsg::InitTransfer))
            .near_expect(BridgeError::ParseMsg);

        // We can't trust sender_id to pay for storage as it can be spoofed.
        let signer_id = env::signer_account_id();
        let promise_or_promise_index_or_value = match parsed_msg {
            BridgeOnTransferMsg::InitTransfer(init_transfer_msg) => {
                self.init_transfer(sender_id, signer_id, token_id, amount, init_transfer_msg)
            }
            BridgeOnTransferMsg::FastFinTransfer(fast_fin_transfer_msg) => {
                self.fast_fin_transfer(token_id, amount, signer_id, fast_fin_transfer_msg)
            }
            BridgeOnTransferMsg::UtxoFinTransfer(utxo_fin_transfer_msg) => self.utxo_fin_transfer(
                token_id,
                amount,
                &signer_id,
                &sender_id,
                utxo_fin_transfer_msg,
            ),
            BridgeOnTransferMsg::SwapMigratedToken => {
                self.swap_migrated_token(sender_id, token_id, amount)
                    .detach();
                PromiseOrPromiseIndexOrValue::Value(U128(0))
            }
        };

        promise_or_promise_index_or_value.as_return();
    }

    pub fn sign_transfer(
        &mut self,
        transfer_id: TransferId,
        fee_recipient: Option<AccountId>,
        fee: &Option<Fee>,
    ) -> Promise {
        let transfer_message = self.get_transfer_message(transfer_id);

        if let Some(fee) = &fee {
            require!(
                &transfer_message.fee == fee,
                BridgeError::InvalidFee.as_ref()
            );
        }

        let token_address = self
            .get_token_address(
                transfer_message.get_destination_chain(),
                self.get_token_id(&transfer_message.token),
            )
            .unwrap_or_else(|| {
                env::panic_str(BridgeError::FailedToGetTokenAddress.to_string().as_str())
            });

        let decimals = self
            .token_decimals
            .get(&token_address)
            .near_expect(BridgeError::TokenDecimalsNotFound);
        let amount_to_transfer = Self::normalize_amount(
            transfer_message
                .amount_without_fee()
                .near_expect(BridgeError::InvalidFee),
            decimals,
        );

        require!(
            amount_to_transfer > 0,
            BridgeError::InvalidAmountToTransfer.as_ref()
        );

        let message = DestinationChainMsg::from_json(&transfer_message.msg)
            .and_then(|s| s.destination_msg())
            .unwrap_or_default();

        let transfer_payload = TransferMessagePayload {
            prefix: PayloadType::TransferMessage,
            destination_nonce: transfer_message.destination_nonce,
            transfer_id,
            token_address,
            amount: U128(amount_to_transfer),
            recipient: transfer_message.recipient,
            fee_recipient,
            message,
        };

        let payload = near_sdk::env::keccak256_array(
            transfer_payload
                .encode_hashable()
                .near_expect(BridgeError::Borsh),
        );

        ext_signer::ext(self.mpc_signer.clone())
            .with_static_gas(MPC_SIGNING_GAS)
            .with_attached_deposit(env::attached_deposit())
            .sign(SignRequest {
                payload,
                path: SIGN_PATH.to_owned(),
                key_version: 0,
            })
            .then(
                Self::ext(env::current_account_id())
                    .with_static_gas(SIGN_TRANSFER_CALLBACK_GAS)
                    .sign_transfer_callback(transfer_payload, &transfer_message.fee),
            )
    }

    fn init_transfer_internal(
        &mut self,
        transfer_message: TransferMessage,
        storage_owner: AccountId,
    ) -> U128 {
        let required_storage_balance = self
            .add_transfer_message(transfer_message.clone(), storage_owner.clone())
            .saturating_add(NearToken::from_yoctonear(transfer_message.fee.native_fee.0));

        if self
            .try_update_storage_balance(
                storage_owner,
                required_storage_balance,
                NearToken::from_yoctonear(0),
            )
            .is_err()
        {
            self.remove_transfer_message_without_refund(transfer_message.get_transfer_id());
            return transfer_message.amount;
        }

        if let OmniAddress::Near(token_id) = transfer_message.token.clone() {
            self.burn_tokens_if_needed(token_id.clone(), transfer_message.amount);

            self.lock_tokens_if_needed(
                transfer_message.get_destination_chain(),
                &token_id,
                transfer_message.amount.0,
            );
        } else {
            self.remove_transfer_message_without_refund(transfer_message.get_transfer_id());
            return transfer_message.amount;
        }

        env::log_str(&OmniBridgeEvent::InitTransferEvent { transfer_message }.to_log_string());
        U128(0)
    }
