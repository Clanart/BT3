### Title
Integer Division Truncation in `NewDynamicFeeChecker` Allows Cosmos SDK Transactions to Bypass Fee Deduction When `baseFee = 0` — (`File: ante/evm/fee_checker.go`)

---

### Summary

`NewDynamicFeeChecker` derives a per-gas price cap by integer-dividing the user-supplied fee by the gas limit (`feeCap = fee / gas`). When `baseFee = 0` — a reachable state when `MinGasPrice < 1` because `CalculateBaseFee` truncates `MinGasPrice` to an integer lower bound of 0 — the truncated `feeCap` collapses to zero for any fee smaller than the gas limit. The resulting `effectiveFee` is zero, so `DeductFeeDecorator` deducts nothing from the sender even though `MinGasPriceDecorator` already accepted the transaction as fee-sufficient.

---

### Finding Description

**Step 1 — `feeCap` truncation** (`ante/evm/fee_checker.go` line 83):

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))   // integer division, rounds down
```

`sdkmath.Int.Quo` is truncating integer division. If `fee < gas`, `feeCap = 0`.

**Step 2 — guard passes when `baseFee = 0`** (line 86):

```go

```
