### Title
Gateway Silently Swallows `ProofManager` Storage Failure, Admitting Transactions Whose Proofs Are Permanently Unrecoverable - (`File: crates/apollo_gateway/src/gateway.rs`)

### Summary
`GenericGateway::store_proof_and_spawn_archiving` logs a `ProofManager` storage error but does not propagate it. The transaction is forwarded to the mempool regardless. Later, when the consensus orchestrator reconstructs the full RPC transaction for P2P broadcast, it calls `get_proof` from the proof manager; because the proof was never persisted, the call returns `ProofNotFound`, breaking the consensus broadcast path for any block that contains the affected transaction.

### Finding Description

In `add_tx_inner`, after proof verification succeeds, the gateway calls `store_proof_and_spawn_archiving`: [1](#0-0) 

Inside that function, the proof manager write result is matched, but the `Err` arm only logs and falls through — it does **not** return early or propagate the error: [2](#0-1) 

Execution then continues unconditionally: the GCS archive task is spawned, and `Some((handle, tx_hash))` is returned. Back in `add_tx_inner`, the transaction is sent to the mempool and the gateway returns `Ok(gateway_output)`: [3](#0-2) 

The internal transaction carries `proof_facts` (included in the transaction hash): [4](#0-3) 

But the corresponding `Proof` blob is absent from the proof manager. When the consensus orchestrator later needs to reconstruct the full RPC transaction for P2P broadcast, it calls: [5](#0-4) 

`get_proof` returns `Err(ProofNotFound)` because the proof was never stored: [6](#0-5) 

The blockifier's `validate_proof_facts` only checks the `proof_facts` fields (program hash, block hash, config hash) — it does not require the proof blob to be present in the proof manager: [7](#0-6) 

So the transaction executes successfully in the blockifier and is included in a block, but the consensus orchestrator cannot broadcast the block proposal.

### Impact Explanation

A transaction with non-empty `proof_facts` is admitted to the mempool and executed by the blockifier. When the consensus orchestrator attempts to reconstruct the full RPC transaction (to broadcast the block proposal to peers), `convert_internal_rpc_tx_to_rpc_tx` fails with `ProofNotFound`. The block proposal containing this transaction cannot be broadcast, stalling consensus for that block. This matches the **High** impact: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

### Likelihood Explanation

Any transient I/O error, disk-full condition, or proof manager service interruption during the `set_proof` call triggers the path. An adversary who can exhaust proof manager storage (by flooding the gateway with large valid proofs) can reliably induce the failure for a subsequent targeted transaction. Even without a deliberate attacker, ordinary infrastructure instability silently corrupts the admission invariant.

### Recommendation

Propagate the `store_proof_in_proof_manager` error as a gateway rejection. Change the `Err` arm in `store_proof_and_spawn_archiving` to return an error (or change the function signature to `Result<ProofArchiveHandle, ...>`) and have `add_tx_inner` reject the transaction when proof storage fails, analogous to how `ERC20.safeTransfer` reverts on failure rather than silently continuing.

```rust
Err(e) => {
    error!("Failed to set proof in proof manager: {}", e);
    return Err(/* gateway error */);  // reject the transaction
}
```

### Proof of Concept

1. Configure the proof manager with a read-only or full filesystem path so `set_proof` returns an `IoError`.
2. Submit a valid Invoke V3 transaction carrying non-empty `proof_facts` and a valid `proof` to the gateway.
3. Observe that the gateway returns `200 OK` with a transaction hash (proof verification passed, storage error only logged).
4. Confirm the transaction is present in the mempool.
5. Trigger block building; the blockifier executes the transaction successfully (proof_facts fields pass `validate_proof_facts`).
6. Observe that `convert_internal_rpc_tx_to_rpc_tx` returns `ProofNotFound` when the consensus orchestrator attempts to reconstruct the transaction for P2P broadcast, preventing the block proposal from being sent.

    fn validate_proof_facts(
        &self,
        block_context: &BlockContext,
        state: &mut dyn State,
    ) -> TransactionPreValidationResult<()> {
        // Only Invoke V3 transactions can carry proof facts.
        let Transaction::Invoke(invoke_tx) = &self.tx else {
            return Ok(());
        };
        if invoke_tx.version() < TransactionVersion::THREE {
            return Ok(());
        }

        // Parse proof facts.
        let proof_facts = invoke_tx.proof_facts();
        let snos_proof_facts = match ProofFactsVariant::try_from(&proof_facts)
            .map_err(|e| TransactionPreValidationError::InvalidProofFacts(e.to_string()))?
        {
            ProofFactsVariant::Empty => return Ok(()),
            ProofFactsVariant::Snos(snos_proof_facts) => snos_proof_facts,
        };
        let os_constants = &block_context.versioned_constants.os_constants;

        // Validate the program hash.
        let allowed = &os_constants.allowed_virtual_os_program_hashes;
        if !allowed.contains(&snos_proof_facts.program_hash) {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS program hash {} is not allowed",
                snos_proof_facts.program_hash
            )));
        }

        // Validate the block hash and block number.
        let proof_block_hash = snos_proof_facts.block_hash.0;
        let proof_block_number = snos_proof_facts.block_number.0;
        Self::validate_proof_block_number(
            proof_block_number,
            block_context.block_info.block_number,
        )?;
        Self::validate_proof_block_hash(proof_block_hash, proof_block_number, os_constants, state)?;

        // Validate the config hash.
        let chain_info = &block_context.chain_info;
        // TODO(Meshi): Cache this computation as part of the chain context.
        let virtual_os_config_hash = OsChainInfo::from(chain_info)
            .compute_virtual_os_config_hash()
            .expect("Failed to compute OS config hash");
        let proof_config_hash = snos_proof_facts.config_hash;
        if virtual_os_config_hash != proof_config_hash {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS config hash mismatch. Computed virtual OS config hash: \
                 {virtual_os_config_hash}, expected virtual OS config hash: {proof_config_hash}."
            )));
        }

        Ok(())
    }
