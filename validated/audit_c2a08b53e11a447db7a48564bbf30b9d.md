Audit Report

## Title
Stranded Native ETH on Router Is Consumed by Any Subsequent WETH-Input Swap — (File: metric-periphery/contracts/base/PeripheryPayments.sol)

## Summary
`PeripheryPayments.pay()` uses the router's global `address(this).balance` to cover WETH-input swap payments with no per-user attribution. Native ETH left on the router by one user (e.g., by omitting `refundETH()` after sending excess `msg.value`) is silently consumed to settle a later, unrelated user's WETH obligation. The original depositor receives nothing in return and their ETH is permanently lost.

## Finding Description
`PeripheryPayments.pay()` handles WETH-input payments via a three-branch native-ETH priority path:

```solidity
// metric-periphery/contracts/base/PeripheryPayments.sol lines 73-84
} else if (token == WETH) {
    uint256 nativeBalance = address(this).balance;
    if (nativeBalance >= value) {
        IWETH9(WETH).deposit{value: value}();
        IERC20(WETH).safeTransfer(recipient, value);
    } else if (nativeBalance > 0) {
        IWETH9(WETH).deposit{value: nativeBalance}();
        IERC20(WETH).safeTransfer(recipient, nativeBalance);
        IERC20(WETH).safeTransferFrom(payer, recipient, value - nativeBalance);
    } else {
        IERC20(WETH).safeTransferFrom(payer, recipient, value);
    }
}
```

`address(this).balance` is a global, unattributed pool. There is no accounting variable that tracks which user's `msg.value` contributed to the current balance.

The `receive()` gate only blocks direct ETH pushes from non-WETH addresses:

```solidity
// metric-periphery/contracts/base/PeripheryPayments.sol lines 32-34
receive() external payable {
    if (msg.sender != WETH) revert NotWETH();
}
```

This guard does not prevent ETH sent via `msg.value` in a payable `multicall` or direct `exactInputSingle` call from persisting on the router across transactions. `exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`, and `multicall` are all `payable`. Any ETH sent with these calls that is not fully consumed by the swap amount, and for which the user omits a trailing `refundETH()`, remains on the router indefinitely.

The next user who calls any WETH-input swap triggers `pay(WETH, nextUser, pool, value)`. If `address(this).balance >= value`, the router wraps the stranded ETH and forwards it to the pool — charging the stranded ETH instead of pulling WETH from `nextUser`. The `payer` field (set to `msg.sender` of the second user via transient storage) is never consulted in the first branch.

The `refundETH()` call is optional and must be composed manually by the caller. The test suite itself demonstrates the expected pattern but does not enforce it:

```solidity
// metric-periphery/test/MetricOmmSimpleRouter.native.t.sol lines 128-129
calls[1] = abi.encodeWithSelector(router.refundETH.selector);
router.multicall{value: msgValue}(calls);
```

## Impact Explanation
Direct loss of user principal. User A's native ETH, stranded on the router, is consumed to settle User B's WETH swap obligation. User A receives nothing in return; User B receives a free or partially-free swap. The loss is bounded only by the stranded ETH amount, which can be arbitrarily large (e.g., a user who sends 1 ETH for a small swap and forgets `refundETH()`). This constitutes a direct loss of user principal above Sherlock thresholds and falls squarely within the "Critical/High direct loss of user principal" allowed impact gate.

## Likelihood Explanation
High. The `refundETH()` call is optional and must be manually appended to a multicall. Any user who sends excess ETH without appending `refundETH()` — a common mistake with payable multicalls — leaves exploitable residue. An attacker can monitor the router's ETH balance on-chain and immediately follow with a WETH-input swap for any amount up to the stranded balance, draining it at zero cost. No special privileges or preconditions are required beyond being an ordinary router caller.

## Recommendation
Track the ETH legitimately attributable to the current transaction's `msg.value` in a transient storage slot (e.g., `T_SLOT_MSG_VALUE`). At the start of each top-level payable entry point (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`), record `msg.value` in that slot. In `pay()`, deduct from that slot before falling through to `safeTransferFrom`. If the slot is exhausted, pull from `payer` directly. Clear the slot at the end of the call. This ensures only the current caller's ETH is ever used to cover their own WETH obligation.

Alternatively, require callers to pass `amountIn` exactly as `msg.value` when paying with native ETH, and revert if `address(this).balance` exceeds the declared amount at entry.

## Proof of Concept
```
Setup:
  - pool: WETH / Token1
  - router deployed with WETH address

Step 1 — Victim strands ETH:
  vm.prank(victim);
  // victim sends 1 ETH but swap only needs 1000 wei; no refundETH() appended
  router.exactInputSingle{value: 1 ether}(ExactInputSingleParams({
      pool: pool,
      tokenIn: WETH,
      tokenOut: token1,
      zeroForOne: true,
      amountIn: 1000,
      amountOutMinimum: 0,
      recipient: victim,
      deadline: block.timestamp + 1,
      priceLimitX64: 0,
      extensionData: ""
  }));
  // router.balance == 1 ether - 1000 wei  (victim's ETH, stranded)

Step 2 — Attacker exploits:
  vm.prank(attacker);
  // attacker has zero ETH, zero WETH approved, but calls WETH-input swap
  router.exactInputSingle(ExactInputSingleParams({
      pool: pool,
      tokenIn: WETH,
      tokenOut: token1,
      zeroForOne: true,
      amountIn: 500,          // <= router.balance
      amountOutMinimum: 0,
      recipient: attacker,
      deadline: block.timestamp + 1,
      priceLimitX64: 0,
      extensionData: ""
  }));
  // pay(WETH, attacker, pool, 500) fires:
  //   nativeBalance = 1 ether - 1000 wei >= 500
  //   router wraps 500 wei of victim's ETH → transfers WETH to pool
  //   attacker receives token1 output, spending none of their own funds

Assert:
  assertGt(attacker_token1_balance, 0);              // attacker received output
  assertEq(address(router).balance, 1 ether - 1500); // victim's ETH partially consumed
```
