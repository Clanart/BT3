### Title
Missing Declared-Class Validation in `execute_replace_class` Enables Permanent Contract Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary class hash and immediately commits it to state without verifying that the hash corresponds to a declared contract class. Validation is deferred to the next call to the contract, at which point `execute_entry_point` fails irrecoverably. Any funds held by the contract are permanently frozen.

---

### Finding Description

In `execute_replace_class`, the OS reads the requested class hash from the syscall and updates `contract_state_changes` with no check that the hash was ever declared: [1](#0-0) 

The explicit TODO at line 898 acknowledges the missing check:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
```

The class hash is committed to `contract_state_changes` unconditionally: [2](#0-1) 

Validation is deferred to the next invocation of the contract. At that point, `execute_entry_point` performs: [3](#0-2) 

`dict_read` on an undeclared class hash returns `0` (the Cairo dict default). `find_element` with `key=0` then panics because no compiled class with hash `0` exists. This is not a graceful revert — it is a hard OS-level failure. The contract becomes permanently inaccessible.

**The deferred-validation pattern is exact:**

| Step | NTT Manager (external report) | StarkNet OS (this finding) |
|---|---|---|
| Step 1 | Transfer queued, funds locked, instruction order not validated | `replace_class` accepted, class hash committed, declared-class check skipped |
| Step 2 | `completeOutboundQueuedTransfer` validates order, always reverts | Next call to contract hits `find_element` with hash `0`, panics |
| Consequence | Funds permanently stuck in NTT Manager | Contract permanently inaccessible, funds frozen |

---

### Impact Explanation

Any contract holding funds — a multisig wallet, a DeFi vault, an escrow — whose class hash is replaced with an undeclared value becomes permanently inaccessible. No entry point can be called, no withdrawal executed, no further upgrade applied. This is **Critical: Permanent freezing of funds**.

---

### Likelihood Explanation

Any contract can issue the `replace_class` syscall with an arbitrary `felt` value as the class hash. No privilege is required. A realistic path:

1. A user deploys an upgradeable contract holding funds, with an `upgrade(new_class_hash)` function that calls `replace_class`.
2. An attacker (or the user by mistake) calls `upgrade` with a class hash that was never declared.
3. The OS executes `execute_replace_class` with no validation and commits the invalid hash.
4. The contract is permanently broken.

The TODO comment confirms the team is aware the check is absent. The OS is the protocol's final arbiter of validity; if it accepts the state transition, the proof is valid and the state is committed on L1.

---

### Recommendation

Before committing the new class hash in `execute_replace_class`, verify it exists in `contract_class_changes` (i.e., it was declared in the current or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` with the new `class_hash` and assert the result is non-zero before calling `dict_update`. [4](#0-3) 

---

### Proof of Concept

1. Deploy contract `Vault` holding 1000 tokens, exposing `upgrade(new_class_hash: felt)` which calls `replace_class(new_class_hash)`.
2. Call `Vault.upgrade(0xdeadbeef)` where `0xdeadbeef` is not a declared class hash.
3. OS executes `execute_replace_class`: no validation, `contract_state_changes` updated with `class_hash = 0xdeadbeef`.
4. Call `Vault.withdraw(...)`.
5. `execute_entry_point` reads `contract_class_changes[0xdeadbeef]` → returns `0` (dict default).
6. `find_element(..., key=0)` panics — no compiled class with hash `0` exists.
7. `Vault` is permanently inaccessible; 1000 tokens are frozen forever. [5](#0-4)

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.

    %{ GetContractAddressStateEntry %}

        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

        key=contract_address,
        prev_value=cast(state_entry, felt),

    );
