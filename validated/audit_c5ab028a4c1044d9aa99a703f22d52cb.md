### Title
Missing Duplicate Key/Hash Validation in `batch_insert` Corrupts DataLayer Merkle Tree Root - (File: crates/chia-datalayer/src/merkle/blob.rs)

### Summary
`MerkleBlob::batch_insert` bypasses the duplicate-key and duplicate-hash guards that `MerkleBlob::insert` enforces for all items beyond the first two, allowing an untrusted caller to silently insert duplicate leaves into the DataLayer Merkle tree. The resulting tree has an incorrect root hash and produces forged or invalid inclusion/exclusion proofs.

### Finding Description
`MerkleBlob::insert` enforces two guards before writing any leaf:

```rust
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`batch_insert` handles the first two items (when the tree has ≤ 1 existing leaf) by calling `self.insert(...)`, which runs those guards. All remaining items are written directly via `insert_entry_to_blob` with no duplicate check whatsoever:

```rust
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    // ... builds LeafNode ...
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
``` [2](#0-1) 

Because `pop()` removes from the tail of the input vector, the first two items processed by `insert()` are the last two elements of the caller-supplied slice; all earlier elements go through the unchecked bulk path. A caller supplying a batch of ≥ 3 entries where any entry duplicates a key or hash already present (or duplicates another entry in the same batch) will silently produce a structurally invalid tree.

The Python binding `py_batch_insert` exposes this path directly to untrusted callers: [3](#0-2) 

### Impact Explanation
A corrupted `MerkleBlob` has an incorrect root hash. Any inclusion or exclusion proof generated from it will be invalid, allowing untrusted input to prove membership of keys that are not legitimately present, or to deny membership of keys that are. This directly matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

### Likelihood Explanation
The `batch_insert` API is the recommended high-throughput insertion path and is exposed via the Python wheel. Any caller that passes a batch containing a repeated `KeyId` or a repeated `Hash` — whether by mistake or by deliberate construction — triggers the corruption silently (no error is returned). The `block_status_cache` is not updated for the bulk-path leaves, so subsequent `check_integrity` calls will also detect the inconsistency, but only after the corrupted state has already been committed.

### Recommendation
Add the same duplicate-key and duplicate-hash guards at the start of the bulk loop in `batch_insert`, mirroring the checks in `insert`:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // ... existing bulk-insert logic ...
}
```

Alternatively, pre-validate the entire input batch for uniqueness before any writes begin, and update `block_status_cache` atomically with each bulk write so the cache stays consistent.

### Proof of Concept
```python
from chia_rs.datalayer import MerkleBlob

blob = MerkleBlob(bytearray())

# Seed the tree with 2 leaves so leaf_count > 1 and the bulk path is taken
blob.insert(1, 1, b'\x01' * 32)
blob.insert(2, 2, b'\x02' * 32)

# batch_insert with a duplicate key (key=1 already present)
# The duplicate bypasses the KeyAlreadyPresent guard in the bulk path
blob.batch_insert(
    [(3, 3), (1, 99)],          # key 1 is a duplicate
    [b'\x03' * 32, b'\x04' * 32]
)

# Tree now contains two leaves with key=1; root hash is incorrect
# Proofs generated from this tree are invalid
blob.calculate_lazy_hashes()
print("Corrupted root:", blob.get_root_hash())
```

    #[pyo3(name = "batch_insert")]
    pub fn py_batch_insert(
        &mut self,
        keys_values: Vec<(KeyId, ValueId)>,
        hashes: Vec<Hash>,
    ) -> PyResult<()> {
        if keys_values.len() != hashes.len() {
            Err(Error::UnmatchedKeysAndValues(
                keys_values.len(),
                hashes.len(),
            ))?;
        }

        self.batch_insert(zip(keys_values, hashes).collect())?;

        Ok(())
    }
