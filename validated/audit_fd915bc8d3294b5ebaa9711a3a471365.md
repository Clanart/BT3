Looking at the `upsert` function and the `insert_entry_to_blob` / `add_leaf` internals carefully.

### Title
Missing `HashAlreadyPresent` Guard in `MerkleBlob::upsert` Silently Corrupts `leaf_hash_to_index` Cache — (`crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::upsert` is missing the `HashAlreadyPresent` check that `insert` enforces. When called with a `new_hash` that already belongs to a different leaf, it silently overwrites that leaf's entry in `leaf_hash_to_index`, leaving the DataLayer Merkle blob in a permanently inconsistent state that fails `check_integrity`.

---

### Finding Description

`insert` guards against hash collisions at lines 372–374: [1](#0-0) 

`upsert`, when the key already exists, has no equivalent guard. Its update path is: [2](#0-1) 

Step-by-step:

1. **Line 798** — `remove_leaf(&leaf)` removes the old hash H1 from `leaf_hash_to_index` and the key from `key_to_index`. [3](#0-2) 

2. **Line 803** — `insert_entry_to_blob` writes the updated block and then calls `add_leaf`. [4](#0-3) 

3. **`add_leaf`** calls `HashMap::insert(leaf.hash, index)` which **silently overwrites** any existing entry for H2 — the entry that previously pointed to k2's index. [5](#0-4) 

**Post-upsert state when `upsert(k1, v1_new, H2)` is called and H2 is already k2's hash:**

| Structure | Expected | Actual |
|---|---|---|
| blob | k1→H2, k2→H2 | k1→H2, k2→H2 (two leaves share H2) |
| `key_to_index` | k1→idx1, k2→idx2 | k1→idx1, k2→idx2 (correct) |
| `leaf_hash_to_index` | H1→idx1, H2→idx2 | H2→idx1 only (k2's entry silently overwritten) |

The note about "midway failure" in the question is a red herring — no partial failure is needed. The corruption is unconditional on every successful call.

---

### Impact Explanation

- `check_integrity` at lines 868–874 asserts `leaf_count == leaf_hash_to_index_cache_length`. With 2 leaves but only 1 entry in `leaf_hash_to_index`, it returns `IntegrityLeafHashToIndexCacheLength`. [6](#0-5) 

- `get_node_by_hash(H2)` returns k1's `(KeyId, ValueId)` instead of k2's — wrong data is served for a valid hash lookup. [7](#0-6) 

- The DataLayer committed state is permanently corrupted; the only recovery is rebuilding the blob from scratch via `BlockStatusCache::new`. [8](#0-7) 

---

### Likelihood Explanation

The Python binding exposes `upsert` with `new_hash` as a direct caller-controlled parameter: [9](#0-8) [10](#0-9) 

Existing leaf hashes are observable through `get_proof_of_inclusion`, `get_node_by_hash`, and `get_hashes_indexes`. No SHA-256 collision is required — the attacker simply reuses a known existing leaf hash as `new_hash` for a different key. The call succeeds with `Ok(())` and leaves the tree corrupted.

---

### Recommendation

Add the same guard that `insert` uses, but conditioned on the hash actually changing (to allow value-only updates with the same hash, as exercised by existing tests):

```rust
pub fn upsert(&mut self, key: KeyId, value: ValueId, new_hash: &Hash) -> Result<(), Error> {
    let Ok((leaf_index, mut leaf, mut block)) = self.get_leaf_by_key(key) else {
        self.insert(key, value, new_hash, InsertLocation::Auto {})?;
        return Ok(());
    };

+       return Err(Error::HashAlreadyPresent());
+   }

}
```

### Proof of Concept

```rust
let mut blob = MerkleBlob::new(vec![]).unwrap();
let h1 = Hash(Bytes32::new([1u8; 32]));
let h2 = Hash(Bytes32::new([2u8; 32]));

blob.insert(KeyId(1), ValueId(10), &h1, InsertLocation::Auto {}).unwrap();
blob.insert(KeyId(2), ValueId(20), &h2, InsertLocation::Auto {}).unwrap();

// upsert k1 to use h2 — no error returned
blob.upsert(KeyId(1), ValueId(99), &h2).unwrap();

// leaf_hash_to_index now has only one entry (h2 → k1's index)
// k2's hash entry is silently gone
assert!(blob.check_integrity().is_err()); // IntegrityLeafHashToIndexCacheLength
```

```rust
        let mut key_to_index: HashMap<KeyId, TreeIndex> = HashMap::default();
        let mut leaf_hash_to_index: HashMap<Hash, TreeIndex> = HashMap::default();

        for item in LeftChildFirstIterator::new(blob, None) {
            let (index, block) = item?;
            seen_indexes.set(index.0 as usize, true);

            if let Node::Leaf(leaf) = block.node {
                if key_to_index.insert(leaf.key, index).is_some() {
                    return Err(Error::KeyAlreadyPresent());
                }
                if leaf_hash_to_index.insert(leaf.hash, index).is_some() {
                    return Err(Error::HashAlreadyPresent());
                }
            }
        }

        let mut free_indexes: IndexSet<TreeIndex> = IndexSet::new();
        for (index, seen) in seen_indexes.iter().enumerate() {
            if !seen {
                free_indexes.insert(TreeIndex(index as u32));
            }
        }

        Ok(Self {
            free_indexes,
            key_to_index,
            leaf_hash_to_index,
        })
```
