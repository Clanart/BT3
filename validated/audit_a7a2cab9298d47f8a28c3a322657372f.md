### Title
`ProofOfInclusion::valid()` Final Root-Hash Check Is Tautological ‚Äî Forged Inclusion Proofs Pass Validation - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` is the sole public API for verifying a DataLayer Merkle proof of inclusion. Its final assertion ‚Äî `existing_hash == self.root_hash()` ‚Äî is always `true` when the loop completes without returning `false`, making it a tautology. The function therefore only verifies internal self-consistency of the proof chain, not that the proof's root matches any external trusted tree root. An attacker who supplies a crafted, internally-consistent `ProofOfInclusion` (deserialized from untrusted bytes) will always pass `valid()`, regardless of whether the claimed key-value pair is actually in the DataLayer tree.

### Finding Description

`ProofOfInclusion::valid()` iterates over `self.layers`, computing `calculated_hash` from the running `existing_hash` and each layer's `other_hash`. At each step it asserts `calculated_hash == layer.combined_hash`; if not, it returns `false`. After the loop, `existing_hash` holds the last `calculated_hash`, which by the loop invariant equals the last `layer.combined_hash`. The final line then compares `existing_hash` against `self.root_hash()`: [1](#0-0) 

`root_hash()` is defined as: [2](#0-1) 

When `layers` is non-empty, `root_hash()` returns `last.combined_hash`. After the loop, `existing_hash` is exactly that same `last.combined_hash`. The comparison `existing_hash == self.root_hash()` is therefore always `true` ‚Äî it is a tautology that adds no security guarantee.

The `ProofOfInclusion` struct derives `Streamable`, meaning it can be deserialized from arbitrary bytes supplied by an untrusted peer: [3](#0-2) 

An attacker can craft a `ProofOfInclusion` with any `node_hash` (e.g., `SHA256(fake_key || fake_value)`) and a chain of `layers` whose `combined_hash` values are internally consistent but correspond to a completely different (or non-existent) tree root. `valid()` returns `true` for this forged proof.

The Python binding exposes `valid()` as the primary verification method: [4](#0-3) 

The fuzz target and integration tests call only `proof.valid()` without cross-checking `proof.root_hash()` against a trusted external root: [5](#0-4) 

### Impact Explanation

Any verifier (Python or Rust) that calls `proof.valid()` as the sole check accepts forged DataLayer inclusion proofs. An attacker can prove that an arbitrary key-value pair is "in" the DataLayer without it actually being present, or prove membership under a fabricated root. This lets untrusted input prove invalid state ‚Äî matching the allowed High impact: *DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.*

### Likelihood Explanation

`ProofOfInclusion` is `Streamable` and exposed over the Python/WASM boundary. Any DataLayer client that receives proofs from peers and validates them with `proof.valid()` alone is vulnerable. The attack requires only constructing a self-consistent proof chain, which is trivial arithmetic over SHA-256.

### Recommendation

`valid()` must accept an external trusted root hash parameter and compare the computed root against it, rather than against the proof's own `root_hash()`. Alternatively, rename the current method to `internally_consistent()` and add a separate `valid(trusted_root: &Hash) -> bool` that performs the external root check. All call sites (Python bindings, fuzz targets, tests) must be updated to supply the trusted root.

```rust
pub fn valid(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root  // compare against external trusted root
}
```

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
import hashlib

# Forge a proof claiming key_hash is in the tree
fake_node_hash = hashlib.sha256(b"fake_key_value").digest()
fake_other_hash = hashlib.sha256(b"sibling").digest()

# Compute a consistent combined_hash
import struct
# Side 0 = Left: combined = SHA256(1 || fake_node_hash || fake_other_hash)
combined = hashlib.sha256(b"\x01" + fake_node_hash + fake_other_hash).digest()

layer = ProofOfInclusionLayer(
    other_hash_side=0,
    other_hash=fake_other_hash,
    combined_hash=combined,
)
proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

# valid() returns True for a completely forged proof
assert proof.valid(), "Forged proof accepted!"
# proof.root_hash() == combined, which is attacker-controlled
```

The forged proof passes `valid()` because the tautological final check `existing_hash == self.root_hash()` always holds. No knowledge of the real DataLayer tree is required.
