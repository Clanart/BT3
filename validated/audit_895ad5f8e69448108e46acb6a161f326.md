### Title
Hardcoded `DEFAULT_INITIAL_GAS_COST` in Deprecated Syscall Path Enables Infinite Recursion Leading to Network Halt — (File: `execution/deprecated_execute_syscalls.cairo`)

---

### Summary

The `contract_call_helper` function in `deprecated_execute_syscalls.cairo` assigns a hardcoded `DEFAULT_INITIAL_GAS_COST` (10,000,000,000) as the gas budget for every call from a CairoZero (deprecated) contract into a Cairo 1 contract. This gas is not derived from the caller's remaining gas, breaking the gas-depletion invariant that prevents infinite recursion. An unprivileged attacker can construct a cross-version recursive call cycle (CairoZero → Cairo1 → CairoZero → …) where the gas is refreshed on every deprecated-path leg, enabling unbounded recursion that exhausts node memory and halts block production.

---

### Finding Description

In the deprecated syscall handler, when a CairoZero contract issues a `call_contract` or `library_call` syscall, the OS executes the callee via `contract_call_helper`: [1](#0-0) 

```cairo
// Set enough gas for this call to succeed.
// This is needed since the caller contract is of version 0 and has no notion of gas, and
// the callee may be of version 1.0.
let remaining_gas = DEFAULT_INITIAL_GAS_COST;
with remaining_gas {
    let (is_reverted, retdata_size, retdata, _is_deprecated) = select_execute_entry_point_func(
        block_context=block_context, execution_context=execution_context
    );
}
```

`DEFAULT_INITIAL_GAS_COST` is hardcoded to `10_000_000_000`: [2](#0-1) 

This gas is **always reset to the same constant**, regardless of how much gas the transaction or the calling contract has already consumed. It is never derived from the caller's `remaining_gas`.

By contrast, the Cairo 1 syscall path correctly forwards only the gas remaining after deducting the syscall cost: [3](#0-2) 

```cairo
let (success, remaining_gas) = reduce_syscall_base_gas(
    specific_base_gas_cost=CALL_CONTRACT_GAS_COST, ...
);
// ...
contract_call_helper(remaining_gas=remaining_gas, ...);
```

The deprecated path has no such deduction — it always provides a fresh `10_000_000_000` gas budget to the callee.

A secondary instance of the same pattern exists in `charge_fee`: [4](#0-3) 

```cairo
let remaining_gas = DEFAULT_INITIAL_GAS_COST;
non_reverting_select_execute_entry_point_func{remaining_gas=remaining_gas}(
    block_context=block_context, execution_context=&execution_context
);
```

---

### Impact Explanation

**High — Network not being able to confirm new transactions (total network shutdown).**

The StarkNet OS is a Cairo program that must be fully executed and proven for each block. If a transaction causes the OS Cairo program to recurse infinitely, the prover process will consume unbounded memory and either crash (OOM) or run forever. Either outcome halts block production: the sequencer cannot finalize the block, consensus times out, and the network stops confirming transactions. This matches the exact impact described in the reference report (Nibiru localnet memory ramp-up and block production halt).

---

### Likelihood Explanation

**High.** The attack requires only:
1. Declaring a CairoZero class (permissionless; CairoZero declarations with version 1/2 transactions are still accepted by the OS — `execute_declare_transaction` only skips version 0).
2. Deploying a CairoZero contract A whose function calls a Cairo1 contract B.
3. Deploying a Cairo1 contract B whose function calls back into A.
4. Submitting a single invoke transaction to A.

No privileged access, leaked keys, or external dependencies are required. The attack is fully self-contained and reachable by any unprivileged transaction sender.

---

### Recommendation

Replace the hardcoded `DEFAULT_INITIAL_GAS_COST` in the deprecated `contract_call_helper` with a value derived from the transaction's actual remaining gas at the point of the call. Since CairoZero callers have no gas concept, the OS should cap the gas forwarded to the callee at the remaining transaction-level gas budget (analogous to the EIP-150 63/64 rule), not at an unbounded constant. Concretely:

- Thread the transaction-level `remaining_gas` through the deprecated syscall execution path.
- In `contract_call_helper` (deprecated), use `min(remaining_gas, DEFAULT_INITIAL_GAS_COST)` rather than the constant alone, and deduct the consumed gas after the call returns.

The same fix should be applied to `charge_fee` in `transaction_impls.cairo`.

---

### Proof of Concept

**Attack contracts:**

```cairo
// CairoZero contract A (Solidity-style pseudocode for clarity)
@external
func attack{syscall_ptr: felt*, ...}() {
    // Calls Cairo1 contract B — OS gives B DEFAULT_INITIAL_GAS_COST (10B) gas
    call_contract(contract_address=B, function_selector='recurse', ...);
    return ();
}
```

```cairo
// Cairo1 contract B
#[external]
fn recurse(ref self: ContractState) {
    // Calls CairoZero contract A — OS deducts CALL_CONTRACT_GAS_COST (91,560) from B's gas
    // Then A calls B again — OS gives B a FRESH 10,000,000,000 gas
    IContractA::attack(contract_address: A);
}
```

**Execution trace:**

```
Tx → A::attack()
  → [deprecated path] B::recurse()  [B gets 10,000,000,000 gas]
    → [Cairo1 path]   A::attack()   [B loses 91,560 gas]
      → [deprecated path] B::recurse()  [B gets 10,000,000,000 gas AGAIN]
        → [Cairo1 path]   A::attack()
          → ... (infinite)
```

Each iteration of the cycle allocates new Cairo memory frames. The node's memory grows without bound until the process is killed, halting block production.
