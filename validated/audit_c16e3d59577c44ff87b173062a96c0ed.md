Audit Report

## Title
Router's `pay()` Consumes Stale Native ETH From Prior Transactions to Subsidize Any Subsequent WETH Swap — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary

`PeripheryPayments.pay()` reads `address(this).balance` — the router's **total** native ETH balance — to decide how much ETH to wrap when `token == WETH`. It does not scope this to the current transaction's `msg.value`. Any ETH stranded in the router from a prior caller who omitted `refundETH()` is silently consumed to settle the next caller's WETH swap input, constituting an unauthorized transfer of the victim's funds.

## Finding Description

`pay()` branches on `nativeBalance = address(this).balance`: [1](#0-0) 

If `nativeBalance >= value`, the router wraps its entire ETH balance up to `value` and forwards it to the pool — pulling nothing from `payer`. This balance is not scoped to the current call.

The `receive()` guard only fires on plain ETH transfers (no calldata): [2](#0-1) 

All four swap entry points are `payable`, so ETH sent alongside a function call bypasses `receive()` entirely and is credited silently to the contract balance: [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) 

**Attack path:**

1. **User A** calls `exactInputSingle{value: 2 ether}(tokenIn=WETH, amountIn=500)` without `refundETH()`. The callback `_justPayCallback` wraps 500 wei; `~2 ether − 500 wei` remains in the router.
2. **User B** calls `exactInputSingle(tokenIn=WETH, amountIn=1000)` with zero ETH.
3. The callback invokes `pay(WETH, userB, pool, 1000)`: [7](#0-6) 

4. `nativeBalance ≈ 2 ether − 500 wei ≥ 1000`, so the router wraps 1000 wei of **User A's ETH** and transfers it to the pool. User B's WETH balance is untouched.

No existing guard prevents this: there is no check that `address(this).balance == msg.value` at call entry, no per-transaction ETH budget, and no mechanism to reject calls when stale ETH is present.

## Impact Explanation

Direct loss of user principal. User A's stranded ETH is permanently transferred to the pool on behalf of an unrelated caller. The victim loses the full stranded amount; the attacker receives a free or discounted swap. This meets the Sherlock Critical/High threshold for direct loss of user funds via an unprivileged, externally reachable path.

## Likelihood Explanation

All four swap entry points are `payable`, so ETH can be stranded with any of them. `refundETH()` is optional and must be composed manually (typically via `multicall`); omitting it is a realistic user error, especially for direct (non-multicall) invocations. An attacker can monitor the mempool or simply call a WETH swap after any block where the router's ETH balance is non-zero. The attack is repeatable with no special privileges.

## Recommendation

Scope the native ETH available to `pay()` to only the ETH sent in the current transaction. The standard pattern is to pass `msg.value` (or a tracked per-call ETH budget) into `pay()` and decrement it as it is consumed, rather than reading `address(this).balance`. Alternatively, revert at the start of each top-level call if `address(this).balance` exceeds `msg.value` (i.e., reject calls when stale ETH is present).

## Proof of Concept

```solidity
function test_staleEthSubsidizesSwap() public {
    // User A sends 2 ETH but only swaps 500 wei; forgets refundETH
    vm.prank(userA);
    router.exactInputSingle{value: 2 ether}(ExactInputSingleParams({
        pool: wethPool, tokenIn: weth, tokenOut: token1,
        zeroForOne: true, amountIn: 500, amountOutMinimum: 0,
        recipient: userA, deadline: block.timestamp + 1,
        priceLimitX64: 0, extensionData: ""
    }));
    assertGt(address(router).balance, 0); // ~2 ether stranded

    uint256 userBWethBefore = IERC20(weth).balanceOf(userB);

    // User B swaps 1000 WETH with no ETH sent
    vm.prank(userB);
    router.exactInputSingle(ExactInputSingleParams({
        pool: wethPool, tokenIn: weth, tokenOut: token1,
        zeroForOne: true, amountIn: 1000, amountOutMinimum: 0,
        recipient: userB, deadline: block.timestamp + 1,
        priceLimitX64: 0, extensionData: ""
    }));

    // User B's WETH balance unchanged — router's ETH paid for them
    assertEq(IERC20(weth).balanceOf(userB), userBWethBefore);
    // 1000 wei of User A's ETH consumed for User B
    assertEq(address(router).balance, 2 ether - 500 - 1000);
}
```
