### Title
Partial Batch EVM Transaction Replay / Frontrunning via Per-Message Signature Not Bound to Complete Batch Payload — (`ante/eth.go`)

### Summary
Ethermint supports batching multiple `MsgEthereumTx` messages in a single Cosmos transaction. Each message carries a self-contained Ethereum signature covering only its own fields (nonce, to, value, data, chainID). The ante handler increments nonces for all messages upfront before execution. If the batch fails, the entire Cosmos transaction reverts including nonce increments. Because each individual `MsgEthereumTx` is a valid, independently-submittable signed Ethereum transaction, an attacker who observes the mempool can extract a subset of messages from a pending or failed batch and submit them as a separate transaction, executing only a partial subset of the intended atomic operation.

### Finding Description

In `CheckAndSetEthSenderNonce` (`ante/eth.go`), the ante handler iterates over all messages in a batch and increments each sender's sequence number upfront before any EVM execution occurs: [1](#0-0) 

For a batch of 3 messages from the same sender, nonces are pre-incremented N → N+3 before any message executes. If the Cosmos transaction fails (e.g., one `MsgEthereumTx` reverts at the EVM level), the entire transaction reverts including these nonce increments, returning nonces to N.

Each `MsgEthereumTx` is a complete, self-contained signed Ethereum transaction. Its signature covers only its own fields — not the complete batch payload. Because Ethermint's EVM ante handler authenticates via the Ethereum signature (not a Cosmos-level signature over the full batch): [2](#0-1) 

any individual `MsgEthereumTx` can be wrapped in a new Cosmos transaction and submitted independently. The `newEthAnteHandler` in `evmd/ante/handler_options.go` calls `VerifyEthSig` per-message, with no check that the message belongs to a specific batch: [3](#0-2) 

**Attack vectors:**

1. **Frontrunning**: Attacker observes a pending batch `[MsgA(N), MsgB(N+1), MsgC(N+2)]` in the mempool

func VerifyEthSig(tx sdk.Tx, signer ethtypes.Signer) error {
	for _, msg := range tx.GetMsgs() {
		msgEthTx, ok := msg.(*evmtypes.MsgEthereumTx)
		if !ok {
			return errorsmod.Wrapf(errortypes.ErrUnknownRequest, "invalid message type %T, expected %T", msg, (*evmtypes.MsgEthereumTx)(nil))
		}

		if err := msgEthTx.VerifySender(signer); err != nil {
			return errorsmod.Wrapf(errortypes.ErrorInvalidSigner, "signature verification failed: %s", err.Error())
		}
	}

	return nil
