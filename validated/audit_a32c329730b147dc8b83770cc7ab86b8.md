### Title
`starknet_getStorageAt` Pending-Block Contract Existence Check Omits Pending Deployed Contracts, Returning Authoritative `CONTRACT_NOT_FOUND` for Valid Pending Contracts - (File: crates/apollo_rpc/src/v0_8/api/api_impl.rs)

### Summary

The `get_storage_at` RPC handler checks contract existence only against committed (finalized) state when the queried storage value is zero, completely ignoring pending deployed contracts. The sibling handler `get_class_hash_at` (via `maybe_get_class_hash_at`) correctly consults pending deployed contracts. Any unprivileged caller can observe an authoritative `CONTRACT_NOT_FOUND` error for a contract that genuinely exists in the pending block, producing an inconsistent and wrong RPC view of pending state.

### Finding Description

In `crates/apollo_rpc/src/v0_8/api/api_impl.rs`, `get_storage_at` (lines 339–384) handles a `BlockId::Tag(Tag::Pending)` request as follows:

1. It reads `pending_data.state_update.state_diff.storage_diffs` and passes them to `execution_utils::get_storage_at`.
2. `execution_utils::get_storage_at` (lines 151–169 of `crates/apollo_rpc_execution/src/execution_utils.rs`) returns the pending storage value if the key is present in the pending diff; otherwise it falls back to committed storage.
3. When the result `res` equals `Felt::default()` (zero), the handler performs a contract existence guard at lines 375–381:

```rust
if res == Felt::default() && contract_address != BLOCK_HASH_TABLE_ADDRESS {
    txn.get_state_reader()
        .map_err(internal_server_error)?
        .get_class_hash_at(state_number, &contract_address)   // ← committed state only
        .map_err(internal_server_error)?
        .ok_or_else(|| ErrorObjectOwned::from(CONTRACT_NOT_FOUND))?;
}
```

This guard calls `txn.get_state_reader().get_class_hash_at(state_number, &contract_address)`, which queries **only committed storage**. It does not consult `pending_state_diff.deployed_contracts` or `pending_state_diff.replaced_classes`.

By contrast, `maybe_get_class_hash_at` (lines 1649–1675) correctly reads both pending deployed contracts and replaced classes from `pending_data`, then passes them to `execution_utils::get_class_hash_at` (lines 190–215 of `execution_utils.rs`), which searches the pending lists before falling back to committed state:

```rust
let maybe_pending_deployed_contracts_and_replaced_classes =
    if let BlockId::Tag(Tag::Pending) = block_id {
        let pending_state_diff =
            read_pending_data(&self.pending_data, &txn).await?.state_update.state_diff;
        Some((pending_state_diff.deployed_contracts, pending_state_diff.replaced_classes))
    } else { None };
// ...
execution_utils::get_class_hash_at(&txn, state_number,
    maybe_pending_deployed_contracts_and_replaced_classes.as_ref().map(|t| (&t.0, &t.1)),
    contract_address)
```

**Concrete scenario triggering the wrong value:**

A contract is deployed only in the pending block (present in `pending_state_diff.deployed_contracts`, absent from committed state). The caller queries a storage key whose value is zero (either never written, or written as zero in the pending diff).

- `execution_utils::get_storage_at` finds no matching entry in the pending diff and falls back to committed storage, returning `Felt::ZERO`.
- `res == Felt::default()` → the guard fires.
- `txn.get_state_reader().get_class_hash_at(state_number, &contract_address)` returns `None` because the contract is not in committed state.
- The handler returns `CONTRACT_NOT_FOUND`.

Meanwhile, `starknet_getClassHashAt` with the same `block_id = "pending"` and the same address correctly returns the class hash, because `maybe_get_class_hash_at` checks pending deployed contracts.

The two RPC methods give contradictory authoritative answers about the same pending state.

### Impact Explanation

`starknet_getStorageAt` is a standard Starknet JSON-RPC method. Clients (wallets, explorers, dApps, fee estimators) treat its response as authoritative. Returning `CONTRACT_NOT_FOUND` for a contract that exists in the pending block is a wrong authoritative value. A client that uses `getStorageAt` to probe whether a pending-deployed contract is live will incorrectly conclude the contract does not exist, while `getClassHashAt` on the same address returns a valid class hash. This inconsistency breaks any logic that cross-checks pending state across these two methods, and can cause fee estimation or simulation calls that depend on pending storage reads to fail or produce wrong results for newly deployed contracts whose storage slots are at their default zero value.

This matches: **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

### Likelihood Explanation

The trigger requires no privilege. Any caller can:
1. Submit a `deploy_account` or `deploy` transaction that lands in the pending block.
2. Immediately call `starknet_getStorageAt` with `block_id = "pending"` for any storage key that has not been explicitly written (i.e., value is zero).

This is a routine operation during normal pending-block interaction. The inconsistency is permanent until the contract is committed to a finalized block.

### Recommendation

The contract existence guard in `get_storage_at` must also consult pending deployed contracts, mirroring the logic in `maybe_get_class_hash_at`. Specifically, replace the raw `txn.get_state_reader().get_class_hash_at(...)` call with `execution_utils::get_class_hash_at(...)` supplied with the same pending deployed contracts and replaced classes that were already read for the storage diff lookup:

```rust
// Read pending deployed contracts alongside storage diffs
let (maybe_pending_storage_diffs, maybe_pending_deployed) =
    if let BlockId::Tag(Tag::Pending) = block_id {
        let diff = read_pending_data(&self.pending_data, &txn).await?.state_update.state_diff;
        (Some(diff.storage_diffs), Some((diff.deployed_contracts, diff.replaced_classes)))
    } else { (None, None) };

// ... existing storage lookup ...

if res == Felt::default() && contract_address != BLOCK_HASH_TABLE_ADDRESS {
    execution_utils::get_class_hash_at(
        &txn,
        state_number,
        maybe_pending_deployed.as_ref().map(|t| (&t.0, &t.1)),
        contract_address,
    )
    .map_err(internal_server_error)?
    .ok_or_else(|| ErrorObjectOwned::from(CONTRACT_NOT_FOUND))?;
}
```

### Proof of Concept

1. Ensure the node has a live pending block (parent hash matches latest committed block).
2. Submit a `deploy_account` transaction; confirm it appears in the pending block's transaction list.
3. Call `starknet_getClassHashAt` with `block_id = "pending"` and the deployed contract address → returns the class hash (contract found in pending).
4. Call `starknet_getStorageAt` with `block_id = "pending"`, the same contract address, and any storage key (e.g., `0x0`) → returns `CONTRACT_NOT_FOUND` (wrong; should return `0x0`).
5. The two responses are contradictory for the same pending state, confirming the inconsistency.

pub fn get_storage_at<Mode: TransactionKind>(
    txn: &StorageTxn<'_, Mode>,
    state_number: StateNumber,
    pending_storage_diffs: Option<&IndexMap<ContractAddress, Vec<StorageEntry>>>,
    contract_address: ContractAddress,
    key: StorageKey,
) -> StorageResult<Felt> {
    if let Some(pending_storage_diffs) = pending_storage_diffs {
        if let Some(storage_entries) = pending_storage_diffs.get(&contract_address) {
            if let Some(StorageEntry { key: _, value }) = storage_entries
                .iter()
                .find(|StorageEntry { key: other_key, value: _ }| key == *other_key)
            {
                return Ok(*value);
            }
        }
    }
    txn.get_state_reader()?.get_storage_at(state_number, &contract_address, &key)
}

pub fn get_class_hash_at<Mode: TransactionKind>(
    txn: &StorageTxn<'_, Mode>,
    state_number: StateNumber,
    pending_deployed_contracts_and_replaced_classes: Option<(
        &Vec<DeployedContract>,
        &Vec<ReplacedClass>,
    )>,
    contract_address: ContractAddress,
) -> StorageResult<Option<ClassHash>> {
    if let Some((pending_deployed_contracts, pending_replaced_classes)) =
        pending_deployed_contracts_and_replaced_classes
    {
        // Searching first in the replaced classes because if the contract was deployed and
        // replaced, the replaced class is the contract's class.
        for ReplacedClass { address, class_hash } in pending_replaced_classes {
            if *address == contract_address {
                return Ok(Some(*class_hash));
            }
        }
        for DeployedContract { address, class_hash } in pending_deployed_contracts {
            if *address == contract_address {
                return Ok(Some(*class_hash));
            }
        }
    }
    txn.get_state_reader()?.get_class_hash_at(state_number, &contract_address)
