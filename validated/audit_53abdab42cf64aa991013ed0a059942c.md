Let me look at how `set_state_header` is called from the network layer and check the `verify_path` implementation to understand if it's position-sensitive.

Looking at the production code in `set_state_header` and the `verify_path` implementation, I need to determine whether the shard_id embedded in the chunk is ever compared to the `shard_id` parameter before the DB write.

**Key findings:**

**1. `verify_path` is NOT position-sensitive** [1](#0-0) 

`verify_path` calls `compute_root_from_path(path, item_hash) == root`. It only checks that the path leads from the leaf hash to the root — it does NOT check which position (shard index) in the tree the leaf occupies. A valid proof for shard B's position passes `verify_path` even when the caller expects shard A's position.

**2. The chunk proof check in `set_state_header` does not bind the chunk to `shard_id`** [2](#0-1) 

This verifies that the chunk hash is *somewhere* in the block's `chunk_headers_root` merkle tree, but never verifies that the proof corresponds to shard A's index. An attacker supplying chunk_B with proof_B (valid for shard B's position) satisfies this check while `shard_id=A`.

**3. No `chunk.shard_id() == shard_id` guard exists anywhere in `set_state_header`** [3](#0-2) 

The DB write uses `StateHeaderKey(shard_id, sync_hash)` — the `shard_id` parameter — without ever asserting that `chunk.shard_id()` matches it. `validate_chunk_proofs` checks internal chunk consistency (signatures, etc.), not the shard_id parameter alignment.

**4. `verify_path_with_index` exists but is not used here** [4](#0-3) 

This function explicitly checks that the path's direction bits correspond to a specific index. `set_state_header` uses the weaker `verify_path` instead.

**5. The downstream `set_state_part` reads the corrupted header** [5](#0-4) 

`set_state_part` reads the stored header for `shard_id=A`, extracts `prev_state_root` from the chunk embedded in it (which is chunk_B's state root), and validates/installs state parts against that root. If the header was corrupted, shard B's state root drives shard A's state installation.

### Title
Missing shard_id-to-chunk binding in `set_state_header` allows cross-shard state root substitution — (`chain/chain/src/state_sync/adapter.rs`)

### Summary
`set_state_header` accepts a `ShardStateSyncResponseHeader` from a peer for `shard_id=A` but never asserts that the embedded `chunk.shard_id()` equals `A`. The merkle proof check uses `verify_path`, which is position-agnostic: a valid proof for shard B's chunk at shard B's position in the block satisfies the check even when the caller intends shard A. The header is then written to `DBCol::StateHeaders` under `StateHeaderKey(shard_id=A, sync_hash)` with shard B's `prev_state_root`, causing `set_state_part` and `set_state_finalize` to install shard B's trie state into shard A.

### Finding Description
In `set_state_header` (`chain/chain/src/state_sync/adapter.rs`, lines 368–532):

- The function receives `shard_id` (A) and `shard_state_header` (containing chunk_B).
- Step 3a calls `verify_path(chunk_headers_root, chunk_proof, ChunkHashHeight(chunk_B.hash, chunk_B.height_included))`. Because `verify_path` only reconstructs the root from the path without checking which leaf index the path encodes, a proof for shard B's position passes when chunk_B is supplied.
- No statement of the form `chunk.shard_id() == shard_id` appears anywhere in the function.
- The write at line 527–528 stores the header under `StateHeaderKey(shard_id=A, sync_hash)`.

An attacker who is selected as the state-sync provider for shard A (any unprivileged peer can be selected) constructs a `ShardStateSyncResponseHeader` containing:
- chunk_B (a real, validly-signed chunk from shard B, public blockchain data)
- proof_B (the real merkle proof for chunk_B at shard B's position, public blockchain data)
- incoming receipts for shard A (public blockchain data, satisfying the receipt proof loop at lines 447–503 which hashes against the `shard_id` parameter)
- state_root_node for shard B (public blockchain data)

All five validation steps pass. The header is stored under shard A's key with shard B's `prev_state_root`.

### Impact Explanation
`set_state_part` reads `StateHeaderKey(shard_id=A, sync_hash)`, extracts `prev_state_root` from the embedded chunk (chunk_B's root), and validates incoming state parts against that root. The attacker then supplies state parts consistent with shard B's trie. `set_state_finalize` installs this trie as shard A's state. The syncing node operates with shard B's account/access-key/contract state under shard A's identity, corrupting all subsequent execution for shard A.

### Likelihood Explanation
Exploitability requires the attacker to be the peer selected by the syncing node for state sync. Peer selection is not restricted to validators; any reachable peer advertising the snapshot can be chosen. The attacker needs only public, on-chain data (chunk, proof, receipts, state_root_node for shard B). No cryptographic forgery is required.

### Recommendation
Add an explicit shard_id guard immediately after extracting the chunk in `set_state_header`:

```rust
if chunk.shard_id() != shard_id {
    return Err(Error::Other(
        "set_shard_state failed: chunk.shard_id() does not match shard_id parameter".into(),
    ));
}
```

Alternatively, replace the `verify_path` call with `verify_path_with_index` (already present in `core/primitives/src/merkle.rs`) using the shard's index in the block, which would bind the proof to the correct leaf position and make the shard_id implicit in the proof verification.

    pub fn set_state_header(
        &self,
        shard_id: ShardId,
        sync_hash: CryptoHash,
        shard_state_header: ShardStateSyncResponseHeader,
    ) -> Result<(), Error> {
        let sync_block_header = self.chain_store.get_block_header(&sync_hash)?;

        let chunk = shard_state_header.cloned_chunk();
        let prev_chunk_header = shard_state_header.cloned_prev_chunk_header();

        // 1-2. Checking chunk validity
        if !validate_chunk_proofs(&chunk, self.epoch_manager.as_ref())? {
            byzantine_assert!(false);
            return Err(Error::Other(
                "set_shard_state failed: chunk header proofs are invalid".into(),
            ));
        }

        // Consider chunk itself is valid.

        // 3. Checking that chunks `chunk` and `prev_chunk` are included in appropriate blocks
        // 3a. Checking that chunk `chunk` is included into block at last height before sync_hash
        // 3aa. Also checking chunk.height_included
        let sync_prev_block_header =
            self.chain_store.get_block_header(sync_block_header.prev_hash())?;
        if !verify_path(
            *sync_prev_block_header.chunk_headers_root(),
            shard_state_header.chunk_proof(),
            &ChunkHashHeight(chunk.chunk_hash().clone(), chunk.height_included()),
        ) {
            byzantine_assert!(false);
            return Err(Error::Other(
                "set_shard_state failed: chunk isn't included into block".into(),
            ));
        }

        let block_header = get_block_header_on_chain_by_height(
            &self.chain_store,
            &sync_hash,
            chunk.height_included(),
        )?;
        // 3b. Checking that chunk `prev_chunk` is included into block at height before chunk.height_included
        // 3ba. Also checking prev_chunk.height_included - it's important for getting correct incoming receipts
        match (&prev_chunk_header, shard_state_header.prev_chunk_proof()) {
            (Some(prev_chunk_header), Some(prev_chunk_proof)) => {
                let prev_block_header =
                    self.chain_store.get_block_header(block_header.prev_hash())?;
                if !verify_path(
                    *prev_block_header.chunk_headers_root(),
                    prev_chunk_proof,
                    &ChunkHashHeight(prev_chunk_header.chunk_hash().clone(), prev_chunk_header.height_included()),
                ) {
                    byzantine_assert!(false);
                    return Err(Error::Other(
                        "set_shard_state failed: prev_chunk isn't included into block".into(),
                    ));
                }
            }
            (None, None) => {
                if chunk.height_included() != 0 {
                    return Err(Error::Other(
                    "set_shard_state failed: received empty state response for a chunk that is not at height 0".into()
                ));
                }
            }
            _ =>
                return Err(Error::Other("set_shard_state failed: `prev_chunk_header` and `prev_chunk_proof` must either both be present or both absent".into()))
        };

        // 4. Proving incoming receipts validity
        // 4a. Checking len of proofs
        if shard_state_header.root_proofs().len()
            != shard_state_header.incoming_receipts_proofs().len()
        {
            byzantine_assert!(false);
            return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
        }
        let mut hash_to_compare = sync_hash;
        for (i, receipt_response) in
            shard_state_header.incoming_receipts_proofs().iter().enumerate()
        {
            let ReceiptProofResponse(block_hash, receipt_proofs) = receipt_response;

            // 4b. Checking that there is a valid sequence of continuous blocks
            if *block_hash != hash_to_compare {
                byzantine_assert!(false);
                return Err(Error::Other(
                    "set_shard_state failed: invalid incoming receipts".into(),
                ));
            }
            let header = self.chain_store.get_block_header(&hash_to_compare)?;
            hash_to_compare = *header.prev_hash();

            let block_header = self.chain_store.get_block_header(block_hash)?;
            // 4c. Checking len of receipt_proofs for current block
            if receipt_proofs.len() != shard_state_header.root_proofs()[i].len()
                || receipt_proofs.len() != block_header.chunks_included() as usize
            {
                byzantine_assert!(false);
                return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
            }
            // We know there were exactly `block_header.chunks_included` chunks included
            // on the height of block `block_hash`.
            // There were no other proofs except for included chunks.
            // According to Pigeonhole principle, it's enough to ensure all receipt_proofs are distinct
            // to prove that all receipts were received and no receipts were hidden.
            let mut visited_shard_ids = HashSet::<ShardId>::new();
            for (j, receipt_proof) in receipt_proofs.iter().enumerate() {
                let ReceiptProof(receipts, shard_proof) = receipt_proof;
                let ShardProof { from_shard_id, to_shard_id: _, proof } = shard_proof;
                // 4d. Checking uniqueness for set of `from_shard_id`
                match visited_shard_ids.get(from_shard_id) {
                    Some(_) => {
                        byzantine_assert!(false);
                        return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
                    }
                    _ => visited_shard_ids.insert(*from_shard_id),
                };
                let RootProof(root, block_proof) = &shard_state_header.root_proofs()[i][j];
                let receipts_hash = CryptoHash::hash_borsh(ReceiptList(shard_id, receipts));
                // 4e. Proving the set of receipts is the subset of outgoing_receipts of shard `shard_id`
                if !verify_path(*root, proof, &receipts_hash) {
                    byzantine_assert!(false);
                    return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
                }
                // 4f. Proving the outgoing_receipts_root matches that in the block
                if !verify_path(
                    *block_header.prev_chunk_outgoing_receipts_root(),
                    block_proof,
                    root,
                ) {
                    byzantine_assert!(false);
                    return Err(Error::Other("set_shard_state failed: invalid proofs".into()));
                }
            }
        }
        // 4g. Checking that there are no more heights to get incoming_receipts
        let header = self.chain_store.get_block_header(&hash_to_compare)?;
        if header.height() != prev_chunk_header.map_or(0, |h| h.height_included()) {
            byzantine_assert!(false);
            return Err(Error::Other("set_shard_state failed: invalid incoming receipts".into()));
        }

        // 5. Checking that state_root_node is valid
        let chunk_inner = chunk.take_header().take_inner();
        if matches!(
            self.runtime_adapter.validate_state_root_node(
                shard_state_header.state_root_node(),
                chunk_inner.prev_state_root(),
            ),
            StateRootNodeValidationResult::Invalid
        ) {
            byzantine_assert!(false);
            return Err(Error::Other("set_shard_state failed: state_root_node is invalid".into()));
        }

        // Saving the header data.
        let mut store_update = self.chain_store.store().store_update();
        let key = borsh::to_vec(&StateHeaderKey(shard_id, sync_hash)).unwrap();
        store_update.set_ser(DBCol::StateHeaders, &key, &shard_state_header);
        store_update.commit();

        Ok(())
    }
