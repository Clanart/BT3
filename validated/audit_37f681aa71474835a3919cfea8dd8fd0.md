### Title
`ProofOfInclusion::valid()` Is Self-Referential and Never Validates Against an External Root Hash, Enabling Forged DataLayer Inclusion Proofs — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` only checks the internal consistency of the proof chain. Its terminal assertion `existing_hash == self.root_hash()` is tautologically true whenever the loop completes without returning `false`, because `root_hash()` is derived from the same `layers` field that was just validated. The method never compares the computed root against any caller-supplied expected root. Any `ProofOfInclusion` whose layer hashes are internally consistent will pass `valid()` regardless of which tree it actually belongs to, allowing an attacker to present a structurally valid proof for an entirely different tree root.

### Finding Description

`ProofOfInclusion` is a Streamable type exposed through Python bindings. Its `valid()` method is the sole self-contained verification primitive:

```rust
// crates/chia-datalayer/src/merkle/proof_of_inclusion.rs  lines 40-58
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()   // ← always true when loop completes
}
```

`root_hash()` is:

```rust
// lines 32-38
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← same field just assigned to existing_hash
    } else {
        self.node_hash
    }
}
```

After the loop, `existing_hash` holds the last `calculated_hash`, which equals `last.combined_hash` (the loop would have returned `false` otherwise). `root_hash()` returns that same `last.combined_hash`. Therefore `existing_hash == self.root_hash()` is always `true` when the loop exits normally. The final guard is a no-op.

The analog to the FlywheelCore bug is direct: in that report, `setFlywheelRewards` never checks that `newFlywheelRewards.flywheel == address(this)` — the back-reference from the new component to the parent is never validated. Here, `valid()` never checks that the proof's computed root matches the caller's expected tree root — the back-reference from the proof to the tree it claims to belong to is never validated.

### Impact Explanation

A DataLayer peer or client that receives a `ProofOfInclusion` from an untrusted source and calls only `proof.valid()` to accept or reject it receives no assurance that the proof belongs to the expected tree. An attacker can:

1. Pick any target `node_hash` (claiming arbitrary data is present).
2. Build a chain of `ProofOfInclusionLayer` values whose `combined_hash` fields are each correctly computed from the previous hash and a chosen sibling — a trivial SHA-256 chain.
3. Transmit the resulting `ProofOfInclusion` to a victim.
4. `proof.valid()` returns `true`; `proof.root_hash()` returns the attacker-chosen root.

If the victim does not separately compare `proof.root_hash()` to the authoritative root it holds, it accepts the forged proof. This lets untrusted input prove invalid state in the DataLayer, matching the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."*

### Likelihood Explanation

The Python binding exposes `valid()` as the primary (and only named) verification method on `ProofOfInclusion`. All Rust and Python tests call `assert!(proof.valid())` / `assert proof_of_inclusion.valid()` without a separate root-hash comparison. There is no documentation warning that `root_hash()` must be checked independently. DataLayer sync code that receives proofs over the network and calls `valid()` is the natural, expected usage pattern, making misuse highly probable.

### Recommendation

Replace the tautological terminal check with a comparison against a caller-supplied expected root:

```rust
pub fn valid_for_root(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root   // compare against external root
}
```

Either deprecate `valid()` in favour of `valid_for_root(expected_root)`, or make `valid()` itself require an expected root parameter. Update the Python binding accordingly and add a test that a proof generated for tree A is rejected when verified against tree B's root.

use chia_datalayer::{Hash, Side};
use chia_datalayer::merkle::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};

fn forge_proof(target_node_hash: Hash, sibling: Hash) -> ProofOfInclusion {
    // Compute a single-layer proof whose combined_hash is attacker-chosen.
    let combined = crate::calculate_internal_hash(&target_node_hash, Side::Left, &sibling);
    ProofOfInclusion {
        node_hash: target_node_hash,
        layers: vec![ProofOfInclusionLayer {
            other_hash_side: Side::Left,
            other_hash: sibling,
            combined_hash: combined,   // correctly computed → loop passes
        }],
    }
}

let forged = forge_proof(arbitrary_node_hash, arbitrary_sibling);
assert!(forged.valid());          // passes — no external root checked
// forged.root_hash() == attacker-chosen combined hash, not the real tree root

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
