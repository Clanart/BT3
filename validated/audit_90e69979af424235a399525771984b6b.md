### Title
Missing Declared-Class Existence Check in `execute_replace_class` Allows Permanent Contract Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS Cairo program updates a contract's class hash in the state without verifying that the new class hash corresponds to a previously declared class. An unprivileged contract can call `replace_class` with an arbitrary, undeclared class hash. The OS will produce a valid proof for this state transition, permanently bricking the contract and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class`, the OS reads `request.class_hash` directly from the syscall request and writes it into the contract's `StateEntry` via `dict_update` on `contract_state_changes`. There is no cross-check against `contract_class_changes` (the set of declared classes). The code itself contains an explicit acknowledgment of this missing check:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

Compare this to `execute_declare_transaction`, which enforces `prev_value=0` on `contract_class_changes` to guarantee a class is only declared once:

```cairo
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [2](#0-1) 

The `replace_class` syscall is dispatched to any executing contract without privilege restriction: [3](#0-2) 

**Analogy to BvB bug:** In BvB, `reclaimContract()` transferred funds to `bulls[contractId]` without checking it was ever set (i.e., `!= address(0)`), because the order was never matched. Here, `execute_replace_class` writes an arbitrary `class_hash` into the contract state without checking it was ever declared — the "matching" step (declaration) is never verified.

---

### Impact Explanation

After `replace_class` is called with an undeclared `class_hash`:

1. The contract's `StateEntry.class_hash` is set to an undeclared value in the proven state.
2. Every subsequent call to the contract will attempt to execute using this undeclared class hash.
3. No compiled class exists for this hash, so all future calls to the contract will fail permanently.
4. Any ERC-20 tokens, NFTs, or other assets held in the contract's storage become permanently inaccessible.

**Impact: Critical — Permanent freezing of funds.**

The OS produces a cryptographically valid STARK proof for this invalid state transition, meaning the L1 verifier accepts it. There is no on-chain recovery path.

---

### Likelihood Explanation

- Any unprivileged user can deploy a contract containing a `replace_class(undeclared_hash)` call.
- The syscall is available to all contracts with no access control at the OS level.
- A malicious sequencer can include such a transaction in a block and generate a valid proof.
- The attack requires only a standard `invoke` transaction — no special privileges, leaked keys, or external dependencies.
- The explicit `TODO` comment confirms the developers are aware the check is absent.

---

### Recommendation

Before writing the new `class_hash` into the contract's `StateEntry`, verify that it exists in `contract_class_changes` (i.e., it was declared in the current or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert the result is non-zero (i.e., a compiled class hash was registered for it):

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the enforcement already present in `deploy_contract`, which asserts `state_entry.class_hash = UNINITIALIZED_CLASS_HASH` before deploying: [4](#0-3) 

---

### Proof of Concept

1. Attacker declares a valid contract class `C` (to be able to deploy).
2. Attacker deploys contract `C` at address `A`, which holds or will receive funds.
3. Contract `C` contains an entry point that calls `replace_class(class_hash=0xdeadbeef)` where `0xdeadbeef` is never declared.
4. Attacker (or anyone) calls that entry point on `A`.
5. The OS executes `execute_replace_class`:
   - Reads `request.class_hash = 0xdeadbeef`.
   - Skips the missing existence check (the TODO).
   - Writes `StateEntry(class_hash=0xdeadbeef, ...)` for address `A` into `contract_state_changes`.
6. The OS produces a valid proof. The L1 verifier accepts it.
7. All subsequent calls to `A` fail — no compiled class for `0xdeadbeef` exists.
8. All funds in `A`'s storage are permanently frozen. [5](#0-4)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
