### Title
`SyncStateReader::get_compiled_class_hash` Panics via `todo!()` Instead of Returning Default, Causing Gateway to Reject Valid Declare Transactions - (File: crates/apollo_gateway/src/sync_state_reader.rs)

### Summary

The `SyncStateReader` struct implements the `BlockifierStateReader` (which extends `StateReader`) trait for the gateway's stateful transaction validation path. The `StateReader` trait explicitly documents that `get_compiled_class_hash` must return `CompiledClassHash::default()` when no compiled class hash is found. However, `SyncStateReader::get_compiled_class_hash` is implemented as `todo!()`, which panics unconditionally at runtime. Any Declare transaction submitted to the gateway that causes this method to be called on an uncached class hash will trigger the panic, which is caught by `spawn_blocking` and surfaced as an internal error, causing the gateway to reject the valid transaction.

### Finding Description

The `StateReader` trait defines the contract for all state readers in the sequencer:

```rust
/// Returns the compiled class hash of the given class hash.
/// Returns CompiledClassHash::default() if no v1_class is found for the given class hash.
fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash>;
``` [1](#0-0) 

The `SyncStateReader` — the production state reader used by the gateway for stateful validation — implements this method as:

```rust
fn get_compiled_class_hash(&self, _class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    todo!()
}
``` [2](#0-1) 

This is a direct violation of the trait's documented contract. Every other production `StateReader` implementation correctly returns `CompiledClassHash::default()` for missing entries. For example, `ApolloReader`:

```rust
Ok(None) => Ok(CompiledClassHash::default()),
``` [3](#0-2) 

And `ExecutionStateReader`:

```rust
let Some(block_number) = maybe_block_number else {
    return Ok(CompiledClassHash::default());
};
``` [4](#0-3) 

The call chain that reaches the panic during gateway stateful validation is:

1. `StatefulTransactionValidator::run_validate_entry_point` wraps `SyncStateReader` in `StateReaderAndContractManager`, then in `CachedState`, and calls `blockifier_validator.validate(account_tx)` inside `spawn_blocking`. [5](#0-4) 

2. For Declare transactions, `StatefulValidator::perform_validations` calls `self.execute(tx)` which runs the full transaction execution. [6](#0-5) 

3. During Declare execution, the blockifier calls `CachedState::get_compiled_class_hash` to check whether the class is already declared. If the class hash is not in the cache, it falls through to the underlying state reader. [7](#0-6) 

4. `CachedState` delegates to `StateReaderAndContractManager::get_compiled_class_hash`, which delegates directly to `SyncStateReader::get_compiled_class_hash` → `todo!()` → **panic**. [8](#0-7) 

The panic inside `spawn_blocking` is caught as a `JoinError` and converted to a `StarknetError::InternalError`, causing the gateway to return an error for the valid Declare transaction.

### Impact Explanation

**High. Mempool/gateway/RPC admission rejects valid transactions before sequencing.**

Any Declare transaction submitted to the gateway triggers stateful validation. When the blockifier checks whether the declared class hash is already registered (a mandatory step), it calls `get_compiled_class_hash` on the `SyncStateReader`. Because this method is `todo!()`, it panics. The panic is caught by `spawn_blocking` and the gateway returns an internal error, rejecting the valid Declare transaction. No Declare transaction can pass gateway stateful validation through the `SyncStateReader` path.

### Likelihood Explanation

**High.** The `SyncStateReader` is the primary production state reader used by the gateway in the distributed/hybrid deployment topology. Every Declare transaction submitted to such a gateway will hit this code path. The `todo!()` is not gated behind any feature flag or configuration option.

### Recommendation

Implement `SyncStateReader::get_compiled_class_hash` to query the state sync client for the compiled class hash, returning `CompiledClassHash::default()` when the class is not found — consistent with the `StateReader` trait contract and all other production implementations. Additionally, add a test case for this method in `crates/apollo_gateway/src/sync_state_reader_test.rs` analogous to the existing `test_get_storage_at`, `test_get_nonce_at`, and `test_get_class_hash_at` tests. [9](#0-8) 

### Proof of Concept

1. Deploy a gateway node using `SyncStateReader` (distributed/hybrid topology).
2. Submit a valid Declare transaction (Cairo 1 class, valid signature, valid nonce, sufficient fee).
3. The gateway's `StatefulTransactionValidator::run_validate_entry_point` is invoked.
4. Inside `spawn_blocking`, `StatefulValidator::perform_validations` calls `self.execute(tx)` for the Declare transaction.
5. The blockifier calls `CachedState::get_compiled_class_hash(class_hash)` for the newly declared class hash (not yet in cache).
6. The call propagates: `CachedState` → `StateReaderAndContractManager` → `SyncStateReader::get_compiled_class_hash` → `todo!()` → **panic**.
7. `spawn_blocking` catches the panic as a `JoinError`; the gateway returns `StarknetError::InternalError`.
8. The valid Declare transaction is rejected at the gateway admission layer, never reaching the mempool or sequencer.
