### Title
Missing Class Existence Check in `execute_replace_class` Allows Permanent Contract Freezing and Network Halt - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS Cairo program accepts an arbitrary `class_hash` from a contract without verifying that the hash corresponds to a declared class. This is the direct analog of the reported Proxy vulnerability: just as a missing contract existence check before `delegatecall` allows silent success on a non-existent target, the missing class existence check here allows a contract to silently replace its class with an undeclared hash, making the contract permanently uncallable and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class` (`syscall_impls.cairo`, lines 877–916), the OS reads the requested `class_hash` directly from the syscall request and writes it into `contract_state_changes` without any validation that the hash is a declared class:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The explicit `TODO` comment at line 898 acknowledges this missing check. The function performs no lookup into `contract_class_changes` to confirm the new `class_hash` was ever declared.

When the affected contract is subsequently called, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [2](#0-1) 

If `class_hash` was never declared, `dict_read` on `contract_class_changes` returns `0` (the uninitialized default, as `UNINITIALIZED_CLASS_HASH = 0`). `find_element` then searches for a compiled class with key `0`, which does not exist, causing the OS to fail and the block to be unprovable. [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds / High — Network halt.**

Two concrete impacts arise:

1. **Permanent freezing of funds:** Any contract that calls `replace_class` with an undeclared class hash becomes permanently uncallable. All tokens or assets held by that contract are irrecoverably frozen, because every future OS execution attempt for that contract will fail at `find_element`.

2. **Network halt:** If the sequencer includes a block containing such a transaction (and a subsequent call to the affected contract in the same or a later block), the OS cannot generate a valid STARK proof for that block. The network cannot confirm new transactions until the block is dropped or the issue is resolved.

---

### Likelihood Explanation

The entry path is fully unprivileged:

- Any deployed contract can issue a `replace_class` syscall with an arbitrary felt value as the class hash.
- The OS enforces no constraint on the value — the TODO comment confirms the check is simply absent.
- If the sequencer-side blockifier also lacks this validation (consistent with the OS TODO), the transaction is included in a block and the OS fails to prove it.
- Even if the blockifier rejects it at the gateway, the OS-level gap means the invariant is not enforced at the proof layer, which is the security boundary that validators rely on.

---

### Recommendation

In `execute_replace_class`, before writing the new `class_hash` into `contract_state_changes`, assert that the class hash exists as a key in `contract_class_changes` with a non-zero compiled class hash:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the check already implicitly required by `execute_entry_point` and closes the gap acknowledged by the existing TODO comment.

---

### Proof of Concept

1. Attacker deploys contract `C` holding user funds.
2. Attacker calls a function on `C` that internally invokes the `replace_class` syscall with `class_hash = 0xdeadbeef` (an undeclared felt).
3. `execute_replace_class` in the OS writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes` with no validation.
4. The state commitment now records `C` as having class hash `0xdeadbeef`.
5. Any subsequent call to `C` (in the same or a future block) causes `execute_entry_point` to call `dict_read(contract_class_changes, 0xdeadbeef)` → returns `0` → `find_element(..., key=0)` → element not found → OS execution fails → block is unprovable.
6. Funds in `C` are permanently frozen; if the call to `C` is in the same block, the entire block cannot be proven, halting the network. [4](#0-3) [5](#0-4)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,

    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,

    );
