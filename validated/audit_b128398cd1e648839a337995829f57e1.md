### Title
Unbounded `Predecessors` Execution in gRPC `TraceTx` Enables Node DoS — (File: `x/evm/keeper/grpc_query.go`)

---

### Summary

The `TraceTx` gRPC handler accepts a caller-supplied `Predecessors` array of arbitrary length and re-executes every entry through `ApplyMessageWithConfig` under an infinite gas meter, with no length cap or budget check. Any unprivileged caller can directly invoke this public gRPC endpoint with thousands of complex EVM transactions as predecessors, forcing the node into unbounded EVM computation.

---

### Finding Description

`Keeper.TraceTx` in `x/evm/keeper/grpc_query.go` processes a `QueryTraceTxRequest` whose `Predecessors` field is a `repeated MsgEthereumTx`. The handler iterates over every entry and calls `ApplyMessageWithConfig` for each one:

```go
for i, tx := range req.Predecessors {
    ethTx := tx.AsTransaction()
    msg, err := core.TransactionToMessage(ethTx, signer, cfg.BaseFee)
    ...
    rsp, err := k.ApplyMessageWithConfig(ctx, msg, cfg, true)
    ...
    cfg.TxConfig.LogIndex += uint(len(rsp.Logs))
}
``` [1](#0-0) 

There is **no check on `len(req.Predecessors)`** before this loop. The query context is set up with an infinite gas meter (standard for Cosmos SDK query handlers), so there is no gas-based ceiling on the total computation across all predecessors. Each predecessor runs the full EVM state transition — including contract creation, storage writes, and arbitrary bytecode execution.

The `rpc/backend/tracing.go` `TraceTransaction` function populates predecessors from the block and forwards them via gRPC:

```go
var predecessors []*evmtypes.MsgEthereumTx
for _, txBz := range blk.Block.Txs[:transaction.TxIndex] {
    ...
    predecessors = append(predecessors, ethMsg)
}
...
traceResult, err := b.queryClient.TraceTx(rpctypes.ContextWithHeight(contextHeight), &traceTxRequest)
``` [2](#0-1) 

However, the gRPC `TraceTx` endpoint (Cosmos SDK port 9090) is **directly reachable** by any caller without authentication. An attacker bypasses the RPC backend entirely and crafts a raw gRPC `QueryTraceTxRequest` with an arbitrarily large `Predecessors` slice, each entry being a maximally expensive EVM transaction (e.g., a contract deployment with large initcode, or a transaction performing thousands of `SSTORE` operations). The handler will execute all of them sequentially with no bound.

The `QueryTraceTxRequest` proto definition confirms `Predecessors` is an unbounded repeated field:

```proto
message QueryTraceTxRequest {
  repeated MsgEthereumTx predecessors = 2;
  ...
}
``` [3](#0-2) 

---

### Impact Explanation

The gRPC server and the CometBFT consensus engine run in the **same OS process**. Saturating the gRPC goroutine pool with CPU-bound EVM re-execution starves the consensus engine of CPU and memory. A validator node subjected to this attack can miss `PrepareProposal`/`ProcessProposal` deadlines, causing it to be slashed or to stall block production. A coordinated attack against multiple validators can halt the chain.

This matches the allowed High impact: *"Public JSON-RPC, gRPC, simulation, tracing, receipt/log, or indexer path … exposes a reachable route to the impacts above"* (chain halt).

---

### Likelihood Explanation

- The gRPC endpoint is public by default (port 9090, no authentication).
- No special privilege, key, or on-chain state is required — only the ability to send a gRPC request.
- The attack payload is a single well-formed `QueryTraceTxRequest` with a large `Predecessors` slice; no prior transactions need to be mined.
- The analog pattern (unbounded iteration over attacker-supplied list) is well-known and straightforward to exploit.

---

### Recommendation

1. **Cap `Predecessors` length** before the loop in `TraceTx`. A safe upper bound is `blockGasLimit / params.TxGas` (the maximum number of minimum-cost transactions that can fit in one block):

```go
maxPredecessors := cfg.BlockGasLimit / params.TxGas
if uint64(len(req.Predecessors)) > maxPredecessors {
    return nil, status.Errorf(codes.InvalidArgument,
        "too many predecessors: %d > %d", len(req.Predecessors), maxPredecessors)
}
```

2. **Apply a per-request gas budget** across all predecessor executions, analogous to the `RPCGasCap` used in `EthCall`.

3. **Rate-limit or authenticate** the gRPC `TraceTx` and `TraceBlock` endpoints at the server layer.

---

### Proof of Concept

```python
import grpc
from ethermint.evm.v1 import query_pb2, query_pb2_grpc, tx_pb2

channel = grpc.insecure_channel("validator-node:9090")
stub = query_pb2_grpc.QueryStub(channel)

# Craft N maximally expensive predecessor transactions
# (e.g., contract deployments with large initcode)
predecessors = [make_expensive_eth_tx() for _ in range(5000)]

req = query_pb2.QueryTraceTxRequest(
    msg=make_minimal_eth_tx(),       # valid target tx
    predecessors=predecessors,        # 5000 unbounded re-executions
    block_number=1,
    chain_id=9000,
)

# Node now executes 5000 full EVM state transitions with infinite gas meter
resp = stub.TraceTx(req)
```

The node will spend unbounded CPU time executing all 5000 predecessors sequentially via `ApplyMessageWithConfig`, with no gas ceiling, matching the exact DoS pattern from the `StakePet` report: an attacker-controlled list of unbounded length is iterated without a minimum-cost barrier. [4](#0-3)

func (k Keeper) TraceTx(c context.Context, req *types.QueryTraceTxRequest) (*types.QueryTraceTxResponse, error) {
	var baseFee *big.Int
	if req == nil || req.Msg == nil {
		return nil, status.Error(codes.InvalidArgument, "request and message cannot be empty")
	}
	if req.BaseFee != nil && !req.BaseFee.IsNil() {
		baseFee = req.BaseFee.BigInt()
	}
	resultData, err := execTrace(
		c,
		req,
		k,
		baseFee,
		func(ctx sdk.Context, cfg *EVMConfig, traceConfig *types.TraceConfig) (*core.Message, error) {
			signer := ethtypes.MakeSigner(cfg.ChainConfig, big.NewInt(ctx.BlockHeight()), uint64(ctx.BlockTime().Unix())) //#nosec G115
			tracer, err := newTacer(&logger.Config{}, cfg.ChainConfig, cfg.TxConfig, traceConfig)
			if err != nil {
				return nil, status.Error(codes.Internal, err.Error())
			}
			cfg.Tracer = tracer.Hooks
			cfg.DebugTrace = true
			cfg.TraceReplay = traceConfig.GetTraceReplay()
			for i, tx := range req.Predecessors {
				ethTx := tx.AsTransaction()
				msg, err := core.TransactionToMessage(ethTx, signer, cfg.BaseFee)
				if err != nil {
					k.Logger(ctx).Debug("trace: skipping predecessor, failed to convert tx to message",
						"index", i, "hash", ethTx.Hash().Hex(), "err", err.Error())
					continue
				}
				cfg.TxConfig.TxHash = ethTx.Hash()
				cfg.TxConfig.TxIndex, err = ethermint.SafeUint(i)
				if err != nil {
					k.Logger(ctx).Debug("trace: skipping predecessor, invalid tx index",
						"index", i, "hash", ethTx.Hash().Hex(), "err", err.Error())
					continue
				}
				rsp, err := k.ApplyMessageWithConfig(ctx, msg, cfg, true)
				if err != nil {
					k.Logger(ctx).Error("trace: predecessor replay failed, trace state may be incomplete",
						"index", i, "hash", ethTx.Hash().Hex(), "err", err.Error())
					continue
				}
				cfg.TxConfig.LogIndex += uint(len(rsp.Logs))
			}

			tx := req.Msg.AsTransaction()
			cfg.TxConfig.TxHash = tx.Hash()
			if len(req.Predecessors) > 0 {
				cfg.TxConfig.TxIndex++
			}

			return core.TransactionToMessage(tx, signer, cfg.BaseFee)
		},
	)
	if err != nil {
		return nil, err
	}

	return &types.QueryTraceTxResponse{
		Data: resultData,
	}, nil
}
