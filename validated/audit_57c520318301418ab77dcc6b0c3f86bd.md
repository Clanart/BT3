### Title
Missing Payload Size Validation in `execute_send_message_to_l1` Enables Unbounded `memcpy` in OS Execution — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_send_message_to_l1` function in `syscall_impls.cairo` computes `payload_size = request.payload_end - payload_start` using raw felt subtraction with **no upper-bound or sign check**. A malicious contract can set `payload_end < payload_start`, causing felt-arithmetic wraparound to produce a `payload_size` close to the STARK prime P ≈ 2²⁵¹. The OS then calls `memcpy` with this astronomically large length, making proof generation infeasible and halting the network. The fixed gas cost of `SEND_MESSAGE_TO_L1_GAS_COST = 14470` — with no per-element payload charge — means this costs the attacker almost nothing.

---

### Finding Description

In `execute_send_message_to_l1` (`syscall_impls.cairo` lines 1346–1374):

```cairo
tempvar payload_start = request.payload_start;
tempvar payload_size = request.payload_end - payload_start;

assert [outputs.messages_to_l1] = MessageToL1Header(
    from_address=contract_address, to_address=request.to_address, payload_size=payload_size
);
memcpy(
    dst=outputs.messages_to_l1 + MessageToL1Header.SIZE, src=payload_start, len=payload_size
);
``` [1](#0-0) 

`payload_size` is a raw felt subtraction. There is **no** `assert_nn`, `assert_nn_le`, or `assert_le` guard on this value before it is passed to `memcpy`. In Cairo felt arithmetic (mod P ≈ 2²⁵¹), if `payload_end < payload_start`, the result is `P − (payload_start − payload_end)` — a value near 2²⁵¹.

The gas cost charged is a **flat** `SEND_MESSAGE_TO_L1_GAS_COST = 14470`:

```cairo
let success = reduce_syscall_gas_and_write_response_header(
    total_gas_cost=SEND_MESSAGE_TO_L1_GAS_COST, request_struct_size=SendMessageToL1Request.SIZE
);
``` [2](#0-1) [3](#0-2) 

**Contrast with other syscalls** that correctly charge per-element gas to bound their inputs:

- `execute_deploy`: `DEPLOY_GAS_COST + DEPLOY_CALLDATA_FACTOR_GAS_COST * constructor_calldata_size`
- `execute_meta_tx_v0`: `META_TX_V0_GAS_COST + META_TX_V0_CALLDATA_FACTOR_GAS_COST * calldata_size` [4](#0-3) [5](#0-4) 

`execute_send_message_to_l1` has no such per-element charge and no size guard.
