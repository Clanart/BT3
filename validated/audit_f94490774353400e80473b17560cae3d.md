### Title
`upsert` Missing Hash-Collision Guard Corrupts `leaf_hash_to_index` and Merkle Tree State — (`crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::upsert` does not check whether the supplied `new_hash` already belongs to a different leaf before overwriting the in-memory `leaf_hash_to_index` cache. The `insert` path has an explicit guard; the update path inside `upsert` does not. Supplying a `new_hash` equal to an existing leaf's hash silently overwrites the cache entry for that other leaf, corrupts `get_node_by_hash` lookups, and leaves the blob with two leaves sharing the same hash, producing an incorrect Merkle root.

    let Ok((leaf_index, mut leaf, mut block)) = self.get_leaf_by_key(key) else {

``` [2](#0-1) 

**Step-by-step cache corruption:**

1. `remove_leaf(&leaf_A)` removes `hash_A` from `leaf_hash_to_index` and adds `idx_A` to `free_indexes`. [3](#0-2) 

2. `insert_entry_to_blob(idx_A, block_with_hash_B)` writes the updated block to the blob, then calls `add_leaf(idx_A, leaf_with_hash_B)`. [4](#0-3) 

3. `add_leaf` calls `leaf_hash_to_index.insert(hash_B, idx_A)`. Because `HashMap::insert` silently overwrites, the existing entry `hash_B → idx_B` (key_B's index) is replaced with `hash_B → idx_A` (key_A's index). [5](#0-4) 

**Post-corruption state:**

| Structure | Before | After |
|---|---|---|
| `leaf_hash_to_index` | `{hash_A: idx_A, hash_B: idx_B}` | `{hash_B: idx_A}` |
| blob at `idx_A` | key_A / hash_A | key_A / hash_B |
| blob at `idx_B` | key_B / hash_B | key_B / hash_B (unchanged) |

Two leaves in the blob now carry the same hash (`hash_B`). The parent of `idx_A` is marked dirty and will be recalculated using `hash_B` for key_A's slot, producing a wrong Merkle root. Key_B's subtree is not re-hashed, so the root is permanently inconsistent.

`get_node_by_hash` resolves `hash_B` exclusively through `leaf_hash_to_index`:

After the attack, `get_node_by_hash(hash_B)` returns `(key_A, new_value)` instead of `(key_B, value_B)`. Key_B's leaf is permanently unreachable via hash lookup.

### Impact Explanation

- **Corrupted Merkle root**: two leaves share `hash_B`; the root hash computed after `calculate_lazy_hashes` is wrong. Any proof of inclusion/exclusion derived from this root is invalid.
- **`get_node_by_hash` returns wrong key/value**: DataLayer consumers that resolve data by leaf hash receive key_A's data when querying hash_B.
- **Key_B permanently unreachable by hash**: `get_node_by_hash(hash_B)` never returns key_B's data again for the lifetime of this `MerkleBlob` instance.

This matches the allowed High impact: *DataLayer Merkle proof/blob/delta logic corrupts tree roots or lets untrusted input prove invalid state.*

### Likelihood Explanation

`upsert` is a public method exposed through Python bindings (`MerkleBlob.upsert(key, value, new_hash: bytes32)`). [9](#0-8) 

The `new_hash` parameter is caller-supplied. Any DataLayer component or binding consumer that passes a hash equal to an already-present leaf's hash (whether by mistake or by a malicious actor who can influence DataLayer update operations) triggers the corruption. No cryptographic break is required — the attacker only needs to know an existing leaf hash (observable from the public tree) and supply it as `new_hash` for a different key's update.

### Recommendation

Add the same guard that `insert` uses, at the top of the update branch in `upsert`, before `remove_leaf` is called:

```rust
pub fn upsert(&mut self, key: KeyId, value: ValueId, new_hash: &Hash) -> Result<(), Error> {
    let Ok((leaf_index, mut leaf, mut block)) = self.get_leaf_by_key(key) else {
        self.insert(key, value, new_hash, InsertLocation::Auto {})?;
        return Ok(());
    };

    // Guard: reject if new_hash already belongs to a *different* leaf
    if leaf.hash != *new_hash && self.block_status_cache.contains_leaf_hash(new_hash) {
        return Err(Error::HashAlreadyPresent());
    }

    self.block_status_cache.remove_leaf(&leaf)?;
    ...
}
```

The `leaf.hash != *new_hash` condition allows a no-op hash update (same hash, new value) without a false positive.
