### Title
Legacy EIP-712 `timeout_height` Excluded from Typed Data Hash Enables Replay of Expired Transactions - (File: `ethereum/eip712/eip712_legacy.go`)

### Summary
The `timeout_height` field is intentionally omitted from the EIP-712 `Tx` type definition in the legacy EIP-712 path. Because EIP-712 only hashes fields declared in the type schema, `timeout_height` is never committed to by the user's signature. An unprivileged attacker who observes a signed legacy EIP-712 transaction in the mempool can strip or zero out `timeout_height` and resubmit the transaction after the user's intended expiry block, bypassing the user's explicit time-bound authorization.

### Finding Description

**Root cause — excluded field in type schema**

In `extractMsgTypes` (`ethereum/eip712/eip712_legacy.go`), the `Tx` type definition hard-codes the fields that enter the EIP-712 hash. `timeout_height` is explicitly commented out:

```go
"Tx": {
    {Name: "account_number", Type: "string"},
    {Name: "chain_id", Type: "string"},
    {Name: "fee", Type: "Fee"},
    {Name: "memo", Type: "string"},
    {Name: "msgs", Type: "Msg[]"},
    {Name: "sequence", Type: "string"},
    // Note timeout_height was removed because it was not getting filled with the legacyTx
    // {Name: "timeout_height", Type: "string"},
},
``` [1](#0-0) 

Per EIP-712, any key present in the message map but absent from the type definition is silently ignored during hashing. Therefore the EIP-712 `sigHash` is identical regardless of what value `timeout_height` carries.

**How the ante handler exposes this**

`VerifySignature` in `ante/cosmos/eip712.go` reconstructs `txBytes` from the *live* transaction, including `tx.GetTimeoutHeight()`:

```go
txBytes := legacytx.StdSignBytes(
    signerData.ChainID,
    signerData.AccountNumber,
    signerData.Sequence,
    tx.GetTimeoutHeight(),   // ← taken from the submitted tx, not the signed doc
    legacytx.StdFee{Amount: tx.GetFee(), Gas: tx.GetGas()},
    msgs, tx.GetMemo(),
)
...
typedData, err := eip712.LegacyWrapTxToTypedData(
    ethermintCodec, extOpt.TypedDataChainID, msgs[0], txBytes, feeDelegation)
``` [2](#0-1) 

`LegacyWrapTxToTypedData` unmarshals `txBytes` into a raw `map[string]interface{}` and sets it as the EIP-712 `Message`. Because `timeout_height` is absent from the `Tx` type schema, `apitypes.TypedDataAndHash` ignores it when computing `sigHash`. The resulting hash is therefore identical for `timeout_height = 0` and `timeout_height = X`. [3](#0-2) 

**Contrast with the non-legacy path**

The non-legacy Protobuf decoder (`decodeProtobufSignDoc`) and the legacy Protobuf decoder (`legacyDecodeProtobufSignDoc`) both explicitly reject any sign doc that carries a non-zero `timeout_height`:

```go
if body.TimeoutHeight != 0 || body.GetTimeoutTimestamp() != nil || ... {
    return apitypes.TypedData{}, errors.New(
        "body contains unsupported fields: TimeoutHeight, ...")
}
``` [4](#0-3) [5](#0-4) 

The legacy Amino path (`legacyDecodeAminoSignDoc`) and the `VerifySignature` ante handler have **no equivalent guard**, leaving the field silently unprotected.

**Entry point**

The vulnerable decorator is wired into the live ante chain for every transaction carrying `ExtensionOptionsWeb3Tx`:

```go
case "/ethermint.types.v1.ExtensionOptionsWeb3Tx":
    anteHandler = newLegacyCosmosAnteHandlerEip712(ctx, options, ...)
``` [6](#0-5) 

```go
cosmos.NewLegacyEip712SigVerificationDecorator(options.AccountKeeper, options.SignModeHandler),
``` [7](#0-6) 

### Impact Explanation

A user who sets `timeout_height = X` on a legacy EIP-712 Cosmos transaction intends the transaction to be invalid after block X. Because `timeout_height` is not covered by the signature, an attacker can:

1. Observe the signed transaction in the mempool.
2. Mutate `timeout_height` from `X` to `0` (no expiry).
3. Resubmit after block X.

`authante.NewTxTimeoutHeightDecorator` passes `timeout_height = 0` without rejection. `LegacyEip712SigVerificationDecorator` computes the same `sigHash` as the original and accepts the signature. The transaction executes with the original message content (fund transfer, delegation, governance vote, etc.) at a time the signer explicitly intended to prohibit.

This matches the allowed High impact: **EIP-712 authorization bypass enabling forged/unauthorized execution** — the signer's authorization is used to execute an action outside the time window the signer consented to.

### Likelihood Explanation

- The legacy EIP-712 path (`ExtensionOptionsWeb3Tx`) is still active and reachable by any unprivileged user.
- Mempool transactions are publicly observable on most Cosmos nodes.
- The mutation is trivial: zero out one protobuf field before rebroadcast.
- The attack is only useful when the original transaction was not included before its timeout (e.g., low-fee transactions, intentionally short-lived authorizations, or transactions the user cancelled by letting them expire).

### Recommendation

Apply the same guard that the Protobuf decoders already use. In `VerifySignature` (`ante/cosmos/eip712.go`), reject the transaction if `tx.GetTimeoutHeight() != 0` before constructing `txBytes`:

```go
if tx.GetTimeoutHeight() != 0 {
    return errorsmod.Wrap(errortypes.ErrNotSupported,
        "legacy EIP-712 does not support timeout_height; use the standard EIP-712 path")
}
```

Alternatively, add `timeout_height` back to the `Tx` type definition in `extractMsgTypes` so the field is committed to by the signature. The same fix should be applied to `legacyDecodeAminoSignDoc` in `encoding_legacy.go`, which also lacks the guard.

### Proof of Concept

```
1. Alice creates a MsgSend with timeout_height = 1000 (current block ~990).
   She signs it via the legacy EIP-712 path (ExtensionOptionsWeb3Tx).
   The EIP-712 hash H does NOT include timeout_height.

2. Alice broadcasts the transaction. It sits in the mempool.

3. Block 1001 arrives. TxTimeoutHeightDecorator drops Alice's original tx.

4. Bob (attacker) copies Alice's signed transaction, sets timeout_height = 0,
   and broadcasts the mutated transaction.

5. Ante handler:
   a. TxTimeoutHeightDecorator: timeout_height == 0 → no rejection.
   b. LegacyEip712SigVerificationDecorator:
      - Builds txBytes with timeout_height = 0.
      - Calls LegacyWrapTxToTypedData → txData contains timeout_height:0
        but Tx type schema omits it → hash H' == H.
      - ethcrypto.VerifySignature(pubKey, H', sig) → true.
   c. Transaction executes: Alice's funds are transferred at block 1001+,
      after the window Alice intended to restrict.
```
