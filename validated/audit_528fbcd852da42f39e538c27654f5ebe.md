### Title
Stale `feemarketParams.BaseFee` Captured at Ante-Handler Construction Used for All Subsequent Cosmos-SDK Tx Fee Validation - (File: `evmd/ante/handler_options.go`, `ante/evm/fee_checker.go`)

### Summary

`newCosmosAnteHandler` snapshots `feemarketParams` (including `BaseFee`) once at construction time and passes a pointer to the captured value into `NewDynamicFeeChecker`. Because the EIP-1559 base fee is updated every block in `BeginBlock`, the fee checker permanently uses a stale base fee for all Cosmos-SDK native transactions, allowing them to be accepted with fees below the actual current base fee.

### Finding Description

In `evmd/ante/handler_options.go`, `newCosmosAnteHandler` reads `feemarketParams` from the keeper once at construction:

```go
func newCosmosAnteHandler(ctx sdk.Context, options HandlerOptions, ...) sdk.AnteHandler {
    evmParams := options.EvmKeeper.GetParams(ctx)
    feemarketParams := options.FeeMarketKeeper.GetParams(ctx)   // ← snapshot at construction
    ...
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams) // ← pointer to snapshot
    ...
    cosmos.NewMinGasPriceDecorator(options.FeeMarketKeeper, evmDenom, &feemarketParams) // ← same snapshot
``` [1](#0-0) 

`NewDynamicFeeChecker` closes over the pointer and, on every invocation, calls:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
``` [2](#0-1) 

`GetBaseFee` reads `feemarketParams.BaseFee` directly from the captured struct:

```go
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    ...
    baseFee := feemarketParams.GetBaseFee()   // reads captured BaseFee, never refreshed
``` [3](#0-2) 

Meanwhile, every block `BeginBlock` recalculates and persists a new base fee to the KVStore:

```go
func (k *Keeper) BeginBlock(ctx sdk.Context) error {
    baseFee := k.CalculateBaseFee(ctx)
    ...
    k.SetBaseFee(ctx, baseFee)
``` [4](#0-3) 

`SetBaseFee` writes to the KVStore but never touches the pointer held by the already-constructed `NewDynamicFeeChecker` closure. The captured `feemarketParams.BaseFee` is therefore permanently frozen at the value present when `newCosmosAnteHandler` was called (typically app startup / genesis).

By contrast, the Ethereum-transaction path (`newEthAnteHandler`) correctly re-reads `blockCfg` (including `BaseFee`) on every invocation:

```go
blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
...
baseFee := blockCfg.BaseFee
``` [5](#0-4) 

The asymmetry is the root cause: EVM transactions always see the current base fee; Cosmos-SDK transactions always see the genesis-time base fee.

### Impact Explanation

When the live base fee rises above the genesis-time snapshot (the normal case as the chain processes load), any Cosmos-SDK transaction whose fee satisfies the stale (lower) base fee will pass `NewDynamicFeeChecker` and be committed, even though it underpays relative to the current EIP-1559 floor. This is a fee-market ante-handler bug that permits invalid transactions to commit and causes fees to be mis-accounted — matching the **High** allowed impact: *"fee market, ante handler … bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

### Likelihood Explanation

- `DynamicFeeChecker` must be enabled (`options.DynamicFeeChecker = true`), which is the intended production configuration for chains that want EIP-1559 fee semantics on Cosmos-SDK transactions.
- The base fee changes every block; after even a single block of above-target gas usage the snapshot diverges.
- Any unprivileged user can craft a Cosmos-SDK transaction with a fee calibrated to the (lower) genesis base fee and submit it via the standard mempool/RPC path.
- No special privileges, governance access, or validator collusion are required.

### Recommendation

Move the `feemarketParams` read inside the returned closure so it is refreshed on every transaction, mirroring the pattern used in `newEthAnteHandler`:

```go
func NewDynamicFeeChecker(ethCfg *params.ChainConfig, evmParams *types.Params, fmKeeper FeeMarketKeeper) authante.TxFeeChecker {
    return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
        feemarketParams := fmKeeper.GetParams(ctx)   // fresh every call
        baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, &feemarketParams)
        ...
    }
}
```

Alternatively, pass the `FeeMarketKeeper` directly and call `GetBaseFee(ctx)` inside the closure, exactly as `newEthAnteHandler` does via `EVMBlockConfig`.

### Proof of Concept

1. Chain starts; genesis `BaseFee = 1_000_000_000` (1 Gwei). `newCosmosAnteHandler` is called; `feemarketParams.BaseFee` is snapshotted at 1 Gwei.
2. After several blocks of high gas usage, `BeginBlock` raises the live base fee to, say, 2_000_000_000 (2 Gwei) and writes it to the KVStore.
3. Attacker submits a Cosmos-SDK transaction (e.g., `MsgSend`) with `ExtensionOptionDynamicFeeTx` and a fee of 1.1 Gwei/gas — above the stale snapshot but below the live base fee.
4. `NewDynamicFeeChecker` evaluates `feeCap (1.1 Gwei) >= baseFee (1 Gwei, stale)` → passes.
5. The transaction is included in a block and committed, despite paying less than the current EIP-1559 floor of 2 Gwei.

func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
	if !IsLondon(ethCfg, height) {
		return nil
	}
	baseFee := feemarketParams.GetBaseFee()
	// should not be nil if london hardfork enabled
	if baseFee == nil {
		return new(big.Int)
	}
	return baseFee
}
