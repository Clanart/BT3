### Title
Missing Zero-Value Validation in `execute_replace_class` Allows Permanent Contract Bricking - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` function in the StarkNet OS accepts an attacker-supplied `class_hash` of `0` (i.e., `UNINITIALIZED_CLASS_HASH`) without any validation. This is the direct Cairo analog of the reported Solidity zero-address assignment bug. A contract can call the `replace_class` syscall with `class_hash = 0`, permanently setting its own class hash to the uninitialized sentinel value. Any subsequent call to that contract causes the OS to panic inside `find_element` when it cannot locate a compiled class for hash `0`, permanently bricking the contract and freezing all funds it holds.

### Finding Description

`execute_replace_class` in `syscall_impls.cairo` reads `class_hash` directly from the caller-controlled syscall request and writes it into the contract's `StateEntry` with no validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The developer-inserted TODO comment at line 898 explicitly acknowledges the missing check. The identical gap exists in the deprecated path: [2](#0-1) 

When a subsequent call is made to the bricked contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // = 0
);
// compiled_class_hash is now 0 (no declared class for key 0)
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,           // = 0, not present → Cairo panic
);
``` [3](#0-2) 

`find_element` is a Cairo built-in that panics (hard assertion failure) when the key is absent. Since `UNINITIALIZED_CLASS_HASH = 0` is never a valid entry in `compiled_class_facts`, the OS proof for any block containing a call to the bricked contract cannot be generated. [4](#0-3) 

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` is set to `0` via `replace_class(0)`:
- The state change is committed to the Merkle trie and is irreversible.
- Every future call to the contract causes the OS to panic, so no transaction touching that contract can ever be proven.
- All ERC-20 balances, ETH, or other assets held by the contract are permanently inaccessible.

### Likelihood Explanation

Any deployed Sierra contract that exposes a code path invoking `replace_class` with an attacker-controlled argument (or hardcoded `0`) can trigger this. The syscall is available to all non-privileged contract callers. The missing check is explicitly flagged by a developer TODO, confirming it is a known gap rather than an intentional design choice.

### Recommendation

Add an `assert_not_zero` guard on `class_hash` immediately after reading it from the request, in both `execute_replace_class` implementations:

```cairo
// In syscall_impls.cairo
let class_hash = request.class_hash;
assert_not_zero(class_hash);  // Reject UNINITIALIZED_CLASS_HASH (0)
```

```cairo
// In deprecated_execute_syscalls.cairo
let class_hash = syscall_ptr.class_hash;
assert_not_zero(class_hash);  // Reject UNINITIALIZED_CLASS_HASH (0)
```

Additionally, resolve the TODO by verifying that `class_hash` maps to a declared entry in `contract_class_changes` before committing the state update.

### Proof of Concept

1. Deploy a Sierra contract `Bricker` containing:
   ```
   fn brick_self(ref self: ContractState) {
       starknet::replace_class_syscall(0_felt252.try_into().unwrap()).unwrap();
   }
   ```
2. Call `brick_self()`. The OS executes `execute_replace_class` with `class_hash = 0`; no assertion fires. The contract's `StateEntry.class_hash` is written as `0`.
3. In any subsequent block, send any call to `Bricker`. The OS enters `execute_entry_point`, reads `compiled_class_hash = dict_read(key=0) = 0`, then calls `find_element(..., key=0)`. Because `0` is not present in `compiled_class_facts`, Cairo panics. The block is unprovable and all funds in `Bricker` are permanently frozen. [5](#0-4) [6](#0-5)

    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
