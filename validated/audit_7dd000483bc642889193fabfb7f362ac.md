### Title
Stale Cached `feemarketParams.BaseFee` in `NewDynamicFeeChecker` Allows Cosmos Tx Fee Underpayment — (`evmd/ante/handler_options.go`)

### Summary

`newCosmosAnteHandler` reads `feemarketParams` once at construction time and passes a pointer to that local value copy into `NewDynamicFeeChecker`. Because the EIP-1559 base fee is updated every block in `BeginBlock`, but the captured params struct is never refreshed, Cosmos-SDK transactions can be submitted with fees anchored to the stale genesis-era base fee, permanently bypassing the live fee-market floor for the lifetime of the node process.

### Finding Description

In `evmd/ante/handler_options.go`, `newCosmosAnteHandler` is called once at app startup with an initial `sdk.Context`. It reads a **value copy** of the fee-market params from the store and immediately passes a pointer to that local copy into the fee-checker and the min-gas-price decorator:

```go
// evmd/ante/handler_options.go  lines 179-187
func newCosmosAnteHandler(ctx sdk.Context, options HandlerOptions, ...) sdk.AnteHandler {
    evmParams        := options.EvmKeeper.GetParams(ctx)
    feemarketParams  := options.FeeMarketKeeper.GetParams(ctx)   // value copy, never updated
    ...
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
    ...
    cosmos.NewMinGasPriceDecorator(options.FeeMarketKeeper, evmDenom, &feemarketParams)
``` [1](#0-0) 

`NewDynamicFeeChecker` closes over the pointer and, on every invocation, calls:

```go
// ante/evm/fee_checker.go  line 56
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
``` [2](#0-1) 

`types.GetBaseFee` simply reads `feemarketParams.GetBaseFee()` — the value frozen at construction time:

```go
// x/evm/types/utils.go  lines 244-254
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    if !IsLondon(ethCfg, height) { return nil }
    baseFee := feemarketParams.GetBaseFee()
    ...
    return baseFee
}
``` [3](#0-2) 

Meanwhile, every block `x/feemarket/keeper/abci.go` `BeginBlock` recalculates and **writes** a new base fee to the persistent store:

```go
// x/feemarket/keeper/abci.go  lines 30-38
func (k *Keeper) BeginBlock(ctx sdk.Context) error {
    baseFee := k.CalculateBaseFee(ctx)
    k.SetBaseFee(ctx, baseFee)
    ...
}
``` [4](#0-3) 

The `SetBaseFee` path writes into the params KV store:

```go
// x/feemarket/keeper/params.go  lines 72-78
func (k Keeper) SetBaseFee(ctx sdk.Context, baseFee *big.Int) {
    params := k.GetParams(ctx)
    params.BaseFee = ethermint.SaturatedNewInt(baseFee)
    k.SetParams(ctx, params)
}
``` [5](#0-4) 

The in-memory `feemarketParams` struct captured by the closure is **never touched again**. Its `BaseFee` field remains the genesis value for the entire lifetime of the node process.

The same stale pointer is also passed to `MinGasPriceDecorator`, which reads `mpd.feemarketParams.MinGasPrice` from the frozen copy rather than from the live keeper, meaning a governance-raised `MinGasPrice` is also silently ignored for Cosmos txs:

```go
// ante/cosmos/min_gas_price.go  line 54
minGasPrice := mpd.feemarketParams.MinGasPrice
``` [6](#0-5) 

By contrast, the **Ethereum** ante handler (`newEthAnteHandler`) correctly re-reads live params on every call via `EVMBlockConfig`:

```go
// evmd/ante/handler_options.go  lines 88-95
blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
feemarketParams := &blockCfg.FeeMarketParams
baseFee := blockCfg.BaseFee
``` [7](#0-6) 

This asymmetry means Cosmos-SDK transactions are validated against a permanently stale base fee while Ethereum transactions are validated against the live one.

### Impact Explanation

Any user submitting a Cosmos-SDK transaction (bank send, governance vote, IBC relay, authz exec, etc.) when `DynamicFeeChecker` is enabled can set `feeCap` anywhere above the frozen genesis base fee (e.g. 1 Gwei) even if the live base fee has risen to 10× or 100× that value. The `NewDeductFeeDecorator` then deducts only `effectivePrice × gas` computed from the stale base fee, so the user pays far less than the protocol requires. This constitutes:

- **Fee mis-accounting / underpayment** for every Cosmos tx after the first block — matching the "valid user funds/fees to be mis-accounted" High impact criterion.
- **Fee-market bypass**: the EIP-1559 congestion-pricing mechanism is entirely non-functional for Cosmos txs, allowing sustained spam at genesis-era prices regardless of chain load.

### Likelihood Explanation

The condition is triggered automatically from block 2 onward on any chain where `DynamicFeeChecker: true` and the base fee is not `NoBaseFee`. No special privileges, keys, or coordination are required — any wallet submitting a standard Cosmos tx benefits from the stale floor. The base fee diverges from genesis immediately under any non-zero block utilization.

### Recommendation

Remove the one-time param snapshot from `newCosmosAnteHandler`. Instead, have `NewDynamicFeeChecker` accept the `FeeMarketKeeper` interface and call `keeper.GetBaseFee(ctx)` (or `keeper.GetParams(ctx).BaseFee`) on every invocation, mirroring the pattern already used correctly in `newEthAnteHandler`. The same fix applies to `NewMinGasPriceDecorator`, which should read `MinGasPrice` from the keeper rather than from the frozen pointer.

### Proof of Concept

```
Block 1  – genesis base fee = 1 Gwei  (feemarketParams frozen here)
Blocks 2–N – high utilization; CalculateBaseFee raises live base fee to 20 Gwei
             (stored in KV store, but feemarketParams.BaseFee still = 1 Gwei)

Attacker submits Cosmos bank-send tx:
  fee = 2 Gwei × gasLimit

NewDynamicFeeChecker:
  baseFee  = feemarketParams.GetBaseFee() = 1 Gwei   ← stale
  feeCap   = 2 Gwei / gasLimit × gasLimit = 2 Gwei
  check:   feeCap (2) >= baseFee (1)  → PASS

DeductFeeDecorator deducts 2 Gwei × gasLimit.
Correct deduction should be 20 Gwei × gasLimit.
Attacker underpays by 18 Gwei × gasLimit per tx, indefinitely.
```

func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
	if !IsLondon(ethCfg, height) {
		return nil
	}
	baseFee := feemarketParams.GetBaseFee()
	// should not be nil if london hardfork enabled
	if baseFee == nil {
		return new(big.Int)
	}
	return baseFee
}
