### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Contract Bricking - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS does not verify that the caller-supplied class hash corresponds to a declared contract class before writing it into the contract state. An unprivileged contract can call `replace_class` with any arbitrary felt value as the new class hash. Once committed, any subsequent OS execution that attempts to call that contract will fail to resolve the compiled class, permanently bricking the contract and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the caller-supplied `class_hash` from the syscall request and writes it directly into `contract_state_changes` without any check that the hash corresponds to a declared class:

```cairo
// execute_replace_class (syscall_impls.cairo, lines 877–916)
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The explicit `TODO` at line 898 confirms the validation is intentionally deferred but not yet implemented.

When the contract is subsequently called, `execute_entry_point` resolves the compiled class in two steps:

1. `dict_read{dict_ptr=contract_class_changes}(key=execution_context.class_hash)` — returns `0` (the dict default) if the class hash was never declared.
2. `find_element(array_ptr=compiled_class_facts, key=compiled_class_hash)` — called with `key=0`, which is not present in the compiled class facts array, causing an unrecoverable assertion failure in the OS. [2](#0-1) 

The `contract_class_changes` dict is initialized with `dict_new()` (default value `0`), so any undeclared class hash silently resolves to compiled class hash `0`. [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value and the block is proven and committed to L1, the contract state is permanently poisoned. No future block can include a successful call to that contract: any attempt causes the OS to fail at `find_element`, making the block unprovable. All assets held by the contract (tokens, NFTs, protocol reserves) become permanently inaccessible. The committed L1 state root reflects the bricked contract, and there is no recovery path within the protocol.

---

### Likelihood Explanation

Any unprivileged user can deploy a contract whose code calls `replace_class` with an arbitrary felt (e.g., `0xdeadbeef`). No special role, leaked key, or operator access is required. The attack requires only two transactions: one to deploy the contract and one to trigger the `replace_class` call. If the contract is a shared vault, DEX pool, or any protocol contract holding user funds, the impact extends to all depositors. The missing check is explicitly acknowledged in the source with a dated TODO, confirming it is a known gap in the current production code.

---

### Recommendation

Inside `execute_replace_class`, before writing the new class hash to `contract_state_changes`, assert that the class hash exists in `contract_class_changes` (i.e., it has been declared in the current or a prior block). Concretely:

```cairo
// After reading class_hash from request:
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);  // Revert if class is not declared.
```

This mirrors the validation already performed implicitly in `execute_entry_point` and makes the OS reject the syscall with a failure response rather than silently committing an invalid state. [4](#0-3) 

---

### Proof of Concept

**Step 1 — Deploy a malicious contract** whose constructor or any external entry point contains:

```cairo
// Pseudocode for the malicious contract
@external
func brick_self{syscall_ptr: felt*}() {
    replace_class(class_hash=0xdeadbeef);  // 0xdeadbeef is never declared
    return ();
}
```

**Step 2 — Fund the contract.** Users or the attacker deposit tokens into the contract (e.g., it presents itself as a yield vault).

**Step 3 — Call `brick_self`.** The attacker sends a transaction invoking `brick_self`. The OS executes `execute_replace_class`:

- `request.class_hash = 0xdeadbeef`
- No validation is performed (line 898 TODO)
- `contract_state_changes[contract_address].class_hash` is set to `0xdeadbeef`
- Transaction succeeds; block is proven and committed to L1.

**Step 4 — Attempt any future call to the contract.** The OS executes `execute_entry_point`:

```
dict_read(contract_class_changes, key=0xdeadbeef) → 0   // not declared
find_element(compiled_class_facts, key=0)               // FAILS: no class with hash 0
``` [5](#0-4) 

The OS execution aborts. The block is unprovable. All funds in the contract are permanently frozen with no recovery mechanism.
