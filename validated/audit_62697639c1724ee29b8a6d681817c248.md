### Title
`MerkleBlob::batch_insert` Bypasses Duplicate-Key/Hash Validation for All Items When Tree Has ≥2 Leaves — (`File: crates/chia-datalayer/src/merkle/blob.rs`)

### Summary

`MerkleBlob::batch_insert` skips the duplicate-key and duplicate-hash checks that `MerkleBlob::insert` enforces. When the tree already contains two or more leaves, **every** item in the batch bypasses validation entirely. This allows a caller — or a malicious DataLayer peer — to insert duplicate `KeyId`/`Hash` entries, silently corrupting the `block_status_cache`, producing a wrong Merkle root, and enabling forged proofs of inclusion.

### Finding Description

`MerkleBlob::insert` guards against duplicate keys and duplicate leaf hashes before writing any node:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  insert()
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
```

`batch_insert` delegates to `insert` only for the last two items **and only when the tree currently has ≤1 leaf**:

```rust
// batch_insert – lines 578-603
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else { return Ok(()); };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;   // validated
    }
}

for ((key, value), hash) in keys_values_hashes {   // ← NO duplicate check
    let new_leaf_index = self.get_new_index();
    let new_block = Block {
        node: Node::Leaf(LeafNode { parent: Parent(None), hash, key, value }),
        ..
    };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
```

Two distinct bypass scenarios exist:

| Tree state at call time | Items validated | Items bypassing validation |
|---|---|---|
| 0–1 leaves | Last 2 items (via `pop`) | All remaining (first N-2) |
| **≥2 leaves** | **None** | **All N items** |

The second case is the common production case: a DataLayer store that has been synced before will always have ≥2 leaves, so every subsequent `batch_insert` call skips all duplicate checks.

`BlockStatusCache` uses `HashMap::insert` internally, which silently overwrites on collision. After a duplicate-key batch, the blob contains two leaf nodes sharing the same `KeyId`, but the cache records only the last-written index. The tree root is then computed over a structurally invalid tree.

### Impact Explanation

- **Corrupted Merkle root**: The root hash is computed over a tree containing duplicate leaf nodes. Any node that syncs this store will compute the same wrong root, so the root diverges from the canonical value that would result from a correctly validated sequence of single inserts.
- **Forged proof of inclusion**: `get_proof_of_inclusion` looks up the leaf via the cache, which points to only one of the two duplicate leaves. A proof generated for the cached leaf is structurally valid (passes `ProofOfInclusion::valid()`) but proves membership of the wrong `(key, value)` pair, because the other duplicate leaf — with a different `ValueId` — is invisible to the cache.
- **Persistent state corruption**: The corrupted blob is persisted to disk via `to_path` / `zstd_encode`, so the corruption survives restarts.

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

### Likelihood Explanation

The DataLayer sync protocol calls `batch_insert` with data received from peer nodes. A malicious peer (or a store owner who controls the on-chain DataLayer coin) can craft a batch containing a repeated `KeyId` or `Hash`. Because the tree will already have ≥2 leaves after the first sync, the entire batch bypasses validation. No privileged key material beyond owning or impersonating a DataLayer peer is required.

### Recommendation

Move the duplicate-key and duplicate-hash guards into `batch_insert` itself, mirroring the checks already present in `insert`:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // ... existing write logic
}
```

This follows the long-term recommendation from the external report: keep validation as close as possible to where the data is consumed and stored, rather than relying on callers to pre-validate.

### Proof of Concept

```rust
// Demonstrates that batch_insert accepts duplicate keys when tree has ≥2 leaves
let mut blob = MerkleBlob::new(vec![]).unwrap();

// Seed the tree with 2 leaves so leaf_count() > 1
blob.insert(KeyId(1), ValueId(1), &hash_of(1), InsertLocation::Auto {}).unwrap();
blob.insert(KeyId(2), ValueId(2), &hash_of(2), InsertLocation::Auto {}).unwrap();

// Now batch_insert skips ALL validation
let duplicate_batch = vec![
    ((KeyId(1), ValueId(99)), hash_of(99)),  // KeyId(1) already present – not caught
    ((KeyId(3), ValueId(3)),  hash_of(3)),
];
blob.batch_insert(duplicate_batch).unwrap();  // succeeds, no error

// Tree now has two leaf nodes with KeyId(1); cache points to only one.
// Root hash is wrong; proof for KeyId(1) proves the wrong ValueId.
blob.calculate_lazy_hashes().unwrap();
let proof = blob.get_proof_of_inclusion(KeyId(1)).unwrap();
assert!(proof.valid());  // passes, but proves the wrong (key, value) pair
```

        let index_count = blob.len() / BLOCK_SIZE;

        let mut seen_indexes: BitVec<u64, bitvec::order::Lsb0> = BitVec::repeat(false, index_count);
        let mut key_to_index: HashMap<KeyId, TreeIndex> = HashMap::default();
        let mut leaf_hash_to_index: HashMap<Hash, TreeIndex> = HashMap::default();

        for item in LeftChildFirstIterator::new(blob, None) {
            let (index, block) = item?;
            seen_indexes.set(index.0 as usize, true);

            if let Node::Leaf(leaf) = block.node {
                if key_to_index.insert(leaf.key, index).is_some() {
                    return Err(Error::KeyAlreadyPresent());
                }
                if leaf_hash_to_index.insert(leaf.hash, index).is_some() {
                    return Err(Error::HashAlreadyPresent());
                }
            }
        }

        let mut free_indexes: IndexSet<TreeIndex> = IndexSet::new();
        for (index, seen) in seen_indexes.iter().enumerate() {
            if !seen {
                free_indexes.insert(TreeIndex(index as u32));
            }
        }

        Ok(Self {
            free_indexes,
            key_to_index,
            leaf_hash_to_index,
        })
    }
