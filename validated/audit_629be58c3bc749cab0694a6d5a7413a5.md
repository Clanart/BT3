### Title
`CommitmentStateDiff::from(StateMaps)` silently drops `declared_contracts`, causing wrong `state_diff_commitment` and `concatenated_counts` in every block containing a deprecated (Cairo 0) class declaration — (`crates/blockifier/src/state/cached_state.rs`)

---

### Summary

`StateMaps` carries a `declared_contracts: HashMap<ClassHash, bool>` field that records every class declared in a block, including deprecated (Cairo 0) classes. The conversion `CommitmentStateDiff::from(StateMaps)` silently discards this field. Because `CommitmentStateDiff` has no `deprecated_declared_classes` member, the downstream conversion `ThinStateDiff::from(CommitmentStateDiff)` is forced to hard-code `deprecated_declared_classes: Vec::new()`. The `ThinStateDiff` produced this way is then passed to `calculate_block_commitments`, which uses it to compute both the `state_diff_commitment` (Poseidon hash) and the `concatenated_counts` field (which encodes `state_diff_length`). Both values are wrong whenever the block contains a deprecated Declare transaction, producing a corrupted `PartialBlockHashComponents` and, ultimately, a wrong block hash.

---

### Finding Description

**Analog to the external bug.** In the Perennial bug, `reserve.redeem(amount)` returns a pro-rata amount that is less than `amount` when the reserve is undercollateralised, but the code discards the return value and continues to use the original `amount`. The sequencer analog is structurally identical: a conversion function is called, its output is quantitatively smaller than the input (missing entries), and the downstream computation uses the truncated output as if it were complete.

**Root cause — three-step information loss.**

Step 1 — `StateMaps` records deprecated declarations:

```rust
// crates/blockifier/src/state/cached_state.rs
pub struct StateMaps {
    pub nonces: HashMap<ContractAddress, Nonce>,
    pub class_hashes: HashMap<ContractAddress, ClassHash>,
    pub storage: HashMap<StorageEntry, Felt>,
    pub compiled_class_hashes: HashMap<ClassHash, CompiledClassHash>,
    pub declared_contracts: HashMap<ClassHash, bool>,   // ← Cairo-0 hashes live here
}
```

Step 2 — `CommitmentStateDiff::from(StateMaps)` silently drops `declared_contracts`:

```rust
// crates/blockifier/src/state/cached_state.rs  (lines 679-688)
impl From<StateMaps> for CommitmentStateDiff {
    fn from(diff: StateMaps) -> Self {
        Self {
            address_to_class_hash: IndexMap::from_iter(diff.class_hashes),
            storage_updates: StorageDiff::from(StorageView(diff.storage)),
            class_hash_to_compiled_class_hash: IndexMap::from_iter(diff.compiled_class_hashes),
            address_to_nonce: IndexMap::from_iter(diff.nonces),
            // diff.declared_contracts is never read
        }
    }
}
```

Step 3 — `ThinStateDiff::from(CommitmentStateDiff)` hard-codes an empty list:

```rust
// crates/blockifier/src/state/cached_state.rs  (lines 690-701)
impl From<CommitmentStateDiff> for ThinStateDiff {
    fn from(commitment_state_diff: CommitmentStateDiff) -> Self {
        Self {
            deployed_contracts: commitment_state_diff.address_to_class_hash,
            storage_diffs: commitment_state_diff.storage_updates,
            class_hash_to_compiled_class_hash: commitment_state_diff.class_hash_to_compiled_class_hash,
            nonces: commitment_state_diff.address_to_nonce,
            // TODO(AlonH): Remove this when the structure of storage diffs changes.
            deprecated_declared_classes: Vec::new(),   // ← always empty
        }
    }
}
```

**Corrupted commitment path.** `BlockExecutionArtifacts::new` calls `calculate_block_commitments` with this truncated `ThinStateDiff`:

```rust
// crates/apollo_batcher/src/block_builder.rs  (lines 160-166)
let (header_commitments, measurements) = calculate_block_commitments(
    &transactions_data,
    ThinStateDiff::from(commitment_state_diff.clone()),   // deprecated_declared_classes = []
    l1_da_mode,
    &block_info.starknet_version,
)
.await;
```

Inside `calculate_block_commitments`:

```rust
// crates/starknet_api/src/block_hash/block_hash_calculator.rs  (lines 318-327)
let concatenated_counts = concat_counts(
    transactions_data.len(),
    event_leaf_elements.len(),
    state_diff.len(),   // ThinStateDiff::len() counts deprecated_declared_classes.len() → 0 instead of N
    l1_da_mode,
);
let state_diff_length = state_diff.len();   // same wrong value
```

And `calculate_state_diff_hash` chains the (empty) `deprecated_declared_classes` into the Poseidon hash:

```rust
// crates/starknet_api/src/block_hash/state_diff_hash.rs  (lines 30-41)
hash_chain = chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
// chains length=0 and zero entries instead of the real N entries
```

Both `state_diff_commitment` and `concatenated_counts` are stored in `BlockHeaderCommitments`, which feeds `PartialBlockHashComponents::new` and ultimately `calculate_block_hash`:

```rust
// crates/starknet_api/src/block_hash/block_hash_calculator.rs  (lines 260-261)
.chain(&block_commitments.concatenated_counts)       // wrong
.chain(&block_commitments.state_diff_commitment.0.0) // wrong
```

The `partial_block_hash` used for consensus and the final block hash are therefore both wrong for any block that contains a deprecated Declare transaction.

**Contrast with the RPC path.** The RPC execution utility correctly threads the deprecated class hash through manually:

```rust
// crates/apollo_rpc_execution/src/execution_utils.rs  (lines 130-144)
pub fn induced_state_diff(..., deprecated_declared_class_hash: Option<ClassHash>) -> ... {
    Ok(ThinStateDiff {
        ...
        deprecated_declared_classes: deprecated_declared_class_hash
            .map_or_else(Vec::new, |class_hash| vec![class_hash]),
        ...
    })
}
```

The block-building path has no equivalent mechanism.

---

### Impact Explanation

Every block that includes a deprecated (Cairo 0) Declare transaction will have:

- A wrong `state_diff_commitment` (Poseidon hash over the state diff omits the declared class hashes).
- A wrong `concatenated_counts` field (the packed `state_diff_length` sub-field is too small by the number of deprecated declarations).
- A wrong `partial_block_hash` (used as the proposal commitment exchanged during consensus).
- A wrong final block hash (stored in storage and propagated to L1).

This matches the **Critical** impact category: *Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input.*

---

### Likelihood Explanation

Deprecated (Cairo 0) Declare transactions are still part of the Starknet protocol and the blockifier explicitly handles them. The `StateMaps.declared_contracts` field is populated during execution. Any block that includes even one such transaction triggers the wrong commitment. The trigger requires no privileged access — any user can submit a deprecated Declare transaction.

---

### Recommendation

Add `deprecated_declared_classes` to `CommitmentStateDiff` and populate it from `StateMaps.declared_contracts` (filtering for entries where the `bool` value marks a deprecated class):

```rust
pub struct CommitmentStateDiff {
    pub address_to_class_hash: IndexMap<ContractAddress, ClassHash>,
    pub address_to_nonce: IndexMap<ContractAddress, Nonce>,
    pub storage_updates: IndexMap<ContractAddress, IndexMap<StorageKey, Felt>>,
    pub class_hash_to_compiled_class_hash: IndexMap<ClassHash, CompiledClassHash>,
    pub deprecated_declared_classes: Vec<ClassHash>,   // ← add this
}

impl From<StateMaps> for CommitmentStateDiff {
    fn from(diff: StateMaps) -> Self {
        let deprecated_declared_classes = diff.declared_contracts
            .iter()
            .filter_map(|(hash, &is_new)| if !is_new { Some(*hash) } else { None })
            .collect();
        Self {
            address_to_class_hash: IndexMap::from_iter(diff.class_hashes),
            storage_updates: StorageDiff::from(StorageView(diff.storage)),
            class_hash_to_compiled_class_hash: IndexMap::from_iter(diff.compiled_class_hashes),
            address_to_nonce: IndexMap::from_iter(diff.nonces),
            deprecated_declared_classes,
        }
    }
}
```

Then remove the hard-coded `Vec::new()` in `ThinStateDiff::from(CommitmentStateDiff)` and propagate the field normally.

---

### Proof of Concept

1. Submit a deprecated (Cairo 0) Declare transaction to the sequencer.
2. The blockifier executes it; `StateMaps.declared_contracts` is populated with `(class_hash, false)`.
3. `CommitmentStateDiff::from(state_maps)` is called; `declared_contracts` is ignored.
4. `BlockExecutionArtifacts::new` calls `calculate_block_commitments` with `ThinStateDiff { deprecated_declared_classes: vec![], … }`.
5. `calculate_state_diff_hash` produces hash `H_wrong` (missing the class hash entry).
6. `concat_counts` produces `concatenated_counts_wrong` with `state_diff_length` one less than the true value.
7. The block header stores `state_diff_commitment = H_wrong` and `concatenated_counts = concatenated_counts_wrong`.
8. `calculate_block_hash` chains both wrong values, producing a block hash that does not match what any honest verifier or prover would compute from the actual state diff. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) [7](#0-6)

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
