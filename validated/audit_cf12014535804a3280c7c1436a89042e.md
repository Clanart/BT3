### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS Cairo program does not verify that the new class hash supplied by a contract corresponds to a previously declared contract class. This is an explicitly acknowledged missing check (marked with a `TODO` comment). An unprivileged contract can call `replace_class` with an arbitrary, undeclared class hash, permanently bricking itself and freezing any funds it holds, because subsequent calls to that contract will cause the OS proof to fail.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall. After deducting gas, it reads the requested new class hash directly from the syscall request and writes it into `contract_state_changes` without any validation that the hash corresponds to a declared class: [1](#0-0) 

The comment at line 898 explicitly acknowledges the missing check:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
```

The `class_hash` value from `request.class_hash` is written directly into the new `StateEntry` and committed to `contract_state_changes` with no constraint that it exists in `contract_class_changes` (the declared-class mapping).

When any subsequent call is made to this contract (in the same or a future block), `execute_entry_point` performs: [2](#0-1) 

A `dict_read` on `contract_class_changes` for an undeclared class hash returns the default value `0`. The subsequent `find_element` call then searches for a compiled class with hash `0`. If no such compiled class exists in the block's facts bundle, `find_element` panics, causing the OS proof to fail. The sequencer can never include any call to this contract in a valid block again — the contract is permanently inaccessible.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any funds (ERC-20 tokens, ETH, or other assets) held in a contract's storage become permanently inaccessible once the contract's class hash is replaced with an undeclared value. The contract cannot be called, upgraded, or recovered because:

1. Every call to the contract causes the OS proof to fail at `find_element`.
2. No valid block can ever include a transaction targeting this contract.
3. There is no recovery mechanism — the state change is committed and irreversible.

---

### Likelihood Explanation

The attack path is directly reachable by any unprivileged actor who can trigger a contract to execute `replace_class`. Concrete scenarios:

- A malicious contract developer deploys a contract that accepts user deposits, then calls `replace_class(arbitrary_undeclared_felt)` to permanently freeze all deposited funds.
- A contract with a logic bug that allows an attacker to supply an arbitrary class hash to `replace_class` (e.g., via a misconfigured upgrade path).

No privileged role, leaked key, or external dependency is required. The `replace_class` syscall is a standard, gas-metered syscall available to any Sierra contract. [3](#0-2) 

---

### Recommendation

Before committing the new class hash to `contract_state_changes`, verify that it exists as a key in `contract_class_changes` (i.e., it was declared in a prior `declare` transaction and is present in the block's class-change dictionary). The check should assert:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the validation already performed in `execute_entry_point` when resolving a class for execution, and closes the gap between what the blockifier enforces off-chain and what the OS enforces on-chain. [4](#0-3) 

---

### Proof of Concept

1. Deploy contract `Vault` that holds user funds and exposes an `upgrade(class_hash: felt)` function that calls `replace_class(class_hash)`.
2. Users deposit funds into `Vault`.
3. Attacker calls `Vault.upgrade(0xDEAD)` where `0xDEAD` is not a declared class hash.
4. The OS processes `execute_replace_class`: gas is deducted, `contract_state_changes[vault_address].class_hash` is set to `0xDEAD`, no validation occurs.
5. The block containing step 3 is proven successfully (no call to `Vault` follows in the same block).
6. In any subsequent block, any call to `Vault` causes `execute_entry_point` to do `dict_read(contract_class_changes, 0xDEAD)` → returns `0`, then `find_element(..., key=0)` → panic → OS proof fails.
7. The sequencer can never include a call to `Vault` in a valid block. All funds are permanently frozen.

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    let revert_log = &revert_log[1];

    return ();
}
