Looking at the exact code path in `upsert` and the cache methods:

### Title
`upsert` Silently Overwrites `leaf_hash_to_index` Bijection, Enabling False Proof of Inclusion — (`crates/chia-datalayer/src/merkle/blob.rs`)

### Summary
`MerkleBlob::upsert` does not check whether `new_hash` is already present for a different key before updating the `leaf_hash_to_index` cache. When a caller upserts key1 with a hash that already belongs to key2, the cache entry for that hash is silently redirected to key1's index. Subsequent calls to `get_node_by_hash` with key2's hash return key1's key/value pair, producing a cryptographically false proof of inclusion.

### Finding Description

`insert` correctly guards against duplicate hashes: [1](#0-0) 

`upsert` has no equivalent guard. Its update path is: [2](#0-1) 

Step by step:

1. `remove_leaf(&leaf)` removes key1's old hash from `leaf_hash_to_index` and adds `index1` to `free_indexes`. [3](#0-2) 

2. `insert_entry_to_blob(leaf_index, &block)` is called with the block now carrying `new_hash = hash2`. [4](#0-3) 

3. Inside `insert_entry_to_blob`, `add_leaf` is called unconditionally: [5](#0-4) 

`HashMap::insert` silently overwrites the existing `leaf_hash_to_index[hash2] = index2` with `index1`. No error is returned. key2 still occupies `index2` in the blob with `hash2` stored in its node, but the cache now maps `hash2 → index1`.

### Impact Explanation

`get_node_by_hash` resolves entirely through `leaf_hash_to_index`: [6](#0-5) 

After the corrupt upsert, `get_node_by_hash(hash2)` returns `(key1, value1)` instead of `(key2, value2)`. Any DataLayer inclusion proof built on top of this lookup maps a leaf hash to the wrong key/value pair, constituting a forged proof of inclusion. Additionally, `leaf_hash_to_index` now has one fewer entry than `key_to_index` (key2's hash is orphaned), violating the bijection invariant that `check_integrity` explicitly enforces: [7](#0-6) 

This falls under: **High — DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.**

### Likelihood Explanation

Any caller with write access to a `MerkleBlob` (including Python/wasm bindings that expose `upsert`) can trigger this with two ordinary insert calls followed by one upsert. No privileged role, leaked key, or network-level attack is required.

### Recommendation

Add a duplicate-hash check at the top of the `upsert` update branch, mirroring the guard already present in `insert`:

```rust
pub fn upsert(&mut self, key: KeyId, value: ValueId, new_hash: &Hash) -> Result<(), Error> {
    let Ok((leaf_index, mut leaf, mut block)) = self.get_leaf_by_key(key) else {
        self.insert(key, value, new_hash, InsertLocation::Auto {})?;
        return Ok(());
    };

    // Guard: new_hash must not already belong to a different key
    if leaf.hash != *new_hash && self.block_status_cache.contains_leaf_hash(new_hash) {
        return Err(Error::HashAlreadyPresent());
    }

    self.block_status_cache.remove_leaf(&leaf)?;
    // ... rest unchanged
}
```

### Proof of Concept

```rust
use chia_datalayer::merkle::blob::{InsertLocation, MerkleBlob};
use chia_datalayer::{Hash, KeyId, ValueId};

let mut blob = MerkleBlob::new(vec![]).unwrap();

let key1 = KeyId(1);
let key2 = KeyId(2);
let hash1 = Hash([1u8; 32].into());
let hash2 = Hash([2u8; 32].into());

// Insert two leaves with distinct hashes
blob.insert(key1, ValueId(10), &hash1, InsertLocation::Auto {}).unwrap();
blob.insert(key2, ValueId(20), &hash2, InsertLocation::Auto {}).unwrap();

// Upsert key1 to use key2's hash — no error returned
blob.upsert(key1, ValueId(99), &hash2).unwrap();

// get_node_by_hash(hash2) now returns key1's data, not key2's
let (returned_key, returned_value) = blob.get_node_by_hash(hash2).unwrap();
assert_eq!(returned_key, key1);   // PASSES — wrong key returned
assert_eq!(returned_value, ValueId(99));
// key2 still exists but its hash is orphaned in leaf_hash_to_index
```

    fn insert_entry_to_blob(&mut self, index: TreeIndex, block: &Block) -> Result<(), Error> {
        let new_block_bytes = block.to_bytes()?;
        let extend_index = self.extend_index();
        match index.cmp(&extend_index) {
            Ordering::Greater => return Err(Error::BlockIndexOutOfBounds(index)),
            Ordering::Equal => self.blob.extend_from_slice(&new_block_bytes),
            Ordering::Less => {
                self.blob[block_range(index)].copy_from_slice(&new_block_bytes);
            }
        }

        match block.node {
            Node::Leaf(leaf) => self.block_status_cache.add_leaf(index, leaf),
            Node::Internal(..) => self.block_status_cache.add_internal(index),
        }

        Ok(())
    }
