### Title
Deterministic Panic in `ProofOfSpace::update_digest` for Invalid v2 Proof Bytes — (`crates/chia-protocol/src/proof_of_space.rs`)

---

### Summary

`ProofOfSpace::update_digest()` unconditionally calls `.expect()` on the result of `quality_string()` for v2 proofs. Because `parse()` accepts any proof byte sequence without validating its content, an attacker can craft a syntactically valid but semantically invalid v2 `ProofOfSpace` that passes deserialization, then trigger a deterministic Rust panic the moment any code calls `get_hash()` on any struct containing that `ProofOfSpace` (`ChallengeBlockInfo`, `RewardChainBlock`, `FullBlock`, `RewardChainBlockUnfinished`).

---

### Finding Description

**Root cause — `update_digest` panics on `None`:** [1](#0-0) 

```rust
// for v2 proofs, we don't hash the full proof directly.
self.quality_string()
    .expect("internal error. Can't compute hash of invalid ProofOfSpace")
    .update_digest(digest);
```

`quality_string()` returns `Option<Bytes32>` and explicitly documents that it returns `None` for invalid proofs: [2](#0-1) 

The `k_size` is derived from proof length (`proof.len() * 8 / 128`), and `chia_pos2::quality_string_from_proof()` returns `None` when the proof content is invalid for the given `k_size`/`strength` combination. The Python-facing wrapper in `wheel/src/api.rs` confirms this: [3](#0-2) 

> "Returns the serialized quality string, or None if proof format is invalid."

**`parse()` does not validate proof bytes:**

For v2 proofs, `parse()` only checks that exactly one of `pool_public_key`/`pool_contract_puzzle_hash` is set. The proof bytes are accepted unconditionally: [4](#0-3) 

**`stream()` is safe but `update_digest()` is not — a critical asymmetry:**

`stream()` for v2 proofs writes the raw proof bytes directly without calling `quality_string()`: [5](#0-4) 

This means `to_bytes()` / `parse()` round-trips succeed silently, but `get_hash()` (which calls `update_digest()`) panics. The bug is invisible until hashing is attempted.

**Propagation through consensus structs:**

`RewardChainBlock::update_digest()` calls `self.proof_of_space.update_digest(digest)` directly: [6](#0-5) 

`FullBlock::update_digest()` calls `self.reward_chain_block.update_digest(digest)`: [7](#0-6) 

`ChallengeBlockInfo` uses `#[streamable]` (not `no_streamable`), so its derived `update_digest()` also calls `proof_of_space.update_digest(digest)`: [8](#0-7) 

---

### Impact Explanation

Any code path that calls `get_hash()` on a `FullBlock`, `RewardChainBlock`, `ChallengeBlockInfo`, or `RewardChainBlockUnfinished` containing a crafted v2 `ProofOfSpace` with invalid proof bytes will trigger a deterministic Rust `panic!`. In a native Rust consensus validator this crashes the process. Via PyO3, the panic is converted to a `PanicException`; if the Python node does not explicitly catch `PanicException` at the call site (which is non-standard — normal Python code catches `Exception`, not `BaseException` subclasses), the exception propagates and can crash the node process or corrupt its internal state. Because the panic is deterministic and reproducible from a single crafted network message, a single attacker can repeatedly crash any node that processes the block, constituting a network-wide DoS / chain halt vector.

---

### Likelihood Explanation

- No privileges required. Any peer can send a `FullBlock` or `ChallengeBlockInfo` over the Chia p2p protocol.
- `parse()` accepts the malformed object without error.
- The panic fires the first time `get_hash()` is called, which happens early in block validation (header hash computation precedes proof-of-space verification in standard Chia node logic).
- The attack is trivially reproducible and requires no cryptographic capability.

---

### Recommendation

Replace the `.expect()` with a graceful error return. `update_digest` should not be infallible for externally-supplied data. Options:

1. Change `update_digest` signature to `Result<(), Error>` for `ProofOfSpace` (and propagate through callers).
2. Validate the proof during `parse()` for v2 proofs (call `quality_string_from_proof` and return `Err(Error::InvalidPoS)` if it returns `None`), so the invariant "a parsed v2 ProofOfSpace always has a valid quality string" holds before `update_digest` is ever reached.

Option 2 is simpler and consistent with the existing `parse()` guard for the pool key/contract exclusivity check.

### Proof of Concept

```rust
use chia_protocol::{Bytes, Bytes32, ProofOfSpace};
use chia_bls::G1Element;
use chia_traits::Streamable;

let pos = ProofOfSpace::new(
    Bytes32::default(),
    Some(G1Element::default()),  // pool_public_key set
    None,                         // pool_contract_puzzle_hash absent
    G1Element::default(),
    1,                            // version = 1 (v2 proof)
    0,                            // plot_index
    0,                            // meta_group
    10,                           // strength = 10 (requires specific proof length)
    0,                            // size (unused for v2)
    Bytes::from(vec![0xAB; 3]),   // proof: 3 bytes — wrong length for strength=10
);

// parse() succeeds — no proof content validation
let buf = pos.to_bytes().unwrap();  // stream() also succeeds — no quality_string() call
let parsed = ProofOfSpace::parse::<false>(
    &mut std::io::Cursor::new(&buf)
).unwrap();

// PANICS: "internal error. Can't compute hash of invalid ProofOfSpace"
let _hash = parsed.get_hash();
```

Embed this `ProofOfSpace` in a `ChallengeBlockInfo` or `FullBlock`, serialize it, and send it to any Chia node. The panic fires when the node computes the block's header hash.

    /// returns the quality string of the v2 proof of space.
    /// returns None if this is a v1 proof or if the proof is invalid.
    pub fn quality_string(&self) -> Option<Bytes32> {
        if self.version != 1 {
            return None;
        }

        let k_size = (self.proof.len() * 8 / 128) as u8;
        let plot_id = self.compute_plot_id().to_bytes();
        chia_pos2::quality_string_from_proof(&plot_id, k_size, self.strength, self.proof.as_slice())
            .map(|quality| {
                let mut sha256 = Sha256::new();
                sha256.update(chia_pos2::serialize_quality(
                    &quality.chain_links,
                    self.strength,
                ));
                sha256.finalize().into()
            })
    }
