### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary class hash as the replacement class for a contract without verifying that the hash is actually declared in the contract class tree. An unprivileged contract can replace its own class with a non-existent class hash, permanently bricking the contract and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall. It reads the requested new class hash from the syscall request and directly writes it into the contract's `StateEntry` in `contract_state_changes` — with no check that the class hash exists in `contract_class_changes` (the declared class tree):

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The `contract_class_changes` dictionary — which tracks which class hashes are actually declared — is not an implicit argument to `execute_replace_class` and is never consulted. The function signature does not even receive it: [2](#0-1) 

Compare this to `execute_deploy`, which also sets a class hash on a new contract but at least derives it from a `DeployRequest` that was validated upstream. The `replace_class` path has no equivalent guard.

The `execute_replace_class` handler is dispatched from `execute_syscalls` when the selector matches `REPLACE_CLASS_SELECTOR`: [3](#0-2) 

The `contract_class_changes` dict is available at the `execute_syscalls` level but is not threaded into `execute_replace_class`: [4](#0-3) 

The analogy to the external report is direct: just as the farm admin is set at initialization time and never re-validated when the lending market owner changes (a stale relationship between two linked entities), here the contract's `class_hash` field in the state tree is updated without re-validating its relationship to the class declaration tree. The two trees — contract state and contract class — are supposed to be consistent, but `replace_class` breaks that invariant.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

After a successful `replace_class` call with an undeclared hash:

- The contract's `StateEntry.class_hash` in the Patricia state tree points to a class hash that has no entry in the contract class tree.
- Every future call to that contract will fail at class resolution time, because the OS will attempt to look up the class and find nothing.
- Any ERC-20 tokens, ETH, or other assets held in the contract's storage become permanently inaccessible.
- The state transition is committed to the proven block output and cannot be rolled back.

The `get_contract_state_hash` function commits `class_hash` into the Merkle leaf: [5](#0-4) 

Once the block is proven and the state root is updated, the corrupt class hash is part of the canonical state.

---

### Likelihood Explanation

**High.** The `replace_class` syscall is a standard, publicly documented StarkNet syscall callable by any contract. No privileged role, leaked key, or operator cooperation is required. An attacker needs only to:

1. Deploy a contract (or control one that holds user funds).
2. Invoke a function in that contract that calls `replace_class` with an arbitrary felt value as the class hash.

The OS will accept the syscall, write the invalid class hash into the state, and include it in the proven output. The TODO comment in the code (`// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.`) confirms the developers are aware the check is absent. [6](#0-5) 

---

### Recommendation

Add `contract_class_changes: DictAccess*` as an implicit argument to `execute_replace_class` and perform a `dict_read` on it with `request.class_hash` as the key before writing the new `StateEntry`. Assert that the returned value is non-zero (i.e., the class is declared). This mirrors the invariant enforced during `execute_declare_transaction`, where `prev_value=0` and `assert_not_zero(compiled_class_hash)` together guarantee only valid classes enter the class tree: [7](#0-6) 

---

### Proof of Concept

1. Attacker deploys a shared vault contract (e.g., a simple ERC-20 wrapper) that accepts deposits from users. Users deposit funds; the contract's storage now holds balances.
2. Attacker calls a function on the vault contract that internally issues the `replace_class` syscall with `class_hash = 0xdeadbeef` (an arbitrary felt that was never declared).
3. The OS processes the block. `execute_replace_class` in `syscall_impls.cairo` writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes` with no validation against `contract_class_changes`.
4. `state_update` in `state.cairo` squashes and commits the state, including the corrupt entry, into the Patricia tree. The new global state root reflects the invalid class hash.
5. All subsequent calls to the vault contract fail: the OS cannot resolve class hash `0xdeadbeef` to any compiled class. All user funds are permanently frozen.

        // Not enough gas to execute the syscall.
        return ();

func get_contract_state_hash{hash_ptr: HashBuiltin*}(
    class_hash: felt, storage_root: felt, nonce: felt
) -> (hash: felt) {
    const CONTRACT_STATE_HASH_VERSION = 0;
    if (class_hash == UNINITIALIZED_CLASS_HASH) {
        if (storage_root == 0) {
            if (nonce == 0) {
                return (hash=0);
            }
        }
    }

    // Set res = H(H(class_hash, storage_root), nonce).
    let (hash_value) = hash2(class_hash, storage_root);
    let (hash_value) = hash2(hash_value, nonce);

    // Return H(hash_value, CONTRACT_STATE_HASH_VERSION). CONTRACT_STATE_HASH_VERSION must be in the
    // outermost hash to guarantee unique "decoding".
    let (hash) = hash2(hash_value, CONTRACT_STATE_HASH_VERSION);
    return (hash=hash);
