### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Bricking — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary `class_hash` value and writes it directly into the contract state without verifying that the hash corresponds to a previously declared class. This is an exact analog of the PriceFeed state-transition bypass: just as the PriceFeed transitions to a new price source without validating the price is in range, the OS transitions a contract to a new class without validating the class exists. An unprivileged user can exploit this to permanently brick any contract they control, freezing all funds held within it.

---

### Finding Description

In `execute_replace_class` (lines 878–916 of `syscall_impls.cairo`), the syscall handler reads `request.class_hash` and immediately writes it into `contract_state_changes` via `dict_update`. There is no check that `class_hash` is present in `contract_class_changes` (the declared-class registry) or in `compiled_class_facts_bundle`.

The code itself acknowledges this with an explicit TODO:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

Contrast this with `execute_entry_point`, which, when a contract is later *called*, must look up the class hash in `contract_class_changes` and then find the corresponding `CompiledClassFact` via `find_element`. If the class hash stored by `replace_class` does not exist in the compiled class facts bundle, `find_element` will fail, making it impossible to generate a valid proof for any block that includes a call to the bricked contract. [2](#0-1) 

The missing validation is the exact check that `execute_declare_transaction` enforces when a class is first introduced: it verifies the class hash is a valid Sierra class hash pre-image via `finalize_class_hash`. [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with a non-existent hash:

1. The state transition is committed to the Merkle tree and published on L1.
2. Any future transaction that calls the contract will cause the OS to fail at `find_element` (no compiled class fact for the garbage hash), making it impossible to produce a valid proof for any block containing such a call.
3. The sequencer is forced to permanently exclude all calls to the contract.
4. All funds (ERC-20 balances, NFTs, ETH) held in the contract's storage are permanently inaccessible — no withdrawal, transfer, or recovery transaction can ever be proven.

---

### Likelihood Explanation

**High.** The attack path requires only standard, unprivileged protocol access:

1. Deploy any contract (permissionless on StarkNet).
2. Have the contract execute the `replace_class` syscall with an arbitrary non-existent hash (e.g., `felt(1)`).
3. The OS accepts the call, charges gas, and commits the invalid class hash to state.

No privileged role, leaked key, or operator cooperation is needed. The attacker can target their own contract (self-bricking to grief users who deposited funds) or any contract they have write access to via a `replace_class` call path.

---

### Recommendation

**Short term:** In `execute_replace_class`, add a check that `request.class_hash` is present in `contract_class_changes` (i.e., it was declared in the current or a prior block). This mirrors the check already enforced in `execute_declare_transaction` and is exactly what the existing TODO comment calls for.

**Long term:** Enumerate all state-transition points in the OS (class replacement, contract deployment, storage writes) and audit each for missing invariant checks before the state is committed. Establish a policy that every class hash written into `contract_state_changes` must be traceable to a validated declaration.

---

### Proof of Concept

1. Attacker deploys contract `C` (which holds user funds or is a wallet).
2. Attacker calls a function on `C` that internally invokes `replace_class(class_hash=0x1)`.
3. `execute_replace_class` in `syscall_impls.cairo` (line 896) reads `class_hash = 0x1`, skips the missing declared-class check, and writes `StateEntry(class_hash=0x1, ...)` into `contract_state_changes`.
4. The block is proven and the state root (including `C`'s new class hash `0x1`) is committed to L1.
5. In any subsequent block, a user submits a withdrawal transaction to `C`. The OS calls `execute_entry_point` for `C`, reads `class_hash = 0x1` from state, calls `dict_read` on `contract_class_changes` (returns `compiled_class_hash = 0`), then calls `find_element(..., key=0)` — which fails because no compiled class with hash `0` exists.
6. The block proof is invalid. The sequencer must drop all calls to `C`. All funds in `C` are permanently frozen. [4](#0-3)
