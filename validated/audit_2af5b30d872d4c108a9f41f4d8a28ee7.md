### Title
Missing State Diff Commitment Verification in P2P Sync Allows Malicious Peer to Inject Arbitrary State — (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

### Summary

The P2P sync client accepts and permanently stores a `ThinStateDiff` whose Poseidon hash does not match the `state_diff_commitment` field in the stored block header. A malicious peer controls both the header (including `state_diff_length` and `state_diff_commitment`) and the subsequent state diff chunks. Because no commitment check is ever performed, the peer can supply exactly `state_diff_length` units of attacker-chosen content and the node will store it as authoritative state.

---

### Finding Description

**Header path — no cryptographic verification**

`HeaderStreamBuilder::parse_data_for_block` accepts a `SignedBlockHeader` from the network and stores it verbatim. The only checks performed are:

- Block number ordering [1](#0-0) 
- Signature vector length == 1 [2](#0-1) 

`verify_block_signature` exists in `starknet_api` and signs `poseidon_hash(block_hash, state_diff_commitment)`, but it is **never called** in this path. [3](#0-2) 

The header — including the peer-supplied `state_diff_commitment` and `state_diff_length` — is written to storage unconditionally. [4](#0-3) 

**State diff path — length-only guard, no hash guard**

`StateDiffStreamBuilder::parse_data_for_block` reads `target_state_diff_len` directly from the stored (peer-supplied) header: [5](#0-4) 

It then loops, accumulating chunks until `current_state_diff_len == target_state_diff_len`: [6](#0-5) 

After the loop, the assembled `ThinStateDiff` is stored with no check that `calculate_state_diff_hash(&result) == header.state_diff_commitment`. [7](#0-6) 

**Correction to the question's specific scenario**: sending length 1 when the header declares 100 would be rejected by the `WrongStateDiffLength` guard. The actual attack requires sending **exactly N chunks** (where N == `state_diff_length`) with arbitrary content. The proof idea at the end of the question (send 5 chunks with attacker-chosen content, assert hash mismatch) is correct.

---

### Impact Explanation

The node stores a `ThinStateDiff` that does not correspond to the `state_diff_commitment` committed in the block header. This corrupted state diff becomes the authoritative state for that block number. Any subsequent RPC call (`starknet_getStorageAt`, `starknet_getNonce`, `starknet_getClassHashAt`, etc.) will return wrong values derived from the injected state. If the node also executes transactions on top of this state (e.g., as a sequencer syncing historical blocks), execution will operate on wrong storage values, nonces, and class hashes — satisfying the "Wrong state… from blockifier/syscall/execution logic" criterion.

---

### Likelihood Explanation

Any peer that can establish a P2P connection to the syncing node can perform this attack. No operator or validator privileges are required. The attack is deterministic and requires no brute force.

---

### Recommendation

After assembling the final `ThinStateDiff` (after the loop, before returning `Ok(Some(...))`), compute its Poseidon commitment and compare it against the `state_diff_commitment` stored in the header:

```rust
let computed = calculate_state_diff_hash(&result);
let expected = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .expect("...")
    .state_diff_commitment
    .ok_or(...)?;
if computed != expected {
    return Err(ParseDataError::BadPeer(BadPeerError::StateDiffCommitmentMismatch { ... }));
}
```

Additionally, `HeaderStreamBuilder::parse_data_for_block` should call `verify_block_signature` against the known sequencer public key before accepting any header, so that neither `state_diff_commitment` nor `state_diff_length` can be forged.

    fn parse_data_for_block<'a>(
        state_diff_chunks_response_manager: &'a mut ClientResponsesManager<
            DataOrFin<StateDiffChunk>,
        >,
        block_number: BlockNumber,
        storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            let mut result = ThinStateDiff::default();
            let mut prev_result_len = 0;
            let mut current_state_diff_len = 0;
            let target_state_diff_len = storage_reader
                .begin_ro_txn()?
                .get_block_header(block_number)?
                .expect("A header with number lower than the header marker is missing")
                .state_diff_length
                .ok_or(P2pSyncClientError::OldHeaderInStorage {
                    block_number,
                    missing_field: "state_diff_length",
                })?;

            while current_state_diff_len < target_state_diff_len {
                let maybe_state_diff_chunk = state_diff_chunks_response_manager
                    .next()
                    .await
                    .ok_or(ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }))?;
                let Some(state_diff_chunk) = maybe_state_diff_chunk?.0 else {
                    if current_state_diff_len == 0 {
                        return Ok(None);
                    } else {
                        return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                            expected_length: target_state_diff_len,
                            possible_lengths: vec![current_state_diff_len],
                        }));
                    }
                };
                prev_result_len = current_state_diff_len;
                if state_diff_chunk.is_empty() {
                    return Err(ParseDataError::BadPeer(BadPeerError::EmptyStateDiffPart));
                }
                // It's cheaper to calculate the length of `state_diff_part` than the length of
                // `result`.
                current_state_diff_len += state_diff_chunk.len();
                unite_state_diffs(&mut result, state_diff_chunk)?;
            }

            if current_state_diff_len != target_state_diff_len {
                return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                    expected_length: target_state_diff_len,
                    possible_lengths: vec![prev_result_len, current_state_diff_len],
                }));
            }

            validate_deprecated_declared_classes_non_conflicting(&result)?;
            Ok(Some((result, block_number)))
        }
        .boxed()
    }
