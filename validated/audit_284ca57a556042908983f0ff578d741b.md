### Title
Zero-Fee Cosmos Tx Bypass via Non-EVM Denom Fee When baseFee=0 — (`ante/evm/fee_checker.go`, `ante/evm/nativefee.go`)

### Summary

`NewDynamicFeeChecker` computes `feeCap` exclusively from the EVM-denom amount of `feeCoins`, ignoring any non-EVM denom coins. When `baseFee=0` (London HF enabled, `NoBaseFee=false`, `params.BaseFee=0`), a tx carrying only non-EVM denom fees passes the `feeCap >= baseFeeInt` check with `feeCap=0 >= baseFeeInt=0`, produces `effectiveFee=[{aevmos,0}]`, and `checkDeductFee` skips `DeductFees` because `fee.IsZero()=true`. The original non-EVM denom fee is silently discarded. The tx commits with zero fee deducted in any denom.

---

### Finding Description

**Step 1 — `types.GetBaseFee` returns `big.Int(0)`, not nil, when London HF is enabled and `NoBaseFee=false` with `params.BaseFee=0`:** [1](#0-0) 

`feemarketParams.GetBaseFee()` returns `p.BaseFee.BigInt()` when `NoBaseFee=false`. If `params.BaseFee=0`, this is `big.Int(0)`. The nil-check in `GetBaseFee` only fires when `NoBaseFee=true`; a zero value passes through as a non-nil `*big.Int`. [2](#0-1) 

**Step 2 — `NewDynamicFeeChecker` only reads the EVM denom amount:** [3](#0-2) 

`fee = feeCoins.AmountOf(denom)` returns `0` when the tx carries only `uatom`. `feeCap = 0 / gas = 0`. `baseFeeInt = 0`. The guard `feeCap.LT(baseFeeInt)` evaluates `0 < 0 = false` — it passes.

**Step 3 — `effectiveFee` is constructed as `[{aevmos, 0}]`:** [4](#0-3) 

`EffectiveGasPrice(0, 0, MaxInt64) = min(0+MaxInt64, 0) = 0`. `effectiveFee = [{aevmos, 0*gas}] = [{aevmos, 0}]`. The original `1000uatom` is completely discarded — it is never returned or deducted.

**Step 4 — `checkDeductFee` skips `DeductFees` because `fee.IsZero()=true`:** [5](#0-4) 

`sdk.Coins{{aevmos, 0}}.IsZero()` returns `true` (all coin amounts are zero). `DeductFees` is skipped. The tx commits with zero fee deducted.

**When is `baseFee=0` reachable?**

The keeper test explicitly documents this state: [6](#0-5) 

`"enable london HF, not enable feemarket"` → `big.NewInt(0)`. Additionally, when `NoBaseFee=false` and `params.BaseFee` has been driven to 0 by the dynamic fee calculation (possible when `MinGasPrice=0`, the default), the same condition holds. [7](#0-6) 

With `MinGasPrice=0` (default), `BigMax(baseFee, 0)` allows baseFee to reach 0.

---

### Impact Explanation

Any unprivileged user can submit Cosmos txs (non-EVM messages) with fee denominated only in a non-EVM denom (e.g., `uatom`) and have them commit with **zero fee deducted in any denom**. The fee collector receives nothing. This violates the invariant that every committed tx must pay at least `baseFee * gas` in the EVM denom. This fits the High impact category: *ante handler bug that permits invalid transactions to commit with fees mis-accounted*.

---

### Likelihood Explanation

`baseFee=0` is a documented, reachable chain state (London HF enabled, feemarket disabled or `params.BaseFee=0`). The default `MinGasPrice=0` means the dynamic fee calculation can drive baseFee to 0 on a low-traffic chain. No privileged access is required — any tx submitter can exploit this.

---

### Recommendation

In `NewDynamicFeeChecker`, after computing `effectiveFee`, verify that the original `feeCoins` actually cover the `effectiveFee` in the EVM denom. Specifically:

1. If `baseFee > 0`: reject if `feeCoins.AmountOf(denom) < effectivePrice * gas`.
2. If `baseFee == 0` and `effectiveFee.IsZero()`: still require that `feeCoins` are empty or contain only the EVM denom — do not silently accept non-EVM denom fees as satisfying the EVM fee requirement.
3. Alternatively, return the original `feeCoins` (not `effectiveFee`) when `effectiveFee` is zero, so that non-EVM denom fees are still deducted.

---

### Proof of Concept

```
Preconditions:
  - London HF enabled (LondonBlock <= current height)
  - NoBaseFee=false, params.BaseFee=0 (or feemarket disabled)
  - MinGasPrice=0 (default)

Attack:
  1. Construct a Cosmos tx (e.g., MsgSend) with:
       fee = [1000uatom], gas = 21000
  2. Submit via BroadcastTx.

Trace through NewDynamicFeeChecker:
  baseFee = GetBaseFee(...) = big.Int(0)  // not nil, London enabled
  fee = feeCoins.AmountOf("aevmos") = 0
  feeCap = 0 / 21000 = 0
  baseFeeInt = 0
  feeCap.LT(baseFeeInt) → 0 < 0 → false  // PASSES
  effectivePrice = EffectiveGasPrice(0, 0, MaxInt64) = 0
  effectiveFee = [{aevmos, 0}]            // 1000uatom discarded

Trace through checkDeductFee:
  fee = [{aevmos, 0}]
  fee.IsZero() = true                     // DeductFees SKIPPED

Result: tx commits, fee_collector balance unchanged, 1000uatom NOT deducted.

Assert: fee_collector.GetBalance("aevmos") == 0 (unchanged)
        fee_collector.GetBalance("uatom")  == 0 (unchanged)
        tx result: success
```
