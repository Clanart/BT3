### Title
`env::signer_account_id()` (tx.origin Equivalent) Used for Trusted-Relayer Check in `fast_fin_transfer` — (File: near/omni-bridge/src/lib.rs)

---

### Summary

`ft_on_transfer` in the NEAR bridge resolves the "signer" using `env::signer_account_id()` — the NEAR equivalent of EVM's `tx.origin` — and passes it as `signer_id` into `fast_fin_transfer`, where it is used as the sole authorization gate for the trusted-relayer restriction. Any contract that a trusted relayer calls can re-use that relayer's `signer_account_id` to pass the check and execute an unauthorized fast-finalization of a cross-chain transfer.

---

### Finding Description

In `ft_on_transfer`, the bridge explicitly avoids trusting `sender_id` (the direct predecessor, i.e. the token contract) and instead reads the original transaction signer:

```rust
// We can't trust sender_id to pay for storage as it can be spoofed.
let signer_id = env::signer_account_id();   // ← tx.origin equivalent
``` [1](#0-0) 

This `signer_id` is then forwarded directly into `fast_fin_transfer`:

```rust
BridgeOnTransferMsg::FastFinTransfer(fast_fin_transfer_msg) => {
    self.fast_fin_transfer(token_id, amount, signer_id, fast_fin_transfer_msg)
}
``` [2](#0-1) 

Inside `fast_fin_transfer`, the only authorization check is:

```rust
require!(self.is_trusted_relayer(&signer_id), "Relayer is not active");
``` [3](#0-2) 

In NEAR, `env::signer_account_id()` is the account that cryptographically signed the **original** transaction and remains constant across the entire cross-contract call chain. `env::predecessor_account_id()` is the direct caller (the token contract in this context). By using `signer_account_id()` for the relayer check, the bridge inherits the same class of vulnerability as EVM's `tx.origin`: any contract that a trusted relayer interacts with can piggyback on that relayer's identity.

**Attack path:**

1. Attacker deploys a malicious contract `M` that holds some amount of a bridge-registered token.
2. Attacker lures a trusted relayer `R` into calling `M` (e.g., through a seemingly legitimate DeFi interaction, airdrop claim, or any public-facing function).
3. Inside its execution, `M` calls `ft_transfer_call` on the bridge token, specifying the bridge contract as receiver and embedding a `FastFinTransfer` message that names an attacker-controlled account as `relayer` and a target `transfer_id` corresponding to a real pending cross-chain transfer.
4. The token contract calls `ft_on_transfer` on the bridge. At this point `env::signer_account_id()` == `R` (the trusted relayer), so `is_trusted_relayer` returns `true`.
5. The bridge executes `fast_fin_transfer`: it immediately sends `amount - fee` tokens to the legitimate recipient and records the attacker's account as the `relayer` for `transfer_id`.
6. When the legitimate `fin_transfer` proof arrives later, the bridge pays the fee to the attacker's account instead of the legitimate relayer. [4](#0-3) 

The storage-balance deduction path in `init_transfer` is also affected — `signer_id` is used to deduct pre-deposited storage from the victim relayer's balance — but the fast-transfer path is the higher-impact vector. [5](#0-4) 

---

### Impact Explanation

An attacker can bypass the trusted-relayer restriction and execute unauthorized fast-finalization of cross-chain transfers. The concrete financial impact is theft of relayer fees: the attacker front-runs legitimate relayers on any pending transfer, records themselves as the relayer, and collects the fee when the proof is submitted. Additionally, the attacker can drain a trusted relayer's pre-deposited storage balance by triggering `init_transfer` through the same mechanism. This maps to: **High — proof/signature/MPC verification bypass enabling unauthorized transfer finalization**.

---

### Likelihood Explanation

A trusted relayer is a privileged but not isolated account. It routinely interacts with other contracts on NEAR (e.g., staking, DEX, governance). Any such interaction is sufficient to trigger the attack if the counterparty contract is malicious or upgradeable. The attacker only needs to hold a small amount of the bridge token to initiate `ft_transfer_call`. No leaked keys or colluding MPC signers are required.

---

### Recommendation

Replace `env::signer_account_id()` with `env::predecessor_account_id()` for the relayer identity check. Because `ft_on_transfer` is called by the token contract (the predecessor), the actual relayer must be identified from the `sender_id` parameter passed by the token contract, which is the account that called `ft_transfer_call`. `sender_id` is the NEAR equivalent of `msg.sender` in this context and cannot be forged by an intermediate contract.

```rust
// Before (vulnerable):
let signer_id = env::signer_account_id();

// After (safe):
// Use sender_id (the ft_transfer_call caller) for relayer authorization.
// sender_id is already passed into ft_on_transfer by the token contract.
```

In `fast_fin_transfer`, change the trusted-relayer check to use `sender_id` instead of `signer_id`:

```rust
require!(self.is_trusted_relayer(&sender_id), "Relayer is not active");
```

For storage-balance deductions in `init_transfer`, the same substitution applies: deduct from `sender_id`, not from `signer_id`.

---

    pub fn ft_on_transfer(&mut self, sender_id: AccountId, amount: U128, msg: String) {
        let token_id = env::predecessor_account_id();
        let parsed_msg: BridgeOnTransferMsg = serde_json::from_str(&msg)
            .or_else(|_| serde_json::from_str(&msg).map(BridgeOnTransferMsg::InitTransfer))
            .near_expect(BridgeError::ParseMsg);

        // We can't trust sender_id to pay for storage as it can be spoofed.
        let signer_id = env::signer_account_id();
        let promise_or_promise_index_or_value = match parsed_msg {
            BridgeOnTransferMsg::InitTransfer(init_transfer_msg) => {
                self.init_transfer(sender_id, signer_id, token_id, amount, init_transfer_msg)
            }
            BridgeOnTransferMsg::FastFinTransfer(fast_fin_transfer_msg) => {
                self.fast_fin_transfer(token_id, amount, signer_id, fast_fin_transfer_msg)
            }
            BridgeOnTransferMsg::UtxoFinTransfer(utxo_fin_transfer_msg) => self.utxo_fin_transfer(
                token_id,
                amount,
                &signer_id,
                &sender_id,
                utxo_fin_transfer_msg,
            ),
            BridgeOnTransferMsg::SwapMigratedToken => {
                self.swap_migrated_token(sender_id, token_id, amount)
                    .detach();
                PromiseOrPromiseIndexOrValue::Value(U128(0))
            }
        };

        promise_or_promise_index_or_value.as_return();
    }

    #[allow(clippy::needless_pass_by_value)]
    fn fast_fin_transfer(
        &mut self,
        token_id: AccountId,
        amount: U128,
        signer_id: AccountId,
        fast_fin_transfer_msg: FastFinTransferMsg,
    ) -> PromiseOrPromiseIndexOrValue<U128> {
        require!(self.is_trusted_relayer(&signer_id), "Relayer is not active");

        let origin_token = self
            .get_token_address(
                fast_fin_transfer_msg.transfer_id.origin_chain,
                token_id.clone(),
            )
            .near_expect(BridgeError::TokenNotFound);

        let decimals = self
            .token_decimals
            .get(&origin_token)
            .near_expect(BridgeError::TokenDecimalsNotFound);

        let denormalized_amount =
            Self::denormalize_amount(fast_fin_transfer_msg.amount.0, decimals);
        let denormalized_fee = Self::denormalize_fee(&fast_fin_transfer_msg.fee, decimals);
        require!(
            denormalized_amount == amount.0 + denormalized_fee.fee.0,
            BridgeError::InvalidFastTransferAmount.as_ref()
        );

        if self.is_unified_transfer_finalised(&fast_fin_transfer_msg.transfer_id) {
            env::panic_str(BridgeError::TransferAlreadyFinalised.to_string().as_str());
        }

        let fast_transfer = FastTransfer {
            token_id: token_id.clone(),
            recipient: fast_fin_transfer_msg.recipient.clone(),
            amount: U128(denormalized_amount),
            fee: denormalized_fee,
            transfer_id: fast_fin_transfer_msg.transfer_id,
            msg: fast_fin_transfer_msg.msg,
        };

        if let OmniAddress::Near(recipient) = fast_fin_transfer_msg.recipient {
            let storage_deposit_amount = fast_fin_transfer_msg
                .storage_deposit_amount
                .map(|amount| amount.0)
                .unwrap_or_default();
            if storage_deposit_amount > 0 {
                self.update_storage_balance(
                    signer_id.clone(),
                    NearToken::from_yoctonear(storage_deposit_amount),
                    NearToken::from_yoctonear(0),
                );
            }

            let deposit_action = StorageDepositAction {
                account_id: recipient,
                token_id,
                storage_deposit_amount: fast_fin_transfer_msg
                    .storage_deposit_amount
                    .map(|amount| amount.0),
            };

            Self::check_or_pay_ft_storage(
                &deposit_action,
                &mut NearToken::from_yoctonear(storage_deposit_amount),
            )
            .then(
                Self::ext(env::current_account_id())
                    .with_static_gas(
                        FAST_TRANSFER_CALLBACK_GAS.saturating_add(FT_TRANSFER_CALL_GAS),
                    )
                    .fast_fin_transfer_to_near_callback(
                        &fast_transfer,
                        signer_id,
                        fast_fin_transfer_msg.relayer,
                    ),
            )
            .into()
        } else {
            self.fast_fin_transfer_to_other_chain(
                &fast_transfer,
                signer_id,
                fast_fin_transfer_msg.relayer,
            );
            PromiseOrPromiseIndexOrValue::Value(U128(0))
        }
    }
