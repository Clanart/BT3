### Title
`meta_tx_v0` Transaction Replay via Missing Nonce in Hash — (`crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_meta_tx_v0` syscall computes a transaction hash that contains **no nonce** (`additional_data_size=0`). The OS also explicitly skips nonce checking and signature validation for version-0 transactions. As a result, any valid `meta_tx_v0` signature observed on-chain can be replayed indefinitely by any unprivileged actor, enabling direct theft of funds from accounts that use this syscall.

---

### Finding Description

The `execute_meta_tx_v0` function in `syscall_impls.cairo` is the OS-level handler for the `meta_tx_v0` syscall. It constructs a version-0 transaction context and calls the target contract's `__execute__` entry point. The transaction hash is computed by `compute_meta_tx_v0_hash`:

```cairo
// transaction_hash/transaction_hash.cairo lines 294-315
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(...) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        tx_hash_prefix=INVOKE_HASH_PREFIX,
        version=0,
        contract_address=contract_address,
        entry_point_selector=entry_point_selector,
        calldata_size=calldata_size,
        calldata=calldata,
        max_fee=0,
        chain_id=chain_id,
        additional_data_size=0,          // ← no nonce
        additional_data=cast(0, felt*),  // ← no nonce
    );
    return tx_hash;
}
``` [1](#0-0) 

The hash is a pure function of `(contract_address, selector, calldata, chain_id)` — no nonce, no block number, no uniqueness token of any kind.

The OS then sets `nonce=0` in the constructed `TxInfo` and never increments any counter:

```cairo
// syscall_impls.cairo lines 343-363
tempvar new_tx_info = new TxInfo(
    version=0,
    ...
    nonce=0,   // ← always zero
    ...
);
``` [2](#0-1) 

Two OS-level guards that would normally prevent replay are both explicitly bypassed for version-0 transactions:

**1. Nonce check skipped:**
```cairo
// execute_transaction_utils.cairo lines 63-67
func check_and_increment_nonce{...}(tx_info: TxInfo*) -> () {
    // Do not handle nonce for version 0.
    if (tx_info.version == 0) {
        return ();
    }
``` [3](#0-2) 

**2. `__validate__` skipped:**
```cairo
// execute_transaction_utils.cairo lines 127-130
    // Do not run "__validate__" for version 0.
    if (tx_execution_info.tx_info.version == 0) {
        return ();
    }
``` [4](#0-3) 

The `meta_tx_v0` syscall is reachable from any Sierra contract via the normal syscall dispatch path:

```cairo
// execute_syscalls.cairo lines 343-344
assert selector = META_TX_V0_SELECTOR;
execute_meta_tx_v0(block_context=block_context, caller_execution_context=execution_context);
``` [5](#0-4) 

---

### Impact Explanation

**Critical — Direct loss of funds.**

An account contract's `__execute__` entry point receives a `TxInfo` with `transaction_hash = H(contract_address, selector, calldata, chain_id)`. If the account verifies the user's ECDSA signature against this hash (the standard pattern), a signature that was valid once is valid forever for the same calldata, because the hash never changes across blocks or invocations.

An attacker who observes a legitimate `meta_tx_v0` call on-chain (the signature is part of the outer transaction's calldata and is fully public) can:
1. Deploy a contract that calls `meta_tx_v0` with the same `(contract_address, selector, calldata, signature)`.
2. Submit an outer v3 invoke transaction calling that contract.
3. The OS computes the identical hash, the target account's `__execute__` verifies the same signature successfully, and the action (e.g., token transfer) executes again.

This can be repeated until the victim account is drained.

---

### Likelihood Explanation

**Medium-High.** The `meta_tx_v0` syscall is a live protocol feature. Any account that uses it for fund-moving operations (transfers, approvals, multicalls) exposes its signatures to replay. The attacker only needs to monitor the mempool or chain history for a `meta_tx_v0` call, extract the signature from the outer transaction's calldata, and replay it. No privileged access, leaked keys, or external conditions are required.

---

### Recommendation

Include a nonce in the `meta_tx_v0` hash, analogous to how `compute_l1_handler_transaction_hash` includes the L1 nonce as `additional_data`:

```cairo
// transaction_hash.cairo — compute_l1_handler_transaction_hash (reference pattern)
additional_data_size=1,
additional_data=&nonce,
``` [6](#0-5) 

For `meta_tx_v0`, the nonce should be sourced from the target contract's on-chain nonce (read from `contract_state_changes`) and incremented after each successful execution, mirroring `check_and_increment_nonce`. The nonce must be included in `compute_meta_tx_v0_hash` as `additional_data` so that each invocation produces a unique hash and old signatures cannot be reused.

---

### Proof of Concept

1. Alice submits a v3 invoke transaction. Her contract calls `meta_tx_v0` targeting Bob's account with calldata `[transfer, victim, 1000_ETH]` and a valid ECDSA signature `sig_A`.
2. The OS computes `H = hash(Bob_addr, __execute__, calldata, chain_id)` — no nonce. Bob's `__execute__` verifies `sig_A` against `H` and executes the transfer. 1000 ETH leaves Bob's account.
3. Attacker Eve observes Alice's transaction on-chain, extracts `(Bob_addr, calldata, sig_A)`.
4. Eve deploys `AttackContract` with a single function that calls `meta_tx_v0(Bob_addr, __execute__, calldata, sig_A)`.
5. Eve submits a v3 invoke calling `AttackContract`. The OS again computes the identical `H`, Bob's `__execute__` again verifies `sig_A` successfully (same hash, same signature), and another 1000 ETH is transferred.
6. Eve repeats step 5 until Bob's account is empty.

The root cause — `additional_data_size=0` in `compute_meta_tx_v0_hash` combined with the version-0 nonce bypass in `check_and_increment_nonce` — is entirely within the OS Cairo code in scope. [7](#0-6) [1](#0-0)

        return ();

    local contract_address = request.contract_address;

    if (selector != EXECUTE_ENTRY_POINT_SELECTOR) {
