### Title
`MerkleBlob::batch_insert` Silently Accepts Duplicate Keys, Corrupting the Merkle Tree Root — (File: `crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::batch_insert` does not validate for duplicate keys or hashes within the input batch. When duplicates are present, both leaves are written to the blob, but `BlockStatusCache` is silently overwritten by the last duplicate via `HashMap::insert`, causing the cache to diverge from the actual blob state. The resulting Merkle tree has an incorrect root hash, invalidating all proofs of inclusion and exclusion derived from it.

---

### Finding Description

`batch_insert` (lines 570–656) processes a batch of `((KeyId, ValueId), Hash)` tuples in two phases:

**Phase 1** — only when `leaf_count <= 1`: pops the last 2 items from the vector and calls `self.insert()`, which explicitly guards against duplicates: [1](#0-0) 

**Phase 2** — for all remaining items (or the entire batch when `leaf_count > 1`): iterates and calls `self.insert_entry_to_blob()` directly, with **no duplicate check**: [2](#0-1) 

`insert_entry_to_blob` calls `block_status_cache.add_leaf()` for each leaf: [3](#0-2) 

`add_leaf` calls `HashMap::insert`, which **silently overwrites** existing entries: [4](#0-3) 

When two batch entries share the same `KeyId`:

1. Both leaf nodes are written to the blob at distinct indexes.
2. `key_to_index` and `leaf_hash_to_index` are overwritten to point only to the second duplicate's index.
3. The first duplicate's index is never added to `free_indexes` — it is permanently orphaned.
4. Internal nodes are built from **all** leaf indexes (including both duplicates), producing an incorrect subtree hash.
5. The Merkle root is computed from this corrupted structure.

This is structurally identical to the reported checkpoint bug: a duplicate entry in the same "batch" (same block timestamp there, same `batch_insert` call here) causes a counter/cache to advance past the correct state, leaving an orphaned entry that corrupts the aggregate output.

The Python binding `py_batch_insert` only validates that `keys_values` and `hashes` have equal length — it performs no uniqueness check: [5](#0-4) 

By contrast, `insert()` explicitly rejects duplicates: [6](#0-5) 

The `Error` type defines both `KeyAlreadyPresent` and `HashAlreadyPresent` variants, confirming that uniqueness is a required invariant: [7](#0-6) 

---

### Impact Explanation

The corrupted Merkle root causes:

1. **`check_integrity()` failure** — `check_just_integrity` iterates the blob and finds the first duplicate leaf at index N, but the cache reports index M (the second duplicate). This triggers `IntegrityKeyToIndexCacheIndex(key, N, M)`: [8](#0-7) 

2. **Incorrect Merkle root** — the root hash is computed over a tree that contains a phantom duplicate leaf, so it does not represent the intended key-value set.

3. **Invalid proofs of inclusion** — `get_proof_of_inclusion` walks the lineage from the cache-tracked index; the orphaned duplicate leaf is unreachable, and the root hash it produces will not match any honest node's root. [9](#0-8) 

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

---

### Likelihood Explanation

`check_integrity_on_drop` is `false` in production builds (only `true` under `cfg!(test)`), so the corruption is silent at runtime: [10](#0-9) 

The Python binding is a network-facing API used by DataLayer sync. Any DataLayer peer that sends a delta containing duplicate `KeyId` values — or any Python caller that constructs a batch without pre-deduplication — will silently corrupt the local Merkle tree. The fuzz target for `batch_insert` does not exist (only `insert` is fuzzed), leaving this path untested:



---

### Recommendation

Add a pre-pass deduplication check at the start of `batch_insert`, mirroring the guards in `insert()`:

```rust
pub fn batch_insert(
    &mut self,
    mut keys_values_hashes: Vec<((KeyId, ValueId), Hash)>,
) -> Result<(), Error> {
    // Validate uniqueness within the batch and against existing keys
    let mut seen_keys: HashSet<KeyId> = HashSet::new();
    let mut seen_hashes: HashSet<Hash> = HashSet::new();
    for ((key, _), hash) in &keys_values_hashes {
        if self.block_status_cache.contains_key(*key) || !seen_keys.insert(*key) {
            return Err(Error::KeyAlreadyPresent());
        }
        if self.block_status_cache.contains_leaf_hash(hash) || !seen_hashes.insert(*hash) {
            return Err(Error::HashAlreadyPresent());
        }
    }
    // ... rest of existing logic
}
```

Additionally, add a fuzz target for `batch_insert` with duplicate-containing inputs.

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
