Looking at the code carefully, I need to trace the full path: `parse_data_for_block` → `write_to_storage` → `add_deprecated_class` in the class manager, and check whether any layer verifies `hash(class_body) == class_hash`.

### Title
P2P Sync Stores Unverified Deprecated Class Body Under Declared Class Hash — (File: crates/apollo_p2p_sync/src/client/class.rs)

---

### Summary

`ClassStreamBuilder::parse_data_for_block` verifies only that a peer-supplied `class_hash` appears in the stored state diff's `deprecated_declared_classes` set. It never verifies `hash(deprecated_class_body) == class_hash`. The `write_to_storage` path then calls `add_deprecated_class(class_hash, deprecated_class_body)`, which stores the body under the peer-supplied key with no hash check. A malicious peer can substitute an arbitrary Cairo 0 class body for any declared class hash, causing the wrong bytecode to be served to execution.

---

### Finding Description

**Step 1 — `parse_data_for_block` accepts without body-hash verification.**

For the `DeprecatedContractClass` branch (lines 136–141), the only check is whether `class_hash` is present in `deprecated_declared_classes` (line 137). The class body itself is never hashed and compared to `class_hash`. [1](#0-0) 

**Step 2 — `write_to_storage` passes the peer-supplied hash directly to the class manager.**

The peer-supplied `class_hash` is used as the storage key with no intermediate check. [2](#0-1) 

**Step 3 — `add_deprecated_class` stores without verification.**

`ClassManager::add_deprecated_class` calls `set_deprecated_class(class_id, class)` directly. There is no hash computation or comparison. [3](#0-2) 

**Contrast with Sierra classes.** For Cairo 1 classes, `add_class` recomputes `class_hash = sierra_class.calculate_class_hash()` from the body (line 66) and stores under that computed key. A wrong body is therefore stored under the wrong key (DoS / missing class), not under the declared class_hash. The critical path is exclusively through deprecated classes. [4](#0-3) 

The developers are aware of the analogous gap for Sierra classes (the TODO at line 39 reads "Verify class hash matches class manager response. report if not."), but no equivalent guard exists for deprecated classes at any layer. [5](#0-4) 

**Step 4 — Execution consumes the stored class without re-verification.**

`StateReaderAndContractManager::get_compiled_from_class_manager` fetches whatever the class manager holds for the given `class_hash` and returns it directly to the blockifier. No hash re-check occurs at retrieval time. [6](#0-5) 

---

### Impact Explanation

A node syncing via p2p stores an attacker-controlled Cairo 0 class body under a legitimate, committed `class_hash`. Every subsequent execution of a contract deployed with that class hash runs the wrong bytecode, producing wrong state transitions, wrong receipts, and wrong events. Impact: **Critical — wrong compiled class selected for execution**.

---

### Likelihood Explanation

Any p2p peer can exploit this. The `class_hash` values are public (they appear in the committed, already-stored state diff). The attacker only needs to send a crafted `(DeprecatedContractClass, class_hash)` message during the class-sync phase for a block whose state diff is already stored. No operator or validator privileges are required.

---

### Recommendation

In `parse_data_for_block`, after receiving a `DeprecatedContractClass`, compute its hash and compare it to the peer-supplied `class_hash`. If they differ, return `ParseDataError::BadPeer(BadPeerError::ClassHashMismatch { class_hash })` and report the peer. The same check should be added in `write_to_storage` as a defence-in-depth guard. The existing TODO for Sierra classes (line 39) should also be resolved by comparing the `class_hash` returned by `add_class` against the expected key.

---

### Proof of Concept

1. Obtain a valid state diff for block N that lists `class_hash_A` in `deprecated_declared_classes`.
2. As a p2p peer, respond to the class query for block N with `DataOrFin(Some((DeprecatedContractClass { bytecode: [0xdeadbeef, …], … }, class_hash_A)))`.
3. `parse_data_for_block` accepts the pair — `class_hash_A` is in `deprecated_declared_classes`, count matches.
4. `write_to_storage` calls `add_deprecated_class(class_hash_A, wrong_body)`.
5. The class manager stores `wrong_body` under `class_hash_A` with no verification.
6. A subsequent call to `get_compiled_class(class_hash_A)` returns `wrong_body`.
7. Any contract deployed with `class_hash_A` executes the attacker-supplied bytecode.

    fn get_compiled_from_class_manager(
        &self,
        class_hash: ClassHash,
    ) -> StateResult<RunnableCompiledClass> {
        if let Some(runnable_class) =
            self.contract_class_manager.get_runnable(&class_hash, &self.native_classes_whitelist)
        {
            match &runnable_class {
                RunnableCompiledClass::V0(_) => {}
                _ => {
                    // The Cairo1 class is cached; verify it is declared,
                    // since existence in the cache does not guarantee that
                    // (it might contain a declared class from a reverted block, for example).
                    if !self.state_reader.is_declared(class_hash)? {
                        return Err(StateError::UndeclaredClassHash(class_hash));
                    }
                }
            }
            self.increment_cache_hit_metric();
            self.update_native_metrics(&runnable_class);
            return Ok(runnable_class);
        }
        self.increment_cache_miss_metric();

        let compiled_class = self.state_reader.get_compiled_classes(class_hash)?;
        self.contract_class_manager.set_and_compile(class_hash, compiled_class.clone());
        // Access the cache again in case the class was compiled.
        let runnable_class = self
            .contract_class_manager
            .get_runnable(&class_hash, &self.native_classes_whitelist)
            .unwrap_or_else(|| {
                // Edge case that should not be happen if the cache size is big enough.
                // TODO(Yoni): consider having an atomic set-and-get.
                log::error!("Class is missing immediately after being cached.");
                compiled_class.to_runnable()
            });
        self.update_native_metrics(&runnable_class);
        Ok(runnable_class)
    }
