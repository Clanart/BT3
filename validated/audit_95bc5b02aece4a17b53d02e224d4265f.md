Audit Report

## Title
Router `pay()` Consumes Stranded Native ETH Without Ownership Attribution, Enabling Residue Theft - (File: metric-periphery/contracts/base/PeripheryPayments.sol)

## Summary
The `pay()` function uses `address(this).balance` to fund WETH payments without any per-transaction attribution. ETH stranded on the router from a prior user's `multicall{value: X}()` call (where `refundETH()` was omitted) is silently consumed by the next WETH swap, causing irreversible loss of the prior user's principal equal to `min(stranded_ETH, swap_value)`.

## Finding Description
In `PeripheryPayments.sol` L73–84, when `pay()` is called with `token == WETH` and `payer != address(this)`, it reads the router's entire native ETH balance and uses it to fund the WETH deposit before pulling from the payer's allowance:

```solidity
uint256 nativeBalance = address(this).balance;
if (nativeBalance >= value) {
    IWETH9(WETH).deposit{value: value}();
    IERC20(WETH).safeTransfer(recipient, value);
} else if (nativeBalance > 0) {
    IWETH9(WETH).deposit{value: nativeBalance}();
    IERC20(WETH).safeTransfer(recipient, nativeBalance);
    IERC20(WETH).safeTransferFrom(payer, recipient, value - nativeBalance);
} else {
    IERC20(WETH).safeTransferFrom(payer, recipient, value);
}
```

There is no guard limiting native ETH consumption to the current transaction's `msg.value`. ETH arrives legitimately via `multicall{value: X}()` (L39–44 of `MetricOmmSimpleRouter.sol`), which is `payable` and passes `msg.value` into the router's balance. If a user sends excess ETH and omits `refundETH()`, the surplus is stranded. The `receive()` guard at L32–34 only blocks direct `.call{value}` transfers from non-WETH addresses; it does not prevent ETH from arriving as `msg.value` in a payable multicall.

Once ETH is stranded, any subsequent caller executing a WETH swap triggers `_justPayCallback` → `pay(WETH, msg.sender, pool, value)`. Since `payer != address(this)` and `token == WETH`, the stranded ETH is deposited as WETH and transferred to the pool on behalf of the new caller. The original depositor cannot recover it via `refundETH()` because the ETH has already been converted to WETH and transferred out.

## Impact Explanation
Direct loss of user principal. User A's stranded ETH is irreversibly consumed by User B's swap. The loss equals `min(stranded_ETH, swap_value)` per consuming swap. A single stranded deposit can be drained across multiple subsequent WETH swaps. This meets the Sherlock threshold for High severity: unprivileged loss of user funds with no recovery path.

## Likelihood Explanation
The stranded-ETH precondition is reachable through normal usage. The documented pattern for native-ETH input is `multicall{value: X}([exactInputSingle(...), refundETH()])`. Omitting `refundETH()` — or sending excess ETH to ensure a swap succeeds — strands the surplus. Once stranded, any unprivileged caller executing a WETH swap triggers consumption. No special permissions, malicious setup, or non-standard tokens are required.

## Recommendation
Track the ETH balance attributable to the current transaction in transient storage: store `msg.value` at `multicall` entry and decrement it as ETH is consumed in `pay()`. In `pay()`, limit native ETH consumption to the per-transaction budget rather than `address(this).balance`. Alternatively, restrict native ETH usage in `pay()` to the case where `payer == address(this)` (mid-path hops only), and always pull WETH via `safeTransferFrom` for external payers.

## Proof of Concept
```solidity
// Step 1: User A strands ETH — sends 1 ETH, swaps only 1000 wei, omits refundETH()
router.multicall{value: 1 ether}([
    abi.encodeCall(router.exactInputSingle, ExactInputSingleParams({
        tokenIn: WETH, amountIn: 1000, /* ... */ }))
    // no refundETH() — ~1 ETH stranded on router
]);

// Step 2: User B calls a WETH swap with msg.value = 0, no WETH approval needed
router.exactInputSingle(ExactInputSingleParams({
    tokenIn: WETH, amountIn: 500, /* ... recipient: userB ... */
}));
// pay() at L75-77: nativeBalance (~1 ETH) >= 500, deposits 500 wei of User A's ETH as WETH,
// transfers to pool. User B receives token1 without spending any of their own WETH.
// User A's stranded ETH is reduced by 500 wei with no recovery path.
```
