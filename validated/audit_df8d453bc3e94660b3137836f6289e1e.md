### Title
`ProofOfInclusion::valid()` Never Validates Against a Trusted Root Hash, Enabling Forged DataLayer Inclusion Proofs - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` in `chia-datalayer` only checks that the proof's internal hash chain is self-consistent. It never compares the computed root against any externally-supplied trusted root hash. Because `root_hash()` returns `last.combined_hash` — the same value that `existing_hash` was just verified to equal inside the loop — the final check `existing_hash == self.root_hash()` is a tautology and is always `true` when the loop completes. Any attacker can construct a `ProofOfInclusion` from arbitrary bytes (the struct is `Streamable` and fully exposed to Python) that passes `valid()` without corresponding to any real DataLayer tree.

### Finding Description

`ProofOfInclusion::valid()` is defined as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()   // ← always true
}
``` [1](#0-0) 

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← same field just verified in the loop
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

After the loop, `existing_hash` equals the last `calculated_hash`, which was already asserted to equal `layer.combined_hash`. `self.root_hash()` returns that same `last.combined_hash`. The final comparison is therefore a tautology — it never rejects anything. `valid()` accepts any internally self-consistent proof chain regardless of what tree root it claims to correspond to.

The struct is `Streamable` and `PyStreamable`, fully constructible from raw bytes or via the Python `__new__` constructor, and `valid()` is exposed directly to Python: [3](#0-2) [4](#0-3) 

The Python API registers `ProofOfInclusion` as a first-class type in the `chia_rs.datalayer` submodule: [5](#0-4) 

### Impact Explanation

Any code that calls `proof.valid()` to decide whether a DataLayer key-value pair is included in a committed tree root is completely bypassable. An attacker supplies a crafted `ProofOfInclusion` whose `node_hash` encodes any key-value pair they wish to assert, with a single internally-consistent layer (choose any `other_hash`, compute `combined_hash = calculate_internal_hash(node_hash, side, other_hash)`). `valid()` returns `true`. The forged proof proves inclusion of an arbitrary key-value pair in a tree that never contained it.

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."**

### Likelihood Explanation

The attack requires only the ability to construct a `ProofOfInclusion` object — achievable by any unprivileged caller via `ProofOfInclusion.from_bytes(...)` or the Python constructor. No keys, no privileged access, no chain state are needed. The attacker only needs to know the hash function (`calculate_internal_hash`) which is public. Any system that receives a `ProofOfInclusion` from an untrusted peer and calls `.valid()` without separately checking `.root_hash()` against a trusted on-chain root is fully exploitable.

### Recommendation

`valid()` must accept a trusted root hash parameter and compare against it instead of `self.root_hash()`:

```rust
pub fn valid(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against externally-supplied root
}
```

The Python binding should be updated accordingly, and all call sites must supply the trusted root obtained from the on-chain DataLayer singleton coin state, not from the proof itself.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
from chia_rs.sized_ints import uint8

# Attacker wants to forge proof that arbitrary node_hash is in some tree.
# Pick any 32-byte node_hash (the "leaf" the attacker wants to prove).
fake_node_hash = bytes([0xAB] * 32)

# Pick any other_hash and side.
other_hash = bytes([0xCD] * 32)
side = uint8(0)  # Left

# Compute a valid combined_hash using the public hash function.
# (In practice: combined_hash = sha256(b"\x01" + left + right) per DataLayer convention)
import hashlib
left = fake_node_hash
right = other_hash
combined = hashlib.sha256(b"\x01" + left + right).digest()

layer = ProofOfInclusionLayer(
    other_hash_side=side,
    other_hash=bytes(other_hash),
    combined_hash=bytes(combined),
)

forged_proof = ProofOfInclusion(node_hash=bytes(fake_node_hash), layers=[layer])

# valid() returns True — no real tree needed.
assert forged_proof.valid() == True

# root_hash() returns the attacker-chosen combined_hash, not any real tree root.
print("Forged root:", forged_proof.root_hash().hex())
# Any verifier that only calls proof.valid() without checking
# proof.root_hash() == trusted_on_chain_root is fully bypassed.
