### Title
Missing Nonce in `compute_meta_tx_v0_hash` Enables Signature Replay Attack - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/transaction_hash/transaction_hash.cairo`)

---

### Summary

The `compute_meta_tx_v0_hash` function computes the hash for a v0 meta transaction without including any nonce or unique replay-prevention field. Because the OS-computed hash is what account contracts sign against during `meta_tx_v0` execution, any valid signature produced for a given `(contract_address, selector, calldata, chain_id)` tuple is permanently replayable by any unprivileged caller who can invoke the `meta_tx_v0` syscall from their own contract.

---

### Finding Description

`compute_meta_tx_v0_hash` in `transaction_hash.cairo` delegates to `deprecated_get_transaction_hash` with `additional_data_size=0` and `additional_data=cast(0, felt*)`:

```cairo
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(
    contract_address: felt,
    entry_point_selector: felt,
    calldata: felt*,
    calldata_size: felt,
    chain_id: felt,
) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        ...
        additional_data_size=0,          // ← no nonce
        additional_data=cast(0, felt*),
    );
    return tx_hash;
}
``` [1](#0-0) 

The resulting hash is a pure function of `(INVOKE_HASH_PREFIX, 0, contract_address, selector, calldata, 0, chain_id)`. No nonce, timestamp, or any other uniqueness token is mixed in.

In `execute_meta_tx_v0` (`syscall_impls.cairo`), this hash is placed into a freshly constructed `TxInfo` with `nonce=0` hardcoded, and the execution context is handed to the target contract's `__execute__` entry point:

```cairo
tempvar new_tx_info = new TxInfo(
    version=0,
    ...
    transaction_hash=meta_tx_hash,
    nonce=0,          // ← always zero, never checked or incremented
    ...
);
``` [2](#0-1) 

Crucially, `call_contract` explicitly **blocks** calling `__execute__` directly:

```cairo
if (request.selector == EXECUTE_ENTRY_POINT_SELECTOR) {
    write_failure_response(...ERROR_INVALID_ARGUMENT);
    return ();
}
``` [3](#0-2) 

`meta_tx_v0` is therefore the **only** protocol path that can invoke `__execute__` from within a contract. It is specifically designed for account contracts that perform signature verification inside `__execute__` (using `tx_info.transaction_hash` and `tx_info.signature_start/end`). For any such contract, the OS-computed hash is the security anchor — and that hash contains no replay-prevention field.

There is no `check_and_increment_nonce` call anywhere in the `execute_meta_tx_v0` flow, consistent with the v0 design, but unlike regular v3 outer transactions which do carry nonce protection. [4](#0-3) 

---

### Impact Explanation

**Critical — Direct loss of funds.**

An account contract that uses `meta_tx_v0` for signature-gated execution (e.g., a session-key wallet, a paymaster, or any contract that verifies a signature inside `__execute__`) signs over a hash that is permanently fixed for a given `(contract_address, calldata)` pair. Once a valid signature is broadcast on-chain, any attacker can:

- Extract the `(contract_address, calldata, signature)` triple from the observed transaction.
- Deploy a contract that issues `meta_tx_v0` with those exact parameters.
- Execute the same `__execute__` call an unlimited number of times in subsequent blocks.

If `__execute__` performs a token transfer, the attacker drains the account. If it performs any state-mutating operation, the attacker can repeat it indefinitely. The outer transaction's nonce protects only the attacker's own transaction from being replayed, not the inner meta_tx_v0 call.

---

### Likelihood Explanation

**Medium-High.** The attack requires:

1. Observing a live `meta_tx_v0` syscall (visible in any block explorer or mempool feed once the feature is used).
2. Deploying a single malicious contract that re-issues the syscall — a trivial one-time cost.

No privileged access, leaked keys, or operator cooperation is needed. Any unprivileged L2 user can execute this attack the moment a victim account uses `meta_tx_v0`.

---

### Recommendation

Include a nonce in the `compute_meta_tx_v0_hash` computation. The nonce must be:

- Unique per invocation (e.g., derived from the outer transaction's nonce and a per-call counter, or from a dedicated storage slot in the target contract that the OS reads and increments).
- Passed as `additional_data` to `deprecated_get_transaction_hash` (i.e., `additional_data_size=1`, `additional_data=&nonce`), mirroring how regular v1/v2 transactions include the nonce.

Alternatively, enforce that the OS reads and atomically increments a nonce from the target contract's state before computing the hash, analogous to `check_and_increment_nonce` used for account transactions.

---

### Proof of Concept

1. **Victim setup**: Alice deploys an account contract `A` whose `__execute__` verifies `get_execution_info().tx_info.signature` against `get_execution_info().tx_info.transaction_hash`. She authorises a one-time transfer of 1000 STRK by signing the corresponding `meta_tx_v0` hash and submitting a transaction `T_alice` that calls a trusted relay contract, which in turn calls `meta_tx_v0(contract_address=A, selector=__execute__, calldata=[transfer_1000_STRK], signature=sig_alice)`.

2. **Attacker observes**: Bob reads `T_alice` from the block. He extracts `(A, calldata, sig_alice)`.

3. **Attacker replays**: Bob deploys contract `B` with a single function that calls `meta_tx_v0(contract_address=A, selector=__execute__, calldata=[transfer_1000_STRK], signature=sig_alice)`. Bob submits `T_bob` invoking `B`.

4. **Result**: The OS computes `meta_tx_hash = compute_meta_tx_v0_hash(A, __execute__, calldata, chain_id)` — identical to the hash Alice signed — and calls `A.__execute__` with `tx_info.transaction_hash = meta_tx_hash` and `tx_info.signature = sig_alice`. Alice's `__execute__` verifies the signature successfully and executes the transfer again. Bob repeats until Alice's account is drained. [5](#0-4) [6](#0-5)

    range_check_ptr,
    syscall_ptr: felt*,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    revert_log: RevertLogEntry*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, caller_execution_context: ExecutionContext*) {
    alloc_locals;

    let request = cast(syscall_ptr + RequestHeader.SIZE, MetaTxV0Request*);
    local calldata_start: felt* = request.calldata_start;
    local calldata_size = request.calldata_end - calldata_start;

    let specific_base_gas_cost = (
        META_TX_V0_GAS_COST + META_TX_V0_CALLDATA_FACTOR_GAS_COST * calldata_size
    );
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=specific_base_gas_cost, request_struct_size=MetaTxV0Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    local contract_address = request.contract_address;
    local selector = request.selector;
    local caller_execution_info: ExecutionInfo* = caller_execution_context.execution_info;
    local old_tx_info: TxInfo* = caller_execution_info.tx_info;

    if (selector != EXECUTE_ENTRY_POINT_SELECTOR) {
        write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
        return ();
    }

    // Sanity check: Verify that `signature` is a valid Sierra array.
    assert_nn_le(request.signature_end - request.signature_start, SIERRA_ARRAY_LEN_BOUND - 1);

    let (state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
        key=contract_address
    );

    // Compute the meta-transaction hash.
    let pedersen_ptr = builtin_ptrs.selectable.pedersen;
    with pedersen_ptr {
        let meta_tx_hash = compute_meta_tx_v0_hash(
            contract_address=contract_address,
            entry_point_selector=selector,
            calldata=calldata_start,
            calldata_size=calldata_size,
            chain_id=old_tx_info.chain_id,
        );
    }
    update_pedersen_in_builtin_ptrs(pedersen_ptr=pedersen_ptr);

    // Prepare execution context.
    tempvar new_tx_info = new TxInfo(
        version=0,
        account_contract_address=contract_address,
        max_fee=0,
        signature_start=request.signature_start,
        signature_end=request.signature_end,
        transaction_hash=meta_tx_hash,
        chain_id=old_tx_info.chain_id,
        nonce=0,
        resource_bounds_start=cast(0, ResourceBounds*),
        resource_bounds_end=cast(0, ResourceBounds*),
        tip=0,
        paymaster_data_start=cast(0, felt*),
        paymaster_data_end=cast(0, felt*),
        nonce_data_availability_mode=0,
        fee_data_availability_mode=0,
        account_deployment_data_start=cast(0, felt*),
        account_deployment_data_end=cast(0, felt*),
        proof_facts_start=cast(0, felt*),
        proof_facts_end=cast(0, felt*),
    );

    let (deprecated_tx_info_ptr: DeprecatedTxInfo*) = alloc();
    tempvar execution_context: ExecutionContext* = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=state_entry.class_hash,
        calldata_size=calldata_size,
        calldata=calldata_start,
        execution_info=new ExecutionInfo(
            block_info=caller_execution_info.block_info,
            tx_info=new_tx_info,
            caller_address=ORIGIN_ADDRESS,
            contract_address=contract_address,
            selector=selector,
        ),
        deprecated_tx_info=deprecated_tx_info_ptr,
    );
    fill_deprecated_tx_info(tx_info=new_tx_info, dst=execution_context.deprecated_tx_info);

    // Since we process the revert log backwards, entries before this point belong to the calling
    // contract.
    assert [revert_log] = RevertLogEntry(
        selector=CHANGE_CONTRACT_ENTRY, value=caller_execution_info.contract_address
    );
    let revert_log = &revert_log[1];

    contract_call_helper(
        remaining_gas=remaining_gas,
        block_context=block_context,
        execution_context=execution_context,
    );

    // Entries before this point belong to the callee.
    assert [revert_log] = RevertLogEntry(selector=CHANGE_CONTRACT_ENTRY, value=contract_address);
    let revert_log = &revert_log[1];

    return ();
