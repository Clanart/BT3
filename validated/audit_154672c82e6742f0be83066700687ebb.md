### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Fund Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary felt value as a new class hash without verifying that the hash corresponds to an actually-declared class. A contract can replace its own class with a hash that was never declared, permanently making the contract uncallable and freezing any funds it holds. The missing check is explicitly acknowledged by a TODO comment in the code.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The critical line is the TODO comment at line 898: `// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.` The OS unconditionally writes the caller-supplied `class_hash` into the contract's `StateEntry` in `contract_state_changes` with no membership check against `contract_class_changes` (the declared class registry).

This is the direct analog of the original report's bug: just as `Tap.updateController()` allowed replacing a critical component reference without enforcing that the new address was valid/consistent, `execute_replace_class` allows replacing the contract's class reference (its execution logic) with an arbitrary, undeclared hash.

The `contract_class_changes` dict, which tracks declared classes, is a separate implicit argument that `execute_replace_class` does not receive or consult: [2](#0-1) 

Compare with `execute_storage_write` or `execute_deploy`, which both receive `contract_class_changes` when they need to interact with class state. `execute_replace_class` does not.

The `StateEntry` struct stores `class_hash` as a plain `felt` with no on-chain validation: [3](#0-2) 

The state commitment (`compute_contract_state_commitment`) then hashes this arbitrary class hash into the global Merkle root, permanently committing the invalid state: [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` is set to an undeclared hash and committed to the global state root, every future call to that contract will fail at class lookup time (the class bytecode does not exist). Any ETH/ERC-20 tokens or other assets held in the contract's storage become permanently inaccessible. The state root is cryptographically committed, so there is no rollback path at the protocol level.

If the affected contract is a widely-used account contract or a DeFi vault, the frozen funds could be substantial.

---

### Likelihood Explanation

**High.** The `replace_class` syscall is a standard, permissionless syscall available to any Cairo contract. No special role, key, or privilege is required. An attacker simply:

1. Deploys any contract (or exploits a bug in an existing contract that calls `replace_class`).
2. Has the contract invoke `replace_class` with a felt value that is not a declared class hash (e.g., `1` or any random felt).
3. The OS accepts the syscall, writes the invalid class hash to state, and commits it.

The attacker does not need operator access, a leaked key, or any external dependency. The entry path is a standard user-submitted invoke transaction.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that the requested `class_hash` exists in `contract_class_changes` (i.e., maps to a non-zero compiled class hash). Concretely:

- Add `contract_class_changes: DictAccess*` as an implicit argument to `execute_replace_class`.
- Perform a `dict_read` on `contract_class_changes` keyed by `request.class_hash`.
- Assert the returned compiled class hash is non-zero before proceeding with the state update.

This mirrors how `execute_declare_transaction` enforces `assert_not_zero(compiled_class_hash)` before writing to `contract_class_changes`: [5](#0-4) 

---

### Proof of Concept

1. Attacker deploys contract `C` holding 1000 STRK.
2. `C`'s constructor or any external entry point calls `replace_class(class_hash=0xdeadbeef)` where `0xdeadbeef` is never declared.
3. The OS executes `execute_replace_class`:
   - Reads `request.class_hash = 0xdeadbeef`.
   - Skips the missing declared-class check (TODO at line 898).
   - Writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes`.
4. `state_update` squashes and commits this entry into the global Merkle root via `compute_contract_state_commitment`.
5. All future invoke transactions targeting `C` fail at class resolution — the 1000 STRK are permanently frozen. [6](#0-5) [7](#0-6)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
