### Title
Missing Declared Class Validation in `execute_replace_class` Enables Permanent Fund Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the replacement class hash corresponds to a previously declared contract class. This allows any contract to replace its class with an arbitrary, undeclared felt value, rendering the contract permanently inaccessible and freezing any funds it holds. The missing check is explicitly acknowledged by a developer TODO comment in the production code.

---

### Finding Description

In `syscall_impls.cairo`, the function `execute_replace_class` processes the `replace_class` syscall. After deducting gas, it reads `request.class_hash` and directly updates the contract's `StateEntry` in `contract_state_changes` with no validation that the new class hash exists in `contract_class_changes` (i.e., has ever been declared).

The critical section:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO at line 898 explicitly acknowledges the missing invariant enforcement. Because the OS imposes no constraint on `class_hash`, a contract can call `replace_class` with any arbitrary felt value. If that value does not correspond to a declared class, the execution engine will be unable to load the contract's code for any future call, making the contract permanently non-functional. [2](#0-1) 

The analogous invariant is enforced for `declare` transactions: `dict_update` uses `prev_value=0` to enforce that a class may be declared only once, and `assert_not_zero(compiled_class_hash)` rejects zero hashes. No equivalent guard exists for `replace_class`. [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any funds (ETH, STRK, or ERC-20 tokens) held by a contract whose class is replaced with an undeclared hash are permanently frozen. There is no recovery path: the contract cannot execute any function (including withdrawal functions) because its class cannot be resolved by the execution engine. The state transition is committed to the global state root via `compute_contract_state_commitment`, making it irreversible on-chain. [4](#0-3) 

---

### Likelihood Explanation

The attack is reachable by an unprivileged user in two concrete ways:

1. **Direct path**: A contract deployer deploys a contract with an upgrade function that calls `replace_class` without access control (e.g., `fn upgrade(new_class: felt) { replace_class(new_class); }`). If other users deposit funds into this contract, the deployer — or any attacker who can call the unguarded function — can supply an undeclared hash and permanently freeze those funds.

2. **Exploit path**: Any existing deployed contract that exposes an upgrade entrypoint without proper caller validation is vulnerable. An attacker submits a transaction calling that entrypoint with an arbitrary undeclared felt as the class hash. The OS processes the `replace_class` syscall, updates the state, and the contract is permanently broken.

The OS-level missing validation is the root cause: the OS should reject `replace_class` calls with undeclared hashes at the protocol layer, providing a safety net regardless of application-level access control. Without it, a single missing `assert_caller_is_owner` in any contract's upgrade function is sufficient for permanent fund loss.

---

### Recommendation

Before updating the contract's `StateEntry`, verify that `request.class_hash` is present as a key in `contract_class_changes` with a non-zero `new_value` (i.e., it has been declared in the current block or a prior block). If the class hash is not declared, write a failure response via `write_failure_response` instead of updating the state. This mirrors the invariant already enforced for `declare` transactions.

---

### Proof of Concept

1. Deploy **ContractA** with a public function `upgrade(new_class: felt)` that calls `replace_class(new_class)` with no access control. ContractA holds user-deposited funds.
2. An attacker calls `ContractA.upgrade(0xdeadbeef)` where `0xdeadbeef` is an arbitrary felt value that has never been declared.
3. The OS processes the `replace_class` syscall inside `execute_replace_class`. It reads `class_hash = 0xdeadbeef` and calls `dict_update` to set ContractA's `StateEntry.class_hash = 0xdeadbeef` — with no check against `contract_class_changes`.
4. The state transition is squashed and committed via `state_update` → `compute_contract_state_commitment`. ContractA's class hash in the global state root is now `0xdeadbeef`. [5](#0-4) 

5. Any subsequent call to ContractA fails: the execution engine looks up class hash `0xdeadbeef`, finds no corresponding compiled class, and the call reverts.
6. All funds in ContractA are permanently frozen with no recovery mechanism.

**Analogy to the external report**: Just as `TokenFactory1155.createToken()` accepted any `IRoyalties` contract without validation — granting the caller `ROYALTIES_ROLE` and enabling arbitrary manipulation of royalties accounting — `execute_replace_class` accepts any class hash without validation, enabling permanent corruption of a contract's executable state and the irreversible freezing of its funds.

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    let revert_log = &revert_log[1];

    return ();
}
