### Title
Unverified Hint-Provided Block Hash Written to Block Hash Contract Without Cairo Constraint Enforcement - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/os_utils.cairo`)

---

### Summary

The `write_block_number_to_block_hash_mapping` function in `os_utils.cairo` writes a block hash value to the reserved block hash contract (address `0x1`) that is sourced entirely from an unverified hint (`%{ GetOldBlockNumberAndHash %}`). No Cairo constraint verifies that this hint-provided value equals the actual historical block hash. The OS itself acknowledges this with an explicit comment: *"Currently, the block hash mapping is not enforced by the OS."* A malicious prover can supply an arbitrary value for `old_block_hash`, generate a proof that satisfies all remaining Cairo constraints, and have that proof accepted — causing every subsequent `get_block_hash` syscall to return a fabricated value to any contract that queries it.

---

### Finding Description

In `write_block_number_to_block_hash_mapping`:

```cairo
// Currently, the block hash mapping is not enforced by the OS.
// TODO(Yoni, 1/1/2026): output this hash.
local old_block_hash;
%{ GetOldBlockNumberAndHash %}

...
assert [storage_ptr] = DictAccess(key=old_block_number, prev_value=0, new_value=old_block_hash);
``` [1](#0-0) 

`old_block_hash` is declared as a `local` variable and populated exclusively by the hint `%{ GetOldBlockNumberAndHash %}`. In Cairo's proving model, hints are prover-supplied witnesses that are **not** part of the constraint system — they carry no cryptographic guarantee. The only Cairo assertion present is `prev_value=0`, which enforces that the slot was previously unwritten, but places **no constraint** on `new_value=old_block_hash`. Any felt value satisfies the proof.

This value is then committed to the `contract_state_changes` dictionary under `BLOCK_HASH_CONTRACT_ADDRESS = 0x1`: [2](#0-1) 

The `execute_get_block_hash` syscall in `syscall_impls.cairo` subsequently reads from this same storage slot via `read_block_hash_from_storage`, asserting only that the stored value equals the response value — it does not independently verify the hash:

```cairo
assert [storage_ptr] = DictAccess(
    key=block_number, prev_value=expected_block_hash, new_value=expected_block_hash
);
``` [3](#0-2) 

The `OsOutputHeader` struct in `output.cairo` also carries a `new_block_hash` field with the comment *"Currently, the block hash is not enforced by the OS"*: [4](#0-3) 

Similarly, in `get_block_hashes` in `block_hash.cairo`, `previous_block_hash`, `header_commitments`, `gas_prices_hash`, and `starknet_version` are all hint-provided locals with no Cairo constraint verifying their correctness. The only "check" is `%{ CheckBlockHashConsistency %}`, which is itself a hint and therefore not part of the proof: [5](#0-4) 

The `get_block_os_output_header` function in `os_utils.cairo` explicitly documents that the previous block hash and previous state root are guessed and unverified by the OS: [6](#0-5)

### Impact Explanation

**Direct loss of funds (Critical).**

Any on-chain contract that calls the `get_block_hash` syscall and uses the returned value for a security-sensitive purpose — such as pseudo-random number generation, replay-attack prevention, or cross-contract commitment verification — will receive a prover-controlled arbitrary value instead of the true historical block hash. A malicious prover can:

1. Choose a target contract that uses `get_block_hash` for randomness or access control.
2. Precompute which `old_block_hash` value causes the target contract to behave favorably (e.g., win a lottery, bypass a guard).
3. Supply that value via the `GetOldBlockNumberAndHash` hint.
4. Generate a valid OS proof — all Cairo constraints are satisfied because `new_value` is unconstrained.
5. Submit the proof; the L1 verifier accepts it.
6. The block hash contract now stores the fabricated value.
7. The target contract's `get_block_hash` call returns the fabricated value, enabling the exploit.

The `STORED_BLOCK_HASH_BUFFER = 10` constant means the manipulated entry covers the block 10 blocks prior to the current one, which is exactly the range accessible to contracts via `get_block_hash`. [7](#0-6) 

---

### Likelihood Explanation

The codebase explicitly supports client-side proving via the virtual OS path (`ALLOWED_VIRTUAL_OS_PROGRAM_HASHES_0` in `constants.cairo`), meaning the prover role is not permanently restricted to a single trusted operator. [8](#0-7) 

Even in the centralized sequencer model, the sequencer controls hint execution. The developers have acknowledged the gap with a TODO dated `1/1/2026`, confirming the constraint is missing and the fix is deferred. Any prover — current or future — who controls hint execution can exploit this without any additional privilege beyond the ability to generate a proof.

---

### Recommendation

Replace the unconstrained hint-provided `old_block_hash` with a value that is cryptographically committed to the OS output and verified against the chain's canonical block hash sequence. Concretely:

1. **Output the block hash**: As the TODO notes, include `old_block_hash` in the OS output segment so the L1 verifier contract can cross-check it against the on-chain block hash registry.
2. **Add a Cairo assertion**: After computing the new block hash via `calculate_block_hash`, assert that `previous_block_hash` equals the value stored in the block hash contract for `block_number - 1`, making the chain linkage a provable constraint rather than a hint.
3. **Enforce `header_commitments` and `gas_prices_hash`**: Remove the TODO deferral in `get_block_hashes`; compute these values inside the OS rather than accepting them from hints, so the `new_block_hash` is fully determined by verified inputs.

---

### Proof of Concept

1. A prover targets a deployed lottery contract `L` that calls `get_block_hash(N - 10)` and uses the result modulo some number to pick a winner.
2. The prover precomputes a value `X` such that `X mod M == winning_index`.
3. When generating the OS proof for block `N`, the prover supplies `old_block_hash = X` via the `GetOldBlockNumberAndHash` hint in `write_block_number_to_block_hash_mapping`.
4. The Cairo constraint `assert [storage_ptr] = DictAccess(key=N-10, prev_value=0, new_value=X)` is satisfied because `prev_value=0` holds (first write) and `new_value` is unconstrained.
5. The proof is valid; the L1 verifier accepts it.
6. Block hash contract at address `0x1` now maps `N-10 → X`.
7. Contract `L` calls `get_block_hash(N-10)`, receives `X`, and the prover-controlled account wins the lottery, draining funds. [9](#0-8) [10](#0-9)

// Gets the block hash of the block at given block number.
func execute_get_block_hash{
    range_check_ptr, syscall_ptr: felt*, contract_state_changes: DictAccess*
}(block_context: BlockContext*) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, GetBlockHashRequest*);

    // Reduce gas.
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=GET_BLOCK_HASH_GAS_COST, request_struct_size=GetBlockHashRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall; in that case, 'reduce_syscall_base_gas' already
        // wrote the response objects and advanced the syscall pointer.
        return ();
    }

    // Handle out of range block number.
    let request_block_number = request.block_number;
    let current_block_number = block_context.block_info_for_execute.block_number;

    // A block number is a u64. STORED_BLOCK_HASH_BUFFER is 10.
    // The following computations will not overflow.
    local is_block_number_in_block_hash_buffer;
    %{ IsBlockNumberInBlockHashBuffer %}
    if (is_block_number_in_block_hash_buffer != FALSE) {
        assert_lt(current_block_number, request_block_number + STORED_BLOCK_HASH_BUFFER);
        write_failure_response(
            remaining_gas=remaining_gas, failure_felt=ERROR_BLOCK_NUMBER_OUT_OF_RANGE
        );
        return ();
    }

    assert_nn_le(request_block_number, current_block_number - STORED_BLOCK_HASH_BUFFER);

    // Gas reduction has succeeded and the request is valid; write the response header.
    let response_header = cast(syscall_ptr, ResponseHeader*);
    let syscall_ptr = syscall_ptr + ResponseHeader.SIZE;
    assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=0);

    let response = cast(syscall_ptr, GetBlockHashResponse*);
    // Advance syscall pointer to the next syscall.
    let syscall_ptr = syscall_ptr + GetBlockHashResponse.SIZE;

    let block_hash = response.block_hash;

    read_block_hash_from_storage(block_number=request_block_number, expected_block_hash=block_hash);

    return ();
