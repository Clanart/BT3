### Title
Missing Declared Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not validate that the new class hash supplied by a contract is actually a declared class. Any unprivileged contract can call `replace_class` with an arbitrary, undeclared class hash. The OS will commit this invalid class hash to the contract state without complaint. Any subsequent call to that contract will fail irrecoverably at the OS level because the class hash cannot be resolved to a compiled class, permanently freezing all funds held by the contract.

---

### Finding Description

In `execute_replace_class`, the OS reads the requested class hash directly from the syscall request and writes it into `contract_state_changes` with no check that the hash corresponds to a previously declared class:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The TODO comment at line 898 is an explicit acknowledgment by the developers that this validation is missing. [2](#0-1) 

The downstream consequence is in `execute_entry_point`, which resolves a contract's class hash to a compiled class by performing a `find_element` lookup in the `compiled_class_facts_bundle`. If the class hash stored in state does not correspond to any declared class, `dict_read` on `contract_class_changes` returns `0` (the default for an undeclared key), and `find_element` will fail to locate a matching `CompiledClassFact`, making it impossible to generate a valid proof for any future call to that contract:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is overwritten with an undeclared value, no valid STARK proof can ever be produced for a transaction that calls that contract. The contract becomes permanently uncallable at the protocol level. Any ERC-20 tokens, ETH, or other assets held in the contract's storage are irrecoverably frozen. There is no upgrade path or recovery mechanism available to users of the bricked contract.

---

### Likelihood Explanation

The `replace_class` syscall is available to any Sierra contract. An unprivileged user needs only to:

1. Deploy a contract (or control an existing one) that invokes `replace_class` with an arbitrary felt value as the class hash.
2. Submit a transaction that triggers that code path.

No privileged role, leaked key, or operator cooperation is required. The attack is a single transaction and is fully deterministic.

---

### Recommendation

Before committing the new class hash to `contract_state_changes`, add an explicit check that the hash exists in `contract_class_changes` (i.e., it has been declared in the current or a prior block). The check should mirror the lookup already performed in `execute_entry_point`:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This ensures the OS enforces the same invariant at write-time that it implicitly requires at read-time, closing the gap identified in the TODO comment.

---

### Proof of Concept

1. Declare class `A` (a valid Sierra class). Deploy contract `C` using class `A`. Fund `C` with tokens.
2. Submit an invoke transaction that causes `C` to call the `replace_class` syscall with `class_hash = 0xDEAD` (an arbitrary undeclared felt).
3. The OS executes `execute_replace_class`, writes `class_hash=0xDEAD` into `contract_state_changes[C]`, and returns success — no validation occurs.
4. The block is proven and settled on L1 with `C`'s class hash now set to `0xDEAD`.
5. Submit any subsequent transaction calling `C`. The OS executes `execute_entry_point`, calls `dict_read{dict_ptr=contract_class_changes}(key=0xDEAD)` → returns `0`, then calls `find_element(..., key=0)` → assertion failure. No valid proof can be generated. The funds in `C` are permanently frozen.
