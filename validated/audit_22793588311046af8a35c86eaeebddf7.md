### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Bricking — (`File: execution/syscall_impls.cairo`)

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the replacement class hash corresponds to a previously declared contract class. An explicit `TODO` comment acknowledges this missing check. A contract can therefore set its own class hash to an arbitrary undeclared value. Any subsequent call to that contract will cause the OS to attempt to look up a compiled class that does not exist, producing an irrecoverable Cairo assertion failure that prevents the block from being proven, or permanently freezing any funds held by the contract.

### Finding Description

In `execute_replace_class` (lines 878–916 of `syscall_impls.cairo`), the OS reads the requested new class hash directly from the syscall request and writes it into `contract_state_changes` without any check that the hash exists in `contract_class_changes` (the dictionary of declared class → compiled class mappings):

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The state update is committed unconditionally. When the contract is subsequently called, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // now the invalid hash
);
// compiled_class_hash == 0 (default for undeclared class)

let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,           // searching for 0 — not present
);
``` [2](#0-1) 

`find_element` is a Cairo standard-library function that **asserts** the key is present. If the key is absent, the Cairo VM raises an irrecoverable assertion failure. The OS proof for that block cannot be generated.

### Impact Explanation

Two concrete impacts exist:

1. **Critical — Permanent freezing of funds.** Any ERC-20 token balance, NFT, or vault balance held by a contract whose class hash has been replaced with an undeclared value is permanently inaccessible. No withdrawal, transfer, or recovery call can succeed because every entry-point dispatch fails.

2. **High — Network halt.** If the sequencer includes a subsequent call to the bricked contract in a block, the OS Cairo execution fails with an assertion error during `find_element`. The block cannot be proven, and the network cannot advance until the sequencer is patched to skip or revert such transactions.

### Likelihood Explanation

The attack path is reachable by any unprivileged contract deployer:

1. Deploy a contract whose logic calls `replace_class(arbitrary_undeclared_hash)` — either directly or via a user-supplied argument.
2. Invoke the contract. The OS accepts the syscall and commits the invalid class hash to state (no validation gate exists in the OS Cairo code).
3. Any subsequent call to the contract (same block or future block) triggers the fatal `find_element` assertion.

The TODO comment at line 898 confirms the development team is aware the check is absent. No privileged role or leaked key is required; a standard `INVOKE` transaction suffices.

### Recommendation

Inside `execute_replace_class`, before updating `contract_state_changes`, assert that the requested class hash is present in `contract_class_changes`:

```cairo
// Verify the new class hash is a declared class.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the validation already performed implicitly in `execute_entry_point` and makes the OS the authoritative enforcement point, consistent with the defense-in-depth model.

### Proof of Concept

1. Declare and deploy contract `A` that exposes: `fn brick_self(new_hash: felt) { replace_class(new_hash); }`.
2. Send funds (e.g., ERC-20 tokens) to contract `A`.
3. Call `brick_self(0xdeadbeef)` where `0xdeadbeef` is never declared. The OS writes `class_hash=0xdeadbeef` into `contract_state_changes` with no error.
4. In the same or next block, call any entry point of contract `A`. The OS executes `dict_read(key=0xdeadbeef)` → returns `0`; then `find_element(..., key=0)` → Cairo assertion failure.
5. The block proof fails (network halt path) **or** the funds in contract `A` are permanently frozen (fund-freeze path), depending on sequencer behavior.
