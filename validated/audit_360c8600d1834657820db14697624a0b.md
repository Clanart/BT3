### Title
`ProofOfInclusion::valid()` Validates Against Its Own Claimed Root, Not a Trusted External Root — Forged DataLayer Inclusion Proofs Always Pass - (File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs)

### Summary

`ProofOfInclusion::valid()` in `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs` only checks the internal self-consistency of the proof chain. Its final assertion compares `existing_hash` against `self.root_hash()`, but `root_hash()` is derived entirely from the proof's own last layer (`last.combined_hash`). Because the loop already enforces that `existing_hash` equals `last.combined_hash` before the final comparison, the final check is a tautology — it is always `true` when the loop completes. An attacker can craft any internally consistent `ProofOfInclusion` for any arbitrary `node_hash` (i.e., any fake key/value pair), and `valid()` will return `true` without ever comparing against a trusted external root. Any DataLayer consumer that calls only `proof.valid()` — the sole public validation API — accepts forged inclusion proofs.

### Finding Description

**Root cause — tautological final check in `valid()`:**

```rust
// crates/chia-datalayer/src/merkle/proof_of_inclusion.rs

pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // <-- taken from the proof itself
    } else {
        self.node_hash
    }
}

pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;   // existing_hash == layer.combined_hash here
    }

    existing_hash == self.root_hash()      // always true: both sides == last layer.combined_hash
}
```

After the loop, `existing_hash` equals the last `layer.combined_hash` (the loop would have returned `false` otherwise). `self.root_hash()` also returns `self.layers.last().combined_hash` — the same value. The final comparison is therefore `last_layer.combined_hash == last_layer.combined_hash`, which is unconditionally `true`.

**What `valid()` actually checks:** only that each layer's `combined_hash` is consistent with the hash computed from the previous layer and `other_hash`. It never compares the computed root against any externally trusted root.

**Analog to the SAM vulnerability:** In SAM, `onlyBeforeMintConcluded` relied on the external (attacker-controlled) contract's `mintConcluded()` return value rather than an internally tracked flag. Here, `valid()` relies on the proof's own `combined_hash` field (attacker-controlled) rather than an externally committed root. In both cases, the guard check is satisfied by untrusted data supplied by the attacker.

**Exploit path:**

1. Attacker constructs a `ProofOfInclusion` with `node_hash = H(fake_key || fake_value)` and a chain of layers whose `combined_hash` values are computed consistently (trivially done by computing forward from any chosen `other_hash` values).
2. The struct is `Streamable` and exposed via Python bindings; it can be serialized and sent over the DataLayer peer-to-peer sync protocol.
3. A DataLayer client receives the serialized proof, deserializes it, and calls `proof.valid()` — the only public validation method.
4. `valid()` returns `true`. The client accepts that `fake_key` is present in the tree.
5. The attacker has proven inclusion of a key that does not exist in the actual committed tree.

**Entry point:** Any unprivileged peer participating in DataLayer sync can send a crafted serialized `ProofOfInclusion`. The `ProofOfInclusion` struct derives `Streamable`, making it directly deserializable from network bytes.

### Impact Explanation

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."**

A DataLayer client that calls only `proof.valid()` — the sole public validation API — will accept any internally consistent proof for any claimed key/value pair, regardless of whether that pair exists in the actual committed tree. This allows an attacker to deceive DataLayer consumers into accepting false state, undermining the integrity guarantees of the DataLayer.

### Likelihood Explanation

The `valid()` method is the only public proof-validation API exposed by `ProofOfInclusion`. The Python bindings expose it directly, and all internal tests call only `proof.valid()` without a separate root check. There is no documentation or API-level mechanism that forces callers to also compare `proof.root_hash()` against a trusted external root. Any DataLayer sync consumer that follows the natural API usage pattern is vulnerable.

### Recommendation

The `valid()` method must accept a trusted external root and compare the computed root against it:

```rust
pub fn valid_for_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against externally trusted root
}
```

The no-argument `valid()` should either be removed or renamed to `is_internally_consistent()` to make clear it does not validate against any committed root. All DataLayer sync consumers must be updated to pass the on-chain committed root.

### Proof of Concept

```rust
use chia_datalayer::{Hash, Side};
use chia_datalayer::merkle::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};

fn forge_proof_of_inclusion() {
    // Attacker wants to prove that fake_node_hash is in the tree.
    let fake_node_hash = Hash::from([0xAA; 32]);
    let fake_other_hash = Hash::from([0xBB; 32]);

    // Compute a consistent combined_hash for one layer.
    let combined = chia_datalayer::calculate_internal_hash(
        &fake_node_hash,
        Side::Left,
        &fake_other_hash,
    );

    let forged_proof = ProofOfInclusion {
        node_hash: fake_node_hash,
        layers: vec![ProofOfInclusionLayer {
            other_hash_side: Side::Left,
            other_hash: fake_other_hash,
            combined_hash: combined,   // attacker sets this to match
        }],
    };

    // valid() returns true even though fake_node_hash is not in any real tree.
    assert!(forged_proof.valid());
    // root_hash() returns the attacker-chosen combined value, not the real tree root.
    assert_eq!(forged_proof.root_hash(), combined);
}
```
