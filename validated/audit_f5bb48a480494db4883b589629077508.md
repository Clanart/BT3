### Title
Missing `chunk.shard_id() == shard_id` Guard in `set_state_header` Allows Cross-Shard State Root Substitution — (`chain/chain/src/state_sync/adapter.rs`)

---

### Summary

`ChainStateSyncAdapter::set_state_header` accepts a peer-supplied `ShardStateSyncResponseHeader` for shard A but never asserts that the embedded `ShardChunk` actually belongs to shard A. An unprivileged peer can substitute shard B's on-chain chunk (with its valid Merkle proof at shard B's position) and pass every validation gate. The corrupted header is then persisted under `StateHeaderKey(A, H)`, causing all subsequent `set_state_part` calls to validate parts against shard B's `prev_state_root`, and `set_state_finalize` to apply shard B's transactions and state to shard A's `shard_uid`.

---

### Finding Description

**Production entrypoint:** `ChainStateSyncAdapter::set_state_header` in `chain/chain/src/state_sync/adapter.rs`.

The function receives `shard_id: ShardId` as the target shard and a peer-supplied `shard_state_header: ShardStateSyncResponseHeader`. It extracts the embedded chunk:

```rust
let chunk = shard_state_header.cloned_chunk();
``` [1](#0-0) 

**Gate 1 — `validate_chunk_proofs`:** Checks hash consistency, `tx_root`, and `outgoing_receipts_root` of the chunk body. It never inspects `chunk.shard_id()` relative to the `shard_id` parameter. [2](#0-1) 

**Gate 2 — `verify_path` for chunk inclusion:** Verifies that `ChunkHashHeight(chunk.chunk_hash(), chunk.height_included())` hashes to `sync_prev_block_header.chunk_headers_root()` via the supplied `chunk_proof`. The implementation is:

```rust
pub fn verify_path<T: BorshSerialize>(root: MerkleHash, path: &MerklePath, item: T) -> bool {
    verify_hash(root, path, CryptoHash::hash_borsh(item))
}
pub fn verify_hash(root: MerkleHash, path: &MerklePath, item_hash: MerkleHash) -> bool {
    compute_root_from_path(path, item_hash) == root
}
``` [3](#0-2) 

`verify_path` only checks that the leaf reaches the root via the given path. **It does not check which leaf index (shard position) the path encodes.** A proof for shard B's chunk at shard B's position in the tree is structurally valid and passes this check even when the caller expects shard A. [4](#0-3) 

**Gate 3 — `prev_chunk_header` / `prev_chunk_proof`:** Same `verify_path` call, same absence of position check. [5](#0-4) 

**Gate 4 — incoming receipts:** The receipts hash is computed as `ReceiptList(shard_id, receipts)` where `shard_id` is the parameter (shard A). The attacker supplies valid incoming receipts for shard A (publicly available from the chain), so these checks pass independently of which chunk is embedded. [6](#0-5) 

**Gate 5 — `validate_state_root_node`:** Validates `state_root_node` against `chunk_inner.prev_state_root()`. Since `chunk_inner` is shard B's chunk, `prev_state_root()` is shard B's state root. The attacker supplies shard B's `state_root_node`, which is consistent with shard B's state root. This check passes. [7](#0-6) 

**Storage:** The header is persisted under `StateHeaderKey(shard_id=A, sync_hash=H)` but contains shard B's chunk. The invariant `StateHeaderKey(A, H).chunk.shard_id() == A` is violated. [8](#0-7) 

---

### Impact Explanation

**`set_state_part` corruption:** When validating state parts for shard A, the code reads the stored header, extracts `prev_state_root` from the embedded chunk (shard B's state root), and validates parts against it:

```rust
let state_root = *chunk.take_header().take_inner().prev_state_root();
self.runtime_adapter.validate_state_part(shard_id, &state_root, part_id, part)
``` [9](#0-8) 

The attacker can supply state parts for shard B's trie, which pass `validate_state_part` because the state root matches. Legitimate parts for shard A's trie are rejected.

**`set_state_finalize` corruption:** The finalization step applies the embedded chunk using `shard_uid` derived from the `shard_id` parameter (shard A), but uses `chunk_header.prev_state_root()` (shard B's state root) and shard B's transactions:

```rust
let shard_uid = shard_id_to_uid(self.epoch_manager.as_ref(), shard_id, block_header.epoch_id())?;
let apply_result = self.runtime_adapter.apply_chunk(
    RuntimeStorageConfig::new(chunk_header.prev_state_root(), true),
    ...
    ApplyChunkShardContext { shard_uid, ... },
    ...
    transactions,  // shard B's transactions
)?;
``` [10](#0-9) 

This applies shard B's state and transactions to shard A's `shard_uid`, producing a permanently corrupted post-state for shard A.

---

### Likelihood Explanation

The attack requires only that the adversary be a peer reachable by the syncing node. All inputs needed — shard B's on-chain chunk, its Merkle proof at shard B's position in `chunk_headers_root`, shard B's `prev_chunk_header` and proof, shard A's incoming receipts proofs, and shard B's `state_root_node` — are publicly available from the chain. No validator, chunk producer, or operator privileges are required. The syncing node actively solicits state headers from peers, so the attacker does not need to initiate contact.

---

### Recommendation

Add an explicit shard identity check immediately after extracting the chunk in `set_state_header`:

```rust
let chunk = shard_state_header.cloned_chunk();
if chunk.shard_id() != shard_id {
    return Err(Error::Other(
        "set_shard_state failed: chunk.shard_id() does not match requested shard_id".into(),
    ));
}
``` [11](#0-10) 

A parallel check should be added for `prev_chunk_header.shard_id()` when present.

### Proof of Concept

Construct a `ShardStateSyncResponseHeader::V2` where:
- `chunk` = the on-chain `ShardChunk` for shard B at `sync_prev_block` (valid signature, valid internal proofs)
- `chunk_proof` = the Merkle path for shard B's `ChunkHashHeight` in `sync_prev_block.chunk_headers_root` (at shard B's index)
- `prev_chunk_header` = shard B's previous chunk header
- `prev_chunk_proof` = its Merkle path in the preceding block's `chunk_headers_root`
- `incoming_receipts_proofs` = valid incoming receipts for shard A (copied from a legitimate header)
- `root_proofs` = corresponding root proofs for shard A's receipts
- `state_root_node` = shard B's `StateRootNode` (consistent with shard B's `prev_state_root`)

Send this as the response to a `StateRequestHeader { shard_id: A, sync_hash: H }` request. `set_state_header(A, H, header)` will accept it and write it to `DBCol::StateHeaders` under key `StateHeaderKey(A, H)`. Subsequent `set_state_part(A, H, ...)` calls will validate parts against shard B's state root, and `set_state_finalize` will apply shard B's state to shard A.

A property test in `chain/chain/src/state_sync/adapter.rs` should generate headers with `chunk.shard_id() != shard_id` and assert `set_state_header` returns an error.

pub fn validate_chunk_proofs(
    chunk: &ShardChunk,
    epoch_manager: &dyn EpochManagerAdapter,
) -> Result<bool, Error> {
    let correct_chunk_hash = chunk.compute_header_hash();

    // 1. Checking chunk.header.hash
    let header_hash = chunk.header_hash();
    if header_hash != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }

    // 2. Checking that chunk body is valid
    // 2a. Checking chunk hash
    if chunk.chunk_hash() != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }
    let height_created = chunk.height_created();
    let outgoing_receipts_root = chunk.prev_outgoing_receipts_root();
    let (transactions, receipts) = (chunk.to_transactions(), chunk.prev_outgoing_receipts());

    // 2b. Checking that chunk transactions are valid
    let (tx_root, _) = merklize(transactions);
    if &tx_root != chunk.tx_root() {
        byzantine_assert!(false);
        return Ok(false);
    }
    // 2c. Checking that chunk receipts are valid
    if height_created == 0 {
        return Ok(receipts.is_empty() && outgoing_receipts_root == &CryptoHash::default());
    } else {
        let shard_layout = {
            let prev_block_hash = chunk.prev_block_hash();
            epoch_manager.get_shard_layout_from_prev_block(&prev_block_hash)?
        };
        let outgoing_receipts_hashes = Chain::build_receipts_hashes(receipts, &shard_layout)?;
        let (receipts_root, _) = merklize(&outgoing_receipts_hashes);
        if &receipts_root != outgoing_receipts_root {
            byzantine_assert!(false);
            return Ok(false);
        }
    }
    Ok(true)
