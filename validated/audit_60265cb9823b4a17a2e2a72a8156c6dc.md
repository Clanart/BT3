### Title
Missing Declared Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not validate that the new class hash supplied by a contract is actually declared in the protocol. Any contract can call `replace_class` with an arbitrary undeclared class hash. The OS commits this invalid state transition without rejection, permanently rendering the contract inaccessible and freezing all funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall by directly updating the contract's class hash in `contract_state_changes` with no check that the new hash corresponds to a declared class:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The inline `TODO` comment at line 898 explicitly acknowledges the missing check. The function accepts any arbitrary felt as `class_hash` and writes it directly into the state.

The downstream consequence is deterministic. When any subsequent call targets this contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
```

If `execution_context.class_hash` was never declared, `dict_read` returns the default value `0`. The subsequent `find_element` call over `compiled_class_facts` then fails to locate a fact with hash `0`, and the entry point returns `is_reverted=1` with `ERROR_ENTRY_POINT_NOT_FOUND`. This failure is permanent and applies to every entry point of the contract, including any withdrawal or recovery function.

---

### Impact Explanation

Any funds (ERC-20 balances, ETH, or other assets tracked in the contract's storage) become permanently inaccessible once the contract's class hash is replaced with an undeclared value. The OS proves and commits this state transition to L1 via the ZK proof, making it irreversible on-chain. This satisfies the **Critical — Permanent freezing of funds** impact category.

---

### Likelihood Explanation

The `replace_class` syscall is available to every deployed contract with no caller restriction. A malicious contract developer can:

1. Deploy a contract that appears to be a legitimate vault or DeFi protocol.
2. Attract user deposits.
3. Invoke an internal function that calls `replace_class` with an arbitrary undeclared felt (e.g., `0x1`).
4. The OS accepts the syscall unconditionally.
5. All user funds are permanently frozen.

No privileged role, leaked key, or network-level attack is required. The attacker is an ordinary contract deployer — an unprivileged transaction sender — which is an explicitly allowed attacker model.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that it is declared by asserting that its entry in `contract_class_changes` is non-zero:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the validation already performed in `execute_entry_point` and closes the gap acknowledged by the existing TODO comment.

---

### Proof of Concept

1. Deploy a contract `Vault` that holds user token balances and exposes a hidden `nuke()` function:
   ```
   fn nuke() {
       replace_class(class_hash=0x1);  // 0x1 is never declared
   }
   ```
2. Users deposit funds; `Vault` accumulates balances.
3. Attacker calls `nuke()`. The OS routes the syscall to `execute_replace_class`.
4. `execute_replace_class` writes `class_hash=0x1` into `contract_state_changes` for `Vault`'s address — **no declared-class check is performed** (line 898–910 of `syscall_impls.cairo`).
5. The block is proven and committed to L1.
6. Any subsequent call to `Vault` reaches `execute_entry_point` (line 154 of `execute_entry_point.cairo`), performs `dict_read(key=0x1)` → returns `0`, `find_element` finds no compiled class fact, returns `ERROR_ENTRY_POINT_NOT_FOUND`.
7. All user funds in `Vault` are permanently frozen with no recovery path. [1](#0-0) [2](#0-1) [3](#0-2)
