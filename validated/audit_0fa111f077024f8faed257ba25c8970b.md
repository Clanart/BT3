### Title
Unauthenticated P2P Peer Can Inject Arbitrary `state_root` into Header Storage, Corrupting `starknet_getStateUpdate` `new_root` — (`crates/apollo_p2p_sync/src/client/header.rs`)

### Summary

`HeaderStreamBuilder::parse_data_for_block` performs no cryptographic or structural validation of the `state_root` (or `block_hash`) fields in a peer-supplied `SignedBlockHeader`. The value is stored verbatim via `write_to_storage` → `append_header` and is subsequently returned as the authoritative `new_root` by `starknet_getStateUpdate`.

### Finding Description

**Entrypoint:** `HeaderStreamBuilder::parse_data_for_block` [1](#0-0) 

The only checks performed on the incoming `SignedBlockHeader` are:
1. `block_number` matches the expected sequence number.
2. `signatures.len() == ALLOWED_SIGNATURES_LENGTH` (exactly 1 signature present). [2](#0-1) 

No check is made on:
- `state_root` (the Patricia global root)
- `block_hash` (not verified against a locally computed hash)
- `parent_hash` (there is even an explicit TODO acknowledging this gap)
- The `BlockSignature` is stored but **never verified** against `block_hash` or `state_diff_commitment` in this path

The `_storage_reader` parameter is unused (underscore-prefixed), confirming no cross-reference against stored state occurs. [3](#0-2) 

**Storage write path:** `write_to_storage` calls `append_header` unconditionally with the peer-supplied header. [4](#0-3) 

`append_header` copies `block_header.block_header_without_hash.state_root` directly into `StorageBlockHeader.state_root` with no validation. [5](#0-4) 

**RPC exposure:** `get_state_update` reads the stored header and returns `header.new_root` (which maps to `block_header_without_hash.state_root`) as the `new_root` field of `AcceptedStateUpdate`. [6](#0-5) 

The mapping `new_root ← state_root` is explicit in the RPC block conversion: [7](#0-6) 

**`verify_block_signature` is never called in this path.** The function exists and signs `poseidon_hash(block_hash, state_diff_commitment)`, but even if it were called, it does not bind `state_root` — so the state_root would remain unprotected regardless. [8](#0-7) 

### Impact Explanation

A low-trust P2P peer can send a `SignedBlockHeader` with `state_root = GlobalRoot(Felt::from(0xbad))`. After `parse_data_for_block` accepts it and `write_to_storage` commits it, any subsequent call to `starknet_getStateUpdate` for that block number returns `new_root = 0xbad` — an authoritative-looking wrong global root. This matches the allowed impact: **High — RPC returns an authoritative-looking wrong value.**

The `old_root` of the next block is also derived from the stored `state_root` of the previous block's header, so the corruption chains forward across blocks. [9](#0-8) 

### Likelihood Explanation

Any node operating in P2P sync mode (i.e., not central/feeder-gateway sync) is exposed. The attacker needs only to be a reachable P2P peer — no credentials, keys, or operator access required. The attack is deterministic and requires no brute force.

### Recommendation

1. **Verify the block signature** in `parse_data_for_block` using `verify_block_signature` against a known sequencer public key before accepting the header. This binds `block_hash` and `state_diff_commitment`.
2. **Verify `block_hash` consistency**: recompute the block hash from the received header fields and reject if it does not match the peer-supplied `block_hash`.
3. **Verify `state_root` against the committer**: after syncing the state diff, confirm that the Patricia trie root produced by the committer matches the `state_root` in the stored header.
4. Resolve the existing TODO for `parent_hash` verification. [10](#0-9) 

### Proof of Concept

The proof idea in the question is directly realizable:

```rust
// In a Rust integration test (mirrors the production write path):
let bad_state_root = GlobalRoot(Felt::from(0xbadu64));
let signed_header = SignedBlockHeader {
    block_header: BlockHeader {
        block_hash: BlockHash(StarkHash::ZERO),
        block_header_without_hash: BlockHeaderWithoutHash {
            block_number: BlockNumber(0),
            state_root: bad_state_root,  // attacker-controlled
            ..Default::default()
        },
        state_diff_length: Some(0),
        ..Default::default()
    },
    signatures: vec![BlockSignature::default()], // length == 1, passes the only check
};

// parse_data_for_block accepts this (block_number matches, signatures.len() == 1)
// write_to_storage stores it via append_header
storage_writer
    .begin_rw_txn().unwrap()
    .append_header(BlockNumber(0), &signed_header.block_header).unwrap()
    .commit().unwrap();

// Read back — state_root is stored verbatim
let stored = storage_reader.begin_ro_txn().unwrap()
    .get_block_header(BlockNumber(0)).unwrap().unwrap();
assert_eq!(stored.block_header_without_hash.state_root, bad_state_root);
// RPC starknet_getStateUpdate would return new_root = 0xbad
```

The `append_header` call succeeds because `StorageBlockHeader.state_root` is set directly from the input with no trie verification. [11](#0-10)

                timeout(Duration::from_secs(15), signed_headers_response_manager.next())

                        .block_number,

                }));
            }
            Ok(Some(signed_block_header))

    }

pub fn verify_block_signature(
    sequencer_pub_key: &SequencerPublicKey,
    signature: &BlockSignature,
    state_diff_commitment: &GlobalRoot,
    block_hash: &BlockHash,
) -> Result<bool, BlockVerificationError> {
    let message_hash = Poseidon::hash_array(&[block_hash.0, state_diff_commitment.0]);
    verify_message_hash_signature(&message_hash, &signature.0, &sequencer_pub_key.0).map_err(
        |err| BlockVerificationError::BlockSignatureVerificationFailed {
            block_hash: *block_hash,
            error: err,
        },
    )
}
