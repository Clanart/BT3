### Title
`nativeFee` ETH Permanently Locked in `OmniBridgeWormhole` on Every ERC-20 `initTransfer` with Non-Zero Native Fee — (`evm/src/omni-bridge/contracts/OmniBridgeWormhole.sol`)

### Summary
In `OmniBridgeWormhole`, when a user calls `initTransfer` for an ERC-20 token with `nativeFee > 0`, the contract receives `msg.value = nativeFee + wormholeFee` ETH but only forwards `wormholeFee` (the `extensionValue`) to Wormhole. The `nativeFee` portion is silently retained by the OmniBridge contract with no mechanism to release it to the relayer or refund it to the sender. This ETH is permanently locked.

### Finding Description

In `OmniBridge.sol::initTransfer`, for ERC-20 tokens, the Wormhole extension value is computed as:

```solidity
extensionValue = msg.value - nativeFee;
``` [1](#0-0) 

This `extensionValue` is then passed to `initTransferExtension`. In `OmniBridgeWormhole`, the override forwards only `value` (i.e., `extensionValue`) to Wormhole:

```solidity
_wormhole.publishMessage{value: value}(
    wormholeNonce,
    payload,
    _consistencyLevel
);
``` [2](#0-1) 

The contract receives the full `msg.value` but only forwards `msg.value - nativeFee` to Wormhole. The `nativeFee` ETH remains in the OmniBridge contract. There is no `receive()` fallback, no relayer claim function, and no ETH withdrawal path in the contract other than `finTransfer` for native-ETH tokens (`tokenAddress == address(0)`). The `nativeFee` ETH is therefore irrecoverable.

The base `OmniBridge.initTransferExtension` enforces `value == 0` (reverting with `InvalidValue` if `msg.value != nativeFee`), which confirms that `nativeFee` is intended to be actual ETH sent to the contract — not merely a message parameter:

```solidity
function initTransferExtension(..., uint256 value) internal virtual {
    if (value != 0) {
        revert InvalidValue();
    }
}
``` [3](#0-2) 

The Wormhole override replaces this check with a `publishMessage` call, silently accepting any `nativeFee` ETH without forwarding or refunding it.

### Impact Explanation

Every ERC-20 `initTransfer` call on `OmniBridgeWormhole` where `nativeFee > 0` permanently locks `nativeFee` wei in the bridge contract. There is no admin withdrawal, no relayer claim, and no refund path. This constitutes an irrecoverable lock of user funds in the bridge flow. The locked ETH accumulates across all such transfers and cannot be recovered by any party.

**Allowed impact matched**: Permanent freezing / irrecoverable lock of user funds in bridge flow.

### Likelihood Explanation

The `nativeFee` parameter is a first-class field in the bridge protocol, documented as the native-token fee offered to relayers. Any user who sets `nativeFee > 0` when initiating an ERC-20 transfer on the Wormhole-backed EVM bridge loses that ETH permanently. Relayer tooling or SDK integrations that populate `nativeFee` to incentivize faster processing would trigger this on every call. The entry path is fully unprivileged — any bridge user calling `initTransfer` is affected.

### Recommendation

1. In `OmniBridgeWormhole::initTransferExtension`, validate that `value == _wormhole.messageFee()` and revert otherwise, preventing any ETH from being silently retained.
2. Alternatively, if `nativeFee` ETH is intentionally held by the contract as a relayer incentive, implement an explicit claim mechanism (e.g., a signed claim by the relayer after proof of delivery) so the ETH is not permanently locked.
3. Add a guarded ETH sweep function (admin-only) as a safety net for any ETH that accumulates due to this or similar accounting gaps.

### Proof of Concept

1. Deploy `OmniBridgeWormhole` with a Wormhole address where `messageFee() = 1e15` (0.001 ETH).
2. User calls:
   ```solidity
   OmniBridgeWormhole.initTransfer(
       erc20Token,   // tokenAddress (ERC-20, not address(0))
       1000,         // amount
       0,            // fee
       5e17,         // nativeFee = 0.5 ETH
       "recipient.near",
       "",
       { value: 5e17 + 1e15 }  // nativeFee + wormholeFee
   );
   ```
3. `extensionValue = (5e17 + 1e15) - 5e17 = 1e15` → Wormhole receives `1e15` (correct fee).
4. OmniBridge retains `5e17` ETH (the `nativeFee`).
5. No function exists to withdraw or claim this `5e17` ETH. It is permanently locked. [4](#0-3) [5](#0-4)
