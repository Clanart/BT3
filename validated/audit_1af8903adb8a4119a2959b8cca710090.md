### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the new class hash supplied by the caller corresponds to a previously declared class. This is the direct analog of the ERC721 `safeTransferFrom` bug: just as that function dropped the required `IERC721Receiver` check, `execute_replace_class` drops the required "class is declared" check. A contract can replace its own class with an arbitrary, undeclared felt value. The OS accepts the state update unconditionally. Any subsequent call to that contract will fail irrecoverably at the prover level, permanently freezing all funds held by the contract.

---

### Finding Description

In `execute_replace_class`, after deducting gas, the OS reads the caller's current `StateEntry` and writes a new one with the attacker-supplied `class_hash`:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment on line 898 is an explicit acknowledgment by the developers that this check is **missing**. No assertion, `dict_read`, or `find_element` call is made to confirm that `class_hash` exists in `contract_class_changes` (the declared-class registry).

When any future call targets this contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [2](#0-1) 

`dict_read` on an undeclared key returns `0`. `find_element` with `key=0` (or any value absent from the compiled class facts bundle) **asserts** the element is present — it is not a soft search. The proof fails, and the contract becomes permanently unexecutable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with an undeclared value, every future entry-point call to that contract causes an irrecoverable proof failure at `find_element`. The contract can never be called again. Any ERC-20 tokens, ETH, STRK, or other assets held in the contract's storage are permanently inaccessible. There is no recovery path: `replace_class` cannot be called again because the contract itself is no longer executable.

---

### Likelihood Explanation

**High.** The attack requires only:

1. Deploying a contract (permissionless on StarkNet).
2. Calling the `replace_class` syscall from within that contract with an arbitrary felt value as the new class hash.

No privileged role, leaked key, or operator cooperation is needed. The entry path is fully controlled by an unprivileged transaction sender. The missing check is confirmed by the developers' own TODO comment.

---

### Recommendation

Before writing the new `StateEntry`, verify that `class_hash` is a declared class by performing a `dict_read` on `contract_class_changes` and asserting the returned compiled class hash is non-zero:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the check already performed in `execute_entry_point` and ensures the OS rejects `replace_class` calls that reference undeclared classes, consistent with the StarkNet protocol specification.

---

### Proof of Concept

1. Deploy contract `VictimVault` that holds user funds and exposes a `drain_self()` entry point that calls `replace_class(0xdeadbeef)` — an arbitrary undeclared felt.
2. Attacker calls `drain_self()`. The OS executes `execute_replace_class`:
   - Gas is deducted.
   - No declared-class check is performed.
   - `contract_state_changes[VictimVault_address].class_hash` is set to `0xdeadbeef`.
3. The block is accepted by the sequencer (the state transition is valid from the sequencer's perspective).
4. Any subsequent call to `VictimVault` (e.g., a withdrawal) reaches `execute_entry_point`:
   - `dict_read(contract_class_changes, 0xdeadbeef)` returns `0` (undeclared).
   - `find_element(..., key=0)` asserts the element exists — it does not — proof generation fails.
5. `VictimVault` is permanently unexecutable. All funds are frozen. [3](#0-2) [4](#0-3)

    alloc_locals;

    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
