### Title
Missing Class Hash Validation in `execute_replace_class` Enables Permanent Fund Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS accepts an attacker-controlled `class_hash` from the syscall request and writes it directly into the contract state without verifying that the hash corresponds to a declared class. This is the StarkNet analog of the JOJO "unsafe call" vulnerability: just as JOJO's helper contracts accepted an unvalidated call target that could be any contract, the OS here accepts an unvalidated class hash that can be any felt value. A contract that replaces its class with an undeclared hash becomes permanently unexecutable, permanently freezing any funds it holds.

---

### Finding Description

In `execute_replace_class`, the new class hash is read from the syscall request and stored into the contract state with no validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment at line 898 is a self-admission by the developers that this check is missing. [2](#0-1) 

When a subsequent `call_contract` syscall targets this contract, the OS reads the stored `class_hash` from the state entry and uses it to look up the compiled class:

```cairo
let (state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
    key=contract_address
);
...
tempvar execution_context: ExecutionContext* = new ExecutionContext(
    ...
    class_hash=state_entry.class_hash,
    ...
);
``` [3](#0-2) 

If `class_hash` does not correspond to any declared class, the prover cannot supply a valid compiled class fact, and execution permanently fails. Because `replace_class` can only be called by the contract itself, and the contract can no longer execute after the replacement, there is no recovery path.

The `replace_class` syscall is dispatched unconditionally from `execute_syscalls` with no pre-call whitelist or class-existence check:

```cairo
if (selector == REPLACE_CLASS_SELECTOR) {
    execute_replace_class(contract_address=execution_context.execution_info.contract_address);
    ...
}
``` [4](#0-3) 

---

### Impact Explanation

Any funds held by a contract whose class hash has been replaced with an undeclared value are **permanently frozen**. The contract cannot execute any entry point, cannot transfer funds out, and cannot call `replace_class` again to self-recover. This satisfies the **Critical — Permanent freezing of funds** impact category.

---

### Likelihood Explanation

The `replace_class` syscall is reachable by any unprivileged contract deployer or transaction sender. A malicious actor can:

1. Deploy a contract that accepts user deposits and exposes a function that internally calls `replace_class` with an arbitrary undeclared hash.
2. Collect user deposits.
3. Trigger the `replace_class` call, permanently bricking the contract and freezing all deposited funds.

Additionally, any legitimate contract that uses a proxy/upgrade pattern and has insufficient access control on its upgrade function is exploitable by any external caller who can invoke that function.

---

### Recommendation

Implement the validation noted in the TODO: before writing the new class hash into the contract state, assert that `class_hash` exists as a key in `contract_class_changes` (i.e., it was declared in the current or a prior block). This is the direct analog of the JOJO fix — implementing a whitelist of valid targets before allowing the privileged operation.

---

### Proof of Concept

1. Attacker deploys contract `C` with logic:
   - `deposit()`: accepts funds from users.
   - `brick()`: calls the `replace_class` syscall with `class_hash = 0x1337...` (an arbitrary undeclared felt).
2. Users call `deposit()` and transfer funds to `C`.
3. Attacker calls `brick()`. The OS executes `execute_replace_class`, reads `class_hash = 0x1337...` from the request, and writes it into `C`'s `StateEntry` with no validation.
4. All subsequent calls to `C` fail: the OS reads `class_hash = 0x1337...` from the state, cannot find a compiled class for it, and the execution aborts.
5. Funds in `C` are permanently frozen. No recovery is possible because `C` itself cannot execute `replace_class` again. [5](#0-4)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
