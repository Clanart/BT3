### Title
Missing Declared-Class Validation in `execute_replace_class` Enables Single-Step Irreversible Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS Cairo program does not verify that the new class hash supplied by a contract corresponds to a previously declared contract class. This allows any contract — deployed by an unprivileged user — to replace its own class hash with an arbitrary, undeclared felt value in a single irreversible step. Once replaced, the contract becomes permanently non-callable, freezing all funds it holds. The missing check is explicitly acknowledged in the production code as a TODO.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the requested new class hash directly from the syscall request and writes it into `contract_state_changes` without any validation that the hash corresponds to a declared class:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The OS then commits this undeclared class hash into the global state root via `state_update` → `compute_contract_state_commitment`. Once committed and proven on L1, the state is final.

In any subsequent block, when a caller attempts to invoke the affected contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // the undeclared hash
);
// compiled_class_hash == 0 (dict default)

let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,   // 0 — no such entry exists
);
```

`find_element` with key `0` finds no matching compiled class fact, causing the OS execution to abort. The sequencer, aware that the class is undeclared, will refuse to include any call to the contract in future blocks, permanently freezing it. The state transition that set the undeclared class hash was already proven and accepted on L1 — it cannot be undone.

This is a direct structural analog to the reported "single-step ownership transfer" vulnerability: a single, unconfirmed, irreversible state mutation — replacing the contract's class — is accepted by the OS without requiring any verification that the target state is valid.

---

### Impact Explanation

**Impact: Critical — Permanent freezing of funds.**

Any contract holding ERC-20 tokens, ETH, or other assets whose class hash is replaced with an undeclared value becomes permanently non-callable. The OS proof for the block containing the `replace_class` call is valid and accepted by the L1 verifier. No subsequent block can include calls to the broken contract because the OS cannot resolve its class. The funds are irrecoverably locked.

---

### Likelihood Explanation

The `replace_class` syscall is callable by any contract on itself. An attacker can:

1. Deploy a contract (permissionless via `deploy_account` or the `deploy` syscall).
2. Include a function in that contract that calls `replace_class` with an attacker-controlled or hardcoded undeclared class hash.
3. Submit an invoke transaction calling that function.

The OS processes the syscall and commits the undeclared class hash to state without any validation. The missing check is confirmed by the explicit `TODO` comment at line 898 of `syscall_impls.cairo`. No privileged role is required. The attack is reachable by any unprivileged transaction sender or contract deployer.

---

### Recommendation

Add a validation step inside `execute_replace_class` to confirm that `request.class_hash` exists in `contract_class_changes` (i.e., has a non-zero compiled class hash entry) before updating the contract state. This mirrors the two-step confirmation pattern recommended in the external report: the new class must be verifiably declared before the replacement is committed.

```cairo
// After reading class_hash from request:
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);  // Reject undeclared class hashes
```

---

### Proof of Concept

1. **Deploy** a contract `C` containing an entry point `bad_upgrade()` that executes the `replace_class` syscall with `new_class_hash = 0xdeadbeef` (an undeclared felt).
2. **Submit** an invoke transaction calling `C.bad_upgrade()`.
3. The OS executes `execute_replace_class`:
   - Reads `class_hash = 0xdeadbeef` from the syscall request.
   - Skips the missing declared-class check (line 898 TODO).
   - Writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes`.
4. `state_update` squashes and commits the new state root including `C`'s updated class hash.
5. The OS proof is generated and verified on L1. The state is final.
6. In the next block, any call to `C` causes `execute_entry_point` to call `dict_read(key=0xdeadbeef)` → returns `0` → `find_element(key=0)` → no compiled class found → OS execution aborts.
7. The sequencer permanently excludes `C` from future blocks. All funds in `C` are frozen.
