### Title
`ProofOfInclusion::valid()` Validates Only Internal Self-Consistency, Not Against Any Committed Root — Forged Inclusion Proofs Always Pass - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` in `chia-datalayer` derives the "root" it checks against from the proof's own last `combined_hash` field — a value fully controlled by the proof submitter. The final equality check is a tautology: it always passes whenever the internal chain is self-consistent. An attacker can construct a `ProofOfInclusion` for any arbitrary `node_hash` (e.g., a key-value pair not in the tree) that passes `valid()` without corresponding to any actual committed tree root.

### Finding Description

`ProofOfInclusion::valid()` is implemented as follows:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← taken from the proof itself
    } else {
        self.node_hash
    }
}

pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;   // ← becomes last.combined_hash after final iteration
    }

    existing_hash == self.root_hash()      // ← tautology: both sides are last.combined_hash
}
``` [1](#0-0) 

After the loop completes, `existing_hash` holds the `calculated_hash` from the last iteration. The loop body already asserted `calculated_hash == layer.combined_hash`, so `existing_hash` equals `last.combined_hash`. `root_hash()` also returns `last.combined_hash`. The final check `existing_hash == self.root_hash()` is therefore always `true` when layers are non-empty — it compares a value to itself. [2](#0-1) 

The analog to the UNI_V3 report is direct: just as `slot0.poolSQ96` is a manipulable spot value used in place of a committed oracle price, `last.combined_hash` is an attacker-supplied value used in place of a committed tree root. In both cases, the "validation" is circular — it checks the submitted value against itself.

`ProofOfInclusion` is a `Streamable` type exposed through Python bindings with `from_bytes()`, `from_bytes_unchecked()`, and `py_valid()`. Any Python consumer that deserializes an externally-supplied proof and calls `proof.valid()` as its sole check will accept forged proofs. [3](#0-2) [4](#0-3) [5](#0-4) 

### Impact Explanation

An attacker who can submit a `ProofOfInclusion` to any verifier that calls only `proof.valid()` can prove the inclusion of any arbitrary `node_hash` in any tree. The attacker:

1. Chooses an arbitrary `node_hash` (e.g., a key-value pair not in the actual DataLayer tree).
2. Chooses arbitrary `other_hash` values for each layer.
3. Computes each `combined_hash` correctly using `calculate_internal_hash` — this is the only real constraint.
4. The resulting `ProofOfInclusion` passes `valid()` unconditionally.

The `root_hash()` of the forged proof will be whatever the attacker computed, not the actual committed tree root. Any verifier that does not separately compare `proof.root_hash()` against a known committed root is fully bypassed.

This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, letting untrusted input prove invalid state.** [6](#0-5) 

### Likelihood Explanation

The `ProofOfInclusion` struct is a first-class Python-exposed type with serialization support. The `valid()` method is the only verification method on the struct. Any Python DataLayer client that receives proofs over the network and calls `proof.valid()` without also checking `proof.root_hash() == known_committed_root` is vulnerable. The misleading name `valid()` strongly encourages callers to treat it as a complete validity check. [7](#0-6) 

### Recommendation

`valid()` must accept a `known_root: Hash` parameter and replace the final tautological check with a comparison against that committed root:

```rust
pub fn valid(&self, known_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;
    }

    &existing_hash == known_root   // ← compare against externally committed root
}
```

Alternatively, keep `valid()` as an internal-consistency check but rename it to `is_internally_consistent()` and add a separate `verify_against_root(known_root: &Hash) -> bool` method. All call sites — including the Python binding — must be updated to pass the committed root obtained from a trusted source (e.g., the on-chain DataLayer singleton coin). [8](#0-7)

use chia_datalayer::{Hash, ProofOfInclusion, ProofOfInclusionLayer, Side};

fn forge_proof(fake_node_hash: Hash) -> ProofOfInclusion {
    // Choose arbitrary sibling hash
    let other_hash: Hash = [0xAB; 32];
    // Compute combined_hash correctly so the internal chain check passes
    let combined_hash = chia_datalayer::calculate_internal_hash(
        &fake_node_hash,
        Side::Left,
        &other_hash,
    );
    ProofOfInclusion {
        node_hash: fake_node_hash,
        layers: vec![ProofOfInclusionLayer {
            other_hash_side: Side::Left,
            other_hash,
            combined_hash,   // attacker-computed, not from any real tree
        }],
    }
}

fn main() {
    let fake_node_hash: Hash = [0xFF; 32]; // not in any real tree
    let proof = forge_proof(fake_node_hash);
    assert!(proof.valid()); // passes — forged proof accepted
    // proof.root_hash() returns attacker-chosen combined_hash, not any real committed root
}

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
