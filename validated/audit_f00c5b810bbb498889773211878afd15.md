Audit Report

## Title
ETH stranded via `payable` entry points is silently consumed by the next WETH-pool depositor — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary
`PeripheryPayments.pay` unconditionally uses `address(this).balance` to cover WETH payments before pulling from the actual payer. Because `addLiquidityExactShares` is `payable` and `_addLiquidity` issues no automatic refund after settlement, any ETH left in the contract from a prior transaction is consumed on behalf of the next WETH-pool depositor, causing the original sender to lose that ETH permanently.

## Finding Description
**ETH accumulation:** `addLiquidityExactShares` is `payable` [1](#0-0)  and `multicall` is `payable` [2](#0-1) . The `receive()` guard only blocks plain ETH transfers (no calldata), not ETH sent via a `payable` function call. [3](#0-2) 

**No automatic refund:** `_addLiquidity` calls `_clearPayContext()` after settlement but never calls `refundETH`. [4](#0-3)  `refundETH` is opt-in and must be called explicitly by the user. [5](#0-4) 

**Stranded ETH consumed by next depositor:** In `pay`, when `token == WETH` and `payer != address(this)`, the function reads `address(this).balance` first. If `nativeBalance >= value`, it wraps the contract's ETH entirely and transfers it to the pool, pulling nothing from the actual payer. [6](#0-5) 

**Exploit path:**
1. User A calls `addLiquidityExactShares{value: 1 ether}(wethPool, ...)`. Pool requests 0.5 WETH in callback. `pay` wraps 0.5 ETH from `address(this).balance` (= 1 ETH), leaving 0.5 ETH stranded. User A does not call `refundETH`.
2. Attacker calls `addLiquidityExactShares(wethPool, ...)` with no ETH. Pool requests 0.5 WETH. `pay` sees `nativeBalance = 0.5 ETH >= 0.5 ETH`, wraps User A's stranded ETH, and transfers it to the pool. Attacker's own WETH is never pulled.
3. User A loses 0.5 ETH; attacker receives pool shares funded by User A's ETH.

The `payer` in the callback is always `msg.sender` of the outer call, confirming the attacker's own address is the `payer` yet pays nothing. [7](#0-6) 

## Impact Explanation
Direct, irrecoverable loss of ETH for any user who sends excess ETH to a `payable` entry point without appending a `refundETH` call. The attacker receives pool shares at zero cost, with the deficit funded by the victim's stranded ETH. This is a direct loss of user principal meeting Sherlock Medium thresholds: real fund loss, contingent on user error and a race condition between two independent transactions.

## Likelihood Explanation
The precondition — a user omitting `refundETH` after sending excess ETH — is realistic because `addLiquidityExactShares` is independently callable as a `payable` function without requiring a `multicall` wrapper. An attacker can monitor the contract's ETH balance on-chain or via mempool and submit a WETH-pool deposit before the victim recovers their ETH. The `receive()` guard prevents the attacker from *forcing* ETH into the contract, so exploitation is entirely opportunistic.

## Recommendation
Inside `_addLiquidity`, after settlement, automatically refund any remaining `address(this).balance` to the payer:
```solidity
if (address(this).balance > 0) {
    _transferETH(payer, address(this).balance);
}
```
Alternatively, snapshot `address(this).balance` before the call and use only that snapshot amount in `pay`, ignoring any ETH that arrived in prior transactions.

## Proof of Concept
```solidity
function test_strandedEthConsumedByAttacker() public {
    // User A sends 1 ETH; pool only needs 0.5 WETH
    vm.deal(userA, 1 ether);
    vm.prank(userA);
    adder.addLiquidityExactShares{value: 1 ether}(
        wethPool, userA, 0, deltas, 0.5 ether, 0, ""
    );
    // 0.5 ETH remains; userA did not call refundETH
    assertEq(address(adder).balance, 0.5 ether);

    // Attacker deposits into same WETH pool needing 0.5 WETH
    // pay() sees nativeBalance=0.5 ether >= value=0.5 ether
    // wraps adder's ETH, pulls 0 WETH from attacker
    uint256 attackerWethBefore = IERC20(WETH).balanceOf(attacker);
    vm.prank(attacker);
    adder.addLiquidityExactShares(wethPool, attacker, 0, deltas, 0.5 ether, 0, "");

    assertEq(IERC20(WETH).balanceOf(attacker), attackerWethBefore); // attacker paid nothing
    assertEq(address(adder).balance, 0);                            // userA's ETH gone
}
```
