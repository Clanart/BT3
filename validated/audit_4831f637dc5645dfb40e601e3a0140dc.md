### Title
`todo!()` in `SyncStateReader::get_compiled_class_hash` Panics on Any Declare Transaction, Crashing the Gateway - (File: `crates/apollo_gateway/src/sync_state_reader.rs`)

---

### Summary

`SyncStateReader`, the production state reader used by the gateway for stateful transaction validation, implements `get_compiled_class_hash` with a bare `todo!()` macro. Any Declare V2/V3 transaction submitted to the gateway triggers this code path, causing an unconditional panic that crashes the gateway process and halts admission of all subsequent valid transactions.

---

### Finding Description

`SyncStateReader` implements the `BlockifierStateReader` trait for the gateway's stateful validation path. Every method is implemented except `get_compiled_class_hash`, which is left as a stub:

```rust
fn get_compiled_class_hash(&self, _class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    todo!()
}
``` [1](#0-0) 

`SyncOrGenesisStateReader` wraps `SyncStateReader` and unconditionally delegates `get_compiled_class_hash` to it:

```rust
fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    match self {
        Self::Sync(state_reader) => state_reader.get_compiled_class_hash(class_hash),
        Self::Genesis(genesis_state_reader) => {
            genesis_state_reader.get_compiled_class_hash(class_hash)
        }
    }
}
``` [2](#0-1) 

`SyncOrGenesisStateReader` is the concrete type returned by `SyncStateReaderFactory` and handed to the blockifier for stateful validation: [3](#0-2) 

During stateful validation of a Declare V2/V3 transaction, the blockifier's `CachedState` calls `get_compiled_class_hash` on the underlying state reader to check whether the class has already been declared. This call reaches `SyncStateReader::get_compiled_class_hash`, hits `todo!()`, and panics. In a Tokio async task without `catch_unwind`, this propagates as an unrecoverable panic, crashing the gateway worker.

The `StateReader` trait itself also carries a second stub — `get_compiled_class_hash_v2` — with an `unimplemented!()` default body, which would fire for any `SyncStateReader` caller that invokes it without an override: [4](#0-3) 

---

### Impact Explanation

The gateway is the sole admission point for transactions before sequencing. A panic in the stateful-validation task crashes the gateway process (or at minimum the handling task), causing it to stop accepting any transactions — valid or otherwise — until the process is restarted. This maps directly to:

> **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

---

### Likelihood Explanation

Triggering the panic requires only submitting a syntactically valid Declare V2/V3 transaction to the public gateway endpoint. No privileged access, special account, or unusual network condition is required. Any unprivileged user can perform this action at will, making the likelihood **high**.

---

### Recommendation

Implement `get_compiled_class_hash` in `SyncStateReader` by querying the state-sync client, analogous to how `get_storage_at`, `get_nonce_at`, and `get_class_hash_at` are implemented. If the state-sync client does not yet expose a `get_compiled_class_hash` RPC, add that endpoint and wire it through `StateSyncClient` / `SharedStateSyncClientMetricWrapper`. Until then, replace `todo!()` with a `StateError::StateReadError` return so the gateway returns a proper error instead of panicking.

---

### Proof of Concept

1. Stand up a sequencer node with the gateway enabled and at least one synced block.
2. Construct any valid Declare V2 or V3 transaction (any Sierra class, any account).
3. Submit it to the gateway's `add_declare_transaction` endpoint.
4. The gateway's stateful-validation task calls `CachedState::get_compiled_class_hash` → `SyncOrGenesisStateReader::get_compiled_class_hash` → `SyncStateReader::get_compiled_class_hash` → `todo!()`.
5. The Tokio task panics; the gateway process crashes or stops serving requests.
6. All subsequent valid transactions (Invoke, DeployAccount, Declare) are rejected with a connection error until the process is restarted.
