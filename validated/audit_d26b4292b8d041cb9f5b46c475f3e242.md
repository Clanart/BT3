### Summary

`execute_declare_transaction` contains a special "bootstrap" path that completely skips signature verification, nonce enforcement, and fee payment. The trigger conditions (`sender_address == 'BOOTSTRAP'`, `nonce == 0`, `version == 3`, `max_fee == 0`) can be satisfied by any unprivileged user. An attacker can therefore declare arbitrary class hashes with an attacker-chosen `compiled_class_hash`, poisoning the class registry. This is the direct analog of the external report: a guarded path (normal declare with `__validate_declare__`) is bypassed via an alternative unguarded path (bootstrap declare), exactly as `check_lock` was bypassed via a direct token transfer.
