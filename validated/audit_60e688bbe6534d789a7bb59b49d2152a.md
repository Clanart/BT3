### Title
`validate_virtual_os_state_diff` unconditionally rejects zero-value storage writes, causing valid client-side-proving transactions to fail — (File: `crates/starknet_transaction_prover/src/running/committer_utils.rs`)

---

### Summary

`create_commitment_infos_with_state_changes` in the transaction prover unconditionally calls `validate_virtual_os_state_diff`, which rejects any state diff containing a storage write of `Felt::ZERO`. Writing zero to a storage slot is a valid Starknet operation (resetting a counter, clearing a flag). Any transaction that does so cannot be proved, and is rejected at the gateway when client-side proving is required.

---

### Finding Description

In `crates/starknet_transaction_prover/src/running/storage_proofs.rs`, `get_storage_proofs` dispatches to `create_commitment_infos_with_state_changes` when `config.include_state_changes == true`. [1](#0-0) 

Inside that function, `validate_virtual_os_state_diff` is called unconditionally on every state diff before the Patricia-trie commit: [2](#0-1) 

`validate_virtual_os_state_diff` treats any storage value of `Felt::ZERO` as a forbidden "storage deletion": [3](#0-2) 

The Starknet OS itself imposes no such restriction. The Cairo `execute_storage_write` syscall writes `request.value` verbatim, including zero: [4](#0-3) 

The conversion path that feeds the check is:

```
StateMaps.storage  →  state_maps_to_committer_state_diff  →  StateDiff.storage_updates
``` [5](#0-4) 

Zero-value entries survive this conversion unchanged. A transaction that calls `storage_write_syscall(0, addr, 0)` produces a `StateDiff` entry `(addr, key) → StarknetStorageValue(Felt::ZERO)`, which immediately triggers the error.

**Structural analog to the external report:**

| External (Incentivizer) | Sequencer (transaction prover) |
|---|---|
| `rescue(token)` is for **any** ERC20 | `create_commitment_infos_with_state_changes` is for **any** state diff |
| Unconditionally calls `verifyRewardBalance()` | Unconditionally calls `validate_virtual_os_state_diff()` |
| Fails when reward balance is low, even for non-reward tokens | Fails when any storage value is zero, even for valid zero-writes |
| Valid rescue of non-reward token is blocked | Valid proof of zero-write transaction is blocked |

---

### Impact Explanation

When a transaction uses client-side proving (`proof_facts` attached), the gateway invokes `VirtualSnosProver::prove_transaction`, which calls `get_storage_proofs` with `include_state_changes = true`. If the transaction's state diff contains any zero-value storage write, `validate_virtual_os_state_diff` returns `ProofProviderError::InvalidStateDiff`, the proof pipeline aborts, and the gateway rejects the transaction.

**Impact:** High — Mempool/gateway admission rejects a valid transaction before sequencing. [6](#0-5) 

---

### Likelihood Explanation

Writing zero to storage is routine: resetting a counter, clearing an approval, zeroing a flag. Any such transaction submitted with client-side proving enabled hits this path. The trigger requires no privilege — any user account can craft it.

---

### Recommendation

Apply `validate_virtual_os_state_diff` only when the state diff is known to be free of zero-value writes, or remove the zero-value storage check if the virtual OS Patricia-trie committer can handle leaf deletion. At minimum, the check should be conditional on whether the storage key previously held a non-zero value (i.e., distinguish a genuine deletion from a first-time zero write).

---

### Proof of Concept

1. Deploy a contract with an entry point that writes zero to storage:
   ```cairo
   fn clear(ref self: ContractState) {
       syscalls::storage_write_syscall(0, some_key, 0).unwrap_syscall();
   }
   ```
2. Submit an invoke transaction calling `clear`, with `proof_facts` populated (client-side proving mode).
3. The gateway calls `VirtualSnosProver::prove_transaction`.
4. `get_storage_proofs` is called with `config.include_state_changes = true`.
5. `state_maps_to_committer_state_diff` converts `StateMaps { storage: {(addr, key) → Felt::ZERO} }` into a `StateDiff` with `storage_updates[(addr, key)] = StarknetStorageValue(Felt::ZERO)`.
6. `validate_virtual_os_state_diff` hits the branch at line 75 and returns `Err(ProofProviderError::InvalidStateDiff("Storage deletion not allowed …"))`.
7. Proof generation fails; the gateway returns an error and the valid transaction is dropped. [7](#0-6) [8](#0-7)

/// Converts blockifier's StateMaps to committer's StateDiff format.
pub fn state_maps_to_committer_state_diff(state_maps: StateMaps) -> StateDiff {
    StateDiff {
        address_to_class_hash: state_maps.class_hashes,
        address_to_nonce: state_maps.nonces,
        class_hash_to_compiled_class_hash: state_maps
            .compiled_class_hashes
            .into_iter()
            .map(|(class_hash, compiled_class_hash)| {
                (class_hash, CompiledClassHash(compiled_class_hash.0))
            })
            .collect(),
        storage_updates: StorageDiff::from(StorageView(state_maps.storage))
            .into_iter()
            .map(|(address, updates)| {
                (
                    address,
                    updates
                        .into_iter()
                        .map(|(key, value)| (StarknetStorageKey(key), StarknetStorageValue(value)))
                        .collect(),
                )
            })
            .collect(),
    }
}

/// Validates that the committer state diff contains only allowed state transitions.
///
/// This function enforces the following constraints:
/// * **No Storage Deletions:** Storage entries cannot be updated to `Felt::ZERO`.
/// * **No Class Declarations:** The `class_hash_to_compiled_class_hash` map must be empty.
/// * **No Contract Deployments:** The `address_to_class_hash` map must be empty.
pub(crate) fn validate_virtual_os_state_diff(
    state_diff: &StateDiff,
) -> Result<(), ProofProviderError> {
    // validate no storage deletions.
    for (address, storage_diffs) in &state_diff.storage_updates {
        for (key, value) in storage_diffs {
            if value.0 == Felt::ZERO {
                return Err(ProofProviderError::InvalidStateDiff(format!(
                    "Storage deletion not allowed: try to delete storage at address {address:?}, \
                     key {key:?}"
                )));
            }
        }
    }
    // validate no contract deployments (or replaced classes).
    if !state_diff.address_to_class_hash.is_empty() {
        return Err(ProofProviderError::InvalidStateDiff(format!(
            "Contract deployments not allowed: try to deploy contracts(address to class hash): \
             {0:?}",
            state_diff.address_to_class_hash
        )));
    }
    // validate no contract declarations (or compiled class hash updates).
    if !state_diff.class_hash_to_compiled_class_hash.is_empty() {
        return Err(ProofProviderError::InvalidStateDiff(format!(
            "Contract declarations not allowed: try to declare classes(class hash to compiled \
             class hash): {0:?}",
            state_diff.class_hash_to_compiled_class_hash
        )));
    }
    Ok(())
}
