### Title
`ProofOfSpace::update_digest` Panics on Parsed-but-Cryptographically-Invalid v2 Proof — (`crates/chia-protocol/src/proof_of_space.rs`)

### Summary
`ProofOfSpace::parse()` for `version == 1` (v2 proofs) accepts any arbitrary proof byte string without cryptographic validation. `update_digest()` subsequently calls `self.quality_string().expect(...)` on that same proof. Because `quality_string()` returns `None` for any proof that fails `chia_pos2::quality_string_from_proof`, the `.expect()` unconditionally panics. Any node that receives a crafted v2 `ProofOfSpace` embedded in a block and then calls `.hash()` or `.get_hash()` on it will crash.

### Finding Description

**Step 1 — `parse()` accepts garbage proof bytes.**

`parse::<false>()` for `version == 1` reads `proof` as a raw `Bytes` field with no cryptographic check. The only structural guard is that exactly one of `pool_public_key` / `pool_contract_puzzle_hash` is set: [1](#0-0) 

An attacker can satisfy that guard trivially (set `pool_public_key`, clear `pool_contract_puzzle_hash`) and supply any garbage bytes as `proof`. `parse()` returns `Ok(pos)`.

**Step 2 — `quality_string()` returns `None` for invalid proofs.**

`quality_string()` delegates to `chia_pos2::quality_string_from_proof`, which returns `None` when the proof bytes do not satisfy the PoS2 constraints: [2](#0-1) 

**Step 3 — `update_digest()` panics via `.expect()` on `None`.**

In the `version == 1` branch of `update_digest()`, the result of `quality_string()` is unwrapped with `.expect()`: [3](#0-2) 

If `quality_string()` returns `None`, this line panics unconditionally. There is no `Result` propagation — it is a hard process abort.

**Step 4 — `hash()` calls `update_digest()` directly.**

The `Streamable` trait's `hash()` method calls `update_digest()` with no error handling: [4](#0-3) 

**Step 5 — `RewardChainBlock::update_digest()` propagates the panic.**

`RewardChainBlock::update_digest()` calls `self.proof_of_space.update_digest(digest)` unconditionally: [5](#0-4) 

So hashing any `RewardChainBlock` that embeds a crafted v2 `ProofOfSpace` also panics. The Python binding `get_hash()` exposed on both types reaches this same path.

### Impact Explanation

A Rust `panic!` / `.expect()` failure in a non-`catch_unwind` context terminates the process. Any full node that receives a network block containing a v2 `ProofOfSpace` with garbage proof bytes and then calls `.hash()` / `.get_hash()` (e.g., to verify the block hash during validation) will crash. Because the crash is deterministic and reproducible from a single crafted message, an attacker can halt every reachable node simultaneously, constituting a chain halt.

Impact: **Critical** — serialized network object triggers deterministic node crash / chain halt.

### Likelihood Explanation

The attacker needs only to:
1. Construct a byte string where the prefix byte encodes `version == 1` (bit 1 set), exactly one of pool key / contract hash is present, and `proof` is arbitrary garbage.
2. Embed it in a `RewardChainBlock` (or any other struct containing `ProofOfSpace`) and broadcast it.

No keys, no privileges, no chain state are required. The crafted message passes `parse()` and the panic fires the first time any node calls `.hash()` on the object.

### Recommendation

The invariant mismatch must be closed at the parse boundary. `parse()` for `version == 1` should validate the proof bytes via `quality_string_from_proof` and return `Err(Error::InvalidPoS)` if they are invalid, before returning `Ok`. Alternatively, `update_digest()` should be refactored to return a `Result` and propagate the failure instead of calling `.expect()`. The `.expect()` at line 252 is the immediate root cause and must not remain reachable from externally-supplied data.

### Proof of Concept

```rust
// Craft a syntactically valid v2 ProofOfSpace with garbage proof bytes.
// prefix byte = 0b10 => version = 1, pool_contract_puzzle_hash absent
// pool_public_key = Some(valid G1Element), pool_contract_puzzle_hash = None
// proof = 128 bytes of 0xFF (wrong values, quality_string_from_proof returns None)

let mut bytes: Vec<u8> = Vec::new();
// challenge (32 bytes)
bytes.extend_from_slice(&[0u8; 32]);
// pool_public_key: present (prefix 0x01) + 48 bytes of a valid G1 point
bytes.push(0x01);
bytes.extend_from_slice(&VALID_G1_BYTES);
// pool_contract_puzzle_hash prefix: 0b10 => version=1, not present
bytes.push(0b10);
// plot_public_key: 48 bytes of a valid G1 point
bytes.extend_from_slice(&VALID_G1_BYTES);
// plot_index (u16 LE), meta_group (u8), strength (u8)
bytes.extend_from_slice(&[0u8, 0u8, 0u8, 2u8]);
// proof: 4-byte length prefix + 128 bytes of garbage
bytes.extend_from_slice(&128u32.to_be_bytes());
bytes.extend_from_slice(&[0xFFu8; 128]);

let pos = ProofOfSpace::parse::<false>(&mut Cursor::new(&bytes)).unwrap();
// parse() succeeds — no proof validation

pos.hash(); // <- PANIC: "internal error. Can't compute hash of invalid ProofOfSpace"
```

The same panic fires when `RewardChainBlock::hash()` is called on a block embedding this `ProofOfSpace`, which is the realistic network-facing trigger.

    pub fn quality_string(&self) -> Option<Bytes32> {
        if self.version != 1 {
            return None;
        }

        let k_size = (self.proof.len() * 8 / 128) as u8;
        let plot_id = self.compute_plot_id().to_bytes();
        chia_pos2::quality_string_from_proof(&plot_id, k_size, self.strength, self.proof.as_slice())
            .map(|quality| {
                let mut sha256 = Sha256::new();
                sha256.update(chia_pos2::serialize_quality(
                    &quality.chain_links,
                    self.strength,
                ));
                sha256.finalize().into()
            })
    }
