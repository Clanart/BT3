Looking at the full `initTransfer1155` and `finTransfer` code paths in `OmniBridge.sol`.

**Key findings:**

**1. `initTransfer1155` does NOT populate `multiTokens[deterministicToken]`** [1](#0-0) 

The function computes `deterministicToken`, locks the ERC1155 tokens, and emits `InitTransfer` — but never writes to `multiTokens[deterministicToken]`. There is no guard requiring prior `logMetadata1155`.

**2. `logMetadata1155` is the only writer to `multiTokens`** [2](#0-1) 

**3. `finTransfer` requires `multiTokens[deterministicToken]` to release ERC1155 tokens** [3](#0-2)

**4. The test suite confirms `initTransfer1155` succeeds without prior `logMetadata1155`** [4](#0-3) 

Tokens are locked, `multiTokens` remains empty.

### Title
Missing `multiTokens` population in `initTransfer1155` causes permanent ERC1155 token lock — (`evm/src/omni-bridge/contracts/OmniBridge.sol`)

### Summary
`initTransfer1155` locks ERC1155 tokens in the bridge but never writes to `multiTokens[deterministicToken]`. The only function that populates this mapping is `logMetadata1155`. Without this mapping, `finTransfer` cannot identify the token as ERC1155 and falls through to an ERC20 `safeTransfer` call on a non-contract address, which reverts. Tokens are permanently locked with no refund or recovery mechanism.

### Finding Description
`initTransfer1155` computes `deterministicToken = deriveDeterministicAddress(tokenAddress, tokenId)`, transfers ERC1155 tokens from the user into the bridge, and emits `InitTransfer(deterministicToken, ...)`. It does not write to `multiTokens[deterministicToken]`.

`finTransfer` dispatches on `multiTokens[payload.tokenAddress]`:
- If `multiToken.tokenAddress != address(0)` → ERC1155 release path (correct)
- Otherwise → falls through to ERC20 `safeTransfer(deterministicToken, ...)` → reverts because `deterministicToken` is a hash-derived address with no code

Since `logMetadata1155` is a separate, optional call with no enforcement in `initTransfer1155`, any user can skip it and trigger the lock.

### Impact Explanation
ERC1155 tokens sent via `initTransfer1155` without a prior `logMetadata1155` are permanently locked in the bridge contract. There is no admin rescue function, no refund path, and no way to call `finTransfer` successfully for these tokens. This matches the Critical impact: irrecoverable lock of user funds in bridge flows.

### Likelihood Explanation
`initTransfer1155` is a public, unprivileged function. Any user who calls it without first calling `logMetadata1155` (which is easy to miss since it is a separate, undocumented prerequisite) will permanently lose their tokens. The test suite itself demonstrates this path succeeds silently.

### Recommendation
`initTransfer1155` should populate `multiTokens[deterministicToken]` itself, mirroring the logic in `logMetadata1155`:

```solidity
MultiTokenInfo storage multiToken = multiTokens[deterministicToken];
if (multiToken.tokenAddress == address(0)) {
    multiToken.tokenAddress = tokenAddress;
    multiToken.tokenId = tokenId;
} else if (multiToken.tokenAddress != tokenAddress || multiToken.tokenId != tokenId) {
    revert ERC1155MappingMismatch();
}
```

Alternatively, add a guard that reverts if `multiTokens[deterministicToken]` is not already populated, forcing users to call `logMetadata1155` first.

### Proof of Concept
1. Deploy `OmniBridge` and an ERC1155 token. Mint tokens to `user`.
2. `user` approves bridge and calls `initTransfer1155(tokenAddress, tokenId, 1, 0, 0, "user.near", "")` — **no prior `logMetadata1155`**.
3. Confirm `erc1155.balanceOf(bridge, tokenId) == 1` (tokens locked).
4. Confirm `bridge.multiTokens(deterministicToken).tokenAddress == address(0)` (mapping empty).
5. Attempt `finTransfer` with a valid MPC signature for `deterministicToken` → reverts at `IERC20(deterministicToken).safeTransfer(...)` because `deterministicToken` has no code.
6. No other code path in the contract can release the locked ERC1155 tokens.

    function initTransfer1155(
        address tokenAddress,
        uint256 tokenId,
        uint128 amount,
        uint128 fee,
        uint128 nativeFee,
        string calldata recipient,
        string calldata message
    ) external payable whenNotPaused(PAUSED_INIT_TRANSFER) {
        currentOriginNonce += 1;
        if (fee >= amount) {
            revert InvalidFee();
        }

        address deterministicToken = deriveDeterministicAddress(
            tokenAddress,
            tokenId
        );

        IERC1155(tokenAddress).safeTransferFrom(
            msg.sender,
            address(this),
            tokenId,
            amount,
            ""
        );

        uint256 extensionValue = msg.value - nativeFee;

        initTransferExtension(
            msg.sender,
            deterministicToken,
            currentOriginNonce,
            amount,
            fee,
            nativeFee,
            recipient,
            message,
            extensionValue

            msg.sender,
            deterministicToken,

            amount,
