### Title
Gateway Admits Transactions with Zero L1/L1DataGas `max_price_per_unit` Due to Asymmetric Minimum Price Validation — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway enforces a minimum price floor only on `l2_gas.max_price_per_unit`, leaving `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` completely unchecked. A transaction with zero L1/L1DataGas prices passes all gateway admission checks, enters the mempool, and is only rejected later by the blockifier's `check_fee_bounds` during execution. This is the direct sequencer analog of the UMA "missing minimum collateral ratio" bug: just as UMA liquidators could set a *maximum* collateral ratio but not a *minimum*, the gateway here enforces a minimum L2 gas price but no minimum on L1/L1DataGas prices.

---

### Finding Description

**Root cause — stateless validator (`crates/apollo_gateway/src/stateless_transaction_validator.rs`, lines 71–76):**

```rust
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
```

Only `l2_gas.max_price_per_unit` is compared against `min_gas_price`. `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never checked against any floor. [1](#0-0) 

**Root cause — stateful validator (`crates/apollo_gateway/src/stateful_transaction_validator.rs`, lines 330–362):**

```rust
// TODO(Arni): Consider running this validation for all gas prices.
fn validate_tx_l2_gas_price_within_threshold(...)
```

The function is explicitly scoped to L2 gas only. The TODO comment acknowledges the gap. The stateful validator already reads `get_block_info()` which contains all three gas prices, but only uses the L2 price. [2](#0-1) 

**Where the check actually lives — blockifier (`crates/blockifier/src/transaction/account_transaction.rs`, lines 437–445):**

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(
        ResourceBoundsError::MaxGasPriceTooLow { resource, ... }
    );
}
```

This check covers all three resources (L1Gas, L1DataGas, L2Gas), but it runs inside `perform_pre_validation_stage` at *execution time* inside the batcher — not at gateway admission. [3](#0-2) 

**Attack path:**

1. Attacker submits `AllResources` invoke transactions with `l2_gas.max_price_per_unit ≥ min_gas_price` (e.g., 8 Gwei) but `l1_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0`.
2. `StatelessTransactionValidator::validate_resource_bounds` passes — L2 price meets threshold; L1/L1DataGas not checked. [4](#0-3) 
3. `StatefulTransactionValidator::validate_resource_bounds` passes — only L2 price checked against previous block. [5](#0-4) 
4. Transactions enter the mempool.
5. Batcher picks them up; `check_fee_bounds` fails with `MaxGasPriceTooLow` for L1Gas (block L1 gas price > 0).
6. Transactions are rejected from the block but have already consumed mempool slots and batcher resources.

The `StatefulTransactionValidatorConfig` documents `min_gas_price_percentage` as applying to "all gas prices" conceptually, but the implementation only applies it to L2 gas. [6](#0-5) 

---

### Impact Explanation

**High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

Transactions that will deterministically fail at blockifier execution (because their L1/L1DataGas `max_price_per_unit` is below the block's actual gas price) are admitted through the gateway and mempool. An unprivileged attacker can flood the mempool with such transactions, degrading sequencer throughput and wasting batcher resources. No state corruption occurs, but the gateway's admission invariant is broken: it accepts transactions it should reject.

---

### Likelihood Explanation

Any user can craft and submit such transactions with no special privileges. The attack is cheap because rejected transactions incur no fee. The gap is confirmed by the explicit TODO comment in production code.

---

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or create a parallel function) to also validate `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the previous block's L1 gas and L1 data gas prices, applying the same `min_gas_price_percentage` threshold. The stateful validator already calls `get_block_info()` which returns all three gas prices — the fix requires reading `l1_gas_price` and `l1_data_gas_price` from the same `GasPriceVector` and applying the threshold check symmetrically.

---

### Proof of Concept

```
Submit RpcInvokeTransactionV3 with:

    l2_gas:      { max_amount: 1_000_000, max_price_per_unit: 8_000_000_000 },  // passes min_gas_price
    l1_gas:      { max_amount: 1_000_000, max_price_per_unit: 0 },              // NOT checked at gateway
    l1_data_gas: { max_amount: 1_000_000, max_price_per_unit: 0 },              // NOT checked at gateway
  }

Gateway (stateless):  ACCEPTED  — l2_gas price ≥ min_gas_price (8 Gwei); l1/l1_data not checked
Gateway (stateful):   ACCEPTED  — only l2_gas price compared to previous block threshold
Mempool:              ACCEPTED
Blockifier:           REJECTED  — MaxGasPriceTooLow { resource: L1Gas, max_amount: 0, actual: <block_l1_price> }
```

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

pub struct StatefulTransactionValidatorConfig {
    // If true, ensures the max L2 gas price exceeds (a configurable percentage of) the base gas
    // price of the previous block.
    pub validate_resource_bounds: bool,
    pub max_allowed_nonce_gap: u32,
    pub reject_future_declare_txs: bool,
    pub max_nonce_for_validation_skip: Nonce,
    pub versioned_constants_overrides: Option<VersionedConstantsOverrides>,
    // Minimum gas price as percentage of threshold to accept transactions.
    pub min_gas_price_percentage: u8, // E.g., 80 to require 80% of threshold.
}

impl Default for StatefulTransactionValidatorConfig {
    fn default() -> Self {

            max_nonce_for_validation_skip: Nonce(Felt::ONE),
            min_gas_price_percentage: 100,
