### Title
EIP-712 Legacy Ante Handler Does Not Validate That `feePayer` Matches the Message Signer, Enabling Unauthorized Cosmos Message Execution on Behalf of Arbitrary Accounts - (File: `ante/cosmos/eip712.go`)

---

### Summary

The `LegacyEip712SigVerificationDecorator` in `ante/cosmos/eip712.go` verifies that the `feePayer` field in `ExtensionOptionsWeb3Tx` matches the recovered signature, but **never checks that the `feePayer` is the same account as the actual message signer declared in the Cosmos messages**. An attacker who controls any valid Cosmos account can craft a Legacy EIP-712 transaction where the `msgs` array names a victim's address as the message signer (e.g., `delegator_address`, `from_address`), while the `feePayer` in the extension is set to the attacker's own address. The attacker signs the EIP-712 typed data with their own key, which passes all signature checks. The transaction is then executed with the victim's address as the authoritative message signer, allowing the attacker to execute arbitrary Cosmos messages (bank sends, staking delegations, governance votes, etc.) on behalf of any account.

---

### Finding Description

The vulnerability class from the external report is **insufficient input validation allowing signing of arbitrary objects on behalf of other parties**. The analog in Ethermint is in the Legacy EIP-712 ante handler path.

**Root cause — `VerifySignature` in `ante/cosmos/eip712.go`:**

The function `VerifySignature` (lines 171–302) performs the following checks:
1. Recovers the public key from `extOpt.FeePayerSig` over the EIP-712 typed data hash.
2. Checks that the recovered public key equals `pubKey` (the account key of `signerAddrs[0]`, i.e., the Cosmos tx signer).
3. Checks that the recovered address equals `feePayer` (from `extOpt.FeePayer`).

What it **does not check** is whether `feePayer` equals the signer declared inside the Cosmos messages themselves.

```go
// ante/cosmos/eip712.go lines 232–248
if len(extOpt.FeePayer) == 0 {
    return errorsmod.Wrap(...)
}
feePayer, err := sdk.AccAddressFromBech32(extOpt.FeePayer)
...
feeDelegation := &eip712.FeeDelegationOptions{
    FeePayer: feePayer,
}

if err := eip712.LegacyValidatePayloadMessages(msgs); err != nil { ... }

typedData, err := eip712.LegacyWrapTxToTypedData(ethermintCodec, extOpt.TypedDataChainID, msgs[0], txBytes, feeDelegation)
```

`LegacyValidatePayloadMessages` (lines 236–274 of `ethereum/eip712/encoding_legacy.go`) only checks that all messages share the same type and the same signer address among themselves — it does **not** check that this shared signer equals `feePayer`.

`LegacyWrapTxToTypedData` (lines 49–103 of `ethereum/eip712/eip712_legacy.go`) builds the EIP-712 typed data by:
- Deriving the type schema from the Go struct fields of `msgs[0]` via reflection (`walkFields`/`legacyTraverseFields`).
- Injecting `feeDelegation.FeePayer` into the `fee.feePayer` field of the JSON payload.

The EIP-712 hash is computed over the `fee.feePayer` field (the attacker's address) and the `msgs[].value` fields (which contain the victim's address as the message signer). The attacker signs this hash with their own key. The ante handler verifies the signature against `feePayer` (attacker's address) — which passes — but never asserts that `feePayer == msgs[0].GetSigners()[0]` (the victim's address).

The Cosmos SDK message handler then executes the message using the signer declared inside the message itself (the victim), not the `feePayer`.

**The critical missing check** is:

```go
// This check is ABSENT from VerifySignature:
msgSigners, _, _ := protoCodec.GetMsgV1Signers(msgs[0])
if !feePayer.Equals(sdk.AccAddress(msgSigners[0])) {
    return errorsmod.Wrap(errortypes.ErrorInvalidSigner, "feePayer must match message signer")
}

### Impact Explanation

An unprivileged attacker who holds any valid Cosmos account (even a freshly created one with dust balance) can:

1. Craft a Legacy EIP-712 Cosmos transaction where `msgs` contains a message (e.g., `MsgSend`, `MsgDelegate`, `MsgVote`) with the **victim's address** as the declared signer.
2. Set `ExtensionOptionsWeb3Tx.FeePayer` to the **attacker's own address**.
3. Sign the EIP-712 typed data hash with the **attacker's own private key**.
4. Submit the transaction. The ante handler verifies the attacker's signature against the attacker's `feePayer` address — this passes. The message is then executed by the Cosmos SDK with the victim's address as the authoritative signer.

This enables:
- **Direct theft of funds**: `MsgSend` from victim to attacker, draining the victim's bank balance.
- **Unauthorized staking/governance actions**: delegations, undelegations, votes on behalf of any account.

This matches the allowed Critical impact: *"Unauthorized theft, mint, burn bypass, or balance transfer of EVM-denom or Cosmos bank funds through Ethermint transaction execution."* [5](#0-4) [6](#0-5) 

---

### Likelihood Explanation

The Legacy EIP-712 path (`newLegacyCosmosAnteHandlerEip712`) is still present and active in the codebase, used when a transaction carries an `ExtensionOptionsWeb3Tx` extension option. The attack requires only:
- A valid Cosmos account with any balance (for fees).
- Knowledge of the victim's bech32 address (public information).
- The ability to submit a Cosmos transaction (standard RPC access).

No privileged access, validator compromise, or governance action is required. The attacker constructs the malicious transaction entirely client-side. [7](#0-6) [8](#0-7) 

---

### Recommendation

In `VerifySignature` (`ante/cosmos/eip712.go`), after parsing `feePayer` from `extOpt.FeePayer` and after calling `LegacyValidatePayloadMessages`, add an explicit check that `feePayer` equals the message signer:

```go
msgSigners, _, err := protoCodec.GetMsgV1Signers(msgs[0])
if err != nil {
    return errorsmod.Wrap(err, "failed to get msg signers for feePayer validation")
}
if len(msgSigners) != 1 || !feePayer.Equals(sdk.AccAddress(msgSigners[0])) {
    return errorsmod.Wrap(errortypes.ErrorInvalidSigner,
        "feePayer must match the message signer in EIP-712 legacy transactions")
}
```

This mirrors the check already present in `legacyDecodeAminoSignDoc` and `legacyDecodeProtobufSignDoc` (which derive `feePayer` from the message signer), but those paths are not used by the ante handler — the ante handler accepts an attacker-supplied `feePayer` from the extension option without cross-checking it against the message signer. ### Proof of Concept

**Setup**: Attacker controls account `A` (bech32: `cosmos1attacker...`). Victim is account `V` (bech32: `cosmos1victim...`) with funds.

**Step 1**: Attacker constructs a `MsgSend` with `from_address = cosmos1victim...`, `to_address = cosmos1attacker...`, `amount = victim's full balance`.

**Step 2**: Attacker builds a Legacy EIP-712 Cosmos transaction:
- `body.messages = [MsgSend{from: victim, to: attacker, amount: X}]`
- `body.extension_options = [ExtensionOptionsWeb3Tx{typedDataChainID: N, feePayer: "cosmos1attacker...", feePayerSig: <attacker_sig>}]`
- `auth_info.signer_infos[0].public_key = attacker's pubkey`

**Step 3**: Attacker computes `txBytes = StdSignBytes(chainID, attacker_accNum, attacker_seq, 0, fee, [MsgSend], "")`.

**Step 4**: Attacker calls `LegacyWrapTxToTypedData(codec, chainID, MsgSend, txBytes, FeeDelegationOptions{FeePayer: attacker})` to get the typed data, then signs the EIP-712 hash with attacker's private key → `feePayerSig`.

**Step 5**: Attacker submits the transaction. In `VerifySignature`:
- `pubKey` = attacker's pubkey (from `signerAddrs[0]` = attacker's account).
- `feePayer` = attacker's address (from `extOpt.FeePayer`).
- Signature verification: `recoveredFeePayerAcc == feePayer` ✓, `pubKey.Equals(pk)` ✓.
- **No check that `feePayer == MsgSend.from_address`**.

**Step 6**: Transaction passes ante handler. Cosmos SDK executes `MsgSend` with `from_address = victim` → victim's funds transferred to attacker. [11](#0-10) [4](#0-3)

// LegacyValidatePayloadMessages ensures that the transaction messages can be represented in an EIP-712
// encoding by checking that messages exist, are of the same type, and share a single signer.
func LegacyValidatePayloadMessages(msgs []sdk.Msg) error {
	if len(msgs) == 0 {
		return errors.New("unable to build EIP-712 payload: transaction does contain any messages")
	}

	var msgType string
	var msgSigner sdk.AccAddress

	for i, m := range msgs {
		t, err := getMsgType(m)
		if err != nil {
			return err
		}

		signers, _, err := protoCodec.GetMsgV1Signers(m)
		if err != nil {
			return err
		}
		if len(signers) != 1 {
			return errors.New("unable to build EIP-712 payload: expect exactly 1 signer")
		}

		if i == 0 {
			msgType = t
			msgSigner = signers[0]
			continue
		}

		if t != msgType {
			return errors.New("unable to build EIP-712 payload: different types of messages detected")
		}

		if !msgSigner.Equals(sdk.AccAddress(signers[0])) {
			return errors.New("unable to build EIP-712 payload: multiple signers detected")
		}
	}

	return nil
}

func LegacyWrapTxToTypedData(
	cdc codectypes.AnyUnpacker,
	chainID uint64,
	msg sdk.Msg,
	data []byte,
	feeDelegation *FeeDelegationOptions,
) (apitypes.TypedData, error) {
	value, err := ethermint.SafeInt64(chainID)
	if err != nil {
		return apitypes.TypedData{}, err
	}
	txData := make(map[string]interface{})

	if err := json.Unmarshal(data, &txData); err != nil {
		return apitypes.TypedData{}, errorsmod.Wrap(errortypes.ErrJSONUnmarshal, "failed to JSON unmarshal data")
	}

	domain := apitypes.TypedDataDomain{
		Name:              "Cosmos Web3",
		Version:           "1.0.0",
		ChainId:           math.NewHexOrDecimal256(value),
		VerifyingContract: "cosmos",
		Salt:              "0",
	}

	msgTypes, err := extractMsgTypes(cdc, "MsgValue", msg)
	if err != nil {
		return apitypes.TypedData{}, err
	}

	if feeDelegation != nil {
		feeInfo, ok := txData["fee"].(map[string]interface{})
		if !ok {
			return apitypes.TypedData{}, errorsmod.Wrap(errortypes.ErrInvalidType, "cannot parse fee from tx data")
		}

		feeInfo["feePayer"] = feeDelegation.FeePayer.String()

		// also patching msgTypes to include feePayer
		msgTypes["Fee"] = []apitypes.Type{
			{Name: "feePayer", Type: "string"},
			{Name: "amount", Type: "Coin[]"},
			{Name: "gas", Type: "string"},
		}
	}

	typedData := apitypes.TypedData{
		Types:       msgTypes,
		PrimaryType: "Tx",
		Domain:      domain,
		Message:     txData,
	}

	return typedData, nil
}
