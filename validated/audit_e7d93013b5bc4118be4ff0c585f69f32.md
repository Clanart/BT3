### Title
Hardcoded Validator Signing Key in Production `create_signature_manager()` Allows Precommit Vote Forgery - (File: `crates/apollo_signature_manager/src/lib.rs`)

### Summary

`LocalKeyStore::new_for_testing()` embeds a hardcoded ECDSA private key directly in the production binary. The public factory function `create_signature_manager()` — used by `apollo_node` to instantiate the live signing component — unconditionally calls this testing constructor. Every node that ships this binary therefore uses the same well-known private key to sign BFT precommit votes over block hashes, allowing any reader of the source to forge those signatures.

### Finding Description

`LocalKeyStore::new_for_testing()` in `crates/apollo_signature_manager/src/signature_manager.rs` hard-codes a Stark-curve private key:

```rust
pub(crate) const fn new_for_testing() -> Self {
    const PRIVATE_KEY: PrivateKey = PrivateKey(Felt::from_hex_unchecked(
        "0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133",
    ));
    ...
}
``` [1](#0-0) 

The function carries no `#[cfg(test)]` gate. `LocalKeyStoreSignatureManager::new()` in `lib.rs` calls it unconditionally:

```rust
impl LocalKeyStoreSignatureManager {
    pub fn new() -> Self {
        Self(GenericSignatureManager::new(LocalKeyStore::new_for_testing()))
    }
}
``` [2](#0-1) 

The public production entry-point `create_signature_manager()` delegates directly to this constructor, with a TODO acknowledging the key store is not yet production-ready:

```rust
// TODO(Elin): understand how key store would look in production ...
pub fn create_signature_manager() -> SignatureManager {
    SignatureManager::new()
}
``` [3](#0-2) 

`create_signature_manager` is wired into the live node in `crates/apollo_node/src/components.rs` (2 call-sites). The `SignatureManager` produced here is the component that calls `sign_precommit_vote(block_hash)` to produce the ECDSA signature attached to every BFT precommit vote:

```rust
pub async fn sign_precommit_vote(
    &self,
    block_hash: BlockHash,
) -> SignatureManagerResult<RawSignature> {
    let message_digest = build_precommit_vote_message_digest(block_hash);
    self.sign(message_digest).await
}
``` [4](#0-3) 

Because every node running this binary shares the identical private key, the corresponding public key `0x125d56b1fbba593f1dd215b7c55e384acd838cad549c4a2b9c6d32d264f4e2a` is the registered validator identity for all of them. [5](#0-4) 

### Impact Explanation

An attacker who reads the open-source repository (or disassembles the shipped binary) obtains the private key at zero cost. They can then:

1. Compute `sign(PRIVATE_KEY, H("PRECOMMIT_VOTE" || block_hash))` for any `block_hash` of their choice.
2. Because all nodes share the same key, the attacker can produce valid-looking precommit signatures attributed to every validator simultaneously.
3. Injecting 2/3+ forged precommit votes for a crafted block hash causes `upon_decision` in the BFT state machine to fire and emit `SMRequest::DecisionReached`, committing a block the honest nodes never built. [6](#0-5) 

The committed block hash propagates through `set_global_root_and_block_hash` into storage, poisoning the canonical chain state root and every downstream proof, sync, and RPC result derived from it.

**Matching impact:** *High — Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload.* The precommit signature logic binds the wrong signer (a publicly known test key) to every block-hash vote, enabling an unprivileged network peer to forge authoritative consensus signatures.

### Likelihood Explanation

The private key is a compile-time constant in a public repository. No special access, privileged position, or cryptographic attack is required. Any party who clones the repository or inspects the binary can extract it immediately. The `create_signature_manager` path is the only production path; there is no runtime override.

### Recommendation

1. Remove the hardcoded key from `new_for_testing()` and gate the entire function with `#[cfg(any(test, feature = "testing"))]`.
2. Implement a production `LocalKeyStore::new(private_key: PrivateKey)` constructor that reads the key from a secrets manager, environment variable, or encrypted key file at startup.
3. Replace the `create_signature_manager()` body with a constructor that requires an explicit `PrivateKey` argument, making it a compile error to instantiate the production component without supplying a real key.
4. Add a CI lint that rejects any non-test call-site of `new_for_testing`.

### Proof of Concept

```
# 1. Extract the key from the source (or binary strings):
PRIVATE_KEY=0x608bf2cdb1ad4138e72d2f82b8c5db9fa182d1883868ae582ed373429b7a133

# 2. For any target block hash B, compute the precommit message digest:
#    digest = blake2s("PRECOMMIT_VOTE" || B.to_bytes_be())

# 3. Sign with the known key:
#    (r, s) = ecdsa_sign(PRIVATE_KEY, digest)

# 4. Broadcast Vote { vote_type: Precommit, proposal_commitment: B,
#    voter: <any registered validator id>, signature: (r,s) }
#    Repeat for 2/3+ distinct voter IDs (all share the same key).

# 5. The BFT state machine receives quorum, fires upon_decision(round),
#    and emits DecisionReached — committing block B to storage.
```

The forged votes pass `verify_precommit_vote_signature` because the public key stored for every validator is the one derived from the leaked private key. [7](#0-6)
