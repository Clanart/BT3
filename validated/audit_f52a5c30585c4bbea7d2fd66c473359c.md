### Title
Missing Declared-Class Validation in `execute_replace_class` Enables Permanent Contract Bricking — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary
The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary felt as a replacement class hash without verifying that the hash corresponds to a declared contract class. A contract can therefore replace its own class with an undeclared hash, permanently preventing any future successful execution against that contract and freezing all funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` reads the requested class hash directly from the syscall request and writes it into the contract state with no validation:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The developer-acknowledged `TODO` comment on line 898 confirms this check is intentionally absent from the current implementation. [2](#0-1) 

Contrast this with `execute_declare_transaction`, which enforces `assert_not_zero(compiled_class_hash)` and explicitly records the class in `contract_class_changes` before the class hash is considered valid: [3](#0-2) 

Because the OS is a Cairo program that generates ZK proofs, when a subsequent transaction attempts to call a contract whose class hash has no corresponding compiled class in the `compiled_class_facts_bundle`, the OS hint resolution fails and no valid proof can be produced for any block containing such a call. The contract becomes permanently non-executable at the proof level — not merely reverting, but making proof generation impossible for any interaction with it.

---

### Impact Explanation

Any contract that calls `replace_class` with an undeclared class hash (including `0`, or any random felt) will be permanently bricked. All funds held by that contract become permanently inaccessible: no valid proof can be generated for a block that includes a call to the contract, so the sequencer is forced to exclude all such transactions indefinitely. This constitutes **Critical — Permanent freezing of funds**.

The OS is the last enforcement layer. Without this check, there is no protocol-level guarantee preventing a contract from entering an irrecoverable state.

---

### Likelihood Explanation

**Medium.** The `replace_class` syscall is reachable by any contract without privilege. Two realistic paths exist:

1. **Malicious deployer**: An attacker deploys a contract that accepts user deposits (e.e., a fake yield vault), accumulates funds, then calls `replace_class(0)` or any undeclared hash. User funds are permanently frozen — a more severe variant of a rug-pull because even the attacker cannot recover the funds.
2. **Vulnerable existing contract**: A legitimate contract with a logic flaw that allows an attacker to trigger `replace_class` with an attacker-controlled argument. The OS provides no safety net.

No privileged role, leaked key, or external dependency is required. The entry path is a standard user-submitted transaction.

---

### Recommendation

Before writing the new class hash to `contract_state_changes`, verify that `class_hash` exists as a key in `contract_class_changes` (i.e., it was declared in the current or a prior block). The check should be enforced at the OS level, analogous to how `execute_declare_transaction` enforces `assert_not_zero(compiled_class_hash)` and records the class before it is considered valid.

---

### Proof of Concept

1. Attacker deploys a contract `VaultContract` that accepts ETH/token deposits from users.
2. Users deposit funds; `VaultContract` accumulates a balance.
3. Attacker submits a transaction that calls `replace_class(0xdeadbeef)` (or any felt not present in `contract_class_changes`) from within `VaultContract`.
4. `execute_replace_class` in the OS accepts the call, writes `class_hash=0xdeadbeef` into `contract_state_changes` for `VaultContract`'s address. No validation occurs. [4](#0-3) 
5. In any subsequent block, any transaction calling `VaultContract` causes the OS to attempt to resolve compiled class facts for `0xdeadbeef`. The hint `GetContractAddressStateEntry` / class lookup fails because no compiled class with that hash exists in `compiled_class_facts_bundle`. [5](#0-4) 
6. No valid proof can be generated for any block containing a call to `VaultContract`. The sequencer must permanently exclude all such transactions.
7. All user funds in `VaultContract` are permanently frozen with no recovery path.

func execute_replace_class{
    range_check_ptr,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;
