### Title
Signed Integer Overflow in `SuggestGasTipCap` Returns Incorrect `eth_gasPrice` / `eth_maxPriorityFeePerGas` - (File: `rpc/backend/chain_info.go`)

### Summary
`SuggestGasTipCap` in `rpc/backend/chain_info.go` performs a raw `int64` multiplication on `baseFee.Int64()` without overflow protection. When `baseFee` exceeds `math.MaxInt64`, `big.Int.Int64()` silently returns the low-order 64 bits as a signed value, and the subsequent multiplication can overflow `int64`. The only guard (`if maxDelta < 0`) catches negative results but not positive wrap-arounds. The corrupted value is returned directly to callers of `eth_gasPrice` and `eth_maxPriorityFeePerGas`.

### Finding Description

In `rpc/backend/chain_info.go`, `SuggestGasTipCap` computes the maximum expected base-fee delta using native `int64` arithmetic:

```go
maxDelta := baseFee.Int64() * (int64(params.Params.ElasticityMultiplier) - 1) / int64(params.Params.BaseFeeChangeDenominator)
if maxDelta < 0 {
    maxDelta = 0
}
return big.NewInt(maxDelta), nil
``` [1](#0-0) 

Two distinct overflow paths exist:

1. **`baseFee.Int64()` silent truncation**: Go's `big.Int.Int64()` returns the low-order 64 bits as a signed `int64` when the value exceeds `math.MaxInt64`. No error is returned and no check is performed before the call.

2. **Multiplication overflow**: Even when `baseFee.Int64()` returns a valid positive `int64`, the product `baseFee.Int64() * (int64(ElasticityMultiplier) - 1)` can overflow. For example, with `baseFee = 5×10¹⁸` (below `math.MaxInt64 ≈ 9.2×10¹⁸`) and `ElasticityMultiplier = 3`, the product `10¹⁹` exceeds `math.MaxInt64` and wraps to a large negative or incorrect positive value.

The `if maxDelta < 0` guard only catches the case where the overflow produces a negative result. A positive wrap-around passes through unchecked and is returned as the suggested tip cap.

This result is consumed by two public JSON-RPC endpoints:

- `eth_maxPriorityFeePerGas` → `SuggestGasTipCap` directly
- `eth_gasPrice` → `GasPrice()` → `SuggestGasTipCap`, then adds the result to `baseFee` [2](#0-1) [3](#0-2) 

### Impact Explanation

When `baseFee` is large enough to trigger the overflow, `eth_gasPrice` returns a wildly inflated gas price. Clients (wallets, relayers, automated bots) that rely on `eth_gasPrice` to construct transactions will embed an overflowed gas price into their transactions. Depending on the direction of the overflow:

- **Positive wrap-around**: `eth_gasPrice` returns an astronomically large value. Clients submitting transactions with this price will have their funds drained by fees far beyond the actual cost, constituting a mis-accounting of user funds via the fee path.
- **Negative wrap-around clamped to 0**: `eth_gasPrice` returns only `baseFee` with zero tip. Transactions submitted with this price may be rejected by the ante handler if the effective price falls below `MinGasPrice`, silently breaking transaction submission for all clients relying on the RPC node.

This matches the allowed High impact: **"Public JSON-RPC path feeds incorrect consensus-critical data into transaction execution"** — specifically, the gas price fed into transaction construction is corrupted, causing fee mis-accounting or invalid transaction admission failures.

### Likelihood Explanation

The base fee is stored as `sdkmath.Int` (an arbitrary-precision `big.Int`) and can be set to any value via governance `MsgUpdateParams`. A governance proposal setting `BaseFee` above `math.MaxInt64 / (ElasticityMultiplier - 1)` immediately triggers the overflow on the next call to `SuggestGasTipCap`. Additionally, through sustained full-block conditions, the EIP-1559 algorithm can organically grow the base fee toward this threshold over time. The RPC endpoint is public and requires no authentication, so any client querying `eth_gasPrice` or `eth_maxPriorityFeePerGas` after the threshold is crossed receives the corrupted value.

### Recommendation

Replace the raw `int64` arithmetic with `big.Int` arithmetic throughout `SuggestGasTipCap`:

```go
elasticity := new(big.Int).SetInt64(int64(params.Params.ElasticityMultiplier) - 1)
denom := new(big.Int).SetInt64(int64(params.Params.BaseFeeChangeDenominator))
maxDelta := new(big.Int).Mul(baseFee, elasticity)
maxDelta.Div(maxDelta, denom)
if maxDelta.Sign() < 0 {
    maxDelta.SetInt64(0)
}
return maxDelta, nil
```

This eliminates both the `Int64()` truncation and the multiplication overflow.

### Proof of Concept

1. Via governance, set `BaseFee` to `6 × 10¹⁸` (valid `sdkmath.Int`, below `math.MaxUint64` but above `math.MaxInt64 / 2`).
2. With default `ElasticityMultiplier = 2`, `BaseFeeChangeDenominator = 8`:
   - `baseFee.Int64()` returns `6×10¹⁸` (fits in `int64` since `6×10¹⁸ < 9.2×10¹⁸`).
   - `6×10¹⁸ * (2-1) = 6×10¹⁸` — no overflow here, but with `ElasticityMultiplier = 3`: `6×10¹⁸ * 2 = 1.2×10¹⁹ > math.MaxInt64`, wrapping to a large negative value, clamped to 0.
3. With `ElasticityMultiplier = 3` and `BaseFee = 5×10¹⁸`: `5×10¹⁸ * 2 = 10¹⁹`, which overflows `int64` (max `≈ 9.2×10¹⁸`) and wraps to `10¹⁹ - 2^64 ≈ -8.4×10¹⁸` → clamped to 0.
4. Call `eth_maxPriorityFeePerGas` — returns `0x0` instead of the correct `~6.25×10¹⁷`.
5. Call `eth_gasPrice` — returns only `baseFee` with no tip, causing all client-submitted EIP-1559 transactions to carry zero priority fee, potentially falling below `MinGasPrice` and being rejected. [4](#0-3)

func (b *Backend) SuggestGasTipCap(baseFee *big.Int) (*big.Int, error) {
	if baseFee == nil {
		// london hardfork not enabled or feemarket not enabled
		return big.NewInt(0), nil
	}

	params, err := b.queryClient.FeeMarket.Params(b.ctx, &feemarkettypes.QueryParamsRequest{})
	if err != nil {
		return nil, err
	}
	// calculate the maximum base fee delta in current block, assuming all block gas limit is consumed
	// ```
	// GasTarget = GasLimit / ElasticityMultiplier
	// Delta = BaseFee * (GasUsed - GasTarget) / GasTarget / Denominator
	// ```
	// The delta is at maximum when `GasUsed` is equal to `GasLimit`, which is:
	// ```
	// MaxDelta = BaseFee * (GasLimit - GasLimit / ElasticityMultiplier) / (GasLimit / ElasticityMultiplier) / Denominator
	//          = BaseFee * (ElasticityMultiplier - 1) / Denominator
	// ```
	maxDelta := baseFee.Int64() * (int64(params.Params.ElasticityMultiplier) - 1) / int64(params.Params.BaseFeeChangeDenominator)
	if maxDelta < 0 {
		// impossible if the parameter validation passed.
		maxDelta = 0
	}
	return big.NewInt(maxDelta), nil
```
