### Title
Unbounded `req.Txs` / `req.Predecessors` Iteration in `TraceBlock` and `TraceTx` Enables Resource-Exhaustion DoS via Public gRPC/JSON-RPC - (File: `x/evm/keeper/grpc_query.go`)

### Summary
The `TraceBlock` and `TraceTx` gRPC handlers iterate over caller-supplied arrays (`req.Txs` and `req.Predecessors` respectively) with no size limit. Each element triggers a full EVM state transition via `ApplyMessageWithConfig`. An unprivileged caller can submit a single request containing thousands of high-gas transactions, causing unbounded CPU and memory consumption on the node. If a validator co-hosts the JSON-RPC server, this can starve the consensus engine of resources and cause missed blocks or consensus failure.

### Finding Description

**`TraceBlock`** iterates over `req.Txs` — a `repeated MsgEthereumTx` protobuf field — with no upper-bound check:

```go
// x/evm/keeper/grpc_query.go:619-644
txsLength := len(req.Txs)
results := make([]*types.TxTraceResult, 0, txsLength)

for i, tx := range req.Txs {          // ← no size limit
    ...
    traceResult, logIndex, err := k.prepareTrace(ctx, cfg, msg, req.TraceConfig, true)
    ...
}
```

`prepareTrace` calls `ApplyMessageWithConfig`, which executes a full EVM state transition for every element. [1](#0-0) 

**`TraceTx`** iterates over `req.Predecessors` — also a `repeated MsgEthereumTx` — with no upper-bound check:

```go
// x/evm/keeper/grpc_query.go:542-563
for i, tx := range req.Predecessors {   // ← no size limit
    ...
    rsp, err := k.ApplyMessageWithConfig(ctx, msg, cfg, true)
    ...
}
``` [2](#0-1) 

The proto definition confirms both fields are unbounded `repeated` arrays: [3](#0-2) [4](#0-3) 

By contrast, `SimulateV1` correctly enforces `MaxSimulateBlocks = 256` and a cross-block `GasBudget`: [5](#0-4) [6](#0-5) 

No equivalent guard exists for `TraceBlock` or `TraceTx`.

### Impact Explanation

Each element in `req.Txs` / `req.Predecessors` causes a full EVM execution (including storage reads/writes, gas accounting, and state-DB operations). A single malicious request with N=10,000 high-gas transactions can monopolize the node's goroutine for seconds to minutes. Because Cosmos SDK validators commonly expose the JSON-RPC server in the same process as the consensus engine, this resource exhaustion can cause the validator to miss `propose`/`prevote`/`precommit` deadlines, leading to consensus timeouts and potential chain liveness failure. This matches the allowed High impact: *"Public JSON-RPC, gRPC, simulation, tracing, receipt/log, or indexer path … exposes a reachable route to the impacts above."*

### Likelihood Explanation

Both `debug_traceBlockByNumber` and `debug_traceTransaction` are standard Ethereum JSON-RPC methods exposed publicly. No authentication is required. The attacker only needs to craft a JSON-RPC request with a large `txs` / `predecessors` array. The `Backend.TraceBlock` path collects all Ethereum messages from a block and forwards them directly to the keeper: [7](#0-6) 

The `Backend.TraceTransaction` path collects all predecessor messages from the same block and forwards them: [8](#0-7) 

An attacker can also call the gRPC endpoint directly with an arbitrarily crafted `QueryTraceBlockRequest` or `QueryTraceTxRequest`.

### Recommendation

1. Add a maximum size check on `req.Txs` in `TraceBlock` and on `req.Predecessors` in `TraceTx`, analogous to the `MaxSimulateBlocks` guard in `SimulateV1`.
2. Introduce a cross-request gas budget (similar to `GasBudget` in the simulator) that caps the total EVM gas that can be consumed by a single trace request.
3. Consider running the JSON-RPC server in a separate process from the consensus engine to isolate resource exhaustion.

### Proof of Concept

**`TraceBlock` DoS:**
```bash
# Craft a QueryTraceBlockRequest with 10,000 identical high-gas MsgEthereumTx entries
# and send it directly to the gRPC endpoint:
grpcurl -d '{"txs": [<10000 copies of a high-gas MsgEthereumTx>], "block_number": 1}' \
  localhost:9090 ethermint.evm.v1.Query/TraceBlock
# The node will execute 10,000 full EVM state transitions synchronously,
# blocking the goroutine for an extended period.
```

**`TraceTx` DoS:**
```bash
# Craft a QueryTraceTxRequest with 10,000 predecessor transactions:
grpcurl -d '{"msg": <valid MsgEthereumTx>, "predecessors": [<10000 entries>], "block_number": 1}' \
  localhost:9090 ethermint.evm.v1.Query/TraceTx
# Each predecessor triggers ApplyMessageWithConfig, causing unbounded execution.
```

The root cause is the absence of any size limit on `req.Txs` at `x/evm/keeper/grpc_query.go:619` and on `req.Predecessors` at `x/evm/keeper/grpc_query.go:542`, both of which are reachable from the public `debug_traceBlockByNumber` and `debug_traceTransaction` JSON-RPC methods.

func (b *Backend) TraceBlock(height rpctypes.BlockNumber,
	config *rpctypes.TraceConfig,
	block *tmrpctypes.ResultBlock,
) ([]*evmtypes.TxTraceResult, error) {
	txs := block.Block.Txs
	txsLength := len(txs)

	if txsLength == 0 {
		// If there are no transactions return empty array
		return []*evmtypes.TxTraceResult{}, nil
	}
	blockRes, err := b.TendermintBlockResultByNumber(&block.Block.Height)
	if err != nil {
		b.logger.Debug("block result not found", "height", block.Block.Height, "error", err.Error())
		return nil, nil
	}
	txDecoder := b.clientCtx.TxConfig.TxDecoder()

	var txsMessages []*evmtypes.MsgEthereumTx
	for i, tx := range txs {
		if !rpctypes.TxSuccessOrExceedsBlockGasLimit(blockRes.TxsResults[i]) {
			b.logger.Debug("invalid tx result code", "cosmos-hash", hexutil.Encode(tx.Hash()))
			continue
		}
		decodedTx, err := txDecoder(tx)
		if err != nil {
			b.logger.Error("failed to decode transaction", "hash", txs[i].Hash(), "error", err.Error())
			continue
		}

		for _, msg := range decodedTx.GetMsgs() {
			ethMessage, ok := msg.(*evmtypes.MsgEthereumTx)
			if !ok {
				// Just considers Ethereum transactions
				continue
			}
			txsMessages = append(txsMessages, ethMessage)
		}
	}

	// minus one to get the context at the beginning of the block
	contextHeight := height - 1
	if contextHeight < 1 {
		// 0 is a special value for `ContextWithHeight`.
		contextHeight = 1
	}
	ctxWithHeight := rpctypes.ContextWithHeight(int64(contextHeight))

	traceBlockRequest := &evmtypes.QueryTraceBlockRequest{
		Txs:             txsMessages,
		TraceConfig:     b.convertConfig(config),
		BlockNumber:     block.Block.Height,
		BlockTime:       block.Block.Time,
		BlockHash:       common.Bytes2Hex(block.BlockID.Hash),
		ProposerAddress: sdk.ConsAddress(block.Block.ProposerAddress),
		ChainId:         b.chainID.Int64(),
	}

	res, err := b.queryClient.TraceBlock(ctxWithHeight, traceBlockRequest)
