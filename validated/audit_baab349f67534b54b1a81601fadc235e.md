### Title
Stale L1 Gas Price Fallback Without Freshness Validation Causes Wrong Block Header Gas Prices to Pass Proposal Validation — (`crates/apollo_consensus_orchestrator/src/utils.rs`)

### Summary

`get_l1_prices_in_fri_and_wei_and_conversion_rate()` silently falls back to `previous_block_info` (the L1 gas prices from the last committed block) when the L1 gas price provider or ETH-to-STRK oracle fails, without any check on how old that info is. This stale reference is then used by `is_block_info_valid()` to validate a proposer's claimed L1 gas prices. Because both proposer and validator fall back to the same unboundedly-stale reference, a block embedding arbitrarily outdated L1 gas prices passes the margin check and is committed to the chain, causing wrong fee accounting for every transaction in that block and propagating the stale prices to all subsequent blocks.

### Finding Description

`get_l1_prices_in_fri_and_wei_and_conversion_rate()` implements a three-tier fallback:

1. Fresh oracle data (correct path)
2. `previous_block_info` — prices from the last committed block, **no staleness check**
3. Hardcoded `DEFAULT_ETH_TO_FRI_RATE` / `min_l1_gas_price_wei` [1](#0-0) 

When `get_price_info()` returns `StaleL1GasPricesError` (its own staleness guard fires) or the oracle returns any error, execution falls through to the `previous_block_info` branch. The code extracts `l1_prices_fri` and `l1_prices_wei` directly from `block_info` and returns them as the current reference price: [2](#0-1) 

There is no comparison of `block_info.timestamp` against the current wall-clock time. If the oracle has been failing for N consecutive blocks, the reference price is N blocks old, with no upper bound enforced.

This stale reference is consumed by `is_block_info_valid()` in the proposal validator: [3](#0-2) 

The validator accepts the proposal if the proposer's claimed L1 gas price is within `l1_gas_price_margin_percent` (10%) of the stale reference: [4](#0-3) 

Because both the proposer (`initiate_build()`) and every validator call the same `get_l1_prices_in_fri_and_wei` with the same `previous_block_info`, they converge on the same stale value. The 10% margin check passes, and the block is committed with stale L1 gas prices embedded in its header.

The same stale prices then become the new `previous_block_info` for the next block, propagating the error indefinitely until the oracle recovers.

The `L1GasPriceProvider.get_price_info()` does have its own staleness guard: [5](#0-4) 

But this guard only prevents the provider from returning stale L1 data; it does not prevent the orchestrator from using the equally-stale `previous_block_info` as a substitute. The `EthToStrkOracleClient.eth_to_fri_rate()` has an analogous silent fallback — it returns the cached rate from `quantized_timestamp - 1` without any age check when the current query is in-flight: [6](#0-5) 

### Impact Explanation

The stale L1 gas prices are written into `ProposalInit.l1_gas_price_fri` / `l1_data_gas_price_fri` and forwarded to `convert_to_sn_api_block_info()`, which constructs the `BlockInfo` passed to the blockifier: [7](#0-6) 

Every transaction executed in that block has its L1 fee calculated against the wrong gas price. If L1 gas prices have risen significantly during the oracle outage, users underpay; if they have fallen, users overpay. The committed block header contains the wrong authoritative value, which RPC fee-estimation endpoints will serve to clients.

### Likelihood Explanation

The trigger is an L1 gas price provider or oracle failure lasting more than one block. This can occur naturally (network partition, oracle API downtime) and does not require any privileged action. The `oracle_fails_on_second_block` test confirms the fallback is reachable in practice: [8](#0-7) 

### Recommendation

Add a staleness check on `previous_block_info.timestamp` before using it as a fallback reference. If the previous block's timestamp is older than `max_time_gap_seconds` relative to the current proposal timestamp, skip the `previous_block_info` fallback and proceed to the default minimum values (or return an error). This mirrors the existing guard in `get_price_info()` and closes the gap between the two staleness policies.

### Proof of Concept

1. The L1 gas price scraper stops updating (e.g., L1 RPC outage). After `max_time_gap_seconds`, `get_price_info()` returns `StaleL1GasPricesError`.
2. `get_l1_prices_in_fri_and_wei_and_conversion_rate()` falls through to `previous_block_info`, returning the prices from the last committed block — no age check performed.
3. The proposer builds `ProposalInit` with those stale prices. The validator calls `is_block_info_valid()`, which calls the same function and obtains the same stale reference.
4. `within_margin(proposed, stale_reference, 10%)` passes. The block is accepted and committed with stale `l1_gas_price_fri` / `l1_data_gas_price_fri` in its header.
5. The committed block's prices become the new `previous_block_info`. For every subsequent block while the oracle is down, the stale prices propagate one block further, with no mechanism to detect or bound the drift.
6. All transactions in affected blocks have their L1 fees calculated against the wrong price, producing incorrect fee accounting with direct economic impact on users and the sequencer.

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let previous_block_info = PreviousBlockInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)

                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {

                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })

    // attempt to build_proposal on block number 1.
    deps.setup_deps_for_validate(SetupDepsArgs::default());
    deps.setup_deps_for_build(SetupDepsArgs { start_block_number: HEIGHT_1, ..Default::default() });

    // set up batcher decision_reached
    deps.batcher.expect_decision_reached().times(1).return_once(|_| {
        Ok(DecisionReachedResponse {
            state_diff: ThinStateDiff::default(),
            central_objects: CentralObjects::default(),
        })
    });

    deps.state_sync_client.expect_add_new_block().times(1).return_once(|_| Ok(()));
