### Title
Gas Refund "Push" Transfer Failure Causes Irrecoverable Fee Loss After EVM Execution - (File: x/evm/keeper/gas.go, x/evm/keeper/state_transition.go)

### Summary

`ApplyTransaction` uses a "push" pattern to return leftover gas to the sender via `RefundGas` after EVM execution. If this bank transfer fails, `ApplyTransaction` returns a cosmos-level error. The Cosmos SDK then reverts the entire message-handler cache context (including committed EVM state), but the ante-handler fee deduction â€” which was committed to the outer context before message execution â€” is **not** reverted. The sender loses their full gas fee with no execution taking effect.

### Finding Description

In `ApplyTransaction`, the execution flow is:

1. **Ante handler** (outer context, committed before message handler): `CheckEthGasConsume` â†’ `DeductTxCostsFromUserBalance` deducts `gasLimit Ă— effectiveGasPrice` from the sender and deposits it into the fee collector.

2. **Message handler** (wrapped in SDK cache context): `ApplyMessageWithConfig` runs the EVM, commits the `StateDB`, and (if hooks are set) calls `commit()` to flush the cache context into `ctx`.

3. **`RefundGas`** is then called on `ctx` to push `leftoverGas Ă— gasPrice` back to the sender from the fee collector:

```go
// x/evm/keeper/state_transition.go:252-254
if err = k.RefundGas(ctx, msg, leftoverGas, cfg.Params.EvmDenom); err != nil {
    return nil, errorsmod.Wrapf(err, "failed to refund leftover gas to sender %s", msg.From)
}
```

Inside `RefundGasWithPrice`:

```go
// x/evm/keeper/gas.go:78-82
err := k.bankKeeper.SendCoinsFromModuleToAccountVirtual(ctx, authtypes.FeeCollectorName, msg.From.Bytes(), refundedCoins)
if err != nil {
    err = errorsmod.Wrapf(errortypes.ErrInsufficientFunds, "fee collector account failed to refund fees: %s", err.Error())
    return errorsmod.Wrapf(err, "failed to refund %d leftover gas (%s)", leftoverGas, refundedCoins.String())
}
```

If `SendCoinsFromModuleToAccountVirtual` returns an error, `RefundGas` returns an error, `ApplyTransaction` returns `nil, err`, and `EthereumTx` propagates it:

```go
// x/evm/keeper/msg_server.go:58-61
evmResult, err := k.ApplyTransaction(ctx, msg)
if err != nil {
    return nil, errorsmod.Wrap(err, "failed to apply transaction")
}
```

The SDK's `RunMsgs` discards the message-handler cache context on error, reverting all EVM state changes. However, the ante-handler fee deduction â€” committed to the outer context â€” is **not** rolled back.

**Failure conditions for `SendCoinsFromModuleToAccountVirtual`:**
- The fee collector module account holds insufficient balance (e.g., due to concurrent fee distribution or an accounting inconsistency).
- The recipient address (`msg.From`) is a blocked address in the bank module (e.g., if an EVM address collides with a Cosmos module account address, since both are 20-byte addresses on EVM-compatible chains).

The `BankKeeper` interface explicitly exposes `BlockedAddr(addr sdk.AccAddress) bool`, confirming that blocked-address checks are enforced:

```go
// x/evm/types/interfaces.go:54

```

### Impact Explanation

A user submits a valid EVM transaction. The ante handler deducts the full gas fee. The EVM executes successfully. `RefundGas` fails (e.g., sender address is blocked, or fee collector is transiently underfunded). The cosmos-level error causes the SDK to revert all EVM state changes. The user's gas fee is permanently lost â€” they paid `gasLimit Ă— effectiveGasPrice` but received no execution and no refund. This is a direct mis-accounting of valid user funds.

This matches the allowed High impact: **"EVM state transition, gas refund, fee market, ante handler, mempool, or proposal handling bug that permits â€¦ valid user funds/fees to be mis-accounted."**

### Likelihood Explanation

The likelihood is low-to-moderate:
- **Blocked address collision**: On EVM-compatible Cosmos chains, EVM addresses and Cosmos `sdk.AccAddress` share the same 20-byte space. A user whose EVM address maps to a Cosmos module account address (which is blocked) would trigger this on every transaction. While rare, it is deterministic and reproducible.
- **Fee collector underfunding**: Less likely in normal operation, but possible under edge conditions (e.g., fee distribution running concurrently, or a bug in the virtual accounting layer).
- The path is reachable by any unprivileged user submitting a standard `MsgEthereumTx`.

### Recommendation

Wrap `RefundGas` so that failure is treated as a VM-level error (surfaced in `res.VmError`) rather than a cosmos-level error, consistent with how `PostTxProcessing` hook failures are handled:

```go
// Instead of returning a cosmos-level error:
if err = k.RefundGas(ctx, msg, leftoverGas, cfg.Params.EvmDenom); err != nil {
    // Treat as a non-fatal accounting failure; log and continue.
    // The user forfeits the refund but the EVM state is preserved.
    k.Logger(ctx).Error("failed to refund gas", "error", err, "sender", msg.From)
}
```

Alternatively, validate that `msg.From` is not a blocked address before accepting the transaction in the ante handler, so the condition is caught before fee deduction.

### Proof of Concept

1. Identify or craft an EVM sender address that maps to a Cosmos blocked address (e.g., a module account address).
2. Submit a `MsgEthereumTx` from that address with `gasLimit > intrinsicGas` (so `leftoverGas > 0` and a refund is attempted).
3. The ante handler deducts `gasLimit Ă— gasPrice` from the sender.
4. The EVM executes successfully; `stateDB.Commit()` succeeds.
5. `RefundGas` calls `SendCoinsFromModuleToAccountVirtual(feeCollector â†’ blockedAddr)`, which fails.
6. `ApplyTransaction` returns a cosmos-level error.
7. `EthereumTx` returns an error; the SDK reverts the message-handler cache context.

```go
// RefundGasWithPrice transfers the leftover gas to sender using the provided gas price.
func (k *Keeper) RefundGasWithPrice(
	ctx sdk.Context,
	msg *core.Message,
	leftoverGas uint64,
	gasPrice *big.Int,
	denom string,
) error {
	if gasPrice == nil {
		gasPrice = new(big.Int)
	}

	// Return EVM tokens for remaining gas, exchanged at the original rate.
	remaining := new(big.Int).Mul(new(big.Int).SetUint64(leftoverGas), gasPrice)

	switch remaining.Sign() {
	case -1:
		// negative refund errors
		return errorsmod.Wrapf(types.ErrInvalidRefund, "refunded amount value cannot be negative %d", remaining.Int64())
	case 1:
		// positive amount refund
		refundedCoins := sdk.Coins{sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(remaining))}

		// refund to sender from the fee collector module account, which is the escrow account in charge of collecting tx fees
		err := k.bankKeeper.SendCoinsFromModuleToAccountVirtual(ctx, authtypes.FeeCollectorName, msg.From.Bytes(), refundedCoins)
		if err != nil {
			err = errorsmod.Wrapf(errortypes.ErrInsufficientFunds, "fee collector account failed to refund fees: %s", err.Error())
			return errorsmod.Wrapf(err, "failed to refund %d leftover gas (%s)", leftoverGas, refundedCoins.String())
		}
	default:
		// no refund, consume gas and update the tx gas meter
	}

	return nil
}
```
