### Title
Unrestricted `receive()` with No ETH Recovery Mechanism Permanently Locks Ether — (`evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.sol` exposes a bare, unrestricted `receive() external payable {}` that silently accepts ETH from any caller. Unlike ETH deposited through `initTransfer` (which is tracked on-chain and can be released via a signed `finTransfer`), ETH arriving through `receive()` is never recorded in any state variable or event, and no admin rescue or withdrawal function exists anywhere in the contract. That ETH is permanently irrecoverable.

---

### Finding Description

`OmniBridge.sol` line 574 contains:

```solidity
receive() external payable {}
``` [1](#0-0) 

There are zero restrictions on the sender or the amount. Compare this with the only legitimate ETH-inflow path — `initTransfer` with `tokenAddress == address(0)`:

```solidity
if (tokenAddress == address(0)) {
    if (fee != 0) { revert InvalidFee(); }
    extensionValue = msg.value - amount - nativeFee;
}
``` [2](#0-1) 

When ETH enters via `initTransfer`, the `InitTransfer` event is emitted, the NEAR-side bridge observes it, and the MPC can later sign a `finTransfer` to release the corresponding amount to a recipient. ETH that enters via `receive()` emits nothing, is recorded nowhere, and therefore can never be matched to a `finTransfer` payload.

The only ETH-outflow path is `finTransfer` when `payload.tokenAddress == address(0)`:

```solidity
(bool success, ) = payload.recipient.call{value: payload.amount}("");
if (!success) revert FailedToSendEther();
``` [3](#0-2) 

This path requires a valid ECDSA signature from `nearBridgeDerivedAddress` over a specific `(destinationNonce, amount, recipient)` tuple. No such signature will ever be produced for ETH that arrived silently through `receive()`.

A comprehensive search confirms there is no `withdraw`, `rescue`, `recoverETH`, or equivalent admin function anywhere in `OmniBridge.sol` or `OmniBridgeWormhole.sol`. [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent, irrecoverable lock of ETH in the bridge contract.**

Any ETH that reaches the contract via `receive()` — whether sent by a user who mistakenly transfers ETH directly to the bridge address, by a smart-contract integration that calls the bridge with ETH and no calldata, or by a self-destruct/coinbase injection — is frozen forever. There is no admin escape hatch, no rescue function, and no MPC-signed path that can release untracked ETH. This matches the allowed impact: *"Permanent freezing, irrecoverable lock, or unclaimable settlement of user or protocol funds in bridge, token, or vault flows."*

---

### Likelihood Explanation

**Medium.** The bridge contract address is publicly known and handles real ETH. Common scenarios that trigger `receive()` instead of `initTransfer`:

- A user sends ETH directly to the contract address from a wallet (no calldata).
- A smart-contract integration calls the bridge address with ETH but without the correct function selector (e.g., due to an ABI mismatch or upgrade).
- A contract uses `address(bridge).transfer(amount)` or `.send(amount)` expecting the bridge to handle it.
- A `selfdestruct` from another contract forces ETH into the bridge.

None of these require any privileged access.

---

### Recommendation

Remove the bare `receive()` function entirely if there is no legitimate use case for receiving ETH without calldata. If ETH must be receivable from a specific trusted source (e.g., a WETH contract for unwrapping), restrict the sender:

```solidity
// Only accept ETH from the designated WETH contract
receive() external payable {
    if (msg.sender != address(wethToken)) revert UnauthorizedEthSender();
}
```

Additionally, add an admin ETH-rescue function to recover any ETH that arrives through unintended paths:

```solidity
function rescueETH(address payable to, uint256 amount) external onlyRole(DEFAULT_ADMIN_ROLE) {
    (bool ok,) = to.call{value: amount}("");
    require(ok, "transfer failed");
}
```

### Proof of Concept

1. Deploy `OmniBridgeWormhole` (or `OmniBridge`) on a testnet.
2. Send ETH directly to the contract address with no calldata (plain ETH transfer from an EOA or `address(bridge).call{value: 1 ether}("")`).
3. Observe that `receive()` accepts the ETH silently — no event is emitted, no state is updated.
4. Confirm that `finTransfer` cannot release this ETH: any attempt requires a valid MPC signature over a `destinationNonce` that was never registered, so no valid signature exists.
5. Confirm no `withdraw` or rescue function exists — the ETH is permanently locked. [1](#0-0) [5](#0-4)
