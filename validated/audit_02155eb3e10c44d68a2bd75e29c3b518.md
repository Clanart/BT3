### Title
`execute_meta_tx_v0` Syscall Allows Any Contract to Invoke `__execute__` on Arbitrary Accounts Without `__validate__` — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_meta_tx_v0` syscall, callable by any executing contract, directly invokes `__execute__` on an arbitrary target account contract without ever calling `__validate__`. Because StarkNet's account abstraction model places signature verification exclusively in `__validate__`, this allows an unprivileged attacker to execute arbitrary calls on behalf of any victim account — including token transfers — without a valid signature from the account owner.

---

### Finding Description

`execute_meta_tx_v0` is registered as a live syscall in `execute_syscalls.cairo` under `META_TX_V0_SELECTOR`: [1](#0-0) [2](#0-1) 

The implementation in `syscall_impls.cairo` enforces only one constraint on the selector — it must be `EXECUTE_ENTRY_POINT_SELECTOR` (i.e., `__execute__`): [3](#0-2) 

There is **no authorization check** on who the caller is, and **no check** that the caller has any relationship to the target `contract_address`. Any contract in any transaction can invoke this syscall against any target.

The syscall then constructs a synthetic `TxInfo` for the target: [4](#0-3) 

Key properties of this synthetic context:
- `version = 0` — causes `run_validate` to skip validation (see `execute_transaction_utils.cairo` line 128–130)
- `nonce = 0` — hardcoded, no OS-level nonce check is performed for this inner execution
- `signature_start/end` = **attacker-supplied** values from `request`
- `caller_address = ORIGIN_ADDRESS` (address 0) — passes the standard "top-level call" check in most account contracts

The execution is then dispatched directly to `__execute__` via `contract_call_helper` → `select_execute_entry_point_func`: [5](#0-4) 

`__validate__` is **never called** on the target contract. In StarkNet's account abstraction model, `__validate__` is the sole location where signature verification is performed. `__execute__` is designed to trust that `__validate__` has already run. Bypassing `__validate__` therefore bypasses all signature verification for the target account.

The `run_validate` function in `execute_transaction_utils.cairo` confirms that version-0 transactions skip validation entirely: [6](#0-5) 

---

### Impact Explanation

**Critical — Direct loss of funds.**

An attacker can call `meta_tx_v0` from any contract they deploy, targeting any victim account's `__execute__` entry point with arbitrary calldata (e.g., a `transfer` call to the attacker's address). Because `__validate__` is skipped, the victim's signature is never checked. The victim's `__execute__` runs the attacker's calldata under the victim's contract address, draining the victim's token balances. This affects every standard account contract on the network whose `__execute__` does not independently re-verify signatures (which is the universal design assumption for StarkNet accounts).

---

### Likelihood Explanation

**High.** The attack requires only:
1. Deploying a malicious contract (permissionless).
2. Submitting a standard invoke transaction that calls the malicious contract (permissionless).
3. The malicious contract issues the `meta_tx_v0` syscall targeting any victim.

No privileged role, leaked key, or sequencer cooperation is required. The sequencer includes the attacker's transaction as a normal fee-paying invoke. The OS proof system accepts it because `execute_meta_tx_v0` imposes no caller authorization constraint.

---

### Recommendation

1. **Require `__validate__` before `__execute__`**: Before dispatching to `contract_call_helper`, call the target contract's `__validate__` entry point and assert it returns `VALIDATED`. This mirrors the flow in `run_validate` used for top-level invoke transactions.

2. **Bind the meta-tx hash to the outer transaction's sender or a user-provided authorization**: The `compute_meta_tx_v0_hash` currently binds only to `contract_address`, `selector`, `calldata`, and `chain_id` — it does not bind to any authorization from the target account. The hash should commit to a nonce from the target account's state to prevent replay.

3. **Enforce a nonce check**: The OS must read and increment the target contract's nonce for the inner meta-tx execution, just as `check_and_increment_nonce` does for top-level transactions. [7](#0-6) 

---

### Proof of Concept

**Setup:**
- Victim account `V` holds 1000 STRK. `V` uses a standard account contract where `__validate__` verifies the owner's ECDSA signature and `__execute__` runs the provided calls without re-verifying.
- Attacker deploys contract `M` with the following logic in its entry point:

```
// Pseudocode for M's attack function
fn attack(victim: ContractAddress, attacker: ContractAddress) {
    // Construct calldata: transfer(attacker, 1000)
    let calldata = [transfer_selector, attacker, 1000, 0];
    
    // Issue meta_tx_v0 syscall targeting victim's __execute__
    meta_tx_v0(
        contract_address = victim,
        selector = EXECUTE_ENTRY_POINT_SELECTOR,  // __execute__
        calldata = calldata,
        signature = [],  // empty — __validate__ is never called
    );
}
```

**Execution trace in the OS:**

1. Attacker submits `invoke(M.attack(V, attacker))` — a normal, fee-paying transaction. `M.__validate__` and `M.__execute__` run normally.

2. Inside `M.__execute__`, the `meta_tx_v0` syscall is issued. The OS enters `execute_meta_tx_v0` in `syscall_impls.cairo`.

3. The OS constructs `new_tx_info` with `version=0`, `nonce=0`, `account_contract_address=V`, `signature=[]`, and `caller_address=ORIGIN_ADDRESS`.

4. The OS calls `contract_call_helper` → `select_execute_entry_point_func` with `selector=__execute__` on contract `V`. **`V.__validate__` is never called.**

5. `V.__execute__` runs with `caller_address=0` (passes top-level check), `tx_info.version=0` (no resource bounds check), and the attacker's calldata. It executes `transfer(attacker, 1000)` on the fee token contract.

6. 1000 STRK is transferred from `V` to the attacker. The proof is valid; the block is accepted. [8](#0-7)

func execute_meta_tx_v0{
    range_check_ptr,
    syscall_ptr: felt*,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    revert_log: RevertLogEntry*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, caller_execution_context: ExecutionContext*) {
    alloc_locals;

    let request = cast(syscall_ptr + RequestHeader.SIZE, MetaTxV0Request*);
    local calldata_start: felt* = request.calldata_start;
    local calldata_size = request.calldata_end - calldata_start;

    let specific_base_gas_cost = (
        META_TX_V0_GAS_COST + META_TX_V0_CALLDATA_FACTOR_GAS_COST * calldata_size
    );
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=specific_base_gas_cost, request_struct_size=MetaTxV0Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    local contract_address = request.contract_address;
    local selector = request.selector;
    local caller_execution_info: ExecutionInfo* = caller_execution_context.execution_info;
    local old_tx_info: TxInfo* = caller_execution_info.tx_info;

    if (selector != EXECUTE_ENTRY_POINT_SELECTOR) {
        write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
        return ();
    }

    // Sanity check: Verify that `signature` is a valid Sierra array.
    assert_nn_le(request.signature_end - request.signature_start, SIERRA_ARRAY_LEN_BOUND - 1);

    let (state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
        key=contract_address
    );

    // Compute the meta-transaction hash.
    let pedersen_ptr = builtin_ptrs.selectable.pedersen;
    with pedersen_ptr {
        let meta_tx_hash = compute_meta_tx_v0_hash(
            contract_address=contract_address,
            entry_point_selector=selector,
            calldata=calldata_start,
            calldata_size=calldata_size,
            chain_id=old_tx_info.chain_id,
        );
    }
    update_pedersen_in_builtin_ptrs(pedersen_ptr=pedersen_ptr);

    // Prepare execution context.
    tempvar new_tx_info = new TxInfo(
        version=0,
        account_contract_address=contract_address,
        max_fee=0,
        signature_start=request.signature_start,
        signature_end=request.signature_end,
        transaction_hash=meta_tx_hash,
        chain_id=old_tx_info.chain_id,
        nonce=0,
        resource_bounds_start=cast(0, ResourceBounds*),
        resource_bounds_end=cast(0, ResourceBounds*),
        tip=0,
        paymaster_data_start=cast(0, felt*),
        paymaster_data_end=cast(0, felt*),
        nonce_data_availability_mode=0,
        fee_data_availability_mode=0,
        account_deployment_data_start=cast(0, felt*),
        account_deployment_data_end=cast(0, felt*),
        proof_facts_start=cast(0, felt*),
        proof_facts_end=cast(0, felt*),
    );

    let (deprecated_tx_info_ptr: DeprecatedTxInfo*) = alloc();
    tempvar execution_context: ExecutionContext* = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=state_entry.class_hash,
        calldata_size=calldata_size,
        calldata=calldata_start,
        execution_info=new ExecutionInfo(
            block_info=caller_execution_info.block_info,
            tx_info=new_tx_info,
            caller_address=ORIGIN_ADDRESS,
            contract_address=contract_address,
            selector=selector,
        ),
        deprecated_tx_info=deprecated_tx_info_ptr,
    );
    fill_deprecated_tx_info(tx_info=new_tx_info, dst=execution_context.deprecated_tx_info);

    // Since we process the revert log backwards, entries before this point belong to the calling
    // contract.
    assert [revert_log] = RevertLogEntry(
        selector=CHANGE_CONTRACT_ENTRY, value=caller_execution_info.contract_address
    );
    let revert_log = &revert_log[1];

    contract_call_helper(
        remaining_gas=remaining_gas,
        block_context=block_context,
        execution_context=execution_context,
    );

    // Entries before this point belong to the callee.
    assert [revert_log] = RevertLogEntry(selector=CHANGE_CONTRACT_ENTRY, value=contract_address);
    let revert_log = &revert_log[1];

    return ();

func check_and_increment_nonce{contract_state_changes: DictAccess*}(tx_info: TxInfo*) -> () {
    // Do not handle nonce for version 0.
    if (tx_info.version == 0) {
        return ();
    }

    tempvar state_entry: StateEntry*;
    %{ SetStateEntryToAccountContractAddress %}

    tempvar current_nonce = state_entry.nonce;
    with_attr error_message("Unexpected nonce.") {
        assert current_nonce = tx_info.nonce;
    }

    // Update contract_state_changes.
    tempvar new_state_entry = new StateEntry(
        class_hash=state_entry.class_hash,
        storage_ptr=state_entry.storage_ptr,
        nonce=current_nonce + 1,
    );
    dict_update{dict_ptr=contract_state_changes}(
        key=tx_info.account_contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    return ();
