### Title
Arithmetic Overflow in `compute_max_possible_fee` Enables Complete Fee Bypass — (File: `execution/transaction_impls.cairo`)

---

### Summary

The `compute_max_possible_fee` function in `transaction_impls.cairo` performs unchecked felt-arithmetic multiplication on user-controlled `max_price_per_unit` values. Because `max_price_per_unit` has no enforced upper bound beyond `assert_nn` (i.e., it can be any value in `[0, (PRIME-1)/2]`), an attacker can craft resource-bounds values whose sum of products wraps to exactly `0 mod PRIME`. When `charge_fee` observes `max_fee == 0`, it returns immediately without executing the ERC-20 fee transfer, granting the attacker free transaction execution.

---

### Finding Description

**Root cause — `compute_max_possible_fee` (transaction_impls.cairo, lines 87–102):**

```cairo
func compute_max_possible_fee(tx_info: TxInfo*) -> felt {
    ...
    return l1_gas_bounds.max_amount * l1_gas_bounds.max_price_per_unit
         + l2_gas_bounds.max_amount * (l2_gas_bounds.max_price_per_unit + tx_info.tip)
         + l1_data_gas_bounds.max_amount * l1_data_gas_bounds.max_price_per_unit;
}
``` [1](#0-0) 

All arithmetic is performed in the StarkNet field (`mod PRIME`). Each product `max_amount × max_price_per_unit` can be up to `(2^64 − 1) × (PRIME−1)/2 ≈ 2^314`, far exceeding PRIME ≈ 2^251, so each term wraps freely.

**Insufficient validation — `pack_resource_bounds` (transaction_hash.cairo, lines 103–108):**

```cairo
func pack_resource_bounds{range_check_ptr}(resource_bounds: ResourceBounds) -> felt {
    assert_nn_le(resource_bounds.max_amount, 2 ** 64 - 1);   // bounded
    assert_nn(resource_bounds.max_price_per_unit);            // only ≥ 0, NO upper bound
    ...
}
``` [2](#0-1) 

`max_price_per_unit` is constrained only to `[0, (PRIME−1)/2]` by `assert_nn`. No upper bound prevents overflow.

**Exploited skip path — `charge_fee` (transaction_impls.cairo, lines 111–165):**

```cairo
func charge_fee(...) {
    local tx_info: TxInfo* = tx_execution_context.execution_info.tx_info;
    let max_fee = compute_max_possible_fee(tx_info=tx_info);

    if (max_fee == 0) {
        return ();          // ← fee transfer is skipped entirely
    }
    ...
    assert_nn_le(calldata.amount.low, max_fee);   // actual_fee ≤ max_fee = 0 → actual_fee = 0
    ...
}
``` [3](#0-2) 

When `max_fee == 0`, the function returns before executing the ERC-20 transfer. This path is reached for invoke, deploy-account, and declare transactions.

---

### Impact Explanation

**Critical — Direct loss of funds.**

The sequencer never receives the fee for the transaction. Because the OS proof enforces `actual_fee ≤ max_fee = 0`, the sequencer is cryptographically prevented from charging any fee for the block. Any number of such transactions can be included in a block, draining sequencer revenue entirely. Additionally, free execution enables unbounded spam, which maps to the secondary impact of network halt.

---

### Likelihood Explanation

**High.** The attack requires only arithmetic knowledge of the StarkNet field prime. The attacker constructs a valid V3 transaction (invoke/deploy-account/declare) with crafted `max_price_per_unit` values. No privileged access, leaked keys, or external dependencies are required. The transaction is signed normally and submitted through the standard mempool interface.

---

### Recommendation

Add an explicit upper-bound range check on `max_price_per_unit` in `pack_resource_bounds` (e.g., `assert_nn_le(resource_bounds.max_price_per_unit, MAX_PRICE_PER_UNIT)` where `MAX_PRICE_PER_UNIT` is chosen so that the maximum possible sum of three products cannot overflow PRIME). Alternatively, validate in `compute_max_possible_fee` that the computed result is non-zero and consistent with the individual terms before returning.

---

### Proof of Concept

Let `PRIME = 2^251 + 17·2^192 + 1`.

Choose:
- L1 gas: `max_amount = 1`, `max_price_per_unit = (PRIME − 1) / 2`
- L2 gas: `max_amount = 1`, `max_price_per_unit = 1`, `tip = 0`
- L1 data gas: `max_amount = 1`, `max_price_per_unit = (PRIME − 1) / 2`

All three `max_price_per_unit` values are in `[0, (PRIME−1)/2]`, so they pass `assert_nn`.

```
compute_max_possible_fee
  = (PRIME−1)/2 + 1 + (PRIME−1)/2
  = PRIME − 1 + 1
  = PRIME
  ≡ 0  (mod PRIME)
```

`charge_fee` observes `max_fee == 0` and returns immediately. The ERC-20 fee transfer is never executed. The attacker's transaction (invoke/deploy-account/declare) executes with zero fee charged, and the proof is valid.

func compute_max_possible_fee(tx_info: TxInfo*) -> felt {
