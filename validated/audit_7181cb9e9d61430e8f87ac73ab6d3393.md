### Title
`deprecated_declared_classes` Silently Dropped from State Diff Commitment — (`crates/blockifier/src/state/cached_state.rs`)

### Summary

The `From<CommitmentStateDiff> for ThinStateDiff` conversion unconditionally sets `deprecated_declared_classes: Vec::new()`. This `ThinStateDiff` is the sole input to `calculate_block_commitments` inside `BlockExecutionArtifacts::new`, which computes both the `state_diff_commitment` and the `state_diff_length` that are chained into the block hash. Any Cairo 0 (`DECLARE` v0/v1) class declaration executed in a block is therefore silently absent from the state diff commitment and from the `concatenated_counts` field, producing a wrong block hash for every block that contains such a transaction.

### Finding Description

**Root cause — `CommitmentStateDiff` → `ThinStateDiff` conversion:**

```rust

**End-to-end effect:** when a `DECLARE` v0/v1 transaction is executed, the resulting deprecated class hash is present in `StateMaps.declared_contracts`, dropped by `From<StateMaps> for CommitmentStateDiff`, absent from the `ThinStateDiff` passed to `calculate_block_commitments`, and therefore absent from both `state_diff_commitment` and `concatenated_counts`. Both values feed directly into `calculate_block_hash`, so the produced block hash is wrong. The same `ThinStateDiff::from(commitment_state_diff)` is also stored via `append_state_diff`, so the stored state diff is equally wrong.

The `state_maps_to_committer_state_diff` function used in the SNOS/transaction-prover path also omits `declared_contracts`:

```rust
pub fn state_maps_to_committer_state_diff(state_maps: StateMaps) -> StateDiff {
    StateDiff {
        address_to_class_hash: state_maps.class_hashes,
        address_to_nonce: state_maps.nonces,
        class_hash_to_compiled_class_hash: ...,
        storage_updates: ...,
        // declared_contracts dropped
    }
}
``` [9](#0-8) 

This means the proof is internally consistent with the wrong block hash, so the wrong commitment propagates to L1 without being caught by the prover.

### Impact Explanation

For every block that contains a `DECLARE` v0/v1 (Cairo 0) transaction:

1. **Wrong `state_diff_commitment`** — the Poseidon hash over the state diff omits the deprecated class hash(es), producing a value that does not match the true state diff.
2. **Wrong `state_diff_length` in `concatenated_counts`** — the count of state diff entries is under-reported by the number of deprecated class declarations.
3. **Wrong block hash** — both corrupted values are chained into the block hash via `calculate_block_hash`.
4. **Wrong stored state diff** — `append_state_diff` persists the same empty `deprecated_declared_classes`, so `starknet_getStateUpdate` returns an incorrect state diff.
5. **Proof consistency with wrong hash** — the SNOS path drops the same field, so the ZK proof attests to the wrong block hash, which is then accepted on L1.

This matches the **Critical** impact: *Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input.*

### Likelihood Explanation

Any user can submit a `DECLARE` v0 or v1 transaction (Cairo 0 class declaration). The Starknet protocol still supports these transaction types. No special privilege is required. The trigger is a normal, unprivileged user action.

### Recommendation

Add `deprecated_declared_classes` to `CommitmentStateDiff` and populate it from `StateMaps.declared_contracts` (filtering for `true` entries that represent new declarations). Update `From<StateMaps> for CommitmentStateDiff` and `From<CommitmentStateDiff> for ThinStateDiff` accordingly, and propagate the field through `state_maps_to_committer_state_diff` in the SNOS/prover path.

### Proof of Concept

1. Submit a `DECLARE` v1 transaction declaring a Cairo 0 contract class to the sequencer.
2. The blockifier executes it; `StateMaps.declared_contracts` is populated with the class hash.
3. `From<StateMaps> for CommitmentStateDiff` drops `declared_contracts`; `CommitmentStateDiff` has no deprecated class field.
4. `BlockExecutionArtifacts::new` calls `ThinStateDiff::from(commitment_state_diff)`, producing `deprecated_declared_classes: Vec::new()`.
5. `calculate_block_commitments` computes `state_diff_commitment = Poseidon([..., 0 deprecated classes, ...])` and `state_diff_length` without counting the declared class.
6. `calculate_block_hash` chains these wrong values; the resulting block hash differs from the hash that would be produced if the deprecated class were included.
7. The same wrong `ThinStateDiff` is stored via `append_state_diff`; `starknet_getStateUpdate` returns a state diff with an empty `deprecated_declared_classes`.
8. The SNOS/prover path (`state_maps_to_committer_state_diff`) also drops the deprecated class, so the proof attests to the wrong block hash, which is submitted to and accepted by the L1 verifier.

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}

pub fn state_maps_to_committer_state_diff(state_maps: StateMaps) -> StateDiff {
    StateDiff {
        address_to_class_hash: state_maps.class_hashes,
        address_to_nonce: state_maps.nonces,
        class_hash_to_compiled_class_hash: state_maps
            .compiled_class_hashes
            .into_iter()
            .map(|(class_hash, compiled_class_hash)| {
                (class_hash, CompiledClassHash(compiled_class_hash.0))
            })
            .collect(),
        storage_updates: StorageDiff::from(StorageView(state_maps.storage))
            .into_iter()
            .map(|(address, updates)| {
                (
                    address,
                    updates
                        .into_iter()
                        .map(|(key, value)| (StarknetStorageKey(key), StarknetStorageValue(value)))
                        .collect(),
                )
            })
            .collect(),
    }
}
