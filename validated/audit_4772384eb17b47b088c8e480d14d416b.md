### Title
`deprecated_declared_classes` Always Zeroed in `CommitmentStateDiff`→`ThinStateDiff` Conversion Causes Wrong `state_diff_commitment` and `concatenated_counts` in Block Hash — (`File: crates/blockifier/src/state/cached_state.rs`)

---

### Summary

`CommitmentStateDiff` has no `deprecated_declared_classes` field. The `From<CommitmentStateDiff> for ThinStateDiff` conversion hardcodes that field to `Vec::new()`. Because `BlockExecutionArtifacts::new` feeds `ThinStateDiff::from(commitment_state_diff.clone())` into both `calculate_block_commitments` (which computes `state_diff_commitment` and `concatenated_counts`) and `thin_state_diff()` (which is stored and committed), any block that contains a deprecated (Cairo 0) class declaration will produce a wrong `state_diff_commitment`, a wrong `state_diff_length` inside `concatenated_counts`, and therefore a wrong block hash — all accepted and stored without error.

---

### Finding Description

**Step 1 — The missing field in `CommitmentStateDiff`.**

`CommitmentStateDiff` tracks four categories of state change but has no slot for deprecated (Cairo 0) class declarations:

```rust
pub struct CommitmentStateDiff {
    pub address_to_class_hash: IndexMap<ContractAddress, ClassHash>,
    pub address_to_nonce: IndexMap<ContractAddress, Nonce>,
    pub storage_updates: IndexMap<ContractAddress, IndexMap<StorageKey, Felt>>,
    pub class_hash_to_compiled_class_hash: IndexMap<ClassHash, CompiledClassHash>,
}
``` [1](#0-0) 

`StateMaps` does carry `declared_contracts: HashMap<ClassHash, bool>` for deprecated classes, but `From<StateMaps> for CommitmentStateDiff` silently drops it — `declared_contracts` is never read. [2](#0-1) 

**Step 2 — The hardcoded empty vec in the conversion.**

The only conversion from `CommitmentStateDiff` to `ThinStateDiff` unconditionally sets `deprecated_declared_classes` to an empty vector, with a TODO acknowledging the gap:

```rust
impl From<CommitmentStateDiff> for ThinStateDiff {
    fn from(commitment_state_diff: CommitmentStateDiff) -> Self {
        Self {
            deployed_contracts: commitment_state_diff.address_to_class_hash,
            storage_diffs: commitment_state_diff.storage_updates,
            class_hash_to_compiled_class_hash: commitment_state_diff
                .class_hash_to_compiled_class_hash,
            nonces: commitment_state_diff.address_to_nonce,
            // TODO(AlonH): Remove this when the structure of storage diffs changes.
            deprecated_declared_classes: Vec::new(),   // ← always empty
        }
    }
}
``` [3](#0-2) 

**Step 3 — Both commitment paths use this conversion.**

`BlockExecutionArtifacts::new` calls `calculate_block_commitments` with `ThinStateDiff::from(commitment_state_diff.clone())`:

```rust
let (header_commitments, measurements) = calculate_block_commitments(
    &transactions_data,
    ThinStateDiff::from(commitment_state_diff.clone()),   // deprecated_declared_classes = []
    l1_da_mode,
    &block_info.starknet_version,
).await;
``` [4](#0-3) 

`thin_state_diff()`, which is the state diff stored to storage and sent to the committer, uses the same conversion:

```rust
pub fn thin_state_diff(&self) -> ThinStateDiff {
    ThinStateDiff::from(self.commitment_state_diff.clone())
}
``` [5](#0-4) 

**Step 4 — `state_diff_length` and `concatenated_counts` are computed from the deficient diff.**

Inside `calculate_block_commitments`:

```rust
let concatenated_counts = concat_counts(
    transactions_data.len(),
    event_leaf_elements.len(),
    state_diff.len(),          // ThinStateDiff::len() counts deprecated_declared_classes.len() = 0
    l1_da_mode,
);
let state_diff_length = state_diff.len();   // same wrong value
``` [6](#0-5) 

`ThinStateDiff::len()` explicitly adds `deprecated_declared_classes.len()`, which is always 0 here:

```rust
result += self.deprecated_declared_classes.len();
``` [7](#0-6) 

**Step 5 — `calculate_state_diff_hash` also receives the empty list.**

```rust
hash_chain =
    chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
``` [8](#0-7) 

`chain_deprecated_declared_classes` chains the count and each class hash into the Poseidon hash. With an empty list it chains `0` instead of the real count and real class hashes, producing a wrong `StateDiffCommitment`. [9](#0-8) 

**Step 6 — Both wrong values enter the block hash.**

`calculate_block_hash` chains `concatenated_counts` (which embeds the wrong `state_diff_length`) and `state_diff_commitment.0.0` (the wrong hash):

```rust
.chain(&block_commitments.concatenated_counts)
.chain(&block_commitments.state_diff_commitment.0.0)
``` [10](#0-9) 

---

### Impact Explanation

For every block that contains at least one deprecated (Cairo 0) class declaration:

| Value | Expected | Actual |
|---|---|---|
| `deprecated_declared_classes` in `ThinStateDiff` | non-empty | `[]` |
| `state_diff_length` in `concatenated_counts` | N + k | N (k = number of deprecated declarations) |
| `state_diff_commitment` | hash over full diff | hash over diff missing deprecated classes |
| block hash | correct | wrong, but accepted and stored |
| `ThinStateDiff` written to storage / sent to committer | includes deprecated classes | missing deprecated classes |

This matches the **Critical** impact: *Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input* — specifically a wrong `state_diff_commitment` and wrong `concatenated_counts` baked into the accepted block hash, and a wrong `ThinStateDiff` committed to storage.

---

### Likelihood Explanation

**Low.** Deprecated (Cairo 0) declare transactions (v0/v1) are still valid Starknet transaction types. Any unprivileged user can submit one. If the gateway admits it and the batcher includes it, the block commitment is silently wrong. The TODO comment in the conversion confirms the developers are aware the field is always empty but have not yet closed the gap.

---

### Recommendation

Add `deprecated_declared_classes` to `CommitmentStateDiff` and populate it from `StateMaps::declared_contracts` (entries where `value == true` and the initial read was `false`):

```rust
pub struct CommitmentStateDiff {
    pub address_to_class_hash: IndexMap<ContractAddress, ClassHash>,
    pub address_to_nonce: IndexMap<ContractAddress, Nonce>,
    pub storage_updates: IndexMap<ContractAddress, IndexMap<StorageKey, Felt>>,
    pub class_hash_to_compiled_class_hash: IndexMap<ClassHash, CompiledClassHash>,
    pub deprecated_declared_classes: Vec<ClassHash>,   // add this
}
```

Then in `From<StateMaps> for CommitmentStateDiff`, populate it from `diff.declared_contracts` (keys where value is `true`), and in `From<CommitmentStateDiff> for ThinStateDiff`, forward it instead of hardcoding `Vec::new()`.

---

### Proof of Concept

1. Submit a `DECLARE` v1 transaction (deprecated Cairo 0 class) to the sequencer gateway.
2. The batcher includes it; `StateMaps::declared_contracts` records `{class_hash: true}`.
3. `CommitmentStateDiff::from(state_maps)` drops `declared_contracts`; `CommitmentStateDiff` has no deprecated class entry.
4. `BlockExecutionArtifacts::new` calls `ThinStateDiff::from(commitment_state_diff)` → `deprecated_declared_classes = []`.
5. `calculate_block_commitments` computes `state_diff_length = N` (missing the 1 deprecated declaration) and `state_diff_commitment = H(... 0 deprecated classes ...)`.
6. `calculate_block_hash` chains the wrong `concatenated_counts` and wrong `state_diff_commitment` → wrong block hash stored and broadcast.
7. `thin_state_diff()` returns the same deficient diff → committer writes a `ThinStateDiff` missing the deprecated class to storage.
8. Any node syncing or verifying the proof receives a state diff that omits the deprecated class declaration, breaking state consistency.

fn chain_deprecated_declared_classes(
    deprecated_declared_classes: &[ClassHash],
    hash_chain: HashChain,
) -> HashChain {
    let mut sorted_deprecated_declared_classes = deprecated_declared_classes.to_vec();
    sorted_deprecated_declared_classes.sort_unstable();
    hash_chain
        .chain(&sorted_deprecated_declared_classes.len().into())
        .chain_iter(sorted_deprecated_declared_classes.iter().map(|class_hash| &class_hash.0))
}
