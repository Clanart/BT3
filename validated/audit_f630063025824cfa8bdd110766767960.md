### Title
Missing Declared Class Validation in `execute_replace_class` Allows Permanent Contract Fund Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in `syscall_impls.cairo` does not validate that the new class hash provided by a contract corresponds to a declared contract class. This allows any contract to replace its class with an arbitrary undeclared hash, permanently making the contract inaccessible and freezing any funds held within it. The missing check is explicitly acknowledged by a TODO comment in the code.

---

### Finding Description

In `execute_replace_class`, the OS accepts any class hash provided by the calling contract without checking whether that class hash corresponds to a declared contract class. The TODO comment at line 898 explicitly acknowledges this missing check: [1](#0-0) 

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The function directly updates the contract's class hash in `contract_state_changes` with no validation. When a contract calls `replace_class` with an undeclared class hash, the OS commits the update to state. Subsequently, when any transaction attempts to call this contract, `execute_entry_point` will:

1. Read the undeclared class hash from `contract_state_changes` as `execution_context.class_hash`
2. Call `dict_read{dict_ptr=contract_class_changes}(key=undeclared_hash)` — returns `0` because the class was never declared [2](#0-1) 

3. Call `find_element(..., key=0)` to find the compiled class fact with hash `0` — panics/fails because no such compiled class fact exists [3](#0-2) 

This makes the contract permanently inaccessible. The sequencer will reject any transaction that calls this contract (because it would cause the OS to fail), permanently freezing any funds held in the contract. There is no recovery path — the class hash is committed to the Patricia Merkle tree and there is no mechanism to "undeclare" a class or reverse a `replace_class` call.

This is directly analogous to M-02: just as `_setupRole` grants a role without setting up the admin (making revocation impossible), `execute_replace_class` sets a class hash without validating it is declared (making the state change irreversible and destructive).

---

### Impact Explanation

Any funds (ETH, STRK, or tokens) held in a contract whose class hash has been replaced with an undeclared hash are **permanently frozen**. The contract cannot execute any code, so funds cannot be withdrawn. This constitutes a **Critical impact: Permanent freezing of funds**.

---

### Likelihood Explanation

Any contract can call the `replace_class` syscall on itself. A contract with a bug, or a malicious shared contract (e.g., a DEX or lending protocol) that holds other users' funds, can call `replace_class` with an arbitrary class hash. Since the OS does not validate the new class hash, this can happen without any privileged access. The missing check is explicitly acknowledged in the codebase with a TODO, confirming the developers are aware the invariant is not enforced. [4](#0-3) 

---

### Recommendation

Add a check in `execute_replace_class` to verify that the new class hash is a declared class before updating state. This can be done by asserting that the class hash exists in `contract_class_changes` with a non-zero compiled class hash:

```cairo
// Check that the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the fix recommended in M-02: just as the Neuron constructor should grant `DEFAULT_ADMIN_ROLE` to enforce role governance, `execute_replace_class` should enforce that the replacement class is declared before committing the state change.

---

### Proof of Concept

1. Deploy contract A that holds user funds (e.g., a shared vault).
2. Contract A calls `replace_class(0xdeadbeef)` where `0xdeadbeef` is not a declared class hash.
3. The OS accepts the replacement — the TODO check is absent, no validation occurs.
4. Contract A's class hash is committed to state as `0xdeadbeef`.
5. Any subsequent call to contract A fails:
   - `dict_read{dict_ptr=contract_class_changes}(key=0xdeadbeef)` → returns `0`
   - `find_element(..., key=0)` → panics (no compiled class with hash `0`)
   - The sequencer rejects the transaction as it would invalidate the OS proof.
6. All funds in contract A are permanently frozen with no recovery path. [5](#0-4)
