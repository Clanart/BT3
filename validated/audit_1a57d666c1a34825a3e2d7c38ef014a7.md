### Title
Unprotected BOOTSTRAP Path in `execute_declare_transaction` Bypasses All Signature, Nonce, and Fee Checks — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

The `execute_declare_transaction` function in the StarkNet OS contains a special-cased "BOOTSTRAP" initialization path that completely skips signature verification, nonce validation, fee payment, and sender contract existence checks. Any unprivileged user who can get the sequencer to include a crafted declare transaction with `sender_address == 'BOOTSTRAP'`, `nonce == 0`, `version == 3`, and zero resource bounds can declare an arbitrary class hash mapped to an arbitrary `compiled_class_hash` — with no cryptographic authorization whatsoever. This is the direct analog of the "front-runnable initializer" class from the external report: an unguarded initialization path that any actor can trigger before the legitimate operator does.

---

### Finding Description

In `execute_declare_transaction`, after computing the transaction hash and filling `tx_info`, the OS checks a special condition:

```cairo
if (sender_address == 'BOOTSTRAP' and tx_info.nonce == 0 and tx_info.version == 3) {
    let max_possible_fee = compute_max_possible_fee(tx_info=tx_info);
    if (max_possible_fee == 0) {
        assert_not_zero(compiled_class_hash);
        dict_update{dict_ptr=contract_class_changes}(
            key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
        );
        %{ SkipTx %}
        return ();
    }
}
``` [1](#0-0) 

When this branch is taken, the OS **entirely skips**:

1. **`check_and_increment_nonce`** — no nonce validation or increment occurs. [2](#0-1) 

2. **`run_validate`** (i.e., `__validate_declare__`) — no signature verification is performed. [3](#0-2) 

3. **`charge_fee`** — no fee is charged. [4](#0-3) 

4. **`dict_read` on `contract_state_changes` for `sender_address`** — the normal path (line 782) reads the sender's state entry to obtain its class hash for validation. The bootstrap path skips this entirely, meaning no deployed contract at address `'BOOTSTRAP'` is required. [5](#0-4) 

The only guards in the bootstrap path are:
- `compiled_class_hash != 0`
- `prev_value == 0` (the class must not have been declared before)

The felt value of `'BOOTSTRAP'` is the ASCII big-endian encoding `0x424F4F545354524150`. This is a concrete, known value. A transaction with `sender_address = 0x424F4F545354524150`, `nonce = 0`, `version = 3`, and all resource bounds set to zero satisfies every condition. The transaction hash is computed and verified by `%{ AssertTransactionHash %}`, but since no signature check is performed in this path, the hash verification only confirms internal consistency — it does not prove ownership of the sender address. [6](#0-5) 

The `compute_max_possible_fee` function confirms that setting all three resource bounds (`L1_GAS`, `L2_GAS`, `L1_DATA_GAS`) to zero amounts satisfies `max_possible_fee == 0`: [7](#0-6) 

---

### Impact Explanation

**Critical — Direct loss of funds.**

An attacker who successfully triggers the BOOTSTRAP path can write an arbitrary `(class_hash → compiled_class_hash)` mapping into `contract_class_changes` with no authorization. Specifically:

- The attacker chooses any `class_hash` not yet declared and any `compiled_class_hash` (including one pointing to malicious CASM bytecode they control).
- This entry is committed into the class commitment tree via `dict_update` with `prev_value=0`. [8](#0-7) 
- The class commitment tree update is then proven and committed to L1 via `compute_class_commitment`. [9](#0-8) 

Once a malicious `class_hash → compiled_class_hash` mapping is committed on-chain, any subsequent `deploy` syscall or `deploy_account` transaction using that `class_hash` will instantiate a contract running the attacker's CASM. Contracts holding funds (ERC-20 balances, ETH, STRK) that are deployed or upgraded to this class hash will execute attacker-controlled logic, enabling direct theft of funds.

Additionally, the `get_contract_class_leaf_hash` function used during commitment hashing does not validate that the `compiled_class_hash` corresponds to any real, registered Sierra class: [10](#0-9) 

This means the corrupted class tree entry passes through the Patricia commitment update without any integrity check on the CASM content.

---

### Likelihood Explanation

**Medium.**

The conditions required are entirely attacker-controllable at the transaction-field level:
- `sender_address = 'BOOTSTRAP'` — a known felt constant, no contract deployment needed
- `nonce = 0` — trivially set
- `version = 3` — standard
- All resource bounds = 0 — trivially set

The primary barrier is that the sequencer's mempool layer may reject transactions from a `sender_address` with no deployed contract. However, the OS-level Cairo code — which is the authoritative source of truth for the ZK proof — imposes no such check in the bootstrap path. A malicious or compromised sequencer, or a sequencer that processes the transaction before its own mempool guard fires, would produce a valid proof accepted by the L1 verifier. Furthermore, the BOOTSTRAP path is intended for system bootstrapping, meaning it may be deliberately enabled during network initialization windows, precisely when an attacker can race to front-run it.

---

### Recommendation

1. **Remove or gate the BOOTSTRAP path behind an operator-controlled flag** that is set in `BlockContext` and verified by the OS, rather than relying on a magic `sender_address` value that any user can supply.
2. **Require a valid signature** even in the bootstrap path, or restrict it to a hardcoded sequencer/operator address whose public key is embedded in the OS constants.
3. **Add a `dict_read` check** on `contract_state_changes` for `sender_address` in the bootstrap path to at minimum require a deployed contract at that address.
4. **Emit a distinct output marker** for bootstrap declarations so L1 verifiers can apply additional scrutiny.

---

### Proof of Concept

**Attacker crafts the following declare transaction fields:**

```
sender_address  = 0x424F4F545354524150  // felt('BOOTSTRAP')
nonce           = 0
version         = 3
tip             = 0
resource_bounds = [
    { resource: L1_GAS,      max_amount: 0, max_price_per_unit: 0 },
    { resource: L2_GAS,      max_amount: 0, max_price_per_unit: 0 },
    { resource: L1_DATA_GAS, max_amount: 0, max_price_per_unit: 0 },
]
class_hash         = <target class hash not yet declared>
compiled_class_hash = <hash of attacker-controlled malicious CASM>
```

**OS execution path:**

1. `execute_declare_transaction` is called.
2. `%{ DeclareTxFields %}` loads the attacker's fields.
3. `compute_declare_transaction_hash` computes a valid hash (no signature needed).
4. `%{ AssertTransactionHash %}` passes (hash is internally consistent).
5. The condition `sender_address == 'BOOTSTRAP' and nonce == 0 and version == 3` evaluates to true.
6. `compute_max_possible_fee` returns `0` (all bounds are zero).
7. `assert_not_zero(compiled_class_hash)` passes (attacker provides non-zero value).
8. `dict_update` writes `class_hash → malicious_compiled_class_hash` into `contract_class_changes`.
9. `%{ SkipTx %}` returns — no signature check, no nonce increment, no fee.
10. The class commitment tree is updated and proven; the malicious mapping is committed to L1. [1](#0-0) [9](#0-8)

func compute_max_possible_fee(tx_info: TxInfo*) -> felt {
    tempvar resource_bounds: ResourceBounds* = tx_info.resource_bounds_start;
    let n_resource_bounds = (tx_info.resource_bounds_end - resource_bounds) / ResourceBounds.SIZE;

    // Only V3 transactions with all resource bounds are supported.
    assert tx_info.version = 3;
    assert n_resource_bounds = 3;

    tempvar l1_gas_bounds: ResourceBounds = resource_bounds[L1_GAS_INDEX];
    tempvar l2_gas_bounds: ResourceBounds = resource_bounds[L2_GAS_INDEX];
    tempvar l1_data_gas_bounds = resource_bounds[L1_DATA_GAS_INDEX];

    return l1_gas_bounds.max_amount * l1_gas_bounds.max_price_per_unit + l2_gas_bounds.max_amount *
        (l2_gas_bounds.max_price_per_unit + tx_info.tip) + l1_data_gas_bounds.max_amount *
        l1_data_gas_bounds.max_price_per_unit;
}

    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    outputs: OsCarriedOutputs*,

        tx_hash_prefix=DECLARE_HASH_PREFIX,
        sender_address=sender_address,

    with poseidon_ptr {

            assert [class_hash_ptr] = expected_class_hash;
        }

    update_poseidon_in_builtin_ptrs(poseidon_ptr=poseidon_ptr);

    // Get the account transaction info.

            );
            %{ SkipTx %}
            return ();
        }
    }

func get_contract_class_leaf_hash{poseidon_ptr: PoseidonBuiltin*}(compiled_class_hash: felt) -> (
    hash: felt
) {
    if (compiled_class_hash == UNINITIALIZED_CLASS_HASH) {
        return (hash=0);
    }

    // Return H(CONTRACT_CLASS_LEAF_VERSION, compiled_class_hash).
    let (hash_value) = poseidon_hash(CONTRACT_CLASS_LEAF_VERSION, compiled_class_hash);
    return (hash=hash_value);
}

func compute_class_commitment{poseidon_ptr: PoseidonBuiltin*, range_check_ptr}(
    class_changes_start: DictAccess*,
    n_class_updates: felt,
    patricia_update_constants: PatriciaUpdateConstants*,
) -> (contract_class_tree_update_output: CommitmentUpdate) {
    alloc_locals;

    // Guess the initial and final roots of the contract class tree.
    local initial_root;
    local final_root;
    %{ SetPreimageForClassCommitments %}

    // Create a dictionary mapping class hash to the contract class leaf hash,
    // to prepare the input for the commitment tree update.
    let (local hashed_class_changes: DictAccess*) = alloc();
    hash_class_changes(
        n_class_updates=n_class_updates,
        class_changes=class_changes_start,
        hashed_class_changes=hashed_class_changes,
    );

    // Call patricia_update_using_update_constants() instead of patricia_update()
    // in order not to repeat globals_pow2 calculation.
    patricia_update_using_update_constants_with_poseidon(
        patricia_update_constants=patricia_update_constants,
        update_ptr=hashed_class_changes,
        n_updates=n_class_updates,
        height=MERKLE_HEIGHT,
        prev_root=initial_root,
        new_root=final_root,
    );

    return (
        contract_class_tree_update_output=CommitmentUpdate(
            initial_root=initial_root, final_root=final_root
        ),
    );
