### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezal of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the new class hash supplied by a contract corresponds to a previously declared class. The OS will generate a valid proof for a state transition that sets a contract's class hash to an arbitrary, undeclared value. Once committed on-chain, the contract becomes permanently uncallable, freezing all funds held in its storage.

### Finding Description
In `syscall_impls.cairo`, the `execute_replace_class` function reads the caller's state entry and unconditionally writes the attacker-supplied `class_hash` into the contract state without any check that the hash exists in `contract_class_changes` (the declared-class dictionary):

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The in-code `TODO` at line 898 explicitly acknowledges the missing check. [1](#0-0) 

The analogous issue in the external report is that `create_market` did not verify the authority on vault accounts before accepting them as legitimate program-controlled vaults. Here, `execute_replace_class` does not verify the new class hash is a legitimate declared class before committing it to the proven state.

The `contract_class_changes` dictionary (populated only by `execute_declare_transaction` via `dict_update`) is the authoritative record of declared classes. [2](#0-1)  The `execute_replace_class` path never consults this dictionary. [3](#0-2) 

### Impact Explanation
**Critical — Permanent freezing of funds.**

Once the OS produces and the verifier accepts a proof containing a state transition where `contract.class_hash = <undeclared_hash>`, the on-chain state is irrevocably committed. Any subsequent call to that contract will fail at the class-lookup stage because no entry-point table exists for the undeclared hash. All ERC-20 balances, ERC-721 tokens, or any other assets stored in the contract's storage slots become permanently inaccessible with no recovery path.

### Likelihood Explanation
An unprivileged contract deployer is the attacker. The attack requires no privileged key, no operator collusion, and no external dependency:

1. Attacker deploys a contract that appears legitimate (e.g., a token vault or DEX pool).
2. Users deposit funds into the contract.
3. Attacker calls `replace_class` from within the contract, supplying `class_hash = 0` (or any value not in `contract_class_changes`).
4. The OS, lacking the validation check, generates a valid proof for this state transition.
5. The proof is verified and the state is committed; the contract is permanently bricked.

The `replace_class` syscall is reachable from any Cairo 1 contract's execution context via the `REPLACE_CLASS_SELECTOR` dispatch in `execute_syscalls`. [4](#0-3) 

### Recommendation
Before writing the new state entry, assert that the supplied `class_hash` exists in `contract_class_changes`. Concretely, perform a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert the returned compiled class hash is non-zero:

```cairo
// Verify the class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the pattern used in `execute_declare_transaction` where `assert_not_zero(compiled_class_hash)` is enforced before updating `contract_class_changes`. [5](#0-4) 

### Proof of Concept

1. Declare a malicious class `MaliciousVault` whose `__execute__` entry point:
   - Accepts deposits normally.
   - Exposes an admin function that calls `replace_class(0)`.
2. Deploy `MaliciousVault` at address `A`.
3. Victims call `deposit()` on `A`, storing balances in `A`'s storage.
4. Attacker calls the admin function; the OS executes `execute_replace_class` with `class_hash = 0`.
5. Because no check against `contract_class_changes` is performed, the OS writes `StateEntry(class_hash=0, ...)` for address `A`. [1](#0-0) 
6. The OS generates a valid proof; the verifier accepts it; the state is committed.
7. All subsequent calls to `A` fail — no entry-point table exists for class hash `0` — and all deposited funds are permanently frozen.

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
