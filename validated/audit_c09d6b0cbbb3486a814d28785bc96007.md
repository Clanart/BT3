### Title
`ProofOfInclusion::valid()` Never Anchors to an External Root — Forged Inclusion Proofs Always Pass - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate only verifies the internal self-consistency of the proof chain. Its final comparison is tautological: it compares `existing_hash` (which equals `layers.last().combined_hash` after the loop) against `self.root_hash()` (which also returns `layers.last().combined_hash`). No external, trusted root hash is ever checked. An attacker who can supply a serialized `ProofOfInclusion` to any consumer that calls `.valid()` can forge an arbitrary inclusion proof that always returns `true`, regardless of the actual DataLayer tree state.

### Finding Description

**Root cause — tautological final check in `valid()`:**

```rust
// crates/chia-datalayer/src/merkle/proof_of_inclusion.rs  lines 40-58
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;   // ← always equals layer.combined_hash here
    }

    existing_hash == self.root_hash()      // ← always true; see root_hash() below
}
```

`root_hash()` (lines 32-38):

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← same value as `existing_hash` after the loop
    } else {
        self.node_hash
    }
}
```

After the loop, `existing_hash` equals the last `layer.combined_hash` (the loop would have returned `false` otherwise). `self.root_hash()` also returns `last.combined_hash`. The comparison `existing_hash == self.root_hash()` is therefore always `true` — it is a comparison of a value with itself. The empty-layers case is equally tautological: `existing_hash` stays as `self.node_hash`, and `root_hash()` returns `self.node_hash`.

**What is missing:** a parameter `trusted_root: &Hash` that the caller supplies from a source they trust (e.g., a committed on-chain root, a locally computed root from `MerkleBlob::get_root_hash()`). Without it, `valid()` only proves that the proof is internally self-consistent — it says nothing about whether the proof corresponds to any real tree.

**Attacker-controlled entry path:**

1. `ProofOfInclusion` derives `Streamable` and is exposed via Python bindings (`wheel/python/chia_rs/datalayer.pyi` lines 237-266), so it can be deserialized from arbitrary bytes received over the network or from an untrusted peer.
2. An attacker constructs a `ProofOfInclusion` with an arbitrary `node_hash` (e.g., the hash of a key-value pair they want to falsely prove is present) and a single `ProofOfInclusionLayer` where `other_hash`, `other_hash_side`, and `combined_hash` are chosen so that `calculate_internal_hash(node_hash, side, other_hash) == combined_hash`. This is trivially achievable by the attacker: they pick any `other_hash` and `side`, compute the hash, and set `combined_hash` to the result.
3. The victim calls `proof.valid()` → `true`. The victim believes the key-value pair is present in the DataLayer tree at whatever root the forged proof implies.

### Impact Explanation

**High — DataLayer Merkle proof logic accepts forged inclusion proofs.**

Any DataLayer consumer that relies on `ProofOfInclusion::valid()` as the sole check before trusting a proof (e.g., a light client, a cross-chain bridge, a smart-coin puzzle that reads DataLayer state) can be deceived into accepting fabricated state. An attacker can prove the presence of any key-value pair in any tree, with any root hash they choose, without possessing any real tree data. This enables forged state attestation, which can lead to unauthorized actions in any system that gates behavior on DataLayer inclusion proofs.

### Likelihood Explanation

The Python binding exposes `valid()` as the canonical validation method with no required root parameter. The method name `valid()` strongly implies complete validation. Any developer who calls `proof.valid()` without separately checking `proof.root_hash()` against a locally trusted root is fully exposed. The `Streamable` derivation means proofs can be received from untrusted network peers and deserialized without any prior filtering.

### Recommendation

Replace the parameterless `valid()` with a method that requires a trusted root:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // ← compare against caller-supplied trusted root
}
```

The existing `valid()` (which only checks internal consistency) should either be removed or clearly renamed to `is_internally_consistent()` with documentation warning that it does not validate against any real tree root.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
from chia_rs import calculate_internal_hash  # or equivalent

# Attacker wants to forge proof that fake_node_hash is in some tree
fake_node_hash = bytes([0xAA] * 32)
other_hash     = bytes([0xBB] * 32)
other_side     = 1  # Side.Right

# Attacker computes combined_hash so the layer is internally consistent
combined_hash = calculate_internal_hash(fake_node_hash, other_side, other_hash)

layer = ProofOfInclusionLayer(
    other_hash_side=other_side,
    other_hash=other_hash,
    combined_hash=combined_hash,
)
forged_proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

assert forged_proof.valid()          # ← returns True; proof is entirely fabricated
assert forged_proof.root_hash() == combined_hash  # attacker controls the claimed root
```
