### Title
`get_compiled_class_hash` in `ExecutionStateReader` ignores `state_number` bound, returning a future-declared class's compiled hash at past-state queries — (`crates/apollo_rpc_execution/src/state_reader.rs`)

---

### Summary

`ExecutionStateReader::get_compiled_class_hash` retrieves the block at which a Cairo 1 class was declared and returns its compiled class hash **without checking whether that declaration block falls before or after `self.state_number`**. When the RPC layer calls `trace_block_transactions`, `simulate_transactions`, or `estimate_fee` at a historical state number, the function silently returns a non-zero compiled class hash for a class that was not yet declared at that state, causing the blockifier to treat the class as already declared. This produces authoritative-looking wrong execution results from those RPC endpoints.

---

### Finding Description

`ExecutionStateReader::get_compiled_class_hash` (lines 163–208) works as follows:

1. It calls `get_class_definition_block_number(&class_hash)` — a direct key-value lookup that returns the block at which the class was stored, with no awareness of `self.state_number`.
2. If a block number is found, it fetches the state diff at that block and returns the compiled class hash from it.
3. **It never checks whether `block_number <= self.state_number.block_after()`.** [1](#0-0) 

Every other historical lookup in the codebase performs this guard. `get_contract_class` in `execution_utils.rs` explicitly returns `None` when the declaration block is after the queried state: [2](#0-1) 

`get_class_definition_at` in the storage layer does the same: [3](#0-2) 

The RPC `get_compiled_class` handler also guards correctly: [4](#0-3) 

`get_compiled_class_hash` is the sole path that skips this check. The underlying `get_class_definition_block_number` is a raw key-value lookup with no temporal filtering: [5](#0-4) 

The codebase itself acknowledges the analogous gap for Cairo 0 classes in the class-manager path with a TODO: [6](#0-5) 

---

### Impact Explanation

`trace_block_transactions` sets `state_number = StateNumber::right_before_block(block_number)` for non-pending blocks: [7](#0-6) 

`simulate_transactions` and `estimate_fee` use `StateNumber::unchecked_right_after_block(block_number)`, which is also before any class declared in a later block. [8](#0-7) 

When any of these endpoints is called at block M and a class C was declared at block N > M:

- `get_compiled_class_hash(C)` returns the non-zero compiled class hash from block N's state diff.
- The blockifier's `CachedState` caches this value and treats C as already declared.
- A `Declare` transaction for C at block M is incorrectly rejected (class appears already declared).
- `trace_block_transactions` at block N itself is broken: the state is `right_before_block(N)`, yet `get_compiled_class_hash` returns the hash written by block N, so the `Declare` transaction inside block N is replayed as a failure even though it originally succeeded.

The corrupted value flows through `CachedState::get_compiled_class_hash`: [9](#0-8) 

This matches the **High** impact: *RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.*

---

### Likelihood Explanation

Any caller of `starknet_traceBlockTransactions`, `starknet_simulateTransactions`, or `starknet_estimateFee` that targets a block containing a `Declare` transaction triggers the bug. No special privilege is required; the block number and class hash are public information. The `trace_block_transactions` path is the most direct trigger because it always sets `state_number` to `right_before_block`, placing it exactly one step before the declaration.

---

### Recommendation

Add a state-number guard in `get_compiled_class_hash` immediately after obtaining `block_number`, mirroring the pattern used in `get_contract_class`:

```rust
let Some(block_number) = maybe_block_number else {
    return Ok(CompiledClassHash::default());
};

// NEW: mirror the guard in get_contract_class / get_class_definition_at
if self.state_number.is_before(block_number) {
    return Ok(CompiledClassHash::default());
}
``` [10](#0-9) 

---

### Proof of Concept

1. Append block 0 with an empty state diff (no class declared).
2. Append block 1 with a `Declare` for class hash `0xABC` → compiled class hash `0x1`.
3. Construct an `ExecutionStateReader` with `state_number = StateNumber::right_before_block(BlockNumber(1))` (i.e., the state before block 1).
4. Call `reader.get_compiled_class_hash(ClassHash(0xABC))`.
5. **Expected**: `CompiledClassHash::default()` (class not yet declared).
6. **Actual**: `CompiledClassHash(0x1)` — the hash from block 1's state diff is returned, because `get_class_definition_block_number` returns `Some(BlockNumber(1))` and no temporal guard exists.

The blockifier then treats `0xABC` as already declared at the pre-block-1 state, causing any simulated or traced `Declare` transaction for `0xABC` at that state to be incorrectly rejected. [11](#0-10)

    fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
        if let Some(pending_data) = &self.maybe_pending_data {
            for DeclaredClassHashEntry { class_hash: other_class_hash, compiled_class_hash } in
                &pending_data.declared_classes
            {
                if class_hash == *other_class_hash {
                    return Ok(*compiled_class_hash);
                }
            }
        }

        let maybe_block_number = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_reader()
            .map_err(storage_err_to_state_err)?
            .get_class_definition_block_number(&class_hash)
            .map_err(storage_err_to_state_err)?;

        // Cairo 0 classes (and undeclared classes) do not have a compiled class hash.
        // According to the trait, return the default value.
        let Some(block_number) = maybe_block_number else {
            return Ok(CompiledClassHash::default());
        };

        let state_diff = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_diff(block_number)
            .map_err(storage_err_to_state_err)?
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing state diff at block {block_number}."
            )))?;

        let compiled_class_hash = state_diff
            .class_hash_to_compiled_class_hash
            .get(&class_hash)
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing class declaration at block {block_number}, class \
                 {class_hash}."
            )))?;

        Ok(*compiled_class_hash)
    }
