### Title
`ProofOfInclusion::valid()` Performs a Tautological Self-Check Instead of Verifying Against an External Root Hash - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` in the DataLayer Merkle proof module performs a self-referential final check — it compares the last computed hash against `self.root_hash()`, which is defined as the `combined_hash` of the last layer. Because the loop already verified that `calculated_hash == layer.combined_hash` before assigning `existing_hash = calculated_hash`, the final assertion `existing_hash == self.root_hash()` is always trivially true. The function never validates the proof against any externally trusted root hash, allowing a forged `ProofOfInclusion` to pass validation for any claimed leaf.

### Finding Description

`ProofOfInclusion` is a `Streamable` type (serializable/deserializable) used to prove DataLayer key-value inclusion. Its `valid()` method is the sole verification gate:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;   // ← existing_hash is now layer.combined_hash
    }

    existing_hash == self.root_hash()      // ← self.root_hash() returns last.combined_hash
}
``` [1](#0-0) 

`root_hash()` is defined as:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← same value as existing_hash after the loop
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

After the loop body executes `existing_hash = calculated_hash` and the loop has verified `calculated_hash == layer.combined_hash`, `existing_hash` holds exactly `last.combined_hash`. The final line `existing_hash == self.root_hash()` therefore reduces to `last.combined_hash == last.combined_hash`, which is unconditionally `true`. No external, trusted root hash is ever consulted.

The analog to the original report is direct: `withdrawRevenue()` transferred funds back to the contract itself (a self-referential no-op); `valid()` checks the proof's own internally-derived hash against itself (a self-referential no-op), providing no actual security guarantee.

### Impact Explanation

`ProofOfInclusion` is a `Streamable` type exposed through Python bindings: [3](#0-2) 

Any party that receives a `ProofOfInclusion` over the network and calls `valid()` to authenticate it receives no protection. An attacker can construct a proof for an arbitrary `node_hash` (any key-value pair they wish to falsely claim is present) by:

1. Choosing any `node_hash`.
2. Choosing arbitrary `other_hash` values for each layer.
3. Computing each `combined_hash` correctly from the previous hash and the chosen sibling — this is public computation.
4. Submitting the proof; `valid()` returns `true`.

This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.**

### Likelihood Explanation

`ProofOfInclusion` is a `Streamable` type with Python bindings, meaning it is designed to cross trust boundaries. The `valid()` method is the only verification function and is called directly in tests and the fuzz harness: [4](#0-3) 

Any consumer of DataLayer proofs from an untrusted peer who relies on `valid()` is fully exposed. The construction of a forged proof requires only public arithmetic (hash computation), no secret knowledge.

### Recommendation

`valid()` must accept an externally trusted root hash parameter and compare the final computed hash against it, not against the proof's own last `combined_hash`:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against external trusted root
}
```

The current `valid()` (no parameter) should either be removed or clearly documented as only checking internal self-consistency, not security.

### Proof of Concept

```python
from chia_rs import MerkleBlob, ProofOfInclusion, ProofOfInclusionLayer
import hashlib

# Build a real tree with key 1 → value 1
blob = MerkleBlob(bytearray())
real_hash = hashlib.sha256(b"real").digest()
blob.insert(1, 1, real_hash)
blob.calculate_lazy_hashes()

# Forge a proof claiming key 999 (not in tree) is included
fake_leaf_hash = hashlib.sha256(b"fake_leaf").digest()
sibling_hash   = hashlib.sha256(b"sibling").digest()

# Compute combined_hash honestly from fake inputs
import hashlib
combined = hashlib.sha256(fake_leaf_hash + sibling_hash).digest()  # simplified

forged = ProofOfInclusion(
    node_hash=fake_leaf_hash,
    layers=[ProofOfInclusionLayer(
        other_hash_side=0,          # Left
        other_hash=sibling_hash,
        combined_hash=combined,     # self-consistent but not the real root
    )]
)

# valid() returns True even though fake_leaf_hash is not in the tree
assert forged.valid()   # passes — tautological check
```

The `valid()` call succeeds because the final check compares `combined` against `combined` (the last `combined_hash`), never against the actual tree root. [1](#0-0) [5](#0-4)

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
