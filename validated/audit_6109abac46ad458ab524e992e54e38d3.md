### Title
Zero-Fee EIP-712 Cosmos Tx Bypass via `NoBaseFee`/London Divergence in `NewDynamicFeeChecker` — (`ante/evm/fee_checker.go`)

### Summary

When `NoBaseFee=true` and the London hardfork is active, `evmtypes.GetBaseFee()` returns `new(big.Int)` (zero) instead of `nil`. `NewDynamicFeeChecker` interprets a non-nil zero as a valid EIP-1559 base fee, so any Cosmos EIP-712 transaction with `fee=0` satisfies `feeCap(0) >= baseFeeInt(0)` and commits state without paying fees.

### Finding Description

**Step 1 — `Params.GetBaseFee()` returns nil when `NoBaseFee=true`** [1](#0-0) 

**Step 2 — `evmtypes.GetBaseFee()` converts that nil into `new(big.Int)` (zero) when London is enabled** [2](#0-1) 

The comment "should not be nil if london hardfork enabled" documents the intent, but the result is a semantic mismatch: `NoBaseFee=true` means "no floor", yet the function returns zero, which is a floor of zero.

**Step 3 — `NewDynamicFeeChecker` receives `baseFee = new(big.Int)` (zero, not nil)** [3](#0-2) 

The nil guard at line 57 does **not** trigger, so the checker does not fall back to `checkTxFeeWithValidatorMinGasPrices`. It proceeds with EIP-1559 logic.

**Step 4 — Fee comparison with fee=0 and baseFee=0 always passes** [4](#0-3) 

With `fee=0`, `gas>0`: `feeCap = 0 / gas = 0`; `baseFeeInt = 0`; `0 < 0` is false → no error. `effectiveFee = min(0+maxPriority, 0) * gas = 0`. `DeductFeeDecorator` deducts 0.

**Step 5 — EIP-712 Cosmos tx routes through this checker** [5](#0-4) [6](#0-5) 

`newLegacyCosmosAnteHandlerEip712` installs `NewDynamicFeeChecker` when `options.DynamicFeeChecker=true` (the production setting confirmed in `evmd/app.go`).

**Step 6 — `MinGasPriceDecorator` does not save the situation** [7](#0-6) 

The default `MinGasPrice` is `sdkmath.LegacyZeroDec()` (zero), so the decorator short-circuits unconditionally in default deployments.

### Impact Explanation

An unprivileged attacker can submit a valid EIP-712 Cosmos transaction with `fee=0` that passes all ante-handler fee checks and commits state. The fee floor invariant is bypassed: the chain processes transactions for free, mis-accounting fees and allowing state mutation without economic cost. This fits the **High** impact category: "ante handler bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."

### Likelihood Explanation

Requires `NoBaseFee=true` (a governance-settable parameter used by chains that want fixed or zero fees) **and** London HF active (the default for virtually all Ethermint deployments) **and** `DynamicFeeChecker=true` (the production default). All three conditions co-occur in any chain that disables the fee market while keeping London active — a documented and supported configuration.

### Recommendation

In `evmtypes.GetBaseFee()`, when `feemarketParams.GetBaseFee()` returns nil (i.e., `NoBaseFee=true`), return `nil` rather than `new(big.Int)`, so `NewDynamicFeeChecker` correctly falls back to `checkTxFeeWithValidatorMinGasPrices`:

```go
// x/evm/types/utils.go
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    if !IsLondon(ethCfg, height) {
        return nil
    }
    return feemarketParams.GetBaseFee() // return nil as-is when NoBaseFee=true
}
```

Alternatively, add an explicit `NoBaseFee` guard in `NewDynamicFeeChecker` before the nil check.
