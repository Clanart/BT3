### Title
Missing Cairo 0 Class Existence Check at Historical State Number in `ExecutionStateReader::get_compiled_class` - (File: crates/apollo_rpc_execution/src/state_reader.rs)

### Summary

`ExecutionStateReader::get_compiled_class` skips the state-number existence check for Cairo 0 (deprecated) classes when the class manager path is active. For Cairo 1 classes the code explicitly calls `is_contract_class_declared` to verify the class was declared at or before the requested `state_number`; for Cairo 0 classes that branch is absent and the class is returned unconditionally. Any RPC endpoint that executes against a historical block (`starknet_call`, `starknet_estimateFee`, `starknet_simulateTransactions`, `starknet_traceTransaction`) will therefore run with a Cairo 0 class that did not exist at the requested state, producing an authoritative-looking wrong execution result, fee estimate, or trace.

### Finding Description

In `ExecutionStateReader::get_compiled_class`, when `self.class_manager_handle` is `Some`, the code fetches the compiled class from the class manager and then branches on the class type:

```rust
// Cairo 1 path — existence check IS performed
ContractClass::V1(casm_contract_class) => {
    let is_declared = is_contract_class_declared(
        &self.storage_reader.begin_ro_txn()...,
        &class_hash,
        self.state_number,   // ← checked against historical state
    )?;
    if is_declared {
        Ok(RunnableCompiledClass::V1(casm_contract_class.try_into()?))
    } else {
        Err(StateError::UndeclaredClassHash(class_hash))
    }
}

// Cairo 0 path — NO existence check
// TODO(shahak): Verify cairo0 as well after get_class_definition_block_number is fixed.
ContractClass::V0(deprecated_contract_class) => {
    Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
}
``` [1](#0-0) 

The helper `is_contract_class_declared` checks `get_class_definition_block_number` and verifies the declaration block is at or before `state_number`: [2](#0-1) 

For Cairo 0 classes the equivalent check would use `get_deprecated_class_definition_block_number`, but the TODO comment acknowledges this is not yet done. The fallback storage path (no class manager) correctly enforces the state-number boundary via `get_deprecated_class_definition_at`: [3](#0-2) 

The `ExecutionStateReader` is constructed with the `class_manager_handle` in `execute_transactions`, which is the shared execution kernel for all RPC simulation/estimation/call endpoints: [4](#0-3) 

Those endpoints pass `class_manager_client` explicitly: [5](#0-4) [6](#0-5) 

### Impact Explanation

When a caller invokes `starknet_estimateFee`, `starknet_simulateTransactions`, `starknet_call`, or `starknet_traceTransaction` against a historical block ID (block N−1), and the class manager already holds a Cairo 0 class that was declared at block N, the execution engine receives and runs that class as if it existed at block N−1. The returned fee estimate, execution retdata, state diff, or trace is computed against a wrong class — one that did not exist at the requested state. The result is presented to the caller as authoritative.

This matches the allowed impact: **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

### Likelihood Explanation

- Cairo 0 (deprecated) contracts remain widely deployed on Starknet mainnet.
- The class manager is enabled in production node configurations (the `class_manager_client` field is populated in the RPC server).
- Any user can trigger the condition by submitting an RPC call against a block number that precedes the declaration block of a Cairo 0 class that the node has already synced.
- No privileged access is required; the attacker only needs to know a Cairo 0 class hash and the block at which it was declared.

### Recommendation

Apply the same `is_contract_class_declared`-style guard to the Cairo 0 branch. The correct check uses `get_deprecated_class_definition_block_number` (already available on the state reader) rather than `get_class_definition_block_number` (which is Cairo 1-only):

```rust
ContractClass::V0(deprecated_contract_class) => {
    let declared_block = self
        .storage_reader
        .begin_ro_txn()
        .map_err(storage_err_to_state_err)?
        .get_state_reader()
        .map_err(storage_err_to_state_err)?
        .get_deprecated_class_definition_block_number(&class_hash)
        .map_err(|e| StateError::StateReadError(e.to_string()))?;

    let is_declared = declared_block
        .is_some_and(|block_number| self.state_number.is_after(block_number));

    if is_declared {
        Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
    } else {
        Err(StateError::UndeclaredClassHash(class_hash))
    }
}
```

Remove the TODO comment once the fix is in place.

### Proof of Concept

1. Node is synced to block N. A Cairo 0 class `C` was declared at block N. The class manager holds `C`.
2. Caller sends:
   ```json
   {
     "method": "starknet_estimateFee",
     "params": {
       "transactions": [{ "type": "INVOKE", "calldata": ["<addr deploying C>", ...] }],
       "block_id": { "block_number": N-1 }
     }
   }
   ```
3. `estimate_fee` resolves `state_number = StateNumber::unchecked_right_after_block(N-1)`.
4. `execute_transactions` constructs `ExecutionStateReader { state_number, class_manager_handle: Some(...) }`.
5. During execution, `get_compiled_class(C)` is called. The class manager returns `ContractClass::V0(C)`. The Cairo 0 branch returns `Ok(RunnableCompiledClass::V0(...))` without checking `state_number`.
6. Execution proceeds with class `C` as if it existed at block N−1. The returned fee estimate reflects execution of code that did not exist at that state.

The correct behavior (matching the fallback storage path and the Cairo 1 class manager path) would be to return `Err(StateError::UndeclaredClassHash(C))`, causing the RPC call to fail with `CONTRACT_NOT_FOUND` or `CLASS_HASH_NOT_FOUND`. [7](#0-6) [8](#0-7)
