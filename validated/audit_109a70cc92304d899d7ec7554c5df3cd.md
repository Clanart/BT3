### Title
`ProofOfInclusion::valid()` Accepts Forged Inclusion Proofs — No External Root Binding — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` only checks internal self-consistency of the proof structure. Its final comparison is a tautology: it compares `existing_hash` against `self.root_hash()`, which is defined as `last.combined_hash` — the same value `existing_hash` was just set to inside the loop. The function never validates the proof against any external trusted root. An attacker can construct a `ProofOfInclusion` with an arbitrary `node_hash` (a key-value pair not in the real tree) and internally consistent layers, and `valid()` will return `true`.

### Finding Description

`ProofOfInclusion` and `ProofOfInclusionLayer` are both `Streamable` types exposed to Python via `pyclass(get_all, from_py_object)`: [1](#0-0) 

The Python bindings expose direct construction from arbitrary inputs: [2](#0-1) 

The `valid()` method: [3](#0-2) 

Trace through the logic:
1. The loop checks `calculated_hash != layer.combined_hash` → returns `false` if mismatch.
2. On success: `existing_hash = calculated_hash = layer.combined_hash` (they are equal by the check above).
3. After the loop: `existing_hash` equals the last `layer.combined_hash`.
4. `self.root_hash()` returns `last.combined_hash`: [4](#0-3) 

Therefore `existing_hash == self.root_hash()` is **always `true`** when the loop completes. The final check is a tautology and provides zero additional validation. `valid()` never compares against any externally trusted root hash.

### Impact Explanation

An attacker can construct a `ProofOfInclusion` from Python via `__new__`, `from_bytes`, or `from_json_dict` with:
- `node_hash` = SHA256 of any arbitrary key-value pair not present in the real DataLayer tree
- `layers` = a chain of `ProofOfInclusionLayer` objects where each `combined_hash` is correctly computed from the previous hash and an arbitrary `other_hash`

Calling `proof.valid()` on this forged object returns `True`. Any Python consumer that relies on `proof.valid()` as the sole validity gate — without separately checking `proof.root_hash() == trusted_on_chain_root` — will accept the forged inclusion proof. This allows an attacker to falsely prove that a key-value pair exists in a committed DataLayer tree root, enabling invalid state to be proven as valid.

This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, or lets untrusted input prove invalid state.**

### Likelihood Explanation

- `ProofOfInclusion` is a `Streamable` type with full Python construction and deserialization APIs, making it trivially constructable from attacker-controlled bytes.
- The `valid()` method is the only validation method on the type; `root_hash()` is a separate method that callers must remember to check independently.
- The misleading name `valid()` strongly implies complete proof validation, increasing the probability that callers omit the separate root-hash check.
- The Python type stub exposes both `valid()` and `root_hash()` without any documentation indicating that both must be checked: [5](#0-4) 

### Recommendation

1. Add a `trusted_root: Hash` parameter to `valid()` and compare `existing_hash` against it at the end of the loop, replacing the tautological self-comparison.
2. Alternatively, rename the current method to `internally_consistent()` and add a new `valid(trusted_root: Hash) -> bool` that performs the root-binding check.
3. Update the Python type stub and any documentation to make the root-binding requirement explicit.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
import hashlib

# Arbitrary "fake" leaf hash (not in any real tree)
fake_node_hash = bytes([0xAA] * 32)
sibling_hash   = bytes([0xBB] * 32)

# Compute a combined_hash that is internally consistent
# (matches what calculate_internal_hash would produce for these two inputs)
# The exact hash function used is calculate_internal_hash(fake_node_hash, Left, sibling_hash)
# For PoC purposes, any value works as long as combined_hash == calculate_internal_hash(...)
# We use the real function by constructing a single-layer proof:

layer = ProofOfInclusionLayer(
    other_hash_side=0,          # Side::Left
    other_hash=sibling_hash,
    combined_hash=<calculate_internal_hash(fake_node_hash, Left, sibling_hash)>,
)

forged_proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

# valid() returns True despite fake_node_hash never being in any real tree
assert forged_proof.valid() == True

# root_hash() returns an attacker-controlled value, not any on-chain root
print(forged_proof.root_hash())  # attacker-chosen combined_hash
```

    pub fn valid(&self) -> bool {
        let mut existing_hash = self.node_hash;

        for layer in &self.layers {
            let calculated_hash = crate::calculate_internal_hash(
                &existing_hash,
                layer.other_hash_side,
                &layer.other_hash,
            );

            if calculated_hash != layer.combined_hash {
                return false;
            }

            existing_hash = calculated_hash;
        }

        existing_hash == self.root_hash()
    }
