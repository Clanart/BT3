### Title
`batch_insert` Bypasses Duplicate-Key Guard, Silently Corrupting DataLayer Merkle Tree Root — (`File: crates/chia-datalayer/src/merkle/blob.rs`)

### Summary

`MerkleBlob::batch_insert` skips the duplicate-key and duplicate-hash checks that `MerkleBlob::insert` enforces. A caller who supplies a batch containing a repeated `KeyId` (or a key already present in the tree) causes two leaf nodes with the same key to be written into the flat blob. The in-memory `BlockStatusCache` silently overwrites the first entry with the second, leaving the blob and the cache in an inconsistent state. The Merkle root subsequently computed over this corrupted structure is wrong, and any proof of inclusion or exclusion derived from it is invalid.

### Finding Description

`MerkleBlob::insert` guards against duplicate keys and hashes before writing anything:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 369-374
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
```

`batch_insert` uses `insert` only for the last one or two items popped from the vector when the tree currently has ≤ 1 leaves. All remaining items — and **all** items when the tree already has ≥ 2 leaves — are written directly through `insert_entry_to_blob` with no duplicate check:

```rust
// lines 587-602
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ... Node::Leaf(LeafNode { hash, key, value, ... }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;   // ← no duplicate guard
    indexes.push(new_leaf_index);
}
```

`insert_entry_to_blob` calls `block_status_cache.add_leaf`, which uses `HashMap::insert` — a silent overwrite:

```rust
// lines 188-192
fn add_leaf(&mut self, index: TreeIndex, leaf: LeafNode) {
    self.free_indexes.shift_remove(&index);
    self.key_to_index.insert(leaf.key, index);        // overwrites silently
    self.leaf_hash_to_index.insert(leaf.hash, index); // overwrites silently
}
```

After a batch with a repeated key `K` (values `V1`/`H1` then `V2`/`H2`):

- The blob contains **two** leaf blocks for `K` (at indices `I1` and `I2`).
- `key_to_index` maps `K → I2` only; `I1` is orphaned but still wired into the tree structure.
- `leaf_count()` returns 1 (from `key_to_index.len()`), but the blob has 2 leaf nodes.
- `calculate_lazy_hashes()` propagates hashes over the corrupted structure, producing a root that does not correspond to any valid key-value mapping.
- `get_proof_of_inclusion(K)` returns a proof anchored to `I2` against the corrupted root; no valid proof for `I1` can be produced.

The Python binding `py_batch_insert` passes the caller-supplied list directly into `batch_insert` with no additional deduplication:

```rust
// lines 1503-1518
pub fn py_batch_insert(&mut self, keys_values: Vec<(KeyId, ValueId)>, hashes: Vec<Hash>) -> PyResult<()> {
    ...
    self.batch_insert(zip(keys_values, hashes).collect())?;
    Ok(())
}
```

### Impact Explanation

The corrupted Merkle root is the committed state of a DataLayer store. Any downstream consumer that verifies proofs against this root will accept proofs for the wrong value of `K` (the second insertion) and will be unable to verify the first. If the corrupted root is persisted and shared, all inclusion and exclusion proofs derived from it are unsound. This directly matches the allowed High impact: *DataLayer Merkle proof/blob/delta logic corrupts tree roots or lets untrusted input prove invalid state.*

### Likelihood Explanation

`batch_insert` is the primary bulk-load path for the DataLayer and is exposed as a first-class Python API. Any code path that feeds externally-sourced key-value pairs into `batch_insert` without pre-deduplicating them triggers the bug. Because `insert` already rejects duplicates, callers have no reason to expect `batch_insert` to behave differently. The inconsistency is invisible at the call site — `batch_insert` returns `Ok(())`.

### Recommendation

Add a duplicate-key and duplicate-hash check inside the fast path of `batch_insert`, mirroring the guards in `insert`:

```rust
for ((key, value), hash) in &keys_values_hashes {
    if self.block_status_cache.contains_key(*key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(hash) {
        return Err(Error::HashAlreadyPresent());
    }
}
```

Alternatively, update `add_leaf` to return an error (instead of silently overwriting) when a key or hash is already present, so that every code path through `insert_entry_to_blob` is protected uniformly.

### Proof of Concept

```python
from chia_rs.datalayer import MerkleBlob, KeyId, ValueId
import hashlib

def h(n):
    return bytes(hashlib.sha256(n.to_bytes(8, "big")).digest())

blob = MerkleBlob(blob=bytearray())

# Pre-populate so leaf_count > 1, bypassing the insert() path entirely
blob.insert(KeyId(100), ValueId(100), h(100))
blob.insert(KeyId(200), ValueId(200), h(200))

# batch_insert with a duplicate key — no error raised
blob.batch_insert(
    [(KeyId(1), ValueId(10)), (KeyId(1), ValueId(99))],  # key 1 appears twice
    [h(1), h(2)],
)

blob.calculate_lazy_hashes()

# The blob now has two leaf nodes for KeyId(1).
# The root hash is computed over this corrupted structure.
# check_integrity() will raise IntegrityKeyToIndexCacheLength
# because leaf_count in the blob (4) != key_to_index length (3).
blob.check_integrity()  # raises Error

    fn insert_entry_to_blob(&mut self, index: TreeIndex, block: &Block) -> Result<(), Error> {
        let new_block_bytes = block.to_bytes()?;
        let extend_index = self.extend_index();
        match index.cmp(&extend_index) {
            Ordering::Greater => return Err(Error::BlockIndexOutOfBounds(index)),
            Ordering::Equal => self.blob.extend_from_slice(&new_block_bytes),
            Ordering::Less => {
                self.blob[block_range(index)].copy_from_slice(&new_block_bytes);
            }
        }

        match block.node {
            Node::Leaf(leaf) => self.block_status_cache.add_leaf(index, leaf),
            Node::Internal(..) => self.block_status_cache.add_internal(index),
        }

        Ok(())
    }
