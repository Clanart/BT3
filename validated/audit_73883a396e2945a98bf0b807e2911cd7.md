### Title
Residual ETH in Router Consumed by Subsequent WETH Swap via Hybrid `pay` Branch — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

---

### Summary

The `pay` function's hybrid WETH branch reads `address(this).balance` globally, without restricting consumption to the current transaction's `msg.value`. Any ETH left in the router from a prior user's `multicall` can be silently consumed by a later caller's WETH swap, causing direct ETH loss for the prior user.

---

### Finding Description

`PeripheryPayments.pay` contains three branches for `token == WETH`: [1](#0-0) 

The middle branch (`nativeBalance > 0 && nativeBalance < value`) at lines 78–81 deposits **all** of `address(this).balance` as WETH and transfers it to the pool, then pulls only the shortfall from `payer`. This balance is not scoped to the current transaction — it includes any ETH left over from any prior call.

ETH accumulates in the router legitimately because:

1. `multicall` is `payable` and forwards `msg.value` to the contract balance. [2](#0-1) 

2. `refundETH()` is a separate, optional step that must be explicitly included in the multicall — there is no automatic refund. [3](#0-2) 

3. `receive()` only blocks *direct* ETH sends (not `msg.value` in payable calls), so it provides no protection here. [4](#0-3) 

**Concrete attack path:**

- **Tx 1 (User A):** Calls `multicall{value: X+Y}(...)`. The swap consumes `X` ETH. User A omits `refundETH()`. `Y` ETH remains in the router.
- **Tx 2 (User B / attacker):** Calls `exactInputSingle` with `tokenIn=WETH, amountIn=Z` where `Y < Z`. The swap callback triggers `pay(WETH, userB, pool, Z)`. Since `nativeBalance = Y > 0` and `Y < Z`, the hybrid branch fires:
  - Deposits User A's `Y` ETH → transfers `Y` WETH to pool.
  - Pulls only `Z - Y` WETH from User B's wallet.
- **Result:** Pool receives `Z` WETH. User B paid only `Z - Y`. User A's `Y` ETH is gone.

---

### Impact Explanation

User A suffers a direct, unrecoverable loss of `Y` ETH. User B (or a deliberate attacker monitoring the router's balance) receives a subsidy equal to `Y` ETH on their swap. This is a principal-level fund loss, not a fee or dust issue.

---

### Likelihood Explanation

Stranded ETH in the router is a realistic and common condition:
- Users routinely send a round-number `msg.value` slightly above `amountIn` to avoid reverts.
- Multicall bundles that omit `refundETH()` are common in practice (e.g., when the user only cares about the swap output).
- An attacker can monitor the router's ETH balance on-chain and front-run or immediately follow any transaction that leaves a non-zero balance.

---

### Recommendation

Scope ETH consumption in `pay` to the current transaction's `msg.value` only. One approach: track how much ETH has been consumed within the current call context (e.g., via a transient storage slot initialized to `msg.value` at the router entry point) and use that instead of `address(this).balance`. Alternatively, require that the router's ETH balance is zero at the start of each non-multicall entry point, or automatically refund residual ETH at the end of every swap function.

// Foundry test sketch
contract TestResidualETHSteal is Test {
    MetricOmmSimpleRouter router;
    address weth;
    address pool; // WETH/TOKEN pool, zeroForOne=true

    function test_residualETHStolenByWETHSwap() public {
        address userA = makeAddr("userA");
        address userB = makeAddr("userB");

        // Fund userA with ETH, userB with WETH
        vm.deal(userA, 2 ether);
        deal(weth, userB, 1 ether);
        vm.prank(userB);
        IERC20(weth).approve(address(router), type(uint256).max);

        // Tx 1: userA multicall with 2 ETH, swap only needs 1 ETH, no refundETH step
        vm.prank(userA);
        bytes[] memory calls = new bytes[](1);
        calls[0] = abi.encodeCall(router.exactInputSingle, (
            ExactInputSingleParams({
                tokenIn: weth, amountIn: 1 ether, /* ... */ recipient: userA
            })
        ));
        router.multicall{value: 2 ether}(calls);
        // 1 ETH remains in router

        assertEq(address(router).balance, 1 ether);
        uint256 userABalanceBefore = userA.balance; // 0 (sent 2, got nothing back)

        // Tx 2: userB swaps 1.5 WETH exactInputSingle
        // hybrid branch fires: 1 ETH from router + 0.5 WETH from userB = 1.5 WETH to pool
        vm.prank(userB);
        router.exactInputSingle(ExactInputSingleParams({
            tokenIn: weth, amountIn: 1.5 ether, /* ... */ recipient: userB
        }));

        // userB only spent 0.5 WETH from wallet; router's 1 ETH (userA's) was consumed
        assertEq(address(router).balance, 0);
        assertEq(IERC20(weth).balanceOf(userB), 0.5 ether); // saved 1 ETH worth
        // userA's 1 ETH is permanently lost
    }
}
