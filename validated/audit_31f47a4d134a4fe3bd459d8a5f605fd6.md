### Title
Missing Class Hash Existence Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall in the StarkNet OS accepts any arbitrary class hash without verifying it has been declared, while `execute_declare_transaction` strictly validates and registers class hashes before they can be used. This asymmetry — directly analogous to the deposit/withdraw inconsistency in the reference report — allows a contract to replace its own class with an undeclared hash, permanently breaking the contract and freezing any funds held within it. The missing check is explicitly acknowledged by a TODO comment in the code.

---

### Finding Description

**Declare transaction — strict validation:**

`execute_declare_transaction` in `transaction_impls.cairo` enforces a two-step validation before a class hash is usable:

1. It calls `finalize_class_hash` to verify the class hash is a valid Sierra class hash pre-image.
2. It writes the compiled class hash into `contract_class_changes` with `prev_value=0`, enforcing that the class is new and properly registered. [1](#0-0) [2](#0-1) 

**`replace_class` syscall — no validation:**

`execute_replace_class` in `syscall_impls.cairo` takes `request.class_hash` directly from the syscall request and writes it into `contract_state_changes` with zero validation. The code contains an explicit TODO acknowledging the missing check:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [3](#0-2) 

There is no lookup into `contract_class_changes` to confirm the supplied class hash was ever declared. Any felt value — including `0`, a random value, or a hash of a non-existent class — is accepted and committed to the contract state.

---

### Impact Explanation

**Permanent freezing of funds (Critical).**

If a contract's class hash is replaced with an undeclared hash:

- All subsequent calls to the contract will fail at the class resolution step, because the class does not exist in the class tree.
- The contract's storage (and any ERC-20 balances, locked collateral, or other assets tracked within it) becomes permanently inaccessible.
- The state update is committed to L1 via the OS proof, making the freeze irreversible at the protocol level.

The OS proof is the authoritative source of truth accepted by the L1 verifier. Because the OS does not enforce the invariant, a proof containing an invalid `replace_class` is indistinguishable from a valid one.

---

### Likelihood Explanation

**Medium.** The `replace_class` syscall can only be invoked by a contract against itself. Exploitation requires one of the following realistic paths:

1. **Buggy access control in an upgradeable contract**: A DeFi protocol or token contract exposes an upgrade function that calls `replace_class`. If the access control has a flaw (e.g., missing `assert caller == admin`), an unprivileged user can supply an arbitrary class hash and permanently break the contract, freezing all deposited user funds.

2. **Malicious contract developer**: A developer deploys a contract that accepts user deposits and includes a hidden or time-delayed call to `replace_class` with an invalid hash. The OS provides no backstop to prevent this.

Both paths are reachable by an unprivileged transaction sender without any privileged key or operator access.

---

### Recommendation

In `execute_replace_class`, before updating `contract_state_changes`, verify that the supplied `class_hash` exists in `contract_class_changes` (i.e., its compiled class hash is non-zero). This mirrors the invariant enforced by `execute_declare_transaction`:

```cairo
// Verify the class hash has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This closes the asymmetry between declaration (strict) and replacement (currently unchecked).

---

### Proof of Concept

1. Deploy a vault contract `V` that holds user funds and exposes an `upgrade(new_class_hash)` function that calls `replace_class(new_class_hash)`. Assume a missing or bypassable access-control check.
2. Users deposit funds into `V` (e.g., via ERC-20 transfers tracked in `V`'s storage).
3. An unprivileged attacker calls `V.upgrade(0xdeadbeef)` where `0xdeadbeef` is not a declared class hash.
4. The OS executes `execute_replace_class` with `class_hash = 0xdeadbeef`. No check against `contract_class_changes` is performed.
5. `contract_state_changes` is updated: `V`'s `class_hash` is now `0xdeadbeef`.
6. The OS generates a valid proof. The L1 verifier accepts the state update.
7. All subsequent calls to `V` fail — the class `0xdeadbeef` does not exist in the class tree.
8. All user funds in `V` are permanently frozen with no recovery path. [4](#0-3)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    );

        key=contract_address,

        new_value=cast(new_state_entry, felt),
