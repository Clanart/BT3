### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Fund Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS processes the `replace_class` syscall without verifying that the new class hash corresponds to a declared contract class. This is the direct analog to "missed safeApprove": a critical, irreversible state-changing operation is performed without the safety check that would prevent an unsafe outcome. Any contract can replace its own class with an arbitrary, undeclared hash, rendering itself permanently inaccessible and freezing all funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function explicitly acknowledges the missing check via a TODO comment, then proceeds to unconditionally update the contract's class hash in `contract_state_changes` without verifying that `class_hash` is present in `contract_class_changes` (i.e., has been declared):

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The OS writes the new `StateEntry` with the unvalidated `class_hash` directly into the state dictionary. After this state update is committed, any future call to the contract will attempt to load the class identified by `class_hash`. If that hash is not declared, the call fails unconditionally and permanently.

Compare this to how `execute_declare_transaction` enforces the invariant that `compiled_class_hash != 0` and that `prev_value=0` (class declared only once): [2](#0-1) 

No equivalent guard exists in `execute_replace_class`. The "safe" pattern — verifying the target class is declared before committing the replacement — is entirely absent.

---

### Impact Explanation

Once a contract's class hash is set to an undeclared value and the block is proven:

- Every subsequent call to the contract fails at class-loading time.
- There is no recovery path: the state change is committed to the Merkle tree and the proven output.
- All tokens, balances, or assets held in the contract's storage are permanently inaccessible.

This satisfies **Critical — Permanent freezing of funds**.

---

### Likelihood Explanation

The attack is reachable by an unprivileged **contract deployer**:

1. A malicious deployer publishes a contract that appears legitimate (e.g., a yield vault or escrow).
2. Users deposit funds into the contract.
3. The deployer calls a function on the contract (or the contract itself calls `replace_class`) with an arbitrary undeclared hash.
4. The OS, lacking the validation, commits the replacement.
5. All user funds are permanently frozen.

Alternatively, any contract with insufficient access control on a `replace_class` call path is exploitable by an unprivileged transaction sender. The OS provides zero protocol-level backstop. The TODO comment confirms the developers are aware this check is missing.

---

### Recommendation

Before writing the new `StateEntry` into `contract_state_changes`, verify that `class_hash` exists as a key in `contract_class_changes` (i.e., it has a non-zero compiled class hash mapping). This mirrors the existing invariant enforced during `execute_declare_transaction` and closes the gap at the OS level, regardless of what individual contracts do.

---

### Proof of Concept

1. Declare class `C` (valid, compiled class hash recorded in `contract_class_changes`).
2. Deploy contract `A` using class `C`; fund it with 1000 STRK.
3. From contract `A` (or via a public entry point), invoke the `replace_class` syscall with `new_class_hash = 0xdeadbeef` (never declared).
4. The OS executes `execute_replace_class`:
   - Gas is deducted. [3](#0-2) 
   - The TODO check is skipped. [4](#0-3) 
   - `contract_state_changes[A].class_hash` is set to `0xdeadbeef`.
5. Block is proven and committed.
6. Any subsequent call to contract `A` fails — class `0xdeadbeef` does not exist.
7. The 1000 STRK held in `A`'s storage is permanently frozen.
