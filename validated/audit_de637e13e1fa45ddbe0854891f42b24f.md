### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezal of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts an arbitrary attacker-supplied `class_hash` and writes it directly to the contract state without verifying that the hash corresponds to a previously declared class. This is structurally identical to the reported XSS vulnerability class — untrusted external data is propagated into a trusted context (protocol state) without sanitization. A contract owner can permanently brick their own contract (or any contract they control) by replacing its class with an undeclared hash, permanently freezing any funds held by that contract.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the `class_hash` field directly from the syscall request struct and writes it into `contract_state_changes` via `dict_update` with no check that the hash exists in `contract_class_changes` (the set of declared classes):

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
```

The TODO comment at line 898 explicitly acknowledges the missing check. The OS Cairo code has no constraint — no `assert`, no `dict_read` against `contract_class_changes` — that would prevent an arbitrary felt from being committed as the contract's class hash. [1](#0-0) 

Compare this to the `execute_declare_transaction` flow in `transaction_impls.cairo`, where a class hash is only committed to `contract_class_changes` after `finalize_class_hash` verifies the Sierra component hashes: [2](#0-1) 

No equivalent guard exists in `execute_replace_class`.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once `replace_class` is executed with an undeclared class hash:

1. The OS Cairo code successfully proves the state transition (the `dict_update` is valid Cairo; no constraint is violated).
2. The contract's `class_hash` field in `StateEntry` is committed to the global state root via `compute_contract_state_commitment` → `get_contract_state_hash`.
3. Any subsequent call to the contract (via `call_contract`, `invoke`, etc.) will attempt to look up the class by hash. Since the hash was never declared, the class does not exist, and execution fails unconditionally.
4. There is no recovery path: the contract cannot call `replace_class` again because it cannot be called at all.
5. All ERC-20 tokens, ETH, or other assets held in the contract's storage are permanently inaccessible. [3](#0-2) 

---

### Likelihood Explanation

**High.**

- The entry path requires only an unprivileged contract owner (a standard `invoke` transaction calling `replace_class`).
- No privileged role, leaked key, or external dependency is required.
- The attacker controls the `class_hash` field in `ReplaceClassRequest` directly from transaction calldata.
- The missing check is explicitly acknowledged in the source with a TODO dated 2026, confirming it is a known gap in the current implementation.
- The syscall is dispatched unconditionally from `execute_syscalls` whenever `selector == REPLACE_CLASS_SELECTOR`. [4](#0-3) 

---

### Recommendation

Before writing the new class hash to `contract_state_changes`, verify that it exists in `contract_class_changes` (i.e., it was previously declared). The check should be a Cairo-level assertion, not a hint, so it is enforced by the proof system:

```cairo
// Verify the class has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the pattern used in `execute_declare_transaction`, where `dict_update` with `prev_value=0` enforces that a class may only be declared once and must exist before use. [5](#0-4) 

---

### Proof of Concept

1. Attacker deploys a wallet contract holding 100 ETH. The contract's `class_hash` in state is `0xABC` (a valid declared class).
2. Attacker submits an `invoke` transaction calling `replace_class(class_hash=0xDEADBEEF)`, where `0xDEADBEEF` is never declared.
3. The OS executes `execute_replace_class`. At line 896, `class_hash = 0xDEADBEEF` is read from the request. The TODO-guarded missing check is skipped. `dict_update` writes `StateEntry(class_hash=0xDEADBEEF, ...)` to `contract_state_changes`.
4. The OS proves the block. The new state root commits `class_hash=0xDEADBEEF` for the contract address.
5. Any future `call_contract` or `invoke` targeting the wallet attempts to execute class `0xDEADBEEF`. The class does not exist. Execution fails at the VM level for every future call.
6. The 100 ETH is permanently frozen. No recovery is possible because the contract cannot execute any entry point, including a corrective `replace_class`. [6](#0-5)

func get_contract_state_hash{hash_ptr: HashBuiltin*}(
    class_hash: felt, storage_root: felt, nonce: felt
) -> (hash: felt) {
    const CONTRACT_STATE_HASH_VERSION = 0;
    if (class_hash == UNINITIALIZED_CLASS_HASH) {
        if (storage_root == 0) {
            if (nonce == 0) {
                return (hash=0);
            }
        }
    }

    // Set res = H(H(class_hash, storage_root), nonce).
    let (hash_value) = hash2(class_hash, storage_root);
    let (hash_value) = hash2(hash_value, nonce);

    // Return H(hash_value, CONTRACT_STATE_HASH_VERSION). CONTRACT_STATE_HASH_VERSION must be in the
    // outermost hash to guarantee unique "decoding".
    let (hash) = hash2(hash_value, CONTRACT_STATE_HASH_VERSION);
    return (hash=hash);
}
