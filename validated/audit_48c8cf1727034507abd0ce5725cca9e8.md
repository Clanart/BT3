### Title
Stale Fee-Market Snapshot in Cosmos Ante Handler Allows Fee Bypass for Non-EVM Transactions - (File: `evmd/ante/handler_options.go`)

---

### Summary

`newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712` capture `feemarketParams` as a **value-type snapshot** at construction time and pass a pointer to that frozen copy into both `NewDynamicFeeChecker` and `NewMinGasPriceDecorator`. Because the snapshot is never refreshed, every Cosmos (non-EVM) transaction processed after the base fee or `MinGasPrice` has changed is validated against stale values. The Ethereum ante handler (`newEthAnteHandler`) does not share this flaw — it re-fetches live params on every invocation.

---

### Finding Description

**Root cause — `newCosmosAnteHandler`** (`evmd/ante/handler_options.go`, lines 178–211):

```go
func newCosmosAnteHandler(ctx sdk.Context, options HandlerOptions, ...) sdk.AnteHandler {
    evmParams        := options.EvmKeeper.GetParams(ctx)          // one-time snapshot
    feemarketParams  := options.FeeMarketKeeper.GetParams(ctx)    // one-time snapshot
    ...
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
    ...
    cosmos.NewMinGasPriceDecorator(options.FeeMarketKeeper, evmDenom, &feemarketParams)
```

`feemarketParams` is a local `feemarkettypes.Params` value. `&feemarketParams` is passed to both decorators and captured in their closures/structs. The pointer keeps the local alive on the heap, but its contents are **never mutated** after construction.

**`NewDynamicFeeChecker`** (`ante/evm/fee_checker.go`, line 56) reads the base fee from the snapshot:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
```

`types.GetBaseFee` (`x/evm/types/utils.go`, line 248) calls `feemarketParams.GetBaseFee()` — always the construction-time value.

**`MinGasPriceDecorator.AnteHandle`** (`ante/cosmos/min_gas_price.go`, line 54) reads `MinGasPrice` from the same snapshot:

```go
minGasPrice := mpd.feemarketParams.MinGasPrice
```

**Contrast with `newEthAnteHandler`** (`evmd/ante/handler_options.go`, line 88), which fetches live params on every call:

```go
blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
feemarketParams := &blockCfg.FeeMarketParams   // fresh every tx
```

The base fee is updated every block in `BeginBlock` (`x/feemarket/keeper/abci.go`, line 38: `k.SetBaseFee(ctx, baseFee)`). `MinGasPrice` can be changed at any time via governance (`x/feemarket/keeper/params.go`, line 40: `k.SetParams`). Neither change is ever reflected in the Cosmos ante handler's snapshot.

---

### Impact Explanation

**High — ante handler bug that permits invalid transactions to commit and causes fee mis-accounting.**

Two concrete sub-impacts:

1. **Base-fee underpayment**: After the EIP-1559 base fee rises (e.g., due to sustained high block utilization), Cosmos-SDK transactions whose `feeCap` satisfies the stale (lower) base fee but not the current one pass `NewDynamicFeeChecker` and are included in blocks. The `DeductFeeDecorator` deducts the stale-fee-derived `effectiveFee`, so the chain collects less fee revenue than the protocol requires and the fee-market invariant is violated.

2. **`MinGasPrice` governance bypass**: After a governance proposal raises `MinGasPrice`, `MinGasPriceDecorator` continues enforcing the old, lower floor for Cosmos transactions. Transactions that should be rejected as spam are admitted.

Both paths allow unprivileged users to submit transactions that the current fee-market rules should reject, constituting invalid-tx admission and fee mis-accounting.

---

### Likelihood Explanation

**Medium.** The base fee adjusts every block whenever gas usage deviates from the target. On a busy chain the base fee can drift significantly within minutes. Any long-running node (no restart since genesis or last upgrade) will have a stale snapshot. A user who monitors the on-chain base fee via `eth_gasPrice` / `feemarket` query and compares it to the snapshot value (observable by querying the node at startup height) can craft Cosmos transactions that exploit the gap. No privileged access is required; only a standard Cosmos SDK transaction broadcast.

---

### Recommendation

Replace the construction-time snapshot with a live read inside the ante handler closure, mirroring the pattern already used in `newEthAnteHandler`:

1. In `newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712`, remove the upfront `GetParams` calls and the `&feemarketParams` pointer passing.
2. Pass the `FeeMarketKeeper` interface directly to `NewDynamicFeeChecker` and `NewMinGasPriceDecorator` so they can call `keeper.GetParams(ctx)` on each invocation, consistent with how `newEthAnteHandler` calls `EVMBlockConfig` per transaction.
3. Alternatively, restructure `newCosmosAnteHandler` to accept no `ctx` at construction and instead fetch all params lazily inside the returned closure.

---

### Proof of Concept

1. Node starts; genesis base fee = 1 000 000 000 (1 Gwei). `newCosmosAnteHandler` is called once; snapshot `BaseFee = 1e9`.
2. Network is busy for N blocks; `BeginBlock` raises the live base fee to 2 000 000 000 (2 Gwei) via `k.SetBaseFee`.
3. Attacker submits a Cosmos SDK `MsgSend` with `fee = 1.5e9 * gasLimit` (above stale snapshot, below live base fee).
4. `NewDynamicFeeChecker` evaluates `feeCap = 1.5e9 >= baseFee(snapshot) = 1e9` → passes.
5. `MinGasPriceDecorator` evaluates against stale `MinGasPrice` → passes.
6. `DeductFeeDecorator` deducts `effectiveFee` computed from the stale base fee.

func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
	if !IsLondon(ethCfg, height) {
		return nil
	}

		return new(big.Int)
	}
	return baseFee

func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
