### Title
Integer Division Truncation in `NewDynamicFeeChecker` Silently Under-Deducts Cosmos Tx Fees — (File: ante/evm/fee_checker.go)

### Summary
`NewDynamicFeeChecker` derives the per-gas fee cap for Cosmos SDK transactions by integer-dividing the total stated fee by the gas limit. Because Go integer division truncates, the computed `feeCap` is always `≤` the true per-gas value. The `effectiveFee` returned to `DeductFeeDecorator` is therefore `floor(fee/gas) × gas`, which is strictly less than the user's stated `fee` whenever `fee mod gas ≠ 0`. The fee collector receives less than the amount the transaction declared it would pay, constituting a systematic fee mis-accounting in the ante handler.

---

### Finding Description

In `ante/evm/fee_checker.go`, `NewDynamicFeeChecker` computes:

```go
gas     := feeTx.GetGas()                          // line 79
fee     := feeCoins.AmountOf(denom)                // line 81
feeCap  := fee.Quo(sdkmath.NewIntFromUint64(gas))  // line 83  ← integer division
```

`sdkmath.Int.Quo` is integer (floor) division. Whenever `fee` is not an exact multiple of `gas`, the fractional part is silently discarded.

The truncated `feeCap` is then fed directly into the EIP-1559 effective-price formula:

```go
effectivePrice := sdkmath.NewIntFromBigInt(
    types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
// effectivePrice = min(baseFee + tip, feeCap)

effectiveFee := sdk.Coins{{
    Denom:  denom,
    Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas)),  // line 97
}}
```

When `effectivePrice` is capped at the truncated `feeCap`, the returned `effectiveFee` equals `floor(fee/gas) × gas`, not `fee`. The shortfall is exactly `fee mod gas` (up to `gas − 1` atoms per transaction).

`DeductFeeDecorator` uses this returned value as the amount to deduct from the sender and credit to the fee collector:

```go
// ante/evm/nativefee.go line 59
fee, priority, err = dfd.txFeeChecker(ctx, tx)
...
if err := dfd.checkDeductFee(ctx, tx, fee); err != nil { ... }
```

The `fee mod gas` remainder stays in the sender's account; the fee collector never receives it.

---

### Impact Explanation

This is a fee-accounting bug in the ante handler path for Cosmos SDK transactions that use the dynamic fee checker (`ExtensionOptionDynamicFeeTx` or the default Cosmos ante handler when `DynamicFeeChecker = true`). Every such transaction where `fee mod gas ≠ 0` causes the fee collector to receive less than the declared fee. The maximum shortfall per transaction is `gas − 1` atoms. For a transaction with `gas = 10 000 000` and `baseFee = 1 000 000 000` (1 Gwei), the maximum per-transaction shortfall is `9 999 999` atoms (≈ 0.00001 native token). Across many transactions this accumulates into a systematic under-collection of protocol fees. This matches the allowed High impact: *"ante handler … bug that permits … valid user funds/fees to be mis-accounted."*

---

### Likelihood Explanation

The condition `fee mod gas ≠ 0` is the common case: any fee amount that is not an exact integer multiple of the gas limit triggers the truncation. Standard wallets and SDKs routinely produce such values (e.g., `fee = gasPrice × gasLimit` where `gasPrice` is a decimal). The path is reachable by any unprivileged user submitting a Cosmos SDK transaction on a chain with London HF enabled and `DynamicFeeChecker = true`.

---

### Recommendation

Replace the truncating integer division with a ceiling division, or compute the effective fee as the minimum of the stated fee and `effectivePrice × gas`:

```go
// Option A: ceiling division for feeCap
feeCap := fee.Add(sdkmath.NewIntFromUint64(gas - 1)).Quo(sdkmath.NewIntFromUint64(gas))

// Option B: cap effectiveFee at the stated fee
effectiveFeeAmt := effectivePrice.Mul(sdkmath.NewIntFromUint64(gas))
if effectiveFeeAmt.GT(fee) {
    effectiveFeeAmt = fee
}
```

Option A preserves the EIP-1559 semantics (feeCap = ceil(fee/gas)) and ensures the fee collector always receives at least the effective fee. Option B directly prevents over-refunding.

### Proof of Concept

| Parameter | Value |

**Execution**:

1. `feeCap = 1 999 999 / 1 000 000 = 1` (truncated from 1.999999) [1](#0-0)
