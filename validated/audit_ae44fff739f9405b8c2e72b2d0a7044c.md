### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary felt value as the new class hash without verifying that it corresponds to a previously declared contract class. This is an exact structural analog to the CVGT report's root cause: a state reference is accepted and committed without legitimacy validation. An unprivileged contract deployer can exploit this to permanently freeze funds held by a contract.

---

### Finding Description

In `execute_replace_class`, the new class hash is read directly from the user-controlled syscall request and written into the contract state dictionary with no validation:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The `TODO` comment at line 898 explicitly acknowledges the missing check. The `class_hash` field originates from `request.class_hash`, which is fully attacker-controlled via the syscall interface. [2](#0-1) 

Once an invalid class hash is committed to `contract_state_changes`, any subsequent call to that contract reaches `execute_entry_point`, which performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // ← the invalid hash
);
// compiled_class_hash == 0 (default for undeclared key)

let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,           // ← 0, never present
);
``` [3](#0-2) 

`find_element` with key `0` finds no compiled class fact and panics, making the contract permanently uncallable. All funds held in or exclusively accessible through that contract are frozen.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once `replace_class` commits an undeclared class hash to the on-chain state, the contract's entry point dispatch is irrecoverably broken. No future transaction can successfully call the contract; every attempt fails at the OS level when resolving the compiled class. Any ERC-20 balances, ETH, or protocol-controlled assets stored in the contract's storage become permanently inaccessible.

---

### Likelihood Explanation

**High.** The attack requires only:
1. Deploying a contract (permissionless on StarkNet).
2. Attracting user deposits (social engineering or a legitimate-looking DeFi interface).
3. Calling `replace_class` with an arbitrary felt value (e.g., `1`) as the new class hash.

No privileged access, leaked keys, or external dependency compromise is required. The missing check is explicitly flagged in the source with a `TODO`, confirming it is a known gap in the current implementation.

---

### Recommendation

In `execute_replace_class`, before writing the new class hash to `contract_state_changes`, verify that `request.class_hash` exists as a key in `contract_class_changes` (i.e., it has been declared in the current or a prior block). Concretely:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(
    key=class_hash
);
assert_not_zero(compiled_class_hash);
```

This mirrors the check already performed implicitly in `execute_entry_point` but must be enforced at the point of mutation, not at the point of use.

---

### Proof of Concept

1. Attacker deploys contract `C` with a legitimate class hash.
2. Users deposit tokens into `C` (e.g., via a `deposit` entry point).
3. Attacker sends an invoke transaction from `C` that calls the `replace_class` syscall with `class_hash = 0xdeadbeef` (any undeclared felt).
4. `execute_replace_class` writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes` for address `C` with no validation. [4](#0-3) 
5. The state is committed on-chain.
6. Any subsequent call to `C` reaches `execute_entry_point`, which calls `dict_read(key=0xdeadbeef)` on `contract_class_changes`, returning `0` (undeclared). [5](#0-4) 
7. `find_element(key=0)` fails — the contract is permanently uncallable and all deposited funds are frozen. [6](#0-5)
