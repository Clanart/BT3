### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary class hash as the replacement class without verifying that the hash corresponds to a declared (known) contract class. A malicious or buggy contract can replace its own class with an undeclared hash, rendering the contract permanently uncallable and freezing all funds held in its storage.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The code itself contains a `TODO` comment explicitly acknowledging the missing check. The function writes `class_hash` directly from the syscall request into `contract_state_changes` with no validation against `contract_class_changes` (the dictionary of declared classes) or `compiled_class_facts_bundle` (the set of compiled classes available for execution).

This is the direct analog to the Gearbox `creditManager` vulnerability: just as unrestricted parameter changes to `creditManager` can break dependent accounts, unrestricted class replacement in the OS can permanently break a contract and freeze its funds.

The OS global context holds the compiled class facts bundle, which is the authoritative set of classes available for execution:

```cairo
struct OsGlobalContext {
    starknet_os_config: StarknetOsConfig,
    ...
    compiled_class_facts_bundle: CompiledClassFactsBundle,
    ...
}
``` [2](#0-1) 

When a contract is later called, the OS resolves its class hash from `contract_state_changes` and looks up the compiled class in `compiled_class_facts_bundle`. If the class hash set by `replace_class` is not present in that bundle (because it was never declared), execution of the contract fails unconditionally and permanently.

---

### Impact Explanation

**Critical — Permanent Freezing of Funds.**

Once a contract's class hash is replaced with an undeclared hash:

1. Every subsequent call to the contract fails at the class-lookup stage.
2. The contract's storage (including token balances, locked collateral, LP positions, etc.) becomes permanently inaccessible.
3. There is no recovery path: the contract cannot be called to withdraw funds, and the OS has no mechanism to revert a committed class replacement.

This matches the allowed impact: **Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

The attack is reachable by any **unprivileged contract deployer**:

1. Attacker deploys a contract that appears legitimate (e.g., a vault, DEX, or lending pool).
2. Users deposit funds into the contract.
3. Attacker triggers `replace_class` from within the contract (e.g., via an admin function, a rug-pull backdoor, or by exploiting a re-entrancy path) with an arbitrary undeclared class hash.
4. The OS accepts the replacement without validation.
5. The contract's class is committed to state as an undeclared hash.
6. All future calls to the contract revert. Funds are frozen.

The `replace_class` syscall is a standard, publicly available syscall. No privileged role, leaked key, or operator cooperation is required beyond the ability to deploy and call a contract — both of which are unprivileged operations on StarkNet.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, the OS must verify that the requested class hash exists in the declared class set. Concretely, `execute_replace_class` should assert that `class_hash` has a non-zero entry in `contract_class_changes` (i.e., it was declared in a prior or current block). This is exactly what the existing `TODO` comment calls for:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
```

The fix should perform a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert the result is non-zero before proceeding with the state update.

---

### Proof of Concept

**Step 1 — Deploy a malicious vault contract** that holds user funds and exposes a `replace_class(new_hash)` entry point callable by the deployer.

**Step 2 — Users deposit funds.** The contract accumulates token balances in its storage.

**Step 3 — Attacker calls `replace_class`** with a class hash that has never been declared (e.g., `0xdeadbeef`). The syscall reaches `execute_replace_class` in the OS.

**Step 4 — OS processes the syscall** at `syscall_impls.cairo` lines 896–910:
```cairo
let class_hash = request.class_hash;  // = 0xdeadbeef, undeclared
// No check performed here (TODO acknowledged)
tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, ...  // 0xdeadbeef written to state
);
dict_update{dict_ptr=contract_state_changes}(...);
``` [3](#0-2) 

**Step 5 — State is committed.** The contract's class hash in the Patricia tree is now `0xdeadbeef`.

**Step 6 — Any subsequent call to the contract** causes the OS to look up `0xdeadbeef` in `compiled_class_facts_bundle`. The class does not exist. Execution fails. The contract is permanently bricked. All user funds are frozen.

    range_check_ptr,
    syscall_ptr: felt*,

    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;
