### Title
L1Provider In-Memory State Not Persisted Across Restarts Causes Valid L1Handler Transactions to Return `NotFound` During Block Validation — (File: `crates/apollo_l1_provider/src/l1_provider.rs`, `crates/apollo_l1_provider/src/l1_scraper.rs`)

---

### Summary

The `L1Provider`'s entire transaction state (`TransactionManager` with `records`, `proposable_index`, `consumed_queue`) is held purely in memory and is never written to durable storage. On every restart the `L1Scraper` rewinds by `startup_rewind_time_seconds` (6 hours in the production deployment config) and re-scrapes only that window of L1 events. Any L1 handler transaction that was sent to L1 **more than 6 hours before the restart** and has **not yet been committed to L2** is permanently absent from the provider's records after restart. When a validator subsequently calls `validate()` for such a transaction, the provider returns `InvalidValidationStatus::NotFound`, which causes `ValidateTransactionProvider` to return `L1HandlerTransactionValidationFailed`, rejecting the block.

---

### Finding Description

**Root cause — no persistence of L1 provider state.**

`L1Provider::new()` initialises an empty `TransactionManager` on every process start: [1](#0-0) 

The `TransactionManager` itself is a pure in-memory structure with no storage backend: [2](#0-1) 

**Root cause — fixed rewind window on every startup.**

`L1Scraper::fetch_start_block()` always rewinds by `startup_rewind_time_seconds` from the current L1 tip, regardless of what was previously scraped: [3](#0-2) 

The production deployment sets this to 21 600 seconds (6 hours): [4](#0-3) 

`L1Scraper::run()` calls `fetch_start_block()` unconditionally on every startup, then calls `initialize()` which sends only the events from that window to the provider: [5](#0-4) 

**Broken invariant — missing transactions return `NotFound`.**

`TransactionManager::validate_tx()` returns `InvalidValidationStatus::NotFound` for any transaction hash not present in `records`: [6](#0-5) 

The codebase itself acknowledges that old transactions may be absent after restart, but only for cancellation events — the same gap exists for the transaction payloads: [7](#0-6) 

**Impact path — block validation fails.**

`ValidateTransactionProvider::get_txs()` treats any `Invalid` status (including `NotFound`) as a hard block-validation failure: [8](#0-7) 

---

### Impact Explanation

**High. Valid L1 handler transactions are rejected during block validation before sequencing.**

A validator node whose `L1Provider` was restarted within the last 6 hours will return `NotFound` for any L1 handler transaction that was sent to L1 more than 6 hours ago and has not yet been committed to L2. The `ValidateTransactionProvider` converts this into a hard `L1HandlerTransactionValidationFailed` error, causing the entire block to fail validation. The valid L1 handler transaction is never sequenced.

---

### Likelihood Explanation

**Medium.** The conditions are:

1. An L1 handler transaction is sent to L1 and remains uncommitted to L2 for longer than `startup_rewind_time_seconds` (6 hours in production). This can occur during periods of high L2 backlog or sequencer downtime.
2. The sequencer restarts (planned upgrade, crash, pod eviction). Restarts are routine in production Kubernetes deployments.

Both conditions are reachable without any privileged access. Any user who sends an L1 handler transaction and observes a sequencer restart more than 6 hours later can trigger this path.

---

### Recommendation

**Short term:** Persist the `TransactionManager` state (records, proposable index, consumed queue, and the last scraped L1 block reference) to the existing MDBX storage atomically with each batch of processed events. On restart, load the persisted state and resume scraping from the last persisted L1 block reference rather than rewinding by a fixed time window.

**Long term:** Add integration tests that restart the scraper/provider pair after a configurable delay and assert that all previously scraped, uncommitted L1 handler transactions are still present and validatable. Enforce that `startup_rewind_time_seconds` is always larger than the maximum observed L1-to-L2 commitment latency, or remove the time-based rewind entirely in favour of the persisted offset.

---

### Proof of Concept

1. User sends an L1 handler transaction to the Starknet core contract on L1 at time **T**. The `L1Scraper` scrapes it and `L1Provider::add_events()` inserts it into `TransactionManager::records`.
2. The transaction is not yet committed to L2 (e.g., due to a sequencer backlog).
3. At time **T + 7 h**, the sequencer process restarts (upgrade, crash, pod eviction).
4. `L1Scraper::fetch_start_block()` rewinds 6 hours from the current L1 tip, yielding a start block at approximately **T + 1 h**. The transaction from time **T** is before this window and is never re-scraped.
5. `L1Provider` is initialised with an empty `TransactionManager`; the transaction is absent from `records`.
6. The proposer includes the transaction in a new block proposal (it is still valid on L1).
7. The validator calls `L1Provider::validate(tx_hash, height)`.
8. `TransactionManager::validate_tx()` finds no record for `tx_hash` and returns `InvalidValidationStatus::NotFound`.
9. `ValidateTransactionProvider::get_txs()` receives `Invalid(NotFound)` and returns `Err(L1HandlerTransactionValidationFailed { tx_hash, validation_status: NotFound })`.
10. Block validation fails. The valid L1 handler transaction is rejected and never sequenced.

#[derive(Clone, Debug, PartialEq, Eq)]
pub struct TransactionManager {
    /// Storage of all l1 handler transactions --- keeps transactions until they can be safely
    /// removed, like when they are consumed on L1, or fully cancelled on L1.
    pub records: Records,
    pub config: TransactionManagerConfig,
    /// Ordered lexicographically by scraping moment timestamp, then order-of-arrival for
    /// identical timestamps, also at any point the staged transactions are a prefix of the
    /// structure under this order.
    /// Invariant: contains all hashes of transactions that are proposable, and only them.
    /// Invarariant 2: Once removed from this index, a transaction will never be proposed again.
    proposable_index: BTreeMap<UnixTimestamp, Vec<TransactionHash>>,
    /// Generation counter used to prevent double usage of an l1 handler transaction in a single
    /// block.
    /// Calling `get_txs` or `validate_tx` tags the touched transactions with the current block
    /// counter, so that further calls will know not to touch them again.
    /// At the start and end (commit) of every block, the counter is incremented, thus "unstaging"
    /// all tagged transactions from the previous block attempt.
    // TODO(Gilad): remove "for rejected" from name when uncommitted is migrated to records DS.
    current_staging_epoch: StagingEpoch,
    /// All consumed transactions that are waiting to be removed from the transaction manager.
    /// Invariant: Ordered lexicographically by the block timestamp where they were marked as
    /// consumed, then order-of-arrival for identical timestamps.
    /// Invariant 2: A transaction is in the queue iff it is in the records and marked as consumed.
    consumed_queue: BTreeMap<BlockTimestamp, Vec<TransactionHash>>,
}

impl TransactionManager {
    pub fn new(
        l1_handler_proposal_cooldown_seconds: Duration,
        l1_handler_cancellation_timelock_seconds: Duration,
        l1_handler_consumption_timelock_seconds: Duration,
    ) -> Self {
        Self {
            config: TransactionManagerConfig {
                l1_handler_proposal_cooldown_seconds,
                l1_handler_cancellation_timelock_seconds,
                l1_handler_consumption_timelock_seconds,
            },
            records: Default::default(),
            proposable_index: Default::default(),
            current_staging_epoch: StagingEpoch::new(),
            consumed_queue: Default::default(),
        }
    }

    pub fn validate_tx(&mut self, tx_hash: TransactionHash, unix_now: u64) -> ValidationStatus {
        let current_staging_epoch_cloned = self.current_staging_epoch;

        let policy = TransactionRecordPolicy {
            cancellation_timelock: self.config.l1_handler_cancellation_timelock_seconds,
        };

        let validation_status = self.with_record(tx_hash, |record| {
            // If the current time affects the state, update state now.
            record.update_time_based_state(unix_now, policy);
            if !record.is_validatable() {
                match record.state {
                    TransactionState::Committed => {
                        InvalidValidationStatus::AlreadyIncludedOnL2.into()
                    }
                    TransactionState::CancelledOnL2 => {
                        InvalidValidationStatus::CancelledOnL2.into()
                    }
                    TransactionState::Consumed => InvalidValidationStatus::ConsumedOnL1.into(),
                    _ => unreachable!(),
                }
            } else if record.try_mark_staged(current_staging_epoch_cloned) {
                ValidationStatus::Validated
            } else {
                InvalidValidationStatus::AlreadyIncludedInProposedBlock.into()
            }
        });

        validation_status.unwrap_or(InvalidValidationStatus::NotFound.into())
    }

    #[instrument(skip(self), err)]
    pub async fn run(&mut self) -> L1ScraperResult<(), BaseLayerType> {
        // Each of the following function is use as a closure sent to
        // retry_until_base_layer_success. This will retry each of them, in turn, in an
        // endless loop until they succeed. Since they have different return types, we have
        // to use box/pin to pass them as a closure.

        // Try to get all the information needed for initialization.
        self.scrape_from_this_l1_block = Some(
            self.retry_until_base_layer_success(
                |this| this.fetch_start_block().boxed(),
                "fetching start block",
            )
            .await?,
        );

        // The last historic L2 height is returned, to be using in initialization.
        let historic_l2_height = self
            .retry_until_base_layer_success(
                |this| this.get_last_historic_l2_height().boxed(),
                "fetching last historic L2 height",
            )
            .await?;

        // Initialize fetches events from start block up to latest, sends them to the provider.
        // Update the L1 block number we need to scrape from.
        self.scrape_from_this_l1_block = Some(
            self.retry_until_base_layer_success(
                |this| this.initialize(historic_l2_height).boxed(),
                "initializing",
            )
            .await?,
        );

        // This is the main (steady state) loop.
        loop {
            // Sleep at start of loop, as we get here right after successful initialize+break.
            sleep(self.config.polling_interval_seconds).await;

            match self.send_events_to_l1_provider().await {
                Err(L1ScraperError::BaseLayerError(e)) => {
                    L1_MESSAGE_SCRAPER_BASELAYER_ERROR_COUNT.increment(1);
                    warn!("BaseLayerError during scraping: {e:?}");
                }
                Ok(_) => {
                    L1_MESSAGE_SCRAPER_SUCCESS_COUNT.increment(1);
                    set_unix_now_seconds(&L1_MESSAGE_SCRAPER_LAST_SUCCESS_TIMESTAMP_SECONDS);
                }
                Err(e) => return Err(e),
            }
        }
    }

    /// Use config.startup_rewind_time_seconds to estimate an L1 block number
    /// that is far enough back to start scraping from.
    pub async fn fetch_start_block(
        &mut self,
    ) -> Result<L1BlockReference, L1ScraperError<BaseLayerType>> {
        // Define the safety margin (e.g., extra 50% over the required number of blocks).
        const SAFTEY_MARGIN_NUMERATOR: u64 = 3;
        const SAFTEY_MARGIN_DENOMINATOR: u64 = 2;
        const_assert!(SAFTEY_MARGIN_NUMERATOR > SAFTEY_MARGIN_DENOMINATOR);
        let finality = self.config.finality;
        let latest_l1_block_number = self
            .base_layer
            .latest_l1_block_number()
            .await
            .map_err(L1ScraperError::BaseLayerError)?;
        let latest_l1_block_number = latest_l1_block_number.checked_sub(finality).ok_or(
            L1ScraperError::LatestBlockNumberTooLow {
                latest_l1_block_no_finality: latest_l1_block_number,
                finality,
            },
        )?;
        debug!("Latest L1 block number: {latest_l1_block_number:?}");

        // Estimate the number of blocks in the interval, to rewind from the latest block.
        let blocks_in_interval = self.config.startup_rewind_time_seconds.as_secs()
            / self.config.l1_block_time_seconds.as_secs();
        debug!("Blocks in interval: {blocks_in_interval}");

        // Add 50% safety margin.
        let safe_blocks_in_interval = blocks_in_interval + blocks_in_interval / 2;
        debug!("Safe blocks in interval: {safe_blocks_in_interval}");

        let l1_block_number_rewind = latest_l1_block_number.saturating_sub(safe_blocks_in_interval);
        debug!("L1 block number rewind: {l1_block_number_rewind}");

        let block_reference_rewind = self
            .base_layer
            .l1_block_at(l1_block_number_rewind)
            .await
            .map_err(L1ScraperError::BaseLayerError)?
            .unwrap_or_else(|| {
                panic!(
                    "Rewound L1 block number is between 0 and the verified latest L1 block \
                     {latest_l1_block_number}, so should exist",
                )
            });
        debug!("Block reference rewind: {block_reference_rewind:?}");

        Ok(block_reference_rewind)
    }
