### Title
`replace_class` Syscall Accepts Zero Class Hash, Enabling Contract State Reset and Fund Loss - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

`execute_replace_class` does not validate that the supplied `class_hash` is non-zero. Because `UNINITIALIZED_CLASS_HASH = 0` is the exact sentinel value that `deploy_contract` uses to decide whether an address is available for fresh deployment, a contract can call `replace_class(0)` to make itself appear undeployed. For contracts whose nonce is still `0` (all contracts deployed via the `deploy` syscall, not `deploy_account`), this satisfies both preconditions checked by `deploy_contract`, enabling re-deployment to the same address, re-execution of the constructor, and overwriting of storage that previously held user funds.

---

### Finding Description

**Root cause — missing zero-check in `execute_replace_class`** [1](#0-0) 

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The in-code `TODO` explicitly acknowledges the missing validation. No `assert_not_zero(class_hash)` is present. Passing `class_hash = 0` is accepted silently.

**The sentinel value `UNINITIALIZED_CLASS_HASH = 0` is the exact guard used by `deploy_contract`** [2](#0-1) [3](#0-2) 

```cairo
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}
assert state_entry.class_hash = UNINITIALIZED_CLASS_HASH;   // must be 0
assert state_entry.nonce = 0;                               // must be 0

tempvar new_state_entry = new StateEntry(
    class_hash=constructor_execution_context.class_hash,
    storage_ptr=state_entry.storage_ptr,   // ← existing storage is PRESERVED
    nonce=0,
);
```

After `replace_class(0)`, the contract's `class_hash` is `0`. Both assertions pass for any contract whose nonce is still `0`.

**Nonce stays `0` for all contracts deployed via the `deploy` syscall**

`check_and_increment_nonce` is called only inside account-transaction handlers (`execute_invoke_function_transaction`, `execute_deploy_account_transaction`, `execute_declare_transaction`). [4](#0-3) 

Contracts deployed via the `deploy` syscall (ERC-20 tokens, vaults, AMM pools, etc.) are never account contracts and their nonce remains `0` indefinitely.

**Address computation is deterministic — re-deployment to the same address is trivial** [5](#0-4) 

```cairo
let (contract_address) = get_contract_address(
    salt=request.contract_address_salt,
    class_hash=request.class_hash,
    constructor_calldata_size=constructor_calldata_size,
    constructor_calldata=constructor_calldata_start,
    deployer_address=deployer_address,
);
```

Re-submitting the original `(salt, class_hash, calldata, deployer)` tuple yields the same address. The attacker already knows these values because they performed the original deployment.

**`get_contract_state_hash` treats `class_hash = 0` as the "empty contract" leaf** [6](#0-5) 

```cairo
if (class_hash == UNINITIALIZED_CLASS_HASH) {
    if (storage_root == 0) {
        if (nonce == 0) {
            return (hash=0);
        }
    }
}
```

Setting `class_hash = 0` also corrupts the Patricia-tree leaf hash for the contract, making the state commitment inconsistent with the actual on-chain storage.

---

### Impact Explanation

**Critical — Direct loss of funds / Permanent freezing of funds.**

*Path A — Re-deployment and fund theft (nonce = 0 contracts):*

1. Attacker deploys contract C (e.g., a fake vault) via `deploy` syscall → `{class_hash: X, nonce: 0}`.
2. Users deposit funds; storage slot `balances[user]` accumulates value.
3. Attacker triggers `replace_class(0)` on C → `{class_hash: 0, nonce: 0}`.
4. Attacker re-deploys with identical parameters → address resolves to C again.
5. `deploy_contract` passes both assertions; constructor re-runs with `storage_ptr = old_storage_ptr`.
6. Constructor overwrites `balances[user]` → all user funds stolen.

*Path B — Permanent freeze (any contract):*

Even without re-deployment, `class_hash = 0` means no entry points exist. Every subsequent call to C reverts. Funds locked in C are permanently inaccessible.

---

### Likelihood Explanation

The attacker only needs to:
1. Deploy a contract (public, unprivileged role) that exposes a path calling `replace_class(0)` on itself.
2. Attract deposits.
3. Execute the attack.

No privileged key, operator access, or external dependency is required. The missing check is explicitly flagged in the source with a `TODO` comment, confirming the developers are aware the validation is absent. The attack is fully self-contained within a single block.

---

### Recommendation

Inside `execute_replace_class`, before updating `contract_state_changes`, add:

```cairo
assert_not_zero(class_hash);  // Prevent reset to UNINITIALIZED_CLASS_HASH sentinel
```

Additionally, implement the acknowledged TODO: verify that `class_hash` exists as a key in `contract_class_changes` (i.e., it has been declared in the current or a prior block) before accepting the replacement. [7](#0-6)

### Proof of Concept

```
Block N:
  Tx 1: deploy(salt=S, class_hash=X, calldata=D, deployer=A)
        → address C = hash(S, X, D, A)
        → state[C] = {class_hash: X, nonce: 0, storage: {}}

  Tx 2..N-1: users call C.deposit() → storage[C][balances[u]] = V

Block N+1:
  Tx 1: invoke C.evil_replace()  (calls replace_class syscall with class_hash=0)
        → state[C] = {class_hash: 0, nonce: 0, storage: {balances[u]: V}}

  Tx 2: deploy(salt=S, class_hash=X, calldata=D, deployer=A)
        → address = hash(S, X, D, A) = C  ← same address
        → deploy_contract checks:
              state[C].class_hash == UNINITIALIZED_CLASS_HASH (0)  ✓
              state[C].nonce == 0                                   ✓
        → new_state[C] = {class_hash: X, storage_ptr: old_ptr, nonce: 0}
        → constructor runs, writes storage[C][balances[u]] = 0
        → all user funds erased / redirected to attacker
```

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
