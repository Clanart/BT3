### Title
Unregistered Token Accepted in `initTransfer()` Causes Permanent Irrecoverable Lock of User Funds - (File: `evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.initTransfer()` accepts any arbitrary ERC20 token address without verifying the token is registered in the bridge's `ethToNearToken` mapping. When a user passes an unregistered token, the tokens are transferred into the bridge contract and an `InitTransfer` event is emitted, but the NEAR side will panic with `TokenNotRegistered` when the relayer attempts to finalize the transfer. There is no refund or recovery path on the EVM side, resulting in permanent loss of user funds.

---

### Finding Description

`initTransfer()` contains three branches for handling the token:

1. `customMinters[tokenAddress] != address(0)` → burn via custom minter
2. `isBridgeToken[tokenAddress]` → burn bridge token
3. **else** → `IERC20(tokenAddress).safeTransferFrom(msg.sender, address(this), amount)`

The third branch is a catch-all that accepts **any** ERC20 token address, including tokens that have never been registered in the bridge via `deployToken()` or `addCustomToken()`. There is no check that `ethToNearToken[tokenAddress]` is non-empty before accepting the deposit. [1](#0-0) 

After the tokens are locked, the `InitTransfer` event is emitted with the unregistered `tokenAddress`: [2](#0-1) 

When the relayer picks up this event and calls `sign_transfer` on NEAR, the bridge calls `get_token_id()` on the token address. For a non-NEAR address, this performs a lookup in `token_address_to_id` and panics with `BridgeError::TokenNotRegistered` if the token is not registered: [3](#0-2) 

The EVM contract has no admin rescue function, no refund mechanism, and no way to recover tokens locked via `initTransfer()`. The `finTransfer()` path only releases tokens that were locked by a NEAR-originated transfer signed by the MPC. The user's tokens are permanently stuck.

The same pattern exists in the StarkNet bridge's `init_transfer()`, which also has an `else` branch that accepts any ERC20 via `transfer_from` without checking `starknet_to_near_token`: [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent irrecoverable lock of user funds in bridge flows.**

Any user who calls `initTransfer()` with an ERC20 token that is not registered in the bridge (i.e., `ethToNearToken[tokenAddress]` is empty) will have their tokens permanently locked in the `OmniBridge` contract with no recovery path. The NEAR side will reject the transfer, and the EVM side has no mechanism to refund or rescue the locked tokens.

---

### Likelihood Explanation

**High.** `initTransfer()` is a public, unpermissioned function callable by any user. No off-chain tooling or UI guard is enforced at the contract level. A user who mistakenly (or deliberately) passes an unregistered token address will trigger this path. The `else` branch is the default for all non-bridge, non-custom-minter tokens, making it easy to reach.

---

### Recommendation

Add a validation check at the start of `initTransfer()` to require that the token is registered in the bridge before accepting it:

```solidity
require(
    tokenAddress == address(0) ||
    isBridgeToken[tokenAddress] ||
    customMinters[tokenAddress] != address(0) ||
    bytes(ethToNearToken[tokenAddress]).length > 0,
    "ERR_TOKEN_NOT_REGISTERED"
);
```

Apply the same guard to `initTransfer1155()` and the StarkNet `init_transfer()`.

---

### Proof of Concept

1. Deploy any arbitrary ERC20 token `ArbitraryToken` (not registered in the bridge).
2. Approve `OmniBridge` to spend `ArbitraryToken`.
3. Call `OmniBridge.initTransfer(address(ArbitraryToken), 1000, 0, 0, "victim.near", "")`.
4. `ArbitraryToken.safeTransferFrom(user, bridge, 1000)` succeeds; `InitTransfer` event is emitted.
5. Relayer calls `sign_transfer` on NEAR with `transfer_id` referencing `ArbitraryToken`'s EVM address.
6. NEAR's `get_token_id()` panics: `ERR_TOKEN_NOT_REGISTERED` — the transfer is never finalized.
7. The 1000 `ArbitraryToken` units are permanently locked in `OmniBridge` with no recovery path. [5](#0-4) [6](#0-5)
