### Title
Stale `feemarketParams` Snapshot in `newCosmosAnteHandler` Bypasses Dynamic Fee Validation - (File: evmd/ante/handler_options.go)

### Summary
`newCosmosAnteHandler` captures `feemarketParams` (including `BaseFee` and `MinGasPrice`) once at handler construction time and passes a pointer to this frozen snapshot into both `NewDynamicFeeChecker` and `MinGasPriceDecorator`. Because `BaseFee` is updated every block by `feemarket.BeginBlock`, Cosmos SDK transactions are validated against a permanently stale base fee for the entire lifetime of the node, allowing fee-market bypass.

### Finding Description

In `evmd/ante/handler_options.go`, `newCosmosAnteHandler` reads `feemarketParams` from the keeper once at construction time and passes a pointer to this local snapshot into both fee-checking components:

```go
func newCosmosAnteHandler(ctx sdk.Context, options HandlerOptions, ...) sdk.AnteHandler {
    evmParams := options.EvmKeeper.GetParams(ctx)
    feemarketParams := options.FeeMarketKeeper.GetParams(ctx)  // captured once at init
    ...
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
    ...
    cosmos.NewMinGasPriceDecorator(options.FeeMarketKeeper, evmDenom, &feemarketParams),
``` [1](#0-0) 

`NewDynamicFeeChecker` uses this stale pointer to derive `baseFee` on every invocation:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
``` [2](#0-1) 

`MinGasPriceDecorator` reads `MinGasPrice` from the same stale pointer:

```go
minGasPrice := mpd.feemarketParams.MinGasPrice
``` [3](#0-2) 

By contrast, `newEthAnteHandler` correctly reads fresh params per-transaction via `EVMBlockConfig`, which is cached in the object store and refreshed each block:

```go
blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
feemarketParams := &blockCfg.FeeMarketParams
baseFee := blockCfg.BaseFee
``` [4](#0-3) 

`EVMBlockConfig` itself reads `feemarketParams.GetBaseFee()` from the keeper at the start of each block: [5](#0-4) 

The `BaseFee` stored in params is updated every block by `feemarket.BeginBlock`: [6](#0-5) 

Since `newCosmosAnteHandler` is called once at app initialization (standard Cosmos SDK ante handler setup pattern), `feemarketParams` is a local variable whose address is captured in the closure. It is never refreshed. The `BaseFee` field inside it remains the genesis value for the entire node lifetime.

### Impact Explanation

`BaseFee` changes every block whenever block gas usage diverges from the target. Once the base fee rises above the genesis value (e.g., due to sustained high block utilization), any Cosmos SDK transaction whose fee-per-gas falls between the stale genesis base fee and the current live base fee will pass `NewDynamicFeeChecker` and `MinGasPriceDecorator` and be committed â€” even though it is below the required minimum. This is a fee-market bypass for all Cosmos SDK (non-EVM) transactions that use the dynamic fee checker, allowing users to underpay fees. Conversely, when the base fee drops below genesis, valid Cosmos SDK transactions are incorrectly rejected, causing a liveness issue for those senders.

This matches the allowed impact: **High â€” fee market / ante handler bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted.**

### Likelihood Explanation

The base fee diverges from its genesis value from the very first block with non-target gas usage. Any Cosmos SDK transaction submitted after that point is validated against the wrong base fee. No special attacker capability is required â€” a normal unprivileged user submitting a Cosmos SDK transaction (e.g., a governance vote, IBC relay, staking message) with a fee calibrated to the stale genesis base fee triggers the bug. The condition is persistent and worsens over time as the base fee drifts further from genesis.

### Recommendation

Move the `feemarketParams` (and `evmParams`) reads inside the returned handler closure so they are refreshed on every transaction, mirroring the `newEthAnteHandler` approach:

```go
func newCosmosAnteHandler(options HandlerOptions, extra ...sdk.AnteDecorator) sdk.AnteHandler {
    return func(ctx sdk.Context, tx sdk.Tx, simulate bool) (sdk.Context, error) {
        evmParams := options.EvmKeeper.GetParams(ctx)
        feemarketParams := options.FeeMarketKeeper.GetParams(ctx)
        // build decorators using fresh params ...
    }
}
```

Alternatively, use `EVMBlockConfig` (which is already cached per-block in the object store) to obtain `feemarketParams` and `baseFee`, exactly as `newEthAnteHandler` does.

### Proof of Concept

1. Chain starts; genesis `BaseFee` = 1 gwei. `newCosmosAnteHandler` is called once, capturing `feemarketParams.BaseFee = 1 gwei`.
2. Blocks fill up; `feemarket.BeginBlock` updates `BaseFee` to 2 gwei in params store after block N.
3. Attacker submits a Cosmos SDK governance-vote transaction with `fee = 1.5 gwei Ă— gasLimit`.
4. `newEthAnteHandler` (EVM path) would correctly reject an EVM tx at this fee level â€” it reads the live 2 gwei base fee via `EVMBlockConfig`.
5. `newCosmosAnteHandler` (Cosmos path) reads `feemarketParams.BaseFee = 1 gwei` (stale snapshot) â†’ `baseFee = 1 gwei` â†’ `feeCap (1.5 gwei) >= baseFeeInt (1 gwei)` â†’ **passes**.
6. Transaction is committed with fees 25% below the current required minimum, bypassing the fee market for Cosmos SDK messages.

```go
func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
}
