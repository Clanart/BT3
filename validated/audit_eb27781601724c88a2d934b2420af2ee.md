### Title
Native ETH `finTransfer` Permanently Locks Bridged Funds When Recipient Contract Cannot Receive ETH — (`evm/src/omni-bridge/contracts/OmniBridge.sol`)

### Summary

`OmniBridge.finTransfer` delivers native ETH to `payload.recipient` via a bare `.call{value: amount}("")`. If the recipient is a contract that reverts on ETH receipt, every finalization attempt reverts, the nonce is never consumed, and the ETH is permanently locked in the bridge with no recovery path.

### Finding Description

In `finTransfer`, when `payload.tokenAddress == address(0)` (native ETH delivery), the contract executes:

```solidity
(bool success, ) = payload.recipient.call{value: payload.amount}("");
if (!success) revert FailedToSendEther();
``` [1](#0-0) 

The nonce is marked used optimistically before the transfer:

```solidity
completedTransfers[payload.destinationNonce] = true;
``` [2](#0-1) 

Because `revert FailedToSendEther()` rolls back the entire transaction, the `completedTransfers` write is also rolled back. The nonce is never permanently consumed. However, the `payload.recipient` is cryptographically committed inside the MPC-signed message — it is part of the Borsh-encoded blob that is verified against `nearBridgeDerivedAddress`: [3](#0-2) 

No alternative delivery address can be substituted without invalidating the signature. There is no WETH fallback, no escrow, and no admin rescue function in the contract: [4](#0-3) 

The contract does hold a `receive()` function so ETH accumulates in it, but there is no corresponding withdrawal path for stuck ETH tied to a failed delivery. [5](#0-4) 

### Impact Explanation

Any ETH bridged from NEAR (or another chain) to an EVM contract address that does not implement a payable `receive`/`fallback` function is permanently locked in the bridge. The ETH cannot be delivered (every `finTransfer` call reverts), cannot be redirected (the recipient is signature-bound), and cannot be rescued (no admin withdrawal exists). This is an irrecoverable loss of user funds matching the **Critical** impact class: *Permanent freezing, irrecoverable lock, or unclaimable settlement of user or protocol funds in bridge, token, or vault flows.*

### Likelihood Explanation

The scenario is realistic and reachable by any unprivileged bridge user:

- A user on NEAR initiates a transfer of native ETH to an EVM address that is a contract (e.g., a Gnosis Safe with a guard that reverts on ETH receipt, a DeFi vault, a proxy with no fallback, or any contract that intentionally rejects plain ETH sends).
- The user controls the recipient field in `initTransfer` on the source chain; no privileged role is required.
- The relayer (or anyone) calls `finTransfer` on EVM; it reverts every time.
- The user has no recourse.

### Recommendation

Replace the bare `.call` with a try/escrow pattern: if the ETH push fails, wrap the amount into WETH and credit it to the recipient's claimable balance (or to a per-nonce escrow mapping), then allow the recipient to pull it later. This mirrors the mitigation applied in the referenced PartyDAO fix (PR #126). Concretely:

1. On failure of `.call{value: amount}("")`, deposit `amount` into WETH and record `pendingEth[recipient] += amount`.
2. Add a `claimEth(address recipient)` function that transfers the WETH or retries the ETH push.
3. Alternatively, mark the nonce as completed regardless of delivery outcome and emit a `DeliveryFailed` event so off-chain tooling can trigger a refund on the source chain.

### Proof of Concept

1. On NEAR, user calls `ft_transfer_call` to bridge 1 ETH to EVM, specifying `recipient = address(VaultContract)` where `VaultContract` has no `receive()` function.
2. MPC signs the `TransferMessagePayload` with `tokenAddress = address(0)`, `amount = 1e18`, `recipient = address(VaultContract)`.
3. Relayer calls `OmniBridge.finTransfer(sig, payload)` on EVM.
4. Execution reaches line 319: `VaultContract.call{value: 1e18}("")` → returns `success = false`.
5. Line 322: `revert FailedToSendEther()` — entire tx reverts, including the `completedTransfers[nonce] = true` write.
6. Any subsequent call with the same payload repeats steps 3–5 indefinitely.
7. The 1 ETH sits in `OmniBridge` forever; `VaultContract` never receives it; no admin function can recover it. [6](#0-5)

    function finTransfer(
        bytes calldata signatureData,
        BridgeTypes.TransferMessagePayload calldata payload
    ) external payable whenNotPaused(PAUSED_FIN_TRANSFER) {
        if (completedTransfers[payload.destinationNonce]) {
            revert NonceAlreadyUsed(payload.destinationNonce);
        }

        completedTransfers[payload.destinationNonce] = true;

        bytes memory borshEncoded = bytes.concat(
            bytes1(uint8(BridgeTypes.PayloadType.TransferMessage)),
            Borsh.encodeUint64(payload.destinationNonce),
            bytes1(payload.originChain),
            Borsh.encodeUint64(payload.originNonce),
            bytes1(omniBridgeChainId),
            Borsh.encodeAddress(payload.tokenAddress),
            Borsh.encodeUint128(payload.amount),
            bytes1(omniBridgeChainId),
            Borsh.encodeAddress(payload.recipient),
            bytes(payload.feeRecipient).length == 0 // None or Some(String) in rust
                ? bytes("\x00")
                : bytes.concat(
                    bytes("\x01"),
                    Borsh.encodeString(payload.feeRecipient)
                ),
            bytes(payload.message).length == 0
                ? bytes("")
                : Borsh.encodeBytes(payload.message)
        );
        bytes32 hashed = keccak256(borshEncoded);

        if (ECDSA.recover(hashed, signatureData) != nearBridgeDerivedAddress) {
            revert InvalidSignature();
        }

        MultiTokenInfo memory multiToken = multiTokens[payload.tokenAddress];

        if (payload.tokenAddress == address(0)) {
            // slither-disable-next-line arbitrary-send-eth
            (bool success, ) = payload.recipient.call{value: payload.amount}(
                ""
            );
            if (!success) revert FailedToSendEther();
        } else if (multiToken.tokenAddress != address(0)) {
            IERC1155(multiToken.tokenAddress).safeTransferFrom(
                address(this),
                payload.recipient,
                multiToken.tokenId,
                payload.amount,
                ""
            );
        } else if (customMinters[payload.tokenAddress] != address(0)) {
            ICustomMinter(customMinters[payload.tokenAddress]).mint(
                payload.tokenAddress,
                payload.recipient,
                payload.amount
            );
        } else if (isBridgeToken[payload.tokenAddress]) {
            if (payload.message.length == 0) {
                IBridgeToken(payload.tokenAddress).mint(
                    payload.recipient,
                    payload.amount
                );
            } else {
                IBridgeToken(payload.tokenAddress).mint(
                    payload.recipient,
                    payload.amount,
                    payload.message
                );
            }
        } else {
            IERC20(payload.tokenAddress).safeTransfer(
                payload.recipient,
                payload.amount
            );
        }

        finTransferExtension(payload);

        emit BridgeTypes.FinTransfer(
            payload.originChain,
            payload.originNonce,
            payload.tokenAddress,
            payload.amount,
            payload.recipient,
            payload.feeRecipient
        );
    }
