Audit Report

## Title
ETH silently trapped on non-WETH swap/liquidity paths and drainable by any caller via `refundETH()` — (`metric-periphery/contracts/base/PeripheryPayments.sol`, `metric-periphery/contracts/MetricOmmSimpleRouter.sol`, `metric-periphery/contracts/MetricOmmPoolLiquidityAdder.sol`)

## Summary
Every user-facing entry point in `MetricOmmSimpleRouter` and `MetricOmmPoolLiquidityAdder` is declared `payable`, yet `PeripheryPayments.pay()` only consumes native ETH when `token == WETH`. For any other token the `else` branch executes a plain ERC-20 pull and leaves `address(this).balance` untouched, silently trapping any ETH the caller attached. Because `refundETH()` transfers the entire contract balance to `msg.sender` with no access control, any third party can immediately drain the trapped ETH.

## Finding Description
`PeripheryPayments.pay()` branches on the token address:

```solidity
// PeripheryPayments.sol L69-88
} else {
    // ETH is completely ignored here
    IERC20(token).safeTransferFrom(payer, recipient, value);
}
```

When `token != WETH`, the final `else` branch executes a plain ERC-20 pull and never touches `address(this).balance`. Any ETH the caller attached to the transaction accumulates in the contract. [1](#0-0) 

The `receive()` guard only blocks direct ETH transfers (no calldata); it does not prevent ETH from being accepted by `payable` functions: [2](#0-1) 

`refundETH()` then sends the full balance to whoever calls it, with no restriction on caller identity: [3](#0-2) 

All four router entry points are `payable` with no `msg.value == 0` guard: [4](#0-3) [5](#0-4) [6](#0-5) [7](#0-6) 

Both liquidity adder entry points are similarly `payable` with no guard: [8](#0-7) [9](#0-8) 

## Impact Explanation
Any ETH attached to a non-WETH swap or liquidity call is permanently lost from the caller's perspective and immediately claimable by any unprivileged third party via `refundETH()`. This is a direct loss of user principal with no recovery path once the ETH is swept. The exact corrupted value is `address(this).balance` — the full ETH balance of the router or liquidity adder contract.

## Likelihood Explanation
The affected functions are the primary user-facing entry points. Wallets, aggregators, and SDK wrappers that batch calls via `multicall` may forward ETH intended for a WETH hop to a subsequent non-WETH call in the same batch. User error (sending ETH to a non-WETH swap) is a well-documented class of mistake. No special privilege or malicious setup is required; any unprivileged caller can trigger the loss and any unprivileged caller can steal the trapped ETH.

## Recommendation
Add a zero-value guard on every code path where native ETH is not consumed:

```solidity
// In exactInputSingle, exactInput, exactOutputSingle, exactOutput
// when tokenIn != WETH (or when the first token in the path is not WETH):
if (params.tokenIn != WETH) require(msg.value == 0, "ETH not accepted for ERC20 swap");

// In addLiquidityExactShares / addLiquidityWeighted
// when neither pool token is WETH:
require(msg.value == 0, "ETH not accepted for ERC20 liquidity");
```

Alternatively, remove `payable` from functions that have no WETH-wrapping code path, or add an automatic `refundETH()` call at the end of each entry point.

## Proof of Concept
1. Alice calls `exactInputSingle` with `tokenIn = USDC`, `amountIn = 1000e6`, and attaches `1 ether` as `msg.value`.
2. Inside the swap callback, `pay(USDC, Alice, pool, 1000e6)` is invoked. Because `USDC != WETH`, the `else` branch fires: `IERC20(USDC).safeTransferFrom(Alice, pool, 1000e6)`. The `1 ether` is never touched.
3. The swap completes successfully. Alice's `1 ether` now sits in `MetricOmmSimpleRouter`.
4. Bob, an unrelated address, calls `refundETH()`. `address(this).balance == 1 ether`, so `_transferETH(Bob, 1 ether)` executes. Bob receives Alice's ETH.
5. Alice has no recourse; the ETH is gone.

The same sequence applies to `exactInput`, `exactOutputSingle`, `exactOutput`, `addLiquidityExactShares`, and `addLiquidityWeighted` whenever the input token(s) are not WETH.
