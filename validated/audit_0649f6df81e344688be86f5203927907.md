### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the replacement class hash corresponds to a previously declared contract class. Any contract can therefore replace its own class hash with an arbitrary, undeclared value. The OS accepts and commits this state transition without complaint. Every subsequent call to that contract fails because the class does not exist, permanently freezing all funds held by the contract.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` (lines 878–916) writes the caller-supplied class hash directly into `contract_state_changes` with no check against `contract_class_changes` (the dict that tracks declared classes). The code itself acknowledges the gap with an explicit TODO:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The same missing check exists in the deprecated path:

```cairo
func execute_replace_class{contract_state_changes: DictAccess*, revert_log: RevertLogEntry*}(
    contract_address, syscall_ptr: ReplaceClass*
) {
    ...
    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );
    dict_update{dict_ptr=contract_state_changes}(...);
``` [2](#0-1) 

The syscall dispatcher routes `REPLACE_CLASS_SELECTOR` directly to this function, passing only the currently-executing contract address — no external guard exists:

```cairo
if (selector == REPLACE_CLASS_SELECTOR) {
    execute_replace_class(contract_address=execution_context.execution_info.contract_address);
``` [3](#0-2) 

The protocol invariant that every contract's `class_hash` field must point to a declared class is therefore unenforced at the OS level. A state transition that violates this invariant is proven and committed without error.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once the OS commits a state in which a contract's `class_hash` points to an undeclared value, no entry point of that contract can ever be executed again: the OS cannot locate the class bytecode. Any ERC-20 tokens, NFTs, or native STRK held by the contract become permanently inaccessible. There is no recovery path because the only way to change the class hash again is to call `replace_class` — which itself requires executing an entry point of the now-broken contract.

---

### Likelihood Explanation

The attack is straightforward for a malicious contract deployer:

1. Deploy a contract whose `__execute__` (or any callable entry point) contains a conditional `replace_class` call with an attacker-chosen, undeclared hash.
2. Attract user deposits (e.g., present the contract as a vault or multisig).
3. Submit an invoke transaction that triggers the `replace_class` branch.
4. The OS processes the syscall, writes the undeclared hash into `contract_state_changes`, and commits the block.
5. All future calls to the contract revert; user funds are frozen.

No privileged access is required. The attacker is a standard contract deployer and transaction sender — both explicitly listed as valid entry-point actors in the bounty scope. The attack requires only one on-chain transaction after users have deposited.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that the hash exists either in `contract_class_changes` (declared in the current block) or in the pre-block state (previously declared). Concretely, add a `dict_read` on `contract_class_changes` and assert the returned compiled-class hash is non-zero. Apply the same fix to `execute_replace_class` in `deprecated_execute_syscalls.cairo`.

---

### Proof of Concept

```
1. Attacker deploys MaliciousVault with class hash X (legitimately declared).
   MaliciousVault.__execute__ contains:
       if (storage_read(TRIGGER_SLOT) != 0):
           replace_class(0xdeadbeef)   # 0xdeadbeef is never declared

2. Users deposit 1 000 000 STRK into MaliciousVault.

3. Attacker calls:
       storage_write(TRIGGER_SLOT, 1)
   in a prior transaction, then submits an invoke to MaliciousVault.__execute__.

4. OS executes execute_replace_class(contract_address=MaliciousVault, class_hash=0xdeadbeef).
   No validation is performed (TODO comment, lines 898).
   contract_state_changes[MaliciousVault].class_hash = 0xdeadbeef is committed.

5. Any subsequent call to MaliciousVault fails: the OS cannot find class 0xdeadbeef.
   1 000 000 STRK are permanently frozen.
``` [4](#0-3)

func execute_replace_class{

    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
