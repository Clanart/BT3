### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Fund Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary `class_hash` value from the caller without verifying that the hash corresponds to a previously declared contract class. An unprivileged contract deployer can call `replace_class` with a non-existent class hash, permanently rendering the contract uncallable and freezing any funds held within it. The OS itself acknowledges this gap with a `TODO` comment.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the requested `class_hash` directly from the syscall request and writes it into `contract_state_changes` with no check that the hash exists in `contract_class_changes` (i.e., was ever declared via a `DECLARE` transaction):

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

Compare this with `execute_declare_transaction`, which is the only legitimate path to register a class hash into `contract_class_changes`: [2](#0-1) 

The OS enforces `prev_value=0` on declaration (a class may only be declared once), but `execute_replace_class` never cross-references `contract_class_changes` to confirm the target hash was ever declared. The `TODO` comment at line 898 is an explicit acknowledgment of this missing invariant.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

If a contract's class hash is set to an undeclared value, the OS will commit this state to the global Merkle root. On any subsequent call to that contract, the execution layer will find no bytecode for the class hash and the call will permanently fail. Any ERC-20 balances, ETH, or other assets held in the contract's storage become irrecoverable. Because the invalid state is committed and proven, there is no rollback path.

---

### Likelihood Explanation

The `replace_class` syscall is a standard, permissionless syscall callable by any contract on itself. No privileged role is required. A malicious contract deployer can:

1. Deploy a contract and attract user deposits (e.g., a vault or multi-sig).
2. Call `replace_class` with a fabricated, never-declared class hash.
3. The OS accepts the syscall, updates `contract_state_changes`, and commits the invalid class hash to the proven state root.
4. All subsequent calls to the contract revert permanently; deposited funds are frozen.

This is directly reachable by an unprivileged transaction sender with no dependency on sequencer collusion or any external party.

---

### Recommendation

Before updating `contract_state_changes` in `execute_replace_class`, verify that `class_hash` is present in `contract_class_changes` (i.e., it was previously declared). Concretely, perform a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert the returned compiled class hash is non-zero:

```cairo
// Verify the class hash was declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the invariant already enforced in `execute_declare_transaction` (`prev_value=0` / `assert_not_zero(compiled_class_hash)`). [3](#0-2) 

---

### Proof of Concept

1. Attacker deploys `VaultContract` (a contract that accepts user token deposits).
2. Users call `deposit()` on `VaultContract`; balances are stored in its storage.
3. Attacker sends an `INVOKE` transaction calling `replace_class` syscall on `VaultContract` with `class_hash = 0xdeadbeef` (never declared).
4. `execute_replace_class` in `syscall_impls.cairo` (line 878–916) reads `class_hash = 0xdeadbeef` from the request, skips any declared-class check (the `TODO` at line 898), and writes `new StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes`.
5. `state_update` in `state/state.cairo` squashes and commits this entry into the Patricia Merkle Tree, producing a valid proof with `VaultContract.class_hash = 0xdeadbeef`.
6. All future calls to `VaultContract` fail: no bytecode exists for `0xdeadbeef`. User funds are permanently frozen. [4](#0-3) [5](#0-4)

func execute_replace_class{

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    );

        key=contract_address,

    );

    return ();
}
