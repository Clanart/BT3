### Title
Missing duplicate-hash guard in `MerkleBlob::upsert` allows two leaves to share the same hash, corrupting the DataLayer tree state — (`crates/chia-datalayer/src/merkle/blob.rs`)

### Summary
`MerkleBlob::upsert` updates an existing leaf's hash without checking whether the new hash is already held by a different leaf. `insert` enforces this invariant; `upsert` does not. The result is two blob leaves sharing the same hash while the `leaf_hash_to_index` cache maps that hash to only one of them, breaking `get_node_by_hash` and causing `check_integrity` to return an error.

### Finding Description
`insert` guards against duplicate leaf hashes: [1](#0-0) 

`upsert`, when the key is already present, skips this check entirely: [2](#0-1) 

`remove_leaf` removes the old hash from the cache: [3](#0-2) 

`insert_entry_to_blob` then calls `add_leaf`, which uses `HashMap::insert` — silently overwriting any existing entry for the new hash: [4](#0-3) [5](#0-4) 

### Impact Explanation
After the attack:
- Two leaves in the blob share hash H; `leaf_hash_to_index` maps H to only one of them.
- `get_node_by_hash(H)` returns the wrong key/value.
- `check_integrity` detects the cache length mismatch and returns `IntegrityLeafHashToIndexCacheLength`: [6](#0-5) 
- Any proof-of-inclusion built on the corrupted cache is unreliable; the tree root derived from the blob is inconsistent with the cache state.

This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic corrupts tree roots or lets untrusted input prove invalid state.**

### Likelihood Explanation
`upsert` is a public API exposed through both the Rust library and the Python/wasm bindings: [7](#0-6) 

Any caller that controls the `new_hash` argument — including a DataLayer client that can issue upsert operations — can trigger this with two API calls. No privileged access is required.

### Recommendation
Add the same duplicate-hash guard to `upsert` that `insert` already has, before modifying the leaf:

```rust
pub fn upsert(&mut self, key: KeyId, value: ValueId, new_hash: &Hash) -> Result<(), Error> {
    let Ok((leaf_index, mut leaf, mut block)) = self.get_leaf_by_key(key) else {
        self.insert(key, value, new_hash, InsertLocation::Auto {})?;
        return Ok(());
    };

    // Guard: only check if the hash is actually changing
    if leaf.hash != *new_hash && self.block_status_cache.contains_leaf_hash(new_hash) {
        return Err(Error::HashAlreadyPresent());
    }

    self.block_status_cache.remove_leaf(&leaf)?;
    leaf.hash.clone_from(new_hash);
    leaf.value = value;
    block.node = Node::Leaf(leaf);
    self.insert_entry_to_blob(leaf_index, &block)?;
    ...
}
```

#[test]
fn test_upsert_duplicate_hash_corruption() {
    let mut blob = MerkleBlob::new(vec![]).unwrap();
    let hash_a = sha256_num(&1u64);
    let hash_b = sha256_num(&2u64);

    // Insert key A with hash_a, key B with hash_b
    blob.insert(KeyId(1), ValueId(10), &hash_a, InsertLocation::Auto {}).unwrap();
    blob.insert(KeyId(2), ValueId(20), &hash_b, InsertLocation::Auto {}).unwrap();

    // Upsert key B with hash_a (already owned by key A) — no error returned
    blob.upsert(KeyId(2), ValueId(99), &hash_a).unwrap();

    // check_integrity detects the corruption
    blob.check_integrity().expect_err("should detect duplicate hash");

    // get_node_by_hash returns key B's data for hash_a, key A's leaf is orphaned
    let (key, _) = blob.get_node_by_hash(hash_a).unwrap();
    assert_ne!(key, KeyId(1)); // key A is unreachable via its own hash
}
