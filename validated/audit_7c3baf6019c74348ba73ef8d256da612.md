### Title
DataLayer `ProofOfInclusion::valid()` Tautological Root Check Accepts Forged Inclusion Proofs — (`File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` contains a tautological final check that makes it verify only the internal consistency of the proof chain, never the proof's computed root against any external trusted root. An attacker can craft a `ProofOfInclusion` claiming any key is included in the DataLayer tree, and `valid()` will return `true`.

### Finding Description

The `valid()` method walks the proof layers, verifying that each `calculated_hash == layer.combined_hash`, and updates `existing_hash = calculated_hash` on each iteration. After the loop it performs:

```rust
existing_hash == self.root_hash()
```

`root_hash()` is defined as:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← same value as existing_hash after the loop
    } else {
        self.node_hash
    }
}
```

After the loop, `existing_hash` holds the last `calculated_hash`, which equals the last `layer.combined_hash` (because the loop only continues when `calculated_hash == layer.combined_hash`). `root_hash()` returns that same `layer.combined_hash`. The final comparison is therefore always `true` whenever the loop completes without returning `false`.

The function never compares the computed root against any externally-supplied trusted root. An attacker can construct a `ProofOfInclusion` with:

- An arbitrary `node_hash` (claiming any key-value pair is included)
- Layers whose `combined_hash` values are internally consistent (each equals `calculate_internal_hash(prev, side, other)`)
- A final `combined_hash` that is whatever the attacker chooses — it becomes the self-reported "root"

`valid()` returns `true` for this forged proof.

`ProofOfInclusion` implements `Streamable`, so it can be deserialized from untrusted bytes. Any DataLayer client that receives a serialized proof from a peer and calls only `proof.valid()` — without separately comparing `proof.root_hash()` against the on-chain committed root — will accept the forged proof. ### Impact Explanation

Matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."*

A DataLayer client that trusts `valid()` alone will accept a proof claiming any arbitrary key-value pair is present in the tree, regardless of the actual committed root. This lets an untrusted peer prove invalid DataLayer state to any receiver that does not separately verify `proof.root_hash()` against the on-chain root.

### Likelihood Explanation

`ProofOfInclusion` is `Streamable` and exposed through Python bindings. DataLayer sync protocols exchange proofs between peers. The function name `valid()` strongly implies complete proof validation; callers are likely to rely on it without a separate root check. The fuzz target and all existing tests generate proofs from the actual tree (so they never exercise the forged-proof path), leaving the bug undetected. [3](#0-2) 

### Recommendation

`valid()` must accept an external trusted root and compare against it:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against caller-supplied root
}
```

Alternatively, rename the current function to `is_internally_consistent()` and add clear documentation that callers must separately verify `root_hash()` against the on-chain committed root.

### Proof of Concept

```rust
use chia_datalayer::{ProofOfInclusion, ProofOfInclusionLayer, Side, Hash};

// Compute a single internally-consistent layer for an arbitrary node_hash
let node_hash: Hash = [0xAA; 32];
let other_hash: Hash = [0xBB; 32];
// combined_hash = calculate_internal_hash(node_hash, Left, other_hash)
// (attacker computes this offline using the public hash function)
let combined_hash: Hash = calculate_internal_hash(&node_hash, Side::Left, &other_hash);

let forged = ProofOfInclusion {
    node_hash,
    layers: vec![ProofOfInclusionLayer {
        other_hash_side: Side::Left,
        other_hash,
        combined_hash,
    }],
};

// valid() returns true even though this proof was never generated
// from the actual DataLayer tree and combined_hash is not the real root
assert!(forged.valid());
// forged.root_hash() == combined_hash  (attacker-chosen, not the real root)
