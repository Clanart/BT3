### Title
Missing Per-Element Gas Cost for `execute_send_message_to_l1` Variable-Length Payload — (File: `execution/syscall_impls.cairo`)

### Summary

The `execute_send_message_to_l1` syscall handler charges a flat `SEND_MESSAGE_TO_L1_GAS_COST = 14470` gas regardless of the payload size. The OS then `memcpy`s the entire variable-length payload into the OS output segment. Unlike analogous variable-length syscalls (`execute_deploy`, `execute_keccak`, `execute_meta_tx_v0`), there is no per-element gas factor applied to the payload. An unprivileged contract caller can send arbitrarily large L1 messages at a fixed gas cost, creating disproportionate proving and L1 verification load not commensurate with the fee paid.

### Finding Description

In `execute_send_message_to_l1`:

```cairo
let success = reduce_syscall_gas_and_write_response_header(
    total_gas_cost=SEND_MESSAGE_TO_L1_GAS_COST,   // flat 14470, no payload factor
    request_struct_size=SendMessageToL1Request.SIZE
);
...
tempvar payload_size = request.payload_end - payload_start;
memcpy(
    dst=outputs.messages_to_l1 + MessageToL1Header.SIZE, src=payload_start, len=payload_size
);
``` [1](#0-0) 

The gas charged is always `SEND_MESSAGE_TO_L1_GAS_COST - SYSCALL_BASE_GAS_COST = 4470` net, regardless of `payload_size`. [2](#0-1) 

Compare with the correct pattern used by other variable-length syscalls:

- `execute_deploy`: `DEPLOY_GAS_COST + DEPLOY_CALLDATA_FACTOR_GAS_COST * constructor_calldata_size`
- `execute_keccak`: `KECCAK_GAS_COST + q * KECCAK_ROUND_COST_GAS_COST`
- `execute_meta_tx_v0`: `META_TX_V0_GAS_COST + META_TX_V0_CALLDATA_FACTOR_GAS_COST * calldata_size` [3](#0-2) [4](#0-3) [5](#0-4) 

There is also no `SIERRA_ARRAY_LEN_BOUND` check on `payload_size` in `execute_send_message_to_l1`, unlike the signature bound check in `execute_meta_tx_v0`. [6](#0-5) 

The payload is written directly into the OS output (`OsCarriedOutputs.messages_to_l1`), which forms part of the STARK proof's public input and is committed on-chain. [7](#0-6) 

### Impact Explanation

**High — Network not being able to confirm new transactions.**

The OS output written by `memcpy` is proportional to `payload_size` and becomes part of the STARK proof's public input. A very large payload:

1. Requires proportionally more Cairo VM steps to `memcpy` into the output segment, consuming prover capacity without commensurate gas payment.
2. Inflates the OS output segment, increasing proof generation time and L1 calldata cost for the sequencer.
3. Because the gas cost is flat, an attacker can pack a block with many large-payload `send_message_to_l1` calls at minimal L2 gas cost, causing blocks to become extremely expensive to prove, potentially stalling block confirmation.

### Likelihood Explanation

Any deployed contract can call `send_message_to_l1` with an arbitrarily large payload array. No privileged role is required. The attacker only needs to:
1. Deploy a contract that constructs a large felt array and calls `send_message_to_l1`.
2. Submit invoke transactions calling that contract.

The L2 gas budget (`EXECUTE_MAX_SIERRA_GAS = 1,100,000,000`) limits the total gas per transaction, but since the payload gas cost is flat (4,470 net), a single transaction can issue many large-payload messages. The actual OS proving work scales with total payload bytes across all messages in the block, while the fee does not.

### Recommendation

Apply a per-element gas factor to the payload, consistent with the pattern used by `execute_deploy` and `execute_meta_tx_v0`:

```cairo
let specific_base_gas_cost = SEND_MESSAGE_TO_L1_GAS_COST +
    SEND_MESSAGE_TO_L1_PAYLOAD_FACTOR_GAS_COST * payload_size;
let success = reduce_syscall_gas_and_write_response_header(
    total_gas_cost=specific_base_gas_cost,
    request_struct_size=SendMessageToL1Request.SIZE
);
```

Additionally, add a `SIERRA_ARRAY_LEN_BOUND` check on `payload_size` before the gas reduction, consistent with other array-length validations in the codebase.

### Proof of Concept

1. Deploy a contract with the following logic:
```cairo
// Pseudocode
fn exploit() {
    let mut payload: Array<felt252> = ArrayTrait::new();
    let mut i = 0;
    loop {
        if i == 10000 { break; }
        payload.append(0xdeadbeef);
        i += 1;
    };
    // Gas cost: flat SEND_MESSAGE_TO_L1_GAS_COST = 14470 regardless of 10000-felt payload
    starknet::send_message_to_l1_syscall(
        to_address: 0x1.try_into().unwrap(),
        payload: payload.span()
    ).unwrap();
}
```

2. Submit an invoke transaction calling `exploit()`. The OS charges 14,470 gas for the syscall but must `memcpy` 10,000 felts into the output segment.

3. Repeat across multiple transactions in the same block. The block's OS output grows proportionally to total payload, while the gas charged does not. The prover must handle the inflated output at no additional cost to the attacker. [8](#0-7) [9](#0-8)

// Sends a message to L1.
func execute_send_message_to_l1{range_check_ptr, syscall_ptr: felt*, outputs: OsCarriedOutputs*}(
    contract_address: felt
) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, SendMessageToL1Request*);
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=SEND_MESSAGE_TO_L1_GAS_COST, request_struct_size=SendMessageToL1Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    tempvar payload_start = request.payload_start;
    tempvar payload_size = request.payload_end - payload_start;

    assert [outputs.messages_to_l1] = MessageToL1Header(
        from_address=contract_address, to_address=request.to_address, payload_size=payload_size
    );
    memcpy(
        dst=outputs.messages_to_l1 + MessageToL1Header.SIZE, src=payload_start, len=payload_size
    );
    let (outputs) = os_carried_outputs_new(
        messages_to_l1=outputs.messages_to_l1 + MessageToL1Header.SIZE + payload_size,
        messages_to_l2=outputs.messages_to_l2,
    );

    return ();
}
