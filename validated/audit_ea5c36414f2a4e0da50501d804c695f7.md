### Title
Unverified Class Hash in `execute_replace_class` Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

`execute_replace_class` in the StarkNet OS updates a contract's class hash in `contract_state_changes` without verifying that the supplied class hash has actually been declared. This is the direct analog of the reported `GovPool::delegateTreasury` bug: state accounting is advanced (voting power / class hash) without confirming the underlying precondition (token transfer / class declaration) was satisfied. Any contract can call `replace_class` with an arbitrary, undeclared class hash, permanently corrupting its own state entry and freezing any funds it holds.

---

### Finding Description

`execute_replace_class` reads the requested class hash directly from the syscall request and writes it into `contract_state_changes` with no check against `contract_class_changes` (the dictionary of declared classes):

```cairo
// execute_replace_class (syscall_impls.cairo, lines 878-916)
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The in-code `TODO` comment explicitly acknowledges the missing check. The OS never cross-references `class_hash` against `contract_class_changes` before committing the new state entry. The parallel in the external report is exact:

| External report | StarkNet OS |
|---|---|
| `IERC20.transfer()` called; voting power increased with no balance check | `replace_class` called; class hash updated with no declaration check |
| `delegateTokensTreasury(delegatee, amount)` runs unconditionally | `dict_update(contract_state_changes, …, new_value=new_state_entry)` runs unconditionally |
| Invariant broken: accounting balance ≠ actual token balance | Invariant broken: committed class hash ∉ declared class set | [1](#0-0)

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value and the block is proven:

1. The state root permanently encodes the invalid class hash.
2. Every future attempt to execute the contract causes the OS hint machinery to fail when looking up the class in the compiled-class-facts bundle, making the contract permanently unexecutable.
3. Any ERC-20 balances, NFTs, or other assets held by the contract become irretrievably frozen — no `transfer`, `withdraw`, or `upgrade` entry point can ever be reached again.

If the affected contract is a widely-used token or DeFi vault, the frozen value can be arbitrarily large.

---

### Likelihood Explanation

The `replace_class` syscall is available to every Sierra contract with no privilege restriction. A single malicious or buggy contract calling `replace_class(undeclared_hash)` is sufficient to trigger the issue. The attacker-controlled entry path is:

1. Deploy any contract (unprivileged).
2. From within that contract, emit a `replace_class` syscall with an arbitrary felt as the class hash.
3. The OS processes the syscall, writes the invalid class hash to `contract_state_changes`, and includes it in the proven state update.
4. The contract is permanently bricked; all funds inside are frozen.

No privileged role, leaked key, or external dependency is required.

---

### Recommendation

Before writing the new class hash to `contract_state_changes`, verify that it exists in `contract_class_changes` (i.e., that it was declared in the current or a prior block). The check should mirror the pattern already used for `execute_declare_transaction`, which enforces `prev_value=0` to guarantee uniqueness:

```cairo
// Proposed addition inside execute_replace_class, before dict_update:
let (declared_compiled_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("replace_class: class hash not declared.") {
    assert_not_zero(declared_compiled_hash);
}
```

This follows the "trust but verify" principle recommended in the external report's mitigation.

---

### Proof of Concept

1. Declare class `A` (valid, compiled hash `H_A`).
2. Deploy contract `C` using class `A`; fund it with 1000 STRK.
3. From contract `C`, call syscall `replace_class(class_hash=0xdeadbeef)` — an undeclared hash.
4. The OS executes `execute_replace_class`:
   - `request.class_hash = 0xdeadbeef`
   - No check against `contract_class_changes`
   - `dict_update(contract_state_changes, key=C, new_value=StateEntry(class_hash=0xdeadbeef, …))`
5. Block is proven; state root now encodes `C → class_hash=0xdeadbeef`.
6. Any subsequent invoke of `C` causes the OS to look up `0xdeadbeef` in the compiled-class-facts bundle, find nothing, and panic — the 1000 STRK are permanently frozen. [2](#0-1)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
