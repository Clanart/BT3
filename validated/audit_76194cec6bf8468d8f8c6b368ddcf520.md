Audit Report

## Title
Cross-User ETH Isolation Failure in `PeripheryPayments.pay` — Stranded ETH Subsidises Another User's WETH Input — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary
`PeripheryPayments.pay` uses `address(this).balance` — the router's entire native ETH balance — when settling a WETH-input swap. Because `multicall` is `payable` and `refundETH()` is a separate, optional call, ETH sent by User A in a prior transaction can remain in the router and be silently consumed to cover User B's WETH obligation in a later, independent transaction. User A suffers a permanent, uncompensated ETH loss.

## Finding Description
The root cause is in `pay()` at [1](#0-0)  — when `token == WETH`, the function reads `address(this).balance` which aggregates all ETH held by the router regardless of origin. There is no per-user or per-transaction accounting.

ETH becomes stranded because `multicall` is `payable` [2](#0-1)  and `refundETH()` is a separate, optional call that users must explicitly include in their multicall bundle. [3](#0-2) 

The `receive()` guard [4](#0-3)  only blocks direct ETH sends; it does not prevent ETH accumulation via `msg.value` in payable entry points (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`, `multicall`). If a user sends ETH for a non-WETH swap or sends excess ETH and omits `refundETH()`, the surplus is stranded.

Exploit flow:
1. User A calls `multicall{value: 1 ether}([exactInputSingle({tokenIn: tokenA, tokenOut: tokenB, ...})])` — no `refundETH()`. The swap settles in ERC-20; 1 ETH remains in the router.
2. User B calls `exactInputSingle({tokenIn: WETH, amountIn: 2 ether, ...})`. The pool callback fires `pay(WETH, userB, pool, 2 ether)`.
3. Inside `pay`: `nativeBalance = address(this).balance = 1 ether`. The branch at line 78 wraps and transfers User A's 1 ETH as WETH to the pool, then pulls only 1 WETH from User B via `transferFrom`. [5](#0-4) 
4. User A's 1 ETH is permanently gone; User B's WETH cost is halved.

## Impact Explanation
Direct, permanent loss of User A's ETH principal with no corresponding output. User A's ETH is wrapped into WETH and transferred to a pool to settle a different user's swap obligation. This violates the cross-user ETH isolation invariant and constitutes a Critical/High direct loss of user principal, meeting Sherlock contest thresholds.

## Likelihood Explanation
No privileged access is required. Users routinely call `multicall{value: X}` for ETH-paying swaps and may omit `refundETH()` (e.g., swap consumed less than `X`, or the token was not WETH). Any subsequent WETH-input swap by any user silently drains the stranded balance. A griever can deliberately exploit this by observing a stranded-ETH transaction in the mempool and immediately following with a WETH swap sized to consume exactly the stranded amount. The scenario is realistic, repeatable, and requires no special permissions.

## Recommendation
Replace `address(this).balance` with only the ETH attributable to the current call. One approach: record `msg.value` in transient storage at each payable entry point and use that stored value as the cap in `pay()`, rather than the entire contract balance. Alternatively, auto-refund any remaining ETH at the end of each top-level swap entry point, or require `refundETH()` as the mandatory last call in any `multicall` that sends ETH.

## Proof of Concept
1. Deploy `MetricOmmSimpleRouter` with real WETH, a pool for `tokenA/tokenB` (non-WETH), and a pool for `WETH/tokenX`.
2. User A calls `multicall{value: 1 ether}([exactInputSingle({tokenIn: tokenA, tokenOut: tokenB, amountIn: ..., ...})])` — no `refundETH()`. Assert `address(router).balance == 1 ether`.
3. User B approves 1 WETH to the router, then calls `exactInputSingle({tokenIn: WETH, tokenOut: tokenX, amountIn: 2 ether, ...})`.
4. Pool callback fires → `pay(WETH, userB, pool, 2 ether)`: `nativeBalance = 1 ether` → wraps 1 ETH as WETH to pool, pulls only 1 WETH from User B.
5. Assert: `address(router).balance == 0` (User A's ETH gone), User B's WETH decreased by 1 (not 2), User A's ETH balance permanently reduced by 1 ETH with no output received.
