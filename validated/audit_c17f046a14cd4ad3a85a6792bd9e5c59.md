### Title
P2P State Diff Sync Validates Length But Not Commitment Hash, Allowing Wrong State Diff Injection — (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

---

### Summary

The P2P sync client terminates state diff chunk collection when the accumulated chunk length equals `header.state_diff_length`, but never verifies the assembled `ThinStateDiff` against `header.state_diff_commitment`. These are two independent fields in the stored block header, both derived from the same state diff at block production time. A malicious peer can send chunks whose lengths sum to the expected value while carrying entirely different content, causing the node to store and serve a wrong state diff as authoritative.

---

### Finding Description

**Two independent sources of truth in the block header**

At block production, `calculate_block_commitments` computes `state_diff_commitment = Poseidon(ThinStateDiff)` and `state_diff_length = ThinStateDiff::len()` from the same object. Both are stored in the block header. [1](#0-0) [2](#0-1) 

**Validation uses only `state_diff_length`**

`parse_data_for_block` reads `target_state_diff_len` from the stored header and loops until `current_state_diff_len == target_state_diff_len`. Once the length matches, the assembled `ThinStateDiff` is written to storage with no hash check. [3](#0-2) 

The only post-assembly check is `validate_deprecated_declared_classes_non_conflicting`, which only detects duplicate deprecated class hashes — it does not verify the Poseidon commitment. [4](#0-3) 

**`state_diff_commitment` is stored but never used for verification**

The header stores `state_diff_commitment` alongside `state_diff_length`: [5](#0-4) 

The P2P server also falls back to computing `state_diff_length` from the stored state diff when the header field is absent, further decoupling the two values: [6](#0-5) 

**Wrong state diff is written to storage and served as authoritative**

Once accepted, the wrong `ThinStateDiff` is committed directly to the MDBX storage tables (nonces, deployed contracts, storage diffs, compiled class hashes) and the state marker is advanced: [7](#0-6) [8](#0-7) 

---

### Impact Explanation

**Exact corrupted value**: The `ThinStateDiff` written to storage for block `N` differs from the one whose Poseidon hash is recorded in `header.state_diff_commitment`. Every downstream consumer of that block's state — `starknet_getStateUpdate` RPC, state sync to other nodes, fee estimation, and any read of storage/nonce/class-hash values — operates on the injected wrong state. This matches the **High** impact: *"RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."*

---

### Likelihood Explanation

The attacker must be a P2P peer of the victim node. No privileged access is required. The attack is triggered by sending well-formed `StateDiffChunk` messages whose individual `len()` values sum to `target_state_diff_len` while encoding different contract addresses, storage keys, or nonces. The P2P network is open to any participant, making this reachable without special trust.

---

### Recommendation

After the loop in `parse_data_for_block`, compute the Poseidon hash of the assembled `ThinStateDiff` and compare it against `header.state_diff_commitment`:

```rust
// After: if current_state_diff_len != target_state_diff_len { ... }
let calculated_commitment = calculate_state_diff_hash(&result);
let stored_commitment = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .expect("header must exist")
    .state_diff_commitment
    .ok_or(P2pSyncClientError::OldHeaderInStorage {
        block_number,
        missing_field: "state_diff_commitment",
    })?;
if calculated_commitment != stored_commitment {
    return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffHash {
        expected: stored_commitment,
        got: calculated_commitment,
    }));
}
```

This mirrors the check already present in `Committer::commit_or_load` when `verify_state_diff_hash` is enabled: [9](#0-8) 

---

### Proof of Concept

1. Node A (victim) syncs headers from a peer; block 100's header is stored with `state_diff_length = 3` and `state_diff_commitment = H(real_diff)`.
2. Malicious peer sends three `StateDiffChunk` messages each of `len() == 1`, totalling 3, but encoding fabricated storage updates (e.g., setting attacker's balance slot to `MAX_FELT`).
3. `parse_data_for_block` exits the loop because `current_state_diff_len (3) == target_state_diff_len (3)`.
4. `validate_deprecated_declared_classes_non_conflicting` passes (no deprecated classes involved).
5. `write_to_storage` calls `append_state_diff(block_number_100, fabricated_diff)`, advancing the state marker.
6. `starknet_getStateUpdate(block_id=100)` now returns the fabricated storage update. Any node that syncs state from Node A inherits the corrupted state. [10](#0-9)

impl<T: StorageTransaction<Mode = RW>> StateStorageWriter for T {
    #[sequencer_latency_histogram(STORAGE_APPEND_THIN_STATE_DIFF_LATENCY, false)]
    fn append_state_diff(
        self,
        block_number: BlockNumber,
        thin_state_diff: ThinStateDiff,
    ) -> StorageResult<Self> {
        let tables = self.tables();
        let inner_txn = self.txn();
        let file_offset_table = inner_txn.open_table(&tables.file_offsets)?;
        let markers_table = self.open_table(&tables.markers)?;
        let state_diffs_table = self.open_table(&tables.state_diffs)?;
        let nonces_table = self.open_table(&tables.nonces)?;
        let deployed_contracts_table = self.open_table(&tables.deployed_contracts)?;
        let storage_table = self.open_table(&tables.contract_storage)?;
        let declared_classes_block_table = self.open_table(&tables.declared_classes_block)?;
        let deprecated_declared_classes_block_table =
            self.open_table(&tables.deprecated_declared_classes_block)?;
        let compiled_class_hash_table = self.open_table(&tables.compiled_class_hash)?;

        // Write state.
        write_deployed_contracts(
            &thin_state_diff.deployed_contracts,
            inner_txn,
            block_number,
            &deployed_contracts_table,
            &nonces_table,
        )?;
        write_storage_diffs(
            &thin_state_diff.storage_diffs,
            inner_txn,
            block_number,
            &storage_table,
        )?;
        // Must be called after write_deployed_contracts since the nonces are updated there.
        write_nonces(&thin_state_diff.nonces, inner_txn, block_number, &nonces_table)?;

        for (class_hash, _) in &thin_state_diff.class_hash_to_compiled_class_hash {
            let not_declared = declared_classes_block_table.get(inner_txn, class_hash)?.is_none();
            if not_declared {
                declared_classes_block_table.insert(inner_txn, class_hash, &block_number)?;
            }
        }

        write_compiled_class_hashes(
            &thin_state_diff.class_hash_to_compiled_class_hash,
            inner_txn,
            block_number,
            &compiled_class_hash_table,
        )?;

        for class_hash in thin_state_diff.deprecated_declared_classes.iter() {
            // Cairo0 classes can be declared in different blocks. The first block to declare the
            // class is recorded here.
            if deprecated_declared_classes_block_table.get(inner_txn, class_hash)?.is_none() {
                deprecated_declared_classes_block_table.insert(
                    inner_txn,
                    class_hash,
                    &block_number,
                )?;
            }
        }

        // Write state diff.
        let location = self.file_handlers().append_state_diff(&thin_state_diff);
        state_diffs_table.append(inner_txn, &block_number, &location)?;
        file_offset_table.upsert(inner_txn, &OffsetKind::ThinStateDiff, &location.next_offset())?;

        update_marker_to_next_block(inner_txn, &markers_table, MarkerKind::State, block_number)?;

        advance_compiled_class_marker_over_blocks_without_classes(
            inner_txn,
            &markers_table,
            &state_diffs_table,
            self.file_handlers(),
        )?;

        Ok(self)
    }
