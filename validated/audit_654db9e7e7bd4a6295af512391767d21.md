### Title
Missing `state_diff_commitment` Hash Verification in P2P State Diff Sync Allows Malicious Peer to Inject Arbitrary State — (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

---

### Summary

The P2P state-diff sync client validates that the total length of received `StateDiffChunk` messages equals the `state_diff_length` stored in the block header, but never verifies that the assembled `ThinStateDiff` hashes to the `state_diff_commitment` also stored in that same header. A malicious peer can send chunks whose lengths sum correctly but whose content is entirely fabricated, causing the syncing node to persist a wrong state diff and serve authoritative-looking wrong values from every state-related RPC endpoint.

---

### Finding Description

`parse_data_for_block` in `crates/apollo_p2p_sync/src/client/state_diff.rs` reads two fields from the already-stored block header:

```
state_diff_length   → used as the loop termination condition
state_diff_commitment → never read
```

The loop accumulates chunks until `current_state_diff_len == target_state_diff_len`, then calls `validate_deprecated_declared_classes_non_conflicting` (a duplicate-class-hash check) and returns the assembled diff: [1](#0-0) 

The `state_diff_commitment` field is present in the stored header: [2](#0-1) 

and is transmitted over the wire alongside `state_diff_length`: [3](#0-2) 

The canonical hash function `calculate_state_diff_hash` exists and is used in the committer path: [4](#0-3) 

The committer itself optionally verifies the hash when `verify_state_diff_hash` is enabled: [5](#0-4) 

But the P2P sync path writes directly to `apollo_storage` via `append_state_diff` without any such check: [6](#0-5) 

`unite_state_diffs` only rejects duplicate keys (conflicting parts); it does not verify content against any commitment: [7](#0-6) 

The `StateDiffChunk::len()` method counts fields in a way that is consistent with `ThinStateDiff::len()`, so an attacker can craft chunks with arbitrary contract addresses, storage keys, values, class hashes, and nonces while keeping the total count equal to `target_state_diff_len`: [8](#0-7) [9](#0-8) 

---

### Impact Explanation

A syncing node that accepts a length-correct but content-wrong state diff will:

1. Persist the fabricated diff to MDBX via `append_state_diff`, overwriting the correct per-contract storage, nonce, and class-hash tables.
2. Serve wrong values from `starknet_getStorageAt`, `starknet_getClassHashAt`, `starknet_getNonce`, and `starknet_getStateUpdate` — all of which read directly from those MDBX tables.
3. Produce a wrong global root if the committer is later invoked with the stored diff, breaking proof inputs and SNOS output verification.

This matches the allowed impact: **"High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."** and potentially **"Critical. Wrong state … storage value … from blockifier/syscall/execution logic."**

---

### Likelihood Explanation

Any peer in the P2P network can respond to a state-diff query. The syncing node connects to multiple peers and accepts the first valid-length response. No privileged position is required. The attack is silent: the node does not log a mismatch because no mismatch check exists. The only existing guard (`validate_deprecated_declared_classes_non_conflicting`) is trivially satisfied by sending each deprecated class hash exactly once.

---

### Recommendation

After the accumulation loop completes and before calling `write_to_storage`, verify the assembled diff against the stored commitment:

```rust
if let Some(expected_commitment) = header.state_diff_commitment {
    let actual_commitment = calculate_state_diff_hash(&result);
    if actual_commitment != expected_commitment {
        return Err(ParseDataError::BadPeer(
            BadPeerError::StateDiffHashMismatch { block_number }
        ));
    }
}
```

`calculate_state_diff_hash` already sorts all sub-maps internally, so ordering of the received chunks does not matter. [4](#0-3) 

---

### Proof of Concept

**Structural analog to the Ditto M-08 bug:**

| Ditto | Sequencer |
|---|---|
| Sorted list of shorts with CRs | Sorted/accumulated `StateDiffChunk` list |
| Dispute checks only `disputeCR < currentProposal.CR` | Sync checks only `sum(chunk.len()) == target_state_diff_len` |
| Missing check: `proposal[N-1].CR <= disputeCR` | Missing check: `calculate_state_diff_hash(result) == state_diff_commitment` |
| Attacker submits individually valid items that collectively violate the ordering invariant | Attacker submits individually valid chunks that collectively violate the hash commitment invariant |
| Result: higher penalty extracted than intended | Result: wrong state stored and served as authoritative |

**Concrete exploit scenario:**

1. Honest block N has state diff: `{addr_A: {key_1: val_1}}`, `state_diff_length = 1`, `state_diff_commitment = H`.
2. Malicious peer responds to the syncing node's state-diff query for block N with: `ContractDiff { contract_address: addr_B, storage_diffs: {key_2: val_2} }` — length = 1, satisfies `current_state_diff_len == target_state_diff_len`.
3. `unite_state_diffs` inserts `addr_B → {key_2: val_2}` with no conflict.
4. `validate_deprecated_declared_classes_non_conflicting` passes (no deprecated classes).
5. The fabricated diff is written to MDBX.
6. `starknet_getStorageAt(addr_A, key_1)` returns `0` (not found); `starknet_getStorageAt(addr_B, key_2)` returns `val_2` — both wrong. [10](#0-9)

```rust
fn unite_state_diffs(
    state_diff: &mut ThinStateDiff,
    state_diff_chunk: StateDiffChunk,
) -> Result<(), BadPeerError> {
    match state_diff_chunk {
        StateDiffChunk::ContractDiff(contract_diff) => {
            if let Some(class_hash) = contract_diff.class_hash {
                if state_diff
                    .deployed_contracts
                    .insert(contract_diff.contract_address, class_hash)
                    .is_some()
                {
                    return Err(BadPeerError::ConflictingStateDiffParts);
                }
            }
            if let Some(nonce) = contract_diff.nonce {
                if state_diff.nonces.insert(contract_diff.contract_address, nonce).is_some() {
                    return Err(BadPeerError::ConflictingStateDiffParts);
                }
            }
            if !contract_diff.storage_diffs.is_empty() {
                match state_diff.storage_diffs.get_mut(&contract_diff.contract_address) {
                    Some(storage_diffs) => {
                        for (k, v) in contract_diff.storage_diffs {
                            if storage_diffs.insert(k, v).is_some() {
                                return Err(BadPeerError::ConflictingStateDiffParts);
                            }
                        }
                    }
                    None => {
                        state_diff
                            .storage_diffs
                            .insert(contract_diff.contract_address, contract_diff.storage_diffs);
                    }
                }
            }
        }
        StateDiffChunk::DeclaredClass(declared_class) => {
            if state_diff
                .class_hash_to_compiled_class_hash
                .insert(declared_class.class_hash, declared_class.compiled_class_hash)
                .is_some()
            {
                return Err(BadPeerError::ConflictingStateDiffParts);
            }
        }
        StateDiffChunk::DeprecatedDeclaredClass(deprecated_declared_class) => {
            state_diff.deprecated_declared_classes.push(deprecated_declared_class.class_hash);
        }
    }
    Ok(())
}
```

```rust
pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
```

```rust
impl StateDiffChunk {
    pub fn len(&self) -> usize {
        match self {
            StateDiffChunk::ContractDiff(contract_diff) => {
                let mut result = contract_diff.storage_diffs.len();
                if contract_diff.class_hash.is_some() {
                    result += 1;
                }
                if contract_diff.nonce.is_some() {
                    result += 1;
                }
                result
            }
            StateDiffChunk::DeclaredClass(_) => 1,
            StateDiffChunk::DeprecatedDeclaredClass(_) => 1,
        }
    }

    pub fn is_empty(&self) -> bool {
        self.len() == 0
    }
}
```
