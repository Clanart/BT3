### Title
Integer Division Truncation in `NewDynamicFeeChecker` Collapses Effective Fee to Zero, Enabling Free Cosmos Transactions - (File: ante/evm/fee_checker.go)

---

### Summary

`NewDynamicFeeChecker` derives a per-gas price (`feeCap`) from the total fee coins by integer division. When the total fee amount is smaller than the gas limit, the division truncates to zero. With `baseFee = 0` (the reachable lower bound when `MinGasPrice < 1` or equals the default of 0), the zero `feeCap` passes the base-fee comparison, `EffectiveGasPrice` returns 0, and `DeductFeeDecorator` deducts nothing. A user who provides any nonzero fee coins passes `MinGasPriceDecorator` but pays zero effective fee.

---

### Finding Description

In `ante/evm/fee_checker.go`, `NewDynamicFeeChecker` computes the per-gas price cap as:

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))   // line 83 – integer division
```

`fee` is the raw integer amount of EVM-denom coins attached to the tx; `gas` is the gas limit. `sdkmath.Int.Quo` truncates toward zero. Whenever `fee < gas`, `feeCap = 0`.

The subsequent check is:

```go
if feeCap.LT(baseFeeInt) {   // line 86
    return nil, 0, errorsmod.Wrapf(...)
}
```

When `baseFee = 0` (see below), `feeCap = 0` satisfies `0 >= 0`, so the check passes.

`EffectiveGasPrice` is then:

```go
effectivePrice := sdkmath.NewIntFromBigInt(
    types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
// x/evm/types/utils.go:232 → min(tipCap + baseFee, feeCap) = min(MaxInt64+0, 0) = 0
```

So `effectiveFee = 0 * gas = 0`.

`DeductFeeDecorator.checkDeductFee` in `ante/evm/nativefee.go` only deducts when the fee is nonzero:

```go
if !fee.IsZero() {          // line 110
    err := evmkeeper.DeductFees(...)
}
```

With `effectiveFee = 0`, no coins are transferred to the fee collector.

**When is `baseFee = 0` reachable?**

`CalculateBaseFee` in `x/feemarket/keeper/eip1559.go` clamps the base fee at:

```go
minGasPrice := params.MinGasPrice.TruncateInt().BigInt()   // line 103
return ethermint.BigMax(x.Sub(parentBaseFee, baseFeeDelta), minGasPrice)
```

`MinGasPrice` is a `sdk.Dec` with a **default of 0** (`x/feemarket/spec/07_params.md`). Any value `< 1` truncates to 0, so the base fee can decrease to 0 on any chain with low traffic. `GetBaseFee` in `x/evm/types/utils.go:244` returns `new(big.Int)` (i.e., 0, not nil) when the stored base fee is 0 and London HF is active, so the code does **not** fall back to the validator min-gas-price path.

---

### Impact Explanation

When `baseFee = 0`:

- A user submits a Cosmos tx with `gas = 1_000_000` and `fee = 1` (one indivisible unit).
- `MinGasPriceDecorator` is skipped (`minGasPrice.IsZero()`).
- `feeCap = 1 / 1_000_000 = 0`; `effectiveFee = 0`.
- `DeductFeeDecorator` deducts nothing.
- The user executes the tx for free; the fee collector receives 0.

With a fractional `MinGasPrice` (e.g., `0.5`):

- `MinGasPriceDecorator` requires `fee >= ceil(0.5 × 1_000_000) = 500_000`.
- User provides `fee = 500_001` → passes the check.
- `feeCap = 500_001 / 1_000_000 = 0` → `effectiveFee = 0`.
- The 500,001 coins are **never deducted**; the user can reuse them for unlimited txs.

This is a fee mis-accounting bug in the ante handler: valid user funds that should be collected as fees are never transferred, and the fee-market invariant (every accepted tx pays at least `baseFee × gasUsed`) is violated.

---

### Likelihood Explanation

- `MinGasPrice` defaults to 0, so `baseFee` can reach 0 on any low-traffic Ethermint chain without additional configuration.
- Any chain that sets a sub-integer `MinGasPrice` (e.g., `0.001`) is permanently vulnerable because `TruncateInt() = 0`.
- The attacker only needs to submit a standard Cosmos tx with a large gas limit and a tiny fee — no privileged access, no special tooling.
- The `NewDynamicFeeChecker` is activated whenever `DynamicFeeChecker = true` in `HandlerOptions` (the recommended production setting).

---

### Recommendation

Replace the integer division with a ceiling division or a decimal comparison so that a fee of `k` coins with gas `g` yields `feeCap = ceil(k / g)` rather than `floor(k / g)`. Alternatively, reject the tx when `fee < baseFee * gas` before performing the division:

```go
// Before dividing, verify the raw fee covers at least baseFee * gas
requiredFee := baseFeeInt.Mul(sdkmath.NewIntFromUint64(gas))
if fee.LT(requiredFee) {
    return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, ...)
}
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
```

Additionally, `CalculateBaseFee` should use `params.MinGasPrice.Ceil().RoundInt()` (or a round-up) as the lower bound instead of `TruncateInt()`, so that a nonzero `MinGasPrice` always produces a nonzero integer floor for the base fee.

---

### Proof of Concept

**Setup:** Ethermint chain with London HF active, `NoBaseFee = false`, `MinGasPrice = 0` (default), base fee has decayed to 0 after many empty blocks.

**Step 1 – Confirm base fee is 0:**
```
baseFee = CalculateBaseFee(ctx)
// parentBaseFee=1, gasUsed=0, gasTarget=T
// baseFeeDelta = 1*T / T / 8 = 0  (integer division)
// result = max(1 - 0, TruncateInt(0)) = max(1, 0) = 1
// ... after enough blocks with zero gas used, baseFee reaches 0
```

**Step 2 – Submit Cosmos tx:**
```
gas    = 1_000_000
fee    = 1  (1 aevmos)
```

**Step 3 – Trace through ante handler:**
```
MinGasPriceDecorator:  minGasPrice = 0 → skipped
NewDynamicFeeChecker:
  feeCap = 1 / 1_000_000 = 0          ← integer truncation
  baseFeeInt = 0
  feeCap(0) >= baseFeeInt(0)           ← passes
  effectivePrice = min(MaxInt64+0, 0) = 0
  effectiveFee = 0 * 1_000_000 = 0
DeductFeeDecorator:
  fee.IsZero() == true → no deduction  ← 0 coins sent to fee collector
```
