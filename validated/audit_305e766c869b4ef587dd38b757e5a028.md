### Title
Missing Class Declaration Validation in `execute_replace_class` Allows Permanent Fund Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary class hash as the replacement class without verifying that the hash corresponds to a declared class. This is directly analogous to the external report's pattern: a critical state-gating check is absent, allowing a state transition that should be rejected to proceed, with irreversible financial consequences.

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function updates a contract's class hash in `contract_state_changes` without checking whether the new class hash has ever been declared (i.e., exists in `contract_class_changes`):

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
``` [1](#0-0) 

The developers themselves flagged this with a `TODO` at line 898. The check is entirely absent — `class_hash` from the request is used directly to update state with no lookup into `contract_class_changes`.

When a future transaction calls the affected contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,   // returns 0 for undeclared class
);
``` [2](#0-1) 

If the class hash was never declared, `dict_read` returns `0` (the default dict value), and `find_element` fails to locate a compiled class fact with hash `0`, making the contract permanently uncallable.

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once `replace_class` is called with an undeclared class hash, the contract's `StateEntry.class_hash` is permanently set to a value that has no corresponding compiled class. Every subsequent transaction attempting to call that contract will fail at the OS execution level. Any ERC-20 balances, NFTs, or other assets stored in or controlled by that contract's storage become permanently inaccessible. The state change is committed to the Merkle tree and cannot be undone without a protocol-level intervention.

### Likelihood Explanation

The attack path is fully reachable by any unprivileged transaction sender who controls (or can influence) a contract that exposes a `replace_class` call path. No privileged role, leaked key, or external dependency is required. The attacker simply:

1. Deploys or controls a contract holding funds (or tricks users into depositing into it).
2. Sends a transaction that triggers `replace_class` with an arbitrary felt value that is not a declared class hash.
3. The OS accepts the syscall — no validation is performed.
4. The contract's class is permanently replaced with a non-existent class.
5. All funds in the contract are frozen.

The `TODO` comment confirms the developers are aware the check is missing, meaning the code is in a known-incomplete state that is exploitable today.

### Recommendation

Before updating `contract_state_changes` in `execute_replace_class`, verify that `class_hash` is present in `contract_class_changes` (i.e., its compiled class hash is non-zero):

```cairo
// Verify the new class hash has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the check already performed in `execute_entry_point` when resolving a class for execution, and closes the gap between what the OS accepts during `replace_class` and what it can actually execute later.

### Proof of Concept

1. Attacker deploys `VictimVault` — a contract that holds user funds and exposes a `replace_class(new_hash)` external function that calls the `replace_class` syscall.
2. Users deposit tokens into `VictimVault`.
3. Attacker calls `VictimVault.replace_class(0xdeadbeef)` where `0xdeadbeef` is never declared.
4. The OS executes `execute_replace_class`: no check is performed; `contract_state_changes` is updated with `class_hash = 0xdeadbeef`.
5. The block is proven successfully (the `replace_class` transaction itself does not fail).
6. In any subsequent block, any transaction calling `VictimVault` reaches `execute_entry_point`, which does `dict_read(key=0xdeadbeef)` → returns `0`, then `find_element(..., key=0)` → fails to find a compiled class fact.
7. The contract is permanently uncallable; all deposited funds are frozen. [3](#0-2)
