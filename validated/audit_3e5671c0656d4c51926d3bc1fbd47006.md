Missing Upper-Bound Validation on Derived `max_fee` Causes Unprovable Blocks — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

### Summary

`compute_max_possible_fee` can return a value exceeding `2^128 − 1`. The subsequent `assert_nn_le(actual_fee, max_fee)` call in `charge_fee` uses Cairo's range-check builtin, which can only verify values in `[0, 2^128)`. When `max_fee ≥ 2^128`, no valid `actual_fee` exists that satisfies the check, making the block unprovable and halting the network.
