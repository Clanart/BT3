### Title
Unvalidated L1 WEI Gas Prices in `ProposalInit` Allow Malicious Proposer to Commit Wrong Block Hash and Corrupt ETH-Denominated Fee Calculations — (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_block_info_valid` validates only the FRI-denominated L1 gas prices against the oracle, silently discarding the validator's own computed WEI prices. The WEI prices carried in `ProposalInit` (`l1_gas_price_wei`, `l1_data_gas_price_wei`) are never checked. Because these WEI values are (a) committed verbatim into the block hash and (b) used to derive the implicit ETH/STRK conversion rate that sets `l2_gas_price_wei` for execution, a malicious proposer can embed arbitrary WEI prices while keeping FRI prices within the allowed margin, causing every validator to accept a block whose hash and ETH-denominated fee schedule are wrong.

---

### Finding Description

**Root cause — discarded WEI prices in `is_block_info_valid`**

`is_block_info_valid` calls `get_l1_prices_in_fri_and_wei`, which returns both FRI and WEI prices computed from the oracle. The WEI half is immediately discarded:

```rust
let (l1_gas_prices_fri, _l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(
    l1_gas_price_provider,
    init_proposed.timestamp,
    block_info_validation.previous_block_info.as_ref(),
    gas_price_params,
)
.await;
```

Only the FRI prices are then checked:

```rust
if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, l1_gas_price_margin_percent)
    && within_margin(l1_data_gas_price_fri_proposed, l1_data_gas_price_fri, l1_gas_price_margin_percent))
```

`init_proposed.l1_gas_price_wei` and `init_proposed.l1_data_gas_price_wei` are never compared against `_l1_gas_prices_wei`. [1](#0-0) 

**Downstream corruption 1 — block hash**

`gas_prices_to_hash` chains all four WEI/FRI price fields (and `l2_gas_price_wei`) into the Poseidon block hash. Arbitrary WEI values produce an arbitrary block hash commitment:

```rust
.chain(&l1_gas_price.price_in_wei.0.into())
.chain(&l1_gas_price.price_in_fri.0.into())
.chain(&l1_data_gas_price.price_in_wei.0.into())
.chain(&l1_data_gas_price.price_in_fri.0.into())
.chain(&l2_gas_price.price_in_wei.0.into())
.chain(&l2_gas_price.price_in_fri.0.into())
``` [2](#0-1) 

**Downstream corruption 2 — derived `eth_to_fri_rate` and `l2_gas_price_wei`**

`convert_to_sn_api_block_info` derives the ETH/STRK conversion rate from the ratio of the proposal's own WEI and FRI prices, then uses it to compute `l2_gas_price_wei`:

```rust
let previous_block_info = PreviousBlockInfo::from(init);
let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;
let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
```

If `l1_gas_price_wei` is set to 1 while `l1_gas_price_fri` is valid, the implied rate becomes `fri / 1 = fri` (orders of magnitude too high), making `l2_gas_price_wei` orders of magnitude too low. [3](#0-2) 

**Analog to the Pyth confidence-interval bug**

The external bug: Pyth provides a price *and* a confidence interval; the system uses the price without checking whether the interval is too wide. Here: the oracle provides both WEI and FRI prices; the validator checks only the FRI price and ignores the WEI price. The WEI/FRI ratio is the implicit "confidence interval" — a large discrepancy between the two signals an unreliable or manipulated reading — but no such cross-check is performed.

---

### Impact Explanation

| Corrupted value | Effect |
|---|---|
| `l1_gas_price_wei` / `l1_data_gas_price_wei` in block header | Block hash diverges from the correct value; wrong hash committed to L1 |
| Derived `eth_to_fri_rate` | All ETH↔STRK conversions in the block are wrong |
| Derived `l2_gas_price_wei` | ETH-denominated L2 gas price is wrong; transactions paying fees in ETH are over- or under-charged |
| L1 handler fee in ETH | L1 handlers are charged the wrong ETH amount |

A proposer who sets `l1_gas_price_wei = 1` while keeping `l1_gas_price_fri` within the allowed margin causes `l2_gas_price_wei` to collapse to near-zero, letting any transaction pay negligible ETH fees. Conversely, inflating `l1_gas_price_wei` makes `l2_gas_price_wei` enormous, causing legitimate ETH-paying transactions to revert with insufficient balance.

Matches: **Critical — Incorrect fee, gas, resource accounting, balance, or L1 gas price effect with economic impact** and **Critical — Wrong receipt/revert result from execution logic**.

---

### Likelihood Explanation

The trigger requires the attacker to be the consensus proposer for a round. In the current single-sequencer deployment this means a compromised sequencer operator. In a decentralized validator set (the stated roadmap), any staked validator that wins a proposer slot can trigger this without any other privilege. The `within_margin` check on FRI prices provides no constraint on WEI prices, so the attack requires only crafting a `ProposalInit` with a mismatched WEI value — no cryptographic break or state manipulation is needed.

---

### Recommendation

In `is_block_info_valid`, after computing `_l1_gas_prices_wei`, validate the proposed WEI prices against the oracle-computed WEI prices using the same `within_margin` check already applied to FRI prices:

```rust
let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(...).await;

// existing FRI checks ...

if !(within_margin(init_proposed.l1_gas_price_wei, l1_gas_prices_wei.l1_gas_price, margin)
    && within_margin(init_proposed.l1_data_gas_price_wei, l1_gas_prices_wei.l1_data_gas_price, margin))
{
    return Err(ValidateProposalError::InvalidBlockInfo(...));
}
```

Additionally, add a cross-consistency check that the ratio `l1_gas_price_fri / l1_gas_price_wei` (the implied ETH/STRK rate) is within an acceptable margin of the oracle's reported rate, analogous to the Pyth fix of rejecting prices whose confidence interval is too wide.

---

### Proof of Concept

1. Attacker is the proposer for round R.
2. Attacker queries the oracle: oracle returns `eth_to_fri_rate = 1000`, `l1_gas_price_wei = 100 gwei`, `l1_gas_price_fri = 100_000 fri`.
3. Attacker crafts `ProposalInit` with:
   - `l1_gas_price_fri = 100_000` (within margin → passes `within_margin` check)
   - `l1_gas_price_wei = 1` (1 wei, not validated)
4. Validators call `is_block_info_valid`: FRI check passes; WEI check absent → proposal accepted.
5. `convert_to_sn_api_block_info` computes `eth_to_fri_rate = 100_000 / 1 = 100_000` (100× too high).
6. `l2_gas_price_wei = l2_gas_price_fri / 100_000` → 100× too small.
7. `gas_prices_to_hash` commits `price_in_wei = 1` into the block hash → wrong block hash stored on L1.
8. All ETH-paying transactions in the block are charged 100× less than they should be. [4](#0-3) [5](#0-4) [6](#0-5)

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let previous_block_info = PreviousBlockInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}
