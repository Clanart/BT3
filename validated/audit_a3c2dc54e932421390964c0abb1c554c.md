### Title
Missing Class Hash Existence Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS program does not validate that the new class hash provided by a calling contract corresponds to an actually declared class. Any unprivileged contract can invoke `replace_class` with an arbitrary, undeclared class hash. The OS accepts this state transition unconditionally, permanently bricking the contract and freezing any funds stored in it. The code itself acknowledges this missing check with an explicit TODO comment.

---

### Finding Description

In `execute_replace_class` (`syscall_impls.cairo`, lines 877–916), the OS processes a `replace_class` syscall by reading the new class hash directly from the user-controlled syscall request and writing it into the contract state dictionary without any validation that the hash corresponds to a declared class:

```cairo
let class_hash = request.class_hash;
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}
tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The `class_hash` value is taken verbatim from `request.class_hash`, which is fully attacker-controlled. The hint `%{ GetContractAddressStateEntry %}` only fetches the *current* state entry of the calling contract; it performs no existence check on the *new* class hash. The OS then commits this invalid state transition into `contract_state_changes`, which is later squashed and committed to the state root via `state_update`.

This is the direct analog of the VRF Handler report's vulnerability class: an immutable OS program directly accepts and commits a dependency (class hash) without verifying the dependency exists, with no recovery path once the invalid state is committed. [1](#0-0) 

---

### Impact Explanation

Once a contract's class hash is replaced with an undeclared value, the OS can no longer resolve any entry points for that contract. Every subsequent call to the contract — including calls to withdraw funds — will fail at entry point lookup. Any tokens or assets stored in the contract's storage are permanently inaccessible.

**Impact: Critical — Permanent freezing of funds.**

This matches the allowed impact scope. The state transition is irreversible: the invalid class hash is committed to the state root, and there is no OS-level mechanism to recover a contract whose class hash points to a non-existent class.

---

### Likelihood Explanation

The attack path is reachable by any unprivileged transaction sender:

1. A shared vault or pool contract (holding other users' funds) calls `replace_class` as part of an upgrade flow, but passes an incorrect or non-existent class hash (e.g., due to a bug in the upgrade logic, or a malicious contract owner).
2. The OS accepts the syscall without validation.
3. The contract is permanently bricked; all depositors' funds are frozen.

No privileged role, leaked key, or external dependency compromise is required. The `replace_class` syscall is a standard, publicly accessible syscall available to any deployed contract. The missing check is explicitly acknowledged in the codebase as a known gap (`TODO(Yoni, 1/1/2026)`), confirming the developers are aware the validation is absent. [2](#0-1) 

---

### Recommendation

Before committing the new class hash into `contract_state_changes`, the OS must verify that the class hash exists in `contract_class_changes` (i.e., it has been declared in the current or a prior block). Concretely, a lookup into the `contract_class_changes` dictionary should confirm that `class_hash` maps to a non-zero compiled class hash. If the class hash is not declared, the syscall should write a failure response and leave the contract state unchanged, consistent with how other invalid syscall arguments are handled (e.g., `write_failure_response`). [3](#0-2) 

---

### Proof of Concept

1. Deploy a shared vault contract `V` that holds ERC-20 tokens on behalf of multiple users.
2. The vault's upgrade function calls `replace_class(new_class_hash)` where `new_class_hash` is any felt value not corresponding to a declared class (e.g., `0xdeadbeef`, or a hash that was never declared via a `declare` transaction).
3. The OS executes `execute_replace_class`: it reads `class_hash = 0xdeadbeef` from the syscall request, fetches the current `StateEntry` for `V`, constructs a new `StateEntry` with `class_hash=0xdeadbeef`, and writes it into `contract_state_changes` — **no existence check is performed**.
4. The block is proven and the state root is updated. The vault contract `V` now has class hash `0xdeadbeef` in the committed state.
5. Any subsequent `call_contract` or `invoke` targeting `V` fails: the OS looks up `0xdeadbeef` in the compiled class facts bundle, finds no entry, and the call reverts.
6. All user funds stored in `V`'s storage are permanently frozen with no recovery path. [4](#0-3) [5](#0-4)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
