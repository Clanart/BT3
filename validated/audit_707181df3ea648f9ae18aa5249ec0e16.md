### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Freezal of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts an arbitrary, user-controlled class hash from the syscall request and writes it directly to the contract state without verifying that the class hash corresponds to a previously declared contract class. This is the direct analog of the "arbitrary token addresses" vulnerability class: a public, user-reachable function accepts an unvalidated address/hash parameter. A malicious contract can replace its own class with an undeclared hash, permanently corrupting its state and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads `request.class_hash` directly from the user-supplied syscall request and writes it into `contract_state_changes` with no check that the hash corresponds to a declared class:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The TODO comment at line 898 explicitly acknowledges the missing validation. [1](#0-0) 

When the contract is subsequently called, `execute_entry_point` performs:

1. A `dict_read` on `contract_class_changes` keyed by the (now-corrupted) `class_hash` — returning `0` if the hash was never declared.
2. A `find_element` lookup in `compiled_class_facts_bundle` for that compiled class hash.

If the class hash is undeclared, `find_element` with key `0` finds no matching entry and the OS execution fails. [2](#0-1) 

The `execute_replace_class` function is reachable via the `REPLACE_CLASS_SELECTOR` branch in `execute_syscalls`, which is callable by any executing contract: [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

A contract that holds token balances or acts as a vault can call `replace_class` with an arbitrary undeclared class hash (e.g., `felt(0xdeadbeef)`). After this point:

- The contract's `class_hash` field in `contract_state_changes` is set to an undeclared value.
- Any future call to the contract causes `execute_entry_point` to fail when looking up the compiled class, making the contract permanently uncallable.
- All funds (ERC-20 balances, NFTs, ETH) held by the contract are permanently frozen with no recovery path, since the contract can never execute again.

---

### Likelihood Explanation

**Medium.** Any unprivileged user who deploys a contract can immediately trigger this by issuing a `replace_class` syscall with an arbitrary felt value as the class hash. No special privileges, leaked keys, or operator cooperation are required. The attacker only needs to deploy a contract and call `replace_class` from within it. The missing check is explicitly documented as a TODO, confirming it is a known gap in the current implementation.

---

### Recommendation

Before writing the new class hash to `contract_state_changes`, validate that the provided `class_hash` exists as a key in `contract_class_changes` (i.e., it has been declared in the current or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` with `key=class_hash` and assert the returned `compiled_class_hash` is non-zero before proceeding with the state update. This mirrors the validation already implicitly required by `execute_entry_point` when it calls `find_element`.

---

### Proof of Concept

1. Deploy a contract `Vault` that holds user funds and exposes a `replace_and_freeze` external function.
2. Inside `replace_and_freeze`, issue the `replace_class` syscall with `class_hash = 0xdeadbeef` (an undeclared hash).
3. The OS processes `execute_replace_class`: reads `request.class_hash = 0xdeadbeef`, skips the missing validation (line 898 TODO), and writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes`.
4. In any subsequent block, a user calls `Vault.withdraw(...)`.
5. `execute_entry_point` calls `dict_read(key=0xdeadbeef)` on `contract_class_changes` → returns `0` (undeclared).
6. `find_element(..., key=0)` on `compiled_class_facts_bundle` finds no match → OS execution fails.
7. The `Vault` contract is permanently uncallable; all funds are frozen. [4](#0-3) [5](#0-4)
