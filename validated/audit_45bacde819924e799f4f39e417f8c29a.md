### Title
Zero Normalized Amount Permanently Freezes User Funds in `sign_transfer` - (File: `near/omni-bridge/src/lib.rs`)

### Summary
When a user initiates a NEARâ†’EVM (or NEARâ†’Solana/StarkNet) transfer with an amount so small that `normalize_amount(amount - fee)` rounds down to zero, the `sign_transfer` function permanently reverts with `InvalidAmountToTransfer`. Because no cancel/refund path exists for this state, the user's tokens are irrecoverably locked in the bridge contract.

### Finding Description

`init_transfer` (called via `ft_on_transfer`) validates only that `fee.fee < amount`: [1](#0-0) 

It does **not** validate that `normalize_amount(amount - fee) > 0`. Normalization divides by `10^(origin_decimals - decimals)`: [2](#0-1) 

For a token with `origin_decimals = 24` (NEAR) and `decimals = 6` (EVM), the divisor is `10^18`. Any `amount_without_fee < 10^18` normalizes to zero.

`sign_transfer` then computes the normalized amount and enforces `> 0`: [3](#0-2) 

When this `require!` fires, the function panics before reaching the MPC signing call. The transfer message is **never removed** from `pending_transfers` because:

- `sign_transfer_callback` is never reached (MPC was never called)
- `claim_fee_callback` requires a proof of destination-chain finalization, which never happened
- There is no `cancel_transfer` or user-accessible refund function [4](#0-3) 

The transfer message and the locked tokens remain in the bridge contract with no recovery path.

### Impact Explanation

**Permanent freezing of user funds.** Any user who initiates a transfer where `(amount - fee) < 10^(origin_decimals - decimals)` will have their tokens permanently locked. The bridge's `locked_tokens` counter is incremented at `init_transfer_internal`: [5](#0-4) 

and is never decremented because the only decrement paths (`claim_fee_callback`, `fin_transfer_send_tokens_callback`) are unreachable for this transfer.

### Likelihood Explanation

Any unprivileged bridge user can trigger this by calling `ft_transfer_call` with a small amount to a destination chain whose token has fewer decimals than the NEAR-side token. For example, bridging 1 yoctoNEAR-denominated token (24 decimals) to an EVM chain where the token has 6 decimals requires `amount_without_fee >= 10^18` to survive normalization. Amounts below this threshold are silently accepted by `init_transfer` and permanently frozen. The user has no indication at `init_transfer` time that the transfer is unprocessable.

### Recommendation

Add a normalization check inside `init_transfer` (or `init_transfer_internal`) before locking tokens:

```rust
let token_address = self.get_token_address(
    transfer_message.get_destination_chain(),
    self.get_token_id(&transfer_message.token),
);
if let Some(addr) = token_address {
    if let Some(decimals) = self.token_decimals.get(&addr) {
        let normalized = Self::normalize_amount(
            transfer_message.amount_without_fee().unwrap_or(0),
            decimals,
        );
        require!(normalized > 0, BridgeError::InvalidAmountToTransfer.as_ref());
    }
}
```

Alternatively, reject the transfer at `init_transfer` time if `amount_without_fee < 10^(origin_decimals - decimals)`.

### Proof of Concept

1. Token is registered with `origin_decimals = 24`, `decimals = 6` (diff = 18, divisor = `10^18`).
2. User calls `ft_transfer_call` with `amount = 500_000_000_000_000_000` (< `10^18`), `fee = 0`, valid EVM recipient.
3. `init_transfer` passes the `fee < amount` check, stores the transfer message, locks tokens, increments `current_origin_nonce`.
4. Relayer calls `sign_transfer` for this `transfer_id`.
5. `normalize_amount(500_000_000_000_000_000, {decimals:6, origin_decimals:24})` = `500_000_000_000_000_000 / 10^18` = `0`.
6. `require!(0 > 0, ...)` panics â†’ `sign_transfer` reverts.
7. Transfer message remains in `pending_transfers`; tokens remain locked.
8. No subsequent call can remove the transfer or recover the tokens. [6](#0-5) [7](#0-6)

    fn init_transfer_internal(
        &mut self,
        transfer_message: TransferMessage,
        storage_owner: AccountId,
    ) -> U128 {
        let required_storage_balance = self
            .add_transfer_message(transfer_message.clone(), storage_owner.clone())
            .saturating_add(NearToken::from_yoctonear(transfer_message.fee.native_fee.0));

        if self
            .try_update_storage_balance(
                storage_owner,
                required_storage_balance,
                NearToken::from_yoctonear(0),
            )
            .is_err()
        {
            self.remove_transfer_message_without_refund(transfer_message.get_transfer_id());
            return transfer_message.amount;
        }

        if let OmniAddress::Near(token_id) = transfer_message.token.clone() {
            self.burn_tokens_if_needed(token_id.clone(), transfer_message.amount);

            self.lock_tokens_if_needed(
                transfer_message.get_destination_chain(),
                &token_id,
                transfer_message.amount.0,
            );
        } else {
            self.remove_transfer_message_without_refund(transfer_message.get_transfer_id());
            return transfer_message.amount;
        }

        env::log_str(&OmniBridgeEvent::InitTransferEvent { transfer_message }.to_log_string());
        U128(0)
    }
