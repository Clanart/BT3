### Title
Missing Declared Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall in the StarkNet OS does not verify that the caller-supplied class hash is a previously declared class before updating the contract's state entry. This is directly analogous to M-13's missing `allowedVaults` check: a sensitive state-mutation operation lacks the authorization/existence check that should gate it. Any contract can replace its own class hash with an arbitrary, undeclared value, permanently rendering itself non-executable and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class` (`syscall_impls.cairo`, lines 877–916), the OS reads the caller-supplied `class_hash` from the syscall request and immediately writes it into `contract_state_changes` with no check that the hash exists in `contract_class_changes` (the declared-class registry):

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment at line 898 is an explicit, developer-acknowledged admission that the required check is absent. Compare this to `execute_declare_transaction` (`transaction_impls.cairo`, line 817), which enforces `prev_value=0` to prevent re-declaration, and `deploy_contract` (`deploy_contract.cairo`, line 53), which asserts `state_entry.class_hash = UNINITIALIZED_CLASS_HASH` before deployment — both of which enforce existence/authorization invariants. `execute_replace_class` enforces none.

The class hash written into `contract_state_changes` is the authoritative source used by the OS when executing any future call to that contract. If it points to a class that was never declared in `contract_class_changes`, the OS hint that resolves the compiled class will find no entry, making the contract permanently non-executable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value via `replace_class`, every subsequent transaction targeting that contract will fail at class resolution time. The contract's storage and any token balances it holds (ETH, ERC-20, etc.) become permanently inaccessible. There is no recovery path: the class hash is committed to the global state tree, and no future transaction can execute the contract to withdraw funds.

---

### Likelihood Explanation

The `replace_class` syscall is available to every executing contract with no privileged gating. The attacker-controlled entry path requires only:

1. Deploying (or controlling) any contract.
2. Having that contract issue a `replace_class` syscall with a felt value that was never passed through `execute_declare_transaction`.
3. The OS accepts the update unconditionally.

The missing check is explicitly flagged with a TODO dated 2026, confirming it is a known, unimplemented invariant in production code. No leaked key, trusted operator, or network-level attack is required.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that it exists in `contract_class_changes`:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the pattern already used in `execute_declare_transaction` where `prev_value=0` enforces that only genuinely new classes are registered.

---

### Proof of Concept

1. Attacker deploys contract `C` and funds it with ETH (or any token).
2. `C`'s constructor (or any entry point) issues:
   ```
   replace_class(class_hash=0xdeadbeefdeadbeef)  // never declared
   ```
3. The OS executes `execute_replace_class`:
   - Reads `class_hash = 0xdeadbeefdeadbeef` from the request.
   - Skips the missing declared-class check (line 898 TODO).
   - Writes `StateEntry(class_hash=0xdeadbeefdeadbeef, ...)` into `contract_state_changes`.
4. The block is proven and committed. The global state tree now records `C`'s class hash as `0xdeadbeefdeadbeef`.
5. Any future transaction calling `C` causes the OS to look up `0xdeadbeefdeadbeef` in `contract_class_changes` — it is absent — and execution fails.
6. All funds in `C` are permanently frozen with no recovery mechanism. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

        // Not enough gas to execute the syscall.
        return ();
