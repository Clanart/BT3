### Title
Unbounded Storage Iteration in `DeleteAccount` During `StateDB.Commit()` Enables Consensus-Path DoS - (File: `x/evm/keeper/statedb.go`)

### Summary

`DeleteAccount` iterates over every storage slot of a self-destructed contract and deletes them one-by-one via `ForEachStorage`. This loop runs inside `StateDB.Commit()`, which is called on the consensus-critical block-processing path, outside the EVM gas meter. An attacker who pre-fills a contract with a large number of storage slots (paying EVM gas only for the writes) can trigger a SELFDESTRUCT that forces an unbounded KV-store iteration during block commit, potentially causing the block to time out and halting consensus.

### Finding Description

`DeleteAccount` in `x/evm/keeper/statedb.go` clears all storage slots by iterating over the entire KV prefix for the contract address:

```go
k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool {
    k.SetState(ctx, addr, key, nil)
    return true
})
``` [1](#0-0) 

`ForEachStorage` opens a `KVStorePrefixIterator` and steps through every entry under the contract's storage prefix with no upper bound:

```go
iterator := storetypes.KVStorePrefixIterator(store, prefix)
defer iterator.Close()
for ; iterator.Valid(); iterator.Next() { ... }
``` [2](#0-1) 

`DeleteAccount` is invoked from `StateDB.Commit()` for every self-destructed object:

```go
if err := s.keeper.DeleteAccount(cacheCtx, obj.Address()); err != nil {
    return errorsmod.Wrap(err, "failed to delete account")
}
``` [3](#0-2) 

`StateDB.Commit()` is called from `ApplyMessageWithConfig` on the consensus path (`commit=true`):

```go
if commit {
    if err := stateDB.Commit(); err != nil { ... }
}
``` [4](#0-3) 

Critically, `Commit()` executes under the **infinite Cosmos SDK gas meter** that was installed for EVM execution. The KV-store deletions inside `ForEachStorage` consume no EVM gas and are not reflected in `totalGasUsed` that is later reported to the block gas meter:

```go
k.ResetGasMeterAndConsumeGas(ctx, totalGasUsed)
``` [5](#0-4) 

This creates a direct asymmetry: the attacker pays EVM gas only for `SSTORE` writes (20,000 gas each), but the corresponding `DELETE` operations during `Commit()` are entirely unmetered at the Cosmos layer.

### Impact Explanation

An attacker who accumulates N storage slots in a contract (across multiple transactions, each bounded only by the per-transaction gas limit) and then triggers `SELFDESTRUCT` forces the block proposer to execute N sequential KV-store deletions during `Commit()`. Because this work is outside the EVM gas meter and outside the block gas meter, it is invisible to the fee/gas accounting system. If N is large enough, the block processing time exceeds the Tendermint/CometBFT block timeout, causing the proposer to miss its turn and potentially halting consensus.

This matches the allowed Critical impact: **"Valid unprivileged transaction, RPC submission, or block-processing path can halt the chain."**

### Likelihood Explanation

- **Pre-EIP-6780 (pre-Shanghai) chains**: An attacker can fill a contract over many blocks (each block contributing up to `blockGasLimit / 20000` new slots), then SELFDESTRUCT in a single transaction. The cost is proportional to the number of slots written, but the deletion work is free. This is a direct analog to the Canto finding.
- **Post-EIP-6780 (Shanghai+) chains**: `SelfDestruct6780` only destructs contracts created in the same transaction (`newContract == true`), bounding slots to `txGasLimit / 20000`. At a 30M gas limit this is ~1,500 slots per transaction — less severe but still an unmetered cost spike during `Commit()`. [6](#0-5) 

The entry path requires no privilege: any EOA can deploy a contract, fill its storage, and call `SELFDESTRUCT`.

### Recommendation

1. **Bound the deletion loop**: Charge Cosmos SDK gas (or a synthetic "deletion gas") for each slot deleted in `ForEachStorage` inside `DeleteAccount`, so the work is reflected in the block gas meter.
2. **Lazy deletion**: Instead of eagerly iterating and deleting all slots in `DeleteAccount`, mark the account as deleted and perform storage cleanup asynchronously (e.g., in `EndBlock` with a per-block slot-deletion budget), matching go-ethereum's lazy trie-pruning approach.
3. **Per-transaction slot cap**: Enforce a maximum number of storage slots that can be deleted in a single `DeleteAccount` call, returning an error or deferring the remainder if the cap is exceeded.

### Proof of Concept

1. Attacker deploys contract `C` and calls a function that executes `SSTORE` in a loop, filling `C`'s storage with `N` slots across multiple transactions (each paying EVM gas for the writes).
2. Attacker submits a transaction calling `SELFDESTRUCT` on `C` (pre-EIP-6780) or deploys-and-destructs in one transaction (post-EIP-6780).
3. The block proposer includes the transaction and calls `ApplyTransaction` → `ApplyMessageWithConfig(commit=true)`.
4. After EVM execution completes (SELFDESTRUCT costs a fixed EVM gas amount), `StateDB.Commit()` is called.
5. `Commit()` detects `obj.selfDestructed == true` and calls `keeper.DeleteAccount()`.
6. `DeleteAccount` calls `ForEachStorage`, opening a KV iterator over all `N` slots and issuing `N` `store.Delete()` calls — all under the infinite gas meter, unaccounted in `totalGasUsed`.
7. For sufficiently large `N`, the block processing time exceeds the consensus timeout, causing the proposer to miss its slot and potentially halting the chain. [7](#0-6) [8](#0-7)

func (k *Keeper) ForEachStorage(ctx sdk.Context, addr common.Address, cb func(key, value common.Hash) bool) {
	store := ctx.KVStore(k.storeKey)
	prefix := types.AddressStoragePrefix(addr)

	iterator := storetypes.KVStorePrefixIterator(store, prefix)
	defer iterator.Close()

	for ; iterator.Valid(); iterator.Next() {
		key := common.BytesToHash(iterator.Key())
		value := common.BytesToHash(iterator.Value())

		// check if iteration stops
		if !cb(key, value) {
			return
		}
	}
}

func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
	cosmosAddr := sdk.AccAddress(addr.Bytes())
	acct := k.accountKeeper.GetAccount(ctx, cosmosAddr)
	if acct == nil {
		return nil
	}

	// NOTE: only Ethereum accounts (contracts) can be selfdestructed
	_, ok := acct.(ethermint.EthAccountI)
	if !ok {
		return errorsmod.Wrapf(types.ErrInvalidAccount, "type %T, address %s", acct, addr)
	}

	// clear storage
	k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool {
		k.SetState(ctx, addr, key, nil)
		return true
	})

	// remove auth account
	k.accountKeeper.RemoveAccount(ctx, acct)

	k.debugLog(ctx, "account suicided",
		"ethereum-address", addr,
		"cosmos-address", cosmosAddr,
	)

	return nil
