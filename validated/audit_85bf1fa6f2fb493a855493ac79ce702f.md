### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the new class hash supplied by a contract corresponds to an actually-declared class. The OS unconditionally writes the caller-supplied hash into contract state. A contract (or a contract tricked by an attacker) can therefore set its own class hash to an arbitrary undeclared value, permanently rendering itself non-executable and freezing any funds it holds.

---

### Finding Description

`execute_replace_class` in `syscall_impls.cairo` processes the `replace_class` syscall. After deducting gas it reads the caller-supplied `request.class_hash` and writes it directly into `contract_state_changes` with no check that the hash exists in `contract_class_changes` (the dictionary of declared classes):

```cairo

    range_check_ptr,
    syscall_ptr: felt*,

    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.

**Analogy to the external report.** In the Streaming Protocol bug, `incentives[who]` was checked to be non-zero as a guard, but after `claimIncentive()` the mapping was zeroed, silently removing the guard and allowing `arbitraryCall()` to target the token. Here the guard — "the new class hash must be declared" — is simply never implemented. The OS accepts any felt as a valid class hash, just as `arbitraryCall()` accepted any address once the incentive mapping was cleared.

The `replace_class` syscall is dispatched unconditionally for any contract that issues it: [3](#0-2) 

---

### Impact Explanation

Once the OS commits an undeclared class hash to a contract's `StateEntry`, every subsequent call to that contract will fail at class-lookup time: the OS cannot find the bytecode for the hash. The contract becomes permanently non-executable. Any ERC-20 balances, ETH, or other assets stored in that contract's storage are irrecoverably frozen.

**Impact category: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

`replace_class` is a standard syscall available to every Cairo 1 and Cairo 0 contract without any privilege requirement. The attack surface is therefore the entire set of deployed contracts. Concrete realistic paths:

1. **Malicious upgradeable proxy**: A contract that exposes an upgrade function accepting a caller-supplied class hash (common in proxy patterns) can be called by any user with an undeclared hash. The OS will accept it, permanently bricking the proxy and all funds inside.
2. **Deliberate griefing**: An attacker deploys a contract that accepts user deposits, then calls `replace_class` with `class_hash = 1` (or any undeclared felt). Deposited funds are frozen with no recovery path.
3. **Reentrancy-assisted**: A contract that calls `replace_class` inside a callback can be triggered by any external caller who controls the reentrant call path.

No privileged role, leaked key, or external dependency is required. The entry point is a standard unprivileged transaction.

---

### Recommendation

Inside `execute_replace_class` (and its deprecated counterpart), after reading `request.class_hash`, perform a lookup in `contract_class_changes` to confirm the hash has a non-zero compiled class hash entry:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the pattern already used in `execute_declare_transaction`, which enforces `prev_value=0` and `assert_not_zero(compiled_class_hash)` before writing to `contract_class_changes`. [4](#0-3) 

---

### Proof of Concept

1. Attacker deploys contract `VaultAttack` that:
   - Accepts ETH/token deposits from users.
   - Exposes a public `freeze()` function that calls `replace_class(class_hash=0xdeadbeef)` where `0xdeadbeef` is never declared.
2. Users deposit funds into `VaultAttack`.
3. Attacker calls `freeze()`. The OS executes `execute_replace_class`, reads `class_hash = 0xdeadbeef`, finds no declared-class guard, and writes the invalid hash into `contract_state_changes`.
4. The state update is committed to the Merkle tree via `state_update` in `os.cairo`. [5](#0-4) 

5. Any subsequent `call_contract` or `invoke` targeting `VaultAttack` fails: the OS resolves `class_hash = 0xdeadbeef`, finds no compiled class, and reverts. All deposited funds are permanently frozen with no recovery mechanism in the protocol.

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
