### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS stores a caller-supplied class hash into contract state without verifying that the hash corresponds to a declared contract class. An unprivileged contract can invoke `replace_class` with an arbitrary, undeclared class hash. Any subsequent call to that contract will cause the OS proof to fail at `find_element`, permanently freezing any funds held by the contract.

---

### Finding Description

In `execute_replace_class`, the `class_hash` field from the syscall request is read directly and written into the contract's `StateEntry` with no existence check against the set of declared classes:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The `TODO` comment at line 898 explicitly acknowledges the missing check. [1](#0-0) 

When a subsequent transaction calls the affected contract, `execute_entry_point` reads the (now-invalid) class hash from `execution_context.class_hash` and performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
```

For an undeclared class hash, `dict_read` returns the default value `0`. The OS then calls:

```cairo
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [2](#0-1) 

`find_element` is a Cairo primitive that asserts the key exists in the array. If `compiled_class_hash = 0` (or any undeclared value) is not present in the compiled class facts bundle, the assertion fails and the OS proof is invalid. The sequencer cannot produce a valid proof for any block containing a call to the affected contract.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with an undeclared value and the transaction is finalized in a block, the state is committed. Every future transaction that calls the affected contract causes the OS Cairo program to fail at `find_element`, making it impossible to generate a valid ZK proof for any block containing such a call. Funds held by the contract become permanently inaccessible with no recovery path.

---

### Likelihood Explanation

**High.** The `replace_class` syscall is accessible to any contract without privilege restrictions. An attacker only needs to:
1. Deploy or control a contract that holds (or will receive) funds.
2. Issue a transaction that invokes `replace_class` with an arbitrary undeclared felt value as the class hash.

No leaked keys, operator cooperation, or network-level attack is required. The attack is a single on-chain transaction.

---

### Recommendation

Remove the `TODO` and implement the missing validation inside `execute_replace_class`. Before writing the new class hash to `contract_state_changes`, verify that the hash exists as a key in `contract_class_changes` (i.e., it was previously declared via a `declare` transaction). If the class hash is not declared, write a failure response instead of updating state, consistent with how other invalid-argument cases are handled in the syscall layer (e.g., `write_failure_response(..., ERROR_INVALID_ARGUMENT)`).

---

### Proof of Concept

1. Attacker deploys `VictimContract` which accepts ETH/STRK deposits from users and holds a balance.
2. Attacker calls a function on `VictimContract` that internally invokes the `replace_class` syscall with `class_hash = 0xdeadbeef` (an undeclared felt).
3. The OS processes `execute_replace_class` at lines 896–910 of `syscall_impls.cairo` with no validation; `contract_state_changes` is updated so `VictimContract`'s `StateEntry.class_hash = 0xdeadbeef`.
4. The transaction is included in a block and finalized.
5. In the next block, any user attempts to call `VictimContract` (e.g., to withdraw funds).
6. The OS runs `execute_entry_point`: `dict_read` on `contract_class_changes` with key `0xdeadbeef` returns `0`; `find_element` with key `0` fails because no compiled class with hash `0` exists in the bundle.
7. The OS proof for any block containing this call cannot be generated.
8. All funds deposited in `VictimContract` are permanently frozen. [3](#0-2) [4](#0-3)
