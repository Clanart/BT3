### Title
Missing Validation of New Class Hash in `execute_replace_class` Enables Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the new class hash provided by a contract corresponds to a previously declared class. Any contract can call `replace_class` with an arbitrary, undeclared class hash, permanently rendering itself uncallable and freezing all funds it holds. The missing check is explicitly acknowledged in the codebase via a TODO comment.

---

### Finding Description

In `execute_replace_class`, the OS reads the requested `class_hash` directly from the syscall request and writes it into the contract state without any validation: [1](#0-0) 

The critical missing check is explicitly acknowledged at line 898:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
```

After `replace_class` is called with an undeclared hash, the contract's `class_hash` field in `contract_state_changes` is permanently set to that undeclared value. [2](#0-1) 

When any subsequent transaction attempts to call this contract, `execute_entry_point` reads the class hash from state and performs a `dict_read` on `contract_class_changes`: [3](#0-2) 

For an undeclared class hash, `dict_read` returns the dict's default value of `0` (set by `dict_new()` in `initialize_state_changes`). The OS then calls `find_element` to locate a compiled class with hash `0`: [4](#0-3) 

`find_element` is a hint-based Cairo primitive that asserts the element exists. If no compiled class with hash `0` is present in the bundle, the assertion fails, causing the OS execution to abort. The sequencer's blockifier detects this failure during block building and permanently excludes all calls to the affected contract. Since `replace_class` can only be invoked from within the contract itself, and the contract is now permanently uncallable, there is no recovery path.

---

### Impact Explanation

Any funds held in a contract whose class has been replaced with an undeclared hash are **permanently frozen**. The contract becomes permanently uncallable by any party, including the original deployer. Because `replace_class` is the only mechanism to change a contract's class, and it can only be called from within the contract, there is no on-chain recovery mechanism. This is a **Critical — Permanent freezing of funds** impact.

---

### Likelihood Explanation

The attack is directly reachable by any unprivileged contract deployer or any contract that exposes a `replace_class` call path. No privileged role, leaked key, or external dependency is required. The missing validation is confirmed by the in-code TODO comment, meaning the OS was shipped without this guard. Any contract that calls `replace_class(undeclared_hash)` — whether maliciously designed or exploited via a reentrancy/logic bug — triggers the permanent freeze.

---

### Recommendation

Add a validation step inside `execute_replace_class` to verify that the provided `class_hash` corresponds to a declared class before writing it to state. Concretely, perform a `dict_read` on `contract_class_changes` for the new `class_hash` and assert the result is non-zero (i.e., a compiled class hash has been registered for it). This mirrors the existing pattern used in `execute_entry_point` and closes the gap identified by the TODO comment.

---

### Proof of Concept

1. Attacker deploys `MaliciousVault`, a contract that:
   - Accepts token deposits from users (storing balances in its storage).
   - Exposes a `freeze()` function that calls the `replace_class` syscall with an arbitrary undeclared felt (e.g., `0xdeadbeef...` or simply `1`).

2. Users deposit funds into `MaliciousVault`, trusting its declared class logic.

3. Attacker calls `freeze()`. The OS executes `execute_replace_class`:
   - Reads `class_hash = 0xdeadbeef...` from the request.
   - Skips the missing declared-class check (the TODO).
   - Writes `StateEntry(class_hash=0xdeadbeef..., ...)` into `contract_state_changes`.
   - Transaction succeeds and is committed to state.

4. In any subsequent block, a user attempts to call `MaliciousVault` to withdraw funds:
   - `execute_entry_point` reads `class_hash = 0xdeadbeef...` from state.
   - `dict_read` on `contract_class_changes` returns `0` (undeclared).
   - `find_element` with key `0` fails — no compiled class with hash `0` exists.
   - The sequencer's blockifier aborts the call and excludes it from the block.

5. All user funds in `MaliciousVault` are permanently frozen with no recovery path. [5](#0-4) [6](#0-5)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
