### Title
Missing Declared Class Existence Check in `execute_replace_class` Allows Permanent Contract Bricking - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary `class_hash` value from the caller without verifying that the hash corresponds to a previously declared contract class. This is the StarkNet analog of the DSProxy/DSExec missing contract existence check: just as calling a non-existent or self-destructed EVM contract succeeds silently, replacing a contract's class with an undeclared class hash succeeds silently in the OS. The result is a permanently bricked contract â€” any subsequent call to it will fail at the OS level because no compiled class can be found for the new hash, permanently freezing any funds held by the contract.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(...);
    ...
}
```

The developer-inserted TODO at line 898 explicitly acknowledges the missing check. The function unconditionally writes the caller-supplied `class_hash` into the contract's `StateEntry` with no validation that the hash exists in `contract_class_changes` (the declared class registry).

The same defect exists in the deprecated path:

```cairo

    contract_address, syscall_ptr: ReplaceClass*
) {
    alloc_locals;
    let class_hash = syscall_ptr.class_hash;
    // No existence check here either.
    ...
    dict_update{dict_ptr=contract_state_changes}(...);

```

When a subsequent call is made to the bricked contract, `execute_entry_point` reads the class hash from state and calls `find_element` to locate the compiled class:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
```

If `class_hash` was never declared, `dict_read` returns `UNINITIALIZED_CLASS_HASH` (0), and `find_element` â€” which panics when the key is absent â€” causes proof generation to fail for the entire block.

---

### Impact Explanation

**Critical â€” Permanent freezing of funds.**

Any contract that holds funds (e.g., an account contract, a multisig, a vault) and calls `replace_class` with an undeclared class hash becomes permanently inaccessible. Because the OS has no mechanism to recover from an undeclared class hash written into state, all assets held by that contract are frozen forever. No future transaction can successfully execute against the contract, and no revert mechanism can undo the state change once the block is finalized.

---

### Likelihood Explanation

The likelihood is **medium-high**:

1. The `replace_class` syscall is a standard, publicly documented StarkNet syscall callable by any deployed contract.
2. The OS performs no validation of the supplied class hash â€” the TODO comment at line 898 confirms this is a known, unimplemented guard.
3. A buggy or malicious contract can trivially invoke `replace_class(0)` or `replace_class(<arbitrary_felt>)` and the OS will commit the change to state.
4. No privileged role is required; any unprivileged contract deployer or transaction sender can trigger this path.

---

### Recommendation

Before committing the new class hash in `execute_replace_class`, verify that the hash exists in `contract_class_changes` (i.e., it was previously declared via a `declare` transaction). Concretely, perform a `dict_read` on `contract_class_changes` with `key=class_hash` and assert the returned compiled class hash is non-zero (`UNINITIALIZED_CLASS_HASH` indicates an undeclared class). Apply the same fix to both the Cairo 1 path (`syscall_impls.cairo`) and the deprecated Cairo 0 path (`deprecated_execute_syscalls.cairo`).

---

### Proof of Concept

1. Deploy contract `Vault` holding user funds, implemented with Cairo 1.
2. `Vault` exposes a public function `brick_self()` that calls `replace_class(0x1337_undeclared_hash)`.
3. An attacker (or a bug) triggers `brick_self()`.
4. The OS processes the `replace_class` syscall via `execute_replace_class` in `syscall_impls.cairo` lines 877â€“916. No existence check is performed; `dict_update` writes `class_hash=0x1337_undeclared_hash` into `Vault`'s `StateEntry`.
5. In the same or any subsequent block, any call to `Vault` reaches `execute_entry_point` (`execute_entry_point.cairo` lines 154â€“166). `dict_read` on `contract_class_changes` for key `0x1337_undeclared_hash` returns 0. `find_element` cannot locate a `CompiledClassFact` with hash 0 and panics, making the block unprovable â€” or, if the sequencer correctly reverts the inner call, `Vault` is permanently bricked and all funds are frozen.

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
