### Title
`ProofOfInclusion::valid()` Trivially Returns `true` for Empty-Layers Proof, Enabling Forged DataLayer Inclusion Proofs — (`File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` contains a tautological self-comparison when `layers` is empty: `existing_hash` is initialized to `self.node_hash`, the loop is skipped, and `root_hash()` also returns `self.node_hash` in the empty-layers branch — so the final check `existing_hash == self.root_hash()` reduces to `self.node_hash == self.node_hash`, which is unconditionally `true`. An attacker can craft a `ProofOfInclusion` with any arbitrary `node_hash` and an empty `layers` vector, and `valid()` will accept it.

### Finding Description

`ProofOfInclusion::valid()` is implemented as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()
}
``` [1](#0-0) 

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash   // ← returns node_hash when layers is empty
    }
}
``` [2](#0-1) 

When `self.layers` is empty:
1. `existing_hash` is set to `self.node_hash`.
2. The `for` loop body never executes.
3. `self.root_hash()` returns `self.node_hash` (the `else` branch).
4. The final check becomes `self.node_hash == self.node_hash` → always `true`.

This is structurally identical to the reported bug: a value (`existing_hash`) that should be updated through a loop is compared against a target that degenerates to the same initial value when the loop is skipped, making the validation a no-op.

The `ProofOfInclusion` struct is `Streamable` (deserializable from bytes) and its `valid()` / `root_hash()` methods are exposed directly via Python bindings: [3](#0-2) 

An attacker can construct a serialized `ProofOfInclusion` with:
- `node_hash` = SHA-256 hash of any key-value pair they wish to falsely prove is in the tree
- `layers` = `[]` (empty vector)

`valid()` returns `true`, and `root_hash()` returns the attacker-chosen `node_hash`. Any caller that relies solely on `proof.valid()` — without separately asserting `proof.root_hash() == known_tree_root` — will accept the forged proof.

### Impact Explanation

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."**

A forged `ProofOfInclusion` with empty `layers` passes `valid()` for any `node_hash`. Any DataLayer consumer (Python or Rust) that calls `proof.valid()` as its sole gate — a natural and expected usage given the function's name and self-contained signature — will accept fabricated membership claims. This lets an untrusted party prove that arbitrary key-value data exists in a DataLayer tree when it does not.

### Likelihood Explanation

- The `ProofOfInclusion` struct is `Streamable` and fully deserializable from untrusted bytes.
- The Python bindings expose `valid()` as a standalone method with no required root-hash argument, making it the natural single call for callers.
- The empty-layers case is reachable by any party who can submit a serialized proof; no privileged access is required.
- The legitimate single-element-tree case (where empty layers is correct) makes the bug non-obvious: the code looks intentional.

### Recommendation

`valid()` must be anchored to an externally supplied, trusted root hash rather than computing the root from the proof itself. Change the signature to:

```rust
pub fn valid(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root
}
```

This eliminates the tautological self-comparison and forces callers to supply the known tree root, making forgery impossible regardless of `layers` length.

### Proof of Concept

```rust
use chia_datalayer::{Hash, ProofOfInclusion, Side};

// Attacker wants to "prove" that some arbitrary hash is in the tree.
let fake_node_hash: Hash = [0xde; 32]; // any hash the attacker chooses

let forged_proof = ProofOfInclusion {
    node_hash: fake_node_hash,
    layers: vec![],  // empty — loop is skipped
};

// valid() returns true unconditionally:
// existing_hash = fake_node_hash
// loop skipped
// root_hash() returns fake_node_hash (else branch)
// fake_node_hash == fake_node_hash → true
assert!(forged_proof.valid());          // passes — forged proof accepted
assert_eq!(forged_proof.root_hash(), fake_node_hash); // "root" is attacker-controlled
```

Any DataLayer verifier calling only `proof.valid()` will accept this forged proof for an arbitrary key.
