### Title
P2P Sync Accepts Sierra Body Whose Hash Mismatches State-Diff Class Hash, Permanently Corrupting Class Manager Storage — (`crates/apollo_p2p_sync/src/client/class.rs`)

---

### Summary

The P2P sync client's `write_to_storage` for `(DeclaredClasses, DeprecatedDeclaredClasses, BlockNumber)` calls `class_manager_client.add_class(class.clone())` and **silently discards** the returned `ClassHashes`. The class manager internally recomputes the class hash from the Sierra body and stores the class under that computed hash Z. If a malicious peer sends a Sierra body that hashes to Z while claiming it belongs to state-diff key X (Z ≠ X), the class is permanently stored under Z, not X. The class manager block marker is then advanced past block N, so the node never re-syncs. Any subsequent execution that looks up class X receives `None` from the class manager.

---

### Finding Description

**Step 1 — Peer-controlled input passes the only guard.**

In `parse_data_for_block`, the peer sends `(api_contract_class, class_hash)` pairs. The sole membership check is:

```rust

``` [1](#0-0) 

This confirms that the peer-supplied `class_hash` (X) is a key in the committed state diff. It does **not** verify that the Sierra body hashes to X. The body is stored in `declared_classes_result` under the peer-supplied X: [2](#0-1) 

**Step 2 — `write_to_storage` discards the authoritative hash returned by the class manager.**

```

The `Ok(ClassHashes)` arm is never inspected. The TODO at line 39 explicitly acknowledges the missing check: [3](#0-2) 

**Step 3 — The class manager computes and stores under the actual hash Z.**

`ClassManager::add_class` derives the storage key entirely from the body:

```rust
let class_hash = sierra_class.calculate_class_hash();   // = Z
...
self.classes.set_class(class_hash, class, ...);          // stored under Z
``` [4](#0-3) 

`calculate_class_hash` is the canonical Starknet Poseidon Sierra class hash: [5](#0-4) 

**Step 4 — The class manager marker is advanced, making the corruption permanent.**

```rust
storage_writer
    .begin_rw_txn()?
    .update_class_manager_block_marker(&self.2.unchecked_next())?
    .commit()?;
``` [6](#0-5) 

After this commit, the sync loop will never revisit block N's classes. `get_sierra(X)` returns `None`; `get_sierra(Z)` returns `Some(body)`.

**Contrast with central sync — the guard that p2p sync is missing.**

The central sync path does perform the check that p2p sync omits:

```rust
let class_hash = class_manager_client.add_class(class.clone()).await?.class_hash;
if class_hash != *expected_class_hash {
    panic!("Class hash mismatch. Expected: {expected_class_hash}, got: {class_hash}.");
}
``` [7](#0-6) 

The p2p sync path has no equivalent guard.

---

### Impact Explanation

After the attack, `class_manager.get_sierra(X)` and `class_manager.get_executable(X)` both return `None`. The blockifier uses the class manager as the authoritative source for compiled classes. Any transaction that deploys a contract of class X, or calls an already-deployed contract whose class is X, will fail with a class-not-found error. This constitutes **wrong compiled class / contract code served to the blockifier** for every block that references class X, for the lifetime of the node (until manual intervention or a full re-sync from scratch).

---

### Likelihood Explanation

The attacker must be the sole peer responding to the `ClassQuery` for block N at the moment the p2p sync client issues it. In a sparse or adversarially partitioned p2p network this is achievable. The state diff for block N is already committed to storage before the class query is issued (the `BLOCK_NUMBER_LIMIT: BlockNumberLimit::StateDiffMarker` constraint), so the attacker knows exactly which class hashes are valid keys to claim.

---

### Recommendation

In `write_to_storage`, inspect the `ClassHashes` returned by `add_class` and compare `returned.class_hash` against the state-diff key. On mismatch, treat the peer as bad (disconnect / blacklist) and do **not** advance the class manager block marker. This mirrors the guard already present in the central sync path at `apollo_central_sync/src/lib.rs:517`. [8](#0-7)

### Proof of Concept

```
state_diff for block N: { class_hash_to_compiled_class_hash: { X -> compiled_X } }

Attacker sends: (sierra_body_B, class_hash=X)
  where Poseidon(B) = Z, Z ≠ X

parse_data_for_block:
  declared_classes.get(&X).is_some() == true  → accepted
  declared_classes_result.insert(X, B)

write_to_storage:
  class_manager_client.add_class(B).await
    → ClassManager::add_class computes hash(B) = Z
    → set_class(Z, B, ...)
    → returns ClassHashes { class_hash: Z, ... }
  return value discarded

  update_class_manager_block_marker(N+1) committed

Post-condition:
  class_manager.get_sierra(X) == None   ← execution fails for class X
  class_manager.get_sierra(Z) == Some(B) ← stored under wrong key
```

    pub async fn add_class(&mut self, class: RawClass) -> ClassManagerResult<ClassHashes> {
        let sierra_class = SierraContractClass::try_from(&class)?;
        let class_hash = sierra_class.calculate_class_hash();
        if let Ok(Some(executable_class_hash_v2)) =
            self.classes.get_executable_class_hash_v2(class_hash)
        {
            // Class already exists.
            return Ok(ClassHashes { class_hash, executable_class_hash_v2 });
        }

        let compilation_start_time = Instant::now();
        let (raw_executable_class, executable_class_hash_v2) =
            self.compiler.compile(class.clone()).await.map_err(|err| match err {
                SierraCompilerClientError::SierraCompilerError(error) => {
                    ClassManagerError::SierraCompiler { class_hash, error }
                }
                SierraCompilerClientError::ClientError(error) => {
                    ClassManagerError::Client(error.to_string())
                }
            })?;
        debug!(
            %class_hash,
            compiled_class_hash = %executable_class_hash_v2,
            compilation_elapsed_ms = compilation_start_time.elapsed().as_millis(),
            class_size_bytes =
                class.size().map_or("Failed to get class size".to_owned(), |size| size.to_string()),
            casm_size_bytes =
                raw_executable_class.size().map_or("Failed to get casm size".to_owned(), |size| size.to_string()),
            "Finished compiling class."
        );

        self.validate_class_length(&raw_executable_class)?;
        Self::validate_class_version(&sierra_class)?;
        self.classes.set_class(
            class_hash,
            class,
            executable_class_hash_v2,
            raw_executable_class,
        )?;
