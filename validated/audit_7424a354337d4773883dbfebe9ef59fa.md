### Title
EIP-712 Type Schema Silently Omits Nested Array Fields, Allowing Signature Bypass — (`ethereum/eip712/types.go`)

---

### Summary

`recursivelyAddTypesToRoot` silently drops any message field whose value is a nested array (array-of-arrays). Because the field is absent from the EIP-712 type schema, the typed-data hash does not commit to it. An attacker can therefore submit a transaction whose nested-array field differs from what the signer authorized, and signature verification will still pass.

---

### Finding Description

In `recursivelyAddTypesToRoot`, when a field is an array the code unwraps its first element and calls `getEthTypeForJSON` on that element: [1](#0-0) 

For a nested array (e.g. `routes: [[addr1, addr2], [addr3]]`), `fieldAsArray[0]` is itself an array. `getEthTypeForJSON` returns `""` for any `gjson.JSON` value (arrays and objects alike): [2](#0-1) 

Back in the loop, `ethType == ""` skips the primitive branch, and `field.IsObject()` is `false` for an array, so the object-recursion branch is also skipped. The field is **silently dropped** — no error is returned, and nothing is appended to `typesToAdd`: [3](#0-2) 

The comment at line 213 acknowledges nested arrays are unsupported but treats omission as acceptable rather than returning an error. This violates the invariant that every message field must appear in the EIP-712 type schema.

---

### Impact Explanation

The EIP-712 hash is computed over `TypedData.Types` and `TypedData.Message`. Fields absent from `Types` are not encoded into the hash. Both verification paths re-derive the hash from the submitted transaction:

**Legacy path** — `VerifySignature` in `ante/cosmos/eip712.go` calls `LegacyWrapTxToTypedData` then `apitypes.TypedDataAndHash`: [4](#0-3) 

**Non-legacy path** — `verifySignatureAsEIP712` in `crypto/ethsecp256k1/ethsecp256k1.go` calls `GetEIP712BytesForMsg` which invokes the same `WrapTxToTypedData` → `recursivelyAddTypesToRoot` chain: [5](#0-4) 

Because both signing and verification use the same buggy schema builder, the hash is identical whether `routes` is `[[addr1]]` or `[[malicious_addr]]`. The Cosmos-SDK signature field is intentionally empty for EIP-712 transactions (the only real signature is `extOpt.FeePayerSig`): [6](#0-5) 

There is therefore no secondary check that would catch the field mutation.

---

### Likelihood Explanation

The bug is reachable by any unprivileged user who submits a Cosmos SDK message containing a nested array field through the standard EIP-712 transaction path (`ExtensionOptionsWeb3Tx` or `ethsecp256k1.VerifySignature`). Any chain built on Ethermint that registers a module with such a message type (e.g. a DEX with a `routes [][]string` field, a governance proposal with nested option arrays, etc.) is directly exploitable. No privileged access, key compromise, or validator collusion is required.

---

### Recommendation

In `recursivelyAddTypesToRoot`, after the array-unwrap block, add an explicit check: if the unwrapped element is itself an array (`field.IsArray()`), return an error immediately rather than falling through silently:

```go
if field.IsArray() {
    return "", errorsmod.Wrap(errortypes.ErrInvalidRequest,
        "nested array types are not supported in EIP-712")
}
```

This ensures the invariant — all message fields must be covered by the type schema — is enforced, and callers receive a clear error instead of a silently incomplete hash.

---

### Proof of Concept

1. Register a Cosmos SDK message type with a field `routes [][]string` (Amino type `"test/MsgNestedArray"`).
2. Construct an Amino sign-doc JSON:
   ```json
   {"msgs":[{"type":"test/MsgNestedArray","value":{"routes":[["addr1","addr2"],["addr3"]]}}], ...}
   ```
3. Call `WrapTxToTypedData` / `GetEIP712BytesForMsg` on this sign-doc. Observe that `TypedData.Types["TypeMsgNestedArray"]` contains no entry for `routes`.
4. Sign the resulting EIP-712 hash with an Ethereum key.
5. Mutate the sign-doc to `"routes":[["malicious"]]` and call `GetEIP712BytesForMsg` again. The hash is identical to step 3.
6. Submit the mutated transaction with the original signature. `VerifySignature` / `verifySignatureAsEIP712` accepts it, and the transaction executes with `routes = [["malicious"]]` — a value the signer never authorized. [7](#0-6)

func recursivelyAddTypesToRoot(
	typeMap apitypes.Types,
	rootType string,
	prefix string,
	payload gjson.Result,
) (string, error) {
	typesToAdd := []apitypes.Type{}

	// Must sort the JSON keys for deterministic type generation.
	sortedFieldNames, err := sortedJSONKeys(payload)
	if err != nil {
		return "", errorsmod.Wrap(err, "unable to sort object keys")
	}

	typeDef := typeDefForPrefix(prefix, rootType)

	for _, fieldName := range sortedFieldNames {
		field := payload.Get(fieldName)
		if !field.Exists() {
			continue
		}

		// Handle array type by unwrapping the first element.
		// Note that arrays with multiple types are not supported
		// using EIP-712, so we can ignore that case.
		isCollection := false
		if field.IsArray() {
			fieldAsArray := field.Array()

			if len(fieldAsArray) == 0 {
				// Arbitrarily add string[] type to handle empty arrays,
				// since we cannot access the underlying object.
				emptyArrayType := "string[]"
				typesToAdd = appendedTypesList(typesToAdd, fieldName, emptyArrayType)

				continue
			}

			field = fieldAsArray[0]
			isCollection = true
		}

		ethType := getEthTypeForJSON(field)

		// Handle JSON primitive types by adding the corresponding
		// EIP-712 type to the types schema.
		if ethType != "" {
			if isCollection {
				ethType += "[]"
			}
			typesToAdd = appendedTypesList(typesToAdd, fieldName, ethType)

			continue
		}

		// Handle object types recursively. Note that nested array types are not supported
		// in EIP-712, so we can exclude that case.
		if field.IsObject() {
			fieldPrefix := prefixForSubField(prefix, fieldName)

			fieldTypeDef, err := recursivelyAddTypesToRoot(typeMap, rootType, fieldPrefix, field)
			if err != nil {
				return "", err
			}

			fieldTypeDef = sanitizeTypedef(fieldTypeDef)
			if isCollection {
				fieldTypeDef += "[]"
			}

			typesToAdd = appendedTypesList(typesToAdd, fieldName, fieldTypeDef)

			continue
		}
	}

	return addTypesToRoot(typeMap, typeDef, typesToAdd)
}

func getEthTypeForJSON(json gjson.Result) string {
	switch json.Type {
	case gjson.True, gjson.False:
		return ethBool
	case gjson.Number:
		return ethInt64
	case gjson.String:
		return ethString
	case gjson.JSON:
		// Array or Object type
		return ""
	default:
		return ""
	}
}

func (pubKey PubKey) verifySignatureAsEIP712(msg, sig []byte) bool {
	eip712Bytes, err := eip712.GetEIP712BytesForMsg(msg)
	if err != nil {
		return false
	}

	if pubKey.verifySignatureECDSA(eip712Bytes, sig) {
		return true
	}

	// Try verifying the signature using the legacy EIP-712 encoding
	legacyEIP712Bytes, err := eip712.LegacyGetEIP712BytesForMsg(msg)
	if err != nil {
		return false
	}

	return pubKey.verifySignatureECDSA(legacyEIP712Bytes, sig)
