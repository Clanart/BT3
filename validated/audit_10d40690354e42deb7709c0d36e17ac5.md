Audit Report

## Title
Stranded ETH on the router is attributionless and can be stolen by any caller via `refundETH()` — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary
When a user calls `exactOutputSingle` or `exactOutput` with ETH as the input token (WETH) and sends `amountInMaximum` ETH, the pool callback consumes only the actual `amountIn ≤ amountInMaximum` via `pay()`, leaving the surplus `amountInMaximum − amountIn` stranded on the router with no per-caller attribution. Any unprivileged caller can then invoke `refundETH()` to drain the entire router ETH balance, stealing the victim's surplus ETH.

## Finding Description

**Root cause — `pay()` uses total router ETH balance without attribution:**

In `PeripheryPayments.sol` lines 73–84, when `token == WETH` and `payer != address(this)`, the function reads `nativeBalance = address(this).balance` — the router's aggregate ETH balance — and deposits exactly `value` (the actual swap `amountIn`) as WETH, forwarding it to the pool. Any ETH sent by the caller beyond `value` is silently left on the router with no record of who deposited it. [1](#0-0) 

**How ETH becomes stranded — `exactOutputSingle` with ETH:**

`exactOutputSingle` is `external payable` and accepts `msg.value` up to `amountInMaximum`. The pool callback fires `_justPayCallback` → `pay(tokenIn, msg.sender, pool, amountIn)` where `amountIn ≤ amountInMaximum`. Only `amountIn` ETH is deposited as WETH; the function returns without refunding the surplus `amountInMaximum − amountIn`. [2](#0-1) 

**Theft vector — `refundETH()` sends all ETH to `msg.sender` with no attribution check:**

`refundETH()` unconditionally transfers `address(this).balance` to `msg.sender`. Any caller — including an attacker who contributed zero ETH — receives the full router balance, including ETH stranded by a prior user. [3](#0-2) 

**Existing guards are insufficient:**

The `receive()` fallback reverts for non-WETH senders, but this does not prevent ETH from entering via `payable` function calls. The `amountInMaximum` check (line 145) only reverts if `amountIn > amountInMaximum`; it does not refund the difference. The `multicall` pattern (lines 39–44) can compose `exactOutputSingle` + `refundETH()` correctly, but this is not enforced — users calling `exactOutputSingle` directly are silently exposed. [4](#0-3) [5](#0-4) 

## Impact Explanation

Direct, unconditional loss of user principal. A user who calls `exactOutputSingle{value: amountInMaximum}` without composing a `multicall` ending in `refundETH()` loses `amountInMaximum − amountIn` ETH to the first caller who invokes `refundETH()`. There is no recovery path. This meets the Sherlock High/Medium threshold for direct loss of user funds via an unprivileged call.

## Likelihood Explanation

`exactOutputSingle` and `exactOutput` are the natural entry points for exact-output ETH swaps. The pattern of sending `amountInMaximum` and expecting the router to refund the difference is standard across Uniswap-style routers, but here the refund is not automatic. The attack requires no privilege, no special token, no coordination, and no front-running — simply observing `address(router).balance > 0` and calling `refundETH()`. It is repeatable on every unguarded exact-output ETH swap.

## Recommendation

1. **Auto-refund excess ETH at the end of `exactOutputSingle` / `exactOutput`:** After the pool callback settles `amountIn`, compute `address(this).balance` and transfer any remainder back to `msg.sender` before returning.
2. **Track per-caller ETH in transient storage:** Record `msg.value` keyed to `msg.sender` at entry of each `payable` swap function; restrict `refundETH()` to return only the caller's own recorded contribution.
3. **Document the `multicall` + `refundETH()` requirement** in NatSpec on every `external payable` swap function so integrators are not silently exposed.

## Proof of Concept

```solidity
function test_strandedEthStolenViaRefundETH() public {
    uint128 amountOut = 1_000;
    uint256 amountInMaximum = 2 ether;

    // Victim calls exactOutputSingle directly (no multicall, no refundETH).
    vm.prank(swapper);
    uint256 amountIn = router.exactOutputSingle{value: amountInMaximum}(
        IMetricOmmSimpleRouter.ExactOutputSingleParams({
            pool: address(pool),
            tokenIn: address(weth),
            tokenOut: address(token1),
            zeroForOne: true,
            amountOut: amountOut,
            amountInMaximum: uint128(amountInMaximum),
            recipient: swapper,
            deadline: block.timestamp + 1,
            priceLimitX64: 0,
            extensionData: ""
        })
    );

    // Surplus ETH is stranded on the router.
    uint256 stranded = amountInMaximum - amountIn;
    assertGt(stranded, 0);
    assertEq(address(router).balance, stranded);

    // Attacker steals it with a single unprivileged call.
    vm.prank(attacker);
    router.refundETH();

    assertEq(attacker.balance, attackerEthBefore + stranded);
    assertEq(address(router).balance, 0);
}
```
