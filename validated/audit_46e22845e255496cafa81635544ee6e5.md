### Title
Missing Declared-Class Existence Check in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS Cairo program does not verify that the new class hash supplied by a contract is actually declared (i.e., exists in the class tree or in the current block's `contract_class_changes` dict). This is structurally identical to the Argo `mark_vault` bug: a state-transition function that should enforce a precondition on the target state omits the check entirely. An unprivileged contract deployer can exploit this to permanently freeze any funds held by a contract by replacing its class with an arbitrary, undeclared class hash, rendering the contract permanently uncallable.

---

### Finding Description

`execute_replace_class` in `syscall_impls.cairo` processes the `replace_class` syscall. It reads the caller's current `StateEntry`, constructs a new one with the attacker-supplied `class_hash`, and writes it back to `contract_state_changes` — all without verifying that `class_hash` corresponds to a declared class.

The missing check is explicitly acknowledged by a TODO comment in the code itself:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The OS proof accepts this state transition unconditionally. Once the block is proven and committed to L1, the contract's class hash in the state tree is permanently set to the undeclared value. Because the class does not exist in the class tree, every subsequent call to the contract will fail at class resolution, making the contract permanently uncallable and any funds it holds permanently frozen.

By contrast, class declaration in `execute_declare_transaction` correctly enforces `prev_value=0` to guarantee a class is declared at most once: [2](#0-1) 

And contract deployment in `deploy_contract` correctly asserts `state_entry.class_hash = UNINITIALIZED_CLASS_HASH` before proceeding: [3](#0-2) 

`execute_replace_class` is the only state-mutating OS function that performs no analogous existence check on its target class.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once the OS proof is accepted and the state root is committed to L1, the contract's class hash in the Merkle state tree is permanently set to an undeclared value. The class entry does not exist in the class tree, so any future call to the contract — including calls to withdraw or transfer funds — will fail at class resolution. There is no recovery path: the state is committed on-chain and the class can never be retroactively declared to match the already-committed hash (class declaration enforces `prev_value=0`, so a new declaration would produce a different leaf). All funds held by the contract are permanently inaccessible.

---

### Likelihood Explanation

The attack is directly reachable by any unprivileged contract deployer:

1. Deploy a contract whose constructor or any callable entry point issues a `replace_class` syscall with an arbitrary felt value as the class hash (e.g., `1`).
2. Induce users to deposit funds into the contract (e.g., by presenting it as a vault or token contract).
3. Call the entry point that triggers `replace_class`.
4. The OS processes the syscall without checking class existence, includes the state update in the proven block, and commits it to L1.
5. The contract is now permanently uncallable; all deposited funds are frozen.

No privileged access, leaked keys, or external dependency compromise is required. The attacker only needs the ability to deploy a contract and submit transactions — both are public protocol operations.

---

### Recommendation

Before writing the new `StateEntry`, verify that `class_hash` is declared. This requires checking either:

1. That `class_hash` appears as a key in `contract_class_changes` with a non-zero `new_value` (declared in the current block), **or**
2. That a `dict_read` on `contract_class_changes` for `class_hash` returns a non-zero compiled class hash (previously declared and present in the class tree).

The fix is analogous to the Argo patch — assert the required precondition before mutating state:

```cairo
// Assert that the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the pattern already used in `execute_declare_transaction` where `assert_not_zero(compiled_class_hash)` is called before the `dict_update`. [4](#0-3) 

---

### Proof of Concept

1. **Deploy attacker contract** with the following logic in its `__execute__` entry point:
   ```
   replace_class(class_hash=0xdeadbeef)  // 0xdeadbeef is never declared
   ```
2. **Induce deposits**: users send funds to the contract (e.g., via ERC-20 `transfer`).
3. **Trigger**: call the contract's `__execute__` entry point.
4. **OS processing**: `execute_replace_class` in `syscall_impls.cairo` reads the current `StateEntry`, constructs a new one with `class_hash=0xdeadbeef`, and calls `dict_update` — no existence check is performed.
5. **Block proven and committed**: the state root on L1 now reflects `contract.class_hash = 0xdeadbeef`.
6. **Funds frozen**: any subsequent call to the contract fails because `0xdeadbeef` does not exist in the class tree. The funds are permanently inaccessible. [5](#0-4)

    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
