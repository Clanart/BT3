### Title
DataLayer `ProofOfInclusion::valid()` Tautological Root-Hash Check Accepts Forged Inclusion Proofs - (File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs)

### Summary
`ProofOfInclusion::valid()` compares the final computed hash against `self.root_hash()`, which returns the `combined_hash` field stored inside the proof itself. Because the loop already enforces `calculated_hash == layer.combined_hash` before advancing, the final comparison is a tautology — it always passes for any internally self-consistent proof, regardless of whether the proof corresponds to the actual DataLayer tree root. An attacker can craft a `ProofOfInclusion` with arbitrary leaf data and a fake root, serialize it via the `Streamable` interface, and `valid()` will return `true`.

### Finding Description

`ProofOfInclusion::valid()` is implemented as follows:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {   // ← enforces consistency
            return false;
        }

        existing_hash = calculated_hash;              // ← existing_hash := combined_hash
    }

    existing_hash == self.root_hash()                 // ← tautology
}
``` [1](#0-0) 

`root_hash()` returns the `combined_hash` of the last layer:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← same field the loop already verified
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

After the last loop iteration, `existing_hash` has been set to `calculated_hash`, and the loop guard already confirmed `calculated_hash == last.combined_hash`. Therefore `existing_hash == self.root_hash()` reduces to `last.combined_hash == last.combined_hash`, which is unconditionally `true`. The function never compares against an externally-supplied, trusted root hash.

This is the direct analog of the reported Solidity bug: instead of comparing against the authoritative external value (`maxAllocation` / the real tree root), the code compares against the attacker-controlled value already embedded in the proof (`userMaxAllc` / `layer.combined_hash`).

`ProofOfInclusion` derives `Streamable`, so it can be deserialized from arbitrary bytes. The Python binding exposes both `from_bytes` and `valid()` directly: [3](#0-2) [4](#0-3) 

### Impact Explanation

Any DataLayer consumer that receives a `ProofOfInclusion` from an untrusted peer, deserializes it, and calls `proof.valid()` to decide whether to trust the claimed key→value mapping will accept a completely fabricated proof. The attacker constructs a chain where each `combined_hash` is the correct hash of the previous hash and a chosen `other_hash`, making the chain internally consistent, but the final `combined_hash` (the "root") is arbitrary and does not correspond to any real DataLayer tree. `valid()` returns `true`.

This lets untrusted input prove invalid state — a forged inclusion proof for a key that is not in the tree, or for a key mapped to a different value — satisfying the allowed High impact: **DataLayer Merkle proof logic accepts forged inclusion/exclusion, lets untrusted input prove invalid state**.

### Likelihood Explanation

The `ProofOfInclusion` type is a first-class Streamable/Python-exposed object intended to be transmitted between DataLayer peers. Any node that validates received proofs using only `proof.valid()` — without separately comparing `proof.root_hash()` against a locally-known trusted root — is exploitable by any peer that sends a crafted proof. The attack requires no special privileges, only the ability to send a serialized `ProofOfInclusion`.

### Recommendation

`valid()` must accept an externally-trusted root hash parameter and compare against it, not against the proof's own stored `combined_hash`:

```rust
pub fn valid(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against externally-supplied root
}
```

Alternatively, remove `root_hash()` from the public API and require callers to always supply the trusted root when calling `valid()`.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
import hashlib

# Build a fake leaf hash and a fake "other" hash
fake_leaf  = bytes([0xAA] * 32)
fake_other = bytes([0xBB] * 32)

# Compute a combined_hash that is internally consistent with the layer
# (matches what calculate_internal_hash would produce for these inputs)
# The exact hash function doesn't matter for the PoC — any consistent
# combined_hash will satisfy the loop guard.
import chia_rs.datalayer as dl
# Construct a single-layer proof with arbitrary data
layer = ProofOfInclusionLayer(
    other_hash_side=0,          # left
    other_hash=fake_other,
    combined_hash=<computed_hash_of_fake_leaf_and_fake_other>,
)
proof = ProofOfInclusion(node_hash=fake_leaf, layers=[layer])

# valid() returns True even though this proof was never generated
# from any real MerkleBlob and the "root" is attacker-chosen.
assert proof.valid()   # passes — tautology confirmed
```

The `valid()` call succeeds because the loop verifies `calculated_hash == layer.combined_hash` (which the attacker controls), then the final check compares `existing_hash` (= `calculated_hash`) against `self.root_hash()` (= `layer.combined_hash`) — the same value. [1](#0-0) [5](#0-4) [6](#0-5)

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
