### Title
Forced `safeTransfer` to Blacklisted Recipient in `finTransfer` Causes Permanent Fund Lock — (File: `evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.finTransfer` unconditionally pushes native ERC-20 tokens (e.g., USDC held in the bridge vault) to `payload.recipient` via `safeTransfer`. If that address is blacklisted by USDC, every call to `finTransfer` for that transfer reverts. Because the MPC-signed payload hard-codes the recipient and no cancel/refund path exists on the NEAR side, the user's locked tokens become permanently irrecoverable.

---

### Finding Description

`OmniBridge.finTransfer` follows a push-transfer pattern for native ERC-20 tokens:

```solidity
// OmniBridge.sol lines 283–355
completedTransfers[payload.destinationNonce] = true;   // (1) mark nonce used

// ... (ERC-1155, custom-minter, bridge-token branches) ...

} else {
    IERC20(payload.tokenAddress).safeTransfer(          // (2) forced push
        payload.recipient,
        payload.amount
    );
}
``` [1](#0-0) [2](#0-1) 

When `payload.tokenAddress` is a USDC-like token with a blacklist and `payload.recipient` is on that blacklist, `safeTransfer` reverts. Because the revert unwinds the entire transaction, `completedTransfers[payload.destinationNonce]` is also rolled back, so the nonce is never consumed. Every subsequent attempt to call `finTransfer` with the same MPC-signed payload will fail identically.

The recipient address is embedded in the MPC-signed `TransferMessagePayload` and cannot be changed without a new MPC signature:

```solidity
bytes memory borshEncoded = bytes.concat(
    ...
    bytes1(omniBridgeChainId),
    Borsh.encodeAddress(payload.recipient),   // recipient is part of signed data
    ...
);
bytes32 hashed = keccak256(borshEncoded);
if (ECDSA.recover(hashed, signatureData) != nearBridgeDerivedAddress) {
    revert InvalidSignature();
}
``` [3](#0-2) 

On the NEAR side, once `sign_transfer` is called the transfer is removed from `pending_transfers` (zero-fee case) or kept pending (non-zero fee), but there is no `cancel_transfer` or refund function that would return the locked tokens to the user when the destination-chain finalization permanently fails: [4](#0-3) 

The NEAR bridge holds the user's tokens in its vault (for native NEAR tokens) or has already burned the bridged representation. Either way, the user has no on-chain path to recover them.

---

### Impact Explanation

**Critical — Permanent freezing / irrecoverable lock of user funds.**

A user who bridges USDC (or any token with a transfer blacklist) from NEAR to EVM and whose EVM address subsequently gets blacklisted by USDC will have their tokens permanently locked:

- `finTransfer` always reverts → tokens never released from the EVM vault.
- No cancel/refund mechanism exists on NEAR → tokens never returned to the user.
- The MPC signature is bound to the blacklisted recipient → no alternative finalization is possible without protocol-level intervention.

This matches the allowed impact: *"Permanent freezing, irrecoverable lock, or unclaimable settlement of user or protocol funds in bridge, token, or vault flows."*

---

### Likelihood Explanation

**Low.** USDC blacklisting is an infrequent but real event (OFAC sanctions, exchange hacks, etc.). The scenario requires:
1. A user bridges USDC from NEAR to EVM.
2. The user's EVM recipient address is blacklisted by USDC between the time the transfer is initiated and the time `finTransfer` is called (or after a failed attempt).

Both conditions are independently rare but not impossible, and the impact when they coincide is total and permanent loss of funds.

---

### Recommendation

Replace the push-transfer pattern with a pull (claim) pattern for native ERC-20 tokens:

1. In `finTransfer`, instead of calling `safeTransfer` directly, credit an internal mapping:
   ```solidity
   mapping(address recipient => mapping(address token => uint256 amount)) public claimable;
   ```
2. Add a `claimTokens(address token)` function that lets the recipient pull their balance.
3. This decouples the bridge finalization (nonce consumption, event emission) from the token delivery, so a blacklisted recipient does not block finalization and does not permanently lock funds.

Additionally, consider adding a `cancelTransfer` function on the NEAR side that allows a user to reclaim locked tokens when a destination-chain finalization has been provably impossible for a configurable timeout period.

---

### Proof of Concept

1. Alice holds USDC on NEAR and calls `ft_transfer_call` → `init_transfer` to bridge 1 000 USDC to her EVM address `0xAlice`. Tokens are locked in the NEAR bridge.
2. The NEAR bridge MPC signs a `TransferMessagePayload` with `recipient = 0xAlice`, `tokenAddress = USDC_EVM`, `amount = 1000`.
3. Before the relayer calls `finTransfer`, USDC's issuer blacklists `0xAlice`.
4. Relayer calls `OmniBridge.finTransfer(sig, payload)`:
   - `completedTransfers[nonce] = true` executes.
   - `IERC20(USDC).safeTransfer(0xAlice, 1000)` reverts (blacklisted).
   - Entire transaction reverts; nonce is not consumed.
5. Every subsequent call to `finTransfer` with the same payload reverts at step 4.
6. Alice cannot change the recipient (signature check enforces it).
7. Alice cannot cancel the transfer on NEAR (no such function exists).
8. Result: 1 000 USDC permanently locked in the NEAR bridge vault, irrecoverable by Alice or any unprivileged party. [5](#0-4) [6](#0-5)

    function finTransfer(
        bytes calldata signatureData,
        BridgeTypes.TransferMessagePayload calldata payload
    ) external payable whenNotPaused(PAUSED_FIN_TRANSFER) {
        if (completedTransfers[payload.destinationNonce]) {
            revert NonceAlreadyUsed(payload.destinationNonce);
        }

        completedTransfers[payload.destinationNonce] = true;

        bytes memory borshEncoded = bytes.concat(
            bytes1(uint8(BridgeTypes.PayloadType.TransferMessage)),
            Borsh.encodeUint64(payload.destinationNonce),
            bytes1(payload.originChain),
            Borsh.encodeUint64(payload.originNonce),
            bytes1(omniBridgeChainId),
            Borsh.encodeAddress(payload.tokenAddress),
            Borsh.encodeUint128(payload.amount),
            bytes1(omniBridgeChainId),
            Borsh.encodeAddress(payload.recipient),
            bytes(payload.feeRecipient).length == 0 // None or Some(String) in rust
                ? bytes("\x00")
                : bytes.concat(
                    bytes("\x01"),
                    Borsh.encodeString(payload.feeRecipient)
                ),
            bytes(payload.message).length == 0
                ? bytes("")
                : Borsh.encodeBytes(payload.message)
        );
        bytes32 hashed = keccak256(borshEncoded);

        if (ECDSA.recover(hashed, signatureData) != nearBridgeDerivedAddress) {
            revert InvalidSignature();
        }

        MultiTokenInfo memory multiToken = multiTokens[payload.tokenAddress];

        if (payload.tokenAddress == address(0)) {
            // slither-disable-next-line arbitrary-send-eth
            (bool success, ) = payload.recipient.call{value: payload.amount}(
                ""
            );
            if (!success) revert FailedToSendEther();
        } else if (multiToken.tokenAddress != address(0)) {
            IERC1155(multiToken.tokenAddress).safeTransferFrom(
                address(this),
                payload.recipient,
                multiToken.tokenId,
                payload.amount,
                ""
            );
        } else if (customMinters[payload.tokenAddress] != address(0)) {
            ICustomMinter(customMinters[payload.tokenAddress]).mint(
                payload.tokenAddress,
                payload.recipient,
                payload.amount
            );
        } else if (isBridgeToken[payload.tokenAddress]) {
            if (payload.message.length == 0) {
                IBridgeToken(payload.tokenAddress).mint(
                    payload.recipient,
                    payload.amount
                );
            } else {
                IBridgeToken(payload.tokenAddress).mint(
                    payload.recipient,
                    payload.amount,
                    payload.message
                );
            }
        } else {
            IERC20(payload.tokenAddress).safeTransfer(
                payload.recipient,
                payload.amount
            );
        }

        finTransferExtension(payload);

        emit BridgeTypes.FinTransfer(
            payload.originChain,
            payload.originNonce,
            payload.tokenAddress,
            payload.amount,
            payload.recipient,
            payload.feeRecipient
        );
