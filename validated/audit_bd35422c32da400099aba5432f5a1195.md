### Title
Reverted L1 Handler Transactions Skip `consume_l1_to_l2_message`, Leaving L1→L2 Messages Unconsumed in OS Output — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

In `execute_l1_handler_transaction`, when the sequencer marks a transaction as reverted, the function returns early and never calls `consume_l1_to_l2_message`. This means the L1→L2 message is never written into `outputs.messages_to_l2`, so it is absent from the OS output. The L1 Starknet core contract, which relies on the OS output to consume message hashes, will not consume the message. The message remains permanently pending on L1, causing any funds locked in the message (e.g., bridge deposits) to be frozen, or enabling the message to be replayed in a future block.

---

### Finding Description

`execute_l1_handler_transaction` begins by reading the sequencer-provided `is_reverted` hint. If the transaction is reverted, it immediately returns:

```cairo
if (is_reverted != FALSE) {
    %{ EndTx %}
    return ();
}
```

The entire remainder of the function — including `consume_l1_to_l2_message` — is skipped. [1](#0-0) 

In the non-reverted path, `consume_l1_to_l2_message` is called at line 444, which writes a `MessageToL2Header` entry into `outputs.messages_to_l2`: [2](#0-1) 

`consume_l1_to_l2_message` advances the `outputs.messages_to_l2` pointer, recording the message in the OS carried outputs: [3](#0-2) 

These carried outputs are serialized into the final OS output via `serialize_messages`, which relocates the `messages_to_l2` segment into the output segment for L1 verification: [4](#0-3) 

The OS output is then processed and serialized in `process_os_output` → `serialize_os_output`: [5](#0-4) 

**Contrast with all other transaction types.** For `INVOKE_FUNCTION`, `DEPLOY_ACCOUNT`, and `DECLARE`, even when `is_reverted == TRUE`, the execution is skipped but critical accounting calls (`charge_fee`) are still made unconditionally after the conditional branch: [6](#0-5) [7](#0-6) [8](#0-7) 

For L1 handlers, there is no equivalent unconditional call to `consume_l1_to_l2_message` outside the revert guard. The missing call is the direct analog of the H-01 pattern: a necessary state-recording function is called in one code path (non-reverted) but entirely absent in another (reverted).

---

### Impact Explanation

**Impact: Critical — Permanent freezing of funds.**

The L1 Starknet core contract consumes L1→L2 message hashes only when they appear in the proven OS output. If a reverted L1 handler does not emit the message into `outputs.messages_to_l2`, the message hash is absent from the OS output. The L1 contract therefore never marks the message as consumed.

Consequences:
1. **Permanent fund freeze**: For bridge-style L1→L2 messages (e.g., ETH or ERC-20 deposits), the funds are locked in the L1 bridge contract. The L2 minting never occurred (execution reverted), and the L1 message is never consumed, so the funds cannot be recovered through normal protocol flow.
2. **Potential replay**: If the L1 contract allows the same message to be re-queued (because it was never consumed), the sequencer could include it again in a future block, leading to double-execution of the L1 handler.

---

### Likelihood Explanation

**Likelihood: High.**

Any unprivileged user can trigger this path:
1. Deploy a contract on L2 whose `@l1_handler` function always reverts (e.g., panics unconditionally, or reverts when called with specific calldata).
2. Send an L1→L2 message to that contract from L1 (paying the required L1 fee).
3. The sequencer attempts to execute the L1 handler, which reverts.
4. The sequencer sets `is_reverted = TRUE` via the `%{ IsReverted %}` hint.
5. The OS skips `consume_l1_to_l2_message`.
6. The message is absent from the OS output; the L1 contract does not consume it.

This requires no privileged access, no leaked keys, and no operator collusion. Any L1 message sender is a valid attacker entry point.

---

### Recommendation

Move `consume_l1_to_l2_message` to execute **before** the revert check, or restructure the function so that message consumption is unconditional regardless of revert status — mirroring how `charge_fee` is called unconditionally for all other transaction types. The execution of the entry point should remain conditional on `is_reverted`, but the message recording must not be gated by it.

Concretely, the structure should be:

```cairo
func execute_l1_handler_transaction{...}(block_context: BlockContext*) {
    alloc_locals;
    %{ StartTx %}

    // ... build execution context, compute tx hash, fill TxInfo ...

    // Consume L1-to-L2 message UNCONDITIONALLY (regardless of revert status).
    consume_l1_to_l2_message(execution_context=tx_execution_context, nonce=nonce);

    local is_reverted;
    %{ IsReverted %}
    if (is_reverted != FALSE) {
        %{ EndTx %}
        return ();
    }

    // Execute entry point only for non-reverted transactions.
    let remaining_gas = L1_HANDLER_L2_GAS_MAX_AMOUNT;
    non_reverting_select_execute_entry_point_func{remaining_gas=remaining_gas}(...);

    %{ EndTx %}
    return ();
}
```

---

### Proof of Concept

1. Deploy an L2 contract with an `@l1_handler` that always panics.
2. From L1, call the Starknet core contract's `sendMessageToL2` targeting that contract, paying the required fee.
3. The sequencer picks up the message and attempts `execute_l1_handler_transaction`.
4. The `%{ IsReverted %}` hint is set to `TRUE` (execution failed).
5. The OS executes the branch at line 387–390, returning immediately. [9](#0-8) 
6. `consume_l1_to_l2_message` (line 444) is never reached. [10](#0-9) 
7. The `outputs.messages_to_l2` pointer is not advanced; the message is absent from the OS output serialized by `serialize_messages`. [11](#0-10) 
8. The L1 Starknet contract verifies the proof but finds no record of the message being consumed. The message hash remains live on L1. The funds paid with the message are permanently frozen.

```text
func consume_l1_to_l2_message{outputs: OsCarriedOutputs*}(
    execution_context: ExecutionContext*, nonce: felt
) {
    assert_not_zero(execution_context.calldata_size);
    // The payload is the calldata without the from_address argument (which is the first).
    let payload: felt* = execution_context.calldata + 1;
    tempvar payload_size = execution_context.calldata_size - 1;

    tempvar execution_info = execution_context.execution_info;

    // Write the given transaction to the output.
    assert [outputs.messages_to_l2] = MessageToL2Header(
        from_address=[execution_context.calldata],
        to_address=execution_info.contract_address,
        nonce=nonce,
        selector=execution_info.selector,
        payload_size=payload_size,
    );

    let message_payload = cast(outputs.messages_to_l2 + MessageToL2Header.SIZE, felt*);
    memcpy(dst=message_payload, src=payload, len=payload_size);

    let (outputs) = os_carried_outputs_new(
        messages_to_l1=outputs.messages_to_l1,
        messages_to_l2=outputs.messages_to_l2 + MessageToL2Header.SIZE +
        outputs.messages_to_l2.payload_size,
    );
    return ();
```

```text
func serialize_messages{output_ptr: felt*}(
    initial_carried_outputs: OsCarriedOutputs*, final_carried_outputs: OsCarriedOutputs*
) {
    let messages_to_l1_segment_size = (
        final_carried_outputs.messages_to_l1 - initial_carried_outputs.messages_to_l1
    );
    serialize_word(messages_to_l1_segment_size);

    // Relocate 'messages_to_l1_segment' to the correct place in the output segment.
    relocate_segment(src_ptr=initial_carried_outputs.messages_to_l1, dest_ptr=output_ptr);
    let output_ptr = cast(final_carried_outputs.messages_to_l1, felt*);

    let messages_to_l2_segment_size = (
        final_carried_outputs.messages_to_l2 - initial_carried_outputs.messages_to_l2
    );
    serialize_word(messages_to_l2_segment_size);

    // Relocate 'messages_to_l2_segment' to the correct place in the output segment.
    relocate_segment(src_ptr=initial_carried_outputs.messages_to_l2, dest_ptr=output_ptr);
    let output_ptr = cast(final_carried_outputs.messages_to_l2, felt*);

    return ();
```
