### Title
`ProofOfInclusion::valid()` Does Not Verify Against a Trusted Root Hash, Enabling Forged Inclusion Proofs — (`File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` only checks internal hash-chain self-consistency. It derives its "root" from the proof's own last layer (`self.root_hash()` returns `self.layers.last().combined_hash`), so the final equality check is trivially satisfied by any internally consistent proof. An attacker who can supply a `ProofOfInclusion` object — via Python bindings or Streamable deserialization — can forge a proof for any key/value pair in any fake tree and have `valid()` return `true`.

### Finding Description

`root_hash()` is defined as:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← derived from the proof itself
    } else {
        self.node_hash
    }
}
``` [1](#0-0) 

`valid()` then ends with:

```rust
existing_hash == self.root_hash()
``` [2](#0-1) 

After the loop, `existing_hash` equals the last `calculated_hash`, which the loop already verified equals `layer.combined_hash`. `self.root_hash()` returns that same `combined_hash`. The final comparison is therefore **always true** for any proof that passes the loop. No external trusted root is ever consulted.

The struct is fully constructible from untrusted input via the Python `__new__` constructor and via `from_bytes` / `from_bytes_unchecked` Streamable deserialization: [3](#0-2) 

`valid()` is the only verification method exposed on `ProofOfInclusion` — there is no `valid_for_root(root: Hash)` variant: [4](#0-3) 

### Impact Explanation

Any verifier that calls `proof.valid()` as its sole check accepts a proof for any key/value pair in any attacker-chosen fake tree root. This lets untrusted input prove invalid DataLayer state — a forged inclusion proof passes without the proof's root ever being compared to the actual committed tree root. This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."*

### Likelihood Explanation

`ProofOfInclusion` is a public, Streamable, Python-constructible type exposed through the `chia_rs.datalayer` submodule. The `valid()` method is the only verification entry point and its name implies completeness. Any downstream Python caller (e.g., chia-blockchain) that uses `proof.valid()` without separately asserting `proof.root_hash() == trusted_root` is vulnerable. The Python binding registration confirms the type is fully accessible to unprivileged callers: [5](#0-4) 

### Recommendation

Add a `valid_for_root(trusted_root: Hash) -> bool` method that accepts an externally-supplied trusted root and compares against it, and deprecate or remove the no-argument `valid()` from the public API. At minimum, rename `valid()` to `internally_consistent()` to prevent callers from treating it as a complete security check. The corrected check should be:

```rust
pub fn valid_for_root(&self, trusted_root: &Hash) -> bool {
    self.internally_consistent() && &self.root_hash() == trusted_root
}
```

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
import hashlib

# Craft a fake leaf hash
fake_leaf = bytes([0xAB] * 32)

# Craft a fake sibling hash
fake_sibling = bytes([0xCD] * 32)

# Compute the "combined" hash the same way calculate_internal_hash does:
# internal_hash = SHA256(b"\x02" + left + right)
combined = hashlib.sha256(b"\x02" + fake_sibling + fake_leaf).digest()

layer = ProofOfInclusionLayer(
    other_hash_side=0,   # Side::Left → sibling is on the left
    other_hash=fake_sibling,
    combined_hash=combined,
)

proof = ProofOfInclusion(node_hash=fake_leaf, layers=[layer])

# valid() returns True for a completely fabricated proof
assert proof.valid(), "Forged proof accepted!"
# proof.root_hash() == combined — an attacker-chosen fake root
```

`valid()` returns `True` for this entirely fabricated proof because it never compares against any real tree root. [2](#0-1)

    pub fn valid(&self) -> bool {
        let mut existing_hash = self.node_hash;

        for layer in &self.layers {
            let calculated_hash = crate::calculate_internal_hash(
                &existing_hash,
                layer.other_hash_side,
                &layer.other_hash,
            );

            if calculated_hash != layer.combined_hash {
                return false;
            }

            existing_hash = calculated_hash;
        }

        existing_hash == self.root_hash()
    }
