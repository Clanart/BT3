### Title
Missing Compiled Class Hash Validation During Declare Transaction Registration — (`execution/transaction_impls.cairo`)

### Summary

In `execute_declare_transaction`, the OS registers a `(class_hash → compiled_class_hash)` pair in the contract class tree without verifying that `compiled_class_hash` corresponds to any validated compiled class fact. Only the Sierra class hash is cryptographically verified; the compiled class hash is accepted as long as it is non-zero. This is the direct analog of the NFT bridge report's first concern: a linkage check is performed at usage time but not at registration time, allowing an invalid pair to be permanently committed to state.

### Finding Description

In `execute_declare_transaction` (`transaction_impls.cairo`, lines 693–828), the declare flow proceeds as follows:

1. `compiled_class_hash` is loaded from the hint `%{ DeclareTxFields %}` (line 715) — it is sequencer/user-supplied.
2. The transaction hash is computed and verified, binding the `compiled_class_hash` to the signed transaction (lines 725–732).
3. The **Sierra** class hash is cryptographically validated via `finalize_class_hash` (lines 738–743).
4. The only constraint placed on `compiled_class_hash` before it is written to state is `assert_not_zero(compiled_class_hash)` (line 816).
5. `dict_update` permanently commits `class_hash → compiled_class_hash` to `contract_class_changes` (lines 817–819). [1](#0-0) 

There is no assertion that `compiled_class_hash` is a member of the set of compiled class facts that were validated in the same block by `validate_compiled_class_facts` (`compiled_class.cairo`, lines 99–137). [2](#0-1) 

`validate_compiled_class_facts` validates the hashes of compiled classes that are *provided* in the block, but there is no cross-reference step that checks every newly declared class's `compiled_class_hash` against that validated set. The two registrations — Sierra class hash (validated) and compiled class hash (unvalidated) — are therefore decoupled, exactly mirroring the separate-layer registration mismatch described in the external report. [3](#0-2) 

### Impact Explanation

Once `dict_update` writes the pair to `contract_class_changes`, it is squashed and committed to the contract class Patricia tree by `compute_class_commitment` (`state/commitment.cairo`, lines 255–291) with no further validation of the compiled class hash value. [4](#0-3) 

The class commitment tree entry is permanent and immutable — there is no re-declaration or update path. Any contract subsequently deployed under this class hash will reference a `compiled_class_hash` for which no valid compiled class exists. Execution of any entry point on such a contract will fail irrecoverably. Funds deposited into such a contract are **permanently frozen** — matching the "Critical: Permanent freezing of funds" impact category.

### Likelihood Explanation

The `compiled_class_hash` field is part of the signed transaction body and is user-controlled. A malicious or misconfigured sequencer can include a declare transaction whose `compiled_class_hash` is an arbitrary non-zero felt that does not correspond to any real compiled class. Because the OS proof does not enforce the linkage, the resulting proof is valid and the invalid pair is accepted into the canonical state. A victim user who later deploys a contract under the declared class hash (which appears legitimate from the Sierra side) and receives funds will have those funds permanently locked. No privileged role beyond submitting a declare transaction is required.

### Recommendation

Inside `execute_declare_transaction`, after loading `compiled_class_hash` and before calling `dict_update`, add an assertion that `compiled_class_hash` is present in the set of compiled class facts that have been validated for the current block. Concretely, the OS should maintain a set (or sorted array) of validated compiled class hashes produced by `validate_compiled_class_facts` and perform a membership check — analogous to moving the linkage check from `_initiateNFTDeposit` into `registerNFTPair` in the external report's recommendation.

### Proof of Concept

1. Attacker submits a v3 declare transaction for a valid Sierra class but sets `compiled_class_hash = 0xdeadbeef` (any non-zero felt with no corresponding compiled class).
2. A sequencer (malicious or buggy) includes this transaction in a block.
3. The OS executes `execute_declare_transaction`: Sierra class hash passes `finalize_class_hash` validation; `assert_not_zero(0xdeadbeef)` passes; `dict_update(key=class_hash, prev_value=0, new_value=0xdeadbeef)` succeeds.
4. `compute_class_commitment` commits `class_hash → H(CONTRACT_CLASS_LEAF_VERSION, 0xdeadbeef)` to the class tree.
5. A victim deploys a contract under `class_hash` and receives funds.
6. Any call to the contract requires the OS to locate the compiled class for `0xdeadbeef`; none exists. All entry points fail permanently. Funds are frozen with no recovery path. [5](#0-4) [6](#0-5)

func validate_compiled_class_facts{poseidon_ptr: PoseidonBuiltin*, range_check_ptr}(
    n_compiled_class_facts, compiled_class_facts: CompiledClassFact*, builtin_costs: felt*
) {
    if (n_compiled_class_facts == 0) {
        return ();
    }
    alloc_locals;

    let compiled_class_fact = &compiled_class_facts[0];
    let compiled_class = compiled_class_fact.compiled_class;

    validate_entry_points(
        n_entry_points=compiled_class.n_external_functions,
        entry_points=compiled_class.external_functions,
    );

    validate_entry_points(
        n_entry_points=compiled_class.n_l1_handlers, entry_points=compiled_class.l1_handlers
    );
    // Compiled classes are expected to end with a `ret` opcode followed by a pointer to the
    // builtin costs.
    assert compiled_class.bytecode_ptr[compiled_class.bytecode_length] = 0x208b7fff7fff7ffe;
    assert compiled_class.bytecode_ptr[compiled_class.bytecode_length + 1] = cast(
        builtin_costs, felt
    );

    // Calculate the compiled class hash.
    // This hint enters a new scope that contains the bytecode segment structure of the class.
    %{ EnterScopeWithBytecodeSegmentStructure %}
    let (hash) = blake_compiled_class_hash(compiled_class, full_contract=FALSE);
    %{ LoadClass %}

    assert compiled_class_fact.hash = hash;

    return validate_compiled_class_facts(
        n_compiled_class_facts=n_compiled_class_facts - 1,
        compiled_class_facts=&compiled_class_facts[1],
        builtin_costs=builtin_costs,
    );

func get_contract_class_leaf_hash{poseidon_ptr: PoseidonBuiltin*}(compiled_class_hash: felt) -> (
    hash: felt
) {
    if (compiled_class_hash == UNINITIALIZED_CLASS_HASH) {
        return (hash=0);
    }

    // Return H(CONTRACT_CLASS_LEAF_VERSION, compiled_class_hash).
    let (hash_value) = poseidon_hash(CONTRACT_CLASS_LEAF_VERSION, compiled_class_hash);
    return (hash=hash_value);
}

func compute_class_commitment{poseidon_ptr: PoseidonBuiltin*, range_check_ptr}(
    class_changes_start: DictAccess*,
    n_class_updates: felt,
    patricia_update_constants: PatriciaUpdateConstants*,
) -> (contract_class_tree_update_output: CommitmentUpdate) {
    alloc_locals;

    // Guess the initial and final roots of the contract class tree.
    local initial_root;
    local final_root;
    %{ SetPreimageForClassCommitments %}

    // Create a dictionary mapping class hash to the contract class leaf hash,
    // to prepare the input for the commitment tree update.
    let (local hashed_class_changes: DictAccess*) = alloc();
    hash_class_changes(
        n_class_updates=n_class_updates,
        class_changes=class_changes_start,
        hashed_class_changes=hashed_class_changes,
    );

    // Call patricia_update_using_update_constants() instead of patricia_update()
    // in order not to repeat globals_pow2 calculation.

        patricia_update_constants=patricia_update_constants,
        update_ptr=hashed_class_changes,
        n_updates=n_class_updates,
        height=MERKLE_HEIGHT,
        prev_root=initial_root,
        new_root=final_root,
    );

    return (
        contract_class_tree_update_output=CommitmentUpdate(
            initial_root=initial_root, final_root=final_root
        ),
    );
