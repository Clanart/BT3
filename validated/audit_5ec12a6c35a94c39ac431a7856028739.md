### Title
Wrong Amount Field Used in `TransferMessage` Construction in `fast_fin_transfer_to_other_chain` — (`near/omni-bridge/src/lib.rs`)

### Summary

In `fast_fin_transfer_to_other_chain`, the `TransferMessage` sent to the destination chain is constructed with `amount: fast_transfer.amount` (the gross total, including fee) instead of `amount_without_fee` (the net amount the recipient should receive). Only `amount_without_fee` of tokens are burned/locked on NEAR, but the MPC-signed message instructs the destination chain to release the full gross total to the recipient, creating unbacked token supply equal to the fee.

### Finding Description

In `near/omni-bridge/src/lib.rs`, the function `fast_fin_transfer_to_other_chain` (line 914) handles fast-finalization of transfers whose recipient is on a non-NEAR chain. It correctly computes `amount_without_fee` and uses it for both burning and locking:

```rust
let amount_without_fee = fast_transfer
    .amount_without_fee()
    .near_expect(BridgeError::InvalidFee);

self.burn_tokens_if_needed(fast_transfer.token_id.clone(), amount_without_fee.into());
self.lock_tokens_if_needed(
    fast_transfer.get_destination_chain(),
    &fast_transfer.token_id,
    amount_without_fee,
);
``` [1](#0-0) 

However, when constructing the `TransferMessage` that the MPC will sign and the destination chain will execute, the code uses `fast_transfer.amount` (the gross total, which equals `amount_without_fee + fee`) instead of `amount_without_fee`:

```rust
let transfer_message = TransferMessage {
    origin_nonce: self.current_origin_nonce,
    token: OmniAddress::Near(fast_transfer.token_id.clone()),
    amount: fast_transfer.amount,   // ← BUG: should be amount_without_fee
    recipient: fast_transfer.recipient.clone(),
    fee: fast_transfer.fee.clone(),
    ...
};
``` [2](#0-1) 

On the EVM destination chain, `finTransfer` decodes `payload.amount` from the MPC-signed borsh blob and releases that exact value to `payload.recipient` — there is no fee deduction on the EVM side, because `TransferMessagePayload` carries no fee field and the borsh encoding does not include the fee amount:

```solidity
struct TransferMessagePayload {
    uint64 destinationNonce;
    uint8 originChain;
    uint64 originNonce;
    address tokenAddress;
    uint128 amount;       // ← released in full to recipient
    address recipient;
    string feeRecipient;  // only emitted in event, never paid
    bytes message;
}
``` [3](#0-2) 

The EVM `finTransfer` releases `payload.amount` directly to `payload.recipient` with no fee subtraction: [4](#0-3) 

The borsh encoding signed by MPC also omits the fee amount, confirming the EVM cannot reconstruct `amount_without_fee` independently: [5](#0-4) 

### Impact Explanation

- NEAR burns/locks `amount_without_fee` tokens.
- The MPC signs a `TransferMessage` with `amount = fast_transfer.amount` (gross total).
- The EVM `finTransfer` releases the gross total to the recipient.
- The delta — equal to the fee — is released to the recipient but was never backed by locked collateral on NEAR.
- This directly breaks bridge collateralization: for every fast transfer to a non-NEAR recipient, unbacked tokens equal to the fee are minted/released on the destination chain.
- Repeated exploitation inflates unbacked supply proportional to the fee on every such fast transfer.

**Impact class:** High — balance/accounting corruption that breaks bridge collateralization.

### Likelihood Explanation

This is triggered by normal, non-malicious operation of any trusted relayer performing a fast finalization for a transfer whose recipient is on a non-NEAR chain (e.g., EVM). No adversarial intent is required; the bug fires on every such fast transfer. The trusted-relayer check at line 756 gates entry, but the bug is in the protocol logic executed by legitimate relayers in the course of their normal duties. [6](#0-5) 

### Recommendation

Replace `fast_transfer.amount` with `amount_without_fee` in the `TransferMessage` construction inside `fast_fin_transfer_to_other_chain`:

```rust
let transfer_message = TransferMessage {
    origin_nonce: self.current_origin_nonce,
    token: OmniAddress::Near(fast_transfer.token_id.clone()),
    amount: amount_without_fee.into(),  // ← corrected
    recipient: fast_transfer.recipient.clone(),
    fee: fast_transfer.fee.clone(),
    sender: OmniAddress::Near(env::current_account_id()),
    msg: fast_transfer.msg.clone(),
    destination_nonce,
    origin_transfer_id: Some(fast_transfer.transfer_id.clone()),
};
```

This ensures the MPC-signed message instructs the destination chain to release only the net recipient amount, consistent with the collateral actually burned/locked on NEAR.

### Proof of Concept

1. A transfer of 1000 tokens with fee=100 is initiated from EVM to EVM (routed through NEAR fast-transfer path).
2. A trusted relayer calls `fast_fin_transfer` on NEAR, sending 900 tokens (`amount_without_fee`) to the bridge.
3. `fast_fin_transfer_to_other_chain` is invoked:
   - Burns/locks 900 tokens on NEAR. ✓
   - Constructs `TransferMessage { amount: 1000, recipient: <EVM addr>, ... }`. ← bug
4. MPC signs the message with `amount = 1000`.
5. Relayer submits to EVM `finTransfer` with `payload.amount = 1000`.
6. EVM releases 1000 tokens to recipient (100 unbacked).
7. Bridge collateral is short by 100 tokens per such fast transfer. [7](#0-6)

    fn fast_fin_transfer_to_other_chain(
        &mut self,
        fast_transfer: &FastTransfer,
        storage_payer: AccountId,
        relayer_id: AccountId,
    ) {
        if fast_transfer.recipient.is_utxo_chain() {
            let btc_account_id = self.get_utxo_chain_token(fast_transfer.get_destination_chain());
            require!(
                fast_transfer.token_id == btc_account_id,
                BridgeError::NativeTokenRequiredForChain.as_ref()
            );
        }

        let amount_without_fee = fast_transfer
            .amount_without_fee()
            .near_expect(BridgeError::InvalidFee);

        self.burn_tokens_if_needed(fast_transfer.token_id.clone(), amount_without_fee.into());

        self.lock_tokens_if_needed(
            fast_transfer.get_destination_chain(),
            &fast_transfer.token_id,
            amount_without_fee,
        );

        let mut required_balance =
            self.add_fast_transfer(fast_transfer, relayer_id, storage_payer.clone());

        let destination_nonce =
            self.get_next_destination_nonce(fast_transfer.get_destination_chain());
        self.current_origin_nonce += 1;

        let transfer_message = TransferMessage {
            origin_nonce: self.current_origin_nonce,
            token: OmniAddress::Near(fast_transfer.token_id.clone()),
            amount: fast_transfer.amount,
            recipient: fast_transfer.recipient.clone(),
            fee: fast_transfer.fee.clone(),
            sender: OmniAddress::Near(env::current_account_id()),
            msg: fast_transfer.msg.clone(),
            destination_nonce,
            origin_transfer_id: Some(fast_transfer.transfer_id.clone()),
        };
        let new_transfer_id = transfer_message.get_transfer_id();

        required_balance = self
            .add_transfer_message(transfer_message, storage_payer.clone())
            .saturating_add(required_balance);

        env::log_str(
            &OmniBridgeEvent::FastTransferEvent {
                fast_transfer: fast_transfer.clone(),
                new_transfer_id: Some(new_transfer_id),
            }
            .to_log_string(),
        );

        self.update_storage_balance(storage_payer, required_balance, NearToken::from_near(0));
    }
