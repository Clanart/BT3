### Title
Gateway L2 Gas Price Admission Check Uses Stale Current-Block Price Instead of `next_l2_gas_price` — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The `validate_resource_bounds` function in `StatefulTransactionValidator` validates a transaction's L2 gas `max_price_per_unit` against the **current block's** `l2_gas_price`, but the transaction will execute in the **next block** which uses `next_l2_gas_price` (the EIP-1559-adjusted price). This is the direct sequencer analog of the OracleMaker spread bug: a protection check uses only the pre-action state rather than the post-action state, causing the gateway to admit transactions that will fail during execution and to reject transactions that would succeed.

---

### Finding Description

`validate_resource_bounds` reads the L2 gas price reference from `GatewayFixedBlockSyncStateClient::get_block_info()`:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs:196-213
async fn validate_resource_bounds(...) {
    if self.config.validate_resource_bounds {
        // TODO(Arni): getnext_l2_gas_price from the block header.
        let previous_block_l2_gas_price = self
            .gateway_fixed_block_state_reader
            .get_block_info()
            .await?
            .gas_prices
            .strk_gas_prices
            .l2_gas_price;
        self.validate_tx_l2_gas_price_within_threshold(
            executable_tx.resource_bounds(),
            previous_block_l2_gas_price,
        )?;
    }
}
``` [1](#0-0) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client()` constructs a `BlockInfo` from the block header, mapping only `block_header.l2_gas_price` into `strk_gas_prices.l2_gas_price`. It never reads `block_header.next_l2_gas_price`:

```rust
// crates/apollo_gateway/src/gateway_fixed_block_state_reader.rs:42-62
let block_info = BlockInfo {
    gas_prices: GasPrices {
        strk_gas_prices: GasPriceVector {
            l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
            ...
        },
    },
    ...
};
``` [2](#0-1) 

`StorageBlockHeader` stores both fields separately:

```rust
pub l2_gas_price: GasPricePerToken,      // current block's price
pub next_l2_gas_price: GasPrice,          // EIP-1559 price for the NEXT block
``` [3](#0-2) 

The `next_l2_gas_price` is computed by `calculate_next_l2_gas_price_for_fin` after each block is finalized and stored in the block header. It is the price that will actually govern execution in the next block:

```rust
// crates/apollo_consensus_orchestrator/src/sequencer_consensus_context.rs:328-334
block_header_without_hash = BlockHeaderWithoutHash {
    l2_gas_consumed: l2_gas_used,
    next_l2_gas_price: self.l2_gas_price,  // EIP-1559 adjusted
    ...
};
``` [4](#0-3) 

The admission threshold is therefore computed as:

```
threshold = min_gas_price_percentage% × block_N.l2_gas_price
```

But the correct reference is:

```
threshold = min_gas_price_percentage% × block_N.next_l2_gas_price
```

The TODO comment in the code explicitly acknowledges this gap: `// TODO(Arni): getnext_l2_gas_price from the block header.` [5](#0-4) 

---

### Impact Explanation

**Admits invalid transactions (High — gateway accepts invalid transactions before sequencing):**
When block N is congested, `next_l2_gas_price > l2_gas_price`. A transaction with `max_price_per_unit` in the range `[threshold_current, next_l2_gas_price)` passes the gateway check but fails in the batcher because the blockifier's `check_fee_bounds` enforces `max_price_per_unit >= actual_block_gas_price` (which equals `next_l2_gas_price`). The transaction is admitted to the mempool, consumes batcher resources, and then fails execution.

**Rejects valid transactions (High — gateway rejects valid transactions before sequencing):**
When block N is under-utilized, `next_l2_gas_price < l2_gas_price`. A transaction with `max_price_per_unit` in the range `[threshold_next, threshold_current)` is rejected by the gateway even though it would succeed in the next block. This is the direct analog of the OracleMaker case where a position that crosses zero is mispriced.

---

### Likelihood Explanation

The EIP-1559 mechanism (`calculate_next_base_gas_price`) adjusts `next_l2_gas_price` by up to `1/gas_price_max_change_denominator` per block relative to `l2_gas_price`. In any block where gas usage differs from the target (the normal case), the two values diverge. The discrepancy is systematic and continuous, not edge-case. [6](#0-5) 

---

### Recommendation

In `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client()`, expose `next_l2_gas_price` from the block header and pass it separately to `validate_resource_bounds`. The `validate_tx_l2_gas_price_within_threshold` call should use `block_header.next_l2_gas_price` as the reference price, not `block_header.l2_gas_price.price_in_fri`. This resolves the acknowledged TODO and aligns the admission check with the price that will actually govern execution.

---

### Proof of Concept

1. Block N executes at full capacity: `l2_gas_price = 100 fri`, `l2_gas_consumed = max_block_size`.
2. EIP-1559 computes `next_l2_gas_price ≈ 112 fri` (stored in block N's header).
3. User submits a V3 transaction with `l2_gas.max_price_per_unit = 105 fri`. `min_gas_price_percentage = 100`.
4. Gateway calls `validate_resource_bounds`:
   - Reads `previous_block_l2_gas_price = 100 fri` (from `l2_gas_price`, not `next_l2_gas_price`).
   - Threshold = `100% × 100 = 100 fri`.
   - Check: `105 ≥ 100` → **PASS**. Transaction admitted to mempool.
5. Batcher builds block N+1 with `l2_gas_price = 112 fri` (= block N's `next_l2_gas_price`).
6. Blockifier `check_fee_bounds` enforces `max_price_per_unit ≥ actual_l2_gas_price`:
   - `105 < 112` → **FAIL**. Transaction reverts with `MaxGasPriceTooLow`.

The transaction passed gateway admission but fails execution — the gateway accepted an invalid transaction before sequencing. The root cause is identical in structure to the OracleMaker bug: the protection check evaluates only the pre-action state (`l2_gas_price`) rather than the state that will apply when the action takes effect (`next_l2_gas_price`).     // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants =
        orchestrator_versioned_constants::VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
