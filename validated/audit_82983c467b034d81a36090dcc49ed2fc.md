### Title
`batch_insert` Skips Duplicate Key/Hash Validation, Silently Corrupting the DataLayer Merkle Tree Root - (File: `crates/chia-datalayer/src/merkle/blob.rs`)

### Summary
`MerkleBlob::batch_insert` bypasses the duplicate-key and duplicate-hash guards that `MerkleBlob::insert` enforces. When a batch containing repeated `KeyId` or `Hash` values is supplied, leaf nodes are written directly into the blob and the `BlockStatusCache` is silently overwritten, producing a structurally invalid Merkle tree whose root hash is wrong. Any DataLayer inclusion proof derived from that root is therefore forged.

### Finding Description
`MerkleBlob::insert` (the single-item path) guards against duplicates at lines 369–374:

```rust
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
```

`batch_insert` takes a completely different code path for the bulk of its input. When the tree already holds ≥ 2 leaves the `if self.block_status_cache.leaf_count() <= 1` guard is skipped entirely, and every item in the caller-supplied vector is written via `insert_entry_to_blob` with no duplicate check:

```rust
for ((key, value), hash) in keys_values_hashes {          // no guard
    let new_leaf_index = self.get_new_index();
    let new_block = Block { … Node::Leaf(LeafNode { hash, key, value, … }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?; // writes unconditionally
    indexes.push(new_leaf_index);
}
```

`insert_entry_to_blob` calls `BlockStatusCache::add_leaf`, which uses `HashMap::insert` — a silent overwrite:

```rust
fn add_leaf(&mut self, index: TreeIndex, leaf: LeafNode) {
    self.free_indexes.shift_remove(&index);
    self.key_to_index.insert(leaf.key, index);       // silently replaces old entry
    self.leaf_hash_to_index.insert(leaf.hash, index);// silently replaces old entry
}
```

Result: the blob now contains two `LeafNode` blocks with the same `KeyId`/`Hash`, but the cache only tracks the last one. The tree structure (parent/child pointers) is built around both nodes, so the root hash is computed over a tree that violates the uniqueness invariant. The `check_integrity` path (only active in test builds) would catch this, but production builds have `check_integrity_on_drop: cfg!(test)` set to `false`.

Even when the tree starts with 0–1 leaves, only the last two items popped from the vector go through `self.insert()` (the safe path); all earlier items in the vector still bypass the duplicate check. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) 

### Impact Explanation
A corrupted `MerkleBlob` has a root hash that does not correspond to any valid set of unique key-value pairs. Any `get_proof_of_inclusion` call on the corrupted tree produces a `ProofOfInclusion` whose `combined_hash` chain leads to the wrong root. A DataLayer subscriber or verifier that trusts this root will accept inclusion proofs for keys that were never legitimately inserted, or reject proofs for keys that were. This directly satisfies the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."* [6](#0-5) 

### Likelihood Explanation
`batch_insert` is the primary high-throughput insertion API, exposed directly through the Python wheel (`py_batch_insert`) and used by the DataLayer sync path. Any caller that supplies a batch with a repeated key — whether by mistake or by a malicious DataLayer publisher — triggers the corruption. No special privilege is required beyond the ability to call `batch_insert` with attacker-controlled input. [7](#0-6) 

### Recommendation
Add the same duplicate guards that `insert` uses at the start of the fast-path loop in `batch_insert`:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // … existing code …
}
```

Alternatively, pre-validate the entire input vector for uniqueness before any blob mutation begins, so the operation is atomic (all-or-nothing).

### Proof of Concept

```python
from chia_rs.datalayer import MerkleBlob, KeyId, ValueId
import hashlib

def h(n: int) -> bytes:
    return hashlib.sha256(n.to_bytes(8, "big")).digest()

blob = MerkleBlob(bytearray())

# Seed the tree with 3 leaves so leaf_count > 1 (bypasses the safe path)
for i in range(3):
    blob.insert(KeyId(i), ValueId(i), h(i))
blob.calculate_lazy_hashes()
root_before = blob.get_root_hash()

# batch_insert with a duplicate key (KeyId(0) already exists)
# The fast path writes it unconditionally; no KeyAlreadyPresent error is raised.
blob.batch_insert(
    [(KeyId(0), ValueId(99)), (KeyId(10), ValueId(10))],
    [h(0), h(10)],
)
blob.calculate_lazy_hashes()
root_after = blob.get_root_hash()

# root_after is computed over a tree with two leaves sharing KeyId(0).
# A proof of inclusion for KeyId(0) now points to the *second* leaf (index
# overwritten in the cache), while the first leaf is an orphan in the blob.
assert root_before != root_after  # tree root silently changed
proof = blob.get_proof_of_inclusion(KeyId(0))
# proof.node_hash corresponds to h(0) (the duplicate), not the original leaf,
# yet it verifies against root_after — a forged state.
```

    fn insert_entry_to_blob(&mut self, index: TreeIndex, block: &Block) -> Result<(), Error> {
        let new_block_bytes = block.to_bytes()?;
        let extend_index = self.extend_index();
        match index.cmp(&extend_index) {
            Ordering::Greater => return Err(Error::BlockIndexOutOfBounds(index)),
            Ordering::Equal => self.blob.extend_from_slice(&new_block_bytes),
            Ordering::Less => {
                self.blob[block_range(index)].copy_from_slice(&new_block_bytes);
            }
        }

        match block.node {
            Node::Leaf(leaf) => self.block_status_cache.add_leaf(index, leaf),
            Node::Internal(..) => self.block_status_cache.add_internal(index),
        }

        Ok(())
    }

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
