### Title
Missing Declared Class Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts an arbitrary `class_hash` from the caller and writes it directly into `contract_state_changes` without verifying that the supplied hash corresponds to a class that has actually been declared in `contract_class_changes`. This is structurally identical to the reported BankV2 UID vulnerability: a critical identifier is consumed without identity validation against the authoritative registry. Any contract can exploit this to permanently brick itself (or any contract it controls), freezing all funds held in that contract's storage forever.

---

### Finding Description

In `execute_replace_class` (lines 878–916 of `syscall_impls.cairo`), the OS processes the `replace_class` syscall as follows:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;          // <-- attacker-supplied

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash,                    // <-- written without validation
        storage_ptr=state_entry.storage_ptr,
        nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The TODO comment at line 898 explicitly acknowledges the missing check. The function never consults `contract_class_changes` (the authoritative dictionary mapping declared class hashes to their compiled class hashes) to confirm that `request.class_hash` is a valid, declared class. Any felt value — including one that has never been declared — is accepted and committed to state.

Compare this to the analogous correct pattern used everywhere else in the OS: when `execute_declare_transaction` registers a class, it enforces `prev_value=0` (class declared at most once) and writes to `contract_class_changes`. When `execute_call_contract` or `charge_fee` look up a class, they read `state_entry.class_hash` from the already-validated state. `execute_replace_class` breaks this invariant by allowing an arbitrary, unvalidated hash to be injected into `contract_state_changes`. ### Impact Explanation

Once a contract's `class_hash` in `contract_state_changes` is set to an undeclared value, every subsequent call to that contract — whether via `call_contract`, `charge_fee`, or a direct transaction — will attempt to dispatch to a class that does not exist in the compiled class facts bundle. Execution will fail unconditionally. The contract's storage (including all token balances, vault shares, or other assets it holds) remains intact but permanently inaccessible. This constitutes **Critical: Permanent freezing of funds**.

The `charge_fee` function in `transaction_impls.cairo` illustrates the dependency: it reads `fee_state_entry.class_hash` from `contract_state_changes` and dispatches to it. If the fee token contract itself were targeted, fee collection would halt for every subsequent transaction, constituting a **High: Network not being able to confirm new transactions**. [3](#0-2) 

---

### Likelihood Explanation

The `replace_class` syscall is available to any deployed contract with no privilege requirement. An attacker needs only to:
1. Deploy a contract (permissionless on StarkNet).
2. Have that contract call `replace_class` with a felt value that is not a declared class hash.

The OS will accept and commit the invalid hash. No operator cooperation, leaked key, or network-level attack is required. The entry path is fully reachable by an unprivileged transaction sender or contract deployer.

---

### Recommendation

Before writing the new `class_hash` to `contract_state_changes`, verify that it exists as a key in `contract_class_changes` (i.e., it has been declared in the current or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` with `key=class_hash` and assert the returned compiled class hash is non-zero. This mirrors the validation already implicitly enforced during `execute_declare_transaction` via `prev_value=0` and `assert_not_zero(compiled_class_hash)`. [4](#0-3) 

---

### Proof of Concept

1. Attacker deploys contract `C` that holds user funds (e.g., acts as a vault or ERC20 token).
2. Attacker (or a bug in `C`) calls the `replace_class` syscall from within `C`, supplying `class_hash = 0xdeadbeef` (an arbitrary felt never declared on-chain).
3. The OS executes `execute_replace_class`: it reads `request.class_hash = 0xdeadbeef`, skips the missing declared-class check (per the TODO at line 898), and writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes` for address `C`.
4. The block is proven and the state root is updated with `C`'s class hash set to `0xdeadbeef`.
5. Any subsequent transaction targeting `C` — including user withdrawals — causes the OS to look up class `0xdeadbeef` in the compiled class facts bundle, find nothing, and fail.
6. All funds stored in `C`'s storage slots are permanently frozen with no recovery path. [5](#0-4)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
