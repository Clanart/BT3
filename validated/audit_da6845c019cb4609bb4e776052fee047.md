### Title
DataLayer `ProofOfInclusion::valid()` Never Verifies Against an External Root — Forged Inclusion Proofs Always Pass - (File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs)

### Summary
`ProofOfInclusion::valid()` in the DataLayer Merkle implementation contains a tautological final check: it computes `existing_hash == self.root_hash()`, but `root_hash()` returns `last.combined_hash`, which is the same value `existing_hash` was just set to in the final loop iteration. The function therefore only verifies internal hash-chain consistency and never validates the proof against any external, trusted tree root. An attacker can trivially construct a `ProofOfInclusion` for any arbitrary `node_hash` that passes `valid()` without the key-value pair being present in the actual DataLayer store.

### Finding Description

`ProofOfInclusion::valid()` is defined as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;   // existing_hash := layer.combined_hash
    }
    existing_hash == self.root_hash()      // always true — see below
}
``` [1](#0-0) 

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // returns the last layer's combined_hash
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

After the loop completes, `existing_hash` holds `calculated_hash` from the last iteration, which was already verified to equal `last_layer.combined_hash`. `self.root_hash()` also returns `last_layer.combined_hash`. The comparison `existing_hash == self.root_hash()` is therefore `last_layer.combined_hash == last_layer.combined_hash` — a tautology that is always `true`.

For an empty `layers` vec, `existing_hash` stays as `self.node_hash` and `root_hash()` also returns `self.node_hash`, so the check is again always `true`.

**Consequence:** `valid()` only verifies that the supplied hash chain is internally self-consistent. It never checks that the computed root matches any known, on-chain-committed DataLayer root. An attacker can:

1. Choose any arbitrary `node_hash` (e.g., a key-value pair not in the store).
2. Build any number of `ProofOfInclusionLayer` entries where each `combined_hash` equals `calculate_internal_hash(prev, side, other_hash)` — trivially computable.
3. Deserialize this as a `ProofOfInclusion` (it is a `Streamable` type, fully deserializable from bytes).
4. Call `proof.valid()` → returns `true`. [3](#0-2) 

The Python binding exposes `valid()` directly and the Python stub documents it as the proof verification method: [4](#0-3) 

The `get_proof_of_inclusion` method on `MerkleBlob` is also exposed to Python, and all tests verify proofs solely by calling `proof.valid()` without any separate root comparison: [5](#0-4) 

### Impact Explanation

DataLayer clients and puzzles that rely on `ProofOfInclusion::valid()` to confirm that a key-value pair is present in a DataLayer store can be deceived by a forged proof. Any caller that does not separately compare `proof.root_hash()` against a known, trusted on-chain root (a step the API does not require or document) will accept the forged proof as valid. This lets untrusted input prove invalid DataLayer state, matching the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.**

### Likelihood Explanation

The `ProofOfInclusion` struct is `Streamable` and fully deserializable from attacker-controlled bytes. The `valid()` method is the sole documented proof-verification entry point in both the Rust API and the Python bindings. No existing test or usage in the repository separately checks `proof.root_hash()` against an external root before trusting the result. Any DataLayer consumer that follows the natural API usage pattern is vulnerable.

### Recommendation

`valid()` must accept a trusted root hash parameter and compare against it instead of `self.root_hash()`:

```rust
pub fn valid(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against the known on-chain root
}
```

All call sites must supply the on-chain-committed DataLayer root hash. The `root_hash()` helper can remain as a convenience accessor but should not be used as the validation target.

### Proof of Concept

```python
from chia_rs.datalayer import MerkleBlob, ProofOfInclusion, ProofOfInclusionLayer
from chia_rs.sized_bytes import bytes32
from chia_rs.sized_ints import uint8
import hashlib

# Compute a fake "internal hash" matching chia_datalayer's calculate_internal_hash
# (SHA-256 of prefix || left || right, where prefix encodes node type)
INTERNAL_PREFIX = bytes(30) + b'\x01\x00'  # example; adjust to actual prefix

def fake_internal_hash(left: bytes, right: bytes) -> bytes:
    return hashlib.sha256(INTERNAL_PREFIX + left + right).digest()

# Arbitrary leaf hash — NOT in any real DataLayer store
fake_leaf = bytes32(b'\xab' * 32)
fake_sibling = bytes32(b'\xcd' * 32)

# Build one internally consistent layer
combined = bytes32(fake_internal_hash(fake_leaf, fake_sibling))
layer = ProofOfInclusionLayer(
    other_hash_side=uint8(1),   # Right side
    other_hash=fake_sibling,
    combined_hash=combined,
)

# Construct the forged proof
forged_proof = ProofOfInclusion(node_hash=fake_leaf, layers=[layer])

# valid() returns True even though fake_leaf is not in any real tree
assert forged_proof.valid(), "Forged proof should pass the broken valid() check"
print("Forged proof accepted as valid — root_hash:", forged_proof.root_hash().hex())
```

The forged proof passes `valid()` because the final check `existing_hash == self.root_hash()` reduces to `combined == combined`, which is always true regardless of whether `fake_leaf` is in any real DataLayer store. [1](#0-0)
