### Title
DataLayer `Block::from_bytes()` Accepts Metadata/Node-Type Mismatch, Enabling Forged Inclusion Proofs — (File: `crates/chia-datalayer/src/merkle/format.rs`)

---

### Summary

The DataLayer Merkle blob format stores each block as a 2-byte `NodeMetadata` header followed by a 53-byte data section. `Block::from_bytes()` dispatches on `metadata.node_type` to decide whether to deserialize the data section as an `InternalNode` (45 bytes) or a `LeafNode` (53 bytes), but **never validates that the resulting parsed node variant agrees with the metadata type**. This is explicitly acknowledged in the source with the comment `// NOTE: metadata node type and node's type not verified for agreement`. An attacker who can supply a crafted blob can flip the metadata byte of any `InternalNode` block from `Internal` (0) to `Leaf` (1), causing the InternalNode's `left`/`right` child-index bytes to be reinterpreted as a `LeafNode`'s `key` field and the 8 zero-padding bytes to be reinterpreted as `value = 0`. The `BlockStatusCache` then registers this fabricated key, and `get_proof_of_inclusion()` returns a cryptographically self-consistent proof for a key that does not exist in the tree.

---

### Finding Description

**Struct layout and size mismatch (the analog):**

`InternalNode` serializes to 45 bytes; `LeafNode` serializes to 53 bytes. `DATA_SIZE` is hard-coded to 53 (the larger size) with the comment `// TODO: figure out the real max better than trial and error?`. When an `InternalNode` is written, `Node::to_bytes()` pads the 45-byte payload with 8 zero bytes to fill the fixed 53-byte slot:

```rust
base.resize(DATA_SIZE, 0);   // zero-pad to 53 bytes
```

When reading back, `streamable_from_bytes_ignore_extra_bytes` is used, which stops parsing as soon as the struct is satisfied and silently ignores any remaining bytes. This is correct when the metadata byte is honest. However, if the metadata byte is `Leaf` (1) while the data section holds an `InternalNode` payload, the parser reads all 53 bytes as a `LeafNode`:

| Bytes | InternalNode meaning | Misread as LeafNode |
|-------|---------------------|---------------------|
| 0–31  | `hash` (32 B)       | `hash` (32 B) — same |
| 32–36 | `parent` (5 B)      | `parent` (5 B) — same |
| 37–40 | `left` (u32)        | first 4 bytes of `key` (i64) |
| 41–44 | `right` (u32)       | last 4 bytes of `key` (i64) |
| 45–52 | zero-padding        | `value` = 0 |

The fabricated `key` is `(left as i64) << 32 | (right as i64)`, and `value` is 0.

**Root cause — no cross-validation in `Block::from_bytes()`:**

```rust
pub fn from_bytes(blob: BlockBytes) -> Result<Self, Error> {
    let metadata = NodeMetadata::from_bytes(&metadata_blob)...;
    let node = Node::from_bytes(&metadata, &data_blob)...;  // dispatches on metadata.node_type only
    Ok(Block { metadata, node })  // no check that metadata.node_type == node variant
}
```

`Node::from_bytes()` itself:

```rust
Ok(match metadata.node_type {
    NodeType::Internal => Node::Internal(streamable_from_bytes_ignore_extra_bytes(blob)?),
    NodeType::Leaf     => Node::Leaf(streamable_from_bytes_ignore_extra_bytes(blob)?),
})
```

There is no post-parse assertion that the resulting `Node` enum variant matches `metadata.node_type`.

**Attack chain:**

1. Attacker crafts a blob where block at index *N* has metadata byte `0x01` (Leaf) but the data section contains a valid `InternalNode` payload (hash H, parent P, left L, right R, zero-padded to 53 bytes).
2. `MerkleBlob::new(crafted_blob)` calls `BlockStatusCache::new()`, which iterates all blocks. Block *N* is classified as a leaf with fabricated key `K = (L << 32) | R` and value `0`.
3. `get_proof_of_inclusion(K)` traverses the lineage from block *N* to the root, collecting sibling hashes. Because block *N*'s stored hash is the actual hash of the internal node (which is consistent with the tree's hash chain), the resulting `ProofOfInclusion` passes `valid()`.
4. The caller receives a self-consistent proof that key `K` is included in the tree, even though `K` was never inserted.

---

### Impact Explanation

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."**

A crafted blob causes `get_proof_of_inclusion()` to return a `ProofOfInclusion` whose `valid()` method returns `true` for a key that was never inserted. Any consumer that trusts `proof.valid()` without independently verifying the root against a known-good source will accept the forged proof. This undermines the integrity guarantee of the DataLayer Merkle tree.

---

### Likelihood Explanation

`MerkleBlob::new()` accepts a raw `Vec<u8>` with no structural pre-validation beyond checking that the length is a multiple of `BLOCK_SIZE`. The `DeltaReader` and `DeltaFileCache` read blobs from files that may originate from peer nodes in a distributed DataLayer deployment. Any peer that can supply a delta file or blob to a node can trigger this path. The `check_integrity()` function is not called automatically during `new()`, and even if called, none of its checks (`IntegrityParentChildMismatch`, `IntegrityKeyToIndexCacheLength`, etc.) explicitly verify that `metadata.node_type` agrees with the actual node variant.

---

### Recommendation

1. **Add a cross-validation step in `Block::from_bytes()`**: after parsing, assert that the `metadata.node_type` matches the `Node` enum variant:
   ```rust
   match (&metadata.node_type, &node) {
       (NodeType::Internal, Node::Internal(_)) => {}
       (NodeType::Leaf,     Node::Leaf(_))     => {}
       _ => return Err(Error::FailedLoadingNode(...)),
   }
   ```
2. **Derive `DATA_SIZE` from the actual struct sizes** rather than using a magic number with a TODO comment. Add a compile-time assertion that `DATA_SIZE >= max(sizeof InternalNode serialized, sizeof LeafNode serialized)`.
3. **Call `check_integrity()` (or a lightweight variant) during `MerkleBlob::new()`** when loading from untrusted sources.

---

### Proof of Concept

```rust
// Craft a block where metadata says Leaf but data is an InternalNode
// InternalNode: hash=H(32), parent=None(5), left=1(4), right=2(4) → 45 bytes, pad 8 zeros
let mut crafted_block = [0u8; BLOCK_SIZE];
crafted_block[0] = 1; // NodeType::Leaf
crafted_block[1] = 0; // dirty = false
// bytes 2..34: hash H
// bytes 34..39: parent = None (0x00 + 4 zero bytes)
// bytes 39..43: left = 1 (u32 big-endian)
crafted_block[42] = 1;
// bytes 43..47: right = 2 (u32 big-endian)
crafted_block[46] = 2;
// bytes 47..55: zero-padding (value = 0)

let blob = crafted_block.to_vec();
let merkle = MerkleBlob::new(blob).unwrap();
// fabricated key = (1i64 << 32) | 2i64 = 4294967298
let fabricated_key = KeyId((1i64 << 32) | 2i64);
// get_proof_of_inclusion returns Ok(...) and proof.valid() == true
```

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
