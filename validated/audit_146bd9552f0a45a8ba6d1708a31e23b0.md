### Title
Unbounded `max_price_per_unit` Enables Felt-Arithmetic Overflow in `compute_max_possible_fee`, Allowing Fee-Free Transaction Execution - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

`compute_max_possible_fee` computes the maximum chargeable fee using raw felt arithmetic (mod the Stark prime P ≈ 2²⁵¹). The `max_price_per_unit` field of each `ResourceBounds` entry is only validated as non-negative (`assert_nn`), with no upper bound. An unprivileged transaction sender can craft resource bounds whose products sum to exactly 0 mod P, causing `compute_max_possible_fee` to return 0 and `charge_fee` to skip fee collection entirely — executing the transaction for free.

---

### Finding Description

**Validation gap in `pack_resource_bounds`:**

In `transaction_hash/transaction_hash.cairo`, `pack_resource_bounds` enforces:

```cairo
assert_nn_le(resource_bounds.max_amount, 2 ** 64 - 1);  // bounded to u64
assert_nn(resource_bounds.max_price_per_unit);           // only: >= 0
```

`assert_nn` constrains `max_price_per_unit` to `[0, (P−1)/2]` — up to ≈ 2²⁵⁰. There is no `assert_nn_le(max_price_per_unit, 2**128 - 1)` as the SNIP-8 spec intends (u128).

**Overflow in `compute_max_possible_fee`:**

In `execution/transaction_impls.cairo`, the fee ceiling is computed as:

```cairo
return l1_gas_bounds.max_amount * l1_gas_bounds.max_price_per_unit +
       l2_gas_bounds.max_amount * (l2_gas_bounds.max_price_per_unit + tx_info.tip) +
       l1_data_gas_bounds.max_amount * l1_data_gas_bounds.max_price_per_unit;
```

All arithmetic is in the Cairo felt field (mod P). With `max_price_per_unit` up to (P−1)/2 and `max_amount` up to 2⁶⁴−1, each product can reach ≈ 2³¹⁴, wrapping around P many times. The attacker controls all three terms and can engineer their sum to be ≡ 0 (mod P).

**Fee bypass in `charge_fee`:**

```cairo
let max_fee = compute_max_possible_fee(tx_info=tx_info);
if (max_fee == 0) {
    return ();   // <-- fee collection skipped entirely
}
```

When `max_fee == 0`, the function returns without executing the ERC-20 transfer to the sequencer.

---

### Impact Explanation

**Critical — Direct loss of funds.**

An attacker can execute arbitrary transactions (invoke, declare, deploy_account) without paying any fee. The sequencer receives zero compensation for the computational resources consumed. At scale, this drains sequencer revenue and can be used to spam the network with zero-cost transactions, threatening both economic sustainability and liveness.

---

### Likelihood Explanation

The attack requires only crafting a valid v3 transaction with specific `max_price_per_unit` values. No privileged access, leaked keys, or external dependency is needed. The OS itself accepts the transaction as valid (the hash computation passes, the bounds checks pass), and the proof will be valid — the fee bypass is enforced by the proof, not just the mempool.

---

### Recommendation

Add an explicit upper-bound check on `max_price_per_unit` in `pack_resource_bounds`, consistent with the u128 type specified in SNIP-8:

```cairo
func pack_resource_bounds{range_check_ptr}(resource_bounds: ResourceBounds) -> felt {
    assert_nn_le(resource_bounds.max_amount, 2 ** 64 - 1);
-   assert_nn(resource_bounds.max_price_per_unit);
+   assert_nn_le(resource_bounds.max_price_per_unit, 2 ** 128 - 1);
    return (resource_bounds.resource * 2 ** 64 + resource_bounds.max_amount) * 2 ** 128 +
        resource_bounds.max_price_per_unit;
}
```

Similarly, bound `tip` in `hash_fee_fields` (already done: `assert_nn_le(tip, 2 ** 64 - 1)`) and verify that `compute_max_possible_fee` is called only after these bounds are enforced.

### Proof of Concept

The Stark prime is P = 2²⁵¹ + 17·2¹⁹² + 1.

Set the following resource bounds in a v3 invoke transaction:

| Field | Value |
|---|---|
| l1_gas `max_amount` | 1 |
| l1_gas `max_price_per_unit` | (P−1)/2 |
| l2_gas `max_amount` | 1 |
| l2_gas `max_price_per_unit` | (P−1)/2 |
| l1_data_gas `max_amount` | 1 |
| l1_data_gas `max_price_per_unit` | 1 |
| tip | 0 |

All values pass `assert_nn` (each is in `[0, (P−1)/2]`) and `assert_nn_le(max_amount, 2⁶⁴−1)`.

`compute_max_possible_fee` computes:

```
1 * (P−1)/2  +  1 * ((P−1)/2 + 0)  +  1 * 1
= (P−1)/2 + (P−1)/2 + 1
= P − 1 + 1
= P
≡ 0  (mod P)
```

func compute_max_possible_fee(tx_info: TxInfo*) -> felt {
    tempvar resource_bounds: ResourceBounds* = tx_info.resource_bounds_start;
    let n_resource_bounds = (tx_info.resource_bounds_end - resource_bounds) / ResourceBounds.SIZE;

    // Only V3 transactions with all resource bounds are supported.
    assert tx_info.version = 3;
    assert n_resource_bounds = 3;

    tempvar l1_gas_bounds: ResourceBounds = resource_bounds[L1_GAS_INDEX];
    tempvar l2_gas_bounds: ResourceBounds = resource_bounds[L2_GAS_INDEX];
    tempvar l1_data_gas_bounds = resource_bounds[L1_DATA_GAS_INDEX];

    return l1_gas_bounds.max_amount * l1_gas_bounds.max_price_per_unit + l2_gas_bounds.max_amount *
        (l2_gas_bounds.max_price_per_unit + tx_info.tip) + l1_data_gas_bounds.max_amount *
        l1_data_gas_bounds.max_price_per_unit;
}
