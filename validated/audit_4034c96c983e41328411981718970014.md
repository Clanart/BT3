### Title
Unchecked Return Value of Fee Token Transfer in `charge_fee` - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

The `charge_fee` function in the StarkNet OS invokes the fee token's `transfer` entry point via `non_reverting_select_execute_entry_point_func`, but **completely discards the return value** (retdata). If the fee token's `transfer` returns a failure indicator (`false`) without reverting, the OS proceeds as if the fee was successfully collected, resulting in the sequencer receiving no fee while the transaction is included in the block.

---

### Finding Description

In `charge_fee` (lines 111–165 of `transaction_impls.cairo`), the OS constructs an `ExecutionContext` targeting the fee token's `transfer` entry point and calls:

```cairo
let remaining_gas = DEFAULT_INITIAL_GAS_COST;
non_reverting_select_execute_entry_point_func{remaining_gas=remaining_gas}(
    block_context=block_context, execution_context=&execution_context
);
return ();
```

The return tuple `(retdata_size, retdata, is_deprecated)` is silently dropped. `non_reverting_select_execute_entry_point_func` only asserts `is_reverted = 0` — it does **not** inspect the transfer's return data. [1](#0-0) 

Contrast this with how `__validate__`, `__validate_declare__`, and `__validate_deploy__` are handled — all of them capture and assert the return value:

```cairo
let (retdata_size, retdata, is_deprecated) = non_reverting_select_execute_entry_point_func(...);
if (is_deprecated == 0) {
    assert retdata_size = 1;
    assert retdata[0] = VALIDATED;
}
``` [2](#0-1) 

`non_reverting_select_execute_entry_point_func` itself only enforces non-reversion:

```cairo
assert is_reverted = 0;
return (retdata_size, retdata, is_deprecated);
``` [3](#0-2) 

The `TransferCallData` struct and the call site confirm the OS is performing an ERC20-style transfer to the sequencer address: [4](#0-3) [5](#0-4) 

---

### Impact Explanation

**Impact: Critical — Direct loss of funds.**

If the fee token's `transfer` entry point returns `false` (or any non-success value) without reverting — which is a valid behavior for ERC20-compatible tokens — the OS Cairo program accepts the proof as valid, the block is finalized, and the sequencer receives zero fee for the transaction. The user's transaction is executed and committed on-chain without paying the required fee. This constitutes a direct, permanent loss of protocol fee revenue (funds owed to the sequencer/protocol).

---

### Likelihood Explanation

The StarkNet fee token (`STRK`) is a protocol-controlled contract. However:

1. The OS code makes no enforcement that the fee token must revert on failure — it only checks `is_reverted = 0`. Any fee token implementation that returns `false` on insufficient balance (rather than reverting) would trigger this silently.
2. If the fee token contract is ever upgraded or replaced with one following a "return false on failure" pattern (common in older ERC20 designs), all transactions would be processed fee-free.
3. A transaction sender with insufficient balance in a non-reverting fee token can submit transactions and have them included without paying fees.

Likelihood is **medium** given the current STRK token reverts on failure, but the OS-level invariant is entirely unenforced.

---

### Recommendation

Capture and validate the return value of the fee token transfer in `charge_fee`, analogous to how `__validate__` return data is checked. After calling `non_reverting_select_execute_entry_point_func`, assert that the transfer returned a success indicator:

```cairo
let (retdata_size, retdata, is_deprecated) = non_reverting_select_execute_entry_point_func{
    remaining_gas=remaining_gas
}(block_context=block_context, execution_context=&execution_context);
if (is_deprecated == 0) {
    // ERC20 transfer must return (true,) on success.
    assert retdata_size = 1;
    assert retdata[0] = 1;  // true
}
```

This mirrors the pattern already used for `__validate__`, `__validate_declare__`, and `__validate_deploy__`.

---

### Proof of Concept

1. Deploy a fee token contract whose `transfer` function returns `(false,)` without reverting when the sender has insufficient balance.
2. Submit a V3 transaction from an account with zero fee token balance. `max_fee > 0` so `charge_fee` is entered.
3. The OS calls `transfer(sequencer, actual_fee)` on the fee token. The token returns `(false,)` without reverting.
4. `non_reverting_select_execute_entry_point_func` asserts `is_reverted = 0` — passes. The `(retdata_size, retdata, is_deprecated)` tuple is discarded.
5. `charge_fee` returns normally. The block is finalized with the transaction included.
6. The sequencer's fee token balance is unchanged — no fee was collected. The user's transaction was executed for free. [6](#0-5)

    alloc_locals;

    local tx_info: TxInfo* = tx_execution_context.execution_info.tx_info;
    let max_fee = compute_max_possible_fee(tx_info=tx_info);

    if (max_fee == 0) {
        return ();
    }

    local low_actual_fee;
    %{ LoadActualFee %}
    local calldata: TransferCallData = TransferCallData(
        recipient=block_context.block_info_for_execute.sequencer_address,
        amount=Uint256(low=low_actual_fee, high=0),
    );

    // Verify that the charged amount is not larger than the transaction's max_fee field.
    assert_nn_le(calldata.amount.low, max_fee);

    // TODO(ilya, 01/01/2026): Consider caching the fee_token_class_hash.
    local fee_token_address = block_context.os_global_context.starknet_os_config.fee_token_address;
    let (fee_state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
        key=fee_token_address
    );
    let (__fp__, _) = get_fp_and_pc();
    // Use block_info directly from block_context, so that charge_fee will always run in
    // execute-mode rather than validate-mode.
    local execution_context: ExecutionContext = ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=fee_state_entry.class_hash,
        calldata_size=TransferCallData.SIZE,
        calldata=&calldata,
        execution_info=new ExecutionInfo(
            block_info=block_context.block_info_for_execute,
            tx_info=tx_info,
            caller_address=tx_info.account_contract_address,
            contract_address=fee_token_address,
            selector=TRANSFER_ENTRY_POINT_SELECTOR,
        ),
        deprecated_tx_info=tx_execution_context.deprecated_tx_info,
    );

    let remaining_gas = DEFAULT_INITIAL_GAS_COST;
    non_reverting_select_execute_entry_point_func{remaining_gas=remaining_gas}(
        block_context=block_context, execution_context=&execution_context
    );
    return ();
}
