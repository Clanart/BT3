### Title
Missing Declared-Class Validation in `execute_replace_class` Enables Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary class hash from the calling contract without verifying that the hash corresponds to a previously declared class. This is a direct analog to the HSG "swap" attack: just as signers could remove one module and add a malicious one while keeping the count unchanged, a contract can swap its valid class hash for an undeclared one while the OS only records the state change without validating the new value. The result is a permanently non-executable contract and irreversible loss of any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` (lines 878–916) processes the `replace_class` syscall. After deducting gas, it reads `class_hash` directly from the syscall request and writes it into `contract_state_changes` with no check that the hash exists in the declared-class set:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment at line 898 is an explicit acknowledgment by the team that the validation is absent. The OS never cross-references `class_hash` against `contract_class_changes` (the dict of classes declared in the current block) or the existing contract-class state tree. Any felt value — including one that has never been declared — is accepted and committed to the global state.

The structural parallel to the HSG bug is exact:

| HSG (Gnosis Safe) | StarkNet OS |
|---|---|
| `checkAfterExecution` checked module *count* but not module *identity* | `execute_replace_class` records a class-hash *change* but not class *existence* |
| Attacker removes a real module, inserts a malicious one; count stays the same | Contract replaces a real class hash with an undeclared one; the dict entry count stays the same |
| Guard bypass → arbitrary future execution | Invalid class hash → contract permanently non-executable |

The `contract_class_changes` dict, squashed in `squash_class_changes`, and the commitment computed in `compute_class_commitment` both operate only on the entries that were explicitly written during the block. Neither function retroactively validates that a class hash written by `replace_class` is present in the declared-class set. [2](#0-1) [3](#0-2) 

---

### Impact Explanation

Once the block containing the invalid `replace_class` call is proven and committed, the contract's `class_hash` field in the state tree permanently holds an undeclared value. Every subsequent transaction that targets the contract requires the OS (and the sequencer) to load the class bytecode for that hash. Because no such class exists, no valid proof can be generated for any future execution of the contract. The contract is irrecoverably bricked.

Any ERC-20 tokens, ETH, or other assets held in the contract's storage are permanently inaccessible. This satisfies the **Critical — Permanent freezing of funds** impact criterion.

---

### Likelihood Explanation

The attack path requires no privileged access:

1. Any unprivileged user can deploy a contract (e.g., a token vault or liquidity pool).
2. The contract's own code can issue the `replace_class` syscall with an arbitrary felt as the new class hash.
3. The OS accepts the call unconditionally.
4. The block is proven and the invalid state is finalized on-chain.

The missing check is explicitly flagged as a TODO by the development team, confirming it is a known, unresolved gap in the production code path. A malicious deployer can attract user deposits and then trigger this path to rug-pull all deposited funds with no recovery mechanism.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that it exists in the declared-class set. Concretely, perform a `dict_read` on `contract_class_changes` for the requested `class_hash` and assert the returned compiled-class hash is non-zero. If the class was declared in a prior block (not in `contract_class_changes`), a separate read from the contract-class state tree must confirm existence. This mirrors the fix recommended in the HSG report: a full content comparison, not merely a structural update.

---

### Proof of Concept

1. **Deploy** a contract `Vault` that accepts ERC-20 deposits from users. Users deposit funds; `Vault` holds balances in storage.
2. **Craft** a transaction from the `Vault` contract that calls the `replace_class` syscall with `class_hash = 0xdeadbeef` (an arbitrary felt that was never declared).
3. **Submit** the transaction. The sequencer executes it; `execute_replace_class` in the OS reads `class_hash = 0xdeadbeef` from the request and writes it into `contract_state_changes` without any validation.
4. **Block is proven.** The OS generates a valid STARK proof for the block (the missing check means no constraint is violated). The state root is updated; `Vault`'s class hash is now `0xdeadbeef`.
5. **Attempt recovery.** Any user submits a withdrawal transaction targeting `Vault`. The sequencer/OS must load the class for `0xdeadbeef` to execute the entry point. No such class exists. Proof generation fails. No transaction against `Vault` can ever be included in a valid block.
6. **All user funds are permanently frozen** with no on-chain recovery path. [4](#0-3)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}

func squash_class_changes{range_check_ptr}(
    class_changes_start: DictAccess*, class_changes_end: DictAccess*
) -> (n_class_updates: felt, squashed_contract_state_dict: DictAccess*) {
    alloc_locals;

    local squashed_dict: DictAccess*;
    %{ GuessClassesPtr %}
    let (local squashed_dict_end) = squash_dict(
        dict_accesses=class_changes_start,
        dict_accesses_end=class_changes_end,
        squashed_dict=squashed_dict,
    );

    %{ UpdateClassesPtr %}

    return (
        n_class_updates=(squashed_dict_end - squashed_dict) / DictAccess.SIZE,
        squashed_contract_state_dict=squashed_dict,
    );
