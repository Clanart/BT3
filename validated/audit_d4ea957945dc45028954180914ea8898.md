### Title
Unenforced Class Declaration Requirement in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the new class hash supplied by the caller is an already-declared class. The missing check is explicitly acknowledged by a TODO comment in the code. Because the OS is the cryptographic source of truth for state transitions, this unenforced requirement allows any contract to replace its own class hash with an arbitrary, undeclared value. Once committed on-chain, the contract becomes permanently unexecutable, freezing any funds it holds.

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The function accepts any `class_hash` value from the syscall request and writes it directly into `contract_state_changes` without consulting `contract_class_changes` to confirm the hash corresponds to a declared class. The TODO comment at line 898 explicitly acknowledges this missing enforcement.

The analogous enforcement path that *does* exist for class declarations is in `execute_declare_transaction`, where `dict_update` uses `prev_value=0` to enforce that a class may only be declared once:

```cairo
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [2](#0-1) 

No equivalent cross-check is performed in `execute_replace_class`. The `contract_class_changes` dict is never consulted.

When a subsequent call is made to a contract whose class hash is undeclared, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,
);
``` [3](#0-2) 

A `dict_read` on an undeclared class hash returns 0 (the default). `find_element` with key `0` will then fail to locate a matching `CompiledClassFact`, causing the Cairo program to abort. The block containing any call to the corrupted contract cannot be proven, making the contract permanently unexecutable.

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with an undeclared value and the state transition is committed on-chain (the OS produces a valid proof because it performs no validation), the contract is permanently unexecutable. Any ERC-20 tokens, ETH, or other assets held in the contract's storage are irretrievably frozen. The OS proof is accepted by the L1 verifier because the Cairo constraints are satisfied — the missing check means no constraint is violated from the proof system's perspective.

### Likelihood Explanation

The `replace_class` syscall is callable by any contract on behalf of itself. An attacker who:
1. Deploys a contract that accepts user deposits (or exploits a legitimate contract with an exposed `replace_class` path), and
2. Calls `replace_class` with an arbitrary undeclared felt value,

can trigger this with a single transaction. No privileged role is required. The sequencer's blockifier may or may not independently validate the new class hash; the OS — which is the cryptographic enforcement layer — does not, so even if the blockifier is later patched, the OS remains the exploitable gap.

### Recommendation

In `execute_replace_class`, before writing the new class hash to `contract_state_changes`, verify that the hash exists in `contract_class_changes`:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
// Ensure the class is declared (compiled_class_hash != 0).
assert_not_zero(compiled_class_hash);
```

This mirrors the enforcement already present in `execute_entry_point` and closes the gap identified by the TODO comment.

### Proof of Concept

1. Attacker deploys contract `C` that holds user funds and exposes a public `upgrade(new_hash)` function that calls `replace_class(new_hash)`.
2. Attacker calls `C.upgrade(0xdeadbeef)` where `0xdeadbeef` is never declared.
3. `execute_replace_class` in the OS writes `class_hash=0xdeadbeef` into `contract_state_changes` with no validation.
4. The OS produces a valid proof; L1 accepts the state update.
5. Any subsequent transaction calling `C` causes `execute_entry_point` to call `dict_read(contract_class_changes, 0xdeadbeef)` → returns 0 → `find_element(..., key=0)` aborts the Cairo program.
6. No block containing a call to `C` can ever be proven. All funds in `C` are permanently frozen.

func execute_replace_class{

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

        key=contract_address,

    );

    return ();
}
