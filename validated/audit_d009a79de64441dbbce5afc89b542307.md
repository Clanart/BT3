### Title
`EndBlock` Chain Halt via `SafeInt64` Overflow When `MaxGas = -1` — (`x/feemarket/keeper/abci.go`)

### Summary

When CometBFT consensus params set `MaxGas = -1` (unlimited block gas) and the chain has zero minimum gas price, an unprivileged attacker can submit a single Ethereum transaction with `gasLimit = math.MaxUint64`. A uint64 underflow bug in the ante handler's overflow guard lets this transaction pass all checks and be included in a block. At `EndBlock`, `SafeInt64(ctx.BlockGasWanted())` receives a value > `math.MaxInt64`, returns an error, and that error propagates through `ModuleManager.EndBlock` → `app.EndBlocker` → CometBFT `FinalizeBlock`, causing a deterministic chain halt.

---

### Finding Description

**Step 1 — Overflow guard bug in `CheckEthGasConsume`** [1](#0-0) 

The guard is:
```go
if gasWanted > math.MaxInt64-gasLimit {
    return ctx, fmt.Errorf("gasWanted(%d) + gasLimit(%d) overflow", ...)
}
gasWanted += gasLimit
if gasWanted > blockGasLimit {
    return ctx, errorsmod.Wrapf(errortypes.ErrOutOfGas, ...)
}
```

When `gasLimit = math.MaxUint64 (18446744073709551615)`, the expression `math.MaxInt64 - gasLimit` is evaluated in `uint64` arithmetic:

```
uint64(9223372036854775807) - uint64(18446744073709551615)
= 9223372036854775808   (underflow wraps around)
```

So `gasWanted (0) > 9223372036854775808` is **false** — the overflow check silently passes. This is a uint64 underflow bug.

**Step 2 — Block gas limit check is disabled when `MaxGas = -1`** [2](#0-1) 

When `MaxGas = -1`, `BlockGasLimit` returns `math.MaxUint64`. The subsequent check `gasWanted > blockGasLimit` becomes `math.MaxUint64 > math.MaxUint64` = **false** — also passes.

**Step 3 — Fee check passes with zero gas price**

If `NoBaseFee = true` or `baseFee = 0` and `MinGasPrice = 0`, the fee for a tx with `gasPrice = 0` is zero. The balance check passes trivially.

**Step 4 — `EndBlock` receives `BlockGasWanted = math.MaxUint64` and returns an error** [3](#0-2) 

```go
gasWanted := ctx.BlockGasWanted()          // = math.MaxUint64
gw, err := ethermint.SafeInt64(gasWanted)  // returns error: value exceeds MaxInt64
if err != nil {
    return err                             // error propagates up
}
``` [4](#0-3) 

**Step 5 — Error propagates to chain halt** [5](#0-4) 

`EndBlocker` returns the error from `ModuleManager.EndBlock`. In Cosmos SDK v0.50+, an error returned from `EndBlock` causes `FinalizeBlock` to abort, halting the chain deterministically across all validators.

---

### Impact Explanation

A single malformed-but-ante-passing transaction causes every validator to fail `FinalizeBlock` for that block height. Since all validators execute the same deterministic logic, the chain halts permanently at that height. This matches the Critical impact: **valid block-processing path halts the chain**.

---

### Likelihood Explanation

Requires two specific but valid configurations:
1. `MaxGas = -1` (no block gas limit) — explicitly documented in CometBFT as a valid setting; many chains and testnets use it.
2. Zero effective gas price (`NoBaseFee = true` or `MinGasPrice = 0` with `baseFee = 0`) — common on private chains, testnets, and chains that disable EIP-1559.

On any chain meeting both conditions, a single unprivileged transaction triggers the halt. No special privileges, keys, or validator cooperation required.

---

### Recommendation

1. **Fix the overflow guard** in `CheckEthGasConsume` to correctly handle `gasLimit > math.MaxInt64`:
   ```go
   // Replace:
   if gasWanted > math.MaxInt64-gasLimit {
   // With:
   if gasLimit > math.MaxInt64 || gasWanted > math.MaxInt64-gasLimit {
   ``` [6](#0-5) 

2. **Saturate instead of error in `EndBlock`**: replace `SafeInt64` with a saturating clamp to `math.MaxInt64` so that an oversized value degrades gracefully rather than halting the chain. [7](#0-6) 

3. **Validate `MaxGas` at startup**: warn or reject `MaxGas = -1` in combination with zero gas price, or document the interaction explicitly.

---

### Proof of Concept

```go
// Set MaxGas = -1 and NoBaseFee = true in chain config.
// Submit a single Ethereum tx:
tx := evmtypes.NewTxContract(
    chainID, nonce, nil, math.MaxUint64, // gasLimit = MaxUint64
    big.NewInt(0),                        // gasPrice = 0
    nil, nil, nil, nil,
)
// Ante handler passes (overflow check underflows, block gas limit = MaxUint64).
// Block is committed. EndBlock calls:
//   SafeInt64(math.MaxUint64) → error "uint64 value 18446744073709551615 cannot exceed 9223372036854775807"
// ModuleManager.EndBlock returns error → FinalizeBlock aborts → chain halt.
```

func SafeInt64(value uint64) (int64, error) {
	if value > uint64(math.MaxInt64) {
		return 0, fmt.Errorf("uint64 value %v cannot exceed %v", value, math.MaxInt64)
	}

	return int64(value), nil
}
