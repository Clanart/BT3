### Title
Missing Class Hash Existence Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `execution/syscall_impls.cairo`)

---

### Summary
The `execute_replace_class` syscall handler in the StarkNet OS program does not verify that the new class hash supplied by a contract corresponds to a previously declared class. This is an explicitly acknowledged missing check (marked with a `TODO`). Any contract can call `replace_class` with an arbitrary, undeclared class hash, permanently bricking itself. Funds held in the bricked contract are permanently frozen, as no future call to that contract can be proven.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function updates the contract's class hash in `contract_state_changes` without performing any validation that the new class hash is a declared class: [1](#0-0) 

The critical missing check is explicitly acknowledged at line 898:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
``` [2](#0-1) 

The same missing validation exists in the deprecated path: [3](#0-2) 

When a subsequent call is made to the bricked contract, `execute_entry_point` reads the (now invalid) class hash from `contract_state_changes`, then attempts to look it up in `contract_class_changes` via `dict_read`. Since the hash was never declared, the lookup returns 0. The subsequent `find_element` call over the compiled class facts array then fails to locate a compiled class with hash 0, causing the OS to be unable to generate a valid proof for any block containing a call to that contract: [4](#0-3) 

The analog to the external report is direct: the external report describes a **missing post-execution check** (the post-check is bypassed via delegate call + SELFDESTRUCT). Here, the **missing post-condition check** is the validation that the replacement class hash is declared — the OS accepts the `replace_class` state transition without enforcing this invariant, just as the LSP0 account accepted the delegate call without guaranteeing the post-execution check would run.

Additionally, the `library_call` syscall executes code from an arbitrary class in the **calling contract's address context**: [5](#0-4) 

This means a contract that makes a `library_call` to a malicious class can have its own class hash replaced with an invalid value by the callee — a direct structural analog to the delegate call + SELFDESTRUCT pattern in the external report.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with an undeclared value:
1. The OS accepts the block containing the `replace_class` call as valid (no check is performed).
2. Any subsequent transaction targeting that contract causes `execute_entry_point` to fail to locate the compiled class, making it impossible to generate a valid proof for blocks containing such calls.
3. All funds (tokens, ETH, ERC-20 balances) stored in the bricked contract's storage are permanently inaccessible — no withdrawal, transfer, or recovery function can ever be called again.

---

### Likelihood Explanation

**Medium.**

- Any deployed contract can call `replace_class` with an arbitrary felt value as the class hash. No privileged role is required.
- A malicious contract owner can intentionally brick a contract holding other users' funds (e.g., a shared vault, escrow, or token contract).
- The `library_call` vector allows a contract to be bricked without the contract owner's direct intent: if a contract library-calls a class that internally calls `replace_class` with an invalid hash, the calling contract is bricked. This is reachable by any user who can trigger a `library_call` in a vulnerable contract.
- The TODO comment confirms the team is aware the check is absent, meaning the window of exposure is open until the fix is deployed.

---

### Recommendation

In `execute_replace_class` (both `syscall_impls.cairo` and `deprecated_execute_syscalls.cairo`), before updating `contract_state_changes`, verify that the new `class_hash` exists as a key in `contract_class_changes` (i.e., it has been declared in the current or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` with the new class hash and assert the result is non-zero before committing the state update. [6](#0-5) 

---

### Proof of Concept

1. **Deploy** a contract `Vault` that holds user funds and exposes a `brick_self()` function that calls `replace_class(0xdeadbeef)`.
2. **Invoke** `brick_self()`. The OS executes `execute_replace_class`, sets `Vault`'s class hash to `0xdeadbeef` in `contract_state_changes`, and writes the block — no validation of `0xdeadbeef` occurs.
3. **Attempt** any subsequent call to `Vault` (e.g., `withdraw()`). `execute_entry_point` reads class hash `0xdeadbeef`, calls `dict_read{dict_ptr=contract_class_changes}(key=0xdeadbeef)` → returns 0 (undeclared), then `find_element(..., key=0)` fails to locate a compiled class.
4. **Result**: No valid proof can be generated for any block containing a call to `Vault`. All funds in `Vault` are permanently frozen.

The same outcome is achievable via `library_call`: a contract that library-calls a malicious class which internally calls `replace_class(0xdeadbeef)` will have its own class hash replaced, bricking the calling contract without the calling contract's explicit intent.

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
