### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary felt value as a new class hash without verifying that the hash corresponds to a previously declared contract class. This allows any contract — including one deployed by an unprivileged user — to replace its own class hash with an undeclared (non-existent) value, permanently bricking the contract and freezing all funds stored within it.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the new class hash directly from the user-supplied syscall request and writes it into `contract_state_changes` without any Cairo-level constraint confirming the hash is declared:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The developer-acknowledged TODO at line 898 confirms the missing check is known but unimplemented. The `contract_class_changes` dictionary (which maps class hashes to compiled class hashes) is never consulted here. There is no `assert_not_zero(class_hash)` guard, no lookup into `contract_class_changes`, and no cross-reference against the compiled class facts bundle validated post-execution in `os.cairo`. [2](#0-1) 

Compare this to the `execute_declare_transaction` path in `transaction_impls.cairo`, where `dict_update` on `contract_class_changes` uses `prev_value=0` to enforce that a class may only be declared once and `assert_not_zero(compiled_class_hash)` to reject zero hashes: [3](#0-2) 

No equivalent guard exists in `execute_replace_class`.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is overwritten with an undeclared value (e.g., `0xdeadbeef`, `1`, or any felt not present in `contract_class_changes`), every future call to that contract will fail at class resolution time. The contract's storage — including all token balances, collateral, or protocol reserves held within it — becomes permanently inaccessible. Because the state transition is committed to the Merkle tree and included in the OS output, there is no recovery path without a protocol-level rollback. [4](#0-3) 

---

### Likelihood Explanation

The `replace_class` syscall is available to any executing contract with no privilege restriction. An attacker can:

1. Deploy a contract whose code unconditionally calls `replace_class(class_hash=<undeclared_value>)`.
2. Advertise the contract as a legitimate DeFi vault or token bridge to attract user deposits.
3. Trigger the `replace_class` call after funds accumulate.

Alternatively, a buggy upgrade path in any existing protocol contract could accidentally supply an undeclared hash. Because the OS imposes no Cairo constraint preventing this, the proof for such a block is valid and the state transition is final.

---

### Recommendation

Add a Cairo-level constraint inside `execute_replace_class` that verifies the requested `class_hash` exists in `contract_class_changes` (or in the global compiled class facts bundle) before committing the update. Concretely:

- Perform a `dict_read` on `contract_class_changes` for `class_hash` and assert the returned compiled class hash is non-zero.
- Alternatively, assert `class_hash` is present in the `compiled_class_facts` array validated by `validate_compiled_class_facts_post_execution`.

This mirrors the existing protection in `execute_declare_transaction` where `assert_not_zero(compiled_class_hash)` and `prev_value=0` enforce class integrity. [5](#0-4) 

---

### Proof of Concept

1. **Attacker deploys** a contract `VaultAttack` whose `__execute__` entry point calls:
   ```
   replace_class(class_hash=0x1)  // 0x1 is never declared
   ```
2. **Users deposit** funds (e.g., ERC-20 tokens) into `VaultAttack`, trusting its current (legitimate) class.
3. **Attacker invokes** `VaultAttack.__execute__`, triggering the `replace_class` syscall.
4. **OS execution** reaches `execute_replace_class`. `request.class_hash = 0x1`. No Cairo constraint checks `contract_class_changes` for key `0x1`. `dict_update` writes `class_hash=0x1` into `contract_state_changes` for `VaultAttack`'s address.
5. **State is committed.** `VaultAttack` now has class hash `0x1` in the Merkle state.
6. **All subsequent calls** to `VaultAttack` fail at class resolution — the class `0x1` has no compiled bytecode. User funds are permanently frozen.

The OS proof for this block is valid because no Cairo assertion is violated. The `validate_compiled_class_facts_post_execution` call in `os.cairo` only validates classes that were *executed*, not the target of `replace_class`. [2](#0-1) [6](#0-5)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    let revert_log = &revert_log[1];

    return ();
}
