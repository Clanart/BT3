### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the new class hash supplied by a contract corresponds to a previously declared class. An unprivileged contract can call `replace_class` with an arbitrary (undeclared) felt value, permanently rendering itself non-executable and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function accepts the caller-supplied `class_hash` and writes it directly into `contract_state_changes` without checking whether that hash exists in `contract_class_changes` (the declared-class registry):

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;
    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );
    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The developers themselves acknowledge the gap with the TODO comment at line 898. The identical omission exists in the deprecated path: [2](#0-1) 

When the contract is subsequently invoked, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // now an undeclared hash
);
// compiled_class_hash == 0 (default for missing key)

let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,           // find_element(key=0) → panic
);
``` [3](#0-2) 

`find_element` is not a soft-failure primitive; it panics when the key is absent. The OS proof for any block containing a subsequent call to the poisoned contract is therefore unprovable. The sequencer must permanently exclude all transactions targeting that contract, making any funds it holds irrecoverable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is overwritten with an undeclared value:
- Every future call to that contract causes the OS to panic inside `find_element`.
- The sequencer cannot include any such call in a provable block.
- All token balances, collateral, or other assets stored in the contract's storage are permanently inaccessible — there is no upgrade path because the contract itself is no longer callable.

---

### Likelihood Explanation

The attack requires only the ability to deploy a contract and submit a transaction — both are permissionless on StarkNet. Two realistic paths exist:

1. **Direct self-poisoning**: An attacker deploys a contract that accepts deposits, collects funds, then calls `replace_class(0)`. Funds are permanently frozen.
2. **Governance exploit**: A legitimate protocol exposes a class-upgrade governance function without independently validating the new hash against the declared-class registry. An attacker passes an undeclared hash through governance, freezing all protocol funds.

No privileged role, leaked key, or external dependency is required.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that it exists in `contract_class_changes` (i.e., that `dict_read` returns a non-zero compiled class hash). Concretely, inside `execute_replace_class`:

```cairo
// Proposed fix
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

Apply the same guard in the deprecated `execute_replace_class` in `deprecated_execute_syscalls.cairo`.

---

### Proof of Concept

1. **Declare** class `A` (class_hash = `H_A`, compiled_class_hash = `C_A`).
2. **Deploy** a contract at address `ADDR` using class `A`. The contract exposes:
   ```
   fn poison() { replace_class(felt(0)); }
   fn deposit() { /* accept ERC-20 transfer */ }
   ```
3. **Deposit** tokens into `ADDR` via `deposit()`.
4. **Call** `ADDR::poison()`. The OS executes `execute_replace_class` with `class_hash = 0`; no validation fires; `contract_state_changes[ADDR].class_hash` is set to `0`.
5. **Attempt** any subsequent call to `ADDR`. Inside `execute_entry_point`:
   - `dict_read(contract_class_changes, key=0)` → returns `0` (undeclared).
   - `find_element(..., key=0)` → **panic** — no compiled class with hash `0` exists.
6. The sequencer cannot produce a valid proof for any block containing a call to `ADDR`. All deposited tokens are permanently frozen.
