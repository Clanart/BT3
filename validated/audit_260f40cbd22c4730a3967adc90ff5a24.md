### Title
Unvalidated `class_hash` in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in `syscall_impls.cairo` accepts an attacker-controlled `class_hash` from the syscall request and directly commits it to the global contract state without verifying that the class hash corresponds to a declared contract class. This allows any contract to replace its own class with an arbitrary, undeclared hash, permanently rendering the contract uncallable and freezing all funds it holds.

---

### Finding Description

In `execute_replace_class` (lines 878–916 of `syscall_impls.cairo`), the `class_hash` field is read directly from the `ReplaceClassRequest` struct and written into `contract_state_changes` with no validation:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The TODO comment at line 898 is an explicit developer acknowledgment that this check is missing. [1](#0-0) 

The downstream consequence is in `execute_entry_point`, which resolves the class hash of any called contract:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [2](#0-1) 

If `class_hash` is not declared, `dict_read` returns 0 (the default for an absent key). `find_element` then searches for a compiled class fact with key `0`. If no such fact exists, the Cairo program traps, making it impossible for the sequencer to produce a valid STARK proof for any block that includes a call to the affected contract. The contract becomes permanently uncallable at the protocol level.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value and the state is committed to the global Patricia Merkle Tree, no future block can include a transaction that calls that contract. The sequencer cannot produce a valid proof for such a block because `find_element` will fail inside the OS Cairo program. Any ERC-20 tokens, ETH, or other assets held in the contract's storage are permanently inaccessible. There is no recovery path because the class hash is already committed on-chain and the OS has no mechanism to override it without a valid proof.

---

### Likelihood Explanation

**Medium.**

The `replace_class` syscall is callable by any contract on itself — no privileged role is required. An attacker can deploy a contract, fund it (or lure users to fund it), and then call `replace_class` with an arbitrary felt value as the class hash. The attack requires only a single transaction from an unprivileged sender. Additionally, legitimate contracts that implement upgradeable patterns could be manipulated into calling `replace_class` with an invalid hash through reentrancy or a malicious upgrade proposal.

---

### Recommendation

Inside `execute_replace_class`, before writing the new state entry, verify that `class_hash` is present in `contract_class_changes` (i.e., it has been declared in the current or a prior block). The check should assert that `dict_read{dict_ptr=contract_class_changes}(key=class_hash)` returns a non-zero compiled class hash. If the check fails, write a failure response and return without modifying state, consistent with how other invalid syscall arguments are handled.

---

### Proof of Concept

1. Attacker deploys `MaliciousContract` which holds or receives user funds.
2. Attacker calls a function on `MaliciousContract` that internally invokes the `replace_class` syscall with `class_hash = 0xdeadbeef` (any value not present in `contract_class_changes`).
3. The OS executes `execute_replace_class`: no validation is performed; `contract_state_changes` is updated with `class_hash = 0xdeadbeef` for `MaliciousContract`'s address. [3](#0-2) 
4. The block is proven and the state is committed. `MaliciousContract`'s class hash in the global state is now `0xdeadbeef`.
5. Any subsequent transaction targeting `MaliciousContract` reaches `execute_entry_point`, which calls `dict_read` on `contract_class_changes` for key `0xdeadbeef`, receiving `0`. [4](#0-3) 
6. `find_element` is called with `key=0`; no compiled class fact with that key exists, causing the Cairo program to trap. The sequencer cannot produce a proof for any block containing this call.
7. All funds in `MaliciousContract` are permanently frozen.
