### Title
Missing Declared Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the replacement class hash corresponds to a declared contract class. Any contract can replace its own class hash with an arbitrary undeclared felt value. Once committed on-chain, the contract becomes permanently non-executable, and any funds it holds are irreversibly frozen.

---

### Finding Description

In `execute_replace_class`, the OS reads the requested class hash from the syscall request and immediately writes it into `contract_state_changes` with no validation that the hash exists in `contract_class_changes`: [1](#0-0) 

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);

assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
```

The TODO comment at line 898 is the developers' own acknowledgment that this check is absent. There is no lookup into `contract_class_changes` to confirm that `class_hash` maps to a non-zero compiled class hash before the state is mutated.

Compare this to how class declaration enforces existence — `dict_update` with `prev_value=0` in `execute_declare_transaction` ensures a class is written exactly once: [2](#0-1) 

No equivalent guard exists in `execute_replace_class`.

The vulnerability class is identical to M-08: a **missing condition check on a state-mutating operation** allows an actor to drive a protocol invariant (here: "a contract's class hash must always point to a declared class") into a permanently broken state. In M-08 the missing check was `reserveRatioBips < 9000`; here the missing check is `contract_class_changes[class_hash] != 0`.

---

### Impact Explanation

Once a contract's class hash is set to an undeclared value and the block is proven:

1. Any subsequent call to that contract requires the OS to execute the entry point of the stored class hash.
2. The sequencer cannot supply valid class-data hints for an undeclared hash; the OS proof for any transaction that invokes the contract is unprovable.
3. The contract is permanently non-callable. All ERC-20 balances, ETH, or other assets stored in its storage are irreversibly frozen.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

The `replace_class` syscall is available to every deployed contract with no access restriction at the OS level. Realistic trigger paths:

- **Upgradeable contracts with user-supplied class hash**: A common pattern is a proxy or upgradeable contract whose `upgrade(new_class_hash)` function calls `replace_class`. A malicious caller passes any arbitrary felt (e.g., `0xdeadbeef`) as `new_class_hash`. The OS accepts it without validation.
- **Buggy contract logic**: A contract that computes the new class hash incorrectly (e.g., off-by-one, wrong storage slot) can accidentally write an undeclared hash, permanently bricking itself.

No privileged role, leaked key, or operator cooperation is required. Any unprivileged transaction sender who can invoke a contract's upgrade path can trigger this.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify it is present in `contract_class_changes` with a non-zero compiled class hash. In Cairo pseudocode:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the invariant already enforced during `execute_declare_transaction`, where `prev_value=0` and `assert_not_zero(compiled_class_hash)` together guarantee that only valid, declared classes enter the class tree. [3](#0-2) 

---

### Proof of Concept

1. Deploy contract `Vault` that holds user funds and exposes `upgrade(new_class_hash: felt)` which calls the `replace_class` syscall with the caller-supplied value.
2. Attacker calls `Vault.upgrade(0xdeadbeef)`.
3. The OS executes `execute_replace_class`:
   - Reads `class_hash = 0xdeadbeef` from the syscall request.
   - Skips the missing declared-class check (TODO line 898).
   - Writes `StateEntry(class_hash=0xdeadbeef, ...)` into `contract_state_changes`.
   - Logs the old class hash in the revert log (transaction is not reverted, so this log is never replayed).
4. Block is proven. The state root now records `Vault.class_hash = 0xdeadbeef`.
5. Any future transaction calling `Vault` requires the OS to execute class `0xdeadbeef`. No such class exists in `contract_class_changes`; the sequencer cannot produce a valid proof for any such transaction.
6. All funds in `Vault`'s storage are permanently inaccessible. [4](#0-3)

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
