### Title
Stale `feemarketParams` and `evmDenom` Captured at Ante-Handler Construction Cause Cosmos-Tx Fee Mis-Accounting — (`evmd/ante/handler_options.go`)

---

### Summary

`newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712` snapshot `evmParams` and `feemarketParams` (including the live `BaseFee`) once at construction time and close over them permanently. The `NewDynamicFeeChecker` returned by these functions therefore validates every subsequent Cosmos transaction against the **genesis-time base fee**, not the current one. Because `BaseFee` is updated every block by `feemarket.BeginBlock`, the checker is stale from block 2 onward. Any user can submit a Cosmos-wrapped transaction with fees calibrated to the old (lower) base fee and have it accepted and committed, permanently under-collecting protocol fees.

---

### Finding Description

`newEthAnteHandler` is a live closure: on every invocation it calls `options.EvmKeeper.EVMBlockConfig(ctx, ...)`, which reads the current `Params` and `FeeMarketParams` from the KV store (or the per-block object-store cache that is reset each block). [1](#0-0) 

`newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712` do the opposite: they call `GetParams` / `GetFeeMarketParams` **once** at construction time and capture the results in local variables. [2](#0-1) [3](#0-2) 

Those local variables are then passed **by pointer** into `NewDynamicFeeChecker`: [4](#0-3) 

Inside `NewDynamicFeeChecker`, the fee-validation logic reads `feemarketParams.BaseFee` (via `types.GetBaseFee`) and `evmParams.EvmDenom` from those captured pointers — which point to the heap-escaped copies of the construction-time snapshots, never to the live store. [5](#0-4) [6](#0-5) 

`BaseFee` is updated in the KV store every block by `feemarket.BeginBlock → SetBaseFee → SetParams`: [7](#0-6) [8](#0-7) 

The stale `feemarketParams.BaseFee` in the Cosmos ante handler is therefore the genesis value for the entire lifetime of the node process.

The same stale `evmDenom` string is also passed to `NewMinGasPriceDecorator`: [9](#0-8) 

---

### Impact Explanation

After the first block in which gas usage diverges from the target, `BaseFee` moves away from its genesis value. From that point on, every Cosmos transaction (including EIP-712 wrapped Cosmos txs) is fee-validated against the **genesis** base fee, not the current one.

Concrete scenario — base fee rises from 1 Gwei to 2 Gwei:

1. User submits a Cosmos tx with `fee = 1 Gwei × gasLimit`.
2. `NewDynamicFeeChecker` computes `feeCap = fee / gas = 1 Gwei`; `baseFeeInt = feemarketParams.BaseFee = 1 Gwei` (stale). Check passes.
3. `effectiveFee = 1 Gwei × gas` is returned and deducted.
4. The fee collector receives half the protocol-required fee; the block proposer and stakers are under-compensated.

This matches the allowed High impact: *"ante handler… bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

A secondary impact: if governance later changes `EvmDenom` via `MsgUpdateParams`, the stale `evmDenom` causes `feeCoins.AmountOf(oldDenom)` to return zero for any tx paying in the new denom, making `feeCap = 0 < baseFee` and rejecting all Cosmos txs — a complete Cosmos-tx DoS. [10](#0-9) 

---

### Likelihood Explanation

The base fee changes every block whenever block gas usage differs from the target. No governance action, privileged role, or special condition is required — the stale-base-fee path is active from block 2 onward on any live chain. Any unprivileged user who observes the current base fee and submits a Cosmos tx with fees calibrated to the genesis base fee will exploit this automatically.

---

### Recommendation

Replace the construction-time snapshot with a live read inside the returned closure, mirroring `newEthAnteHandler`:

```go
func newCosmosAnteHandler(options HandlerOptions, extra ...sdk.AnteDecorator) sdk.AnteHandler {
    return func(ctx sdk.Context, tx sdk.Tx, simulate bool) (sdk.Context, error) {
        // Read fresh params on every invocation
        blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
        if err != nil { ... }
        evmParams    := &blockCfg.Params
        feemarketParams := &blockCfg.FeeMarketParams
        evmDenom     := evmParams.EvmDenom
        ethCfg       := blockCfg.ChainConfig

        var txFeeChecker ante.TxFeeChecker
        if options.DynamicFeeChecker {
            txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, evmParams, feemarketParams)
        }
        // build and run decorator chain with live values …
    }
}
```

Apply the same fix to `newLegacyCosmosAnteHandlerEip712`.

---

### Proof of Concept

1. Start a chain; note genesis `BaseFee = B₀`.
2. Submit enough EVM transactions to push block gas above the target for several blocks; observe `BaseFee` rise to `B₁ > B₀` via `eth_gasPrice` or `feemarket` query.
3. Construct a Cosmos `MsgSend` wrapped in a standard SDK tx with `fee = B₀ × gasLimit` (using the genesis base fee).
4. Broadcast. The `NewDynamicFeeChecker` inside `newCosmosAnteHandler` computes `feeCap = B₀`, compares against its stale `baseFeeInt = B₀`, passes, and deducts only `B₀ × gas` from the sender.
5. Query the fee-collector balance; it received `B₀ × gas` instead of the required `B₁ × gas`, confirming fee under-collection. [11](#0-10) [12](#0-11) [13](#0-12)

```go
func NewDynamicFeeChecker(ethCfg *params.ChainConfig, evmParams *types.Params, feemarketParams *feemarkettypes.Params) authante.TxFeeChecker {
	return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
		feeTx, ok := tx.(sdk.FeeTx)
		if !ok {
			return nil, 0, fmt.Errorf("tx must be a FeeTx")
		}

		if ctx.BlockHeight() == 0 {
			// genesis transactions: fallback to min-gas-price logic
			return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
		}

		denom := evmParams.EvmDenom

		baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
		if baseFee == nil {
			// london hardfork is not enabled: fallback to min-gas-prices logic
			return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
		}

		// default to `MaxInt64` when there's no extension option.
		maxPriorityPrice := sdkmath.NewInt(math.MaxInt64)

		// get the priority tip cap from the extension option.
		if hasExtOptsTx, ok := tx.(authante.HasExtensionOptionsTx); ok {
			for _, opt := range hasExtOptsTx.GetExtensionOptions() {
				if extOpt, ok := opt.GetCachedValue().(*ethermint.ExtensionOptionDynamicFeeTx); ok {
					maxPriorityPrice = extOpt.MaxPriorityPrice
					break
				}
			}
		}

		if maxPriorityPrice.Sign() == -1 {
			return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, "priority fee is negative")
		}

		gas := feeTx.GetGas()
		feeCoins := feeTx.GetFee()
		fee := feeCoins.AmountOf(denom)

		feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
		baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)

		if feeCap.LT(baseFeeInt) {
			return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, "insufficient gas prices; got: %s required: %s", feeCap, baseFeeInt)
		}

		// calculate the effective gas price using the EIP-1559 logic.
		effectivePrice := sdkmath.NewIntFromBigInt(types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))

		// NOTE: create a new coins slice without having to validate the denom
		effectiveFee := sdk.Coins{
			{
				Denom:  denom,
				Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas)),
			},
		}

		bigPriority := effectivePrice.Sub(baseFeeInt).Quo(types.DefaultPriorityReduction)
		priority := int64(math.MaxInt64)

		if bigPriority.IsInt64() {
			priority = bigPriority.Int64()
		}

		return effectiveFee, priority, nil
	}
}
```

```go
func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
}
```
