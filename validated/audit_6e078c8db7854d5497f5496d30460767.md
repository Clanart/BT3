### Title
Unvalidated Class Hash in `execute_replace_class` Enables Permanent Fund Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary felt value as the new class hash without verifying it corresponds to a declared contract class. A contract deployer can exploit this to permanently freeze funds held in a contract by replacing its class with an undeclared hash, rendering the contract permanently uncallable.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `REPLACE_CLASS` syscall. After deducting gas, it reads the requested class hash directly from the syscall request and updates the contract's state entry — with no check that the new class hash is actually declared on-chain.

The code at lines 896–910 reads:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The inline TODO comment explicitly acknowledges the missing validation. The OS writes the arbitrary `class_hash` into the contract's `StateEntry` and commits it to the state diff without any cross-reference against `contract_class_changes` (the set of classes declared in the current block) or any previously declared class.

Once the state transition is proven and committed to L1, the contract's on-chain class hash is permanently set to the undeclared value. Any subsequent call to that contract will fail at class resolution time because no bytecode exists for the hash. Since `replace_class` can only be invoked by the contract itself, and the contract can no longer execute, the class hash cannot be corrected. All assets stored in the contract are permanently frozen.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

A contract whose class hash is set to an undeclared value becomes permanently uncallable. No entry point — including `replace_class` itself — can be invoked to recover the situation. Any ERC-20 balances, ETH, or other assets held in the contract's storage are irrecoverably locked. The state transition is proven by the OS and accepted by the L1 verifier, so there is no on-chain mechanism to reverse it.

---

### Likelihood Explanation

**Medium.**

The attack requires a contract deployer to publish a contract that (a) accepts user deposits and (b) subsequently calls `replace_class` with an undeclared hash. This is a realistic rug-pull vector: a malicious deployer can attract user funds under the guise of a legitimate vault or token contract, then trigger the syscall to freeze all deposited assets. No privileged role, leaked key, or operator cooperation is required — only the ability to deploy a contract and submit a transaction, both of which are permissionless on StarkNet.

---

### Recommendation

Before updating `contract_state_changes`, verify that `request.class_hash` is present in either `contract_class_changes` (declared in the current block) or the existing on-chain class trie. The TODO comment at line 898 already identifies this as a required check. Concretely, add a lookup analogous to the one performed in `execute_declare_transaction` (which uses `dict_update` with `prev_value=0` to enforce uniqueness) to confirm the target class hash has a corresponding compiled class hash entry before allowing the replacement. [2](#0-1) 

---

### Proof of Concept

1. **Attacker deploys** a contract (`ClassA`) whose `__execute__` entry point accepts deposits and records balances in storage.
2. **Users deposit funds** into the contract; balances are stored in the contract's storage trie.
3. **Attacker submits** a transaction that calls the contract's internal logic, which in turn issues a `REPLACE_CLASS` syscall with `class_hash = 0xdeadbeef` (an arbitrary felt that has never been declared).
4. **The OS executes** `execute_replace_class`. Gas is deducted, the TODO check is absent, and `dict_update` writes `class_hash=0xdeadbeef` into the contract's `StateEntry`. [3](#0-2) 

5. **State is committed.** `state_update` squashes the dict and computes the new global root, which is submitted to L1 as a valid proof. [4](#0-3) 

6. **All subsequent calls** to the contract fail at class resolution because `0xdeadbeef` has no associated compiled class. The contract cannot call `replace_class` again to fix itself. User funds are permanently frozen.

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
