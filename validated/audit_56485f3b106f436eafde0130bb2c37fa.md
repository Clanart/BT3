### Title
`ProofOfInclusion.valid()` Always Returns `true` for Empty `layers` Array, Enabling Forged DataLayer Inclusion Proofs — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` contains no guard against an empty `layers` field. When `layers` is empty, `root_hash()` degenerates to returning `node_hash`, and `valid()` reduces to the tautology `node_hash == node_hash`, which is unconditionally `true`. An attacker who can supply a deserialized `ProofOfInclusion` object (via the `Streamable`/Python/wasm boundary) can forge a structurally "valid" proof of inclusion for any chosen hash value against any chosen root, bypassing the DataLayer Merkle proof check entirely.

### Finding Description

`ProofOfInclusion` is a `Streamable`-derived struct exposed through Python bindings:

```rust
// crates/chia-datalayer/src/merkle/proof_of_inclusion.rs
pub struct ProofOfInclusion {
    pub node_hash: Hash,
    pub layers: Vec<ProofOfInclusionLayer>,
}
```

`root_hash()` has a special case when `layers` is empty:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash   // ← returns node_hash when layers is empty
    }
}
```

`valid()` performs no pre-check on `layers.len()`:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {          // ← loop body never executes when layers == []
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()    // ← node_hash == node_hash → always true
}
```

When `layers` is `[]`:
- The loop body never executes.
- `existing_hash` stays as `self.node_hash`.
- `self.root_hash()` returns `self.node_hash`.
- The final comparison is `self.node_hash == self.node_hash` → **unconditionally `true`**.

The struct is constructible from untrusted bytes via `from_bytes` / `parse_rust` (both exposed in the Python stub at `wheel/python/chia_rs/datalayer.pyi`) and via `__new__` in Python. An attacker can craft:

```python
forged = ProofOfInclusion(node_hash=target_root, layers=[])
assert forged.valid()          # True — no verification performed
assert forged.root_hash() == target_root  # True — tautology
```

This passes both the internal consistency check and the root comparison that callers are expected to perform.

### Impact Explanation

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."**

A caller that receives a `ProofOfInclusion` from an untrusted source (e.g., a DataLayer peer, a wallet RPC response, or a cross-language binding call) and checks `proof.valid() and proof.root_hash() == expected_root` can be deceived into accepting a forged proof of inclusion. The attacker sets `node_hash = expected_root` and `layers = []`; both checks pass. The caller then incorrectly concludes that `expected_root` is a leaf present in the tree, enabling false state attestation, unauthorized DataLayer state acceptance, or corrupted delta synchronization decisions.

### Likelihood Explanation

`ProofOfInclusion` is a first-class serializable type exposed to Python and wasm consumers. Any DataLayer client that receives proofs over the network or from an untrusted store and validates them using only `valid()` + `root_hash()` is directly exploitable. The construction requires zero cryptographic knowledge — only the ability to serialize a two-field struct with an empty list.

### Recommendation

Add an explicit guard at the start of `valid()` that rejects empty `layers` unless the tree is known to be a single-node tree (in which case the proof format itself should be different). The simplest fix:

```rust
pub fn valid(&self) -> bool {
    if self.layers.is_empty() {
        // A single-leaf tree has no parent layers; the root IS the leaf hash.
        // Only valid if the caller explicitly expects a single-node tree.
        // For general use, an empty layers proof is always invalid.
        return false;
    }
    // ... existing logic
}
```

Alternatively, `valid()` should accept an explicit `expected_root: &Hash` parameter and compare against it directly, removing the ambiguity introduced by `root_hash()` doubling as both a getter and an implicit validation target.
