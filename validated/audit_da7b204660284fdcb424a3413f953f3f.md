### Title
Empty `check_is_reverted` Leaves `is_reverted` Hint Unconstrained, Enabling Prover to Suppress L1→L2 Message Consumption and Freeze Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/execution_constraints.cairo`)

---

### Summary

`check_is_reverted` in `execution_constraints.cairo` is a completely empty function. It is the direct analog of the MysteryBox `fulfilled` flag that was never set: a guard that is supposed to enforce a state-transition invariant but performs zero Cairo constraints. Because the `is_reverted` local variable is filled exclusively by a hint (`%{ IsReverted %}`) and `check_is_reverted` asserts nothing, the prover can freely choose any value for `is_reverted` without invalidating the proof. For L1-handler transactions this means the prover can suppress `consume_l1_to_l2_message`, leaving the corresponding L1 funds permanently locked.

---

### Finding Description

**Root cause — `check_is_reverted` is a no-op:** [1](#0-0) 

```cairo
func check_is_reverted(is_reverted: felt) {
    return ();
}
```

The function accepts `is_reverted` and immediately returns. No `assert`, no range-check, no comparison — the Cairo constraint system sees nothing.

**Call site in `execute_invoke_function_transaction`:** [2](#0-1) 

```cairo
local is_reverted;
%{ IsReverted %}
check_is_reverted(is_reverted);          // ← does nothing
if (is_reverted == FALSE) {
    // Execute only non-reverted transactions.
    ...
    non_reverting_select_execute_entry_point_func(...);
} else {
    // stack-alignment tempvars only, no execution
}
// Fee is charged regardless.
charge_fee(...);
```

**Parallel unconstrained path in `execute_l1_handler_transaction`:** [3](#0-2) 

```cairo
local is_reverted;
%{ IsReverted %}
if (is_reverted != FALSE) {
    %{ EndTx %}
    return ();          // ← returns WITHOUT consume_l1_to_l2_message
}
...
consume_l1_to_l2_message(execution_context=tx_execution_context, nonce=nonce);
```

Here `check_is_reverted` is not even called; `is_reverted` is used raw from the hint. The message-consumption step is the only place the OS records that an L1→L2 message was processed.

**Why the proof remains valid:** In Cairo, `local` variables are unconstrained unless an `assert` (or equivalent builtin operation) ties them to other constrained values. The hint `%{ IsReverted %}` fills the cell; `check_is_reverted` adds no constraint; the `if` branch in Cairo is not a constraint. The STARK verifier therefore accepts any value of `is_reverted` the prover supplies.

**Contrast with the correctly constrained inner path:** When `is_reverted == FALSE` and `non_reverting_select_execute_entry_point_func` is called, the inner execution result *is* constrained: [4](#0-3) 

```cairo
let (is_reverted, ...) = select_execute_entry_point_func{...}(...);
assert is_reverted = 0;
```

So the prover *cannot* lie about a reverted transaction being successful (the inner assert would fail). But the prover *can* lie in the other direction: claim any successful transaction is reverted, skip the execute step entirely, and produce a valid proof.

---

### Impact Explanation

**Critical — Permanent freezing of funds (L1-handler path):**

An L1→L2 message carries ETH/ERC-20 value locked in the StarkNet Core contract on L1. The OS records consumption of that message only inside `consume_l1_to_l2_message`: [5](#0-4) 

If the prover sets `is_reverted = TRUE` for an L1-handler transaction, the function returns before reaching `consume_l1_to_l2_message`. The OS output never records the message as consumed. The L1 contract therefore never releases the locked funds. Because the proof is valid, the L1 verifier accepts the state update. The funds are permanently frozen with no on-chain recourse.

**Critical — Direct loss of funds (invoke path):**

For invoke transactions, skipping the execute step while still running `charge_fee` means the user's fee is deducted but their intended state change (e.g., a token transfer) never happens. The fee is a direct, unrecoverable loss.

---

### Likelihood Explanation

The prover is the sequencer. In the current StarkNet architecture the sequencer is a single operator. The OS is designed to be trustless — the proof is supposed to constrain the sequencer so that L1 accepts only correct state transitions. The empty `check_is_reverted` breaks that guarantee: a proof that suppresses L1-handler execution is indistinguishable from a correct proof. Any L1/L2 message sender whose handler is silently marked reverted loses their funds with no ability to detect or contest it from on-chain evidence alone.

---

### Recommendation

`check_is_reverted` must add a Cairo constraint that ties `is_reverted` to the actual execution outcome. The standard pattern is to run the entry point unconditionally, capture the real `is_reverted` result from `select_execute_entry_point_func`, and assert it equals the hint-supplied value:

```cairo
func check_is_reverted(is_reverted: felt) {
    // Constrain is_reverted to {0, 1}.
    assert is_reverted * (1 - is_reverted) = 0;
    // Additional: tie to actual execution result via the revert log or
    // a dedicated boolean output from select_execute_entry_point_func.
    return ();
}
```

For L1-handler transactions, a symmetric call to `check_is_reverted` (or an inline assertion) must be added before the early-return branch.

---

### Proof of Concept

1. User sends an L1→L2 message depositing 10 ETH; the StarkNet Core contract locks the ETH.
2. The sequencer includes the corresponding L1-handler transaction in a block.
3. When generating the OS execution trace, the prover sets `is_reverted = 1` (non-zero) for that transaction via the `%{ IsReverted %}` hint.
4. `check_is_reverted(1)` returns immediately — no constraint fires.
5. `execute_l1_handler_transaction` hits `if (is_reverted != FALSE) { return (); }` and exits before `consume_l1_to_l2_message`.
6. The OS output segment contains no record of the message being consumed.
7. The STARK proof is generated and verified successfully on L1 (no violated constraint).

func consume_l1_to_l2_message{outputs: OsCarriedOutputs*}(
    execution_context: ExecutionContext*, nonce: felt
) {
    assert_not_zero(execution_context.calldata_size);
    // The payload is the calldata without the from_address argument (which is the first).
    let payload: felt* = execution_context.calldata + 1;
    tempvar payload_size = execution_context.calldata_size - 1;

    tempvar execution_info = execution_context.execution_info;

    // Write the given transaction to the output.
    assert [outputs.messages_to_l2] = MessageToL2Header(
        from_address=[execution_context.calldata],
        to_address=execution_info.contract_address,
        nonce=nonce,
        selector=execution_info.selector,
        payload_size=payload_size,
    );

    let message_payload = cast(outputs.messages_to_l2 + MessageToL2Header.SIZE, felt*);
    memcpy(dst=message_payload, src=payload, len=payload_size);

    let (outputs) = os_carried_outputs_new(
        messages_to_l1=outputs.messages_to_l1,
        messages_to_l2=outputs.messages_to_l2 + MessageToL2Header.SIZE +
        outputs.messages_to_l2.payload_size,
    );
    return ();
