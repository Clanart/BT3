### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary felt value as a new class hash without verifying that the hash corresponds to a declared class. The OS itself acknowledges this gap with an explicit TODO comment. An attacker who can trigger a contract's upgrade path (e.g., via a governance mechanism, a compromised admin, or a contract that exposes `replace_class` to user-supplied input) can supply an undeclared class hash, permanently rendering the contract non-executable and freezing all token balances it holds.

---

### Finding Description

In `execute_replace_class`, after deducting gas, the OS reads the current `StateEntry` for the calling contract and writes a new `StateEntry` with the caller-supplied `class_hash`:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The `class_hash` field of the new `StateEntry` is taken directly from `request.class_hash` — a value fully controlled by the transaction sender — with no cross-reference against `contract_class_changes` (the dictionary of declared classes) or any other registry. The TODO at line 898 explicitly acknowledges this missing check.

By contrast, `execute_declare_transaction` enforces that a class may only be declared once and that its hash is a valid Sierra class hash:

```cairo
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [2](#0-1) 

`execute_replace_class` performs no equivalent check. The OS will commit a `StateEntry` whose `class_hash` has no corresponding compiled class, making the contract permanently non-executable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

On StarkNet, ERC-20 token balances are stored in the token contract's own storage, keyed by the holder's address. When contract `A` holds tokens, those balances live in the token contract's storage under key `A`. To move them, contract `A` must call the token contract's `transfer` entry point.

If contract `A`'s class hash is replaced with an undeclared value:

1. The OS commits the new `StateEntry` with the bogus class hash into the Patricia state tree.
2. Any subsequent transaction targeting contract `A` will attempt to look up the class hash in the compiled class facts bundle. No matching class exists.
3. No entry point of contract `A` can ever be executed again.
4. All token balances held by contract `A` are permanently inaccessible — the contract cannot call `transfer`, `approve`, or any other function to recover them.

This satisfies the **Critical: Permanent freezing of funds** impact criterion.

---

### Likelihood Explanation

The attack surface is any contract that exposes an upgrade path accepting a caller-supplied class hash. This is a common pattern:

- Governance-controlled upgradeable contracts (DAO votes supply a new class hash).
- Admin-key-controlled proxies (a compromised or malicious admin supplies an invalid hash).
- Contracts that forward a user-provided class hash to `replace_class` without pre-validating it on-chain.

Because the OS itself performs no validation, the entire burden falls on every individual contract's upgrade logic. A single contract that fails to pre-validate the class hash (e.g., by calling `get_class_hash_at` on itself after the upgrade, or by checking the class registry) will be permanently bricked along with all funds it holds. Given the prevalence of upgradeable contracts and the absence of any OS-level guard, the likelihood of at least one such contract being exploited is high.

---

### Recommendation

Before writing the new `StateEntry`, `execute_replace_class` must verify that `request.class_hash` is present in the declared class registry. Concretely, the OS should perform a `dict_read` on `contract_class_changes` (or an equivalent global class registry) for `request.class_hash` and assert that the returned compiled class hash is non-zero. This mirrors the fix applied in MarinateV2: adding a `require` check for the residual balance before allowing removal.

```cairo
// Proposed fix (pseudocode):
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

---

### Proof of Concept

1. Attacker deploys contract `Vault` that holds 1,000,000 STRK tokens and exposes an `upgrade(new_class_hash: felt)` function that calls `replace_class(new_class_hash)`.
2. Attacker calls `Vault.upgrade(0xdeadbeef)` — an arbitrary felt that is not a declared class hash.
3. The OS executes `execute_replace_class`:
   - Line 896: reads current `StateEntry` for `Vault`.
   - Line 898: **skips** the declared-class check (TODO).
   - Lines 902–910: writes a new `StateEntry` with `class_hash=0xdeadbeef`.
4. The block is proven and the state root is updated. `Vault`'s `StateEntry` now permanently records `class_hash=0xdeadbeef`.
5. Any future transaction targeting `Vault` fails to find a compiled class for `0xdeadbeef`. No entry point can execute.
6. The 1,000,000 STRK tracked in the STRK token contract under key `Vault` are permanently frozen — no `transfer` call can ever be made on behalf of `Vault`. [3](#0-2)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    let revert_log = &revert_log[1];

    return ();
}
