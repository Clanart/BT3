### Title
Meta Transaction V0 Hash Omits Nonce, Enabling Signature Replay for Direct Loss of Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/transaction_hash/transaction_hash.cairo`)

---

### Summary

`compute_meta_tx_v0_hash` computes the cryptographic commitment for a meta transaction v0 without including any nonce. The OS then hardcodes `nonce=0` in the `TxInfo` delivered to the target account contract. Because the hash is fully determined by `(contract_address, selector, calldata, chain_id)`, any attacker who observes a valid meta-tx-v0 signature on-chain can replay it inside a fresh outer transaction, causing the target account to re-execute the same action — including token transfers — without the account holder's consent.

---

### Finding Description

**Root cause — missing nonce in hash:**

`compute_meta_tx_v0_hash` delegates to `deprecated_get_transaction_hash` with `additional_data_size=0`, explicitly passing no nonce:

```cairo
// transaction_hash.cairo lines 295-315
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(
    contract_address: felt,
    entry_point_selector: felt,
    calldata: felt*,
    calldata_size: felt,
    chain_id: felt,
) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        ...
        additional_data_size=0,          // ← nonce is absent
        additional_data=cast(0, felt*),
    );
    return tx_hash;
}
``` [1](#0-0) 

**Root cause — nonce hardcoded to 0 in TxInfo:**

In `execute_meta_tx_v0`, the `TxInfo` constructed for the target contract's `__execute__` call unconditionally sets `nonce=0`:

```cairo
// syscall_impls.cairo lines 343-363
tempvar new_tx_info = new TxInfo(
    version=0,
    account_contract_address=contract_address,
    max_fee=0,

    signature_end=request.signature_end,
    transaction_hash=meta_tx_hash,
    chain_id=old_tx_info.chain_id,
    nonce=0,                             // ← always zero, never checked
    ...
