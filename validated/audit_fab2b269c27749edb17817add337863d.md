### Title
Silent `proving_gas` Underflow in `sierra_gas_to_steps_gas` Produces Wrong Block Capacity Accounting — (`File: crates/blockifier/src/bouncer.rs`)

### Summary

When `cairo_primitives_gas` (builtin contribution computed with Sierra costs) exceeds `vm_resources_sierra_gas`, `sierra_gas_to_steps_gas` silently clamps the result to `GasAmount::ZERO` instead of propagating an error. The wrong `steps_proving_gas = 0` then flows through `proving_gas_from_cairo_primitives_and_sierra_gas` → `compute_proving_gas` → `get_tx_weights` → `Bouncer::try_update` → `BlockExecutionArtifacts::bouncer_weights` → `DecisionReachedResponse::central_objects::bouncer_weights` → `CentralBouncerWeights`. The bouncer enforces block capacity using this understated `proving_gas`, so it accepts transactions that actually exceed the prover's capacity. The resulting block is committed to storage and forwarded to the prover, which then fails.

### Finding Description

`sierra_gas_to_steps_gas` is the sequencer analog of `totalAssetValue()` in the external report: it is a core accounting primitive whose silent failure propagates to every downstream consumer. [1](#0-0) 

When `cairo_primitives_gas > sierra_gas`, the subtraction underflows and the closure returns `GasAmount::ZERO`:

```
steps_proving_gas = GasAmount::ZERO   // wrong; should be negative → error
```

This wrong value is immediately consumed by `proving_gas_from_cairo_primitives_and_sierra_gas`: [2](#0-1) 

```
total_proving_gas = ZERO + cairo_primitives_proving_gas
                  < correct_steps_proving_gas + cairo_primitives_proving_gas
```

`compute_proving_gas` passes this understated value up to `get_tx_weights`: [3](#0-2) 

`get_tx_weights` stores it in `BouncerWeights.proving_gas`: [4](#0-3) 

`Bouncer::try_update` then uses this wrong weight to check block capacity: [5](#0-4) 

Because `proving_gas` is understated, `has_room` returns `true` for transactions that actually exceed the prover's budget. The block is built, committed, and the wrong `proving_gas` is forwarded to the centralized pipeline: [6](#0-5) 

The `CentralBouncerWeights.proving_gas` field is the authoritative resource figure consumed by the Python proving pipeline. A wrong value here means the pipeline receives a block whose actual proving cost exceeds the declared budget.

### Impact Explanation

**Scope: Critical — Incorrect bouncer/resource accounting with economic impact.**

The bouncer is the sole gate that prevents over-capacity blocks from reaching the prover. When `proving_gas` is understated, the gate is bypassed silently. The committed block cannot be proven; the sequencer has wasted execution resources and the block finalization pipeline stalls until the block is reverted or the prover is restarted. Multiple downstream functions are affected by the wrong value:

- `Bouncer::try_update` — wrong capacity check
- `BlockExecutionArtifacts::new` — wrong `bouncer_weights` stored
- `Batcher::decision_reached` — wrong weights committed to storage
- `DecisionReachedResponse` — wrong `central_objects.bouncer_weights` sent to Cende
- `CentralBouncerWeights` — wrong `proving_gas` forwarded to the Python proving pipeline [7](#0-6) 

### Likelihood Explanation

The underflow requires `cairo_primitives_to_gas(cairo_primitives_for_proving_gas, sierra_builtin_gas_costs) > vm_resources_sierra_gas`. The `cairo_primitives_for_proving_gas` aggregates Patricia-update builtins, OS-computation builtins, and `tx_cairo_primitives_counters`: [8](#0-7) 

`vm_resources_sierra_gas` is built from `extended_execution_resources_to_gas` (steps + memory holes + builtins) plus `tx_resources.computation.sierra_gas`. If `tx_cairo_primitives_counters` carries builtin counts that are not fully reflected in `vm_resources` (e.g., due to a discrepancy between how Cairo primitives and VM builtins are tracked), or if a future opcode is assigned a non-zero Sierra cost, the underflow becomes reachable. The developers already acknowledge the condition with an explicit `log::debug!` message, confirming it is a known edge case rather than an impossible state.

### Recommendation

1. Replace the silent `GasAmount::ZERO` fallback with a hard error (`panic!` or `TransactionExecutionError`) so that the condition is surfaced immediately rather than silently corrupting downstream accounting.
2. Add an invariant assertion in `get_tx_weights` that `total_proving_gas >= cairo_primitives_proving_gas` before storing it in `BouncerWeights`.
3. Add a test that exercises the underflow path and verifies the error is propagated rather than swallowed.

### Proof of Concept

1. Craft a transaction whose `tx_cairo_primitives_counters` contains a large number of high-cost builtins (e.g., `ecdsa` at 10 561 gas/unit) that are not fully reflected in `vm_resources_sierra_gas` (e.g., because `sierra_gas` from `tx_resources.computation.sierra_gas` is zero and the step count is minimal).
2. Call `get_tx_weights` with these inputs. `cairo_primitives_to_gas(cairo_primitives_for_proving_gas, sierra_builtin_gas_costs)` exceeds `vm_resources_sierra_gas`.
3. `sierra_gas_to_steps_gas` logs the debug message and returns `GasAmount::ZERO`.
4. `total_proving_gas` is now `cairo_primitives_proving_gas` only — missing the steps contribution.
5. `Bouncer::try_update` accepts the transaction because the understated `proving_gas` fits within `block_max_capacity.proving_gas`.
6. The block is committed with `BlockExecutionArtifacts.bouncer_weights.proving_gas` set to the wrong value.
7. The prover receives a block whose actual proving cost exceeds the declared budget and fails. [9](#0-8) [2](#0-1) [10](#0-9)

fn proving_gas_from_cairo_primitives_and_sierra_gas(

    proving_builtin_gas_costs: &BuiltinGasCosts,

}

/// Computes the steps gas by subtracting the builtins' contribution from the Sierra gas.
pub fn sierra_gas_to_steps_gas(
    sierra_gas: GasAmount,
    cairo_primitives_counters: &CairoPrimitiveCounterMap,
    sierra_builtin_gas_costs: &BuiltinGasCosts,
) -> GasAmount {
    let cairo_primitives_gas =
        cairo_primitives_to_gas(cairo_primitives_counters, sierra_builtin_gas_costs);

    sierra_gas.checked_sub(cairo_primitives_gas).unwrap_or_else(|| {
        log::debug!(
            "Sierra gas underflow: cairo primitives gas exceeds total. Sierra gas: \
             {sierra_gas:?}, Cairo primitives gas: {cairo_primitives_gas:?}, Cairo primitives: \
             {cairo_primitives_counters:?}"
        );
        GasAmount::ZERO
    })
}
