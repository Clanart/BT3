### Title
Missing Declared-Class Guard in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the new class hash supplied by the caller corresponds to a previously declared contract class. Any contract can call `replace_class` with an arbitrary, undeclared class hash. The OS will commit this invalid class hash to the state without error, permanently rendering the contract unexecutable and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class` (lines 878–916 of `syscall_impls.cairo`), after deducting gas and reading the current `StateEntry`, the function unconditionally writes the caller-supplied `class_hash` into `contract_state_changes`:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges the missing guard. There is no assertion or lookup confirming that `class_hash` exists in `contract_class_changes` (current-block declarations) or in the global declared-class state. The OS proof remains valid regardless of whether the supplied class hash was ever declared, because the proof system only validates compiled class facts that were *actually used during execution* in the current block — not the class hash stored in a contract's state entry.

This is the direct analog of the external report's pattern: a state-mutating function (`replace_class` ↔ `withdrawLock`) that is missing a required guard (`class_hash is declared` ↔ `require(lockCrv, "!lock")`), allowing it to be called with invalid inputs that corrupt protocol state.

---

### Impact Explanation

Once the OS commits a state root containing a contract whose `class_hash` points to an undeclared class:

1. Every subsequent transaction that attempts to call that contract will fail at the proof-generation stage, because the prover cannot supply a valid compiled-class fact for the undeclared hash.
2. The contract becomes permanently unexecutable — no upgrade, no withdrawal, no recovery path exists at the protocol level.
3. All ERC-20 tokens, ETH, or other assets held by the contract are permanently frozen.

This matches the **Critical — Permanent freezing of funds** impact category.

---

### Likelihood Explanation

- The `replace_class` syscall is available to every contract without restriction; `execute_syscalls` dispatches to it for any caller that emits `REPLACE_CLASS_SELECTOR`.
- A malicious contract deployer can: (a) advertise a contract as a vault or escrow, (b) accept user deposits, then (c) call `replace_class(0xdeadbeef)` in a single transaction. The OS will include this in a valid proof, commit the state, and the funds are frozen.
- A buggy contract can trigger the same outcome accidentally.
- The sequencer has no OS-enforced reason to reject such a transaction; the proof is sound from the verifier's perspective. [2](#0-1) 

---

### Recommendation

Before writing the new `StateEntry`, add a lookup into the declared-class state to confirm the supplied `class_hash` is valid. Concretely, the OS should assert that `class_hash` resolves to a non-zero compiled class hash — either via a `dict_read` on `contract_class_changes` (for classes declared in the current block) or via a storage read on the global class-registry contract (for classes declared in prior blocks). The fix mirrors the recommendation in the external report: add the missing `require`-equivalent guard before the state mutation proceeds.

---

### Proof of Concept

1. **Deploy** a contract `Vault` that accepts token deposits from users. `Vault` holds 1 000 000 STRK.
2. **Craft** a transaction that calls `Vault.__execute__`, which internally issues the `replace_class` syscall with `class_hash = 0xdeadbeef` (a felt that was never declared on-chain).
3. **Submit** the transaction. The OS executes `execute_replace_class`:
   - Gas is deducted ✓
   - `state_entry` is read ✓
   - `new_state_entry` is written with `class_hash = 0xdeadbeef` — **no declared-class check** ✓
4. The OS generates a valid STARK proof. The L1 verifier accepts it. The state root now encodes `Vault.class_hash = 0xdeadbeef`.
5. Any future transaction targeting `Vault` causes the prover to search for a compiled class for `0xdeadbeef`. None exists. Proof generation fails. The transaction cannot be included in any block.
6. The 1 000 000 STRK inside `Vault` are permanently frozen. [3](#0-2)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
