### Title
Ring-Buffer Wrap-Around in EIP-2935 Block Hash History Returns Wrong Hash for Out-of-Window Blocks - (File: `x/evm/keeper/keeper.go`)

### Summary

`GetHeaderHash` reads from the EIP-2935 ring-buffer storage without verifying that the stored hash actually belongs to the requested block height. After the ring buffer wraps around (every `window` blocks), a slot at index `height % window` contains the hash of a *newer* block, not the requested one. The function returns this stale/wrong hash as if it were correct, causing the EVM `BLOCKHASH` opcode to return an incorrect value during consensus execution.

### Finding Description

`SetHeaderHash` stores the current block's hash at ring slot `blockHeight % window`: [1](#0-0) 

`GetHeaderHash` reads back by computing the same ring index and returning whatever is stored there, with no check that the stored value actually belongs to the requested height: [2](#0-1) 

The only guard is `hash.Cmp(common.Hash{}) != 0` — if the slot is non-zero, the hash is returned immediately. There is no check of the form `requestedHeight + window > currentHeight`.

`GetHashFn`, which implements the EVM `BLOCKHASH` opcode, calls `GetHeaderHash` for **all** historical block numbers without any lower-bound guard on the EIP-2935 path: [3](#0-2) 

The staking fallback path has a `lower` bound check (`num64 >= lower`), but the EIP-2935 path has none.

**Concrete wrap-around scenario** (default `window = 8191`):

| Block height | Slot written | Slot content |
|---|---|---|
| 1 | `1 % 8191 = 1` | hash(block 1) |
| 8192 | `8192 % 8191 = 1` | hash(block 8192) ← overwrites |
| 16383 | `16383 % 8191 = 1` | hash(block 16383) ← overwrites again |

At block 16384, a contract calls `BLOCKHASH(1)`. `GetHeaderHash(ctx, 1)` computes `ringIndex = 1`, reads slot 1, finds hash(block 16383) (non-zero), and returns it as the hash of block 1. The EVM spec requires `BLOCKHASH` to return `0` for blocks outside the accessible window, but instead it returns a hash from a completely different block.

### Impact Explanation

The `BLOCKHASH` opcode is used by smart contracts for commit-reveal schemes, on-chain randomness, and time-lock mechanisms. Returning a wrong non-zero hash instead of `0` for out-of-window blocks:

1. Breaks contracts that check `blockhash(n) == 0` to detect expired commitments — an attacker can supply the wrong-but-non-zero hash to bypass expiry checks.
2. Allows prediction/manipulation of "random" values derived from `BLOCKHASH` in contracts that assume out-of-window hashes are unpredictable (they are now deterministically the hash of a known block).
3. Feeds incorrect consensus-critical data into EVM state transitions on every validator node identically, meaning exploits are deterministic and reproducible across the network.

This matches **High** impact: EVM state transition bug that feeds incorrect data into execution, enabling unauthorized account/state mutation in contracts relying on `BLOCKHASH` correctness.

### Likelihood Explanation

The condition is triggered automatically after `window` (8191) blocks once the EIP-2935 history contract is deployed. Any contract calling `BLOCKHASH` on a block number older than `window` blocks will receive a wrong hash. No special privileges are required — any unprivileged EVM transaction can trigger this via the `BLOCKHASH` opcode.

### Recommendation

In `GetHeaderHash`, after reading the hash from the ring buffer, validate that the requested height is within the current window before returning it:

```go
func (k Keeper) GetHeaderHash(ctx sdk.Context, height uint64) common.Hash {
    acct := k.GetAccount(ctx, ethparams.HistoryStorageAddress)
    if acct != nil && acct.IsContract() {
        window := types.DefaultHistoryServeWindow
        params := k.GetParams(ctx)
        if params.HistoryServeWindow > 0 {
            window = params.HistoryServeWindow
        }

        // Guard: only serve hashes within the current window
        currentHeight, err := ethermint.SafeUint64(ctx.BlockHeight())
        if err == nil && currentHeight > height && currentHeight-height >= window {
            // height is outside the ring buffer window; slot has been overwritten
            goto fallback
        }

        ringIndex := height % window
        var key common.Hash
        binary.BigEndian.PutUint64(key[24:], ringIndex)
        hash := k.GetState(ctx, ethparams.HistoryStorageAddress, key)
        if hash.Cmp(common.Hash{}) != 0 {
            return hash
        }
    }
fallback:
    store := ctx.KVStore(k.storeKey)
    hashByte := store.Get(types.GetHeaderHashKey(height))
    if len(hashByte) > 0 {
        return common.BytesToHash(hashByte)
    }
    return common.Hash{}
}
```

Alternatively, add the same lower-bound guard in `GetHashFn` before calling `GetHeaderHash`, mirroring the existing `lower` check used for the staking fallback path.

### Proof of Concept

1. Deploy any contract with bytecode that executes `BLOCKHASH(1)` and returns the result.
2. Wait for the chain to produce more than `window` (8191) blocks after the EIP-2935 history contract is deployed.
3. Call the contract. Observe that `BLOCKHASH(1)` returns a non-zero hash (the hash of block `1 + 8191 = 8192` or the most recent block that maps to slot `1 % 8191`), instead of `0x0` as required by the EVM specification for out-of-window blocks.
4. A commit-reveal contract that checks `require(blockhash(commitBlock) == 0, "commitment expired")` can be bypassed because the check never triggers — the wrong non-zero hash is returned indefinitely. [4](#0-3) [5](#0-4) [6](#0-5)

func (k Keeper) GetHashFn(ctx sdk.Context, headerHashNum uint64) vm.GetHashFunc {
	return func(num64 uint64) common.Hash {
		h, err := ethermint.SafeInt64(num64)
		if err != nil {
			return common.Hash{}
		}
		upper, err := ethermint.SafeUint64(ctx.BlockHeight())
		if err != nil {
			return common.Hash{}
		}
		if upper == num64 {
			headerHash := ctx.HeaderHash()
			if len(headerHash) > 0 {
				return common.BytesToHash(headerHash)
			}
		}
		// Align check with https://github.com/ethereum/go-ethereum/blob/release/1.11/core/vm/instructions.go#L433
		var lower uint64
		if upper <= headerHashNum {
			lower = 0
		} else {
			lower = upper - headerHashNum
		}

		if upper > num64 {
			// The requested height is historical, query EIP-2935 contract storage
			headerHash := k.GetHeaderHash(ctx, num64)
			if headerHash.Cmp(common.Hash{}) != 0 {
				return headerHash
			} else if num64 >= lower {
				// Pre upgrade case
				// In case EIP-2935 is not supported and data cannot be found, we fetch historical info
				histInfo, err := k.stakingKeeper.GetHistoricalInfo(ctx, h)
				if err != nil {
					k.Logger(ctx).Debug("historical info not found", "height", h, "err", err.Error())
					return common.Hash{}
				}
				header, err := cmttypes.HeaderFromProto(&histInfo.Header)
				if err != nil {
					k.Logger(ctx).Error("failed to cast tendermint header from proto", "error", err)
					return common.Hash{}
				}
				return common.BytesToHash(header.Hash())
			}
		}
		return common.Hash{}
	}
}
