### Title
Underestimated `STORAGE_READ_GAS_COST` / `STORAGE_WRITE_GAS_COST` Omit Patricia Update and Dict-Squash Overhead, Enabling Cheap Block-Resource Exhaustion — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/constants.cairo`)

---

### Summary

`STORAGE_READ_GAS_COST` (18 070) and `STORAGE_WRITE_GAS_COST` (44 970) are charged to callers at syscall time, but the OS itself must also execute per-key Patricia-tree updates and dict-squash during state commitment — work that is explicitly **not** included in those constants (acknowledged by a developer TODO). An unprivileged transaction sender can therefore craft transactions whose user-facing gas cost is far below the actual Cairo-step cost imposed on the prover, allowing a cheap, sustained attack that exhausts the block's provable step budget and prevents the network from finalising new blocks.

---

### Finding Description

In `constants.cairo` the two storage-syscall gas constants carry an explicit acknowledgement that they are incomplete:

```
// TODO(Yoni, 1/1/2026): take into account Patricia updates and dict squash.
const STORAGE_READ_GAS_COST = 18070;
const STORAGE_WRITE_GAS_COST = 44970;
``` [1](#0-0) 

`execute_storage_read` and `execute_storage_write` in `syscall_impls.cairo` deduct exactly these constants (minus the pre-charged `SYSCALL_BASE_GAS_COST = 10 000`) from the caller's gas budget and nothing more:

```cairo
let success = reduce_syscall_gas_and_write_response_header(
    total_gas_cost=STORAGE_READ_GAS_COST, request_struct_size=StorageReadRequest.SIZE
);
``` [2](#0-1) 

```cairo
let success = reduce_syscall_gas_and_write_response_header(
    total_gas_cost=STORAGE_WRITE_GAS_COST, request_struct_size=StorageWriteRequest.SIZE
);
``` [3](#0-2) 

After all transactions in a block are executed, the OS runs `hash_contract_state_changes` for every modified contract. For each contract it calls `patricia_update_using_update_constants` (or the read-optimised variant) over every storage slot that was touched:

```cairo
local n_updates = (state_dict_end - state_dict_start) / DictAccess.SIZE;
if (should_use_read_optimized != 0) {
    patricia_update_read_optimized(..., n_updates=n_updates, height=MERKLE_HEIGHT, ...);
} else {
    patricia_update_using_update_constants(..., n_updates=n_updates, height=MERKLE_HEIGHT, ...);
}
``` [4](#0-3) 

Each Patricia update over a 251-bit-height Merkle tree is a logarithmically expensive Cairo computation (hash operations, range checks, memory writes). None of this cost is reflected in `STORAGE_READ_GAS_COST` or `STORAGE_WRITE_GAS_COST`. The gas accounting therefore systematically under-prices every storage syscall by the Patricia-update and dict-squash overhead.

The `reduce_syscall_base_gas` function confirms that the only cost deducted is `specific_base_gas_cost - SYSCALL_BASE_GAS_COST`: [5](#0-4) 

There is no secondary deduction for commitment-phase work.

---

### Impact Explanation

The Cairo VM has a hard cap on the number of steps (and builtin invocations) that can appear in a single provable block. Because the Patricia-update cost is proportional to the number of distinct storage slots touched across the block, an attacker who floods the block with storage-heavy transactions forces the prover to execute far more Cairo steps than the gas accounting predicted. Once the step cap is reached the block cannot be proved; the sequencer must either discard the block or stall. Repeated across consecutive blocks this constitutes a **sustained network halt** — the allowed High impact "Network not being able to confirm new transactions."

---

### Likelihood Explanation

Any unprivileged account can issue `storage_read` / `storage_write` syscalls. The attacker pays only the underestimated gas price (net 8 070 units per read after the base-gas refund), while imposing unbounded Patricia-update work on the prover. The attack requires no privileged role, no leaked key, and no operator cooperation. The TODO comment confirms the gap is known but unresolved in the current codebase.

---

### Recommendation

1. Measure the actual Cairo-step cost of `patricia_update_using_update_constants` and `patricia_update_read_optimized` per storage slot at the configured Merkle height, and add that cost to `STORAGE_READ_GAS_COST` and `STORAGE_WRITE_GAS_COST`.
2. Similarly account for the dict-squash cost that is proportional to the number of accesses per key.
3. Remove the TODO and enforce the corrected constants before the next protocol upgrade.

---

### Proof of Concept

1. Attacker deploys a contract with an entry point that loops `N` times, each iteration calling `storage_write` to a distinct key.
2. Attacker submits the transaction with `max_l2_gas = N × STORAGE_WRITE_GAS_COST` — a cost the attacker pays.
3. The sequencer includes the transaction; gas accounting passes.
4. At block-commitment time, `hash_contract_state_changes` must run `patricia_update_using_update_constants` for all `N` distinct keys, consuming `O(N × log(2^251))` Cairo steps — steps that were never charged.
5. With a sufficiently large `N` (bounded only by the user's gas budget, not by the prover's step budget), the block's step cap is exceeded and the block cannot be proved.
6. Repeating across blocks prevents the network from finalising any new state, achieving a network halt. [1](#0-0) [6](#0-5) [7](#0-6)

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, StorageReadRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=STORAGE_READ_GAS_COST, request_struct_size=StorageReadRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let response = cast(syscall_ptr, StorageReadResponse*);
    // Advance syscall pointer to the next syscall.
    let syscall_ptr = syscall_ptr + StorageReadResponse.SIZE;

    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    // Update the contract's storage.
    static_assert StorageReadRequest.SIZE == 2;
    assert request.reserved = 0;
    tempvar value = response.value;
    %{ CacheContractStorageRequestKey %}
    tempvar storage_ptr = state_entry.storage_ptr;

            ),

        ),
    );

func hash_contract_state_changes{hash_ptr: HashBuiltin*, range_check_ptr}(
    contract_address: felt,
    prev_state: StateEntry*,
    new_state: StateEntry*,
    patricia_update_constants: PatriciaUpdateConstants*,
    hashed_state_changes: DictAccess*,
) {
    alloc_locals;

    local initial_contract_state_root;
    local final_contract_state_root;

    %{ SetPreimageForCurrentCommitmentInfo %}

    local state_dict_start: DictAccess* = prev_state.storage_ptr;
    local state_dict_end: DictAccess* = new_state.storage_ptr;
    local n_updates = (state_dict_end - state_dict_start) / DictAccess.SIZE;
    // Call patricia_update_using_update_constants() (or the read-optimized variant) instead of
    // patricia_update() in order not to repeat globals_pow2 calculation.
    local should_use_read_optimized: felt;
    %{ ShouldUseReadOptimizedPatriciaUpdate %}
    if (should_use_read_optimized != 0) {
        patricia_update_read_optimized(
            patricia_update_constants=patricia_update_constants,
            update_ptr=state_dict_start,
            n_updates=n_updates,
            height=MERKLE_HEIGHT,
            prev_root=initial_contract_state_root,
            new_root=final_contract_state_root,
        );
    } else {
        patricia_update_using_update_constants(
            patricia_update_constants=patricia_update_constants,
            update_ptr=state_dict_start,
            n_updates=n_updates,
            height=MERKLE_HEIGHT,
            prev_root=initial_contract_state_root,
            new_root=final_contract_state_root,
        );
    }
    local range_check_ptr = range_check_ptr;

    let (prev_value) = get_contract_state_hash(
        class_hash=prev_state.class_hash,
        storage_root=initial_contract_state_root,
        nonce=prev_state.nonce,
    );
    assert hashed_state_changes.prev_value = prev_value;
    let (new_value) = get_contract_state_hash(
        class_hash=new_state.class_hash,
        storage_root=final_contract_state_root,
        nonce=new_state.nonce,
    );

    assert hashed_state_changes.new_value = new_value;
    assert hashed_state_changes.key = contract_address;

    return ();
}
