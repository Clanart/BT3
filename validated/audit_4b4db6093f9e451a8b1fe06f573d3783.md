### Title
`ProofOfInclusion::valid()` Final Check Is a Tautology — Forged Proofs Always Pass — (File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs)

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate contains a self-referential final check that is always `true` when the loop body passes. The function never validates the proof against an external trusted root hash. An attacker who can supply a `ProofOfInclusion` object (via the Streamable/Python/wasm boundary) can forge an internally-consistent proof for any arbitrary `node_hash` and have `valid()` return `true`, without the proof corresponding to the actual committed DataLayer tree root.

### Finding Description

`ProofOfInclusion::valid()` is implemented as follows: [1](#0-0) 

The loop invariant guarantees that after each iteration, `existing_hash` is set to `calculated_hash`, which was just verified to equal `layer.combined_hash`. After the loop completes, `existing_hash` therefore equals the last layer's `combined_hash`.

`root_hash()` is: [2](#0-1) 

It returns `last.combined_hash` — the same value that `existing_hash` holds after the loop. The final statement `existing_hash == self.root_hash()` is therefore always `true` when the loop completes without returning `false`. It is a tautology.

The function never accepts a caller-supplied trusted root. It only verifies that the proof's own layers are internally self-consistent. Any `ProofOfInclusion` whose layers satisfy the hash-chain property will pass `valid()`, regardless of whether the resulting `root_hash()` matches the actual on-chain committed DataLayer root.

`ProofOfInclusion` is a `Streamable` type exposed through the Python wheel and wasm bindings: [3](#0-2) [4](#0-3) 

It can be deserialized from arbitrary bytes via `from_bytes` / `parse_rust`, giving an attacker full control over all fields.

The existing tests confirm the pattern: callers invoke only `proof.valid()` without separately asserting `proof.root_hash() == trusted_root`: [5](#0-4) 

### Impact Explanation

An attacker who can deliver a `ProofOfInclusion` object to a verifier (e.g., over the DataLayer peer protocol or via a Python/wasm API call) can forge proof of inclusion for any `node_hash` of their choice. The forged proof will pass `valid()` while its `root_hash()` is an attacker-chosen value unrelated to the actual committed tree root. Any verifier that relies solely on `proof.valid()` — as the API name implies — will accept the forged proof, allowing the attacker to assert false DataLayer state (e.g., claiming a key-value pair exists in a tree when it does not).

This matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.**

### Likelihood Explanation

- `ProofOfInclusion` is a `Streamable` type fully deserializable from attacker-controlled bytes through the Python and wasm bindings.
- The `valid()` method is the sole public API for proof verification; its name gives no indication that a separate root-hash comparison is required.
- No privileged role or key material is needed; any unprivileged party that can supply bytes to a verifier can exploit this.

### Recommendation

`valid()` must accept a caller-supplied trusted root and compare against it instead of `self.root_hash()`:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root
}
```

The existing `valid()` method (which only checks internal consistency) should be renamed to `is_internally_consistent()` or removed, so callers cannot accidentally rely on it as a complete security check.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
import hashlib

def sha256(a: bytes, b: bytes) -> bytes:
    # mirrors calculate_internal_hash (simplified)
    return hashlib.sha256(a + b).digest()

# Attacker wants to "prove" an arbitrary node_hash is in the tree
node_hash = bytes([0xAA] * 32)          # forged leaf hash
other_hash = bytes([0xBB] * 32)         # arbitrary sibling

# Build one internally-consistent layer
combined = sha256(node_hash, other_hash)  # side = Right (node on left)

layer = ProofOfInclusionLayer(
    other_hash_side=1,                   # Right
    other_hash=other_hash,
    combined_hash=combined,
)

forged = ProofOfInclusion(node_hash=node_hash, layers=[layer])

assert forged.valid()          # True — tautology passes
# forged.root_hash() == combined, which is NOT the real tree root
# but valid() never checks against any external root
```

The forged proof passes `valid()` unconditionally. A verifier that calls only `proof.valid()` accepts the false inclusion claim.
