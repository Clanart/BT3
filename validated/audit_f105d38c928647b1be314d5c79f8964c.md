### Title
P2P Sync State Diff Accepted Without Verifying Hash Against Stored Header's `state_diff_commitment` — (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

---

### Summary

`StateDiffStreamBuilder::parse_data_for_block` assembles a `ThinStateDiff` from untrusted P2P chunks and validates only that its total element count matches the stored header's `state_diff_length`. It never computes `calculate_state_diff_hash(&result)` and compares it against the stored header's `state_diff_commitment`. A malicious peer can therefore supply a state diff with the correct element count but arbitrary wrong content, which is written directly to storage and used to derive the state root.

---

### Finding Description

In `parse_data_for_block` the function reads `target_state_diff_len` from the stored header and loops until the assembled diff reaches that length: [1](#0-0) 

After the loop it performs only a length equality check and a duplicate-class check, then returns the assembled diff unconditionally: [2](#0-1) 

The stored `BlockHeader` also carries `state_diff_commitment: Option<StateDiffCommitment>` — the Poseidon hash of the canonical state diff — but this field is never read in the state diff parsing path. The function `calculate_state_diff_hash` is already used in the committer for exactly this purpose: [3](#0-2) 

The missing invariant is: `calculate_state_diff_hash(assembled_diff) == stored_header.state_diff_commitment`.

This is the direct sequencer analog of the original bug: the "component" (P2P peer) provides a state diff, but there is no check that it matches the "node's" (stored block header's) expected commitment. The same gap is explicitly acknowledged in the central sync path: [4](#0-3) 

The assembled diff is then written to storage without any further validation: [5](#0-4) 

---

### Impact Explanation

A malicious P2P peer can send state diff chunks whose total length matches `state_diff_length` in the header but whose content (storage values, class hashes, nonces, deployed contracts) is arbitrary. The wrong `ThinStateDiff` is written to storage via `append_state_diff`. The committer then applies this diff to the Patricia trie, producing a wrong global state root: [6](#0-5) 

All subsequent RPC calls (`starknet_getStorageAt`, `starknet_getClassAt`, `starknet_getClassHashAt`, `starknet_call`, `starknet_estimateFee`, `starknet_simulateTransactions`) return authoritative-looking wrong values derived from the corrupted state. This matches **High: RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

---

### Likelihood Explanation

Any node participating in P2P sync is exposed. The P2P network is open; any peer can respond to a state diff query for a given block range. The check is entirely absent (not gated on a config flag). The attacker needs only to be the peer that answers the state diff query — no special privilege is required.

---

### Recommendation

After assembling the full `ThinStateDiff` in `parse_data_for_block`, read the stored header's `state_diff_commitment` and verify the hash before returning:

```rust
// After the length check at line 99:
let header = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .expect("A header with number lower than the header marker is missing");

if let Some(expected_commitment) = header.state_diff_commitment {
    let calculated = calculate_state_diff_hash(&result);
    if calculated != expected_commitment {
        return Err(ParseDataError::BadPeer(
            BadPeerError::StateDiffCommitmentMismatch {
                expected: expected_commitment,
                calculated,
            },
        ));
    }
}
```

This mirrors the check already present in `apollo_committer` and resolves the acknowledged TODO in `apollo_central_sync`. [7](#0-6) 

---

### Proof of Concept

1. Node A starts P2P sync and stores a valid header for block N with `state_diff_commitment = H` and `state_diff_length = L`.
2. Attacker peer responds to the state diff query for block N with L elements of valid structure but wrong storage values (e.g., all storage slots set to `0xdead`).
3. `parse_data_for_block` accepts the diff: length matches `L`, no conflicting parts, no hash check performed.
4. `write_to_storage` calls `append_state_diff(N, wrong_diff)`. [8](#0-7) 

5. The committer applies `wrong_diff` to the Patricia trie, producing a wrong global root stored under `ForestMetadataType::StateRoot(N)`.
6. `starknet_getStorageAt(contract, key, block=N)` returns `0xdead` instead of the correct value — an authoritative-looking wrong answer served to all RPC clients.

    fn parse_data_for_block<'a>(
        state_diff_chunks_response_manager: &'a mut ClientResponsesManager<
            DataOrFin<StateDiffChunk>,
        >,
        block_number: BlockNumber,
        storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            let mut result = ThinStateDiff::default();
            let mut prev_result_len = 0;
            let mut current_state_diff_len = 0;
            let target_state_diff_len = storage_reader
                .begin_ro_txn()?
                .get_block_header(block_number)?
                .expect("A header with number lower than the header marker is missing")
                .state_diff_length
                .ok_or(P2pSyncClientError::OldHeaderInStorage {
                    block_number,
                    missing_field: "state_diff_length",
                })?;

            while current_state_diff_len < target_state_diff_len {
                let maybe_state_diff_chunk = state_diff_chunks_response_manager
                    .next()
                    .await
                    .ok_or(ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }))?;
                let Some(state_diff_chunk) = maybe_state_diff_chunk?.0 else {
                    if current_state_diff_len == 0 {
                        return Ok(None);
                    } else {
                        return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                            expected_length: target_state_diff_len,
                            possible_lengths: vec![current_state_diff_len],
                        }));
                    }
                };
                prev_result_len = current_state_diff_len;
                if state_diff_chunk.is_empty() {
                    return Err(ParseDataError::BadPeer(BadPeerError::EmptyStateDiffPart));
                }
                // It's cheaper to calculate the length of `state_diff_part` than the length of
                // `result`.
                current_state_diff_len += state_diff_chunk.len();
                unite_state_diffs(&mut result, state_diff_chunk)?;
            }

            if current_state_diff_len != target_state_diff_len {
                return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                    expected_length: target_state_diff_len,
                    possible_lengths: vec![prev_result_len, current_state_diff_len],
                }));
            }

            validate_deprecated_declared_classes_non_conflicting(&result)?;
            Ok(Some((result, block_number)))
        }
        .boxed()
    }
