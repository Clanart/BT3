### Title
Missing Class Hash Existence Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not validate that the supplied new class hash refers to a previously declared contract class. The OS accepts any arbitrary felt value as the replacement class hash without checking its existence in `contract_class_changes`. This is an acknowledged gap (marked with a `TODO` comment). Any contract that replaces its class with an undeclared hash becomes permanently non-callable, freezing all funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The `class_hash` value comes directly from the syscall request (user-controlled) and is written into the contract's `StateEntry` without any check that it exists in `contract_class_changes`. The developers themselves acknowledge this with the `TODO` comment at line 898. [2](#0-1) 

By contrast, the `execute_declare_transaction` flow in `transaction_impls.cairo` enforces `assert_not_zero(compiled_class_hash)` and uses `dict_update` with `prev_value=0` to ensure a class is declared exactly once — but no analogous existence check is performed when a class hash is consumed by `replace_class`. [3](#0-2) 

---

### Impact Explanation

Once a contract's `class_hash` field in `StateEntry` is set to an undeclared hash, every subsequent call to that contract — including `call_contract`, `library_call`, and direct invocations — will fail at entry-point resolution with `ENTRYPOINT_NOT_FOUND`. There is no recovery path: the OS has no mechanism to revert a committed `replace_class` after the transaction is finalized. Any ERC-20 tokens, ETH, or STRK held in the contract's storage are permanently inaccessible.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

The `replace_class` syscall is callable by any Cairo contract during execution. Exploitation requires one of:

1. A contract that intentionally or accidentally passes an undeclared class hash to `replace_class` (e.g., off-by-one in a hash computation, or a contract upgrade flow that declares the new class in a separate transaction that fails).
2. An attacker who finds a reentrancy or input-injection bug in a target contract that lets them control the argument to `replace_class`.

Because the OS provides no safety net, even an accidental misconfiguration by a legitimate contract developer permanently destroys the contract. The likelihood is low for a deliberate external attack but non-trivial for accidental misuse in complex upgrade flows.

**Likelihood: Low.**

---

### Recommendation

Before writing the new `class_hash` into `contract_state_changes`, the OS should verify that the hash is present in `contract_class_changes` with a non-zero compiled class hash. The fix mirrors the pattern already used in `execute_declare_transaction`:

```cairo
// Verify the class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This check should be inserted immediately after `let class_hash = request.class_hash;` and before the `dict_update` call in `execute_replace_class`.

---

### Proof of Concept

1. Deploy contract `A` holding 1000 STRK.
2. Contract `A` executes the `replace_class` syscall with `class_hash = 0xdeadbeef` (an arbitrary undeclared felt).
3. The OS processes `execute_replace_class`: gas is deducted, the response header is written with `failure_flag=0`, and `contract_state_changes` is updated so `A`'s `StateEntry.class_hash = 0xdeadbeef`.
4. The block is proven and finalized on L1.
5. Any subsequent `call_contract` to `A` resolves `class_hash = 0xdeadbeef`, finds no matching entry point, and returns `ENTRYPOINT_NOT_FOUND`.
6. The 1000 STRK in `A`'s storage are permanently inaccessible — no withdrawal, no upgrade, no recovery.
