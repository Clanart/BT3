### Title
P2P Sync State Diff Accepted Without Commitment Verification, Enabling Wrong Global Root and Block Hash — (`crates/apollo_p2p_sync/src/client/state_diff.rs`)

---

### Summary

`StateDiffStreamBuilder::parse_data_for_block` validates only the *length* of received state-diff chunks against the header's `state_diff_length` field, but never verifies the assembled `ThinStateDiff` hash against the `state_diff_commitment` stored in the already-trusted, signed block header. Any peer can respond to a state-diff query with chunks that sum to the correct length but carry wrong storage values, class hashes, or nonces. The corrupted diff is written to storage unconditionally, causing the committer to derive a wrong `GlobalRoot`, the commitment manager to record a wrong block hash, and every downstream consumer (RPC, proof inputs, state sync) to operate on wrong authoritative state.

---

### Finding Description

**Root cause — missing provenance check in `parse_data_for_block`** [1](#0-0) 

The function reads `target_state_diff_len` from the stored header and accumulates `StateDiffChunk` messages until the running length equals that target. The only structural check after assembly is `validate_deprecated_declared_classes_non_conflicting`, which only tests for duplicate entries in the deprecated-class list. [2](#0-1) 

The stored block header already contains a `state_diff_commitment` field — a Poseidon hash of the canonical state diff — that is part of the signed block hash: [3](#0-2) 

Because the block hash is signed by validators, the `state_diff_commitment` in the stored header is trusted. The missing check is:

```rust
let stored_commitment = txn.get_block_header(block_number)?
    .state_diff_commitment
    .ok_or(P2pSyncClientError::OldHeaderInStorage { ... })?;
let computed = calculate_state_diff_hash(&result);
if computed != stored_commitment {
    return Err(ParseDataError::BadPeer(BadPeerError::StateDiffCommitmentMismatch { ... }));
}
```

Without this check, the assembled diff is passed directly to `write_to_storage`: [4](#0-3) 

which calls `append_state_diff` with no further validation: [5](#0-4) 

**Downstream corruption chain**

1. Wrong `ThinStateDiff` is persisted (wrong storage values, class hashes, nonces).
2. The batcher's commitment manager sends a `CommitBlockRequest` carrying this diff to the committer: [6](#0-5) 

3. The committer applies the diff to the Patricia trie and returns a wrong `GlobalRoot`: [7](#0-6) 

4. The commitment manager calls `calculate_block_hash` with the wrong global root, producing a wrong block hash that diverges from the signed header's block hash: [8](#0-7) 

5. `StateSync::get_block` serves the wrong `ThinStateDiff` to every consumer (batcher `add_sync_block`, RPC, L1 provider catch-up): [9](#0-8) 

**Corroborating evidence — central sync has the same gap acknowledged**

The central sync `store_state_diff` carries an explicit TODO admitting the same missing check: [10](#0-9) 

**Design intent vs. reality**

`SyncBlock` is documented as trusted: [11](#0-10) 

But in the P2P path the diff arrives from an arbitrary, untrusted peer — the trust assumption is violated before the data reaches the `SyncBlock` abstraction.

---

### Impact Explanation

**Impact: Critical — Wrong state, storage value, class hash stored from accepted P2P input.**

A syncing node that processes a crafted state diff will:
- Store wrong contract storage values, nonces, and class hashes in its Patricia trie.
- Compute and persist a wrong `GlobalRoot`, breaking the state-root invariant for every subsequent block.
- Compute a wrong block hash (different from the consensus-signed hash), corrupting the block-hash chain used by proof inputs and retrospective block-hash lookups.
- Serve wrong values from `get_storage_at`, `get_nonce_at`, `get_class_hash_at` via RPC — authoritative-looking wrong values.
- Feed wrong state into the transaction prover's storage proofs, invalidating SNOS proof inputs.

---

### Likelihood Explanation

Any participant in the P2P network can respond to a state-diff query. The syncing node sends a `StateDiff` protocol query; any connected peer may answer. No authentication or reputation gate prevents a peer from crafting a response with the correct `state_diff_length` but wrong entries. The attack requires only knowledge of the target block's `state_diff_length` (publicly available from the signed header) and the ability to connect as a peer.

---

### Recommendation

After assembling the full `ThinStateDiff` from chunks in `parse_data_for_block`, compute its Poseidon commitment and compare it against the `state_diff_commitment` stored in the signed block header:

```rust
// After the while loop, before returning Ok(Some(...)):
let stored_commitment = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .expect("header must exist")
    .state_diff_commitment
    .ok_or(P2pSyncClientError::OldHeaderInStorage {
        block_number,
        missing_field: "state_diff_commitment",
    })?;
let computed_commitment = calculate_state_diff_hash(&result);
if computed_commitment != stored_commitment {
    return Err(ParseDataError::BadPeer(BadPeerError::StateDiffCommitmentMismatch {
        block_number,
        expected: stored_commitment,
        actual: computed_commitment,
    }));
}
```

Apply the same fix to `store_state_diff` in `apollo_central_sync` (the existing `TODO(dan)` comment).

### Proof of Concept

```
Precondition: attacker controls a peer node reachable by the victim syncing node.

1. Victim node queries the attacker peer for the state diff of block N.
   (StateDiff protocol, block_number = N)

2. Attacker reads the signed header for block N from the network.
   Extracts state_diff_length = L (e.g., 3 entries).

3. Attacker crafts StateDiffChunk messages whose total len() == L but whose
   storage values differ from the canonical diff
   (e.g., sets storage slot 0x1 at address 0xA to 0xDEAD instead of 0x0).

4. Victim's parse_data_for_block accumulates chunks:
     current_state_diff_len reaches L  →  loop exits
     validate_deprecated_declared_classes_non_conflicting passes (no duplicates)
     → returns Ok(Some((crafted_diff, N)))

5. write_to_storage calls append_state_diff(N, crafted_diff) — no hash check.

6. Committer receives CommitBlockRequest { state_diff: crafted_diff, height: N }.
   Patricia trie update produces wrong GlobalRoot R'.

7. calculate_block_hash(partial_components, R', parent_hash) → wrong_hash ≠ signed_hash.

8. Victim's storage now has:
     - wrong storage value at (0xA, 0x1)
     - wrong GlobalRoot R' for block N
     - wrong block hash for block N
   All RPC queries for block N return wrong values.
   Proof inputs derived from this state are invalid.
```

```rust
    fn parse_data_for_block<'a>(
        state_diff_chunks_response_manager: &'a mut ClientResponsesManager<
            DataOrFin<StateDiffChunk>,
        >,
        block_number: BlockNumber,
        storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            let mut result = ThinStateDiff::default();
            let mut prev_result_len = 0;
            let mut current_state_diff_len = 0;
            let target_state_diff_len = storage_reader
                .begin_ro_txn()?
                .get_block_header(block_number)?
                .expect("A header with number lower than the header marker is missing")
                .state_diff_length
                .ok_or(P2pSyncClientError::OldHeaderInStorage {
                    block_number,
                    missing_field: "state_diff_length",
                })?;

            while current_state_diff_len < target_state_diff_len {
                let maybe_state_diff_chunk = state_diff_chunks_response_manager
                    .next()
                    .await
                    .ok_or(ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }))?;
                let Some(state_diff_chunk) = maybe_state_diff_chunk?.0 else {
                    if current_state_diff_len == 0 {
                        return Ok(None);
                    } else {
                        return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                            expected_length: target_state_diff_len,
                            possible_lengths: vec![current_state_diff_len],
                        }));
                    }
                };
                prev_result_len = current_state_diff_len;
                if state_diff_chunk.is_empty() {
                    return Err(ParseDataError::BadPeer(BadPeerError::EmptyStateDiffPart));
                }
                // It's cheaper to calculate the length of `state_diff_part` than the length of
                // `result`.
                current_state_diff_len += state_diff_chunk.len();
                unite_state_diffs(&mut result, state_diff_chunk)?;
            }

            if current_state_diff_len != target_state_diff_len {
                return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                    expected_length: target_state_diff_len,
                    possible_lengths: vec![prev_result_len, current_state_diff_len],
                }));
            }

            validate_deprecated_declared_classes_non_conflicting(&result)?;
            Ok(Some((result, block_number)))
        }
        .boxed()
    }
```

```rust
impl StateStorageWriter for StorageTxn<'_, RW> {
    #[sequencer_latency_histogram(STORAGE_APPEND_THIN_STATE_DIFF_LATENCY, false)]
    fn append_state_diff(
        self,
        block_number: BlockNumber,
        thin_state_diff: ThinStateDiff,
    ) -> StorageResult<Self> {
        let file_offset_table = self.txn.open_table(&self.tables.file_offsets)?;
        let markers_table = self.open_table(&self.tables.markers)?;
        let state_diffs_table = self.open_table(&self.tables.state_diffs)?;
        let nonces_table = self.open_table(&self.tables.nonces)?;
        let deployed_contracts_table = self.open_table(&self.tables.deployed_contracts)?;
        let storage_table = self.open_table(&self.tables.contract_storage)?;
        let declared_classes_block_table = self.open_table(&self.tables.declared_classes_block)?;
        let deprecated_declared_classes_block_table =
            self.open_table(&self.tables.deprecated_declared_classes_block)?;
        let compiled_class_hash_table = self.open_table(&self.tables.compiled_class_hash)?;

        // Write state.
        write_deployed_contracts(
            &thin_state_diff.deployed_contracts,
            &self.txn,
            block_number,
            &deployed_contracts_table,
            &nonces_table,
        )?;
        write_storage_diffs(
            &thin_state_diff.storage_diffs,
            &self.txn,
            block_number,
            &storage_table,
        )?;
        // Must be called after write_deployed_contracts since the nonces are updated there.
        write_nonces(&thin_state_diff.nonces, &self.txn, block_number, &nonces_table)?;

        for (class_hash, _) in &thin_state_diff.class_hash_to_compiled_class_hash {
            let not_declared = declared_classes_block_table.get(&self.txn, class_hash)?.is_none();
            if not_declared {
                declared_classes_block_table.insert(&self.txn, class_hash, &block_number)?;
            }
        }

        write_compiled_class_hashes(
            &thin_state_diff.class_hash_to_compiled_class_hash,
            &self.txn,
            block_number,
            &compiled_class_hash_table,
        )?;

        for class_hash in thin_state_diff.deprecated_declared_classes.iter() {
            // Cairo0 classes can be declared in different blocks. The first block to declare the
            // class is recorded here.
            if deprecated_declared_classes_block_table.get(&self.txn, class_hash)?.is_none() {
                deprecated_declared_classes_block_table.insert(
                    &self.txn,
                    class_hash,
                    &block_number,
                )?;
            }
        }

        // Write state diff.
        let location = self.file_handlers.append_state_diff(&thin_state_diff);
        state_diffs_table.append(&self.txn, &block_number, &location)?;
        file_offset_table.upsert(&self.txn, &OffsetKind::ThinStateDiff, &location.next_offset())?;

        update_marker_to_next_block(&self.txn, &markers_table, MarkerKind::State, block_number)?;
```
