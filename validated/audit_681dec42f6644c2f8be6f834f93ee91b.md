### Title
Missing Declared Class Hash Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the new class hash supplied by the calling contract is actually declared in `contract_class_changes`. Any contract can therefore replace its own class with an arbitrary, undeclared class hash, permanently rendering itself unexecutable and freezing all funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` reads `request.class_hash` directly from the syscall request and writes it into the contract's state entry with no existence check:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The developer-acknowledged TODO at line 898 confirms the check is intentionally deferred but never enforced. The OS proof system therefore accepts a state transition where a contract's `class_hash` field is set to a value that has no corresponding entry in `contract_class_changes`. Once committed, the contract cannot be executed (the class does not exist), and because execution is impossible, the contract can never call `replace_class` again to self-recover.

The `execute_replace_class` handler is dispatched unconditionally from `execute_syscalls` for any contract that issues the `REPLACE_CLASS_SELECTOR` syscall: [2](#0-1) 

The `contract_address` passed in is always the currently-executing contract's own address, so the syscall is self-targeted and reachable by any unprivileged contract author.

---

### Impact Explanation

Once a contract's class hash is set to an undeclared value, every future call to that contract fails at class lookup. The contract's storage — including all token balances or collateral deposited by users — is permanently inaccessible with no on-chain recovery path. This is **Critical: Permanent freezing of funds**.

---

### Likelihood Explanation

**Medium.** The attack is directly reachable by any unprivileged transaction sender who deploys or controls a contract. The most realistic trigger is an upgradeable contract pattern where the new class hash is caller-supplied and the contract's own upgrade logic does not independently validate it, relying (incorrectly) on the OS to enforce the constraint. Because the OS currently enforces no such constraint, a single malicious or buggy `replace_class` call with an undeclared hash is sufficient to permanently brick the contract.

---

### Recommendation

Add a `dict_read` on `contract_class_changes` inside `execute_replace_class` before updating the state entry, and assert that the returned compiled class hash is non-zero (i.e., the class has been declared). This mirrors the pattern already used elsewhere in the OS to verify class existence before execution.

---

### Proof of Concept

1. Attacker deploys contract `C` (e.g., a token vault) that accepts user deposits and exposes an `upgrade(new_class_hash: felt)` entry point that internally calls the `replace_class` syscall with the supplied hash.
2. Legitimate users deposit funds into `C`.
3. Attacker calls `C.upgrade(0xdeadbeef)` where `0xdeadbeef` is not present in `contract_class_changes`.
4. The OS dispatches `execute_replace_class` with `class_hash = 0xdeadbeef`.
5. The missing check at line 898 is skipped; `dict_update` writes `class_hash = 0xdeadbeef` into `C`'s `StateEntry`.
6. The proof is generated and accepted — the OS output commits this state transition.
7. All subsequent calls to `C` fail: the class `0xdeadbeef` does not exist, so no entry point can be resolved.
8. `C` cannot call `replace_class` again (execution is impossible).
9. All user funds stored in `C`'s storage slots are permanently frozen. [3](#0-2)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

        key=contract_address,
        prev_value=cast(state_entry, felt),

    );

    return ();
}
