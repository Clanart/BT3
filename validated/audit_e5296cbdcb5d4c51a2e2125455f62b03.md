### Title
Missing Declared Class Validation in `execute_replace_class` Allows Permanent Contract Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the new class hash provided corresponds to a previously declared contract class. This allows any contract to replace its own class hash with an arbitrary, undeclared hash, permanently bricking the contract and freezing any funds it holds. The same flaw exists in the deprecated syscall path.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall. After reducing gas, it reads the new `class_hash` from the request and directly updates the contract's `StateEntry` in `contract_state_changes` without verifying that a contract class with that hash has ever been declared. The code itself acknowledges this missing check with an explicit TODO:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0)

The analog to the external report is direct: just as the Bridge contract was missing a function to perform a critical state transition on a contract it owns, the OS here is missing a validation step on a critical state transition (`replace_class`) that it is responsible for enforcing. In both cases, the absence of a necessary check on a class/implementation change creates an irreversible bad state.

---

### Impact Explanation

If a contract calls `replace_class` with an undeclared class hash:

1. The OS accepts the state transition and writes the new (invalid) class hash into `contract_state_changes`.
2. The contract's class hash now points to a non-existent class in the declared class set.
3. Any subsequent call to the contract will fail at the OS level because no compiled class can be loaded for that hash.
4. If the contract holds any funds (ERC-20 tokens, ETH via L1→L2 messages, etc.), those funds are **permanently frozen** — no entry point of the contract can ever execute again.

**Impact: Critical — Permanent freezing of funds.**

---

### Likelihood Explanation

This is directly reachable by any unprivileged contract deployer or transaction sender:

- No privileged role, leaked key, or operator cooperation is required.
- Any user can deploy a contract whose constructor or any entry point calls `replace_class` with an arbitrary felt value that is not a declared class hash.
- The OS will accept the syscall without any validation, as confirmed by the TODO comment in the production code.
- The attack is deterministic and requires only a single transaction.

---

### Recommendation

Before updating the contract's class hash in `contract_state_changes`, verify that the new class hash exists in `contract_class_changes` (i.e., it was declared in the current block) or in the pre-block declared class set. Concretely, replace the TODO with an assertion such as:

```cairo
// Assert that the new class hash has been declared.
let (compiled_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_hash);
```

This check should be applied to both `execute_replace_class` in `syscall_impls.cairo` and `execute_replace_class` in `deprecated_execute_syscalls.cairo`.

---

### Proof of Concept

1. An unprivileged user deploys a contract `C` that holds funds and whose `__execute__` entry point calls `replace_class(0xdeadbeef_undeclared_hash)`.
2. The user submits an invoke transaction targeting `C`.
3. The OS processes the `replace_class` syscall via `execute_replace_class` in `syscall_impls.cairo`.
4. No validation is performed — the TODO at line 898 confirms the check is absent.
5. `contract_state_changes` is updated: `C`'s class hash is now `0xdeadbeef_undeclared_hash`.
6. The block is finalized and the state is committed.
7. Any future transaction targeting `C` fails because the OS cannot resolve `0xdeadbeef_undeclared_hash` to a compiled class.
8. All funds held by `C` are permanently frozen. [3](#0-2) [4](#0-3)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
