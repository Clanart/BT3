### Title
L1→L2 Message Marked as Consumed Before Handler Execution Succeeds, Enabling Permanent Message Freezing — (File: `execution/transaction_impls.cairo`)

---

### Summary

In `execute_l1_handler_transaction`, `consume_l1_to_l2_message` is called at line 444 — writing the message to the OS output and marking it as consumed — **before** the handler is executed at line 446. The handler is dispatched through `non_reverting_select_execute_entry_point_func`, which hard-asserts `is_reverted = 0`. If the handler always reverts, no valid proof can ever be generated for that message, and the only sequencer escape hatch (`is_reverted = TRUE` pre-skip) leaves the message permanently unconsumed in the L1→L2 queue. Funds locked in such a message are permanently frozen.

---

### Finding Description

**Root cause — ordering of consume vs. execute:**

In `execute_l1_handler_transaction` (`transaction_impls.cairo` lines 374–452):

```cairo
// Consume L1-to-L2 message.
consume_l1_to_l2_message(execution_context=tx_execution_context, nonce=nonce);   // line 444
let remaining_gas = L1_HANDLER_L2_GAS_MAX_AMOUNT;
non_reverting_select_execute_entry_point_func{remaining_gas=remaining_gas}(       // line 446
    block_context=block_context, execution_context=tx_execution_context
);
```

`consume_l1_to_l2_message` writes the `MessageToL2Header` and payload into `outputs.messages_to_l2` unconditionally before the handler runs. [1](#0-0) 

**Root cause — non-reverting assertion:**

`non_reverting_select_execute_entry_point_func` (`execute_transaction_utils.cairo` lines 181–197) hard-asserts the handler cannot revert:

```cairo
assert is_reverted = 0;   // line 195
```

If the handler's `failure_flag` is 1, this assertion fails and the entire execution trace is invalid — no valid STARK proof can be generated. [2](#0-1) 

**Root cause — `check_is_reverted` is a no-op:**

The only OS-level guard that could validate the sequencer's `is_reverted` hint is `check_is_reverted` (`execution_constraints.cairo` lines 20–22):

```cairo
func check_is_reverted(is_reverted: felt) {
    return ();
}
```

It performs **zero validation**. The sequencer can freely set `is_reverted = TRUE` for any L1 handler. [3](#0-2) 

**Combined effect:**

When the sequencer encounters an L1 handler that always reverts, it has exactly two choices:

| Sequencer choice | What happens |
|---|---|
| `is_reverted = FALSE` (try to execute) | Handler reverts → `assert is_reverted = 0` fails → proof invalid → block cannot be submitted |
| `is_reverted = TRUE` (pre-skip) | Function returns at line 387–390 **before** `consume_l1_to_l2_message` → message is **not** consumed → message stays in queue forever | [4](#0-3) 

In neither path can the message ever be consumed. The L1→L2 message is permanently stuck.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any ETH or ERC-20 tokens bridged via an L1→L2 message whose target L2 handler always reverts (e.g., the contract was upgraded to remove the handler, or the handler panics on all inputs) are permanently unrecoverable through the normal protocol path. The OS provides no mechanism to consume a message while allowing the handler to fail gracefully — the `non_reverting_select_execute_entry_point_func` constraint makes handler failure a proof-level abort, not a recoverable revert.

---

### Likelihood Explanation

The trigger is fully attacker-controlled via the public L1→L2 messaging interface:

1. An attacker (or any user) deploys an L2 contract whose `@l1_handler` always panics.
2. The attacker sends an L1→L2 message to that contract carrying bridged funds.
3. The sequencer is permanently unable to include a valid proof for that message.
4. The funds are frozen with no on-chain recovery path.

No privileged role, leaked key, or network-level attack is required. The attacker is an ordinary L1 message sender — an explicitly allowed unprivileged role in the bounty scope.

---

### Recommendation

**Short term:** Move `consume_l1_to_l2_message` to **after** `non_reverting_select_execute_entry_point_func` returns successfully, so the message is only committed to the OS output when the handler is confirmed to have succeeded.

**Long term:** Implement `check_is_reverted` to actually validate the sequencer's `is_reverted` hint against the execution result, and define a protocol-level "failed L1 handler" path that consumes the message (marking it as failed) so it cannot block the queue indefinitely. Document the expected behavior when an L1 handler reverts.

---

### Proof of Concept

1. Attacker deploys L2 contract `C` with `@l1_handler fn handle_deposit(...) { panic!(...) }`.
2. Attacker calls the L1 StarkNet core contract's `sendMessageToL2` targeting `C`, attaching bridged ETH.
3. The sequencer picks up the L1→L2 message and attempts `execute_l1_handler_transaction`.
4. **Path A (execute):** `consume_l1_to_l2_message` writes the message to `outputs.messages_to_l2`; handler runs and sets `failure_flag = 1`; `assert is_reverted = 0` at line 195 of `execute_transaction_utils.cairo` fails; proof is invalid; block cannot be submitted.
5. **Path B (pre-skip):** Sequencer sets `is_reverted = TRUE`; function returns at line 387–390 before consuming the message; message remains in the L1→L2 queue unconsumed.
6. Neither path ever produces a valid proof that consumes the message. The bridged ETH is permanently frozen. [5](#0-4) [6](#0-5) [3](#0-2)

```text
func consume_l1_to_l2_message{outputs: OsCarriedOutputs*}(
    execution_context: ExecutionContext*, nonce: felt
) {
    assert_not_zero(execution_context.calldata_size);
    // The payload is the calldata without the from_address argument (which is the first).
    let payload: felt* = execution_context.calldata + 1;
    tempvar payload_size = execution_context.calldata_size - 1;

    tempvar execution_info = execution_context.execution_info;

    // Write the given transaction to the output.
    assert [outputs.messages_to_l2] = MessageToL2Header(
        from_address=[execution_context.calldata],
        to_address=execution_info.contract_address,
        nonce=nonce,
        selector=execution_info.selector,
        payload_size=payload_size,
    );

    let message_payload = cast(outputs.messages_to_l2 + MessageToL2Header.SIZE, felt*);
    memcpy(dst=message_payload, src=payload, len=payload_size);

    let (outputs) = os_carried_outputs_new(
        messages_to_l1=outputs.messages_to_l1,
        messages_to_l2=outputs.messages_to_l2 + MessageToL2Header.SIZE +
        outputs.messages_to_l2.payload_size,
    );
    return ();
```
