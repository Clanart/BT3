### Title
P2P State Diff Assembly Verifies Length But Not `state_diff_commitment`, Allowing a Malicious Peer to Inject a Wrong State Diff - (File: crates/apollo_p2p_sync/src/client/state_diff.rs)

---

### Summary

The `parse_data_for_block` function in the P2P sync client assembles a `ThinStateDiff` from peer-supplied chunks and terminates the loop only when the accumulated chunk-length counter reaches the `state_diff_length` stored in the block header. It never computes the hash of the assembled diff and compares it against the `state_diff_commitment` that is also stored in the same header. A malicious peer can relay a legitimate, correctly-signed header and then supply state-diff chunks whose content is entirely fabricated, as long as the total `StateDiffChunk::len()` sum equals the header's `state_diff_length`. The fabricated diff passes every check and is written to storage, giving the syncing node a permanently wrong view of the chain state.

---

### Finding Description

**Analog mapping.** The external report describes a `merge()` that transfers voting power from a "claimed" token to a "fresh" token without resetting the claimed flag, so the same underlying power is counted twice. The sequencer analog is: `unite_state_diffs` merges peer-supplied chunks into a result `ThinStateDiff` without ever verifying the final result against the authoritative commitment hash that was already stored in the block header. The "claimed" marker in the external bug maps to `state_diff_commitment`; the "merge without reset" maps to the missing post-assembly hash check.

**Code path.**

### Impact Explanation

A malicious P2P peer that has received a legitimate block header (correctly signed by the sequencer) can:

1. Forward that header to the victim node — it is stored with the correct `state_diff_commitment` and `state_diff_length`.
2. Send fabricated `StateDiffChunk` messages whose `len()` values sum to `state_diff_length` but whose content (contract addresses, storage keys, values, class hashes, nonces) is entirely attacker-controlled.
3. The victim node writes the fabricated `ThinStateDiff` to storage via `append_state_diff`.

Consequences:
- The node's storage now contains wrong deployed-contract class hashes, wrong storage values, wrong nonces, and wrong declared-class mappings for that block.
- Any subsequent RPC call (`starknet_getStorageAt`, `starknet_getClassAt`, `starknet_getNonce`, `starknet_getStateUpdate`, etc.) returns attacker-chosen values.
- If the node later re-executes transactions against this state (e.g., for fee estimation or tracing), execution uses the wrong state, producing wrong results.
- The global state root computed by the committer from this diff will diverge from the canonical root, breaking proof verification.

This matches the allowed impact: **"High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."**

---

### Likelihood Explanation

Any peer on the P2P network can perform this attack. No privileged access is required. The attacker only needs to:
- Observe a valid block header (publicly available on the network).
- Craft replacement chunks whose lengths sum correctly.

The attack is silent — the victim node logs no error and reports no bad-peer event, because every check (`is_empty`, `ConflictingStateDiffParts`, `WrongStateDiffLength`, `validate_deprecated_declared_classes_non_conflicting`) passes.

---

### Recommendation

After the assembly loop exits and before writing to storage, compute the hash of the assembled diff and compare it against the stored commitment:

```rust
if let Some(expected_commitment) = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .and_then(|h| h.state_diff_commitment)
{
    let actual_commitment = calculate_state_diff_hash(&result);
    if actual_commitment != expected_commitment {
        return Err(ParseDataError::BadPeer(
            BadPeerError::WrongStateDiffCommitment { block_number },
        ));
    }
}
```

This mirrors the existing `verify_state_diff_hash` guard in the committer and closes the gap between the length-only check and the cryptographic commitment.

### Proof of Concept

**Setup:** Block N has been produced with `state_diff_length = 3` and `state_diff_commitment = H` (hash of the real diff: one deployed contract at address `0xA` with class hash `0xC`, one storage update `(0xA, key=0x1, val=0x2)`, one nonce update `(0xA, nonce=5)`).

**Attack:**

1. Malicious peer sends the legitimate `SignedBlockHeader` for block N (obtained from an honest node). The victim stores it with `state_diff_length = 3`, `state_diff_commitment = H`.

2. Malicious peer sends three fabricated chunks (total `len()` = 3):
   - `ContractDiff { address: 0xB, class_hash: Some(0xDEAD), nonce: None, storage_diffs: {} }` → `len() = 1`
   - `ContractDiff { address: 0xB, class_hash: None, nonce: None, storage_diffs: {0x10: 0xFF} }` → `len() = 1`
   - `DeclaredClass { class_hash: 0xEEEE, compiled_class_hash: 0xFFFF }` → `len() = 1`

3. `parse_data_for_block` loop: `current_state_diff_len` reaches 3 = `target_state_diff_len`. Loop exits. `validate_deprecated_declared_classes_non_conflicting` passes (no deprecated classes). `Ok(Some((result, N)))` is returned.

4. `write_to_storage` calls `append_state_diff(N, fabricated_diff)`. The victim node now believes contract `0xB` was deployed with class `0xDEAD` and that class `0xEEEE` was declared — none of which is true.

5. `starknet_getClassHashAt(block=N, address=0xA)` returns `None` (contract 0xA is absent from the fabricated diff). `starknet_getClassHashAt(block=N, address=0xB)` returns `0xDEAD`. Both answers are wrong.

impl StateDiffChunk {
    pub fn len(&self) -> usize {
        match self {
            StateDiffChunk::ContractDiff(contract_diff) => {
                let mut result = contract_diff.storage_diffs.len();
                if contract_diff.class_hash.is_some() {
                    result += 1;
                }
                if contract_diff.nonce.is_some() {
                    result += 1;
                }
                result
            }
            StateDiffChunk::DeclaredClass(_) => 1,
            StateDiffChunk::DeprecatedDeclaredClass(_) => 1,
        }
    }

    pub fn is_empty(&self) -> bool {
        self.len() == 0
    }
