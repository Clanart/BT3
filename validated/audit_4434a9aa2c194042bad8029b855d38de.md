### Title
`compute_max_possible_fee` Can Wrap to Zero Due to Unbounded `max_price_per_unit`, Enabling Fee-Free Transaction Execution — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

The `compute_max_possible_fee` function in the StarkNet OS computes the maximum fee a V3 transaction may be charged. Because `max_price_per_unit` in each `ResourceBounds` entry has no upper-bound validation, an attacker can supply crafted values that cause the entire fee sum to wrap to zero modulo the Cairo field prime. When `max_fee == 0`, `charge_fee` returns immediately without collecting any fee, allowing the transaction to execute for free.

---

### Finding Description

`pack_resource_bounds` — the only place where `ResourceBounds` fields are range-checked — enforces:

```cairo
assert_nn_le(resource_bounds.max_amount, 2 ** 64 - 1);   // max_amount ∈ [0, 2^64-1]
assert_nn(resource_bounds.max_price_per_unit);            // max_price_per_unit ∈ [0, (P-1)/2]
``` [1](#0-0) 

`max_price_per_unit` is only checked to be non-negative; there is **no upper bound** (e.g., `≤ 2^128 - 1`). It can therefore be any felt value up to `(P-1)/2 ≈ 2^250`.

`compute_max_possible_fee` then multiplies these raw field values together with no overflow protection:

```cairo
return l1_gas_bounds.max_amount * l1_gas_bounds.max_price_per_unit
     + l2_gas_bounds.max_amount * (l2_gas_bounds.max_price_per_unit + tx_info.tip)
     + l1_data_gas_bounds.max_amount * l1_data_gas_bounds.max_price_per_unit;
``` [2](#0-1) 

All arithmetic is modular over the Cairo prime P ≈ 2^251. With `max_amount ≤ 2^64-1` and `max_price_per_unit ≤ (P-1)/2 ≈ 2^250`, each product can reach ≈ 2^314, wrapping around P multiple times. An attacker can choose values such that the entire sum reduces to **0 mod P**.

`charge_fee` then short-circuits:

```cairo
let max_fee = compute_max_possible_fee(tx_info=tx_info);
if (max_fee == 0) {
    return ();
}
``` [3](#0-2) 

No ERC-20 transfer is executed, and the sequencer receives nothing for the transaction.

---

### Impact Explanation

**Direct loss of funds (Critical).** The sequencer collects zero fee for a fully-executed transaction. An attacker can repeat this for every transaction they submit, draining sequencer revenue and, at scale, making block production economically unviable. This maps directly to the "Direct loss of funds" impact category.

---

### Likelihood Explanation

**High.** The attack requires only crafting a valid V3 transaction with specific `max_price_per_unit` values. No privileged role, leaked key, or external dependency is needed. The attacker controls all three `ResourceBounds` entries and the `tip`, giving ample degrees of freedom to solve the modular equation. The transaction still passes hash verification because `pack_resource_bounds` is called during hash computation and only checks `assert_nn` (no upper bound), so the crafted values are accepted as valid.

---

### Recommendation

Add an explicit upper-bound check on `max_price_per_unit` inside `pack_resource_bounds`, consistent with the StarkNet transaction specification (e.g., `assert_nn_le(resource_bounds.max_price_per_unit, 2 ** 128 - 1)`). This ensures that `max_amount * max_price_per_unit ≤ (2^64-1) * (2^128-1) < 2^192 ≪ P`, making field-modular wrap-around impossible and keeping `compute_max_possible_fee` semantically correct. ### Proof of Concept

**Setup:** Attacker constructs a V3 invoke transaction where the three `ResourceBounds` entries are chosen so that:

```
A₁ * B₁  +  A₂ * (B₂ + tip)  +  A₃ * B₃  ≡  0  (mod P)
```

where `Aᵢ = max_amount` (≤ 2^64-1) and `Bᵢ = max_price_per_unit` (≤ (P-1)/2).

**Concrete example (simplified):**
- Set L1_GAS and L1_DATA_GAS `max_amount = 0` (zeroing those terms).
- Set L2_GAS `max_amount = 1`, `tip = 0`.
- Set L2_GAS `max_price_per_unit = P` — but since `assert_nn` only checks `≥ 0`, choose `max_price_per_unit = 0` to trivially get sum = 0, **or** choose any non-zero `max_price_per_unit` and a complementary `max_amount` such that `max_amount * max_price_per_unit ≡ 0 (mod P)` (impossible for prime P unless one factor is 0 — so use the multi-term cancellation path with all three terms).

**Multi-term cancellation path:**
- L1_GAS: `max_amount = k`, `max_price_per_unit = v₁` such that `k * v₁ mod P = X`
- L2_GAS: `max_amount = 1`, `max_price_per_unit = P - X - Y` (where Y is the L1_DATA term)
- L1_DATA_GAS: `max_amount = 1`, `max_price_per_unit = Y`
- Total = `X + (P - X - Y) + Y = P ≡ 0 (mod P)` ✓

All values satisfy `assert_nn` (they are in [0, (P-1)/2] by construction). `pack_resource_bounds` passes. The transaction hash is computed and verified normally. `compute_max_possible_fee` returns 0. `charge_fee` returns at line 123 without transferring any tokens. The transaction executes fully for free. [4](#0-3) [1](#0-0)

func compute_max_possible_fee(tx_info: TxInfo*) -> felt {
    tempvar resource_bounds: ResourceBounds* = tx_info.resource_bounds_start;
    let n_resource_bounds = (tx_info.resource_bounds_end - resource_bounds) / ResourceBounds.SIZE;

    // Only V3 transactions with all resource bounds are supported.
    assert tx_info.version = 3;
    assert n_resource_bounds = 3;

    tempvar l1_gas_bounds: ResourceBounds = resource_bounds[L1_GAS_INDEX];
    tempvar l2_gas_bounds: ResourceBounds = resource_bounds[L2_GAS_INDEX];
    tempvar l1_data_gas_bounds = resource_bounds[L1_DATA_GAS_INDEX];

    return l1_gas_bounds.max_amount * l1_gas_bounds.max_price_per_unit + l2_gas_bounds.max_amount *
        (l2_gas_bounds.max_price_per_unit + tx_info.tip) + l1_data_gas_bounds.max_amount *
        l1_data_gas_bounds.max_price_per_unit;
}

// Charges a fee from the user.
// If max_fee is not 0, validates that the selector matches the entry point of an account contract
// and executes an ERC20 transfer on the behalf of that account contract.
//
// Arguments:
// block_context - a global context that is fixed throughout the block.
// tx_execution_context - The execution context of the transaction that pays the fee.
func charge_fee{
    range_check_ptr,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, tx_execution_context: ExecutionContext*) {
    alloc_locals;

    local tx_info: TxInfo* = tx_execution_context.execution_info.tx_info;
    let max_fee = compute_max_possible_fee(tx_info=tx_info);

    if (max_fee == 0) {
        return ();
    }
