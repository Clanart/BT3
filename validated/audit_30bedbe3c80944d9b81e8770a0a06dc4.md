### Title
`INTERNED_GENERATOR` Flag Has No Activation Path in `get_flags_for_height_and_constants` — (`File: crates/chia-consensus/src/spendbundle_validation.rs`)

---

### Summary

The `INTERNED_GENERATOR` consensus flag is fully defined and wired into consensus-critical cost-computation branches in `run_block_generator2` and `calculate_base_cost`, but it is **never emitted** by `get_flags_for_height_and_constants` — the sole canonical function that maps block height to active consensus flags. No corresponding height constant exists in `ConsensusConstants`. The flag is therefore permanently dead in every real consensus path, meaning the "generator-identity hard fork" it was designed to enforce can never activate.

---

### Finding Description

`ConsensusFlags::INTERNED_GENERATOR` (bit `0x0800_0000`) is declared in `flags.rs` with the explicit comment:

> "After the generator-identity hard fork, generators must be validated from the INTERNED (canonical) tree so atom/pair limits and cost apply to the same structure independent of serialization." [1](#0-0) 

The flag gates two distinct consensus-critical branches:

**Branch 1 — `run_block_generator2` (block validation path):**

```rust
let (mut a, base_cost, program) = if flags.contains(ConsensusFlags::INTERNED_GENERATOR) {
    // intern_tree_limited → interned_vbytes * cost_per_byte
} else {
    // node_from_bytes_backrefs → program.len() * cost_per_byte
};
``` [2](#0-1) 

**Branch 2 — `calculate_base_cost` (mempool / spend-bundle validation path):**

```rust
fn calculate_base_cost(...) -> Result<u64, ValidationErr> {
    if flags.contains(ConsensusFlags::INTERNED_GENERATOR) {
        // intern_tree_limited → interned_vbytes * cost_per_byte
    } else {
        // raw generator length * cost_per_byte
    }
}
``` [3](#0-2) 

**The activation function never sets the flag:**

```rust
pub fn get_flags_for_height_and_constants(
    prev_tx_height: u32,
    constants: &ConsensusConstants,
) -> ConsensusFlags {
    let mut flags = ConsensusFlags::empty();
    if prev_tx_height >= constants.hard_fork2_height {
        flags |= ENABLE_KECCAK_OPS_OUTSIDE_GUARD | COST_CONDITIONS | ENABLE_SECP_OPS | RELAXED_BLS;
    }
    if prev_tx_height >= constants.soft_fork8_height {
        flags |= DISABLE_OP;
    }
    if prev_tx_height >= constants.soft_fork9_height {
        flags |= SIMPLE_GENERATOR | CANONICAL_INTS | LIMIT_SPENDS;
        // INTERNED_GENERATOR is never added here
    }
    flags
}
``` [4](#0-3) 

`ConsensusConstants` contains no `interned_generator_height` field, so there is no mechanism to ever activate this flag through the normal consensus path. [5](#0-4) 

The only place `INTERNED_GENERATOR` is used in production code is inside `run_block_generator2` and `calculate_base_cost`. It is also used in a single internal test that explicitly passes the flag by hand: [6](#0-5) 

---

### Impact Explanation

**Cost-model split between block builder and consensus validator.**

`InternedBlockBuilder` computes block cost using `interned_vbytes * cost_per_byte` (canonical tree size). Consensus validation, because `INTERNED_GENERATOR` is never set, always uses `program.len() * cost_per_byte` (back-reference-compressed serialized size). When a generator contains many repeated subtrees, the back-reference serialized form is strictly smaller than the canonical interned form. This creates a permanent, unresolvable disagreement between what the block builder charges and what consensus charges.

**Atom/pair limit enforcement gap.**

The flag's stated purpose is to ensure "atom/pair limits and cost apply to the same structure independent of serialization." Without it, atom/pair limits are enforced on the back-reference-deserialized tree, not the canonical tree. A generator can be crafted so that its back-reference form is within limits while its canonical expansion is not. Because `INTERNED_GENERATOR` is never activated, this canonical-form check is permanently skipped in all consensus paths.

**Consequence:** A block producer can craft a generator whose back-reference-compressed serialized size is small (low byte cost charged by consensus) while its canonical tree is large (high interned vbytes). This allows the generator's byte cost to be systematically undercharged relative to the intended design, permitting more total computation within a block than `max_block_cost_clvm` was designed to allow. This is a cost-accounting disagreement in consensus-critical data.

---

### Likelihood Explanation

Exploiting the cost undercharge requires constructing a block generator — a role limited to farmers (block producers). The flag's absence is not exploitable by a passive observer or a transaction submitter. Likelihood is therefore **low** in practice, but the structural defect is permanent and deterministic: every block validated after any future "generator-identity hard fork" would still use the wrong cost model because the activation path does not exist.

---

### Recommendation

1. Add an `interned_generator_height: u32` field to `ConsensusConstants`.
2. In `get_flags_for_height_and_constants`, add:
   ```rust
   if prev_tx_height >= constants.interned_generator_height {
       flags |= ConsensusFlags::INTERNED_GENERATOR;
   }
   ```
3. Set the height in `TEST_CONSTANTS` and the mainnet constants to the planned hard-fork block. [7](#0-6)

fn calculate_base_cost(
    spend_bundle: &SpendBundle,
    flags: ConsensusFlags,
    constants: &ConsensusConstants,
) -> Result<u64, ValidationErr> {
    if flags.contains(ConsensusFlags::INTERNED_GENERATOR) {
        let mut gen_allocator = Allocator::new();
        let generator = build_generator(
            &mut gen_allocator,
            spend_bundle
                .coin_spends
                .iter()
                .map(|cs| (cs.coin, cs.puzzle_reveal.as_slice(), cs.solution.as_slice())),
        )
        .map_err(|_| ValidationErr::Err(ErrorCode::GeneratorRuntimeError))?;
        let interned = intern_tree_limited(&gen_allocator, generator, u32::MAX as usize)
            .map_err(|_| ValidationErr::Err(ErrorCode::GeneratorRuntimeError))?;
        Ok(interned_vbytes(&interned) * constants.cost_per_byte)
    } else {
        // We don't pay the size cost (nor execution cost) of being wrapped by a
        // quote (in solution_generator).
        let generator_length_without_quote =
            calculate_generator_length(&spend_bundle.coin_spends) - QUOTE_BYTES;
        Ok(generator_length_without_quote as u64 * constants.cost_per_byte)
    }
}

pub fn get_flags_for_height_and_constants(
    prev_tx_height: u32,
    constants: &ConsensusConstants,
) -> ConsensusFlags {
    //  the hard-fork initiated with 2.0. To activate June 2024
    //  * costs are ascribed to some unknown condition codes, to allow for
    // soft-forking in new conditions with cost
    //  * a new condition, SOFTFORK, is added which takes a first parameter to
    //    specify its cost. This allows soft-forks similar to the softfork
    //    operator
    //  * BLS operators introduced in the soft-fork (behind the softfork
    //    guard) are made available outside of the guard.
    //  * division with negative numbers are allowed, and round toward
    //    negative infinity
    //  * AGG_SIG_* conditions are allowed to have unknown additional
    //    arguments
    //  * Allow the block generator to be serialized with the improved clvm
    //   serialization format (with back-references)

    // The soft fork initiated with 2.5.0. The activation date is still TBD.
    // Adds a new keccak256 operator under the softfork guard with extension 1.
    // This operator can be hard forked in later, but is not included in a hard fork yet.

    // In hard fork 2, we enable the keccak operator outside the softfork guard
    let mut flags = ConsensusFlags::empty();
    if prev_tx_height >= constants.hard_fork2_height {
        flags |= ConsensusFlags::ENABLE_KECCAK_OPS_OUTSIDE_GUARD
            | ConsensusFlags::COST_CONDITIONS
            | ConsensusFlags::ENABLE_SECP_OPS
            | ConsensusFlags::RELAXED_BLS;
    }

    if prev_tx_height >= constants.soft_fork8_height {
        flags |= ConsensusFlags::DISABLE_OP;
    }

    if prev_tx_height >= constants.soft_fork9_height {
        flags |= ConsensusFlags::SIMPLE_GENERATOR
            | ConsensusFlags::CANONICAL_INTS
            | ConsensusFlags::LIMIT_SPENDS;
    }
    flags
}
