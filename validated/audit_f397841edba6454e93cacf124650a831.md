### Title
`batch_insert` Bypasses Duplicate-Key/Hash Guard for Bulk Items, Corrupting DataLayer Merkle Tree Root — (`crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::batch_insert` applies the duplicate-key and duplicate-hash guards only to the first two items it processes (via `self.insert()`), but silently skips those same guards for every remaining item in the batch (via direct `insert_entry_to_blob` calls). An untrusted caller supplying a batch with a repeated `KeyId` or `Hash` can insert phantom duplicate leaf nodes into the blob, producing a corrupted Merkle root and enabling forged inclusion proofs against the committed DataLayer state.

---

### Finding Description

`MerkleBlob::insert` enforces two invariants before writing any leaf:

```rust
// blob.rs:369-374
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
```

`batch_insert` calls `self.insert()` (with those guards) only for the first two items when the tree has ≤ 1 existing leaves:

```rust
// blob.rs:578-585
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else { return Ok(()); };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;   // ← guarded
    }
}
```

All remaining items — and **all** items when the tree already has ≥ 2 leaves — are written directly through `insert_entry_to_blob` with no duplicate check:

```rust
// blob.rs:587-603
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ..., node: Node::Leaf(LeafNode { hash, key, value, ... }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;   // ← no guard
    indexes.push(new_leaf_index);
}
```

`insert_entry_to_blob` calls `block_status_cache.add_leaf`, which performs a silent HashMap overwrite:

```rust
// blob.rs:188-193
fn add_leaf(&mut self, index: TreeIndex, leaf: LeafNode) {
    self.free_indexes.shift_remove(&index);
    self.key_to_index.insert(leaf.key, index);          // silently overwrites
    self.leaf_hash_to_index.insert(leaf.hash, index);   // silently overwrites
}
```

When a duplicate `KeyId` is supplied, the old leaf node remains physically in the blob (still referenced by its parent internal node), while the cache's `key_to_index` entry is overwritten to point to the new, disconnected leaf. The new leaf is then wired into a fresh subtree and grafted onto the existing tree. The result is a blob that contains **two** leaf nodes for the same key, one reachable via the tree structure and one reachable via the cache — a split that makes the Merkle root inconsistent with the cache-visible state.

The Python binding `py_batch_insert` (line 1503) exposes this path directly to untrusted callers.

---

### Impact Explanation

The DataLayer Merkle root is committed on-chain. After a corrupted `batch_insert`:

1. **Tree root corruption**: The root hash is computed from the full blob structure, which now contains both the old and new leaf for the same key. The root no longer faithfully represents the key→value mapping visible through the cache.
2. **Forged inclusion proofs**: `get_proof_of_inclusion(key)` follows the cache to the *new* leaf's index and constructs a Merkle path. That path does not match the root hash (which includes the *old* leaf's contribution), so the proof is structurally invalid — but the mismatch is silent and only detectable by re-verifying the proof against the root. A caller that trusts the proof without re-checking the root accepts a false state.
3. **Persistent corruption**: `check_integrity` is only called in test builds (`check_integrity_on_drop: cfg!(test)`), so production deployments never detect the inconsistency.

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

---

### Likelihood Explanation

The Python binding is the primary public API for DataLayer operations. Any caller that constructs a `batch_insert` payload — including one that re-submits an existing key — triggers the bypass whenever the batch contains more than two items or the tree already has two or more leaves. No special privilege is required; the attacker only needs to control the contents of the batch vector.

---

### Recommendation

Move the duplicate-key and duplicate-hash checks out of `insert` and into `batch_insert` so they apply uniformly to every item in the batch before any blob writes occur:

```rust
pub fn batch_insert(
    &mut self,
    keys_values_hashes: Vec<((KeyId, ValueId), Hash)>,
) -> Result<(), Error> {
    // Guard the entire batch first
    for ((key, _value), hash) in &keys_values_hashes {
        if self.block_status_cache.contains_key(*key) {
            return Err(Error::KeyAlreadyPresent());
        }
        if self.block_status_cache.contains_leaf_hash(hash) {
            return Err(Error::HashAlreadyPresent());
        }
    }
    // ... existing logic unchanged ...
}
```

Alternatively, deduplicate the input vector before processing and return an error on any collision.

### Proof of Concept

```python
from chia_rs import MerkleBlob, KeyId, ValueId
import hashlib

def h(n: int) -> bytes:
    return hashlib.sha256(n.to_bytes(8, "big")).digest()

blob = MerkleBlob(blob=bytearray())

# Pre-populate with 2 leaves so leaf_count >= 2 → ALL batch items bypass the guard
blob.insert(KeyId(100), ValueId(100), h(100), None, None)
blob.insert(KeyId(200), ValueId(200), h(200), None, None)

# Now batch_insert with a duplicate key (KeyId(100) already exists)
# The guard is never reached; insert_entry_to_blob writes a second leaf for key 100
blob.batch_insert(
    [(KeyId(100), ValueId(999)), (KeyId(300), ValueId(300))],
    [h(100), h(300)],
)

blob.calculate_lazy_hashes()

# Two leaves for KeyId(100) now exist in the blob; the root is corrupted.
# get_proof_of_inclusion returns a path that does not match the actual root.
proof = blob.get_proof_of_inclusion(KeyId(100))
print("proof valid:", proof.valid())   # may print True while root is inconsistent

    fn insert_entry_to_blob(&mut self, index: TreeIndex, block: &Block) -> Result<(), Error> {
        let new_block_bytes = block.to_bytes()?;
        let extend_index = self.extend_index();
        match index.cmp(&extend_index) {
            Ordering::Greater => return Err(Error::BlockIndexOutOfBounds(index)),
            Ordering::Equal => self.blob.extend_from_slice(&new_block_bytes),
            Ordering::Less => {
                self.blob[block_range(index)].copy_from_slice(&new_block_bytes);
            }
        }

        match block.node {
            Node::Leaf(leaf) => self.block_status_cache.add_leaf(index, leaf),
            Node::Internal(..) => self.block_status_cache.add_internal(index),
        }

        Ok(())
    }
