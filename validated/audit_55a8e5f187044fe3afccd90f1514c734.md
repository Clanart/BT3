### Title
Stale Base Fee Snapshot in `NewDynamicFeeChecker` Causes Fee Market Bypass for Cosmos SDK Transactions - (`ante/evm/fee_checker.go`, `evmd/ante/handler_options.go`)

---

### Summary

`NewDynamicFeeChecker`, used for Cosmos SDK (non-EVM) transactions, captures `feemarketParams` at ante handler construction time and never refreshes it. Because the EIP-1559 `BaseFee` is stored inside `feemarketParams.BaseFee` and changes every block, the fee checker permanently operates against a stale base fee. This is the direct analog of the Unlock Protocol bug: a price/fee value is read once and frozen, while the live value diverges.

---

### Finding Description

`newCosmosAnteHandler` (and `newLegacyCosmosAnteHandlerEip712`) are called once — at ante handler construction time — and snapshot `feemarketParams` into a local struct:

```go
// evmd/ante/handler_options.go
func newCosmosAnteHandler(ctx sdk.Context, options HandlerOptions, ...) sdk.AnteHandler {
    evmParams := options.EvmKeeper.GetParams(ctx)
    feemarketParams := options.FeeMarketKeeper.GetParams(ctx)   // ← one-time snapshot
    ...
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
    ...
}
``` [1](#0-0) 

The pointer `&feemarketParams` is passed into `NewDynamicFeeChecker`, which closes over it:

```go
// ante/evm/fee_checker.go
func NewDynamicFeeChecker(ethCfg *params.ChainConfig, evmParams *types.Params, feemarketParams *feemarkettypes.Params) authante.TxFeeChecker {
    return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
        ...
        baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)  // ← reads stale snapshot
        ...
        effectivePrice := sdkmath.NewIntFromBigInt(types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
        ...
    }
}
``` [2](#0-1) 

`types.GetBaseFee` reads `feemarketParams.GetBaseFee()` — the frozen struct field — not the live KV store:

```go
// x/evm/types/utils.go
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    ...
    baseFee := feemarketParams.GetBaseFee()   // ← reads captured snapshot, never updated
    ...
}
``` [3](#0-2) 

Meanwhile, every block `BeginBlock` calls `CalculateBaseFee` and `SetBaseFee`, which writes the new base fee into the KV store via `SetParams`:

```go
// x/feemarket/keeper/params.go
func (k Keeper) SetBaseFee(ctx sdk.Context, baseFee *big.Int) {
    params := k.GetParams(ctx)
    params.BaseFee = ethermint.SaturatedNewInt(baseFee)
    err := k.SetParams(ctx, params)   // ← updates KV store only
    ...
}
``` [4](#0-3) 

The local struct captured in the closure is **never updated**. The live KV store diverges from the snapshot with every block.

**Contrast with EVM transactions**: `newEthAnteHandler` correctly reads the current block's base fee dynamically on every transaction via `EVMBlockConfig(ctx, ...)`:

```go
// evmd/ante/handler_options.go
func newEthAnteHandler(options HandlerOptions) sdk.AnteHandler {
    return func(ctx sdk.Context, tx sdk.Tx, simulate bool) (sdk.Context, error) {
        blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
        ...
        baseFee := blockCfg.BaseFee   // ← live, per-transaction read
        ...
    }
}
``` [5](#0-4) 

`EVMBlockConfig` reads `feemarketParams.GetBaseFee()` fresh from the KV store on every call (or from the object-store cache that is invalidated per block): [6](#0-5) 

---

### Impact Explanation

When `DynamicFeeChecker: true` is set (the production configuration for Cosmos SDK transactions on EIP-1559-enabled chains):

1. **Fee market bypass (under-charging)**: If the base fee has risen since ante handler construction (e.g., due to sustained high block utilization), Cosmos SDK transactions are validated against the stale lower base fee. A transaction with `feeCap` between the stale base fee and the current base fee passes `feeCap.LT(baseFeeInt)` check and is accepted and committed, even though it would be rejected under the correct current base fee. This permits invalid transactions to commit.

2. **User fund mis-accounting (over-charging)**: If the base fee has fallen since construction, the `effectivePrice` is computed against the stale higher base fee, causing users to be charged more than the correct current effective fee.

The base fee can change up to ~12.5% per block (bounded by `BaseFeeChangeDenominator`), so the divergence compounds over time.

---

### Likelihood Explanation

- Triggered by any unprivileged Cosmos SDK transaction (not EVM) submitted to a chain with `DynamicFeeChecker: true` and EIP-1559 enabled.
- The base fee changes every block, so the stale snapshot diverges immediately after the first block following ante handler construction.
- No special privileges, governance access, or validator compromise required — any user submitting a Cosmos SDK transaction exercises this path.
- The `newLegacyCosmosAnteHandlerEip712` path has the identical bug, extending the attack surface to EIP-712 Cosmos transactions. [7](#0-6) 

---

### Recommendation

`NewDynamicFeeChecker` must not close over a snapshot of `feemarketParams`. Instead, it should read the current base fee from the live keeper on every invocation, matching the pattern used by `newEthAnteHandler`:

```go
// Pass the keeper, not a snapshot pointer
func NewDynamicFeeChecker(ethCfg *params.ChainConfig, evmParams *types.Params, feeMarketKeeper FeeMarketKeeper) authante.TxFeeChecker {
    return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
        ...
        baseFee := feeMarketKeeper.GetBaseFee(ctx)  // live read per transaction
        ...
    }
}
```

Alternatively, `newCosmosAnteHandler` should be restructured as a closure (like `newEthAnteHandler`) that reads params from the keeper on every transaction rather than at construction time.

---

### Proof of Concept

1. Chain starts with base fee = 1,000,000,000 (1 Gwei). `newCosmosAnteHandler` is called once, capturing `feemarketParams.BaseFee = 1_000_000_000`.
2. Over 100 blocks of high utilization, the base fee rises to 3,000,000,000 (3 Gwei) via `BeginBlock` → `CalculateBaseFee` → `SetBaseFee`.
3. Attacker submits a Cosmos SDK transaction with `fee = 1,500,000,000 * gasLimit` (1.5 Gwei effective price).
4. `NewDynamicFeeChecker` computes `baseFeeInt = 1_000_000_000` (stale), `feeCap = 1_500_000_000`. Check `feeCap.LT(baseFeeInt)` → false (1.5 > 1.0). Transaction passes.
5. The transaction is committed with a fee below the current 3 Gwei base fee, bypassing the fee market.
6. Under the correct live base fee, `feeCap (1.5) < baseFee (3.0)` → the transaction would be rejected. [8](#0-7)

func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {

		return nil
	}
	baseFee := feemarketParams.GetBaseFee()

	if baseFee == nil {
		return new(big.Int)
	}
	return baseFee
