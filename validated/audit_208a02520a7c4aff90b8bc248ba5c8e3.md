### Title
Missing Nonce in `meta_tx_v0` Hash Enables Unlimited Signature Replay — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/transaction_hash/transaction_hash.cairo` and `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

`compute_meta_tx_v0_hash` constructs the signed digest for the `meta_tx_v0` syscall without including a nonce. Because the hash commits only to `(prefix, version=0, contract_address, selector, calldata, max_fee=0, chain_id)`, any valid signature produced for a `meta_tx_v0` invocation is permanently replayable by any unprivileged contract. Combined with the fact that `execute_meta_tx_v0` never calls `__validate__` and never checks or increments the account nonce, an attacker who observes a single signed `meta_tx_v0` can drain the victim account by replaying the signature an unlimited number of times.

---

### Finding Description

**Root cause — `compute_meta_tx_v0_hash` omits the nonce**

`compute_meta_tx_v0_hash` calls `deprecated_get_transaction_hash` with `additional_data_size=0` and `additional_data=cast(0, felt*)`:

```cairo
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(
    contract_address: felt,
    entry_point_selector: felt,
    calldata: felt*,
    calldata_size: felt,
    chain_id: felt,
) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        tx_hash_prefix=INVOKE_HASH_PREFIX,
        version=0,
        contract_address=contract_address,
        entry_point_selector=entry_point_selector,
        calldata_size=calldata_size,
        calldata=calldata,
        max_fee=0,
        chain_id=chain_id,
        additional_data_size=0,          // ← nonce is NOT included
        additional_data=cast(0, felt*),
    );
    return tx_hash;
}
``` [1](#0-0) 

Compare this with `compute_l1_handler_transaction_hash`, which passes `additional_data_size=1` and `additional_data=&nonce` to bind the hash to a specific nonce: [2](#0-1) 

**Root cause — `execute_meta_tx_v0` hardcodes `nonce=0` and skips `__validate__`**

Inside `execute_meta_tx_v0`, the `new_tx_info` struct is constructed with `nonce=0` unconditionally, and no `check_and_increment_nonce` call is made anywhere in the function:

```cairo
tempvar new_tx_info = new TxInfo(
    version=0,
    account_contract_address=contract_address,
    max_fee=0,
    signature_start=request.signature_start,
    signature_end=request.signature_end,
    transaction_hash=meta_tx_hash,
    chain_id=old_tx_info.chain_id,
    nonce=0,                             // ← always 0, never incremented
    ...
);
``` [3](#0-2) 

The function then directly calls `contract_call_helper`, which invokes `__execute__` on the target contract — bypassing `run_validate` entirely: [4](#0-3) 

For comparison, `execute_invoke_function_transaction` always calls `run_validate` and `check_and_increment_nonce` before execution: [5](#0-4) 

**Why the account contract cannot self-protect**

Standard StarkNet account contracts perform nonce management inside `__validate__`, not inside `__execute__`. Since `meta_tx_v0` only calls `__execute__` (enforced by the `selector != EXECUTE_ENTRY_POINT_SELECTOR` guard): [6](#0-5) 

…the account's nonce guard is never reached. The OS-level nonce state for the account is also never touched, so the on-chain nonce remains unchanged after every replay.

---

### Impact Explanation

**Critical — Direct loss of funds.**

Once a victim account owner (or any contract acting on their behalf) emits a single `meta_tx_v0` syscall whose calldata encodes a token transfer or other value-moving operation, the signature is permanently valid. An attacker who extracts that signature from the transaction trace can replay it an unlimited number of times by deploying a contract that issues the same `meta_tx_v0` syscall. Each replay executes `__execute__` on the victim account with the original calldata, draining the account's balance until it is empty.

---

### Likelihood Explanation

**High.** The `meta_tx_v0` syscall is callable by any unprivileged contract deployer or transaction sender — no privileged role is required. The signature is observable from public transaction data. The only cost to the attacker is gas per replay. The attack is deterministic and requires no probabilistic assumptions.

---

### Recommendation

Include the account nonce in the `meta_tx_v0` hash, mirroring how `compute_l1_handler_transaction_hash` binds its hash to a nonce via `additional_data`:

```cairo
func compute_meta_tx_v0_hash{pedersen_ptr: HashBuiltin*}(
    contract_address: felt,
    entry_point_selector: felt,
    calldata: felt*,
    calldata_size: felt,
    chain_id: felt,
    nonce: felt,                          // ← add nonce parameter
) -> felt {
    let (tx_hash) = deprecated_get_transaction_hash{hash_ptr=pedersen_ptr}(
        ...
        additional_data_size=1,           // ← include nonce
        additional_data=&nonce,
    );
    return tx_hash;
}
```

Additionally, `execute_meta_tx_v0` must read the current account nonce from `contract_state_changes`, assert it matches the nonce in the request, and increment it — analogous to `check_and_increment_nonce` used for regular account transactions.

---

### Proof of Concept

1. Alice's account is at address `A`. She authorises a contract `C1` to call `meta_tx_v0` on her behalf, providing a signature `σ` over `hash(INVOKE_HASH_PREFIX, 0, A, __execute__, calldata_transfer_100_ETH, 0, chain_id)`.
2. `C1` calls `meta_tx_v0(contract_address=A, selector=__execute__, calldata=calldata_transfer_100_ETH, signature=σ)`. The OS computes the same hash (no nonce), verifies `σ` inside Alice's `__execute__`, and executes the transfer. Alice's on-chain nonce is **not incremented**.
3. Bob observes `σ` and `calldata_transfer_100_ETH` from the public transaction trace.
4. Bob deploys contract `C2` with a single function that calls `meta_tx_v0(contract_address=A, selector=__execute__, calldata=calldata_transfer_100_ETH, signature=σ)`.
5. Bob submits a transaction invoking `C2`. The OS recomputes the identical hash (still no nonce, still `nonce=0` in `new_tx_info`), `σ` verifies correctly, and the transfer executes again.
6. Bob repeats step 5 until Alice's account is empty. Each iteration costs only gas.
