### Title
Missing Class Hash Declaration Validation in `execute_replace_class` Allows OS Proof Failure via Undeclared Class Invocation — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in `syscall_impls.cairo` does not verify that the new class hash supplied by the caller is a declared class. An unprivileged contract can replace its own class hash with an arbitrary, undeclared value. If that contract is subsequently called within the same transaction (via `call_contract`), the OS will attempt to resolve the undeclared class hash through `find_element` in `execute_entry_point.cairo`. Because `find_element` has no graceful failure path — it panics when the key is absent — the OS proof fails, preventing the block from being finalized and causing a network halt.

---

### Finding Description

**Root cause — `execute_replace_class` (syscall_impls.cairo, lines 878–916):**

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The TODO comment is an explicit acknowledgment that the check is absent. `class_hash` is taken verbatim from the syscall request and written into `contract_state_changes` without any assertion that it corresponds to a previously declared class in `contract_class_changes`.

**Propagation — `execute_entry_point` (execute_entry_point.cairo, lines 154–166):**

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);

let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
```

When the class hash is undeclared, `dict_read` on `contract_class_changes` returns the default value `0`. `find_element` then searches for a `CompiledClassFact` with `hash = 0`. No such fact exists in an honest block. `find_element` has **no success/failure return path** — unlike `search_sorted_optimistic` used in `get_entry_point` for missing selectors, `find_element` panics when the key is absent, terminating the Cairo program and invalidating the proof.

**Contrast with graceful handling elsewhere:**

`get_entry_point` (execute_entry_point.cairo, lines 122–132) uses `search_sorted_optimistic`, which returns a `success` flag and allows the OS to return `is_reverted=1` gracefully when a selector is missing. No equivalent fallback exists for a missing compiled class.

---

### Impact Explanation

**Impact: High — Network not being able to confirm new transactions (total network shutdown).**

When the OS proof fails, the block cannot be finalized on L1. The sequencer's execution model handles a call to a contract with an undeclared class hash by reverting the inner call gracefully and continuing the outer transaction. The OS, however, panics unconditionally at `find_element` before any revert logic can execute. This mismatch between the sequencer's execution model and the OS proof means the sequencer includes a transaction it considers valid, but the OS cannot prove it. The block is unprovable, and no further blocks can be confirmed until the issue is resolved — a total network halt.

---

### Likelihood Explanation

**Likelihood: High.**

The attack requires no privileged access. Any user can:
1. Declare a class and deploy a contract from it.
2. Have the contract call `replace_class` with an arbitrary undeclared felt value.
3. Have the same contract call itself (or be called by another contract in the same transaction) after the class replacement.

The sequencer executes step 3 as a gracefully reverted inner call and includes the transaction in the block. The OS then fails to prove the block. The TODO comment in the production code confirms the check is intentionally deferred, making the window of exploitation open until the fix is applied.

---

### Recommendation

Inside `execute_replace_class`, before writing the new class hash to `contract_state_changes`, assert that the class hash exists in `contract_class_changes` (i.e., it has a non-zero compiled class hash entry). This is exactly what the existing TODO comment calls for:

```cairo
// Verify the class is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```
