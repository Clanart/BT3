### Title
Permanently Locked `nativeFee` ETH in OmniBridge — No Admin Withdrawal Function - (File: `evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.initTransfer` accepts a `nativeFee` amount in ETH from every caller. This ETH accumulates inside the contract but there is no admin or privileged function anywhere in `OmniBridge` or `OmniBridgeWormhole` to withdraw it. The contract also exposes an unconditional `receive() external payable {}`, compounding the accumulation surface. The result is that all `nativeFee` ETH paid by bridge users is permanently locked and unclaimable.

---

### Finding Description

`initTransfer` computes `extensionValue` as `msg.value - nativeFee` (ERC-20 path) or `msg.value - amount - nativeFee` (native ETH path) and forwards only `extensionValue` to `initTransferExtension`. [1](#0-0) 

In `OmniBridgeWormhole.initTransferExtension`, only `value` (i.e., `extensionValue`) is forwarded to the Wormhole `publishMessage` call; `nativeFee` is never forwarded anywhere. [2](#0-1) 

The base `OmniBridge` contract lists every admin function (`addCustomToken`, `removeCustomToken`, `setMetadata`, `pause`, `upgradeToken`, `setNearBridgeDerivedAddress`) but contains no ETH withdrawal function. [3](#0-2) 

The unconditional `receive()` further allows arbitrary ETH to be sent directly to the contract with no recovery path. [4](#0-3) 

On the NEAR side, `claim_fee_callback` pays out only the token-denominated fee (`transfer_message.amount.0 - denormalized_amount`) in the bridged token; it never touches or returns the `nativeFee` ETH that was locked on the EVM side. [5](#0-4) 

---

### Impact Explanation

Every user who calls `initTransfer` with a non-zero `nativeFee` permanently loses that ETH. The ETH is held by the contract with no on-chain path for the user, the relayer, or the admin to recover it. This constitutes an **irrecoverable lock of user funds** in the bridge contract — matching the Critical impact class: *"Permanent freezing, irrecoverable lock, or unclaimable settlement of user or protocol funds in bridge, token, or vault flows."*

---

### Likelihood Explanation

`nativeFee` is a first-class parameter of the public `initTransfer` interface and is explicitly emitted in the `InitTransfer` event. Any user or relayer that sets a non-zero `nativeFee` (e.g., to incentivize relayers to cover destination-chain gas) triggers the lock. The likelihood is **medium-to-high**: the parameter exists precisely to be used, and the NEAR-side fee claim mechanism does not compensate for it, so users and integrators who follow the documented fee pattern will lose funds.

---

### Recommendation

Add an admin-only ETH withdrawal function to `OmniBridge`, analogous to the fix described in the referenced report:

```solidity
function withdrawNativeFees(address payable recipient, uint256 amount)
    external
    onlyRole(DEFAULT_ADMIN_ROLE)
{
    (bool success, ) = recipient.call{value: amount}("");
    if (!success) revert FailedToSendEther();
}
```

Alternatively, track accumulated `nativeFee` per relayer and allow each relayer to claim their own share.

---

### Proof of Concept

1. Alice calls `initTransfer(usdcAddress, 1000e6, 0, 0.005 ether, "near:alice.near", "")` sending `msg.value = 0.005 ether`.
2. Inside `initTransfer`: `extensionValue = 0.005 ether - 0.005 ether = 0`. The USDC is pulled from Alice. `initTransferExtension` is called with `value = 0`.
3. In `OmniBridgeWormhole.initTransferExtension`, `_wormhole.publishMessage{value: 0}(...)` is called. The `0.005 ether` `nativeFee` remains in the contract.
4. The relayer finalizes on NEAR and calls `claim_fee`. NEAR pays out the token fee in USDC. The `0.005 ether` is never referenced.
5. No function in `OmniBridge` or `OmniBridgeWormhole` can move that ETH. It is permanently locked. [6](#0-5) [2](#0-1) [7](#0-6)

    function initTransfer(
        address tokenAddress,
        uint128 amount,
        uint128 fee,
        uint128 nativeFee,
        string calldata recipient,
        string calldata message
    ) external payable whenNotPaused(PAUSED_INIT_TRANSFER) {
        currentOriginNonce += 1;
        if (fee >= amount) {
            revert InvalidFee();
        }

        uint256 extensionValue;
        if (tokenAddress == address(0)) {
            if (fee != 0) {
                revert InvalidFee();
            }
            extensionValue = msg.value - amount - nativeFee;
        } else {
            extensionValue = msg.value - nativeFee;
            if (customMinters[tokenAddress] != address(0)) {
                IERC20(tokenAddress).safeTransferFrom(
                    msg.sender,
                    customMinters[tokenAddress],
                    amount
                );
                ICustomMinter(customMinters[tokenAddress]).burn(
                    tokenAddress,
                    amount
                );
            } else if (isBridgeToken[tokenAddress]) {
                BridgeToken(tokenAddress).burn(msg.sender, amount);
            } else {
                IERC20(tokenAddress).safeTransferFrom(
                    msg.sender,
                    address(this),
                    amount
                );
            }
        }

            msg.sender,
            tokenAddress,
            currentOriginNonce,
            amount,
            fee,
            nativeFee,
            recipient,
            message,
            extensionValue

            amount,
            fee,

            message

    pub fn claim_fee(&mut self, #[serializer(borsh)] args: ClaimFeeArgs) -> Promise {
        self.verify_proof(args.chain_kind, args.prover_args).then(
            Self::ext(env::current_account_id())
                .with_attached_deposit(env::attached_deposit())
                .with_static_gas(CLAIM_FEE_CALLBACK_GAS)
                .claim_fee_callback(&env::predecessor_account_id()),
        )
    }

    #[private]
    #[payable]
    pub fn claim_fee_callback(
        &mut self,
        #[serializer(borsh)] predecessor_account_id: &AccountId,
        #[callback_result]
        #[serializer(borsh)]
        call_result: Result<ProverResult, PromiseError>,
    ) -> PromiseOrValue<()> {
        let Ok(ProverResult::FinTransfer(fin_transfer)) = call_result else {
            env::panic_str(BridgeError::InvalidProofMessage.to_string().as_str())
        };

        let fee_recipient = fin_transfer.fee_recipient.unwrap_or_else(|| {
            env::panic_str(BridgeError::FeeRecipientNotSetOrEmpty.to_string().as_str());
        });

        require!(
            fee_recipient == *predecessor_account_id,
            BridgeError::OnlyFeeRecipientCanClaim.as_ref()
        );
        require!(
            self.factories
                .get(&fin_transfer.emitter_address.get_chain())
                == Some(fin_transfer.emitter_address),
            BridgeError::UnknownFactory.as_ref()
        );

        let transfer_message = self.remove_transfer_message(fin_transfer.transfer_id);

        if let Some(origin_transfer_id) = transfer_message.origin_transfer_id.clone() {
            let mut fast_transfer = FastTransfer::from_transfer(
                transfer_message.clone(),
                self.get_token_id(&transfer_message.token),
            );
            fast_transfer.transfer_id = origin_transfer_id;

            if let Some(fast_transfer_status) = self.get_fast_transfer_status(&fast_transfer.id()) {
                // For fast transfers we need to wait for finalization of the first leg (Origin chain -> Near) before allowing fee claim.
                // This confirms that fast transfer was executed with correct parameters.
                // Othewise malicious relayer can create a fast transfer with arbitrary high fee and claim it here.
                if fast_transfer_status.finalised {
                    self.remove_fast_transfer(&fast_transfer.id());
                } else {
                    env::panic_str(BridgeError::FastTransferNotFinalised.to_string().as_str());
                }
            }
        }

        let token = self.get_token_id(&transfer_message.token);
        let token_address = self
            .get_token_address(transfer_message.get_destination_chain(), token.clone())
            .unwrap_or_else(|| {
                env::panic_str(BridgeError::FailedToGetTokenAddress.to_string().as_str())
            });

        let denormalized_amount = Self::denormalize_amount(
            fin_transfer.amount.0,
            self.token_decimals
                .get(&token_address)
                .near_expect(BridgeError::TokenDecimalsNotFound),
        );
        // Fee includes both the user-specified fee and any dust lost during decimal
        // normalization (see `normalize_amount`). Since `denormalize(normalize(x)) <= x`
        // due to floor division, the difference naturally captures the normalization remainder.
        let fee = transfer_message.amount.0 - denormalized_amount;

        self.send_fee_internal(&transfer_message, fee_recipient, fee)
    }
