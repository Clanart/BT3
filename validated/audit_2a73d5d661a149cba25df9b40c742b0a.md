### Title
`deprecated_declared_classes` Excluded from `CommitmentStateDiff→ThinStateDiff` Conversion Produces Wrong `state_diff_commitment` and `concatenated_counts` in Block Hash — (`crates/blockifier/src/state/cached_state.rs`)

### Summary

`From<CommitmentStateDiff> for ThinStateDiff` unconditionally sets `deprecated_declared_classes: Vec::new()`. Every block-commitment path that starts from a `CommitmentStateDiff` (the sequencer's own block production path) therefore computes a `state_diff_commitment` and a `state_diff_length`-encoded `concatenated_counts` that silently omit any Cairo-0 (deprecated) class declarations present in the block. The resulting `PartialBlockHashComponents` — and therefore the final block hash — are wrong for any block that contains a `DeclareV0`/`DeclareV1` transaction.

---

### Finding Description

**Root cause — `crates/blockifier/src/state/cached_state.rs` lines 690–701**

```rust
impl From<CommitmentStateDiff> for ThinStateDiff {
    fn from(commitment_state_diff: CommitmentStateDiff) -> Self {
        Self {
            deployed_contracts: commitment_state_diff.address_to_class_hash,
            storage_diffs: commitment_state_diff.storage_updates,
            class_hash_to_compiled_class_hash: commitment_state_diff
                .class_hash_to_compiled_class_hash,
            nonces: commitment_state_diff.address_to_nonce,
            // TODO(AlonH): Remove this when the structure of storage diffs changes.
            deprecated_declared_classes: Vec::new(),   // ← always empty
        }
    }
}
```

`CommitmentStateDiff` has no field for deprecated (Cairo-0) class declarations because `StateMaps` — from which it is built — does not track them. The `Vec::new()` is therefore a structural omission, not a deliberate exclusion.

**Commitment path — `crates/apollo_batcher/src/block_builder.rs` lines 160–166**

```rust

    &transactions_data,
    ThinStateDiff::from(commitment_state_diff.clone()),  // deprecated_declared_classes = []
    l1_da_mode,
    &block_info.starknet_version,
)

```

`calculate_block_commitments` (`crates/starknet_api/src/block_hash/block_hash_calculator.rs` lines 318–327) calls:

```rust
let concatenated_counts = concat_counts(
    transactions_data.len(),
    event_leaf_elements.len(),
    state_diff.len(),          // ← undercounted: missing deprecated_declared_classes
    l1_da_mode,
);

```

and spawns `calculate_state_diff_hash(&state_diff)` (`crates/starknet_api/src/block_hash/state_diff_hash.rs` lines 30–42):

```rust
pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    ...
    hash_chain = chain_deprecated_declared_classes(
        &state_diff.deprecated_declared_classes,  // always []
        hash_chain,
    );
    ...
}
```

`chain_deprecated_declared_classes` chains `[0]` (count = 0) instead of `[N, class_hash_0, …, class_hash_N-1]`.

The resulting `BlockHeaderCommitments` — containing the wrong `state_diff_commitment` and wrong `concatenated_counts` — is stored in `PartialBlockHashComponents` and propagated to the final block hash.

**Stored state diff — `BlockExecutionArtifacts::thin_state_diff()` line 200**

```rust
pub fn thin_state_diff(&self) -> ThinStateDiff {
    ThinStateDiff::from(self.commitment_state_diff.clone())  // deprecated_declared_classes = []
}
```

The `ThinStateDiff` written to MDBX storage via `append_state_diff` also has empty `deprecated_declared_classes`, so the stored `state_diff_length` (derived from `ThinStateDiff::len()`) is equally undercounted.

**P2P sync impact — `crates/apollo_p2p_sync/src/client/state_diff.rs` lines 62–104**

The P2P sync client reads `state_diff_length` from the stored block header as `target_state_diff_len` and loops until `current_state_diff_len == target_state_diff_len`. Because the sequencer stored an undercounted length, a syncing peer terminates the loop early and commits a `ThinStateDiff` that is missing all `DeprecatedDeclaredClass` chunks for that block.

---

### Impact Explanation

For any block containing one or more `DeclareV0`/`DeclareV1` transactions:

| Corrupted value | Location | Effect |
|---|---|---|
| `state_diff_commitment` | `BlockHeaderCommitments` / block header | Wrong Poseidon hash; block hash is wrong |
| `concatenated_counts` | `BlockHeaderCommitments` / block hash | `state_diff_length` field undercounted by N deprecated declarations |
| Stored `ThinStateDiff` | MDBX `state_diffs` table | Missing `deprecated_declared_classes`; wrong `state_diff_length` |
| P2P-synced `ThinStateDiff` | Syncing node storage | Incomplete state diff accepted as complete |

The block hash mismatch means consensus nodes that independently recompute the hash will disagree, and any downstream proof that commits to the block hash (SNOS `block_hash` field in `SnosProofFacts`) will be invalid.

---

### Likelihood Explanation

`DeclareV0`/`DeclareV1` transactions are still valid Starknet transaction types and can be submitted by any unprivileged user. The trigger requires only that one such transaction be included in a sequenced block. No privileged access or malicious peer is needed.

---

### Recommendation

`CommitmentStateDiff` (or `StateMaps`) must be extended to carry deprecated class declarations so that the `From<CommitmentStateDiff> for ThinStateDiff` conversion can populate `deprecated_declared_classes` correctly. Until then, the batcher should reconstruct the full `ThinStateDiff` from the complete `StateDiff` (which does carry `deprecated_declared_classes`) rather than from `CommitmentStateDiff`.

---

### Proof of Concept

1. Submit a `DeclareV1` transaction declaring a Cairo-0 class hash `C`.
2. The sequencer executes the block; `BlockExecutionSummary.state_diff` is a `CommitmentStateDiff` with no record of `C`.
3. `BlockExecutionArtifacts::new()` calls `ThinStateDiff::from(commitment_state_diff.clone())` → `deprecated_declared_classes = []`.
4. `calculate_block_commitments` computes:
   - `state_diff.len()` = actual length − 1 (missing `C`)
   - `calculate_state_diff_hash` chains `[0]` for deprecated classes instead of `[1, C]`
5. The stored block header carries `state_diff_commitment = H_wrong` and `state_diff_length = L − 1`.
6. A second node syncing via P2P reads `target_state_diff_len = L − 1`, receives chunks summing to `L − 1`, and stores a `ThinStateDiff` without `C` — accepting it as complete.
7. Both nodes now hold a state that omits the declaration of class `C`, and the block hash diverges from any node that correctly computes it. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) [7](#0-6)

    hash_chain =

    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}

    /// This has the same value as `state_diff_length` in the corresponding `BlockHeader`.
    pub fn len(&self) -> usize {

        result += self.class_hash_to_compiled_class_hash.len();
