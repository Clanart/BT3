### Title
EIP-2935 Ring Buffer Collision in `GetHeaderHash` Causes `BLOCKHASH` Opcode to Return Wrong Non-Zero Hash for Out-of-Window Blocks — (File: `x/evm/keeper/keeper.go`, `x/evm/keeper/state_transition.go`)

---

### Summary

`GetHashFn` in Ethermint calls `GetHeaderHash` for **all** historical block numbers without first checking whether the requested block is within the valid `headerHashNum`-block window. After the EIP-2935 history contract is deployed, `GetHeaderHash` reads from a ring buffer keyed by `height % window`. For any block number `N` that is outside the valid window (`N < upper - headerHashNum`), the ring slot `N % window` already holds the hash of a **different, more recent block** at the same ring index. Because that hash is non-zero, `GetHashFn` returns it as if it were the correct hash for block `N`, instead of returning `common.Hash{}` (zero) as the EVM specification requires.

---

### Finding Description

**Root cause — `GetHashFn` in `x/evm/keeper/state_transition.go` lines 121–140:**

```go
if upper > num64 {
    // The requested height is historical, query EIP-2935 contract storage
    headerHash := k.GetHeaderHash(ctx, num64)
    if headerHash.Cmp(common.Hash{}) != 0 {
        return headerHash          // ← returned WITHOUT checking num64 >= lower
    } else if num64 >= lower {
        // Pre upgrade case / staking fallback
        ...
    }
}
```

The `lower` bound (`upper - headerHashNum`, default `upper - 256`) is computed but is only applied to the staking-fallback branch. The EIP-2935 path (`GetHeaderHash`) is invoked for every historical block, including those far outside the window.

**Ring buffer collision — `GetHeaderHash` in `x/evm/keeper/keeper.go` lines 384–391:**

```go
ringIndex := height % window          // window = 8191 by default
var key common.Hash
binary.BigEndian.PutUint64(key[24:], ringIndex)
hash := k.GetState(ctx, ethparams.HistoryStorageAddress, key)

if hash.Cmp(common.Hash{}) != 0 {
    return hash                        // ← returns hash for a DIFFERENT block
}
```

The ring buffer stores one hash per slot `height % 8191`. After the chain has run for more than 8191 blocks, every slot is populated. When `GetHeaderHash(ctx, N)` is called for an out-of-window `N`, slot `N % 8191` holds the hash of the most recent block `N + k*8191` that falls within the window — a completely different block. That non-zero hash is returned and propagated back through `GetHashFn` as the answer for block `N`.

**`SetHeaderHash` writes the current block's hash to the ring slot unconditionally:**

```go
ringIndex := uint64(ctx.BlockHeight()) % window
k.SetState(ctx, ethparams.HistoryStorageAddress, key, ctx.HeaderHash())
```

Every block overwrites the slot, so after `window` blocks every slot is non-zero.

---

### Impact Explanation

The `BLOCKHASH(N)` opcode in the EVM is supposed to return `0x00…00` for any block number outside the most-recent-256-block window (per the Yellow Paper and go-ethereum). Ethermint instead returns the hash of a **different block** at the same ring index. This is incorrect consensus-critical data injected into EVM execution.

Concrete consequences:
1. **Randomness manipulation**: Any on-chain contract that uses `BLOCKHASH(N)` for randomness and guards against the "zero means unavailable" case will receive a predictable, attacker-known non-zero value. An attacker who knows the chain height can compute exactly which block's hash will be returned and pre-simulate the outcome.
2. **Commit-reveal bypass**: Contracts that use `blockhash(commitBlock) == 0` as a sentinel to detect expired commitments will never see zero after the ring fills, allowing stale commitments to remain valid indefinitely.
3. **Cross-block determinism**: The wrong hash is deterministic across all validators (they all compute the same wrong value), so it does not cause consensus failure, but it silently corrupts EVM application logic that depends on the Ethereum-specified behavior.

This matches the allowed High impact: **"EVM state transition… bug that permits… valid user funds/fees to be mis-accounted"** and **"Public JSON-RPC… path feeds incorrect consensus-critical data into transaction execution."**

---

### Likelihood Explanation

- **Trigger condition**: EIP-2935 history contract deployed (it is in `DefaultPreinstalls`) AND chain height > `HistoryServeWindow` (8191 blocks, roughly a few hours on a 1-second block chain). Both conditions are met on any production deployment.
- **Attacker entry path**: Any unprivileged EVM transaction or `eth_call` that invokes `BLOCKHASH(N)` with `N < currentBlock - 256`. No special privileges required.
- **Predictability**: The wrong hash returned is fully deterministic and publicly computable from on-chain data, making exploitation straightforward for any attacker who monitors the chain.

---

### Recommendation

Apply the `lower` bound check **before** calling `GetHeaderHash`, not only in the staking-fallback branch:

```go
if upper > num64 {
    if num64 >= lower {
        headerHash := k.GetHeaderHash(ctx, num64)
        if headerHash.Cmp(common.Hash{}) != 0 {
            return headerHash
        }
        // Pre upgrade case / staking fallback
        histInfo, err := k.stakingKeeper.GetHistoricalInfo(ctx, h)
        ...
    }
}
return common.Hash{}
```

This ensures that out-of-window block numbers always return `common.Hash{}`, matching Ethereum's specified behavior.

### Proof of Concept

**Setup**: Chain has been running for > 8191 blocks. EIP-2935 contract is deployed. `headerHashNum = 256`, `HistoryServeWindow = 8191`. Current block height `upper = 10000`.

**Valid window**: blocks 9744–10000.

**Attack**:
1. Deploy a gambling contract that uses `BLOCKHASH(1000)` as a randomness source, with a guard `require(blockhash(1000) != 0, "expired")`.
2. On Ethereum, `BLOCKHASH(1000)` returns `0` (block 1000 is 9000 blocks ago, outside the 256-block window), so the guard reverts.
3. On Ethermint with EIP-2935 deployed:
   - `GetHashFn` calls `GetHeaderHash(ctx, 1000)`.
   - Ring index: `1000 % 8191 = 1000`.
   - Slot 1000 was last written at block `1000 + 8191 = 9191` (within the last 8191 blocks, so it is populated).
   - `GetHeaderHash` returns `hash(block 9191)` — non-zero.
   - `GetHashFn` returns this hash without checking `1000 >= lower (9744)`.
4. The guard passes. The attacker pre-computes `keccak256(hash(block 9191))` off-chain, knows the "random" outcome, and submits only winning transactions. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

func (k Keeper) GetHashFn(ctx sdk.Context, headerHashNum uint64) vm.GetHashFunc {
	return func(num64 uint64) common.Hash {
		h, err := ethermint.SafeInt64(num64)
		if err != nil {
			return common.Hash{}
		}
		upper, err := ethermint.SafeUint64(ctx.BlockHeight())
		if err != nil {
			return common.Hash{}
		}
		if upper == num64 {
			headerHash := ctx.HeaderHash()
			if len(headerHash) > 0 {
				return common.BytesToHash(headerHash)
			}
		}
		// Align check with https://github.com/ethereum/go-ethereum/blob/release/1.11/core/vm/instructions.go#L433
		var lower uint64
		if upper <= headerHashNum {
			lower = 0
		} else {
			lower = upper - headerHashNum
		}

		if upper > num64 {
			// The requested height is historical, query EIP-2935 contract storage
			headerHash := k.GetHeaderHash(ctx, num64)
			if headerHash.Cmp(common.Hash{}) != 0 {
				return headerHash
			} else if num64 >= lower {
				// Pre upgrade case
				// In case EIP-2935 is not supported and data cannot be found, we fetch historical info
				histInfo, err := k.stakingKeeper.GetHistoricalInfo(ctx, h)
				if err != nil {
					k.Logger(ctx).Debug("historical info not found", "height", h, "err", err.Error())
					return common.Hash{}
				}
				header, err := cmttypes.HeaderFromProto(&histInfo.Header)
				if err != nil {
					k.Logger(ctx).Error("failed to cast tendermint header from proto", "error", err)
					return common.Hash{}
				}
				return common.BytesToHash(header.Hash())
			}
		}
		return common.Hash{}
	}
}

// SetHeaderHash stores the hash of the current block header in the store.
func (k Keeper) SetHeaderHash(ctx sdk.Context) {
	acct := k.GetAccount(ctx, ethparams.HistoryStorageAddress)
	if acct != nil && acct.IsContract() {
		window := types.DefaultHistoryServeWindow
		params := k.GetParams(ctx)
		if params.HistoryServeWindow > 0 {
			window = params.HistoryServeWindow
		}
		// set current block hash in the contract storage, compatible with EIP-2935
		ringIndex := uint64(ctx.BlockHeight()) % window //nolint:gosec // G115 // won't exceed uint64

		binary.BigEndian.PutUint64(key[24:], ringIndex)
		k.SetState(ctx, ethparams.HistoryStorageAddress, key, ctx.HeaderHash())

		if err != nil {
			panic(err)
		}

	}
}

// GetHeaderHash sets block hash into EIP-2935 compatible storage contract.
func (k Keeper) GetHeaderHash(ctx sdk.Context, height uint64) common.Hash {
	// check if history contract has been deployed
	acct := k.GetAccount(ctx, ethparams.HistoryStorageAddress)
	if acct != nil && acct.IsContract() {
		window := types.DefaultHistoryServeWindow
		params := k.GetParams(ctx)
		if params.HistoryServeWindow > 0 {
			window = params.HistoryServeWindow
		}

		ringIndex := height % window
		var key common.Hash
		binary.BigEndian.PutUint64(key[24:], ringIndex)
		hash := k.GetState(ctx, ethparams.HistoryStorageAddress, key)

		if hash.Cmp(common.Hash{}) != 0 {
			return hash
		}
	}
	// fall back to old behavior for retro compatibility
	// TODO can be removed along with DeleteHeaderHash once HistoryStorage has been filled up in next protocol upgrade
	store := ctx.KVStore(k.storeKey)
	hashByte := store.Get(types.GetHeaderHashKey(height))
	if len(hashByte) > 0 {
		return common.BytesToHash(hashByte)
	}
	return common.Hash{}
}
