### Title
Integer Division Truncation in `NewDynamicFeeChecker` Causes Zero Effective Fee for Cosmos SDK Transactions When Base Fee Is Driven to Zero - (File: ante/evm/fee_checker.go)

### Summary

`NewDynamicFeeChecker` computes `feeCap = fee / gas` via integer division. When `fee < gas` (which occurs whenever `MinGasPrice < 1`), `feeCap` truncates to 0. A second truncation in `CalculateBaseFee` — `minGasPrice := params.MinGasPrice.TruncateInt().BigInt()` — allows the base fee to fall to 0 when `MinGasPrice` is a fractional value. Together, these two truncations allow Cosmos SDK transactions to pass all ante-handler fee checks while paying zero effective fees.

### Finding Description

**Root cause 1 — `feeCap` truncation in `NewDynamicFeeChecker`:** [1](#0-0) 

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))   // integer division
...
if feeCap.LT(baseFeeInt) { return error }
effectivePrice := EffectiveGasPrice(baseFeeInt, feeCap, maxPriorityPrice)
effectiveFee = effectivePrice.Mul(gas)
```

When `fee < gas`, `feeCap = floor(fee/gas) = 0`. If `baseFeeInt = 0`, the guard passes and `effectiveFee = 0 * gas = 0`.

**Root cause 2 — `MinGasPrice` truncation in `CalculateBaseFee`:** [2](#0-1) 

```go
minGasPrice := params.MinGasPrice.TruncateInt().BigInt()
return ethermint.BigMax(x.Sub(parentBaseFee, baseFeeDelta), minGasPrice)
```

`MinGasPrice` is a `sdk.Dec`. When governance sets it to any fractional value (e.g., `0.5`), `TruncateInt()` yields `0`, so the base fee lower-bound becomes `0`. The base fee is then free to decrease all the way to `0` over successive low-utilization blocks.

**Combined exploit path:**

1. Governance sets `MinGasPrice = 0.5` (a valid, non-zero fractional value).
2. Due to root cause 2, `CalculateBaseFee` clamps the base fee at `0` instead of `0.5`, so `baseFee` eventually reaches `0`.
3. A user submits a Cosmos SDK tx with `gas = 100`, `fee = 50` (satisfies `MinGasPriceDecorator`: `50 >= ceil(0.5 × 100) = 50`).
4. `NewDynamicFeeChecker` computes `feeCap = floor(50/100) = 0`.
5. Guard: `feeCap (0) >= baseFeeInt (0)` → passes.
6. `effectivePrice = min(0 + MaxInt64, 0) = 0`.
7. `effectiveFee = 0 × 100 = 0`.
8. `DeductFeeDecorator` deducts `0` coins. [3](#0-2) 

The `fee` variable is overwritten with `effectiveFee = 0` from `txFeeChecker`, and `checkDeductFee` skips deduction because `fee.IsZero()`. [4](#0-3) 

The `MinGasPriceDecorator` checks the *submitted* `feeCoins`, not the effective fee, so it passes while `DeductFeeDecorator` deducts nothing. [5](#0-4) 

### Impact Explanation

Cosmos SDK transactions are executed with zero fees deducted from the sender, even when `MinGasPrice > 0`. The fee collector module receives nothing. This is a direct fee mis-accounting bug in the ante handler: valid user fees that the protocol requires are never collected. Any user can exploit this once `MinGasPrice` is set to a fractional value and the base fee has dropped to zero.

### Likelihood Explanation

`MinGasPrice` is a governance-settable `sdk.Dec` parameter with a documented purpose of enforcing a global minimum. Setting it to a sub-integer value (e.g., `0.5`) is a natural and expected configuration on chains where the EVM denom has high precision (e.g., `aevmos` at 10⁻¹⁸). Once set, the base fee automatically drops to zero within a few low-utilization blocks, requiring no further attacker action. Any user can then submit Cosmos SDK transactions for free.

### Recommendation

1. **Fix `CalculateBaseFee`**: Use the full decimal `MinGasPrice` as the lower bound, not its truncated integer. Either compare using `sdk.Dec` arithmetic or multiply by `10^18` before truncating to preserve sub-unit precision:

```go
// Instead of:
minGasPrice := params.MinGasPrice.TruncateInt().BigInt()
// Use:
minGasPrice := params.MinGasPrice.TruncateInt().BigInt()
// Only if MinGasPrice >= 1; otherwise use ceiling:
minGasPrice := params.MinGasPrice.Ceil().TruncateInt().BigInt()
```

2. **Fix `NewDynamicFeeChecker`**: Compute `feeCap` using decimal arithmetic to avoid truncation to zero:

```go
// Instead of integer division:
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
// Use ceiling division or decimal comparison:
feeCapDec := sdkmath.LegacyNewDecFromInt(fee).Quo(sdkmath.LegacyNewDecFromUint64(gas))
if feeCapDec.LT(sdkmath.LegacyNewDecFromBigInt(baseFee)) { return error }
```

### Proof of Concept

```

  MinGasPrice = 0.5 (sdk.Dec, set via governance)
  BaseFeeChangeDenominator = 8, ElasticityMultiplier = 2
  Chain runs several empty blocks → baseFee decreases each block
  
In CalculateBaseFee (each empty block):
  minGasPrice = TruncateInt(0.5) = 0
  baseFee = max(parentBaseFee - delta, 0) → eventually 0

Attack tx (Cosmos SDK, not EVM):
  gas    = 100
  fee    = 50  (passes MinGasPriceDecorator: 50 >= ceil(0.5 * 100) = 50)

In NewDynamicFeeChecker:
  feeCap        = floor(50 / 100) = 0
  baseFeeInt    = 0
  0 >= 0        → guard passes
  effectivePrice = min(0 + MaxInt64, 0) = 0
  effectiveFee  = 0 * 100 = 0

In DeductFeeDecorator:
  fee = effectiveFee = 0
  fee.IsZero() → skip deduction

Result: transaction executes, 0 fees collected, 50 coins remain in attacker's account.
