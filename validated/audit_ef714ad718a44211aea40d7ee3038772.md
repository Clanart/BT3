### Title
Missing Class Hash Existence Validation in `execute_replace_class` Enables Permanent Freezing of Contract Funds — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not validate that the new class hash supplied by a contract corresponds to any declared class. Any contract can therefore replace its own class with an arbitrary, undeclared felt, permanently breaking the contract and freezing all funds it holds. The missing check is explicitly acknowledged by a TODO comment in the production OS code.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` reads the caller-supplied `class_hash` from the syscall request and writes it directly into `contract_state_changes` with no existence check against `contract_class_changes` (the current-block declaration dict) or the existing class tree:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The TODO comment at line 898 is the developers' own acknowledgement that the class-existence guard is absent. The OS performs only a gas deduction and a state-dict write; it never consults `contract_class_changes` or the Patricia class tree to confirm the hash is real. [2](#0-1) 

For contrast, the class-declaration path in `execute_declare_transaction` does enforce `prev_value=0` (one-time declaration) and validates the class hash against `finalize_class_hash`: [3](#0-2) 

No equivalent guard exists in `execute_replace_class`.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once the OS commits a state update that sets a contract's `class_hash` to an undeclared felt, the contract is permanently broken:

- The global state root is updated to reflect the new (invalid) class hash via `compute_contract_state_commitment` → `get_contract_state_hash`.
- Every future call to the contract reaches the class-loading step and fails because no class data exists for that hash.
- There is no on-chain recovery path: `replace_class` can only be called by the contract itself, and the contract can no longer execute.
- All ERC-20 balances, ETH, NFTs, or any other assets held by the contract are permanently inaccessible. [4](#0-3) 

---

### Likelihood Explanation

**Medium.**

The attack surface is any contract that can be induced to call `replace_class` with an attacker-chosen hash. Two realistic paths exist:

1. **Honeypot contract**: An attacker deploys a contract that advertises a yield-bearing or escrow function. After users deposit funds, the attacker calls a function that internally invokes `replace_class(undeclared_hash)`. The OS accepts the syscall without validation, the state is committed, and the funds are frozen.

2. **Vulnerability in an existing contract**: If any deployed contract has a logic flaw that allows an external caller to influence the argument to `replace_class`, an attacker can supply an undeclared hash and freeze the contract.

The TODO comment strongly implies the blockifier also lacks this check (otherwise the OS TODO would be moot — the blockifier would have already rejected such transactions before they reached the OS). This means the sequencer would include the transaction, the OS would prove it, and the resulting state would be accepted by the network.

---

### Recommendation

Before writing the new `class_hash` into `contract_state_changes`, verify that the hash is declared. The check must cover both the current block's declarations and previously declared classes:

```cairo
// Option A: verify the hash exists in contract_class_changes (current block).
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);

// Option B (more complete): also accept hashes declared in prior blocks by
// reading the class leaf from the Patricia class tree and asserting it is non-zero.
```

Until this guard is in place, `replace_class` is an unconstrained state-mutation primitive that any contract can use to permanently destroy itself and freeze its funds.

---

### Proof of Concept

1. Attacker deploys contract `M` (class `C`) containing:
   ```
   fn freeze() {
       replace_class(0xdeadbeef);  // 0xdeadbeef is never declared
   }
   ```
2. Users deposit funds into `M` (e.g., via an ERC-20 `transfer` or ETH deposit).
3. Attacker calls `M.freeze()`.
4. The OS dispatches `execute_replace_class` with `class_hash = 0xdeadbeef`.
5. No existence check is performed (line 898 TODO). The state entry for `M` is updated: `class_hash ← 0xdeadbeef`.
6. `compute_contract_state_commitment` hashes the new state entry and updates the global root — the proof is valid.
7. All subsequent calls to `M` fail at class loading. All funds are permanently frozen. [5](#0-4) [6](#0-5)

func get_contract_state_hash{hash_ptr: HashBuiltin*}(
    class_hash: felt, storage_root: felt, nonce: felt
) -> (hash: felt) {
    const CONTRACT_STATE_HASH_VERSION = 0;
    if (class_hash == UNINITIALIZED_CLASS_HASH) {
        if (storage_root == 0) {
            if (nonce == 0) {
                return (hash=0);
            }
        }
    }

    // Set res = H(H(class_hash, storage_root), nonce).
    let (hash_value) = hash2(class_hash, storage_root);
    let (hash_value) = hash2(hash_value, nonce);

    // Return H(hash_value, CONTRACT_STATE_HASH_VERSION). CONTRACT_STATE_HASH_VERSION must be in the
    // outermost hash to guarantee unique "decoding".
    let (hash) = hash2(hash_value, CONTRACT_STATE_HASH_VERSION);
    return (hash=hash);
```
