### Title
Missing State Diff Commitment Verification in P2P Sync Allows Malicious Peer to Inject Wrong Storage Values into `apollo_storage` - (File: `crates/apollo_p2p_sync/src/client/state_diff.rs`)

---

### Summary

`parse_data_for_block` in the P2P sync client assembles `StateDiffChunk` messages from peers into a `ThinStateDiff` and validates only the total entry count against `header.state_diff_length`. It never verifies that `calculate_state_diff_hash(&assembled_diff) == header.state_diff_commitment`. A malicious peer can send chunks whose entry counts sum to the correct `state_diff_length` but whose storage values, class hashes, or nonces differ from the canonical state diff. The wrong `ThinStateDiff` is written to `apollo_storage` before the committer's downstream hash check runs, creating a window in which RPC returns authoritative-looking wrong state values and leaving the storage layer in an inconsistent state if the committer subsequently rejects the diff.

---

### Finding Description

`parse_data_for_block` reads `target_state_diff_len` from the already-stored block header's `state_diff_length` field and loops over incoming `StateDiffChunk` messages, accumulating `current_state_diff_len += state_diff_chunk.len()` until the target is reached. [1](#0-0) 

`StateDiffChunk::len()` counts individual entries (storage key-value pairs, deployed-contract entries, nonce entries, declared-class entries) in exactly the same way as `ThinStateDiff::len()`. [2](#0-1) 

After the loop the function calls `validate_deprecated_declared_classes_non_conflicting` and returns `Ok(Some((result, block_number)))`. There is **no call to `calculate_state_diff_hash`** and no comparison against `header.state_diff_commitment`. [3](#0-2) 

The assembled `ThinStateDiff` is immediately written to `apollo_storage`: [4](#0-3) 

The block header already contains the correct `state_diff_commitment` (a Poseidon hash over the canonical state diff content): [5](#0-4) 

`state_diff_commitment` is also embedded in `concatenated_counts` and therefore in the block hash and partial block hash components: [6](#0-5) 

A malicious peer can craft chunks that:
1. Contain the correct number of entries per type (passes the length check).
2. Contain no duplicate keys (passes `unite_state_diffs` conflict checks).
3. Carry **different storage values, class hashes, or nonces** than the canonical diff.

The assembled wrong `ThinStateDiff` is stored in `apollo_storage` and is immediately visible to RPC queries (`starknet_getStateUpdate`, `starknet_getStorageAt`, etc.) before the committer ever sees it.

The committer does perform a hash check when `verify_state_diff_hash: true` (the default): [7](#0-6) [8](#0-7) 

However, this check is:
- **Downstream and asynchronous** — the wrong diff is already in `apollo_storage` when the committer runs.
- **Conditional** — it is skipped when `state_diff_commitment` is `None` (old blocks) or when the flag is disabled.
- **Not a rollback** — if the committer returns `StateDiffHashMismatch`, the wrong diff remains in `apollo_storage`; no automatic cleanup is performed. [9](#0-8) 

---

### Impact Explanation

**High — RPC returns authoritative-looking wrong values.**

During the window between `append_state_diff` and the committer's verification, any RPC call that reads from `apollo_storage` (storage-at, nonce-at, class-hash-at, get-state-update) returns the attacker-injected values. If `verify_state_diff_hash` is disabled or the commitment is absent, the wrong diff is committed to the Patricia trie, producing a wrong `GlobalRoot`, wrong block hash, and wrong proof inputs — escalating to Critical.

---

### Likelihood Explanation

Any node that connects to the victim as a P2P peer for state diff sync can trigger this. No privileged access is required. The attacker only needs to serve state diff chunks; the block header (which carries the correct `state_diff_commitment`) is synced independently and is not under the attacker's control.

---

### Recommendation

After assembling the full `ThinStateDiff` and before returning it for storage, verify the Poseidon commitment:

```rust
// After the while loop and validate_deprecated_declared_classes_non_conflicting:
let header = storage_reader.begin_ro_txn()?.get_block_header(block_number)? ...;
if let Some(expected_commitment) = header.state_diff_commitment {
    let actual_commitment = calculate_state_diff_hash(&result);
    if actual_commitment != expected_commitment {
        return Err(ParseDataError::BadPeer(BadPeerError::InvalidStateDiffCommitment));
    }
}
```

This mirrors the invariant already enforced by the committer but moves it to the point of ingress, preventing wrong data from ever entering `apollo_storage`. [10](#0-9) 

---

### Proof of Concept

1. Victim node starts P2P state diff sync for block N. The block header (with correct `state_diff_commitment = H` and `state_diff_length = L`) is already stored in `apollo_storage`.
2. Attacker's peer receives the `StateDiffQuery` for block N.
3. Attacker sends `StateDiffChunk::ContractDiff` messages whose `len()` values sum to `L` but whose storage values differ from the canonical diff (e.g., replace `value_canonical` with `value_attacker` for key K of contract C).
4. `parse_data_for_block` accepts the chunks: `current_state_diff_len == target_state_diff_len` ✓, no conflicting parts ✓, no duplicate deprecated classes ✓.
5. `write_to_storage` calls `append_state_diff(block_number, tampered_diff)` — the wrong diff is now in `apollo_storage`.
6. Any `starknet_getStorageAt(C, K, block_N)` RPC call now returns `value_attacker` instead of `value_canonical`.
7. The committer later calls `calculate_state_diff_hash(tampered_diff)` and finds it ≠ `H`, returning `StateDiffHashMismatch` — but the wrong diff remains in `apollo_storage` and no rollback occurs. [11](#0-10) [7](#0-6)

    #[latency_histogram("p2p_sync_state_diff_parse_data_for_block_latency_seconds", true)]
    fn parse_data_for_block<'a>(
        state_diff_chunks_response_manager: &'a mut ClientResponsesManager<
            DataOrFin<StateDiffChunk>,
        >,
        block_number: BlockNumber,
        storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            let mut result = ThinStateDiff::default();
            let mut prev_result_len = 0;
            let mut current_state_diff_len = 0;
            let target_state_diff_len = storage_reader
                .begin_ro_txn()?
                .get_block_header(block_number)?
                .expect("A header with number lower than the header marker is missing")
                .state_diff_length
                .ok_or(P2pSyncClientError::OldHeaderInStorage {
                    block_number,
                    missing_field: "state_diff_length",
                })?;

            while current_state_diff_len < target_state_diff_len {
                let maybe_state_diff_chunk = state_diff_chunks_response_manager
                    .next()
                    .await
                    .ok_or(ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }))?;
                let Some(state_diff_chunk) = maybe_state_diff_chunk?.0 else {
                    if current_state_diff_len == 0 {
                        return Ok(None);
                    } else {
                        return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                            expected_length: target_state_diff_len,
                            possible_lengths: vec![current_state_diff_len],
                        }));
                    }
                };
                prev_result_len = current_state_diff_len;
                if state_diff_chunk.is_empty() {
                    return Err(ParseDataError::BadPeer(BadPeerError::EmptyStateDiffPart));
                }
                // It's cheaper to calculate the length of `state_diff_part` than the length of
                // `result`.
                current_state_diff_len += state_diff_chunk.len();
                unite_state_diffs(&mut result, state_diff_chunk)?;
            }

            if current_state_diff_len != target_state_diff_len {
                return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffLength {
                    expected_length: target_state_diff_len,
                    possible_lengths: vec![prev_result_len, current_state_diff_len],
                }));
            }

            validate_deprecated_declared_classes_non_conflicting(&result)?;
            Ok(Some((result, block_number)))
        }
        .boxed()
    }

impl StateDiffChunk {
    pub fn len(&self) -> usize {
        match self {
            StateDiffChunk::ContractDiff(contract_diff) => {
                let mut result = contract_diff.storage_diffs.len();
                if contract_diff.class_hash.is_some() {
                    result += 1;
                }
                if contract_diff.nonce.is_some() {
                    result += 1;
                }
                result
            }
            StateDiffChunk::DeclaredClass(_) => 1,
            StateDiffChunk::DeprecatedDeclaredClass(_) => 1,
        }
    }

    pub fn is_empty(&self) -> bool {
        self.len() == 0
    }

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
