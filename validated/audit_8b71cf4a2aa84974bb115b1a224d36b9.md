### Title
Missing Declared Class Hash Validation in `execute_replace_class` Enables Permanent Contract Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the replacement class hash corresponds to a declared contract class. Any contract can replace its own class with an arbitrary, undeclared class hash. Once committed to the chain, the contract becomes permanently inoperable and all funds it holds are permanently frozen, with no recovery path.

---

### Finding Description

In `execute_replace_class`, the function reads the new class hash from the syscall request and directly updates the contract's state entry without checking whether the class hash is actually declared in the class registry. The TODO comment at line 898 of `syscall_impls.cairo` explicitly acknowledges this missing check:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The OS accepts any felt value as the new `class_hash` in the `replace_class` request without verifying it against the declared class registry (`contract_class_changes`). Once the block is finalized and the state committed, the contract's class hash is permanently set to the undeclared value. Any subsequent call to the contract will fail at the class lookup stage because the OS cannot find the corresponding compiled class. The contract becomes permanently inoperable.

The syscall dispatch in `execute_syscalls.cairo` routes `REPLACE_CLASS_SELECTOR` directly to this function, passing the current contract's address:

```cairo
if (selector == REPLACE_CLASS_SELECTOR) {
    execute_replace_class(contract_address=execution_context.execution_info.contract_address);
``` [2](#0-1) 

This is the direct analog of the external report's vulnerability: just as `PublicLock.initialize()` lacked access control allowing anyone to call it and corrupt the initialization state, `execute_replace_class` lacks input validation allowing any class hash — including undeclared ones — to be permanently written into a contract's state entry, with irreversible consequences.

---

### Impact Explanation

**Critical. Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value, the contract is permanently frozen. All funds held by the contract — tokens, ETH bridged via L1→L2 messages, user deposits in a DeFi protocol — become permanently inaccessible. The state change is committed to the chain via `state_update` in `state.cairo`: [3](#0-2) 

The OS validates this as a legitimate state transition, so the proof is accepted by L1. There is no recovery mechanism at the protocol level.

---

### Likelihood Explanation

The `replace_class` syscall is callable by any executing contract. An attacker who controls a contract holding user funds, or who can exploit a reentrancy or logic vulnerability in a shared contract (e.g., a multi-sig wallet, a DEX, or a lending protocol) to trigger `replace_class` with an undeclared hash, can permanently freeze all funds in that contract. The missing check is explicitly documented as a TODO in the source code, confirming it is a known gap in the OS's validation logic that has not yet been addressed.

---

### Recommendation

Before updating the contract's state entry in `execute_replace_class`, add a check that the new class hash exists in the `contract_class_changes` dictionary (i.e., it has been declared in the current or a previous block). If the class hash is not declared, the syscall should write a failure response instead of updating the state. This mirrors the existing `prev_value=0` guard used in `execute_declare_transaction` to enforce single-declaration: [4](#0-3) 

---

### Proof of Concept

1. A contract `C` holding user funds calls the `replace_class` syscall with `class_hash = 0xdeadbeef` (an undeclared hash).
2. The OS routes the call to `execute_replace_class` via `execute_syscalls`.
3. No check is performed to verify that `0xdeadbeef` is a declared class hash — the TODO at line 898 of `syscall_impls.cairo` confirms this gap.
4. The contract's state entry is updated: `C.class_hash = 0xdeadbeef`.
5. The block is finalized; `state_update` commits the new state root to the chain.
6. Any subsequent call to contract `C` fails because `0xdeadbeef` is not present in the compiled class facts bundle.
7. All funds in contract `C` are permanently frozen with no recovery path. [5](#0-4)

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
