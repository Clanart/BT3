### Title
Missing CASM Integrity Verification Against Committed `compiled_class_hash` in Central Sync — (`crates/apollo_central_sync/src/lib.rs`)

### Summary

`store_compiled_class` in the central sync path receives the committed `compiled_class_hash` from the state diff but discards it (parameter is named `_compiled_class_hash_v1` with a leading underscore). The `CasmContractClass` bytes fetched from the central source are written directly to storage via `append_casm` with no check that `hash(received_casm) == committed_compiled_class_hash`. An explicit developer TODO acknowledges the missing check. Once the wrong CASM is stored it is permanent (the storage layer rejects re-writes with `KeyAlreadyExists`), so every subsequent execution of any contract deployed from that class hash runs the wrong bytecode, producing wrong state, wrong receipts, and a wrong block commitment.

### Finding Description

The parameter `_compiled_class_hash_v1` carries the `CompiledClassHash` that was committed to the state diff (and therefore to the block hash, which is L1-verified). The function computes `compiled_class_hash_v2` from the bytes it just received, but never asserts:

```
compiled_class.hash(&HashVersion::V2) == committed_compiled_class_hash
```

The `append_casm` writer stores whatever bytes it receives, keyed only by `class_hash`:

```rust

``` [2](#0-1) 

Once written, the storage layer enforces uniqueness — a second write for the same `class_hash` returns `KeyAlreadyExists`: [3](#0-2) 

The `CommitmentStateDiff` that feeds `calculate_block_commitments` and ultimately the block hash carries `class_hash_to_compiled_class_hash` as the authoritative binding between a Sierra class hash and its CASM hash: [4](#0-3) 

`calculate_state_diff_hash` hashes this binding into the `state_diff_commitment` field of the block header: [5](#0-4) 

Because the CASM stored in the `casms` table is never checked against the `compiled_class_hash` committed in the state diff, the two can silently diverge.

When the blockifier later executes a transaction that calls a contract of this class, it fetches the CASM from storage via `get_casm_unchecked`: [6](#0-5) 

and runs whatever bytecode is there, with no re-verification against the committed hash.

The `add_class_and_executable_unsafe` path (called when `!is_compiler_backward_compatible`) also propagates the unverified CASM into the class manager under the name "unsafe": [7](#0-6) [8](#0-7) 

### Impact Explanation

A syncing node that receives a `CasmContractClass` whose bytes do not match the `compiled_class_hash` committed in the state diff will:

1. Store the wrong CASM permanently (write is irreversible due to `KeyAlreadyExists` guard).
2. Execute every future transaction that deploys or calls a contract of that class using the wrong bytecode.
3. Produce wrong storage updates, events, receipts, and return values — all committed into the block hash via `BlockExecutionArtifacts::new` → `calculate_block_commitments`.
4. Generate a block commitment that diverges from what the prover expects, breaking proof generation for that block.

This matches the allowed impact: **Critical — Wrong compiled class / CASM artifact selected for execution**, and **Critical — Wrong state, receipt, event, or revert result from blockifier/execution logic**.

### Likelihood Explanation

The central source fetches the CASM over the network from the Starknet feeder gateway. Any of the following can deliver a mismatched CASM without the sequencer detecting it:

- A network-level MITM between the syncing node and the gateway.
- A bug or regression in the gateway's CASM serialization.
- A compromised or malicious feeder gateway endpoint.

The sequencer already holds the ground truth — the `compiled_class_hash` from the L1-verified state diff — but throws it away. The developer TODO at line 774 confirms the team is aware the check is absent.

### Recommendation

Inside `store_compiled_class`, after computing `compiled_class_hash_v2`, assert it equals the committed hash before writing:

```rust
async fn store_compiled_class(
    &mut self,
    block_number: BlockNumber,
    class_hash: ClassHash,
    committed_compiled_class_hash: CompiledClassHash,  // rename: no longer ignored
    compiled_class: CasmContractClass,
    is_compiler_backward_compatible: bool,
) -> StateSyncResult {
    let compiled_class_hash_v2 = compiled_class.hash(&HashVersion::V2);
    // Also check V1 for pre-migration blocks where the state diff still carries V1.
    let compiled_class_hash_v1 = compiled_class.hash(&HashVersion::V1);
    if compiled_class_hash_v2 != committed_compiled_class_hash
        && compiled_class_hash_v1 != committed_compiled_class_hash
    {
        return Err(StateSyncError::CompiledClassHashMismatch {
            class_hash,
            committed: committed_compiled_class_hash,
            received_v2: compiled_class_hash_v2,
        });
    }
    // ... rest of function
```

The same check should be applied before `add_class_and_executable_unsafe` to prevent the class manager from caching a mismatched artifact.

### Proof of Concept

1. A syncing node calls `store_compiled_class` for `class_hash = H`, `committed_compiled_class_hash = C` (from the L1-verified state diff), and `compiled_class = CASM_WRONG` where `CASM_WRONG.hash(V2) ≠ C`.
2. `append_casm(&H, &CASM_WRONG)` succeeds — no hash check is performed.
3. A subsequent transaction deploys a contract at address `A` with class hash `H`.
4. An invoke transaction calls `A::some_function(...)`.
5. The blockifier calls `get_casm(&H)` → returns `CASM_WRONG`.
6. `CASM_WRONG` is executed; its bytecode produces storage writes `S_wrong`, events `E_wrong`, and return value `R_wrong`.
7. `CommitmentStateDiff` records `S_wrong`; `calculate_block_commitments` hashes it into `state_diff_commitment`.
8. The block hash commits to `S_wrong` — diverging from what any honest node or the prover would compute from the correct CASM `C`.
9. The SNOS proof for this block fails or attests to the wrong state root, breaking the L1 state commitment.

    fn append_casm(self, class_hash: &ClassHash, casm: &CasmContractClass) -> StorageResult<Self> {
        let casm_table = self.open_table(&self.tables().casms)?;
        let markers_table = self.open_table(&self.tables().markers)?;
        let state_diff_table = self.open_table(&self.tables().state_diffs)?;
        let file_offset_table = self.txn().open_table(&self.tables().file_offsets)?;

        let location = self.file_handlers().append_casm(casm);
        casm_table.insert(self.txn(), class_hash, &location)?;
        file_offset_table.upsert(self.txn(), &OffsetKind::Casm, &location.next_offset())?;
        update_marker(
            self.txn(),
            &markers_table,
            &state_diff_table,
            self.file_handlers(),
            class_hash,
        )?;
        Ok(self)
    }
