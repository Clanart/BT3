### Title
Wrong `state_diff_commitment` and Block Hash Due to `deprecated_declared_classes` Silently Dropped in `CommitmentStateDiff` → `ThinStateDiff` Conversion — (`File: crates/blockifier/src/state/cached_state.rs`)

---

### Summary

The sequencer's block-building path converts `CommitmentStateDiff` (the blockifier's execution output) to `ThinStateDiff` for commitment calculation. `CommitmentStateDiff` has no `deprecated_declared_classes` field, so the conversion always hardcodes `deprecated_declared_classes: Vec::new()`. Any deprecated (Cairo 0) class declarations included in a block are silently dropped from the state diff used to compute `state_diff_commitment`, `state_diff_length`, `concatenated_counts`, and the final block hash. This is the direct sequencer analog of the USDT `approve()` incompatibility: a sub-standard interface (`CommitmentStateDiff`) that omits a required field, causing the downstream consumer (`calculate_block_commitments`) to silently operate on incomplete data and produce a wrong authoritative commitment.

---

### Finding Description

**Root cause — the missing field:**

`CommitmentStateDiff` in `crates/blockifier/src/state/cached_state.rs` has four fields and no `deprecated_declared_classes`: [1](#0-0) 

The `From<CommitmentStateDiff> for ThinStateDiff` conversion hardcodes the missing field to an empty `Vec`: [2](#0-1) 

The `TODO` comment acknowledges this is a known structural gap, not an intentional design choice.

**Propagation through the commitment pipeline:**

`BlockExecutionArtifacts::new()` calls `calculate_block_commitments()` with `ThinStateDiff::from(commitment_state_diff.clone())` — the defective conversion: [3](#0-2) 

`calculate_block_commitments()` passes the truncated `ThinStateDiff` to both `calculate_state_diff_hash()` and `concat_counts()`: [4](#0-3) 

`ThinStateDiff::len()` counts `deprecated_declared_classes.len()` as part of `state_diff_length`: [5](#0-4) 

`calculate_state_diff_hash()` chains `deprecated_declared_classes` into the Poseidon hash: [6](#0-5) 

`concat_counts()` packs the wrong `state_diff_length` into the `concatenated_counts` felt: [7](#0-6) 

`calculate_block_hash()` chains both the wrong `state_diff_commitment` and the wrong `concatenated_counts` into the final block hash: [8](#0-7) 

**Downstream storage and sync impact:**

The wrong `PartialBlockHashComponents` (containing wrong `state_diff_commitment` and `concatenated_counts`) is stored and later used by `finalize_commitment_output()` to compute the final block hash: [9](#0-8) 

The wrong `state_diff_length` is also stored in `StorageBlockHeader`: [10](#0-9) 

The P2P sync client uses `state_diff_length` from the stored header as the authoritative termination condition for state diff chunk collection: [11](#0-10) 

---

### Impact Explanation

For any block containing a deprecated (Cairo 0) class declaration (DeclareV1 transaction):

1. **Wrong `state_diff_commitment`** — the Poseidon hash over the state diff omits the deprecated class hashes, producing a commitment that does not match the actual state transition.
2. **Wrong `state_diff_length`** — undercounted by the number of deprecated declared classes; this wrong value is packed into `concatenated_counts`.
3. **Wrong `concatenated_counts`** — the packed felt encoding transaction count, event count, and state diff length is wrong.
4. **Wrong block hash** — `calculate_block_hash` chains both wrong values; the sequencer produces and stores a block hash that diverges from the Starknet specification and from what the SNOS/prover would independently compute.
5. **Wrong `state_diff_length` in stored header** — the P2P sync termination guard uses this value, so syncing nodes would accept a truncated state diff as complete.

This matches the impact scope: **Critical — Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input** and **Critical — Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact** (wrong block hash breaks L1 settlement).

---

### Likelihood Explanation

Any user can submit a DeclareV1 (deprecated class declaration) transaction. The gateway does not explicitly reject DeclareV1 transactions; the blockifier handles them. If such a transaction is included in a block, the defect is triggered deterministically. The trigger is unprivileged and requires no special access.

---

### Recommendation

Add `deprecated_declared_classes` to `CommitmentStateDiff` in `crates/blockifier/src/state/cached_state.rs` and populate it from `StateMaps` (which must also be extended to track deprecated class declarations). Remove the hardcoded `Vec::new()` in `From<CommitmentStateDiff> for ThinStateDiff`. This mirrors the fix in the external report: replace the sub-standard interface with one that returns the full required data, analogous to replacing `approve()` with `safeApprove()`/`forceApprove()`.

---

### Proof of Concept

1. Submit a DeclareV1 transaction declaring a Cairo 0 class to the sequencer gateway.
2. The transaction is validated and included in a block by the batcher.
3. `BlockExecutionArtifacts::new()` calls `ThinStateDiff::from(commitment_state_diff.clone())` — `deprecated_declared_classes` is `[]`.
4. `calculate_block_commitments()` computes `state_diff_commitment` = `Poseidon(STARKNET_STATE_DIFF0, ..., chain_deprecated_declared_classes([]), ...)` — missing the declared class hash.
5. `ThinStateDiff::len()` returns N (correct count minus the deprecated class count); `concat_counts()` packs this wrong N into `concatenated_counts`.
6. `calculate_block_hash()` produces a block hash that differs from the hash the SNOS would compute over the same block (since SNOS uses the full state diff including deprecated class declarations).
7. The wrong block hash is stored in `StorageBlockHeader` and propagated to L1 settlement, causing proof verification failure.

pub fn concat_counts(
    transaction_count: usize,
    event_count: usize,
    state_diff_length: usize,
    l1_data_availability_mode: L1DataAvailabilityMode,
) -> Felt {
    let l1_data_availability_byte: u8 = match l1_data_availability_mode {
        L1DataAvailabilityMode::Calldata => 0,
        L1DataAvailabilityMode::Blob => 0b10000000,
    };
    let concat_bytes = [
        to_64_bits(transaction_count).as_slice(),
        to_64_bits(event_count).as_slice(),
        to_64_bits(state_diff_length).as_slice(),
        &[l1_data_availability_byte],
        &[0_u8; 7], // zero padding
    ]
    .concat();
    Felt::from_bytes_be_slice(concat_bytes.as_slice())
}

    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);

    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);

    fn finalize_commitment_output<R: BatcherStorageReader + ?Sized>(
        storage_reader: Arc<R>,
        CommitmentTaskOutput { response: CommitBlockResponse { global_root }, height }: CommitmentTaskOutput,
        should_finalize_block_hash: bool,
    ) -> CommitmentManagerResult<FinalBlockCommitment> {
        match should_finalize_block_hash {
            false => {
                debug!("Finalized commitment for block {height} without calculating block hash.");
                Ok(FinalBlockCommitment { height, block_hash: None, global_root })
            }
            true => {
                debug!("Finalizing commitment for block {height} with calculating block hash.");
                let (previous_block_hash, partial_block_hash_components) =
                    storage_reader.get_parent_hash_and_partial_block_hash_components(height)?;
                let previous_block_hash = previous_block_hash.ok_or_else(|| {
                    CommitmentManagerError::MissingBlockHash(height.prev().expect(
                        "For the genesis block, the block hash is constant and should not be \
                         fetched from storage.",
                    ))
                })?;
                let partial_block_hash_components = partial_block_hash_components
                    .ok_or(CommitmentManagerError::MissingPartialBlockHashComponents(height))?;
                debug!(
                    "Calculating block hash for block {height} with partial block hash \
                     components: {partial_block_hash_components:?}"
                );
                debug!(
                    "Global root: {global_root:?}, previous block hash: {previous_block_hash:?}"
                );
                let block_hash = calculate_block_hash(
                    &partial_block_hash_components,
                    global_root,
                    previous_block_hash,
                )?;
                Ok(FinalBlockCommitment { height, block_hash: Some(block_hash), global_root })
            }
        }
