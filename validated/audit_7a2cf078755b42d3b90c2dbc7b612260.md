### Title
Self-Destruct + Same-Transaction CREATE2 Re-Deploy Leaves Stale Storage in Committed KV Store - (`x/evm/statedb/statedb.go`)

### Summary

`StateDB.Commit()` determines whether to call `DeleteAccount` (which clears contract storage) by checking `obj.selfDestructed` on the object currently held in `s.stateObjects[addr]`. When a contract self-destructs and is re-created at the same address within the same transaction (via CREATE2), `createObject()` replaces the self-destructed object in `stateObjects` with a fresh one. `Commit()` then sees the new, non-self-destructed object and skips `DeleteAccount` entirely. The old storage slots are never cleared from the underlying KV store, and the re-deployed contract silently inherits them.

---

### Finding Description

**Root cause — `createObject` overwrites the self-destructed entry in `stateObjects`:** [1](#0-0) 

When `createObject` is called for an address that already has a live (self-destructed) object, it appends a `resetObjectChange{prev: prev}` to the journal and then calls `s.setStateObject(newobj)`, replacing the old entry in `s.stateObjects`. Crucially, `resetObjectChange.Dirtied()` returns `nil`: [2](#0-1) 

So the address remains dirty (from the earlier `selfDestructChange`), but `stateObjects[addr]` now points to the new, non-self-destructed object.

**`Commit()` dispatches on the current `stateObjects` entry:** [3](#0-2) 

The loop fetches `obj = s.stateObjects[addr]`. Because that is now the re-created object (`obj.selfDestructed == false`), the `else` branch runs: it writes the new object's dirty code/nonce/storage, but **never calls `DeleteAccount`**. The old storage slots written by the original contract remain verbatim in the KV store.

**`DeleteAccount` is the only path that clears storage:** [4](#0-3) 

`DeleteAccount` iterates `ForEachStorage` and calls `SetState(..., nil)` for every slot. Since this is never invoked for the self-destructed-then-re-created address, all prior storage persists.

**The new contract reads stale storage via `GetCommittedState`:** [5](#0-4) 

The re-created object starts with an empty `originStorage` cache. Any `SLOAD` for a slot not yet written by the new contract falls through to `keeper.GetState`, which reads directly from the KV store — returning the old, non-zero value.

**go-ethereum avoids this with a separate `stateObjectsDestruct` map** that records every object that was self-destructed, independently of whether it was later re-created. Ethermint's `StateDB` struct has no such map: [6](#0-5) 

---

### Impact Explanation

A re-deployed contract at the same CREATE2 address inherits arbitrary non-zero storage from its predecessor. If the re-deployed contract is a financial primitive (token, vault, escrow) that assumes clean storage on initialization — e.g., uses a storage slot as an "initialized" flag, an owner address, or a balance mapping — the stale values bypass those checks. This enables unauthorized state mutation and potential theft or mis-accounting of EVM-denom funds, matching the **High** impact category: *EVM state transition bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted.*

---

### Likelihood Explanation

The attack requires only an unprivileged EVM transaction. CREATE2 re-deployment after self-destruct is a well-known and documented Ethereum pattern. An attacker who controls the deployed bytecode (or can predict the CREATE2 address of a victim contract) can craft a single transaction that: (1) calls the existing contract to trigger `SELFDESTRUCT`, and (2) immediately re-deploys at the same address. No privileged role, governance action, or validator cooperation is needed.

---

### Recommendation

Track self-destructed objects in a separate map (analogous to go-ethereum's `stateObjectsDestruct`) so that `Commit()` can call `DeleteAccount` for every address that was self-destructed during the transaction, regardless of whether it was subsequently re-created. Concretely:

1. Add `stateObjectsDestruct map[common.Address]*stateObject` to `StateDB`.
2. In `SelfDestruct` (or `markSelfDestructed`), record the object in that map.
3. In `Commit()`, iterate `stateObjectsDestruct` first and call `DeleteAccount` for each entry before processing the re-created objects.

---

### Proof of Concept

```
Block N, single EVM transaction:
  1. Attacker calls FactoryContract.deployAndDestroy(salt, initcode)
     - Factory CREATE2-deploys VictimContract at address X
       VictimContract constructor writes: storage[0] = 0xDEAD
     - Factory immediately calls VictimContract.destroy()
       VictimContract executes SELFDESTRUCT → stateObject[X].selfDestructed = true
     - Factory CREATE2-deploys NewContract at address X (same salt, same initcode)
       createObject(X) replaces stateObjects[X] with fresh object (selfDestructed=false)

  2. stateDB.Commit() runs:
     - sortedDirties() includes X (dirtied by selfDestructChange)
     - stateObjects[X] = NewContract object (selfDestructed=false)
     - else-branch: writes NewContract's dirty storage only
     - DeleteAccount(X) is NEVER called
     - KV store still contains: storage[X][0] = 0xDEAD

  3. Next transaction calls NewContract.readSlot(0):
     - GetCommittedState(0) → originStorage miss → keeper.GetState(X, 0)
     - Returns 0xDEAD  ← stale value from destroyed predecessor
```

func (s *StateDB) createObject(addr common.Address) *stateObject {
	prev := s.getStateObject(addr)

	newobj := newObject(s, addr, nil)
	if prev == nil {
		s.journal.append(createObjectChange{account: &addr})
	} else {
		s.journal.append(resetObjectChange{prev: prev})
	}
	s.setStateObject(newobj)
	return newobj
}

		obj, exist := s.stateObjects[addr]
		if !exist {

		}
		if obj.selfDestructed {
			// Burn any balance that arrived after SelfDestruct was called (e.g., via a

			// SelfDestruct already burned the balance present at destruction time, but
			// subsequent AddBalance calls write to the bank without a matching burn.
			// DeleteAccount only removes auth metadata and storage; it never touches the
			// bank balance, so we must drain it here before removing the account.
			//
			// Both operations run inside a single CacheContext so that if DeleteAccount
			// fails after SubBalance, the partial burn is rolled back and the bank is
			// left consistent.
			cosmosAddr := sdk.AccAddress(obj.Address().Bytes())
			cacheCtx, writeCache := s.origCtx.CacheContext()
			// Only the EVM denom is burned here. Non-EVM-native tokens (IBC, CosmWasm
			// bridge) held by the destroyed address are not drained and may remain as
			// orphaned bank balances.
			if remaining := s.keeper.GetBalance(cacheCtx, cosmosAddr, s.evmDenom); remaining.Sign() > 0 {
				coin := sdk.NewCoin(s.evmDenom, sdkmath.NewIntFromBigInt(remaining.ToBig()))
				if _, err := s.keeper.SubBalance(cacheCtx, cosmosAddr, coin); err != nil {
					return errorsmod.Wrap(err, "failed to burn post-selfdestruct balance")
				}
			}
			if err := s.keeper.DeleteAccount(cacheCtx, obj.Address()); err != nil {
				return errorsmod.Wrap(err, "failed to delete account")
			}
			writeCache()
		} else {
			codeDirty := obj.codeDirty()
			if codeDirty && obj.code != nil {
				s.keeper.SetCode(s.origCtx, obj.CodeHash(), obj.code)
			}
			if codeDirty || obj.nonceDirty() {

					return errorsmod.Wrap(err, "failed to set account")
				}

				if value == obj.originStorage[key] {
					continue
				}

func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
	cosmosAddr := sdk.AccAddress(addr.Bytes())
	acct := k.accountKeeper.GetAccount(ctx, cosmosAddr)
	if acct == nil {
		return nil
	}

	// NOTE: only Ethereum accounts (contracts) can be selfdestructed
	_, ok := acct.(ethermint.EthAccountI)
	if !ok {
		return errorsmod.Wrapf(types.ErrInvalidAccount, "type %T, address %s", acct, addr)
	}

	// clear storage
	k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool {
		k.SetState(ctx, addr, key, nil)
		return true
	})

	// remove auth account
	k.accountKeeper.RemoveAccount(ctx, acct)

	k.debugLog(ctx, "account suicided",
		"ethereum-address", addr,
		"cosmos-address", cosmosAddr,
	)

	return nil

func (s *stateObject) GetCommittedState(key common.Hash) common.Hash {
	if s.overrideStorage != nil {
		if value, ok := s.overrideStorage[key]; ok {
			return value
		}
		return common.Hash{}
	}

	if value, cached := s.originStorage[key]; cached {
		return value
	}
	// If no live objects are available, load it from keeper
	value := s.db.keeper.GetState(s.db.ctx, s.Address(), key)
	s.originStorage[key] = value
	return value
}
