### Title
`execute_replace_class` Accepts Undeclared Class Hash Without Validation — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS directly stores a caller-supplied `class_hash` into the contract state without verifying that the hash corresponds to a previously declared class. An explicit `TODO` comment in the code acknowledges this missing check. A malicious or buggy contract can replace its own class with an arbitrary, undeclared hash, permanently breaking all future calls to that contract and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class` (both the Cairo 1 path in `syscall_impls.cairo` and the deprecated path in `deprecated_execute_syscalls.cairo`), the OS reads `class_hash` directly from the syscall request and writes it into `contract_state_changes` with no validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The same pattern exists in the deprecated syscall handler: [2](#0-1) 

The OS never checks whether `class_hash` is present in `contract_class_changes` (the dict mapping declared class hashes to their compiled class hashes). Any felt value is accepted and committed to state.

When a subsequent transaction calls the affected contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
``` [3](#0-2) 

If `class_hash` was never declared, `dict_read` returns 0. `find_element` then searches for a compiled class with hash 0, which does not exist, causing the call to trap. The contract becomes permanently uncallable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with an undeclared value and the block is proven and settled on L1, the state is canonical and irreversible. Every future call to that contract will fail at the OS level because no compiled class can be found for the stored hash. Any ERC-20 balances, NFTs, or other assets held by the contract are permanently inaccessible.

---

### Likelihood Explanation

**High.** The attacker-controlled entry path is a single syscall available to any Cairo 1 or Cairo 0 contract. No privileged role is required. The missing check is explicitly acknowledged by a `TODO` comment in the production code, confirming the gap is real and not an artifact of test scaffolding. Any contract that calls `replace_class` with a felt value that is not a declared class hash — whether maliciously or accidentally — triggers the freeze.

---

### Recommendation

Before writing the new class hash to `contract_state_changes`, verify that it exists as a key in `contract_class_changes` (i.e., it has been declared in the current block or in a prior block). Concretely, perform a `dict_read` on `contract_class_changes` with the supplied `class_hash` and assert the returned `compiled_class_hash` is non-zero before proceeding with the state update.

---

### Proof of Concept

1. Deploy a contract `Victim` holding user funds.
2. From `Victim` (or any contract that can call it), invoke the `replace_class` syscall with `class_hash = 0xdeadbeef` (any felt not corresponding to a declared class).
3. The OS processes `execute_replace_class`, skips the missing validation, and writes `class_hash = 0xdeadbeef` into `contract_state_changes` for `Victim`'s address.
4. The block is proven; L1 accepts the proof and the state transition is finalized.
5. Any subsequent transaction calling `Victim` reaches `execute_entry_point`, which does `dict_read(contract_class_changes, 0xdeadbeef)` → returns 0, then `find_element(..., key=0)` → not found → execution fails.
6. All funds in `Victim` are permanently frozen with no recovery path.
