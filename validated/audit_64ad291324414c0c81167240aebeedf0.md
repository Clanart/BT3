### Title
Unvalidated `l1_gas_price_wei` / `l1_data_gas_price_wei` in Proposal Validation Corrupts Block Hash and ETH Fee Calculations - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_block_info_valid` validates the proposer's FRI gas prices against the local oracle but **silently discards** the locally-computed wei prices and accepts the proposer's `l1_gas_price_wei` / `l1_data_gas_price_wei` without any oracle comparison. Those unvalidated wei values flow directly into ETH-denominated fee execution, the block hash, and the fallback used to derive the next block's conversion rate.

---

### Finding Description

In `is_block_info_valid`, the locally-computed wei prices are explicitly thrown away:

```rust
let (l1_gas_prices_fri, _l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(...)
```

Only the FRI prices are then checked with `within_margin`. The proposer's `init.l1_gas_price_wei` and `init.l1_data_gas_price_wei` are never compared to anything. [1](#0-0) 

Those unvalidated wei values then propagate through three distinct paths:

**Path 1 – ETH fee execution.** `convert_to_sn_api_block_info` maps `init.l1_gas_price_wei` directly to `eth_gas_prices.l1_gas_price` in the `BlockInfo` handed to the batcher. Every ETH-denominated fee check in the blockifier uses this value. [2](#0-1) 

**Path 2 – Block hash.** `PartialBlockHashComponents::new` reads `block_info.gas_prices.l1_gas_price_per_token()`, which returns a `GasPricePerToken` containing both `price_in_wei` and `price_in_fri`. For Starknet ≥ 0.13.4 (current version is 0.14.2), `gas_prices_to_hash` hashes all six wei+fri fields into the block hash. [3](#0-2) [4](#0-3) 

**Path 3 – Fallback corruption.** After `decision_reached`, `PreviousBlockInfo::from(init)` stores the proposer's unvalidated wei prices. When the oracle fails for the next block, `calculate_eth_to_fri_rate` derives the conversion rate as `l1_prices_fri / l1_prices_wei` from this stored data. A manipulated wei value produces a wrong rate, which then drives wrong FRI prices for the next block. [5](#0-4) [6](#0-5) 

---

### Impact Explanation

A malicious proposer sets `l1_gas_price_fri` within the 10 % margin (passes validation) while setting `l1_gas_price_wei` to an arbitrary value (no check exists):

1. **Wrong block hash committed to storage and L1.** The block hash for Starknet ≥ 0.13.4 is `Poseidon("STARKNET_GAS_PRICES0", wei, fri, data_wei, data_fri, l2_wei, l2_fri)`. A manipulated wei value produces a different hash. All validators agree on this wrong hash (they all receive the same `ProposalInit`), so it is accepted by consensus and written to storage. Future proof facts that reference this block hash (`validate_proof_block_hash`) will bind to the wrong value.

2. **Wrong ETH-denominated fee charged / accepted.** `eth_gas_prices.l1_gas_price` is set to the manipulated wei value. Pre-validation (`check_actual_cost_within_bounds`) and post-execution fee checks for ETH-paying transactions use this price, producing wrong fee amounts — either under-charging (economic loss to the sequencer) or over-charging (economic loss to users).

3. **Cascading wrong FRI prices on oracle failure.** The corrupted `previous_block_info` is the only fallback when the oracle is unavailable. The derived `eth_to_fri_rate` will be wrong, causing the next block's FRI prices to be wrong, which in turn causes the next block's proposal to fail the FRI margin check — a liveness impact.

Matching impact scopes: **Critical** (wrong block hash / wrong fee with economic impact) and **High** (RPC returns authoritative-looking wrong `price_in_wei` for the block).

---

### Likelihood Explanation

Any consensus validator can be the proposer for a given round. No majority is required; a single round of proposership suffices. The manipulation is trivial: set `l1_gas_price_wei` to any value in the `ProposalInit` struct before broadcasting. The FRI prices remain within the 10 % margin, so `is_block_info_valid` returns `Ok(())`.

---

### Recommendation

Stop discarding `_l1_gas_prices_wei`. Apply the same `within_margin` check to the proposer's wei prices against the locally-computed wei prices:

```rust
let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(...).await;

// existing FRI checks ...

if !(within_margin(init_proposed.l1_gas_price_wei,
                   l1_gas_prices_wei.l1_gas_price, margin)
  && within_margin(init_proposed.l1_data_gas_price_wei,
                   l1_gas_prices_wei.l1_data_gas_price, margin))
{
    return Err(InvalidBlockInfo(...));
}
```

This mirrors the existing FRI validation and closes the gap.

---

### Proof of Concept

1. Attacker is the proposer for round R.
2. Attacker calls `get_l1_prices_in_fri_and_wei` locally to obtain valid FRI prices (within margin).
3. Attacker constructs `ProposalInit` with `l1_gas_price_fri` = valid, `l1_gas_price_wei` = `1` (1 wei).
4. Validators receive the `ProposalInit` and call `is_block_info_valid`.
5. `_l1_gas_prices_wei` is discarded; only FRI prices are checked — validation passes.
6. `convert_to_sn_api_block_info` sets `eth_gas_prices.l1_gas_price = NonzeroGasPrice(1)`.
7. All ETH-paying transactions in the block are fee-checked against 1 wei/gas — effectively zero cost.
8. `gas_prices_to_hash` hashes `price_in_wei = 1` into the block hash; the committed block hash differs from what any honest proposer would have produced.
9. `PreviousBlockInfo` stores `l1_gas_price_wei = 1`; on the next oracle failure, `calculate_eth_to_fri_rate` returns `l1_gas_price_fri * 10^18 / 1` — an astronomically large rate — causing the next block's FRI prices to overflow or be clamped to `max`, failing the margin check and stalling consensus.

fn calculate_eth_to_fri_rate(block_info: &PreviousBlockInfo) -> Result<u128, StarknetApiError> {
    let eth_to_fri_rate = block_info
        .l1_prices_fri
        .l1_gas_price
        .0
        .checked_mul(WEI_PER_ETH)
        .ok_or_else(|| {
            StarknetApiError::GasPriceConversionError(format!(
                "Gas price in Fri should be small enough to multiply by WEI_PER_ETH. Previous \
                 block info: {:?}",
                block_info
            ))
        })?
        .checked_div(block_info.l1_prices_wei.l1_gas_price.0)
        .ok_or_else(|| {
            StarknetApiError::GasPriceConversionError(format!(
                "Gas price in Wei should be non-zero. Previous block info: {:?}",
                block_info
            ))
        })?;
    if eth_to_fri_rate == 0 {
        return Err(StarknetApiError::GasPriceConversionError(format!(
            "Eth to fri rate is zero. Previous block info: {:?}",
            block_info
        )));
    }
    Ok(eth_to_fri_rate)
