### Title
Proof-Facts Cache Key Omits Transaction Identity, Enabling Proof Verification Bypass — (File: crates/apollo_proof_manager/src/proof_manager.rs)

---

### Summary

The `ProofManager` caches verified proofs keyed solely by `proof_facts.hash()`. That hash is computed over block-level fields only (program hash, block number, block hash, config hash) and never includes the submitting transaction's `sender_address`, `calldata`, `nonce`, or transaction hash. Any unprivileged user who observes a transaction that already has its proof cached can submit a new transaction carrying the same `proof_facts` but an entirely fabricated or invalid `proof`; the gateway skips cryptographic proof verification and admits the transaction.

---

### Finding Description

**Root cause — `ProofFacts.hash()` omits transaction identity**

`ProofFacts` is a flat `Vec<Felt>` whose content is the SNOS virtual-OS output: `[PROOF_VERSION, VIRTUAL_SNOS, program_hash, VIRTUAL_OS_OUTPUT_VERSION, block_number, block_hash, config_hash, l2_to_l1_messages_segment_size, …]`. None of these fields are transaction-specific.

```rust
// crates/starknet_api/src/transaction/fields.rs  line 643-645
pub fn hash(&self) -> Felt {
    HashChain::new().chain_iter(self.0.iter()).get_poseidon_hash()
}
``` [1](#0-0) 

The `SnosProofFacts` struct that is parsed from those bytes contains only `proof_version`, `program_hash`, `block_number`, `block_hash`, and `config_hash` — no `sender_address`, no `calldata` hash, no transaction hash. [2](#0-1) 

**Cache keyed by that hash alone**

`ProofManager.contains_proof` and `ProofManager.set_proof` both derive the storage/cache key exclusively from `proof_facts.hash()`:

```rust
// crates/apollo_proof_manager/src/proof_manager.rs  line 62-64
let facts_hash = proof_facts.hash();
self.proof_storage.set_proof(facts_hash, proof.clone()).await?;
self.cache.insert(facts_hash, proof);
``` [3](#0-2) 

**Verification skipped on cache hit**

`run_proof_verification` (shared by both the gateway and consensus flows) returns immediately without calling `starknet_proof_verifier::verify_proof` when `contains_proof` is true:

```rust
// crates/apollo_transaction_converter/src/transaction_converter.rs  line 403-407
let contains_proof = proof_manager_client.contains_proof(proof_facts.clone()).await?;
if contains_proof {
    return Ok(false);   // ← proof bytes are never inspected
}
``` [4](#0-3) 

**Proof facts ARE included in the transaction hash — but the proof is not**

`compute_invoke_transaction_hash` in the Cairo OS hashes `proof_facts` into the transaction hash (so the account signature covers them), but the raw `proof` bytes are never part of any commitment:

```cairo
// transaction_hash.cairo  line 209-213
if (proof_facts_size != 0) {
    poseidon_hash_update_with_nested_hash(
        data_ptr=proof_facts, data_length=proof_facts_size
    );
}
``` [5](#0-4) 

The OS-level `check_proof_facts` validates only the block-level fields (program hash allowlist, block number staleness, block hash storage lookup, config hash). It never verifies the cryptographic proof itself; that is the sequencer's job — which is bypassed on a cache hit. [6](#0-5) 

**Attack path**

Two distinct `InvokeTransactionV3` transactions from different senders that reference the same past block and produce no L2→L1 messages will have byte-for-byte identical `proof_facts` (same `block_number`, `block_hash`, `config_hash`, `program_hash`, `l2_to_l1_messages_segment_size = 0`). Their `proof_facts.hash()` values are therefore identical, so they share the same `ProofManager` cache slot.

---

### Impact Explanation

Once a legitimate transaction `tx_A` has been admitted and its proof cached under key `H = hash(proof_facts_P)`, any subsequent transaction `tx_B` that:

1. carries the same `proof_facts_P` (publicly visible in `tx_A`), and  
2. carries an arbitrary, invalid, or empty-padded `proof` blob,

will pass `validate_proof_facts_and_proof_consistency` (both fields non-empty), pass `validate_proof_facts` in the blockifier (block-level fields are valid), and have its proof verification silently skipped by `run_proof_verification`. The gateway admits `tx_B` into the mempool with an unverified proof.

This matches the **High** impact tier: *"Mempool/gateway/RPC admission accepts invalid transactions … before sequencing."* [7](#0-6) [8](#0-7) 

---

### Likelihood Explanation

`proof_facts` are a public field of every submitted `InvokeTransactionV3`; they appear verbatim in the RPC response and in the central-objects serialization. No privileged access is required. Any observer of the mempool or block history can extract a valid `proof_facts` vector and reuse it. The only constraint is that the attacker's transaction must pass normal account `__validate__` (their own signature over their own transaction hash, which includes `proof_facts`), which is trivially satisfied by signing with their own key.

---

### Recommendation

Bind the proof-manager cache key to the transaction that submitted the proof. The minimal fix is to key storage by `(transaction_hash, proof_facts.hash())` rather than `proof_facts.hash()` alone. Alternatively, include the `sender_address` and `nonce` in the `ProofFacts` hash computation so that the proof is cryptographically tied to the submitting account and replay position, mirroring how the Starknet transaction hash already binds `sender_address` and `nonce`. [9](#0-8) [10](#0-9) 

---

### Proof of Concept

```
1. Alice submits InvokeV3 tx_A:
     sender_address = 0xAAAA
     calldata       = [transfer(Bob, 100)]
     proof_facts    = P  (block_number=N, block_hash=H, config_hash=C, program_hash=PH, l2l1=0)
     proof          = Q  (valid SNOS proof)
   → gateway calls run_proof_verification(P, Q)
   → contains_proof(P) == false  → verify_proof(P, Q) succeeds
   → ProofManager stores  hash(P) → Q

2. Eve observes tx_A, extracts proof_facts P.

3. Eve submits InvokeV3 tx_B:
     sender_address = 0xEEEE
     calldata       = [transfer(Eve, 999999)]
     proof_facts    = P  (identical to tx_A)
     proof          = R  (garbage bytes, cryptographically invalid)
   → Eve signs tx_B with her own key (valid account signature over tx_B's hash, which includes P)

4. gateway calls run_proof_verification(P, R):
     contains_proof(P) == true   ← cache hit from step 1
     → returns Ok(false)         ← verify_proof(P, R) is NEVER called

5. tx_B is admitted to the mempool with an unverified proof.
   The OS-level check_proof_facts passes (P's block fields are valid).
   Eve's transaction executes.
```

The proof `R` is never cryptographically verified against `tx_B`'s execution. The sequencer has accepted a transaction whose client-side proof claim is entirely unsubstantiated, directly analogous to the MoonscapeDefi pattern where the signed message omits `msg.sender`, allowing any caller to replay a valid owner signature for their own benefit.

func check_proof_facts{range_check_ptr, contract_state_changes: DictAccess*}(
    proof_facts_size: felt,
    proof_facts: felt*,
    current_block_number: felt,
    virtual_os_config_hash: felt,
) {
    if (proof_facts_size == 0) {
        return ();
    }

    assert_le(ProofHeader.SIZE + VirtualOsOutputHeader.SIZE, proof_facts_size);

    // Validate the proof header.
    let proof_header = cast(proof_facts, ProofHeader*);
    assert is_program_hash_allowed(proof_header.program_hash) = TRUE;
    // Proof version and variant are for future compatibility.
    assert [proof_header] = ProofHeader(
        proof_version=PROOF_VERSION,
        proof_variant=VIRTUAL_SNOS,
        program_hash=proof_header.program_hash,
    );

    // Validate the virtual OS output header.
    let os_output_header = cast(&proof_facts[ProofHeader.SIZE], VirtualOsOutputHeader*);

    with_attr error_message("Virtual OS output version is not supported") {
        assert os_output_header.output_version = VIRTUAL_OS_OUTPUT_VERSION;
    }

    // Validate that the proof facts block number is not too recent.
    // (This is a sanity check - the following non-zero check ensures that the block hash is
    // not trivial).
    assert_nn_le(
        os_output_header.base_block_number, current_block_number - STORED_BLOCK_HASH_BUFFER
    );
    // Not all block hashes are stored in the contract; Make sure the requested one is not trivial.
    assert_not_zero(os_output_header.base_block_hash);

    // validate that the proof facts block hash is the true hash of the proof facts block number.
    read_block_hash_from_storage(
        block_number=os_output_header.base_block_number,
        expected_block_hash=os_output_header.base_block_hash,
    );

    // validate that the proof facts config hash is the true hash of the OS config.
    assert os_output_header.starknet_os_config_hash = virtual_os_config_hash;

    return ();
}

    fn validate_proof_facts(
        &self,
        block_context: &BlockContext,
        state: &mut dyn State,
    ) -> TransactionPreValidationResult<()> {
        // Only Invoke V3 transactions can carry proof facts.
        let Transaction::Invoke(invoke_tx) = &self.tx else {
            return Ok(());
        };
        if invoke_tx.version() < TransactionVersion::THREE {
            return Ok(());
        }

        // Parse proof facts.
        let proof_facts = invoke_tx.proof_facts();
        let snos_proof_facts = match ProofFactsVariant::try_from(&proof_facts)
            .map_err(|e| TransactionPreValidationError::InvalidProofFacts(e.to_string()))?
        {
            ProofFactsVariant::Empty => return Ok(()),
            ProofFactsVariant::Snos(snos_proof_facts) => snos_proof_facts,
        };
        let os_constants = &block_context.versioned_constants.os_constants;

        // Validate the program hash.
        let allowed = &os_constants.allowed_virtual_os_program_hashes;
        if !allowed.contains(&snos_proof_facts.program_hash) {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS program hash {} is not allowed",
                snos_proof_facts.program_hash
            )));
        }

        // Validate the block hash and block number.
        let proof_block_hash = snos_proof_facts.block_hash.0;
        let proof_block_number = snos_proof_facts.block_number.0;
        Self::validate_proof_block_number(
            proof_block_number,
            block_context.block_info.block_number,
        )?;
        Self::validate_proof_block_hash(proof_block_hash, proof_block_number, os_constants, state)?;

        // Validate the config hash.
        let chain_info = &block_context.chain_info;
        // TODO(Meshi): Cache this computation as part of the chain context.
        let virtual_os_config_hash = OsChainInfo::from(chain_info)
            .compute_virtual_os_config_hash()
            .expect("Failed to compute OS config hash");
        let proof_config_hash = snos_proof_facts.config_hash;
        if virtual_os_config_hash != proof_config_hash {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS config hash mismatch. Computed virtual OS config hash: \
                 {virtual_os_config_hash}, expected virtual OS config hash: {proof_config_hash}."
            )));
        }

        Ok(())
    }
