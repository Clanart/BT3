### Title
`ProofOfInclusion::valid()` Does Not Verify Against a Trusted Root — Forged DataLayer Inclusion Proofs Pass Validation - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` only checks the internal hash-chain consistency of the proof object itself. It never compares the computed root against any externally-supplied, trusted root hash. An attacker who can supply a `ProofOfInclusion` (via `from_bytes` / `from_py_object`) can craft a fully self-consistent but entirely fabricated proof for any arbitrary `node_hash`, and `valid()` will return `true`.

### Finding Description

`ProofOfInclusion::valid()` is implemented as follows:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()
}
``` [1](#0-0) 

`root_hash()` is defined as:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← derived from the proof itself
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

The final check `existing_hash == self.root_hash()` is a **tautology**: after the loop, `existing_hash` holds the last `calculated_hash`, which was already verified to equal `layer.combined_hash`; and `self.root_hash()` returns exactly `self.layers.last().combined_hash`. The check always passes if the loop passes. No external, trusted root is ever consulted.

An attacker can construct a `ProofOfInclusion` with:
- `node_hash` = SHA-256 of any arbitrary leaf they wish to "prove"
- `layers` = a chain of internally-consistent hashes built from attacker-chosen `other_hash` values

This proof will pass `valid()` while proving nothing about any real DataLayer tree.

The struct is fully constructible from untrusted bytes via the `Streamable` / `from_bytes` / `from_py_object` interfaces exposed through the Python wheel: [3](#0-2) [4](#0-3) 

### Impact Explanation

Any DataLayer consumer (Python node software, wallet, or DataLayer client) that calls `proof.valid()` on a peer-supplied `ProofOfInclusion` without separately comparing `proof.root_hash()` against a trusted on-chain root will accept forged inclusion proofs. This allows an attacker to assert that any arbitrary key-value pair is present in any DataLayer tree, enabling unauthorized DataLayer state transitions and potentially forged DataLayer-backed coin spend authorizations.

This matches the allowed impact: **High — DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, letting untrusted input prove invalid state.**

### Likelihood Explanation

- `ProofOfInclusion` is a `Streamable` type deserializable from raw bytes by any caller.
- The Python bindings expose `from_bytes`, `from_bytes_unchecked`, and `from_py_object` constructors, making attacker-controlled construction trivial.
- The method is named `valid()`, strongly implying full proof validity; callers are unlikely to know they must also check `root_hash()` against a separate trusted source.
- The fuzz target and test suite only call `valid()` on proofs generated internally by `MerkleBlob`, masking the issue for externally-supplied proofs. [5](#0-4) 

### Recommendation

`valid()` must accept a trusted root parameter and compare against it:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    // existing internal-consistency checks ...
    existing_hash == *trusted_root   // compare against externally-trusted root
}
```

Alternatively, rename the current method to `is_internally_consistent()` and add a separate `valid(trusted_root: &Hash) -> bool` that performs the full check. Update all call sites — including the Python bindings — to require the trusted root.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer
import hashlib

# Attacker-chosen leaf to "prove"
fake_leaf = b'\xab' * 32

# Build a single-layer proof with attacker-chosen sibling
sibling = b'\xcd' * 32
# Compute combined_hash as the DataLayer internal hash would
combined = hashlib.sha256(b'\x00' * 30 + b'\x01' + fake_leaf + sibling).digest()

layer = ProofOfInclusionLayer(
    other_hash_side=1,       # sibling is on the right
    other_hash=sibling,
    combined_hash=combined,
)
proof = ProofOfInclusion(node_hash=fake_leaf, layers=[layer])

assert proof.valid()          # ← returns True for a completely fabricated proof
assert proof.root_hash() == combined  # attacker-controlled "root"
# No real DataLayer tree was consulted.
```
