### Title
Stale Cached `MinGasPrice` in `MinGasPriceDecorator` Bypasses Governance-Updated Fee Floor — (`File: ante/cosmos/min_gas_price.go`)

### Summary
`MinGasPriceDecorator` caches a `*feemarkettypes.Params` pointer at construction time and reads `MinGasPrice` exclusively from that pointer in every `AnteHandle` call. The stored `feesKeeper` is never consulted. After a governance parameter-change proposal raises `MinGasPrice`, the decorator permanently enforces the old (lower) value, allowing any Cosmos SDK transaction with fees below the new floor to pass ante-handler validation and be committed.

### Finding Description

`MinGasPriceDecorator` is constructed with a pointer to a `Params` struct that was snapshotted at app-startup (or ante-handler-chain build time):

```go
// ante/cosmos/min_gas_price.go:36-46
type MinGasPriceDecorator struct {
    feesKeeper      interfaces.FeeMarketKeeper
    evmDenom        string
    feemarketParams *feemarkettypes.Params   // ← captured once, never refreshed
}

func NewMinGasPriceDecorator(fk interfaces.FeeMarketKeeper, evmDenom string,
    feemarketParams *feemarkettypes.Params) MinGasPriceDecorator {
    return MinGasPriceDecorator{feesKeeper: fk, evmDenom: evmDenom, feemarketParams: feemarketParams}
}
``` [1](#0-0) 

Inside `AnteHandle`, the minimum gas price is read from the cached pointer, not from the keeper:

```go
// ante/cosmos/min_gas_price.go:54
minGasPrice := mpd.feemarketParams.MinGasPrice
``` [2](#0-1) 

The `feesKeeper` field is stored in the struct but is **never called** anywhere in `AnteHandle`. The keeper's `GetParams(ctx)` always reads the live on-chain value from the KV store:

```go
// x/feemarket/keeper/params.go:28-37
func (k Keeper) GetParams(ctx sdk.Context) types.Params {
    var params types.Params
    bz := ctx.KVStore(k.storeKey).Get(types.ParamsKey)
    ...
    k.cdc.MustUnmarshal(bz, &params)
    return params
}
``` [3](#0-2) 

`SetParams` writes a freshly marshalled struct to the store, so the pointer held by the decorator is never updated in place:

```go
// x/feemarket/keeper/params.go:40-48
func (k Keeper) SetParams(ctx sdk.Context, p types.Params) error {
    ...
    store := ctx.KVStore(k.storeKey)
    bz := k.cdc.MustMarshal(&p)
    store.Set(types.ParamsKey, bz)
    return nil
}
``` [4](#0-3) 

The test suite confirms the construction pattern — a local variable's address is passed, so the pointer is permanently stale after any subsequent `SetParams` call:

```go
// evmd/ante/min_gas_price_test.go:125-126
feemarketParams := s.app.FeeMarketKeeper.GetParams(ctx)
dec := cosmos.NewMinGasPriceDecorator(s.app.FeeMarketKeeper, evmtypes.DefaultEVMDenom, &feemarketParams)
``` [5](#0-4) 

### Impact Explanation

After a governance proposal raises `MinGasPrice` (e.g., from `0` to `10`), the decorator continues to enforce the old value. Any Cosmos SDK transaction (bank send, IBC, staking, etc.) with fees below the new floor passes `AnteHandle` and is committed. This is a fee-market ante-handler bug that permits invalid transactions to commit and causes user fees to be mis-accounted — matching the **High** allowed impact: *"fee market, ante handler … bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

### Likelihood Explanation

`MinGasPrice` defaults to `0`, so the bug is dormant until governance raises it. Once any governance proposal successfully updates `MinGasPrice` to a non-zero value — a routine operational action on live chains — the bypass is permanent for the lifetime of the running node process. No special attacker capability is required beyond submitting a normal Cosmos SDK transaction with a low fee.

### Recommendation

Replace the cached-pointer read with a live keeper lookup inside `AnteHandle`:

```go
func (mpd MinGasPriceDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
    feeTx, ok := tx.(sdk.FeeTx)
    if !ok { ... }

    // Read live params from keeper instead of stale cached pointer
    minGasPrice := mpd.feesKeeper.GetParams(ctx).MinGasPrice
    ...
}
```

The `feemarketParams *feemarkettypes.Params` field and constructor parameter can then be removed entirely, since the keeper already provides the authoritative value.

### Proof of Concept

1. Chain starts with `MinGasPrice = 0`. `MinGasPriceDecorator` is constructed; its `feemarketParams.MinGasPrice = 0`.
2. Governance proposal passes: `MinGasPrice` is raised to `10`. `SetParams` writes the new value to the KV store. The decorator's cached pointer still holds `MinGasPrice = 0`.
3. Attacker submits a `MsgSend` with `fee = 0`.
4. `AnteHandle` evaluates `minGasPrice := mpd.feemarketParams.MinGasPrice` → `0`.
5. The short-circuit `if minGasPrice.IsZero()` fires at line 57, skipping all fee checks entirely.
6. The transaction passes ante-handling and is committed despite violating the governance-set fee floor. [6](#0-5)

func (mpd MinGasPriceDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
	feeTx, ok := tx.(sdk.FeeTx)
	if !ok {
		return ctx, errorsmod.Wrapf(errortypes.ErrInvalidType, "invalid transaction type %T, expected sdk.FeeTx", tx)
	}

	minGasPrice := mpd.feemarketParams.MinGasPrice

	// Short-circuit if min gas price is 0 or if simulating
	if minGasPrice.IsZero() || simulate {
		return next(ctx, tx, simulate)
	}
```
