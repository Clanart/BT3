### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts an arbitrary felt value as the new class hash without verifying it corresponds to a declared contract class. This allows any contract to permanently brick itself by replacing its class hash with an undeclared value, freezing all funds held in its storage irreversibly.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall. The function reads the caller-supplied `class_hash` from the request and directly writes it into the contract's `StateEntry` in `contract_state_changes` — with no check that the hash exists in the declared class registry (`contract_class_changes`).

The missing check is explicitly acknowledged by the codebase itself: [1](#0-0) 

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The `class_hash` field comes directly from the syscall request (attacker-controlled). No cross-reference against `contract_class_changes` or the class commitment tree is performed. The state update is committed unconditionally.

This is the direct analog of the WERC721 bug: just as `totalSupply` was decremented without verifying the NFT was actually unlocked (leaving assets permanently inaccessible), here the contract's `class_hash` is updated without verifying the class is actually declared — leaving the contract permanently inaccessible and its funds frozen.

The class hash is committed into the global state root via `compute_contract_state_commitment` → `hash_contract_state_changes` → `get_contract_state_hash`, which hashes `(class_hash, storage_root, nonce)` without any validity check on `class_hash`: [2](#0-1) 

Once committed, the invalid class hash is part of the canonical state root and cannot be undone.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

When a contract's `class_hash` is set to an undeclared value:

- Every future call to the contract will fail at the OS level because the class cannot be resolved for execution.
- All token balances, ETH, or other assets stored in the contract's storage slots are permanently inaccessible.
- The broken state is committed into the Patricia Merkle Tree and reflected in the global state root, making it irreversible on-chain.

A vault, pool, or escrow contract holding user funds can be permanently bricked by a single `replace_class` call with an arbitrary felt value.

---

### Likelihood Explanation

**Medium.** The attack path requires no special privilege beyond the ability to deploy or interact with a contract that calls `replace_class`:

1. **Malicious deployer (rug pull):** A contract owner deploys a contract, attracts user deposits, then calls `replace_class(arbitrary_felt)` to freeze all deposited funds.
2. **Accidental breakage:** A legitimate contract's upgrade logic passes an incorrect or stale class hash (e.g., a hash that was never declared on this chain), permanently bricking the contract.
3. **Exploit via upgrade path:** An attacker who can trigger an upgrade function in a third-party contract passes an invalid class hash through it.

All three paths are reachable by an unprivileged transaction sender or contract deployer with no operator access required.

---

### Recommendation

In `execute_replace_class`, before writing the new `StateEntry`, verify that `class_hash` exists in `contract_class_changes` (i.e., has a non-zero compiled class hash entry). Concretely:

```cairo
// After reading class_hash from request:
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the invariant enforced in `execute_declare_transaction`, where `prev_value=0` is asserted to ensure a class is declared exactly once: [3](#0-2) 

---

### Proof of Concept

1. Deploy contract `C` (e.g., a vault) that holds user funds and exposes an upgrade function calling `replace_class(new_hash)`.
2. Invoke the upgrade function with `new_hash = 0xdeadbeef` (any felt not present in `contract_class_changes`).
3. The OS executes `execute_replace_class` in `syscall_impls.cairo`:
   - `class_hash = 0xdeadbeef` is read from the request.
   - No validation against declared classes is performed (TODO at line 898).
   - `contract_state_changes` is updated: `C`'s `StateEntry.class_hash = 0xdeadbeef`.
4. `state_update` in `state.cairo` squashes and commits the state; `compute_contract_state_commitment` hashes the invalid class hash into the Patricia tree.
5. The block is finalized. `C`'s class hash in the global state root is now `0xdeadbeef`.
6. Any subsequent call to `C` fails — the OS cannot resolve the class for execution.
7. All user funds stored in `C`'s storage are permanently frozen with no recovery path.

```text
func get_contract_state_hash{hash_ptr: HashBuiltin*}(
    class_hash: felt, storage_root: felt, nonce: felt
) -> (hash: felt) {
    const CONTRACT_STATE_HASH_VERSION = 0;
    if (class_hash == UNINITIALIZED_CLASS_HASH) {
        if (storage_root == 0) {
            if (nonce == 0) {
                return (hash=0);
            }
        }
    }

    // Set res = H(H(class_hash, storage_root), nonce).
    let (hash_value) = hash2(class_hash, storage_root);
    let (hash_value) = hash2(hash_value, nonce);

    // Return H(hash_value, CONTRACT_STATE_HASH_VERSION). CONTRACT_STATE_HASH_VERSION must be in the
    // outermost hash to guarantee unique "decoding".
    let (hash) = hash2(hash_value, CONTRACT_STATE_HASH_VERSION);
    return (hash=hash);
```
