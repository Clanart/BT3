### Title
Missing Validation That Replacement Class Hash Is Declared in `execute_replace_class` — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary
The `execute_replace_class` syscall handler in the StarkNet OS does not validate that the new class hash supplied by the caller is actually a declared class. The developers have explicitly acknowledged this gap with a TODO comment. An unprivileged contract can call `replace_class` with an arbitrary, undeclared class hash, permanently corrupting its own class pointer and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the requested class hash directly from the syscall request and writes it into the contract state without any check that the hash corresponds to a previously declared class:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges the missing check. [2](#0-1) 

By contrast, `execute_entry_point` later performs a `dict_read` on `contract_class_changes` to resolve the class hash to a compiled class hash, and then calls `find_element` to locate the compiled class fact. If the class hash was never declared, the dict read returns 0 (the default), and `find_element` cannot locate a compiled class fact with key 0, causing the OS to fail to produce a valid proof for any block that calls the corrupted contract. [3](#0-2) 

The analogous pattern to the original report is exact: just as `registerToken` accepted `localToken == remoteToken` without validation, `execute_replace_class` accepts any arbitrary `class_hash` without verifying it is declared, allowing invalid state to be committed to the protocol.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is overwritten with an undeclared value, every subsequent call to that contract causes the OS to fail to resolve the class, making the contract permanently uncallable. All ERC-20 balances, NFTs, or other assets held by or managed through that contract become permanently inaccessible. The state corruption is committed to the global state root and cannot be reversed without a protocol-level intervention.

---

### Likelihood Explanation

Any deployed contract can invoke the `replace_class` syscall. No privileged role or special permission is required. A malicious or buggy contract can pass an arbitrary felt value as the new class hash. The OS imposes no on-chain guard, so the invalid state is committed as long as the sequencer includes the transaction — and the sequencer's off-chain validation is not part of the OS proof system's trust boundary.

---

### Recommendation

Add an explicit validation inside `execute_replace_class` that the supplied `class_hash` exists in `contract_class_changes` (i.e., has a non-zero compiled class hash entry), mirroring the check already enforced during `execute_declare_transaction` via `assert_not_zero(compiled_class_hash)` and `dict_update(..., prev_value=0, ...)`. [4](#0-3) 

Concretely, before writing the new state entry in `execute_replace_class`, perform:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This ensures only declared classes can be used as replacement targets, matching the invariant enforced everywhere else in the OS.

---

### Proof of Concept

1. Deploy contract `Victim` holding user funds (e.g., an ERC-20 vault).
2. `Victim` (or an attacker who controls it) calls the `replace_class` syscall with `class_hash = 0xdeadbeef` — an arbitrary value never declared on-chain.
3. The OS executes `execute_replace_class`: no validation is performed; `contract_state_changes` is updated so `Victim`'s `class_hash` field becomes `0xdeadbeef`. [5](#0-4) 
4. The block is finalized and the global state root is updated to reflect `Victim.class_hash = 0xdeadbeef`.
5. In any subsequent block, any transaction calling `Victim` reaches `execute_entry_point`, which does `dict_read(contract_class_changes, key=0xdeadbeef)` → returns 0, then `find_element(..., key=0)` → fails (no compiled class with hash 0 exists). [3](#0-2) 
6. `Victim` is permanently uncallable. All funds it holds are permanently frozen.
