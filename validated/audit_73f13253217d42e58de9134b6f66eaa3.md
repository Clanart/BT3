### Title
Unbounded Predecessor Replay Loop in `TraceTx` gRPC Handler Enables Node Resource Exhaustion - (File: `x/evm/keeper/grpc_query.go`)

### Summary
The `TraceTx` gRPC handler iterates over a caller-supplied `Predecessors` list with no size limit and no timeout, executing a full EVM state transition (`ApplyMessageWithConfig`) for every entry. An unprivileged caller can submit a `QueryTraceTxRequest` containing thousands of computationally expensive predecessor transactions, causing unbounded CPU and memory consumption on the node.

### Finding Description

`TraceTx` in `x/evm/keeper/grpc_query.go` accepts a `QueryTraceTxRequest` whose `Predecessors` field is a `repeated MsgEthereumTx` with no enforced upper bound. [1](#0-0) 

Inside the handler, every predecessor is replayed unconditionally via a full EVM execution: [2](#0-1) 

There is no check on `len(req.Predecessors)` before entering the loop, no per-iteration timeout, and no gas budget that caps the total work. The 5-second `defaultTraceTimeout` is applied only to the final traced transaction inside `prepareTrace`, not to the predecessor replay loop: [3](#0-2) 

The analogous `TraceBlock` handler has the same pattern — it iterates over `req.Txs` with no size limit: [4](#0-3) 

The proto definition confirms `predecessors` is an unbounded repeated field: [5](#0-4) 

### Impact Explanation

Each predecessor call to `ApplyMessageWithConfig` performs a full EVM state transition including storage reads/writes, intrinsic gas computation, and EVM opcode execution. An attacker can craft a request with N predecessors each containing maximum-gas contract calls (e.g., tight loops, large calldata). With N = 10,000 predecessors at the block gas limit each, the node must execute O(N × block_gas_limit) worth of EVM work in a single synchronous gRPC call, with no interruptibility. This exhausts CPU and can trigger OOM on the node process. A crashed validator node misses blocks and may be slashed; if multiple validators are targeted simultaneously, the chain halts.

This matches: **High — Public gRPC path exposes a reachable route to chain halt.**

### Likelihood Explanation

The Cosmos SDK gRPC query port (default 9090) is publicly accessible on most deployments. No authentication is required to call `TraceTx`. The `Predecessors` field is a standard protobuf repeated field — any gRPC client can populate it with arbitrarily many entries. The attack requires only a single malformed gRPC request and no on-chain funds.

### Recommendation

1. Enforce a hard cap on `len(req.Predecessors)` before entering the replay loop (e.g., reject if `> maxBlockTxs` or a configurable constant).
2. Apply the same `defaultTraceTimeout` deadline to the entire predecessor replay loop, not just the final traced transaction.
3. Apply the same fix to `TraceBlock`: cap `len(req.Txs)` before iterating.

```go
    // Each predecessor: infinite-loop bytecode, max gas limit
    predecessors[i] = buildMaxGasLoopTx(chainID, nonce+uint64(i))
}

    Msg:          anyValidTx,
    Predecessors: predecessors,
    BlockNumber:  currentHeight,
    ChainId:      chainID,
}
// Single gRPC call; node enters unbounded loop in TraceTx,
// executing ApplyMessageWithConfig 10,000 times with no timeout.
client.TraceTx(ctx, req)
```
