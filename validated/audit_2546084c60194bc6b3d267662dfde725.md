### Title
Unchecked Return Value of Fee Transfer Execution in `charge_fee` Allows Fee-Free Transaction Execution - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

The `charge_fee` function in `transaction_impls.cairo` calls `non_reverting_select_execute_entry_point_func` to execute the ERC20 fee token transfer but **discards all return values**. If the fee transfer entry point fails (returns `is_reverted=1`), the OS proceeds without detecting the failure, generating a valid proof for a block where fees were never collected. This is the direct Cairo analog of the `transfer` vs `safeTransfer` pattern described in the external report.

---

### Finding Description

In `charge_fee` at lines 160–163, the OS executes the ERC20 `transfer` call on the fee token contract:

```cairo
let remaining_gas = DEFAULT_INITIAL_GAS_COST;
non_reverting_select_execute_entry_point_func{remaining_gas=remaining_gas}(
    block_context=block_context, execution_context=&execution_context
);
return ();
```

The function `non_reverting_select_execute_entry_point_func` returns a tuple `(retdata_size: felt, retdata: felt*, is_deprecated: felt)` and internally calls `execute_entry_point`, which returns `(is_reverted: felt, retdata_size: felt, retdata: felt*)`. When `is_reverted=1`, `execute_entry_point` calls `handle_revert` to undo all storage changes made by the fee transfer — meaning **no fee is deducted from the user and no fee is credited to the sequencer** — but `charge_fee` never inspects this outcome.

This is a direct inconsistency with other call sites of the same function in the same file. In `execute_deploy_account_transaction` (lines 677–684) and `execute_declare_transaction` (lines 804–812), the return values are captured and the `VALIDATED` magic value is asserted:

```cairo
let (retdata_size, retdata, is_deprecated) = non_reverting_select_execute_entry_point_func(
    block_context=block_context, execution_context=validate_deploy_execution_context
);
if (is_deprecated == 0) {
    assert retdata_size = 1;
    assert retdata[0] = VALIDATED;
}
```

No equivalent check exists for the fee transfer in `charge_fee`. [1](#0-0) [2](#0-1) [3](#0-2) 

---

### Impact Explanation

**Critical — Direct loss of funds.**

The OS is the trustless enforcement layer whose output is verified on L1. If the OS proof does not enforce that the fee transfer succeeded, L1 accepts a state transition where:

- The user's fee token balance is **not reduced** (the revert log undoes the storage write).
- The sequencer's fee token balance is **not increased**.
- The transaction is **fully executed** and its effects are committed.

The sequencer receives no compensation for executing the transaction. Because the proof is valid (the OS never asserts fee transfer success), the L1 verifier accepts the block. This constitutes a direct, provable loss of funds at the protocol level — the sequencer is deprived of fee revenue that the protocol guarantees it should receive.

---

### Likelihood Explanation

A user can reliably trigger a failed fee transfer by crafting a transaction whose execution body transfers the user's entire fee token (STRK/ETH) balance to another address. The sequence is:

1. User holds exactly `B` fee tokens.
2. User submits a transaction whose body calls `transfer(other, B)` on the fee token contract.
3. The sequencer's pre-validation passes (user has `B ≥ max_fee` at submission time).
4. The OS executes the transaction body: user's balance drops to 0.
5. The OS calls `charge_fee`, which attempts `transfer(sequencer, actual_fee)`.
6. The fee token's `transfer` fails (balance is 0); `execute_entry_point` returns `is_reverted=1` and calls `handle_revert`.
7. `charge_fee` ignores the return value and returns normally.
8. The OS generates a valid proof. L1 accepts the block. The user paid no fee.

This path requires only an unprivileged transaction sender and no privileged cooperation. The sequencer's pre-validation checks balance at submission time, not post-execution, so it does not prevent this. [4](#0-3) 

---

### Recommendation

Capture and assert the return value of `non_reverting_select_execute_entry_point_func` inside `charge_fee`. Specifically, assert that `is_reverted == 0` (and, for non-deprecated contracts, that `retdata_size == 1` and `retdata[0] == TRUE` per the ERC20 `transfer` return convention). This mirrors the pattern already used for `__validate__` and `__validate_deploy__` entry points in the same file.

```cairo
let (retdata_size, retdata, is_deprecated) = non_reverting_select_execute_entry_point_func{
    remaining_gas=remaining_gas
}(block_context=block_context, execution_context=&execution_context);
// Assert the fee transfer did not revert.
// (is_deprecated path omits the boolean return; non-deprecated ERC20 returns TRUE.)
if (is_deprecated == 0) {
    assert retdata_size = 1;
    assert retdata[0] = 1;  // TRUE: transfer succeeded
}
```

### Proof of Concept

1. Deploy a contract holding `B` STRK where `B ≥ max_fee`.
2. Submit an invoke transaction whose `__execute__` body calls `STRK.transfer(attacker_address, B)`.
3. The sequencer includes the transaction (pre-validation passes: balance `B ≥ max_fee`).
4. OS executes `__execute__`: STRK balance of sender → 0.
5. OS calls `charge_fee` → calls `STRK.transfer(sequencer, actual_fee)`.
6. STRK `transfer` fails (balance = 0); `execute_entry_point` returns `is_reverted=1`, `handle_revert` undoes the storage write.
7. `charge_fee` returns without checking `is_reverted`.
8. OS produces a valid STARK proof. L1 verifier accepts it.
9. Result: transaction executed, state committed, sequencer received 0 fee. [5](#0-4) [6](#0-5) [7](#0-6)

// If max_fee is not 0, validates that the selector matches the entry point of an account contract
// and executes an ERC20 transfer on the behalf of that account contract.
//
// Arguments:
// block_context - a global context that is fixed throughout the block.
// tx_execution_context - The execution context of the transaction that pays the fee.
func charge_fee{
    range_check_ptr,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, tx_execution_context: ExecutionContext*) {
    alloc_locals;

    local tx_info: TxInfo* = tx_execution_context.execution_info.tx_info;
    let max_fee = compute_max_possible_fee(tx_info=tx_info);

    if (max_fee == 0) {
        return ();
    }

    local low_actual_fee;
    %{ LoadActualFee %}
    local calldata: TransferCallData = TransferCallData(
        recipient=block_context.block_info_for_execute.sequencer_address,
        amount=Uint256(low=low_actual_fee, high=0),
    );

    // Verify that the charged amount is not larger than the transaction's max_fee field.
    assert_nn_le(calldata.amount.low, max_fee);

    // TODO(ilya, 01/01/2026): Consider caching the fee_token_class_hash.
    local fee_token_address = block_context.os_global_context.starknet_os_config.fee_token_address;
    let (fee_state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
        key=fee_token_address
    );
    let (__fp__, _) = get_fp_and_pc();
    // Use block_info directly from block_context, so that charge_fee will always run in
    // execute-mode rather than validate-mode.
    local execution_context: ExecutionContext = ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=fee_state_entry.class_hash,
        calldata_size=TransferCallData.SIZE,
        calldata=&calldata,
        execution_info=new ExecutionInfo(
            block_info=block_context.block_info_for_execute,
            tx_info=tx_info,
            caller_address=tx_info.account_contract_address,
            contract_address=fee_token_address,
            selector=TRANSFER_ENTRY_POINT_SELECTOR,
        ),
        deprecated_tx_info=tx_execution_context.deprecated_tx_info,
    );

    let remaining_gas = DEFAULT_INITIAL_GAS_COST;
    non_reverting_select_execute_entry_point_func{remaining_gas=remaining_gas}(
        block_context=block_context, execution_context=&execution_context
    );
    return ();
}
