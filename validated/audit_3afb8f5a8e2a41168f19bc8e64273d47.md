### Title
Missing Nonce Enforcement in `execute_meta_tx_v0` Enables Replay Attacks Leading to Direct Loss of Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_meta_tx_v0` syscall handler in the StarkNet OS accepts a user-controlled `contract_address` and `signature` from the syscall request, computes a transaction hash that **does not include a nonce**, and executes `__execute__` on the target account without any OS-level nonce check or increment. Any unprivileged contract can invoke this syscall. Once a valid meta-tx v0 signature is observed on-chain, an attacker can replay it indefinitely, causing the target account to re-execute the same `__execute__` calldata (e.g., a token transfer) without the account owner's fresh consent.

---

### Finding Description

`execute_meta_tx_v0` is a syscall reachable by any contract via the `META_TX_V0_SELECTOR` dispatch in `execute_syscalls`. [1](#0-0) 

Inside `execute_meta_tx_v0`, the target `contract_address` and `signature` are taken directly from the caller-supplied syscall request: [2](#0-1) 

The meta-tx hash is computed over `(contract_address, entry_point_selector, calldata, chain_id)` — **no nonce is passed**: [3](#0-2) 

The `TxInfo` presented to the account contract hardcodes `nonce=0` and `version=0`: [4](#0-3) 

The OS then calls `__execute__` directly on the target account without invoking `check_and_increment_nonce`: [5](#0-4) 

Compare this to every other transaction type (invoke, deploy_account, declare), where the OS explicitly calls `check_and_increment_nonce` before execution: [6](#0-5) 

Because the hash is nonce-free and the OS never increments `StateEntry.nonce` for meta-tx v0, the same `(contract_address, calldata, signature)` triple produces an identical, valid hash on every invocation. An attacker who observes a submitted meta-tx v0 (signatures are public in transaction data) can replay it by deploying a contract that issues the same `execute_meta_tx_v0` syscall with the captured parameters.

The analog to the original report is direct: just as `depositEUR` accepts a user-controlled `from` address and moves tokens from any account that has granted an allowance, `execute_meta_tx_v0` accepts a user-controlled `contract_address` and re-executes `__execute__` on any account for which a valid signature has ever been seen — because the hash commits to no nonce and the OS enforces no uniqueness.

---

### Impact Explanation

**Critical — Direct loss of funds.**

If the replayed `__execute__` calldata encodes a token transfer (e.g., "send 100 STRK to attacker"), each replay drains the same amount from the victim account. Because the OS never increments the account's `StateEntry.nonce` for meta-tx v0, and because the hash is identical across replays, the attack can be repeated until the account balance is exhausted. Accounts that rely on the OS for nonce management (the standard StarkNet account design) have no OS-enforced protection against this replay.

---

### Likelihood Explanation

**High.**

- No preconditions beyond observing one valid meta-tx v0 on-chain (signatures are public).
- Any unprivileged contract can issue the `META_TX_V0_SELECTOR` syscall.
- The replay requires no privileged role, no leaked key, and no network-level attack.
- The meta-tx v0 mechanism is explicitly designed to be callable by relayers, making the attack surface intentionally broad.

---

### Recommendation

Include a nonce in the meta-tx v0 hash computation (analogous to how `compute_invoke_transaction_hash` includes the nonce), and call `check_and_increment_nonce` (or an equivalent OS-level nonce enforcement) after constructing the `TxInfo` for meta-tx v0. The nonce should be taken from the target account's current `StateEntry.nonce` and incremented atomically, preventing any replay of a previously submitted meta-tx v0.

---

### Proof of Concept

1. **Victim** submits a meta-tx v0 via a relayer: `contract_address=VICTIM`, `calldata=[transfer, ATTACKER, 100]`, `signature=SIG`.
2. The OS computes `hash = H(VICTIM, __execute__, calldata, chain_id)` (no nonce). `SIG` is valid for this hash.
3. **Attacker** deploys `MaliciousRelayer` which issues `execute_meta_tx_v0` syscall with `contract_address=VICTIM`, same `calldata`, same `SIG`.
4. The OS recomputes the identical hash, the account's `__execute__` verifies `SIG` against it (valid), and executes the transfer again.
5. `StateEntry.nonce` for `VICTIM` is never incremented by the OS for meta-tx v0. [7](#0-6) 

6. Step 3–5 repeats until `VICTIM`'s balance is zero.

func execute_meta_tx_v0{
    range_check_ptr,
    syscall_ptr: felt*,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    revert_log: RevertLogEntry*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, caller_execution_context: ExecutionContext*) {
    alloc_locals;

    let request = cast(syscall_ptr + RequestHeader.SIZE, MetaTxV0Request*);
    local calldata_start: felt* = request.calldata_start;
    local calldata_size = request.calldata_end - calldata_start;

    let specific_base_gas_cost = (
        META_TX_V0_GAS_COST + META_TX_V0_CALLDATA_FACTOR_GAS_COST * calldata_size
    );
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=specific_base_gas_cost, request_struct_size=MetaTxV0Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    local contract_address = request.contract_address;
    local selector = request.selector;
    local caller_execution_info: ExecutionInfo* = caller_execution_context.execution_info;
    local old_tx_info: TxInfo* = caller_execution_info.tx_info;

    if (selector != EXECUTE_ENTRY_POINT_SELECTOR) {
        write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
        return ();
    }

    // Sanity check: Verify that `signature` is a valid Sierra array.
    assert_nn_le(request.signature_end - request.signature_start, SIERRA_ARRAY_LEN_BOUND - 1);

    let (state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
        key=contract_address
    );

    // Compute the meta-transaction hash.
    let pedersen_ptr = builtin_ptrs.selectable.pedersen;
    with pedersen_ptr {
        let meta_tx_hash = compute_meta_tx_v0_hash(
            contract_address=contract_address,
            entry_point_selector=selector,
            calldata=calldata_start,
            calldata_size=calldata_size,
            chain_id=old_tx_info.chain_id,
        );
    }
    update_pedersen_in_builtin_ptrs(pedersen_ptr=pedersen_ptr);

    // Prepare execution context.
    tempvar new_tx_info = new TxInfo(
        version=0,
        account_contract_address=contract_address,
        max_fee=0,
        signature_start=request.signature_start,
        signature_end=request.signature_end,
        transaction_hash=meta_tx_hash,
        chain_id=old_tx_info.chain_id,
        nonce=0,
        resource_bounds_start=cast(0, ResourceBounds*),
        resource_bounds_end=cast(0, ResourceBounds*),
        tip=0,
        paymaster_data_start=cast(0, felt*),
        paymaster_data_end=cast(0, felt*),
        nonce_data_availability_mode=0,
        fee_data_availability_mode=0,
        account_deployment_data_start=cast(0, felt*),
        account_deployment_data_end=cast(0, felt*),
        proof_facts_start=cast(0, felt*),
        proof_facts_end=cast(0, felt*),
    );

    let (deprecated_tx_info_ptr: DeprecatedTxInfo*) = alloc();
    tempvar execution_context: ExecutionContext* = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=state_entry.class_hash,
        calldata_size=calldata_size,
        calldata=calldata_start,
        execution_info=new ExecutionInfo(
            block_info=caller_execution_info.block_info,
            tx_info=new_tx_info,
            caller_address=ORIGIN_ADDRESS,
            contract_address=contract_address,
            selector=selector,
        ),
        deprecated_tx_info=deprecated_tx_info_ptr,
    );
    fill_deprecated_tx_info(tx_info=new_tx_info, dst=execution_context.deprecated_tx_info);

    // Since we process the revert log backwards, entries before this point belong to the calling
    // contract.
    assert [revert_log] = RevertLogEntry(
        selector=CHANGE_CONTRACT_ENTRY, value=caller_execution_info.contract_address
    );
    let revert_log = &revert_log[1];

    contract_call_helper(
        remaining_gas=remaining_gas,
        block_context=block_context,
        execution_context=execution_context,
    );

    // Entries before this point belong to the callee.
    assert [revert_log] = RevertLogEntry(selector=CHANGE_CONTRACT_ENTRY, value=contract_address);
    let revert_log = &revert_log[1];

    return ();
}

func check_and_increment_nonce{contract_state_changes: DictAccess*}(tx_info: TxInfo*) -> () {
    // Do not handle nonce for version 0.
    if (tx_info.version == 0) {
        return ();
    }

    tempvar state_entry: StateEntry*;
    %{ SetStateEntryToAccountContractAddress %}

    tempvar current_nonce = state_entry.nonce;
    with_attr error_message("Unexpected nonce.") {
        assert current_nonce = tx_info.nonce;
    }

    // Update contract_state_changes.
    tempvar new_state_entry = new StateEntry(
        class_hash=state_entry.class_hash,
        storage_ptr=state_entry.storage_ptr,
        nonce=current_nonce + 1,
    );
    dict_update{dict_ptr=contract_state_changes}(
        key=tx_info.account_contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    return ();
