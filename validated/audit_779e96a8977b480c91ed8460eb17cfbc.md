### Title
Recipient Token Blacklist Causes Permanent Fund Lock in `finTransfer` - (`evm/src/omni-bridge/contracts/OmniBridge.sol`)

---

### Summary

`OmniBridge.sol::finTransfer()` pushes tokens directly to `payload.recipient` via `IERC20.safeTransfer`. If the recipient address is blacklisted by the token contract (e.g., USDC, USDT), every finalization attempt reverts. Because there is no cancel or redirect mechanism on either the EVM or NEAR side, the user's tokens locked on NEAR during `init_transfer` become permanently irrecoverable.

---

### Finding Description

`finTransfer` in `OmniBridge.sol` follows this sequence:

1. Check and mark nonce used: `completedTransfers[payload.destinationNonce] = true`
2. Verify MPC signature
3. Push tokens to recipient:

```solidity
} else {
    IERC20(payload.tokenAddress).safeTransfer(
        payload.recipient,
        payload.amount
    );
}
``` [1](#0-0) 

Because Solidity reverts atomically, the nonce marking at line 287 is rolled back when `safeTransfer` reverts. The nonce is therefore not permanently consumed — but the transfer is permanently unexecutable. Every subsequent call with the same signed payload will hit the same revert. There is no alternative delivery path, no redirect-to-sender function, and no expiry/cancel mechanism anywhere in the NEAR bridge contract. [2](#0-1) 

Once `init_transfer` is called on NEAR, tokens are locked (or burned for deployed tokens) and a `TransferMessage` is stored. The only way to release those tokens is a successful `fin_transfer` on the destination chain. No `cancel_transfer` or `refund_transfer` function exists. [3](#0-2) 

The same structural issue exists in the StarkNet bridge: `_set_transfer_finalised` is called before the ERC-20 transfer, and if `transfer` returns `false` the assert reverts the whole transaction — but again, no recovery path exists. [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent freezing / irrecoverable lock of user funds.**

A user who initiates a NEAR → EVM transfer of a blacklist-capable token (USDC, USDT, etc.) to a recipient address that is or becomes blacklisted will have their tokens locked on NEAR forever. The bridge holds the locked balance, the destination transfer can never succeed, and no on-chain recovery path exists. The locked amount is bounded only by the transfer size — potentially large for institutional users.

---

### Likelihood Explanation

**Medium.**

USDC and USDT are among the most bridged assets and both maintain operator-controlled blacklists. Blacklisting can happen:
- After transfer initiation due to regulatory action against the recipient address.
- Deliberately: a malicious actor could initiate a transfer to a known-blacklisted address to grief themselves or, in a more complex scenario, to grief a relayer's collateral in fast-transfer flows.
- By mistake: users sometimes copy a wrong or sanctioned address.

The bridge explicitly handles USDC as a native (non-bridge) token via the `safeTransfer` branch, making this path reachable in production.

---

### Recommendation

1. **Pull-over-push for failed deliveries:** If `safeTransfer` fails (wrap in a try/catch or check return value), store the amount in a claimable mapping keyed by `(token, recipient)` so the recipient can pull funds later, or allow the sender to redirect to an alternative address after a timeout.
2. **Cancel/redirect mechanism:** Add a `cancelTransfer` function on NEAR that, given proof that the destination `finTransfer` is permanently unexecutable (e.g., after N failed attempts or after a timeout), releases the locked tokens back to the original sender.
3. **Recipient validation:** Optionally, validate that the recipient is not a zero address before signing the transfer on NEAR, though blacklist status cannot be checked cross-chain at signing time.

---

### Proof of Concept

**Step 1 — Initiate transfer on NEAR:**
```
User calls ft_on_transfer → init_transfer on NEAR bridge
  → 1,000 USDC locked in NEAR bridge contract
  → TransferMessage stored with recipient = 0xBLACKLISTED_EVM_ADDRESS
```

**Step 2 — Relayer attempts finalization on EVM:**
```solidity
// OmniBridge.sol::finTransfer called with:
// payload.tokenAddress = USDC (0xA0b86991...)
// payload.recipient    = 0xBLACKLISTED_EVM_ADDRESS
// payload.amount       = 1_000_000_000 (1000 USDC, 6 decimals)

completedTransfers[nonce] = true;          // set
// ... signature verified ...
IERC20(USDC).safeTransfer(0xBLACKLISTED, 1_000_000_000);
// ↑ USDC reverts: "Blacklistable: account is blacklisted"
// Entire tx reverts → completedTransfers[nonce] rolled back to false
```

**Step 3 — Permanent lock:**
- Relayer retries → same revert, indefinitely.
- No `cancelTransfer` exists on NEAR.
- User's 1,000 USDC is permanently locked in the NEAR bridge contract.
- `get_locked_tokens(ChainKind::Eth, usdc_account_id)` still shows the locked balance, unreachable. [5](#0-4) [6](#0-5)

    function finTransfer(
        bytes calldata signatureData,
        BridgeTypes.TransferMessagePayload calldata payload
    ) external payable whenNotPaused(PAUSED_FIN_TRANSFER) {
        if (completedTransfers[payload.destinationNonce]) {
            revert NonceAlreadyUsed(payload.destinationNonce);
        }

        completedTransfers[payload.destinationNonce] = true;

        bytes memory borshEncoded = bytes.concat(
            bytes1(uint8(BridgeTypes.PayloadType.TransferMessage)),
            Borsh.encodeUint64(payload.destinationNonce),
            bytes1(payload.originChain),
            Borsh.encodeUint64(payload.originNonce),
            bytes1(omniBridgeChainId),
            Borsh.encodeAddress(payload.tokenAddress),
            Borsh.encodeUint128(payload.amount),
            bytes1(omniBridgeChainId),
            Borsh.encodeAddress(payload.recipient),
            bytes(payload.feeRecipient).length == 0 // None or Some(String) in rust
                ? bytes("\x00")
                : bytes.concat(
                    bytes("\x01"),
                    Borsh.encodeString(payload.feeRecipient)
                ),
            bytes(payload.message).length == 0
                ? bytes("")
                : Borsh.encodeBytes(payload.message)
        );
        bytes32 hashed = keccak256(borshEncoded);

        if (ECDSA.recover(hashed, signatureData) != nearBridgeDerivedAddress) {
            revert InvalidSignature();
        }

        MultiTokenInfo memory multiToken = multiTokens[payload.tokenAddress];

        if (payload.tokenAddress == address(0)) {
            // slither-disable-next-line arbitrary-send-eth
            (bool success, ) = payload.recipient.call{value: payload.amount}(
                ""
            );
            if (!success) revert FailedToSendEther();
        } else if (multiToken.tokenAddress != address(0)) {
            IERC1155(multiToken.tokenAddress).safeTransferFrom(
                address(this),
                payload.recipient,
                multiToken.tokenId,
                payload.amount,
                ""
            );
        } else if (customMinters[payload.tokenAddress] != address(0)) {
            ICustomMinter(customMinters[payload.tokenAddress]).mint(
                payload.tokenAddress,
                payload.recipient,
                payload.amount
            );
        } else if (isBridgeToken[payload.tokenAddress]) {
            if (payload.message.length == 0) {
                IBridgeToken(payload.tokenAddress).mint(
                    payload.recipient,
                    payload.amount
                );
            } else {
                IBridgeToken(payload.tokenAddress).mint(
                    payload.recipient,
                    payload.amount,
                    payload.message
                );
            }
        } else {
            IERC20(payload.tokenAddress).safeTransfer(
                payload.recipient,
                payload.amount
            );
        }

        finTransferExtension(payload);

        emit BridgeTypes.FinTransfer(
            payload.originChain,
            payload.originNonce,
            payload.tokenAddress,
            payload.amount,
            payload.recipient,
            payload.feeRecipient
        );
    }

        fn fin_transfer(
            ref self: ContractState, signature: Signature, payload: TransferMessagePayload,
        ) {
            assert(!_is_paused(@self, PAUSE_FIN_TRANSFER), 'ERR_FIN_TRANSFER_PAUSED');

            assert(
                !self.is_transfer_finalised(payload.destination_nonce), 'ERR_NONCE_ALREADY_USED',
            );
            _set_transfer_finalised(ref self, payload.destination_nonce);

            _verify_borsh_signature(
                ref self, @payload.to_borsh(self.omni_bridge_chain_id.read()), signature,
            );

            if self.is_bridge_token(payload.token_address) {
                IBridgeTokenDispatcher { contract_address: payload.token_address }
                    .mint(payload.recipient, payload.amount.into());
            } else {
                let success = IERC20Dispatcher { contract_address: payload.token_address }
                    .transfer(payload.recipient, payload.amount.into());
                assert(success, 'ERR_TRANSFER_FAILED');
            }

            self
                .emit(
                    Event::FinTransfer(
                        FinTransfer {
                            origin_chain: payload.origin_chain,
                            origin_nonce: payload.origin_nonce,
                            token_address: payload.token_address,
                            amount: payload.amount,
                            recipient: payload.recipient,
                            fee_recipient: payload.fee_recipient,
                            message: payload.message,
                        },
                    ),
                )
        }

    fn unlock_tokens(
        &mut self,
        chain_kind: ChainKind,
        token_id: &AccountId,
        amount: u128,
    ) -> LockAction {
        let key = (chain_kind, token_id.clone());
        let Some(available) = self.locked_tokens.get(&key) else {
            return LockAction::Unchanged;
        };
        require!(
            available >= amount,
            TokenLockError::InsufficientLockedTokens.as_ref()
        );

        let remaining = available - amount;
        self.locked_tokens.insert(&key, &remaining);

        LockAction::Unlocked {
            chain_kind,
            token_id: token_id.clone(),
            amount,
        }
    }
