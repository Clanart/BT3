### Title
`CommitmentStateDiff`→`ThinStateDiff` Conversion Silently Drops `deprecated_declared_classes`, Producing Wrong `state_diff_commitment`, `state_diff_length`, and Block Hash for Blocks Containing Deprecated Declare Transactions - (File: `crates/blockifier/src/state/cached_state.rs`)

---

### Summary

The `From<CommitmentStateDiff> for ThinStateDiff` conversion unconditionally sets `deprecated_declared_classes: Vec::new()`. This is the sole path used by `BlockExecutionArtifacts::new` to derive the `ThinStateDiff` that feeds `calculate_block_commitments` and `thin_state_diff()`. Any block that includes a deprecated (Cairo 0) Declare transaction will therefore have a wrong `state_diff_commitment`, a wrong `state_diff_length`, a wrong `concatenated_counts` field, and ultimately a wrong block hash. The stored state diff is also wrong, so the error propagates consistently through storage, P2P sync, and the proof pipeline.

---

### Finding Description

**Root cause — the pseudo-state conversion**

`CommitmentStateDiff` has no `deprecated_declared_classes` field:

```rust
pub struct CommitmentStateDiff {
    pub address_to_class_hash: IndexMap<ContractAddress, ClassHash>,
    pub address_to_nonce: IndexMap<ContractAddress, Nonce>,
    pub storage_updates: IndexMap<ContractAddress, IndexMap<StorageKey, Felt>>,
    pub class_hash_to_compiled_class_hash: IndexMap<ClassHash, CompiledClassHash>,
}
``` [1](#0-0) 

When it is converted to `ThinStateDiff`, the missing field is silently filled with an empty vector:

```rust
impl From<CommitmentStateDiff> for ThinStateDiff {
    fn from(commitment_state_diff: CommitmentStateDiff) -> Self {
        Self {
            ...
            // TODO(AlonH): Remove this when the structure of storage diffs changes.
            deprecated_declared_classes: Vec::new(),
        }
    }
}
``` [2](#0-1) 

**How the wrong value propagates into the block commitment**

`BlockExecutionArtifacts::new` calls `calculate_block_commitments` with the result of this conversion:

```rust
let (header_commitments, measurements) = calculate_block_commitments(
    &transactions_data,
    ThinStateDiff::from(commitment_state_diff.clone()),  // deprecated_declared_classes = []
    l1_da_mode,
    &block_info.starknet_version,
).await;
``` [3](#0-2) 

`calculate_block_commitments` computes both the `state_diff_commitment` (Poseidon hash) and the `state_diff_length`, which are both wrong:

```rust
let concatenated_counts = concat_counts(
    transactions_data.len(),
    event_leaf_elements.len(),
    state_diff.len(),   // wrong: deprecated_declared_classes not counted
    l1_da_mode,
);
...
let state_diff_task = spawn_measured_task(move || calculate_state_diff_hash(&state_diff));
``` [4](#0-3) 

`calculate_state_diff_hash` explicitly chains `deprecated_declared_classes` into the Poseidon hash:

```rust
hash_chain = chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
``` [5](#0-4) 

`ThinStateDiff::len()` counts `deprecated_declared_classes`:

```rust
result += self.deprecated_declared_classes.len();
``` [6](#0-5) 

The wrong `state_diff_commitment` and wrong `concatenated_counts` (which encodes `state_diff_length`) are both chained into the final block hash:

```rust
.chain(&block_commitments.concatenated_counts)
.chain(&block_commitments.state_diff_commitment.0.0)
``` [7](#0-6) 

**The stored state diff is also wrong**

`thin_state_diff()` uses the same conversion, so the `ThinStateDiff` written to storage via `append_state_diff` also has `deprecated_declared_classes: Vec::new()`:

```rust
pub fn thin_state_diff(&self) -> ThinStateDiff {
    ThinStateDiff::from(self.commitment_state_diff.clone())
}
``` [8](#0-7) 

**Contrast with the RPC execution path**

The RPC simulation/fee-estimation path (`induced_state_diff`) correctly handles deprecated classes by accepting the class hash as a separate parameter and injecting it:

```rust
deprecated_declared_classes: deprecated_declared_class_hash
    .map_or_else(Vec::new, |class_hash| vec![class_hash]),
``` [9](#0-8) 

No equivalent injection exists in the block-building path.

---

### Impact Explanation

For any block that includes a deprecated (Cairo 0) Declare transaction:

1. **Wrong `state_diff_commitment`** — the Poseidon hash stored in the block header omits the deprecated class hashes. Any verifier (prover, syncing node) that recomputes the hash from the actual state changes will get a mismatch.
2. **Wrong `state_diff_length`** — the count stored in the block header is too small by the number of deprecated declared classes.
3. **Wrong `concatenated_counts`** — this packed field encodes `state_diff_length` and is chained into the block hash.
4. **Wrong block hash** — the final `BlockHash` is wrong because both `state_diff_commitment` and `concatenated_counts` are wrong inputs.
5. **Wrong stored state diff** — the `ThinStateDiff` written to MDBX storage also has empty `deprecated_declared_classes`, so the error is consistent across storage, P2P sync, and the proof pipeline. The P2P sync server serves the wrong state diff, and the `state_diff_length` in the header matches the wrong stored diff, making the error invisible to length-based validation.

This matches the impact category: **Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input** and **Wrong compiled class, CASM/native artifact, class hash, or contract code selected for execution**.

---

### Likelihood Explanation

Deprecated Declare (V0/V1) transactions are still valid in the Starknet protocol. If the gateway accepts them (which the blockifier is capable of executing, as evidenced by the `induced_state_diff` handling), any such transaction included in a sequenced block triggers the bug. The TODO comment in the conversion (`// TODO(AlonH): Remove this when the structure of storage diffs changes`) confirms the gap is known but unresolved. The bug requires no privileged access — any user who can submit a deprecated Declare transaction to the sequencer can trigger it.

---

### Recommendation

1. Add `deprecated_declared_classes` to `CommitmentStateDiff` and populate it in the blockifier when a deprecated Declare transaction is executed, mirroring the approach already used in `induced_state_diff`.
2. Remove the hardcoded `deprecated_declared_classes: Vec::new()` from `From<CommitmentStateDiff> for ThinStateDiff` once the field is populated.
3. Until the fix is in place, reject deprecated Declare transactions at the gateway to prevent the wrong commitment from being committed.

---

### Proof of Concept

1. Submit a deprecated Declare (V0 or V1) transaction declaring Cairo 0 class `C` to the sequencer.
2. The blockifier executes it; `CommitmentStateDiff` is produced with no record of `C`.
3. `BlockExecutionArtifacts::new` calls `ThinStateDiff::from(commitment_state_diff)` → `deprecated_declared_classes = []`.
4. `calculate_state_diff_hash` computes `Poseidon("STARKNET_STATE_DIFF0", ..., 0 /* n_deprecated */, ...)` — missing `C`.
5. `ThinStateDiff::len()` returns `N` instead of `N+1`.
6. `concat_counts(txs, events, N, da_mode)` encodes the wrong `state_diff_length`.
7. `calculate_block_hash` chains the wrong `state_diff_commitment` and wrong `concatenated_counts` → wrong `BlockHash`.
8. An independent verifier that recomputes the state diff hash from the actual class trie (which does contain `C`) will compute a different `state_diff_commitment` and a different block hash, detecting the inconsistency.

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
