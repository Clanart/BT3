### Title
Deploy Syscall Hardcodes `failure_flag=0`, Silently Discarding Constructor Reverts and Enabling Permanent Fund Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_deploy` syscall implementation in the StarkNet OS unconditionally writes `failure_flag=0` in the deploy response header, regardless of whether the deployed contract's constructor actually reverted. A calling factory contract therefore cannot distinguish a successful deploy from a failed one. Any funds transferred to the "deployed" address after a silently-failed constructor are permanently frozen, because the contract at that address remains uninitialized (`class_hash = 0`, no entry points).

---

### Finding Description

In `syscall_impls.cairo`, `execute_deploy` calls `deploy_contract` and then unconditionally asserts a success response:

```cairo
with remaining_gas {
    let (retdata_size, retdata) = deploy_contract(
        block_context=block_context, constructor_execution_context=constructor_execution_context
    );
}

// Write the response header.
// TODO(Yoni, 1/1/2026): support failures.
assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=0);
``` [1](#0-0) 

The `TODO` comment explicitly acknowledges that failure reporting is unimplemented. The `failure_flag` field in `ResponseHeader` is the only mechanism by which a calling contract can learn whether a syscall succeeded or failed. Because it is hardcoded to `0`, the calling contract's execution always continues on the "success" branch after a `deploy` syscall, even when the constructor reverted.

The revert mechanism (`revert_log`) is an implicit argument of `execute_deploy` and is threaded into `deploy_contract`. When the constructor reverts, the revert log correctly undoes the class-hash assignment for the new contract address, leaving it at `class_hash = 0` (uninitialized). However, this internal state correction is invisible to the calling contract because `failure_flag` is never set to `1`. [2](#0-1) 

For contrast, `contract_call_helper` (used by `execute_call_contract` and `execute_library_call`) correctly propagates the `is_reverted` flag:

```cairo
assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=is_reverted);
``` [3](#0-2) 

The deploy syscall is the sole exception that never propagates failure.

The revert log structure and its class-hash restoration path are confirmed in `revert.cairo`:

```cairo
if (selector == CHANGE_CLASS_ENTRY) {
    // Change class entry.
    let class_hash = revert_log_end[0].value;
    return revert_contract_changes();
}
``` [4](#0-3) 

So after a constructor revert: the on-chain state has `class_hash = 0` for the new address, but the calling contract's execution proceeds as if `class_hash` was set correctly.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

A factory contract that deploys child contracts and transfers initial funds to them (a standard DeFi pattern: DEX pool factories, vault factories, NFT collection factories) cannot detect a failed constructor. The sequence:

1. Factory calls `deploy` syscall with attacker-supplied constructor calldata.
2. Constructor reverts; revert log restores `class_hash = 0` for the child address.
3. OS writes `failure_flag = 0`; factory's execution continues on the success branch.
4. Factory transfers tokens to the child address (e.g., initial liquidity, seed funds).
5. Child address has `class_hash = 0` — no entry points exist.
6. Tokens are permanently frozen; no withdrawal path exists.

There is no recovery mechanism: the child address cannot be re-deployed to the same address with a different salt (the address derivation is deterministic and the nonce/state slot is already touched), and the funds cannot be retrieved because no callable entry point exists.

---

### Likelihood Explanation

Factory-deploy-and-fund patterns are ubiquitous in DeFi protocols on StarkNet. Any permissionless factory that accepts user-supplied constructor calldata (salt, class hash, constructor arguments) is directly exploitable. An attacker needs only to craft constructor calldata that causes the constructor to revert (e.g., by violating an invariant check inside the constructor, passing an out-of-range argument, or exhausting gas). The OS-level bug guarantees the factory cannot guard against this regardless of how carefully it is written, because the failure signal is suppressed at the protocol layer.

---

### Recommendation

Propagate the constructor's revert status through the deploy response. `deploy_contract` should return an `is_reverted` flag (analogous to `select_execute_entry_point_func`), and `execute_deploy` should write:

```cairo
assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=is_reverted);
```

This mirrors the already-correct behavior in `contract_call_helper`. Additionally, when `is_reverted = 1`, the response body should carry the constructor's failure reason so the calling contract can handle it gracefully.

---

### Proof of Concept

```
1. Attacker identifies a permissionless factory contract F that:
      a. Accepts (class_hash, salt, constructor_calldata) from the caller.
      b. Calls the `deploy` syscall.
      c. Transfers `initial_funds` tokens to the returned contract_address.

2. Attacker supplies constructor_calldata crafted to trigger a revert
   inside the constructor (e.g., assert(0) or an out-of-range value).

3. OS executes the constructor → constructor reverts.
   Revert log restores class_hash = 0 for the new address.
   OS writes failure_flag = 0 to the deploy response.

4. Factory reads failure_flag = 0, concludes deploy succeeded.
   Factory executes: transfer(child_address, initial_funds).

5. child_address now holds `initial_funds` tokens.
   child_address.class_hash = 0 → no entry points → no withdrawal path.

6. `initial_funds` are permanently frozen.
   Attack cost: one transaction + gas.
   No privileged access required.
```

    range_check_ptr,
    syscall_ptr: felt*,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    revert_log: RevertLogEntry*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, caller_execution_context: ExecutionContext*) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, DeployRequest*);
    local constructor_calldata_start: felt* = request.constructor_calldata_start;
    local constructor_calldata_size = request.constructor_calldata_end - constructor_calldata_start;

    let specific_base_gas_cost = DEPLOY_GAS_COST + DEPLOY_CALLDATA_FACTOR_GAS_COST *
        constructor_calldata_size;
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=specific_base_gas_cost, request_struct_size=DeployRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    local caller_execution_info: ExecutionInfo* = caller_execution_context.execution_info;
    local caller_address = caller_execution_info.contract_address;

    // Verify deploy_from_zero is either 0 (FALSE) or 1 (TRUE).
    tempvar deploy_from_zero = request.deploy_from_zero;
    assert deploy_from_zero * (deploy_from_zero - 1) = 0;
    // Set deployer_address to 0 if request.deploy_from_zero is TRUE.
    let deployer_address = (1 - deploy_from_zero) * caller_address;

    let selectable_builtins = &builtin_ptrs.selectable;
    let hash_ptr = selectable_builtins.pedersen;
    with hash_ptr {
        let (contract_address) = get_contract_address(
            salt=request.contract_address_salt,
            class_hash=request.class_hash,
            constructor_calldata_size=constructor_calldata_size,
            constructor_calldata=constructor_calldata_start,
            deployer_address=deployer_address,
        );
    }
    tempvar builtin_ptrs = new BuiltinPointers(
        selectable=SelectableBuiltins(
            pedersen=hash_ptr,
            range_check=selectable_builtins.range_check,
            ecdsa=selectable_builtins.ecdsa,
            bitwise=selectable_builtins.bitwise,
            ec_op=selectable_builtins.ec_op,
            poseidon=selectable_builtins.poseidon,
            segment_arena=selectable_builtins.segment_arena,
            range_check96=selectable_builtins.range_check96,
            add_mod=selectable_builtins.add_mod,
            mul_mod=selectable_builtins.mul_mod,
        ),
        non_selectable=builtin_ptrs.non_selectable,
    );

    tempvar constructor_execution_context = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_CONSTRUCTOR,
        class_hash=request.class_hash,
        calldata_size=constructor_calldata_size,
        calldata=constructor_calldata_start,
        execution_info=new ExecutionInfo(
            block_info=caller_execution_info.block_info,
            tx_info=caller_execution_info.tx_info,
            caller_address=caller_address,
            contract_address=contract_address,
            selector=CONSTRUCTOR_ENTRY_POINT_SELECTOR,
        ),
        deprecated_tx_info=caller_execution_context.deprecated_tx_info,
    );

    with remaining_gas {
        let (retdata_size, retdata) = deploy_contract(
            block_context=block_context, constructor_execution_context=constructor_execution_context
        );
    }

    // TODO(Yoni, 1/1/2026): consider sharing code with call_contract_helper.
    let response_header = cast(syscall_ptr, ResponseHeader*);
    let syscall_ptr = syscall_ptr + ResponseHeader.SIZE;

    // Write the response header.
    // TODO(Yoni, 1/1/2026): support failures.
    assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=0);

    assert [response] = DeployResponse(
        contract_address=contract_address,
        constructor_retdata_start=retdata,
        constructor_retdata_end=retdata + retdata_size,
    );

    return ();
