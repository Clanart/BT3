### Title
Stale `feemarketParams` Snapshot in `NewDynamicFeeChecker` Allows Cosmos SDK Transactions to Bypass the Current Base Fee - (`evmd/ante/handler_options.go`)

### Summary
`newCosmosAnteHandler` captures `feemarketParams` once at app-startup time and passes a pointer to that frozen snapshot into `NewDynamicFeeChecker`. Because the EIP-1559 base fee is updated every block in `BeginBlock`, the fee checker for Cosmos SDK dynamic-fee transactions always validates against the genesis base fee rather than the live one. An unprivileged user can therefore submit Cosmos SDK transactions whose fee satisfies only the genesis base fee, bypassing the current (potentially much higher) base fee and committing with mis-accounted fees.

### Finding Description

**Root cause — `newCosmosAnteHandler` captures params once at construction time**

`newCosmosAnteHandler` is called exactly once, during `setAnteHandler` at application startup:

```go
// evmd/app.go
func (app *EthermintApp) setAnteHandler(...) {
    anteHandler, err := ante.NewAnteHandler(ante.HandlerOptions{...})
    app.SetAnteHandler(anteHandler)
}
```

Inside `newCosmosAnteHandler`, the fee-market params are read from the store at that single moment and a pointer to the local copy is handed to `NewDynamicFeeChecker`:

**The base fee is updated every block but the snapshot is never refreshed**

`feemarket.BeginBlock` calls `CalculateBaseFee` and persists the result via `SetBaseFee` → `SetParams`:

The EIP-1559 formula can raise the base fee by up to `baseFee × (ElasticityMultiplier − 1) / BaseFeeChangeDenominator` per block (≈ 12.5 % with defaults), so after sustained high utilisation the live base fee can be orders of magnitude above the genesis value.

**Contrast with the Ethereum-tx path, which is correct**

### Impact Explanation

Any user can craft a Cosmos SDK transaction carrying `ExtensionOptionDynamicFeeTx` whose `feeCap` satisfies only the genesis base fee. Because `NewDynamicFeeChecker` validates against the frozen genesis value, the transaction passes `DeductFeeDecorator`, is committed to the chain, and the fee collector receives far less than the current protocol-mandated minimum. This is a direct fee mis-accounting bug in the ante handler: invalid transactions (insufficient fees relative to the live base fee) are committed, and user funds/fees are mis-accounted. The `DynamicFeeChecker` flag is `true` in the production app, so the path is always active.

### Likelihood Explanation

The base fee drifts upward whenever blocks are above the gas target. On any chain with sustained moderate-to-high utilisation the live base fee will exceed the genesis value within hours. The attack requires no special privilege: any account can submit a Cosmos SDK transaction with `ExtensionOptionDynamicFeeTx`. The attacker simply sets `feeCap` to the well-known genesis base fee (e.g. `1_000_000_000` = 1 Gwei, the `InitialBaseFee` constant) and the stale checker accepts it regardless of the current value.

### Recommendation

`newCosmosAnteHandler` should not capture `feemarketParams` at construction time. Instead, `NewDynamicFeeChecker` should accept a `FeeMarketKeeper` interface and read the current base fee from the store on every invocation, mirroring the pattern used in `newEthAnteHandler` / `EVMBlockConfig`. Alternatively, the ante handler constructor should be rebuilt per-block (as is done for the Ethereum path) so the snapshot is always fresh.

### Proof of Concept

1. Start a chain with default genesis base fee `1_000_000_000` (1 Gwei).
2. Fill every block to `2 × gasTarget` for ~100 blocks; the live base fee rises to roughly `1_000_000_000 × (1.125)^100 ≈ 1.7 × 10^10` (17 Gwei).
3. Construct a Cosmos SDK `Tx` with `ExtensionOptionDynamicFeeTx{MaxPriorityPrice: 0}` and `fee = 1_000_000_000 × gasLimit` (1 Gwei × gas).
4. Submit via `BroadcastTx`. `newCosmosAnteHandler` invokes `NewDynamicFeeChecker`, which reads `feemarketParams.BaseFee = 1_000_000_000` (genesis snapshot). The check `feeCap >= baseFee` passes (1 Gwei ≥ 1 Gwei).
5. The transaction is included in the block and committed. The fee collector receives `1 Gwei × gas` instead of the required `17 Gwei × gas` — a ~17× underpayment — with no error.

	baseFee := k.CalculateBaseFee(ctx)

	if baseFee == nil {

	}

	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),

// EVMBlockConfig creates the EVMBlockConfig based on current state
func (k *Keeper) EVMBlockConfig(ctx sdk.Context, chainID *big.Int) (*EVMBlockConfig, error) {
	objStore := ctx.ObjectStore(k.objectKey)
	v := objStore.Get(types.KeyPrefixObjectParams)
	if v != nil {
		return v.(*EVMBlockConfig), nil
	}

	params := k.GetParams(ctx)
	ethCfg := params.ChainConfig.EthereumConfig(chainID)

	feemarketParams := k.feeMarketKeeper.GetParams(ctx)

	// get the coinbase address from the block proposer
	coinbase, err := k.GetCoinbaseAddress(ctx)
	if err != nil {
		return nil, errorsmod.Wrap(err, "failed to obtain coinbase address")
	}

	var baseFee *big.Int
	if types.IsLondon(ethCfg, ctx.BlockHeight()) {
		baseFee = feemarketParams.GetBaseFee()
		// should not be nil if london hardfork enabled
		if baseFee == nil {
			baseFee = new(big.Int)
		}
	}
	time := ctx.BlockHeader().Time
	var blockTime uint64
	if !time.IsZero() {
		blockTime, err = ethermint.SafeUint64(time.Unix())
		if err != nil {
			return nil, err
		}
	}
	blockNumber := big.NewInt(ctx.BlockHeight())
	rules := ethCfg.Rules(blockNumber, ethCfg.MergeNetsplitBlock != nil, blockTime)

	// Build the default precompile set once per block.
	contracts := make(map[common.Address]vm.PrecompiledContract)
	for addr, c := range vm.DefaultPrecompiles(rules) {
		contracts[addr] = c
	}

	var zero common.Hash
	cfg := &EVMBlockConfig{
		Params:             params,
		FeeMarketParams:    feemarketParams,
		ChainConfig:        ethCfg,
		CoinBase:           coinbase,
		BaseFee:            baseFee,
		Difficulty:         new(big.Int),
		Random:             &zero,
		BlobBaseFee:        new(big.Int),
		BlockNumber:        blockNumber,
		BlockTime:          blockTime,
		Rules:              rules,
		DefaultPrecompiles: contracts,
	}
	objStore.Set(types.KeyPrefixObjectParams, cfg)
	return cfg, nil
}
