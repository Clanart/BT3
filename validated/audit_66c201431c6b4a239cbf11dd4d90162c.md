### Title
Missing `global_roots` consistency check across chunked RPC proof fetches silently produces a cross-state Patricia trie input to the transaction prover — (`crates/starknet_transaction_prover/src/running/storage_proofs.rs`)

---

### Summary

`merge_storage_proofs` unconditionally adopts `global_roots` from the **first** chunk response and never asserts that every subsequent chunk returned the same roots. When a large query is split into sequential RPC calls and any chunk is served by a backend at a different sync height (or returns stale data), the merged `RpcStorageProof` silently contains Patricia trie nodes rooted at two different state roots. The `FactsDb` built from this proof feeds wrong `previous_root` / `updated_root` values into the SNOS proof input, producing a proof for a wrong state transition.

---

### Finding Description

`fetch_proofs` splits any query exceeding `MAX_KEYS_PER_REQUEST = 100` keys into sequential sub-requests:

```rust
// storage_proofs.rs:281-288
let chunks = split_query(query, MAX_KEYS_PER_REQUEST);
let mut proofs = Vec::with_capacity(chunks.len());
for chunk in &chunks {
    proofs.push(self.fetch_single_proof(block_number, chunk).await?);
}
Ok(merge_storage_proofs(proofs, &chunks, query))
``` [1](#0-0) 

`merge_storage_proofs` then blindly takes `global_roots` from `proofs[0]` and unions all trie nodes from every chunk without any cross-chunk root consistency check:

```rust
// storage_proofs.rs:125
let global_roots = proofs[0].global_roots.clone();
// ... all nodes from proofs[1..] are merged in without checking their global_roots
``` [2](#0-1) 

`GlobalRoots` carries three fields: `contracts_tree_root`, `classes_tree_root`, and `block_hash`. If chunk 2 is served by a load-balanced backend that has advanced to block N+1 while chunk 1 was served at block N, the merged proof will contain:

- `global_roots` anchored at block N's state root (from chunk 1)
- Patricia trie nodes from block N+1's trie (from chunk 2)

These nodes have different hashes than the nodes expected under block N's root. The downstream `create_facts_db_from_storage_proof` inserts all of them into the same `FactsDb` without any root-consistency validation: [3](#0-2) 

`commit_state_diff` then initialises the Patricia committer with `contracts_trie_root_hash` taken from the (chunk-1) `global_roots`:

```rust
// storage_proofs.rs:328-329
let contracts_trie_root_hash = HashOutput(rpc_proof.global_roots.contracts_tree_root);
let classes_trie_root_hash   = HashOutput(rpc_proof.global_roots.classes_tree_root);
``` [4](#0-3) 

The committer traverses the trie starting from block N's root but finds nodes belonging to block N+1's trie. The `add_dummy_nodes_for_orphan_hashes` mechanism silently fills in missing nodes with zero-hash dummies rather than erroring: [5](#0-4) 

This causes `commit_state_diff` to return an incorrect `new_roots`, which propagates into `CommitmentInfo.previous_root` and `CommitmentInfo.updated_root` for both the contracts trie and the classes trie — the exact values the SNOS proof system uses to verify the state transition.

The existing test suite (`assert_split_merge_identity`) only verifies structural round-trip correctness using a single deterministic `global_roots` fixture; it never tests the case where chunks return differing roots: [6](#0-5) 

---

### Impact Explanation

The corrupted `CommitmentInfo` (wrong `previous_root` / `updated_root`) is fed directly into the SNOS proof input. The proof is generated for a state transition that never occurred on-chain. This is a wrong state root in proof inputs/outputs, matching the audit pivot:

> *"transaction prover storage proofs must agree on block number, state root, class hash, storage key, proof facts hash, L1 event identity, and pending markers."*

Impact category: **Critical — Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input** (wrong state root in SNOS input/output).

---

### Likelihood Explanation

The trigger requires >100 keys in a single proof request (realistic for blocks with many contract reads) and an RPC endpoint that is either:

1. A load balancer routing sequential requests to backends at different sync heights (common in production), or
2. A node that has advanced to a new block between the first and second sequential chunk fetch.

The sequential fetch loop (`for chunk in &chunks`) introduces a real time gap between requests. The TODO comment `// TODO(Aviv): Consider fetching chunks in parallel with try_join_all` confirms this is a known async gap. No privileged access is required; any sufficiently large block naturally triggers the chunking path.

---

### Recommendation

Add a consistency assertion inside `merge_storage_proofs` before accepting any chunk's nodes:

```rust
pub(crate) fn merge_storage_proofs(
    proofs: Vec<RpcStorageProof>,
    split_queries: &[RpcStorageProofsQuery],
    original_query: &RpcStorageProofsQuery,
) -> Result<RpcStorageProof, ProofProviderError> {   // change return type
    assert_eq!(proofs.len(), split_queries.len(), "proofs/queries length mismatch");
    assert!(!proofs.is_empty(), "cannot merge zero proofs");

    let global_roots = proofs[0].global_roots.clone();

    // NEW: validate all chunks agree on the same state root
    for (i, proof) in proofs.iter().enumerate().skip(1) {
        if proof.global_roots != global_roots {
            return Err(ProofProviderError::InconsistentGlobalRoots {
                chunk: i,
                expected: global_roots.clone(),
                got: proof.global_roots.clone(),
            });
        }
    }
    // ... rest unchanged
}
```

Propagate the `Result` through `fetch_proofs` and `get_storage_proofs`. This converts a silent data-corruption path into a hard error that can be retried with a consistent RPC endpoint.

---

### Proof of Concept

1. A block has 150 accessed keys (e.g. 80 contract addresses + 70 storage slots), triggering the chunking path in `fetch_proofs`.
2. The RPC endpoint is a load balancer. Chunk 1 (keys 1–100) is served by backend A at block N; `global_roots` = `{contracts_root: R_N, classes_root: C_N, block_hash: H_N}`.
3. Between the two sequential `fetch_single_proof` calls, backend B advances to block N+1. Chunk 2 (keys 101–150) is served by backend B; `global_roots` = `{contracts_root: R_{N+1}, classes_root: C_{N+1}, block_hash: H_{N+1}}`.
4. `merge_storage_proofs` sets `global_roots = proofs[0].global_roots` (block N roots) and unions all trie nodes from both chunks without error.
5. `create_facts_db_from_storage_proof` inserts nodes from both block N and block N+1 into the same `FactsDb`.
6. `commit_state_diff` initialises the Patricia committer with `contracts_trie_root_hash = R_N` but traverses nodes that belong to `R_{N+1}`; orphan hashes are silently replaced with zero-hash dummies.
7. The committer returns `new_roots` that does not correspond to any real on-chain state.
8. `CommitmentInfo.previous_root = R_N` and `CommitmentInfo.updated_root = <wrong>` are written into the SNOS proof input.
9. The generated proof attests to a state transition `R_N → <wrong>` that never occurred, and will be rejected by the L1 verifier — or, if the wrong root happens to collide with a real root, accepted for the wrong state.

pub(crate) fn merge_storage_proofs(
    proofs: Vec<RpcStorageProof>,
    split_queries: &[RpcStorageProofsQuery],
    original_query: &RpcStorageProofsQuery,
) -> RpcStorageProof {
    assert_eq!(proofs.len(), split_queries.len(), "proofs/queries length mismatch");
    assert!(!proofs.is_empty(), "cannot merge zero proofs");

    let global_roots = proofs[0].global_roots.clone();
    let mut classes_proof: IndexMap<Felt, MerkleNode> = IndexMap::default();
    let mut contracts_proof =
        ContractsProof { nodes: IndexMap::default(), contract_leaves_data: Vec::new() };

    let addr_to_idx: HashMap<Felt, usize> = original_query
        .contract_storage_keys
        .iter()
        .enumerate()
        .map(|(i, csk)| (csk.contract_address, i))
        .collect();
    let mut contracts_storage_proofs: Vec<IndexMap<Felt, MerkleNode>> =
        vec![IndexMap::default(); original_query.contract_storage_keys.len()];

    for (chunk_query, proof) in split_queries.iter().zip(proofs) {
        classes_proof.extend(proof.classes_proof);
        contracts_proof.nodes.extend(proof.contracts_proof.nodes);
        contracts_proof.contract_leaves_data.extend(proof.contracts_proof.contract_leaves_data);
        for (csk, storage_proof) in
            chunk_query.contract_storage_keys.iter().zip(proof.contracts_storage_proofs)
        {
            let idx = addr_to_idx
                .get(&csk.contract_address)
                .expect("chunk address not in original query");
            contracts_storage_proofs[*idx].extend(storage_proof);
        }
    }

    RpcStorageProof { classes_proof, contracts_proof, contracts_storage_proofs, global_roots }
}

pub(crate) fn create_facts_db_from_storage_proof(
    rpc_proof: &RpcStorageProof,
    query: &RpcStorageProofsQuery,
    initial_reads: &StateMaps,
) -> Result<FactsDb<MapStorage>, ProofProviderError> {
    let mut db_map: DbHashMap = DbHashMap::new();

    insert_inner_nodes(&mut db_map, &rpc_proof.contracts_proof.nodes)?;
    insert_inner_nodes(&mut db_map, &rpc_proof.classes_proof)?;
    for storage_proof in &rpc_proof.contracts_storage_proofs {
        insert_inner_nodes(&mut db_map, storage_proof)?;
    }

    insert_contract_leaves(&mut db_map, rpc_proof, query)?;
    insert_storage_leaves(&mut db_map, initial_reads)?;
    insert_compiled_class_leaves(&mut db_map, initial_reads)?;

    // Add dummy nodes for orphan hashes (sibling hashes without preimages).
    add_dummy_nodes_for_orphan_hashes(&mut db_map, &rpc_proof.contracts_proof.nodes)?;
    add_dummy_nodes_for_orphan_hashes(&mut db_map, &rpc_proof.classes_proof)?;
    for storage_proof in &rpc_proof.contracts_storage_proofs {
        add_dummy_nodes_for_orphan_hashes(&mut db_map, storage_proof)?;
    }

    Ok(FactsDb::new(MapStorage(db_map)))
}

fn add_dummy_nodes_for_orphan_hashes(
    db_map: &mut DbHashMap,
    nodes: &IndexMap<Felt, MerkleNode, impl BuildHasher>,
) -> Result<(), ProofProviderError> {
    // Build set of hashes that have preimages in current proof batch.
    let has_preimage: HashSet<&Felt> = nodes.keys().collect();

    // Create dummy binary node value (both children point to zero hash).
    let dummy_hash = HashOutput(Felt::ZERO);
    let dummy_binary = FactDbFilledNode::<StarknetStorageValue>(FilledNode {
        hash: dummy_hash,
        data: NodeData::Binary(BinaryData { left_data: dummy_hash, right_data: dummy_hash }),
    });
    let dummy_value = dummy_binary.serialize()?;

    // Insert dummy nodes for orphan child hashes.
    for (_, node) in nodes {
        match node {
            MerkleNode::BinaryNode(bn) => {
                add_dummy_node_for_orphan_child(db_map, &bn.left, &has_preimage, &dummy_value);
                add_dummy_node_for_orphan_child(db_map, &bn.right, &has_preimage, &dummy_value);
            }
            MerkleNode::EdgeNode(en) => {
                add_dummy_node_for_orphan_child(db_map, &en.child, &has_preimage, &dummy_value);
            }
        }
    }

    Ok(())
