### Title
`MerkleBlob::batch_insert` Bypasses Duplicate-Key Guard, Silently Corrupting the DataLayer Merkle Tree Root - (File: `crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::batch_insert` skips the duplicate-key and duplicate-hash existence checks that `MerkleBlob::insert` enforces. When a batch contains a key already present in the tree (or a duplicate key within the batch itself), the function silently writes a second leaf node for the same key, producing a structurally invalid tree with a corrupted Merkle root. Proofs of inclusion derived from the corrupted tree are unsound.

---

### Finding Description

`MerkleBlob::insert` guards against duplicate keys and hashes before writing any node:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 369-374
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`batch_insert` delegates to `insert` only for the first two items when the tree has ≤ 1 existing leaf. For every subsequent item it calls `insert_entry_to_blob` directly, with no existence check at all:

```rust
// lines 587-602
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ... Node::Leaf(LeafNode { hash, key, value, ... }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;   // ← no key/hash guard
    indexes.push(new_leaf_index);
}
``` [2](#0-1) 

`insert_entry_to_blob` unconditionally writes the block and registers the leaf in the cache: [3](#0-2) 

The Python binding `py_batch_insert` is the public entry point and passes caller-supplied data straight through: [4](#0-3) 

The Python API stub confirms `batch_insert` is a first-class public method: [5](#0-4) 

---

### Impact Explanation

When a duplicate key is inserted via `batch_insert`:

1. Two `LeafNode` entries with the same `KeyId` exist in the blob.
2. The `block_status_cache` records both, so `get_keys_values` returns an inconsistent map.
3. Internal-node hashes are computed over the corrupted subtree, producing a wrong Merkle root.
4. `get_proof_of_inclusion` generates proofs against the wrong root; a verifier using the pre-corruption root will reject them, and a verifier using the post-corruption root will accept forged state.

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic … corrupts tree roots, or lets untrusted input prove invalid state."*

---

### Likelihood Explanation

The `batch_insert` Python binding is the performance-optimised path for bulk DataLayer writes. Any caller that does not deduplicate its input before calling `batch_insert` — including a DataLayer node processing a user-submitted batch that contains a key already committed to the store — will silently corrupt the tree. No privilege is required; the function is unconditionally callable from Python.

---

### Recommendation

Add the same guards that `insert` uses at the top of the fast-path loop in `batch_insert`:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // ... existing insert_entry_to_blob call
}
```

Alternatively, refactor `batch_insert` to call the validated `insert` path for every item, accepting the small per-item overhead in exchange for correctness.

### Proof of Concept

```python
from chia_rs.datalayer import MerkleBlob, KeyId, ValueId
from chia_rs.sized_ints import int64
import hashlib

def h(n: int) -> bytes:
    return hashlib.sha256(n.to_bytes(8, "little")).digest()

blob = MerkleBlob(bytearray())

# Seed the tree with 3 leaves so leaf_count > 1 (bypasses the guarded path)
for i in range(3):
    blob.insert(KeyId(int64(i)), ValueId(int64(i)), h(i))

root_before = blob.get_root_hash()

# batch_insert with a key already present (key=0) — no error raised
blob.batch_insert(
    [(KeyId(int64(0)), ValueId(int64(99)))],  # duplicate key 0
    [h(99)],
)

root_after = blob.get_root_hash()

# Root has changed silently; two leaves with key=0 now exist
assert root_before != root_after, "tree root corrupted without error"

# Proof generated against corrupted root is unsound
proof = blob.get_proof_of_inclusion(KeyId(int64(0)))
# proof.valid() may return True against the wrong root

    fn insert_entry_to_blob(&mut self, index: TreeIndex, block: &Block) -> Result<(), Error> {
        let new_block_bytes = block.to_bytes()?;
        let extend_index = self.extend_index();
        match index.cmp(&extend_index) {
            Ordering::Greater => return Err(Error::BlockIndexOutOfBounds(index)),
            Ordering::Equal => self.blob.extend_from_slice(&new_block_bytes),
            Ordering::Less => {
                self.blob[block_range(index)].copy_from_slice(&new_block_bytes);
            }
        }

        match block.node {
            Node::Leaf(leaf) => self.block_status_cache.add_leaf(index, leaf),
            Node::Internal(..) => self.block_status_cache.add_internal(index),
        }

        Ok(())
    }
