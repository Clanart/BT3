### Title
Stale L1 Gas Prices Silently Committed to Block Hash After Sequencer Downtime via Unbounded `previous_block_info` Fallback — (`crates/apollo_consensus_orchestrator/src/utils.rs`)

---

### Summary

When the sequencer is offline for longer than `max_time_gap_seconds` (default 900 s), `L1GasPriceProvider::get_price_info` returns `StaleL1GasPricesError`. The fallback path in `get_l1_prices_in_fri_and_wei_and_conversion_rate` silently substitutes the gas prices from the **previous committed block** (`previous_block_info`) with no bound on how old those prices are. Because the validator runs the identical fallback against the same `previous_block_info`, the `within_margin` check in `is_block_info_valid` passes, and the stale prices are accepted, committed into `PartialBlockHashComponents`, hashed into the block hash, and used for all fee calculations in that block.

---

### Finding Description

**Step 1 – Staleness guard in the provider.**

`L1GasPriceProvider::get_price_info` rejects any query whose timestamp exceeds the newest scraped sample by more than `max_time_gap_seconds` (default 900 s):

```rust
if timestamp.0 > (*last_timestamp + self.config.max_time_gap_seconds) {
    return Err(L1GasPriceProviderError::StaleL1GasPricesError { … });
}
``` [1](#0-0) 

**Step 2 – Fallback silently uses arbitrarily old prices.**

`get_l1_prices_in_fri_and_wei_and_conversion_rate` catches both `StaleL1GasPricesError` and `MissingDataError` and falls back to `previous_block_info` **without any staleness check on that struct**:

```rust
if let Some(block_info) = previous_block_info {
    …
    return (prev_l1_gas_price, prev_l1_gas_price_wei, eth_to_fri_rate);
}
``` [2](#0-1) 

`previous_block_info` carries only the gas prices and timestamp of the last committed block; there is no field recording when it was last refreshed, and no caller checks its age before using it. [3](#0-2) 

**Step 3 – Stale prices flow into the proposal and block hash.**

`initiate_build` passes the result directly into `ProposalInit`:

```rust
let (l1_prices_fri, l1_prices_wei) = get_l1_prices_in_fri_and_wei(
    args.deps.l1_gas_price_provider.clone(),
    timestamp,
    args.previous_block_info.as_ref(),
    &args.gas_price_params,
).await;
let init = ProposalInit { …, l1_gas_price_fri: l1_prices_fri.l1_gas_price, … };
``` [4](#0-3) 

`PartialBlockHashComponents::new` then copies those prices verbatim into the structure that feeds `calculate_block_hash`:

```rust
l1_gas_price: block_info.gas_prices.l1_gas_price_per_token(),
l1_data_gas_price: block_info.gas_prices.l1_data_gas_price_per_token(),
``` [5](#0-4) 

`calculate_block_hash` chains those prices (via `gas_prices_to_hash`) into the Poseidon hash that becomes the canonical block hash: [6](#0-5) 

**Step 4 – Validator accepts the stale prices.**

`is_block_info_valid` calls the same `get_l1_prices_in_fri_and_wei` with `init_proposed.timestamp` and the same `previous_block_info`. After the downtime both nodes have identical stale fallback prices, so `within_margin` passes:

```rust
let (l1_gas_prices_fri, _) = get_l1_prices_in_fri_and_wei(
    l1_gas_price_provider,
    init_proposed.timestamp,          // current wall-clock time
    block_info_validation.previous_block_info.as_ref(),
    gas_price_params,
).await;
…
if !(within_margin(l1_gas_price_fri_proposed, l1_gas_price_fri, …) && …) {
    return Err(…);
}
``` [7](#0-6) 

The block is committed with stale gas prices that no validator rejects.

---

### Impact Explanation

The stale `l1_gas_price` and `l1_data_gas_price` values are embedded in the committed block hash and used by the blockifier to compute every transaction fee in that block. If L1 gas prices moved significantly during the downtime (e.g., a spike or crash), every transaction in the first post-recovery block pays fees computed from pre-downtime prices. This is a direct, on-chain, irreversible incorrect fee/gas accounting effect with economic impact — matching the Critical scope item: *"Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact."*

---

### Likelihood Explanation

The default `max_time_gap_seconds` is 900 s (15 minutes). [8](#0-7) 

Sequencer downtime exceeding 15 minutes is a realistic operational event (restart, network partition, L1 scraper crash). No privileged access is required; the condition is met whenever the scraper falls behind the wall clock by more than the configured gap.

---

### Recommendation

Add a staleness bound to the `previous_block_info` fallback. Before returning the cached prices, compare `previous_block_info.timestamp` against the current wall-clock time. If the gap exceeds a configurable threshold (e.g., `max_time_gap_seconds`), either refuse to build a block until fresh prices are available, or surface an explicit warning and apply a conservative price adjustment. This mirrors the intent of the existing `max_time_gap_seconds` guard in `get_price_info` and closes the bypass introduced by the silent fallback.

---

### Proof of Concept

1. Sequencer is running normally; `previous_block_info` holds gas prices from block N (timestamp T₀).
2. Sequencer goes offline at T₀ + 1 s; L1 gas price scraper stops.
3. L1 gas price spikes 10× during the outage.
4. Sequencer restarts at T₀ + 1800 s (30 min later).
5. Proposer calls `initiate_build`; `get_proposal_timestamp` returns T₀ + 1800.
6. `get_price_info(T₀ + 1800)` returns `StaleL1GasPricesError` because `last_timestamp ≈ T₀` and `1800 > 900`.
7. Fallback returns `previous_block_info` prices (pre-spike, 30 min old).
8. `ProposalInit` is broadcast with the pre-spike prices.
9. Validator runs `is_block_info_valid`; its own `get_l1_prices_in_fri_and_wei` call hits the same stale fallback and returns identical prices; `within_margin` passes.
10. Block N+1 is committed with pre-spike L1 gas prices baked into its block hash.
11. All transactions in block N+1 are charged fees 10× too low, causing a direct economic loss to the protocol proportional to the L1 gas price delta and the block's total gas consumption.

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
