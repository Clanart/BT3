### Title
Native Fee (STRK) Permanently Locked in Starknet OmniBridge — No Withdrawal Mechanism (File: `starknet/src/omni_bridge.cairo`)

---

### Summary

The Starknet `OmniBridge` contract collects `native_fee` in STRK tokens from every user who calls `init_transfer` with `native_fee > 0`, transferring those tokens directly into the contract itself. The contract exposes no function to withdraw, distribute, or claim these accumulated STRK fees. They are permanently locked with no recovery path in the current code.

---

### Finding Description

In `starknet/src/omni_bridge.cairo`, the `init_transfer` function accepts a `native_fee` parameter denominated in STRK (the Starknet native gas token). When `native_fee > 0`, the contract pulls STRK from the caller into itself:

```cairo
// starknet/src/omni_bridge.cairo  lines 309-313
if native_fee > 0 {
    let native_token = self.strk_token_address.read();
    let success = IERC20Dispatcher { contract_address: native_token }
        .transfer_from(caller, get_contract_address(), native_fee.into());
    assert(success, 'ERR_FEE_TRANSFER_FAILED');
}
```

The destination is `get_contract_address()` — the bridge contract itself. The full public interface of the contract is:

- `log_metadata`, `deploy_token`, `fin_transfer`, `init_transfer`
- `upgrade_token`, `set_pause_flags`, `pause_all`
- `get_token_address`, `is_bridge_token`, `is_transfer_finalised`
- `upgrade` (admin-only contract upgrade)

None of these functions withdraw, transfer, or distribute the accumulated STRK balance. There is no `withdraw_fees`, `claim_fee`, or equivalent function anywhere in the contract.

By contrast, the NEAR `omni-bridge` contract imports `ClaimFeeArgs` and has a dedicated fee-claiming flow, confirming that native-fee distribution to relayers is an intended protocol feature — but it is entirely absent on Starknet.

---

### Impact Explanation

Every `init_transfer` call on Starknet that includes a non-zero `native_fee` permanently deposits STRK into the bridge contract with no recovery path. Over time, the total STRK locked grows proportionally to bridge usage. Relayers who are supposed to receive these fees as compensation for gas costs cannot claim them. The funds are irrecoverably locked in the current contract code.

This matches the allowed impact: **Critical — Permanent freezing / irrecoverable lock of user or protocol funds in bridge flows.**

---

### Likelihood Explanation

`native_fee` is a standard parameter in the bridge's transfer flow, present across all chains (EVM, NEAR, Solana, Starknet). Any user bridging tokens out of Starknet who sets `native_fee > 0` — which is the normal operating mode when a relayer is expected to submit the proof on the destination chain — triggers this lock. No special attacker action is required; ordinary bridge usage is sufficient.

---

### Recommendation

Add an admin-restricted (e.g., `DEFAULT_ADMIN_ROLE`) function to withdraw accumulated STRK fees to a designated treasury or fee-distributor address:

```cairo
fn withdraw_native_fees(ref self: ContractState, recipient: ContractAddress, amount: u256) {
    self.accesscontrol.assert_only_role(DEFAULT_ADMIN_ROLE);
    let native_token = self.strk_token_address.read();
    let success = IERC20Dispatcher { contract_address: native_token }
        .transfer(recipient, amount);
    assert(success, 'ERR_FEE_WITHDRAWAL_FAILED');
}
```

Alternatively, route `native_fee` directly to a relayer or treasury address at collection time rather than accumulating it in the contract.

---

### Proof of Concept

1. User calls `init_transfer` on the Starknet `OmniBridge` with `native_fee = 1_000_000_000_000_000_000` (1 STRK).
2. The contract executes `transfer_from(caller, get_contract_address(), 1e18)` — 1 STRK enters the contract.
3. The `InitTransfer` event is emitted, logging `native_fee = 1e18`.
4. A relayer observes the event and submits the proof on the destination chain, expecting to be compensated.
5. No function exists in the Starknet contract to send the 1 STRK to the relayer or any treasury.
6. The 1 STRK remains locked in the contract indefinitely.
7. Repeat for every `init_transfer` with `native_fee > 0`; the locked STRK balance grows monotonically with no recovery path. [1](#0-0) [2](#0-1) [3](#0-2)
