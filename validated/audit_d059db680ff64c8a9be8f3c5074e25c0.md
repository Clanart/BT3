### Title
DataLayer `ProofOfInclusion::valid()` Does Not Verify Against an External Root Hash, Enabling Forged Inclusion Proofs - (File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs)

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate only verifies internal hash-chain consistency. Its final check is a tautology: it compares `existing_hash` (which the loop already forced to equal `last.combined_hash`) against `self.root_hash()` (which also returns `last.combined_hash`). The function never verifies the proof against any external trusted root. An attacker who can supply a crafted `ProofOfInclusion` (via the `Streamable`/`from_bytes` interface) can forge a proof of inclusion for any arbitrary key-value pair and pass `valid()`.

### Finding Description

`ProofOfInclusion::valid()` is implemented as follows:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()   // ← tautological
}
```

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← same value the loop just set existing_hash to
    } else {
        self.node_hash
    }
}
```

After the loop completes without returning `false`, `existing_hash` equals `last.combined_hash` (because the loop enforces `calculated_hash == layer.combined_hash` and then sets `existing_hash = calculated_hash`). The final check `existing_hash == self.root_hash()` therefore reduces to `last.combined_hash == last.combined_hash`, which is always `true`.

**Consequence:** `valid()` only checks that the hash chain is self-consistent. It never checks that the chain terminates at a specific, externally-trusted root. An attacker can:

1. Choose any `node_hash` (the leaf they wish to falsely claim is included).
2. Choose arbitrary `other_hash` values for each layer.
3. Compute each `combined_hash` correctly using `calculate_internal_hash`.
4. Serialize the resulting `ProofOfInclusion` via `to_bytes()` / `Streamable`.
5. The forged proof passes `valid()` unconditionally.

The `ProofOfInclusion` struct is `Streamable` and fully deserializable from untrusted bytes via `from_bytes()` / `parse_rust()`, and is exposed through both the Rust API and the Python wheel bindings. Any DataLayer verification path that calls only `proof.valid()` without also asserting `proof.root_hash() == known_on_chain_root` accepts forged proofs.

This is the direct analog of the external report: just as the CircuitBreaker did not account for base-token flows (allowing bypass of the withdrawal cap), `valid()` does not account for the external root (allowing bypass of the inclusion check).

### Impact Explanation

An attacker who can deliver a crafted `ProofOfInclusion` to a DataLayer peer or verifier can prove inclusion of any key-value pair in any DataLayer tree, regardless of the actual committed on-chain root. This lets untrusted input prove invalid state, matching the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

### Likelihood Explanation

`ProofOfInclusion` is a `Streamable` type exposed over the Python wheel and intended to be exchanged between DataLayer peers. Any peer-facing verification code that calls `proof.valid()` without the companion `proof.root_hash() == trusted_root` check is immediately exploitable by any network peer. The misleading name `valid()` makes this misuse highly likely.

### Recommendation

The `valid()` method should accept the trusted root hash as a parameter and verify against it:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    // ... existing chain check ...
    existing_hash == *trusted_root   // compare against external root, not self.root_hash()
}
```

Alternatively, rename the current method to `is_internally_consistent()` and add a separate `verify(trusted_root: &Hash) -> bool` that performs the complete check. Update all call sites accordingly.

### Proof of Concept

```rust
use chia_datalayer::{Hash, Side, ProofOfInclusion, ProofOfInclusionLayer};
use chia_datalayer::calculate_internal_hash;

// Attacker wants to forge inclusion of an arbitrary node_hash
let fake_node_hash: Hash = [0xAA; 32];
let other_hash:     Hash = [0xBB; 32];

// Compute a valid combined_hash for one layer
let combined = calculate_internal_hash(&fake_node_hash, Side::Left, &other_hash);

let forged = ProofOfInclusion {
    node_hash: fake_node_hash,
    layers: vec![ProofOfInclusionLayer {
        other_hash_side: Side::Left,
        other_hash,
        combined_hash: combined,   // correctly computed → loop passes
    }],
};

// valid() returns true even though fake_node_hash is not in any real tree
assert!(forged.valid());
// root_hash() returns `combined`, which is attacker-controlled
assert_eq!(forged.root_hash(), combined);
```

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
