### Title
Stranded Router ETH from Prior Multicall Silently Subsidizes Subsequent User's WETH Input Payment — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

---

### Summary

`PeripheryPayments.pay()` reads `address(this).balance` to decide how much native ETH to wrap as WETH. Because this balance is the *entire* router's ETH balance — not scoped to the current caller's `msg.value` — any ETH left on the router from a prior payable call is silently consumed to fund a later user's WETH input. The prior user loses that ETH; the later user pays proportionally less than owed.

---

### Finding Description

`pay()` contains the following branch for WETH payments: [1](#0-0) 

```solidity
} else if (token == WETH) {
    uint256 nativeBalance = address(this).balance;   // ← entire router balance
    if (nativeBalance >= value) {
        IWETH9(WETH).deposit{value: value}();
        IERC20(WETH).safeTransfer(recipient, value);
    } else if (nativeBalance > 0) {
        IWETH9(WETH).deposit{value: nativeBalance}();
        IERC20(WETH).safeTransfer(recipient, nativeBalance);
        IERC20(WETH).safeTransferFrom(payer, recipient, value - nativeBalance);
    } else {
        IERC20(WETH).safeTransferFrom(payer, recipient, value);
    }
```

`nativeBalance` is `address(this).balance`, which is **global router state**, not the current transaction's `msg.value`. ETH can be stranded on the router whenever a caller sends more `msg.value` than a swap consumes and omits the `refundETH()` cleanup step.

`multicall` is `payable` and loops via `delegatecall`: [2](#0-1) 

No automatic refund is issued after any swap function. The `refundETH()` helper exists but is opt-in: [3](#0-2) 

The `receive()` guard only blocks *direct* ETH pushes from non-WETH addresses; it does **not** prevent ETH from accumulating via `msg.value` in any payable entry point: [4](#0-3) 

---

### Impact Explanation

**Direct loss of user principal.** A victim's stranded ETH is wrapped into WETH and transferred to a pool on behalf of a different user. The victim receives nothing in return; the beneficiary pays less than the full swap input amount. This is a silent, unrecoverable fund transfer between unrelated users.

---

### Likelihood Explanation

Stranding ETH is easy and common:
- Any user who sends `msg.value > amountIn` to `exactInputSingle` / `exactInput` / `multicall` without appending a `refundETH()` call leaves the excess on the router.
- An attacker can monitor the router's ETH balance on-chain and immediately follow with a WETH `exactInputSingle` to drain it.
- No privileged access, no malicious pool, no non-standard token is required.

---

### Recommendation

Scope the native ETH available for WETH wrapping to the **current transaction's** `msg.value`, not the global contract balance. One approach: track `msg.value` consumed in transient storage per call and use only that amount in `pay()`. Alternatively, enforce that `address(this).balance` at the start of each top-level entry equals `msg.value` (i.e., require no pre-existing ETH balance), or automatically refund any residual ETH at the end of every swap entry point.

### Proof of Concept

```
// Foundry test sketch
function test_strandedEthSubsidizesWethSwap() public {
    // Step 1: Victim sends 2 ETH for a 1 ETH WETH exactInputSingle, forgets refundETH
    vm.prank(victim);
    router.exactInputSingle{value: 2 ether}(
        ExactInputSingleParams({tokenIn: WETH, amountIn: 1 ether, ...})
    );
    // 1 ETH is now stranded on the router

    assertEq(address(router).balance, 1 ether);

    // Step 2: Attacker calls exactInputSingle with WETH input, sends 0 ETH
    uint256 attackerWethBefore = IERC20(WETH).balanceOf(attacker);
    vm.prank(attacker);
    router.exactInputSingle{value: 0}(
        ExactInputSingleParams({tokenIn: WETH, amountIn: 1 ether, ...})
    );

    // Attacker's WETH balance unchanged — stranded ETH paid for the swap
    assertEq(IERC20(WETH).balanceOf(attacker), attackerWethBefore);
    // Router ETH drained
    assertEq(address(router).balance, 0);
    // Victim lost 1 ETH with no recourse
}
```

In `pay()`, when `nativeBalance = 1 ETH` and `value = 1 ETH`, the `nativeBalance >= value` branch fires, wraps the stranded ETH, and transfers it to the pool — pulling **zero** from the attacker's wallet.
