### Title
`replace_class` Syscall Accepts Undeclared Class Hash Without Validation, Enabling Permanent Fund Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in the StarkNet OS Cairo program processes the `replace_class` syscall without verifying that the supplied class hash has been declared (i.e., exists in `contract_class_changes`). An unprivileged contract can call `replace_class` with an arbitrary, undeclared class hash. The OS will update the contract's state entry unconditionally, permanently bricking the contract and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class`, the OS reads the requested class hash directly from the syscall request and updates the contract's `StateEntry` without checking whether that class hash is present in `contract_class_changes` (the dict that tracks declared classes):

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges the missing check. The OS Cairo code is the authoritative constraint system that gets proven — if it does not enforce the class-existence check, a valid STARK proof can be generated for a block containing a `replace_class` call with an undeclared class hash.

The analog to the external report is direct: just as the bridge's `withdrawTokens` unconditionally whitelists a newly deployed token without checking `_whiteListEnabled`, the OS's `execute_replace_class` unconditionally updates a contract's class hash without checking whether the class is declared. In both cases, a guard condition that should gate a state mutation is absent.

For context, a class is "declared" when it has an entry in `contract_class_changes` (mapping class hash → compiled class hash), enforced by `dict_update` with `prev_value=0` in `execute_declare_transaction`: [2](#0-1) 

`execute_replace_class` performs no equivalent lookup into `contract_class_changes` before writing the new class hash into `contract_state_changes`.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value, every subsequent call to that contract will fail at the class-resolution step (the class bytecode does not exist). If the contract holds ERC-20 tokens, ETH, NFTs, or any other assets, those assets become permanently inaccessible. The state commitment computed by `compute_contract_state_commitment` will include the corrupted class hash: [3](#0-2) 

The resulting proof is valid according to the Cairo constraints, so the L1 verifier accepts it. The on-chain state is permanently updated with the bricked contract entry. There is no recovery path.

---

### Likelihood Explanation

**High.** The `replace_class` syscall is callable by any contract on behalf of itself — no privileged role is required. An attacker needs only to:

1. Deploy a contract (or use an existing self-owned contract).
2. Have that contract invoke `replace_class` with any felt value that is not a declared class hash (e.g., `0xdeadbeef`, or even `0` which equals `UNINITIALIZED_CLASS_HASH`). [4](#0-3) 

The OS Cairo code imposes no barrier. A sequencer (even a non-malicious one following the blockifier) that does not independently enforce this check would include such a transaction, and the resulting proof would be accepted by the L1 verifier.

---

### Recommendation

Before updating `contract_state_changes`, verify that `request.class_hash` has a non-zero entry in `contract_class_changes`. Concretely, perform a `dict_read` on `contract_class_changes` keyed by `request.class_hash` and assert the result is non-zero (i.e., the class has been declared). This mirrors the existing guard in `deploy_contract` which asserts `state_entry.class_hash = UNINITIALIZED_CLASS_HASH` before writing: [5](#0-4) 

The fix for `execute_replace_class` should be symmetric: assert that the new class hash IS initialized (declared) before accepting it.

---

### Proof of Concept

```
1. Attacker deploys ContractA (holds 1000 ETH worth of tokens).
2. ContractA contains a function `brick_self()` that calls:
       replace_class(0xdeadbeef)   // 0xdeadbeef is never declared
3. Attacker calls `brick_self()`.
4. The OS processes the `replace_class` syscall in `execute_replace_class`:
   - `request.class_hash = 0xdeadbeef`
   - No check against `contract_class_changes` is performed (TODO comment, line 898)
   - `dict_update` writes `StateEntry(class_hash=0xdeadbeef, ...)` for ContractA
5. The block is proven; the L1 verifier accepts the proof.
6. ContractA's on-chain class hash is now `0xdeadbeef`.
7. Any subsequent call to ContractA fails — the class does not exist.
8. All 1000 ETH worth of tokens in ContractA are permanently frozen.
``` [6](#0-5)

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
