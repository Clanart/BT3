### Title
Permanent Lock of Source-Chain Funds via Zero-Address Recipient in `fin_transfer` — (`starknet/src/omni_bridge.cairo`)

---

### Summary

`OmniBridge::fin_transfer` marks the destination nonce as finalised **before** calling `BridgeToken::mint`. Because OpenZeppelin Cairo ERC20's internal `mint` panics when `recipient.is_zero()`, any signed payload carrying `recipient = ContractAddress(0)` causes the entire transaction to revert — unwinding the nonce write — so the nonce is never consumed, every retry reverts identically, and the user's source-chain funds are permanently locked with no on-chain recovery path.

---

### Finding Description

In `fin_transfer`, the execution order is:

1. Pause check
2. Nonce-already-used check
3. **`_set_transfer_finalised(ref self, payload.destination_nonce)`** ← nonce written to storage
4. Signature verification
5. `IBridgeTokenDispatcher { … }.mint(payload.recipient, payload.amount.into())` [1](#0-0) 

Because Starknet transactions are atomic, a panic inside `mint` rolls back **all** state changes in the call, including the nonce write at step 3. The nonce therefore remains unconsumed after the revert.

`BridgeToken::mint` delegates directly to the OpenZeppelin Cairo ERC20 component's internal `mint`: [2](#0-1) 

OpenZeppelin Cairo ERC20 (≥ 0.14) asserts `!recipient.is_zero()` and panics with `MINT_TO_ZERO` when the recipient is the zero address. There is no zero-address guard anywhere in `fin_transfer` before the dispatcher call, and there is no refund or rescue path in the contract. [3](#0-2) 

The `TransferMessagePayload` struct accepts any `ContractAddress` for `recipient`, including zero: [4](#0-3) 

---

### Impact Explanation

- Source-chain funds (burned or locked on NEAR) are irrecoverable: the destination nonce is never consumed, so the transfer can never be finalised.
- Every retry of the same NEAR-signed payload produces an identical revert.
- No on-chain refund or rescue mechanism exists in the Starknet bridge.

Impact: **Critical — permanent, irrecoverable lock of user funds.**

---

### Likelihood Explanation

The precondition is that NEAR's MPC signs a `fin_transfer` payload whose `recipient` field encodes `ContractAddress(0)`. This can occur via:

- A user supplying `0x0` as their Starknet destination address (a realistic user mistake, since Starknet addresses are opaque hex strings to most users).
- A bug in NEAR-side address derivation or encoding that produces the zero address.

NEAR's `OmniAddress` for Starknet is a raw felt252 value; address 0 is a valid felt252 and is not rejected at the NEAR signing layer. Once signed, the payload is valid and will pass `_verify_borsh_signature` on Starknet. [5](#0-4) 

Likelihood: **Medium** (requires a specific bad input, but the input is user-controlled and the NEAR layer does not filter it).

---

### Recommendation

Add an explicit zero-address guard in `fin_transfer` before dispatching to `mint` or `transfer`:

```cairo
assert(!payload.recipient.is_zero(), 'ERR_ZERO_RECIPIENT');
```

Place this check immediately after signature verification and before the bridge-token branch. This converts an unrecoverable permanent lock into a clean, early revert that leaves the nonce unconsumed and allows the user or protocol to take corrective action (e.g., re-sign with a valid recipient via a NEAR-side governance or refund flow). [6](#0-5) 

---

### Proof of Concept

1. Deploy `BridgeToken` and `OmniBridge` on a Starknet devnet; register the token as a bridge token.
2. Construct a `TransferMessagePayload` with `recipient = ContractAddress(0)` and a valid `destination_nonce`.
3. Sign the borsh-encoded payload with the `omni_bridge_derived_address` key.
4. Call `fin_transfer(sig, payload)`.
5. **Observe**: transaction reverts with `MINT_TO_ZERO` (from OZ ERC20).
6. Call `is_transfer_finalised(destination_nonce)` → returns `false` (nonce not consumed).
7. Retry step 4 → identical revert every time.
8. Confirm no function in the contract allows recovery or refund of the source-chain funds.

        fn fin_transfer(
            ref self: ContractState, signature: Signature, payload: TransferMessagePayload,
        ) {
            assert(!_is_paused(@self, PAUSE_FIN_TRANSFER), 'ERR_FIN_TRANSFER_PAUSED');

            assert(
                !self.is_transfer_finalised(payload.destination_nonce), 'ERR_NONCE_ALREADY_USED',
            );
            _set_transfer_finalised(ref self, payload.destination_nonce);

            _verify_borsh_signature(
                ref self, @payload.to_borsh(self.omni_bridge_chain_id.read()), signature,
            );

            if self.is_bridge_token(payload.token_address) {
                IBridgeTokenDispatcher { contract_address: payload.token_address }
                    .mint(payload.recipient, payload.amount.into());
            } else {
                let success = IERC20Dispatcher { contract_address: payload.token_address }
                    .transfer(payload.recipient, payload.amount.into());
                assert(success, 'ERR_TRANSFER_FAILED');
            }

            self
                .emit(
                    Event::FinTransfer(
                        FinTransfer {
                            origin_chain: payload.origin_chain,
                            origin_nonce: payload.origin_nonce,
                            token_address: payload.token_address,
                            amount: payload.amount,
                            recipient: payload.recipient,
                            fee_recipient: payload.fee_recipient,
                            message: payload.message,
                        },
                    ),
                )
        }

    fn _verify_borsh_signature(
        ref self: ContractState, borsh_bytes: @ByteArray, signature: Signature,
    ) {
        let message_hash_le = compute_keccak_byte_array(borsh_bytes);
        let message_hash = reverse_u256_bytes(message_hash_le);

        let sig = signature_from_vrs(signature.v, signature.r, signature.s);
        verify_eth_signature(message_hash, sig, self.omni_bridge_derived_address.read());
    }
