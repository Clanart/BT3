### Title
`ProofOfInclusion::valid()` Accepts Forged Inclusion Proofs — No External Root Verification - (File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs)

### Summary
The `ProofOfInclusion::valid()` method in the DataLayer Merkle proof implementation only checks internal hash-chain consistency. It never verifies the proof against an external trusted root hash. Because `root_hash()` returns the proof's own last `combined_hash` (attacker-controlled), the final equality check is trivially self-referential and always passes. An attacker can construct a `ProofOfInclusion` with empty `layers` and any arbitrary `node_hash` that passes `valid()`, forging inclusion of any key-value pair in any tree.

### Finding Description

`ProofOfInclusion` is defined as a `Streamable` struct with a `node_hash` and a `Vec<ProofOfInclusionLayer>`: [1](#0-0) 

The `root_hash()` helper returns the last layer's `combined_hash`, or `self.node_hash` when `layers` is empty — both are attacker-supplied fields: [2](#0-1) 

The `valid()` method iterates over layers, checking internal hash consistency, then performs a final check `existing_hash == self.root_hash()`: [3](#0-2) 

**Two independent bypass paths exist:**

**Path 1 — Empty layers:** When `layers` is empty, the loop body never executes. `existing_hash` remains `self.node_hash`, and `self.root_hash()` also returns `self.node_hash`. The final check becomes `self.node_hash == self.node_hash`, which is unconditionally `true`. Any `ProofOfInclusion { node_hash: X, layers: [] }` passes `valid()` for any `X`.

**Path 2 — Non-empty layers (tautological final check):** In the last loop iteration, `existing_hash` is set to `calculated_hash`, which was already asserted equal to `layer.combined_hash`. After the loop, `self.root_hash()` returns `self.layers.last().combined_hash` — the same value. The final check `existing_hash == self.root_hash()` is therefore always `true` if all per-layer checks pass. The function never compares against any externally trusted root; it only verifies that the attacker-supplied chain is internally self-consistent.

The struct is `Streamable` (deserializable from bytes) and fully exposed via Python bindings: [4](#0-3) [5](#0-4) 

The Python stub confirms the constructor accepts arbitrary `node_hash` and `layers`: [6](#0-5) 

The `get_proof_of_inclusion` binding returns a `ProofOfInclusion` that any peer can receive, deserialize, and re-submit with tampered fields: [7](#0-6) 

### Impact Explanation
Any DataLayer consumer that receives a `ProofOfInclusion` from an untrusted peer and calls only `proof.valid()` — without separately asserting `proof.root_hash() == trusted_root` — will accept a forged proof. An attacker can prove that an arbitrary key-value pair is included in any tree root, enabling false state attestation, forged inclusion/exclusion claims, and potential corruption of DataLayer Merkle state. This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

### Likelihood Explanation
The `ProofOfInclusion` API exposes only `valid()` and `root_hash()` as verification primitives. There is no `valid_for_root(expected: Hash) -> bool` method. The naming `valid()` strongly implies complete proof validity, making it likely that callers rely on it exclusively. The struct is `Streamable`, so any network-received proof can be deserialized and submitted. The Python DataLayer layer receives proofs from peers over the network, making the attacker-controlled entry path direct and unprivileged.

### Recommendation
1. Add a `valid_for_root(expected_root: Hash) -> bool` method that checks `self.root_hash() == expected_root` in addition to the internal chain consistency.
2. Rename the existing `valid()` to `internally_consistent()` to prevent callers from mistaking it for a complete security check.
3. Audit all call sites of `valid()` / `py_valid()` to ensure they also verify the root hash against a trusted external value.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion

# Forge inclusion of any node_hash with empty layers
forged = ProofOfInclusion(
    node_hash=bytes(range(32)),   # attacker-chosen hash
    layers=[],
)

assert forged.valid()             # True — no external root checked
assert forged.root_hash() == bytes(range(32))  # root == node_hash (self-referential)

# Also works with a non-empty internally-consistent chain:
# attacker builds layers where each combined_hash = H(prev, side, other)
# valid() returns True regardless of which actual tree root this corresponds to

    pub fn valid(&self) -> bool {
        let mut existing_hash = self.node_hash;

        for layer in &self.layers {
            let calculated_hash = crate::calculate_internal_hash(
                &existing_hash,
                layer.other_hash_side,
                &layer.other_hash,
            );

            if calculated_hash != layer.combined_hash {
                return false;
            }

            existing_hash = calculated_hash;
        }

        existing_hash == self.root_hash()
    }
