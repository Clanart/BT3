### Title
Inverted Comparison Arms in `queued_proposals` Drain Loop Causes Valid Proposals to Be Silently Dropped - (`File: crates/apollo_consensus_orchestrator/src/sequencer_consensus_context.rs`)

### Summary

In `set_height_and_round`, the loop that drains stale entries from `queued_proposals` has its `Less` and `Greater` match arms swapped. The result is the exact structural analog of the APR bug: a comparison whose direction is inverted relative to the intended semantics. Future-round proposals are deleted when they should be kept, and past-round (stale) proposals halt the loop when they should be discarded, preventing the current round's proposal from ever being processed.

### Finding Description

`queued_proposals` is a `BTreeMap<Round, …>` sorted ascending by round. `first_entry()` always returns the entry with the **minimum** key. The loop is intended to:

1. Discard entries whose round is **less than** `current_round` (stale).
2. Extract and process the entry whose round **equals** `current_round`.
3. Stop when the minimum remaining entry is **greater than** `current_round` (all remaining are future rounds).

The actual code at lines 924–934 does the opposite:

```rust
while let Some(entry) = self.queued_proposals.first_entry() {
    match self.current_round.cmp(entry.key()) {
        std::cmp::Ordering::Less => {
            entry.remove();          // ← removes a FUTURE-round proposal
        }
        std::cmp::Ordering::Equal => {
            to_process = Some(entry.remove());
            break;
        }
        std::cmp::Ordering::Greater => return Ok(()),  // ← stops when a PAST-round proposal exists
    }
}
``` [1](#0-0) 

- **`Less` arm** (`current_round < entry.key()`): the minimum queued round is in the **future**; the correct action is `return Ok(())` (stop, keep all remaining). Instead the code **removes** that future proposal.
- **`Greater` arm** (`current_round > entry.key()`): the minimum queued round is in the **past** (stale); the correct action is `entry.remove()` (discard and continue). Instead the code **returns immediately**, leaving the stale entry in the map and never reaching the current-round entry.

The correct implementation is:

```rust
std::cmp::Ordering::Less    => return Ok(()),   // all remaining are future rounds
std::cmp::Ordering::Equal   => { to_process = Some(entry.remove()); break; }
std::cmp::Ordering::Greater => { entry.remove(); }  // stale, discard and continue
```

Proposals are queued in `validate_proposal` whenever `init.round > self.current_round`:

```rust
std::cmp::Ordering::Greater => {
    self.queued_proposals
        .insert(init.round, ((init, timeout, content_receiver), fin_sender));
    fin_receiver
}
``` [2](#0-1) 

### Impact Explanation

**Path A – stale past-round entry blocks current-round processing (`Greater` arm):**

1. Validator is at round 0; a proposal for round 1 is queued (`{1: P1}`).
2. Consensus times out and advances directly to round 2 (round skip, normal in Tendermint).
3. `set_height_and_round(height, 2)` is called; `current_round = 2`.
4. `first_entry()` returns key `1`; `2.cmp(1) = Greater` → `return Ok(())`.
5. The stale round-1 entry remains in the map; the round-2 proposal (if queued) is never extracted.
6. `validate_current_round_proposal` is never called; the validator never sends a prevote for the current round's proposal.
7. Consensus stalls or the node votes nil, preventing block finalization.

**Path B – future-round proposal deleted (`Less` arm):**

1. Validator is at round 1; a proposal for round 3 arrives and is queued (`{3: P3}`).
2. Consensus advances to round 2; `set_height_and_round(height, 2)` is called.
3. `first_entry()` returns key `3`; `2.cmp(3) = Less` → `entry.remove()`.
4. The round-3 proposal is permanently deleted from the queue.
5. When round 3 is reached, the proposal is absent; the validator must re-receive it from the network. If the proposer does not retransmit, the validator cannot validate the proposal.

Both paths are reachable without any privileged access: round skipping is a standard Tendermint liveness mechanism, and receiving proposals for future rounds is normal network behavior.

### Likelihood Explanation

Round skipping occurs whenever a round times out without a quorum (e.g., the proposer is offline or slow). This is a routine event in any live Tendermint deployment. A single timeout at round 0 advancing to round 2 is sufficient to trigger Path A. Path B is triggered whenever a proposal for round R+2 is received while the validator is at round R, which is also routine under network jitter. No attacker capability beyond sending a valid consensus proposal is required.

### Recommendation

Swap the `Less` and `Greater` arms:

```rust
while let Some(entry) = self.queued_proposals.first_entry() {
    match self.current_round.cmp(entry.key()) {
        std::cmp::Ordering::Less => {
            // All remaining entries are for future rounds; nothing to do.
            return Ok(());
        }
        std::cmp::Ordering::Equal => {
            to_process = Some(entry.remove());
            break;
        }
        std::cmp::Ordering::Greater => {
            // Stale past-round entry; discard and continue.
            entry.remove();
        }
    }
}

    async fn set_height_and_round(
        &mut self,
        height: BlockNumber,
        round: Round,
    ) -> Result<(), ConsensusError> {
        // First height or a new (higher) height.
        if self.current_height.is_none_or(|h| height > h) {
            self.update_dynamic_config().await;
            // On first height: initialize l2_gas_price to the configured minimum for this height,
            // ensuring correct startup after restart/revert.
            if self.current_height.is_none() {
                let min_gas_price_for_height = get_min_gas_price_for_height(
                    height,
                    &self.config.dynamic_config.min_l2_gas_price_per_height,
                );
                self.l2_gas_price = max(self.l2_gas_price, min_gas_price_for_height);
                let gas_price_u64 = u64::try_from(self.l2_gas_price.0).unwrap_or(u64::MAX);
                CONSENSUS_L2_GAS_PRICE.set_lossy(gas_price_u64);
            }
            self.current_height = Some(height);
            self.current_round = round;
            self.queued_proposals.clear();
            // The Batcher must be told when we begin to work on a new height. The implicit model is
            // that consensus works on a given height until it is done (either a decision is reached
            // or sync causes us to move on) and then moves on to a different height, never to
            // return to the old height.
            return Ok(self.deps.batcher.start_height(StartHeightInput { height }).await?);
        }
        assert_eq!(
            Some(height),
            self.current_height,
            "height {} is not equal to current height {:?}",
            height,
            self.current_height
        );
        if round == self.current_round {
            return Ok(());
        }
        assert!(
            round > self.current_round,
            "round {} is not greater than current round {}",
            round,
            self.current_round
        );
        self.interrupt_active_proposal().await;
        self.current_round = round;
        self.update_dynamic_config().await;

        let mut to_process = None;
        while let Some(entry) = self.queued_proposals.first_entry() {
            match self.current_round.cmp(entry.key()) {
                std::cmp::Ordering::Less => {
                    entry.remove();
                }
                std::cmp::Ordering::Equal => {
                    to_process = Some(entry.remove());
                    break;
                }
                std::cmp::Ordering::Greater => return Ok(()),
            }
        }
        // Validate the proposal for the current round if exists.
        let Some(((init, timeout, content), fin_sender)) = to_process else {
            return Ok(());
        };
        let block_info_validation = BlockInfoValidation {
            height: init.height,
            block_timestamp_window_seconds: self
                .config
                .static_config
                .block_timestamp_window_seconds,
            previous_block_info: self.previous_block_info.clone(),
            l1_da_mode: self.l1_da_mode,
            l2_gas_price_fri: self
                .config
                .dynamic_config
                .override_l2_gas_price_fri
                .map(GasPrice)
                .unwrap_or(self.l2_gas_price),
        };
        self.validate_current_round_proposal(
            init,
            block_info_validation,
            timeout,
            self.config.static_config.validate_proposal_margin_millis,
            content,
            fin_sender,
        )
        .await;
        Ok(())
    }
