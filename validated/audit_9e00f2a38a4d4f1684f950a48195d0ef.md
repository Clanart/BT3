Looking at the actual code for `set_state_header` and `set_state_finalize`, I need to trace whether the missing `shard_id` check is real and whether the corruption path is reachable by an unprivileged peer.

### Title
Missing `shard_id` binding in `set_state_header` allows a malicious peer to install cross-shard state into a syncing node — (`chain/chain/src/state_sync/adapter.rs`)

---

### Summary

`set_state_header` validates that the supplied chunk is cryptographically included in the block's `chunk_headers_root` Merkle tree, but never checks that the chunk's own `shard_id` field equals the `shard_id` argument. A malicious peer can therefore respond to a state-sync header request for shard A with a `ShardStateSyncResponseHeader` that embeds shard B's chunk (plus a valid Merkle proof for shard B's position in the tree). The header passes all existing guards and is persisted under the key `StateHeaderKey(shard_id=A, sync_hash)`. Downstream, `set_state_finalize` retrieves that header and applies shard B's transactions and receipts to shard A's trie (using `shard_uid` derived from the caller's `shard_id=A` argument), producing a wrong `chunk_extra.state_root` for shard A.

---

### Finding Description

**Entry point — any network peer:**

The syncing node broadcasts `NetworkRequests::StateRequestHeader { shard_id, sync_hash }` to a randomly selected peer. No validator or operator privilege is required to respond; any connected peer may reply with a `VersionedStateResponse`. [1](#0-0) 

**`set_state_header` — missing shard_id guard:**

After extracting the chunk from the received header, the function runs five checks:

1. `validate_chunk_proofs` — verifies the chunk's internal hash, tx root, and receipts root. Does **not** inspect `chunk.shard_id()`. [2](#0-1) 

2. `verify_path` against `sync_prev_block_header.chunk_headers_root()` — verifies that `ChunkHashHeight(chunk.chunk_hash(), chunk.height_included())` is present somewhere in the Merkle tree. The tree covers **all** shards; the proof encodes the leaf's position but the check does **not** assert that position equals the shard index for `shard_id=A`. Shard B's chunk, which legitimately appears in the same block, passes this check with its own valid Merkle path. [3](#0-2) 

3. Receipt proof validation — computes `receipts_hash = CryptoHash::hash_borsh(ReceiptList(shard_id, receipts))` using the **caller's** `shard_id=A`, not the chunk's shard_id. An attacker supplies empty (or legitimately empty) receipt lists for shard A, which produce valid Merkle proofs against the block's `prev_chunk_outgoing_receipts_root`. [4](#0-3) 

4. `validate_state_root_node` — validates the `state_root_node` against `chunk_inner.prev_state_root()`, which is shard B's state root. Passes trivially when the attacker supplies shard B's genuine `state_root_node`. [5](#0-4) 

5. **Storage** — the header is written under `StateHeaderKey(shard_id=A, sync_hash)` with no shard_id binding check. [6](#0-5) 

**`set_state_finalize` / `chain_update::set_state_finalize` — no re-check:**

`Chain::set_state_finalize` retrieves the stored header by `(shard_id=A, sync_hash)` and passes it directly to `chain_update::set_state_finalize`. That function:

- Derives `shard_uid` from the caller's `shard_id=A`.
- Calls `apply_chunk` with `shard_uid=A` but `chunk_header.prev_state_root()` from shard B's chunk — the state root that was installed into shard A's trie storage by the preceding `apply_state_part` calls.
- Saves the resulting `ChunkExtra` (with shard B's post-application state root) under shard A's key. [7](#0-6) [8](#0-7) 

---

### Impact Explanation

The syncing node's `chunk_extra.state_root` for shard A is permanently set to the result of applying shard B's chunk to shard B's state. All subsequent block validation and chunk application for shard A on this node will use the wrong state root, causing the node to diverge from the canonical chain. The corruption is local to the syncing node but is persistent and silent — no error is raised during or after `set_state_finalize`.

---

### Likelihood Explanation

Any peer on the NEAR p2p network can respond to `StateRequestHeader`. The attacker needs only to:
1. Run a standard nearcore node (no validator key required).
2. Serve a `ShardStateSyncResponseHeader` for shard A that embeds shard B's chunk and its genuine Merkle proof — all data is publicly available from any full node.
3. Provide valid state parts for shard B's state (also publicly available).

The syncing node retries on failure but accepts the first response that passes `set_state_header`; a single malicious peer that wins the race corrupts the node.

---

### Recommendation

Add an explicit shard_id binding check immediately after extracting the chunk in `set_state_header`:

```rust
// In set_state_header, after: let chunk = shard_state_header.cloned_chunk();
if chunk.shard_id() != shard_id {
    byzantine_assert!(false);
    return Err(Error::Other(
        "set_shard_state failed: chunk shard_id does not match requested shard_id".into(),
    ));
}
``` [9](#0-8) 

A symmetric guard in `chain_update::set_state_finalize` provides defence-in-depth:

```rust
// After extracting chunk from shard_state_header
if chunk.shard_id() != shard_id {
    return Err(Error::Other(
        "set_state_finalize: chunk shard_id mismatch".into(),
    ));
}
``` [10](#0-9)

pub fn validate_chunk_proofs(
    chunk: &ShardChunk,
    epoch_manager: &dyn EpochManagerAdapter,
) -> Result<bool, Error> {
    let correct_chunk_hash = chunk.compute_header_hash();

    // 1. Checking chunk.header.hash
    let header_hash = chunk.header_hash();
    if header_hash != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }

    // 2. Checking that chunk body is valid
    // 2a. Checking chunk hash
    if chunk.chunk_hash() != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }
    let height_created = chunk.height_created();
    let outgoing_receipts_root = chunk.prev_outgoing_receipts_root();
    let (transactions, receipts) = (chunk.to_transactions(), chunk.prev_outgoing_receipts());

    // 2b. Checking that chunk transactions are valid
    let (tx_root, _) = merklize(transactions);
    if &tx_root != chunk.tx_root() {
        byzantine_assert!(false);
        return Ok(false);
    }
    // 2c. Checking that chunk receipts are valid
    if height_created == 0 {
        return Ok(receipts.is_empty() && outgoing_receipts_root == &CryptoHash::default());
    } else {
        let shard_layout = {
            let prev_block_hash = chunk.prev_block_hash();
            epoch_manager.get_shard_layout_from_prev_block(&prev_block_hash)?
        };
        let outgoing_receipts_hashes = Chain::build_receipts_hashes(receipts, &shard_layout)?;
        let (receipts_root, _) = merklize(&outgoing_receipts_hashes);
        if &receipts_root != outgoing_receipts_root {
            byzantine_assert!(false);
            return Ok(false);
        }
    }
    Ok(true)
}

    pub fn set_state_finalize(
        &mut self,
        shard_id: ShardId,
        sync_hash: CryptoHash,
    ) -> Result<(), Error> {
        let shard_state_header = self.state_sync_adapter.get_state_header(shard_id, sync_hash)?;
        let mut height = shard_state_header.chunk_height_included();
        let mut chain_update = self.chain_update();
        let shard_uid = chain_update.set_state_finalize(shard_id, sync_hash, shard_state_header)?;
        chain_update.commit()?;

        // We restored the state on height `shard_state_header.chunk.header.height_included`.
        // Now we should build a chain up to height of `sync_hash` block.
        loop {
            height += 1;
            let mut chain_update = self.chain_update();
            // Result of successful execution of set_state_finalize_on_height is bool,
            // should we commit and continue or stop.
            if chain_update.set_state_finalize_on_height(height, shard_id, sync_hash)? {
                chain_update.commit()?;
            } else {
                break;
            }
        }

        let flat_storage_manager = self.runtime_adapter.get_flat_storage_manager();
        if let Some(flat_storage) = flat_storage_manager.get_flat_storage_for_shard(shard_uid) {
            let header = self.get_block_header(&sync_hash)?;
            flat_storage.update_flat_head(header.prev_hash()).unwrap();
        }

        Ok(())
    }
