### Title
Stale Feemarket Params Snapshot in Cosmos Ante Handler Bypasses Dynamic Base-Fee Enforcement — (`evmd/ante/handler_options.go`)

---

### Summary

`newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712` snapshot `feemarketParams` (including `BaseFee`) once at construction time and pass a pointer to that frozen copy into `NewDynamicFeeChecker` and `NewMinGasPriceDecorator`. Because the feemarket `BeginBlock` updates `BaseFee` in the KVStore every block, the Cosmos-tx ante path always enforces a stale base fee. The Ethereum-tx path (`newEthAnteHandler`) does not share this flaw — it reads fresh `blockCfg` on every invocation.

---

### Finding Description

**`newCosmosAnteHandler` — stale snapshot**

```go
// evmd/ante/handler_options.go:178-187
func newCosmosAnteHandler(ctx sdk.Context, options HandlerOptions, extra ...sdk.AnteDecorator) sdk.AnteHandler {
    evmParams        := options.EvmKeeper.GetParams(ctx)       // snapshot at construction
    feemarketParams  := options.FeeMarketKeeper.GetParams(ctx) // snapshot at construction
    ...
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
``` [1](#0-0) 

`feemarketParams` is a value-type struct. `&feemarketParams` is a pointer to a local variable that is captured by the `NewDynamicFeeChecker` closure. After construction the struct is never mutated; `BaseFee` inside it is frozen at whatever value was in the store when the handler was built.

The same pattern appears in `newLegacyCosmosAnteHandlerEip712`:

```go
// evmd/ante/handler_options.go:29-37
evmParams        := options.EvmKeeper.GetParams(ctx)
feemarketParams  := options.FeeMarketKeeper.GetParams(ctx)
...
txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
``` [2](#0-1) 

**`NewDynamicFeeChecker` — uses the frozen `feemarketParams.BaseFee`**

```go
// ante/evm/fee_checker.go:56
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
``` [3](#0-2) 

`GetBaseFee` reads `feemarketParams.GetBaseFee()` — the frozen snapshot — not the live KVStore value. [4](#0-3) 

**`NewMinGasPriceDecorator` — same frozen pointer**

```go
cosmos.NewMinGasPriceDecorator(options.FeeMarketKeeper, evmDenom, &feemarketParams)
``` [5](#0-4) 

**Contrast: `newEthAnteHandler` reads fresh state on every call**

```go
// evmd/ante/handler_options.go:88-95
blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
...
feemarketParams := &blockCfg.FeeMarketParams
baseFee         := blockCfg.BaseFee
``` [6](#0-5) 

The Ethereum-tx path is correct; the Cosmos-tx path is not.

**`BeginBlock` updates `BaseFee` every block**

```go
// x/feemarket/keeper/abci.go:31-38
baseFee := k.CalculateBaseFee(ctx)
...
k.SetBaseFee(ctx, baseFee)
``` [7](#0-6) 

`SetBaseFee` writes the new value into the KVStore params, but the frozen snapshot held by the Cosmos ante handler is never refreshed. [8](#0-7) 

---

### Impact Explanation

When the live base fee rises above the frozen snapshot value, Cosmos-native transactions (and legacy EIP-712 transactions) whose `feeCap` lies between the stale base fee and the real current base fee pass `NewDynamicFeeChecker` and `NewMinGasPriceDecorator` and are committed to the chain. These transactions pay less than the EIP-1559 market rate. Simultaneously, the EVM-tx path correctly rejects the same fee level. The result is:

- **Fee mis-accounting**: Cosmos-tx senders underpay fees relative to the enforced market rate, violating the fee market invariant.
- **Asymmetric enforcement**: EVM transactions and Cosmos transactions are held to different base-fee standards within the same block, breaking the unified fee market.
- **Mempool ordering corruption**: `NewDynamicFeeChecker` also computes `priority` from `effectivePrice - baseFeeInt`; a stale (lower) base fee inflates the computed priority of under-priced Cosmos transactions, causing them to be ordered ahead of correctly-priced EVM transactions.

This matches the allowed High impact: *"fee market, ante handler, or proposal handling bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

---

### Likelihood Explanation

The base fee changes every block whenever gas usage deviates from the target. On any active chain the frozen snapshot diverges from the live base fee within minutes of startup. Any user who submits a Cosmos-native transaction (e.g., a `MsgSend`, an authz `MsgExec`, or a legacy EIP-712 tx) with a fee cap calibrated to the stale base fee will trigger this path. No special privileges are required; a standard unprivileged transaction is sufficient.

---

### Recommendation

**Short term**: Replace the snapshot pattern in `newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712` with the same live-read pattern used by `newEthAnteHandler`. Specifically, move the `GetParams` calls inside the returned closure so they execute against the live `ctx` on every transaction:

```go
func newCosmosAnteHandler(options HandlerOptions, extra ...sdk.AnteDecorator) sdk.AnteHandler {
    return func(ctx sdk.Context, tx sdk.Tx, simulate bool) (sdk.Context, error) {
        evmParams       := options.EvmKeeper.GetParams(ctx)
        feemarketParams := options.FeeMarketKeeper.GetParams(ctx)
        // build decorators with fresh params ...
    }
}
```

**Long term**: Unify the Cosmos and Ethereum ante paths to share the same `EVMBlockConfig` read, eliminating the possibility of divergence.

---

### Proof of Concept

1. Chain starts; genesis `BaseFee = 1_000_000_000` (1 Gwei). `newCosmosAnteHandler` is constructed; `feemarketParams.BaseFee` is frozen at 1 Gwei.
2. Blocks fill above the gas target. After N blocks, `BeginBlock` has raised the live `BaseFee` to 2 Gwei in the KVStore.
3. Alice submits a Cosmos `MsgSend` with `feeCap = 1.5 Gwei`.
4. `NewDynamicFeeChecker` reads `baseFee` from the frozen snapshot → 1 Gwei. `feeCap (1.5) >= baseFee (1.0)` → **passes**.
5. Bob submits an EVM transfer with `gasFeeCap = 1.5 Gwei`. `newEthAnteHandler` reads live `BaseFee = 2 Gwei`. `feeCap (1.5) < baseFee (2.0)` → **rejected**.
6. Alice's under-priced Cosmos transaction is committed; Bob's correctly-priced EVM transaction is rejected. The fee market is violated.

func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
	if !IsLondon(ethCfg, height) {
		return nil
	}
	baseFee := feemarketParams.GetBaseFee()
	// should not be nil if london hardfork enabled
	if baseFee == nil {
		return new(big.Int)
	}
	return baseFee
}

func (k Keeper) SetBaseFee(ctx sdk.Context, baseFee *big.Int) {
	params := k.GetParams(ctx)
	params.BaseFee = ethermint.SaturatedNewInt(baseFee)
	err := k.SetParams(ctx, params)

	}
