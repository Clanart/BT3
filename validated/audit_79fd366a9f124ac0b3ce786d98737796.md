### Title
`ProofOfInclusion::valid()` Tautological Root Check Allows Forged DataLayer Inclusion Proofs — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

---

### Summary

`ProofOfInclusion::valid()` contains a tautological final check: after the loop, `existing_hash` is always equal to `self.root_hash()` by construction, so the check is never meaningful. The function only verifies internal self-consistency of the proof structure, not that the proof's root matches any external, authoritative tree root. An attacker can construct a `ProofOfInclusion` with an arbitrary `node_hash` (claiming any key is included) and fabricated layers, and `valid()` will return `true`.

---

### Finding Description

In `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`, the `valid()` function is:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()   // ← always true
}
```

And `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← same value as existing_hash after loop
    } else {
        self.node_hash
    }
}
```

**Trace of the tautology:**

1. The loop body checks `calculated_hash != layer.combined_hash` and returns `false` if they differ; otherwise it sets `existing_hash = calculated_hash`.
2. After the loop, `existing_hash` equals the last `layer.combined_hash`.
3. `root_hash()` returns `last.combined_hash`.
4. Therefore `existing_hash == self.root_hash()` is `last.combined_hash == last.combined_hash` — always `true`.

The function never compares the computed root against any externally-known, authoritative tree root. An attacker who knows the `calculate_internal_hash` function (which is the public DataLayer hash primitive) can:

1. Pick any `node_hash` (e.g., the hash of a key-value pair that does **not** exist in the tree).
2. Pick arbitrary `other_hash` values for each layer.
3. Compute each `combined_hash` correctly using `calculate_internal_hash`.
4. Serialize the resulting `ProofOfInclusion` (it is a `Streamable` type and has a Python binding).
5. Call `proof.valid()` — it returns `true`.

The forged proof's `root_hash()` will not match the real tree root, but `valid()` never checks that.

The Python binding `py_valid()` exposes this function directly to Python callers:

```rust
#[pyo3(name = "valid")]
pub fn py_valid(&self) -> bool {
    self.valid()
}
```

The existing test `test_proof_of_inclusion_invalid_identified` only mutates `combined_hash` to confirm internal-consistency detection; it never checks the proof against an external root, confirming the intended (but broken) API.

---

### Impact Explanation

Any DataLayer node or client that receives a `ProofOfInclusion` from an untrusted peer and calls `proof.valid()` without also asserting `proof.root_hash() == known_root` will accept a completely fabricated proof. This lets an attacker:

- Prove that an arbitrary key-value pair is **included** in a DataLayer store when it is not.
- Prove that an arbitrary key-value pair is **excluded** from a store when it is actually present (by forging a proof for a different leaf that happens to produce the same claimed root).
- Corrupt the perceived state of a DataLayer tree, enabling forged inclusion/exclusion attestations.

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

---

### Likelihood Explanation

`ProofOfInclusion` is a `Streamable` type and is exposed via Python bindings. DataLayer nodes exchange proofs over the network to attest to key-value membership. The function name `valid()` strongly implies a complete validity check; callers are not warned that they must separately assert `proof.root_hash() == known_root`. The existing test suite itself calls only `assert proof_of_inclusion.valid()` without a root check, reinforcing the incorrect usage pattern. Any downstream Python code (e.g., in `chia-blockchain`) that follows the same pattern is immediately exploitable by a peer that sends a crafted `ProofOfInclusion`.

---

### Recommendation

Replace the tautological final comparison with a check against a caller-supplied authoritative root:

```rust
pub fn valid(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;
    }

    &existing_hash == expected_root   // compare against the real tree root
}
```

Update the Python binding accordingly, and update all call sites (including tests) to pass the authoritative root obtained from the `MerkleBlob`.

### Proof of Concept

```python
# Attacker constructs a forged ProofOfInclusion for a key not in the tree.
# calculate_internal_hash is the public DataLayer hash combiner.

fake_node_hash = bytes([0xAA] * 32)   # hash of a non-existent key-value pair
other_hash     = bytes([0xBB] * 32)   # arbitrary sibling hash

# Compute combined_hash correctly — attacker knows the hash function
combined_hash = calculate_internal_hash(fake_node_hash, Side.Left, other_hash)

layer = ProofOfInclusionLayer(
    other_hash_side=Side.Left,
    other_hash=other_hash,
    combined_hash=combined_hash,
)

forged_proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

# valid() returns True even though fake_node_hash is not in any real tree
assert forged_proof.valid()

# The forged root does NOT match the real tree root, but valid() never checks:
