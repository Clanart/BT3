### Title
Missing Declared Class Validation in `execute_replace_class` Allows Arbitrary Class Hash Assignment - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the requested replacement class hash corresponds to a previously declared contract class. Any contract can call `replace_class` with an arbitrary, undeclared class hash, permanently corrupting its own state and freezing any funds it holds.

### Finding Description
In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall by reading `request.class_hash` and directly writing it into `contract_state_changes` without checking whether that class hash exists in `contract_class_changes` (the OS-level registry of declared classes):

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-acknowledged TODO at line 898 explicitly confirms the missing check. The `contract_class_changes` dictionary (populated by `execute_declare_transaction` via `dict_update` with `prev_value=0`) maps class hashes to compiled class hashes. The `execute_replace_class` function never consults this dictionary to validate the incoming `class_hash`.

By contrast, the declare flow in `transaction_impls.cairo` enforces `prev_value=0` to ensure a class is only declared once and `assert_not_zero(compiled_class_hash)` to ensure a valid compiled class exists. No equivalent guard exists in `execute_replace_class`.

### Impact Explanation
When a contract's `class_hash` field in `contract_state_changes` is set to an arbitrary undeclared value, the OS commits this invalid state to the chain. Any subsequent transaction targeting that contract will fail at class resolution time because no compiled class facts exist for the hash. All assets (ERC-20 balances, NFTs, or any storage-backed value) held by the contract become permanently inaccessible. This constitutes **Critical — Permanent freezing of funds**.

### Likelihood Explanation
The `replace_class` syscall is a standard, permissionless operation callable by any Sierra contract during its own execution. No privileged role or special condition is required beyond having enough gas. The attacker-controlled entry path is a single transaction: deploy a contract, invoke it with calldata that triggers `replace_class` with an arbitrary felt value as the class hash. The OS will accept and commit the resulting state without rejection.

### Recommendation
Before writing the new `StateEntry`, verify that the requested `class_hash` is present in `contract_class_changes` with a non-zero compiled class hash. Concretely, perform a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert the returned value is non-zero, mirroring the validation already enforced in `execute_declare_transaction`.

### Proof of Concept

1. Deploy contract `C` holding token balances.
2. From within `C`, issue the `replace_class` syscall with `class_hash = 0xdeadbeef` (any felt not present in `contract_class_changes`).
3. The OS executes `execute_replace_class`:
   - Gas check passes.
   - `class_hash = 0xdeadbeef` is read from the request.
   - **No validation against `contract_class_changes` is performed** (the TODO at line 898 is skipped).
   - `contract_state_changes[C]` is updated with `class_hash = 0xdeadbeef`.
4. The block is proven and committed. The state root now encodes `C` with class hash `0xdeadbeef`.
5. Any future `invoke` targeting `C` fails: the OS cannot find compiled class facts for `0xdeadbeef` in `compiled_class_facts_bundle`.
6. All funds in `C` are permanently frozen. [1](#0-0) [2](#0-1) [3](#0-2)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.

        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

        key=contract_address,

    );
