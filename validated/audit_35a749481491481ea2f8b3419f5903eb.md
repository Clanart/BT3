### Title
Unhandled Constructor Revert in `deploy_contract` Causes Unprovable Block and Network Halt — (File: `execution/deploy_contract.cairo`)

---

### Summary

The `deploy_contract` function in the StarkNet OS unconditionally asserts `is_reverted = 0` after invoking a contract constructor. Because the OS still processes syscalls for reverted outer transactions, an unprivileged user can submit an invoke transaction that calls the `deploy` syscall with a class whose constructor always reverts. The sequencer includes the outer transaction as a standard reverted transaction, but the OS hits the hard assertion and fails to generate a valid proof for the block, permanently halting the network.

---

### Finding Description

**Root cause — `deploy_contract.cairo` line 91:**

```cairo
// The deprecated deploy syscalls do not support reverts.
assert is_reverted = 0;
```

The function header at line 28 acknowledges the gap:

```cairo
// TODO(Yoni, 1/1/2027): handle failures.
func deploy_contract{...}(...)
``` [1](#0-0) 

**Compounding issue — `syscall_impls.cairo` lines 538–539:**

The `execute_deploy` syscall hardcodes `failure_flag=0` in the response header regardless of the constructor outcome:

```cairo
// TODO(Yoni, 1/1/2026): support failures.
assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=0);
``` [2](#0-1) 

**Why syscalls are processed even for reverted transactions — `execute_entry_point.cairo` lines 288–307:**

Even when `is_reverted != FALSE`, `call_execute_syscalls` is unconditionally called to advance the syscall pointer. This means the `deploy` syscall is replayed by the OS even when the outer entry point is reverted.

```cairo
if (is_reverted != FALSE) {
    let revert_log = init_revert_log();
} else { ... }
let builtin_ptrs = return_builtin_ptrs;
with syscall_ptr {
    call_execute_syscalls(...);   // always called
}
``` [3](#0-2) 

**Attack path:**

1. Attacker declares a Sierra class whose constructor always calls `panic` (reverts).
2. Attacker submits an invoke transaction that calls the `deploy` syscall with this class hash.
3. The sequencer simulates: constructor reverts → outer invoke is marked reverted → transaction is included in the block (reverted transactions pay fees and are included in Starknet blocks).
4. The OS processes the block. For the reverted invoke, it still calls `call_execute_syscalls`, which reaches `execute_deploy` → `deploy_contract` → constructor → `is_reverted = 1`.
5. `assert is_reverted = 0` fires. The OS proof fails. The block is permanently unprovable.

---

### Impact Explanation

The block containing the malicious transaction cannot be proven. Since the sequencer has already committed to the block and the OS cannot generate a valid STARK proof for it, the network cannot advance its state. This is a **total network shutdown** — no new transactions can be confirmed until the issue is resolved out-of-band.

**Impact: High — Network not being able to confirm new transactions.**

---

### Likelihood Explanation

Any unprivileged user can:
- Submit a `declare` transaction for a class with a reverting constructor (standard operation).
- Submit an invoke transaction calling `deploy` with that class hash.

No privileged access, leaked keys, malicious operator cooperation, or external dependencies are required. The attack is cheap, deterministic, and repeatable.

---

### Recommendation

Remove the `assert is_reverted = 0` constraint in `deploy_contract.cairo` and implement proper constructor revert handling:

1. When `is_reverted == 1`, undo the `dict_update` that set the new class hash (revert the state entry back to `UNINITIALIZED_CLASS_HASH`).
2. Propagate `is_reverted=1` back to `execute_deploy`.
3. In `execute_deploy` (`syscall_impls.cairo`), write `failure_flag=is_reverted` in the response header instead of hardcoding `0`.

This mirrors how `execute_entry_point.cairo` already handles reverts via `handle_revert` and the revert log mechanism. [4](#0-3) 

---

### Proof of Concept

```
1. Declare class `RevertingCtor` with constructor:
       fn constructor(...) { panic!("always revert"); }

2. Submit invoke tx from any account:
       contract.call_deploy(class_hash=RevertingCtor_hash, salt=0, calldata=[])

3. Sequencer simulation:
       - Constructor reverts → deploy syscall fails → outer invoke reverts
       - Sequencer includes the reverted invoke in the next block (standard behavior)

4. OS proof generation for that block:
       execute_invoke_function_transaction()
         → execute_entry_point()          [is_reverted=1 for outer invoke]
           → call_execute_syscalls()      [always called, even for reverted]
             → execute_deploy()           [syscall_impls.cairo:452]
               → deploy_contract()        [deploy_contract.cairo:29]
                 → select_execute_entry_point_func()  → is_reverted=1
                 → assert is_reverted = 0  ← FAILS

5. OS proof fails. Block is unprovable. Network halts.
```

    range_check_ptr,

    retdata_size: felt, retdata: felt*

    alloc_locals;

    local contract_address = constructor_execution_context.execution_info.contract_address;

    // Assert that we don't deploy to one of the reserved addresses.
    assert_not_zero(
        (contract_address - ORIGIN_ADDRESS) * (contract_address - BLOCK_HASH_CONTRACT_ADDRESS) * (
            contract_address - ALIAS_CONTRACT_ADDRESS
        ) * (contract_address - RESERVED_CONTRACT_ADDRESS),
    );

    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}
    assert state_entry.class_hash = UNINITIALIZED_CLASS_HASH;
    assert state_entry.nonce = 0;

    tempvar new_state_entry = new StateEntry(
        class_hash=constructor_execution_context.class_hash,
        storage_ptr=state_entry.storage_ptr,
        nonce=0,
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    // Entries before this point belong to the caller.
    // Note the caller is the deployer (not the caller of the deployer).
    assert [revert_log] = RevertLogEntry(

        value=constructor_execution_context.execution_info.caller_address,
    );
    let revert_log = &revert_log[1];

    assert [revert_log] = RevertLogEntry(
        selector=CHANGE_CLASS_ENTRY, value=UNINITIALIZED_CLASS_HASH
    );
    let revert_log = &revert_log[1];

    // Invoke the contract constructor.
    let (is_reverted, retdata_size, retdata, _is_deprecated) = select_execute_entry_point_func(
        block_context=block_context, execution_context=constructor_execution_context
    );

    // Entries before this point belong to the deployed contract.
    assert [revert_log] = RevertLogEntry(selector=CHANGE_CONTRACT_ENTRY, value=contract_address);
    let revert_log = &revert_log[1];

    // The deprecated deploy syscalls do not support reverts.
    assert is_reverted = 0;
    return (retdata_size=retdata_size, retdata=retdata);
