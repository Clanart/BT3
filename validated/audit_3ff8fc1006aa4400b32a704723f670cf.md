### Title
Proof-facts cache bypass allows any transaction to reuse a previously verified proof and skip cryptographic verification — (`crates/apollo_transaction_converter/src/transaction_converter.rs`)

---

### Summary

`run_proof_verification` in `TransactionConverter` skips cryptographic proof verification whenever `contains_proof(proof_facts)` returns `true`. Because `ProofFacts` are keyed solely by a hash of `(proof_version, program_hash, block_number, block_hash, config_hash)` — with no binding to the submitting transaction's `sender_address`, `calldata`, or transaction hash — any attacker who copies a legitimate transaction's `proof_facts` can submit their own Invoke V3 transaction with an arbitrary (invalid) proof and have it admitted by the gateway without any cryptographic check.

---

### Finding Description

**Root cause — `ProofFacts` contain no transaction-specific identity**

`SnosProofFacts` is parsed from the raw `ProofFacts` felt array and contains exactly:

```
proof_version | program_hash | block_number | block_hash | config_hash
``` [1](#0-0) 

`ProofFacts::hash()` is a Poseidon hash over this same felt array — no `sender_address`, no `calldata`, no transaction hash is included. [2](#0-1) 

The proof manager stores and looks up proofs keyed exclusively by this hash: [3](#0-2) 

**Root cause — `run_proof_verification` skips verification on cache hit**

```rust
async fn run_proof_verification(
    proof_facts: ProofFacts,
    proof: Proof,
    proof_manager_client: SharedProofManagerClient,
) -> Result<bool, TransactionConverterError> {
    let contains_proof = proof_manager_client.contains_proof(proof_facts.clone()).await?;

    if contains_proof {
        return Ok(false);   // ← proof argument is silently discarded
    }
    // cryptographic verification only reached on first submission
    starknet_proof_verifier::verify_proof(proof_facts, proof)
    ...
}
``` [4](#0-3) 

When `contains_proof` is `true`, the supplied `proof` bytes are never examined. Any non-empty byte string satisfies the only other proof-related stateless check (`validate_proof_facts_and_proof_consistency` requires both fields to be simultaneously empty or non-empty): [5](#0-4) 

**Blockifier pre-validation does not re-verify the proof**

`validate_proof_facts` in the blockifier checks only `program_hash`, `block_number`, `block_hash`, and `config_hash` — all of which are valid because the attacker copied a legitimate `proof_facts`. It never re-runs the cryptographic verifier and never checks that the proof covers the attacker's specific transaction: [6](#0-5) 

The same structural gap exists in the Cairo OS `check_proof_facts`, which validates the same four fields and nothing else: [7](#0-6) 

**Transaction hash does not close the gap**

`get_invoke_transaction_v3_hash` does chain the `proof_facts` hash into the transaction hash, so the attacker's transaction hash differs from the victim's: [8](#0-7) 

This means the attacker must produce a valid ECDSA signature over their own transaction hash — which they can do with their own key. The transaction hash binding does not prevent reuse of `proof_facts`; it only means the attacker signs their own (different) transaction.

---

### Impact Explanation

The gateway admits an Invoke V3 transaction whose attached `proof` is cryptographically invalid (it was generated for a different transaction). The proof manager's `contains_proof` gate is the only barrier between submission and acceptance, and it is bypassed by copying any previously accepted `proof_facts`. Because client-side proving transactions carry zero `max_price_per_unit` across all resource bounds (enforced by `validate_zero_fee_resource_bounds`), the attacker executes arbitrary calldata with zero fee cost, constituting both:

- **High** — Mempool/gateway admission accepts an invalid transaction (one whose proof does not cover its execution).
- **Critical** — Incorrect fee/resource accounting with economic impact: the sequencer bears execution cost while the attacker pays nothing. [9](#0-8) 

---

### Likelihood Explanation

**High.** The attack requires no privileged access and no cryptographic capability:

1. `proof_facts` are public — they appear in the transaction body on-chain and in RPC responses (confirmed by the `block_post_0_14_2.json` fixture showing multiple transactions sharing identical `proof_facts` arrays).
2. The attacker only needs to copy the felt array verbatim into their own transaction.
3. The `proof` field only needs to be non-empty (any byte string passes `validate_proof_facts_and_proof_consistency`).
4. The attack is repeatable for every block whose hash is stored, not just once.

---

### Recommendation

Bind `proof_facts` storage and lookup to the submitting transaction's hash. Replace the single-key lookup:

```rust
proof_manager_client.contains_proof(proof_facts)
```

with a two-key lookup keyed by `(proof_facts_hash, tx_hash)`. Alternatively, always re-run `verify_proof` regardless of cache state, using the cache only to avoid redundant filesystem I/O, not to skip cryptographic verification. The OS-level `check_proof_facts` should additionally assert that the proof output includes the hash of the transaction being executed.

---

### Proof of Concept

1. **Alice** submits a valid client-side proving Invoke V3 transaction with `proof_facts = P` (containing a legitimate `block_number`/`block_hash`/`config_hash`/`program_hash`) and a valid cryptographic `proof = Q`. The gateway calls `verify_proof(P, Q)` → passes; stores `(P.hash() → Q)` in the proof manager.

2. **Bob** observes Alice's transaction (via RPC or on-chain) and copies her `proof_facts = P` verbatim.

3. Bob constructs his own Invoke V3 transaction:
   - `sender_address` = Bob's address
   - `calldata` = arbitrary payload
   - `proof_facts` = P (copied from Alice)
   - `proof` = `[0x01]` (one non-empty byte — passes `validate_proof_facts_and_proof_consistency`)
   - `max_price_per_unit` = 0 for all resources (client-side proving fee structure)
   - Signs with his own key over his own transaction hash (which includes `P.hash()`)

4. Gateway calls `run_proof_verification(P, [0x01], ...)`:
   - `contains_proof(P)` → `true` (Alice's proof is stored)
   - Returns `Ok(false)` — **cryptographic verification skipped entirely**

5. Stateful validation passes (Bob has a valid nonce and account).

6. Bob's transaction is admitted to the mempool and executed with zero fees.

7. `validate_proof_facts` in the blockifier checks `program_hash`, `block_number`, `block_hash`, `config_hash` — all valid because P is legitimate — and returns `Ok(())`.

8. Bob's arbitrary `calldata` executes at zero cost to Bob. [10](#0-9) [11](#0-10)

    fn validate_proof_facts(
        &self,
        block_context: &BlockContext,
        state: &mut dyn State,
    ) -> TransactionPreValidationResult<()> {
        // Only Invoke V3 transactions can carry proof facts.
        let Transaction::Invoke(invoke_tx) = &self.tx else {
            return Ok(());
        };
        if invoke_tx.version() < TransactionVersion::THREE {
            return Ok(());
        }

        // Parse proof facts.
        let proof_facts = invoke_tx.proof_facts();
        let snos_proof_facts = match ProofFactsVariant::try_from(&proof_facts)
            .map_err(|e| TransactionPreValidationError::InvalidProofFacts(e.to_string()))?
        {
            ProofFactsVariant::Empty => return Ok(()),
            ProofFactsVariant::Snos(snos_proof_facts) => snos_proof_facts,
        };
        let os_constants = &block_context.versioned_constants.os_constants;

        // Validate the program hash.
        let allowed = &os_constants.allowed_virtual_os_program_hashes;
        if !allowed.contains(&snos_proof_facts.program_hash) {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS program hash {} is not allowed",
                snos_proof_facts.program_hash
            )));
        }

        // Validate the block hash and block number.
        let proof_block_hash = snos_proof_facts.block_hash.0;
        let proof_block_number = snos_proof_facts.block_number.0;
        Self::validate_proof_block_number(
            proof_block_number,
            block_context.block_info.block_number,
        )?;
        Self::validate_proof_block_hash(proof_block_hash, proof_block_number, os_constants, state)?;

        // Validate the config hash.
        let chain_info = &block_context.chain_info;
        // TODO(Meshi): Cache this computation as part of the chain context.
        let virtual_os_config_hash = OsChainInfo::from(chain_info)
            .compute_virtual_os_config_hash()
            .expect("Failed to compute OS config hash");
        let proof_config_hash = snos_proof_facts.config_hash;
        if virtual_os_config_hash != proof_config_hash {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS config hash mismatch. Computed virtual OS config hash: \
                 {virtual_os_config_hash}, expected virtual OS config hash: {proof_config_hash}."
            )));
        }

        Ok(())
    }

func check_proof_facts{range_check_ptr, contract_state_changes: DictAccess*}(
    proof_facts_size: felt,
    proof_facts: felt*,
    current_block_number: felt,
    virtual_os_config_hash: felt,
) {
    if (proof_facts_size == 0) {
        return ();
    }

    assert_le(ProofHeader.SIZE + VirtualOsOutputHeader.SIZE, proof_facts_size);

    // Validate the proof header.
    let proof_header = cast(proof_facts, ProofHeader*);
    assert is_program_hash_allowed(proof_header.program_hash) = TRUE;
    // Proof version and variant are for future compatibility.
    assert [proof_header] = ProofHeader(
        proof_version=PROOF_VERSION,
        proof_variant=VIRTUAL_SNOS,
        program_hash=proof_header.program_hash,
    );

    // Validate the virtual OS output header.
    let os_output_header = cast(&proof_facts[ProofHeader.SIZE], VirtualOsOutputHeader*);

    with_attr error_message("Virtual OS output version is not supported") {
        assert os_output_header.output_version = VIRTUAL_OS_OUTPUT_VERSION;
    }

    // Validate that the proof facts block number is not too recent.
    // (This is a sanity check - the following non-zero check ensures that the block hash is
    // not trivial).
    assert_nn_le(
        os_output_header.base_block_number, current_block_number - STORED_BLOCK_HASH_BUFFER
    );
    // Not all block hashes are stored in the contract; Make sure the requested one is not trivial.
    assert_not_zero(os_output_header.base_block_hash);

    // validate that the proof facts block hash is the true hash of the proof facts block number.
    read_block_hash_from_storage(
        block_number=os_output_header.base_block_number,
        expected_block_hash=os_output_header.base_block_hash,
    );

    // validate that the proof facts config hash is the true hash of the OS config.
    assert os_output_header.starknet_os_config_hash = virtual_os_config_hash;

    return ();
}

fn validate_zero_fee_resource_bounds(
    tx: &RpcInvokeTransactionV3,

    let bounds = &tx.resource_bounds;
    let mut violations = Vec::new();

    if bounds.l1_gas.max_price_per_unit != GasPrice(0) {
        violations
            .push(format!("l1_gas.max_price_per_unit = {}", bounds.l1_gas.max_price_per_unit.0));
    }
    if bounds.l2_gas.max_price_per_unit != GasPrice(0) {
        violations
            .push(format!("l2_gas.max_price_per_unit = {}", bounds.l2_gas.max_price_per_unit.0));
    }
    if bounds.l1_data_gas.max_price_per_unit != GasPrice(0) {
        violations.push(format!(
            "l1_data_gas.max_price_per_unit = {}",
            bounds.l1_data_gas.max_price_per_unit.0
        ));
    }
    if tx.tip != Tip(0) {
        violations.push(format!("tip = {}", tx.tip.0));
