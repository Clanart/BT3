### Title
`ProofOfInclusion::valid()` Verifies Hash Chain Only, Not Key-to-Hash Binding — Forged Inclusion Proof Accepted from Untrusted Input - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary

`ProofOfInclusion::valid()` validates only the internal hash chain from `node_hash` up to the root, but never verifies that `node_hash` actually encodes the specific key being claimed. Because `ProofOfInclusion` is constructable from Python with arbitrary field values (via `pyclass(from_py_object)`), an attacker can supply a structurally valid proof whose `node_hash` belongs to a completely different leaf (or even an internal node), causing `valid()` to return `true` while proving nothing about the claimed key. This is the direct analog of the external report's "hostname-only" partial-identifier bypass: the check validates one dimension of identity (the hash chain) while ignoring the binding between that hash and the claimed data item.

### Finding Description

**Root cause — `valid()` is a partial check:**

```rust
// crates/chia-datalayer/src/merkle/proof_of_inclusion.rs  lines 40-58
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;          // ← taken from untrusted struct field

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }

    existing_hash == self.root_hash()                // ← only proves hash chain is consistent
}
```

The function never checks: *"does `node_hash` equal the hash that was stored for the key this proof claims to cover?"* The struct carries no `key` field at all:

```rust
// lines 26-29

    pub node_hash: Hash,
    pub layers: Vec<ProofOfInclusionLayer>,

```

**Attacker-controlled construction path:**

Both `ProofOfInclusion` and `ProofOfInclusionLayer` carry `pyclass(get_all, from_py_object)`, making them directly constructable from Python with arbitrary bytes:

```rust
// lines 8-11

    feature = "py-bindings",
    pyclass(get_all, from_py_object),
    derive(PyJsonDict, PyStreamable)

```

The Python type stub confirms the public constructor:

```python

```

**Exploit flow:**

A DataLayer peer that knows the current tree root and the hash of any leaf `K1` (hash `H1`) can craft a `ProofOfInclusion` claiming to prove inclusion of a *different* key `K2`:

1. Set `node_hash = H1` (hash legitimately present in the tree for key `K1`).
2. Populate `layers` with the real sibling hashes along the path from `H1` to the root.
3. Send the struct to a verifying client.
4. Client calls `proof.valid()` → returns `True`.
5. Client incorrectly concludes that key `K2` is present in the tree.

The `valid()` function passes because the hash chain from `H1` to the root is internally consistent; it never checks whether `H1` is the hash registered for `K2`.

**Contrast with correct generation:**

`get_proof_of_inclusion(key)` sets `node_hash` to `node.hash` — the externally-supplied hash stored for that key — but `valid()` never re-derives or re-checks this binding:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 1192-1195
Ok(proof_of_inclusion::ProofOfInclusion {
    node_hash: node.hash,   // externally-provided hash, not derived from key
    layers,
})
```

Because the DataLayer leaf hash is an *external commitment* (not `SHA256(key || value)`), there is no cryptographic way for `valid()` to recover the key from `node_hash` without it being included in the struct.

### Impact Explanation

A DataLayer client that calls `proof.valid()` as its sole verification step can be convinced that an arbitrary key is present in a tree when it is not. This lets untrusted input prove invalid state — matching the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."*

Concrete consequences include: a DataLayer application accepting a forged proof of data availability, authorizing state transitions based on data that was never committed, or a light client being deceived about the contents of a DataLayer store.

### Likelihood Explanation

- The `ProofOfInclusion` struct is directly constructable from Python with no privilege required.
- The tree root and all leaf hashes are public in a DataLayer store.
- The `valid()` method name strongly implies a complete validity check; callers are unlikely to add a separate key-to-hash binding check on top of it.
- No cryptographic break is required; the attack is purely structural.

### Recommendation

1. Add a `key` field to `ProofOfInclusion` (or accept the key as a parameter to `valid()`).
2. Inside `valid()`, recompute the expected leaf hash from the key (or compare `node_hash` against the caller-supplied expected hash) before accepting the proof.
3. Alternatively, document explicitly that callers *must* separately assert `proof.node_hash == expected_hash_for_key` after calling `valid()`, and add an assertion or a dedicated `valid_for_key(key, expected_hash)` method to make the two-step check impossible to omit.

### Proof of Concept

```python
from chia_rs.datalayer import (
    MerkleBlob, ProofOfInclusion, ProofOfInclusionLayer, KeyId, ValueId, Side
)

# Build a tree with two keys
blob = MerkleBlob(blob=bytearray())
H1 = bytes([0x01] * 32)
H2 = bytes([0x02] * 32)
blob.batch_insert([((KeyId(1), ValueId(10)), H1),
                   ((KeyId(2), ValueId(20)), H2)])
blob.calculate_lazy_hashes()

# Obtain a legitimate proof for key 1
real_proof = blob.get_proof_of_inclusion(KeyId(1))
assert real_proof.valid()

# Forge a proof claiming key 2 is present, but reusing key 1's node_hash
forged = ProofOfInclusion(
    node_hash=real_proof.node_hash,   # H1 — belongs to key 1, not key 2
    layers=real_proof.layers,
)

# valid() returns True even though this proves nothing about key 2
assert forged.valid(), "Forged proof passes valid() — key-to-hash binding not checked"
print("Forged proof accepted: node_hash =", forged.node_hash.hex())
