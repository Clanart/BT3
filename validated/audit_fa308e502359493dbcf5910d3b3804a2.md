### Title
`ProofOfInclusion::valid()` Is a Self-Referential Tautology — Forged DataLayer Inclusion Proofs Pass Validation Without Matching Any Trusted Root - (File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs)

### Summary

`ProofOfInclusion::valid()` in the DataLayer crate performs only internal self-consistency checks. Its final comparison is a logical tautology: it compares `existing_hash` against `self.root_hash()`, but `root_hash()` is derived from the same proof fields that `existing_hash` was just computed from. The function never verifies the proof against any external, trusted tree root. An attacker who can supply a serialized `ProofOfInclusion` to any consumer that calls only `.valid()` can forge an inclusion proof for an arbitrary key-value pair and have it accepted as valid.

### Finding Description

`root_hash()` returns the `combined_hash` of the last layer in the proof:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← taken directly from the proof itself
    } else {
        self.node_hash
    }
}
```

`valid()` iterates the layers, verifying that each `calculated_hash == layer.combined_hash`, then sets `existing_hash = calculated_hash`. After the loop, `existing_hash` is exactly `last_layer.combined_hash`. The final guard is:

```rust
existing_hash == self.root_hash()
// expands to:
last_layer.combined_hash == last_layer.combined_hash   // always true
```

This means `valid()` returns `true` for **any** proof whose layers are internally chained, regardless of whether the resulting root matches the actual committed tree root. An attacker constructs:

1. A chosen `node_hash` (the leaf they wish to falsely prove is included).
2. A chain of `ProofOfInclusionLayer` values where each `combined_hash` equals `calculate_internal_hash(prev_hash, side, other_hash)` — trivially satisfiable by choosing arbitrary `other_hash` values and computing `combined_hash` forward.

The resulting `ProofOfInclusion` serializes cleanly as a `Streamable` object, deserializes at the receiver, and passes `.valid()` unconditionally.

### Impact Explanation

Any DataLayer consumer that deserializes a `ProofOfInclusion` from an untrusted peer and calls only `.valid()` — without separately comparing `.root_hash()` against a locally-trusted tree root — will accept the forged proof. This allows an attacker to assert that an arbitrary key-value mapping exists in a DataLayer store when it does not, corrupting the verified state visible to that node. Because `ProofOfInclusion` is a `Streamable` type exposed through the Python wheel (`from_bytes`, `valid()`, `root_hash()` are all public API), the attack surface spans every Python-layer DataLayer client.

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

### Likelihood Explanation

The `ProofOfInclusion` struct is `Streamable` and fully exposed over the Python API. Any peer-to-peer DataLayer proof exchange that relies on `.valid()` as the sole gate is vulnerable. Constructing a valid-looking forged proof requires only forward hash computation — no preimage attacks, no key material. Likelihood is high wherever proofs cross a trust boundary.

### Recommendation

`valid()` must accept an external trusted root and compare against it, or the final line must be changed to compare against a caller-supplied root rather than `self.root_hash()`. The minimal fix:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == trusted_root   // compare against external root, not self
}
```

All call sites that currently call `.valid()` on proofs received from untrusted sources must be updated to supply the locally-committed root.

### Proof of Concept

**Step 1 — Choose a target leaf to falsely prove:**
```
node_hash = sha256("fake_key" || "fake_value")   # any 32-byte value
```

**Step 2 — Build one internally-consistent layer:**
```
other_hash = [0u8; 32]                           # arbitrary
other_hash_side = Side::Left
combined_hash = calculate_internal_hash(&node_hash, Side::Left, &other_hash)
```

**Step 3 — Construct and serialize the proof:**
```rust
let forged = ProofOfInclusion {
    node_hash,
    layers: vec![ProofOfInclusionLayer { other_hash_side, other_hash, combined_hash }],
};
let bytes = forged.to_bytes();
```

**Step 4 — Deserialize at the victim and call `valid()`:**
```python
proof = ProofOfInclusion.from_bytes(bytes)
assert proof.valid()   # returns True — forged proof accepted
# proof.root_hash() returns attacker-chosen combined_hash, not the real tree root
```

The victim node now believes the attacker-chosen key-value pair is committed in the DataLayer store.
