### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS Cairo program accepts an arbitrary `class_hash` from the caller without verifying that the hash corresponds to a previously declared class. A contract can replace its own class with any felt value, including one that has never been declared. After the replacement, any subsequent call to that contract causes the OS to look up a compiled class fact that does not exist, making the contract permanently inaccessible and freezing all funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the requested class hash directly from the syscall request and writes it into `contract_state_changes` without any membership check against the set of declared classes:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment in the code explicitly acknowledges the missing check. [1](#0-0) 

When the contract is subsequently called, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [2](#0-1) 

If `class_hash` was replaced with an undeclared value, `dict_read` returns 0 (the uninitialized default), and `find_element` panics because no compiled class fact with hash 0 exists. This causes the entire OS block execution to fail for any block that includes a call to the affected contract.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with an undeclared value:

1. The state change is committed to the Starknet state trie (the `replace_class` call itself succeeds and is not reverted).
2. Every subsequent transaction that calls the contract causes `find_element` to panic inside `execute_entry_point`, making the block proof invalid.
3. The sequencer is forced to permanently exclude all transactions targeting that contract.
4. All ERC-20 tokens, ETH, STRK, or any other assets held in the contract's storage become permanently inaccessible — a direct, irreversible loss of user funds.

---

### Likelihood Explanation

**Medium.** The `replace_class` syscall is a standard, publicly documented StarkNet syscall available to any Sierra contract. An attacker can:

- Deploy a contract whose constructor or any publicly callable function invokes `replace_class` with an arbitrary felt (e.g., `1`).
- Lure users into depositing funds into the contract before triggering the replacement.
- Call the function to freeze all deposited funds.

No privileged role, leaked key, or operator cooperation is required. The attacker only needs to be a contract deployer and a transaction sender — both are unprivileged roles.

---

### Recommendation

Add a Cairo-level assertion inside `execute_replace_class` that verifies the requested `class_hash` exists in `contract_class_changes` (i.e., has a non-zero compiled class hash mapping). This is the check the TODO comment already identifies as missing:

```cairo
// Verify the class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the validation already performed implicitly in `execute_entry_point` but must be enforced eagerly at the point of replacement, not lazily at the point of invocation.

---

### Proof of Concept

1. **Declare and deploy** a Sierra contract `Trap` whose `__execute__` function calls `replace_class(class_hash=1)` (an undeclared hash) and then returns normally.
2. **Lure victims** into depositing tokens into `Trap`.
3. **Send a transaction** invoking `Trap.__execute__`. The OS processes `execute_replace_class`:
   - `request.class_hash = 1` is written to `contract_state_changes` with no validation. [3](#0-2) 
   - The syscall returns success; the transaction completes; the block is proven.
4. **In the next block**, any transaction calling `Trap` causes `execute_entry_point` to call `dict_read` for class hash `1`, get compiled hash `0`, then call `find_element` with key `0`. [2](#0-1) 
5. `find_element` panics — the block proof fails. The sequencer must drop all transactions targeting `Trap`.
6. All victim funds are permanently frozen.
