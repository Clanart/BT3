### Title
Unclamped `origin_decimals` in `DeployTokenResponse` Causes u128 Overflow/Division-by-Zero in NEAR Amount Normalization — (`solana/programs/bridge_token_factory/src/instructions/user/deploy_token.rs`)

---

### Summary

The Solana `initialize_token_metadata` function captures `origin_decimals` from the raw MPC-signed payload **before** clamping, then emits a `DeployTokenResponse` with `decimals=9` (clamped) and `origin_decimals=255` (raw). NEAR's `bind_token` stores this pair verbatim. Every subsequent call to `denormalize_amount` or `normalize_amount` for this token computes `10_u128.pow(255 - 9)` = `10_u128.pow(246)`, which wraps to **0** in Rust release mode (since `10^246 mod 2^128 = 0`). This causes `denormalize_amount` to silently return 0 for all transfer amounts, and `normalize_amount` to panic with division-by-zero. The entire attack is executable by an unprivileged user who deploys a NEAR token with `decimals=255`.

---

### Finding Description

**Step 1 — Solana `initialize_token_metadata` splits decimals without validation** [1](#0-0) 

`origin_decimals` is captured from the raw payload (e.g., 255), then `metadata.decimals` is clamped to `MAX_ALLOWED_DECIMALS = 9`. [2](#0-1) 

**Step 2 — The response posts the divergent pair via Wormhole** [3](#0-2) 

`DeployTokenResponse { decimals: 9, origin_decimals: 255 }` is serialized and posted. [4](#0-3) 

**Step 3 — NEAR `bind_token_callback` stores the pair without any bounds check** [5](#0-4) 

`add_token` stores `Decimals { decimals: 9, origin_decimals: 255 }` permanently. [6](#0-5) 

**Step 4 — Amount arithmetic overflows** [7](#0-6) 

With `diff_decimals = 255 - 9 = 246`:

- `10_u128.pow(246)`: since `10 = 2 × 5`, `10^246 = 2^246 × 5^246`. In `u128` arithmetic (mod 2^128), because `246 > 128`, `2^246 mod 2^128 = 0`, so **`10_u128.pow(246)` wraps to 0** in Rust release mode.
- `denormalize_amount`: `amount * 0 = 0` — every incoming Solana→NEAR transfer mints **0 tokens** to the recipient (silent fund loss).
- `normalize_amount`: `amount / 0` — **panics** (division by zero always panics in Rust even in release mode) — every outgoing NEAR→Solana transfer is permanently blocked.

These functions are called in `fin_transfer_callback`, `sign_transfer`, `fast_fin_transfer`, and `claim_fee_callback`: [8](#0-7) [9](#0-8) 

**Step 5 — The MPC signature precondition is trivially satisfiable**

`log_metadata` on NEAR is a public, unpermissioned call. It fetches `ft_metadata()` from any NEAR token and requests the MPC to sign the result with no decimal validation: [10](#0-9) 

An attacker deploys a NEAR NEP-141 token returning `decimals=255` from `ft_metadata`, calls `log_metadata`, and receives a valid MPC-signed `MetadataPayload { decimals: 255 }`. This is the exact payload consumed by Solana's `deploy_token`.

---

### Impact Explanation

For any token registered via this path with `origin_decimals > 38` (where `10^(origin_decimals - decimals)` exceeds `u128::MAX ≈ 3.4 × 10^38`):

- **Solana → NEAR transfers**: `denormalize_amount` returns 0; users deposit tokens on Solana and receive nothing on NEAR. Funds are locked in the Solana vault with no recovery path.
- **NEAR → Solana transfers**: `normalize_amount` panics; `sign_transfer` always reverts, permanently blocking withdrawals.
- The `Decimals` mapping is write-once (guarded by `BridgeError::TokenExists`), so the corruption is **irreversible** without a contract upgrade.

This matches: **High — Balance, decimal, fee, token-mapping, or accounting corruption that breaks bridge collateralization or misdirects value.**

---

### Likelihood Explanation

The attack requires no privileged access:
1. Deploy a NEAR NEP-141 token with `decimals=255` — trivial.
2. Call `log_metadata(token_id)` on the NEAR bridge — public, no role required.
3. Submit the MPC-signed payload to Solana `deploy_token` — public instruction.
4. Call NEAR `bind_token` with the resulting Wormhole proof — public, no role required.

The only cost is gas and storage deposit. The attack is fully self-contained and locally reproducible.

---

### Recommendation

1. **Validate the decimal gap before storing**: In `bind_token_callback` (or in `add_token`), assert `origin_decimals >= decimals` and `origin_decimals - decimals <= 38` (since `10^39 > u128::MAX`).
2. **Use `checked_pow` / `checked_mul`**: Replace bare `10_u128.pow(diff_decimals)` and `amount * ...` with checked arithmetic that returns an error instead of wrapping or panicking.
3. **Clamp `origin_decimals` on the Solana side**: If `metadata.decimals` was clamped, set `origin_decimals = metadata.decimals` (i.e., the already-clamped value) so the gap is always 0 for tokens whose native decimals exceed `MAX_ALLOWED_DECIMALS`, or enforce a maximum gap.

---

### Proof of Concept

```rust
// Rust pseudocode — locally reproducible
let decimals = Decimals { decimals: 9, origin_decimals: 255 };
let diff: u32 = (decimals.origin_decimals - decimals.decimals).into(); // 246

// In release mode: wraps to 0 (2^246 mod 2^128 = 0)
let factor = 10_u128.pow(diff);
assert_eq!(factor, 0); // true in release mode

// denormalize_amount: silent fund loss
let amount: u128 = 1_000_000_000; // 1 SOL-side token
let denormalized = amount * factor; // 0 — user receives nothing
assert_eq!(denormalized, 0);

// normalize_amount: division by zero panic
// let _ = amount / factor; // panics: attempt to divide by zero
```

Fuzz `origin_decimals` from 39 to 255 with any `decimals <= 9`; for all values where `diff >= 39`, `10_u128.pow(diff)` overflows u128 and the invariant `denormalize(normalize(x)) == x` breaks catastrophically.

pub struct DeployTokenResponse {
    pub token: String,
    pub solana_mint: Pubkey,
    pub decimals: u8,
    pub origin_decimals: u8,
}

impl Payload for DeployTokenResponse {
    type AdditionalParams = ();

    fn serialize_for_near(&self, _params: Self::AdditionalParams) -> Result<Vec<u8>> {
        let mut writer = BufWriter::new(Vec::with_capacity(DEFAULT_SERIALIZER_CAPACITY));
        // 0. OutgoingMessageType::DeployToken
        OutgoingMessageType::DeployToken.serialize(&mut writer)?;
        // 1. token
        self.token.serialize(&mut writer)?;
        // 2. solana_mint
        writer.write_all(&[SOLANA_OMNI_BRIDGE_CHAIN_ID])?;
        self.solana_mint.serialize(&mut writer)?;
        // 3. decimals
        self.decimals.serialize(&mut writer)?;
        // 4. origin_decimals
        self.origin_decimals.serialize(&mut writer)?;

        writer
            .into_inner()
            .map_err(|_| error!(ErrorCode::InvalidArgs))
    }

    fn denormalize_amount(amount: u128, decimals: Decimals) -> u128 {
        let diff_decimals: u32 = (decimals.origin_decimals - decimals.decimals).into();
        amount * (10_u128.pow(diff_decimals))
    }

    /// Uses floor division — any sub-unit remainder ("dust") is truncated and not transferred

    fn normalize_amount(amount: u128, decimals: Decimals) -> u128 {
