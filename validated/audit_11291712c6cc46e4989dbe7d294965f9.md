### Title
Missing Existence Validation on `replace_class` Class Hash Allows Permanent Contract Bricking - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary
The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary felt value as the new class hash without verifying that the hash corresponds to a declared contract class. A contract can replace its own class hash with an undeclared or zero value, permanently bricking itself and freezing any funds it holds. The OS code itself contains a `TODO` comment acknowledging this missing check.

---

### Finding Description

In `execute_replace_class` (`syscall_impls.cairo`, lines 878–916), after deducting gas, the OS reads `class_hash` directly from the syscall request and writes it into `contract_state_changes` without any validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

There are two missing checks:
1. **No zero-check**: `class_hash` is not asserted to be non-zero.
2. **No existence check**: There is no verification that `class_hash` is a key present in `contract_class_changes` (i.e., that a compiled class with that hash was declared in the current block or any prior block).

The TODO comment at line 898 explicitly acknowledges this gap. [2](#0-1) 

When a subsequent transaction calls the bricked contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // now an undeclared hash
);
// compiled_class_hash == 0 (default for missing key)
let (compiled_class_fact: CompiledClassFact*) = find_element(
    ...
    key=compiled_class_hash,           // find_element(key=0) → panic
);
``` [3](#0-2) 

`find_element` panics when the key is absent, making the contract permanently uncallable. The sequencer must exclude any transaction targeting the bricked contract, so funds held there are irrecoverable.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any ERC-20 tokens, ETH, or other assets held in a contract whose class hash has been replaced with an undeclared value are permanently inaccessible. No valid transaction can ever call the contract again because the OS will always panic when attempting to resolve the compiled class. The state change is committed on-chain and irreversible.

---

### Likelihood Explanation

**Low.**

Triggering this requires a contract to call `replace_class` with an invalid hash — either through a deliberate attack (a malicious contract designed to self-destruct after draining or trapping funds) or through a fat-finger/misconfiguration error in a contract's upgrade logic. The syscall is available to any deployed contract without privilege restrictions, making the attack surface reachable by any unprivileged contract deployer or user who can invoke a vulnerable contract.

---

### Recommendation

Add the following validations inside `execute_replace_class` before writing to `contract_state_changes`:

1. **Non-zero check**: `assert_not_zero(class_hash)` — reject class hash of 0.
2. **Existence check**: Verify that `class_hash` is present as a key in `contract_class_changes` (i.e., it was declared in this block or a prior block). This is exactly what the existing TODO comment requests.

The analogous pattern already exists for `execute_declare_transaction`, which enforces `assert_not_zero(compiled_class_hash)` before writing to `contract_class_changes`. [4](#0-3) 

---

### Proof of Concept

1. **Attacker deploys** a contract `VictimVault` holding user funds, whose class implements `replace_class`.
2. **Attacker (or a bug) triggers** a call that executes `replace_class(0x1337dead...)` where `0x1337dead...` is not a declared class hash.
3. **OS processes** `execute_replace_class`: gas is deducted, no existence check is performed, `contract_state_changes[VictimVault].class_hash = 0x1337dead...` is committed.
4. **In the next block**, any transaction calling `VictimVault` reaches `execute_entry_point`, which calls `dict_read(contract_class_changes, 0x1337dead...)` → returns 0, then `find_element(..., key=0)` → OS panics.
5. **Sequencer excludes** all transactions targeting `VictimVault`. Funds are permanently frozen. [5](#0-4) [6](#0-5)

    alloc_locals;

    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
