### Title
`starknet_getStateUpdate` Returns State Diff Inconsistent with `state_diff_commitment` in Block Hash — (`crates/apollo_rpc/src/v0_8/api/api_impl.rs`)

---

### Summary

`starknet_getStateUpdate` silently strips empty storage-diff entries from the `ThinStateDiff` it returns, while the `state_diff_commitment` baked into the block hash was computed from the **full** diff (including those empty entries). Any client or proof system that uses the RPC response to reconstruct or verify the block hash will compute a wrong `state_diff_commitment` and therefore a wrong block hash — an authoritative-looking wrong value from the RPC.

---

### Finding Description

In `get_state_update`, after reading the stored `ThinStateDiff`, the handler calls:

```rust
thin_state_diff.storage_diffs.retain(|_k, v| !v.is_empty());
```

The inline comment explicitly acknowledges the invariant this breaks:

> *"Some blocks contain empty storage diffs that must be kept for the computation of state diff commitment."* [1](#0-0) 

The `state_diff_commitment` that is chained into the block hash is computed by `calculate_state_diff_hash`, which iterates over `state_diff.storage_diffs` — including any contract entries whose inner map is empty: [2](#0-1) 

That commitment is then chained into `calculate_block_hash` as a first-class field: [3](#0-2) 

The commitment is also stored in `StorageBlockHeader.state_diff_commitment` and in `PartialBlockHashComponents.header_commitments.state_diff_commitment`, which is the value the batcher writes to storage and later uses to finalize the block hash: [4](#0-3) [5](#0-4) 

The commitment is computed at block-build time from the **unstripped** diff: [6](#0-5) 

So the pipeline is:

```
Full ThinStateDiff (with empty entries)
  → calculate_state_diff_hash()  → state_diff_commitment  → block_hash  ✓
  → RPC retain() strips empties  → returned state_diff                  ✗
```

The two outputs are structurally different. A client that calls `starknet_getStateUpdate`, recomputes `calculate_state_diff_hash` over the returned diff, and then tries to verify the block hash will obtain a mismatched commitment and therefore a wrong block hash.

---

### Impact Explanation

**High — RPC returns an authoritative-looking wrong value.**

`starknet_getStateUpdate` is the canonical source of truth for a block's state changes. The response carries `block_hash` and `new_root` alongside `state_diff`, implying the three are mutually consistent. They are not: the `state_diff` field is missing entries that were used to derive `block_hash`. Any downstream consumer that:

- reconstructs the `state_diff_commitment` to verify the block hash,
- feeds the RPC-returned diff into a proof system that checks the commitment, or
- uses the diff to reproduce the Patricia-trie root,

will silently compute a wrong value. This is the sequencer analog of the vault user burning tokens and receiving nothing: the client "commits" to the RPC's authoritative state diff, but the diff has been silently drained of the entries needed to satisfy the commitment invariant.

---

### Likelihood Explanation

Medium-High. Empty storage-diff entries arise whenever a transaction touches a contract's storage slot and the net effect is zero (write-then-revert, or a slot that was already zero). The comment confirms this occurs in production blocks. Any block containing such a transaction triggers the discrepancy for every caller of `starknet_getStateUpdate`.

---

### Recommendation

Choose one consistent policy and apply it everywhere:

1. **Include empty entries in the RPC response** — remove the `retain` call so the returned `state_diff` matches the committed hash exactly.
2. **Exclude empty entries from the commitment** — strip them before calling `calculate_state_diff_hash` at block-build time, and keep the `retain` in the RPC handler.

Option 1 is safer because it preserves the invariant that `state_diff_commitment == calculate_state_diff_hash(returned_state_diff)` without touching the commitment pipeline.

---

### Proof of Concept

1. Produce a block containing a transaction that writes a storage slot and then reverts (or writes back the original value), leaving an empty inner map for that contract in the stored `ThinStateDiff`.
2. Call `starknet_getStateUpdate` for that block number. The response omits the empty-map entry due to `retain`.
3. Independently fetch the block header (e.g., via `starknet_getBlockWithTxHashes`) and extract `block_hash`.
4. Recompute `calculate_state_diff_hash` over the RPC-returned `state_diff` → obtain commitment **C₁**.
5. Recompute `calculate_state_diff_hash` over the **full** stored diff (including the empty entry) → obtain commitment **C₂**.
6. Reconstruct `calculate_block_hash` using C₁ vs C₂: only C₂ reproduces the real `block_hash`. The RPC-returned diff yields a wrong block hash, confirming the authoritative-looking wrong value. [7](#0-6) [2](#0-1) [8](#0-7)

    async fn get_state_update(&self, block_id: BlockId) -> RpcResult<StateUpdate> {
        let txn = self.storage_reader.begin_ro_txn().map_err(internal_server_error)?;
        if let BlockId::Tag(Tag::Pending) = block_id {
            let state_update = read_pending_data(&self.pending_data, &txn).await?.state_update;
            return Ok(StateUpdate::PendingStateUpdate(PendingStateUpdate {
                old_root: state_update.old_root,
                state_diff: state_update.state_diff.into(),
            }));
        }

        // Get the block header for the block hash and state root.
        let block_number = get_accepted_block_number(&txn, block_id)?;
        let header: BlockHeader = get_block_header_by_number(&txn, block_number)?.into();

        // Get the old root.
        let old_root = match get_accepted_block_number(
            &txn,
            BlockId::HashOrNumber(BlockHashOrNumber::Hash(header.parent_hash)),
        ) {
            Ok(parent_block_number) => {
                BlockHeader::from(get_block_header_by_number(&txn, parent_block_number)?).new_root
            }
            Err(_) => GlobalRoot::ROOT_OF_EMPTY_STATE,
        };

        // Get the block state diff.
        let mut thin_state_diff = txn
            .get_state_diff(block_number)
            .map_err(internal_server_error)?
            .ok_or_else(|| ErrorObjectOwned::from(BLOCK_NOT_FOUND))?;
        // Remove empty storage diffs. Some blocks contain empty storage diffs that must be kept for
        // the computation of state diff commitment.
        thin_state_diff.storage_diffs.retain(|_k, v| !v.is_empty());

        let state_diff =
            self.convert_thin_state_diff(thin_state_diff, block_id, block_number).await?;
        Ok(StateUpdate::AcceptedStateUpdate(AcceptedStateUpdate {
            block_hash: header.block_hash,
            new_root: header.new_root,
            old_root,
            state_diff,
        }))
    }

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
