### Title
Missing Calldata Bounds Validation in `execute_meta_tx_v0` Enables Unprovable Block Inclusion — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_meta_tx_v0` function in the StarkNet OS Cairo program validates the `signature` array bounds but **omits the equivalent validation for `calldata`**. A malicious contract can craft a `META_TX_V0` syscall with `calldata_end < calldata_start`, producing a `calldata_size` that wraps around the field prime. This causes the gas-cost calculation to overflow (felt arithmetic is modular), and the downstream hash computation (`hash_update_with_hashchain`) to iterate over an astronomically large number of elements, making the OS execution non-terminating within any practical step budget. The block containing such a transaction cannot be proven, resulting in a network halt.
