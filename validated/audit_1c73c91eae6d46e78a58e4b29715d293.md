### Title
`batch_insert` Bypasses Duplicate-Key and Duplicate-Hash Validation, Enabling Corrupted DataLayer Merkle Root and Forged Inclusion/Exclusion Proofs — (File: crates/chia-datalayer/src/merkle/blob.rs)

### Summary

`MerkleBlob::batch_insert` omits the uniqueness guards that the single-item `insert` function enforces. When the tree already holds two or more leaves, every item in the batch is written directly to the blob without any duplicate-key or duplicate-hash check. A caller who supplies a batch containing a repeated `KeyId` or `Hash` silently produces a Merkle tree with two leaf nodes sharing the same key, corrupting the committed root hash and enabling forged DataLayer inclusion/exclusion proofs.

### Finding Description

`insert` (line 362) enforces two explicit guards before writing any leaf:

```rust
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`batch_insert` (line 570) takes a structurally different path. When `leaf_count <= 1`, it pops the **last two** items from the input vector and routes them through `insert` (with checks). All remaining items—and **every** item when `leaf_count >= 2`—are written directly via `insert_entry_to_blob` with no uniqueness check at all:

```rust
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ... Node::Leaf(LeafNode { hash, key, value, ... }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;  // ← no duplicate check
    indexes.push(new_leaf_index);
}
``` [2](#0-1) 

`insert_entry_to_blob` then calls `self.block_status_cache.add_leaf(index, leaf)`, which overwrites the cache entry for the key with the new index. The old leaf node remains in the blob at its original slot, unreachable through the cache, while the new leaf occupies a different slot. The tree now contains two physical leaf nodes with the same `KeyId`. [3](#0-2) 

The Python-facing binding `py_batch_insert` passes caller-supplied `keys_values` and `hashes` directly into `batch_insert` with no pre-validation of key uniqueness: [4](#0-3) 

The single-item path correctly rejects duplicates, as confirmed by the existing test: [5](#0-4) 

No equivalent test exists for `batch_insert` with duplicate keys.

### Impact Explanation

The Merkle root is computed bottom-up from all leaf hashes. With two leaves sharing the same `KeyId` (but potentially different hashes), the root hash is derived from an invalid tree structure. Any DataLayer proof of inclusion or exclusion derived from this root is based on corrupted committed state. A verifier that trusts the root will:

- **Accept forged inclusion proofs**: a key can be proven present at two different tree positions simultaneously.
- **Produce false exclusion results**: the orphaned leaf is invisible to the cache, so `get_leaf_by_key` will not find it, making a present key appear absent.

`check_integrity` will also detect the corruption and fail with `IntegrityKeyNotInCache` for the orphaned leaf, confirming committed state corruption. [6](#0-5) 

### Likelihood Explanation

`batch_insert` is the primary bulk-load path for the DataLayer and is called directly from the Python wheel. The DataLayer node software in `chia-blockchain` calls `py_batch_insert` when applying store updates submitted by users. Because `batch_insert` itself performs no uniqueness enforcement for the fast path, any caller that does not pre-deduplicate its input—or any attacker who controls the batch contents—can trigger this silently. The trigger condition (tree has ≥ 2 leaves, which is the normal operating state) is always satisfied in production.

### Recommendation

Add the same duplicate-key and duplicate-hash checks to the fast path of `batch_insert` that `insert` already performs. Before calling `insert_entry_to_blob` for each leaf, check `self.block_status_cache.contains_key(key)` and `self.block_status_cache.contains_leaf_hash(&hash)` and return the appropriate error. Alternatively, accumulate a local `HashSet` of keys and hashes seen within the batch itself to catch intra-batch duplicates before any blob mutation occurs.

### Proof of Concept

```rust
let mut blob = MerkleBlob::new(vec![]).unwrap();

// Pre-populate with 2 leaves so leaf_count > 1,
// bypassing the `if self.block_status_cache.leaf_count() <= 1` guard entirely.
blob.insert(KeyId(100), ValueId(100), &sha256_num(&100i64), InsertLocation::Auto {}).unwrap();
blob.insert(KeyId(200), ValueId(200), &sha256_num(&200i64), InsertLocation::Auto {}).unwrap();

// batch_insert with a key already present in the tree — no error is returned.
blob.batch_insert(vec![
    ((KeyId(100), ValueId(999)), sha256_num(&999i64)), // duplicate KeyId(100)
]).unwrap(); // succeeds silently

// The tree now has two leaf nodes with KeyId(100), corrupting the root hash.
blob.calculate_lazy_hashes().unwrap();
// blob.check_integrity() → Err(IntegrityKeyNotInCache(...))
// Any proof derived from this root is invalid.
```

    fn check_just_integrity(&self) -> Result<(), Error> {
        let mut leaf_count: usize = 0;
        let mut internal_count: usize = 0;
        let mut child_to_parent: HashMap<TreeIndex, TreeIndex> = HashMap::new();

        for item in ParentFirstIterator::new(&self.blob, None) {
            let (index, block) = item?;
            if let Some(parent) = block.node.parent().0 {
                if child_to_parent.remove(&index) != Some(parent) {
                    return Err(Error::IntegrityParentChildMismatch(index));
                }
            }
            match block.node {
                Node::Internal(node) => {
                    internal_count += 1;
                    child_to_parent.insert(node.left, index);
                    child_to_parent.insert(node.right, index);
                }
                Node::Leaf(node) => {
                    leaf_count += 1;
                    let cached_index = self
                        .block_status_cache
                        .get_index_by_key(node.key)
                        .ok_or(Error::IntegrityKeyNotInCache(node.key))?;
                    if *cached_index != index {
                        return Err(Error::IntegrityKeyToIndexCacheIndex(
                            node.key,
                            index,
                            *cached_index,
                        ));
                    }
                    assert!(
                        !self.block_status_cache.is_index_free(index),
                        "{}",
                        format!("active index found in free index list: {index:?}")
                    );
                }
            }
        }

        let key_to_index_cache_length = self.block_status_cache.key_to_index.len();
        if leaf_count != key_to_index_cache_length {
            return Err(Error::IntegrityKeyToIndexCacheLength(
                leaf_count,
                key_to_index_cache_length,
            ));
        }
        let leaf_hash_to_index_cache_length = self.block_status_cache.leaf_hash_to_index.len();
        if leaf_count != leaf_hash_to_index_cache_length {
            return Err(Error::IntegrityLeafHashToIndexCacheLength(
                leaf_count,
                leaf_hash_to_index_cache_length,
            ));
        }
        let total_count = leaf_count + internal_count + self.block_status_cache.free_index_count();
        let extend_index = self.extend_index();
        if total_count != extend_index.0 as usize {
            return Err(Error::IntegrityTotalNodeCount(extend_index, total_count));
        }
        if !child_to_parent.is_empty() {
            return Err(Error::IntegrityUnmatchedChildParentRelationships(
                child_to_parent.len(),
            ));
        }

        Ok(())
    }

    fn insert_entry_to_blob(&mut self, index: TreeIndex, block: &Block) -> Result<(), Error> {
        let new_block_bytes = block.to_bytes()?;
        let extend_index = self.extend_index();
        match index.cmp(&extend_index) {
            Ordering::Greater => return Err(Error::BlockIndexOutOfBounds(index)),
            Ordering::Equal => self.blob.extend_from_slice(&new_block_bytes),
            Ordering::Less => {
                self.blob[block_range(index)].copy_from_slice(&new_block_bytes);
            }
        }

        match block.node {
            Node::Leaf(leaf) => self.block_status_cache.add_leaf(index, leaf),
            Node::Internal(..) => self.block_status_cache.add_internal(index),
        }

        Ok(())
    }
