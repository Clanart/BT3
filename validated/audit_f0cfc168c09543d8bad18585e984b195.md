### Title
Forged Internal Node Hash Accepted Without Verification in `inner_build_blob_from_node_list` — (`crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::inner_build_blob_from_node_list` blindly stores the map-key hash as the `InternalNode.hash` field without ever computing or comparing `internal_hash(left_child_hash, right_child_hash)`. Because the node is written with `dirty: false`, neither `calculate_lazy_hashes` nor `check_just_integrity` will recompute or validate it. Any caller — including the Python-exposed `DeltaReader.add_internal_nodes()` — can inject an arbitrary root hash into a committed DataLayer Merkle blob.

---

### Finding Description

**Root cause — `inner_build_blob_from_node_list` (blob.rs)**

When processing an `Internal` node from the `nodes` map, the function uses `node_hash` (the map key supplied by the caller) verbatim as the stored hash: [1](#0-0) 

No call to `internal_hash(left_child_hash, right_child_hash)` is made; the computed left/right child hashes are never compared against `node_hash`. The block is also written with `dirty: false`: [2](#0-1) 

**Entry point — `py_add_internal_nodes` (deltas.rs)**

The Python binding inserts caller-supplied `(hash → (left, right))` pairs directly into `self.nodes` with zero validation: [3](#0-2) 

The `InternalNodesMap` type is `HashMap<Hash, (Hash, Hash)>`, so any 32-byte value is a valid key.

**No detection by `check_integrity`**

`check_just_integrity` verifies parent-child index linkage and cache consistency, but never checks that an internal node's stored hash equals `internal_hash(left_child_hash, right_child_hash)`: [4](#0-3) 

`calculate_lazy_hashes` only recomputes hashes for nodes where `block.metadata.dirty == true`. Because the forged node is inserted with `dirty: false`, it is silently skipped: [5](#0-4) 

`check_integrity` therefore passes on a blob whose root hash is entirely attacker-controlled: [6](#0-5) 

---

### Impact Explanation

A DataLayer node that syncs tree state from peers calls `add_internal_nodes` with data received over the network, then calls `create_merkle_blob_and_filter_unused_nodes` to materialise the blob: [7](#0-6) 

After this call, `get_root_hash()` returns the forged value and `check_integrity()` raises no error. The committed DataLayer tree root is corrupted: inclusion/exclusion proofs derived from it reference a hash that does not correspond to the actual leaf contents, enabling a malicious peer to make a victim node accept and store an invalid tree state.

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."*

---

### Likelihood Explanation

The Python binding `add_internal_nodes` is the documented interface for feeding delta data into a `DeltaReader`. Any DataLayer peer that participates in the sync protocol can supply the `internal_nodes` map. No privilege, key material, or local access is required beyond being a reachable peer. The forged blob passes `check_integrity`, so no existing runtime guard catches the corruption.

---

### Recommendation

In `inner_build_blob_from_node_list`, after recursing into both children and obtaining their actual stored hashes, compute the expected hash and reject any mismatch:

```rust
// after left_index and right_index are resolved:
let left_hash  = self.get_hash(left_index)?;
let right_hash = self.get_hash(right_index)?;
let expected   = internal_hash(&left_hash, &right_hash);
if expected != node_hash {
    return Err(Error::InternalHashMismatch(node_hash, expected));
}
```

Alternatively, discard `node_hash` entirely for internal nodes and always derive the hash from the children, consistent with how every other insertion path in `MerkleBlob` operates (e.g., `insert_second`, `batch_insert`). [8](#0-7) 

---

### Proof of Concept

```python
from chia_rs import DeltaReader, MerkleBlob
from hashlib import sha256

# Arbitrary leaf hashes
left_hash  = bytes([0x01] * 32)
right_hash = bytes([0x02] * 32)

# Forged root: does NOT equal internal_hash(left, right)
forged_root = bytes([0xFF] * 32)

# Real internal_hash would be sha256(b"\x02" + left_hash + right_hash)
real_root = sha256(b"\x02" + left_hash + right_hash).digest()
assert forged_root != real_root  # confirm the forgery

# Build a DeltaReader with forged internal node and two valid leaves
reader = DeltaReader(
    internal_nodes={forged_root: (left_hash, right_hash)},
    leaf_nodes={
        left_hash:  (1, 10),   # key=1, value=10
        right_hash: (2, 20),   # key=2, value=20
    },
)

blob = reader.create_merkle_blob_and_filter_unused_nodes(
    forged_root, set()
)

# get_root_hash() returns the forged value
assert blob.get_root_hash() == forged_root

# check_integrity() raises no error despite the hash mismatch
blob.check_integrity()   # passes silently

print("Forged root accepted:", blob.get_root_hash().hex())
```
