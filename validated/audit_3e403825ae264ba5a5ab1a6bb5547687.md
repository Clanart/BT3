Unrestricted `execute_meta_tx_v0` Syscall Bypasses `__validate__` and Enables Arbitrary Execution on Any Account — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary

The `execute_meta_tx_v0` function in `syscall_impls.cairo` allows **any executing contract** to invoke `__execute__` on an arbitrary target account with fully attacker-controlled calldata and signature, while completely bypassing the `__validate__` entry point. Because standard StarkNet account contracts perform signature verification exclusively in `__validate__` — not in `__execute__` — this creates a direct path for an unprivileged attacker to execute arbitrary calls from any victim account, leading to direct loss of funds.

The only guard present is a selector check: [2](#0-1) 

This enforces that the target selector must be `EXECUTE_ENTRY_POINT_SELECTOR` (`__execute__`), but imposes **no restriction** on:
- Which contract may call this syscall (any caller is accepted)
- Which target `contract_address` may be specified (any account is targetable)
- What `calldata` is provided (fully attacker-controlled)
- What `signature` is provided (fully attacker-controlled)

The syscall then constructs a synthetic `TxInfo` with `version=0`, `nonce=0`, `max_fee=0`, and the attacker-supplied signature: [3](#0-2)
