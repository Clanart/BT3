### Title
Missing Declared Class Hash Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall in the StarkNet OS does not verify that the new class hash supplied by a contract has been previously declared (i.e., exists in `contract_class_changes`). Every other code path that uses a class hash — specifically `execute_entry_point` — implicitly enforces this invariant by looking up the compiled class hash and panicking if it is absent. `execute_replace_class` bypasses this check entirely, allowing any unprivileged contract to permanently freeze itself (and any funds it holds) by replacing its class with an undeclared hash. The missing check is even acknowledged by a TODO comment in the source.

---

### Finding Description

The StarkNet OS enforces a two-step lifecycle for contract classes:

**Step 1 — Declaration.** `execute_declare_transaction` writes the mapping `class_hash → compiled_class_hash` into `contract_class_changes`: [1](#0-0) 

**Step 2 — Execution.** `execute_entry_point` reads the contract's current `class_hash` from `contract_state_changes`, then looks up its `compiled_class_hash` from `contract_class_changes`. If the class was never declared, `dict_read` returns `0`, and the subsequent `find_element` call panics (Cairo assertion failure), making the contract permanently uncallable: [2](#0-1) 

**The gap — `execute_replace_class`.** This syscall directly overwrites the contract's `class_hash` in `contract_state_changes` with whatever value the calling contract supplies, with **no check** that the new hash exists in `contract_class_changes`. The TODO comment at line 898 explicitly acknowledges the missing validation: [3](#0-2) 

The critical lines:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The OS accepts the syscall and commits the state change. No lookup into `contract_class_changes` is performed at this point.

**Contrast with `deploy_contract`.** When a contract is deployed, its constructor is immediately invoked via `select_execute_entry_point_func`, which internally calls `execute_entry_point`. This means an undeclared class hash would cause an immediate failure during deployment — the class hash is implicitly validated. `replace_class` has no such implicit validation because no entry point is invoked at the time of the call. [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's `class_hash` is set to an undeclared hash via `replace_class`, every subsequent call to that contract reaches `execute_entry_point`, which performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // undeclared hash → returns 0
);
// find_element with key=0 panics if no compiled class fact has hash 0
let (compiled_class_fact: CompiledClassFact*) = find_element(..., key=compiled_class_hash);
```

The contract becomes permanently uncallable. Any ERC-20 tokens, ETH, or other assets held in the contract's storage are irreversibly frozen. There is no recovery path because the contract cannot execute any entry point (including one that would call `replace_class` again to fix the hash).

---

### Likelihood Explanation

The attack surface is broad:

- **Any unprivileged contract** can issue the `replace_class` syscall — it requires no special role or privilege.
- A malicious contract deployer can intentionally deploy a contract that calls `replace_class` with an arbitrary undeclared hash, then fund it (or trick users into funding it) before triggering the freeze.
- A buggy contract (e.g., one that computes a class hash incorrectly) can trigger this accidentally, permanently locking user funds.
- The `execute_syscalls` dispatcher routes `REPLACE_CLASS_SELECTOR` directly to `execute_replace_class` with no pre-flight checks: [5](#0-4) 

---

### Recommendation

Add a validation step inside `execute_replace_class` to confirm that the requested `class_hash` has a non-zero entry in `contract_class_changes` before committing the state update:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,   // add this implicit arg
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // ADD: Verify the class hash has been declared.
    let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
    with_attr error_message("Class hash is not declared.") {
        assert_not_zero(compiled_class_hash);
    }

    // existing state update follows ...
}
```

This mirrors the implicit validation already present in `execute_entry_point` and closes the bypass.

---

### Proof of Concept

1. Attacker deploys contract `C` (with a valid declared class hash) and funds it with tokens.
2. Attacker calls contract `C`, which internally executes:
   ```
   replace_class(class_hash=0xDEAD)   // 0xDEAD is never declared
   ```
3. The OS processes `execute_replace_class`: no declared-class check is performed; `contract_state_changes[C].class_hash` is set to `0xDEAD`.
4. The transaction succeeds and is included in the block.
5. Any subsequent call to contract `C` reaches `execute_entry_point`:
   - `dict_read(contract_class_changes, 0xDEAD)` → returns `0` (undeclared)
   - `find_element(..., key=0)` → panics (no compiled class fact with hash `0`)
6. Contract `C` is permanently uncallable. All funds held in its storage are permanently frozen.

    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
