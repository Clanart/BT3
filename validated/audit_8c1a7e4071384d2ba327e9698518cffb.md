### Title
Path Traversal via `ExpandHome` Allows Arbitrary File Write Through Public `debug_startGoTrace` / `debug_startCPUProfile` RPC — (`rpc/namespaces/ethereum/debug/utils.go`)

### Summary

`ExpandHome` expands `~/` to the user's home directory but applies no containment check. `filepath.Clean` normalizes traversal sequences after expansion, so `~/../../../etc/passwd` resolves to `/etc/passwd`. The only guard (`restrict-user-input`) defaults to `false`, leaving the traversal fully reachable when the `debug` namespace is enabled.

### Finding Description

`ExpandHome` (utils.go:44–53) replaces the leading `~` with `usr.HomeDir` and then calls `filepath.Clean`. It does not verify that the cleaned result is a descendant of `HomeDir`. `validatePath` (utils.go:59–91) calls `ExpandHome`, then `filepath.Abs`, and only applies a containment check when `srvflags.JSONRPCRestrictUserInput` is `true`. The default configuration sets `RestrictUserInput: false` (config.go:287). `restrictedCreate` (utils.go:96–101) then calls `os.Create(fp)` — a truncating open — at the attacker-supplied path.

Every `debug_*` file-writing method (`StartGoTrace`, `StartCPUProfile`, `BlockProfile`, `GoTrace`, `CpuProfile`, `writeProfile`) flows through `validatePath` → `restrictedCreate`, so all are affected.

### Impact Explanation

An RPC caller who can reach the `debug` namespace can create or truncate any file writable by the node process:

- Overwrite the node's private key or config files, causing a validator outage or key compromise.
- Write to `/etc/cron.d/` or similar scheduler directories for OS-level code execution as the node user.
- Truncate the node's data files, corrupting committed state.

This fits the allowed High impact: "Public JSON-RPC path exposes a reachable route to the impacts above."

### Likelihood Explanation

Requires the operator to explicitly enable the `debug` namespace (non-default) and leave `restrict-user-input=false` (default). Many operators enable `debug` for troubleshooting without reading the config comment warning. The RPC endpoint is typically network-accessible. No authentication is required once the namespace is enabled.

### Recommendation

1. **Enforce containment in `ExpandHome` itself** — after `filepath.Clean`, assert `strings.HasPrefix(result, home+string(filepath.Separator))` and return an error if not.
2. **Change the default** of `RestrictUserInput` to `true`.
3. **Reject `..` components before expansion** in `validatePath` as a defense-in-depth measure.

### Proof of Concept

```
# Node running with debug namespace enabled, restrict-user-input=false (default)
curl -X POST http://node:8545 \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"debug_startGoTrace",
       "params":["~/../../../tmp/evil"],"id":1}'

# ExpandHome("~/../../../tmp/evil")
#   → filepath.Clean("/home/nodeuser/../../../tmp/evil")
#   → "/tmp/evil"
# restrictedCreate → os.Create("/tmp/evil")  ← file created outside home
```

Fuzzing `ExpandHome` with `"~/" + strings.Repeat("../", N) + "target"` and asserting the output has prefix `usr.HomeDir+string(filepath.Separator)` will fail for all `N ≥ 1`.

func validatePath(ctx *server.Context, file string) (string, error) {
	fp, err := ExpandHome(file)
	if err != nil {
		return "", err
	}
	fp, err = filepath.Abs(fp)
	if err != nil {
		return "", err
	}
	if ctx.Viper.GetBool(srvflags.JSONRPCRestrictUserInput) {
		absDataDir, err := filepath.Abs(ctx.Config.RootDir)
		if err != nil {
			return "", err
		}
		realDataDir, err := filepath.EvalSymlinks(absDataDir)
		if err != nil {
			return "", err
		}
		// Resolve parent dir symlinks; the file itself may not exist yet.
		realParent, err := filepath.EvalSymlinks(filepath.Dir(fp))
		if err != nil {
			return "", err
		}
		fp = filepath.Join(realParent, filepath.Base(fp))
		if !strings.HasPrefix(fp, realDataDir+string(filepath.Separator)) {
			return "", errors.New("file path must be in the data directory")
		}
		// Reject a pre-existing symlink at the final component.
		if fi, err := os.Lstat(fp); err == nil && fi.Mode()&os.ModeSymlink != 0 {
			return "", errors.New("file path must not be a symlink")
		}
	}
	return fp, nil
