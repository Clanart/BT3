### Title
`ProofOfSpace::update_digest` Panics on Invalid v2 Proof Bytes, Enabling Remote Chain Halt via Crafted Block - (File: `crates/chia-protocol/src/proof_of_space.rs`)

### Summary

`ProofOfSpace::update_digest` — the method used to compute the consensus block hash — calls `self.quality_string().expect(...)` without any prior validation of the proof bytes. Because `Streamable::parse` for v2 proofs accepts any proof byte sequence as long as the structural fields are valid, an attacker can craft a network block containing a v2 `ProofOfSpace` with invalid proof bytes. When any node attempts to compute the block hash (calling `update_digest`), it panics, crashing the node process.

### Finding Description

`ProofOfSpace` is a consensus-critical structure embedded in `RewardChainBlock`. Its `Streamable` implementation provides `update_digest`, which is called to compute the block hash during validation.

For v2 proofs (`version == 1`), `update_digest` calls:

```rust
self.quality_string()
    .expect("internal error. Can't compute hash of invalid ProofOfSpace")
    .update_digest(digest);
``` [1](#0-0) 

`quality_string()` returns `Option<Bytes32>` — it returns `None` when `chia_pos2::quality_string_from_proof` fails on invalid proof bytes:

```rust
pub fn quality_string(&self) -> Option<Bytes32> {
    if self.version != 1 { return None; }
    let k_size = (self.proof.len() * 8 / 128) as u8;
    let plot_id = self.compute_plot_id().to_bytes();
    chia_pos2::quality_string_from_proof(&plot_id, k_size, self.strength, self.proof.as_slice())
        .map(|quality| { ... })
}
``` [2](#0-1) 

The `parse` method for v2 proofs validates only the structural constraint (exactly one of `pool_public_key` / `pool_contract_puzzle_hash` must be set), but **does not validate the proof bytes themselves**:

```rust
} else if version == 1 {
    let plot_index = <u16 as Streamable>::parse::<TRUSTED>(input)?;
    let meta_group  = <u8  as Streamable>::parse::<TRUSTED>(input)?;
    let strength    = <u8  as Streamable>::parse::<TRUSTED>(input)?;
    let proof       = <Bytes as Streamable>::parse::<TRUSTED>(input)?;

    if pool_public_key.is_some() == pool_contract_puzzle_hash.is_some() {
        return Err(Error::InvalidPoS);
    }
    Ok(ProofOfSpace { ..., proof, ... })
}
``` [3](#0-2) 

Therefore, a v2 `ProofOfSpace` with garbage proof bytes parses successfully, but panics when `update_digest` is called. The panic propagates through `RewardChainBlock::update_digest`:

```rust
fn update_digest(&self, digest: &mut Sha256) {
    ...
    self.proof_of_space.update_digest(digest);
    ...
}
``` [4](#0-3) 

### Impact Explanation

This matches the allowed impact: **"Valid unprivileged CLVM program, spend bundle, block, proof, or serialized network object can trigger deterministic consensus divergence, chain halt, or committed state corruption."**

Any node that receives a crafted block containing a v2 `ProofOfSpace` with structurally valid but cryptographically invalid proof bytes will panic when computing the block hash. Depending on the Rust panic handler configuration (`unwind` vs `abort`), this either crashes the node process or terminates the validation thread. Nodes that crash cannot participate in consensus, causing a chain halt. The panic is deterministic — every node receiving the same crafted block will panic at the same point.

### Likelihood Explanation

The attacker only needs to:
1. Construct a `RewardChainBlock` (or any block message containing a `ProofOfSpace`) with `version = 1`, exactly one of `pool_public_key`/`pool_contract_puzzle_hash` set (to pass parse validation), and arbitrary garbage bytes as the proof.
2. Broadcast it to the network.

No privileged access, leaked keys, or governance role is required. The crafted block is accepted by `parse` and the panic is triggered the moment any node calls `update_digest` to compute the block hash.

### Recommendation

Move proof validation into `parse` for v2 proofs. Validate that `quality_string_from_proof` succeeds during deserialization and return `Err(Error::InvalidPoS)` if it does not. Replace the `.expect(...)` in `update_digest` with a graceful error path (e.g., return a sentinel hash or propagate an error through the `Streamable` trait), so that an invalid proof never reaches the panic site.

### Proof of Concept

1. Craft a byte sequence representing a `ProofOfSpace` with:
   - Valid `challenge` (32 bytes)
   - `pool_public_key = None` (prefix byte `0x00`)
   - Prefix byte `0b10` (version=1, no contract)
   - Valid `plot_public_key` (48 bytes)
   - `plot_index`, `meta_group`, `strength` (any values)
   - `proof` = arbitrary garbage bytes (e.g., 128 bytes of `0xFF`)
2. Embed this in a `RewardChainBlock` and send it to a node.
3. The node calls `ProofOfSpace::parse` — succeeds (structural check passes).
4. The node calls `RewardChainBlock::update_digest` to compute the block hash.
5. `ProofOfSpace::update_digest` calls `self.quality_string().expect(...)`.
6. `quality_string_from_proof` returns `None` on the garbage proof bytes.
7. `.expect(...)` panics: `"internal error. Can't compute hash of invalid ProofOfSpace"`.
8. Node process crashes or thread terminates. [5](#0-4)

impl Streamable for ProofOfSpace {
    fn update_digest(&self, digest: &mut Sha256) {
        self.challenge.update_digest(digest);
        self.pool_public_key.update_digest(digest);

        if self.version == 0 {
            self.pool_contract_puzzle_hash.update_digest(digest);
            self.plot_public_key.update_digest(digest);
            self.size.update_digest(digest);
            self.proof.update_digest(digest);
        } else if self.version == 1 {
            if let Some(pool_contract) = self.pool_contract_puzzle_hash {
                0b11_u8.update_digest(digest);
                pool_contract.update_digest(digest);
            } else {
                0b10_u8.update_digest(digest);
            }

            self.plot_public_key.update_digest(digest);
            self.plot_index.update_digest(digest);
            self.meta_group.update_digest(digest);
            self.strength.update_digest(digest);

            // for v2 proofs, we don't hash the full proof directly. The full
            // proof is the witness to this quality string commitment.
            self.quality_string()
                .expect("internal error. Can't compute hash of invalid ProofOfSpace")
                .update_digest(digest);
        } else {
            panic!("version field must be 0 or 1, but it's {}", self.version);
        }
    }
