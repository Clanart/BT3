### Title
Missing Declaration Check for Cairo V0 Classes in `ExecutionStateReader::get_compiled_class` Allows Execution of Undeclared Class at Historical Block - (File: crates/apollo_rpc_execution/src/state_reader.rs)

### Summary

`ExecutionStateReader::get_compiled_class`, when backed by the class manager, verifies that a Cairo V1 class is declared at the queried `state_number` before returning it. The identical check is explicitly skipped for Cairo V0 (deprecated) classes with a `TODO` comment. A class manager that holds a Cairo V0 class declared in a future block will silently return it for any earlier `state_number`, causing RPC execution, fee estimation, and simulation to run the wrong class and return authoritative-looking wrong values.

### Finding Description

In `ExecutionStateReader::get_compiled_class`, when `class_manager_handle` is `Some`, the code fetches the executable class from the class manager and then branches on its version:

```rust
// crates/apollo_rpc_execution/src/state_reader.rs  L121-L141
return match contract_class {
    ContractClass::V1(casm_contract_class) => {
        let is_declared = is_contract_class_declared(
            &self.storage_reader.begin_ro_txn()...,
            &class_hash,
            self.state_number,          // ← checked against state_number
        )?;
        if is_declared {
            Ok(RunnableCompiledClass::V1(casm_contract_class.try_into()?))
        } else {
            Err(StateError::UndeclaredClassHash(class_hash))
        }
    }
    // TODO(shahak): Verify cairo0 as well after get_class_definition_block_number is fixed.
    ContractClass::V0(deprecated_contract_class) => {
        Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
        // ↑ NO declaration check — class returned regardless of state_number
    }
};
``` [1](#0-0) 

The class manager is a global, state-number-agnostic store. It is populated by the central sync process as blocks arrive. A Cairo V0 class declared at block N is stored in the class manager permanently. When an RPC caller queries at any `state_number` before block N, `get_executable` still returns the class, but the V0 branch returns it without calling `is_contract_class_declared`, so the class is treated as available at the earlier state.

The V1 branch correctly guards against this with `is_contract_class_declared`: [2](#0-1) 

The `StateReader` trait documents that `get_compiled_class_hash` returns `CompiledClassHash::default()` for Cairo V0 classes, and `StateReaderAndContractManager::get_compiled_class_hash_v2` even has an `assert_ne!` guard against the default value being used: [3](#0-2) 

This confirms the codebase treats the default value as a sentinel for "not declared / Cairo 0", yet the execution path for V0 classes in `ExecutionStateReader` never validates the sentinel condition.

### Impact Explanation

Any RPC method that invokes `ExecutionStateReader` with a class manager handle — `starknet_call`, `starknet_estimateFee`, `starknet_simulateTransactions`, `starknet_traceTransaction` — can be made to execute a Cairo V0 class that is not declared at the queried block. The execution result (return data, events, fee, revert status) is computed against the wrong class, and the RPC response is returned as authoritative. This matches the allowed impact: **"High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."**

### Likelihood Explanation

The trigger requires:
1. A Cairo V0 class to be present in the class manager (declared in any block ≥ 1).
2. An RPC query at a `block_id` whose `state_number` precedes that declaration block.
3. A `library_call` syscall (or direct `CallEntryPoint` with an explicit `class_hash`) targeting that class hash — bypassing the `UninitializedStorageAddress` guard that protects normal contract calls.

Condition 1 is satisfied by any historical Cairo V0 declaration. Condition 2 is satisfied by any historical block query. Condition 3 is reachable by any unprivileged caller who can craft calldata that triggers a `library_call` to the target class hash inside an already-deployed contract. The combination is low-probability in practice but fully unprivileged and requires no special access.

### Recommendation

Apply the same `is_contract_class_declared` guard to the Cairo V0 branch, mirroring the V1 branch. The `TODO` comment references a `get_class_definition_block_number` issue; once that is resolved, the check should be:

```rust
ContractClass::V0(deprecated_contract_class) => {
    let is_declared = is_contract_class_declared(
        &self.storage_reader.begin_ro_txn().map_err(storage_err_to_state_err)?,
        &class_hash,
        self.state_number,
    )
    .map_err(|e| StateError::StateReadError(e.to_string()))?;

    if is_declared {
        Ok(RunnableCompiledClass::V0(deprecated_contract_class.try_into()?))
    } else {
        Err(StateError::UndeclaredClassHash(class_hash))
    }
}
```

Until `get_class_definition_block_number` is fixed for deprecated classes, an interim guard using `get_deprecated_class_definition_at(self.state_number, &class_hash)` (which already performs the state-number check) can be used: [4](#0-3) 

### Proof of Concept

1. Deploy a Cairo V0 contract class `C` at block 10. The class manager now holds `C`.
2. Issue `starknet_call` at `block_id = { block_number: 5 }` (before block 10) against any deployed contract whose `__execute__` or any entry point performs `library_call(class_hash=C, ...)`.
3. `ExecutionStateReader::get_compiled_class(C)` is called with `state_number = right_after_block(5)`.
4. `class_manager_client.get_executable(C)` returns `ContractClass::V0(...)` — the class exists in the manager.
5. The V0 branch returns `Ok(RunnableCompiledClass::V0(...))` without checking `is_contract_class_declared`.
6. The library call executes class `C` at block 5, even though `C` was not declared until block 10.
7. The RPC response reflects execution results from the wrong class, returning an authoritative-looking wrong value. [5](#0-4)
