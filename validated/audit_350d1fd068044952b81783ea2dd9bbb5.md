### Title
Excess ETH Sent to `initTransfer` Is Permanently Lost to Wormhole Fee Collector — (`evm/src/omni-bridge/contracts/OmniBridgeWormhole.sol`)

---

### Summary

`OmniBridgeWormhole.initTransferExtension` forwards the entire `extensionValue` (= `msg.value − nativeFee`) to `_wormhole.publishMessage`. Wormhole's `publishMessage` only requires `msg.value >= messageFee()` and silently absorbs any surplus into its fee collector. Because `OmniBridge.initTransfer` never enforces `msg.value == nativeFee + wormhole.messageFee()`, any ETH a user sends beyond that exact sum is irrecoverably lost.

---

### Finding Description

`OmniBridge.initTransfer` computes `extensionValue` as the ETH left over after accounting for the user-supplied `nativeFee`:

```
// OmniBridge.sol lines 391–393
extensionValue = msg.value - amount - nativeFee;   // native-ETH path
extensionValue = msg.value - nativeFee;             // ERC-20 path
``` [1](#0-0) 

This `extensionValue` is passed unchanged to `initTransferExtension`, which in `OmniBridgeWormhole` forwards it verbatim to Wormhole:

```
// OmniBridgeWormhole.sol line 143
_wormhole.publishMessage{value: value}(wormholeNonce, payload, _consistencyLevel);
``` [2](#0-1) 

Wormhole's `publishMessage` accepts any `msg.value >= messageFee()` and routes the entire amount to its fee collector — it does **not** refund the surplus. There is no guard anywhere in the call chain that enforces `extensionValue == _wormhole.messageFee()`.

The same pattern exists in `deployTokenExtension`, `logMetadataExtension`, and `finTransferExtension`, which pass the raw `msg.value` directly:

```
_wormhole.publishMessage{value: msg.value}(...)   // lines 63, 87, 109
``` [3](#0-2) [4](#0-3) [5](#0-4) 

The `initTransfer` path is the highest-risk surface because it is called by ordinary bridge users, not privileged operators.

---

### Impact Explanation

Any ETH a user sends beyond `nativeFee + _wormhole.messageFee()` is permanently transferred to the Wormhole fee collector and is unrecoverable. This constitutes direct, irrecoverable loss of user funds and misdirection of value — matching the **High** allowed impact: *"Balance, decimal, fee, token-mapping, or accounting corruption that breaks bridge collateralization or misdirects value."*

---

### Likelihood Explanation

**Medium.** Users must supply `msg.value` manually and have no on-chain helper to query the exact required amount (`nativeFee + _wormhole.messageFee()`). A user who pads their ETH to guarantee the transaction succeeds, or who miscalculates the Wormhole fee, silently loses the surplus with no revert or warning. The `initTransfer` function is a primary, publicly accessible entry point for every EVM bridge user.

---

### Recommendation

In `OmniBridgeWormhole.initTransferExtension`, validate that `value` equals exactly the Wormhole message fee before forwarding, and revert otherwise:

```solidity
uint256 wormholeFee = _wormhole.messageFee();
require(value == wormholeFee, "Incorrect Wormhole fee");
_wormhole.publishMessage{value: wormholeFee}(wormholeNonce, payload, _consistencyLevel);
```

Apply the same fix to `deployTokenExtension`, `logMetadataExtension`, and `finTransferExtension` by replacing `msg.value` with `_wormhole.messageFee()` and requiring `msg.value == _wormhole.messageFee()` at the start of each public entry point.

---

### Proof of Concept

1. Wormhole is deployed with `messageFee() = 0.001 ETH`.
2. User calls `OmniBridgeWormhole.initTransfer(tokenAddress, amount, fee, nativeFee=0, recipient, "")` and attaches `msg.value = 1 ETH` (overpaying to be safe).
3. `extensionValue = 1 ETH − 0 = 1 ETH`.
4. `_wormhole.publishMessage{value: 1 ETH}(...)` is called; Wormhole accepts it (≥ 0.001 ETH) and credits its fee collector with the full 1 ETH.
5. The transfer is recorded on-chain with `nativeFee = 0`; the user's 0.999 ETH surplus is gone with no revert, no event, and no refund path. [6](#0-5) [7](#0-6)

    function initTransferExtension(
        address sender,
        address tokenAddress,
        uint64 originNonce,
        uint128 amount,
        uint128 fee,
        uint128 nativeFee,
        string calldata recipient,
        string calldata message,
        uint256 value
    ) internal override {
        bytes memory payload = bytes.concat(
            bytes1(uint8(MessageType.InitTransfer)),
            bytes1(omniBridgeChainId),
            Borsh.encodeAddress(sender),
            bytes1(omniBridgeChainId),
            Borsh.encodeAddress(tokenAddress),
            Borsh.encodeUint64(originNonce),
            Borsh.encodeUint128(amount),
            Borsh.encodeUint128(fee),
            Borsh.encodeUint128(nativeFee),
            Borsh.encodeString(recipient),
            Borsh.encodeString(message)
        );
        // slither-disable-next-line reentrancy-eth
        _wormhole.publishMessage{value: value}(
            wormholeNonce,
            payload,
            _consistencyLevel
        );

        wormholeNonce++;
    }
