### Title
`CommitmentStateDiff` Missing `deprecated_declared_classes` Causes Wrong `state_diff_commitment` and Block Hash for Blocks Containing Deprecated Class Declarations â€” (`File: crates/blockifier/src/state/cached_state.rs`)

---

### Summary

The `From<CommitmentStateDiff> for ThinStateDiff` conversion in `crates/blockifier/src/state/cached_state.rs` hardcodes `deprecated_declared_classes: Vec::new()`. Because `CommitmentStateDiff` has no `deprecated_declared_classes` field and `From<StateMaps> for CommitmentStateDiff` silently drops `StateMaps::declared_contracts`, any deprecated (Cairo 0) class declaration executed in a block is invisible to every downstream commitment and hash computation. The resulting `state_diff_commitment`, `state_diff_length` packed into `concat_counts`, and final block hash are all wrong for any block that contains a deprecated class declaration.

---

### Finding Description

**Root cause â€” `From<StateMaps> for CommitmentStateDiff` drops `declared_contracts`**

`StateMaps` carries a `declared_contracts: HashMap<ClassHash, bool>` field that is populated whenever a deprecated (Cairo 0) class is declared. The conversion to `CommitmentStateDiff` silently omits it:

```rust
impl From<StateMaps> for CommitmentStateDiff {
    fn from(diff: StateMaps) -> Self {
        Self {
            address_to_class_hash: IndexMap::from_iter(diff.class_hashes),
            storage_updates: StorageDiff::from(StorageView(diff.storage)),
            class_hash_to_compiled_class_hash: IndexMap::from_iter(diff.compiled_class_hashes),
            address_to_nonce: IndexMap::from_iter(diff.nonces),
            // diff.declared_contracts is never transferred
        }
    }
}
``` [1](#0-0) 

**Propagation â€” `From<CommitmentStateDiff> for ThinStateDiff` hardcodes the empty list**

Because `CommitmentStateDiff` has no `deprecated_declared_classes` field, the conversion to `ThinStateDiff` is forced to hardcode an empty `Vec`:

```rust
impl From<CommitmentStateDiff> for ThinStateDiff {
    fn from(commitment_state_diff: CommitmentStateDiff) -> Self {
        Self {
            deployed_contracts: commitment_state_diff.address_to_class_hash,
            storage_diffs: commitment_state_diff.storage_updates,
            class_hash_to_compiled_class_hash: commitment_state_diff.class_hash_to_compiled_class_hash,
            nonces: commitment_state_diff.address_to_nonce,
            // TODO(AlonH): Remove this when the structure of storage diffs changes.
            deprecated_declared_classes: Vec::new(),
        }
    }
}
``` [2](#0-1) 

**Commitment path â€” `BlockExecutionArtifacts::new` feeds this truncated diff into `calculate_block_commitments`**

**Hash computation â€” `calculate_state_diff_hash` and `ThinStateDiff::len` both consume `deprecated_declared_classes`**

`calculate_state_diff_hash` chains the deprecated declared classes into the Poseidon hash:

---

### Impact Explanation

For any block that contains at least one deprecated (Cairo 0) class declaration:

1. **Wrong `state_diff_commitment`** â€” `calculate_state_diff_hash` hashes an empty `deprecated_declared_classes` list instead of the actual one. The `StateDiffCommitment` stored in `PartialBlockHashComponents` and written to storage is incorrect.
2. **Wrong `state_diff_length` / `concat_counts`** â€” `ThinStateDiff::len()` returns a value that is too small by the number of deprecated declared classes, corrupting the packed `concatenated_counts` field.
3. **Wrong block hash** â€” Both corrupted values feed `calculate_block_hash`, producing a block hash that does not match what any honest verifier or full node would compute from the actual state diff.
4. **Wrong state diff in storage** â€” `BlockExecutionArtifacts::thin_state_diff()` calls the same conversion, so the `ThinStateDiff` committed to storage via `append_state_diff` also has an empty `deprecated_declared_classes`, causing downstream state readers and syncing peers to miss those class declarations.

This matches the **"Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input"** critical impact.

---

### Likelihood Explanation

The blockifier fully supports deprecated (Cairo 0) `DECLARE` transactions. The `StateCache::declare_contract` method writes to `StateMaps::declared_contracts` during execution of any deprecated class declaration. The gateway does not categorically reject deprecated class declarations. Therefore, any operator or user who submits a valid deprecated class declaration transaction that is included in a sequenced block will silently trigger this corruption. No privileged access is required.

---

### Recommendation

1. Add `deprecated_declared_classes: Vec<ClassHash>` to `CommitmentStateDiff`.
2. Populate it in `From<StateMaps> for CommitmentStateDiff` from `diff.declared_contracts` (keys where value is `true`).
3. Remove the hardcoded `deprecated_declared_classes: Vec::new()` in `From<CommitmentStateDiff> for ThinStateDiff` and map the new field instead.

```diff

    pub class_hash_to_compiled_class_hash: IndexMap<ClassHash, CompiledClassHash>,

### Proof of Concept

1. Submit a valid deprecated (Cairo 0) `DECLARE` transaction to the sequencer gateway.
2. The transaction is accepted, executed by the blockifier, and included in a block proposal.
3. During `BlockExecutionArtifacts::new`, `ThinStateDiff::from(commitment_state_diff)` produces `deprecated_declared_classes: []`.
4. `calculate_state_diff_hash` hashes `0` deprecated classes instead of `1`, producing a wrong `StateDiffCommitment`.
5. `ThinStateDiff::len()` returns `N` instead of `N+1`, so `concat_counts` encodes the wrong `state_diff_length`.
6. `calculate_block_hash` chains both wrong values, producing a block hash that differs from the hash any honest verifier would compute from the actual on-chain state diff.
7. Any full node or proof verifier that independently computes the state diff hash from the stored `ThinStateDiff` will also see the wrong value, because `append_state_diff` stores the same truncated diff.

```rust
pub fn concat_counts(
    transaction_count: usize,
    event_count: usize,
    state_diff_length: usize,
    l1_data_availability_mode: L1DataAvailabilityMode,
) -> Felt {
    let l1_data_availability_byte: u8 = match l1_data_availability_mode {
        L1DataAvailabilityMode::Calldata => 0,
        L1DataAvailabilityMode::Blob => 0b10000000,
    };
    let concat_bytes = [
        to_64_bits(transaction_count).as_slice(),
        to_64_bits(event_count).as_slice(),
        to_64_bits(state_diff_length).as_slice(),
        &[l1_data_availability_byte],
        &[0_u8; 7], // zero padding
    ]
    .concat();
    Felt::from_bytes_be_slice(concat_bytes.as_slice())
}
