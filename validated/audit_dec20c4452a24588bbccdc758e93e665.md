### Title
Unprivileged `get_class_hash_at(ALIAS_CONTRACT_ADDRESS)` Poisons Block State Diff, Breaking OS Proof Generation - (File: `state/state.cairo`)

---

### Summary

Any contract can call the `get_class_hash_at` syscall with `ALIAS_CONTRACT_ADDRESS` as the argument. This causes a `dict_read` on `contract_state_changes` keyed to `ALIAS_CONTRACT_ADDRESS`, permanently inserting that address into the block's state diff. At end-of-block, `squash_state_changes_and_maybe_allocate_aliases` asserts that `ALIAS_CONTRACT_ADDRESS` is **not** present in the squashed diff before alias allocation. Because it is present, the Cairo assertion fails and the OS cannot produce a valid proof for the block, halting the network's ability to confirm transactions.

---

### Finding Description

**Root cause — missing address guard in `execute_get_class_hash_at`**

`syscall_impls.cairo` implements the `get_class_hash_at` syscall:

```cairo
// Read the state entry of the requested address.
let (raw_state_entry) = dict_read{dict_ptr=contract_state_changes}(
    key=request.contract_address
);
``` [1](#0-0) 

There is no guard preventing `request.contract_address` from being `ALIAS_CONTRACT_ADDRESS`. The `dict_read` unconditionally inserts a `DictAccess` entry `(key=ALIAS_CONTRACT_ADDRESS, prev_value=V, new_value=V)` into `contract_state_changes`. Cairo's `squash_dict` includes **all** accessed keys in the squashed output, even pure reads where `prev_value == new_value`.

The same unguarded `dict_read` pattern exists in `execute_call_contract`:

```cairo
tempvar contract_address = request.contract_address;
let (state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
    key=contract_address
);
``` [2](#0-1) 

**Failing assertion — `state/state.cairo`**

After all transactions execute, `squash_state_changes_and_maybe_allocate_aliases` runs:

```cairo
// Sanity check: ensure ALIAS_CONTRACT_ADDRESS is not included in the state diff
// (before the allocation).
let (_, success) = search_sorted(
    array_ptr=squashed_contract_state_dict,
    elm_size=DictAccess.SIZE,
    n_elms=n_contract_state_changes,
    key=ALIAS_CONTRACT_ADDRESS,
);
assert success = 0;
``` [3](#0-2) 

Because `ALIAS_CONTRACT_ADDRESS` was inserted by the attacker's `get_class_hash_at` call, `search_sorted` returns `success=1`. The Cairo constraint `assert success = 0` is violated; the prover cannot satisfy it, so no valid STARK proof can be generated for the block.

**Deployment guard does not protect reads**

`deploy_contract.cairo` blocks *deploying* to `ALIAS_CONTRACT_ADDRESS`:

```cairo
assert_not_zero(
    (contract_address - ORIGIN_ADDRESS) * (contract_address - BLOCK_HASH_CONTRACT_ADDRESS) * (
        contract_address - ALIAS_CONTRACT_ADDRESS
    ) * (contract_address - RESERVED_CONTRACT_ADDRESS),
);
``` [4](#0-3) 

But this guard is irrelevant to syscalls that *read* the state of an arbitrary address. No analogous guard exists in `execute_get_class_hash_at` or `execute_call_contract`.

---

### Impact Explanation

If a block containing the malicious transaction is submitted to the prover, the OS Cairo program hits an unsatisfiable constraint and cannot produce a proof. The block cannot be finalized on L1. If the sequencer does not detect and filter this condition during pre-inclusion simulation, any attacker can permanently stall block finalization — matching the **High: Network not being able to confirm new transactions (total network shutdown)** impact class.

---

### Likelihood Explanation

- Any unprivileged user can deploy a contract (or use an existing one) that calls `get_class_hash_at(ALIAS_CONTRACT_ADDRESS)`.
- `ALIAS_CONTRACT_ADDRESS` is a public constant in the OS.
- The attack costs only the gas for one syscall.
- No privileged role, leaked key, or external dependency is required.
- The attack is repeatable: every block that includes such a transaction is unprovable.

---

### Recommendation

Add an explicit address guard in `execute_get_class_hash_at` and `execute_call_contract` (and any other syscall that performs a `dict_read` on `contract_state_changes` with a caller-supplied address) to reject reserved addresses before the `dict_read` occurs:

```cairo
// Reject reserved addresses before touching contract_state_changes.
with_attr error_message("Reserved contract address.") {
    assert_not_zero(
        (request.contract_address - ALIAS_CONTRACT_ADDRESS) *
        (request.contract_address - BLOCK_HASH_CONTRACT_ADDRESS) *
        (request.contract_address - RESERVED_CONTRACT_ADDRESS)
    );
}
```

Alternatively, the `assert success = 0` sanity check in `squash_state_changes_and_maybe_allocate_aliases` should be replaced with a graceful error path that allows the sequencer to identify and exclude the offending transaction rather than failing the entire proof.

---

### Proof of Concept

```
1. Attacker deploys ContractA with an external function:
       fn poison_block() {
           get_class_hash_at(ALIAS_CONTRACT_ADDRESS);  // syscall
       }

2. Attacker submits an invoke transaction calling ContractA::poison_block().

3. The OS processes the transaction:
   - execute_get_class_hash_at() calls
       dict_read{dict_ptr=contract_state_changes}(key=ALIAS_CONTRACT_ADDRESS)
   - A DictAccess entry (key=ALIAS_CONTRACT_ADDRESS, prev=V, new=V) is appended
     to contract_state_changes.

4. End-of-block: state_update() → squash_state_changes_and_maybe_allocate_aliases()
   - squash_dict includes ALIAS_CONTRACT_ADDRESS in the squashed dict.
   - search_sorted(..., key=ALIAS_CONTRACT_ADDRESS) returns success=1.
   - assert success = 0  ← FAILS.

5. The prover cannot satisfy the constraint.
   No valid proof is generated for the block.
   The block is never finalized on L1.
   Network halts.
```
