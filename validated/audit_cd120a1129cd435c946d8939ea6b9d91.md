### Title
Missing Zero-Value Validation in `execute_replace_class` Allows Setting Contract Class Hash to `UNINITIALIZED_CLASS_HASH` — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS Cairo program accepts any `class_hash` value — including `0` (`UNINITIALIZED_CLASS_HASH`) — without validation. An unprivileged contract can call `replace_class(0)` on itself, setting its own class hash to zero. Any subsequent call to that contract causes the OS to attempt a `find_element` lookup for compiled class hash `0`, which panics and makes the block unprovable, halting the network.

---

### Finding Description

In `execute_replace_class` (lines 877–916 of `syscall_impls.cairo`), the function reads `request.class_hash` from the syscall pointer and directly writes it into the contract's `StateEntry` with no validation: [1](#0-0) 

The developer-acknowledged TODO at line 898 explicitly states the missing check: [2](#0-1) 

`UNINITIALIZED_CLASS_HASH` is defined as `0` in `commitment.cairo`: [3](#0-2) 

When a subsequent transaction calls the affected contract, `get_invoke_tx_execution_context` reads the updated `StateEntry` and sets `class_hash=0` in the `ExecutionContext`: [4](#0-3) 

`execute_entry_point` then performs a `dict_read` on `contract_class_changes` with `key=0`, returning `compiled_class_hash=0` (no class is declared with hash 0). It then calls `find_element` with `key=0`: [5](#0-4) 

`find_element` panics (hard Cairo failure, not a graceful revert) when the key is absent from the array. This terminates the OS execution, making the block unprovable.

---

### Impact Explanation

**High — Network not being able to confirm new transactions (total network shutdown).**

If a sequencer includes a block where:
1. Transaction T1 calls `replace_class(0)` on contract `C`, and
2. Transaction T2 (in the same block) calls contract `C`,

the OS will panic during proving of T2. The block is committed via consensus but cannot be proven. The network halts until the block is discarded or the issue is patched.

---

### Likelihood Explanation

Any deployed contract can issue the `replace_class` syscall with an arbitrary `class_hash` value, including `0`. No privileged role is required. The attacker only needs to:
- Deploy a contract that calls `replace_class(0)` on itself.
- Ensure a subsequent call to that contract appears in the same block (e.g., via a second transaction or an inner `call_contract` syscall within the same transaction).

The attack is fully reachable by an unprivileged transaction sender.

---

### Recommendation

Add an explicit zero-check in `execute_replace_class` before updating the state:

```cairo
// In execute_replace_class, after reading class_hash:
let class_hash = request.class_hash;
with_attr error_message("Zero class hash not allowed.") {
    assert_not_zero(class_hash);
}
```

Additionally, resolve the existing TODO by verifying that `class_hash` corresponds to a declared class in `contract_class_changes` before accepting the replacement.

---

### Proof of Concept

1. Attacker deploys contract `C` (valid class hash `H`).
2. Attacker sends **Transaction T1**: calls `C.__execute__()`, which internally issues `replace_class(0)`.
3. OS processes T1: `contract_state_changes[C].class_hash` is updated to `0`.
4. Attacker sends **Transaction T2** in the same block: calls any entry point on `C`.
5. OS processes T2:
   - `get_invoke_tx_execution_context` reads `state_entry.class_hash = 0` → `ExecutionContext.class_hash = 0`.
   - `execute_entry_point` calls `dict_read(contract_class_changes, key=0)` → returns `compiled_class_hash = 0`.
   - `find_element(compiled_class_facts, key=0)` → **panics** (no compiled class with hash 0 exists).
6. OS execution terminates with a hard failure. The block is unprovable. **Network halts.** [6](#0-5)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,

    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
