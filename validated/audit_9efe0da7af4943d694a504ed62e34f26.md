### Title
Missing Declared Class Validation in `execute_replace_class` Allows Permanent Fund Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the new class hash supplied by the calling contract is actually a declared class. Any contract can call `replace_class` with an arbitrary, undeclared class hash. The OS will accept it, commit the corrupted state, and permanently render the contract uncallable — freezing all funds it holds.

---

### Finding Description

**Vulnerability class:** Missing identifier validity check (fee/accounting bug / commitment-output flaw).

In `syscall_impls.cairo`, `execute_replace_class` (lines 877–916) reads `class_hash` directly from the user-controlled syscall request and writes it into `contract_state_changes` without any check that the hash corresponds to a declared class:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment on line 898 explicitly acknowledges the missing check. Compare this with `execute_declare_transaction`, which enforces uniqueness by requiring `prev_value=0` in `dict_update` for `contract_class_changes`:

```cairo
// Note that prev_value=0 enforces that a class may be declared only once.
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [2](#0-1) 

`execute_replace_class` has no equivalent guard. It does not read from `contract_class_changes` and does not verify the hash against the compiled class facts bundle loaded in `get_os_global_context`. [3](#0-2) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once `replace_class` is executed with an undeclared class hash `H`:

1. The contract's `StateEntry.class_hash` is committed to the Patricia Merkle tree as `H`.
2. Every subsequent invocation of the contract causes the OS to look up the compiled class for `H` in `compiled_class_facts`. Because `H` was never declared, no entry exists.
3. Execution of the contract is permanently impossible.
4. Any ERC-20 tokens, ETH, or other assets held in the contract's storage are irrecoverably frozen.

The state commitment path confirms the corrupted entry flows into the final global root: [4](#0-3) 

---

### Likelihood Explanation

**Low.**

The attack requires a contract to call `replace_class` with an invalid hash. Two realistic paths exist:

1. **Malicious deployer:** An attacker deploys a contract whose upgrade function does not validate the new class hash. The attacker advertises the contract as a legitimate vault or DeFi primitive, waits for users to deposit funds, then calls the upgrade function with an arbitrary undeclared hash, freezing all deposited assets.
2. **Buggy upgrade mechanism:** A legitimate contract's permissioned upgrade path contains a logic error that allows an invalid hash to be passed. An attacker who can reach that code path (e.g., through a governance exploit or a missing access-control check in the contract) triggers the freeze.

Both paths are reachable by an unprivileged transaction sender or contract deployer — no privileged operator access is required.

---

### Recommendation

Before committing the new class hash in `execute_replace_class`, verify that it is a declared class. Concretely, perform a `dict_read` on `contract_class_changes` (for classes declared in the current block) or verify membership in the global class commitment tree. If the hash is not found in either location, write a failure response and abort the syscall — analogous to how `execute_call_contract` rejects the `EXECUTE_ENTRY_POINT_SELECTOR`: [5](#0-4) 

---

### Proof of Concept

1. Attacker deploys contract `C` with a public `upgrade(new_class_hash)` function that calls the `replace_class` syscall with the supplied argument and no access-control guard.
2. Users deposit funds into `C` (e.g., via an ERC-20 `transfer`).
3. Attacker submits an `INVOKE_FUNCTION` transaction calling `C.upgrade(0xdeadbeef)`, where `0xdeadbeef` is an arbitrary felt that was never passed to `execute_declare_transaction`.
4. The OS executes `execute_replace_class` at lines 877–916 of `syscall_impls.cairo`. The check on line 898 is absent; `dict_update` writes `class_hash=0xdeadbeef` into `contract_state_changes`.
5. `state_update` in `state.cairo` squashes and commits this entry to the Patricia tree, producing a new global root that encodes `C.class_hash = 0xdeadbeef`.
6. Any future invocation of `C` causes the OS to search `compiled_class_facts` for `0xdeadbeef`, finds nothing, and the execution fails unconditionally.
7. All funds in `C` are permanently frozen with no recovery path. [6](#0-5) [7](#0-6)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
