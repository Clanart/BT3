### Title
IBC/Non-EVM Token Orphaning on SELFDESTRUCT — (`x/evm/statedb/statedb.go`)

### Summary
`StateDB.Commit()` only burns the EVM-denom balance of a self-destructed contract. All other Cosmos bank balances (IBC tokens, CosmWasm bridge tokens, etc.) are silently left at the destroyed address. Because `DeleteAccount` only removes auth-module metadata and EVM storage — never bank balances — those tokens become orphaned but remain spendable by whoever redeploys a contract at the same address via CREATE2.

### Finding Description

**`SelfDestruct` — only EVM denom is cleared**

`SelfDestruct` calls `SubBalance` which is hardcoded to `s.evmDenom`: [1](#0-0) 

**`Commit` — same single-denom burn, with an explicit acknowledgment of the gap** [2](#0-1) 

The comment at lines 813–815 is a direct admission: non-EVM-native tokens "may remain as orphaned bank balances."

**`DeleteAccount` — never touches bank balances** [3](#0-2) 

`RemoveAccount` removes the auth-module record; the x/bank store entry for every non-EVM denom is untouched.

**`SubBalance` — denom-specific, only burns what it is told** [4](#0-3) 

### Impact Explanation

After `Commit()`:
- The auth account at address A is deleted.
- The EVM-denom bank balance is burned.
- Every IBC / non-EVM bank balance at address A **remains in the bank store**, associated with the same bech32 address.

An attacker who deployed the contract with CREATE2 can redeploy an identical contract at the same address (same deployer, same salt, same init-code hash). The new contract account is created at address A, and the bank module immediately sees the pre-existing IBC balances. The new contract can then drain them via a bank precompile or `ExecuteNativeAction`.

This is a **burn bypass**: tokens that the protocol (or users) expect to be destroyed on SELFDESTRUCT survive and are recoverable.

### Likelihood Explanation

- Any unprivileged user can deploy a contract, send it IBC tokens (via `x/bank` `MsgSend` or IBC transfer), call SELFDESTRUCT, and redeploy with CREATE2.
- No governance, validator, or privileged role is required.
- The gap is explicitly documented in the production source, confirming it is a known, unmitigated condition.
- IBC tokens are common on Cosmos chains; the attack surface is broad.

### Recommendation

In the `selfDestructed` branch of `Commit()`, iterate over **all** bank balances held by the destroyed address (not just `evmDenom`) and burn or redirect each one before calling `DeleteAccount`. The Cosmos SDK `BankKeeper` exposes `GetAllBalances(ctx, addr)` for this purpose. Example fix:

```go
allBalances := s.keeper.GetAllBalances(cacheCtx, cosmosAddr)
for _, coin := range allBalances {
    if _, err := s.keeper.SubBalance(cacheCtx, cosmosAddr, coin); err != nil {
        return errorsmod.Wrap(err, "failed to burn post-selfdestruct non-evm balance")
    }
}
```

Alternatively, send non-EVM balances to a community pool or burn address rather than attempting to burn tokens that may not be burnable by the EVM module.

### Proof of Concept

```
1. Deploy contract C at address A using CREATE2(salt=S, initcode=I).
2. Send 100 uatom (IBC denom) to address A via x/bank MsgSend.
3. Call C.selfdestruct() — triggers StateDB.SelfDestruct → SubBalance(evmDenom only).
4. Transaction commits → StateDB.Commit() burns evmDenom, calls DeleteAccount(A).
5. Query bank balance of A for "uatom" → returns 100 uatom (non-zero, orphaned).
6. Redeploy contract at A using CREATE2(same deployer, same S, same I).
7. New contract at A calls bank precompile transfer(A → attacker, "uatom", 100).
8. Attacker receives 100 uatom that should have been burned.
```

The assertion at step 5 directly matches the proof idea in the question and is confirmed by the production comment at: [5](#0-4)

func (k *Keeper) SubBalance(ctx sdk.Context, addr sdk.AccAddress, coin sdk.Coin) (uint256.Int, error) {
	coins := sdk.NewCoins(coin)
	prevBalance := k.GetBalance(ctx, addr, coin.Denom)
	if err := k.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, coins); err != nil {
		return uint256.Int{}, err
	}
	if err := k.bankKeeper.BurnCoins(ctx, types.ModuleName, coins); err != nil {
		return uint256.Int{}, err
	}
	return prevBalance, nil

// DeleteAccount handles contract's suicide call:
// - remove code
// - remove states
// - remove auth account
//
// NOTE: balance should be cleared separately
func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
	cosmosAddr := sdk.AccAddress(addr.Bytes())
	acct := k.accountKeeper.GetAccount(ctx, cosmosAddr)
	if acct == nil {
		return nil
	}

	// NOTE: only Ethereum accounts (contracts) can be selfdestructed
	_, ok := acct.(ethermint.EthAccountI)
	if !ok {
		return errorsmod.Wrapf(types.ErrInvalidAccount, "type %T, address %s", acct, addr)
	}

	// clear storage
	k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool {
		k.SetState(ctx, addr, key, nil)
		return true
	})

	// remove auth account
	k.accountKeeper.RemoveAccount(ctx, acct)

	k.debugLog(ctx, "account suicided",
		"ethereum-address", addr,
		"cosmos-address", cosmosAddr,
	)

	return nil
