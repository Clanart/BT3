### Title
Stale Base Fee Snapshot in `NewDynamicFeeChecker` Allows Cosmos Transactions to Bypass Current Fee Market Minimum - (`ante/evm/fee_checker.go`)

---

### Summary

`NewDynamicFeeChecker`, used for Cosmos (non-EVM) transactions, captures `feemarketParams` once at app initialization and never refreshes it. Because the EIP-1559 base fee is stored inside `feemarketParams.BaseFee` and is updated every block by `BeginBlock`, the checker always enforces the **startup-time base fee** rather than the current block's base fee. Any unprivileged user can submit a Cosmos transaction with a fee calibrated to the stale (potentially much lower) base fee and have it accepted and committed, paying less than the protocol requires.

---

### Finding Description

**Root cause — stale params pointer captured at app initialization**

`newCosmosAnteHandler` is called once during `setAnteHandler` at app startup:

```go
// evmd/ante/handler_options.go  lines 178-188
func newCosmosAnteHandler(ctx sdk.Context, options HandlerOptions, ...) sdk.AnteHandler {
    evmParams      := options.EvmKeeper.GetParams(ctx)       // snapshot at startup
    feemarketParams := options.FeeMarketKeeper.GetParams(ctx) // snapshot at startup
    ...
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
``` [1](#0-0) 

The pointer `&feemarketParams` is passed into `NewDynamicFeeChecker`, which closes over it:

```go
// ante/evm/fee_checker.go  lines 56-60
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
if baseFee == nil {
    return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
}
``` [2](#0-1) 

`types.GetBaseFee` reads `feemarketParams.BaseFee` from the passed struct pointer — it does **not** query the live KV store. The local variable `feemarketParams` in `newCosmosAnteHandler` is heap-allocated (captured by the closure) but is never mutated after initialization.

**How the base fee changes at runtime**

Every block, `feemarket.BeginBlock` calls `CalculateBaseFee` and writes the new value to the KV store via `SetBaseFee`:

```go
// x/feemarket/keeper/abci.go  lines 30-51
func (k *Keeper) BeginBlock(ctx sdk.Context) error {
    baseFee := k.CalculateBaseFee(ctx)
    ...
    k.SetBaseFee(ctx, baseFee)
``` [3](#0-2) 

`SetBaseFee` updates `params.BaseFee` in the KV store:

```go
// x/feemarket/keeper/params.go  lines 72-78
func (k Keeper) SetBaseFee(ctx sdk.Context, baseFee *big.Int) {
    params := k.GetParams(ctx)
    params.BaseFee = ethermint.SaturatedNewInt(baseFee)
    err := k.SetParams(ctx, params)
``` [4](#0-3) 

The in-memory snapshot held by the `NewDynamicFeeChecker` closure is **never updated** by this path.

**Contrast with the EVM ante handler**

`newEthAnteHandler` correctly fetches fresh params on every transaction via `EVMBlockConfig`:

```go
// evmd/ante/handler_options.go  lines 86-96

        blockCfg, err := options.EvmKeeper.EVMBlockConfig(ctx, options.EvmKeeper.ChainID())
        ...
        feemarketParams := &blockCfg.FeeMarketParams   // fresh every call
        baseFee         := blockCfg.BaseFee
``` [5](#0-4) 

`EVMBlockConfig` reads live params from the KV store on each invocation:

```go
// x/evm/keeper/config.go  lines 82-100
params          := k.GetParams(ctx)
feemarketParams := k.feeMarketKeeper.GetParams(ctx)
...
baseFee = feemarketParams.GetBaseFee()
``` [6](#0-5) 

**Fee check and deduction using the stale base fee**

The stale `baseFee` is used to compute `feeCap` and `effectiveFee`:

```go
// ante/evm/fee_checker.go  lines 83-98
feeCap    := fee.Quo(sdkmath.NewIntFromUint64(gas))
baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)          // stale value

if feeCap.LT(baseFeeInt) {                               // check against stale base fee
    return nil, 0, errorsmod.Wrapf(...)
}
effectivePrice := sdkmath.NewIntFromBigInt(
    types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
effectiveFee := sdk.Coins{{Denom: denom,
    Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas))}}
``` [7](#0-6) 

The `effectiveFee` returned here is what `DeductFeeDecorator` actually deducts from the sender and credits to the fee collector. If `baseFee` is stale-low, the deducted amount is below the protocol-required minimum.

The `MinGasPriceDecorator` that runs before `DeductFeeDecorator` also uses the same stale `feemarketParams.MinGasPrice`:

```go
// ante/cosmos/min_gas_price.go  lines 54-58
minGasPrice := mpd.feemarketParams.MinGasPrice   // also stale
if minGasPrice.IsZero() || simulate {
    return next(ctx, tx, simulate)               // short-circuits when MinGasPrice == 0
``` [8](#0-7) 

When `MinGasPrice` is zero (the common default), this decorator is a no-op, leaving `NewDynamicFeeChecker` as the sole fee gate — and it uses the stale base fee.

---

### Impact Explanation

Any unprivileged user submitting a Cosmos SDK transaction (governance votes, staking delegations, IBC relays, etc.) can craft the `fee` field using the base fee that was in effect at node startup rather than the current block's base fee. After a period of sustained high block utilization — which causes the EIP-1559 base fee to ratchet upward — the gap between the stale and live base fee can be substantial (the base fee can increase by up to ~12.5 % per block with default parameters). The transaction passes the stale-base-fee check, is committed, and the fee collector receives less than the protocol requires. This is a mis-accounting of user fees / protocol fee revenue that matches the allowed High impact: *"ante handler… bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

---

### Likelihood Explanation

The condition is always present after the first block that changes the base fee. No special privileges are required — any account can submit a Cosmos transaction. The attacker only needs to observe the current live base fee (via `eth_gasPrice` or the feemarket query), compare it to the startup base fee (observable from node logs or genesis), and set their fee accordingly. The longer a node runs without restart, the larger the exploitable gap.

---

### Recommendation

Replace the stale-params closure in `NewDynamicFeeChecker` with a live read from the KV store on every invocation, mirroring the pattern used in `newEthAnteHandler`. One approach: pass the `FeeMarketKeeper` interface into `NewDynamicFeeChecker` and call `keeper.GetBaseFee(ctx)` inside the returned closure instead of reading from the captured `feemarketParams` pointer.

Alternatively, restructure `newCosmosAnteHandler` to be a per-block factory (as `newEthAnteHandler` already is) so that `feemarketParams` is fetched fresh from the store on each transaction.

---

### Proof of Concept

1. Node starts. `feemarketParams.BaseFee` is captured at startup value `B_0` (e.g., 1 000 000 000).
2. Blocks are produced at full capacity. After N blocks the live base fee in the KV store is `B_N` (e.g., 2 000 000 000 after ~6 full blocks with default denominator 8).
3. Attacker submits a Cosmos governance `MsgVote` with `fee = B_0 * gasLimit` (half the current required fee).
4. `newCosmosAnteHandler` → `NewDynamicFeeChecker` computes `feeCap = B_0`, checks `feeCap >= B_0` (stale) → **passes**.
5. `DeductFeeDecorator` deducts `B_0 * gasLimit` from the sender and credits the fee collector.
6. The protocol should have received `B_N * gasLimit` but received only `B_0 * gasLimit` — a shortfall of `(B_N - B_0) * gasLimit` per transaction.

func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil
