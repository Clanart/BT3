### Title
Integer Division Truncation in `NewDynamicFeeChecker` Allows Zero-Fee Cosmos Transactions When `NoBaseFee = true` - (File: `ante/evm/fee_checker.go`)

### Summary

`NewDynamicFeeChecker` computes the per-gas fee cap via integer division (`fee / gas`). When the London hardfork is active but `NoBaseFee = true`, `types.GetBaseFee` returns `big.NewInt(0)` instead of `nil`, so the checker does not fall back to the min-gas-prices path. An attacker can inflate the `gas` field so that `feeCap` truncates to zero, which then passes the `feeCap >= baseFee (0)` check, producing an `effectiveFee` of zero. The `DeductFeeDecorator` skips deduction when the fee is zero, so the transaction is processed at no cost.

### Finding Description

`types.GetBaseFee` (called at line 56) is:

```go

    if !IsLondon(ethCfg, height) {
        return nil
    }
    baseFee := feemarketParams.GetBaseFee()
    if baseFee == nil {          // NoBaseFee=true → GetBaseFee returns nil
        return new(big.Int)      // returns 0, NOT nil
    }
    return baseFee
}
``` [2](#0-1) 

When `NoBaseFee = true`, `feemarketParams.GetBaseFee()` returns `nil`:

```go
func (p Params) GetBaseFee() *big.Int {
    if p.NoBaseFee {
        return nil
    }
    return p.BaseFee.BigInt()
}
``` [3](#0-2) 

So `types.GetBaseFee` returns `big.NewInt(0)` — **not** `nil` — meaning the checker does **not** fall back to the min-gas-prices path. The checker proceeds with `baseFee = 0`.

With `baseFee = 0`:
- Attacker submits a Cosmos tx with `fee = 1 aevmos`, `gas = 1_000_000_000`
- `feeCap = 1 / 1_000_000_000 = 0` (integer truncation)
- `feeCap (0) >= baseFeeInt (0)` → passes
- `EffectiveGasPrice(0, 0, maxPriorityPrice) = min(0 + maxPriorityPrice, 0) = 0`
- `effectiveFee = 0 * gas = 0`

`EffectiveGasPrice` is:
```go
func EffectiveGasPrice(baseFee *big.Int, feeCap *big.Int, tipCap *big.Int) *big.Int {
    return ethermint.BigMin(new(big.Int).Add(tipCap, baseFee), feeCap)
}
``` [4](#0-3)

```go

    err := evmkeeper.DeductFees(dfd.bankKeeper, ctx, deductFeesFromAcc, fee)
``` [5](#0-4) 

The `NoBaseFee = true` + London HF enabled configuration is explicitly documented and tested:

### Impact Explanation

Any unprivileged user can submit Cosmos (non-EVM) transactions with a negligible fee (e.g., 1 aevmos) and a large gas limit, causing the ante handler to compute an effective fee of zero. The transaction is admitted and executed without paying any fees. This is a fee bypass that permits invalid transactions (those with insufficient fees) to commit, matching the allowed High impact: **"ante handler bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."**

### Likelihood Explanation

**Medium.** The `NoBaseFee = true` configuration is a first-class, documented feature used by chains that want zero-price EVM calls or a static base fee. The default `MinGasPrice = 0` means no fallback protection. The attack requires only a standard Cosmos SDK transaction with a large `gas` field — no special privileges, keys, or contracts needed. The attacker's only cost is the 1-unit fee they include (which is never deducted).

### Recommendation

1. **Use ceiling division** instead of floor division when computing `feeCap`:
   ```go
   // Replace:
   feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
   // With (ceiling):
   gasInt := sdkmath.NewIntFromUint64(gas)
   feeCap := fee.Add(gasInt.Sub(sdkmath.OneInt())).Quo(gasInt)
   ```
2. **Enforce a minimum `feeCap`** equal to `MinGasPrice` when `baseFee = 0`, mirroring the EVM path's `EthMinGasPriceDecorator`.
3. Alternatively, when `baseFee == 0` (not nil), treat it the same as the min-gas-prices fallback path to ensure the global `MinGasPrice` is enforced.

### Proof of Concept

**Precondition:** Chain has London HF enabled and `NoBaseFee = true` (feemarket disabled), `MinGasPrice = 0` (default).

1. Construct a Cosmos SDK tx with:
   - `fee = sdk.NewCoins(sdk.NewCoin("aevmos", sdkmath.NewInt(1)))`
   - `gas = 1_000_000_000` (1 billion)
2. Submit via `BroadcastTx`.
3. In `NewDynamicFeeChecker`:
   - `baseFee = types.GetBaseFee(...) = big.NewInt(0)` (not nil, so no fallback)
   - `feeCap = 1 / 1_000_000_000 = 0`
   - `0 >= 0` → passes
   - `effectivePrice = min(0 + MaxInt64, 0) = 0`
   - `effectiveFee = {aevmos: 0}`
4. `DeductFeeDecorator`: `fee.IsZero() == true` → no deduction.
5. Transaction executes with zero fees paid.
