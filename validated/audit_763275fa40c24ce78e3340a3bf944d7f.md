### Title
Missing Zero Class Hash Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` function in `syscall_impls.cairo` accepts `class_hash = 0` without validation. Because `UNINITIALIZED_CLASS_HASH = 0` is the OS sentinel for a non-deployed contract, any contract that calls `replace_class(0)` — intentionally or via a governance/upgrade exploit — will have its class hash overwritten with the uninitialized sentinel, permanently preventing execution of any entry point and freezing all funds held in that contract.

---

### Finding Description

In `execute_replace_class`, the OS reads the requested class hash directly from the syscall request and writes it into the contract state with no zero-check:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The sentinel value for an undeployed contract is defined as:

```cairo
const UNINITIALIZED_CLASS_HASH = 0;
``` [2](#0-1) 

The `get_contract_state_hash` function treats `class_hash == 0` as the uninitialized state, returning hash `0` when storage and nonce are also zero, and producing a hash that maps to a class-less contract otherwise: [3](#0-2) 

The codebase already establishes the correct pattern: in `execute_declare_transaction`, the OS explicitly asserts `compiled_class_hash` is non-zero before writing it to the class tree:

```cairo
assert_not_zero(compiled_class_hash);
dict_update{dict_ptr=contract_class_changes}(
    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [4](#0-3) 

This check is absent in `execute_replace_class`. The inline TODO comment at line 898 even acknowledges that class hash validation is missing: [5](#0-4) 

---

### Impact Explanation

Once a contract's `class_hash` is set to `UNINITIALIZED_CLASS_HASH` (0), the OS has no class to dispatch entry points to. Every subsequent call to that contract — including any `transfer` or withdrawal function — will fail. Any ERC-20 tokens, STRK, ETH, or other assets stored in the contract's storage become permanently inaccessible. This is **permanent freezing of funds**, matching the Critical impact tier.

---

### Likelihood Explanation

The `replace_class` syscall is callable by any deployed contract from within its own execution context — it requires no privileged role. Realistic trigger paths include:

1. **Malicious contract**: A contract intentionally calls `replace_class(0)` after collecting user deposits (rug-pull variant that also destroys the contract's own code).
2. **Governance/upgrade exploit**: A contract with an on-chain upgrade mechanism (e.g., a DAO-controlled proxy) is passed `class_hash = 0` through a malicious governance proposal, freezing all deposited funds.
3. **Off-by-one / uninitialized variable bug**: A contract's upgrade logic reads a class hash from storage before it is set; the default felt value is 0, and the OS accepts it silently.

The TODO comment in the production OS code confirms the developers are aware validation is absent, increasing the likelihood that this gap will be exploited before it is patched.

---

### Recommendation

Add an explicit non-zero assertion for `class_hash` at the start of `execute_replace_class`, immediately after reading the request, mirroring the pattern already used in `execute_declare_transaction`:

```cairo
let class_hash = request.class_hash;
assert_not_zero(class_hash);  // Prevent replacing with UNINITIALIZED_CLASS_HASH.
```

Additionally, implement the existing TODO to verify that `class_hash` corresponds to a previously declared class in `contract_class_changes`, preventing replacement with arbitrary undeclared hashes.

---

### Proof of Concept

1. Deploy contract `Vault` that accepts user deposits and stores balances in its storage.
2. `Vault` (or an attacker who controls its upgrade path) issues the `replace_class` syscall with `request.class_hash = 0`.
3. The OS executes `execute_replace_class`; no assertion fires because there is no zero-check.
4. `dict_update` writes a new `StateEntry` with `class_hash = 0` for `Vault`'s address into `contract_state_changes`.
5. The state commitment is computed and finalized with `Vault.class_hash = UNINITIALIZED_CLASS_HASH`.
6. All future transactions targeting `Vault` find no class to dispatch to; every entry point call fails.
7. All user funds stored in `Vault`'s storage slots are permanently frozen with no recovery path. [6](#0-5)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.

    return ();
}

func get_contract_state_hash{hash_ptr: HashBuiltin*}(
    class_hash: felt, storage_root: felt, nonce: felt
) -> (hash: felt) {
    const CONTRACT_STATE_HASH_VERSION = 0;
    if (class_hash == UNINITIALIZED_CLASS_HASH) {
        if (storage_root == 0) {
            if (nonce == 0) {
                return (hash=0);
            }
        }
    }

    // Set res = H(H(class_hash, storage_root), nonce).
    let (hash_value) = hash2(class_hash, storage_root);
    let (hash_value) = hash2(hash_value, nonce);

    // Return H(hash_value, CONTRACT_STATE_HASH_VERSION). CONTRACT_STATE_HASH_VERSION must be in the
    // outermost hash to guarantee unique "decoding".
    let (hash) = hash2(hash_value, CONTRACT_STATE_HASH_VERSION);
    return (hash=hash);
}
