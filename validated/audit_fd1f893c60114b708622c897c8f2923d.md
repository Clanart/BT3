### Title
Missing Nonce in `compute_meta_tx_v0_hash` Enables Indefinite Signature Replay via `MetaTxV0` Syscall - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/transaction_hash/transaction_hash.cairo`)

---

### Summary

The `MetaTxV0` syscall allows a calling contract to execute `__execute__` on a target contract using a caller-supplied signature. The hash that the OS computes and presents to the target contract for signature verification — `compute_meta_tx_v0_hash` — omits any nonce or replay-prevention field. Because the hash is purely a function of `(contract_address, selector, calldata, chain_id)`, a valid signature over that tuple remains valid forever and can be replayed an unlimited number of times by any party that has observed it, leading to direct loss of funds.

---

`compute_meta_tx_v0_hash` delegates to `deprecated_get_transaction_hash` with `additional_data_size=0`:

```cairo

The resulting hash covers only: `(INVOKE_HASH_PREFIX, version=0, contract_address, entry_point_selector, calldata_hash, max_fee=0, chain_id)`. No nonce, no deadline, no block number, and no counter of any kind is included.

Compounding this, `execute_meta_tx_v0` constructs the inner `TxInfo` with a hardcoded `nonce=0` and performs **no nonce check or increment** against the target contract's on-chain nonce:

```cairo
tempvar new_tx_info = new TxInfo(
    version=0,
    ...
    nonce=0,
    ...
);
``` [3](#0-2) 

Compare this with how regular account transactions are protected: `check_and_increment_nonce` explicitly skips nonce enforcement for version 0, and the meta-tx path never calls it for the target contract:

```cairo
func check_and_increment_nonce(...) {

    if (tx_info.version == 0) {
        return ();
    }
    ...
}

### Impact Explanation

**Critical — Direct loss of funds.**

Any account contract that supports meta-transactions signs a hash of `(contract_address, calldata, chain_id)`. Because this hash is static for a given intent (e.g., "transfer 100 STRK to Bob"), the signature is valid for every future block. A relayer or any on-chain observer who has seen the signature once can replay it indefinitely. Each replay triggers the target contract's `__execute__`, which may transfer tokens, approve allowances, or perform other irreversible financial operations. The victim has no on-chain mechanism to invalidate the signature short of replacing their account class.

---

### Likelihood Explanation

**High.** The `MetaTxV0` syscall is a first-class protocol feature designed for gasless/sponsored transactions. Any user who has ever submitted a meta-transaction has produced a replayable signature. The attacker's entry path requires only: (1) observing a past meta-transaction on-chain to extract the signature, and (2) submitting a new outer invoke transaction that calls `MetaTxV0` with the same `(contract_address, calldata, signature)` tuple. No privileged access, leaked key, or off-chain coordination is required.

---

### Recommendation

Include a nonce in the meta-transaction hash. The nonce should be read from the target contract's on-chain state before execution and incremented atomically, mirroring how `check_and_increment_nonce` works for regular account transactions. Alternatively, include a caller-supplied deadline (block number or timestamp) in the hash and enforce it inside `execute_meta_tx_v0`. The `compute_meta_tx_v0_hash` function must pass this value as `additional_data` to `deprecated_get_transaction_hash`.

---

### Proof of Concept

1. Alice's account contract supports `MetaTxV0`. She signs a meta-transaction hash for calldata `[transfer, Bob, 100_STRK]`. The hash is `H = Pedersen(invoke, 0, Alice_addr, __execute__, H(calldata), 0, chain_id)`.
2. Alice gives the signature `σ` to a relayer. The relayer submits an outer invoke transaction that calls `execute_meta_tx_v0(Alice_addr, __execute__, calldata, σ)`. Alice's `__execute__` runs, transferring 100 STRK to Bob.
3. The relayer (or any observer of the on-chain calldata) submits the same outer invoke again in a later block with the identical `(Alice_addr, calldata, σ)`. Because `compute_meta_tx_v0_hash` produces the same `H`, Alice's `__validate__` accepts `σ` again, and another 100 STRK is transferred.
4. This repeats until Alice's account is drained or she manually revokes the class. [2](#0-1) [6](#0-5)

// * The signature is replaced with the given signature.
// * The caller is the OS (address 0).
// * The transaction version is replaced by 0.
// * The transaction hash is replaced by the corresponding version-0 transaction hash.
// The changes apply to the called contract and the inner contracts it calls.
func execute_meta_tx_v0{

    syscall_ptr: felt*,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,

    outputs: OsCarriedOutputs*,

    alloc_locals;

    let request = cast(syscall_ptr + RequestHeader.SIZE, MetaTxV0Request*);
    local calldata_start: felt* = request.calldata_start;
    local calldata_size = request.calldata_end - calldata_start;

    let specific_base_gas_cost = (
        META_TX_V0_GAS_COST + META_TX_V0_CALLDATA_FACTOR_GAS_COST * calldata_size
    );
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=specific_base_gas_cost, request_struct_size=MetaTxV0Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    local contract_address = request.contract_address;
    local selector = request.selector;
    local caller_execution_info: ExecutionInfo* = caller_execution_context.execution_info;
    local old_tx_info: TxInfo* = caller_execution_info.tx_info;

    if (selector != EXECUTE_ENTRY_POINT_SELECTOR) {
        write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
        return ();
    }

    // Sanity check: Verify that `signature` is a valid Sierra array.

            entry_point_selector=selector,
            calldata=calldata_start,
            calldata_size=calldata_size,
            chain_id=old_tx_info.chain_id,

        max_fee=0,
        signature_start=request.signature_start,
        signature_end=request.signature_end,
        transaction_hash=meta_tx_hash,
        chain_id=old_tx_info.chain_id,
        nonce=0,
        resource_bounds_start=cast(0, ResourceBounds*),
        resource_bounds_end=cast(0, ResourceBounds*),

        paymaster_data_end=cast(0, felt*),
        nonce_data_availability_mode=0,
        fee_data_availability_mode=0,
        account_deployment_data_start=cast(0, felt*),
        account_deployment_data_end=cast(0, felt*),
        proof_facts_start=cast(0, felt*),
        proof_facts_end=cast(0, felt*),
    );

    let (deprecated_tx_info_ptr: DeprecatedTxInfo*) = alloc();
    tempvar execution_context: ExecutionContext* = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=state_entry.class_hash,
        calldata_size=calldata_size,
        calldata=calldata_start,
        execution_info=new ExecutionInfo(
            block_info=caller_execution_info.block_info,
            tx_info=new_tx_info,
            caller_address=ORIGIN_ADDRESS,
            contract_address=contract_address,
            selector=selector,

    assert [revert_log] = RevertLogEntry(
        selector=CHANGE_CONTRACT_ENTRY, value=caller_execution_info.contract_address
    );
    let revert_log = &revert_log[1];

    contract_call_helper(
        remaining_gas=remaining_gas,
        block_context=block_context,
        execution_context=execution_context,
    );

    // Entries before this point belong to the callee.
    assert [revert_log] = RevertLogEntry(selector=CHANGE_CONTRACT_ENTRY, value=contract_address);
    let revert_log = &revert_log[1];

    return ();
}
