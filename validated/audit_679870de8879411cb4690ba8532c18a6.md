### Title
`deprecated_declared_classes` Silently Dropped from State Diff Commitment and Block Hash — (File: `crates/blockifier/src/state/cached_state.rs`)

---

### Summary

The `From<CommitmentStateDiff> for ThinStateDiff` conversion unconditionally sets `deprecated_declared_classes: Vec::new()`. Because `calculate_state_diff_hash` and `concat_counts` both consume this field, any block that includes a Cairo 0 (`Declare` v0) transaction produces a wrong `state_diff_commitment` and a wrong `concatenated_counts` field, both of which are chained into the final block hash. The wrong commitment is also written to storage via `thin_state_diff()`.

---

### Finding Description

**Step 1 — The silent drop.**

`CommitmentStateDiff` has no `deprecated_declared_classes` field:

```rust
pub struct CommitmentStateDiff {
    pub address_to_class_hash: IndexMap<ContractAddress, ClassHash>,
    pub address_to_nonce: IndexMap<ContractAddress, Nonce>,
    pub storage_updates: IndexMap<ContractAddress, IndexMap<StorageKey, Felt>>,
    pub class_hash_to_compiled_class_hash: IndexMap<ClassHash, CompiledClassHash>,
}
``` [1](#0-0) 

The conversion to `ThinStateDiff` hard-codes the field to empty:

```rust
impl From<CommitmentStateDiff> for ThinStateDiff {
    fn from(commitment_state_diff: CommitmentStateDiff) -> Self {
        Self {
            ...
            // TODO(AlonH): Remove this when the structure of storage diffs changes.
            deprecated_declared_classes: Vec::new(),
        }
    }
}
``` [2](#0-1) 

Meanwhile, `StateMaps` does track whether a class is declared via `declared_contracts: HashMap<ClassHash, bool>`, but `From<StateMaps> for CommitmentStateDiff` never maps this field into `CommitmentStateDiff`, so the information is lost before the conversion even occurs. [3](#0-2) [4](#0-3) 

**Step 2 — Propagation into commitment construction.**

`BlockExecutionArtifacts::new()` calls `calculate_block_commitments` with the corrupted `ThinStateDiff`:

```rust
let (header_commitments, measurements) = calculate_block_commitments(
    &transactions_data,
    ThinStateDiff::from(commitment_state_diff.clone()),  // deprecated_declared_classes = []
    l1_da_mode,
    &block_info.starknet_version,
)
.await;
``` [5](#0-4) 

`calculate_block_commitments` derives two values that are wrong:

1. **`state_diff_commitment`** — via `calculate_state_diff_hash(&state_diff)`. The hash function explicitly chains `deprecated_declared_classes`:

   ```
   Poseidon("STARKNET_STATE_DIFF0", deployed_contracts, declared_classes,
            deprecated_declared_classes,   ← always 0 count + no hashes
            1, 0, storage_diffs, nonces)
   ``` [6](#0-5) [7](#0-6) 

2. **`concatenated_counts`** — via `concat_counts(..., state_diff.len(), ...)`. `ThinStateDiff::len()` counts `deprecated_declared_classes.len()`, which is always 0:

   ```rust
   result += self.deprecated_declared_classes.len();  // always 0
   ``` [8](#0-7) [9](#0-8) 

**Step 3 — Wrong block hash.**

Both corrupted values are chained into `calculate_block_hash`:

```rust
.chain(&block_commitments.state_diff_commitment.0.0)   // wrong
.chain(&block_commitments.concatenated_counts)          // wrong (state_diff_length field)
``` [10](#0-9) 

**Step 4 — Wrong value written to storage.**

`BlockExecutionArtifacts::thin_state_diff()` also returns the corrupted `ThinStateDiff` (same conversion), which is the value written to the storage layer: [11](#0-10) 

---

### Impact Explanation

For any block containing a Cairo 0 `Declare` (v0) transaction:

- `state_diff_commitment` is computed as if `deprecated_declared_classes = []`, omitting the actual declared class hashes from the Poseidon chain. The committed value is wrong.
- `state_diff_length` in `concatenated_counts` is undercounted by the number of deprecated declared classes in the block.
- The final `block_hash` is wrong because it chains both corrupted values.
- The wrong `ThinStateDiff` (with empty `deprecated_declared_classes`) is persisted to storage, so RPC `starknet_getStateUpdate` and sync peers receive a state diff that omits the declared classes.

This matches the **Critical** impact: *Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input.*

---

### Likelihood Explanation

Cairo 0 `Declare` transactions (version 0) remain valid in the Starknet protocol. An unprivileged user can submit one through the standard RPC/HTTP gateway. The gateway does not reject them. Once such a transaction is included in a block, the corruption is deterministic. In practice, Cairo 0 declarations are rare in modern Starknet, making this a low-frequency but fully reachable trigger.

---

### Recommendation

1. Add a `deprecated_declared_classes: Vec<ClassHash>` field to `CommitmentStateDiff`.
2. Populate it from `StateMaps.declared_contracts` (entries where `value == true` and `compiled_class_hashes` does not contain the key, i.e., Cairo 0 classes) during the `From<StateMaps> for CommitmentStateDiff` conversion.
3. Remove the hard-coded `deprecated_declared_classes: Vec::new()` from `From<CommitmentStateDiff> for ThinStateDiff` and use the new field instead.
4. Remove the `// TODO(AlonH)` comment once the fix is in place.

---

### Proof of Concept

1. Submit a Cairo 0 `Declare` transaction (v0) with a valid Cairo 0 class bytecode to the sequencer gateway.
2. Wait for the transaction to be included in a block.
3. Retrieve the block's `state_diff_commitment` from the sequencer's storage or RPC.
4. Independently compute `calculate_state_diff_hash` on the `ThinStateDiff` that includes the declared class hash in `deprecated_declared_classes`.
5. Observe that the two hashes differ: the sequencer's stored commitment chains `0` for the deprecated declared classes count, while the correct hash chains `1` followed by the class hash.
6. Observe that `concatenated_counts` encodes `state_diff_length` as N instead of N+1, and that the final `block_hash` differs from the correctly computed value.

    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);

fn chain_deprecated_declared_classes(
    deprecated_declared_classes: &[ClassHash],
    hash_chain: HashChain,
) -> HashChain {
    let mut sorted_deprecated_declared_classes = deprecated_declared_classes.to_vec();

}
