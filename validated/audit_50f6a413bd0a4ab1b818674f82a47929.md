Audit Report

## Title
Excess ETH Stranded and Stealable After Exact-Output Swaps — (`metric-periphery/contracts/MetricOmmSimpleRouter.sol`)

## Summary

`exactOutputSingle` and `exactOutput` are `payable` and accept native ETH as input via WETH wrapping in `PeripheryPayments.pay`. The swap callback wraps and pays only the pool-requested `amountIn` (≤ `amountInMaximum`), leaving any excess ETH (`msg.value − amountIn`) in the router with no automatic refund. Because `refundETH()` is a public function that sends the entire router ETH balance to `msg.sender`, any third party can immediately steal the stranded excess ETH before the original caller can recover it.

## Finding Description

In `exactOutputSingle` (L130–147), the user sends `msg.value ≥ amountInMaximum`. The pool fires `metricOmmSwapCallback`, which routes to `_justPayCallback` (L192–199):

```solidity
pay(_getTokenToPay(), _getPayer(), msg.sender,
    uint256(MetricOmmSwapResults.extractPositiveAmount(amount0Delta, amount1Delta)));
```

Inside `pay` (PeripheryPayments.sol L73–77), when `token == WETH` and `nativeBalance >= value`, only the pool-requested `value` is wrapped and transferred — the remainder stays as raw ETH in the router:

```solidity
if (nativeBalance >= value) {
    IWETH9(WETH).deposit{value: value}();
    IERC20(WETH).safeTransfer(recipient, value);
}
```

After the callback, `exactOutputSingle` checks the slippage bound and calls `_clearExpectedCallbackPool()` (L145–146) with no ETH refund. The same pattern applies to `exactOutput` (L154–188), which also ends with `_clearExpectedCallbackPool()` and no refund.

The stranded ETH is then immediately claimable by any caller via the public `refundETH()` (PeripheryPayments.sol L58–63):

```solidity
function refundETH() external payable override {
    uint256 balance = address(this).balance;
    if (balance > 0) { _transferETH(msg.sender, balance); }
}
```

`refundETH()` sends the **entire** router ETH balance to `msg.sender` with no access control, so a front-running attacker or any subsequent caller can drain the excess ETH that belongs to the original swapper.

## Impact Explanation

Direct loss of user principal. A user who calls `exactOutputSingle` or `exactOutput` standalone (not via a `multicall` that also encodes `refundETH`) with `msg.value > actual amountIn` permanently loses the difference to the first caller of `refundETH()`. The loss is proportional to the gap between `amountInMaximum` and the actual pool-consumed input, which can be substantial for volatile pools or conservative slippage settings. This meets the Sherlock High/Medium threshold for direct loss of user funds.

## Likelihood Explanation

Any unprivileged user who:
1. Uses native ETH as the input token for an exact-output swap, and
2. Calls `exactOutputSingle` or `exactOutput` directly rather than via a `multicall([..., refundETH])` batch

triggers the vulnerability. Exact-output swaps with ETH are a primary DEX router use case. The theft is permissionless and can be executed by any EOA or bot monitoring the mempool or router ETH balance.

## Recommendation

Add an automatic ETH refund at the end of both `exactOutputSingle` and `exactOutput`, after `_clearExpectedCallbackPool()`:

```solidity
uint256 ethBalance = address(this).balance;
if (ethBalance > 0) {
    _transferETH(msg.sender, ethBalance);
}
```

Alternatively, document prominently (with a code-level `require` or wrapper check) that callers **must** batch a `refundETH()` call via `multicall` when using ETH as input for exact-output swaps, consistent with the pattern already tested for `exactInputSingle` in `MetricOmmSimpleRouter.native.t.sol`.

## Proof of Concept

1. User calls `exactOutputSingle({amountOut: 1000e6, amountInMaximum: 1 ether, tokenIn: WETH, ...})` with `msg.value = 1 ether`.
2. Pool determines it needs only 0.6 ETH; `_justPayCallback` wraps and pays 0.6 ETH to the pool.
3. `exactOutputSingle` returns `amountIn = 0.6 ether`, satisfies `≤ amountInMaximum`, calls `_clearExpectedCallbackPool()` — no refund.
4. 0.4 ETH remains in the router.
5. Attacker calls `refundETH()` in the same block; `refundETH` sends `address(this).balance` (0.4 ETH) to the attacker.
6. User has permanently lost 0.4 ETH.

Foundry test sketch:
```solidity
router.exactOutputSingle{value: 1 ether}(params); // actual cost 0.6 ETH
vm.prank(attacker);
router.refundETH(); // attacker receives 0.4 ETH
assertEq(attacker.balance, 0.4 ether);
```
