### Title
Integer Division Truncation in `NewDynamicFeeChecker` Allows Zero-Fee Cosmos Transactions When `NoBaseFee = true` - (File: `ante/evm/fee_checker.go`)

---

### Summary

In `NewDynamicFeeChecker`, the per-gas fee cap is derived from the total fee via integer division (`fee / gas`). When the total fee is smaller than the gas limit, this truncates to zero. On chains running with `NoBaseFee = true` and London hardfork enabled, the base fee is `0` (not `nil`), so the zero `feeCap` passes the base-fee check, the effective price collapses to zero, and the `DeductFeeDecorator` deducts nothing from the sender — even though the `MinGasPriceDecorator` already validated that the provided fee was sufficient.

---

### Finding Description

**Root cause — `ante/evm/fee_checker.go`, line 83:**

```go
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
```

`sdkmath.Int.Quo` is integer (truncating) division. Any Cosmos tx where `fee < gas` produces `feeCap = 0`.

**Why the nil-guard does not save it — `x/evm/types/utils.go`, lines 244–253:**

```go
func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
    if !IsLondon(ethCfg, height) {
        return nil
    }
    baseFee := feemarketParams.GetBaseFee()
    if baseFee == nil {          // NoBaseFee=true → GetBaseFee returns nil
        return new(big.Int)      // returns 0, NOT nil
    }
    return baseFee
}
```

When `NoBaseFee = true` and London is active, `GetBaseFee` returns `0` (a non-nil `*big.Int`). The nil-check in `NewDynamicFeeChecker` therefore does **not** fall back to `checkTxFeeWithValidatorMinGasPrices`.

**Confirmed by `x/feemarket/types/params.go`, lines 139–145:**

```go
func (p Params) GetBaseFee() *big.Int {
    if p.NoBaseFee {
        return nil   // causes utils.GetBaseFee to return new(big.Int) = 0
    }
    return p.BaseFee.BigInt()
}
```

**Fee check passes with zero feeCap — `ante/evm/fee_checker.go`, lines 84–88:**

```go
baseFeeInt := sdkmath.NewIntFromBigInt(baseFee)   // = 0
if feeCap.LT(baseFeeInt) {                         // 0 < 0 → false → no error
    return nil, 0, errorsmod.Wrapf(...)
}
```

**Effective price collapses to zero — `x/evm/types/utils.go`, lines 232–234:**

```go
func EffectiveGasPrice(baseFee, feeCap, tipCap *big.Int) *big.Int {
    return ethermint.BigMin(new(big.Int).Add(tipCap, baseFee), feeCap)
    // = min(tipCap + 0, 0) = 0
}
```

**Zero fee returned and deducted — `ante/evm/fee_checker.go`, lines 94–99 and `ante/evm/nativefee.go`, lines 57–64:**

```go
effectiveFee := sdk.Coins{{Denom: denom, Amount: effectivePrice.Mul(...)}}
// effectivePrice = 0 → effectiveFee = 0

// In DeductFeeDecorator:
fee, priority, err = dfd.txFeeChecker(ctx, tx)  // returns effectiveFee = 0
dfd.checkDeductFee(ctx, tx, fee)                 // deducts 0
```

---

### Impact Explanation

Any user submitting a Cosmos (non-EVM) transaction on a chain configured with `NoBaseFee = true` and London hardfork active can set `fee < gas` (e.g., `fee = 1 aevmos`, `gas = 21000`). The `MinGasPriceDecorator` may accept the tx (if `minGasPrice` is small enough that `ceil(minGasPrice × gas) ≤ fee`), but the `DeductFeeDecorator` deducts **zero** fees. The fee collector module receives nothing. This is a fee market / ante handler bug that permits valid user fees to be mis-accounted, matching the **High** allowed impact category.

---

### Likelihood Explanation

`NoBaseFee = true` is a documented, legitimate operational mode explicitly designed for chains that want a static or zero base fee. The `keeper_test.go` test case `"enable london HF, not enable feemarket"` confirms `GetBaseFee` returns `big.NewInt(0)` in this configuration. Any chain running this common configuration is affected. The attacker needs only to craft a standard Cosmos tx with `fee < gas`, which is trivially achievable.

---

### Recommendation

Replace the integer division with a check that preserves the original fee amount as the effective fee when `baseFee = 0`, or use ceiling division so that `feeCap` is never zero when `fee > 0`:

```go
// Option: use ceiling division
feeCap := fee.Add(sdkmath.NewIntFromUint64(gas - 1)).Quo(sdkmath.NewIntFromUint64(gas))
```

Alternatively, when `baseFee = 0` and `feeCap = 0`, return `feeCoins` (the original provided fee) as the effective fee rather than `effectiveFee = 0 × gas`.

---

### Proof of Concept

**Chain configuration:**
- `NoBaseFee = true`, London hardfork enabled, `minGasPrice = 0.5`

**Attacker submits a Cosmos tx:**
- `gas = 21000`, `fee = 10500 aevmos` (passes `MinGasPriceDecorator`: `ceil(0.5 × 21000) = 10500 ≤ 10500`)

**Trace through `NewDynamicFeeChecker`:**
1. `types.GetBaseFee(...)` → `new(big.Int)` = `0` (not nil, no fallback)
2. `feeCap = 10500 / 21000 = 0` (integer truncation)
3. `baseFeeInt = 0`; check `0 < 0` → false → passes
4. `effectivePrice = EffectiveGasPrice(0, 0, MaxInt64) = min(MaxInt64, 0) = 0`
5. `effectiveFee = 0 × 21000 = 0`
6. `DeductFeeDecorator` calls `checkDeductFee` with `fee = 0` → deducts nothing

**Result:** Attacker executes a Cosmos tx paying 0 fees despite providing 10500 aevmos in the tx fee field (the coins are never actually deducted). [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)
