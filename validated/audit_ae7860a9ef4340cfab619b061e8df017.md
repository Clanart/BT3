Audit Report

## Title
Stranded Router ETH Consumed by Arbitrary Subsequent WETH Swap — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary
`PeripheryPayments.pay` uses `address(this).balance` as the native ETH cap when wrapping ETH into WETH, rather than the current transaction's `msg.value`. Because `exactOutputSingle` and other payable entry points have no automatic `refundETH`, any ETH overpayment is permanently stranded on the router within that transaction. A subsequent caller with `tokenIn == WETH` and `msg.value == 0` can consume that stranded ETH to fund their own swap at zero cost.

## Finding Description
In `PeripheryPayments.pay` (lines 73–84), when `token == WETH` and `payer != address(this)`, the function reads `nativeBalance = address(this).balance` — the router's total ETH balance across all transactions — and uses it as the source for WETH wrapping before falling back to `safeTransferFrom`:

```solidity
uint256 nativeBalance = address(this).balance;   // total router ETH, not msg.value
if (nativeBalance >= value) {
    IWETH9(WETH).deposit{value: value}();
    IERC20(WETH).safeTransfer(recipient, value);
} else if (nativeBalance > 0) { ... }
else { IERC20(WETH).safeTransferFrom(payer, recipient, value); }
```

The `receive()` guard (lines 32–34) only blocks plain ETH pushes from non-WETH addresses; it does not intercept ETH arriving as `msg.value` in any `payable` function call. ETH sent with `exactOutputSingle`, `exactInputSingle`, `exactInput`, `exactOutput`, or `multicall` lands directly in `address(this).balance` and is never automatically returned.

**Stranding path:** Alice calls `exactOutputSingle{value: 2 ether}` with `amountInMaximum: 2 ether`. The pool charges 1 ether actual input. `_justPayCallback` calls `pay(WETH, Alice, pool, 1 ether)`. `nativeBalance (2 ether) >= value (1 ether)`, so exactly 1 ether is wrapped and sent to the pool. The remaining 1 ether stays on the router. `exactOutputSingle` has no automatic `refundETH` call (line 130–147), so the 1 ether is stranded.

**Exploit path:** Bob calls `exactInputSingle{value: 0}(amountIn: 1 ether, tokenIn: WETH)`. `_justPayCallback` calls `pay(WETH, Bob, pool, 1 ether)`. `nativeBalance = 1 ether >= value = 1 ether`, so the first branch fires: Alice's stranded ETH is wrapped and sent to the pool. `safeTransferFrom(Bob, ...)` is never reached. Bob receives full swap output having paid nothing.

The partial-native branch (lines 78–81) is equally exploitable: stranded ETH less than `value` reduces the pull from Bob's address by exactly the stranded amount.

## Impact Explanation
Direct loss of user principal. Alice's ETH is consumed to fund Bob's swap; Bob receives swap output without paying ETH or WETH. This is a swap conservation failure: the pool receives owed input, but it is sourced from a prior user's funds rather than the current caller's. Impact is **High**: victim loses 100% of stranded ETH, attacker gains equivalent swap output for free.

## Likelihood Explanation
**Medium.** ETH stranding requires a user to call a payable swap function directly (not via multicall with `refundETH`) with `msg.value > actual amountIn`, and omit a manual `refundETH` call. This is a realistic mistake: `exactOutputSingle` is a standalone `payable` function, users routinely overshoot `msg.value` for slippage headroom on exact-output swaps, and the function provides no automatic refund. Once stranded, exploitation requires only a single public call with `msg.value = 0` and `tokenIn = WETH`.

## Recommendation
Track only the current transaction's contributed ETH. The standard fix is to record `msg.value` at entry (e.g., in transient storage or a local variable passed through the call chain) and use that as the cap for native wrapping instead of `address(this).balance`. Alternatively, add an automatic `refundETH` at the end of every payable swap entry point (`exactInputSingle`, `exactOutputSingle`, `exactInput`, `exactOutput`) so no ETH can survive past a single transaction.

## Proof of Concept
```solidity
// Step 1: Alice overpays on exactOutputSingle (no multicall, no refundETH)
router.exactOutputSingle{value: 2 ether}(ExactOutputSingleParams({
    tokenIn: address(weth), amountOut: Y, amountInMaximum: 2 ether, ...
}));
// Pool charges 1 ether; 1 ether remains stranded on router
assert(address(router).balance == 1 ether);

// Step 2: Bob swaps with msg.value = 0, consuming Alice's stranded ETH
router.exactInputSingle{value: 0}(ExactInputSingleParams({
    tokenIn: address(weth), amountIn: 1 ether, ...
}));
// pay(WETH, Bob, pool, 1 ether): nativeBalance=1 ether >= value=1 ether
// → wraps Alice's ETH, safeTransferFrom(Bob) never called
assert(address(router).balance == 0);
// Bob received swap output, paid nothing
```
