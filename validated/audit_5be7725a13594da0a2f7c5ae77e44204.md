### Title
AnteCache Entry Silently Deleted via Simulation Path in `CheckAndSetEthSenderNonce` — (`ante/eth.go`)

### Summary

`CheckAndSetEthSenderNonce` never inspects the `simulate bool` parameter passed to the ante handler. When the Cosmos SDK's `SimulateTx` gRPC endpoint invokes the ante handler with `ctx.IsCheckTx()==false` and `simulate==true`, the function's `else if ex` branch at line 315–317 fires and calls `nonceCache.Delete(fromStr, txNonce)` on the shared `AnteCache`, destroying a live cache entry that must persist until DeliverTx.

---

### Finding Description

**Relevant code — `ante/eth.go` lines 289–318:** [1](#0-0) 

The logic has three branches after the nonce-match check:

| Condition | Action |
|---|---|
| `IsCheckTx && !IsReCheckTx && ex` | `continue` (tx-replacement shortcut) |
| `txNonce != expectedNonce && ex` | `Delete` + return `ErrInvalidSequence` |
| `!IsCheckTx && ex && txNonce==expectedNonce` | **`Delete`** (intended for DeliverTx) |

The third branch is the bug. It assumes `!IsCheckTx` implies DeliverTx. It does not: the Cosmos SDK's `SimulateTx` path also sets `ctx.IsCheckTx()==false` while passing `simulate==true` to the ante handler.

**Dispatch path — `evmd/ante/ante.go` lines 72–93:** [2](#0-1) 

A properly formed Ethereum tx (carrying `ExtensionOptionsEthereumTx`) submitted via `SimulateTx` is routed to `newEthAnteHandler`.

**`newEthAnteHandler` — `evmd/ante/handler_options.go` lines 86–148:** [3](#0-2) 

The `simulate` parameter is forwarded only to `CheckEthMempoolFee` (line 110). It is **not** forwarded to `CheckAndSetEthSenderNonce` (line 147–148). The shared `options.AnteCache` pointer is passed directly.

**Shared cache — `evmd/app.go` line 802:** [4](#0-3) 

A single `*cache.AnteCache` instance is created at startup and captured by the ante handler closure. Every invocation — CheckTx, DeliverTx, and SimulateTx — operates on the same in-memory map.

**`AnteCache.Delete` — `ante/cache/antecache.go` lines 60–71:** [5](#0-4) 

The delete is unconditional and immediately visible to all concurrent goroutines via the mutex.

---

### Impact Explanation

After the attacker deletes `(A, N)` from the cache:

1. The next replacement tx for `(A, N)` submitted during CheckTx misses the `continue` shortcut (line 295 requires `ex==true`).
2. It falls through to normal processing: `txNonce==expectedNonce` passes, the sequence is incremented to `N+1` in CheckTx state, and the replacement tx is admitted to the mempool.
3. Any further replacement tx now sees `expectedNonce==N+1` but carries `txNonce==N` → `ErrInvalidSequence` → rejected.
4. At DeliverTx time the replacement tx also fails (`expectedNonce==N+1` after the original tx executes).

The user's pending tx is locked: the original cannot be replaced and the first replacement will be evicted at DeliverTx. This is a targeted, silent DoS on the mempool tx-replacement mechanism for any chosen address.

---

### Likelihood Explanation

- The Cosmos SDK `SimulateTx` gRPC endpoint is public and unauthenticated.
- The attacker only needs to know the victim's address and current nonce (both are public on-chain).
- A valid signed Ethereum tx with `ExtensionOptionsEthereumTx` is trivially constructable.
- The attack is repeatable: every time the victim re-seeds the cache via a new CheckTx, the attacker can immediately delete it again.

---

### Recommendation

Guard the cache-mutation branches in `CheckAndSetEthSenderNonce` against simulation mode. The `simulate bool` parameter must be threaded through from `newEthAnteHandler` to `CheckAndSetEthSenderNonce`, and the `else if ex` DeliverTx-delete branch must be skipped when `simulate==true`:

```go
// proposed guard
} else if ex && !simulate {
    nonceCache.Delete(fromStr, txNonce)
}
```

Similarly, the `else if ex` branch at line 302–304 (recheck mismatch delete) should be reviewed for the same simulation-context exposure.

---

### Proof of Concept

```go
// 1. Seed the cache as if a real CheckTx succeeded.
anteCache := cache.NewAnteCache(0)
anteCache.Set(addrStr, 0)
require.True(t, anteCache.Exists(addrStr, 0))

// 2. Build a simulate context: IsCheckTx==false, IsReCheckTx==false.
simCtx := sdk.Context{}.WithIsCheckTx(false).WithIsReCheckTx(false)

// 3. Call CheckAndSetEthSenderNonce as the ante handler does during SimulateTx.
//    txNonce==0 == expectedNonce (account sequence not yet advanced).
_, err := ante.CheckAndSetEthSenderNonce(
    simCtx, tx, ak, false, accountGetter, anteCache)
require.NoError(t, err)

// 4. Cache entry is gone — invariant violated.
require.False(t, anteCache.Exists(addrStr, 0),
    "simulate path must not delete live cache entries")

// 5. Replacement tx now bypasses the shortcut, increments CheckTx sequence,
//    and any subsequent replacement fails with ErrInvalidSequence.
```

func (c *AnteCache) Delete(address string, nonce uint64) {
	if c.maxTx < 0 {
		return
	}
	c.mu.Lock()
	defer c.mu.Unlock()
	key := TxNonce{address, nonce}
	if elem, ok := c.cache[key]; ok {
		c.order.Remove(elem)
		delete(c.cache, key)
	}
}
