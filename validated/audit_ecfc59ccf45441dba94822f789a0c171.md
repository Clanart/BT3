### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary `class_hash` value — including `0` (the `UNINITIALIZED_CLASS_HASH` sentinel) or any undeclared hash — without verifying that the new class is actually declared in the protocol's class tree. A contract deployer can exploit this to permanently brick a contract holding user funds, causing irreversible freezing of those assets. The OS itself acknowledges the missing check via a `TODO` comment but does not enforce it.

---

### Finding Description

In `execute_replace_class` (`syscall_impls.cairo`, lines 878–916), the OS processes the `REPLACE_CLASS` syscall as follows:

1. It reads `request.class_hash` from the syscall segment.
2. It updates `contract_state_changes` with the new `StateEntry` containing the caller-supplied `class_hash`.
3. **No validation is performed** to confirm that `class_hash` corresponds to a declared class in `contract_class_changes`.

The code contains an explicit developer acknowledgment of this gap:

```cairo
let class_hash = request.class_hash;
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
```

After the update, the contract's on-chain `class_hash` is set to the attacker-supplied value. If that value is `0` (`UNINITIALIZED_CLASS_HASH`) or any hash not present in the compiled class facts bundle, every subsequent call to the contract will fail at class-lookup time. Because `replace_class` is the only mechanism to change a contract's class, and the contract can no longer execute any code once its class is invalid, the state is irreversible. All storage — including user token balances or ETH deposits — is permanently inaccessible.

The contrast with the `execute_declare_transaction` path is instructive: when a class is declared, the OS enforces `prev_value=0` and `assert_not_zero(compiled_class_hash)` to guarantee the class is valid before writing it to `contract_class_changes`. No equivalent guard exists in `execute_replace_class`.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any contract that holds user assets (ERC-20 balances, ETH, NFTs, vault shares) can be permanently bricked by calling `replace_class(0)` or `replace_class(<undeclared_hash>)`. Once the contract's `class_hash` is set to an invalid value:

- All `call_contract` syscalls targeting it fail at class resolution.
- All top-level invoke transactions targeting it fail.
- The contract cannot call `replace_class` again to recover, because it cannot execute any code.
- The funds stored in the contract's storage trie remain in the global state root but are permanently inaccessible — a textbook permanent freeze.

---

### Likelihood Explanation

**Moderate-to-High.**

The attack path requires only that a contract deployer (an unprivileged role explicitly listed in the bounty scope) deploy a contract that:
1. Accepts deposits from users.
2. Calls `replace_class(0)` in a subsequent transaction.

No privileged key, operator access, or external dependency is needed. The syscall is available to every Sierra contract. The missing check is a single-line omission that the developers themselves flagged as a known gap. Any contract that exposes an upgrade path (a common pattern) is a potential vector if the upgrade logic does not independently validate the new class hash before invoking the syscall.

---

### Recommendation

Inside `execute_replace_class`, before writing the new `StateEntry`, add a lookup into `contract_class_changes` (or the global class commitment) to confirm that `class_hash` maps to a non-zero compiled class hash:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This mirrors the guard already present in `execute_declare_transaction` (`assert_not_zero(compiled_class_hash)`) and closes the gap acknowledged by the existing TODO comment.

---

### Proof of Concept

**Step 1 — Deploy a vault contract** that accepts user ETH/token deposits and stores balances in its storage.

**Step 2 — Users deposit funds.** The contract's storage now holds user balances; the global state root commits to them.

**Step 3 — Attacker (deployer) calls `replace_class(class_hash=0)`.**

The OS executes `execute_replace_class`:

```cairo
// syscall_impls.cairo lines 896–910
let class_hash = request.class_hash;  // class_hash = 0
// No validation performed (TODO comment only).
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}
tempvar new_state_entry = new StateEntry(
    class_hash=class_hash,          // 0 = UNINITIALIZED_CLASS_HASH
    storage_ptr=state_entry.storage_ptr,
    nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The OS accepts this update without error. The block is proven and finalized.

**Step 4 — Any user attempts to withdraw.**

`execute_call_contract` reads `state_entry.class_hash = 0` and constructs an `ExecutionContext` with `class_hash=0`. The OS attempts to resolve the compiled class for hash `0`; no such class exists in the compiled class facts bundle. Execution fails. The call reverts. The user's funds remain in storage but are permanently inaccessible.

    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
