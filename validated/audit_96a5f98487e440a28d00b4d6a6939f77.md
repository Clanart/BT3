### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Fund Freezing — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts any arbitrary felt value as the new class hash and writes it directly to contract state without verifying that a class with that hash has actually been declared. A contract that calls `replace_class` with an undeclared or garbage class hash will have its class permanently set to an invalid value. All subsequent calls to that contract will fail at entry-point dispatch, permanently freezing any funds held in the contract's storage.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the requested class hash from the syscall request and immediately writes it to `contract_state_changes` without any existence check:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The in-code `TODO` comment explicitly acknowledges the missing check. There is no call to `contract_class_changes` to verify that `class_hash` maps to a compiled class hash, and no assertion that the value is non-zero or corresponds to any previously declared Sierra class.

Compare this with `execute_declare_transaction` in `transaction_impls.cairo`, which enforces `assert_not_zero(compiled_class_hash)` and uses `dict_update` with `prev_value=0` to guarantee the class exists before recording it. `execute_replace_class` performs neither check. [1](#0-0) [2](#0-1) 

---

### Impact Explanation

Once a contract's `class_hash` in `contract_state_changes` is set to an undeclared value, the StarkNet OS cannot resolve any entry point for that contract. Every future `call_contract`, `invoke`, or fee-charge targeting that address will fail at entry-point dispatch. Any ERC-20 balances, ETH, or other assets stored in the contract's storage segment are permanently inaccessible — a direct analog to the original report's "tokens sent to a blacklisted address become frozen."

This satisfies the **Critical — Permanent freezing of funds** impact category. [3](#0-2) 

---

### Likelihood Explanation

The attack path requires only that a contract calls the `replace_class` syscall with an arbitrary felt. This is reachable by:

1. **A malicious contract owner** who deploys a contract, attracts user funds (e.g., acts as a vault or token contract), then calls `replace_class(0xdeadbeef)` — an undeclared hash — to permanently freeze all deposited assets.
2. **A buggy contract** that passes an incorrect hash to `replace_class` (e.g., passing a storage key instead of a class hash). The OS provides no safety net.
3. **A contract that self-destructs by design**, replacing its class with zero or any invalid felt, making recovery impossible.

No privileged role, leaked key, or external dependency is required. Any unprivileged transaction sender who controls or interacts with a contract that exposes a `replace_class` call path can trigger this. [4](#0-3) 

---

### Recommendation

Before writing the new class hash to `contract_state_changes`, the OS must verify that the hash is present in `contract_class_changes` (i.e., it was previously declared). Concretely, perform a `dict_read` on `contract_class_changes` keyed by `class_hash` and assert the returned compiled class hash is non-zero:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the validation already present in `execute_declare_transaction` and closes the gap acknowledged by the existing TODO comment. [5](#0-4) 

---

### Proof of Concept

1. Attacker deploys `VaultContract` with a `drain_class()` external function that calls `replace_class(0x1)` (an undeclared hash).
2. Users deposit ETH/ERC-20 tokens into `VaultContract` (balances stored in its storage).
3. Attacker invokes `drain_class()`. The OS executes `execute_replace_class`:
   - `class_hash = 0x1` is read from the request.
   - No check against `contract_class_changes` is performed.
   - `contract_state_changes[VaultContract].class_hash` is set to `0x1`.
4. Any subsequent `call_contract(VaultContract, ...)` or `invoke` targeting `VaultContract` fails: the OS cannot find an entry point for class `0x1`.
5. All token balances stored in `VaultContract`'s storage are permanently frozen — no withdrawal, transfer, or recovery is possible. [6](#0-5)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,

    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce

        key=contract_address,

    );

    return ();
}
