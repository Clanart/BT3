### Title
`CommitmentStateDiff → ThinStateDiff` silently drops `deprecated_declared_classes`, producing wrong `state_diff_commitment`, `state_diff_length`, and block hash for any block containing a deprecated (Cairo 0) declare transaction — (File: `crates/blockifier/src/state/cached_state.rs`)

### Summary
`From<CommitmentStateDiff> for ThinStateDiff` hardcodes `deprecated_declared_classes: Vec::new()`. Because `CommitmentStateDiff` has no field for Cairo 0 class hashes (they carry no compiled-class hash and are absent from `class_hash_to_compiled_class_hash`), every `ThinStateDiff` produced from block execution silently omits deprecated declared classes. This incomplete `ThinStateDiff` is fed directly into `calculate_block_commitments`, producing a wrong `state_diff_commitment` (Poseidon hash), a wrong `state_diff_length`, and a wrong `concatenated_counts` — all of which feed into the block hash. P2P-syncing nodes use `state_diff_length` as the loop-termination condition for state-diff streaming and will accept a truncated state diff, leaving their state without the declared Cairo 0 class.

### Finding Description

**Root cause — silent drop in the conversion:**

`CommitmentStateDiff` has four fields: `address_to_class_hash`, `address_to_nonce`, `storage_updates`, and `class_hash_to_compiled_class_hash`. A deprecated declare transaction adds the class hash to `StateMaps.declared_contracts` but adds nothing to `StateMaps.compiled_class_hashes` (Cairo 0 has no compiled class hash). Therefore `CommitmentStateDiff::from(StateMaps)` never captures the deprecated declared class.

The conversion then hardcodes the missing field:
