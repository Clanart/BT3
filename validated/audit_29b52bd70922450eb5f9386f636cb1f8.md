### Title
Unchecked ERC20 Transfer Return Value in `charge_fee` Allows Silent Fee Evasion - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo`)

---

### Summary

The `charge_fee` function in `transaction_impls.cairo` calls `non_reverting_select_execute_entry_point_func` to execute an ERC20 `transfer` for fee collection but **completely discards the return value**. If the fee token's `transfer` returns `false` (indicating failure) without reverting, the OS silently proceeds as if the fee was successfully charged. Every other critical entry-point call in the same file captures and asserts its return data; `charge_fee` is the sole exception.

---

### Finding Description

`charge_fee` constructs an `ExecutionContext` targeting the fee token contract with `TRANSFER_ENTRY_POINT_SELECTOR` and calls `non_reverting_select_execute_entry_point_func`. The call site discards all return values:

```cairo
// lines 161-163
non_reverting_select_execute_entry_point_func{remaining_gas=remaining_gas}(
    block_context=block_context, execution_context=&execution_context
);
return ();
``` [1](#0-0) 

`non_reverting_select_execute_entry_point_func` returns `(retdata_size: felt, retdata: felt*, is_deprecated: felt)` and internally asserts only that `is_reverted = 0`:

```cairo
// execute_transaction_utils.cairo lines 192-196
let (is_reverted, retdata_size, retdata, is_deprecated) = select_execute_entry_point_func{...}(...);
assert is_reverted = 0;
return (retdata_size, retdata, is_deprecated);
``` [2](#0-1) 

The assertion only catches a hard revert. An ERC20 `transfer` that returns `(false,)` — i.e., `retdata[0] = 0` — without reverting passes the `is_reverted = 0` check and its failure is never detected.

**Contrast with every other critical call in the same file:**

- `run_validate` captures `retdata` and asserts `retdata[0] = VALIDATED`: [3](#0-2) 

- `execute_declare_transaction` captures `retdata` and asserts `retdata[0] = VALIDATED`: [4](#0-3) 

- `execute_deploy_account_transaction` captures `retdata` and asserts `retdata[0] = VALIDATED`: [5](#0-4) 

`charge_fee` is the only call site that ignores the return data entirely. [6](#0-5) 

---

### Impact Explanation

**Critical — Direct loss of funds.**

If the fee token's `transfer` returns `false` without reverting (a valid ERC20 behavior per the standard), the OS proof is still valid: `is_reverted = 0` is satisfied, the block is finalized on L1, and the sequencer's state commitment reflects the transaction as executed. The sequencer receives zero fee tokens for processing the transaction. Because the OS proof is accepted on L1, this loss is permanent and irreversible. At scale, an attacker who can reliably trigger a `false`-returning transfer can drain sequencer revenue across an entire block.

---

### Likelihood Explanation

The fee token address is a protocol-level constant (`block_context.os_global_context.starknet_os_config.fee_token_address`), so an attacker cannot substitute an arbitrary token. [7](#0-6) 

However, the risk is real in two concrete scenarios:

1. **Insufficient balance path:** If a user's account balance is exactly zero and the fee token implementation returns `false` rather than reverting on an underflow (a valid implementation choice), the OS accepts the block without the sequencer receiving payment.
2. **Fee token upgrade:** StarkNet's fee token contracts are upgradeable. A future version that returns `false` on any edge case would silently break fee collection for all transactions hitting that path, with no OS-level protection.

The entry path is fully unprivileged: any transaction sender triggers `charge_fee` for every invoke, declare, and deploy-account transaction. [8](#0-7) 

---

### Recommendation

Capture the return value of the `non_reverting_select_execute_entry_point_func` call inside `charge_fee` and assert that the transfer succeeded:

```cairo
let remaining_gas = DEFAULT_INITIAL_GAS_COST;
let (retdata_size, retdata, is_deprecated) =
    non_reverting_select_execute_entry_point_func{remaining_gas=remaining_gas}(
        block_context=block_context, execution_context=&execution_context
    );
// Assert the ERC20 transfer returned true (success).
if (is_deprecated == 0) {
    assert retdata_size = 1;
    assert retdata[0] = 1;  // ERC20 success = true
}
```

This mirrors the pattern already used for `__validate__`, `__validate_declare__`, and `__validate_deploy__` throughout the same file.

---

### Proof of Concept

1. Deploy an account contract with zero STRK balance.
2. Submit an invoke transaction (v3) with non-zero resource bounds so `max_fee != 0` and `charge_fee` is entered.
3. The fee token's `transfer(sequencer, amount)` is called. If the token returns `(false,)` instead of reverting (e.g., due to zero balance handling), `non_reverting_select_execute_entry_point_func` returns normally with `retdata[0] = 0`.
4. `charge_fee` returns without checking `retdata`. The OS continues to `EndTx`.
5. The block proof is generated and submitted to L1. The L1 verifier accepts it.
6. The sequencer's fee token balance is unchanged; the user's transaction was processed for free.

func charge_fee{
    range_check_ptr,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, tx_execution_context: ExecutionContext*) {
    alloc_locals;

    local tx_info: TxInfo* = tx_execution_context.execution_info.tx_info;
    let max_fee = compute_max_possible_fee(tx_info=tx_info);

    if (max_fee == 0) {
        return ();
    }

    local low_actual_fee;
    %{ LoadActualFee %}
    local calldata: TransferCallData = TransferCallData(
        recipient=block_context.block_info_for_execute.sequencer_address,
        amount=Uint256(low=low_actual_fee, high=0),
    );

    // Verify that the charged amount is not larger than the transaction's max_fee field.
    assert_nn_le(calldata.amount.low, max_fee);

    // TODO(ilya, 01/01/2026): Consider caching the fee_token_class_hash.
    local fee_token_address = block_context.os_global_context.starknet_os_config.fee_token_address;
    let (fee_state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
        key=fee_token_address
    );
    let (__fp__, _) = get_fp_and_pc();
    // Use block_info directly from block_context, so that charge_fee will always run in
    // execute-mode rather than validate-mode.
    local execution_context: ExecutionContext = ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=fee_state_entry.class_hash,
        calldata_size=TransferCallData.SIZE,
        calldata=&calldata,
        execution_info=new ExecutionInfo(
            block_info=block_context.block_info_for_execute,
            tx_info=tx_info,
            caller_address=tx_info.account_contract_address,
            contract_address=fee_token_address,
            selector=TRANSFER_ENTRY_POINT_SELECTOR,
        ),
        deprecated_tx_info=tx_execution_context.deprecated_tx_info,
    );

    let remaining_gas = DEFAULT_INITIAL_GAS_COST;
    non_reverting_select_execute_entry_point_func{remaining_gas=remaining_gas}(
        block_context=block_context, execution_context=&execution_context
    );
    return ();
}
