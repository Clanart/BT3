### Title
Unclaimable Native Fee Lock — `PayloadType::ClaimNativeFee` Defined But Never Implemented, Permanently Locking ETH in EVM Bridge (`evm/src/omni-bridge/contracts/OmniBridge.sol`, `evm/src/omni-bridge/contracts/BridgeTypes.sol`)

---

### Summary

`BridgeTypes.sol` and `near/omni-types/src/lib.rs` both define a `ClaimNativeFee` payload type alongside `TransferMessage` and `Metadata`, signalling a planned mechanism for relayers to claim native fees (ETH) paid by users on EVM chains. However, no `claimNativeFee` function exists anywhere in `OmniBridge.sol`, and no corresponding MPC-signing flow exists in the NEAR bridge. Every wei of ETH paid as `nativeFee` in `initTransfer` is permanently locked in the contract with no extraction path.

---

### Finding Description

`BridgeTypes.sol` defines three payload types:

```solidity
enum PayloadType {
    TransferMessage,
    Metadata,
    ClaimNativeFee   // ← defined but never used
}
```

The NEAR-side type mirror is identical:

```rust
pub enum PayloadType {
    TransferMessage,
    Metadata,
    ClaimNativeFee,  // ← defined but never used
}
```

`OmniBridge.sol::initTransfer` accepts a `nativeFee` parameter and retains the corresponding ETH inside the contract:

```solidity
uint256 extensionValue;
if (tokenAddress == address(0)) {
    extensionValue = msg.value - amount - nativeFee;
} else {
    extensionValue = msg.value - nativeFee;   // nativeFee stays in contract
    ...
}
initTransferExtension(..., extensionValue);
```

The base `initTransferExtension` reverts if `value != 0`, so `extensionValue` must be zero, meaning the full `nativeFee` portion of `msg.value` is retained by the contract. The contract has a bare `receive() external payable {}` fallback but no withdrawal function for accumulated native fees. There is no `claimNativeFee` function, no admin ETH sweep, and no MPC-signed payload of type `ClaimNativeFee` is ever constructed or verified anywhere in the NEAR bridge (`sign_transfer` produces only `PayloadType::TransferMessage`; `log_metadata_callback` produces only `PayloadType::Metadata`).

---

### Impact Explanation

Every ETH unit paid as `nativeFee` by bridge users is permanently locked inside `OmniBridge.sol`. This constitutes an irrecoverable lock of protocol funds (relayer compensation) in the bridge vault flow. The `ClaimNativeFee` payload type's presence in both the EVM and NEAR type systems confirms the feature was planned but never implemented — an exact structural analog to the USSD missing-redeem finding. The locked ETH cannot be recovered by users, relayers, or admins through any on-chain path.

---

### Likelihood Explanation

Any user who passes `nativeFee > 0` to `initTransfer` triggers the lock. This is a normal, documented user action to incentivize relayers. The `nativeFee` parameter is a first-class field in both the EVM function signature and the NEAR `InitTransferMsg` struct, so real usage is expected. The lock is silent — no revert, no event indicating the fee is unclaimable.

---

### Recommendation

Implement the `ClaimNativeFee` flow end-to-end:

1. On the NEAR bridge: add a `claim_native_fee` entry point that constructs and MPC-signs a `ClaimNativeFeePayload` (using `PayloadType::ClaimNativeFee`) referencing the transfer ID and fee recipient.
2. On `OmniBridge.sol`: add a `claimNativeFee(bytes calldata signatureData, ClaimNativeFeePayload calldata payload)` function that verifies the MPC signature against `nearBridgeDerivedAddress` and transfers the accumulated native fee to the designated recipient.

Until this is implemented, either disable `nativeFee` on EVM (revert if `nativeFee > 0`) or document that native fees are non-functional.

---

### Proof of Concept

1. User calls `initTransfer(USDC, 100e6, 1e6, 1e18, "near:alice.near", "")` with `msg.value = 1e18` (1 ETH as native fee).
2. Inside `initTransfer`: `extensionValue = msg.value - nativeFee = 1e18 - 1e18 = 0`. The 1 ETH stays in `OmniBridge.sol`.
3. `initTransferExtension` is called with `value = 0` — no revert.
4. The relayer calls `sign_transfer` on NEAR, which produces a `PayloadType::TransferMessage` payload — `ClaimNativeFee` is never signed.
5. No `claimNativeFee` function exists on `OmniBridge.sol`. The 1 ETH is permanently locked.
6. Repeat for every user who pays a native fee. The locked ETH accumulates indefinitely.
