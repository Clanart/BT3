### Title
Missing Validation of New Class Hash in `replace_class` Enables Permanent Contract Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary
The `execute_replace_class` syscall handler in the StarkNet OS does not validate that the new class hash supplied by a contract is actually a declared class. An attacker can replace a contract's class hash with an arbitrary undeclared value, permanently rendering the contract unexecutable and freezing any funds held within it.

### Finding Description
In `execute_replace_class`, the OS accepts `request.class_hash` and writes it directly into `contract_state_changes` without checking whether that hash exists in `contract_class_changes` (the dict of declared classes). The function signature does not even include `contract_class_changes` as an implicit argument, making the check structurally impossible at this call site. A TODO comment in the code explicitly acknowledges this gap:

```cairo
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

Compare the function signature of `execute_replace_class` — which only receives `contract_state_changes` — against `execute_syscalls`, which receives both `contract_state_changes` and `contract_class_changes`. The replace-class handler has no access to the declared-class dictionary and therefore cannot perform the cross-parameter consistency check: [2](#0-1) [3](#0-2) 

This is the direct analog of the external report's root cause: a swap output token is used without validating it matches the withdrawal target. Here, a new class hash is accepted without validating it matches any declared class.

### Impact Explanation
Once a contract's class hash is set to an undeclared value and committed to state via `state_update`, the OS can never again find compiled class facts for that contract. Every future call to the contract will fail at the OS level (no compiled class to execute). Any ERC-20 tokens, ETH, or other assets held in the contract's storage are permanently inaccessible — a **critical permanent freezing of funds**. [4](#0-3) 

### Likelihood Explanation
The `replace_class` syscall is callable by any contract on itself — no privileged role is required. The attack path is straightforward:

1. Attacker deploys a contract that appears legitimate (e.g., a yield vault or escrow).
2. Users deposit funds into the contract.
3. Attacker calls `replace_class` from within the contract, supplying an arbitrary undeclared felt (e.g., `1`) as the new class hash.
4. The OS writes the invalid class hash to `contract_state_changes` with no validation.
5. After `state_update` commits the change, the contract is permanently unexecutable.
6. All deposited funds are frozen forever.

No external pre-conditions, privileged keys, or third-party compromise are required.

### Recommendation
Pass `contract_class_changes` as an implicit argument to `execute_replace_class` and assert that the requested class hash exists as a key in that dictionary before updating `contract_state_changes`. Concretely:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,   // ADD THIS
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;
    // Validate the class hash is declared.
    let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
    assert_not_zero(compiled_class_hash);
    ...
}
```

This mirrors the fix pattern from the external report: add a cross-parameter consistency check between the supplied value and the authoritative source of truth.

### Proof of Concept

1. Deploy a contract `VaultContract` with a legitimate class hash `C_valid`. Users deposit 1000 ETH worth of tokens.
2. `VaultContract` internally calls the `replace_class` syscall with `new_class_hash = 0xdeadbeef` (not declared anywhere).
3. The OS executes `execute_replace_class`: gas is deducted, `contract_state_changes[vault_address].class_hash` is set to `0xdeadbeef`, no error is raised.
4. `state_update` squashes and commits the change. The on-chain state now records `VaultContract`'s class as `0xdeadbeef`.
5. Any subsequent invoke targeting `VaultContract` causes the OS to look up compiled class facts for `0xdeadbeef` — none exist. The proof cannot be generated for any transaction touching this contract.
6. All 1000 ETH of user funds are permanently frozen with no recovery path. [5](#0-4)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
