### Title
Silent Transaction Drop in FIFO/Echonet Mempool Path Returns Success to Caller - (`crates/apollo_mempool/src/communication.rs`)

### Summary

In `MempoolCommunicationWrapper::add_tx`, when the mempool is in FIFO (Echonet) mode, a failure to fetch block metadata from the recorder causes the function to silently return `Ok(())` without adding the transaction to the mempool. Because the gateway has already completed validation and returned a successful `tx_hash` to the caller before this point, the transaction is permanently lost with no error surfaced.

### Finding Description

The gateway's `add_tx_inner` completes all validation steps — stateless checks, stateful checks, proof archiving — and then calls `self.mempool_client.add_tx(add_tx_args).await`. This call reaches `MempoolCommunicationWrapper::add_tx`: [1](#0-0) 

When `self.mempool.is_fifo()` is true (i.e., `BehaviorMode::Echonet`), the code calls `fetch_and_update_tx_block_metadata`. If that fetch fails after all retries (recorder unavailable, network error, invalid URL), the function logs a warning and returns `Ok(())`: [2](#0-1) 

The `Ok(())` propagates back to the gateway, which interprets it as a successful mempool insertion and returns `Ok(gateway_output)` — including the `tx_hash` — to the HTTP caller: [3](#0-2) 

The transaction was never inserted into `self.mempool` and will never be sequenced. The retry policy in the wrapper is bounded at 10 seconds total: [4](#0-3) 

The `try_fetch_tx_block_metadata` call itself has a 2-second per-request timeout: [5](#0-4) 

### Impact Explanation

A user submitting a transaction to a node running in Echonet/FIFO mode receives a valid-looking `tx_hash` response. The transaction is silently discarded at the mempool boundary. It will never appear in a block, never be committed to L1, and no error is ever returned. This is the direct Starknet analog of the LES bug: an alternative admission path (FIFO mode) that accepts the transaction at the RPC layer but fails to forward it into the sequencing pipeline, while returning a success response.

Impact: **High. Mempool/gateway/RPC admission rejects valid transactions before sequencing** (silently, with a misleading success response).

### Likelihood Explanation

Any Echonet-mode deployment where the recorder service is temporarily unavailable, misconfigured, or slow will trigger this. The recorder URL is a configurable external HTTP endpoint: [6](#0-5) 

Transient network failures, recorder restarts, or an invalid `recorder_url` in config are all realistic triggers. The existing TODO comment at line 134 acknowledges the forwarding invariant is not yet verified: [7](#0-6) 

### Recommendation

Replace the silent `return Ok(())` with a propagated error so the gateway can surface the failure to the caller instead of returning a false success:

```rust
if !self.fetch_and_update_tx_block_metadata(tx_hash).await {
    return Err(MempoolError::FetchTxBlockMetadataFailed { tx_hash });
}
```

Alternatively, if silent dropping is intentional for Echonet replay (e.g., to skip unknown transactions), the gateway must not return `Ok(gateway_output)` for transactions that were not actually enqueued. The mempool response must distinguish "enqueued" from "silently skipped."

### Proof of Concept

1. Configure a node with `BehaviorMode::Echonet` and point `recorder_url` to an unreachable host.
2. Submit any valid invoke transaction via `POST /add_transaction`.
3. The gateway completes stateless + stateful validation and calls `mempool_client.add_tx`.
4. `fetch_and_update_tx_block_metadata` exhausts retries (≤10 s) and returns `false`.
5. `add_tx` returns `Ok(())` without calling `self.mempool.add_tx(...)`.
6. The gateway returns `{"transaction_hash": "0x..."}` — HTTP 200.
7. The transaction hash never appears in any block; `starknet_getTransactionStatus` returns unknown indefinitely.

    pub(crate) async fn add_tx(
        &mut self,
        args_wrapper: AddTransactionArgsWrapper,
    ) -> MempoolResult<()> {
        if self.mempool.is_fifo() {
            let tx_hash = args_wrapper.args.tx.tx_hash();
            if !self.fetch_and_update_tx_block_metadata(tx_hash).await {
                warn!("Failed to fetch tx block metadata for tx {}, skipping transaction", tx_hash);
                return Ok(());
            }
        }

        self.mempool.add_tx(args_wrapper.args.clone())?;

        // TODO(AlonH): Verify that only transactions that were added to the mempool are sent.

        {
            warn!("Failed to send transaction to P2P: {:?}", p2p_client_err);
        }

        Ok(())
    }

    async fn try_fetch_tx_block_metadata(
        &self,
        url: &reqwest::Url,
    ) -> Result<TxBlockMetadata, String> {
        const REQUEST_TIMEOUT_SECS: u64 = 2;
        let response = self
            .echonet_client
            .get(url.as_str())
            .timeout(std::time::Duration::from_secs(REQUEST_TIMEOUT_SECS))
            .send()
            .await
            .map_err(|e| format!("request failed: {}", e))?;

        if !response.status().is_success() {
            return Err(format!("HTTP {}", response.status()));
        }

        response.json::<TxBlockMetadata>().await.map_err(|e| format!("invalid response: {}", e))
    }
