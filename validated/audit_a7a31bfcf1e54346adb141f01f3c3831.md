### Title
Fee-on-Transfer Token Accounting Discrepancy Enables Unbacked Cross-Chain Minting - (File: evm/src/omni-bridge/contracts/OmniBridge.sol)

### Summary
`OmniBridge.initTransfer()` records the caller-supplied `amount` in the `InitTransfer` event without measuring the actual tokens received. For fee-on-transfer ERC20 tokens, the bridge locks fewer tokens than the event claims, causing the destination chain to mint or release more tokens than are backed by the EVM vault.

### Finding Description
In `OmniBridge.initTransfer()`, when the token is a plain ERC20 (not a bridge token and not a custom minter), the contract executes:

```solidity
IERC20(tokenAddress).safeTransferFrom(
    msg.sender,
    address(this),
    amount          // caller-supplied parameter
);
``` [1](#0-0) 

Immediately after, the same caller-supplied `amount` is emitted verbatim in the cross-chain event:

```solidity
emit BridgeTypes.InitTransfer(
    msg.sender,
    tokenAddress,
    currentOriginNonce,
    amount,          // same parameter, not actual received balance
    ...
);
``` [2](#0-1) 

The `InitTransfer` event is the authoritative source of truth consumed by the NEAR bridge's `fin_transfer_callback` to determine how many tokens to mint or unlock on the destination chain. [3](#0-2) 

For a fee-on-transfer token (one that deducts a percentage on every `transferFrom`), the bridge contract receives `amount * (1 - fee_rate)` tokens, but the event records `amount`. The NEAR side then mints or releases the full `amount`, creating supply that is not backed by the EVM vault.

The `InitTransfer` event definition confirms `amount` is a `uint128` field passed directly from the function argument with no balance-before/after check: [4](#0-3) 

### Impact Explanation
This is a **High** impact finding under the "Balance, decimal, fee, token-mapping, or accounting corruption that breaks bridge collateralization or misdirects value" category.

- The EVM vault holds `N * (1 - fee_rate)` tokens.
- The destination chain (NEAR or others) mints/releases `N` tokens.
- The bridge is immediately undercollateralized by `N * fee_rate` tokens per transfer.
- Repeated transfers compound the deficit.
- When users bridge back, the EVM vault cannot honor all redemptions. The last users to redeem find the vault drained, resulting in permanent loss of funds — an irrecoverable lock of user assets.

### Likelihood Explanation
Fee-on-transfer tokens are a well-known ERC20 pattern (e.g., tokens with reflection mechanics, deflationary tokens). The bridge imposes no whitelist on which ERC20 tokens can be used with `initTransfer`. Any unprivileged user can call `initTransfer` with such a token. No special role, leaked key, or privileged access is required. The entry path is fully public.

### Recommendation
Measure the actual received balance using a before/after balance check and use that value in the event:

```solidity
uint256 balanceBefore = IERC20(tokenAddress).balanceOf(address(this));
IERC20(tokenAddress).safeTransferFrom(msg.sender, address(this), amount);
uint256 actualReceived = IERC20(tokenAddress).balanceOf(address(this)) - balanceBefore;
require(uint128(actualReceived) == actualReceived, "amount overflow");
amount = uint128(actualReceived);
```

Then use `amount` (now the actual received value) in the `InitTransfer` event and all downstream accounting. This ensures the emitted event always reflects the true locked balance.

### Proof of Concept
1. Deploy or use any existing fee-on-transfer ERC20 token `FeeToken` with a 5% transfer fee.
2. Approve `OmniBridge` to spend 1,000 `FeeToken`.
3. Call `OmniBridge.initTransfer(FeeToken, 1000, 0, 0, "recipient.near", "")`.
4. The bridge receives 950 `FeeToken` (5% fee deducted by the token contract).
5. The `InitTransfer` event is emitted with `amount = 1000`.
6. The NEAR bridge's `fin_transfer_callback` processes the event and mints/releases 1000 tokens to the recipient on NEAR.
7. The bridge is now undercollateralized by 50 tokens.
8. Repeat until the vault is drained; the final redeemer on EVM cannot withdraw their tokens. [5](#0-4)

    function initTransfer(
        address tokenAddress,
        uint128 amount,
        uint128 fee,
        uint128 nativeFee,
        string calldata recipient,
        string calldata message
    ) external payable whenNotPaused(PAUSED_INIT_TRANSFER) {
        currentOriginNonce += 1;
        if (fee >= amount) {
            revert InvalidFee();
        }

        uint256 extensionValue;
        if (tokenAddress == address(0)) {
            if (fee != 0) {
                revert InvalidFee();
            }
            extensionValue = msg.value - amount - nativeFee;
        } else {
            extensionValue = msg.value - nativeFee;
            if (customMinters[tokenAddress] != address(0)) {
                IERC20(tokenAddress).safeTransferFrom(
                    msg.sender,
                    customMinters[tokenAddress],
                    amount
                );
                ICustomMinter(customMinters[tokenAddress]).burn(
                    tokenAddress,
                    amount
                );
            } else if (isBridgeToken[tokenAddress]) {
                BridgeToken(tokenAddress).burn(msg.sender, amount);
            } else {
                IERC20(tokenAddress).safeTransferFrom(
                    msg.sender,
                    address(this),
                    amount
                );
            }
        }

        initTransferExtension(
            msg.sender,
            tokenAddress,
            currentOriginNonce,
            amount,
            fee,
            nativeFee,
            recipient,
            message,
            extensionValue
        );

            tokenAddress,

            amount,
            fee,

        if let OmniAddress::Near(recipient) = transfer_message.recipient.clone() {
