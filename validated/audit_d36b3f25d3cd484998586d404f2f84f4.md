### Title
`migrated_compiled_classes` silently dropped in `client_pending_data_to_execution_pending_data`, causing `get_compiled_class_hash` to return stale v1 hash and `should_migrate` to double-count migration resources in RPC pending execution — (`crates/apollo_rpc/src/pending.rs`)

### Summary

`client_pending_data_to_execution_pending_data` maps every field of `ClientStateDiff` into `ExecutionPendingData` except `migrated_compiled_classes`. Because `ExecutionPendingData` has no such field, the migration entries are silently discarded. `ExecutionStateReader::get_compiled_class_hash` only consults `pending_data.declared_classes`; for a class that was migrated (v1→v2 hash) in the pending block it falls through to storage and returns the stale v1 `CompiledClassHash`. The blockifier bouncer's `CasmHashMigrationData::from_state` calls `should_migrate` for every executed class hash; `should_migrate` calls `get_compiled_class_hash`, sees the old v1 hash, compares it with the v2 hash, and concludes the class still needs migration — even though the pending block already migrated it. This causes incorrect migration-resource accounting, wrong bouncer weights, and wrong fee estimates returned by `estimate_fee` / `simulate_transactions` for the pending block.

### Finding Description

**Step 1 — the drop.**

`client_pending_data_to_execution_pending_data` constructs `ExecutionPendingData` field-by-field: [1](#0-0) 

`migrated_compiled_classes` is present in `ClientStateDiff`: [2](#0-1)

**Step 2 — wrong hash returned.**

`ExecutionStateReader::get_compiled_class_hash` only iterates `pending_data.declared_classes`. For a class that appears only in `migrated_compiled_classes` it falls through to storage and returns the block-at-declaration's `class_hash_to_compiled_class_hash` entry — the old v1 hash: [4](#0-3) 

**Step 3 — `should_migrate` fires incorrectly.**

`CasmHashMigrationData::from_state` calls `should_migrate` for every class hash executed in the pending block: [5](#0-4) 

`should_migrate` calls `get_compiled_class_hash`, gets the stale v1 hash, compares it with the v2 hash, and returns `Some(...)` — signalling that migration is still needed: [6](#0-5) 

**Step 4 — wrong fee estimate served.**

`estimate_fee` and `simulate_transactions` both call `client_pending_data_to_execution_pending_data` and pass the result into `execute_transactions`: [7](#0-6) [8](#0-7) 

The migration resources accumulated by the bouncer inflate the gas/fee estimate returned to the caller.

### Impact Explanation

Any user querying `estimate_fee` or `simulate_transactions` against the pending block during a protocol upgrade that emits `migrated_compiled_classes` entries receives an inflated fee estimate. The wrong `CompiledClassHash` is the concrete corrupted value; it propagates through `should_migrate` into the bouncer's resource accounting and ultimately into the RPC response. This matches **High — RPC fee estimation or simulation returns an authoritative-looking wrong value**.

The impact does not reach Critical because:
- `get_compiled_class` (CASM artifact selection) uses `pending_data.classes` (`PendingClasses`), which is correctly populated; the actual bytecode executed is correct.
- `migrated_compiled_classes` is a temporary field (`// TODO(Aviv): Remove this field after we upgrade to 0.14.1.`) so the window is bounded to the upgrade period.
- The wrong value is served only through the RPC pending view, not committed to storage or included in a proof.

### Likelihood Explanation

The condition requires a pending block that contains non-empty `migrated_compiled_classes` entries AND a user querying the pending RPC endpoints during that window. During the 0.14.1 upgrade this is the normal operating state of the network, making the window non-trivial.

### Recommendation

1. Add a `migrated_compiled_classes: IndexMap<ClassHash, CompiledClassHash>` field to `ExecutionPendingData`.
2. Populate it in `client_pending_data_to_execution_pending_data` from `client_pending_data.state_update.state_diff.migrated_compiled_classes`.
3. In `ExecutionStateReader::get_compiled_class_hash`, after checking `declared_classes`, also check `migrated_compiled_classes` and return the migrated hash if found.

```rust
    fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
        if let Some(pending_data) = &self.maybe_pending_data {
            for DeclaredClassHashEntry { class_hash: other_class_hash, compiled_class_hash } in
                &pending_data.declared_classes
            {
                if class_hash == *other_class_hash {
                    return Ok(*compiled_class_hash);
                }
            }
        }

        let maybe_block_number = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_reader()
            .map_err(storage_err_to_state_err)?
            .get_class_definition_block_number(&class_hash)
            .map_err(storage_err_to_state_err)?;

        // Cairo 0 classes (and undeclared classes) do not have a compiled class hash.
        // According to the trait, return the default value.
        let Some(block_number) = maybe_block_number else {
            return Ok(CompiledClassHash::default());
        };

        let state_diff = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_diff(block_number)
            .map_err(storage_err_to_state_err)?
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing state diff at block {block_number}."
            )))?;

        let compiled_class_hash = state_diff
            .class_hash_to_compiled_class_hash
            .get(&class_hash)
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing class declaration at block {block_number}, class \
                 {class_hash}."
            )))?;

        Ok(*compiled_class_hash)
    }
```

```rust
    fn from_state<S: StateReader>(
        state_reader: &S,
        executed_class_hashes: &HashSet<ClassHash>,
        versioned_constants: &VersionedConstants,
    ) -> TransactionExecutionResult<Self> {
        if !versioned_constants.enable_casm_hash_migration {
            return Ok(Self::empty());
        }

        executed_class_hashes.iter().try_fold(Self::empty(), |mut migration_data, &class_hash| {
            if let Some((class_hash, casm_hash_v2_to_v1)) =
                should_migrate(state_reader, class_hash)?
            {
                // Add class hash mapping to the migration data.
                migration_data.class_hashes_to_migrate.insert(class_hash, casm_hash_v2_to_v1);

                // Accumulate the class's migration resources.
                let class = state_reader.get_compiled_class(class_hash)?;
                migration_data.resources +=
                    &class.estimate_compiled_class_hash_migration_resources();
            }
            Ok(migration_data)
        })
    }
```

```rust
pub fn should_migrate(
    state_reader: &impl StateReader,
    class_hash: ClassHash,
) -> StateResult<Option<(ClassHash, CompiledClassHashV2ToV1)>> {
    let state_compiled_class_hash = state_reader.get_compiled_class_hash(class_hash)?;
    match state_compiled_class_hash {
        // Class hash does not exist in the state, or is a Cairo 0 class.
        CompiledClassHash(hash) if hash == StarkHash::ZERO => Ok(None),
        state_compiled_class_hash => {
            let compiled_class_hash_v2 = state_reader.get_compiled_class_hash_v2(
                class_hash,
                &state_reader.get_compiled_class(class_hash)?,
            )?;
            // If the state compiled class hash is compiled class hash v2, the class should not
            // migrate.
            if state_compiled_class_hash == compiled_class_hash_v2 {
                return Ok(None);
            }
            Ok(Some((class_hash, (compiled_class_hash_v2, state_compiled_class_hash))))
        }
    }
```
