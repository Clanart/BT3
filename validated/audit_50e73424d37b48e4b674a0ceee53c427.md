### Title
`stateObject.originStorage` Cache Becomes Stale After `ExecuteNativeAction` Modifies EVM Storage, Enabling Invariant Bypass in EVM Contracts — (`File: x/evm/statedb/state_object.go`, `x/evm/statedb/statedb.go`)

---

### Summary

`stateObject.GetCommittedState` caches storage slot values in `originStorage` on first read and never invalidates that cache. When a precompile calls `ExecuteNativeAction` to write the same EVM storage slot through the keeper, the EVM's in-memory `originStorage` entry is not updated. Any subsequent EVM `SLOAD` for that slot returns the stale pre-action value. The commit-time conflict detector only scans EVM-dirty keys, so a slot that the EVM only reads (never writes) is never checked — the stale read silently drives incorrect EVM execution while the committed state reflects the native write.

---

### Finding Description

**Root cause — `GetCommittedState` never invalidates `originStorage`:** [1](#0-0) 

```go
func (s *stateObject) GetCommittedState(key common.Hash) common.Hash {
    ...
    if value, cached := s.originStorage[key]; cached {
        return value          // ← always returns cached value; no invalidation path
    }
    value := s.db.keeper.GetState(s.db.ctx, s.Address(), key)
    s.originStorage[key] = value
    return value
}
```

Once a slot is cached in `originStorage`, every subsequent `GetState` / `GetCommittedState` call for that slot returns the cached value, regardless of what the keeper store actually contains.

**Native writes land in the keeper store but bypass the cache:**

`ExecuteNativeAction` creates a new `cachemulti.Store` branch, runs the action, then advances `s.ctx` to point at the new branch: [2](#0-1) 

After the action, `s.ctx` reflects the native write. A fresh `keeper.GetState(s.ctx, addr, key)` call would return the updated value. But `GetCommittedState` never reaches the keeper for a slot already in `originStorage`, so the EVM sees the pre-action value for the rest of the transaction.

**Commit-time conflict detection has a blind spot:** [3](#0-2) 

The loop iterates only over `obj.dirtyStorage.SortedKeys()` — slots the EVM has written. A slot that the EVM only read (cached in `originStorage`, absent from `dirtyStorage`) is never examined. The native write commits silently, but the EVM's execution logic already ran against the stale cached value.

---

### Impact Explanation

An EVM contract that:
1. reads storage slot `X` (value `V0` is cached in `originStorage`),
2. calls a precompile that executes a native action writing `X = V1` via `keeper.SetState`,
3. reads `X` again — receives stale `V0`,
4. makes a security-critical decision (access control, ownership check, balance guard) based on `V0`,

will execute that decision incorrectly. The transaction commits with `X = V1` in the store (native write wins) but the EVM's branching logic already ran against `V0`. This enables bypassing invariant checks in EVM contracts that interact with precompiles — directly analogous to the Stylus `SmartWallet` ownership-bypass scenario in the reference report.

The impact maps to: **High — EVM state transition bug that permits invalid transactions to commit, and unauthorized account/code mutation via forged execution path.**

---

### Likelihood Explanation

Any precompile that (a) has access to the EVM keeper and (b) calls `keeper.SetState` inside an `ExecuteNativeAction` closure triggers this path. The `Keeper` interface exposes `SetState` as a callable method: [4](#0-3) 

The comment "only called by `StateDB.Commit()`" is non-enforced documentation. A precompile author who calls `keeper.SetState` from within a native action — a natural pattern for precompiles that manage EVM-side state — silently introduces the cache desync. No privileged role or validator collusion is required; a normal user transaction invoking the precompile is sufficient.

---

### Recommendation

**Short term:** In `ExecuteNativeAction`, after a successful native action, iterate over all `stateObjects` and delete any `originStorage` entries whose on-store value has changed (i.e., call `keeper.GetState` for each cached key and evict stale entries). Alternatively, prohibit `keeper.SetState` calls from within native action closures by enforcing it at the API level (e.g., pass a read-only keeper variant to the action context).

**Long term:** Document explicitly that `originStorage` is a write-through cache that is not invalidated by native actions, and enforce the invariant that native actions must not write EVM storage slots that may already be cached by the surrounding EVM execution frame. Add a test that reads a slot, runs a native action writing that slot, reads again, and asserts the fresh value is returned.

---

### Proof of Concept

```
1. Deploy contract C with storage slot owner = Alice (slot 0x00).

2. C.execute() does:
   a. local_owner = SLOAD(0x00)          // originStorage[0x00] = Alice (cached)
   b. CALL precompile P with args
   c. assert SLOAD(0x00) == local_owner  // invariant check

3. Precompile P.run():
   ExecuteNativeAction(func(ctx) {
       keeper.SetState(ctx, C, 0x00, Bob) // writes Bob into nextStore
   })                                     // s.ctx now points at nextStore

4. Back in C.execute():
   SLOAD(0x00) → GetCommittedState(0x00)
              → originStorage[0x00] hit → returns Alice  ← STALE

5. assert Alice == Alice  → passes (invariant bypassed)

6. Commit():
   - slot 0x00 not in dirtyStorage → no conflict check
   - flushNativeCacheLayers() writes Bob to store
   - committed state: owner = Bob

Result: C's invariant check passed with stale Alice, but committed state has owner = Bob.
        Ownership was changed without triggering the guard.
```

The stale read at step 4 is caused by `GetCommittedState` returning the cached `originStorage` entry set at step 2a, never seeing the native write at step 3. The commit-time detector at `statedb.go:766–788` skips slot `0x00` because it is absent from `dirtyStorage`. [5](#0-4) [3](#0-2)

```go
func (s *stateObject) GetCommittedState(key common.Hash) common.Hash {
	if s.overrideStorage != nil {
		if value, ok := s.overrideStorage[key]; ok {
			return value
		}
		return common.Hash{}
	}

	if value, cached := s.originStorage[key]; cached {
		return value
	}
	// If no live objects are available, load it from keeper
	value := s.db.keeper.GetState(s.db.ctx, s.Address(), key)
	s.originStorage[key] = value
	return value
}
```

```go
func (s *StateDB) ExecuteNativeAction(contract common.Address, converter EventConverter, action func(ctx sdk.Context) error) error {
	prevStore := s.cacheMS
	prevLayerCount := len(s.cacheLayers)

	nextStore, ok := s.cacheMS.CacheMultiStore().(cachemulti.Store)
	if !ok {
		panic("expect nested CacheMultiStore result to be cachemulti.Store")
	}

	eventManager := sdk.NewEventManager()
	actionCtx := s.ctx.WithMultiStore(nextStore).WithEventManager(eventManager)

	if err := action(actionCtx); err != nil {
		return err
	}

	s.cacheMS = nextStore
	s.cacheLayers = append(s.cacheLayers, nextStore)
	s.ctx = s.ctx.WithMultiStore(nextStore)

	events := eventManager.Events()
	s.emitNativeEvents(contract, converter, events)
	s.nativeEvents = s.nativeEvents.AppendEvents(events)
	s.journal.append(nativeChange{
		previousStore:      prevStore,
		previousLayerCount: prevLayerCount,
		events:             len(events),
	})
	return nil
```
