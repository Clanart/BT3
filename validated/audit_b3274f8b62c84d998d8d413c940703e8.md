### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Contract Bricking — (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS accepts an arbitrary, user-controlled `class_hash` from the syscall request and writes it directly to `contract_state_changes` without verifying that the hash corresponds to a previously declared contract class. An explicit developer TODO acknowledges this missing check. A malicious or buggy contract can call `replace_class` with an undeclared hash, permanently bricking itself and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class` (`syscall_impls.cairo`, lines 877–916), the OS reads `class_hash` directly from the user-supplied syscall request and updates the contract's state entry with no validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The developer TODO at line 898 explicitly acknowledges the missing check. Compare this to `execute_declare_transaction` in `transaction_impls.cairo`, where the class hash is cryptographically verified via `finalize_class_hash` before being committed:

```cairo
let expected_class_hash = finalize_class_hash(
    contract_class_component_hashes=contract_class_component_hashes
);
with_attr error_message("Invalid class hash pre-image.") {
    assert [class_hash_ptr] = expected_class_hash;
}
``` [2](#0-1) 

No equivalent validation exists for `replace_class`. The `class_hash` field of `ReplaceClassRequest` is fully attacker-controlled.

Once the invalid hash is committed, any future call to the contract reaches `execute_entry_point`, which performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [3](#0-2) 

`dict_read` returns 0 for an undeclared hash. `find_element` with key 0 (or any non-existent compiled class hash) fails with a Cairo assertion error, making it impossible to generate a valid OS proof for any block that includes a call to the bricked contract.

Furthermore, the revert mechanism in `revert.cairo` only reverts changes within the same transaction's revert log. Once the `replace_class` transaction is finalized and committed to state, the change is permanent — the contract cannot call `replace_class` again to fix itself because its class is now invalid and no code can execute. [4](#0-3) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any contract holding user funds (e.g., a multisig wallet, a token vault, a DeFi protocol contract) that calls `replace_class` with an arbitrary undeclared hash becomes permanently uncallable. Because the contract's own execution is broken, it cannot call `replace_class` again to recover. All funds held by the contract are permanently frozen with no recovery path.

---

### Likelihood Explanation

**Medium.** The attack requires a contract to issue the `replace_class` syscall with an undeclared hash. This is reachable by:
- A malicious contract owner who intentionally bricks a shared contract (e.g., a rug-pull variant)
- A buggy contract that computes or accepts an arbitrary class hash from user input and passes it to `replace_class`

No privileged sequencer access is required. Any deployed contract can issue this syscall. The missing check is acknowledged by the developers themselves via the TODO comment, confirming the gap is known and unmitigated.

---

### Recommendation

Add a validation step in `execute_replace_class` to assert that `request.class_hash` maps to a non-zero entry in `contract_class_changes` (i.e., it has been previously declared). This mirrors the validation already enforced during `execute_declare_transaction` via `finalize_class_hash`. Specifically, after reading `class_hash` from the request, perform:

```cairo
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This ensures only declared classes can be used as replacement targets, closing the gap noted in the TODO.

---

### Proof of Concept

1. Attacker deploys `VaultContract` holding user funds, with a valid declared class.
2. `VaultContract` calls the `replace_class` syscall with `class_hash = 0xdeadbeef` (an arbitrary undeclared felt value).
3. `execute_replace_class` in `syscall_impls.cairo` (line 896–913) writes `class_hash=0xdeadbeef` to `contract_state_changes` with no validation. The transaction succeeds and is finalized.
4. In the next block, any user attempts to call `VaultContract` to withdraw funds.
5. `execute_entry_point` calls `dict_read` on `contract_class_changes` with key `0xdeadbeef`, receiving 0 (undeclared). `find_element` fails — the OS cannot generate a proof for this block.
6. The sequencer skips the call (marks it reverted or excludes it). No withdrawal is ever possible.
7. All funds in `VaultContract` are permanently frozen.

func handle_revert{contract_state_changes: DictAccess*}(
    contract_address, revert_log_end: RevertLogEntry*
) {
    alloc_locals;

    local state_entry: StateEntry*;

    %{ PrepareStateEntryForRevert %}

    let class_hash = state_entry.class_hash;
    let storage_ptr = state_entry.storage_ptr;
    with class_hash, storage_ptr, revert_log_end {
        revert_contract_changes();
    }

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(
            new StateEntry(class_hash=class_hash, storage_ptr=storage_ptr, nonce=state_entry.nonce),
            felt,
        ),
    );

    // `revert_contract_changes()` stops where
    // `revert_log_end[0].selector == CHANGE_CONTRACT_ENTRY`.
    tempvar next_contract_address = revert_log_end[0].value;

    if (next_contract_address == CONTRACT_ADDRESS_UPPER_BOUND) {
        // Finish backward processing: this entry marks the beginning of the revert log.
        return ();
    }

    return handle_revert(contract_address=next_contract_address, revert_log_end=revert_log_end);
}
