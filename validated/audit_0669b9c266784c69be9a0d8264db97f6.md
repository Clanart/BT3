### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS accepts any arbitrary class hash value without verifying that the hash corresponds to a previously declared contract class. A malicious contract deployer can exploit this to permanently freeze funds held by a contract by replacing its class with an undeclared hash, rendering the contract permanently non-executable.

---

### Finding Description

In `execute_replace_class` (the new syscall path), the OS reads the requested class hash directly from the syscall request and immediately writes it into the contract state without any validation:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The developer-acknowledged `TODO` comment at line 898 explicitly confirms the missing check. The same omission exists in the deprecated syscall path at `deprecated_execute_syscalls.cairo` lines 307–329, which also performs no validation of the supplied class hash.

The analog to the external report is direct: just as `POOL_INIT_CODE_HASH` being an incorrect hardcoded value caused the periphery contract to compute a wrong pool address (pointing to non-existent code), here the OS accepts an attacker-supplied class hash that points to non-existent (undeclared) code, with the same consequence — the target contract becomes permanently non-executable. [1](#0-0) [2](#0-1) 

---

### Impact Explanation

Once a contract's class hash is set to an undeclared value, every subsequent call to that contract will fail at the OS level when it attempts to look up and execute the class — because no compiled class fact exists for that hash. Any ERC-20 tokens, ETH, or other assets held in the contract's storage become permanently inaccessible. This constitutes **Critical — Permanent freezing of funds**.

The state update is committed to the Patricia trie via `dict_update` and propagated through `compute_contract_state_commitment` → `calculate_global_state_root`, making it part of the canonical on-chain state with no recovery path. [3](#0-2) 

---

### Likelihood Explanation

The attacker-controlled entry path requires only that the attacker be a **contract deployer** — an explicitly allowed unprivileged role. The attack proceeds as follows:

1. Attacker deploys a contract (e.g., a token vault, staking pool, or any contract that accepts user deposits).
2. Users deposit funds into the contract.
3. The contract's code (controlled by the attacker) calls `replace_class` with an arbitrary undeclared hash (e.g., `0x1`).
4. The OS handler `execute_replace_class` accepts the syscall without checking whether `0x1` is a declared class.
5. The contract's class hash in state is updated to `0x1`.
6. All subsequent calls to the contract revert because no class exists for `0x1`.
7. User funds are permanently frozen.

No privileged access, leaked keys, or external dependency compromise is required. The attack is fully reachable from a standard contract deployment and invocation.

---

### Recommendation

Before committing the new class hash to state, verify that it exists in `contract_class_changes` (i.e., it has been declared in the current or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` with the supplied `class_hash` as key and assert the returned compiled class hash is non-zero (`UNINITIALIZED_CLASS_HASH` check). This mirrors the validation already enforced during `execute_declare_transaction`, where `assert_not_zero(compiled_class_hash)` and `dict_update(..., prev_value=0, ...)` together guarantee a class is declared exactly once before use. [4](#0-3) 

---

### Proof of Concept

1. Deploy contract `VaultAttack` with the following logic:
   - `deposit()`: accepts ETH/tokens from users, records balances.
   - `freeze()`: calls the `replace_class` syscall with `class_hash = 0x1` (an undeclared hash).

2. Users call `deposit()` and transfer funds.

3. Attacker calls `freeze()`. The OS executes `execute_replace_class`:
   - `request.class_hash = 0x1`
   - No check against declared classes is performed (line 898 TODO).
   - `dict_update` writes `StateEntry { class_hash: 0x1, ... }` for `VaultAttack`'s address.

4. State root is updated via `compute_contract_state_commitment` to include the new leaf.

5. Any subsequent `call_contract` targeting `VaultAttack` fails: the OS cannot find a compiled class for hash `0x1`, the call reverts, and all deposited funds are permanently frozen. [5](#0-4)

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
