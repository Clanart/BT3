### Title
Missing Duplicate-Key Guard in `MerkleBlob::batch_insert` Allows Merkle Tree State Corruption — (`File: crates/chia-datalayer/src/merkle/blob.rs`)

---

### Summary

`MerkleBlob::batch_insert` bypasses the duplicate-key and duplicate-hash checks that `MerkleBlob::insert` enforces. An unprivileged caller supplying a batch that contains a repeated `KeyId`, or a `KeyId` already present in a tree with two or more existing leaves, silently inserts a second leaf node for the same key. This corrupts the internal `block_status_cache`, orphans the original leaf in the blob, and produces an incorrect Merkle root — invalidating all subsequent inclusion/exclusion proofs derived from that tree.

---

### Finding Description

`MerkleBlob::insert` guards against duplicate state at lines 369–374:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 362-374
pub fn insert(...) -> Result<TreeIndex, Error> {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(hash) {
        return Err(Error::HashAlreadyPresent());
    }
    ...
}
```

`MerkleBlob::batch_insert` (lines 570–657) takes a completely different code path for all items when the tree already has two or more leaves (`leaf_count > 1`), and for items beyond the first two when the tree starts with zero or one leaf:

```rust
// lines 587-602  — no duplicate check before insert_entry_to_blob
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ..., node: Node::Leaf(LeafNode { hash, key, value, ... }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;   // ← no KeyAlreadyPresent guard
    indexes.push(new_leaf_index);
}
```

`insert_entry_to_blob` writes the new block to the blob **and** updates `block_status_cache` (the key→index and leaf-hash→index maps). When a duplicate key is written:

1. The cache entry for that key is overwritten to point to the new index.
2. The original leaf node remains in the blob at its old index but is now unreachable through the cache.
3. The tree's internal node structure still references the old leaf index, creating a parent/child mismatch.
4. `check_integrity()` will detect `leaf_count != key_to_index_cache_length` or `IntegrityParentChildMismatch`, confirming corruption.
5. Any Merkle root computed after the corruption is wrong; proofs generated from it are invalid.

The Python binding `py_batch_insert` (lines 1503–1519) passes caller-supplied `keys_values` and `hashes` directly into `batch_insert` with no pre-filtering:

```rust
// lines 1503-1518
pub fn py_batch_insert(&mut self, keys_values: Vec<(KeyId, ValueId)>, hashes: Vec<Hash>) -> PyResult<()> {
    ...
    self.batch_insert(zip(keys_values, hashes).collect())?;
    Ok(())
}
```

---

### Impact Explanation

This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."**

A corrupted tree root means:

- Proofs of inclusion generated by `get_proof_of_inclusion` reference hashes that no longer correspond to the actual leaf set, so a verifier will accept a proof for a key/value pair that was silently overwritten.
- Proofs of exclusion may falsely succeed or fail for keys that are structurally present but cache-invisible.
- Delta-sync logic (`DeltaFileCache`) that compares blob hashes across snapshots will produce incorrect deltas, propagating the corrupted state to downstream consumers.

---

### Likelihood Explanation

Any code path that calls `MerkleBlob::batch_insert` (or `py_batch_insert` from Python) with attacker-influenced data is directly reachable. The DataLayer is designed to accept external key/value insertions from application-layer clients. A client that submits a batch with a repeated key — whether accidentally or maliciously — triggers the corruption with no privilege required. The regular `insert` path correctly rejects this with `KeyAlreadyPresent`, so the gap is specific to the batch path and is not caught by existing tests for the batch case with pre-existing duplicate keys.

---

### Recommendation

Add the same duplicate guards at the top of the fast path inside `batch_insert`, before calling `insert_entry_to_blob` for each leaf:

```rust
for ((key, value), hash) in keys_values_hashes {
    if self.block_status_cache.contains_key(key) {
        return Err(Error::KeyAlreadyPresent());
    }
    if self.block_status_cache.contains_leaf_hash(&hash) {
        return Err(Error::HashAlreadyPresent());
    }
    // ... existing logic
}
```

Alternatively, collect all keys into a temporary `HashSet` before the loop and reject the entire batch if any duplicate is detected, consistent with the atomicity expectation of a batch operation.

---

### Proof of Concept

```python
from chia_rs import MerkleBlob, KeyId, ValueId
import hashlib

blob = MerkleBlob(bytearray())

# Pre-populate with 3 leaves so leaf_count > 1 (bypasses the guarded path)
for i in range(3):
    blob.insert(KeyId(i), ValueId(i), hashlib.sha256(str(i).encode()).digest())

root_before = blob.get_root()   # correct root

# batch_insert with a duplicate key (key=0 already exists)
dup_key   = KeyId(0)
dup_value = ValueId(999)
dup_hash  = hashlib.sha256(b"attacker").digest()

blob.batch_insert([(dup_key, dup_value)], [dup_hash])
# No exception raised — duplicate silently accepted

root_after = blob.get_root()    # corrupted root, differs from root_before
assert root_before != root_after, "Root was silently corrupted"

    fn check_just_integrity(&self) -> Result<(), Error> {
        let mut leaf_count: usize = 0;
        let mut internal_count: usize = 0;
        let mut child_to_parent: HashMap<TreeIndex, TreeIndex> = HashMap::new();

        for item in ParentFirstIterator::new(&self.blob, None) {
            let (index, block) = item?;
            if let Some(parent) = block.node.parent().0 {
                if child_to_parent.remove(&index) != Some(parent) {
                    return Err(Error::IntegrityParentChildMismatch(index));
                }
            }
            match block.node {
                Node::Internal(node) => {
                    internal_count += 1;
                    child_to_parent.insert(node.left, index);
                    child_to_parent.insert(node.right, index);
                }
                Node::Leaf(node) => {
                    leaf_count += 1;
                    let cached_index = self
                        .block_status_cache
                        .get_index_by_key(node.key)
                        .ok_or(Error::IntegrityKeyNotInCache(node.key))?;
                    if *cached_index != index {
                        return Err(Error::IntegrityKeyToIndexCacheIndex(
                            node.key,
                            index,
                            *cached_index,
                        ));
                    }
                    assert!(
                        !self.block_status_cache.is_index_free(index),
                        "{}",
                        format!("active index found in free index list: {index:?}")
                    );
                }
            }
        }

        let key_to_index_cache_length = self.block_status_cache.key_to_index.len();
        if leaf_count != key_to_index_cache_length {
            return Err(Error::IntegrityKeyToIndexCacheLength(
                leaf_count,
                key_to_index_cache_length,
            ));
        }
        let leaf_hash_to_index_cache_length = self.block_status_cache.leaf_hash_to_index.len();
        if leaf_count != leaf_hash_to_index_cache_length {
            return Err(Error::IntegrityLeafHashToIndexCacheLength(
                leaf_count,
                leaf_hash_to_index_cache_length,
            ));
        }
        let total_count = leaf_count + internal_count + self.block_status_cache.free_index_count();
        let extend_index = self.extend_index();
        if total_count != extend_index.0 as usize {
            return Err(Error::IntegrityTotalNodeCount(extend_index, total_count));
        }
        if !child_to_parent.is_empty() {
            return Err(Error::IntegrityUnmatchedChildParentRelationships(
                child_to_parent.len(),
            ));
        }

        Ok(())
    }
