### Title
Stale Fee-Market Params Snapshot in `newCosmosAnteHandler` Allows Cosmos Txs to Bypass Updated Fee Requirements After Governance Change - (File: `evmd/ante/handler_options.go`)

### Summary
`newCosmosAnteHandler` (and `newLegacyCosmosAnteHandlerEip712`) snapshot `feemarketParams` and `evmParams` at construction time and pass them by pointer into `NewDynamicFeeChecker` and `MinGasPriceDecorator` closures. Because these closures capture the initial snapshot rather than reading live on-chain state per transaction, any governance-executed update to `feemarket` parameters (e.g., enabling the base fee, raising `MinGasPrice`, or changing `BaseFee`) is silently ignored for all subsequent Cosmos-type transactions. The Ethereum ante handler (`newEthAnteHandler`) does not share this flaw — it calls `EVMBlockConfig` inside its closure on every invocation.

### Finding Description

In `evmd/ante/handler_options.go`, `newCosmosAnteHandler` reads params once at construction:

```go
func newCosmosAnteHandler(ctx sdk.Context, options HandlerOptions, ...) sdk.AnteHandler {
    evmParams := options.EvmKeeper.GetParams(ctx)           // snapshot
    feemarketParams := options.FeeMarketKeeper.GetParams(ctx) // snapshot
    ...
    txFeeChecker = evm.NewDynamicFeeChecker(ethCfg, &evmParams, &feemarketParams)
    ...
    cosmos.NewMinGasPriceDecorator(options.FeeMarketKeeper, evmDenom, &feemarketParams),
``` [1](#0-0) 

Both `&evmParams` and `&feemarketParams` are pointers to local variables captured in the returned closure. They are never refreshed from the KV store after construction.

`NewDynamicFeeChecker` uses the captured `feemarketParams` to compute the base fee:

```go
baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
``` [2](#0-1) 

`types.GetBaseFee` calls `feemarketParams.GetBaseFee()`, which checks `NoBaseFee` and returns `params.BaseFee.BigInt()` — both from the stale snapshot: [3](#0-2) 

`MinGasPriceDecorator` similarly reads `mpd.feemarketParams.MinGasPrice` from the stale pointer: [4](#0-3) 

By contrast, `newEthAnteHandler` calls `EVMBlockConfig` inside the closure on every transaction, reading live state: [5](#0-4) 

The same stale-snapshot pattern exists in `newLegacyCosmosAnteHandlerEip712`: [6](#0-5) 

### Impact Explanation

After a governance `MsgUpdateParams` executes on `x/feemarket`:

1. **`NoBaseFee` toggled `true → false` (fee market enabled):** The stale snapshot still has `NoBaseFee=true`, so `feemarketParams.GetBaseFee()` returns `nil`. `NewDynamicFeeChecker` falls back to `checkTxFeeWithValidatorMinGasPrices` instead of enforcing the EIP-1559 base fee. Cosmos-type transactions are accepted with fees below the newly required base fee — a direct fee mis-accounting bug permitting invalid transactions to commit.

2. **`MinGasPrice` raised via governance:** `MinGasPriceDecorator` reads the old lower value from the snapshot. Cosmos txs with fees between the old and new `MinGasPrice` are incorrectly accepted, bypassing the spam-prevention floor.

3. **`BaseFee` directly updated:** `NewDynamicFeeChecker` computes `effectiveFee` using the stale `BaseFee`, causing incorrect fee deduction from user balances — funds mis-accounted.

This matches the allowed High impact: "ante handler bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."

### Likelihood Explanation

Governance parameter updates to `x/feemarket` are a normal, documented chain operation (e.g., activating EIP-1559 post-launch, adjusting `MinGasPrice` for spam control). The `MsgUpdateParams` path is unprivileged in the sense that any governance proposal that passes triggers it. After the update, every Cosmos-type transaction processed by the stale ante handler is affected. No special attacker capability is needed beyond submitting a standard Cosmos transaction after the governance change takes effect.

### Recommendation

Move the params reads inside the returned closure so they are fetched from the live KV store on every transaction, mirroring the pattern used in `newEthAnteHandler`:

```go
func newCosmosAnteHandler(options HandlerOptions, ...) sdk.AnteHandler {
    return func(ctx sdk.Context, tx sdk.Tx, simulate bool) (sdk.Context, error) {
        evmParams := options.EvmKeeper.GetParams(ctx)
        feemarketParams := options.FeeMarketKeeper.GetParams(ctx)
        chainID := options.EvmKeeper.ChainID()
        ethCfg := evmParams.GetChainConfig().EthereumConfig(chainID)
        // build decorators using fresh params ...
    }
}
```

The same fix applies to `newLegacyCosmosAnteHandlerEip712`.

### Proof of Concept

1. Chain starts with `NoBaseFee=true` (fee market disabled). `newCosmosAnteHandler` is called; snapshot captures `NoBaseFee=true`, `MinGasPrice=0`.
2. Governance proposal passes: `MsgUpdateParams` sets `NoBaseFee=false`, `BaseFee=1000000000`, `MinGasPrice=100`.
3. `x/feemarket` KV store now reflects the new params. `BeginBlock` calls `CalculateBaseFee` and sets the live base fee.
4. An attacker submits a Cosmos tx (e.g., `MsgSend`) with `fee=1uatom` (below the new `MinGasPrice=100`).
5. `MinGasPriceDecorator.AnteHandle` reads `mpd.feemarketParams.MinGasPrice` → stale `0` → short-circuits at `minGasPrice.IsZero()` → passes.
6. `NewDynamicFeeChecker` calls `types.GetBaseFee(height, ethCfg, feemarketParams)` → stale `NoBaseFee=true` → returns `nil` → falls back to `checkTxFeeWithValidatorMinGasPrices` → passes (no local min-gas-price set).
7. Transaction commits with fees far below the governance-mandated minimum, bypassing the fee market entirely. [7](#0-6) [8](#0-7) [9](#0-8)

func GetBaseFee(height int64, ethCfg *params.ChainConfig, feemarketParams *feemarkettypes.Params) *big.Int {
