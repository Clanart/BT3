### Title
Locked Native Fee (STRK) Accumulation in StarkNet OmniBridge with No Withdrawal Path — (File: starknet/src/omni_bridge.cairo)

---

### Summary
In `OmniBridge::init_transfer` on StarkNet, the `native_fee` in STRK tokens is transferred from the caller directly into the bridge contract itself. No function exists in the contract or its interface to withdraw or distribute these accumulated STRK fees to relayers or a treasury. Every `init_transfer` call with `native_fee > 0` permanently locks STRK inside the bridge until an admin performs an out-of-band contract upgrade.

---

### Finding Description

In `starknet/src/omni_bridge.cairo`, the `init_transfer` function collects `native_fee` in STRK:

```cairo
if native_fee > 0 {
    let native_token = self.strk_token_address.read();
    let success = IERC20Dispatcher { contract_address: native_token }
        .transfer_from(caller, get_contract_address(), native_fee.into());
    assert(success, 'ERR_FEE_TRANSFER_FAILED');
}
``` [1](#0-0) 

The STRK tokens are transferred to `get_contract_address()` — the bridge itself. The `IOmniBridge` interface exposes no function to withdraw or distribute these accumulated STRK balances: [2](#0-1) 

The relayer compensation model on NEAR uses `claim_fee` to pay the relayer in the **bridged token** (the `fee` field), not in STRK. There is no corresponding STRK payout to the relayer on StarkNet. Consequently, every STRK `native_fee` paid by a user accumulates in the bridge contract with no accounting path for distribution.

The bridge is upgradeable via `IUpgradeable::upgrade`, but recovery requires privileged out-of-band admin intervention — exactly the same condition the external report identified as justifying High severity. [3](#0-2) 

---

### Impact Explanation

Every `init_transfer` call on StarkNet with `native_fee > 0` silently locks STRK in the bridge contract. The STRK is:
- Deducted from the user's wallet (user pays it as a fee)
- Never forwarded to the relayer on StarkNet
- Not tracked in any accounting structure
- Irrecoverable without an admin contract upgrade

This constitutes **fee accounting corruption that misdirects value**: the STRK fee paid by the user does not reach the intended recipient (the relayer), and the bridge accumulates an ever-growing untracked STRK balance. This matches the allowed impact: *"High. Balance, decimal, fee, token-mapping, or accounting corruption that breaks bridge collateralization or misdirects value."*

---

### Likelihood Explanation

Likelihood is **High**. The trigger is any unprivileged user calling `init_transfer` with `native_fee > 0` on the StarkNet bridge — a normal, expected user action. No special role or condition is required. The issue is systematic: it affects every such call, not an edge case.

---

### Recommendation

Add an admin-callable withdrawal function for accumulated STRK native fees, forwarding them to a designated fee recipient or treasury:

```cairo
fn withdraw_native_fees(ref self: ContractState, recipient: ContractAddress) {
    self.accesscontrol.assert_only_role(DEFAULT_ADMIN_ROLE);
    let native_token = self.strk_token_address.read();
    let balance = IERC20Dispatcher { contract_address: native_token }
        .balance_of(get_contract_address());
    if balance > 0 {
        let success = IERC20Dispatcher { contract_address: native_token }
            .transfer(recipient, balance);
        assert(success, 'ERR_FEE_WITHDRAWAL_FAILED');
    }
}
```

Alternatively, forward `native_fee` directly to a pre-configured fee recipient address at the time of `init_transfer`, rather than retaining it in the bridge contract.

---

### Proof of Concept

1. User calls `init_transfer(token_address, 1000, fee=10, native_fee=50, recipient, message)` on the StarkNet OmniBridge.
2. Bridge executes `transfer_from(caller, get_contract_address(), 50)` — 50 STRK moves from the user into the bridge contract.
3. The `InitTransfer` event is emitted; the relayer observes it and submits proof on NEAR.
4. Relayer calls `claim_fee` on NEAR and receives the bridged-token fee (10 units of the bridged token). No STRK is paid to the relayer anywhere.
5. The 50 STRK remains in the StarkNet bridge contract indefinitely.
6. Repeat for every user who pays a `native_fee`: STRK accumulates with no distribution path.
7. Recovery requires the admin to upgrade the contract and add a withdrawal function — out-of-band intervention with no current mechanism. [4](#0-3)

        fn init_transfer(
            ref self: ContractState,
            token_address: ContractAddress,
            amount: u128,
            fee: u128,
            native_fee: u128,
            recipient: ByteArray,
            message: ByteArray,
        ) {
            assert(!_is_paused(@self, PAUSE_INIT_TRANSFER), 'ERR_INIT_TRANSFER_PAUSED');

            assert(amount > 0, 'ERR_ZERO_AMOUNT');
            assert(fee < amount, 'ERR_INVALID_FEE');

            let origin_nonce = self.current_origin_nonce.read() + 1;
            self.current_origin_nonce.write(origin_nonce);

            let caller = get_caller_address();

            if self.is_bridge_token(token_address) {
                IBridgeTokenDispatcher { contract_address: token_address }
                    .burn(caller, amount.into());
            } else {
                let success = IERC20Dispatcher { contract_address: token_address }
                    .transfer_from(caller, get_contract_address(), amount.into());
                assert(success, 'ERR_TRANSFER_FROM_FAILED');
            }

            if native_fee > 0 {
                let native_token = self.strk_token_address.read();
                let success = IERC20Dispatcher { contract_address: native_token }
                    .transfer_from(caller, get_contract_address(), native_fee.into());
                assert(success, 'ERR_FEE_TRANSFER_FAILED');
            }

            self
                .emit(
                    Event::InitTransfer(
                        InitTransfer {
                            sender: caller,
                            token_address,
                            origin_nonce,
                            amount,
                            fee,
                            native_fee,
                            recipient,
                            message,
                        },
                    ),
                )
        }
