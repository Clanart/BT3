### Title
P2P State Diff Sync Accepts Length-Matching but Commitment-Divergent State Diffs, Corrupting Stored State and RPC Outputs - (File: crates/apollo_p2p_sync/src/client/state_diff.rs)

---

### Summary

`parse_data_for_block` in the P2P sync client assembles a `ThinStateDiff` from peer-supplied chunks and terminates only when the accumulated chunk-length equals `header.state_diff_length`. It never verifies that `calculate_state_diff_hash(&assembled_diff) == header.state_diff_commitment`. A malicious peer can relay a legitimately-signed header and then supply wrong-content chunks whose lengths sum to the correct total, causing the node to persist a state diff that diverges from the on-chain commitment. The resulting wrong state diff is then served authoritatively by `starknet_getStateUpdate` and fed into the committer's Patricia-trie update, producing a wrong global root and corrupted proof inputs.

---

### Finding Description

**Root cause — missing commitment check in `parse_data_for_block`** [1](#0-0) 

The loop reads `target_state_diff_len` from the stored header's `state_diff_length` field and exits when `current_state_diff_len == target_state_diff_len`. After the loop, the function returns `Ok(Some((result, block_number)))` with no call to `calculate_state_diff_hash` and no comparison against `header.state_diff_commitment`.

**The commitment that is never checked**

`state_diff_commitment` is stored in `StorageBlockHeader` and is the Poseidon hash of the correct state diff, computed during block production: [2](#0-1) 

It is also the value chained into the block hash: [3](#0-2) 

**What `state_diff_length` actually measures**

`ThinStateDiff::len()` counts deployed contracts, declared classes, deprecated classes, nonces, and individual storage-key entries — it does **not** count empty inner maps: [4](#0-3) 

Two distinct state diffs can therefore share the same `len()` while producing different Poseidon hashes, making the length check a necessary but insufficient guard.

**How the wrong state diff propagates**

After `parse_data_for_block` returns, the assembled diff is written to MDBX via `append_state_diff`: [5](#0-4) 

`starknet_getStateUpdate` reads directly from this table: [6](#0-5) 

The committer then receives the wrong state diff and computes a wrong global root, which it stores as `ForestMetadataType::StateRoot`: [7](#0-6) 

The committer's `verify_state_diff_hash` flag can catch this only if it is enabled and the caller passes the header's `state_diff_commitment` as `Some(commitment)`: [8](#0-7) 

In the P2P sync path the flag may be disabled or the commitment may be passed as `None`, leaving the wrong root silently stored.

---

### Impact Explanation

| Layer | Corrupted value | Matching allowed impact |
|---|---|---|
| MDBX state diff table | Wrong `ThinStateDiff` for block N | High — `starknet_getStateUpdate` returns authoritative-looking wrong storage/nonce/class values |
| Committer forest | Wrong `GlobalRoot` for block N | Critical — wrong state root fed into subsequent block execution and proof inputs |
| Block hash chain | `state_root` in `calculate_block_hash` is wrong | Critical — wrong block hash commitment |
| SNOS / transaction prover | Wrong Patricia-trie preimages | Critical — invalid proof facts accepted or generated | [9](#0-8) 

---

### Likelihood Explanation

Any node reachable over the P2P network can act as the malicious peer. The attacker does not forge a header; it relays a legitimately-signed header (obtained from the honest network) and then substitutes wrong state diff chunks whose individual `StateDiffChunk::len()` values sum to the correct `state_diff_length`. No privileged key material is required. The only prerequisite is that the victim node accepts P2P connections from the attacker, which is the normal operating mode of a syncing node.

---

### Recommendation

After the assembly loop in `parse_data_for_block`, before returning `Ok(Some((result, block_number)))`, add:

```rust
let expected_commitment = storage_reader
    .begin_ro_txn()?
    .get_block_header(block_number)?
    .expect("header must exist")
    .state_diff_commitment
    .ok_or(P2pSyncClientError::OldHeaderInStorage {
        block_number,
        missing_field: "state_diff_commitment",
    })?;

let actual_commitment = calculate_state_diff_hash(&result);
if actual_commitment != expected_commitment {
    return Err(ParseDataError::BadPeer(BadPeerError::WrongStateDiffCommitment {
        expected: expected_commitment,
        actual: actual_commitment,
    }));
}
```

This mirrors the check already present in the committer: [10](#0-9) 

and closes the gap between the length guard and the full commitment invariant.

---

### Proof of Concept

1. Victim node V is syncing block N via P2P.
2. Attacker peer A sends V the legitimately-signed header for block N. The header contains `state_diff_length = L` and `state_diff_commitment = H_correct`.
3. V stores the header and opens a state-diff query session with A.
4. A constructs wrong `StateDiffChunk` values — e.g., replacing storage value `(addr, key) → v_real` with `(addr, key) → v_fake` — while keeping the total chunk-length equal to `L`.
5. `parse_data_for_block` accepts all chunks (length check passes at line 99), returns `Ok(Some((wrong_diff, N)))`. [11](#0-10) 

6. `wrong_diff` is written to MDBX. `starknet_getStateUpdate(block_id=N)` now returns `v_fake` for `(addr, key)` — an authoritative-looking wrong value.
7. The committer applies `wrong_diff` to the Patricia trie, producing `GlobalRoot_wrong ≠ GlobalRoot_correct`. All subsequent blocks built on top of this node's state inherit the corrupted root.
8. Any proof generated from this node's state uses wrong Patricia-trie preimages, producing an invalid or unverifiable proof.

    fn append_state_diff(
        self,
        block_number: BlockNumber,
        thin_state_diff: ThinStateDiff,
    ) -> StorageResult<Self> {
        let file_offset_table = self.txn.open_table(&self.tables.file_offsets)?;
        let markers_table = self.open_table(&self.tables.markers)?;
        let state_diffs_table = self.open_table(&self.tables.state_diffs)?;
        let nonces_table = self.open_table(&self.tables.nonces)?;
        let deployed_contracts_table = self.open_table(&self.tables.deployed_contracts)?;
        let storage_table = self.open_table(&self.tables.contract_storage)?;
        let declared_classes_block_table = self.open_table(&self.tables.declared_classes_block)?;
        let deprecated_declared_classes_block_table =
            self.open_table(&self.tables.deprecated_declared_classes_block)?;
        let compiled_class_hash_table = self.open_table(&self.tables.compiled_class_hash)?;

        // Write state.
        write_deployed_contracts(
            &thin_state_diff.deployed_contracts,
            &self.txn,
            block_number,
            &deployed_contracts_table,
            &nonces_table,
        )?;
        write_storage_diffs(
            &thin_state_diff.storage_diffs,
            &self.txn,
            block_number,
            &storage_table,
        )?;
        // Must be called after write_deployed_contracts since the nonces are updated there.
        write_nonces(&thin_state_diff.nonces, &self.txn, block_number, &nonces_table)?;

        for (class_hash, _) in &thin_state_diff.class_hash_to_compiled_class_hash {
            let not_declared = declared_classes_block_table.get(&self.txn, class_hash)?.is_none();
            if not_declared {
                declared_classes_block_table.insert(&self.txn, class_hash, &block_number)?;
            }
        }

        write_compiled_class_hashes(
            &thin_state_diff.class_hash_to_compiled_class_hash,
            &self.txn,
            block_number,
            &compiled_class_hash_table,
        )?;

        for class_hash in thin_state_diff.deprecated_declared_classes.iter() {
            // Cairo0 classes can be declared in different blocks. The first block to declare the
            // class is recorded here.
            if deprecated_declared_classes_block_table.get(&self.txn, class_hash)?.is_none() {
                deprecated_declared_classes_block_table.insert(
                    &self.txn,
                    class_hash,
                    &block_number,
                )?;
            }
        }

        // Write state diff.
        let location = self.file_handlers.append_state_diff(&thin_state_diff);
        state_diffs_table.append(&self.txn, &block_number, &location)?;
        file_offset_table.upsert(&self.txn, &OffsetKind::ThinStateDiff, &location.next_offset())?;

        update_marker_to_next_block(&self.txn, &markers_table, MarkerKind::State, block_number)?;
