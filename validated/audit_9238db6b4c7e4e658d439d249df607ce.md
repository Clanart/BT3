### Title
`fee_proposal` Bounds Check Bypassed During V0_14_3 Transition Window Allows Unconstrained L2 Gas Price Manipulation - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

The SNIP-35 `fee_proposal` mechanism uses a sliding window of `window_size` (10) prior blocks to compute `fee_actual` (the median). The validator enforces that any incoming `fee_proposal` stays within a multiplicative margin of `fee_actual`. However, this bounds check is **entirely skipped** when `fee_actual` is `None`. `fee_actual` is `None` for the entire `window_size`-block span immediately after the V0_14_3 upgrade, because the window is populated with pre-V0_14_3 blocks whose `fee_proposal_fri` is `None`. During this window, a proposer can publish any arbitrary `fee_proposal` value and every honest validator will accept it, poisoning the median that governs the L2 gas price for all subsequent blocks.

### Finding Description

`compute_fee_actual` iterates over heights `[height - window_size, height - 1]` and returns `None` the moment it encounters any entry that is `None` or absent:

```rust
Some(None) | None => {
    warn!("...");
    return None;
}
``` [1](#0-0) 

Pre-V0_14_3 blocks are stored with `fee_proposal_fri = None`. Therefore, for the first `window_size` blocks after the upgrade (heights `[H, H + window_size - 1]`), the window always contains at least one `None` entry, so `compute_fee_actual` always returns `None`.

The validator's bounds check is guarded by an `if let (Some(fee_actual), Some(fee_proposal))` pattern:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    // ... bounds check ...
}
``` [2](#0-1) 

When `fee_actual` is `None`, the entire bounds check is skipped. A malicious proposer can therefore publish `fee_proposal_fri = u128::MAX` (or any other extreme value) and every validator will accept it, because `proposal_init_validation.fee_actual` is `None` for all heights in the transition window.

The `fee_actual` passed to the validator is computed at call sites in `validate_proposal` and `set_height_and_round`:

```rust
fee_actual: compute_fee_actual(
    &self.fee_proposals_window,
    init.height,
    VersionedConstants::latest_constants().fee_proposal_window_size,
),
```
