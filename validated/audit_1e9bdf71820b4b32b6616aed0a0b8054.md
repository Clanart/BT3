Audit Report

## Title
Unattributed Native ETH Residue in `pay()` Allows Cross-Transaction Theft of Victim's Stranded ETH — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary

`PeripheryPayments.pay()` reads `address(this).balance` as an unattributed shared pool when settling a WETH-input swap. Any native ETH left on the router by a prior user's transaction is silently consumed by the next caller whose `tokenIn` is WETH, giving that caller a free swap at the victim's expense. A second, simpler theft path exists via `refundETH()`, which sends the entire router ETH balance to any caller with no access control.

## Finding Description

`exactInputSingle` is `payable` and accepts arbitrary `msg.value`. When a user sends `value: X` but specifies `amountIn: Y < X`, the swap callback fires `_justPayCallback` → `pay(WETH, victim, pool, Y)`. [1](#0-0) 

Inside `pay()`, `nativeBalance = address(this).balance` equals `X` (the full ETH sent). Since `X ≥ Y`, it deposits exactly `Y` ETH as WETH and sends it to the pool. The remaining `X − Y` stays on the router with no per-user accounting. [2](#0-1) 

The `receive()` guard only blocks plain ETH transfers from non-WETH addresses; it does **not** intercept ETH sent via a `payable` function call, so `exactInputSingle{value: X}` successfully deposits `X` onto the router without reverting. [3](#0-2) 

In a subsequent transaction, an attacker calls `exactInputSingle{value: 0}` with `tokenIn = WETH` and `amountIn = Z ≤ X − Y`. The callback fires `pay(WETH, attacker, pool, Z)`. Because `address(this).balance = X − Y ≥ Z`, the function deposits the victim's stranded ETH and sends WETH to the pool — the attacker receives swap output without spending any ETH or WETH. [4](#0-3) 

The second theft path: `refundETH()` sends the entire router ETH balance to `msg.sender` with no access control, allowing any caller to drain stranded ETH directly without needing a WETH swap. [5](#0-4) 

## Impact Explanation

Direct loss of user ETH principal. Any victim who calls `exactInputSingle` or `exactInput` with `msg.value > amountIn` and omits a `refundETH()` step loses the excess `msg.value − amountIn`. The attacker recovers it in the next block with a zero-cost call. This is a direct, quantifiable loss of user funds meeting Critical/High severity thresholds.

## Likelihood Explanation

Sending a small ETH surplus to guarantee a swap succeeds is a common user pattern. The router is `payable` and does not enforce that callers wrap their call in a `multicall` with `refundETH()`. An MEV bot can monitor the mempool for any `exactInputSingle{value > amountIn}` transaction and back-run with a `refundETH()` call or a zero-value WETH swap in the same block. The exploit is repeatable, requires no special privileges, and is fully automatable.

## Recommendation

Track the `msg.value` contributed by the current top-level call and limit `pay()` to consuming only that amount of native ETH. One approach: record `msg.value` in a transient storage slot at the start of each public entry point and decrement it as ETH is consumed in `pay()`; revert if `pay()` would consume more than the recorded amount. Alternatively, require callers to pre-wrap ETH to WETH before calling the router and remove the native-ETH branch from `pay()` entirely, relying on `multicall` + `unwrapWETH9` for the ETH-in / ETH-out UX.

## Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.35;

contract NativeResidueTheftTest is SimpleRouterTestBase {

    function test_attackerStealsVictimLeftoverETH() public {
        uint128 amountIn = 1_000;
        uint256 victimSends = 2_000; // overpays by 1000 wei

        // --- Victim transaction ---
        vm.deal(victim, victimSends);
        vm.prank(victim);
        router.exactInputSingle{value: victimSends}(
            IMetricOmmSimpleRouter.ExactInputSingleParams({
                pool: address(pool), tokenIn: address(weth),
                recipient: victim, zeroForOne: true,
                amountIn: amountIn, amountOutMinimum: 0,
                deadline: block.timestamp + 1,
                priceLimitX64: type(uint128).max, extensionData: ""
            })
        );
        assertEq(address(router).balance, 1_000, "residue on router");

        // --- Attacker transaction: value 0, no WETH approval needed ---
        uint256 attackerWethBefore = weth.balanceOf(attacker);
        vm.prank(attacker);
        router.exactInputSingle{value: 0}(
            IMetricOmmSimpleRouter.ExactInputSingleParams({
                pool: address(pool), tokenIn: address(weth),
                recipient: attacker, zeroForOne: true,
                amountIn: amountIn, amountOutMinimum: 0,
                deadline: block.timestamp + 1,
                priceLimitX64: type(uint128).max, extensionData: ""
            })
        );

        assertEq(weth.balanceOf(attacker), attackerWethBefore, "attacker WETH unchanged");
        assertEq(address(router).balance, 0, "victim residue consumed");
    }
}
```
