### Title
Missing Cancel/Recovery Function for Pending Transfers Causes Permanent Fund Lock — (`near/omni-bridge/src/lib.rs`)

---

### Summary

The NEAR bridge contract (`near/omni-bridge/src/lib.rs`) has no public function to cancel or recover a pending transfer message once it has been stored. When a user initiates a transfer via `ft_transfer_call` → `init_transfer_internal`, their tokens are immediately burned or locked and a `TransferMessage` is written to `pending_transfers`. The only way to clear that record is through `sign_transfer`, which is restricted to trusted relayers and panics if the token's destination-chain address is not registered. There is no on-chain cancel path for the user or any other party to recover the locked tokens.

---

### Finding Description

**Entry path — user-initiated transfer:**

A user calls `ft_transfer_call` on a NEAR token contract, which triggers `ft_on_transfer` in the bridge: [1](#0-0) 

`ft_on_transfer` dispatches to `init_transfer`, which builds a `TransferMessage` and calls `init_transfer_internal`: [2](#0-1) 

Inside `init_transfer_internal`, the tokens are **immediately burned or locked** and the transfer message is persisted in `pending_transfers`. There is no pre-check that the token has a registered address on the destination chain.

**Stuck path — `sign_transfer` panics:**

The only way to process a pending transfer is `sign_transfer`, which is `#[trusted_relayer]`-gated. It calls `get_token_address` and panics with `FailedToGetTokenAddress` if the mapping is absent: [3](#0-2) 

It also panics if the normalized amount rounds to zero: [4](#0-3) 

In either case the transfer message remains in `pending_transfers` indefinitely.

**No cancel function exists:**

A search of the entire `near/omni-bridge/src/` directory reveals that `remove_transfer_message` is only called from internal callbacks (`sign_transfer_callback`, `submit_transfer_to_utxo_chain_connector`). There is no public `cancel_transfer`, `withdraw_transfer`, or equivalent function that would allow the user, the relayer, or the DAO to reclaim the burned/locked tokens. [5](#0-4) 

**Additional compounding factor — `fast_fin_transfer_to_other_chain`:**

When a relayer performs a fast transfer to a non-NEAR chain, the bridge creates a secondary `TransferMessage` with `sender` set to the bridge contract itself, not the relayer: [6](#0-5) 

Even if a cancel function were added that checked `sender`, the relayer could not invoke it for this secondary message because the bridge contract is recorded as the sender.

---

### Impact Explanation

If a user initiates a transfer for a token that has no registered address on the destination chain (or whose decimal normalization produces zero), their tokens are burned or locked on-chain and the resulting `TransferMessage` can never be cleared. Because no public cancel function exists, the funds are **permanently and irrecoverably locked** in the bridge contract. This matches the allowed impact: *"Permanent freezing, irrecoverable lock, or unclaimable settlement of user or protocol funds in bridge, token, or vault flows."*

---

### Likelihood Explanation

The scenario is reachable by any unprivileged bridge user via the standard `ft_transfer_call` path. Concrete triggers include:

1. A token is registered on NEAR but its destination-chain address mapping (`token_id_to_address`) has not yet been set, or was removed after the transfer was initiated.
2. A token with high origin decimals is transferred in a small amount such that `normalize_amount` returns zero.
3. The MPC signer is permanently decommissioned or the key is rotated without migrating pending transfers.

None of these require privileged access; condition 1 in particular is a realistic operational state during token onboarding.

---

### Recommendation

1. **Add a public `cancel_transfer` function** that allows the transfer owner (stored in `TransferMessageStorageValue::owner`) to cancel a pending transfer and receive a refund of the locked/burned tokens (via mint for burned bridged tokens, or unlock for native tokens).
2. **Pre-validate token registration** inside `init_transfer` before burning or locking tokens: verify that `get_token_address(destination_chain, token_id)` returns `Some` and that the normalized amount is non-zero, and refund the user immediately if either check fails.
3. For the `fast_fin_transfer_to_other_chain` path, record the relayer (not `env::current_account_id()`) as the `sender` of the secondary `TransferMessage` so that a future cancel function can be correctly gated.

---

### Proof of Concept

1. Token `foo.near` is registered in the bridge but has no address mapping for `ChainKind::Sol`.
2. User calls `ft_transfer_call` on `foo.near` with `receiver_id = bridge`, `amount = 1000`, `msg = InitTransferMsg { recipient: Sol("..."), fee: 0, ... }`.
3. `init_transfer_internal` burns 1000 `foo.near` tokens and writes the `TransferMessage` to `pending_transfers`.
4. Any trusted relayer calls `sign_transfer` for this transfer ID.
5. `sign_transfer` executes `get_token_address(Sol, foo.near)` → `None` → `env::panic_str("ERR_FAILED_TO_GET_TOKEN_ADDRESS")`.
6. The transfer message remains in `pending_transfers`. The 1000 tokens are gone. No cancel function exists. Funds are permanently lost. [7](#0-6) [8](#0-7)

    fn init_transfer_internal(
        &mut self,
        transfer_message: TransferMessage,
        storage_owner: AccountId,
    ) -> U128 {
        let required_storage_balance = self
            .add_transfer_message(transfer_message.clone(), storage_owner.clone())
            .saturating_add(NearToken::from_yoctonear(transfer_message.fee.native_fee.0));

        if self
            .try_update_storage_balance(
                storage_owner,
                required_storage_balance,
                NearToken::from_yoctonear(0),
            )
            .is_err()
        {
            self.remove_transfer_message_without_refund(transfer_message.get_transfer_id());
            return transfer_message.amount;
        }

        if let OmniAddress::Near(token_id) = transfer_message.token.clone() {
            self.burn_tokens_if_needed(token_id.clone(), transfer_message.amount);

            self.lock_tokens_if_needed(
                transfer_message.get_destination_chain(),
                &token_id,
                transfer_message.amount.0,
            );
        } else {
            self.remove_transfer_message_without_refund(transfer_message.get_transfer_id());
            return transfer_message.amount;
        }

        env::log_str(&OmniBridgeEvent::InitTransferEvent { transfer_message }.to_log_string());
        U128(0)
    }

impl Contract {
    #[payable]
    pub fn storage_deposit(&mut self, account_id: Option<AccountId>) -> StorageBalance {
        let account_id = account_id.unwrap_or_else(env::predecessor_account_id);
        let amount = env::attached_deposit();
        let storage = self.accounts_balances.get(&account_id).map_or_else(
            || {
                let min_required_storage_balance = self.required_balance_for_account();
                let available = amount
                    .checked_sub(min_required_storage_balance)
                    .near_expect(StorageError::NotEnoughStorageBalanceAttached {
                        required: min_required_storage_balance,
                        attached: amount,
                    });
                StorageBalance {
                    total: amount,
                    available,
                }
            },
            |mut storage| {
                storage.total = storage.total.saturating_add(amount);
                storage.available = storage.available.saturating_add(amount);
                storage
            },
        );
        self.accounts_balances.insert(&account_id, &storage);

        self.resume_promise(&account_id).detach();

        storage
    }

    #[private]
    pub fn resume_promise(&self, account_id: &AccountId) -> PromiseOrValue<()> {
        if let Some(promise_id) = &self.init_transfer_promises.get(account_id) {
            let result = env::promise_yield_resume(promise_id, []);
            env::log_str(&format!("Resume promise. Result: {result}"));

            if !result {
                return Self::ext(env::current_account_id())
                    .resume_promise(account_id)
                    .into();
            }
        }
        PromiseOrValue::Value(())
    }

    #[payable]
    pub fn storage_withdraw(&mut self, amount: Option<NearToken>) -> StorageBalance {
        assert_one_yocto();
        let account_id = env::predecessor_account_id();
        let mut storage = self
            .storage_balance_of(&account_id)
            .near_expect(StorageError::AccountNotRegistered(account_id.clone()));
        let to_withdraw = amount.unwrap_or(storage.available);
        storage.total = storage.total.checked_sub(to_withdraw).near_expect(
            StorageError::NotEnoughStorageBalance {
                requested: to_withdraw,
                available: storage.total,
            },
        );
        storage.available = storage.available.checked_sub(to_withdraw).near_expect(
            StorageError::NotEnoughStorageBalance {
                requested: to_withdraw,
                available: storage.available,
            },
        );

        self.accounts_balances.insert(&account_id, &storage);

        Promise::new(account_id).transfer(to_withdraw).detach();

        storage
    }
