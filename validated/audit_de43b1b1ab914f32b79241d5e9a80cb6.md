### Title
Unregistered ERC20 Token Permanently Locked in EVM Bridge With No Recovery Path - (File: evm/src/omni-bridge/contracts/OmniBridge.sol)

### Summary
`OmniBridge.sol::initTransfer` accepts any arbitrary ERC20 token and locks it in the contract without verifying the token is registered in the bridge's cross-chain mapping. If the token has no corresponding NEAR-side registration, the NEAR `fin_transfer_callback` will always revert, making the locked funds irrecoverable.

### Finding Description
`initTransfer` in `OmniBridge.sol` handles three ERC20 cases: custom minter tokens, bridge tokens (`isBridgeToken`), and a catch-all fallback for any other ERC20. In the fallback branch, the contract performs a `safeTransferFrom` to lock the tokens with no validation that `ethToNearToken[tokenAddress]` is populated: [1](#0-0) 

Specifically, lines 406–412 show the unchecked fallback:

```solidity
} else {
    IERC20(tokenAddress).safeTransferFrom(
        msg.sender,
        address(this),
        amount
    );
}
```

There is no guard such as `require(bytes(ethToNearToken[tokenAddress]).length > 0, "ERR_TOKEN_NOT_REGISTERED")` before locking.

On the NEAR side, when a relayer attempts to finalize the transfer via `fin_transfer_callback`, the first thing it does is look up the token's decimal configuration: [2](#0-1) 

```rust
let decimals = self
    .token_decimals
    .get(&init_transfer.token)
    .near_expect(BridgeError::TokenDecimalsNotFound);
```

For any token not registered on NEAR, `token_decimals` returns `None`, causing an unconditional panic. The `fin_transfer` call fails permanently.

The EVM contract has no admin rescue function, no `withdraw`, and no mechanism to return locked ERC20 tokens to users. The only egress path for locked tokens is `finTransfer`, which requires a valid MPC-signed payload from NEAR — impossible to produce for an unregistered token. [3](#0-2) 

### Impact Explanation
**Critical — Permanent freezing of user funds.** Any ERC20 token sent through `initTransfer` that lacks a NEAR-side registration is irrecoverably locked in the EVM bridge contract. There is no admin escape hatch, no refund path, and no way to generate the MPC signature needed to call `finTransfer` back to the user. The locked amount is bounded only by the user's token balance and approval.

### Likelihood Explanation
**Medium.** The bridge is intended to support arbitrary ERC20 tokens (native assets, not just pre-deployed bridge tokens). A user who holds a legitimate ERC20 that has not yet been registered on NEAR — or who mistypes a token address — can trigger this path with a single transaction. No special privileges are required; `initTransfer` is a public, unpermissioned function callable by any EOA.

### Recommendation
Add a registration check at the top of the `else` branch (or at the function entry for non-ETH tokens) before accepting custody of any ERC20:

```solidity
require(
    bytes(ethToNearToken[tokenAddress]).length > 0 ||
    customMinters[tokenAddress] != address(0) ||
    isBridgeToken[tokenAddress],
    "ERR_TOKEN_NOT_REGISTERED"
);
```

Alternatively, restrict the fallback `safeTransferFrom` path to tokens that appear in `ethToNearToken`, mirroring the existing `isBridgeToken` guard pattern.

### Proof of Concept

1. Token `0xABCD` is a standard ERC20 deployed on Ethereum. It has not been registered via `deployToken` or `addCustomToken`, so `ethToNearToken[0xABCD]` is empty and `isBridgeToken[0xABCD]` is `false`.
2. Alice calls:
   ```solidity
   OmniBridge.initTransfer(
       0xABCD,   // tokenAddress
       1_000e18, // amount
       0,        // fee
       0,        // nativeFee
       "alice.near",
       ""
   );
   ```
3. The fallback branch executes `IERC20(0xABCD).safeTransferFrom(alice, bridge, 1_000e18)`. Alice's 1,000 tokens are now held by the bridge. [4](#0-3) 
4. `InitTransfer` event is emitted. A relayer picks it up and submits a `fin_transfer` proof to the NEAR bridge.
5. `fin_transfer_callback` on NEAR reaches:
   ```rust
   self.token_decimals.get(&init_transfer.token)
       .near_expect(BridgeError::TokenDecimalsNotFound); // panics
   ``` [2](#0-1) 
6. Every `fin_transfer` attempt reverts. No MPC signature for a `finTransfer` back to Alice can ever be produced. Alice's 1,000 tokens are permanently locked in the EVM bridge contract with no recovery path.
