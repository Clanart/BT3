### Title
`execute_replace_class` Allows Replacing Contract Class With Undeclared Class Hash, Enabling Permanent Fund Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS performs a contract class-hash state transition without first verifying that the new class hash corresponds to a declared class. This is the direct analog of the reported `uninstallModule` bug: a state-clearing/transition operation proceeds without validating the prerequisite state. Any contract can call `replace_class` with an arbitrary, undeclared class hash, permanently making itself non-callable and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` (lines 878–916) reads the current `StateEntry` for the calling contract and unconditionally overwrites its `class_hash` field with whatever value the caller supplies in the request:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The developer-acknowledged TODO at line 898 confirms the missing check: `// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.` [1](#0-0) 

By contrast, the `deploy_contract` function (the analogous "install" path) **does** enforce the prerequisite state check — it asserts `state_entry.class_hash = UNINITIALIZED_CLASS_HASH` before proceeding, ensuring a contract is not already deployed: [2](#0-1) 

Similarly, `execute_declare_transaction` enforces `prev_value=0` in the `dict_update` call, ensuring a class can only be declared once: [3](#0-2) 

`execute_replace_class` has no equivalent guard. It does not consult `contract_class_changes` to verify that `request.class_hash` has a corresponding compiled class entry before committing the state update.

---

### Impact Explanation

When a contract's `class_hash` is set to a value that has no entry in `contract_class_changes` (i.e., no compiled class was declared for it), the contract becomes permanently non-callable in all subsequent blocks. The OS cannot load the compiled class for execution, so every future call to the contract will fail. Any funds (tokens, ETH, or other assets) held by the contract are permanently frozen with no recovery path.

This matches the **Critical — Permanent freezing of funds** impact category.

---

### Likelihood Explanation

The `replace_class` syscall is a standard, permissionless operation available to any executing contract. A malicious contract can call it with a crafted, undeclared class hash in a single transaction. No privileged role, leaked key, or external dependency is required. The attacker only needs to deploy a contract and invoke `replace_class` from within it. The attack is deterministic and requires no special network conditions.

---

### Recommendation

Before updating `contract_state_changes`, verify that `request.class_hash` is present in `contract_class_changes` (i.e., it was declared in the current or a prior block). The check should mirror the invariant enforced by `deploy_contract` and `execute_declare_transaction`:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,  // <-- add this implicit arg
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // Verify the new class hash is declared.
    let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
    assert_not_zero(compiled_class_hash);  // 0 means undeclared

    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}
    ...
}
```

---

### Proof of Concept

1. Attacker deploys contract `A` holding user funds (e.g., an ERC20 balance or ETH).
2. Contract `A` calls the `replace_class` syscall with `class_hash = 0xdeadbeef` (any value not present in `contract_class_changes`).
3. `execute_replace_class` in `syscall_impls.cairo` (lines 896–910) reads the current `StateEntry`, constructs a `new_state_entry` with `class_hash = 0xdeadbeef`, and calls `dict_update` — no declared-class check is performed.
4. The OS commits the block. Contract `A`'s on-chain class hash is now `0xdeadbeef`.
5. In any subsequent block, any call to contract `A` requires the OS to load the compiled class for `0xdeadbeef`. No such class exists in `contract_class_changes`.
6. All calls to contract `A` fail permanently. All funds held by contract `A` are frozen with no recovery mechanism. [4](#0-3)

        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }
