Audit Report

## Title
Native ETH sent to non-WETH swap functions is silently stranded and immediately stealable via `refundETH` - (File: metric-periphery/contracts/MetricOmmSimpleRouter.sol / metric-periphery/contracts/base/PeripheryPayments.sol)

## Summary
All four swap entry points in `MetricOmmSimpleRouter` are declared `payable`, causing the EVM to unconditionally accept `msg.value` into the contract's balance before execution. When `tokenIn != WETH`, the internal `pay()` helper falls through to `safeTransferFrom` and never touches `address(this).balance`, leaving any sent ETH stranded. The public, attribution-free `refundETH()` then allows any caller to drain the entire ETH balance, enabling theft of victim funds in the same block.

## Finding Description

**Payable swap functions accept ETH unconditionally.** All four swap entry points — `exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput` — are declared `external payable`. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

The `receive()` guard that rejects ETH from non-WETH senders only applies to plain ETH transfers (no calldata). It does **not** intercept ETH sent alongside a `payable` function call. [5](#0-4) 

**`pay()` only consumes native ETH in the WETH branch.** When `token != WETH` and `payer != address(this)`, execution reaches the `else` branch and calls `safeTransferFrom`. `address(this).balance` is never read or spent; any ETH that arrived via `msg.value` remains on the router after the swap completes. [6](#0-5) 

**`refundETH()` is public and attribution-free.** It sends the router's **entire** ETH balance to `msg.sender` with no record of who deposited the ETH, making stranded funds immediately claimable by any third party. [7](#0-6) 

## Impact Explanation
A user who sends ETH alongside a non-WETH swap loses that ETH permanently to any caller of `refundETH()`. The loss is unbounded — any `msg.value` amount is at risk. Because `refundETH` is stateless and public, the stranded ETH is extractable in the same block by an unrelated party with no special privileges. This constitutes a direct, unprivileged loss of user principal above Sherlock thresholds (High severity).

## Likelihood Explanation
The Uniswap v3-style multicall pattern encourages users to pair swaps with `refundETH()` in a single multicall. Any user or integrator who calls a swap function directly with `msg.value > 0` and `tokenIn != WETH`, or who builds a multicall that sends ETH for a WETH hop but routes through a non-WETH pool first, will strand ETH. MEV bots routinely monitor well-known router addresses for stranded balances and call `refundETH` immediately. No privileged access or special setup is required.

## Recommendation
1. **Reject non-zero `msg.value` when `tokenIn != WETH`:** Add a guard at the top of each swap entry point (or inside `pay()`) that reverts if `msg.value > 0 && token != WETH`.
2. **Alternatively, add per-sender attribution to `refundETH`:** Track per-sender ETH deposits in transient storage and only refund the caller's own contribution.
3. **Post-swap leftover check:** After the swap completes, if `address(this).balance > 0` and `tokenIn != WETH`, revert or auto-refund `msg.sender`.

## Proof of Concept
```
Actors: Alice (victim), Bob (attacker)

1. Alice calls:
   router.exactInputSingle{value: 1 ether}(ExactInputSingleParams({
       pool:             address(pool),
       tokenIn:          address(token1),   // NOT WETH
       tokenOut:         address(token2),
       zeroForOne:       true,
       amountIn:         1_000,
       amountOutMinimum: 0,
       recipient:        alice,
       deadline:         block.timestamp + 1,
       priceLimitX64:    0,
       extensionData:    ""
   }));

   - EVM deposits 1 ETH into router.balance before execution.
   - Swap executes; callback calls pay(token1, alice, pool, 1000).
   - token1 != WETH → safeTransferFrom(alice, pool, 1000) succeeds.
   - address(router).balance == 1 ether (untouched).
   - Alice's 1 ETH is stranded.

2. Bob (same or next block) calls:
   router.refundETH();

   - balance = address(router).balance = 1 ether
   - _transferETH(bob, 1 ether) → Bob receives Alice's 1 ETH.
```
