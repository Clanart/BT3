### Title
Missing `chunk.shard_id() == shard_id` Binding Check in `set_state_header` Allows Cross-Shard State Root Substitution — (`chain/chain/src/state_sync/adapter.rs`)

---

### Summary

`set_state_header` accepts a `ShardStateSyncResponseHeader` from an unprivileged peer and stores it under `StateHeaderKey(shard_id, sync_hash)` without ever verifying that the embedded chunk belongs to `shard_id`. A peer can supply a header whose chunk belongs to a different shard (S2) with a valid merkle proof for S2's position in the block, causing S2's `prev_state_root` to be stored as the authoritative state root for S1. Subsequent `set_state_part` and `apply_state_part` calls then install S2's trie data into S1's `shard_uid`.

---

### Finding Description

`set_state_header` in `chain/chain/src/state_sync/adapter.rs` performs two structural checks on the supplied chunk:

**Check 1 — `validate_chunk_proofs`** (line 380): [1](#0-0) 

This function (in `chain/chain/src/validate.rs`) validates only the chunk's internal consistency: its hash, `tx_root`, and `prev_outgoing_receipts_root`. It does not inspect `chunk.shard_id()` at all. [2](#0-1) 

**Check 2 — `verify_path` against `chunk_headers_root`** (lines 394–403): [3](#0-2) 

This verifies that `ChunkHashHeight(chunk.chunk_hash(), chunk.height_included())` is a leaf in the block's chunk merkle tree at the position encoded by `chunk_proof`. It proves the chunk is *somewhere* in the block — it does **not** prove the chunk is at the index corresponding to `shard_id`. A chunk from S2 with S2's valid merkle proof passes this check unconditionally.

**No `shard_id` binding check exists anywhere in the function.** The `grep` for `shard_id()` in `adapter.rs` returns zero matches, confirming no call to `chunk.shard_id()` is made.

The header is then stored unconditionally: [4](#0-3) 

The key stored is `StateHeaderKey(shard_id=S1, sync_hash)` but the value is the header for S2, whose embedded chunk carries S2's `prev_state_root`.

**Receipt proof check (step 4) does not close the gap.** The inner loop at line 488 hashes receipts as `ReceiptList(shard_id, receipts)` using the *requested* shard S1, but the attacker controls both `receipts` and `root_proofs`. If the attacker provides an empty `incoming_receipts_proofs` list, the loop body never executes. The terminal check at line 506–510 then only requires that `sync_hash.height == prev_chunk_header.height_included()`, which the attacker can satisfy by choosing a chunk included at the appropriate height. [5](#0-4) [6](#0-5) 

**Downstream impact in `set_state_part`** (lines 534–561): it reads the stored header, extracts `state_root` from the embedded chunk's `prev_state_root`, and calls `validate_state_part(shard_id=S1, &state_root_of_S2, ...)`. S2's trie parts validate against S2's root and are then installed into S1's storage. [7](#0-6) 

---

### Impact Explanation

The corrupted value is `DBCol::StateHeaders[StateHeaderKey(S1, sync_hash)]` = a `ShardStateSyncResponseHeader` whose `chunk.prev_state_root()` is S2's state root. Every subsequent state part for S1 is validated against S2's root and written into S1's shard storage. After `apply_state_part` completes, S1's trie contains S2's data. This is a complete state substitution for the syncing shard.

---

### Likelihood Explanation

Any peer acting as a state sync provider (no validator privileges required) can trigger this. State sync is initiated by the syncing node itself by sending `StateRequestHeader` messages to peers; any responding peer can supply a cross-shard header. The attack requires only knowledge of a valid block and its chunk merkle proofs, which are public on-chain data.

---

### Recommendation

Add an explicit shard binding check immediately after `validate_chunk_proofs` in `set_state_header`:

```rust
// After validate_chunk_proofs succeeds:
if chunk.shard_id() != shard_id {
    byzantine_assert!(false);
    return Err(Error::Other(
        "set_shard_state failed: chunk shard_id does not match requested shard_id".into(),
    ));
}
```

The same check should be applied to `prev_chunk_header.shard_id()` when present. The fix location is `chain/chain/src/state_sync/adapter.rs` between lines 385 and 387. [8](#0-7)

### Proof of Concept

Call path (all production code, no test privileges needed):

1. Syncing node sends `StateRequestHeader { shard_id: S1, sync_hash }` to a peer.
2. Peer (attacker) constructs a `ShardStateSyncResponseHeader` containing:
   - `chunk`: a valid `ShardChunk` for S2 fetched from the canonical block at `sync_prev_block`
   - `chunk_proof`: S2's merkle proof from `sync_prev_block.chunk_headers_root`
   - `incoming_receipts_proofs`: empty `Vec`
   - `root_proofs`: empty `Vec`
   - `state_root_node`: S2's state root node (publicly obtainable)
3. Peer sends this as the response for S1.
4. `set_state_header(shard_id=S1, sync_hash, header_for_S2)` is called:
   - `validate_chunk_proofs(chunk_S2)` → passes (chunk is internally valid)
   - `verify_path(chunk_headers_root, chunk_proof_S2, ChunkHashHeight(chunk_S2.hash, height))` → passes (S2's chunk IS in the block at S2's index)
   - No `chunk.shard_id() == shard_id` check
   - Empty receipt proof loop executes 0 iterations
   - Terminal height check passes if `prev_chunk_header.height_included()` is chosen correctly
   - `validate_state_root_node(state_root_node_S2, prev_state_root_S2)` → passes
5. `DBCol::StateHeaders[StateHeaderKey(S1, sync_hash)]` is written with S2's header.
6. `set_state_part(shard_id=S1, ...)` reads S2's `prev_state_root`, validates S2's trie parts against it, and installs them into S1's storage. [9](#0-8)

```rust
pub fn validate_chunk_proofs(
    chunk: &ShardChunk,
    epoch_manager: &dyn EpochManagerAdapter,
) -> Result<bool, Error> {
    let correct_chunk_hash = chunk.compute_header_hash();

    // 1. Checking chunk.header.hash
    let header_hash = chunk.header_hash();
    if header_hash != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }

    // 2. Checking that chunk body is valid
    // 2a. Checking chunk hash
    if chunk.chunk_hash() != &correct_chunk_hash {
        byzantine_assert!(false);
        return Ok(false);
    }
    let height_created = chunk.height_created();
    let outgoing_receipts_root = chunk.prev_outgoing_receipts_root();
    let (transactions, receipts) = (chunk.to_transactions(), chunk.prev_outgoing_receipts());

    // 2b. Checking that chunk transactions are valid
    let (tx_root, _) = merklize(transactions);
    if &tx_root != chunk.tx_root() {
        byzantine_assert!(false);
        return Ok(false);
    }
    // 2c. Checking that chunk receipts are valid
    if height_created == 0 {
        return Ok(receipts.is_empty() && outgoing_receipts_root == &CryptoHash::default());
    } else {
        let shard_layout = {
            let prev_block_hash = chunk.prev_block_hash();
            epoch_manager.get_shard_layout_from_prev_block(&prev_block_hash)?
        };
        let outgoing_receipts_hashes = Chain::build_receipts_hashes(receipts, &shard_layout)?;
        let (receipts_root, _) = merklize(&outgoing_receipts_hashes);
        if &receipts_root != outgoing_receipts_root {
            byzantine_assert!(false);
            return Ok(false);
        }
    }
    Ok(true)
}
```
