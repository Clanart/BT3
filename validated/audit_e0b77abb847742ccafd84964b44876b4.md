### Title
Missing `shard_id` Binding Check in `set_state_header` Allows Cross-Shard State Corruption During State Sync - (File: `chain/chain/src/state_sync/adapter.rs`)

---

### Summary

`ChainStateSyncAdapter::set_state_header` accepts a caller-supplied `shard_id` and a peer-supplied `ShardStateSyncResponseHeader` but never verifies that the chunk embedded in the header actually belongs to the requested `shard_id`. The merkle-inclusion proof only proves the chunk exists somewhere in the block's `chunk_headers_root` tree — it does not bind the chunk to a specific shard position. A malicious peer can supply a valid chunk for shard X while the syncing node requested shard Y, causing the header to be stored under the wrong shard key. All downstream operations (`set_state_part`, `apply_state_part`) then apply shard X's trie data into shard Y's storage slot, permanently corrupting the syncing node's state for shard Y.

---

### Finding Description

In `set_state_header`, the function receives `shard_id` (the shard the node wants to sync) and `shard_state_header` (a `ShardStateSyncResponseHeader` from a peer containing an embedded `ShardChunk`). The validation sequence is:

1. `validate_chunk_proofs(&chunk, ...)` — validates internal chunk body consistency (tx root, receipts root, etc.)
2. `verify_path(*sync_prev_block_header.chunk_headers_root(), chunk_proof, &ChunkHashHeight(chunk.chunk_hash(), chunk.height_included()))` — proves the chunk is included in the block's chunk-headers merkle tree

**The missing check**: at no point is `chunk.shard_id()` compared against the caller-supplied `shard_id`.

The merkle proof in step 2 only proves that a chunk with a given hash is *somewhere* in the block's `chunk_headers_root` tree. A block contains chunks for all shards. A malicious peer can provide a valid chunk for shard X (with a valid merkle proof) in response to a request for shard Y's header, and the proof passes because the chunk genuinely exists in the block.

The corrupted header is then stored under `StateHeaderKey(shard_Y, sync_hash)`:

```rust
// Saving the header data.
let mut store_update = self.chain_store.store().store_update();
let key = borsh::to_vec(&StateHeaderKey(shard_id, sync_hash)).unwrap();
store_update.set_ser(DBCol::StateHeaders, &key, &shard_state_header);
store_update.commit();
```

`set_state_part` then reads this stored header, extracts `prev_state_root` from the embedded chunk (shard X's state root), validates the state part against it, and stores the part under `StatePartKey(sync_hash, shard_Y, part_id)`.

`apply_state_part` then applies the trie changes to `shard_uid` derived from `shard_Y` and `epoch_id`, but the `state_root` and trie data belong to shard X. The comment in `apply_state_part` even asserts "Part was already validated earlier, so could never fail here" — meaning no re-validation occurs at apply time. ### Impact Explanation

A syncing node that accepts a malicious peer's state header for shard Y will:

1. Store shard X's chunk data under shard Y's `StateHeaderKey`
2. Validate and store state parts against shard X's `prev_state_root` under shard Y's `StatePartKey`
3. Apply shard X's trie changes into shard Y's `ShardUId` storage slot (both the trie column and flat storage)
4. Call `set_state_finalize` which applies the chunk (shard X's chunk) as if it were shard Y's chunk, writing incorrect `ChunkExtra`, outgoing receipts, and incoming receipts for shard Y

The node ends up with a permanently corrupted state for shard Y: wrong state root, wrong flat storage, wrong `ChunkExtra`. All subsequent block processing for shard Y will produce incorrect results, causing the node to be unable to participate in consensus or to produce invalid blocks/chunks.

**Severity: High** — state corruption is permanent and undetectable by the node itself (no re-validation at apply time). [5](#0-4) 

---

### Likelihood Explanation

During state sync, a node downloads headers from peers. The `StateSyncDownloadSourcePeer` selects peers to download from, and any peer in the network can serve state sync responses. A single malicious peer that is selected as the download source for a given shard can execute this attack. The attack requires no special privileges — any network peer can respond to state sync requests with a crafted `ShardStateSyncResponseHeader` containing a chunk for the wrong shard. [6](#0-5) 

---

### Recommendation

Add an explicit shard binding check immediately after extracting the chunk from the header in `set_state_header`, before any other validation:

```rust
let chunk = shard_state_header.cloned_chunk();
// Verify the chunk in the header actually belongs to the requested shard.
if chunk.shard_id() != shard_id {
    byzantine_assert!(false);
    return Err(Error::Other(format!(
        "set_shard_state failed: chunk shard_id {:?} does not match requested shard_id {:?}",
        chunk.shard_id(), shard_id
    )));
}
```

Similarly, if `prev_chunk_header` is present, verify `prev_chunk_header.shard_id() == shard_id`. [7](#0-6) 

---

### Proof of Concept

1. A syncing node N begins state sync for shard Y (e.g., `shard_id = ShardId(1)`) at `sync_hash = H`.
2. Malicious peer M intercepts the `StateRequestHeader { shard_id: 1, sync_hash: H }` request.
3. M responds with a `ShardStateSyncResponseHeader` containing:
   - The chunk for shard X (e.g., `shard_id = ShardId(0)`) from the block at `sync_hash - 1`
   - A valid merkle proof that this shard-0 chunk is included in `sync_prev_block.chunk_headers_root()` (this proof is trivially constructable from the block data)
   - Valid receipt proofs and a valid `state_root_node` for shard 0
4. N calls `set_state_header(shard_id=1, sync_hash=H, header_from_M)`.
5. `validate_chunk_proofs` passes (chunk 0 is internally valid).
6. `verify_path` passes (chunk 0 is genuinely in the block's merkle tree).
7. The header is stored under `StateHeaderKey(shard_id=1, sync_hash=H)` containing shard 0's chunk.
8. N calls `set_state_part(shard_id=1, ...)` — reads the stored header, gets shard 0's `prev_state_root`, validates parts against it, stores them under `StatePartKey(H, shard_id=1, part_id)`.
9. N calls `apply_state_part(shard_id=1, state_root=shard0_root, ...)` — applies shard 0's trie data to `ShardUId` for shard 1.
10. N calls `set_state_finalize(shard_id=1, sync_hash=H)` — applies shard 0's chunk transactions and receipts as if they were shard 1's, writing corrupted `ChunkExtra` for shard 1.

Node N now has shard 1's storage populated with shard 0's state, with no indication of corruption. [8](#0-7) [9](#0-8)

    pub fn set_state_finalize(
        &mut self,

        sync_hash: CryptoHash,

            tracing::debug_span!(target: "sync", "chain_update_set_state_finalize", %shard_id, ?sync_hash).entered();
        let (chunk, incoming_receipts_proofs) = match shard_state_header {
            ShardStateSyncResponseHeader::V1(shard_state_header) => (
                ShardChunk::V1(shard_state_header.chunk),
                shard_state_header.incoming_receipts_proofs,
            ),
            ShardStateSyncResponseHeader::V2(shard_state_header) => {
                (shard_state_header.chunk, shard_state_header.incoming_receipts_proofs)
            }
        };

        // Note that block headers are already synced and can be taken
        // from store on disk.
        let block_header = get_block_header_on_chain_by_height(
            &self.chain_store_update.chain_store(),
            &sync_hash,
            chunk.height_included(),
        )?;

        // Getting actual incoming receipts.
        let mut receipt_proof_responses: Vec<ReceiptProofResponse> = vec![];
        for incoming_receipt_proof in &incoming_receipts_proofs {
            let ReceiptProofResponse(hash, _) = incoming_receipt_proof;
            let block_header = self.chain_store_update.get_block_header(hash)?;
            if block_header.height() <= chunk.height_included() {
                receipt_proof_responses.push(incoming_receipt_proof.clone());
            }
        }
        let receipts = collect_receipts_from_response(&receipt_proof_responses);
        let is_genesis = block_header.height() == self.chain_store_update.get_genesis_height();
        let prev_block_header = (!is_genesis)
            .then(|| self.chain_store_update.get_block_header(block_header.prev_hash()))
            .transpose()?;

        // Prev block header should be present during state sync, since headers have been synced at
        // this point, except for genesis.
        let gas_price = if let Some(prev_block_header) = &prev_block_header {
            prev_block_header.next_gas_price()
        } else {
            block_header.next_gas_price()
        };

        let chunk_header = chunk.cloned_header();
        let gas_limit = chunk_header.gas_limit();
        let block = self.chain_store_update.get_block(block_header.hash())?;
        let transactions = chunk.to_transactions().to_vec();
        let transaction_validity = if let Some(prev_block_header) = prev_block_header {
            self.chain_store_update
                .chain_store()
                .compute_transaction_validity(&prev_block_header, &chunk)
        } else {
            vec![true; transactions.len()]
        };
        let transactions = SignedValidPeriodTransactions::new(transactions, transaction_validity);
        let shard_uid =
            shard_id_to_uid(self.epoch_manager.as_ref(), shard_id, block_header.epoch_id())?;

            .runtime_adapter
            .get_tries()
            .maybe_pin_memtrie_root(shard_uid, chunk_header.prev_state_root())?;
        let apply_result = self.runtime_adapter.apply_chunk(
            RuntimeStorageConfig::new(chunk_header.prev_state_root(), true),
            ApplyChunkReason::UpdateTrackedShard,
            ApplyChunkShardContext {
                shard_uid,
                gas_limit,
                last_validator_proposals: chunk_header.prev_validator_proposals(),
                is_new_chunk: true,
                on_post_state_ready: None,
                memtrie_pin,
            },
            ApplyChunkBlockContext {
                block_type: BlockType::Normal,
                height: chunk_header.height_included(),
                prev_block_hash: *chunk_header.prev_block_hash(),
                block_timestamp: block_header.raw_timestamp(),
                gas_price,
                random_seed: *block_header.random_value(),
                congestion_info: block.block_congestion_info(),
                bandwidth_requests: block.block_bandwidth_requests(),
            },
            &receipts,
            transactions,
        )?;

        self.chain_store_update.save_chunk(chunk);

        // inline; everything else goes through the shared helper.

            block.as_ref(),

            &config,

            self.chain_store_update.save_incoming_receipt(
                &receipt_proof_response.0,
                shard_id,
                receipt_proof_response.1,
            );
        }
        Ok(shard_uid)

    pub fn set_state_finalize(
        &mut self,
        shard_id: ShardId,
        sync_hash: CryptoHash,
    ) -> Result<(), Error> {
        let shard_state_header = self.state_sync_adapter.get_state_header(shard_id, sync_hash)?;
        let mut height = shard_state_header.chunk_height_included();
        let mut chain_update = self.chain_update();
        let shard_uid = chain_update.set_state_finalize(shard_id, sync_hash, shard_state_header)?;
        chain_update.commit()?;

        // We restored the state on height `shard_state_header.chunk.header.height_included`.
        // Now we should build a chain up to height of `sync_hash` block.
        loop {
            height += 1;
            let mut chain_update = self.chain_update();
            // Result of successful execution of set_state_finalize_on_height is bool,
            // should we commit and continue or stop.
            if chain_update.set_state_finalize_on_height(height, shard_id, sync_hash)? {
                chain_update.commit()?;
            } else {
                break;
            }
        }

        let flat_storage_manager = self.runtime_adapter.get_flat_storage_manager();
        if let Some(flat_storage) = flat_storage_manager.get_flat_storage_for_shard(shard_uid) {
            let header = self.get_block_header(&sync_hash)?;
            flat_storage.update_flat_head(header.prev_hash()).unwrap();
        }

        Ok(())
    }
