### Title
Integer Truncation in `feeCap` Derivation Enables Complete Fee Bypass for Cosmos Txs When BaseFee Is Zero - (File: `ante/evm/fee_checker.go`)

---

### Summary

`NewDynamicFeeChecker` derives the per-gas fee cap via integer (truncating) division `feeCap = fee / gas`. When a user submits a Cosmos tx whose total fee amount is less than the gas limit, `feeCap` truncates to 0. If the chain's EIP-1559 `baseFee` is also 0 at that moment, the insufficient-fee guard passes, `effectivePrice` resolves to 0, and the function returns `effectiveFee = 0` coins â€” meaning the Cosmos SDK `DeductFeeDecorator` deducts nothing from the sender while the transaction is still admitted and executed.

---

### Finding Description

In `ante/evm/fee_checker.go`, `NewDynamicFeeChecker` computes the per-gas fee cap as:

```go
// ante/evm/fee_checker.go:83
feeCap := fee.Quo(sdkmath.NewIntFromUint64(gas))
```

`sdkmath.Int.Quo` performs truncating integer division. Whenever the total fee amount (`fee`) is smaller than the gas limit (`gas`), the quotient truncates to 0.

The guard that follows is:

```go
// ante/evm/fee_checker.go:86-88
if feeCap.LT(baseFeeInt) {
    return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, ...)
}
```

When `baseFee = 0`, the comparison is `0 < 0 = false`, so the check is silently bypassed.

The effective price is then computed via:

```go
// ante/evm/fee_checker.go:91
effectivePrice := sdkmath.NewIntFromBigInt(types.EffectiveGasPrice(baseFeeInt.BigInt(), feeCap.BigInt(), maxPriorityPrice.BigInt()))
```

`EffectiveGasPrice` is defined as:

```go
// x/evm/types/utils.go:232-234
func EffectiveGasPrice(baseFee *big.Int, feeCap *big.Int, tipCap *big.Int) *big.Int {
    return ethermint.BigMin(new(big.Int).Add(tipCap, baseFee), feeCap)
}
```

With `baseFee = 0` and `feeCap = 0`: `min(tipCap + 0, 0) = 0`. Therefore:

```go
// ante/evm/fee_checker.go:94-99
effectiveFee := sdk.Coins{{
    Denom:  denom,
    Amount: effectivePrice.Mul(sdkmath.NewIntFromUint64(gas)), // 0 * gas = 0
}}
return effectiveFee, priority, nil
```

The function returns `effectiveFee = 0`. The Cosmos SDK `DeductFeeDecorator` uses the `TxFeeChecker` return value as the amount to deduct; it deducts 0 coins from the sender. The transaction is admitted and executed with no fee payment.

`baseFee = 0` is reachable via two paths:
1. `CalculateBaseFee` in `x/feemarket/keeper/eip1559.go:103-104` clamps the base fee to `max(computed, minGasPrice)`. When `MinGasPrice = 0` (a common default) and blocks are consistently under-utilized, `baseFeeDelta` can drive `baseFee` to 0.
2. `GetBaseFee` in `x/evm/types/utils.go:244-254` returns `new(big.Int)` (= 0) when `feemarketParams.GetBaseFee()` returns nil, which can occur during certain param states.

---

### Impact Explanation

An unprivileged user can submit a Cosmos tx (e.g., a governance vote, IBC transfer, or any SDK message) with `fee = 1 aevmos` and `gas = 1,000,000`. The `feeCap` truncates to 0, the zero-baseFee guard passes, and `effectiveFee = 0` is returned. The `DeductFeeDecorator` deducts 0 from the sender. The transaction executes with no fee payment, bypassing the fee market entirely. This is a direct mis-accounting of user fees in the ante handler, matching the allowed High impact: "ante handler bug that permits valid user funds/fees to be mis-accounted."

---

### Likelihood Explanation

Medium. The condition requires `baseFee = 0`, which occurs when `MinGasPrice = 0` (the default on many Ethermint-based chains) and blocks are consistently below the gas target. This is a realistic steady-state on low-traffic chains. The attacker-controlled input (`fee < gas`) is trivially constructable by any user.

---

### Recommendation

Replace the truncating integer division with a ceiling division or a direct comparison of the total fee against `baseFee * gas`:

```go
// Option A: reject if total fee < baseFee * gas (no per-gas derivation needed)
requiredFee := baseFeeInt.Mul(sdkmath.NewIntFromUint64(gas))
if fee.LT(requiredFee) {
    return nil, 0, errorsmod.Wrapf(errortypes.ErrInsufficientFee, ...)
}

// Option B: ceiling division for feeCap
// feeCap = ceil(fee / gas) = (fee + gas - 1) / gas
feeCap := fee.Add(sdkmath.NewIntFromUint64(gas - 1)).Quo(sdkmath.NewIntFromUint64(gas))
```

Additionally, add an explicit guard for `gas = 0` before line 83 to prevent a division-by-zero panic.

---

### Proof of Concept

**Setup**: Chain with London hardfork enabled, `MinGasPrice = 0`, blocks consistently under-utilized so `baseFee` has decayed to 0.

**Attack**:
1. Attacker constructs a Cosmos tx (e.g., `MsgSend`) with:
   - `fee = [{denom: "aevmos", amount: 1}]`
   - `gas = 1_000_000`
2. Tx reaches `NewDynamicFeeChecker`:
   - `fee = 1`, `gas = 1_000_000`
   - `feeCap = 1 / 1_000_000 = 0` (truncated) [1](#0-0) 
   - `baseFeeInt = 0` (baseFee decayed to 0) [2](#0-1) 
   - `feeCap.LT(baseFeeInt)` â†’ `0 < 0` = false â†’ guard bypassed [3](#0-2) 
   - `effectivePrice = EffectiveGasPrice(0, 0, MaxInt64) = min(MaxInt64, 0) = 0` [4](#0-3) 
   - `effectiveFee = 0 * 1_000_000 = 0` [5](#0-4) 
3. `DeductFeeDecorator` deducts 0 aevmos from the attacker.
4. Tx executes successfully with no fee paid.
