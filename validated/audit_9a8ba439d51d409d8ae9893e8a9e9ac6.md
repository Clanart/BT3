### Title
`SyncStateReader::get_compiled_class_hash` Is `todo!()` — Gateway Panics on Declare Stateful Validation - (File: `crates/apollo_gateway/src/sync_state_reader.rs`)

---

### Summary

`SyncStateReader` implements the `BlockifierStateReader` (`StateReader`) trait but leaves `get_compiled_class_hash` as a bare `todo!()`. Any Declare transaction that reaches stateful validation in the gateway causes a runtime panic at this call site, making the gateway reject every valid Declare transaction before it can be sequenced.

---

### Finding Description

The `StateReader` trait in `crates/blockifier/src/state/state_api.rs` declares `get_compiled_class_hash` as a required, non-defaulted method:

```rust
fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash>;
``` [1](#0-0) 

`SyncStateReader` is the production `BlockifierStateReader` implementation used during gateway stateful validation. Its implementation of this method is:

```rust
fn get_compiled_class_hash(&self, _class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    todo!()
}
``` [2](#0-1) 

`SyncStateReader` is instantiated inside `SyncStateReaderFactory::get_blockifier_state_reader_and_gateway_fixed_block_from_latest_block` and handed to the blockifier as the underlying reader for `CachedState` during stateful validation. [3](#0-2) 

During stateful validation of a Declare transaction, `CachedState` calls `get_compiled_class_hash` on its underlying reader to check whether the class has already been declared with a different compiled class hash. This call propagates to `SyncStateReader::get_compiled_class_hash`, which panics unconditionally via `todo!()`.

The `SyncOrGenesisStateReader` enum delegates `get_compiled_class_hash` directly to `SyncStateReader`, so the panic is not intercepted: [4](#0-3) 

The `get_compiled_class_hash` method is called in at least 12 places inside `cached_state.rs` alone, confirming it is a live production call path.

---

### Impact Explanation

Every Declare transaction submitted to the gateway triggers stateful validation. During that validation the blockifier reads the compiled class hash of the declared class through `CachedState → SyncStateReader::get_compiled_class_hash`, which panics. The gateway either propagates the panic as an internal error (rejecting the transaction) or, depending on the async runtime's panic handling, crashes the gateway process entirely. In either case, **no Declare transaction can be successfully admitted to the mempool**, which means no new Cairo 1 contract class can ever be deployed through the live sequencer.

This matches the impact: **High — Mempool/gateway/RPC admission rejects valid transactions before sequencing.**

---

### Likelihood Explanation

Any user submitting a Declare transaction to the sequencer triggers this path unconditionally. No special privileges, malformed input, or race conditions are required. The trigger is the normal, documented transaction type.

---

### Recommendation

Implement `get_compiled_class_hash` on `SyncStateReader` by delegating to the state-sync client, mirroring the pattern used for `get_storage_at`, `get_nonce_at`, and `get_class_hash_at`:

```rust
fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
    self.runtime
        .block_on(
            self.state_sync_client
                .get_compiled_class_hash(self.block_number, class_hash),
        )
        .map_err(|e| StateError::StateReadError(e.to_string()))
}
```

The `StateSyncClient` trait already exposes `get_compiled_class_hash` at `crates/apollo_state_sync_types/src/communication.rs`, so no new RPC surface is needed. [5](#0-4) 

---

### Proof of Concept

1. Start the sequencer with `SyncStateReaderFactory` (the default production path).
2. Submit any valid Declare V2/V3 transaction via `starknet_addDeclareTransaction`.
3. The gateway calls `extract_state_nonce_and_run_validations`, which instantiates a `CachedState<SyncStateReader>` and runs blockifier validation.
4. The blockifier calls `state.get_compiled_class_hash(class_hash)` on the `CachedState`.
5. `CachedState` misses its local cache and calls `self.state.get_compiled_class_hash(class_hash)` on the underlying `SyncStateReader`.
6. `SyncStateReader::get_compiled_class_hash` executes `todo!()`, panicking with `"not yet implemented"`.
7. The gateway returns an internal error to the caller; the transaction is never forwarded to the mempool. [6](#0-5) [2](#0-1)
