### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Freezing — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS does not verify that the new class hash supplied by a contract is actually a declared class. An unprivileged contract deployer can call `replace_class` with an arbitrary, undeclared class hash. The OS accepts this state transition and generates a valid proof. After the block is finalized on L1, the contract's class hash in the global state permanently points to an undeclared class, making the contract uncallable and freezing any funds it holds.

---

### Finding Description

In `execute_replace_class`, the OS reads the requested `class_hash` from the syscall request and writes it directly into `contract_state_changes` without any check that the hash corresponds to a declared class:

```cairo
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The TODO comment at line 898 explicitly acknowledges the missing check. The `contract_class_changes` dictionary (which tracks declared classes) is not consulted at all during `replace_class` execution. [2](#0-1) 

The syscall is dispatched unconditionally for any executing contract:

```cairo
if (selector == REPLACE_CLASS_SELECTOR) {
    execute_replace_class(contract_address=execution_context.execution_info.contract_address);
``` [3](#0-2) 

The class declaration path (`execute_declare_transaction`) enforces `prev_value=0` to prevent double-declaration, and `assert_not_zero(compiled_class_hash)` to prevent zero hashes. But `replace_class` has no symmetric check against the declared-class dictionary. [4](#0-3) 

The analogy to the external report is direct: just as the veNFT report shows that a resource (staking power) can be used after it has been transferred away — because the "used" state is not invalidated — here a class hash reference can be written into contract state without the OS verifying the referenced resource (the class) actually exists. In both cases, the OS/contract accepts a state transition referencing a resource it never validates.

---

### Impact Explanation

After a block containing a `replace_class(undeclared_hash)` call is proven and accepted on L1:

1. The contract's `class_hash` field in the global Patricia state permanently equals `undeclared_hash`.
2. Any future transaction that calls this contract requires the prover to supply class facts for `undeclared_hash`. Since the class was never declared, no such facts exist.
3. No valid proof can ever be generated for a block that executes this contract.
4. All funds (tokens, ETH) held by the contract are permanently inaccessible.

This matches the allowed impact: **Critical — Permanent freezing of funds**.

The state commitment machinery in `compute_contract_state_commitment` and `hash_contract_state_changes` faithfully commits the corrupted `class_hash` into the Merkle root, making the freeze irreversible once the proof is accepted on L1. [5](#0-4) 

---

### Likelihood Explanation

- **Reachable by unprivileged users**: Any user can deploy a contract (via `deploy_account` or the `deploy` syscall) and call it. No privileged role is required.
- **No external dependency**: The vulnerability is entirely within the OS Cairo code.
- **Accidental trigger**: A contract developer could accidentally call `replace_class` with a hash that has not yet been declared (e.g., a hash computed off-chain but not yet submitted in a `declare` transaction). The OS would accept this and freeze the contract.
- **Malicious trigger**: An attacker deploys a contract that accepts user deposits, then calls `replace_class` with an arbitrary undeclared hash, permanently freezing deposited funds.
- The TODO comment at line 898 confirms the development team is aware this check is missing and has not yet implemented it. [6](#0-5) 

---

### Recommendation

In `execute_replace_class`, before writing the new class hash to `contract_state_changes`, verify that `class_hash` exists in `contract_class_changes` (i.e., it has been declared in the current or a prior block). Concretely:

1. Perform a `dict_read` on `contract_class_changes` with `key=class_hash`.
2. Assert the returned `compiled_class_hash` is non-zero (i.e., the class is declared).
3. Only then proceed with the `dict_update` on `contract_state_changes`.

This mirrors the existing protection in `execute_declare_transaction` where `assert_not_zero(compiled_class_hash)` and `prev_value=0` together enforce class uniqueness and existence. [2](#0-1) 

---

### Proof of Concept

**Attack sequence (block-level):**

1. **Block N — Setup**: Attacker deploys contract `Vault` that accepts ETH deposits. Victims deposit funds.

2. **Block N+1 — Attack**: Attacker calls `Vault.freeze()`, which internally executes:
   ```
   replace_class(0xdeadbeef...undeclared_hash)
   ```
   The OS executes `execute_replace_class`:
   - Reads `class_hash = 0xdeadbeef...` from the syscall request.
   - Skips the missing declared-class check (TODO at line 898).
   - Writes `StateEntry(class_hash=0xdeadbeef..., ...)` into `contract_state_changes`.
   - The revert log records the old class hash (for potential revert within the same tx), but the transaction is not reverted.

3. **Block N+1 — Proof**: The OS generates a valid STARK proof for block N+1. The proof is valid because `0xdeadbeef...` is never *executed* in this block — only stored. `validate_compiled_class_facts_post_execution` only checks classes that were actually executed. [7](#0-6) 

4. **L1 Verification**: The proof is submitted and verified on L1. The new global state root — which encodes `Vault.class_hash = 0xdeadbeef...` — is accepted.

5. **Block N+2 onward — Freeze**: Any transaction attempting to call `Vault` requires the prover to load class facts for `0xdeadbeef...`. Since this class was never declared, no facts exist. The prover cannot produce a valid proof. `Vault` is permanently uncallable. All deposited funds are frozen forever.

// Replaces the class.
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}

func hash_contract_state_changes{hash_ptr: HashBuiltin*, range_check_ptr}(
    contract_address: felt,

    patricia_update_constants: PatriciaUpdateConstants*,
    hashed_state_changes: DictAccess*,

    alloc_locals;

    local initial_contract_state_root;
    local final_contract_state_root;

    %{ SetPreimageForCurrentCommitmentInfo %}

    local state_dict_start: DictAccess* = prev_state.storage_ptr;
    local state_dict_end: DictAccess* = new_state.storage_ptr;

    // Call patricia_update_using_update_constants() (or the read-optimized variant) instead of

    local should_use_read_optimized: felt;
    %{ ShouldUseReadOptimizedPatriciaUpdate %}
    if (should_use_read_optimized != 0) {
        patricia_update_read_optimized(

            update_ptr=state_dict_start,
            n_updates=n_updates,
            height=MERKLE_HEIGHT,
            prev_root=initial_contract_state_root,
            new_root=final_contract_state_root,
        );

        patricia_update_using_update_constants(
            patricia_update_constants=patricia_update_constants,
            update_ptr=state_dict_start,
            n_updates=n_updates,
            height=MERKLE_HEIGHT,
            prev_root=initial_contract_state_root,
            new_root=final_contract_state_root,
        );
    }
    local range_check_ptr = range_check_ptr;

    let (prev_value) = get_contract_state_hash(
        class_hash=prev_state.class_hash,
        storage_root=initial_contract_state_root,
        nonce=prev_state.nonce,
    );
    assert hashed_state_changes.prev_value = prev_value;
    let (new_value) = get_contract_state_hash(
        class_hash=new_state.class_hash,
        storage_root=final_contract_state_root,
        nonce=new_state.nonce,
    );

    assert hashed_state_changes.new_value = new_value;
    assert hashed_state_changes.key = contract_address;

    return ();
