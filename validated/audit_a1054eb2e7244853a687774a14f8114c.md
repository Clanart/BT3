### Title
Validator Discards Oracle-Computed WEI Prices and Never Checks Proposer's `l1_gas_price_wei`, Allowing Arbitrary ETH-Denominated Gas Prices to Corrupt `l2_gas_price_wei` in Committed Block Header - (File: `crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

---

### Summary

`is_block_info_valid` fetches both FRI and WEI L1 gas prices from the oracle but silently discards the WEI result (`_l1_gas_prices_wei`). It only validates the proposer's FRI prices within a 10 % margin. The proposer's `l1_gas_price_wei` and `l1_data_gas_price_wei` fields in `ProposalInit` are never compared against the oracle. `convert_to_sn_api_block_info` then uses those unvalidated WEI values verbatim to (a) set `eth_gas_prices.l1_gas_price` / `l1_data_gas_price` in the executed `BlockInfo`, and (b) derive `eth_to_fri_rate` via `calculate_eth_to_fri_rate`, which is immediately used to compute `l2_gas_price_wei`. A proposer who keeps `l1_gas_price_fri` within the 10 % window while setting `l1_gas_price_wei` to an arbitrary value can therefore embed any `l2_gas_price_wei` it chooses into the block that every honest validator will accept and execute.

---

`calculate_eth_to_fri_rate` computes the rate as `l1_gas_price_fri * WEI_PER_ETH / l1_gas_price_wei`:

```rust

``` [3](#0-2) 

Because `l1_gas_price_wei` is proposer-controlled and unvalidated, the derived `eth_to_fri_rate` is also proposer-controlled, and therefore so is `l2_gas_price_wei`.

The resulting `BlockInfo` is passed directly to the batcher for execution:

```rust

``` [4](#0-3) 

**`ProposalInit` carries both WEI and FRI fields, but only FRI is guarded:** [5](#0-4) 

---

### Impact Explanation

The corrupted `l2_gas_price_wei` (and `l1_gas_price_wei` / `l1_data_gas_price_wei`) is committed into the executed `BlockInfo` and propagated to:

1. **Fee calculation for every ETH-denominated transaction in the block.** `eth_gas_prices.l2_gas_price` is used by the blockifier to charge users paying in ETH. A proposer who sets `l1_gas_price_wei` to 1/N of the true oracle value inflates `eth_to_fri_rate` by N, deflating `l2_gas_price_wei` by N — making ETH-denominated L2 gas effectively free.

2. **`eth_gas_prices.l1_gas_price` and `l1_data_gas_price`** are set directly from the unvalidated WEI fields and govern fees for L1 handler transactions.

3. **Block header commitment.** `CentralBlockInfo` serialises `price_in_wei` for all three gas types and is uploaded to the central blob; the wrong values become part of the canonical state diff sent to L1. [6](#0-5) 

This matches the allowed impact: **"Critical. Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact."**

---

### Likelihood Explanation

Any consensus proposer (a sequencer node that wins a round) can trigger this without any external dependency. The proposer constructs `ProposalInit` locally in `initiate_build` and can freely choose `l1_gas_price_wei`. Honest validators will accept the proposal because `is_block_info_valid` only checks FRI prices. The attack requires no special privilege beyond being selected as proposer for one round. [7](#0-6) 

---

### Recommendation

In `is_block_info_valid`, validate the proposer's WEI prices against the oracle-computed WEI prices with the same `within_margin` check already applied to FRI prices:

```rust
let (l1_gas_prices_fri, l1_gas_prices_wei) = get_l1_prices_in_fri_and_wei(...).await;

// existing FRI check ...

// add WEI check:
if !(within_margin(init_proposed.l1_gas_price_wei,     l1_gas_prices_wei.l1_gas_price,      margin)
  && within_margin(init_proposed.l1_data_gas_price_wei, l1_gas_prices_wei.l1_data_gas_price, margin))
{
    return Err(ValidateProposalError::InvalidBlockInfo(...));
}
```

This mirrors the existing FRI validation and ensures the ETH-denominated prices embedded in the block are consistent with the oracle, preventing manipulation of `eth_to_fri_rate` and the derived `l2_gas_price_wei`. [8](#0-7) 

---

### Proof of Concept

1. A proposer node wins a consensus round and calls `initiate_build`.
2. It constructs `ProposalInit` with:
   - `l1_gas_price_fri` = oracle value (passes the 10 % FRI margin check)
   - `l1_gas_price_wei` = `oracle_wei / 1000` (1000× lower than actual; **never checked**)
3. `is_block_info_valid` on every honest validator node:
   - Fetches oracle FRI prices → within 10 % → **passes**
   - Fetches oracle WEI prices → **discards them** (`_l1_gas_prices_wei`)
   - Returns `Ok(())`
4. `convert_to_sn_api_block_info` runs on every validator:
   - `eth_to_fri_rate` = `l1_gas_price_fri * 10^18 / (oracle_wei/1000)` = **1000× the true rate**
   - `l2_gas_price_wei` = `l2_gas_price_fri * 10^18 / eth_to_fri_rate` = **1/1000 of the correct value**
5. The batcher executes all transactions with `l2_gas_price_wei` = near-zero → ETH-denominated L2 gas fees are effectively zero.
6. The block is finalised; `CentralBlockInfo` with the manipulated WEI prices is uploaded to the central blob and committed to L1. [9](#0-8) [10](#0-9)

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let previous_block_info = PreviousBlockInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}

fn calculate_eth_to_fri_rate(block_info: &PreviousBlockInfo) -> Result<u128, StarknetApiError> {
    let eth_to_fri_rate = block_info
        .l1_prices_fri
        .l1_gas_price
        .0
        .checked_mul(WEI_PER_ETH)
        .ok_or_else(|| {
            StarknetApiError::GasPriceConversionError(format!(
                "Gas price in Fri should be small enough to multiply by WEI_PER_ETH. Previous \
                 block info: {:?}",
                block_info
            ))
        })?
        .checked_div(block_info.l1_prices_wei.l1_gas_price.0)
        .ok_or_else(|| {
            StarknetApiError::GasPriceConversionError(format!(
                "Gas price in Wei should be non-zero. Previous block info: {:?}",
                block_info
            ))
        })?;
    if eth_to_fri_rate == 0 {
        return Err(StarknetApiError::GasPriceConversionError(format!(
            "Eth to fri rate is zero. Previous block info: {:?}",
            block_info
        )));
    }
    Ok(eth_to_fri_rate)
