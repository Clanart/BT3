### Title
`ProofOfInclusion::valid()` Final Root-Hash Check Is Tautological — Forged Inclusion Proofs Always Pass - (File: crates/chia-datalayer/src/merkle/proof_of_inclusion.rs)

### Summary

`ProofOfInclusion::valid()` ends with `existing_hash == self.root_hash()`, but `self.root_hash()` is derived from the proof's own last layer (`last.combined_hash`). Because the loop body already enforces `existing_hash == layer.combined_hash` at every step, the final comparison is a tautology — it is always `true` when the loop completes. The function therefore never checks the proof against the actual committed tree root, directly analogous to the external report's use of a derived tick-price instead of the stored `sqrt_price`.

### Finding Description

`ProofOfInclusion::valid()` is defined as:

```rust
pub fn valid(&self) -> bool {
    let mut existing_hash = self.node_hash;

    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );

        if calculated_hash != layer.combined_hash {   // ← enforces equality
            return false;
        }

        existing_hash = calculated_hash;              // ← so existing_hash == layer.combined_hash
    }

    existing_hash == self.root_hash()                 // ← self.root_hash() == last.combined_hash == existing_hash
}
``` [1](#0-0) 

`root_hash()` is:

```rust
pub fn root_hash(&self) -> Hash {
    if let Some(last) = self.layers.last() {
        last.combined_hash          // ← same field the loop already verified
    } else {
        self.node_hash
    }
}
``` [2](#0-1) 

After the loop, `existing_hash` equals the last layer's `combined_hash` (because the loop returned `false` otherwise). `self.root_hash()` also returns the last layer's `combined_hash`. The final comparison is therefore `combined_hash_last == combined_hash_last`, which is unconditionally `true`.

The correct value to compare against is the **externally-known, on-chain-committed tree root** — not a value extracted from the proof itself. This is the exact same class of error as the external report: a derived/self-referential value (`get_sqrt_ratio_at_tick(current_tick)` / `self.root_hash()`) is used in place of the authoritative stored value (`self.sqrt_price.get()` / the actual committed root).

### Impact Explanation

An attacker who can supply a `ProofOfInclusion` to any caller that relies solely on `proof.valid()` can:

1. Choose an arbitrary `node_hash` (claiming any key/value pair is present in the tree).
2. Build any number of `ProofOfInclusionLayer` entries whose `combined_hash` fields are internally self-consistent (each `combined_hash` = `calculate_internal_hash(prev, side, other_hash)`).
3. Call `proof.valid()` → returns `true`.

The proof is accepted as valid even though it has no relationship to the actual committed DataLayer tree root. This matches the allowed High impact: **"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion … or lets untrusted input prove invalid state."**

The fuzz target and all existing tests generate proofs from the real blob and then call `proof.valid()`, so the tautology is never exposed by the test suite. [3](#0-2) 

The Python binding exposes `valid()` directly to callers: [4](#0-3) 

Any Python DataLayer client that calls `proof.valid()` without separately comparing `proof.root_hash()` to the on-chain committed root is fully vulnerable to forged proofs.

### Likelihood Explanation

- The `valid()` method is the primary, self-contained validation API. Its name implies completeness.
- No caller in the repository passes an expected root to `valid()` — the function signature does not even accept one.
- The fuzz target and all Rust/Python tests call only `proof.valid()`, establishing the pattern that this call alone is sufficient.
- An attacker needs only to craft a `ProofOfInclusion` struct with internally consistent hashes — no knowledge of the real tree is required.

### Recommendation

`valid()` must accept the expected root hash as a parameter and compare `existing_hash` against it instead of against `self.root_hash()`:

```rust
// Before (tautological):
existing_hash == self.root_hash()

// After (correct):
pub fn valid(&self, expected_root: &Hash) -> bool {
    let mut existing_hash = self.node_hash;
    for layer in &self.layers {
        let calculated_hash = crate::calculate_internal_hash(
            &existing_hash,
            layer.other_hash_side,
            &layer.other_hash,
        );
        if calculated_hash != layer.combined_hash {
            return false;
        }
        existing_hash = calculated_hash;
    }
    &existing_hash == expected_root   // compare against the authoritative root
}
```

All call sites (Rust, Python, WASM) must be updated to supply the committed tree root. The `root_hash()` helper can remain as a convenience accessor but must not be used as the validation target inside `valid()`.

### Proof of Concept

```rust
use chia_datalayer::{Hash, MerkleBlob, KeyId, ValueId, InsertLocation};
use chia_datalayer::merkle::proof_of_inclusion::{ProofOfInclusion, ProofOfInclusionLayer};
use chia_datalayer::{Side, calculate_internal_hash};

// Real tree with one entry
let mut blob = MerkleBlob::new(Vec::new()).unwrap();
let real_key   = KeyId(1);
let real_value = ValueId(1);
let real_hash: Hash = [0xAA; 32];
blob.insert(real_key, real_value, &real_hash, InsertLocation::Auto {}).unwrap();
blob.calculate_lazy_hashes().unwrap();

// Attacker wants to forge a proof for a key that does NOT exist
let fake_node_hash: Hash = [0xBB; 32];
let fake_other:     Hash = [0xCC; 32];
// Build one internally-consistent layer
let fake_combined = calculate_internal_hash(&fake_node_hash, Side::Left, &fake_other);
let forged_proof = ProofOfInclusion {
    node_hash: fake_node_hash,
    layers: vec![ProofOfInclusionLayer {
        other_hash_side: Side::Left,
        other_hash:      fake_other,
        combined_hash:   fake_combined,   // self-referential; not the real root
    }],
};

// valid() returns true even though fake_combined != real tree root
assert!(forged_proof.valid());   // ← passes; forged proof accepted
