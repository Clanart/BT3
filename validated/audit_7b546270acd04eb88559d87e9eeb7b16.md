### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Arbitrary Class Hash Injection Leading to Network Halt - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS program does not verify that the requested replacement class hash corresponds to a previously declared class. An unprivileged contract can set its own class hash to any arbitrary felt value. When the OS subsequently attempts to execute a call to that contract, it will fail to locate the compiled class fact, causing the OS to panic and making the affected block unprovable — a network halt.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function accepts a caller-controlled `class_hash` from the syscall request and writes it directly into `contract_state_changes` without any validation:

```cairo
// execute_replace_class, lines 896–910
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
```

The TODO comment explicitly acknowledges the missing check. [1](#0-0) 

The analog to the external report is direct: just as the iframe was granted extensive permissions with no CSP `frame-src` restriction to limit which origins could be loaded, `execute_replace_class` grants any contract the ability to inject any arbitrary class hash into the protocol state with no "allowlist" check against declared classes.

When the OS later attempts to execute a call to the affected contract, `execute_entry_point` performs:

```cairo
// execute_entry_point.cairo, lines 154–166
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [2](#0-1) 

For an undeclared class hash, `dict_read` on `contract_class_changes` returns the default value `0`. `find_element` then searches for a compiled class fact with hash `0`. If no such fact exists (the normal case), the hint inside `find_element` raises an exception, causing the OS to abort. The block becomes unprovable.

---

### Impact Explanation

**High — Network not being able to confirm new transactions (total network shutdown).**

Once a contract's class hash is set to an undeclared value, every subsequent transaction that calls that contract (in any future block) will cause the OS to fail at `find_element`. The sequencer cannot produce a valid STARK proof for any block containing such a call. If the affected contract is widely used (e.g., a token or account contract), this permanently prevents proof generation for any block that includes interactions with it, halting the network's ability to confirm new transactions.

---

### Likelihood Explanation

**Medium.** The attack requires only:
1. Deploying a contract (permissionless).
2. Calling `replace_class` from within that contract with an arbitrary undeclared felt value as the class hash.

No privileged role, leaked key, or external dependency compromise is needed. The entry path is a standard unprivileged `invoke` transaction. The TODO comment confirms the check is intentionally absent in the current codebase.

---

### Recommendation

In `execute_replace_class`, before writing the new class hash to `contract_state_changes`, verify that the requested `class_hash` has a corresponding entry in `contract_class_changes` (i.e., it has been declared). Concretely:

```cairo
// After reading class_hash from request:
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the CSP fix in the external report: explicitly restricting the set of allowed values to a trusted, pre-validated set (declared classes only), rather than accepting any caller-supplied value.

---

### Proof of Concept

1. Attacker deploys `MaliciousContract` with a valid class.
2. `MaliciousContract.__execute__` calls the `replace_class` syscall with `class_hash = 0xdeadbeef` (an undeclared felt).
3. The OS processes `execute_replace_class`:
   - No validation is performed (the TODO check is absent). [3](#0-2) 
   - `contract_state_changes[MaliciousContract.address].class_hash` is set to `0xdeadbeef`.
4. In the next block, any transaction calling `MaliciousContract` triggers `execute_entry_point`:
   - `dict_read(contract_class_changes, 0xdeadbeef)` → returns `0` (undeclared). [4](#0-3) 
   - `find_element(..., key=0)` → hint fails, OS aborts. [5](#0-4) 
5. The block is unprovable. The network cannot advance past this block.
