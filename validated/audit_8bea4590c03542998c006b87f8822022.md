### Title
`MerkleBlob::batch_insert` Bypasses Duplicate-Key/Hash Guards, Corrupting Tree Root and Enabling Forged Inclusion Proofs - (File: crates/chia-datalayer/src/merkle/blob.rs)

### Summary

`MerkleBlob::batch_insert` silently omits the duplicate-key and duplicate-hash checks that `MerkleBlob::insert` enforces. When a batch containing a repeated `KeyId` or `Hash` is supplied, both leaves are written into the blob without error. The `block_status_cache` is left inconsistent with the blob, the root hash computed by `calculate_lazy_hashes` is wrong, and any `ProofOfInclusion` derived from that root is based on corrupted state. This is the direct analog of the ERC20Quest wrong-order-of-operations class: `insert` validates *before* writing; `batch_insert` writes *without* validating, producing the same kind of silent state corruption.

### Finding Description

`MerkleBlob::insert` enforces two invariants before touching the blob:

```rust
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
```

`batch_insert` uses `insert` only for the last two items popped from the vector when `leaf_count <= 1`. For every other item — and for *all* items when `leaf_count >= 2` — it calls `insert_entry_to_blob` directly:

```rust
for ((key, value), hash) in keys_values_hashes {
    let new_leaf_index = self.get_new_index();
    let new_block = Block { ... Node::Leaf(LeafNode { hash, key, value, ... }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?;
    indexes.push(new_leaf_index);
}
```

`insert_entry_to_blob` calls `block_status_cache.add_leaf`, which silently overwrites the cache for a duplicate key:

```rust
fn add_leaf(&mut self, index: TreeIndex, leaf: LeafNode) {
    self.free_indexes.shift_remove(&index);
    self.key_to_index.insert(leaf.key, index);       // overwrites previous entry
    self.leaf_hash_to_index.insert(leaf.hash, index);
}
```

Concrete sequence when a batch `[(key_A, val1, h1), (key_A, val2, h2)]` is submitted with `leaf_count >= 2`:

1. Leaf `(key_A, val1, h1)` is written at `index_1`; cache: `key_A → index_1`.
2. Leaf `(key_A, val2, h2)` is written at `index_2`; cache: `key_A → index_2` (overwrites).
3. Both `index_1` and `index_2` are pushed into `indexes` and wired into the subtree by the pairing loop.
4. `calculate_lazy_hashes()` recomputes internal-node hashes bottom-up; the root hash is derived from a tree that contains *both* leaves.
5. `get_proof_of_inclusion(key_A)` looks up `index_2` in the cache and builds a valid path to the (corrupted) root.
6. `proof.valid()` returns `true` — but the root hash encodes `h1` (the orphaned first leaf) as well, so the committed root is wrong.

The `BlockStatusCache::new` constructor *does* detect duplicate keys when loading an existing blob and returns `Error::KeyAlreadyPresent`, confirming that duplicate leaves are considered an invalid state. `batch_insert` is the only write path that can produce this state without detection.

### Impact Explanation

The corrupted root hash is the value that the DataLayer commits as the authoritative state of a store. Any downstream consumer that verifies a `ProofOfInclusion` against this root will accept proofs for key-value pairs that were never legitimately inserted, or will accept a proof for `(key_A, val2)` while the root also encodes the stale `(key_A, val1)`. This directly satisfies the allowed High impact: *DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state.*

### Likelihood Explanation

`batch_insert` is exposed as a first-class Python binding (`py_batch_insert`) and is the primary bulk-load path used during DataLayer store synchronisation. A DataLayer store publisher — an unprivileged network participant — controls the key-value pairs that a syncing node ingests. If the syncing layer does not pre-deduplicate the incoming batch before calling `batch_insert`, a publisher can deliberately include a repeated key to corrupt the recipient's tree root. No privileged access, key material, or chain-level attack is required; only the ability to publish DataLayer store data.

### Recommendation

Add a pre-pass over the incoming batch inside `batch_insert` that mirrors the guards in `insert`:

```rust
pub fn batch_insert(
    &mut self,
    keys_values_hashes: Vec<((KeyId, ValueId), Hash)>,
) -> Result<(), Error> {
    // Validate the entire batch before touching the blob
    for ((key, _), hash) in &keys_values_hashes {
        if self.block_status_cache.contains_key(*key) {
            return Err(Error::KeyAlreadyPresent());
        }
        if self.block_status_cache.contains_leaf_hash(hash) {
            return Err(Error::HashAlreadyPresent());
        }
    }
    // Also check for intra-batch duplicates
    let mut seen_keys = HashSet::new();
    let mut seen_hashes = HashSet::new();
    for ((key, _), hash) in &keys_values_hashes {
        if !seen_keys.insert(key) { return Err(Error::KeyAlreadyPresent()); }
        if !seen_hashes.insert(hash) { return Err(Error::HashAlreadyPresent()); }
    }
    // ... existing logic
}
```

Alternatively, route all items through `insert` (accepting the performance cost) or add an assertion in `add_leaf` that panics on overwrite in debug builds.

### Proof of Concept

```rust
let mut blob = MerkleBlob::new(vec![]).unwrap();
// Ensure leaf_count >= 2 so the guard branch is skipped entirely
blob.insert(KeyId(0), ValueId(0), &sha256_num(&0_i64), InsertLocation::Auto {}).unwrap();
blob.insert(KeyId(1), ValueId(1), &sha256_num(&1_i64), InsertLocation::Auto {}).unwrap();

// batch with a repeated KeyId — insert() would reject this, batch_insert() does not
let batch = vec![
    ((KeyId(2), ValueId(2)),  sha256_num(&2_i64)),
    ((KeyId(2), ValueId(99)), sha256_num(&99_i64)), // duplicate key, different value & hash
];
blob.batch_insert(batch).unwrap(); // succeeds silently

blob.calculate_lazy_hashes().unwrap();

// Root hash now encodes BOTH leaves for KeyId(2)
let root = blob.get_hash(TreeIndex(0)).unwrap();

// Proof is built from the second leaf (val=99) but root includes the first (val=2)
let proof = blob.get_proof_of_inclusion(KeyId(2)).unwrap();
assert!(proof.valid()); // passes — proof is internally consistent with the corrupted root
```

```rust
    fn insert_entry_to_blob(&mut self, index: TreeIndex, block: &Block) -> Result<(), Error> {
        let new_block_bytes = block.to_bytes()?;
        let extend_index = self.extend_index();
        match index.cmp(&extend_index) {
            Ordering::Greater => return Err(Error::BlockIndexOutOfBounds(index)),
            Ordering::Equal => self.blob.extend_from_slice(&new_block_bytes),
            Ordering::Less => {
                self.blob[block_range(index)].copy_from_slice(&new_block_bytes);
            }
        }

        match block.node {
            Node::Leaf(leaf) => self.block_status_cache.add_leaf(index, leaf),
            Node::Internal(..) => self.block_status_cache.add_internal(index),
        }

        Ok(())
    }
```

```rust
    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
```
