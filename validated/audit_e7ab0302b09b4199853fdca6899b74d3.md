### Title
`SetCodeTx.Validate()` Omits Length Bounds on Outer `R`/`S` Bytes, Causing `uint256.MustFromBig` Panic on Chain - (File: `x/evm/types/set_code_tx.go`)

---

### Summary

`SetCodeTx.Validate()` enforces a 1-byte constraint on each authorization-list entry's `V` field but imposes **no upper-bound on the outer transaction's `R` and `S` byte slices**. Because `SetCodeTx.AsEthereumData()` converts those slices to `uint256.Int` via `uint256.MustFromBig`, any value ≥ 2²⁵⁶ (encoded as a byte slice longer than 32 bytes with a non-zero leading byte) causes an unrecovered panic that crashes the node.

---

### Finding Description

`SetCodeAuthorization.R`, `SetCodeAuthorization.S`, and the outer `SetCodeTx.R`/`SetCodeTx.S` are all declared as `[]byte` (variable-length) in the protobuf schema: [1](#0-0) [2](#0-1) 

`SetCodeTx.Validate()` only enforces a length constraint on the authorization-list `V` byte; it never checks the outer `R` or `S`: [3](#0-2) 

`rawSignatureValues` converts the raw bytes to `*big.Int` with no overflow guard: [4](#0-3) 

`SetCodeTx.AsEthereumData()` then passes those `*big.Int` values directly to `uint256.MustFromBig`: [5](#0-4) 

`uint256.MustFromBig` is documented to **panic** when the supplied `*big.Int` does not fit in 256 bits. A byte slice of length ≥ 33 with a non-zero leading byte produces exactly such a value.

The same gap exists in `LegacyTx.Validate()`, `AccessListTx.Validate()`, and `DynamicFeeTx.Validate()` — none check `R`/`S` length — but those types pass `*big.Int` to go-ethereum structs that accept arbitrary-precision integers, so no panic occurs there. The panic path is unique to `SetCodeTx` because go-ethereum's `ethtypes.SetCodeTx` uses `uint256.Int` (not `*big.Int`) for `R` and `S`. [6](#0-5) [7](#0-6) 

---

### Impact Explanation

`uint256.MustFromBig` panics with an unrecovered `panic("overflow")`. A panic in the EVM keeper's transaction-execution path is not caught by any deferred recovery in the block-processing loop, causing the node process to terminate. Every validator that processes the block containing this transaction crashes simultaneously, halting the chain. This matches the **Critical** impact class: *valid unprivileged transaction can halt the chain or corrupt committed state*.

---

### Likelihood Explanation

Any unprivileged user can broadcast a Cosmos SDK transaction wrapping a `MsgEthereumTx` whose inner `SetCodeTx` carries a 33-byte `R` or `S` field. The Cosmos mempool and ante handler call `Validate()`, which passes. The panic fires only when `AsEthereumData()` is invoked during block execution. Because EIP-7702 (`SetCodeTxType`) is a newly supported transaction type in this codebase, the attack surface is fresh and the missing length check is easy to overlook.

---

### Recommendation

1. In `SetCodeTx.Validate()`, add explicit length guards for the outer `R` and `S` fields alongside the existing `V` check:

```go
if len(tx.R) > 32 {
    return errorsmod.Wrap(ErrInvalidSignature, "R must be at most 32 bytes")
}
if len(tx.S) > 32 {
    return errorsmod.Wrap(ErrInvalidSignature, "S must be at most 32 bytes")
}
```

2. Apply the same guards to `LegacyTx`, `AccessListTx`, and `DynamicFeeTx` for defence-in-depth, even though those types do not currently panic.

3. Add analogous length checks for `SetCodeAuthorization.R` and `SetCodeAuthorization.S` in the authorization-list loop, since silent truncation by `uint256.SetBytes` in `ToEthAuthList` could allow a forged authorization to pass with a different scalar than the one submitted.

---

        ChainID: &validChainID,

        S:       []byte{0x01},

func (tx *SetCodeTx) AsEthereumData() ethtypes.TxData {
	v, r, s := tx.GetRawSignatureValues()
	to := tx.GetTo()
	if to == nil {
		// Validate() rejects empty To; zero address is a nil-safe fallback only.
		to = &common.Address{}
	}
	authList := tx.GetAuthList()
	if authList == nil {
		authList = &[]ethtypes.SetCodeAuthorization{}

// Validate performs a stateless validation of the tx fields.
func (tx SetCodeTx) Validate() error {
	if len(tx.To) == 0 {
		return errorsmod.Wrap(core.ErrSetCodeTxCreate, "to address cannot be empty")
	}

	if len(tx.AuthList) == 0 {
		return errorsmod.Wrap(core.ErrEmptyAuthList, "auth list cannot be empty")
	}

	// V is the signature y-parity byte; an empty slice would panic at auth.V[0]
	// in ToEthAuthList.
	for i := range tx.AuthList {
		if len(tx.AuthList[i].V) != 1 {
			return errorsmod.Wrapf(core.ErrAuthorizationInvalidSignature, "auth %d: V must be a single byte", i)
		}
	}

func rawSignatureValues(vBz, rBz, sBz []byte) (v, r, s *big.Int) {
	if len(vBz) > 0 {
		v = new(big.Int).SetBytes(vBz)
	}
	if len(rBz) > 0 {
		r = new(big.Int).SetBytes(rBz)
	}
	if len(sBz) > 0 {
		s = new(big.Int).SetBytes(sBz)
	}
	return v, r, s
}

// Validate performs a stateless validation of the tx fields.
func (tx LegacyTx) Validate() error {
	gasPrice := tx.GetGasPrice()
	if gasPrice == nil {
		return errorsmod.Wrap(ErrInvalidGasPrice, "gas price cannot be nil")
	}

	if gasPrice.Sign() == -1 {
		return errorsmod.Wrapf(ErrInvalidGasPrice, "gas price cannot be negative %s", gasPrice)
	}
	if !types.IsValidInt256(gasPrice) {
		return errorsmod.Wrap(ErrInvalidGasPrice, "out of bound")
	}
	if !types.IsValidInt256(tx.Fee()) {
		return errorsmod.Wrap(ErrInvalidGasFee, "out of bound")
	}

	amount := tx.GetValue()
	// Amount can be 0
	if amount != nil && amount.Sign() == -1 {
		return errorsmod.Wrapf(ErrInvalidAmount, "amount cannot be negative %s", amount)
	}
	if !types.IsValidInt256(amount) {
		return errorsmod.Wrap(ErrInvalidAmount, "out of bound")
	}

	if tx.To != "" {
		if err := types.ValidateAddress(tx.To); err != nil {
			return errorsmod.Wrap(err, "invalid to address")
		}
	}

	return nil
}

// Validate performs a stateless validation of the tx fields.
func (tx DynamicFeeTx) Validate() error {
	if tx.GasTipCap == nil {
		return errorsmod.Wrap(ErrInvalidGasCap, "gas tip cap cannot nil")
	}

	if tx.GasFeeCap == nil {
		return errorsmod.Wrap(ErrInvalidGasCap, "gas fee cap cannot nil")
	}

	if tx.GasTipCap.IsNegative() {
		return errorsmod.Wrapf(ErrInvalidGasCap, "gas tip cap cannot be negative %s", tx.GasTipCap)
	}

	if tx.GasFeeCap.IsNegative() {
		return errorsmod.Wrapf(ErrInvalidGasCap, "gas fee cap cannot be negative %s", tx.GasFeeCap)
	}

	if !types.IsValidInt256(tx.GetGasTipCap()) {
		return errorsmod.Wrap(ErrInvalidGasCap, "out of bound")
	}

	if !types.IsValidInt256(tx.GetGasFeeCap()) {
		return errorsmod.Wrap(ErrInvalidGasCap, "out of bound")
	}

	if tx.GasFeeCap.LT(*tx.GasTipCap) {
		return errorsmod.Wrapf(
			ErrInvalidGasCap, "max priority fee per gas higher than max fee per gas (%s > %s)",
			tx.GasTipCap, tx.GasFeeCap,
		)
	}

	if !types.IsValidInt256(tx.Fee()) {
		return errorsmod.Wrap(ErrInvalidGasFee, "out of bound")
	}

	amount := tx.GetValue()
	// Amount can be 0
	if amount != nil && amount.Sign() == -1 {
		return errorsmod.Wrapf(ErrInvalidAmount, "amount cannot be negative %s", amount)
	}
	if !types.IsValidInt256(amount) {
		return errorsmod.Wrap(ErrInvalidAmount, "out of bound")
	}

	if tx.To != "" {
		if err := types.ValidateAddress(tx.To); err != nil {
			return errorsmod.Wrap(err, "invalid to address")
		}
	}

	if tx.GetChainID() == nil {
		return errorsmod.Wrap(
			errortypes.ErrInvalidChainID,
			"chain ID must be present on AccessList txs",
		)
	}

	return nil
}
