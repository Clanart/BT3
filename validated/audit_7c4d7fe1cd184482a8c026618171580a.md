### Title
Stale `cancellation_requested_at` in `mark_rejected` allows `update_time_based_state` to silently override `Rejected` → `CancelledOnL2`, causing `validate_tx` to return wrong `ValidationStatus` — (`crates/apollo_l1_events/src/transaction_record.rs`)

### Summary
`mark_rejected` sets `state=Rejected` but does not clear `cancellation_requested_at`. `update_time_based_state` only guards on `self.committed` (false for any rejected tx) and `cancellation_requested_at` (still `Some`), so after the timelock it unconditionally overwrites `Rejected` with `CancelledOnL2`. `validate_tx` then returns `InvalidValidationStatus::CancelledOnL2` for a tx that should return `Validated`, corrupting the admission decision.

### Finding Description
`TransactionRecord::mark_rejected` (line 63–72) transitions `state` to `Rejected` and sets `rejected=true`, but leaves `cancellation_requested_at` intact:

```rust
pub fn mark_rejected(&mut self) {
    assert!(!self.committed, ...);
    self.state = TransactionState::Rejected;
    self.rejected = true;
    // cancellation_requested_at is NOT cleared
}
```

`update_time_based_state` (line 194–208) has no guard on the current state or the `rejected` flag:

```rust
pub fn update_time_based_state(&mut self, unix_now: u64, policy: TransactionRecordPolicy) {
    if let Some(requested_at) = self.cancellation_requested_at {
        if self.committed { return; }   // only guard: committed flag
        // no check: self.rejected, self.state == Rejected
        if is_cancellation_timelock_passed {
            self.state = TransactionState::CancelledOnL2;  // overwrites Rejected
        }
    }
}
```

`validate_tx` calls `update_time_based_state` before checking `is_validatable()`, so the corrupted state is observed immediately: [7](#0-6) 

### Impact Explanation
A validator node calling `validate_tx` on a tx that legitimately reached `Rejected` state (but had a prior cancellation request) will receive `CancelledOnL2` instead of `Validated`. This causes the validator to reject a block proposal that the proposer considers valid, breaking consensus or causing the validator to diverge from the proposer's view of which L1 handler transactions are includable. The corrupted `TransactionState` is also reflected in `snapshot()`, misclassifying the tx from `rejected` to `cancelled_on_l2` in all downstream consumers. [8](#0-7) 

### Likelihood Explanation
The race condition (cancellation event arriving between proposal and commit) is realistic: `add_events` is called asynchronously from the scraper while block processing proceeds. No special privileges are required — any L1 actor who can submit an L1 handler tx and also trigger a cancellation request on the L1 contract can set up the precondition. The timelock expiry is the only time constraint, and it is bounded.

### Recommendation
In `mark_rejected`, clear `cancellation_requested_at`:
```rust
pub fn mark_rejected(&mut self) {
    assert!(!self.committed, ...);
    self.state = TransactionState::Rejected;
    self.rejected = true;
    self.cancellation_requested_at = None; // prevent stale metadata from driving future transitions
}
```
Alternatively, add an explicit state guard in `update_time_based_state`:
```rust
if !matches!(self.state, TransactionState::CancellationStartedOnL2) {
    return;
}
```
The first fix is preferable because it enforces the invariant at the source.

### Proof of Concept
```rust
#[test]
fn rejected_tx_must_not_transition_to_cancelled_on_l2() {
    use std::time::Duration;
    use crate::transaction_record::{
        TransactionRecord, TransactionPayload, TransactionState, TransactionRecordPolicy,
    };
    use starknet_api::block::BlockTimestamp;
    use starknet_api::transaction::TransactionHash;

    let hash = TransactionHash::default();
    let mut record = TransactionRecord::new(TransactionPayload::from(hash));

    // Step 1: cancellation requested at t=100
    record.mark_cancellation_request(BlockTimestamp(100));
    assert_eq!(record.state, TransactionState::CancellationStartedOnL2);

    // Step 2: tx rejected (e.g. blockifier execution failure)
    record.mark_rejected();
    assert_eq!(record.state, TransactionState::Rejected);

    // Step 3: timelock expires, update_time_based_state called
    let policy = TransactionRecordPolicy {
        cancellation_timelock: Duration::from_secs(3600),
    };
    record.update_time_based_state(100 + 3600 + 1, policy);

    // BUG: state is CancelledOnL2, not Rejected
    assert_eq!(
        record.state,
        TransactionState::Rejected,  // FAILS: actual is CancelledOnL2
        "Rejected tx must not silently transition to CancelledOnL2 via stale cancellation metadata"
    );
}
```

    pub fn validate_tx(&mut self, tx_hash: TransactionHash, unix_now: u64) -> ValidationStatus {
        let current_staging_epoch_cloned = self.current_staging_epoch;

        let policy = TransactionRecordPolicy {
            cancellation_timelock: self.config.l1_handler_cancellation_timelock_seconds,
        };

        let validation_status = self.with_record(tx_hash, |record| {
            // If the current time affects the state, update state now.
            record.update_time_based_state(unix_now, policy);
            if !record.is_validatable() {
                match record.state {
                    TransactionState::Committed => {
                        InvalidValidationStatus::AlreadyIncludedOnL2.into()
                    }
                    TransactionState::CancelledOnL2 => {
                        InvalidValidationStatus::CancelledOnL2.into()
                    }
                    TransactionState::Consumed => InvalidValidationStatus::ConsumedOnL1.into(),
                    _ => unreachable!(),
                }
            } else if record.try_mark_staged(current_staging_epoch_cloned) {
                ValidationStatus::Validated
            } else {
                InvalidValidationStatus::AlreadyIncludedInProposedBlock.into()
            }
        });

        validation_status.unwrap_or(InvalidValidationStatus::NotFound.into())
    }

        if self.state.is_uninitialized() {
            return Err(L1EventsProviderError::Uninitialized);
        }

        // TODO(guyn): can we remove this "every sec" since the polling interval is rather long?
        info_every_n_ms!(1000, "Adding {} l1 events", events.len());
        trace!("Adding events: {events:?}");

        for event in events {
            match event {
                Event::L1HandlerTransaction {
                    l1_handler_tx,
                    block_timestamp,
                    scrape_timestamp,
                } => {
                    self.tx_manager.add_tx(l1_handler_tx, block_timestamp, scrape_timestamp);
                }
                Event::TransactionCancellationStarted {
                    tx_hash,
                    cancellation_request_timestamp,
                } => {
                    if !self.tx_manager.exists(tx_hash) {
                        warn!(
                            "Dropping cancellation request for old L1 handler transaction \
                             {tx_hash}: not in the provider and will never be scraped at this \
                             point."
                        );
                        continue;
                    }

                    self.tx_manager
                        .request_cancellation(tx_hash, cancellation_request_timestamp)

                            // Re-requesting a cancellation is meaningful for the L1 timelock, but
                            // for the l2 timelock we only consider the first cancellation
                            // relevant.
                            info!(

                        });
                }

                    info!(
                        "Cancellation finalized for tx_hash: {tx_hash}. Deleting the tx from the \

                    );
                    self.tx_manager.finalize_cancellation(tx_hash);
                }
                Event::TransactionConsumed { tx_hash, timestamp: consumed_at } => {
                    if let Err(previously_consumed_at) =

                        panic!(

                             consumed at {previously_consumed_at}."
                        );
                    }
                }
            }
        }

    }
