### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Freezing of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

`execute_replace_class` in the StarkNet OS does not verify that the caller-supplied `class_hash` corresponds to a previously declared class before committing the replacement to state. An unprivileged contract can replace its own class with an arbitrary, undeclared hash. Any subsequent call to that contract will cause the OS to fail when resolving the compiled class, permanently freezing all funds held in the contract.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The developer-acknowledged TODO at line 898 confirms the missing check. The function unconditionally writes `class_hash` — which is fully attacker-controlled — into `contract_state_changes` with no assertion that the hash exists in `contract_class_changes` (the declared-class registry). [1](#0-0) 

When any subsequent call targets the affected contract, `execute_entry_point` reads the class hash from state and attempts to resolve it:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // ← undeclared hash
);
// compiled_class_hash == 0 (default for uninitialized dict key)

let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,           // ← key=0, not present → OS fails
);
```

`dict_read` returns `0` for an undeclared key. `find_element` then panics because no compiled class with hash `0` exists in the bundle. The OS cannot produce a valid proof for any block containing a call to the affected contract. [2](#0-1) 

**Analog to the external report:** In the DebtDAO bug, `close()` accepted a non-existent credit ID, operated on a zero-initialized struct, and decremented `count` to trigger the `REPAID` state without actual repayment. Here, `execute_replace_class` accepts an undeclared class hash, writes it to state without validation, and causes the OS to operate on a zero-initialized compiled-class lookup — triggering an unrecoverable failure for all future calls to the contract.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with an undeclared value:
- Every future call to that contract causes `find_element` to fail inside `execute_entry_point`.
- The OS cannot prove any block that includes such a call.
- The sequencer will reject all transactions targeting the contract.
- All ERC-20 balances, NFTs, or other assets held in the contract's storage become permanently inaccessible with no recovery path. [3](#0-2) 

---

### Likelihood Explanation

The attack requires only:
1. Deploying a contract (permissionless on StarkNet).
2. Sending a single invoke transaction that calls `replace_class` with an arbitrary felt value.

No privileged role, leaked key, or external dependency is needed. The `replace_class` syscall is a standard, publicly accessible syscall available to any Sierra contract. The missing check is explicitly flagged by a TODO comment, confirming it is a known gap in the current implementation. [4](#0-3) 

---

### Recommendation

Before committing the class replacement to `contract_state_changes`, assert that the supplied `class_hash` has a non-zero entry in `contract_class_changes` (i.e., it has been declared):

```cairo
// After: let class_hash = request.class_hash;
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the existing pattern in `execute_entry_point` where `dict_read` on `contract_class_changes` is used to resolve the compiled class, and ensures that `replace_class` can only target classes that are provably resolvable by the OS. [5](#0-4) 

---

### Proof of Concept

1. **Deploy** a malicious contract (class `A`) that implements a function `attack()` which calls `replace_class(0xdeadbeef)` — an arbitrary felt that has never been declared.

2. **Invoke** `attack()` on the deployed contract. Inside `execute_replace_class`:
   - `class_hash = 0xdeadbeef` (attacker-controlled, no validation).
   - `dict_update` writes `StateEntry(class_hash=0xdeadbeef, ...)` for the contract address.
   - The syscall returns success.

3. **Send any subsequent transaction** targeting the same contract address (e.g., a transfer of funds out).

4. Inside `execute_entry_point` for the subsequent call:
   - `execution_context.class_hash = 0xdeadbeef`.
   - `dict_read(contract_class_changes, 0xdeadbeef)` → returns `0` (never declared).
   - `find_element(compiled_class_facts, key=0)` → **OS panics / fails** because no compiled class with hash `0` exists.

5. **Result:** The contract is permanently unexecutable. All funds stored in its storage are frozen with no recovery mechanism. [6](#0-5) [7](#0-6)

    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
