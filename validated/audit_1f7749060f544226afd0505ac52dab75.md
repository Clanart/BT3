### Title
`execute_meta_tx_v0` Syscall Bypasses Target Account `__validate__`, Enabling Unauthorized Execution of Arbitrary Calldata — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_meta_tx_v0` syscall in the StarkNet OS allows any contract to invoke another account's `__execute__` entry point with attacker-controlled calldata, without running the target account's `__validate__` entry point. This is a direct analog to the UTB `receiveFromBridge` vulnerability: a publicly accessible execution path that bypasses the mandatory validation gate (`__validate__`), enabling unauthorized execution of arbitrary calls on behalf of any account and resulting in direct loss of funds.

---

### Finding Description

In the normal invoke transaction flow, `execute_invoke_function_transaction` enforces the following sequence:

1. Compute and verify transaction hash
2. Check and increment nonce (`check_and_increment_nonce`)
3. Run `__validate__` (`run_validate`) — the account's signature verification gate
4. Run `__execute__`
5. Charge fee [1](#0-0) 

`run_validate` is the critical security gate: it calls the account contract's `__validate__` entry point, which is responsible for verifying the transaction signature. [2](#0-1) 

The `execute_meta_tx_v0` function in `syscall_impls.cairo` is a syscall available to **any executing contract**. It accepts an arbitrary `contract_address`, `selector`, `calldata`, and `signature` from the calling contract, constructs a version-0 `TxInfo`, and then calls `contract_call_helper` directly: [3](#0-2) 

`contract_call_helper` calls `select_execute_entry_point_func` directly — **it never calls `run_validate`**. There is no call to `check_and_increment_nonce` either. The target account's `__validate__` is entirely skipped. [4](#0-3) 

The only restriction enforced is that the `selector` must equal `EXECUTE_ENTRY_POINT_SELECTOR` — i.e., the syscall can only target `__execute__`, which is the primary execution entry point of every account contract. [5](#0-4) 

Critically, the `caller_address` in the constructed `ExecutionInfo` is set to `ORIGIN_ADDRESS` (0): [6](#0-5) 

This is identical to the `caller_address` used in normal top-level invoke transactions: [7](#0-6) 

Standard StarkNet account contracts check `assert_caller_is_null()` (caller == 0) in `__execute__` — a check that **passes** for both normal transactions and meta-tx v0. The account cannot distinguish between a legitimately validated transaction and a meta-tx v0 call based on `caller_address` alone. Since standard accounts rely entirely on `__validate__` for signature verification and do not re-verify signatures inside `__execute__`, the bypass is complete.

---

### Impact Explanation

An attacker can drain any account's funds:

1. Attacker deploys a malicious contract.
2. Attacker submits a valid invoke transaction to their malicious contract (passing their own `__validate__`).
3. The malicious contract issues the `execute_meta_tx_v0` syscall targeting the victim account's `__execute__` with calldata encoding a token transfer to the attacker.
4. The OS executes `victim.__execute__(transfer_calldata)` without calling `victim.__validate__`.
5. The victim's `__execute__` processes the transfer; funds are lost.

**Impact: Direct loss of funds (Critical).**

The victim's nonce is not incremented, so the attacker can issue multiple `execute_meta_tx_v0` calls within a single outer transaction to fully drain the account.

---

### Likelihood Explanation

- No privileged role, leaked key, or trusted operator is required.
- Any unprivileged user can deploy a contract and submit transactions.
- The syscall has no access control — it is available to all contracts.
- The attack is fully on-chain and deterministic.

---

### Recommendation

`execute_meta_tx_v0` must either:

1. **Call `run_validate` on the target account before `__execute__`**, using the provided signature and the computed meta-tx hash, so the target account's signature verification gate is not bypassed; or
2. **Require the target account to opt-in** to meta-transaction support via a registry or storage flag, so only accounts explicitly designed for meta-tx relaying can be targeted; or
3. **Restrict which contracts may issue this syscall** via an allowlist enforced in the OS.

The current design places the burden of signature verification on `__execute__`, which contradicts the standard StarkNet account security model where `__validate__` is the sole verification gate.

---

### Proof of Concept

```
1. Attacker deploys MaliciousRelay contract.

2. Attacker submits invoke tx:
   - sender: attacker_account
   - target: MaliciousRelay.__execute__
   - attacker_account.__validate__ passes (attacker signs their own tx)

3. MaliciousRelay.__execute__ issues execute_meta_tx_v0 syscall:
   - contract_address = victim_account_address
   - selector        = EXECUTE_ENTRY_POINT_SELECTOR  (__execute__)
   - calldata        = [transfer(attacker_address, victim_full_balance)]
   - signature       = []  (empty; __validate__ will not run)

4. OS executes execute_meta_tx_v0:
   - Constructs TxInfo with version=0, nonce=0, caller_address=0
   - Calls contract_call_helper → select_execute_entry_point_func
   - victim.__validate__ is NEVER called
   - victim.__execute__([transfer(attacker, balance)]) executes

5. Victim's token balance is transferred to attacker.
   Victim's nonce is unchanged; attack can be repeated in same tx.
```

func execute_meta_tx_v0{
    range_check_ptr,
    syscall_ptr: felt*,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    revert_log: RevertLogEntry*,
    outputs: OsCarriedOutputs*,
}(block_context: BlockContext*, caller_execution_context: ExecutionContext*) {
    alloc_locals;

    let request = cast(syscall_ptr + RequestHeader.SIZE, MetaTxV0Request*);
    local calldata_start: felt* = request.calldata_start;
    local calldata_size = request.calldata_end - calldata_start;

    let specific_base_gas_cost = (
        META_TX_V0_GAS_COST + META_TX_V0_CALLDATA_FACTOR_GAS_COST * calldata_size
    );
    let (success, remaining_gas) = reduce_syscall_base_gas(
        specific_base_gas_cost=specific_base_gas_cost, request_struct_size=MetaTxV0Request.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    local contract_address = request.contract_address;
    local selector = request.selector;
    local caller_execution_info: ExecutionInfo* = caller_execution_context.execution_info;
    local old_tx_info: TxInfo* = caller_execution_info.tx_info;

    if (selector != EXECUTE_ENTRY_POINT_SELECTOR) {
        write_failure_response(remaining_gas=remaining_gas, failure_felt=ERROR_INVALID_ARGUMENT);
        return ();
    }

    // Sanity check: Verify that `signature` is a valid Sierra array.
    assert_nn_le(request.signature_end - request.signature_start, SIERRA_ARRAY_LEN_BOUND - 1);

    let (state_entry: StateEntry*) = dict_read{dict_ptr=contract_state_changes}(
        key=contract_address
    );

    // Compute the meta-transaction hash.
    let pedersen_ptr = builtin_ptrs.selectable.pedersen;
    with pedersen_ptr {
        let meta_tx_hash = compute_meta_tx_v0_hash(
            contract_address=contract_address,
            entry_point_selector=selector,
            calldata=calldata_start,
            calldata_size=calldata_size,
            chain_id=old_tx_info.chain_id,
        );
    }
    update_pedersen_in_builtin_ptrs(pedersen_ptr=pedersen_ptr);

    // Prepare execution context.
    tempvar new_tx_info = new TxInfo(
        version=0,
        account_contract_address=contract_address,
        max_fee=0,
        signature_start=request.signature_start,
        signature_end=request.signature_end,
        transaction_hash=meta_tx_hash,
        chain_id=old_tx_info.chain_id,
        nonce=0,
        resource_bounds_start=cast(0, ResourceBounds*),
        resource_bounds_end=cast(0, ResourceBounds*),
        tip=0,
        paymaster_data_start=cast(0, felt*),
        paymaster_data_end=cast(0, felt*),
        nonce_data_availability_mode=0,
        fee_data_availability_mode=0,
        account_deployment_data_start=cast(0, felt*),
        account_deployment_data_end=cast(0, felt*),
        proof_facts_start=cast(0, felt*),
        proof_facts_end=cast(0, felt*),
    );

    let (deprecated_tx_info_ptr: DeprecatedTxInfo*) = alloc();
    tempvar execution_context: ExecutionContext* = new ExecutionContext(
        entry_point_type=ENTRY_POINT_TYPE_EXTERNAL,
        class_hash=state_entry.class_hash,
        calldata_size=calldata_size,
        calldata=calldata_start,
        execution_info=new ExecutionInfo(
            block_info=caller_execution_info.block_info,
            tx_info=new_tx_info,
            caller_address=ORIGIN_ADDRESS,
            contract_address=contract_address,
            selector=selector,
        ),
        deprecated_tx_info=deprecated_tx_info_ptr,
    );
    fill_deprecated_tx_info(tx_info=new_tx_info, dst=execution_context.deprecated_tx_info);

    // Since we process the revert log backwards, entries before this point belong to the calling
    // contract.
    assert [revert_log] = RevertLogEntry(
        selector=CHANGE_CONTRACT_ENTRY, value=caller_execution_info.contract_address
    );
    let revert_log = &revert_log[1];

    contract_call_helper(
        remaining_gas=remaining_gas,
        block_context=block_context,
        execution_context=execution_context,
    );

    // Entries before this point belong to the callee.
    assert [revert_log] = RevertLogEntry(selector=CHANGE_CONTRACT_ENTRY, value=contract_address);
    let revert_log = &revert_log[1];

    return ();

// Executes the entry point and writes the corresponding response to the syscall_ptr.
// Assumes that syscall_ptr points at the response header.
func contract_call_helper{
    range_check_ptr,
    syscall_ptr: felt*,
    builtin_ptrs: BuiltinPointers*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,
    revert_log: RevertLogEntry*,
    outputs: OsCarriedOutputs*,
}(remaining_gas: felt, block_context: BlockContext*, execution_context: ExecutionContext*) {
    with remaining_gas {
        let (is_reverted, retdata_size, retdata, _is_deprecated) = select_execute_entry_point_func(
            block_context=block_context, execution_context=execution_context
        );
    }

    if (is_reverted != FALSE) {
        // Append `ERROR_ENTRY_POINT_FAILED` to the retdata.
        assert retdata[retdata_size] = ERROR_ENTRY_POINT_FAILED;
        tempvar retdata_size = retdata_size + 1;
    } else {
        ap += 2;  // Align the stack to avoid revoked references.
        tempvar retdata_size = retdata_size;
    }

    let response_header = cast(syscall_ptr, ResponseHeader*);
    let syscall_ptr = syscall_ptr + ResponseHeader.SIZE;

    // Write the response header.
    with_attr error_message("Predicted gas costs are inconsistent with the actual execution.") {
        assert [response_header] = ResponseHeader(gas=remaining_gas, failure_flag=is_reverted);
    }

    let response = cast(syscall_ptr, CallContractResponse*);
    // Advance syscall pointer to the next syscall.
    let syscall_ptr = syscall_ptr + CallContractResponse.SIZE;

    %{ CheckNewCallContractResponse %}

    // Write the response.
    relocate_segment(src_ptr=response.retdata_start, dest_ptr=retdata);
    assert [response] = CallContractResponse(
        retdata_start=retdata, retdata_end=retdata + retdata_size
    );

    return ();
}
