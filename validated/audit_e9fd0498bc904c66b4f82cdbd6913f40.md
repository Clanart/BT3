### Title
Missing Class Hash Validation in `execute_replace_class` Allows Permanent Freezal of Contract Funds — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS unconditionally accepts any arbitrary felt value as a new class hash without verifying that the hash corresponds to a declared contract class. This is the direct analog to the external report's vulnerability class: a critical state variable (the contract's `class_hash` in `StateEntry`) is mutated without proper validation, allowing a caller to corrupt the state in a way that permanently freezes all funds held by the contract.

---

### Finding Description

In `syscall_impls.cairo`, the function `execute_replace_class` processes the `replace_class` syscall:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
```

The `class_hash` field from the request is written directly into the contract's `StateEntry` in `contract_state_changes` with **no check** that `class_hash` corresponds to any declared class in `contract_class_changes`. The TODO comment at line 898 explicitly acknowledges this missing validation. [1](#0-0) 

After the block is finalized and the state root is committed, the contract's on-chain class hash is permanently set to the arbitrary value. Any future invocation of the contract will attempt to look up this class hash in the class commitment tree, fail to find it, and the call will fail unconditionally. There is no recovery path.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any funds (ERC-20 balances, ETH, NFTs) stored in the contract's storage become permanently inaccessible. The contract cannot be called, upgraded again, or self-destructed because all entry points require a valid class to dispatch to. The state root committed to L1 encodes the invalid class hash, making the freeze irreversible at the protocol level.

---

### Likelihood Explanation

The `replace_class` syscall is a standard, permissionless syscall available to every Cairo 1 contract. No privileged role is required. A malicious contract deployer can:

1. Deploy a contract that accepts user deposits.
2. Accumulate funds from victims.
3. Issue a `replace_class` syscall with `class_hash = 0` (or any undeclared felt).
4. The OS commits the block with the corrupted `StateEntry`.
5. All deposited funds are permanently frozen.

This is a straightforward, one-transaction attack requiring no special access.

---

### Recommendation

Before writing the new `class_hash` into `contract_state_changes`, the OS must verify that the hash exists in `contract_class_changes` (i.e., it was declared in the current block) or in the existing class commitment tree. Concretely, add a `dict_read` on `contract_class_changes` for `class_hash` and assert the result is non-zero, or perform a Patricia tree membership proof for the class hash against the current class root. The TODO comment at line 898 already identifies this as a required fix. [2](#0-1) 

---

### Proof of Concept

1. Attacker deploys contract `VaultMalicious` (Cairo 1) with:
   - A public `deposit()` function that accepts funds.
   - A public `brick()` function that calls `replace_class(class_hash=0x1337dead)`.
2. Users call `deposit()`, transferring tokens into `VaultMalicious`.
3. Attacker calls `brick()`. The OS executes `execute_replace_class`:
   - `request.class_hash = 0x1337dead` (undeclared).
   - Line 898 TODO is skipped — no validation occurs.
   - `dict_update` writes `StateEntry(class_hash=0x1337dead, ...)` into `contract_state_changes`.
4. Block is finalized. `state_update` in `state.cairo` squashes and commits the state diff, including the corrupted entry, into the global state root posted to L1.
5. All subsequent calls to `VaultMalicious` fail: the OS cannot find class `0x1337dead` in the class tree. User funds are permanently frozen with no recovery mechanism. [3](#0-2) [4](#0-3)

func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    alloc_locals;
    let request = cast(syscall_ptr + RequestHeader.SIZE, ReplaceClassRequest*);

    // Reduce gas.
    let success = reduce_syscall_gas_and_write_response_header(
        total_gas_cost=REPLACE_CLASS_GAS_COST, request_struct_size=ReplaceClassRequest.SIZE
    );
    if (success == FALSE) {
        // Not enough gas to execute the syscall.
        return ();
    }

    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );

    assert [revert_log] = RevertLogEntry(selector=CHANGE_CLASS_ENTRY, value=state_entry.class_hash);
    let revert_log = &revert_log[1];

    return ();
}
