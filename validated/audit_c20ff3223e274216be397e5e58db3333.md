### Title
Peer-Supplied `StateRootNode::memory_usage` Causes Arithmetic Overflow in `get_num_state_parts` During State Sync - (File: `core/primitives/src/state_sync.rs`)

### Summary

`get_num_state_parts(memory_usage)` performs an unchecked addition `memory_usage + STATE_PART_MEMORY_LIMIT.as_u64() - 1` that overflows when `memory_usage` is near `u64::MAX`. The `memory_usage` value originates from the `StateRootNode.memory_usage` field inside a peer-supplied `ShardStateSyncResponseHeader`. The header validation path (`set_state_header`) never calls `validate_state_root_node`, so a malicious peer can inject an arbitrary `memory_usage` while supplying otherwise-valid chunk proofs and receipt proofs. The overflow panics the node.

### Finding Description

`get_num_state_parts` computes the number of state parts from the trie root's memory usage:

```rust
// core/primitives/src/state_sync.rs:353-355
pub fn get_num_state_parts(memory_usage: u64) -> u64 {
    (memory_usage + STATE_PART_MEMORY_LIMIT.as_u64() - 1) / STATE_PART_MEMORY_LIMIT.as_u64()
}
``` [1](#0-0) 

`STATE_PART_MEMORY_LIMIT` is 30 MiB = 31,457,280. Any `memory_usage > u64::MAX - 31,457,279` causes the addition to overflow and panic.

`memory_usage` is read from `ShardStateSyncResponseHeader::num_state_parts()`:

```rust
// core/primitives/src/state_sync.rs:240-242
pub fn num_state_parts(&self) -> u64 {
    get_num_state_parts(self.state_root_node().memory_usage)
}
``` [2](#0-1) 

The `state_root_node` field is part of `ShardStateSyncResponseHeaderV2`, which is received from a peer and stored by `set_state_header`: [3](#0-2) 

`set_state_header` validates chunk Merkle proofs, chunk inclusion in blocks, and receipt proofs — but **never calls `validate_state_root_node`**: [4](#0-3) 

`validate_state_root_node` is the function that would cross-check `StateRootNode.memory_usage` against the value encoded in `StateRootNode.data` (which is hash-committed to the state root): [5](#0-4) 

Because `set_state_header` skips this check, a peer can supply:
- `state_root_node.data` = valid serialized trie node whose `hash(data)` equals the committed state root
- `state_root_node.memory_usage` = `u64::MAX` (arbitrary, not validated against `data`)

The header passes all existing checks and is persisted to `DBCol::StateHeaders`. Subsequently, any call to `num_state_parts()` on the stored header triggers the overflow.

### Impact Explanation

A syncing node panics (crashes) the first time it calls `num_state_parts()` on the stored header — for example inside `get_state_response_part` or the state sync shard download loop. The crash is deterministic and repeatable: every restart re-downloads the same header from the same malicious peer, re-stores it, and crashes again, creating a permanent denial-of-service for the syncing node. [6](#0-5) 

### Likelihood Explanation

Any peer reachable by a syncing node can serve state sync headers. The attacker only needs to know the committed state root (public, on-chain) to construct a valid `data` blob while setting `memory_usage = u64::MAX`. No validator role or privileged position is required. State sync is triggered automatically when a node falls behind, making this reachable without any special coordination.

### Recommendation

1. **Validate `state_root_node` in `set_state_header`**: call `validate_state_root_node` (or inline the equivalent check) before persisting the header. Reject the header if `state_root_node.memory_usage` does not match the value decoded from `state_root_node.data`.

2. **Harden `get_num_state_parts` defensively**: use `checked_add` to avoid a panic even if a bad value somehow reaches the function:

```rust
pub fn get_num_state_parts(memory_usage: u64) -> u64 {
    let limit = STATE_PART_MEMORY_LIMIT.as_u64();
    memory_usage.checked_add(limit - 1)
        .map(|v| v / limit)
        .unwrap_or(u64::MAX / limit)
}
```

### Proof of Concept

```rust
// Craft a ShardStateSyncResponseHeaderV2 with valid chunk/receipts
// but state_root_node.memory_usage = u64::MAX.
// Pass it to set_state_header — it is accepted (no memory_usage check).
// Then call header.num_state_parts():
//   get_num_state_parts(u64::MAX)
//   = (u64::MAX + 31_457_280 - 1) / 31_457_280
//   ^^^ panics: attempt to add with overflow
```

The overflow threshold is `memory_usage > 18_446_744_042_252_271_615` (i.e., `u64::MAX - 31_457_279`). Any value above this, freely injectable via the `StateRootNode.memory_usage` wire field, triggers the panic.

        &self,
        shard_id: ShardId,
        sync_hash: CryptoHash,
        shard_state_header: ShardStateSyncResponseHeader,
    ) -> Result<(), Error> {
        let sync_block_header = self.chain_store.get_block_header(&sync_hash)?;

        let chunk = shard_state_header.cloned_chunk();
        let prev_chunk_header = shard_state_header.cloned_prev_chunk_header();

        // 1-2. Checking chunk validity
        if !validate_chunk_proofs(&chunk, self.epoch_manager.as_ref())? {
            byzantine_assert!(false);
            return Err(Error::Other(
                "set_shard_state failed: chunk header proofs are invalid".into(),
            ));
        }

        // Consider chunk itself is valid.

        // 3. Checking that chunks `chunk` and `prev_chunk` are included in appropriate blocks
        // 3a. Checking that chunk `chunk` is included into block at last height before sync_hash
        // 3aa. Also checking chunk.height_included
        let sync_prev_block_header =
            self.chain_store.get_block_header(sync_block_header.prev_hash())?;
        if !verify_path(
            *sync_prev_block_header.chunk_headers_root(),
            shard_state_header.chunk_proof(),
            &ChunkHashHeight(chunk.chunk_hash().clone(), chunk.height_included()),
        ) {
            byzantine_assert!(false);
            return Err(Error::Other(
                "set_shard_state failed: chunk isn't included into block".into(),
            ));
        }

        let block_header = get_block_header_on_chain_by_height(
            &self.chain_store,
            &sync_hash,
            chunk.height_included(),
        )?;
        // 3b. Checking that chunk `prev_chunk` is included into block at height before chunk.height_included
        // 3ba. Also checking prev_chunk.height_included - it's important for getting correct incoming receipts
        match (&prev_chunk_header, shard_state_header.prev_chunk_proof()) {
            (Some(prev_chunk_header), Some(prev_chunk_proof)) => {
                let prev_block_header =
                    self.chain_store.get_block_header(block_header.prev_hash())?;
                if !verify_path(
                    *prev_block_header.chunk_headers_root(),
                    prev_chunk_proof,
                    &ChunkHashHeight(prev_chunk_header.chunk_hash().clone(), prev_chunk_header.height_included()),

                    byzantine_assert!(false);
                    return Err(Error::Other(
                        "set_shard_state failed: prev_chunk isn't included into block".into(),
                    ));
                }

                if chunk.height_included() != 0 {
                    return Err(Error::Other(
                    "set_shard_state failed: received empty state response for a chunk that is not at height 0".into()
                ));
                }

    fn validate_state_root_node(
        &self,
        state_root_node: &StateRootNode,
        state_root: &StateRoot,
    ) -> StateRootNodeValidationResult {
        if state_root == &Trie::EMPTY_ROOT {
            return if state_root_node == &StateRootNode::empty() {
                StateRootNodeValidationResult::Valid
            } else {
                StateRootNodeValidationResult::Invalid
            };
        }
        if hash(&state_root_node.data) != *state_root {
            return StateRootNodeValidationResult::Invalid;
        }
        match Trie::get_memory_usage_from_serialized(&state_root_node.data) {
            Ok(memory_usage) => {
                if memory_usage == state_root_node.memory_usage {
                    StateRootNodeValidationResult::Valid
                } else {
                    StateRootNodeValidationResult::Invalid
                }
            }
            Err(_) => StateRootNodeValidationResult::Invalid, // Invalid state_root_node
        }
    }
