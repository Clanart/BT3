### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Bricking - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS program accepts any arbitrary `class_hash` value from the caller without verifying that the hash corresponds to a previously declared contract class. An explicit `TODO` in the code acknowledges this missing check. A contract can call `replace_class` with an undeclared class hash, permanently setting its own class hash to an invalid value. Because the contract's class hash is now unresolvable, any subsequent call to that contract will fail irrecoverably at the OS level, permanently freezing all funds held by the contract.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function reads the requested new class hash directly from the syscall request and writes it into `contract_state_changes` without any validation:

```cairo
let class_hash = request.class_hash;
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}
tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);
dict_update{dict_ptr=contract_state_changes}(...);
``` [1](#0-0) 

The `TODO` comment at line 898 explicitly acknowledges the missing check. No assertion, range check, or dict lookup is performed to confirm that `class_hash` exists in `contract_class_changes` (i.e., was previously declared via a `declare` transaction).

When any subsequent call is made to the bricked contract, `execute_entry_point` performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash
);
// ...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    elm_size=CompiledClassFact.SIZE,
    n_elms=compiled_class_facts_bundle.n_compiled_class_facts,
    key=compiled_class_hash,
);
``` [2](#0-1) 

If `execution_context.class_hash` is an undeclared value, `dict_read` on `contract_class_changes` returns `0` (the default for an uninitialized dict entry). `find_element` then searches for a compiled class with hash `0`, which does not exist in `compiled_class_facts_bundle`, causing the OS execution to fail irrecoverably. The contract can never be called again, and there is no recovery path: calling `replace_class` again to fix the class hash requires executing the contract, which is now impossible.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is set to an undeclared value via `replace_class`, the contract is permanently bricked. All ERC-20 tokens, ETH, or other assets held in the contract's storage become permanently inaccessible. There is no upgrade path, no admin override, and no protocol-level recovery mechanism. The state corruption is committed to the proven state and propagated to L1.

---

### Likelihood Explanation

**Medium.**

The `replace_class` syscall is callable by any unprivileged contract without any special role or permission. A contract with a bug in its upgrade logic (e.g., computing a class hash incorrectly, or accepting an attacker-supplied hash as the new class) can trigger this path. The explicit `TODO` comment confirms the OS developers are aware this check is absent. Any contract that holds significant funds and implements an upgrade mechanism is at risk.

---

### Recommendation

In `execute_replace_class`, before writing the new class hash to `contract_state_changes`, validate that the class hash is present in `contract_class_changes` (i.e., it was previously declared). Concretely:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
with_attr error_message("Class hash is not declared.") {
    assert_not_zero(compiled_class_hash);
}
```

This mirrors the validation already implicitly required by `execute_entry_point` when resolving a class hash to its compiled form, and closes the gap between what the OS accepts and what it can actually execute.

---

### Proof of Concept

1. **Attacker deploys a contract** whose logic calls `replace_class(0xdeadbeef)`, where `0xdeadbeef` is an arbitrary value that was never declared on-chain.

2. **Attacker invokes the contract**, triggering the `replace_class` syscall. In `execute_replace_class`, the OS writes `class_hash = 0xdeadbeef` into `contract_state_changes` for the contract address with no validation. [3](#0-2) 

3. **The transaction is included in a block** and the state is committed. The contract's class hash is now `0xdeadbeef` in the proven state.

4. **Any subsequent call to the contract** reaches `execute_entry_point`. `dict_read{dict_ptr=contract_class_changes}(key=0xdeadbeef)` returns `0`. `find_element` searches for compiled class hash `0`, fails, and the OS execution aborts. [2](#0-1) 

5. **The contract is permanently bricked.** All funds held in its storage are frozen with no recovery path, since executing the contract (required to call `replace_class` again) is now impossible.
