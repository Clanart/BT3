### Title
Non-EVM-Denom (IBC/CosmWasm) Tokens Permanently Orphaned on Contract Self-Destruct — (File: x/evm/statedb/statedb.go)

---

### Summary

When a contract self-destructs, `StateDB.Commit()` burns only the EVM-native denom balance before deleting the auth account. Non-EVM-native tokens (IBC tokens, CosmWasm bridge tokens) held by the destroyed address are explicitly skipped and remain as orphaned bank balances after the auth account is removed. No recovery mechanism exists; those tokens are permanently locked.

---

### Finding Description

In `StateDB.Commit()`, the self-destruct cleanup path is:

```go
if obj.selfDestructed {
    cosmosAddr := sdk.AccAddress(obj.Address().Bytes())
    cacheCtx, writeCache := s.origCtx.CacheContext()
    // Only the EVM denom is burned here. Non-EVM-native tokens (IBC, CosmWasm
    // bridge) held by the destroyed address are not drained and may remain as
    // orphaned bank balances.
    if remaining := s.keeper.GetBalance(cacheCtx, cosmosAddr, s.evmDenom); remaining.Sign() > 0 {
        coin := sdk.NewCoin(s.evmDenom, sdkmath.NewIntFromBigInt(remaining.ToBig()))
        if _, err := s.keeper.SubBalance(cacheCtx, cosmosAddr, coin); err != nil {
            return errorsmod.Wrap(err, "failed to burn post-selfdestruct balance")
        }
    }
    if err := s.keeper.DeleteAccount(cacheCtx, obj.Address()); err != nil {
        return errorsmod.Wrap(err, "failed to delete account")
    }
    writeCache()
}
``` [1](#0-0) 

The code queries only `s.evmDenom` (e.g., `uphoton`) and burns that single denomination. The comment at lines 813–815 explicitly acknowledges the gap:

> *"Non-EVM-native tokens (IBC, CosmWasm bridge) held by the destroyed address are not drained and may remain as orphaned bank balances."* [2](#0-1) 

`DeleteAccount` then removes the auth account record and all EVM storage, but never touches the bank module for non-EVM denoms:

```go
func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
    ...
    k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool { ... })
    k.accountKeeper.RemoveAccount(ctx, acct)
    ...
}
``` [3](#0-2) 

After `writeCache()` commits, the bank module still holds any IBC or other non-EVM-denom coins at the now-deleted address. Because the auth account no longer exists, no signer can spend those coins. The Cosmos SDK bank module requires a valid auth account to send coins; without one, the balance is permanently inaccessible.

The `SelfDestruct()` function itself only burns the EVM denom at destruction time:

```go
balance := s.GetBalance(addr)   // reads only evmDenom
if balance.Sign() > 0 {
    s.SubBalance(addr, balance, tracing.BalanceDecreaseSelfdestructBurn)
}
``` [4](#0-3) 

Non-EVM-denom balances are never read or touched at any point in the selfdestruct lifecycle.

---

### Impact Explanation

Any non-EVM-denom tokens (IBC vouchers, CosmWasm bridge tokens, etc.) held by a self-destructing contract are permanently orphaned in the bank module. The bank module retains a positive balance at the deleted address, but no auth account exists to authorize a spend. The tokens cannot be burned, transferred, or recovered by any on-chain mechanism. This is a permanent, irreversible loss of Cosmos bank funds triggered through normal EVM transaction execution — matching the allowed High impact: *"valid user funds/fees to be mis-accounted."*

---

### Likelihood Explanation

IBC tokens held inside EVM contracts are a realistic and growing use case on Cosmos EVM chains (e.g., IBC-transferred assets used as collateral in DeFi contracts). `SELFDESTRUCT` (and EIP-6780 same-transaction destruction) is a standard EVM opcode reachable by any unprivileged transaction. No special privilege is required: any user who deploys a contract, receives IBC tokens into it, and triggers selfdestruct will silently lose those tokens. The bug is triggered by the normal EVM state-transition path through `ApplyMessageWithConfig` → `StateDB.Commit()`.

---

### Recommendation

Before calling `DeleteAccount`, iterate over **all** bank balances at the address (not just `s.evmDenom`) and either burn them or transfer them to a designated recovery module account. The Cosmos SDK `BankKeeper` exposes `GetAllBalances(ctx, addr)` for this purpose:

```go
allBalances := s.keeper.GetAllBalances(cacheCtx, cosmosAddr)
for _, coin := range allBalances {
    if _, err := s.keeper.SubBalance(cacheCtx, cosmosAddr, coin); err != nil {
        // handle or log; non-EVM denoms may need SendCoinsFromAccountToModule
    }
}
```

Alternatively, transfer non-EVM-denom balances to a community pool or governance-controlled module account so they remain recoverable via governance.

---

### Proof of Concept

1. Deploy a Solidity contract `C` on an Ethermint chain with IBC enabled. The contract has a `receive()` payable function and a `destroy()` function calling `selfdestruct(owner)`.
2. IBC-transfer 1000 `ibc/ATOM` tokens to the contract address `C` via the IBC transfer precompile or a standard IBC MsgTransfer targeting the EVM address.
3. In a single Ethereum transaction, call `C.destroy()`. The EVM executes `SELFDESTRUCT`, which calls `StateDB.SelfDestruct(C)` — this burns only the EVM-denom balance.
4. `StateDB.Commit()` processes the self-destructed object: burns the EVM denom, calls `DeleteAccount(C)` which removes the auth account and EVM storage.
5. After the block commits, query `bank.GetAllBalances(C)`: the 1000 `ibc/ATOM` tokens are still present at address `C`.
6. Attempt any spend from `C`: fails because `accountKeeper.GetAccount(C)` returns `nil` — no auth account exists.
7. The 1000 `ibc/ATOM` tokens are permanently orphaned. No governance or admin path exists to recover them.

The root cause is exclusively in `StateDB.Commit()` at lines 813–815 of `x/evm/statedb/statedb.go`, which is the necessary and only step where non-EVM-denom balances must be drained before account deletion. [5](#0-4)

func (k *Keeper) DeleteAccount(ctx sdk.Context, addr common.Address) error {
	cosmosAddr := sdk.AccAddress(addr.Bytes())
	acct := k.accountKeeper.GetAccount(ctx, cosmosAddr)
	if acct == nil {
		return nil
	}

	// NOTE: only Ethereum accounts (contracts) can be selfdestructed
	_, ok := acct.(ethermint.EthAccountI)
	if !ok {
		return errorsmod.Wrapf(types.ErrInvalidAccount, "type %T, address %s", acct, addr)
	}

	// clear storage
	k.ForEachStorage(ctx, addr, func(key, _ common.Hash) bool {
		k.SetState(ctx, addr, key, nil)
		return true
	})

	// remove auth account
	k.accountKeeper.RemoveAccount(ctx, acct)

	k.debugLog(ctx, "account suicided",
		"ethereum-address", addr,
		"cosmos-address", cosmosAddr,
	)

	return nil
```
