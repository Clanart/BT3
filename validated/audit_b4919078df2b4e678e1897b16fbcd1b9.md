### Title
Missing Backward-Compatibility Branch in `calculate_global_state_root` Produces Incorrect Global State Root — (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/state/commitment.cairo`)

---

### Summary

The function `calculate_global_state_root` in `commitment.cairo` is explicitly documented to handle three distinct cases for computing the global state root. However, the implementation only handles two of them. The missing case — when the contract class tree is empty (`contract_class_root == 0`) but the contract state tree is non-empty (`contract_state_root != 0`) — causes the OS to emit an incorrect global state root into its proof output. Because L1 expects the backward-compatible value (`contract_state_root`) for this case, the proof is rejected, and the network cannot confirm new transactions for any block that triggers this condition.

---

### Finding Description

The function comment at `commitment.cairo` lines 31–37 documents three cases:

```
// If both the contract class and contract state trees are empty, the global root is set to 0.
// If the contract class tree is empty, the global state root is equal to the
// contract state root (for backward compatibility);
// Otherwise, the global root is obtained by:
//     global_root = H(GLOBAL_STATE_VERSION, contract_state_root, contract_class_root).
```

The implementation at lines 41–48 is:

```cairo
if (contract_state_root == 0 and contract_class_root == 0) {
    return (global_root=0);
}
tempvar elements: felt* = new (GLOBAL_STATE_VERSION, contract_state_root, contract_class_root);
let (global_root) = poseidon_hash_many(n=3, elements=elements);
return (global_root=global_root);
```

Only two of the three documented cases are implemented:

| Condition | Documented behavior | Actual behavior |
|---|---|---|
| Both roots == 0 | Return 0 | Return 0 ✓ |
| `contract_class_root == 0`, `contract_state_root != 0` | Return `contract_state_root` | Return `H(VERSION, state_root, 0)` ✗ |
| Both roots != 0 | Return `H(VERSION, state_root, class_root)` | Return `H(VERSION, state_root, class_root)` ✓ |

The missing branch is the backward-compatibility case. When `contract_class_root == 0` and `contract_state_root != 0`, the code falls through to the hash computation and returns `H(GLOBAL_STATE_VERSION, contract_state_root, 0)` — a value that differs from the expected `contract_state_root`.

This incorrect value is then propagated through `state_update` in `state.cairo` (lines 90–97), stored in the `CommitmentUpdate` struct, placed into `OsOutputHeader.state_update_output` (line 35 of `output.cairo`), and serialized to the proof output via `serialize_output_header` (lines 161–162 of `output.cairo`). The L1 verifier receives and checks this root against its own stored state root, which was computed using the backward-compatible formula. The mismatch causes proof rejection.

---

### Impact Explanation

**High — Network not being able to confirm new transactions (total network shutdown).**

The global state root is the single value that anchors the entire StarkNet state on L1. When the OS emits an incorrect `final_root` (or `initial_root`) in the `CommitmentUpdate`, the L1 verifier rejects the proof. No state update can be applied to L1, and no further blocks can be finalized. This is a complete halt of the network for any block that triggers the condition.

The incorrect root is not a minor rounding error — it is a cryptographically distinct value (`H(VERSION, state_root, 0)` vs. `state_root`), so there is no possibility of accidental agreement.

---

### Likelihood Explanation

The condition `contract_class_root == 0` with `contract_state_root != 0` is triggered whenever:

1. **A new network deployment** processes its first block(s) with state changes (e.g., fee token contract initialization, pre-deployed system contracts) but before any Sierra class is declared. This is a realistic and common scenario for testnets and new chain deployments.
2. **Historical block replay** for blocks produced before the contract class tree was introduced. The OS is used for proof generation over historical ranges; any such block would trigger the bug.

No privileged access is required. Any unprivileged transaction sender whose transaction is included in such a block contributes to triggering the condition. The attacker does not need to do anything special — the network state itself (empty class tree) is the trigger.

---

### Recommendation

Add the missing backward-compatibility branch to `calculate_global_state_root`:

```cairo
func calculate_global_state_root{poseidon_ptr: PoseidonBuiltin*, range_check_ptr}(
    contract_state_root: felt, contract_class_root: felt
) -> (global_root: felt) {
    if (contract_state_root == 0 and contract_class_root == 0) {
        return (global_root=0);
    }
    // Backward compatibility: if the class tree is empty, return the state root directly.
    if (contract_class_root == 0) {
        return (global_root=contract_state_root);
    }
    tempvar elements: felt* = new (GLOBAL_STATE_VERSION, contract_state_root, contract_class_root);
    let (global_root) = poseidon_hash_many(n=3, elements=elements);
    return (global_root=global_root);
}
```

This aligns the implementation with the documented specification and with the L1 verifier's expected formula for the backward-compatible case.

### Proof of Concept

**Setup:** Deploy a new StarkNet-compatible network. At genesis, the contract class tree is empty (`class_root = 0`). The contract state tree is also empty (`state_root = 0`). Both roots are 0, so `calculate_global_state_root` correctly returns 0.

**Trigger:** Submit a transaction that modifies contract state (e.g., a fee payment or a pre-deployed system contract write) but does not declare any new class. After execution, `final_state_root != 0` and `final_class_root == 0`.

**OS execution path:**

1. `execute_blocks` in `os.cairo` (line 230) calls `state_update`.
2. `state_update` in `state.cairo` (lines 90–97) calls `calculate_global_state_root` twice — once for the initial roots and once for the final roots.
3. For the final roots: `contract_state_root != 0`, `contract_class_root == 0`. The missing branch is not taken. The function computes `H(GLOBAL_STATE_VERSION, final_state_root, 0)` and returns it as `final_global_root`.
4. This incorrect root is stored in `CommitmentUpdate.final_root` and serialized by `serialize_output_header` (lines 161–162 of `output.cairo`).
5. L1 receives the proof. Its stored state root is `final_state_root` (the backward-compatible value). The proof asserts `H(GLOBAL_STATE_VERSION, final_state_root, 0)`. The values differ. L1 rejects the proof.
6. No further blocks can be finalized. The network halts. [1](#0-0) [2](#0-1) [3](#0-2)
