### Title
L1 Gas Price Mean Computed from Insufficient History During Startup Enables Fee Manipulation - (File: `crates/apollo_l1_gas_price/src/l1_gas_price_provider.rs`)

---

### Summary

The `L1GasPriceProvider` silently falls back to computing the sliding-window mean from as few as **1 block** when the ring buffer holds fewer entries than `number_of_blocks_for_mean`. This is the direct sequencer analog of the "observation cardinality of 1" TWAP flaw: during sequencer startup or restart, an attacker who has recently spiked Ethereum base fees can cause the sequencer to embed a manipulated `l1_gas_price` / `l1_data_gas_price` into block proposals, producing wrong fee values that are committed to the block header and block hash.

---

### Finding Description

`L1GasPriceProvider::get_price_info` computes the mean gas price over a sliding window of `number_of_blocks_for_mean` (production value: 300) L1 blocks. When the ring buffer contains fewer than that many entries, the code silently shrinks the window to whatever is available:

```rust
// crates/apollo_l1_gas_price/src/l1_gas_price_provider.rs  lines 147-158
let first_index = if last_index >= num_blocks {
    last_index - num_blocks
} else {
    warn!(
        "Not enough history to calculate the mean gas price. Using blocks {}-{}, inclusive.",
        samples[0].block_number,
        samples[last_index - 1].block_number,
    );
    L1_GAS_PRICE_PROVIDER_INSUFFICIENT_HISTORY.increment(1);
    0   // ← window shrinks to [0, last_index), possibly just 1 block

``` [1](#0-0) 

The function then divides the sum by `actual_number_of_blocks = last_index - first_index`, which can be 1. No error is returned; the caller receives a `PriceInfo` that looks authoritative. [2](#0-1) 

**Startup window.** The `L1GasPriceScraper` is configured to pre-fill the buffer by scraping from `latest_l1_block - (number_of_blocks_for_mean × startup_num_blocks_multiplier)` = `latest - 600` blocks. There is no synchronization barrier that prevents the consensus orchestrator from calling `get_price_info` before those 300+ blocks have been ingested. During the fill-up phase the buffer holds 1 → 299 blocks, and every call to `get_price_info` uses the insufficient-history path. [3](#0-2) 

**Attacker-controlled data path.** The resulting `PriceInfo` flows directly into `get_l1_prices_in_fri_and_wei_and_conversion_rate`, which converts it to `L1PricesInFri` / `L1PricesInWei` and places both into the `ProposalInit` message broadcast to all validators: [4](#0-3) 

Validators accept the proposer's gas prices from `ProposalInit` and reconstruct `BlockInfo` via `convert_to_sn_api_block_info`, embedding the manipulated prices into the executed block: [5](#0-4) 

The `l1_gas_price` and `l1_data_gas_price` fields (both wei and fri) are stored in the block header and are inputs to `calculate_block_hash`, so a manipulated value produces a wrong committed block hash. [6](#0-5) 

---

### Impact Explanation

**Impact: High**

The manipulated `l1_gas_price` / `l1_data_gas_price` values are:

1. **Committed to the block header and block hash** — every block produced during the startup window carries a wrong gas price, which is an authoritative wrong value returned by RPC (`starknet_getBlockWithTxHashes`, `starknet_getBlockWithTxs`, etc.).
2. **Used for fee charging** — all transactions in those blocks have their actual fees computed against the manipulated price, causing users to overpay (spike) or underpay (suppression) relative to the true L1 cost. This is a direct economic impact.
3. **Used by `starknet_estimateFee` / `starknet_simulateTransactions`** — the pending block's gas price is sourced from the same provider, so RPC fee estimation returns wrong values during the window.

This matches the allowed impact: *"Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact"* and *"RPC execution, fee estimation … returns an authoritative-looking wrong value."*

---

### Likelihood Explanation

**Likelihood: Medium**

- The insufficient-history window is **always present** at every sequencer startup or restart — no attacker action is needed to create the condition.
- Manipulating Ethereum base fees requires capital (filling L1 blocks with high-fee transactions) but is a well-known technique; the attacker only needs to have spiked fees within the last ~600 L1 blocks (~2 hours) before the restart.
- The window during which the buffer is under-filled is bounded by how fast the scraper can ingest 300 historical blocks; in practice this is seconds to a few minutes, but the sequencer can produce multiple L2 blocks in that time.
- The `apply_fee_transformations` clamp (`min_l1_gas_price_wei` / `max_l1_gas_price_wei`) provides partial mitigation, but only if `max_l1_gas_price_wei` is set conservatively; the production config does not show a tight upper bound. [7](#0-6) 

---

### Recommendation

1. **Block proposal building until the buffer is warm.** Add a readiness check in `get_price_info` (or expose a separate `is_ready() -> bool`) that returns `InsufficientHistoryError` instead of silently succeeding when `last_index < num_blocks`. The orchestrator should treat this as a non-retryable error and delay proposal building (or use the explicit fallback to previous-block prices) until the buffer is full.

2. **Enforce a minimum window size.** Rather than falling back to `first_index = 0`, return `L1GasPriceProviderError::InsufficientHistoryError` so callers can decide whether to use the previous block's prices or the default, both of which are safer than a 1-block mean.

3. **Synchronize scraper pre-fill with sequencer start.** The scraper should signal "pre-fill complete" (e.g., via a `tokio::sync::watch` channel) before the consensus orchestrator is allowed to call `get_price_info` for the first time.

---

### Proof of Concept

```
Precondition: Attacker has spiked Ethereum base fee to X (e.g., 10× normal)
              for blocks [L - 10 .. L] on L1, where L is the current L1 tip.

1. Sequencer restarts (upgrade, crash, or normal maintenance).

2. L1GasPriceScraper initialises the provider (empty buffer) and begins
   scraping from block L - 600.

3. After ingesting only 5 blocks (all with spiked base fee X), the consensus
   orchestrator calls get_price_info(timestamp_T).

4. In get_price_info:
     last_index = 5
     num_blocks = 300
     → insufficient history branch: first_index = 0
     → mean = sum(X, X, X, X, X) / 5 = X   (10× normal)

5. The orchestrator embeds l1_gas_price = X into ProposalInit and broadcasts
   it to validators.

6. Validators accept the proposal; the block is committed with l1_gas_price = X.

7. All transactions in that block are charged fees based on X (10× normal).
   Users who set max_fee ≥ X are overcharged; the block hash commits to the
   wrong price.

8. starknet_getBlockWithTxHashes returns l1_gas_price = X as authoritative.
``` [8](#0-7) [9](#0-8)

    pub fn get_price_info(&self, timestamp: BlockTimestamp) -> L1GasPriceProviderResult<PriceInfo> {
        let Some(samples) = &self.price_samples_by_block else {
            return Err(L1GasPriceProviderError::NotInitializedError);
        };
        // timestamp of the newest price sample
        let last_timestamp = samples
            .back()
            .ok_or(L1GasPriceProviderError::MissingDataError {
                timestamp: timestamp.0,
                lag: self.config.lag_margin_seconds.as_secs(),
            })?
            .timestamp;

        // Check if the prices are stale.
        if timestamp.0 > (*last_timestamp + self.config.max_time_gap_seconds) {
            return Err(L1GasPriceProviderError::StaleL1GasPricesError {
                current_timestamp: timestamp.0,
                last_valid_price_timestamp: *last_timestamp,
            });
        }

        // This index is for the last block in the mean (inclusive).
        let index_last_timestamp_rev = samples.iter().rev().position(|data| {
            data.timestamp <= timestamp.saturating_sub(&self.config.lag_margin_seconds.as_secs())
        });

        // Could not find a block with the requested timestamp and lag.
        let Some(last_index_rev) = index_last_timestamp_rev else {
            return Err(L1GasPriceProviderError::MissingDataError {
                timestamp: timestamp.0,
                lag: self.config.lag_margin_seconds.as_secs(),
            });
        };
        // Convert the index to the forward direction.
        // `last_index` should be one past the final entry we will include in our calculation.
        // The index returned from `position` is guaranteed to be less than `len()`,
        // so `last_index` is guaranteed to be >= 1.
        let last_index = samples.len() - last_index_rev;

        let num_blocks = usize::try_from(self.config.number_of_blocks_for_mean)
            .expect("number_of_blocks_for_mean is too large to fit into a usize");

        let first_index = if last_index >= num_blocks {
            last_index - num_blocks
        } else {
            warn!(
                "Not enough history to calculate the mean gas price. Using blocks {}-{}, \
                 inclusive.",
                samples[0].block_number,
                samples[last_index - 1].block_number,
            );
            L1_GAS_PRICE_PROVIDER_INSUFFICIENT_HISTORY.increment(1);
            0
        };

pub(crate) async fn get_l1_prices_in_fri_and_wei_and_conversion_rate(
    l1_gas_price_provider_client: Arc<dyn L1GasPriceProviderClient>,
    timestamp: u64,
    previous_block_info: Option<&PreviousBlockInfo>,
    gas_price_params: &GasPriceParams,
) -> (L1PricesInFri, L1PricesInWei, u128) {
    // One of these paths should fill the return values:
    // 1. Both L1 gas price and eth/strk rate are Ok, use those.
    // 2. Otherwise, use previous block info.
    // 3. If that isn't available either, use min gas prices and default eth/strk rate.

    // Get the eth to fri rate from the oracle, and the L1 gas price (in wei) from the provider.
    let (eth_to_fri_rate, price_info) = tokio::join!(
        l1_gas_price_provider_client.get_eth_to_fri_rate(timestamp),
        l1_gas_price_provider_client.get_price_info(BlockTimestamp(timestamp))
    );
    if price_info.is_err() {
        warn!("Failed to get l1 gas price from provider: {:?}", price_info);
        CONSENSUS_L1_GAS_PRICE_PROVIDER_ERROR.increment(1);
    }
    if eth_to_fri_rate.is_err() {
        warn!("Failed to get eth to fri rate from oracle: {:?}", eth_to_fri_rate);
    }
    if let (Ok(eth_to_fri_rate), Ok(mut price_info)) = (eth_to_fri_rate, price_info) {
        // Both L1 prices and rate are Ok, so we can use them.
        info!(
            "raw eth_to_fri_rate (from oracle): {eth_to_fri_rate}, raw l1 gas price wei (from \
             provider): {price_info:?}"
        );
        apply_fee_transformations(&mut price_info, gas_price_params);
        let prices_in_wei = L1PricesInWei {
            l1_gas_price: price_info.base_fee_per_gas,
            l1_data_gas_price: price_info.blob_fee,
        };
        // Apply the eth/strk rate to get prices in fri.
        let l1_gas_prices_fri_result =
            L1PricesInFri::convert_from_wei(&prices_in_wei, eth_to_fri_rate);
        // If conversion fails, leave return_value=None to try backup methods.
        if let Ok(prices_in_fri) = l1_gas_prices_fri_result {
            return (prices_in_fri, prices_in_wei, eth_to_fri_rate);
        } else {
            warn!(
                "Failed to convert L1 gas prices to FRI: {:?}",
                l1_gas_prices_fri_result.clone().err()
            );
        }
    }

pub(crate) fn apply_fee_transformations(
    price_info: &mut PriceInfo,
    gas_price_params: &GasPriceParams,
) {
    price_info.base_fee_per_gas = price_info
        .base_fee_per_gas
        .saturating_add(gas_price_params.l1_gas_tip_wei)
        .clamp(gas_price_params.min_l1_gas_price_wei, gas_price_params.max_l1_gas_price_wei);

    price_info.blob_fee = GasPrice(
        (gas_price_params.l1_data_gas_price_multiplier * price_info.blob_fee.0).to_integer(),
    )
    .clamp(gas_price_params.min_l1_data_gas_price_wei, gas_price_params.max_l1_data_gas_price_wei);
}

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let previous_block_info = PreviousBlockInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,

                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),

    })
