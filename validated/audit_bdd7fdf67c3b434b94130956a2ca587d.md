### Title
Stale Fee Market Params in `NewDynamicFeeChecker` and `MinGasPriceDecorator` Allow Cosmos SDK Transactions to Bypass Current EIP-1559 Base Fee - (File: `ante/evm/fee_checker.go`, `evmd/ante/handler_options.go`)

### Summary

`newCosmosAnteHandler` and `newLegacyCosmosAnteHandlerEip712` snapshot `feemarketParams` and `evmParams` once at ante-handler construction time (app startup). These stale snapshots are closed over by `NewDynamicFeeChecker` and `MinGasPriceDecorator`. Because the EIP-1559 base fee changes every block, Cosmos SDK transactions are fee-validated against a permanently stale base fee, allowing under-fee transactions to be admitted and committed whenever the live base fee has risen above the startup value.

The same pattern exists in `newLegacyCosmosAnteHandlerEip712`: [2](#0-1) 

The returned `sdk.AnteHandler` closure is invoked on every transaction, but it always uses the frozen `feemarketParams`. Inside `NewDynamicFeeChecker`, the base fee is derived from this stale snapshot:

```go

``` [3](#0-2) 

`feemarketParams.BaseFee` is the value stored in the KV-store at app startup. The live base fee is updated every block by `BeginBlock` → `CalculateBaseFee` → `SetBaseFee`: [4](#0-3) 

The base fee stored in the KV-store changes every block based on gas utilization: [5](#0-4) 

**Contrast with EVM transactions**: `newEthAnteHandler` fetches fresh params on every invocation via `EVMBlockConfig`, so EVM transactions are not affected:

```go

``` [6](#0-5) 

`MinGasPriceDecorator` also uses the stale snapshot for `MinGasPrice`:

```go

``` [7](#0-6) 

### Impact Explanation

**High.** This is a fee market / ante handler bug that permits invalid Cosmos SDK transactions (fees below the current EIP-1559 base fee) to be admitted to the mempool and committed to the chain. Whenever the live base fee rises above the startup value — which is the normal operating condition on a busy chain — any Cosmos SDK transaction priced at the stale startup base fee passes `NewDynamicFeeChecker` and `MinGasPriceDecorator` and is committed. The fee distribution mechanism receives less than the protocol-mandated base fee, constituting a mis-accounting of user fees. This matches the allowed impact: *"fee market, ante handler, mempool, or proposal handling bug that permits invalid transactions to commit or valid user funds/fees to be mis-accounted."*

### Likelihood Explanation

**Medium.** The base fee changes every block on any chain with non-zero block utilization. After even a few blocks of above-target gas usage, the live base fee diverges from the startup snapshot. Any unprivileged user who submits a Cosmos SDK transaction (e.g., governance, IBC, staking) with fees calibrated to the stale base fee will bypass the current fee floor. No special privileges or coordination are required; the attacker only needs to know the startup base fee (publicly observable from genesis or the first block).

### Recommendation

1. **`NewDynamicFeeChecker`**: Remove the `feemarketParams` closure parameter. Instead, read the live base fee from the store on every invocation using the `ctx` already available inside the closure:
   ```go
   // Read live base fee from keeper, not stale snapshot
   baseFee := feeMarketKeeper.GetBaseFee(ctx)
   ```
2. **`MinGasPriceDecorator`**: Replace `mpd.feemarketParams.MinGasPrice` with a live read: `mpd.feesKeeper.GetParams(ctx).MinGasPrice`.
3. **`newCosmosAnteHandler` / `newLegacyCosmosAnteHandlerEip712`**: Pass the live keeper references into decorators rather than snapshotted param structs, mirroring the pattern already used correctly in `newEthAnteHandler`.

### Proof of Concept

1. Chain starts. `feemarketParams.BaseFee` = `B0` (e.g., `1_000_000_000`). `newCosmosAnteHandler` captures this value.
2. Over N blocks of above-target gas usage, `BeginBlock` updates the live base fee to `B_N = 2_000_000_000`.
3. Attacker submits a Cosmos SDK governance `MsgVote` transaction with `fee = B0 * gasLimit` (half the current required fee).
4. `NewDynamicFeeChecker` computes `baseFee = types.GetBaseFee(..., feemarketParams)` → returns stale `B0`. The check `feeCap >= baseFeeInt` passes.
5. `MinGasPriceDecorator` checks against stale `feemarketParams.MinGasPrice` — also passes.
6. The transaction is deducted at `B0 * gasLimit` and committed, paying half the protocol-mandated fee. The fee collector receives `B0 * gasLimit` instead of `B_N * gasLimit`, mis-accounting `(B_N - B0) * gasLimit` tokens per such transaction.

func NewDynamicFeeChecker(ethCfg *params.ChainConfig, evmParams *types.Params, feemarketParams *feemarkettypes.Params) authante.TxFeeChecker {
	return func(ctx sdk.Context, tx sdk.Tx) (sdk.Coins, int64, error) {
		feeTx, ok := tx.(sdk.FeeTx)
		if !ok {
			return nil, 0, fmt.Errorf("tx must be a FeeTx")
		}

		if ctx.BlockHeight() == 0 {
			// genesis transactions: fallback to min-gas-price logic
			return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
		}

		denom := evmParams.EvmDenom

		baseFee := types.GetBaseFee(ctx.BlockHeight(), ethCfg, feemarketParams)
		if baseFee == nil {
			// london hardfork is not enabled: fallback to min-gas-prices logic
			return checkTxFeeWithValidatorMinGasPrices(ctx, feeTx)
		}

func (k *Keeper) BeginBlock(ctx sdk.Context) error {
	baseFee := k.CalculateBaseFee(ctx)

	// return immediately if base fee is nil
	if baseFee == nil {
		return nil
	}

	k.SetBaseFee(ctx, baseFee)

	defer func() {
		telemetry.SetGauge(float32(baseFee.Int64()), "feemarket", "base_fee") //nolint:staticcheck
	}()

	// Store current base fee in event
	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeFeeMarket,
			sdk.NewAttribute(types.AttributeKeyBaseFee, baseFee.String()),
		),
	})
	return nil

type MinGasPriceDecorator struct {
	feesKeeper      interfaces.FeeMarketKeeper
	evmDenom        string
	feemarketParams *feemarkettypes.Params
}

// NewMinGasPriceDecorator creates a new MinGasPriceDecorator instance used only for
// Cosmos transactions.
func NewMinGasPriceDecorator(fk interfaces.FeeMarketKeeper, evmDenom string, feemarketParams *feemarkettypes.Params) MinGasPriceDecorator {
	return MinGasPriceDecorator{feesKeeper: fk, evmDenom: evmDenom, feemarketParams: feemarketParams}
}

func (mpd MinGasPriceDecorator) AnteHandle(ctx sdk.Context, tx sdk.Tx, simulate bool, next sdk.AnteHandler) (newCtx sdk.Context, err error) {
	feeTx, ok := tx.(sdk.FeeTx)
	if !ok {
		return ctx, errorsmod.Wrapf(errortypes.ErrInvalidType, "invalid transaction type %T, expected sdk.FeeTx", tx)
	}

	minGasPrice := mpd.feemarketParams.MinGasPrice

	// Short-circuit if min gas price is 0 or if simulating
	if minGasPrice.IsZero() || simulate {
		return next(ctx, tx, simulate)
	}
