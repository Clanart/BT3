### Title
Missing EOA Sender Validation for Prague/EIP-7702 Allows Contract Accounts to Send Transactions - (File: `ante/eth.go`)

### Summary

When the Prague hardfork is active (enabling EIP-7702), Ethermint's `VerifyEthAccount` ante handler completely skips the EOA sender check instead of narrowing it to allow only EIP-7702 delegation code. This allows any account with non-delegation contract code (and a known private key) to send transactions, bypassing the standard Ethereum requirement that transaction senders must be EOAs or EIP-7702 delegated accounts.

### Finding Description

In `ante/eth.go`, `VerifyEthAccount` guards the EOA check with a `!rules.IsPrague` condition:

```go
// check whether the sender address is EOA
acct := statedb.NewAccountFromSdkAccount(accountGetter(from))

if !rules.IsPrague {
    if acct.IsContract() {
        fromAddr := common.BytesToAddress(from)
        return errorsmod.Wrapf(errortypes.ErrInvalidType,
            "the sender is not EOA: address %s, codeHash <%s>", fromAddr, acct.CodeHash)
    }
}
``` [1](#0-0) 

When `rules.IsPrague` is `true`, the **entire block is skipped** — no check is performed at all. The correct behavior per EIP-7702 and go-ethereum's `core/state_transition.go` `TransitionDb` is to allow senders whose code is an EIP-7702 delegation designator, but **still reject** senders with non-delegation contract code:

```go
// go-ethereum TransitionDb (Prague-aware):
if codeHash := st.state.GetCodeHash(msg.From); codeHash != (common.Hash{}) && codeHash != types.EmptyCodeHash {
    if _, ok := types.ParseDelegation(st.state.GetCode(msg.From)); !ok {
        return nil, fmt.Errorf("%w: address %v, codehash: %s", ErrSenderNoEOA, msg.From.Hex(), codeHash)
    }
}
```

Ethermint's `ApplyMessageWithConfig` calls `evm.Call` directly (not go-ethereum's `TransitionDb`), so this go-ethereum check is never reached. The ante handler is the **only** enforcement point, and it is disabled for Prague. [2](#0-1) 

The `validateAuthorization` function in `set_code_authorizations.go` does check that the **authority** (the account being delegated) has no non-delegation code, but this is a different check — it applies to the authorization target, not to the outer transaction sender. [3](#0-2) 

### Impact Explanation

An attacker who controls the private key for an address that has non-delegation contract code deployed to it can, on a Prague-enabled Ethermint chain:

1. **Transfer funds** held by the contract account to an arbitrary address.
2. **Mutate the contract account's state** by calling arbitrary functions as the contract account.
3. **Execute arbitrary calls** as the contract account, bypassing any access controls that assume the address is a pure contract (not a transaction sender).

This is a signer verification bypass enabling forged execution and unauthorized account/code mutation, matching the allowed High impact: *"Ethereum transaction, EIP-7702 authorization, nonce, chain-id, or signer verification bypass enabling replay, forged execution, or unauthorized account/code mutation."*

### Likelihood Explanation

The attack requires:
1. Prague hardfork enabled on the chain (required for EIP-7702 support).
2. An address for which the attacker knows the private key **and** which has non-delegation contract code deployed to it.

The second condition is achievable via CREATE2 vanity address mining: an attacker pre-computes a deployer salt such that the resulting CREATE2 address matches an address for which they hold the private key. This is computationally feasible for addresses with short vanity prefixes and is a known technique. Once the contract is deployed to that address by any party, the attacker can exploit the missing check.

### Recommendation

Replace the `if !rules.IsPrague` guard with a check that allows delegation code but rejects non-delegation code, mirroring go-ethereum's behavior:

```go
acct := statedb.NewAccountFromSdkAccount(accountGetter(from))
if acct.IsContract() {
    if rules.IsPrague {
        // EIP-7702: allow delegation designators, reject plain contract code
        code := evmKeeper.GetCode(ctx, from)
        if _, ok := ethtypes.ParseDelegation(code); !ok {
            return errorsmod.Wrapf(errortypes.ErrInvalidType,
                "the sender is not EOA: address %s, codeHash <%s>", fromAddr, acct.CodeHash)
        }
    } else {
        return errorsmod.Wrapf(errortypes.ErrInvalidType,
            "the sender is not EOA: address %s, codeHash <%s>", fromAddr, acct.CodeHash)
    }
}
```

### Proof of Concept

1. Mine a CREATE2 salt `s` such that `CREATE2(deployer, s, initcode)` produces address `X`, where `X` is also the Ethereum address derived from a known private key `k`.
2. Deploy any non-delegation contract to `X` using the mined salt (can be done by any party).
3. Enable Prague hardfork on the Ethermint chain.
4. Construct a `MsgEthereumTx` signed with private key `k` (sender = `X`) transferring the contract's ETH balance to an attacker-controlled address.
5. Submit the transaction. `VerifyEthAccount` skips the EOA check because `rules.IsPrague == true`.
6. `CheckEthCanTransfer` passes (balance check only).
7. `CheckAndSetEthSenderNonce` increments `X`'s sequence.
8. `ApplyMessageWithConfig` executes the call — funds are transferred, state is mutated. [4](#0-3) [5](#0-4)

// VerifyEthAccount validates checks that the sender balance is greater than the total transaction cost.
// The account will be created in memory if it doesn't exist, i.e cannot be found on store, which will eventually set to
// store when increasing nonce.
// This AnteHandler decorator will fail if:
// - any of the msgs is not a MsgEthereumTx
// - from address is empty
// - account balance is lower than the transaction cost
func VerifyEthAccount(
	ctx sdk.Context, tx sdk.Tx,
	evmKeeper interfaces.EVMKeeper, evmDenom string,
	accountGetter AccountGetter,
	rules params.Rules,
) error {
	for _, msg := range tx.GetMsgs() {
		msgEthTx, ok := msg.(*evmtypes.MsgEthereumTx)
		if !ok {
			return errorsmod.Wrapf(errortypes.ErrUnknownRequest, "invalid message type %T, expected %T", msg, (*evmtypes.MsgEthereumTx)(nil))
		}

		ethTx := msgEthTx.AsTransaction()

		// sender address should be in the tx cache from the previous AnteHandle call
		from := msgEthTx.GetFrom()
		if from.Empty() {
			return errorsmod.Wrap(errortypes.ErrInvalidAddress, "from address cannot be empty")
		}

		// check whether the sender address is EOA
		acct := statedb.NewAccountFromSdkAccount(accountGetter(from))

		if !rules.IsPrague {
			if acct.IsContract() {
				fromAddr := common.BytesToAddress(from)
				return errorsmod.Wrapf(errortypes.ErrInvalidType,
					"the sender is not EOA: address %s, codeHash <%s>", fromAddr, acct.CodeHash)
			}
		}

		balance := evmKeeper.GetBalance(ctx, from, evmDenom)
		if err := keeper.CheckSenderBalance(sdkmath.NewIntFromBigIntMut(balance.ToBig()), ethTx); err != nil {
			return errorsmod.Wrap(err, "failed to check sender balance")
		}
	}
	return nil
}
