### Title
Unconstrained `old_block_hash` in `write_block_number_to_block_hash_mapping` Allows Malicious Prover to Forge Block Hash Mapping — (File: os_utils.cairo)

---

### Summary

In `os_utils.cairo`, the function `write_block_number_to_block_hash_mapping` writes an `old_block_hash` value into the block hash mapping contract storage. This value is a `local` variable assigned exclusively by a hint (`%{ GetOldBlockNumberAndHash %}`), with **no Cairo constraint** binding it to any verified data. A malicious prover can set `old_block_hash` to any arbitrary felt, and this forged value will be committed into the state root — which is verified by L1 — making it indistinguishable from a legitimate block hash. Contracts that call `get_block_hash` will receive the forged value.

---

### Finding Description

In `os_utils.cairo`, `write_block_number_to_block_hash_mapping` is responsible for writing the hash of a past block into the dedicated block hash mapping contract storage (the contract at `BLOCK_HASH_CONTRACT_ADDRESS`):

```cairo
// Currently, the block hash mapping is not enforced by the OS.
// TODO(Yoni, 1/1/2026): output this hash.
local old_block_hash;
%{ GetOldBlockNumberAndHash %}

assert state_entry.class_hash = 0;
assert state_entry.nonce = 0;
tempvar storage_ptr = state_entry.storage_ptr;
assert [storage_ptr] = DictAccess(key=old_block_number, prev_value=0, new_value=old_block_hash);
``` [1](#0-0) 

`old_block_hash` is declared as a `local` variable and assigned by the hint `%{ GetOldBlockNumberAndHash %}`. In Cairo's proof model, hints are **not constraints** — they are only used to help the prover fill in witness values. The verifier does not check hints. There is no `assert` statement anywhere that constrains `old_block_hash` to equal any verified value (e.g., the actual hash of the old block as committed on L1 or as output by the OS for the corresponding block).

The `old_block_hash` value is then written as `new_value` in a `DictAccess` entry for the block hash mapping contract storage. This storage update flows into the state diff, which is committed to the state root via the Patricia tree in `compute_contract_state_commitment`. [2](#0-1) 

The state root **is** constrained and verified by the L1 contract. Therefore, the forged `old_block_hash` value is accepted as part of the valid, L1-verified state — exactly analogous to how the sha256-air's unconstrained `final_hash` was accepted as a valid hash output.

The comment in the code explicitly acknowledges the missing constraint: `// Currently, the block hash mapping is not enforced by the OS.` [3](#0-2) 

---

### Impact Explanation

Contracts on StarkNet call the `get_block_hash` syscall to retrieve the hash of a past block. This syscall reads from the block hash mapping contract storage populated by `write_block_number_to_block_hash_mapping`. If a malicious prover writes a forged `old_block_hash`, every contract that calls `get_block_hash` for that block number receives the forged value — committed into the verified state root.

Contracts that use `get_block_hash` for security-critical purposes (e.g., as a source of on-chain randomness, to implement commit-reveal schemes, or to verify L1 event inclusion) can be directly exploited. A malicious prover who controls the forged block hash in advance can manipulate the outcome of these contracts, leading to **direct loss of funds**.

**Impact: Critical — Direct loss of funds.**

---

### Likelihood Explanation

A malicious prover trivially exploits this by modifying the hint `%{ GetOldBlockNumberAndHash %}` to assign any desired felt to `old_block_hash`. The Cairo verifier does not check hints; the proof remains valid. The forged value is committed to the state root and accepted by the L1 contract. No special access, leaked key, or privileged role is required beyond the ability to generate a proof — which is the exact threat model ZK proofs are supposed to defend against.

---

### Recommendation

`old_block_hash` must be constrained in the Cairo circuit. The fix should verify that `old_block_hash` equals the actual hash of the old block as previously output by the OS for the corresponding block. Concretely:

1. Include the old block hash in the OS output for the block that produced it (the existing TODO acknowledges this: `// TODO(Yoni, 1/1/2026): output this hash.`).
2. Add a Cairo `assert` that constrains `old_block_hash` to equal the value output by the OS for that block, analogous to how `final_hash` in sha256-air was fixed by asserting `final_hash[i] = prev_hash[i] + work_vars[i]`. [4](#0-3) 

---

### Proof of Concept

1. A malicious prover modifies the hint `%{ GetOldBlockNumberAndHash %}` in `os_utils.cairo` to assign `old_block_hash = 0` (or any attacker-chosen value) instead of the real block hash.
2. The prover generates a valid Cairo proof. The verifier accepts it because hints are not checked.
3. The forged `old_block_hash` is written into the block hash mapping contract storage as `new_value` in the `DictAccess` entry.
4. This storage update is committed to the state root via the Patricia tree and accepted by the L1 contract.
5. Any contract that subsequently calls `get_block_hash(old_block_number)` receives `0` (or the attacker-chosen value) instead of the real block hash.
6. Contracts using this value for randomness, commit-reveal, or L1 event verification are now exploitable, leading to direct loss of funds. [5](#0-4) [6](#0-5)

func write_block_number_to_block_hash_mapping{range_check_ptr, contract_state_changes: DictAccess*}(
    block_context: BlockContext*
) {
    alloc_locals;
    tempvar old_block_number = block_context.block_info_for_execute.block_number -
        STORED_BLOCK_HASH_BUFFER;
    let is_old_block_number_non_negative = is_nn(old_block_number);
    if (is_old_block_number_non_negative == FALSE) {
        // Not enough blocks in the system - nothing to write.
        return ();
    }

    // Fetch the (block number -> block hash) mapping contract state.
    local state_entry: StateEntry*;
    %{ GetBlockHashMapping %}

    // Currently, the block hash mapping is not enforced by the OS.
    // TODO(Yoni, 1/1/2026): output this hash.
    local old_block_hash;
    %{ GetOldBlockNumberAndHash %}

    // Update mapping.
    assert state_entry.class_hash = 0;
    assert state_entry.nonce = 0;
    tempvar storage_ptr = state_entry.storage_ptr;
    assert [storage_ptr] = DictAccess(key=old_block_number, prev_value=0, new_value=old_block_hash);
    let storage_ptr = storage_ptr + DictAccess.SIZE;
    %{ WriteOldBlockToStorage %}

    // Update contract state.
    tempvar new_state_entry = new StateEntry(class_hash=0, storage_ptr=storage_ptr, nonce=0);
    dict_update{dict_ptr=contract_state_changes}(
        key=BLOCK_HASH_CONTRACT_ADDRESS,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    return ();
