### Title
Unbounded Fallback to Stale `previous_block_info` L1 Gas Prices Embeds Wrong Fee Values in Block Commitments — (`File: crates/apollo_consensus_orchestrator/src/utils.rs`)

---

### Summary

When the L1 gas price scraper is unavailable, `get_l1_prices_in_fri_and_wei_and_conversion_rate` silently falls back to `previous_block_info` L1 prices with no staleness check on that cached value. Because both the proposer and validator use the same fallback path, proposals carrying arbitrarily old L1 gas prices pass `is_block_info_valid` and are committed to the chain, embedding wrong `l1_gas_price_fri` / `l1_data_gas_price_fri` values in every subsequent block header.

---

### Finding Description

`L1GasPriceProvider::get_price_info` already detects stale L1 data and returns `StaleL1GasPricesError` when the newest sample is older than `max_time_gap_seconds`: [1](#0-0) 

When that error (or any provider/oracle error) is returned, `get_l1_prices_in_fri_and_wei_and_conversion_rate` falls back to `previous_block_info`: [2](#0-1) 

`PreviousBlockInfo` carries the L2 block timestamp, not the L1 data timestamp: [3](#0-2) 

There is no comparison of `previous_block_info.timestamp` against the current wall-clock time before returning the cached prices. The L2 block timestamp advances with every new block, so the cached prices always appear "fresh" even though the underlying L1 data may be hours old.

Once the scraper goes down, every new block is built with the same frozen L1 prices. `decision_reached` updates `previous_block_info` from the committed block, so the stale prices propagate forward indefinitely — each block's `previous_block_info` carries the same stale values as the one before it.

The validator calls the identical fallback inside `is_block_info_valid`: [4](#0-3) 

Because both proposer and validator resolve to the same stale `previous_block_info`, the `within_margin` check passes and the proposal is accepted: [5](#0-4) 

The stale `l1_gas_price_fri` and `l1_data_gas_price_fri` are then committed into the block header via `convert_to_sn_api_block_info`: [6](#0-5) 

---

### Impact Explanation

Every transaction in every block produced during the outage has its L1 gas fee calculated against the stale price. If actual L1 gas prices have risen (e.g., during Ethereum network congestion — the exact scenario that causes the scraper to fall behind), users underpay for L1 data posting and the sequencer absorbs the difference. The wrong prices are permanently embedded in committed block headers and state diffs, making the error irreversible without a reorg. This matches: **Critical — Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact.**

---

### Likelihood Explanation

The L1 gas price scraper is an off-chain process that can fail due to Ethereum RPC unavailability, L1 reorg handling, oracle endpoint downtime, or bugs. The `L1GasPriceScraperError` variants show these are anticipated failure modes: [7](#0-6) 

The scraper loop logs errors and sleeps, but does not halt block production: [8](#0-7) 

Likelihood is **Low** (requires scraper downtime) but the window is unbounded — the chain continues indefinitely with stale prices.

---

### Recommendation

Add a wall-clock staleness check on `previous_block_info` before using it as a fallback. Compare `clock.unix_now()` against `previous_block_info.timestamp` and reject the fallback (falling through to the hard-coded minimum defaults) if the age exceeds a configurable threshold (e.g., `max_time_gap_seconds` already used by the provider). This mirrors the existing staleness guard in `L1GasPriceProvider::get_price_info` and closes the bypass introduced by the fallback path.

---

### Proof of Concept

1. The L1 gas price scraper is running normally; `previous_block_info` is populated with prices from block N (e.g., `l1_gas_price_fri = X`).
2. Ethereum network congestion causes the scraper to stop advancing. `L1GasPriceProvider::get_price_info` begins returning `StaleL1GasPricesError` because `timestamp.0 > last_timestamp + max_time_gap_seconds`.
3. The proposer calls `get_l1_prices_in_fri_and_wei_and_conversion_rate`. Both `price_info` and `eth_to_fri_rate` fail (or only `price_info` fails). The function falls back to `previous_block_info`, returning `l1_gas_price_fri = X` — the value from block N.
4. The proposer embeds `l1_gas_price_fri = X` in `ProposalInit` and broadcasts it.
5. The validator calls `is_block_info_valid`, which calls the same fallback and also computes `l1_gas_price_fri = X`. `within_margin(X, X, margin)` is trivially true. The proposal is accepted.
6. `decision_reached` updates `previous_block_info` from the newly committed block, which also carries `l1_gas_price_fri = X`. The cycle repeats for block N+1, N+2, … indefinitely.
7. Meanwhile, actual L1 gas prices have risen to `10X` due to congestion. All transactions in blocks N+1 … N+k pay fees computed at `X` instead of `10X`, causing the sequencer to under-collect L1 gas fees and subsidize L1 data posting costs for the entire outage window.

pub(crate) fn convert_to_sn_api_block_info(
    init: &ProposalInit,
) -> Result<starknet_api::block::BlockInfo, StarknetApiError> {
    if init.l1_gas_price_fri.0 == 0
        || init.l1_gas_price_wei.0 == 0
        || init.l1_data_gas_price_fri.0 == 0
        || init.l1_data_gas_price_wei.0 == 0
        || init.l2_gas_price_fri.0 == 0
    {
        warn!("Zero gas price detected in block info: {:?}", init);
    }

    let l1_gas_price_fri = NonzeroGasPrice::new(init.l1_gas_price_fri)?;
    let l1_data_gas_price_fri = NonzeroGasPrice::new(init.l1_data_gas_price_fri)?;
    let l1_gas_price_wei = NonzeroGasPrice::new(init.l1_gas_price_wei)?;
    let l1_data_gas_price_wei = NonzeroGasPrice::new(init.l1_data_gas_price_wei)?;
    let l2_gas_price_fri = NonzeroGasPrice::new(init.l2_gas_price_fri)?;
    let previous_block_info = PreviousBlockInfo::from(init);
    let eth_to_fri_rate = calculate_eth_to_fri_rate(&previous_block_info)?;

    let l2_gas_price_wei = NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)
        .inspect_err(|_| {
            warn!(
                "L2 gas price in wei is zero! Conversion rate: {eth_to_fri_rate}, L2 gas price in \
                 FRI: {}",
                init.l2_gas_price_fri
            )
        })?;
    Ok(starknet_api::block::BlockInfo {
        block_number: init.height,
        block_timestamp: BlockTimestamp(init.timestamp),
        sequencer_address: init.builder,
        gas_prices: GasPrices {
            strk_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_fri,
                l1_data_gas_price: l1_data_gas_price_fri,
                l2_gas_price: l2_gas_price_fri,
            },
            eth_gas_prices: GasPriceVector {
                l1_gas_price: l1_gas_price_wei,
                l1_data_gas_price: l1_data_gas_price_wei,
                l2_gas_price: l2_gas_price_wei,
            },
        },
        use_kzg_da: init.l1_da_mode.is_use_kzg_da(),
        starknet_version: init.starknet_version,
    })
}
