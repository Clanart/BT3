### Title
Floor Division in `induced_gas_costs` Allows Blocks to Exceed Configured Builtin Instance Limits — (File: crates/blockifier/src/bouncer.rs)

### Summary
`BuiltinInstanceLimits::induced_gas_costs` derives each builtin's per-instance proving-gas cost using integer floor division. Because the cost is floored, the effective per-block instance ceiling enforced by the bouncer is strictly higher than the configured `BuiltinInstanceLimits`, allowing a block to be accepted that contains more builtin instances than the prover can handle.

### Finding Description
`induced_gas_costs` computes each builtin's per-instance cost as:

```rust
let derive = |limit: NonZeroU64| -> u64 { proving_gas.0 / limit.get() };
```

This is `floor(proving_gas / limit)`. [1](#0-0) 

The bouncer then accumulates `instances × cost` and rejects a block only when the total exceeds `block_max_capacity.proving_gas`. [2](#0-1)  Because the cost is floored, the maximum number of instances the bouncer will admit is:

```
effective_limit = floor(proving_gas / floor(proving_gas / configured_limit))
```

which is ≥ `configured_limit` whenever `proving_gas` is not exactly divisible by `configured_limit`.

**Concrete example with production defaults** (`proving_gas = 5_000_000_000`, `range_check96 = 33_519_553`): [3](#0-2) 

| Step | Value |
|------|-------|
| `cost_per_instance = floor(5_000_000_000 / 33_519_553)` | **149** |
| `effective_limit = floor(5_000_000_000 / 149)` | **33_557_046** |
| Configured limit | **33_519_553** |
| **Excess instances admitted** | **+37,493** |

A block containing `33_557_046` range_check96 instances accumulates `33_557_046 × 149 = 4_999_999_854` proving gas, which is below the `5_000_000_000` cap, so the bouncer accepts it — yet the block exceeds the configured per-builtin ceiling by 37,493 instances.

The same arithmetic applies to every builtin whose limit does not evenly divide `proving_gas`. For `poseidon` (limit `600_000`): `floor(5e9 / 600_000) = 8_333`, effective limit `floor(5e9 / 8_333) = 600_024` — 24 extra instances. For `ecop` (limit `130_000`): `floor(5e9 / 130_000) = 38_461`, effective limit `floor(5e9 / 38_461) = 130_007` — 7 extra instances. [4](#0-3) 

There is no separate raw-instance-count check in the bouncer; the only guard is the `proving_gas` accumulator. [5](#0-4) 

### Impact Explanation
The `BuiltinInstanceLimits` represent the prover's per-block capacity for each Cairo primitive. When the bouncer admits a block that exceeds these limits, the prover (Stone/Stwo) receives a block it cannot prove, breaking the sequencer→prover pipeline.

The corrupted value is the `proving_gas` field accumulated in `Bouncer::accumulated_weights.bouncer_weights.proving_gas`: it is computed from an underestimated per-instance cost, so it underreports the true resource consumption, causing the block to be accepted when it should be rejected. [6](#0-5) 

This is the direct sequencer analog of the Eggs Finance rounding bug: just as floored token conversion left the protocol short of tokens on a round-trip, floored cost derivation leaves the bouncer's instance guard short of its configured ceiling.

**Impact: Critical — Incorrect bouncer/resource accounting with economic impact.** A block that cannot be proved is committed to consensus, stalling finality and wasting sequencer resources.

### Likelihood Explanation
Any unprivileged user can craft transactions that use many instances of a builtin (e.g., range_check96 via arithmetic-heavy Cairo contracts). No special privilege is required. The attacker only needs to push the block's range_check96 count above `33_519_553` but below `33_557_046` — a window of 37,493 instances that the bouncer accepts but the prover rejects.

### Recommendation
Replace floor division with ceiling division in `induced_gas_costs`:

```rust
// Current (floor — underestimates cost):
let derive = |limit: NonZeroU64| -> u64 { proving_gas.0 / limit.get() };

// Fixed (ceil — rounds in favor of the protocol):
let derive = |limit: NonZeroU64| -> u64 {
    proving_gas.0.div_ceil(limit.get())
};
```

Ceiling division ensures `instances × cost ≥ proving_gas` whenever `instances = limit`, so the bouncer will reject any block that would exceed the configured per-builtin ceiling. This mirrors the Eggs Finance recommendation: always round in favor of the protocol (overestimate cost rather than underestimate it). [7](#0-6) 

### Proof of Concept
With production defaults (`proving_gas = 5_000_000_000`, `range_check96 = 33_519_553`): [8](#0-7) 

1. `cost_per_instance = floor(5_000_000_000 / 33_519_553) = 149`
2. Attacker submits transactions totaling `33_557_046` range_check96 instances.
3. Bouncer accumulates `33_557_046 × 149 = 4_999_999_854 ≤ 5_000_000_000` → **block accepted**.
4. Block contains `33_557_046 > 33_519_553` range_check96 instances → **prover rejects block**.
5. Finality stalls; sequencer must discard the block and re-propose.

With ceiling division: `cost_per_instance = ceil(5_000_000_000 / 33_519_553) = 150`. Then `33_519_553 × 150 = 5_027_932_950 > 5_000_000_000`, so the bouncer correctly rejects any block exceeding the configured limit. [9](#0-8)

```rust
impl BuiltinInstanceLimits {
    /// Induces the per-instance proving-gas cost of each Cairo primitive from its per-block
    /// instance limit: `cost = floor(proving_gas / limit)`.
    pub fn induced_gas_costs(&self, proving_gas: GasAmount) -> BuiltinGasCosts {
        let derive = |limit: NonZeroU64| -> u64 { proving_gas.0 / limit.get() };
        BuiltinGasCosts {
            pedersen: derive(self.pedersen),
            range_check: derive(self.range_check),
            range_check96: derive(self.range_check96),
            poseidon: derive(self.poseidon),
            ecdsa: derive(self.ecdsa),
            ecop: derive(self.ecop),
            bitwise: derive(self.bitwise),
            keccak: derive(self.keccak),
            add_mod: derive(self.add_mod),
            mul_mod: derive(self.mul_mod),
            blake: derive(self.blake),
        }
    }
```

```rust
fn proving_gas_from_cairo_primitives_and_sierra_gas(
    sierra_gas: GasAmount,

    proving_builtin_gas_costs: &BuiltinGasCosts,
    sierra_builtin_gas_costs: &BuiltinGasCosts,
) -> GasAmount {
    let cairo_primitives_proving_gas =
        cairo_primitives_to_gas(cairo_primitives_counters, proving_builtin_gas_costs);

    steps_proving_gas.checked_add_panic_on_overflow(cairo_primitives_proving_gas)
}
```
