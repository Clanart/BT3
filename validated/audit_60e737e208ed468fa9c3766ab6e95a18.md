### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Contract Freezing â€” (`File: crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not verify that the caller-supplied `class_hash` corresponds to a previously declared class. The OS itself acknowledges this with an explicit TODO. An unprivileged contract can invoke `replace_class` with an arbitrary, undeclared class hash; the OS will accept the state update, permanently making the contract un-callable and freezing any funds it holds.

---

### Finding Description

**Vulnerable function â€” new syscall path**

`execute_replace_class` in `syscall_impls.cairo`:

```cairo
func execute_replace_class{...}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
    local state_entry: StateEntry*;
    %{ GetContractAddressStateEntry %}

    tempvar new_state_entry = new StateEntry(
        class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
    );

    dict_update{dict_ptr=contract_state_changes}(
        key=contract_address,
        prev_value=cast(state_entry, felt),
        new_value=cast(new_state_entry, felt),
    );
    ...
}
``` [1](#0-0) 

The function reads `class_hash` directly from the caller-controlled request and writes it into `contract_state_changes` with **no lookup into `contract_class_changes`** to confirm the hash was ever declared. The TODO comment at line 898 is the OS developers' own acknowledgement that this check is absent.

**Same issue in the deprecated syscall path**

`execute_replace_class` in `deprecated_execute_syscalls.cairo` (lines 307â€“329) is identical in structure: it reads `syscall_ptr.class_hash` and writes it to `contract_state_changes` with no declared-class check. [2](#0-1) 

**Contrast with the correct pattern used elsewhere**

`execute_declare_transaction` correctly enforces that a class can only be written once by requiring `prev_value=0`:

```cairo
// Note that prev_value=0 enforces that a class may be declared only once.

    key=[class_hash_ptr], prev_value=0, new_value=compiled_class_hash
);
``` [3](#0-2) 

`execute_replace_class` performs no analogous check against `contract_class_changes`.

### Impact Explanation

After a successful `replace_class` call with an undeclared hash:

1. The contract's `class_hash` field in `contract_state_changes` is set to a value that has no entry in the class trie.
2. Every subsequent call to the contract (invoke, L1 handler, etc.) will attempt to look up the class and fail â€” the contract is permanently bricked.
3. All ERC-20 balances, ERC-721 tokens, or any other assets stored in the contract's storage become permanently inaccessible.

This matches the allowed impact: **Critical â€” Permanent freezing of funds**.

---

### Likelihood Explanation

- `replace_class` is a standard, permissionless syscall available to every Sierra contract.
- Any contract owner can write a contract that calls `replace_class(undeclared_hash)`.
- No privileged role is required; the attacker only needs to deploy a contract and submit an invoke transaction.
- The OS will produce a valid proof for the resulting state, so the freeze is irreversible on-chain.

---

### Recommendation

Before writing the new class hash into `contract_state_changes`, verify that the hash exists in `contract_class_changes` (i.e., it was declared either in a prior block or in the current block before this syscall). A minimal fix:

```cairo
func execute_replace_class{
    range_check_ptr,
    syscall_ptr: felt*,
    contract_state_changes: DictAccess*,
    contract_class_changes: DictAccess*,   // <-- add this implicit arg
    revert_log: RevertLogEntry*,
}(contract_address: felt) {
    ...
    let class_hash = request.class_hash;

    // Verify the class has been declared.
    let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
    assert_not_zero(compiled_class_hash);   // reverts if undeclared

    // Proceed with state update.
    ...
}
```

Apply the same fix to the deprecated `execute_replace_class` in `deprecated_execute_syscalls.cairo`.

### Proof of Concept

1. **Declare** a legitimate contract class `C` and **deploy** a contract at address `A` with class `C`. Fund `A` with tokens.
2. Write a malicious contract `M` whose `__execute__` calls `replace_class(0xdeadbeef)` â€” a hash that has never been declared.
3. Submit an invoke transaction targeting `M`.
4. The OS executes `execute_replace_class`:
   - `class_hash = 0xdeadbeef`
   - No check against `contract_class_changes` is performed (line 898 TODO).
   - `dict_update` writes `class_hash=0xdeadbeef` into `contract_state_changes[A]`.
5. The OS produces a valid proof; the state root is updated.
6. Any subsequent call to `A` resolves class hash `0xdeadbeef`, finds no compiled class, and reverts permanently.
7. All funds in `A` are frozen with no recovery path. [4](#0-3) [2](#0-1)

        // Not enough gas to execute the syscall.
