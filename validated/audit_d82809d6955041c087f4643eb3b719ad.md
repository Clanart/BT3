### Title
`batch_insert` Bypasses Duplicate Key/Hash Validation Present in `insert`, Enabling Merkle Tree Root Corruption — (File: `crates/chia-datalayer/src/merkle/blob.rs`)

### Summary

`MerkleBlob::insert` guards against duplicate keys and duplicate leaf hashes before writing to the tree. `MerkleBlob::batch_insert` calls `insert` only for the first two items when the tree has ≤1 leaves; all remaining items are written directly to the blob via `insert_entry_to_blob` with **no duplicate checks at all**. An attacker-controlled batch containing repeated `KeyId` or `Hash` values silently corrupts the tree structure and produces a wrong Merkle root, enabling forged inclusion/exclusion proofs.

### Finding Description

`MerkleBlob::insert` enforces two invariants before any write:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 369-374
if self.block_status_cache.contains_key(key) {
    return Err(Error::KeyAlreadyPresent());
}
if self.block_status_cache.contains_leaf_hash(hash) {
    return Err(Error::HashAlreadyPresent());
}
``` [1](#0-0) 

`batch_insert` takes a different path. When the tree already has ≥2 leaves (`leaf_count > 1`), the early-exit branch that calls `insert` is skipped entirely. For every item in the batch, the code directly calls `insert_entry_to_blob` — bypassing both guards:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 578-603
if self.block_status_cache.leaf_count() <= 1 {
    for _ in 0..2 {
        let Some(((key, value), hash)) = keys_values_hashes.pop() else {
            return Ok(());
        };
        self.insert(key, value, &hash, InsertLocation::Auto {})?;  // ← validated
    }
}

for ((key, value), hash) in keys_values_hashes {          // ← remaining items
    let new_leaf_index = self.get_new_index();
    let new_block = Block { … Node::Leaf(LeafNode { hash, key, value, … }) };
    self.insert_entry_to_blob(new_leaf_index, &new_block)?; // ← NO duplicate check
    indexes.push(new_leaf_index);
}
``` [2](#0-1) 

Even when `leaf_count <= 1`, only the last two items (popped from the end of the vector) go through `insert`; all earlier items in the vector still reach the unchecked loop.

The Python binding `py_batch_insert` passes the caller-supplied lists directly into this function with no pre-filtering:

```rust
// crates/chia-datalayer/src/merkle/blob.rs  lines 1503-1518
pub fn py_batch_insert(
    &mut self,
    keys_values: Vec<(KeyId, ValueId)>,
    hashes: Vec<Hash>,
) -> PyResult<()> {
    …
    self.batch_insert(zip(keys_values, hashes).collect())?;
    Ok(())
}
``` [3](#0-2) 

The existing fuzz target and the `test_double_insert_fails` unit test both exercise only the single-item `insert` path, leaving the `batch_insert` bypass untested: [4](#0-3) [5](#0-4) 

### Impact Explanation

When a duplicate `KeyId` is written via `insert_entry_to_blob`, the blob gains a second leaf node for the same key. The subsequent subtree-building loop computes `internal_hash` over these duplicate leaves and grafts the resulting subtree into the existing tree: [6](#0-5) 

The resulting root hash is computed over a structurally invalid tree. `get_proof_of_inclusion` traverses the `block_status_cache` key-to-index mapping, which now points to whichever leaf was written last for that key; the earlier leaf is an orphan whose hash is still folded into the root. A proof generated from this corrupted root will not match the true committed state, allowing:

- **Forged inclusion**: a key that was deleted (or never legitimately inserted) can appear provably included because its orphaned leaf hash is still part of the root.
- **Root mismatch**: the on-chain DataLayer coin commitment diverges from the actual key-value state, letting untrusted input prove invalid state.

This matches the allowed High impact: *"DataLayer Merkle proof/blob/delta logic accepts forged inclusion/exclusion, corrupts tree roots, or lets untrusted input prove invalid state."* [7](#0-6) [8](#0-7) 

### Likelihood Explanation

`py_batch_insert` is exposed directly to Python callers via the `chia_rs` wheel. Any DataLayer delta-sync path that constructs the batch from remote or partially-trusted data can supply a repeated `KeyId` or `Hash`. The `check_integrity_on_drop` flag is `false` in production (`cfg!(test)` only), so the corruption is not caught at runtime: [9](#0-8) 

### Recommendation

Add the same duplicate-key and duplicate-hash guards at the top of `batch_insert` (or inside the unchecked loop) that `insert` already enforces. Alternatively, collect all incoming keys and hashes into a temporary set before writing any of them to the blob, and return `Error::KeyAlreadyPresent` / `Error::HashAlreadyPresent` on collision. Extend the fuzz target to exercise `batch_insert` with repeated keys.

### Proof of Concept

```rust
let mut blob = MerkleBlob::new(vec![]).unwrap();

// Pre-populate so leaf_count > 1, bypassing the insert() branch entirely
for i in 0i64..3 {
    blob.insert(KeyId(i), ValueId(i), &sha256_num(&i), InsertLocation::Auto {}).unwrap();
}

let dup_key = KeyId(99);
let hash_a = sha256_num(&100i64);
let hash_b = sha256_num(&101i64);

// batch_insert with the same KeyId twice — no error is returned
blob.batch_insert(vec![
    ((dup_key, ValueId(100)), hash_a),
    ((dup_key, ValueId(101)), hash_b),   // duplicate key, different hash
]).unwrap();   // succeeds silently

blob.calculate_lazy_hashes().unwrap();

// Root is now computed over an orphaned duplicate leaf — proofs are wrong
let proof = blob.get_proof_of_inclusion(dup_key).unwrap();
// proof.valid() may return true, but the root does not reflect the true state
```

    pub fn get_proof_of_inclusion(
        &self,
        key: KeyId,
    ) -> Result<proof_of_inclusion::ProofOfInclusion, Error> {
        let mut index = *self
            .block_status_cache
            .get_index_by_key(key)
            .ok_or(Error::UnknownKey(key))?;

        let node = self
            .get_node(index)?
            .expect_leaf("key to index mapping should only have leaves");

        let parents = self.get_lineage_blocks_with_indexes(index)?;
        let mut layers: Vec<proof_of_inclusion::ProofOfInclusionLayer> = Vec::new();
        let mut parents_iter = parents.iter();
        // first in the lineage is the index itself, second is the first parent
        parents_iter.next();
        for (next_index, block) in parents_iter {
            if block.metadata.dirty {
                return Err(Error::Dirty(*next_index));
            }
            let parent = block
                .node
                .expect_internal("all nodes after the first should be internal");
            let sibling_index = parent.sibling_index(index)?;
            let sibling_block = self.get_block(sibling_index)?;
            let sibling = sibling_block.node;
            let layer = proof_of_inclusion::ProofOfInclusionLayer {
                other_hash_side: parent.get_sibling_side(index)?,
                other_hash: sibling.hash(),
                combined_hash: parent.hash,
            };
            layers.push(layer);
            index = *next_index;
        }

        Ok(proof_of_inclusion::ProofOfInclusion {
            node_hash: node.hash,
            layers,
        })
    }
