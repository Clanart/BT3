### Title
Missing Declared-Class Validation in `execute_replace_class` Allows Permanent Fund Freezing - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall implementation in the StarkNet OS performs gas and state-entry checks but omits the critical validation that the supplied `class_hash` corresponds to a previously declared class. An attacker can exploit this to permanently freeze funds held by any contract by replacing its class with an undeclared hash, after which no future call to that contract can be proven.

---

### Finding Description

In `syscall_impls.cairo`, `execute_replace_class` processes the `replace_class` syscall as follows:

1. Reduces gas (line 888–893).
2. Reads `request.class_hash` (line 896).
3. **Skips validation that `class_hash` is declared** — acknowledged by a TODO comment at line 898.
4. Writes the new `StateEntry` with the arbitrary `class_hash` into `contract_state_changes` (lines 902–910). [1](#0-0) 

The explicit TODO at line 898 reads:
```
// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
``` [2](#0-1) 

After this state update is committed, any subsequent call to the affected contract reaches `execute_entry_point`, which performs:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // the undeclared hash
);
// compiled_class_hash == 0 (default for uninitialized dict entry)
let (compiled_class_fact: CompiledClassFact*) = find_element(
    ...
    key=compiled_class_hash,           // 0 — no such fact exists
);
``` [3](#0-2) 

`find_element` is not a soft search — it asserts the element exists. Because no compiled class fact with hash `0` (or the arbitrary undeclared hash) is present in the bundle, the proof for any future transaction touching that contract is unprovable. The contract becomes permanently inaccessible.

### Impact Explanation

**Critical — Permanent freezing of funds.**

Once a contract's class hash is replaced with an undeclared value and the transaction is finalized, the contract's state entry in the global Merkle tree permanently records the invalid class hash. No future transaction that invokes this contract can produce a valid proof. Any ERC-20 tokens, ETH, or other assets held in the contract's storage are irrecoverably locked.

---

### Likelihood Explanation

**Medium.**

The attack requires a contract that exposes `replace_class` with attacker-influenced input. Two realistic paths exist:

1. **Attacker-deployed honeypot**: An attacker deploys a contract that appears legitimate, attracts user deposits, then calls `replace_class` with an undeclared hash as the final operation of a transaction (so no subsequent call in the same transaction triggers the unprovable `find_element`). The transaction is proven successfully, the state is committed, and the funds are frozen.

2. **Legitimate contract with insufficient input validation**: A contract implements an upgrade mechanism that calls `replace_class` with a caller-supplied hash, trusting the OS to reject invalid hashes. Because the OS does not reject them, an attacker passes an undeclared hash and freezes the contract.

The `replace_class` syscall is a standard, publicly documented StarkNet feature reachable by any contract deployer or transaction sender.

---

### Recommendation

Inside `execute_replace_class`, before writing the new `StateEntry`, verify that `class_hash` has a corresponding entry in `contract_class_changes` (i.e., it has been declared in the current or a prior block). Concretely, perform a `dict_read` on `contract_class_changes` for `class_hash` and assert the returned compiled class hash is non-zero, mirroring the lookup already performed in `execute_entry_point`:

```cairo
// Verify the class has been declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);
```

This resolves the acknowledged TODO and closes the gap between what `replace_class` permits and what `execute_entry_point` requires.

---

### Proof of Concept

1. Attacker deploys contract `C` with class hash `H_declared` (a valid declared class).
2. `C` exposes a function `freeze()` that calls `replace_class(H_undeclared)` where `H_undeclared` is any felt not present in the declared class set.
3. Users deposit funds into `C`.
4. Attacker calls `freeze()`. The OS processes `execute_replace_class`:
   - Gas check passes.
   - No declared-class check is performed (the TODO gap).
   - `contract_state_changes[C].class_hash` is set to `H_undeclared`.
   - Transaction is proven and committed (no further call to `C` in this tx, so `find_element` is never reached).
5. In any subsequent block, any transaction calling `C` reaches `execute_entry_point`:
   - `dict_read(contract_class_changes, H_undeclared)` → `0`.
   - `find_element(..., key=0)` → assertion failure; proof is unprovable.
6. All funds in `C` are permanently frozen. [5](#0-4)

        // Not enough gas to execute the syscall.
        return ();
