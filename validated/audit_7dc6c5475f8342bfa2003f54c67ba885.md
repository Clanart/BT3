### Title
Unbounded Predecessor Replay Loop in `TraceTx` Enables Unbounded CPU/Memory Exhaustion via Single Unprivileged gRPC/RPC Call - (File: x/evm/keeper/grpc_query.go)

### Summary
The `TraceTx` gRPC handler iterates over a caller-supplied `req.Predecessors` slice with no count limit and no timeout, executing each predecessor as a full EVM state transition via `ApplyMessageWithConfig`. A single unprivileged `debug_traceTransaction` call carrying thousands of complex predecessor transactions can exhaust node CPU and memory indefinitely. In the standard Ethermint deployment where the JSON-RPC server and the CometBFT validator share the same process, this starves the consensus engine of CPU time and can cause the validator to miss blocks, halting the chain.

### Finding Description

In `TraceTx`, the `execTrace` callback iterates over `req.Predecessors` without any bound: [1](#0-0) 

Each iteration calls `k.ApplyMessageWithConfig(ctx, msg, cfg, true)` — a full EVM execution including intrinsic-gas computation, access-list preparation, EVM call/create, gas refund, and StateDB commit. There is no maximum predecessor count enforced anywhere in the call chain: [2](#0-1) 

The timeout mechanism that exists in `prepareTrace` (default 5 s, user-overridable) is installed **after** the predecessor loop returns and only governs the single traced transaction via `tracer.Stop`: [3](#0-2) 

It has no effect on the predecessor replay. The `TraceTx` proto definition carries `Predecessors` as a plain repeated field with no `validate_basic` size cap: [4](#0-3) 

By contrast, the `SimulateV1` path enforces `MaxSimulateBlocks = 256` and a cross-block `GasBudget`: [5](#0-4) [6](#0-5) 

No equivalent guard exists for `TraceTx` predecessors.

### Impact Explanation

Each predecessor invokes `ApplyMessageWithConfig`, which allocates a fresh `StateDB`, branches the multistore, runs the EVM, and commits dirty state objects: [7](#0-6) 

With N predecessors each deploying a large contract or running a gas-heavy loop, the node spends O(N × gas_per_tx) CPU time and O(N) memory in a single goroutine before the traced transaction even begins. Because Ethermint runs the JSON-RPC server and the CometBFT consensus engine in the same OS process, a goroutine monopolising CPU for minutes starves the consensus reactor, causing the validator to miss `propose`/`prevote`/`precommit` timeouts and halting block production. This matches the **Critical** allowed impact: "Valid unprivileged RPC submission… can halt the chain."

### Likelihood Explanation

`debug_traceTransaction` is a standard Ethereum JSON-RPC method exposed by every Ethermint node with the debug namespace enabled. No authentication, stake, or special privilege is required. A single HTTP POST with a crafted `predecessors` array of, say, 10 000 transactions is sufficient. The attack is trivially scriptable and requires no on-chain activity.

### Recommendation

1. **Enforce a hard cap on predecessor count** in `TraceTx` (e.g., `MaxTracePredecessors = 1024`) and return `codes.InvalidArgument` if exceeded.
2. **Apply a context deadline to the entire predecessor loop**, not just the traced transaction. Pass `deadlineCtx` into the loop and check `ctx.Err()` after each iteration.
3. **Mirror the `GasBudget` pattern** from `SimulateV1`: track cumulative gas consumed across all predecessors and abort when a configurable cap is reached.
4. Apply the same fix to `TraceBlock`, which has an analogous unbounded loop over `req.Txs` with no overall timeout.

### Proof of Concept

```python
import json, requests

# Build 5000 predecessor "transactions" (any valid MsgEthereumTx proto bytes)
predecessors = [build_heavy_contract_deploy_tx() for _ in range(5000)]

payload = {
  "jsonrpc": "2.0",
  "method": "debug_traceTransaction",
  "params": [
    "0xTARGET_TX_HASH",
    {"predecessors": predecessors}   # no server-side limit
  ],
  "id": 1
}
# Single request; node spends hours replaying predecessors,
# starving the CometBFT consensus reactor.
requests.post("http://node:8545", json=payload)
```

The node's CPU pegs at 100 % for the duration of the predecessor replay. With a co-located validator, `propose` timeouts fire, the validator is slashed for downtime, and block production halts until the request completes or the process is restarted.

func (k Keeper) TraceTx(c context.Context, req *types.QueryTraceTxRequest) (*types.QueryTraceTxResponse, error) {
	var baseFee *big.Int
	if req == nil || req.Msg == nil {
		return nil, status.Error(codes.InvalidArgument, "request and message cannot be empty")
	}
	if req.BaseFee != nil && !req.BaseFee.IsNil() {
		baseFee = req.BaseFee.BigInt()
	}
	resultData, err := execTrace(
		c,
		req,
		k,
		baseFee,
		func(ctx sdk.Context, cfg *EVMConfig, traceConfig *types.TraceConfig) (*core.Message, error) {
			signer := ethtypes.MakeSigner(cfg.ChainConfig, big.NewInt(ctx.BlockHeight()), uint64(ctx.BlockTime().Unix())) //#nosec G115
			tracer, err := newTacer(&logger.Config{}, cfg.ChainConfig, cfg.TxConfig, traceConfig)
			if err != nil {
				return nil, status.Error(codes.Internal, err.Error())
			}
			cfg.Tracer = tracer.Hooks
			cfg.DebugTrace = true
			cfg.TraceReplay = traceConfig.GetTraceReplay()
			for i, tx := range req.Predecessors {
				ethTx := tx.AsTransaction()
				msg, err := core.TransactionToMessage(ethTx, signer, cfg.BaseFee)
				if err != nil {
					k.Logger(ctx).Debug("trace: skipping predecessor, failed to convert tx to message",
						"index", i, "hash", ethTx.Hash().Hex(), "err", err.Error())
					continue
				}
				cfg.TxConfig.TxHash = ethTx.Hash()
				cfg.TxConfig.TxIndex, err = ethermint.SafeUint(i)
				if err != nil {
					k.Logger(ctx).Debug("trace: skipping predecessor, invalid tx index",
						"index", i, "hash", ethTx.Hash().Hex(), "err", err.Error())
					continue
				}
				rsp, err := k.ApplyMessageWithConfig(ctx, msg, cfg, true)
				if err != nil {
					k.Logger(ctx).Error("trace: predecessor replay failed, trace state may be incomplete",
						"index", i, "hash", ethTx.Hash().Hex(), "err", err.Error())
					continue
				}
				cfg.TxConfig.LogIndex += uint(len(rsp.Logs))
			}

			tx := req.Msg.AsTransaction()
			cfg.TxConfig.TxHash = tx.Hash()
			if len(req.Predecessors) > 0 {
				cfg.TxConfig.TxIndex++
			}

			return core.TransactionToMessage(tx, signer, cfg.BaseFee)
		},
	)
	if err != nil {
		return nil, err
	}

	return &types.QueryTraceTxResponse{
		Data: resultData,
	}, nil
}
