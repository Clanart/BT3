### Title
Silent consumption of excess ETH via unchecked `extensionValue` forwarded to Wormhole in `initTransfer` (native ETH path) — (`evm/src/omni-bridge/contracts/OmniBridgeWormhole.sol`)

### Summary
An unprivileged user calling `initTransfer` with `tokenAddress=address(0)` and `msg.value > amount + nativeFee + _wormhole.messageFee()` will have the excess ETH permanently forwarded to and retained by the Wormhole core contract, with no recovery path.

### Finding Description
In `OmniBridge.initTransfer`, when `tokenAddress == address(0)`, `extensionValue` is computed as `msg.value - amount - nativeFee` with no upper-bound enforcement. This value is passed to `OmniBridgeWormhole.initTransferExtension`, which calls `_wormhole.publishMessage{value: value}(...)`. The Wormhole core contract only requires `msg.value >= messageFee()` and silently absorbs any surplus. The override removes the base contract's `value != 0` revert guard without substituting any equivalent check.

### Impact Explanation
Any ETH sent beyond `amount + nativeFee + messageFee()` is permanently lost to the Wormhole contract. There is no refund mechanism, no recovery path, and no event indicating the discrepancy. This constitutes irrecoverable loss of user funds matching the **Critical** impact category: permanent, irrecoverable lock of user-supplied ETH.

### Likelihood Explanation
The path is reachable by any unprivileged user with a single public call. No special role, leaked key, or external compromise is required. A user who miscalculates fees or a front-end that passes a rounded `msg.value` can trigger this silently. Likelihood is **Medium** (requires a user to overpay, but the contract provides no protection against it).

### Recommendation
In `OmniBridgeWormhole.initTransferExtension`, enforce that the forwarded value exactly equals the required fee:

```solidity
uint256 fee = _wormhole.messageFee();
require(value == fee, "InvalidValue");
_wormhole.publishMessage{value: fee}(...);
```

Alternatively, refund `value - fee` to the original sender after the `publishMessage` call.

### Proof of Concept

    function initTransferExtension(
        address sender,
        address tokenAddress,
        uint64 originNonce,
        uint128 amount,
        uint128 fee,
        uint128 nativeFee,
        string calldata recipient,
        string calldata message,
        uint256 value
    ) internal override {
        bytes memory payload = bytes.concat(
            bytes1(uint8(MessageType.InitTransfer)),
            bytes1(omniBridgeChainId),
            Borsh.encodeAddress(sender),
            bytes1(omniBridgeChainId),
            Borsh.encodeAddress(tokenAddress),
            Borsh.encodeUint64(originNonce),
            Borsh.encodeUint128(amount),
            Borsh.encodeUint128(fee),
            Borsh.encodeUint128(nativeFee),
            Borsh.encodeString(recipient),
            Borsh.encodeString(message)
        );
        // slither-disable-next-line reentrancy-eth
        _wormhole.publishMessage{value: value}(
            wormholeNonce,
            payload,
            _consistencyLevel
        );

        wormholeNonce++;
    }
