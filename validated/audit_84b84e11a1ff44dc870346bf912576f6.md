### Title
Wrong Validator Function for `MinGasMultiplier` in `ParamSetPairs` Bypasses Upper-Bound Check, Allowing Fee Accounting Manipulation - (File: `x/feemarket/types/params.go`)

### Summary

In `x/feemarket/types/params.go`, the `ParamSetPairs()` function registers `MinGasMultiplier` with the validator `validateMinGasPrice` instead of `validateMinGasMultiplier`. The `validateMinGasPrice` function only checks that the value is non-negative and non-nil — it does **not** enforce the upper bound of `1`. This means a governance proposal (or legacy subspace parameter update) can set `MinGasMultiplier > 1`, bypassing the intended cap, which directly corrupts the `gasWanted` accounting used in every block's EIP-1559 base fee calculation.

### Finding Description

In `ParamSetPairs()`, the last entry incorrectly uses `validateMinGasPrice` as the validator for `MinGasMultiplier`:

```go
// x/feemarket/types/params.go line 64
paramtypes.NewParamSetPair(ParamStoreKeyMinGasMultiplier, &p.MinGasMultiplier, validateMinGasPrice),
``` [1](#0-0) 

The correct validator is `validateMinGasMultiplier`, which enforces `0 ≤ value ≤ 1`:

```go
// x/feemarket/types/params.go lines 215-233
func validateMinGasMultiplier(i interface{}) error {
    ...
    if v.GT(sdkmath.LegacyOneDec()) {
        return fmt.Errorf("value cannot be greater than 1: %s", v)
    }
    return nil
}
``` [2](#0-1) 

The `validateMinGasPrice` function used instead only checks non-negative and non-nil: [3](#0-2) 

The same bug is present in the migration types: [4](#0-3) 

Note that `Params.Validate()` and `SetParams()` do call `validateMinGasMultiplier` correctly, so the full-params update path is protected. However, the legacy Cosmos SDK `x/params` subspace path — used during chain upgrades and legacy parameter change proposals — calls the per-field validator registered in `ParamSetPairs`, which is the wrong one. This allows `MinGasMultiplier > 1` to be stored via the legacy subspace.

### Impact Explanation

`MinGasMultiplier` is used in `EndBlock` to compute the effective `gasWanted` stored for the next block's EIP-1559 base fee calculation:

```go
// x/feemarket/keeper/abci.go lines 72-74
minGasMultiplier := k.GetParams(ctx).MinGasMultiplier
limitedGasWanted := sdkmath.LegacyNewDec(gw).Mul(minGasMultiplier)
gasWanted = sdkmath.LegacyMaxDec(limitedGasWanted, sdkmath.LegacyNewDec(gasUsed)).TruncateInt().Uint64()
``` [5](#0-4) 

If `MinGasMultiplier` is set to a value greater than 1 (e.g., 100), `limitedGasWanted` becomes a multiple of the actual gas limit, which can exceed `uint64` bounds or produce a `gasWanted` value far above the block gas limit. This feeds into `CalculateBaseFee`: [6](#0-5) 

An inflated `parentGasUsed` (from the corrupted `gasWanted`) causes the base fee to spike artificially on every block, making all transactions fail the `gasFeeCap >= baseFee` check in `VerifyFee` and `CheckEthGasConsume`. This effectively halts all EVM transaction processing — a chain-halting condition — because no user transaction can meet the inflated base fee requirement. [7](#0-6) 

### Likelihood Explanation

The legacy `x/params` subspace path is exercised during chain upgrades (migration `v4` reads from the subspace) and by any legacy parameter change governance proposal that updates `MinGasMultiplier` individually. The migration code at `x/feemarket/migrations/v4/migrate.go` calls `legacySubspace.GetParamSetIfExists` which uses the `ParamSetPairs` validators, and the same wrong validator is present in the migration types as well. [8](#0-7) 

A governance participant submitting a parameter update proposal for `MinGasMultiplier` via the legacy subspace path can set it to any value ≥ 0, including values > 1, without triggering a validation error.

### Recommendation

Change line 64 of `x/feemarket/types/params.go` to use `validateMinGasMultiplier` instead of `validateMinGasPrice`:

```go
paramtypes.NewParamSetPair(ParamStoreKeyMinGasMultiplier, &p.MinGasMultiplier, validateMinGasMultiplier),
```

Apply the same fix to `x/feemarket/migrations/v4/types/params.go` line 50.

### Proof of Concept

1. Submit a legacy governance parameter change proposal setting `MinGasMultiplier = 10` (or any value > 1) via the `x/params` subspace path.
2. The per-field validator `validateMinGasPrice` is called — it only checks non-negative, so the value passes.
3. On the next `EndBlock`, `limitedGasWanted = gasWanted * 10`, producing a value 10× the actual gas used.
4. `CalculateBaseFee` in `BeginBlock` sees `parentGasUsed >> parentGasTarget`, causing the base fee to increase by the maximum delta every block.
5. Within a few blocks, the base fee exceeds any practical `gasFeeCap` a user can set, causing all EVM transactions to be rejected by `VerifyFee` with `ErrInsufficientFee`.
6. The chain continues producing blocks but no EVM transaction can be included — effective EVM halt.

	parentGasUsed := k.GetBlockGasWanted(ctx)

	// NOTE: a MaxGas equal to -1 means that block gas is unlimited
	if consParams.Block == nil || consParams.Block.MaxGas <= -1 {
		panic(fmt.Sprintf("get invalid consensus params: %s", consParams))
	}
	gasLimit := big.NewInt(consParams.Block.MaxGas)
	// CONTRACT: ElasticityMultiplier cannot be 0 as it's checked in the params
	// validation
	parentGasTargetBig := new(big.Int).Div(gasLimit, new(big.Int).SetUint64(uint64(params.ElasticityMultiplier)))
	if !parentGasTargetBig.IsUint64() {
		return nil
	}

	parentGasTarget := parentGasTargetBig.Uint64()
	baseFeeChangeDenominator := new(big.Int).SetUint64(uint64(params.BaseFeeChangeDenominator))

	// If the parent gasUsed is the same as the target, the baseFee remains
	// unchanged.
	if parentGasUsed == parentGasTarget {
		return new(big.Int).Set(parentBaseFee)
	}

	if parentGasUsed > parentGasTarget {
		// If the parent block used more gas than its target, the baseFee should
		// increase.
		gasUsedDelta := new(big.Int).SetUint64(parentGasUsed - parentGasTarget)
		x := new(big.Int).Mul(parentBaseFee, gasUsedDelta)
		y := x.Div(x, parentGasTargetBig)
		baseFeeDelta := ethermint.BigMax(
			x.Div(y, baseFeeChangeDenominator),
			common.Big1,
		)

		return x.Add(parentBaseFee, baseFeeDelta)
	}

	// Otherwise if the parent block used less gas than its target, the baseFee
	// should decrease.
	gasUsedDelta := new(big.Int).SetUint64(parentGasTarget - parentGasUsed)
	x := new(big.Int).Mul(parentBaseFee, gasUsedDelta)
	y := x.Div(x, parentGasTargetBig)
	baseFeeDelta := x.Div(y, baseFeeChangeDenominator)

	// Set global min gas price as lower bound of the base fee, transactions below
	// the min gas price don't even reach the mempool.
	minGasPrice := params.MinGasPrice.TruncateInt().BigInt()
	return ethermint.BigMax(x.Sub(parentBaseFee, baseFeeDelta), minGasPrice)
