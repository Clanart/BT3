### Title
Missing Class Hash Validation in `execute_replace_class` Enables Permanent Freezing of Funds - (File: `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/syscall_impls.cairo`)

---

### Summary

The `execute_replace_class` syscall handler in the StarkNet OS does not validate that the new class hash supplied by a contract corresponds to a previously declared contract class. This is explicitly acknowledged by a TODO comment in the code. As a result, any contract can replace its own class hash with an arbitrary, undeclared value, permanently bricking itself and freezing any funds it holds.

---

### Finding Description

In `syscall_impls.cairo`, the `execute_replace_class` function processes the `replace_class` syscall. After deducting gas, it reads the requested new class hash directly from the syscall request and writes it into the contract state — without any check that the hash corresponds to a declared class:

```cairo
// execute_replace_class (syscall_impls.cairo ~line 878)
let class_hash = request.class_hash;

// TODO(Yoni, 1/1/2026): Check that there is a declared contract class with the given hash.
local state_entry: StateEntry*;
%{ GetContractAddressStateEntry %}

tempvar new_state_entry = new StateEntry(
    class_hash=class_hash, storage_ptr=state_entry.storage_ptr, nonce=state_entry.nonce
);

dict_update{dict_ptr=contract_state_changes}(
    key=contract_address,
    prev_value=cast(state_entry, felt),
    new_value=cast(new_state_entry, felt),
);
``` [1](#0-0) 

The missing validation is the direct analog of the M-3 bug: just as `notDelegated` silently broke the upgrade authorization path in the Cork Protocol, the absent class-hash check here silently breaks the class-replacement safety invariant — the OS accepts a state transition it should reject.

Once the contract's `class_hash` field in `contract_state_changes` is set to an undeclared value, any subsequent call to that contract reaches `execute_entry_point`, which does:

```cairo
let (compiled_class_hash: felt) = dict_read{dict_ptr=contract_class_changes}(
    key=execution_context.class_hash   // now an undeclared hash → returns 0
);
...
let (compiled_class_fact: CompiledClassFact*) = find_element(
    array_ptr=compiled_class_facts_bundle.compiled_class_facts,
    ...
    key=compiled_class_hash,           // 0 → not found → assertion failure
);
``` [2](#0-1) 

`dict_read` on an uninitialized key returns the default value `0`. `find_element` with key `0` will not find a matching `CompiledClassFact`, causing an irrecoverable Cairo assertion failure for every future call to that contract. The contract — and all funds it holds — are permanently inaccessible.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

A malicious contract can:
1. Accept user deposits.
2. Call `replace_class` with an arbitrary undeclared hash (e.g., `1`).
3. The OS writes the invalid class hash into the contract's `StateEntry` without any check.
4. All future calls to the contract fail at the proof level (no compiled class found).
5. All deposited funds are permanently frozen with no recovery path.

This satisfies the "Critical. Permanent freezing of funds" impact category.

---

### Likelihood Explanation

The `replace_class` syscall is available to every deployed contract — no privileged role is required. The attack requires only:
- Deploying a contract that accepts funds and then calls `replace_class` with an invalid hash.
- No external dependency, no operator collusion, no key leak.

The OS is the necessary vulnerable step: it is the only place where the class hash could be validated against the set of declared classes, and it explicitly skips that check.

---

### Recommendation

Remove the TODO and implement the missing validation. Before writing the new class hash into `contract_state_changes`, verify that the hash exists in `contract_class_changes` (i.e., it was previously declared via a `declare` transaction). The check should mirror the lookup already performed in `execute_entry_point`:

```cairo
// Verify the new class hash is declared.
let (compiled_class_hash) = dict_read{dict_ptr=contract_class_changes}(key=class_hash);
assert_not_zero(compiled_class_hash);  // 0 means undeclared
```

This ensures `replace_class` can only target classes that the OS can actually execute.

---

### Proof of Concept

1. Deploy contract `Vault` that:
   - Exposes a `deposit` entry point that accepts ETH/STRK.
   - Exposes a `rug` entry point that calls `replace_class(class_hash=1)`.
2. Users call `deposit`, locking funds in `Vault`.
3. Attacker calls `rug`.
4. `execute_replace_class` in the OS writes `class_hash=1` into `Vault`'s `StateEntry` with no validation.
5. Any subsequent call to `Vault` reaches `execute_entry_point`:
   - `dict_read(contract_class_changes, key=1)` → returns `0` (undeclared).
   - `find_element(..., key=0)` → no matching `CompiledClassFact` → assertion failure.
6. All calls to `Vault` permanently fail; deposited funds are frozen forever. [3](#0-2) [4](#0-3)
